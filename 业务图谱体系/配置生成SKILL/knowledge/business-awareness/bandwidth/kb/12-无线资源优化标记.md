### K1201: 无线资源优化标记机制原理 [原理]

通过 **DSCP / FPI 标记**或**终端 OS 差异化**，影响无线侧调度优先级（非直接用户面限速），**间接优化带宽体验**。

**核心机制链**（`01-business-graph.md` §2.7）：
```
SA 识别业务
  → DSCP / FPI 标记 / OSTYPE 差异化
  → RAN 调度队列优化（非 UPF 限速）
  → 间接提升业务带宽体验
```

> 与 BWM/CAR/Shaping 关键差异：无线优化标记**不在用户面执行限速**，而是通过标记影响基站调度。

### K1202: DSCP/FPI 标记 [配置]

> Task T-601。特性 GWFD-020359（IM 业务 DSCP 标记）、GWFD-110331（FPI 标记）。

按业务类型映射 DSCP 或 FPI 值，通过 **GTP-U 扩展头**影响无线调度：

| 特性 | 标记类型 | 应用场景 |
|------|---------|---------|
| GWFD-020359 | DSCP | IM 类业务（微信等）无线资源管控 |
| GWFD-110331 | FPI | 业务流标识无线资源优化 |

配置通过 ADD RULE（含 DSCP/FPI 动作参数）实现，具体参数依特性版本而定。

### K1203: 终端 OS 差异化（SET APNOSLELBWMSW） [配置]

> Task T-602。特性 GWFD-110301。

按终端 OS 类型（Android/iOS/Windows）匹配不同 BWM 策略，适配不同编码格式：

```mml
SET APNOSLELBWMSW:APNNAME="{apn}",OSTYPEBWMSW=ENABLE;
```

- `APNNAME`：APN/DNN 名
- `OSTYPEBWMSW`：`ENABLE` / `DISABLE`（终端 OS BWM 差异化开关）

使能后，UDG 按终端 OS 类型差异化码率控制策略。

### K1204: 视频上下行解耦 [配置]

> 特性 GWFD-110302。Task T-301（QoS 属性）的增强应用。

视频业务上下行解耦的信令控制，依赖 QoS 保证（GWFD-020358/WSFD-109107）作为基础：
- 上行（视频上传）与下行（视频播放）使用不同 QoS 参数
- 通过 QOSPROP 分别配置 MBRUL/MBRDL/GBRUL/GBRDL
- 优化视频业务的上下行带宽分配

### K1206: 异常流量检测（辅助能力） [配置]

> 特性 GWFD-020305。作为辅助能力，通过 Feature 依赖和 BusinessRule 表达，不单独建方案。

检测终端异常下行流量，阻断恶意终端：
- 配合 BWM Gate=Closed 或 CAR REDACT=DISCARD 实现阻断
- 作为 BWM 限速的补充，用于恶意流量识别后的阻断动作

### K1208: 无线优化标记与 BWM 的边界 [隐性规则]

| 维度 | BWM 限速 | 无线优化标记 |
|------|---------|-------------|
| 作用层 | 用户面（UPF） | 无线侧（RAN 调度） |
| 机制 | 令牌桶丢弃/缓冲 | DSCP/FPI 影响队列优先级 |
| 效果 | 直接带宽上限 | 间接体验优化 |
| 配置入口 | BWMCONTROLLER | RULE 的标记动作 / SET APNOSLELBWMSW |

两者可叠加：BWM 控制用户面上限，无线标记优化 RAN 调度顺序。
