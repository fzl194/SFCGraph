### K1301: RULE PRIORITY 通用规则（引用） [原理]

> **详见 `../业务感知域规则.md §3`**。本节仅复述要点，完整规则以域规则为准。

- **数字越小，优先级越高**（固定规则，不可更改）
- "第 N 优先" = 第 N 高优先级 = PRIORITY 数值第 N 小
- 分析前必须用 Grep 提取现网**所有 RULE 的 PRIORITY**（含非本场景 RULE），按数值排序后定位
- 跨场景共享：计费、带宽控制、访问限制三场景共用此规则

---

### K1302: 带宽场景的两套独立优先级 [原理]

带宽控制场景有**两套独立的优先级体系**：

| 优先级字段 | 所属对象 | 作用 |
|-----------|---------|------|
| `RULE.PRIORITY` | PCC 规则（POLICYTYPE=BWM/PCC/QOS/ADC） | PCC 规则间裁决 |
| `BWMRULE.BWMRULEPRI` | BWM 规则（BWM 独有） | BWM 规则间裁决 |

两套优先级独立计算，互不影响。Phase 3 参数收集时必须**分别提取**两套优先级分布。

---

### K1303: FUP 降速优先级覆盖（BR-BW-02） [隐性规则]

> BusinessRule BR-BW-02。

FUP 降速规则的特殊优先级要求：
- **PRIORITY 最高**（数值最小，如 1）— 确保覆盖原保障规则
- **FlowFilter 完全覆盖**原保障规则（端口号范围一致）— 确保所有相关流量都降速

违反：部分流量降速、部分不受影响，降速不彻底。

---

### K1304: BWM 与 PCC 规则独立匹配（BR-BW-03） [原理]

> BusinessRule BR-BW-03。来源：`01-business-graph.md` §4。

BWM 规则与 PCC 规则在 SA 七步流程中**独立匹配、互不干扰、可叠加执行**：
- BWM 管有线侧（用户面限速整形）
- PCC 管无线侧（QoS 调度）
- 两者命中后结果叠加下发转发面

误以为冲突而删除其一 → 某一段带宽失控。

---

### K1305: POLICYTYPE 跨类型 RULENAME 不冲突（CR-BW-01） [隐性规则]

> CommandRule CR-BW-01。

| POLICYTYPE 组合 | 是否可同名 |
|----------------|-----------|
| PCC vs QOS | **不可同名** |
| BWM vs CHARGING | 可同名 |
| BWM vs PCC | 可同名（不同 POLICYTYPE） |

PCC 类型 RULENAME 与 QOS 类型 RULENAME 不能相同；同名规则可存在于不同 POLICYTYPE。

---

### K1306: 预定义规则三网元一致性（BR-BW-01 / CR-BW-05 / TR-BW-06） [隐性规则]

> 三条规则协同。

凡动态 PCC 场景（PCRF/PCF 下发预定义规则名）的特性：
- `RULENAME` 在 PCF / SMF / UPF 三处**完全一致**
- 参数（VOLUMETHRESHOLD、MBR 等）三处一致
- `FLOWFILTER` 三处一致
- ADC 场景 `FLOWFILTERNAME` / appid 三处一致
- 定向业务必须用预定义规则（PCF 无 L7 识别能力）

违反：跨网元规则名不一致 → 策略无法匹配，带宽控制失效。

### K1307: BWM 规则匹配流程 [原理]

> 来源：`topic-knowledge/Batch-18-UDG业务感知专题-规则匹配流程与套餐配置.md`。

BWM 规则匹配在 SA 七步流程中独立执行：
```
报文 → L2/L3/L4 解析 → SA L7 识别（SVC/APP）
  → BWMRULE 匹配（按 BWMRULETYPE + BWMSERVICENAME）
    → 命中 → 执行 UPBWMCTRLNAME1/DNBWMCTRLNAME1 的 CAR/Shaping
    → 未命中 → 走默认（无 BWM 控制）
```

---

### K1308: PCC 规则匹配流程 [原理]

PCC 规则（RULE）匹配独立于 BWM：
```
报文 → FLOWFILTER 匹配（L3/L4/L7）
  → RULE 匹配（按 PRIORITY 裁决，POLICYTYPE 分支）
    → POLICYTYPE=BWM → BWM 控制（与 BWMRULE 体系协同）
    → POLICYTYPE=PCC → FUP 监控（URR 上报）
    → POLICYTYPE=QOS → GBR 保证（QOSPROP）
    → POLICYTYPE=ADC → 应用检测（ADCPARA 事件）
```

> 两套匹配结果叠加下发转发面，体现 BR-BW-03 独立匹配原则。
