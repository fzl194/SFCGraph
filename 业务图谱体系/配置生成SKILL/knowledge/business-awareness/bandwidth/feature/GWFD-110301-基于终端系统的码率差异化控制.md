# GWFD-110301 基于终端系统的码率差异化控制 - 特性知识文档

> **场景归属**: 业务感知 → 带宽控制子场景（辅助特性：视频码率差异化）
> **产品**: UDG (UPF网元)
> **场景定位**: 识别终端系统 → 差异化视频码率 → 带宽差异化控制
> **辅助特性说明**: 针对视频业务，按终端系统维度做码率差异化

---

## 1. 特性概述

| 属性 | 说明 |
|------|------|
| **特性ID** | GWFD-110301 |
| **特性名称** | 基于终端系统的码率差异化控制 |
| **所属产品** | UDG |
| **适用网元** | PGW-U、UPF |
| **支持版本** | UDG 20.3.0及后续版本 |
| **License控制项** | 82209784 LKV3G5RDSC01 基于终端系统的码率差异化控制 |

**核心能力一句话描述**：运营商基于用户的应用及终端操作系统（Android/iOS/Windows）进行差异化业务控制，通过为不同终端系统配置不同的带宽管理策略，实现视频码率的差异化控制，提升用户体验，优化网络效率。

**在带宽控制场景中的定位**：本特性是带宽控制子场景的辅助特性，聚焦"视频码率差异化"这一垂直维度。主特性 GWFD-110311（基于业务感知的带宽控制）提供通用带宽控制能力，本特性在其基础上增加终端系统识别维度，使得同一业务在不同终端系统上可以采用不同的码率策略。

**与BWM、SA的协作关系**：
- **SA（Service Awareness）依赖**：本特性基于SA执行业务识别，必须同时开启SA功能。SA负责识别应用类型（如HTTP视频流量）。
- **BWM（Bandwidth Management）依赖**：本特性基于GWFD-110311（基于业务感知的带宽控制）的BWM框架实现差异化控制，必须开启带宽控制功能。
- **OSTYPE维度**：在BWM业务定义中增加OSTYPE参数，实现按终端系统区分的精细化控制。

**应用场景**：
- 运营商需基于不同的终端系统对某一应用进行精细化管理（如：iOS终端和Android终端的视频流采用不同码率上限）。
- 在现有网络资源前提下，精细区分用户群（按终端系统），进行差异化管理，提升带宽资源利用率。

## 2. 核心概念

### 2.1 终端系统（OSTYPE）

终端系统指用户设备的操作系统类型。本特性支持识别以下三种终端系统：

| OSTYPE取值 | 说明 |
|------------|------|
| **Android** | 安卓系统终端 |
| **IOS** | 苹果iOS系统终端 |
| **Windows** | Windows系统终端 |

终端系统识别是通过BWM业务（BWMSERVICE）中的OSTYPE参数实现的。UPF在处理用户面流量时，结合SA的业务识别结果和终端系统信息，将流量匹配到对应的BWM业务规则。

### 2.2 视频码率（Bitrate）

视频码率是指视频数据在单位时间内的数据量。视频OTT（Over-The-Top）提供商为了保障播放流畅性，会自动感知带宽变化并调整视频分辨率。每档分辨率视频都有一个最小播放带宽以保证流畅播放，这个最小播放带宽就是视频码率控制的依据。

### 2.3 码率差异化的业务背景

不同终端系统对于部分视频OTT的同一分辨率视频采用差异化格式编码，导致同分辨率视频的最小播放带宽不一致。例如，同一视频内容在Android和iOS上可能采用不同的编码格式，所需的带宽不同。这为运营商精确控制业务带来挑战，本特性正是为解决这一问题而设计。

### 2.4 终端系统识别机制

终端系统识别不是通过单独的信令交互完成的，而是通过以下机制实现：
- UPF在建立用户会话时获取终端相关信息。
- 结合SA的业务识别能力，将终端系统类型与业务类型关联。
- 在BWM业务定义中通过OSTYPE参数标记该业务适用的终端系统。

### 2.5 与SA、BWM的关系

```
SA（业务识别）          BWM（带宽控制）           本特性（终端系统维度）
┌──────────────┐      ┌──────────────────┐      ┌─────────────────────┐
│ 识别应用类型   │      │ 带宽管理业务定义    │      │ OSTYPE维度区分       │
│ (如HTTP视频)  │ ──→  │ 带宽管理控制器(CAR) │ ←──  │ Android/iOS/Windows │
│              │      │ 带宽管理规则        │      │ 差异化码率策略        │
└──────────────┘      └──────────────────┘      └─────────────────────┘
      ↓                       ↓                         ↓
   识别视频业务            执行流量监管              按终端系统差异化
```

## 3. 实现原理与流程

### 3.1 核心原理

视频OTT为了保障播放的流畅性会自动感知带宽变化调整视频分辨率。每档分辨率视频都有一个最小播放带宽以保证流畅播放，此最小播放带宽是视频码率控制的依据。当前不同的终端系统对于部分视频OTT的同一分辨率视频采用差异化格式编码，导致同分辨率视频的最小播放带宽不一致。

运营商开启基于终端系统的码率差异化控制功能后，当不同用户在不同的终端平台上进行某一业务时，可基于终端平台进行差异化业务控制。在现有的网络资源下，提高带宽资源的利用率，服务更多的用户。

### 3.2 配置架构

基于终端系统的码率差异化控制的配置架构由以下BWM核心对象组成：

| 配置对象 | MML命令 | 职责 |
|----------|---------|------|
| **BWM业务（BWMSERVICE）** | ADD BWMSERVICE | 定义业务类型+协议+操作系统类型，是终端系统差异化控制的入口 |
| **BWM控制器（BWMCONTROLLER）** | ADD BWMCONTROLLER | 定义CAR参数（CIR/CBS/PIR/PBS/三色动作），控制实际带宽 |
| **BWM参数（BANDWIDTHMNG）** | SET BANDWIDTHMNG | 配置带宽管理生效范围（业务处理单元级或整机级） |
| **BWM用户组（BWMUSERGROUP）** | ADD BWMUSERGROUP | 定义用户组，配置优先级和业务类型 |
| **BWM规则（BWMRULE）** | ADD BWMRULE | 将用户组与BWM业务、BWM控制器绑定，定义带宽控制规则 |
| **APN绑定（APNBINDBWMUSRG）** | ADD APNBINDBWMUSRG | 将APN下的用户包含到用户组 |
| **APN OS级开关（APNOSLELBWMSW）** | SET APNOSLELBWMSW | 开启APN的操作系统级带宽管理功能 |

```

       │
       ▼
┌─────────────────────┐
│ 步骤1: SA业务识别    │  SA识别流量所属应用类型

└─────────┬───────────┘

│ 步骤2: 终端系统识别  │  识别终端操作系统类型
│ Android/iOS/Windows │  （结合会话信息）
└─────────┬───────────┘
          │
          ▼
┌─────────────────────────┐
│ 步骤3: BWM业务匹配       │  根据业务类型+OSTYPE
│ 匹配BWMSERVICE(含OSTYPE)│  匹配到对应BWM业务
└─────────┬───────────────┘

│ (CIR/CBS/PIR/PBS)      │

```

### 3.4 三色令牌桶控制原理

BWM控制器使用CAR（Committed Access Rate）进行流量监管，基于三色令牌桶机制：

| 颜色 | 含义 | 常见动作配置 | 控制效果 |
|------|------|------------|----------|
| **绿色（GREEN）** | 流量速率 <= CIR | PASS | 承诺带宽内的流量放行 |
| **黄色（YELLOW）** | CIR < 流量速率 <= PIR | PASS或DISCARD | 超出承诺但低于峰值的流量 |
| **红色（RED）** | 流量速率 > PIR | DISCARD | 超出峰值的流量丢弃 |

**两种典型控制效果**：
- 绿色PASS + 黄色PASS + 红色DISCARD = 流量最大速率不超过PIR（峰值速率）
- 绿色PASS + 黄色DISCARD + 红色DISCARD = 流量最大速率不超过CIR（承诺速率）

### 4.2 License开关配置

**SET LICENSESWITCH** - 打开本特性的License开关。

License控制项为：`LKV3G5RDSC01`（82209784 基于终端系统的码率差异化控制）。

```
SET LICENSESWITCH:LICITEM="LKV3G5RDSC01",SWITCH=ENABLE;
```

### 4.3 BWM业务配置（核心：OSTYPE维度）

**ADD BWMSERVICE** - 定义带宽管理业务，关键是OSTYPE参数。

| 参数 | 名称 | 示例值 | 说明 |
|------|------|--------|------|
| BWMSERVICENAME | 带宽控制业务名称 | bs_test | 自定义名称 |
| BWMSERVICETYPE | 业务类型 | NONTOS | 非TOS业务类型 |
| NONTOSSRVTYPE | 非TOS业务类型 | PROTOCOL | 协议类型 |
| PROTOCOLNAME | 协议名称 | http | 匹配的协议 |
| **OSTYPE** | **操作系统类型** | **IOS** | **终端系统识别关键参数** |

**OSTYPE是本特性的核心参数**，决定该BWM业务匹配哪种终端系统的流量。要实现差异化控制，需要为每种终端系统分别创建BWMSERVICE。

### 4.4 BWM控制器配置

**ADD BWMCONTROLLER** - 配置CAR参数，控制实际带宽。

| 参数 | 名称 | 示例值 | 推荐配置规则 |
|------|------|--------|------------|
| BWMCNAME | 控制器名称 | bc-up | 自定义名称 |
| CTRLTYPE | 控制器类型 | CAR | 固定为CAR |
| CIR | 承诺信息速率(kbps) | 300 | CIR = 0.5 * PIR |
| CBS | 承诺突发缓存(bytes) | 10000 | CBS >= CIR * 1.5 * 1000 / 8 |
| PIR | 峰值信息速率(kbps) | 600 | 最大带宽 |
| PBS | 峰值突发缓存(bytes) | 12000 | PBS >= PIR * 1.5 * 1000 / 8 |
| GREEN | 绿色动作 | PASS | 承诺带宽内放行 |
| YELLOW | 黄色动作 | PASS | 超承诺但低于峰值 |
| RED | 红色动作 | DISCARD | 超峰值丢弃 |

通常需要配置上行和下行两个控制器（如bc-up和bc-down）。

### 4.5 带宽管理参数配置

**SET BANDWIDTHMNG** - 配置带宽管理生效范围。

| 参数 | 名称 | 示例值 | 说明 |
|------|------|--------|------|
| RANGE | 生效范围 | SYSTEM | SYSTEM表示整机级 |

BWM控制器中的用户组级业务带宽和连接数等配置数据是基于业务处理单元生效还是整机生效，由此参数控制。

### 4.6 BWM用户组配置

**ADD BWMUSERGROUP** - 定义带宽管理用户组。

| 参数 | 名称 | 示例值 | 说明 |
|------|------|--------|------|
| USERGROUPTYPE | 用户组类型 | SPECIFIC_GROUP | 特定用户组 |
| USERGROUPNAME | 用户组名称 | ug_test | 自定义名称 |
| USERGROUPPRI | 用户组优先级 | 1 | 优先级 |
| USERGLEVSRVTYPE | 用户组级业务类型 | NONTOS | 非TOS业务 |

### 4.7 BWM规则配置

**ADD BWMRULE** - 将用户组、BWM业务、BWM控制器三者绑定。

| 参数 | 名称 | 示例值 | 说明 |
|------|------|--------|------|
| USERGROUPTYPE | 用户组类型 | SPECIFIC_GROUP | 特定用户组 |
| USERGROUPNAME | 用户组名称 | ug_test | 已配置的用户组 |
| BWMRULETYPE | 规则类型 | GROUP_SPECIFIC | 用户组级别特定规则 |
| BWMRULENAME | 规则名称 | br_test | 自定义名称 |
| BWMSERVICENAME | BWM业务名称 | bs_test | 已配置的BWM业务 |

**ADD APNBINDBWMUSRG** - 将APN下的用户包含到用户组。

| 参数 | 名称 | 示例值 | 说明 |
|------|------|--------|------|
| USERGROUPNAME | 用户组名称 | ug_test | 已配置的用户组 |
| APN | APN名称 | apn-test | 已配置的APN |

| 参数 | 名称 | 示例值 | 说明 |
|------|------|--------|------|
| APNNAME | APN名称 | apn-test | 已配置的APN |

### 4.9 与PCC规则的联动

本特性基于BWM框架实现，与PCC规则的关系：
- BWM业务定义中通过PROTOCOLNAME匹配协议流量（如http），而非直接依赖PCC规则中的过滤器。
- SA识别业务后，BWM框架根据BWMSERVICE定义（协议+OSTYPE）匹配流量。
- PCC规则负责会话级策略控制（如QoS），BWM负责带宽级精细控制，两者并行工作。

## 5. 配置案例

### 案例一：iOS终端HTTP业务带宽控制（基础场景）

**场景描述**：对同一用户组下的iOS系统用户的上行、下行用户面数据流量（HTTP协议）进行带宽监管。

**配置步骤**：

**步骤1：打开License开关**
```
SET LICENSESWITCH:LICITEM="LKV3G5RDSC01",SWITCH=ENABLE;
```

**步骤2：配置BWM业务（OSTYPE=IOS）**
```
ADD BWMSERVICE:BWMSERVICENAME="bs_test",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=PROTOCOL,PROTOCOLNAME="http",OSTYPE=IOS;
```

**步骤3：配置上行和下行BWM控制器**

上行控制器（CIR=300kbps, PIR=600kbps）：
```
ADD BWMCONTROLLER:BWMCNAME="bc-up",CTRLTYPE=CAR,CIR=300,CBS=10000,PIR=600,PBS=12000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
```

下行控制器（CIR=300kbps, PIR=600kbps）：
```
ADD BWMCONTROLLER:BWMCNAME="bc-down",CTRLTYPE=CAR,CIR=300,CBS=10000,PIR=600,PBS=12000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
```

**步骤4：配置带宽管理生效范围**
```
SET BANDWIDTHMNG:RANGE=SYSTEM;
```

**步骤5：配置BWM用户组**
```
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",USERGROUPPRI=1,USERGLEVSRVTYPE=NONTOS;
```

**步骤6：配置BWM规则（绑定业务、控制器）**
```
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="br_test",BWMSERVICENAME="bs_test",UPBWMCTRLNAME1="bc-up",DNBWMCTRLNAME1="bc-down",BWMRULEPRI=1;
```

**步骤7：将APN绑定到用户组**
```
ADD APNBINDBWMUSRG:USERGROUPNAME="ug_test",APN="apn-test";
```

**步骤8：开启APN操作系统级带宽管理开关**
```
SET APNOSLELBWMSW:APNNAME="apn-test",ISENABLE=ENABLE;
```

**控制效果**：iOS系统终端的HTTP流量上行最大速率600kbps（PIR），下行最大速率600kbps（PIR），超出PIR的流量被丢弃。

### 案例二：多终端系统差异化码率控制（进阶场景）

**场景描述**：对同一APN下的用户，按终端系统差异化控制HTTP视频业务带宽：
- iOS终端：上行PIR=4Mbps，下行PIR=8Mbps（高质量视频体验）
- Android终端：上行PIR=2Mbps，下行PIR=4Mbps（标准视频体验）

**配置步骤**：

**步骤1：打开License开关**
```
SET LICENSESWITCH:LICITEM="LKV3G5RDSC01",SWITCH=ENABLE;
```

**步骤2：配置两个BWM业务（按OSTYPE区分）**
```
ADD BWMSERVICE:BWMSERVICENAME="bs_ios_http",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=PROTOCOL,PROTOCOLNAME="http",OSTYPE=IOS;
ADD BWMSERVICE:BWMSERVICENAME="bs_android_http",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=PROTOCOL,PROTOCOLNAME="http",OSTYPE=ANDROID;
```

**步骤3：配置iOS的BWM控制器（高带宽）**
```
ADD BWMCONTROLLER:BWMCNAME="bc_ios_up",CTRLTYPE=CAR,CIR=2000,CBS=375000,PIR=4000,PBS=750000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
ADD BWMCONTROLLER:BWMCNAME="bc_ios_down",CTRLTYPE=CAR,CIR=4000,CBS=750000,PIR=8000,PBS=1500000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
```

**步骤4：配置Android的BWM控制器（低带宽）**
```
ADD BWMCONTROLLER:BWMCNAME="bc_android_up",CTRLTYPE=CAR,CIR=1000,CBS=187500,PIR=2000,PBS=375000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
ADD BWMCONTROLLER:BWMCNAME="bc_android_down",CTRLTYPE=CAR,CIR=2000,CBS=375000,PIR=4000,PBS=750000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
```

> CBS/PBS计算说明：CBS >= CIR * 1.5 * 1000 / 8，PBS >= PIR * 1.5 * 1000 / 8

**步骤5：配置带宽管理生效范围**
```
SET BANDWIDTHMNG:RANGE=SYSTEM;
```

**步骤6：配置BWM用户组**
```
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_ios",USERGROUPPRI=1,USERGLEVSRVTYPE=NONTOS;
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_android",USERGROUPPRI=2,USERGLEVSRVTYPE=NONTOS;
```

**步骤7：配置BWM规则（分别绑定不同终端系统的业务和控制器）**
```
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_ios",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="br_ios",BWMSERVICENAME="bs_ios_http",UPBWMCTRLNAME1="bc_ios_up",DNBWMCTRLNAME1="bc_ios_down",BWMRULEPRI=1;
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_android",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="br_android",BWMSERVICENAME="bs_android_http",UPBWMCTRLNAME1="bc_android_up",DNBWMCTRLNAME1="bc_android_down",BWMRULEPRI=1;
```

**步骤8：将APN分别绑定到用户组**
```
ADD APNBINDBWMUSRG:USERGROUPNAME="ug_ios",APN="apn-test";
ADD APNBINDBWMUSRG:USERGROUPNAME="ug_android",APN="apn-test";
```

**步骤9：开启APN OS级带宽管理开关**
```
SET APNOSLELBWMSW:APNNAME="apn-test",ISENABLE=ENABLE;
```

**控制效果**：
- iOS终端HTTP视频流量：上行最大4Mbps，下行最大8Mbps
- Android终端HTTP视频流量：上行最大2Mbps，下行最大4Mbps
- 超出PIR的流量被丢弃，实现了按终端系统的码率差异化控制

## 6. 验证与调测

### 6.1 验证APN OS级带宽管理开关

使用LST APNOSLELBWMSW命令检查APN操作系统级带宽管理开关是否开启：

```
LST APNOSLELBWMSW:APNNAME="apn-test";
```

预期输出：
```
RETCODE = 0 Operation Success.

APN OS Level Bandwidth Managament Switch Information
--------------------------------------------------------------------------------
                                        APN = apn-test
 APN OS Level Bandwidth Managament Switch = ENABLE
(Number of results = 1)
--- END
```

如果开关为DISABLE，需重新执行SET APNOSLELBWMSW命令开启。

### 6.2 验证用户会话与APN绑定

使用DSP SESSIONINFO命令通过用户IMSI查看用户激活所使用的APN：

```
DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="633451234612111";
```

预期输出中应包含：
```
IMSI  =  633451234612111
APN name  =  apn-test
```

如果IMSI绑定的APN与规划值不一致，需要重新配置并重新激活用户。

LST APNBINDBWMUSRG:USERGROUPNAME="ug_test";

预期输出：

LST BWMUSERGROUP:QRYUSRGRPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test";

### 6.6 验证BWM业务与控制器

- BWM Service的PROTOCOLNAME（协议名称）与规划一致
- 上行和下行BWM Controller的CIR/CBS/PIR/PBS/三色动作与规划一致

### 6.7 码率控制效果验证

**测试条件**：
- 测试终端为对应OSTYPE的系统（如IOS系统测试终端）
- 使用可控制发包速率的发包工具
- 访问业务的带宽大于所配置PIR
- 只有当前测试用户发出流量

**测试步骤**：
1. 打开接入侧/PDN侧镜像接口上的抓包工具
2. 激活APN下的测试用户，使用发包工具发送HTTP用户报文
3. 使用Wireshark的Statistics > IO Graphs功能观察报文速率曲线
4. 验证流量速率是否符合配置的带宽控制参数

**三色动作对应的控制效果验证标准**：

| CAR配置 | 验证标准 | 测试条件 |
|---------|----------|----------|
| 绿色PASS + 黄色PASS + 红色DISCARD | 流量最大速率不超过PIR | 发包速率大于PIR |
| 绿色PASS + 黄色DISCARD + 红色DISCARD | 流量最大速率不超过CIR | 发包速率大于CIR |
| 绿色DISCARD + 黄色DISCARD + 红色DISCARD | 流量全部被丢弃 | 无限制 |

**抓包过滤条件设置**（以观察两个用户为例）：
- Graph 1：两个用户S1-U/N3口下行报文总和，过滤条件：`gtp && (ip.addr==10.0.0.1 || ip.addr == 10.0.0.2)`
- Graph 2：单个用户S1-U/N3口下行报文，过滤条件：`!gtp && ip.addr ==10.0.0.1`
- Graph 3：另一用户S1-U/N3口下行报文，过滤条件：`!gtp && ip.addr ==10.0.0.2`
- Graph 4：两个用户SGi/N6接口侧下行报文，过滤条件：`!gtp && (ip.addr==10.0.0.1 || ip.addr == 10.0.0.2)`

X Axis选择时间单位1秒，Y Axis选择Bytes/Tick。

**验证结论**：如果隶属于同一个用户组的两个用户的总流量受控（速率不超过配置上限），则带宽管理功能正常。

### 6.8 常见问题排查

| 问题现象 | 可能原因 | 排查方法 |
|----------|----------|----------|
| 流量不受控 | APN OS级开关未开启 | LST APNOSLELBWMSW检查开关状态 |
| 流量不受控 | 用户IMSI未绑定正确APN | DSP SESSIONINFO验证APN绑定 |
| 流量不受控 | 用户组未绑定APN | LST APNBINDBWMUSRG验证绑定关系 |
| 流量不受控 | BWM规则配置错误 | LST BWMRULE验证规则配置 |
| 流量不受控 | OSTYPE或协议不匹配 | LST BWMSERVICE验证OSTYPE和PROTOCOLNAME |
| 流量速率异常 | CAR参数配置不合理 | 检查CIR/CBS/PIR/PBS是否符合推荐配置 |
| 终端无法匹配 | 终端系统不支持 | 确认终端为Android/iOS/Windows之一 |

**排查失败后的信息收集**：
1. 在镜像接口开启抓包工具并保存报文
2. 执行EXP MML命令导出当前配置数据为MML脚本文件
3. 执行DSP ROUTE命令查看路由信息
4. 执行LST INTERFACE命令查看接口信息
5. 收集对端设备配置及接口状态信息
6. 联系技术支持

### 7.9 CAR参数推荐配置规则

| 参数 | 推荐规则 | 示例（PIR=600kbps） |
|------|----------|---------------------|
| CIR | CIR = 0.5 * PIR | CIR = 300 |
| CBS | CBS >= CIR * 1.5 * 1000 / 8 | CBS >= 56250（保证1.5秒数据队列缓存） |
| PBS | PBS >= PIR * 1.5 * 1000 / 8 | PBS >= 112500（保证1.5秒数据队列缓存） |

> CBS和PBS的设计目的是保证1.5秒数据的队列缓存能力。

### 7.10 应用场景说明

**场景一：终端系统精细化管理**
运营商需基于不同的终端系统对某一应用进行精细化管理。例如，视频OTT在不同终端系统上使用不同的编码格式，导致相同分辨率视频的带宽需求不同。通过本特性，运营商可以针对每种终端系统配置最适合的带宽策略。

**场景二：用户群精细化区分**
在现有网络资源前提下，精细区分用户群（按终端系统维度），进行差异化管理。在网络拥塞时，可为运营商提供更精确的无线宽带管理方式。在不增加基站的前提下，更加合理地利用现有无线和核心网设备资源，优化运营商的网络效率。
