# GWFD-110311 基于业务感知的带宽控制 知识文档

## 概述

### 特性定义

基于业务感知的带宽控制（Bandwidth Management, BWM）是 UDG 通过带宽管理对流量进行控制的特性。随着 P2P、VoIP、网络游戏等应用的普及，P2P 类业务占用大量网络带宽资源导致拥塞。本特性使 UDG 能够：

1. 通过 SA（业务感知）识别报文业务类型（P2P、VoIP、HTTP 等），或通过 TOS/DSCP 值识别业务。
2. 根据业务类型、流量属性（上/下行方向、时间段）、用户属性（RAT、Roaming）匹配业务控制规则。
3. 按照匹配到的规则执行带宽控制动作（CAR 流量监管或 Shaping 流量整形），实现基于**用户级 / 用户组级 / 整机级**三个层次的层次化带宽管理。

### 适用NF

| NF | 角色 | 说明 |
|----|------|------|
| PGW-U / UPF（UDG 侧） | 执行带宽控制 | 根据业务类型完成带宽控制，实现用户/用户组/整机三级层次化控制 |
| PGW-C / SMF（UNC 侧） | 策略下发 | 通过 N4 接口给 UDG 下发带宽控制规则 |
| PCRF / PCF | 策略管理中心 | 基于用户签约、套餐等动态安装/删除预定义规则 |
| SGSN / eNodeB / gNodeB | 接入侧 | 提供 2/3/4/5G 用户接入能力 |

## 核心概念

### 关键术语

| 术语 | 全称/含义 | 说明 |
|------|----------|------|
| **BWM** | Bandwidth Management，带宽管理 | UDG 对用户业务流量进行速率控制的统称 |
| **CAR** | Committed Access Rate，流量监管 | 基于令牌桶算法，超出速率的报文直接丢弃或重标记 |
| **Shaping** | 流量整形 | 对报文进行缓存排队，平滑输出速率（仅用户级支持） |
| **CIR** | Committed Information Rate，承诺信息速率 | 保证带宽，单位 kbps |
| **PIR** | Peak Information Rate，峰值信息速率 | 最大带宽，单位 kbps |
| **CBS** | Committed Burst Size，承诺突发大小 | 保证带宽对应的令牌桶缓存，单位字节 |
| **PBS** | Peak Burst Size，峰值突发大小 | 峰值带宽对应的令牌桶缓存，单位字节 |
| **TOS** | Type of Service，业务类型 | IP 报文中的 TOS 域，用于区分业务优先级 |
| **GREEN/YELLOW/RED** | 三色标记 | GREEN=符合 CIR；YELLOW=超过 CIR 但未超 PIR；RED=超过 PIR |
| **ServiceLevel** | 业务级别 | 20.9.0 版本引入，用于分级带宽管理控制 |

### 对象模型

基于业务感知的带宽控制的核心配置对象及其关系：

```
BWM Service（带宽管理业务）
  └── 定义业务识别方式（TOS 或 非TOS）
      ├── TOS 业务：按 TOS/DSCP 值识别
      └── 非TOS 业务：按 Protocol、Protocol Group、CategoryProp 识别

BWM Controller（带宽管理控制器）
  └── 定义带宽控制动作
      ├── CTRLTYPE：CAR（流量监管）或 SHAPING（流量整形，仅用户级）
      ├── CIR/CBS/PIR/PBS：令牌桶参数
      ├── GREEN/YELLOW/RED：三色处理策略（PASS / REMARK / DISCARD）
      ├── COLORISAWARE：是否感知颜色（优先保证绿色流量）
      ├── PRIORITYISAWARE：是否感知优先级
      └── SRVLEVELSPEC：业务等级规格（分级带宽控制用）

BWM User Group（带宽管理用户组）
  └── 定义需要进行带宽控制的用户集合
      ├── USERGROUPTYPE：用户组类型
      ├── USERGROUPNAME：用户组名称
      ├── USERGROUPPRI：用户组优先级
      ├── USERLEVSRVTYPE：用户级业务类型（TOS/NONTOS）
      └── USERGLEVSRVTYPE：用户组级业务类型（TOS/NONTOS）

BWM Rule（带宽管理规则）
  └── 绑定 BWM Service + BWM Controller + 控制属性
      ├── BWMRULETYPE：SUBSCRIBER_SPECIFIC（用户级）/ GROUP_SPECIFIC（用户组级）/ GROUP_DEFAULT（缺省）
      ├── UPBWMCTRLNAME / DNBWMCTRLNAME：上/下行带宽管理控制器
      ├── TIMERANGENAME：时间段（可选）
      ├── SERVICELEVEL：业务级别（分级带宽控制）
      └── BWMRULEPRI：规则优先级

BWM Rule Global（全局/整机级规则）
  └── ADD BWMRULEGLOBAL 配置整机级业务控制规则

BCSrvLevelPly（业务级别策略）
  └── 为每个 ServiceLevel 的 BWM Controller 设置 CIR/PIR 比例
```

### CAR vs Shaping 差异

| 对比项 | CAR（流量监管） | Shaping（流量整形） |
|--------|----------------|-------------------|
| 工作方式 | 基于令牌桶，超限报文丢弃或重标记 | 对报文缓存排队，平滑输出 |
| 延迟影响 | 不增加延迟 | 增加延迟和抖动 |
| 适用层级 | 用户级、用户组级、整机级 | **仅用户级支持** |
| CTRLTYPE 取值 | CAR | SHAPING |
| 适用场景 | P2P 限速、带宽保护 | 精细化流量平滑 |

### TOS 业务 vs 非TOS 业务

| 对比项 | TOS 业务 | 非TOS 业务 |
|--------|---------|-----------|
| BWMSERVICETYPE | TOS | NONTOS |
| 识别方式 | IP 报文 TOS/DSCP 值 | 三四层 Filter、七层协议、协议特征 |
| 是否需要 SA | 不需要（直接读 DSCP） | 需要 SA 识别和解析 |
| NONTOSSRVTYPE | 不适用 | PROTOCOL / CATEGORY_PROP / PROTOCOL_GROUP |
| 典型场景 | 按 IP 优先级限速 | 按 P2P/HTTP 协议限速 |

### 基础版 vs 增强版（分级带宽控制）

| 对比项 | 基础版（20.0.0） | 增强版（20.9.0） |
|--------|-----------------|-----------------|
| 分级带宽控制 | 不支持 | 支持 ADD BCSRVLEVELPLY |
| 业务级别策略 | 不支持 | 支持 ServiceLevel 1~15 |
| 调度机制 | 无分级调度 | 高优先级业务优先获带宽，低优先级保基础带宽 |
| 核心命令 | ADD BWMSERVICE / BWMCONTROLLER / BWMRULE | 额外支持 ADD BCSRVLEVELPLY |
| 适用场景 | 简单限速 | 差异化业务分级管理 |

## 原理与流程

### 业务控制域

带宽管理基于三个作用范围域进行层次化控制：

```
整机级（Global）
  └── 对整机所有用户的业务流量进行控制
       └── 用户组级（APN/DNN / 切片 / UserProfile）
            └── 对一个用户组下所有用户的业务流量进行控制
                 └── 用户级（Subscriber）
                      └── 对单个用户的业务流量进行控制
```

- **用户级**：基于用户级别的业务控制和带宽管理。**只有用户级支持 SHAPING**。
- **用户组级**：通过 UserProfile / APN / 切片 划分用户组，进行组级控制。
- **整机级**：通过 `ADD BWMRULEGLOBAL` 配置整机级业务控制规则。

### 实现原理

1. **业务识别**：UDG 识别报文的业务类型。
   - TOS 业务：通过用户报文的 TOS/DSCP 值进行识别。
   - 非TOS 业务：通过三四层过滤规则、七层协议类型、七层协议特征信息进行识别。
2. **规则匹配**：对于识别出的业务，UDG 根据用户属性和流量属性匹配业务控制规则。
   - 用户属性：RAT（无线接入类型）、Roaming（漫游属性）。
   - 流量属性：上行/下行方向、所处时间段。
3. **执行控制**：按照匹配到的业务控制规则执行 CAR/Shaping 动作。

### 业务流程

```
PGW-C/SMF ──N4──> PGW-U/UPF（下发带宽控制规则）
                      │
                      ▼
N3/S1-U/N6/Gi ──> PGW-U/UPF（收到上下行业务流量）
                      │
                      ▼
                 识别业务类型（SA/TOS）
                      │
                      ▼
                 匹配业务控制规则
                 （业务 + 用户属性 + 流量属性）
                      │
                      ▼
                 执行带宽控制（CAR/Shaping）
                 （用户级 -> 用户组级 -> 整机级）
```

### 分级带宽控制原理

20.9.0 版本引入的分级带宽管理：

1. 通过 `ADD BWMCONTROLLER` 的 `SRVLEVELSPEC` 参数设置业务等级规格（最大 15 级）。
2. 通过 `ADD BCSRVLEVELPLY` 为每个 ServiceLevel 设置 CIR 比例和 PIR 比例。
3. 所有 ServiceLevel 的 CIRRATE 之和不能超过 100%。
4. 高优先级业务在保障低优先级业务基础带宽的前提下优先获得带宽；高优先级业务需求低时剩余带宽共享给低优先级业务。

### 带宽管理生效范围

通过 `SET BANDWIDTHMNG` 命令的 `RANGE` 参数控制用户组级业务带宽和连接数的生效范围：

| RANGE 取值 | 说明 | 建议场景 |
|-----------|------|---------|
| CPU | 基于业务处理单元（CPU）生效 | 已有用户组级带宽配置的现网 |
| SYSTEM | 基于整机生效 | 新建局点 |

## 配置规则

### 激活步骤（通用流程）

所有带宽控制场景的配置遵循以下通用步骤：

```
步骤1: SET LICENSESWITCH     -- 打开 License 开关
步骤2: ADD BWMSERVICE        -- 配置 BWM 业务（定义业务识别方式）
步骤3: ADD BWMCONTROLLER     -- 配置 BWM 控制器（定义 CAR/Shaping 参数）
步骤4: SET BANDWIDTHMNG      -- 配置带宽管理参数（生效范围等，按需）
步骤5: ADD BWMUSERGROUP      -- 配置带宽管理用户组
步骤6: ADD BWMRULE           -- 配置带宽管理规则（绑定 Service + Controller）
步骤7: ADD APNBINDBWMUSRG    -- 绑定 APN 到用户组
步骤8: ADD BWMRULEGLOBAL     -- 配置整机级规则（仅三级控制场景）
步骤9: SET REFRESHSRV        -- 刷新生效（仅涉及 Rule/L7Filter 时）
```

### 参数说明

#### BWM Controller 关键参数

| 参数 | 含义 | 推荐取值 |
|------|------|---------|
| BWMCNAME | 带宽管理控制器名称 | 自定义命名 |
| CTRLTYPE | 带宽管理控制类型 | CAR 或 SHAPING |
| CIR | 承诺信息速率（kbps） | CIR = 0.5 * PIR（最大带宽建议为保证带宽的2倍） |
| CBS | 承诺突发大小（字节） | CBS >= CIR * 1.5 * 1000 / 8（保证1.5秒数据缓存） |
| PIR | 峰值信息速率（kbps） | 根据带宽控制需求规划 |
| PBS | 峰值突发大小（字节） | PBS >= PIR * 1.5 * 1000 / 8（保证1.5秒数据缓存） |
| GREEN | 绿色处理策略 | PASS（放行） |
| YELLOW | 黄色处理策略 | PASS / REMARK（重标记） |
| RED | 红色处理策略 | DISCARD（丢弃） |
| COLORISAWARE | 颜色感知 | ENABLE 时优先保证组内绿色用户流量 |
| PRIORITYISAWARE | 优先级感知 | ENABLE 时按 BWMRULEPRI 区分业务优先级 |
| SRVLEVELSPEC | 业务等级规格 | 1~15（分级带宽控制用） |

#### BWM Service 业务类型参数

| 参数 | 取值 | 说明 |
|------|------|------|
| BWMSERVICETYPE | TOS | 以 TOS/DSCP 值区分业务 |
| BWMSERVICETYPE | NONTOS | 以协议/特征区分业务 |
| TOSCFGTYPE | CLASS | TOS 配置类型为分类模式 |
| TOSCLASSTYPE | BE/AF1/AF2/.../CS7 | TOS 分类类型（DSCP 等级） |
| NONTOSSRVTYPE | PROTOCOL | 非TOS 类型为协议 |
| NONTOSSRVTYPE | CATEGORY_PROP | 非TOS 类型为分类属性 |
| PROTOCOLNAME | http/bt/... | 协议名称 |
| CATEPROPNAME | 自定义 | 分类属性名称（需与 ADD RULE 的 POLICYNAME 一致） |

#### BWM Rule 类型参数

| BWMRULETYPE | 含义 | 典型场景 |
|-------------|------|---------|
| SUBSCRIBER_SPECIFIC | 用户级规则 | 对每个用户的业务流量分别控制 |
| GROUP_SPECIFIC | 用户组级规则 | 对用户组下所有用户流量总量控制 |
| GROUP_DEFAULT | 用户组缺省规则 | APN/DNN 级整体限速（不区分业务） |

### 多维度配置场景

#### 场景一：基于 APN/DNN 的用户组级带宽控制

**适用场景**：对某个 APN/DNN 下的一个用户组进行整体流量监管（不区分业务类型）。

**关键配置差异**：
- BWMSERVICE 不需要配置（该功能不区分业务类型）。
- BWMRULETYPE = GROUP_DEFAULT。
- USERGLEVSRVTYPE = NONTOS。

```mml
SET LICENSESWITCH:LICITEM="LKV3G5TCSA01",SWITCH=ENABLE;

ADD APN:APN="apn-test", HASVPN=DISABLE, HASVPNIPV6=DISABLE;

// 配置上行/下行 BWM 控制器
ADD BWMCONTROLLER:BWMCNAME="bc-up",CTRLTYPE=CAR,CIR=2000000,CBS=320000000,PIR=4000000,PBS=800000000,GREEN=PASS,YELLOW=PASS,RED=DISCARD,PRIORITYISAWARE=DISABLE;
ADD BWMCONTROLLER:BWMCNAME="bc-down",CTRLTYPE=CAR,CIR=2000000,CBS=320000000,PIR=4000000,PBS=800000000,GREEN=PASS,YELLOW=PASS,RED=DISCARD,PRIORITYISAWARE=DISABLE;

SET BANDWIDTHMNG:RANGE=SYSTEM,CIRMINTHRES=300000,PIRMINTHRES=500000;

// 配置用户组（仅支持 NONTOS）
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",USERGROUPPRI=1,USERLEVSRVTYPE=NONTOS,USERGLEVSRVTYPE=NONTOS;

// GROUP_DEFAULT 规则（不关联 BWMSERVICE）
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_DEFAULT,UPBWMCTRLNAME1="bc-up",DNBWMCTRLNAME1="bc-down";

ADD APNBINDBWMUSRG:USERGROUPNAME="ug_test",APN="apn-test";
```

#### 场景二：基于 PCC 预定义规则的带宽控制

**适用场景**：PCF/PCRF 动态安装/删除预定义规则触发带宽控制。典型用途：
- 套餐耗尽后降低用户访问速率（FUP）。
- 用户进入指定 PRA 区域时限速。

**关键配置差异**：
- 需要配置 ADD CATEGORYPROP 和 ADD RULE（POLICYTYPE=BWM）。
- BWMSERVICE 的 CATEPROPNAME 必须与 ADD RULE 的 POLICYNAME 一致。
- RULENAME 必须与 PCF/PCRF、UNC 规划一致。

```mml
SET LICENSESWITCH:LICITEM="LKV3G5TCSA01",SWITCH=ENABLE;

// 配置分类属性和 BWM 业务（CategoryProp 关联方式）
ADD CATEGORYPROP:CATEPROPNAME="cp_700_600";
ADD BWMSERVICE:BWMSERVICENAME="my-bwm-service1",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=CATEGORY_PROP,CATEPROPNAME="cp_700_600";

// 配置上行 700kbps、下行 600kbps 控制器
ADD BWMCONTROLLER:BWMCNAME="bc-up-p2p",CTRLTYPE=CAR,CIR=100,PIR=700,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
ADD BWMCONTROLLER:BWMCNAME="bc-dn-p2p",CTRLTYPE=CAR,CIR=100,PIR=600,GREEN=PASS,YELLOW=PASS,RED=DISCARD;

ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug-bwm",USERGROUPPRI=1,USERLEVSRVTYPE=NONTOS;

ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug-bwm",BWMRULETYPE=SUBSCRIBER_SPECIFIC,BWMRULENAME="br-p2p",BWMSERVICENAME="my-bwm-service1",UPBWMCTRLNAME1="bc-up-p2p",DNBWMCTRLNAME1="bc-dn-p2p",BWMRULEPRI=1;

ADD APNBINDBWMUSRG:USERGROUPNAME="ug-bwm",APN="testapn";

// 配置业务感知规则
ADD L7FILTER:L7FILTERNAME="l7-p2p",SUBL7FLTNAME="sl7-p2p",URL="www.example.com*/*";
ADD FLOWFILTER:FLOWFILTERNAME="flowfilter_p2p";
ADD PROTBINDFLOWF:FLOWFILTERNAME="flowfilter_p2p",PROTOCOLNAME="http",L7FILTERNAME="l7-p2p";

SET REFRESHSRV:REFRESHTYPE=USERPROFILE;

// RULE 关联 BWM 策略：POLICYNAME 必须与 CATEPROPNAME 一致
ADD RULE:RULENAME="rule_p2pbwm",POLICYTYPE=BWM,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="flowfilter_p2p",PRIORITY=1,POLICYNAME="cp_700_600";
```

#### 场景三：基于切片的带宽控制

**适用场景**：对特定网络切片下的用户组进行带宽控制。

**关键配置差异**：
- 需要配置 ADD SNSSAI / ADD SNSSAIUPINTF / ADD SNSSAIBWMUSRG。
- SET BANDWIDTHMNG 的 BWMSUBSCRCTRL=PCC_RULE_IGNORE（使用 BWM Rule 而非 PCC Rule）。

```mml
SET LICENSESWITCH:LICITEM="LKV3G5TCSA01",SWITCH=ENABLE;

ADD BWMSERVICE:BWMSERVICENAME="bs_test",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=PROTOCOL,PROTOCOLNAME="bt";
```

#### 场景四：基于时间段的带宽控制

**适用场景**：在不同时间段对 P2P 流量进行不同规则的流量控制（如高峰期限速更严格）。

**关键配置差异**：
- 需要配置 ADD TIMERANGE + ADD PERITIMERANGE。
- BWMRULE 中通过 TIMERANGENAME1/2/3 关联不同时间段，分别对应 UPBWMCTRLNAME1/2/3 和 DNBWMCTRLNAME1/2/3。

```mml
SET LICENSESWITCH:LICITEM="LKV3G5TCSA01",SWITCH=ENABLE;

ADD BWMSERVICE:BWMSERVICENAME="my-bwm-service8",BWMSERVICETYPE=TOS,TOSCFGTYPE=CLASS,TOSCLASSTYPE=CS7;

// 三个时段的带宽控制器（严格 -> 宽松 -> 最严格）
ADD BWMCONTROLLER:BWMCNAME="bc1",CTRLTYPE=CAR,CIR=30,CBS=1000,PIR=50,PBS=1200,GREEN=PASS,YELLOW=REMARK,YELLOWREMARKT=DSCPVALUE,YELLOWREMARKV=10,RED=DISCARD;
ADD BWMCONTROLLER:BWMCNAME="bc2",CTRLTYPE=CAR,CIR=300,CBS=10000,PIR=500,PBS=12000,GREEN=PASS,YELLOW=REMARK,YELLOWREMARKT=DSCPVALUE,YELLOWREMARKV=15,RED=DISCARD;
ADD BWMCONTROLLER:BWMCNAME="bc3",CTRLTYPE=CAR,CIR=3000,CBS=100000,PIR=5000,PBS=120000,GREEN=PASS,YELLOW=REMARK,YELLOWREMARKT=DSCPVALUE,YELLOWREMARKV=20,RED=DISCARD;

// 配置两个时间段
ADD TIMERANGE:TIMERANGENAME="timerange1";
ADD TIMERANGE:TIMERANGENAME="timerange2";
ADD PERITIMERANGE:TIMERANGENAME="timerange1",PERITMRANGESEQ=1,PERITIMETYPE=DAYSOFWEEK,DAYOFWEEK=DAILY,STARTTIME=8&30,ENDTIME=10&00;
ADD PERITIMERANGE:TIMERANGENAME="timerange2",PERITMRANGESEQ=1,PERITIMETYPE=DAYSOFWEEK,DAYOFWEEK=DAILY,STARTTIME=22&00,ENDTIME=23&00;

ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ugroup8",USERGROUPPRI=1,USERGLEVSRVTYPE=TOS;

// 一条 BWMRULE 绑定三个时间段和三个控制器
// 时段1(8:30-10:00) -> bc2(中速), 时段2(22:00-23:00) -> bc3(高速), 其他时段 -> bc1(低速)
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ugroup8",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="my-bw-rule8",BWMSERVICENAME="my-bwm-service8",
  UPBWMCTRLNAME1="bc2",DNBWMCTRLNAME1="bc2",TIMERANGENAME1="timerange1",
  UPBWMCTRLNAME2="bc3",DNBWMCTRLNAME2="bc3",TIMERANGENAME2="timerange2",
  UPBWMCTRLNAME3="bc1",DNBWMCTRLNAME3="bc1",
  BWMRULEPRI=1;

ADD APNBINDBWMUSRG:USERGROUPNAME="ugroup8",APN="apn-test";
```

#### 场景七：基于用户/用户组/整机三级的带宽控制

**适用场景**：同时配置用户级、用户组级和整机级三层层次化带宽控制。

**关键配置差异**：
- 需要为三级分别配置 BWM Controller（带宽值递增：用户级 < 用户组级 < 整机级）。
- 用户级和用户组级用 ADD BWMRULE，整机级用 ADD BWMRULEGLOBAL。
- USERLEVSRVTYPE 和 USERGLEVSRVTYPE 可以不同（如用户级 NONTOS + 用户组级 TOS）。

#### 场景八：基于用户组的分级带宽管理控制（增强版）

**适用场景**：总带宽一定时实现差异化业务的分级管理：多个业务优先级共享带宽，高优先级业务优先获带宽，低优先级业务保基础带宽。

**关键配置差异**：
- BWMCONTROLLER 配置 SRVLEVELSPEC（业务等级规格）。
- 通过 ADD BCSRVLEVELPLY 为每个 ServiceLevel 配置 CIRRATE 和 PIRRATE。
- BWMRULE 中通过 SERVICELEVEL 指定业务级别。
- BWMUSERGROUP 中 USERGLEVRULESW=ENABLE 启用用户组级规则。

```mml
SET LICENSESWITCH:LICITEM="LKV3G5TCSA01",SWITCH=ENABLE;

// 三个不同 TOS 等级的 BWM 业务
ADD BWMSERVICE:BWMSERVICENAME="bs_test1",BWMSERVICETYPE=TOS,TOSCFGTYPE=CLASS,TOSCLASSTYPE=BE;
ADD BWMSERVICE:BWMSERVICENAME="bs_test2",BWMSERVICETYPE=TOS,TOSCFGTYPE=CLASS,TOSCLASSTYPE=AF1;
ADD BWMSERVICE:BWMSERVICENAME="bs_test3",BWMSERVICETYPE=TOS,TOSCFGTYPE=CLASS,TOSCLASSTYPE=AF2;

// 带宽控制器设置 SRVLEVELSPEC=3
ADD BWMCONTROLLER:BWMCNAME="bc_test",CTRLTYPE=CAR,CIR=300,CBS=60000,PIR=500,PBS=100000,GREEN=PASS,YELLOW=PASS,RED=DISCARD,SRVLEVELSPEC=3;

SET BANDWIDTHMNG:RANGE=SYSTEM;

// 业务级别策略：CIRRATE 之和 <= 100%
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=1,CIRRATE=50,PIRRATE=70;  // 高优先级
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=2,CIRRATE=30,PIRRATE=50;  // 中优先级
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=3,CIRRATE=20,PIRRATE=100; // 低优先级

ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",USERGROUPPRI=1,USERGLEVSRVTYPE=TOS,USERGLEVRULESW=ENABLE;

// 三条规则绑定不同 ServiceLevel
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="br_test1",BWMSERVICENAME="bs_test1",UPBWMCTRLNAME1="bc_test",DNBWMCTRLNAME1="bc_test",BWMRULEPRI=10,SERVICELEVEL=1;
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="br_test2",BWMSERVICENAME="bs_test2",UPBWMCTRLNAME1="bc_test",DNBWMCTRLNAME1="bc_test",BWMRULEPRI=15,SERVICELEVEL=2;

ADD APN:APN="apn-test",HASVPN=DISABLE,HASVPNIPV6=DISABLE;
ADD APNBINDBWMUSRG:USERGROUPNAME="ug_test",APN="apn-test";
```

### 多维度配置场景对比

| 场景 | 业务识别 | 规则类型 | 控制层级 | 特殊命令 |
|------|---------|---------|---------|---------|
| APN/DNN 用户组级 | 不区分业务 | GROUP_DEFAULT | 用户组 | ADD APNBINDBWMUSRG |
| PCC 预定义规则 | CategoryProp | SUBSCRIBER_SPECIFIC | 用户 | ADD CATEGORYPROP + ADD RULE |
| 切片 | Protocol/TOS | GROUP_SPECIFIC | 用户组 | ADD SNSSAI + SNSSAIBWMUSRG |
| 时间段 | TOS | GROUP_SPECIFIC | 用户组 | ADD TIMERANGE + PERITIMERANGE |
| 用户 TOS | TOS | SUBSCRIBER_SPECIFIC | 用户 | - |
| 用户组 TOS | TOS | GROUP_SPECIFIC | 用户组 | SET BANDWIDTHMNG |
| 三级控制 | NONTOS + TOS | SUBSCRIBER + GROUP + GLOBAL | 用户/组/整机 | ADD BWMRULEGLOBAL |
| 分级带宽控制 | TOS | GROUP_SPECIFIC | 用户组 | ADD BCSRVLEVELPLY |
| HTTP 用户限速 | Protocol | SUBSCRIBER_SPECIFIC | 用户 | - |

### 约束条件

1. **License 必须先开启**：执行 `SET LICENSESWITCH:LICITEM="LKV3G5TCSA01",SWITCH=ENABLE` 是第一步。
2. **CAR 参数推荐值**：CIR = 0.5 * PIR；CBS >= CIR * 1.5 * 1000 / 8；PBS >= PIR * 1.5 * 1000 / 8。
3. **APN/DNN 用户组级限制**：只支持 CAR 控制类型，只支持 NONTOS 业务类型，不关联 BWMSERVICE。
4. **切片场景**：BWMSUBSCRCTRL 必须设为 PCC_RULE_IGNORE。
5. **PCC 预定义规则**：CATEPROPNAME 与 RULE 的 POLICYNAME 必须一致；RULENAME 必须与 PCF/PCRF、UNC 规划一致。
6. **时间段控制**：一条 BWMRULE 可绑定最多 3 个时间段（TIMERANGENAME1/2/3），分别对应 3 组上/下行控制器。
7. **分级带宽控制**：所有 ServiceLevel 的 CIRRATE 之和不能超过 100%。
8. **COLORISAWARE**：ENABLE 时优先保证用户组内绿色用户流量，避免黄色流量抢占绿色流量。
9. **SHAPING 限制**：只有用户级带宽管理支持 SHAPING，用户组级和整机级不支持。
10. **SET REFRESHSRV**：涉及 Rule/L7Filter 变更时必须执行刷新命令使配置生效。
11. **三色策略**：GREEN 通常为 PASS，YELLOW 可为 PASS/REMARK，RED 通常为 DISCARD。

## 配置案例

### 案例一：APN/DNN 级带宽限制（基础版）

**需求**：UDG 对某个 APN 下的用户组进行整体流量监管，上行 CIR=2Gbps/PIR=4Gbps，下行同。

**完整 MML**：见 [场景一：基于 APN/DNN 的用户组级带宽控制](#场景一基于-apndnn-的用户组级带宽控制) 的脚本。

**验证要点**：
- APN 下用户的总流量不超过 PIR（4Gbps）。
- 总流量在 CIR（2Gbps）以下时保证通过。
- 超过 PIR 的报文被丢弃（RED=DISCARD）。

### 案例二：PCC 预定义规则 FUP 限速（增强版）

**需求**：用户购买 P2P 套餐，套餐耗尽前不限速；套餐耗尽后，PCF 通过下发预定义规则（rule_p2pbwm）触发带宽控制，限制上行 700kbps、下行 600kbps。

**完整 MML**：见 [场景二：基于 PCC 预定义规则的带宽控制](#场景二基于-pcc-预定义规则的带宽控制) 的脚本。

**工作流程**：
1. 套餐未耗尽：PCF 不安装 rule_p2pbwm，用户正常上网不限速。
2. 套餐耗尽：PCF 通过 N4 下发安装 rule_p2pbwm 的指令。
3. UDG 匹配到 rule_p2pbwm，通过 POLICYNAME=cp_700_600 关联到 BWM Service。
4. 执行 CAR 控制上行 700kbps、下行 600kbps。

### 案例三：时间段差异化带宽控制

**需求**：对 CS7 等级的 TOS 流量，在不同时段执行不同限速策略。

| 时段 | 时间范围 | CIR(kbps) | PIR(kbps) | 控制器 |
|------|---------|-----------|-----------|--------|
| 高峰时段 | 8:30-10:00 | 300 | 500 | bc2 |
| 低谷时段 | 22:00-23:00 | 3000 | 5000 | bc3 |
| 其他时段 | 其他 | 30 | 50 | bc1 |

**完整 MML**：见 [场景四：基于时间段的带宽控制](#场景四基于时间段的带宽控制) 的脚本。

## 验证与调测

### 验证方法

1. **License 验证**：确认 License 已开启。
2. **配置查询**：使用 `DSP SESSIONINFO` 查看用户上下文信息，确认带宽控制规则是否生效。
3. **流量统计验证**：发送流量测试，验证限速效果。
   - CIR 以下流量应正常通过（GREEN=PASS）。
   - CIR~PIR 之间流量按 YELLOW 策略处理（PASS 或 REMARK）。
   - PIR 以上流量按 RED 策略处理（DISCARD）。
4. **PCC 规则验证**（PCC 预定义规则场景）：确认 PCF 下发安装/删除预定义规则后，UDG 能正确匹配和执行。

### 常见问题与排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|---------|
| 带宽控制不生效 | License 未开启 | 执行 SET LICENSESWITCH 开启 LKV3G5TCSA01 |
| Rule 不生效 | 未执行 SET REFRESHSRV | 配置完成后执行 SET REFRESHSRV:REFRESHTYPE=USERPROFILE |
| 切片带宽控制不生效 | BWMSUBSCRCTRL 未设为 PCC_RULE_IGNORE | SET BANDWIDTHMNG 中设置 BWMSUBSCRCTRL=PCC_RULE_IGNORE |
| PCC 预定义规则不生效 | CATEPROPNAME 与 POLICYNAME 不一致 | 确保 ADD BWMSERVICE 的 CATEPROPNAME 与 ADD RULE 的 POLICYNAME 一致 |
| 用户组级控制不准确 | RANGE 配置不当 | 检查 SET BANDWIDTHMNG 的 RANGE（CPU 或 SYSTEM） |
| 分级带宽控制不生效 | CIRRATE 之和超过 100% | 检查所有 ADD BCSRVLEVELPLY 的 CIRRATE，确保之和 <= 100% |
| SHAPING 不支持 | 非用户级场景使用了 SHAPING | SHAPING 仅支持用户级，用户组级和整机级使用 CAR |
| 时间段切换异常 | TIMERANGE/PERITIMERANGE 配置有误 | 检查时间段定义和周期时间段配置 |
| 组内黄色流量抢占绿色流量 | COLORISAWARE 未开启 | ADD BWMCONTROLLER 中设置 COLORISAWARE=ENABLE |

## 参考信息

### 接口与信元

本特性通过 PGW-C/SMF 与 PGW-U/UPF 之间的 **N4 接口**（5G）或 **Sxa 接口**（4G）下发带宽控制规则。PCF/PCRF 通过 Gx/N7 接口向 SMF 下发 PCC 规则。
