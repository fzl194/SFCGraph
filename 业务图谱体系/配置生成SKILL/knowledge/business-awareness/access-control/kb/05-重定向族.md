# 05 · 重定向族（SMARTREDIRECT 子轨 + WEBPROXY 子轨 + Portal）

> **定位**：详解 4 种重定向实现（HTTP 重定向 / DNS 纠错 / Portal captive / WebProxy）的配置差异与协议限制。
> **数据来源**：`访问限制场景/feature-knowledge/GWFD-110281-用户Portal.md` / `GWFD-110282-WebProxy.md` / `GWFD-110283-DNS纠错.md` / `GWFD-110284-HTTP智能重定向.md`、`04-command-graph.md`（§2.7 SMARTREDIRECT、§2.8 WEBPROXY/Portal、§3.4/3.5 关系边、§5.3/5.4/5.5 参数）。
> **关联文件**：`02-双轨道动作机制与POLICYTYPE路由.md`（POLICYTYPE=SMARTREDIRECT/WEBPROXY 路由）。

## 1. 重定向族 4 种实现对比（CS-AC-03/04/05）

| 维度 | HTTP 智能重定向（110284） | DNS 纠错（110283） | 用户 Portal（110281） | WebProxy（110282） |
|------|--------------------------|------------------|---------------------|-------------------|
| 方案 | CS-AC-03 | CS-AC-04 | CS-AC-05 Portal | CS-AC-05 WebProxy |
| 协议层 | L7 HTTP 响应改写 | DNS 解析层重写 | L7 HTTP 响应改写 + captive | **L3 IP NAT**（透明代理） |
| POLICYTYPE | SMARTREDIRECT | SMARTREDIRECT | PCC（captive 在 USERPROFILE） | **WEBPROXY** |
| POLICYNAME 指向 | SMARTHTTPREDIR | DNSOVERWRITING | PCCPOLICYGRP | IPFARM |
| 触发条件 | EXTENDEDFILTER + ERRORCODE(HTTP GT400) | EXTENDEDFILTER + ERRORCODE(DNS NXDOMAIN=3) | L7FILTER（首次 HTTP 请求） | FILTER（TCP SYN，L3 不依赖 L7） |
| 重定向目标 | SERVERURL（第三方服务器/内容过滤页） | SERVERIP1（第三方 Platform IP） | Portal Server 集群（IPFarm） | Proxy Server 集群（IPFarm） |
| 携带信息 | REDIRAPPENDINFO（MSISDN/IMSI/IMEI，可选） | — | — | — |
| IPFarm 集群 | 否 | 否 | **是**（Portal Server，心跳检测） | **是**（Proxy Server，心跳检测 + 黑名单） |
| captive 周期 | 否 | 否 | **USERPROFILE.CAPMODETHRES**（如 6 分钟） | 否 |
| ★ 协议支持 | **仅 HTTP1.x**，不支持 HTTPS/HTTP2.0 | **仅 UDP DNS**，不支持 TCP DNS | **仅 HTTP1.x/WAP** | **★ 唯一支持 HTTPS/HTTP2.0/任意 TCP** |
| 兜底机制 | — | — | IPFarm 全 DOWN 时 **DEFAULTACT=BLOCK** | 服务器 DOWN 丢包 |
| 介入时机 | L7（已建 TCP） | DNS 层（最早，未建 TCP） | L7（首次访问） | L3（TCP SYN） |

## 2. SMARTREDIRECT 子轨配置对象（CR-AC-03）

### 2.1 SMARTHTTPREDIR（HTTP 智能重定向，OBJ-AC-SMARTHTTPREDIR）

| 参数 | 类型 | 取值范围 | 说明 |
|------|------|---------|------|
| SMTHTTPREDINAME | string | — | HTTP 智能重定向动作策略名（RULE.POLICYNAME 指向此） |
| SERVERURL | string | — | 重定向目标 URL |
| EXTFLTTYPE1/2 | enum | AND / OR | 扩展过滤器组合逻辑 |
| EXTFLTNAME1/2 | string | — | 引用 EXTENDEDFILTER（URL/UserAgent/ContentType 多维） |
| APPENDINFONAME | string | — | 引用 REDIRAPPENDINFO（携带 MSISDN/IMSI/IMEI） |
| BINDErrCODENAME | string | — | 引用 ERRORCODE（HTTP 错误码 GT 400 触发） |

### 2.2 REDIRAPPENDINFO（重定向携带信息，OBJ-AC-REDIRAPPENDINFO）

| 参数 | 类型 | 说明 |
|------|------|------|
| APPENDINFONAME | string | 携带信息对象名 |
| REQURLFLAG | enum | 是否携带原请求 URL |
| IMSIFLAG | enum | 是否携带 IMSI |
| IMEIFLAG | enum | 是否携带 IMEI |

> HTTP 智能重定向独有；DNS 纠错不携带信息。

### 2.3 DNSOVERWRITING（DNS 重写，OBJ-AC-DNSOVERWRITING）

| 参数 | 类型 | 取值范围 | 说明 |
|------|------|---------|------|
| DNSOVERWRTNAME | string | — | DNS 重写动作名（RULE.POLICYNAME 指向此，与 SMARTHTTPREDIR 共用 POLICYTYPE=SMARTREDIRECT） |
| EXTFLTTYPE1 | enum | AND | 扩展过滤器逻辑 |
| EXTFLTNAME1 | string | — | 引用 EXTENDEDFILTER（错误域名匹配） |
| SERVERIP1 | string | IP | 第三方 Platform IP（DNS 响应改写目标） |
| BINDERRCODENAME | string | — | 引用 ERRORCODE（DNS 错误码 EQUAL 3 NXDOMAIN 触发） |

## 3. WEBPROXY 子轨 + Portal：IPFarm 集群（CR-AC-10）

### 3.1 IPFARMGLOBAL（全局参数，OBJ-AC-IPFARMGLOBAL）

| 参数 | 类型 | 取值范围 | 说明 |
|------|------|---------|------|
| SERVERTYPE | enum | — | 服务器类型 |
| LBMETHOD | enum | ROUND_ROBIN / LEAST_RECENTLY_USED / LEAST_LOAD | 负荷分担方式（默认 LEAST_LOAD） |
| TIMETHRESHOLD | int | — | 时间阈值 |
| HEALTHSUCCLIMIT | int | — | 心跳成功阈值（UP↔DOWN 切换） |
| HEALTHFAILLIMIT | int | — | 心跳失败阈值 |

### 3.2 IPFARM（重定向服务器集群，OBJ-AC-IPFARM）

| 参数 | 类型 | 取值范围 | 说明 |
|------|------|---------|------|
| IPFARMNAME | string | — | IPFarm 名（RULE.POLICYTYPE=WEBPROXY 时 POLICYNAME 指向此） |
| IPVERSION | enum | — | IP 版本 |
| INTERFACENAME | string | — | 心跳检测接口（不同 IPFarm 必须用不同接口） |
| HEALTHCHECKFLAG | enum | ENABLE / DISABLE | 心跳检测开关 |
| DEFAULTACT | enum | BLOCK / (放行) | Portal 全 DOWN 时缺省动作=BLOCK（兜底阻塞） |

> 规格：整机 64 个 IPFarm，每个最多 512 个 IP。

### 3.3 IPFARMSERVER（IPFarm 服务器，OBJ-AC-IPFARMSERVER）

| 参数 | 类型 | 说明 |
|------|------|------|
| IPFARMNAME | string | 归属 IPFarm |
| SERVERIPV4 / SERVERIPV6 | string | Portal Server / Proxy Server IP |

### 3.4 LOGICINF（逻辑接口，OBJ-AC-LOGICINF）

| 参数 | 类型 | 说明 |
|------|------|------|
| NAME | string | 逻辑接口名 |
| IPVERSION | enum | IP 版本 |
| IPV4ADDRESS1 / IPV4MASK1 | string | IPv4 地址/掩码 |
| VPNINSTANCE | string | VPN 实例（ICAP 互通时引用） |

> LOGICINF 双用途：心跳检测（Portal/WebProxy）+ ICAP 互通（URL 过滤）共用对象类；不同 IPFarm 必须用不同接口。

### 3.5 BLACKLISTRULE（WebProxy 黑名单，OBJ-AC-BLACKLISTRULE）

WebProxy 独有黑名单匹配规则。

## 5. ★ Portal captive 配置位置（TR-AC-07）

Portal 的 captive 周期配置在 **USERPROFILE.CAPMODETHRES**，**不在 RULE**（与其他重定向族关键差异）：

```mml
ADD RULE:RULENAME="{rule_portal}", POLICYTYPE=PCC, ..., POLICYNAME="{ppg_name}";
ADD USERPROFILE:USERPROFILENAME="{up_portal}", CAPMODETHRES=6;  (★ captive 周期 6 分钟)
ADD RULEBINDING:USERPROFILENAME="{up_portal}", RULENAME="{rule_portal}";
```

## 6. ★ WebProxy 双规则约束（TR-AC-06）

WebProxy 必须同时配置**两条 RULE**：

```mml
!-- 规则 1：WebProxy 重定向（POLICYTYPE=WEBPROXY，POLICYNAME=IPFarm）
ADD RULE:RULENAME="{rule_webproxy}", POLICYTYPE=WEBPROXY, PRIORITY={p1}, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="{ff_name}", POLICYNAME="{ipfarm_name}";

!-- 规则 2：PCC 计费（POLICYTYPE=PCC，POLICYNAME=PCCPOLICYGRP）
ADD RULE:RULENAME="{rule_pcc}", POLICYTYPE=PCC, PRIORITY={p2}, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="{ff_name}", POLICYNAME="{ppg_name}";
```

**约束**：
- 两条 RULE 共用同一 FLOWFILTER
- RULENAME 不能同名（CR-AC-01）
- POLICYTYPE 不同（WEBPROXY vs PCC）

## 7. 协议盲区与优势（BR-AC-09, CR-AC-10）

| 重定向类型 | HTTP1.x | HTTPS / HTTP2.0 | 原因 |
|-----------|---------|-----------------|------|
| HTTP 智能重定向 | 支持 | **不支持** | 加密无法获取 HTTP 响应特征 |
| DNS 纠错 | 支持（UDP DNS） | 不适用（DNS 独立） | 仅 UDP DNS，不支持 TCP DNS |
| Portal | 支持 | **不支持** | 加密无法改写 HTTP 响应 |
| **WebProxy** | 支持 | **★ 支持** | L3 IP NAT 不依赖 L7 解析 |
| HTTPS 头增强 | 不适用 | 部分支持（SSL Extension TLV） | 仅基于 SNI 触发 |

> **加密协议场景（HTTPS/HTTP2.0）的唯一选择**：WebProxy（BR-AC-09）。

## 8. 配置链示例

### 8.1 HTTP 智能重定向完整链

```mml
ADD EXTENDEDFILTER:EXTFLTNAME="{extflt_url}", URL="{target_domain}", USERAGENT="{...}";
ADD ERRORCODE:ERRORCODENAME="{err_400}", ERRORCODEOP=GT, ERRORCODESTART=400;
ADD REDIRAPPENDINFO:APPENDINFONAME="{append_msisdn}", IMSIFLAG=ENABLE;
ADD SMARTHTTPREDIR:SMTHTTPREDINAME="{redir_http}", SERVERURL="{target_url}", EXTFLTTYPE1=AND, EXTFLTNAME1="{extflt_url}", APPENDINFONAME="{append_msisdn}", BINDErrCODENAME="{err_400}";
ADD FILTER / L7FILTER / FLOWFILTER / FLTBINDFLOWF / PROTBINDFLOWF ...
ADD RULE:RULENAME="{rule_http_redir}", POLICYTYPE=SMARTREDIRECT, PRIORITY={p}, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="{ff}", POLICYNAME="{redir_http}";
ADD USERPROFILE / ADD RULEBINDING ...
SET REFRESHSRV:REFRESHTYPE=ALL;
```

### 8.2 WebProxy 完整链（加密协议场景）

```mml
SET IPFARMGLOBAL:SERVERTYPE={...}, LBMETHOD=LEAST_LOAD, HEALTHSUCCLIMIT={...}, HEALTHFAILLIMIT={...};
ADD LOGICINF:NAME="{logicinf_1}", IPVERSION={...}, IPV4ADDRESS1="{...}", IPV4MASK1="{...}";
ADD IPFARM:IPFARMNAME="{ipfarm_proxy}", IPVERSION={...}, INTERFACENAME="{logicinf_1}", HEALTHCHECKFLAG=ENABLE;
ADD IPFARMSERVER:IPFARMNAME="{ipfarm_proxy}", SERVERIPV4="{proxy_ip}";
ADD BLACKLISTRULE: ...;
ADD FILTER:FILTERNAME="{filter_l3}", ...;  (L3 TCP SYN)
ADD FLOWFILTER / ADD FLTBINDFLOWF ...
ADD URR / ADD URRGROUP / ADD PCCPOLICYGRP ...
ADD RULE:RULENAME="{rule_webproxy}", POLICYTYPE=WEBPROXY, ..., POLICYNAME="{ipfarm_proxy}";
ADD RULE:RULENAME="{rule_pcc}", POLICYTYPE=PCC, ..., POLICYNAME="{ppg}";  (★ 双规则)
ADD USERPROFILE / ADD RULEBINDING / ADD APN ...
SET REFRESHSRV:REFRESHTYPE=ALL;
```

## 9. 关键规则索引

| 规则 ID | 规则名 | 核查要点 |
|---------|--------|---------|
| BR-AC-09 | 加密协议仅 WebProxy 可处理 | HTTPS/HTTP2.0 场景选错重定向方式导致失效 |
| CR-AC-03 | SMARTREDIRECT 两特性共用 POLICYTYPE | POLICYNAME 指向区分 HTTP 重定向 vs DNS 纠错 |
| CR-AC-10 | HTTPS/HTTP2.0 加密协议盲区 | HTTP 重定向/Portal 仅 HTTP1.x |
| TR-AC-06 | WebProxy 双规则约束 | 必须同时配 WEBPROXY + PCC 两条 RULE |
| TR-AC-07 | Portal captive 配置位置约束 | CAPMODETHRES 在 USERPROFILE 不在 RULE |
