## 1. 访问限制核心业务语义：三大动作 + PERMIT

访问限制场景的核心业务语义是 **DISCARD（阻塞）/ HEADEN（头增强）/ REDIRECT（重定向）** 三大动作，加上 URL 过滤特有的 **PERMIT（放行）**。

| 动作类型 | 直接产生此动作的特性 | 实现机制 |
|---------|---------------------|---------|
| **DISCARD（阻塞）** | ADC（兜底阻塞）、URL 过滤（CFTEMPLATE.ACTION=BLOCK）、Portal（IPFarm 全 DOWN 时 DEFAULTACT=BLOCK） | 轨道 A 隐式阻塞（规则匹配不上丢弃）；轨道 B 显式 BLOCK |
| **HEADEN（头增强）** | HTTP/HTTPS/RTSP 头增强族 | 轨道 A；POLICYTYPE=HEADEN 引用 HEADEN 对象 |
| **REDIRECT（重定向）** | Portal、WebProxy、DNS 纠错、HTTP 智能重定向、URL 过滤（CFTEMPLATE.ACTION=REDIRECT） | 4 种协议层实现（DNS / L3-IPNAT / L7-HTTP 响应 / L7-Portal） + URL 过滤轨道 B |
| **PERMIT（放行）** | URL 过滤（CFTEMPLATE.ACTION=PERMIT） | **仅 URL 过滤显式支持 PERMIT**（轨道 A 特性主要是"不做动作"） |

## 2. ★ 双轨道+五子轨架构（本场景核心，区别于计费单轨 CHARGING）

### 2.1 轨道定义

| 轨道 | 驱动核心 | 动作指定方式 | 涉及特性 |
|------|---------|-------------|----------|
| **轨道 A：PCC 体系** | RULE.POLICYTYPE | 隐式（POLICYTYPE 决定动作） | ADC、头增强族、头防欺诈、Portal、WebProxy、DNS 纠错、HTTP 智能重定向 |
| **轨道 B：URL 过滤体系** | CFTEMPLATE.ACTION / CONTCATEGBIND.ACTION | 显式（BLOCK/PERMIT/REDIRECT） | URL 过滤 |

### 2.2 轨道 A 内五子轨（POLICYTYPE 枚举）

POLICYTYPE 是轨道 A 的动作总开关，5 个枚举值对应 5 个子轨：

| POLICYTYPE | 子轨 | POLICYNAME 指向 | 动作语义 | 涉及方案 |
|-----------|------|----------------|---------|---------|
| **PCC** | PCC 子轨 | PCCPOLICYGRP | 标准 PCC（Portal 计费 / 阻塞 / URL 过滤触发） | CS-AC-01 / CS-AC-05 Portal / CS-AC-06 URL 过滤触发 / CS-AC-08 / CS-AC-09 |
| **ADC** | ADC 子轨 | (直接在 RULE) + ADCPARA | 应用检测（兜底阻塞） | ADC 兜底阻塞（CS-AC-01 衍生） |
| **HEADEN** | HEADEN 子轨 | HEADEN 对象 | 头增强（字段插入，含防欺诈子模式） | CS-AC-02 |
| **SMARTREDIRECT** | SMARTREDIRECT 子轨 | SMARTHTTPREDIR / DNSOVERWRITING | HTTP 重定向 / DNS 纠错（共用 POLICYTYPE，区分点在 POLICYNAME 指向） | CS-AC-03 / CS-AC-04 |
| **WEBPROXY** | WEBPROXY 子轨 | IPFARM | Web Proxy（L3 IP NAT，唯一处理加密协议） | CS-AC-05 WebProxy |

### 2.3 轨道 B 独立动作体系

| 动作指定 | 配置对象 | 动作枚举 | 特点 |
|---------|---------|---------|------|
| **模板级缺省动作** | CFTEMPLATE.ACTION | BLOCK / PERMIT / REDIRECT | 模板粒度，PERMIT 仅轨道 B |
| **分类级精确动作** | CONTCATEGBIND.ACTION | BLOCK / PERMIT / REDIRECT | 覆盖模板级缺省，基于 CATEGORYID 精确匹配 |

> **URL 过滤 RULE 用 POLICYTYPE=PCC 触发**，但实际 BLOCK/PERMIT/REDIRECT 动作**不走 PCCPOLICYGRP**，而走 CFTEMPLATE/CONTCATEGBIND（轨道 A→B 混合，建模难点，CR-AC-02 / TR-AC-03）。

## 3. 与计费/带宽场景的差异

| 维度 | 计费场景 | 带宽控制场景 | 访问限制场景 |
|------|---------|------------|------------|
| POLICYTYPE 枚举 | CHARGING（固定） | BWM/PCC/QOS/ADC（按策略分支） | **ADC/PCC/HEADEN/SMARTREDIRECT/WEBPROXY**（轨道 A 五子轨，最丰富） |
| 动作机制 | 单轨（CHARGING） | 单轨（POLICYTYPE 分支） | **双轨道+五子轨**（轨道 A POLICYTYPE 隐式 + 轨道 B CFTEMPLATE.ACTION 显式） |
| PERMIT 支持 | 不适用（计费无放行概念） | 不显式支持 | **仅轨道 B 显式支持 PERMIT**（BR-AC-10, CR-AC-06） |
| 独有对象族 | 计费三件套（URR/URRGROUP/PCCPOLICYGRP） | BWM 三级控制 | 重定向族（SMARTHTTPREDIR/DNSOVERWRITING/IPFarm）、头增强族（HEADEN）、URL 过滤（CFTEMPLATE/CONTCATEGBIND/ICAP*）、接入控制（USRLOCATION） |
| ConfigurationSolution | 7 个 | 7 个 | **9 个**（按动作类型×轨道×触发维度分，最多） |

## 4. 双轨并存（CR-AC-14）

同一用户业务流可先后被轨道 A（PCC 体系）和轨道 B（URL 过滤体系）检查：
- 轨道 A 处理 ADC / 头增强 / 重定向
- 轨道 B 处理 URL 分类过滤
- 最终动作取决于配置优先级和规则匹配顺序
- 双轨优先级需查参考信息或现网验证（已知建模难点）

**典型双轨并存场景**：头增强（轨道 A · HEADEN 子轨） + URL 过滤（轨道 B），同一报文可同时命中并执行两个动作（BR-AC-02 规则匹配类型独立原则）。

## 5. 9 方案闭包总览（按动作类型×轨道×触发维度）

| 方案 ID | 方案名 | 动作类型 | 动作轨道 | core_mechanism_combo |
|---------|--------|---------|---------|---------------------|
| CS-AC-01 | PCC 阻塞 | DISCARD | 轨道 A · PCC 子轨 | FILTER/FLOWFILTER → RULE(PCC) → PCCPOLICYGRP → UDG DISCARD |
| CS-AC-02 | 头增强 | HEADEN | 轨道 A · HEADEN 子轨 | FILTER/L7FILTER → HEADEN → RULE(HEADEN) → UDG 插入字段 |
| CS-AC-03 | HTTP 重定向 | REDIRECT | 轨道 A · SMARTREDIRECT 子轨 | EXTENDEDFILTER+ERRORCODE → SMARTHTTPREDIR → RULE(SMARTREDIRECT) → UDG 改写 HTTP 响应 |
| CS-AC-04 | DNS 重定向 | REDIRECT | 轨道 A · SMARTREDIRECT 子轨 | EXTENDEDFILTER+ERRORCODE → DNSOVERWRITING → RULE(SMARTREDIRECT) → UDG 改写 DNS 响应 |
| CS-AC-05 | Portal/WebProxy 重定向 | REDIRECT | 轨道 A · PCC/WEBPROXY 子轨 | IPFarm 集群 → PCCPOLICYGRP(USERPROFILE CAPMODETHRES) / RULE(WEBPROXY) |
| CS-AC-06 | URL 过滤 | BLOCK/PERMIT/REDIRECT | **轨道 B** | ICAP 链 + CFTEMPLATE/CONTCATEGBIND.ACTION → UDG 执行 |
| CS-AC-07 | 接入控制 | (粗粒度准入) | UNC 侧 | SAR/区域/ODB + USRLOCATION+USRLOCATIONGRP+UPBINDUPG |
| CS-AC-08 | 配额耗尽重定向 | REDIRECT | 轨道 A · PCC 子轨 + PCF 决策 | URR 配额 → US_RE/NO_CREDIT → PCF 下发 RedirectInformation → UPF 执行 |
| CS-AC-09 | 区域引导重定向 | REDIRECT | 轨道 A · PCC 子轨 + 位置触发 | WSFD-211001 + UPCF 触发器(PRA/PLMN/SAREA) → PCF → SMF → UPF |
