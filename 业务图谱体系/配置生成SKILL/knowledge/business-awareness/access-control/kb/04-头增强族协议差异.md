# 04 · 头增强族协议差异（HEADEN 子轨）

> **定位**：详解 HEADEN 对象跨 HTTP/HTTPS/RTSP 三协议的配置差异与防欺诈子模式。
> **数据来源**：`访问限制场景/feature-knowledge/GWFD-110261-HTTP头增强.md` / `GWFD-110262-RTSP头增强.md` / `GWFD-110263-HTTPS头增强.md` / `GWFD-110401-HTTP头防欺诈.md`、`04-command-graph.md`（§2.6 OBJ-AC-HEADEN、CR-AC-04/11、§5.2 ADD HEADEN）。
> **关联文件**：`02-双轨道动作机制与POLICYTYPE路由.md`（POLICYTYPE=HEADEN 路由）。

---

## 1. HEADEN 对象跨协议复用（发现五）

HEADEN 是协议无关的统一头增强对象，跨 4 个特性复用：
- GWFD-110261 HTTP 头增强（协议：HTTP1.x）
- GWFD-110262 RTSP 头增强（协议：RTSP）
- GWFD-110263 HTTPS 头增强（协议：HTTPS，TLS 1.2/1.3）
- GWFD-110401 HTTP 头防欺诈（内嵌于 HEADEN，不独立产生动作）

RULE.POLICYTYPE=HEADEN，POLICYNAME 指向 HEADEN 对象名（统一入口）。

---

## 2. ADD HEADEN 关键参数（CR-AC-04，§5.2）

| 参数 | 类型 | 取值范围 | 说明 |
|------|------|---------|------|
| HEADERENNAME | string | — | 头增强对象名 |
| DATATYPE | enum | IMSI1 / MSISDN1 / IMEI1 / APN / ULI / MULTIPARA 等 25 类 | 插入的用户信息字段类型 |
| PREFIXNAME | string | — | 字段前缀名（建议非 HTTP 标准头，如 X-msisdn / X-imsi，避免防欺诈失败） |
| ENCRYALGORI | enum | MD5 / RC4 / AES128 / AES256 / RSA1024 / RSA2048 / SHA256 | 加密算法（**HTTPS 头增强无 RSA**） |
| PSWDKEY / PSWDKEYCONFIRM | string | — | 密钥（两字段必须一致） |
| **ANTIFRAUD** | enum | **ENABLE / DISABLE** | **使能 HTTP 头防欺诈**（内嵌前置检测；CR-AC-04） |
| GRAYLIST | enum | ENABLE / DISABLE | 灰名单模式（跳过头增强插入，仅防欺诈） |

## 3. ★ 协议差异对比（BR-AC-08, CR-AC-10/11）

| 维度 | HTTP 头增强（110261） | HTTPS 头增强（110263） | RTSP 头增强（110262） |
|------|---------------------|----------------------|---------------------|
| 协议支持 | **仅 HTTP1.x**，不支持 HTTPS/HTTP2.0 | HTTPS（TLS 1.2/1.3），通过 SSL Extension 部分支持 | RTSP |
| ★ ENCRYALGORI 限制 | MD5/RC4/AES-128/256/RSA-1024/2048/SHA-256 全支持 | **无 RSA** | MD5/RC4/AES-128/256/SHA-256（视版本） |
| ★ 字段插入格式 | 扩展字段（HTTP 头） | **TLS TLV 格式插入 SSL Extension** | 扩展字段 |
| ★ 触发方式 | 基于特定 IP/URL/HTTP 协议 | **仅基于 SNI** | 基于 RTSP 协议 |
| 编码方式 | base64 + ASCII | base64 + ASCII + **十六进制**（额外） | base64 + ASCII |
| ★ 防欺诈支持（ANTIFRAUD） | **支持**（ENABLE 需 License 双开） | **支持**（同 HTTP） | **不支持**（族内唯一例外，CR-AC-11） |
| 知名端口 | WELLKNOWNPORT 80/8080 | 443 | 554 |

## 4. ★ 头防欺诈子模式（BR-AC-06, CR-AC-04, TR-AC-04）

### 4.1 强耦合依赖

头防欺诈（GWFD-110401）完全**寄生**头增强配置链路，无独立 MML：
- 通过 ADD HEADEN 的 **ANTIFRAUD=ENABLE** 参数内嵌
- 启用防欺诈必须同时开启头增强 License（双开）：
  - HHAS License：`LKV3G5HHAS01`（82209786）
  - HTHE License：`LKV3G5HTHE01`（82209777）
- RTSP 头增强不支持防欺诈（CR-AC-11，族内唯一例外）

### 4.2 执行顺序（头防欺诈检测流程）

```
1. 防欺诈检测：检查已有字段是否伪造
   ├─ 不存在字段：继续头增强
   ├─ 字段正确：跳过头增强（已存在合法字段）
   ├─ 字段错误：纠正后继续（或灰名单模式停止）
   └─ 多个字段：清理冗余后继续
2. 字段纠正/冗余清理（GRAYLIST=ENABLE 时仅此步，不插入）
3. 头增强插入：插入 MSISDN/IMSI/IMEI 等字段
```

### 4.3 GRAYLIST 灰名单模式

- GRAYLIST=ENABLE：只执行防欺诈检测与纠正，**跳过头增强插入**
- GRAYLIST=DISABLE：完整执行防欺诈 + 头增强插入

## 5. 配置链示例（HTTP 头增强 + 防欺诈）

```mml
!-- 0. SA 前置（仅首次）
LOD SIGNATUREDB: ...;
LOD PARSERDB: ...;
ADD WELLKNOWNPORT:IDENPROTNAME="{...}", PROTOCOLNAME="http", PORTOP=EQUAL, STARTPORT=80;

!-- 1. 底层过滤
ADD FILTER:FILTERNAME="{filter_http}", ...;
ADD L7FILTER:L7FILTERNAME="{l7_http}", URL="{...}", METHODTYPE={...};
ADD FLOWFILTER:FLOWFILTERNAME="{ff_http}";
ADD FLTBINDFLOWF:FLOWFILTERNAME="{ff_http}", FILTERNAME="{filter_http}";
ADD PROTBINDFLOWF:FLOWFILTERNAME="{ff_http}", PROTOCOLNAME="http", L7FILTERNAME="{l7_http}";

!-- 2. 头增强对象（含防欺诈子模式）
ADD HEADEN:HEADERENNAME="{headen_http_msisdn}", DATATYPE=MSISDN1, PREFIXNAME="X-msisdn", ENCRYALGORI=SHA256, PSWDKEY="{key}", PSWDKEYCONFIRM="{key}", ANTIFRAUD=ENABLE, GRAYLIST=DISABLE;
SET BASE64: ..., ENABLE;  (base64 编码开关)

!-- 3. 头增强规则
ADD RULE:RULENAME="{rule_headen_http}", POLICYTYPE=HEADEN, PRIORITY={priority}, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="{ff_http}", POLICYNAME="{headen_http_msisdn}";

!-- 4. 容器与绑定 + 刷新
ADD USERPROFILE:USERPROFILENAME="{up_headen}";
ADD RULEBINDING:USERPROFILENAME="{up_headen}", RULENAME="{rule_headen_http}";
SET REFRESHSRV:REFRESHTYPE=ALL;
```

## 6. 应用限制（来自 GWFD-110261 §1.9）

| # | 限制项 | 说明 |
|---|--------|------|
| 1 | METHODTYPE 限制 | 若整机未配置 METHODTYPE，只支持默认（GET/POST/CONNECT）；软参 BIT316 控制扩展 |
| 2 | IPv6 分片 | IPv6 不支持中间路由器分片，需限制插入字段长度（建议 TCP MSS ≤ PMTU - 40B - 20B - 字段长度） |
| 3 | MTU 过小分片 | 用户报文到达 UDG 前被分片，UDG **不对分片报文**做头增强 |
| 4 | HTTP 头域分片 | HTTP 头域分片的请求报文不支持头增强 |
| 5 | 外置 PCEF 计费误差 | 头增强改变报文长度，导致外置 PCEF 计费误差；建议头增强与计费部署在同一网元 |
| 6 | 协议版本 | HTTP 头增强**只支持 HTTP1.x，不支持 HTTP2.0** |
| 7 | 加密场景计费 | 不支持加密场景基于 host 的计费和控制 |
| 8 | HTTPS 协议 | HTTP 头增强**不支持 HTTPS**（需用 HTTPS 头增强 GWFD-110263） |
| 9 | 拼接字段数量 | 最多 3 个拼接字段，每字段最多 4 参数拼接（MULTIPARA，仅 MSISDN/SGSNIP/MSIP） |
