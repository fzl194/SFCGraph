# GWFD-010171 离线计费 知识文档

## 概述

离线计费（Offline Charging）是一种**非实时**的计费方式。UDG（SGW-U/PGW-U）收集移动用户的计费信息（如时长、流量、事件等），上报给UNC（SGW-C/PGW-C），UNC将这些信息格式化为话单，通过**Ga接口**将话单消息发送到CG（Charging Gateway）。CG完成话单的存储、标准化等预处理工作，并将标准计费数据送到BS（Billing System）作最终处理。

- **适用NF**：SGW-U、PGW-U
- **支持版本**：UDG 20.0.0及后续版本、UNC 20.0.0及后续版本
- **License**：无需License许可
- **特性首次发布版本**：20.0.0

### 客户价值

| 受益方 | 受益描述 |
|--------|----------|
| 运营商 | 根据用户使用数据业务的信息进行精确收费；为运营商之间、运营商与ISP之间的结算提供依据 |
| 用户 | 记录用户使用数据业务的详细情况，便于核查消费情况 |

### 遵循标准

| 标准编号 | 标准名称 |
|----------|----------|
| 3GPP 23.502 | Procedures for the 5G System; Stage 2 (Release 15) |
| 3GPP 29.244 | Interface between the Control Plane and the User Plane Nodes; Stage 3 |

## 核心概念

### 离线计费 vs 在线计费

| 维度 | 离线计费 | 在线计费 |
|------|----------|----------|
| 实时性 | 非实时 | 实时 |
| 接口 | Ga接口（UNC → CG） | Gy接口（PGW-C → OCS） |
| 配额管理 | 不需要申请配额 | 需要向OCS申请配额 |
| 话单生成 | UNC生成原始话单，发送到CG | OCS实时跟踪配额使用 |
| 计费统计 | 时长、流量、事件默认均开启 | 取决于OCS下发的配额类型 |
| 适用场景 | 后付费用户、结算 | 预付费用户、实时控制 |

### 涉及网元与职责

| 网元 | 职责 |
|------|------|
| SGW-C/PGW-C（UNC） | 通过N4接口下发URR信元，指示计费原则；按照计费规则生成原始话单，通过Ga接口发送到CG |
| SGW-U/PGW-U（UDG） | 收集用户使用网络、业务的资源信息（时长、流量等），通过N4接口上报给SGW-C/PGW-C |
| CG（Charging Gateway） | 对UNC发送来的话单进行存储、合并、标准化预处理、增加运营商自定义字段，将标准计费数据发送到BS |
| BS（Billing System） | 处理CG发送来的计费数据，生成最终用户账单 |

### URR（Usage Reporting Rule）

URR是离线计费的核心配置对象，定义了使用量上报规则。

**关键参数：**

| 参数 | 含义 | 说明 |
|------|------|------|
| URRNAME | 使用量上报规则名称 | 字符串，1~31字符，不支持空格 |
| URRID | URR标识 | 整数，0~2147483646，全网唯一，建议从1000开始分配 |
| USAGERPTMODE | 使用量上报模式 | OFFLINE / ONLINE / MONITORINGKEY / QOS |
| OFFMETERINGTYPE | 离线计费统计类型 | 仅USAGERPTMODE=OFFLINE时有效 |
| RG | 计费组 | 可选，报表功能开启时需配置，需与SMF保持一致 |
| SID | 业务标识 | 可选，报表功能开启时需配置，需与SMF保持一致 |

**USAGERPTMODE枚举值：**

| 取值 | 含义 |
|------|------|
| OFFLINE | 离线计费URR |
| ONLINE | 在线计费URR |
| MONITORINGKEY | 流量累计Monitoring-Key |
| QOS | QoS保证 |

### OFFMETERINGTYPE（离线计费统计类型）

**前提条件**：USAGERPTMODE=OFFLINE时可选配置。

| 取值 | 含义 | 说明 |
|------|------|------|
| VOLUME | 按流量计费 | **默认值** |
| TIME | 按时长计费 | - |
| EVENT | 按事件计费 | 对性能有一定影响 |
| FREE | 免费 | 表示该URR有计费欺诈风险，需要做计费防欺诈判断。要实现流量免费需配合UNC配置 |
| VOLUME_TIME | 按流量和时长计费 | - |
| EVENT_VOLUME | 按事件和流量计费 | - |
| EVENT_TIME | 按事件和时长计费 | - |
| EVENT_VOLUME_TIME | 按事件、流量和时长计费 | 全量统计 |

**性能注意**：开启事件计费（EVENT相关取值）时，对性能会产生一定影响，执行命令前需评估对性能的影响。

**默认值**：VOLUME（按流量计费）。

### 离线计费方式的选择机制

离线计费方式的选择有PCRF下发和本地配置两种来源，本地配置有三种粒度（优先级从高到低）：

1. **UserProfile粒度**（最高）
2. **APN粒度**
3. **CC（Charging Characteristics）粒度**（最低）

### 计费方式选择优先级（在线/离线接口选择）

1. 通过 `ADD APNCHGMODE` 命令配置基于APN选择计费接口（高优先级）
2. 通过 `SET CHGMODE` 配置全局选择计费接口（低优先级）

## 原理与流程

### 组网架构

离线计费的组网结构如下：

```
SGW-U/PGW-U ──(N4接口)──> SGW-C/PGW-C ──(Ga接口)──> CG ──(FTP)──> BS
     (UDG)                    (UNC)               (Charging Gateway)  (Billing System)
```

- UDG与UNC之间通过**N4接口**（PFCP协议）交互计费信息
- UNC与CG之间通过**Ga接口**传输话单
- CG与BS之间通过**FTP协议**传输标准计费数据

### 离线计费流程

完整离线计费流程包含以下步骤：

**阶段一：会话建立与URR下发**

1. **用户发起承载建立请求**
2. **SGW-C/PGW-C发送PFCP Session Establishment Request**，携带Create URR信元组
   - URR ID：唯一标识一个Create URR
   - Measurement Method：计费方式（流量/时长/事件）
   - Reporting Triggers：上报触发条件
   - Volume Threshold：流量阈值（总/上行/下行）
   - Time Threshold：时间阈值
   - Linked URR ID：关联URR（A link B时，B上报也触发A上报）
3. **SGW-U/PGW-U返回PFCP Session Establishment Response**，会话建立成功
4. **用户承载建立成功**

**阶段二：使用量统计与上报**

5. **用户访问数据业务**，当到达下发阈值时，SGW-U/PGW-U发送PFCP Session Report Request
   - Usage Report包含：
     - URR ID：标识上报的URR
     - UR-SEQN：Usage Report唯一标识
     - Start Time / End Time：起止时间
     - Volume Measurement：流量使用情况（总/上行/下行）
     - Duration Measurement：时长使用情况
     - Time of First Packet：首包时间
     - Time of Last Packet：尾包时间
6. **SGW-C/PGW-C返回PFCP Session Report Response**

**阶段三：话单生成与传输**

7~8. **SGW-C/PGW-C生成话单并通过Ga接口发送到CG**
   - 在话单中记录用户使用的流量、承载持续时间等信息
   - 达到话单关闭条件后产生原始话单
   - 通过Data Record Transfer Request消息将话单发送到CG

### Reporting Triggers（上报触发条件）

| 触发条件 | 含义 |
|----------|------|
| Periodic Reporting | 定期上报 |
| Volume Threshold | 流量阈值触发 |
| Time Threshold | 时间阈值触发 |
| Linked Usage Reporting | 关联URR触发 |

### 内容计费匹配过程

离线计费支持**内容计费**（Content-based Charging），即按业务类型差异化计费。

匹配流程：
1. 用户激活时，通过APN确定UsrProfGroup，根据用户信息（IMSI、MSISDN、RAT、CC等）匹配UserProfile
2. UserProfile下绑定一组Rule，Rule包含三四层/七层协议过滤条件
3. UDG使用SA（Service Awareness）技术对报文进行解析
4. 数据流命中Rule的FlowFilter → 使用Rule携带的URR组计费
5. 数据流未匹配Rule或匹配但Rule无URR组 → 继承UserProfile下的缺省URR组计费
6. UserProfile下无URR组 → 不进行计费

**最佳实践**：一般UserProfile下要配置缺省URR组，避免漏计费。

## 配置规则

### 配置三件套（核心）

离线计费的核心配置遵循标准三件套模式：**URR → URRGROUP → PCCPOLICYGRP**

```
ADD URR（定义计费规则）
    ↓
ADD URRGROUP（组织URR为组）
    ↓
ADD PCCPOLICYGRP（绑定URR组到PCC策略组）
```

### 一致性约束

以下参数在SGW-C/PGW-C和SGW-U/PGW-U上的配置**必须保持一致**：

- **URRID**：使用量上报规则标识
- **USAGERPTMODE**：使用量上报模式（OFFLINE）
- **OFFMETERINGTYPE**：离线计费统计类型

不一致会导致计费行为异常。

### RULE一致性约束

- SGW-C/PGW-C和SGW-U/PGW-U上的**RuleName**必须一致
- RULE下配置的**POLICYTYPE、POLICYNAME**必须一致

### USERPROFILE一致性约束

- USERPROFILE名称在PCF、SGW-C/PGW-C、SGW-U/PGW-U间必须一致
- 不一致时PCF下发的策略无法生效
- USERPROFILE下绑定的**RULENAME、POLICYTYPE**必须一致，不一致策略无法按预期生效

### 配置命令清单

| 命令 | 用途 | 备注 |
|------|------|------|
| ADD URR | 增加使用量上报规则 | 核心命令，定义URRID、USAGERPTMODE=OFFLINE、OFFMETERINGTYPE |
| ADD URRGROUP | 增加URR组 | 组织上行/下行URR |
| ADD PCCPOLICYGRP | 增加PCC策略组 | 绑定URR组 |
| ADD FILTER | 增加过滤器 | 定义三四层协议匹配条件 |
| ADD FLOWFILTER | 增加流过滤器 | 流过滤容器 |
| ADD FLTBINDFLOWF | 流过滤器绑定过滤器 | 将Filter绑定到FlowFilter |
| ADD RULE | 增加业务规则 | 关联FlowFilter和PCC策略组 |
| ADD USERPROFILE | 增加用户模板 | 用户策略容器 |
| ADD RULEBINDING | 增加规则绑定 | 将Rule绑定到UserProfile |
| SET REFRESHSRV | 刷新服务 | REFRESHTYPE=USERPROFILE，使配置生效 |

| 软参 | 功能 |
|------|------|
| BIT1273 | 控制某运营商对于无法获取正常费率的异常报文的定制计费处理 |
| BYTE57 | 设置内容计费资源耗尽或阻塞后的动作 |
| BYTE301 | 控制强制归并流量计费方式 |
| BIT496 | 控制UsageReport中UsageExtention字段key为URL时的填充方式 |
| BIT878 | 控制是否删除话单URL中的不可见字符 |

| BIT341 | 控制用户计费方式是否为Normal计费 |

## 配置案例

### 案例一：Ga接口离线计费基础配置

**场景描述**：用户进行离线计费，统计到URR ID为1000的URR中，使用事件+流量+时长全量统计。

**配置脚本：**

```mml
// 第一步：配置离线计费的URR三件套
ADD URR:URRNAME="offlineURR",URRID=1000,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=EVENT_VOLUME_TIME;
ADD URRGROUP:URRGROUPNAME="urrgroup1",UPURRNAME1="offlineURR",DOWNURRNAME1="offlineURR";
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="cg-test",URRGROUPNAME="urrgroup1";

// 第二步：配置三四层匹配条件
ADD FILTER:FILTERNAME="filter_red",L34PROTTYPE=STRING,L34PROTOCOL=TCP,SVRSTARTPORT=80,SVRENDPORT=80;
ADD FLOWFILTER:FLOWFILTERNAME="fg_red";
ADD FLTBINDFLOWF:FLOWFILTERNAME="fg_red",FILTERNAME="filter_red";

// 第三步：配置内容计费规则
ADD RULE:RULENAME="Rule001",POLICYTYPE=PCC,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="fg_red",POLICYNAME="cg-test",PRIORITY=110;
SET REFRESHSRV:REFRESHTYPE=USERPROFILE;

// 第四步：配置规则绑定用户模板
ADD USERPROFILE:USERPROFILENAME="up_test";
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="Rule001";
```

### 案例参数说明

| 配置项 | 参数 | 取值 | 说明 |
|--------|------|------|------|
| URR | URRNAME | offlineURR | 离线计费URR名称 |
| URR | URRID | 1000 | URR标识，需全网唯一 |
| URR | USAGERPTMODE | OFFLINE | 离线计费模式 |
| URR | OFFMETERINGTYPE | EVENT_VOLUME_TIME | 事件+流量+时长全量统计 |
| URRGROUP | UPURRNAME1 | offlineURR | 上行使用同一URR |
| URRGROUP | DOWNURRNAME1 | offlineURR | 下行使用同一URR |
| PCCPOLICYGRP | URRGROUPNAME | urrgroup1 | 绑定URR组 |
| FILTER | L34PROTOCOL | TCP | 匹配TCP协议 |
| FILTER | SVRSTARTPORT/SVRENDPORT | 80 | 匹配HTTP端口 |
| RULE | POLICYTYPE | PCC | PCC策略类型 |
| RULE | PRIORITY | 110 | 优先级（数字越小优先级越高） |

### 不同OFFMETERINGTYPE配置示例

```mml
// 仅流量计费（默认）
ADD URR:URRNAME="volumeURR",URRID=1001,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=VOLUME;

// 仅时长计费
ADD URR:URRNAME="timeURR",URRID=1002,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=TIME;

// 流量+时长计费
ADD URR:URRNAME="vtURR",URRID=1003,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=VOLUME_TIME;

// 免费计费（防欺诈）
ADD URR:URRNAME="freeURR",URRID=1004,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=FREE;

// 事件+流量计费
ADD URR:URRNAME="evURR",URRID=1005,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=EVENT_VOLUME;
```

## 验证与调测

### 告警

本特性无相关告警。

### 测量指标

本特性无相关测量指标。

### 注意事项

1. **ADD URR命令执行后立即生效**，无需重启
2. **ADD URR最大记录数为65000**，配置数超过规格90%时会上报告警ALM-135602215
3. **开启事件计费对性能有影响**，需评估后使用
4. **URRID建议从1000开始分配**，在所有URR内保持唯一
5. **SET REFRESHSRV必须在RULE配置完成后执行**，使配置生效
6. **仅绑定三四层filter的RULE优先级应低于绑定七层filter的RULE优先级**（数字越小优先级越高）
7. **UPURRNAME1/2/3和DOWNURRNAME1/2/3只是编号**，无优先级语义
