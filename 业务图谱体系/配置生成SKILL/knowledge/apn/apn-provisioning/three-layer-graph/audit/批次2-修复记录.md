### 2.3 HIGH-04：`04-command-graph.md` 修复

#### 2.3.1 140 MMLCommand 补 status 列（§1）

> **Schema 依据**：§11.3 MMLCommand.status 必选（原 §1 所有 140 命令表均无 status 列）。

**修复方式**：新增 §0.3 全局字段声明块（与 `02-feature-graph.md` §0.3 同模式），声明 140 个 MMLCommand 的 `status` 字段值均为 `active`。MPLS 推导命令（CMD-UDG-062/063/064，§1.11）虽 `status=active`（已在命令图谱登记），但 §1.11 表中 `command_summary` 列已用 "★推导" 前缀标注其源自 MPLS L3VPN 标准实践推导，并在 §0.3 集中说明 + 引用 CR-APN-18。

> **为什么不逐行加列**：140 命令分布在 24 个表中，逐行加列会破坏表结构且与项目既有模式（全局声明）不一致；全局声明方式已被 `02-feature-graph.md` §0.3（License/Feature status）确立为本项目记录 Schema 必选字段的标准做法，Schema 合规性等价。

#### 2.3.2 CommandParameter 补 required_mode 列（§5）

> **Schema 依据**：§11.4 CommandParameter.required_mode 必选（取值 `required / optional / conditional_required`；原 §5 九个核心命令参数表均无此列）。

**修复方式**：§5 九个核心命令参数表均从 4 列扩为 5 列，新增 `required_mode` 列。共补 9 个表：

| 子节 | 命令 | 参数数 | required_mode 分布 |
|------|------|--------|-------------------|
| §5.1 | ADD POOL（UDG） | 6 | required×3 / optional×1 / conditional_required×2 |
| §5.2 | ADD ADDRPOOL（UNC） | 3 | required×3 |
| §5.3 | SET APNADDRESSATTR | 6 | required×2 / optional×2 / conditional_required×2 |
| §5.4 | SET APNL2TPATTR（U） | 4（含 ... 占位） | required×2 / optional×1 |
| §5.5 | SET APNL2TPCTRL（C） | 2 | required×2 |
| §5.6 | SET APNAUTHATTR | 4 | required×2 / conditional_required×2 |
| §5.7 | ADD IPSECPOLICY | 4 | required×3 / conditional_required×1 |
| §5.8 | ADD GRETUNNEL | 5 | required×3 / conditional_required×2 |
| §5.9 | ADD APNACTNUM | 5 | optional×3 / conditional_required×1 + required×1（APN） |

> **conditional_required 判定依据**：参数依赖另一参数取值或场景条件时标 conditional_required（如 VPNINSTANCE 依赖 HASVPN=ENABLE；COMMONUSERNAME 依赖 ACCESSMODE=TRANS_AUTH；PDNCONNREJCAUSE 依赖配置了任一并发阈值）。

#### 2.3.3 ★CR-APN-07 rule_type 枚举修正（HIGH）

> **Schema 依据**：§11.6 CommandRule.rule_type 枚举值 —— `syntax_rule / semantic_rule / precondition_rule / delete_constraint / parameter_mutex / parameter_dependency / object_reference_rule / runtime_check_rule`。

**修复前**（非法）：
```
CR-APN-07 | ... | `consistency_rule`（mapped: `semantic_rule`） | ...
```

**修复后**（合法）：
```
CR-APN-07 | ... | `semantic_rule` | ...
```

**修正理由**：
1. `consistency_rule` **不在** Schema §11.6 CommandRule.rule_type 枚举中（该枚举值属于 §9.6 FeatureRule.rule_type，跨层混用错误）。
2. `（mapped: semantic_rule）` 标注是过渡期占位，应直接采用目标枚举值，删除映射标注。
3. 该规则逻辑（"APN VPN 与地址池 VPN 必须一致"）属于跨对象语义一致性约束，`semantic_rule` 是 Schema §11.6 最贴切的枚举值。

> **范围限定**：本次仅修正 CR-APN-07（任务指定）。CR-APN-10（`validation_rule（mapped: parameter_dependency）`）存在同类问题但不在本批次范围，留待后续批次处理。
