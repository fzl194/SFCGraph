# 05-U+C 不对称与跨侧一致性

> 数据源：`04-command-graph.md` §4 CommandRule CR-APN-01/02/03/04 + §2 ConfigObject + `03-task-layer.md` §10 TR-APN-03/05/06
> 作用：Phase 3 跨侧参数收集 / Phase 5 双侧生成 / Phase 6 跨网元一致性核查（CRITICAL 级）

---

## 1. 三组核心不对称对象族

### 1.1 POOL(UDG) vs ADDRPOOL(UNC)（CR-APN-01/04）

| 维度 | UDG POOL | UNC ADDRPOOL |
|------|---------|--------------|
| 命令 | `ADD POOL`（CMD-UDG-014） | `ADD ADDRPOOL`（CMD-UNC-002） |
| 命名前缀 | POOL / POOLGROUP / POOLBIND**GROUP** | ADDRPOOL / ADDRPOOLGRP / POOLBIND**GRP** |
| **POOLTYPE** | **LOCAL**（本地池）/ EXTERNAL | **UDM**（仅静态签约），无 LOCAL |
| 使用场景 | 动态分配（UPF-APN/UPF-SMF/UPF-LOCATION/RADIUS/DHCP/LNS） | 仅 UDM 静态签约（CS-APN-01/04） |
| 池组绑定 | ADD POOLBINDGROUP（GROUP） | ADD POOLBINDGRP（GRP）+ ADD POOLBINDAPN（UNC 独有） |

> **★关键约束**：两侧命令前缀不对称，ConfigObject 必须分离建模；POOLTYPE 跨产品同名参数取值集不同。**不可混用**。

### 1.2 APNL2TPATTR(U,10+参数) vs APNL2TPCTRL(C,2参数)（CR-APN-02）

| 维度 | UDG APNL2TPATTR | UNC APNL2TPCTRL |
|------|----------------|-----------------|
| 命令 | `SET APNL2TPATTR`（CMD-UDG-065） | `SET APNL2TPCTRL`（CMD-UNC-064） |
| 参数数 | **10+**（L2TPSWITCH/SUPPORTIPV6/L2TPGROUPID/RDSLNSMODE/ICRQ_CALLINGNO/ICCN_AUTH/IPCP_NEGO/...） | **2**（APN/L2TPSWITCH） |
| 角色 | U 面 LAC 执行（封装 PPP/L2TP） | C 面决策（下发 LNS 参数经 N4） |
| License | LKV3G5L2TP01（UDG 必须） | 无（UNC WSFD-104410 无 License） |

> **C 决策 U 执行模式典型不对称**。参数集不匹配会导致 L2TP 隧道建链失败。

### 1.3 L2TPN4KEY(U) vs L2TPKEY(C)（CR-APN-03）

| 维度 | UDG L2TPN4KEY | UNC L2TPKEY |
|------|--------------|-------------|
| 命令 | `SET L2TPN4KEY`（CMD-UDG-074） | `SET L2TPKEY`（CMD-UNC-066） |
| 参数 | KEY（N4 加密） | KEY（N4 加密） |
| **一致性** | **两端密钥必须相同** | **两端密钥必须相同** |

> 加密协商失败导致 L2TP 隧道无法建立。

## 2. 其他 U+C 共用对象（对称）

| 对象 | UDG 命令 | UNC 命令 | 备注 |
|------|---------|---------|------|
| APN | `ADD APN`（CMD-UDG-009） | 复用同命令族（不独立编号） | 跨域共用挂载点 |
| SECTION | `ADD SECTION`（CMD-UDG-015） | `ADD SECTION`（CMD-UNC-006） | 同命令 |
| APNADDRESSATTR | `SET APNADDRESSATTR`（CMD-UDG-013） | `SET APNADDRESSATTR`（CMD-UNC-014） | 同命令名（U+C 共用） |
| POOLGRPMAP | `ADD POOLGRPMAP`（CMD-UDG-018） | `ADD POOLGRPMAP`（CMD-UNC-009） | 同命令（参数名差异） |
| IPALLOCBYLOCGLBSW | `SET IPALLOCBYLOCGLBSW`（CMD-UDG-029） | `SET IPALLOCBYLOCGLBSW`（CMD-UNC-017） | 同命令名 |
| REDUNDUSER | `SET REDUNDUSER`（CMD-UDG-076） | `SET REDUNDUSER`（CMD-UNC-084） | U+C 共用对象 |
| LICENSESWITCH | `SET LICENSESWITCH`（CMD-UDG-001） | `SET LICENSESWITCH`（CMD-UNC-001） | 参数名差异：LICITEM vs LICENSECODE |
| REFRESHSRV | `SET REFRESHSRV`（CMD-UDG-088） | `SET REFRESHSRV`（CMD-UNC-087） | U+C 各一个，配置链终点 |

## 3. 跨侧一致性 CRITICAL 项（Phase 6 核查）

| 参数项 | 一致性要求 | 规则 |
|--------|-----------|------|
| APN/DNN 名 | U+C 必须一致 | 通用 |
| SUPPORTIPV4/SUPPORTIPV6 | U+C 必须一致 | SET APNADDRESSATTR |
| VPNINSTANCE（若两侧都配 VPN） | U+C 必须一致 | CR-APN-07 |
| L2TPN4KEY（U）↔ L2TPKEY（C）密钥 | **必须相同** | CR-APN-03 |
| POOLTYPE | 跨侧取值集不同（UDG=LOCAL/EXTERNAL，UNC=UDM） | CR-APN-04（不是一致，是分离） |
| APNL2TPATTR（U）↔ APNL2TPCTRL（C） | 两侧都需配置（参数集不对称） | CR-APN-02 |
| ADD APN 前 U+C 都需配置 | APN 是共用挂载点 | 通用 |

---

## 4. 配置链终点：REFRESHSRV

UDG 与 UNC 两侧配置链**各自以 SET REFRESHSRV 结尾**：

- `SET REFRESHSRV:;`（CMD-UDG-088）— UDG 侧：地址分配/隧道/L2TP/QoS 变更后必须
- `SET REFRESHSRV:;`（CMD-UNC-087）— UNC 侧：Radius/UPF选择/ARD 变更后必须

> Task 级 `must_be_last=true`。无业务参数，触发下发动作。
