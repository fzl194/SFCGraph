# 12-接入控制与并发限制

> 数据源：`04-command-graph.md` §1.14 + §1.24/1.27 + §5.9 + CR-APN-15/16/17 + `feature/WSFD-106003-用户接入控制功能.md` + `feature/GWFD-010151-接入控制.md` + `feature/WSFD-010503-多PDN-PDU功能.md`
> 作用：Phase 3 接入控制/并发参数 / Phase 5 接入控制生成
> **TR-APN-10**：U 面带宽流控（APNQOSATTR）与 C 面接入权限（ARD/NGMMSUBDATA）机制完全不同，非 C-U 对称

---

## 1. U 面 QoS 带宽流控（GWFD-010151，UDG）

### SET APNQOSATTR（CMD-UDG-080）

| 参数 | 取值 | 说明 |
|------|------|------|
| APN | string | APN 名 |
| CARSHAPESWUL | ENABLE / DISABLE | 上行 CAR/Shaping 开关 |
| CARSHAPEUL | CAR / SHAPE | 上行方式（CAR=限速 / SHAPE=缓存整形） |
| CARSHAPESWDL | ENABLE / DISABLE | 下行 CAR/Shaping 开关 |
| CARSHAPEDL | CAR / SHAPE | 下行方式 |

**特点**：
- 仅 APN 粒度（无法针对单用户）
- 上下行独立配置（CAR 或 SHAPE）
- 与 C 面 WSFD-106003 机制完全不同（TR-APN-10）

---

## 2. C 面接入权限 ARD（WSFD-106003 子特性 B，UNC）

> **TR-APN-09**：卡类型控制必须先启用 WSFD-010301 AKA 鉴权（通过三元组/五元组区分 SIM/USIM）

| 命令 | command_id | 关键参数 | 代际 |
|------|-----------|---------|------|
| ADD GBARD | CMD-UNC-053 | IMSI, APNNI, CARDTYPE(SIM/USIM), ARD, CTRLTYPE(ALLOW/DENY), CAUSE | 2G |
| ADD IUARD | CMD-UNC-054 | IMSI, APNNI, CARDTYPE, ARD, CTRLTYPE | 3G |
| ADD S1ARD | CMD-UNC-055 | IMSI, APNNI, CARDTYPE, ARD, CTRLTYPE | 4G |
| ADD NGMMSUBDATA | CMD-UNC-056 | USER_RANGE, IMSIPRE, RATRESTRICT, CORERESTRICT | 5GC（子特性 A） |
| SET NGMMPROCTRL | CMD-UNC-057 | NGMM 处理控制 | 5GC |

**BR-APN-AMF-LOCAL-FIRST**：AMF 本地配置优先于签约；本地 Null 强制允许；紧急注册跳过接入控制。

## 3. 单 APN 并发限制（WSFD-010503，UNC）

### ADD APNACTNUM（CMD-UNC-070，§5.9）

| 参数 | 类型 | 取值 | required_mode | 说明 |
|------|------|------|--------------|------|
| APN | string | — | required | APN 名 |
| PDNNUM | int | — | optional | PDN 并发数阈值 |
| IPV4ADDRNUM | int | — | optional | IPv4 地址数阈值 |
| IPV6ADDRNUM | int | — | optional | IPv6 地址数阈值 |
| PDNCONNREJCAUSE | int | — | conditional_required | 超限拒绝原因值（CR-APN-15；配置阈值后必填） |

**BR-APN-CONCURRENCY-11-15**：EPC 单用户 PDN 连接 ≤ 11；5GC 单用户 PDU 会话 ≤ 15；超限 MME/SMF 拒绝（原因值 55）。

**CR-APN-15**：单 APN 并发超过 PDNNUM/IPV4ADDRNUM/IPV6ADDRNUM 阈值时，触发 PDNCONNREJCAUSE 拒绝。

---

## 4. 配置实例

### 4.1 U 面 QoS

```mml
// 上下行 CAR 限速
SET APNQOSATTR:APN="apn-test1",CARSHAPESWUL=ENABLE,CARSHAPEUL=CAR,CARSHAPESWDL=ENABLE,CARSHAPEDL=CAR;
```

### 4.2 单 APN 并发限制

```mml
ADD APNACTNUM:APN="apn-test1",PDNNUM=11,IPV4ADDRNUM=2,IPV6ADDRNUM=2,PDNCONNREJCAUSE=55;
```

## 5. UPF 选择（WSFD-107010，三轮筛选，UNC）

> TR-APN-08：第一轮 7 必选条件全满足无优先级；4G 接入 PNFNSINDEX=0（不用切片）；SMF 和 UPF 必须同厂商；静态 IP 段绑定 UPF 与 SMF 主锚点 UPF 选择冲突时 SMF 优先（CR-APN-16）

| 轮次 | 命令族 | 说明 |
|------|--------|------|
| 第一轮（必选） | ADD PNFDNN / PNFNS / PNFDNAI / PNFUPFINFO / UPAREA / PNFTAI / UPBINDS11 / UPBINDGNGP | 7 必选条件 |
| 第二轮（优选） | SET UPSELECTPRI / SET UPSELECTFLAG | 策略次序/开关 |
| 第三轮（权重） | SET UPLOADBALANCE | 负载均衡 |
| APN 级 | SET APNUPSELPLY | APN 级 UPF 选择策略 |

**BR-APN-UPF-VENDOR-LOCK**：SMF 和 UPF 设备必须同时为 HUAWEI（WSFD-107010 硬约束）。

---

## 6. 约束

- **TR-APN-10**：U 面 APNQOSATTR（仅 APN 粒度）vs C 面 ARD/NGMMSUBDATA（可到单用户），机制完全不同，非 C-U 对称
- **TR-APN-09**：ARD 卡类型控制必须先启用 WSFD-010301 AKA 鉴权
- **BR-APN-AMF-LOCAL-FIRST**：AMF 本地配置优先于签约
- **BR-APN-CONCURRENCY-11-15**：EPC ≤11，5GC ≤15，超限拒绝（原因值 55）
- **BR-APN-UPF-VENDOR-LOCK**：SMF + UPF 必须同厂商（HUAWEI）
- **CR-APN-15**：APNACTNUM 阈值超限触发 PDNCONNREJCAUSE 拒绝
- **CR-APN-16**：STATICADDRPARA × SMF 主锚点 UPF 冲突时 SMF 优先
