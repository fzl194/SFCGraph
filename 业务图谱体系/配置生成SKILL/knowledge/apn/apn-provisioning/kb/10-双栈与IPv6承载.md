# 10-双栈与 IPv6 承载

> 数据源：`04-command-graph.md` §1.2/1.6 + CR-APN-05/06 + `feature/GWFD-020403-IPv4v6双栈接入.md` + `feature/GWFD-020401-IPv6承载上下文.md` + `feature/GWFD-020406-IPv6PrefixDelegation.md`
> 作用：Phase 3 双栈参数（DP-APN-ADDR-TYPE=IPv4v6/IPv6）/ Phase 5 双栈生成（CS-APN-03/04/09）
> **License 串联**：LKV3G5V6PB01（承载）→ LKV3G5VDSA01（双栈）→ LKV3G5P6PD01（PD），BR-APN-IPV6-CASCADE

---

## 1. VPNINSTAF 地址族（CR-APN-06）

| 场景 | ADD VPNINSTAF 配置 |
|------|-------------------|
| IPv4 单栈 | AFTYPE=ipv4uni（或无 VPNINSTAF） |
| IPv6 单栈 | AFTYPE=ipv6uni（★IPv6 必需，CR-APN-06） |
| IPv4v6 双栈 | ipv4uni + ipv6uni 双地址族（双 VPN 实例） |

```mml
// IPv4 VPN
ADD L3VPNINST:VRFNAME="vpn1";
ADD VPNINSTAF:VRFNAME="vpn1",AFTYPE=ipv4uni,VRFRD=100:1;

// IPv6 VPN（双栈场景额外）
ADD L3VPNINST:VRFNAME="vpn1_v6";
ADD VPNINSTAF:VRFNAME="vpn1_v6",AFTYPE=ipv6uni,VRFRD=100:2;
```

> **CR-APN-06**：IPv6/双栈必须 AFTYPE=ipv6uni 激活 IPv6 地址族，否则 IPv6 地址无法分配。

---

## 2. APN 双栈双绑定

```mml
ADD APN:APN="{apn}",HASVPN=ENABLE,VPNINSTANCE="vpn1",HASVPNIPV6=ENABLE,VPNINSTANCEIPV6="vpn1_v6";
```

- HASVPN=ENABLE + VPNINSTANCE=IPv4 VPN
- HASVPNIPV6=ENABLE + VPNINSTANCEIPV6=IPv6 VPN（双栈）

---

## 3. 双池双段（IPv4 POOL + IPv6 POOL 并存）

```mml
// IPv4 池
ADD POOL:POOLNAME="pool_v4",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="pool_v4",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.10.1.1",V4ENDIP="10.10.1.10";

// IPv6 池
ADD POOL:POOLNAME="pool_v6",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1_v6";
ADD SECTION:POOLNAME="pool_v6",SECTIONNUM=1,IPVERSION=IPV6,V6PREFIXSTART="FC00:1::",V6PREFIXEND="FC00:1::FF",V6PREFIXLENGTH=64;
```

> **CR-APN-05 / TR-APN-04**：V6PREFIXLENGTH=64 为普通 IPv6 单栈；<64 切换 IPv6 Prefix Delegation 模式（需 License LKV3G5P6PD01）。

---

## 4. 双池绑定同组（双优先级算法）

```mml
// 池组（双优先级算法使能）
ADD POOLGROUP:POOLGRPNAME="poolgrp1",IPV4ALLOCPRIALG=ENABLE,IPV6ALLOCPRIALG=ENABLE;

// 双池绑定（同类池间比较优先级，跨类型不比）
ADD POOLBINDGROUP:POOLGROUPNAME="poolgrp1",POOLNAME="pool_v4",PRIORITY=10;
ADD POOLBINDGROUP:POOLGROUPNAME="poolgrp1",POOLNAME="pool_v6",PRIORITY=10;
```

---

## 5. APN 级双栈属性

```mml
SET APNADDRESSATTR:APN="{apn}",SUPPORTIPV4=ENABLE,SUPPORTIPV6=ENABLE;
```

## 6. 下行路由发布（OSPF + OSPFv3 双进程）

```mml
// IPv4 OSPF
ADD OSPF:PROCID=100,VRFNAME="vpn1",SCHEMAROUID="10.10.10.1";
ADD OSPFAREA:PROCID=100,AREAID="0.0.0.0";
ADD OSPFNETWORK:PROCID=100,AREAID="0.0.0.0",IPADDRESS="10.13.21.0",WILDCARDMASK="0.0.0.255";
ADD OSPFIMPORTROUTE:PROCID=100,TOPOID=0,PROTOCOL=wlr;

// IPv6 OSPFv3（IPv6 承载链必需）
ADD OSPFV3:PROCID=101,ROUTERID="10.10.10.1";
ADD OSPFV3AREA:PROCID=101,AREAID="0.0.0.0";
ADD OSPFV3IMPORTROUTE:PROCID=101,PROTOCOL=wlr;
```

## 8. License 串联（BR-APN-IPV6-CASCADE）

```
LKV3G5V6PB01（IPv6 承载上下文，GWFD-020401，承载链根）
   ↓ 前级必须先激活
LKV3G5VDSA01（IPv4v6 双栈，GWFD-020403，使能 GWFD-010105 的 IPv4v6 取值）
   ↓ V6PREFIXLENGTH<64 触发
LKV3G5P6PD01（IPv6 Prefix Delegation，GWFD-020406）
```

**违反效果**：IPv6 承载缺失会导致双栈/PD 无法生效；License 串联断裂；缺 License 时双栈配置命令成功但功能不生效（BR-APN-DUALSTACK-NEED-LICENSE）。

---

## 9. IPv6 Prefix Delegation（V6PREFIXLENGTH<64）

- V6PREFIXLENGTH=64：普通 IPv6 单栈地址分配
- V6PREFIXLENGTH<64：切换 PD 模式（前缀代理给下游）
- PD 模式需 License LKV3G5P6PD01
- APNADDRESSATTR 需配 V6POOLID（PD 场景必填）

---

## 10. 约束（CR-APN-05/06/07，BR-APN-IPV6-CASCADE/DUALSTACK-NEED-LICENSE，TR-APN-04）

- **VPNINSTAF AFTYPE=ipv6uni 必需**（CR-APN-06）
- **V6PREFIXLENGTH<64 切换 PD**（CR-APN-05 / TR-APN-04）
- **APN VPN 与 POOL VPN 必须一致**（CR-APN-07），双栈对应双 VPN 实例
- **双栈 License 串联**：承载→双栈→PD，前级必须先激活
- **双栈双优先级算法**：IPV4ALLOCPRIALG + IPV6ALLOCPRIALG 都需使能
- **优先级仅同类池比较**：IPv4 vs IPv4，IPv6 vs IPv6，跨类型不比
- **OSPF Router ID 全网唯一**（SCHEMAROUID）
