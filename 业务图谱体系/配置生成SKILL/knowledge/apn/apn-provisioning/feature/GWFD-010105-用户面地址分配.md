## 1. 概述

### 1.1 特性定义（是什么）

UE IP地址是UE在网络中进行通信和寻址的唯一标识，用户上网过程中必须被分配IP地址才可正常通信。**用户面地址分配是指 UDG（PGW-U/UPF）本地为UE分配地址**，可提供三种类型的IP地址：IPv4地址、IPv6地址、IPv4v6双栈地址。

由于 UDG 为用户面网关，处于分组核心网和外部DN交换的位置，因此 UDG 在为UE分配IP地址的同时，还可生成下行路由用于下行报文的回程寻址。

> 来源：`output/UDG_Product_Documentation_CH_20.15.2/特性部署/特性指南/UDG特性指南/基本接入功能/GWFD-010105 用户面地址分配/GWFD-010105 用户面地址分配特性概述_68664421.md`

## 2. 激活

> 与 SA-Basic 不同，本特性无独立 License。激活即"配置使能"：通过 MML 命令创建地址池、配置地址分配规则并使能开关。

### 2.1 激活通用前置条件

- 已完成数据面逻辑接口（Sa/Sc/Pa）和 N4/Sxa/Sxb 逻辑接口的配置（参考初始配置手册"配置业务接口数据"）
- 已规划 VPN 实例、APN/DNN 实例、本地地址池

> 来源：各激活文档"必备事项"

### 2.2 通用激活步骤骨架

所有 4 种子方式共享以下骨架（差异在步骤 3 映射条件和步骤 4/5 的开关）：

```
步骤1：基于APN/DNN实例使能地址分配属性
  ├── ADD L3VPNINST         → 创建 L3VPN 实例
  ├── ADD VPNINSTAF         → 设置 VPN 实例地址族
  ├── ADD VPNINST           → 配置 VPN 实例（用户面侧）
  ├── ADD APN               → 配置 APN/DNN 实例（绑定 VPN）
  └── SET APNADDRESSATTR    → 使能该 APN 的 SUPPORTIPV4 / SUPPORTIPV6

步骤2：配置地址池绑定到地址池组
  ├── ADD POOL              → 配置 LOCAL 类型地址池
  ├── ADD SECTION           → 配置地址段（IPv4 / IPv6）
  ├── ADD POOLGROUP         → 配置地址池组
  ├── ADD POOLBINDGROUP     → 绑定地址池到地址池组
  └── ADD CONFLICTIP（可选） → 标识冲突 IPv4 地址

步骤3：配置映射关系（按子方式不同）
  ├── 基于APN/DNN          → ADD POOLGRPMAP: APN=..., POOLGROUPNAME=...
  ├── 基于SMF              → ADD CPNODEID + ADD POOLGRPMAP: SMF=..., POOLGROUPNAME=...
  ├── 基于SMF+APN/DNN      → ADD CPNODEID + ADD POOLGRPMAP: APN=..., SMF=..., POOLGROUPNAME=...
  └── 基于RADIUS下发地址池名 → ADD POOLGRPMAP: APN=..., POOLGROUPNAME=...（关键在 SET APNADDRESSATTR IGNOREV4/V6POOLID=DISABLE）

步骤4：配置地址分配规则
  ├── SET IPALLOCRULE       → 配置全局三级规则（FIRSTRULE/SECONDRULE/THIRDRULE）
  └── SET APNIPALLOCRULE（可选） → 配置基于 APN 的地址分配规则

步骤5：（仅基于SMF相关子方式）配置基于SMF分配地址的开关
  ├── SET IPALLOCBYSMFGLBSW → 配置全局开关
  └── SET IPALLOCBYSMFSW（可选） → 配置指定 SMF 的开关

步骤6：配置手机下行路由（生成下行子段路由发布到骨干网）
  ├── ADD OSPF
  ├── ADD OSPFAREA
  ├── ADD OSPFNETWORK
  └── ADD OSPFIMPORTROUTE   → 引入 WLR（用户路由）到 OSPF
```

> 来源：综合各激活文档"操作步骤"章节

## 3. 原理

### 3.1 实现原理：能力协商与地址分配决策

UDG 通过 N4/Sxa/Sxb 接口的**偶联建立响应消息**中的**私有扩展域**告知 C 面本地分配地址能力，C 面获取 UDG 该能力。

用户激活向 C 面发起会话建立请求，申请分配IP地址，C 面决策由 U 面分配指定类型的IP地址（IPv4、IPv6或IPv4v6双栈）。在 N4/Sxa/Sxb 接口通过 PFCP 会话建立请求消息（PFCP Session Establishment Request）指示 UDG 分配地址。UDG 根据指示为用户分配地址，并通过会话建立响应消息将分配的IP地址传递给UE，UE获取IP地址后便可发起上下行数据业务。

> 来源：`GWFD-010105 用户面地址分配特性概述_68664421.md`（"原理概述"章节）

### 3.2 地址分配方式（4种子方式 + 任意叠加组合）

UDG 本地地址池分配地址有如下多种方式，也支持这些场景的任意叠加组合方式（例如基于位置+APN/DNN、SMF+APN/DNN的地址分配）：

| 地址分配方式 | 使用场景 | 文档位置 |
|-------------|---------|---------|
| 基于APN/DNN分配地址 | 按照APN/DNN规划和管理用户地址池，适用于根据APN规划地址池的场景 | 激活文档：`基于APN_DNN分配地址_72547232.md` |
| 基于位置分配地址 | 按照位置规划和管理用户地址池，根据用户所处的位置分配特殊的IP地址，实现IP地址细分 | 详细参考 GWFD-020421 基于位置的地址分配 |
| 基于SMF分配地址 | 按照SMF Node ID规划和管理用户地址池，适用于运营商部署SMF内置BSF（Binding Support Function）的场景 | 激活文档：`基于SMF分配地址_87605805.md` |
| 基于RADIUS下发地址池名称分配地址 | 按照运营商规划，适用于Radius管理端到端动态地址分配的场景 | 激活文档：`基于RADIUS下发地址池名称分配地址_13796101.md` |
| 基于SMF+APN/DNN组合 | 第一优先级按SMF+APN双维度匹配，第二优先级回退到SMF单维度 | 激活文档：`基于SMF+APN_DNN分配地址_87787640.md` |

> 来源：`实现原理/地址分配方式_87787955.md`

### 3.3 地址分配规则（全局 vs APN两级 + 三级优先级）

UDG 上地址分配可以基于**全局**（`SET IPALLOCRULE`）和 **APN/DNN**（`SET APNIPALLOCRULE`）分别使能：
- 如果设置了基于APN的地址分配规则，则使用APN下配置的地址分配规则
- 否则就使用基于全局的地址分配规则

UDG 上地址分配规则支持**三级优先级规则**，三级优先级可分别选择上述 APN/DNN、位置、SMF 进行组合：
- 如果第一优先级规则匹配成功，则使用第一优先级类型的地址池进行地址分配
- 否则依次是第二优先级和第三优先级类型的地址池进行地址分配
- 如果都匹配失败，则地址分配失败

每级规则中所选的条件之间为"**与**"的关系，表示从所选条件映射的地址范围内分配地址。此处设置的规则需要与地址池的映射条件一致。例如按照 SMF+APN（`APN-1&LOCATION-0&SMF-1`）设置条件，则规划的地址池组需绑定 SMF&APN。

**规则条件字符串格式**：`APN-{0|1}&LOCATION-{0|1}&SMF-{0|1}`（1=启用此维度，0=不启用）

> 来源：`实现原理/地址分配方式_87787955.md`、各激活文档"操作步骤4"

### 3.4 IPv4/IPv6/双栈地址类型参数（C-U面PFCP信元协商）

UE IP地址在 PDU 会话建立过程中被分配，在整个 PDU 会话过程中固定唯一。C 面通过 PFCP Session Establishment Request 消息中的 **UE IP Address / UE IP address Pool Identity** 子信元（其 CHV4、CHV6、V4、V6 BIT 位），指示 UDG 是否分配地址以及分配的地址类型。

#### 3.4.1 关键信元（在 Create PDR/PDI 或 Create Traffic Endpoint 中携带）

- **UE IP Address** 子信元：含 CHV4/CHV6/V4/V6 控制位
- **UE IP address Pool Identity** 子信元：当地址分配方式为用户面地址分配时，UDG 会解析其中的 Framed-Pool 或 Framed-IPv6-Pool，根据地址池名称从指定地址池分配；若地址池名称为 `*` 字符串，则使用普通本地地址池

#### 3.4.2 UDG 地址分配方式选择真值表（BIT391控制标准/华为私有）

| BIT391 | Spare bit | CHV4 | CHV6 | V4 | V6 | IPv4 addr | IPv6 addr | UDG处理方式 | 分配方式 |
|--------|-----------|------|------|----|----|-----------|-----------|------------|---------|
| 0（标准） | - | 0 | 0 | 1 | 0 | 携带 | - | 取该IPv4地址为UE IP | 外部地址分配 |
| 0（标准） | - | 0 | 0 | 0 | 1 | - | 携带 | 取该IPv6地址为UE IP | 外部地址分配 |
| 0（标准） | - | 0 | 0 | 1 | 1 | 携带 | 携带 | 取IPv4+IPv6为UE IP | 外部地址分配 |
| 0（标准） | - | 1 | 0 | 0 | 0 | - | - | 本地地址池分配IPv4 | **用户面地址分配** |
| 0（标准） | - | 0 | 1 | 0 | 0 | - | - | 本地地址池分配IPv6 | **用户面地址分配** |
| 0（标准） | - | 1 | 1 | 0 | 0 | - | - | 本地地址池分配IPv4+IPv6 | **用户面地址分配** |
| 0（标准） | - | 1 | 0 | 0 | 1 | - | 携带 | 取外部IPv6 + 本地分配IPv4 | 混合 |
| 0（标准） | - | 0 | 1 | 1 | 0 | 携带 | - | 取外部IPv4 + 本地分配IPv6 | 混合 |
| 1（华为私有） | 1 | - | - | 1 | 0 | 全0 | - | 本地分配IPv4 | **用户面地址分配** |
| 1（华为私有） | 1 | - | - | 0 | 1 | - | 全0 | 本地分配IPv6 | **用户面地址分配** |
| 1（华为私有） | 1 | - | - | 1 | 1 | 全0 | 全0 | 本地分配IPv4+IPv6 | **用户面地址分配** |

> 说明：对于表中未给出的场景，PGW-U/UPF 均以激活失败处理。

                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐

```

> 来源：`实现原理/地址分配流程_74936046.md`

### 3.6 协议交互

| 接口 | 交互网元 | 关键信元 | 说明 |
|------|---------|---------|------|
| N4（5G）/ Sxa / Sxb（4G） | SMF/PGW-C ↔ UPF/PGW-U | PFCP Association Setup Response（私有扩展 UP Function Flag, ADUP 标识位） | U面能力上报 |
| N4 / Sxa / Sxb | SMF/PGW-C → UPF/PGW-U | PFCP Session Establishment Request（Create PDR/PDI 的 UE IP Address、UE IP Address Pool Identity） | C面指示U面分配地址及类型 |
| N4 / Sxa / Sxb | UPF/PGW-U → SMF/PGW-C | PFCP Session Establishment Response | U面回传分配的UE IP |
| Gi/SGi/N6 | UPF/PGW-U ↔ DN | OSPF/BGP 路由发布 | 用户下行子段路由发布 |

> 来源：`实现原理/地址分配流程_74936046.md`

### 3.7 其他增值功能

| 增值功能 | 详细解释 |
|---------|---------|
| 地址池优先级功能 | 同一粒度（APN/DNN、位置、SMF）下绑定多个地址池时，支持按地址池优先级分配地址。只有同类地址池之间才有优先级比较，不同类型的地址池间（IPv4和IPv6）没有优先级比较。**优先级仅限于地址池，地址池下的地址段没有优先级顺序** |
| 地址延迟释放功能 | 当新用户接入时，如果获取到的地址为之前用户刚释放的地址，则对地址敏感的周边网元其数据处理可能会存在异常。为避免这类异常，用户去激活后其使用的地址不能被立即释放，而适当等待一段时间后再释放 |
| 地址租期续约功能 | 当用户重新接入时，如果需要继续处理之前在网时可接续的业务，则UDG支持将该用户之前使用的IP地址再次分配给该用户，以接续中断的业务 |
| 地址强制回收功能 | 如果要动态修改或删除某个地址池或地址段，但该地址池或地址段内已经有地址分配完成，这时可通过地址强制回收功能回收已分配的地址。**执行地址强制回收之前，请首先锁定地址池或地址段。强制回收正在使用的地址，会使正常在线的用户将被强制下线，中断用户访问的业务** |
| 防止地址冲突功能 | 当本地地址池中的某些IPv4/IPv6地址与系统或其他设备的地址冲突时，可以通过命令标识该IPv4/IPv6地址为冲突地址，这样可以限制这些IPv4/IPv6地址分配给用户，避免地址冲突 |

> 来源：`实现原理/其他增值功能_87787968.md`

### 4.5 约束条件

| 约束类型 | 约束内容 |
|---------|---------|
| C面网元限制 | 对接的C面网元**必须为华为配套设备**（因N4/Sxa/Sxb对PFCP协议有私有扩展） |
| VPN三者一致 | 地址池绑定的VPN实例、用户激活使用的APN绑定的VPN实例、与PDN/DN连接的Gi/SGi/N6接口的外联口绑定的VPN实例三者必须一致 |
| APN/DNN一致 | UDG本端配置的APN/DNN实例应与C面（SMF/SGW-C/PGW-C）配置的APN/DNN实例一致 |
| 地址池绑定数量 | 每APN/DNN支持绑定的地址池最多16个 |
| 地址段绑定数量 | 每本地地址池支持绑定的地址段最多64个 |
| 主机路由规格 | 每 UDG 支持的主机路由规格为40万 |
| 地址池优先级范围 | 优先级仅限于地址池，地址池下的地址段没有优先级顺序；只有同类地址池之间才有优先级比较，不同类型的地址池间（IPv4和IPv6）没有优先级比较 |
| 冲突IP生效时机 | 若某IPv4地址已分配给用户，此时配置其为冲突地址命令仍可执行成功，但配置在**下次分配该地址时生效**，不影响已在线用户 |
| 强制回收风险 | 执行地址强制回收前必须先锁定地址池或地址段；强制回收正在使用的地址会使正常在线用户被强制下线 |
| 规则匹配失败 | 三级优先级规则都匹配失败时地址分配失败 |
| 真值表未列场景 | PFCP Session Establishment Request 中 BIT 组合未在真值表列出的场景，PGW-U/UPF 均以激活失败处理 |
| ROUTER ID唯一 | OSPF Router ID（SCHEMAROUID）必须全网唯一 |

## 5. 配置案例

### 5.1 典型场景一：基于APN/DNN分配地址（单级优先级）

**场景描述**：UE接入核心网，UDG采用本地地址池为UE动态分配IPv4地址。第一优先级按从APN/DNN映射的地址范围内分配地址，匹配不上则分配失败。指定地址池中的10.1.1.1地址为冲突地址。

**MML命令序列（原样保留）**：

```
//配置VPN实例。
ADD L3VPNINST:VRFNAME="vpn1";
ADD VPNINSTAF:VRFNAME="vpn1",AFTYPE=ipv4uni,VRFRD=100:1;

//基于APN使能地址分配属性。
ADD VPNINST:VPNINSTANCE="vpn1";
ADD APN:APN="apn-test1",HASVPN=ENABLE,VPNINSTANCE="vpn1";
SET APNADDRESSATTR:APN="apn-test1",SUPPORTIPV4=ENABLE,SUPPORTIPV6=ENABLE;

//配置地址池。
ADD POOL:POOLNAME="testpool1",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="testpool1",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.10.1.1",V4ENDIP="10.10.1.10";

//配置地址池绑定到地址池组。
ADD POOLGROUP: POOLGRPNAME="poolgroup1";
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool1";
ADD CONFLICTIP:POOLNAME="testpool1",IPADDRESS="10.1.1.1";

//配置APN与地址池组的映射关系。
ADD POOLGRPMAP: MAPPINGNAME="mapping1", APN="apn-test1", POOLGROUPNAME="poolgroup1";

//配置地址分配规则。
SET IPALLOCRULE: FIRSTRULESW=ENABLE, FIRSTRULE=APN-1&LOCATION-0&SMF-0, SECONDRULESW=DISABLE, THIRDRULESW=DISABLE;

//配置手机下行路由。
ADD OSPF: PROCID=100, VRFNAME="vpn1", SCHEMAROUID="10.10.10.1", LSAARRMAXINTV=1000, LSAARRSTARINTV=500, LSAARRHLDINTV=500;
ADD OSPFAREA:PROCID=100,AREAID="0.0.0.0";
ADD OSPFNETWORK:PROCID=100,AREAID="0.0.0.0", IPADDRESS="10.13.21.0", WILDCARDMASK="0.0.0.255";
ADD OSPFIMPORTROUTE:PROCID=100,TOPOID=0,PROTOCOL=wlr;
```

> 来源：`激活用户面地址分配/基于APN_DNN分配地址_72547232.md`

### 5.2 典型场景二：基于SMF分配地址（单级优先级 + SMF全局开关）

**场景描述**：第一优先级按从SMF映射的地址范围内分配地址。需配置 SMF NodeID 并使能 IPALLOCBYSMFGLBSW。

**MML命令序列（仅展示与场景一差异部分）**：

```
//配置SMF实例与地址池组的映射关系。
ADD CPNODEID: CPNAME="smfnode1", IPV4NODEID="10.0.0.1", IPV6NODEID="FC00:1111:1001:0001:0100:1100:0000:0001", FQDNNODEID="consumer.huawei.com";
ADD POOLGRPMAP: MAPPINGNAME="mapping1", SMF="smfnode1", POOLGROUPNAME="poolgroup1";

//配置地址分配规则。
SET IPALLOCRULE: FIRSTRULESW=ENABLE, FIRSTRULE=APN-0&LOCATION-0&SMF-1, SECONDRULESW=DISABLE, THIRDRULESW=DISABLE;

//配置基于SMF分配地址的开关（两级控制：全局 + 指定SMF；此处配置全局开关）。
SET IPALLOCBYSMFGLBSW: SWITCH=ENABLE;
```

> 说明：文档原文 `ADD POOL:POOLNAME="testpoo11"` 疑似笔误，实际应为 `testpool1`。地址段 `V4STARTIP="10.10.1.21", V4ENDIP="10.10.1.30"`（区别于场景一的 .1.1-.1.10）。
> 来源：`激活用户面地址分配/基于SMF分配地址_87605805.md`

### 5.3 典型场景三：基于SMF+APN/DNN组合分配地址（两级优先级回退）

**场景描述**：第一优先级按SMF+APN映射的地址范围分配，第二优先级回退到从SMF映射的地址范围分配。两条映射规则、两个地址池组，演示地址池优先级（IPV4ALLOCPRIALG/PRIORITY）。

**MML命令序列（核心差异部分）**：

```
//配置地址池2（第二个池，用于SMF单维度回退）。
ADD POOL:POOLNAME="testpool2",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="testpool2",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.10.1.21",V4ENDIP="10.10.1.30";

//配置地址池组2（带优先级算法）。
ADD POOLGROUP: POOLGRPNAME="poolgroup2", IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE;
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup2", POOLNAME="testpool2", PRIORITY=10;
ADD CONFLICTIP:POOLNAME="testpool1",IPADDRESS="10.1.1.1";

//配置SMF实例、APN/DNN实例与地址池组的双映射关系（第一优先级 mapping1=APN+SMF；第二优先级 mapping2=仅SMF）。
ADD CPNODEID: CPNAME="smfnode1", IPV4NODEID="10.0.0.1", IPV6NODEID="FC00:1111:1001:0001:0100:1100:0000:0001", FQDNNODEID="consumer.huawei.com";

### 5.4 典型场景四：基于RADIUS下发地址池名称分配地址

**场景描述**：RADIUS下发地址池名称给SMF，SMF将地址池名称发送给UPF，由UPF从指定地址池内为用户分配IP地址。关键差异：必须配置 IGNOREV4POOLID=DISABLE 和 IGNOREV6POOLID=DISABLE，并使能地址池通配功能。

**MML命令序列（核心差异部分）**：

```
//基于RADIUS下发地址池名称使能地址分配属性（IGNOREV4/V6POOLID必须为DISABLE）。
ADD VPNINST:VPNINSTANCE="vpn1";
ADD APN:APN="apn-test1",HASVPN=ENABLE,VPNINSTANCE="vpn1";
SET APNADDRESSATTR:APN="apn-test1",SUPPORTIPV4=ENABLE,SUPPORTIPV6=ENABLE,IGNOREV4POOLID=DISABLE,IGNOREV6POOLID=DISABLE;

//配置APN与地址池组的映射关系。
ADD POOLGRPMAP: MAPPINGNAME="mapping1", APN="apn-test1", POOLGROUPNAME="poolgroup1";

//配置基于APN的地址分配规则（使用 SET APNIPALLOCRULE，ALLOCATTR=LOCAL）。
SET APNIPALLOCRULE: APN="apn-test1",ALLOCATTR=LOCAL,FIRSTRULESW=ENABLE, FIRSTRULE=APN-1&LOCATION-0&SMF-0, SECONDRULESW=DISABLE, THIRDRULESW=DISABLE;

//配置地址池名称使能通配功能（控制AAA未下发通配符时是否开启通配）。
SET ADDRESSATTR: V4POOLWILDCARDSW=ENABLE, V6POOLWILDCARDSW=ENABLE;
```

> 来源：`激活用户面地址分配/基于RADIUS下发地址池名称分配地址_13796101.md`

## 6. 验证与调测

### 6.1 验证方法

#### 6.1.1 调测前提与目的

用户激活采用用户面分配UE IP地址时，需要调测该功能，确保用户可以正常接入。

> 适用：PGW-U、UPF

#### 6.1.2 调测数据准备

| 类别 | 参数名称 | 取值样例 | 获取方法 |
|------|---------|---------|---------|
| 用户信息查询 | 用户IMSI（IMSI） | 460000123456789 | 测试终端自带 |
| 测试终端使用的APN | APN/DNN实例名（APN） | apn-test1 | 已配置数据中获取 |

工具：测试终端

#### 6.1.3 调测执行步骤

**步骤1**：测试终端使用"apn-test"APN发起接入网络请求。

**步骤2**：执行 `DSP SESSIONINFO` 命令，查看测试终端的IP地址是否在 UDG 本地规划的地址池内。

```
DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
```

预期输出：
```
......
-------------------------------
                                 IMSI  =  460000123456789
                                     ......              
                             PDP Type  =  IPv4 
                                     ......    
                     IPv4 PDP address  =  10.10.10.1         
                                     ......    
                             APN name  =  apn-test1
                                     ......       
(Number of results = 1)
---    END
```

判断：
- 测试终端激活成功，且"IPv4 PDP address"在规划地址池内 → 调测结束
- 测试终端激活成功，但"IPv4 PDP address"不在规划地址池内 → 执行步骤3

**步骤3**：继续查看 `DSP SESSIONINFO`，确认测试终端IP地址类型。

```
DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
```

预期输出：
```
......
-------------------------
         APN name  =  apn-test1
                 ......        
IPv4 Address type  =  UPF ALLOC IP ADDRESS 
                 ......      
(Number of results = 1)
---    END
```

判断：
- `UPF ALLOC IP ADDRESS` → 本地地址池分配（符合预期），执行步骤4进一步定位
- `EXTERNAL ALLOC IP ADDRESS` → 外部地址池分配，需返回配置页面重新配置

**步骤4**：收集信息并寻求技术支持（仅当步骤3显示UPF ALLOC但地址仍不在规划池内）。
- a. 在OM Portal上创建用户跟踪任务并保存报文
- b. 执行 `EXP MML` 命令将当前配置数据导出为MML脚本文件并保存
- c. 收集并保存上述所有查询信息
- d. 收集归纳所有信息并联系华为技术支持解决

> 来源：`调测用户面地址分配_72547233.md`

### 6.4 常见问题与排查

| 问题现象 | 可能原因 | 排查方法 |
|---------|---------|---------|
| 用户激活失败 | 三级优先级规则全部匹配失败 | LST POOLGROUP、LST POOLGRPMAP 检查映射关系；对照 SET IPALLOCRULE 规则字符串 |
| 用户激活成功但地址不在规划池内 | 实际为外部地址池分配 | DSP SESSIONINFO 查 IPv4 Address type，若为 EXTERNAL ALLOC IP ADDRESS 需重新配置 |
| 基于SMF分配不生效 | 未使能 IPALLOCBYSMFGLBSW 或指定SMF开关状态不同 | LST IPALLOCBYSMFGLBSW；确认全局=ENABLE；检查 SET IPALLOCBYSMFSW 是否针对该SMF覆盖 |
| RADIUS下发地址池名不生效 | IGNOREV4/V6POOLID 未置 DISABLE | LST APNADDRESSATTR 确认 IGNORE 标志为 DISABLE |
| 地址池占用率高告警 | ALM-81215/ALM-81216 | DSP POOLUSAGE、DSP SECTIONUSAGE；扩充地址段或新建地址池 |
| 冲突IP仍被分配 | 已在线用户不受影响，下次分配才生效 | 属预期行为；等待用户释放后再观察 |
| 双栈地址仅分到一种 | 仅配置了IPv4或IPv6一种SECTION | 为同一地址池或不同地址池补配另一种 IPVERSION 的 SECTION |
| 下行报文无法回程 | 下行路由未发布 | 检查 ADD OSPFIMPORTROUTE PROTOCOL=wlr 是否配置；检查 OSPF 进程/区域/网络配置 |
| C-U 协商失败 | 对接C面非华为设备 | 应用限制：C面必须为华为配套设备（PFCP私有扩展） |
