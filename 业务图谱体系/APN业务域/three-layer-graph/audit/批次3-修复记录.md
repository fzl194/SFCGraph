## 2. 收尾清理：CR-APN-10 遗留 `mapped:` 标注

### 2.1 问题定位

批次2 修复 CR-APN-07 时，已发现 CR-APN-10 存在同类 `mapped:` 标注（`validation_rule（mapped: parameter_dependency）`），但范围限定为 CR-APN-07，留待本批次处理。本批次完成全图谱 `mapped:` 标注彻底清理。

**清理前 grep 结果**：
```
04-command-graph.md:659: CR-APN-10 | ... | `validation_rule`（mapped: `parameter_dependency`） | ...
```

### 2.2 枚举值选择依据

**Schema §11.6 CommandRule.rule_type 合法枚举**：
```
syntax_rule / semantic_rule / precondition_rule / delete_constraint
/ parameter_mutex / parameter_dependency / object_reference_rule / runtime_check_rule
```

**CR-APN-10 规则逻辑**：IPSec IKE 提议的 DHGROUP 不能为 None；否则 IKE 协商无法完成密钥交换。

**枚举值判定**：
- `validation_rule`：**不在 §11.6 枚举**（属 §10.4 TaskRule.rule_type 与 §9.6 FeatureRule.rule_type）
- `parameter_dependency`（mapped 目标）：语义不符 —— 本规则无依赖参数，DHGROUP 是独立的取值约束
- **`runtime_check_rule`（选择）**：DHGROUP=None 的违规在 IKE 协商运行时才暴露（密钥交换失败），属于运行时检查；与 CR-APN-15（APNACTNUM 并发限制触发拒绝，`runtime_check_rule`）同类模式

### 2.3 修复操作

**文件**：`04-command-graph.md` §4 第 659 行

| 修复前（非法 + mapped） | 修复后（合法枚举） |
|-----------------------|-------------------|
| `\| CR-APN-10 \| IPSec IKE DH 组不能为 None \| \`validation_rule\`（mapped: \`parameter_dependency\`） \| explicit \| restriction \| parameter \| ...` | `\| CR-APN-10 \| IPSec IKE DH 组不能为 None \| \`runtime_check_rule\` \| explicit \| restriction \| parameter \| ...` |
