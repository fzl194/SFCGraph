> **特性范围**：
> - WSFD-010502 地址分配方式（UNC 侧，4 种分配方式：UDM/Radius/DHCP/SMF 本地分配），含 3 个激活子文档（静态地址分配调测、静态IP网段绑定UPF、静态地址黑名单）
> - WSFD-010504 控制面地址分配方式（UNC 侧，仅 SMF 分配；**无激活文档**，仅有参考信息+特性概述）
> **Schema参考**：`三层图谱Schema-最终版-v0.1.md` §11 命令图谱（§11.3 MMLCommand / §11.4 CommandParameter / §11.5 ConfigObject / §11.6 CommandRule / §11.7 关系边）

> **数据来源**：7 篇特性文档（010502：5 篇 + 010504：2 篇）+ 15 篇 MML 命令手册原文（路径见 §抽取核对清单）

### 0.3 全局字段声明（status）

> **适用范围**：本文件 §1 所有 MMLCommand。
> **声明**：本文件 §1 所有 MMLCommand 的 `status` 字段值均为 `active`（Schema §11.3 MMLCommand.status 必备）。WSFD-010502/010504 所有正式命令均处于启用状态。

### 0.4 ★ UDG vs UNC 命令区分核对（铁律：不混）

| 功能域 | UDG 命令（GWFD-010105 样板） | UNC 命令（本文件） | 关键差异 |
|--------|---------------------------|------------------|---------|
| 地址池 | **ADD POOL**（POOLTYPE=LOCAL/EXTERNAL/**MULTICAST**） | **ADD ADDRPOOL**（POOLTYPE=**Local/UDM/Radius/DHCP**） | ★POOLTYPE 取值完全不同：UDG 三类型（本地/外部/组播）；UNC 四类型（本地/UDM签约/Radius/DHCP），UNC 无 MULTICAST，新增 UDM/Radius/DHCP 三种静态分配类型 |
| 地址池组 | ADD POOLGROUP | **ADD ADDRPOOLGRP**（带 POOLGRPTYPE=Local/UDM/Radius/DHCP） | UNC 命令名带 ADDR 前缀 + 类型参数（与 ADDRPOOL 的 POOLTYPE 对应） |
| 池-池组绑定 | ADD POOLBIND**GROUP** | ADD POOLBIND**GRP**（缩写 GRP） | ★命名差异：GROUP vs GRP |
| UPF 组 | （UDG 无此体系） | **ADD ADDRUPGROUP** + **ADD UPFBINDGRP** | ★UNC 独有「地址-UPF 绑定体系」：UPF 组（ADDRUPGROUP）+ UPF-UPF组绑定（UPFBINDGRP），支撑静态地址选 UPF |
| UPF 节点 | （UDG 无） | **ADD UPNODE**（ADDRALLOCMODE=INHERIT/SMF_ALLOC/UPF_FIRST） | ★UNC 独有 UPF 节点命令 + 地址分配属性三态（控制面/用户面分配切换） |
| 池组映射 | ADD POOLGRPMAP（SMF/APN/LOCATION 任意组合） | ADD POOLGRPMAP（UPFGRPNAME/APN/LOCATION 任意组合；**当前版本不支持 LOCATION 参数**） | ★UNC 用 UPFGRPNAME 替代 SMF；位域规则用 `UPNODE-X` 替代 `SMF-X`；010502 静态场景映射 POOLGRP↔UPFGRP |
| 池-APN 绑定 | （UDG 无独立命令，走 POOLGRPMAP 的 APN） | **ADD POOLBINDAPN**（白名单检查场景） | ★UNC 独有：地址池直接绑 APN（白名单检查开关 CHECKIPVALID 配套） |
| 全局分配规则 | SET IPALLOCRULE（位域 `APN-X&LOCATION-X&SMF-X`） | SET IPALLOCRULE（UNC，位域 `APN-X&LOCATION-X&UPNODE-X`） | ★位域第三字段不同：UDG=SMF，UNC=UPNODE |
| APN 地址属性 | SET APNADDRESSATTR（UDG，15 参数） | SET APNADDRESSATTR（UNC，★30+ 参数，含 IPV4/IPV6ALLOCTYPE 分配方式切换） | ★UNC 版参数量翻倍：新增 IPV4/IPV6ALLOCTYPE(LOCAL/RADIUS/DHCP)、IPV4/IPV6RADIUSPRIOR、RDSSVRCTRL、ALLOCPRECEDENCE 等 |
| 冲突地址 | ADD CONFLICTIP（仅 IPv4，2 参数） | ADD CONFLICTIP（UNC，★5 参数，支持 IPv4+IPv6） | ★UNC 版支持 IPv6（IPV6PREFIX/IPV6PREFIXLEN） |
| 黑名单 | （UDG 无） | **ADD BLACKLIST** | ★UNC 独有：静态地址黑名单（基于 VPN，每 VPN 20 列表） |
| 静态地址扫描 | （UDG 无） | **SET STATICADDRPARA** | ★UNC 独有：静态地址用户去活扫描（主锚点与静态绑定UPF不一致时） |
| 位置区全局开关 | （UDG 用 SET IPALLOCBYSMFGLBSW） | **SET IPALLOCBYLOCGLBSW** | ★UDG 是 SMF 分配开关；UNC 是位置区分配开关（功能域不同） |

## 8. 使用实例脚本（保留手册原文 + 激活文档脚本）

### 8.1 配置静态IP地址网段与UPF的绑定（来源：配置静态IP地址网段与UPF的绑定_67401512.md，★UDM 静态分配完整脚本）

```
//创建本地地址池（UDM 类型）。
ADD ADDRPOOL: POOLNAME="pool01", IPVERSION=IPv4, POOLTYPE=UDM, CHECKIPVALID=Enable;

//配置地址池与APN的绑定关系（白名单检查）。
ADD POOLBINDAPN:APN="huawei.com",POOLNAME="pool01";

//配置本地地址池里的IPv4地址段。
ADD SECTION: POOLNAME="pool01", SECTIONNUM=1, IPVERSION=IPv4, V4STARTIP="10.0.0.0", V4ENDIP="10.1.2.3";

//配置地址池组名称（UDM 类型）。
ADD ADDRPOOLGRP: POOLGRPNAME="poolgroup1", POOLGRPTYPE=UDM;

//配置地址池与地址池组绑定。
ADD POOLBINDGRP: POOLGRPNAME="poolgroup1", POOLNAME="pool01";

//配置静态地址用户使用的UPF（地址分配属性 INHERIT）。
ADD UPNODE: NFINSTANCENAME="UPF_Instance_1", ADDRALLOCMODE=INHERIT;

//配置静态地址用户使用的UPF组（UDM 类型）。
ADD ADDRUPGROUP: UPFGRPNAME="upfgroup1", UPFGRPTYPE=UDM;

//配置UPF与UPF组绑定。
ADD UPFBINDGRP: UPFGRPNAME="upfgroup1", UPFID="UPF_Instance_1";

//配置地址池组与UPF组绑定。
ADD POOLGRPMAP: MAPPINGNAME="mapping01", POOLGRPNAME="poolgroup1", UPFGRPNAME="upfgroup1";
```
