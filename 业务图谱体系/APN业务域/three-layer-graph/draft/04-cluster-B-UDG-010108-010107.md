> **Schema 参考**：§11.4 CommandParameter。`required_mode` 取值 `required / optional / conditional_required`（Schema §11.4 必备字段）。本节 6 个独有命令的参数表已从手册原文抽取全量参数。引用的簇B/E 命令参数见对应样板，此处不重复。

### 5.1 STR PDNROUTETST（GWFD-010108 探测启动，★13 参数含完整 IPv4/IPv6 双栈 + 三探测方式）

| 参数 | 类型 | 取值范围 | `required_mode` | 默认值 | 说明 |
|------|------|---------|-----------------|--------|------|
| IPPOOL | string | 1~79 字节，由 `_`/`-`/数字/大小写字母/`.` 组成，不能以 `.` 开头且无连续 `..`，不区分大小写 | `required` | 无 | 地址池名称（引用 ADD POOL 生成） |
| IPVERSION | enum | IPV4 / IPV6 | `required` | 无 | IP 地址类型 |
| SRCIPV4ADDR | IPv4 | 点分十进制 | `conditional_required` | 无 | IPVERSION=IPV4 时可选；探测源 IPv4；★必须是地址池中的 IP，若是冲突地址则先去活会话再探测；不配置时 PING/DNS 从每个路由网段随机选 IP，TRACERT 随机选一个地址网段 IP |
| DSTIPV4ADDR | IPv4 | 点分十进制，不可为广播/全0，必须有效 | `conditional_required` | 无 | IPVERSION=IPV4 时必填；探测目的 PDN Server IP；★不允许配置为 UE 地址或 UPF 设备已配置地址 |
| SRCIPV6PREFIX | IPv6 | 冒分十六进制 X:X:X:X:X:X:X:X | `conditional_required` | 无 | IPVERSION=IPV6 时可选；探测源 IPv6 前缀地址 |
| DSTIPV6PREFIX | IPv6 | 冒分十六进制 X:X:X:X:X:X:X:X | `conditional_required` | 无 | IPVERSION=IPV6 时必填；探测目的 PDN Server IPv6 前缀地址 |
| DSCPV | int | 0~63 | `conditional_required` | 无 | IPVERSION=IPV4 时可选；探测报文 DSCP 值 |
| TCV | int | 0~63 | `conditional_required` | 无 | IPVERSION=IPV6 时可选；探测报文 traffic-class 值 |
| TESTMETHOD | enum | PING / DNS / TRACERT | `optional` | PING | ★路由探测方式（三选一）；默认 PING |
| LENGTH | int | 20~1800 | `conditional_required` | 100 | TESTMETHOD=PING 时可选；探测报文净荷长度 |

> **★手册原文核对**：STR PDNROUTETST 手册「参数说明」表共 13 行参数。IPv4 探测参数（SRCIPV4ADDR/DSTIPV4ADDR/DSCPV）与 IPv6 探测参数（SRCIPV6PREFIX/DSTIPV6PREFIX/TCV）完全对称。TESTMETHOD 默认 PING。三种探测方式的参数依赖：PING→LENGTH、DNS→DOMAINVALUE、TRACERT（无额外参数）。WHOLEDETECT/SECTIONNUM 仅 PING/DNS 可用。

## 8. 使用实例脚本（保留手册原文）

### 8.1 GWFD-010108 用户面地址自动检测（来源：激活用户面地址自动检测_04636908.md）

```
//任务一：以PING探测方式启动PDN/DN侧路由探测功能。
//全量探测。
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=PING, WHOLEDETECT=ENABLE, SECTIONNUM=0;
//非全量探测。
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, SRCIPV4ADDR="10.10.1.1", DSTIPV4ADDR="10.1.1.1", TESTMETHOD=PING;

//任务二：以DNS探测方式启动PDN/DN侧路由探测功能。
//全量探测。
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=DNS, DOMAINVALUE="www.example.com", WHOLEDETECT=ENABLE, SECTIONNUM=0;
//非全量探测。
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=DNS, DOMAINVALUE="www.example.com";

//任务三：以Tracert探测方式启动PDN/DN侧路由探测功能。Tracert探测方式不支持全量探测。
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=TRACERT;

//查询PDN/DN侧路由探测结果。
//所有的探测结果，包括探测成功的结果和探测失败的结果。
DSP PDNTSTRESULT:;
//所有探测成功的结果。
DSP PDNTSTRESULT:DISPLAYMODE=SUCCESS;
//所有探测失败的结果。
DSP PDNTSTRESULT: DISPLAYMODE=FAIL;
```

### 8.2 GWFD-010107 静态地址用户路由冗余 · 主用 UDG 配置（来源：激活静态地址用户路由冗余_19612403.md）

```
//a. 配置地址池信息（★POOLTYPE=EXTERNAL + REDUNDFUNC=ENABLE）。
ADD POOL: POOLNAME="pool1", POOLTYPE=EXTERNAL, IPVERSION=IPV4, HASVPN=ENABLE, VPNINSTANCE="vpn1", REDUNDFUNC=ENABLE;
ADD POOL: POOLNAME="pool2", POOLTYPE=EXTERNAL, IPVERSION=IPV6, HASVPN=ENABLE, VPNINSTANCE="vpn1", REDUNDFUNC=ENABLE;
ADD SECTION: POOLNAME="pool1", SECTIONNUM=1, IPVERSION=IPV4, V4STARTIP="10.1.1.0", V4ENDIP="10.1.1.225";
ADD SECTION: POOLNAME="pool2", SECTIONNUM=2, IPVERSION=IPV6, V6PREFIXSTART="fc00:1::1", V6PREFIXEND="fc00:1::1111", V6PREFIXLENGTH=64;
ADD POOLGROUP: POOLGRPNAME="poolgroup1", IPV4ALLOCPRIALG=ENABLE;
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup1", POOLNAME="pool1";
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup1", POOLNAME="pool2";
ADD APN: APN="test", HASVPN=ENABLE, VPNINSTANCE="vpn1";
ADD POOLGRPMAP: MAPPINGNAME="mapping1", APN="test", POOLGROUPNAME="poolgroup1";

//b. 配置冗余备份重定向IP（★虚拟IP，重定向业务流到GRE Tunnel）。
ADD REDUNDRDTIP: IPVERSION=IPv4, IPV4ADDRESS="192.168.0.1";
ADD REDUNDRDTIP: IPVERSION=IPv6, IPV6ADDRESS="fc00:3::1";

//c. 配置loopBack接口（★绑vpn2，与Tunnel的vpn1不同）。
ADD INTERFACE: IFNAME="LoopBack1", IFADMINSTATUS=up;
ADD IPBINDVPN: IFNAME="LoopBack1", VRFNAME="vpn2";
ADD IFIPV4ADDRESS: IFNAME="LoopBack1", IFIPADDR="10.11.1.1", SUBNETMASK="255.255.255.255";

//d. 配置GRE Tunnel（★REDUNDANCYEN=TRUE）。
ADD INTERFACE: IFNAME="Tunnel1", IFMTU=1500;
ADD GRETUNNEL: TNLNAME="Tunnel1", SRCTYPE=if_name, DSTIPADDR="10.10.0.1", TNLTYPE=gre, SRCIFNAME="LoopBack1", DSTVPNNAME="vpn2", REDUNDANCYEN=TRUE, KEEPALVEN=TRUE;
ADD IPBINDVPN: IFNAME="Tunnel1", VRFNAME="vpn1";
ADD IFIPV4ADDRESS: IFNAME="Tunnel1", IFIPADDR="10.11.1.3", SUBNETMASK="255.255.255.0";
SET IFIPV6ENABLE: IFNAME="Tunnel1", ENABLEFLAG=TRUE, MTU=1500;
ADD IFIPV6ADDRESS: IFNAME="Tunnel1", IPV6ADDRESS="fc00:2::1", PREFIXLEN=64;

//e. 配置IPv4静态路由（★目的=虚拟重定向IP 192.168.0.1）。
ADD SRROUTE: AFTYPE=ipv4unicast, PREFIX="192.168.0.1", MASKLENGTH=32, VRFNAME="vpn1", IFNAME="Tunnel1";
//f. 配置IPv6静态路由（★目的=虚拟重定向IP fc00:3::1）。
ADD SRROUTE6: AFTYPE=ipv6unicast, PREFIX="fc00:3::1", MASKLENGTH=128, VRFNAME="vpn1", IFNAME="Tunnel1";

//g. （可选）下行缺省低优先级静态地址用户网段路由入冗余GRE隧道。
ADD SRROUTE: AFTYPE=ipv4unicast, PREFIX="10.1.1.0", MASKLENGTH=24, VRFNAME="vpn1", IFNAME="Tunnel1", BFDENABLE=FALSE, DHCPENABLE=FALSE;
ADD SRROUTE6: AFTYPE=ipv6unicast, PREFIX="fc00:1::1", MASKLENGTH=64, VRFNAME="vpn1", IFNAME="Tunnel1", BFDENABLE=FALSE, DHCPENABLE=FALSE;

//h. 主备LoopBack接口路由互通（OSPF）。
ADD OSPFINTERFACE: PROCID=6, AREAID="0.0.0.5", IFNAME="LoopBack1";
//i. N6口路由优先级（★主UPF优先级高，COST=100 < 备200）。
MOD OSPFIMPORTROUTE: PROCID=600, TOPOID=0, PROTOCOL=wlr, IMPTCOSTCFG=TRUE, COST=100;
//j. 使能静态地址用户路由冗余功能（★与ADD POOL.REDUNDFUNC双使能）。
SET REDUNDUSER: SWITCH=ENABLE;
```
