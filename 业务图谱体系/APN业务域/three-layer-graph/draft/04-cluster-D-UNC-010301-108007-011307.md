## 3. ConfigObject 间关系边（§11.7）

### 3.1 contains 边（2条）

| 起点 | 关系 | 终点 | 说明 |
|------|------|------|------|
| UPLIST4RDS | `contains` | UPINSTANCEID | RADIUS 使用的 UP 列表包含 UP 实例（每列表最多 64 个 UP；被 UPFRDSCLIENTIP/UPFRDSSVR 绑定时禁止增删） |
| RDSSVRGRP | `contains` | RDSSVR | Radius 服务器组包含 Radius 服务器（主16+备16+抄送64） |

### 3.2 refers_to 边（5条）

| 起点 | 关系 | 终点 | 说明 |
|------|------|------|------|
| NGUSRSECPARA | `refers_to` | UE IMSI | IMSI_PREFIX 匹配用户群（SUBRANGE=IMSI_PREFIX 时 IMSIPRE 5~15 位十进制） |
| CPGTPUADDR | `refers_to` | LOGICIP | GTP-U IP+VPN 必须在 LOGICIP 表已配置（LST LOGICIP 查询） |
| RDSUPFCTRL | `refers_to` | UPLIST4RDS | UPF 控制属性引用 UP 列表同名记录（UPLISTNAME+UPINSTANCEID 必须已存在） |
| UPFRDSSVR | `refers_to` | RDSSVR | 中转服务器 Server IP 引用 ADD RDSSVR 生成的服务器（★中转与直连 Server IP 不可相同） |
| RDSSVR | `refers_to` | VPNINST | Radius 服务器引用 VPN 实例（主备 VPN 必须相同，抄送可不同，最多 5 个 VPN） |

### 3.3 depends_on 边（4条）

| 起点 | 关系 | 终点 | 说明 |
|------|------|------|------|
| RDSUPFCTRL | `depends_on` | UPLIST4RDS | UPF 控制属性依赖 UP 列表先配同名记录（手册原文：执行时需保证 UPLIST4RDS 已添加相同 UPLISTNAME 和 UPINSTANCEID） |
| UPFRDSSVR | `depends_on` | RADIUSCLIENTIP / APNRDSCLIENTIP | 中转服务器依赖 Client IP 接口先配（手册原文：新增中转 UPF 连接 RADIUS 服务器时，需先用 ADD RADIUSCLIENTIP 或 ADD APNRDSCLIENTIP 配 Client IP） |
| UPFRDSCLIENTIP | `depends_on` | UPFRDSSVR | ★顺序强约束：必须先配 ADD UPFRDSSVR 再配 ADD UPFRDSCLIENTIP，否则 AAA 计费业务无法正常（手册原文） |
| UPFRDSCLIENTIP | `depends_on` | UPF: NETWORKINSTVPNMAP | UPF 侧需先执行 ADD NETWORKINSTVPNMAP，否则 SMF 建立 N4-U 会话失败，延迟最大 15 分钟重试（手册原文） |

### 3.4 links 边（2条）

| 起点 | 关系 | 终点 | 说明 |
|------|------|------|------|
| UPFRDSSVR | `links` | (UPLIST4RDS → DN-AAA Server) | 中转 UPF 与 Radius 服务器绑定，建立 SMF→UPF→DN-AAA 的 N4 GTP-U 隧道转接路径 |
| APNRDSSVRGRP | `links` | APN → RDSSVRGRP | APN 绑定 Radius 服务器组（★一个 APN 只能绑一个组） |

## 8. 使用实例脚本（保留手册/激活文档原文，3 个特性各 1 个）

### 8.1 WSFD-010301 5G 鉴权（来源：激活鉴权功能(适用于5G)_66260458.md + 手册使用实例）

```
// 配置所有用户在服务请求流程中需要强制鉴权（激活文档任务示例）。
ADD NGUSRSECPARA: SUBRANGE=ALL_USER, AUTHEVENT=SERVICE_REQUEST, AUTHPERIOD=24;

// 为指定 IMSI 前缀用户增加安全配置，支持所有完整性算法和加密算法（手册使用实例）。
ADD NGUSRSECPARA: SUBRANGE=IMSI_PREFIX, IMSIPRE="123031200100001", INTEGALG=NIA1-1&NIA2-1&NIA3-1, ENCRYALG=NEA0-1&NEA1-1&NEA2-1&NEA3-1;
```

### 8.2 WSFD-108007 终端二次鉴权（来源：激活终端二次鉴权_24743509.md）

```
// 增加与UPF对接的本端GTP-U地址。
ADD CPGTPUADDR:IPVERSION=IPV4,IPV4ADDR="192.168.0.20";

// 向RADIUS服务器使用的UPF列表中增加UPF。
ADD UPLIST4RDS: UPLISTNAME="uplist1", UPINSTANCEID="up1";

// 可选：增加RADIUS服务器的UPF列表中指定UPF的功能属性。
ADD RDSUPFCTRL: UPLISTNAME="uplist1", UPINSTANCEID="up1",PREFERENCE=1,LOCKED=ENABLE;

// 增加中转UPF与Radius服务器的绑定关系（★必须先于 UPFRDSCLIENTIP）。
ADD UPFRDSSVR: SERVERTYPE=AUTHENTICATION, IPVERSION=IPV4, SERVERIPV4="192.168.10.2", UPLISTNAME="uplist1";

// 增加中转UPF与Radius客户端的绑定关系（★必须最后执行，执行后立即触发建链）。
ADD UPFRDSCLIENTIP: CLIENTYPE=AUTHENTICATION, IPVERSION=IPV4, UPLISTNAME="uplist1", VPNINSTANCE="vpnInstance1", CLIENTIPV4="192.168.10.1";
```

### 8.3 WSFD-011307 Radius 抄送（来源：激活支持RADIUS抄送功能_33769357.md）

```
// 配置AAA服务器组（主备模式）。
ADD RDSSVRGRP: RDSSVRGRPNAME="isprg", MODE=MASTER_SLAVE, SIGOPTACCTMSG=DISABLE;

ADD VPNINST: VPNINSTANCE="vpntest";

// 配置AAA计费服务器（主）。
ADD RDSSVR: RDSSVRGRPNAME="isprg", SERVERTYPE=ACCOUNTING, IPVERSION=IPV4, SERVERIPV4="192.168.8.181", PORT=1819, VPNINSTANCE="vpntest", CIPHERKEY="*****", PRIFLAG=PRIMARY;

// 配置AAA鉴权服务器（备）。
ADD RDSSVR: RDSSVRGRPNAME="isprg", SERVERTYPE=AUTHENTICATION, IPVERSION=IPV4, SERVERIPV4="192.168.8.182", PORT=1819, VPNINSTANCE="vpntest", CIPHERKEY="*****", PRIFLAG=SECONDARY;

// 配置抄送服务器（★CARBON_COPY，需先有主备）。
ADD RDSSVR: RDSSVRGRPNAME="isprg", SERVERTYPE=AUTHENTICATION, IPVERSION=IPV4, SERVERIPV4="192.168.8.183", PORT=1819, VPNINSTANCE="vpntest", CIPHERKEY="*****", PRIFLAG=CARBON_COPY;

// 配置APN下绑定AAA服务器组。
ADD APN: APN="apn-test";
ADD APNRDSSVRGRP: APN="apn-test",RDSSVRGRPNAME="isprg";
```

### 9.5 关键发现（★重点核对项）

| 发现项 | 内容 | 影响 |
|--------|------|------|
| **5G 鉴权向量归属澄清** | 5G AKA 四元组(RAND/AUTN/XRES*/K_AUSF) + EAP-AKA' 五元组(RAND/AUTN/XRES/CK'/IK') 由 **UDM 生成**，非本特性 ADD NGUSRSECPARA 命令产出。本特性命令仅控制强制鉴权流程(AUTHEVENT)和算法选择(INTEGALG/ENCRYALG)。★5G AKA 不支持请求多个鉴权向量也不支持预获取。 | CR-010301-02 标注，避免误将鉴权向量建模为本特性命令参数 |
| **108007 协议限制** | 启用终端二次鉴权后 SMF ★不支持 EAP，仅支持 PAP/CHAP；SMF-DN-AAA 仅支持 Radius，不支持 Diameter。 | CR-108007-01 标注为 critical 限制 |
| **108007 顺序强约束** | ADD UPFRDSCLIENTIP ★必须最后执行（执行后立即触发建链）；必须先 ADD UPFRDSSVR；UPF 侧需先 ADD NETWORKINSTVPNMAP（否则 N4-U 会话失败延迟 15 分钟重试）。 | CR-108007-03 标注为 critical 顺序约束 |
