## 1. 概述

### 1.1 特性定义（是什么）

UNC 支持**通过 UDM 分配、Radius 分配、SMF 分配、DHCP 分配四种方式**为用户分配 IP 地址，可以为用户分配 IPv4 地址、IPv6 地址或双栈地址（IPv4 和 IPv6 地址）。

本特性是**控制面（C 面）地址分配决策特性的总集**：SMF/PGW-C/GGSN 决定"地址从哪里来、由谁分配"，并据此指示用户面（U 面）执行或由 C 面直接下发地址给 UE。与 UDG 侧 GWFD-010105（用户面地址分配）形成 C-U 协同分工：C 面决策、U 面执行。

> 来源：`output/UNC 20.15.2 产品文档(裸机容器) 05/网络部署/特性部署/UNC特性指南/会话管理功能/WSFD-010502 地址分配方式/WSFD-010502 地址分配方式特性概述_64915075.md`（"定义"章节）

- 已阅读 `WSFD-010502 地址分配方式特性概述`

> 来源：`配置静态IP地址网段与UPF的绑定_67401512.md`（"必备事项"章节）

### 2.2 通用激活步骤骨架（静态IP段→UPF绑定场景，10步）

```
步骤1：创建本地静态地址地址池（POOLTYPE=UDM，CHECKIPVALID=Enable）
  └── ADD ADDRPOOL

步骤2（可选）：开启静态地址用户白名单检查
  └── ADD POOLBINDAPN     → 绑定地址池与APN，激活时按用户地址查池再选UPF

  └── ADD ADDRPOOLGRP

步骤7：配置静态地址用户使用的UPF组（UPFGRPTYPE=UDM）
```

## 3. 原理

### 3.1 地址分配决策权总览（4种分配方式 + 接入代际差异）

UNC 根据组网情况和用户签约，从以下 4 种地址分配方式中选择决策：

| 分配方式 | 决策源 | UNC 角色 | 适用代际 |
|---------|--------|---------|---------|
| **UDM 分配（静态）** | HLR/HSS/UDM 签约时确定的静态 IP | 接收并下发签约 IP 给 UE；可选基于静态段选择 UPF | 2G&3G / 4G / 5G |
| **Radius 分配** | Radius 服务器在鉴权 Access-Response 中下发 | 从 Radius 获取用户 IP，下发 UE | 2G&3G / 4G / 5G |
| **SMF 分配（本地池）** | SMF 本地地址池 | 从 APN 绑定的本地池或 Radius 池分配 IP | 主要 5G（PGW-C/SMF） |
| **DHCP 分配** | 外部 DHCPv4/DHCPv6 Server | **作 DHCP Client 代理用户申请** | 5G（SMF） |

> 来源：`WSFD-010502 地址分配方式特性概述_64915075.md`（"原理概述"章节）

### 3.2 UDM 静态分配方式（按接入代绩的信元携带差异）

#### 3.2.1 2/3G 场景

承载上下文激活过程中，UE 发送 **Activate PDP Context Request** 给 SGSN，消息的 **PDP Address 信元**携带 UE 的 IPv4 地址/IPv6 前缀。SGSN 验证后通过 **Create PDP Context Request 的 End User Address 信元**将 UE IP 发送给 GGSN，GGSN 将收到的静态 IP/IPv6 前缀发送给 UE 使用。

#### 3.2.2 4G 场景

默认承载上下文激活过程中，MME 根据请求和 HSS 签约数据，在 **Create Session Request 的 PDN Address Allocation（PAA）信元**携带已获取的 IPv4 地址或 IPv6 前缀给 SGW-C，SGW-C 再通过 Create Session Request 将 PAA 发送给 PGW-C，PGW-C 将静态 IP 发送给 UE。

#### 3.2.3 5G 场景

PDU 会话建立时，SMF 通过调用 UDM 服务操作 **Nudm_SDM_Get** 获取 UE 在 UDM 签约时确定的静态 IP 地址/前缀，之后通过 PDU 会话建立响应消息发送给 UE。

#### 3.2.4 静态 IP 段→UPF 选择增强（v02 新增）

为了让指定用户选择到特定的 UPF（如企业用户定制服务），可在 UNC 上配置**静态 IP 地址网段与 UPF 的绑定**：

- 当静态 IP 地址网段绑定的 UPF 与 SMF 选择的主锚点 UPF **不一致**时 → **优先使用 SMF 选择的主锚点 UPF 建立会话**
- 如果已有会话的主锚点 UPF 与静态 IP 绑定的 UPF 不一致 → 可根据 `SET STATICADDRPARA` 命令中的开关配置**时间扫描会话去活**

> 来源：`WSFD-010502 地址分配方式特性概述_64915075.md`（"原理概述/UDM 分配方式"）、`配置静态IP地址网段与UPF的绑定_67401512.md`（"操作步骤/说明"）

### 3.3 Radius 分配方式

为用户分配的 IP 是在 PDP/承载/会话上下文激活阶段，在**用户鉴权过程**中从 Radius 服务器获取的 IP 地址，通过 Radius 认证过程从 Radius 服务器获取，地址直接携带在服务器返回的**接入响应消息（Access-Accept）**中。

> 来源：`WSFD-010502 地址分配方式特性概述_64915075.md`（"原理概述/Radius 地址分配方式"）

### 3.4 SMF 本地分配方式（含延迟释放与租期续约）

SMF 为用户分配的 IP 是在 PDU 会话建立阶段直接从 **SMF 的本地地址池**中获取的 IP 地址。

**分配逻辑**：
- UNC 收到激活请求 → 如果用户所属 APN 配置的是本地地址分配方式 → UNC 从用户所属 APN 绑定的**本地地址池**或 **Radius 地址池**中为用户分配一个 IP（IPv4 地址或 IPv6 地址前缀）

**地址延迟释放**：
- 当用户去激活后，UNC 可设置为该用户分配的 IP **延迟释放**，待本地地址池中 IP 地址的**等待释放时间**到达后，UNC 才将该地址重新分配给其他用户

**地址租期续约**：
- 如果在租约期内该用户再次创建 PDU 会话，可**继续使用原 IP 地址**（接续中断业务）

> 来源：`WSFD-010502 地址分配方式特性概述_64915075.md`（"原理概述/SMF 分配方式"）

### 3.5 DHCP 代理申请方式

DHCP 分配地址的方式指移动用户所分配的 IP 是在 PDU 会话建立阶段**从 DHCP 服务器上配置的地址池中获取的动态 IP**。**UNC 作为 DHCP Client，代理用户向 DHCP Server 申请 IP 地址**。

**代理流程**：
- UNC 收到用户激活请求 → **携带代理 IP 向 DHCP Server/DHCPv6 Server 申请 IP**
- DHCP Server/DHCPv6 Server 从**与该代理 IP 相同网段的地址池**中为用户分配一个 IPv4/IPv6 地址

> 来源：`WSFD-010502 地址分配方式特性概述_64915075.md`（"原理概述/DHCP 地址分配方式"）

### 3.6 静态地址黑名单（地址合法性校验）

UNC 在本地配置静态地址黑名单，判断用户携带地址的合法性。若用户地址出现于黑名单中，标明地址非法，**不允许接入**。支持绑定 VPN 实例（HASVPN/VPNINSTANCE）。

> 来源：`配置静态地址黑名单_48452479.md`（"操作场景"）

### 4.4 约束条件

| 约束类型 | 约束内容 |
|---------|---------|
| C-U 数据一致性 | UNC 配置的用户静态地址网段信息**必须包含在 UDG（GWFD-010104）的地址池内**（硬约束） |

| 黑名单粒度 | 按 IPv4/IPv6 地址段范围匹配；命中即拒绝接入 |
| UPF 实例来源 | ADD UPNODE 的 NFINSTANCENAME 必须已通过 ADD PNFPROFILE 配置 |

## 5. 配置案例

### 5.1 典型场景一：静态IP地址网段→UPF绑定（企业园区场景）

**场景描述**：某企业用户接入网络时采用静态 IP 分配方式。开启静态地址用户白名单检查功能，需要为 **10.0.0.0~10.1.2.3 地址段内的用户选择到 UPF 名称为 UPF_Instance_1 的园区 UPF**。UPF_Instance_1 上已配置包含网段 10.0.0.0~10.1.2.3 的地址池信息，需要在 UNC 上配置静态 IP 段到 UPF_Instance_1 的关系，为用户选择到指定 UPF。

**MML命令序列（原样保留）**：

```
//创建本地地址池。
ADD ADDRPOOL: POOLNAME="pool01", IPVERSION=IPv4, POOLTYPE=UDM, CHECKIPVALID=Enable;

//配置地址池与APN的绑定关系。
ADD POOLBINDAPN:APN="huawei.com",POOLNAME="pool01";

//配置本地地址池里的IPv4地址段。
ADD SECTION: POOLNAME="pool01", SECTIONNUM=1, IPVERSION=IPv4, V4STARTIP="10.0.0.0", V4ENDIP="10.1.2.3";

//配置地址池组名称。
ADD ADDRPOOLGRP: POOLGRPNAME="poolgroup1", POOLGRPTYPE=UDM;

//配置地址池与地址池组绑定。
ADD POOLBINDGRP: POOLGRPNAME="poolgroup1", POOLNAME="pool01";

//配置静态地址用户使用的UPF。
ADD UPNODE: NFINSTANCENAME="UPF_Instance_1", ADDRALLOCMODE=INHERIT;

//配置静态地址用户使用的UPF组。
ADD ADDRUPGROUP: UPFGRPNAME="upfgroup1", UPFGRPTYPE=UDM;

//配置UPF与UPF组绑定。
ADD UPFBINDGRP: UPFGRPNAME="upfgroup1", UPFID="UPF_Instance_1";

//配置地址池组与UPF组绑定。
ADD POOLGRPMAP: MAPPINGNAME="mapping01", POOLGRPNAME="poolgroup1", UPFGRPNAME="upfgroup1";
```

> 来源：`配置静态IP地址网段与UPF的绑定_67401512.md`（"任务示例"）

### 5.2 典型场景二：配置静态地址黑名单

**场景描述**：运营商判断 **10.1.1.1~10.1.1.10 地址段内的地址非法**，将其配置为静态地址黑名单，用户地址出现于黑名单中时不允许接入。

**MML命令序列（原样保留）**：

```
//创建静态地址黑名单。
ADD BLACKLIST: NAME="testblacklist", IPVERSION=IPv4, V4STARTIP="10.1.1.1", V4ENDIP="10.1.1.10", HASVPN=Enable, VPNINSTANCE="testvpn";
```

> 来源：`配置静态地址黑名单_48452479.md`（"任务示例"）

### 5.3 场景变体对照

| 变体 | 核心差异 | 关键命令/参数 | 文档覆盖度 |
|------|---------|--------------|-----------|
| UDM 静态分配 | 来自 HLR/HSS/UDM 签约 | 原理信元：PDP Address / PAA / Nudm_SDM_Get | 仅原理，无独立激活脚本 |
| UDM 静态 + 段→UPF绑定 | 静态段绑定园区 UPF | ADD ADDRPOOL POOLTYPE=UDM + ADD UPNODE/ADDRUPGROUP/UPFBINDGRP/POOLGRPMAP | 完整激活脚本（场景一） |
| Radius 分配 | 鉴权响应下发 IP | 原理：Access-Accept 携带；UNC 侧激活未覆盖 | 仅原理 |
| SMF 本地池分配 | SMF 本地池/Radius 池 | 文档未提供独立 MML 脚本 | 仅原理 |
| DHCP 代理申请 | UNC 作 DHCP Client | 文档未提供独立 MML 脚本 | 仅原理 |
| 静态地址黑名单 | 黑名单拒绝接入 | ADD BLACKLIST（独立体系） | 完整激活脚本（场景二） |

> 说明：5 篇产品文档中，仅"静态IP段→UPF绑定"和"静态地址黑名单"两个子场景提供了完整 MML 激活脚本；Radius/SMF本地池/DHCP 三种分配方式**仅在原理章节描述**，对应 MML 命令需参考 Radius 功能（WSFD-011306）、DHCP功能（WSFD-104413）等关联特性（Stage 3 补充）。

## 6. 验证与调测

### 6.1 验证方法（静态地址分配 IPv4 功能）

#### 6.1.1 调测前提与目的

当已经部署地址分配方式特性时，需要对本功能进行调测，确保功能可以正常使用。

> 适用：GGSN、PGW-C、SMF

#### 6.1.2 调测数据准备

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
|------|---------|---------|---------|------|
| 用户信息查询 | 用户IMSI号（IMSI） | 123000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN名称（APN） | apn-test | 从已完成的配置数据中获取 | 取自 UE 透明接入运营商自身 IP 网络中配置的 APN 实例名 |
| 地址池名称 | 地址池名称（POOLNAME） | pool01 | 从已完成的配置数据中获取 | - |

工具：测试终端

#### 6.1.3 调测执行步骤

**步骤1**：测试终端使用 "apn-test" APN 发起接入网络请求，创建 PDU 会话标识为 1 的 PDU 会话。

**步骤2**：进入 "MML命令行-UNC" 窗口。执行 **DSP PDUSESSION** 命令，通过测试终端的 IMSI 查看测试终端是否成功激活。

- 如果测试终端激活成功，且使用的 "用户IP地址" 与 "UP的PFCP IPv4地址" **与规划值一致** → 测试终端接入成功，调测结束
- 如果测试终端激活成功，但使用的 "用户IP地址" 或 "UP的PFCP IPv4地址" **与规划值不一致** → 执行步骤3

**步骤3**：收集信息并寻求技术支持。
- a. 在 OM Portal 创建用户跟踪任务，并保存报文
- b. 执行 **EXP MML** 命令查看当前配置并保存查询结果
- c. 收集并保存上述所有查询信息
- d. 收集归纳所有信息并联系华为技术支持解决

> 来源：`调测静态地址分配IPv4地址功能_67401513.md`（"操作步骤"）

### 6.5 常见问题与排查

| 问题现象 | 可能原因 | 排查方法 |
|---------|---------|---------|
| 静态地址用户选错 UPF | 静态IP段绑定UPF 与 SMF主锚点UPF 冲突，且SMF主锚点UPF优先级更高 | 确认预期：文档明确 SMF 主锚点 UPF 优先；若需切换需调整 SMF UPF 选择策略或 SET STATICADDRPARA 会话去活开关 |
| 静态地址用户激活失败 | UNC 静态段未包含在 UDG 地址池内（C-U 数据不一致） | 核对 UNC 静态段 ⊆ UDG（GWFD-010104）地址池 |
| 静态地址用户激活失败 | ADDRPOOL 配置但未配 SECTION | 补配 ADD SECTION |
| 地址池地址段超限 | 一个池超过 64 个地址段 | 拆分多个地址池 |
| 静态地址用户被拒绝接入 | 用户 IP 命中黑名单 | LST BLACKLIST 查黑名单配置；评估是否应移出黑名单 |
| UPF 节点未找到 | NFINSTANCENAME 未通过 ADD PNFPROFILE 配置 | 先配置 ADD PNFPROFILE，再 ADD UPNODE |

### 8.2 与 GWFD-010105（UDG 侧）的 C-U 协同与不一致

| # | 维度 | GWFD-010105（UDG/U面） | WSFD-010502（UNC/C面） | 协同/差异 |
|---|------|------------------------|------------------------|----------|
| 1 | 角色定位 | U面执行方：按C面指示从本地池选地址 | C面决策方：决定地址来源（4种方式） | **C-U 协同分工**：决策在C面，执行在U面 |
| 2 | 分配方式粒度 | 4种子方式（APN/DNN、位置、SMF、Radius下发池名）+ 组合 | 4种分配方式（UDM、Radius、SMF本地池、DHCP） | **不对称**：U面按"匹配维度"细分（APN/SMF/位置），C面按"地址来源"分类（UDM/Radius/SMF/DHCP） |
| 3 | 地址池命令命名 | ADD POOL（POOLTYPE=LOCAL） | ADD ADDRPOOL（POOLTYPE=UDM） | **命令前缀差异**：U面为 POOL，C面为 ADDRPOOL；POOLTYPE 取值不同（LOCAL vs UDM） |
| 4 | 地址池组命令命名 | ADD POOLGROUP / ADD POOLBINDGROUP | ADD ADDRPOOLGRP / ADD POOLBINDGRP | **命名风格差异**：U面 POOLGROUP，C面 ADDRPOOLGRP |
| 5 | IPv4/IPv6类型参数 | 文档中主要使用 IPV4（大写） | 文档中 IPv4/IPv6（混合大小写） | **取值风格差异**：执行时需按各侧命令规范填入 |
| 6 | IPV4ALLOCTYPE/IPV6ALLOCTYPE | GWFD-010105 文档未明确出现此参数名 | WSFD-010502 文档亦未明确出现此参数名 | **任务关注点澄清**：用户重点提到的 IPV4ALLOCTYPE/IPV6ALLOCTYPE 参数在两侧 WSFD-010502/GWFD-010105 产品文档中**均未显式定义**；实际指示 IPv4/IPv6 分配类型的是 PFCP Session Establishment Request 中的 **CHV4/CHV6/V4/V6 BIT位组合**（GWFD-010105 真值表，详见该文档 §3.4.2） |
| 7 | 静态地址段约束 | UDG 配置地址池（含网段） | UNC 静态段**必须包含在** UDG 地址池内 | **C-U 数据一致性硬约束**（文档明确引用 GWFD-010104） |
| 8 | UPF选择冲突协调 | UDG 侧无此问题（执行方） | 静态IP段绑定UPF 与 SMF主锚点UPF冲突时 → **SMF主锚点UPF优先**，由 SET STATICADDRPARA 控制去活 | **C面决策权优先级**：SMF 选择 > 静态IP段绑定 |
| 9 | 地址分配决策权 | U面无决策权（按C面指示） | **C面拥有决策权**：选哪种方式、指示U面是否分配及类型 | 决策权统一在C面 |
| 10 | License | 无License | 无License | 一致 |
| 11 | 应用限制 | 必须对接华为C面（PFCP私有扩展） | 无应用限制 | **差异**：限制在U面侧（对接华为C面），C面侧本身不涉及此限制 |
| 12 | 特性规格 | 40万主机路由、16池、64段 | 无特殊规格 | 差异：规格约束在U面侧 |
| 13 | 告警/软参/测量指标 | 有（ALM-81215/81216、BYTE44/120/BIT1260/BYTE658、9条指标） | **全部为"无"** | 差异：运维观测点在U面侧，C面侧无独立告警/软参/指标 |
| 14 | 代际支持 | UPF/PGW-U 不分代际（5G/4G执行方） | 明确分 **2/3G（GGSN）、4G（PGW-C）、5G（SMF）** 三代理绩 | 差异：C面需处理代际信元差异（PDP Address/PAA/Nudm_SDM_Get） |
