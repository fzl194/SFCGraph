## 1. 概述

### 1.1 特性定义（是什么）

UDG 支持创建 IPv4v6 双栈，使 UE 可以用 IPv4v6 地址进行数据传输。UDG 能够为用户创建 IPv4v6 双栈类型的 PDP 上下文 / EPS 承载 / PDU Session / QoS Flow，IPv6 类型数据包与 IPv4 类型数据包一样在接口上透明传输；当 UE 进行数据业务时，UDG 可将 IPv4 报文或 IPv6 报文传输至 PDN/DN 网络。

> 来源：`output/UDG_Product_Documentation_CH_20.15.2/特性部署/特性指南/UDG特性指南/IPv6功能/GWFD-020403 IPv4v6双栈接入/GWFD-020403 IPv4v6双栈接入特性概述_38276179.md`（"定义"、"原理概述"章节）

### 1.8 应用场景

本特性应用于网络中存在**双栈类型的UE接入**的情况。当运营商因 IPv4 地址枯竭而向 IPv6 演进，但现网仍存在大量 IPv4 业务需要同时承载时，通过部署本特性使同一用户在同一次 PDU 会话中同时获得 IPv4 和 IPv6 地址。

> 来源：`GWFD-020403 IPv4v6双栈接入特性概述_38276179.md`（"应用场景"章节）

### 1.9 对系统的影响

- 随着 IPv4v6 双栈用户数的增加，该特性对系统的影响越大。
- IPv4/v6 双栈改造后，UE 需通过 UDG 下发"Router Advertisement"数据报文以获取 IPv6 地址。当 UE 处于弱覆盖的场景下，eNodeB 无法下发该报文给终端，**掉话率会增加**。相关原理请参考 UNC 产品文档的特性"IPv6 前缀代理"中 5GC 业务流程的介绍。

> 来源：`GWFD-020403 IPv4v6双栈接入特性概述_38276179.md`（"对系统的影响"章节）

## 2. 核心概念

### 2.1 关键术语

| 术语 | 全称 | 说明 |
|------|------|------|
| IPv4v6双栈 | IPv4v6 Dual Stack | 同一用户上下文中同时承载 IPv4 与 IPv6 两种协议栈，UE 同时拥有 IPv4 地址与 IPv6 地址 |
| PDP Type | Packet Data Protocol Type | PDP上下文/EPS承载/PDU Session的协议类型，取值 IPv4 / IPv6 / IPv4v6 / IPv4orIPv6（本特性聚焦 IPv4v6） |
| PDP上下文 | PDP Context | 2G/3G 分组数据上下文（双栈在此层创建） |
| EPS承载 | EPS Bearer | 4G 演进分组系统承载（双栈在此层创建） |
| PDU Session | 协议数据单元会话 | 5G PDU会话（双栈在此层创建） |
| QoS Flow | 服务质量流 | 5G QoS Flow（双栈在此层创建） |
| Router Solicitation (RS) | 路由器请求 | UE 发往 UDG 的请求消息，用于请求 IPv6 路由信息 |
| Router Advertisement (RA) | 路由器通告 | UDG 发往 UE 的通告消息，给 IPv6 用户通告路由、IP 地址前缀等信息 |
| IPv6地址前缀 | IPv6 Address Prefix | IPv6 地址的网络前缀部分，双栈配置中 IPv6 池需规划 V6PREFIXLENGTH（49~64） |
| IPv6 Prefix Delegation | IPv6前缀代理 | 当 V6PREFIXLENGTH < 64 时表示该地址池采用 IPv6 Prefix Delegation 方式分配（对应 GWFD-020406） |
| 双栈VPN | Dual Stack VPN | APN 同时绑定 IPv4 VPN（HASVPN）与 IPv6 VPN（HASVPNIPV6）的配置形态 |

### 2.2 对象模型（双栈配置体系）

IPv4v6 双栈接入的配置架构基于"双栈 APN + 双类型地址池 + APN-地址池组映射"的对称双栈模型：

```
                      ┌──────────────────────────────┐
                      │   License 使能层（前置开关）   │
                      │   SET LICENSESWITCH           │
                      │   LICITEM=LKV3G5VDSA01        │
                      └───────────────┬──────────────┘
                                      │
                                      ▼
        ┌─────────────────────────────────────────────────┐
        │  双栈 APN（IPv4 + IPv6 对称绑定）                 │
        │  ADD APN                                         │
        │    HASVPN=ENABLE, VPNINSTANCE="VRF_Internet"     │
        │    HASVPNIPV6=ENABLE, VPNINSTANCEIPV6="VRF_Internet"│
        └─────────────────────┬───────────────────────────┘
                              │
                ┌─────────────┴─────────────┐
                ▼                           ▼
   ┌────────────────────────┐   ┌────────────────────────┐
   │  IPv4 地址池（testpool）│   │ IPv6 地址池（testpoolv6）│
   │  ADD POOL               │   │  ADD POOL               │
   │    POOLTYPE=LOCAL       │   │    POOLTYPE=LOCAL       │
   │    IPVERSION=IPV4       │   │    IPVERSION=IPV6       │
   │    HASVPN=ENABLE        │   │    HASVPN=ENABLE        │
   │    VPNINSTANCE=...      │   │    VPNINSTANCE=...      │
   │  ADD SECTION            │   │  ADD SECTION            │
   │    IPVERSION=IPV4       │   │    IPVERSION=IPV6       │
   │    V4STARTIP/V4ENDIP    │   │    V6PREFIXSTART/END    │
   └───────────┬────────────┘   └────────────┬───────────┘
               │                            │
               └────────────┬───────────────┘
                            │
                  ADD POOLBINDGROUP（双池绑定到同一组）
                            │
                            ▼
              ┌──────────────────────────────┐
              │   地址池组 poolgroup1          │
              │   IPV4ALLOCPRIALG=ENABLE      │
              │   IPV6ALLOCPRIALG=ENABLE      │
              └──────────────┬───────────────┘
                             │
                  ADD POOLGRPMAP
                             │
                             ▼
              ┌──────────────────────────────┐
              │   APN ↔ 地址池组 映射          │
              │   APN="apn-test"              │
              │   POOLGROUPNAME="poolgroup1"  │
              └──────────────────────────────┘
```

核心配置对象说明：

| 对象 | 作用 | 双栈场景角色 |
|------|------|------------|
| APN（双栈形态） | 接入点配置，同时绑定 IPv4 VPN 和 IPv6 VPN | 通过 HASVPN + HASVPNIPV6 双开关对称使能双栈 |
| POOL（IPv4） | IPv4 本地地址池 | 承载 V4STARTIP/V4ENDIP 地址段 |
| POOL（IPv6） | IPv6 本地地址池 | 承载 V6PREFIXSTART/V6PREFIXEND 地址段，含 V6PREFIXLENGTH |
| SECTION（IPv4） | IPv4 地址段 | 双栈场景的 IPv4 段 |
| SECTION（IPv6） | IPv6 地址段 | 双栈场景的 IPv6 段，前缀长度 49~64 |
| POOLGROUP | 地址池组 | 同时绑定 IPv4 与 IPv6 两类 POOL，双栈分配的统一容器 |
| POOLBINDGROUP | 地址池到组绑定 | 将 IPv4 池与 IPv6 池分别绑定到同一 POOLGROUP |
| POOLGRPMAP | APN 到地址池组映射 | 指示双栈 APN 从该 POOLGROUP 分配（双栈）地址 |

### 2.3 在 APN 地址分配场景的角色

GWFD-020403 在 APN 地址分配场景中扮演"**IPv4v6 双栈能力使能层**"的角色，与 GWFD-010105（用户面地址分配）形成"使能 / 分配机制"分层协同关系（详见 §7.1）：

```
┌──────────────────────────────────────────────────────────────┐
│ Layer A: 双栈能力使能层（GWFD-020403，License LKV3G5VDSA01） │
│   - 控制是否允许创建 IPv4v6 类型的 PDP上下文/EPS承载/PDU会话  │
│   - 控制 UDG 是否向双栈用户主动下发 Router Advertisement     │
│   - 控制双栈对称配置形态（双VPN、双池、双段）                │
└──────────────────────────┬───────────────────────────────────┘
                           │ 依赖其地址分配基础设施
                           ▼
┌──────────────────────────────────────────────────────────────┐
│ Layer B: 地址分配机制层（GWFD-010105，无独立License）        │
│   - 4种子方式（APN/DNN、位置、SMF、RADIUS下发地址池名）      │
│   - 三级优先级规则                                            │
│   - PFCP 信元协商（CHV4/CHV6/V4/V6）地址类型                 │
│   - 本地地址池体系（POOL/SECTION/POOLGROUP/POOLGRPMAP）      │
└──────────────────────────────────────────────────────────────┘
```

具体角色：

1. **能力使能**：GWFD-020403 通过 License + 双栈 APN 配置，使 UDG 具备创建 IPv4v6 双栈上下文的能力。没有本特性使能时，UDG 不能为用户提供双栈接入。
2. **地址类型参数的上层语义**：GWFD-010105 的 PDP Type 参数（IPv4/IPv6/IPv4v6）的 **IPv4v6 取值**，需本特性的 License 使能才可在 UDG 侧落地为真正的双栈上下文。
3. **IPv6 通告机制**：本特性明确描述了 UDG 下发 Router Advertisement 给 IPv6/双栈用户的机制（GWFD-010105 的 10 个文档未涉及此 RA 机制）。

## 3. 原理与流程

### 3.1 实现原理：双栈创建与IPv6地址通告

#### 3.1.1 双栈上下文创建机制

承载建立完成后，UDG 能够为用户创建 **IPv4v6 双栈类型**的 PDP 上下文 / EPS 承载 / PDU Session / QoS Flow。在双栈上下文中：

- IPv6 类型的数据包与 IPv4 类型的数据包**一样，在接口上透明传输**
- 当 UE 进行数据业务时，UDG 可将 IPv4 报文或 IPv6 报文传输至 PDN/DN 网络

```
┌─────────────────────────────────────────────────────────┐
│                  双栈用户报文转发                          │
│                                                         │
│   UE (持有 IPv4 + IPv6 双地址)                           │
│      │                                                  │
│      ├── IPv4 报文 ──→ UDG ──→ PDN/DN (IPv4 网络)       │
│      │                                                  │
│      └── IPv6 报文 ──→ UDG ──→ PDN/DN (IPv6 网络)       │
│                                                         │
│   两种协议栈在 UDG 接口上均透明传输，互不干扰              │
└─────────────────────────────────────────────────────────┘
```

> 来源：`GWFD-020403 IPv4v6双栈接入特性概述_38276179.md`（"原理概述"章节）

#### 3.1.2 IPv6 地址通告机制（Router Advertisement）

承载建立完成后，UE 可能会向 UDG 发送 **Router Solicitation** 消息，获取 IPv6 地址信息。关键行为：

- **无论是否收到 Router Solicitation 消息**，只要用户的 **PDP Type** 是 **IPv6** 或者 **IPv4v6**，UDG 都会向 UE 发送 **Router Advertisement** 消息
- RA 消息用于给 IPv6 用户通告路由、IP 地址前缀等信息

> 文档原文：承载建立完成后，UE可能会向UDG发送Router Solicitation消息，获取IPv6地址信息。无论是否收到Router Solicitation消息，只要用户的PDP Type是IPv6或者IPv4v6，UDG都会向UE发送Router Advertisement消息，给IPv6用户通告路由、IP地址前缀等信息。

> 来源：`GWFD-020403 IPv4v6双栈接入特性概述_38276179.md`（"原理概述"章节）

#### 3.1.3 IPv6 地址前缀长度与 PD 关系

激活文档明确了 IPv6 地址段的前缀长度语义：

- IPv6 前缀长度范围为 **49~64**
- **当前缀长度小于 64 时**，则表示该地址池采用 **IPv6 Prefix Delegation 方式分配**（即与 GWFD-020406 IPv6 Prefix Delegation 特性协同）

> 来源：`激活IPv4v6双栈接入_38276180.md`（"数据"表 ADD SECTION V6PREFIXLENGTH 行说明）

### 3.2 业务流程（IPv4+IPv6 并发分配端到端）

基于 GWFD-020403 文档（RA 通告机制）与 GWFD-010105 文档（PFCP 地址类型协商、真值表 CHV4=1&CHV6=1）综合，双栈地址并发分配流程如下：

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 承载建立（PDU会话建立请求携带 PDU Type = IPv4v6）              │
│    UE 通过 SMF/PGW-C 发起会话建立，请求双栈接入                   │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. C面决策 + PFCP 地址类型协商                                    │
│    SGW-C/PGW-C/SMF 通过 PFCP Session Establishment Request 指示  │
│    UDG 分配 IPv4v6 双栈（CHV4=1, CHV6=1, V4=0, V6=0）            │
│    （详见 GWFD-010105 §3.4 真值表第6行）                          │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. UDG 双栈地址分配（双池并发）                                   │
│    UDG 根据 APN→POOLGROUP 映射，从 poolgroup1 中：               │
│    - 从 IPv4 池 testpool 分配一个 IPv4 地址                       │
│    - 从 IPv6 池 testpoolv6 分配一个 IPv6 地址                     │
│    两类地址在同一次会话中并发分配（非先后）                       │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. IPv6 路由通告（RA 机制，★本特性核心）                         │
│    无论是否收到 UE 的 RS，只要 PDP Type = IPv4v6：                │
│    UDG 主动向 UE 发送 Router Advertisement 消息                  │
│    通告 IPv6 路由、IP 地址前缀                                    │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 5. 地址回传与双栈上下文建立完成                                   │
│    a. UDG 通过 PFCP Session Establishment Response 回传          │
│       IPv4 PDP address + IPv6 PDP address 给 C 面                │
│    b. C 面传递给 UE，UE 同时获得 IPv4 与 IPv6 地址                │
│    c. IPv4v6 双栈 PDP 上下文/EPS承载/PDU Session 创建完成        │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 6. 双栈数据传输                                                   │
│    UE 后续可同时进行 IPv4 报文和 IPv6 报文的收发                  │
│    UDG 在接口上对两类报文均透明传输至 PDN/DN                      │
└─────────────────────────────────────────────────────────────────┘
```

> 说明：步骤2/3/5的PFCP信元协商细节见 GWFD-010105 §3.4；步骤4的 RA 机制是 GWFD-020403 文档的**独有贡献**（010105未涉及）。

> 来源：`GWFD-020403 IPv4v6双栈接入特性概述_38276179.md`（"原理概述"章节）；GWFD-010105 文档 §3.4 真值表（双栈对应 CHV4=1&CHV6=1）

### 3.3 协议交互

| 接口 | 交互网元 | 消息类型 | 说明 |
|------|---------|---------|------|
| N4（5G）/ Sxa / Sxb（4G） | SMF/PGW-C ↔ UPF/PGW-U | PFCP Session Establishment Request/Response | C面指示双栈分配（CHV4=1&CHV6=1）；U面回传分配的 IPv4+IPv6 地址 |
| Uu / 空口（用户面） | UDG → UE | **Router Advertisement** | UDG 向 IPv4v6 双栈用户通告 IPv6 路由与前缀（★本特性核心机制） |
| Uu / 空口（用户面） | UE → UDG | Router Solicitation（可选） | UE 请求 IPv6 路由信息（UDG 不依赖此消息也会主动发 RA） |
| Gi/SGi/N6 | UPF/PGW-U ↔ DN | IPv4 报文 / IPv6 报文透明传输 | 双栈数据转发 |

> 来源：`GWFD-020403 IPv4v6双栈接入特性概述_38276179.md`（"原理概述"章节）

## 4. 配置规则

### 4.1 激活步骤

IPv4v6 双栈接入特性的激活和配置流程：

```
步骤1：打开本特性的 License 配置开关
  └── SET LICENSESWITCH:LICITEM="LKV3G5VDSA01",SWITCH=ENABLE;

步骤2：配置双栈 APN（IPv4 + IPv6 对称绑定）
  └── ADD APN:APN=..., HASVPN=ENABLE, VPNINSTANCE=...,
              HASVPNIPV6=ENABLE, VPNINSTANCEIPV6=...

步骤3：配置地址池组
  └── ADD POOLGROUP:POOLGRPNAME=..., 
        IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE

步骤4：配置 IPv4 及 IPv6 的地址池（双池）
  ├── ADD POOL:POOLNAME="testpool",   IPVERSION=IPV4, ...
  └── ADD POOL:POOLNAME="testpoolv6", IPVERSION=IPV6, ...

步骤5：配置 IPv4 及 IPv6 的地址池里的地址段（双段）
  ├── ADD SECTION:POOLNAME="testpool",   IPVERSION=IPV4, 
  │             V4STARTIP=..., V4ENDIP=...
  └── ADD SECTION:POOLNAME="testpoolv6", IPVERSION=IPV6, 
              V6PREFIXSTART=..., V6PREFIXEND=..., V6PREFIXLENGTH=64

步骤6：将 IPv4 及 IPv6 的地址池绑定到地址池组
  ├── ADD POOLBINDGROUP:POOLGROUPNAME=..., POOLNAME="testpool",   PRIORITY=10
  └── ADD POOLBINDGROUP:POOLGROUPNAME=..., POOLNAME="testpoolv6", PRIORITY=10

步骤7：配置地址池组与 APN 的映射关系
  └── ADD POOLGRPMAP:MAPPINGNAME=..., APN=..., POOLGROUPNAME=...
```

> 说明：文档原操作步骤标注"步骤2 可选：配置APN及地址分配功能"。但任务示例与数据规划表明，实际部署中 APN 与地址池配置是双栈激活的实质性步骤（License 开关为前置，步骤2-7为实质配置）。

> 来源：`激活IPv4v6双栈接入_38276180.md`（"操作步骤"、"任务示例"章节）

### 4.5 约束条件

| 约束类型 | 约束内容 |
|---------|---------|
| License 强制要求 | 必须加载 License 控制项 82209829 LKV3G5VDSA01 并通过 SET LICENSESWITCH 打开开关，否则特性不可用 |

| IPv6 前缀长度 | V6PREFIXLENGTH 范围 49~64；< 64 时切换为 IPv6 Prefix Delegation 分配方式（与 GWFD-020406 协同） |
| VPN 三者一致 | 双栈场景下 IPv4 VPN 链与 IPv6 VPN 链各自需满足"APN 绑定的 VPN / 地址池绑定的 VPN / Gi/SGi/N6 外联口绑定的 VPN"三者一致 |
| 弱覆盖场景影响 | IPv4/v6 双栈改造后，UE 需通过 UDG 下发 RA 获取 IPv6 地址；弱覆盖场景下 eNodeB 无法下发该报文给终端时，**掉话率会增加** |
| 对系统的影响 | 随 IPv4v6 双栈用户数增加，对系统的影响越大（具体影响依赖话务模型） |
| 地址池资源独立 | IPv4 池与 IPv6 池各自规划地址段；IPv4 地址枯竭不影响 IPv6 地址分配，反之亦然 |

## 5. 配置案例

### 5.1 典型场景：APN 下本地地址池分配 IPv4v6 双栈地址

**场景描述**：配置 APN "apn-test" 下的地址分配方式为**本地地址池分配 IPv4v6 双栈地址**。UE 接入后同时获得 IPv4 地址（10.0.x.x 段）与 IPv6 地址（fc00:... 段），可同时进行 IPv4 与 IPv6 通信。

**配置步骤和 MML 命令序列（原样保留文档脚本）**：

```
//打开本特性的License配置开关。

SET LICENSESWITCH:LICITEM="LKV3G5VDSA01",SWITCH=ENABLE;

//配置APN名称。

ADD APN:APN="apn-test", HASVPN=ENABLE, VPNINSTANCE="VRF_Internet", HASVPNIPV6=ENABLE, VPNINSTANCEIPV6="VRF_Internet";

//配置地址池绑定到地址池组。

ADD POOLGROUP: POOLGRPNAME="poolgroup1", IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE;
ADD POOL:POOLNAME="testpool",POOLTYPE=LOCAL,IPVERSION=IPV4, HASVPN=ENABLE,VPNINSTANCE="VRF_Internet";
ADD POOL:POOLNAME="testpoolv6",POOLTYPE=LOCAL,IPVERSION=IPV6, HASVPN=ENABLE,VPNINSTANCE="VRF_Internet";
ADD SECTION:POOLNAME="testpool", SECTIONNUM=1, IPVERSION=IPV4, V4STARTIP="10.0.0.0", V4ENDIP="10.0.127.255";
ADD SECTION:POOLNAME="testpoolv6", SECTIONNUM=1, IPVERSION=IPV6, V6PREFIXSTART="fc00:0000:0000:fcee:0000:0000:0000:0000", V6PREFIXEND="fc00:0000:0000:fcef:ffff:ffff:ffff:ffff", V6PREFIXLENGTH=64;
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool", PRIORITY=10;
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpoolv6", PRIORITY=10;

//配置APN与地址池组的映射关系。

ADD POOLGRPMAP:MAPPINGNAME="mapping1", APN="apn-test", POOLGROUPNAME="poolgroup1";
```

> 来源：`激活IPv4v6双栈接入_38276180.md`（"任务示例-脚本"章节，原样保留）

**脚本要点解读**：

| 配置片段 | 双栈语义 |
|---------|---------|
| `SET LICENSESWITCH:LICITEM="LKV3G5VDSA01"` | 打开 IPv4v6 双栈能力使能开关（前置必配） |
| `ADD APN ... HASVPN=ENABLE, VPNINSTANCE=... HASVPNIPV6=ENABLE, VPNINSTANCEIPV6=...` | APN 同时绑定 IPv4 与 IPv6 两个 VPN（对称双栈） |
| `ADD POOLGROUP ... IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE` | 地址池组同时使能 IPv4 与 IPv6 优先级算法 |
| `ADD POOL ... IPVERSION=IPV4` + `ADD POOL ... IPVERSION=IPV6` | 两个独立的 IPv4 池与 IPv6 池 |
| `ADD SECTION ... IPVERSION=IPV4, V4STARTIP/V4ENDIP` | IPv4 地址段 |
| `ADD SECTION ... IPVERSION=IPV6, V6PREFIXSTART/V6PREFIXEND, V6PREFIXLENGTH=64` | IPv6 地址段，前缀长度 64（非 PD 方式） |
| `ADD POOLBINDGROUP ... POOLNAME="testpool"` + `ADD POOLBINDGROUP ... POOLNAME="testpoolv6"` | 两个池分别绑定到同一地址池组 |
| `ADD POOLGRPMAP ... APN="apn-test"` | APN 与地址池组的映射（双栈统一） |

### 5.2 场景变体

| 变体 | 核心差异 | 关键参数变化 |
|------|---------|------------|
| IPv6 PD 双栈（V6PREFIXLENGTH < 64） | IPv6 地址采用 Prefix Delegation 方式分配，需与 GWFD-020406 协同 | `ADD SECTION ... V6PREFIXLENGTH=60`（或 49~63 中任一）；UDG 行为切换为 PD 模式 |
| 双栈 VPN 隔离（IPv4/IPv6 分属不同 VPN） | IPv4 业务与 IPv6 业务走不同 VPN 实例 | `VPNINSTANCE="VRF_V4"` 与 `VPNINSTANCEIPV6="VRF_V6"` 不同名 |
| 仅 IPv4 单栈（退化场景） | 不使能 HASVPNIPV6，仅配置 IPv4 池；本特性 License 仍需加载但不发挥双栈作用 | `ADD APN ... HASVPNIPV6=DISABLE`；不配置 testpoolv6 |
| 仅 IPv6 单栈（退化场景） | 仅配置 IPv6 池 | `ADD APN ... HASVPN=DISABLE`；不配置 testpool |

## 6. 验证与调测

### 6.1 验证方法

#### 6.1.1 调测前提与目的

支持用户入网时以 IPv4v6 双栈接入。适用于 SGW-U、PGW-U、UPF。

#### 6.1.2 调测数据准备

| 类别 | 参数名称 | 取值样例 | 获取方法 |
|------|---------|---------|---------|
| 用户信息查询 | 用户IMSI（IMSI） | 460000123456789 | 测试终端 |
| 测试APN | APN | apn-test | 已配置数据中获取 |

工具：测试终端、OM Portal

> 该操作无需准备数据。

#### 6.1.3 调测执行步骤

**步骤1**：测试终端使用 "apn-test" APN 发起接入网络请求。

**步骤2**：执行 `DSP SESSIONINFO` 命令，通过测试终端的 IMSI 查看测试终端是否成功激活。

```
DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
```

预期输出（原样保留文档示例）：
```
......
-------------------------------
                                 IMSI  =  460000123456789
                                     ......              
                             PDP Type  =  IPv4v6
                     IPv4 PDP address  =  10.1.1.42     
                     IPv6 PDP address  =  fc00:2000:2000:1600:1000:2300:1000:15
                                     ......                 
                             APN name  =  apn-test         
                                     ......                
 (Number of results = 1)
---    END
```

判断规则：
- 如果测试终端激活成功，且 **"PDP Type"、"IPv4 PDP address" 和 "IPv6 PDP address"** 与规划值一致 → 测试终端接入成功，调测结束
- 如果测试终端激活成功，但 "PDP Type"、"IPv4 PDP address" 和 "IPv6 PDP address" 与规划值不一致 → 执行步骤3

**步骤3**：收集信息并寻求技术支持。
- a. 在 OM Portal 上创建用户跟踪任务并保存报文
- b. 执行 `EXP MML` 命令将当前配置数据导出为 MML 脚本文件并保存
- c. 收集并保存上述所有查询信息
- d. 收集归纳所有信息并联系华为技术支持解决

> 来源：`调测IPv4v6双栈接入_38276241.md`（"操作步骤"章节）

### 6.2 验证要点（双栈特有的三元组判断）

与单栈地址分配不同，双栈接入调测需**同时验证三个字段**：

| 验证字段 | 期望值 | 判读 |
|---------|-------|------|
| PDP Type | **IPv4v6** | 必须为 IPv4v6（非 IPv4 也非 IPv6），证明双栈上下文创建成功 |
| IPv4 PDP address | 在 10.0.0.0 ~ 10.0.127.255 范围 | IPv4 地址在规划的 testpool 段内 |
| IPv6 PDP address | 在 fc00:0000:0000:fcee:: ~ fc00:0000:0000:fcef:ffff:ffff:ffff:ffff 范围 | IPv6 地址在规划的 testpoolv6 段内 |

三者缺一则双栈未真正建立：仅 IPv4 address → 退化为 IPv4 单栈；仅 IPv6 address → 退化为 IPv6 单栈；PDP Type 非 IPv4v6 → 双栈能力未使能（检查 License 与 APN 双栈配置）。

### 6.3 常见问题与排查

| 问题现象 | 可能原因 | 排查方法 |
|---------|---------|---------|
| 用户激活失败，PDP Type 为 IPv4 或 IPv6（非双栈） | License 未加载或 SET LICENSESWITCH 未使能 | LST LICENSESWITCH 确认 LKV3G5VDSA01=ENABLE；检查 License 文件加载状态 |
| PDP Type=IPv4v6 但仅获得 IPv4 地址 | IPv6 池（testpoolv6）或 IPv6 段未配置；IPv6 VPN 未绑定 | LST POOL / LST SECTION 确认 IPv6 池与段；LST APN 确认 HASVPNIPV6=ENABLE |
| PDP Type=IPv4v6 但仅获得 IPv6 地址 | IPv4 池（testpool）或 IPv4 段未配置；IPv4 VPN 未绑定 | LST POOL / LST SECTION 确认 IPv4 池与段；LST APN 确认 HASVPN=ENABLE |
| IPv6 地址不在规划段内 | V6PREFIXSTART/V6PREFIXEND 配置错误；V6PREFIXLENGTH 与规划不符 | LST SECTION 核对 IPv6 段参数；特别检查 V6PREFIXLENGTH |
| 弱覆盖场景掉话率上升 | eNodeB 无法下发 RA 给终端 | UNC 侧 IPv6 前缀代理 5GC 业务流程相关排查（参考 UNC IPv6 PD 特性） |
| 用户激活成功但无法访问 IPv6 业务 | RA 通告异常，UE 未获得 IPv6 路由 | 抓包确认 UDG 是否下发 Router Advertisement；检查 IPv6 下行路由（OSPFv3） |
| 双栈 VPN 隔离不生效 | IPv4 VPN 与 IPv6 VPN 配置为同一实例 | LST APN 核对 VPNINSTANCE 与 VPNINSTANCEIPV6 是否按要求隔离或合一 |
| 性能下降 | 双栈用户数增长 | 评估话务模型；扩容 UDG 资源 |

### 7.2 ★ GWFD-020403 与 GWFD-010105 的关系结论（文档依据）

**结论：GWFD-020403（IPv4v6双栈接入）与 GWFD-010105（用户面地址分配）是"使能层 / 分配机制层"的协同关系，非父子关系，非包含关系。**

#### 7.2.1 关系定位

| 维度 | GWFD-020403（双栈接入） | GWFD-010105（用户面地址分配） |
|------|----------------------|--------------------------|
| 核心职责 | **能力使能**：控制 UDG 是否允许创建 IPv4v6 双栈上下文；控制 RA 通告机制 | **分配机制**：提供 4 种地址分配子方式、三级优先级规则、本地地址池体系 |
| License要求 | **必须**（82209829 LKV3G5VDSA01） | **无**（无需 License） |
| 文档目录 | IPv6功能 | 基本接入功能 |
| PDP Type 参数 | 消费方（依赖 IPv4v6 取值落地为双栈） | 提供方（PDP Type 是其地址类型参数之一） |
| 双栈配置形态 | 定义（双 VPN、双池对称） | 承载（提供 ADD POOL/SECTION 等机制承载双池） |
| Router Advertisement | **定义并描述**（RA 主动下发机制） | **未涉及**（10个文档均未提及 RA） |

#### 7.2.2 文档依据

1. **GWFD-010105 明确将"IPv4v6双栈"列为其支持的 3 种地址类型之一**：
   > "用户面地址分配是指 UDG（PGW-U/UPF）本地为UE分配地址，可提供三种类型的IP地址：**IPv4地址、IPv6地址、IPv4v6双栈地址**。"
   > （来源：`GWFD-010105 用户面地址分配特性概述_68664421.md` §定义）

   `variant_dimensions` 也明确包含 `"IP地址类型(IPv4 / IPv6 / IPv4v6双栈)"`。

2. **GWFD-010105 的地址池机制支持双栈**：
   > "本地地址池支持IPv4和IPv6类型，如果规划UE IPv4v6双栈地址，则需要配置两种类型的地址池。"
   > （来源：`GWFD-010105` ADD SECTION 参数说明）

3. **GWFD-020403 的激活脚本复用 GWFD-010105 的地址池命令体系**：
   SET LICENSESWITCH + ADD APN + ADD POOL/SECTION/POOLGROUP/POOLBINDGROUP/POOLGRPMAP，与 GWFD-010105 的命令完全一致，仅 License 项与双栈对称配置（HASVPNIPV6、双池、IPV4/IPV6ALLOCPRIALG）为新增。

4. **GWFD-020403 独有贡献（010105 未覆盖）**：
   - Router Advertisement 主动下发机制
   - 双栈能力的 License 使能
   - 双栈配置的对称形态定义（APN 双 VPN、POOLGROUP 双优先级算法）

#### 7.2.3 关系图

```
GWFD-010105 用户面地址分配（无License，分配机制层）
   │
   │  提供地址分配基础设施：
   │  - 4种子方式（APN/位置/SMF/RADIUS）
   │  - 三级优先级规则
   │  - 本地地址池 POOL/SECTION/POOLGROUP/POOLGRPMAP
   │  - PFCP 地址类型协商（CHV4/CHV6/V4/V6）
   │
   ▼ 消费其地址类型参数的 IPv4v6 取值
   │
GWFD-020403 IPv4v6双栈接入（License LKV3G5VDSA01，使能层）
   │
   │  在 010105 的基础上叠加：
   │  - License 使能 IPv4v6 双栈能力
   │  - 双栈对称配置形态（双VPN、双池、双优先级算法）
   │  - RA 通告机制（IPv6 地址通告给 UE）
   │  - 创建 IPv4v6 类型的 PDP上下文/EPS承载/PDU Session/QoS Flow
   │
   ▼
   完整的 IPv4v6 双栈用户体验
```

#### 7.2.4 实践配置意义

部署 IPv4v6 双栈时，需**同时激活两个特性**：
1. 先完成 GWFD-010105 的地址分配基础设施配置（APN、地址池、映射规则、地址分配规则、下行路由）
2. 再加载 GWFD-020403 的 License 并使能开关（SET LICENSESWITCH LKV3G5VDSA01=ENABLE）
3. 在 APN 与 POOLGROUP 配置中采用双栈对称形态（HASVPN+HASVPNIPV6、双池、IPV4ALLOCPRIALG+IPV6ALLOCPRIALG）

GWFD-020403 的激活脚本本质上是 GWFD-010105 配置的"双栈对称化"特化版本。
