## 1. 概述

### 1.1 特性定义（是什么）

**基于位置的地址分配**是指 UDG（PGW-U/UPF）根据用户激活时所处的位置 **LAC（Location Area Code）或 TAC（Tracking Area Code）** 为用户动态分配 IP 地址，从而为特定的区域分配特定的 IP 地址，实现 IP 地址的精细化管理。

在 UDG 上配置地址池组，每个地址池组通过绑定位置区组（LAC/TAC）可映射到更小一些的地理区域（如区/县）。当 UE 发起激活请求时，UDG 根据激活请求消息（PFCP Session Establishment Request）中携带的位置信息 LAC/TAC 从对应的地址池中为用户分配特定的 IP 地址。用户 IP 地址与所在位置区域对应，外部 PDN 服务器根据用户 IP 实施相应的控制策略，从而实现 IP 地址精细管理和业务精细管理。

> 来源：`GWFD-020421 基于位置的地址分配特性概述_02846766.md`（"定义"、"原理概述"章节）[EV-FK-01]

## 3. 原理

### 3.1 实现原理：位置区绑定地址池组

在 UDG 上配置地址池组，每个地址池组通过绑定位置区组（LAC/TAC）可映射到更小一些的地理区域（如区/县）。当 UE 发起激活请求时，UDG 根据激活请求消息（**PFCP Session Establishment Request**）中携带的位置信息 LAC/TAC 从对应的地址池中为用户分配特定的 IP 地址。

**位置区划分原则**：
- 一个区域可以是一个或多个 LAC
- 一个区域可以是一个或多个 TAC
- 一个区域可以是 LAC 和 TAC 的组合
- **一个 LAC、TAC 只能属于一个位置区**（唯一归属约束）

**白名单机制**：UDG 上可以基于 IPv4 和 IPv6 地址分别设置基于位置区的地址分配开关，也可以设置白名单，**位于白名单中的用户不参与基于位置区的地址分配**（即白名单用户走普通的非位置敏感的地址分配逻辑）。

> 来源：`GWFD-020421 基于位置的地址分配特性概述_02846766.md`（"原理概述"章节）[EV-FK-01]

### 3.2 地址分配方式：与 APN/DNN、SMF 任意组合

UDG 本地基于位置的地址分配方式，**可以同时与基于 APN/DNN、基于 SMF 的地址方式任意组合**，组合后的地址分配时，所设置的条件间是"**与**"的关系，即同时满足所设置条件的分配要求。这样多样的地址分配方式可以满足运营商多样化的地址管理和网络运维诉求。

| 组合方式 | 规则字符串 | 场景说明 |
|---------|-----------|---------|
| 仅基于位置 | `APN-0&LOCATION-1&SMF-0` | 按位置区规划地址池，最简形式 |
| 位置 + APN/DNN | `APN-1&LOCATION-1&SMF-0` | 同一位置不同 APN 分配不同地址段 |
| 位置 + SMF | `APN-0&LOCATION-1&SMF-1` | 位置 + SMF 双维度规划 |
| 位置 + APN + SMF | `APN-1&LOCATION-1&SMF-1` | 三维度同时匹配（最高粒度） |

> 来源：`GWFD-020421 基于位置的地址分配特性概述_02846766.md`（"原理概述-地址分配方式"章节）[EV-FK-01]
> **规则字符串格式**：`APN-{0|1}&LOCATION-{0|1}&SMF-{0|1}`（1=启用此维度，0=不启用），与母特性 GWFD-010105 完全一致。

### 3.3 地址分配规则（全局 vs APN 两级 + 三级优先级）

UDG 上地址分配可以基于**全局**（`SET IPALLOCRULE`）和 **APN/DNN**（`SET APNIPALLOCRULE`）分别使能：
- 如果设置了基于 APN 的地址分配规则，则使用 APN 下配置的地址分配规则
- 否则就使用基于全局的地址分配规则

UDG 上地址分配规则支持**三级优先级规则**，三级优先级可分别选择 APN/DNN、位置、SMF 进行组合：
- 如果第一优先级规则匹配成功，则使用第一优先级类型的地址池进行地址分配
- 否则依次是第二优先级和第三优先级类型的地址池进行地址分配
- 如果都匹配失败，则地址分配失败

**示例**：第一优先级按位置+APN/DNN（`APN-1&LOCATION-1&SMF-0`）分配，第二优先级回退到仅按位置（`APN-0&LOCATION-1&SMF-0`）分配，第三优先级不使能。若第一和第二优先级规则都无法分配地址，则地址分配失败。

每级规则中所选的条件之间为"**与**"的关系，表示从所选条件映射的地址范围内分配地址。此处设置的规则需要与地址池的映射条件一致（例如按位置区+APN 设置规则，则规划的地址池组也需绑定 LOC&APN）。

> 来源：`GWFD-020421 基于位置的地址分配特性概述_02846766.md`（"原理概述-地址分配方式"章节）；`基于APN_DNN+位置分配地址_02846776.md`（操作步骤7说明）[EV-FK-01, EV-FK-04]

### 3.4 基于位置区分配地址的开关（两级控制）★本特性独有

本特性在三级优先级规则之上，额外增加了一个**基于位置区分配地址的开关**，支持两级控制：

| 开关层级 | 命令 | 作用 |
|---------|------|------|
| 全局开关 | `SET IPALLOCBYLOCGLBSW` | 控制 IPv4 和 IPv6 地址是否使能基于位置区分配（可分别设置） |
| 指定位置区开关 | `SET IPALLOCBYLOCSW` | 针对指定位置区覆盖全局开关（用于个别位置区差异化的启用/禁用） |

**工作机制**：即使规则字符串中 LOCATION=1，若全局开关或对应位置区的开关未使能，基于位置的地址分配逻辑也不会触发。这是本特性区别于母特性其他子方式的关键控制点。

> 来源：`基于位置分配地址_02846773.md`（操作步骤8说明）[EV-FK-03]

## 5. 配置案例

### 5.1 典型场景一：仅基于位置分配地址（单级优先级）

**场景描述**：UE 接入核心网，UDG 基于用户激活位置为 UE 分配 IPv4 地址。第一优先级按照从 LAC 位置区映射的地址范围内分配地址，如果第一优先级的规则匹配不上，则分配地址失败。

**MML命令序列（原样保留）**：

```
//打开本特性的License配置开关。
SET LICENSESWITCH:LICITEM="LKV3G5LBAA01",SWITCH=ENABLE;

//基于APN使能地址分配属性。
ADD VPNINST:VPNINSTANCE="vpn1";
ADD APN:APN="apn-test1",HASVPN=ENABLE,VPNINSTANCE="vpn1";
SET APNADDRESSATTR:APN="apn-test1",SUPPORTIPV4=ENABLE,SUPPORTIPV6=ENABLE;

//配置地址池。
ADD POOL:POOLNAME="testpool1",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="testpool1",SECTIONNUM=0,IPVERSION=IPV4,V6PREFIXSTART="10.10.1.11",V6PREFIXEND="10.10.1.20";

//配置地址池绑定到地址池组。
ADD POOLGROUP: POOLGRPNAME="poolgroup1";
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool1";

//配置LAC位置区组与地址池组的映射关系。
ADD LACGROUP:LACGROUPNAME="lac1";
ADD LACID:LACGROUPNAME="lac1",LACSECNUM=2,LACSTARTID="0x000a",LACENDID="0x000b";
ADD POOLGRPMAP: MAPPINGNAME="mapping1", LOCATIONGRPTYPE=LAC, LOCATIONGRPNAME="lac1", POOLGROUPNAME="poolgroup1";

//配置地址分配规则。
SET IPALLOCRULE: FIRSTRULESW=ENABLE, FIRSTRULE=APN-0&LOCATION-1&SMF-0, SECONDRULESW=DISABLE, THIRDRULESW=DISABLE;

//配置基于位置区分配地址的开关。
SET IPALLOCBYLOCGLBSW: SWITCH=ENABLE;

//配置手机下行路由。
ADD OSPF:PROCID=100,VRFNAME="vpn1",SCHEMAROUID="10.10.10.1";
ADD OSPFAREA:PROCID=100,AREAID="0.0.0.0";
ADD OSPFNETWORK:PROCID=100,AREAID="0.0.0.0", IPADDRESS="10.13.21.0", WILDCARDMASK="0.0.0.255";
ADD OSPFIMPORTROUTE:PROCID=100,TOPOID=0,PROTOCOL=wlr;

//配置基于位置区域地址分配用户白名单，白名单中的用户不基于位置区分配IP地址。
ADD ADRLOCWHITELST: MSISDN="123456";
```

> 来源：`基于位置分配地址_02846773.md`（任务示例）[EV-FK-03]
> **文档原文笔误说明**：`ADD SECTION` 任务示例原文使用 `V6PREFIXSTART="10.10.1.11", V6PREFIXEND="10.10.1.20"` 参数名，但 `IPVERSION=IPV4`，按 GWFD-010105 命名规范应为 `V4STARTIP/V4ENDIP`。此处属文档内部不一致，已原样保留并标注，实际配置应使用 `V4STARTIP/V4ENDIP`。

### 5.2 典型场景二：基于位置 + APN/DNN 组合分配地址（两级优先级回退）

**场景描述**：UE 接入核心网，UDG 采用本地地址池为 UE 动态分配 IPv4 地址。第一优先级按照从位置区+APN/DNN 映射的地址范围内分配地址，第二优先级按照从位置区映射的地址范围分配地址，如果第一优先级和第二优先级的规则都匹配不上，则分配地址失败。演示地址池优先级（IPV4ALLOCPRIALG/PRIORITY）。

**MML命令序列（原样保留，展示与场景一的差异）**：

```
//打开本特性的License配置开关。
SET LICENSESWITCH:LICITEM="LKV3G5LBAA01",SWITCH=ENABLE;

//基于APN使能地址分配属性。
ADD VPNINST:VPNINSTANCE="vpn1";
ADD APN:APN="apn-test1",HASVPN=ENABLE,VPNINSTANCE="vpn1";
SET APNADDRESSATTR:APN="apn-test1",SUPPORTIPV4=ENABLE,SUPPORTIPV6=ENABLE;

//配置地址池绑定到地址池组（双池双组，第二组启用优先级算法）。
ADD POOL:POOLNAME="testpool1",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="testpool1",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.10.1.11",V4ENDIP="10.10.1.20";
ADD POOLGROUP: POOLGRPNAME="poolgroup1";
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool1", PRIORITY=10;
ADD POOL:POOLNAME="testpool2",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="testpool2",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.0.1.11",V4ENDIP="10.0.1.20";
ADD POOLGROUP: POOLGRPNAME="poolgroup2",IPV4ALLOCPRIALG=ENABLE,IPV6ALLOCPRIALG=ENABLE;
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup2", POOLNAME="testpool2";

//配置LAC位置区组。
ADD LACGROUP:LACGROUPNAME="lac1";
ADD LACID:LACGROUPNAME="lac1",LACSECNUM=2,LACSTARTID="0x000a",LACENDID="0x000b";

//配置LAC位置区组、APN/DNN与地址池组的映射关系（第一优先级 mapping1=LOC+APN；第二优先级 mapping2=仅LOC）。
ADD POOLGRPMAP: MAPPINGNAME="mapping1", LOCATIONGRPTYPE=LAC, LOCATIONGRPNAME="lac1", APN="apn-test1", POOLGROUPNAME="poolgroup1";
ADD POOLGRPMAP: MAPPINGNAME="mapping2", LOCATIONGRPTYPE=LAC, LOCATIONGRPNAME="lac1", POOLGROUPNAME="poolgroup2";

//配置地址分配规则（两级使能）。
SET IPALLOCRULE: FIRSTRULESW=ENABLE, FIRSTRULE=APN-1&LOCATION-1&SMF-0, SECONDRULESW=ENABLE, SECONDRULE=APN-0&LOCATION-1&SMF-0, THIRDRULESW=DISABLE;

//配置基于位置区分配地址的开关。
SET IPALLOCBYLOCGLBSW: SWITCH=ENABLE;

//配置手机下行路由。
ADD OSPF:PROCID=100,VRFNAME="vpn1",SCHEMAROUID="10.10.10.1";
ADD OSPFAREA:PROCID=100,AREAID="0.0.0.0";
ADD OSPFNETWORK:PROCID=100,AREAID="0.0.0.0", IPADDRESS="10.13.21.0", WILDCARDMASK="0.0.0.255";
ADD OSPFIMPORTROUTE:PROCID=100,TOPOID=0,PROTOCOL=wlr;

//配置基于位置区域地址分配用户白名单，白名单中的用户不基于位置区分配IP地址。
ADD ADRLOCWHITELST: MSISDN="123456";
```

> 来源：`基于APN_DNN+位置分配地址_02846776.md`（任务示例）[EV-FK-04]

## 6. 验证与调测

### 6.1 验证方法

#### 6.1.1 调测前提与目的

用户激活采用基于位置区域分配 UE IP 地址时，需要调测该功能，确保用户可以正常接入。

> 适用：PGW-U、UPF

#### 6.1.2 调测数据准备

| 类别 | 参数名称 | 取值样例 | 获取方法 |
|------|---------|---------|---------|
| 用户信息查询 | 用户IMSI（IMSI） | 460000123456789 | 测试终端自带 |
| 测试终端使用的APN | APN/DNN实例名（APN name） | apn-test1 | 已配置数据中获取 |

工具：测试终端

#### 6.1.3 调测执行步骤

**步骤1**：使用测试终端**在规划的位置范围内**通过"apn-test1"APN发起接入网络请求。

> ★关键：与母特性 GWFD-010105 调测不同，本特性调测必须在规划的位置范围内发起接入，否则位置匹配会失败。

**步骤2**：执行 `DSP SESSIONINFO` 命令，查看测试终端的 IP 地址是否在 UDG 本地预先按位置规划的地址池内。

```
DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
```

预期输出：
```
......
-------------------------------
                                 IMSI  =  460000123456789
                                     ......              
                             PDP Type  =  IPv4 
                                     ......    
                     IPv6 PDP address  =  10.10.1.15
                                     ......    
                             APN name  =  apn-test1
                                     ......       
(Number of results = 1)
---    END
```

判断：
- 测试终端使用的"IPv4 PDP address"在预先按位置规划的地址池内 → 测试终端接入成功，调测结束
- 测试终端使用的"IPv4 PDP address"不在预先按位置规划的地址池内 → 执行步骤3

> 说明：调测文档原文输出字段名为"IPv6 PDP address"显示 IPv4 地址 `10.10.1.15`，疑为文档笔误，实际应为"IPv4 PDP address"。

**步骤3**：继续查看 `DSP SESSIONINFO`，确认测试终端 IP 地址类型。

```
DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
```

预期输出：
```
......
-------------------------
         APN name  =  apn-test1
                 ......        
IPv4 Address type  =  UPF ALLOC IP ADDRESS 
                 ......      
(Number of results = 1)
---    END
```

判断：
- `UPF ALLOC IP ADDRESS` → 本地地址池分配，执行步骤4进一步核对位置映射
- `EXTERNAL ALLOC IP ADDRESS` → 外部地址池分配，需返回 UDG 配置页面检查本地配置以及检查 SMF 上配置的地址分配能力，是否符合预先规划。如果不符合，请调整配置；如果符合，执行步骤5

**步骤4**：★本特性独有的位置映射核对步骤。查看当前使用的 IP 地址所归属的 UDG 上的地址池组，执行 `LST POOLGRPMAP` 检查该地址池组绑定的位置信息是否与预期一致。

```
LST POOLGRPMAP: MAPPINGNAME="mapping2",POOLGROUPNAME="poolgroup2";
```

预期输出：
```
RETCODE = 0  Operation Success.  
Mapping of Pool Group 
--------------------- 
Name of Pool Group Mapping    Type of Location Group    Name of Location Group    APN         SMF     Name of Pool Group  
mapping1                      LAC                       lac1                      apn-test1   NULL    poolgroup1         (Number of results = 1) 
---    END
```

判断：
- 地址池组绑定的位置信息与预期一致 → 执行步骤5
- 不一致 → 返回 UDG 配置页面重新配置

**步骤5**：收集信息并寻求技术支持。
- a. 在 OM Portal 上创建用户跟踪任务并保存报文
- b. 执行 `EXP MML` 命令将当前配置数据导出为 MML 脚本文件并保存
- c. 收集并保存上述所有查询信息
- d. 收集归纳所有信息并联系华为技术支持解决

> 来源：`调测基于位置的地址分配_02846777.md`[EV-FK-05]

### 6.4 常见问题与排查

| 问题现象 | 可能原因 | 排查方法 |
|---------|---------|---------|
| 用户激活失败（不在规划地址池） | 三级优先级规则全部匹配失败（位置区组未映射或规则字符串错） | LST LACGROUP/LST TACGROUP 检查位置区组；LST POOLGRPMAP 检查 LOCATION 映射；对照 SET IPALLOCRULE 规则字符串中 LOCATION=1 |
| 用户激活失败（License 未生效） | 未加载 License 或 SET LICENSESWITCH 未打开 | LST LICENSE 确认 LKV3G5LBAA01 已加载；确认 SET LICENSESWITCH:LICITEM="LKV3G5LBAA01",SWITCH=ENABLE 已执行 |
| 基于位置分配不生效 | SET IPALLOCBYLOCGLBSW 全局开关未 ENABLE，或指定位置区 SET IPALLOCBYLOCSW 覆盖为 DISABLE | 确认全局开关=ENABLE；检查是否有指定位置区开关覆盖 |
| 用户未按位置分配地址 | 用户 MSISDN 在白名单中 | LST ADRLOCWHITELST 确认用户是否被豁免 |
| 用户激活成功但地址不在规划池内 | 实际为外部地址池分配 | DSP SESSIONINFO 查 IPv4 Address type，若为 EXTERNAL ALLOC IP ADDRESS 需重新配置并检查 SMF 地址分配能力配置 |
| 地址池占用率高告警 | ALM-81215/ALM-81216 | 扩充地址段或新建地址池 |
| 与 L2TP VPN 同时配置失败 | 本特性与 GWFD-020412 互斥 | 取消二者之一的配置（文档明确不支持同时应用） |
| C-U 协商失败 | 对接 C 面非华为设备 | 应用限制：C 面必须为华为配套设备（PFCP 私有扩展） |
| 5G 场景位置匹配失败 | LAC/TAC 类型不匹配（4G 用 LAC，5G 用 TAC/N2TAC） | 确认 LOCATIONGRPTYPE 与实际网络制式一致；5G 场景应配置 TACGROUP+N2TACID |
| 跨位置区移动后地址异常 | 文档未明确移动性重分配机制 | ★ Stage 3 需横向核实：检查是否触发会话重建；当前文档范围内无明确处置流程 |
