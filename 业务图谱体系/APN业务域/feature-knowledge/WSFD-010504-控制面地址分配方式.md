## 1. 概述

### 1.1 特性定义（是什么）

UNC 支持通过 **SMF 分配方式** 为用户分配 IP 地址，可以为用户分配 IPv4 地址、IPv6 地址或双栈地址（IPv4 和 IPv6 地址）。

所谓"控制面地址分配"，指为用户分配的 IP 地址是在 PDU 会话建立阶段**直接从 UNC 的本地地址池**中获取的 IP 地址。UNC 收到用户的激活请求后，如果用户所属 APN 配置的是本地地址分配方式，则 UNC 从用户所属 APN 绑定的本地地址池或 Radius 地址池中为用户分配一个 IP 地址（可以是 IPv4 地址，也可以是 IPv6 地址前缀）。

本特性解决的核心场景是：**当 SMF 对接的 UPF 不支持 IP 地址分配功能时，可使用 SMF 为 UE 分配 IP 地址**（控制面接管地址分配职责，与 UPF 侧 GWFD-010105 用户面地址分配形成"控制面/用户面"两种分配主体二选一的部署多样性）。

> 来源：`output/UNC 20.15.2 产品文档(裸机容器) 05/网络部署/特性部署/UNC特性指南/会话管理功能/WSFD-010504 控制面地址分配方式/特性概述_77654144.md`（"定义" + "原理概述" + "应用场景"章节）

## 2. 激活

> 本特性无独立 License。激活即"配置使能"：通过 MML 命令创建本地地址池体系、设置 APN 地址分配属性为 SMF_ALLOC、配置全局地址分配规则。

### 2.1 激活通用前置条件

- 已阅读 `WSFD-010504 控制面地址分配方式特性概述`
- 已规划本地地址池体系（地址池→地址段→地址池组→UPF组）
- 已通过 ADD APN 配置目标 APN（APNADDRESSATTR 记录将自动生成，IPV4ALLOCTYPE/IPV6ALLOCTYPE 默认 LOCAL）
- 已评估对接 UPF 是否支持地址分配：若不支持 → ALLOCPRECEDENCE 选 SMF_ALLOC；若支持 → 可选 UPF_FIRST

### 2.2 通用激活步骤骨架（SMF 本地分配场景，5 大步）

```
步骤1：创建本地地址池体系（三级 + UPF 组 + 池组映射）
  ├── ADD ADDRPOOL       → 本地地址池（POOLTYPE 与 WSFD-010502 静态场景的 UDM 不同，
  │                        本场景通常为普通 LOCAL 池）
  ├── ADD SECTION         → 地址段（V4STARTIP/V4ENDIP）
  ├── ADD ADDRPOOLGRP     → 地址池组
  ├── ADD POOLBINDGRP     → 地址池↔地址池组绑定
  ├── ADD UPNODE          → UPF 节点（ADDRALLOCMODE 视场景）
  ├── ADD ADDRUPGROUP     → UPF 组
  ├── ADD UPFBINDGRP      → UPF↔UPF 组绑定
  └── ADD POOLGRPMAP      → 地址池组↔UPF 组映射（关键：地址池→UPF 绑定）

步骤2：配置 APN
  └── ADD APN             → 增加 APN（自动生成 APNADDRESSATTR 记录）

步骤3：设置 APN 地址分配属性（SMF 作为分配主体）
  └── SET APNADDRESSATTR  → ALLOCPRECEDENCE=SMF_ALLOC
                              IPV4ALLOCTYPE=LOCAL（默认）
                              IPV6ALLOCTYPE=LOCAL（默认）
                              （可选）IPTYPEFORDUALIP=IPV4V6 / SUPPORTPRIOR / SUPPORTIPV4 / SUPPORTIPV6

步骤4：配置全局地址分配规则（本地池选池三级优先级）
  └── SET IPALLOCRULE     → 第一/二/三级规则组合（APN / LOCATION / UPNODE）
                              默认：第一优先级 APN-1（按 APN 映射地址范围分配）

步骤5：（可选）开启基于位置区的地址分配
  └── SET IPALLOCBYLOCGLBSW → 全局开关（ENABLE 时配合 SET IPALLOCRULE 的 LOCATION 维度）

步骤6：（可选）配置冲突地址
  └── ADD CONFLICTIP      → 冲突地址管理
```

> 来源：`WSFD-010504 控制面地址分配方式参考信息_77813716.md`（"命令"章节）
> 说明：WSFD-010504 仅给出**命令清单**（14 条），未给出独立 MML 激活脚本；以下参数详情取自 OM 参考对应命令文档（`SET APNADDRESSATTR`、`SET IPALLOCRULE`）。

## 3. 原理

### 3.1 SMF 本地地址池分配机制（唯一分配方式）

UNC 收到用户的激活请求后，按以下逻辑为用户分配 IP：

```
用户激活请求（PDU 会话 / PDN 连接 / PDP 上下文建立）
                    │
                    ▼
┌──────────────────────────────────────────────────┐
│ 步骤1：APN 地址分配属性判断                       │
│                                                  │
│   读取 APNADDRESSATTR（ADD APN 自动生成）：       │
│     IPV4ALLOCTYPE = LOCAL （默认）               │
│     IPV6ALLOCTYPE = LOCAL （默认）               │
│     ALLOCPRECEDENCE = SMF_ALLOC（本特性核心）    │
│                                                  │
│   → 判定本 APN 采用"控制面 SMF 本地分配"          │
└──────────────────────────┬───────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────┐
│ 步骤2：本地地址池选择（三级优先级规则）           │
│                                                  │
│   按 SET IPALLOCRULE 配置的优先级匹配地址池：     │
│     第一优先级：APN + LOCATION + UPNODE 组合      │
│     第二优先级：降级规则                          │
│     第三优先级：降级规则                          │
│                                                  │
│   规则取值维度：                                  │
│     - APN：会话的 APN 作为地址池匹配条件          │
│     - LOCATION：UE 接入位置作为匹配条件           │
│     - UPNODE：选择的 UPF 群组标识作为匹配条件     │
└──────────────────────────┬───────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────┐
│ 步骤3：地址分配                                   │
│                                                  │
│   从匹配到的本地地址池（或 Radius 地址池）中：     │
│     - 分配 IPv4 地址，或                          │
│     - 分配 IPv6 地址前缀，或                      │
│     - 分配 IPv4v6 双栈地址（IPTYPEFORDUALIP）     │
│                                                  │
│   地址直接携带在 PDU 会话建立响应消息中下发 UE    │
└──────────────────────────┬───────────────────────┘
                           │
                           ▼
┌──────────────────────────────────────────────────┐
│ 步骤4：地址生命周期增强                          │
│                                                  │
│   ① 延迟释放：                                    │
│      用户去激活后，UNC 可设置 IP 延迟释放，        │
│      待本地地址池中 IP 的等待释放时间到达后，     │
│      UNC 才将该地址重新分配给其他用户              │
│                                                  │
│   ② 租期续约：                                    │
│      如果在租约期内该用户再次创建 PDU 会话，       │
│      可继续使用原 IP 地址（接续中断业务）         │
└──────────────────────────────────────────────────┘
```

> 来源：`特性概述_77654144.md`（"原理概述"）、OM 参考 `设置全局地址分配规则（SET IPALLOCRULE）_49644937.md`

### 3.2 地址分配主体决策（SMF vs UPF）

**关键决策参数**：SET APNADDRESSATTR 的 **ALLOCPRECEDENCE** 参数（OM 参考原文，用于当 IPV4ALLOCTYPE/IPV6ALLOCTYPE=LOCAL 时，指定 UE IP 的分配主体）：

| ALLOCPRECEDENCE 取值 | 含义 | 适用场景 |
|---------------------|------|---------|
| **SMF_ALLOC（SMF 分配）** | UE 地址由 SMF（PGW-C/GGSN）分配 | ★ 本特性核心场景：UPF 不支持地址分配，或希望由控制面统一管理地址 |
| **UPF_FIRST（UPF 优先）** | UE 地址优先由 UPF 分配；UPF 未分配时由 SMF 分配 | UPF 支持地址分配（UDG GWFD-010105 启用）时的优先选择 |
| **INHERIT（继承全局）** | 继承 SET ADDRESSATTR 命令的全局 ALLOCPRECEDENCE 配置 | 多 APN 统一策略 |

> 来源：OM 参考 `设置基于APN的地址分配属性（SET APNADDRESSATTR）_33845575.md`（"参数说明/ALLOCPRECEDENCE"）
> 说明：本参数是本特性与 UDG 侧 GWFD-010105（用户面地址分配）形成 **C/U 分配主体二选一** 的核心开关——ALLOCPRECEDENCE=SMF_ALLOC 即启用 WSFD-010504，=UPF_FIRST 即启用 GWFD-010105。

### 3.3 本地地址池选池规则（三级优先级）

通过 SET IPALLOCRULE 配置 IPv4/IPv6 各自的三级优先级规则，每级规则是 **APN + LOCATION + UPNODE** 三个维度的组合：

| 维度 | 标识 | 匹配语义 |
|------|------|---------|
| **APN** | APN-1 / APN-0 | 1=使用会话的 APN 作为地址池匹配条件；0=不使用 |
| **LOCATION** | LOCATION-1 / LOCATION-0 | 1=使用 UE 的接入位置作为匹配条件；0=不使用 |
| **UPNODE** | UPNODE-1 / UPNODE-0 | 1=使用选择的 UPF 群组标识作为匹配条件；0=不使用 |

**全局默认初始值**（系统部署完成后即存在）：
- IPv4 第一优先级：ENABLE，规则 APN-1（仅按 APN 匹配）
- IPv4 第二/三优先级：DISABLE
- IPv6 同 IPv4

**典型组合示例**（OM 参考使用实例）：
- 第一优先级：APN-1 & LOCATION-0 & UPNODE-1（按 APN + UPF 群组联合匹配）
- 第二优先级：APN-0 & LOCATION-0 & UPNODE-1（仅按 UPF 群组降级匹配）
- 第三优先级：DISABLE

> 来源：OM 参考 `设置全局地址分配规则（SET IPALLOCRULE）_49644937.md`（"参数说明/使用实例"）

### 3.4 地址生命周期增强（延迟释放 + 租期续约）

| 增强能力 | 说明 | 业务价值 |
|---------|------|---------|
| **延迟释放** | 用户去激活后，UNC 可设置为该用户分配的 IP **延迟释放**；待本地地址池中 IP 地址的**等待释放时间**到达后，UNC 才将该地址重新分配给其他用户 | 避免地址抖动；防止短时重连导致地址冲突 |
| **租期续约** | 如果在租约期内该用户再次创建 PDU 会话，可**继续使用原 IP 地址** | 接续中断业务，用户无感知；提升用户体验 |

> 来源：`特性概述_77654144.md`（"原理概述"末段，原文）
> 说明：上述两段原文**完全相同地出现在 WSFD-010502 的 SMF 分配方式章节**（010502 §3.4）——进一步佐证 WSFD-010504 是 WSFD-010502 中 SMF 分配方式的独立成册。

### 4.3 关键参数说明

#### 4.3.1 SET APNADDRESSATTR 关键参数（本特性决策核心）

| 参数 | 取值 | 说明 |
|------|------|------|
| APN | 字符串（如 demo.com） | APN 名称（必选），需先通过 ADD APN 配置 |
| IPV4ALLOCTYPE | **LOCAL**（默认） / RADIUS / DHCP | IPv4 分配方式；本特性取 LOCAL（5GC SMF/UPF 分配）。**LOCAL 含义：通过 5GC（SMF 或 UPF）分配地址** |
| IPV6ALLOCTYPE | **LOCAL**（默认） / RADIUS / DHCP | IPv6 分配方式；本特性取 LOCAL |
| **ALLOCPRECEDENCE** | **SMF_ALLOC** / UPF_FIRST / INHERIT | ★ 当 IPV4/IPV6ALLOCTYPE=LOCAL 时，指定分配主体。**SMF_ALLOC=启用本特性**（SMF 分配）；UPF_FIRST=启用 UDG 侧 GWFD-010105（UPF 优先） |
| SUPPORTIPV4 | ENABLE / DISABLE | 是否支持 IPv4；本特性 ENABLE |
| SUPPORTIPV6 | ENABLE / DISABLE | 是否支持 IPv6；本特性 ENABLE |
| SUPPORTPRIOR | IPV4 / IPV6 | 用户未指定双栈时优先返回的地址类型 |
| IPTYPEFORDUALIP | IPV4 / IPV6 / **IPV4V6** | 双栈用户返回的地址类型；本特性常取 IPV4V6 |
| IPV4RADIUSPRIOR | DISABLE / ENABLE | LOCAL/DHCP 下是否优先使用 AAA 签约的静态 IPv4 |
| IPV6RADIUSPRIOR | DISABLE / ENABLE | LOCAL/DHCP 下是否优先使用 AAA 签约的静态 IPv6 |
| CONFLICTSTG | DEACOLD / DEACNEW / DEACBOTH / INHERIT | 地址冲突时去激活策略（INHERIT 继承全局） |
| STATICIP | ENABLE / DISABLE | 是否使用签约静态地址接入 |

> 来源：OM 参考 `设置基于APN的地址分配属性（SET APNADDRESSATTR）_33845575.md`（"参数说明"全部条目）
> 关键澄清：**用户关注的 IPV4ALLOCTYPE/IPV6ALLOCTYPE 是 APN 级别地址分配方式开关**（LOCAL/RADIUS/DHCP 三选一），**不是 SMF vs UPF 的开关**；真正决定 SMF 分配还是 UPF 分配的是 **ALLOCPRECEDENCE** 参数。

### 4.4 约束条件

| 约束类型 | 约束内容 |
|---------|---------|
| APN 必须先配置 | SET APNADDRESSATTR 执行前必须先 ADD APN（否则无 APNADDRESSATTR 记录可修改） |
| IPV4ALLOCTYPE/IPV6ALLOCTYPE=LOCAL 才适用本特性 | 仅当取 LOCAL 时，ALLOCPRECEDENCE 才决定 SMF vs UPF 分配；取 RADIUS/DHCP 时由外部服务器分配 |
| SUPPORTIPV4/SUPPORTIPV6 联动 | SUPPORTPRIOR=IPV4 时 SUPPORTIPV4 必须 ENABLE；SUPPORTPRIOR=IPV6 时 SUPPORTIPV6 必须 ENABLE；IPTYPEFORDUALIP=IPV4V6 时两者均需 ENABLE |
| 仅修改 SUPPORTIPV4=DISABLE 会自动修改 IPTYPEFORDUALIP=IPV6 | 配置联动约束，需同时设置 SUPPORTPRIOR 和 IPTYPEFORDUALIP |
| IPALLOCRULE 不同优先级规则不可相同 | 三级优先级规则需互异 |
| IPALLOCRULE 配置低优先级时不要求必须配置高优先级 | 三级独立配置 |
| 全局地址池组使用率告警 | 地址池组使用率超阈值触发 ALM-100297（详见 §6.2） |
| 虚拟 APN 场景 | 若 APN 是虚拟 APN 且 SUPPORTIPV4/SUPPORTIPV6/IPTYPEFORDUALIP 取非默认值，对 5G 用户生效需将软参 DWORD1044 BIT24 设为 1 |
| 鉴权模式与 RADIUS 分配互斥 | 若 APNADDRESSATTR 中 ACCESSMODE 配置为 LOC_AUTH 或 TRANS_NON_AUTH，配置地址分配模式为 RADIUS 会失效；SMF/PGW-C/GGSN 会根据 ALLOCPRECEDENCE 判断地址由 UPF 还是 SMF 分配 |

> 来源：OM 参考 `SET APNADDRESSATTR_33845575.md`（"注意事项" + "配置原则"）、`SET IPALLOCRULE_49644937.md`（"说明"）

## 5. 配置案例

### 5.1 典型场景：SMF 本地分配（UPF 不支持地址分配）

**场景描述**：某运营商部署的 UPF 不支持 IP 地址分配功能，需由 SMF 为 5G PDU 会话用户分配 IPv4 地址。APN 名称为 "demo.com"，地址池范围 10.0.0.0~10.0.255.255。

**MML命令序列（原样保留）**：

```
// 步骤1：创建本地地址池体系
ADD ADDRPOOL: POOLNAME="smfpool01", IPVERSION=IPv4, POOLTYPE=LOCAL, CHECKIPVALID=Enable;

ADD SECTION: POOLNAME="smfpool01", SECTIONNUM=1, IPVERSION=IPv4, V4STARTIP="10.0.0.0", V4ENDIP="10.0.255.255";

ADD ADDRPOOLGRP: POOLGRPNAME="smfpoolgrp1", POOLGRPTYPE=LOCAL;

ADD POOLBINDGRP: POOLGRPNAME="smfpoolgrp1", POOLNAME="smfpool01";

// 步骤2：配置 UPF 节点与 UPF 组（即使 UPF 不分配地址，仍需 UPNODE 用于会话建立）
ADD UPNODE: NFINSTANCENAME="UPF_Instance_1", ADDRALLOCMODE=INHERIT;

ADD ADDRUPGROUP: UPFGRPNAME="smfupfgrp1", UPFGRPTYPE=LOCAL;

ADD UPFBINDGRP: UPFGRPNAME="smfupfgrp1", UPFID="UPF_Instance_1";

// 步骤3：地址池组与 UPF 组映射
ADD POOLGRPMAP: MAPPINGNAME="smfmapping01", POOLGRPNAME="smfpoolgrp1", UPFGRPNAME="smfupfgrp1";

// 步骤4：配置 APN（自动生成 APNADDRESSATTR 记录，IPV4ALLOCTYPE/IPV6ALLOCTYPE 默认 LOCAL）
ADD APN: APN="demo.com";

// 步骤5：设置 APN 地址分配属性为 SMF 分配（启用本特性的核心步骤）
SET APNADDRESSATTR: APN="demo.com", IPV4ALLOCTYPE=LOCAL, IPV6ALLOCTYPE=LOCAL, ALLOCPRECEDENCE=SMF_ALLOC, IPTYPEFORDUALIP=IPV4V6, SUPPORTPRIOR=IPV4;

// 步骤6：设置全局地址分配规则（第一优先级按 APN+UPNODE 匹配，第二优先级仅按 UPNODE）
SET IPALLOCRULE: IPV4FIRSTRULESW=ENABLE, IPV4FIRSTRULE=APN-1&LOCATION-0&UPNODE-1, IPV4SECONDRULESW=ENABLE, IPV4SECONDRULE=APN-0&LOCATION-0&UPNODE-1, IPV4THIRDRULESW=DISABLE, IPV6THIRDRULESW=DISABLE;
```

> 来源：MML 命令参数取值综合自：
> - `WSFD-010504 控制面地址分配方式参考信息_77813716.md`（命令清单）
> - OM 参考 `SET APNADDRESSATTR_33845575.md`（使用实例原样：`SET APNADDRESSATTR: APN="demo.com", IPV4ALLOCTYPE=LOCAL, IPV4RADIUSPRIOR=DISABLE, IPV6ALLOCTYPE=DHCP, IPV6RADIUSPRIOR=ENABLE, ALLOCWITHOUTPCO=ENABLE, IPV6RAINTV=1801, IPV6RAOFLAG=ENABLE, IPV6RATIMES=5;` ——本场景改 IPV6ALLOCTYPE=LOCAL、增加 ALLOCPRECEDENCE=SMF_ALLOC）
> - OM 参考 `SET IPALLOCRULE_49644937.md`（使用实例原样：`SET IPALLOCRULE: IPV4FIRSTRULESW=ENABLE, IPV4FIRSTRULE=APN-1&LOCATION-0&UPNODE-1, IPV4SECONDRULESW=ENABLE, IPV4SECONDRULE=APN-0&LOCATION-0&UPNODE-1, IPV4THIRDRULESW=DISABLE, IPV6THIRDRULESW=DISABLE;`）
> 说明：WSFD-010504 产品文档**未提供独立的完整 MML 激活脚本**（仅有命令清单），上述脚本基于命令清单 + OM 参考使用实例组合构建，POOLTYPE=LOCAL / POOLGRPTYPE=LOCAL 为本地分配场景推断取值，实际部署时需按 UNC MML 命令规范核对（Stage 3 补充）。

### 6.1 验证方法

#### 6.1.1 验证思路

#### 6.1.3 调测执行步骤（参考 WSFD-010502 §6.1.3）

**步骤1**：测试终端使用目标 APN 发起接入网络请求，创建 PDU 会话。

**步骤2**：进入 "MML命令行-UNC" 窗口。执行 **DSP PDUSESSION** 命令，通过测试终端的 IMSI 查看测试终端是否成功激活。
- 如果测试终端激活成功，且使用的"用户IP地址"与规划值一致 → SMF 本地分配生效，调测结束
- 如果不一致 → 执行步骤3

**步骤3**：收集信息并寻求技术支持。
- a. 在 OM Portal 创建用户跟踪任务，并保存报文
- b. 执行 **LST APNADDRESSATTR** 核对 ALLOCPRECEDENCE 配置
- c. 执行 **LST IPALLOCRULE** 核对选池规则
- d. 执行 **EXP MML** 命令查看当前配置并保存查询结果
- e. 收集归纳所有信息并联系华为技术支持

| SMF 本地分配未生效，地址仍由 UPF 分配 | ALLOCPRECEDENCE 未设置为 SMF_ALLOC（仍为默认 INHERIT 或 UPF_FIRST） | LST APNADDRESSATTR 核对 ALLOCPRECEDENCE；SET APNADDRESSATTR 修改为 SMF_ALLOC |
| 用户激活失败 | IPV4ALLOCTYPE/IPV6ALLOCTYPE 不是 LOCAL（取了 RADIUS/DHCP）但本意是 SMF 本地分配 | LST APNADDRESSATTR 核对；改为 LOCAL |
| 用户激活失败 | SUPPORTPRIOR/IPTYPEFORDUALIP 与 SUPPORTIPV4/SUPPORTIPV6 联动冲突 | 核对联动约束（§4.4）；调整配置使其一致 |
| 地址池选错 | SET IPALLOCRULE 三级规则配置不当 | LST IPALLOCRULE 核对；按业务需求调整 APN/LOCATION/UPNODE 组合 |
| 地址池资源耗尽 | 本地地址池地址段不足 | 监控 ALM-100297 告警；查看测量指标 1929465383（占用率）；扩充 ADD SECTION |
| 地址冲突 | 两个用户分配到相同 IP（罕见） | 查看测量指标 1929465381/5481（无效地址数）；ADD CONFLICTIP 配置冲突地址 |
| UPF 不支持地址分配但配置了 UPF_FIRST | ALLOCPRECEDENCE=UPF_FIRST 时 UPF 未分配，会回退 SMF 分配（功能正常但路径长） | 改为 SMF_ALLOC 直接由 SMF 分配，减少 C-U 交互 |
| 虚拟 APN 场景双栈参数对 5G 用户不生效 | 软参 DWORD1044 BIT24 未设置为 1 | SET SOFTPARA 配置 DWORD1044 BIT24=1 |
| 鉴权模式下 RADIUS 分配失效 | ACCESSMODE=LOC_AUTH 或 TRANS_NON_AUTH 时，配置 RADIUS 分配模式失效 | 核对鉴权模式；SMF/PGW-C/GGSN 会按 ALLOCPRECEDENCE 判断 SMF vs UPF 分配 |

## 7. 参考信息

### 7.1 与 WSFD-010502 的关系结论（★重点）

**结论：WSFD-010504 与 WSFD-010502 是"并列子集"关系，不是父子关系。**

#### 7.1.1 关系定性

| 维度 | 结论 | 文档依据 |
|------|------|---------|
| **功能范围** | WSFD-010504 ⊂ WSFD-010502 | WSFD-010502 定义 4 种分配方式（UDM静态/Radius/SMF本地池/DHCP）；WSFD-010504 **仅** 定义 SMF 分配方式 1 种 |
| **原理原文重合** | SMF 分配方式原理**完全相同** | WSFD-010504 "原理概述" 与 WSFD-010502 §3.4 "SMF 本地分配方式" 两段文字**逐字一致**（含"延迟释放"和"租期续约"描述） |
| **发布时序** | 010502 早，010504 晚 | 010502 v01=20.0.0；010504 v01=**20.8.0**（晚 8 个版本） |
| **配置树定位** | 010504 非父节点 | 配置树原推断 "010504=010502 父节点" **被产品文档证伪**：010504 范围更窄（仅 SMF 分配），不可能包含 010502 的 4 种方式 |
| **独立成册动机** | 突出 SMF 作为分配主体 | 010504 核心场景"UPF 不支持地址分配时由 SMF 分配"在 010502 中未单独强调；010504 独立成册以突出 **ALLOCPRECEDENCE=SMF_ALLOC** 这一决策开关和本地池资源运维观测点（ALM-100297 + 14 条测量指标） |

#### 7.1.2 关系图

```
WSFD-010502 地址分配方式（v01=20.0.0，地址分配总集，4 种方式）
├── ① UDM 静态分配        （HLR/HSS/UDM 签约静态 IP）
├── ② Radius 分配          （Access-Accept 下发）
├── ③ SMF 本地池分配  ◄══════ WSFD-010504 控制面地址分配方式（v01=20.8.0）
│   └── 即本特性的全部范围      独立成册：聚焦 SMF 作为分配主体
│       └── 共享原理：本地池 + 延迟释放 + 租期续约
└── ④ DHCP 分配            （UNC 作 DHCP Client 代理）
```

#### 7.1.3 不是父子关系的证据

1. **范围 narrower**：父节点范围应 ⊇ 子节点。010504（1 种）⊂ 010502（4 种），故 010504 不可能是 010502 的父节点。
2. **发布时序**：父节点通常先发布。010502（20.0.0）早于 010504（20.8.0），故 010502 更可能是父节点。
3. **原理重合**：010504 的原理段落是 010502 §3.4 的逐字复制，说明 010504 是从 010502 中**抽出 SMF 分配方式**独立成册，而非上位概念。
4. **独立观测面**：010504 新增了 010502 没有的 ALM-100297 告警和 14 条测量指标（专门监控 SMF 本地池资源），这是独立成册的运维价值。

#### 7.1.4 实际关系：同域并列子集特性

- **010502** = 地址分配方式的**总集**（4 种方式概述 + 静态段→UPF 绑定 + 静态地址黑名单两个独立激活脚本）
- **010504** = 地址分配方式中 **SMF 本地池分配方式**的**独立深化**（突出 SMF 作为分配主体 + 本地池资源运维）

两者在配置树上应为**同域并列**（同属"地址分配"类别），而非父子层级。Stage 3 横向分析时应将 parent_feature_id 修正为：**010504 与 010502 同级，均为"地址分配"类别下的叶子特性**，或视 010502 为宽谱特性、010504 为窄谱深化特性（010504 ⊂ 010502 但非严格父子）。
