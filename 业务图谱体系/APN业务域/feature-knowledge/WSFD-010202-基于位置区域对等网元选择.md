## 1. 概述

### 1.1 特性定义（是什么）

UNC 在 **Attach/RAU/TAU** 等流程中查询源侧网元节点 SGSN，在 **Handover/Relocation** 等流程中查询目标侧网元节点 SGSN/MME，以及在 **SRVCC** 流程中查询目标 MSC 的过程均为**对等网元选择**。基于 **LAI/RAI/TAI** 定制 DNS 域名进行对等网元查询，称为**基于位置区域的对等网元选择**。

> **对等网元（华为定义概念）**：指与本网元处在相同网络层级的设备。对 MME 和 SGSN 来说，同样处在控制域、进行移动性管理的 SGSN、MME 以及 MSC 网元都是对等网元。

本特性的核心价值是**减少 DNS 配置记录数量**：通过将一定范围的 LAI/RAI/TAI 映射到起始位置区域或区域名称（ZONE），用聚合后的单一域名替代大量逐区域域名记录，降低运维成本并减少对 DNS Server 的查询次数。

> 来源：EV-FK-01（特性概述 §定义、§应用场景）

| 涉及NF | 网元角色 | 支持版本 | 功能说明 |

## 2. 原理

### 2.1 域名定制方法（两种映射方式）

对等网元选择的 DNS 域名定制包含如下两种方法：

**方法一：映射到起始 LAI/RAI/TAI**

将一定范围的 LAI/RAI/TAI 映射到该范围的起始 LAI/RAI/TAI。
> 示例：将 RAI_1 ~ RAI_100 组成一个区域集合，并将该集合统一映射到 RAI_1。当位于 RAI_1 ~ RAI_100 的用户终端触发对等网元选择时，UNC 将统一使用 RAI_1 构造域名进行 DNS 解析。

**方法二：映射到区域名称（ZONE）**

将一定范围的 LAI/RAI/TAI 映射到某区域名称（ZONE）。运营商可根据自身 LAI/RAI/TAI 所覆盖的物理区域合理规划符合运维习惯的区域名称。
> 示例：将 RAI_1 ~ RAI_100 组成一个区域集合，并将该集合映射到某区域名称（ZONE）。当位于 RAI_1 ~ RAI_100 的用户终端触发对等网元选择时，UNC 将统一使用该区域名称（ZONE）构造域名进行 DNS 解析。

**实现命令**：
- 域名定制通过 **ADD AREADNS** 实现
- 定制 SGSN 域名时，可通过 **ADD SGSNDNS** 匹配与现网 DNS 域名后缀格式的差异

> 来源：EV-FK-02（实现原理 §域名定制方法）

| MME | TAI | TAC-LB01.TAC-HB49.TAC.EPC.MNC003.MCC123.3GPPNETWORK.ORG<br>TAC-LB02.TAC-HB49.TAC.EPC.MNC003.MCC123.3GPPNETWORK.ORG<br>TAC-LB03.TAC-HB49.TAC.EPC.MNC003.MCC123.3GPPNETWORK.ORG | TAC-LB01.TAC-HB49.TAC.EPC.MNC003.MCC123.3GPPNETWORK.ORG | ZONE.EPC.MNC003.MCC123.3GPPNETWORK.ORG |

**结论**：定制前每个区域码（LAC/RAC/TAC）需要一条独立的 DNS 记录，记录数随区域数量线性增长；定制后，整个区域集合只需 1 条记录（映射到起始码或 ZONE 名称），记录数大幅减少。

> 来源：EV-FK-02（实现原理 §域名定制方法，表1 对等网元选择域名定制）

### 2.3 域名定制流程（按触发场景与目标网元）

文档以 3 个典型场景为例介绍对等网元选择过程中的域名定制流程：

| # | 场景 | 触发流程 | 查询目标 | 位置区域类型 | 备注 |
|---|------|---------|---------|------------|------|
| 1 | LTE 网络至 UMTS 网络的切换 | Handover | 目标网络 SGSN | LAI/RAI | UNC 用聚合后的域名做 DNS 解析 |
| 2 | UMTS 网络至 LTE 网络的切换 | Handover | 目标网络 MME | TAI | UNC 用聚合后的域名做 DNS 解析 |
| 3 | SRVCC | SRVCC | 目标 MSC | LAI/RAI | 可通过 **SET MSCSELPLCY** 选择基于 RAI FQDN 或 LAI FQDN 去解析 MSC |

> 来源：EV-FK-02（实现原理 §域名定制流程，图1 选择对等网元的域名定制流程）

### 2.4 位置区（LAI/RAI/TAI）到网元的映射机制

本特性建立的映射关系链为：

```
用户位置（LAI/RAI/TAI）
    ↓ ADD AREADNS 配置（区域集合 → 起始码 或 ZONE 名称）
聚合后的 DNS 域名（如 ZONE.MNC123.MCC123.GPRS）
    ↓ DNS 解析（A 记录 / NAPTR 记录，本地 Hostfile 或 DNS Server）
对等网元（SGSN / MME / MSC）的 IP/接口
```

关键点：
1. **区域聚合**是本特性的核心机制——把多个细粒度位置区域码聚合为一个粗粒度的查询键
2. **两种聚合目标**：起始码（保留位置语义）或 ZONE 名称（脱离具体位置、纯运维命名）
3. **DNS 解析的两类记录**：A 记录（IPV4DNSH，域名→IPv4）和 NAPTR 记录（DNSN，FQDN→网元接口，如 S10/Sv）
4. **SRVCC 场景的额外策略**：SET MSCSELPLCY 控制 MSC 选择基于 RAI 还是 LAI

> 来源：EV-FK-02（实现原理 §域名定制方法、§域名定制流程）；EV-FK-03（激活文档 §操作步骤）

## 4. 配置案例

### 4.1 任务1：基于 RAI 区域集合（映射到 ZONE）选择 SGSN

**场景描述**：选择 SGSN 过程中，配置基于 RAI 区域集合（映射到区域名称 ZONE）选择 SGSN 的 DNS 域名定制方式。对应的位置区域为 "0x0001"，路由区集合为 "0x01" ~ "0x05"。

**MML 脚本（文档原文）**：

```
// 步骤1：增加 Hostfile 的 A 解析记录（可选，DNS Server 已配则跳过）
ADD IPV4DNSH: HSINDEX=1, HOSTNAME="ZONE.MNC123.MCC123.GPRS", ADDRSECTION=SECTION1, IPV4ADDR1="10.2.3.10";

// 步骤2：增加基于位置区域的 DNS 域名定制策略
ADD AREADNS: DNTYPE=SGSN_RAI, LAC="0x0001", RAC="0x01", RACRANGE="0x05", ZONESW=YES, ZONENAME="ZONE";
```

> 来源：EV-FK-03（激活文档 §任务示例 任务1）

### 4.2 任务2：基于 TAI 区域集合（映射到起始 TAI）选择 MME

**场景描述**：选择 MME 过程中，配置基于 TAI 区域集合（映射到起始 TAI）选择 MME 的 DNS 域名定制方式。对应的跟踪区集合为 "0x0001" ~ "0x0005"。

**MML 脚本（文档原文）**：

```
// 步骤1：增加 Hostfile 的 A 解析记录
ADD IPV4DNSH: HSINDEX=2, HOSTNAME="TOPON.MME-S10.MMEC02.MMEGI8001.MME.EPC.MNC000.MCC123.3GPPNETWORK.ORG", ADDRSECTION=SECTION1, IPV4ADDR1="10.2.3.11";

// 步骤2：增加 Hostfile 的 NAPTR 解析记录
ADD DNSN: FQDN="TAC-LB01.TAC-HB00.TAC.EPC.MNC123.MCC123.3GPPNETWORK.ORG", HSINDEX=2, ENTITY=MME, INTYPE=S10;

// 步骤3：增加基于位置区域的 DNS 域名定制策略
ADD AREADNS: DNTYPE=MME_TAI, TAC="0x0001", TACRANGE="0x0005";
```

> 注：本任务未配置 ZONESW=YES，即默认映射到起始 TAI（TAC="0x0001"）。

> 来源：EV-FK-03（激活文档 §任务示例 任务2）

### 4.3 任务3：基于 LAI 区域集合（映射到 ZONE）选择 MSC（SRVCC）

**场景描述**：选择 MSC 过程中，配置基于 LAI 区域集合（映射到区域名称 ZONE）选择 MSC 的 DNS 域名定制方式。对应的位置区集合为 "0x0001" ~ "0x0005"。

**MML 脚本（文档原文）**：

```
// 步骤1：增加 Hostfile 的 A 解析记录
ADD IPV4DNSH: HSINDEX=3, HOSTNAME="NODES.EPC.ENVID68.MNC003.MCC123.3GPPNETWORK.ORG", ADDRSECTION=SECTION1, IPV4ADDR1="10.2.3.12";

// 步骤2：增加 Hostfile 的 NAPTR 解析记录
ADD DNSN: FQDN="ZONE.EPC.MNC003.MCC123.3GPPNETWORK.ORG", HSINDEX=3, ENTITY=MSC, INTYPE=Sv;

// 步骤3：增加基于位置区域的 DNS 域名定制策略
ADD AREADNS: DNTYPE=MSC_LAI, LAC="0x0001", LACRANGE="0x0005", ZONESW=YES, ZONENAME="ZONE";
```

> 来源：EV-FK-03（激活文档 §任务示例 任务3）

### 7.1 与配置树/文档清单的差异

| # | 维度 | 配置树/文档清单描述 | 产品文档实际内容 | 差异类型 |
|---|------|---------------------|-----------------|---------|
| 1 | 文档数量 | 文档清单列 **5 个文件** | 5 个文件齐全（概述/原理/激活/调测/参考信息） | 一致：文档覆盖完整 |
| 2 | 文档覆盖度 | 标注"[辅助]" | 文档完整：含概述、原理、激活脚本（3 个任务）、调测方法（OM Portal 跟踪）、参考信息（12 条命令 CRUD） | **覆盖度优于"辅助"标注**：本特性虽标"辅助"，但文档齐全度（5 文件、完整 MML 脚本、调测图示）接近"★核心"标准 |
| 3 | 适用 NF | 文档清单标题"UNC/SMF，网元选择类" | 特性概述 §适用NF 明确为 "SGSN、MME"（非 SMF）；MSC 为被查询目标 | **差异**：文档清单写"SMF"，实际文档为 SGSN/MME（SMF 是 5G 实体，本特性是 2G/3G/4G 特性，不涉及 SMF） |
| 4 | License 要求 | 文档清单未标注 | 产品文档明确 **无需 License**（与 GWFD-020421、WSFD-107010 均不同） | 补全：License 信息明确 |
| 5 | 命令清单完整性 | - | 参考信息列出 **12 条 MML 命令**（AREADNS/IPV4DNSH/DNSN 各 4 条 CRUD）+ 原理文档提及 ADD SGSNDNS、SET MSCSELPLCY | 补全：命令清单较完整，但 ADD SGSNDNS/SET MSCSELPLCY 未进入参考信息正式清单 |
| 6 | 配置脚本完整性 | - | 激活文档提供 **3 个完整任务示例的 MML 脚本**（SGSN/MME/MSC 各一），覆盖两种映射方式（起始码 + ZONE） | 一致：脚本完整，质量高 |
| 7 | 代际覆盖 | - | 覆盖 2G/3G（SGSN/MSC，LAI/RAI）与 4G（MME，TAI），**不涉及 5G** | 一致：本特性是 Pre-5G 特性 |

### 7.2 与 UPF 选择（WSFD-107010）的关系（★ 重点）

| # | 维度 | WSFD-010202（本特性） | WSFD-107010（UPF 选择） | 关系说明 |
|---|------|----------------------|------------------------|---------|
| 1 | 代际 | 2G/3G/4G（Pre-5G） | 5G | 代际互补，不重叠 |
| 2 | 选择目标 | 对等控制面网元（SGSN/MME/MSC） | 用户面网元（UPF） | 一个选 C 面对等网元，一个选 U 面网元 |
| 3 | 选择机制 | DNS 域名定制 + DNS 解析（位置区域聚合） | 多维属性筛选（DNN/切片/DNAI/位置/接口/EPS互通）+ 权重/负载 | 本特性机制简单（仅 DNS），UPF 选择机制复杂（三轮筛选） |
| 4 | 位置区域角色 | 作为 DNS 域名的聚合键 | 作为 UPF 属性的筛选条件（TAI 范围匹配） | 位置区域在两者中都参与，但作用方式不同 |
| 5 | License | 否 | 是（82209917/82200BES） | 本特性免费，UPF 选择需 License |
| 6 | 配置复杂度 | 低（主要 ADD AREADNS + Hostfile） | 高（20+ 条命令，NF 实例/业务能力/位置区/接口/策略 5 大类） | 本特性轻量，UPF 选择重量级 |
| 7 | 同厂商约束 | 无 | SMF 与 UPF 必须同为 HUAWEI | 本特性无此约束（DNS 是开放协议） |

### 7.3 与基于位置的地址分配（GWFD-020421）的关系（★ 重点）

| # | 维度 | WSFD-010202（本特性） | GWFD-020421（基于位置的地址分配） | 关系说明 |
|---|------|----------------------|--------------------------------|---------|
| 1 | 解决问题 | 选对等控制面网元 | 为用户分配 IP 地址 | 完全不同的目标 |
| 2 | 作用网元 | UNC（SGSN/MME） | UDG（PGW-U/UPF）为主 | 一在控制面，一在用户面 |
| 3 | 位置区域作用 | DNS 域名聚合键（LAI/RAI/TAI → 起始码/ZONE） | 地址池绑定键（LAC/TAC → 地址池组） | 同样用位置区域，但映射目标不同（域名 vs 地址池） |
| 4 | License | 否 | 是（82200BAK LKV3G5LBAA01） | 本特性免费 |
| 5 | 位置区划分原则 | 由 ADD AREADNS 的起始码+范围定义集合 | 一个区域可含一个或多个 LAC/TAC，或组合；一个 LAC/TAC 只能属于一个位置区 | 划分粒度规则在 GWFD-020421 文档更详细 |
| 6 | 应用限制 | 无 | 仅支持 UDG 本地地址池；对接控制面须为华为配套版本 | GWFD-020421 限制更多 |
| 7 | 互斥关系 | 无 | 与 GWFD-020412 L2TP VPN 互斥 | 本特性无互斥声明 |
