# WSFD-011202 支持热计费 知识文档

## 概述

### 定义

热计费（Hot Billing）是一种**特殊的离线计费方式**，具有普通离线计费的所有功能，但 CG 会优先处理带有热计费标志的话单，话单产生速度比普通离线计费话单更快。

### 与普通离线计费的区别

热计费与普通离线计费的唯一区别在于话单中的计费属性字段 `chargingCharacteristics` 取值不同：

| 计费类型 | chargingCharacteristics 字段值 | 十六进制值 |
|----------|-------------------------------|-----------|
| 热计费 | hotbilling | 0x0100 |
| 普通计费 | normal | 0x0800 |
| 预付费 | prepaid | - |
| 统一费率 | flat | - |

### 适用网元（NF）

| 适用范围 | 涉及网元 | 版本要求 |
|----------|---------|---------|
| GGSN/SGW-C/PGW-C 场景 | GGSN-C、SGW-C、PGW-C | UNC 20.3.0 及后续版本 |
| | GGSN-U、SGW-U、PGW-U | UDG 20.3.0 及后续版本 |
| | CG | 无特殊要求 |
| | BS（Billing System） | 无特殊要求 |
| SGSN 场景 | SGSN | UNC 20.3.0 及后续版本 |
| | CG | 无特殊要求 |
| | BS | 无特殊要求 |

### 客户价值

| 受益方 | 受益描述 |
|--------|---------|
| 运营商 | 热计费的话单产生速度更快，可以降低欠费风险 |
| 用户 | 用户不感知该特性 |

### 遵循标准

3GPP TS 32.015：Telecommunications management; Charging management; 3G call and event data for the Packet Switched (PS) domain。

## 核心概念

### 热计费的定位

1. **热计费属于离线计费的增强特性**，不是独立的计费模式。
2. 开启热计费必须先开启离线计费功能（依赖 WSFD-011201 支持离线计费）。
3. 热计费话单的产生、传送和缓存机制与普通离线计费完全相同。
4. 区别仅在于 CG 优先处理带热计费标志的话单，使话单处理速度更快。

### 计费属性（Charging Characteristics）

计费属性是区分热计费与普通计费的关键字段：

- **GGSN/SGW-C/PGW-C**：计费属性在控制面配置，也可以从其他网元（SGSN/SGW/HLR）获得。
- **SGSN**：SGSN 支持四种计费属性——普通计费、热计费、预付费、包月制（统一费率）。
- 热计费用户的 `chargingCharacteristics` 字段在话单中体现为热计费标志（0x0100）。

### 话单中的计费属性显示

在最终生成的话单中，`chargingCharacteristics` 字段显示内容：

| 计费类型 | 话单中显示 |
|----------|-----------|
| 热计费 | Hot billing |
| 普通计费 | Normal billing |
| 预付费 | Prepaid billing |
| 统一费率 | Flat billing |

### 对系统的影响

- **GGSN/SGW-C/PGW-C**：热计费对话单产生速度有一定要求，对系统性能有影响，需要在全网规划中合理设置话单产生阈值。
- **SGSN**：对系统无影响。

## 原理与流程

### GGSN/SGW-C/PGW-C 热计费流程

```
步骤1~4：用户携带热计费属性激活，成功建立承载。
步骤5~6：GGSN-C/SGW-C/PGW-C 与 CG 交互链路状态检测消息，保证链路正常。
步骤7~8：用户访问数据业务时，GGSN-U/SGW-U/PGW-U 记录取计费信息（数据流量、时长、费率变更等），
         并将相关信息传递给 GGSN-C/SGW-C/PGW-C。
步骤9~11：GGSN-C/SGW-C/PGW-C 根据配置的话单触发条件，在话单中记录热计费信息以及
          数据流量、时长、费率变更，打上热计费标志。达到话单关闭条件后产生原始话单，
          通过 GTP' 协议封装后传给 CG。CG 通过 FTP 协议传给 BS。
```

关键要点：
- 用户面（GGSN-U/SGW-U/PGW-U）负责收集用户计费信息（时长、流量等）。
- 控制面（GGSN-C/SGW-C/PGW-C）负责判断计费属性、产生话单、打热计费标志。
- 话单通过 GTP' 协议封装传给 CG，CG 再通过 FTP 协议传给 BS。
- BS 可对 CG 下发专有指令，优先处理带有热计费属性的话单。

### SGSN 热计费流程

```
步骤1：SGSN 与 CG 进行状态交互，确认与 SGSN 互通的 CG 可用。
步骤2：热计费用户业务接入，触发热计费，产生话单。
步骤3：SGSN 判断用户为热计费用户，在话单中添加热计费标志。
       - 与 CG 互通时：通过 GTP' 协议封装后发送给 CG。
       - 与 CG 断链时：SGSN 将已生成的话单缓存到本地。
步骤4：CG 优先处理有热计费标记的话单，对话单进行存储、合并、标准化等预处理，
       然后将计费数据发送到 BS（Billing System）。
步骤5：BS 结算用户的 GPRS 费用，如果用户余额不足，则记录下用户所欠的金额，
       并向 HLR 申请终止 GPRS 业务，禁止用户使用 GPRS 服务。
```

关键差异（与 GGSN/SGW-C/PGW-C 对比）：
- SGSN 自身完成计费信息收集和话单生成（无用户面/控制面分离）。
- SGSN 支持断链缓存机制：与 CG 断链时，话单缓存到本地。
- BS 侧可主动触发欠费停机：余额不足时向 HLR 申请终止 GPRS 业务。
- SGSN 可以在 30 秒内产生话单。

### 话单产生速度

- 热计费话单产生速度比普通离线计费话单更快。
- SGSN 热计费可以在 **30 秒内** 产生话单。
- GGSN/SGW-C/PGW-C 通过配置更小的阈值参数来加快话单产生速度。

## 配置规则

### 前置条件

| 适用范围 | 前置条件 |
|----------|---------|
| GGSN/SGW-C/PGW-C | 已配置离线计费功能（WSFD-011201） |
| SGSN | 已配置离线计费功能（WSFD-011201），已加载 License |

### License 要求

| 适用范围 | License 要求 |
|----------|-------------|
| GGSN/SGW-C/PGW-C | **无需 License** |
| SGSN | 需要 License：`82206574 LKV2HBILL02 支持热计费功能` |

### GGSN/SGW-C/PGW-C 配置步骤

热计费的配置与离线计费配置类似，核心差异点：
1. 必须开启离线计费开关。
2. 热计费话单参数阈值应设置得比普通离线计费更小，以加快话单产生速度。

#### 配置流程

```
1. 使能离线计费开关
   SET CHARGECTRL: HOMEOFFLINE=ENABLE, VISITOFFLINE=ENABLE;

2. 配置热计费属性
   ADD CHARGECHAR: CCNAME="xxx", HOME="0x100", ROAM="0x100", VISIT="0x100",
                   HOMESGSN=DISABLE, ROAMSGSN=DISABLE, VISITSGSN=DISABLE;

3. 配置离线计费模板（含话单阈值）
   ADD OFCTEMPLATE: OFCTEMPLATENAME="xxx", GCDRVERSION=R7V740_EGCDR,
                     TQM=QCT, QCTVALUE=30;
   SET OFCTHRESHOLD: OFCTEMPLATENAME="xxx", THRESHINHERIT=NON_INHERIT,
                      CONDITIONCHANGE=3, VOLUMETHRESHOLD=1000,
                      TIMETHRESHOLD=30, SERVINGNODECHNG=3;

4. 绑定到 APN
   ADD APN: APN="apn-test";
   SET APNCHARGECTRL: APN="apn-test", ONLINESW=DISABLE, OFFLINESW=ENABLE,
                       PGWOFCTEMPLATE="xxx", SGWOFCTEMPLATE="xxx",
                       GGSNOFCTEMPLATE="xxx", CCNAME="xxx";
```

#### 关键配置参数说明

| 参数 | 说明 | 热计费配置要点 |
|------|------|---------------|
| HOME/ROAM/VISIT (ADD CHARGECHAR) | 本地/漫游/拜访用户计费属性 | 设为 `0x100`（热计费） |
| HOMESGSN/ROAMSGSN/VISITSGSN | 使用 Serving Node 计费属性 | 设为 `DISABLE`（使用本地配置） |
| HOMEOFFLINE/VISITOFFLINE (SET CHARGECTRL) | 离线计费开关 | 设为 `ENABLE` |
| CONDITIONCHANGE (SET OFCTHRESHOLD) | 计费条件改变阈值 | 建议值 3 |
| VOLUMETHRESHOLD (SET OFCTHRESHOLD) | 流量阈值 | 建议值 1000（比普通离线计费更小） |
| TIMETHRESHOLD (SET OFCTHRESHOLD) | 时长阈值 | 建议值 30（比普通离线计费更小） |
| SERVINGNODECHNG (SET OFCTHRESHOLD) | Serving Node 改变次数阈值 | 建议值 3 |
| GCDRVERSION (ADD OFCTEMPLATE) | G-CDR 版本 | 如 `R7V740_EGCDR` |

### SGSN 配置步骤

#### 配置流程

```
1. 打开 License 配置开关
   SET LICENSESWITCH: LICITEM="LKV2HBILL02", SWITCH=ENABLE;

2. 设置计费属性为热计费
   SET CHGCHAR: CC=HOTBILLING, SP=YES, SPP=PERMIT, SPL=2;
```

#### 关键配置参数说明

| 参数 | 说明 | 热计费配置要点 |
|------|------|---------------|
| LICITEM (SET LICENSESWITCH) | License 项 | `LKV2HBILL02` |
| SWITCH (SET LICENSESWITCH) | 开关 | `ENABLE` |
| CC (SET CHGCHAR) | 计费属性 | `HOTBILLING`（实时计费） |
| SP (SET CHGCHAR) | 生成 S-CDR | `YES` |
| SPP (SET CHGCHAR) | 周期生成 S-CDR | `PERMIT` |
| SPL (SET CHGCHAR) | S-CDR 生成周期（分钟） | 建议值 2（加快话单产生） |

## 配置案例

### 案例1：GGSN/SGW-C/PGW-C 激活热计费

场景：为 APN "apn-test" 配置热计费功能。

```mml
// 1. 使能离线计费开关和热计费属性
SET CHARGECTRL:HOMEOFFLINE=ENABLE,VISITOFFLINE=ENABLE;

ADD CHARGECHAR: CCNAME="testchgcha", HOME="0x100", ROAM="0x100", VISIT="0x100",
                HOMESGSN=DISABLE, ROAMSGSN=DISABLE, VISITSGSN=DISABLE;

// 2. 配置计费参数（阈值较小以加快话单产生）
ADD OFCTEMPLATE:OFCTEMPLATENAME="offlinecharg-test",GCDRVERSION=R7V740_EGCDR,
                 TQM=QCT,QCTVALUE=30;

SET OFCTHRESHOLD:OFCTEMPLATENAME="offlinecharg-test",THRESHINHERIT=NON_INHERIT,
                  CONDITIONCHANGE=3,VOLUMETHRESHOLD=1000,TIMETHRESHOLD=30,
                  SERVINGNODECHNG=3;

// 3. 绑定到 APN
ADD APN:APN="apn-test";

SET APNCHARGECTRL: APN="apn-test", ONLINESW=DISABLE, OFFLINESW=ENABLE,
                    PGWOFCTEMPLATE="offlinecharg-test",
                    SGWOFCTEMPLATE="offlinecharg-test",
                    GGSNOFCTEMPLATE="offlinecharg-test",
                    CCNAME="testchgcha";
```

### 案例2：SGSN 激活热计费

场景：设置实时计费用户的 S-CDR 生成周期为 2 分钟。

```mml
// 1. 打开 License 配置开关
SET LICENSESWITCH: LICITEM="LKV2HBILL02", SWITCH=ENABLE;

// 2. 计费属性设为"实时计费"，生成 S-CDR 设为"YES"，周期生成 S-CDR 设为"PERMIT"，
//    S-CDR 生成周期设为 2 分钟
SET CHGCHAR: CC=HOTBILLING, SP=YES, SPP=PERMIT, SPL=2;
```

## 验证与调测

### GGSN/SGW-C/PGW-C 调测步骤

#### 步骤 1：激活用户测试

- 测试终端使用配置了热计费的 APN 接入网络，访问业务。
- 如果无法接入，先调测接入功能。

#### 步骤 2：产生话单

- 用户访问业务，产生话单。
- 一段时间超过设置的时间阈值或流量阈值后，去激活用户。

#### 步骤 3：查询话单计费属性

- 登录 CG 客户端，查询此用户刚生成的话单。
- 检查 `chargingCharacteristics` 字段：
  - 如果显示 **Hot billing**：热计费功能正常，调测完成。
  - 如果不是 Hot billing：进入步骤 4 排查。

#### 步骤 4：检查 APN 计费属性绑定

```mml
LST APNCHARGECTRL:APN="apn-test";
```

检查输出中"计费属性名称"是否绑定了正确的热计费属性名。
- 如果不正确：重新配置该 APN 的计费属性实例。

注意：如果配置了 AAA 鉴权下发 Charge-characteristic，需检查 AAA 鉴权响应消息是否下发 `Charge-characteristic:Hotbilling` 字段。如果存在其他网元可以下发 Charge-characteristic，需检查其他网元是否下发该字段。

#### 步骤 5：检查计费属性值

```mml
LST CHARGECHAR:CCNAME="testchgcha";
```

期望输出：
```
计费属性名称 = testchgcha
本地用户计费属性 = 0x0100
漫游用户计费属性 = 0x0100
拜访用户计费属性 = 0x0100
本地用户使用Serving Node计费属性 = 禁止
漫游用户使用Serving Node计费属性 = 禁止
拜访用户使用Serving Node计费属性 = 禁止
```

- 如果计费属性值不是 `0x0100`（hotbilling）：重新配置计费属性实例。

#### 步骤 6：检查后付费用户的计费选择

```mml
LST CHARGEMETHOD
```

- 如果用户为后付费用户，而"Offline"配置为"DISABLE"：改为"ENABLE"并重新调测。

#### 步骤 7：检查计费开关

```mml
LST CHARGECTRL
```

- 如果用户所属类型被配置为"NONE"或"ONLINE"：重新配置为"OFFLINE"并重新调测。
- 如果配置为"OFFLINE"或"BOTH"：进入步骤 8。

#### 步骤 8：收集信息寻求技术支持

1. 执行 `EXP MML` 命令导出当前配置。
2. 收集并保存上述所有查询信息。
3. 联系技术支持解决。

### SGSN 调测步骤

- 使用 OM Portal 跟踪进行调测。
- 前提是已完成激活热计费功能的配置。

## GGSN/SGW-C/PGW-C 与 SGSN 差异对比

| 对比维度 | GGSN/SGW-C/PGW-C | SGSN |
|----------|-------------------|------|
| 架构 | 控制面/用户面分离：U 面收集计费信息，C 面产生话单 | 单网元完成所有功能 |
| License | 无需 License | 需要 LKV2HBILL02 License |
| 配置命令 | ADD CHARGECHAR（0x100）、SET CHARGECTRL、SET OFCTHRESHOLD、ADD OFCTEMPLATE、SET APNCHARGECTRL | SET LICENSESWITCH、SET CHGCHAR（CC=HOTBILLING） |
| 话单类型 | G-CDR（可配置版本如 R7V740_EGCDR） | S-CDR |
| 阈值配置 | 通过 SET OFCTHRESHOLD 配置流量阈值、时长阈值、计费条件改变阈值、SGSN 改变次数阈值 | 通过 SET CHGCHAR 的 SPL 参数配置 S-CDR 生成周期 |
| 断链处理 | 与普通离线计费机制相同 | SGSN 与 CG 断链时缓存话单到本地 |
| 欠费处理 | 与普通离线计费机制相同 | BS 可向 HLR 申请终止 GPRS 业务 |
| 话单产生速度 | 通过配置更小的阈值参数实现 | 30 秒内产生话单，S-CDR 周期建议 2 分钟 |
| Serving Node 属性 | 支持 HOMESGSN/ROAMSGSN/VISITSGSN 参数 | 无此参数 |
