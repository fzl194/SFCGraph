# WSFD-011206 支持融合计费 知识文档

## 概述

### 特性定义

5G网络中引入融合计费（Converged Charging），SMF与CHF（Charging Function）间通过统一的Nchf接口（也称N40接口）和融合计费服务进行实时计费和后计费，实现2/3/4G网络中在线计费（Gy/Diameter）和离线计费（Ga/GTP'）的功能。

- **适用NF**：SMF（主控面），UPF（收集上报计费信息），CHF（配额管理、话单生成），PCF（下发计费规则）
- **支持版本**：UNC 20.0.0及后续版本
- **License**：无需额外License许可；内容计费需内容计费基本功能License支持

### 融合计费 vs 在线计费 vs 离线计费

| 计费机制 | 功能 | 计费接口 | 协议 | 对端NF |
|----------|------|----------|------|--------|
| Gy/Diameter在线计费 | 实时在线监控，余额用尽可停止业务 | Gy | Diameter协议 | OCS |
| 离线计费 | 指定时间更新余额，不实时影响业务 | Ga | GTP'协议 | CG |
| 融合计费 | 同时支持在线与离线计费功能 | Nchf(N40) | HTTP/2协议 | CHF |

### 融合计费分类

1. **按是否申请配额**：
   - 在线计费：SMF向CHF申请配额，配额不足时可停止业务
   - 离线计费：SMF不申请配额，仅上报资源使用情况

2. **按计费费率粒度**：
   - 会话计费（PDU Session Charging）：一个PDU会话所有业务统一费率
   - 内容计费（FBC, Flow Based Charging）：不同业务使用不同费率（需License）
   - QoS flow计费（QBC）：不同QoS flow使用不同费率

3. **按网络资源统计方式**：流量计费、时长计费、事件计费

### 应用限制

- 不支持多UPF场景共享在线计费RG配额
- 同一计费会话内，同一RG的在线、离线属性应唯一
- 话单由CHF生成，SMF向CHF上报计费信息

### 依赖特性

| 依赖特性 | 控制项 | 说明 |
|----------|--------|------|
| WSFD-109002 内容计费基本功能 | LKV3W9BCC12 | 基于业务粒度融合计费时需开启 |
| WSFD-109004 基于业务流量的计费 | LKV3WPVBCS11 | 基于业务流量融合计费时需开启 |
| WSFD-109003 基于业务时长的计费 | LKV3W9TBCS12 | 基于业务时长融合计费时需开启 |

### 遵循标准

3GPP TS 23.501, 23.502, 23.503, 29.512, 32.240, 32.255, 32.290, 32.291, 32.298, 29.244

## 核心概念

### Nchf(N40)接口

SMF与CHF之间的服务化接口，基于HTTP/2协议，实现融合计费。接口API版本通过 `SET N40APIVER` 命令配置（如F30版本），需与CHF使用相同协议版本。

### 融合计费模板（CCT, Converged Charging Template）

CCT包含本地配置的融合计费参数（时间/流量门限、Trigger、故障处理等）。

- 默认存在全局模板 `global`（CCT-global）
- 支持按UserProfile、DNN、CC粒度配置普通CCT模板
- 优先级：UserProfile > DNN > CC > 整机通用参数（global）
- 命令：`ADD CCT`（新增）、`MOD CCT`（修改global）、`ADD SELECTCCTBYCC`（绑定CC）、`SET APNCHARGECTRL`（绑定DNN）、`SET USRPROFCHARGE`（绑定UserProfile）

### 费率标识（Rating Identifier）

- **费率组（Rating Group, RG）**：SMF与CHF交互的费率粒度
- **费率组+业务ID（RG, SID）**：更细粒度的配额管理
- 费率标识通过URR（使用量上报规则）配置，`ONLCOMPOUNDTYPE=ONLYRG` 使用RG，`ONLCOMPOUNDTYPE=RGSID` 使用(RG,SID)

### 计费属性（Charging Characteristics, CC）

3GPP协议定义的用户签约付费属性：
- 热计费 0x0100、统一费率 0x0200、预付费 0x0400、普通计费 0x0800、自定义
- CC来源优先级：RADIUS Server下发 > UserProfile配置 > DNN配置 > 全局配置 > normal(0x0800)
- UDM下发的CC通过 `ADD CHARGECHAR` 的HOMESGSN/ROAMSGSN/VISITSGSN控制是否使用

### 计费Trigger

SMF与CHF建立计费会话后进行后续交互的条件。分为Session级和RG级。

- **立即上报**：Trigger发生时立即上报CHF
- **延迟上报**：记录数据但不立即发送，在下一条Charging Data Request中携带
- Session级和RG级同时生效的Trigger冲突时，以Session级优先
- CHF下发的Trigger优先级高于SMF本地配置（`ADD PDUTRIGGER`/`ADD RGTRIGGER`）
- `SET CNVRGDCHGPARA` 的 `IGNORETRIGGER` 参数可忽略CHF下发的指定Trigger

## 原理与流程

### 计费会话创建流程

PDU会话建立过程中建立计费会话，分两种场景：

**UPF分配用户IP地址**：
1. UE发起PDU会话建立请求
2. SMF向PCF获取PCC规则和计费策略（或使用本地配置）
3. SMF向UPF发送PFCP Session Establishment Request（含UE IP Address信元指示地址分配方式）
4. UPF分配IP地址并返回PFCP Session Establishment Response
5. SMF选择CHF（按优先级，见CHF选择机制）
6. SMF发送Nchf_ConvergedCharging_ChargingDataCreate Request到CHF（携带用户标识、PDU会话信息、RG配额申请等）
7. CHF检查余额、执行预留、开启CDR话单
8. CHF返回ChargingDataCreate Response（携带授权配额、Trigger）
9. SMF向UPF发送PFCP Session Modification Request（含Create URR）
10. PDU会话建立成功

**SMF分配用户IP地址**：SMF直接携带用户IP地址向CHF发送Create Request，CHF返回配额后SMF再向UPF发送PFCP Session Establishment Request。

**在线 vs 离线计费信元差异**：
- 离线计费Create Request不含 `requestedUnit` 等配额管理信元
- 离线计费Create Response不含 `grantedUnit`、`timeQuotaThreshold`、`volumeQuotaThreshold` 等配额信元
- 离线计费PFCP Create URR不含 Volume Quota、Time Quota、Event Quota

### 计费会话更新流程

**内部事件UPF触发**：配额耗尽/剩余配额达阈值/配额空闲时间门限达到/访问无配额新业务
- UPF发送PFCP Session Report Request（含Usage Report）
- SMF向CHF发送ChargingDataUpdate Request（携带用量和配额申请）
- CHF扣费处理并返回Update Response（携带新配额和Trigger）
- SMF通过PFCP Session Modification下发配额到UPF

> 达到阈值或配额耗尽时，UPF启动定时器（T3RESPONSE * N3REQUEST + 4秒），超时未收到配额更新消息则阻塞业务。

**内部事件SMF触发**：在线配额有效时长超期（CCT模板VT参数配置）
- SMF向UPF发起计费即时查询（PFCP Modification携带Query URR）
- SMF向CHF更新计费会话
- `SET CNVRGDCHGPARA` 的 `MERGERGVTSW` 参数控制是否合并RG级VT事件

**外部事件触发**：接入条件更新（Serving Node变更、PRA变更、QoS改变、用户位置更新、终端时区改变、PLMN变更等）
- SMF向UPF查询用量
- SMF向CHF上报并申请重授权

**CHF发起重授权**：CHF发送Nchf_ConvergedCharging_ChargingDataNotify Request（notificationType=REAUTHORIZATION）
- SMF向UPF查询用量，向CHF更新计费会话

### 计费会话释放流程

**UE发起PDU会话释放**：
1. SMF向UPF发送PFCP Session Deletion Request
2. UPF返回Deletion Response（携带计费统计信息）
3. SMF向CHF发送ChargingDataRelease Request（triggerType=FINAL，sessionStopIndicator=true）
4. CHF扣费并关闭CHF-CDR
5. CHF返回Release Response

> 异常去活时triggerType的控制：`ADD N40DIAGTRIGGER` 的 RELEASETRIGGER 参数优先级高于 `SET CNVRGDCHGPARA` 的 SPTABNTRIGGER 参数。

**CHF发起用户去活**：
- CHF发送Notify Request（notificationType=ABORT_CHARGING）
- SMF释放UPF资源，向CHF发送Release Request

### 计费事件触发条件

| 计费事件 | 级别 | 缺省类别 | CHF能否变更 | 上报消息 |
|----------|------|----------|------------|----------|
| Start of PDU Session | Session | 立即上报 | 不适用 | ChargingDataRequest[Initial] |
| Start of service data flow, no charging session | RG | 立即上报 | 否 | ChargingDataRequest[Initial] |
| QoS change | Session/RG | 延迟上报 | 是 | ChargingDataRequest[Update] |
| User Location change | Session/RG | 延迟上报 | 是 | ChargingDataRequest[Update] |
| Serving Node change | Session/RG | 延迟上报 | 是 | ChargingDataRequest[Update] |
| UE time zone change | Session/RG | 立即上报 | 是 | ChargingDataRequest[Update] |
| PLMN change | Session/RG | 立即上报 | 是 | ChargingDataRequest[Update] |
| RAT type change | Session/RG | 立即上报 | 是 | ChargingDataRequest[Update] |
| Addition/Removal of UPF | Session/RG | 立即上报 | 是 | ChargingDataRequest[Update] |
| Tariff time change | Session/RG | 延迟上报 | 否 | ChargingDataRequest[Update] |
| PDU会话时间/流量/事件门限到达 | Session | 立即上报 | 否(仅能去使能) | ChargingDataRequest[Update] |
| RG时间/流量门限到达 | RG | 延迟上报 | 是 | ChargingDataRequest[Update] |
| 配额阈值/耗尽/有效时间超时/空闲超时 | RG | 立即上报 | 否(仅能去使能) | ChargingDataRequest[Update] |
| CHF重授权 | RG | 立即上报 | 否 | ChargingDataRequest[Update] |
| End of PDU session | Session | 立即上报 | 否 | ChargingDataRequest[Termination] |
| CHF session termination | Session | 立即上报 | 否 | ChargingDataRequest[Termination] |
| Abort request from CHF | Session | 立即上报 | 否 | ChargingDataRequest[Termination] |

### MEC计费流程

MEC场景下通过UL CL（上行分流器）分流到不同锚点UPF（PSA1、PSA2）。

- UL CL UPF没有计费规则，不计费
- SMF按UPF向CHF申请配额（Multiple Unit Usage字段携带UPF ID），每个UPF独享配额
- 主/辅锚点均可独立申请配额、收集使用量

### 互操作计费流程

基于N26接口的5GS与EPS互操作：
- PGW-C与SMF必须共部署，采用一致的融合计费机制和Nchf接口
- 切换时通过RAT更新的Trigger感知切换事件，PGW-C/SMF关闭所有统计，发送ChargingDataRequest[Update]到CHF，CHF生成CHF-CDR

## 配置规则

### 1. 使能融合计费功能

融合计费使能有三级开关粒度（优先级从高到低）：

**总开关**（`SET CHARGECTRL`）：控制本地/漫游/拜访用户是否使能融合计费
```
SET CHARGECTRL: HOMECONVERGED=ENABLE, VISITCONVERGED=ENABLE, ROAMCONVERGED=ENABLE;
```

**UserProfile级**（`SET USRPROFCHARGE`）：
```
ADD USERPROFILE: USERPROFILENAME="up-test", UPTYPE=PCC;
SET USRPROFCHARGE: USRPROFNAME="up-test", CONVERGEDSW=ENABLE, RGAPPLIED=ONLINERGONLY;
```

**DNN级**（`SET APNCHARGECTRL`）：
```
ADD APN: APN="apn-test";
SET APNCHARGECTRL: APN="apn-test", CONVERGEDSW=ENABLE, RGAPPLIED=ONLINERGONLY;
```

**CC级**（`ADD CHARGEMETHOD`）：
```
ADD CHARGEMETHOD: CHARGECHARACT=SPECIFIC_VALUE, CCVALUE="0x400", MASK="0x400", PRIORITY=2, CONVERGED=ENABLE, RGAPPLIED=ONLINERGONLY;
```

> `RGAPPLIED` 参数：DEFAULT时优先选择在线URR；ONLINERGONLY/OFFLINERGONLY时建议同时配置在线和离线URR。

### 2. 配置计费和策略模式

计费接口选择优先级：漫游属性（`ADD ROAMCHGMODE`）> APN/DNN（`ADD APNCHGMODE`）> 全局（`SET CHGMODE`）

**全局配置Nchf模式**：
```
SET CHGMODE: TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=False, FORCED=NchfMode;
```

**基于APN/DNN配置**：
```
ADD APNCHGMODE: APN="apn-test", TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=False, FORCED=NchfMode;
```

**基于漫游属性配置**：
```
ADD ROAMCHGMODE: ROAMINGTYPE="ROAMING", TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=False, FORCED=NchfMode;
```

**策略接口配置**：
```
SET POLICYMODE: TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=DISABLE, FORCED=Npcf;
```

**可选：基于PCF实例调整计费接口**：
```
ADD PCCCHGMODEBYPCFID: PCFINSID="pcf1", POLICYINTF=N7, CHGINTF=N40;
```

### 3. CHF选择方式

SMF选择CHF的优先级（从高到低）：

1. **基于IMSI选择**（`ADD SELCHFGBYIMSI`）：拨测场景
2. **基于IMSI号段选择**（`ADD SELCHFGBIMSISEG`）：多CHF主备组负载均衡
3. **基于PCF下发的chargingInfo选择**：FQDN或IP地址
4. **基于UDM签约CC选择**：DNN/APN级CC优先于Session级CC
5. **基于NRF发现**：不支持主备方式，仅支持优先级和权重
6. **基于本地配置CC选择**（`ADD SELECTCHFGBYCC`）
7. **基于全局默认CHF组选择**（`SET GLBDFTCHFGROUP`）

**CHF主备配置**：
```
ADD TNFINS: TNFINSINDEX=1, TNFTYPE=CHF, TNFINSNAME="target_chf_0", SCHEMA=http;
ADD TNFINSIP: TNFINSINDEX=1, IPTYPE=IPV4, IPV4ADDR="192.168.94.2", PORT=3220;
ADD TNFINS: TNFINSINDEX=2, TNFTYPE=CHF, TNFINSNAME="target_chf_1", SCHEMA=http;
ADD TNFINSIP: TNFINSINDEX=2, IPTYPE=IPV4, IPV4ADDR="192.168.104.2", PORT=3230;
ADD TNFGRP: TNFGRPINDEX=1, TNFTYPE=CHF, TNFGRPNAME="ChfGroup1";
ADD TNFGRP: TNFGRPINDEX=2, TNFTYPE=CHF, TNFGRPNAME="ChfGroup2";
ADD TNFBINDGRP: TNFGRPINDEX=1, TNFINSINDEX=1;
ADD TNFBINDGRP: TNFGRPINDEX=2, TNFINSINDEX=2;
ADD SELECTCHFGBYCC: CCTYPE=VALUE, CCVALUE="0x1111", MASK="0xffff", PRIORITY=2, PRIMARYCHFGRP="ChfGroup1", SECONDARYCHFGRP="ChfGroup2";
```

### 4. 配置融合计费模板（CCT）

**修改全局CCT模板**：
```
MOD CCT: CCTMPLTNAME="global", QHT=5;
```

**新增CCT模板**：
```
ADD CCT: CCTMPLTNAME="cct_test", QHT=0, VQT=20, TQT=20, UQT=0, VT=30, MAXNUMOFCCC=3, PDUVOLUMELIMIT=500, PDUTIMELIMIT=30, RGVOLUMELIMIT=500, RGTIMELIMIT=30, FUATERMINATE=TERM_SERVICE, TIMEFORMAT=LocalTime, MAXSVCCONTAINER=100, SECRUTHRESHOLD=100;
```

**CCT模板关键参数**：
| 参数 | 说明 |
|------|------|
| QHT | 配额空闲时间门限值(秒)，数据包停止传送后计时，达到门限向CHF上报 |
| VQT | 流量阈值触发百分比(%) |
| TQT | 时间阈值触发百分比(%) |
| VT | 在线配额有效时长(分)，超时后SMF立即向CHF上报并申请新配额 |
| MAXNUMOFCCC | 计费条件改变几次后触发Update消息 |
| PDUVOLUMELIMIT | PDU级流量阈值(MB) |
| PDUTIMELIMIT | PDU级时长阈值(分钟) |
| RGVOLUMELIMIT | RG级流量阈值(MB) |
| RGTIMELIMIT | RG级时长阈值(分钟) |
| MAXSVCCONTAINER | 最大携带业务容器数量（0=不限） |
| FUATERMINATE | 最终配额动作指示终结方式 |
| TIMEFORMAT | 时间格式（LocalTime/UTC） |

**绑定CCT到CC/DNN/UserProfile**：
```
ADD SELECTCCTBYCC: CCTYPE=VALUE, CCVALUE="0x0800", CCTMPLTNAME="cct_test";
SET APNCHARGECTRL: APN="apn-test", CCTEMPLATE="cct_test";
SET USRPROFCHARGE: USRPROFNAME="up-test", CCTEMPLATE="cct_test";
```

### 5. 配置SMF与CHF交互的条件和内容

**配置PDU会话建立时是否与CHF交互**（`SET CHFINIT`）：
```
SET CHFINIT: CCTMPLTNAME="cct_test", CHFINIT=SENDREQ, CCRINITRGNUM=5;
```
- `CHFINIT=SENDREQ`：建立PDU会话时发送ChargingDataRequest与CHF交互
- `CHFINIT=NOTSENDREQ`：不发送
- `CCRINITRGNUM`：初始请求RG个数

**配置预申请配额的RG**：
```
ADD URR: URRNAME="urr_test", URRID=1111, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=111, ONLMETERINGTYPE=VOLUME;
ADD URRGROUP: URRGROUPNAME="urrgroup_test", UPURRNAME1="urr_test", DOWNURRNAME1="urr_test";
ADD USERPROFILE: USERPROFILENAME="up_test", UPTYPE=PCC;
SET CTXSTARTRATING: USERPROFILENAME="up_test", CTXRURRGRPNAME1="urrgroup_test";
```

**配置Session级Trigger**：
```
ADD PDUTRIGGER: CCTMPLTNAME="cct_test", QOSCHG=IMMEDIATE, SRVNDCHG=DEFERRED, RATCHG=IMMEDIATE, PRACHG=DEFERRED, ADDUPF=IMMEDIATE, INSERTISMF=DEFERRED, CGISAICHG=DEFERRED, RAICHG=DEFERRED;
```

**配置RG级Trigger**：
```
ADD RGTRIGGER: CCTMPLTNAME="cct_test", ULCHG=NONREPORT, PRACHG=IMMEDIATE, PSDATAOFFCHG=IMMEDIATE;
```

**配置RG老化功能**：
```
SET RGRESCTRL: ONLRGAGESW=ENABLE, ONLAGETIMER=20, ONLBLKTIMER=0, OFLRGAGESW=ENABLE, OFLAGETIMER=20;
```

### 6. 配置异常返回码和异常信元动作

**故障处理动作**（`SET FAILHANDLING`）：
```
SET FAILHANDLING: CCTMPLTNAME="cct-test", FHACTION=CONTINUE, FAILOVERSUP=ENABLE, AUTOFAILBACK=ENABLE, TXTIMER=5, HOLDINGTIME=10;
```

| 参数 | 说明 |
|------|------|
| FHACTION | 超时处理动作：CONTINUE（继续业务）/RETRY_AND_TERM（重试后终止）/TERMINATE（终止） |
| FAILOVERSUP | Failover使能：ENABLE时主CHF故障向备CHF发消息 |
| AUTOFAILBACK | Failback使能：主CHF恢复后回切 |
| TXTIMER | Tx定时器时长(秒)，超时未收到CHF响应触发故障处理 |
| HOLDINGTIME | 用户保持时长(分钟) |

**PDU级异常返回码动作**（`ADD PDUSCACT`）：
```
ADD PDUSCACT: CCTMPLTNAME="cct-test", CODETYPE=VALUE, STATUSCODE=502, SCACT=FAILOVER;
```
- SCACT可选：FAILOVER（向备CHF重发）、CONTINUE（继续业务）、TERMINATE（终止会话）

**RG级异常返回码动作**（`ADD RGRCACT`）：
```
ADD RGRCACT: CCTMPLTNAME="cct-test", RCCODE=QUOTAMNOTAPPL, RCACT=BLCK_IMMED_RPT, BLKTIMER=30;
```

**忽略CHF响应消息信元**（`SET CNVRGDCHGPARA`）：
```
SET CNVRGDCHGPARA: IGNORERSPAVP=FUI, BADRSPACT=TERMINATE;
```
- `IGNORERSPAVP` 位域取值：INVOCATIONNUM（忽略序列号）、FUI（忽略FinalUnitIndication）、SESSIONFAILOVER（忽略sessionFailover）、FAILUREHANDLING（忽略FailureHandling）
- `BADRSPACT`：TERMINATE（异常信元导致去活）/CONTINUE（继续业务，不再配额管理）

### 7. 配置融合计费费率标识

费率标识配置链路：URR -> URRGROUP -> PCCPOLICYGRP -> RULE -> RULEBINDING -> UserProfile

**配置URR**：
```
ADD URR: URRNAME="urr_online_URL", URRID=2300, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=100, ONLMETERINGTYPE=VOLUME;
ADD URR: URRNAME="urr_offline_URL", URRID=12500, USAGERPTMODE=OFFLINE, OFFCOMPOUNDTYPE=ONLYRG, RG=1100, OFFMETERINGTYPE=VOLUME;
```

**配置URRGROUP**（建议同时配置在线和离线URR）：
```
ADD URRGROUP: URRGROUPNAME="UrrGp_URL", UPURRNAME1="urr_online_URL", DOWNURRNAME1="urr_online_URL", UPURRNAME2="urr_offline_URL", DOWNURRNAME2="urr_offline_URL";
```

**配置PCC策略组和Rule**：
```
ADD PCCPOLICYGRP: PCCPOLICYGRPNM="pccpolicy_URL", URRGROUPNAME="UrrGp_URL";

```

**RG配额耗尽后的默认动作**（`ADD QUOTAEXHAUSTACT`）：
- BLOCK：阻塞业务（FAR动作为drop）
- REDIRECT：重定向到指定地址
- FORWARD：继续转发业务

### 11. 配置计费消息缓存

主备CHF均故障时，SMF缓存计费消息，CHF恢复后回放。

```
SET FAILHANDLING: CCTMPLTNAME="cct-test", FHACTION=CONTINUE, TXTIMER=5, UPDCONTTERM=ENABLE, CNVCHGRECOVER=ENABLE, RECOVERINTERVAL=5;
MOD PDUSCACT: CCTMPLTNAME="cct-test", CODETYPE=VALUE, STATUSCODE=502, SCACT=FAILOVER;
SET N40MSGSTG: STGSWITCH=ENABLE, REPLAYINTERVAL=30, REPLAYRATE=500;
SET STGTRIGGER: VOLUMELIMIT=IMMEDIATE, PDUVOLUMELIMIT=500;
SET CNVRGDCHGPARA: CHGDATAREFGEN=SMF;
SET GLBDFTCHFGROUP: PRIMARYCHFGRP="ChfGroup1", SECONDARYCHFGRP="ChfGroup2";
SET CDRSTORAGECTRL: CDREXPIREDAY=2, CDREXPIREWEEK=4;
SET STGALARMCTRL: ALMTHD=5120, MONITORTIME=60;
SET N4CHGMSGCTRL: MSGCACHEEPDSW=ENABLE, CACHEFULLACT=CONTINUE;
```

**缓存前提条件**：
- FHACTION必须为CONTINUE（放通业务）
- SCACT必须为FAILOVER或CONTINUE
- STGSWITCH必须为ENABLE
- CHGDATAREFGEN必须为SMF
- 需配置全局默认CHF组

**关键参数**：
| 参数 | 说明 |
|------|------|
| REPLAYINTERVAL | 回放间隔(秒) |
| REPLAYRATE | 回放速率 |
| CNVCHGRECOVER | 融合计费自动恢复功能开关 |
| RECOVERINTERVAL | 自动恢复间隔(分钟) |
| CDREXPIREDAY/CDREXPIREWEEK | 缓存文件超期时长 |
| MSGCACHEEPDSW | N4计费消息缓存池扩展开关（20条扩展到40条） |
| CACHEFULLACT | 缓存池满动作：CONTINUE（继续业务） |

## 验证与调测

### 调测计费会话创建功能

**检查清单**（按优先级排查）：
1. 检查License开关：`LST LICENSESWITCH:LICITEM="LKV3W9BCC12"`
2. 检查计费模式是否为Nchf：`LST CHGMODE` / `LST APNCHGMODE`（FORCED应为NchfMode）
3. 检查融合计费是否使能：`LST CHARGECTRL`、`LST USRPROFCHARGE`、`LST APNCHARGECTRL`、`LST CHARGEMETHOD`
4. 检查CHF交互开关：`LST CCT`（CHF交互使能开关应为"激活发送"）
5. 检查是否有可用CHF：`LST GLBDFTCHFGROUP`、`LST SELECTCHFGBYCC`
6. 检查N40接口通信是否正常：检查ALM-100072告警

### 调测流量计费功能

**排查不一致的方法**：
1. 检查是否有免费业务URR（`LST URR` 查看ONLMETERINGTYPE/OFFMETERINGTYPE=FREE）
2. 检查PCF动态Rule是否指定计费RG
3. 检查预定义Rule是否绑定计费RG（`LST RULE` -> `LST PCCPOLICYGRP` -> `LST URRGROUP` -> `LST URR`）

### 调测主备CHF可靠性

**排查方法**：
1. 检查本地是否配置备CHF组：`LST SELECTCHFGBYCC`、`LST TNFBINDGRP`、`LST TNFGRP`、`LST TNFINSIP`、`LST TNFINS`
2. 检查CHF响应中sessionFailover是否为FAILOVER_NOT_SUPPORTED
3. 检查SMF本地Failover使能：`LST CCT`（FAILOVERSUP应为ENABLE）
4. 检查备CHF接口通信：检查ALM-100072告警

### 调测计费Trigger上报功能

**排查方法**：
1. 检查CHF是否下发对应Trigger类型（查看ChargingDataResponse消息triggers信元）
2. 检查SMF本地Trigger配置：`LST PDUTRIGGER`、`LST RGTRIGGER`
3. 注意Trigger冲突规则：Session级和RG级同时生效时以Session级优先

### 调测缓存消息回放功能

**前提条件**：
- SET FAILHANDLING FHACTION=CONTINUE
- ADD PDUSCACT SCACT=FAILOVER或CONTINUE
- SET N40MSGSTG STGSWITCH=ENABLE
- SET CNVRGDCHGPARA CHGDATAREFGEN=SMF

**排查方法**：
1. 检查主CHF接口通信：检查ALM-100072告警
2. 检查缓存文件是否超期：检查ALM-81059告警，`LST CDRSTORAGECTRL`
3. 检查回放时间间隔是否到达：`LST N40MSGSTG`
