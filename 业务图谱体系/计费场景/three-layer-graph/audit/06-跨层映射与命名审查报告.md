| Command → 哪些Task invokes它 | **部分** | 03各Task的command_refs可反查，05§4的invokes可反查，但无统一反向索引 |
| Task → 哪些Feature decomposes_to它 | **部分** | 05§3可从Feature侧查Task集，但无Task→Feature反向表；03的feature_ref可辅助 |
| Feature → 哪些CS uses_feature它 | **不支持** | 05§1只列CS→Feature方向；需手工反查 |
| ConfigObject → 哪些SO realized_by它 | **部分** | 01§5 SO的realized_by列可查，但反向(SO→Object)和(Object→SO)不完整 |

### 7.2 反向导航薄弱点

1. **Feature→CS反向不可查**：WSFD-109002无CS引用就是一例——如果支持反向查询，这个缺口能被工具自动发现
2. **Command→Task反向不可查**：如ADD QUOTAEXHAUSTACT被T-204 invokes，但无法从Command侧快速反查到Task
3. **建议**：在05文件中增加反向索引表，或在图谱数据库落地后通过双向边实现
