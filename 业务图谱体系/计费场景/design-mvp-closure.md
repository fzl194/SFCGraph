# 计费场景MVP闭包设计

## 目标
通过7批次阅读555篇原始文档，构建完整的计费场景知识库和业务图谱，
使Agent能直接基于知识库+业务图谱进行配置核查和配置生成。

## 三阶段流程

### Phase 1: 批次阅读与知识沉淀 (7轮)
每批读取原始md → 提炼知识（含可追踪来源） → 知识草稿

知识范围：方案设计、原理、配置、协议、运维、故障案例
每条知识必须标注来源文件路径

产出：knowledge-drafts/draft-batch-{01..07}.md

### Phase 2: 知识库融合
7份知识草稿 + 现有计费知识库(680行) → 融合为统一知识库

### Phase 3: 业务图谱更新
融合知识库 + 现有图谱(543行) + Schema(E1-E5/A1-A2) → 更新最终图谱

## 知识草稿格式

每条知识按以下结构记录，确保可追溯：

```markdown
## {主题}

### 知识条目
> 来源: {UNC相对路径}

**摘要**: 一句话概括
**详细内容**: 具体描述（方案设计/原理/配置/协议/运维）
**适用条件**: 何时适用
**关联知识**: 与其他知识条目的关系
```
