# WSFD-011201 支持离线计费 知识文档

## 概述

### 定义

离线计费是一种非实时的计费方式。SGSN/GGSN-C/SGW-C/PGW-C将GGSN-U/SGW-U/PGW-U上报的移动用户的计费信息（如时长、流量、事件等）按照一定的计费规则格式化为话单，通过Ga接口将话单消息发送到CG（Charging Gateway）。CG完成话单的存储、标准化等预处理工作，并将标准计费数据送到BS（Billing System）作最终处理。

> 3GPP TS 23.125 PCC架构中定义的离线计费的接口是Gz。从计费架构方面来说，它等同于Ga接口。

### 适用NF

SGSN、GGSN、SGW-C、PGW-C

### 版本可获得性

| 涉及NF | 支持版本 | 功能说明 |
|--------|----------|----------|
| SGSN | UNC 20.3.0及后续 | 按照预先设置的计费规则生成原始话单，发送到CG |
| GGSN/SGW-C/PGW-C | UNC 20.3.0及后续 | 通过N4接口下发URR信元，指示SGW-U/PGW-U的计费原则；按照计费规则生成话单发送到CG |
| SGW-U/PGW-U | UDG 20.3.0及后续 | 收集用户使用网络、业务等资源的信息，上报给控制面 |
| CG | 无特殊要求 | 对话单进行存储、合并、标准化预处理 |
| BS | 无特殊要求 | 处理CG发送来的计费数据，生成最终用户账单 |

### 特性规格

| 规格名称 | 规格指标 |
|----------|----------|
| ADD CHGIMSICFG 控制S-CDR话单生成的配置命令个数 | 2,048 |
| ADD CHGPLMNCFG 控制S-CDR话单生成的配置命令个数 | 1,500 |
| 整系统话单缓存能力（天） | 7 |
| 最多支持对接的CG个数 | 32 |

### 应用限制

- UNC支持最多与32个CG对接
- CG支持的话单版本字段需要与UNC的话单版本字段支持能力一致

### 与其他特性的交互

| 交互类型 | 相关特性 | 交互说明 |
|----------|----------|----------|
| 依赖 | WSFD-109002 内容计费基本功能 | 需要基于业务粒度对用户业务进行离线计费时需同时开启 |
| 依赖 | WSFD-109004 基于业务流量的计费 | 需要基于业务流量对用户业务进行离线计费时需开启 |
| 依赖 | WSFD-109003 基于业务时长的计费 | 需要基于业务时长对用户业务进行在线计费时需开启 |

## 核心概念

### 1. CC（Charging Characteristics，计费属性）

**定义**：计费属性指对用户所采用的计费类型，不同计费类型可以有不同的话单生成方式。CC由16位的字符串构成，用于决定会话的具体计费类型。CC往往由HLR/HSS/PGW-C提供，支持本地用户、漫游用户、拜访用户三种用户。

**CC优先级（GGSN/SGW-C/PGW-C）**：

```
UserProfile下的配置 > APN下的配置 > 全局配置 > normal（缺省：普通计费）
```

**CC选择逻辑**：
- 当左侧网元（SGSN/MME/SGW-C）携带签约CC，且本地基于UserProfile/APN配置CC时，由 `ADD CHARGECHAR` 命令的 HOMESGSN/ROAMSGSN/VISITSGSN 参数控制是否使用签约下发的CC
- 当左侧网元携带签约CC，且本地基于全局配置CC时，由 `SET GLBCHARGECHAR` 命令控制
- RADIUS Server下发CC优先级大于UNC本地配置
- 对于特定用户，无论是只做在线计费、离线计费，还是两者都做，一个用户只会对应一个CC

**4G网络中签约CC的来源**：
- SP合设场景：由左侧MME携带CC（MME从HSS获取的签约CC）
- SP分离部署场景：SGW-C由MME携带，PGW-C由SGW-C携带

**2/3G网络**：UNC作为GGSN，由左侧SGSN携带签约CC（SGSN从HLR获取）

**SGSN侧CC来源**（五个来源，优先级从高到低）：
1. HLR签约的APN计费属性
2. HLR签约的用户计费属性
3. SGSN根据APN配置的计费属性
4. SGSN根据用户MCC和MNC配置的计费属性
5. SGSN配置的本地/拜访/漫游用户缺省计费属性

**SGSN计费属性分类**：
| 类型 | 说明 |
|------|------|
| 普通计费属性（Normal） | 后付费用户，按照常规方式支付费用 |
| 预付费计费属性（Prepaid） | 获取服务前需要预支付费用 |
| 包月制计费属性（Flat rate） | 一个月内收费固定 |
| 实时计费属性（Hot billing） | 短时间或流量达到某值时及时生成话单 |

**SGSN是否忽略HLR签约CC的配置项**（六个参数）：
- HOME_SUBSCRIBER：是否忽略本网用户的签约计费属性
- ROAM_SUBSCRIBER：是否忽略漫游用户的签约计费属性
- VISIT_SUBSCRIBER：是否忽略拜访用户的签约计费属性
- HOME_APN：是否忽略本网用户的签约APN计费属性
- ROAM_APN：是否忽略漫游用户的签约APN计费属性
- VISIT_APN：是否忽略拜访用户的签约APN计费属性

### 2. RG（Rating Group，费率组）

**定义**：使用相同的计费类型的服务形成的集合叫费率组，话单中的费率组信息用来标识和区分业务费率。可以对PDP上下文/EPS承载内全部分组业务进行统一费率计费，或者为特殊业务进行独立费率计费。

**配置要点**：
- RG是必须的费率标识，SID是可选的
- 通过 `ADD URR` 命令配置，参数 `USAGERPTMODE=OFFLINE`，`RG` 指定费率组值
- 参数 `OFFCOMPOUNDTYPE`、`RG`、`SID` 组合必须唯一
- URR通过 `SET GLBURRGROUP` 绑定到全局URR组，或通过 `ADD URRGROUP` 创建URR组

### 3. SID（Service Identifier，业务识别标识）

**定义**：SID代表一条业务数据流，是RG的子集。在计费过程中，如果需要使用SID，则必须和绑定的RG一起使用。

**RG与SID关系示例**：

| RG | RG描述 | SID描述 | SID |
|----|--------|---------|-----|
| 1 | 流量5元/Gb，通话免费 | 流量5块钱/Gb | 1 |
| 1 | 流量5元/Gb，通话免费 | 通话免费 | 2 |
| 7 | 使用A公司旗下所有软件流量不限量 | A公司旗下B软件流量不限量 | 11 |

### 4. 话单（CDR，Charging Data Record）

**定义**：话单是离线计费记录用户业务使用情况的最小单元。基于每个PDP上下文/EPS承载生成，用于统计用户的服务使用状况。PDP上下文/EPS承载生命周期内产生一张或多张话单，产生多张时称为部分话单（Partial CDR）。

## 原理与流程

### 离线计费总体流程（GGSN/SGW-C/PGW-C）

```
1. 用户发起承载建立请求
2. SGW-C/PGW-C发送PFCP Session Establishment Request，携带Create URR信元组
   - URR ID：唯一标识
   - Measurement Method：计费方式（流量/时长/事件）
   - Reporting Triggers：上报触发条件
   - Volume Threshold / Time Threshold / Event Threshold
   - Linked URR ID：关联URR
3. SGW-U/PGW-U返回PFCP Session Establishment Response
4. 用户承载建立成功
5. 用户访问业务，到达阈值时SGW-U/PGW-U发送PFCP Session Report Request
   - 携带Usage Report信元组（URR ID、流量、时长、首尾包时间等）
6. SGW-C/PGW-C返回PFCP Session Report Response
7. SGW-C/PGW-C产生话单，通过Data Record Transfer Request发送到CG
8. CG返回Data Record Transfer Response
```

### CG交互流程

**链路状态检测**：
1. 配置完CG后（ADD CG），主动发送Node Alive Request，未收到则重发直到收到响应
2. 固定每隔一分钟发送Echo Request，缺省3次未收到响应则触发告警ALM-81021 CG无响应
3. 话单发送无响应时根据重发次数和时间间隔（SET CDRTRANSFER）重发
4. CG故障或负载过重时发送Redirection Request通知重定向到另一个CG

**链路状态判定**：
- 高通态（通断比>80%）：允许话单发送，允许历史缓存话单发送
- 低通态（50%~80%）：允许话单发送，不允许历史缓存话单发送
- 阻塞态（<50%）：禁止话单发送

### CG组网可靠性（GGSN/SGW-C/PGW-C）

**CG选择方式**：

| 方式 | 说明 |
|------|------|
| PCRF下发CG | 通过CCA-I消息下发主备CG的IP和端口，优先使用。必须在本地已配置才生效 |
| 本地CG服务器选择 | 从CG服务器池中选择状态正常、优先级最高的CG |

**CG负荷分担算法**（通过SET CDRTRANSFER设置）：

| 算法 | 说明 |
|------|------|
| 基于负载的负荷分担 | 每次发送话单时选择负载最少的CG。同一用户的话单可能发往不同CG |
| 基于用户的负荷分担 | 用户激活时选定CG，同一用户的话单每次发往同一CG。不同用户轮选CG |

**CG组选择逻辑**：
1. 如果配置了CG组，通过号段匹配CG组
2. 匹配上CG组则在CG组内按原则选择CG
3. 未匹配上CG组但配置了全局CG组，在全局CG组内选择
4. 否则在非CG组的全局CG中选择状态正常、优先级最高的CG

**CG过载保护**：提供WAL（Windows Access Limit）过载保护算法，CG待处理负荷超出配置的WAL值时置为不可用，负荷转移到其他CG。

### SGSN侧特殊流程

**话单发送架构**：
```
SPP/UPP进程 -> CDP进程 -> 话单池 -> CG话单发送队列 -> CSLB/VNRS微服务 -> CG
```

**异常处理机制**：
- CDP进程话单池空闲比例<=50%时，通知UPP/SPP不生成中间话单
- 话单池空闲比例<=40%时，话单写入硬盘；大于40%时读回话单池发送
- 话单挂载到CG发送队列失败时，写入硬盘

**CG选择流程**（SGSN）：
CG配置依次有五个来源：
1. 配置的PLMNCG
2. 配置的计费行为CG
3. 配置的计费属性CG
4. GGSN推荐的CG
5. 配置的"缺省CG标识"为"YES"的CG

"PLMNCG扩展范围"控制PLMN用户话单发送到PLMNCG还是非PLMNCG。

## 配置规则

### 一、配置到CG的数据（GGSN/SGW-C/PGW-C）

**配置步骤**：
1. 创建L3VPN实例：`ADD L3VPNINST`
2. 创建VPN实例：`ADD VPNINST`
3. 配置Ga接口集中点模式：`SET CONCENPOINT`
4. 配置Ga逻辑接口：`ADD LOGICIP` + `ADD LOGICINF`
5. 配置CG信息：`ADD CG`
6. 可选-配置CG组：`ADD CGGROUP` -> `ADD CGBINDING` -> `ADD CGGRPBINDING`
7. 可选-配置全局CG组：`SET GLBCGGROUP`
8. 配置CG负荷分担算法：`SET CDRTRANSFER`

**关键参数**：
| 命令 | 参数 | 说明 |
|------|------|------|
| ADD CG | IPVERSION/IPV4ADDR/PORT | CG的IP版本、地址和端口 |
| ADD CG | CDRTYPE | 话单类型（R7等） |
| ADD CG | PRIORITY | CG优先级，数值越小越高 |
| SET CDRTRANSFER | CGSELECTIONMODE | CG负荷分担算法 |
| SET CDRTRANSFER | RETRANSTIMES/RETRANSINTERVAL | 重传次数和间隔 |
| SET CDRTRANSFER | GTPPMAXPAYLOAD | GTP'消息最大字节数（1200~7180） |
| SET CDRTRANSFER | NARESTRANSINTVL | Node Alive消息重传间隔 |

### 二、配置离线/在线计费方式（GGSN/PGW-C）

**GGSN/PGW-C离线计费方式优先级**（从高到低）：
```
PCRF下发 -> SET CHARGECTRL总开关 -> RADIUS下发 -> UserProfile -> APN -> 计费属性CC
```

**总开关**：`SET CHARGECTRL` 控制HOMEOFFLINE/VISITOFFLINE/ROAMOFFLINE和HOMEONLINE/VISITONLINE/ROAMONLINE。PCRF下发不受此总开关控制。

**注意**：针对同一个用户，不同属性下规划的计费方式要一致，否则用户计费出错。

**配置命令**：
| 粒度 | 命令 | 说明 |
|------|------|------|
| 用户归属属性 | SET CHARGECTRL | 总开关，控制本地/漫游/拜访用户的在线/离线计费 |
| UserProfile | SET USRPROFCHARGE | OFFLINESW/ONLINESW |
| APN | SET APNCHARGECTRL | OFFLINESW/ONLINESW，默认INHERIT |
| 计费属性CC | ADD CHARGEMETHOD | CHARGECHARACT/CCVALUE/MASK + OFFLINE/ONLINE |

**免费业务配置**：`ADD URR` 中 USAGERPTMODE=OFFLINE，OFFMETERINGTYPE=FREE

### 三、配置SGW-C离线计费方式

**SGW-C特点**：
- 不受PCRF控制，纯本地配置
- 本地配置有两种粒度：APN和计费属性CC
- 不支持UserProfile粒度（区别于PGW-C/GGSN）

**配置命令**：
| 粒度 | 命令 | 说明 |
|------|------|------|
| 用户归属属性 | SET SGWCHARGECFG | HOMEOFFLINE/ROAMOFFLINE/VISITOFFLINE + SGWCGFLAG |
| 计费属性CC | SET SGWCHGMETH | CHARGECHAR + OFFLINEFLAG |
| APN | SET SGWAPNCHGMETH | APN + OFFLINE，默认INHERIT |
| 号段组 | ADD SGWSEGGCHGMETH | 基于IMSI/MSISDN号段组 |

**SGW-C Home PLMN配置**（可选）：`ADD NGSGWPLMN` -- 当PGW-C与SGW-C合设但需要按不同漫游属性计费时使用。

**SP合一网关话单控制**：`SET OFCCDRPARA` 的 SPCDRCONTROL=PGW_SGW_CDR 配置SP合一用户产生SGW话单。

### 四、配置离线计费参数（GGSN/SGW-C/PGW-C）

**四级优先级**（从高到低）：
```
UserProfile > APN > 计费属性CC（Charge Characteristic） > 全局（GLOBAL）
```

**OFCTemplate模板配置步骤**：
1. `ADD OFCTEMPLATE` -- 配置模板名、话单版本、时长配额机制、定时器等
2. `SET OFCTHRESHOLD` -- 配置阈值：TIMETHRESHOLD/VOLUMETHRESHOLD/CONDITIONCHANGE/SERVINGNODECHNG
3. `SET CDRTRIGGER` -- 配置话单产生条件：CDRTRIGRATCHNG/CDRTRIGSNCHNG/CDRTRIGMAXCHNG/CDRTRIGTIMEZONE/CDRTRIGPLMNCHNG
4. `SET CONTAINERTRIGGER` -- 配置容器产生条件：CONTTRIGRATCHNG/CONTTRIGQOSCHNG等

**模板绑定方式**：
| 绑定对象 | 命令 | 说明 |
|----------|------|------|
| UserProfile | SET USRPROFCHARGE | PGWOFCTEMPLATE + SGWOFCTEMPLATE |
| APN | SET APNCHARGECTRL | PGWOFCTEMPLATE + SGWOFCTEMPLATE |
| 计费属性CC | ADD GLBOFCTEMPLATE | GLOBALFLAG=CHARGE_CHARACT + CCVALUE/CCMASK/CCPRIORITY |
| 全局 | ADD GLBOFCTEMPLATE | GLOBALFLAG=GLOBAL |

**关键参数说明**：
| 参数 | 说明 |
|------|------|
| TQM | 时长配额机制：QCT（配额关闭定时器） |
| QCTVALUE | QCT时长（秒） |
| QHTVALUE | QHT时长（秒），建议30 |
| GCDRVERSION | G-CDR版本，如R7V740_GCDR |
| PGWCDRVERSION | PGW-CDR版本，如R8V850_PGW_CDR |
| SGWCDRVERSION | SGW-CDR版本，如R8V850_SGW_CDR |
| THRESHINHERIT | NON_INHERIT表示不继承上一级 |

### 五、配置计费属性CC（GGSN/SGW-C/PGW-C）

**CC配置命令**：
| 粒度 | 命令 | 说明 |
|------|------|------|
| CC实例 | ADD CHARGECHAR | CCNAME + HOME/ROAM/VISIT（CC值）+ HOMESGSN/ROAMSGSN/VISITSGSN |
| UserProfile绑定 | SET USRPROFCHARGE | CCNAME |
| APN绑定 | SET APNCHARGECTRL | CCNAME |
| 全局CC | SET GLBCHARGECHAR | HOME/ROAM/VISIT + HOMESGSN/ROAMSGSN/VISITSGSN |

**CC选择图中的控制参数**：HOMESGSN/ROAMSGSN/VISITSGSN=ENABLE时使用签约下发的CC，DISABLE时使用本地配置的CC。

### 六、配置费率标识（GGSN/SGW-C/PGW-C）

**三种费率标识配置粒度**：

#### 6.1 全局粒度的Session级计费费率标识
```mml
ADD URR: URRNAME="offlineURR", URRID=1000, USAGERPTMODE=OFFLINE, OFFCOMPOUNDTYPE=RGSID, RG=10, SID=20, OFFMETERINGTYPE=VOLUME;
SET GLBURRGROUP:UPOFFURRNAME="offlineURR", DOWNOFFURRNAME="offlineURR";
```

#### 6.2 UserProfile粒度的Session级计费费率标识
```mml
ADD URR: ...（同上）
ADD URRGROUP: URRGROUPNAME="urrgroup1", UPURRNAME1="offlineURR", DOWNURRNAME1="offlineURR";
ADD USERPROFILE:USERPROFILENAME="up-test", UPTYPE=PCC;
SET URRGRPBINDING:USERPROFILENAME="up-test",DFTURRGRPNAME="urrgroup1";
```
> 没有绑定Rule的UserProfile下的用户数据包不会创建五元组，不占用五元组资源

#### 6.3 内容计费费率标识
```mml
SET LICENSESWITCH: LICITEM="LKV3W9BCC12", SWITCH=ENABLE;  // 打开内容计费License
ADD URR: ... USAGERPTMODE=OFFLINE, OFFMETERINGTYPE=EVENT_VOLUME_TIME;
ADD URRGROUP: ...
ADD PCCPOLICYGRP: PCCPOLICYGRPNM="cg-test", URRGROUPNAME="urrgroup1";
ADD RULE: RULENAME="Rule001", POLICYTYPE=PCC, PRIORITY=110, POLICYNAME="cg-test";
ADD USERPROFILE: ...
ADD RULEBINDING: USERPROFILENAME="up-test", RULENAME="Rule001";
```

**URR参数说明**：
| 参数 | 说明 |
|------|------|
| USAGERPTMODE | OFFLINE表示离线计费 |
| OFFCOMPOUNDTYPE | RGSID/RG_ONLY/SID_ONLY |
| RG | 费率组值（必须） |
| SID | 业务标识值（可选） |
| OFFMETERINGTYPE | 统计类型：VOLUME/TIME/EVENT_VOLUME_TIME等 |

### 七、配置费率切换

#### 7.1 GGSN/SGW-C/PGW-C费率切换

**费率切换配置优先级**（从高到低）：
```
UserProfile实例 > APN实例 > 计费属性CC > 整机 > 默认值
```

**配置步骤**：
1. 配置节假日：`ADD FESTIVAL` -- GLOBALFLG/CCVALUE/MONTH/DAY
2. 配置星期：`ADD WEEKDAY` -- GLOBALFLG/CCVALUE/DAYOFWEEK/TARIFFTYPE
3. 配置费率切换时间段：`ADD TARIFFGROUP` -- TARIFFGRPNAME/GLOBALFLG/CCVALUE/TARIFFTYPE/STARTTIME/ENDTIME
4. 配置全局费率切换组：`SET GLBTARIFFGROUP`
5. 配置APN下费率切换组：`SET APNCHARGECTRL` 的 TARIFFGRPNAME
6. 配置UserProfile下费率切换组：`SET USRPROFCHARGE` 的 TARIFFGRPNAME

**费率类型**：WORKDAY（工作日）、WEEKEND（周末）、FESTIVAL（节假日）

**重要约束**：
- CC值不能取0x0200（统一费率类型，不会发生费率切换）
- 每个APN/UserProfile只能绑定一个费率切换组
- 具有相同CC值的时间段不允许重叠

#### 7.2 SGSN费率切换

**SGSN专用命令**：
| 命令 | 说明 |
|------|------|
| ADD CHGHOLI | 增加节假日配置（YEAR/MONTH/DAY/CC） |
| SET CHGWKDY | 设置星期配置（WKDAY/CC/TT） |
| ADD CHGTARI | 增加费率时段（TT/ST/ET/CC） |

### 八、配置计费和策略接口选择（GGSN/SGW-C/PGW-C）

**计费接口选择优先级**（从高到低）：
```
基于漫游属性（ADD ROAMCHGMODE） > 基于APN/DNN（ADD APNCHGMODE） > 全局配置（SET CHGMODE）
```

**策略接口选择优先级**：
```
基于APN/DNN（ADD APNPOLICYMODE） > 全局配置（SET POLICYMODE）
```

**计费接口类型**：GaGyMode（传统模式）、NchfMode（5G融合计费模式）

**策略接口类型**：Gx（4G）、Npcf/N7（5G）

**5G终端4G接入场景**：
- 5G终端4G接入是指在4G接入的PDN激活流程中，激活消息的PCO中携带5G PDU session ID信息
- 可通过BY5GSIWKI参数按5GS互操作标识选择接口
- SGW独立部署场景通过FORSGWONLY指定计费接口
- 5G终端国漫场景通过FORVSMFONLY指定计费接口

**基于PCF的最终决策**（可选）：`ADD PCCCHGMODEBYPCFID` -- 当策略接口选N7时，可决策计费接口是否由N40回落GaGy

### 九、配置话单字段（GGSN/SGW-C/PGW-C）

**话单字段优先级**：APN > 全局配置 > 默认值

**配置命令**：
1. `ADD CDRFIELDTEMP` -- 创建话单字段模板，控制各字段是否携带
2. `SET GLBCDRFLDTEMP` -- 绑定全局话单字段模板
3. `SET APNCHARGECTRL` -- 为APN绑定话单字段模板

**主要可控字段**：

| 类别 | 字段 | 说明 |
|------|------|------|
| 用户信息 | MSISDN、served-imeisv、CCSelectionMode、user5GFlag | 用户标识相关 |
| 接入信息 | ApnNetworkId、ApnSelectMode、PDPType、SGSNSGWPLMNId、ULI、RATType、MSTimeZone、DynAddrFlag | 接入方式相关 |
| 业务信息 | ListOfTrafficVolume、ListOfServiceData、RecordExtension、ImsSigContext、StartTime、StopTime | 业务统计相关 |
| 其他 | Diagnostics、RecSeqNumber、LocalRecSeqNum、PSFCI、NodeId、PDNConnectionId | 话单管理相关 |

**特殊字段控制**：
- `SET OFCCDRPARA` 控制PGW-CDR的流量容器/业务容器字段
- `BIT836` 软参控制是否携带华为CG的5G定制字段（user5GFlag），与华为CG对接时设为1

### 十、SGSN特殊配置

#### 10.1 SGSN话单生成策略

三种S-CDR话单生成策略：
1. **PLMN+APN NI策略**：`ADD CHGPLMNCFG`
2. **IMSI前缀+APN NI+拜访类型+CC策略**：`ADD CHGIMSICFG`
3. **全局计费属性策略**：`SET CHGCHAR`

**PLMN ID控制策略**：`SET CHGCDR` 的 PLMNCTRL 参数
- IMSI：使用基于IMSI前缀的策略
- LOCINFO：使用基于PLMN和APN NI的策略

#### 10.2 SGSN强制生成话单

两种方式：
1. `SET CHGCDR` -- 在特定时间内强制生成原因值为Management Intervention的S-CDR或M-CDR
2. `CRE CHGCDR` -- 为特定用户强制生成话单

## 话单类型与格式

### 话单类型（GGSN/SGW-C/PGW-C）

| 话单类型 | 版本 | 描述 | 适用网元 |
|----------|------|------|----------|
| G-CDR | R98/R99/R4/R5/R6/R7 | GPRS/UMTS用户计费，不支持内容计费 | GGSN |
| eG-CDR | R6/R7 | GPRS/UMTS用户计费，支持内容计费 | GGSN |
| SGW-CDR | R8/R9/R10 | LTE用户计费，不支持内容计费 | SGW-C |
| PGW-CDR | R8/R9/R10 | LTE/EPC融合用户计费，支持内容计费 | PGW-C |

> GPRS/UMTS网络可采用任意类型话单；EPC融合组网必须采用PGW-CDR以保持GUL切换前后话单类型一致。

### 话单容器

| 容器类型 | 描述 | 新容器产生条件 |
|----------|------|----------------|
| 流量容器 | 不区分业务，记录流量、时长 | QoS改变、费率时段改变、用户位置改变、产生话单 |
| 业务容器 | 按业务粒度记录流量、时长 | IP-CAN承载修改、费率时段改变、在线计费CCFH处理失败、产生话单 |

通过 `SET CONTAINERTRIGGER` 配置容器产生条件。

### 话单产生条件

| 话单类型 | 产生条件 |
|----------|----------|
| 部分话单 | 按时间（SET OFCTHRESHOLD的TIMETHRESHOLD）、按流量（VOLUMETHRESHOLD）、按计费条件改变次数（CONDITIONCHANGE）、强制产生（FOC GENERATECDR） |
| 最后话单 | MS/UE去激活 |

### SGSN话单类型

SGSN主要产生S-CDR和M-CDR：
- **S-CDR**：基于PDP上下文产生
- **M-CDR**：基于MM（移动性管理）产生

**S-CDR关闭原因**：用户PDP上下文去激活、到达流量阈值、到达时间阈值、手动生成CDR、系统切换

**M-CDR关闭原因**：用户去附着、到达时间阈值、SGSN变更、手动生成CDR、系统切换

### 话单编码与传输

- 话单按照ASN.1（Abstract Syntax Notation One）定义生成，编码采用BER（Basic Encoding Rules）方式
- 符合3GPP TS 32.298标准
- 话单通过GTP'协议、UDP、IP依次封装后通过Ga接口发送给CG

## 验证与调测

### 调测到CG的数据

1. `DSP CGSTATUS` 查看CG状态
2. 如果abnormal，执行PING检查连通性
3. `LST CG` 检查CG配置是否与对端一致
4. 检查告警ALM-81021 CG无响应
5. 创建Ga接口跟踪任务查看Echo Request/Response
6. 如果出现IP分片报文，调整 `SET CDRTRANSFER` 的GTPPMAXPAYLOAD

### 调测时间阈值功能

1. 设置时间阈值：`SET OFCTHRESHOLD` 的 TIMETHRESHOLD
2. 激活用户并访问业务，保持超过时间阈值
3. CG客户端查看话单的 `causeForRecClosing` 字段是否为 `Time Limit`，`duration` 是否符合阈值

### 调测流量阈值功能

1. 设置流量阈值：`SET OFCTHRESHOLD` 的 VOLUMETHRESHOLD
2. 激活用户访问业务，使流量超过阈值
3. CG客户端查看话单的 `causeForRecClosing` 字段是否为 `Volume Limit`

### 调测费率切换功能

1. 设置合适的话单时长阈值
2. 在费率切换点前激活用户
3. 经过费率切换点后去激活用户
4. CG客户端查看话单的List of Traffic Volumes字段中是否有费率切换关闭原因

### 调测CG指示重定向功能

1. 配置两个CG（CG1优先级高于CG2）
2. 激活用户访问业务
3. 停止CG1服务
4. 查看CG1是否发送重定向消息，话单是否转发到CG2

### 调测话单缓存功能

1. 激活用户访问业务
2. 断开CG连接（删除路由），等待CG状态变为abnormal
3. 继续生成话单，查看话单是否缓存到硬盘
4. 恢复CG连接，检查缓存话单是否正常发送

### 调测LastActivity计费

1. 确保LastActivity功能已开启：`SET OFCCDRPARA` 的 LASTACTIVITYSW=ENABLE
2. 确保话单时间格式为UTC：`ADD OFCTEMPLATE` 的 CDRTIMEFORMAT=UTC
3. 激活用户访问业务后去激活
4. 检查话单中RecordExtensions字段的LastActivityTime是否正确

### 关键查询命令

| 命令 | 用途 |
|------|------|
| DSP CGSTATUS | 查看CG状态 |
| LST CG | 查看CG配置 |
| LST CDRTRANSFER | 查看GTP'消息控制参数 |
| LST OFCTEMPLATE | 查看离线计费模板配置 |
| LST APNCHARGECTRL | 查看APN计费配置 |
| LST TARIFFGROUP | 查看费率切换组 |
| LST WEEKDAY | 查看星期表 |
| LST FESTIVAL | 查看节假日 |
| LST OFCCDRPARA | 查看离线话单参数 |
| FOC GENERATECDR | 强制产生话单 |
| DSP SMPDPNUM | 查询PDP上下文数 |

## 话单可靠性

### GGSN/SGW-C/PGW-C可靠性机制

| 机制 | 描述 |
|------|------|
| 预防重复话单 | 发送给备用CG的GTP'消息携带"可能重复"标记 |
| 缓存话单信息 | 所有CG链路阻塞时缓存话单，缓存目录charge1/charge2，超期产生ALM-81059 |
| 抑制零流量话单 | 用户未进行业务时不产生中间话单（去激活和强制产生除外），通过SET ZEROCHGSKIPSW控制 |
| 抑制话单产生 | 话单池空间不足时（ALM-81005），抑制更新类trigger的话单产生，控制用户接入 |
| 备份计费信息 | 计费信息定时备份到CSDB_VNFC，POD故障时其他计费POD接替 |

### SGSN可靠性机制

| 机制 | 描述 |
|------|------|
| 重复话单禁止 | CG故障恢复后，SGSN询问是否已收到话单，通知重定向CG删除重复话单 |
| 硬盘话单文件覆盖 | 硬盘空间不足时自动删除最旧话单文件（SET CHGGA: CDROVERWRITE=YES） |

**硬盘空间状态**：
| 状态 | 条件 |
|------|------|
| 硬盘空间正常 | 占用量 < 告警门限 |
| 硬盘空间不足 | 占用量 > 告警门限（可配置50%~90%） |
| 硬盘满 | 占用量 > 99% |
