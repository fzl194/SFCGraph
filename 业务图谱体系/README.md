> 本目录是全部业务图谱的根目录，同时承载**构建工具**（SOP + 三层图谱定义）和**构建产物**（各场景图谱）。

## 文件定位与阅读顺序

### 首次构建（新业务域/新子场景）

| 步骤 | 文件 | 目的 |
|------|------|------|
| 1 | `三层图谱本体标准定义.md` | 理解为什么要分三层、每层回答什么问题 |
| 2 | `三层图谱对象与关系设计.md` | 通过计费案例理解对象间关系怎么走 |
| 3 | `三层图谱Schema-最终版-v0.1.md` | 确认对象字段、关系边、禁止关系（实例化时查） |
| 4 | `知识流转与图谱构建SOP.md` | 按SOP 5阶段执行全流程 |

### 日常构建（已理解图谱模型）

直接从SOP §1（全链路总览）开始，Schema作为字典按需查阅。

## 使用约束

- **Schema是唯一权威**：对象字段、关系边、禁止关系以 `三层图谱Schema-最终版-v0.1.md` 为准
- **EV ID编号格式**：全库统一用编号格式（EV-FK-01），不用语义别名（EV-FK-BWM）
- **并行Agent限制**：最多2个并行Agent
- **审查必须追溯权威源**：每次修改前验证原始产品文档
