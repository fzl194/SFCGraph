## 知识文档模板

每个特性知识文档包含以下section：

```
# {特性ID} {特性名称} 知识文档

## 概述（是什么、适用NF、License、前置条件）
## 核心概念（关键术语、对象模型）
## 原理与流程（实现原理、业务流程、协议交互）
## 配置规则（激活步骤、MML命令、参数说明、约束条件）
## 配置案例（典型场景配置示例）
## 验证与调测（验证方法、常见问题）
## 参考信息（接口、信元、与其他特性的关系）
## 知识来源（文档清单）
```

## 执行方式

- 每次阅读1个特性（1个batch），14个batch串行执行
- 每个batch用Agent并行读取该特性的所有文档并产出知识md
- 14个全部完成后，产出跨特性分析文档
- 文档路径来源：`charging-feature-doc-list.md`

## 约束

- 全量沉淀：不遗漏特性文档中的有用信息
- 知识必须来源于原始特性文档，不凭空推断
- 每个特性知识文档自成体系，不依赖其他特性文档
- 跨特性关系仅在14个特性全部读完后构建
