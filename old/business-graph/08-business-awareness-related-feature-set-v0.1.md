# 业务感知相关特性子集 v0.1

## 1. 本页定位

本页不是特性图谱全景。

本页只收：

```text
当前已经通过原始语料确认，
且与业务感知主线直接相关的第一批特性子集
```

## 2. 第一批直接相关特性实例

## 3. 每个特性在业务感知里的作用

### 3.1 GWFD-110101 SA-Basic

作用：

- 提供三四层解析
- 提供协议识别
- 提供七层解析
- 提供业务识别后的多样化处理基础

为什么是业务感知底座：

- 文档定义里直接写了“判断用户业务，并根据规划，对这些业务进行多样化处理，如计费、执行重定向、头增强”
- 参考信息页直接给出了业务感知命令主集

### 3.2 GWFD-020301 内容计费基本功能

作用：

- 在 UDG 侧承接“基于业务粒度的计费”
- 依赖 SA-Basic 先识别业务类型，再标记对应 URR ID

为什么直接相关：

- 文档定义里直接写了“基于业务的计费”
- 与其他特性的交互里直接依赖 `GWFD-110101 SA-Basic`

### 3.3 WSFD-109002 内容计费基本功能

作用：

- 在 SMF/UNC 侧承接“内容计费”
- 根据本地或 PCF 下发规则，把业务粒度计费信息通过 PDI/URR 下发到 UDG/UPF

为什么直接相关：

- 文档原理明确描述了：
  - SMF 获取计费规则
  - 通过 PDI 下发到 UDG/UPF
  - UDG/UPF 匹配后标记 URR ID
  - 触发 Trigger 后上报计费信息

### 3.4 WSFD-011206 支持融合计费

作用：

- 承接 5G 场景的 Nchf 融合计费
- 统一在线和离线计费能力
- 为“业务粒度内容计费”提供统一对端交互模型

为什么直接相关：

- 文档中明确将“内容计费（FBC）”列为融合计费分类之一
- 直接依赖 `WSFD-109002 内容计费基本功能`

## 4. 当前子集和全景的关系

当前子集只覆盖业务感知的“计费主线”。

还未正式并入的业务感知相关特性，后续应逐步扩展，例如：

- URL 过滤相关特性
- 七层重定向相关特性
- 带宽控制相关特性
- 异常下行流量检测 / 防欺诈相关特性
- 头增强相关特性

所以本页的定位是：

```text
业务感知第一阶段已确认的直接相关特性子集
```

不是最终全集。
