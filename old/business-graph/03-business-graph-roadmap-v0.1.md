# 业务感知业务图谱路线图 v0.1

## 1. 目标

本页回答：

```text
业务感知业务图谱不是只做一个最小MVP，
而是如何从一条闭包逐步扩展到整个业务感知工作面。
```

## 2. 当前边界

当前边界是：

- 命令图谱全景正在构建
- 特性图谱全景正在构建
- 业务图谱不能等待前两层 100% 完成后再开始
- 业务图谱允许直接回查原始命令和特性文档

所以当前做法是：

```text
用前两层定义做骨架
同时直接回查原始语料
按业务感知主线逐步扩展
```

## 3. 路线分期

### 3.1 第一阶段：计费主线打通

目标：

- 差异化计费
- 默认兜底计费
- 免费 / 独立 RG
- UDG/UNC 计费对象链闭包

输出：

- 第一批 `NetworkScenario`
- 第一批 `DeliverySolution`
- 第一批 `delivery_solution_capability_mapping`
- 第一批 `feature_capability_mapping`
- 第一批 `delivery_solution_task_mapping`
- 第一批 `engineering_task_config_mapping`

### 3.2 第二阶段：配额与动作扩展

目标：

- 配额耗尽重定向
- 配额耗尽阻断
- 继续转发
- QuotaExhaustAct、重定向、运行态切换

### 3.3 第三阶段：七层识别与访问控制扩展

目标：

- URL/Host/Path 识别
- URL/域名访问限制
- 七层重定向
- 规则优先级和兜底逻辑

### 3.4 第四阶段：体验与增强扩展

目标：

- 带宽控制
- 头增强
- 安全增强 / 防欺诈

### 3.5 第五阶段：业务感知全景收口

目标：

- 收口业务感知相关特性全集
- 收口业务感知相关命令/对象链全集
- 收口业务感知主要场景全集
- 形成业务感知完整工作面

## 4. 每一阶段都要同步收什么

每推进一个阶段，不只收业务场景本身，还要同步收：

### A. 相关特性子集

例如：

- 哪些特性真正支撑该阶段场景
- 这些特性的可获得性、典型路径、限制是什么

### B. 相关命令/对象链子集

例如：

- 该阶段涉及哪些配置对象
- 哪些命令形成对象链
- 哪些关系是 `references / binds / contains`

### C. 场景闭包

例如：

- 业务目标
- 语义实例
- 方案候选
- 工程任务
- 验收和诊断

## 5. 当前最小闭包的作用

当前最小闭包：

```text
差异化计费 + 默认兜底计费
```

它不是最终范围，而是：

- 第一条工作主线
- 第一批审查样板
- 第一套业务图谱构建方法验证

## 6. 当前进展

当前已经完成两条可审闭包：

1. 计费主线
   - 差异化计费
   - 免费业务
   - 默认兜底计费
2. 配额与动作主线
   - 配额耗尽动作
   - 配额耗尽后重定向
   - 七层重定向

## 7. 当前下一步

当前最合理的下一步是：

1. 对现有两条闭包做业务建模审查
2. 补齐 `RuntimeFlow`、`ValidationPlan`、`DiagnosisRule`、`RiskConstraint`
3. 再进入第三条主线：带宽控制或访问控制
4. 每新增一条主线，都同步更新场景、能力、任务、对象链和证据映射

通过后，再继续 Phase 2、Phase 3、Phase 4、Phase 5。
