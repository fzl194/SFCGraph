# 命令层 YAML 字段定义

> 命令层两类对象（**命令 / 配置对象**）的 YAML 字段。**参数不单独建**（在命令 md 原文内）。

## 一、命令 (MMLCommand) — 11 字段

### 标识字段（必有）
| 字段 | 类型 | 必填 | 来源 / 提取规则 |
|---|---|---|---|
| `id` | string | ✅ | 逻辑ID `{nf}@MMLCommand@{name}`，如 `UDG@MMLCommand@ADD URR`（= 文件名 = 引用键，网元在最前）|
| `type` | string | ✅ | 固定 `MMLCommand` |
| `name` | string | ✅ | 英文命令名 = H1 括号内，如 `ADD URR`（保留空格）|
| `name_zh` | string | - | 中文命令名 = H1 括号前，如 `增加URR` |
| `nf` | string | ✅ | 网元，`--nf` |
| `version` | string | ✅ | 版本，`--version` |
| `category_path` | list | - | 目录链 = `--mml-dir` 下到文件的各级目录名 |
| `status` | string | ✅ | `active`（默认）/ `draft` / `stale` |

### 检索预留（轻解析，可空）
| 字段 | 类型 | 来源 / 提取规则 |
|---|---|---|
| `applicable_nf` | list | 命令功能章节 `适用NF：` 行 |
| `effect_mode` | string | 注意事项匹配 `立即生效 / 对新流 / 对新用户 / 延迟` |
| `is_dangerous` | bool | 注意事项/功能含 `高危 / 谨慎 / 严重影响 / 破坏性` → true |

## 二、配置对象 (ConfigObject) — 9 字段

> **产生规则**：只有配置类命令（ADD/MOD/DEL/RMV/SET）产生配置对象；查询类（LST/DSP）和动作类（ACT）**不产生**（但可关联已存在的）。

| 字段 | 类型 | 必填 | 来源 / 提取规则 |
|---|---|---|---|
| `id` | string | ✅ | `{nf}@ConfigObject@{object}`，如 `UDG@ConfigObject@URR`（object = 命令族 object_keyword）|
| `type` | string | ✅ | 固定 `ConfigObject` |
| `name` | string | ✅ | object_keyword（命令名第二 token，如 `URR`）|
| `name_zh` | string | - | ADD 命令中文去动词前缀（`增加URR` → `URR`）|
| `nf` | string | ✅ | `--nf` |
| `version` | string | ✅ | `--version` |
| `object_kind` | string | ✅ | 按族内动词推导：有 ADD/MOD/DEL/RMV → `entity`；仅 SET → `global_setting`；默认 `entity` |
| `applicable_nf` | list | - | ADD 命令的 `applicable_nf` |
| `status` | string | ✅ | `active` |

> 配置对象**无原始 md**（聚合而来），正文仅 `## 说明`（取自 ADD 命令功能）；关系在 `## 边`（被操作 = 同 object 的所有命令，含查询/动作类）。

## 扩展接口 / 已删字段
- 扩展：未来加字段直接在 YAML 末尾追加，脚本原样保留未知字段。
- 已删（不进 YAML）：`verb` / `object_keyword`（可从 name 解析）/ `command_category`。
- 命令正文（功能/注意事项/**参数说明**/实例/输出/参考）保留在原始 md，不进 YAML。

## 参数（不单独建）
**参数说明表在命令 md 原文内**，不进 YAML、不单独建 Parameter md。查参数读对应命令 md。
