# 查询接口

> Agent 访问三层图谱的唯一通道。命令行 curl 直调，BASE `http://127.0.0.1:8000/api/v1`。
> 端点分两类：**目录/定位类**（找锚点 ID）+ **取内容类**（拿 md）。

---

## 端点总览

| 端点 | 用途 | 阶段 |
|---|---|---|
| `GET /stats` | 版图统计（四层规模 + 域/场景 + 网元/版本） | ① 理解全景 |
| `GET /domains` | 全部业务域 md（业务层入口） | ② 选层/定位 |
| `GET /names` | `{id: name}` 字典（把正文 `[[ID]]` 渲染成可读名） | 定位辅助 |
| `GET /objects?layer=&type=&q=&nf=&version=&domain=` | 列对象（**层内定位主力**） | ② 选层/定位 |
| `GET /objects/{id}?version=` | 单对象结构化（frontmatter + 正文 + 出边） | ③ 看 |
| `GET /objects/{id}/neighbors?version=` | 单跳邻居（**含反链**，仅 curated 边 ⚠） | ③ 看周边 |
| `GET /objects/{id}/md?version=` | 原始 md 全文 | ③ 看 |
| `POST /md` | **批量取 md（fetch+expand 主力）** | ④ expand |
| `GET /subgraph?center=&hops=&type=&version=` | N 跳子图 | ③ 看局部图 |

> ⚠ `/neighbors` 和 `/subgraph` 只含 **curated 边**（`## 边` 段 + 无 `## 边` 对象的 ref/正文）。完备遍历必须 `POST /md` + 扫全文，见 [检索纪律.md](检索纪律.md) §1。

---

## 1. GET /stats — 理解全景（每次查询先调）

```bash
curl -s "http://127.0.0.1:8000/api/v1/stats"
```

关键字段：
- `per_layer`：四层（命令层/特性层/任务层/业务层）各多少对象 → 看完备度形状
- `per_domain` / `per_domain_scenario`：**当前到底有哪些业务域/场景** → 确认业务层+任务层覆盖范围
- `nfs` / `versions_per_nf`：网元（UDG/UNC）+ 版本

---

## 2. GET /objects — 层内定位主力

```bash
# 按 UI 层（命令层/特性层/任务层/业务层）
curl -s "http://127.0.0.1:8000/api/v1/objects?layer=特性层&q=计费"

# 按类型（更窄，层内进一步收窄）
curl -s "http://127.0.0.1:8000/api/v1/objects?type=MMLCommand&nf=UDG&version=20.15.2&q=URR"

# 业务域/场景过滤
curl -s "http://127.0.0.1:8000/api/v1/objects?layer=业务层&domain=business-awareness"
```

参数：`layer`（UI 层）/ `type`（单类型，优先于 layer）/ `q`（id + name 子串）/ `nf` / `version` / `domain` / `scenario` / `page` / `size`。

返回行：`{id, type, layer, nf, domain, scenario, name, versions[]}`。按 id 聚合（多版本合并）。

> `q` 是 id+name 的**子串匹配**（非语义）。中文/特殊字符直接放 query 一般可用；若异常，URL-encode。

---

## 3. GET /domains — 业务层入口

```bash
curl -s "http://127.0.0.1:8000/api/v1/domains"
```

返回 `[{id, name, md}, ...]`，每个 md 含 frontmatter + 正文 + `## 边`。**仅业务域**有此特化；其他层用 `/objects?layer=`。

---

## 4. POST /md — 批量取 md（fetch+expand 主力）

**Windows cmd.exe**（echo 管道喂 stdin，避开 JSON 引号地狱）：
```cmd
echo {"ids":["NetworkScenario@charging","UDG@AtomTask@ADD URR"]} | curl -s -X POST http://127.0.0.1:8000/api/v1/md -H "Content-Type: application/json" --data-binary @-
```

**Git Bash / Linux**：
```bash
curl -s -X POST http://127.0.0.1:8000/api/v1/md -H "Content-Type: application/json" -d '{"ids":["NetworkScenario@charging"]}'
```

返回 `{id: {version, md}}`。某 id 不存在 → 该条 `{error, available_versions}`，**其余 id 照常返回**（部分失败容错）。

- 不传 `version` → 每个 id 各取**最新现存版本**。
- 单个 id 也走批量（统一姿势）。

---

## 5. 单对象 / 邻居 / 原文

```bash
# 单对象结构化（frontmatter + 正文 + 出边）
curl -s "http://127.0.0.1:8000/api/v1/objects/UDG%40MMLCommand%40ADD%20URR"

# 单跳邻居（含反链 in，仅 curated 边 ⚠）
curl -s "http://127.0.0.1:8000/api/v1/objects/UDG%40ConfigObject%40URR/neighbors"

# 原始 md 全文
curl -s "http://127.0.0.1:8000/api/v1/objects/UDG%40MMLCommand%40ADD%20URR/md"
```

> `{id}` 含 `@` 和空格，URL 须 encode：`@`→`%40`、空格→`%20`。

---

## 6. ID 格式

- **业务层**（跨 NF，无版本）：`{Type}@{slug}`，2 段。如 `NetworkScenario@charging`。
- **其他层**（NF 隔离，带版本）：`{nf}@{Type}@{name}`，3 段。如 `UDG@AtomTask@ADD URR`（含空格）。
- `@` 与空格在 JSON body 里无需转义；在 URL path 里必须 encode。
