# 层包标准（Layer Package Standard）

> **每个层文件夹必须遵循的统一结构。** 加新层 = 按本标准建文件夹、填内容。
> 确保命令/特性/Task/业务四层同构，层与层独立、互不依赖。

---

## 1. 全局 vs 每层（划分原则）

- **全局的东西放顶层**（所有层共用）：治理、通用约定、共享脚本
- **每层的东西下沉到该层文件夹**：构建师 / 核查 / SKILL / 字段 / 模板 / 脚本 / change-requests

| 类别 | 位置 | 内容 |
|---|---|---|
| 全局 | 顶层 | `README` / `VERSION` / `CHANGELOG` / `演进机制` / `conventions/`（通用命名）/ `scripts/`（共享导出） |
| 每层 | `{layer}/` | 见下"层包必备结构" |

---

## 2. 层包必备结构

每个层文件夹（`command/` `feature/` `task/` `business/`）内部同构：

| 文件 | 必备 | 职责 |
|---|---|---|
| `agent.md` | ✅ | 该层构建师人设（角色/职责边界/输入输出/交接） |
| `check.md` | ✅ | 该层核查（只审不建 + 审查项类别） |
| `SKILL.md` | ✅ | 构建方法（输入/输出/流程/规范），声明 Spec 或 Procedure 体裁 |
| `字段定义.md` | ✅ | 该层对象的字段权威表 |
| `template/` | ✅ | 该层文件骨架 |
| `scripts/` | ✅ | 该层构建脚本 |
| `change-requests/` | ✅ | 该层变更请求（含 README + CR 文件） |
| 辅助文档 | 可选 | 调研、需求梳理等 |

> 每层包**独立可测**：自带"构建（agent）→ 核查（check）→ 反馈（change-requests）"闭环，不依赖其它层。
>
> **scripts/ 可选（Procedure 体裁层）**：`scripts/` 对 Spec 体裁层（command/feature，代码构建）必备；对 **Procedure 体裁层（task/business，Agent 手工构建）若无构建脚本可省略**，但须在该层 `SKILL.md` 说明（如 business 不建 scripts/，构建靠 Agent 读文档理解写 md）。task 层目前仅有迁移/采集辅助脚本，无 `build_all` 编排器。

---

## 3. 命名

- 层文件夹名固定：`command` / `feature` / `task` / `business`（小写英文）。
- 层内文件名见上表（统一）。

---

## 4. 加新层步骤

1. 建层文件夹（如 `feature/`）
2. 复制本标准的"层包必备结构"，逐项填：
   - `agent.md`（构建师人设）
   - `check.md`（核查）
   - `SKILL.md`（构建方法）
   - `字段定义.md` / `template/` / `scripts/`
   - `change-requests/README.md`
3. 在顶层 `README.md` 状态表标记该层进度。
