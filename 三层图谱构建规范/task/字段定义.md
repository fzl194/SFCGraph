# Task 层 YAML 字段定义

> task 层对象（atom / compound / feature_task）的 YAML 字段。每个对象 = YAML + 正文 + `## 边`。
> 体裁：Procedure（agent 理解输入自己写）。命令/特性层是 Spec，task 层不同。

## 一、atom (AtomTask) — 8 字段

> 一条命令一个 atom（1:1，**ID 用命令名做锚，弃编号**）。承载命令的"动态配置方法"（配置方法字典/DP/约束在正文，不进 YAML）。

### 标识字段（必有）

| 字段 | 类型 | 必填 | 来源 / 规则 |
|---|---|---|---|
| `id` | string | ✅ | `{nf}@AtomTask@{命令名}`，如 `UDG@AtomTask@ADD URR`（= 文件名；命令名做锚；local 保留空格；不含 version）|
| `type` | string | ✅ | 固定 `AtomTask` |
| `name` | string | ✅ | 命令名（= id 的 local），如 `ADD URR` |
| `name_zh` | string | ✅ | 配置动作中文名（**= 正文 H1 的配置动作名，同源**；区别于命令层"增加URR"），如 `配置使用量上报规则` |
| `nf` | string | ✅ | 网元，`--nf` |
| `version` | string | ✅ | 版本，`--version` |
| `ref` | string | ✅ | 指向命令层对象 `{nf}@MMLCommand@{命令名}`，如 `UDG@MMLCommand@ADD URR` |
| `status` | string | ✅ | `draft`（默认）/ `active` / `stale` |

### 不进 YAML（在正文 / 不存）

- 动态配置方法（配置方法字典 / DP / 约束）→ 正文（最小抽取，同命令/特性层）
- **DP/约束不编号、无独立锚点**：引用粒度 = md 级（`[[UDG@AtomTask@ADD URR]]` 指整个 atom md），无 md 内部章节级引用
- 命令静态知识（功能/参数真相表/规格）→ 命令层 md，atom 引子链过去，**不抄进来**
- **无证据**：不写 `source` / `source_evidence_ids`（task 层无证据概念）

### 已删（不进 YAML）

- `task_layer`（`type: AtomTask` 已隐含层）
- `task_intent`（并入正文引子/配置方法段叙述）
- 旧 `task_logical_name`（并入 `name_zh`）
- 旧编号 `0-XXXXX`（ID 改用命令名做锚，弃编号）

## 二、compound (CompoundTask) — 8 字段

> 多命令可复用步骤模块。`ref: null`。ID 用**英文名**做锚（Agent 取，kebab-case，稳定）。`command_set` 是复用判定 + 快速查询的核心字段。

| 字段 | 类型 | 必填 | 来源 / 规则 |
|---|---|---|---|
| `id` | string | ✅ | `{nf}@CompoundTask@{英文名}`，如 `UDG@CompoundTask@charging-trilogy`（= 文件名；英文名 kebab-case，Agent 取，稳定） |
| `type` | string | ✅ | 固定 `CompoundTask` |
| `name` | string | ✅ | 英文名（= id 的 local） |
| `name_zh` | string | ✅ | 步骤中文名（如 `计费三件套`） |
| `nf` | string | ✅ | 网元 |
| `version` | string | ✅ | 版本 |
| `command_set` | list | ✅ | 命令名列表（如 `["ADD URR","ADD URRGROUP","ADD PCCPOLICYGRP"]`），复用判定（Jaccard）+ 快速查询 |
| `status` | string | ✅ | `draft`/`active`/`stale` |

> ref 固定 null（compound 无上层对象）。
> compound 不重复 atom 的命令配置方法（atom 讲命令怎么配，compound 讲多命令怎么组装成步骤）。
> 无证据：不写 source/source_evidence_ids。

## 三、feature_task (FeatureTask) — 8 字段

> 特性级配置编排。`ref → Feature`。ID 用 **feature_code** 做锚（1:1 特性）。静态/动态拆分：Feature 层=静态知识，feature_task=动态配置编排（同构 atom→command）。

| 字段 | 类型 | 必填 | 来源 / 规则 |
|---|---|---|---|
| `id` | string | ✅ | `{nf}@FeatureTask@{feature_code}`，如 `UDG@FeatureTask@GWFD-020301`（= 文件名） |
| `type` | string | ✅ | 固定 `FeatureTask` |
| `name` | string | ✅ | feature_code |
| `name_zh` | string | ✅ | 特性名（配置编排视角） |
| `nf` | string | ✅ | 网元 |
| `version` | string | ✅ | 版本 |
| `ref` | string | ✅ | 指向特性层对象 `{nf}@Feature@{feature_code}` |
| `status` | string | ✅ | `draft`/`active`/`stale`；能力型底座特性可标 `foundation`（骨架） |

> 无证据：不写 source/source_evidence_ids。

## 扩展接口

未来加字段直接在 YAML 末尾追加；脚本/agent 原样保留未知字段（同命令/特性层）。
