## Slide 2

### 标题

为什么现在要调整三层图谱的组织方式

### 核心信息

- 旧模式：每个场景各做一套图谱，再放入 SKILL
- 三类耦合：SKILL 与资产耦合、构建与维护耦合、静态与动态知识耦合
- 新目标：图谱资产独立沉淀、Task 独立表达配置方法、SKILL 运行时按需取用

### 画面结构

- 左栏：旧模式闭环图
- 中栏：4 个问题卡片
- 右栏：3 个转向目标箭头块
- 底部：一句结论条

### 视觉要求

- 浅灰白背景
- 主色专业蓝，问题用少量暖红，转向目标用青绿
- 中文排版工整，信息密度高但不拥挤
- 风格像正式体系建设汇报，不像产品海报

## Slide 3

### 标题

三层图谱的核心结构

### 核心信息

- 静态三层：业务层、特性层、命令层
- 动态一层：Task
- Task 内部分层：feature task / compound task / atom task
- SKILL 不承载资产，通过 API 按需取用图谱知识

### 画面结构

- 中央主结构图：三层横排 + 右侧 Task 竖列
- 最右侧：SKILL 模块流程区
- 底部：结构性结论条

### 视觉要求

- 层级颜色明显区分：业务蓝、特性青绿、Task 橙、命令红橙、SKILL 灰蓝
- 关系线尽量简洁，不交叉混乱
- 既有架构图感，也有管理汇报的收束感

## Slide 4

### 标题

业务感知域里，这套结构已经长成什么样

### 核心信息

- 业务感知域下已有 3 个场景：计费、带宽控制、访问限制
- 计费场景下已有 7 个方案
- 方案不会直接落命令，而是逐级落到 feature task / compound task / atom task
- 最终进入 SKILL 的模块化执行流程

### 画面结构

- 左侧：业务感知域树
- 中部：计费场景 7 个方案卡片簇
- 右侧：内容计费基础 CS 到 SKILL 流程的展开链

### 视觉要求

- 信息要丰富，但保持强秩序感
- 用高亮方案卡和右侧展开链形成视觉焦点
- 看起来像“真实资产结构图”，不是概念草图

## 可追加页：图谱验证与迭代优化框架

### 标题

图谱验证与迭代优化框架

### 核心信息

- Agent SOP 构建三层图谱
- SKILL 通过 API 获取图谱并执行配置生成
- Test Agent 基于 input/output 测试集验证图谱效果
- Debug Agent 在测试全流程中定位三层图谱问题
- 反向修改图谱资产并重新进入验证
- 人工使用反馈也可以直接进入图谱修订闭环

### 画面结构

- 中轴主流程：5 个模块，从左到右
  - `Agent SOP 构建图谱`
  - `SKILL 调图谱执行`
  - `Test Agent 测试集验证`
  - `Debug Agent 问题定位`
  - `反向修改三层图谱`
- 形成一个自动闭环箭头
- 底部增加一条人工反馈支路：
  - `人工反馈 -> 标记问题位置 -> 图谱定向修订`
- 底部结论条简洁收束

### 视觉要求

- 清爽专业风，浅色背景
- 主流程强调工程闭环，不做算法感炫技
- 自动闭环和人工闭环都要看得出来
- 文案正式、克制，不夸大
