# Command Graph 工作准则

## 1. 目标

构建 UDG 20.15.2 的完整命令图谱。工作自主执行，无需人工审批。

## 2. 自我迭代规则

- **每个 Step 至少迭代 3 次，最多 5 次**
- 每次迭代必须：
  1. 运行脚本产出数据
  2. 读取产出 CSV，统计质量指标（非空率、解析成功率等）
  3. 随机抽样 10-20 条，读取原始 md 文件对比验证
  4. 发现问题 → 修改代码或 prompt → 重新运行
- 每轮迭代记录：发现的问题、修复措施、质量指标变化
- 当连续 2 轮无新问题发现，或已达到 5 轮上限，进入下一步

## 3. 质量标准

| 指标 | 目标 |
|------|------|
| 命令清单完整率 | ≥ 99%（文件数匹配） |
| 命令元数据解析率 | ≥ 95%（关键 section 成功提取） |
| 参数表解析率 | ≥ 90%（参数表格成功解析） |
| 枚举值提取率 | ≥ 85%（枚举型参数的取值范围成功提取） |
| 配置对象发现率 | ≥ 90%（命令正确归类到配置对象） |
| LLM 语义关系 evidence 准确率 | ≥ 80%（evidence 确实支持关系） |

## 4. LLM 使用准则

- **DeepSeek API**: 从 `.env` 读取配置
- **速率控制**: 请求间隔 ≥ 0.5s，避免触发限流
- **Prompt 设计**: 每条关系必须附带 evidence_text（原文引用）
- **错误处理**: API 失败时重试 3 次，间隔递增
- **成本控制**: 优先处理高价值关系，批量处理减少 API 调用

## 5. 代码规范

- 复用 `feature-graph/` 的代码模式：
  - `BASE_DIR = Path(__file__).resolve().parent`
  - `make_id()` 函数
  - CSV 读写（utf-8-sig, DictWriter）
  - Section 锚点正则解析
- 每个 step 脚本独立可运行
- 输出文件编码统一 utf-8-sig
- 日志输出到 stdout，包含进度和统计

## 6. 文件命名

- 脚本: `step{N}_{描述}.py`
- 数据: `udg_{描述}.csv` / `udg_{描述}.json`
- 审计: `data/audit/step{N}_audit_v{M}.md`
- 最终: `data/final/{表名}.csv`

## 7. 特殊情况处理

- **非命令文件**: 如 `MML命令申明_79861195.md`，在 Step 1 标记为特殊文件
- **格式不一致的文档**: 记录到审计报告，尝试兼容解析
- **解析失败**: 记录原始错误，不跳过，产出 failed 记录供后续分析
- **大型参数表**: 分块处理，避免内存溢出

## 8. Codex 审核

每个 Step 完成全部迭代后，调用 Codex 进行最终审核：
- 审核代码质量
- 审核数据质量
- 提出改进建议
- 如 Codex 发现问题，追加 1-2 轮迭代修复
