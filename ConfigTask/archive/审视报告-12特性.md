## 2. Task 定义结论

**task = 一个连贯的配置动作（意图 + 有序命令组）**，不是单命令。边界判据：一组命令共同产出**一个可命名的配置对象集** + 内部有跨命令约束。一个特性 = **0~N 个 task**：
- **0**：协议底座型（SA-Basic）。
- **1**：单动作（接入控制/VoLTE/eDRX/IPv6承载）；或**多场景同骨架**（IPSec 9 变体 = 1 task，差异是 DecisionPoint 变体）。
- **N**：多阶段/多子能力（内容计费 5、QoS 3）。

**颗粒度决策 = 阶段级**（操作流程的一个阶段 = 一个 task），与计费场景已建实例一致。

## 3. TaskRule 必要性结论 —— 保留

TaskRule 只管**task 内部、跨多条命令、离开此动作即失效**的协同约束。4 类（rule_type）+ 真实例子：

| rule_type | 例子 | 为何不能归别处 |
|---|---|---|
| `binding_rule` | FLOWFILTER 必须绑定 filter/l7filter 否则 Rule 无效 | 跨2命令的绑定强制，非单命令语法 |
| `relative_value_rule` | 只绑 L34 的 RULE 优先级 < 绑 L7 的；any RULE 最低 | 多条 RULE 相对约束，挂不上单命令，也不跨网元 |
| `conditional_param_rule` | 需计费→URRGROUPNAME 必填；ADC task 下 ADCMUTEFLAG=DISABLE | task 上下文决定参数必填/固定，非命令本身约束 |
| `consistency_rule` | IPSec VNRS↔IPsec 微服务双配原则（14+命令）；VoLTE PRIORITY1/2+PROTOCOL6/17 配对；eDRX 全局与 APN 缓存优先级 | 跨多命令一致性/配对，离开动作即失效 |

数量可控（12 特性约 10-12 条 TaskRule），不爆炸。

## 4. 规则归口总表（CommandRule / TaskRule / FeatureRule）

| 规则类型 | 归属 | 真实例子 |
|---|---|---|
| 单命令参数语法/取值/唯一性 | **CommandRule** | RULENAME+POLICYTYPE 唯一；DHGROUP 不能 None；反掩码语义 |
| task 内跨命令引用绑定强制 | **TaskRule** | FLOWFILTER 必绑 filter |
| task 内多对象相对取值约束 | **TaskRule** | RULE 优先级相对关系 |
| task 内条件→参数必填/固定值 | **TaskRule** | URRGROUPNAME 条件必填；ADCMUTEFLAG=DISABLE |
| task 内跨命令一致性/配对 | **TaskRule** | IPSec 双配；VoLTE 配对；eDRX 优先级 |
| 跨网元参数一致性 | **FeatureRule** | URRID/USAGERPTMODE 跨 SMF/UPF 一致；USERPROFILENAME 跨 PCF/SMF/UPF |
| 特性间能力依赖 | **FeatureRule** | ADC 依赖 SA+PCC；eDRX 依赖基本接入；URL过滤按协议依赖 SA-Web/SA-Mobile/HTTP2.0Host |
| 协议/能力边界 | **FeatureRule** | 内容计费不含加密协议解析；URL过滤仅 HTTP/HTTPS/WAP/非加密QUIC |
| 软参(BITxxx/BYTExxx)、License装载 | 产品级/FeatureRule | 各特性参考信息软参 |

**归口判据**：涉及 ≥2 条不同命令 + 不跨特性/不跨网元 + 离开当前 task 语境即失效 → TaskRule。否则单命令→CommandRule，跨网元/跨特性→FeatureRule。
