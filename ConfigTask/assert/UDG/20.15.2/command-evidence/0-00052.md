## ② 命令真相（mml_commands.jsonl）
**功能**：**适用NF：SGW-U、PGW-U、UPF**

![](设置N4U路径相关属性（SET UPN4UPATH）_18761567.assets/notice_3.0-zh-cn.png)

本命令属于高危命令，配置UPF主动向SMF发送GTP请求消息的重发时间间隔和最大尝试发送次数，如果配置不合理，可能导致用户激活失败或资源残留。

该命令用来设置N4U路径的Echo探测属性。
**notes（规格/上限→应投影 atom rule）**：
- - 该命令执行后立即生效。
- 该命令最大记录数为1。
- N4ECHOSW设置为使能ENABLE时，参数N4ECHOTIME、N4ECHOINTERVAL、DEACTIVEFLAG、N4T3RESPONSE、N4N3REQUEST才能生效。
- 在基站数目较多时，建议配置较大ECHOINTERVAL，防止单位时间内发送太多ECHO消息导致系统CPU上升影响基本业务。
- 如果基站数目较多但配置E

**参数真相表（代码解析）**：

| 参数 | 描述 | 数据来源 | 必选 | 默认 | 取值范围 |
|---|---|---|---|---|---|
| N4ECHOINTERVAL | N4接口发送GTP心跳请求的间隔时间 | local_planned | optional | 无 | 整数类型，取值范围为60～3600，单位是秒。 |
| N4T3RESPONSE | N4接口的GTP请求消息的重发时间间隔 | local_planned | optional | 无 | 整数类型，取值范围为1～20，单位是秒。 |
| N4N3REQUEST | N4接口的GTP请求消息的最大尝试发送次数 | local_planned | optional | 无 | 整数类型，取值范围为1～6。 |
| DEACTIVEFLAG | 是否去活路径上已激活的上下文 | local_planned | optional | 无 | 枚举类型。 |
| N4ECHOTIME | N4接口的GTP-U路径断告警后发送echo消息的次数 | local_planned | conditional | 无 | 整数类型，取值范围为0～60。 |
| N4ECHOSW | Echo开关 | local_planned | optional | 无 | 枚举类型。 |
