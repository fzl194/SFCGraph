### 1.7 互斥关系双向性

互斥在语义上是对称关系（A互斥B ⇔ B互斥A）。为同时满足"忠实原文档可追溯"和"前端展示对称"两个诉求，采用 `conflict_pair_id` 字段把两个方向的边关联起来。

**CSV 设计（2026-06-08 P1-1.5）**：
- FeatureDependency 表保留双向两条记录，每条记录带同一 `conflict_pair_id`，按 `sorted([source, target])` 拼接生成（如 `conflict:GWFD-010107-GWFD-010108`）
- 非 conflicts_with 类型的边 `conflict_pair_id` 为空
- 双向记录合并证据：两条记录各自的 `description`、`source_type`、`license_control_item` 都是独立证据源

**前端展示规则**：
- 按 `conflict_pair_id` 去重显示（46条独立互斥关系，而非64条CSV记录）
- 对双向匹配的 pair，合并展示两个方向的描述（双方各自如何声明）
- 对单向 pair，标注"仅一方声明"

**审查规则**：
- 同一 `conflict_pair_id` 的多条记录合并审查一次

**最终统计（P1-1.5 应用后）**：

| 指标 | 数值 |
|------|------|
| conflicts_with CSV 记录 | 64 |
| unique `conflict_pair_id` | **46** |
| 双向匹配（pair 出现2次） | 18 对 |
| 单向（pair 出现1次） | 28 条 |
| 异常（pair 出现≥3次） | 0 |

**P1-1.4 URL-ID优先修复（2026-06-08）**: 发现原始md中存在链接文本ID与链接URL路径ID不一致的笔误，例：

### 2.4 20个漏提取License的根因分析

**全部20个都是UNC (WSFD) 特性，UDG无漏提取。**

```
| 特性 | 适用NF | License控制项 |
|------|--------|-------------|
| 支持多HPLMN功能 | SGSN、MME | 82205877 LKV2HPLMN02 支持多HPLMN功能 |
| 支持多HPLMN功能 | AMF | 82209967 LKV2MHAM01 支持多HPLMN功能-UAM |
```

而当前代码的格式2只识别**2列表格**：
```
| 适用NF | License控制项 |
```

**格式分类统计**：

| 表格格式 | 漏提特性数 |
|----------|-----------|
| `特性 \| 适用NF \| License控制项` (3列) | 15 |

| `N7 \| 编码 名称` (接口名替代NF) | 1 |
| `涉及NF \| 支持版本 \| 功能说明` (License嵌入功能说明列) | 1 |
