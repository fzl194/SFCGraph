# VoLTE承载故障快速恢复

`WSFD-201204` · enhanced · required

## 所属目录

- [WSFD-201200](feature/UNC/20.15.2/WSFD-201200.md)

## 适用性

- 适用NF：MME、SGW-C、PGW-C
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=M; S/PGW-C(4G)=M; NB-IoT(4G)=-; 5G MME(5G NSA)=M; 5G S/PGW-C(5G NSA)=M; AMF(5G SA)=-; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

MME、SGW-C、PGW-C

## 定义

随着LTE网络的普及，语音业务会逐步从CS域切换到PS+IMS域。一旦PS网元故障，将会导致用户的语音业务中断，如果运维人员未及时处理故障，UE有可能长时间中断被叫语音业务，影响用户的业务体验。解决故障的关键在于尽快让UE重新附着PS网络。

- 在普通场景下，该特性让UE快速附着PS网络，降低MME/SGW-C/PGW-C故障对语音业务的影响。
- 在部署了VoLTE语音业务的网络中，MME Pool的组网场景下，用户正在VoLTE语音业务，如果MME故障，Pool内其它MME可以快速接替故障MME上的用户的业务，实现VoLTE承载的快速恢复，确保通话继续。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 减少网络故障和操作维护对VoLTE语音业务的影响，增强VoLTE语音可靠性，提升运营商的服务质量。<br>- 在IMS语音业务中断的情况下，快速恢复语音业务，减少用户投诉。 |
| 用户 | 缩短语音业务中断时间，提升用户体验。 |

## 应用场景

1. 本特性应用于部署了VoLTE语音业务的MME Pool网络，对VoLTE语音通话可靠性要求高时，比如执警、救援等呼叫业务中不会因为网络故障和操作维护中断，建议开启本特性。
2. 一般故障/重启场景，建议开启本特性：

## 对系统的影响

- 在普通故障/重启场景下，性能影响程度：低
    - 在MME故障/重启场景下，SGW-C会保留语音业务承载，会占用承载资源，具体影响跟保留的语音承载数量有关。
    - 在SGW-C故障/重启场景下，PGW-C会保留语音业务承载，会占用承载资源，具体影响跟保留的语音承载数量有关。
    - 在PGW-C故障/重启场景下，SGW-C发送PGW Restart Notification消息给MME，对系统性能影响可以忽略。
- 在部署了VoLTE语音业务的网络中，MME Pool的组网场景对系统无影响。

## 应用限制

- 漫游用户不支持SGW-C/PGW-C故障/重启场景故障恢复功能。
- MME/SGW-C故障场景下，SGW-C/PGW-C保留承载，保留承载期间，业务中断，如果采用时长计费，用户还会被计费，建议PS核心网不部署时长计费。
- 该特性不适用于PGW-C和MME同时故障的场景。
- 所有加入到MME pool/SGW pool的设备，都必须开启VoLTE承载故障快速恢复特性。
- 不支持SGW-U/PGW-U故障/重启时由SGW-C/PGW-C选择其它SGW-U/PGW-U（重启时可能选择原SGW-U/PGW-U）重建上下文。
- SGW-U故障重启场景主叫二次恢复依赖于SGW-C给MME返回MBR的失败响应时MME会删除PDN连接并且PDN连接删除后UE能重建PDN连接。
- N4接口消息超时时间应小于S5接口的N3T3时间。

## 原理概述

- 在普通故障/重启场景下，VoLTE承载故障快速恢复在不同故障场景的系统实现如[表2](#ZH-CN_CONCEPT_0286652960__table1865856818)所示。
  *表2 VoLTE承载故障快速恢复故障场景*

  | 故障场景 | 实现原理 |
  | --- | --- |
  | MME故障与恢复 | - SGW-C通过Echo消息探测到MME故障。<br>- SGW-C通过Recovery更新探测到MME重启，该MME恢复正常。<br>- SGW-C通过发送携带IMSI的DDN（Downlink Data Notification）消息给pool内的MME触发恢复。 |
  | SGW-C故障与恢复 | - SGW-C故障或重启后，PGW-C根据配置识别保留承载并进行标识，对于保留承载不执行去激活处理，并且通知PGW-U标识相关的保留承载。<br>- PGW-C通过发送PDTN(PGW Downlink Triggering Notification)消息给pool内的SGW-C触发恢复。 |
  | PGW-C故障与恢复 | PGW-C故障或重启后，SGW-C向各MME发送PRN消息，并去激活相关的PDN Connection。如果MME启动了PGW-C故障下的业务恢复，会通知UE重建相关会话。 |
  | SGW-U故障与恢复 | - SGW-U故障/重启时，SGW-C直接给PGW-C返回Context not found的失败响应，触发PGW-C保留承载。<br>- PGW-C通过发送PDTN(PGW Downlink Triggering Notification)消息给pool内的SGW-C触发恢复。 |
  | PGW-U故障与恢复 | PGW-U故障/重启时，PGW-C去激活相关的PDN Connection。去激活流程中通知UE重建相关会话。 |
- 在部署了VoLTE语音业务的网络中，MME Pool的组网场景中：在MME故障或重启时，正在进行的VoLTE语音通话会中断，对于紧急业务可能造成重要信息的丢失。VoLTE承载故障快速恢复支持GBR承载的链式备份，在MME故障或者重启情况下，Pool内的其它MME能快速接替故障MME帮助用户恢复VoLTE承载，通话继续。
