# SBI IMR信令上报

`WSFD-220002` · enhanced · none

## 所属目录

- [WSFD-220000](feature/UNC/20.15.2/WSFD-220000.md)

## 适用性

- 适用NF：AMF、SMF、NRF
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=-; S/PGW-C(4G)=-; NB-IoT(4G)=-; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=-; AMF(5G SA)=M; SMF(5G SA)=M; NRF(5G SA)=M`

## 适用NF（原文）

AMF、SMF、NRF

## 定义

SBI IMR信令上报是指网元支持将SBI接口解密后的原始信令使用IMR（initial message record）机制发送给单据服务器（如CloudUDN），单据服务器将原始信令通过xDR流北向接口发送给服务探针/SEQ系统/第三方服务器等。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 提升外置探针采集数据的完整性和准确性。<br>- 通过软探针，解决5G外置探针部署的问题。<br>- 降低网络运维复杂度。 |
| 用户 | 该特性对用户不可见。 |

## 应用场景

为了应对SBI接口加密的场景下，外置探针对消息解密的准确性有影响的问题，开通本特性支持内置软探针功能，实现SBI接口信令通过IMR上报。

## 可获得性

| **产品** | **支持版本** | **功能说明** |
| --- | --- | --- |
| UNC | 20.7.1<br>及后续版本 | 支持数据采集，上报SBI接口信令，生成IMR单据，并向单据服务器上报。 |
| 单据服务器 | 无特殊要求 | - 支持采集UNC上报的原始信令，并进行海量存储。<br>- 预处理原始信令，输出北向数据至第三方服务器。 |

**License** **支持**

本特性只有获得了License许可后才能获得该特性的服务，对应的License控制项为

| 适用NF | License控制项 |
| --- | --- |
| SMF | 81203788 LKV2IMRSDRSM01 HTTP IMR信令上报-USM |
| AMF | 81203789 LKV2IMRSDRAM01 HTTP IMR信令上报-UAM |

本特性无需配置，完成vProbe微服务安装、HTTP的内置探针插件包安装后即可使用。

## 与其他特性的交互（原文）

本特性不涉及与其他特性的交互。

## 对系统的影响

开启本功能会最多会增加40%的系统负荷，需提前预留约40%的硬件资源，请谨慎使用。

## 应用限制

- SMS over NAS、data over NAS的相关SBI接口的原始信令不做采集。
- HTTP IMR是SBI解密并且解压缩的原始信令，该IMR发送的时候，HTTP压缩字典不需要再发送。
- 业务地址和CHR单独上报业务地址共用。

## 原理概述

**组网**

SBI IMR信令上报功能主要包括三层架构体系，统一拉通数据采集和数据分析平台，组网架构如 [图1](#ZH-CN_TOPIC_0000001208194485__fig677091211430) 所示：

**图1** SBI IMR信令上报架构图

<br>

![](WSFD-220002 SBI IMR信令上报特性概述_08194485.assets/zh-cn_image_0000001468003793_2.png)

- 采集层：
  SDI信令指UNC的SBI接口上接收的HTTPS协议信令消息。UNC基于内置探针，实现SBI接口解密后的原始信令的采集。节省部署新探针所需要的硬件投资。软采通过IMR接口上报给单据服务器。
- 数据层：一般由全新云化架构的CloudUDN网元提供统一数据采集、汇聚、建模、存储、定位等基础功能，提供业务扩展能力，支持上层应用系统通过插件形式定制数据处理业务。
- 应用层：对数据层开放的数据进行分析和可视化呈现，包括如下3种功能系统：
    - SEQ Analyst
    - Netprobe
    - 第三方IT系统

**系统实现**

当SBI IMR消息上报开关打开时，UNC将接收/发送的SBI接口消息通过IMR事件上报接口发送给单据服务器。SBI IMR信令上报流程图如 [图2](#ZH-CN_TOPIC_0000001208194485__fig1940601671915) 。

**图2** SBI IMR信令上报流程图

![](WSFD-220002 SBI IMR信令上报特性概述_08194485.assets/zh-cn_image_0000002173112085_2.png)

流程描述如下：

1. UNC判断SBI IMR信令上报功能开启，启动信令上报功能。
2. 周边NF和UNC间HTTP信令交互。
3. UNC的SBIM微服务将收到/发送的HTTP消息发送给vProbe微服务。
4. vProbe微服务通过IMR接口，把解密后的HTTP信令封装，上报给单据服务器。
5. 单据服务器预处理，通过xDR接口输出北向数据至第三方服务器等。
  > **说明**
  > UNC 20.10.0 及后续版本UCF将不再转发IMR数据，vProbe服务用于承接IMR数据转发功能。
