`WSFD-011503` · protocol · required

## 适用NF（原文）

MME

## 定义

支持基于终端能力接入管理，是指 UNC 针对支持DCNR（Dual connectivity with New Radio）能力的终端，如果在HSS上的签约信息指示该终端签约支持NSA，则 UNC 允许此终端使用DCNR，否则 UNC 将此终端当成4G终端接入处理。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 允许支持DCNR能力的终端用户接入NSA网络，并通过4G基站（eNodeB）和5G基站（gNodeB ）接入EPC网络，实现NSA双连接，保证能够为用户提供高带宽业务。 |
| 用户 | 享受高带宽无线接入业务。 |

## 应用场景

该特性适用于NSA Option3/3a/3x组网场景。

## 原理概述

本节包括以下内容:

**DCNR简介**

DCNR（Dual connectivity with New Radio）即终端可以通过eNodeB和gNodeB接入EPC网络，实现NSA双连接。

eNodeB和gNodeB共用4G核心网（EPC），eNodeB为主站，gNodeB为从站，控制面信令走4G通道至EPC。

沿用4G核心网（EPC），以4G作为控制面的锚点，采用LTE与5G NR双连接的方式，利用现有的LTE网络部署5G。

共建模式在投资、用户体验、可运维性方面具有明显优势，是一种分组域核心网建设的理想模式。

支持基于终端能力接入管理是实施NSA部署的前提，DCNR组网形式如 [图1](#ZH-CN_CONCEPT_0193270616__fig1636411162612) 所示。

**NSA接入**

NSA接入流程如 [图2](#ZH-CN_CONCEPT_0193270616__fig76468922711) 所示。

**图2** NSA接入流程

<br>

![](特性概述_93270616.assets/zh-cn_image_0231429012_2.png)

1. UE新建RRC连接，通过Attach Request消息（携带UE Network Capability信息）首次接入eNodeB。
  消息中UE Network Capability信元新增DCNR指示位，DCNR指示位指示该终端的DCNR能力。
2. eNodeB向MME发送Initial UE Message（携带UE Network Capability信息），MME根据UE network capability信元确认UE是否支持DCNR。
3. MME上的 [**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 和 [**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md) 的 “SF2” 配置确定Update Location Request消息中的Supported-Features信元是否携带NR as Secondary RAT比特位给HSS，Features of Feature-List-ID 2中NR as Secondary RAT信元比特位指示MME的DCNR能力。
4. 如果用户签约了ARD（Access-Restriction-Data），则HSS在Update Location Answer消息中携带ARD，用于指示该用户NSA的签约情况。
5. 发送Create Session Request消息，MME根据 [**ADD GTPCV2CMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C V2协议兼容性/增加GTP-C V2协议兼容性(ADD GTPCV2CMPT)_26305734.md) 命令的 “CRTSESREQ” 配置，决定消息中是否新增UP Function Selection Indication Flags信元。
  该信元的DCNR指示位置为1，指示S-GW/P-GW支持NSA接入。
6. S-GW/P-GW给MME发送Create Session Response。
7. MME给eNodeB的Initial Context Setup Request消息（该消息中还包含了Attach Accept消息）中，Handover Restriction List信元中增加NR Restrict in EPS as Secondary RAT标识位，指示eNodeB是否允许为该用户创建NR承载。Handover Restriction List中携带NR Restrict in EPS as Secondary RAT=0，表示不允许eNodeB为该用户创建NR承载。
8. Attach Accept消息中，EPS network feature support information中增加指示位RestrictDCNR，指示终端网络侧是否允许该用户NR接入。RestrictDCNR=1 表示不允许该用户NR接入，RestrictDCNR=0 表示允许该用户NR接入。
9. eNodeB发送Initial Context Setup Response消息给MME。
10. UE发送Direct Transfer消息给eNodeB，该消息中还包含了Attach Complete消息。
11. eNodeB把上一步得到的Attach Complete包含在一个S1-MME控制信息里转发给MME。
12. MME发送Modify Bearer Request消息给S-GW。
13. S-GW向MME发送Modify Bearer Response消息，表明它已经收到eNodeB的TEID ，这样S-GW就能发送缓存的下行数据包了。

**NSA双连接**

UNC 针对支持DCNR（Dual connectivity with New Radio）能力的终端，如果在HSS上的签约信息指示该终端签约支持NSA，则 UNC 允许此终端使用DCNR，否则 UNC 将此终端当成4G终端接入处理。

NSA双连接流程如 [图3](#ZH-CN_CONCEPT_0193270616__fig75641358162710) 所示。

**图3** NSA双连接流程

<br>

![](特性概述_93270616.assets/zh-cn_image_0231429207_2.png)

1. eNodeB根据当前用户业务类型、空口状态等信息判断需要为用户新建NR承载后，向MME发送E-RAB Modification Indication消息将需要分流的承载IP地址指向gNodeB。
2. MME向SGW-C发送Modify Bearer Request消息（携带gNodeB TEID和IP地址），通知SGW-C将用户面传输路径从eNodeB切换到gNodeB。
3. SGW-C更新对应承载的IP地址信息后通过Modify Bearer Response消息将更新结果返回给MME。
4. MME通过将E-RAB Modification Confirm消息更新结果通知给eNodeB。
