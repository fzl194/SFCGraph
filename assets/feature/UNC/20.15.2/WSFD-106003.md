## 适用NF（原文）

AMF

## 定义

用户接入控制功能是指 UNC 支持根据用户签约或者本地配置的移动性接入限制信息判断是否允许用户注册到5G核心网的功能。 UNC 按以下两种策略实施用户接入控制：

- 根据用户接入和移动签约数据/AMF本地配置中的“RAT限制（RAT restriction）”信息控制用户的无线接入方式。
- 根据用户接入和移动签约数据/AMF本地配置中的“核心网类型限制（Core Network type restriction）”信息控制用户的核心网接入方式。

- 运营商希望根据用户签约/AMF本地配置的“RAT限制（RAT restriction）”信息控制用户的无线接入方式。
- 运营商希望根据用户签约/AMF本地配置的“核心网类型限制（Core Network type restriction）”信息控制用户的核心网接入方式。

## 应用限制

- 用户接入控制功能对于紧急注册不生效，如果是紧急注册，则跳过用户接入控制检查。
- 签约与本地配置的移动性接入限制信息优先级默认以本地配置为准。
- AMF本地配置的RAT限制信息、核心网限制信息发生变化时，仅针对新注册用户生效，对于已注册到本AMF的存量用户不生效。

## 原理概述

****5G移动性限制背景介绍****

网络的移动性管理既要保证UE可达和数据传输连续，又要防止UE接入某些受限区域或者请求不该使用的服务。移动性限制就是指3GPP内UE的接入访问受限、移动受限等。

5G协议中则明确定义了移动性接入限制信息有RAT限制、核心网限制、禁止区域、服务区限制四种类型：

- **RAT限制（RAT restriction ）**
- **核心网限制（Core Network type restriction）**
- **禁止区域****（****Forbidden Area****）**
- **服务区限制（Service Area Restriction）**

5G系统中的移动性限制是UE、基站和核心网相互协同完成的，作用如下：

1. UE在某区域尝试接入如果由于限制被核心网拒绝，UE支持根据注册被拒的原因值在本地生成和维护禁止区域列表，并且在一定时长内禁止尝试在该区域内注册到核心网。
2. 基站从核心网拿到用户移动性接入限制信息后，在切换流程选择目标网络时会过滤受限的RAT、跟踪区等。
3. 核心网作为移动性限制的决策中心，综合运营商的各种策略（如签约数据或AMF本地配置移动性接入限制信息）执行接入限制功能。

UNC 作为移动性限制的决策中心，支持实现移动性限制，并且根据不同类型的 **移动性接入限制信息** 实现不同的限制功能。包括：

- 根据**RAT限制（RAT restriction ）**和**核心网限制（Core Network type restriction）**实现用户接入控制功能，如本特性。
- 根据**禁止区域****（****Forbidden Area****）**实现区域漫游限制功能，如[WSFD-105003 区域漫游限制（适用于AMF）](../../../漫游管理功能/WSFD-105003 区域漫游限制/WSFD-105003 区域漫游限制（适用于AMF）_48669740.md)。
- 根据**服务区限制（Service Area Restriction）**实现服务区域限制功能。

本特性主要介绍用户接入控制功能。

**接入控制流程涉及的移动性接入限制信息介绍**

- **RAT限制（RAT restriction ）**
  RAT限制（RAT restriction ）表示限制的3GPP内的无线接入方式，该信息中指示了受限的RAT（Radio Access Technologies）类型列表，用于表明是否限制用户接入NG-RAN或E-UTRAN。对应于2/3/4G签约数据中的ARD（Access Restriction Data），在互操作流程中可以部分转换。
  RAT限制信息获取方式：
    - AMF可以从用户签约数据获取RAT限制信息，也可以在本地配置。
    - 在注册流程结束后，核心网把RAT限制信息下发给基站，基站在发起切换流程时可基于该信息判断是否允许用户切换到目标RAT。

- **核心网限制（Core Network type restriction）**
  核心网限制（Core Network type restriction）表示限制的核心网接入方式，该信息中指示了受限的核心网类型列表，用于表明是否限制用户接入5GC或者EPC。
  AMF可以从用户签约数据获取核心网限制信息，也可以在本地配置。

**用户接入控制判断流程**

在注册、移动性注册和切换等业务流程中，当用户接入，AMF根据UE签约或AMF本地配置的 **RAT限制信息（RAT restriction ）** 和 **核心网限制信息（Core Network type restriction）** 信息，以及UE当前使用的无线接入类型来决策是否允许用户接入。

用户接入控制判断流程如 [图1](#ZH-CN_CONCEPT_0248552635__fig1) 所示。

> **说明**
> 本特性主要介绍用户接入控制流程。如果通过了接入控制检查，再根据签约或本地配置的 **禁止区域** **（** **Forbidden Area** **）** 以及UE当前驻留小区所归属的TA检查是否允许用户在当前区域接入，参见 [WSFD-105003 区域漫游限制（适用于AMF）](../../../漫游管理功能/WSFD-105003 区域漫游限制/WSFD-105003 区域漫游限制（适用于AMF）_48669740.md) 。
