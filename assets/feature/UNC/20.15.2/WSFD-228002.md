## 定义

某行业的业务服务部署在行业总部所在的区域内，希望其职员可以通过终端，在归属地、省内其他区域、省外区域都能够随时访问行业内部业务和Internet。

5G Core通过通用DNN的漫游特性，为终端用户签约通用DNN和专用DNN，通过专用DNN访问行业内部业务，通过通用DNN访问Internet。可以满足用户在归属地同时访问行业内部业务和Internet，并且在跨地市、跨省漫游场景下，仍然能够保持业务不中断。

在本文的之后章节，将支持基于通用DNN和专用DNN的漫游分流用户统一称为智能分流用户，将支持基于双DNN的漫游分流的SMF、UPF统一称为智能分流SMF、智能分流UPF。

> **说明**
> **漫游** ：本特性中所描述的“漫游”是指用户在本运营商的网络覆盖的范围内的跨区域漫游，而非跨运营商网络之间的国际漫游或异网漫游。
>
> **双域专网** ：支持用户同时访问运营商公网业务和园区专网业务的网络，双域即运营商公网域和园区专网域，提供的业务下面将描述为“双域业务”。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 支持终端用户在本地市、省内跨地市、跨省漫游场景下同时访问行业内部业务和Internet，提升用户漫游场景下的业务体验。 |
| 用户 | - 满足用户在出差场景下，随时随地的访问行业内部业务和Internet的诉求。<br>- 用户在跨地市和跨省移动时，则可以一直访问行业内部业务。 |

## 应用场景

相较于UL CL分流（ [原理概述](#ZH-CN_TOPIC_0000001670339689__section141747110546) 有详细UL CL分流与通用DNN漫游的差异），在智能分流场景下，终端用户不仅能在本地市访问行业内部业务，当用户漫游到省内其他地市或其他省份时，也可以访问行业内部业务。

但是对于用户来说，经常存在出差到外地且需要访问行业内部业务的场景。通过在终端安装VPN软件等方式虽然支持通过Internet访问行业内部业务，但是存在操作繁琐、业务不流畅等问题。

部署本特性后，可满足终端用户在本地市、省内跨地市、跨省漫游场景下的同时访问行业内部业务和Internet。

## 应用限制

- SA国际漫游场景下不支持本特性。
- 用户4G接入，通用会话SGW-C、PGW-C分离时，不支持专网会话选择到SGW-C所在网元设备。
- 单用户最大支持300个园区域名（按每域名平均长度32字节计算），单用户最大支持200个园区IP地址段。
- 漫游地部署的非特定人群的本地业务（如国家剧院）和通用DNN漫游分流方案的归属地园区业务互斥。
- 本特性支持根据IP地址和域名进行分流，不支持IP地址+端口号、域名+端口号进行分流。
- 园区内专网UPF分流功能仅适用于用户在园区内5G接入场景，用户在园区内4G接入时不适用。
- 当前开启园区内分流功能后（通过**ADD MULDNNULCLMODE**控制），智能分流SMF外部消息交互量会增加，会造成CPU上涨，影响程度需要基于运营商话务模型具体测算，部署前请联系华为技术支持详细评估。
- 如果园区为IPv6组网并且使用域名分流，要求园区DNS服务器为IPv4v6双栈，能解析IPv4的AAAA记录请求，同时专网UPF上需要配置IPv4 IPv6的双栈UE地址池。

## 原理概述

3GPP标准 [ULCL分流方案](../../../../业务专题/5G Core ULCL分流解决方案/概述_95053859.md) 下，如 [图1](#ZH-CN_TOPIC_0000001670339689__fig41891415432) （左侧）所示，用户在园区/本地访问园区业务时，SMF基于用户当前位置信息，在用户PDU会话中插入ULCL UPF，并向ULCL UPF下发预定义规则的分流策略，用户后续访问双域业务的数据报文均会经过ULCL UPF，匹配到分流规则的报文被分发到园区DN，匹配不到分流规则的报文则分流至主锚点UPF。当用户在本运营商网络覆盖范围内跨区域漫游，重新激活PDU会话时超出园区ULCL UPF的覆盖范围，无法访问园区业务。另外3GPP标准ULCL分流也不支持用户4G接入。

**图1** 标准ULCL分流和通用DNN漫游分流对比示意

<br>

![](特性概述_70339689.assets/zh-cn_image_0000001671764557_2.png "点击放大")

相较于协议标准的ULCL分流方案， **通用DNN漫游分流方案** 做了如下增强：

- 不仅支持园区用户在本地同时访问双域业务，还支持用户在本运营商网络覆盖范围内跨区域漫游场景下同时访问运营商公网业务和位于归属地的园区业务。
- 不仅支持用户5G接入场景下对双域业务的访问，还支持用户4G接入场景下对双域业务的访问。

如 [图1](#ZH-CN_TOPIC_0000001670339689__fig41891415432) （右侧）所示， **通用DNN漫游分流方案** 中引入了智能分流SMF/PGW-C、智能分流UPF、专网SMF以及专网UPF，运营商在归属地HSS/UDM侧为园区用户签约运营商通用数据DNN以及园区专用DNN，归属地PCF侧签约园区动态分流规则，后续通过如下流程支撑用户对双域业务的访问：

- 用户在拜访地接入时，拜访地MME/AMF从归属地HSS/UDM获取用户签约的通用DNN和专用DNN，并基于通用DNN选择智能分流SMF/PGW-C，智能分流SMF/PGW-C基于通用DNN选择智能分流UPF，并从归属地PCF获取园区业务的动态分流策略并下发给智能分流UPF，最终创建用户访问公网的会话，也称为**公网会话**。
- 智能分流SMF/PGW-C模仿AMF基于用户签约的园区专用DNN选择位于归属地的专网SMF，专网SMF再基于园区专用DNN选择专网UPF，并从PCF获取园区PCC策略下发给专网UPF，创建用户访问园区专网业务的会话，也称为**专网会话**。用户在拜访地的公网会话可以是4G会话或5G会话，经过智能分流SMF/PGW-C处理后，网络侧为用户创建访问园区的**专网会话**均为5G PDU会话。

这样用户基于 **双DNN** 访问 **双域业务** 的 **双会话** 就建立起来了，后续用户数据报文经过智能分流UPF汇聚后分流，匹配到分流规则的报文被分发到归属地专网UPF，匹配不到分流规则的报文则分流至公网。 [图1](#ZH-CN_TOPIC_0000001670339689__fig41891415432) （右侧）所示各网元核心功能详细请参见 [表1](#ZH-CN_TOPIC_0000001670339689__table4192050104310) 。

*表1 概念说明*

| 概念 | 说明 |
| --- | --- |
| 智能分流SMF | 位于用户当前所在地（拜访地），<br>承担如下功能：<br>- 从归属地PCF获取园区的动态分流策略并下发给智能分流UPF用于分流园区用户访问公网和园区专网的流量。<br>- 基于通用DNN触发创建用户访问公网业务的会话。<br>- 基于园区签约的专用DNN从NRF发现专网SMF，触发创建用户访问园区的专网会话。<br>- 将智能分流UPF分配的UE IP地址发送给专网SMF。 |
| 智能分流UPF | 位于用户当前所在地（拜访地），承担如下功能：<br>- 根据智能分流SMF下发的动态分流策略分流园区用户访问公网和园区专网的流量，将访问公网的流量在拜访地卸载，将访问园区的流量通过N9口转给归属地专网UPF。<br>- 分配公网会话的UE IP地址。 |
| 专网SMF | 位于用户归属地，根据归属地<br>园区签约的专用<br>DNN<br>选择专网UPF并建立N4会话，将公网会话的UE IP地址下发给专网UPF。 |
