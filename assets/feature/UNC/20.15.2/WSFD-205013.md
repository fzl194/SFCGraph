# 网络拥塞的实时检测

`WSFD-205013` · enhanced · required

## 所属目录

- [WSFD-205000](feature/UNC/20.15.2/WSFD-205000.md)

## 适用性

- 适用NF：MME、SGW-C、PGW-C、AMF、SMF
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=M; S/PGW-C(4G)=M; NB-IoT(4G)=-; 5G MME(5G NSA)=M; 5G S/PGW-C(5G NSA)=M; AMF(5G SA)=M; SMF(5G SA)=M; NRF(5G SA)=-`

## 适用NF（原文）

MME、SGW-C、PGW-C、AMF、SMF

## 定义

网络拥塞的实时检测特性是指用户进行会话创建或跨小区移动时，SMF获取用户的位置信息，并将用户的位置信息通过N4接口发送给UPF，使UPF获取不同小区的上网用户数量，UPF将小区网络质量上报NWDAF进行可视化呈现，从而实时检测不同小区的网络拥塞程度。

## 客户价值

| **受益方** | **受益描述** |
| --- | --- |
| 运营商 | - 运营商可通过本特性探测拥塞小区，更加了解自身网络的拥塞情况。<br>- 运营商可通过本特性进一步进行网络优化，优化小区流量，提高网络效率。<br>- 本特性有助于提升运营商的网络体验竞争力，吸纳更多潜在用户。 |
| 用户 | 通过检测网络拥塞进行对应网络优化，有助于减少网络拥塞，提升用户的上网体验。 |

## 应用场景

在运营商网络流量持续增长的背景下，小区的网络拥塞现象加剧，用户的体验不断下降。运营商为缓解网络拥塞，提升用户体验，需开启本特性对无线侧网络拥塞情况进行实时检测。

| SGW-C | UNC 20.10.0及后续版本 | SGW-C通过S5/S8接口获取MME发送的用户位置信息ULI（User Location Information）以及基站ID，并透传给PGW-C。 |
| PGW-C | UNC 20.10.0及后续版本 | PGW-C获取到SGW-C发送的用户位置信息ULI（User Location Information）以及基站ID后通过N4接口发送给PGW-U。 |
| PGW-U | UDG 20.10.0及后续版本 | PGW-U支持对4G用户会话创建/修改消息中ULI信元进行解析、存储、刷新和上报NWDAF，并支持根据NWDAF下发的小区用户策略进行策略控制。 |

| SMF | UNC 20.10.0及后续版本 | SMF通过N11接口获取AMF发送的用户位置信息ULI（User Location Information）以及基站ID，并通过N4接口发送给UPF。 |
| UPF | UDG 20.10.0及后续版本 | UPF支持对5G用户会话创建/修改消息中ULI信元进行解析、存储、刷新和上报NWDAF，并支持根据NWDAF下发的小区用户策略进行策略控制。 |
| NWDAF | CloudUDN V100R023C10及后续版本 | - 提供网络流量测量和小区质量可视化界面。<br>- 提供质量标准自定义界面。<br>- 提供策略自定义界面和策略下发功能。<br>- 提供智能小区拥塞探测和策略下发功能。 |

## 原理概述

本特性4G组网图如 [图1](#ZH-CN_CONCEPT_0000001430162313__fig143413952217) 所示，5G组网图如 [图2](#ZH-CN_CONCEPT_0000001430162313__fig1945515718247) 所示。UNC产品涉及的NF与UPF、NWDAF配合完成拥塞网络优化功能，其中本特性实现网络拥塞的实时检测，即获取用户的位置信息并下发给UPF进行解析存储，并由UPF进一步将解析后的信息上报NWDAF进行可视化呈现及处理。

- 4G用户的网络拥塞实时检测：
    1. 当用户创建会话时，信令流程请参考[4G用户会话管理](../../../../../快速入门/协议与流程/EPC信令分析/EPC信令与协议分析/业务流程/会话管理_30779036.md)，本特性支持在**MME**发给**SGW-C**的**Create Session Request**消息及**PGW-C**发给**PGW-U**的**PFCP Session Establishment Request**消息中携带ULI信元用于标识用户位置信息。
    2. 当用户进行跨小区移动，用户修改会话时，信令流程请参考[4G用户会话管理](../../../../../快速入门/协议与流程/EPC信令分析/EPC信令与协议分析/业务流程/会话管理_30779036.md)，本特性支持在**MME**发给**SGW-C**的**Modify Bearer Request**消息及**PGW-C**发给**PGW-U**的**PFCP Session Modification Request**消息中携带ULI信元用于标识用户位置移动。
  实现4G用户的位置信息收集，进而实现网络拥塞实时检测。
- 5G用户的网络拥塞实时检测：
    1. 当用户创建会话时，信令流程请参考[PDU会话建立](../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/会话流程/PDU会话建立_97353800.md)，本特性支持在**AMF**发给**SMF**的**Nsmf_PDUSession_CreateSMContext Request**消息及**SMF**发给**UPF**的**PFCP Session Establishment Request**消息中携带ULI信元用于标识用户位置信息。
    2. 当用户进行跨小区移动，用户修改会话时，信令流程请参考[PDU会话修改](../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/会话流程/PDU会话修改（SMF发起）_47593601.md)，信令流程请参考PDU会话修改，本特性支持在**AMF**发给**SMF**的**Nsmf_PDUSession_UpdateSMContext Request**消息及**SMF**发给**UPF**的**PFCP Session Modification Request**消息中携带ULI信元用于标识用户位置移动。
  实现5G用户的位置信息收集，进而实现网络拥塞实时检测。
