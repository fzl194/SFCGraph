# 智能信令风暴平滑处理

`WSFD-010804` · enhanced · required

## 所属目录

- [WSFD-010800](feature/UNC/20.15.2/WSFD-010800.md)

## 适用性

- 适用NF：MME、SGW-C
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=M; S/PGW-C(4G)=M; NB-IoT(4G)=M; 5G MME(5G NSA)=M; 5G S/PGW-C(5G NSA)=M; AMF(5G SA)=-; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

MME、SGW-C

## 定义

LTE网络中，当MME和SGW-C间S11接口故障导致流程失败时，MME会批量分离用户，引发短时间内大批量UE附着，进而引发信令风暴。引入本特性后，当部分流程的失败速率超过一定门限时，MME以较低的速率延迟分离受影响的UE，从而降低引发网络信令风暴的风险。

S11接口故障或MME过载导致DDN无响应时，如果SGW-C立即删除用户承载，大量用户被去激活后会反复尝试重新激活，信令急剧增加可能导致LTE网络拥塞和设备过载。SGW-C支持智能信令风暴平滑处理，避免S11接口故障或MME过载情况下可能引发的信令风暴。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 避免由于S11接口故障而引发网络风暴，减少网络拥塞或过载发生的可能性。<br>- 使UE在S11接口故障场景下优先驻留在LTE网络，避免立即选择GSM/UMTS网络接入，对GSM/UMTS网络造成信令冲击。<br>- 提高S11接口故障或MME过载情况下的整网可靠性。 |
| 用户 | 用户不感知该特性。 |

## 应用场景

本特性适用于如下两种场景：

- S11接口故障场景（非SGW-C复位导致）。该场景下，MME由于流程失败，立即分离大量UE，从而导致短时间内批量UE附着，增加信令风暴的风险。为降低该场景下信令风暴风险，建议配置本特性。
- S11接口故障或MME过载导致DDN无响应。该场景下，如果SGW-C立即删除用户承载，大量用户被去激活后会反复尝试重新激活，信令急剧增加可能导致LTE网络拥塞和设备过载。为降低该场景下信令风暴风险，建议配置本特性。

## 对系统的影响

- 部署本特性后，受S11接口故障影响的UE可能被延迟分离。进入延迟分离状态的UE被分离前，其VoLTE语音和数据业务均不可用。在此期间，若触发了UE的其他流程（且MME与SGW-C交互成功）或成功重选了SGW-C，则可提前恢复受影响业务。
- MME通过在GTP-C路径上向SGW-C发送Echo Request消息感知SGW-C是否可达。为快速感知MME与SGW-C之间链路状态的变化，建议在UNC上将消息发送间隔配置为最小值（60秒）。
- S11接口故障或MME过载导致DDN消息无响应时，SGW-C将删除大量用户的行为均匀的分散到一个时间段内完成，以避免出现信令急剧增加导致网络拥塞和设备过载的情况。
    1. SGW-C支持通过 **[SET DDNTIMEOUTCTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/流控管理/DDN消息无响应处理策略/设置DDN定时器超时参数（SET DDNTIMEOUTCTRL）_46314493.md)** 命令开启DDN消息无响应时延时删除功能。
    2. 当SGW-C判断DDN消息无响应时，SGW-C给用户设置延时删除定时器，并为用户分配延时删除定时器的有效时间。
      通过 **[SET DDNTIMEOUTCTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/流控管理/DDN消息无响应处理策略/设置DDN定时器超时参数（SET DDNTIMEOUTCTRL）_46314493.md)** 的参数MINDELAYTIMER和MAXDELAYTIMER配置SGW-C的延时删除用户功能的有效时间范围。为每个用户在延时删除用户功能的有效时间范围随机选取一个时间值，做为用户的延时删除定时器的有效时间。
      例如：SGW-C的延时删除用户功能的有效时间范围为30-60分钟。用户1的延时删除定时器T1=30分钟、用户2的延时删除定时器T2=32分钟、用户3的延时删除定时器T3=40分钟、…、用户n的延时删除定时器Tn=60分钟。T1、T2、T3、T4、…、Tn的值是SGW-C从延时删除用户功能的有效时间段中随机分配的，不同用户的定时器时间可以重复。
    3. 当用户在延时删除定时器超时前，S11接口出现信令交互，SGW-C认为S11接口故障或MME过载等故障现象恢复正常，SGW-C会删除用户的延时删除定时器。
      S11接口的信令交互消息包括：Create Session Request、Create Bearer Response、Bearer Resource Command、Modify Bearer Request、Delete Session Request、Delete Bearer Response、Downlink Data Notification Acknowledge、Downlink Data Notification Failure Indication、Delete Indirect Data Forwarding Tunnel Request、Modify Bearer Command、Update Bearer Response、Delete Bearer Command、Create Indirect Data Forwarding Tunnel Request、Release Access Bearers Request、Change Notification Request、Suspend Notification、Resume Notification、Trace Session Activation。
    4. 当用户的延时删除定时器超时后，SGW-C发送Update Bearer Request命令进行探测，如果对端无响应，则将用户加入删除队列，并删除用户。
      用户被加入删除队列以后，将被SGW-C删除。即使S11接口故障或MME过载等故障现象恢复正常，用户依然会被删除。
  为了便于观察相关信息，SGW-C支持通过 **[DSP DDNTIMEOUTCNT](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/流控管理/DDN消息无响应处理策略/显示DDN超时延时删除功能运行信息记录（DSP DDNTIMEOUTCNT）_46314491.md)** 命令查询处于延时删除状态的用户数、删除队列中的用户数、已延时删除的用户数。

## 应用限制

本特性要求对端SGW-C为华为设备。

## 原理概述

**特性概述**

为降低S11接口故障场景（非SGW-C复位导致）中信令风暴的风险，本特性针对Service Request，Intra TAU/Combined TAU without S-GW Change，Intra Handover without S-GW Change和Intra X2 Path Switch without S-GW Change这四个流程（以下简称为高频流程），引入信令平滑处理机制。

当MME统计到高频流程的失败速率超过一定门限时，MME对这些流程中的UE进行延迟分离处理，此时UE进入延迟分离状态。MME并不立即分离受影响的UE，而是在延迟一段时间后，再以较低的速率分离这些UE，以达到信令平滑处理的目的。对于进入延迟分离状态的UE，MME只是抑制其当前流程，但仍然可以正常处理UE的后续信令流程。

> **说明**
> 本特性统计因Modify Bearer Request（MBR）发送失败或响应超时的高频流程的失败速率。

**系统实现**

**信令抑制启控**

当MME统计到高频流程的失败速率（单位：次/秒）超过启控门限时，系统启动信令抑制功能，如 [图2](#ZH-CN_CONCEPT_0162734662__fig02) 所示。

- 该门限由[**DSP SIGSUPPRESS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/信令抑制管理/S11接口故障场景信令抑制管理/显示S11接口信令风暴抑制功能的控制与统计信息(DSP SIGSUPPRESS)_72225847.md)命令中的“抑制分离速率（个/秒）”参数决定，启控门限=“抑制分离速率（个/秒）”*80%。
- “抑制分离速率（个/秒）”由[**DSP SIGSUPPRESS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/信令抑制管理/S11接口故障场景信令抑制管理/显示S11接口信令风暴抑制功能的控制与统计信息(DSP SIGSUPPRESS)_72225847.md)命令中“抑制用户数控制值”除以[**SET SIGSUPPRESS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/信令抑制管理/S11接口故障场景信令抑制管理/设置S11接口信令风暴抑制功能参数(SET SIGSUPPRESS)_72345767.md)命令中“抑制时长（分）”计算得出。其中，“抑制用户数控制值”为MME系统规划的4G用户数或系统实时的4G用户数，“抑制时长（分）”为运营商可接受的业务中断时长。

> **说明**
> WSFD-110755-3 S-GW/P-GW故障下的业务恢复特性开启时，该特性中S11接口故障（非GW复位导致）触发的UE分离的速率（包括因SGW-C重选失败导致的UE分离的速率）会影响本特性的启控。此时的启控条件为：MME统计到的高频流程的失败速率与WSFD-110755-3 S-GW/P-GW故障下的业务恢复特性中的UE分离速率之和超过启控门限。

如 [图2 信令抑制启控示意图](#ZH-CN_CONCEPT_0162734662__fig02) 所示，信令抑制启控分为两种状态：

- 状态1：高频流程的失败速率超过启控门限的持续时间不超过5秒时，系统处于状态1。此时通过[**DSP SIGSUPPRESS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/信令抑制管理/S11接口故障场景信令抑制管理/显示S11接口信令风暴抑制功能的控制与统计信息(DSP SIGSUPPRESS)_72225847.md)命令查询到的“抑制功能启控状态”为“未启控状态”。<br>
    - MME对失败速率未超过上述门限的高频流程中的UE进行正常处理，直接分离受影响UE。
    - 对失败速率超过上述门限的高频流程中的UE进行延迟分离处理。
- 状态2：高频流程失败速率超过门限的持续时长超过5秒后，系统进入状态2。<br>
    - MME对所有失败的高频流程中的UE进行延迟分离处理。此时上报告警[ALM-80679 S11接口故障信令风暴抑制启控](../../../../../网络运维/故障处理/UNC告警处理/业务告警/SGSN&MME/ALM-80679 S11接口故障信令风暴抑制启控_89656092.md)。
  > **说明**
  > - 上述5秒判决期内的处理与状态1中的处理相同。
  > - [图2](#ZH-CN_CONCEPT_0162734662__fig02)以MME先进入状态1后进入状态2为例。实际网络环境中，当满足进入条件时，MME可直接进入状态2。

**延迟分离处理**

UE进入延迟分离状态后，MME将在 “抑制时长（分）” 内，基于 “抑制分离速率（个/秒）” 完成对UE的分离。

如果MME收到来自SGW-C的响应消息或者UE通过CSFB回落到GSM/UMTS网络，则清除该UE的延迟分离状态，不对UE进行延迟分离处理。UE的延迟分离状态可通过 [**DSP COMMMCTX**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/融合接入业务管理/融合用户数据库管理/显示移动性管理上下文的相关信息（DSP COMMMCTX）_58365337.md) 命令中的 “用户是否被抑制” 参数查询。

> **说明**
> UE进入延迟分离状态后，
>
> - 如果MME收到SGW-C的Create Bearer Request消息，将不对消息进行处理，且不清除该UE的延迟分离状态。如果MME收到SGW-C的Update Bearer Request或者Delete Bearer Request消息，则对消息进行正常处理，此时可能会触发Service Request流程。如果Service Request流程中MBR交互成功，则清除该UE的延迟分离状态；否则，不清除UE的延迟分离状态。
> - 如果MME收到SGW-C的Downlink Data Notification消息，则判断当前S11接口的GTP-C路径状态。如果路径正常，则对消息进行正常处理，触发UE发起Service Request流程；如果路径故障，则丢弃该消息。

**信令抑制解控**

MME进入启控状态2 后，当同时满足如下3个条件时，MME停止对失败的高频流程中的UE进行抑制，即停止对这些UE进行延迟分离处理。同时恢复告警 [ALM-80679 S11接口故障信令风暴抑制启控](../../../../../网络运维/故障处理/UNC告警处理/业务告警/SGSN&MME/ALM-80679 S11接口故障信令风暴抑制启控_89656092.md) 。

- 延迟分离速率降低：90秒内的UE的实际分离速率均低于[**DSP SIGSUPPRESS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/信令抑制管理/S11接口故障场景信令抑制管理/显示S11接口信令风暴抑制功能的控制与统计信息(DSP SIGSUPPRESS)_72225847.md)命令中“抑制分离速率（个/秒）”的40%。
- 处于延迟分离状态的用户数减少：[**DSP SIGSUPPRESS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/信令抑制管理/S11接口故障场景信令抑制管理/显示S11接口信令风暴抑制功能的控制与统计信息(DSP SIGSUPPRESS)_72225847.md)命令中“抑制用户数”低于“抑制用户数控制值”的5%。
- 用户进入延迟分离状态的速率降低：最近90秒新进入延迟分离状态的用户的速率都低于[**DSP SIGSUPPRESS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/信令抑制管理/S11接口故障场景信令抑制管理/显示S11接口信令风暴抑制功能的控制与统计信息(DSP SIGSUPPRESS)_72225847.md)命令中“抑制分离速率（个/秒）”的50%。
