# 故障管理

`NPFD-010003` · operations · ops_only

## 所属目录

- [NPFD-010000](feature/UNC/20.15.2/NPFD-010000.md)

## 适用性

- 适用NF：UNC
- 代际支持矩阵：`SGSN(2.5&3G)=M; GGSN-C(2.5&3G)=M; MME(4G)=M; S/PGW-C(4G)=M; NB-IoT(4G)=M; 5G MME(5G NSA)=M; 5G S/PGW-C(5G NSA)=M; AMF(5G SA)=M; SMF(5G SA)=M; NRF(5G SA)=M`

## 定义

故障管理功能主要对设备和网络的异常运行情况进行实时监视，完成对故障信息和设备运行情况的监视、报告和存储。而且通过告警的显示可以使用户能尽快采取相应措施，对故障进行隔离和校正，恢复受影响的业务。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 利用设备活动告警列表、系统内部的可靠性保证机制，确保告警及时、可靠地呈现给用户，以支持快速、准确的故障定位和诊断。<br>- 支持告警相关性分析、告警屏蔽、告警抑制、告警持续性分析功能，呈现给用户最直观有效的故障告警信息。<br>- 降低故障管理系统的负荷。<br>- 支持高效的故障定位和诊断。 |
| 用户 | 用户不感知该特性。 |

## 应用场景

系统运行过程中，系统运行相关问题会及时上报告警。

## 对系统的影响

如果告警频繁上报，告警数量过多的情况下会占用大量的系统资源，降低系统性能。

## 应用限制

本特性无应用限制。

## 原理概述

**告警级别**

系统根据告警影响的范围和程度，将告警分为如下四个级别。操作人员可根据告警级别，优先处理紧急重要的告警。

- ![](NPFD-010003 故障管理_18283563.assets/zh-cn_image_0193404426_2.png)紧急告警
  此类级别的故障影响到系统提供的服务，需要紧急处理。如某资源完全不可用，需立即采取措施，进行恢复。
- ![](NPFD-010003 故障管理_18283563.assets/zh-cn_image_0193404424_2.png)重要告警
  此类级别的故障影响到服务质量，需要立即处理。如某资源服务质量下降，需对其进行还原，恢复全部能力。
- ![](NPFD-010003 故障管理_18283563.assets/zh-cn_image_0193404423_2.png)次要告警
  此类级别的故障还未影响到服务质量，但为了避免更严重的故障，需要在适当时候进行处理或进一步观察。
- ![](NPFD-010003 故障管理_18283563.assets/zh-cn_image_0193404422_2.png)提示告警
  此类级别的故障指示会有潜在的错误影响到提供的服务，需要根据不同的错误进行处理。

**活动告警**

用户可以在 OM Portal 上查看最近1000条以内的活动告警，并根据实际情况对告警执行相应操作。

*表1 高级搜索参数说明*

| 参数名 | 参数含义 | 参数取值 |
| --- | --- | --- |
| 应用名称 | 产生告警的应用的名称。 | 系统已安装应用的名称。<br>默认值：全部 |
| 告警ID | 用于确定一类告警，代表一种特定的故障，是告警项的唯一索引，是网管系统内告警配置表的主关键字。 | 默认值：无 |
| 告警名称 | 告警的名称。 | 默认值：无 |
| 定位信息 | 表明发生该条告警的物理或者业务对象，其中包含用户能够定位告警发生对象的所有必须元素，用户根据定位信息就能够唯一确定该条告警的发生对象。 | 默认值：无 |
| 告警级别 | 用于标识故障对业务的影响程度。 | - 紧急：此类级别的故障影响到系统提供的服务，需要紧急处理。如某资源完全不可用，需立即采取措施，进行恢复。<br>- 重要：此类级别的故障影响到服务质量，需要立即处理。如某资源服务质量下降，需对其进行还原，恢复全部能力。<br>- 次要：此类级别的故障还未影响到服务质量，但为了避免更严重的故障，需要在适当时候进行处理或进一步观察。<br>- 提示：此类级别的故障指示会有潜在的错误影响到提供的服务，需要根据不同的错误进行处理。<br>- 全选<br>默认值：全选 |
| 发生时间 | 该告警产生的时间。 | - 最近1天<br>- 最近3天<br>- 最近7天<br>- 最近10天<br>- 自定义<br>- 所有时段<br>默认值：所有时段 |

**历史告警**
