# 指定区域用户位置上报(PRA)-4G

`WSFD-106010-1` · enhanced · required

## 所属目录

- [WSFD-106010](feature/UNC/20.15.2/WSFD-106010.md)

## 适用性

- 适用NF：MME、SGW-C、PGW-C
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=M; S/PGW-C(4G)=-; NB-IoT(4G)=-; 5G MME(5G NSA)=M; 5G S/PGW-C(5G NSA)=-; AMF(5G SA)=M; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

MME、SGW-C、PGW-C

## 定义

基于指定区域的策略控制特性是指基于运营商订阅的PRA（Presence Report Area）区域，当用户进入或离开这些区域时， UNC 获得用户当前位置信息并向PCRF/PCF上报，PCRF/PCF根据用户位置状态进行策略决策并发起策略更新，执行不同的服务策略（如特殊计费、套餐、信息推送等），满足运营商基于位置信息的差异化和定制化需求，用户通过签约享受相应服务。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 运营商通过部署基于指定区域的策略控制特性可以实时掌握用户的位置状态，对用户群进行细分，从而有针对性的向用户进行广告推送、信息推广等增值业务，提升区域内用户体验。<br>- 运营商通过实时掌握用户位置状态，制定精确的位置套餐，从而提升用户粘性，增加运营收益。<br>- 相比其他实时位置特性，部署基于指定区域的策略控制特性对运营商现有网络的信令冲击小，可节约运营成本。 |
| 用户 | - 针对指定区域，用户可享受更精确的广告推送、信息推广等增值业务，比如酒店用户可获取该酒店的优惠策略。<br>- 针对指定区域，用户可享用更合理的套餐业务，比如校园用户进入园区内可享受资费更便宜的套餐。 |

## 应用场景

基于指定区域的策略控制特性支持MME在用户进入或离开指定区域时向SGW-C/PGW-C上报位置信息，PCRF/PCF根据此位置信息更新并向SGW-C/PGW-C下发对应的策略，典型应用场景如下。商场、校园、体育场、地铁沿线等需要向用户提供基于位置的计费优惠、套餐服务时，运营商会部署基于指定区域的策略控制。

- 校园套餐
  校园策略针对用户生效，进入校园区域后，用户资费更优惠，从而能吸引更多的客户群，提升运营商收益。
- 商场套餐
  商场套餐针对进入指定商场或酒店区域的所有用户或指定用户组生效，进入指定区域后，用户可享用更优惠的资费套餐或流量统付业务，从而提升商场或酒店等第三方收益。
- 地铁区域业务推送
  针对进入指定地铁区域的用户进行广告投放，从而使得用户体验更精准。

## 应用限制

- 本特性仅对本地用户生效，对跨运营商的漫游用户不生效。
- 离线计费话单和在线计费消息中不携带PRA相关信元。
- 若网络中部分MME不支持本特性或本特性未开启，当用户进行Inter MME位置移动，会造成PRA订阅信息的丢失，从而影响PCRF等上游网元根据PRA区域开展的各类业务的可靠性。
- 本特性只支持核心网预定义的PRA，在MME上配置的PRA是核心网预定义PRA。在网络规划时由核心网预定义PRA ID以及与PRA ID相关联的PRA区域列表，并配置到MME。
- 本特性以PDN连接粒度进行订阅，即一个PDN连接只允许订阅一个PRA区域。

## 原理概述

**PRA服务**

PRA区域由PLMN中的TAI、eNodeB ID和ECGI等类型的区域列表组成，可包含一个或多个TAI、eNodeB ID、ECGI或TAI+ECGI、eNodeB ID+ECGI组合，由PRA ID唯一标识。例如可以配置一个校园为一个PRA区域，以0x800000作为该区域的PRA ID，并将校园所在的位置区列表以TAI/eNodeB/ECGI类型配置到该PRA区域内。在用户进入运营商订阅的PRA区域中，可以享受到不同策略的服务。例如：在商场、校园、某些场馆、地铁沿线等运营商会提供基于位置的计费优惠、套餐服务等。

除了采用Gx接口连接PGW-C和PCRF的典型EPC组网之外， UNC 还允许采用N7接口连接PGW-C和PCF。PCF通过N7接口下发策略给PGW-C，MME实时将eNodeB侧上报的位置信息给SGW-C/PGW-C，PGW-C上报位置信息给PCF，PCF进行策略决策和更新。

通过配置（ [**SET POLICYMODE**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/基本功能/策略模式选择/设置策略接口的选择方式（SET POLICYMODE）_09653658.md) ）命令， UNC 可以选择是通过Gx接口连接PCRF还是通过N7接口连接PCF。在这种场景下，Gx接口的消息和相关的信元换成了N7接口的相关消息和信元。N7接口和Gx接口对应的消息如下：

| --- | --- |
| CC-Request(INITIAL_REQUEST)/CC-Answer | Npcf_SMPolicyControl_Create request/response |
| CC-Request(UPDATE_REQUEST)/CC-Answer | Npcf_SMPolicyControl_Update request/response |
| RA-Request/RA-Answer | Npcf_SMPolicyControl_UpdateNotify request/response |
| CC-Request(TERMINATION_REQUEST)/CC-Answer | Npcf_SMPolicyControl_UpdateNotify(delete) request/response<br>或Npcf_SMPolicyControl_Delete request/response |
