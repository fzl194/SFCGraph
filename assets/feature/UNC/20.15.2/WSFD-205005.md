## 定义

基于UE接入能力选择网关，是指 UNC 在使用 APN （Access Point Name）进行 DNS (Domain Name Service)解析获取网关 IP 地址前，将UE接入能力信息加入到APN-NI和APN-OI之间，然后将扩展的APN发送到DNS服务器解析，获取合建的GGSN/P-GW的IP地址。

UNC 能够根据UE的接入能力选择不同容量的GGSN/P-GW。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | UNC<br>能够为同时具有GU和LTE接入能力的UE选择一个合建的GGSN/P-GW，保证GERAN/UTRAN/E-UTRAN互操作的成功率。<br>随着具备E-UTRAN能力的UE越来越多，业务可以平滑切换到新部署合建GGSN/P-GW上。 |
| 用户 | 保证用户在GERAN/UTRAN和E-UTRAN网络间平滑迁移，而无需重新附着，保证了用户业务的连续性。 |

## 应用场景

基于UE接入能力选择网关特性适用于传统GGSN和新部署的合建GGSN/P-GW并存的组网场景。

UNC 可以为具有LTE接入能力的UE选择新部署的合建GGSN/P-GW，保证GERAN/UTRAN/E-UTRAN互操作的成功率。对于没有LTE接入能力的UE选择传统的GGSN。由于具有LTE接入能力的UE数量与日俱增，传统GGSN的流量需要平滑的切换到新部署的GGSN。

基于UE接入能力选择网关特性也适用于新部署的独立P-GW和合建GGSN/P-GW并存的组网场景。

UNC 可以为同时具有GU和LTE接入能力的UE选择合建的GGSN/P-GW，保证GERAN/UTRAN/E-UTRAN互操作的成功率。对于没有GU接入能力的LTE数据卡选择独立的P-GW，以方便运营商根据不同的业务规划灵活的部署网络。

## 原理概述

**UE接入能力**

UE接入能力由终端设备的能力和运营商的接入控制策略两方面共同决定。只有两个因素都支持UE在GERAN/UTRAN和E-UTRAN网络接入，系统才认为UE具备GERAN/UTRAN和E-UTRAN接入能力，即GUL能力。其中接入控制策略通过 [**ADD GBARD**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/接入限制管理/Gb模式接入限制参数/增加Gb模式接入限制参数(ADD GBARD)_26305284.md) / [**ADD IUARD**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/接入限制管理/Iu模式接入限制参数/增加Iu模式接入限制参数(ADD IUARD)_72345073.md) 命令控制是否允许UE在GERAN/UTRAN网络接入；通过 [**ADD S1ARD**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/接入限制管理/S1模式接入限制参数/增加S1模式接入限制参数(ADD S1ARD)_26145478.md) 命令控制是否允许UE在E-UTRAN网络接入。如 [图1](实现原理_70166789.md#ZH-CN_TOPIC_0170166789__fig3) 所示：

## 特性关系

**影响（affects）**

- [WSFD-205004](feature/UNC/20.15.2/WSFD-205004.md)：这些特性在为指定用户配置GGSN/P-GW选择策略时，可以选择S-GW优选或P-GW优选，如果不设置，本特性默认为S-GW优选。 在同时选择S-GW/P-GW时，S-GW优选和P-GW优选决定了以哪一个网关的优先级和权重为基准进行选择。

**依赖（depends_on）**

- [WSFD-106003](feature/UNC/20.15.2/WSFD-106003.md)：基于UE接入能力选择网关时需要同时考虑UE终端能力和用户接入控制特性中配置的控制类型。

- [WSFD-106203](feature/UNC/20.15.2/WSFD-106203.md)：先执行APN映射，后执行本功能。

**交互（interacts_with）**

- [WSFD-104601](feature/UNC/20.15.2/WSFD-104601.md)：当本特性与左侧特性同时部署时，不影响左侧特性网关选择功能，仅在其域名查询基础上增加优先选择支持DCNR能力的S/P-GW步骤。 当 UNC 进行网关路由选择时，根据左侧 [WSFD-205001 网关路由选择功能](../../网关选择功能/WSFD-205001 网关路由选择功能_63535123.md) 特性中的 [GGSN/P-GW路由选择](../../网关选择功能/WSFD-205001 网关路由选择功能/实现原理_70040601.md#ZH-CN_TOPIC_0170040601__p1156187501180641) ，如果已经配置根据指定的IP地址或Hostname选择P-GW路由，或者根据HSS签约数据选择P-GW路由，需要确认指定的网关或者签约的网关为高速网关。
