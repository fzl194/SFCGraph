# 本地QOS控制

`WSFD-109203` · enhanced · required

## 所属目录

- [WSFD-109200](feature/UNC/20.15.2/WSFD-109200.md)

## 适用性

- 适用NF：GGSN、SGW-C、PGW-C、SMF
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=M; MME(4G)=-; S/PGW-C(4G)=M; NB-IoT(4G)=-; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=M; AMF(5G SA)=-; SMF(5G SA)=M; NRF(5G SA)=-`

## 适用NF（原文）

GGSN、 SGW-C、PGW-C、SMF

## 定义

本地QoS控制是指 UNC 在本地配置基于全局或APN的QoS 信息 ，对用户携带的QoS进行控制。

> **说明**
> 智能QoS管理增强部署或改造需要评估现有网络的业务模型、资费策略、网络资源占用情况。具体部署或改造时，请联系华为技术支持。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 移动运营商可以灵活地控制用户的QoS参数。 |
| 用户 | 享有更好的业务质量。 |

## 应用场景

UNC 使用本地QoS策略对用户QoS进行控制适用于以下场景：

- 用户业务激活时。
- 用户在线时发生切换。
- 用户漫游属性更新时。

## 应用限制

- 本特性适用于本地Qos控制下的单承载场景，只适用2/3G一次PDP激活/更新流程，4G缺省承载激活流程，2/3G和4G切换场景中一次PDP和缺省承载之间的QoS映射，5G场景激活和4到5切换的场景，其他场景不适用。
- 对于PCC用户，不使用本地配置的QoS参数替换，本特性不关注。

## 原理概述

UNC 接收来自SGSN请求QoS参数后，可进行基于 UNC 本地配置的整机或APN的QoS协商和控制。

本地配置QoS： UNC 在本地配置基于全局或APN的QoS参数。

- 基于APN的QoS
  通过 [**ADD QOSPROFILE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/QoS模板/增加QoS描述配置（ADD QOSPROFILE）_09654430.md) 命令配置指定APN实例下的QosProfile，通过 [**ADD APN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/APN管理/APN/增加APN配置（ADD APN）_09653747.md) 配置APN实例，并通过 [**SET APNQOSATTR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/APN的QoS属性/设置指定APN的QoS属性配置信息（SET APNQOSATTR）_09651825.md) [设置指定APN的QoS属性配置信息（SET APNQOSATTR）](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/APN的QoS属性/设置指定APN的QoS属性配置信息（SET APNQOSATTR）_09651825.md) 为APN实例绑定QoS profile。
    - 若配置了别名APN，则不使用本地QoS策略，使用签约的QoS策略。
    - 若配置了虚拟APN，则使用虚拟APN配置的本地QoS策略。
- 基于全局的QoS
  通过 [**SET QOSGLOBAL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/全局QoS功能配置/全局QoS参数/设置全局QoS配置（SET QOSGLOBAL）_09652976.md) 命令配置全局的QoS，当不需要或者没有基于APN粒度设置QoS时，所有APN都关联到该QoS信息。

**GPRS/UMTS系统实现**

1、SGSN向GGSN发送“Create PDP Context Request”消息，消息中 携带SGSN和UE初步协商的QoS参数 。

2、GGSN根据收到的 QoS参数 进行相应处理，对于非PCC用户，使用本地配置的签约QoS参数进行替换。如果未配置本地签约QoS，则不做替换。本地签约QoS通过 [**ADD PRER8SUBQOS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/PreR8 QoS配置/本地PreR8 QoS/增加Pre-R8签约QoS配置（ADD PRER8SUBQOS）_09653616.md) 命令配置，通过 [**ADD QOSPROFILE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/QoS模板/增加QoS描述配置（ADD QOSPROFILE）_09654430.md) 或 [**SET QOSGLOBAL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/全局QoS功能配置/全局QoS参数/设置全局QoS配置（SET QOSGLOBAL）_09652976.md) 命令绑定到APN或全局上。

3、GGSN本地通过 [**ADD PRER8QOSACTION**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/PreR8 QoS配置/PreR8 QoS控制动作/增加Pre-R8 QoS控制动作配置（ADD PRER8QOSACTION）_09652510.md) 命令配置QoS上下行保证带宽门限、最高带宽门限以及带宽超过门限值时对QoS用户的处理动作，对用户携带的QoS参数进行检查，当用户激活或者更新请求中携带的QoS参数值超过了本地配置的QoS参数范围， UNC 将根据超过范围时的动作配置对用户进行相应处理：

- 如果设定的动作为degrade（降级），则自动降为配置的APN QoS值。
- 如果设定的动作为reject（拒绝），则拒绝创建上下文请求，协商结束，激活或更新失败。

4、上述步骤得到的QoS，经过纠错/映射/协商等操作后， 生成的 QoS ，并通过“Create PDP Context Response”消息反馈给SGSN。

> **说明**
> 纠错： UNC 根据协议检查UE请求QoS参数的合法性。对于业务相关的QoS参数，如果请求QoS参数非法，则根据 [**SET DEFPRER8QOS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/PreR8 QoS配置/全局PreR8 QoS纠错/设置缺省的Pre-R8 QoS参数（SET DEFPRER8QOS）_09654401.md) 命令配置的缺省QoS参数进行纠错和替换，纠错值等于本地配置的缺省QoS参数，否则纠错值等于请求值；对于业务不涉及的QoS参数，纠错值等于请求值。

**EPS系统实现**

- 基于整机的QoS控制
  如果 UNC 采用本地QoS策略，在用户建立缺省承载时， UNC 机对缺省承载携带的QoS参数进行更改，包括QCI、ARP和上下行APN-AMBR。
- 基于APN的QoS控制
  如果 UNC 采用本地QoS策略，在用户缺省承载建立时， UNC 可以基于APN对缺省承载携带的QoS参数进行更改，包括QCI、ARP和上下行APN-AMBR。
  UNC 支持对APN的总保证带宽进行设置，实现基于APN级的带宽控制功能。当APN的最大保证带宽已经被占满，将不允许新的用户再接入该APN，即接入该APN的新用户不能被激活。
- 基于APN下用户的QoS控制
  UNC 支持对用户级的总带宽进行设置，实现基于APN下用户级的带宽控制功能。当用户的最大保证带宽已经被占满，或对应APN的最大带宽已经被占满，则此时将不允许用户的新业务被激活。

1、MME向SGW-C/PGW-C发送“Create Session Request”消息，消息中 携带QoS参数 。

2、PGW-C根据收到的 QoS参数 进行相应处理，对于非PCC用户，在用户建立缺省承载时， UNC 使用本地配置的签约QoS对缺省承载携带的QoS参数进行替换，包括QCI、ARP和上下行APN-AMBR。如果未配置本地签约QoS，则不做替换。本地签约QoS通过 [**ADD EPSSUBQOS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/EPS QoS配置/本地EPS QoS/增加EPS签约QoS配置（ADD EPSSUBQOS）_09653648.md) 命令配置，通过 [**ADD QOSPROFILE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/QoS模板/增加QoS描述配置（ADD QOSPROFILE）_09654430.md) 或 [**SET QOSGLOBAL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/全局QoS功能配置/全局QoS参数/设置全局QoS配置（SET QOSGLOBAL）_09652976.md) 命令绑定到APN或全局上。

3、PGW-C本地通过 [**ADD EPSQOSACTION**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/EPS QoS配置/EPS QoS控制动作/增加EPS QoS控制动作配置（ADD EPSQOSACTION）_09654382.md) 命令配置QoS上下行保证带宽门限、最高带宽门限以及带宽超过门限值时对QoS用户的处理动作，对用户携带的QoS参数进行检查，当用户激活或者更新请求中携带的QoS参数值超过了本地配置的QoS参数范围， UNC 将根据超过范围时的动作配置对用户进行相应处理：

- 如果设定的动作为degrade（降级），则自动降为配置的APN QoS值。
- 如果设定的动作为reject（拒绝），则拒绝创建会话请求，激活或更新失败。

4、上述步骤得到的QoS，经过纠错/映射/协商等操作后， 生成的 QoS ，并反馈给SGW-C。

> **说明**
> 纠错： UNC 根据协议检查UE请求QoS参数的合法性。对于业务相关的QoS参数，如果请求QoS参数非法，则根据 [**SET DEFEPSQOS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/EPS QoS配置/全局EPS QoS纠错/设置EPS缺省QoS参数（SET DEFEPSQOS）_09653732.md) 命令配置的缺省QoS参数进行纠错和替换，纠错值等于本地配置的缺省QoS参数，否则纠错值等于请求值；对于业务不涉及的QoS参数，纠错值等于请求值。

**5GC系统实现**

1、AMF向SMF发送“Nsmf_PDUSession_CreateSMContext Request”消息，消息中 QoS参数 。

2、SMF根据收到的 QoS参数 进行相应处理，对于非PCC用户，使用本地配置的签约QoS参数进行替换。如果未配置本地签约QoS，则不做替换。

本地签约QoS通过 [**ADD 5GCSUBQOS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/5GC QoS配置/本地5GC QoS/增加5GC签约QoS配置（ADD 5GCSUBQOS）_09652601.md) 命令配置，通过 [**ADD QOSPROFILE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/QoS模板/增加QoS描述配置（ADD QOSPROFILE）_09654430.md) 或 [**SET QOSGLOBAL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/全局QoS功能配置/全局QoS参数/设置全局QoS配置（SET QOSGLOBAL）_09652976.md) 命令绑定到APN或全局上。

3、SMF本地通过 [**ADD 5GCQOSACTION**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/5GC QoS配置/5GC QoS控制动作/增加5GC QoS控制动作配置（ADD 5GCQOSACTION）_09652342.md) [增加5GC QoS控制动作配置（ADD 5GCQOSACTION）](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/5GC QoS配置/5GC QoS控制动作/增加5GC QoS控制动作配置（ADD 5GCQOSACTION）_09652342.md) 命令配置QoS上下行保证带宽门限、最高带宽门限以及带宽超过门限值时对QoS用户的处理动作，对用户携带的QoS参数进行检查，当用户激活或者更新请求中携带的QoS参数值超过了本地配置的QoS参数范围， UNC 将根据超过范围时的动作配置对用户进行相应处理：

4、上述步骤得到的QoS，经过纠错/映射/协商等操作后， 生成的 QoS ，并反馈给AMF。

**相关软参**

在2/3G一次PDP激活/更新流程及4G到2/3G切换场景下，当本地配置了签约QoS，可以使用软参 [BIT1111 控制当本地配置了subscriber qos时是使用配置值替换请求的QoS值还是进行协商](../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（SMF_GGSN_SGW-C_PGW-C）/QoS管理/BIT1111 控制当本地配置了subscriber qos时是使用配置值替换请求的QoS值还是进行协商_98883705.md) 控制是使用本地签约QoS直接替换请求的QoS值，还是使用本地签约QoS与请求的QoS值进行协商。

- Bit1111值为0时，表示使用本地配置的subscriber qos对创建请求和4G到2/3G切换的QoS值进行替换，更新场景不进行替换。

- Bit1111值为1时，表示使用本地配置的subscriber qos与创建、更新及4G到2/3G切换的QoS值进行协商。

在2/3/4G互操作、4G到5G切换场景下，当本地配置了基于APN的QoS，可以使用软参 [BIT907 控制基于APN的本地QoS定义功能是否应用于23G、4G和5G切换场景](../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（SMF_GGSN_SGW-C_PGW-C）/QoS管理/BIT907 控制基于APN的本地QoS定义功能是否应用于23G、4G和5G切换场景_11353236.md) 控制是否使用基于APN的本地QoS定义功能。

- Bit907值为0时，表示在23G、4G和5G切换场景不使用基于APN的本地QoS定义功能。
- Bit907值为1时，表示在23G、4G和5G切换场景使用基于APN的本地QoS定义功能。
