# 支持NF属性检查与校验

`WSFD-214101` · enhanced · required

## 所属目录

- [WSFD-214100](feature/UNC/20.15.2/WSFD-214100.md)

## 适用性

- 适用NF：NRF
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=-; S/PGW-C(4G)=-; NB-IoT(4G)=-; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=-; AMF(5G SA)=-; SMF(5G SA)=-; NRF(5G SA)=M`

## 适用NF（原文）

NRF

## 定义

为了避免NF的异常操作影响到NRF设备安全及5G Core业务，NRF支持对NF进行白名单和校验控制：符合NRF白名单的NF才允许注册/更新；NF进行上述服务化流程时，对应参数校验通过的NF才能完成对应的服务化流程。

为了避免号段类NF的号段配置错误影响到5G Core业务，NRF支持对NF Profile的属性冲突核验，对于存在号段冲突的情况可以查询或提示告警。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | NRF对NF进行有效性核查或冲突校验，可以及时发现NF误操作或者误配置，避免影响5G Core业务。 |
| 用户 | 用户不感知该特性。 |

## 应用场景

NF进行基本的服务化流程包含NF注册/更新/订阅通知/发现流程时，可以使用本特性进行相应的参数检查或属性冲突核查，避免误操作或者误配置导致5G Core业务受损。

## 应用限制

- NRF处于流控状态时，NF新注册/更新时MML触发的核验延迟，发现流程时停止NRF内及跨NRF冲突核查及告警，减少对NRF的负荷。
- 跨NRF的NF间属性冲突交叉验证只支持单层东西向互联组网场景。
- NRF上属性冲突自动核验只能核验出满足发现条件且存在冲突的NF，不能覆盖所有的已注册NF。

## 原理概述

NF Profile通过NF向NRF注册时携带，是后续NF发现和Token请求等的基础，因此NF属 Profile的正确性尤为重要。NRF针对NF Profile提供了接入控制、属性检查和属性校验等功能，确保NF业务的正常进行。

- **NF白名单接入控制功能**
  NRF提供了白名单功能，只有在白名单列表中的NF才允许注册/更新。NRF上通过 **[SET NRFWHITELISTSW](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF白名单配置/设置NF白名单开关（SET NRFWHITELISTSW）_35374753.md)** 命令开启NF白名单受控接入控制功能，并通过 **[ADD NRFWHITELIST](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF白名单配置/增加NF白名单（ADD NRFWHITELIST）_35519265.md)** 命令配置NF InstanceID白名单列表。此功能需要NF配合，待注册的NF需要首先进行人工配置核验，确保NF Profile没问题后，NRF才将此NF添加到白名单中，提升注册NF的NF Profile准确率。
- **NF服务化流程中的基础参数校验**
    - NF Profile必带参数校验
      对于大型运营商网络，同一种类型NF全网会规划多个，分别服务于不同范围或用户，每个NF上需要规划配置对应的服务属性范围，如果不携带这些属性，NRF会默认为此NF支持所有范围，可能会导致后面业务过程中选择的NF出现错误。因此，需要对特定NF必须携带的属性参数做校验，只有满足必带参数条件，才允许NF注册/更新/通知/发现等流程。
      NF和NRF侧都支持必带属性参数校验：

      - NF侧支持校验的NF类型及对应的参数参考[表1](#ZH-CN_CONCEPT_0000001137608223__zh-cn_topic_0166303311_table560712920507)，通过**[SET NFNRFMGMTPARA](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/服务化接口管理/注册与服务发现/NRF管理/NRF参数管理/NF与NRF间全局管理参数管理/设置NF与NRF间的全局管理参数（SET NFNRFMGMTPARA）_32041712.md)**命令中的 “NFPROFILECHKSW”参数控制。
                  - NF注册流程中，如果NF Service中注册参数检查不满足必选属性携带规则，NF侧会上报“ [ALM-100225 NF注册失败](../../../../../../网络运维/故障处理/UNC告警处理/业务告警/业务公共/ALM-100225 NF注册失败_60414514.md) ”告警。
                  - NF更新流程中，如果NF更新后的NF Service Profile不满足必选属性携带规则，NF侧会上报“ [ALM-100225 NF注册失败](../../../../../../网络运维/故障处理/UNC告警处理/业务告警/业务公共/ALM-100225 NF注册失败_60414514.md) ”告警。
          - NRF支持校验的NF类型及对应的参数参考[表2](#ZH-CN_CONCEPT_0000001137608223__zh-cn_topic_0166303311_table657784604817)，通过**[SET NRFTAKEPARARULE](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF携带参数处理规则/设置NF携带参数处理规则（SET NRFTAKEPARARULE）_88537110.md)**命令控制。
                  - NF注册流程中，如果NF注册参数检查不满足必选属性携带规则，NRF不允许NF注册并返回对应原因，需要NF侧人工调整注册参数。
                  - NF更新流程中，如果NF更新后的Profile不满足必选属性携带规则，NRF不允许NF更新并返回对应原因，需要NF侧人工调整NF参数。
                  - NF订阅通知流程中，由于NF属性变更触发订阅通知时，NRF会对属性变更后NF包含的属性进行校验，如果变更的NF不满足必选属性携带规则，NRF会将此NF作为Suspended状态发送订阅通知。
                  - NF发现流程中，NRF对服务发现结果的目标NF包含的属性进行校验，如果目标NF不满足必选属性携带规则，NRF不会将此NF作为服务发现结果返回。
                  - NRF还会周期性检查已注册的对应NF中是否携带了[表2](#ZH-CN_CONCEPT_0000001137608223__zh-cn_topic_0166303311_table657784604817)的参数信息，如果没有携带，上报“ [ALM-100315 NF携带信元防呆](../../../../../../网络运维/故障处理/UNC告警处理/业务告警/NRF/ALM-100315 NF携带信元防呆_11207409.md) ”告警。

      *表1 NF侧判断不同类型NF必须携带的NF Service Profile信息*

      | **NF**<br>**类型** | **NF Service**<br>**必带信息** |
      | --- | --- |
      | AMF | TAI List或TAI Range必带一个 |
      | SMF | TAI List或TAI Range必带一个 |
      | NWDAF | TAI List或TAI Range必带一个 |
      | BSF | IPv4或IPv6Prefix必带一个 |

      *表2 NRF侧判断不同类型NF必须携带的NF Profile信息*

      | **NF**<br>**类型** | **NF Profile**<br>**必带信息** |
      | --- | --- |
      | AMF | TAI List或TAI Range必带一个 |
      | SMF | TAI List或TAI Range必带一个 |
      | NWDAF | TAI List或TAI Range必带一个 |
      | AUSF | SUPI或RoutingIndicator必带一个 |
      | UDM | SUPI或GPSI或RoutingIndicator必带一个 |
      | PCF | SUPI或GPSI必带一个 |
      | UDR | SUPI或GPSI必带一个 |
      | CHF | SUPI或GPSI必带一个 |
      | CUSTOM_OCS | SUPI或GPSI必带一个 |
      | BSF | IPv4或IPv6Prefix必带一个 |
    - NF服务发现参数必带参数检查
      服务发现参数如果不规范，例如发现号段类NF时，服务发现参数中没有携带号码参数，可能会导致选择的目标NF不符合预期，影响业务。为了确保服务发现参数的规范性，NRF支持对特定目标NF的服务发现流程中的发现参数进行校验。对于服务发现特定NF，除基本发现参数及NF Type、Service Name外，必须携带其他发现参数，否则NRF会判断服务发现流程失败。 此功能通过 **[SET NRFDISCPARARULE](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF服务发现参数处理规则/设置NF服务发现参数处理规则（SET NRFDISCPARARULE）_35273633.md)** 命令灵活配置不同目标NF必须携带的服务发现参数。
    - NF服务发现参数可忽略参数检查
      NRF上可通过 **[ADD NRFIGNDISCPARA](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF服务发现忽略参数处理规则/增加NF服务发现忽略参数规则（ADD NRFIGNDISCPARA）_98328634.md)** 命令在发现特定NF类型时校验某些可以忽略的参数，减少对各NF发现参数要求，提升兼容性。例如即使请求方NF服务发现携带了preferred-locality参数，NRF可忽略此参数，选择只满足其他选择参数的目标NF。
    - NF其他参数校验检查
          - NRF支持对更新流程中的参数做如下校验：不允许更新NF Type或NF InstanceID，NRF会处理为更新失败。
          - NRF上支持对部分更新流程中更新参数是否携带array下标做校验，对于携带array下标的部分更新流程，NRF会处理为更新失败，避免NF和NRF间对array下标处理不一致而影响业务。此功能通过 **[SET NRFPATUPDIDXSW](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/对端NF管理/NF信息管理/PATCH更新携带下标处理开关/设置PATCH更新携带下标处理开关（SET NRFPATUPDIDXSW）_35519285.md)** 命令实现。其中array下标用于指定要更新的数组元素的位置。
- **NRF对特定NF的NF Profile的属性冲突核验**
    - NF Profile的属性冲突人工核验
      NRF支持通过 **[OPR NRFNFATTRVRY](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF属性冲突核验/操作执行NF属性冲突核验（OPR NRFNFATTRVRY）_35636461.md)** 命令对 [表3](#ZH-CN_CONCEPT_0000001137608223__zh-cn_topic_0166303311_table136111091503) 给出的NF进行对应属性冲突人工核验 **。** NF属性冲突人工核验参数通过 **[SET NRFVERIFYPARA](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF服务发现参数处理规则/设置NF服务发现参数处理规则（SET NRFDISCPARARULE）_35273633.md)** 命令配置。 NRF支持两种核验行为 ，可以通过 **[DSP NRFNFATTRVRY](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF属性冲突核验/显示NF属性冲突核验记录（DSP NRFNFATTRVRY）_35519269.md)** 查看核验结果 ：

      - 本NRF管理的同类型NF间属性冲突交叉验证：对于同一个NRF下，如果已注册的相同类型NF中有与当前NF存在属性值重叠，但两者大区/省份信息不同（大区及省份信息可以通过 **[SET NRFVERIFYPARA](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF属性冲突核验/设置NF属性冲突核验参数（SET NRFVERIFYPARA）_88697044.md)** 中参数设置），则认为当前NF存在NF间属性冲突。
          - NF属性与跨NRF寻址信息冲突交叉验证：如果为NRF东西向分层组网，核验跨NRF寻址信息中与当前NF存在属性值重叠，则认为存在NF属性与跨NRF寻址信息冲突。

      *表3 NF Profile的属性冲突 核验NF类型及对应属性*

      | **待核验NF**<br>**类型** | **可核验的属性** |
      | --- | --- |
      | UDM | IMSI、MSISDN、ROUTINGINDICATOR |
      | PCF | IMSI、MSISDN |
      | CHF | IMSI |
      | AUSF | IMSI、ROUTINGINDICATOR |
      | AMF | TAI/TAI Range |
      | SMF | TAI/TAI Range |
      | NWDAF | TAI/TAI Range |
      | BSF | IPV6PREFIX |
    - NF Profile的属性冲突自动核验
      服务发现流程中，NRF也实现了对 [表3](../../../../../业务专题/5G Core NRF解决方案/NRF解决方案概述/NRF可靠性及最大包处理能力_66303311.md#ZH-CN_TOPIC_0166303311__table136111091503) 给出的NF进行对应属性冲突核验，核验通过服务发现流程 **自动触发** 。如果存在属性冲突，则上报 [ALM-100325 服务发现NF属性核验冲突](../../../../../../网络运维/故障处理/UNC告警处理/业务告警/NRF/ALM-100325 服务发现NF属性核验冲突_19069243.md) 告警 。NRF上通过 **[SET NRFDISCVERIFY](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF属性冲突核验/设置服务发现NF属性冲突核验参数（SET NRFDISCVERIFY）_88377454.md)** 命令配置支持两种核验行为 ：

      - 本NRF管理的同类型NF间属性冲突交叉验证：对于同一个NRF下，如果发现结果中NF的存在大区/省份信息不同（大区及省份信息可以通过 **[SET NRFVERIFYPARA](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF属性冲突核验/设置NF属性冲突核验参数（SET NRFVERIFYPARA）_88697044.md)** 中参数设置），则认为当前NF存在NF间属性冲突，会上报[ALM-100325 服务发现NF属性核验冲突](../../../../../../网络运维/故障处理/UNC告警处理/业务告警/NRF/ALM-100325 服务发现NF属性核验冲突_19069243.md)告警。此种场景下，NRF可以通过**[SET NRFDISCVERIFY](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF属性冲突核验/设置服务发现NF属性冲突核验参数（SET NRFDISCVERIFY）_88377454.md)** 中的 “DISCPRELOCALSW”参数设置只返回与请求者同大区/省份的目标NF。
            > **说明**
            > NFInstanceID需符合运营商的规范。若不符合，强制开启 **[SET NRFDISCVERIFY](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NF属性冲突核验/设置服务发现NF属性冲突核验参数（SET NRFDISCVERIFY）_88377454.md)** 中的 “DISCPRELOCALSW” 参数 对应的冲突优先返回同大区/省份目标NF，则会影响业务。
          - NF属性与跨NRF寻址信息冲突交叉验证：如果为NRF东西向分层组网，本大区已经有满足发现条件的NF，核验跨NRF寻址信息也存在满足发现条件的路由信息，则认为存在NF属性与跨NRF寻址信息冲突，会上报[ALM-100325 服务发现NF属性核验冲突](../../../../../../网络运维/故障处理/UNC告警处理/业务告警/NRF/ALM-100325 服务发现NF属性核验冲突_19069243.md)告警。此种场景下，NRF只返回本大区的目标NF。
      NF侧也支持通过 **[SET NFDISCPLCY](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/服务化接口管理/注册与服务发现/本地NRF功能管理/NF发现策略管理/设置NF的服务发现策略（SET NFDISCPLCY）_09651764.md)** 命令的 “NFIDCHECK” 参数对NRF返回的服务发现结果NF进行属性冲突核验，如果服务发现结果中NF存在大区/省份信息不同，认为存在NF间属性冲突，会上报 [ALM-100306 NFInstanceID冲突](../../../../../../网络运维/故障处理/UNC告警处理/业务告警/业务公共/ALM-100306 NFInstanceID冲突_11538426.md) 告警。
