## 适用NF（原文）

SGSN

## 定义

SMS over GPRS/EDGE/WCDMA是指SGSN通过分组域为移动用户提供收发短消息的业务。SMS（Short Message Service）是电信业务的一种，它允许用户通过SMC（Short Message Center）发送和接收字符信息。SMS可以由电路域或分组域进行承载。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 短消息通过分组域下发给用户，能够更有效的利用<br>无线资源<br>，改善在通过电路域发送短消息时，如果处于话务高峰期，短消息会出现延迟的现象。 |
| 用户 | 用户可以更好地、及时地收发短消息。 |

## 应用场景

当某地区经常因为话务高峰期无线接口通道资源不足，而导致短消息延迟时，建议开启SMS over GPRS/EDGE/WCDMA特性。

## 原理概述

**SMS over GPRS/EDGE/WCDMA简介**

SGSN 支持 分组域 短消息的处理。SMS over GPRS/EDGE/WCDMA特性是指SGSN通过分组域为移动用户提供收发短消息的业务，包括SMMO（Short Message Mobile Originated）、SMMT（Short Message Mobile Terminated）以及SM（Short Message） Alert。

SMS可以承载于 CS 网络，也可以承载于PS网络。 GPRS 附着或GPRS附着 IMSI 未附着的MS通过PS域实现短消息的接收与发送；GPRS附着并且IMSI也附着的MS可以通过PS域或CS域实现短消息的接收与发送。B类手机的分组域短消息只能在通话结束状态才能够进行，可通过挂起恢复流程实现电话业务和分组域短信业务的 切换 。

> **说明**
> - A类手机：A类操作模式的MS可以同时操作GPRS和其他GSM业务。
> - B类手机：B类操作模式的MS可以同时监视GPRS和GSM业务的控制通道，但每次只能操作一种类型的业务。
> - C类手机：C类操作模式的MS只能操作GSM业务。

短消息的实现需要多个实体共同完成，其业务网络结构如 [图1](#ZH-CN_CONCEPT_0185247546__zh-cn_concept_0130429336_fig1) 所示。

**图1** 短消息业务网络结构

<br>

![](特性概述_85247546.assets/zh-cn_image_0193298345_2.png)

由 [图1](#ZH-CN_CONCEPT_0185247546__zh-cn_concept_0130429336_fig1) 可知，SGSN可以从 SMS-IWMSC （Short Message Service Interworking MSC ）接收短消息，并通过Gb或Iu接口发送给MS；也可以从MS接收短消息，通过Gd接口发送给 SMS-GMSC （SMS Gateway MSC）。Gb接口或Iu接口、Gd接口完成短消息的前转功能，C接口为MS接收短消息时提供路由功能，Gr接口在实现短消息提醒功能和短消息 鉴权 时使用。华为短消息系统由短消息中心SMC（Short Message Center）实现，SMC集成了SMS-GMSC、SMS-IWMSC和SM-SC（Short Message-Service Center）的功能。各主要实体的功能如 [表1](#ZH-CN_CONCEPT_0185247546__zh-cn_concept_0130429336_tbl_entity) 所示。

*表1 主要实体的功能*

| 实体 | 功能 |
| --- | --- |
| SGSN | - 签约检查在进行业务之前，对用户签约数据是否允许相应的业务的一种检查，是SGSN收到移动用户发起的短消息后的自动操作，以判断该用户是否能够发起短消息。若该用户可以发送短消息，则将SM发送到SMS-GMSC。<br>- 业务闭锁检查进行业务之前，对该业务是否被闭锁的一种检查。是SGSN收到移动用户发起的短消息后的自动操作，以判断该用户是否能够发起短消息。 |
| SMS-GMSC | 可接收<br>PLMN<br>内的短消息，检查目的MS的地址，向相关HLR获取路由信息并传送给接收方SC（Service Center）。 |
| SMS-IWMSC | 可接收从SC方传来的短消息，从HLR获得路由信息和SMS信息，并下发此短消息给接收方MS的<br>VMSC<br>。 |
| HLR | HLR检查MS的鉴约数据（如ODB数据和呼叫受限信息）判断此MS是否能够接收SMS。如果SMS业务允许，HLR返回用户当前所在的SGSN号和MSC号给SMS-GMSC；如果MT SMS业务受限，HLR返回响应消息，带上相应原因值结束处理。 |
