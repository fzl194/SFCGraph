# 支持Direct Tunnel功能

`WSFD-104506` · base · required

## 所属目录

- [WSFD-104500](feature/UNC/20.15.2/WSFD-104500.md)

## 适用性

- 适用NF：SGSN、GGSN
- 代际支持矩阵：`SGSN(2.5&3G)=M; GGSN-C(2.5&3G)=M; MME(4G)=-; S/PGW-C(4G)=-; NB-IoT(4G)=-; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=-; AMF(5G SA)=-; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

SGSN、 GGSN

## 定义

随着3G业务不断的开展，以及HSPA等技术的应用，UMTS网络越来越需要提高用户面处理能力。原有的网络是分别在RNC与SGSN、SGSN与GGSN-U之间建立GTP-U通道的（Indirect Tunnel）。为此，RNC、SGSN以及GGSN-U等网元需要同时增强相应的用户面处理性能，这样会增加运营商的资金投入和运营成本。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 节约SGSN用户面资源，减少相关运营投资和运营费用。<br>- 实现了信令面和用户面分离，便于以后升级到EPC网络。<br>- 用户面的扩容不再需要升级SGSN，只需要升级GGSN-U和RNC，从而提高了网络的可扩容性。 |
| 用户 | 减少用户数据时延，提升用户体验。 |

## 应用场景

为了减少运营商的投资费用和运营费用，以及便于后续的网络扩容，3GPP提出了Direct Tunnel解决方案。Direct Tunnel功能将RNC与SGSN、SGSN与GGSN-U之间用户面的两段隧道（Indirect Tunnel）优化为一段隧道，优化后用户面转发不经过SGSN，而直接在RNC和GGSN-U之间建立GTP-U隧道，通过节省用户面资源降低了运营商的资金投入和运营成本，同时也优化了UMTS网络用户面的性能。

## 可获得性

**涉及NF**

| 涉及NF | 版本支持 | 功能说明 |
| --- | --- | --- |
| RNC | 无特殊要求 | 使用和GGSN-U之间的直连GTP-U隧道，完成用户上下行数据的转发。 |
| SGSN | UNC 20.3.0及后续版本 | - 根据运营商的配置以及用户的类型来判断是否需要启用Direct Tunnel。<br>- 为满足条件的用户建立RNC和GGSN-U之间的直连GTP-U隧道。 |
| GGSN-C | UNC 20.3.0及后续版本 | 处理SGSN的Direct Tunnel建立请求，完成RNC和GGSN-U之间直连GTP-U隧道的建立。 |
| GGSN-U | UDG 20.3.0及后续版本 | 使用和RNC之间的直连GTP-U隧道，完成用户上下行数据的转发。 |

## 对系统的影响

- 对于信令面，SGSN在PDP Context的激活、保留以及Service Request流程中增加了向GGSN-C发送Update PDP Context消息以及GGSN-C发送PFCP Session Modification消息的操作，增加了SGSN、GGSN-C和GGSN-U的信令处理负荷。
- 对于用户面，GGSN-U的数据直接发送到RNC，减少了SGSN的处理以及数据传递的时延，从而可以提高用户体验；同时还减少了SGSN用户面资源，因此无影响。

## 应用限制

| 无法部署Direct Tunnel解决方案的场景 | 说明 |
| --- | --- |
| 外网用户。<br>说明：外网用户包括漫游用户和拜访用户。<br>- 漫游用户：指使用其归属PLMN的GGSN的外网用户。<br>- 拜访用户：指使用SGSN所在PLMN的GGSN的外网用户。 | 外网用户需要使用拜访网络中SGSN用户面的功能，SGSN需要对用户流量计费，用于运营商之间的结算。 |
| RNC/SGSN/GGSN不支持Direct Tunnel功能。 | - |
| RNC不能启用Direct Tunnel（如RNC与GGSN不能直接通信，缺少相关配置路由）。 | - |
| GGSN不支持V1版本的GTP-C协议。 | V0版本的GTP-C无法携带Direct Tunnel扩展标识。 |
| 运营商的计费策略是需要核减SGSN的S-CDR流量。 | SGSN需要处理用户数据，并收集S-CDR所需的数据，如流量/时长。 |
| [WSFD-206005 Smartphone控制基础功能](../../流控功能/WSFD-206005 Smartphone控制基础功能_85152745.md)<br>中配置禁止使用Direct Tunnel功能。 | - |
| [WSFD-206006 Smartphone异常信令节省](../../流控功能/WSFD-206006 Smartphone异常信令节省_85152749.md)<br>中启用Parking APN假激活功能。 | - |
| 2G用户业务、非HPLMN和MVNO用户业务。 | 只能使用Indirect Tunnel方式。 |
