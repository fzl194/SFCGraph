# VoLTE基础语音业务

`WSFD-102001` · base · required

- [WSFD-102000](feature/UNC/20.15.2/WSFD-102000.md)

- 适用NF：MME、SGW-C、PGW-C
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=M; S/PGW-C(4G)=M; NB-IoT(4G)=-; 5G MME(5G NSA)=M; 5G S/PGW-C(5G NSA)=M; AMF(5G SA)=-; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

MME、SGW-C、PGW-C

## 定义

VoLTE（Voice over LTE），一种基于LTE传输语音的解决方案。随着移动宽带网络的演进，移动语音从软交换向移动宽带语音的方向发展。无线侧从2G/3G演进到LTE，核心网从CS演进到IMS。华为提供基于IMS的VoLTE移动宽带语音方案，实现了语音业务和数据业务融合的新模式（都通过LTE进行传输），该方案具有高带宽、低时延的特性，让用户体验到极致的语音接续服务。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 使运营商由传统的CS网络提供语音服务，演进到LTE网络提供语音服务，提高了运营商的竞争力。<br>- 支持在传统CS网络基础上进行融合部署，降低运营商的成本。 |
| 用户 | - 相对于传统CS语音业务，VoLTE语音业务降低了接入时延，提高了语音质量，提升用户体验。<br>- LTE用户可自助开通VoLTE业务，无需换卡换号，业务开通高效便捷。 |

## 应用场景

用户通过LTE接入，进行语音业务。

## 对系统的影响

- 本特性部署后，UE除了建立用于数据传输的PDN连接，还会建立用于IMS语音的PDN连接，在进行通话时会建立QCI=1的专有承载，因此该特性对运营商的话务模型有影响。部署该特性时需要考虑已有设备的容量是否可以满足新的话务模型，必要的话需要先对设备进行扩容。
- 本特性部署后对系统性能有一定影响，影响程度取决于VoLTE用户比例和相应的话务模型，部署前请先联系华为技术支持进行评估。

## 应用限制

目前暂不支持网络侧发起专有承载创建作为IMS信令上下文的场景。

**VoLTE注册流程**

VoLTE用户进行语音业务时，需要先进行网络注册，包括EPC网络附着和IMS网络注册。

- EPC网络附着：UE发起附着请求后，EPC网络先对UE进行鉴权，鉴权通过后建立用于IMS语音业务的缺省承载（QCI=5），并获得用于IMS注册的P-CSCF地址列表。
- IMS网络注册：包括基本注册和第三方注册。基本注册过程中，IMS网络和UE进行双向鉴权，鉴权通过后，S-CSCF从融合HLR/HSS下载用户数据。第三方注册过程中，S-CSCF根据用户数据中的iFC（initial Filter Criteria）触发到ATS进行注册，ATS从融合HLR/HSS下载UE的业务数据。

**VoLTE** **基本呼叫流程**

VoLTE基本语音呼叫建立过程如下描述。

1. UE拨号发起呼叫，通过缺省承载与IMS网关交互SIP信令，完成号码分析、主被叫信令建立等动作。
2. 通话建立之后，PGW-C/PGW-U根据PCRF下发的策略发起专有承载（QCI=1）建立过程。
3. 用户进入通话，语音数据通过专有承载进行传输。
4. 用户通话结束后，UE发出拆线请求，MME与IMS网关交互SIP信令，完成拆线、资源释放等动作。
