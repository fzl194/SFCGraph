# NSA用户数据话单生成和上报

`WSFD-011511` · protocol · required

## 所属目录

- [WSFD-011510](feature/UNC/20.15.2/WSFD-011510.md)

## 适用性

- 适用NF：SGW-C、PGW-C
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=-; S/PGW-C(4G)=-; NB-IoT(4G)=-; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=M; AMF(5G SA)=-; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

SGW-C、PGW-C

## 定义

NSA组网场景下，UNC支持将E-UTRAN上报的5G用户流量单独生成话单，上报给计费网关CG，并进一步将话单传递给计费中心BS。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 支持5G用户流量离线计费，在4G/5G融合场景下，区分5G用户单独计费。 |
| 用户 | 记录5G用户使用数据业务的详细情况，便于核减消费情况。 |

## 应用场景

该特性适用于NSA Option3/3a/3x组网以及NSA Option3/3a/3x组网与2/3/4G组网共存场景。

## 可获得性

**涉及NF**

| 涉及NF | 支持版本 | 功能说明 |
| --- | --- | --- |
| UE | 无特殊要求 | 支持NSA双连接，即支持DCNR能力。 |
| eNodeB | 无特殊要求 | - 接收MME在消息中携带的NR UE Security Capabilities信元。<br>- 支持在消息中携带5G流量统计报告（Secondary RAT Usage Report List）。 |
| gNodeB | 无特殊要求 | 接收MME在消息中携带的NR UE Security Capabilities信元。 |
| MME | UNC 20.3.0及后续版本 | 支持从既有S1消息或者新增的Secondary RAT Data Usage Report消息中解析到Secondary RAT Usage Report List信元后，根据用户的当前流程状态转发5G计费流程统计报告。 |
| SGW-C/PGW-C | UNC 20.3.0及后续版本 | 按照预先设置的计费规则生成原始话单，发送到CG。 |
| CG | 无特殊要求 | 对SGW-C/PGW-C发送来的话单进行存储、合并、标准化预处理、增加运营商自定义字段，并将处理后的标准计费数据发送到BS。 |

**License** **支持**

本特性无需获得License许可即可获得该特性的服务。

## 对系统的影响

本特性对系统无影响。

## 应用限制

- SGW-C/PGW-C支持最多与32个CG对接。
- CG支持的话单版本字段需要与SGW-C/PGW-C的话单版本字段支持能力一致。

## 原理概述

NSA组网场景下，终端能同时建立到eNodeB和gNodeB的无线承载，并且在各自承载上能独立进行上下行的用户数据传输。EPC侧无法区分是4G流量还是5G流量。当需要对5G流量单独计费时，需由gNodeB进行流量统计，并通过接口发给eNodeB。随后eNodeB以承载的粒度向MME上报5G计费流量统计，并由MME转发给SGW-C/PGW-C。SGW-C/PGW-C根据MME上报的5G计费流量统计，生成话单上报到CG。NSA组网场景下5G数据的计费如 [图1](#ZH-CN_CONCEPT_0228784141__fig1) 所示。

**图1** NSA组网场景下（以Option3x为例）5G数据的计费示意图

<br>

![](特性概述_28784141.assets/zh-cn_image_0228835441_2.png)

1.gNodeB进行5G流量统计。

2.gNodeB通过接口发给eNodeB。

3.eNodeB将消息转发至MME，消息中携带5G计费流量统计报告（Secondary RAT Usage Report List，参见3GPP TS 36413）。eNodeB可以在如下消息中携带5G计费流量统计报告：

- E-RAB MODIFY RESPONSE
- E-RAB RELEASE RESPONSE
- E-RAB RELEASE INDICATION
- E-RAB MODIFICATION INDICATION
- UE CONTEXT RELEASE REQUEST
- UE CONTEXT RELEASE COMPLETE
- UE CONTEXT SUSPEND REQUEST
- SECONDARY RAT REPORT

4.MME从eNodeB发送来的S1接口消息中解析到Secondary RAT Usage Report List信元后，根据当前的流程状态上报5G计费流量到SGW-C/PGW-C。MME可以在如下消息中携带 **Secondary RAT Usage Data Report** 信元：

- Create Session Request
- Modify Bearer Request
- Delete Session Request
- Delete Bearer Response
- Delete Bearer Command
- Release Access Bearers Request
- Change Notification Request

5.SGW-C/PGW-C生成话单并上报到CG。SGW-C/PGW-C需配置 [**ADD CDRFIELDTEMP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/离线计费/话单字段控制/话单字段模板/增加话单字段模板（ADD CDRFIELDTEMP）_09896890.md) 命令中的 “RAN-SecondaryRAT-Usage-Report” 参数，使话单支持携带RAN-SecondaryRAT-Usage-Report字段。

> **说明**
> - 5G数据流量需要在SGW-CDR/PGW-CDR话单中区分上下行。
> - SGW-C/PGW-C支持R8、R9和R10版本话单。
> - SGW-C/PGW-C话单支持携带5G用户扩展QoS字段。

## 计费与话单

话单中携带RAN-SecondaryRAT-Usage-Report字段。
