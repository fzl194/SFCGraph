# 终端二次鉴权

`WSFD-108007` · enhanced · required

## 所属目录

- [WSFD-108000](feature/UNC/20.15.2/WSFD-108000.md)

## 适用性

- 适用NF：SMF
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=-; S/PGW-C(4G)=-; NB-IoT(4G)=-; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=-; AMF(5G SA)=-; SMF(5G SA)=M; NRF(5G SA)=-`

## 适用NF（原文）

SMF

## 定义

终端二次鉴权（UPF转接）功能主要应用在5G场景下，由于UPF下沉，当DN-AAA部署到企业或园区时，SMF与DN-AAA之间不能直接互通，需要经过UPF转接。此场景下SMF和UPF间建立设备级的N4隧道（N4 GTP-U隧道），用来转接SMF与DN-AAA之间二次鉴权的消息。二次鉴权消息对UPF而言为普通的数据报文。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 安全性增强：SMF不直接通过大网跟DN-AAA对接，提高安全性。<br>- 满足特定组网要求：满足DN-AAA部署在企业内部或者园区网络内部，与SMF之间不能直接互通场景的组网要求。 |
| 用户 | 用户不感知该特性。 |

## 应用场景

UPF下沉，DN-AAA部署到企业或园区，SMF与DN-AAA之间不能直接互通，需要UPF转发SMF与DN-AAA之间二次鉴权的消息。

## 对系统的影响

如果UPF到DN-AAA抄送服务器的转发路径故障或者抄送服务器故障时，SMF无法感知抄送服务器的状态。

## 应用限制

- 当启用终端二次鉴权功能后，SMF不支持EAP鉴权方式，只支持PAP、CHAP鉴权；
- SMF与DN-AAA之间的二次鉴权消息只支持Radius协议，不支持Diameter协议。
- 一个SMF网元上部署的所有业务中，直连AAA和经UPF中转AAA场景中的Radius Server IP不可以相同，否则会造成直连AAA场景业务受损。

## 原理概述

**二次鉴权经UPF转接组网方案**

UPF下沉到边缘，连接企业、园区网络。企业、园区部署DN-AAA，与SMF间不能直接互通。SMF和DN-AAA之间的二次鉴权消息经UPF转接。

**二次鉴权消息经UPF转接流程**

**图2** 二次鉴权经UPF转接流程图

<br>

![](WSFD-108007 终端二次鉴权特性概述_73984008.assets/zh-cn_image_0000001150149114_2.png)

1. SMF选择UPF创建Radius消息专用的N4会话，给UPF发送PFCP Session Establishment Request消息。
2. UPF收到会话创建请求，根据PDR/FAR中携带的source interface、destination interface及各自对应的3gpp interface type，识别此会话为创建N4的GTP-U隧道的专用N4会话。
3. UPF收到SMF发送的报文后，解析GTP-U头的信息，匹配到SMF下发的PDR，使用对应的FAR进行报文转发，剥离GTP-U信息，直接转发IP报文（即Radius消息）给DN-AAA。
4. UPF收到DN-AAA发送的Radius消息后，匹配SMF下发的PDR，使用对应的FAR，对收到的IP报文进行GTP-U加封装，转发GTP-U报文给SMF，完成消息转发。

**终端二次鉴权功能：**

1. 对接多个企业/园区
  UPF可能同时对接多个企业、园区， UNC 同时支持不少于6000个UPF转接会话。
2. 支持重选UPF转发
  N4-U会话建立后，如UPF故障，SMF会选择另外的UPF发送二次鉴权消息。
3. 支持重选DN-AAA
  当DN-AAA Server故障，SMF发送二次鉴权消息无响应，SMF会选择另外的DN-AAA发送二次鉴权消息
4. 边缘和中心UPF主备容灾
  边缘UPF和中心UPF之间进行容灾，正常情况下用户通过边缘UPF访问园区业务。当边缘UPF故障时用户通过中心UPF访问园区业务。
