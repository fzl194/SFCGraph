# 服务区域限制

`WSFD-105006` · protocol · required

## 所属目录

- [WSFD-105000](feature/UNC/20.15.2/WSFD-105000.md)

## 适用性

- 适用NF：AMF
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=-; S/PGW-C(4G)=-; NB-IoT(4G)=-; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=-; AMF(5G SA)=M; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

AMF

## 定义

服务区域限制，AMF根据用户签约、AMF本地配置或PCF下发的服务限制区域，识别UE是否在业务允许区域内，如果UE不处于允许区域，UE可以发起注册流程，但是不支持发起会话和其他业务（普通的数据业务）。服务区限制对于紧急呼叫、高优先级业务不做限制。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 限制终端在指定区域的业务可用性，满足运营商基于区域的差异化运营。 |
| 用户 | 满足用户在不同区域内的差异化用户服务和体验。 |

## 应用场景

运营商希望针对指定的区域，差异化用户的服务和体验，可以开启本特性。

## 原理概述

**5G移动性限制背景介绍**

网络的移动性管理既要保证UE可达和数据传输连续，又要防止UE接入某些受限区域或者请求不该使用的服务。移动性限制就是指3GPP内UE的接入访问受限、移动受限等。

5G协议中则明确定义了移动性接入限制信息（Access And Mobility Subscription Data）有RAT限制、核心网限制、禁止区域、服务区限制四种类型， UNC 作为移动性限制的决策中心，支持实现移动性限制，并且根据不同类型的移动性接入限制信息实现不同的限制功能：

*表1 移动性接入限制信息*

| 类型 | 功能 | 相关特性 |
| --- | --- | --- |
| RAT限制（RAT restriction ） | 用户接入过程中检查用户的无线接入方式是否满足RAT限制。 | [用户接入控制功能（适用于AMF）](../../基本接入功能/WSFD-106003 用户接入控制功能/用户接入控制功能（适用于AMF）_48552634.md)<br>。 |
| 核心网限制（Core Network type restriction） | 用户接入过程中检查用户的核心网接入方式是否满足核心网限制，即是否限制用户接入5GC或者EPC。 | [用户接入控制功能（适用于AMF）](../../基本接入功能/WSFD-106003 用户接入控制功能/用户接入控制功能（适用于AMF）_48552634.md)<br>。 |
| 禁止区域（Forbidden Area） | 用户接入过程中检查用户接入的区域是否是禁止区域，如果UE处于禁止区域，将限制用户注册。 | [WSFD-105003 区域漫游限制（适用于AMF）](../WSFD-105003 区域漫游限制/WSFD-105003 区域漫游限制（适用于AMF）_48669740.md)<br>。 |
| 服务区限制（Service Area Restriction） | 用户触发数据业务，UNC检查该用户是否在允许进行数据业务区域。如果UE不处于允许区域，将限制用户使用普通的数据业务（除紧急呼叫、高优先级业务）。 | 本特性。 |

**服务区域限制**

服务区域限制（Service Area Restriction）是对5G终端数据业务的一种控制手段，在限制的服务区域内，UE可以发起注册流程，但是不支持发起会话和其他业务（普通的数据业务）。对于紧急呼叫、MPS/MCS等高优先级业务则不做限制。

服务区域限制是UE、基站和核心网相互协同完成的，在用户初始接入到5GC的时候，AMF综合UE签约、PCF下发的策略将服务区域限制信息下发给UE以及NG-RAN，UE本地保存这些信息，并遵循这些规则。

- 如果接入到5GC的UE在之前的流程中已经获取了服务区域限制信息，那么在发起业务时该UE能主动识别在当前区域的业务可行性。
- 如果UE尚未获取服务区域限制信息（如初次接入），或者服务区域限制信息发生了变化但UE未及时感知，这种情况下就需要核心网（AMF）识别是否限制UE发起业务。

UE注册流程中AMF为其分配服务区域限制信息的主要流程参见 [图1](#ZH-CN_CONCEPT_0276900357__fig014191912617) ，注册流程包括初始注册、5G内的移动性注册更新以及4G到5G的移动性注册更新。关键流程为：

- AMF从UDM取签约数据，包含服务区域限制信息。
- AMF将包括签约的服务区域限制信息在内的数据通过AM策略流程发送给PCF，并从PCF获取调整后的服务区域限制信息。
- AMF如果获取了服务区域限制信息：
    - 在给UE的Registrationaccept/Configurationupdatecommand消息的Service area list信元中包含服务区域限制列表。
    - 在给NG-RAN的Initial Context Setup Request /DownlinkNASTransport/HandoverRequest消息中通过Mobility Restriction List携带服务区域限制列表。
