# 基于服务PLMN的eMTC终端接入速率控制

`WSFD-216105` · enhanced · required

## 所属目录

- [WSFD-216100](feature/UNC/20.15.2/WSFD-216100.md)

## 适用性

- 适用NF：SGW-C、PGW-C
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=-; S/PGW-C(4G)=M; NB-IoT(4G)=-; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=-; AMF(5G SA)=-; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

SGW-C、PGW-C

## 定义

eMTC（enhanced Machine Type Communication）业务在正常场景下，终端发送的都是低频率数据包，但终端在异常场景（如超时重发等）下，可能在短时间内高频率发送数据包，此时可能会导致网络拥塞并影响业务运行，所以3GPP协议引入终端发送数据的速率控制机制。基于服务PLMN的eMTC终端接入速率控制在MME上配置速率门限，并发给UE和SGW-C/PGW-C，用于控制UE和SGW-C/PGW-C固定时间（6分钟）内上行和下行数据包的数目。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 避免由于终端异常而导致网络拥塞，有效提升整网的可靠性。 |
| 用户 | 提高普通终端用户业务的成功率和业务体验。 |

## 应用场景

本特性应用于eMTC控制面优化（Control Plane Optimization）数据传输中，用于限制终端的数据传输速率，在终端异常情况下可以防止网络拥塞而影响业务，提升整网的可靠性。

## 对系统的影响

性能影响程度：低

基于服务PLMN的eMTC终端接入速率控制只对超过门限的数据包丢包处理，对系统影响可忽略。

## 应用限制

- 基于服务PLMN的eMTC终端接入速率控制仅适用于控制面CP优化（Control Plane Optimization）场景。
- 用户在附着后短时间内发送一定的数据包后退网，当该用户再次入网时之前发送的数据包不会被UNC统计到单位时间内发送的数据包内，UNC重新开始进行该用户的速率控制计时和计数。

## 原理概述

**相关概念**

控制面优化：

控制面优化即CP优化 (Control Plane Optimization )，在终端和MME之间通过NAS（Non-access Stratum）消息传输业务层数据，在MME和SGW-C之间通过S11-U接口传递业务层数据，无需反复建立和删除空口无线数据承载DRB（Data Radio Bearer）和S1-U连接，有效提升了数据传输效率，同时也可以降低对网络的信令负荷。具体请参考 [WSFD-215101 基于信令面的数据传输（SGW-C）](../../NB-IoT业务功能/WSFD-215101 基于信令面的数据传输/WSFD-215101 基于信令面的数据传输（SGW-C）_77260994.md) 。

基于服务PLMN的eMTC终端接入速率控制是在MME上配置速率门限，发给UE和SGW-C/PGW-C，SGW-C/PGW-C根据MME发来的速率控制信息，产生数据包的控制速率，对于超过门限的数据包进行丢包处理，未超过门限的数据包进行正常的转发处理。

基于服务PLMN的eMTC终端接入速率控制流程图如 [图2](#ZH-CN_TOPIC_0275993428__fig2) 所示。

- 在UE附着/PDN连接建立流程中，MME在本地配置了速率控制门限参数后，MME把Create Session Request消息发送给SGW-C/PGW-C，SGW-C/PGW-C上超过速率控制的用户报文被丢包；没有超过速率控制的用户报文可以正常处理转发。
- MME通过Activate Default EPS Bearer Request消息把Serving PLMN Rate Control发送给UE，UE限制单位时间数据传输速率。
