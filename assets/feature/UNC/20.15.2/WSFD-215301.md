# HSS免升级

`WSFD-215301` · enhanced · required

## 所属目录

- [WSFD-215300](feature/UNC/20.15.2/WSFD-215300.md)

## 适用性

- 适用NF：MME
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=-; MME(4G)=M; S/PGW-C(4G)=-; NB-IoT(4G)=M; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=-; AMF(5G SA)=-; SMF(5G SA)=-; NRF(5G SA)=-`

## 适用NF（原文）

MME

## 定义

在HSS未升级不支持NB-IoT终端接入、无法进行Non-IP数据传输的情况下，通过配置使MME发送给HSS可以识别的消息和信元，建立Non-IP的PDN上下文，保证NB-IoT终端成功接入网络，采用Non-IP Data Over NAS进行数据传输。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 减少客户在部署NB-IoT时对整个网络的影响，减少运营成本。<br>基于现网HSS设备直接叠加业务，进行端到端的调测，运维简单，节约成本。提升运营商在IoT领域的竞争力。 |
| 用户 | 用户不感知该特性。 |

## 应用场景

NB-IoT技术广泛应用于多种行业，如智能抄表、智能照明、智能垃圾桶、智能宠物健康跟踪、智能农业等，而在诸多行业和业务中，运营商需要确认部署NB-IoT后对周边网元的诉求和影响，开销很大，且有不确定性。

当需要HSS在不升级的情况下，就可以支持NB-IoT用户的接入并可以采用Non-IP Data Over NAS进行数据传输时，建议开启本特性。

[**系统实现**](#ZH-CN_TOPIC_0275808665__sub1)

1. 运营商确认本特性License打开，并收集HSS的版本支持情况，在UNC上进行配置。
    - HSS能够识别NB-IoT RAT，将 [**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 的参数 “NBIOTRAT（是否支持NB-IoT RAT）” 设为 “SUPPORT” 。此时，MME将发送携带RAT-Type为EUTRAN-NB-IoT的消息，MME将结合来自HSS的Update Location Answer消息中的信元ARD（Access Restriction Data）对终端进行接入控制。
      HSS支持Non-IP签约数据下发，无需特别进行配置。
    - HSS不能识别NB-IoT RAT，将 [**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 的参数 “NBIOTRAT（是否支持NB-IoT RAT）” 设为 “NOT_SUPPORT” ，MME将发送携带RAT-Type为EUTRAN的消息。
      HSS不支持签约PDN类型为Non-IP的APN，需要通过 [**ADD NONIPSUBCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/Non-IP APNNI配置/增加Non-IP APNNI配置(ADD NONIPSUBCTRL)_72225451.md) 增加Non-IP APNNI配置。
2. NB-IoT终端进行附着流程或TAU流程，消息按照配置携带相应HSS能够接收的信元。
3. HSS的版本情况不影响NB-IoT终端接入网络，能够采用Non-IP Data Over NAS进行数据传输，MME和HSS之间Update Location Request/Answer、Insert Subscriber Data Request/Answer、Delete Subscriber Data Request/Answer消息成功交互。
4. MME发送Attach Accept/TAU Accept消息给NB-IoT终端，流程完成。

## 特性关系

**影响（affects）**

- [WSFD-011601](feature/UNC/20.15.2/WSFD-011601.md)：当HSS不能支持NB-IoT终端接入时，HSS免升级特性是否开启，影响NB-IoT终端能否成功接入网络。
- [WSFD-011601](feature/UNC/20.15.2/WSFD-011601.md)：当HSS不能支持NB-IoT终端接入或不支持Non-IP签约数据下发时，HSS免升级特性是否开启，影响NB-IoT终端能否成功接入网络。
- [WSFD-215103](feature/UNC/20.15.2/WSFD-215103.md)：当对端HSS不支持NB-IoT业务时，需要开启 [WSFD-215301 HSS免升级](../../WSFD-215301 HSS免升级_75808644.md) ，通过配置使HSS支持NB-IoT业务。
- [WSFD-215103](feature/UNC/20.15.2/WSFD-215103.md)：当HSS不能支持NB-IoT终端接入或不支持Non-IP签约数据下发时，HSS免升级特性是否开启，影响NB-IoT终端能否正常采用Non-IP Data Over NAS进行数据传输。
