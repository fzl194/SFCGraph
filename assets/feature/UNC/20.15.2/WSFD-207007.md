`WSFD-207007` · enhanced · required

## 定义

基于5G的网络共享（5G MOCN）是指在5G网络中采用MOCN（Multi-Operator Core Network）方式的网络共享。在这种模式下，多家运营商共享 无线接入网 设备和资源，但不共享核心网设备，独立建立各自的核心网，有利于节省运营商的投资成本。

## 客户价值

| **受益方** | **受益描述** |
| --- | --- |
| 运营商 | 多个运营商可以共享使用一套接入设备，从而降低设备成本，有利于节省运营商的投资成本。 |
| 用户 | 用户可以自主选择接入的运营商，有机会享受差异化服务。 |

## 应用场景

当多个运营商需要共享使用一套接入设备进行网络运营时需要开启本特性，基于5G的MOCN网络共享典型组网如 [图1](../WSFD-207003 基于LTE的网络共享（MOCN）/特性概述_03670020.md#ZH-CN_CONCEPT_0203670020__fig1990723511311) 所示，其中运营商A、B、C分别采用自己的核心网设备，无线接入网由一个运营商X建设，A、B、C均租用运营商X的接入设备进行组网建设。

**图1** MOCN典型组网图

<br>

![](WSFD-207007 基于5G的网络共享（MOCN）特性概述_29397150.assets/zh-cn_image_0229403616_2.png "点击放大")

> **说明**
> 典型组网图中仅列举了在共享区域内建设一个gNodeB，实际上对于一个运营商在共享区域内建设gNodeB的数量并没有要求限制。

该场景的主要特征如下：

- 共享区域的网络（gNodeB）由一个运营商建设，该网络共享给其他几个运营商。共享网络由运营商Operator X建设，该网络共享给运营商Operator A、Operator B和Operator C。X可以是A、B和C中的任何一个或都不是。
- 共享的无线接入网（gNodeB）需要和多个运营商的核心网设备相连。
- 用户接入网络时，各个运营商签约的UE需要被路由到各自的CN（Core Network）运营商为其服务。

## 原理概述

**相关概念**

- Home PLMN
  HPLMN（Home Public Land Mobile Network ）指归属公共陆地移动网，如果UE的MCC、MNC与某个PLMN匹配时，该PLMN便是这个UE的Home PLMN，用来标识签约用户的归属运营商。一个运营商可以支持多个拥有不同Home PLMN的签约用户，这种情况下我们称该运营商支持多HPLMN。
- Global gNodeB ID
  gNodeB的全局标识，使用基站归属运营商的PLMN，Global gNodeB ID=基站归属运营商的PLMN+gNodeB ID。
- Supporting UE
  支持网络共享的UE，能够解析共享网络广播系统信息中的共享网络信息，在共享网络里选择一个核心网运营商作为自己的服务运营商。
- Selected PLMN
  UE在无线网络中选择的PLMN就是Selected PLMN。基站根据UE的Selected PLMN构造TAI（Selected PLMN+TAC），并且在UE的初始注册消息中带给核心网。所以AMF能通过TAI获取Selected PLMN。Selected PLMN是在无线领域的概念，对应核心网领域就是Serving PLMN。
- Serving PLMN
  最终为Supporting UE提供服务的核心网运营商称之为Serving PLMN。如果用户驻留在自己的归属网络，那么Serving PLMN就等于该用户的Home PLMN（或者EHPLMN）。对于支持多Home PLMN的运营商，为了简化网络管理，往往仅会使用其中某个Home PLMN作为其唯一标识，这个Home PLMN在AMF上映射为Serving PLMN的概念，其它PLMN都是其等价Home PLMN（EHPLMN）。即使运营商的Serving PLMN只有一个，但是其所支持的Home PLMN可能有多个。

**MOCN模式下Serving PLMN选择**

在MOCN网络共享模式下存在多个PLMN，UE需要选择合适的PLMN进行接入，如 [图2](#ZH-CN_CONCEPT_0229397150__fig1044513261671) 。当NG-RAN支持网络共享（MOCN）时，AMF1和AMF2共享NG-RAN，NG-RAN会广播多个运营商的PLMN列表，发送给AMF1和AMF2。各运营商在NG-RAN中有各自独立的TAI，但是TAC是相同的，只不过PLMN不同。AMF将其支持的Serving PLMN列表返回给NG-RAN。NG-RAN侧维护AMF设备标识与各AMF支持的PLMN列表，用于后续UE注册接入时的路由选择。

**图2** MOCN模式下Serving PLMN选择

<br>

![](WSFD-207007 基于5G的网络共享（MOCN）特性概述_29397150.assets/zh-cn_image_0231369965_2.png)

NG-RAN需要分别与多个运营商的AMF建立N2连接，如 [图3](#ZH-CN_CONCEPT_0229397150__fig138621018413) 。

**图3** NG Setup流程

<br>

![](WSFD-207007 基于5G的网络共享（MOCN）特性概述_29397150.assets/zh-cn_image_0231453734_2.png)

- NG-RAN在与AMF建立N2连接的过程中，NG-RAN会将其广播的PLMN列表以及各PLMN下支持的网络切片列表在NG SETUP REQUEST消息中发送给AMF。
- AMF将其支持的Serving PLMN列表，不论NG-RAN是否支持该PLMN，都在NG SETUP RESPONSE消息中返回给NG-RAN。
- 通过N2连接建立过程，NG-RAN侧维护了它连接的AMF设备标识与各AMF支持的PLMN列表的对应关系。
- Supporting UE发起到核心网的初始注册请求时，先获取共享NG-RAN的广播系统信息后解析其中的网络共享信息，按照特定的优先级选择Selected PLMN提供给NG-RAN，NG-RAN根据AMF标识与PLMN之间的对应关系选择合适的AMF，并在发送给AMF的消息中携带Selected PLMN。AMF根据漫游策略、接入策略控制判断是否允许用户接入：
    - 允许，则将该Select PLMN作为该UE的Serving PLMN，并在发送给UE的TAI信元中携带Serving PLMN ID。
    - 不允许，拒绝该用户接入。

**在MOCN网络共享模式下切换** **流程**

**图4** 切换流程

<br>

![](WSFD-207007 基于5G的网络共享（MOCN）特性概述_29397150.assets/zh-cn_image_0000002132082208_2.png)

1. 源NG-RAN发送Handover Required（包含Target ID）给源AMF（S-AMF），通知有用户要进行切换。
2. 如果源AMF不能再服务UE，则源AMF选择一个可服务的目标AMF（T-AMF）。源AMF向NRF发送Nnrf_NFDiscovery_Request消息请求，消息中携带Target ID和特定条件属性等信息。
3. NRF向源AMF发送Nnrf_NFDiscovery_Response消息，确定目标AMF。
4. 源AMF给目标AMF发送Namf_Communication_CreateUEContext Request消息（包含Target ID）发起切换资源分配流程。目标AMF从源AMF拿到Target ID信元后，**同时使用****目标Global gNodeB ID及目标TAI****检查**，判断切换类型（Inter or Intra）。
  > **说明**
  > Target ID包含目标Global gNodeB ID以及目标TAI等，Global gNodeB ID使用的是目标基站归属运营商的PLMN，目标TAI使用的是目标运营商的PLMN，在目标基站没有启用MOCN的场景下，上述两个PLMN是相同的。在目标基站启用MOCN的场景下，上述两个PLMN可能不相同。所以，需要同时使用目标Global gNodeB ID及目标TAI检查。

5. 目标AMF根据Target ID确定目标NG-RAN。目标AMF给目标NG-RAN发送Handover Request消息，请求建立无线侧网络资源。
6. 目标NG-RAN向目标AMF发送Handover Request Acknowledge消息。
7. 目标AMF给源AMF发送Namf_Communication_CreateUEContext Response消息。
8. 后续流程和Handover公共流程相同。
