`WSFD-011603` · protocol · required

## 适用NF（原文）

MME

## 定义

NB-IoT UE状态上报功能是第三方的应用服务器（AS）/SCS（Services Capability Server）通过SCEF（Service Capability Exposure Function）向电信网络获取NB-IoT终端的状态、位置等信息或者下发某些参数（例如Active timer、周期TAU timer）以影响终端的行为。HSS在收到SCEF的能力订阅请求后转给MME，MME将订阅的报告返回给SCEF，以便第三方服务器根据UE的状态更好地对NB-IoT终端提供服务，或者让NB-IoT终端更加省电。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 建立第三方应用厂家跟NB-IoT终端的连接管道，让第三方应用厂家及时感知NB-IoT终端的状态，以辅助各种物联网应用的展开。 |
| 用户 | 用户可享受更优的服务体验，终端更加省电。 |

## 应用场景

本特性应用于SCS/AS向电信网络订阅请求终端状态的场景。典型场景可包括：

- 智能路灯：进行远程路灯控制或者主动查询路灯状态，需要订阅“UE可达监测事件”或“DDN失败后可达监测事件”。
- 智能抄表：进行远程水表（气表）阀门控制，主动查询水表（气表）状态，需要订阅“UE可达监测事件”或“DDN失败后可达监测事件”。
- 智能物流：当需要查询设备最近一次出现的位置信息，需要订阅“用户位置监测事件”。
- 智能运维：NB-IoT网络智能运维场景，需要订阅NB-IoT终端“连接丢失监测事件”和“通信故障事件”，获知RAN/NAS Release Cause标识的通信故障。

## 应用限制

- UE reachability包括数据可达和SMS可达两种类型，目前本版本仅支持数据可达。
- MME不支持External Identifier信元，需要SCEF在收到MME上报的事件后映射为External Identifier发给SCS。
- NB-IoT能力开放订阅流程中设置的周期TAU定时器，仅对支持长周期的M2M终端生效。
- 不支持对Configuration-Information-Request和Configuration-Information-Answer消息的处理。

## 原理概述

第三方的应用服务器（AS）/SCS（如 [图1](#ZH-CN_TOPIC_0000001495011277__fig01) 中的IoT平台）通过SCEF向电信网络获取NB-IoT终端的状态、位置等状态信息，以便更好地对NB-IoT终端提供服务；或者通过IoT平台下发某些参数给MME以影响终端的行为，让NB-IoT终端更加省电。这个过程就是所谓的NB-IoT能力开放。本特性中主要开放与UE的状态信息相关的能力，称为UE状态上报功能。

**图1** NB-IoT业务能力开发组网图

<br>

![](WSFD-011603 NB-IoT UE状态上报功能特性概述_95011277.assets/zh-cn_image_0000001447988586_2.png "点击放大")

<br>

这种能力开放是通过“订阅-报告”的形式实现，即AS通过SCEF向电信网元订阅指定NB-IoT终端的一种或者某几种事件，每种事件表示电信网络可开放的能力。当MME上打开NB-IoT业务能力开发功能后（ [**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 勾选相关能力以及本特性license打开），MME在用户触发Update Location Request流程时，将MME支持的开发能力列表告知HSS。当SCEF向HSS发送订阅请求，HSS将订阅请求发送给MME，MME上报报告。NB-IoT UE状态上报功能支持如 [表1](#ZH-CN_TOPIC_0000001495011277__tab4) 所示的事件上报。

*表1 NB-IoT UE状态上报功能支持的开放能力*

| 订阅能力 | 描述 |
| --- | --- |
| Loss of connectivity | 连接丢失，表示UE在指定的时间内没有触发信令或者进行数据传输，MME判定UE连接丢失。 |
| UE reachability | UE可达，表示UE从休眠状态进入连接或者可寻呼状态。 |
| Location Reportin | 位置信息，表示获取UE的当前实际位置或者当前已知位置。 |
| Communication failure | 通信失败，表示UE的数据承载被异常释放。 |
| Availability after DDN Failure | DDN失败后可用，表示UE在寻呼失败后重新接入到网络。 |
