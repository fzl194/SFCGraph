# DHCPv6地址分配

`WSFD-104005` · protocol · required

## 所属目录

- [WSFD-104000](feature/UNC/20.15.2/WSFD-104000.md)

## 适用性

- 适用NF：GGSN、SGW-C、PGW-C、SMF
- 代际支持矩阵：`SGSN(2.5&3G)=-; GGSN-C(2.5&3G)=M; MME(4G)=-; S/PGW-C(4G)=M; 5G MME(5G NSA)=-; 5G S/PGW-C(5G NSA)=M; AMF(5G SA)=-; SMF(5G SA)=M; NRF(5G SA)=-`

## 适用NF（原文）

GGSN、 SGW-C、PGW-C、SMF

## 定义

UNC 在PDU会话建立阶段，通过DHCPv6 Client功能从外置的DHCPv6服务器的地址池中获取IPv6地址和相关配置信息，然后将获取到的用户IPv6地址、DNS地址 和P-CSCF地址 等信息分配给IPv6用户或者IPv4v6双栈用户，满足基于IPv6解决方案组网下IPv6用户或者IPv4v6双栈用户的接入。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 通过外置DHCPv6服务器分配IPv6地址，实现用户IPv6地址、DNS地址、P-CSCF地址的动态管理。增强了接入业务中地址分配部署选择的多样性，运营商可以根据自身业务场景，灵活选择相关的业务方案，节省地址资源，降低运营成本。 |
| 用户 | 用户不感知。 |

## 应用场景

DHCP地址分配适用于运营商管理大量地址池，也可以用于企业网自己管理地址池。

## 对系统的影响

本特性对系统无影响。

## 应用限制

- UNC分配给用户的P-CSCF地址最大支持2个，包括UNC本地分配或外置DHCPv6服务器分配的P-CSCF地址。
- 仅支持在DHCP过程中获取P-CSCF服务器地址，不支持获取P-CSCF服务器域名。

## 原理概述

DHCPv6分配地址的方式是指移动用户所分配的IPv6地址是在PDU会话建立阶段从DHCPv6服务器上配置的地址池中获取的动态IP地址。 UNC 作为DHCP Client，代理用户向DHCPv6 Server申请IP地址。

UNC 收到用户的激活请求，携带代理IP向DHCPv6 Server申请IP地址，DHCPv6 Server从与该代理IP相同网段的地址池中为用户分配一个IPv6地址。

UNC 向DHCPv6 Server发送DHCPv6 Relay-forward Solicit消息时，如果需要获取用户的DNS地址或P-CSCF地址， 可通过 [**SET DHCPPARAREQ**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/DHCP管理/DHCP请求信元参数信息/设置向外部DHCPv4或者DHCPv6服务器发送的消息中的请求信元中的参数信息（SET DHCPPARAREQ）_32224051.md) 进行设置； 设置后DHCPv6 Solicit消息Option Request Option中将携带DNS Recursive Name Server Option（DNS地址选项）、SIP Servers IPv6 Address List（P-CSCF地址选项）进行地址申请。

如果DHCPv6服务器支持给该用户分配DNS或P-CSCF地址，则在Relay-reply Advertise消息中携带DNS Recursive Name Server Option（DNS地址选项）、SIP Servers IPv6 Address List（P-CSCF地址选项），返回分配的具体地址信息。
