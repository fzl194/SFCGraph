# N6/Gi/SGi接口IPv6组网

`GWFD-020402` · base · required

## 所属目录

- [GWFD-020400](feature/UDG/20.15.2/GWFD-020400.md)

## 适用性

- 适用NF：PGW-U、UPF
- 代际支持矩阵：`GGSN(2G&3G)=M; S/PGW-U(4G)=M; S/PGW-U(5G NSA)=M; UPF(5G)=M`

## 适用NF（原文）

PGW-U、UPF

## 定义

UDG 支持在PDN/DN侧基于IPv6（Internet Protocol Version 6）技术的组网，连接PDN/DN网络的物理接口可以配置为IPv6地址，同时还支持PMTU（Path Maximum Transmission Unit）探测及IPv6动态路由协议，包括OSPFv3（Open Shortest Path First version 3）和BGP4+（Border Gateway Protocol 4+）。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 解决IP地址缺乏问题。IPv6的源地址与目的地址长度都是128位（16字节），可以提供超过3.4×10<br>38<br>种可能的地址空间，可以满足多层次的地址划分需要，以及公有地址和机构内部私有网络地址的分配。在IPv6网络中可以不再使用一些地址节约技术，如NAT（Network Address Translation）等。<br>- 提高中间设备的转发效率。IPv6的地址空间采用了层次化的地址结构，利于路由快速查找，同时借助路由聚合，可减少IPv6路由表的大小。 |
| 用户 | 用户不感知该特性。 |

## 应用场景

- 开展IPv6业务应用。
- 解决IPv4地址枯竭问题。

## 对系统的影响

尽管IPv6地址长度是IPv4地址长度的四倍，但IPv6基本报文头的长度只有40字节，为IPv4报文头长度（不包括选项字段）的两倍，对转发性能影响不大。

## 应用限制

需要对端网元支持IPv6功能。

## 原理概述

**系统实现**

- PMTU
  PMTU是从源端到目的端路径上合适的IPv6 MTU（Maximum Transmission Unit）值，源端按照该值发送报文，使得报文在整个传输过程中不需要分片，减轻中间路由设备的工作压力，以便有效地利用网络资源并得到最佳的吞吐量。 UDG 支持以下两种方式确定PMTU：

  - 手工配置静态PMTU
      UDG 支持为指定的目的IPv6地址配置静态的PMTU值。当 UDG 从接口发送报文时，将比较该接口的MTU与指定目的IPv6地址的静态PMTU值，当报文长度大于二者中最小值时，采用该最小值对报文进行分片。
    - PMTU发现（Path MTU Discovery）
      UDG 支持源端到目的端PMTU的动态发现，源端使用这个MTU值发送后续报文到目的端。 UDG 支持PMTU老化时间的配置，当老化时间到达时，动态确定的PMTU值将被删除，源端通过PMTU发现机制重新确定发送报文的MTU值。
      如 [图1](#ZH-CN_CONCEPT_0166623501__fig15654193143811) 所示，PMTU发现的工作过程是：源端主机 UDG 先使用MTU=1500向目的主机发送报文，如果中间路由设备给源端返回一个错误消息ICMP error:Packet too big，其中包括该设备的MTU值，源端主机使用该MTU值来重新发送这个报文，如此反复，直到目的端主机收到该报文，从而确定网络中两台主机之间能够处理的最大报文的大小。在PMTU发现过程结束之前，可能会出现反复发送报文和收到报文太大消息，这是因为可能会不断发现更远的路径链路有更小的IPv6 MTU。
      **图1** PMTU发现的工作过程

      <br>

      ![](GWFD-020402 N6_Gi_SGi接口IPv6组网特性概述_66623501.assets/zh-cn_image_0000001286746686.png)

  - 6PE/6VPE
      6PE（IPv6 Provider Edge）是在IPv4网络中利用MPLS（Multiprotocol Label Switching）隧道技术为不同地区的被分割的IPv6网络提供连通服务。将被分割的IPv6网络利用隧道技术连接起来的方式有很多，6PE方式的隧道是在ISP的PE设备上实现IPv4/IPv6双协议栈，利用MP-BGP为其分配的标签标识IPv6路由，并通过PE之间的LSP实现IPv6之间的互通。
      6PE/6VPE设备是指在基于IPv4的MPLS核心网络上提供BGP-MPLS IPv6/BGP-MPLS IPv6 VPN服务的PE设备。
      如 [图2](#ZH-CN_CONCEPT_0166623501__fig1617551383715) 所示， UDG 具备6PE的基本功能，支持IPv4/IPv6双协议栈； UDG 与P、其他PE之间利用IPv4路由协议交换路由，将生成或配置的用户下行路由发布给远端PE的IPv6网络，并学习远端PE发布的IPv6路由； UDG 与其他PE之间建立LSP隧道来透明传输IPv6报文。
      **图2** UDG 作6PE设备接入MPLS网络
