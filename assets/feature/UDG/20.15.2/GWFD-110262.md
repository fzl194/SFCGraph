# RTSP头增强

`GWFD-110262` · enhanced · required

## 所属目录

- [GWFD-110260](feature/UDG/20.15.2/GWFD-110260.md)

## 适用性

- 适用NF：PGW-U、UPF
- 代际支持矩阵：`GGSN(2G&3G)=M; S/PGW-U(4G)=M; S/PGW-U(5G NSA)=M; UPF(5G)=M`

## 适用NF（原文）

PGW-U、UPF

## 定义

RTSP（Real Time Streaming Protocol）头增强指对RTSP报文头的内容做增强处理，将 [用户信息](#ZH-CN_CONCEPT_0162388776__table72434169445) （如APN、IMEI等）中的一个或多个信息添加到RTSP报文头，并传递给流媒体服务器Streaming Server，运营商和服务提供商可基于这些信息灵活开展业务和实施控制，例如业务认证、广告宣传、业务推广和账户管理与查询。

> **说明**
> RTSP头增强在RTSP报文头的扩展字段插入特定的用户标识信息，如IMSI/MSISDN/IP地址，存在将用户的个人身份信息泄露的风险。请遵循当地法律允许的目的和范围内启用相应的功能，以确保符合当地通信自由与个人隐私保护相关的法律要求。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 对用户业务进行认证，只允许签约用户访问专有业务。<br>- 联合服务提供商进行广告宣传、业务推广等，从而获得相关收益。 |
| 用户 | - 接收个性化业务内容，享受专有服务。<br>- 使用实时便捷的账户管理与查询功能。 |

## 应用场景

由于Streaming Server通常部署在数据通信网络中，因此在未开启RTSP头增强特性时，Streaming Server无法得知用户的相关信息，运营商无法对用户的业务访问进行控制。而当在 UDG 上开启RTSP头增强特性后，用户的相关信息（如APN、IMEI等）中的一个或多个信息就可以由 UDG 添加到RTSP报文头中传递给Streaming Server，从而很好的满足了运营商针对用户做业务访问认证的需求。

除了可以支持业务访问认证外，如果运营商或服务提供商的Streaming Server需要根据用户的属性信息，向用户返回个性化管理与查询页面以及计费结算，RTSP头增强特性也可以很好的满足该业务需求。

## 对系统的影响

当在 UDG 上开启RTSP头增强特性后，由于匹配到头增强动作的用户所有报文， UDG 都需要对其进行报文的解析和RTSP头增强处理，因此系统处理负荷将增加，报文转发性能和吞吐量将下降。

详细性能影响需要基于流量模型进行评估，请联系华为技术支持获取服务。

## 应用限制

- 由于IPv6报文不支持中间路由器分片，所以对IPv6报文进行RTSP头增强时需要限制头增强插入字段的长度。建议设置TCP MSS ≤ PMTU - IPv6首部（40字节）- TCP首部（20字节）- 头增强字段的长度。头增强插入字段过长，导致报文长度超过PMTU值将导致IPv6业务无法访问。
- 如果网络设备的MTU设置过小，用户数据报文到达 UDG 前被分片， UDG 不对分片报文进行头增强处理。
- RTSP头域分片的请求报文不支持头增强功能。
- 网络中同时部署外置PCEF与内置PCEF的场景下，若采用外置PCEF计费，则内置PCEF上不建议部署RTSP头增强特性。RTSP头增强插入的信息改变了报文长度，导致用户在外置PCEF上计费有误差。建议将头增强特性和计费特性部署在同一个网元上，或者在内置PCEF上计费，在外置PCEF上部署头增强。
- 用户所访问的流媒体服务器使用的协议是RTSP over HTTP时， UDG 不支持对这类报文进行头增强处理。
- RTSP头增强最多支持插入3个拼接字段，每个拼接字段最多支持4个参数拼接。

## 原理概述

根据《RFC 2616: Hypertext Transfer Protocol -- HTTP/1.1》对HTTP协议的解释，HTTP报文头中的字段是灵活可扩展的。HTTP头增强特性的实现即在HTTP请求报文头中插入运营商规划的自定义字段内容。每一组字段都是按照<字段前缀名称: 字段值>的协议标准格式来进行组织插入。实时流协议在语法和操作上与HTTP/1.1类似，因此RTSP的扩展机制与HTTP的扩展机制基本相同。具体说明如下：

- 字段前缀名称
  字段前缀名称的数据源是在 UDG 上预先设定的配置信息，当需要进行RTSP头增强处理时， UDG 会读取本地预先设定好的配置作为字段前缀名称的内容。字段前缀名称的内容可以按照运营商的需求在 UDG 上通过 [**ADD HEADEN**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/头增强控制/头增强/增加头增强（ADD HEADEN）_82837516.md) 命令灵活配置。
- 字段值
  字段值的数据源是业务访问用户的具体信息。字段值的内容是灵活可变的，当用户访问触发RTSP头增强特性的特定数据业务时，用户的相关信息会实时的作为字段值内容由 UDG 插入到RTSP报文中传递给Streaming Server。

RTSP支持对RTSP Setup、Play、Pause、Teardown、Options、Describe请求报文做头增强处理。

RTSP头增强支持插入的字段如 [表1](#ZH-CN_CONCEPT_0162388776__table72434169445) 所示，详细描述请参见 [**ADD HEADEN**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/头增强控制/头增强/增加头增强（ADD HEADEN）_82837516.md) 命令。

*表1 头增强支持插入的字段*

| 字段类型 | 字段值说明 | 加密 |
| --- | --- | --- |
| MSISDN | 用户激活时携带的MSISDN号。 | 支持 |
| IMSI | 用户激活时携带的IMSI号。 | 支持 |
| IMEI | 用户激活时携带的国际移动设备标识。IMEI依赖于SGSN/MME/S-GW是否携带，如果SGSN/MME/S-GW不携带，则RTSP头增强中无法插入该信息。 | 支持 |
| SGSN IP | 用户激活和更新时上下文中携带的SGSN/S-GW信令面IP地址，IP地址类型可以为IPv4或IPv6。 | 支持 |
| SUBPROFILE | 用户在华为PCRF/PCF上开户时存储的用户信息（如性别、年龄），可通过N4接口的私有信元Subscriber Profile下发给<br>UDG<br>。 | 支持 |
| MS IP | 用户的IP地址信息，IP地址类型可以为IPv4或IPv6。 | 支持 |
| APN | 用户激活时所属的APN名称。该APN可以是真实APN或请求APN。 | 支持 |

| User-Profile-alias | user-profile的别名，用来标识一个user-profile，用户业务匹配上相应user-profile时，在RTSP头增强中插入user-profile别名，发送给外部服务器，便于运营商了解该user-profile的使用情况。User-Profile-alias的值来源于<br>UDG<br>本地配置。 | 不支持 |
| MCC | 用户激活时携带的移动国家码。 | 不支持 |

插入字段值安全机制如下：

- RTSP头增强功能支持MD5、RC4、AES-128、AES-256、RSA-1024、RSA-2048、SHA-256加密算法。MD5和SHA256算法支持盐值加密。
  > **注意**
  > MD5、RC4、RSA-1024有安全风险，不建议使用。
- 对端服务器具备base64或ASCII解码能力时，建议在UDG上对通过加密的头增强报文进行base64编码或ASCII编码，可以有如下两种配置方式：
    - 将[**ADD HEADEN**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/头增强控制/头增强/增加头增强（ADD HEADEN）_82837516.md)命令的“CIPHERTXTENCODE”参数配置为“INHERIT”后，可以使用[**SET BASE64**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/业务控制公共配置/BASE64编码/设置Base64编码规则（SET BASE64）_82837587.md)命令开启base64编码功能，可以使用[BIT475](../../../../OM参考/软件参数/UDG软件参数/用户面软件参数/软件参数/业务控制管理/BIT475 控制头增强插入项加密后的值是否进行ASCII编码_79100759.md)软参开启ASCII编码功能。
    - 将[**ADD HEADEN**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/头增强控制/头增强/增加头增强（ADD HEADEN）_82837516.md)命令的“CIPHERTXTENCODE”参数配置为“BASE64”开启base64编码功能，将[**ADD HEADEN**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/头增强控制/头增强/增加头增强（ADD HEADEN）_82837516.md)命令的“CIPHERTXTENCODE”参数配置为“ASCII”开启ASCII编码功能。

## 计费与话单

由于业务处理时都是先做计费再进行头增强处理的（与规则优先级无关），所以RTSP头增强特性对用户计费无影响，由 UDG 通过RTSP头增强特性插入的信息不会作为用户上网使用的业务量进行计费统计。
