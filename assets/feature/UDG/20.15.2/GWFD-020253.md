# P-CSCF故障时IMS业务恢复

`GWFD-020253` · base · required

## 所属目录

- [GWFD-020250](feature/UDG/20.15.2/GWFD-020250.md)

## 适用性

- 适用NF：PGW-U、UPF
- 代际支持矩阵：`GGSN(2G&3G)=M; S/PGW-U(4G)=M; S/PGW-U(5G NSA)=M`

## 适用NF（原文）

PGW-U、UPF

## 定义

UDG 采用ICMP的方式，周期性的检测P-CSCF状态，当检测到 UDG 与P-CSCF链接状态异常时， UDG 会上报链接异常告警给SGW-C/PGW-C，SGW-C/PGW-C将一对状态正常的P-CSCF地址推送给MS/UE，MS/UE从中选择一个P-CSCF地址，重新发起IMS业务，从而保证IMS业务自动恢复。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 及时发现<br>UDG<br>与P-CSCF之间的链接故障，并且<br>UDG<br>主动采取故障处理措施，减少业务中断时间，减少用户投诉。 |
| 用户 | 及时恢复IMS业务，提升用户体验。 |

## 应用场景

P-CSCF是PS域接入IMS域的第一个网络节点。由于 UDG 与P-CSCF设备之间缺少心跳探测消息，用户业务出现故障后，运营商不能及时发现问题和处理问题，所以需要有一种状态探测机制来提醒运营商的运维人员。为了保证用户的IMS业务能够及时恢复， UDG 提供一种故障自处理功能，缩短IMS业务的中断时长。

## 可获得性

**涉及网元**

| 涉及网元 | 支持版本 | 功能说明 |
| --- | --- | --- |
| PGW-C/SMF | UNC 20.0.0及后续版本 | - 在用户激活过程中PGW-C/SMF下发一对P-CSCF地址给MS/UE，供MS/UE选择其中一个发起语音业务。<br>- PGW-C/SMF从PCRF/PCF获取MS/UE注册的P-CSCF地址，将P-CSCF的地址保存在PDP/承载上下文中。<br>- 收到PGW-U/UPF的告警后，PGW-C/SMF会给受影响的MS/UE发起更新请求，请求中携带一对P-CSCF地址，通知UE使用状态正常的P-CSCF设备进行IMS业务。 |
| PGW-U/UPF | UDG 20.2.0及后续版本 | - PGW-U/UPF周期性发送ICMP报文，检测PGW-U/UPF与P-CSCF之间的链路状态。<br>- 一旦探测到PGW-U/UPF与P-CSCF链路中断，<br>UDG<br>会上报告警，并且PGW-C/SMF会给受影响的MS/UE发起更新请求，请求中携带一对P-CSCF地址，通知MS/UE使用状态正常的P-CSCF设备进行IMS业务。 |
| PCRF/PCF | 无特殊要求 | PCRF/PCF将MS/UE地址与P-CSCF地址的对应关系，通过动态rule的方式下发给<br>UNC<br>。 |
| P-CSCF | 无特殊要求 | P-CSCF从注册消息中获取MS/UE地址与P-CSCF地址的对应关系，将对应关系封装在AAR消息中发给PCRF/PCF设备。 |
| DHCP Server | 无特殊要求 | 可选网元。当用户需要从外置DHCP服务器获取P-CSCF地址时，需要部署DHCP Server。 |

## 对系统的影响

IMS业务恢复特性只对P-CSCF进行状态扫描，根据扫描结果判断是否给MS/UE发送P-CSCF地址列表更新消息，对系统影响可忽略。

## 应用限制

UDG 检测P-CSCF的状态仅能检测到传输层链路状态，无法检测到上层应用层故障。如果 UDG 与P-CSCF之间部署有防火墙，要求防火墙允许ICMP echo报文通过。
