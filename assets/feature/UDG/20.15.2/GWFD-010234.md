# Single IP

`GWFD-010234` · base · none

## 所属目录

- [GWFD-010220](feature/UDG/20.15.2/GWFD-010220.md)

## 适用性

- 适用NF：SGW-U、PGW-U、UPF
- 代际支持矩阵：`GGSN(2G&3G)=M; S/PGW-U(4G)=M; S/PGW-U(5G NSA)=M; UPF(5G)=M; NB-IoT=M`

## 适用NF（原文）

SGW-U、PGW-U、UPF

## 定义

UDG 的SingleIP特性提供以下两种能力：

- 不同类型接口的IP地址复用：UDG支持SGW-U/PGW-U/UPF融合部署，需要提供多种逻辑接口。为减少逻辑接口占用的IP地址数量，UDG引入SingleIP的概念，通过Saif/Paif/Scif/N4if接口来实现多种业务逻辑接口（如N3和S1-U、S5-p/S8-p和N9a、S5-s/S8-s和N9c等）只使用一个IP地址。
- 同一类型只配置一个接口：SingleIP支持一类逻辑接口只配置一个，使用一个IP地址，节省IP地址资源，简化组网规划和组网配置。

## 应用场景

在网络部署前， UDG 需要根据当前设备组网支持的逻辑网元形态、对应业务逻辑接口、每个接口的业务处理能力要求，合理分配当前的业务处理资源。本特性主要介绍设备业务处理资源划分所涉及的基本概念和主要划分原则，并提供典型的业务举例进行说明，用于指导完成不同组网模型下，逻辑网元划分、业务接口的资源处理分配。

## 原理概述

**业务逻辑接口**

如 [图1](#ZH-CN_CONCEPT_0124082251__fig133553281449) 所示， UDG 支持网络中定义的标准业务接口，用于实现与周边网元的互联互通。支持的标准接口有：S1-U、N3、S5/S8、N9、SGi /Gi 、N6、Sxa/Sxb、N4等。

UDG 提供简化配置的Saif/Paif/Scif/N4if接口，将网元同一侧的接口归一化为一个业务接口，其包括的逻辑接口如下：

- Saif抽象接口支持S1-U、N3功能，考虑到数据面组网的灵活性， UDG 可提供独立的S1-U、N3接口的配置。
- Paif抽象接口同时支持Gn、Gp、S5-p/S8-p、N9a功能， UDG 不提供独立的Gn、Gp、S5-p/S8-p、N9a接口的配置。
- Scif抽象接口支持S5-s/S8-s、N9c功能，考虑到数据面组网的灵活性，UDG可提供独立的S5-s/S8-s、N9c接口的配置。
- N4if抽象接口支持Sxa、Sxb和N4功能。

**逻辑接口表达方式**

UDG 引入SPU Group和SPU Instance概念，在SPU Group以及SPU Instance上，Saif、Paif和Scif所代表的逻辑接口不同。

逻辑接口配置在SPU Group上，称为组级逻辑接口。逻辑接口配置在SPU Instance上，称为Instance级逻辑接口。以Sa接口为例，其它逻辑接口的接口编号x/y/z与Sa接口的举例完全相同，配置含义如 [表1](#ZH-CN_CONCEPT_0124082251__tab4) 所示。

*表1 逻辑接口配置含义*

| 逻辑接口表达式 | 逻辑接口含义 |
| --- | --- |
| Interface Saif x/y/z | - x：逻辑接口所在的SPU Group号<br>- y：逻辑接口所在的SPU Instance号<br>- z：接口ID<br>说明：y的取值只有0和1。当y为0时，表示配置的接口为组级接口。当y为1时，表示配置的接口为Instance级接口。 |

例如：

- n4if 1/0/0表示：在SPU group 1上配置一个接口ID为0的组级N4接口。
- paif 1/1/0表示： 在SPU group 1的Instance1上配置一个接口ID为0的Instance级Pa接口。

**SPU Group**

SPU Group是一个逻辑概念。SPU Group内的所有SPU之间按照负荷分担原则激活。 UDG 只支持配置一个SPU Group。

**SPU Instance**

此概念仅用于和组级接口区分，是逻辑接口可以配置的最小单元，当前属于一个接口集合。SPU Instance和SPU group的地址生效范围相同，其中配置的接口在整个SPU group内生效。考虑到传输网络的负荷分担，数据面的地址都配置为Instance级接口（例如S1-u等）。 UDG 的激活用户会在多个同类的Instance级接口之间按照负荷分担原则选择使用。
