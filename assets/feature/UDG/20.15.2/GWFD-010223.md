# N9 接口

`GWFD-010223` · base · required

## 所属目录

- [GWFD-010220](feature/UDG/20.15.2/GWFD-010220.md)

## 适用性

- 适用NF：UPF
- 代际支持矩阵：`UPF(5G)=M`

## 适用NF（原文）

UPF

## 定义

N9接口为UPF（User Plane Function）和UPF之间的用户面接口，用于传递UPF间的上行、下行用户数据流。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | UPF的N9接口基于3GPP标准实现，可以实现与同厂家以及不同厂家的UPF间设备对接。 |
| 用户 | 用户不感知该特性。 |

## 应用场景

N9接口是UPF之间的用户面接口，用于传输UPF之间的用户面信息。组网结构如 [图1](#ZH-CN_TOPIC_0124822229__fig158415291417) 所示。

**图1** 5G系统服务化架构接口图

<br>

![](GWFD-010223 N9接口_24822229.assets/zh-cn_image_0221099951.png)

当网络中部署了多个UPF时，需要配置对应的N9接口。以下场景需要配置对应的N9接口：

- 本地I-UPF（Intermediate User Plane Function）和PSA UPF（PDU Session Anchor User Plane Function）之间。
- 用户漫游地I-UPF和归属地PSA UPF之间。
- ULCL UPF（Uplink Classifier User Plane Function）和PSA UPF之间。
- BP UPF（Branching Point User Plane Function）和PSA UPF之间。

本特性对系统无影响。

## 应用限制

- 配置N9接口类型的逻辑接口地址时，IP地址类型必须相同。
- 该接口传输GTP-U报文时携带的GTP-U扩展头总长度最多支持100字节，超出此长度报文会被丢弃。

## 原理概述

**N9** **接口协议栈**

N9接口可以作为UPF与UPF之间的用户面接口，用于在UPF与UPF之间传递上行、下行用户数据流。N9接口采用GTPv1-U协议，支持3GPP TS 29.281协议。对应的用户面接口协议栈如 [图2](#ZH-CN_TOPIC_0124822229__fig1263415401131) 所示。
