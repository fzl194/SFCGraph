# 基于APN的eMTC终端接入速率控制

`GWFD-110646` · protocol · required

## 所属目录

- [GWFD-110645](feature/UDG/20.15.2/GWFD-110645.md)

## 适用性

- 适用NF：SGW-U、PGW-U
- 代际支持矩阵：`S/PGW-U(4G)=M; S/PGW-U(5G NSA)=M`

## 适用NF（原文）

SGW-U、PGW-U

## 定义

对于eMTC (enhanced Machine Type Communication)业务，在正常场景下，终端发送的都是低频率数据包，但是一些场景下（如终端异常），终端可能在短时间高频率发送大量数据包，此时可能会导致网络拥塞并影响业务运行，所以3GPP协议引入终端发送数据的速率控制机制。基于APN的eMTC终端接入速率控制在PGW-C上配置速率门限，并在PGW-U上应用配置的速率门限，用于控制该UE的某PDN连接单位时间内上行和下行数据包的数量。

## 客户价值

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 避免由于终端异常而导致网络拥塞，有效提升整网的可靠性。 |
| 用户 | 提高普通终端用户业务的成功率和业务体验。 |

## 应用场景

本特性应用于eMTC控制面优化（Control Plane Optimization）和用户面优化（User Plane Optimization）数据传输中：

- 如果终端异常，可能在短时间高频率发送大量数据包，此时可能会导致网络拥塞并影响业务运行，限制终端的数据传输速率，在终端异常情况下可以防止网络拥塞而影响业务，提升整网的可靠性。
- 在极端多连接大流量场景下，基于信令面传输数据时，MME可能会因为流量过大导致异常，限制终端的数据传输速率可以有效避免该问题发生。
- 通过基于APN的eMTC速率控制可以对终端与网络的数据交互速率进行控制，从而节约了空口资源。

## 应用限制

用户在附着后短时间内发送一定的数据包后退网，当该用户再次入网时之前发送的数据包不会被 UDG 统计到单位时间内发送的数据包内， UDG 重新开始进行该用户的速率控制计时和计数。

## 原理概述

当eMTC终端接入基于APN进行速率控制时，根据eMTC本身特点，有两种场景：

- 控制面优化：在终端和MME之间通过NAS（Non-access Stratum）消息传输业务层数据，在MME和SGW-U之间通过S11-U接口传递业务层数据，无需反复建立和删除空口无线数据承载DRB（Data Radio Bearer）和S1-U连接，有效提升了数据传输效率，同时也可以降低对网络的信令负荷。
- 用户面优化：在UE、eNodeB和MME上保存恢复DRB和S1-U连接的S1AP（S1 Application Protocol）关联、UE上下文和承载上下文等数据，后续能快速恢复承载，降低对网络的信令负荷。

基于APN的eMTC终端接入速率控制是在PGW-C上配置速率门限，并在PGW-U上生效，对超过速率门限的数据包进行丢弃处理。PGW-C使用PCO/ePCO信元将基于APN的eMTC终端接入速率控制参数发送给UE，用于控制该UE的某PDN连接单位时间内上行数据包的数目。对于数据下行的速率控制，PGW-C则通过PFCP Session将速率控制参数下发给PGW-U。
