# 用户面地址自动检测

`GWFD-010108` · base · required

## 所属目录

- [GWFD-010100](feature/UDG/20.15.2/GWFD-010100.md)

## 适用性

- 适用NF：PGW-U、UPF
- 代际支持矩阵：`GGSN(2G&3G)=M; S/PGW-U(4G)=M; S/PGW-U(5G NSA)=M; UPF(5G)=M; NB-IoT=M`

## 适用NF（原文）

PGW-U、UPF

## 定义

用户面地址自动检测是 UDG 本地分配IP地址时， UDG 通过向PDN/DN侧服务器发送探测消息检查 UDG 到PDN/DN侧服务器之间的路由是否正常的功能。

## 客户价值

| **受益方** | **受益描述** |
| --- | --- |
| 运营商 | 该功能可以非常便利地调测<br>UDG<br>到PDN/DN侧服务器之间的路由连通性，可以帮助运营商快速进行业务调测、故障定位，减少业务访问异常引起的用户投诉和故障定位困难带来的运维成本。 |
| 用户 | 用户不感知该特性。 |

## 应用场景

运营商在进行业务调测、版本升级、地址扩容或有用户业务访问异常投诉时，需要检查 UDG 到PDN/DN侧服务器之间的路由连通性，可以使用该功能。

## 对系统的影响

本特性对系统无影响。

## 应用限制

- 本特性支持在UDG本地地址池分配UE IP地址的场景下使用。但是当UDG上部署内置NAT功能后，由于PGW-U/UPF无法感知外部服务器IP地址公网私网IP类型，因此有可能出现用户面IP地址探测失败的情况。该情况可以联系华为技术支持工程师处理。
- 本特性不支持在外部地址池（如SMF、RADIUS分配地址或静态地址）分配UE IP地址的场景下使用。
- 本特性不支持NAT地址转换。

## 原理概述

**探测方式**

UDG 支持PING、DNS和Tracert三种探测方式。

- PING探测方式
  PING（Packet Internet Groper，因特网包探索器）是测试网络端对端连接性的程序，通过给目的IP地址发送一个数据包，再判断对方是否返回一个同样大小的数据包来确定两端路由连通性。 UDG 在本地构造一个固定格式的ICMP（Internet Control Message Protocol）请求报文，并期待收到对端PDN/DN侧服务器对应的ICMP应答报文，请求报文和应答报文的对应关系可以通过报文中的sequence字段识别。如果能收到对应的应答报文，则判断路由连接正常，否则判断失败。
- DNS探测方式
  实际组网环境中，存在大多数防火墙会禁止PING报文的情况，采用上述PING探测方式导致探测会导致路由探测失败；此时可以选择DNS的探测方式。DNS（Domain Name System，域名系统）是提供域名与 IP 地址相互映射转换的一个分布式数据库，使用户能够方便的访问互联网，而不用去记住IP地址。DNS探测方式在 UDG 本地构造一个固定域名或指定域名的标准查询报文，发送给配置的目的DNS server IP地址。如果能收到对应的响应报文，则判断路由连接正常，否则判断失败。
- Tracert探测方式
  当网络中出现中故障，前面两种方探测方式探测失败时，可以进一步使用Tracert探测方式，查看网络中出现故障的位置，为故障诊断提供依据。Tracert（跟踪路由）是路由跟踪诊断程序，通过给目的IP发送不同IP生存时间 (TTL) 值的ICMP报文，路径上的每个路由器在转发数据包之前将数据包上的 TTL 递减 1，直到减为 0 时，给源头回应“ICMP已超时”或“目的端口不可达”的响应消息。
  当探测方式配置为Tracert方式时， UDG 在本地构造并发送一个TTL为1的探测数据包，收到中间第一跳路由器的“ICMP已超时”响应。随后每次发送的探测数据包TTL递增1，直到收到目的IP响应或TTL达到最大值（30）。通过检查中间路由器发回的“ICMP已超时”消息确定 UDG 到目的IP所经过的路由。由于探测报文指定的端口是一个在目的IP备没有应用的端口，当目的IP收到探测报文时会响应“目的端口不可达（ICMP port unreachable）”消息，说明Tracert探测结束。如果中间路由有故障，则故障路由器不会回复响应，根据Tracert探测上报的结果跳数可判断故障发生的位置。

**探测机制**

UDG 根据本地配置（ [**STR PDNROUTETST**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话连通性检测/网络侧连通性检测/PDN侧路由探测/启动PDN侧路由探测（STR PDNROUTETST）_70824402.md) ）信息（VPN、源IP地址、目的IP地址）构造探测报文，探测源IP地址是本地地址池网段范围内一个空闲地址，如果没有配置该参数，对于PING探测方式和DNS探测方式， UDG 从每一个路由网段内随机选取一个IP作为探测源IP；对于Tracert探测方式，由于探测时间稍长， UDG 随机选取一个地址网段的IP作为探测源IP，不遍历所有的地址网段，因此针对Tracert探测方式可以配置指定的探测源IP地址。

如果需要检查整个网络的路由连通性，可以进行全量探测。全量探测可以选择是否指定地址段号进行探测，如果指定地址段号进行全量探测，会针对此地址段下的所有路由网段触发探测；如果不指定地址段号进行全量探测，会针当前POOL内的所有地址段下的所有路由网段触发探测。Tracert探测方式不支持全量探测。

> **说明**
> - 探测源IP地址建议选取地址网段中的空闲地址，否则会导致在线用户被去激活。
> - 通过[**DSP ROUTE**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/IP服务/路由管理/路由基础/IPv4路由表/显示IPv4路由表（DSP ROUTE）_00441129.md)（IPv4路由表）/[**DSP ROUTE6**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/IP服务/路由管理/路由基础/IPv6路由表/查询IPv6路由表（DSP ROUTE6）_49961994.md)（IPv6路由表）查询探测前后的路由结果进行对比，防止探测影响业务正确的路由处理。

探测目的IP地址是PDN/DN侧的目的服务器IP地址，对于DNS探测方式，探测目的IP地址必须是DNS Server IP地址。

UDG 根据这些信息本地构造探测报文，给PDN/DN侧的目的服务器发送探测消息。如果能收到对端的响应，则判断路由连接正常；反之则路由探测失败。
