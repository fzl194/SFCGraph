## 与三场景关系

| 关系 | 说明 |
|---|---|
| **无 cross-feature 复用** | APN 域特性不依赖业务感知域 PCC/SA backbone；但 APN 会话管理与三场景的 URR/PCCPOLICYGRP 等无交叉 |
| **共享 IPFD-* 隧道** | 接入方式 4 个 GRE/IPSec/MPLS 特性属"通用 IP 特性"，与业务感知域无重叠；跨场景可能复用 VPN 网元（不入业务图谱，配置层自行处理） |
| **独立 BD** | APN 域在 `业务图谱体系/APN业务域/` 独立成库，`feature-knowledge/` + `three-layer-graph/` 已有；task wiki 沿用 `task/UDG/20.15.2/` + `task/UNC/20.15.2/` 路径（独立 BD 不另开目录） |

## 构建顺序建议

```
批次1（APN基础·会话管理根能力）→ 批次2-3（地址分配·IPv4/IPv6 全族）
  → 批次4（鉴权计费·UNC独有）→ 批次5-6（接入方式+网元选择+接入控制）
```

每批次遵循三场景已固化的 SOP：并行构建（UDG/UNC 同时）、benchmark-first（先建 backbone 1-00001/1-00002）、双向回填、R1 审视 per 批。
