# 配置IPv6路径MTU（ADD IPV6PATHMTU）

> 命令静态知识见 [ADD IPV6PATHMTU](command/UDG/20.15.2/ADD-IPV6PATHMTU.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD IPV6PATHMTU 记录一个 IPv6 目的地址的路径 MTU（PMTU），用于在该 VPN 范围内控制分片，提高报文传输效率。IPV6ADDRESS 引用已配的 [接口 IPv6 地址](task/UDG/20.15.2/0-00061.md)（ADD IFIPV6ADDRESS）；VRFNAME 引用上游 [L3VPN 实例](task/UDG/20.15.2/0-00071.md)（ADD L3VPNINST）。

**N6/Gi/SGi 双平面 IPv6 组网典型配法**（GWFD-020402，VPN_CN 下四组目的地址）

| VRFNAME | IPV6ADDRESS | PATHMTU |
|---|---|---|
| `VPN_CN` | `fc00:1000::1` | `1300` |
| `VPN_CN` | `fc00:2000::1` | `1300` |
| `VPN_CN` | `fc00:1000::2` | `1300` |
| `VPN_CN` | `fc00:2000::2` | `1300` |

**参数取值样例**

| 参数 | 取值样例 | 来源 | 必选 | 说明 |
|---|---|---|---|---|
| VRFNAME | `VPN_CN` | reference（← 0-00071 ADD L3VPNINST） | conditional（DP） | VPN 名称，可选值由 [决策点](#决策点) 定；省略则走默认 `_public_`（公网 PMTU）；可用 `LST L3VPNINST` 查询 |
| IPV6ADDRESS | `fc00:1000::1` 等 | reference（← 0-00061 ADD IFIPV6ADDRESS） | required | 已配的接口 IPv6 地址 |
| PATHMTU | `1300` | global_planned（全网规划） | required | 整数 1280~10000（byte） |

**典型脚本**：

```
ADD IPV6PATHMTU:VRFNAME="VPN_CN",IPV6ADDRESS="fc00:1000::1",PATHMTU=1300;
ADD IPV6PATHMTU:VRFNAME="VPN_CN",IPV6ADDRESS="fc00:2000::1",PATHMTU=1300;
ADD IPV6PATHMTU:VRFNAME="VPN_CN",IPV6ADDRESS="fc00:1000::2",PATHMTU=1300;
ADD IPV6PATHMTU:VRFNAME="VPN_CN",IPV6ADDRESS="fc00:2000::2",PATHMTU=1300;
```

**部署时序**：① ADD L3VPNINST 建 VPN（0-00071）→ ② ADD IFIPV6ADDRESS 配接口 IPv6 地址（0-00061）→ ③ ADD IPV6PATHMTU 记录路径 MTU（本 atom）。

## 决策点

**DP 0-00097 · IPV6PATHMTU VPN 作用域（VRFNAME 是否显式配）** — 命令真相 VRFNAME optional 默认 `_public_`，但 UDG 实际部署（020402）脚本均显式配 `VPN_CN` 用于在指定 VPN 范围记录 PMTU，构成配法分叉。

| 选项 | 含义 | 对 VRFNAME 的影响 |
|---|---|---|
| 显式配 VPN（默认） | 在指定 VPN 范围记录 PMTU | requiredness=required，显式传 `VPN_CN` 等 |
| 配默认公网 | 面向公网的 IPv6 PMTU | 不传，省略走默认 `_public_` |

## 约束

- **rule-0-00245**（warning）命令最大记录数 **300**（该命令规格显著小于多数接口/路由类命令）；执行后**立即生效**；VRFNAME 可用 `LST L3VPNINST` 查询 — 后果：超规格（>300）新增失败
- VRFNAME 引用必须已由 ADD L3VPNINST 建出；IPV6ADDRESS 必须已由 ADD IFIPV6ADDRESS 配置
