# 配置IPv6路径MTU（ADD IPV6PATHMTU）

> 命令静态知识见 [ADD IPV6PATHMTU](command/UDG/20.15.2/ADD-IPV6PATHMTU.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。用于 IPv6 报文按路径 MTU 控制分片，提高跨段传输效率。

## 配置方法

ADD IPV6PATHMTU 记录指定 VPN 内一条目的 IPv6 地址的路径 MTU。**IPV6ADDRESS 必选**（目的 IPv6 地址）、**PATHMTU 必选**（1280~10000 byte）、**VRFNAME 可选**（缺省走公网 `_public_`，UDG 实际部署 GWFD-020402 显式配 VPN_CN）。核心配置空间有 3 个维度：**VPN 作用域**（VRFNAME）、**目的 IPv6 地址**（IPV6ADDRESS）、**路径 MTU 取值**（PATHMTU）。1 个特性命中（GWFD-020402 N6/Gi/SGi IPv6 组网）使用 VRFNAME=VPN_CN、PATHMTU=1300、双平面四组目的地址。

### 配置维度 1：VPN 作用域（参数 VRFNAME，可选）

| 取值 | 作用 |
|---|---|
| **已建 L3VPN 实例名**（ref [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST） | PMTU 记录属于该 VPN（020402 范式 `VPN_CN`） |
| **省略 / 空** | 走系统默认公网 `_public_`，PMTU 作用于公网路径 |

> 1~31 字符；省略时命令使用默认 `_public_`；可用 `LST L3VPNINST` 查询已建实例。

### 配置维度 2：目的 IPv6 地址（参数 IPV6ADDRESS，必选）

| 取值 | 作用 |
|---|---|
| **已配的接口 IPv6 地址**（ref [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS） | 跨段路径的目的地址（020402 范式 `fc00:1000::1` / `fc00:2000::1` / `fc00:1000::2` / `fc00:2000::2`） |

> IPv6 地址类型；同一 VPN 内同一目的地址只能配一条；已有记录重复添加命令拒收。

### 配置维度 3：路径 MTU 取值（参数 PATHMTU，必选）

| 取值范围 | 作用 |
|---|---|
| **1280~10000 整数**（byte） | 该目的地址允许的最大不分片载荷；典型 1300 适配隧道场景 |

> PATHMTU 受 IPv6 MTU 上界 10000 约束，且 ≥ IPv6 协议允许最小值 1280；超出范围命令拒收。

**典型脚本（GWFD-020402 N6/Gi/SGi IPv6 组网，VPN_CN 双平面四组目的地址）**：
```
ADD IPV6PATHMTU:VRFNAME="VPN_CN",IPV6ADDRESS="fc00:1000::1",PATHMTU=1300;
ADD IPV6PATHMTU:VRFNAME="VPN_CN",IPV6ADDRESS="fc00:2000::1",PATHMTU=1300;
ADD IPV6PATHMTU:VRFNAME="VPN_CN",IPV6ADDRESS="fc00:1000::2",PATHMTU=1300;
ADD IPV6PATHMTU:VRFNAME="VPN_CN",IPV6ADDRESS="fc00:2000::2",PATHMTU=1300;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：记录 IPv6 PMTU"——前置 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST 建 VPN + [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS 配目的地址。

## 决策点：VPN 作用域显式配置（DP 0-00116）

问：本 PMTU 记录是配到指定 VPN 还是省略走默认公网？

| 选项 | VRFNAME | IPV6ADDRESS | PATHMTU | 适用场景 |
|---|---|---|---|---|
| **显式指定 VPN**（020402 范式） | 必填 `VPN_CN` 等 | 必填 | 必填 | 在指定 VPN 范围记录 PMTU，跨段防分片 |
| **省略走默认公网** | 不传（走 `_public_`） | 必填 | 必填 | 公网接口场景，不挂 VPN 实例 |

> 1 个特性命中（020402）；VRFNAME 可选但 UDG 实际部署均显式配；同 VPN 内同目的地址唯一。

## 约束

- **命令最大记录数 300**（warning，rule-0-00116）：该命令规格显著小于多数接口/路由类命令 — 后果：超规格（>300）新增失败
- **VRFNAME 可省略走默认公网**（info，rule-0-00116）：VRFNAME 可选，省略时记录在系统默认公网 `_public_` — 后果：未显式配 VPN 时 PMTU 在公网范围生效
- **IPV6ADDRESS 必须已配**（critical，rule-0-00116）：目的地址须已通过 `ADD IFIPV6ADDRESS` 在对应接口配置 — 后果：地址未配时命令拒收
- **VRFNAME 必须已建**（critical，rule-0-00116）：VRFNAME 必须已通过 `ADD L3VPNINST` 创建（若有指定） — 后果：VPN 不存在命令拒收
- **PATHMTU 范围约束**（warning，rule-0-00116）：取值 1280~10000 byte（IPv6 PMTU 上界） — 后果：超范围命令拒收
- **执行后立即生效**（warning，rule-0-00116）：本命令执行后立即生效，IPv6 报文分片按新 PMTU 走 — 后果：误配即刻影响跨段报文传输
