# 配置带宽管理业务（ADD BWMSERVICE）

> 命令静态知识见 [ADD BWMSERVICE](command/UDG/20.15.2/ADD-BWMSERVICE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD BWMSERVICE 是带宽控制骨架的"业务范围定义层"：把一条带宽管理业务用 TOS 分类、分类属性、协议名、协议组 4 类锚点之一标识，产出 BWMSERVICENAME 供 ADD BWMRULE(0-00037) 引用做业务匹配。命令核心配置空间有 3 个维度：**业务类型**（BWMSERVICETYPE 二选一：TOS / NONTOS）、**TOS 分类子分支**（TOSCFGTYPE 决定是 CLASS 类名还是 TOS_VALUE 数值）、**非 TOS 业务子分支**（NONTOSSRVTYPE 决定绑 CATEGORY_PROP / PROTOCOL / PROTOCOLGROUP 中的哪一类锚点）。3 个特性命中覆盖 TOS+CLASS、NONTOS+PROTOCOL、NONTOS+CATEGORY_PROP 三种主流组合。

### 配置维度 1：业务类型（参数 BWMSERVICETYPE，必选）

| 取值 | 作用 | 必选联动子分支 |
|---|---|---|
| **TOS** | TOS 类型业务（按报文的 TOS/DSCP 区分） | TOSCFGTYPE（CLASS / TOS_VALUE） |
| **NONTOS** | 非 TOS 类型业务（按 7 层/分类属性/协议组区分） | NONTOSSRVTYPE（CATEGORY_PROP / PROTOCOL / PROTOCOLGROUP） |

### 配置维度 2：TOS 分类子分支（参数 TOSCFGTYPE，条件必选 — BWMSERVICETYPE=TOS 时）

| 取值 | 作用 | 必选联动 |
|---|---|---|
| **CLASS** | 按 TOS 类别名（BE/AF1/AF2/.../CS7） | TOSCLASSTYPE |
| **TOS_VALUE** | 按 TOS 数值（0~63） | TOSVALUE |

| TOSCLASSTYPE 取值 | 含义 |
|---|---|
| **BE / AF1~AF4 / EF / CS1~CS7** | 8 个标准 PHB 类别（110311 演示 BE/AF1/AF2/CS7） |

### 配置维度 3：非 TOS 业务子分支（参数 NONTOSSRVTYPE，条件必选 — BWMSERVICETYPE=NONTOS 时）

| 取值 | 作用 | 必选联动 | 典型场景 |
|---|---|---|---|
| **CATEGORY_PROP** | 绑定分类属性（CATEPROPNAME） | CATEPROPNAME | 020354 / 110311（按业务分类做带宽控制） |
| **PROTOCOL** | 绑定协议名（PROTOCOLNAME） | PROTOCOLNAME | 110301（按协议：http）+ 110311（http/bt） |
| **PROTOCOLGROUP** | 绑定协议组（PROTGROUPNAME） | PROTGROUPNAME | 020354 协议组维度（命令参数齐全） |

### 可选：操作系统类型（参数 OSTYPE，BWMSERVICETYPE=NONTOS 时可选）

| 取值 | 作用 |
|---|---|
| **IOS / ANDROID / WIN / OTHER** | 进一步按操作系统区分（110301 演示 IOS） |

**典型脚本（020354 基于业务的 Shaping，TOS+TOS_VALUE）**：
```
ADD BWMSERVICE:BWMSERVICENAME="bwmservice_test",BWMSERVICETYPE=TOS,TOSCFGTYPE=TOS_VALUE,TOSVALUE=0;
```

**典型脚本（020354 基于业务的 Shaping，NONTOS+CATEGORY_PROP）**：
```
ADD BWMSERVICE:BWMSERVICENAME="bwmservice_test",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=CATEGORY_PROP,CATEPROPNAME="cateprop_test";
```

**典型脚本（110301 码率差异化，NONTOS+PROTOCOL+OSTYPE）**：
```
ADD BWMSERVICE:BWMSERVICENAME="bs_test",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=PROTOCOL,PROTOCOLNAME="http",OSTYPE=IOS;
```

**典型脚本（110311 业务感知带宽控制，TOS+CLASS 多档）**：
```
ADD BWMSERVICE:BWMSERVICENAME="bs_test1",BWMSERVICETYPE=TOS,TOSCFGTYPE=CLASS,TOSCLASSTYPE=BE;
ADD BWMSERVICE:BWMSERVICENAME="bs_test2",BWMSERVICETYPE=TOS,TOSCFGTYPE=CLASS,TOSCLASSTYPE=AF1;
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；CATEGORY_PROP 引用的 CATEPROPNAME 须先于 BWMSERVICE 建（[0-00032](task/UDG/20.15.2/0-00032.md)）；PROTOCOLNAME 须先建协议（命令 wiki 完整参数表）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：业务范围定义"——前置 ADD CATEGORYPROP(0-00032)（如选 CATEGORY_PROP）/ 协议定义（如选 PROTOCOL）；产出 BWMSERVICENAME 被 ADD BWMRULE(0-00037) 引用做业务匹配。

## 决策点：业务类型与子分支组合（DP 0-00034）

问：ADD BWMSERVICE 选哪个业务类型 + 子分支组合？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| TOS + CLASS | + TOSCLASSTYPE（BE/AF1~4/EF/CS1~7） | 按标准 PHB 类别做带宽控制（110311 多档） |
| TOS + TOS_VALUE | + TOSVALUE（0~63） | 按 TOS 数值精确区分（020354 范式） |
| NONTOS + CATEGORY_PROP | + CATEPROPNAME | 按"业务分类属性"做带宽控制（020354 / 110311） |
| NONTOS + PROTOCOL | + PROTOCOLNAME [+ OSTYPE] | 按协议做带宽控制（110301/110311，OSTYPE 进一步按 OS 区分） |
| NONTOS + PROTOCOLGROUP | + PROTGROUPNAME | 按协议组做带宽控制（020354 范式） |

> 选项不互斥：同一 feature 可建多条不同 BWMSERVICE 覆盖不同业务。

## 约束

- **BWMSERVICETYPE 二选一后必入子分支**（critical，rule-0-00334）：选 TOS 必须配 TOSCFGTYPE + TOSCLASSTYPE/TOSVALUE；选 NONTOS 必须配 NONTOSSRVTYPE + CATEPROPNAME/PROTOCOLNAME/PROTGROUPNAME — 后果：缺子分支命令拒收
- **CATEGORY_PROP 引用前置依赖**（critical，rule-0-00334）：配 NONTOSSRVTYPE=CATEGORY_PROP 时 CATEPROPNAME 须先建（[0-00032](task/UDG/20.15.2/0-00032.md)） — 后果：未建引用失败
- **PROTOCOL 引用前置依赖**（critical，rule-0-00334）：配 NONTOSSRVTYPE=PROTOCOL 时 PROTOCOLNAME 须先建 — 后果：未建引用失败
- **OSTYPE 仅 NONTOS 配时生效**（warning，rule-0-00334）：BWMSERVICETYPE=TOS 时 OSTYPE 无意义 — 后果：误配拒收
- **BWMSERVICENAME 全网唯一**（critical，隐含）：不同业务之间 BWMSERVICENAME 不能重复 — 后果：重名 ADD 失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
