# 配置带宽管理业务（ADD BWMSERVICE）

> 命令静态知识见 [ADD BWMSERVICE](command/UDG/20.15.2/ADD-BWMSERVICE.md)

## 配置方法

定义一条带宽管理业务范围，由 BWMSERVICETYPE（TOS/NONTOS）+ NONTOSSRVTYPE（分类属性/协议组/协议/协议+OS）共同决定业务区分维度。UDG 实战主要走 NONTOS 四分支。

| 场景分支 | BWMSERVICETYPE | NONTOSSRVTYPE | 配套参数 | 样例 |
|---|---|---|---|---|
| 分类属性（URL整形/PCC预定义） | NONTOS | CATEGORY_PROP | CATEPROPNAME="cateprop_test" | 关联 ADD CATEGORYPROP |
| 协议组 | NONTOS | PROTOCOLGROUP | PROTGROUPNAME="protgroup_test" | 关联 ADD PROTOCOLGROUP |
| 协议 | NONTOS | PROTOCOL | PROTOCOLNAME="http" | 七层协议名 |
| 协议+操作系统（终端差异化） | NONTOS | PROTOCOL | PROTOCOLNAME="http",OSTYPE=IOS | 110301 专用 |
| TOS（按业务类型区分） | TOS | - | TOSCFGTYPE=CLASS,TOSCLASSTYPE=BE | 走 TOS 分支 |

**BWMSERVICENAME** 跨场景统一规划（如 `bs_test`/`bwmservice_test`），被 BWMRULE 引用。

**典型脚本**：

```
# 分类属性分支（020354 URL整形）
ADD BWMSERVICE:BWMSERVICENAME="bwmservice_test",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=CATEGORY_PROP,CATEPROPNAME="cateprop_test";
# 协议+OS分支（110301 终端系统码率差异）
ADD BWMSERVICE:BWMSERVICENAME="bs_test",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=PROTOCOL,PROTOCOLNAME="http",OSTYPE=IOS;
```

**关键约束**：PCC 预定义规则场景下，BWMSERVICE 的 CATEPROPNAME 必须与 ADD RULE 的 POLICYNAME 规划一致（跨 PCF/PCRF/UNC/UDG 四网元同名）。

**步骤位置**：BWM 三件套首步——先建业务范围，再建 Controller，最后 BWMRULE 把两者绑进用户组。

## 决策点

**DP 0-00099**：BWMSERVICE 业务类型 4 路分叉（CATEGORY_PROP/PROTOCOLGROUP/PROTOCOL/PROTOCOL+OSTYPE），互斥单选，决定 NONTOSSRVTYPE 取值与配套必选参数。

| 选项 | NONTOSSRVTYPE | 必选配套 | 不传 |
|---|---|---|---|
| CATEGORY_PROP（默认 PROTOCOL+OSTYPE） | CATEGORY_PROP | CATEPROPNAME | PROTGROUPNAME/PROTOCOLNAME/OSTYPE |
| PROTOCOLGROUP | PROTOCOLGROUP | PROTGROUPNAME | 其余 absent |
| PROTOCOL | PROTOCOL | PROTOCOLNAME | 其余 absent |
| PROTOCOL+OSTYPE | PROTOCOL | PROTOCOLNAME+OSTYPE | 其余 absent |
