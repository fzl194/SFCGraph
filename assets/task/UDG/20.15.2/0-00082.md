# 创建静态BFD会话(IPv4)（ADD BFDSESSION）

> 命令静态知识见 [ADD BFDSESSION](command/UDG/20.15.2/ADD-BFDSESSION.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD BFDSESSION 是 GWFD-010155 静态路由+BFD 组网独有命令（缺省路由组网/OSPF 组网不含），用于创建 IPv4 静态非自协商 BFD 会话供 ADD SRROUTE 静态路由绑定。单特性 9 个参数已绑全，15 个未绑参数（IPv6/单臂 Echo/源地址类/MTU 等）当前 IPv4 静态场景不涉及，MINTXINT/MINRXINT/DETECTMULTI/WTRTIMERINT/TOSEXP 均默认(200/200/3/0/7)不显式配，不绑。

**取值范式（从证据数据规划表归纳）**

| 参数 | 取值 | 说明 |
|---|---|---|
| SESSNAME | BfdName1（本端规划） | 会话名称，作为 [配置IPv4静态路由](task/UDG/20.15.2/0-00062.md) SESSIONNAME 的引用源 |
| ADDRTYPE | IPv4（固定） | IP 协议版本，当前场景固定 IPv4 |
| CREATETYPE4 | SESS_STATIC（固定） | IPv4 静态非自协商会话 |
| ONEARMECHO | FALSE（固定） | 非单臂 Echo |
| LINKTYPE | IP（固定） | IP 链路 |
| LOCALDISCR | 1（本端规划） | 本地标识符，**须等于对端 REMOTEDISCR** |
| REMOTEDISCR | 2（本端规划） | 远端标识符，**须等于对端 LOCALDISCR** |
| DESTADDR4 | 192.168.94.1（本端规划） | 对端 IPv4 地址 |
| VRFNAME | vpn-s2b（已配置数据中获取） | reference 引用 [创建L3VPN实例](task/UDG/20.15.2/0-00071.md) |

**脚本样例**
```
ADD BFDSESSION:SESSNAME="BfdName1",ADDRTYPE=IPv4,CREATETYPE4=SESS_STATIC,ONEARMECHO=FALSE,DESTADDR4="192.168.94.1",LINKTYPE=IP,VRFNAME="vpn-s2b",LOCALDISCR=1,REMOTEDISCR=2;
```

**关键规则**
- **固定参数多**：ADDRTYPE/CREATETYPE4/ONEARMECHO/LINKTYPE 四参单特性固定无分叉，binding=fixed。
- **标识符交叉配对**：静态非自协商会话两端须提前规划好描述符 ID——本端 LOCALDISCR=对端 REMOTEDISCR，本端 REMOTEDISCR=对端 LOCALDISCR。
- **MINTXINT/MINRXINT/DETECTMULTI/WTRTIMERINT/TOSEXP 不绑**：md 明示"可不输使用默认值"，单特性脚本不显式配。
- **执行位置**：在 SET BFD 全局使能（0-00073）之后、ADD SRROUTE（0-00062）之前。

## 约束

- **rule 0-00186**（dependency/error）使用 ADD BFDSESSION 创建静态 BFD 会话前，必须已通过 SET BFD 命令全局使能 BFD 功能(BFDENABLE=TRUE)，否则会话不生效 — 后果：未全局使能 BFD 时创建的静态会话无法生效 / 检测功能缺失
- **rule 0-00187**（capacity/warning）该命令在非 NP 卡场景和 NP 卡非加速模式场景最大记录数为 7200，在 NP 卡加速模式场景最大记录数为 4096 — 后果：超过场景上限后新增 BFD 静态会话失败
- **rule 0-00188**（consistency/error）当 CREATETYPE4=SESS_STATIC：静态非自协商会话两端须提前规划好描述符 ID，本端 LOCALDISCR 须等于对端 REMOTEDISCR，本端 REMOTEDISCR 须等于对端 LOCALDISCR（两端标识符交叉配对） — 后果：本地/远端标识符未交叉配对导致 BFD 会话无法 Up、检测失效
