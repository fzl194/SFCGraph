# 创建静态BFD会话（ADD BFDSESSION）

> 命令静态知识见 [ADD BFDSESSION](command/UDG/20.15.2/ADD-BFDSESSION.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令是 BFD 配置 2 件套之二**（会话级/双端）——前置 [0-00073](task/UDG/20.15.2/0-00073.md) SET BFD:BFDENABLE=TRUE 全局使能；可为 [0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE 静态路由 BFDENABLE=TRUE 提供 SESSIONNAME。

## 配置方法（配置方法字典）

ADD BFDSESSION 是"**创建 BFD 静态会话（IPv4 / IPv6）+ 静态非自协商 / 静态自协商 / 单臂 Echo**"命令。SESSNAME 必选（作为后续 ADD SRROUTE 的 SESSIONNAME 引用）；ADDRTYPE 必选（IPv4/IPv6）；**静态会话（CREATETYPE=SESS_STATIC）必须同时配 LOCALDISCR 和 REMOTEDISCR**（两端交叉配对）。核心配置空间有 6 个维度：**会话标识**（SESSNAME + DESCRIPTION 描述信息）、**协议版本**（ADDRTYPE IPv4/IPv6 门控 CREATETYPE4/6 + DESTADDR4/6 + SRCADDR4/6）、**会话类型**（CREATETYPE4/6 静态/自协商）、**Echo 模式**（ONEARMECHO/6 FALSE/TRUE）、**会话参数**（MINTXINT/MINRXINT/MINECHORXINT/DETECTMULTI/WTRTIMERINT/TOSEXP/ADMINDOWN/PROCESSPST/LINKTYPE）、**标识符 + 链路端点**（LOCALDISCR/REMOTEDISCR/DESTADDR/SRCADDR/IFNAME/VRFNAME）。

### 配置维度 1：会话标识（参数 SESSNAME 必选 + DESCRIPTION 可选）

| 参数 | 必选性 | 取值范围 | 备注 |
|---|---|---|---|
| **SESSNAME** | 必选 | 字符串 1~64 字符（不区分大小写，不支持空格） | 会话名称，作为 [0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE 的 SESSIONNAME 引用源 |
| **DESCRIPTION** | 可选 | 字符串 1~51 字符（区分大小写，支持空格） | 描述信息 |

### 配置维度 2：协议版本（参数 ADDRTYPE，必选，IPv4/IPv6 门控）

ADDRTYPE 决定后续地址参数走 IPv4 链路还是 IPv6 链路：

| 取值 | 联动必选参数 | 适用场景 |
|---|---|---|
| **IPv4** | DESTADDR4 + 视 CREATETYPE4 配 SRCADDR4 + LOCALDISCR/REMOTEDISCR | GWFD-010155 静态+BFD 组网（192.168.94.1）；ADD SRROUTE BFD 联动主流场景 |
| **IPv6** | DESTADDR6 + 视 CREATETYPE6 配 SRCADDR6 + LOCALDISCR/REMOTEDISCR | IPv6 静态+BFD 组网 |

### 配置维度 3：会话类型（参数 CREATETYPE4/CREATETYPE6，条件必选）

CREATETYPE 决定是静态会话还是静态自协商会话：

| 取值 | 适用场景 | 联动 |
|---|---|---|
| **SESS_STATIC**（静态会话） | 已明确被检测链路的链路信息 + 两端描述符 ID 已提前规划 | 必配 LOCALDISCR + REMOTEDISCR + ONEARMECHO/6 |
| **SESS_AUTO**（静态自协商会话） | **必须配置源 IP 地址**（SRCADDR4/SRCADDR6）；会话参数 MINTXINT/MINRXINT/DETECTMULTI 等可选 | 用于自动协商场景 |

> **关键约束**：**静态自协商会话必须配置源 IP 地址（SRCADDR4 或 SRCADDR6）**。

### 配置维度 4：Echo 模式（参数 ONEARMECHO/ONEARMECHO6，条件必选）

ONEARMECHO 决定是否为单臂 Echo BFD：

| 取值 | 适用场景 | 联动 |
|---|---|---|
| **FALSE** | 双臂 BFD：探测报文 src=SRCADDR4/6, dst=DESTADDR4/6；REMOTEDISCR 条件必选 | 主流场景（GWFD-010155 双臂 BFD） |
| **TRUE** | 单臂 Echo BFD：探测报文 src=dst=DESTADDR4/6；在支持 BFD 的一端建会话，对端不支持 BFD 时环回；**IFNAME 必选**；不支持 BFD 端须禁止 URPF，否则单臂 ECHO 无法生效 | 单端部署 |

> **LST INTERFACE** 查看 IFNAME 接口名。

### 配置维度 5：会话参数（可选参数集）

| 参数 | 取值范围 | 默认值 | 触发条件 | 作用 |
|---|---|---|---|---|
| **MINTXINT** | 30~1000 ms | 200 | CREATETYPE4/6=AUTO 或 ONEARMECHO=FALSE | 最小发送间隔 |
| **MINRXINT** | 30~1000 ms | 200 | CREATETYPE4/6=AUTO 或 ONEARMECHO=FALSE | 最小接收间隔 |
| **MINECHORXINT** | 30~1000 ms | 200 | ONEARMECHO/6=TRUE | 单臂 Echo 接收间隔 |
| **DETECTMULTI** | 3~50 | 3 | 可选 | 检测倍数；检测周期=对端 DETECTMULTI × MAX(对端发送间隔, 本端接收间隔) |
| **WTRTIMERINT** | 0~60 min | 0 | 可选 | 等待恢复时间（防止主备切换震荡；两端需配相同） |
| **TOSEXP** | 0~7 | 7 | CREATETYPE4/6=STATIC | 报文优先级（值越大优先级越高） |
| **ADMINDOWN** | TRUE/FALSE | FALSE | 可选 | 管理 Down（手工置 Down，不影响业务） |
| **PROCESSPST** | TRUE/FALSE | FALSE | 可选 | 操作 PST 表（每个接口只允许 1 个 BFD 会话配此参数） |
| **LINKTYPE** | IP | — | 必选 | 链路类型（当前仅 IP） |

> **典型配置不显式传 MINTXINT/MINRXINT/DETECTMULTI/WTRTIMERINT/TOSEXP**——按缺省值 200ms/200ms/3/0/7 处理。

### 配置维度 6：标识符 + 链路端点

| 参数 | 必选性 | 取值范围 | 联动 |
|---|---|---|---|
| **LOCALDISCR** | CREATETYPE=SESS_STATIC 必选 | 整数 1~16384 | **本端 LOCALDISCR = 对端 REMOTEDISCR**（交叉配对） |
| **REMOTEDISCR** | ONEARMECHO=FALSE 必选 | 整数 1~4294967295 | **本端 REMOTEDISCR = 对端 LOCALDISCR**（交叉配对） |
| **DESTADDR4** | ADDRTYPE=IPv4 必选 | IPv4 地址 | 双臂会话 dst；单臂会话 src=dst |
| **SRCADDR4** | CREATETYPE4=SESS_AUTO 必选；ONEARMECHO TRUE/FALSE 可选 | IPv4 地址 | 通常不配置（系统从出接口选）；URPF 共存时需手工配 |
| **DESTADDR6** | ADDRTYPE=IPv6 必选 | IPv6 地址 | IPv6 双臂 dst |
| **SRCADDR6** | CREATETYPE6=SESS_AUTO 必选；ONEARMECHO6 TRUE/FALSE 可选 | IPv6 地址 | IPv6 源（不配时系统从出接口选） |
| **IFNAME** | ONEARMECHO/6=TRUE 必选；CREATETYPE4/6=AUTO 可选 | 字符串 1~63 | 出接口名称（LST INTERFACE 查看） |
| **VRFNAME** | 可选 | 字符串 1~31（区分大小写，不支持空格） | 会话绑定 VPN 实例（LST L3VPNINST 查看） |

**典型脚本（GWFD-010155 静态+BFD IPv4 组网，非单臂双臂静态会话）**：
```
ADD BFDSESSION: SESSNAME="BfdName1", ADDRTYPE=IPv4, CREATETYPE4=SESS_STATIC, ONEARMECHO=FALSE, DESTADDR4="192.168.94.1", LINKTYPE=IP, VRFNAME="vpn-s2b", LOCALDISCR=1, REMOTEDISCR=2;
```

> 命令 notes：**最大记录数**：非 NP 卡场景和 NP 卡非加速模式场景 7200；NP 卡加速模式场景 4096；**执行后立即生效**；**IPv4 链路 MTU 建议 >52、IPv6 链路 MTU 建议 >72 防止 BFD 震荡**；**静态自协商会话必须配置源 IP 地址**；**使用本命令前必须通过 [0-00073](task/UDG/20.15.2/0-00073.md) SET BFD 命令全局使能 BFD 功能**；**单臂 BFD 模式下不支持 BFD 端须禁止 URPF**；**单臂 BFD 会话 Up 后修改对端直连接口 IP 状态不可预料**——需先删后建。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N+1：创建 BFD 会话"——前置 [0-00073](task/UDG/20.15.2/0-00073.md) SET BFD:BFDENABLE=TRUE 全局使能；下游 [0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE SESSIONNAME 引用。

## 决策点：BFD 会话类型选择（DP 0-00082）

问：ADD BFDSESSION 的 CREATETYPE / ONEARMECHO 取何组合？

| 选项 | 组合 | 联动必配参数 | 适用场景 |
|---|---|---|---|
| **IPv4 静态非单臂双臂**（default） | ADDRTYPE=IPv4 + CREATETYPE4=SESS_STATIC + ONEARMECHO=FALSE | LOCALDISCR + REMOTEDISCR + DESTADDR4 + LINKTYPE | GWFD-010155 静态路由 BFD 联动（主流范式） |
| **IPv4 单臂 Echo** | ADDRTYPE=IPv4 + CREATETYPE4=SESS_STATIC + ONEARMECHO=TRUE | LOCALDISCR + DESTADDR4 + IFNAME + VRFNAME（REMOTEDISCR 条件可选） | 对端不支持 BFD 时环回 |
| **IPv4 静态自协商** | ADDRTYPE=IPv4 + CREATETYPE4=SESS_AUTO | SRCADDR4 + DESTADDR4 | 自动协商场景 |
| **IPv6 静态非单臂双臂** | ADDRTYPE=IPv6 + CREATETYPE6=SESS_STATIC + ONEARMECHO6=FALSE | LOCALDISCR + REMOTEDISCR + DESTADDR6 | IPv6 链路 BFD 联动 |
| **IPv6 单臂 Echo** | ADDRTYPE=IPv6 + CREATETYPE6=SESS_STATIC + ONEARMECHO6=TRUE | LOCALDISCR + DESTADDR6 + IFNAME | IPv6 对端不支持 BFD |
| **IPv6 静态自协商** | ADDRTYPE=IPv6 + CREATETYPE6=SESS_AUTO | SRCADDR6 + DESTADDR6 | IPv6 自动协商 |

> 默认场景（010155 静态路由 BFD 联动）必配 LOCALDISCR+REMOTEDISCR；单臂 Echo 必配 IFNAME。

## 约束

- **最大记录数 7200/4096**（warning，rule-0-00482）：非 NP 卡场景和 NP 卡非加速模式场景最大记录数 7200；NP 卡加速模式场景最大记录数 4096 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00482）：命令执行后立即生效 — 后果：误操作即刻生效
- **必须前置 SET BFD:BFDENABLE=TRUE**（critical，rule-0-00482）：使用此命令前必须通过 [0-00073](task/UDG/20.15.2/0-00073.md) SET BFD 命令全局使能 BFD 功能 — 后果：未全局使能时创建的静态会话无法生效
- **静态会话标识符交叉配对**（critical，rule-0-00482）：静态非自协商会话两端须提前规划好描述符 ID——本端 LOCALDISCR = 对端 REMOTEDISCR，本端 REMOTEDISCR = 对端 LOCALDISCR — 后果：未交叉配对致 BFD 会话无法 Up / 检测失效
- **静态自协商必配源 IP**（critical，rule-0-00482）：静态自协商会话必须配置源 IP 地址（SRCADDR4 或 SRCADDR6） — 后果：未配 SRCADDR 致会话建立失败
- **单臂 BFD 禁 URPF**（critical，rule-0-00482）：在创建单臂模式 ECHO 功能之前，应在不支持 BFD 的一端禁止使用 URPF — 后果：未禁 URPF 单臂 ECHO 无法生效
- **单臂 BFD 改对端 IP 不可预料**（warning，rule-0-00482）：单臂 BFD 会话 Up 后修改对端直连接口 IP 状态无法预料，建议先删后建 — 后果：直接修改对端直连口 IP 后单臂 BFD 可能变 Down
- **IPv4/IPv6 链路 MTU 防震荡**（warning，rule-0-00482）：BFD 检测链路接口 MTU 建议 IPv4 >52、IPv6 >72 — 后果：MTU 过小可能致 BFD 震荡
- **PST 每接口限 1 个会话**（info，隐含）：PROCESSPST=TRUE 时每个接口只允许 1 个 BFD 会话配置 — 后果：违反致配置冲突
- **REMOTEDISCR 与 ONEARMECHO 联动**（info，隐含）：ONEARMECHO=FALSE 时 REMOTEDISCR 必选；ONEARMECHO=TRUE 时 REMOTEDISCR 条件可选 — 后果：双臂会话缺 REMOTEDISCR 命令拒收
- **WTR 两端需一致**（warning，rule-0-00482）：使用 WTR 时需两端配置相同 — 后果：两端不一致会致应用程序感知的 BFD 状态不一致
