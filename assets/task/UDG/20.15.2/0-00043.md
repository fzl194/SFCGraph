# 配置地址池（ADD POOL）

> 命令静态知识见 [ADD POOL](command/UDG/20.15.2/ADD-POOL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

ADD POOL 是地址池族链路的"池定义层"：建立地址池对象（POOLNAME），地址段由 ADD SECTION(0-00044) 补充，归属关系由 ADD POOLBINDGROUP(0-00045) 绑到地址池组，APN 入口由 ADD POOLGRPMAP(0-00047) 完成映射。命令核心配置空间有 5 个维度：**地址池类型**（POOLTYPE 决定地址来源/必选联动参数）、**IP 版本**（IPVERSION 决定 IPv4/IPv6 后续参数组）、**VPN 绑定**（HASVPN+VPNINSTANCE）、**冗余/白名单**（EXTERNAL 池下 REDUNDFUNC+MASTERFLAG 静态路由冗余 / CHECKIPVALID 静态白名单）、**本地池租约**（LOCAL 池下 IPLEASE+RELEASETIME+SINGLEIPFLAG）。6 个特性命中覆盖 010104/010105/010107/020403/020406/020421，POOLTYPE 在 EXTERNAL 与 LOCAL 间分叉。

### 配置维度 1：地址池类型（参数 POOLTYPE，决定地址来源与必选联动参数集）

| 取值 | 作用 | 必选联动 | 适用场景 |
|---|---|---|---|
| **EXTERNAL**（默认） | 地址由 RADIUS/BRAS/DHCP 或静态分配；UDG 仅发布路由 | IPVERSION（可选） + HASVPN+VPNINSTANCE | 地址分配方式、静态地址用户路由冗余、Prefix Delegation |
| **LOCAL** | UDG 自维护地址段、为用户分配本地动态地址；必须配 ADD SECTION 提供地址段 | IPVERSION（强烈建议）+ HASVPN+VPNINSTANCE + ADD SECTION(0-00044) | IPv4v6 双栈、基于位置分配、入不转板 |

### 配置维度 2：IP 版本（参数 IPVERSION，EXTERNAL/LOCAL 均可选）

| 取值 | 作用 | 配套地址段参数 |
|---|---|---|
| **IPV4** | IPv4 地址池 | V4STARTIP/V4ENDIP（ADD SECTION） |
| **IPV6** | IPv6 地址池 | V6PREFIXSTART/V6PREFIXEND/V6PREFIXLENGTH（ADD SECTION） |
| 不配 | 继承默认（IPV4） | V4*（ADD SECTION） |

> 双栈场景按"IPv4 + IPv6 各建一池"配（020403 范式：`testpool`+`testpoolv6`）。

### 配置维度 3：VPN 绑定（参数 HASVPN+VPNINSTANCE，可选/条件必选）

| 取值 | 作用 | 配套必选 |
|---|---|---|
| **HASVPN=DISABLE** | 不绑 VPN | — |
| **HASVPN=ENABLE** | 绑 VPN | VPNINSTANCE 必填，引用 [ADD VPNINST](task/UDG/20.15.2/0-00041.md) |

> 三者一致性约束：APN / 地址池 / SGi-N6 外联口**必须同 VPN**——见约束 rule-0-00443。

### 配置维度 4：路由冗余/白名单（参数 REDUNDFUNC/MASTERFLAG/CHECKIPVALID，仅 EXTERNAL 池可选）

| 参数 | 取值 | 适用场景 |
|---|---|---|
| **REDUNDFUNC** | ENABLE（开静态地址用户路由冗余）/ DISABLE（默认） | 静态地址用户路由冗余（010107 范式） |
| **MASTERFLAG** | ENABLE（主用 UPF）/ DISABLE（备用） | REDUNDFUNC=ENABLE 时条件可选（010107 范式：主用 MASTERFLAG=ENABLE） |
| **CHECKIPVALID** | ENABLE / DISABLE | Prefix Delegation（020406 范式：IPV6 静态地址白名单检测） |

> REDUNDFUNC 需配全局开关 [SET REDUNDUSER](task/UDG/20.15.2/0-00064.md) 一起使用（010107 全局开关+本池开关成对出现）。

### 配置维度 5：本地池租约（参数 IPLEASE/RELEASETIME/SINGLEIPFLAG/ALARMSWITCH/IMSSW，仅 LOCAL 池可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **IPLEASE** | ENABLE / DISABLE | 是否使能 IP 地址租约功能 |
| **RELEASETIME** | 整数（秒） | 用户释放后对应 IP 等待期；**过大为高危**（地址延迟释放→池地址不足→激活失败） |
| **SINGLEIPFLAG** | YES / NO | 是否小地址池（每用户一地址） |
| **ALARMSWITCH** | ENABLE / DISABLE | 池内 ALM-812... 告警使能 |
| **IMSSW** | ENABLE / DISABLE | IP 租约是否对 IMS 用户有效 |

**典型脚本（010104 EXTERNAL 最小配置）**：
```
ADD POOL:POOLNAME="testpool",POOLTYPE=EXTERNAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
```

**典型脚本（020406 EXTERNAL+IPV6+白名单）**：
```
ADD POOL:POOLNAME="testpool",POOLTYPE=EXTERNAL,IPVERSION=IPV6,CHECKIPVALID=ENABLE,HASVPN=ENABLE,VPNINSTANCE="vpn1";
```

**典型脚本（010107 EXTERNAL+路由冗余主备，4 池）**：
```
ADD POOL: POOLNAME="pool1", POOLTYPE=EXTERNAL, IPVERSION=IPV4, HASVPN=ENABLE, VPNINSTANCE="vpn1", REDUNDFUNC=ENABLE;
ADD POOL: POOLNAME="pool2", POOLTYPE=EXTERNAL, IPVERSION=IPV6, HASVPN=ENABLE, VPNINSTANCE="vpn1", REDUNDFUNC=ENABLE;
ADD POOL: POOLNAME="pool3", POOLTYPE=EXTERNAL, IPVERSION=IPV4, HASVPN=ENABLE, VPNINSTANCE="vpn1", REDUNDFUNC=ENABLE;
ADD POOL: POOLNAME="pool4", POOLTYPE=EXTERNAL, IPVERSION=IPV6, HASVPN=ENABLE, VPNINSTANCE="vpn1", REDUNDFUNC=ENABLE;
```

**典型脚本（020403 LOCAL 双栈）**：
```
ADD POOL:POOLNAME="testpool",POOLTYPE=LOCAL,IPVERSION=IPV4,HASVPN=ENABLE,VPNINSTANCE="VRF_Internet";
ADD POOL:POOLNAME="testpoolv6",POOLTYPE=LOCAL,IPVERSION=IPV6,HASVPN=ENABLE,VPNINSTANCE="VRF_Internet";
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；POOLTYPE=LOCAL+RELEASETIME 过大为高危命令。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：地址池定义"——前置无；产出 POOLNAME 被 [0-00044](task/UDG/20.15.2/0-00044.md)（ADD SECTION 配地址段）、[0-00045](task/UDG/20.15.2/0-00045.md)（ADD POOLBINDGROUP 绑地址池组）、[0-00047](task/UDG/20.15.2/0-00047.md)（ADD POOLGRPMAP 建 APN 映射）引用。

## 决策点：地址池类型与子开关组合（DP 0-00043）

问：ADD POOL 选哪个 POOLTYPE + 必选子参数集组合？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| EXTERNAL 最小 | POOLTYPE=EXTERNAL + HASVPN+VPNINSTANCE | 地址分配方式（010104 范式） |
| EXTERNAL+IPV6 白名单 | POOLTYPE=EXTERNAL + IPVERSION=IPV6 + CHECKIPVALID=ENABLE + HASVPN+VPNINSTANCE | IPv6 Prefix Delegation（020406 范式） |
| EXTERNAL+路由冗余 | POOLTYPE=EXTERNAL + IPVERSION=IPV4/IPV6 + HASVPN+VPNINSTANCE + REDUNDFUNC=ENABLE [+ MASTERFLAG=ENABLE] | 静态地址用户路由冗余（010107 范式，主备各 2 池=4 池） |
| LOCAL 双栈 | POOLTYPE=LOCAL + IPVERSION=IPV4/IPV6（各一池）+ HASVPN+VPNINSTANCE | IPv4v6 双栈接入（020403 范式） |
| LOCAL 多池 | POOLTYPE=LOCAL + HASVPN+VPNINSTANCE（同 VPNINST 多池） | SMF 分配（010105）/ Prefix Delegation（020406）/ 基于位置（020421） |

> 6 个特性命中在 EXTERNAL（010104/010107/020406）与 LOCAL（010105/020403/020406/020421）间分叉；同 feature 可建多池（如主备各 2 池=4 池）。

## 约束

- **POOLNAME 全网唯一**（critical，rule-0-00443）：不同地址池之间 POOLNAME 不能重复 — 后果：重名 ADD 失败
- **VPN 三者一致**（critical，rule-0-00443）：HASVPN=ENABLE 时 VPNINSTANCE 与同 APN、SGi-N6 外联口必须同 VPN — 后果：VPN 不一致致地址分配或路由异常
- **HASVPN=ENABLE 时 VPNINSTANCE 必填**（critical，rule-0-00443）：HASVPN=ENABLE 配 VPNINSTANCE 必填；用 `LST VPNINST` 查询已配值 — 后果：缺 VPNINSTANCE 命令拒收
- **LOCAL+RELEASETIME 过大为高危**（critical，rule-0-00443，命令 notes）：用户地址延迟释放→池地址不足→激活失败 — 后果：用户激活失败
- **最大记录数 40000**（warning，rule-0-00443）：>98% 上报 ALM-135602215（≤95% 恢复，阈值可 `MOD CFGTHRESHOLD（★R1.5 atom 待补）` 改） — 后果：超规格新增失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
- **REDUNDFUNC 需全局开关配套**（warning，隐含）：池级 REDUNDFUNC=ENABLE 需配 [SET REDUNDUSER](task/UDG/20.15.2/0-00064.md) 全局开关 — 后果：未开全局开关时冗余功能不生效
