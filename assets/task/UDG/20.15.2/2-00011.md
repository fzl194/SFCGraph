# RTSP头增强（GWFD-110262）

> 特性静态知识见 [GWFD-110262](feature/UDG/20.15.2/GWFD-110262.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性是**头增强协议族成员**，复用族范本 [2-00010](task/UDG/20.15.2/2-00010.md)（HTTP头增强）的 compound 骨架，差异集中在协议绑定（RTSP）+ License + SA 依赖 + 按官方差异表不支持头防欺诈。

## 配置概览

RTSP 头增强将用户信息（MSISDN/IMSI/APN/MSIP 等）以 `<字段前缀: 字段值>` 格式插入 RTSP 请求报文头扩展字段，传给 Streaming Server。**核心机制与 HTTP 头增强完全一致：RULE 经 `POLICYTYPE=HEADEN` 直接引用 HEADEN 对象（ADD HEADEN 定义），RTSP 与 HTTP 共用 ADD HEADEN 命令**。实时流协议在语法和操作上与 HTTP/1.1 类似，RTSP 的扩展机制与 HTTP 基本相同。

与 HTTP 头增强（[2-00010](task/UDG/20.15.2/2-00010.md)）的差异：(1) 协议绑定 PROTOCOLNAME=**rtsp**（HTTP=http）；(2) License LKV3G5RTHE01（HTTP=LKV3G5HTHE01）；(3) SA 依赖 SA-Basic+**SA-Streaming**（HTTP=SA-Basic+SA-Web Browsing）；(4) 知名端口 554（RTSP 默认端口）；(5) **按官方差异表不支持头防欺诈**（HTTP/HTTPS 支持，证据：[头增强功能之间的差异](evidence/UDG/20.15.2/头增强功能之间的差异_10706790.md) §头防欺诈行）；(6) UDG 仅支持基于 TCP 的 RTSP，不支持 RTSP over HTTP。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开 RTSP 头增强 License 开关，单命令）：`SET LICENSESWITCH`（LKV3G5RTHE01）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **配置流过滤器**（流匹配容器，单命令）：`ADD FLOWFILTER`（FLOWFILTERNAME）→ [0-00007](task/UDG/20.15.2/0-00007.md)
3. **配置过滤条件 + 协议识别**（按触发方式选，三四层/七层/协议三选一）：
   - 过滤条件：`ADD FILTER`+`ADD FLTBINDFLOWF` 或 `ADD L7FILTER`+`ADD PROTBINDFLOWF`（**PROTOCOLNAME=rtsp**）+ `SET REFRESHSRV` → [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链 compound，仅含 FILTER/FILTERIPV6/L7FILTER/FLOWFILTER/FLTBINDFLOWF/PROTBINDFLOWF）
   - 协议识别（直引 atom，**非 1-00009 compound 成员**）：`ADD WELLKNOWNPORT`（**端口554，PROTOCOLNAME=rtsp**）→ [0-00039](task/UDG/20.15.2/0-00039.md) + `LOD SIGNATUREDB` → [0-00027](task/UDG/20.15.2/0-00027.md) + `LOD PARSERDB` → [0-00152](task/UDG/20.15.2/0-00152.md)
   - 基于特定 URL（主流，activation 演示）：`ADD L7FILTER`（URL）+ `ADD PROTBINDFLOWF`（PROTOCOLNAME=rtsp, L7FILTERNAME）
   - 基于特定 IP：`ADD FILTER`（SVRIP/端口，L34PROTOCOL=TCP）+ `ADD FLTBINDFLOWF`（**不配协议**，同时影响 HTTP/RTSP/HTTPS）
   - 基于 RTSP 协议：`ADD PROTBINDFLOWF`（**PROTOCOLNAME=rtsp**，**不配 L7FILTER**）
4. **配置头增强对象**（RTSP/HTTP 共用 ADD HEADEN，多字段插入+加密；**RTSP 按官方差异表不支持 ANTIFRAUD/GRAYLIST 防欺诈**，证据见约束）：`ADD HEADEN`（HEADERENNAME+DATATYPE+PREFIXNAME+ENCRYALGORI，**多字段多次执行**）→ [1-00013](task/UDG/20.15.2/1-00013.md)
5. **配置计费属性和策略**（因头增强仅针对上行报文，需上行计费；可选）：`ADD URR`+`ADD URRGROUP`+`ADD PCCPOLICYGRP` → [1-00010](task/UDG/20.15.2/1-00010.md)
6. **配置头增强规则**（POLICYTYPE=HEADEN 触发，规则绑定收尾）：`ADD RULE`（POLICYTYPE=HEADEN + POLICYNAME=HEADERENNAME）+ `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
   - 计费规则（可选，与头增强规则共用同一 FLOWFILTER）：`ADD RULE`（POLICYTYPE=PCC + POLICYNAME=PCCPOLICYGRPNM）

> **参数证据链**（R1.5，配置流程脚注）：以上命令+参数均来自 RTSP activation §表1~表10 + 命令 wiki。
> - SET LICENSESWITCH（**LKV3G5RTHE01**）：activation §表1 + §任务示例脚本（原样）
> - ADD FLOWFILTER / ADD FILTER（L34PROTOCOL=TCP, SVRIP）/ ADD FLTBINDFLOWF / SET REFRESHSRV：activation §表2~表3 + §任务示例脚本（原样）
> - ADD L7FILTER（URL）/ ADD PROTBINDFLOWF（**PROTOCOLNAME=rtsp**, L7FILTERNAME）：activation §表4 + §任务示例脚本（原样，**RTSP 关键差异**）
> - ADD WELLKNOWNPORT（**IDENPROTNAME=rtsp_554, PROTOCOLNAME=rtsp, STARTPORT=554, PRIORITY=5**）/ LOD SIGNATUREDB / LOD PARSERDB：activation §表4 + §任务示例脚本（原样，RTSP 默认端口 554）
> - **ADD HEADEN**（HEADERENNAME/DATATYPE=APN,IMSI1,MSIP1,MSISDN1/PREFIXNAME/ENCRYALGORI=NONE）：activation §表5 + §任务示例脚本（4 字段原样）；加密联动见 [1-00013](task/UDG/20.15.2/1-00013.md) 场景差异 §2
> - ADD URR/URRGROUP/PCCPOLICYGRP：activation §表7 + §任务示例脚本（原样）
> - ADD RULE（POLICYTYPE=HEADEN + POLICYNAME=header_test）：activation §表8 + §任务示例脚本（原样，PCC 策略组名称=NULL）
> - ADD RULE（POLICYTYPE=PCC + POLICYNAME=pg_test）：activation §表9 + §任务示例脚本（原样）
> - ADD USERPROFILE/RULEBINDING：activation §表10 + §任务示例脚本（原样）
>
> **activation 模板复用说明**（R1.4）：RTSP activation 与 HTTP activation 部署模板结构高度一致（10 表 + 同样的 RULE/HEADEN 命令骨架），但 **RTSP activation 的协议绑定参数（PROTOCOLNAME=rtsp）、知名端口（554）、License（LKV3G5RTHE01）均来自 RTSP activation 主脚本原样演示**（非 HTTP 模板复用，是 RTSP 独立 activation 文档）。**防欺诈不支持**（官方差异表明确声明 RTSP 头防欺诈"不支持"，证据：[头增强功能之间的差异](evidence/UDG/20.15.2/头增强功能之间的差异_10706790.md) §头防欺诈行；HTTP/HTTPS 标"支持"，RTSP 标"不支持"）——虽然 ADD HEADEN 命令含 ANTIFRAUD 参数（RTSP 与 HTTP 共用 ADD HEADEN），但按差异表 RTSP 场景防欺诈不生效，配置生成不产出 ANTIFRAUD=ENABLE。

## 决策点

**DP（特性级）：触发方式 + 插入字段类型 + 加密算法 + 编码方式**（影响过滤链走法 + ADD HEADEN 参数）。维度按 RTSP activation §表3/§表4/§表5 + 原理文档 §触发条件 + §安全机制 全列。**RTSP 按官方差异表不支持防欺诈/灰名单 DP**（证据：[头增强功能之间的差异](evidence/UDG/20.15.2/头增强功能之间的差异_10706790.md) §头防欺诈行；与 HTTP 头增强差异，见约束）。

| 场景 | 触发方式 | 过滤链走法 | 插入字段示例 | 加密 |
|---|---|---|---|---|
| **基于 URL 触发（主流，activation 演示）** | 基于 URL | L7FILTER（URL）+ PROTBINDFLOWF（**rtsp**+L7FILTER）| APN/IMSI/MSIP/MSISDN | NONE |
| 基于特定 IP 触发 | 基于特定 IP | FILTER（SVRIP+端口，L34PROTOCOL=TCP，不配协议）+ FLTBINDFLOWF | 任意 | 任意 |
| 基于 RTSP 协议触发 | 基于 RTSP 协议 | PROTBINDFLOWF（**rtsp**，无 L7FILTER）| 任意 | 任意 |
| 加密变体（安全推荐）| 任意 | 同上 | 任意 | **AES128_GCM/AES256_GCM**（+PSWDKEY+SET AES）|
| 加密变体（哈希）| 任意 | 同上 | 任意 | **SHA256**（+PSWDKEY+SALTSW）|
| 多参数拼接（20.10.0+）| 任意 | 同上 | MULTIPARA1/2/3（MSISDN+SGSNIP+MSIP）| 任意 |

**联动**：
- 触发方式 → [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链走法（L7FILTER+PROTBINDFLOWF / FILTER+FLTBINDFLOWF / 仅 PROTBINDFLOWF）
- 协议绑定 → **PROTOCOLNAME=rtsp**（RTSP 关键差异，HTTP=http）；WELLKNOWNPORT 端口 554
- 加密算法 → [1-00013](task/UDG/20.15.2/1-00013.md) 场景差异 §2（联动 SET AES/ADD RSA* 等加密配置命令）
- 编码方式 → [1-00013](task/UDG/20.15.2/1-00013.md) 场景差异 §3（INHERIT/BASE64/ASCII）
- **防欺诈/灰名单 → 不支持**（官方差异表明确声明 RTSP 头防欺诈"不支持"，证据：[头增强功能之间的差异](evidence/UDG/20.15.2/头增强功能之间的差异_10706790.md) §头防欺诈行；与 HTTP/HTTPS 关键差异）
- 5G 格式插入 → SET SOFTPARA BYTE896（仅对 RAT=5G 用户生效，GPSI/SUPI/PEI）

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5RTHE01` — 未开则 RTSP 头增强不生效。
- **强依赖 SA-Basic**（critical）：GWFD-110101（License LKV3G5SABS01）—— RTSP 报文解析基础，未开则无法识别报文。
- **协议依赖 SA-Streaming**（critical）：GWFD-110107（License LKV3G5SAST01）—— RTSP 流媒体协议解析必需（**与 HTTP 头增强差异：HTTP 依赖 SA-Web Browsing，RTSP 依赖 SA-Streaming**）。
- **POLICYTYPE=HEADEN 触发条件**（critical）：RULE.POLICYTYPE=HEADEN + POLICYNAME=HEADERENNAME（直接引用，PCC 策略组名称=NULL）— 指错则头增强不触发。
- **不支持头防欺诈**（critical）：官方差异表明确声明 RTSP 头防欺诈"不支持"（证据：[头增强功能之间的差异](evidence/UDG/20.15.2/头增强功能之间的差异_10706790.md) §头防欺诈行；HTTP/HTTPS 标"支持"，RTSP 标"不支持"）— 与 HTTP/HTTPS 头增强关键差异；配 ANTIFRAUD=ENABLE 在 RTSP 场景不生效。
- **仅基于 TCP 的 RTSP**（critical）：UDG 仅支持基于 TCP 的 RTSP（RFC 2326 允许 TCP/UDP，UDG 不支持 UDP）— UDP 上的 RTSP 报文不做头增强。
- **不支持 RTSP over HTTP**（warning）：用户访问的流媒体服务器使用 RTSP over HTTP 协议时，UDG 不支持对这类报文做头增强 — 属预期行为。
- **仅 RTSP1.x 类 HTTP/1.1 扩展**（warning）：RTSP 扩展机制与 HTTP/1.1 基本相同，但不支持 HTTP2.0；HTTPS 需 GWFD-110263（ADD TLSHEADEN）— 混用则对应协议报文不做头增强。
- **基于 IP 触发同时影响 HTTP/RTSP/HTTPS**（warning）：不区分协议，对用户的 HTTP/RTSP/HTTPS 报文都做头增强（设计如此）— 后果：HTTP/HTTPS 报文意外被头增强；改用基于 URL 触发可规避。
- **IPv6 分片风险**（warning）：头增强字段过长致报文超 PMTU 将导致 IPv6 业务无法访问；建议 TCP MSS ≤ PMTU - IPv6首部(40B) - TCP首部(20B) - 头增强字段长度。
- **分片报文不做头增强**（warning）：MTU 过小致用户报文到达 UDG 前被分片，UDG 不处理分片报文；RTSP 头域分片的请求报文也不支持。
- **外置 PCEF 计费误差**（warning）：头增强改变报文长度，外置 PCEF 计费有误差；建议头增强与计费部署在同一网元，或内置 PCEF 计费 + 外置 PCEF 部署头增强。
- **RTSP 请求类型限制**（info）：仅对 6 类 RTSP 请求做头增强——Setup/Play/Pause/Teardown/Options/Describe；其他请求类型不处理。
- **规格**（info）：整机 ≤100 头增强动作；头增强仅针对上行报文（计费属性须含上行）。
- **计费与话单**（info）：先计费再头增强（与规则优先级无关），头增强插入信息不计入业务量计费统计。
