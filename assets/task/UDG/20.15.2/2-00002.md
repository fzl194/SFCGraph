# URL过滤基本功能（GWFD-110471）

> 特性静态知识见 [GWFD-110471](feature/UDG/20.15.2/GWFD-110471.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

URL 过滤通过外置 ICAP Server 分析 URL 分类，UDG 执行动作（PERMIT/BLOCK/REDIRECT）。**双轨道协同**：
- **轨道A（PCC 触发）**：三四层匹配 + PCC 费率策略组 + 规则绑用户模板，让 RULE(POLICYTYPE=PCC) 触发 URL 过滤业务
- **轨道B（CF 内容匹配）**：CF 策略-模板-分类体系，缺省动作(CFTEMPLATE.ACTION) + 分类绑定动作(CONTCATEGBIND.ACTION，**优先**)

多场景差异由 DP 驱动（动作匹配模式/动作类型/协议类型/粒度/ICAP部署/cache）。

## 配置流程（步骤编排）

1. **ICAP 互通**（前置，轨道B 外置通道）→ [1-00007](task/UDG/20.15.2/1-00007.md)
2. **License + APN 粒度开关**（单命令 atom）：[SET LICENSESWITCH](task/UDG/20.15.2/0-00019.md) → [ADD APN](task/UDG/20.15.2/0-00020.md) → [SET APNCFFUNC](task/UDG/20.15.2/0-00251.md)
3. **CF 内容过滤业务**（轨道B 核心）→ [1-00008](task/UDG/20.15.2/1-00008.md)
4. **本地 cache**（单命令 atom，可选）：[SET CFCACHEPARA](task/UDG/20.15.2/0-00258.md)
5. **过滤链**（轨道A 匹配，跨域通用 backbone；DP 选过滤模式，URL过滤用三四层）→ [1-00009](task/UDG/20.15.2/1-00009.md)
6. **刷新生效**（单命令 atom）：[SET REFRESHSRV](task/UDG/20.15.2/0-00015.md)（Filter 配置后置生效）
7. **计费三件套**（轨道A，跨域通用 backbone）→ [1-00010](task/UDG/20.15.2/1-00010.md)
8. **规则与用户模板绑定**（轨道A 收尾，POLICYTYPE=PCC 触发，跨域通用 backbone）→ [1-00011](task/UDG/20.15.2/1-00011.md)

## 决策点

**DP（特性级）：动作匹配模式 + 动作类型 + 粒度 + 协议**（影响轨道B + 前置依赖）：

| 场景 | 匹配模式 | CFTEMPLATE.ACTION(缺省) | CONTCATEGBIND(分类绑定) | 粒度 | 协议/SA 依赖 |
|---|---|---|---|---|---|
| 分类匹配（主流） | 分类匹配 | BLOCK/PERMIT/REDIRECT（兜底） | 配：CATEGORYID+ACTION（优先） | APN | HTTP/WAP→SA-Web Browsing |
| 直接动作 | 直接动作 | 兜底（如 PERMIT） | 省略 | APN | HTTP/WAP |
| HTTPS SNI | 分类匹配 | 同上 | 同上 | APN | HTTPS（仅 SNI，需 HTTP2.0 Host 识别） |
| 全局粒度 | 分类/直接 | 同上 | 同上 | 全局(SET GLBCFFUNC) | - |
| 白名单 | - | - | - | - | 免过滤豁免(CFWHITEURLLST/CFIPWHITELIST) |

**联动**：
- 动作匹配模式 → [1-00008](task/UDG/20.15.2/1-00008.md) 是否配 CONTCATEGROUP/CONTCATEGBIND
- 粒度 → SET APNCFFUNC(APN) vs SET GLBCFFUNC(全局)
- 协议 → SA 依赖（HTTP/WAP→SA-Web Browsing；WAP1.X→SA-Mobile；HTTPS→HTTP2.0 Host 识别）
- cache → [0-00258](task/UDG/20.15.2/0-00258.md) 是否配

## 约束（特性级 rule）

- **License 前置**：SET LICENSESWITCH 开启 `LKV3G5UFBF01`。
- **依赖特性**：[SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，URL 解析基础，必选）+ 协议对应 SA + [PCC 基本功能](feature/UDG/20.15.2/GWFD-020351.md)（GWFD-020351，PCC 触发）。
- **CONTCATEGBIND.ACTION 优先于 CFTEMPLATE.ACTION**（分类匹配模式核心）。
- **ICAP 前置**：必须先配 [1-00007](task/UDG/20.15.2/1-00007.md)，CFTEMPLATE 引用 ICAPSRVGMNAME。
- 仅解析特定协议 URL：HTTP/WAP1.X/WAP2.0（Get/Post URL）/HTTPS/非加密QUIC（仅 SNI/证书）。
- 规格：≤500 套餐 / ≤30 ICAP Server / 每组 ≤10 / ≤150万 URL。
- 性能：每会话与 ICAP 交互增 CPU + Get 首包时延；开 cache 缓解。
