# 创建L2TP组(本地配置方式专属)（ADD L2TPGROUP）

> 命令静态知识见 [ADD L2TPGROUP](command/UDG/20.15.2/ADD-L2TPGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**适用 NF：PGW-U、UPF**。

## 配置方法

ADD L2TPGROUP 创建**本地配置方式**专属的 L2TP 组（AAA 方式不需要建组）；一组绑定 1~6 个 LNS IP 并配置隧道鉴权/AVP/HELLO/PPP/窗口参数。**GROUPID 必选**（1~1500 整数）。核心配置空间有 4 个维度：**组号定位**（GROUPID）、**隧道身份与保活**（AUTHENTICATION/AVPHIDDEN/HELLO*/LOCALNAME/RETRYTIMES/DOMAINNAME/VPNINSTANCE）、**多 LNS 工作模式**（LOCALLNSMODE / FIRST~SIXTHLNSIP / CFM*PWD）、**PPP 与隧道窗口**（PPPMAGICNUMBER/MAXSENDWINSIZE/INITTUNNELNUM/MAXSESSIONNUM/DSCPV）。1 个特性命中（GWFD-020412 L2TP VPN 本地方式）建组 1：DOMAINNAME=example.com、LOCALNAME=UPF、AVPHIDDEN=ENABLE、HELLOINTERVAL=120、VPNINSTANCE=vpn_l2tp、LOCALLNSMODE=REDUNDANCY、主备 2 个 LNS。

### 配置维度 1：组号定位（参数 GROUPID，必选）

| 取值 | 作用 |
|---|---|
| **1~1500 整数**（020412 范式 `1`） | L2TP 组号；本组在系统内全局唯一 |

> 由 [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR 用 `L2TPGROUPID` 引用；同名重复添加命令拒收。

### 配置维度 2：隧道身份与保活（参数 AUTHENTICATION / DOMAINNAME / LOCALNAME / AVPHIDDEN / HELLO* / RETRYTIMES / VPNINSTANCE，可选）

| 参数 | 取值/范围 | 作用（020412 范式值） |
|---|---|---|
| **AUTHENTICATION** | DISABLE/ENABLE | 隧道鉴权开关（建议 ENABLE） |
| **DOMAINNAME** | 字符串 1~60 | 隧道域名；用户名匹配后命中本组（020412 范式 `example.com`） |
| **LOCALNAME** | 字符串 1~30（区分大小写） | 隧道本端 LAC 名称（020412 范式 `UPF`；未配时取 [0-00131](task/UDG/20.15.2/0-00131.md) SET GLOBALL2TP 缺省） |
| **AVPHIDDEN** | DISABLE/ENABLE | 关键 AVP 属性加密开关（020412 范式 ENABLE） |
| **HELLOINTERVALSW** | DISABLE/ENABLE | HELLO 保活报文开关（020412 范式 ENABLE） |
| **HELLOINTERVAL** | 60~10000 秒 | HELLO 报文间隔（020412 范式 `120`）；HELLOINTERVALSW=ENABLE 时必选 |
| **RETRYTIMES** | 1~16（默认 3） | 报文重发次数 |
| **VPNINSTANCE** | 字符串 1~31 | 隧道绑定的 VPN 实例名（020412 范式 `vpn_l2tp`，ref [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST） |

> HELLOINTERVAL 仅在 HELLOINTERVALSW=ENABLE 时为必选。

### 配置维度 3：多 LNS 工作模式（参数 LOCALLNSMODE / FIRST~SIXTHLNSIP / *PWD / CFM*PWD，条件可选）

| 参数 | 前提 | 取值 | 作用 |
|---|---|---|---|
| **LOCALLNSMODE** | — | ONLY_ONE（默认）/REDUNDANCY/LOAD_SHARING | 多 LNS 工作模式（020412 范式 REDUNDANCY 主备） |
| **FIRSTLNSIP + FIRSTPWD** | REDUNDANCY/LOAD_SHARING | IP+密码 | 第 1 个 LNS（020412 范式 `10.10.10.1` / `0123456`） |
| **SECONDLNSIP + SECONDPWD** | REDUNDANCY/LOAD_SHARING | IP+密码 | 第 2 个 LNS（020412 范式 `10.10.10.2` / `0123456`） |
| **THIRD~SIXTHLNSIP + *PWD** | 仅 LOAD_SHARING | IP+密码 | 第 3~6 个 LNS（LOAD_SHARING 可达 6 个） |
| **CFMFIRSTPWD / CFMSECONDPWD** | REDUNDANCY/LOAD_SHARING | 重复密码 | 确认密码（020412 范式都配 `0123456`） |
| **CFMTHIRD~CFMFIFTHPWD / CFMSIXTHPWD** | LOAD_SHARING | 重复密码 | 第 3~6 个 LNS 密码确认 |

> LOCALLNSMODE=ONLY_ONE 时**不**配 *LNSIP/*PWD；REDUNDANCY 可达 2 个 LNS；LOAD_SHARING 可达 6 个 LNS。

### 配置维度 4：PPP 与隧道窗口（参数 PPPMAGICNUMBER / MAXSENDWINSIZE / INITTUNNELNUM / MAXSESSIONNUM / DSCPV，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **PPPMAGICNUMBER** | DISABLE/ENABLE | PPP LCP 是否支持 Magic-Number 协商（020412 范式 ENABLE） |
| **MAXSENDWINSIZE** | 整数 | LAC 侧隧道发送窗口上限（020412 范式 `32`） |
| **INITTUNNELNUM** | 整数 | LAC 与每 LNS 初始建立隧道数 |
| **MAXSESSIONNUM** | 整数 | 每条隧道承载的会话数上限 |
| **DSCPV** | 0~63、255 | 信令报文 DSCP 值 |

> 这五个参数下放自 [0-00131](task/UDG/20.15.2/0-00131.md) SET GLOBALL2TP——组内未配时走系统缺省。

**典型脚本（020412 L2TP VPN 本地方式，L2TP 组 1 + REDUNDANCY 主备 2 LNS）**：
```
ADD L2TPGROUP:GROUPID=1,DOMAINNAME="example.com",LOCALNAME="UPF",AVPHIDDEN=ENABLE,HELLOINTERVALSW=ENABLE,HELLOINTERVAL=120,VPNINSTANCE="vpn_l2tp",LOCALLNSMODE=REDUNDANCY,FIRSTLNSIP="10.10.10.1",FIRSTPWD="0123456",SECONDLNSIP="10.10.10.2",SECONDPWD="0123456",CFMFIRSTPWD="0123456",CFMSECONDPWD="0123456",PPPMAGICNUMBER=ENABLE,MAXSENDWINSIZE=32;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：建 L2TP 组"——前置 [0-00131](task/UDG/20.15.2/0-00131.md) SET GLOBALL2TP + [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR（含 L2TPGROUPID）；后续 [0-00135](task/UDG/20.15.2/0-00135.md) ADD L2TPCLIENTIP 绑源端 + [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG。

## 决策点：本地 LNS 寻址模式（DP 0-00133）

问：本 L2TP 组下用哪种多 LNS 寻址拓扑？

| 选项 | LOCALLNSMODE | 配 LNS 数 | 配套参数（F/S LNSIP / CFM PWD） | 适用场景 |
|---|---|---|---|---|
| **单 LNS** | ONLY_ONE | 0 个（不配 *LNSIP） | 不配 | 单 LNS 集中接入 |
| **主备 2 LNS**（020412 范式） | REDUNDANCY | 2 个 | F/S LNSIP + F/S PWD + CFM F/S PWD | LNS 主备倒换 |
| **负载分担 ≤6 LNS** | LOAD_SHARING | 1~6 个 | T1~T6 LNSIP + *PWD + CFM*PWD | 多 LNS 负载分担 |

> 1 个特性命中（020412，REDUNDANCY 主备 2 LNS 范式）；ONLY_ONE 时不配任何 *LNSIP。

## 约束

- **GROUPID 必选且唯一**（critical，rule-0-00133）：GROUPID 是必选参数；同一系统内不可重复 — 后果：缺 GROUPID 或重复添加命令拒收
- **HELLOINTERVAL 条件必选**（critical，rule-0-00133）：HELLOINTERVAL 仅在 HELLOINTERVALSW=ENABLE 时为必选 — 后果：缺间隔命令拒收
- **LNS 配置与模式联动**（critical，rule-0-00133）：LOCALLNSMODE=ONLY_ONE 时不配 *LNSIP；REDUNDANCY/LOAD_SHARING 时 FIRST/SECOND 成对配 — 后果：违反联动约束命令拒收
- **VPNINSTANCE 必须已建**（critical，rule-0-00133）：VPNINSTANCE 必须已通过 `ADD L3VPNINST` 创建 — 后果：VPN 不存在命令拒收
- **多 LNS 数约束**（critical，rule-0-00133）：LOAD_SHARING 时最多配 6 个 LNS；REDUNDANCY 时最多 2 个 — 后果：超数命令拒收
- **密码与确认密码一致**（warning，rule-0-00133）：FIRSTPWD/CFMFIRSTPWD、SECONDPWD/CFMSECONDPWD 等须一致 — 后果：密码不一致命令拒收
- **适用 NF**（info，rule-0-00133）：本命令仅适用 PGW-U、UPF — 后果：误在 SGW-U 上执行命令拒收
- **执行后立即生效**（warning，rule-0-00133）：本命令执行后立即生效 — 后果：误配即刻影响用户 L2TP 域名匹配与隧道建立
