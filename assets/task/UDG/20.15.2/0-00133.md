# 创建L2TP组(本地配置方式专属)（ADD L2TPGROUP）

> 命令静态知识见 [ADD L2TPGROUP](command/UDG/20.15.2/ADD-L2TPGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD L2TPGROUP 是**本地配置方式专属**命令（AAA 方式不建 L2TP 组，由 RADIUS 下发 LNS）。组创建后绑定到 [APN 的 L2TP 属性](task/UDG/20.15.2/0-00132.md)，按用户名域名匹配触发建隧道。VPNINSTANCE 引用上游 [VPN 实例](task/UDG/20.15.2/0-00041.md)（ADD VPNINST）。

**主备 LNS（REDUNDANCY）典型配法**（GWFD-020412）

| 参数 | 取值样例 | 来源 | 必选 | 说明 |
|---|---|---|---|---|
| GROUPID | `1` | local_planned | required | L2TP 组号 1~1500 |
| DOMAINNAME | `example.com` | local_planned | optional | 用户名域名匹配触发建隧道 |
| LOCALNAME | `UPF` | local_planned | optional | 本端名称，空则取 [SET GLOBALL2TP](task/UDG/20.15.2/0-00131.md) 缺省 |
| AUTHENTICATION | `ENABLE`（默认） | local_planned | optional | 隧道鉴权开关 |
| AVPHIDDEN | `ENABLE` | local_planned | optional | 隐藏 AVP（默认 DISABLE） |
| HELLOINTERVALSW | `ENABLE` | local_planned | optional | HELLO 报文开关（默认 DISABLE） |
| HELLOINTERVAL | `120` | local_planned | conditional | 仅 HELLOINTERVALSW=ENABLE 时必填，60~10000 秒 |
| RETRYTIMES | `3`（默认） | local_planned | optional | 1~16 |
| VPNINSTANCE | `vpn_l2tp` | reference（← 0-00041 ADD VPNINST） | optional | 无 VPN 时走默认 VRF |
| LOCALLNSMODE | `REDUNDANCY`（默认） | local_planned | optional | 主备 / `LOAD_SHARING`（最多 6 LNS） |
| FIRSTLNSIP | `10.10.10.1` | peer_planned | conditional | 主 LNS IP |
| FIRSTPWD | `0123456` | peer_planned | conditional | 主 LNS 密码，配 CFMFIRSTPWD 确认 |
| CFMFIRSTPWD | `0123456` | peer_planned | conditional | 与 FIRSTPWD 一致 |
| SECONDLNSIP | `10.10.10.2` | peer_planned | conditional | 备 LNS IP（REDUNDANCY 必填） |
| SECONDPWD | `0123456` | peer_planned | conditional | 备 LNS 密码 |
| CFMSECONDPWD | `0123456` | peer_planned | conditional | 与 SECONDPWD 一致 |
| PPPMAGICNUMBER | `ENABLE` | peer_planned | optional | Magic-Number 协商（默认 DISABLE） |
| MAXSENDWINSIZE | `32` | peer_planned | optional | 发送窗口 4~128（默认 64） |

**主备 LNS 典型脚本**：

```
ADD L2TPGROUP: GROUPID=1, DOMAINNAME="example.com", LOCALNAME="UPF", AVPHIDDEN=ENABLE, HELLOINTERVALSW=ENABLE, HELLOINTERVAL=120, VPNINSTANCE="vpn_l2tp", LOCALLNSMODE=REDUNDANCY, FIRSTLNSIP="10.10.10.1", FIRSTPWD="0123456", SECONDLNSIP="10.10.10.2", SECONDPWD="0123456", CFMFIRSTPWD="0123456", CFMSECONDPWD="0123456", PPPMAGICNUMBER=ENABLE, MAXSENDWINSIZE=32;
```

**LNS 模式**：`LOCALLNSMODE=REDUNDANCY`（主备 2 LNS，SECONDLNSIP/SECONDPWD/CFMSECONDPWD 必填）；`LOAD_SHARING`（负荷分担，最多 6 LNS）；IPv4 LNS 数 >6 或对接 IPv6 LNS 须改用 ADD L2TPLNSINFO 追加（不在本批 18 命令范围）。

**部署时序**：① ADD VPNINST 建 VPN 实例（0-00041）→ ② ADD L2TPGROUP 建 L2TP 组（本 atom）→ ③ SET GLOBALL2TP 设缺省（0-00131）→ ④ SET APNL2TPATTR 绑定 APN（0-00132，L2TPGROUPID 引用本组）。

## 约束

- **rule-0-00367**（warning）命令最大记录数 **1500**（GROUPID 1~1500 全局唯一）；执行后**立即生效**；LNS IP 序号唯一——ADD L2TPGROUP 的 LNS IP 序号（FIRST~SIXTH 取 1~6）与 ADD L2TPLNSINFO 的 LNSNO（1~30）**不能相同**；IPv4 LNS 数 >6 须用 ADD L2TPLNSINFO（每 L2TP 组最多 30 IPv4 LNS），IPv6 LNS 必须用 ADD L2TPLNSINFO；同一组内 LNS 序号唯一 — 后果：超 1500 新建失败 / LNS 序号冲突致隧道无法建立 / 未用 L2TPLNSINFO 扩展时 LNS 数受 6 上限
- FIRSTPWD/SECONDPWD 配时须同配 CFMFIRSTPWD/CFMSECONDPWD 一致；HELLOINTERVAL 仅 HELLOINTERVALSW=ENABLE 时必填
- 适用 NF：PGW-U、UPF
