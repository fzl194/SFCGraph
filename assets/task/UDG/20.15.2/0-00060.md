# 使能接口IPv6并配置MTU（SET IFIPV6ENABLE）

> 命令静态知识见 [SET IFIPV6ENABLE](command/UDG/20.15.2/SET-IFIPV6ENABLE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**接口缺省不使能 IPv6**，必须先 SET IFIPV6ENABLE 才能配 [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS。IFNAME 引用 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 已建的接口。

## 配置方法（配置方法字典）

SET IFIPV6ENABLE 是"接口 IPv6 使能开关 + IPv6 MTU + 自动链路层地址"配置命令。核心配置空间有 4 个维度：**使能开关**（ENABLEFLAG，门控其余参数）、**IPv6 MTU**（MTU）、**自动链路层地址**（AUTOLINKLOCAL）、**适用接口类型**（命令 notes 限定）。3 个特性命中（010107、020402、IPFD-015004）覆盖 Tunnel 以太网子接口/Ethernet/Tunnel 三类接口范式。

### 配置维度 1：使能开关（参数 ENABLEFLAG，必选）

ENABLEFLAG 是门控开关，决定 IPv6 能力是否在该接口上启用：

| 取值 | 联动必选参数 | 适用场景 |
|---|---|---|
| **TRUE** | 可选 MTU + 可选 AUTOLINKLOCAL | 接口需承载 IPv6 业务（绝大多数场景） |
| **FALSE** | — | 关闭 IPv6（命令支持，3 个特性均未演示） |

> **缺省规则**：缺省情况下接口不使能 IPv6；**IPv6 使能时 MTU 缺省 1500、AUTOLINKLOCAL 缺省 FALSE**。

### 配置维度 2：IPv6 MTU（参数 MTU，ENABLEFLAG=TRUE 时条件可选）

| 参数 | 取值范围 | 默认值 | 备注 |
|---|---|---|---|
| **MTU** | 1280~9600 字节 | 1500（ENABLEFLAG=TRUE 时缺省） | 建议配 1500~9000 |

> **IPv4/IPv6 MTU 取大规则**：若 IPv4 MTU（来自 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE.IFMTU）与 IPv6 MTU（本命令 MTU）同时配置，**实际取两者之间的较大值**配置至 vNic（非直觉，需注意）。

### 配置维度 3：自动链路层地址（参数 AUTOLINKLOCAL，ENABLEFLAG=TRUE 时条件可选）

| 参数 | 取值 | 缺省值 | 作用 |
|---|---|---|---|
| **AUTOLINKLOCAL** | TRUE / FALSE | FALSE（ENABLEFLAG=TRUE 时） | TRUE=接口自动生成链路地址（链路本地地址用于邻居发现协议和无状态自动配置）；FALSE=不自动生成 |

### 配置维度 4：适用接口类型（命令 notes 限定）

命令 notes 明确：本命令仅在 **VNRS_VNFC 上的 Ethernet 接口、Ethernet 子接口、Eth-Trunk 接口、Eth-Trunk 子接口、Loopback 口、Tunnel 口** 上配置执行。其他接口类型不支持。

**典型脚本（010107 静态路由冗余，Tunnel 接口 + MTU=1500）**：
```
SET IFIPV6ENABLE: IFNAME="Tunnel1", ENABLEFLAG=TRUE, MTU=1500;
SET IFIPV6ENABLE: IFNAME="Tunnel2", ENABLEFLAG=TRUE, MTU=1500;
```

**典型脚本（020402 IPv6 组网，以太网子接口，仅 ENABLEFLAG）**：
```
SET IFIPV6ENABLE:IFNAME="Ethernet70/0/4.1", ENABLEFLAG=TRUE;
SET IFIPV6ENABLE:IFNAME="Ethernet70/0/5.1", ENABLEFLAG=TRUE;
```

**典型脚本（IPFD-015004 IPsec，Ethernet + Tunnel）**：
```
SET IFIPV6ENABLE:IFNAME="Ethernet66/0/5", ENABLEFLAG=TRUE;
SET IFIPV6ENABLE:IFNAME="Tunnel27", ENABLEFLAG=TRUE;
```

> 命令 notes：**执行后立即生效**；**缺省不使能 IPv6**（必须显式 SET TRUE 才能配 IPv6 地址）；仅 VNRS_VNFC 上的 Ethernet/Ethernet 子接口/Eth-Trunk/Eth-Trunk 子接口/Loopback/Tunnel 口生效；**IPv4 MTU 与 IPv6 MTU 同时配时取较大值配置至 vNic**；本命令参与自动化配置——下发前需 `DSP OPSASSISTSTATE（★R1.5 atom 待补）` 查询 autoscaling_autoconfig.py。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：使能 IPv6"——前置 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE（建接口）+ [0-00057](task/UDG/20.15.2/0-00057.md) ADD IPBINDVPN（绑 VPN 后才能配私网 IPv6）；下游 [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS（必须使能后配）。

## 决策点：IPv6 MTU 配置策略（DP 0-00060）

问：SET IFIPV6ENABLE 的 MTU 参数如何决策？

| 选项 | MTU 取值 | 适用场景 |
|---|---|---|
| 显式 MTU=1500 | MTU=1500 | GRE Tunnel IPv6 承载（010107 范式：封装 GRE 出外联口不再次分片） |
| 不配 MTU（取缺省） | 不配（自动取 1500） | 以太网子接口 IPv6 组网（020402 范式）/ IPsec（IPFD-015004 范式）——非必填 |

> 3 个特性命中（010107/020402/IPFD-015004）中**仅 010107 显式配 MTU**；MTU 与 IFMTU（IPv4）取大下发 vNic 需注意；AUTOLINKLOCAL 在 UDG 场景未演示，实例化不用。

## 约束

- **缺省不使能 IPv6**（critical，rule-0-00460）：缺省情况下接口上不使能 IPv6 — 后果：未显式 SET TRUE 的接口后续 ADD IFIPV6ADDRESS 不生效
- **ENABLEFLAG=ENABLEFLAG 门控**（critical，rule-0-00460）：MTU/AUTOLINKLOCAL 仅在 ENABLEFLAG=TRUE 时才生效 — 后果：ENABLEFLAG=FALSE 时配 MTU/AUTOLINKLOCAL 命令拒收或被忽略
- **IPv4/IPv6 MTU 取大值配置**（warning，rule-0-00460）：若 IPv4 MTU（[0-00056](task/UDG/20.15.2/0-00056.md).IFMTU）与 IPv6 MTU（本命令 MTU）都配置，实际取两者之间的较大值配置至 vNic — 后果：配 IPv6 MTU<IPv4 MTU 时 vNic 实际取 IPv4 MTU（非直觉）
- **适用接口类型限定**（critical，rule-0-00460）：仅 VNRS_VNFC 上的 Ethernet/Ethernet 子接口/Eth-Trunk/Eth-Trunk 子接口/Loopback/Tunnel 口生效 — 后果：其他接口类型命令拒收
- **执行后立即生效**（warning，rule-0-00460）：命令执行后立即生效 — 后果：误操作即刻生效
- **参与自动化配置**（critical，rule-0-00460）：ENABLEFLAG/MTU 参与自动化配置——下发前需 `DSP OPSASSISTSTATE（★R1.5 atom 待补）` 查询 autoscaling_autoconfig.py — 后果：自动化配置过程中执行致最终生效配置不可预期
- **MTU 范围与建议**（info，隐含）：MTU 取值 1280~9600；建议 1500~9000 — 后果：配置过小/过大可能不成功
