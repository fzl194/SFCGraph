# 使能接口IPv6并配置MTU（SET IFIPV6ENABLE）

> 命令静态知识见 [SET IFIPV6ENABLE](command/UDG/20.15.2/SET-IFIPV6ENABLE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET IFIPV6ENABLE 是 IPv6 地址配置的**前置使能**——接口缺省不使能 IPv6，必须先 SET IFIPV6ENABLE 才能配 [ADD IFIPV6ADDRESS](task/UDG/20.15.2/0-00061.md)。IFNAME 引用 [ADD INTERFACE](task/UDG/20.15.2/0-00056.md)。

**2 种场景范式（从证据数据规划表归纳）**

| 范式 | 接口 | ENABLEFLAG | MTU | 典型行 |
|---|---|---|---|---|
| GRE Tunnel IPv6 承载（010107） | Tunnel1/Tunnel2 | TRUE | 1500 | `SET IFIPV6ENABLE:IFNAME="Tunnel1",ENABLEFLAG=TRUE,MTU=1500;` |
| 以太网子接口 IPv6 组网（020402） | Ethernet70/0/4.1 等 | TRUE | 不配 | `SET IFIPV6ENABLE:IFNAME="Ethernet70/0/4.1",ENABLEFLAG=TRUE;` |

**ENABLEFLAG 固定 TRUE**（使能，所有场景一致）。MTU 仅 Tunnel 场景配（原则：封装 GRE 隧道出外联口不再次分片），以太网子接口省略 MTU 证明非必填（conditional）。AUTOLINKLOCAL（自动链路层地址）UDG 特性均未用，不绑。

## 约束

- **rule 0-00173**（state_invariant）缺省不使能 IPv6；仅 VNRS_VNFC 的 Ethernet 接口/子接口/Eth-Trunk 接口/子接口/Loopback/Tunnel 口生效；若 IPv4 MTU 与 IPv6 MTU 同时配置，实际取两者**较大值**下发至 vNic — 后果：未显式 SET 的接口 IPv6 不生效；IPv4/IPv6 MTU 同配取大值（非直觉，需注意）
