# 配置路由策略（ADD ROUTEPOLICY）

> 命令静态知识见 [ADD ROUTEPOLICY](command/UDG/20.15.2/ADD-ROUTEPOLICY.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD ROUTEPOLICY 创建一个命名的路由策略容器（POLICYNAME），本身只录策略名。它是"路由策略三件套"的**根对象**：POLICYNAME 作为引用源，供后续 [配置路由策略节点](task/UDG/20.15.2/0-00112.md)、[配置匹配路由类型](task/UDG/20.15.2/0-00113.md) 复合引用。

**单参数命令，配置即取一个全局唯一的策略名。**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| POLICYNAME | `policy1` | 本端规划 | 字符串 1~200，**全局唯一**（命名后不可重复添加） |

**典型脚本**（GWFD-020401 IPv6承载上下文）：

```
ADD ROUTEPOLICY:POLICYNAME="policy1";
```

**三件套引用链**：本 atom 建 POLICYNAME → 节点 atom（NODESEQUENCE+MATCHMODE）按 (POLICYNAME,NODESEQUENCE) 复合键引用 → 匹配类型 atom（ROUTETYPE）再引用同一复合键。

## 约束

- **rule-0-00240**（warning）POLICYNAME **全局唯一**，策略名不可重复添加；命令最大记录数 65535；执行后立即生效 — 后果：同名重复添加失败 / POLICYNAME 是下游节点与匹配类型的引用源，唯一性是引用链正确性的前提
