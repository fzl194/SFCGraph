# 配置路由策略（ADD ROUTEPOLICY）

> 命令静态知识见 [ADD ROUTEPOLICY](command/UDG/20.15.2/ADD-ROUTEPOLICY.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是路由策略三件套的根节点（POLICY 容器），后续挂节点（NODE）和匹配条件（MATCHROUTETYPE）。

## 配置方法

ADD ROUTEPOLICY 是路由策略根容器命令。**POLICYNAME 必选**（1~200 字符），全网唯一。核心配置空间有 1 个维度：**策略命名**（POLICYNAME）。1 个特性命中（GWFD-020401 IPv6 承载上下文）使用 `policy1`。

### 配置维度 1：策略命名（参数 POLICYNAME，必选）

| 取值 | 作用 |
|---|---|
| **本端规划字符串**（1~200 字符） | 路由策略名，作为 ROUTEPOLICYNODE/MATCHROUTETYPE 的引用键 |

> POLICYNAME 必须全网唯一；不区分大小写（建议保持一致）；后续节点通过 (POLICYNAME,NODESEQUENCE) 复合键挂入。

**典型脚本（GWFD-020401 IPv6 承载上下文）**：
```
ADD ROUTEPOLICY:POLICYNAME="policy1";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：建路由策略"——前置无；后续 [0-00112](task/UDG/20.15.2/0-00112.md) ADD ROUTEPOLICYNODE（挂节点）+ [0-00113](task/UDG/20.15.2/0-00113.md) ADD MATCHROUTETYPE（匹配路由类型）。

## 决策点：路由策略命名（DP 0-00111）

（本命令无独立决策分支，仅有 POLICYNAME 命名。）

| 选项 | POLICYNAME | 适用场景 |
|---|---|---|
| **业务化命名**（020401） | `policy1` | 单一策略（如 IPv6 路由策略） |
| **多策略分区命名** | `policy_<scope>_<seq>` | 多策略分区管理，按业务/范围/序号 |

> 1 个特性命中（020401）；POLICYNAME 必选且全网唯一。

## 约束

- **POLICYNAME 必选**（critical，rule-0-00111）：POLICYNAME 是必选参数 — 后果：缺 POLICYNAME 命令拒收
- **POLICYNAME 全网唯一**（critical，rule-0-00111）：POLICYNAME 必须全网唯一 — 后果：重名命令拒收
- **长度限制**（warning，rule-0-00111）：POLICYNAME 字符串长度 1~200 — 后果：超长命令拒收
- **POLICY 是 NODE 容器**（info，rule-0-00111）：本命令仅建策略容器，不挂节点；后续节点用 [0-00112](task/UDG/20.15.2/0-00112.md) ADD ROUTEPOLICYNODE 挂入 — 后果：未挂节点前策略不生效
