# 设置APN Non-IP配置（SET APNNONIPFUNC）

> 命令静态知识见 [SET APNNONIPFUNC](command/UDG/20.15.2/SET-APNNONIPFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

本命令开启/关闭指定 APN 的网关 Non-IP 功能（运营商提供给企业/IoT 用户的无 IP 数据传输服务，UE 报文不经过 IP 协议栈直接走应用层）。APN 必选，NONIPSWITCH 与 USERPORT 可选。

**适用 NF：SGW-U、PGW-U、UPF**（命令真相明确）

### 配置维度 1：Non-IP 开关（参数 NONIPSWITCH，可选）

| 取值 | 作用 | 配套必选参数 | 典型场景 |
|---|---|---|---|
| **ENABLE** | 开启指定 APN 的 Non-IP 数据传输功能 | APN | 物联网/企业 M2M 场景（GWFD-110607 Non-IP 数据传输）|
| **DISABLE** | 关闭 Non-IP 数据传输功能（恢复普通 IP 接入） | APN | 业务下线/迁移回 IP 场景 |

### 配置维度 2：用户端口号（参数 USERPORT，可选）

| 取值 | 作用 | 配套必选参数 | 典型场景 |
|---|---|---|---|
| **整数 1~65535** | 指定用户应用层端口号（运营商网关心跳） | APN + NONIPSWITCH=ENABLE | Non-IP 流量按用户端口分流 |
| **未配置** | 系统默认 5683（命令 notes 初始值） | APN | 接受默认端口的通用场景 |

**典型脚本**：
```
SET APNNONIPFUNC:APN="apn1.com",NONIPSWITCH=ENABLE;
SET APNNONIPFUNC:APN="apn1.com",NONIPSWITCH=ENABLE,USERPORT=5684;
SET APNNONIPFUNC:APN="apn1.com",NONIPSWITCH=DISABLE;  // 关闭功能
```

**步骤位置**：在 ADD APN 后启用 Non-IP 业务时配置；属 IoT 业务激活末段（前置：APN 已配）。

## 决策点

（无。本命令用法单一——APN 必选 + NONIPSWITCH 开关 + USERPORT 端口，无分叉选择，按"开关 + 端口"两维参数组合执行。）

## 约束

- **rule-0-00177-impl**（critical，命令 notes 推论）系统最多支持 10000 条 APNNonIpFunc 配置——超限 ADD 失败。`severity: critical` 后果：物联网业务规模超限后无法新增 Non-IP APN。
- **rule-0-00178-impl**（info，命令 notes 推论）该命令存在系统初始记录（NONIPSWITCH=DISABLE + USERPORT=5683），不配 NONIPSWITCH 时按初始值生效（DISABLE）。`severity: info` 后果：未配 NONIPSWITCH 的 APN 默认不使能 Non-IP 业务。
- **rule-0-00179-impl**（warning，命令 notes 推论）该命令执行后立即生效。`severity: warning` 后果：配置后立即影响 APN 的 Non-IP 流量行为。
- **rule-0-00180-impl**（critical，对端协商要求）USERPORT 取值范围 1~65535。`severity: critical` 后果：非法值配置失败。
- **rule-0-00181-impl**（critical，APN 名规范）APN 名仅含"-"、数字、大小写字母、"."，不能以"."开头，不能连续两个"."，长度 1~64。`severity: critical` 后果：非法 APN 名配置失败。
- **rule-0-00182-impl**（warning，系统资源）每 APN 最多配置 1 条 APNNonIpFunc（重复执行 SET 是覆盖语义）。`severity: warning` 后果：重复 SET 覆盖前配置，不报错但可能丢配置。
