# 设置APN单用户五元组上限（SET APNFLOWMAXNUM）

> 命令静态知识见 [SET APNFLOWMAXNUM](command/UDG/20.15.2/SET-APNFLOWMAXNUM.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`SET APNFLOWMAXNUM` 对**单个 APN** 下**单个用户**所有业务允许创建的五元组总数做上限控制，**仅 PGW-U/UPF 适用**。每个 APN 一条记录（SET 修改），多 APN 重复执行。核心维度是 APN 名 + 节点数阈值；FDMAXNUM 必须 ≤ 全局 `SET SRVCOMMONPARA.FDMAXNUM`。

### 配置维度 1：APN 接入点名（参数 APN）
| 取值 | 作用 | 配套 |
|---|---|---|
| 字符串 1~63 字符（`a-zA-Z0-9.-_`，以字母/数字开头） | 标识要设置的 APN 实例 | 必选 |

### 配置维度 2：APN 最大五元组节点数（参数 FDMAXNUM）
| 取值范围 | 作用 | 约束 |
|---|---|---|
| 1~10000（整数） | 该 APN 单用户允许的五元组节点总数上限 | 必须 ≤ 全局 SRVCOMMONPARA.FDMAXNUM |

## 决策点

（无。本命令在所有特性下用法一致：APN 名引用前置 ADD APN，FDMAXNUM 取值由本端规划决定，无独立 DP 分支。）

## 约束

- **rule-0-00405**（critical）：FDMAXNUM 取值 1~10000，超范围被拒 — 命令执行失败
- **rule-0-00406**（critical）：APN 名必须先经 [ADD APN](task/UDG/20.15.2/0-00020.md) 创建，否则该 APN 不存在 → 命令报错
- **rule-0-00407**（critical）：FDMAXNUM 必须 ≤ 全局 `SET SRVCOMMONPARA.FDMAXNUM`，全局级低于 APN 级时 APN 配置被覆盖 — APN 限流失效
- **rule-0-00408**（warning）：本命令仅 PGW-U/UPF 适用，SGW-U 侧执行被拒 — 跨 NF 误配置
- **rule-0-00409**（warning）：系统最多支持 10000 条记录，超出新增失败
- **rule-0-00410**（info）：每 APN 初始值为 65535；修改后仅对新接入或更新用户立即生效（老会话不追溯）
