# 配置APN单用户五元组上限（SET APNFLOWMAXNUM）

> 命令静态知识见 [SET APNFLOWMAXNUM](command/UDG/20.15.2/SET-APNFLOWMAXNUM.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110910（Routing Behind MS）的 APN 级流数上限配置步骤，针对**单个用户**在该 APN 下所有业务允许创建的五元组总数做控制。一次配置一个 APN 的上限，多个 APN 重复执行。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 |
|---|---|---|---|
| APN | 接入点名 | `apn1` | 引用 ADD APN 已配置数据（[配置APN](task/UDG/20.15.2/0-00020.md)） |
| FDMAXNUM | APN 最大五元组节点数 | `800` | 本端规划（1~10000，须 ≤ 全局 `SET SRVCOMMONPARA.FDMAXNUM`） |

**脚本样例**

```
SET APNFLOWMAXNUM:APN="apn1",FDMAXNUM=800;
```

**操作步骤定位**：该命令位于 Routing Behind MS 激活流程的第 3 步（紧接 [SET APNADDRESSATTR](command/UDG/20.15.2/SET-APNADDRESSATTR.md) 之后），第 4 步配套 `SET SRVCOMMONPARA` 配全局上限并打开自动调整开关——**APN 级上限必须与全局级协同**（全局 ≥ APN 级）。

## 约束

- **rule-0-00405**（warning）系统最多支持 10000 条记录，超出新增失败；对应 APN 须先用 ADD APN 配置存在；每 APN 初始值为 65535；FDMAXNUM 取值 1~10000；修改后仅对新接入或更新用户立即生效（老会话不追溯）
