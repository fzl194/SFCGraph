# 配置上行BA映射（SET QOSBA）

> 命令静态知识见 [SET QOSBA](command/UDG/20.15.2/SET-QOSBA.md)。本页只讲配置生成实例化时这条命令**怎么配**。入口方向，与出口方向 [SET QOSPHB](task/UDG/20.15.2/0-00069.md) 对偶。

## 配置方法

SET QOSBA 配置 DS 域**上行（入口）** VLAN 报文 802.1p 值到 UDG 内部服务等级 + 报文颜色的映射（CoS→ToS）。DSNAME 引用 [ADD QOSDIFFERSERV](task/UDG/20.15.2/0-00066.md)，BATYPE 主场景固定 8021p。

**配置范式（从证据数据规划表归纳）**

| 参数 | 取值 | 来源 | 说明 |
|---|---|---|---|
| DSNAME | default | 引用 0-00066 | DS 域名 |
| BATYPE | 8021p | 固定取值 | BA 类型（802.1p 或 DSCP） |
| BAVALUE | 3 | 本端规划 | BA 值（802.1p 取 0~7，DSCP 取 0~63） |
| SERVICECLASS | be | 本端规划 | 服务分类 |
| COLOR | red | 本端规划 | 颜色分类（命令真相 optional） |

```
SET QOSBA:DSNAME="default",BATYPE=8021p,BAVALUE=3,SERVICECLASS=be,COLOR=red;
```

同一 DSNAME 下入口/出口两方向 802.1p 值可独立规划（主示例出口 SET QOSPHB BAVALUE=3/SERVICECLASS=be/COLOR=green）。本 atom 仅入口方向配置，由 DP（QoS 优先级映射方向）选入口选项时使用。BAVALUE=8021p 时取 0~7，BATYPE=DSCP 时取 0~63；若同时配 CoS→ToS 与 3GPP QoS→DSCP/ToS，前者被后者覆盖。

## 约束

- **rule 0-00283**（dependency，与 [0-00069](task/UDG/20.15.2/0-00069.md) 共担）即时生效；BATYPE/PHBTYPE 跨方向互斥（同一 DSNAME 下入口 BA 上行 802.1p/DSCP 映射入口 SERVICECLASS/COLOR，出口 PHB 下行 SERVICECLASS/COLOR 映射出口 802.1p/DSCP，配套取值 8021p 或 DSCP）；入口+出口方向需各执行一次；BAVALUE/PHBVALUE 取值 0~63（802.1p 子范围 0~7）；BATYPE=DSCP 与 3GPP QoS-DSCP 并存时被后者覆盖 — 后果：方向未配→DSCP 映射缺失 / BATYPE 错配→取值越界被拒
