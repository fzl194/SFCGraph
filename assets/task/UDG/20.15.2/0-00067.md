# 配置上行BA映射（SET QOSBA）

> 命令静态知识见 [SET QOSBA](command/UDG/20.15.2/SET-QOSBA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。入口方向，与出口方向 [SET QOSPHB](task/UDG/20.15.2/0-00069.md) 对偶。

## 配置方法

SET QOSBA 是 QoS 5 件套的"入口映射层"——配置 DS 域**上行（入口）**VLAN 报文 802.1p 值（或 IP DSCP 值）到 UDG 内部服务等级 + 报文颜色的映射（外部优先级→内部优先级）。本命令的核心配置空间有 4 个维度：**映射类型**（BATYPE 二选一）、**外部优先级值**（BAVALUE 取值范围由 BATYPE 决定）、**内部服务等级**（SERVICECLASS 枚举）、**报文颜色**（COLOR 枚举）。

### 配置维度 1：映射类型（参数 BATYPE，必选）

| 取值 | 作用 | BAVALUE 范围 |
|---|---|---|
| **8021p** | VLAN 报文 802.1p 优先级映射（CoS→ToS） | 0~7 |
| **DSCP** | IP 报文 DSCP 优先级映射（DSCP→ToS） | 0~63 |

### 配置维度 2：外部优先级值（参数 BAVALUE，必选）

| 取值范围 | 含义 |
|---|---|
| 整数 0~7（BATYPE=8021p 时） | 802.1p 优先级值（0=BE，1~7 优先级递增） |
| 整数 0~63（BATYPE=DSCP 时） | DSCP 优先级值 |

### 配置维度 3：内部服务等级（参数 SERVICECLASS，必选）

| 取值 | 含义 |
|---|---|
| **be** | 尽力而为（最低） |
| **af1 / af2 / af3 / af4** | 确保转发 4 档 |
| **ef** | 加速转发（语音/视频） |
| **cs1 / cs2 / cs3 / cs4 / cs5 / cs6 / cs7** | 服务类别 7 档 |

### 配置维度 4：报文颜色（参数 COLOR，可选）

| 取值 | 含义 |
|---|---|
| **green** | 报文颜色（合规） |
| **yellow** | 报文颜色（超 CIR） |
| **red** | 报文颜色（超 PIR，需丢弃或重标记） |

| 参数 | 必选性 | 取值 | 作用 |
|---|---|---|---|
| `DSNAME` | 必选 | 字符串 | 引用 [0-00066](task/UDG/20.15.2/0-00066.md) ADD QOSDIFFERSERV |
| `BATYPE` | 必选 | 8021p / DSCP | BA 类型 |
| `BAVALUE` | 必选 | 0~7 或 0~63 | 外部优先级值 |
| `SERVICECLASS` | 必选 | be/af1~4/ef/cs1~7 | 内部服务等级 |
| `COLOR` | 可选 | green/yellow/red | 报文颜色（命令真相 optional） |

**典型脚本（010201 QoS 入口映射）**：
```
SET QOSBA:DSNAME="default",BATYPE=8021p,BAVALUE=3,SERVICECLASS=be,COLOR=red;
```

**步骤位置**：在所属 feature 编排中位于"步骤 3a：入口映射"——前置 [0-00066](task/UDG/20.15.2/0-00066.md) ADD QOSDIFFERSERV（DSNAME 来源）+ [0-00065](task/UDG/20.15.2/0-00065.md) SET SRVCOMMONPARA（TunnelMarking）；后续 [0-00068](task/UDG/20.15.2/0-00068.md) ADD QOSIFTRUST（接口绑定）。

## 决策点：BATYPE 映射类型选择（DP 0-00067）

问：SET QOSBA 的 BATYPE 取何值？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 8021p（VLAN 报文 CoS） | BAVALUE 0~7 | 入口方向 VLAN 报文 802.1p→ToS 映射（010201 主场景） |
| DSCP（IP 报文） | BAVALUE 0~63 | 入口方向 IP DSCP→ToS 映射 |

## 约束

- **BATYPE/PHBTYPE 跨方向互斥**（critical，rule-0-00283）：同一 DSNAME 下入口 BA 上行 802.1p/DSCP 映射入口 SERVICECLASS/COLOR，出口 PHB 下行 SERVICECLASS/COLOR 映射出口 802.1p/DSCP；配套取值 8021p 或 DSCP — 后果：BATYPE/PHBTYPE 错配→取值越界被拒
- **入口+出口方向各执行一次**（critical，rule-0-00283）：同一 DSNAME 入口 BA + 出口 PHB 须成对配置 — 后果：方向未配→DSCP 映射缺失
- **BAVALUE 取值范围**（critical，隐含）：8021p 时 0~7，DSCP 时 0~63 — 后果：超范围命令拒收
- **BATYPE=DSCP 与 3GPP QoS-DSCP 并存时被后者覆盖**（warning，rule-0-00283）：若同时配 CoS→ToS 与 3GPP QoS→DSCP/ToS，前者被后者覆盖 — 后果：BATYPE=DSCP 配置被覆盖
- **DSNAME 前置依赖**（critical，隐含）：DSNAME 须先建（[0-00066](task/UDG/20.15.2/0-00066.md)） — 后果：未建域命令拒收
