# 配置全局缺省URR组（SET SPECTRAFURRGRP）

> 命令静态知识见 [SET SPECTRAFURRGRP](command/UDG/20.15.2/SET-SPECTRAFURRGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`SET SPECTRAFURRGRP` 配置**整机级**全局缺省费率组，承接 `SET URRGRPBINDING` 的全局继承开关（`GLBDFTURRGSW=ENABLE` / `GLBDFTSIGURRGSW=ENABLE`）。**仅 PGW-U/UPF 适用**。**高危命令**：修改会导致整机计费使用的全局缺省费率变化或业务不计费。仅 1 个必填参数（URRGROUPNAME 引用上游 URRGROUP），整机最大 1 条（重复配置即覆盖）。

### 配置维度 1：URR 组名（参数 URRGROUPNAME）
| 取值 | 作用 | 约束 |
|---|---|---|
| 字符串 1~31 字符 | 引 `ADD URRGROUP`（0-00002）产出的 URRGROUP；典型取异常/兜底费率组 | 必选；必须先经 `ADD URRGROUP` 配置生成；绑定的 URR 欠费会影响特殊流量使用量上报 |

**典型脚本**：

```
SET SPECTRAFURRGRP: URRGROUPNAME="UrrGp_abnormal";
```

**步骤位置**：计费收尾序列——位于 `SET URRGRPBINDING`（0-00013，全局继承开关）之后、`ADD SPECURRGRPLIST`（0-00014）/`SET REFRESHSRV`（0-00015）之前；开关 ENABLE 时，USERPROFILE 未单独配缺省费率则继承本全局缺省。

## 决策点

（无。本命令在所有特性下用法一致：单条全局配置，URRGROUPNAME 一个必选 reference。无 DP 分支；"是否开启特殊流量"由 BIT1232 软参控制，不属本命令维度。）

## 约束

- **rule-0-00290-1**（critical）：高危——修改 URRGROUPNAME 会导致整机计费使用的全局缺省费率变化或业务不计费，造成运营商计费损失；立即生效
- **rule-0-00290-2**（warning）：最大记录数 1（整机级唯一全局缺省），重复配置即覆盖；立即生效
- **rule-0-00290-3**（warning）：本命令仅支持时长和流量计费方式，不支持基于事件的计费方式 — 事件计费场景配此全局缺省对事件计费无效
- **rule-0-00290-4**（warning）：URRGROUPNAME 必须已通过 `ADD URRGROUP`（0-00002）配置生成；若绑定的 URR 出现欠费，会影响特殊流量使用量上报
- **rule-0-00290-5**（info）：特殊流量走本组计费需 BIT1232 软参值设置为 1（开启特殊流量使用量上报功能）；本命令不区分业务费率和信令费率
