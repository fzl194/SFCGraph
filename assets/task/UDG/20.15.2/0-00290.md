# 配置全局缺省URR组（SET SPECTRAFURRGRP）

> 命令静态知识见 [SET SPECTRAFURRGRP](command/UDG/20.15.2/SET-SPECTRAFURRGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

可选收尾命令——配置整机级全局缺省URR组，承接 `SET URRGRPBINDING` 的全局继承开关（`GLBDFTURRGSW=ENABLE` / `GLBDFTSIGURRGSW=ENABLE`）。仅 URRGROUPNAME 一个必填参数（reference），取上游产出的 URRGROUP 名称。典型取异常/兜底费率组作为整机级全局缺省，处理两类流量：**特殊流量**（业务报文丢失未能完成七层精确匹配时已转发的流量，如在线用户下线时仅剩的信令报文，需 BIT1232 软参=1 才走本组计费）与**未配任何费率的业务流量**。

```
SET SPECTRAFURRGRP: URRGROUPNAME="UrrGp_abnormal";
```

**步骤位置**：原始部署文档步骤 10（计费收尾序列中），位于 `SET URRGRPBINDING`（步骤 9）之后、`ADD SPECURRGRPLIST`（步骤 11）/`SET REFRESHSRV`（步骤 12）之前。与 `SET URRGRPBINDING` 的全局继承开关配套——开关 ENABLE 时，USERPROFILE 未单独配缺省费率则继承本全局缺省。

**取值样例**：

| 参数 | 取值样例 | 来源 |
|---|---|---|
| URRGROUPNAME | `UrrGp_abnormal`（异常/兜底组，典型） | 引 [0-00002](0-00002.md) URRGROUP |

## 约束

- **rule-0-00290-1**（critical）高危：修改 URRGROUPNAME 会导致整机计费使用的全局缺省费率变化或业务不计费，造成运营商计费损失。**立即生效**。
- **rule-0-00290-2**（warning）规格：最大记录数 **1**（整机级唯一全局缺省），重复配置即覆盖。**立即生效**。
- **rule-0-00290-3**（warning）计费方式：本命令仅支持**时长和流量**计费方式，**不支持基于事件的计费方式** — 事件计费场景（如 GWFD-020306）配此全局缺省对事件计费无效。
- **rule-0-00290-4**（warning）前置依赖：URRGROUPNAME 必须已通过 [ADD URRGROUP](0-00002.md) 配置生成；若绑定的 URR 出现欠费，会影响特殊流量使用量上报。
- **rule-0-00290-5**（info）特殊流量触发：特殊流量走本组计费需 `BIT1232` 软参值设置为 1（开启特殊流量使用量上报功能）；本命令不区分业务费率和信令费率。
