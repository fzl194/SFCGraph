# 增加ICAP服务器（ADD ICAPSERVER）

> 命令静态知识见 [ADD ICAPSERVER](command/UDG/20.15.2/ADD-ICAPSERVER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务配置 UDG 与外部 ICAP Server（URL 分类查询服务）的互通参数。`ICAPSERVERNAME` 供后续 [ADD ICAPSVRBINDISG](task/UDG/20.15.2/0-00261.md) 引用绑定到 ICAP 服务器组。适用 NF：PGW-U、UPF。

本特性场景统一走 IPv4 路径（ICAPSVRIPTYPE=IPV4 固定），ICAPSERVERTYPE 固定 URL_FILTERING。

**取值样例（数据规划表，主特性 GWFD-110471）**

| 参数 | 含义 | 样例值 | 获取方法 |
|---|---|---|---|
| ICAPSERVERNAME | ICAP服务器名称 | icapserver1 | 全网规划（供 0-00261 引用） |
| ICAPSERVERTYPE | ICAP服务器类型 | URL_FILTERING | 固定取值 |
| ICAPSVRIPTYPE | IP地址类型 | IPV4 | 全网规划（本特性 IPv4 固定） |
| ICAPSERVERIPV4 | IPv4地址 | 172.16.39.136 | 全网规划（不支持 0.0.0.0 和 255.255.255.255） |
| VPNINSTANCE | VPN实例 | vpn-gcfif | 已配置数据中获取（引用 [ADD VPNINST](task/UDG/20.15.2/0-00041.md)，与 ADD LOGICINF 的 VPNINSTANCE 保持一致） |

**主场景脚本样例**

```
ADD ICAPSERVER:ICAPSERVERNAME="icapserver1", ICAPSERVERTYPE=URL_FILTERING, ICAPSVRIPTYPE=IPV4, ICAPSERVERIPV4="172.16.39.136", VPNINSTANCE="vpn-gcfif";
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：ICAPSERVERIPV6（IPv6 路径，本特性走 IPv4 未用）；ICAPSERVERPORT（默认 **1344**）；TCPRETRYTIME（默认 **5** 秒）；OPTIONINTERVAL（默认 **10** 秒）；CFGDOMAINNAME（配置域，optional）。均为 optional/conditional 且特性 md 未用，按冷门不绑，使用默认值即可。

## 约束

- 命令最大记录数 **30**；每种类型的 ICAP 服务器最多 **30 个**。
- ICAPSERVERIPV4 不支持 0.0.0.0 和 255.255.255.255；ICAPSERVERIPV6 不支持全 F。
- VPNINSTANCE 需与 ADD LOGICINF 的 VPNINSTANCE 保持一致（跨 atom 一致性编排）。
- 命令执行后**立即生效**。
