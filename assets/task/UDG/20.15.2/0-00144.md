# 配置静态路由双向BFD自动化模板(路由交叉)（ADD AUTOSCALINGSRBFD）

> 命令静态知识见 [ADD AUTOSCALINGSRBFD](command/UDG/20.15.2/ADD-AUTOSCALINGSRBFD.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是路由交叉功能（GWFD-020423）静态路由双向 BFD 检测的自动化模板，仅在 DP 0-00024 取 opt-bfd-two-way 时构建（与单臂 echo BFD [0-00143](task/UDG/20.15.2/0-00143.md) 二选一）。用于检测网络 B 链路双向可达性。采用双向 BFD 时，上行静态路由 [0-00142](task/UDG/20.15.2/0-00142.md) 不填 BFDTEMPLATENAME，且本任务须先于静态路由生效。双向 BFD 需配收/发间隔与检测倍数。

**配置范式**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| SERVICENAME | `ServName_networkB` | 引用 [0-00141](task/UDG/20.15.2/0-00141.md) | 已配置服务名 |
| IPVERSION | `IPv4` | 固定 | 当前路由交叉用 IPv4 |
| MINRXINTERVAL | 如 `500` | 全网规划（LLD min-rx-interval 毫秒） | 最小接收间隔，30~1000ms |
| MINTXINTERVAL | 如 `500` | 全网规划（LLD min-tx-interval 毫秒） | 最小发送间隔，30~1000ms |
| MULTIPLIER | 如 `4` | 全网规划（LLD detect-multiplier） | 检测倍数，3~50 |

**示例脚本**：
```
ADD AUTOSCALINGSRBFD: SERVICENAME="ServName_networkB", IPVERSION=IPv4,
  MINRXINTERVAL=500, MINTXINTERVAL=500, MULTIPLIER=4;
```

## 决策点

- **DP-0-00024**（BFD 类型）：选 opt-bfd-two-way（双向）→ 构建本任务，且须先于静态路由生效；[0-00142](task/UDG/20.15.2/0-00142.md) 不填 BFDTEMPLATENAME。选单臂→ 改用 [0-00143](task/UDG/20.15.2/0-00143.md)，本任务不建。

## 约束

- **rule-0-00232**（warning）最大记录数 2048；前置 SERVICENAME 值相同的 ADD AUTOSCALINGSERVICE（0-00141）须已配置；自动化配置开关须先关闭（SET AUTOCONFIG 关闭 → 执行 → 开启）；使用前须全局使能 BFD 功能；配置了使能 BFD 且 SERVICENAME 相同的静态路由（0-00142）须同步生效；立即生效 — 后果：超规格/前置缺失/开关未关/BFD 未全局使能→新增失败
