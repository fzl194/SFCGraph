# 配置静态路由双向BFD自动化模板(路由交叉)（ADD AUTOSCALINGSRBFD）

> 命令静态知识见 [ADD AUTOSCALINGSRBFD](command/UDG/20.15.2/ADD-AUTOSCALINGSRBFD.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD AUTOSCALINGSRBFD 用于配置路由交叉功能（GWFD-020423）静态路由**双向 BFD 检测**的自动化模板。**仅在 DP 0-00144 取 opt-bfd-two-way 时构建**（与单臂 echo BFD 0-00143 二选一）。用于检测网络 B 链路双向可达性。**采用双向 BFD 时，上行静态路由 [0-00142](task/UDG/20.15.2/0-00142.md) 不填 BFDTEMPLATENAME**，且本任务须先于静态路由生效。**双向 BFD 需配收/发间隔与检测倍数**。1 个特性命中（020423 路由交叉）。

### 配置维度 1：地址族（参数 IPVERSION，必选）

| 取值 | 配套探测参数 | 适用场景 |
|---|---|---|
| **IPv4**（固定） | MINRXINTERVAL/MINTXINTERVAL/MULTIPLIER | 020423 当前唯一场景 |
| IPv6 | 同上 IPv6 参数族 | IPv6 路由交叉（未用） |

### 配置维度 2：探测间隔与倍数（参数 MINRXINTERVAL + MINTXINTERVAL + MULTIPLIER）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **MINRXINTERVAL** | 30~1000 ms | 最小接收间隔（参见 LLD min-rx-interval） |
| **MINTXINTERVAL** | 30~1000 ms | 最小发送间隔（参见 LLD min-tx-interval） |
| **MULTIPLIER** | 3~50 | 检测倍数（参见 LLD detect-multiplier） |

> 三者均全网规划；双向 BFD 需配收/发**双**间隔，单臂 echo 仅需收包间隔。

### 配置维度 3：服务名引用（参数 SERVICENAME，必选）

| 取值 | 来源 |
|---|---|
| **与 ADD AUTOSCALINGSERVICE 一致的服务名** | 引用 [0-00141](task/UDG/20.15.2/0-00141.md).SERVICENAME |

**典型脚本（020423 双向 BFD IPv4 范式）**：

```
ADD AUTOSCALINGSRBFD: SERVICENAME="ServName_networkB", IPVERSION=IPv4,
  MINRXINTERVAL=500, MINTXINTERVAL=500, MULTIPLIER=4;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：配双向 BFD 自动化模板"——前置 [0-00141](task/UDG/20.15.2/0-00141.md) ADD AUTOSCALINGSERVICE 配 SERVICENAME；后续 [0-00142](task/UDG/20.15.2/0-00142.md) ADD AUTOSCALINGSRROUTE **不填 BFDTEMPLATENAME** 并须同步生效。

## 决策点：BFD 类型（DP 0-00144）

| 选项 | 联动模板 | 适用场景 |
|---|---|---|
| **opt-bfd-two-way（双向）** | 本任务（须先于静态路由生效）+ [0-00142](task/UDG/20.15.2/0-00142.md) 不填 BFDTEMPLATENAME | 双向 BFD 探测 |
| **opt-bfd-one-arm（单臂 echo）** | [0-00143](task/UDG/20.15.2/0-00143.md) ADD AUTOSCALINGBFD | 单臂 echo BFD |

> 单臂/双向二选一：选双向 → 构建本任务 + SRROUTE 不填 BFDTEMPLATENAME；选单臂 → 改用 0-00143，本任务不建。

## 约束

- **最大记录数 2048**（warning，rule-0-00144）：系统最多 2048 条 AUTOSCALINGSRBFD — 后果：超规格新增失败
- **SERVICENAME 前置必备**（critical，rule-0-00144）：前置 SERVICENAME 值相同的 ADD AUTOSCALINGSERVICE(0-00141) 须已配置 — 后果：前置缺失命令拒收
- **静态路由同步生效**（critical，rule-0-00144）：配置了使能 BFD 且 SERVICENAME 相同的静态路由(0-00142) 须同步生效 — 后果：静态路由未同步则 BFD 不可用
- **自动化配置开关先关闭**（critical，rule-0-00144）：执行前须先 `SET AUTOCONFIG 关闭`，执行后再开启 — 后果：未关开关命令拒收
- **BFD 全局使能**（critical，rule-0-00144）：使用前须全局使能 BFD 功能 — 后果：BFD 未全局使能命令拒收
- **探测参数范围**（warning，rule-0-00144）：MINRX/MINTX 30~1000ms / MULTIPLIER 3~50 — 后果：超范围命令拒收
- **立即生效**（warning，rule-0-00144）：执行后立即生效 — 后果：误配即刻影响 BFD 双向探测
