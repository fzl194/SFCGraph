# 创建逻辑接口（ADD LOGICINF）

> 命令静态知识见 [ADD LOGICINF](command/UDG/20.15.2/ADD-LOGICINF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD LOGICINF 是用户面逻辑接口创建命令，按"用途维度"区分 5 类场景。8 个特性数据规划表仅用 7 个参数，5 个未绑参数（IPV4ADDRESS2/MASK2 次地址、IPV4MTU/IPV6MTU 默认 1500/1800、SLICEATTRSW 切片）6 特性均未用，不绑。NAME 取值随用途变化（Paif1/1/0、phif1/0/0、giif1/0/1、s11-uif1/1/0），IPVERSION 由 DP 0-00017 IPv4/IPv6 分叉，VPNINSTANCE 用途相关。

**用途维度范式（从证据数据规划表归纳）**

| 用途 | 典型特性 | NAME | IPVERSION | 地址 | VPNINSTANCE | 脚本样例 |
|---|---|---|---|---|---|---|
| Paif（ePDG IPv4 互通） | GWFD-010155 VPN+OSPF/缺省路由/静态+BFD | Paif1/1/0 | IPv4 | IPV4ADDRESS1=10.10.10.10,IPV4MASK1=255.255.255.255 | vpn-s2b（已配置数据中获取） | `ADD LOGICINF:NAME="Paif1/1/0",IPVERSION=IPV4,IPV4ADDRESS1="10.10.10.10",IPV4MASK1="255.255.255.255",VPNINSTANCE="vpn-s2b";` |
| Paif（ePDG IPv6 互通） | GWFD-010155 OSPFv3 | Paif1/0/0 | IPV6 | IPV6ADDRESS1=fc00::1,IPV6PREFIXLEN1=128 | vpn-paif（从已配置数据中获取） | `ADD LOGICINF:NAME="Paif1/0/0",IPVERSION=IPV6,IPV6ADDRESS1="fc00::1",VPNINSTANCE="vpn-paif",IPV6PREFIXLEN1=128;` |
| IPFarm 心跳本端接口 | GWFD-110281 Portal、GWFD-110282 Web Proxy | phif1/0/0（或 phif1/0/1） | IPV4 | IPV4ADDRESS1=10.10.10.10（地址池预留）,IPV4MASK1=255.255.255.255 | 不配 | `ADD LOGICINF:NAME="phif1/0/0",IPVERSION=IPV4,IPV4ADDRESS1="10.10.10.10",IPV4MASK1="255.255.255.255";` |
| IPFarm-IMS（P-CSCF 故障恢复） | GWFD-020253 | phif1/0/0 | IPV4 | IPV4ADDRESS1=192.168.6.6,IPV4MASK1=255.255.255.255 | ims_vpn（全网规划） | `ADD LOGICINF:NAME="phif1/0/0",IPVERSION=IPV4,IPV4ADDRESS1="192.168.6.6",VPNINSTANCE="ims_vpn",IPV4MASK1="255.255.255.255";` |
| L2TP 源端 Giif | GWFD-020412 AAA/本地配置 | giif1/0/1（或 giif1/0/0） | IPV4 | IPV4ADDRESS1=10.8.20.2,IPV4MASK1=255.255.255.255 | vpn_l2tp（须与 APN 绑定 VPN 一致） | `ADD LOGICINF:NAME="giif1/0/1",IPVERSION=IPV4,IPV4ADDRESS1="10.8.20.2",IPV4MASK1="255.255.255.255",VPNINSTANCE="vpn_l2tp";` |
| S11-U 信令面数据传输 | GWFD-110606 | s11-uif1/1/0 | IPv4 | IPV4ADDRESS1=10.2.151.15,IPV4MASK1=255.255.255.255 | 不配 | `ADD LOGICINF:NAME="s11-uif1/1/0",IPVERSION=IPV4,IPV4ADDRESS1="10.2.151.15",IPV4MASK1="255.255.255.255";` |
| ICAP Server 互通 | GWFD-110471 | gcfif1/0/0 | IPV4 | IPV4ADDRESS1=10.10.1.7,IPV4MASK1=255.255.255.255 | vpn-gcfif（已配置数据中获取） | `ADD LOGICINF:NAME="gcfif1/0/0",IPVERSION=IPV4,IPV4ADDRESS1="192.168.1.1",IPV4MASK1="255.255.255.255",VPNINSTANCE="vpn-gcfif";` |

**参数取值规则**
- **NAME**：本端规划，取值随用途约定（Paif/phif/giif/s11-uif/gcfif 等），外联口 VPN 须与逻辑接口 VPN 相同否则报文查路由失败（GWFD-010155 md 说明）。
- **IPVERSION**：decision_driven→DP 0-00017，IPv4 配 IPV4ADDRESS1/IPV4MASK1，IPv6 配 IPV6ADDRESS1/IPV6PREFIXLEN1。
- **VPNINSTANCE**：conditional，**用途相关**——Paif→0-00071 L3VPNINST / L2TP-Giif→0-00041 VPNINST（须与 APN 绑定 VPN 一致）/ IPFarm-IMS→ims_vpn / IPFarm·S11-U 无；绑定时须先建对应 VPN 实例。binding 级单 source_ref 为 schema 限制，默认 0-00071。
- **IPFarm 心跳地址**：心跳检测的本端 IP 一般从终端地址池中获取，配置地址池时需预留一个 IP 给 IP Farm 心跳检测。

## 约束

- **rule 0-00175**（cardinality/warning）ADD LOGICINF 命令最大记录数为 4096；当配置记录数大于规格 98% 时上报 ALM-135602215 配置数量超阈值告警（≤95% 恢复，阈值可 MOD CFGTHRESHOLD 修改）；若需绑定 VPN（VPNINSTANCE），须提前配置 ADD VPNINST/L3VPNINST；执行后立即生效 — 后果：超规格新增失败；未前置 VPN 即绑 VPNINSTANCE 将失败；接近阈值触发告警
