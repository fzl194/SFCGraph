# IPSec 隧道建立（VNRS+IPsec 双配 + ACL + 提议 + IKE Peer + 策略 + 应用）

> IPSec 隧道建立的**9 步可复用骨架**（接入方式隧道族共享）。被引用于 [2-00044](task/UDG/20.15.2/2-00044.md) IPFD-015004 IPSec。
> 2026-07-12 建（防平铺，SOP §3.5——9 步阶段内聚 + 隧道族复用：IPSec/GRE over IPSec/OSPF over IPSec/主备隧道/多 Sequence/国密 IKEv1）。
> **★ UDG 独有 VNRS + IPsec 微服务双配架构**：VNRS 负责外部通信/流量引流，IPsec 负责 IKE 协商/加解密；两微服务 Tunnel 接口名/IP/类型/VPN **必须一对一配置**（硬约束）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. VNRS 微服务 VPN 与隧道接口 | `ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD INTERFACE` + `ADD IPSECINTFCFG` + `ADD IPBINDVPN` + `ADD IFIPV4ADDRESS` | VRFNAME=vrf1；IFNAME=Tunnel10, IFADMINSTATUS=up；IPSECINTFCFG: INTERFACENAME=Tunnel10, **TNLTYPE=IPSEC**；IFIPADDR=隧道接口 IP /32 |
| 2.（VNRS 侧）静态路由引流 | `ADD SRROUTE` | AFTYPE=ipv4unicast, PREFIX=对端网段, MASKLENGTH, IFNAME=Tunnel10, NEXTHOP=对端 Tunnel IP |
| 3. IPsec 微服务双配 VPN 与隧道接口（★关键）| `ADD L3VPNINSTIPSEC` + `ADD VPNINSTAFIPSEC` + `ADD INTERFACEIPSEC` + `ADD IPBINDVPNIPSEC` + `ADD IFIPV4ADDRESSIPSEC` | **接口名/IP/VPN 必须与步骤 1 VNRS 侧完全一致**（IFNAME=Tunnel10, 同 IP, 同 VRFNAME） |
| 4. 定义需保护的数据流（ACL）| `ADD ACLGROUPIPSEC` + `ADD ACLRULEADV4IPSEC` | ACLNAME=3000；ACLPROTOCOL=0, ACLSOURCEIP=源 IP, ACLSRCWILD, ACLDESTIP=目的 IP, ACLDESTWILD（**仅源/目的 IP，端口不生效**）|
| 5. IPsec 安全提议 | `ADD IPSECPROPOSALIPSEC` | PROPOSALNAME=tran1, **IPSECPROTOCOL=Esp**, ESPAUTHALGO=Sha2_256, ESPENCRYPTALGO=Aes_256, ENCAPMODE=**Tunnel** |
| 6. IKE 安全提议 | `ADD IKEPROPOSAL` | PROPOSALNUMBER=10, AUTHMETHOD=**Pre_share**, AUTHALGORITHM=Sha2_256, DHGROUP=**Dh_group19**（不可为 None）, SADURATION=3600 |
| 7. IKE 对等体 | `ADD IKEPEER` | PEERNAME=b, **PRESHAREDKEY=abcde**（两端必一致）, EXCHANGEMODE=Main/Aggressive, PROPOSAL=10, LOWREMOTEADDR=对端 IP（**不能为地址段**）, NATTRAVERSAL=TRUE/FALSE, INVRFNAME/OUTVRFNAME |
| 8. IPsec 安全策略 + 引用提议 + 绑定 IKE Peer | `ADD IPSECPOLICY` + `ADD PROPATTACHIPSECPROPOSAL` + `ADD ATTACHIKEPEER` | POLICYNAME=map1, SEQUENCENUMBER=10, POLICYMODE=Isakmp, ACLNUMBER=3000；IPSECPROPNAME=tran1；IKEPEERNAME=b, PEERPRIORITY=1（支持主备多 Peer）|
| 9. 应用策略到 Tunnel 接口 +（可选）DPD/NAT 保活 | `ADD IPSECINTFCFGIPSEC` + `SET IKEGLOBALCONFIG` | IPSECINTFCFGIPSEC: INTERFACENAME=Tunnel10, TNLTYPE=IPSEC, POLICYNAME=map1；IKEGLOBALCONFIG: DPDTYPE=Periodic/On_demand, DPDINTERVAL=10, DPDRETRYINTRVL=3, NATKLI=30 |

**典型脚本（普通 IPv4 IPSec 隧道，ESP+IKEv2+DH group19）**：
```
// === 步骤 1-2：VNRS 微服务 VPN+Tunnel+静态路由 ===
ADD L3VPNINST:VRFNAME="vrf1";
ADD VPNINSTAF:VRFNAME="vrf1",AFTYPE=ipv4uni;
ADD INTERFACE:IFNAME="Tunnel10",IFADMINSTATUS=up;
ADD IPSECINTFCFG:INTERFACENAME="Tunnel10",TNLTYPE=IPSEC;
ADD IPBINDVPN:IFNAME="Tunnel10",VRFNAME="vrf1";
ADD IFIPV4ADDRESS:IFNAME="Tunnel10",IFIPADDR="192.168.1.1",SUBNETMASK="255.255.255.255";
ADD SRROUTE:VRFNAME="vrf1",AFTYPE=ipv4unicast,PREFIX="10.1.2.0",MASKLENGTH=24,IFNAME="Tunnel10",NEXTHOP="192.168.1.2";

// === 步骤 3：IPsec 微服务双配（接口名/IP/VPN 与 VNRS 一致）===
ADD L3VPNINSTIPSEC:VRFNAME="vrf1";
ADD VPNINSTAFIPSEC:VRFNAME="vrf1",AFTYPE=Ipv4uni;
ADD INTERFACEIPSEC:IFNAME="Tunnel10",IFADMINSTATUS=Up;
ADD IPBINDVPNIPSEC:IFNAME="Tunnel10",VRFNAME="vrf1";
ADD IFIPV4ADDRESSIPSEC:IFNAME="Tunnel10",IFIPADDR="192.168.1.1",SUBNETMASK="255.255.255.255";

// === 步骤 4：ACL 数据流（源/目的 IP）===
ADD ACLGROUPIPSEC:ACLNAME="3000";
ADD ACLRULEADV4IPSEC:ACLNAME="3000",ACLRULENAME="rule_1",ACLACTION=Permit,VRFNAME="vrf1",ACLPROTOCOL=0,ACLSOURCEIP="10.1.1.2",ACLSRCWILD="0.0.0.0",ACLDESTIP="10.1.2.2",ACLDESTWILD="0.0.0.0";

// === 步骤 5-7：IPsec 提议 + IKE 提议 + IKE Peer ===
ADD IPSECPROPOSALIPSEC:PROPOSALNAME="tran1",ESPAUTHALGO=Sha2_256,ESPENCRYPTALGO=Aes_256,IPSECPROTOCOL=Esp,ENCAPMODE=Tunnel;
ADD IKEPROPOSAL:PROPOSALNUMBER=10,AUTHMETHOD=Pre_share,AUTHALGORITHM=Sha2_256,DHGROUP=Dh_group19;
ADD IKEPEER:PEERNAME="b",PRESHAREDKEY="abcde",NATTRAVERSAL=TRUE,PROPOSAL=10,LOWREMOTEADDR="192.168.1.2",INVRFNAME="vrf1",OUTVRFNAME="vrf1";

// === 步骤 8：IPsec 安全策略（引用提议 + 绑定 IKE Peer）===
ADD IPSECPOLICY:POLICYNAME="map1",SEQUENCENUMBER=10,POLICYMODE=Isakmp,TEMPLATEMODE=None,ACLNUMBER=3000;
ADD PROPATTACHIPSECPROPOSAL:POLICYNAME="map1",SEQUENCENUMBER=10,POLICYMODE=Isakmp,TEMPLATEMODE=None,IPSECPROPNAME="tran1";
ADD ATTACHIKEPEER:POLICYNAME="map1",SEQUENCENUMBER=10,POLICYMODE=Isakmp,TEMPLATEMODE=None,IKEPEERNAME="b",PEERPRIORITY=1;

// === 步骤 9：应用策略 + DPD/NAT 保活（可选）===
ADD IPSECINTFCFGIPSEC:INTERFACENAME="Tunnel10",TNLTYPE=IPSEC,POLICYNAME="map1";
SET IKEGLOBALCONFIG:DPDINTERVAL=10,DPDTYPE=Periodic,DPDRETRYINTRVL=3,NATKLI=30;
```

> 对端 Device B 脚本对称：PEERNAME 改指向 A，LOWREMOTEADDR 互换，ACLSOURCEIP/ACLDESTIP 对调。

**步骤位置**：在 [1-00029](task/UDG/20.15.2/1-00029.md) APN 接入域基础设施（若 VPN 组网）之后；IPSec 隧道建立是接入方式核心。GRE over IPSec 场景下，先建 [1-00032](task/UDG/20.15.2/1-00032.md) GRE 隧道，再叠加本骨架。

## 场景差异

| 场景 | 安全协议 | IKE 版本 | 地址族 | 主备 | NAT 穿越 | 复用说明 |
|---|---|---|---|---|---|---|
| 普通 IPv4 IPSec | ESP | IKEv2（缺省）| IPv4 | 无 | 可选 | 本骨架基线 |
| 普通 IPv6 IPSec | ESP | IKEv2 | IPv6 | 无 | 不支持（NAT 穿越不支持 IPv6） | 步骤 1/3 改 IPv6 地址族 + ADD IFIPV6ADDRESSIPSEC；UDG 20.8.0+（v02）|
| IPv4 主备隧道 | ESP | IKEv2 | IPv4 | **步骤 8 多 ATTACHIKEPEER + PEERPRIORITY 优先级**（热备一主一备 / 冷备最多一主两备）| 可选 | 步骤 7/8 多 IKE Peer |
| GRE over IPSec | ESP | IKEv2 | IPv4 | 无 | 可选 | **前置** [1-00032](task/UDG/20.15.2/1-00032.md) GRE 隧道（Tunnel1）封装组播/广播；本骨架 Tunnel 用 Tunnel2；ACL 匹配 GRE 源/目的（LoopBack IP），非用户 IP |
| OSPF over IPSec | ESP | IKEv2 | IPv4 | 无 | 可选 | OSPF 引流到 IPSec；步骤 4 ACL 匹配 OSPF 组播（需 GRE 封装）|
| 多 Sequence 策略 | ESP | IKEv2 | IPv4 | 无 | 可选 | 步骤 8 同 POLICYNAME 多 SEQUENCENUMBER + 多 ACL |
| 指定本端接口建立 | ESP | IKEv2 | IPv4 | 无 | 可选 | 步骤 7 IKE Peer 指定非 Tunnel 接口为 IKE 协商 IP |
| 国密 IKEv1 | ESP | **IKEv1** + 国密 | IPv4 | 无 | 可选 | 步骤 5/6 算法走国密（SM3/SM4/SM2）+ 步骤 7 EXCHANGEMODE + 国密证书 |
| IKEv1 主模式 | ESP/AH/Ah_esp | IKEv1 主模式 | IPv4 | 无 | 可选 | 步骤 7 EXCHANGEMODE=Main + LOCALIDTYPE=Ip |
| IKEv1 野蛮模式 | ESP/AH/Ah_esp | IKEv1 野蛮模式 | IPv4 | 无 | 可选 | 步骤 7 EXCHANGEMODE=Aggressive（远程访问/地址动态场景）|
| 仅 AH 认证（不加密）| AH | IKEv2 | IPv4 | 无 | 不支持 | 步骤 5 IPSECPROTOCOL=Ah + AHAUTHALGO（冷门）|

> **★ 互斥约束**：IPSec 隧道源地址**不能与 GRE 隧道源地址相同**（应用限制硬约束，见 [1-00032](task/UDG/20.15.2/1-00032.md) GRE 隧道）。GRE over IPSec 场景下 GRE 用 LoopBack1、IPSec 用 Tunnel10+物理接口 IP，规避源地址冲突。

## 决策点

| 选项 | 影响（参数 + 联动） |
|---|---|
| 安全协议=Esp（主推）| 步骤 5 IPSECPROTOCOL=Esp + ESPAUTHALGO + ESPENCRYPTALGO；步骤 6 IKE 提议独立 |
| 安全协议=Ah_esp（双认证）| 步骤 5 IPSECPROTOCOL=Ah_esp + ESPAUTHALGO + ESPENCRYPTALGO + AHAUTHALGO |
| 安全协议=Ah（仅认证）| 步骤 5 IPSECPROTOCOL=Ah + AHAUTHALGO；不加密 |
| IKE 版本=IKEv2（缺省）| 步骤 7 IKEPEER 不显式设或 MOD IKEPEER VERSION1=FALSE |
| IKE 版本=IKEv1 | 步骤 7 EXCHANGEMODE=Main/Aggressive + VERSION1=TRUE；存在安全风险 |
| 主备=无 | 步骤 8 单 ATTACHIKEPEER |
| 主备=热备 | 步骤 8 多 ATTACHIKEPEER，PEERPRIORITY 区分优先级；主备自动协商 SA，立即切换（一主一备）|
| 主备=冷备 | 步骤 8 多 ATTACHIKEPEER；备用默认不协商，切换时再协商（最多一主两备）|
| DPD=Periodic（周期）| 步骤 9 DPDTYPE=Periodic + DPDINTERVAL + DPDRETRYINTRVL |
| DPD=On_demand（按需）| 步骤 9 DPDTYPE=On_demand；本端无加密流量时不发 DPD |
| NAT 穿越=TRUE | 步骤 7 NATTRAVERSAL=TRUE + 步骤 9 NATKLI=30；**仅 ESP 隧道模式，不支持 IPv6** |

> 联动特性级 DP：安全协议 × IKE 版本 × 主备 × NAT 穿越由 [2-00044](task/UDG/20.15.2/2-00044.md) 场景 DP 决定。

## 约束

- **VNRS/IPsec 微服务双配**（critical）：两微服务 Tunnel 接口名/IP/类型/VPN/指定本端接口**必须一对一配置**，否则业务不通 — 后果：隧道 DOWN。
- **GRE 源地址互斥**（critical）：IPSec 隧道源地址不能与 GRE 隧道源地址相同 — 后果：应用限制违反（见 [1-00032](task/UDG/20.15.2/1-00032.md)）。
- **PSK 两端必一致**（critical）：PRESHAREDKEY 两端必须相同，否则 SA 建立不成功 — 后果：SA 协商失败。
- **DHGROUP 不可为 None**（critical）：DHGROUP 不能为 None 或不配置，建议 Dh_group19 — 后果：SA 协商失败。
- **远程地址不可为地址段**（critical）：LOWREMOTEADDR 必须为具体 IP — 后果：命令拒收。
- **ACL 仅源/目的 IP**（critical）：端口配置不生效；ACLPROTOCOL=0（任意）；步骤 4 仅配 ACLSOURCEIP/ACLDESTIP — 后果：数据流不匹配，隧道不传输。
- **隧道两端路由可达**（critical）：IKE 协商前提 — 后果：隧道 DOWN。
- **算法两端须一致**（critical）：IPSECPROTOCOL/ESPAUTHALGO/ESPENCRYPTALGO/AHAUTHALGO/ENCAPMODE/AUTHALGORITHM/ENCRALGORITHM/INTEGALGORITHM/DHGROUP 两端协商一致 — 后果：SA 协商失败。
- **NAT 穿越仅 ESP 隧道模式**（warning）：不支持 IPv6 组网；仅 IPSECPROTOCOL=Esp + ENCAPMODE=Tunnel 场景 — 后果：NAT 场景隧道建立失败。
- **AH+ESP 须同封装模式**（warning）：同时采用 AH 和 ESP 时必须相同 ENCAPMODE — 后果：协商失败。
- **IKEv1 安全风险**（warning）：IKEv1 存在公认密码学漏洞；UDG 缺省 IKEv2；MOD IKEPEER VERSION1=FALSE 关闭 IKEv1 — 后果：弱安全。
- **对系统性能有影响**（info）：IPSec 封装+加密运算开销；ACL 优化减少不必要加密。
- **无需 License**（info）：本特性无需 License。
- **C-U 对称**（info）：UDG/UNC 两侧配置同构；UNC 对应 IPFD-016000（UNC 20.6.0+）。
- **首次发布版本**（info）：UDG 20.5.0；IPv6/NAT 穿越/主备需 UDG 20.8.0+（v02）。
