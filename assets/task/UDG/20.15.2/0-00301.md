# 配置地址池冲突IPv4地址（ADD CONFLICTIP）

> 命令静态知识见 [ADD CONFLICTIP](command/UDG/20.15.2/ADD-CONFLICTIP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。POOLNAME 引用 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL 已建地址池；典型前置链：建 POOL(0-00043) → 配地址段(0-00044) → 绑地址池组(0-00045) → 标记冲突 IP(0-00301)。

## 配置方法（配置方法字典）

ADD CONFLICTIP 是地址池族链路的"冲突地址标记层"：在已建的本地地址池(POOLNAME)中标识指定 IPv4 地址为冲突状态，禁止设备地址被分配给终端用户（如该地址与系统或其他设备本身的地址冲突时使用）。命令核心配置空间有 3 个维度：**地址池引用**（POOLNAME 必选，引用已建 POOL）、**冲突地址来源**（IPADDRESS 必选，全网规划）、**配额上限**（每池最多 16 条冲突 IPv4，全局记录数最大 10000）。1 个特性命中（GWFD-010105 用户面地址分配基于 SMF 分配地址）覆盖典型范式。

### 配置维度 1：地址池引用（参数 POOLNAME，必选）

POOLNAME 决定该冲突 IPv4 隶属于哪个已建地址池：

| 取值 | 作用 | 联动必选参数 | 前置依赖 |
|---|---|---|---|
| **已建 POOLNAME** | 字符串 1~79 字节，由 `_`、`-`、数字、大小写字母和 `.` 组成，不以 `.` 开头且无连续 `..` | IPADDRESS | 必须先 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL 建好；POOLTYPE=LOCAL 时必须先 [0-00044](task/UDG/20.15.2/0-00044.md) ADD SECTION 配地址段 |

> **POOLNAME 命名约束**：1~79 字节，字符集 `_`、`-`、数字、大小写字母、`.`，不能以 `.` 开头且不能出现连续两个 `.`，不区分大小写。

### 配置维度 2：冲突 IPv4 地址（参数 IPADDRESS，必选）

IPADDRESS 决定被标记为冲突状态的 IPv4 地址本身：

| 取值 | 作用 | 数据来源 |
|---|---|---|
| **IPv4 地址**（如 10.1.1.1） | 标记该 IPv4 为冲突状态，禁止分配给终端用户 | 全网规划（待标记的冲突地址） |

> **生效机制**：命令执行后**只对新激活用户生效**；若地址已分配给终端用户，命令仍可执行成功，但配置生效要等到下次分配该地址时。

### 配置维度 3：配额上限（命令 notes 投影）

| 配额维度 | 上限 | 说明 |
|---|---|---|
| **每池冲突 IPv4 数** | 最多 **16** 条/POOL | 超过时命令失败 |
| **全局总记录数** | 最多 **10000** 条 | 命令 notes 硬规格 |

**典型脚本（010105 用户面地址分配，SMF 分配地址场景，标记冲突 IP）**：
```
ADD CONFLICTIP:POOLNAME="testpool1",IPADDRESS="10.1.1.1";
```

> 命令 notes：执行后只对新激活用户生效；最大记录数 10000；配置前需先 ADD POOL 建池；冲突地址池可用地址数减少；地址若已分配给用户则在下次分配时再生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 5：地址池冲突标记"——前置依赖 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL（建地址池，POOLNAME）+ [0-00044](task/UDG/20.15.2/0-00044.md) ADD SECTION（LOCAL 池配地址段，POOLTYPE=LOCAL 时必选）；上游被 [0-00045](task/UDG/20.15.2/0-00045.md) ADD POOLBINDGROUP（绑地址池组）穿透引用；冲突地址不需后续命令。

## 决策点：地址池与冲突 IPv4 选择（DP 0-00301）

问：在哪个地址池下标记哪个 IPv4 为冲突？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| LOCAL 池单条冲突 IPv4 | POOLNAME（LOCAL 池） + IPADDRESS（单 IPv4） | 010105 SMF 分配地址场景，LOCAL 池标记特定冲突 IPv4 |
| EXTERNAL 池单条冲突 IPv4 | POOLNAME（EXTERNAL 池） + IPADDRESS（单 IPv4） | EXTERNAL 池下若仍需标记冲突 IP（010105 子场景） |

> 1 个特性命中（010105）以 LOCAL 池单条冲突 IPv4 为典型范式；EXTERNAL 池理论上也可标记但官方示例未覆盖。

## 约束

- **POOLNAME 必须先建**（critical，rule-0-00301）：POOLNAME 必须先由 ADD POOL 命令建好 — 后果：未建池命令拒收
- **每池最多 16 条冲突 IPv4**（critical，rule-0-00301）：一个地址池最多可设置 16 个 IPv4 地址为冲突状态 — 后果：超规格新增失败
- **最大记录数 10000**（warning，rule-0-00301）：该命令全局最大记录数为 10000 — 后果：超规格新增失败
- **执行后只对新激活用户生效**（warning，rule-0-00301）：命令执行后只对新激活用户生效；地址已分配给用户则在下次分配时再生效 — 后果：误以为立即生效致地址仍被分配
- **适用 NF 限定**（warning，rule-0-00301）：PGW-U、UPF — 后果：SGW-U 等不适用
- **POOLNAME 命名约束**（warning，rule-0-00301）：1~79 字节，字符集 `_`、`-`、数字、大小写字母、`.`，不以 `.` 开头且无连续 `..` — 后果：不符合命名规范命令拒收
- **冲突地址池可用地址数减少**（info，rule-0-00301）：配置为冲突状态的 IPv4 不会分配给用户，该地址池可用地址数减少 — 后果：池可用地址减少
