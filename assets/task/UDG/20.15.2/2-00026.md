# 基于累计流量的策略控制（GWFD-020353）

> 特性静态知识见 [GWFD-020353](feature/UDG/20.15.2/GWFD-020353.md)。本页讲配置生成实例化时这个特性**怎么配**。
> **文档完整度说明**：本特性无独立激活/配置文档（仅有特性概述+实现原理），是 **License 门控 + 复用 PCC backbone** 的能力型特性（类似 [2-00020](task/UDG/20.15.2/2-00020.md) SA特征库更新管控）。配置流程按特性概述 + PCC backbone 归纳。

## 配置概览

基于累计流量的策略控制（FUP 基础版，2/3/4G 场景）针对用户**会话级累计流量**（整个 PDU 会话，不区分业务类型）进行监控。当累计流量达预设阈值时，由 **PCRF/PCF 下发策略**（降速/阻塞/改费率）。**核心机制：License 门控 + 复用 PCC 基本功能**——UDG 侧仅开 License，累计流量经 PCC 触发链（URR 上报）→ PGW-C/SMF → PCRF/PCF 决策 → 下发新策略。区别于 [2-00027](task/UDG/20.15.2/2-00027.md)（GWFD-110312 业务级累计流量，区分业务类型）。

## 配置流程（步骤编排，每步显示含命令）

1. **License 前置**（核心，单命令）：`SET LICENSESWITCH`（LKV3G5PCBT01）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **复用 PCC 基本功能 backbone**（前置依赖，详见 [2-00018](task/UDG/20.15.2/2-00018.md)）：PCC 策略组 + 过滤链 + 规则绑定（URR 上报累计流量→PCRF 决策）→ [1-00009](task/UDG/20.15.2/1-00009.md) + [1-00010](task/UDG/20.15.2/1-00010.md) + [1-00011](task/UDG/20.15.2/1-00011.md)

> **参数证据链**（R1.5）：SET LICENSESWITCH（LKV3G5PCBT01）来自特性概述（License 82200AFM）；FUP 阈值/动作策略由 PCRF/PCF 下发（非 UDG 侧配置），UDG 仅承载累计流量上报（PCC 触发链）。
>
> **activation 模板复用说明**（R1.4）：本特性无独立 activation 文档（仅有特性概述+实现原理）。配置流程（License + 复用 PCC backbone）按特性概述 §License（"加载 License 后即可使用，通过 SET LICENSESWITCH 打开开关"）+ §依赖（"需开启 PCC 基本功能以获取策略"）归纳。FUP 的阈值判定与策略下发在 PCRF/PCF 侧，UDG 侧无独立配置对象。

## 决策点

| 选项 | 影响 |
|---|---|
| 已加载 License LKV3G5PCBT01 + PCC backbone | 会话级累计流量经 URR 上报→PCRF/PCF，达阈值下发 FUP 策略 |
| 未加载 License | FUP 功能不可用（累计流量不触发策略变更）|

> FUP 阈值/周期/动作（降速/阻塞/改费率）由 PCRF/PCF 侧策略决定，UDG 侧不配置——配置生成不产出 FUP 阈值参数（无证据，R1.5）。

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5PCBT01` — 未开则 FUP 不生效。
- **依赖 PCC 基本功能**（critical）：GWFD-020351（[2-00018](task/UDG/20.15.2/2-00018.md)）—— 累计流量上报与策略下发经 PCC 触发链 — 未开则 FUP 无策略执行路径。
- **会话级粒度**（info）：本特性以整个 PDU 会话为粒度统计累计流量（不区分业务）；业务级累计流量见 GWFD-110312（[2-00027](task/UDG/20.15.2/2-00027.md)）。
- **策略在 PCRF/PCF 侧**（info）：阈值判定与降速/阻塞/改费率动作由 PCRF/PCF 下发，UDG 侧仅承载上报与执行。
