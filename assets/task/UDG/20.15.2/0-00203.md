# 增加流策略并绑定CB对（ADD MQCPOLICY + ADD MQCPOLICYNODE）

> 命令静态知识见 [ADD MQCPOLICY](command/UDG/20.15.2/ADD-MQCPOLICY.md)、[ADD MQCPOLICYNODE](command/UDG/20.15.2/ADD-MQCPOLICYNODE.md)。本页只讲配置生成实例化时这两条命令**怎么配**。

## 配置方法

聚合 atom：先 [ADD MQCPOLICY] 建流策略容器，再用 [ADD MQCPOLICYNODE] 把 C 侧（CLASSIFIERNAME）和 B 侧（BEHAVIORNAME）绑成 CB 对挂进策略。两者都引用上游已建对象，不本端规划。

**数据规划取值样例**（证据③，IPFD-012001 数据规划表）：

| 参数 | 取值样例 | 获取方法 | 所属命令 | 说明 |
|---|---|---|---|---|
| POLICYNAME（策略名称） | `policy1`、`policy2` | 本端规划 | ADD MQCPOLICY | 策略容器唯一标识 |
| CLASSIFIERNAME（分类名称） | `class1` | 引用（0-00201） | ADD MQCPOLICYNODE | CB 对的 C 侧 |
| BEHAVIORNAME（行为名称） | `behavior1` | 引用（0-00199） | ADD MQCPOLICYNODE | CB 对的 B 侧 |
| PRECEDENCE（优先级） | `1` | 本端规划 | ADD MQCPOLICYNODE | optional，多 CB 节点匹配顺序，值小优先 |

**典型范式**（证据③任务示例脚本，两步：建策略 + 绑 CB 对）：
```
ADD MQCPOLICY: POLICYNAME="policy1";
ADD MQCPOLICYNODE: POLICYNAME="policy1", CLASSIFIERNAME="class1", BEHAVIORNAME="behavior1";
```

> 一个 POLICYNAME 下可挂多个 CB 节点（多次执行 MQCPOLICYNODE），按 PRECEDENCE 决定多规则匹配优先级。主场景示例 PRECEDENCE 未显式传参。MQCPOLICY 命令真相另有 STCENABLE（统计使能，default disable）/SHAREMODE（共享模式），本特性未用，主场景不绑。

## 约束

- **rule-0-00404**（warning）ADD MQCPOLICY 最大记录数 **1023**；执行后立即生效；绑 CB 对（MQCPOLICYNODE）前，流分类 [ADD MQCCLASSIFIER](0-00201.md) 与流行为 [ADD MQCBEHAVIOR](0-00199.md) 必须已存在——CLASSIFIERNAME/BEHAVIORNAME 为 reference 引用，引用对象不存在则绑定失败。
