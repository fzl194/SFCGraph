# 配置地址池冲突IPv6前缀（ADD CONFLICTIPV6）

> 命令静态知识见 [ADD CONFLICTIPV6](command/UDG/20.15.2/ADD-CONFLICTIPV6.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。POOLNAME 引用 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL 已建地址池；V6PREFIXLENGTH 必须与对应地址池地址段前缀长度一致才能生效。典型前置链：建 POOL(0-00043, IPVERSION=IPV6) → 配 IPv6 地址段(0-00044, V6PREFIXLENGTH) → 绑地址池组(0-00045) → 标记冲突 IPv6 前缀(0-00302)。

## 配置方法（配置方法字典）

ADD CONFLICTIPV6 是地址池族链路的"IPv6 冲突前缀标记层"：在已建的本地地址池(POOLNAME)中标识指定 IPv6 前缀为冲突状态，禁止该前缀内地址被分配给终端用户。命令核心配置空间有 4 个维度：**地址池引用**（POOLNAME 必选，引用已建 POOL）、**冲突前缀来源**（V6PREFIX 必选，全网规划）、**前缀长度**（V6PREFIXLENGTH 可选，默认 64）、**配额与约束**（每池最多 16 条冲突前缀，**同一地址池下前缀不允许重叠**，**冲突前缀长度必须与对应地址池地址段前缀长度一致才生效**）。1 个特性命中（GWFD-020406 IPv6 Prefix Delegation）覆盖典型范式。

### 配置维度 1：地址池引用（参数 POOLNAME，必选）

POOLNAME 决定该冲突 IPv6 前缀隶属于哪个已建地址池：

| 取值 | 作用 | 联动必选参数 | 前置依赖 |
|---|---|---|---|
| **已建 POOLNAME**（IPVERSION=IPV6 池） | 字符串 1~79 字节，由 `_`、`-`、数字、大小写字母和 `.` 组成，不以 `.` 开头且无连续 `..` | V6PREFIX | 必须先 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL 建 IPV6 池；POOLTYPE=LOCAL 时必须先 [0-00044](task/UDG/20.15.2/0-00044.md) ADD SECTION 配 IPv6 地址段 |

> **POOLNAME 命名约束**：1~79 字节，字符集 `_`、`-`、数字、大小写字母、`.`，不能以 `.` 开头且不能出现连续两个 `.`，不区分大小写。

### 配置维度 2：冲突 IPv6 前缀（参数 V6PREFIX，必选）

V6PREFIX 决定被标记为冲突状态的 IPv6 地址前缀：

| 取值 | 作用 | 数据来源 |
|---|---|---|
| **IPv6 前缀**（如 `fc00:0000:0000:fcee:0000:0000:0000:0001`） | 标记该前缀为冲突状态，禁止前缀内地址被分配给终端用户 | 全网规划（待标记的冲突前缀） |

> **V6PREFIX 取值约束**：冒号分十六进制 `X:X:X:X:X:X:X:X`；除组播地址、链路本地地址、环回地址或未指定的地址为非法地址外，其他都为合法地址。

### 配置维度 3：IPv6 前缀长度（参数 V6PREFIXLENGTH，可选，默认 64）

V6PREFIXLENGTH 决定该冲突前缀的掩码长度：

| 取值范围 | 默认值 | 作用 | 生效约束 |
|---|---|---|---|
| **整数 49~64** | 64 | 该冲突前缀的前缀长度 | **冲突前缀长度必须与对应地址池下地址段前缀长度相同时，冲突前缀才生效** |

> **典型范式**：020406 IPv6 Prefix Delegation 范式 V6PREFIXLENGTH=63（与对应 ADD SECTION 的 V6PREFIXLENGTH=63 一致）。

### 配置维度 4：配额与约束（命令 notes 投影）

| 约束维度 | 上限/规则 | 说明 |
|---|---|---|
| **每池冲突地址数** | 最多 **16** 条/POOL | 超过时命令失败 |
| **全局总记录数** | 最多 **10000** 条 | 命令 notes 硬规格 |
| **同池前缀不可重叠** | 同一地址池下添加的冲突 prefix 不允许重叠 | 重叠时命令失败 |
| **前缀长度匹配** | 冲突地址的前缀长度与对应地址池地址段前缀长度相同时才生效 | 不匹配则配置不生效 |

**典型脚本（020406 IPv6 Prefix Delegation，标记 IPv6 冲突前缀）**：
```
ADD CONFLICTIPV6:POOLNAME="testpool3",V6PREFIX="fc00:0000:0000:fcee:0000:0000:0000:0001", V6PREFIXLENGTH=63;
```

> 命令 notes：执行后只对新激活用户生效；最大记录数 10000；配置前需先 ADD POOL 建 IPv6 池；冲突地址池可用地址数减少；地址已分配给用户时命令仍可执行成功但下次分配时再生效；冲突前缀长度需与地址段前缀长度一致才生效；同一地址池下冲突 prefix 不允许重叠。

**步骤位置**：在所属 feature 编排中通常位于"步骤 5：IPv6 冲突前缀标记"——前置依赖 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL（建 IPV6 池，POOLNAME）+ [0-00044](task/UDG/20.15.2/0-00044.md) ADD SECTION（配 IPv6 地址段，V6PREFIXLENGTH 必须与本命令冲突前缀长度一致）；上游被 [0-00045](task/UDG/20.15.2/0-00045.md) ADD POOLBINDGROUP（绑地址池组）穿透引用。

## 决策点：IPv6 冲突前缀长度选择（DP 0-00302）

问：该冲突 IPv6 前缀用哪个前缀长度？

| 选项 | V6PREFIXLENGTH | 联动约束 | 适用场景 |
|---|---|---|---|
| Prefix Delegation 63 位前缀 | 63 | 对应地址池地址段 V6PREFIXLENGTH=63 | 020406 IPv6 Prefix Delegation 范式（testpool3/testpool30/testpool） |
| 默认 64 位前缀 | 64 | 对应地址池地址段 V6PREFIXLENGTH=64 | 默认场景 |

> 1 个特性命中（020406）以 63 位前缀为典型范式（需与对应 SECTION 一致）。

## 约束

- **POOLNAME 必须先建且为 IPV6 池**（critical，rule-0-00302）：POOLNAME 必须先由 ADD POOL 命令建好且 IPVERSION=IPV6 — 后果：未建池或池非 IPV6 命令拒收
- **V6PREFIXLENGTH 必须与地址段前缀长度一致**（critical，rule-0-00302）：冲突地址的前缀长度与对应地址池地址段前缀长度相同时才生效 — 后果：长度不一致配置不生效
- **同一地址池冲突前缀不可重叠**（critical，rule-0-00302）：同一地址池下添加的冲突 prefix 不允许重叠 — 后果：重叠命令失败
- **V6PREFIXLENGTH 取值范围 49~64**（critical，rule-0-00302）：前缀长度必须在 49~64 之间 — 后果：超范围命令拒收
- **每池最多 16 条冲突地址**（critical，rule-0-00302）：一个地址池最多可设置 16 个地址为冲突状态 — 后果：超规格新增失败
- **最大记录数 10000**（warning，rule-0-00302）：该命令全局最大记录数为 10000 — 后果：超规格新增失败
- **执行后只对新激活用户生效**（warning，rule-0-00302）：命令执行后只对新激活用户生效；地址已分配给用户则在下次分配时再生效 — 后果：误以为立即生效致前缀内地址仍被分配
- **适用 NF 限定 UPF**（warning，rule-0-00302）：UPF（PGW-U 不支持 IPv6 冲突）— 后果：PGW-U 等不适用
- **POOLNAME 命名约束**（warning，rule-0-00302）：1~79 字节，字符集 `_`、`-`、数字、大小写字母、`.`，不以 `.` 开头且无连续 `..` — 后果：不符合命名规范命令拒收
- **冲突前缀池可用地址数减少**（info，rule-0-00302）：配置为冲突状态的前缀内地址不会分配给用户，该地址池可用地址数减少 — 后果：池可用地址减少
- **V6PREFIX 取值约束**（warning，rule-0-00302）：冒号分十六进制 `X:X:X:X:X:X:X:X`；除组播/链路本地/环回/未指定地址外合法 — 后果：非法地址命令拒收
