# PCC基本功能（GWFD-020351）

> 特性静态知识见 [GWFD-020351](feature/UDG/20.15.2/GWFD-020351.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性是 **PCC backbone 基准**（POLICYTYPE=PCC 标准范例），计费族/带宽族/URL过滤/重定向族的 PCC 规则均复用此骨架。

## 配置概览

PCC 基本功能在用户业务流程中实现策略与计费控制（PCRF/PCF 决策，PGW-C/SMF 下发，PGW-U/UPF 执行业务流检测+策略执行）。PCC 规则分两种下发方式：

- **动态规则**（网络中有 PCRF/PCF，配置动态PCC功能_74096530）：PCRF/PCF 下发该规则对应的业务流特征+动作，SMF/UPF 不需特殊配置，PCRF/PCF 可随时增删规则。跨网元一致性的关键是 **RULENAME**（规则标识，须与 SMF 上配置的规则名称相同）。
- **预定义规则**（无 PCRF/PCF，或 PCRF/PCF 下发规则标识激活本地规则）：UPF/SMF/PCF 上需定义**相同的规则标识**——ADD RULE 的 RULENAME（预定义规则）或 ADD USERPROFILE 的 USERPROFILENAME（预定义规则组）。UPF 配业务流特征+动作，SMF/PCF 下发该标识激活 UPF 本地静态规则。本地PCC（配置本地PCC功能_74096529）即此方式，跨网元一致性的关键是 **USERPROFILENAME**（规则组标识，由 SMF 下发）。

核心对象链（两方式共用）：`PCCPOLICYGRP`（绑 URRGROUP 计费属性，计费时配 URRGROUPNAME）→ 过滤链（`FILTER`+`FLOWFILTER`+`FLTBINDFLOWF`）→ `RULE`（POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, POLICYNAME=PCCPOLICYGRPNM）→ `USERPROFILE` + `RULEBINDING`。两份 activation 均演示**三四层 any to any 全匹配 filter**（L34PROTOCOL=ANY，所有报文都匹配，未配置七层规则）。

## 配置流程（步骤编排，每步显示含命令）

1. **License 前置**（单命令）：`SET LICENSESWITCH`（`LICITEM=LKV3G5PCCB01,SWITCH=ENABLE`）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **PCC 策略组**（单命令，计费时绑 URRGROUPNAME）：`ADD PCCPOLICYGRP`（`PCCPOLICYGRPNM` + `URRGROUPNAME`，URRGROUPNAME 来自计费三件套）→ 复用 [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套的 PCCPOLICYGRP 步；不计费时直引 atom [0-00003](task/UDG/20.15.2/0-00003.md)（URRGROUPNAME 可省）
3. **过滤链**（三四层 any to any 全匹配）：`ADD FILTER`（`FILTERNAME + L34PROTTYPE=STRING + L34PROTOCOL=ANY`）+ `ADD FLOWFILTER`（`FLOWFILTERNAME`）+ `ADD FLTBINDFLOWF`（`FLOWFILTERNAME + FILTERNAME`）→ [1-00009](task/UDG/20.15.2/1-00009.md)
4. **业务规则**（POLICYTYPE=PCC）：`ADD RULE`（`RULENAME + POLICYTYPE=PCC + FILTERINGMODE=FLOWFILTER + FLOWFILTERNAME + PRIORITY + POLICYNAME=PCCPOLICYGRPNM`）→ [1-00011](task/UDG/20.15.2/1-00011.md)
5. **用户模板 + 绑定**：`ADD USERPROFILE`（`USERPROFILENAME`）+ `ADD RULEBINDING`（`USERPROFILENAME + RULENAME`）→ [1-00011](task/UDG/20.15.2/1-00011.md)
6. **刷新生效**（单命令）：`SET REFRESHSRV`（`REFRESHTYPE=ALL`）→ [0-00015](task/UDG/20.15.2/0-00015.md)

> **参数证据链**（R1.5，配置流程脚注）：以上命令+参数均来自两份 activation §数据规划表 + §任务示例脚本（两份脚本原样相同）。

> - ADD FLOWFILTER（flowfilter_test）：两份 activation §数据规划表 ADD FLOWFILTER 段 + §任务示例脚本（原样）

## 决策点

**DP（特性级）：规则下发方式 + 过滤匹配层 + 是否计费**（影响 RULE/USERPROFILE 一致性语义 + 过滤链走法 + PCCPOLICYGRP 是否绑 URRGROUP）。维度按两份 activation §操作场景 + §数据规划表全列。

| 场景 | 规则下发方式 | 跨网元一致性关键 | 过滤匹配 | 是否计费 | 走法 |
|---|---|---|---|---|---|
| **动态PCC 标准部署（activation 演示）** | 动态规则（PCRF/PCF 下发业务流特征+动作）| RULENAME（须与 SMF 上规则名称相同，PCRF/PCF 可随时增删）| 三四层 any to any（L34PROTOCOL=ANY）| 计费（PCCPOLICYGRP 绑 URRGROUPNAME=urrgp_test）| 骨架 + [1-00010](task/UDG/20.15.2/1-00010.md)（计费三件套）+ [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链）+ [1-00011](task/UDG/20.15.2/1-00011.md)（规则绑定）|
| 本地PCC 变体（无 PCRF/PCF）| 预定义规则组（SMF 下发 USERPROFILENAME 激活本地规则，SMF 不支持实时更新）| **USERPROFILENAME**（须与 SMF 上用户模板名称相同，规则组标识）| 三四层 any to any | 计费（同上）| 同上（activation 脚本一致），语义在 USERPROFILENAME |
| 预定义规则变体（PCRF/PCF 下发 RULENAME 激活）| 预定义规则（PCF 下发规则标识，激活本地静态规则）| RULENAME（须 UPF/SMF/PCF 定义相同）| 三四层 any to any | 计费（同上）| 同上 |
| 七层精细匹配变体 | 动态/预定义任一 | 同对应方式 | 七层 L7FILTER+PROTBINDFLOWF（具体规则内容见业务感知配置）| 计费/纯策略均可 | [1-00009](task/UDG/20.15.2/1-00009.md) 加 L7 段（L7FILTER+PROTBINDFLOWF）|
| 仅策略不计费变体 | 动态/预定义任一 | 同对应方式 | 三四层 any | **不计费**（PCCPOLICYGRP 不配 URRGROUPNAME）| 直引 atom [0-00003](task/UDG/20.15.2/0-00003.md)（省 [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套）|

**联动**：
- 规则下发方式 → [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定：动态规则的跨网元一致点是 RULENAME；本地PCC 预定义规则组的一致点是 USERPROFILENAME；预定义规则的一致点是 RULENAME（PCF 下发规则标识）。无论哪种方式，ADD RULE/ADD USERPROFILE 的命名都须"与对端协商"（两份 activation §数据规划表的"获取方法=与对端协商"）。
- 过滤匹配 → [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链：activation 演示三四层 any to any 全匹配（所有报文都匹配）；七层精细匹配的具体规则内容（L7FILTER+PROTBINDFLOWF）见 [业务感知配置](evidence/UDG/20.15.2/业务感知配置_90565483.md)，activation 未演示（零证据差异，配置生成按三四层 any 产出，七层待补业务感知配置）。
- 是否计费 → [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套：计费时 PCCPOLICYGRP 绑 URRGROUPNAME（走计费三件套）；纯策略不计费时 URRGROUPNAME 可省（直引 atom [0-00003](task/UDG/20.15.2/0-00003.md)）。

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5PCCB01` — 未开则 PCC 功能不生效。
- **RULENAME/USERPROFILENAME 与 SMF/PCF 一致**（critical）：动态规则的 RULENAME 须与 SMF 上规则名称相同；本地PCC 的 USERPROFILENAME 须与 SMF 上用户模板名称相同；预定义规则的 RULENAME 须 UPF/SMF/PCF 定义相同 — 不一致则跨网元会话匹配失败，规则无法激活。
- **FLOWFILTER 至少绑一个 filter/l7filter**（critical）：ADD FLOWFILTER 须至少绑定一个 ADD FILTER 或 ADD L7FILTER — 否则所有报文都匹配不上该流过滤器（两份 activation §操作步骤3 明示）。
- **PCCPOLICYGRP 计费时绑 URRGROUP**（warning）：需要计费时 PCCPOLICYGRP.URRGROUPNAME 须指向已配的 URRGROUP（来自计费三件套 [1-00010](task/UDG/20.15.2/1-00010.md)）— 漏配则计费不生效；纯策略控制可省 URRGROUPNAME（两份 activation 脚本注释明示）。
- **依赖业务感知配置 / SA-Basic**（info）：具体规则内容（精细业务流特征+动作）需根据业务策略确定，配置方法见 [业务感知配置](evidence/UDG/20.15.2/业务感知配置_90565483.md)；HTTP 等协议解析依赖 SA-Basic（[GWFD-110101](feature/UDG/20.15.2/GWFD-110101.md)）。
- **PRIORITY 仅对 PCC 用户生效**（info）：RULE.PRIORITY（值越小优先级越高，activation 演示 65535）— 非 PCC 用户不受此规则约束。
- **性能影响**（info）：使用 PCC 功能系统性能将下降（用户激活需与 PCRF/PCF 交互，报文处理增加时延），下降程度需基于流量模型评估（见特性 wiki §对系统的影响）。
