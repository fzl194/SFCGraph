# 配置SA触发专有承载（ADD SADEDICBEARER）

> 命令静态知识见 [ADD SADEDICBEARER](command/UDG/20.15.2/ADD-SADEDICBEARER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD SADEDICBEARER 是 PCC 业务属性链的"SA 专有承载触发模式定义层"：把"哪一协议/协议组、按什么方向触发专有承载创建"用一条记录描述，与 [0-00038](task/UDG/20.15.2/0-00038.md)（ADD QOSPROP）的 DECOUPLINGSW 共同决定七层业务触发的承载创建行为。命令核心配置空间有 2 个维度：**协议等级**（PROTOCOLLEVEL 二选一：PROTOCOL / PROTOCOLGROUP）、**触发模式**（TRIGGERMODE 决定上/下行/双向/不触发）。仅 1 个特性命中（020358 七层 QoS 范式，演示 http 协议 + DOWNLINK_ONLY）。

### 配置维度 1：协议等级（参数 PROTOCOLLEVEL，必选）

| 取值 | 作用 | 必选联动 |
|---|---|---|
| **PROTOCOL** | 单协议维度 | PROTOCOLNAME |
| **PROTOCOLGROUP** | 协议组维度 | PROTGROUPNAME |

### 配置维度 2：协议/协议组引用（参数 PROTOCOLNAME/PROTGROUPNAME，条件必选）

| 参数 | 条件 | 作用 |
|---|---|---|
| **PROTOCOLNAME** | PROTOCOLLEVEL=PROTOCOL 时必选 | 引用 ADD PROTOCOL（★R1.5 atom 待补） 配置的协议名 |
| **PROTOCOLNAME/PROTGROUPNAME** | PROTOCOLLEVEL=PROTOCOLGROUP 时必选 | 引用 ADD PROTOCOLGROUP（★R1.5 atom 待补） 配置的协议组名 |

### 配置维度 3：触发模式（参数 TRIGGERMODE，必选）

| 取值 | 作用 |
|---|---|
| **DOWNLINK_ONLY** | 仅下行触发专有承载创建（020358 范式） |
| **UPLINK_ONLY** | 仅上行触发 |
| **BOTH** | 上下行均触发 |
| **NONE** | 不触发 |

**典型脚本（020358 业务触发的 QoS 保证，PROTOCOL + DOWNLINK_ONLY）**：
```
ADD SADEDICBEARER:PROTOCOLLEVEL=PROTOCOL,PROTOCOLNAME="http",TRIGGERMODE=DOWNLINK_ONLY;
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；与 [0-00038](task/UDG/20.15.2/0-00038.md)（ADD QOSPROP）的 DECOUPLINGSW 联动：DECOUPLINGSW=ENABLE 时 QOSPROP 的下行链路模式优先级高于本命令配置。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：SA 触发模式定义"——前置 ADD PROTOCOL（★R1.5 atom 待补）（PROTOCOLNAME 来源）；产出 (PROTOCOLLEVEL, PROTOCOLNAME/PROTGROUPNAME, TRIGGERMODE) 三元组供 SA 评估使用。

## 决策点

（无。本命令 PROTOCOLLEVEL 二选一 + TRIGGERMODE 四选一，均为参数级条件必选；无独立特性级 DP。）

## 约束

- **PROTOCOLLEVEL 决定协议引用**（critical，rule-0-00340）：PROTOCOL 时 PROTOCOLNAME 必选；PROTOCOLGROUP 时 PROTGROUPNAME 必选 — 后果：缺协议名命令拒收
- **协议/协议组引用前置依赖**（critical，rule-0-00340）：PROTOCOLNAME 须先建（ADD PROTOCOL（★R1.5 atom 待补））；PROTGROUPNAME 须先建（ADD PROTOCOLGROUP（★R1.5 atom 待补）） — 后果：未建引用失败
- **DECOUPLINGSW 优先级冲突**（critical，rule-0-00340）：[0-00038](task/UDG/20.15.2/0-00038.md) 的 DECOUPLINGSW=ENABLE 时本命令的下行配置被覆盖 — 后果：解耦/承载配置冲突时以 QOSPROP 为准
- **同协议多模式覆盖**（warning，隐含）：同 (PROTOCOLLEVEL, PROTOCOLNAME/PROTGROUPNAME) 重复 ADD 会以最后一条为准 — 后果：误覆盖
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
