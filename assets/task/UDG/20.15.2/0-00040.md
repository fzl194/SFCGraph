# 配置SA触发专有承载（ADD SADEDICBEARER）
> 命令静态知识见 [ADD SADEDICBEARER](command/UDG/20.15.2/ADD-SADEDICBEARER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

为某协议或协议组使能 SA（业务感知）能力触发专有承载创建，并配置触发模式——把指定协议的所有用户报文汇聚到一个特定专有承载（默认仅下行汇聚）。仅 L7 业务触发 QoS 场景需要，位于 ADD WELLKNOWNPORT 之后。

| 参数 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| PROTOCOLLEVEL（协议等级） | PROTOCOL / PROTOCOLGROUP | 本端规划 | 二选一，决定下一行配哪个 |
| PROTOCOLNAME（协议名） | http | 本端规划 | PROTOCOLLEVEL=PROTOCOL 时必选，须 SubProtocol 表存在 |
| PROTGROUPNAME（协议组名） | （PROTOCOLGROUP 时填） | 本端规划 | PROTOCOLLEVEL=PROTOCOLGROUP 时必选 |
| TRIGGERMODE（触发专有承载模式） | NOT_TRIGGER / FLOWNODE_BASED / DOWNLINK_ONLY | 本端规划 | 默认 NOT_TRIGGER；七层 QoS 下行汇聚用 DOWNLINK_ONLY |

典型脚本样例（七层 QoS 下行汇聚，PROTOCOL 路径）：
```
ADD SADEDICBEARER:PROTOCOLLEVEL=PROTOCOL,PROTOCOLNAME="http",TRIGGERMODE=DOWNLINK_ONLY;
```

配置要点：PROTOCOLLEVEL 决定走"协议级"还是"协议组级"，二选一互斥——详见决策点。TRIGGERMODE=DOWNLINK_ONLY 时 UDG 创建 rule-filter-type=l7-downlink 模板，关联一个特定专有承载，所有用户下行报文汇聚到该承载，上行仍走原专有承载。本 atom 仅 1 特性（GWFD-020358）使用，默认走 PROTOCOL 路径配具体协议名。

## 决策点

PROTOCOLLEVEL 二选一互斥（[[UDG@20.15.2@DecisionPoint@0-00102]]）。

| 选项 | PROTOCOLLEVEL | 必选联动参数 | 不配参数 |
|---|---|---|---|
| 协议级（默认） | PROTOCOL | PROTOCOLNAME | PROTGROUPNAME |
| 协议组级 | PROTOCOLGROUP | PROTGROUPNAME | PROTOCOLNAME |

## 约束
- 最大记录数 100（PROTOCOLLEVEL+协议/协议组对总数）/ 立即生效 · warning · 超规格新增失败
- PROTOCOLLEVEL/PROTOCOLNAME/PROTGROUPNAME 互斥错配 · warning · 命令拒收（见 [[UDG@20.15.2@TaskRule@0-00281]]）
- TRIGGERMODE 默认 NOT_TRIGGER；非触发值需 SA-Basic 场景特性启用 · warning · 误配触发无效
- 适用 NF：PGW-U、UPF
