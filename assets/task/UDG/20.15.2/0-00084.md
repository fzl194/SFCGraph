# 设置DDoS防攻击流控阈值（SET DDOS）

> 命令静态知识见 [SET DDOS](command/UDG/20.15.2/SET-DDOS.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET DDOS 是 DDoS 攻击防护的"全局流量门限"命令：通过限制单位时间（固定 1 秒）内允许通过的 TCP SYN 报文包数，丢弃超过阈值的部分，实现对用户上行 TCP SYN 报文（建链请求）的流控。本命令的核心配置空间是单一维度的防护阈值 THRESHOLD，整机最多记录数 **1**（即全局唯一），**立即生效**。

### 配置维度 1：防护阈值（参数 THRESHOLD，必选）

| 取值 | 防护强度 | 适用 NF | 备注 |
|---|---|---|---|
| **1~10000**（整数） | 越高放通越多（防护越弱），越低拦截越严（防护越强） | SGW-U、PGW-U、UPF | 单位：包/秒；默认初始值 100；典型场景 10（高防护）、150（中等）、5000（高放通） |
| **100（缺省）** | 系统初始默认值 | 同上 | 恢复系统缺省时配置 |

> 配置原则：THRESHOLD 配过小会影响用户新建流速率，建议使用默认值；典型场景见 [修改DDoS流控阈值_24082265](evidence/UDG/20.15.2/修改DDoS流控阈值_24082265.md)（10/150 两档）。

**步骤位置**：在所属 feature 编排中通常为"步骤 1：设置 DDoS 阈值"——整机仅 1 条记录，独立于其它特性配置；生效还需配合 ADD USERPROFILE.DDOSCHECK=ENABLE（[0-00011](task/UDG/20.15.2/0-00011.md)）。

## 约束

- **整机最大记录数为 1**（warning，隐含）：单板仅 1 条 SET DDOS 记录；后续 SET 覆盖前值；存在系统初始记录 — 后果：重复执行仅覆盖初始值；不存在多版本并存
- **立即生效**（info，隐含）：该命令执行后立即生效，无需重启 — 后果：阈值变更实时影响所有 TCP SYN 流控
- **THRESHOLD 取值范围 1~10000**（warning，隐含）：整数类型；过小影响用户新建流速率 — 后果：业务感知的新建连接受限
- **THRESHOLD 单位固定为包/秒**（info，隐含）：时间窗口固定 1 秒 — 后果：与"每分钟/每小时"类阈值口径不同
- **DDOSCHECK 需配套使能**（critical，隐含）：本命令设阈值为全局门限，须在 ADD USERPROFILE 上 DDOSCHECK=ENABLE 才对该 UserProfile 生效 — 后果：仅设阈值但 DDOSCHECK=DISABLE 则防护不生效
