# 设置DDoS防攻击流控阈值（SET DDOS）

> 命令静态知识见 [SET DDOS](command/UDG/20.15.2/SET-DDOS.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

DDoS 阈值是整机级全局开关，单参数 `THRESHOLD`，本端规划取值。语义=每个用户每秒允许转发的 TCP 建链请求包个数；超阈值时 UDG 丢弃报文。命令全局唯一（仅 1 条记录），系统初始值 100。

**取值样例（来自激活/修改任务数据规划表）**

| 场景 | THRESHOLD | 说明 |
|---|---|---|
| 防DDoS功能激活 | 10 | 允许发起最大TCP连接数 |
| 修改阈值 | 150 | 每用户每秒允许转发的TCP建链请求数 |

取值范围整数 1~10000；阈值过小会影响用户新建流速率，建议沿用默认值 100。

**步骤**：配置阈值后，还需在 ADD USERPROFILE 上将 `DDOSCHECK=ENABLE` 才对具体 UserProfile 生效——阈值是全局门限，DDOSCHECK 决定防护作用在哪些用户模板上。

**脚本**

```
SET DDOS:THRESHOLD=10;
```

## 约束

- 全局唯一实例（最大记录数 1），重复执行覆盖前值；立即生效；系统初始记录 THRESHOLD=100。
- 阈值过小影响用户新建流速率，建议默认值；DDOSCHECK 未启用则阈值不生效。
