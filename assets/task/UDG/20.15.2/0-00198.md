# 增加高级ACL规则(VNRS IPv4)（ADD ACLRULEADV4）

> 命令静态知识见 [ADD ACLRULEADV4](command/UDG/20.15.2/ADD-ACLRULEADV4.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令创建高级 IPv4 ACL 规则，基于**五元组**（协议/源IP/源反掩码/目的IP/目的反掩码）匹配报文，TCP/UDP 时扩展端口段。ACLNAME 引用 [ADD ACLGROUP](task/UDG/20.15.2/0-00197.md)（高级范围段 3000-3999）。源/目的 IP 及反掩码取自全网规划（需与对端主机 IP 一致）。

**配置范式（按协议类型分支，2 特性归纳）**

| 协议 ACLPROTOCOL | 必配参数 | 示例行（IPFD-012002） |
|---|---|---|
| IP(0) | 五元组 | `ADD ACLRULEADV4:ACLNAME="3001", ACLRULENAME="adv_2", ACLACTION=Permit, ACLPROTOCOL=0, ACLSOURCEIP="10.1.1.2", ACLSRCWILD="0.0.0.0", ACLDESTIP="192.168.1.1", ACLDESTWILD="0.0.0.0";` |
| TCP(6) / UDP(17) | 五元组 + 端口段（ACLSRCPORTOP/BEGIN[/END] + ACLDESTPORTOP/BEGIN[/END]） | 见命令 wiki 端口段模板 |
| ICMP(1) | 五元组 + ACLICMPNAME[/TYPE/CODE] | 见命令 wiki ICMP 模板 |

**参数取值约定**

| 参数 | 取值 | 来源 | 说明 |
|---|---|---|---|
| ACLNAME | 高级范围段 3000-3999 | 引用 [ADD ACLGROUP](task/UDG/20.15.2/0-00197.md) | 规则组标识 |
| ACLRULENAME | 字符串 1~32 | 本端规划 | 规则名称 |
| ACLACTION | Permit / Deny | 本端规划 | 规则行为 |
| ACLPROTOCOL | 整数 0~255（IP=0, ICMP=1, TCP=6, UDP=17） | 本端规划 | 协议号，决定后续参数组合 |
| ACLSOURCEIP / ACLDESTIP | IPv4 | 全网规划 | 源/目的 IP |
| ACLSRCWILD / ACLDESTWILD | IPv4 反掩码 | 全网规划 | 源/目的反掩码 |
| ACLSRCPORTOP / ACLDESTPORTOP | 枚举（Lt/Gt/Eq/Neq/Range） | 本端规划 | **TCP/UDP 时必填**（rule 0-00270） |
| ACLSRCPORTBEGIN / ACLDESTPORTBEGIN | 整数 0~65535 | 本端规划 | **TCP/UDP 时必填**起始端口 |
| ACLSRCPORTEND / ACLDESTPORTEND | 整数 0~65535 | 本端规划 | PORTOP=Lt/Range 时必填结束端口 |

**未绑冷门参数**（命令真相有但特性脚本未用）：ACLRULEID / ACLFRAGTYPE / VRFNAME / ACLRULEDESCRIPTION / ACLPRECEDENCE / ACLTOS / ACLDSCP / ACLICMPNAME/TYPE/CODE / ACLSYNFLAG / ACLTTLOP/TTL/TTLEND——按需（ICMP/TTL/分片等垂直场景）时查命令 wiki。

## 约束

- **rule-0-00270**（error）ACLPROTOCOL=TCP(6)/UDP(17) 时端口段参数（ACLSRCPORTOP/ACLDESTPORTOP + ACLSRCPORTBEGIN/ACLDESTPORTBEGIN）必填；PORTOP=Lt/Range 时 ACLSRCPORTEND/ACLDESTPORTEND 必填 — 后果：端口匹配参数缺失→规则无法精确匹配
- 最大记录数 65535；单组最多 16384 条规则；前置必须先 ADD ACLGROUP — 后果：超限/无组→新增失败
