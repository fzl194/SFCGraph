# 增加高级ACL规则（ADD ACLRULEADV4）

> 命令静态知识见 [ADD ACLRULEADV4](command/UDG/20.15.2/ADD-ACLRULEADV4.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。高级 ACL 基于五元组（源目的 IP+反掩码+协议号）+ TCP/UDP 时扩展端口号（TCP/UDP 端口号才有意义）；ACLNAME 引用 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP 高级组（3000~3999）或命名组（"acl1"）。

## 配置方法（配置方法字典）

ADD ACLRULEADV4 是"增加高级 ACL 规则"命令。3 参必选（ACLNAME/ACLRULENAME/ACLPROTOCOL/ACLACTION）+ 22 参可选。配置空间有 3 个维度：**协议分叉**（ACLPROTOCOL 决定 IP/TCP/UDP/ICMP 后续参数组合）、**规则行为**（ACLACTION Permit/Deny）、**端口范围类型**（仅 TCP/UDP 涉及 PORTOP 决定起止端口参数）。1 个特性命中（IPFD-012002 ACL）覆盖 IP 协议（0=IP）两种范例。

### 配置维度 1：协议分叉（参数 ACLPROTOCOL，必选，整数 0~255）

| 协议号 | 协议名 | 联动必选参数 | 典型场景 |
|---|---|---|---|
| **0** | IP | 五元组（ACLSOURCEIP/SRCWILD + ACLDESTIP/DESTWILD） | IPFD-012002 范例 1+2 范式（IP 层匹配） |
| **1** | ICMP | 五元组 + ACLICMPNAME [/TYPE /CODE 当 ACLICMPNAME=Custom 时] | ICMP 报文类型匹配 |
| **6** | TCP | 五元组 + ACLSRCPORT 系列 + ACLDESTPORT 系列 | TCP 端口段匹配（HTTP/HTTPS/SSH 等） |
| **17** | UDP | 五元组 + ACLSRCPORT 系列 + ACLDESTPORT 系列 | UDP 端口段匹配（DNS/SIP 等） |

> **IPFD-012002 范例**：`ACLPROTOCOL=0` 五元组即可（IP 协议级匹配）。TCP/UDP 时端口参数必配。

### 配置维度 2：规则行为（参数 ACLACTION，必选）

| 取值 | 作用 |
|---|---|
| **Permit** | 允许匹配报文通过 |
| **Deny** | 拒绝匹配报文 |

> IPFD-012002 范例 1= Permit / 范例 2= Deny，两者形成"先允许后拒绝"的策略组合。

### 配置维度 3：端口范围类型（参数 ACLSRCPORTOP/ACLDESTPORTOP，仅 TCP/UDP 时可选）

| 取值 | 联动必选端口参数 | 适用场景 |
|---|---|---|
| **Eq** | ACL[SRCDEST]PORTBEGIN | 单端口匹配（如 HTTP 80） |
| **Gt** | ACL[SRCDEST]PORTBEGIN | 大于起始端口 |
| **Lt** | ACL[SRCDEST]PORTBEGIN + ACL[SRCDEST]PORTEND | 小于结束端口 |
| **Neq** | ACL[SRCDEST]PORTBEGIN | 不等于某端口 |
| **Range** | ACL[SRCDEST]PORTBEGIN + ACL[SRCDEST]PORTEND | 端口区间匹配 |

> **铁律**（rule-0-00270）：TCP/UDP 时端口段参数必填；不配 PORTOP 即不校验端口。

**典型脚本（IPFD-012002 高级组 3001，IP 协议 Permit）**：
```
ADD ACLRULEADV4:ACLNAME="3001", ACLRULENAME="adv_2", ACLACTION=Permit, ACLPROTOCOL=0, ACLSOURCEIP="10.1.1.2", ACLSRCWILD="0.0.0.0", ACLDESTIP="192.168.1.1", ACLDESTWILD="0.0.0.0";
```

**典型脚本（IPFD-012002 高级组 3001，IP 协议 Deny）**：
```
ADD ACLRULEADV4:ACLNAME="3001", ACLRULENAME="adv_3", ACLACTION=Deny, ACLPROTOCOL=0, ACLSOURCEIP="10.1.2.2", ACLSRCWILD="0.0.0.255", ACLDESTIP="192.168.1.1", ACLDESTWILD="0.0.0.0";
```

**典型脚本（IPFD-012002 命名组 acl1，IP 协议 Deny+Permit 组合）**：
```
ADD ACLRULEADV4: ACLNAME="acl1", ACLRULENAME="adv_3", ACLACTION=Deny, ACLPROTOCOL=0, ACLSOURCEIP="192.168.1.3", ACLSRCWILD="0.0.0.0", ACLDESTIP="192.168.2.1", ACLDESTWILD="0.0.0.0";
ADD ACLRULEADV4: ACLNAME="acl1", ACLRULENAME="adv_4", ACLACTION=Permit, ACLPROTOCOL=0, ACLSOURCEIP="192.168.1.4", ACLSRCWILD="0.0.0.0", ACLDESTIP="192.168.2.1", ACLDESTWILD="0.0.0.0";
```

> 命令 notes：**最大记录数 65535**（整机 ACL 规则）；**单组最多 16384 条**；**执行后立即生效**；ACLFRAGTYPE（首片/非首片/非分片报文匹配，可选）；VRFNAME（VPN 实例限定，1~31）；ACLICMPNAME=Custom 时 ACLICMPTYPE + ACLICMPCODE 必选；ACLPROTOCOL=0 范例未配端口段。

**步骤位置**：在所属 feature 编排中位于"步骤 N：建 ACL 规则"——前置 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP（建组）；下游 [MOD ACLRULEADV4](command/UDG/20.15.2/MOD-ACLRULEADV4.md)（修改规则）；[RMV ACLRULEADV4](command/UDG/20.15.2/RMV-ACLRULEADV4.md)（删除）；[0-00202](task/UDG/20.15.2/0-00202.md) ADD QOSRULEACL（流分类下挂本规则 ACLNAME）。

## 决策点：协议类型与端口选择（DP 0-00198）

问：ADD ACLRULEADV4 针对哪种协议配置？（决定联动必选参数组合）

| 选项 | ACLPROTOCOL | ACLACTION | 端口段 | 适用场景 |
|---|---|---|---|---|
| **IP 层匹配** | 0 | Permit/Deny | 不配端口 | IPFD-012002 范例（高层报文分类） |
| **TCP 单端口** | 6 | Permit/Deny | PORTOP=Eq + BEGIN=N | 单 TCP 端口（如 HTTP 80） |
| **TCP 端口段** | 6 | Permit/Deny | PORTOP=Range + BEGIN/END | TCP 端口区间（如 1024-65535） |
| **UDP 单端口** | 17 | Permit/Deny | PORTOP=Eq + BEGIN=N | 单 UDP 端口（如 DNS 53） |
| **ICMP 类型匹配** | 1 | Permit/Deny | 不配端口；配 ACLICMPNAME | ICMP 报文（如 ping/不可达） |

> 1 个特性命中（IPFD-012002）覆盖 ACLPROTOCOL=0 范式；TCP/UDP/ICMP 按需参考命令 wiki 取端口/ICMP 详细参数。

## 约束

- **TCP/UDP 端口段必填**（critical，rule-0-00270）：ACLPROTOCOL=TCP(6)/UDP(17) 时端口段参数必填；PORTOP=Lt/Range 时 PORTEND 必填 — 后果：端口参数缺失规则无法精确匹配
- **最大记录数 65535**（critical，rule-0-00198）：整机 ACL 规则最大 65535 条 — 后果：超规格新增失败
- **单组最多 16384 条规则**（critical，rule-0-00198）：单个 ACL 规则组下最多 16384 条 — 后果：超限新增失败
- **前置必须先 ADD ACLGROUP**（critical，rule-0-00198）：ACLNAME 引用的规则组必须先建 — 后果：未建引用失败
- **执行后立即生效**（warning，rule-0-00198）：执行后立即生效 — 后果：误操作即刻生效
- **ACLICMPNAME=Custom 时 ICMPTYPE/COD 必填**（warning，rule-0-00198）：ICMP Custom 模式下必配 ACLICMPTYPE + ACLICMPCODE — 后果：不配参数命令拒收
- **ACLRULENAME 命名约束**（info，rule-0-00198）：字符串 1~32（不支持空格，区分大小写） — 后果：超长命令拒收
- **ACLNAME 范围段**（info，rule-0-00198）：数字 ACLNAME 需在 3000~3999 高级范围段；命名 ACLNAME 直接引用 [0-00197](task/UDG/20.15.2/0-00197.md) — 后果：跨段数字命令拒收
