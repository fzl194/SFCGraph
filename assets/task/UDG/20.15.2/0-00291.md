# 增加头增强（ADD HEADEN）

> 命令静态知识见 [ADD HEADEN](command/UDG/20.15.2/ADD-HEADEN.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`ADD HEADEN` 新建一个头增强对象（HEADERENNAME），通过 DATATYPE 指定要插入的用户信息字段类型，PREFIXNAME 设定 HTTP 头扩展字段前缀。**多字段插入需多次执行本命令**（同一 HEADERENNAME 下最多 40 种不同 DATATYPE）。**仅 PGW-U/UPF 适用**。

### 配置维度 1：头增强名（参数 HEADERENNAME）
| 取值 | 作用 | 约束 |
|---|---|---|
| 字符串 1~31 字符，无空格 | 头增强唯一标识 | 必选；不能配为 HTTP 标准头域名（如 host、accept），否则防欺诈失效 |

### 配置维度 2：数据类型（参数 DATATYPE，必选）
| 取值分组 | 代表取值 | 作用 |
|---|---|---|
| 用户标识类 | MSISDN1/2/3、IMSI1/2/3、IMEI1/2/3、MSIP1/2/3、USERPROFALIAS | 插入用户标识信息 |
| 网络信息类 | APN、SGSNIP1/2/3、GGSNIP1/2/3、UPIPV4、UPIPV6、ZONEID | 插入网络/接入信息 |
| 计费/会话类 | BILLINGTYPE、SESSIONID、CHGID、CHGCHAR1/2/3、MCC、MNC、ULI1/2/3 | 插入计费/会话属性 |
| 移动属性类 | RAT1/2/3、ROAMING1/2/3、SGSN_MCC_MNC1/2/3 | 插入移动属性 |
| 通用类 | USERDEF1~4（必配 USERDEFINEVAL）、TIMESTAMP1/2/3、SUBPROFILE1/2/3、RANDNUM | 自定义/时间戳/签约 |
| 拼接类 | MULTIPARA1/2/3 | 多参数拼接，PARADATATYPE1/2/3 至少 1 个非 NONE（仅 MSISDN/SGSNIP/MSIP） |

### 配置维度 3：前缀名称（参数 PREFIXNAME）
| 取值 | 作用 | 约束 |
|---|---|---|
| 字符串 1~31 字符 | HTTP 头扩展字段前缀 | 必选（除 MULTIPARA 与 USERDEF 的特殊值外）；同 HEADERENNAME 内不可重复 |

### 配置维度 4：加密算法（参数 ENCRYALGORI）
| 取值 | 作用 | 安全风险 |
|---|---|---|
| AES128_GCM（默认） / AES256_GCM / AES128_CTR / AES256_CTR | 对称加密（推荐安全） | 低 |
| AES128 / AES256（CBC） | 对称 CBC | 有安全风险，不建议 |
| SHA256 | 哈希 | 低；可配 SALTSW（ENABLE 默认） |
| MD5 | 哈希 | 有安全风险，不建议 |
| RC4 | 对称流 | 有安全风险，不建议 |
| RSA1024 / RSA2048 / RSA3072 | 非对称（需配 RSA*NAME 公钥） | RSA1024 不建议 |
| NONE | 不加密 | 有安全风险，不建议 |

### 配置维度 5：防欺诈（参数 ANTIFRAUD）
| 取值 | 作用 |
|---|---|
| DISABLE（默认） | 不使能防欺诈 |
| ENABLE | 插入前先检测/纠正已有错误字段；整机最多 32 个前缀开防欺诈，单报文 ≤16 个 |

### 配置维度 6：灰名单（参数 GRAYLIST）
| 取值 | 作用 |
|---|---|
| DISABLE（默认） | 关闭灰名单 |
| ENABLE | 只防欺诈不插入头增强字段（配合 ANTIFRAUD=ENABLE） |

### 配置维度 7：密文编码（参数 CIPHERTXTENCODE）
| 取值 | 作用 |
|---|---|
| INHERIT（默认） | SET BASE64 + 软参 BIT475 控（base64/ASCII） |
| NONE | 不对密文做特殊编码 |
| BASE64 | 命令内直接开 base64（可配等号/斜杠/加号替换字符） |
| ASCII | 命令内直接开 ASCII |

**典型脚本**：

```
// 4 字段不加密
ADD HEADEN:HEADERENNAME="header_test",DATATYPE=APN,PREFIXNAME="pre-apn",ENCRYALGORI=NONE;
ADD HEADEN:HEADERENNAME="header_test",DATATYPE=IMSI1,PREFIXNAME="pre-imsi",ENCRYALGORI=NONE;
// 防欺诈 + AES128 加密
ADD HEADEN:HEADERENNAME="headen1",DATATYPE=MSISDN1,PREFIXNAME="TEST",ENCRYALGORI=AES128,PSWDKEY="******",PSWDKEYCONFIRM="******",ANTIFRAUD=DISABLE;
```

**步骤位置**：承接 [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链）+ 协议识别（WELLKNOWNPORT/SIGNATUREDB/PARSERDB）；本命令产出 HEADERENNAME，供下游 [0-00010](task/UDG/20.15.2/0-00010.md) RULE 经 `POLICYTYPE=HEADEN, POLICYNAME=<HEADERENNAME>` 直接引用（**不经 PCCPOLICYGRP**）。

## 决策点：加密算法选择（DP 0-00291）

| 选项 | ENCRYALGORI 取值 | 配套参数 | 适用 |
|---|---|---|---|
| 推荐 GCM | AES128_GCM / AES256_GCM | PSWDKEY + PSWDKEYCONFIRM | 生产环境默认 |
| 推荐 CTR | AES128_CTR / AES256_CTR | PSWDKEY + PSWDKEYCONFIRM | 高吞吐场景 |
| CBC（不建议） | AES128 / AES256 | PSWDKEY + PSWDKEYCONFIRM | 老系统兼容 |
| 哈希 | SHA256 / MD5 | PSWDKEY + SALTSW（RANDNUM 作盐） | 不可逆场景 |
| 非对称 | RSA2048 / RSA3072 / RSA1024 | RSA*NAME（公钥）+ METHOD + PADDINGTYPE | 对端固定公钥 |

## 约束

- **rule-0-00291-1**（critical）：PREFIXNAME 不可为 HTTP 标准头域名（host/accept 等），否则防欺诈失效 — 后果：防欺诈无法识别/纠正已有字段
- **rule-0-00291-2**（critical）：单条 ADD HEADEN 只能配一个 DATATYPE；同 HEADERENNAME 下最多 40 种不同 DATATYPE — 后果：超规格新增失败
- **rule-0-00291-3**（warning）：整机规格 ≤100 个不同 HEADERENNAME；≤500 个前缀字段；≤32 个前缀开防欺诈；单报文 ≤16 个前缀开防欺诈 — 后果：超规格新增失败
- **rule-0-00291-4**（warning）：加密密钥强度建议 ≥8 字符且含大小写/数字/特殊字符 ≥2 类 — 后果：弱密钥有安全风险
- **rule-0-00291-5**（critical）：DATATYPE=MULTIPARA 时 PARADATATYPE1/2/3 不能同时 NONE 且任意两值不可相同 — 后果：新增失败
- **rule-0-00291-6**（info）：插 BILLINGTYPE 需 SMF `SET PFCPPVTEXT:FEATURE=CHARGING_TYPE,SWITCH=ENABLE`；插 SESSIONID 需 `FEATURE=AUTH_CLASS,SWITCH=ENABLE`
- **rule-0-00291-7**（info）：加密仅作用于字段取值（前缀名称不加密；抓包可见 `msisdn:` 明文 + 密文取值）
- **rule-0-00291-8**（info）：UDG 不支持 ZONEID 字段的头增强插入（即使配 ERRFLAGNAME 也无效）
- **rule-0-00291-9**（info）：当用户 RAT 类型为 NR 时，不支持插入 SGSNIP 类型的字段
