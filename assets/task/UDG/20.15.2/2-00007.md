# 基于业务时长的计费（GWFD-020302）

> 特性静态知识见 [GWFD-020302](feature/UDG/20.15.2/GWFD-020302.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

基于业务时长的计费是**[内容计费基本功能](task/UDG/20.15.2/2-00003.md)（GWFD-020301）的增强形式**：核心差异是 URR 统计类型固定为 **`OFFMETERINGTYPE=DURATION`**（离线）/ **`ONLMETERINGTYPE=DURATION`**（在线）——按匹配业务的**持续时间**累计统计（话单/CCR 上报时长），而非字节数或事件。

配置结构与内容计费/流量计费完全相同：核心对象链 **URR**（费率，DURATION 统计）→ **URRGROUP**（费率组，同时绑在线+离线 URR）→ **PCCPOLICYGRP**（策略组）→ **RULE**（绑流过滤器+策略组）→ **USERPROFILE**（规则容器）。**复用通用 backbone compound**（与内容计费/流量计费/在线计费共享），仅 URR 统计类型差异化为 DURATION。

时长统计有两种方式——**CTP 连续时长计费**（QCT=0，从业务访问到终止连续累计）与 **QCT 时长计费**（QCT≠0，QCT 定时器统计，空闲超时则扣除）。两种方式的 QCT 取值由 **UNC 侧** `ADD DCCTEMPLATE`（在线）/ `ADD OFCSRVTEMPLATE`（离线）配置，或由 OCS/CHF 下发——UDG 本身只设 DURATION 统计类型，CTP/QCT 选择是控制面维度。4 场景（URL/IMS/any/异常）由 DP（时长方式 + 业务过滤模式）驱动，参数差异化，不建多套流程。

## 配置流程（步骤编排，每步显示含命令）

1. **License**（单命令前置）：`SET LICENSESWITCH`（开启 `LKV3G5TBCS01`）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **计费三件套**（费率定义，URR 统计类型=DURATION）：`ADD URR`（`USAGERPTMODE`+`OFFMETERINGTYPE=DURATION`/`ONLMETERINGTYPE=DURATION`）+ `ADD URRGROUP`（同时绑在线+离线 URR）+ `ADD PCCPOLICYGRP` → [1-00010](task/UDG/20.15.2/1-00010.md)
3. **过滤链**（业务匹配，DP 选）：`ADD FILTER`/`ADD FILTERIPV6`/`ADD L7FILTER`/`ADD FLOWFILTER`/`ADD FLTBINDFLOWF`/`ADD PROTBINDFLOWF` → [1-00009](task/UDG/20.15.2/1-00009.md)
4. **规则与用户模板绑定**：`ADD RULE`（POLICYTYPE=PCC，绑 FLOWFILTER+PCCPOLICYGRP）+ `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
5. **计费收尾**（缺省费率+防欺诈+刷新）：`SET URRGRPBINDING`（缺省业务+信令费率，可带 `GLBDFTURRGSW=ENABLE`/`GLBDFTSIGURRGSW=ENABLE`）+ `ADD SPECURRGRPLIST`（可选防欺诈）+ `SET REFRESHSRV`（必须最后）→ [1-00012](task/UDG/20.15.2/1-00012.md)
6. **全局缺省 URR 组**（单命令，可选，承接原始文档步骤 10）：`SET SPECTRAFURRGRP`（处理特殊流量/未配 URR 的业务，配合 `GLBDFTURRGSW=ENABLE` 继承）→ [SET SPECTRAFURRGRP](task/UDG/20.15.2/0-00290.md)（atom 待建）

## 决策点

**DP（特性级）：时长方式（CTP/QCT）+ 业务过滤场景**（影响 URR 费率 + 过滤链）。时长计费场景下 URR 统计类型固定 DURATION（异常场景用 FREE 兜底），CTP/QCT 由控制面（UNC/OCS/CHF）决定：

| 场景 | 业务 | 过滤模式（[1-00009](task/UDG/20.15.2/1-00009.md) DP） | URR 统计类型（[1-00010](task/UDG/20.15.2/1-00010.md)） | 时长方式（控制面配置） | 费率组 |
|---|---|---|---|---|---|
| 场景1 URL | www.huawei.com | L34 + L7（URL 匹配） | OFFMETERINGTYPE/ONLMETERINGTYPE=**DURATION** | CTP 或 QCT（UNC ADD DCCTEMPLATE/OFCSRVTEMPLATE，OCS 下发优先） | UrrGp_URL |
| 场景2 IMS | 语音 | L34 IPv6（服务器 IP） | OFFMETERINGTYPE/ONLMETERINGTYPE=**DURATION** | CTP 或 QCT（同上） | UrrGp_IMS |
| 场景3 any | 基础业务（兜底） | L34 any to any（IPv4+IPv6） | OFFMETERINGTYPE/ONLMETERINGTYPE=**DURATION** | CTP 或 QCT（同上） | UrrGp_any |
| 场景4 异常 | 未匹配 | 不匹配（走缺省） | OFFMETERINGTYPE/ONLMETERINGTYPE=**FREE**（不计费） | — | UrrGp_abnormal |

**CTP vs QCT 时长方式**（核心 DP 维度，**控制面 UNC/OCS/CHF 决定，UDG 不配**）：

| 时长方式 | 标识 | 配置来源（UNC） | 统计起止 | 适用 |
|---|---|---|---|---|
| CTP 连续时长 | QCT=0 | `ADD DCCTEMPLATE`（在线，QCT=0）/ `ADD OFCSRVTEMPLATE`（离线，TQM=QCT 且 QCT=0） | 在线：收到 OCS 授权配额→去活；离线：收到首包→业务终止 | 精度低，连续累计 |
| QCT 定时统计 | QCT≠0 | `ADD DCCTEMPLATE`（在线，QCT≠0）/ `ADD OFCSRVTEMPLATE`（离线，TQM=QCT 且 QCT≠0） | 收到首包起；空闲启动 QCT 定时器，空闲≤QCT 计入、>QCT 超时停止 | 精度高，剔除空闲 |

> **优先级**：OCS/CHF 下发的 QCT/TQM（Time-Quota-Mechanism）优先于 UNC 本地配置；若同时下发 QCT 与 TQM，TQM 优先。

**联动**：
- 统计类型 DURATION → URR（[1-00010](task/UDG/20.15.2/1-00010.md)）按业务持续时间累计，话单/CCR 上报时长字段（离线话单时间信元 / 在线 `CC-Time`）。
- 计费方式（在线/离线/融合）→ URR 的 `USAGERPTMODE` + URRGROUP 同时绑在线+离线 URR（`UPURRNAME1`/`DOWNURRNAME1`=离线，`UPURRNAME2`/`DOWNURRNAME2`=在线）。
- 时长方式（CTP/QCT）→ **UNC 侧** `ADD DCCTEMPLATE`/`ADD OFCSRVTEMPLATE` 的 QCT/TQM 参数；UDG 配置不直接体现，但影响统计行为。
- 过滤模式 → 过滤链（[1-00009](task/UDG/20.15.2/1-00009.md)）出现哪些过滤器。
- 缺省费率 → 收尾（[1-00012](task/UDG/20.15.2/1-00012.md)）的 `DFTURRGRPNAME`（通常 any 组）+ `DFTSIGURRGNAME`（异常组），可带 `GLBDFTURRGSW=ENABLE`/`GLBDFTSIGURRGSW=ENABLE` 继承全局缺省；可选 `SET SPECTRAFURRGRP` 配全局缺省处理特殊流量。

## 约束（特性级 rule）

- **License 前置**（critical）：`SET LICENSESWITCH` 开启 `LKV3G5TBCS01`（82209823） — 未开 License 业务命令不生效。
- **依赖特性**（critical）：[SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，业务识别必选）+ [内容计费基本功能](feature/UDG/20.15.2/GWFD-020301.md)（GWFD-020301，时长计费是其增强） — 依赖链 SA-Basic → 内容计费 → 时长计费。
- **CTP↔QCT 不可切换**（critical）：计费会话过程中不支持 CTP 与 QCT 两种时长计费方式切换 — 会话建立后切换则统计异常。
- **URRGROUP 必须同时配在线+离线 URR**（critical）：只配一种且与用户计费模式不匹配则无法计费 — 计费失败。
- **融合计费特殊**（critical）：SMF `RGAPPLIED=DEFAULT`（SET USRPROFCHARGE/SET APNCHARGECTRL/ADD CHARGEMETHOD）时 URRGROUP 不能同时配离线+在线 URR — 否则计费冲突。
- **SMF/UPF 一致性**（critical）：`URRID`/`USAGERPTMODE`/`OFFMETERINGTYPE`/`ONLMETERINGTYPE`/`RULENAME`/`POLICYTYPE`/`POLICYNAME`/`USERPROFILENAME` 在 SMF 与 UPF（及 PCF）必须一致 — 不一致则计费异常/策略不生效。
- **FLOWFILTER 必须至少绑一个过滤条件**（critical）：否则 Rule 无效。
- **REFRESHSRV 必须最后执行**（critical）：Filter 变更才生效。
- **每个 USERPROFILE 必绑缺省费率**（critical）：`DFTURRGRPNAME`（业务）+ `DFTSIGURRGNAME`（信令） — 否则未匹配费率的报文无法计费。
- **RULE 优先级**（warning）：数字越小越高；只绑三四层的 RULE 优先级低于绑七层的；any 规则优先级最低（如 65000）。
- **防欺诈绑定约束**（warning）：防欺诈开启时，RULE 绑 URL 类型七层 filter 必须同时绑三四层 filter；防欺诈未开启时可仅绑七层。
- **统计类型 DURATION 由 URR 决定**（info）：CTP/QCT 选择由控制面 UNC/OCS/CHF 决定，UDG 仅设 DURATION；时长方式影响统计精度，不影响 UDG 配置结构。
- 性能（info）：已部署内容计费时开启时长计费基本无影响（5% 以内）；未部署内容计费直接开启则报文解析增加负荷，吞吐下降。
