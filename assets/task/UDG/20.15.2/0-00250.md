# 设置地址分配规则(入不转板)（SET IPALLOCRULE）

> 命令静态知识见 [SET IPALLOCRULE](command/UDG/20.15.2/SET-IPALLOCRULE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务配置全局地址分配规则。SET IPALLOCRULE 支持 IPv4 三级优先级规则，每级可选用 APN/DNN、位置（LOCATION）、SMF 三个维度组合（位域"与"关系），从同时满足所有条件映射的地址范围内分配地址。地址分配失败则用户无法上线。适用 NF：PGW-U、UPF。

**规则位域格式**：`APN-{0|1}&LOCATION-{0|1}&SMF-{0|1}`（1=按该维度，0=不按）。如 `APN-1&LOCATION-0&SMF-0` 表示仅按 APN/DNN 分配。

**多特性取值样例（数据规划表）**

| 特性场景 | FIRSTRULESW | FIRSTRULE | SECONDRULESW / SECONDRULE | THIRDRULESW |
|---|---|---|---|---|
| 入不转板（GWFD-020482） | `ENABLE` | `APN-1&LOCATION-0&SMF-0` | `DISABLE` / — | `DISABLE` |
| 基于 APN/DNN 分配（GWFD-020406/010105） | `ENABLE` | `APN-1&LOCATION-0&SMF-0` | `DISABLE` / — | `DISABLE` |
| 基于 SMF 分配（GWFD-020406/010105） | `ENABLE` | `APN-0&LOCATION-0&SMF-1` | `DISABLE` / — | `DISABLE` |
| 基于位置分配（GWFD-020406/020421） | `ENABLE` | `APN-0&LOCATION-1&SMF-0` | `DISABLE` / — | `DISABLE` |
| 基于 SMF+APN/DNN 分配（GWFD-020406/010105） | `ENABLE` | `APN-1&LOCATION-0&SMF-1` | `ENABLE` / `APN-0&LOCATION-0&SMF-1` | `DISABLE` |
| 基于 APN/DNN+位置分配（GWFD-020421） | `ENABLE` | `APN-1&LOCATION-1&SMF-0` | `ENABLE` / `APN-0&LOCATION-1&SMF-0` | `DISABLE` |

**规律**：第一级默认使能；单维度场景第二/三级默认 DISABLE；多维度叠加场景把次优先维度放第二级（SECONDRULESW=ENABLE），第三级默认 DISABLE。规则按优先级匹配，第一级匹配成功即用，否则依次降级，全失败则分配失败。

**脚本样例**（基于 APN/DNN 单维度，最常见）

```
SET IPALLOCRULE:FIRSTRULESW=ENABLE,FIRSTRULE=APN-1&LOCATION-0&SMF-0,SECONDRULESW=DISABLE,THIRDRULESW=DISABLE;
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：SECONDRULE/THIRDRULE（IPv4 二/三级规则内容）为 conditional，本特性二/三级开关 DISABLE 故不适用，不绑；IPV6FIRSTRULESW/IPV6FIRSTRULE/IPV6SECRULESW/IPV6SECONDRULE/IPV6THIRDRULESW/IPV6THIRDRULE（6 个 IPv6 规则集）为独立 IPv6 规则，本特性仅 IPv4 未用到，待 IPv6 场景特性揭示时增量补。

**两级控制说明**：UDG 上地址分配可基于全局（SET IPALLOCRULE）和基于 APN/DNN（SET APNIPALLOCRULE）分别使能。若配置了基于 APN 的地址分配规则，则用 APN 下的规则；否则用全局规则。

## 决策点：分配维度组合 — APN/LOCATION/SMF 三选多（DP 0-00127）

问：FIRSTRULE 的位域格式 `APN-{0|1}&LOCATION-{0|1}&SMF-{0|1}` 取哪种组合？决定按哪些维度匹配地址范围。

| 选项 | FIRSTRULE | 含义 |
|---|---|---|
| 仅 APN/DNN | `APN-1&LOCATION-0&SMF-0` | 按 APN/DNN 分配（最常用） |
| 仅 LOCATION | `APN-0&LOCATION-1&SMF-0` | 按位置分配 |
| 仅 SMF | `APN-0&LOCATION-0&SMF-1` | 按 SMF 分配 |
| APN+SMF 叠加 | `APN-1&LOCATION-0&SMF-1` | 双维度叠加 |
| APN+LOCATION 叠加 | `APN-1&LOCATION-1&SMF-0` | 双维度叠加 |

各维度按"与"关系组合匹配，需同时满足才命中。

## 决策点：三级优先级开关启用模式（DP 0-00128）

问：第二/三级规则开关是否使能？决定多维度叠加时次优维度的处理。

| 选项 | SECONDRULESW / THIRDRULESW | 含义 | 适用场景 |
|---|---|---|---|
| 单维度（默认） | DISABLE | 仅用第一级规则 | 单维度分配场景 |
| 多维度叠加 | 第二级 ENABLE + 第二级规则内容 | 降级维度放第二级 | APN+SMF / APN+LOCATION 等多维度场景 |

## 决策点：本命令与 SET APNIPALLOCRULE 的作用域（DP 0-00129）

问：地址分配走全局规则还是 APN 级规则？决定优先级顺序。

| 选项 | 优先级 | 含义 |
|---|---|---|
| SET APNIPALLOCRULE 已配 | 优先生效 | 用 APN 下的规则 |
| 全局 SET IPALLOCRULE 兜底 | 否则生效 | 用全局规则 |

## 约束

- 命令最大记录数 **1**（全局唯一实例，非 per-APN）。
- 支持 IPv4 三种地址分配规则优先级，**不同优先级的规则不可以配置为相同**（避免分配歧义）。
- 三级优先级独立配置，配置低优先级时不要求必须配置高优先级。
- 全局默认：第一优先级开关使能、第一优先级规则为基于 APN 映射的地址范围分配，第二/三优先级开关不使能。
- 命令执行后**只对新激活用户生效**（存量用户不感知）。
- 若按 SMF 映射的地址范围分配，需另行配置相关前置（见命令 wiki）。
