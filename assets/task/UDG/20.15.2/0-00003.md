# 配置PCC策略组（ADD PCCPOLICYGRP）

> 命令静态知识见 [ADD PCCPOLICYGRP](command/UDG/20.15.2/ADD-PCCPOLICYGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

PCCPOLICYGRP 是计费/策略的"集装器"：把 URR 组（计费）、PCC 动作属性（PCCACTPROPNAME，策略动作）、QoS 属性（QOSPROPNAME）、信令 URR 组（SIGURRGRPNAME，防欺诈）按业务聚合成策略集，后续 RULE 引用本组名。

**典型行（内容计费，按业务一组）**
```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="PPG_URL",URRGROUPNAME="UrrGp_URL";
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="PPG_IMS",URRGROUPNAME="UrrGp_IMS";
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="PPG_any",URRGROUPNAME="UrrGp_any",SIGURRGRPNAME="UrrGp_abnormal";
```

**绑定对象选择（从证据数据规划表归纳）**

| 参数 | 何时配 | 引用来源 |
|---|---|---|
| PCCPOLICYGRPNM | 总配（本端规划，名称不能重复） | 本命令产出 |
| URRGROUPNAME | 计费场景必配；纯策略控制可省 | [ADD URRGROUP](command/UDG/20.15.2/ADD-URRGROUP.md) 的 URRGROUPNAME |
| SIGURRGRPNAME | 需要信令/防欺诈计费时配（如 any 兜底业务） | 已配的 URRGROUP |
| PCCACTPROPNAME | 需要策略动作控制时配（IPv6 SA/ADC/无线资源优化/ProxyUPF） | 已配的 PCCACTPROP |
| QOSPROPNAME | 需要 QoS 控制时配（视频承载/双故障 Bypass） | 已配的 QOSPROP |

**命名约定**：内容计费按业务命名（`PPG_URL`/`PPG_IMS`/`PPG_any`）；纯 PCC/本地 PCC 命名 `pg_test`/`cg_red`/`cg-test` 等。多条数据时 PCCPOLICYGRPNM 不能相同。

## 约束

- **rule-0-00014**（warning）URRGROUPNAME 仅计费时必填：纯策略控制可省，计费场景漏配则计费不生效
- **rule-0-00130**（warning）最大记录数 40000（超 90% 上报 ALM-135602215，阈值可 MOD CFGTHRESHOLD 修改）；PCCPOLICYGRPNM 多条不能重复；执行后立即生效
