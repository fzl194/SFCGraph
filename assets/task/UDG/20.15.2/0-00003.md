# 配置PCC策略组（ADD PCCPOLICYGRP）

> 命令静态知识见 [ADD PCCPOLICYGRP](command/UDG/20.15.2/ADD-PCCPOLICYGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

PCCPOLICYGRP 是计费三件套的"集装器层"：把已配好的 URR 组（PCCPOLICYGRP 的"计费维度"）+ 可选 PCC 动作属性（策略动作维度）+ 可选 QoS 属性（QoS 维度）+ 可选信令 URR 组（防欺诈维度）按业务聚合成一个策略集，供下游 RULE 引用。本命令的核心配置空间有 3 个维度：**绑定对象组合**（决定配哪些引用参数）、**命名约定**（PCCPOLICYGRPNM 是 RULE 引用锚点）、**特性专属开关**（按特性使能的参数）。一个业务一组（多业务配多条）。

### 配置维度 1：绑定对象组合（参数 URRGROUPNAME / SIGURRGRPNAME / PCCACTPROPNAME / QOSPROPNAME，按业务需要组合）

PCCPOLICYGRP 可包含 **URRGroup / PccActionP / ServiceProp**（命令 notes 明确说明）。逐参数按需出现：

| 参数 | 何时配 | 引用来源 | 典型场景 |
|---|---|---|---|
| `URRGROUPNAME` | 计费场景必配；纯策略控制可省 | [ADD URRGROUP](task/UDG/20.15.2/0-00002.md) 产出的 URRGROUPNAME | 内容计费/在线/离线/融合计费、纯 PCC 计费场景 |
| `SIGURRGRPNAME` | 需要信令/防欺诈计费时配 | 已配的 URRGROUPNAME（如 `UrrGp_abnormal`） | 防欺诈兜底（any 业务带 `URR_abnormal` 配套） |
| `PCCACTPROPNAME` | 需要策略动作控制时配 | 已配的 PCCACTPROP | IPv6 SA、ADC、无线资源优化、ProxyUPF |
| `QOSPROPNAME` | 需要 QoS 控制时配 | 已配的 QOSPROP | 视频承载信令控制、双故障 Bypass |
| `EXTENDPROPNAME` | 需要扩展属性时配 | 已配的扩展属性 | 预留扩展场景 |

**典型组合（内容计费，按业务一组）**：
```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="PPG_URL",URRGROUPNAME="UrrGp_URL";
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="PPG_IMS",URRGROUPNAME="UrrGp_IMS";
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="PPG_any",URRGROUPNAME="UrrGp_any",SIGURRGRPNAME="UrrGp_abnormal";
```

**典型组合（纯 PCC，本地/动态 PCC 场景）**：
```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pg_test",URRGROUPNAME="urrgp_test";
```

**典型组合（多维度绑定，IPv6 SA + ADC + 视频承载等）**：
```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="<name>",URRGROUPNAME="<ugrp>",PCCACTPROPNAME="<pccact>",QOSPROPNAME="<qos>";
```

### 配置维度 2：命名约定（参数 PCCPOLICYGRPNM）

PCCPOLICYGRPNM 是 RULE.POLICYNAME 的引用锚点，**命名必须与下游 RULE 引用一致**：

| 命名风格 | 典型取值 | 适用场景 |
|---|---|---|
| 按业务语义 | `PPG_URL` / `PPG_IMS` / `PPG_any` | 内容计费（按业务分组的 PCC 策略组） |
| 通用默认 | `pg_test` / `cg_red` / `cg-test` | 纯 PCC / 在线 / 离线调试用 |

> 多条 PCCPOLICYGRP 数据时 `PCCPOLICYGRPNM` 不能相同（跨 PCC 策略组间不重复）——见约束 rule-0-00130。

### 配置维度 3：特性专属开关（按特性使能的参数）

命令真相表提供 18 个可选参数，绝大多数为冷门/特性专属；按特性使能分类如下：

| 参数 | 默认值 | 适用特性 | 作用 |
|---|---|---|---|
| `FUPSESSIONEXC` | DISABLE | FUP 类 | Session 级 FUP 累计标识 |
| `SIGNALASSOCIATE` | ENABLE | 信号关联计费 | 信令关联计费标识（防欺诈） |
| `ADCMUTEFLAG` | ENABLE | ADC 类 | ADC 静默通知标识 |
| `HTTP2DEGRADESW` | INHERIT | HTTP2 承载 | HTTP2.0 协议回落开关 |
| `EVENTCHARGEFLAG` | DISABLE | 事件计费 | 事件计费标识 |
| `EVENTCHGPOINT` | RESPONSE | 事件计费 | 事件计费点（REQUEST/RESPONSE/FINISH） |
| `MMSCHARGETYPE` | SINGLE | 彩信计费 | 彩信计费类型（单发/群发） |
| `TOKENFUNCFLAG` / `TOKENENCRYALG` / `TOKENSECRETKEY` | DISABLE / SHA256 | Token 检测 | Token 检测功能 |
| `CFGDOMAINNAME` | — | 配置域分组 | 配置域名称 |

> 配置方法段只列**已实证有特性使用的开关**；冷门参数不展开（命令 wiki 全参数表为准）。典型脚本中不写这些参数时按默认值生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：配置 PCC 策略组"——紧随 ADD URRGROUP 之后，过滤链绑定之前。

## 约束

- **URRGROUPNAME 仅计费时必填**（warning，rule-0-00014）：纯策略控制可省 URRGROUPNAME，计费场景漏配则计费不生效 — 后果：纯策略无影响；计费场景漏配则计费不生效
- **PCCPOLICYGRP 配置规格与唯一性**（warning，rule-0-00130）：单板最大 40000；超 90% 上报 ALM-135602215（阈值可 `MOD CFGTHRESHOLD（★R1.5 atom 待补）` 修改）；PCCPOLICYGRPNM 多条不能重复；执行后立即生效 — 后果：超规格新增失败；名称重复导致策略冲突
