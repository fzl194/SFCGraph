# 配置TWAMP客户端Light模式（ADD TWAMPCLIENT）

> 命令静态知识见 [ADD TWAMPCLIENT](command/UDG/20.15.2/ADD-TWAMPCLIENT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110921（TWAMP）**Light 模式独有**的第 4 步——UDG 作发送端主动发起探测，用静态配置替代 TCP 控制连接。Full 模式不用此命令。其 CLIENTID 将被下游 ADD TWAMPSENDER（0-00228）引用。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| CLIENTID | 客户端 ID | `1` | 全网规划，1~12000（命令最大记录数 12000） |
| TWAMPARCH | TWAMP 架构 | `LIGHT` | 固定 LIGHT（本任务核心标识） |
| AFTYPE | 地址族类型 | `IPV4` | 固定 IPV4 |
| LOCALIPV4 | 本端 IP | `10.1.1.5` | 引用 [创建TWAMP逻辑接口](task/UDG/20.15.2/0-00225.md) 的 IPV4ADDRESS（条件必填，AF=IPV4 时） |
| PEERIPV4 | 对端 IP | `10.2.2.5` | 对端规划，gNodeB/eNodeB 侧地址（条件必填，AF=IPV4 时） |
| LLOCALUDPPORT | 本端 UDP 端口 | `65450` | 全网规划，65450~65530（条件必填） |
| LPEERUDPPORT | 对端 UDP 端口 | `1025` | 全网规划，1~65535（条件必填） |
| VRFNAME | VPN 实例名 | `VPN_CN` | 引用 [创建TWAMP VPN实例](task/UDG/20.15.2/0-00224.md)（可选，本特性必填） |

**脚本样例**

```
ADD TWAMPCLIENT:CLIENTID=1,TWAMPARCH=LIGHT,AFTYPE=IPV4,LOCALIPV4="10.1.1.5",PEERIPV4="10.2.2.5",LLOCALUDPPORT=65450,LPEERUDPPORT=1025,VRFNAME="VPN_CN";
```

**操作步骤定位**：紧接 ADD TWAMPLOGICINF 之后、ADD TWAMPSENDER 之前。LOCALIPV4 须与上游逻辑接口 IP 一致（feature-rule 0-00081 TWAMP 一致性）；VRFNAME 须与 ADD TWAMPVPNINST 一致。模式互斥由 feature-DP 0-00036 承担（LIGHT/FULL 二选一）。可选参数 DSCP（默认 0）、DESCRIPTION 本特性未用。
