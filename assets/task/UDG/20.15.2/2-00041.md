# IPv6 Prefix Delegation（GWFD-020406）

> 特性静态知识见 [GWFD-020406](feature/UDG/20.15.2/GWFD-020406.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-020406-IPv6PrefixDelegation.md)。
>
> **本页定位**：配置生成实例化视角——IPv6 Prefix Delegation（IPv6 PD / UDNG / delegated-prefix / Prefix Delegation）的完整配置体系。在 GWFD-020401（IPv6 承载上下文）之上叠加"前缀代理"能力，**唯一配置标志** = `ADD SECTION V6PREFIXLENGTH=49~63`（<64）。
>
> **★ 关键差异**：变体1-4 下行路由走 OSPFv3（纯 IPv6）；**变体5 下行路由走 OSPF**（IPv4，因位置场景双栈混合）。复用 [1-00024](task/UDG/20.15.2/1-00024.md) 时变体5 勿按 OSPFv3 典型脚本盲目套用，必须改 OSPF 段。

## 配置概览

本特性按 **5 种激活变体 × 9~10 步通用骨架** 组织配置：

**对象链**：
```
                       License使能（变体1-4=LKV3G5P6PD01 / 变体5=LKV3G5LBAA01）
                                    ↓
                    地址池体系（POOLTYPE=EXTERNAL/LOCAL + V6PREFIXLENGTH<64 ★）
                                    ↓
                    地址分配规则（APN-1&LOCATION-0&SMF-0 等规则字符串）
                                    ↓
                    下行路由发布（变体1-4=OSPFv3 增强 / 变体5=OSPF）
```

**通用骨架（按变体组合步骤，差异在步骤 3-9）**：

| 步骤 | 干啥 | 主要命令 |
|---|---|---|
| 1. License 开关 | 打开 LKV3G5P6PD01（变体1-4）/ LKV3G5LBAA01（变体5） | `SET LICENSESWITCH` |
| 2. VPN + APN 准备 | VPN/APN 实例 + 地址分配属性使能 | `ADD VPNINST` / `ADD APN` / `SET APNADDRESSATTR` |
| 3. 地址池三级 | POOL（EXTERNAL/LOCAL）+ SECTION（V6PREFIXLENGTH=63 ★）+ POOLGROUP + POOLBINDGROUP | `ADD POOL` / `ADD SECTION` / `ADD POOLGROUP` / `ADD POOLBINDGROUP` |
| 4. PD 冲突前缀（可选） | 标识已使用 IPv6 前缀不参与分配 | `ADD CONFLICTIPV6` |
| 5. POOLGRPMAP 映射 | APN/SMF/LOCATION → POOLGROUP | `ADD POOLGRPMAP` |
| 6. SMF NodeID（变体3-4 必配） | SMF 节点标识 | `ADD CPNODEID` |
| 7. 地址分配规则 | 全局三级规则 + 变体规则字符串 | `SET IPALLOCRULE` |
| 8. SMF/位置 开关 | 变体3-4 配 IPALLOCBYSMFGLBSW；变体5 配 IPALLOCBYLOCGLBSW | `SET IPALLOCBYSMFGLBSW` / `SET IPALLOCBYLOCGLBSW` |
| 9. 下行路由发布 | **变体1-4 用 OSPFv3（含 VPNINSCAPSIMFLG=TRUE 增强）**；**变体5 用 OSPF** | `ADD OSPFV3` 系列 或 `ADD OSPF` 系列 |
| 10. 位置白名单（变体5 可选） | 白名单用户不参与位置分配 | `ADD ADRLOCWHITELST` |

**关键约束**：
- **必须加载 License**：变体1-4 → LKV3G5P6PD01；变体5 → LKV3G5LBAA01（★变体5 License 不一致）
- **强依赖 GWFD-020401 IPv6 承载上下文**（UDG 20.0.0+，License LKV3G5V6PB01）
- **条件依赖 GWFD-020403 IPv4v6 双栈**（双栈 PD 用户需业务解析时）
- **V6PREFIXLENGTH<64 是 PD 唯一配置标志**（49~63；=64 为普通 IPv6 单栈）
- 5 条互斥：L2TP VPN、用户面地址自动检测、NAT、入不转板、通用DNN漫游分流（手机后路由用户）
- VPN 三者一致：地址池 / APN / Gi/SGi/N6 外联口

## 约束

- **License 必配**（critical，rule-0-00528）：变体1-4 必须加载 LKV3G5P6PD01 + SET LICENSESWITCH；变体5 必须加载 LKV3G5LBAA01 + SET LICENSESWITCH — 后果：License 未使能致 PD 特性不工作
- **V6PREFIXLENGTH PD 标志**（critical，rule-0-00529）：ADD SECTION 的 V6PREFIXLENGTH 取值 49~63（<64）即触发 PD 方式分配；=64 为普通 IPv6 单栈 — 后果：=64 致 PD 流程不触发，无 delegated-prefix 下发
- **强依赖 GWFD-020401 承载**（critical，rule-0-00530）：UDG 需要建立 IPv6 承载上下文，必须先开启 GWFD-020401（License LKV3G5V6PB01）— 后果：020401 未开启致 PD 承载无法建立
- **条件依赖 GWFD-020403 双栈**（warning，rule-0-00531）：若 PD 用户为 IPv4v6 双栈且需业务解析，必须先开启 GWFD-020403（License LKV3G5VDSA01）— 后果：020403 未开启致双栈 PD 业务解析失败
- **VPNINSCAPSIMFLG 固定 TRUE**（critical，rule-0-00532）：ADD OSPFV3 的 VPNINSCAPSIMFLG 必须配 TRUE（去使能 VPN 路由环路检测），否则 OSPFv3 路由引入失败 — 后果：未配致 OSPFv3 路由发布失败
- **OSPFv3 PROTOCOL=wlr**（critical，rule-0-00533）：OSPFV3IMPORTROUTE 的 PROTOCOL 必须为 wlr（不能只选 wlr_sp/wlr_ud）— 后果：选错致下行路由不发布
- **OSPFv3 DRPRI=0**（warning，rule-0-00534）：OSPFV3INTERFACE 的 DRPRI 配 0 避免 VNF 被选为 DR/BDR — 后果：被选为 DR 致异常
- **本地池同 APN 前缀长度一致**（critical，rule-0-00535）：本地地址池方式分配时，同一 APN 下的 IPv6 delegated-prefix 长度必须相同 — 后果：长度不一致致用户激活失败
- **外部分配同 APN 前缀长度可异**（info，rule-0-00536）：外部网元（SMF/RADIUS）分配时，同一 APN 下的前缀长度可以不同 — 后果：正常变体
- **Router ID 全网唯一**（critical，rule-0-00537）：OSPFv3 ROUTERID / OSPF SCHEMAROUID 必须全网唯一 — 后果：重复致邻接建立失败
- **VPN 三者一致**（critical，rule-0-00538）：地址池 / APN / Gi/SGi/N6 外联口 VPN 必须一致 — 后果：VPN 不一致致地址分配或路由异常
- **5 条互斥约束**（critical，rule-0-00539）：与 L2TP VPN（GWFD-020412）、用户面地址自动检测（GWFD-010108）、NAT、入不转板（GWFD-020482）、通用DNN漫游分流（GWFD-020531）互斥 — 后果：互斥特性同时部署致 PD 用户异常
- **DHCP Server 不支持**（info，rule-0-00540）：IPv6 PD 不支持从 DHCP Server 获取并分配 IPv6 delegated-prefix — 后果：依赖外部 DHCP 不可行
- **单 UE 单 delegated-prefix**（info，rule-0-00541）：DHCPv6 Solicit 多 IA_PD option 时，仅第一个分配，其余返回 NoPrefixAvail — 后果：单 UE 多网段申请仅首个成功
- **delegated-prefix 永久有效**（info，rule-0-00542）：T1/T2/preferred-lifetime/valid-lifetime 固定 0xFFFFFFFF，不需更新 — 后果：移动路由器场景稳定性需求
- **★ 变体5 License 不一致**（critical，rule-0-00543）：变体5 激活脚本使用 LKV3G5LBAA01（基于位置的地址分配 License），非 LKV3G5P6PD01（PD 特性 License）；前提条件指向 GWFD-020421 — 后果：变体5 按 LKV3G5P6PD01 配致特性不工作
- **★ 变体5 下行路由 OSPF 非 OSPFv3**（critical，rule-0-00544）：变体5 因位置场景双栈混合（SUPPORTIPV4+SUPPORTIPV6），IPv4 侧需 OSPF 发布；不执行 OSPFv3 段 — 后果：按 OSPFv3 盲目套用致下行路由不发布
- **C 面网元华为配套**（critical，rule-0-00545）：对接控制面（SMF/SGW-C/PGW-C）必须为华为解决方案配套版本（PFCP 私有扩展）— 后果：非华为 C 面激活失败
