# 配置七层过滤器（ADD L7FILTER）

> 命令静态知识见 [ADD L7FILTER](command/UDG/20.15.2/ADD-L7FILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD L7FILTER 是过滤链 backbone 的"七层 L7 原子"：把"URL/HOST/User-Agent/Method 等七层关键字段"用一条记录描述（含子过滤器），供 ADD PROTBINDFLOWF 引用。本命令的核心配置空间有 3 个维度：**匹配类型**（URLTYPE 决定按 URL 还是 HOST 匹配）、**Referer 关联计费开关**（ISREFEREREN）、**子过滤器命名风格**（按协议语义命名 SUBL7FLTNAME）。

### 配置维度 1：匹配类型（参数 URLTYPE，必选）

URLTYPE 决定本条 L7FILTER 按哪个七层关键字段匹配：

| 取值 | 含义 | 配套必选参数 | 典型场景 |
|---|---|---|---|
| **URL**（默认/高频） | 按 URL 匹配 | URL（含通配符 `*`） | 内容计费 DNS/HTTP、URL 过滤、PCC 等绝大多数场景 |
| 其他（HOST / USERAGENT 等） | 按主机名/客户端类型匹配 | HOST / USERAGENT / METHODTYPE 等 | 冷门，14 特性均未用到，由 L7FILTER URL 通配覆盖 |

**典型脚本（URL 匹配）**：
```
ADD L7FILTER: L7FILTERNAME="L7_URL", SUBL7FLTNAME="dns", URLTYPE=URL, URL="www.huawei.com";
```

> 9 特性证据中 URLTYPE 固定为 `URL`；HOST/USERAGENT/METHODTYPE/XHEADERNAME/MMSBLURMATCH 等参数实例化不展开。

### 配置维度 2：Referer 关联计费开关（参数 ISREFEREREN，可选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **DISABLE**（默认，不配置） | 普通七层 URL 过滤，不关联来源 URL 计费 | 内容计费/PCC/URL过滤 等默认场景 |
| **ENABLE** | 使能 Referer 关联计费——把经链接访问目标 URL 的流量计回链接所在 URL | 关联 URL 核减（GWFD-020304） |

### 配置维度 3：子过滤器命名风格（参数 SUBL7FLTNAME）

SUBL7FLTNAME 是子七层过滤器的名字（在所属 L7FILTERNAME 下唯一），按协议语义命名：

| 命名风格 | 典型取值 | 适用场景 |
|---|---|---|
| 按协议 | `dns`（DNS）、`http`（HTTP） | 计费 DNS/HTTP 等 |
| 按业务 | `url_test` / `http_test` | 测试场景 |

**典型范式（DNS 计费场景）**：
```
ADD L7FILTER: L7FILTERNAME="L7_URL", SUBL7FLTNAME="dns", URLTYPE=URL, URL="www.huawei.com";
```

随后由 [ADD FLOWFILTER](task/UDG/20.15.2/0-00007.md) 建流过滤器 → [ADD PROTBINDFLOWF](task/UDG/20.15.2/0-00009.md) 绑 L7FILTERNAME 到 FLOWFILTER，完成七层+三四层组合过滤。

> **防欺诈约束**（来自命令 notes 跨命令提示）：RULE 绑 URL 类型 7 层 filter 时，若防欺诈功能开启，必须同时绑三四层 filter；未开启可仅绑 7 层。

**步骤位置**：在所属 feature 编排中通常位于"步骤 3：配置七层过滤条件"——ADD FILTER/L7FILTER 后，ADD FLOWFILTER 之前，ADD PROTBINDFLOWF 引用本命令产出的 L7FILTERNAME。

## 决策点：Referer 关联 URL 核减开关（DP 0-00044）

问：是否对该七层过滤器使能 Referer 关联计费？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不使能 Referer 关联（OFF，默认） | 不配 ISREFEREREN（缺省） | 普通七层 URL 过滤（内容计费/PCC 等默认场景） |
| 使能 Referer 关联 URL 核减（ON） | ISREFEREREN=ENABLE | 关联URL核减（GWFD-020304）：将用户经链接访问目标URL的流量计回链接所在URL |

默认 OFF（绝大多数场景）；GWFD-020304 选 ON。

## 约束

- **ADD L7FILTER 延时生效与记录数上限**（critical，rule-0-00122）：执行后 **60s 固定延时生效**（非 SET REFRESHSRV 刷新，而是固定延时）；L7Filter 最大记录数 **20000**，超规格 90% 上报 ALM-135602215；单 L7FILTERNAME 下最多 **50000** 个子七层过滤器（SUBL7FLTNAME） — 后果：配置延时生效 / 容量超限告警 / 子过滤器超限
- **L7FILTERNAME+SUBL7FLTNAME 联合唯一**（critical，隐含）：同一 L7FILTERNAME 下子过滤器名字不能重复 — 后果：重复新增失败
- **L7FILTER 与三四层 filter 防欺诈联合约束**（warning，隐含）：防欺诈功能开启时，RULE 绑 URL 类型 7 层 filter 必须同时绑三四层 filter — 后果：仅绑 7 层时防欺诈不生效
