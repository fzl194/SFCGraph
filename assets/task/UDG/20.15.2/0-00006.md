# 配置七层过滤器（ADD L7FILTER）

> 命令静态知识见 [ADD L7FILTER](command/UDG/20.15.2/ADD-L7FILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

L7FILTER 定义七层 URL 过滤条件，与 FLOWFILTER/PROTBINDFLOWF 搭配组成业务流过滤器的七层维度。证据③中 5 个特性配置范式高度一致：

| 参数 | 实际取值约定 | 来源 |
|---|---|---|
| L7FILTERNAME | 统一命名 `L7_URL`（七层过滤器名，引用入口） | 本端规划 |
| SUBL7FLTNAME | 按协议命名 `dns`/`http`（DNS 场景=dns，HTTP 场景=http） | 本端规划 |
| URLTYPE | 固定 `URL` | 本端规划 |
| URL | 具体 URL，样例 `www.huawei.com`；可含 `*` 通配 | 本端规划 |

**典型范式（DNS 计费场景）**：
```
ADD L7FILTER: L7FILTERNAME="L7_URL", SUBL7FLTNAME="dns", URLTYPE=URL, URL="www.huawei.com";
```
随后由 [ADD FLOWFILTER](0-00007.md) 建 ff_L7 → [ADD FLTBINDFLOWF](0-00008.md) 绑三四层 → [ADD PROTBINDFLOWF](0-00009.md) 绑 `L7_URL` 到 ff_L7，完成七层+三四层组合过滤。

**防欺诈约束**：RULE 绑 URL 类型 7 层 filter 时，若防欺诈功能开启，必须同时绑三四层 filter；未开启可仅绑 7 层。

## 决策点

**DP 0-00044**（boolean, optional, always）是否对该七层过滤器使能 Referer 关联计费？

| 选项 | 联动 | 适用 |
|---|---|---|
| opt-referer-off（默认） | 不配 ISREFEREREN | 普通 URL 过滤/内容计费/PCC 等默认场景 |
| opt-referer-on | ISREFEREREN=ENABLE | 关联 URL 核减（GWFD-020304）：把经链接访问目标 URL 的流量计回链接所在 URL |

## 约束

- **rule-0-00044→0-00122**（critical）执行后 **60s 固定延时生效**（非 REFRESHSRV 刷新）；L7Filter 最大记录数 **20000**，超 90% 上报 ALM-135602215（阈值可 `MOD CFGTHRESHOLD` 修）；单 L7FILTERNAME 下最多 **50000** 个子七层过滤器。
