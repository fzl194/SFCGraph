# L2TP LNS 建立（L2TP 组 + LNS 扩展 + 组绑定源端接口）

> **本地配置方式（方式A）LNS 容器建立的可复用多命令模块**。被引用于 [2-00046](task/UDG/20.15.2/2-00046.md) GWFD-020412 L2TP VPN。
> 2026-07-12 建（防平铺，SOP §3.5——3 条命令阶段内聚 + L2TP 隧道族复用：主备/负荷分担/IPv6 LNS/多 LNS 扩展）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. L2TP 组（★LNS 容器）| `ADD L2TPGROUP` | GROUPID=组号, DOMAINNAME=域名, LOCALNAME=本端名, LOCALLNSMODE=REDUNDANCY/LOAD_SHARING, FIRSTLNSIP/SECONDLNSIP=主备 LNS, FIRSTPWD/SECONDPWD=隧道密码, HELLOINTERVALSW/HELLOINTERVAL/RETRYTIMES=HELLO 检测, VPNINSTANCE=绑 VPN, PPPMAGICNUMBER=ENABLE, MAXSENDWINSIZE=窗口上限, MAXSESSIONNUM=会话上限 |
| 2.（可选）扩展 LNS（>6 个或 IPv6 LNS）| `ADD L2TPLNSINFO` | GROUPID=同组, LNSNO=序号 1~30, LNSIPVER=IPV4/IPV6, IPV4ADDRESS/IPV6ADDRESS=LNS 地址, PWD/CFMPWD=隧道密码 |
| 3. 组绑定源端接口 | `ADD L2TPCLIENTIP` | L2TPGROUPID=组号, INTERFACENAME=Giif 接口名 |

**典型脚本（本地配置主备 LNS，IPv4）**：
```
// 步骤1：配置 L2TP 组（主备模式 REDUNDANCY，2 个内置 LNS）
ADD L2TPGROUP:GROUPID=1,DOMAINNAME="example.com",LOCALNAME="UPF",AVPHIDDEN=ENABLE,HELLOINTERVALSW=ENABLE,HELLOINTERVAL=120,VPNINSTANCE="vpn_l2tp",LOCALLNSMODE=REDUNDANCY,FIRSTLNSIP="10.10.10.1",FIRSTPWD="0123456",CFMFIRSTPWD="0123456",SECONDLNSIP="10.10.10.2",SECONDPWD="0123456",CFMSECONDPWD="0123456",PPPMAGICNUMBER=ENABLE,MAXSENDWINSIZE=32;

// 步骤2：将 L2TP 组绑定源端接口
ADD L2TPCLIENTIP:L2TPGROUPID=1,INTERFACENAME="giif1/0/0";

// （LNS 超过 6 个或 IPv6 LNS 时扩展，否则跳过）
// ADD L2TPLNSINFO:GROUPID=1,LNSNO=7,LNSIPVER=IPV4,IPV4ADDRESS="10.10.10.7",PWD="0123456",CFMPWD="0123456";
```

**步骤位置**：在 License 开关（[0-00019](task/UDG/20.15.2/0-00019.md)）+ VPN 实例（[0-00041](task/UDG/20.15.2/0-00041.md)）+ SET GLOBALL2TP（[0-00131](task/UDG/20.15.2/0-00131.md)）之后；SET APNL2TPATTR（[0-00132](task/UDG/20.15.2/0-00132.md)）的 L2TPGROUPID 指向本组的 GROUPID。

## 场景差异

| 场景 | LOCALLNSMODE | LNS 数量 | LNS IP 版本 | 走法 |
|---|---|---|---|---|
| **本地配置主备 LNS**（默认）| REDUNDANCY | 2（FIRSTLNSIP + SECONDLNSIP 内置）| IPv4 | 步骤 1（内置 2 LNS）+ 步骤 3 |
| **本地配置负荷分担** | LOAD_SHARING | 2~6（内置 + ADD L2TPLNSINFO 扩展）| IPv4 | 步骤 1（内置 2 LNS）+ 步骤 2 扩展（>2 LNS 时）+ 步骤 3 |
| **IPv6 LNS** | REDUNDANCY（主备强制）| 2（ADD L2TPLNSINFO LNSNO=1/2）| IPv6 | 步骤 1（不配内置 LNS IP）+ 步骤 2（LNSIPVER=IPV6，LNSNO=1 主/2 备）+ 步骤 3 |
| **多 LNS（>6）** | LOAD_SHARING | 7~30 | IPv4 | 步骤 1（内置 6 LNS）+ 步骤 2（扩展到 30 上限）+ 步骤 3 |

> **★ 关键约束**：
> - **主备模式仅支持 IPv4 LNS**——主备模式（LOCALLNSMODE=REDUNDANCY）下 LNS IP 必须用 ADD L2TPGROUP 的 FIRSTLNSIP/SECONDLNSIP 内置参数，不支持 IPv6 LNS
> - **IPv6 LNS 必须用 ADD L2TPLNSINFO**——IPv6 LNS 场景下 ADD L2TPGROUP 不配内置 LNS IP，改用 ADD L2TPLNSINFO LNSIPVER=IPV6 配置，且 LNSNO 只能 1（主）/2（备）
> - **同组内 IP 不重复**——ADD L2TPGROUP 内置 LNS 与 ADD L2TPLNSINFO 扩展 LNS 的 IP 地址不能重复，LNSNO 也不能相同
> - **同组内 IP 版本不混用**——同一 L2TP 组内不支持 IPv4 LNS 与 IPv6 LNS 共存
> - **同组 LNS 总数 ≤30**——ADD L2TPGROUP 内置 6 + ADD L2TPLNSINFO 扩展，合计不超过 30

## 决策点

| 选项（LNS 工作模式） | 影响（LOCALLNSMODE + LNS IP 配置方式） |
|---|---|
| REDUNDANCY（主备，默认推荐）| LOCALLNSMODE=REDUNDANCY + FIRSTLNSIP/SECONDLNSIP 内置（最多 2 个 IPv4 LNS）；主 LNS 故障自动切备 |
| LOAD_SHARING（负荷分担）| LOCALLNSMODE=LOAD_SHARING + 内置 2~6 + ADD L2TPLNSINFO 扩展到 30；多 LNS 间分担会话 |
| IPv6 LNS | ADD L2TPGROUP 不配内置 LNS IP + ADD L2TPLNSINFO LNSIPVER=IPV6 LNSNO=1/2（主备模式强制）|

> 联动特性级 DP：LNS 参数来源（本地配置 vs AAA 下发）由 [2-00046](task/UDG/20.15.2/2-00046.md) 场景 DP 决定。AAA 下发方式（方式B）不走本 compound，改走 [0-00134](task/UDG/20.15.2/0-00134.md) ADD L2TPRDSCLIENT。

## 约束

- **GROUPID 前置创建**（critical，rule-0-02001）：ADD L2TPCLIENTIP / ADD L2TPLNSINFO 的 GROUPID 必须先经 ADD L2TPGROUP 创建 — 后果：引用失败
- **GROUPID 唯一**（critical，rule-0-02002）：GROUPID 1~1500，全局唯一，不允许重复创建 — 后果：同名冲突命令拒收
- **DOMAINNAME 域名匹配**（critical，rule-0-02003）：用户 PPP 用户名的域名必须匹配 L2TPGROUP 的 DOMAINNAME 才能触发本组建立隧道 — 后果：L2TP 隧道不建立
- **主备模式仅 IPv4**（critical，rule-0-02004）：LOCALLNSMODE=REDUNDANCY 时 LNS IP 必须为 IPv4（FIRSTLNSIP/SECONDLNSIP），不支持 IPv6 LNS — 后果：配置拒收
- **IPv6 LNS 用 L2TPLNSINFO**（critical，rule-0-02005）：IPv6 LNS 场景必须用 ADD L2TPLNSINFO LNSIPVER=IPV6，LNSNO 只能 1（主）/2（备）— 后果：配置拒收
- **同组 IP 版本不混用**（critical，rule-0-02006）：同一 L2TP 组内不支持 IPv4 LNS 与 IPv6 LNS 共存 — 后果：配置拒收
- **同组 LNS 总数 ≤30**（warning，rule-0-02007）：ADD L2TPGROUP 内置 + ADD L2TPLNSINFO 扩展，合计不超过 30 — 后果：超规格失败
- **LNS 路由可达**（critical，rule-0-02008）：UDG 到 LNS 的物理链路必须连通（PING 验证）— 后果：隧道 DOWN
- **隧道密码两端一致**（warning，rule-0-02009）：FIRSTPWD/SECONDPWD 或 L2TPLNSINFO PWD 必须与 LNS 侧配置一致 — 后果：SCCRP 拒绝，隧道建立失败
- **GROUP 规格 1500**（info，rule-0-02010）：本地 L2TP Group 规格 1500 — 后果：超规格无法新建
- **INTERFACENAME 存在**（critical，rule-0-02011）：ADD L2TPCLIENTIP 的 INTERFACENAME 必须已通过 ADD LOGICINF 创建 — 后果：绑定失败
- **C-U 对称加密**（warning，rule-0-02012）：启用 SET L2TPN4KEY 加密时对端 SMF（UNC 侧 SET L2TPKEY）必须配置相同密钥 — 后果：L2TP 业务失败
