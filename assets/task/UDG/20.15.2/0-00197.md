# 增加ACL规则组（ADD ACLGROUP）

> 命令静态知识见 [ADD ACLGROUP](command/UDG/20.15.2/ADD-ACLGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。ACL 是一种 IP 包过滤技术，通过对 IP 报文的源/目的地址、通配位、协议号、源/目的端口号（五元组）进行匹配，符合规则集合的报文视为同一数据流，统一采取 Permit/Deny 等处理策略；**使用 ACL 前必须先建组**。

## 配置方法（配置方法字典）

ADD ACLGROUP 是"增加 ACL 规则组"命令。仅 1 参必选（ACLNAME）+ 4 参可选（ACLSTEP/ACLTYPE/ACLMATCHORDER/ACLDESCRIPTION）。配置空间有 4 个维度：**规则组标识**（ACLNAME，决定是数字段还是字符串名）、**规则组类型**（ACLTYPE，决定基本/高级/以太）、**步长**（ACLSTEP，仅未指定规则 ID 时生效）、**匹配顺序**（ACLMATCHORDER）。1 个特性命中（IPFD-012002 ACL）覆盖数字和字符串两种命名范式。

### 配置维度 1：规则组标识（参数 ACLNAME，必选）

| 取值形式 | 数据来源 | 适用范围 |
|---|---|---|
| **数字字符串**（如 "3001"） | 本端规划 | 数字段 ACL（IPFD-012002 范例 "3001"，高级范围 3000~3999） |
| **字母开头的字符串**（如 "acl1"） | 本端规划 | 命名 ACL（IPFD-012002 范例 "acl1"，长度 1~32） |

> **范围段约定（数字段）**：基本 2000~2999 / 高级 3000~3999 / 以太 4000~4999 / 接口 1000~1999。

### 配置维度 2：规则组类型（参数 ACLTYPE，可选）

| 取值 | 适用 ACLNAME 形式 | 默认行为 |
|---|---|---|
| **Basic** | 数字字符串 | 数字字符串不配 ACLTYPE 时按数字段推断类型 |
| **Advance**（默认） | 非数字字符串 | 非数字字符串不配 ACLTYPE 时默认高级 |
| **Ethernet** | — | 以太 ACL |
| **Interface** | — | 接口 ACL |

> **默认行为关键**：非数字字符串 ACLNAME → 默认 Advance；数字字符串 ACLNAME → 不配 ACLTYPE 时按数字段推断；ACLTYPE 与 ACLNAME 必须语义一致（数字 3001 不可配 Basic）。

### 配置维度 3：步长（参数 ACLSTEP，可选）

| 取值范围 | 默认 | 作用 |
|---|---|---|
| 整数（命令 wiki 给出范围） | 5 | 规则 ID 增长步长；仅当不指定规则 ID 时生效 |

### 配置维度 4：匹配顺序（参数 ACLMATCHORDER，可选）

| 取值 | 默认 | 作用 |
|---|---|---|
| **Config**（默认） | ✓ | 配置优先（按配置顺序） |
| **Auto** | — | 深度优先（按规则精确度匹配） |

**典型脚本（IPFD-012002 高级组 3001，数字段）**：
```
ADD ACLGROUP:ACLNAME="3001";
```

**典型脚本（IPFD-012002 命名组 acl1，非数字字符串）**：
```
ADD ACLGROUP: ACLNAME="acl1";
```

> 命令 notes：**最大记录数 65535**（整机 ACL 组总数）；**单组最多 16384 条规则**；**执行后立即生效**；**ACLTYPE 默认推断规则**：非数字 ACLNAME → 高级；数字 ACLNAME → 可省。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：建 ACL 规则组"——前置 无；下游 [0-00198](task/UDG/20.15.2/0-00198.md) ADD ACLRULEADV4（高级 ACL 规则，本组 ACLNAME 引用）；[0-00202](task/UDG/20.15.2/0-00202.md) ADD QOSRULEACL（流分类下挂 ACL）。

## 决策点：ACL 规则组类型选择（DP 0-00197）

问：ADD ACLGROUP 的 ACLNAME/ACLTYPE 选哪种组合？

| 选项 | ACLNAME | ACLTYPE | 适用场景 |
|---|---|---|---|
| **数字段-高级（3001）** | "3001" | 省略（数字段推断） | IPFD-012002 经典高级 ACL 范例 |
| **命名-高级（acl1）** | "acl1" | 省略（非数字默认 Advance） | IPFD-012002 命名高级 ACL 范例 |
| **基本** | "2001" | Basic | 基本 ACL（仅源 IP 匹配） |
| **以太** | "4001" | Ethernet | 以太网帧 ACL |
| **接口** | "1001" | Interface | 接口入方向 ACL |

> 1 个特性命中（IPFD-012002）覆盖数字+字符串两种；其余类型（Basic/Ethernet/Interface）按需取对应段号和类型。

## 约束

- **最大记录数 65535**（critical，rule-0-00197）：整机 ACL 规则组最大 65535 条 — 后果：超规格新增失败
- **单组最多 16384 条规则**（critical，rule-0-00197）：单个 ACL 规则组下最多 16384 条规则 — 后果：超限新增失败
- **执行后立即生效**（warning，rule-0-00197）：执行后立即生效 — 后果：误操作即刻生效
- **ACLTYPE 与 ACLNAME 必须语义一致**（critical，rule-0-00177）：数字 3001 不可配 Basic；非数字默认高级 — 后果：ACLTYPE 与数字段不一致命令拒收
- **ACLNAME 长度约束**（warning，rule-0-00197）：ACLNAME 字符串 1~32（不支持特殊字符） — 后果：超长命令拒收
- **ACLNAME 范围段语义**（info，rule-0-00197）：数字字符串须落在 1000-1999（接口）/ 2000-2999（基本）/ 3000-3999（高级）/ 4000-4999（以太） — 后果：跨段数字命令拒收
