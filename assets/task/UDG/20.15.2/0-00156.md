# 设置APN的IMS属性（SET APNIMSATTR）

> 命令静态知识见 [SET APNIMSATTR](command/UDG/20.15.2/SET-APNIMSATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令用于**将 APN 标记为 IMS 信令承载**，使 VoLTE 语音业务在该 APN 上可用（GWFD-020251 VoLTE 基础语音业务专属）。整机最多 10000 条 ApnImsAttr，单实例即用即配。

**配置范式**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| APN | `ims` | 已配置数据中获取 | 引用 [0-00020](task/UDG/20.15.2/0-00020.md)（ADD APN）已建实例，可用 LST APN 查询 |
| IMSSWITCH | `ENABLE` | 本端规划 | VoLTE 信令承载前置开关；本特性固定 ENABLE，不允许关闭 |

**配置步骤**

1. 前置：先执行 [0-00020](task/UDG/20.15.2/0-00020.md)（ADD APN）创建 APN 实例。
2. 配套前置：先执行 [0-00155](task/UDG/20.15.2/0-00155.md)（ADD APNIMSSIGFLTR）配置 IMS 信令分类器——filter 匹配依赖此命令。
3. 执行 SET APNIMSATTR 开启 IMS 使能：
   ```
   SET APNIMSATTR:APN="ims",IMSSWITCH=ENABLE;
   ```

**示例脚本**：`SET APNIMSATTR:APN="ims",IMSSWITCH=ENABLE;`

## 约束

- **rule-0-00356**（warning）整机最多 10000 条 ApnImsAttr，超限 SET 失败；命令立即生效；属高危命令，影响用户 VoLTE 业务；SGW-U/I-UPF 上使能会触发 PF 的 SIP 信令解析、可能影响性能；PGW-U/PSA-UPF 上若数据 APN 下发 QCI=5 业务承载会触发 filter 匹配与 SIP 解析；filter 匹配依赖 [0-00155](task/UDG/20.15.2/0-00155.md) 配置，配置错误影响 IMS 业务；APN 须先经 [0-00020](task/UDG/20.15.2/0-00020.md) 配置 — 后果：超限 SET 失败 / 未配 APNIMSSIGFLTR 则 filter 匹配失败致 IMS 业务异常 / 未先建 APN 则引用失败
