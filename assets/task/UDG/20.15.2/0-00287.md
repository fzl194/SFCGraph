# 配置时间段容器（ADD TIMERANGE）

> 命令静态知识见 [ADD TIMERANGE](command/UDG/20.15.2/ADD-TIMERANGE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`ADD TIMERANGE` 创建一个具名时间段容器，本身不包含时间细节——具体生效时段由 `ADD PERITIMERANGE`（0-00288，周期）和 `ADD ABSTIMERANGE`（绝对）填充。容器随后被 `ADD BWMRULE` 的 `TIMERANGENAME1~4` 引用，实现"同一规则在不同时间段用不同带宽控制器"。**仅 PGW-U/UPF 适用**。

### 配置维度 1：容器命名（参数 TIMERANGENAME）
| 取值 | 作用 | 约束 |
|---|---|---|
| 字符串 1~31 字符 | 容器唯一标识，字母开头，不支持空格，不区分大小写 | 必选；整机最大 200 条 |

### 配置维度 2：配置域（参数 CFGDOMAINNAME，可选）
| 取值 | 作用 |
|---|---|
| 字符串 1~31 字符 | 命令所属公共配置域名称，时段带宽控制场景未使用 |

**典型脚本**：

```
// 建两个时段容器（白天/夜间各一套带宽控制器）
ADD TIMERANGE:TIMERANGENAME="timerange1";
ADD TIMERANGE:TIMERANGENAME="timerange2";
```

**步骤位置**：时段带宽控制链路的第 1 步——必须先于 `ADD PERITIMERANGE`（0-00288）和 `ADD BWMRULE` 的时段绑定；先建容器，再填周期时段，最后把容器挂到规则上。

## 决策点：容器数（DP 0-00287）

一个 TIMERANGENAME 承载一组（最多 32 周期 + 12 绝对）时段，需按"一套带宽控制器对应一个时段容器"规划。

| 选项 | 容器数 | 说明 |
|---|---|---|
| 单容器（简单分时段） | 1 个 | 单一时段窗口用一套控制器 |
| 多容器（多套控制器切换） | 2~4 个 | BWMRULE 支持 TIMERANGENAME1~4，每容器绑一组上下行控制器，实现按时段切换限速策略 |

时段生效规则：同时含周期+绝对时，生效范围 = 绝时间段并集 ∩ 周期时段并集；仅含其一则取该类并集。当前时间落在生效范围内才生效。

## 约束

- **rule-0-00287-1**（critical）：最大记录数 200 — 超规格新增失败
- **rule-0-00287-2**（warning）：命令执行后立即生效 — 无需刷新
- **rule-0-00287-3**（critical）：每个时间段下最多 32 条周期时段 + 12 条绝对时段 — 子记录上限
- **rule-0-00287-4**（info）：周期+绝对共存时生效范围取交集 — 计算规则
- **rule-0-00287-5**（critical）：被 BWMRULE 绑定后，新增周期时段不能与该规则已绑其它时段重叠 — 重叠校验
- **rule-0-00287-6**（critical）：被 BWMRULE 绑定后，新增周期时段间隔不能小于 30 分钟 — 最小粒度
