# 配置OSPF引入外部路由(IPv4)（ADD OSPFIMPORTROUTE）

> 命令静态知识见 [ADD OSPFIMPORTROUTE](command/UDG/20.15.2/ADD-OSPFIMPORTROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD OSPFIMPORTROUTE 将外部路由（wlr）引入 OSPF 域并在域中发布。`PROCID` 引用上游 ADD OSPF（0-00074）；`TOPOID` 拓扑标识在 8 个特性脚本中均显式传入 `TOPOID=0`（虽命令真相为 optional/默认0，但按"脚本实际用到"提为 required/fixed）；`PROTOCOL` 协议分类在主流场景（GWFD-010104/010105/010155/020406/020421）固定取 `wlr`。引入路由开销/标签/类型等进阶参数（9 项）按通用配置筛不绑。

**取值范式（从证据数据规划表归纳）**

| 特性 / 场景 | PROCID | TOPOID | PROTOCOL | 脚本样例 |
|---|---|---|---|---|
| GWFD-010104/010105/020406/020421 地址分配类（引入 WLR 路由） | 100（引用 0-00074） | 0（全网规划，固定） | wlr（全网/本端规划，固定） | `ADD OSPFIMPORTROUTE:PROCID=100,TOPOID=0,PROTOCOL=wlr;` |
| GWFD-010155 PGW-U 到 ePDG（引入 WLR 路由，含默认 FALSE 开关） | 6（引用 0-00074） | 0（固定） | wlr（全网规划） | `ADD OSPFIMPORTROUTE:PROCID=6,TOPOID=0,PROTOCOL=wlr,IMPTCOSTCFG=FALSE,IMPTTAGCFG=FALSE,IMPTTYPECFG=FALSE;` |
| IPFD-015004 OSPF over IPsec（引入 Direct 直连路由） | 20214（引用 0-00074） | （脚本未显式传） | Direct（本端规划） | `ADD OSPFIMPORTROUTE:PROCID=20214,PROTOCOL=Direct;` |

> PROTOCOL 取值：主流场景固定 wlr（010104 本端规划 / 010155 全网规划，取值同构无分叉，保 global_planned）。GWFD-010155 脚本显式带的 IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG=FALSE 属默认 FALSE 空操作且 010104 脚本未含（分叉+默认值），按通用配置筛不绑。

**关键规则**
- **PROCID 必须引用已创建的 OSPF 进程**：binding=reference→[创建OSPF进程(IPv4)](task/UDG/20.15.2/0-00074.md)，先 ADD OSPF 再 ADD OSPFIMPORTROUTE（命令前置约束）。
- **PROTOCOL 不能只选 WLR_SP 或 WLR_UD**：md 说明 GWFD-010104 强调，采用路由策略方式发布路由时，必须选 `WLR` 方式发布所有路由（不能只发布 WLR_SP 或 WLR_UD），否则发布路由不全。
- **TOPOID=0 固定**：8 个特性脚本均显式传入 TOPOID=0（命令真相 optional/默认0），按脚本实际用到提为 required/fixed。

## 约束

- **rule 0-00193**（constraint_rule/execution_constraint）ADD OSPFIMPORTROUTE 须满足：(1) 前置——必须先执行 ADD OSPF 配置了 OSPF 进程（PROCID）才能使用此命令；(2) 命令最大记录数 8000；(3) 采用路由策略方式发布路由时，PROTOCOL 不能只选 WLR_SP 或 WLR_UD，必须选 WLR 方式发布所有路由；(4) IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG 为引入路由开销/标签/类型的配置开关（默认 FALSE=不配置） — 后果：路由引入失败 / 发布路由不全 / 进程未创建报错（critical）。
