# 配置OSPF引入外部路由(IPv4)（ADD OSPFIMPORTROUTE）

> 命令静态知识见 [ADD OSPFIMPORTROUTE](command/UDG/20.15.2/ADD-OSPFIMPORTROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令是 OSPF backbone 4 件套之四**——必须在 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 创建 PROCID 进程之后执行；将外部路由引入 OSPF 域并在域中发布为 AS External LSA。

## 配置方法（配置方法字典）

ADD OSPFIMPORTROUTE 是"**将外部路由引入 OSPF 域**"命令。PROCID 必选；TOPOID 固定 0（拓扑标识）；PROTOCOL 必选（5 种协议类型）。核心配置空间有 4 个维度：**进程号**（PROCID 引用上游）、**拓扑标识**（TOPOID 固定 0）、**协议类型**（PROTOCOL 5 选 1 enum）、**引入路由参数化**（IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG 三开关 + 对应 COST/TAG/TYPE 实际值 + 路由策略 ROUPOLINAME 可选）。N 个特性命中（GWFD-010104/010105/020406/020421 / GWFD-010155 / IPFD-015004）覆盖 2 种 PROTOCOL 取值（wlr / Direct）。

### 配置维度 1：进程号（参数 PROCID，必选，引用上游）

| 参数 | 取值范围 | 数据来源 | 关联 |
|---|---|---|---|
| **PROCID** | 1~4294967295 | ref [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 已建进程 | 必须已存在，使用 LST OSPF 查看 |

> **关键约束**：**必须先执行 ADD OSPF 配置了 OSPF 进程才能使用本命令**。

### 配置维度 2：拓扑标识（参数 TOPOID，可选，固定 0）

| 参数 | 取值范围 | 默认值 | 备注 |
|---|---|---|---|
| **TOPOID** | 0（目前仅支持默认拓扑 0） | 0 | 8 个特性脚本均显式传入 TOPOID=0，按"脚本实际用到"提为 fixed |

### 配置维度 3：协议类型（参数 PROTOCOL，必选，5 选 1 enum）

PROTOCOL 决定引入什么类型的外部路由：

| 取值 | 协议类型 | 典型场景 |
|---|---|---|
| **Direct** | 直连路由 | IPFD-015004 OSPF over IPsec（引入直连网段到 OSPF 域） |
| **OSPF** | OSPF 路由 | OSPF 进程间路由引入（PROTOCOLPROCID 选源进程号） |
| **Static** | 静态路由 | 静态路由引入 OSPF 域 |
| **BGP** | BGP 路由 | BGP 路由引入 OSPF 域（PERMITIBGPCFG 控制是否引入 IBGP） |
| **wlr** | 无线路由 | GWFD-010104 地址分配类 / GWFD-010155 PGW-U→ePDG 主流场景 |

> **PROTOCOLPROCID**：PROTOCOL=OSPF 时可选配引入路由协议的源进程号（默认 1）。**PROTOCOL 不能只选 WLR_SP 或 WLR_UD**：采用路由策略方式发布路由时，必须选 `WLR` 方式发布所有路由（不能只发布 WLR_SP 或 WLR_UD），否则发布路由不全。

### 配置维度 4：引入路由参数化（IMPT*CFG 开关 × COST/TAG/TYPE 实际值）

三个 CFG 开关独立门控其对应值：

| 子维度 | 开关参数 | 实际值参数 | 取值范围 | 联动 |
|---|---|---|---|---|
| **Cost** | IMPTCOSTCFG（FALSE/TRUE） | COST | 0~16777214 | IMPTCOSTCFG=TRUE 时 COST 条件必选 |
| **Tag** | IMPTTAGCFG（FALSE/TRUE） | TAG | 0~4294967295 | IMPTTAGCFG=TRUE 时 TAG 条件必选 |
| **Type** | IMPTTYPECFG（FALSE/TRUE） | TYPE | 1~2（外部路由类型） | IMPTTYPECFG=TRUE 时 TYPE 条件必选 |

> 3 个 CFG 默认值均为 FALSE（=不配置，等价于系统计算）。
> **路由策略**（ROUPOOLNAME 1~200 字符串，区分大小写，可选）——用于路由策略控制引入路由。
> **PERMITIBGPCFG**（PROTOCOL=BGP 时条件可选，默认 FALSE）——是否使能引入 IBGP 路由。

**典型脚本（010104 地址分配，引入 wlr 路由）**：
```
ADD OSPFIMPORTROUTE: PROCID=100, TOPOID=0, PROTOCOL=wlr;
```

**典型脚本（010155 PGW-U→ePDG，引入 wlr 路由 + 显式 FALSE 三开关）**：
```
ADD OSPFIMPORTROUTE: PROCID=6, TOPOID=0, PROTOCOL=wlr, IMPTCOSTCFG=FALSE, IMPTTAGCFG=FALSE, IMPTTYPECFG=FALSE;
```

**典型脚本（IPFD-015004 OSPF over IPsec，引入 Direct 直连路由）**：
```
ADD OSPFIMPORTROUTE: PROCID=20214, PROTOCOL=Direct;
```

**典型脚本（OSPF 进程间路由引入）**：
```
ADD OSPFIMPORTROUTE: PROCID=1, TOPOID=0, PROTOCOL=OSPF, PROTOCOLPROCID=2;
```

> 命令 notes：**最大记录数 8000**；**执行后立即生效**；**必须先 ADD OSPF 才能使用本命令**；**TOPOID 目前只支持默认拓扑 0**。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N+3：引入外部路由"——前置 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF。

## 决策点：引入协议类型（DP 0-00077）

问：ADD OSPFIMPORTROUTE 的 PROTOCOL 取何值？

| 选项 | PROTOCOL | 联动 | 适用场景 |
|---|---|---|---|
| **wlr（无线路由，default）** | wlr | — | 地址分配类对接（GWFD-010104/010105/020406/020421）；PGW-U→ePDG（GWFD-010155） |
| **Direct 直连路由** | Direct | — | OSPF over IPsec 引入直连网段（IPFD-015004） |
| **OSPF 进程间** | OSPF | PROTOCOLPROCID 源进程号 | OSPF 多进程间路由引入 |
| **BGP** | BGP | PERMITIBGPCFG 控制 IBGP | 引入 BGP 路由 |
| **Static** | Static | — | 引入静态路由 |

> 主流场景 PROTOCOL 固定 `wlr`；不能只选 `WLR_SP` 或 `WLR_UD` 子类，必须选 `WLR` 发布所有路由。PERMITIBGPCFG/PROTOCOLPROCID/ROUPOOLNAME 仅在特定 PROTOCOL 下条件可选。

## 约束

- **最大记录数 8000**（warning，rule-0-00477）：本命令最大记录数为 8000 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00477）：命令执行后立即生效 — 后果：误操作即刻生效
- **必须先 ADD OSPF 才能使用本命令**（critical，rule-0-00477）：只有执行 ADD OSPF 配置了 OSPF 进程才能使用此命令 — 后果：未建进程命令拒收
- **PROTOCOL 不能只选 WLR_SP/WLR_UD**（critical，rule-0-00477）：采用路由策略方式发布路由时，PROTOCOL 不能只选 WLR_SP 或 WLR_UD，必须选 WLR 方式发布所有路由 — 后果：选子类致路由发布不全
- **PROTOCOL 必选**（critical，rule-0-00477）：PROTOCOL 是必选参数 — 后果：未指定 PROTOCOL 命令拒收
- **TOPOID 固定 0**（info，隐含）：TOPOID 目前只支持默认拓扑 0 — 后果：选非 0 命令拒收
- **PROTOCOLPROCID 默认 1**（info，隐含）：PROTOCOL=OSPF 时 PROTOCOLPROCID 默认 1 — 后果：多进程场景需明确指定源进程号
- **CFG 三开关默认 FALSE**（info，隐含）：IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG 默认 FALSE=不配置（用系统计算） — 后果：选 TRUE 时 COST/TAG/TYPE 必填
