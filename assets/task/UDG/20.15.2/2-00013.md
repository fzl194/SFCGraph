# HTTP头防欺诈（GWFD-110401）

> 特性静态知识见 [GWFD-110401](feature/UDG/20.15.2/GWFD-110401.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性**强耦合** [2-00010](task/UDG/20.15.2/2-00010.md)（HTTP头增强）——防欺诈开关内嵌于 `ADD HEADEN` 的 `ANTIFRAUD` 参数，**不独立配置策略对象**，共用 HTTP 头增强的 HEADEN/RULE/FILTER/L7FILTER/FLOWFILTER 全套配置载体。

## 配置概览

HTTP 头防欺诈是 UDG 在 HTTP 头增强过程中进行**检测和修正**，保证 HTTP 报文头增强的内容正确，防止用户使用欺诈的头增强信息进行业务。**核心机制：防欺诈开关内嵌于 `ADD HEADEN` 命令的 `ANTIFRAUD` 参数**（不独立配置策略对象），业务流通过 RULE 的 `POLICYTYPE=HEADEN` 触发（共用 HTTP 头增强动作机制），在头增强插入动作执行**之前**先执行防欺诈检测。

防欺诈检测逻辑：插入前先检测用户数据报文 HTTP 头中**是否已存在**要插入的字段——存在且取值不正确 → 纠正为正确值；存在多个 → 删除冗余保留一个正确值；不存在 → 执行正常头增强插入。执行顺序由软参 BIT1030 控制。

**与 HTTP 头增强（[2-00010](task/UDG/20.15.2/2-00010.md)）的关系**：配置结构零差异（完全复用头增强的 compound 骨架），差异仅三点——(1) License 双开：LKV3G5HHAS01（防欺诈）+ LKV3G5HTHE01（头增强）；(2) `ADD HEADEN` 必须配 `ANTIFRAUD=ENABLE`（头增强默认 DISABLE）；(3) `ADD HEADEN` 的 PREFIXNAME **不能为 HTTP 标准头域名**（如 host/accept），否则防欺诈失效。可选灰名单模式 `GRAYLIST=ENABLE`（只防欺诈不插入头增强字段）。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置（双开）**（打开防欺诈 + 头增强 License 开关，单命令执行两次）：`SET LICENSESWITCH`（**LKV3G5HHAS01** 防欺诈）+ `SET LICENSESWITCH`（LKV3G5HTHE01 头增强，强耦合必开）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **配置三四层过滤条件**（基于三四层触发时配，单命令链）：`ADD FILTER`（FILTERNAME+L34PROTOCOL=ANY+SVRSTARTPORT/SVRENDPORT）+ `ADD FLOWFILTER`（FLOWFILTERNAME）+ `ADD FLTBINDFLOWF` + `SET REFRESHSRV` → [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链 compound，仅含 FILTER/FILTERIPV6/L7FILTER/FLOWFILTER/FLTBINDFLOWF/PROTBINDFLOWF）
3. **配置七层过滤条件 + 协议识别**（基于七层触发时配，URL+METHODTYPE）：
   - 过滤条件：`ADD L7FILTER`（**URL+METHODTYPE=GET-1/POST-1**）+ `ADD PROTBINDFLOWF`（**PROTOCOLNAME=http**, L7FILTERNAME）+ `SET REFRESHSRV` → [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链 compound）
   - 协议识别（直引 atom，**非 1-00009 compound 成员**）：`ADD WELLKNOWNPORT`（http 端口）→ [0-00039](task/UDG/20.15.2/0-00039.md) + `LOD SIGNATUREDB` → [0-00027](task/UDG/20.15.2/0-00027.md) + `LOD PARSERDB` → [0-00152](task/UDG/20.15.2/0-00152.md)
4. **配置头增强对象（使能防欺诈）—— 防欺诈核心步骤**：`ADD HEADEN`（HEADERENNAME+DATATYPE+**PREFIXNAME（非标准头域名）**+ENCRYALGORI+PSWDKEY+**ANTIFRAUD=ENABLE**，多字段多次执行；可选 GRAYLIST=ENABLE 灰名单模式）→ [1-00013](task/UDG/20.15.2/1-00013.md)
5. **配置头增强规则**（POLICYTYPE=HEADEN 触发，规则绑定收尾）：`ADD RULE`（POLICYTYPE=HEADEN + POLICYNAME=HEADERENNAME）+ `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)

## 决策点

**DP（特性级）：触发方式 + 防欺诈字段类型 + 加密算法 + 灰名单模式**（影响过滤链走法 + ADD HEADEN 参数）。维度按防欺诈 activation §表2/§表3/§表4 + 原理文档 §HTTP头防欺诈类型 全列。

| 场景 | 触发方式 | 过滤链走法 | 防欺诈字段 | 加密 | 灰名单 |
|---|---|---|---|---|---|
| **基于七层 URL 触发（主流，activation 演示）** | 基于 URL+METHODTYPE | L7FILTER（URL+GET-1/POST-1）+ PROTBINDFLOWF（http+L7FILTER）| IMSI1（PREFIXNAME=X-imsi，非标准头域名）| AES128（+PSWDKEY）| DISABLE |
| 基于三四层触发 | 基于三四层特征 | FILTER（L34PROTOCOL=ANY）+ FLTBINDFLOWF | 任意（非标准头域名）| 任意 | 任意 |
| 灰名单模式（只防欺诈不插入）| 任意 | 同上 | MSISDN/IMSI 等 | 任意 | **ENABLE**（须配合 ANTIFRAUD=ENABLE）|
| 加密变体（安全推荐）| 任意 | 同上 | 任意（非标准头域名）| **AES128_GCM/AES256_GCM**（+PSWDKEY+SET AES）| DISABLE |

**联动**：
- 触发方式 → [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链走法（L7FILTER+PROTBINDFLOWF / FILTER+FLTBINDFLOWF）
- 防欺诈字段类型 → **PREFIXNAME 不能为 HTTP 标准头域名**（如 host/accept），否则防欺诈失效（critical 约束）
- 加密算法 → [1-00013](task/UDG/20.15.2/1-00013.md) 场景差异 §2（联动 SET AES/ADD RSA* 等加密配置命令）
- 灰名单模式 → [1-00013](task/UDG/20.15.2/1-00013.md) 场景差异 §5（GRAYLIST=ENABLE 只防欺诈不插入，须配合 ANTIFRAUD=ENABLE）
- 执行顺序 → 软参 BIT1030 控制防欺诈是否在头增强前执行（0=中执行/1=前执行，初始值 0；证据：软件参数文档 `BIT1030 控制HTTP头防欺诈功能是否在头增强业务前执行_08922610.md`；默认防欺诈→纠正→插入）

## 约束（特性级 rule）

- **License 双开**（critical）：SET LICENSESWITCH 开启 `LKV3G5HHAS01`（防欺诈）**且** `LKV3G5HTHE01`（头增强，强耦合必开）— 任一未开则防欺诈不生效。
- **强耦合 HTTP 头增强**（critical）：GWFD-110261（License LKV3G5HTHE01）—— 防欺诈开关内嵌于 ADD HEADEN 的 ANTIFRAUD 参数，启用防欺诈必须开启头增强；防欺诈不独立配置策略对象。
- **强依赖 SA-Basic**（critical）：GWFD-110101（License LKV3G5SABS01）—— HTTP 报文解析基础。
- **依赖 SA-Web Browsing**（critical）：GWFD-110103（License LKV3G5SAWB01）—— HTTP/WAP2.0 协议解析必需（activation 前提条件明确要求）。
- **ANTIFRAUD=ENABLE 必配**（critical）：ADD HEADEN 的 ANTIFRAUD=ENABLE 是本特性核心开关 — 不配则防欺诈不生效（默认 DISABLE）。
- **PREFIXNAME 不可为 HTTP 标准头域名**（critical）：如 host/accept/user-agent 会导致防欺诈失效 — 后果：UDG 无法识别已有字段，防欺诈失败（activation §表4 说明 + 特性知识 §1.9 约束 #2）。
- **POLICYTYPE=HEADEN 触发条件**（critical）：RULE.POLICYTYPE=HEADEN + POLICYNAME=HEADERENNAME（直接引用，PCC 策略组名称=NULL）— 指错则防欺诈不触发。
- **仅 HTTP1.x，不支持 HTTP2.0/HTTPS**（warning）：本特性仅对 HTTP/WAP2.0 的 Post/Get 报文上行报文有效；HTTPS 防欺诈需 GWFD-110263（ADD TLSHEADEN ANTIFRAUD=ENABLE）— 混用则对应协议报文不做防欺诈。
- **不支持加密场景下基于 host 的计费和控制**（warning）：防欺诈与 host 计费控制存在互斥（特性知识 §1.9 约束 #4）。
- **灰名单模式须配合 ANTIFRAUD=ENABLE**（warning）：GRAYLIST=ENABLE 单独配无效，必须同时 ANTIFRAUD=ENABLE — 否则灰名单模式不生效。
- **非 HTTP 协议业务流不处理**（info）：配置了三四层或七层特征触发的防欺诈，对非 HTTP 协议的业务流即使匹配到规则也不做防欺诈处理（设计如此）。
- **执行顺序由 BIT1030 控制**（info）：软参 BIT1030 控制防欺诈是否在头增强业务前执行（0=在头增强中执行/1=在头增强前执行，初始值 0；证据：软件参数文档 `BIT1030 控制HTTP头防欺诈功能是否在头增强业务前执行_08922610.md`；默认防欺诈→纠正→插入；灰名单模式跳过插入）。
- **计费与话单**（info）：本特性不涉及计费与话单（产品文档明确声明）。
