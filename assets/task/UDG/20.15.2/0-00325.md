# 查询地址池使用情况（DSP POOLUSAGE）

> 命令静态知识见 [DSP POOLUSAGE](command/UDG/20.15.2/DSP-POOLUSAGE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

DSP POOLUSAGE 是地址池族**资源占用查询**命令，触发查询输出当前地址池的占用情况（占用率/已分配IP数/可用IP数/总数），用于触发 ALM-81216 地址池占用率高告警诊断与扩容决策。命令核心配置空间有 4 个维度：**地址池引用**（IPPOOL 必选 + 必须已建）、**IP 版本**（IPVERSION 决定 V4/V6 统计维度，默认 IPV4）、**整池查询**（WHOLEPOOL 控制是否遍历 POOL 内所有 SECTION）、**地址段**（SECTIONNUM 指定单一地址段号）。

### 配置维度 1：地址池引用（参数 IPPOOL，必选）

| 参数 | 范围 | 备注 |
|---|---|---|
| **IPPOOL** | 字符串 1~79 | 引用已配的 [0-00043](task/UDG/20.15.2/0-00043.md) POOLNAME；前置依赖：POOL 必须先建 |

### 配置维度 2：IP 版本（参数 IPVERSION，必选，默认 IPV4）

| 取值 | 统计维度 |
|---|---|
| **IPV4**（默认） | 统计 IPv4 地址占用 |
| **IPV6** | 统计 IPv6 前缀占用 |

### 配置维度 3：整池查询（参数 WHOLEPOOL，可选）

| 取值 | 行为 |
|---|---|
| ENABLE | 遍历 POOL 内所有 SECTION 输出占用情况 |
| 不配 | 仅输出该 POOL 总占用情况 |

### 配置维度 4：地址段（参数 SECTIONNUM，可选）

| 参数 | 范围 | 适用 |
|---|---|---|
| **SECTIONNUM** | 整数 0~63 | 指定 POOL 内单一地址段（010105 调测范式：`SECTIONNUM=0`） |

**典型脚本（010105 调测：查询指定地址段 0）**：
```
DSP POOLUSAGE:IPPOOL="testpool1",IPVERSION=IPV4,WHOLEPOOL=ENABLE,SECTIONNUM=0;
```

> 命令 notes：适用 NF：PGW-U、UPF；查询输出触发 ALM-81216 告警（占用率超门限）。

**步骤位置**：在所属 feature 编排中通常位于"调测/验证阶段"——前置依赖 [0-00043](task/UDG/20.15.2/0-00043.md)（POOLNAME）+ [0-00044](task/UDG/20.15.2/0-00044.md)（SECTION）；用于告警诊断/地址扩容决策。

## 决策点：查询粒度选择（DP 0-00325）

问：查询采用哪种粒度？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单地址段查询 | SECTIONNUM=0 | 定位某个 SECTION 是否占满（010105 调测范式） |
| 整池全段查询 | WHOLEPOOL=ENABLE 不配 SECTIONNUM | 评估 POOL 整体水位（扩容决策） |
| 整池不分段 | 不配 WHOLEPOOL | 概要查询 POOL 总占用 |

## 约束

- **POOLNAME 前置依赖**（critical，rule-0-00484）：IPPOOL 必须先由 [0-00043](task/UDG/20.15.2/0-00043.md) 建好 — 后果：未建引用命令拒收
- **SECTIONNUM 仅当 WHOLEPOOL=ENABLE 时有效**（warning，rule-0-00484）：二者配合使用 — 后果：单独配 SECTIONNUM 取值受 POOL 内段数限制
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
- **查询结果实时刷新**（info，命令 notes）：当前占用情况实时输出，不缓存 — 后果：高频查询不影响后续决策但增加开销
