# 接口应用流策略（ADD QOSAPPLICATION）

> 命令静态知识见 [ADD QOSAPPLICATION](command/UDG/20.15.2/ADD-QOSAPPLICATION.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是 QoS 复杂流分类**生效的最后一步**：把 [0-00203](task/UDG/20.15.2/0-00203.md) MQCPOLICY 已定义的流策略，绑定到具体以太接口的入/出方向上；策略不应用到接口就不生效。

## 配置方法（配置方法字典）

ADD QOSAPPLICATION 是"在接口应用流量策略"命令。**3 参必选**（IFNAME/DIRECTION/POLICYNAME）+ 1 参可选（LAYER）。POLICYNAME 引用 [0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY 已建策略；IFNAME/DIRECTION/LAYER 决定策略作用于哪个接口哪个方向哪一层。1 个特性命中（IPFD-012001 QoS 复杂流分类）覆盖以太主接口 + outbound 范式。

### 配置维度 1：策略名称（参数 POLICYNAME，必选，引用上游）

| 取值 | 来源 | 适用范围 |
|---|---|---|
| **字符串 1~31** | 引用 [0-00203](task/UDG/20.15.2/0-00203.md):POLICYNAME | 必填，已建的流量策略名 |

### 配置维度 2：接口名称（参数 IFNAME，必选）

| 接口类型 | 支持范围 | 典型场景 |
|---|---|---|
| **以太主接口** | 字符串 1~63 | IPFD-012001 范例 `Ethernet64/0/3` |
| **以太子接口** | 字符串 1~63 | 子接口流策略应用 |

### 配置维度 3：报文方向（参数 DIRECTION，必选）

| 取值 | 作用 | 限制 |
|---|---|---|
| **inbound** | 入方向应用流策略 | 不支持非单播协议报文过滤 |
| **outbound** | 出方向应用流策略 | IPFD-012001 范例；不支持 IPv6 报文过滤，IPv4 仅支持 Permit/Deny |

### 配置维度 4：层信息（参数 LAYER，可选）

| 取值 | 默认 | 适用场景 |
|---|---|---|
| **default**（默认） | ✓ | 策略应用在 IP 层（单层场景主推） |
| **link_layer** | — | 策略应用在链路层（复杂分层场景） |

**典型脚本（IPFD-012001 主流范式，主接口 outbound IP 层）**：
```
ADD QOSAPPLICATION:POLICYNAME="policy1", IFNAME="Ethernet64/0/3", DIRECTION=outbound;
```

**典型脚本（入方向链路层应用）**：
```
ADD QOSAPPLICATION:IFNAME="Ethernet66/0/3", DIRECTION=inbound, POLICYNAME="p1", LAYER=link_layer;
```

> 命令 notes：**最大记录数 131072**；**执行后立即生效**；**自动化配置维护期不可下发**——下发前须用 `DSP OPSASSISTSTATE（★R1.5 atom 待补）` 查询，确认设备不在 autoscaling_autoconfig.py 自动化过程中（否则本命令参数会被覆盖，最终生效配置不可预期）。

**步骤位置**：在所属 feature 编排中位于"步骤 N+3：接口应用策略"——前置 [0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY（POLICYNAME 引用源）+ [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE（IFNAME 来源）+ [0-00201](task/UDG/20.15.2/0-00201.md) MQCCLASSIFIER（CLASSIFIER 内流分类）+ [0-00199](task/UDG/20.15.2/0-00199.md) MQCBEHAVIOR（BEHAVIOR 流动作）；下游 [MOD QOSAPPLICATION（★R1.5 atom 待补）](command/UDG/20.15.2/MOD-QOSAPPLICATION.md)、[RMV QOSAPPLICATION（★R1.5 atom 待补）](command/UDG/20.15.2/RMV-QOSAPPLICATION.md)。

## 决策点

（无。本命令在所有特性下用法一致——POLICYNAME 引用上游已建策略 + IFNAME/DIRECTION 决定作用接口与方向，无特性分叉。）

## 约束

- **最大记录数 131072**（warning，rule-0-00204）：系统最大记录数 131072 — 后果：超规格新增失败
- **自动化配置维护期不可下发**（critical，rule-0-00204）：下发本命令前须用 `DSP OPSASSISTSTATE（★R1.5 atom 待补）` 查询自动化配置维护助手（autoscaling_autoconfig.py），确认当前设备不在自动化配置过程中 — 后果：自动化过程中参数会被覆盖，最终生效配置不可预期
- **执行后立即生效**（warning，rule-0-00204）：执行后立即生效 — 后果：误操作即刻生效
- **POLICYNAME 前置依赖**（critical，rule-0-00204）：POLICYNAME 必须先经 [0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY 创建 — 后果：引用不存在的 POLICYNAME 命令拒收
- **IFNAME 前置依赖**（critical，rule-0-00204）：IFNAME 必须先经 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 创建 — 后果：引用不存在的 IFNAME 命令拒收
- **inbound 不支持非单播协议**（warning，rule-0-00204）：inbound 入方向流量策略不支持对非单播协议报文过滤 — 后果：非单播报文过 inbound 策略不生效
- **outbound IPv6 不支持**（warning，rule-0-00204）：outbound 出方向流量策略不支持对 IPv6 报文过滤 — 后果：IPv6 报文过 outbound 策略不生效
- **outbound IPv4 仅支持 Permit/Deny**（warning，rule-0-00204）：IPv4 报文在 outbound 仅支持 Permit/Deny 动作 — 后果：复杂流动作（CAR/重定向等）在 outbound 不生效
- **POLICYNAME 长度**（info，rule-0-00204）：字符串 1~31 — 后果：超长命令拒收
- **IFNAME 类型约束**（info，rule-0-00204）：仅支持以太主接口/子接口 — 后果：其他接口类型命令拒收
