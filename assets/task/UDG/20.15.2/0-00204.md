# 接口应用流策略（ADD QOSAPPLICATION）

> 命令静态知识见 [ADD QOSAPPLICATION](command/UDG/20.15.2/ADD-QOSAPPLICATION.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是 **QoS 复杂流分类生效的最后一步**——把在 [ADD MQCPOLICY](task/UDG/20.15.2/0-00203.md) 上定义好的流策略，应用到具体的以太主接口/子接口的 inbound 或 outbound 方向。策略不应用到接口就不生效。

**配置范式（IPFD-012001 数据规划表归纳）**

| 步骤 | POLICYNAME | IFNAME | DIRECTION | 示例行 |
|---|---|---|---|---|
| 应用流策略 | 引用 0-00203 已建策略名 | 本端规划的以太主接口/子接口 | outbound（出方向，主场景） | `ADD QOSAPPLICATION:POLICYNAME="policy1", IFNAME="Ethernet64/0/3", DIRECTION=outbound;` |

**参数取值约定**

| 参数 | 取值 | 来源 | 说明 |
|---|---|---|---|
| POLICYNAME | 字符串 1~31 | 引用 [ADD MQCPOLICY](task/UDG/20.15.2/0-00203.md):POLICYNAME | 已建的流量策略名 |
| IFNAME | 字符串 1~63，以太主接口/子接口 | 本端规划 | 仅支持以太主接口/子接口 |
| DIRECTION | 枚举（inbound / outbound） | 本端规划 | 报文方向 |
| LAYER | 枚举，默认 default | 本端规划，主场景可省 | **未绑冷门参数**：单层场景默认 default 满足；复杂分层场景需显式配 |

## 约束

- **rule-0-00290**（warning）最大记录数 131072；下发前须先用 `DSP OPSASSISTSTATE` 查询自动化配置维护助手，确认设备不在自动化配置过程中——否则本命令参数会被自动覆盖，最终生效配置不可预期；命令执行后立即生效
- LAYER 不配 → 默认 default 满足主场景，复杂分层场景需显式配
