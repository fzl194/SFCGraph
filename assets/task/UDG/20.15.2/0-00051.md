# 配置GTP路径相关属性（SET UPGTPPATH）

> 命令静态知识见 [SET UPGTPPATH](command/UDG/20.15.2/SET-UPGTPPATH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令全局唯一（最大记录数 1），承载**两类配置**（按一命令一 atom 合并，参数集并集）：

| 子场景 | 参数 | 来源 md |
|---|---|---|
| ① 非 N4 接口路径探测参数（7 参数） | V1DATAECHOSW / ECHOINTERVAL / T3RESPONSE / N3REQUEST / DEACTIVEFLAG / ECHOTIME / LOGICINFTYPE | 配置GTP_PFCP路径管理参数 |
| ② 黑白名单开关（2 参数） | ECHOLISTSWITCH / ECHOLISTTYPE | 配置GTP路径管理黑白名单 |

**探测参数组（场景①，Echo 心跳机制）**

| 参数 | 含义 | 实例值 | 取值范围 |
|---|---|---|---|
| V1DATAECHOSW | V1 数据路径 Echo 开关 | ENABLE | 枚举 |
| ECHOINTERVAL | 发送 GTP 心跳请求间隔 | 60 | 60~3600 秒 |
| T3RESPONSE | Echo 请求重发间隔 | 10 | 1~20 秒 |
| N3REQUEST | Echo 请求最大重发次数 | 5 | 1~6 次 |
| DEACTIVEFLAG | 路径断是否去活上下文 | ENABLE | 枚举 |
| ECHOTIME | 路径断告警后发 Echo 次数 | 20 | 0~60 |
| LOGICINFTYPE（可选） | 限定探测接口类型 | N9c | N3/S1-U/S11-U/N9c/S5-S\S8-S/Pa；不配=所有非 N4/Sxb 接口 |

**告警-去活链路**：Echo 重发达 N3REQUEST 仍无响应 → 上报 ALM-81018 数据路径断 → 启动 ECHOINTERVAL 定时器 → 发送 ECHOTIME 次 Echo → 仍无响应且 DEACTIVEFLAG=ENABLE → 去活该路径上所有上下文。

**黑白名单组（场景②）**

| 参数 | 含义 | 实例值 |
|---|---|---|
| ECHOLISTSWITCH | EchoList 开关 | ENABLE |
| ECHOLISTTYPE | EchoList 类型 | WHITE（默认）/ BLACK |

启用黑白名单（ECHOLISTSWITCH=ENABLE）须先打开 Echo 开关（V1DATAECHOSW=ENABLE）才进行路径探测，且**必须**配置 ADD ECHOIPLIST（0-00054）名单成员。

**典型行**
```
// 场景① 路径探测参数（全量）
SET UPGTPPATH: V1DATAECHOSW=ENABLE, ECHOINTERVAL=60, T3RESPONSE=10, N3REQUEST=5, DEACTIVEFLAG=ENABLE, ECHOTIME=20, LOGICINFTYPE=N9c;
// 场景② 黑白名单（白名单）
SET UPGTPPATH: ECHOLISTSWITCH=ENABLE, ECHOLISTTYPE=WHITE;
// 仅改重发参数（修改Echo消息的重发数据 md）
SET UPGTPPATH: T3RESPONSE=10, N3REQUEST=3;
```

## 决策点：GTP路径管理名单类型（DP 0-00015）

触发条件：SET UPGTPPATH.ECHOLISTSWITCH=ENABLE。问：黑白名单采用哪种类型？（决定 ECHOLISTTYPE 取值并门控是否需配 ADD ECHOIPLIST 成员）

| 选项 | ECHOLISTTYPE | 联动任务 |
|---|---|---|
| 白名单（默认） | WHITE | 仅探测 ADD ECHOIPLIST 列出的对端 IP，**必须**配置成员 |
| 黑名单 | BLACK | 探测除 ADD ECHOIPLIST 外的所有对端，可选配置排除成员 |

## 约束

- **rule-0-00325**（critical）高危命令 + 全局唯一：最大记录数 1；配置重发间隔/最大次数不合理可能导致用户激活失败或资源残留；命令立即生效；启用黑白名单（ECHOLISTSWITCH=ENABLE）需先开 Echo 开关（V1DATAECHOSW=ENABLE）才进行路径探测，且后续**必须**配 ADD ECHOIPLIST（0-00054）名单成员；LOGICINFTYPE 不配时针对所有非 N4/Sxb 接口路径探测 — 后果：高危配置错误导致用户激活失败/资源残留；黑白名单启用但未配 ECHOIPLIST 导致路径探测不到对端
