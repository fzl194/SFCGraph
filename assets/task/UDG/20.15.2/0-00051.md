# 配置GTP路径相关属性（SET UPGTPPATH）

> 命令静态知识见 [SET UPGTPPATH](command/UDG/20.15.2/SET-UPGTPPATH.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**仅作用于非 N4/Sxb 接口**（N4/Sxb 接口由 [SET UPN4UPATH](task/UDG/20.15.2/0-00052.md) 配置）。

## 配置方法（配置方法字典）

SET UPGTPPATH 是**全局唯一**（最大记录数 1）的"GTP 路径探测参数配置"：承载**非 N4/Sxb 接口**的 Echo 心跳探测参数（V1DATAECHOSW/ECHOINTERVAL/T3RESPONSE/N3REQUEST/DEACTIVEFLAG/ECHOTIME/UAC*）、可选探测接口类型（LOGICINFTYPE）、黑白名单开关（ECHOLISTSWITCH/ECHOLISTTYPE）三组。命令核心配置空间有 4 个维度：**路径探测参数**（V1DATAECHOSW + ECHOINTERVAL + T3RESPONSE + N3REQUEST + DEACTIVEFLAG + ECHOTIME）、**探测接口类型**（LOGICINFTYPE 限定探测接口，不配=所有非 N4/Sxb 接口）、**UAC 探测参数**（UACT3RESPONSE/UACN3REQUEST）、**黑白名单**（ECHOLISTSWITCH + ECHOLISTTYPE）。1 个特性命中（GWFD-010102 路径管理）覆盖 7 探测参数全量脚本、黑白名单开关、仅改 T3RESPONSE/N3REQUEST 3 个脚本。

### 配置维度 1：路径探测参数（V1DATAECHOSW/ECHOINTERVAL/T3RESPONSE/N3REQUEST/DEACTIVEFLAG/ECHOTIME）

| 参数 | 必选性 | 取值范围 | 作用 |
|---|---|---|---|
| **V1DATAECHOSW** | 可选 | ENABLE / DISABLE | V1 数据路径 Echo 开关；开启后才进行路径探测 |
| **ECHOINTERVAL** | 可选 | 60~3600 秒 | 发送 GTP 心跳请求间隔 |
| **T3RESPONSE** | 可选 | 1~20 秒 | Echo 请求重发间隔 |
| **N3REQUEST** | 可选 | 1~6 次 | Echo 请求最大重发次数 |
| **DEACTIVEFLAG** | 可选 | ENABLE / DISABLE | 路径断告警后是否去活该路径上已激活的上下文 |
| **ECHOTIME** | DEACTIVEFLAG=ENABLE 时条件必选 | 0~60 | 路径断告警后发送 echo 消息的次数 |

> **告警-去活链路**：Echo 重发达 N3REQUEST 仍无响应 → 上报 ALM-81018 数据路径断 → 启动 ECHOINTERVAL 定时器 → 发送 ECHOTIME 次 Echo → 仍无响应且 DEACTIVEFLAG=ENABLE → 去活该路径上所有上下文。

### 配置维度 2：探测接口类型（参数 LOGICINFTYPE，V1DATAECHOSW=ENABLE 时条件可选）

| 取值 | 作用 |
|---|---|
| **不配** | 针对所有非 N4/Sxb 接口路径探测 |
| **N3** | 仅 N3 接口探测 |
| **S1-U** | 仅 S1-U 接口探测 |
| **S11-U** | 仅 S11-U 接口探测 |
| **N9c** | 仅 N9c 接口探测（010102 范式） |
| **S5-S8-S** | 仅 S5/S8/S 接口探测 |
| **Pa** | 仅 Pa 接口探测 |

### 配置维度 3：UAC 探测参数（参数 UACT3RESPONSE/UACN3REQUEST，可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **UACT3RESPONSE** | 1~20 秒 | UAC GTP 请求消息的重发时间间隔 |
| **UACN3REQUEST** | 1~6 次 | UAC GTP 请求消息的最大尝试发送次数 |

### 配置维度 4：黑白名单（参数 ECHOLISTSWITCH/ECHOLISTTYPE，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **ECHOLISTSWITCH** | ENABLE / DISABLE | EchoList 开关；ENABLE 开启黑白名单功能 |
| **ECHOLISTTYPE** | WHITE / BLACK | EchoList 类型；WHITE 仅探测 [ADD ECHOIPLIST](task/UDG/20.15.2/0-00054.md) 列出的对端 IP；BLACK 探测除 [ADD ECHOIPLIST](task/UDG/20.15.2/0-00054.md) 外的所有对端 |

> 启用黑白名单（ECHOLISTSWITCH=ENABLE）须先开 Echo 开关（V1DATAECHOSW=ENABLE）才进行路径探测，且**必须**配 [0-00054](task/UDG/20.15.2/0-00054.md) 名单成员。

**典型脚本（010102 路径探测参数全量）**：
```
SET UPGTPPATH:V1DATAECHOSW=ENABLE,ECHOINTERVAL=60,T3RESPONSE=10,N3REQUEST=5,DEACTIVEFLAG=ENABLE,ECHOTIME=20,LOGICINFTYPE=N9c;
```

**典型脚本（010102 黑白名单 - 白名单）**：
```
SET UPGTPPATH:ECHOLISTSWITCH=ENABLE,ECHOLISTTYPE=WHITE;
```

**典型脚本（010102 仅改重发参数）**：
```
SET UPGTPPATH:T3RESPONSE=10,N3REQUEST=3;
```

> 命令 notes：适用 NF：SGW-U、PGW-U、UPF；本命令属高危命令（配置重发间隔/最大次数不合理可能导致用户激活失败或资源残留）；执行后立即生效；**全局唯一最大记录数 1**；LOGICINFTYPE 不配时针对所有非 N4/Sxb 接口路径探测。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：GTP 路径探测参数"——前置无；黑白名单启用后必须接 [0-00054](task/UDG/20.15.2/0-00054.md)（ADD ECHOIPLIST 名单成员）。

## 决策点：黑白名单类型（DP 0-00051）

触发条件：SET UPGTPPATH.ECHOLISTSWITCH=ENABLE。问：黑白名单采用哪种类型？

| 选项 | ECHOLISTTYPE | 联动任务 | 适用场景 |
|---|---|---|---|
| 白名单（默认） | WHITE | 必须配 [0-00054](task/UDG/20.15.2/0-00054.md)（ADD ECHOIPLIST） | 仅探测白名单列出的对端 IP（010102 范式） |
| 黑名单 | BLACK | 可选配 [0-00054](task/UDG/20.15.2/0-00054.md)（ADD ECHOIPLIST） | 探测除黑名单外的所有对端（命令支持，010102 未演示） |

## 约束

- **全局唯一最大记录数 1**（critical，rule-0-00451）：本命令全局唯一 — 后果：重复配置命令拒收
- **高危命令**（critical，rule-0-00451）：配置重发间隔/最大次数不合理可能导致用户激活失败或资源残留 — 后果：高危配置错误
- **ECHOTIME 必填条件**（critical，rule-0-00451）：DEACTIVEFLAG=ENABLE 时 ECHOTIME 必填 — 后果：缺 ECHOTIME 命令拒收
- **黑白名单启用前必开 Echo**（critical，rule-0-00451）：ECHOLISTSWITCH=ENABLE 需先 V1DATAECHOSW=ENABLE — 后果：未开 Echo 启用黑白名单命令拒收
- **黑白名单启用后必配 ECHOIPLIST**（critical，rule-0-00451）：白名单必须配 [0-00054](task/UDG/20.15.2/0-00054.md) 成员 — 后果：黑白名单启用但未配 ECHOIPLIST 致路径探测不到对端
- **LOGICINFTYPE 条件可选**（warning，rule-0-00451）：仅 V1DATAECHOSW=ENABLE 时可配 — 后果：未开 Echo 配 LOGICINFTYPE 命令拒收
- **适用 NF 限定**（warning，隐含）：SGW-U、PGW-U、UPF — 后果：其他 NF 不适用
- **仅作用于非 N4/Sxb 接口**（warning，隐含）：N4/Sxb 接口 GTP 路径由 [0-00052](task/UDG/20.15.2/0-00052.md)（SET UPN4UPATH）配置 — 后果：误用本命令配 N4 接口不生效
- **执行后立即生效**（warning，rule-0-00451）：不需重启 — 后果：执行后立即对所有非 N4/Sxb 路径生效
