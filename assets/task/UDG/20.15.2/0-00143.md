# 配置BFD单臂echo会话模板(路由交叉)（ADD AUTOSCALINGBFD）

> 命令静态知识见 [ADD AUTOSCALINGBFD](command/UDG/20.15.2/ADD-AUTOSCALINGBFD.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是路由交叉功能（GWFD-020423）BFD 单臂 echo 检测的会话模板，仅在 DP 0-00024 取 opt-bfd-one-arm 时构建（与双向 BFD [0-00144](task/UDG/20.15.2/0-00144.md) 二选一）。模板名供上行静态路由 [0-00142](task/UDG/20.15.2/0-00142.md) 的 BFDTEMPLATENAME 引用，检测网络 B 网关接口 IP 是否可达（目的地址 = 网关 IP）。单臂 echo 仅需收包间隔，不配双向收/发间隔。

**配置范式（全部固定/本端为主）**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| TEMPLATENAME | 如 `BFD_10` | 本端规划 | 1~59 位字符串，供 SRROUTE.BFDTEMPLATENAME 引用 |
| BFDTYPE | `Static` | 固定 | 单臂 echo 模板固定取 Static |
| SERVICENAME | `ServName_networkB` | 引用 [0-00141](task/UDG/20.15.2/0-00141.md) | 已配置服务名 |
| IPVERSION | `IPv4` | 固定 | 当前路由交叉用 IPv4 |
| DESTADDR4 | 如 `192.168.20.1` | 从对端获取 | 网关接口 IP（参见 5G Core LLD BFD 设计 对端IP地址） |
| MINECHORXINT | 如 `500` | 全网规划（LLD min-rx-interval 毫秒） | 单臂 echo 收包间隔，30~1000ms |
| DETECTMULTI | 如 `4` | 全网规划（LLD detect-multiplier） | 检测倍数，3~50 |
| ONEARMECHO | `TRUE` | 固定 | 单臂 echo 固定 TRUE |

**示例脚本**：
```
ADD AUTOSCALINGBFD: TEMPLATENAME="BFD_10", BFDTYPE=Static, SERVICENAME="ServName_networkB",
  IPVERSION=IPv4, DESTADDR4="192.168.20.1", MINECHORXINT=500, DETECTMULTI=4, ONEARMECHO=TRUE;
```

## 决策点

- **DP-0-00024**（BFD 类型）：选 opt-bfd-one-arm（单臂 echo / BFD Echo / 单臂 BFD）→ 构建本任务；选双向→ 改用 [0-00144](task/UDG/20.15.2/0-00144.md)，本任务不建。
