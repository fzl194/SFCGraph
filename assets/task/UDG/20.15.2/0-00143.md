# 配置BFD单臂echo会话模板(路由交叉)（ADD AUTOSCALINGBFD）

> 命令静态知识见 [ADD AUTOSCALINGBFD](command/UDG/20.15.2/ADD-AUTOSCALINGBFD.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD AUTOSCALINGBFD 用于配置路由交叉功能（GWFD-020423）BFD 单臂 echo 检测的会话模板。**仅在 DP 0-00143 取 opt-bfd-one-arm 时构建**（与双向 BFD 0-00144 二选一）。模板名供上行静态路由 [0-00142](task/UDG/20.15.2/0-00142.md) 的 BFDTEMPLATENAME 引用，**检测网络 B 网关接口 IP 是否可达**（DESTADDR4 = 网关 IP）。**单臂 echo 仅需收包间隔**，不配双向收/发间隔。1 个特性命中（020423 路由交叉）。

### 配置维度 1：BFD 模式（参数 BFDTYPE，必选）

| 取值 | 联动参数 | 适用场景 |
|---|---|---|
| **Static**（固定） | ONEARMECHO=TRUE | 单臂 echo 模板（020423 唯一场景） |

> 单臂 echo 模板固定 BFDTYPE=Static、ONEARMECHO=TRUE。

### 配置维度 2：探测参数（参数 MINECHORXINT + DETECTMULTI）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **MINECHORXINT** | 30~1000 ms | 单臂 echo 收包间隔（参见 5G Core LLD min-rx-interval） |
| **DETECTMULTI** | 3~50 | 检测倍数（参见 LLD detect-multiplier） |

> 收包间隔 30~1000ms；检测倍数 3~50；二者均全网规划。

### 配置维度 3：探测目的（参数 DESTADDR4/IPVERSION）

| IPVERSION | DESTADDR 参数 | 作用 |
|---|---|---|
| **IPv4**（固定） | DESTADDR4 | 网络 B 网关接口 IP（对端，020423 范式） |
| IPv6 | DESTADDR6 | IPv6 网关（020423 当前未用） |

> 当前路由交叉用 IPv4；变更须与对端 UPF 协商（参见 5G Core LLD BFD 设计 对端 IP 地址）。

**典型脚本（020423 单臂 echo IPv4 范式）**：

```
ADD AUTOSCALINGBFD: TEMPLATENAME="BFD_10", BFDTYPE=Static, SERVICENAME="ServName_networkB",
  IPVERSION=IPv4, DESTADDR4="192.168.20.1", MINECHORXINT=500, DETECTMULTI=4, ONEARMECHO=TRUE;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：配 BFD 单臂 echo 模板"——前置 [0-00141](task/UDG/20.15.2/0-00141.md) ADD AUTOSCALINGSERVICE 配 SERVICENAME；后续 [0-00142](task/UDG/20.15.2/0-00142.md) ADD AUTOSCALINGSRROUTE 引用本任务 TEMPLATENAME。

## 决策点：BFD 类型（DP 0-00143）

| 选项 | 联动模板 | 适用场景 |
|---|---|---|
| **opt-bfd-one-arm（单臂 echo / BFD Echo / 单臂 BFD）** | 本任务 | 单臂 echo BFD 探测 |
| **opt-bfd-two-way（双向）** | [0-00144](task/UDG/20.15.2/0-00144.md) ADD AUTOSCALINGSRBFD | 双向 BFD 探测 |

> 单臂/双向二选一：选单臂 → 构建本任务 + SRROUTE 填 BFDTEMPLATENAME；选双向 → 改用 0-00144，本任务不建。

## 约束

- **最大记录数 2048**（warning，rule-0-00143）：系统最多 2048 条 AUTOSCALINGBFD — 后果：超规格新增失败
- **SERVICENAME 前置必备**（critical，rule-0-00143）：前置 SERVICENAME 值相同的 ADD AUTOSCALINGSERVICE(0-00141) 须已配置 — 后果：前置缺失命令拒收
- **DESTADDR4 范围**（warning，rule-0-00143）：DESTADDR4 取值须为合法 IPv4 地址 — 后果：非法地址命令拒收
- **探测参数范围**（warning，rule-0-00143）：MINECHORXINT 30~1000ms / DETECTMULTI 3~50 — 后果：超范围命令拒收
- **自动化配置开关先关闭**（critical，rule-0-00143）：执行前须先 `SET AUTOCONFIG（★R1.5 atom 待补） 关闭`，执行后再开启 — 后果：未关开关命令拒收
- **BFD 全局使能**（critical，rule-0-00143）：使用前须全局使能 BFD 功能 — 后果：BFD 未全局使能命令拒收
- **立即生效**（warning，rule-0-00143）：执行后立即生效 — 后果：误配即刻影响 BFD 探测
