# 主备 UDG + GRE 隧道冗余保护体系

> 静态地址用户路由冗余（GWFD-010107）的**可靠性执行核心**（2-00038 步骤 7-15）。被引用于 [2-00038](task/UDG/20.15.2/2-00038.md)。
> 2026-07-11 从 2-00038 步骤 7-15 平铺中抽取（防平铺，SOP §3.5——9 步阶段内聚 + 可靠性关键阶段 + 未来隧道冗余可复用）。

## 配置方法

本 compound 建"虚拟重定向 IP + GRE 备份隧道 + 主备路由优先级 + 双开关"完整可靠性链路。**主备 UDG 各配一遍**，差异在 GRE 对端 IP / COST / MASTERFLAG / 上行转发。

| 步骤 | 命令 | 关键参数（主用 UDG 基线） |
|---|---|---|
| 1. 虚拟重定向 IP | `ADD REDUNDRDTIP` | IPVERSION=IPv4/IPv6, IPV4ADDRESS=192.168.0.1（主备可相同或不同）|
| 2. LoopBack 接口（GRE 源，绑 vpn2）| `ADD INTERFACE` + `ADD IPBINDVPN` + `ADD IFIPV4ADDRESS` | LoopBack1, IFADMINSTATUS=up；IPBINDVPN VRFNAME=**vpn2**；IFIPADDR=10.11.1.1 主/10.10.0.1 备 |
| 3. Tunnel 接口（GRE 逻辑接口）| `ADD INTERFACE` | Tunnel1, IFMTU=1500 |
| 4. GRE 备份隧道（核心）| `ADD GRETUNNEL` + `ADD IPBINDVPN` + `ADD IFIPV4ADDRESS` | SRCTYPE=if_name, SRCIFNAME=LoopBack1, DSTIPADDR=对端 LoopBack IP, TNLTYPE=gre, DSTVPNNAME=vpn2, **REDUNDANCYEN=TRUE**, **KEEPALVEN=TRUE**；IPBINDVPN VRFNAME=**vpn1** |
| 5. v02 IPv6 GRE 增强（可选）| `SET IFIPV6ENABLE` + `ADD IFIPV6ADDRESS` | Tunnel ENABLEFLAG=TRUE, MTU=1500；IPV6ADDRESS=fc00:2::1/::2 |
| 6. 静态路由（目的=REDUNDRDTIP，出接口=Tunnel）| `ADD SRROUTE` / `ADD SRROUTE6` | PREFIX=192.168.0.1, MASKLENGTH=32, VRFNAME=vpn1, IFNAME=Tunnel |
| 7. 主备 LoopBack 路由互通 | `ADD OSPFINTERFACE` | PROCID, AREAID, IFNAME=LoopBack1/2 |
| 8. 路由优先级（主备区分）| `MOD OSPFIMPORTROUTE` | PROTOCOL=wlr, IMPTCOSTCFG=TRUE, **COST=100 主/200 备** |
| 9. 全局冗余开关 | `SET REDUNDUSER` | SWITCH=ENABLE（与 ADD POOL REDUNDFUNC 双使能）|

**典型脚本（主用 UDG，IPv4）**：
```
ADD REDUNDRDTIP:IPVERSION=IPv4,IPV4ADDRESS="192.168.0.1";
ADD INTERFACE:IFNAME=LoopBack1,IFTYPE=loopback,IFADMINSTATUS=up;
ADD IPBINDVPN:IFNAME=LoopBack1,VRFNAME="vpn2";
ADD IFIPV4ADDRESS:IFNAME=LoopBack1,IFIPADDR="10.11.1.1",SUBNETMASK="255.255.255.255";
ADD INTERFACE:IFNAME=Tunnel1,IFTYPE=tunnel,IFMTU=1500;
ADD GRETUNNEL:TNLNAME="Tunnel1",TNLTYPE=gre,SRCTYPE=if_name,SRCIFNAME=LoopBack1,DSTIPADDR="10.10.0.1",DSTVPNNAME="vpn2",REDUNDANCYEN=TRUE,KEEPALVEN=TRUE;
ADD IPBINDVPN:IFNAME=Tunnel1,VRFNAME="vpn1";
ADD IFIPV4ADDRESS:IFNAME=Tunnel1,IFIPADDR="10.12.1.1",SUBNETMASK="255.255.255.252";
ADD SRROUTE:AFTYPE=ipv4unicast,PREFIX="192.168.0.1",MASKLENGTH=32,VRFNAME="vpn1",IFNAME=Tunnel1;
ADD OSPFINTERFACE:PROCID=1,AREAID=0.0.0.0,IFNAME=LoopBack1;
MOD OSPFIMPORTROUTE:PROCID=1,TOPOID=0,PROTOCOL=wlr,IMPTCOSTCFG=TRUE,COST=100;
SET REDUNDUSER:SWITCH=ENABLE;
```

**步骤位置**：在 [1-00029](task/UDG/20.15.2/1-00029.md) APN 接入域基础设施 + [1-00022](task/UDG/20.15.2/1-00022.md) 地址池体系（POOLTYPE=EXTERNAL+REDUNDFUNC=ENABLE）之后；本 compound 是可靠性执行收尾。

## 场景差异（主备 UDG + IP 版本双向回填）

| 角色 × IP 版本 | GRE 对端 IP | COST | MASTERFLAG（ADD POOL）| SET APNREDUNDUPSW | IPv6 GRE |
|---|---|---|---|---|---|
| 主用 UDG IPv4 | 备 LB IP（10.10.0.1）| **100** | ENABLE（UE 互访场景）/ DISABLE（默认）| **不配** | 不配 |
| 主用 UDG IPv6 | 备 LB IPv6 | 100 | 同上 | 不配 | 步骤 5 配 |
| 备用 UDG IPv4 | 主 LB IP（10.11.1.1）| **200** | DISABLE | **IPV4SWITCH=ENABLE** | 不配 |
| 备用 UDG IPv6 | 主 LB IPv6 | 200 | DISABLE | IPV6SWITCH=ENABLE | 步骤 5 配 |
| v02 双栈（主/备） | 双栈 | 同上 | 同上 | 双栈 | 步骤 5 配 |

> **关键**：主备对称配置——除 GRE 对端 IP / COST / MASTERFLAG / SET APNREDUNDUPSW 外，命令集完全对称；备用 UDG 多步骤 9（SET APNREDUNDUPSW）。

## 决策点

| 选项（角色） | 影响（COST + APNREDUNDUPSW + MASTERFLAG）|
|---|---|
| 主用 UDG | COST=100；MASTERFLAG=ENABLE（UE 互访）/DISABLE（默认）；不配 APNREDUNDUPSW |
| 备用 UDG | COST=200；MASTERFLAG=DISABLE；APNREDUNDUPSW IPV4/V6SWITCH=ENABLE |

> 联动特性级 DP：角色（主/备）由 [2-00038](task/UDG/20.15.2/2-00038.md) 场景 DP 决定。

## 约束

- **双开关使能**（critical）：SET REDUNDUSER（全局）+ ADD POOL REDUNDFUNC（地址池）必须同时 ENABLE — 后果：缺一不生效。
- **LoopBack/Tunnel VPN 划分**（critical）：LoopBack 绑 vpn2、Tunnel 绑 vpn1，不同 VPN 完成物理与业务划分 — 后果：混淆致 GRE 隧道不可用。
- **主备 LoopBack 路由互通**（critical）：必须通过 OSPF/BGP/静态路由打通主备 LoopBack — 后果：GRE 隧道 DOWN。
- **N6 口路由优先级**（critical）：OSPF COST 主=100/备=200（或 BGP MED；或 PDN/DN 侧配）— 后果：路由不能正确选主用 UDG。
- **上行转发仅备 UDG**（critical）：SET APNREDUNDUPSW 仅备用 UDG 配置 — 后果：主备反配置致流量路径错误。
- **重定向 IP 不冲突**（warning）：REDUNDRDTIP 仅内部重定向不对外发布 — 后果：路由冲突。
- **GRE Keepalive 检测**（warning）：KEEPALVEN=TRUE 源端周期发送，对端不可达时本端 Tunnel 置 Down。
- **互斥 NAT / GWFD-010108**（critical）：与 NAT、用户面地址自动检测双向互斥。
- **依赖 IPFD-015002 GRE**（critical）：GRE 隧道依赖 GRE 特性已使能。
- **v02 IPv6 GRE**（warning）：仅 v02 UDG 20.9.0+ 支持 IPv6 GRE 隧道。
