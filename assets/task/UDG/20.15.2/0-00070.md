# 配置QoS CAR功能（SET QOSCAR）

> 命令静态知识见 [SET QOSCAR](command/UDG/20.15.2/SET-QOSCAR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET QOSCAR 配置用户上下行 QoS CAR（Committed Access Rate）的全局开关，按"用户漫游类型 × RAT"组合限速。USERTYPE 必填、ULRAT/DLRAT 按 RAT 维度配置；DP 0-00016 选 opt-qos-car 时本 atom 与 [基于APN的QoS属性](task/UDG/20.15.2/0-00049.md)（SET APNQOSATTR）协同完成 QoS CAR 功能链。

**两种用户场景范式（从证据数据规划表归纳）**

| 场景 | USERTYPE | ULRAT | DLRAT | 典型行 |
|---|---|---|---|---|
| 4G 本地用户限速（GWFD-010201，本地 breakout） | HOME | eutran-1 | eutran-1 | `SET QOSCAR:USERTYPE=HOME,ULRAT=eutran-1,DLRAT=eutran-1;` |
| 漫游用户限速（GWFD-020381，避免异常大话单） | ROAMING | geran-1 | 不配 | `SET QOSCAR:USERTYPE=ROAMING,ULRAT=geran-1;` |

**关键规则**
- **仅对新激活用户生效**：对已在线用户不生效（设置时机策略）。
- **CAR/Shaping 互斥**：CAR 与 shaping 同开时系统优先按 shaping 控制（不可同时启用 SET QOSSHAPE）。
- **系统存在初始记录**：首配本命令为 ADD 而非 SET；ULRAT/DLRAT 在系统初始记录中不生效，后续新增记录才作用于 RAT 匹配。
- **USERTYPE 跨特性分叉**：GWFD-020381=ROAMING、GWFD-010201=HOME，由主特性默认值决定（rule 0-00028 已覆盖会话类 QoS ROAMING 维度基数）。
- **开启会导致性能下降**：容量评估需考虑 PERF 影响。

## 约束

- **rule 0-00285**（capacity）系统最大记录数 3 — 后果：超限 ADD 失败
- **rule 0-00285**（dependency）仅对新激活用户生效；CAR/Shaping 互斥（同开 shaping 优先）；ULRAT/DLRAT 在初始记录不生效；开启导致性能下降 — 后果：已激活用户不变→期望限速不生效；CAR/Shaping 同开→本 atom 配置无效
- 与 [基于APN的QoS属性](task/UDG/20.15.2/0-00049.md) 协作完成 QoS CAR 功能链（DP 0-00016 opt-qos-car）
