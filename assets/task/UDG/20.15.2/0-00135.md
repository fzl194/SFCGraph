# 增加L2TP源端接口(本地配置方式专属)（ADD L2TPCLIENTIP）

> 命令静态知识见 [ADD L2TPCLIENTIP](command/UDG/20.15.2/ADD-L2TPCLIENTIP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**适用 NF：PGW-U、UPF**。

## 配置方法

ADD L2TPCLIENTIP 在已建 L2TP 组上绑定源端 Gi 接口，作为本地配置方式下系统与 LNS 交互时的本地源端。**L2TPGROUPID 必选**（1~1500）、**INTERFACENAME 必选**（1~31 字符）。本命令是**本地配置方式**专属源端接口绑命令——AAA 下发方式下应在 [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR 下用 [0-00134](task/UDG/20.15.2/0-00134.md) ADD L2TPRDSCLIENT 绑源端。核心配置空间有 2 个维度：**L2TP 组号定位**（L2TPGROUPID）、**源端接口定位**（INTERFACENAME）。1 个特性命中（GWFD-020412 L2TP VPN 本地方式）L2TPGROUPID=1 + INTERFACENAME=giif1/0/0。

### 配置维度 1：L2TP 组号定位（参数 L2TPGROUPID，必选）

| 取值 | 作用 |
|---|---|
| **已建 L2TP 组号**（ref [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP） | 源端接口归属的 L2TP 组（020412 本地范式 `1`） |

> L2TPGROUPID 必须已通过 ADD L2TPGROUP 创建；同组下可绑多个源端 Gi 接口。

### 配置维度 2：源端接口定位（参数 INTERFACENAME，必选）

| 取值 | 作用 |
|---|---|
| **已建的源端 Gi 接口名**（ref [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE） | 系统与 LNS 交互时的源端；020412 本地范式 `giif1/0/0` |

> 接口名 1~31 字符；同一 (L2TPGROUPID, INTERFACENAME) 不可重复添加。

**典型脚本（020412 L2TP VPN 本地配置方式，组 1 绑 giif1/0/0）**：
```
ADD L2TPCLIENTIP:L2TPGROUPID=1,INTERFACENAME="giif1/0/0";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：L2TP 组上绑源端接口"——前置 [0-00131](task/UDG/20.15.2/0-00131.md) SET GLOBALL2TP + [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR（含 L2TPGROUPID=1）+ [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP + [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 建源端接口；后续 [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG。

## 决策点：L2TP 源端接口绑定粒度（DP 0-00135）

问：本 L2TP 源端接口是绑在 L2TP 组还是绑在 APN 上？

| 选项 | 命令 | 绑定粒度 | 适用场景 |
|---|---|---|---|
| **L2TP 组绑定**（020412 本地范式） | ADD L2TPCLIENTIP（本 atom） | GROUPID | 本地配置方式，源端由组决定 |
| **APN 绑定** | ADD L2TPRDSCLIENT（[0-00134](task/UDG/20.15.2/0-00134.md)） | APN | AAA 下发方式，源端由 APN 决定 |

> 1 个特性命中（020412 本地范式）；两种命令互斥，不可在同一 L2TP 实例中混用。

## 约束

- **L2TPGROUPID 必须已建**（critical，rule-0-00135）：L2TPGROUPID 必须已通过 `ADD L2TPGROUP` 创建 — 后果：组不存在命令拒收
- **INTERFACENAME 必须已建**（critical，rule-0-00135）：INTERFACENAME 必须已通过 `ADD INTERFACE` 创建 — 后果：接口不存在命令拒收
- **(L2TPGROUPID, INTERFACENAME) 联合唯一**（critical，rule-0-00135）：同一 L2TP 组下同一 Gi 接口不可重复添加 — 后果：重复添加命令拒收
- **L2TPGROUPID 范围**（warning，rule-0-00135）：L2TPGROUPID 取值 1~1500 — 后果：超范围命令拒收
- **适用 NF**（info，rule-0-00135）：本命令仅适用 PGW-U、UPF — 后果：误在 SGW-U 上执行命令拒收
- **与 AAA 方式命令互斥**（warning，rule-0-00135）：同一 L2TP 实例下不应同时使用本命令与 ADD L2TPRDSCLIENT — 后果：源端绑定混乱
- **执行后立即生效**（warning，rule-0-00135）：本命令执行后立即生效 — 后果：误配即刻影响该 L2TP 组下用户源端连接
