# 绑定L2TP组到源端接口(本地配置方式专属)（ADD L2TPCLIENTIP）

> 命令静态知识见 [ADD L2TPCLIENTIP](command/UDG/20.15.2/ADD-L2TPCLIENTIP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令用于 **本地配置方式激活 L2TP VPN** 场景（GWFD-020412）：把已建 L2TP 组绑到指定源端 Gi 接口，L2TP 用户入网时系统以此接口作源端与 LNS 交互。是 GWFD-020412-本地流程步骤 8。AAA 下发方式用 [0-00134](task/UDG/20.15.2/0-00134.md) 绑 APN，不用本命令。

**配置范式**

| 场景 | 参数 | 典型行 |
|---|---|---|
| 本地配置方式绑源端接口（GWFD-020412） | L2TPGROUPID + INTERFACENAME（均引用） | `ADD L2TPCLIENTIP:L2TPGROUPID=1,INTERFACENAME="giif1/0/0";` |

**参数取值约定**

| 参数 | 取值来源 | 说明 |
|---|---|---|
| L2TPGROUPID | 引用 [0-00133](task/UDG/20.15.2/0-00133.md) 已建 L2TP 组号（整数 1~1500） | 每 L2TP 组只能绑 1 源端接口（GROUPID 唯一约束） |
| INTERFACENAME | 引用 ADD LOGICINF.NAME 的 Giif 接口名（字符串 1~31） | 源端 Gi 接口，须先 [[UDG@20.15.2@Task@0-00136]] 建好 |

两个参数均 `reference` 绑定，取值来自已配置数据，本命令本身无本端规划值。

## 约束

- **rule-0-00369**（warning）系统最多 1500 条；每 L2TP 组唯一绑 1 源端接口；立即生效；前置须先 ADD L2TPGROUP + ADD LOGICINF；适用 PGW-U/UPF — 后果：超限或重复绑定新增失败、缺前置则无引用源
