# IPv6承载上下文（GWFD-020401）

> 特性静态知识见 [GWFD-020401](feature/UDG/20.15.2/GWFD-020401.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-020401-IPv6承载上下文.md)。
>
> **本页定位**：配置生成实例化视角——GWFD-020401 IPv6 承载上下文的完整配置体系。**承载基础设施层 + IPv6 下行路由发布层** 双层骨架，按 9 步激活流程组织。

## 配置概览

本特性按 **2 层架构 × 9 步激活流程** 组织配置：

**对象链**：
```
Layer 1: 基础设施层（★本特性核心）
┌────────────────────────────────────────────────────────┐
│ SET LICENSESWITCH（LKV3G5V6PB01）                       │
│   ↓                                                     │
│ ADD VPNINST（逻辑口） + ADD L3VPNINST（外联口）         │
│   ↓                                                     │
│ ADD VPNINSTAF AFTYPE=ipv6uni（★IPv6 地址族激活）         │
└────────────────────────────────────────────────────────┘
                          ↓
Layer 2: 路由策略层（控制 WLR 路由发布过滤）
┌────────────────────────────────────────────────────────┐
│ ADD ROUTEPOLICY（policy1）                              │
│   ↓                                                     │
│ ADD ROUTEPOLICYNODE（permit, NODESEQUENCE=10）          │
│   ↓                                                     │
│ ADD MATCHROUTETYPE（nssaExternal1）                     │
└────────────────────────────────────────────────────────┘
                          ↓
Layer 3: IPv6 下行路由发布层（OSPFv3 + WLR）
┌────────────────────────────────────────────────────────┐
│ ADD OSPFV3（PROCID=100, ROUTERID, VRFNAME）            │
│   ↓                                                     │
│ ADD OSPFV3AREA（AREAID=0.0.0.1）                        │
│   ↓                                                     │
│ ADD OSPFV3IMPORTROUTE PROTOCOL=wlr（★WLR 路由发布）     │
└────────────────────────────────────────────────────────┘
```

**9 步通用骨架**（IPv6 承载上下文激活流程全 9 步）：

| 步骤 | 干啥 | 主要命令 | 骨架归属 |
|---|---|---|---|
| 1. License 开关 | 打开 LKV3G5V6PB01 配置开关 | `SET LICENSESWITCH` | [1-00027](task/UDG/20.15.2/1-00027.md) |
| 2. 逻辑口 VPN | 建逻辑口 VPN 实例 | `ADD VPNINST` | [1-00027](task/UDG/20.15.2/1-00027.md) |
| 3. 外联口 VPN | 建外联口 L3VPN + IPv6 地址族 | `ADD L3VPNINST` + `ADD VPNINSTAF` | [1-00027](task/UDG/20.15.2/1-00027.md) |
| 4. 路由策略 | 建 ROUTEPOLICY | `ADD ROUTEPOLICY` | 直接 atom |
| 5. 路由策略节点 | 建 ROUTEPOLICYNODE | `ADD ROUTEPOLICYNODE` | 直接 atom |
| 6. 路由类型匹配 | 建 MATCHROUTETYPE | `ADD MATCHROUTETYPE` | 直接 atom |
| 7. OSPFv3 进程 | 建 OSPFv3 进程（IPv6） | `ADD OSPFV3` | [1-00026](task/UDG/20.15.2/1-00026.md) |
| 8. OSPFv3 区域 | 建 OSPFv3 区域 | `ADD OSPFV3AREA` | [1-00026](task/UDG/20.15.2/1-00026.md) |
| 9. OSPFv3 引入 WLR | 引入 WLR 路由发布到骨干网 | `ADD OSPFV3IMPORTROUTE PROTOCOL=wlr` | [1-00026](task/UDG/20.15.2/1-00026.md) |

**关键约束**：
- **必须加载 License LKV3G5V6PB01**（控制项编号 82209828）并打开 SET LICENSESWITCH
- 逻辑口 VPN 与外联口 VPN 必须同名（文档操作步骤2"说明"原文警示）
- 外联口 VPN 必须配 AFTYPE=ipv6uni IPv6 地址族
- OSPFv3 ROUTERID 全网唯一
- **WLR 路由发布必须用 wlr，不能只选 wlr_sp/wlr_ud**（文档操作步骤9"说明"原文强制约束）
- 本特性激活脚本**不含 ADD POOL/SECTION/POOLGRPMAP/APN**——地址分配由 GWFD-010105 承载
- 本特性激活脚本**不含 OSPF（IPv4 侧）**——纯 IPv6 路由发布（OSPFv3）

## 配置流程

> 每步显示"干啥 + 含哪些命令"。可按 IPv6 单栈/双栈/PD 联合部署选择不同参数变种。

1. **License 开关**（LKV3G5V6PB01）：
   - `SET LICENSESWITCH` → [0-00019](task/UDG/20.15.2/0-00019.md)（LICITEM="LKV3G5V6PB01", SWITCH=ENABLE）
   - 复用 → [1-00027](task/UDG/20.15.2/1-00027.md) IPv6 承载基础设施

2. **配置逻辑口 VPN 实例**（**必须与外联口 VPN 同名**；接入域基础设施 IPv6 变种）：
   - → 复用 [1-00029](task/UDG/20.15.2/1-00029.md) APN 接入域基础设施（**LKV3G5V6PB01 License 前置 + AFTYPE=ipv6uni IPv6 变种**；与步骤 3 [1-00027](task/UDG/20.15.2/1-00027.md) IPv6 承载基础设施协同——1-00029 承担 VPN 实例族，1-00027 承担 IPv6 地址族 + 外联口 L3VPN）

3. **配置外联口 VPN + IPv6 地址族**（外联口 L3VPN + AFTYPE=ipv6uni）：
   - 复用 → [1-00027](task/UDG/20.15.2/1-00027.md) IPv6 承载基础设施（外联口 L3VPN + IPv6 地址族段；与步骤 2 [1-00029](task/UDG/20.15.2/1-00029.md) 接入域基础设施协同）

4. **配置路由策略**（policy1）：
   - `ADD ROUTEPOLICY` → [0-00111](task/UDG/20.15.2/0-00111.md)（POLICYNAME="policy1"）

5. **添加路由策略节点**（permit, 节点号 10）：
   - `ADD ROUTEPOLICYNODE` → [0-00112](task/UDG/20.15.2/0-00112.md)（POLICYNAME="policy1", NODESEQUENCE=10, MATCHMODE=permit）

6. **添加匹配路由类型**（nssaExternal1）：
   - `ADD MATCHROUTETYPE` → [0-00113](task/UDG/20.15.2/0-00113.md)（POLICYNAME="policy1", NODESEQUENCE=10, ROUTETYPE=nssaExternal1）

7. **启动 OSPFv3 进程**（PROCID=100, ROUTERID, VRFNAME）：
   - `ADD OSPFV3` → [0-00078](task/UDG/20.15.2/0-00078.md)（PROCID=100, ROUTERID="10.10.10.1", VRFNAME="VPN_Internet"）
   - 复用 → [1-00026](task/UDG/20.15.2/1-00026.md) IPv6 下行路由发布

8. **增加 OSPFv3 区域**（AREAID=0.0.0.1）：
   - `ADD OSPFV3AREA` → [0-00079](task/UDG/20.15.2/0-00079.md)（PROCID=100, AREAID="0.0.0.1"）
   - 复用 → [1-00026](task/UDG/20.15.2/1-00026.md) IPv6 下行路由发布

9. **引入 WLR 路由**（PROTOCOL=wlr, ROUPOOLINAME 引用步骤4-6 路由策略）：
   - `ADD OSPFV3IMPORTROUTE` → [0-00080](task/UDG/20.15.2/0-00080.md)（PROCID=100, TOPOID=0, **PROTOCOL=wlr**, ROUPOLINAME="policy1"）
   - 复用 → [1-00026](task/UDG/20.15.2/1-00026.md) IPv6 下行路由发布
   - **★ 关键约束**：必须选 wlr 方式发布所有路由，不能只选 wlr_sp 或 wlr_ud（文档强制约束）

## 决策点

**DP（特性级）：IPv6 承载场景 × 联动特性 × 走法（★一张表覆盖全场景）**

| 场景 | 联动特性 | 走法（compound/atom 编排） | 关键参数差异 |
|---|---|---|---|
| IPv6 单栈承载（仅 020401） | 无 | [1-00027](task/UDG/20.15.2/1-00027.md)（License+VPN+AFTYPE=ipv6uni）+ 直接 atom 步骤 4-6（路由策略 3 步）+ [1-00026](task/UDG/20.15.2/1-00026.md)（OSPFv3+WLR，3 步） | AFTYPE=ipv6uni，ROUTERID=10.10.10.1 |
| IPv6 + IPv6 PD 联合部署 | GWFD-020406（PD 前缀代理） | [1-00027](task/UDG/20.15.2/1-00027.md) + [1-00026](task/UDG/20.15.2/1-00026.md)（**020406 共网段范式：BFDALLINTFFLG=TRUE + VPNINSCAPSIMFLG=TRUE + VIRTUALSYSFLAG=TRUE + ADD OSPFV3INTERFACE 接口级 VIRTUALSYSFLAG 必配**） | AFTYPE=ipv6uni + 020406 PD 专用地址池（< 64 位前缀） |
| IPv4v6 双栈 | GWFD-020403（双栈） | [1-00027](task/UDG/20.15.2/1-00027.md) + [1-00026](task/UDG/20.15.2/1-00026.md) + GWFD-010105 地址池（双 SECTION/IPV4+IPV6） | AFTYPE=ipv6uni + IPv4 侧 OSPF（[1-00024](task/UDG/20.15.2/1-00024.md)） |
| 与 020402（N6/Gi/SGi 接口 IPv6 组网）联动 | GWFD-020402（020401 文档显式依赖） | [1-00027](task/UDG/20.15.2/1-00027.md) + [1-00026](task/UDG/20.15.2/1-00026.md) + 020402 配套配置 | 020402 负责 N6/Gi/SGi 接口 IPv6 组网，020401 负责承载建立 |
| 仅 IPv4 单栈（不涉及 020401） | 无 | 走 GWFD-010105 用户面地址分配，不激活 020401 | — |

> **关键规则**：
> - **必须首先 SET LICENSESWITCH**（步骤 1），否则特性不工作
> - **逻辑口 VPN 与外联口 VPN 必须同名**（步骤 2/3 约束）
> - **AFTYPE=ipv6uni 必配**（步骤 3），否则 IPv6 承载无法工作
> - **WLR 路由必须用 wlr**（步骤 9），不能只选 wlr_sp/wlr_ud
> - **ROUTERID 全网唯一**（步骤 7）
> - **020406 联合部署**须在 020401 基础上叠加 PD，**020401 是 020406 的强制前置**（020406 文档"与其他特性的交互"章节显式声明）
> - **020403 联合部署**的 IPv6 地址族**由本骨架承担**（020403 激活脚本不含 ADD VPNINSTAF）

### 决策点：OSPFv3 路由策略匹配（DP 0-00341）

问：ROUTEPOLICY 的 MATCHROUTETYPE 选哪个路由类型？

| 选项 | ROUTETYPE | 适用场景 | 备注 |
|---|---|---|---|
| NSSA 外部类型1 | `nssaExternal1` | **020401 范式典型**（IPv6 用户路由在 NSSA 区域外部类型1） | 文档数据规划表样例 |
| OSPF 外部类型1 | `external1` | 常规 OSPF 区域外部路由 | 需根据组网规划调整 |
| OSPF 外部类型2 | `external2` | OSPF 区域外部类型2 | 需根据组网规划调整 |

> **关键规则**：MATCHROUTETYPE 与实际 OSPFv3 区域类型必须匹配——020401 范式 NSSA 区域典型配 nssaExternal1；常规骨干区域配 external1/external2。

## 约束

- **License 必需**（critical，rule-0-01602）：必须加载 License 控制项 82209828 LKV3G5V6PB01 并通过 SET LICENSESWITCH 打开开关 — 后果：特性不可用
- **逻辑口 vs 外联口 VPN 一致**（critical，rule-0-01602）：逻辑接口的 VPN 一定要与其对应的外联口的 VPN 相同（文档操作步骤2"说明"原文） — 后果：逻辑接口报文查询路由失败
- **AFTYPE=ipv6uni 必配**（critical，rule-0-01602）：外联口 VPN 必须配置 AFTYPE=ipv6uni IPv6 地址族 — 后果：IPv6 承载无法工作
- **WLR 路由发布完整性**（critical，rule-0-01602）：采用路由策略方式发布路由时，**不能只选 WLR_SP 或 WLR_UD**，必须选 WLR 方式发布所有路由（文档操作步骤9"说明"原文强制约束） — 后果：下行路由只发布部分用户
- **ROUTER ID 全网唯一**（critical，rule-0-01602）：OSPFv3 ROUTERID 必须全网唯一 — 后果：OSPFv3 邻接建立失败
- **ROUTERID 必配**（critical，rule-0-01602）：OSPFv3 进程下未配 ROUTERID 则无法生成 LSA — 后果：进程能建但无 LSA
- **VRFNAME 与外联口 VPN 一致**（critical，rule-0-01602）：OSPFv3 进程的 VRFNAME 与外联口 L3VPNINST 的 VRFNAME 必须一致 — 后果：VPN 不一致致下行路由不在目标 VRF 中
- **依赖 GWFD-010105**（critical，rule-0-01602）：本特性激活脚本不含地址池（ADD POOL/SECTION/POOLGRPMAP）——必须先完成 GWFD-010105 配置（IPv6 地址池） — 后果：地址池缺失致用户无法分配 IPv6 地址
- **依赖 GWFD-020402**（critical，rule-0-01602）：在 N6/Gi/SGi 接口具备支持 IPv6 的能力时，需要使用 N6/Gi/SGi 接口 IPv6 组网（文档显式声明） — 后果：接口不支持 IPv6 致承载无法建立
- **被 GWFD-020406 显式依赖**（critical，rule-0-01602）：GWFD-020406 IPv6 Prefix Delegation 文档"与其他特性的交互"章节明确列出依赖本特性（控制项 82209828） — 后果：未激活 020401 致 020406 PD 路由发布基础设施缺失
- **进程 VRFNAME 不可更改**（critical，rule-0-01602）：OSPFv3 进程实例 VRFNAME 仅第一次使能时指定，后续不可修改 — 后果：错填 VRFNAME 必须删除进程并重建
- **IPv6 地址族未激活**（warning，rule-0-01602）：未配 ADD VPNINSTAF AFTYPE=ipv6uni — 后果：IPv6 承载无法工作（LST VPNINSTAF 确认 ipv6uni 地址族存在）
- **MATCHROUTETYPE 与区域类型匹配**（warning，rule-0-01602）：MATCHROUTETYPE 与实际 OSPFv3 区域类型必须匹配 — 后果：路由策略匹配失败
- **UDG 起始版本 20.0.0**（warning）：本特性首次发布版本为 UDG 20.0.0 — 后果：低版本网元不支持
- **对系统的影响**（info）：随 IPv6 承载数增加，系统资源占用增大，CPU 占用率上升 — 后果：话务模型压力
