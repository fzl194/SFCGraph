# 增加以太ACL规则（ADD ACLRULEETH）

> 命令静态知识见 [ADD ACLRULEETH](command/UDG/20.15.2/ADD-ACLRULEETH.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。以太 ACL 是 ACL 家族二层形态——基于以太帧字段（源/目的 MAC + VLAN ID + 帧协议号）匹配，**不涉及 IP 层**；ACLNAME 引用 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP 以太范围段（4000~4999）；IP 层匹配改用 [0-00198](task/UDG/20.15.2/0-00198.md) ADD ACLRULEADV4 或 [0-00207](task/UDG/20.15.2/0-00207.md) ADD ACLRULEBAS4。

## 配置方法（配置方法字典）

ADD ACLRULEETH 是"增加以太 ACL 规则"命令。**3 参必选**（ACLNAME/ACLRULENAME/ACLACTION）+ 8 参可选（ACLRULEID/ACLFRAMTYPE/ACLFRAMMASK/ACLSRCMAC/ACLSRCMACMASK/ACLDESTMAC/ACLDESTMACMASK/ACLVLANID/ACLVLANIDMASK/ACLRULEDESCRIPTION）。配置空间有 3 个维度：**规则组标识**（ACLNAME 以太段）、**规则行为**（ACLACTION Permit/Deny）、**二层匹配维度**（源 MAC/目的 MAC/VLAN ID/帧协议号，可叠加）。1 个特性命中（IPFD-012002 ACL）覆盖 Deny+精确源 MAC 范式。

### 配置维度 1：规则组标识（参数 ACLNAME，必选，引用上游）

| 取值形式 | 数据来源 | 适用场景 |
|---|---|---|
| **数字字符串 4000~4999** | 引用 [0-00197](task/UDG/20.15.2/0-00197.md):ACLNAME | IPFD-012002 范例 `4000`（以太段） |
| **字符串 1~32** | 引用 [0-00197](task/UDG/20.15.2/0-00197.md):ACLNAME | 命名 ACL |

> **范围段铁律**：ACLNAME 数字串必须在 4000~4999 范围段（以太 ACL）；跨段数字串（如 3001）命令拒收。

### 配置维度 2：规则行为（参数 ACLACTION，必选）

| 取值 | 作用 |
|---|---|
| **Permit** | 允许匹配报文通过 |
| **Deny** | 拒绝匹配报文 |

> IPFD-012002 范例 = Deny（拒绝源 MAC=00E0-FC12-3456 报文）。

### 配置维度 3：二层匹配维度（4 套可选参数，可叠加）

| 匹配维度 | 必选参数 | 配套掩码 | 典型场景 |
|---|---|---|---|
| **源 MAC** | ACLSRCMAC（H-H-H，1~18 字符） | ACLSRCMACMASK（默认 FFFF-FFFF-FFFF） | IPFD-012002 范例 Deny 00E0-FC12-3456 |
| **目的 MAC** | ACLDESTMAC（H-H-H，1~18 字符） | ACLDESTMACMASK（默认 FFFF-FFFF-FFFF） | 目的 MAC 过滤 |
| **VLAN ID** | ACLVLANID（1~4094） | ACLVLANIDMASK（0x0~0xFFF，默认 0xFFF） | VLAN 域内二层 ACL |
| **帧协议号** | ACLFRAMTYPE（0~0xFFFF 十六进制） | ACLFRAMMASK（默认 0xFFFF） | 以太帧协议类型过滤 |

> **掩码联动规则**：ACLSRCMAC/ACLDESTMAC 配置时对应 MASK 默认 FFFF-FFFF-FFFF（精确匹配）；ACLVLANID 配置时 ACLVLANIDMASK 默认 0xFFF；ACLFRAMTYPE 配置时 ACLFRAMMASK 默认 0xFFFF；不配该字段则对应 MASK 无默认值。

**典型脚本（IPFD-012002 以太组 4000，精确 Deny 源 MAC）**：
```
ADD ACLRULEETH:ACLNAME="4000", ACLRULENAME="rule_5", ACLACTION=Deny, ACLSRCMAC="00E0-FC12-3456";
```

> 命令 notes：**最大记录数 65535**（整机 ACL 规则）；**单组最多 16384 条规则**；**执行后立即生效**；**前置必须先 ADD ACLGROUP**；ACLRULEID（规则 ID，0~4294967294，未配时按规则组步长增长）；ACLRULEDESCRIPTION（字符串 1~127，支持空格）；**MAC 格式 H-H-H**：H 为 4 位十六进制数（可输入 1~4 位）。

**步骤位置**：在所属 feature 编排中位于"步骤 N+1：建以太 ACL 规则"——前置 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP（建以太组）；下游 [MOD ACLRULEETH（★R1.5 atom 待补）](command/UDG/20.15.2/MOD-ACLRULEETH.md)（修改规则）、[RMV ACLRULEETH（★R1.5 atom 待补）](command/UDG/20.15.2/RMV-ACLRULEETH.md)（删除）。

## 决策点

（无。本命令在所有特性下用法一致——基于二层帧字段匹配，行为 Permit/Deny 二选一，无特性分叉。）

## 约束

- **最大记录数 65535**（critical，rule-0-00208）：整机 ACL 规则最大 65535 条 — 后果：超规格新增失败
- **单组最多 16384 条规则**（critical，rule-0-00208）：单个 ACL 规则组下最多 16384 条 — 后果：超限新增失败
- **前置必须先 ADD ACLGROUP**（critical，rule-0-00208）：ACLNAME 引用的规则组必须先建 — 后果：未建组新增失败
- **执行后立即生效**（warning，rule-0-00208）：执行后立即生效 — 后果：误操作即刻生效
- **ACLNAME 范围段约束**（critical，rule-0-00208）：数字 ACLNAME 必须在 4000~4999 以太范围段 — 后果：跨段数字命令拒收
- **最低规则号生效原则**（info，rule-0-00208）：某类数据流能与多条规则匹配时，实际生效的是规则号最低的规则 — 后果：规则号顺序决定优先级
- **ACLRULENAME 命名约束**（info，rule-0-00208）：字符串 1~32（不支持空格，区分大小写） — 后果：超长命令拒收
- **MAC 地址格式**（warning，rule-0-00208）：H-H-H 格式，H 为 4 位十六进制（可输入 1~4 位） — 后果：格式错误命令拒收
- **VLAN ID 范围**（info，rule-0-00208）：1~4094 — 后果：超范围命令拒收
- **帧协议号格式**（info，rule-0-00208）：0~0xFFFF（十六进制整数） — 后果：格式错误或超范围命令拒收
- **ACLRULEDESCRIPTION 长度**（info，rule-0-00208）：字符串 1~127（支持空格，区分大小写） — 后果：超长命令拒收
