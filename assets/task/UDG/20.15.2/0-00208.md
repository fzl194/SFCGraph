# 增加以太/接口ACL规则(VNRS)（ADD ACLRULEETH）

> 命令静态知识见 [ADD ACLRULEETH](command/UDG/20.15.2/ADD-ACLRULEETH.md)（以太规则）、[ADD ACLRULEIF](command/UDG/20.15.2/ADD-ACLRULEIF.md)（接口规则）。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本 atom 是**聚合 atom**：把两条互斥命令合并为一条 task——`ADD ACLRULEETH`（以太 ACL，范围段 4000-4999，按源 MAC 匹配）与 `ADD ACLRULEIF`（接口 ACL，范围段 1000-1999，按接口名匹配）。两条命令作用域互斥，**按 ACLNAME 范围段选用其一**。ACLNAME 引用 [ADD ACLGROUP](task/UDG/20.15.2/0-00197.md)。

> 聚合债：CONSOLIDATION-BACKLOG.md §B/E.2 已标记应拆为 `ADD ACLRULEETH` / `ADD ACLRULEIF` 两个独立 atom，本 pass 不拆。

**配置范式（按范围段分支，IPFD-012002 数据规划表归纳）**

| 范围段 | 选用命令 | 条件必填匹配参数 | 示例行（以太 ACL） |
|---|---|---|---|
| 4000-4999（以太 ACL） | ADD ACLRULEETH | ACLSRCMAC（源 MAC，条件必填） | `ADD ACLRULEETH:ACLNAME="<组>", ACLRULENAME="<规则名>", ACLACTION=Permit, ACLSRCMAC="00e0-fc00-0001";` |
| 1000-1999（接口 ACL） | ADD ACLRULEIF | IFNAME（接口名，条件必填） | `ADD ACLRULEIF:ACLNAME="<组>", ACLRULENAME="<规则名>", ACLACTION=Permit, IFNAME="Ethernet64/0/3";` |

**参数取值约定**

| 参数 | 取值 | 来源 | 说明 |
|---|---|---|---|
| ACLNAME | 字符串 1~32，以太段 4000-4999 / 接口段 1000-1999 | 引用 [ADD ACLGROUP](task/UDG/20.15.2/0-00197.md):ACLNAME | 规则组标识（决定走哪条命令） |
| ACLRULENAME | 字符串 1~32 | 本端规划 | 规则名称 |
| ACLACTION | Permit / Deny | 本端规划 | 规则行为 |
| ACLSRCMAC | H-H-H 格式（如 00e0-fc00-0001） | 本端规划 | **以太 ACL 条件必填**（IFNAME 时不填） |
| IFNAME | 字符串，接口名 | 本端规划 | **接口 ACL 条件必填**（ACLSRCMAC 时不填） |

**未绑冷门参数**（命令真相有但特性脚本未用）：ACLRULEID / ACLFRAMTYPE / ACLFRAMMASK / ACLSRCMACMASK / ACLDESTMAC / ACLDESTMACMASK / ACLVLANID / ACLVLANIDMASK / ACLRULEDESCRIPTION——按需（协议号/VLAN/目的 MAC 等垂直场景）时查命令 wiki。

## 约束

- **rule-0-00271**（warning）聚合 atom 互斥：ACLSRCMAC（以太 ACL，4000-4999）与 IFNAME（接口 ACL，1000-1999）互斥，按 ACLNAME 范围段选用其一——后果：同一规则组混用以太/接口匹配会导致规则语义不清或被拒收
- 最大记录数 65535；单组最多 16384 条规则；前置必须先 `ADD ACLGROUP`——后果：超限/无组→新增失败
