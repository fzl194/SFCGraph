# 配置并组装IPsec安全策略（ADD IPSECPOLICY）

> 命令静态知识见 [ADD IPSECPOLICY](command/UDG/20.15.2/ADD-IPSECPOLICY.md)、[ADD PROPATTACHIPSECPROPOSAL](command/UDG/20.15.2/ADD-PROPATTACHIPSECPROPOSAL.md)、[ADD ATTACHIKEPEER](command/UDG/20.15.2/ADD-ATTACHIKEPEER.md)。本页只讲配置生成实例化时这三条命令**怎么配**。

## 配置方法

聚合 atom：先 [ADD IPSECPOLICY] 建 IPsec 安全策略（引用 ACL），再用 [ADD PROPATTACHIPSECPROPOSAL] 绑定 IPsec 安全提议，最后用 [ADD ATTACHIKEPEER] 绑定 IKE 对等体。多 Sequence 场景（主备隧道/多感兴趣流）按 SEQUENCENUMBER 重复该三步组装。

**数据规划取值样例**（证据③，IPFD-015004 各激活 md 数据规划表）：

| 参数 | 取值样例 | 获取方法 | 所属命令 | 说明 |
|---|---|---|---|---|
| POLICYNAME（策略名称） | `policy1`、`map1` | 本端规划 | ADD IPSECPOLICY | 策略容器标识 |
| SEQUENCENUMBER（序列号） | `1`、`10`、`20`；多 Sequence 场景 `10`+`20` | 对端协商 | ADD IPSECPOLICY | 1~10000，多 Sequence 各一条 |
| POLICYMODE（策略模式） | `Isakmp`（ISAKMP 模式） | 对端协商 | ADD IPSECPOLICY | 用 IKE 建 SA |
| TEMPLATEMODE（模板模式） | `None` | 对端协商 | ADD IPSECPOLICY | 主场景无模板 |
| ACLNUMBER（ACL 编号） | `3000`、`3001`、`3017` | 引用（[0-00211](0-00211.md)） | ADD IPSECPOLICY | 引用 ACLGROUPIPSEC 编号；若上游 ACLNAME 填名称则此处用 ACLNAME 参数 |
| IPSECPROPNAME（IPsec 提议名称） | `tran1`、`Proposal1` | 引用（[0-00212](0-00212.md)） | ADD PROPATTACHIPSECPROPOSAL | 引用 IPsec 安全提议 |
| IKEPEERNAME（IKE 对等体名称） | `a`、`b`、`1` | 引用（[0-00214](0-00214.md)） | ADD ATTACHIKEPEER | 引用 IKE 对等体 |
| PEERPRIORITY（对等体优先级） | 主备隧道场景必填 | 本端规划 | ADD ATTACHIKEPEER | 多 Peer 主备区分，仅 016004 主备场景显式 |

> WORKMODE（工作模式，主备场景 `Master_standby`）/AUTOSWITCHBACK（自动切回，主备场景 `Disable`）仅在 016004 主备隧道场景显式；22 个进阶参数（SA 计长/PFS/DSCP/抗重放/限速/日志/ESN/TFC 等）主场景冷门不绑。10009 场景显式带全套 SA 计长/PFS/DSCP/抗重放等（取缺省值），属完整示例非主流。

**典型范式**（证据③任务示例脚本，普通 IPv4 IPsec 隧道，三步组装）：
```
ADD IPSECPOLICY: POLICYNAME="map1", SEQUENCENUMBER=10, POLICYMODE=Isakmp, TEMPLATEMODE=None, ACLNUMBER=3000;
ADD PROPATTACHIPSECPROPOSAL: POLICYNAME="map1", SEQUENCENUMBER=10, IPSECPROPNAME="tran1";
ADD ATTACHIKEPEER: POLICYNAME="map1", SEQUENCENUMBER=10, IKEPEERNAME="b";
```

**典型范式**（多 Sequence IPsec 策略场景，按序列号重复组装）：
```
ADD IPSECPOLICY: POLICYNAME="map1", SEQUENCENUMBER=10, ..., ACLNUMBER=3000;
ADD IPSECPOLICY: POLICYNAME="map1", SEQUENCENUMBER=20, ..., ACLNUMBER=3001;
# 每个序列号各绑一次提议 + 对等体
```

> ACL 引用规则：上游 [0-00211](0-00211.md) 的 ACLNAME 若填**编号**则此处用 ACLNUMBER；若填**名称**则此处用 ACLNAME 参数（IPSECPOLICY 真相表 conditional）。

## 约束

- **rule-0-00339**（critical）命令立即生效；影响协商及加解密性能，部分配置可能不安全；SEQUENCENUMBER 取值 1~10000；DH 组 dh_group1/2/5 不推荐；ACLNUMBER 引用前置 [0-00211](0-00211.md):ACLNAME；IPSECPROPNAME 引用前置 [0-00212](0-00212.md):PROPOSALNAME；IKEPEERNAME 引用前置 [0-00214](0-00214.md):PEERNAME；多 Sequence 场景按 SEQUENCENUMBER 重复组装；多 Peer 场景配 PEERPRIORITY。
