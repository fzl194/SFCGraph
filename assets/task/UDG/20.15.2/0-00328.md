# 查询地址池（LST POOL）

> 命令静态知识见 [LST POOL](command/UDG/20.15.2/LST-POOL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

LST POOL 是地址池族**配置查询**命令，输出 POOLNAME / POOLTYPE / HASVPN / VPNINSTANCE 等配置字段。命令核心配置空间有 3 个维度：**查询范围**（POOLNAME 可选：指定单个/查全部）、**IP 版本**（IPVERSION 可选，按版本过滤）、**详细度**（VERBOSE 可选，是否输出完整字段）。

### 配置维度 1：查询范围（参数 POOLNAME，可选）

| 取值 | 行为 |
|---|---|
| 指定 POOLNAME | 仅查该地址池配置 |
| 不配 | 查所有地址池配置 |

### 配置维度 2：IP 版本（参数 IPVERSION，可选）

| 取值 | 行为 |
|---|---|
| IPV4 | 仅查 IPv4 地址池 |
| IPV6 | 仅查 IPv6 地址池 |
| 不配 | 查所有版本地址池 |

### 配置维度 3：详细度（参数 VERBOSE，可选）

| 取值 | 行为 |
|---|---|
| ENABLE | 输出完整字段（含租约参数等） |
| 不配 | 输出关键字段 |

**典型脚本（GWFD-010105 调测：查所有地址池配置）**：
```
LST POOL:;
```

**典型脚本（指定 POOLNAME 查询）**：
```
LST POOL:POOLNAME="testpool1";
```

> 命令 notes：适用 NF：PGW-U、UPF；查询输出用于配置核查与排障。

**步骤位置**：在所属 feature 编排中通常位于"调测/验证阶段"——前置条件：POOL 已建；用于 POOL 配置核查。
