# 设置全局策略管理参数（SET UPGLBPMPARA）

> 命令静态知识见 [SET UPGLBPMPARA](command/UDG/20.15.2/SET-UPGLBPMPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`SET UPGLBPMPARA` 配置计费/策略相关全局控制参数。**仅 PGW-U/UPF 适用**。**全局唯一（最大记录数 1，存在系统初始记录）**。GWFD-020358 业务触发 QoS 保证用 `FLOWDANYFMT` 配置 N4 接口 Qos Rule 触发专有承载时 Flow-Description 携带 Any IP 的字符串格式（双栈场景需 IPMASK）。

### 配置维度 1：Qos Urr 老化时间（参数 QOSURRAGETIME）
| 取值 | 作用 | 初始 |
|---|---|---|
| 整数 60~7200 秒 | 内部异常致承载资源无法回收时的回收删除时间 | 120 |

### 配置维度 2：PDR 匹配优化（参数 PDRMATCHOPT）
| 取值 | 作用 |
|---|---|
| DISABLE | 否（参数不生效，功能与 ENABLE 相同） |
| ENABLE（初始） | 是 |

### 配置维度 3：重定向迟滞时间（参数 REDIRECTHYSTMR）
| 取值 | 作用 | 初始 |
|---|---|---|
| 整数 1~3600 秒 | 相邻两次重定向上报的最小时间间隔（避免系统负荷增长） | 600 |

### 配置维度 4：删除未被引用 URR 开关（参数 RMVNOREFURRSW）
| 取值 | 作用 | 初始 |
|---|---|---|
| DISABLE | 否 | DISABLE |
| ENABLE | N4 去激活/动态 PDR 删除致 URR 未引用时主动删除 URR | — |

### 配置维度 5：删除未被引用 URR 时延（参数 RMVNOREFURRTMR，RMVNOREFURRSW=ENABLE 时可选）
| 取值 | 作用 | 初始 |
|---|---|---|
| 整数 2~60 秒 | 主动删除 URR 的延迟时长 | 2 |

### 配置维度 6：流描述中 Any IP 地址格式（参数 FLOWDANYFMT）
| 取值 | 作用 | 初始 |
|---|---|---|
| ANYFMT | Flow-Description 携带 Any 时字符串为 "any to any" | ANYFMT |
| IPMASK | Flow-Description 携带 Any 时字符串为 "ip mask" | — |

**典型脚本**：

```
// 专有承载老化时长
SET UPGLBPMPARA:QOSURRAGETIME=60;
// GWFD-020358：双栈 + AnyToAny 触发专有承载时配 IPMASK
SET UPGLBPMPARA:FLOWDANYFMT=IPMASK;
```

**配置链路**（GWFD-020358 步骤 11，收尾）：业务规则 + 用户模板绑定后，配 UPF 上报 Flow-Description 格式（双栈专有承载场景）。

## 决策点：FLOWDANYFMT（DP 0-00297）

| 选项 | 适用场景 |
|---|---|
| ANYFMT（默认） | 单栈地址；Flow-Description 字符串 "any to any" |
| IPMASK | **双栈地址 + AnyToAny 规则触发专有承载**；Flow-Description 字符串 "ip mask" |

## 约束

- **rule-0-00297-1**（warning）：命令执行后对新数据流生效 — 非立即全局
- **rule-0-00297-2**（info）：全局唯一，最大记录数 1，系统存在初始记录（QOSURRAGETIME=120 / PDRMATCHOPT=ENABLE / REDIRECTHYSTMR=600 / RMVNOREFURRSW=DISABLE / RMVNOREFURRTMR=2 / FLOWDANYFMT=ANYFMT）
- **rule-0-00297-3**（warning）：QOSURRAGETIME 建议 ≥ 五元组老化时间（SET FLOWAGETIME（★R1.5 atom 待补））；若配了 QoS URR Stop 迟滞（SET QOSURRRPTCTRL（★R1.5 atom 待补） QOSURRHYSTIMER），建议 ≥ 老化时间+迟滞时间总和
- **rule-0-00297-4**（warning）：会话激活使用双栈地址 + 已配置 AnyToAny 规则的业务触发专有承载时，FLOWDANYFMT 须配 IPMASK — 否则 Flow-Description 格式不匹配
- **rule-0-00297-5**（info）：PDRMATCHOPT 配成 DISABLE 时不生效，功能和配成 ENABLE 时相同
