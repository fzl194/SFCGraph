# 设置全局策略管理参数（SET UPGLBPMPARA）

> 命令静态知识见 [SET UPGLBPMPARA](command/UDG/20.15.2/SET-UPGLBPMPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET UPGLBPMPARA 配置计费/策略相关全局控制参数。**全局唯一（最大记录数 1，存在系统初始记录）**。GWFD-020358 业务触发 QoS 保证用 `FLOWDANYFMT` 配置 N4 接口 Qos Rule 触发专有承载时 Flow-Description 携带 Any IP 的字符串格式（双栈场景需 IPMASK）。

**关键参数**

| 参数 | 取值 | 用途 | 典型行 |
|---|---|---|---|
| **FLOWDANYFMT** | ANYFMT（默认）/ IPMASK | GWFD-020358：Flow-Description 的 Any IP 格式；双栈地址+AnyToAny规则触发专有承载时需 IPMASK | `SET UPGLBPMPARA:FLOWDANYFMT=IPMASK;` |
| QOSURRAGETIME | 60-7200 秒（初始 120）| QoS 专有承载 URR 老化时间（内部异常致承载资源无法回收时的回收删除时间）| `SET UPGLBPMPARA:QOSURRAGETIME=60;` |
| REDIRECTHYSTMR | 1-3600 秒（初始 600）| 重定向迟滞时间（相邻两次重定向上报最小间隔）| — |
| RMVNOREFURRSW/TMR | DISABLE/ENABLE + 2-60秒 | 删除未被引用 URR 开关+时延（N4 去激活/动态PDR删除致URR未引用时主动删除）| — |
| PDRMATCHOPT | ENABLE（初始，DISABLE 不生效）| PDR 匹配优化 | — |

**脚本样例（GWFD-020358 activation 演示：Flow-Description Any 格式）**
```
SET UPGLBPMPARA:FLOWDANYFMT=ANYFMT;
```

**配置链路**（GWFD-020358 步骤 11，收尾）：业务规则+用户模板绑定后，配 UPF 上报 Flow-Description 格式（双栈专有承载场景）。

## 约束

- **对新数据流生效**（warning）：命令执行后对新数据流生效（非立即全局）。
- **全局唯一**（info）：最大记录数 1，系统存在初始记录（QOSURRAGETIME=120/PDRMATCHOPT=ENABLE/REDIRECTHYSTMR=600/RMVNOREFURRSW=DISABLE/RMVNOREFURRTMR=2/FLOWDANYFMT=ANYFMT）。
- **QOSURRAGETIME 建议**（info）：建议 ≥ 五元组老化时间（SET FLOWAGETIME）；若配了 QoS URR Stop 迟滞（SET QOSURRRPTCTRL QOSURRHYSTIMER），建议 ≥ 老化时间+迟滞时间总和。
- **双栈需 IPMASK**（warning）：会话激活使用双栈地址 + 已配置 AnyToAny 规则的业务触发专有承载时，FLOWDANYFMT 须配 IPMASK — 否则 Flow-Description 格式不匹配。
