# 定义IPsec保护数据流（ADD ACLGROUPIPSEC）

> 命令静态知识见 [ADD ACLGROUPIPSEC](command/UDG/20.15.2/ADD-ACLGROUPIPSEC.md)、[ADD ACLRULEADV4IPSEC](command/UDG/20.15.2/ADD-ACLRULEADV4IPSEC.md)。本页只讲配置生成实例化时这两条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。IPsec ACL 是定义"哪些流需要被 IPsec 保护"（感兴趣流）的专用 ACL——**仅匹配源/目的 IP，ACLACTION 固定 Permit**，不带端口。下游 IPsec 安全策略 [0-00215](task/UDG/20.15.2/0-00215.md) 经 ACLNUMBER/ACLNAME 引用本规则组。

## 配置方法（配置方法字典）

本 atom 是**聚合 atom**：先 [ADD ACLGROUPIPSEC] 建 IPsec 规则组容器，再用 [ADD ACLRULEADV4IPSEC] 在组内下高级 IPv4 规则定义"感兴趣流"。**关键差异点**（vs [0-00198](task/UDG/20.15.2/0-00198.md) ADD ACLRULEADV4 通用高级 ACL）：(1) ACLACTION 固定 Permit；(2) 仅源/目的 IP 匹配，不支持端口段；(3) VRFNAME 引用 IPsec 微服务侧 VPN；(4) 范围段独立于通用 ACL。

### 配置维度 1：规则组标识（参数 ACLNAME，必选）

| 取值形式 | 适用范围 | 典型场景 |
|---|---|---|
| **数字字符串 3000-3999** | IPsec 专用高级 ACL | IPFD-015004 10002 范例 `3000`；多 Sequence 场景可分多组（`3000`+`3001`） |
| **命名字符串 1~32** | 命名 IPsec ACL | 同上可改命名形式 |

> **范围段**：IPsec ACL 沿用高级 ACL 数字段（3000~3999）；**引用方式**：下游 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 的 ACLNUMBER（数字）或 ACLNAME（字符串）参数引用本组——ACLNAME 类型决定引用方式。

### 配置维度 2：规则行为（参数 ACLACTION，必选，固定 Permit）

| 取值 | 作用 | 说明 |
|---|---|---|
| **Permit** | 标记匹配流为"需要 IPsec 保护" | **IPsec 保护流必须 Permit**——非 Permit/Deny 任选 |

### 配置维度 3：匹配维度（仅源/目的 IP，不支持端口）

| 匹配维度 | 必选参数 | 配套反掩码 | 典型场景 |
|---|---|---|---|
| **源 IP** | ACLSOURCEIP（IPv4） | ACLSRCWILD（默认 255.255.255.255） | IPFD-015004 保护子网源端 |
| **目的 IP** | ACLDESTIP（IPv4） | ACLDESTWILD（默认 255.255.255.255） | IPFD-015004 保护子网目的端 |

> **不支持端口段**（vs 通用高级 ACL）：IPsec ACL 不支持 ACLPROTOCOL=TCP/UDP 时的端口段参数（SVRPORT*/ACLPORTOP* 等不适用）。

### 配置维度 4：VPN 上下文（参数 VRFNAME，可选）

| 取值形式 | 来源 | 适用场景 |
|---|---|---|
| **字符串 1~31** | 引用 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC:VRFNAME | 限定 IPsec ACL 生效 VPN 实例 |

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，两步：建组 + 建规则）**：
```
ADD ACLGROUPIPSEC: ACLNAME="3000";
ADD ACLRULEADV4IPSEC: ACLNAME="3000", ACLRULENAME="rule_1", ACLACTION=Permit, ACLPROTOCOL=0,
                     ACLSOURCEIP="10.102.101.0", ACLSRCWILD="0.0.0.255",
                     ACLDESTIP="10.102.102.0", ACLDESTWILD="0.0.0.255", VRFNAME="vrf1";
```

> 命令 notes：**最大记录数 65535**（整机 ACL 规则）；**单组最多 16384 条规则**；**执行后立即生效**；ACLSTEP（步长）/ACLTYPE（规则组类型，缺省 Advance）/ACLMATCHORDER（匹配顺序，缺省 Config）/ACLDESCRIPTION 在主场景未显式传参；多 Sequence IPsec 策略场景（多感兴趣流）建多个 ACL 规则组（如 `3000`+`3001`），下游每个 IPSECPOLICY SEQUENCENUMBER 各引用一个。

**步骤位置**：在所属 feature 编排中位于"IPsec 协商链最前端"——前置 无；下游 [0-00212](task/UDG/20.15.2/0-00212.md) IPSECPROPOSALIPSEC（IPsec 提议）+ [0-00213](task/UDG/20.15.2/0-00213.md) IKEPROPOSAL（IKE 提议）+ [0-00214](task/UDG/20.15.2/0-00214.md) IKEPEER（IKE 对等体）+ [0-00215](task/UDG/20.15.2/0-00215.md) IPSECPOLICY（IPsec 策略引用本规则组）。

## 决策点

（无。本命令在所有 IPsec 子场景下用法一致——ACLACTION 固定 Permit + 仅源/目的 IP 匹配 + 引用 IPsec VPN，无特性分叉。）

## 约束

- **最大记录数 65535**（critical，rule-0-00211）：整机 ACL 规则最大 65535 条 — 后果：超规格新增失败
- **单组最多 16384 条规则**（critical，rule-0-00211）：单个 ACL 规则组下最多 16384 条 — 后果：超限新增失败
- **前置必须先 ADD ACLGROUPIPSEC**（critical，rule-0-00211）：ADD ACLRULEADV4IPSEC 必须在 ACLGROUPIPSEC 建组后 — 后果：未建组新增规则失败
- **执行后立即生效**（warning，rule-0-00211）：执行后立即生效 — 后果：误操作即刻生效
- **ACLACTION 固定 Permit**（critical，rule-0-00211）：IPsec 保护流必须 Permit（Deny 无意义） — 后果：配 Deny 致规则无业务含义
- **不支持端口段**（warning，rule-0-00211）：IPsec ACL 仅支持源/目的 IP 匹配，SVRPORT*/ACLPORTOP* 不适用 — 后果：配端口参数命令拒收
- **VRFNAME 引用 IPsec VPN**（info，rule-0-00211）：VRFNAME 应引用 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC:VRFNAME — 后果：引用非 IPsec VPN 致流定义与协商不一致
- **ACLNAME 范围段**（info，rule-0-00211）：数字 ACLNAME 应在 3000~3999 高段 — 后果：跨段数字命令拒收
- **ACLNAME 长度**（info，rule-0-00211）：字符串 1~32（不支持空格，区分大小写） — 后果：超长命令拒收
- **VRFNAME 长度**（info，rule-0-00211）：字符串 1~31 — 后果：超长命令拒收
- **IPsec 协商链前置**（critical，rule-0-00211）：本规则组位于"ACL→Proposal→Peer→Policy"协商链最前端，须先于 [0-00212](task/UDG/20.15.2/0-00212.md) / [0-00213](task/UDG/20.15.2/0-00213.md) / [0-00214](task/UDG/20.15.2/0-00214.md) / [0-00215](task/UDG/20.15.2/0-00215.md) 建 — 后果：未建组下游策略引用失败
