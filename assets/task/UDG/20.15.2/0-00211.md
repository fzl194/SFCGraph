# 定义IPsec保护数据流（ADD ACLGROUPIPSEC）

> 命令静态知识见 [ADD ACLGROUPIPSEC](command/UDG/20.15.2/ADD-ACLGROUPIPSEC.md)、[ADD ACLRULEADV4IPSEC](command/UDG/20.15.2/ADD-ACLRULEADV4IPSEC.md)。本页只讲配置生成实例化时这两条命令**怎么配**。

## 配置方法

聚合 atom：先 [ADD ACLGROUPIPSEC] 建 IPsec ACL 规则组（容器），再用 [ADD ACLRULEADV4IPSEC] 在组内下高级 IPv4 规则定义"感兴趣流"。规则只匹配源/目的 IP，**不带端口**，ACLACTION 固定 **Permit**（保护流定义，非 Permit/Deny 任选）。

**数据规划取值样例**（证据③，IPFD-015004 各激活 md 数据规划表）：

| 参数 | 取值样例 | 获取方法 | 所属命令 | 说明 |
|---|---|---|---|---|
| ACLNAME（ACL规则组标识） | `3000`、`3001`、`3017`；多 Sequence 场景可分多组 `3000`+`3001` | 本端规划 | ADD ACLGROUPIPSEC | 规则组标识；若填编号则下游 IPSECPOLICY 用 ACLNUMBER 引用，若填名称则用 ACLNAME 引用 |
| ACLRULENAME（高级规则名称） | 规划值 | 本端规划 | ADD ACLRULEADV4IPSEC | 组内规则标识 |
| ACLACTION（ACL动作） | `Permit`（固定） | 固定 | ADD ACLRULEADV4IPSEC | IPsec 保护流必须 Permit |
| ACLPROTOCOL（协议） | 规划值 | 本端规划 | ADD ACLRULEADV4IPSEC | 感兴趣流协议 |
| ACLSOURCEIP / ACLSRCWILD（源 IP/反掩码） | 全网规划的 IPsec 保护子网 | 全网规划 | ADD ACLRULEADV4IPSEC | 仅源 IP 匹配 |
| ACLDESTIP / ACLDESTWILD（目的 IP/反掩码） | 全网规划的 IPsec 保护子网 | 全网规划 | ADD ACLRULEADV4IPSEC | 仅目的 IP 匹配 |
| VRFNAME（VPN 实例） | `vrf1` | 引用（[0-00071](0-00071.md)） | ADD ACLRULEADV4IPSEC | 引用 VNRS VPN 实例 |

> ACLSTEP（步长）/ACLTYPE（规则组类型，缺省 Advance）/ACLMATCHORDER（匹配顺序，缺省 Config）/ACLDESCRIPTION 在主场景未显式传参（冷门，不绑）。仅 03567841/10009 类"指定本端接口"场景显式带 ACLSTEP/ACLTYPE/ACLMATCHORDER。

**典型范式**（证据③任务示例脚本，两步：建组 + 建规则）：
```
ADD ACLGROUPIPSEC: ACLNAME="3000";
ADD ACLRULEADV4IPSEC: ACLNAME="3000", ACLRULENAME=..., ACLACTION=Permit, ACLPROTOCOL=...,
                     ACLSOURCEIP=..., ACLSRCWILD=..., ACLDESTIP=..., ACLDESTWILD=..., VRFNAME="vrf1";
```

> 多 Sequence IPsec 策略场景（多感兴趣流）建多个 ACL 规则组（如 `3000` + `3001`），下游每个 IPSECPOLICY SEQUENCENUMBER 各引用一个。

## 约束

- **rule-0-00273**（warning，聚合 atom 内部前置）(1) 顺序约束：ACLGROUPIPSEC 必须先发；(2) 端口约束：IPsec ACL 仅支持源/目的 IP 匹配，不支持端口（SVRPORT*/ACLPORTOP* 等不适用）；(3) 动作约束：ACLACTION 固定 Permit；(4) 上下文：ACLSOURCEIP/ACLDESTIP 取自 IPsec 保护子网，VRFNAME 引用 [0-00071](0-00071.md) VNRS VPN 实例。
- **rule-0-00077**（IPsec 协商链前置）ACLGROUPIPSEC 位于"ACL→Proposal→Peer→Policy→接口应用"协商链最前端，须先于 [0-00212](0-00212.md) IPsec 提议、[0-00214](0-00214.md) IKE 对等体、[0-00215](0-00215.md) IPsec 策略建。
