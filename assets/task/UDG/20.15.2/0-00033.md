# 配置带宽管理用户组（ADD BWMUSERGROUP）

> 命令静态知识见 [ADD BWMUSERGROUP](command/UDG/20.15.2/ADD-BWMUSERGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD BWMUSERGROUP 是带宽控制骨架的"用户组定义层"：把带宽管理对象按全局/默认/具体 3 档用户组划分，产出 USERGROUPNAME 供 ADD APNBINDBWMUSRG(0-00035) 引用做 APN 绑定、ADD BWMRULE(0-00037) 引用做规则承载。命令核心配置空间有 3 个维度：**用户组类型**（USERGROUPTYPE 决定是否需要 USERGROUPNAME + USERGROUPPRI 必选）、**业务类型**（USERLEVSRVTYPE/USERGLEVSRVTYPE 二选一或都选，决定是 TOS 还是 NONTOS 业务）、**规则使能**（USERLEVRULESW/USERGLEVRULESW 可选，控制对应层级规则开关）。3 维度组合出 N 种用户组形态；目前 3 个特性命中都仅用 SPECIFIC_GROUP 形态。

### 配置维度 1：用户组类型（参数 USERGROUPTYPE，必选）

| 取值 | 作用 | 必选联动 | 备注 |
|---|---|---|---|
| **DEFAULT_GROUP** | 默认用户组（系统保留） | 无 | 命令级默认入口；实际不演示 |
| **SPECIFIC_GROUP** | 具体用户组（按用户/业务自定义） | USERGROUPNAME + USERGROUPPRI | 3 个特性命中均选此档 |
| **GLOBAL** | 全局用户组（整机级） | 无 | 整机带宽规则挂接点 |

### 配置维度 2：业务类型（参数 USERLEVSRVTYPE/USERGLEVSRVTYPE，可选/2 选 1 或都选）

| 参数 | 层级 | 取值 | 作用 |
|---|---|---|---|
| **USERLEVSRVTYPE** | 用户级 | TOS / NONTOS | 决定该用户组对"用户级"业务的区分（按 TOS 标记 / 不按） |
| **USERGLEVSRVTYPE** | 用户组级 | TOS / NONTOS | 决定该用户组对"用户组级"业务的区分（110301/110311 常用） |

### 配置维度 3：规则使能（参数 USERLEVRULESW/USERGLEVRULESW，可选）

| 参数 | 层级 | 取值 | 作用 |
|---|---|---|---|
| **USERLEVRULESW** | 用户级 | ENABLE / DISABLE（默认） | 是否启用用户级带宽管理规则 |
| **USERGLEVRULESW** | 用户组级 | ENABLE / DISABLE（默认） | 是否启用用户组级带宽管理规则（110311 范式） |

**典型脚本（020354 基于业务的 Shaping，NONTOS + TOS 双演示）**：
```
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="usergroup_test",USERGROUPPRI=2,USERLEVSRVTYPE=NONTOS;
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="usergroup_test",USERGROUPPRI=2,USERLEVSRVTYPE=TOS;
```

**典型脚本（110301 码率差异化，用户组级 NONTOS）**：
```
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",USERGROUPPRI=1,USERGLEVSRVTYPE=NONTOS;
```

**典型脚本（110311 业务感知带宽控制，组级 TOS+规则使能）**：
```
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",USERGROUPPRI=1,USERGLEVSRVTYPE=TOS,USERGLEVRULESW=ENABLE;
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；最大记录数 / 优先级 范围未明示，需查命令 wiki 完整参数表。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：用户组定义"——前置无；产出 USERGROUPNAME 被 ADD APNBINDBWMUSRG(0-00035) 引用做 APN 绑定，USERGROUPNAME + USERGROUPPRI 被 ADD BWMRULE(0-00037) 引用做规则承载。

## 决策点：用户组类型档位选择（DP 0-00033）

问：ADD BWMUSERGROUP 选哪个用户组档位？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| SPECIFIC_GROUP（默认，特性全部命中） | + USERGROUPNAME + USERGROUPPRI | 按 APN/用户/业务自定义（020354/110301/110311） |
| DEFAULT_GROUP | 无 | 系统默认组（命令级默认入口，冷门） |
| GLOBAL | 无 | 整机级带宽规则挂接（110313 组级带宽控制候选） |

## 约束

- **USERGROUPTYPE=SPECIFIC_GROUP 时 USERGROUPNAME + USERGROUPPRI 必选**（critical，rule-0-00333）：USERGROUPNAME/USERGROUPPRI 是 SPECIFIC_GROUP 的"标识+优先级"两件套 — 后果：不配 ADD 拒收
- **USERGROUPNAME 全网唯一**（critical，隐含）：不同用户组之间 USERGROUPNAME 不能重复 — 后果：重名 ADD 失败
- **用户级 / 用户组级业务类型独立**（warning，rule-0-00333）：USERLEVSRVTYPE 与 USERGLEVSRVTYPE 互不覆盖，可同时配或任选其一 — 后果：业务区分粒度受所选层影响
- **USERGLEVRULESW 仅当 USERGLEVSRVTYPE 已配时生效**（warning，rule-0-00333）：组级规则使能必须有组级业务类型作为承载 — 后果：未配 USERGLEVSRVTYPE 时 USERGLEVRULESW=ENABLE 无意义
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
