# 配置带宽管理用户组（ADD BWMUSERGROUP）

> 命令静态知识见 [ADD BWMUSERGROUP](command/UDG/20.15.2/ADD-BWMUSERGROUP.md)

## 配置方法

新增一个具体用户组（USERGROUPTYPE 恒为 `SPECIFIC_GROUP`），指定名称、优先级与业务类型。业务类型字段按用户级/用户组级二选一（见决策点），实战普遍显式配 `NONTOS`/`TOS`。

| 参数 | 取值样例 | 说明 |
|---|---|---|
| USERGROUPTYPE | `SPECIFIC_GROUP` | 全特性统一；全局组为系统默认不可建，默认组用 DEFAULT_GROUP |
| USERGROUPNAME | `ug_test` / `usergroup_test` | 1~31 字符，无空格，全网唯一 |
| USERGROUPPRI | `1`（组级）/ `2`（用户级） | 1~32768，同级别下唯一 |
| USERLEVSRVTYPE | `NONTOS` / `TOS` | 用户级路径（020354）显式传 |
| USERGLEVSRVTYPE | `NONTOS` / `TOS` | 用户组级路径（110301/110311）显式传 |

**典型脚本**：

```
# 用户级整形（020354）
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="usergroup_test",USERGROUPPRI=2,USERLEVSRVTYPE=NONTOS;
# 用户组级差异化（110301）
ADD BWMUSERGROUP:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",USERGROUPPRI=1,USERGLEVSRVTYPE=NONTOS;
```

**步骤位置**：在 BWM 三件套中位于 BWMSERVICE 之后、BWMRULE 之前——先定义用户组容器，再配规则把 Service+Controller 绑入。

## 决策点

**DP 0-00098**：BWMUSERGROUP 业务类型（用户级 vs 用户组级）——USERLEVSRVTYPE 与 USERGLEVSRVTYPE 互斥二选一，命令真相均 optional 默认 TOS，UDG 实际部署显式二选一。

| 选项 | 取值 | 联动参数 |
|---|---|---|
| 用户级（默认，020354 路径） | USERLEVSRVTYPE | USERLEVSRVTYPE=required；USERGLEVSRVTYPE=absent |
| 用户组级（110301 路径） | USERGLEVSRVTYPE | USERGLEVSRVTYPE=required；USERLEVSRVTYPE=absent |

## 约束

- 最大记录数 1999（默认+具体组合计，全局组不计）· warning · 超规格新增失败；90% 阈值上报告警 ALM-135602215，MOD CFGTHRESHOLD 可调
- 新增用户组对已在线用户不生效，仅对新激活用户生效 · warning · 在线用户需重激活才纳入管控
- 同一用户级/用户组级下 USERGROUPPRI 不能相同 · warning · 优先级冲突
