# 加载协议特征库（LOD SIGNATUREDB）

> 命令静态知识见 [LOD SIGNATUREDB](command/UDG/20.15.2/LOD-SIGNATUREDB.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

LOD SIGNATUREDB 是七层协议识别的"前置原料加载"命令：把 SIGNATUREDB 特征库文件加载到 UPF 内存，供后续协议识别（IM 类、IPv6 SA、用户 Portal、HTTPS 业务地址识别、DNS 纠错、HTTPS 业务行为识别）使用。首次加载用于初始化；已加载时本命令用于**升级**新版本特征库（升级不中断业务）。命令核心配置空间有 2 个维度：**加载模式**（LOADMODE，必选，决定 VERSION 是否出现）+ **版本号**（VERSION，条件必选，LOADMODE=SPECIFIC 时必填）。与 LOD PARSERDB(0-00152) 配套使用（IM 场景需两者；IPv6 SA 仅需本库）。

### 配置维度 1：加载模式（参数 LOADMODE，必选）

| 取值 | 作用 | VERSION 必选 | 适用场景 |
|---|---|---|---|
| **LATEST** | 加载/升级到当前最新版本特征库 | 不配 | 020352 IPv6 SA、020359 IM 类业务、110203 HTTPS 地址识别、110281 用户 Portal（多数场景默认） |
| **SPECIFIC** | 加载指定版本特征库 | 必填 | 110283 DNS 纠错、110404 HTTPS 业务行为识别（需固定版本） |

### 配置维度 2：版本号（参数 VERSION，条件必选）

| 取值 | 格式 | 适用 |
|---|---|---|
| 字符串 1~16 | 版本号形式如 `01.0002.0358.01` | 仅 LOADMODE=SPECIFIC 时必填 |

**典型脚本（LATEST）**：
```
LOD SIGNATUREDB:LOADMODE=LATEST;
```

**典型脚本（SPECIFIC）**：
```
LOD SIGNATUREDB:LOADMODE=SPECIFIC,VERSION="01.0002.0358.01";
```

> 命令 notes：升级不中断业务；加载约 5 分钟；加载后 DSP SIGNATUREDB（★R1.5 atom 待补） 验证状态（查询命令不建 atom）；新库相比当前版本不允许删除协议或修改协议名称（违规需先清理相关配置）；未加载特征库将无法进行七层协议解析；库文件须经网管/MAE 或 OM Portal 文件传输上传。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：协议特征库加载/升级"——前置无；产出供七层识别链路（ADD WELLKNOWNPORT + ADD EXTENDEDFILTER/ADD FLOWFILTER）使用。

## 决策点：特征库加载模式（DP 0-00027）

问：LOD SIGNATUREDB 的加载模式取哪个分支？

| 选项 | 联动参数 | 适用特性 |
|---|---|---|
| 最新版加载（LATEST，默认） | 仅 LOADMODE=LATEST（VERSION 缺省） | 020352 IPv6 SA、020359 IM 类业务、110203 HTTPS 地址识别、110281 用户 Portal |
| 指定版本加载（SPECIFIC） | LOADMODE=SPECIFIC + VERSION（必填） | 110283 DNS 纠错（`01.0002.0358.01`）、110404 HTTPS 业务行为识别（`01.0002.1061.01`） |

## 约束

- **VERSION 配套约束**（critical，rule-0-00305）：LOADMODE=SPECIFIC 时 VERSION 必选；LOADMODE=LATEST 时不允许配 VERSION — 后果：VERSION 缺配但 LOADMODE=SPECIFIC→ADD 拒收；LATEST 误配 VERSION→拒收
- **库文件前置上传**（critical，rule-0-00305）：新增/升级知识库须先经网管/MAE"网元文件传输"或 OM Portal"文件传输"上传库文件 — 后果：未上传→LOD 命令找不到文件→失败
- **协议一致性约束**（critical，rule-0-00305）：新库相比当前版本不允许删除协议或修改协议名称 — 后果：违规需先清理相关配置，否则新库加载失败
- **升级不中断业务**（warning，rule-0-00305）：升级知识库文件不影响当前业务 — 后果：无须停机
- **加载时长与验证**（warning，rule-0-00305）：加载约 5 分钟；加载后须用 DSP SIGNATUREDB（★R1.5 atom 待补） 验证状态 — 后果：未验证可能误判未生效
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
- **配套 PARSERDB**（warning，隐含）：IM 类业务（020359）等场景需同时加载 PARSERDB（[0-00152](task/UDG/20.15.2/0-00152.md)）— 后果：仅加载本库而未加载 PARSERDB 时 IM 类识别失败
