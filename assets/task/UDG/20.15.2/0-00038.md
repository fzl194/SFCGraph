# 配置QoS属性（ADD QOSPROP）

> 命令静态知识见 [ADD QOSPROP](command/UDG/20.15.2/ADD-QOSPROP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD QOSPROP 是 PCC 业务属性链的"QoS 参数定义层"：把 PCC 预定义规则（或 L3/4 / L7 PCC Rule）的 QoS 参数（带宽+计费+解耦+关键流）用一条命名记录描述，产出 QOSPROPNAME 供 ADD PCCPOLICYGRP(0-00003) 的 QOSPROPNAME 引用。命令核心配置空间有 4 个维度：**带宽维度**（GBR/ MBR × UPLK/DNLK 四件套）、**QoS 使用量上报**（QOSURRNAME 引用 URR）、**上下行解耦**（DECOUPLINGSW）、**关键流检测**（KEYFLOWDETECTSW + KEYFLOWTIME/KEYFLOWSPEED）。3 个特性命中覆盖 020358 GBR/MBR+QOSURR 基础、110302 仅解耦、112000 GBR/MBR+QOSURR+解耦+关键流全配置。

### 配置维度 1：带宽四件套（参数 GBRUPLKVALUE/GBRDNLKVALUE/MBRUPLKVALUE/MBRDNLKVALUE，可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **GBRUPLKVALUE** | 0~20000000（kbps） | 保证的上行比特率（上行 GBR） |
| **GBRDNLKVALUE** | 0~20000000（kbps） | 保证的下行比特率（下行 GBR） |
| **MBRUPLKVALUE** | 0~20000000（kbps） | 最大上行比特率（上行 MBR） |
| **MBRDNLKVALUE** | 0~20000000（kbps） | 最大下行比特率（下行 MBR） |

### 配置维度 2：QoS 使用量上报（参数 QOSURRNAME，可选）

| 取值 | 作用 |
|---|---|
| 引用 [0-00001](task/UDG/20.15.2/0-00001.md) 的 URRNAME | QoS 触发的话单/上报走该 URR 计量 |

### 配置维度 3：上下行解耦（参数 DECOUPLINGSW，可选）

| 取值 | 作用 |
|---|---|
| **ENABLE** | 开启上下行解耦视频差异化调度（本命令配置的下行链路模式优先级高于 ADD SADEDICBEARER 的配置） |
| **DISABLE**（默认） | 关闭上下行解耦 |

### 配置维度 4：关键流检测（参数 KEYFLOWDETECTSW + KEYFLOWTIME/KEYFLOWSPEED，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **KEYFLOWDETECTSW** | ENABLE / DISABLE（默认） | 是否开启关键业务流识别 |
| **KEYFLOWTIME** | 整数（秒，KEYFLOWDETECTSW=ENABLE 时可选） | 识别关键流的时长阈值 |
| **KEYFLOWSPEED** | 整数（千比特每秒，KEYFLOWDETECTSW=ENABLE 时可选） | 识别关键流的速率阈值 |

**典型脚本（020358 业务触发的 QoS 保证，三四层 GBR+MBR+QOSURR 基础）**：
```
ADD QOSPROP:QOSPROPNAME="qos-property1", QOSURRNAME="urr_1", GBRUPLKVALUE=110, GBRDNLKVALUE=110, MBRUPLKVALUE=220, MBRDNLKVALUE=220;
```

**典型脚本（110302 上下行解耦视频承载信令控制，仅解耦开关）**：
```
ADD QOSPROP:QOSPROPNAME="qos-property1", QOSURRNAME="urr_1",DECOUPLINGSW=ENABLE;
```

**典型脚本（112000 双故障 Bypass，GBR+MBR+QOSURR+解耦+关键流全配置）**：
```
ADD QOSPROP: QOSPROPNAME="qosprop_voice", GBRUPLKVALUE=64, GBRDNLKVALUE=64, MBRUPLKVALUE=64, MBRDNLKVALUE=64, QOSURRNAME="urr_qos_01", DECOUPLINGSW=DISABLE, KEYFLOWDETECTSW=DISABLE;
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；L3/4 与 L7 PCC Rule 的 QoS-information 都可由此命令配置；带宽四件套可选 0~20000000 kbps；DECOUPLINGSW=ENABLE 时本命令的下行优先级高于 [0-00040](task/UDG/20.15.2/0-00040.md)（ADD SADEDICBEARER）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：QoS 属性定义"——前置 ADD URR(0-00001)（如配 QOSURRNAME）；产出 QOSPROPNAME 被 ADD PCCPOLICYGRP(0-00003) 的 QOSPROPNAME 引用。

## 决策点：QoS 属性形态选择（DP 0-00038）

问：ADD QOSPROP 选哪些维度组合？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅带宽四件套 | GBRUPLKVALUE/GBRDNLKVALUE/MBRUPLKVALUE/MBRDNLKVALUE | 业务触发 QoS 保带宽（020358 范式） |
| 仅解耦开关 | QOSURRNAME + DECOUPLINGSW=ENABLE | 上下行解耦视频承载（110302 范式） |
| 关键流检测 + 带宽 + 解耦 | 全部配置 | 语音双故障 Bypass 关键流保障（112000 范式） |
| 最小配置（仅 QOSURRNAME） | QOSURRNAME | 仅做使用量上报（命令参数齐全） |

> 选项不互斥：同一 feature 可建多条 QOSPROP 覆盖不同业务。

## 约束

- **QOSURRNAME 引用前置依赖**（critical，rule-0-00338）：配 QOSURRNAME 时 URR 须先建（[0-00001](task/UDG/20.15.2/0-00001.md)） — 后果：未建引用失败
- **KEYFLOWTIME/KEYFLOWSPEED 仅 KEYFLOWDETECTSW=ENABLE 时生效**（warning，rule-0-00338）：未开关键流检测时配关键流阈值无意义 — 后果：误配拒收或被忽略
- **带宽四件套独立可选**（warning，rule-0-00338）：GBR/MBR 上下行 4 个参数互不绑定，可单配或全配 — 后果：未配项按命令默认行为生效
- **DECOUPLINGSW 优先级高于 SADEDICBEARER**（critical，rule-0-00338）：DECOUPLINGSW=ENABLE 时下行链路模式优先级高于 [0-00040](task/UDG/20.15.2/0-00040.md) 配置 — 后果：解耦/承载配置冲突时以 QOSPROP 为准
- **QOSPROPNAME 全网唯一**（critical，隐含）：不同 QoS 属性之间 QOSPROPNAME 不能重复 — 后果：重名 ADD 失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
