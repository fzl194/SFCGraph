# 配置QoS属性（ADD QOSPROP）
> 命令静态知识见 [ADD QOSPROP](command/UDG/20.15.2/ADD-QOSPROP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

命名一个 QoS 属性对象并填四档比特率（上下行保证/最大），再挂一个已配的 QoS URR 做用量上报，供下游 ADD PCCPOLICYGRP 的 QOSPROPNAME 引用——PCC 策略命中时把这套 QoS 参数作用到匹配的数据流。

| 参数 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| QOSPROPNAME（QoS 属性名） | qos-property1 / qosprop_voice | 本端规划 | 1~31 字符串，全局唯一 |
| GBRUPLKVALUE（保证上行比特率） | 110 / 64 | 本端规划 | 0~20000000 kbps |
| GBRDNLKVALUE（保证下行比特率） | 110 / 64 | 本端规划 | 同上 |
| MBRUPLKVALUE（最大上行比特率） | 220 / 64 | 本端规划 | 同上 |
| MBRDNLKVALUE（最大下行比特率） | 220 / 64 | 本端规划 | 同上 |
| QOSURRNAME（QoS 用量上报规则名） | urr_1 / urr_qos_01 | 引用 | 须引用 ADD URR(USAGERPTMODE=QOS) 的 [0-00001](task/UDG/20.15.2/0-00001.md) |
| DECOUPLINGSW（上下行解耦开关） | DISABLE / ENABLE | 本端规划 | 默认 DISABLE；视频差异化调度场景必 ENABLE |
| KEYFLOWDETECTSW（关键流检测开关） | DISABLE | 本端规划 | 默认 DISABLE |

典型脚本样例（按场景二选一）：
```
# 普通保证比特率（GWFD-020358 L7/L3L4 QoS 保证）
ADD QOSPROP:QOSPROPNAME="qos-property1", QOSURRNAME="urr_1", GBRUPLKVALUE=110, GBRDNLKVALUE=110, MBRUPLKVALUE=220, MBRDNLKVALUE=220;

# 视频上下行解耦（GWFD-110302）——只配 QoS URR + 解耦开关
ADD QOSPROP:QOSPROPNAME="qos-property1", QOSURRNAME="urr_1", DECOUPLINGSW=ENABLE;

# 语音 Bypass（GWFD-112000）——四档相等、关闭解耦/关键流
ADD QOSPROP:QOSPROPNAME="qosprop_voice", GBRUPLKVALUE=64, GBRDNLKVALUE=64, MBRUPLKVALUE=64, MBRDNLKVALUE=64, QOSURRNAME="urr_qos_01", DECOUPLINGSW=DISABLE, KEYFLOWDETECTSW=DISABLE;
```

配置要点：四档比特率按业务 SLA 规划（保证型 GBR≤最大 MBR）；QOSURRNAME 必须是 ADD URR 且 USAGERPTMODE=QOS 的 QoS URR（强约束 rule 0-00027）；视频差异化调度场景 DECOUPLINGSW=ENABLE 是特性硬性要求。条件可选 KEYFLOWTIME/KEYFLOWSPEED/CFGDOMAINNAME 冷门暂不投影，待后续特性揭示时增量补全。

## 约束
- QoS URR 上报模式：QOSURRNAME 须为 ADD URR(USAGERPTMODE=QOS) 的 QoS URR · critical · 否则 QoS 用量上报失效（详见 [0-00027](task/UDG/20.15.2/0-00027.md)）
- 最大记录数 500 / 立即生效 · warning · 超规格新增失败
- 一个 PCCPOLICYGRP 仅绑定一个 QOSPROP · warning · 多挂取最后命中
- 适用 NF：PGW-U、UPF
