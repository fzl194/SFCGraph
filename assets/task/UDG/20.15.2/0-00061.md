# 配置接口IPv6地址（ADD IFIPV6ADDRESS）

> 命令静态知识见 [ADD IFIPV6ADDRESS](command/UDG/20.15.2/ADD-IFIPV6ADDRESS.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**前置条件**：必须先 [0-00060](task/UDG/20.15.2/0-00060.md) SET IFIPV6ENABLE（缺省不使能 IPv6）；IFNAME 引用 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 已建的接口。

## 配置方法（配置方法字典）

ADD IFIPV6ADDRESS 是"手工配置接口 IPv6 地址"命令。三参 IFNAME/IPV6ADDRESS/PREFIXLEN 必填，**地址与前缀全网规划**（无特性分叉）。核心配置空间有 3 个维度：**接口类型**（决定典型前缀）、**地址类型**（ADDRTYPE）、**计算类型**（IDGENTYPE）。3 个特性命中（010107、020402、IPFD-015004）覆盖 Tunnel/以太网子接口/Tunnel 三类接口范式。

### 配置维度 1：接口类型（参数 IFNAME，必选）

IFNAME 引用已配的逻辑/物理接口（已 SET IFIPV6ENABLE）：

| 接口类型 | 典型 IFNAME | 典型前缀 | 典型场景 |
|---|---|---|---|
| **Tunnel** | `Tunnel<n>` | `/64` 或 `/96` | GRE Tunnel IPv6 承载（010107 范式：/64）；IPsec IPv6 隧道（IPFD-015004 范式：/64 或 /96） |
| **以太网子接口** | `Ethernet<slot>/<card>/<port>.<vlan>` | `/64` | N6/Gi/SGi IPv6 外联口（020402 范式） |

> **双平面组网**（020402）：两 trunk 允许通过的 Vlan 须不同，避免环路风险；每子接口配独立 IPv6 地址（如 `fc00:1000::1` / `fc00:2000::1`）。

### 配置维度 2：地址类型（参数 ADDRTYPE，可选，默认 global）

| 取值 | 作用 |
|---|---|
| **global**（默认） | 全球单播地址 |
| **linkLocal** | 链路本地地址 |
| **anycast** | 任播地址 |

> 3 个特性命中脚本均不显式 ADDRTYPE，使用缺省 `global`。

### 配置维度 3：计算类型（参数 IDGENTYPE，可选，默认 none）

| 取值 | 作用 |
|---|---|
| **none**（默认） | 无 |
| **cga** | CGA |
| **eui64** | Eui64（接口标识由 MAC 地址 EUI-64 派生） |

> 3 个特性命中脚本均不显式 IDGENTYPE，使用缺省 `none`。

**典型脚本（010107 静态路由冗余，Tunnel /64）**：
```
ADD IFIPV6ADDRESS: IFNAME="Tunnel1", IPV6ADDRESS="fc00:2::1", PREFIXLEN=64;
ADD IFIPV6ADDRESS: IFNAME="Tunnel2", IPV6ADDRESS="fc00:2::2", PREFIXLEN=64;
```

**典型脚本（020402 IPv6 组网，以太网子接口 /64）**：
```
ADD IFIPV6ADDRESS:IFNAME="Ethernet70/0/4.1",IPV6ADDRESS="fc00:1000::1",PREFIXLEN=64;
ADD IFIPV6ADDRESS:IFNAME="Ethernet70/0/5.1",IPV6ADDRESS="fc00:2000::1",PREFIXLEN=64;
```

**典型脚本（IPFD-015004 IPsec IPv6 隧道，Ethernet + Tunnel）**：
```
ADD IFIPV6ADDRESS:IFNAME="Ethernet66/0/5", IPV6ADDRESS="2001:db8:1::1", PREFIXLEN=96;
ADD IFIPV6ADDRESS:IFNAME="Tunnel27", IPV6ADDRESS="2001:db8:1::1:0:1", PREFIXLEN=96;
```

> 命令 notes：**最大记录数 65535**；**执行后立即生效**；仅在 VNRS_VNFC 的 Ethernet 接口/Ethernet 子接口/Eth-Trunk/Eth-Trunk 子接口/Loopback/Tunnel 口配置；**前置必须 SET IFIPV6ENABLE ENABLEFLAG=TRUE**（缺省不使能 IPv6）；修改 IPv6 地址需 `RMV IFIPV6ADDRESS` 删除后再 `ADD` 新地址；本命令参与自动化配置——下发前需 `DSP OPSASSISTSTATE` 查询 autoscaling_autoconfig.py。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N+1：配置接口 IPv6 地址"——前置 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE（建接口）+ [0-00057](task/UDG/20.15.2/0-00057.md) ADD IPBINDVPN（绑 VPN 后才能配私网 IPv6）+ [0-00060](task/UDG/20.15.2/0-00060.md) SET IFIPV6ENABLE（使能 IPv6）。

## 决策点：IPv6 前缀长度（DP 0-00061）

问：ADD IFIPV6ADDRESS 的 PREFIXLEN 取何值？

| 选项 | PREFIXLEN | 适用场景 |
|---|---|---|
| **/64 标准前缀** | 64 | GRE Tunnel IPv6 承载（010107）、以太网子接口 IPv6 组网（020402）、IPsec IPv6 隧道（IPFD-015004 默认） |
| **/96 长前缀** | 96 | IPsec IPv6 隧道 IPsec 端到端地址（IPFD-015004 范式） |

> 3 个特性命中（010107/020402/IPFD-015004）以 /64 为主流；PREFIXLEN 取值范围 1~128；ADDRTYPE/IDGENTYPE 在 UDG 场景实例化不用，使用缺省值。

## 约束

- **最大记录数 65535**（warning，rule-0-00461）：接口 IPv6 地址最大 65535 条 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00461）：命令执行后立即生效 — 后果：误操作即刻生效
- **前置必须使能 IPv6**（critical，rule-0-00461）：缺省不使能 IPv6，必须先 [0-00060](task/UDG/20.15.2/0-00060.md) SET IFIPV6ENABLE 才能配 — 后果：未使能则命令拒收
- **修改地址需先删除**（warning，rule-0-00461）：修改接口 IPv6 地址必须先 RMV IFIPV6ADDRESS 删除再 ADD 新地址 — 后果：直接 ADD 新地址会冲突
- **参与自动化配置**（critical，rule-0-00461）：IPV6ADDRESS/PREFIXLEN 参与自动化配置——下发前需 `DSP OPSASSISTSTATE` 查询 autoscaling_autoconfig.py — 后果：自动化配置过程中执行致最终生效配置不可预期
- **适用接口类型限定**（critical，rule-0-00461）：仅 VNRS_VNFC 的 Ethernet/Ethernet 子接口/Eth-Trunk/Eth-Trunk 子接口/Loopback/Tunnel 口生效 — 后果：其他接口类型命令拒收
- **PREFIXLEN 范围**（info，隐含）：PREFIXLEN 取值范围 1~128 — 后果：超出范围命令拒收
