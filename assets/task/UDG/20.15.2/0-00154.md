# 设置SA业务公共参数（SET SACOMMONPARA）

> 命令静态知识见 [SET SACOMMONPARA](command/UDG/20.15.2/SET-SACOMMONPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET SACOMMONPARA 用于配置 SA 业务相关控制参数集合（含 QUIC 协议识别/解析增强、CDN 增强、用户关联刷新等 7 个可选参数）。**所有参数均为可选**，按需配置；典型场景是 HTTP3.0（QUIC）Host 识别/分析 前置开关。**单命令最大记录数 1**（SA 业务公共参数全局配置）。核心配置空间有 7 个维度：**QUIC 协议识别增强**（QUICIDENFUNCEN）、**QUIC 协议解析增强**（QUICPSRFUNCEN）、**CDN 增强识别**（CDNENHANCEDSW）、**用户关联刷新**（USRRLTRFRSHSW）、**无确定业务报文数**（NORSTPKTNUM）、**HTTP 特殊方法解析**（SPECMETHODSW）、**Quic SA 协议匹配**（QUICSASW）。2 个特性命中（110251 HTTP3.0 Host 识别 / 110252 HTTP3.0 Host 分析）。

### 配置维度 1：QUIC 协议识别功能增强（参数 QUICIDENFUNCEN，可选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 关闭对全加密 Quic 承载应用的协议识别增强（默认） |
| **ENABLE** | 开启对全加密 Quic 承载应用的协议识别增强（110251 范式） |

### 配置维度 2：QUIC 协议解析功能增强（参数 QUICPSRFUNCEN，可选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 关闭对全加密 Quic 承载应用的协议解析增强（默认） |
| **ENABLE** | 开启对全加密 Quic 承载应用的协议解析增强（110252 范式） |

### 配置维度 3：CDN 增强识别功能（参数 CDNENHANCEDSW，可选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 关闭 CDN 识别增强功能（默认） |
| **ENABLE** | 开启 CDN 识别增强功能 |

### 配置维度 4：用户关联识别刷新开关（参数 USRRLTRFRSHSW，可选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 关闭协议识别的刷新功能（默认） |
| **ENABLE** | 开启协议识别的刷新功能 |

### 配置维度 5：无确定业务报文个数（参数 NORSTPKTNUM，可选）

| 取值 | 作用 |
|---|---|
| **整数 0~255**（默认 0） | 控制连续多少个报文没有确定业务，则确定业务；0=不开启此控制 |

### 配置维度 6：HTTP 特殊方法解析开关（参数 SPECMETHODSW，可选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 关闭 HTTP 特殊方法解析（默认） |
| **ENABLE** | 开启 HTTP 特殊方法解析 |

### 配置维度 7：Quic SA 协议匹配开关（参数 QUICSASW，可选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 关闭（默认） |
| **ENABLE** | 在 QUIC SA 协议识别状态确定后进行策略匹配 |

**典型脚本（GWFD-110251 HTTP3.0 Host 识别，识别增强）**：
```
SET SACOMMONPARA:QUICIDENFUNCEN=ENABLE;
```

**典型脚本（GWFD-110252 HTTP3.0 Host 分析，解析增强）**：
```
SET SACOMMONPARA:QUICPSRFUNCEN=ENABLE;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：开 SA 业务公共参数"——前置 [0-00152](task/UDG/20.15.2/0-00152.md) LOD PARSERDB（加载协议解析库）+ [0-00027](task/UDG/20.15.2/0-00027.md) LOD SIGNATUREDB（加载协议特征库）后启用 QUIC 增强；QUICIDENFUNCEN 与 QUICPSRFUNCEN 二选一，互斥。

## 决策点：QUIC 增强特性路由（DP 0-00154）

问：当前特性需 QUIC 协议识别增强还是 QUIC 协议解析增强？

| 选项 | QUICIDENFUNCEN | QUICPSRFUNCEN | 适用特性 |
|---|---|---|---|
| **QUIC 识别增强**（110251） | ENABLE | DISABLE | HTTP3.0 Host 识别 |
| **QUIC 解析增强**（110252） | DISABLE | ENABLE | HTTP3.0 Host 分析 |

> 2 个特性命中（110251/110252）；两开关功能互斥（识别 vs 解析），**不应同开**；仅针对 Quic 密文业务增强 SA，对非 Quic 业务无效。

## 约束

- **所有参数均为可选**（info，rule-0-00154）：NORSTPKTNUM/CDNENHANCEDSW/QUICSASW/USRRLTRFRSHSW/QUICIDENFUNCEN/QUICPSRFUNCEN/SPECMETHODSW 均为可选参数 — 后果：缺省时使用默认值
- **单命令最大记录数 1**（warning，rule-0-00154）：SA 业务公共参数全局配置，单命令最大记录数 1 — 后果：超限 SET 失败
- **执行后立即生效**（warning，rule-0-00154）：本命令执行后立即生效 — 后果：误开 QUIC 增强影响非 Quic 业务性能
- **NORSTPKTNUM 过小风险**（warning，rule-0-00154）：NORSTPKTNUM 配置过小会导致报文解析提前返回失败、影响规则匹配 — 后果：默认 0=不开启此控制
- **QUIC 增强仅对密文业务有效**（warning，rule-0-00154）：QUICIDENFUNCEN/QUICPSRFUNCEN 只针对 Quic 密文业务增强 SA — 后果：对非 Quic 业务无效
