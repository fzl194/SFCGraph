# 增加HTTPS头增强（ADD TLSHEADEN）

> 命令静态知识见 [ADD TLSHEADEN](command/UDG/20.15.2/ADD-TLSHEADEN.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`ADD TLSHEADEN` 新建一个 HTTPS 头增强对象（HEADERENNAME），通过 DATATYPE 指定要插入的用户信息字段类型，**TLSTYPEVAL 标识 TLS 报文头域的 Extension 字段类型（HTTPS 独有，必配）**。**多字段插入需多次执行本命令**（同一 HEADERENNAME 下最多 40 种不同 DATATYPE；**不支持 MULTIPARA 拼接**，与 `ADD HEADEN` 差异）。**仅 PGW-U/UPF 适用**。

### 配置维度 1：头增强名（参数 HEADERENNAME）
| 取值 | 作用 | 约束 |
|---|---|---|
| 字符串 1~31 字符，无空格 | HTTPS 头增强唯一标识 | 必选；**无 PREFIXNAME 参数**（字段名不进报文，由 TLSTYPEVAL/SUBTLSTYPEVAL 在 Extension 字段内 TLV 编码） |

### 配置维度 2：数据类型（参数 DATATYPE，必选）
| 取值分组 | 代表取值 | 作用 |
|---|---|---|
| 用户标识类 | MSISDN1/2/3、IMSI1/2/3、IMEI1/2/3、MSIP1/2/3、USERPROFALIAS | 插入用户标识 |
| 网络信息类 | APN、SGSNIP1/2/3、GGSNIP1/2/3、UPIPV4、UPIPV6、ZONEID | 插入网络/接入信息 |
| 计费/会话类 | BILLINGTYPE、SESSIONID、CHGID、CHGCHAR1/2/3、MCC、MNC、ULI1/2/3 | 插入计费/会话属性 |
| 移动属性类 | RAT1/2/3、ROAMING1/2/3、SGSN_MCC_MNC1/2/3 | 插入移动属性 |
| 通用类 | USERDEF1~4（必配 USERDEFINEVAL）、TIMESTAMP1/2/3、SUBPROFILE1/2/3、RANDNUM | 自定义/时间戳/签约 |

**不支持 MULTIPARA1/2/3**（与 `ADD HEADEN` 差异）。

### 配置维度 3：TLS Type 值（参数 TLSTYPEVAL，必选，HTTPS 独有）
| 取值 | 作用 |
|---|---|
| 整数 0~65534 | TLS 报文头域的 Extension 字段类型；多字段时建议不同字段配不同 TLSTYPEVAL（便于 Web Server 区分） |

### 配置维度 4：Sub TLS Type 值（参数 SUBTLSTYPEVAL，HTTPS 独有）
| 取值 | 作用 |
|---|---|
| 整数 0~254 | 同一 TLSTYPEVAL 下用 Sub Type 区分多字段（取 10/20/30/40 递增） |

### 配置维度 5：加密算法（参数 ENCRYALGORI）
| 取值 | 作用 | 安全风险 |
|---|---|---|
| AES128_GCM（默认） / AES256_GCM / AES128_CTR / AES256_CTR | 对称加密（推荐） | 低 |
| AES128 / AES256（CBC） | 对称 CBC | 有安全风险，不建议 |
| SHA256 / MD5 | 哈希 | MD5 不建议 |
| RC4 | 对称流 | 不建议 |
| NONE | 不加密（可配软参 Byte837 对 MSISDN/IMSI 等做十六进制编码） | 有安全风险 |
| **不支持 RSA1024/2048/3072** | — | 需 RSA 加密改用 `ADD HEADEN` |

### 配置维度 6：防欺诈（参数 ANTIFRAUD）
| 取值 | 作用 |
|---|---|
| DISABLE（默认） | 不使能 |
| ENABLE | 插入前先检测/纠正；整机最多 32 个 TLS Type 启动防欺诈，新增检测项 30 秒内生效 |

### 配置维度 7：灰名单（参数 GRAYLIST）
| 取值 | 作用 |
|---|---|
| DISABLE（默认） | 关闭 |
| ENABLE | 只防欺诈不插入（配合 ANTIFRAUD=ENABLE） |

**典型脚本**：

```
// 4 字段不加密（不同 SUBTLSTYPEVAL 区分）
ADD TLSHEADEN:HEADERENNAME="header_test",DATATYPE=APN,TLSTYPEVAL=1000,SUBTLSTYPEVAL=10,ENCRYALGORI=NONE;
ADD TLSHEADEN:HEADERENNAME="header_test",DATATYPE=IMSI1,TLSTYPEVAL=1000,SUBTLSTYPEVAL=20,ENCRYALGORI=NONE;
// 单字段 AES128_GCM 加密
ADD TLSHEADEN:HEADERENNAME="headen1",DATATYPE=MSISDN1,TLSTYPEVAL=10000,ENCRYALGORI=AES128_GCM,PSWDKEY="XXXXXX",PSWDKEYCONFIRM="XXXXXX",ANTIFRAUD=DISABLE;
```

**步骤位置**：承接 [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链）+ HTTPS 协议识别（WELLKNOWNPORT 端口=443/SIGNATUREDB/PARSERDB）；本命令产出 HEADERENNAME，供下游 [0-00010](task/UDG/20.15.2/0-00010.md) RULE 经 `POLICYTYPE=HEADEN, POLICYNAME=<HEADERENNAME>` 直接引用（**不经 PCCPOLICYGRP**）。

## 决策点：TLS Type × 加密（DP 0-00292）

| 选项 | TLSTYPEVAL × SUBTLSTYPEVAL | ENCRYALGORI | 适用 |
|---|---|---|---|
| 多字段共享 TLSTYPEVAL | 同 TLSTYPEVAL + 不同 SUBTLSTYPEVAL | 按需 | 字段少、激活演示场景 |
| 多字段独立 TLSTYPEVAL | 每字段独立 TLSTYPEVAL | 按需 | Web Server 易于区分 |
| 推荐 GCM | — | AES128_GCM / AES256_GCM | 生产环境默认 |
| 无 RSA | — | 不支持 RSA | 需 RSA 改 HTTP 头增强 |
| 无 MULTIPARA | — | — | 需拼接改 HTTP 头增强 |

## 约束

- **rule-0-00292-1**（critical）：TLSTYPEVAL 必配（HTTPS 独有）；多字段同 TLSTYPEVAL 时须配 SUBTLSTYPEVAL 区分 — 后果：缺配则字段无法在 TLS Extension 中定位
- **rule-0-00292-2**（critical）：不支持 MULTIPARA 拼接（与 `ADD HEADEN` 差异）— 后果：需拼接改 HTTP 头增强
- **rule-0-00292-3**（critical）：不支持 RSA 加密（与 `ADD HEADEN` 差异）— 后果：需 RSA 改 HTTP 头增强
- **rule-0-00292-4**（critical）：单条 ADD TLSHEADEN 只能配一个 DATATYPE；同 HEADERENNAME 下最多 40 种不同 DATATYPE — 后果：超规格新增失败
- **rule-0-00292-5**（warning）：整机规格 ≤100 个不同 HEADERENNAME；≤500 个 TLSHeadEn 前缀字段；≤32 个 TLS Type 启动防欺诈；单报文 ≤16 个前缀做防欺诈 — 后果：超规格新增失败
- **rule-0-00292-6**（warning）：启用后用户**所有报文**都需 SA 处理 — 后果：SA 处理性能下降
- **rule-0-00292-7**（warning）：新增防欺诈检测项 30 秒内生效 — 时间窗口
- **rule-0-00292-8**（info）：插 BILLINGTYPE 需 SMF `SET PFCPPVTEXT:FEATURE=CHARGING_TYPE,SWITCH=ENABLE`；插 SESSIONID 需 `FEATURE=AUTH_CLASS,SWITCH=ENABLE`
- **rule-0-00292-9**（info）：USERPROFALIAS 插入总字符长度超 HEADERFLDLEN 配置值时，超长部分不做头增强插入
- **rule-0-00292-10**（info）：UDG 不支持 ZONEID 字段的 HTTPS 头增强插入
- **rule-0-00292-11**（info）：当用户 RAT 类型为 NR 时，不支持插入 SGSNIP 类型的字段
