# 增加HTTPS头增强（ADD TLSHEADEN）

> 命令静态知识见 [ADD TLSHEADEN](command/UDG/20.15.2/ADD-TLSHEADEN.md)

## 配置方法

新建一个 HTTPS 头增强对象（HEADERENNAME），通过 DATATYPE 指定要插入的用户信息字段类型，**TLSTYPEVAL 标识 TLS 报文头域的 Extension 字段类型（HTTPS 独有，必配）**。**多字段插入需多次执行本命令**（同一 HEADERENNAME 下最多 40 种不同 DATATYPE；**不支持 MULTIPARA 拼接**，与 ADD HEADEN 差异）。加密/防欺诈/灰名单/编码按需配。

| 参数 | 取值样例 | 说明 |
|---|---|---|
| HEADERENNAME | `header_test` | 头增强名称；1~31 字符无空格；**注意：HTTPS 头增强无 PREFIXNAME 参数**（字段名不进报文，由 TLSTYPEVAL/SUBTLSTYPEVAL 在 Extension 字段内 TLV 编码）|
| DATATYPE | `APN` / `IMSI1` / `MSIP1` / `MSISDN1` / `IMEI1` / `SGSNIP1` / `GGSNIP1` / `RAT1` / `ULI1` / `CHGCHAR1` / `ROAMING1` / `USERDEF1` / `TIMESTAMP1` / `UPIPV4` / `UPIPV6` / `RANDNUM` / `MCC` / `MNC` / `SESSIONID` / `USERPROFALIAS` / `SUBPROFILE1` / `BILLINGTYPE` / `CHGID` / `ZONEID` 等 | 数据类型（要插入的用户信息）；**不支持 MULTIPARA1/2/3**（与 ADD HEADEN 差异）；同一 HEADERENNAME 多字段需多次执行 |
| **TLSTYPEVAL** | `1000` / `10000` | **TLS 报文头域的 TLS Type 值（HTTPS 独有必配）**；整数 0~65534；插入多字段时建议不同字段配不同值（便于 Web Server 区分）|
| **SUBTLSTYPEVAL** | `10` / `20` / `30` / `40` | **TLS 报文头域的 Sub TLS Type 值（HTTPS 独有）**；整数 0~254；若多字段同 TLSTYPEVAL 时需配本参数区分（取值根据 Web Server 能力规划）|
| USERDEFINEVAL | `XXXXXX` | DATATYPE=USERDEF1/2/3/4 时必配；用户自定义字段值；1~63 字符（支持空格）|
| IMEITYPE | `IMEI`（默认）/ `TAC` | DATATYPE=IMEI1/2/3 时可选；IMEI=插入完整 IMEI，TAC=仅插入 TAC |
| REMOVEDMCC | `86` | DATATYPE=MSISDN1/2/3 时可选；插入 MSISDN 时需去除的国家码；空=不去除 |
| CCFORMAT | `DECIMAL`（默认）/ `HEX` | DATATYPE=CHGCHAR1/2/3 时可选；charge characteristic 格式 |
| TSPRECISION | `SECOND` / `MILLISECOND` | DATATYPE=TIMESTAMP1/2/3 时可选；时间精度 |
| HEADERFLDLEN | `31`（默认，范围 31~255）| DATATYPE=USERPROFALIAS 时可选；插入项总长度（含前缀名/冒号/空格/加号/结束符）|
| ENCRYALGORI | `NONE` / `MD5` / `RC4` / `AES128` / `AES256` / `SHA256` / `AES128_GCM` / `AES128_CTR` / `AES256_GCM` / `AES256_CTR` | 加密算法；默认 AES128_GCM；**不支持 RSA1024/RSA2048/RSA3072**（与 ADD HEADEN 差异，HTTPS 无 RSA）；MD5/RC4/AES128(CBC)/AES256(CBC)/NONE 有安全风险不建议 |
| PSWDKEY / PSWDKEYCONFIRM | `XXXXXX` | 加密密钥（ENCRYALGORI≠NONE 时必配）；两参数须一致；MD5:1~16、RC4:1~255、AES128/AES128_GCM/AES128_CTR:1~16、AES256/AES256_GCM/AES256_CTR:1~32、SHA256:1~255 |
| CIPHERTXTENCODE | `INHERIT`（默认）/ `NONE` / `BASE64` / `ASCII` | 密文编码方式；INHERIT 由 SET BASE64 + 软参 BIT475 控（base64/ASCII）|
| **ANTIFRAUD** | `ENABLE` / `DISABLE`（默认） | **防欺诈标识（HTTPS 支持，与 HTTP 一致；RTSP 不支持）**；ENABLE=使能头防欺诈（插入前先检测/纠正已有错误字段）|
| GRAYLIST | `ENABLE` / `DISABLE`（默认） | 灰名单；ENABLE=**只防欺诈不插入头增强字段**（配合 ANTIFRAUD=ENABLE）|
| SALTSW | `ENABLE`（默认）/ `DISABLE` | 是否加盐（仅 MD5/SHA256 可配）；建议 ENABLE |
| RC4KEYMD5EN | `INHERIT`（默认）/ `ENABLE` / `DISABLE` | ENCRYALGORI=RC4 时可选；INHERIT 由 SET HEADENGLBPARA 控 |
| HEADENEQ*/HEADENSL*/HEADENPL* | 替换字符串 | CIPHERTXTENCODE=BASE64 时可选；等号/斜杠/加号替换字符 |
| CFGDOMAINNAME | 字符串 1~31 | 可选；命令所属公共配置域名称 |

**典型脚本（基于 SNI 触发的标准部署：4 字段不加密 + 不同 SUBTLSTYPEVAL，activation 原样）**：
```
ADD TLSHEADEN:HEADERENNAME="header_test",DATATYPE=APN,TLSTYPEVAL=1000,SUBTLSTYPEVAL=10,ENCRYALGORI=NONE;
ADD TLSHEADEN:HEADERENNAME="header_test",DATATYPE=IMSI1,TLSTYPEVAL=1000,SUBTLSTYPEVAL=20,ENCRYALGORI=NONE;
ADD TLSHEADEN:HEADERENNAME="header_test",DATATYPE=MSIP1,TLSTYPEVAL=1000,SUBTLSTYPEVAL=30,ENCRYALGORI=NONE;
ADD TLSHEADEN:HEADERENNAME="header_test",DATATYPE=MSISDN1,TLSTYPEVAL=1000,SUBTLSTYPEVAL=40,ENCRYALGORI=NONE;
```

**变体脚本（单字段 + AES128_GCM 加密 + TLS Type=10000，命令 wiki 使用实例）**：
```
ADD TLSHEADEN:HEADERENNAME="headen1",DATATYPE=MSISDN1,TLSTYPEVAL=10000,ENCRYALGORI=AES128_GCM,PSWDKEY="XXXXXX",PSWDKEYCONFIRM="XXXXXX",ANTIFRAUD=DISABLE;
```

**步骤位置**：承接 [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链，产出 FLOWFILTERNAME）+ HTTPS 协议识别（WELLKNOWNPORT 端口=443/SIGNATUREDB/PARSERDB）；本命令产出 HEADERENNAME，供下游 [0-00010](task/UDG/20.15.2/0-00010.md) RULE 经 `POLICYTYPE=HEADEN, POLICYNAME=<HEADERENNAME>` 直接引用（**不经 PCCPOLICYGRP**）。

## 决策点

**DP（阶段级）**：TLS Type 规划 × 加密算法 × 编码方式（联动特性级 feature DP）。

| 选项 | 取值 | 联动 |
|---|---|---|
| 多字段同 TLSTYPEVAL | TLSTYPEVAL 相同 + **SUBTLSTYPEVAL 不同** | 不同字段共享 TLS Type，靠 Sub Type 区分（activation 4 字段演示：TLSTYPEVAL=1000 + SUB=10/20/30/40）|
| 多字段不同 TLSTYPEVAL | 每字段**独立 TLSTYPEVAL** | 便于 Web Server 区分（产品文档建议）|
| 不加密 | ENCRYALGORI=NONE | 无 PSWDKEY；**可配软参 Byte837 对 MSISDN/IMSI/SGSNIP/IMEI/MSIP/GGSNIP/TIMESTAMP 做十六进制编码**（HTTPS 独有）|
| 对称加密（推荐安全）| ENCRYALGORI=AES128_GCM/AES256_GCM/AES128_CTR/AES256_CTR | 配 PSWDKEY；SET AES128/SET AES256 配 IV 值（HEADENRANDOMIV=ENABLE 支持随机 IV）|
| 对称加密（CBC，有风险）| ENCRYALGORI=AES128/AES256 | 不建议；同上 |
| 哈希 | ENCRYALGORI=MD5/SHA256 | 配 PSWDKEY+SALTSW；MD5 不建议；RANDNUM 可作盐 |
| **非对称 RSA（HTTPS 不支持）** | — | **HTTPS 头增强不支持 RSA1024/RSA2048/RSA3072**（与 ADD HEADEN 差异）；需 RSA 加密改用 HTTP 头增强（ADD HEADEN）|
| 编码继承（默认）| CIPHERTXTENCODE=INHERIT | SET BASE64 控 base64；软参 BIT475 控 ASCII |
| 编码显式 base64 | CIPHERTXTENCODE=BASE64 | 命令内直接开 base64；可配等号/斜杠/加号替换字符 |
| 编码显式 ASCII | CIPHERTXTENCODE=ASCII | 命令内直接开 ASCII |

## 约束

- **TLSTYPEVAL 必配（HTTPS 独有 critical）**：开启 HTTPS 头增强必须配 TLSTYPEVAL；多字段同 TLSTYPEVAL 时须配 SUBTLSTYPEVAL 区分 — 后果：缺配则字段无法在 TLS Extension 中定位。
- **不支持 MULTIPARA 拼接**（info，与 ADD HEADEN 差异）：HTTPS 头增强不支持 MULTIPARA1/2/3 — 后果：需拼接场景无法实现，改 HTTP 头增强（ADD HEADEN）。
- **不支持 RSA 加密**（info，与 ADD HEADEN 差异）：HTTPS 头增强 ENCRYALGORI 不支持 RSA* — 后果：需 RSA 加密改用 HTTP 头增强。
- **多字段多次执行**（info）：单条 ADD TLSHEADEN 只能配一个 DATATYPE；同 HEADERENNAME 下最多 40 种不同 DATATYPE — 后果：超规格新增失败。
- **整机规格**（warning）：≤100 个不同名称的 HTTPS 头增强实例；≤500 个 TLSHeadEn 前缀字段；≤32 个 TLS Type 启动防欺诈检测；单报文 ≤16 个不同前缀字段做防欺诈检测；新增防欺诈检测项 30 秒内生效 — 后果：超规格新增失败。
- **所有报文走 SA 处理**（warning）：启用 HTTPS 头增强后用户**所有报文**都需 SA 处理，影响 SA 处理性能 — 后果：系统性能下降。
- **加密密钥强度**（warning）：建议 ≥8 字符且含大小写/数字/特殊字符 ≥2 类 — 后果：弱密钥有安全风险。
- **BILLINGTYPE/SESSIONID 前置**（info）：插 BILLINGTYPE 需 SMF `SET PFCPPVTEXT:FEATURE=CHARGING_TYPE,SWITCH=ENABLE`；插 SESSIONID 需 `FEATURE=AUTH_CLASS,SWITCH=ENABLE`。
- **USERPROFALIAS 长度截断**（info）：插入 USERPROFALIAS 总字符长度超 HEADERFLDLEN 配置值时，超长部分不做头增强插入。
- **加密仅作用于字段取值**（info）：字段名（TLS Type 标识）不加密，属预期设计。
