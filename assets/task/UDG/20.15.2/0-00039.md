# 配置知名端口（ADD WELLKNOWNPORT）
> 命令静态知识见 [ADD WELLKNOWNPORT](command/UDG/20.15.2/ADD-WELLKNOWNPORT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

为某协议绑定一个知名端口（或端口段）做基于端口的协议识别，供下游 L7 业务（QoS/ADC/HTTPS识别/SA-Basic）引用 PROTOCOLNAME。仅 L7 业务触发 QoS 场景必配；执行前必须先 LOD SIGNATUREDB 加载协议识别特征库。

| 参数 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| IDENPROTNAME（知名端口名） | http / port_dns | 本端规划 | 1~63 字符串 |
| PROTOCOLNAME（协议名） | http / dns | 本端规划 | 1~31 字符串，须 SubProtocol 表已存在 |
| PORTOP（端口范围操作码） | EQUAL / RANGE | 本端规划 | EQUAL=单端口，RANGE=端口段 |
| STARTPORT（起始端口号） | 80 / 53 | 本端规划 | 0~65535 |
| ENDPORT（结束端口号） | （PORTOP=RANGE 时填） | 本端规划 | 0~65535，单端口场景不配 |
| PRIORITY（优先级） | 5 / 20 | 本端规划 | 1~65535，建议按 5/10/15 阶梯便于后续插入 |

典型脚本样例：
```
# 单端口（HTTP=80，GWFD-020358 七层 QoS）
ADD WELLKNOWNPORT:IDENPROTNAME="http",PROTOCOLNAME="http",PORTOP=EQUAL,STARTPORT=80,PRIORITY=5;

# DNS 知名端口（GWFD-110203 HTTPS 地址识别）
ADD WELLKNOWNPORT:IDENPROTNAME="port_dns",PROTOCOLNAME="dns",PORTOP=EQUAL,STARTPORT=53,PRIORITY=20;
```

配置要点：本 atom 当前主场景为单端口（PORTOP=EQUAL，仅 STARTPORT），ENDPORT 在端口段场景（PORTOP=RANGE）才必填。配置 PROTOCOLNAME 前须先通过 LOD SIGNATUREDB 加载特征库且 SubProtocol 表存在该协议。多协议端口重叠时按 PRIORITY 匹配。该命令通常位于 ADD QOSPROP / ADD PCCPOLICYGRP 之后、ADD SADEDICBEARER 之前（七层 QoS 流序）。

## 约束
- 最大记录数 200 / 立即生效 · warning · 超规格新增失败
- 前置：执行前须 LOD SIGNATUREDB 加载协议识别特征库 · critical · 否则命令拒收、协议识别失效（见 [[UDG@20.15.2@TaskRule@0-00280]]）
- PROTOCOLNAME 配置时 SubProtocol 表记录必须存在 · critical · 否则 ADD 失败
- 端口段：PORTOP=EQUAL 仅 STARTPORT 必填；PORTOP=RANGE 时 STARTPORT+ENDPORT 均必填 · warning · 缺失则端口匹配失效
- 适用 NF：PGW-U、UPF
