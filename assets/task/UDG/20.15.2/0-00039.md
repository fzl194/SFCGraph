# 配置知名端口（ADD WELLKNOWNPORT）

> 命令静态知识见 [ADD WELLKNOWNPORT](command/UDG/20.15.2/ADD-WELLKNOWNPORT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD WELLKNOWNPORT 是七层过滤的"知名端口锚点定义层"：用一个协议名 + 端口范围 + 优先级描述一个常用端口（http=80, dns=53 等），产出 IDENPROTNAME 供 ADD WELLKNOWNPORTBIND 等场景引用。命令核心配置空间有 3 个维度：**端口范围操作**（PORTOP 决定是单值 EQUAL 还是范围 GT/LT/RANGE）、**端口号**（STARTPORT/ENDPORT 条件必选）、**优先级**（PRIORITY 必选）。仅 1 个特性命中（020358 七层 QoS 范式，演示 http=80）。

### 配置维度 1：端口范围操作（参数 PORTOP，必选）

| 取值 | 作用 | 必选联动 |
|---|---|---|
| **EQUAL** | 等于指定值（单端口） | STARTPORT |
| **GT** | 大于指定值 | STARTPORT |
| **LT** | 小于指定值 | ENDPORT |
| **RANGE** | 区间（含两端） | STARTPORT + ENDPORT |

### 配置维度 2：端口号（参数 STARTPORT/ENDPORT，条件必选）

| 参数 | 条件 | 作用 |
|---|---|---|
| **STARTPORT** | PORTOP=EQUAL/GT/RANGE 时必选 | 起始端口号（含）/ 比较基准 |
| **ENDPORT** | PORTOP=LT/RANGE 时必选 | 结束端口号（含） |

### 配置维度 3：优先级（参数 PRIORITY，必选）

| 取值范围 | 作用 |
|---|---|
| 1~65535 整数 | 端口匹配优先级，命令 notes 建议按 5、10、15...5 的步进递增（值越大优先级越低） |

**典型脚本（020358 业务触发的 QoS 保证，http=80 EQUAL 范式）**：
```
ADD WELLKNOWNPORT: IDENPROTNAME="http",PROTOCOLNAME="http",PORTOP=EQUAL,STARTPORT=80,PRIORITY=5;
```

> 命令 notes：适用 NF：PGW-U、UPF；最大记录数 / 协议名规则未明示，需查命令 wiki 完整参数表；IDENPROTNAME 字符串 1~63，PROTOCOLNAME 字符串 1~31。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：知名端口定义"——前置 ADD PROTOCOL（PROTOCOLNAME 来源）；产出 IDENPROTNAME 供引用方使用。

## 决策点

（无。本命令 PORTOP 决定端口取值结构，是参数级条件必选；4 个 PORTOP 取值是"必选子分支"而非"特性级决策点"，无独立 DP。）

## 约束

- **PORTOP 决定端口号必选集**（critical，rule-0-00339）：EQUAL/GT 必配 STARTPORT；LT 必配 ENDPORT；RANGE 必配 STARTPORT+ENDPORT — 后果：缺端口号命令拒收
- **PROTOCOLNAME 引用前置依赖**（critical，rule-0-00339）：PROTOCOLNAME 须先建（命令 wiki 协议注册流程） — 后果：未建引用失败
- **IDENPROTNAME 全网唯一**（critical，隐含）：不同知名端口之间 IDENPROTNAME 不能重复 — 后果：重名 ADD 失败
- **PRIORITY 范围 1~65535**（warning，rule-0-00339）：命令 notes 建议按 5 的步进 — 后果：超出范围被拒
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
