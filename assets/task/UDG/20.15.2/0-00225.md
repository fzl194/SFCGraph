# 创建TWAMP探测逻辑接口（ADD TWAMPLOGICINF）

> 命令静态知识见 [ADD TWAMPLOGICINF](command/UDG/20.15.2/ADD-TWAMPLOGICINF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`ADD TWAMPLOGICINF` 创建本端 TWAMP 探测报文收发用的逻辑接口，是 TWAMP 探测链路搭建的第 2 步。**Full / Light 两模式共用**同一命令，区别仅在接口名/本端 IP（Full 用 N3/S1-U 口、Light 用业务口）。IPV4ADDRESS 须被下游 TWAMPRESPONDER（Full）/ TWAMPCLIENT（Light）的 LOCALIPV4 引用且一致。

### 配置维度 1：逻辑接口名（参数 NAME）
| 取值 | 作用 |
|---|---|
| 字符串 1~63 字符（按 N3/S1-U 接口编号命名，如 `n3if1/1/0`/`n3if1/1/1`） | 标识 TWAMP 探测逻辑接口 |

### 配置维度 2：地址族类型（参数 AFTYPE）
| 取值 | 作用 | 配套 |
|---|---|---|
| IPV4 | 地址族为 IPv4 | 必带 IPV4ADDRESS+IPV4MASK |
| IPV6 | 地址族为 IPv6（TWAMP 探测链路主流为 IPv4） | 必带 IPV6ADDRESS+IPV6MASK |

### 配置维度 3：IPV4ADDRESS/MASK（AFTYPE=IPV4 时条件必选）
| 参数 | 取值 | 作用 |
|---|---|---|
| IPV4ADDRESS | IPv4 点分十进制（如 `10.1.1.10`） | 逻辑接口地址，必须与业务 N3/S1-U 接口 IP 一致 |
| IPV4MASK | IPv4 掩码（TWAMP 探测用 32 位主机掩码） | 固定 `255.255.255.255` |

### 配置维度 4：VPN 实例名（参数 VRFNAME）
| 取值 | 作用 |
|---|---|
| 引用前置 [ADD TWAMPVPNINST](task/UDG/20.15.2/0-00224.md) 产出的 VRFNAME（如 `VPN_CN`） | 探测报文所属 VPN，必须与 0-00224 一致 |

### 配置维度 5：是否共享业务 IP（参数 SHAREDTYPE）
| 取值 | 作用 |
|---|---|
| TRUE | 与业务 N3/S1-U 接口共用同一 IP（TWAMP 探测主流场景） |
| FALSE | TWAMP 探测使用独立 IP（极少场景） |

## 决策点

（无。本命令在所有特性下用法一致：单一逻辑接口维度，无独立 DP 分支。）

## 约束

- **rule-0-00414**（critical）：VRFNAME 须与上游 [ADD TWAMPVPNINST](task/UDG/20.15.2/0-00224.md) 一致 — 不一致导致探测报文路由失败
- **rule-0-00415**（critical）：AFTYPE=IPV4 时 IPV4ADDRESS/IPV4MASK 必须同时配齐；AFTYPE=IPV6 时 IPV6ADDRESS/IPV6MASK 必须同时配齐 — 条件必选参数缺配被拒
- **rule-0-00416**（critical）：IPV4ADDRESS 须被下游 [ADD TWAMPRESPONDER](task/UDG/20.15.2/0-00226.md).LOCALIPV4（Full 模式）或 [ADD TWAMPCLIENT](task/UDG/20.15.2/0-00227.md).LOCALIPV4（Light 模式）引用且完全一致 — 不一致导致探测报文收发错位
- **rule-0-00417**（warning）：与业务 N3/S1-U 共用 IP 时 SHAREDTYPE 必须 = TRUE — 探测报文与业务报文混发，可能影响业务 QoS
