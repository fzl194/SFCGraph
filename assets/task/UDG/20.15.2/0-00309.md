# 添加LNS与L2TP组绑定关系（ADD L2TPLNSINFO）

> 命令静态知识见 [ADD L2TPLNSINFO](command/UDG/20.15.2/ADD-L2TPLNSINFO.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。GROUPID 引用 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 的 L2TP 组号；同组内 LNS 信息与 ADD L2TPGROUP 内置的 6 个 LNS 合计不超过 30 个；本命令解决 LNS 超过 6 个的扩展场景。

## 配置方法（配置方法字典）

ADD L2TPLNSINFO 是"L2TP 组内 LNS 信息添加"命令。3 个必选参数（GROUPID + LNSNO + LNSIPVER）+ 2 个条件必选（IPV4ADDRESS/IPV6ADDRESS 二选一）+ 2 个可选（PWD + CFMPWD 双确认）。配置空间有 4 个维度：**L2TP 组**（GROUPID 引已建 L2TP 组）、**LNS 序号**（LNSNO 1~30 决定 LNS 在组内位置）、**LNS IP 版本**（LNSIPVER=IPV4/IPV6 决定地址参数）、**隧道认证密码**（PWD+CFMPWD 双确认可选）。

### 配置维度 1：L2TP 组（参数 GROUPID，必选）

| 取值形式 | 来源 | 同组合并上限 |
|---|---|---|
| **整数 1~1500** | 引用 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 已建的 L2TP 组号 | 同组 LNS 总数 ≤ 30（ADD L2TPGROUP 内置 + 本命令扩展） |

> **前置铁律**：GROUPID 必须先经 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 创建；未建组引用本命令将失败。同组内 LNSIPVER 不可混用——一旦组内已有 IPv4 LNS，本命令只允许配置 IPv4 LNS，反之亦然（不支持 IPv4 LNS 与 IPv6 LNS 共存）。

### 配置维度 2：LNS 序号（参数 LNSNO，必选）

| 取值形式 | 与 ADD L2TPGROUP 内置序号的冲突 |
|---|---|
| **整数 1~30** | 不允许与 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 的内置 LNS IP 序号相同（如 ADD L2TPGROUP 已配 FIRSTLNSIP，则 LNSNO=1 不可再用） |

> **序号冲突铁律**：LNSNO 与同组 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 的 LNS IP 序号不能相同；这是 ADD/MOD L2TPGROUP（★R1.5 atom 待补） 与 ADD L2TPLNSINFO 共存场景下的硬约束。

### 配置维度 3：LNS IP 版本（参数 LNSIPVER，必选，二选一固定）

| 取值 | 配套地址参数 | 适用场景 |
|---|---|---|
| **IPV4** | `IPV4ADDRESS`（必选，IPv4 地址格式 X.X.X.X） | 单栈 L2TPv4 场景 |
| **IPV6** | `IPV6ADDRESS`（必选，IPv6 地址格式 X:X:X:X:X:X:X:X） | 单栈 L2TPv6 场景 |

> **IPv4/IPv6 不共存铁律**：同 L2TP 组内不允许 IPv4 LNS 与 IPv6 LNS 共存——ADD/MOD L2TPGROUP（★R1.5 atom 待补） 与 ADD L2TPLNSINFO 共享此约束，整个组只能走 IPv4 或 IPv6 单链路族。
> **主备模式固定**：LNSIPVER=IPV6 + 主备模式下 LNSNO 只能为 1（主）或 2（备）；IPV4 主备模式仅允许 ADD/MOD L2TPGROUP（★R1.5 atom 待补） 配置，不允许 ADD L2TPLNSINFO 配 IPv4。

### 配置维度 4：隧道认证密码（参数 PWD + CFMPWD，可选，条件双确认）

| 参数 | 取值形式 | 配套关系 |
|---|---|---|
| **PWD**（隧道认证密码） | 字符串 1~432（密码类型，不支持空格，区分大小写） | 不加密输入；与 CFMPWD 同时配且必须相同 |
| **CFMPWD**（确认密码） | 字符串 1~432（密码类型，不支持空格，区分大小写） | 必须与 PWD 完全相同 |

> **双确认铁律**：输入 PWD 时必须同时输入 CFMPWD，且两值相同——任一缺失或不一致命令拒收；这是 LNS 端隧道协商密码的安全配对约束。

**典型脚本（IPV4 主 LNS，LNSNO=2 与 ADD L2TPGROUP 内置 LNSNO=1 区分）**：
```
ADD L2TPLNSINFO:GROUPID=1,LNSNO=2,LNSIPVER=IPV4,IPV4ADDRESS="10.1.1.0";
```

**典型脚本（IPV6 主备模式，主 LNSNO=1）**：
```
ADD L2TPLNSINFO:GROUPID=1,LNSNO=1,LNSIPVER=IPV6,IPV6ADDRESS="2001:db8::1",PWD="secret123",CFMPWD="secret123";
```

> 命令 notes：**最大记录数 45000**（系统全局 L2TP 绑定关系上限）；**执行后立即生效**；**前置依赖**：必须先经 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 建组；**同组 LNS 上限**：每个 L2TP 组最多绑定 30 个 LNSINFO（含 ADD L2TPGROUP 内置 6 个 + 本命令扩展）；**IP 不重复**：同组 LNS IP 地址不允许重复；**IPv4/IPv6 不共存**：同组内不支持 IPv4 LNS 与 IPv6 LNS 共存；**序号冲突**：LNSNO 不能与 ADD L2TPGROUP 的内置 LNS 序号相同；**主备模式约束**：仅 IPv6 主备模式可用 ADD L2TPLNSINFO（IPv4 主备仅允许 ADD/MOD L2TPGROUP（★R1.5 atom 待补）），且 LNSNO 只能为 1（主）或 2（备）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：扩展 L2TP 组 LNS 上限"——前置 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP（建组 + 内置 ≤6 LNS）+ [0-00131](task/UDG/20.15.2/0-00131.md) SET GLOBALL2TP（全局 L2TP 缺省属性）+ [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR（APN 级 L2TP 启用）；下游 [MOD L2TPLNSINFO（★R1.5 atom 待补）](command/UDG/20.15.2/MOD-L2TPLNSINFO.md)（修改密码/IP）+ [RMV L2TPLNSINFO（★R1.5 atom 待补）](command/UDG/20.15.2/RMV-L2TPLNSINFO.md)（删除 LNS）；本命令解决 L2TP 接入用户超过 6 个 LNS 场景的扩展。

## 决策点：LNS 配置场景选择（DP 0-00309）

问：本 LNS 信息属于哪种配置场景？（决定 LNSNO + LNSIPVER + 是否使用 PWD/CFMPWD）

| 选项 | LNSIPVER | LNSNO | PWD/CFMPWD | 典型场景 |
|---|---|---|---|---|
| **IPV4 扩展 LNS**（与 ADD L2TPGROUP 内置同链路族） | IPV4 | 2~30（避开 ADD L2TPGROUP 已用序号） | 可选（按对端协商决定） | 命令示例 `GROUPID=1,LNSNO=2`；L2TPv4 单栈扩展 |
| **IPV6 扩展 LNS** | IPV6 | 2~30（避开 ADD L2TPGROUP 已用序号） | 可选（按对端协商决定） | L2TPv6 单栈扩展 |
| **IPV6 主 LNS（主备模式）** | IPV6 | **固定 1** | 可选（按对端协商决定） | IPV6 主备模式主端 |
| **IPV6 备 LNS（主备模式）** | IPV6 | **固定 2** | 可选（按对端协商决定） | IPV6 主备模式备端 |
| **带隧道认证密码的 LNS** | IPV4 或 IPV6 | 按场景 | **必填 PWD+CFMPWD 双确认** | 对端需要隧道认证场景 |

> 5 种场景均要求前置 [0-00133](task/UDG/20.15.2/0-00133.md) 已建同 GROUPID；同组 LNS 总数（含本命令扩展 + ADD L2TPGROUP 内置）≤ 30；IPV4/IPV6 在同组内不共存。

## 约束

- **最大记录数 45000**（warning，rule-0-00309）：系统全局 L2TP 绑定关系最大记录数 45000 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00309）：命令执行后立即生效 — 后果：误操作即刻生效
- **前置依赖：必须先建 L2TP 组**（critical，rule-0-00309）：必须先经 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 配置 L2TP 组信息 — 后果：未建组引用命令拒收
- **同组 LNS 总数 ≤ 30**（critical，rule-0-00309）：每个 L2TP 组最多 30 个 LNS（ADD L2TPGROUP 内置 + 本命令） — 后果：超上限新增失败
- **同组 LNS IP 不重复**（critical，rule-0-00309）：使用 ADD L2TPLNSINFO 或 ADD L2TPGROUP 配置的同组 LNS IP 不能重复 — 后果：IP 冲突命令拒收
- **IPv4/IPv6 不共存**（critical，rule-0-00309）：同组内不支持 IPv4 LNS 与 IPv6 LNS 共存 — 后果：混用链路族命令拒收
- **LNSNO 序号冲突**（critical，rule-0-00309）：LNSNO 不能与同组 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 内置 LNS IP 序号相同 — 后果：序号冲突命令拒收
- **主备模式：仅 IPv6 LNS**（critical，rule-0-00309）：主备模式下 IPv4 LNS 仅允许 ADD/MOD L2TPGROUP（★R1.5 atom 待补） 配置；ADD L2TPLNSINFO 仅允许 IPv6 — 后果：IPv4 主备场景下用本命令配 LNS 失败
- **主备模式：LNSNO 固定 1/2**（critical，rule-0-00309）：IPv6 主备模式下 LNSNO 只能配置 1（主）或 2（备） — 后果：LNSNO=3+ 在主备模式命令拒收
- **PWD+CFMPWD 双确认铁律**（critical，rule-0-00309）：输入 PWD 必须同时输入 CFMPWD，且两值相同 — 后果：缺失或不一致命令拒收
- **密码长度**（info，rule-0-00309）：PWD/CFMPWD 字符串 1~432（密码类型，不支持空格，区分大小写） — 后果：超长或含空格命令拒收
- **GROUPID 范围**（info，rule-0-00309）：整数 1~1500 — 后果：超范围命令拒收
- **LNSNO 范围**（info，rule-0-00309）：整数 1~30 — 后果：超范围命令拒收
- **权限要求**（info，rule-0-00309）：需 G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
