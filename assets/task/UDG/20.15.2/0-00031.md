# 配置ADC参数（ADD ADCPARA）
> 命令静态知识见 [ADD ADCPARA](command/UDG/20.15.2/ADD-ADCPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

每个待检测应用流过滤器配一条 ADC 参数：绑定流过滤器 + 开关流信息上报 + （可选）延迟上报。仅当 PCRF/PCF 经预定义规则下发 APPLICATION_START/STOP Event-Trigger 时才需要配置。

| 参数 | 必选 | 取值样例 | 数据来源 | 语义 |
|---|---|---|---|---|
| FLOWFILTERNAME（流过滤器名称） | required | flowfilter_ADC01 | reference → [0-00007 配置流过滤器](0-00007.md) 的 FLOWFILTERNAME | 检测目标 |
| FLOWINFORPT（流信息上报开关） | optional | ENABLE | 本端规划 | PCRF/PCF 需获取流信息时 ENABLE；缺省 DISABLE |
| ADCHYSTIMER（应用级迟滞时间 秒） | optional | 0 | 全网规划 | 0=关闭延迟上报；Start/Stop 过频时调大延迟上报（0~3600） |

脚本样例（增强的 ADC 基本功能）：
```
ADD ADCPARA:FLOWFILTERNAME="flowfilter_ADC01",FLOWINFORPT=ENABLE,ADCHYSTIMER=0;
```

配置要点：
- FLOWINFORPT 是核心开关：UDG 是否把"应用↔流"对应关系随 APPLICATION_START/STOP 上报给 PCRF/PCF，决定后续策略精度。
- ADCHYSTIMER 控制 PGW-U/UPF 向 PGW-C/SMF 上报事件的迟滞；调大可抑制频繁 Start/Stop 抖动，但代价是事件延迟。
- 关键流检测（KEYFLOWDETECTSW/KEYFLOWTIME/KEYFLOWSPEED）与重定向（REDIRECTNAME）仅 1 特性未用到，属冷门场景，通用配置不绑。
