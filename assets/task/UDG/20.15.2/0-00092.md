# 加载外置OTT业务规则库（LOD EXTERNALDB）

> 命令静态知识见 [LOD EXTERNALDB](command/UDG/20.15.2/LOD-EXTERNALDB.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

LOD EXTERNALDB 是 GWFD-110321 支持内置百万级业务规则库特性专属的"外置规则库加载"命令：系统通过外部规则数据库中的规则文件进行匹配时，可通过无损升级数据库的方式定义大规模的匹配条件。本命令的核心配置空间有 2 个维度：**数据库类型 DBTYPE**（OTT 单枚举，本特性固定）、**版本号 VERSION**（按上传文件决定）。**加载/卸载两次执行间隔≥20 秒**；**加载后须经 SET REFRESHSRV 刷新生效**。

### 配置维度 1：数据库类型（参数 DBTYPE，必选）

| 取值 | 作用 | 适用特性 |
|---|---|---|
| **OTT** | OTT 业务规则数据库（百万级） | [GWFD-110321 支持内置百万级业务规则库](evidence/UDG/20.15.2/激活支持OTT业务规则库功能_05455409.md) |

> 单特性固定 OTT 取值；DBTYPE 还有其它枚举值（命令真相枚举），但仅 OTT 与本特性相关。

### 配置维度 2：版本号（参数 VERSION，必选，本端规划）

字符串 1~21 字符；形式如 `AAAA.BB.CC.DDDD.EEE`（如 `0020.01.0000.0000.001`），由本端运维按上传文件决定；前置 OTT 规则库须已上传至 UDG。

> **典型范式（GWFD-110321 加载 OTT 库）**：
> ```
> LOD EXTERNALDB:DBTYPE=OTT,VERSION="0020.01.0000.0000.001";
> ```

**步骤位置**：在所属 feature 编排中通常为"步骤 2：加载外置 OTT 规则库"——SET LICENSESWITCH（[0-00019](task/UDG/20.15.2/0-00019.md)）使能特性后、SET REFRESHSRV（[0-00015](task/UDG/20.15.2/0-00015.md)）刷新生效前。

## 决策点：无

（无。本命令在所有特性下用法一致，单一类型 OTT，无分支决策。）

## 约束

- **加载/卸载两次执行间隔 ≥20 秒**（warning，rule-0-00306）：规则库加载/卸载操作最小时间间隔 — 后果：间隔 <20s 拒收
- **加载过程系统负荷高、CPU 占用升高**（warning，隐含）：须评估性能后避开业务高峰 — 后果：高峰加载致 CPU 过高丢包
- **加载 OTT ISU 库后触发 FlowFilter 冲突检查**（info，隐含）：可 [DSP COLLISIONCHECK](command/UDG/20.15.2/DSP-COLLISIONCHECK.md) 查询 — 后果：冲突未查影响匹配正确性
- **加载后须 SET REFRESHSRV 刷新生效**（critical，隐含）：本命令加载规则库后业务流未刷新，须经 [SET REFRESHSRV](task/UDG/20.15.2/0-00015.md) 后置刷新 — 后果：未刷新致规则库加载但业务流匹配未生效
- **前置 OTT 规则库须已上传至 UDG 且 ESA VMPod 正常运行**（critical，隐含）：文件上传与 VM 运行时是加载前提 — 后果：前置不满足导致加载失败
- **支持无损升级不中断业务**（info，隐含）：升级过程中业务不中断 — 后果：升级窗口可业务正常运行
- **加载后用 DSP 命令检查成功**（info，隐含）：可 DSP EXTERNALDB 查询加载状态 — 后果：未检查加载失败状态
