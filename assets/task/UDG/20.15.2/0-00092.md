# 加载外置OTT业务规则库（LOD EXTERNALDB）

> 命令静态知识见 [LOD EXTERNALDB](command/UDG/20.15.2/LOD-EXTERNALDB.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

LOD EXTERNALDB 是 GWFD-110321 专属命令，加载外置 OTT 业务规则库。DBTYPE 单特性固定为 OTT（已收紧为 fixed），VERSION 由本端运维按上传文件决定。前置 OTT 规则库已上传至 UDG，后置须 SET REFRESHSRV 刷新生效。加载后须用 DSP 命令检查是否成功。

**主场景范式（从证据数据规划表归纳）**

| 字段 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| DBTYPE | OTT | 固定（本特性固定为 OTT） | 单特性单取值，无需 variable |
| VERSION | 0020.01.0000.0000.001 | 本端规划 | 外置规则库版本号，1~21 字符，形式如 AAAA.BB.CC.DDDD.EEE |

**脚本样例**
```
LOD EXTERNALDB: DBTYPE=OTT,VERSION="0020.01.0000.0000.001";
```

**配置链路**（GWFD-110321 步骤 1→2→3→4）：SET LICENSESWITCH → **LOD EXTERNALDB** → SET REFRESHSRV(0-00015) 刷新生效 → ADD FLOWFILTER 使能外部 OTT 规则库。

## 约束

- **rule 0-00306**（requires/warning）加载/卸载两次执行间隔≥20 秒；加载过程系统负荷高、CPU 占用升高，须评估性能后避开业务高峰；加载 OTT ISU 库后触发 FlowFilter 冲突检查（可 DSP COLLISIONCHECK 查询）；立即生效；加载后须 SET REFRESHSRV(0-00015) 刷新生效；前置 OTT 规则库须已上传至 UDG 且 ESA VMPod 正常运行；加载后用 DSP 命令检查成功；支持无损升级不中断业务 — 后果：间隔<20s 拒收；未做性能评估致高峰 CPU 过高丢包；冲突未查影响匹配正确性；未 REFRESHSRV 致规则库加载但业务流未刷新
