# MPLS VPN（GWFD-020411）

> 特性静态知识见 [GWFD-020411](feature/UDG/20.15.2/GWFD-020411.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-020411-MPLSVPN.md)。
>
> **本页定位**：配置生成实例化视角——MPLS VPN 完整配置体系。**UDG 作 PE**，承担 VRF 维护、MP-BGP、标签处理三大职责。9 篇产品文档**均无激活 MML 脚本**（区别于 GRE/IPSec 激活文档），下文配置流程基于原理推导，部分命令需 Stage 3 从 UDG 命令字典（VPN实例/BGP-VPN/LDP/MPLS-TE 命令族）补全。

## 配置概览

本特性按 **8 步通用骨架 + 6 类组网变体** 组织配置：

**对象链**：
```
MPLS VPN 体系（PE 角色）
┌──────────────────────────────────────────────────────────┐
│ VPN 实例（VRF）★核心                                      │
│  ├─ RD (8字节，例 100:1)                                  │
│  ├─ VPN Target ERT / IRT (32位扩展团体属性)              │
│  ├─ 地址族：IPv4 / IPv6 (6VPE)                            │
│  └─ 分标签方式：perInstance（每实例每标签，推荐）          │
│           │                                               │
│           │ 绑定 PE-CE 接口                                │
│           ▼                                               │
│ PE-CE 路由协议（静态/OSPF/BGP，方案 A/B/C）                │
│           │                                               │
│           │ MP-BGP PE-PE                                  │
│           ▼                                               │
│ BGP-VPNv4 / VPNv6 路由（12字节 VPN-IPv4 = RD+IPv4）        │
│           │                                               │
│           │ MPLS 转发                                     │
│           ▼                                               │
│ 两层标签：O-L（公网到 BGP 下一跳）+ I-L（VPN 出接口）       │
└──────────────────────────────────────────────────────────┘
```

**8 步通用骨架**：

| 步骤 | 干啥 | 主要命令 |
|---|---|---|
| 1. License 开关 | 打开 MPLS VPN License 开关 | `SET LICENSESWITCH` |
| 2. 骨干网基础 | IGP（OSPF/OSPFv3）路由可达 + LDP/RSVP-TE 建立 LSP | 已有 OSPF 骨架（[0-00074](task/UDG/20.15.2/0-00074.md)~）+ LDP 系列 |
| 3. VPN 实例（★核心） | 创建 VRF + RD + VPN Target + perInstance | 需新增 MPLS 专属 atom（MPLS-VPN-INSTANCE 系列，待 Stage 3 建） |
| 4. 接口绑 VPN 实例 | PE-CE 接口绑到 VPN 实例 | 复用 `ADD IPBINDVPN`（[0-00057](task/UDG/20.15.2/0-00057.md)）+ VPN 绑定扩展 |
| 5. PE-CE 路由协议 | 静态 / OSPF 多实例 / BGP（PE-CE） | 复用 OSPF（[0-00074](task/UDG/20.15.2/0-00074.md)）+ BGP 对等体族（待 Stage 3 建） |
| 6. MP-BGP | PE-PE VPNv4/VPNv6 对等体 + 地址族 + RR + perNexthop | 待 Stage 3 建 MP-BGP 专属 atom |
| 7. 组网变体 | Intranet/Extranet/Hub&Spoke/跨域A-B/MCE/6VPE | 通过 VPN Target 双 RT 控制 |
| 8. 可靠性 | VPN FRR / VPN GR Helper / VPN NSR | 待 Stage 3 建可靠性 atom |

**关键约束**：
- **必须配置 perInstance + perNexthop 节约标签**（特性规格 299008），需对端同步修改
- **VPN GR 仅支持 Helper**（不支持 Restarter）
- **VPN NSR 需双主控硬件**
- **PE-CE 接口手工绑定**到 VPN 实例（路由协议自身不能区分）
- **标签规格上限 299008**——大规模部署需规划分标签方式
- **需 License**（控制项编号 Stage 3 从 license 数据表补全）

## 配置流程

> 每步显示"干啥 + 含哪些命令"。**说明**：本特性 9 篇文档无激活 MML 脚本，下文命令级引用含「待 Stage 3 建」标记的为 MPLS-VPN 专属 atom（命令已存在于 `assets/command/` 但尚未建 task atom）；其余为已建通用 atom。

1. **License 开关**（MPLS VPN License 控制项）：
   - `SET LICENSESWITCH` → [0-00019](task/UDG/20.15.2/0-00019.md)（LICITEM="MPLS_VPN_控制项编号待补", SWITCH=ENABLE）

2. **骨干网 IGP + MPLS 基础**（非本特性专属，前置）：
   - 已有 OSPF 骨架：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFNETWORK` → [0-00074](task/UDG/20.15.2/0-00074.md) + [0-00075](task/UDG/20.15.2/0-00075.md) + [0-00076](task/UDG/20.15.2/0-00076.md)
   - LDP 全局配置 → `SET LDPGLOBAL`（待 Stage 3 建 atom）+ LDP 接口 → `ADD LDPIF`（待 Stage 3 建 atom）

3. **创建 VPN 实例（★核心使能）**：
   - `ADD VPNINSTANCE`（待 Stage 3 建 atom，命名空间：MPLS-VPN-INSTANCE）
     - NAME="vpn1"
     - RD="100:1"
     - VPN Target ERT="100:1"（Extension Community）
     - VPN Target IRT="100:1"
   - **地址族使能**：`ADD VPNINSTANCEAF` 或 `ADD VPNINSTAF` → [0-00072](task/UDG/20.15.2/0-00072.md)（AFTYPE=Ipv4uni，IPv6 VPN 场景为 Ipv6uni）
   - **分标签方式（★关键节约）**：VPN 实例地址族视图下配置 perInstance
     - `MOD VPNINSTANCE_AF_PERINSTANCE`（待 Stage 3 建）
   - 复用 → [0-00071](task/UDG/20.15.2/0-00071.md) L3VPNINST（用户面 VPN 实例，骨架可复用但 VRF 视图不同）

4. **PE-CE 接口绑定 VPN 实例**：
   - `ADD IPBINDVPN` → [0-00057](task/UDG/20.15.2/0-00057.md)（VRFNAME="vpn1"，复用现有 IPBINDVPN 逻辑，VRFNAME 绑定）
   - 或 `MOD INTERFACE` + `ADD VPNINSTANCEBINDIF`（MPLS VPN 专属接口绑定命令，待 Stage 3 建）

5. **PE-CE 路由协议**（按子方案选一）：
   - **方案 A：静态路由** → `ADD SRROUTE` → [0-00062](task/UDG/20.15.2/0-00062.md)（VRFNAME="vpn1"）
   - **方案 B：OSPF 多实例** → 复用 OSPF 骨架（[0-00074](task/UDG/20.15.2/0-00074.md) 系列）+ OSPFVRFBIND（VRFNAME 绑定，待 Stage 3 建）
   - **方案 C：BGP（PE-CE EBGP）** → `ADD BGPPEER` + `ADD BGPVRF`（MPLS-VPN-BGP 系列，待 Stage 3 建）

6. **MP-BGP PE-PE 配置**（核心路由发布通道）：
   - **建立 MP-IBGP 对等体**（同 AS 内）：`ADD BGPPEER`（带 VPNv4 能力）+ `ADD BGPPEERAF AFTYPE=ipv4vpn`（待 Stage 3 建）
   - **跨域 OptionB 场景**：建立 MP-EBGP 对等体（不同 AS 间）
   - **大规模组网**：配置 RR（路由反射器）`ADD BGPPEER REFLECTORCLIENT=TRUE`（待 Stage 3 建）
   - **节约标签（关键）**：`MOD BGPGLOBAL LABELEMODE=perNexthop` 或 `SET BGPLABELMODE`（待 Stage 3 建）
   - **IPv6 VPN（6VPE）**：`ADD BGPPEERAF AFTYPE=ipv6vpn`（IPv4 隧道承载 IPv6 VPN）

7. **组网变体配置**（通过 VPN Target RT 控制，多个 VPN 实例配合）：
   - **Intranet**（闭合用户群）：所有 PE 同 RT（如 RT=100:1）
   - **Extranet**（部分交叉）：多 RT 匹配（如 Site1/Site2/Site3 不同 RT 组合）
   - **Hub&Spoke**（Spoke 经 Hub 互通）：双 RT 控制——Spoke-PE（ERT=Spoke/IRT=Hub），Hub-PE（接收接口 IRT=Spoke/发布接口 ERT=Hub）
   - **跨域 OptionA**（VRF-to-VRF）：ASBR 互视为 CE，普通 IPv4 EBGP
   - **跨域 OptionB**（MP-EBGP 标签 VPN-IPv4）：ASBR 间 MP-EBGP + perNexthop
   - **MCE**（多实例 CE）：CE 侧多 VPN 实例共享设备
   - **6VPE**（IPv6 VPN over IPv4 backbone）：VPNv6 地址族

8. **可靠性（可选）**：
   - **VPN FRR**：PE 间链路故障时快速切换 → `SET VPNFRR` 或 `ADD VPNFRRPOLICY`（待 Stage 3 建）
   - **VPN GR Helper**：主备倒换时 VPN 流量不中断（UDG 仅支持 Helper）→ `SET BGPGR HELPER=ENABLE`（GR capability 协商）
   - **VPN NSR**：控制平面故障时转发+控制均不中断，**需双主控硬件** → `SET BGPNSR ENABLE`

## 决策点

**DP（特性级）：组网模型 × 地址族 × PE-CE 协议 × 走法**

| 场景 | 组网模型 | 地址族 | PE-CE 协议 | 走法（compound/atom 编排） |
|---|---|---|---|---|
| Intranet（最简） | 闭合用户群 | IPv4 | OSPF | [LDP 骨架](task/UDG/20.15.2/1-00030.md)（待 Stage 3 建）+ [0-00071](task/UDG/20.15.2/0-00071.md) L3VPNINST（复用）+ VPN-INSTANCE+RT（同 RT）+ [0-00057](task/UDG/20.15.2/0-00057.md) IPBINDVPN + [0-00074](task/UDG/20.15.2/0-00074.md)~ OSPF 骨架（VRFNAME 绑定）+ MP-BGP + perInstance+perNexthop |
| Intranet 静态 | 闭合用户群 | IPv4 | 静态路由 | 同上，PE-CE 换 [0-00062](task/UDG/20.15.2/0-00062.md) SRROUTE（VRFNAME） |
| Intranet EBGP | 闭合用户群 | IPv4 | BGP PE-CE | 同上，PE-CE 换 BGP-PE-CE EBGP 对等体 |
| Extranet | 部分交叉 | IPv4 | OSPF/静态 | 同 Intranet，但 VPN Target RT 多值匹配（如 Site1+Site2 可访问 Site3） |
| Hub&Spoke | Spoke 经 Hub | IPv4 | OSPF | 同 Intranet + 双 RT（Spoke/Hub）+ Hub-PE 双接口 + **EBGP AS-Loop 允许本地 AS 重复** |
| 跨域 OptionA | VRF-to-VRF | IPv4 | 静态/EBGP | ASBR 互视为 CE 普通 IPv4 EBGP；**ASBR 需为每个 VPN 创建实例**（可扩展性差） |
| 跨域 OptionB | MP-EBGP 标签 | IPv4 | 同 Intranet | ASBR 间 MP-EBGP + perNexthop；**PE 必须 perInstance** |
| MCE（多实例 CE） | 共享 CE | IPv4 | OSPF/静态 | CE 侧多 VPN 实例 + PE-CE 路由协议带 VRF 多实例 |
| IPv6 VPN（6VPE） | 任意 | IPv6 | OSPFv3/静态 IPv6/BGP4+ | IPv4 骨干 + VPNv6 地址族 + IPv6 VRF；PE-CE IPv6 路由协议（OSPFv3 [0-00078](task/UDG/20.15.2/0-00078.md) ~） |

> **关键规则**：
> - **必须配置 perInstance（VPN 侧）+ perNexthop（BGP 侧）**——节约标签 299008 资源
> - **MP-BGP 必须使能 VPNv4 地址族**才能承载 VPN-IPv4 路由
> - **VPN Target RT 必须双向匹配**：本端 IRT 需匹配对端 ERT，否则路由被丢弃
> - **PE-CE 接口绑 VRF 是手工操作**——路由协议自身不能区分
> - **VPN GR 仅 Helper**——不支持 GR Restarter 角色
> - **VPN NSR 必须双主控硬件**——单主控设备无法使能
> - **跨域 OptionB ASBR 配置 perNexthop 时 PE 必须 perInstance**——绑定约束

## 约束

- **License 必需**（critical，rule-0-01740）：必须加载 MPLS VPN License 控制项（编号 Stage 3 补全）并通过 SET LICENSESWITCH 打开 — 后果：特性不工作
- **标签节约铁律（critical，rule-0-01741）**：必须配置 VPN 每实例每标签（perInstance）+ BGP 每下一跳每标签（perNexthop）— 后果：标签资源耗尽，路由发布失败
- **对端同步约束（critical，rule-0-01742）**：本端标签配置无法解决对端，需对端设备同步修改 — 后果：对端消耗大量本端标签资源
- **PE-CE 接口手工绑定（critical，rule-0-01743）**：VPN 实例与 PE-CE 接口的关联必须手工设置（路由协议自身不具备区分能力）— 后果：路由错乱
- **VPN Target RT 匹配（critical，rule-0-01744）**：本端 IRT 需匹配对端 ERT，否则路由被丢弃；Hub&Spoke 任意 Spoke-PE 的 IRT 不与其它 Spoke-PE 的 ERT 相同 — 后果：路由不通
- **跨域 OptionB 配套（critical，rule-0-01745）**：ASBR 配置 perNexthop 时必须在 PE 上配置 perInstance — 后果：标签不收敛
- **Hub&Spoke EBGP AS-Loop（critical，rule-0-01746）**：Hub-PE 与 Hub-CE 用 EBGP 时必须手工配置允许本地 AS 编号重复 — 后果：Hub-PE 丢弃含自身 AS 号的路由
- **VPN GR 仅 Helper（warning，rule-0-01747）**：UDG 仅支持 GR Helper 角色，不支持 GR Restarter — 后果：不能作为 GR 倒换发起端
- **VPN NSR 需双主控（critical，rule-0-01748）**：支持 NSR 的设备必须硬件具备双主控结构 — 后果：单主控设备无法使能 NSR
- **分标签切换丢包（warning，rule-0-01749）**：每路由每标签与按下一跳分标签切换过程中，本地和 ASBR 更新标签转发表会造成业务短暂丢包 — 后果：业务瞬断
- **PE 性能要求高（warning，rule-0-01750）**：所有 VPN 处理（VRF 维护、MP-BGP、标签处理）都在 PE 上发生 — 后果：大规模部署需考虑 UDG 性能规格
- **骨干网 IGP/LDP 前置（critical，rule-0-01751）**：PE-P-PE 之间 IP 骨干网已通过 IGP 路由可达 + LSP 已建立 — 后果：MPLS VPN 路由无法迭代到 Tunnel
- **标签规格上限（warning，rule-0-01752）**：MPLS 标签规格 299008 — 后果：超规格路由丢弃
- **UDG 用户面起始版本 20.2.0**（warning）：MPLS VPN 首次发布版本 20.2.0 晚于 GRE/SA-Basic — 后果：低版本网元不支持
