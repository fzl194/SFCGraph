# 配置错误码（ADD ERRORCODE）

> 命令静态知识见 [ADD ERRORCODE](command/UDG/20.15.2/ADD-ERRORCODE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD ERRORCODE 是 DNS 纠错（GWFD-110283）与 HTTP 智能重定向（GWFD-110284）共用的前置命令，按"协议错误码范围"区分两类范式。ERRORCODENAME 全网规划（命名属全网策略），操作码/起始值/终止值本端按业务决定。产出的 ERRORCODENAME 对象被 DNSOVERWRITING(0-00090)/SMARTHTTPREDIR(0-00091) 的 BINDERRCODENAME 引用，必须先于重定向动作策略配置。

**协议范式（从证据数据规划表归纳）**

| 协议 | 典型特性 | ERRORCODENAME | ERRORCODEOP | ERRORCODESTART | ERRORCODEEND | 脚本样例 |
|---|---|---|---|---|---|---|
| DNS 纠错（NXDOMAIN） | GWFD-110283 | dnserrorcode（全网规划） | EQUAL | 3（DNS 响应码 NXDOMAIN） | 不配（EQUAL 仅 START 有效） | `ADD ERRORCODE: ERRORCODENAME="dnserrorcode", ERRORCODEOP=EQUAL, ERRORCODESTART=3;` |
| HTTP 智能重定向（4xx/5xx） | GWFD-110284 | httperrorcode（全网规划） | GT | 400（HTTP 4xx/5xx 错误码） | 不配（GT 仅 START 有效） | `ADD ERRORCODE: ERRORCODENAME="httperrorcode", ERRORCODEOP=GT, ERRORCODESTART=400;` |

**参数取值规则**
- **ERRORCODENAME**：全网规划，1~31 字符（不区分大小写，不支持空格）；命名按业务语义约定（dnserrorcode/httperrorcode）。
- **ERRORCODEOP**：本端规划，决定 START/END 生效语义——EQUAL 仅 START 有效；GE/LE/GT/LT 时 START+END 共同控制范围。
- **ERRORCODESTART/END**：本端规划，整数 0~1000；③脚本均未显式配 END（EQUAL/GT 时 START 即门槛）。

## 约束

- **rule 0-00377**（capacity_limit/warning）最大记录数 100（ErrorCodeName 数量）；单名下最多 500 组错误码范围；整机最多 2000 组；立即生效；ERRORCODENAME 1~31 字符；ERRORCODESTART/END 整数 0~1000，EQUAL 时仅 START 有效；须先建 ErrorCode 才能在 DNS纠错/HTTP智能重定向策略中绑定 — 后果：超 100/500/2000 上限新增失败；未建 ErrorCode 直接引用导致重定向策略无法匹配错误码
