# 融合计费（GWFD-010173）

> 特性静态知识见 [GWFD-010173](feature/UDG/20.15.2/GWFD-010173.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

融合计费（Converged Charging）是 5G 引入的统一计费架构：**同一个 Nchf（N40）会话同时支持在线 RG（带配额管理，相当于在线计费）和离线 RG（不带配额，相当于离线计费）**。UDG 收集时长/流量/事件，经 N4 上报 SMF，SMF 通过统一 Nchf 接口与 CHF 完成实时计费和后计费，简化网络、降低消息数量。架构：UDG ↔(N4/PFCP)↔ SMF ↔(Nchf)↔ CHF。

配置结构与内容计费/流量计费/事件计费完全相同（核心对象链 URR→URRGROUP→PCCPOLICYGRP→RULE→USERPROFILE），**复用通用 backbone compound**。**计费族最复杂的特性，核心差异两点**：

1. **URR 双配置**（★核心）：每个业务配 **2 个 URR**——1 个 offline URR（`USAGERPTMODE=OFFLINE` + `OFFMETERINGTYPE`）+ 1 个 online URR（`USAGERPTMODE=ONLINE` + `ONLMETERINGTYPE`），组装到**同一个 URRGROUP**：`UPURRNAME1/DOWNURRNAME1`=离线 URR，`UPURRNAME2/DOWNURRNAME2`=在线 URR。
2. **RGAPPLIED 约束**（★核心 DP + 硬约束）：UNC 侧 `SET USRPROFCHARGE`/`SET APNCHARGECTRL`/`ADD CHARGEMETHOD` 的 `RGAPPLIED` 参数（ONLINERGONLY/OFFLINERGONLY/DEFAULT）决定 URRGROUP 怎么配；UDG 侧 URRGROUP 配置必须匹配。

4 场景（URL/IMS/any/异常）由 DP（业务过滤场景）驱动，参数差异化，不建多套流程。

## 配置流程（步骤编排，每步显示含命令）

1. **License**（单命令前置，本特性无独立 License）：`SET LICENSESWITCH`（确认内容计费 `LKV3G5BC01` 已开；融合计费自身无需 License）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **计费三件套**（费率定义，★URR 双配置）：`ADD URR`（每业务 2 条：1 个 offline URR `USAGERPTMODE=OFFLINE`+`OFFMETERINGTYPE` + 1 个 online URR `USAGERPTMODE=ONLINE`+`ONLMETERINGTYPE`）+ `ADD URRGROUP`（**按 RGAPPLIED 配**：正常融合模式 `UPURRNAME1/DOWNURRNAME1`=离线 + `UPURRNAME2/DOWNURRNAME2`=在线）+ `ADD PCCPOLICYGRP`（绑 URRGROUP；异常组作为 `SIGURRGRPNAME` 挂到 any 策略组）→ [1-00010](task/UDG/20.15.2/1-00010.md)
3. **过滤链**（业务匹配，DP 选）：`ADD FILTER`/`ADD FILTERIPV6`/`ADD L7FILTER`/`ADD FLOWFILTER`/`ADD FLTBINDFLOWF`/`ADD PROTBINDFLOWF` → [1-00009](task/UDG/20.15.2/1-00009.md)
4. **规则与用户模板绑定**：`ADD RULE`（POLICYTYPE=PCC，绑 FLOWFILTER+PCCPOLICYGRP）+ `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
5. **计费收尾**（缺省费率+防欺诈+刷新）：`SET URRGRPBINDING`（每 USERPROFILE 缺省业务+信令费率）+ `ADD SPECURRGRPLIST`（可选防欺诈，如 URL 业务）+ `SET REFRESHSRV`（`REFRESHTYPE=ALL`，必须最后）→ [1-00012](task/UDG/20.15.2/1-00012.md)
6. **全局缺省 URR 组**（单命令，可选）：`SET SPECTRAFURRGRP`（处理特殊流量/未配 URR 的业务，配合 `GLBDFTURRGSW=ENABLE` 继承）→ [SET SPECTRAFURRGRP](task/UDG/20.15.2/0-00290.md)（atom 待建）
7. **配额超时阻塞放通**（单命令，应急可选）：`SET URRFAILACTION`（`RETRYFAILACT=CONTINUE`，配额耗尽申请新配额超时被阻塞业务时临时放通）→ [0-00018](task/UDG/20.15.2/0-00018.md)

> **注意**：融合计费**不需要** `SET UPDEFAULTQUOTA`（配额由 CHF 经 Nchf 统一下发，非 UDG 本地默认配额开关）。

## 决策点

### DP-A（★核心，融合计费特有）：RGAPPLIED × URRGROUP 配置

`RGAPPLIED` 由 **UNC 侧**（SMF）`SET USRPROFCHARGE`/`SET APNCHARGECTRL`/`ADD CHARGEMETHOD` 命令设置，决定 RG（Rating Group）应用方式；**UDG 侧 URRGROUP 配置必须与之匹配**，否则计费冲突：

| RGAPPLIED（UNC 侧） | URRGROUP 配法（UDG 侧） | 含义 | 异常场景 |
|---|---|---|---|
| **ONLINERGONLY** | 只配在线 URR（`UPURRNAME1`/`DOWNURRNAME1`=online URR，不配 `UPURRNAME2`/`DOWNURRNAME2`） | 仅在线 RG，CHF 配额管理 | 同在线计费（[2-00004](task/UDG/20.15.2/2-00004.md)） |
| **OFFLINERGONLY** | 只配离线 URR（`UPURRNAME1`/`DOWNURRNAME1`=offline URR，不配 `UPURRNAME2`/`DOWNURRNAME2`） | 仅离线 RG，无配额管理 | 同离线计费（[2-00005](task/UDG/20.15.2/2-00005.md)） |
| **DEFAULT** | **只能配一种模式**（仅离线 URR **或** 仅在线 URR，二选一） | SMF 按用户签约决定在线或离线 | **不能同时配离线+在线 URR**，否则计费冲突（critical 约束） |
| 未设 / 融合正常模式 | **双 URR**（`UPURRNAME1`/`DOWNURRNAME1`=离线 + `UPURRNAME2`/`DOWNURRNAME2`=在线） | 在线+离线 RG 同接口并存（原始文档任务示例取此值） | 仅配一种且与用户计费模式不匹配则无法计费 |

> ★ RGAPPLIED=DEFAULT 的"只能配一种模式"与其他三种"必须双配或单配匹配"**正好相反**——这是融合计费最易配错的点（见下方 critical 约束）。

### DP-B：业务过滤场景（影响 URR 统计类型 + 过滤链 + 费率组）

原始文档任务示例 4 场景，统计类型均为 VOLUME（异常场景用 FREE 兜底）：

| 场景 | 业务 | 过滤模式（[1-00009](task/UDG/20.15.2/1-00009.md) DP） | URR 统计类型（[1-00010](task/UDG/20.15.2/1-00010.md)） | 计费方式 | 费率组 | RULE 优先级 |
|---|---|---|---|---|---|---|
| 场景1 URL | www.huawei.com | L34（dns）+ L7（URL） | `OFFMETERINGTYPE`/`ONLMETERINGTYPE`=**VOLUME** | 在线+离线 | UrrGp_URL | 100 |
| 场景2 IMS | 语音 | L34 IPv6（服务器 IP） | `OFFMETERINGTYPE`/`ONLMETERINGTYPE`=**VOLUME** | 在线+离线 | UrrGp_IMS | 110 |
| 场景3 any | 基础业务（兜底） | L34 any to any（IPv4+IPv6） | `OFFMETERINGTYPE`/`ONLMETERINGTYPE`=**VOLUME** | 在线+离线 | UrrGp_any | 65000/65010 |
| 场景4 异常 | 未匹配（信令） | 不匹配（走 `SIGURRGRPNAME`） | `OFFMETERINGTYPE`/`ONLMETERINGTYPE`=**FREE**（不计费） | FREE | UrrGp_abnormal | （作 SIGURRGRP） |

**联动**：
- 统计类型 VOLUME → URR（[1-00010](task/UDG/20.15.2/1-00010.md)）按业务上下行字节数累计，话单/CCR 上报流量使用信息（离线 `datavolumeFBCUplink`/`Downlink`；在线 `CC-Input-Octets`/`CC-Output-Octets`）。
- **双 URR 配置**（DP-A 正常融合模式）→ 每业务 `ADD URR` 两条（offline + online），URRGROUP 同时绑：`UPURRNAME1`/`DOWNURRNAME1`=离线 URR，`UPURRNAME2`/`DOWNURRNAME2`=在线 URR。
- 过滤模式 → 过滤链（[1-00009](task/UDG/20.15.2/1-00009.md)）出现哪些过滤器（URL 场景 L34+L7+PROTBIND；IMS 场景 FILTERIPV6；any 场景 FILTER+FILTERIPV6）。
- 缺省费率 → 收尾（[1-00012](task/UDG/20.15.2/1-00012.md)）的 `DFTURRGRPNAME`（通常 any 组）+ `DFTSIGURRGNAME`（异常组）；异常 URRGROUP 通过 PCCPOLICYGRP 的 `SIGURRGRPNAME` 挂到 any 策略组。

## 约束（特性级 rule）

- **License 依赖**（info）：本特性**无独立 License**；调测时确认内容计费基本功能 License `LKV3G5BC01`（GWFD-020301）已开 + SA-Basic（GWFD-110101） — 否则业务识别/计费基础不成立。
- **依赖特性**（critical）：[SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，业务识别必选）+ [内容计费基本功能](feature/UDG/20.15.2/GWFD-020301.md)（GWFD-020301） — 依赖链 SA-Basic → 内容计费 → 融合计费。
- **RGAPPLIED=DEFAULT 时 URRGROUP 不能同时配离线+在线 URR**（critical）：SMF 侧 `SET USRPROFCHARGE`/`SET APNCHARGECTRL`/`ADD CHARGEMETHOD` 的 `RGAPPLIED=DEFAULT` 时，URRGROUP 只能配一种模式（仅离线或仅在线） — 否则计费冲突。
- **URRGROUP 必须同时配在线+离线 URR**（critical）：非 DEFAULT 场景（含融合正常模式），URRGROUP 必须双配 — 只配一种且与用户计费模式不匹配则无法计费。
- **SMF/UPF 一致性**（critical）：`URRID`/`USAGERPTMODE`/`OFFMETERINGTYPE`/`ONLMETERINGTYPE`/`RULENAME`/`POLICYTYPE`/`POLICYNAME`/`USERPROFILENAME` 在 SMF 与 UPF（及 PCF）必须一致 — 不一致则计费异常/策略不生效（告警 ALM-81054/ALM-81026）。
- **每个 USERPROFILE 必绑缺省费率**（critical）：`SET URRGRPBINDING` 的 `DFTURRGRPNAME`（缺省业务费率）+ `DFTSIGURRGNAME`（缺省信令费率） — 否则未匹配费率的报文无法计费。
- **FLOWFILTER 必须至少绑一个过滤条件**（critical）：否则 Rule 无效。
- **REFRESHSRV 必须最后执行**（critical）：`SET REFRESHSRV:REFRESHTYPE=ALL` — 不执行则 FILTER 配置变更不生效。
- **RULE 优先级**（warning）：数字越小越高；只绑三四层的 RULE 优先级低于绑七层的；any 规则优先级最低（如 65000/65010）。
- **配额超时阻塞放通**（warning）：配额耗尽申请新配额超时（`T3RESPONSE×N3REQUEST+4s`，SET UPPFCPPATH 配）→ PGW-U 阻塞业务；应急放通用 `SET URRFAILACTION:RETRYFAILACT=CONTINUE`。
- **防欺诈绑定约束**（warning）：防欺诈开启时（`ADD SPECURRGRPLIST`），RULE 绑 URL 类型七层 filter 必须同时绑三四层 filter；防欺诈未开启时可仅绑七层。
- **UPURRNAME1/2、DOWNURRNAME1/2 仅编号无优先级语义**（info）：1/2 是槽位编号，非计费优先级。
- **性能**（info）：七层解析对吞吐量有影响；不解析加密协议（SSL/FTPS）。
