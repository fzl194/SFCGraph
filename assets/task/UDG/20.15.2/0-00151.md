# 增加Host主机名域名（ADD HOST）

> 命令静态知识见 [ADD HOST](command/UDG/20.15.2/ADD-HOST.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD HOST 用于配置 Host 条目，支撑 SA 通过 DNS 解析将业务域名映射到 IP，再通过三四层过滤识别该业务的后续报文。**HOSTNAME 与 DOMAINVALUE 必选**（成对存在）；**PRIORITY 与 CFGDOMAINNAME 可选**。核心配置空间有 4 个维度：**Host 名**（HOSTNAME）、**域名**（DOMAINVALUE）、**优先级**（PRIORITY）、**配置域**（CFGDOMAINNAME）。1 个特性命中（GWFD-110203 HTTPS 业务地址识别）使用 `ht_https` + `www.example.com` 范式。

### 配置维度 1：Host 名（参数 HOSTNAME，必选）

| 取值 | 作用 |
|---|---|
| **字符串，1~31 字符** | Host 配置名称；同一 HOSTNAME 唯一标识一条 Host 条目；Filter 按此键引用 |

### 配置维度 2：域名（参数 DOMAINVALUE，必选）

| 取值 | 作用 |
|---|---|
| **字符串，1~63 字符（不区分大小写，转义后）** | 域名；SA 通过对该域名的 DNS 解析获取业务 IP，建立 Domain→ServerIP 映射 |

### 配置维度 3：优先级（参数 PRIORITY，可选）

| 取值 | 作用 |
|---|---|
| **整数 1~65535**（默认 65535） | Host 优先级；值越小越优先 |
| 未配 | 取默认值 65535 |

### 配置维度 4：配置域（参数 CFGDOMAINNAME，可选）

| 取值 | 作用 |
|---|---|
| **字符串，1~31 字符** | 命令所属公共配置域的名称 |

**典型脚本（GWFD-110203 HTTPS 业务地址识别）**：
```
ADD HOST:HOSTNAME="ht_https",DOMAINVALUE="www.example.com";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：建 Host 域名条目"——配合 [0-00152](task/UDG/20.15.2/0-00152.md) LOD PARSERDB（加载最新协议解析库）+ [0-00154](task/UDG/20.15.2/0-00154.md) SET SACOMMONPARA（SA 业务公共参数）使用，触发 SA 域名→IP 映射建立。

## 决策点

（本命令无独立决策分支，仅有 HOSTNAME+DOMAINVALUE 取值差异。）

| 选项 | HOSTNAME | DOMAINVALUE | 适用场景 |
|---|---|---|---|
| **HTTPS Host**（110203） | `ht_https` | `www.example.com` | HTTPS 业务地址识别 |

> 1 个特性命中（110203）；HOSTNAME 与 DOMAINVALUE 必选且成对；PRIORITY 缺省取默认值 65535。

## 约束

- **HOSTNAME+DOMAINVALUE 联合必选**（critical，rule-0-00151）：HOSTNAME 与 DOMAINVALUE 是必选参数 — 后果：缺一命令拒收
- **HOSTNAME 长度与字符**（warning，rule-0-00151）：HOSTNAME 字符串长度 1~31，不支持空 — 后果：超长/空字符串命令拒收
- **DOMAINVALUE 长度与大小写**（warning，rule-0-00151）：DOMAINVALUE 字符串长度 1~63，不区分大小写，转义后比较 — 后果：超长命令拒收；同名 Host 重复添加时按大小写不敏感判定
- **PRIORITY 取值范围**（warning，rule-0-00151）：PRIORITY 取值 1~65535，值越小越优先 — 后果：超范围命令拒收
- **执行后 60s 生效**（warning，rule-0-00151）：本命令执行后 60s 生效 — 后果：下发后立即跑业务流不会命中，须等 60s 再验证
- **DNS 解析依赖**（info，rule-0-00151）：SA 域名→IP 映射依赖 DNS 解析，须保证 [0-00152](task/UDG/20.15.2/0-00152.md) LOD PARSERDB 已加载协议解析库 — 后果：未加载解析库时 DNS 解析失败
