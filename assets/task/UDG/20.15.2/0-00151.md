# 配置Host（ADD HOST）

> 命令静态知识见 [ADD HOST](command/UDG/20.15.2/ADD-HOST.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD HOST 是 HTTPS 业务地址识别场景的"DNS 解析桥"：UDG 解析 DNS 响应报文得到 Domain→ServerIP 映射，FILTER 按 HOSTNAME 引用本 Host，把后续访问该 IP 的业务流关联到规则。每条 Host = 一个域名条目。

**参数规划（取值样例）**

| 参数 | 含义 | 取值样例 | 来源 | 说明 |
|---|---|---|---|---|
| HOSTNAME | Host 配置名称（Filter 引用键） | `ht_https` | 本端规划 | 1~31 字符，不支持空格，不区分大小写；建议带业务语义前缀 |
| DOMAINVALUE | 域名 | `www.example.com` | 本端规划 | 1~63 字符；**含空格必须用 `%20` 转义**，转义后 ≤120 字符 |

**典型脚本**

```
ADD HOST:HOSTNAME="ht_https",DOMAINVALUE="www.example.com";
```

**配置步骤**（在 HTTPS 业务地址识别特性 GWFD-110203 中的位置）
1. 先配计费三件套（URR / URRGROUP / PCCPOLICYGRP）。
2. 配 HTTPS 报文解析规则：**ADD HOST**（本命令，建域名→IP 映射）→ ADD FILTER（按 Host 匹配的三四层条件）。
3. Host 名需与 ADD FILTER 的 HOSTNAME 参数一致，否则 Filter 引用失效。

**冷门参数（主场景未用）**：`PRIORITY`（默认 65535，值越小优先级越高，optional 不配）、`CFGDOMAINNAME`（配置域名称，optional 不配）。

## 约束

- **rule-0-00307**（warning）执行后 **60s 生效**——下发后立即跑业务流不会命中，须等 60s 再验证。
- **rule-0-00307**（warning）最大记录数 **100000**，超限 ADD 失败；超 90% 上报 `ALM-135602215 配置数量超阈值`（阈值可 MOD CFGTHRESHOLD 修改）。
- **rule-0-00307**（warning）DOMAINVALUE 含空格必须 `%20` 转义，转义后 ≤120 字符；HOSTNAME 1~31 字符不支持空格。
