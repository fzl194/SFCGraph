# 过滤链

> **跨域通用 backbone**（对齐 ConfigTask 计费 1-00002）：流匹配基础链，计费/BWM/URL过滤共用。哪些过滤器出现由 DP（过滤模式）决定，FLOWFILTER 至少绑一个。被引用于 [2-00002](task/UDG/20.15.2/2-00002.md)。

## 配置方法

过滤器（三四层/七层/IPv6，按场景选）→ 聚合为 FLOWFILTER → 绑定。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 三四层过滤器 IPv4 | `ADD FILTER` | FILTERNAME；L34PROTTYPE=STRING；L34PROTOCOL=ANY（URL过滤全匹配）/计费三四层 |
| 三四层过滤器 IPv6 | `ADD FILTERIPV6` | FILTERNAME（IMS/IPv6 场景） |
| 七层过滤器 | `ADD L7FILTER` | L7FILTERNAME（PCC/BWM 七层匹配） |
| 流过滤器（必配） | `ADD FLOWFILTER` | FLOWFILTERNAME（聚合容器） |
| 三四层绑定 | `ADD FLTBINDFLOWF` | FLOWFILTERNAME + FILTERNAME（FILTER→FLOWFILTER） |
| 七层绑定 | `ADD PROTBINDFLOWF` | FLOWFILTERNAME + PROTOCOLNAME + L7FILTERNAME（L7FILTER→FLOWFILTER） |

**典型脚本（URL过滤：三四层全匹配）**：
```
ADD FILTER:FILTERNAME="filter_test",L34PROTTYPE=STRING,L34PROTOCOL=ANY;
ADD FLOWFILTER:FLOWFILTERNAME="flowfilter_test1";
ADD FLTBINDFLOWF:FLOWFILTERNAME="flowfilter_test1",FILTERNAME="filter_test";
```

**步骤位置**：承接业务配置；产出 FLOWFILTERNAME 供下游 RULE 引用。刷新生效 SET REFRESHSRV 不在本步骤（见 feature 编排，单命令 atom）。

## 决策点

| 选项（过滤模式，联动特性级） | 影响（参数/命令/联动） |
|---|---|
| URL过滤 | FILTER（三四层全匹配，内容过滤在 CF 层做） |
| 计费 URL 场景 | FILTER + L7FILTER + PROTBINDFLOWF |
| IMS 场景 | FILTERIPV6 |
| any 兜底场景 | FILTER + FILTERIPV6 |

## 约束

- **FLOWFILTER 至少绑一个过滤条件**（critical）：三四层或七层至少其一 — 否则规则匹配失效。
- **绑定参数须已配置**（warning）：FLTBINDFLOWF/PROTBINDFLOWF 的 FILTERNAME/L7FILTERNAME/FLOWFILTERNAME 须已配置 — 否则绑定失败。
