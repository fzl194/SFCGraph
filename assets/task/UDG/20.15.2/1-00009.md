# 过滤链

> **跨域通用 backbone**（对齐 ConfigTask 计费 1-00002）：流匹配基础链，计费/BWM/URL过滤共用。哪些过滤器出现由 DP（过滤模式）决定，FLOWFILTER 至少绑一个。被引用于 [2-00002](task/UDG/20.15.2/2-00002.md)。

## 配置方法

过滤器（三四层/七层/IPv6，按场景选）→ 聚合为 FLOWFILTER → 绑定。

| 过滤器 | 命令 | 适用场景 |
|---|---|---|
| 三四层 IPv4 | ADD FILTER | URL过滤(L34PROTOCOL=ANY)/计费三四层 |
| 三四层 IPv6 | ADD FILTERIPV6 | IMS/IPv6 场景 |
| 七层 | ADD L7FILTER | PCC/BWM 七层匹配 |
| 流过滤器 | ADD FLOWFILTER | 聚合容器（必配） |
| 三四层绑定 | ADD FLTBINDFLOWF | FILTER→FLOWFILTER |
| 七层绑定 | ADD PROTBINDFLOWF | L7FILTER→FLOWFILTER |

**典型脚本（URL过滤：三四层全匹配）**：
```
ADD FILTER:FILTERNAME="filter_test",L34PROTTYPE=STRING,L34PROTOCOL=ANY;
ADD FLOWFILTER:FLOWFILTERNAME="flowfilter_test1";
ADD FLTBINDFLOWF:FLOWFILTERNAME="flowfilter_test1",FILTERNAME="filter_test";
```

**步骤位置**：承接业务配置；产出 FLOWFILTERNAME 供下游 RULE 引用。刷新生效 SET REFRESHSRV 不在本步骤（见 feature 编排，单命令 atom）。

## 决策点

**DP：过滤模式**（联动特性级）—— 决定出现哪些过滤器：
- URL过滤 → FILTER（三四层全匹配，内容过滤在 CF 层做）
- 计费 URL 场景 → FILTER + L7FILTER + PROTBINDFLOWF
- IMS 场景 → FILTERIPV6
- any 兜底场景 → FILTER + FILTERIPV6

## 约束

- **FLOWFILTER 必须至少绑一个过滤条件**（三四层或七层），否则规则匹配失效。
- FLTBINDFLOWF/PROTBINDFLOWF 的 FILTERNAME/L7FILTERNAME/FLOWFILTERNAME 须已配置。
