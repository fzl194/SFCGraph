# 会话建立依赖-地址分配

> **骨架 compound（能力型底座特性维度）**：归属 [2-00033](task/UDG/20.15.2/2-00033.md) 会话管理（GWFD-010101）。本 compound **无独立配置步骤**，仅显式记录 PDU/EPS/PDP 会话建立对地址分配特性的依赖关系——会话级 PFCP 信令携带地址分配信元，UPF 侧地址池/地址分配规则的配置在 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md) 特性下完成。

## 配置方法

> 本骨架 compound **无独立命令**。会话建立的地址分配决策在 PFCP Session Establishment Request 的 `Create PDR` 信元中下发（`UE IP Address`/`UE IP Address Pool Identity`，由 SMF→UPF）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| **本骨架无配置步骤** | — | — |
| **依赖**：地址池/地址分配规则 | 详见 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md) 任务 wiki（待建） | `ADD POOL` / `ADD POOLGROUP` / `ADD POOLBINDGROUP` / `SET APNADDRESSATTR` 等 |
| **依赖**：会话建立流程触发 | SMF→UPF PFCP 信令（非 UDG 配置） | N4 PDR 信元携带地址分配指令 |

**配置生成说明**：
- 配置生成器遇到本骨架时，**直接跳转到 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md) 地址分配特性**生成地址池/地址分配规则
- 5G 场景：PFCP Create PDR 的 `UE IP Address Pool Identity` 字段需引用 UDG 已建的地址池名
- 4G/2-3G 场景：通过 APN→地址池绑定（`SET APNADDRESSATTR`）实现地址分配
- 本骨架的"决策点"**直接引用地址分配特性决策**，不在本骨架重复定义

## 场景差异

| 接入制式 | 会话类型 | PFCP/GTP 信元 | 地址分配触发点 |
|---|---|---|---|
| 5G | PDU 会话 | N4 Create PDR: UE IP Address / UE IP Address Pool Identity | SMF→UPF N4 PFCP Session Establishment Request 步骤16 |
| 4G | EPS 缺省承载 | Sxb PDR: UE IP Address（PGW-U分配时） | Sxb PFCP 承载建立请求 |
| 4G | EPS 专有承载 | 沿用缺省承载 IP，不重新分配 | 专有承载建立时不携带地址分配信元 |
| 2/3G | PDP 上下文 | Gn/Gp GTP-C PDP Context Activation：PDP Address | Gn 接口 PDP 激活请求 |

## 决策点

| 选项（接入制式） | 影响（被依赖特性配置） |
|---|---|
| 5G PDU 会话 | 走 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md) 5G 地址分配（基于 APN/DNN、基于 SMF、IPv4/IPv6/IPv4v6） |
| 4G EPS 承载 | 走 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md) 4G 地址分配（基于 APN 的 SGW-U/PGW-U 本地池） |
| 2/3G PDP 上下文 | 走 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md) 2/3G 地址分配（Gn/Gp 接口 PGW-U 本地池） |

> 联动特性级场景 DP：会话建立类型由 [2-00033](task/UDG/20.15.2/2-00033.md) 场景 DP 决定（5G PDU/4G EPS/2-3G PDP）。

## 约束

- **地址池必须先建**（critical）：会话建立的 PFCP 信令携带地址池名引用 UDG 已建资源，否则 PFCP Session Establishment Response 失败 — 地址池配置归 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md)。
- **C/U 面偶联必须就绪**（critical）：N4（5G）/Sxb（4G）/Gn/Gp（2-3G）接口偶联是会话信令的前置基础。
- **IPv4v6 双栈决策**（warning）：双栈场景（PDU session type=IPv4v6）需 PFCP Create PDR 同时携带 CHV4+CHV6 信元，对应 [GWFD-020403](feature/UDG/20.15.2/GWFD-020403.md) IPv4v6双栈接入。
