# 会话建立依赖-地址分配（★已填实骨架）

> **骨架 compound（能力型底座特性维度，已填实）**：归属 [2-00033](task/UDG/20.15.2/2-00033.md) 会话管理（GWFD-010101）。本 compound **无独立配置步骤**，仅显式记录 PDU/EPS/PDP 会话建立对地址分配特性的依赖关系——会话级 PFCP 信令携带地址分配信元，UPF 侧地址池/地址分配规则的配置在 [2-00034](task/UDG/20.15.2/2-00034.md)（GWFD-010105）特性下完成。

> **★ 2026-07-11 填实说明**：GWFD-010105 用户面地址分配特性（[2-00034](task/UDG/20.15.2/2-00034.md)）已首建，本骨架的"无配置"占位已被真实配置取代。本文件保留作为"会话建立→地址分配"依赖关系的可追溯锚点。

## 配置方法

> 本骨架 compound **无独立命令**。会话建立的地址分配决策在 PFCP Session Establishment Request 的 `Create PDR` 信元中下发（`UE IP Address`/`UE IP Address Pool Identity`，由 SMF→UPF）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| **本骨架无配置步骤** | — | — |
| **依赖 1**：地址池体系（地址池三级+映射） | 详见 [1-00022](task/UDG/20.15.2/1-00022.md) | `ADD POOL` / `ADD SECTION` / `ADD POOLGROUP` / `ADD POOLBINDGROUP` / `ADD POOLGRPMAP` / `ADD CONFLICTIP` |
| **依赖 2**：地址分配规则（全局+APN两级+三级优先级） | 详见 [1-00021](task/UDG/20.15.2/1-00021.md) | `SET IPALLOCRULE` / `SET APNIPALLOCRULE` |
| **依赖 3**：SMF 子方式开关（仅基于 SMF/SMF+APN 子方式） | 详见 [1-00023](task/UDG/20.15.2/1-00023.md) | `ADD CPNODEID` / `SET IPALLOCBYSMFGLBSW` / `SET IPALLOCBYSMFSW` |
| **依赖 4**：下行路由发布（OSPF 引入 wlr） | 详见 [1-00024](task/UDG/20.15.2/1-00024.md) | `ADD OSPF` / `ADD OSPFAREA` / `ADD OSPFNETWORK` / `ADD OSPFIMPORTROUTE PROTOCOL=wlr` |
| **依赖**：APN/DNN 实例与地址分配属性 | [0-00020](task/UDG/20.15.2/0-00020.md) + [0-00046](task/UDG/20.15.2/0-00046.md) | `ADD APN` / `SET APNADDRESSATTR` |
| **依赖**：会话建立流程触发 | SMF→UPF PFCP 信令（非 UDG 配置） | N4 PDR 信元携带地址分配指令 |

**配置生成说明**：
- 配置生成器遇到本骨架时，**直接跳转到 [2-00034](task/UDG/20.15.2/2-00034.md) GWFD-010105 地址分配特性**生成完整配置（含 4 个子方式 + 调测命令）
- 5G 场景：PFCP Create PDR 的 `UE IP Address Pool Identity` 字段需引用 UDG 已建的地址池名
- 4G/2-3G 场景：通过 APN→地址池绑定（`SET APNADDRESSATTR`）实现地址分配
- 基于 SMF 子方式（5G BSF 场景）：需配 CPNODEID + IPALLOCBYSMFGLBSW
- 基于 RADIUS 子方式：需配 APNADDRESSATTR IGNORE*=DISABLE + ADDRESSATTR V4/V6POOLWILDCARDSW=ENABLE
- 本骨架的"决策点"**直接引用地址分配特性决策**，不在本骨架重复定义

## 约束

- **地址池必须先建**（critical）：会话建立的 PFCP 信令携带地址池名引用 UDG 已建资源，否则 PFCP Session Establishment Response 失败 — 地址池配置归 [2-00034](task/UDG/20.15.2/2-00034.md)。
- **C/U 面偶联必须就绪**（critical）：N4（5G）/Sxb（4G）/Gn/Gp（2-3G）接口偶联是会话信令的前置基础。
- **IPv4v6 双栈决策**（warning）：双栈场景（PDU session type=IPv4v6）需 PFCP Create PDR 同时携带 CHV4+CHV6 信元，对应 [GWFD-020403](feature/UDG/20.15.2/GWFD-020403.md) IPv4v6双栈接入。
- **基于 SMF 子方式必配 IPALLOCBYSMFGLBSW**（critical）：SMF 子方式需全局开关 ENABLE，否则 SMF→UPF 地址分配链路不生效 — 配置归 [2-00034](task/UDG/20.15.2/2-00034.md) 中 [1-00023](task/UDG/20.15.2/1-00023.md)。
