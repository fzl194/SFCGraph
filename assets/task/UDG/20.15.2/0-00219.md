# 配置F-TEID分配能力（SET CPTEIDUALLOC）

> 命令静态知识见 [SET CPTEIDUALLOC](command/UDG/20.15.2/SET-CPTEIDUALLOC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

单命令原子，全局唯一记录（SET 修改）。激活 GWFD-020161（CU Full Mesh 组网 UPF 侧）第二步：配置用户面支持分配 F-TEID。该命令决定 UDG 与 SMF/SGW-C/PGW-C 协商 F-TEID 分配能力，CH BIT 位不匹配会直接导致会话建立失败（返回 Invalid F-TEID allocation option）。适用 SGW-U/PGW-U/UPF。

**数据规划取值样例**（证据③，GWFD-020161/激活CU Full Mesh组网（UPF）_51809001.md 数据规划表）：

| 参数 | 含义 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|---|
| SWITCH | UP 分配开关 | DISABLE | 本端规划 | Full Mesh 组网下必须 = DISABLE（强制 UP 本地分配 F-TEID） |

**示例脚本**（取自激活 md）：

```
SET CPTEIDUALLOC: SWITCH=DISABLE;
```

**配置要点**：
- CU Full Mesh 组网下 SWITCH 取值唯一：DISABLE。md 原文"该参数必须配置为 DISABLE"。
- 当前产品仅支持 UP 本地分配 TEID-U，不支持 CP 分配；配错直接导致用户激活失败。
- 该配置不参与 N4 偶联建立条件判断，但影响后续会话建立。
- 前提（基础组网，非本特性步骤）：UDG 本端已完成 Sa/Sc/Pa 数据面逻辑接口与 N4 逻辑接口配置、对端信息（ADD CPASSOCIATION）。

## 约束

- 最大记录数 1（全局唯一）；**只对新激活用户生效**，存量用户不回溯。
- **高危**：系统是否支持 CP 分配 TEID-U 的配置一般不要修改；如需修改（如组网变化），必须先 `DSP SESSIONNUMBER` 确认无用户，若有用户须先 `DEA SESSION` 去活所有用户，下线后再改。
- SWITCH 非 DISABLE（CU Full Mesh 场景）→ 用户激活失败。
- 存在系统初始记录（SWITCH 按产品初值）。
