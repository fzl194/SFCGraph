# 增加L3VPN实例地址族(IPsec微服务)（ADD VPNINSTAFIPSEC）

> 命令静态知识见 [ADD VPNINSTAFIPSEC](command/UDG/20.15.2/ADD-VPNINSTAFIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是 IPsec 微服务侧"VPN + 隧道接口"双配的"加地址族"步骤——在 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 建好 VPN 之后下发，后置 [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC 建 Tunnel/LoopBack + [0-00306](task/UDG/20.15.2/0-00306.md) ADD IPBINDVPNIPSEC 接口绑 VPN + [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC 接口配 IP。**双配铁律**：VRFNAME 必须与 VNRS 侧 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST:VRFNAME **完全相同**（同一 VPN 实例），AFTYPE 决定后续 Tunnel 接口走 IPv4/IPv6 链路。

## 配置方法（配置方法字典）

ADD VPNINSTAFIPSEC 是"IPsec 微服务侧 VPN 实例加地址族"命令。2 个必选参数（VRFNAME + AFTYPE），无可选参数。配置空间有 3 个维度：**VPN 实例**（VRFNAME 引用 L3VPNINSTIPSEC 产出）、**地址族类型**（AFTYPE Ipv4uni/Ipv6uni 二选一固定）、**保留实例黑名单**（`__mpp_vpn_inner__` 禁配）。1 个特性命中（IPFD-015004 IPSec功能 8 子场景：普通 IPv4/IPv6、IPv4/IPv6 主备、GRE over IPsec、OSPF over IPsec、多 Sequence、指定本端接口）共享同一配置空间——IPsec 子场景差异不在本命令（在本命令的上游/下游），仅 VRFNAME 取值（与子场景无关，沿用 `vrf1` 等统一标识）+ AFTYPE 选 ipv4uni/ipv6uni 随子场景分叉。

### 配置维度 1：VPN 实例（参数 VRFNAME，必选，引用上游）

| 取值形式 | 来源 | 双配对应 |
|---|---|---|
| **字符串 1~31**（不支持空格） | 引用 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC:VRFNAME 产出的 VPN 实例名 | 必须与 VNRS 侧 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST:VRFNAME **完全相同**（同一 VPN） |

> **VRFNAME 双配铁律**：本参数引用 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 已建的 IPsec 微服务侧 VPN；同时该 VPN 必须与 VNRS 侧 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST 产出同名——三方对齐（IPsec 侧 L3VPNINSTIPSEC + IPsec 侧 VPNINSTAFIPSEC + VNRS 侧 L3VPNINST），否则 IPsec 隧道不通。

> **必先建 VPN**：VRFNAME 必须先经 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 创建；引用未建 VPN 命令拒收。

> **典型 VRFNAME**：IPFD-015004 8 子场景命中脚本均使用 `vrf1`。

> **黑名单铁律**：`__mpp_vpn_inner__`（系统内部 VPN 保留实例）禁止添加地址族——命令拒收。

### 配置维度 2：地址族类型（参数 AFTYPE，必选，二选一固定）

| 取值 | 作用 | 配套 Tunnel 链路 | 典型场景 |
|---|---|---|---|
| **Ipv4uni**（IPv4 单播地址族） | 给 VPN 加 IPv4 单播地址族 | 后续 Tunnel 走 IPv4 链路（TNLTYPE=IPSEC/IPSEC6 配 IPv4 策略时） | IPFD-015004 普通 IPv4 隧道（10002）/ IPv4 主备（10004）/ GRE over IPsec（10006）/ OSPF over IPsec（10007）/ 多 Sequence（10008）/ 指定本端接口（10009）等所有 IPv4 子场景 |
| **Ipv6uni**（IPv6 单播地址族） | 给 VPN 加 IPv6 单播地址族 | 后续 Tunnel 走 IPv6 链路（TNLTYPE=IPSEC6 配 IPv6 策略时） | IPFD-015004 普通 IPv6 隧道（10003）/ IPv6 主备（10005）等 IPv6 子场景 |

> **地址族双配铁律**：AFTYPE 必须与 VNRS 侧 [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF:AFTYPE + [0-00216](task/UDG/20.15.2/0-00216.md) ADD IPSECINTFCFG:TNLTYPE 链路族**三方一致**（Ipv4uni→IPSEC，Ipv6uni→IPSEC6）——否则 IPsec 隧道协商链路族不匹配。

### 配置维度 3：保留实例黑名单（隐式约束，无对应参数）

| 黑名单 VPN | 约束 | 后果 |
|---|---|---|
| **`__mpp_vpn_inner__`** | 系统内部 VPN 保留实例，禁止添加地址族 | 命令拒收（原始命令 notes 明确禁止） |

> **`_public_` 不在黑名单**：注意 `[0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC` 校验 `_public_`（默认公网 VPN）保留名时拒收，但本命令原始 notes **只禁止 `__mpp_vpn_inner__`**——即 `_public_` 在本命令可加地址族（需先用上游 [0-00210](task/UDG/20.15.2/0-00305.md) 建出该 VPN；实际部署中一般不用）。

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，AFTYPE=Ipv4uni）**：
```
ADD L3VPNINSTIPSEC:VRFNAME="vrf1";
ADD VPNINSTAFIPSEC:VRFNAME="vrf1", AFTYPE=Ipv4uni;
ADD INTERFACEIPSEC:IFNAME="Tunnel10", IFADMINSTATUS=Up;
ADD IPBINDVPNIPSEC:IFNAME="Tunnel10", VRFNAME="vrf1", IPTYPECHECK=Ipv4;
ADD IFIPV4ADDRESSIPSEC:IFNAME="Tunnel10", IFIPADDR="10.10.0.1", SUBNETMASK="255.255.255.252";
```

**典型脚本（IPFD-015004 普通 IPv6 IPsec 隧道，AFTYPE=Ipv6uni）**：
```
ADD L3VPNINSTIPSEC:VRFNAME="vrf1";
ADD VPNINSTAFIPSEC:VRFNAME="vrf1", AFTYPE=Ipv6uni;
```

> 命令 notes：**最大记录数 1001**（一个 VPN 实例最多配 1001 条地址族记录——理论上每 VPN 配 IPv4+IPv6 两条已达上限，剩余可用条数极少）；**执行后立即生效**；**不能给 VPN 实例 `__mpp_vpn_inner__` 添加地址族**；**必先建 VPN**——VRFNAME 引用的 L3VPN 实例必须先经 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 创建。

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：IPsec 微服务五件套——加地址族"——前置 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC（建 VPN）；后置 [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC（建 Tunnel/LoopBack）+ [0-00306](task/UDG/20.15.2/0-00306.md) ADD IPBINDVPNIPSEC（接口绑 VPN）+ [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC（接口配 IP）；下游 IPsec 协商链：[0-00307](task/UDG/20.15.2/0-00307.md) IPsec Policy→Tunnel 接口绑定。

## 决策点：IPsec 隧道链路族选择（DP 0-00317）

问：本地址族用于 IPv4 IPsec 隧道还是 IPv6 IPsec 隧道？（决定 AFTYPE 选 Ipv4uni/Ipv6uni + 后续 Tunnel 链路族 + IPsec 策略族）

| 选项 | AFTYPE | 后续 Tunnel TNLTYPE | 后续 IPsec 策略族 | 典型场景 |
|---|---|---|---|---|
| IPv4 IPsec 隧道（IPFD-015004 10002/10004/10006/10007/10008/10009） | **Ipv4uni** | IPSEC（[0-00216](task/UDG/20.15.2/0-00216.md)） | [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY（IPv4 策略） | 普通 IPv4 隧道/IPv4 主备/GRE over IPsec/OSPF over IPsec/多 Sequence/指定本端接口 |
| IPv6 IPsec 隧道（IPFD-015004 10003/10005） | **Ipv6uni** | IPSEC6（[0-00216](task/UDG/20.15.2/0-00216.md)） | ADD IPSECPOLICY6（★R1.5 atom 待补）（IPv6 策略） | 普通 IPv6 隧道/IPv6 主备 |

> 1 个特性命中（IPFD-015004 IPSec功能 8 子场景）共享同一命令空间；差异在 AFTYPE 二选一（Ipv4uni/Ipv6uni）+ 后续 Tunnel 链路族（IPSEC/IPSEC6）+ IPsec 策略族（IPv4 策略/IPv6 策略）三方对齐；**VRFNAME 双配铁律贯穿所有子场景**——与 [0-00071](task/UDG/20.15.2/0-00071.md) VNRS 侧 VPN 同名。

## 约束

- **最大记录数 1001**（warning，rule-0-00317）：一个 L3VPN 实例最多可输入 1001 条地址族记录（理论 IPv4+IPv6 已用 2 条，剩余极少） — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00317）：命令执行后立即生效 — 后果：误操作即刻生效
- **`__mpp_vpn_inner__` 禁配地址族**（critical，rule-0-00317）：不能给 VPN 实例 `__mpp_vpn_inner__` 添加地址族 — 后果：系统内部保留 VPN 命令拒收
- **VRFNAME 必须先经 L3VPNINSTIPSEC 建**（critical，rule-0-00317）：VRFNAME 必须先经 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 创建 — 后果：引用未建 VPN 命令拒收
- **VRFNAME 双配铁律：与 VNRS 侧 L3VPNINST 同名**（critical，rule-0-00317）：VRFNAME 必须与 VNRS 侧 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST:VRFNAME **完全相同**（同一 VPN 实例） — 后果：VRFNAME 不一致 IPsec 隧道不通
- **AFTYPE 三方对齐铁律**（critical，rule-0-00317）：AFTYPE 必须与 VNRS 侧 [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF:AFTYPE + [0-00216](task/UDG/20.15.2/0-00216.md) ADD IPSECINTFCFG:TNLTYPE 链路族**三方一致**（Ipv4uni→IPSEC，Ipv6uni→IPSEC6） — 后果：链路族不匹配 IPsec 协商失败
- **VRFNAME 长度**（info，rule-0-00317）：字符串 1~31（不支持空格） — 后果：超长/含空格命令拒收
- **VRFNAME 字符约束**（info，rule-0-00317）：不支持空格 — 后果：含空格命令拒收
- **权限要求**（info，rule-0-00317）：G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
