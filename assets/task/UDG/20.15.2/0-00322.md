# 设置基于位置区分配地址的开关（SET IPALLOCBYLOCSW）

> 命令静态知识见 [SET IPALLOCBYLOCSW](command/UDG/20.15.2/SET-IPALLOCBYLOCSW.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典",**不**逐特性罗列取值。该命令**适用 NF 限定 PGW-U/UPF**;max_records=9000;**LOCATIONGRPNAME 必须通过 ADD LACGROUP/ADD TACGROUP 预建**(见 [0-00310](task/UDG/20.15.2/0-00310.md)/[0-00316](task/UDG/20.15.2/0-00316.md))。本命令无汇总 md(走 SOP §2.3 path 2,仅基于原始命令 md)。

## 配置方法(配置方法字典)

SET IPALLOCBYLOCSW 是"基于位置区分配地址的开关"命令,适用于 PGW-U/UPF。作用:按位置区组(LAC/TAC)维度细粒度配置基于位置区分配地址的 IPv4/IPv6 开关;位置区组级开关的 SWITCH=INHERIT 时继承全局 IPALLOCBYLOCGLBSW 配置。核心配置空间有 3 个维度:**位置区组类型**(LOCATIONGRPTYPE 决定 LAC/TAC 二选一)、**位置区组名称**(LOCATIONGRPNAME 引用预建的 LACGROUP/TACGROUP)、**IPv4/IPv6 开关**(SWITCH/IPV6SWITCH 三态 DISABLE/ENABLE/INHERIT)。

### 配置维度 1:位置区组类型(参数 LOCATIONGRPTYPE,必选)

LOCATIONGRPTYPE 决定 LOCATIONGRPNAME 引用的对象类型:

| 取值 | 联动必选参数 | 适用场景 |
|---|---|---|
| **LAC** | LOCATIONGRPNAME 引 [0-00310](task/UDG/20.15.2/0-00310.md) ADD LACGROUP | 2G/3G 位置区组(GSM/UMTS) |
| **TAC** | LOCATIONGRPNAME 引 [0-00316](task/UDG/20.15.2/0-00316.md) ADD TACGROUP | 4G/5G 跟踪区组(LTE/NR) |

### 配置维度 2:位置区组名称(参数 LOCATIONGRPNAME,必选)

| 取值形式 | 取值范围 | 联动 |
|---|---|---|
| **字符串 1~32 位** | 长度 1~32 | 必须已通过 ADD LACGROUP(LAC 类型)/ADD TACGROUP(TAC 类型)预建;LOCATIONGRPTYPE 决定引用对象 |

### 配置维度 3:IPv4/IPv6 开关(参数 SWITCH/IPV6SWITCH,可选,三态)

| 取值 | 作用 | 联动 |
|---|---|---|
| **DISABLE** | 不使能基于位置区分配地址 | 位置分配地址强制关闭 |
| **ENABLE** | 使能基于位置区分配地址 | 位置分配地址开启 |
| **INHERIT**(默认) | 继承全局 IPALLOCBYLOCGLBSW 配置 | 组级不独立决策,跟全局值走 |

> **三态语义关键**:SWITCH=INHERIT 是本命令核心特性——组级"不独立决策,继承全局",适配"位置分配地址全网一致"的场景;组级 DISABLE/ENABLE 是"组级独立决策"——允许特定位置区组按位置分配,其它位置区组跟全局。

> **注意事项**(命令 notes 投影):
> - 该命令执行后**立即生效**。
> - 该命令最大记录数为 **9000**(按 LACGROUP+TACGROUP 总计位置区组上限)。
> - 开关为 INHERIT 时,按全局 IPALLOCBYLOCGLBSW 配置分配地址;非 INHERIT 时按本组级配置分配地址。
> - LOCATIONGRPNAME 必须已通过 ADD LACGROUP(LOCATIONGRPTYPE=LAC)或 ADD TACGROUP(LOCATIONGRPTYPE=TAC)预建。

**典型脚本(LAC 组允许基于位置区分配地址)**:
```
SET IPALLOCBYLOCSW: LOCATIONGRPTYPE=LAC, LOCATIONGRPNAME="lac1", SWITCH=ENABLE, IPV6SWITCH=DISABLE;
```

**典型脚本(TAC 组 IPv6 启用 IPv4 关闭)**:
```
SET IPALLOCBYLOCSW: LOCATIONGRPTYPE=TAC, LOCATIONGRPNAME="tac1", SWITCH=DISABLE, IPV6SWITCH=ENABLE;
```

**典型脚本(TAC 组继承全局配置)**:
```
SET IPALLOCBYLOCSW: LOCATIONGRPTYPE=TAC, LOCATIONGRPNAME="tac1", SWITCH=INHERIT, IPV6SWITCH=INHERIT;
```

**步骤位置**:在所属 feature 编排中位于"位置区组级开关配置"——前置 [0-00321](task/UDG/20.15.2/0-00321.md) SET IPALLOCBYLOCGLBSW(全局开关)+ [0-00310](task/UDG/20.15.2/0-00310.md) ADD LACGROUP 或 [0-00316](task/UDG/20.15.2/0-00316.md) ADD TACGROUP(位置区组对象预建);下游 位置区组维度位置分配地址使能决策生效;入不转板流程 [0-00249](task/UDG/20.15.2/0-00249.md) SET DATAPLANEINFMODE(总开关)。

## 决策点:位置区组级开关取值模式(DP 0-00322)

问:位置区组级 SWITCH/IPV6SWITCH 取 INHERIT 还是 DISABLE/ENABLE?决定位置区组维度的位置分配地址使能策略。

| 选项 | SWITCH/IPV6SWITCH | 含义 | 适用场景 |
|---|---|---|---|
| 继承全局(默认) | INHERIT | 跟全局 IPALLOCBYLOCGLBSW 配置走 | 位置分配地址全网一致的简单场景 |
| 组级强制关闭 | DISABLE | 本组位置分配地址强制关闭,不受全局影响 | 特定位置区组(如漫游边界)禁用位置分配 |
| 组级强制开启 | ENABLE | 本组位置分配地址强制开启,不受全局影响 | 特定位置区组(如热点区域)优先位置分配 |

各取值按位置区组维度独立生效;INHERIT/DISABLE/ENABLE 三态互斥。

## 约束

- **适用 NF 限定 PGW-U/UPF**(critical,rule-0-00322):本命令仅适用于 PGW-U/UPF,控制面 SMF 等网元不适用 — 后果:非 PGW-U/UPF 网元命令拒收
- **LOCATIONGRPNAME 必须预建**(critical,rule-0-00322):LOCATIONGRPNAME 引用的位置区组必须已通过 ADD LACGROUP(LOCATIONGRPTYPE=LAC)或 ADD TACGROUP(LOCATIONGRPTYPE=TAC)预建 — 后果:位置区组未建引用命令拒收
- **LOCATIONGRPTYPE 与 LOCATIONGRPNAME 匹配**(critical,rule-0-00322):LOCATIONGRPTYPE=LAC 必须配 LACGROUP;TAC 必须配 TACGROUP — 后果:类型与组名错配命令拒收
- **LOCATIONGRPNAME 长度 1~32**(warning,rule-0-00322):字符串长度范围为 1~32 — 后果:超长/超短命令拒收
- **最大记录数 9000**(critical,rule-0-00322):整机位置区组级开关最大 9000 条(LACGROUP+TACGROUP 总数) — 后果:超规格新增失败
- **执行后立即生效**(warning,rule-0-00322):命令执行后立即生效 — 后果:误操作即刻生效
- **INHERIT 三态继承全局**(critical,rule-0-00322):SWITCH=INHERIT 时按全局 IPALLOCBYLOCGLBSW 配置分配地址;非 INHERIT 时按本组级配置 — 后果:全局与组级决策混淆时位置分配行为不可预期
- **两参数独立三态**(warning,rule-0-00322):SWITCH 与 IPV6SWITCH 独立三态(DISABLE/ENABLE/INHERIT),无联动关系 — 后果:IPv4/IPv6 协议族使能各自独立
