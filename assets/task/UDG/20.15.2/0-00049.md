# 配置APN的QoS带宽控制属性（SET APNQOSATTR）

> 命令静态知识见 [SET APNQOSATTR](command/UDG/20.15.2/SET-APNQOSATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令为某 APN 打开上下行 QoS 带宽控制开关并选定控制方式（CAR=超出丢弃 / SHAPE=缓存整形）。开关与方式成对出现：上行一对、下行一对，共 4 个参数 + 1 个 APN 名。

**参数对结构**

| 方向 | 开关参数 | 控制方式参数 | 典型取值 |
|---|---|---|---|
| 上行 | CARSHAPESWUL | CARSHAPEUL | ENABLE + CAR |
| 下行 | CARSHAPESWDL | CARSHAPEDL | ENABLE + CAR |

规律：开关=ENABLE 时，对应控制方式参数必填（conditional_required）；开关=INHERIT 时控制方式可留空。

**CAR vs SHAPE 选择**

| 取值 | 行为 |
|---|---|
| CAR | 超出带宽直接丢弃（硬限速，本特性场景统一取此值） |
| SHAPE | 缓存整形（软限速，平滑流量峰值） |

**典型行（三个特性场景一致）**
```
SET APNQOSATTR: APN="apn1", CARSHAPESWUL=ENABLE, CARSHAPEUL=CAR, CARSHAPESWDL=ENABLE, CARSHAPEDL=CAR;
```
> 数据规划表只列 APN + CARSHAPEUL/DL（取值 CAR），但任务示例脚本额外含 CARSHAPESWUL/DL=ENABLE 开关——全录以保正向实例化可复现脚本。三个特性（GWFD-010151 接入控制 / GWFD-020381 会话类 QoS 保证 / GWFD-010201 QoS 与流量管理）数据规划与脚本完全一致。

**前置依赖**：APN 需先由 ADD APN（0-00020）配置，可用 LST APN 查询确认。

## 约束

- **rule-0-00153**（info）最大记录数 10000；命令只对**新激活用户**生效（存量用户不受影响）；系统初始记录为 CARSHAPESWUL/DL=INHERIT、CARSHAPEUL/DL=NULL — 后果：超规格新增失败；改配置后需用户重新激活才生效
