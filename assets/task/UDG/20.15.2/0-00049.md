# 配置APN的QoS带宽控制属性（SET APNQOSATTR）

> 命令静态知识见 [SET APNQOSATTR](command/UDG/20.15.2/SET-APNQOSATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

SET APNQOSATTR 是 APN 级"QoS 带宽控制开关"：针对指定 APN 配上下行 QoS 带宽控制总开关（CARSHAPESWUL/DL）和控制方式（CAR=超出丢弃 / SHAPE=缓存整形），实现该 APN 下用户的接入带宽控制。命令核心配置空间有 5 个维度：**APN 引用**（APN 必选 + 必须已建）、**上行开关**（CARSHAPESWUL）、**上行方式**（CARSHAPEUL 条件必选）、**下行开关**（CARSHAPESWDL）、**下行方式**（CARSHAPEDL 条件必选）。3 个特性命中（010151 接入控制 / 010201 QoS与流量管理 / 020381 会话类QoS保证）的数据规划与脚本完全一致：均取 ENABLE + CAR。

### 配置维度 1：APN 引用（参数 APN，必选）

| 参数 | 范围 | 备注 |
|---|---|---|
| **APN** | 字符串 1~63 | 引用已配的 [ADD APN](task/UDG/20.15.2/0-00020.md) APN；前置依赖：APN 必须先建；每个 APN 仅一条本配置 |

### 配置维度 2：上行开关（参数 CARSHAPESWUL，可选）

| 取值 | 作用 |
|---|---|
| **ENABLE** | 开启上行 QoS 带宽控制；**必填** CARSHAPEUL（条件必选） |
| **DISABLE** | 关闭上行 QoS 带宽控制 |
| **INHERIT**（默认/初始） | 继承系统默认；CARSHAPEUL 可留空 |

### 配置维度 3：上行方式（参数 CARSHAPEUL，开关 ENABLE 时条件必选）

| 取值 | 作用 |
|---|---|
| **CAR** | 超出带宽直接丢弃（**硬限速**，本特性场景统一取此值） |
| **SHAPE** | 缓存整形（**软限速**，平滑流量峰值） |

### 配置维度 4：下行开关（参数 CARSHAPESWDL，可选）

| 取值 | 作用 |
|---|---|
| **ENABLE** | 开启下行 QoS 带宽控制；**必填** CARSHAPEDL（条件必选） |
| **DISABLE** | 关闭下行 QoS 带宽控制 |
| **INHERIT**（默认/初始） | 继承系统默认；CARSHAPEDL 可留空 |

### 配置维度 5：下行方式（参数 CARSHAPEDL，开关 ENABLE 时条件必选）

| 取值 | 作用 |
|---|---|
| **CAR** | 超出带宽直接丢弃（**硬限速**，本特性场景统一取此值） |
| **SHAPE** | 缓存整形（**软限速**，平滑流量峰值） |

**典型脚本（3 特性场景统一范式：上下行均 ENABLE + CAR）**：
```
SET APNQOSATTR:APN="apn",CARSHAPESWUL=ENABLE,CARSHAPEUL=CAR,CARSHAPESWDL=ENABLE,CARSHAPEDL=CAR;
```

> 命令 notes：适用 NF：SGW-U、PGW-U、UPF；执行后**只对新激活用户生效**；系统初始记录为 CARSHAPESWUL/DL=INHERIT、CARSHAPEUL/DL=NULL。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：APN 带宽控制属性配置"——前置依赖 [0-00020](task/UDG/20.15.2/0-00020.md)（ADD APN）；按需在 QoS/带宽控制 backbone（[0-00034](task/UDG/20.15.2/0-00034.md)~[0-00042](task/UDG/20.15.2/0-00042.md)）之后插入。

## 决策点：上下行开关与方式组合（DP 0-00049）

问：SET APNQOSATTR 选哪个上下行开关 + 方式组合？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 上下行均 CAR 限速 | CARSHAPESWUL=ENABLE + CARSHAPEUL=CAR + CARSHAPESWDL=ENABLE + CARSHAPEDL=CAR | 接入控制（010151）/ QoS与流量管理（010201）/ 会话类QoS保证（020381） |
| 上下行均 SHAPE 整形 | CARSHAPESWUL=ENABLE + CARSHAPEUL=SHAPE + CARSHAPESWDL=ENABLE + CARSHAPEDL=SHAPE | 软限速平滑峰值（命令支持，3 个特性未演示） |
| 单方向 CAR 限速 | 仅 ENABLE 一侧 + CARSHAPE*=CAR | 单方向控制（命令支持，3 个特性未演示） |
| 全 INHERIT 继承默认 | CARSHAPESWUL=INHERIT + CARSHAPESWDL=INHERIT | 系统初始状态（默认值） |

> 3 个特性命中以"上下行均 CAR"为统一范式；SHAPE/单方向/INHERIT 等组合命令支持但本场景未演示。

## 约束

- **APN 前置依赖**（critical，rule-0-00449）：APN 必须先由 [0-00020](task/UDG/20.15.2/0-00020.md)（ADD APN）建好 — 后果：未建 APN 命令拒收
- **APN 唯一**（critical，rule-0-00449）：每个 APN 仅一条带宽控制属性配置 — 后果：同 APN 重复配置命令拒收
- **开关 ENABLE 必填方式**（critical，rule-0-00449）：CARSHAPESWUL=ENABLE 必配 CARSHAPEUL；CARSHAPESWDL=ENABLE 必配 CARSHAPEDL — 后果：开关 ENABLE 缺方式命令拒收
- **开关 INHERIT 方式留空**（warning，rule-0-00449）：CARSHAPESW*=INHERIT 时 CARSHAPE* 应留空（系统初始状态） — 后果：INHERIT 配方式行为未定义
- **只对新激活用户生效**（critical，rule-0-00449）：执行后已激活用户不受影响 — 后果：改配置需用户重激活才生效
- **最大记录数 10000**（warning，rule-0-00449）：超规格新增失败 — 后果：超 10000 拒收
- **适用 NF 限定**（warning，隐含）：SGW-U、PGW-U、UPF — 后果：其他 NF 不适用
- **CAR/SHAPE 互斥**（warning，rule-0-00449）：同一方向同一时刻仅一种方式（CAR 硬限速 or SHAPE 软限速） — 后果：同方向重复配方式命令拒收
