# 配置跨VPN上行静态路由(路由交叉)（ADD AUTOSCALINGSRROUTE）

> 命令静态知识见 [ADD AUTOSCALINGSRROUTE](command/UDG/20.15.2/ADD-AUTOSCALINGSRROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是路由交叉功能（GWFD-020423）跨 VPN 上行报文转发的核心。在源 VRF（VRF_networkA）中配到网络 B 服务器 IP 的静态路由，下一跳指向网络 B 网关，实现 UE 访问网络 B 定制服务器。路由前缀/下一跳来自对端（变更须与 UPF 协商），BFD 是否启用、单臂还是双向由 DP 0-00024 决定并联动 BFD 模板选择。

**配置范式**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| SERVICENAME | `ServName_networkB` | 引用 [0-00141](task/UDG/20.15.2/0-00141.md) | 已配置服务名 |
| VRFNAME | `VRF_networkA` | 引用 [0-00071](task/UDG/20.15.2/0-00071.md) | 源 VRF（与网络A互通的接口绑定的VPN） |
| IPVERSION | `IPv4` | 本端规划 | 决定 IPv4 参数族 |
| PREFIX4 | 如 `10.8.8.8` | 从对端获取 | 对端服务器 IP，变更须与 UPF 协商 |
| MASKLENGTH | 如 `32` | 从对端获取 | 与 PREFIX4 配套的服务器掩码 |
| NEXTHOPALLOCTYPE4 | `CONFIG` | 本端规划 | IPv4 下一跳分配方式 |
| NEXTHOP4 | 如 `192.168.20.1` | 从对端获取 | 网关接口IP，UPF 报文发往该直连交换机/路由器 |
| BFDENABLE | `ENABLE` | 本端规划 | 单/双向 BFD 探测时均设为 ENABLE |
| BFDTEMPLATENAME | 如 `BFD_10` | 引用 [0-00143](task/UDG/20.15.2/0-00143.md).TEMPLATENAME | **仅单臂 echo BFD 时填**；双向 BFD 不填（见决策点） |

**示例脚本（单臂 BFD 探测场景）**：
```
ADD AUTOSCALINGSRROUTE: SERVICENAME="ServName_networkB", VRFNAME="VRF_networkA",
  IPVERSION=IPv4, PREFIX4="10.8.8.8", MASKLENGTH=32,
  NEXTHOPALLOCTYPE4=CONFIG, NEXTHOP4="192.168.20.1",
  BFDENABLE=ENABLE, BFDTEMPLATENAME="BFD_10";
```
采用 BFD 双向检测时，省略 BFDTEMPLATENAME（改由 [0-00144](task/UDG/20.15.2/0-00144.md) 联动）。

## 决策点

- **DP-0-00024**（BFD 类型）：决定 BFDTEMPLATENAME 是否填写及联动模板。
  - 选 opt-bfd-one-arm（单臂 echo）→ 本任务 BFDTEMPLATENAME 必填，引用 [0-00143](task/UDG/20.15.2/0-00143.md)；不建 0-00144。
  - 选 opt-bfd-two-way（双向）→ 本任务不填 BFDTEMPLATENAME；依赖 [0-00144](task/UDG/20.15.2/0-00144.md) 先于本任务生效。

## 约束

- **rule-0-00231**（warning）最大记录数 2048；前置 SERVICENAME 值相同的 ADD AUTOSCALINGSERVICE（0-00141）须已配置；使能动态 BFD（DP 0-00024=双向）时 SERVICENAME 值相同的 ADD AUTOSCALINGSRBFD（0-00144）须已配置；DP 0-00024=单臂时 BFDTEMPLATENAME 必填且引用 ADD AUTOSCALINGBFD（0-00143）；IPv4 路由掩码 0~32 / IPv6 0~128；立即生效 — 后果：超规格/前置缺失/BFD 模板不匹配→新增失败或忽略
