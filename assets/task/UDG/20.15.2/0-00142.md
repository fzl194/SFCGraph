# 配置跨VPN上行静态路由(路由交叉)（ADD AUTOSCALINGSRROUTE）

> 命令静态知识见 [ADD AUTOSCALINGSRROUTE](command/UDG/20.15.2/ADD-AUTOSCALINGSRROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD AUTOSCALINGSRROUTE 用于配置路由交叉功能（GWFD-020423）跨 VPN 上行报文转发的静态路由。**核心机制**：在源 VRF（VRF_networkA）中配到网络 B 服务器 IP 的静态路由，下一跳指向网络 B 网关，实现 UE 访问网络 B 定制服务器。**路由前缀/下一跳来自对端**（变更须与 UPF 协商）；BFD 联动与否由决策点决定。1 个特性命中（020423 路由交叉）。

### 配置维度 1：地址族（参数 IPVERSION，必选）

| 取值 | 配套参数族 | 适用场景 |
|---|---|---|
| **IPv4** | PREFIX4/MASKLENGTH/NEXTHOP4/NEXTHOPALLOCTYPE4 | 020423 范式 |
| **IPv6** | PREFIX6/PREFIXLENGTH/NEXTHOP6/NEXTHOPALLOCTYPE6 | IPv6 路由交叉 |

### 配置维度 2：BFD 联动（参数 BFDENABLE + BFDTEMPLATENAME，DP 0-00142 决定）

| BFDENABLE | BFDTEMPLATENAME | 联动模板 | 适用场景 |
|---|---|---|---|
| **DISABLE** | 不填 | 无 | 不启用 BFD 探测 |
| **ENABLE** | 填（单臂 echo） | 引用 [0-00143](task/UDG/20.15.2/0-00143.md).TEMPLATENAME | 单臂 BFD 探测 |
| **ENABLE** | 不填（双向） | 依赖 [0-00144](task/UDG/20.15.2/0-00144.md) 联动 | 双向 BFD 探测 |

> 单臂 BFD 时 BFDTEMPLATENAME **必填**；双向 BFD 时 BFDTEMPLATENAME **不填**，由 SRBFD 模板（0-00144）联动。

### 配置维度 3：下一跳分配方式（参数 NEXTHOPALLOCTYPE4/6）

| 取值 | 作用 |
|---|---|
| **CONFIG** | 静态配置下一跳（020423 范式取此值） |
| **其他** | 见命令真相表枚举 |

**典型脚本（单臂 BFD 探测 IPv4 020423 范式）**：

```
ADD AUTOSCALINGSRROUTE: SERVICENAME="ServName_networkB", VRFNAME="VRF_networkA",
  IPVERSION=IPv4, PREFIX4="10.8.8.8", MASKLENGTH=32,
  NEXTHOPALLOCTYPE4=CONFIG, NEXTHOP4="192.168.20.1",
  BFDENABLE=ENABLE, BFDTEMPLATENAME="BFD_10";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：配跨 VPN 静态路由"——前置 [0-00141](task/UDG/20.15.2/0-00141.md) ADD AUTOSCALINGSERVICE 配 SERVICENAME + [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST 配 VRFNAME；BFD 单臂时前置 [0-00143](task/UDG/20.15.2/0-00143.md)，BFD 双向时前置 [0-00144](task/UDG/20.15.2/0-00144.md)。

## 决策点：BFD 联动类型（DP 0-00142）

| 选项 | BFDENABLE | BFDTEMPLATENAME | 联动模板 | 适用场景 |
|---|---|---|---|---|
| **opt-bfd-disable** | DISABLE | 不填 | 无 | 无 BFD 探测 |
| **opt-bfd-one-arm（单臂 echo）** | ENABLE | 必填（引用 0-00143） | [0-00143](task/UDG/20.15.2/0-00143.md) ADD AUTOSCALINGBFD | 单臂 echo BFD |
| **opt-bfd-two-way（双向）** | ENABLE | 不填 | [0-00144](task/UDG/20.15.2/0-00144.md) ADD AUTOSCALINGSRBFD | 双向 BFD，须 0-00144 先生效 |

## 约束

- **最大记录数 2048**（warning，rule-0-00142）：系统最多 2048 条 AUTOSCALINGSRROUTE — 后果：超规格新增失败
- **SERVICENAME 前置必备**（critical，rule-0-00142）：前置 SERVICENAME 值相同的 ADD AUTOSCALINGSERVICE(0-00141) 须已配置 — 后果：前置缺失命令拒收
- **BFD 双向时 SRBFD 前置**（critical，rule-0-00142）：DP 0-00142=双向时 SERVICENAME 值相同的 ADD AUTOSCALINGSRBFD(0-00144) 须已配置 — 后果：双向 SRBFD 未生效命令拒收
- **BFD 单臂时 BFDTEMPLATENAME 必填**（critical，rule-0-00142）：DP 0-00142=单臂时 BFDTEMPLATENAME 必填且引用 ADD AUTOSCALINGBFD(0-00143) — 后果：模板名缺失或引用错命令拒收
- **路由掩码范围**（warning，rule-0-00142）：IPv4 掩码 0~32 / IPv6 0~128 — 后果：超范围命令拒收
- **立即生效**（warning，rule-0-00142）：执行后立即生效 — 后果：误配即刻影响跨 VPN 转发
