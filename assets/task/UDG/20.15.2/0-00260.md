# 增加ICAP服务器组（ADD ICAPSVRGRP）

> 命令静态知识见 [ADD ICAPSVRGRP](command/UDG/20.15.2/ADD-ICAPSVRGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务创建 ICAP 服务器组（ICAPSVRGRP）——作为 URL 过滤模板（[ADD CFTEMPLATE](task/UDG/20.15.2/0-00253.md) 的 ICAPSRVGMNAME 主用引用）与 ICAP 服务器绑定关系（[ADD ICAPSVRBINDISG](task/UDG/20.15.2/0-00261.md) 引用）共享的容器。`ICAPSVRGRPNAME` 供后续绑定关系引用。适用 NF：PGW-U、UPF。

### 配置维度 1：ICAP 服务器组类型（参数 ICAPSERVERTYPE，必选）
| 取值 | 作用 |
|---|---|
| URL_FILTERING | URL 内容过滤场景的 ICAP Server 容器（主特性样例） |
| CONTENT_FILTERING | 内容过滤场景的 ICAP Server 容器 |

### 配置维度 2：服务器组名称（参数 ICAPSVRGRPNAME，必选）
| 取值 | 含义 | 约束 |
|---|---|---|
| 字符串 | ICAP Server Group 名（供 0-00253/0-00261 引用） | 1~31 字符，**不支持空格**，**不区分大小写** |

### 配置维度 3：配置域（参数 CFGDOMAINNAME，可选）
| 取值 | 作用 |
|---|---|
| 字符串 1~31 字符 | 命令所属公共配置域 |

**主特性脚本样例**（GWFD-110471）

```
ADD ICAPSVRGRP: ICAPSVRGRPNAME="icapsvrgrp1", ICAPSERVERTYPE=URL_FILTERING;
```

**步骤位置**：URL 过滤 / ICAP 互通数据准备阶段（先建组，再 [ADD ICAPSERVER](task/UDG/20.15.2/0-00259.md) + [ADD ICAPSVRBINDISG](task/UDG/20.15.2/0-00261.md)）。

**未绑参数说明（按 baseline §4.1 冷门不绑）**：CFGDOMAINNAME（配置域名称，optional，主场景不配）。

## 决策点：ICAP 服务器组类型分叉（DP 0-00260）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| URL_FILTERING（主特性样例） | ICAPSERVERTYPE=URL_FILTERING | GWFD-110471 URL 过滤 / 内容过滤场景 |
| CONTENT_FILTERING | ICAPSERVERTYPE=CONTENT_FILTERING | 内容过滤细分场景（非本批） |

## 约束

- **rule-0-00453**（critical）：命令最大记录数 **30**；每种类型下最大 30 个；超限 ADD 失败
- **rule-0-00454**（warning）：ICAPSVRGRPNAME 字符串 1~31 字符，不支持空格、不区分大小写 — 名称非法被拒
- **rule-0-00455**（warning）：本任务下游前置——组先建后才允许 [ADD ICAPSERVER](task/UDG/20.15.2/0-00259.md) + [ADD ICAPSVRBINDISG](task/UDG/20.15.2/0-00261.md)（组为容器）
- **rule-0-00456**（info）：命令执行后立即生效
