# 设置FPI差异化控制功能（SET FPIFUNC）

> 命令静态知识见 [SET FPIFUNC](command/UDG/20.15.2/SET-FPIFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。命令为整机级全局开关。

## 配置方法

SET FPIFUNC 是 FPI（Forwarding Plane Identifier）差异化控制功能开关，**全局唯一**。**FPISWITCH 必选**（DISABLE/ENABLE 切换），**FPITRANSMETHOD 仅当 FPISWITCH=ENABLE 时必选**，**FPIPOLICYORIGIN 仅当 FPITRANSMETHOD=GTPU 时必选**。核心配置空间有 3 个维度：**使能开关**（FPISWITCH）、**传递类型**（FPITRANSMETHOD）、**策略来源**（FPIPOLICYORIGIN）。1 个特性命中（GWFD-110331）覆盖 DSCP 传递范式。

### 配置维度 1：使能开关（参数 FPISWITCH，必选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 不使能 FPI 差异化控制（默认） |
| **ENABLE** | 使能 FPI 差异化控制；启用后 FPITRANSMETHOD 必配 |

### 配置维度 2：传递类型（参数 FPITRANSMETHOD，FPISWITCH=ENABLE 时条件必选）

| 取值 | 作用 |
|---|---|
| **GTPU** | 通过 GTP-U 报文头传递 FPI；启用后 FPIPOLICYORIGIN 必配 |
| **DSCP** | 通过报文 DSCP 字段传递 FPI（GWFD-110331 范式） |

> DSCP 模式不需 FPIPOLICYORIGIN；GTPU 模式必须再指定策略来源。

### 配置维度 3：策略来源（参数 FPIPOLICYORIGIN，FPITRANSMETHOD=GTPU 时条件必选）

| 取值 | 作用 |
|---|---|
| **见命令真相表枚举** | 指定 FPI 策略来自哪个控制面（仅 GTPU 模式有意义） |

**典型脚本（GWFD-110331 业务流标识无线资源优化，DSCP 传递）**：
```
SET FPIFUNC:FPISWITCH=ENABLE,FPITRANSMETHOD=DSCP;
```

**典型脚本（GTPU 传递 + 策略来源）**：
```
SET FPIFUNC:FPISWITCH=ENABLE,FPITRANSMETHOD=GTPU,FPIPOLICYORIGIN=...;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：使能 FPI 差异化"——前置业务流识别与策略计算模块已配好（FPI 与策略/BWM 链路协同）。

## 决策点：FPI 传递方式选择（DP 0-00109）

问：本端使用哪种 FPI 传递方式？

| 选项 | FPISWITCH | FPITRANSMETHOD | FPIPOLICYORIGIN | 适用场景 |
|---|---|---|---|---|
| **DSCP 传递**（110331） | ENABLE | DSCP | 不配 | 基于业务流标识的无线资源优化、走 DSCP 链路 |
| **GTPU 传递** | ENABLE | GTPU | 必配 | 通过 GTP-U 报文头传递策略标识 |
| **关闭** | DISABLE | 不配 | 不配 | 不使能 FPI |

> 1 个特性命中（110331）；默认 DISABLE；FPISWITCH=ENABLE 时必须配 FPITRANSMETHOD。

## 约束

- **全局唯一**（warning，rule-0-00109）：FPI 功能全局唯一，重复执行覆盖前值 — 后果：覆盖前配置
- **FPITRANSMETHOD 条件必选**（critical，rule-0-00109）：FPISWITCH=ENABLE 时 FPITRANSMETHOD 必选 — 后果：ENABLE 时缺 FPITRANSMETHOD 命令拒收
- **FPIPOLICYORIGIN 条件必选**（critical，rule-0-00109）：FPITRANSMETHOD=GTPU 时 FPIPOLICYORIGIN 必选 — 后果：GTPU 模式缺 FPIPOLICYORIGIN 命令拒收
- **执行后立即生效**（warning，rule-0-00109）：本命令执行后立即生效 — 后果：误切换即刻影响 FPI 差异化控制链路
- **DSCP 模式不需策略来源**（info，rule-0-00109）：DSCP 模式不需 FPIPOLICYORIGIN — 后果：DSCP 模式配 FPIPOLICYORIGIN 命令忽略/拒收
