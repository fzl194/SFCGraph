# 设置FPI差异化控制功能（SET FPIFUNC）

> 命令静态知识见 [SET FPIFUNC](command/UDG/20.15.2/SET-FPIFUNC.md)

## 配置方法

GWFD-110331 业务流标识无线资源优化的**首步全局开关**——使能 FPI 功能并选择向 RAN 侧传递 FPI 信息的方式。全局 SET 类命令（整机一份），系统初始记录为 FPISWITCH=ENABLE / FPITRANSMETHOD=DSCP / FPIPOLICYORIGIN=NULL。

**配置范式**（110331）：

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| FPISWITCH | ENABLE | 全网规划 | 使能 FPI 功能（需 RAN/PCRF 协同） |
| FPITRANSMETHOD | DSCP | 全网规划（DP 编排） | FPI 向 RAN 侧传递方式，按 RAN 能力选 DSCP/GTPU |

```
SET FPIFUNC:FPISWITCH=ENABLE,FPITRANSMETHOD=DSCP;
```

**FPITRANSMETHOD 传递机制**：
- DSCP：将用户下行报文 DSCP 值映射到 GTP-U 扩展头，要求无线侧识别 DSCP（本特性默认）。
- GTPU：以 GTP-U 扩展头方式传递 FPI，要求无线侧识别 GTP-U 扩展头。

**FPIPOLICYORIGIN 策略来源**（默认 NULL，主特性场景保持）：INTERNAL 表示 FPI 值来源于网关内部策略，需配带 FPI 信息的规则；NULL/EXTERNAL 表示将下行报文 DSCP 映射到 GTP-U 扩展头。

**步骤位置**：特性首步（在 SET LICENSESWITCH 打开 license 之后、配置三四层过滤规则之前）。

## 决策点

**DP 0-00139**（单选）：FPI 向 RAN 侧传递方式 2-way 编排（FPISWITCH=ENABLE 时必选）。

| 选项 | FPITRANSMETHOD | 要求 |
|---|---|---|
| DSCP 传递（本特性默认） | DSCP | 无线侧支持 DSCP 识别，与系统初始记录一致 |
| GTP-U 传递 | GTPU | 无线侧支持 GTP-U 扩展头识别 |

注：DP 0-00138 为 FPIPOLICYORIGIN × FPITRANSMETHOD 联合编排（4-way），与 0-00139 互补；主特性默认组合为 NULL+DSCP，不触发 0-00138。

## 约束

- 最大记录数 1（全局 SET 类） · warning · 超限 SET 失败
- FPITRANSMETHOD 仅 FPISWITCH=ENABLE 时为必选（conditional） · warning · DISABLE 时传会被拒收
- 开启后要求无线侧同时支持该功能 · warning · DSCP/GTPU 按 RAN 能力选
- 系统初始记录 FPISWITCH=ENABLE/FPITRANSMETHOD=DSCP/FPIPOLICYORIGIN=NULL · info
- 命令执行后立即生效 · info
