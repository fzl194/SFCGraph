# 配置QoS重定向下一跳动作（ADD QOSACTRDRNHP）

> 命令静态知识见 [ADD QOSACTRDRNHP](command/UDG/20.15.2/ADD-QOSACTRDRNHP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令把"重定向下一跳"流动作挂到 [0-00199](task/UDG/20.15.2/0-00199.md) ADD MQCBEHAVIOR 已建的空 Behavior 容器；BEHAVIORNAME 引上游已建行为。

## 配置方法（配置方法字典）

ADD QOSACTRDRNHP 是"配置 QoS 重定向下一跳动作"命令。3 参必选（BEHAVIORNAME/IPVERSION/NEXTHOP 或 NEXTHOP6）+ 2 参可选（IFNAME/VRFNAME）。配置空间有 2 个维度：**IP 版本**（IPVERSION 决定 NEXTHOP/NEXTHOP6 配套）、**出接口/VPN**（IFNAME 决定强/弱策略路由）。1 个特性命中（IPFD-012001 QoS 复杂流分类）覆盖 IPv4 强策略路由范式。

### 配置维度 1：IP 版本与下一跳（参数 IPVERSION + NEXTHOP/NEXTHOP6，必选）

| IPVERSION | 联动必选下一跳 | 下一跳取值 | 适用场景 |
|---|---|---|---|
| **IPv4** | NEXTHOP | IPv4 地址（如 192.168.2.3） | IPFD-012001 范例（公网重定向） |
| **IPv6** | NEXTHOP6 | IPv6 地址 | IPv6 流重定向（本 atom 未涉及，按需取） |

> **强/弱策略路由切换**：配 IFNAME（出接口）= **强策略路由**（不查 FIB 表，直接通过指定下一跳+出接口转发）；仅配 NEXTHOP+VRFNAME 不配 IFNAME= **弱策略路由**（重定向到私网 IP，需查 FIB）。

### 配置维度 2：出接口 / VPN（参数 IFNAME + VRFNAME，可选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **IFNAME** | 出接口（强策略路由） | 指定出接口时 NEXTHOP+IFNAME 直转发，不查 FIB |
| **VRFNAME** | VPN 名称（弱策略路由） | 重定向到私网 IP 时必配 VPN 实例 |

> 强策略路由（IFNAME+NEXTHOP）：用户期望流量按指定下一跳+出接口转发；弱策略路由（NEXTHOP+VRFNAME）：重定向到私网 IP，按 VPN 实例转发。

**典型脚本（IPFD-012001 QoS 复杂流分类，公网 IPv4 重定向）**：
```
ADD QOSACTRDRNHP: BEHAVIORNAME="behavior1", IPVERSION=IPv4, NEXTHOP="192.168.2.3";
```

> 命令 notes：**最大记录数 8192**；**NP 卡场景不支持**（主场景适用 PUP 卡）；IFNAME（可选）/VRFNAME（可选）；执行后立即生效；与同流行为下重标记/deny/级联/URPF/CAR **互斥**（permit 可共存）。

**步骤位置**：在所属 feature 编排中位于"步骤 N：挂重定向动作"——前置 [0-00199](task/UDG/20.15.2/0-00199.md) ADD MQCBEHAVIOR（建空 Behavior）；下游 [0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY（CB 节点绑定 BEHAVIORNAME）。

## 决策点：强/弱策略路由选择（DP 0-00200）

问：ADD QOSACTRDRNHP 使用哪种路由模式？（决定 IFNAME/VRFNAME/NEXTHOP 参数组合）

| 选项 | IFNAME | VRFNAME | NEXTHOP/NEXTHOP6 | 行为 | 适用场景 |
|---|---|---|---|---|---|
| **公网强策略路由**（IPFD-012001 范例） | 不配 | 不配 | NEXTHOP=IPv4 | 不查 FIB 直转发下一跳 | IPFD-012001 范式 |
| **公网强策略路由+出接口** | 配 | 不配 | NEXTHOP=IPv4 | 直转发指定下一跳+出接口 | 强制出口接口 |
| **私网弱策略路由** | 不配 | 配 | NEXTHOP=IPv4 | 私网 IP 重定向（VRFNAME 决 VPN） | 重定向到私网 IP |
| **IPv6 重定向** | 可选 | 可选 | NEXTHOP6=IPv6 | IPv6 流重定向 | IPv6 流量分类 |

> 1 个特性命中（IPFD-012001）覆盖公网强策略路由（不带 IFNAME/VRFNAME）范式；其余按需取。

## 约束

- **最大记录数 8192**（critical，rule-0-00288）：整机重定向动作最大 8192 条 — 后果：超规格新增失败
- **NP 卡场景不支持**（critical，rule-0-00288）：NP 卡场景下本命令不可用，仅 PUP 卡支持 — 后果：NP 卡上命令拒收
- **执行后立即生效**（warning，rule-0-00288）：执行后立即生效 — 后果：误操作即刻生效
- **IPVERSION 与 NEXTHOP 必须配套**（critical，rule-0-00200）：IPv4→NEXTHOP；IPv6→NEXTHOP6（IPv6 时不配 NEXTHOP6 命令拒收） — 后果：不配套命令拒收
- **流动作互斥约束**（critical，rule-0-00200）：与同流行为下重标记/deny/级联/URPF/CAR 互斥（permit 可共存） — 后果：互斥动作并发命令拒收
- **BEHAVIORNAME 必须先经 ADD MQCBEHAVIOR 建**（critical，rule-0-00200）：BEHAVIORNAME 引用的 Behavior 必须先由 [0-00199](task/UDG/20.15.2/0-00199.md) ADD MQCBEHAVIOR 定义 — 后果：未建引用失败
- **BEHAVIORNAME 长度**（info，rule-0-00200）：字符串 1~31 — 后果：超长命令拒收
- **IFNAME 必须已建接口**（warning，rule-0-00200）：配 IFNAME 时引用的接口必须先由 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 定义 — 后果：未建接口引用失败
