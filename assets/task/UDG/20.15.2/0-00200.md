# 配置QoS重定向下一跳动作（ADD QOSACTRDRNHP）

> 命令静态知识见 [ADD QOSACTRDRNHP](command/UDG/20.15.2/ADD-QOSACTRDRNHP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

把流动作"重定向下一跳"挂到 [ADD MQCBEHAVIOR](0-00199.md) 建好的空行为容器上。BEHAVIORNAME 引上游已建行为，IPVERSION/NEXTHOP 本端规划。

**数据规划取值样例**（证据③，IPFD-012001 数据规划表）：

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| BEHAVIORNAME | `behavior1` | 引用（0-00199） | 引上游流行为 |
| IPVERSION | `IPv4` | 本端规划 | IPv6 场景用 NEXTHOP6（当前 atom 未绑，缺口） |
| NEXTHOP | `192.168.2.3`（公网IP） | 本端规划 | 下一跳地址 |

**典型范式**（证据③任务示例脚本）：
```
ADD QOSACTRDRNHP: BEHAVIORNAME="behavior1", IPVERSION=IPv4, NEXTHOP="192.168.2.3";
```

> 缺口标注：命令真相另有 IFNAME/VRFNAME（optional，出接口/VPN名，冷门）和 NEXTHOP6（IPVERSION=IPv6 分支必选），主场景未用到，待对应场景揭示后补绑。

## 约束

- **rule-0-00288**（warning）最大记录数 **8192**；**NP 卡场景不支持**（主场景适用 PUP 卡）；与同流行为下重标记/deny/级联/URPF/CAR **互斥**（仅 permit 可共存）；IPVERSION 与 NEXTHOP/NEXTHOP6 配套（IPv4→NEXTHOP，IPv6→NEXTHOP6）；执行后立即生效。
