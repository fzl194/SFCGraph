# 静态地址用户路由冗余（GWFD-010107）

> 特性静态知识见 [GWFD-010107](feature/UDG/20.15.2/GWFD-010107.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-010107-静态地址用户路由冗余.md)。
>
> **本页定位**：配置生成实例化视角——本特性是"主备 UDG + GRE 隧道 + 静态地址池"三要素的**可靠性执行特性**。复用 GWFD-010104 的地址池配置体系，新增 GRE 隧道/重定向 IP/双开关/路由优先级控制。

## 配置概览

本特性按 **3 大要素 × 15 步骨架 × 主备双机对称配置** 组织：

**对象链**：
```
静态地址池（继承 GWFD-010104，POOLTYPE=EXTERNAL + REDUNDFUNC=ENABLE）
   → ADD SECTION / ADD POOLGROUP / ADD POOLBINDGROUP
   → ADD APN + ADD POOLGRPMAP
   ↓
   冗余备份体系（★本特性核心）
   ├── ADD REDUNDRDTIP（虚拟重定向 IP，业务流到 GRE Tunnel）
   ├── ADD INTERFACE（LoopBack + Tunnel 接口） + ADD IPBINDVPN（LB→vpn2 / Tun→vpn1 物理业务划分）
   ├── ADD GRETUNNEL（REDUNDANCYEN=TRUE 备份隧道 + KEEPALVEN=TRUE 保活）
   ├── ADD SRROUTE / ADD SRROUTE6（目的=REDUNDRDTIP，出接口=Tunnel）
   ├── MOD OSPFIMPORTROUTE（主 UDG COST=100 高优先级 / 备 UDG COST=200 低优先级）
   └── SET REDUNDUSER（全局开关，与 REDUNDFUNC 双使能）
   ↓
   备用 UDG 独有：SET APNREDUNDUPSW（上行报文经 GRE 转发至主用 UDG）
```

**主用 UDG 14 步骨架**（备用 UDG 重复步骤 1-14 + 额外步骤 15）：

| 步骤 | 干啥 | 主要命令 |
|---|---|---|
| 1. 静态地址池 | EXTERNAL 类型地址池（含 REDUNDFUNC=ENABLE） | `ADD POOL` |
| 2. 地址段 | IPv4/IPv6 段（主备相同） | `ADD SECTION` |
| 3. 地址池组 | POOLGROUP | `ADD POOLGROUP`（IPV4ALLOCPRIALG=ENABLE） |
| 4. 绑定池到组 | POOLBINDGROUP | `ADD POOLBINDGROUP` |
| 5. APN 实例 | HASVPN=ENABLE | `ADD APN` |
| 6. APN-池组映射 | POOLGRPMAP | `ADD POOLGRPMAP` |
| 7. 冗余备份重定向 IP | 虚拟 IP（IPv4 + IPv6） | `ADD REDUNDRDTIP` |
| 8. LoopBack 接口 | GRE 隧道源接口（绑 vpn2） | `ADD INTERFACE` + `ADD IPBINDVPN` + `ADD IFIPV4ADDRESS` |
| 9. Tunnel 接口 | GRE 隧道逻辑接口（MTU=1500，绑 vpn1） | `ADD INTERFACE` |
| 10. GRE Tunnel | 备份隧道（REDUNDANCYEN=TRUE + KEEPALVEN=TRUE） | `ADD GRETUNNEL` + `ADD IPBINDVPN` + `ADD IFIPV4ADDRESS` + `SET IFIPV6ENABLE` + `ADD IFIPV6ADDRESS`（v02） |
| 11. 静态路由 | 目的=REDUNDRDTIP，出接口=Tunnel | `ADD SRROUTE` / `ADD SRROUTE6` |
| 12. 可选 下行低优先级静态段路由入 GRE | PDN/DN 侧配路由场景 | `ADD SRROUTE` / `ADD SRROUTE6`（可选） |
| 13. 主备 LoopBack 路由互通 + N6 口路由优先级 | OSPF 接口 + MOD COST | `ADD OSPFINTERFACE` + `MOD OSPFIMPORTROUTE COST=100`（主用）/ `200`（备用） |
| 14. 全局冗余开关 | SET REDUNDUSER | `SET REDUNDUSER` |
| **15. （仅备用 UDG）** | 上行报文隧道转发 | `SET APNREDUNDUPSW` |

**关键约束**：
- **本特性无需 License**
- **C-U 协同**：必须与 UNC 侧 WSFD-107021 配合使用（C面决策 + U面执行）
- **双开关使能**：SET REDUNDUSER（全局）+ ADD POOL REDUNDFUNC（地址池）**必须同时使能**
- **互斥**：NAT 功能 / GWFD-010108 用户面地址自动检测
- **依赖 IPFD-015002 GRE**（无控制项，基础设施）
- 主备 UDG 地址段必须**相同**，且包含 SMF 静态段

## 配置流程

> 每步显示"干啥 + 含哪些命令"。主备 UDG 各配一遍；差异在步骤 10（GRE 对端 IP）+ 步骤 13（COST 主=100/备=200）+ 步骤 14 之后备用多步骤 15。

1. **EXTERNAL 地址池（REDUNDFUNC=ENABLE）**（继承 GWFD-010104）：
   - `ADD POOL` → [0-00043](task/UDG/20.15.2/0-00043.md)（POOLTYPE=**EXTERNAL**, HASVPN=ENABLE, VPNINSTANCE, **REDUNDFUNC=ENABLE**, MASTERFLAG=ENABLE 主/ DISABLE 备）
   - 复用 → [1-00022](task/UDG/20.15.2/1-00022.md) 步骤 1-4 地址池三级骨架（POOLTYPE=EXTERNAL 替换 LOCAL）

2. **地址段（IPv4/IPv6）**：
   - `ADD SECTION` → [0-00044](task/UDG/20.15.2/0-00044.md)（V4STARTIP/V4ENDIP 或 V6PREFIX*）

3. **地址池组（含优先级算法）**：
   - `ADD POOLGROUP` → [0-00042](task/UDG/20.15.2/0-00042.md)（IPV4ALLOCPRIALG=ENABLE）

4. **绑定地址池到组**：
   - `ADD POOLBINDGROUP` → [0-00045](task/UDG/20.15.2/0-00045.md)

5. **APN 实例**：
   - `ADD APN` → [0-00020](task/UDG/20.15.2/0-00020.md)（HASVPN=ENABLE, VPNINSTANCE=vpn1）

6. **APN-池组映射**：
   - `ADD POOLGRPMAP` → [0-00047](task/UDG/20.15.2/0-00047.md)（MAPPINGNAME, APN=, POOLGROUPNAME=）

7. **冗余备份重定向 IP**（★本特性核心）：
   - `ADD REDUNDRDTIP` → [0-00055](task/UDG/20.15.2/0-00055.md)（IPVERSION=IPv4/IPv6, IPV4ADDRESS=192.168.0.1 / IPV6ADDRESS=fc00:3::1；主备可相同或不同）

8. **LoopBack 接口（GRE 源接口，绑 vpn2）**：
   - `ADD INTERFACE` → [0-00056](task/UDG/20.15.2/0-00056.md)（LoopBack1/2, IFADMINSTATUS=up）
   - `ADD IPBINDVPN` → [0-00057](task/UDG/20.15.2/0-00057.md)（IFNAME=LoopBack1, **VRFNAME=vpn2**）
   - `ADD IFIPV4ADDRESS` → [0-00058](task/UDG/20.15.2/0-00058.md)（IFIPADDR=10.11.1.1 主 / 10.10.0.1 备, SUBNETMASK=255.255.255.255）

9. **Tunnel 接口（GRE 隧道逻辑接口）**：
   - `ADD INTERFACE` → [0-00056](task/UDG/20.15.2/0-00056.md)（Tunnel1/2, IFMTU=1500）

10. **GRE Tunnel 配置（★本特性核心：备份隧道 + 保活）**：
    - `ADD GRETUNNEL` → [0-00059](task/UDG/20.15.2/0-00059.md)（TNLNAME=Tunnel1/2, SRCTYPE=if_name, **SRCIFNAME=LoopBack1/2**, DSTIPADDR=对端 LoopBack IP（主→10.10.0.1, 备→10.11.1.1）, TNLTYPE=**gre**, DSTVPNNAME=vpn2, **REDUNDANCYEN=TRUE**, **KEEPALVEN=TRUE**）
    - `ADD IPBINDVPN` → [0-00057](task/UDG/20.15.2/0-00057.md)（IFNAME=Tunnel1, **VRFNAME=vpn1** — 与 LoopBack vpn2 不同 VPN 物理业务划分）
    - `ADD IFIPV4ADDRESS` → [0-00058](task/UDG/20.15.2/0-00058.md)（Tunnel 地址）
    - v02 IPv6 GRE 增强：`SET IFIPV6ENABLE` → [0-00060](task/UDG/20.15.2/0-00060.md)（ENABLEFLAG=TRUE, MTU=1500）+ `ADD IFIPV6ADDRESS` → [0-00061](task/UDG/20.15.2/0-00061.md)（IPV6ADDRESS=fc00:2::1/::2, PREFIXLEN=64）

11. **IPv4/IPv6 静态路由**（目的=REDUNDRDTIP，出接口=Tunnel）：
    - `ADD SRROUTE` → [0-00062](task/UDG/20.15.2/0-00062.md)（AFTYPE=ipv4unicast, PREFIX=192.168.0.1, MASKLENGTH=32, VRFNAME=vpn1, IFNAME=Tunnel）
    - `ADD SRROUTE6` → [0-00063](task/UDG/20.15.2/0-00063.md)（AFTYPE=ipv6unicast, PREFIX=fc00:3::1, MASKLENGTH=128, VRFNAME=vpn1, IFNAME=Tunnel）

12. **（可选）下行低优先级静态段路由入 GRE**：
    - `ADD SRROUTE` / `ADD SRROUTE6`（PDN/DN 侧配路由场景）

13. **主备 LoopBack 路由互通 + N6 口路由优先级**：
    - `ADD OSPFINTERFACE` → [0-00313](task/UDG/20.15.2/0-00313.md)（PROCID, AREAID, IFNAME=LoopBack1/2）— 打通主备 LB 路由
    - `MOD OSPFIMPORTROUTE` → [0-00077](task/UDG/20.15.2/0-00077.md)（PROCID, TOPOID=0, PROTOCOL=wlr, IMPTCOSTCFG=TRUE, **COST=100 主 / 200 备**）

14. **全局冗余开关**（★本特性核心，与 REDUNDFUNC 双使能）：
    - `SET REDUNDUSER` → [0-00064](task/UDG/20.15.2/0-00064.md)（SWITCH=ENABLE）

15. **（仅备用 UDG）上行报文隧道转发**：
    - `SET APNREDUNDUPSW` → [0-00320](task/UDG/20.15.2/0-00320.md)（APN=test, IPV4SWITCH=ENABLE, IPV6SWITCH=ENABLE）

## 决策点

**DP（特性级）：角色 × IP 版本 × UE 互访 × 走法**

| 角色 | IP版本 | UE 互访 | 走法（compound/atom 编排） |
|---|---|---|---|
| **主用 UDG** | IPv4 | 不使能（默认） | 步骤 1-14 完整 15 步；ADD POOL MASTERFLAG=DISABLE；STEP 13 COST=100；步骤 14 SET REDUNDUSER ENABLE；**不配步骤 15** |
| **主用 UDG** | IPv4 | **使能**（防重入隧道丢包） | 同上 + ADD POOL **MASTERFLAG=ENABLE** |
| **主用 UDG** | IPv6 | 同 IPv4 | 步骤 10 额外 SET IFIPV6ENABLE + ADD IFIPV6ADDRESS + ADD SRROUTE6 |
| **主用 UDG** | IPv4 + IPv6（v02 双栈） | 同上 | IPv4 + IPv6 命令组合 |
| **备用 UDG** | IPv4 | 不使能 | 步骤 1-14 完整 + **步骤 15 SET APNREDUNDUPSW**；ADD POOL MASTERFLAG=DISABLE；STEP 13 COST=200 |
| **备用 UDG** | IPv4 | **使能** | 同上 + ADD POOL MASTERFLAG=DISABLE（备 UDG 不配 MASTERFLAG，文档未明确，需 Stage 3 横向核实） |
| **备用 UDG** | IPv6 | 同 IPv4 | IPv6 命令组合 + 步骤 15 IPV6SWITCH=ENABLE |

> **关键规则**：
> - **主备对称**：除 GRE 对端 IP/COST/MASTERFLAG/上行转发外，命令集完全对称
> - **MASTERFLAG 仅主用 UDG 配**（UE 互访叠加场景防重入丢包）
> - **步骤 15 仅备用 UDG 配**（上行报文经 GRE 转发回主用，跨 DC 防丢包）
> - **双开关**：SET REDUNDUSER + REDUNDFUNC **必须**同时 ENABLE
> - **COST 主=100，备=200**（OSPF 方式；BGP 走 MED；或 PDN/DN 侧配）
> - **v02 增强**：v02/20.9.0 起支持 IPv6 GRE 隧道

## 约束

- **无需 License**（info）：本特性无需获得 License 许可即可获得该特性的服务
- **C-U 协同硬约束**（critical，rule-0-00534）：本特性与 UNC 配合使用生效，参考 UNC 中 WSFD-107021 — 后果：仅 UDG 单独配置无效
- **依赖 IPFD-015002 GRE**（critical，rule-0-00535）：静态地址路由冗余需要配置 GRE 隧道，依赖 GRE 特性 — 后果：GRE 特性未使能致冗余不生效
- **双开关使能**（critical，rule-0-00536）：SET REDUNDUSER（全局）与 ADD POOL REDUNDFUNC（地址池）必须同时 ENABLE — 后果：缺一不生效
- **静态地址池必配段**（critical，rule-0-00537）：配置 POOL 后必须配 SECTION，否则用户激活失败；一个静态地址池最多 64 个地址段 — 后果：用户激活失败或超规格
- **主备地址段一致**（critical，rule-0-00538）：主备 UDG 必须规划**相同地址段**，且需包含 SMF 配置的静态地址段 — 后果：用户激活失败或路径异常
- **VPN 三者一致**（critical，rule-0-00539）：地址池/APN/Tunnel 接口绑定的 VPN 必须一致 — 后果：VPN 不一致致报文无法转发
- **LoopBack/Tunnel VPN 划分**（critical，rule-0-00540）：LoopBack 接口绑 vpn2、Tunnel 接口绑 vpn1，**不同 VPN 完成物理与业务划分** — 后果：混淆致 GRE 隧道不可用
- **重定向 IP 不冲突**（warning，rule-0-00541）：REDUNDRDTIP 仅内部重定向不对外发布，保证不与其他路由冲突 — 后果：路由冲突致异常
- **主备 LoopBack 路由互通**（critical，rule-0-00542）：必须通过 OSPF/BGP/静态路由打通主备 LoopBack 接口路由 — 后果：GRE 隧道 DOWN
- **N6 口路由优先级**（critical，rule-0-00543）：主 UPF 优先级高于备 UPF（OSPF COST 主=100，备=200；或 BGP MED；或 PDN/DN 侧配） — 后果：路由不能正确选主用 UDG
- **开关前后无用户**（critical，rule-0-00544）：特性开启和关闭前都要确保 APN 下无使用静态路由冗余功能的用户 — 后果：影响现有用户业务
- **上行转发仅备 UDG**（critical，rule-0-00545）：SET APNREDUNDUPSW 仅在备用 UDG 配置 — 后果：主备反配置致流量路径错误
- **互斥 NAT**（critical，rule-0-00546）：使能 NAT 功能后，UDG 无需再使用静态路由冗余功能 — 后果：双向互斥
- **互斥 GWFD-010108**（critical，rule-0-00547）：静态地址用户不能做用户面地址自动探测 — 后果：双向互斥
- **GRE Keepalive 检测**（warning）：KEEPALVEN=TRUE 源端周期发送，对端不可达时本端 Tunnel 置 Down — 后果：故障检测延迟
- **ROUTER ID 全网唯一**（critical）：OSPF Router ID 必须全网唯一 — 后果：OSPF 邻接建立失败
- **系统影响**（warning）：开启后故障/恢复过程中下行流经 GRE Tunnel，系统处理负荷增加，性能与吞吐量下降（具体需话务模型评估） — 后果：转发性能下降
- **应用限制1**（warning，rule-0-00548）：上行数据流经备份 GRE Tunnel 时，**业务重定向功能失效** — 后果：特性互斥
- **应用限制2**（warning，rule-0-00549）：用户上下行报文必须经过同一组防火墙/外置 SA 设备 — 后果：UE/防火墙/外置 SA 设备处理异常
- **首次发布版本**（info）：v01 UDG 20.5.0；v02 20.9.0（IPv6 GRE 增强）
- **v02 IPv6 GRE**（warning）：仅 v02 UDG 20.9.0 及后续版本支持 — 后果：低版本不支持 IPv6 GRE
