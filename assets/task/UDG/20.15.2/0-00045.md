# 绑定地址池到地址池组（ADD POOLBINDGROUP）

> 命令静态知识见 [ADD POOLBINDGROUP](command/UDG/20.15.2/ADD-POOLBINDGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

ADD POOLBINDGROUP 是地址池族链路的"池组绑定层"：把已建的 [ADD POOL](task/UDG/20.15.2/0-00043.md) POOLNAME 绑到已建的 [ADD POOLGROUP](task/UDG/20.15.2/0-00042.md) POOLGROUPNAME 下，多池并存时可配优先级决定地址分配顺序。命令核心配置空间有 3 个维度：**地址池组引用**（POOLGROUPNAME 必选 + 必须已建）、**地址池引用**（POOLNAME 必选 + 必须已建）、**优先级**（PRIORITY 可选，生效范围同 VPN 内）。7 个特性命中覆盖 010104/010105/010107/020403/020406/020421/020482，绝大多数场景在多池并存时配 PRIORITY=10。

### 配置维度 1：地址池组引用（参数 POOLGROUPNAME，必选）

| 参数 | 范围 | 备注 |
|---|---|---|
| **POOLGROUPNAME** | 字符串 1~79 | 引用已配的 [ADD POOLGROUP](task/UDG/20.15.2/0-00042.md) POOLGROUPNAME；前置依赖：POOLGROUP 必须先建 |

### 配置维度 2：地址池引用（参数 POOLNAME，必选）

| 参数 | 范围 | 备注 |
|---|---|---|
| **POOLNAME** | 字符串 1~79 | 引用已配的 [ADD POOL](task/UDG/20.15.2/0-00043.md) POOLNAME；前置依赖：POOL 必须先建 |

### 配置维度 3：优先级（参数 PRIORITY，可选）

| 参数 | 范围 | 作用 |
|---|---|---|
| **PRIORITY** | 整数 | 地址池优先级；**生效范围为地址池组内相同 vpn 的地址池**；**数值越小优先级越高**；同优先级按地址池加入顺序/容量分配 |

> PRIORITY 仅在多池并存场景需要指定分配顺序时配；同组内不同 VPN 池的优先级互不影响。

**典型脚本（010104 单池绑组）**：
```
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup1", POOLNAME="testpool";
```

**典型脚本（010105 多池分组 + PRIORITY）**：
```
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool1";
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup2", POOLNAME="testpool2", PRIORITY=10;
```

**典型脚本（010107 主备 4 池绑 2 组）**：
```
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup1", POOLNAME="pool1";
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup1", POOLNAME="pool2";
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup2", POOLNAME="pool3";
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup2", POOLNAME="pool4";
```

**典型脚本（020403 双栈 + PRIORITY）**：
```
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool", PRIORITY=10;
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpoolv6", PRIORITY=10;
```

**典型脚本（020482 入不转板 + PRIORITY）**：
```
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool", PRIORITY=10;
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；POOLGRPNAME+POOLNAME 联合唯一；PRIORITY 仅组内同 VPN 池生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 3：池组绑定"——前置依赖 [0-00043](task/UDG/20.15.2/0-00043.md)（POOLNAME）+ [0-00042](task/UDG/20.15.2/0-00042.md)（POOLGROUPNAME）；下游 [0-00047](task/UDG/20.15.2/0-00047.md)（POOLGRPMAP 建 APN 映射）。

## 决策点：是否配优先级（DP 0-00045）

问：ADD POOLBINDGROUP 是否需要配 PRIORITY？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 无优先级 | 不配 PRIORITY | 单池绑组（010104 范式：poolgroup1+testpool）；同组多池按加入顺序/容量分配 |
| 等优先级 | PRIORITY=10（同值） | 双栈等优先级场景（020403 范式：testpool+testpoolv6 同 PRIORITY=10，由组内 IPv4/IPv6 各按段决定） |
| 显式优先级 | PRIORITY=N（不同值） | 010105 多分组场景（poolgroup2+testpool2 显式 PRIORITY=10）；020482 入不转板 |
| 主备 4 池 | 不配 PRIORITY | 010107 主备各 2 池分属 poolgroup1/poolgroup2，不需组内排序 |

> 7 个特性命中以"无优先级"（010104/010107/020406/020421）和"等优先级 10"（020403/020482）为主；010105 出现 1 次显式 PRIORITY=10。

## 约束

- **POOLNAME 前置依赖**（critical，rule-0-00445）：POOLNAME 必须先由 [0-00043](task/UDG/20.15.2/0-00043.md) 建好 — 后果：未建引用命令拒收
- **POOLGROUPNAME 前置依赖**（critical，rule-0-00445）：POOLGROUPNAME 必须先由 [0-00042](task/UDG/20.15.2/0-00042.md) 建好 — 后果：未建引用命令拒收
- **POOLGRPNAME+POOLNAME 联合唯一**（critical，rule-0-00445）：同一 POOLNAME 在同一 POOLGROUPNAME 下只能绑一次 — 后果：重复绑定命令拒收
- **PRIORITY 仅组内同 VPN 池生效**（critical，rule-0-00445）：不同 VPN 池的优先级互不影响；数值越小优先级越高 — 后果：跨 VPN 优先级无效
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
- **同优先级按加入顺序/容量分配**（warning，隐含）：同组同 VPN 内 PRIORITY 相同时不报错但分配非确定 — 后果：实际分配结果可能漂移
