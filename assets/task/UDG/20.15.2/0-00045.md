# 绑定地址池到地址池组（ADD POOLBINDGROUP）

> 命令静态知识见 [ADD POOLBINDGROUP](command/UDG/20.15.2/ADD-POOLBINDGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD POOLBINDGROUP 把地址池挂到地址池组下，是地址池族链路中段。**双引用上游**：POOLGROUPNAME ← ADD POOLGROUP 的 POOLGRPNAME（0-00042，`LST POOLGROUP`/`LST POOLBINDGROUP` 可查），POOLNAME ← ADD POOL 的 POOLNAME（0-00043，`LST POOL` 可查）。执行后**只对新激活用户生效**。

**基础范式**：单池绑单组只需两名称；多池同组时按 PRIORITY 控制分配顺序。

**PRIORITY 取值约定**

| 场景 | PRIORITY | 说明 |
|---|---|---|
| 单池绑组 | 省略（默认 16） | 普通地址分配多数场景 |
| 多池按优先级分配 | 显式配 10（1~16） | 双栈/多池场景，只有**同类地址池**（同为 IPv4 或同为 IPv6）间才比较优先级，跨类型（IPv4 vs IPv6）不比 |

**典型行**

普通单池绑组（010104 地址分配方式）：
```
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup1", POOLNAME="testpool";
```
双栈 IPv4/IPv6 池同绑一组（020403，各设 PRIORITY=10）：
```
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool", PRIORITY=10;
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpoolv6", PRIORITY=10;
```
静态路由冗余（010107，pool1~4 分绑 poolgroup1/2）：
```
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup1", POOLNAME="pool1";
ADD POOLBINDGROUP:POOLGROUPNAME="poolgroup2", POOLNAME="pool3";
```

**前置与副作用**：必须先执行 ADD POOLGROUP 建组（否则引用悬空）；地址池一旦绑到组，**不允许再改该池的 VPN 参数**（ADD POOL / MOD POOL）。

## 约束

- **rule-0-00143**（warning）最大记录数 80000；每组下对每个 VPN 实例（含不配 VPN）最多绑 16 个同 VPN 的 IPv4 池 + 16 个同 VPN 的 IPv6 池（一池可被多组绑定）；绑定后不允许改 POOL 的 VPN 参数；前置必须先 ADD POOLGROUP；只对新激活用户生效 — 后果：超规格/超每组每 VPN 16/16 上限失败 / 改已绑池 VPN 致绑定异常 / 未先建 POOLGROUP 致引用悬空
