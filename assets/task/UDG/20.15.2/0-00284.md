# 配置网络切片选择标识（ADD SNSSAI）

> 命令静态知识见 [ADD SNSSAI](command/UDG/20.15.2/ADD-SNSSAI.md)

## 配置方法

按全网规划为每个切片建一条 SNSSAI 记录，SST 必选、SD 可选（无 SD 时默认填全 F）。SST+SD 二元组唯一标识一个切片，是 ADD SNSSAIUPINTF（切片绑逻辑接口）和 ADD SNSSAIBWMUSRG（切片绑带宽用户组）的上游源。

| 参数 | 取值样例 | 说明 |
|---|---|---|
| SST | `1` | 必选，0~255 整数，全网规划 |
| SD | `123456` / `ffffff` | 可选，长度 6 的十六进制字符串；无 SD 时填全 F（不传默认全 F） |
| CFGDOMAINNAME | （不配） | 可选配置域，切片带宽控制场景未使用 |

**典型脚本**：

```
// 定义一个 SST=1、SD=123456 的切片
ADD SNSSAI: SST=1,SD="123456";
```

**步骤位置**：切片带宽控制链路的第 1 步——必须先于 ADD SNSSAIUPINTF 和 ADD SNSSAIBWMUSRG 执行；两条下游命令的 SST/SD 必须与本命令建的记录一致。

## 决策点

**DP 0-00284**：SNSSAI 切片标识构成（SST 是否带 SD）——SD 传或不传决定切片是 SST+SD 双字段切片还是 SST 单字段切片。

| 选项 | SD 取值 | 联动 |
|---|---|---|
| 双字段切片（SST+SD） | 显式传 SD（如 `123456`） | 下游 SNSSAIUPINTF/SNSSAIBWMUSRG 必须传相同 SD |
| 单字段切片（仅 SST） | 不传，系统默认填全 F `ffffff` | 下游命令 SD 可不传，系统自动匹配全 F |

SD 为 6 位十六进制（0~9 / a~f / A~F，大小写不敏感）；非 6 位或含非法字符校验失败。

## 约束

- 最大记录数 8192 · warning · 超规格新增失败
- 命令执行后只对新激活用户生效 · warning · 已在线用户不回溯
- 不配置该命令不影响用户激活 · info · 仅当需基于 SNSSAI 做性能统计/报文转发时才需配
- SD 必须是长度为 6 的字符串，S-NSSAI 无 SD 时填全 F · error · 长度/字符校验
