# 配置网络切片选择标识（ADD SNSSAI）

> 命令静态知识见 [ADD SNSSAI](command/UDG/20.15.2/ADD-SNSSAI.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

按全网规划为每个切片建一条 SNSSAI 记录，SST 必选、SD 可选（无 SD 时默认填全 F）。SST+SD 二元组唯一标识一个切片，是 ADD SNSSAIUPINTF（0-00285 切片绑逻辑接口）和 ADD SNSSAIBWMUSRG（0-00286 切片绑带宽用户组）的上游源。适用 NF：UPF。

### 配置维度 1：切片标识构成（参数 `SST` + `SD`）

| 参数 | 取值范围 | 作用 | 配套必选/条件 |
|---|---|---|---|
| SST | 整数 0～255 | 切片/服务类型，全网规划 | 必选 |
| SD | 6 位十六进制（0~9/a~f/A~F） | 切片区分码 | 可选；不传时系统默认填全 F（`ffffff`） |
| CFGDOMAINNAME | 字符串 1～31 | 配置域 | 可选（冷门不绑，切片带宽场景不使用） |

### 配置维度 2：SD 字符约束

SD 必须为 6 位十六进制字符串，每个字符必须是 0~9 的数字或 a~f/A~F（大小写不敏感）；非 6 位或含非法字符校验失败。

**典型脚本**

```
// 定义一个 SST=1、SD=123456 的切片
ADD SNSSAI: SST=1,SD="123456";

// 无 SD 的单字段切片（系统自动填 ffffff）
ADD SNSSAI: SST=1;
```

**步骤位置**：切片带宽控制链路的第 1 步——必须先于 ADD SNSSAIUPINTF 和 ADD SNSSAIBWMUSRG 执行；两条下游命令的 SST/SD 必须与本命令建的记录一致。

## 决策点：SNSSAI 切片标识构成（DP 0-00284）

**[[UDG@20.15.2@DecisionPoint@0-00284]]**（single_choice，SD 是否带值）

| 选项 | SD 取值 | 联动 |
|---|---|---|
| 双字段切片（SST+SD） | 显式传 SD（如 `123456`） | 下游 SNSSAIUPINTF/SNSSAIBWMUSRG 必须传相同 SD |
| 单字段切片（仅 SST） | 不传，系统默认填全 F `ffffff` | 下游命令 SD 可不传，系统自动匹配全 F |

## 约束

- **命令记录上限**（warning，rule-0-00284-01）：最大记录数 8192，超规格新增失败。
- **新激活生效**（warning，rule-0-00284-02）：命令执行后只对新激活用户生效，已在线用户不回溯。
- **非强制配置**（info，rule-0-00284-03）：不配置本命令不影响用户激活（仅当需基于 SNSSAI 做性能统计/报文转发时才需配）。
- **SD 校验**（critical，rule-0-00284-04）：SD 必须是长度为 6 的字符串，每个字符必须在 0~9 / a~f / A~F 内——长度或字符非法则命令被拒。
