# GRE 隧道建立（LoopBack 源 + GRETUNNEL + Tunnel 接口 + 静态路由）

> GRE 隧道建立的**5 步可复用骨架**（接入方式隧道族共享）。被引用于 [2-00042](task/UDG/20.15.2/2-00042.md) IPFD-015002 GRE。
> 2026-07-12 建（防平铺，SOP §3.5——5 步阶段内聚 + 隧道族复用：GRE/GRE over IPSec/多租户共享）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. LoopBack 接口（推荐隧道源）| `ADD INTERFACE` + `ADD IFIPV4ADDRESS` | IFNAME=LoopBack1, IFTYPE=loopback；IFIPADDR=源 IP, SUBNETMASK=255.255.255.255 |
| 2. GRE 隧道对象（★核心）| `ADD GRETUNNEL` | TNLNAME=Tunnel1, TNLTYPE=**gre**, SRCTYPE=if_name, SRCIFNAME=LoopBack1, DSTIPADDR=对端 IP（可选 DSTVPNNAME 绑 VPN）|
| 3. Tunnel 接口 IP | `ADD IFIPV4ADDRESS` | IFNAME=Tunnel1, IFIPADDR=隧道接口 IP, SUBNETMASK |
| 4. 静态路由（引导流量进隧道）| `ADD SRROUTE` | AFTYPE=ipv4unicast, PREFIX=对端网段, MASKLENGTH, IFNAME=Tunnel1 |
| 5.（可选）隧道增强 | `MOD GRETUNNEL` | CHECKSUMEN=TRUE（端到端校验）/ GREKEYEN=TRUE+GREKEY（识别关键字，两端必须一致）/ KEEPALVEN=TRUE+KEEPALVPERIOD+KEEPALVRETRYCNT（Keepalive 检测）|

**典型脚本（IPv4 GRE 隧道，LoopBack 源）**：
```
ADD INTERFACE:IFNAME="LoopBack1",IFTYPE=loopback,IFADMINSTATUS=up;
ADD IFIPV4ADDRESS:IFNAME="LoopBack1",IFIPADDR="10.1.1.1",SUBNETMASK="255.255.255.255";
ADD GRETUNNEL:TNLNAME="Tunnel1",TNLTYPE=gre,SRCTYPE=if_name,SRCIFNAME=LoopBack1,DSTIPADDR="10.2.2.2";
ADD IFIPV4ADDRESS:IFNAME="Tunnel1",IFIPADDR="192.168.1.1",SUBNETMASK="255.255.255.252";
ADD SRROUTE:AFTYPE=ipv4unicast,PREFIX="10.3.0.0",MASKLENGTH=16,IFNAME=Tunnel1;
// 可选增强
MOD GRETUNNEL:TNLNAME="Tunnel1",TNLTYPE=gre,KEEPALVEN=TRUE,KEEPALVPERIOD=5,KEEPALVRETRYCNT=3;
```

**步骤位置**：在 [1-00029](task/UDG/20.15.2/1-00029.md) APN 接入域基础设施（若 VPN 组网）之后；GRE 隧道建立是接入方式核心。

## 场景差异

| 场景 | 源类型 | 地址族 | VPN 绑定 | 增强项 | 复用说明 |
|---|---|---|---|---|---|
| 基础 GRE 隧道（公网）| if_name（LoopBack）| IPv4 | 无（DSTVPNNAME 不配）| Keepalive 推荐 | 本骨架基线 |
| VPN 组网 GRE | if_name | IPv4 | DSTVPNNAME=vpn 名 | — | 叠加 [1-00029](task/UDG/20.15.2/1-00029.md) VPN 基础设施 |
| IPv6 GRE（v02+）| if_name | IPv6 | 可选 | SET IFIPV6ENABLE+ADD IFIPV6ADDRESS | 步骤 1/3 改 IPv6 地址；需 UDG 20.9.0+ |
| 多租户共享 GRE | if_name | IPv4 | — | **GREKEYEN=TRUE+GREKEY**（同源同宿多隧道以 Key 区分租户）| 步骤 5 必配 GREKey |
| GRE over IPSec | if_name | IPv4 | — | 先建本隧道，再叠加 IPSec 加密 | 复用本骨架 + [1-00033](task/UDG/20.15.2/1-00033.md) IPSec 隧道 |
| 两层嵌套 GRE | if_name | IPv4 | — | 外层+内层各建一隧道 | 本骨架执行两遍 |

> **互斥约束**：GRE 隧道源地址**不能与 IPSec 隧道源地址相同**（应用限制硬约束）。

## 决策点

| 选项（源类型） | 影响（SRCTYPE + 配套参数）|
|---|---|
| SRCTYPE=if_name（推荐）| SRCIFNAME=LoopBack1（源稳定，推荐）|
| SRCTYPE=ip_address | SRCIPADDR=源 IP（重新规划源时）|

> 联动特性级 DP：地址族（IPv4/IPv6/双栈）+ VPN 绑定由 [2-00042](task/UDG/20.15.2/2-00042.md) 场景 DP 决定。

## 约束

- **隧道源/目的路由可达**（critical）：隧道源端口和目的端口已路由可达（PING 验证）— 后果：隧道 DOWN。
- **源地址不与 IPSec 冲突**（critical）：GRE 隧道源地址不能与 IPSec 隧道源地址相同 — 后果：应用限制违反。
- **GREKey 两端一致**（warning）：GREKEYEN=TRUE 时两端 GREKey 必须相同，或两端都不配 — 后果：关键字不匹配致报文丢弃。
- **Tunnel MTU**（warning）：建议 MOD INTERFACE 配置 MTU ≤ 出接口 MTU - GRE 头长度（GRE 头=GRE header + 外层 IP header）— 后果：MTU 过大致分片。
- **最多两层嵌套**（warning）：GRE 隧道嵌套层数最多 2 层 — 后果：超规格失败。
- **Keepalive 单向检测**（info）：KEEPALVEN=TRUE 源端周期发送，对端不可达时本端 Tunnel 置 Down；两端都启用才能双向检测。
- **无需 License**（info）：本特性无需 License。
- **C-U 对称**（info）：UDG/UNC 两侧配置同构，ADD GRETUNNEL 命令对称。
