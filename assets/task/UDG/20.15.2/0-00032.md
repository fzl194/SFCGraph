# 配置分类属性（ADD CATEGORYPROP）

> 命令静态知识见 [ADD CATEGORYPROP](command/UDG/20.15.2/ADD-CATEGORYPROP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD CATEGORYPROP 是带宽控制骨架的"业务分类锚点定义层"：把"分类属性"用一条命名记录描述，产出对象 CATEPROPNAME 供 ADD BWMSERVICE(0-00034) 的 CATEPROPNAME 引用。命令核心配置空间仅 1 个维度：**分类属性命名唯一性**（CATEPROPNAME 全网规划、全局唯一）。分类属性本身只是命名锚点，真正的带宽控制策略由下游 BWMSERVICE 引用此名称展开。

### 配置维度 1：分类属性命名（参数 CATEPROPNAME，必选 + 全局唯一）

| 参数 | 取值范围 | 数据来源 | 唯一性 | 备注 |
|---|---|---|---|---|
| **CATEPROPNAME**（分类属性名称） | 字符串 1~31，不支持空格，不区分大小写 | 全网规划 | 全局唯一 | 引用键 |

**典型脚本（020354 基于业务的 Shaping）**：
```
ADD CATEGORYPROP:CATEPROPNAME="cateprop_test";
```

**典型脚本（110311 基于业务感知的带宽控制）**：
```
ADD CATEGORYPROP:CATEPROPNAME="cp_700_600";
```

> 命令 notes：最大记录数 500；适用 NF：PGW-U、UPF；执行后立即生效；CFGDOMAINNAME 冷门不配。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：分类属性定义"——前置无；产出 CATEPROPNAME 被 ADD BWMSERVICE(0-00034) 的 CATEPROPNAME 引用。

## 决策点

（无。本命令仅命名锚点一个维度，唯一性约束是配置生成自检的核心，详见约束段。）

## 约束

- **CATEPROPNAME 全网唯一**（critical，rule-0-00332）：不同分类属性之间 CATEPROPNAME 不能重复 — 后果：重名 ADD 失败
- **须先于 BWMSERVICE 配置**（critical，rule-0-00332）：CATEPROPNAME 必须先于 ADD BWMSERVICE(0-00034) 建成 — 后果：未建引用失败
- **最大记录数 500**（warning，rule-0-00332）：单板最大 500 条 — 后果：超规格 ADD 失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
