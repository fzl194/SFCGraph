# 配置分类属性（ADD CATEGORYPROP）
> 命令静态知识见 [ADD CATEGORYPROP](command/UDG/20.15.2/ADD-CATEGORYPROP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

命名一个分类属性对象（全网规划、全局唯一），供后续 ADD BWMSERVICE 的 CATEPROPNAME 引用，作为带宽控制的业务分类锚点（如基于 URL 分类）。

| 参数 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| CATEPROPNAME（分类属性名称） | cateprop_test / cp_700_600 | 全网规划 | 1~31 字符串，全局唯一 |

脚本样例：
```
ADD CATEGORYPROP:CATEPROPNAME="cateprop_test";
```

配置要点：分类属性本身只是命名锚点，真正的带宽控制策略由下游 BWMSERVICE 引用此名称展开。配置域 CFGDOMAINNAME 冷门不配。典型用于：基于 URL 的 Shaping 整形（GWFD-020354）、PCC 预定义规则带宽控制（GWFD-110311）、HTTP 计费防欺诈策略承载（GWFD-110403）。
