# 配置IPFarm（ADD IPFARM）

> 命令静态知识见 [ADD IPFARM](command/UDG/20.15.2/ADD-IPFARM.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD IPFARM 是 IP Farm 基础设施的"实例创建"命令：IP farm 是若干个相同 VPN 的 server 集合，可用于重定向（captive portal 动作）或 P-CSCF 连接状态检测。本命令的核心配置空间有 5 个维度：**服务器类型 SERVERTYPE**（REDIRECT/PCSCF/IPMS，决定 IP farm 用途）、**IP 版本 IPVERSION**（IPV4/IPV6，决定虚拟 IP 类型）、**心跳检测开关 HEALTHCHECKFLAG**（决定 INTERFACENAME 是否必选）、**心跳检测接口 INTERFACENAME**（HEALTHCHECKFLAG=ENABLE 时必选）、**Web Proxy 选择模式 WPSELECTMODE/WPCONFLICTACT**（仅 REDIRECT 场景）。

### 配置维度 1：服务器类型（参数 SERVERTYPE，可选/默认 REDIRECT）

| 取值 | 作用 | 配套必选参数 | 典型场景 |
|---|---|---|---|
| **REDIRECT**（默认） | 重定向，captive portal 动作的目的 server 集合 | VIRTUALIP/VIRTUALIPV6（条件可选） | [2-00014](task/UDG/20.15.2/2-00014.md) 用户Portal、[2-00015](task/UDG/20.15.2/2-00015.md) Web Proxy |
| **PCSCF** | 检测 PcscfGroup 中各 P-CSCF 的连接状态 | VPNINSTANCE | [2-00013](task/UDG/20.15.2/2-00013.md) P-CSCF故障时IMS业务恢复 |
| **IPMS** | IPMS Server 状态检测 | VPNINSTANCE | 业务保持 |

### 配置维度 2：IP 版本（参数 IPVERSION，必选）

| 取值 | 作用 | 配套必选参数 |
|---|---|---|
| **IPV4** | IPv4 地址类型 | VIRTUALIP（点分十进制） |
| **IPV6** | IPv6 地址类型 | VIRTUALIPV6（冒号分十六进制） |

> 双栈服务器需配 2 个 IP farm（一 IPV4 + 一 IPV6）。

### 配置维度 3：心跳检测开关（参数 HEALTHCHECKFLAG，可选/默认 ENABLE）

| 取值 | 作用 | 配套参数 |
|---|---|---|
| **ENABLE**（默认） | 开启心跳检测 | INTERFACENAME（条件必选） |
| **DISABLE** | 关闭心跳检测 | INTERFACENAME 可选；VPNINSTANCE 必须与用户 VPN 一致 |

### 配置维度 4：心跳检测接口（参数 INTERFACENAME，HEALTHCHECKFLAG=ENABLE 时必选）

逻辑接口类型必须是 phif；长度 1~31 字符串。

### 配置维度 5：Web Proxy 选择模式（参数 WPSELECTMODE/WPCONFLICTACT，仅 SERVERTYPE=REDIRECT 时可选）

| 参数 | 取值 | 含义 | 适用场景 |
|---|---|---|---|
| **WPSELECTMODE** | INHERIT（默认） | 继承全局软参 BIT596/BIT1541 设置 | 默认场景 |
| | SESSION_BASED | 基于用户会话粒度选 server | 多流关联业务 |
| | FLOW_BASED | 基于流粒度选 server | 业务流之间无关联 |
| **WPCONFLICTACT** | BLOCK（默认） | 无法选到可用 server 时阻塞报文 | proxy server 用于鉴权/分流 |
| | PASS | 无法选到可用 server 时放通 | proxy server 仅用于业务加速 |

> **典型范式**：
> ```
> ADD IPFARM:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERTYPE=PCSCF,VPNINSTANCE="ims_vpn",VIRTUALIP="10.1.1.1",HEALTHCHECKFLAG=ENABLE,INTERFACENAME="phif1/0/0";
> ADD IPFARM:IPFARMNAME="farm_test",IPVERSION=IPV4,HEALTHCHECKFLAG=ENABLE,INTERFACENAME="phif1/0/0";
> ```

**步骤位置**：在所属 feature 编排中通常为"步骤 2：配置 IP Farm"——SET IPFARMGLOBAL 之后、ADD IPFARMSERVER 之前。

## 决策点：服务器类型（DP 0-00087）

问：ADD IPFARM 的 SERVERTYPE 取何值？

| 选项 | SERVERTYPE | 配套必选参数 | 典型场景 |
|---|---|---|---|
| 重定向（REDIRECT，默认） | REDIRECT | VIRTUALIP/VIRTUALIPV6 | [2-00014](task/UDG/20.15.2/2-00014.md)、[2-00015](task/UDG/20.15.2/2-00015.md) |
| P-CSCF 检测（PCSCF） | PCSCF | VPNINSTANCE | [2-00013](task/UDG/20.15.2/2-00013.md) |
| IPMS 检测（IPMS） | IPMS | VPNINSTANCE | 业务保持 |

## 约束

- **整机最大记录数为 3600**（warning，rule-0-00110 同族）：超 90% 上报 ALM-135602215 配置数量超阈值告警；阈值可 MOD CFGTHRESHOLD 修改 — 后果：超规格新增失败
- **VIRTUALIP/VIRTUALIPV6 与 IPFARMNAME 唯一对应**（critical，隐含）：同一 VIRTUALIP 不允许配置到不同 IP farm — 后果：冲突拒收
- **HEALTHCHECKFLAG=ENABLE 时 INTERFACENAME 必选**（critical，隐含）：心跳检测接口必须为 phif 逻辑接口 — 后果：缺失或类型错拒收
- **HEALTHCHECKFLAG=DISABLE 时 VPNINSTANCE 必须与用户 VPN 一致**（warning，隐含）：关闭心跳检测场景下 IP farm 的 VPN 须与用户 VPN 保持一致 — 后果：VPN 不一致导致用户报文无法正确路由
- **双栈需配 2 个 IPFARM**（info，隐含）：同一服务器双栈接入需配 IPv4/IPv6 各一个 IP farm — 后果：单 IP farm 仅支持一种 IP 类型
