# 智能重定向动作链

> **智能重定向族通用 compound**（HTTP智能重定向/DNS纠错 共用骨架）：承载"扩展过滤器+错误码→重写/重定向动作"的动作策略链，POLICYTYPE=SMARTREDIRECT 触发。被引用于 [2-00017](task/UDG/20.15.2/2-00017.md)（HTTP智能重定向）/[2-00016](task/UDG/20.15.2/2-00016.md)（DNS纠错）。

## 配置方法

按协议层选动作命令（HTTP 智能重定向用 ADD SMARTHTTPREDIR，DNS 纠错用 ADD DNSOVERWRITING），统一前置：扩展过滤器（可多个，AND 组合）→ 错误码范围 → 重定向携带信息（**仅 HTTP 智能重定向用**）→ 动作命令（绑过滤器+错误码+目标）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 扩展过滤器（可重复执行配多个）| ADD EXTENDEDFILTER | EXTFLTNAME + URL/USERAGENT/CONTENTTYPE/URLPOSTFIX（至少其一）|
| 错误码范围 | ADD ERRORCODE | ERRORCODENAME + ERRORCODEOP（EQUAL/GT/LT/GE/LE）+ ERRORCODESTART（+ ERRORCODEEND）|
| 重定向携带信息（**仅 HTTP 智能重定向**，可选）| ADD REDIRAPPENDINFO | APPENDINFONAME + REQURLFLAG + IMSIFLAG（+ MSISDNFLAG/IMEIFLAG/MSIPFLAG + 加密）|
| HTTP 智能重定向动作（→SERVERURL）| ADD SMARTHTTPREDIR | SMTHTTPREDINAME + SERVERURL + EXTFLTTYPE1/EXTFLTNAME1（+ EXTFLTTYPE2~5）+ BINDERRCODENAME + APPENDINFONAME（HTTP 专用）|
| DNS 重写动作（→SERVERIP1，场景差异）| ADD DNSOVERWRITING | DNSOVERWRTNAME + EXTFLTTYPE1/EXTFLTNAME1（+ EXTFLTTYPE2~5）+ SERVERIP1（+ SERVERIP2~5/SERVERIPV6）+ BINDERRCODENAME |

**典型脚本（HTTP 智能重定向：URL+USERAGENT 双扩展过滤器 + 错误码 GT 400 + 携带 REQURL+IMSI，activation 原样）**：
```
ADD EXTENDEDFILTER:EXTFLTNAME="filter_http-redirect1",URL="www.huawei.com/*";
ADD EXTENDEDFILTER:EXTFLTNAME="filter_http-redirect2",USERAGENT="Mozilla/*";
ADD ERRORCODE: ERRORCODENAME="httperrorcode", ERRORCODEOP=GT, ERRORCODESTART=400;
ADD REDIRAPPENDINFO:APPENDINFONAME="testredirappendinfo",REQURLFLAG=ENABLE,IMSIFLAG=ENABLE;
ADD SMARTHTTPREDIR:SMTHTTPREDINAME="http-redirect_test", SERVERURL="www.example.com", EXTFLTTYPE1=AND, EXTFLTNAME1="filter_http-redirect1", EXTFLTTYPE2=AND, EXTFLTNAME2="filter_http-redirect2", APPENDINFONAME="testredirappendinfo", BINDERRCODENAME="httperrorcode";
```

**典型脚本（DNS 纠错变体：单 URL 扩展过滤器 + 错误码 EQUAL 3 + 改写 SERVERIP1，activation 原样）**：
```
ADD EXTENDEDFILTER:EXTFLTNAME="l7-filter1",URL="www.example.com/*";
ADD ERRORCODE: ERRORCODENAME="dnserrorcode", ERRORCODEOP=EQUAL, ERRORCODESTART=3;
ADD DNSOVERWRITING:DNSOVERWRTNAME="dec_1",EXTFLTTYPE1=AND,EXTFLTNAME1="l7-filter1",SERVERIP1="192.168.1.1",BINDERRCODENAME="dnserrorcode";
```

**步骤位置**：承接 [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链，产出 FLOWFILTERNAME）；产出 SMTHTTPREDINAME / DNSOVERWRTNAME 供下游 [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定中 RULE 经 `POLICYTYPE=SMARTREDIRECT, POLICYNAME=<动作对象名>` 直接引用（**关键差异：不经 PCCPOLICYGRP，见场景差异 §1**）。

## 场景差异

> 智能重定向族（HTTP智能重定向/DNS纠错）共用本 compound，同属 SMARTREDIRECT 体系（L7 重写），共享"扩展过滤器+错误码"前置，动作命令机制不同（HTTP URL 重定向 vs DNS IP 改写）。

### 1. 动作命令变种（HTTP 重定向 vs DNS 纠错）

| 维度 | HTTP 智能重定向（[2-00017](task/UDG/20.15.2/2-00017.md)）| DNS 纠错（[2-00016](task/UDG/20.15.2/2-00016.md)）|
|---|---|---|
| **触发条件** | HTTP 服务器返回错误码（如 `>400`）| DNS 响应错误码（如 `=3`，NXDOMAIN）|
| **EXTENDEDFILTER** | URL + USERAGENT（2 个扩展过滤器，AND 组合）| URL（1 个扩展过滤器）|
| **ERRORCODE** | `ERRORCODEOP=GT, ERRORCODESTART=400` | `ERRORCODEOP=EQUAL, ERRORCODESTART=3` |
| **携带信息** | ADD REDIRAPPENDINFO（REQURLFLAG + IMSIFLAG）| 无（DNS 不携带）|
| **动作命令** | ADD SMARTHTTPREDIR（→SERVERURL，重定向到第三方 HTTP server）| ADD DNSOVERWRITING（→SERVERIP1，改写 DNS 响应 IP 为第三方 Platform IP）|
| **改写层** | HTTP 层（URL 重定向）| DNS 层（响应 IP 改写）|
| **动作对象被引用** | `RULE.POLICYNAME=SMTHTTPREDINAME` | `RULE.POLICYNAME=DNSOVERWRTNAME` |
| **前置依赖** | 依赖 SA-Basic + SA-Web Browsing | 依赖 PCC 基本功能；LOD SIGNATUREDB 可选（SPECIFIC 版本）|

> 共享点：两特性 RULE 的 `POLICYTYPE=SMARTREDIRECT` 一致，POLICYNAME 均指向动作对象（不经 PCCPOLICYGRP）；共享 ADD EXTENDEDFILTER（[0-00022](task/UDG/20.15.2/0-00022.md)）+ ADD ERRORCODE（[0-00089](task/UDG/20.15.2/0-00089.md)）两前置命令。

> **不含命令说明**：本 compound 不含 SET SRVCOMMONPARA（HTTP 响应码相关，属 Portal 族 [1-00014](task/UDG/20.15.2/1-00014.md) 体系，284 activation 无此命令证据）；不含 ADD FLOWFILTERGRP（可选 FILTERINGMODE 变体，直引 atom [0-00294](task/UDG/20.15.2/0-00294.md)）。

### 2. 错误码操作码变体（ERRORCODEOP）

| 取值 | 语义 | 典型用法 | 生效参数 |
|---|---|---|---|
| **GT**（HTTP 智能重定向默认）| 大于 | `ERRORCODESTART=400`（>400 即 4xx/5xx）| 仅 START |
| **EQUAL**（DNS 纠错默认）| 等于 | `ERRORCODESTART=3`（=3 即 NXDOMAIN）| 仅 START |
| GE / LE / LT | 大于等于 / 小于等于 / 小于 | 范围控制 | START + END 共同 |

### 3. 扩展过滤器逻辑组合（EXTFLTTYPE）

| EXTFLTTYPE | 语义 | 说明 |
|---|---|---|
| **AND**（默认）| 命中所有 AND 组过滤器 | HTTP 智能重定向 URL+USERAGENT 组合用 AND |
| OR | 命中任一 OR 组过滤器 | 多 URL 场景 |
| NOT | 未命中 NOT 组所有过滤器 | 排除特定 URL |

> 动作命令最多 5 组扩展过滤器（EXTFLTTYPE1~5 / EXTFLTNAME1~5），EXTFLTTYPE1/EXTFLTNAME1 必选，其余条件必选。

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| 动作类型 SMARTHTTPREDIR（联动特性级）| ADD SMARTHTTPREDIR → SERVERURL；HTTP URL 重定向；用 REDIRAPPENDINFO；ERRORCODEOP=GT |
| 动作类型 DNSOVERWRITING（联动特性级）| ADD DNSOVERWRITING → SERVERIP1；DNS IP 改写；不用 REDIRAPPENDINFO；ERRORCODEOP=EQUAL |
| 错误码操作 ERRORCODEOP=GT（联动特性级）| ERRORCODESTART=400（HTTP 4xx/5xx）；仅 START 生效 |
| 错误码操作 ERRORCODEOP=EQUAL（联动特性级）| ERRORCODESTART=3（DNS NXDOMAIN）；仅 START 生效 |
| 错误码操作 GE/LE/LT | START+END 共同控制范围（定制场景） |
| 扩展过滤器逻辑 EXTFLTTYPE=AND | 多过滤器全命中（HTTP 默认，URL+USERAGENT）|
| 扩展过滤器逻辑 EXTFLTTYPE=OR | 任一命中（多 URL 场景）|
| 扩展过滤器逻辑 EXTFLTTYPE=NOT | 排除（定制场景）|

## 约束

- **EXTFLTNAME 引用须已配置**（critical）：动作命令的 EXTFLTNAME1/2 须先经 ADD EXTENDEDFILTER（[0-00022](task/UDG/20.15.2/0-00022.md)）配置 — 后果：引用不存在拒收。
- **BINDERRCODENAME 引用须已配置**（critical）：动作命令的 BINDERRCODENAME 须先经 ADD ERRORCODE（[0-00089](task/UDG/20.15.2/0-00089.md)）配置 — 后果：引用不存在拒收。
- **动作对象须先于 RULE 配置**（critical）：SMTHTTPREDINAME / DNSOVERWRTNAME 须先于 `RULE(POLICYTYPE=SMARTREDIRECT)` 配置 — 后果：RULE 引用失败。
- **POLICYTYPE=SMARTREDIRECT 直接引用动作对象名**（critical）：RULE 的 POLICYNAME 指向 SMTHTTPREDINAME / DNSOVERWRTNAME（非 PCCPOLICYGRPNM）— 指错则智能重定向不触发。
- **REDIRAPPENDINFO 仅 HTTP 智能重定向用**（warning）：DNS 纠错不配携带信息；HTTP 场景 APPENDINFONAME 须先经 ADD REDIRAPPENDINFO（[0-00023](task/UDG/20.15.2/0-00023.md)）配置。
- **扩展过滤器可重复执行配多个**（info）：ADD EXTENDEDFILTER 重复执行增加多个不同过滤条件（activation §操作步骤 说明，HTTP 用 2 个，DNS 用 1 个）。
- **规格差异**（warning）：SMARTHTTPREDIR 整机 ≤50（明显小于 DNSOVERWRITING ≤2000 / REDIRAPPENDINFO ≤1000）；动作命令最多 5 组扩展过滤器。
