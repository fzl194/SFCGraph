# 创建IPsec安全提议（ADD IPSECPROPOSAL）

> 命令静态知识见 [ADD IPSECPROPOSAL](command/UDG/20.15.2/ADD-IPSECPROPOSAL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。PROPOSALNAME 被 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY（IPSECPOLICY → IPSECPROPOSALIPSEC）的 PROPOSAL1/2/3 引用；与 IPsec 微服务侧的 [0-00212](task/UDG/20.15.2/0-00212.md) ADD IPSECPROPOSALIPSEC 是双配同源对象。

## 配置方法（配置方法字典）

ADD IPSECPROPOSAL 是"IPsec 提议创建"命令。1 个必选参数（PROPOSALNAME）+ 4 个可选/条件可选参数。配置空间有 4 个维度：**提议名**（PROPOSALNAME 引到 IPSECPOLICY）、**安全协议**（IPSECPROTOCOL 决定走 AH 还是 ESP）、**认证算法**（AHAUTHALGO/ESPAUTHALGO 配套）、**加密算法**（ESPENCRYPTALGO 仅 ESP 配）、**封装模式**（ENCAPMODE=transport 固定）。

### 配置维度 1：提议名（参数 PROPOSALNAME，必选）

| 取值形式 | 来源 | 引用关系 |
|---|---|---|
| **字符串 1~15（不区分大小写）** | 本端规划 | 被 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 的 PROPOSAL1/PROPOSAL2/PROPOSAL3 引用 |

> **同名冲突**：PROPOSALNAME 不允许重复——同一 IPsec 提议名只能创建一次。修改用 [MOD IPSECPROPOSAL](command/UDG/20.15.2/MOD-IPSECPROPOSAL.md)，删除用 [RMV IPSECPROPOSAL](command/UDG/20.15.2/RMV-IPSECPROPOSAL.md)。

### 配置维度 2：安全协议（参数 IPSECPROTOCOL，可选，默认 esp）

| 取值 | 配套认证算法 | 配套加密算法 | 适用场景 |
|---|---|---|---|
| **ah**（默认 esp） | `AHAUTHALGO`（必选） | — | AH 协议仅认证不加密；命令示例 `asdf2` 即 AH 范式 |
| **esp** | `ESPAUTHALGO`（可选，未配/默认 sha2_256） | `ESPENCRYPTALGO`（可选，未配/默认 aes_256） | ESP 协议同时认证+加密（最常用） |

> **三选一铁律**：`IPSECPROTOCOL=ah` 时只能配 `AHAUTHALGO`；`IPSECPROTOCOL=esp` 时只能配 `ESPAUTHALGO` + `ESPENCRYPTALGO`；不能跨协议错配（AHAUTHALGO 不能在 ESP 协议下用，反之亦然）。

### 配置维度 3：认证算法（参数 AHAUTHALGO/ESPAUTHALGO，协议决定条件必选）

#### 3a. AH 协议下的认证算法（参数 AHAUTHALGO，IPSECPROTOCOL=ah 时必选）

| 取值 | 算法 | 安全性 |
|---|---|---|
| **sha2_256**（默认） | SHA2-256 | 安全（推荐） |
| sha2_384 | SHA2-384 | 安全 |
| sha2_512 | SHA2-512 | 安全 |
| sha1 | SHA1 | **不安全**（仅兼容） |
| md5 | MD5 | **不安全**（仅兼容） |
| unconfigure | 未配置 | 使用默认值 |

#### 3b. ESP 协议下的认证算法（参数 ESPAUTHALGO，IPSECPROTOCOL=esp 时可选）

| 取值 | 算法 | 安全性 |
|---|---|---|
| **sha2_256**（默认） | SHA2-256 | 安全（推荐） |
| sha2_384 | SHA2-384 | 安全 |
| sha2_512 | SHA2-512 | 安全 |
| sha1 | SHA1 | **不安全**（仅兼容） |
| md5 | MD5 | **不安全**（仅兼容） |
| null | 空（不认证） | 存在安全风险 |
| unconfigure | 未配置 | 使用默认值 |

> **AH 默认铁律**：AH 协议下 AHAUTHALGO 不能为空——选择 unconfigure 时系统下发默认值 sha2_256。

### 配置维度 4：加密算法（参数 ESPENCRYPTALGO，IPSECPROTOCOL=esp 时可选）

| 取值 | 算法 | 安全性 |
|---|---|---|
| **aes_256**（默认） | AES-256 | 安全（推荐） |
| aes_192 | AES-192 | 安全 |
| aes_128 | AES-128 | 安全 |
| 3des | 3DES | **不安全**（仅兼容） |
| des | DES | **不安全**（仅兼容） |
| null | 空（不加密） | 存在安全风险 |
| unconfigure | 未配置 | 使用默认值 |

> **null/unconfigure 安全告警**：当 ESP 认证/加密算法选择 null 或 unconfigure 时存在安全风险；命令 notes 明示推荐使用 sha2_256 + aes_256 组合。

### 配置维度 5：封装模式（参数 ENCAPMODE，可选，默认 transport）

| 取值 | 作用 |
|---|---|
| **transport**（默认/唯一） | 传输模式，仅加密 IP 负载，不包含 IP 头 |

> **当前仅支持 transport**：命令当前版本 ENCAPMODE 仅取 transport；隧道模式（tunnel）未开放。

**典型脚本（AH 协议 + SHA2-256 认证，仅 IPSECPROTOCOL=AHAUTHALGO 三必选）**：
```
ADD IPSECPROPOSAL:PROPOSALNAME="asdf2",IPSECPROTOCOL=ah,AHAUTHALGO=sha2_256,ENCAPMODE=transport;
```

**典型脚本（ESP 协议 + SHA2-256 认证 + AES-256 加密，使用默认值省略可选参数）**：
```
ADD IPSECPROPOSAL:PROPOSALNAME="map1",IPSECPROTOCOL=esp;
```

**典型脚本（IPv4 IPsec 策略 PROPOSALNAME 引用，0-00215 ADD IPSECPOLICY 的 PROPOSAL1 引用本命名）**：
```
ADD IPSECPROPOSAL:PROPOSALNAME="proposal1",IPSECPROTOCOL=esp,ESPAUTHALGO=sha2_256,ESPENCRYPTALGO=aes_256,ENCAPMODE=transport;
```

> 命令 notes：**最大记录数 100**；**执行后立即生效**；**部分配置不安全**：MD5/SHA1/3DES/DES 仅作兼容用，建议生产环境使用 sha2_256 + aes_256；**AH 默认值**：AHAUTHALGO 选 unconfigure 时下发 sha2_256 默认值；**null 算法告警**：ESP 选 null/unconfigure 时存在安全风险；**当前封装模式**：仅支持 transport。

**步骤位置**：在所属 feature 编排中通常位于"IPsec 协商链·提议创建"——前置 [0-00213](task/UDG/20.15.2/0-00213.md) ADD IKEPROPOSAL（IKE 提议，可平行创建）；下游 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY（PROPOSAL1/2/3 引用本 PROPOSALNAME）；双配同源 [0-00212](task/UDG/20.15.2/0-00212.md) ADD IPSECPROPOSALIPSEC（IPsec 微服务侧 IPSec 提议）。

## 决策点：IPsec 提议组合选择（DP 0-00308）

问：本 IPsec 提议采用哪种安全组合？（决定 IPSECPROTOCOL + 认证算法 + 加密算法取值）

| 选项 | IPSECPROTOCOL | 认证算法 | 加密算法 | 安全性 | 典型场景 |
|---|---|---|---|---|---|
| **AH + SHA2-256（默认安全）** | ah | AHAUTHALGO=sha2_256 | — | 安全 | 命令示例 `asdf2`；纯认证场景 |
| **AH + SHA2-384/512** | ah | AHAUTHALGO=sha2_384/512 | — | 安全 | 高强度纯认证场景 |
| **ESP + SHA2-256 + AES-256（推荐组合）** | esp | ESPAUTHALGO=sha2_256 | ESPENCRYPTALGO=aes_256 | 安全（推荐） | IPFD-015004 10002 等最常用生产环境 |
| **ESP + SHA2-256 + AES-128/192** | esp | ESPAUTHALGO=sha2_256 | ESPENCRYPTALGO=aes_128/192 | 安全 | 性能与安全平衡场景 |
| **ESP + SHA2-384/512 + AES-256** | esp | ESPAUTHALGO=sha2_384/512 | ESPENCRYPTALGO=aes_256 | 高强度 | 强加密需求 |
| **AH + MD5/SHA1（仅兼容）** | ah | AHAUTHALGO=md5/sha1 | — | **不安全** | 仅兼容老旧设备，不推荐生产 |
| **ESP + MD5/SHA1 + 3DES/DES（仅兼容）** | esp | ESPAUTHALGO=md5/sha1 | ESPENCRYPTALGO=3des/des | **不安全** | 仅兼容老旧设备 |
| **ESP + null 认证/加密** | esp | ESPAUTHALGO=null | ESPENCRYPTALGO=null/unconfigure | 存在安全风险 | 调试场景，不推荐生产 |
| **AH + unconfigure 认证** | ah | AHAUTHALGO=unconfigure | — | 系统下发 sha2_256 默认值 | AH 协议下默认行为 |

> 所有算法组合封装模式 ENCAPMODE=transport 固定（当前版本不支持 tunnel）；AH 与 ESP 算法**不能跨协议错配**——AHAUTHALGO 仅在 IPSECPROTOCOL=ah 时生效，反之 ESPAUTHALGO/ESPENCRYPTALGO 仅在 esp 时生效。

## 约束

- **最大记录数 100**（warning，rule-0-00308）：系统最大记录数 100 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00308）：命令执行后立即生效 — 后果：误操作即刻生效
- **MD5/SHA1 不安全告警**（warning，rule-0-00308）：MD5 与 SHA1 认证算法均为不安全算法 — 后果：仅兼容老设备，建议生产环境使用 sha2_256
- **3DES/DES 不安全告警**（warning，rule-0-00308）：3DES 与 DES 加密算法均为不安全算法 — 后果：仅兼容老设备，建议生产环境使用 aes_256
- **null/unconfigure 安全风险**（warning，rule-0-00308）：ESP 选 null 或 unconfigure 时存在安全风险 — 后果：明文/未加密传输，建议生产环境显式选算法
- **AH 认证不能为空**（critical，rule-0-00308）：AHAUTHALGO 选 unconfigure 时下发默认值 sha2_256；不允许真正为空 — 后果：AH 协议无认证报文不完整
- **协议-算法对齐铁律**（critical，rule-0-00308）：IPSECPROTOCOL=ah 时只能配 AHAUTHALGO；=esp 时只能配 ESPAUTHALGO+ESPENCRYPTALGO — 后果：协议与算法错配命令拒收
- **PROPOSALNAME 不重复**（critical，rule-0-00308）：PROPOSALNAME 不允许与已建提议同名 — 后果：同名冲突命令拒收
- **PROPOSALNAME 长度**（info，rule-0-00308）：字符串 1~15（不区分大小写） — 后果：超长命令拒收
- **权限要求**（info，rule-0-00308）：需 G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
