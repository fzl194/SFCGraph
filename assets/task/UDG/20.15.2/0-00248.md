# 配置NTP服务器（ADD NTPSVR）

> 命令静态知识见 [ADD NTPSVR](command/UDG/20.15.2/ADD-NTPSVR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 NPFD-010014（激活支持 NTP 功能）的单步配置——增加一条 NTP 服务器记录，供网元外接 NTP 服务器同步时钟。NTP 服务器提供时钟同步源，各类设备通过外接 NTP 服务器修正自身时间。以下参数需与 NTP 服务器端协商一致：IP 地址、身份验证标志、密钥号、密钥类型、密钥串。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| SVRNAME | NTP 服务器名 | `server3` | 全网规划，长度 ≤32 字母/数字（**必填**） |
| IPTYPE | IP 地址类型 | `IPV4` | 全网规划，枚举 IPV4/IPV6（**必填**） |
| IPADDRESS | IPV4 地址 | `192.168.156.12` | 全网规划，0.0.0.0~255.255.255.255（条件必填：IPTYPE=IPV4 时） |
| PROTOCOLVERSION | NTP 协议版本 | `NTPV4` | 全网规划，枚举 NTPV4（**必填**） |
| AUTHFLAG | 身份验证标志 | `YES` | 全网规划，枚举 YES/NO（**必填**） |
| KEYID | 身份验证密钥号 | `32323` | 全网规划，整数 1~65533（条件必填：AUTHFLAG=YES 时） |
| SHASTRING | 密钥类型 | `SHA256` | 全网规划，枚举 SHA256（条件必填：AUTHFLAG=YES 时） |
| KEYSTRING | 密钥串 | `*****` | 全网规划，十六进制长度 40~128（条件必填：AUTHFLAG=YES 时；**敏感凭据，不硬编码，实际取值由全网规划/密钥管理提供**） |

**脚本样例**（KEYSTRING 为占位）

```
ADD NTPSVR:SVRNAME="server3",IPTYPE=IPV4,IPADDRESS="192.168.156.12",PROTOCOLVERSION=NTPV4,AUTHFLAG=YES,KEYID=32323,SHASTRING=SHA256,KEYSTRING="*****";
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：IPADDRESS6（IPV6 地址）为 conditional（IPTYPE=IPV6 时必填），本特性取 IPV4 故 IPV6 路径不绑；现网 IPv6 场景按需补绑。
