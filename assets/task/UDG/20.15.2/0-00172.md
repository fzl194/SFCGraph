# 设置全局下行数据缓存时长（SET GLBDLBUFTIME）

> 命令静态知识见 [SET GLBDLBUFTIME](command/UDG/20.15.2/SET-GLBDLBUFTIME.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令配置 **eDRX 模式下整机级下行数据缓存时长**（仅在不配 APN 场景生效；APN 级配置由 [0-00174](task/UDG/20.15.2/0-00174.md) SET APNDLBUFTIME 覆盖，优先级 APN>全局）。整机级全局参数，最大记录数为 1。

命令真相有三个用户类型时长参数（NORMALUSER/NBIOTUSER/REDCAPNRUSER），**跨特性各取其一**——本 atom 只承 NB-IoT 路径（NBIOTUSER）；eMTC/5G 时长由 APN 级 atom 0-00174 同参数覆盖，不需重复绑全局参数。

**配置范式（按用户类型选一参数，本端规划）**

| 参数 | 取值样例 | 获取方法 | 取值范围 | 适用特性 |
|---|---|---|---|---|
| NBIOTUSER | `20` | 本端规划 | 3~200 秒 | GWFD-110601 NB-IoT eDRX |
| NORMALUSER | （不绑，由 0-00174 APN 级覆盖） | 本端规划 | 3~15 秒 | GWFD-110641 eMTC eDRX |
| REDCAPNRUSER | （不绑，由 0-00174 APN 级覆盖） | 本端规划 | 3~200 秒 | GWFD-110661 5G eDRX |

> 三参数系统初始值均为 6s。若 MME/SMF 通过 DDN ACK 下发缓存时长，取下发值；否则取本参数。

**配置步骤（以 NB-IoT 为例）**

1. 全网规划确认全局缓存时长（按无线侧能力，过大可能导致性能下降）。
2. 执行全局缓存时长：
   ```
   SET GLBDLBUFTIME: NBIOTUSER=20;
   ```
3. 如需 APN 级差异化，改用 [0-00174](task/UDG/20.15.2/0-00174.md) 配置（优先级 APN>全局）。

**示例脚本**：`SET GLBDLBUFTIME: NBIOTUSER=20;`

## 约束

- **rule-0-00345**（warning）最大记录数 1（整机级），超限 SET 失败；命令立即生效；仅在不配 APN 场景生效，APN 级配置（[0-00174](task/UDG/20.15.2/0-00174.md)）优先级 APN>全局；缓存时长设置过大可能导致性能下降，须按无线侧能力合理取值；系统初始 NORMALUSER/NBIOTUSER/REDCAPNRUSER 均=6s；若 MME 通过 DDN ACK 下发缓存时长则取下发值 — 后果：超限 SET 失败 / 在 APN 场景外执行不生效 / 取值过大性能下降
