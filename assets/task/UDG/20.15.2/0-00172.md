# 设置全局下行数据缓存时长（SET GLBDLBUFTIME）

> 命令静态知识见 [SET GLBDLBUFTIME](command/UDG/20.15.2/SET-GLBDLBUFTIME.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET GLBDLBUFTIME 用于配置 eDRX 模式下整机级下行报文缓存时长（仅在不配 APN 场景生效；APN 级由 [0-00174](task/UDG/20.15.2/0-00174.md) SET APNDLBUFTIME 覆盖，**优先级 APN>全局**）。**所有参数均为可选**（整机级全局配置，**最大记录数 1**）。核心配置空间有 3 个维度：**普通用户时长**（NORMALUSER）、**NB-IoT 时长**（NBIOTUSER）、**RedCap-NR 时长**（REDCAPNRUSER）。3 个特性命中（110601 NB-IoT eDRX / 110641 eMTC eDRX / 110661 5G eDRX）按用户类型选其一参数。

### 配置维度 1：普通用户下行数据缓存时长（参数 NORMALUSER，可选）

| 取值 | 作用 |
|---|---|
| **整数 3~15（秒）**（系统初始值 6s） | 全局普通用户下行报文缓存时长（110641 范式仅在 APN 级覆盖；本 atom 不承典型取值） |

### 配置维度 2：NB-IoT 用户下行数据缓存时长（参数 NBIOTUSER，可选）

| 取值 | 作用 |
|---|---|
| **整数 3~200（秒）**（系统初始值 6s） | 全局 NB-IoT 用户下行报文缓存时长（110601 范式 20） |

### 配置维度 3：RedCap-NR 用户下行数据缓存时长（参数 REDCAPNRUSER，可选）

| 取值 | 作用 |
|---|---|
| **整数 3~200（秒）**（系统初始值 6s） | 全局 RedCap-NR 用户下行报文缓存时长（110661 范式仅在 APN 级覆盖；本 atom 不承典型取值） |

> 若 MME/SMF 通过 DDN ACK 下发缓存时长，取下发值；否则取本参数。

**典型脚本（GWFD-110601 NB-IoT eDRX，全局 NB-IoT 时长）**：
```
SET GLBDLBUFTIME:NBIOTUSER=20;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：设全局下行缓存时长"——配合 [0-00173](task/UDG/20.15.2/0-00173.md) SET GLBDLLTBUFFER（全局下行长时间缓存个数）+ [0-00174](task/UDG/20.15.2/0-00174.md) SET APNDLBUFTIME（APN 级覆盖，优先级更高）使用。

## 决策点：用户类型时长选择（DP 0-00172）

问：本特性需配置哪种用户类型的下行缓存时长？

| 选项 | 主参数 | 取值范围 | 适用场景 |
|---|---|---|---|
| **NB-IoT**（110601） | NBIOTUSER | 3~200s | NB-IoT eDRX 全局时长（典型 20s） |
| **普通用户**（110641） | NORMALUSER | 3~15s | eMTC eDRX 全局时长（典型 12s） |
| **RedCap-NR**（110661） | REDCAPNRUSER | 3~200s | 5G eDRX 全局时长（典型 20s） |

> 3 个特性命中（110601/110641/110661）按用户类型各取其一参数；**APN 级配置（0-00174）优先级更高**，全局设置仅在不配 APN 时生效。

## 约束

- **所有参数均为可选**（info，rule-0-00172）：NORMALUSER/NBIOTUSER/REDCAPNRUSER 均为可选 — 后果：缺省时取系统初始值 6s
- **整机最大记录数 1**（warning，rule-0-00172）：本命令整机级全局配置，最大记录数 1 — 后果：超限 SET 失败
- **执行后立即生效**（warning，rule-0-00172）：本命令执行后立即生效 — 后果：误配置即刻影响下行报文缓存
- **APN 级覆盖优先**（warning，rule-0-00172）：[0-00174](task/UDG/20.15.2/0-00174.md) SET APNDLBUFTIME 优先级 APN>全局 — 后果：配 APN 后全局设置仅对该 APN 外用户生效
- **DDN ACK 下发优先**（info，rule-0-00172）：若 MME/SMF 通过 DDN ACK 下发缓存时长则取下发值 — 后果：本地设置被下发值覆盖
- **取值过大性能下降**（warning，rule-0-00172）：缓存时长设置过大可能导致性能下降 — 后果：须按无线侧能力合理取值
