# QoS专有承载配置链

> **QoS 族通用 compound**（业务触发 QoS 保证）：配置 QoS 属性 + SA 触发专有承载，为业务流提供专有承载/专有 QoS Flow。被引用于 [2-00029](task/UDG/20.15.2/2-00029.md)（业务触发的QoS保证）/[2-00030](task/UDG/20.15.2/2-00030.md)（视频承载信令控制，待建）。

## 配置方法

两步：配置 QoS 属性（绑 QoS URR + GBR/MBR）→ 使能 SA 触发专有承载（TRIGGERMODE）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| QoS 属性 | `ADD QOSPROP` | QOSPROPNAME + QOSURRNAME（绑 QoS URR，即 ADD URR USAGERPTMODE=QOS）+ GBRUPLKVALUE/GBRDNLKVALUE（保证比特率）+ MBRUPLKVALUE/MBRDNLKVALUE（最大比特率）|
| SA 触发专有承载 | `ADD SADEDICBEARER` | PROTOCOLLEVEL（PROTOCOL 协议级/IP 等）+ PROTOCOLNAME + TRIGGERMODE（DOWNLINK_ONLY 下行触发/UPLINK_ONLY/BOTH）|

**典型脚本（GWFD-020358 七层 QoS：HTTP 下行触发专有承载）**：
```
ADD URR:URRNAME="urr_1",URRID=1001,USAGERPTMODE=QOS;
ADD QOSPROP:QOSPROPNAME="qos-property1",QOSURRNAME="urr_1",GBRUPLKVALUE=110,GBRDNLKVALUE=110,MBRUPLKVALUE=220,MBRDNLKVALUE=220;
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="ppg1",URRGROUPNAME="urrgroup1",QOSPROPNAME="qos-property1";
ADD SADEDICBEARER:PROTOCOLLEVEL=PROTOCOL,PROTOCOLNAME="http",TRIGGERMODE=DOWNLINK_ONLY;
```

**步骤位置**：承接 [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链）+ [1-00016](task/UDG/20.15.2/1-00016.md)（SA协议识别，WELLKNOWNPORT）；QoS URR（USAGERPTMODE=QOS）是 [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套的 QoS 变体（URR 统计类型=QOS），PCCPOLICYGRP 增绑 QOSPROPNAME；产出 PCCPOLICYGRPNM 供 [1-00011](task/UDG/20.15.2/1-00011.md) RULE.POLICYNAME 引用。

> 注：URR(QOS) + PCCPOLICYGRP(QOSPROPNAME) 属 [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套的 QoS 变体（命令同，参数变种：USAGERPTMODE=QOS + 增 QOSPROPNAME 绑定）。本 compound 聚焦 QoS 专属的 QOSPROP + SADEDICBEARER 两命令；URR/PCCPOLICYGRP 经 1-00010 引用，QoS 变体见场景差异。

## 场景差异

| 维度 | 2-00029 业务触发QoS（七层）| 2-00029 业务触发QoS（三四层）| 2-00030 视频承载解耦 |
|---|---|---|---|
| SADEDICBEARER | **有**（PROTOCOLLEVEL=PROTOCOL, TRIGGERMODE=DOWNLINK_ONLY）| **无**（按报文匹配触发，可选软参 BIT1104）| **无**（专有承载由 QOSPROP.DECOUPLINGSW+规则匹配驱动，不经 SA 专有承载命令）|
| QOSPROP 参数 | QOSURRNAME + GBR/MBR 四档 | QOSURRNAME + GBR/MBR 四档 | **仅 QOSURRNAME + DECOUPLINGSW=ENABLE（无 GBR/MBR）**|
| WELLKNOWNPORT | 有（协议识别）| 无 | 无 |
| UPGLBPMPARA | 有（FLOWDANYFMT）| 有 | 无 |
| 过滤匹配 | L7FILTER+PROTBINDFLOWF(http+L7FILTER) | 仅 L34（FLOWFILTER 含 TETHERDETFLAG 可选）| PROTBINDFLOWF 仅 PROTOCOLNAME（**无 L7FILTERNAME**，按协议名绑定）|

> **★ 复用警示（防配置生成误带命令）**：2-00030 视频承载解耦**不执行本 compound 的 SADEDICBEARER 步**，仅配 QOSPROP（DECOUPLINGSW 变体）。2-00029 三四层亦无 SADEDICBEARER。配置生成按各 feature 配置流程产出，**勿按本 compound 典型脚本（含 SADEDICBEARER）盲目套用 2-00030/三四层场景**。

## 决策点

| 选项 | 影响 |
|---|---|
| SADEDICBEARER PROTOCOLLEVEL=PROTOCOL（联动特性级）| 协议级触发专有承载（七层，需 SA 识别协议）|
| SADEDICBEARER TRIGGERMODE=DOWNLINK_ONLY | 下行报文触发专有承载创建 |
| QOSPROP GBR<MBR | 保证比特率≤最大比特率（GBR 内保证，超出 best-effort）|

## 约束

- **QOSURRNAME 引用须已配置**（critical）：QOSPROP.QOSURRNAME 须引用已配的 QoS URR（ADD URR USAGERPTMODE=QOS）— 不存在则 QoS 属性引用断链。
- **PCCPOLICYGRP 绑 QOSPROPNAME**（critical）：PCCPOLICYGRP 须绑 QOSPROPNAME + URRGROUPNAME — 漏绑则 QoS 保证不生效。
- **SADEDICBEARER 依赖协议识别**（critical）：PROTOCOLLEVEL=PROTOCOL 时需 SA 识别协议（[1-00016](task/UDG/20.15.2/1-00016.md) + 过滤链 PROTBINDFLOWF）— 未识别则专有承载不触发。
- **GBR ≤ MBR**（warning）：保证比特率须 ≤ 最大比特率 — 否则 QoS 无法保证。
