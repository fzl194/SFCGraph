# 绑定用户组到APN（ADD APNBINDBWMUSRG）

> 命令静态知识见 [ADD APNBINDBWMUSRG](command/UDG/20.15.2/ADD-APNBINDBWMUSRG.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD APNBINDBWMUSRG 是带宽控制骨架的"APN 接入绑定层"：把"哪一 APN 走哪一用户组的带宽策略"用一条命名记录描述，产出 (USERGROUPNAME, APN) 复合绑定使该 APN 下所有用户落入指定用户组。命令核心配置空间仅 1 个维度：**2 必选引用**（USERGROUPNAME + APN 均为必选 reference）。无其他可选参数——命令仅 2 参，组合无变化。

### 配置维度 1：2 必选引用（2 个必选 reference）

| 参数 | 引用对象 | 来源 atom | 何时配 |
|---|---|---|---|
| **USERGROUPNAME** | 带宽管理用户组名称 | 引用 [0-00033](task/UDG/20.15.2/0-00033.md)（ADD BWMUSERGROUP）的 USERGROUPNAME | 总是必选 |
| **APN** | APN 名称 | 引用 [0-00020](task/UDG/20.15.2/0-00020.md)（ADD APN）的 APN | 总是必选 |

**典型脚本（020354 基于业务的 Shaping）**：
```
ADD APNBINDBWMUSRG:USERGROUPNAME="usergroup_test",APN="huawei.com";
```

**典型脚本（110301 码率差异化）**：
```
ADD APNBINDBWMUSRG:USERGROUPNAME="ug_test",APN="apn-test";
```

**典型脚本（110311 业务感知带宽控制）**：
```
ADD APNBINDBWMUSRG:USERGROUPNAME="ug_test",APN="apn-test";
ADD APNBINDBWMUSRG:USERGROUPNAME="ugroup8",APN="apn-test";
ADD APNBINDBWMUSRG:USERGROUPNAME="ug-bwm", APN="testapn";
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；同一 APN 可绑定多个用户组（取决于 BWM 评估策略）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：APN 接入绑定"——前置 ADD BWMUSERGROUP(0-00033) + ADD APN(0-00020)；产出 (USERGROUPNAME, APN) 二元绑定供 BWM 评估时使用。

## 决策点

（无。本命令仅 2 必选引用，无可选分支、无 DP；唯一性约束是配置生成自检的核心，详见约束段。）

## 约束

- **USERGROUPNAME 前置依赖**（critical，rule-0-00335）：USERGROUPNAME 必须先于 ADD APNBINDBWMUSRG 由 [0-00033](task/UDG/20.15.2/0-00033.md)（ADD BWMUSERGROUP）建成 — 后果：未建引用失败
- **APN 前置依赖**（critical，rule-0-00335）：APN 必须先于 ADD APNBINDBWMUSRG 由 [0-00020](task/UDG/20.15.2/0-00020.md)（ADD APN）建成 — 后果：未建引用失败
- **(USERGROUPNAME, APN) 唯一**（critical，隐含）：同一对 (用户组, APN) 不能重复绑定 — 后果：重复 ADD 失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
