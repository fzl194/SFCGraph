# 创建OSPF区域(IPv4)（ADD OSPFAREA）

> 命令静态知识见 [ADD OSPFAREA](command/UDG/20.15.2/ADD-OSPFAREA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD OSPFAREA 在 OSPF 进程下创建区域。`PROCID` 引用上游 ADD OSPF（binding=reference→0-00074）；`AREAID` 取值来源跨特性分叉——GWFD-010104（地址分配方式）标"与对端协商"（backbone 常取 0.0.0.0），GWFD-010155（PGW-U 到 ePDG 动态路由组网）按全网规划取 0.0.0.1，源解析分叉升级为 decision_driven 由 DP 0-00078 选。NSSA/Stub 区域进阶参数（15 项）在 6 个特性数据规划表+脚本中均未用到，按通用配置筛不绑。

**取值范式（从证据数据规划表归纳）**

| 特性 / 场景 | PROCID | AREAID | 脚本样例 |
|---|---|---|---|
| GWFD-010104/010105/020406/020421 地址分配类（与对端协商 backbone） | 100（引用 0-00074） | 0.0.0.0（与对端协商，peer） | `ADD OSPFAREA:PROCID=100,AREAID="0.0.0.0";` |
| GWFD-010155 PGW-U 到 ePDG（全网规划） | 6（引用 0-00074） | 0.0.0.1（全网规划） | `ADD OSPFAREA:PROCID=6,AREAID="0.0.0.1";` |
| IPFD-015004 OSPF over IPsec（对端协商） | 20214（引用 0-00074） | 10.102.105.0（对端协商） | `ADD OSPFAREA:PROCID=20214,AREAID="10.102.105.0";` |

**关键规则**
- **PROCID 必须引用已创建的 OSPF 进程**：binding=reference→[创建OSPF进程(IPv4)](task/UDG/20.15.2/0-00074.md)，先 ADD OSPF 再 ADD OSPFAREA（命令前置约束）。
- **AREAID 由 DP 0-00078 决定**：option opt-areaid-peer（与对端协商，backbone 常取 0.0.0.0，适用地址分配类对接外部路由器）/ option opt-areaid-global（全网规划，适用 PGW-U 到 ePDG 组网）。OSPF 区域号本质须与对端在同一区域一致，否则邻居无法建立；default 取 opt-areaid-peer。
- **AREATYPE 不绑**：md 说明提及可设 NSSA/Normal（NSSA 须配 Loopback 作为 FA），但不在数据规划表，按通用配置筛不绑，其 NSSA 条件要求以 rule 0-00191 投影。
- **AREAID 传递至下游**：ADD OSPFNETWORK（0-00076）引用本 atom 的 AREAID，故本 DP 决策会传递至下游网段配置。

## 决策点

**DP 0-00078（single_choice）OSPF 区域号（AREAID）来源** — ADD OSPFAREA 的 AREAID 取值来源？

| 选项 | 取值 | 适用场景 | 影响 |
|---|---|---|---|
| opt-areaid-peer（与对端协商，default） | peer | 地址分配类场景（GWFD-010104），须与对端路由器在同一区域协商一致，backbone 常取 0.0.0.0 | AREAID variable_source=peer |
| opt-areaid-global（全网规划） | global_planned | PGW-U 到 ePDG 动态路由组网（GWFD-010155），取 0.0.0.1 等，仍须与对端在同一区域 | AREAID variable_source=global_planned |

## 约束

- **rule 0-00191**（constraint_rule/execution_constraint）ADD OSPFAREA 须满足：(1) 前置——必须先执行 ADD OSPF 创建对应 OSPF 进程（PROCID）才能使用此命令；(2) OSPF 区域实例数规格默认单进程 50、命令最大记录数 8016；(3) 若 `AREATYPE=NSSA`，则必须配置 Loopback 地址并在 OSPF 进程中发布出去，否则负载分担不生效，极端情况下业务受损；(4) AREATYPE 默认 Normal，可设 NSSA/Normal；(5) 认证类型 ADD OSPFAREAAUTH 为可选修饰命令 — 后果：负载分担不生效 / 业务受损 / 区域创建失败 / 区域实例超规格（critical）。
