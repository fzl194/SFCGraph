# 创建OSPF区域(IPv4)（ADD OSPFAREA）

> 命令静态知识见 [ADD OSPFAREA](command/UDG/20.15.2/ADD-OSPFAREA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令是 OSPF backbone 4 件套之二**——必须在 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 创建 PROCID 进程之后执行；下游 [0-00076](task/UDG/20.15.2/0-00076.md) ADD OSPFNETWORK 引用本命令的 PROCID+AREAID。

## 配置方法（配置方法字典）

ADD OSPFAREA 是"**在 OSPF 进程下创建区域**"命令。PROCID 必选且引用 [0-00074](task/UDG/20.15.2/0-00074.md) 已建进程；AREAID 必选；AREATYPE 决定是普通区域 / Stub / NSSA。核心配置空间有 4 个维度：**进程号**（PROCID 引用上游）、**区域号**（AREAID 点分十进制 IPv4）、**区域类型**（AREATYPE 决定 NSSA/Stub 进阶参数分支）、**NSSA/Stub 进阶参数**（按 AREATYPE 条件门控的 15 项可选）。N 个特性命中覆盖 3 种 AREAID 取值模式（peer 协商/全网规划/对端协商）。

### 配置维度 1：进程号（参数 PROCID，必选，引用上游）

| 参数 | 取值范围 | 数据来源 | 关联 |
|---|---|---|---|
| **PROCID** | 1~4294967295 | ref [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 已建进程 | 必须已存在，使用 LST OSPF 查看可用进程 |

> **关键约束**：**必须先 ADD OSPF 配置进程才能使用本命令**；PROCID 是同一进程内的多个区域的标识入口。

### 配置维度 2：区域号（参数 AREAID，必选）

| 参数 | 取值范围 | 数据来源 |
|---|---|---|
| **AREAID** | IPv4 地址类型（点分十进制，如 0.0.0.0 / 0.0.0.1 / 10.102.105.0） | 与对端协商 / 全网规划 |

> **典型 AREAID 示例**：地址分配类对接外部路由器 backbone 常取 `0.0.0.0`；PGW-U→ePDG 组网取 `0.0.0.1`；OSPF over IPsec 取 `10.102.105.0`。

### 配置维度 3：区域类型（参数 AREATYPE，可选，默认 Normal）

AREATYPE 决定区域性质和后续进阶参数分支：

| 取值 | 适用场景 | 联动条件 |
|---|---|---|
| **Normal**（默认） | 普通区域 | 无（不触发其他参数） |
| **Stub** | Stub 区域 | 触发 STUBNOSUMMARY/DEFAULTCOST 条件可选 |
| **NSSA** | NSSA 区域 | 触发 NSSA 10 项可选 + Loopback 必备 |

> **NSSA 必备条件**：若 AREATYPE=NSSA，**必须配置 Loopback 地址并在 OSPF 进程中发布出去**，否则负载分担不生效，极端情况下业务受损。

### 配置维度 4：NSSA/Stub 进阶参数（条件门控）

| 子维度 | 关键参数 | 触发条件 | 默认值 |
|---|---|---|---|
| **Stub 缺省路由** | STUBNOSUMMARY（禁止 ABR 向 Stub 区域发 Summary LSA） | AREATYPE=Stub | FALSE |
|  | DEFAULTCOST（Type3 缺省路由开销） | AREATYPE=Stub 或 NSSA | 1 |
| **NSSA 转换** | NSSATRANSLATORALWAYS（指定该 ABR 为转换路由器） | AREATYPE=NSSA | FALSE |
|  | NSSATRANSLATORINTERVAL（转换路由器的失效时间 1~120 秒） | AREATYPE=NSSA | 40 |
|  | NSSASUPPRESSFORWARDINGADDRESS（转换后 Type5 LSA 的 FA=0.0.0.0） | AREATYPE=NSSA | FALSE |
|  | NSSAZEROADDRESSFORWARDING（NSSA ABR 引入外部路由 Type7 LSA FA=0.0.0.0） | AREATYPE=NSSA | FALSE |
| **NSSA 7 类 LSA** | NSSADEFAULTROUTEADVERTISE（产生缺省 7 类 LSA） | AREATYPE=NSSA | FALSE |
|  | NSSANOIMPORTROUTE（不引入外部路由） | AREATYPE=NSSA | FALSE |
|  | NSSANOSUMMARY（禁止 ABR 向 NSSA 发送 Summary LSA） | AREATYPE=NSSA | FALSE |
|  | NSSASETNBIT（DD 报文设置 N-bit 位标志） | AREATYPE=NSSA | FALSE |
| **LDP 同步** | LDPSYNCENABLE（使能 LDP 和 OSPF 同步） | 无 | FALSE |
| **最大开销** | MAXCOSTFLAG/MAXCOSTINTVL（保持最大开销值 0~65535 秒） | MAXCOSTFLAG=TRUE | — |
| **共网段** | 配套 [MOD OSPFAREA](command/UDG/20.15.2/MOD-OSPFAREA.md) VIRTUALSYSFLAG | — | — |
| **自描述** | DESCRIPTIONTEXT（1~80 字符） | — | — |

**典型脚本（地址分配类 backbone 区域 0.0.0.0）**：
```
ADD OSPFAREA: PROCID=100, AREAID="0.0.0.0";
```

**典型脚本（010155 PGW-U→ePDG 区域 0.0.0.1）**：
```
ADD OSPFAREA: PROCID=6, AREAID="0.0.0.1";
```

**典型脚本（IPFD-015004 OSPF over IPsec 区域 10.102.105.0）**：
```
ADD OSPFAREA: PROCID=20214, AREAID="10.102.105.0";
```

> 命令 notes：**最大记录数 8016**；**执行后立即生效**；**OSPF 区域实例数规格默认单进程 50**（实际以服务规格文件为准）；**NSSA 必须配置 Loopback 地址并在 OSPF 进程中发布出去**（否则负载分担不生效）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N+1：在进程下创建区域"——前置 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF；下游 [0-00076](task/UDG/20.15.2/0-00076.md) ADD OSPFNETWORK（PROCID+AREAID 引用）。

## 决策点：OSPF 区域号来源（DP 0-00075）

问：ADD OSPFAREA 的 AREAID 取值来源？

| 选项 | 取值 | 适用场景 |
|---|---|---|
| **与对端协商（默认）** | peer | 地址分配类对接外部路由器（GWFD-010104/010105/020406/020421），与对端路由器在同一区域协商一致；backbone 常取 0.0.0.0 |
| **全网规划** | 全网规划 | PGW-U→ePDG 组网（GWFD-010155），取 0.0.0.1 等，仍须与对端在同一区域 |
| **对端协商** | peer | OSPF over IPsec（IPFD-015004），取对端约定的区域号（如 10.102.105.0） |

> 三个数据模式均要求**对端在同一区域**（否则邻居无法建立）；本 DP 决策会传递至下游 [0-00076](task/UDG/20.15.2/0-00076.md) ADD OSPFNETWORK 的网段 AREAID 字段。

## 约束

- **最大记录数 8016**（warning，rule-0-00475）：本命令最大记录数为 8016 — 后果：超规格新增失败
- **单进程区域实例数默认 50**（warning，rule-0-00475）：OSPF 区域实例数规格默认为单进程 50，实际以服务规格文件为准 — 后果：单进程超过规格新增失败
- **执行后立即生效**（warning，rule-0-00475）：命令执行后立即生效 — 后果：误操作即刻生效
- **必须先 ADD OSPF 才能 ADD OSPFAREA**（critical，rule-0-00475）：只有执行 ADD OSPF 配置了 OSPF 进程才能使用此命令 — 后果：未建进程直接配区域命令拒收
- **AREAID 必须与对端一致**（critical，rule-0-00475）：规划的邻居所在的区域必须一致 — 后果：与对端不一致邻居无法建立
- **NSSA 必须配 Loopback 并发布**（critical，rule-0-00475）：如果 AREATYPE=NSSA，则必须配置 Loopback 地址并在 OSPF 进程中发布出去 — 后果：未配置负载分担不生效，极端情况下业务受损
- **私网 OSPF 不支持 LDP 联动**（info，rule-0-00475）：私网 OSPF 进程下不支持 LDP 和 OSPF 联动功能 — 后果：私网进程配 LDPSYNCENABLE 命令拒收
- **AREATYPE 默认 Normal**（info，rule-0-00475）：AREATYPE 默认值 Normal，可设 NSSA/Normal — 后果：选错区域类型会改变路由汇总行为
