# 配置PFCP路径相关属性（SET UPPFCPPATH）

> 命令静态知识见 [SET UPPFCPPATH](command/UDG/20.15.2/SET-UPPFCPPATH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

PFCP 路径探测是一组全局唯一参数（最大记录数 1，全机一条），定义 UDG（SGW-U/PGW-U/UPF）与对端控制面（SGW-C/PGW-C/SMF）之间的心跳探测节奏与去活策略。一次下发五个参数即可，按"探测间隔 → 重发节奏 → 重发次数 → 告警后探测 → 去活开关"顺序规划。

**参数规划表（取值样例来自 GWFD-010102）**

| 参数 | 含义 | 样例值 | 取值范围 | 规划口径 |
|---|---|---|---|---|
| HEARTBEATINTERVAL | 心跳请求间隔（秒） | 60 | 60~3600 | 本端规划 |
| T3RESPONSE | 请求重发间隔（秒） | 3 | 1~20 | 本端规划 |
| N3REQUEST | 最大重发次数 | 5 | 1~6 | 本端规划 |
| DEACTIVEFLAG | 路径断后去活上下文 | ENABLE | 枚举 | 本端规划 |
| HEARTBEATTIME | 告警后心跳发送次数 | 30 | 0~60 | 本端规划 |

**典型脚本**
```
SET UPPFCPPATH:HEARTBEATINTERVAL=60,T3RESPONSE=3,N3REQUEST=5,DEACTIVEFLAG=ENABLE,HEARTBEATTIME=30;
```

**关键语义（与 SET UPGTPPATH 的差异）**
- 只对新激活用户生效，存量用户沿用旧值（SET UPGTPPATH 立即生效）。
- 高危命令：配置不合理可能触发告警或用户去活，按现网规划设。
- 配额申请定时器联动：对端定时器时长 = T3RESPONSE × N3REQUEST + 4 秒（见 GWFD-010173/GWFD-020300/GWFD-020306 在线计费配额耗尽流程）。

**未绑参数**：USERTYPE（用户类型，命令真相 conditional，010102 数据规划表与脚本均未用到）→ 按通用配置门槛不绑。

## 约束

- **rule-0-00327**（critical）最大记录数 1（全局唯一）；高危命令；执行后只对新激活用户生效；对接 SMF 的 SET T1N1PARA 的 T1+5 秒之和**不能**是本命令 T3RESPONSE 的整数倍 — 违反后果：冗余 DDN 消息 / 用户去活
