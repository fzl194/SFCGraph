# 地址分配规则（全局+APN两级+三级优先级）

> 归属 [2-00034](task/UDG/20.15.2/2-00034.md) 用户面地址分配（GWFD-010105）。本 compound 配置地址分配**规则层**——全局/APN 两级使能 + 三级优先级（APN/LOCATION/SMF 三维条件组合）。被引用于 {2-00034}。

## 配置方法

本 compound 由 2 条命令组成（多命令可复用模块，建 compound 承载）：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 配置全局地址分配规则（整机默认） | [SET IPALLOCRULE](task/UDG/20.15.2/0-00250.md) | `FIRSTRULESW` / `FIRSTRULE` / `SECONDRULESW` / `SECONDRULE` / `THIRDRULESW` |
| 2. 配置 APN 级地址分配规则（覆盖全局，可选） | [SET APNIPALLOCRULE](task/UDG/20.15.2/0-00319.md) | `APN` / `ALLOCATTR=LOCAL` / `FIRSTRULESW` / `FIRSTRULE` |

**典型脚本（010105 场景一：基于 APN/DNN 单级优先级，全局规则）**：
```
SET IPALLOCRULE: FIRSTRULESW=ENABLE, FIRSTRULE=APN-1&LOCATION-0&SMF-0, SECONDRULESW=DISABLE, THIRDRULESW=DISABLE;
```

**典型脚本（010105 场景三：基于 SMF+APN 两级优先级，全局规则）**：
```
SET IPALLOCRULE: FIRSTRULESW=ENABLE, FIRSTRULE=APN-1&LOCATION-0&SMF-1, SECONDRULESW=ENABLE, SECONDRULE=APN-0&LOCATION-0&SMF-1, THIRDRULESW=DISABLE;
```

**典型脚本（010105 场景四：基于 RADIUS 下发地址池名，APN 级规则覆盖全局）**：
```
SET APNIPALLOCRULE: APN="apn-test1", ALLOCATTR=LOCAL, FIRSTRULESW=ENABLE, FIRSTRULE=APN-1&LOCATION-0&SMF-0, SECONDRULESW=DISABLE, THIRDRULESW=DISABLE;
```

## 场景差异

| 场景 | 规则级数 | 规则字符串 | APN 级覆盖 |
|---|---|---|---|
| 基于 APN/DNN（场景一） | 单级（FIRSTRULE） | `APN-1&LOCATION-0&SMF-0` | 可选 |
| 基于位置（场景二，参见 GWFD-020421） | 单级 | `APN-0&LOCATION-1&SMF-0` | 可选 |
| 基于 SMF（场景二） | 单级 | `APN-0&LOCATION-0&SMF-1` | 可选 |
| 基于 SMF+APN（场景三） | 两级（FIRSTRULE + SECONDRULE） | 第一级 `APN-1&LOCATION-0&SMF-1`；第二级 `APN-0&LOCATION-0&SMF-1` | 可选 |
| 基于 RADIUS 下发地址池名（场景四） | 单级（FIRSTRULE） | `APN-1&LOCATION-0&SMF-0`（用 SET APNIPALLOCRULE） | **必配**（覆盖全局） |

## 决策点：规则层级与三级优先级（DP 0-00321）

问：地址分配规则选哪个层级（全局/APN） + 选几级优先级 + 规则字符串怎么写？

| 选项 | 走法 | 规则字符串 | 适用场景 |
|---|---|---|---|
| 全局单级 APN | SET IPALLOCRULE（FIRSTRULE） | `APN-1&LOCATION-0&SMF-0` | 基于 APN/DNN 子方式 |
| 全局单级 SMF | SET IPALLOCRULE（FIRSTRULE） | `APN-0&LOCATION-0&SMF-1` | 基于 SMF 子方式 |
| 全局两级 SMF+APN | SET IPALLOCRULE（FIRSTRULE+SECONDRULE） | 第一级 `APN-1&LOCATION-0&SMF-1`，第二级 `APN-0&LOCATION-0&SMF-1` | 基于 SMF+APN 子方式 |
| APN 级覆盖 | SET APNIPALLOCRULE（ALLOCATTR=LOCAL） | 同上规则字符串 | 基于 RADIUS 下发地址池名（必配覆盖全局） |

## 约束

- **APN 级覆盖全局**（critical，rule-0-00493）：如果设置了基于 APN 的地址分配规则（SET APNIPALLOCRULE），则使用 APN 下配置的规则；否则使用全局规则 — 后果：APN 级规则一旦配置，全局规则对该 APN 不生效
- **三级优先级匹配失败则地址分配失败**（critical，rule-0-00493）：三级优先级规则都匹配失败 → 地址分配失败 — 后果：用户激活失败
- **规则条件"与"关系**（critical，rule-0-00493）：每级规则中所选条件之间为"与"的关系（APN-1&LOCATION-0&SMF-0 = 仅启用 APN 单维度） — 后果：地址分配时按"与"逻辑取交集
- **FIRSTRULE 与地址池映射条件一致**（warning，rule-0-00493）：规则字符串与地址池映射条件必须匹配（FIRSTRULE=`APN-1&SMF-1` 则 POOLGRPMAP 必须含 APN+SMF） — 后果：规则匹配失败
- **SET APNIPALLOCRULE 必配 ALLOCATTR=LOCAL**（warning，rule-0-00493）：RADIUS 子方式必须显式指定 LOCAL 属性（覆盖全局默认） — 后果：缺 ALLOCATTR 命令拒收
