# 配置APN下异常下行流量检测开关（ADD ABNTRAFFICDT）

> 命令静态知识见 [ADD ABNTRAFFICDT](command/UDG/20.15.2/ADD-ABNTRAFFICDT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD ABNTRAFFICDT 是 APN 粒度的"终端异常下行流量检测"使能命令：把"哪一档异常流检测开 / 关"按 APN 维度锁定；前置 APN 已配 + License `LKV3G5ADTD01` 已签。命令核心配置空间有 2 个维度：**总开关**（SWITCH，必选）+ **流行为子开关**（TRAFFICBEHAVIORSWITCH，条件可选，依赖总开关）。两开关独立配置，组合出 4 种语义：全关 / 仅常规异常流 / 仅流行为异常 / 全开；当前 GWFD-020305 默认"全开"。

### 配置维度 1：总检测开关（参数 SWITCH，必选）

| 取值 | 作用 | 联动 |
|---|---|---|
| **ENABLE** | 启用 APN 下终端异常下行流量检测 | TRAFFICBEHAVIORSWITCH 可选配 |
| **DISABLE** | 关闭全部异常流检测 | TRAFFICBEHAVIORSWITCH 不允许配 ENABLE |

### 配置维度 2：基于流行为的检测子开关（参数 TRAFFICBEHAVIORSWITCH，条件可选）

| 取值 | 作用 | 前提 |
|---|---|---|
| **ENABLE** | 在总开关启用基础上额外启用"基于流行为"维度检测 | SWITCH=ENABLE |
| **DISABLE**（默认） | 关闭基于流行为的检测 | SWITCH=ENABLE 或 DISABLE |

**典型脚本（GWFD-020305，全开）**：
```
ADD ABNTRAFFICDT:APN="apn1.com",SWITCH=ENABLE,TRAFFICBEHAVIORSWITCH=ENABLE;
```

> 命令 notes：仅对新激活用户生效（对在线用户不立即生效）；Normal 用户不支持异常流检测；最大记录数 10000；适用 NF：SGW-U、PGW-U、UPF。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：APN 级异常流量检测开关"——前置 ADD APN(0-00020) + License 0-00019 之后。

## 约束

- **总开关与子开关语义约束**（critical，rule-0-00332）：SWITCH=DISABLE 时不允许配 TRAFFICBEHAVIORSWITCH=ENABLE（否则流行为检测无主开关承载） — 后果：语义冲突拒收
- **新增对在线用户不生效**（warning，rule-0-00332）：本命令只对新激活用户生效；对当前在线用户不立即生效 — 后果：配置后用户需重激活才感知
- **Normal 用户不支持流行为检测**（warning，rule-0-00332）：Normal 用户配 TRAFFICBEHAVIORSWITCH=ENABLE 无效（仅异常用户识别） — 后果：Normal 用户不感知
- **最大记录数 10000**（warning，rule-0-00332）：单板最大 10000 条；超限新增失败 — 后果：超规格新增被拒
- **APN 前置依赖**（critical，隐含）：APN 参数必须已 ADD APN(0-00020) 配置 — 后果：APN 未配则 ADD 失败
- **License 前置依赖**（critical，rule-0-00332）：需 License LKV3G5ADTD01（[0-00019](task/UDG/20.15.2/0-00019.md) 入口） — 后果：License 缺失则功能不生效
