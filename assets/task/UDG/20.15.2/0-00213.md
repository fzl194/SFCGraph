# 配置IKE安全提议（ADD IKEPROPOSAL）

> 命令静态知识见 [ADD IKEPROPOSAL](command/UDG/20.15.2/ADD-IKEPROPOSAL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

定义 IKE（密钥交换）阶段的安全参数：认证方法、认证/加密/完整性算法、DH 组、SA 时长。算法取值由 [DP 0-00033](0-00033.md) 编排，**两端协商一致**。PROPOSALNUMBER 为本端编号（无需与对端一致），下游 IKE 对等体 PROPOSAL 字段引用它。

**数据规划取值样例**（证据③，IPFD-015004 各激活 md 数据规划表，PSK 主流 + 国密数字信封场景）：

| 参数 | 取值样例（PSK 主流） | 取值样例（国密数字信封） | 获取方法 | 说明 |
|---|---|---|---|---|
| PROPOSALNUMBER（安全提议号） | `1`、`10` | `1`、`10` | 本端规划 | 本端记录编号，被 IKE 对等体 PROPOSAL 引用；1~4294967295（101 仅查询） |
| AUTHMETHOD（认证方法） | `Pre_share` | `Digital_envelope` | 对端协商（DP 0-00033） | PSK 缺省；国密用数字信封 |
| AUTHALGORITHM（认证算法） | `Sha2_256` | （不绑） | 对端协商（DP 0-00033） | 国密数字信封场景不显式配 |
| ENCRALGORITHM（加密算法） | `Aes_cbc_256` | `Sm4` | 对端协商（DP 0-00033） | 国密用 Sm4 |
| INTEGALGORITHM（完整性算法） | `Hmac_sha2_256` | `Sm3` | 对端协商（DP 0-00033） | 国密用 Sm3 |
| DHGROUP（DH 组） | `Dh_group19`（10009 场景 `Dh_group2`） | （不绑） | 对端协商 | 不能为 None，建议 Dh_group19 |
| SADURATION（SA 持续长度 s） | `3600`、`86400` | `3600`、`86400` | 本端规划 | optional，60~604800 |
| ASYMENCRALG（非对称加密算法） | （不绑） | `Sm2` | 对端协商 | 仅国密数字信封/RSA 场景 |

> REAUTHINTERVAL/SIGHASHALGNEGSW/SIGNPADDING（4 个 conditional 仅 AUTHMETHOD=RSA 时生效，PSK 场景不绑）。设备同时开 IKEv1+IKEv2，缺省 IKEv2，对端不支持则禁用 IKEv2 走 IKEv1。

**典型范式**（证据③任务示例脚本，PSK 主流，GRE/OSPF 场景带 SADURATION）：
```
ADD IKEPROPOSAL: PROPOSALNUMBER=1, AUTHMETHOD=Pre_share, AUTHALGORITHM=Sha2_256,
    ENCRALGORITHM=Aes_cbc_256, INTEGALGORITHM=Hmac_sha2_256, DHGROUP=Dh_group19, SADURATION=3600;
```

**典型范式**（国密数字信封场景）：
```
ADD IKEPROPOSAL: PROPOSALNUMBER=1, AUTHMETHOD=Digital_envelope, ENCRALGORITHM=Sm4,
    INTEGALGORITHM=Sm3, SADURATION=3600, ASYMENCRALG=Sm2;
```

## 决策点

- [DP 0-00033](0-00033.md) 编排 AUTHMETHOD/ENCRALGORITHM/INTEGALGORITHM/AUTHALGORITHM/DHGROUP 的分叉：PSK 标准 IKEv2 算法集 vs 国密数字信封算法集（Sm4/Sm3/Sm2）。

## 约束

- **rule-0-00337**（critical）命令立即生效；含不安全算法提示（提议号 80~91 可软参配 MD5/DES/3DES）；AUTHMETHOD 缺省 Pre_share（PSK），需经 [ADD IKEPEER](0-00214.md) 配 PRESHAREDKEY 否则 SA 协商失败；DHGROUP 不能为 None，建议 Dh_group19；设备同时开 IKEv1+IKEv2 缺省 IKEv2，对端不支持则禁用 IKEv2 走 IKEv1；SIGHASHALGNEGSW/SIGNPADDING/ASYMENCRALG 仅 RSA 场景必填，PSK 不绑。
