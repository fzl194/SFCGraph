# 设置基于APN的下行数据缓存时长（SET APNDLBUFTIME）

> 命令静态知识见 [SET APNDLBUFTIME](command/UDG/20.15.2/SET-APNDLBUFTIME.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET APNDLBUFTIME 用于配置指定 APN 的下行数据缓存时长（eDRX 模式下 APN 级配置，**优先级高于全局** [0-00172](task/UDG/20.15.2/0-00172.md) SET GLBDLBUFTIME）。**APN 必选**；**NORMALUSER/NBIOTUSER/REDCAPNRUSER 均为可选，但单次 SET 仅配一种用户类型时长（互斥）**；**整机最多 10000 条 ApnDLBufTime**。核心配置空间有 4 个维度：**APN 实例**（APN）、**普通用户时长**（NORMALUSER）、**NB-IoT 时长**（NBIOTUSER）、**RedCap-NR 时长**（REDCAPNRUSER）。3 个特性命中（110601 NB-IoT eDRX / 110641 eMTC eDRX / 110661 5G eDRX）按用户类型各取其一参数。

### 配置维度 1：APN 实例（参数 APN，必选）

| 取值 | 作用 |
|---|---|
| **字符串，1~63 字符**（由 `-`/数字/字母/`.` 组成，不以 `.` 开头） | APN 实例名；**引用** [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 已建实例 |

### 配置维度 2：普通用户下行数据缓存时长（参数 NORMALUSER，可选）

| 取值 | 作用 |
|---|---|
| **整数 3~15（秒）**（系统初始值 6s） | APN 下普通用户下行报文缓存时长（110641 范式 12） |

### 配置维度 3：NB-IoT 用户下行数据缓存时长（参数 NBIOTUSER，可选）

| 取值 | 作用 |
|---|---|
| **整数 3~200（秒）**（系统初始值 6s） | APN 下 NB-IoT 用户下行报文缓存时长（110601 范式 10） |

### 配置维度 4：RedCap-NR 用户下行数据缓存时长（参数 REDCAPNRUSER，可选）

| 取值 | 作用 |
|---|---|
| **整数 3~200（秒）**（系统初始值 6s） | APN 下 RedCap-NR 用户下行报文缓存时长（110661 范式 10） |

> 若 SMF 下发 DL Buffering Duration 信元则取下发值，否则取本参数；三时长单次 SET 仅配其一（**互斥**）。

**典型脚本（GWFD-110601 NB-IoT eDRX，APN 级 NB-IoT 时长）**：
```
SET APNDLBUFTIME:APN="apnnb",NBIOTUSER=10;
```

**典型脚本（GWFD-110641 eMTC eDRX，APN 级普通用户时长）**：
```
SET APNDLBUFTIME:APN="apnemtc",NORMALUSER=12;
```

**典型脚本（GWFD-110661 5G eDRX，APN 级 RedCap-NR 时长）**：
```
SET APNDLBUFTIME:APN="apn-test",REDCAPNRUSER=10;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：设 APN 级下行缓存时长"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN（建 APN 实例）；APN 级配置覆盖 [0-00172](task/UDG/20.15.2/0-00172.md) 全局缓存时长。

## 决策点：APN 级用户类型时长三选一（DP 0-00174）

问：SET APNDLBUFTIME 单次执行时，配置哪种用户类型的下行数据缓存时长？

| 选项 | 主参数 | 取值范围 | 适用场景 |
|---|---|---|---|
| **NB-IoT**（110601） | NBIOTUSER | 3~200s | NB-IoT eDRX（典型 APN=`apnnb`，NBIOTUSER=10） |
| **普通用户**（110641） | NORMALUSER | 3~15s | eMTC eDRX（典型 APN=`apnemtc`，NORMALUSER=12） |
| **RedCap-NR**（110661） | REDCAPNRUSER | 3~200s | 5G eDRX（典型 APN=`apn-test`，REDCAPNRUSER=10） |

> 3 个特性命中（110601/110641/110661）按用户类型各取其一参数；三时长互斥不可同时配，单次 SET 仅配一种。

## 约束

- **APN 必选**（critical，rule-0-00174）：APN 是必选参数 — 后果：缺省命令拒收
- **三时长互斥**（critical，rule-0-00174）：NORMALUSER/NBIOTUSER/REDCAPNRUSER 单次 SET 仅配其一 — 后果：三时长混传命令拒收
- **整机最多 10000 条**（warning，rule-0-00174）：整机 ApnDLBufTime 最多 10000 条 — 后果：超限 SET 失败
- **执行后立即生效**（warning，rule-0-00174）：本命令执行后立即生效 — 后果：误配置即刻影响 APN 级下行缓存
- **APN 级覆盖全局**（warning，rule-0-00174）：APN 级配置优先级 APN>全局（[0-00172](task/UDG/20.15.2/0-00172.md)） — 后果：APN 配置覆盖全局时长
- **APN 前置依赖**（warning，rule-0-00174）：APN 须先经 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 配置 — 后果：APN 不存在命令拒收
- **SMF 下发优先**（info，rule-0-00174）：若 SMF 下发 DL Buffering Duration 信元则取下发值 — 后果：本地设置被下发值覆盖
