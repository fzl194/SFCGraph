# 设置PPP配置（SET PPPCFG）

> 命令静态知识见 [SET PPPCFG](command/UDG/20.15.2/SET-PPPCFG.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**适用 NF：PGW-U、UPF**。

## 配置方法

SET PPPCFG 设置系统侧 PPP 协商的**全局唯一**参数集——HOSTNAME（本端主机名）、MRU（最大接收单元）、TIMEOUT（协商超时）。核心配置空间有 3 个维度：**本端身份**（HOSTNAME）、**接收单元上限**（MRU）、**协商超时**（TIMEOUT）。1 个特性命中（GWFD-020412 L2TP VPN）配 HOSTNAME="UPF" + MRU=1500 + TIMEOUT=3。

### 配置维度 1：本端身份（参数 HOSTNAME，可选）

| 取值/范围 | 作用 |
|---|---|
| **字符串 1~30**（020412 范式 `UPF`） | PPP 协商时本端主机名 |
| **省略** | 使用系统默认主机名 |

> 主机名标识 PPP 协商时的本端身份；不同 APN 间共享。

### 配置维度 2：接收单元上限（参数 MRU，可选）

| 取值/范围 | 作用 |
|---|---|
| **64~1500 整数** | PPP 协商时允许的最大接收单元（020412 范式 `1500`） |
| **省略** | 使用系统默认 MRU |

> 仅设置上限；PPP 协商的 LCP MRU 与系统侧 MRU 取**小者**。

### 配置维度 3：协商超时（参数 TIMEOUT，可选）

| 取值/范围 | 作用 |
|---|---|
| **整数（秒）** | 系统未收到用户 PPP 协商报文时重发前的等待时间（020412 范式 `3`） |
| **省略** | 使用系统默认 TIMEOUT |

**典型脚本（020412 L2TP VPN，本端主机名=UPF + MRU=1500 + TIMEOUT=3）**：
```
SET PPPCFG:HOSTNAME="UPF",MRU=1500,TIMEOUT=3;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：设 PPP 协商参数"——前置 [0-00131](task/UDG/20.15.2/0-00131.md) SET GLOBALL2TP；本地方式下与 [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR + [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP + [0-00135](task/UDG/20.15.2/0-00135.md) ADD L2TPCLIENTIP + [0-00139](task/UDG/20.15.2/0-00139.md) SET APNPPPACCESS 配合。

## 决策点：PPP 协商参数全局设置（DP 0-00138）

（本命令无独立决策分支，仅有 3 个参数 HOSTNAME/MRU/TIMEOUT 各按需取值。）

| 选项 | HOSTNAME | MRU | TIMEOUT | 适用场景 |
|---|---|---|---|---|
| **典型配置**（020412 范式） | "UPF" | 1500 | 3 | L2TP 用户 PPP 协商 |
| **省略走系统默认** | 省略 | 省略 | 省略 | 使用系统默认参数 |

> 1 个特性命中（020412）；三个参数均为可选，可单独省略。

## 约束

- **全局唯一实例**（critical，rule-0-00138）：命令最大记录数 1；执行后立即生效 — 后果：超规格（多实例）新建失败；非立即生效会导致 PPP 协商参数未刷新
- **MRU 范围**（warning，rule-0-00138）：MRU 取值 64~1500 — 后果：超范围命令拒收
- **HOSTNAME 范围**（info，rule-0-00138）：HOSTNAME 1~30 字符 — 后果：超长命令拒收
- **适用 NF**（info，rule-0-00138）：本命令仅适用 PGW-U、UPF — 后果：误在 SGW-U 上执行命令拒收
- **存在系统初始记录**（info，rule-0-00138）：系统初始记录走默认参数；不执行时按默认值生效 — 后果：未配则按系统默认
- **与 SET APNPPPACCESS 配合**（warning，rule-0-00138）：系统侧 PPP 参数（SET PPPCFG）与 APN 侧 PPP 鉴权（SET APNPPPACCESS）协同决定 PPP 接入行为 — 后果：任一未配置或与 MRU 不一致时 PPP 协商可能失败
- **执行后立即生效**（warning，rule-0-00138）：本命令执行后立即生效 — 后果：误配即刻影响后续 PPP 协商
