# 创建IPsec微服务VPN与隧道接口（ADD L3VPNINSTIPSEC）

> 命令静态知识见 [ADD L3VPNINSTIPSEC](command/UDG/20.15.2/ADD-L3VPNINSTIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是 IPsec 微服务侧"VPN + 隧道接口"双配的入口——5 条命令依序下发：建 VPN 实例 → 加地址族 → 建隧道接口 → 接口绑 VPN → 接口配 IP。**双配铁律**：IPsec 微服务侧的 VRFNAME/IFNAME/IFIPADDR/AFTYPE 必须与 VNRS 侧（[0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST / [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE / [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS）**一对一一致**，否则 IPsec 隧道不通。

## 配置方法（配置方法字典）

ADD L3VPNINSTIPSEC 自身仅 1 参必选（VRFNAME），但**完整生效依赖 IPsec 微服务侧 5 条命令聚合**。配置空间有 4 个维度：**VPN 实例名**（VRFNAME 与 VNRS 侧一致）、**地址族**（AFTYPE ipv4uni/ipv6uni）、**隧道接口名**（IFNAME 与 VNRS 侧一致）、**接口 IP**（IFIPADDR 与 VNRS 侧一致）。N 个特性命中（IPFD-015004 8 子场景：普通 IPv4/IPv6、GRE over IPsec、主备、OSPF over IPsec、多 Sequence、指定本端接口）共享同一配置。

### 配置维度 1：VPN 实例名（参数 VRFNAME，必选，与 VNRS 侧一致）

| 取值形式 | 范围 | 来源 |
|---|---|---|
| **字符串 1~31** | 不允许 `_public_` / `__mpp_vpn_inner__` | 本端规划，与 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST:VRFNAME 一致 |

> **保留实例名铁律**：VRFNAME 不能用 `_public_`（默认公网 VPN）/ `__mpp_vpn_inner__`（系统内部 VPN）——保留实例命令拒收。

### 配置维度 2：地址族（参数 AFTYPE，必选）

| 取值 | 适用场景 | 说明 |
|---|---|---|
| **ipv4uni** | IPv4 单播（主场景） | IPFD-015004 10002/10004/10006/10007 范式默认 |
| **ipv6uni** | IPv6 单播 | IPFD-015004 IPv6 IPsec 场景显式配 |

### 配置维度 3：隧道接口名（参数 IFNAME，必选，引用上游）

| 取值形式 | 来源 | 双配对应 |
|---|---|---|
| **字符串（隧道接口名）** | 引用 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE:IFNAME | 须与 VNRS 侧接口名一致 |

### 配置维度 4：接口 IP（参数 IFIPADDR，必选，引用上游）

| 取值形式 | 来源 | 双配对应 |
|---|---|---|
| **IPv4/IPv6 地址** | 引用 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS:IFIPADDR | 须与 VNRS 侧接口 IP 一致 |

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，双配 vrf1 + Tunnel10 + 10.102.101.20/32）**：
```
ADD L3VPNINSTIPSEC:VRFNAME="vrf1";
ADD VPNINSTAFIPSEC:VRFNAME="vrf1", AFTYPE=ipv4uni;
ADD INTERFACEIPSEC:IFNAME="Tunnel10";
ADD IPBINDVPNIPSEC:IFNAME="Tunnel10", VRFNAME="vrf1";
ADD IFIPV4ADDRESSIPSEC:IFNAME="Tunnel10", IFIPADDR="10.102.101.20", IFIPMASK="255.255.255.255";
```

> 命令 notes：**最大记录数 1001**；**执行后立即生效**；**双配铁律**：VRFNAME/IFNAME/IFIPADDR/AFTYPE 与 VNRS 侧 [0-00071](task/UDG/20.15.2/0-00071.md) / [0-00056](task/UDG/20.15.2/0-00056.md) / [0-00058](task/UDG/20.15.2/0-00058.md) 一对一一致。

**步骤位置**：在所属 feature 编排中位于"步骤 N：IPsec 微服务侧双配 VPN+隧道接口"——前置 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST / [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE / [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS（VNRS 侧）；下游 IPsec 协商链：[0-00211](task/UDG/20.15.2/0-00211.md) ACL → [0-00212](task/UDG/20.15.2/0-00212.md) Proposal → [0-00213](task/UDG/20.15.2/0-00213.md) IKE Peer → [0-00215](task/UDG/20.15.2/0-00215.md) IPsec Policy。

## 决策点

（无。本命令在所有 IPsec 子场景下用法一致——VRFNAME/IFNAME/IFIPADDR/AFTYPE 与 VNRS 侧一致，仅 ipv4uni/ipv6uni 二选一，无特性分叉。）

## 约束

- **最大记录数 1001**（warning，rule-0-00210）：系统最大记录数 1001 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00210）：命令执行后立即生效 — 后果：误操作即刻生效
- **VRFNAME 保留名约束**（critical，rule-0-00210）：VRFNAME 不能为 `_public_` / `__mpp_vpn_inner__`（保留实例） — 后果：用保留名命令拒收
- **VRFNAME 长度**（info，rule-0-00210）：字符串 1~31 — 后果：超长命令拒收
- **双配铁律：VRFNAME 与 VNRS 侧一致**（critical，rule-0-00210）：VRFNAME 必须与 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST:VRFNAME 一致 — 后果：VRFNAME 不一致 IPsec 隧道不通
- **双配铁律：IFNAME 与 VNRS 侧一致**（critical，rule-0-00210）：IFNAME 必须与 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE:IFNAME 一致 — 后果：IFNAME 不一致 IPsec 隧道不通
- **双配铁律：IFIPADDR 与 VNRS 侧一致**（critical，rule-0-00210）：IFIPADDR 必须与 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS:IFIPADDR 一致 — 后果：IFIPADDR 不一致 IPsec 隧道不通
- **双配铁律：AFTYPE 一致**（critical，rule-0-00210）：AFTYPE（ipv4uni/ipv6uni）必须与 VNRS 侧一致 — 后果：AFTYPE 不一致 IPsec 隧道不通
- **AFTYPE 主场景固定**（info，rule-0-00210）：IPv4 单播场景 AFTYPE=ipv4uni；IPv6 单播场景改 ipv6uni — 后果：AFTYPE 错配协议不匹配
