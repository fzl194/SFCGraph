# 创建IPsec微服务VPN与隧道接口(VNRS+IPSEC双配)（ADD L3VPNINSTIPSEC）

> 命令静态知识见 [ADD L3VPNINSTIPSEC](command/UDG/20.15.2/ADD-L3VPNINSTIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本 atom 是**聚合 atom**：覆盖 IPsec 微服务侧 VPN+隧道接口创建全链——`ADD L3VPNINSTIPSEC`（建 VPN 实例）+ `ADD VPNINSTAFIPSEC`（地址族）+ `ADD INTERFACEIPSEC`（隧道接口）+ `ADD IPBINDVPNIPSEC`（接口绑 VPN）+ `ADD IFIPV4ADDRESSIPSEC`（接口配 IP）。遵循**双配原则**：接口名/接口 IP/AFTYPE/VRFNAME 须与 VNRS 侧（0-00056/0-00058/0-00071）一对一一致，否则业务不通。8 个 IPsec 子场景（普通 IPv4/IPv6、GRE over IPsec、主备、OSPF over IPsec、多 Sequence、指定本端接口）共享同一配置。

> 聚合债：同 0-00147（SET BGP 类型）——5 命令合 1 atom。

**配置范式（IPsec 子场景共享，IPFD-015004 数据规划表归纳）**

| 步骤 | 命令 | 关键参数 | 来源/双配对应 | 示例行 |
|---|---|---|---|---|
| 1. 建 VPN 实例 | ADD L3VPNINSTIPSEC | VRFNAME | 引用 [ADD L3VPNINST](task/UDG/20.15.2/0-00071.md) | `ADD L3VPNINSTIPSEC:VRFNAME="vrf1";` |
| 2. 地址族 | ADD VPNINSTAFIPSEC | AFTYPE=ipv4uni（固定） | 本端规划，IPv6 场景改 ipv6uni | `ADD VPNINSTAFIPSEC:VRFNAME="vrf1", AFTYPE=ipv4uni;` |
| 3. 隧道接口 | ADD INTERFACEIPSEC | IFNAME | 引用 [ADD INTERFACE](task/UDG/20.15.2/0-00056.md) | （见命令 wiki） |
| 4. 接口绑 VPN | ADD IPBINDVPNIPSEC | IFNAME + VRFNAME | 双配一致 | （见命令 wiki） |
| 5. 接口配 IP | ADD IFIPV4ADDRESSIPSEC | IFIPADDR | 引用 [ADD IFIPV4ADDRESS](task/UDG/20.15.2/0-00058.md) | （见命令 wiki） |

**参数取值约定**

| 参数 | 取值 | 来源 | 说明 |
|---|---|---|---|
| VRFNAME | 字符串 1~31 | 引用 [ADD L3VPNINST](task/UDG/20.15.2/0-00071.md):VRFNAME | VPN 实例名，须与 VNRS 侧一致 |
| AFTYPE | ipv4uni（固定）/ ipv6uni | 本端规划 | 主场景 IPv4 单播；**IPv6 场景应改 ipv6uni 由 feature task 投影** |
| IFNAME | 接口名 | 引用 [ADD INTERFACE](task/UDG/20.15.2/0-00056.md):IFNAME | 隧道接口，须与 VNRS 侧一致 |
| IFIPADDR | IPv4 | 引用 [ADD IFIPV4ADDRESS](task/UDG/20.15.2/0-00058.md):IFIPADDR | 接口 IP，须与 VNRS 侧一致 |

## 约束

- **rule-0-00335**（critical）最多 1001 条记录；命令执行后立即生效；**VRFNAME 不能为 `_public_` / `__mpp_vpn_inner__`**（保留实例）；IPsec 微服务侧全链聚合须遵循双配原则：接口名/接口 IP/AFTYPE/VRFNAME 与 VNRS 侧（0-00071/0-00056/0-00058）一对一一致——后果：VRFNAME 用保留名被拒；VNRS 双配不一致导致 IPsec 隧道不通
