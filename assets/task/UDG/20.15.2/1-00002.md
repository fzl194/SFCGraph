# BWM 业务与控制器配置

> BWM 配置的通用骨架阶段（所有场景共享）。承接 [1-00001](task/UDG/20.15.2/1-00001.md) 前置，输出 BWMSERVICE + BWMCONTROLLER 供下游规则引用。归属特性 [2-00001](task/UDG/20.15.2/2-00001.md)。

## 配置方法

**BWM 控制器（ADD BWMCONTROLLER）——所有场景必配**：定义 CAR/Shaping 动作，按上/下行各配一个，名称带方向后缀。

| 参数 | 取值样例 | 说明 |
|---|---|---|
| BWMCNAME | `bc-up` / `bc-down` | 本端规划，带方向语义 |
| CTRLTYPE | `CAR` | 用户组级/整机级只能 CAR；用户级可选 SHAPING |
| CIR / PIR | 2000000 / 4000000 (kbps) | CIR=0.5×PIR |
| CBS / PBS | 320000000 / 800000000 (字节) | CBS≥CIR×1.5×1000/8 |
| GREEN/YELLOW/RED | PASS/PASS/DISCARD | 三色策略 |
| PRIORITYISAWARE | DISABLE | ENABLE 时按 BWMRULEPRI 区分业务优先级 |

**BWM 业务（ADD BWMSERVICE）——按场景**：APN/DNN 用户组级场景**不配**（GROUP_DEFAULT 不区分业务）；其余场景必配。

| 场景族 | BWMSERVICETYPE | 关键参数 |
|---|---|---|
| TOS 族（用户/组 TOS、时间段、分级）| TOS | TOSCFGTYPE=CLASS, TOSCLASSTYPE=BE/AF1/CS7… |
| NONTOS 族（HTTP 限速、切片）| NONTOS | NONTOSSRVTYPE=PROTOCOL, PROTOCOLNAME=http/bt |
| PCC 预定义规则 | NONTOS | NONTOSSRVTYPE=CATEGORY_PROP, CATEPROPNAME（须==RULE.POLICYNAME）|

**典型脚本（APN/DNN 用户组级，无 BWMSERVICE）**：
```
ADD BWMCONTROLLER:BWMCNAME="bc-up",CTRLTYPE=CAR,CIR=2000000,CBS=320000000,PIR=4000000,PBS=800000000,GREEN=PASS,YELLOW=PASS,RED=DISCARD,PRIORITYISAWARE=DISABLE;
ADD BWMCONTROLLER:BWMCNAME="bc-down",CTRLTYPE=CAR,CIR=2000000,CBS=320000000,PIR=4000000,PBS=800000000,GREEN=PASS,YELLOW=PASS,RED=DISCARD,PRIORITYISAWARE=DISABLE;
```

**步骤位置**：承接 [1-00001](task/UDG/20.15.2/1-00001.md)；下游 [1-00003](task/UDG/20.15.2/1-00003.md)（规则引用 BWMCNAME/BWMSERVICENAME）与多命令特化链。

## 场景差异（各场景对本步骤的参数增量）

- **分级带宽**：BWMCONTROLLER 加 `SRVLEVELSPEC`（业务等级规格，如 =3，最大 15，定分级数）；分级通常单控制器上/下行共用（如 `bc_test`）。分级 atom [0-00108](task/UDG/20.15.2/0-00108.md)（BCSRVLEVELPLY）紧随其后配各 ServiceLevel 的 `CIRRATE`+`PIRRATE`（CIRRATE 之和≤100%）。
- **三级控制**：按用户级/用户组级/整机级配三组控制器，带宽递增（如 30/300/3000），名称带级语义（`bc-subs-*`/`bc-group-*`/`bc-global-*`）。
- **切片**：BWMSERVICE 可选 Protocol/NONTOS 识别。
- **APN/DNN 用户组级**：不配 BWMSERVICE（GROUP_DEFAULT 不区分业务）；可选 `COLORISAWARE=ENABLE` 优先保组内绿色流量。

## 决策点

**DP（阶段级）：控制器控制类型** —— `CTRLTYPE`：CAR（所有层级，默认）/ SHAPING（仅用户级 SUBSCRIBER_SPECIFIC）。
**DP（联动特性级场景 DP）：业务识别方式** —— `BWMSERVICETYPE` 由 [2-00001](task/UDG/20.15.2/2-00001.md) 场景 DP 决定（见上表三族）。

## 约束

- CAR 参数推荐：CIR=0.5×PIR，CBS/PBS 保证 1.5s 队列缓存。
- APN/DNN 用户组级：仅 CAR + 不配 BWMSERVICE。
- PCC 场景：CATEPROPNAME 须 == RULE.POLICYNAME（命名铁律）。
