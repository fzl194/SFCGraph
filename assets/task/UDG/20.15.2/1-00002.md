# BWM 业务与控制器配置

> BWM 配置的通用骨架阶段（所有场景共享）。承接 [1-00001](task/UDG/20.15.2/1-00001.md) 前置，输出 BWMSERVICE + BWMCONTROLLER 供下游规则引用。归属特性 [2-00001](task/UDG/20.15.2/2-00001.md)。

## 配置方法

BWM 控制器（所有场景必配，按上/下行各配一个，名称带方向后缀）→ BWM 业务（按场景，APN/DNN 用户组级不配）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| BWM 控制器（所有场景必配，按上/下行各一） | `ADD BWMCONTROLLER` | BWMCNAME=`bc-up`/`bc-down`（带方向语义）；CTRLTYPE=CAR（用户组级/整机级只能 CAR；用户级可选 SHAPING）；CIR/PIR=2000000/4000000(kbps)（CIR=0.5×PIR）；CBS/PBS=320000000/800000000(字节)（CBS≥CIR×1.5×1000/8）；GREEN/YELLOW/RED=PASS/PASS/DISCARD（三色策略）；PRIORITYISAWARE=DISABLE（ENABLE 时按 BWMRULEPRI 区分业务优先级） |
| BWM 业务—TOS 族（用户/组 TOS、时间段、分级） | `ADD BWMSERVICE` | BWMSERVICETYPE=TOS；TOSCFGTYPE=CLASS；TOSCLASSTYPE=BE/AF1/CS7… |
| BWM 业务—NONTOS 族（HTTP 限速、切片） | `ADD BWMSERVICE` | BWMSERVICETYPE=NONTOS；NONTOSSRVTYPE=PROTOCOL；PROTOCOLNAME=http/bt |
| BWM 业务—PCC 预定义规则 | `ADD BWMSERVICE` | BWMSERVICETYPE=NONTOS；NONTOSSRVTYPE=CATEGORY_PROP；CATEPROPNAME（须==RULE.POLICYNAME） |
| BWM 业务—APN/DNN 用户组级 | 不配 | GROUP_DEFAULT 不区分业务；可选 COLORISAWARE=ENABLE 优先保组内绿色流量 |

**典型脚本（APN/DNN 用户组级，无 BWMSERVICE）**：
```
ADD BWMCONTROLLER:BWMCNAME="bc-up",CTRLTYPE=CAR,CIR=2000000,CBS=320000000,PIR=4000000,PBS=800000000,GREEN=PASS,YELLOW=PASS,RED=DISCARD,PRIORITYISAWARE=DISABLE;
ADD BWMCONTROLLER:BWMCNAME="bc-down",CTRLTYPE=CAR,CIR=2000000,CBS=320000000,PIR=4000000,PBS=800000000,GREEN=PASS,YELLOW=PASS,RED=DISCARD,PRIORITYISAWARE=DISABLE;
```

**步骤位置**：承接 [1-00001](task/UDG/20.15.2/1-00001.md)；下游 [1-00003](task/UDG/20.15.2/1-00003.md)（规则引用 BWMCNAME/BWMSERVICENAME）与多命令特化链。

## 场景差异（各场景对本步骤的参数增量）

- **分级带宽**：BWMCONTROLLER 加 `SRVLEVELSPEC`（业务等级规格，如 =3，最大 15，定分级数）；分级通常单控制器上/下行共用（如 `bc_test`）。分级 atom [0-00108](task/UDG/20.15.2/0-00108.md)（BCSRVLEVELPLY）紧随其后配各 ServiceLevel 的 `CIRRATE`+`PIRRATE`（CIRRATE 之和≤100%）。
- **三级控制**：按用户级/用户组级/整机级配三组控制器，带宽递增（如 30/300/3000），名称带级语义（`bc-subs-*`/`bc-group-*`/`bc-global-*`）。
- **切片**：BWMSERVICE 可选 Protocol/NONTOS 识别。
- **APN/DNN 用户组级**：不配 BWMSERVICE（GROUP_DEFAULT 不区分业务）；可选 `COLORISAWARE=ENABLE` 优先保组内绿色流量。
- **Shaping 族（[2-00023](task/UDG/20.15.2/2-00023.md) 基于业务Shaping / [2-00024](task/UDG/20.15.2/2-00024.md) 智能Shaping组级 / [2-00025](task/UDG/20.15.2/2-00025.md) 码率差异化）**：BWMCONTROLLER `CTRLTYPE=SHAPING`，参数集与 CAR **正交不可混配**——SHAPING 用 `RATE/QUEDEPTH/MAXFDNUM`（非 CAR 的 CIR/CBS/PIR/PBS/三色）。智能 Shaping（2-00024）增 `WORKMODE=AUTO`（或 MANUAL）+ `USERFAIREN=ENABLE`（组内公平）+ `MAXPKTLOSTRATE/PKTLOSTRATEDTL/ASSUREMODE/SRVLEVELSPEC`，配 [0-00108](task/UDG/20.15.2/0-00108.md) BCSRVLEVELPLY 各 SERVICELEVEL 的 SHAPRATE。码率差异化（2-00025）BWMSERVICE 用 `BWMSERVICETYPE=NONTOS + NONTOSSRVTYPE=PROTOCOL + OSTYPE=IOS/ANDROID`（按终端系统建多条 Service）+ BWMCONTROLLER `CTRLTYPE=CAR`（上下行双控制器 bc-up/bc-down，CIR/CBS/PIR/PBS/三色）+ SET APNOSLELBWMSW（[0-00107](task/UDG/20.15.2/0-00107.md)）开 OS 级 BWM。

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| 控制类型 CTRLTYPE=CAR（默认，所有层级） | 用户组级/整机级只能 CAR；用户级也可选 |
| 控制类型 CTRLTYPE=SHAPING | 仅用户级 SUBSCRIBER_SPECIFIC 可用 |
| 业务识别 BWMSERVICETYPE=TOS（联动特性级场景 DP） | TOS 族（用户/组 TOS、时间段、分级）→ 配 BWMSERVICE，TOSCFGTYPE=CLASS + TOSCLASSTYPE |
| 业务识别 BWMSERVICETYPE=NONTOS—PROTOCOL（联动特性级场景 DP） | NONTOS 族（HTTP 限速、切片）→ NONTOSSRVTYPE=PROTOCOL + PROTOCOLNAME |
| 业务识别 BWMSERVICETYPE=NONTOS—CATEGORY_PROP（联动特性级场景 DP） | PCC 预定义规则 → NONTOSSRVTYPE=CATEGORY_PROP + CATEPROPNAME（须==RULE.POLICYNAME） |
| 业务识别—不配 BWMSERVICE（联动特性级场景 DP） | APN/DNN 用户组级（GROUP_DEFAULT）→ 仅控制器 |

> `BWMSERVICETYPE` 由 [2-00001](task/UDG/20.15.2/2-00001.md) 场景 DP 决定。

## 约束

- **CAR 参数推荐**（warning）：CIR=0.5×PIR，CBS/PBS 保证 1.5s 队列缓存 — 否则突发流量可能被误丢弃或缓存不足。
- **APN/DNN 用户组级限定**（warning）：仅 CAR + 不配 BWMSERVICE — 违反则与 GROUP_DEFAULT 语义冲突。
- **PCC 命名铁律**（critical）：CATEPROPNAME 须 == RULE.POLICYNAME — 违反则 PCC 规则链断开，限速不生效。
