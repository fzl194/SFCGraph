# 地址分配体系（地址池三级+映射）

> 归属 [2-00034](task/UDG/20.15.2/2-00034.md) 用户面地址分配（GWFD-010105）。本 compound 配置地址分配**地址池体系层**——三级地址池（SECTION→POOL→POOLGROUP）+ 冲突地址标识 + 地址池组映射关系。被引用于 {2-00034}。

## 配置方法

本 compound 由 6 条命令组成（多命令可复用模块，建 compound 承载）：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 创建地址池（LOCAL 类型，绑定 VPN） | [ADD POOL](task/UDG/20.15.2/0-00043.md) | `POOLNAME` / `POOLTYPE=LOCAL` / `HASVPN` / `VPNINSTANCE` |
| 2. 添加地址池 IP 地址段 | [ADD SECTION](task/UDG/20.15.2/0-00044.md) | `POOLNAME` / `SECTIONNUM` / `IPVERSION=IPV4/IPV6` / `V4STARTIP` / `V4ENDIP` |
| 3. 创建地址池组 | [ADD POOLGROUP](task/UDG/20.15.2/0-00042.md) | `POOLGRPNAME` / `IPV4ALLOCPRIALG` / `IPV6ALLOCPRIALG` |
| 4. 绑定地址池到地址池组 | [ADD POOLBINDGROUP](task/UDG/20.15.2/0-00045.md) | `POOLGROUPNAME` / `POOLNAME` / `PRIORITY` |
| 5. 添加冲突 IP 地址（可选，防止地址冲突） | [ADD CONFLICTIP](task/UDG/20.15.2/0-00301.md) | `POOLNAME` / `IPADDRESS` |
| 6. 添加地址池组与 APN 映射关系 | [ADD POOLGRPMAP](task/UDG/20.15.2/0-00047.md) | `MAPPINGNAME` / `APN` / `SMF`（条件）/ `POOLGROUPNAME` |

**典型脚本（010105 场景一：基于 APN/DNN 单级优先级）**：
```
ADD POOL:POOLNAME="testpool1",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="testpool1",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.10.1.1",V4ENDIP="10.10.1.10";
ADD POOLGROUP: POOLGRPNAME="poolgroup1";
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool1";
ADD CONFLICTIP:POOLNAME="testpool1",IPADDRESS="10.1.1.1";
ADD POOLGRPMAP: MAPPINGNAME="mapping1", APN="apn-test1", POOLGROUPNAME="poolgroup1";
```

**典型脚本（010105 场景三：基于 SMF+APN 双池两级映射）**：
```
// 池 1（APN+SMF 映射目标）
ADD POOL:POOLNAME="testpool1",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="testpool1",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.10.1.1",V4ENDIP="10.10.1.10";
ADD POOLGROUP: POOLGRPNAME="poolgroup1";

// 池 2（SMF 单维度回退）
ADD POOL:POOLNAME="testpool2",POOLTYPE=LOCAL,HASVPN=ENABLE,VPNINSTANCE="vpn1";
ADD SECTION:POOLNAME="testpool2",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.10.1.21",V4ENDIP="10.10.1.30";
ADD POOLGROUP: POOLGRPNAME="poolgroup2", IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE;
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup2", POOLNAME="testpool2", PRIORITY=10;

// 映射
ADD CONFLICTIP:POOLNAME="testpool1",IPADDRESS="10.1.1.1";
ADD POOLGRPMAP: MAPPINGNAME="mapping1", APN="apn-test1", SMF="smfnode1", POOLGROUPNAME="poolgroup1";
ADD POOLGRPMAP: MAPPINGNAME="mapping2", SMF="smfnode1", POOLGROUPNAME="poolgroup2";
```

## 场景差异

| 场景 | 池数量 | 池组数量 | POOLGRPMAP 数量 | 关键参数 |
|---|---|---|---|---|
| 基于 APN/DNN | 1 | 1 | 1（APN=） | — |
| 基于 SMF | 1 | 1 | 1（SMF=） | 需配 [0-00303](task/UDG/20.15.2/0-00303.md) CPNODEID |
| 基于 SMF+APN | 2 | 2 | 2（mapping1=APN+SMF / mapping2=SMF） | `IPV4ALLOCPRIALG=ENABLE`、`PRIORITY=10` |
| 基于 RADIUS 下发地址池名 | 1 | 1 | 1（APN=） | 需配 [0-00046](task/UDG/20.15.2/0-00046.md) IGNORE*=DISABLE |
| 基于位置（参见 GWFD-020421） | 1 | 1 | 1（LOCATION=） | 需配 [0-00310](task/UDG/20.15.2/0-00310.md) LACGROUP |

## 约束

- **POOLTYPE=LOCAL 必配**（critical，rule-0-00494）：GWFD-010105 用户面地址分配必须配 LOCAL 类型地址池（由 UDG 自维护地址段） — 后果：EXTERNAL 类型地址由 RADIUS/BRAS/DHCP 分配，不属本特性范围
- **VPN 三者一致**（critical，rule-0-00494）：地址池绑定的 VPN / APN 绑定的 VPN / Gi/SGi/N6 外联口绑定的 VPN 三者必须一致 — 后果：VPN 不一致致地址分配或路由异常
- **每 APN 绑池数 ≤16**（warning，rule-0-00494）：每 APN/DNN 支持绑定的地址池最多 16 个 — 后果：超规格新增失败
- **每 POOL 绑段数 ≤64**（warning，rule-0-00494）：每本地地址池支持绑定的地址段最多 64 个 — 后果：超规格新增失败
- **CONFLICTIP 仅 IPv4**（warning，rule-0-00494）：本命令配 IPv4；IPv6 前缀冲突用 [0-00302](task/UDG/20.15.2/0-00302.md) ADD CONFLICTIPV6 — 后果：IPv6 冲突场景配错命令
- **CONFLICTIP 生效时机**（info，rule-0-00494）：若某 IPv4 地址已分配给用户，此时配置其为冲突地址命令仍可执行成功，但配置在下次分配该地址时生效 — 后果：已在线用户不受影响
