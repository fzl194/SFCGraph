# 配置HTTP智能重定向动作（ADD SMARTHTTPREDIR）

> 命令静态知识见 [ADD SMARTHTTPREDIR](command/UDG/20.15.2/ADD-SMARTHTTPREDIR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD SMARTHTTPREDIR 是 HTTP 智能重定向的"动作策略定义"命令：当 HTTP 服务器返回的错误码（如 ≥400）触发 URL 纠错时，将用户访问的 HTTP 页面重定向到第三方 server。本命令的核心配置空间有 7 个维度：**动作名 SMTHTTPREDINAME**、**目标 server URL**、**错误码携带 ERRCODEFLAG/ERRCODENAME**、**用户定义信息 USERDEFINENAME/USERDEFINEVALUE**、**扩展过滤器组合 EXTFLTTYPE1-5/EXTFLTNAME1-5**、**携带信息 APPENDINFONAME**、**绑定错误码 BINDERRCODENAME**。

### 配置维度 1：动作名（参数 SMTHTTPREDINAME，必选，本端规划）

字符串 1~31（不支持空格、不区分大小写）；产出的 SMTHTTPREDINAME 对象被 ADD RULE 的 POLICYNAME 经 POLICYTYPE=SMARTREDIRECT 引用。

### 配置维度 2：目标 server URL（参数 SERVERURL，必选）

字符串；HTTP 智能重定向后消息中填写的 server URL；SERVERURL 特殊字符须转义（?→%3f、空格→%20、,→%2c 等，详见命令 md）。

### 配置维度 3：错误码携带（参数 ERRCODEFLAG/ERRCODENAME，可选）

| 参数 | 取值 | 含义 |
|---|---|---|
| **ERRCODEFLAG** | ENABLE | 在 HTTP 智能重定向的目标 URL 中携带 Error Code |
| | DISABLE（默认） | 不携带 Error Code |
| **ERRCODENAME** | 字符串 1~31 | ERRCODEFLAG=ENABLE 时可选，引用 [ADD ERRORCODE](task/UDG/20.15.2/0-00089.md) 产出的 ERRORCODENAME |

### 配置维度 4：用户定义信息（参数 USERDEFINENAME/USERDEFINEVALUE，可选）

| 参数 | 含义 |
|---|---|
| **USERDEFINENAME** | 重定向的目标 URL 中携带的用户自定义信息的前缀名称 |
| **USERDEFINEVALUE** | 重定向的目标 URL 中携带的用户自定义值 |

### 配置维度 5：扩展过滤器组合（参数 EXTFLTTYPE1-5/EXTFLTNAME1-5，可选，最多 5 组）

| 参数 | 取值 | 配套参数 |
|---|---|---|
| **EXTFLTTYPE1~5** | AND / OR / NOT（枚举） | EXTFLTNAME1~5（条件必选） |
| **EXTFLTNAME1~5** | 字符串 1~31 | 引用 [ADD EXTENDEDFILTER](task/UDG/20.15.2/0-00022.md) 产出的扩展过滤器名 |

### 配置维度 6：携带信息名称（参数 APPENDINFONAME，可选）

字符串 1~31（不区分大小写、不支持空格）；引用 [ADD REDIRAPPENDINFO](task/UDG/20.15.2/0-00023.md) 产出的 APPENDINFONAME；典型携带信息：REQURLFLAG+IMSIFLAG（原始请求 URL+IMSI）。

### 配置维度 7：绑定错误码（参数 BINDERRCODENAME，可选）

字符串 1~31（不区分大小写、不支持空格）；引用 [ADD ERRORCODE](task/UDG/20.15.2/0-00089.md) 产出的 ERRORCODENAME；典型场景 httperrorcode（HTTP ≥400 GT）。

> **典型范式（HTTP智能重定向，多过滤器+携带信息+错误码）**：
> ```
> ADD SMARTHTTPREDIR:SMTHTTPREDINAME="http-redirect_test",SERVERURL="www.example.com",EXTFLTTYPE1=AND,EXTFLTNAME1="filter_http-redirect1",EXTFLTTYPE2=AND,EXTFLTNAME2="filter_http-redirect2",APPENDINFONAME="testredirappendinfo",BINDERRCODENAME="httperrorcode";
> ```

**步骤位置**：在所属 feature 编排中通常为"步骤 3：配置 HTTP 智能重定向动作"——ADD EXTENDEDFILTER + ADD ERRORCODE + ADD REDIRAPPENDINFO 之后、ADD RULE（POLICYTYPE=SMARTREDIRECT 引 SMTHTTPREDINAME）之前。

## 约束

- **整机最大记录数为 50**（warning，rule-0-00297）：明显小于 DNSOVERWRITING 2000/REDIRAPPENDINFO 1000 — 后果：超规格新增失败
- **EXTFLTNAME 引用须已配置**（critical，隐含）：EXTFLTNAME1/2/3/4/5 须先经 ADD EXTENDEDFILTER（[0-00022](task/UDG/20.15.2/0-00022.md)）配置 — 后果：引用不存在拒收
- **APPENDINFONAME 引用须已配置**（critical，隐含）：APPENDINFONAME 须先经 ADD REDIRAPPENDINFO（[0-00023](task/UDG/20.15.2/0-00023.md)）配置 — 后果：引用不存在拒收
- **BINDERRCODENAME 引用须已配置**（critical，隐含）：BINDERRCODENAME 须先经 ADD ERRORCODE（[0-00089](task/UDG/20.15.2/0-00089.md)）配置 — 后果：引用不存在拒收
- **SMTHTTPREDINAME 全局唯一**（critical，隐含）：动作名全局唯一 — 后果：重名拒收
- **SERVERURL 特殊字符须转义**（warning，隐含）：未转义拒收或解析错误 — 后果：URL 解析异常
- **立即生效**（info，隐含）：该命令执行后立即生效 — 后果：动作策略变更实时影响 HTTP 重定向行为
- **SMTHTTPREDINAME 须先于 RULE 配置**（critical，隐含）：被 RULE.POLICYTYPE=SMARTREDIRECT.POLICYNAME 引用，须先建 — 后果：未先建导致 RULE 引用失败
- **依赖 SA-Basic + SA-Web Browsing**（warning，隐含）：HTTP 智能重定向特性依赖 SA-Basic + SA-Web Browsing 业务感知 — 后果：缺 SA 依赖时 HTTP 业务识别失败，重定向无法触发
