# 配置HTTP智能重定向动作（ADD SMARTHTTPREDIR）

> 命令静态知识见 [ADD SMARTHTTPREDIR](command/UDG/20.15.2/ADD-SMARTHTTPREDIR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD SMARTHTTPREDIR 是 GWFD-110284 HTTP 智能重定向专属命令，按 HTTP 错误码触发重定向到第三方 server。主特性绑 2 组 AND 扩展过滤器（URL+USERAGENT 组合匹配），1~2 组为常规用法，命令真相的 EXTFLTTYPE3-5/EXTFLTNAME3-5/CFGDOMAINNAME 按 3+ 组过滤器场景补充。EXTFLTNAME1/2 引用 EXTENDEDFILTER，APPENDINFONAME 引用 REDIRAPPENDINFO，BINDERRCODENAME 引用 ERRORCODE，SMTHTTPREDINAME 产出对象被 RULE(POLICYTYPE=SMARTREDIRECT) 的 POLICYNAME 引用。

**主场景范式（从证据数据规划表归纳）**

| 字段 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| SMTHTTPREDINAME | http-redirect_test | 本端规划 | HTTP 智能重定向名，1~31 字符 |
| SERVERURL | www.example.com | 本端规划 | 服务器 URL，1~119 字符；含特殊字符须转义（`?`→`%3f`、空格→`%20`、`,`→`%2c`） |
| EXTFLTTYPE1 / EXTFLTNAME1 | AND / filter_http-redirect1 | 本端规划 / 已配置数据中获取 | 第 1 组过滤器，引用 0-00022 |
| EXTFLTTYPE2 / EXTFLTNAME2 | AND / filter_http-redirect2 | 本端规划 / 已配置数据中获取 | 第 2 组过滤器（条件必选，多组时配套） |
| APPENDINFONAME | testredirappendinfo | 已配置数据中获取 | 引用 0-00023 产出的 REDIRAPPENDINFO |
| BINDERRCODENAME | httperrorcode | 已配置数据中获取 | 引用 0-00089 产出的 ERRORCODE |

**脚本样例**
```
ADD SMARTHTTPREDIR:SMTHTTPREDINAME="http-redirect_test", SERVERURL="www.example.com", EXTFLTTYPE1=AND, EXTFLTNAME1="filter_http-redirect1", EXTFLTTYPE2=AND, EXTFLTNAME2="filter_http-redirect2", APPENDINFONAME="testredirappendinfo", BINDERRCODENAME="httperrorcode";
```

**配置链路**（GWFD-110284 步骤 4→5→6）：先 ADD EXTENDEDFILTER(0-00022) → ADD ERRORCODE(0-00089) → ADD REDIRAPPENDINFO(0-00023) → **ADD SMARTHTTPREDIR** → ADD RULE(POLICYTYPE=SMARTREDIRECT)。

## 约束

- **rule 0-00297**（requires/warning）系统最大记录数 50（明显小于 DNSOVERWRITING 2000/REDIRAPPENDINFO 1000）；立即生效；SERVERURL 特殊字符须转义（`?`→`%3f`、空格→`%20`、`,`→`%2c`）；最多 5 组扩展过滤器；EXTFLTTYPE1/EXTFLTNAME1 必选引用 0-00022，EXTFLTTYPE2/EXTFLTNAME2 条件必选；APPENDINFONAME 引用 0-00023；BINDERRCODENAME 引用 0-00089；SMTHTTPREDINAME 须先于 RULE(POLICYTYPE=SMARTREDIRECT) 配置 — 后果：上限超限 ADD 失败；URL 特殊字符未转义拒收或解析错误；引用对象不存在拒收；未先配 SMTHTTPREDINAME 导致 RULE 引用失败
