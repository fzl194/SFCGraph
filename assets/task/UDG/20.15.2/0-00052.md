# 配置N4接口GTP路径相关属性（SET UPN4UPATH）

> 命令静态知识见 [SET UPN4UPATH](command/UDG/20.15.2/SET-UPN4UPATH.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**仅作用于 N4/Sxb 接口**（非 N4/Sxb 接口由 [SET UPGTPPATH](task/UDG/20.15.2/0-00051.md) 配置）。

## 配置方法（配置方法字典）

SET UPN4UPATH 是**全局唯一**（最大记录数 1）的"N4/Sxb 接口 GTP 路径探测参数配置"：与 [SET UPGTPPATH](task/UDG/20.15.2/0-00051.md)（非 N4 接口）配对，按"接口类型"分离。命令核心配置空间有 4 个维度：**Echo 总开关**（N4ECHOSW 门控其余参数）、**心跳间隔**（N4ECHOINTERVAL）、**重发参数**（N4T3RESPONSE + N4N3REQUEST）、**去活策略**（DEACTIVEFLAG + N4ECHOTIME）。1 个特性命中（GWFD-010102 路径管理）覆盖 6 参数全量脚本 + 仅改 T3/N3 脚本。

### 配置维度 1：Echo 总开关（参数 N4ECHOSW）

| 参数 | 取值 | 作用 |
|---|---|---|
| **N4ECHOSW** | ENABLE | 开启 N4 接口 v1 版本数据路径 Echo 探测；**其余 5 参数（N4ECHOINTERVAL/DEACTIVEFLAG/N4T3RESPONSE/N4N3REQUEST/N4ECHOTIME）才生效** |
| | DISABLE | 关闭 N4 Echo 探测；其余 5 参数不生效 |

### 配置维度 2：心跳间隔（参数 N4ECHOINTERVAL，可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **N4ECHOINTERVAL** | 60~3600 秒 | 发送 N4 接口 GTP 心跳请求的间隔时间；**设置后在下一个心跳周期开始生效** |

> 基站数目较多时建议配较大 ECHOINTERVAL（防单位时间内 ECHO 过多致 CPU 上升影响基本业务）。

### 配置维度 3：重发参数（参数 N4T3RESPONSE/N4N3REQUEST，可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **N4T3RESPONSE** | 1~20 秒 | N4 接口 GTP 请求消息的重发时间间隔 |
| **N4N3REQUEST** | 1~6 次 | N4 接口 GTP 请求消息的最大尝试发送次数 |

### 配置维度 4：去活策略（参数 DEACTIVEFLAG/N4ECHOTIME）

| 参数 | 必选性 | 取值 | 作用 |
|---|---|---|---|
| **DEACTIVEFLAG** | 可选 | ENABLE / DISABLE | 路径断告警产生后是否去活该路径上已激活的上下文 |
| **N4ECHOTIME** | DEACTIVEFLAG=ENABLE 时条件必选 | 0~60 | N4 接口的 GTP-U 路径断告警后发送 echo 消息的次数；**0 表示路径断告警后不再发 Echo 消息**，但若 DEACTIVEFLAG=ENABLE 仍会去活上下文 |

> **告警-去活链路**：Echo 重发达 N4N3REQUEST 仍无响应 → 上报 ALM-81018 数据路径断 → 启动 N4ECHOINTERVAL 定时器 → 发送 N4ECHOTIME 次 Echo → 仍无响应且 DEACTIVEFLAG=ENABLE → 去活该路径上所有上下文。

**典型脚本（010102 全量配置，N4T3RESPONSE=3）**：
```
SET UPN4UPATH:N4ECHOSW=ENABLE,N4ECHOINTERVAL=60,N4T3RESPONSE=3,N4N3REQUEST=5,DEACTIVEFLAG=ENABLE,N4ECHOTIME=0;
```

**典型脚本（010102 仅改重发参数，N4T3RESPONSE=10）**：
```
SET UPN4UPATH:N4T3RESPONSE=10,N4N3REQUEST=3;
```

> 命令 notes：适用 NF：SGW-U、PGW-U、UPF；本命令属高危命令（配置 N4 重发间隔/最大次数不合理可能导致用户激活失败或资源残留）；执行后立即生效；**全局唯一最大记录数 1**；N4ECHOINTERVAL 在设置后下一个心跳周期开始生效；**仅作用于 N4/Sxb 接口**。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：N4 接口 GTP 路径探测参数"——前置无；与 [0-00051](task/UDG/20.15.2/0-00051.md)（SET UPGTPPATH，非 N4 接口）配对，按接口类型分别配置。

## 决策点：N4 接口探测开关与去活策略（DP 0-00052）

问：SET UPN4UPATH 选哪个开关 + 去活策略组合？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 全量开启 + 去活 | N4ECHOSW=ENABLE + N4ECHOINTERVAL=60 + N4T3RESPONSE=3 + N4N3REQUEST=5 + DEACTIVEFLAG=ENABLE + N4ECHOTIME=0 | N4 接口路径管理（010102 范式：全量开启 + 路径断去活） |
| 仅改重发参数 | N4T3RESPONSE=10 + N4N3REQUEST=3 | 修改 Echo 消息的重发数据（010102 修改范式） |
| 全关 | N4ECHOSW=DISABLE | 关闭 N4 接口 Echo 探测（命令支持，010102 未演示） |

> 1 个特性命中（010102）覆盖全量+仅改重发 2 个脚本；N4T3RESPONSE 在不同 md 实例值不同（配置 md=3/修改 md=10），按规划值取。

## 约束

- **全局唯一最大记录数 1**（critical，rule-0-00452）：本命令全局唯一 — 后果：重复配置命令拒收
- **高危命令**（critical，rule-0-00452）：配置 N4 重发间隔/最大次数不合理可能导致用户激活失败或资源残留 — 后果：高危配置错误
- **N4ECHOSW=ENABLE 门控**（critical，rule-0-00452）：N4ECHOSW=ENABLE 时其余 5 参数（N4ECHOINTERVAL/DEACTIVEFLAG/N4T3RESPONSE/N4N3REQUEST/N4ECHOTIME）才生效 — 后果：N4ECHOSW 关闭时其余参数不生效
- **N4ECHOTIME 必填条件**（critical，rule-0-00452）：DEACTIVEFLAG=ENABLE 时 N4ECHOTIME 必填 — 后果：缺 N4ECHOTIME 命令拒收
- **N4ECHOTIME=0 语义**（warning，rule-0-00452）：路径断告警后不再发 Echo 消息，但 DEACTIVEFLAG=ENABLE 仍会去活 — 后果：N4ECHOTIME=0 配合 ENABLE 仍触发去活
- **基站多时建议大 ECHOINTERVAL**（warning，rule-0-00452）：基站数目较多时建议配较大 N4ECHOINTERVAL（防单位时间内 ECHO 过多致 CPU 上升影响基本业务） — 后果：ECHOINTERVAL 过小致 CPU 上升
- **N4ECHOINTERVAL 下个周期生效**（warning，rule-0-00452）：设置后在下个心跳周期开始生效 — 后果：当前周期仍按旧值发送
- **适用 NF 限定**（warning，隐含）：SGW-U、PGW-U、UPF — 后果：其他 NF 不适用
- **仅作用于 N4/Sxb 接口**（warning，隐含）：非 N4/Sxb 接口 GTP 路径由 [0-00051](task/UDG/20.15.2/0-00051.md)（SET UPGTPPATH）配置 — 后果：误用本命令配非 N4 接口不生效
