# 配置N4接口GTP路径相关属性（SET UPN4UPATH）

> 命令静态知识见 [SET UPN4UPATH](command/UDG/20.15.2/SET-UPN4UPATH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是 [SET UPGTPPATH](task/UDG/20.15.2/0-00051.md)（非 N4 接口）的**N4/Sxb 接口专用对应命令**：不同命令、参数名统一加 N4 前缀，按一命令一 atom 分离。全局唯一（最大记录数 1），仅 N4 接口生效。

**6 参数组（N4 接口 Echo 心跳机制）**

| 参数 | 含义 | 实例值 | 取值范围 |
|---|---|---|---|
| N4ECHOSW | N4 接口 Echo 总开关 | ENABLE | 枚举 |
| N4ECHOINTERVAL | N4 发送 GTP 心跳间隔 | 60 | 60~3600 秒 |
| N4T3RESPONSE | N4 Echo 请求重发间隔 | 3 | 1~20 秒 |
| N4N3REQUEST | N4 Echo 请求最大重发次数 | 5 | 1~6 次 |
| DEACTIVEFLAG | 路径断是否去活上下文 | ENABLE | 枚举 |
| N4ECHOTIME | N4 路径断告警后发 Echo 次数 | 0 | 0~60 |

**门控关系**：N4ECHOSW=ENABLE 是其余 5 参数生效的前提（N4ECHOSW 关闭时其余 5 参数不生效）。

**告警-去活链路**：Echo 重发达 N4N3REQUEST 仍无响应 → 上报 ALM-81018 数据路径断 → 启动 N4ECHOINTERVAL 定时器 → 发送 N4ECHOTIME 次 Echo → 仍无响应且 DEACTIVEFLAG=ENABLE → 去活该路径上所有上下文。

**N4ECHOTIME=0 语义**：任务示例取 0，表示路径断告警后不再发 Echo 消息，但若 DEACTIVEFLAG=ENABLE 仍会去活该路径上上下文。

**典型行（GWFD-010102 实例）**
```
// 全量配置（配置GTP_PFCP路径管理参数 md）
SET UPN4UPATH: N4ECHOSW=ENABLE, N4ECHOINTERVAL=60, N4T3RESPONSE=3, N4N3REQUEST=5, DEACTIVEFLAG=ENABLE, N4ECHOTIME=0;
// 仅改重发参数（修改Echo消息的重发数据 md）
SET UPN4UPATH: N4T3RESPONSE=10, N4N3REQUEST=3;
```

> 注意两个 md 的 N4T3RESPONSE 实例值不同（配置 md=3、修改 md=10），按各场景原值保留，配置时取当前规划值即可。

## 约束

- **rule-0-00326**（critical）高危命令 + 全局唯一：最大记录数 1；配置 N4 重发间隔/最大次数不合理可能导致用户激活失败或资源残留；命令立即生效；N4ECHOSW=ENABLE 时参数 N4ECHOTIME/N4ECHOINTERVAL/DEACTIVEFLAG/N4T3RESPONSE/N4N3REQUEST 才生效；基站数目较多时建议配较大 ECHOINTERVAL（防单位时间内 ECHO 过多致 CPU 上升影响基本业务） — 后果：高危配置错误导致用户激活失败/资源残留；N4ECHOSW 关闭时其余 5 参数不生效
