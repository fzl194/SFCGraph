# 配置带宽管理控制器（ADD BWMCONTROLLER）

> 命令静态知识见 [ADD BWMCONTROLLER](command/UDG/20.15.2/ADD-BWMCONTROLLER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD BWMCONTROLLER 是带宽控制骨架的"控制算法定义层"：把一个 bwm 控制器用控制类型 + 速率/队列 + 颜色策略三组参数描述，产出 BWMCNAME 供 ADD BWMRULE(0-00037) 引用做 UP/DN 控制器挂接。命令核心配置空间有 3 个维度：**控制类型**（CTRLTYPE 二选一：CAR 令牌桶 / SHAPING 流量整形）、**颜色策略**（CAR 模式下 GREEN/YELLOW/RED 各自 PASS/REMARK 决策）、**组级工作模式**（SHAPING 模式下 WORKMODE AUTO/MANUAL + 丢包/保障参数）。4 个特性命中覆盖 020354 SHAPING 基础、110301 CAR 上行下行、110311 CAR 多档位、110313 SHAPING 组级 AUTO 模式。

### 配置维度 1：控制类型（参数 CTRLTYPE，必选）

| 取值 | 作用 | 必选联动参数 |
|---|---|---|
| **CAR** | 令牌桶限速（CIR/CBS/PIR/PBS 四件套 + 三色策略） | CIR 必选 + CBS/PIR/PBS/颜色对 可选 |
| **SHAPING** | 流量整形（RATE/QUEDEPTH 二件套 + 可选组级模式） | RATE + QUEDEPTH 必选 |

### 配置维度 2：CAR 模式参数（参数 CIR/CBS/PIR/PBS，CTRLTYPE=CAR 时）

| 参数 | 必选性 | 取值范围 | 作用 |
|---|---|---|---|
| **CIR** | 必选 | 整数（kbps） | 承诺信息速率（保证带宽） |
| **CBS** | 可选 | 整数（字节） | 承诺突发尺寸 |
| **PIR** | 可选 | 整数（kbps） | 峰值信息速率 |
| **PBS** | 可选 | 整数（字节） | 峰值突发尺寸 |

### 配置维度 3：颜色策略（参数 GREEN/YELLOW/RED + REMARK 系列，CTRLTYPE=CAR 时可选）

| 参数 | 取值 | 联动 |
|---|---|---|
| **GREEN** | PASS / REMARK | REMARK 时需 GREENREMARKT |
| **YELLOW** | PASS / REMARK | REMARK 时需 YELLOWREMARKT |
| **RED** | PASS / REMARK / DISCARD | REMARK 时需 REDREMARKT |

| 重标记类型 (*REMARKT) | 联动 | 作用 |
|---|---|---|
| **DSCPVALUE** | + *REMARKV（0~63） | 标记为具体 DSCP 数值 |
| **DSCPCLASS** | + *REMARKC（BE/AF1~4/EF/CS1~7） | 标记为标准 PHB 类别 |

### 配置维度 4：SHAPING 模式参数（参数 RATE/QUEDEPTH/MAXFDNUM/SRVLEVELSPEC/USERFAIREN/WORKMODE 等，CTRLTYPE=SHAPING 时）

| 参数 | 必选性 | 取值 | 作用 |
|---|---|---|---|
| **RATE** | 必选 | 整数（kbps） | SHAPING 速率 |
| **QUEDEPTH** | 必选 | 整数（包数） | SHAPING 缓存队列深度 |
| **MAXFDNUM** | 可选 | 整数 | 业务最大五元组数（控制业务流数） |
| **SRVLEVELSPEC** | 可选 | 整数 | 最大可支持业务等级规格数（组级带宽管理） |
| **USERFAIREN** | 可选 | ENABLE / DISABLE | 用户级公平调度使能 |
| **WORKMODE** | 可选 | MANUAL / AUTO | 组级带宽控制工作模式 |
| **MAXPKTLOSTRATE** | WORKMODE=AUTO 时可选 | 整数 | 允许最大丢包率 |
| **PKTLOSTRATEDTL** | WORKMODE=AUTO 时可选 | 整数 | 两个 ServiceLevel 间丢包率差值 |
| **ASSUREMODE** | WORKMODE=AUTO 时可选 | EXPFIRST / RATEFIRST | 优先保障模式 |

**典型脚本（020354 基于业务的 Shaping，SHAPING 基础）**：
```
ADD BWMCONTROLLER:BWMCNAME="bwmc_test",CTRLTYPE=SHAPING,RATE=50,QUEDEPTH=256,MAXFDNUM=2000;
```

**典型脚本（110301 码率差异化，CAR 上行下行对称）**：
```
ADD BWMCONTROLLER:BWMCNAME="bc-up",CTRLTYPE=CAR,CIR=300,CBS=10000,PIR=600,PBS=12000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
ADD BWMCONTROLLER:BWMCNAME="bc-down",CTRLTYPE=CAR,CIR=300,CBS=10000,PIR=600,PBS=12000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
```

**典型脚本（110311 业务感知带宽控制，CAR 多档 + 重标记）**：
```
ADD BWMCONTROLLER:BWMCNAME="bc1",CTRLTYPE=CAR,CIR=30,CBS=1000,PIR=50,PBS=1200,GREEN=PASS,YELLOW=REMARK,YELLOWREMARKT=DSCPVALUE,YELLOWREMARKV=10,RED=DISCARD;
```

**典型脚本（110313 组级带宽控制，SHAPING+AUTO+组级参数）**：
```
ADD BWMCONTROLLER:BWMCNAME="bc1",CTRLTYPE=SHAPING,RATE=400,QUEDEPTH=40,SRVLEVELSPEC=3,USERFAIREN=ENABLE,WORKMODE=AUTO,MAXPKTLOSTRATE=5000,PKTLOSTRATEDTL=50,ASSUREMODE=EXPFIRST;
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；MAXFDNUM 用于限制业务流数；CAR 模式下 COLORISAWARE / PRIORITYISAWARE 控制颜色/优先级敏感（110311 演示 PRIORITYISAWARE=DISABLE）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：控制算法定义"——前置无；产出 BWMCNAME 被 ADD BWMRULE(0-00037) 的 UPBWMCTRLNAME1/DNBWMCTRLNAME1~4 引用。

## 决策点：控制类型与子分支组合（DP 0-00036）

问：ADD BWMCONTROLLER 选哪个控制类型 + 子分支组合？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| SHAPING 基础 | + RATE + QUEDEPTH [+ MAXFDNUM] | 简单整形+限流（020354 范式） |
| SHAPING 组级 AUTO | + SRVLEVELSPEC + USERFAIREN + WORKMODE=AUTO + MAXPKTLOSTRATE/PKTLOSTRATEDTL/ASSUREMODE | 组级智能调度（110313 范式） |
| CAR 三色 PASS | + CIR/CBS/PIR/PBS + GREEN/YELLOW/RED=PASS + RED=DISCARD | 简单限速+丢红色（110301/110311 范式） |
| CAR 三色重标记 | + YELLOW/RED=REMARK + *REMARKT=DSCPVALUE/DSCPCLASS + *REMARKV/*REMARKC | 按档位重标记（110311 范式） |

> 选项不互斥：同 feature 可建多 BWMCONTROLLER（如 bc-up/bc-down/各档独立）。

## 约束

- **CTRLTYPE 决定必选参数集**（critical，rule-0-00336）：选 SHAPING 必配 RATE+QUEDEPTH；选 CAR 必配 CIR — 后果：缺必选参数命令拒收
- **CAR 颜色动作 REMARK 时必入子分支**（critical，rule-0-00336）：*REMARK 配 REMARK 时必配 *REMARKT；*REMARKT=DSCPVALUE 必配 *REMARKV；*REMARKT=DSCPCLASS 必配 *REMARKC — 后果：缺子分支命令拒收
- **SHAPING 组级参数必 WORKMODE=AUTO**（critical，rule-0-00336）：MAXPKTLOSTRATE/PKTLOSTRATEDTL/ASSUREMODE 仅在 WORKMODE=AUTO 时允许配 — 后果：WORKMODE=MANUAL 时配组级参数命令拒收
- **BWMCNAME 全网唯一**（critical，隐含）：不同控制器之间 BWMCNAME 不能重复 — 后果：重名 ADD 失败
- **CAR/SHAPING 互斥**（critical，rule-0-00336）：同一 BWMCNAME 只能选一种控制类型 — 后果：跨类型参数混搭命令拒收
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
