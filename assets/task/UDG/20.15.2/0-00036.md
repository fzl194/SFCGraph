# 配置带宽管理控制器（ADD BWMCONTROLLER）

> 命令静态知识见 [ADD BWMCONTROLLER](command/UDG/20.15.2/ADD-BWMCONTROLLER.md)

## 配置方法

定义一套业务控制动作模板（速率/突发/色动作/队列），由 CTRLTYPE 决定走 SHAPING 整形还是 CAR 限速，配套参数完全不同。BwmController 只有被 BWMRULE 引用才产生实际控制作用。

**三大分支配置范式**：

| 分支 | CTRLTYPE | 必配核心参数 | 适用场景 |
|---|---|---|---|
| SHAPING 基础整形（MANUAL） | SHAPING | RATE / QUEDEPTH | 020354 URL整形、110311 简单场景 |
| SHAPING 智能组级（AUTO） | SHAPING | RATE / QUEDEPTH + SRVLEVELSPEC / USERFAIREN=ENABLE / WORKMODE=AUTO / MAXPKTLOSTRATE / PKTLOSTRATEDTL / ASSUREMODE=EXPFIRST | 110313 智能组级带宽控制 |
| CAR 承诺接入速率 | CAR | CIR / CBS / PIR / PBS / GREEN=PASS / YELLOW=PASS / RED=DISCARD | 110301 终端系统码率差异、110311 PCC预定义 |

**CAR 推荐公式**（命令 notes）：
- CIR = 0.5 × PIR（最大带宽为保证带宽的 2 倍）
- CBS ≥ CIR × 1.5 × 1000 / 8（保证 1.5 秒队列缓存）
- PBS ≥ PIR × 1.5 × 1000 / 8

**典型脚本**：

```
# SHAPING 用户级整形（020354）
ADD BWMCONTROLLER:BWMCNAME="bc-shaping",CTRLTYPE=SHAPING,RATE=10000,QUEDEPTH=1000;

# CAR 终端差异化（110301）
ADD BWMCONTROLLER:BWMCNAME="bc-up",CTRLTYPE=CAR,CIR=2000000,CBS=320000000,PIR=4000000,PBS=800000000,GREEN=PASS,YELLOW=PASS,RED=DISCARD;
```

**上下行双控制器**：CAR 场景通常建 2 个控制器（`bc-up`/`bc-down` 或 `bc-up-p2p`/`bc-dn-p2p`），分别对应 BWMRULE 的 UPBWMCTRLNAME1/DNBWMCTRLNAME1。

**步骤位置**：BWMSERVICE 之后、BWMRULE 之前——控制器是动作模板，规则把模板绑到具体用户组+业务上。

## 决策点

**DP 0-00100**：BWMCONTROLLER 控制类型 3 路分叉（SHAPING-MANUAL / SHAPING-AUTO / CAR），互斥单选，决定 CTRLTYPE 取值与 16 个配套参数的必选/必不传。

| 选项 | CTRLTYPE | 必选 | 不传 |
|---|---|---|---|
| SHAPING-MANUAL（默认） | SHAPING | RATE/QUEDEPTH | 全部 CAR 参数 + AUTO 参数 |
| SHAPING-AUTO（110313） | SHAPING | RATE/QUEDEPTH + SRVLEVELSPEC/USERFAIREN/WORKMODE=AUTO/MAXPKTLOSTRATE/PKTLOSTRATEDTL/ASSUREMODE | 全部 CAR 参数 |
| CAR（110301） | CAR | CIR/CBS/PIR/PBS/GREEN/YELLOW/RED | RATE/QUEDEPTH + 全部 AUTO 参数 |

注：MAXFDNUM 三特性可选，不参与 DP；COLORISAWARE/PRIORITYISAWARE 命令真相 conditional，实战可选 ENABLE。

## 约束

- 最大记录数 2000 · warning · 超规格新增失败；90% 阈值告警 ALM-135602215
