# 配置URR（ADD URR）

> 命令静态知识见 [ADD URR](command/UDG/20.15.2/ADD-URR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

URR 按"业务 × 计费方式"成对规划：同一业务的离线+在线 URR 成对出现，进同一个 URRGROUP。URRID 分段编号、URRNAME 带语义后缀，便于跨网元（SMF/UPF、PGW-C/PGW-U）识别对齐。

**编号约定（从证据数据规划表归纳）**

| 业务 | 离线 URRID | 在线 URRID | URRNAME 命名 | 统计类型 |
|---|---|---|---|---|
| URL / 网址 | 12500 | 2300 | `URR_URL_offline` / `URR_URL_online` | VOLUME |
| IMS / 语音 | 13500 | 3300 | `URR_IMS_offline` / `URR_IMS_online` | VOLUME |
| any / 兜底 | 11500 | 1300 | `URR_any_offline` / `URR_any_online` | VOLUME |
| 异常 / 防欺诈 | 14500 | 4300 | `URR_abnormal_offline` / `URR_abnormal_online` | **FREE** |


**典型行（节选）**
```
ADD URR:URRNAME="URR_URL_offline",URRID=12500,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=VOLUME;
ADD URR:URRNAME="URR_URL_online",URRID=2300,USAGERPTMODE=ONLINE,ONLMETERINGTYPE=VOLUME;
ADD URR:URRNAME="URR_abnormal_offline",URRID=14500,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=FREE;
```

**统计类型取值**：流量计费 VOLUME、时长 TIME、事件 EVENT、流量+时长+事件 EVENT_VOLUME_TIME；选含 EVENT 的枚举前评估性能影响。

**在线计费默认配额**：在线 URR 可配 `DFTQUOTAVOLUME=4294967295`（=继承全局默认配额，非"配 4G 流量"），单位默认 `Byte`；配 0=不使能默认配额。

## 决策点：计费方式选择（DP 0-00001）

问：该费率采用哪种计费方式？

| 选项 | 联动参数 |
|---|---|
| 离线 OFFLINE | `USAGERPTMODE=OFFLINE` + `OFFMETERINGTYPE` 出现 |
| 在线 ONLINE | `USAGERPTMODE=ONLINE` + `ONLMETERINGTYPE` 出现 + 可选 DFTQUOTAVOLUME/DFTQUOTAPARA |
| 融合 CONVERGED | 不是单条 URR 的属性，而是 URRGROUP 内在线+离线 URR 共存 |

例外：SMF 融合计费且 `SET USRPROFCHARGE`/`SET APNCHARGECTRL`/`ADD CHARGEMETHOD` 的 `RGAPPLIED=DEFAULT` 时，URRGROUP 下**不能**同时配离线和在线 URR（会冲突）。

## 约束

- **rule-0-00003**（critical）URRID/USAGERPTMODE/OFFMETERINGTYPE/ONLMETERINGTYPE 在 SMF 与 UPF（或 PGW-C/PGW-U）配置必须完全一致 — 违反后果：跨网元计费不一致、用量上报失败
- **rule-0-00111**（critical）URRID 全网唯一、从 1000 起始（0~2147483646，不复用） — 后果：URRID 重复导致计费规则冲突
- **rule-0-00110**（warning）最大记录数 65000，超 90% 上报 ALM-135602215（阈值可 MOD CFGTHRESHOLD 修改） — 后果：超规格新增失败
