# 配置URR（ADD URR）

> 命令静态知识见 [ADD URR](command/UDG/20.15.2/ADD-URR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

URR 是计费三件套的"费率定义层"：把"哪种业务 × 哪种计费方式 × 哪种统计口径"用一条记录描述，供下游 URRGROUP→PCCPOLICYGRP 引用。本命令的核心配置空间有 4 个维度：**上报模式**（必选，决定后续统计/费率参数是否出现）、**统计类型**（必选，按计费模式分支）、**费率标识组成**（可选，按上报模式分支）、**默认配额单位**（仅 ONLINE）。每个特性选定一组取值后即得一条 URR；同一业务常以离线+在线成对出现，再被 URRGROUP 聚合成费率组。

### 配置维度 1：上报模式（参数 USAGERPTMODE，必选）

| 取值 | 作用 | 联动必选参数 | 联动可选参数 | 典型场景 |
|---|---|---|---|---|
| **OFFLINE** | 离线计费，话单上报 CG（Ga 接口） | `OFFMETERINGTYPE` | `RG` / `SID` | Ga 离线、内容计费（VOLUME/EVENT_VOLUME_TIME） |
| **ONLINE** | 在线计费，实时配额申请（Gy/N40 接口） | `ONLMETERINGTYPE` | `DFTQUOTAVOLUME` / `DFTQUOTAPARA` / `RG` / `SID` | Gy 在线、N40 融合（VOLUME/EVENT） |
| **MONITORINGKEY** | 流量累计监控（不产生费用，仅作 Monitoring Key 累计） | — | — | FUP 流量累计策略控制 |
| **QOS** | QoS 保证标记（不产生费用，仅触发 QoS 控制） | — | — | 业务触发 QoS |

> 4 种上报模式互斥；同一业务通常按"业务 × 计费模式"成对配 2 条 URR（离线一条 + 在线一条）。**CONVERGED 不是 URR 自身的属性**，而是 URRGROUP 内 UP/DOWNURRNAME1（离线）+ UP/DOWNURRNAME2（在线）共存的组装方式——见 [ADD URRGROUP](task/UDG/20.15.2/0-00002.md)。

### 配置维度 2：统计类型（参数 OFFMETERINGTYPE / ONLMETERINGTYPE）

**两参数完全对称的取值集**，按上报模式分支出现：

| 取值 | 统计对象 | 适用上报模式 | 备注 |
|---|---|---|---|
| **VOLUME** | 按流量（默认） | OFFLINE / ONLINE | 默认值；最常用 |
| **TIME** | 按时长 | OFFLINE / ONLINE | 时长计费 |
| **EVENT** | 按事件 | OFFLINE / ONLINE | 防欺诈判断场景配 `FREE` 标志 |
| **FREE** | 免费标记 | OFFLINE / ONLINE | 计费防欺诈判断（不实际计费） |
| **VOLUME_TIME** | 流量 + 时长 | OFFLINE / ONLINE | 多维度累计 |
| **EVENT_VOLUME** | 事件 + 流量 | OFFLINE / ONLINE | 多维度累计 |
| **EVENT_TIME** | 事件 + 时长 | OFFLINE / ONLINE | 多维度累计 |
| **EVENT_VOLUME_TIME** | 事件 + 流量 + 时长 | OFFLINE / ONLINE | 内容计费多维累计（典型） |

> 选含 EVENT 的枚举前评估性能影响（命令 notes 明确提示）。OFFLINE 默认 VOLUME、ONLINE 默认 VOLUME；不配置时按默认值生效。

### 配置维度 3：费率标识组成（参数 RG / SID，条件可选）

**RG 与 SID 配对与否**取决于计费场景对"费率组 vs 业务标识"二级的需要：

| 取值 | 作用 | 适用场景 |
|---|---|---|
| 仅 `RG` | 计费组（一级费率标识） | 标准计费（业务维度合并） |
| `RG` + `SID` | 计费组 + 业务标识（二级费率标识） | 报表功能开启（软参 bit1557），按业务细分 |

| 参数 | 范围 | 约束 |
|---|---|---|
| `RG` | 整数 0~4294967294 | 报表开启时需与 SMF 保持一致；典型多档位：10/15/20/25 |
| `SID` | 整数 0~4294967294 | 报表开启时需与 SMF 保持一致 |

> RG/SID 在 SMF/UPF（或 PGW-C/PGW-U）跨网元必须一致——见约束 §0-00003。

### 配置维度 4：默认配额单位（参数 DFTQUOTAVOLUME / DFTQUOTAPARA，仅 ONLINE）

**在线计费 URR 在 OCS 未及时响应时的"默认放通配额"**：

| 参数 | 取值范围 | 含义 |
|---|---|---|
| `DFTQUOTAVOLUME` | 0、1000~4294967295 | 默认流量配额；0=不使能默认配额；4294967295=继承全局默认配额 |
| `DFTQUOTATIME` | 0~3600、4294967295 | 默认时长配额（秒）；0=不使能；4294967295=继承 |
| `DFTQUOTAPARA` | `Byte` / `KByte`（默认 Byte） | 配额单位；超过 4G 配额需配 KByte |

> 受全局开关 `SET UPDEFAULTQUOTA` 控制——使能后未及时申请到配额期间按 DFTQUOTA 放通业务；不使能则按 `SET UPGLBCHGPARA` 决定缓存或丢弃。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：配置 URR"——紧随过滤链 / License 前置之后，URRGROUP 绑定之前。

## 决策点：计费方式选择（DP 0-00001）

问：该费率采用哪种计费方式（决定上报模式与统计类型参数）？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 离线 OFFLINE | `USAGERPTMODE=OFFLINE` + `OFFMETERINGTYPE` 出现 | Ga 接口离线计费、内容计费、融合计费（与 ONLINE 配对） |
| 在线 ONLINE | `USAGERPTMODE=ONLINE` + `ONLMETERINGTYPE` 出现 + 可选 DFTQUOTAVOLUME/DFTQUOTAPARA | Gy 在线、N40 融合、累计流量策略 |
| 融合 CONVERGED | 同 URRGROUP 内 UP/DOWNURRNAME1（离线）+ UP/DOWNURRNAME2（在线）共存 | 不是单条 URR 的属性 |

例外：SMF 融合计费且 `SET USRPROFCHARGE（★R1.5 atom 待补）` / `SET APNCHARGECTRL（★R1.5 atom 待补）` / `ADD CHARGEMETHOD（★R1.5 atom 待补）` 的 `RGAPPLIED=DEFAULT` 时，URRGROUP 下**不能**同时配离线和在线 URR（会冲突）。

## 约束

- **URR 关键参数跨网元一致**（critical，rule-0-00003）：URRID / USAGERPTMODE / OFFMETERINGTYPE / ONLMETERINGTYPE 在 SMF 与 UPF（或 PGW-C/PGW-U）必须保持一致 — 后果：跨网元计费不一致、用量上报失败
- **URRID 全网唯一且从 1000 起始分配**（critical，rule-0-00111）：URRID 在所有 URR 内唯一（取值 0~2147483646，从 1000 起始） — 后果：URRID 重复导致计费规则冲突/统计错配
- **ADD URR 最大记录数规格**（warning，rule-0-00110）：单板最大 65000；超 90% 上报 ALM-135602215 配置数量超阈值告警（阈值可 `MOD CFGTHRESHOLD（★R1.5 atom 待补）` 修改） — 后果：超规格新增失败；接近阈值时告警
- **DFTQUOTA 默认配额单位隐式约束**（info，隐含）：DFTQUOTAPARA 不配默认 Byte；配额超过 4G 必须改 KByte — 后果：默认单位配错导致配额溢出
