# 配置地址池组与APN映射关系（ADD POOLGRPMAP）

> 命令静态知识见 [ADD POOLGRPMAP](command/UDG/20.15.2/ADD-POOLGRPMAP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD POOLGRPMAP 是地址池族链路收口步骤——把 APN 映射到地址池组，完成"APN → 地址池组 → 地址池 → 地址段"的外部地址分配链路。**双引用上游**：APN ← ADD APN（0-00020，`LST APN` 可查），POOLGROUPNAME ← ADD POOLGROUP 的 POOLGRPNAME（0-00042，`LST POOLGROUP`/`LST POOLBINDGROUP` 可查）。执行后**只对新激活用户生效**。

MAPPINGNAME 是本命令本端规划的唯一映射规则名（1~79 字节）。映射键可选 APN / SMF / 位置区组（LOCATIONGRPTYPE + LOCATIONGRPNAME），按实际组网组合。

**按映射键的范式（数据规划表归纳）**

| 映射维度 | 必配键 | 典型特性 |
|---|---|---|
| APN 级映射 | MAPPINGNAME + APN + POOLGROUPNAME | 地址分配方式、双栈、入不转板、路由冗余 |
| SMF 级映射 | MAPPINGNAME + SMF + POOLGROUPNAME（+ APN 可选） | 基于 SMF 分配、基于 SMF+APN 分配 |
| 位置区级映射 | MAPPINGNAME + LOCATIONGRPTYPE + LOCATIONGRPNAME + POOLGROUPNAME（+ APN 可选） | 基于位置/位置+APN 分配（LAC/TAC-Group） |

**典型行**

APN 级映射（010104 地址分配方式）：
```
ADD POOLGRPMAP: MAPPINGNAME="mapping1", APN="apn-test", POOLGROUPNAME="poolgroup1";
```
SMF 级映射（010105 基于 SMF 分配，SMF=smfnode1 需先 ADD CPNODEID）：
```
ADD POOLGRPMAP: MAPPINGNAME="mapping1", SMF="smfnode1", POOLGROUPNAME="poolgroup1";
```
SMF + APN 级映射（010105 基于 SMF+APN 分配）：
```
ADD POOLGRPMAP: MAPPINGNAME="mapping1", APN="apn-test1", SMF="smfnode1", POOLGROUPNAME="poolgroup1";
ADD POOLGRPMAP: MAPPINGNAME="mapping2", SMF="smfnode1", POOLGROUPNAME="poolgroup2";
```
位置区（LAC）级映射（020421 基于位置分配，需先 ADD LACGROUP + ADD LACID）：
```
ADD POOLGRPMAP: MAPPINGNAME="mapping1", LOCATIONGRPTYPE=LAC, LOCATIONGRPNAME="lac1", POOLGROUPNAME="poolgroup1";
```

**前置依赖链**：必须先 ADD POOLGROUP（建组）；按组网还需先 ADD APN（必配）、ADD CPNODEID（SMF 级）、ADD TACGROUP/ADD LACGROUP + ADD LACID（位置级）。MCC/MNC（PLMN 级）按组网可选。

**未绑冷门可选/conditional**：LOCATIONGRPTYPE/LOCATIONGRPNAME/SMF/MCC/MNC 命令真相为 optional/conditional，仅位置级/SMF 级/PLMN 级组网才用，按通用门槛不绑进基础范式。

## 约束

- **rule-0-00144**（warning）最大记录数 40000；前置必须先 ADD POOLGROUP（按组网还需 ADD APN / 可选 ADD TACGROUP/LACGROUP/ADD CPNODEID）；PoolGrpMap 对象本身为 SMF/APN 级映射规则；只对新激活用户生效 — 后果：超规格新增失败 / 缺前置（POOLGROUP/APN）致映射引用悬空 / 已激活用户不受影响需重激活
