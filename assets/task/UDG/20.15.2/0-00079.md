# 创建OSPFv3区域(IPv6)（ADD OSPFV3AREA）

> 命令静态知识见 [ADD OSPFV3AREA](command/UDG/20.15.2/ADD-OSPFV3AREA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD OSPFV3AREA 在已创建的 OSPFv3 进程下增加区域。3 个特性数据规划表（GWFD-010155/020401/020406）仅用 PROCID + AREAID 两参，其余 17 个区域进阶参数（AREATYPE/NSSA 系列/Stub 等）均未揭示，按通用配置筛不绑。PROCID 引用上游进程，AREAID 由决策点驱动。

**两种 AREAID 来源范式（从证据数据规划表归纳）**

| 组网 | PROCID | AREAID | 脚本样例 |
|---|---|---|---|
| PGW-U 到 ePDG IPv6 动态路由（GWFD-010155） | 6（已配置数据中获取，引用 0-00078） | 0.0.0.1（全网规划） | `ADD OSPFV3AREA:PROCID=6,AREAID="0.0.0.1";` |
| IPv6 承载上下文（GWFD-020401） | 100（全网规划，引用 0-00078） | 0.0.0.1（本端规划） | `ADD OSPFV3AREA:PROCID=100,AREAID="0.0.0.1";` |
| IPv6 Prefix Delegation（GWFD-020406 系列） | 6（本端规划，引用 0-00078） | 0.0.0.5（与对端协商） | `ADD OSPFV3AREA:PROCID=6,AREAID="0.0.0.5";` |

**参数取值规则**
- **PROCID**：reference 引用上游 [创建OSPFv3进程(IPv6)](task/UDG/20.15.2/0-00078.md)，保证进程号一致。
- **AREAID**：IPv4 点分十进制格式，**须与对端路由器在同一区域一致**，否则 OSPFv3 邻居无法建立（backbone 常取 0.0.0.0）。AREAID 同时被 ADD OSPFV3INTERFACE(0-00117) 引用，决策传递下游。

## 决策点

- **DP 0-00082**（OSPFv3 区域号 AREAID 来源）single_choice，必选：
  - **opt-areaid-peer-v3（与对端协商 peer）**：AREAID 须与对端路由器在同一区域协商一致（backbone 常取 0.0.0.0），否则邻居无法建立。适用于 IPv6 Prefix Delegation 等需与外部路由器对接的场景（020406）。
  - **opt-areaid-global-v3（全网规划 global_planned）**[default]：AREAID 由 UDG 侧全网规划（如 ePDG 动态路由组网取 0.0.0.1），仍须与对端在同一区域。适用于 PGW-U 到 ePDG 的 OSPFv3 等价路由组网（010155/020401）。

## 约束

- **rule 0-00201**（constraint/critical）前置——必须先执行 ADD OSPFV3 创建对应 OSPFv3 进程(PROCID)才能使用此命令；OSPFv3 区域实例数规格默认单进程 2000，命令最大记录数 8016；AREAID(IPv4 点分十进制)须与对端路由器在同一区域一致，否则 OSPFv3 邻居无法建立(backbone 常取 0.0.0.0)；若 AREATYPE=NSSA 则必须配置 Loopback 地址并在 OSPFv3 进程中发布出去，否则会造成负载分担不生效，极端情况下会造成业务受损；AREATYPE 默认 Normal，可设 NSSA/Normal — 后果：负载分担不生效 / 业务受损 / 区域创建失败 / 邻居无法建立 / 区域实例超规格
