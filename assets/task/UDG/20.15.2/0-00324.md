# 设置基于SMF分配地址的开关（SET IPALLOCBYSMFSW）

> 命令静态知识见 [SET IPALLOCBYSMFSW](command/UDG/20.15.2/SET-IPALLOCBYSMFSW.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典",**不**逐特性罗列取值。该命令**适用 NF 限定 PGW-U/UPF**;max_records=64;**SMF 必须通过 ADD CPNODEID 命令配置过**(见 [0-00303](task/UDG/20.15.2/0-00303.md))。本命令无汇总 md(走 SOP §2.3 path 2,仅基于原始命令 md)。

## 配置方法(配置方法字典)

SET IPALLOCBYSMFSW 是"基于SMF分配地址的开关"命令,适用于 PGW-U/UPF。作用:按 SMF 实例维度细粒度配置基于SMF分配地址的 IPv4/IPv6 开关;SMF 级开关的 SWITCH=INHERIT 时继承全局 IPALLOCBYSMFGLBSW 配置。核心配置空间有 2 个维度:**SMF 名称**(SMF 必选,引用预建的 CPNODEID 实例)、**IPv4/IPv6 开关**(SWITCH/IPV6SWITCH 三态 DISABLE/ENABLE/INHERIT)。

### 配置维度 1:SMF 名称(参数 SMF,必选)

| 取值形式 | 取值范围 | 联动 |
|---|---|---|
| **字符串 1~255 位** | 长度 1~255,只能由 `_`、`-`、数字、大小写字母和 `.` 组成,不能以 `.` 开头且不能出现连续两个 `.`,不区分大小写 | 必须已通过 ADD CPNODEID(见 [0-00303](task/UDG/20.15.2/0-00303.md))预建该 SMF 实例(CPNAME);命名约束与 CPNAME 一致 |

### 配置维度 2:IPv4/IPv6 开关(参数 SWITCH/IPV6SWITCH,可选,三态)

| 取值 | 作用 | 联动 |
|---|---|---|
| **DISABLE** | 不使能基于SMF分配地址 | SMF分配地址强制关闭 |
| **ENABLE** | 使能基于SMF分配地址 | SMF分配地址开启 |
| **INHERIT**(默认) | 继承全局 IPALLOCBYSMFGLBSW 配置 | SMF级不独立决策,跟全局值走 |

> **三态语义关键**:SWITCH=INHERIT 是本命令核心特性——SMF 级"不独立决策,继承全局",适配"SMF分配地址全网一致"的场景;SMF 级 DISABLE/ENABLE 是"SMF 级独立决策"——允许特定 SMF 按SMF分配,其它 SMF 跟全局。

> **注意事项**(命令 notes 投影):
> - 该命令执行后**立即生效**。
> - 该命令最大记录数为 **64**(整机 SMF 级开关上限)。
> - 开关为 INHERIT 时,按全局 IPALLOCBYSMFGLBSW 配置分配地址;非 INHERIT 时按本 SMF 级配置分配地址。
> - SMF 必须通过 ADD CPNODEID 命令配置过(命名约束与 CPNAME 一致)。

**典型脚本(禁止 smfnode1 基于SMF分配地址)**:
```
SET IPALLOCBYSMFSW: SMF="smfnode1", SWITCH=DISABLE, IPV6SWITCH=DISABLE;
```

**典型脚本(smfnode1 IPv4 启用 IPv6 关闭)**:
```
SET IPALLOCBYSMFSW: SMF="smfnode1", SWITCH=ENABLE, IPV6SWITCH=DISABLE;
```

**典型脚本(smfnode1 双栈继承全局配置)**:
```
SET IPALLOCBYSMFSW: SMF="smfnode1", SWITCH=INHERIT, IPV6SWITCH=INHERIT;
```

**步骤位置**:在所属 feature 编排中位于"SMF级开关配置"——前置 [0-00323](task/UDG/20.15.2/0-00323.md) SET IPALLOCBYSMFGLBSW(全局开关)+ [0-00303](task/UDG/20.15.2/0-00303.md) ADD CPNODEID(SMF 实例对象预建,SMF 命名与 CPNAME 一致);下游 SMF 维度 SMF分配地址使能决策生效。

## 决策点:SMF 级开关取值模式(DP 0-00324)

问:SMF 级 SWITCH/IPV6SWITCH 取 INHERIT 还是 DISABLE/ENABLE?决定 SMF 维度的 SMF分配地址使能策略。

| 选项 | SWITCH/IPV6SWITCH | 含义 | 适用场景 |
|---|---|---|---|
| 继承全局(默认) | INHERIT | 跟全局 IPALLOCBYSMFGLBSW 配置走 | SMF分配地址全网一致的简单场景 |
| SMF 级强制关闭 | DISABLE | 本 SMF SMF分配地址强制关闭,不受全局影响 | 特定 SMF(如漫游边界)禁用 SMF分配 |
| SMF 级强制开启 | ENABLE | 本 SMF SMF分配地址强制开启,不受全局影响 | 特定 SMF(如核心区域)优先 SMF分配 |

各取值按 SMF 维度独立生效;INHERIT/DISABLE/ENABLE 三态互斥。

## 约束

- **适用 NF 限定 PGW-U/UPF**(critical,rule-0-00324):本命令仅适用于 PGW-U/UPF,控制面 SMF 等网元不适用 — 后果:非 PGW-U/UPF 网元命令拒收
- **SMF 必须预建**(critical,rule-0-00324):SMF 引用的 SMF 实例必须已通过 ADD CPNODEID 配置过(命名约束与 CPNAME 一致) — 后果:SMF 未建引用命令拒收
- **SMF 命名约束 1~255**(warning,rule-0-00324):字符串长度 1~255,字符集 `_`、`-`、数字、大小写字母、`.`,不以 `.` 开头且无连续 `..`,不区分大小写 — 后果:不符合命名规范命令拒收
- **最大记录数 64**(critical,rule-0-00324):整机 SMF 级开关最大 64 条 — 后果:超规格新增失败
- **执行后立即生效**(warning,rule-0-00324):命令执行后立即生效 — 后果:误操作即刻生效
- **INHERIT 三态继承全局**(critical,rule-0-00324):SWITCH=INHERIT 时按全局 IPALLOCBYSMFGLBSW 配置分配地址;非 INHERIT 时按本 SMF 级配置 — 后果:全局与 SMF 级决策混淆时SMF分配行为不可预期
- **两参数独立三态**(warning,rule-0-00324):SWITCH 与 IPV6SWITCH 独立三态(DISABLE/ENABLE/INHERIT),无联动关系 — 后果:IPv4/IPv6 协议族使能各自独立
