# L2TP VPN（GWFD-020412，UDG 侧）

> 特性静态知识见 [GWFD-020412](feature/UDG/20.15.2/GWFD-020412.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-020412-L2TPVPN.md)。
>
> **本页定位**：UDG 侧 L2TP VPN 配置生成——UDG 作 LAC 与企业网 LNS 建立 L2TP 隧道，LNS 在内层 PPP 协商中分配企业内网 IP（地址分配主体在 LNS，非 UDG 本地池）。2 大 LNS 参数来源方式（本地配置/AAA 下发）共享 License+VPN+GLOBALL2TP+APNL2TPATTR+SOFTPARAOFBIT+LOGICINF+PPPCFG+APNPPPACCESS+L2TPN4KEY 基础设施，差异在 LNS 容器建立（方式A 建 L2TPGROUP/L2TPCLIENTIP vs 方式B 建 L2TPRDSCLIENT）。调测命令见原始特性信息（[调测本地配置方式激活L2TP VPN](evidence/UDG/20.15.2/GWFD-020412-调测基于本地配置方式激活的L2TP-VPN_40342150.md) / [调测AAA下发方式激活L2TP VPN](evidence/UDG/20.15.2/GWFD-020412-调测基于AAA下发L2TP属性方式激活的L2TP-VPN_41487857.md)），不在配置流程展开。

## 配置概览

本特性按 **2 大 LNS 参数来源方式 × 1 compound 编排（方式A）+ 1 atom 编排（方式B）** 组织。两种方式共享 9 步基础设施，差异在 LNS 容器建立步骤。

**对象链**：
```
License（LKV3G5L2TP01）→ VPN 实例 → SET GLOBALL2TP（缺省本端名/HELLO/重发）
                                        ↓
                                 SET APNL2TPATTR（★APN 粒度使能）
                                        ↓
                                 SET SOFTPARAOFBIT（关闭快速流表 Byte671 bit7=1）
                                        ↓
                                 ADD LOGICINF（Giif 接口）
                                        ↓
              ┌─────────────────────────┴─────────────────────────┐
              ▼ 方式A（本地配置）                       ▼ 方式B（AAA 下发）
   ADD L2TPGROUP（LNS 组容器）+                 （无本地 L2TPGROUP）
   ADD L2TPCLIENTIP（组绑定源端接口）             ADD L2TPRDSCLIENT（APN 绑定源端接口）
   + ADD L2TPLNSINFO（扩展 LNS，可选）           （直接使用 AAA 下发的 LNS 参数）
              └─────────────────────────┬─────────────────────────┘
                                        ↓
               SET PPPCFG（PPP 协商参数，可选）+ SET APNPPPACCESS（PPP 鉴权，可选）
                                        ↓
                        SET L2TPN4KEY（N4 加密，可选）
```

**1 compound + 直接 atom 编排**：

| 步骤 | 干啥 | 编排 |
|---|---|---|
| 0. License 开关 | 打开 L2TP VPN License 82200BVC LKV3G5L2TP01 | [0-00019](task/UDG/20.15.2/0-00019.md) |
| 1. VPN 实例 | L2TP 隧道绑定的 VPN 实例 | [0-00041](task/UDG/20.15.2/0-00041.md) |
| 2. L2TP 缺省参数 | 本端名/HELLO/重发（方式B 额外配 Magic-Number/窗口/初始隧道/会话上限）| [0-00131](task/UDG/20.15.2/0-00131.md) |
| 3. APN L2TP 属性（★核心使能）| APN 粒度 L2TP 开关 + IPv6 支持 + 组绑定（方式A）/RDSLNSMODE（方式B）+ ICRQ/ICCN 信元 + IPCP 协商 + 域名增剥 | [0-00132](task/UDG/20.15.2/0-00132.md) |
| 4. 关闭快速流表 | SET SOFTPARAOFBIT Byte671 bit7=1（应用限制硬约束）| [0-00137](task/UDG/20.15.2/0-00137.md) |
| 5. Giif 接口 | 配置源端逻辑接口（绑 VPN）| [0-00083](task/UDG/20.15.2/0-00083.md) |
| 6a.（方式A）L2TP LNS 建立 | ADD L2TPGROUP（LNS 容器）+ ADD L2TPCLIENTIP（组绑定源端接口）+ ADD L2TPLNSINFO（扩展 LNS，可选）| [1-00034](task/UDG/20.15.2/1-00034.md) |
| 6b.（方式B）APN 绑定源端接口 | ADD L2TPRDSCLIENT（APN 绑定 Giif 接口）| [0-00134](task/UDG/20.15.2/0-00134.md) |
| 7.（可选）PPP 协商参数 | 本端主机名/MRU/超时 | [0-00138](task/UDG/20.15.2/0-00138.md) |
| 8.（可选）APN PPP 鉴权 | 支持 PPP 鉴权功能 | [0-00139](task/UDG/20.15.2/0-00139.md) |
| 9.（可选）N4 加密 | L2TP 私有信元加密（对端 SMF SET L2TPKEY 同步）| [0-00140](task/UDG/20.15.2/0-00140.md) |

**关键约束**：
- **必须 License** 82200BVC LKV3G5L2TP01
- **C-U 协同分工**：UNC 决策+下发 LNS 参数（WSFD-104410），UDG 执行隧道封装+PPP 协商透传
- **地址分配主体在 LNS**（企业网侧），非 UDG 本地池——即使本地配置地址池或 AAA 下发地址，L2TP 用户地址最终由 LNS 分配
- **互斥 GWFD-010108**（用户面地址自动检测）/ **GWFD-020482**（入不转板）
- **应用限制 8 条**：企业网必须部署 LNS / 不支持 PPP 用户接入 / 不支持 DHCP 延迟分配 IPv4 / 不支持 IPv6 PD / 不支持 PFC/ACFC 压缩 / 不支持 NAT / NP 加速不支持 IPv6 / 需关闭快速流表

## 配置流程

> 两种方式共享步骤 0-5 + 7-9；差异在步骤 6（LNS 容器建立）。调测命令见原始特性信息（[调测本地配置方式](evidence/UDG/20.15.2/GWFD-020412-调测基于本地配置方式激活的L2TP-VPN_40342150.md) / [调测AAA下发方式](evidence/UDG/20.15.2/GWFD-020412-调测基于AAA下发L2TP属性方式激活的L2TP-VPN_41487857.md)），不在配置流程展开。

1. **License 开关**（LKV3G5L2TP01）：
   - `SET LICENSESWITCH` LICITEM="LKV3G5L2TP01", SWITCH=ENABLE → [0-00019](task/UDG/20.15.2/0-00019.md)

2. **VPN 实例**（L2TP 隧道绑定的 VPN）：
   - `ADD VPNINST` VPNINSTANCE="vpn_l2tp" → [0-00041](task/UDG/20.15.2/0-00041.md)

3. **L2TP 缺省参数**（本端名/HELLO/重发；方式B 额外配隧道参数）：
   - `SET GLOBALL2TP` LOCALNAME="huawei", HELLOINTERVALSW=ENABLE, HELLOINTERVAL=60, RETRYTIMES=3 → [0-00131](task/UDG/20.15.2/0-00131.md)
   - 方式B 额外：PPPMAGICNUMBER=DISABLE, MAXSENDWINSIZE=64, INITTUNNELNUM=1, MAXSESSIONNUM=32767

4. **APN L2TP 属性**（★核心使能）：
   - `SET APNL2TPATTR` APN="apn-l2tp", L2TPSWITCH=ENABLE, SUPPORTIPV6=DISABLE/ENABLE, ICRQ_CALLINGNO=MSISDN, ICCN_AUTH=ENABLE, IPCP_NEGO=ENABLE, DOMAINNAMEACT=ADD_ENABLE_STRIP_DISABLE, DOMAINNAMEPOS=PREFIX → [0-00132](task/UDG/20.15.2/0-00132.md)
   - 方式A 额外：**L2TPGROUPID=组号**（指向步骤 6a 的 L2TP 组）
   - 方式B 额外：**RDSLNSMODE=REDUNDANCY/TUNNEL_PREFER**（AAA 返回多 LNS 工作模式）

5. **关闭快速流表**（应用限制硬约束）：
   - `SET SOFTPARAOFBIT` DT2=BYTE, BYTENUM=671, BYTEPOSITION=7, BYTEVALUE=1 → [0-00137](task/UDG/20.15.2/0-00137.md)

6. **Giif 接口**（源端逻辑接口，绑 VPN）：
   - `ADD LOGICINF` NAME="giif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.20.1", IPV4MASK1="255.255.255.255", VPNINSTANCE="vpn_l2tp" → [0-00083](task/UDG/20.15.2/0-00083.md)

7. **LNS 容器建立**（按方式选一）：
   - **方式A（本地配置）**：→ 复用 [1-00034](task/UDG/20.15.2/1-00034.md)（ADD L2TPGROUP + ADD L2TPCLIENTIP + ADD L2TPLNSINFO 扩展）
     - 场景变种（按 DP 选）：主备 LNS（REDUNDANCY）/ 负荷分担（LOAD_SHARING）/ IPv6 LNS（L2TPLNSINFO LNSIPVER=IPV6）/ 多 LNS >6（L2TPLNSINFO 扩展）
   - **方式B（AAA 下发）**：`ADD L2TPRDSCLIENT` APN="apn-l2tp", INTERFACENAME="giif1/0/1" → [0-00134](task/UDG/20.15.2/0-00134.md)
     - 场景变种（按 DP 选）：RDSLNSMODE=REDUNDANCY（主备，两 LNS Preference 相同）/ RDSLNSMODE=TUNNEL_PREFER（Preference 不同为主备，相同为负荷分担）

8. **（可选）PPP 协商参数**：
   - `SET PPPCFG` HOSTNAME="UPF", MRU=1500, TIMEOUT=3 → [0-00138](task/UDG/20.15.2/0-00138.md)

9. **（可选）APN PPP 鉴权**：
   - `SET APNPPPACCESS` APN="apn-l2tp", AUTHENTICATION=ENABLE → [0-00139](task/UDG/20.15.2/0-00139.md)

10. **（可选）N4 加密**（对端 SMF SET L2TPKEY 同步）：
    - `SET L2TPN4KEY` N4KEYVALUE="*****", CFMN4KEYVALUE="*****" → [0-00140](task/UDG/20.15.2/0-00140.md)

## 决策点

**DP（特性级）：LNS 参数来源 × LNS 工作模式 × 用户 IP 类型 × 走法**

| 场景 | LNS 参数来源 | LNS 工作模式 | 用户 IP 类型 | 走法（compound/atom 编排） |
|---|---|---|---|---|
| **本地配置主备 LNS**（方式A 基线）| 本地配置（L2TPGROUPID）| REDUNDANCY | IPv4 | 步骤 0-6a + 8-10；[1-00034](task/UDG/20.15.2/1-00034.md) LOCALLNSMODE=REDUNDANCY + FIRSTLNSIP/SECONDLNSIP 内置 |
| **本地配置负荷分担** | 本地配置 | LOAD_SHARING | IPv4 | 同上；[1-00034](task/UDG/20.15.2/1-00034.md) LOCALLNSMODE=LOAD_SHARING + ADD L2TPLNSINFO 扩展 >2 LNS |
| **本地配置 IPv6 LNS** | 本地配置 | REDUNDANCY（强制）| IPv4/IPv6 | 同上；[1-00034](task/UDG/20.15.2/1-00034.md) ADD L2TPLNSINFO LNSIPVER=IPV6 LNSNO=1/2（主备）|
| **本地配置多 LNS（>6）** | 本地配置 | LOAD_SHARING | IPv4 | 同上；[1-00034](task/UDG/20.15.2/1-00034.md) ADD L2TPGROUP 内置 6 + ADD L2TPLNSINFO 扩展到 30 |
| **AAA 下发主备**（方式B）| AAA Server Access-Accept | REDUNDANCY | IPv4 | 步骤 0-5 + 6b + 8-10；[0-00134](task/UDG/20.15.2/0-00134.md) ADD L2TPRDSCLIENT + SET APNL2TPATTR RDSLNSMODE=REDUNDANCY |
| **AAA 下发基于 Preference** | AAA Server Access-Accept | TUNNEL_PREFER | IPv4 | 同上；SET APNL2TPATTR RDSLNSMODE=TUNNEL_PREFER（Preference 不同为主备，相同为负荷分担）|
| **IPv6 用户接入**（v02+）| 本地/AAA 均可 | 同上 | IPv6 | SET APNL2TPATTR **SUPPORTIPV6=ENABLE**；UDG 20.6.0+ |
| **IPv6 隧道**（v03+）| 本地配置 | REDUNDANCY | IPv4/IPv6 | [1-00034](task/UDG/20.15.2/1-00034.md) ADD L2TPLNSINFO LNSIPVER=IPV6；UDG 20.12.0+ |
| **双栈降栈**（异常）| 同上 | 同上 | IPv4+IPv6 | 双栈协商单栈失败按地址来源决定降栈（双栈都 LNS 分配才允许降栈）|

> **关键规则**：
> - **L2TPSWITCH=ENABLE 是 APN 粒度核心使能**——SET APNL2TPATTR 必配
> - **方式A 指定 L2TPGROUPID**（指向 L2TP 组）/ **方式B 指定 RDSLNSMODE**（AAA 返回 LNS 工作模式）——SET APNL2TPATTR 二选一
> - **Byte671 bit7=1 必配**（关闭快速流表，否则报文丢弃）
> - **企业网必须部署 LNS**（应用限制硬约束）
> - **NP 加速场景不支持 IPv6**
> - **L2TP 用户不支持 PPP 用户接入类型**（仅 mobile 用户接入）
> - **不支持 DHCP 延迟分配 IPv4 / 不支持 IPv6 PD**（地址分配主体在 LNS）
> - **不支持 PFC/ACFC 压缩 / 不支持 NAT**
> - **N4 加密对称**：SET L2TPN4KEY（U 面）与对端 SMF SET L2TPKEY（C 面）必须同时 ENABLE 且密钥相同

## 约束

- **必须 License**（critical，rule-0-02050）：必须加载 License 控制项 82200BVC LKV3G5L2TP01 并通过 SET LICENSESWITCH 打开 — 后果：特性不工作
- **C-U 协同**（critical，rule-0-02051）：UDG 作 LAC 执行隧道封装+PPP 协商透传；UNC（WSFD-104410）决策+下发 LNS 参数；两侧版本对齐 20.3.2+ — 后果：单侧配置无效
- **企业网必须部署 LNS**（critical，rule-0-02052）：企业网侧必须部署 LNS 设备 — 后果：L2TP 隧道无法终结
- **地址分配主体在 LNS**（critical，rule-0-02053）：L2TP 用户地址由 LNS 分配（即使本地配置地址池或 AAA 下发地址）— 后果：地址分配失败或与本地池冲突
- **互斥 GWFD-010108**（critical，rule-0-02054）：L2TP VPN 不支持与用户面地址自动检测功能同时应用 — 后果：应用限制违反
- **互斥 GWFD-020482**（critical，rule-0-02055）：APN 下开启 L2TP VPN 则不支持入不转板功能 — 后果：应用限制违反
- **关闭快速流表**（critical，rule-0-02056）：必须 SET SOFTPARAOFBIT Byte671 bit7=1 关闭快速流表 — 后果：快速流表场景报文丢弃
- **不支持 PPP 用户接入**（warning，rule-0-02057）：L2TP 用户接入不支持 PPP 用户接入类型 — 后果：PPP 用户接入失败
- **不支持 DHCP 延迟分配 IPv4**（warning，rule-0-02058）：L2TP 用户接入时不支持 DHCP 延迟分配 IPv4 地址 — 后果：地址分配方式受限
- **不支持 IPv6 PD**（warning，rule-0-02059）：L2TP 用户接入时不支持 IPv6 Prefix Delegation 地址分配方式 — 后果：地址分配方式受限
- **不支持 PFC/ACFC 压缩**（warning，rule-0-02060）：不支持 UDG 与 LNS 协商 PFC 和 ACFC 字段压缩流程 — 后果：PPP 协商参数受限
- **不支持 NAT**（warning，rule-0-02061）：该功能不支持 NAT 地址转换 — 后果：NAT 场景不可用
- **NP 加速不支持 IPv6**（warning，rule-0-02062）：NP 加速场景下 L2TP VPN 特性不支持 IPv6 — 后果：NP 加速+IPv6 场景不可用
- **UDG-LNS 路由可达**（critical，rule-0-02063）：UDG 与 LNS 物理链路连通（PING 验证）— 后果：隧道 DOWN
- **L2TP Group 规格**（info，rule-0-02064）：本地 L2TP Group 规格 1500；AAA 下发方式 Group 规格 1500 — 后果：超规格无法新建
- **UDG 首发版本 20.3.2**（info，rule-0-02065）：UDG 20.3.2 首次发布；v02 20.6.0 支持 IPv6 用户接入；v03 20.12.0 隧道支持 IPv6 地址 — 后果：低版本不支持新功能
