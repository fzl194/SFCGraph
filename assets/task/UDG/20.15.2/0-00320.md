# 配置APN路由冗余上行报文隧道转发功能开关（SET APNREDUNDUPSW）

> 命令静态知识见 [SET APNREDUNDUPSW](command/UDG/20.15.2/SET-APNREDUNDUPSW.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

SET APNREDUNDUPSW 是"APN 级路由冗余上行报文隧道转发功能开关"命令: **仅备 UPF 可配**, 给指定 APN 配 IPv4/IPv6 上行报文隧道转发的使能开关, 用于静态地址用户路由冗余场景。命令核心配置空间有 3 个维度: **APN 引用** (必选)、**IPv4 上行转发开关** (IPV4SWITCH)、**IPv6 上行转发开关** (IPV6SWITCH)。1 个特性命中 (010107 静态地址用户路由冗余) 显式配 IPv4+IPv6 双开。

### 配置维度 1: APN 引用 (参数 APN, 必选)

| 参数 | 范围 | 备注 |
|---|---|---|
| **APN** | 字符串 1~63 | 引用已配的 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 实例名; 不支持空格及 `_`、`#`、`$`、`&` 等特殊字符; 不区分大小写; **每个 APN 仅一条本配置** |

### 配置维度 2: IPv4 上行转发开关 (参数 IPV4SWITCH, 可选)

控制该 APN 是否支持 IPv4 上行报文隧道转发。

| 参数 | 取值 | 作用 |
|---|---|---|
| **IPV4SWITCH** | ENABLE | 使能该 APN 的 IPv4 上行报文隧道转发功能 |
| | DISABLE (默认) | 不使能 |

### 配置维度 3: IPv6 上行转发开关 (参数 IPV6SWITCH, 可选)

控制该 APN 是否支持 IPv6 上行报文隧道转发。

| 参数 | 取值 | 作用 |
|---|---|---|
| **IPV6SWITCH** | ENABLE | 使能该 APN 的 IPv6 上行报文隧道转发功能 |
| | DISABLE (默认) | 不使能 |

> 010107 (静态地址用户路由冗余) 范式显式配 `IPV4SWITCH=ENABLE + IPV6SWITCH=ENABLE` (双栈使能); 系统初始值 IPV4SWITCH=DISABLE, IPV6SWITCH=DISABLE。

**典型脚本 (010107 静态地址用户路由冗余, 双栈上行转发使能)**:
```
SET APNREDUNDUPSW: APN="huawei.com", IPV4SWITCH=ENABLE, IPV6SWITCH=ENABLE;
```

> 命令 notes: 适用 NF: PGW-U、UPF; 执行后立即生效; 系统最多 10000 条; **该命令只能在备 UPF 才能配置** (主 UPF 配命令拒收); 系统初始记录 IPV4SWITCH=DISABLE, IPV6SWITCH=DISABLE。

**步骤位置**: 在所属 feature 编排中通常位于"步骤 N: 备 UPF 侧 APN 冗余上行配置"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN (建 APN 实例) + 全局 [0-00064](task/UDG/20.15.2/0-00064.md) SET REDUNDUSER (静态地址用户路由冗余全局开关) + [0-00055](task/UDG/20.15.2/0-00055.md) ADD REDUNDRDTIP (冗余备份重定向 IP) + [0-00059](task/UDG/20.15.2/0-00059.md) ADD GRETUNNEL (备份 GRE Tunnel) + [0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE (备份 IPv4 静态路由) + [0-00063](task/UDG/20.15.2/0-00063.md) ADD SRROUTE6 (备份 IPv6 静态路由); 010107 场景下与 [0-00313](task/UDG/20.15.2/0-00313.md) ADD OSPFINTERFACE (LoopBack 使能 OSPF) + [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS (Tunnel/LoopBack IPv4 地址) 协同。

## 决策点: 上行转发开关组合 (DP 0-00320)

问: APN 的 IPv4/IPv6 上行转发开关取哪种组合?

| 选项 | IPV4SWITCH | IPV6SWITCH | 典型场景 |
|---|---|---|---|
| 双栈使能 | ENABLE | ENABLE | 010107 静态地址用户路由冗余 (双栈冗余上行) |
| 仅 IPv4 | ENABLE | DISABLE | 纯 IPv4 静态地址用户路由冗余 |
| 仅 IPv6 | DISABLE | ENABLE | 纯 IPv6 静态地址用户路由冗余 |
| 系统默认 (未使能) | DISABLE | DISABLE | 非冗余场景初始记录 |

> 1 个特性命中 (010107) 显式配双栈使能。

## 约束

- **仅备 UPF 可配** (critical, rule-0-00320): 该命令只能在备 UPF 才能配置 — 后果: 主 UPF 配命令拒收
- **APN 前置依赖** (critical, rule-0-00320): APN 必须先由 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 建好 — 后果: 未建 APN 命令拒收
- **APN 唯一 (max_records=10000)** (critical, rule-0-00320): 系统最多支持 10000 条; 每个 APN 仅一条本配置 — 后果: 同 APN 重复配置或超规格命令拒收
- **执行后立即生效** (warning, rule-0-00320): 命令执行后立即生效 — 后果: 误操作即刻影响该 APN 的上行转发
- **全局冗余开关前置** (critical, rule-0-00320): 需先通过 [0-00064](task/UDG/20.15.2/0-00064.md) SET REDUNDUSER 使能静态地址用户路由冗余全局开关 — 后果: 全局开关未开配本命令路由冗余功能无效
- **适用 NF 限定** (warning, 隐含): PGW-U、UPF — 后果: SGW-U 等不适用
- **备 UPF 角色限定** (warning, 隐含): 仅备 UPF 可配 — 后果: 主 UPF 配置无效或拒收
