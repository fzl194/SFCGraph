# 离线计费（GWFD-010171）

> 特性静态知识见 [GWFD-010171](feature/UDG/20.15.2/GWFD-010171.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

离线计费是一种**非实时**后付费方式：UDG（SGW-U/PGW-U）按业务统计时长/流量/事件，经 N4 上报 UNC，UNC 生成话单通过 **Ga 接口**发送到 CG（话单网关），CG 标准化后送 BS 出账单。架构：UDG ↔(N4/PFCP)↔ UNC ↔(Ga)↔ CG ↔(FTP)↔ BS。

URR 取 `USAGERPTMODE=OFFLINE` + `OFFMETERINGTYPE`（VOLUME/TIME/EVENT/FREE/组合等 8 种统计维度）。**复用内容计费的通用 backbone**（计费三件套/过滤链/规则绑定/收尾），**与在线计费（[2-00004](task/UDG/20.15.2/2-00004.md)）的差异**：①URR 离线参数（USAGERPTMODE=OFFLINE + OFFMETERINGTYPE）；②**无需 OCS 实时配额**，不配 `SET UPDEFAULTQUOTA`；③刷新用 `REFRESHTYPE=USERPROFILE`（非 ALL）；④`SET URRFAILACTION` 在离线计费新业务场景下也可用（URR 上报失败放通）。

## 配置流程（步骤编排，每步显示含命令）

1. **License**（单命令，本特性无独立 License）：`SET LICENSESWITCH`（开内容计费 `LKV3G5BCBC01` + SA-Basic License；本特性自身无需 License）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **计费三件套**（费率定义，URR=OFFLINE）：`ADD URR`+`ADD URRGROUP`+`ADD PCCPOLICYGRP`（URR 的 `USAGERPTMODE=OFFLINE`+`OFFMETERINGTYPE`）→ [1-00010](task/UDG/20.15.2/1-00010.md)
3. **过滤链**（业务匹配）：`ADD FILTER`+`ADD FLOWFILTER`+`ADD FLTBINDFLOWF`（三四层匹配，离线计费典型用三四层）→ [1-00009](task/UDG/20.15.2/1-00009.md)
4. **规则与用户模板绑定**：`ADD RULE`（POLICYTYPE=PCC，绑 FLOWFILTER + PCCPOLICYGRP）+`ADD USERPROFILE`+`ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
5. **计费收尾**（缺省费率+刷新）：`SET URRGRPBINDING`（每 USERPROFILE 缺省费率）+`SET REFRESHSRV`（`REFRESHTYPE=USERPROFILE`，**注意与在线计费的 ALL 不同，以原始文档操作步骤为准**）→ [1-00012](task/UDG/20.15.2/1-00012.md)
6. **URR 上报失败动作**（单命令，可选）：`SET URRFAILACTION`（离线计费新业务场景，URR ID 在 UNC 不存在导致上报失败时，配 `RETRYFAILACT=CONTINUE` 放通业务不计费）→ [0-00018](task/UDG/20.15.2/0-00018.md)

> **注意**：离线计费**不需要** `SET UPDEFAULTQUOTA`（无 OCS 实时配额，不申请配额）。

## 决策点

**DP（特性级）：离线统计类型 OFFMETERINGTYPE**（影响 URR 计费维度）：

| 场景 | OFFMETERINGTYPE | 影响（统计维度/联动） |
|---|---|---|
| 按流量计费（默认） | VOLUME | 统计流量；不统计时长/事件 |
| 按时长计费 | TIME | 统计时长；不统计流量/事件 |
| 按事件计费 | EVENT | 统计事件；**对性能有一定影响**，执行前评估 |
| 流量+时长 | VOLUME_TIME | 统计流量和时长 |
| 事件+流量 | EVENT_VOLUME | 统计事件和流量；**性能有影响** |
| 事件+时长 | EVENT_TIME | 统计事件和时长；**性能有影响** |
| 事件+流量+时长（全量） | EVENT_VOLUME_TIME | 全量统计；**性能有影响**（原始文档任务示例取此值） |
| 免费（防欺诈） | FREE | 该 URR 有计费欺诈风险，需做防欺诈判断；实现免费需配合 UNC |

**联动**：
- OFFMETERINGTYPE → URR（[1-00010](task/UDG/20.15.2/1-00010.md)）的离线统计维度，前提是 `USAGERPTMODE=OFFLINE`。
- 含 EVENT 的取值（EVENT/EVENT_VOLUME/EVENT_TIME/EVENT_VOLUME_TIME）→ **性能影响**（warning），评估后使用。
- FREE → 需配合 UNC 实现免费 + 计费防欺诈判断。
- 计费方式选择（在线/离线接口）由 **UNC 侧** `ADD APNCHGMODE`（APN 粒度，高优先级）/`SET CHGMODE`（全局，低优先级）决定，**UDG 用户面 task wiki 不展开**，仅在此提示联动。

## 约束（特性级 rule）

- **License 依赖**（info）：本特性**无独立 License**；依赖内容计费基本功能 License `LKV3G5BCBC01`（GWFD-020301）+ SA-Basic（GWFD-110101）— 否则业务识别/计费基础不成立。
- **依赖特性**：[内容计费基本功能](feature/UDG/20.15.2/GWFD-020301.md)（GWFD-020301）+ [SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，业务识别）。
- **URR 离线一致性**（critical）：`URRID`/`USAGERPTMODE=OFFLINE`/`OFFMETERINGTYPE` 在 SGW-C/PGW-C 与 SGW-U/PGW-U 必须一致 — 不一致则计费行为异常。
- **USERPROFILE/RULE 一致性**（critical）：`USERPROFILENAME`/`RULENAME`/`POLICYTYPE`/`POLICYNAME` 在 PCRF/SGW-C/PGW-C/SGW-U/PGW-U 间必须一致 — 不一致则策略无法按预期生效。
- **刷新类型**（warning）：离线计费用 `SET REFRESHSRV:REFRESHTYPE=USERPROFILE`（非 ALL），使 RULE 配置生效 — 否则配置不生效。
- **REFRESHSRV 必须在 RULE 配置完成后执行**（critical）：在 [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定之后 — 否则配置不生效。
- **每个 USERPROFILE 必绑缺省费率**（critical）：`SET URRGRPBINDING` 的 `DFTURRGRPNAME`+`DFTSIGURRGNAME` — 否则未匹配费率的报文无法计费。
- **URRID 全网唯一**（critical）：建议从 1000 开始分配 — 重复则 URR 冲突。
- **ADD URR 记录数上限**（info）：最大 65000 条，超过规格 90% 上报告警 ALM-135602215。
- **仅绑三四层 filter 的 RULE 优先级应低于绑七层 filter 的 RULE**（warning）：数字越小优先级越高。
- **UPURRNAME1/2/3、DOWNURRNAME1/2/3 仅编号无优先级语义**（info）。
