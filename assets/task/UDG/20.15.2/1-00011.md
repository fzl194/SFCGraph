# 规则与用户模板绑定

> **跨域通用模块**（多命令可复用）：PCC 触发链收尾。计费/BWM/URL过滤共用。被引用于 [2-00002](task/UDG/20.15.2/2-00002.md)。

## 配置方法

三步：规则(绑流过滤器+策略组) → 用户模板 → 规则绑用户模板。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 业务规则 | ADD RULE | RULENAME + POLICYTYPE + FILTERINGMODE + FLOWFILTERNAME + PRIORITY + POLICYNAME |
| 用户模板 | ADD USERPROFILE | USERPROFILENAME |
| 绑定 | ADD RULEBINDING | USERPROFILENAME + RULENAME |

**典型脚本（URL过滤：POLICYTYPE=PCC 触发）**：
```
ADD RULE:RULENAME="rule_test",POLICYTYPE=PCC,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="flowfilter_test1",PRIORITY=1,POLICYNAME="ppg_test";
ADD USERPROFILE:USERPROFILENAME="up_test";
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="rule_test";
```

**步骤位置**：PCC 触发链收尾。承接 [1-00009](task/UDG/20.15.2/1-00009.md)（FLOWFILTERNAME）+ [1-00010](task/UDG/20.15.2/1-00010.md)（POLICYNAME=PCCPOLICYGRPNM）/ [1-00013](task/UDG/20.15.2/1-00013.md)（POLICYNAME=HEADERENNAME）；产出 USERPROFILENAME 供会话匹配。

## 场景差异

> 规则绑定被多特性复用（计费族 7 特性 + URL过滤 + 头增强族），差异集中在 **RULE.POLICYTYPE 决定 POLICYNAME 指向哪种对象**——这是跨特性关键差异。

### POLICYTYPE 变体（RULE.POLICYNAME 指向对象）

| POLICYTYPE | POLICYNAME 指向 | 是否经 PCCPOLICYGRP | 对应 feature |
|---|---|---|---|
| **PCC**（计费/带宽/URL过滤触发）| PCCPOLICYGRPNM（PCC 策略组）| 是（PCCPOLICYGRP 绑 URRGROUP）| [2-00002](task/UDG/20.15.2/2-00002.md) URL过滤、[2-00003](task/UDG/20.15.2/2-00003.md)~[2-00009](task/UDG/20.15.2/2-00009.md) 计费族 |
| **HEADEN**（头增强触发）| HEADERENNAME（HEADEN 对象，ADD HEADEN 定义）| **否（直接引用，PCC 策略组名称=NULL）**| [2-00010](task/UDG/20.15.2/2-00010.md) HTTP头增强、[2-00011](task/UDG/20.15.2/2-00011.md) RTSP头增强、[2-00012](task/UDG/20.15.2/2-00012.md) HTTPS头增强、[2-00013](task/UDG/20.15.2/2-00013.md) HTTP头防欺诈 |
| **SMARTREDIRECT**（重定向触发）| 重定向动作对象名（POLICYNAME 指向随特性不同）| **否（直接引用，不经 PCCPOLICYGRP）**| [2-00014](task/UDG/20.15.2/2-00014.md) 用户Portal（POLICYNAME=**IPFARMNAME**，IP Farm）、[2-00017](task/UDG/20.15.2/2-00017.md) HTTP智能重定向（POLICYNAME=**SMTHTTPREDINAME**）、[2-00016](task/UDG/20.15.2/2-00016.md) DNS纠错（POLICYNAME=**DNSOVERWRTNAME**）|
| **WEBPROXY**（Web代理触发）| IPFARMNAME（IP Farm，L3 IP NAT 重定向）| **否（直接引用）**| [2-00015](task/UDG/20.15.2/2-00015.md) WebProxy |

> 关键：HEADEN / SMARTREDIRECT / WEBPROXY 均为独立于 PCC 的 POLICYTYPE，RULE 经 POLICYNAME 直接引用对应动作对象（HEADERENNAME / IPFARMNAME / SMTHTTPREDINAME / DNSOVERWRTNAME），**不经 PCCPOLICYGRP**（调测输出 PCC 策略组名称=NULL）。SMARTREDIRECT 的 POLICYNAME 指向哪种对象由特性决定（Portal→IPFarm、HTTP重定向→SMARTHTTPREDIR、DNS纠错→DNSOVERWRITING）。这些特性的动作规则可与 PCC 计费规则**绑定同一 USERPROFILE + 共用同一 FLOWFILTER**（用户Portal/WebProxy activation 演示双 RULE：SMARTREDIRECT/WEBPROXY 重定向 + PCC 计费）。DNS纠错/HTTP智能重定向 activation 采用 PCRF 下发预定义规则，无本地 USERPROFILE/RULEBINDING。

### FILTERINGMODE 变体

| FILTERINGMODE | 含义 |
|---|---|
| FLOWFILTER（默认）| 单流过滤器 |
| FLOWFILTERGRP | 流过滤器组 |

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| 策略类型 POLICYTYPE=PCC（联动特性级） | POLICYNAME=PCCPOLICYGRPNM（经 PCCPOLICYGRP）；触发 PCC 体系含 URL过滤（URL过滤必选）；计费/带宽按场景 |
| 策略类型 POLICYTYPE=HEADEN（联动特性级） | POLICYNAME=HEADERENNAME（**直接引用 HEADEN 对象，不经 PCCPOLICYGRP，PCC 策略组名称=NULL**）；触发头增强 |
| 流过滤器选择 FILTERINGMODE=FLOWFILTER | 单流过滤器 |
| 流过滤器选择 FILTERINGMODE=FLOWFILTERGRP | 流过滤器组 |

## 约束

- **规则引用一致**（critical）：RULE.POLICYNAME 须与策略对象一致（PCC→PCCPOLICYGRPNM；HEADEN→HEADERENNAME）；FLOWFILTERNAME 须与 FLOWFILTER 一致 — 不一致则规则链断开。
- **USERPROFILENAME 跨网元一致**（critical）：须 PCF/SMF/UPF 间一致 — 不一致则会话匹配失败。
- **POLICYTYPE 选择正确**（critical）：POLICYTYPE 决定业务动作（PCC→计费/带宽/URL过滤；HEADEN→头增强；SMARTREDIRECT→重定向）— 选错策略类型则对应业务不触发。
- **RULENAME+POLICYTYPE 唯一**（warning）：存在多条数据时，RULENAME+策略类型不能完全相同 — 重复则新增失败。
