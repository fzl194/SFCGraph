# 设置Gy接口一次重定向参数（SET GYONESHOT）

> 命令静态知识见 [SET GYONESHOT](command/UDG/20.15.2/SET-GYONESHOT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET GYONESHOT 是"一次重定向"的全局激活命令（**全局唯一，最大记录数 1**）：把前置配好的 EXTFLTNAME（URL 白名单）与 APPENDINFONAME（携带信息）绑定在一起，匹配白名单的报文被重定向到 OCS 下发的 URL。命令核心配置空间有 2 个维度：**前置引用绑定**（EXTFLTNAME + APPENDINFONAME 两个外部对象引用，必选其一即可，二者全配形成完整重定向动作）+ **重定向完成时间**（REDIRAGETIME，可选时间窗口）。立即生效，不需 REFRESHSRV；本命令是 020356（EXTENDEDFILTER → REDIRAPPENDINFO → SET GYONESHOT）三件套的**收尾激活**。

### 配置维度 1：前置引用绑定（参数 EXTFLTNAME / APPENDINFONAME，条件必选）

| 参数 | 引用对象 | 来源 atom | 作用 |
|---|---|---|---|
| **EXTFLTNAME** | 一次重定向的 URL 白名单 | 引用 [0-00022](task/UDG/20.15.2/0-00022.md)（ADD EXTENDEDFILTER）的 EXTFLTNAME | 决定哪些 URL 触发一次重定向 |
| **APPENDINFONAME** | 重定向携带的用户信息 | 引用 [0-00023](task/UDG/20.15.2/0-00023.md)（ADD REDIRAPPENDINFO）的 APPENDINFONAME | 决定重定向 URL 中携带哪些用户字段 |

> 两个参数独立 optional；典型 020356 场景同时配（URL 白名单 + 携带 IMSI）。

### 配置维度 2：重定向完成时间窗口（参数 REDIRAGETIME，可选）

| 取值 | 作用 |
|---|---|
| 整数 5~600（默认 5） | 重定向完成时间（秒）；不配即默认 5 秒 |

**典型脚本（020356 双引用绑定）**：
```
SET GYONESHOT:EXTFLTNAME="gy-oneshot-url",APPENDINFONAME="testredirappendinfo";
```

> 命令 notes：最大记录数 1（全局唯一）；适用 NF：PGW-U、UPF；立即生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：一次重定向激活"——前置 ADD EXTENDEDFILTER(0-00022) → ADD REDIRAPPENDINFO(0-00023)，本命令收尾。

## 约束

- **EXTFLTNAME 与 APPENDINFONAME 必须已配置**（critical，rule-0-00334）：EXTFLTNAME 引用前置 ADD EXTENDEDFILTER(0-00022)；APPENDINFONAME 引用前置 ADD REDIRAPPENDINFO(0-00023) — 后果：未配前置扩展过滤器或携带信息时一次重定向不生效
- **全局唯一 / 最大记录数 1**（critical，rule-0-00334）：本命令全局仅一条配置 — 后果：重复执行将覆盖或拒绝（按 MML 语义）
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
- **REDIRAGETIME 范围约束**（warning，隐含）：必须 5~600 整数；超出范围拒收 — 后果：参数越界失败
