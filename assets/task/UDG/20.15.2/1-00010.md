# 计费三件套

> **跨域通用 backbone**（对齐 ConfigTask 计费 1-00001）：费率策略组三级链。计费/BWM/URL过滤共用（URL过滤用简化 URR 作计费属性）。被引用于 [2-00002](task/UDG/20.15.2/2-00002.md)。

## 配置方法

三级链：URR → URRGROUP（绑URR）→ PCCPOLICYGRP（绑URRGROUP）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 使用量上报规则 | ADD URR | URRNAME + URRID + USAGERPTMODE |
| URR 组 | ADD URRGROUP | URRGROUPNAME + UPURRNAME1/DOWNURRNAME1 |
| PCC 策略组 | ADD PCCPOLICYGRP | PCCPOLICYGRPNM + URRGROUPNAME |

**典型脚本（URL过滤简化：单一离线 URR 作计费属性）**：
```
ADD URR:URRNAME="urr_2",URRID=2,USAGERPTMODE=OFFLINE;
ADD URRGROUP:URRGROUPNAME="urrgroup1",UPURRNAME1="urr_2",DOWNURRNAME1="urr_2";
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="ppg_test",URRGROUPNAME="urrgroup1";
```

**步骤位置**：承接 [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链）；产出 PCCPOLICYGRPNM 供下游 RULE.POLICYNAME 引用。

## 场景差异

- **URL过滤**：简化单一离线 URR（仅作计费属性，USAGERPTMODE=OFFLINE）。
- **计费场景**：多 URR（在线/离线/融合），见计费特性。
- **BWM**：URR 关联带宽控制（见 GWFD-110311）。

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| 计费方式 USAGERPTMODE=OFFLINE（联动特性级） | 离线计费；URL过滤通常用此 |
| 计费方式 USAGERPTMODE=ONLINE（联动特性级） | 在线计费 |

## 约束

- **URRID 全网唯一**（critical）：从 1000 起始 — 重复则 URR 冲突。
- **URRGROUP 引用一致**（critical）：URRGROUP.UPURRNAME1/DOWNURRNAME1 须与 URR.URRNAME 一致 — 不一致则组引用断链。
- **PCCPOLICYGRP 引用一致**（critical）：PCCPOLICYGRP.URRGROUPNAME 须与 URRGROUP 一致 — 不一致则策略组引用断链。
