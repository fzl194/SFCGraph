# IPv6承载基础设施（License+VPN+IPv6地址族）

> 归属 [2-00040](task/UDG/20.15.2/2-00040.md) IPv6承载上下文（GWFD-020401）。本 compound 配置**IPv6 承载基础设施层**——License 能力使能 + 逻辑口 VPN + 外联口 L3VPN + IPv6 地址族激活，为 IPv6 PDP上下文/EPS承载/PDU Session/QoS Flow 的激活/去激活/更新提供承载。被引用于 {2-00040}。

> **★ 关键约束**：本骨架**逻辑口 VPN 与外联口 VPN 必须同名**（文档操作步骤2"说明"原文警示）——逻辑接口的 VPN 一定要与其对应的外联口的 VPN 相同，否则从逻辑接口发出去的报文会查询路由失败。

> **★ 关键约束**：外联口 VPN 必须配 `AFTYPE=ipv6uni` IPv6 地址族（IPv6 承载必备）——否则 IPv6 承载无法工作。

> **★ 与 020403 / 020406 联动**：本骨架是 020401 的基础设施；020403（IPv4v6 双栈）和 020406（IPv6 PD）的"承载部分"完全复用本骨架，差异仅在地址族使能与上层业务配置。

## 配置方法

本 compound 由 4 条命令组成（多命令可复用模块，建 compound 承载）：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 打开本特性的 License 配置开关 | [SET LICENSESWITCH](task/UDG/20.15.2/0-00019.md) | `LICITEM="LKV3G5V6PB01"` / `SWITCH=ENABLE` |
| 2. 配置逻辑口 VPN 实例 | [ADD VPNINST](task/UDG/20.15.2/0-00041.md) | `VPNINSTANCE`（本端规划，**必须与外联口 VPN 同名**） |
| 3. 配置外联口 L3VPN 实例 | [ADD L3VPNINST](task/UDG/20.15.2/0-00071.md) | `VRFNAME`（本端规划，与逻辑口 VPNINSTANCE 同名） |
| 4. 配置外联口 VPN 的 IPv6 地址族 | [ADD VPNINSTAF](task/UDG/20.15.2/0-00072.md) | `VRFNAME` / `AFTYPE=ipv6uni`（**★ IPv6 承载必备**） |

**典型脚本（020401 场景：IPv6 承载上下文基础设施）**：
```
SET LICENSESWITCH: LICITEM="LKV3G5V6PB01", SWITCH=ENABLE;
ADD VPNINST: VPNINSTANCE="VPN_Internet";
ADD L3VPNINST: VRFNAME="VPN_Internet";
ADD VPNINSTAF: VRFNAME="VPN_Internet", AFTYPE=ipv6uni;
```

**典型脚本（020406 IPv6 PD 场景：PD 沿用本骨架 + 后续 PD 专用地址池）**：
```
SET LICENSESWITCH: LICITEM="LKV3G5V6PB01", SWITCH=ENABLE;
ADD VPNINST: VPNINSTANCE="vpn1";
ADD L3VPNINST: VRFNAME="vpn1";
ADD VPNINSTAF: VRFNAME="vpn1", AFTYPE=ipv6uni;
```

## 场景差异

| 场景 | License | 逻辑口/外联口 VPN 名 | AFTYPE | 备注 |
|---|---|---|---|---|
| IPv6 单栈承载（020401 范式） | LKV3G5V6PB01 | VPN_Internet | ipv6uni | 020401 典型 |
| IPv6 + IPv6 PD 联合部署（020401 + 020406） | LKV3G5V6PB01（020401）+ LKV3G5P6PD01（020406） | vpn1 | ipv6uni | 双 License 都需使能；020406 在 020401 基础上叠加 PD |
| IPv4v6 双栈（020401 + 020403） | LKV3G5V6PB01（020401）+ LKV3G5VDSA01（020403） | 通常 VPN_Internet | ipv6uni（020401 配）+ 020403 通过 APN.HASVPNIPV6 承担 | 020403 激活脚本无 ADD VPNINSTAF，地址族使能由本骨架承载 |
| IPv4 单栈（GWFD-010105 等用户面地址分配） | 无 License | 各自规划 | ipv4uni | 010105 不配 AFTYPE=ipv6uni |

> **★ 复用差异双向回填**：
> - **020406 双 License 场景**必须使能 LKV3G5V6PB01（020401）后再使能 LKV3G5P6PD01（020406）——文档依赖关系明确
> - **020403 联动场景**的 IPv6 地址族**由本骨架使能**（020403 激活脚本不含 ADD VPNINSTAF），不是由 020403 承担——避免双重配置
> - **逻辑口/外联口 VPN 同名**是 020401 文档操作步骤2"说明"原文强制约束，不限于本骨架范围
> - **AFTYPE 必配 ipv6uni**——IPv6 承载场景无 ipv4uni 可选

## 约束

- **License 强制要求**（critical，rule-0-01601）：必须加载 License 控制项 82209828 LKV3G5V6PB01 并通过 SET LICENSESWITCH 打开开关 — 后果：特性不可用
- **逻辑口 vs 外联口 VPN 一致**（critical，rule-0-01601）：逻辑接口的 VPN 一定要与其对应的外联口的 VPN 相同（文档操作步骤2"说明"原文） — 后果：逻辑接口报文查询路由失败
- **AFTYPE=ipv6uni 必配**（critical，rule-0-01601）：外联口 VPN 必须配置 AFTYPE=ipv6uni IPv6 地址族 — 后果：IPv6 承载无法工作
- **执行顺序铁律**（critical，rule-0-01601）：本骨架是 020401 配置的**第 1-4 步**（打开 License + 建 VPN + 使能 IPv6 地址族），之后才配路由策略/路由发布/承载建立 — 后果：未开 License 直接配业务命令失败
- **VPNINSTANCE 全网唯一**（critical，rule-0-01601）：不同 VPN 实例之间 VPNINSTANCE 不能重复 — 后果：重名 ADD 失败
- **VPNINSTANCE 与平台 L3VPNINST 命名一致**（critical，rule-0-01601）：ADD VPNINST 创建的 VPN 命名必须与 ADD L3VPNINST 的 VRFNAME 一致 — 后果：命名不一致业务不通
