# 绑定PCC策略组业务属性（ADD SRVPBINDPCCPG）
> 命令静态知识见 [ADD SRVPBINDPCCPG](command/UDG/20.15.2/ADD-SRVPBINDPCCPG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

把已建成的 PCC策略组、业务属性、（可选）动作属性、（可选）URR组 串成一条"非默认策略组合"。两个必选 reference + 两个可选 reference，全部从已配置数据中获取。

| 参数 | 必选 | 引用对象（source task） | 取值样例 | 何时配 |
|---|---|---|---|---|
| PCCPOLICYGRPNM | required | [0-00003 配置PCC策略组](0-00003.md) 的 PCCPOLICYGRPNM | pg_test | 总是 |
| SRVPROPNAME | required | [0-00029 配置业务属性](0-00029.md) 的 SRVPROPNAME | sp_test | 总是 |
| PCCACTPROPNAME | conditional | [0-00028 配置PCC动作属性](0-00028.md) 的 PCCACTPROPNAME | ap_test | 需要阻塞/转发门控动作时 |
| URRGROUPNAME | conditional | [0-00002 配置URR组](0-00002.md) 的 URRGROUPNAME | — | 需要计费时 |

脚本样例（IPv6 SA：PCC 策略组 + 业务属性 + 阻塞动作属性）：
```
ADD SRVPBINDPCCPG:PCCPOLICYGRPNM="pg_test",SRVPROPNAME="sp_test",PCCACTPROPNAME="ap_test";
```

配置要点：
- 一个 PCC 策略组最多 10 条非默认组合；多条组合时匹配优先级由各组合自身决定。
- 全 optional 时即为"空组合"。扩展属性 EXTENDPROPNAME / Session级FUP累计 FUPSESSIONEXC 三特性场景未用，冷门不配。

## 约束
- max 100000 条 / 每 PCC 策略组 10 组合 / 立即生效 · warning · 超限或超组合数新增失败
- 前置依赖：必须先建成 PCCPOLICYGRPNM(0-00003) 与 SRVPROPNAME(0-00029)；可选 PCCACTPROPNAME(0-00028)、URRGROUPNAME(0-00002) · warning · 未建引用失败
