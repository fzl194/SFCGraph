# 添加位置区域地址分配用户白名单（ADD ADRLOCWHITELST）

> 命令静态知识见 [ADD ADRLOCWHITELST](command/UDG/20.15.2/ADD-ADRLOCWHITELST.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典",**不**逐特性罗列取值。该命令仅 1 个必选参数 MSISDN,配置空间极简;**适用 NF 限定 PGW-U/UPF**,与控制面无关;命令执行后**只对新激活用户生效**(对已激活用户不生效)。

## 配置方法(配置方法字典)

ADD ADRLOCWHITELST 是"位置区域地址分配用户白名单"命令,适用于 PGW-U/UPF。作用:按用户 MSISDN 加入白名单,使该用户跳过位置区域分配地址的开关限制(IPALLOCBYLOCSW/IPALLOCBYLOCGLBSW 等)。核心配置空间极简,**仅 1 个维度**:用户标识(MSISDN)。2 个特性命中(GWFD-020406 IPv6 PD 52579205 + GWFD-020421 基于位置分配地址 02846773)共用同一范式(`MSISDN="123456"`)。

### 配置维度 1:用户标识(参数 MSISDN,必选)

| 取值形式 | 取值范围 | 字符约束 | 典型场景 |
|---|---|---|---|
| **字符串 1~15 位纯数字** | 整数 0~9 字符组合 | 长度 1~15,每位 0~9,MSISDN 号不能为 19 | GWFD-020406 + GWFD-020421 范本 `MSISDN="123456"` |

> **MSISDN 号组成**:1) 用户注册国家的 Country Code (CC);2) 国家移动号 = National Destination Code (NDC) + Subscriber Number (SN)。
>
> **注意事项**(命令 notes 投影):
> - 该命令执行后**只对新激活用户生效**——已激活用户不生效(对在网用户改白名单无效)。
> - 该命令最大记录数为 20(整机白名单上限)。
> - MSISDN 必须为十进制数字,但**不能为 19**(特殊值,命令拒收)。

**典型脚本(GWFD-020406 IPv6 Prefix Delegation + GWFD-020421 基于位置分配地址 共用范式)**:
```
ADD ADRLOCWHITELST: MSISDN="123456";
```

**步骤位置**:在所属 feature 编排中位于"位置分配地址前置白名单"——前置 无;下游 触发位置区域地址分配流程([0-00303](task/UDG/20.15.2/0-00303.md) 类 APN 域命令按位置分配地址)前先把指定 MSISDN 用户加入白名单。

## 决策点

(无。本命令在所有特性下用法一致——仅 1 个必选参数 MSISDN,无特性分叉。)

## 约束

- **适用 NF 限定 PGW-U/UPF**(critical,rule-0-00299):本命令仅适用于 PGW-U/UPF,控制面 SMF 等网元不适用 — 后果:非 PGW-U/UPF 网元命令拒收
- **最大记录数 20**(critical,rule-0-00299):整机白名单最大 20 条记录 — 后果:超规格新增失败
- **执行后只对新激活用户生效**(critical,rule-0-00299):该命令执行后**只对新激活用户生效** — 后果:已激活用户改白名单无效(用户需重激活或拔号重拨)
- **MSISDN 必须为十进制数字**(critical,rule-0-00299):MSISDN 由 0~9 数字组成 — 后果:含非数字字符命令拒收
- **MSISDN 不能为 19**(critical,rule-0-00299):MSISDN 取值不能为字符串"19"(特殊保留值) — 后果:命令拒收
- **MSISDN 长度 1~15 位**(warning,rule-0-00299):字符串长度范围为 1~15 — 后果:超长/超短命令拒收
- **命令无修改/无查询变体**(info,rule-0-00299):本命令仅 ADD/LST/RMV,无 MOD 变体(白名单需先 RMV 再 ADD) — 后果:修改走 RMV+ADD 流程
