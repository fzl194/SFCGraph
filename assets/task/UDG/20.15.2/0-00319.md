# 基于APN配置地址分配规则（SET APNIPALLOCRULE）

> 命令静态知识见 [SET APNIPALLOCRULE](command/UDG/20.15.2/SET-APNIPALLOCRULE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

SET APNIPALLOCRULE 是"APN 级地址分配规则"命令: 给指定 APN 配独立 IPv4/IPv6 三级优先级规则, 覆盖全局 [0-00250](task/UDG/20.15.2/0-00250.md) SET IPALLOCRULE 的默认规则。系统初始 INHERIT (继承全局), 设置 ALLOCATTR=LOCAL 后启用本地规则, 按 APN/DNN × LOCATION × SMF 位域组合匹配地址范围。命令核心配置空间有 4 个维度: **APN 引用** (必选)、**分配属性** (ALLOCATTR/IPV6ALLOCATTR INHERIT/LOCAL)、**IPv4 三级规则** (FIRST/SECOND/THIRDRULESW + 规则内容)、**IPv6 三级规则** (IPV6FIRST/SECRULESW/IPV6SECONDRULE/IPV6THIRDRULESW/IPV6THIRDRULE)。1 个特性命中 (010105 基于 RADIUS 下发地址池名) 显式配 IPv4 LOCAL + APN 单维度 + 二三级 DISABLE。

### 配置维度 1: APN 引用 (参数 APN, 必选)

| 参数 | 范围 | 备注 |
|---|---|---|
| **APN** | 字符串 1~63 | 引用已配的 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 实例名; 字符集 `a-zA-Z0-9-.` (`.` 不可开头, 不连续); 区分大小写; **每个 APN 仅一条本配置** |

### 配置维度 2: 分配属性 (参数 ALLOCATTR/IPV6ALLOCATTR, 可选)

控制该 APN 是否使用本地 IPv4/IPv6 分配规则, 默认 INHERIT 继承全局。

| 参数 | 取值 | 作用 |
|---|---|---|
| **ALLOCATTR** | INHERIT (默认) | 继承 [0-00250](task/UDG/20.15.2/0-00250.md) SET IPALLOCRULE 的全局 IPv4 规则 |
| | LOCAL | 使用本 APN 配置的 IPv4 三级规则 |
| **IPV6ALLOCATTR** | INHERIT (默认) | 继承全局 IPv6 规则 |
| | LOCAL | 使用本 APN 配置的 IPv6 三级规则 |

> 010105 范式显式配 `ALLOCATTR=LOCAL` (IPv4 用本地规则); IPv6 默认 INHERIT。

### 配置维度 3: IPv4 三级规则 (条件可选, 前提 ALLOCATTR=LOCAL)

每级规则有"开关 + 规则内容"对 (开关 ENABLE 时内容为必选)。

| 层级 | 开关参数 | 规则内容参数 | 取值/格式 |
|---|---|---|---|
| 第一级 | **FIRSTRULESW** | **FIRSTRULE** | ENABLE / DISABLE; 内容为位域 `APN-{0|1}&LOCATION-{0|1}&SMF-{0|1}` (1=按该维度) |
| 第二级 | **SECONDRULESW** | **SECONDRULE** | ENABLE / DISABLE; 内容为位域 (FIRSTRULESW=ENABLE 时本参数条件可选) |
| 第三级 | **THIRDRULESW** | **THIRDRULE** | ENABLE / DISABLE; 内容为位域 (SECONDRULESW=ENABLE 时本参数条件可选) |

> **位域"与"关系**: 所选条件同时满足才命中, 从映射的地址范围内分配地址。**不同优先级的规则不可配置为相同** (避免分配歧义)。

### 配置维度 4: IPv6 三级规则 (条件可选, 前提 IPV6ALLOCATTR=LOCAL)

IPv6 与 IPv4 三级规则**完全并行**, 命名以 `IPV6` 前缀区分。

| 层级 | 开关参数 | 规则内容参数 | 取值/格式 |
|---|---|---|---|
| 第一级 | **IPV6FIRSTRULESW** | **IPV6FIRSTRULE** | ENABLE / DISABLE; 位域格式同 IPv4 |
| 第二级 | **IPV6SECRULESW** | **IPV6SECONDRULE** | ENABLE / DISABLE; 位域格式同 IPv4 |
| 第三级 | **IPV6THIRDRULESW** | **IPV6THIRDRULE** | ENABLE / DISABLE; 位域格式同 IPv4 |

**典型脚本 (010105 基于 RADIUS 下发地址池名, APN 单维度 + 二三级 DISABLE)**:
```
SET APNIPALLOCRULE: APN="apn-test1",ALLOCATTR=LOCAL,FIRSTRULESW=ENABLE,FIRSTRULE=APN-1&LOCATION-0&SMF-0,SECONDRULESW=DISABLE,THIRDRULESW=DISABLE;
```

> 命令 notes: 适用 NF: PGW-U、UPF; 执行后立即生效; 系统最多 10000 条; 支持三级优先级规则; 不配置时 ALLOCATTR 默认为 INHERIT (继承全局); 全局配置默认为第一优先级开关使能 (基于 APN 映射); 不同优先级规则不可相同; 三级优先级独立配置 (低优先级可不配高优先级); **若按 SMF 映射分配, 需先通过 SET IPALLOCBYSMFSW 设置开关 + LST IPALLOCBYSMFSW 校验**; **若按 LOCATION 映射分配, 需先通过 SET IPALLOCBYLOCSW 设置开关 + LST IPALLOCBYLOCSW 校验**; **禁止下发 ALLOCATTR=LOCAL 但不下发分配规则的配置**; 系统初始记录 ALLOCATTR=INHERIT, IPV6ALLOCATTR=INHERIT。

**步骤位置**: 在所属 feature 编排中通常位于"步骤 2: APN 地址规则配置"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN (建 APN 实例) + 全局 [0-00250](task/UDG/20.15.2/0-00250.md) SET IPALLOCRULE (或依赖它做兜底); 下游 [0-00047](task/UDG/20.15.2/0-00047.md) ADD POOLGRPMAP (地址池组与 APN 映射); 前置依赖 (按 SMF 维度) [0-00324](task/UDG/20.15.2/0-00324.md) SET IPALLOCBYSMFSW; 前置依赖 (按 LOCATION 维度) [0-00322](task/UDG/20.15.2/0-00322.md) SET IPALLOCBYLOCSW; 与 [0-00046](task/UDG/20.15.2/0-00046.md) SET APNADDRESSATTR (APN 级地址属性, 同 APN 级配合使用) + [0-00318](task/UDG/20.15.2/0-00318.md) SET ADDRESSATTR (全局地址属性, 全局级) 协同。

## 决策点: APN 级规则 vs 全局规则 作用域 (DP 0-00319)

问: APN 的地址分配走 APN 级规则还是全局 [0-00250](task/UDG/20.15.2/0-00250.md) 规则?

| 选项 | 优先级 | ALLOCATTR | 含义 |
|---|---|---|---|
| APN 级规则生效 | 优先 | LOCAL | 本 APN 按 ALLOCATTR/IPV6ALLOCATTR=LOCAL 用本地三级规则 |
| 继承全局规则 | 兜底 | INHERIT (默认) | 本 APN 用 [0-00250](task/UDG/20.15.2/0-00250.md) SET IPALLOCRULE 的全局规则 |

> 010105 范式显式配 IPv4 LOCAL (按 APN 单维度分配), IPv6 INHERIT (走全局)。

## 决策点: 位域维度组合 (DP 0-00127, 复用)

`FIRSTRULE` 的位域格式 `APN-{0|1}&LOCATION-{0|1}&SMF-{0|1}` 取哪种组合? 决定按哪些维度匹配地址范围。

| 选项 | FIRSTRULE | 含义 |
|---|---|---|
| 仅 APN/DNN | `APN-1&LOCATION-0&SMF-0` | 按 APN/DNN 分配 (最常用, 010105 范式) |
| 仅 LOCATION | `APN-0&LOCATION-1&SMF-0` | 按位置分配 (020406/020421 场景) |
| 仅 SMF | `APN-0&LOCATION-0&SMF-1` | 按 SMF 分配 (010105 子场景) |
| APN+SMF 叠加 | `APN-1&LOCATION-0&SMF-1` | 双维度叠加 (次优放 SECONDRULE) |
| APN+LOCATION 叠加 | `APN-1&LOCATION-1&SMF-0` | 双维度叠加 (次优放 SECONDRULE) |

各维度按"与"关系组合匹配, 需同时满足才命中。

## 约束

- **APN 前置依赖** (critical, rule-0-00319): APN 必须先由 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 建好 — 后果: 未建 APN 命令拒收
- **APN 唯一 (max_records=10000)** (critical, rule-0-00319): 系统最多支持 10000 条; 每个 APN 仅一条本配置 — 后果: 同 APN 重复配置或超规格命令拒收
- **执行后立即生效** (warning, rule-0-00319): 命令执行后立即生效 — 后果: 误操作即刻影响该 APN 用户
- **不同优先级规则不可相同** (critical, rule-0-00319): 不同优先级的规则不可以配置为相同 — 后果: 配置违反命令拒收
- **三级优先级独立配置** (warning, rule-0-00319): 低优先级可不配高优先级, 但同一级规则内开关与内容必须配对 — 后果: 规则开关 ENABLE 不配规则内容命令拒收
- **SMF 维度前置依赖** (critical, rule-0-00319): 若按 SMF 映射分配地址, 需通过 [0-00324](task/UDG/20.15.2/0-00324.md) SET IPALLOCBYSMFSW 设置基于 SMF 分配地址的开关 — 后果: 未开 SMF 维度开关配 SMF 规则命令拒收
- **LOCATION 维度前置依赖** (critical, rule-0-00319): 若按 LOCATION 映射分配地址, 需通过 [0-00322](task/UDG/20.15.2/0-00322.md) SET IPALLOCBYLOCSW 设置基于位置区分配地址的开关 — 后果: 未开 LOCATION 维度开关配 LOCATION 规则命令拒收
- **禁下发 LOCAL 不下发规则** (critical, rule-0-00319): 禁止下发分配属性为 LOCAL 但不下发分配规则的配置 — 后果: 分配规则缺失致地址分配失败
- **适用 NF 限定** (warning, 隐含): PGW-U、UPF — 后果: SGW-U 等不适用
- **条件可选约束** (warning, 隐含): FIRSTRULE 仅 FIRSTRULESW=ENABLE 时必选; SECONDRULE 仅 SECONDRULESW=ENABLE 时必选; 依此类推 — 后果: 开关与内容未对齐命令拒收
