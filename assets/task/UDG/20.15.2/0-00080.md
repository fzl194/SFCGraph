# 配置OSPFv3引入外部路由(IPv6)（ADD OSPFV3IMPORTROUTE）

> 命令静态知识见 [ADD OSPFV3IMPORTROUTE](command/UDG/20.15.2/ADD-OSPFV3IMPORTROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令是 OSPFv3 backbone 3 件套之三**——必须在 [0-00078](task/UDG/20.15.2/0-00078.md) ADD OSPFV3 创建 PROCID 进程之后执行；将外部路由引入 OSPFv3 域并在域中发布为 AS External LSA。

## 配置方法（配置方法字典）

ADD OSPFV3IMPORTROUTE 是"**将外部路由引入 OSPFv3 域**"命令。PROCID 必选；TOPOID 固定 0；PROTOCOL 必选（5 种协议类型，含 IPv6 专属的 `ospfv3`）。核心配置空间有 4 个维度：**进程号**（PROCID 引用上游）、**拓扑标识**（TOPOID 固定 0）、**协议类型**（PROTOCOL 5 选 1 enum）、**引入路由参数化**（IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG 三开关 + COST/TAG/TYPE + 路由策略 ROUPOLINAME + INHERITCOSTCFG 继承原开销）。4 个特性命中（GWFD-010155/020401/020403/020406）覆盖 2 种发布模式（全量发布 / 路由策略发布）。

### 配置维度 1：进程号（参数 PROCID，必选，引用上游）

| 参数 | 取值范围 | 数据来源 | 关联 |
|---|---|---|---|
| **PROCID** | 1~4294967295 | ref [0-00078](task/UDG/20.15.2/0-00078.md) ADD OSPFV3 已建进程 | 必须已存在，使用 LST OSPFV3 查看 |

> **关键约束**：**必须在执行 ADD OSPFV3 配置了 OSPFv3 进程后才能使用此命令**。

### 配置维度 2：拓扑标识（参数 TOPOID，可选，固定 0）

| 参数 | 取值范围 | 默认值 | 备注 |
|---|---|---|---|
| **TOPOID** | 0（目前仅支持默认拓扑 0） | 0 | 3 特性脚本均显式传入 TOPOID=0，按脚本实际用到提为 fixed |

### 配置维度 3：协议类型（参数 PROTOCOL，必选，5 选 1 enum，IPv6）

PROTOCOL 决定引入什么类型的外部路由（OSPFv3 场景下含 IPv6 专属的 `ospfv3`）：

| 取值 | 协议类型 | 典型场景 |
|---|---|---|
| **direct** | 直连路由 | 引入直连 IPv6 网段到 OSPFv3 域 |
| **static** | 静态路由 | 引入 IPv6 静态路由 |
| **bgp** | BGP 路由 | 引入 BGP 路由（PERMITIBGPCFG 控制） |
| **ospfv3** | OSPFv3 路由 | OSPFv3 多进程间路由引入（PROTOCOLPROCID 选源进程号） |
| **wlr** | 无线路由 | 主流场景：GWFD-010155/020401/020406 IPv6 全量发布 |

> **PROTOCOLPROCID**：PROTOCOL=ospfv3 时可选配引入路由协议的源进程号（默认 1）。**采用路由策略方式发布路由时，必须选 `WLR` 方式发布所有路由**（不能只发布 WLR_SP 或 WLR_UD），否则发布路由不全（GWFD-020401 md 说明）。

### 配置维度 4：引入路由参数化（IMPT*CFG 开关 × COST/TAG/TYPE）

| 子维度 | 开关参数 | 实际值参数 | 取值范围 | 联动 |
|---|---|---|---|---|
| **Cost** | IMPTCOSTCFG（FALSE/TRUE） | COST | 0~16777214 | IMPTCOSTCFG=TRUE 时 COST 条件必选 |
| **Tag** | IMPTTAGCFG（FALSE/TRUE） | TAG | 0~4294967295 | IMPTTAGCFG=TRUE 时 TAG 条件必选 |
| **Type** | IMPTTYPECFG（FALSE/TRUE） | TYPE | Type1（外部路由类型 1）/ Type2（外部路由类型 2） | IMPTTYPECFG=TRUE 时 TYPE 条件必选 |

> 3 个 CFG 默认值均为 FALSE（=不配置，等价于系统计算）。**INHERITCOSTCFG**（TRUE=继承原路由开销值，默认 FALSE）——**INHERITCOSTCFG 不能与 IMPTCOSTCFG 同时配置为 TRUE**。

### 配置维度 5：路由策略（参数 ROUPOLINAME，可选）

| 参数 | 取值范围 | 备注 |
|---|---|---|
| **ROUPOOLNAME** | 字符串 1~200 字符（区分大小写） | 走路由策略发布（GWFD-020401 配 `policy1`）；不配走全量发布 |

> **PERMITIBGPCFG**（PROTOCOL=bgp 时条件可选，默认 FALSE）——是否使能引入 IBGP 路由。

**典型脚本（010155 PGW-U→ePDG IPv6 动态路由，全量发布）**：
```
ADD OSPFV3IMPORTROUTE: PROCID=6, TOPOID=0, PROTOCOL=wlr, IMPTCOSTCFG=FALSE, IMPTTAGCFG=FALSE, IMPTTYPECFG=FALSE;
```

**典型脚本（020401 IPv6 承载上下文，路由策略发布）**：
```
ADD OSPFV3IMPORTROUTE: PROCID=100, TOPOID=0, PROTOCOL=wlr, ROUPOLINAME="policy1";
```

**典型脚本（020406 IPv6 Prefix Delegation，全量发布精简脚本）**：
```
ADD OSPFV3IMPORTROUTE: PROCID=6, TOPOID=0, PROTOCOL=wlr;
```

**典型脚本（OSPFv3 进程间路由引入）**：
```
ADD OSPFV3IMPORTROUTE: PROCID=1, TOPOID=0, IMPTCOSTCFG=FALSE, IMPTTAGCFG=FALSE, IMPTTYPECFG=FALSE, PROTOCOL=ospfv3, PROTOCOLPROCID=2;
```

> 命令 notes：**最大记录数 8000**；**执行后立即生效**；**必须在 ADD OSPFV3 之后使用**；**TOPOID 目前只支持默认拓扑 0**。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N+2：引入外部路由"——前置 [0-00078](task/UDG/20.15.2/0-00078.md) ADD OSPFV3。

## 决策点：发布模式选择（DP 0-00080）

问：ADD OSPFV3IMPORTROUTE 的发布模式（ROUPOOLNAME）取何策略？

| 选项 | ROUPOOLNAME | 联动 | 适用场景 |
|---|---|---|---|
| **全量发布（default）** | 不配 | PROTOCOL=wlr 全量发布 WLR 路由 | GWFD-010155 PGW-U→ePDG；GWFD-020406 IPv6 Prefix Delegation |
| **路由策略发布** | ref [ADD ROUTEPOLICY](command/UDG/20.15.2/ADD-ROUTEPOLICY.md) 配的策略名（如 `policy1`） | 仅发布策略匹配的路由 | GWFD-020401 IPv6 承载上下文 |

> 两种发布模式均要求 PROTOCOL 固定为 `wlr`，不能只选 `WLR_SP` 或 `WLR_UD` 子类；INHERITCOSTCFG 不能与 IMPTCOSTCFG 同时为 TRUE。

## 约束

- **最大记录数 8000**（warning，rule-0-00480）：本命令最大记录数为 8000 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00480）：命令执行后立即生效 — 后果：误操作即刻生效
- **必须先 ADD OSPFV3 才能使用本命令**（critical，rule-0-00480）：只有在执行 ADD OSPFV3 配置了 OSPFv3 进程后才能使用此命令 — 后果：未建进程命令拒收
- **PROTOCOL 不能只选 WLR_SP/WLR_UD**（critical，rule-0-00480）：采用路由策略方式发布路由时，PROTOCOL 不能只选 WLR_SP 或 WLR_UD，必须选 WLR 方式发布所有路由 — 后果：选子类致路由发布不全
- **PROTOCOL 必选**（critical，rule-0-00480）：PROTOCOL 是必选参数 — 后果：未指定 PROTOCOL 命令拒收
- **INHERITCOSTCFG 与 IMPTCOSTCFG 互斥**（critical，rule-0-00480）：参数 INHERITCOSTCFG 不能与 IMPTCOSTCFG 同时配置为 TRUE — 后果：同时为 TRUE 命令拒收
- **TOPOID 固定 0**（info，隐含）：TOPOID 目前只支持默认拓扑 0 — 后果：选非 0 命令拒收
- **PROTOCOLPROCID 默认 1**（info，隐含）：PROTOCOL=ospfv3 时 PROTOCOLPROCID 默认 1 — 后果：多进程场景需明确指定源进程号
- **CFG 三开关默认 FALSE**（info，隐含）：IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG 默认 FALSE=不配置 — 后果：选 TRUE 时 COST/TAG/TYPE 必填
