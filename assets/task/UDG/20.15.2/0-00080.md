# 配置OSPFv3引入外部路由(IPv6)（ADD OSPFV3IMPORTROUTE）

> 命令静态知识见 [ADD OSPFV3IMPORTROUTE](command/UDG/20.15.2/ADD-OSPFV3IMPORTROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD OSPFV3IMPORTROUTE 将外部 WLR 路由引入 OSPFv3 域并在域内发布。4 个特性数据规划表（GWFD-010155/020401/020403/020406）均固定 PROTOCOL=wlr 全量发布，GWFD-020401 额外配 ROUPOLINAME 走路由策略发布。9 个引入路由属性进阶参数（COST/TAG/TYPE/PROTOCOLPROCID/PERMITIBGPCFG/INHERITCOSTCFG 等）3 特性均未用，不绑。

**两种发布范式（从证据数据规划表归纳）**

| 组网 | PROCID | TOPOID | PROTOCOL | ROUPOLINAME | 脚本样例 |
|---|---|---|---|---|---|
| PGW-U 到 ePDG IPv6 动态路由（GWFD-010155） | 6（已配置数据中获取，引用 0-00078） | 0（全网规划） | wlr（全网规划） | 不配（全量发布） | `ADD OSPFV3IMPORTROUTE:PROCID=6,TOPOID=0,PROTOCOL=wlr,IMPTCOSTCFG=FALSE,IMPTTAGCFG=FALSE,IMPTTYPECFG=FALSE;` |
| IPv6 承载上下文（GWFD-020401） | 100（本端规划，引用 0-00078） | 0（全网规划） | wlr（全网规划） | policy1（已配置数据中获取，路由策略发布） | `ADD OSPFV3IMPORTROUTE:PROCID=100,TOPOID=0,PROTOCOL=wlr,ROUPOLINAME="policy1";` |
| IPv6 Prefix Delegation（GWFD-020406 系列） | 6（本端规划，引用 0-00078） | 0（全网规划） | wlr（本端规划） | 不配（全量发布） | `ADD OSPFV3IMPORTROUTE:PROCID=6,TOPOID=0,PROTOCOL=wlr;` |

**参数取值规则**
- **PROCID**：reference 引用上游 [创建OSPFv3进程(IPv6)](task/UDG/20.15.2/0-00078.md)，保证进程号一致。
- **TOPOID=0**：base 拓扑，3 特性脚本均显式固定为 0。
- **PROTOCOL=wlr**：引入 WLR 路由。采用路由策略方式发布时（指定 ROUPOLINAME）**不能只选 WLR_SP 或 WLR_UD，必须选 WLR 方式发布所有路由**（GWFD-020401 md 说明）。
- **ROUPOLINAME**：conditional，仅 GWFD-020401 配 policy1 走路由策略发布（可 LST ROUTEPOLICY 查询）；010155/020406 不配走全量发布。
- **IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG**：引入路由开销/标签/类型配置开关，默认 FALSE=不配置；GWFD-010155 脚本显式传 FALSE 等同默认空操作，不绑。

## 约束

- **rule 0-00202**（constraint/critical）前置——必须先执行 ADD OSPFV3 配置了 OSPFv3 进程(PROCID)才能使用此命令；命令最大记录数 8000；采用路由策略方式发布路由时(指定 ROUPOLINAME)，PROTOCOL 不能只选 WLR_SP 或 WLR_UD，必须选 WLR 方式发布所有路由；不指定 ROUPOLINAME 时按 PROTOCOL=wlr 全量发布 — 后果：路由引入失败 / 发布路由不全 / 进程未创建报错
