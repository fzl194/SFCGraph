# 设置IKE全局配置（SET IKEGLOBALCONFIG）

> 命令静态知识见 [SET IKEGLOBALCONFIG](command/UDG/20.15.2/SET-IKEGLOBALCONFIG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

可选步骤，紧跟 IPsec 安全策略与隧道接口组装之后（激活 md 中为"步骤 9：配置 DPD 功能（可选）"）。两条主线：开 DPD 周期性探测对端存活；开 NAT 保活（仅 NAT 穿越场景）。命令是 SET（修改全局唯一记录），不存在的参数保持系统当前值，故只写本场景需要的 4 个参数即可。

**数据规划取值样例**（证据③，IPFD-015004 各激活 md 数据规划表，4 场景取值高度一致）：

| 参数 | 含义 | 取值样例 | 获取方法 | 适用条件 |
|---|---|---|---|---|
| DPDTYPE | DPD 类型 | Periodic | 本端规划 | 开启 DPD 时配 |
| DPDINTERVAL | DPD 检查间隔 (s) | 10 | 本端规划 | DPDTYPE=Periodic 时配套生效 |
| DPDRETRYINTRVL | DPD 重试间隔 (s) | 3 | 本端规划 | DPDTYPE=Periodic 时配套生效 |
| NATKLI | NAT 保活时间间隔 (s) | 30 | 本端规划 | 仅 IKEPEER NATTRAVERSAL=TRUE（NAT 穿越）时生效 |

**示例脚本**（取自激活 md）：

```
// 仅开 DPD（无 NAT 穿越）：
SET IKEGLOBALCONFIG:DPDTYPE=Periodic,DPDINTERVAL=10,DPDRETRYINTRVL=3;
// DPD + NAT 穿越（如普通 IPv4 IPsec 隧道场景）：
SET IKEGLOBALCONFIG:DPDTYPE=Periodic,DPDINTERVAL=10,DPDRETRYINTRVL=3,NATKLI=30;
```

**配置要点**：
- 4 个场景（普通 IPv4/IPv6、GRE over IPsec、主备、OSPF over IPsec、多 Sequence）DPD 三参数取值完全相同（Periodic/10/3），可视为本特性的默认推荐组合。
- NATKLI 仅当 IKEPEER 配置了 `NATTRAVERSAL=TRUE` 才生效，未开 NAT 穿越时不必带该参数（默认 20，由系统保持）。
- DPDTYPE=Periodic 时 DPDINTERVAL 与 DPDRETRYINTRVL 必须配套，缺一则探测不生效。
- 命令真相其余 17 个参数（CPU 阈值/分片/抗重放/流控等）在 4 个 IPsec 场景数据规划表均未出现，按通用配置门槛不绑，保持系统默认。

## 约束

- 该命令执行后立即生效；不输入的参数保持系统当前配置（可经 LST IKEGLOBALCONFIG 查询）。
- 需要 CPU 告警上报阈值（CPUREPORTTHRES）≥ CPU 告警恢复阈值（CPUCLEARTHRES）。
- DPDTYPE=Periodic 时 DPDINTERVAL+DPDRETRYINTRVL 配套生效。
- NATKLI 仅在 IKEPEER NATTRAVERSAL=TRUE（NAT 穿越）场景生效。
- 安全日志阈值（DOSTHRESHOLD）不为 0 时，链路协商失败可能导致日志大量打印。
