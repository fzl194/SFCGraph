# 设置OTT业务规则匹配开关（SET EXTOTTMATCHSW）

> 命令静态知识见 [SET EXTOTTMATCHSW](command/UDG/20.15.2/SET-EXTOTTMATCHSW.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET EXTOTTMATCHSW 是 GWFD-110321 支持内置百万级业务规则库特性专属的"OTT 匹配全局开关"命令：使能后 FLOWFILTER(FLOWFILTERTYPE=EXTEROTTDB) 才能匹配 OTT 业务规则库。本命令的核心配置空间是 2 个独立维度的使能开关（IPV4MATCH/IPV6MATCH），整机最多记录数 **1**（全局唯一），**立即生效**，**属高危命令（操作不当导致性能下降明显）**，系统初始值均为 DISABLE。

### 配置维度 1：IPv4 流量 OTT 匹配开关（参数 IPV4MATCH，可选/默认 DISABLE）

| 取值 | 作用 | 适用 NF |
|---|---|---|
| **ENABLE** | 使能 ISU VM 外置匹配基于 IPv4 的 OTT 业务 | PGW-U、UPF |
| **DISABLE**（默认/系统初始） | 关闭 IPv4 流量 OTT 业务匹配 | 同上 |

### 配置维度 2：IPv6 流量 OTT 匹配开关（参数 IPV6MATCH，可选/默认 DISABLE）

| 取值 | 作用 | 适用 NF |
|---|---|---|
| **ENABLE** | 使能 ISU VM 外置匹配基于 IPv6 的 OTT 业务 | PGW-U、UPF |
| **DISABLE**（默认/系统初始） | 关闭 IPv6 流量 OTT 业务匹配 | 同上 |

> **典型范式（GWFD-110321 全量使能）**：
> ```
> SET EXTOTTMATCHSW:IPV4MATCH=ENABLE,IPV6MATCH=ENABLE;
> ```

**步骤位置**：在所属 feature 编排中通常为"步骤 4：使能 OTT 匹配全局开关"——[LOD EXTERNALDB](task/UDG/20.15.2/0-00092.md) 加载规则库 + [SET REFRESHSRV](task/UDG/20.15.2/0-00015.md) 刷新之后、ADD URR 配置在线计费费率标识之前；应先 ADD FLOWFILTER(FLOWFILTERTYPE=EXTEROTTDB) 再 SET EXTOTTMATCHSW。

## 决策点：无

（无。本命令在所有特性下用法一致，2 个独立开关均为 ENABLE/DISABLE 二值，无分支决策。）

## 约束

- **整机最大记录数为 1**（warning，rule-0-00378）：全局唯一实例 — 后果：重复执行覆盖前值
- **属高危命令**（critical，rule-0-00378）：操作不当导致性能下降明显，须谨慎使用并联系华为技术支持 — 后果：性能问题
- **立即生效**（info，隐含）：该命令执行后立即生效 — 后果：开关状态变更实时影响 OTT 匹配
- **系统初始 IPV4MATCH/IPV6MATCH 均为 DISABLE**（info，隐含）：本命令使能前 OTT 匹配默认关闭 — 后果：未显式 ENABLE 时 OTT 匹配不生效
- **应先 ADD FLOWFILTER 再 SET EXTOTTMATCHSW**（warning，隐含）：未先建 FLOWFILTER(FLOWFILTERTYPE=EXTEROTTDB) 时 SET 开关无效 — 后果：开关使能但无匹配流过滤器可作用
