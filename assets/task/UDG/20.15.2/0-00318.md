# 设置全局地址分配属性（SET ADDRESSATTR）

> 命令静态知识见 [SET ADDRESSATTR](command/UDG/20.15.2/SET-ADDRESSATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令属高危命令(参数策略须与AAA一致, 否则用户无法分配地址激活失败)。

## 配置方法（配置方法字典）

SET ADDRESSATTR 是"全局地址分配属性开关"命令: 整局唯一(max_records=1), 设置整局级地址分配参数, 控制 UPF 地址池通配是否开启、IPv4/IPv6 主机路由是否对外发布、用户去激活后 IP 等待释放时间, 以及 CU Full Mesh 场景下 UPF 间的分配优先级。命令核心配置空间有 4 个维度: **地址池通配**(V4/V6POOLWILDCARDSW)、**主机路由发布**(V4/V6IPHOSTROUTE)、**地址租期**(RELEASETIME)、**分配优先级**(ALLOCPRECEDENCE, CU Full Mesh 场景)。2 个特性命中 (010105 基于 RADIUS 下发地址池名、020161 CU Full Mesh) 覆盖典型场景。

### 配置维度 1: 地址池通配开关 (参数 V4POOLWILDCARDSW/V6POOLWILDCARDSW, 可选)

控制 AAA 未下发通配符时, UPF 是否对地址池通配功能进行使能。

| 参数 | 取值 | 作用 |
|---|---|---|
| **V4POOLWILDCARDSW** | ENABLE / DISABLE | IPv4 地址池通配功能开关; 控制 AAA 未下发 IPv4 通配符时是否开启通配。需与 AAA 地址池匹配策略一致, 否则可能导致 UPF 无法分配地址 |
| **V6POOLWILDCARDSW** | ENABLE / DISABLE | IPv6 地址池通配功能开关; 控制 AAA 未下发 IPv6 通配符时是否开启通配 |

> 010105 (基于 RADIUS 下发地址池名) 范式显式配 `V4POOLWILDCARDSW=ENABLE + V6POOLWILDCARDSW=ENABLE`; 系统初始值 DISABLE。

### 配置维度 2: 主机路由发布开关 (参数 V4IPHOSTROUTE/V6IPHOSTROUTE, 可选)

控制 UE 外部地址主机路由是否对外发布。

| 参数 | 取值 | 作用 |
|---|---|---|
| **V4IPHOSTROUTE** | ENABLE / DISABLE | IPv4 的 UE 外部地址主机路由对外发布开关 |
| **V6IPHOSTROUTE** | ENABLE / DISABLE | IPv6 的 UE 外部地址主机路由对外发布开关 |

> ⚠ **关键约束**: V4IPHOSTROUTE/V6IPHOSTROUTE 设为 DISABLE 时, 可能导致用户下行包不通。需与 AAA 配置策略一致, 否则可能致地址分配失败。系统初始值 ENABLE。

### 配置维度 3: 地址租期 (参数 RELEASETIME, 可选)

控制用户因下行异常报文被去激活后, 用户 IP 的等待释放时间。

| 参数 | 范围 | 作用 |
|---|---|---|
| **RELEASETIME** | 整数 0~86400 (秒) | 用户去激活后 IP 等待释放时间; 取值 0 时按地址池配置的 release-time 延迟释放 |

> 系统初始值 0 (按地址池配置延迟释放)。

### 配置维度 4: 分配优先级 (参数 ALLOCPRECEDENCE, CU Full Mesh 场景)

CU Full Mesh 组网场景下, 用户首激活 UPF 不可用时的分配优先级。

| 参数 | 取值 | 作用 |
|---|---|---|
| **ALLOCPRECEDENCE** | UPF_FIRST / LOCAL_FIRST | CU Full Mesh 组网下分配优先级; UPF_FIRST 优先分配至 UPF, LOCAL_FIRST 优先本地 |

> 020161 (CU Full Mesh 组网) 范式显式配 `ALLOCPRECEDENCE=UPF_FIRST`; 本维度仅 CU Full Mesh 场景使用, 单 UPF 部署无须配置。

**典型脚本 (010105 基于 RADIUS 下发地址池名, 双栈地址池通配)**:
```
SET ADDRESSATTR: V4POOLWILDCARDSW=ENABLE, V6POOLWILDCARDSW=ENABLE;
```

**典型脚本 (020161 CU Full Mesh 组网, 分配优先级)**:
```
SET ADDRESSATTR: ALLOCPRECEDENCE=UPF_FIRST;
```

> 命令 notes: 适用 NF: PGW-U、UPF; 本命令属高危命令 (参数策略须与 AAA 保持一致, 否则用户激活失败); 执行后立即生效; 系统仅 1 条记录 (整局唯一); 系统初始记录: V4POOLWILDCARDSW=DISABLE, V6POOLWILDCARDSW=DISABLE, RELEASETIME=0, V4IPHOSTROUTE=ENABLE, V6IPHOSTROUTE=ENABLE。

**步骤位置**: 在所属 feature 编排中通常位于"步骤 1: 全局地址属性配置"——属地址分配链路的前置 (010105 010104 020406 020421 场景下先于地址池/APN 地址规则配置); 与 [0-00046](task/UDG/20.15.2/0-00046.md) SET APNADDRESSATTR (APN 级) + [0-00250](task/UDG/20.15.2/0-00250.md) SET IPALLOCRULE (全局规则) + [0-00319](task/UDG/20.15.2/0-00319.md) SET APNIPALLOCRULE (APN 级规则) + [0-00064](task/UDG/20.15.2/0-00064.md) SET REDUNDUSER (冗余用户总开关) + [0-00249](task/UDG/20.15.2/0-00249.md) SET DATAPLANEINFMODE (入不转板总开关) 协同。CU Full Mesh 场景下与 [0-00219](task/UDG/20.15.2/0-00219.md) SET CPTEIDUALLOC (F-TEID 分配能力) 配合。

## 决策点: 通配功能与 AAA 策略一致性 (DP 0-00318)

问: 通配功能开关与主机路由发布开关如何与 AAA 策略对齐?

| 选项 | V4/V6POOLWILDCARDSW | V4/V6IPHOSTROUTE | 典型场景 |
|---|---|---|---|
| 通配开启+路由发布 | ENABLE | ENABLE (默认) | 010105 基于 RADIUS 下发地址池名场景 (接受 AAA 下发通配) |
| 仅设置分配优先级 | 不配 | 不配 | 020161 CU Full Mesh 组网场景 (仅 ALLOCPRECEDENCE) |
| 系统初始值 | DISABLE | ENABLE | 未启用通配功能的整局默认配置 |

> 2 个特性命中: 010105 显式配双栈通配; 020161 显式配 ALLOCPRECEDENCE; 默认状态下无须改动。

## 约束

- **整局唯一 (max_records=1)** (critical, rule-0-00318): 系统仅支持 1 条全局记录 — 后果: 重复配置命令拒收
- **执行后立即生效** (warning, rule-0-00318): 命令执行后立即生效 — 后果: 误操作即刻影响全局用户
- **高危命令 (与 AAA 策略对齐)** (critical, rule-0-00318): 参数配置策略必须与 AAA 保持一致 (V4/V6POOLWILDCARDSW + V4/V6IPHOSTROUTE) — 后果: 不一致致 UPF 无法分配地址, 用户激活失败
- **主机路由关闭致下行不通** (critical, rule-0-00318): V4IPHOSTROUTE/V6IPHOSTROUTE 设置为 DISABLE 时可能导致用户下行包不通 — 后果: 用户激活后业务异常
- **适用 NF 限定** (warning, 隐含): PGW-U、UPF — 后果: SGW-U 等不适用
- **ALLOCPRECEDENCE 仅 CU Full Mesh 使用** (info, 隐含): ALLOCPRECEDENCE 参数只在 CU Full Mesh 组网 (020161) 场景生效, 单 UPF 部署无须配置 — 后果: 单 UPF 配 ALLOCPRECEDENCE 无实际效果
- **RELEASETIME=0 走地址池配置** (info, 隐含): RELEASETIME=0 时按地址池配置的 release-time 延迟释放 IP — 后果: 与地址池配置协同, 需同步规划
