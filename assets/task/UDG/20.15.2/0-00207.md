# 增加基本IPv4 ACL规则（ADD ACLRULEBAS4）

> 命令静态知识见 [ADD ACLRULEBAS4](command/UDG/20.15.2/ADD-ACLRULEBAS4.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。基本 ACL 是 ACL 家族最简形态——**仅按源 IP 匹配**，不支持目的 IP/端口/协议；ACLNAME 引用 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP 基本范围段（2000~2999）；需五元组匹配时改用 [0-00198](task/UDG/20.15.2/0-00198.md) ADD ACLRULEADV4；二层匹配改用 [0-00208](task/UDG/20.15.2/0-00208.md) ADD ACLRULEETH。

## 配置方法（配置方法字典）

ADD ACLRULEBAS4 是"增加基本 IPv4 ACL 规则"命令。**3 参必选**（ACLNAME/ACLRULENAME/ACLACTION）+ 6 参可选（ACLRULEID/ACLFRAGTYPE/ACLSOURCEIP/ACLSRCWILD/VRFNAME/ACLRULEDESCRIPTION）。配置空间有 3 个维度：**规则组标识**（ACLNAME 决定走哪类组）、**规则行为**（ACLACTION Permit/Deny）、**源 IP 反掩码配对**（ACLSOURCEIP + ACLSRCWILD 决定匹配范围）。1 个特性命中（IPFD-012002 ACL）覆盖 Deny+精确反掩码范式。

### 配置维度 1：规则组标识（参数 ACLNAME，必选，引用上游）

| 取值形式 | 数据来源 | 适用场景 |
|---|---|---|
| **数字字符串 2000~2999** | 引用 [0-00197](task/UDG/20.15.2/0-00197.md):ACLNAME | IPFD-012002 范例 `2000`（基本段） |
| **字符串 1~32** | 引用 [0-00197](task/UDG/20.15.2/0-00197.md):ACLNAME | 命名 ACL |

> **范围段铁律**：ACLNAME 数字串必须在 2000~2999 范围段（基本 ACL）；跨段数字串（如 3001）命令拒收。

### 配置维度 2：规则行为（参数 ACLACTION，必选）

| 取值 | 作用 |
|---|---|
| **Permit** | 允许匹配报文通过 |
| **Deny** | 拒绝匹配报文 |

> IPFD-012002 范例 = Deny（拒绝源 IP=10.1.1.1 报文）。

### 配置维度 3：源 IP 匹配范围（参数 ACLSOURCEIP + ACLSRCWILD）

| ACLSOURCEIP | ACLSRCWILD | 匹配范围 | 适用场景 |
|---|---|---|---|
| **特定 IP（如 10.1.1.1）** | **0.0.0.0** | 精确匹配该 IP | IPFD-012002 范例（精确 Deny 源 IP=10.1.1.1） |
| **网段 IP（如 10.1.1.0）** | **0.0.0.255** | 匹配整段（/24） | 整段子网过滤 |
| **0.0.0.0（默认）** | **255.255.255.255** | 匹配所有 | 等价"任意源" |

> **默认值**：ACLSOURCEIP 默认 0.0.0.0 / ACLSRCWILD 默认 255.255.255.255（即"任意源 IP"）。**典型反掩码**：0.0.0.0 = 精确主机；0.0.0.255 = /24 网段。

**典型脚本（IPFD-012002 基本组 2000，精确 Deny 10.1.1.1）**：
```
ADD ACLRULEBAS4: ACLNAME="2000", ACLRULENAME="rule_5", ACLACTION=Deny, ACLSOURCEIP="10.1.1.1", ACLSRCWILD="0.0.0.0";
```

> 命令 notes：**最大记录数 65535**（整机 ACL 规则）；**单组最多 16384 条规则**；**执行后立即生效**；ACLFRAGTYPE（首片/非首片/非分片报文匹配，6 种枚举）；VRFNAME（VPN 实例限定，默认 _public_，1~31 字符）；**前置必须先 ADD ACLGROUP**；当某类数据流能与指定规则组中的多条规则匹配时，实际生效的是规则号最低的规则。

**步骤位置**：在所属 feature 编排中位于"步骤 N+1：建基本 ACL 规则"——前置 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP（建组）；下游 [MOD ACLRULEBAS4](command/UDG/20.15.2/MOD-ACLRULEBAS4.md)（修改规则）、[RMV ACLRULEBAS4](command/UDG/20.15.2/RMV-ACLRULEBAS4.md)（删除）。

## 决策点

（无。本命令在所有特性下用法一致——仅按源 IP 匹配，行为 Permit/Deny 二选一，无特性分叉。）

## 约束

- **最大记录数 65535**（critical，rule-0-00207）：整机 ACL 规则最大 65535 条 — 后果：超规格新增失败
- **单组最多 16384 条规则**（critical，rule-0-00207）：单个 ACL 规则组下最多 16384 条 — 后果：超限新增失败
- **前置必须先 ADD ACLGROUP**（critical，rule-0-00207）：ACLNAME 引用的规则组必须先建 — 后果：未建组新增失败
- **执行后立即生效**（warning，rule-0-00207）：执行后立即生效 — 后果：误操作即刻生效
- **ACLNAME 范围段约束**（critical，rule-0-00207）：数字 ACLNAME 必须在 2000~2999 基本范围段 — 后果：跨段数字命令拒收
- **最低规则号生效原则**（info，rule-0-00207）：某类数据流能与多条规则匹配时，实际生效的是规则号最低的规则 — 后果：规则号顺序决定优先级
- **ACLRULENAME 命名约束**（info，rule-0-00207）：字符串 1~32（不支持空格，区分大小写） — 后果：超长命令拒收
- **不支持目的 IP/端口/协议匹配**（warning，rule-0-00207）：基本 ACL 仅按源 IP 匹配 — 后果：需五元组匹配应改用 ADD ACLRULEADV4
- **VRFNAME 限制**（info，rule-0-00207）：当前添加 ACL 规则时不支持配置 VPN（VRFNAME 默认 _public_） — 后果：配置 VRFNAME 非 _public_ 时命令拒收
- **VRFNAME 长度**（info，rule-0-00207）：字符串 1~31（字符之间不支持空格，区分大小写） — 后果：超长命令拒收
- **ACLRULEDESCRIPTION 长度**（info，rule-0-00207）：字符串 1~127（支持空格，区分大小写） — 后果：超长命令拒收
