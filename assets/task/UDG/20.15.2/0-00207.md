# 增加基本ACL规则(VNRS IPv4)（ADD ACLRULEBAS4）

> 命令静态知识见 [ADD ACLRULEBAS4](command/UDG/20.15.2/ADD-ACLRULEBAS4.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令创建**基本 IPv4 ACL 规则**，仅按**源 IP 地址**匹配报文（对应范围段 2000-2999）。ACLNAME 引用 [ADD ACLGROUP](task/UDG/20.15.2/0-00197.md)（基本范围段）。基本 ACL 不支持目的 IP/端口/协议匹配——需要五元组匹配时改用 [ADD ACLRULEADV4](task/UDG/20.15.2/0-00198.md)。

**配置范式（IPFD-012002 数据规划表归纳）**

| 步骤 | ACLNAME | ACLRULENAME | ACLACTION | 源 IP 段 | 示例行 |
|---|---|---|---|---|---|
| 增加基本规则 | 引用 0-00197 组（基本段 2000-2999） | 规则名称 | Permit / Deny | 源 IP + 反掩码 | `ADD ACLRULEBAS4:ACLNAME="<组>", ACLRULENAME="<规则名>", ACLACTION=Permit, ACLSOURCEIP="10.1.1.1", ACLSRCWILD="0.0.0.255";` |

**参数取值约定**

| 参数 | 取值 | 来源 | 说明 |
|---|---|---|---|
| ACLNAME | 字符串 1~32，基本范围段 2000-2999 | 引用 [ADD ACLGROUP](task/UDG/20.15.2/0-00197.md):ACLNAME | 规则组标识（前置必须先建） |
| ACLRULENAME | 字符串 1~32 | 本端规划 | 规则名称 |
| ACLACTION | Permit / Deny | 本端规划 | 规则行为 |
| ACLSOURCEIP | IPv4，默认 0.0.0.0 | 全网规划 | 源 IP 地址 |
| ACLSRCWILD | IPv4 反掩码，默认 255.255.255.255 | 全网规划 | 源 IP 反掩码 |

**未绑冷门参数**（命令真相有但特性脚本未用）：ACLRULEID / ACLFRAGTYPE / VRFNAME（默认 _public_）/ ACLRULEDESCRIPTION——按需（报文分片、VPN 实例、规则描述等垂直场景）时查命令 wiki。

## 约束

- 最大记录数 65535；单 ACL 规则组下最多 16384 条规则；前置必须先 `ADD ACLGROUP`——后果：超限/无组→新增失败
- 基本范围段 2000-2999，增加规则时 ACL 规则组必须已创建
