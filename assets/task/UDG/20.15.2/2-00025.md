# 基于终端系统的码率差异化控制（GWFD-110301）

> 特性静态知识见 [GWFD-110301](feature/UDG/20.15.2/GWFD-110301.md)。本页讲配置生成实例化时这个特性**怎么配**。
> **本特性 = [GWFD-110311](feature/UDG/20.15.2/GWFD-110311.md) "HTTP/NONTOS 用户组级" 场景的 OSTYPE 维度扩展**——骨架同 BWM 族（NONTOS+PROTOCOL 识别 + CAR 组级规则），差异在 BWMSERVICE 增 OSTYPE 参数 + 末步加 APNOSLELBWMSW 开关。

## 配置概览

本特性对同一用户组下的 IOS/Android 等操作系统用户差异化执行码率监管（CAR）。核心对象链：**BWMSERVICE**（NONTOS + PROTOCOL + OSTYPE，按 OS 识别业务流）→ **BWMCONTROLLER**（CAR，上下行各一 `bc-up`/`bc-down`）→ **BWMUSERGROUP**（USERGLEVSRVTYPE=NONTOS，组级）→ **BWMRULE**（GROUP_SPECIFIC，绑 Service+Controller）→ 绑定 APN → **APNOSLELBWMSW**（APN 级 OS 差异化开关）。

复用 BWM 族通用骨架：业务与控制器阶段 → 用户组规则绑定阶段。差异（OSTYPE 业务识别 + 组级 NONTOS 规则 + APNOSLELBWMSW 末步开关）由特性级 DP 驱动，记入下表 + 各 compound 场景差异（[1-00002](task/UDG/20.15.2/1-00002.md)/[1-00003](task/UDG/20.15.2/1-00003.md) 待回填 OSTYPE/组级 NONTOS 差异条目）。

## 配置流程（步骤编排，每步显示含命令）

1. **License**（单命令，特性专属 License）：`SET LICENSESWITCH`（`LKV3G5RDSC01`）→ [0-00019](task/UDG/20.15.2/0-00019.md)（[1-00001](task/UDG/20.15.2/1-00001.md) 承载，License 项与 BWM 主特性不同，见脚注¹）
2. **BWM 业务与控制器**（OSTYPE 识别 + CAR 控制器上下行各一）：`ADD BWMSERVICE` + `ADD BWMCONTROLLER`×2 → [1-00002](task/UDG/20.15.2/1-00002.md)
   - BWMSERVICE：`BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=PROTOCOL,PROTOCOLNAME=http,OSTYPE=IOS`（OSTYPE 是本特性差异维度，IOS/Android 等按需）²
   - BWMCONTROLLER×2：`BWMCNAME="bc-up"/"bc-down",CTRLTYPE=CAR,CIR=300,CBS=10000,PIR=600,PBS=12000,GREEN=PASS,YELLOW=PASS,RED=DISCARD`（上下行各一，CAR 参数）²
3. **生效范围**（单命令，并入此步）：`SET BANDWIDTHMNG`（`RANGE=SYSTEM`，组级场景必配）→ [1-00003](task/UDG/20.15.2/1-00003.md) 承载
4. **BWM 用户组、规则与绑定**（组级 NONTOS 规则 + APN 绑定）：`ADD BWMUSERGROUP` + `ADD BWMRULE`(GROUP_SPECIFIC) + `ADD APNBINDBWMUSRG` → [1-00003](task/UDG/20.15.2/1-00003.md)
   - BWMUSERGROUP：`USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPPRI=1,USERGLEVSRVTYPE=NONTOS`（组级 NONTOS 路径）²
   - BWMRULE：`BWMRULETYPE=GROUP_SPECIFIC,BWMSERVICENAME=引用上步,UPBWMCTRLNAME1="bc-up",DNBWMCTRLNAME1="bc-down"`²
   - APNBINDBWMUSRG：`USERGROUPNAME=引用,APN="apn-test"`²
5. **APN 操作系统级开关**（单命令，特性末步，直接 atom）：`SET APNOSLELBWMSW`（`APNNAME=引用上步 APN,ISENABLE=ENABLE`）→ [0-00107](task/UDG/20.15.2/0-00107.md)³

> **脚注¹（License 证据）**：activation 脚本首行 `SET LICENSESWITCH:LICITEM="LKV3G5RDSC01",SWITCH=ENABLE;`（[激活基于终端系统的码率差异化控制](evidence/UDG/20.15.2/激活基于终端系统的码率差异化控制_69722139.md)）。本 License 与 BWM 主特性 `LKV3G5TCSA01`、Shaping `LKV3G5SBTS01` 均不同——终端系统差异化是独立 License 控制项。
>
> **脚注²（BWM 参数证据链）**：BWMSERVICE/BWMCONTROLLER/BWMUSERGROUP/BWMRULE/APNBINDBWMUSRG 命令与参数全部来自 activation 任务示例脚本与数据规划表（同上证据）。`OSTYPE=IOS` 为 activation 明示取值（IOS 演示；Android 等其他 OS 同理按需规划）。CAR 参数 CIR=300/PIR=600/CBS=10000/PBS=12000（kbps/字节）+ 三色 PASS/PASS/DISCARD 为 activation 样例。
>
> **脚注³（APNOSLELBWMSW 证据）**：`SET APNOSLELBWMSW:APNNAME="apn-test",ISENABLE=ENABLE;` 来自 activation 任务示例末行（同上证据）。atom [0-00107](task/UDG/20.15.2/0-00107.md) 已引用同一证据。
>
> **activation 模板复用说明（R1.4）**：本特性 activation（69722139）是**独立模板**（非复用），4 步骤全部独立演示，无模板复用风险。与 GWFD-110311 NONTOS 族共享 BWMSERVICE(NONTOS)/BWMCONTROLLER(CAR)/BWMUSERGROUP/BWMRULE 命令集，但 BWMSERVICE 增 OSTYPE 参数（110311 无）+ 末步 APNOSLELBWMSW（110311 无）——差异进 DP 表。

## 决策点

**DP（特性级）：操作系统类型（OSTYPE）+ 控制层级** —— 决定 BWMSERVICE 识别维度与 BWMRULE 类型。每 option 影响必全：

| 场景 | OSTYPE | BWMSERVICE 识别 | BWMRULETYPE | 控制器类型 | 走法 | 关键联动 |
|---|---|---|---|---|---|---|
| IOS 终端码率差异化（activation 样例） | IOS | NONTOS+PROTOCOL+PROTOCOLNAME+OSTYPE=IOS | GROUP_SPECIFIC | CAR（上下行各一） | [1-00002](task/UDG/20.15.2/1-00002.md)+[1-00003](task/UDG/20.15.2/1-00003.md)+[0-00107](task/UDG/20.15.2/0-00107.md) | USERGLEVSRVTYPE=NONTOS；SET BANDWIDTHMNG RANGE=SYSTEM；末步 APNOSLELBWMSW |
| Android 终端码率差异化（同族扩展） | Android | 同上 OSTYPE=Android | GROUP_SPECIFIC | CAR | 同上 | 同上，BWMSERVICE 按 OS 各建一条 |
| 其他 OS（同族扩展） | 其他 | 同上 OSTYPE=对应值 | GROUP_SPECIFIC | CAR | 同上 | 同上 |

> **OSTYPE 维度**：本特性核心差异——同一 PROTOCOLNAME（如 http）按 OSTYPE 建多条 BWMSERVICE，实现 OS 间差异化码率。activation 演示 IOS，其他 OS 同理。
>
> **组级（GROUP_SPECIFIC）非用户级**：本特性是组级码率差异化（同组内按 OS 分流），非 [2-00023](task/UDG/20.15.2/2-00023.md) 的用户级 Shaping。故 USERGLEVSRVTYPE（组级）非 USERLEVSRVTYPE，BWMRULETYPE=GROUP_SPECIFIC 非 SUBSCRIBER_SPECIFIC。

## 约束（特性级 rule）

- **License 前置**（critical）：`SET LICENSESWITCH` 开启 `LKV3G5RDSC01` 是第一步。
- **依赖特性**（info）：[SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，业务识别）+ [基于业务感知的带宽控制](feature/UDG/20.15.2/GWFD-110311.md)（GWFD-110311，BWM 基础）。
- **APNOSLELBWMSW 必开**（critical）：末步 `SET APNOSLELBWMSW ISENABLE=ENABLE` 是本特性生效前提——不开则 APN 下用户不按 OS 差异化执行码率控制，本特性失效（仅有 BWM 规则不生效 OS 差异）。
- **CAR 参数推荐**（warning）：CIR=0.5×PIR；CBS≥CIR×1.5×1000/8；PBS≥PIR×1.5×1000/8（activation 数据规划表明示推荐配置）。
- **OSTYPE 唯一性**（warning）：同一 PROTOCOLNAME 下不同 OSTYPE 须各建 BWMSERVICE（按 OS 分流），BWMSERVICENAME 全网唯一。
- **SET BANDWIDTHMNG 必配**（warning）：组级场景 `RANGE=SYSTEM` 必配（[1-00003](task/UDG/20.15.2/1-00003.md) 场景差异）——否则组级控制范围不明。
- **新增用户组对在线用户不生效**（warning）：需重激活。
- **APNOSLELBWMSW 与全局开关两级**（info）：本 APN 级开关承继全局 GLBOSLELBWMSW（全局开关 atom 待后续特性揭示）——全局关则 APN 级开亦无效。
