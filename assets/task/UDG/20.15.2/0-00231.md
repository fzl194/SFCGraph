# 配置IPSQM静态整形带宽（ADD IPSQMSHAPING）

> 命令静态知识见 [ADD IPSQMSHAPING](command/UDG/20.15.2/ADD-IPSQMSHAPING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110941（基于基站粒度的 IPSQM）**核心必选**第 2 步——按对端基站 IP + 承诺速率逐条配置静态整形带宽，对下行用户面流量整形，避免突发丢包与重传带宽恶化。适用 NF：SGW-U、UPF。每个基站一条记录，引用上游 SET LICENSESWITCH 开 License 之后执行。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| IPTYPE | IP 地址类型 | `IPv4` | 全网规划（本特性 IPv4） |
| PEERIPV4 | 对端基站 IPv4 地址 | `192.168.1.235` | 全网规划（条件必填，IPTYPE=IPv4 时） |
| CIR | 承诺信息速率（Mbps） | `100` | 本端规划，整数 1~10000 Mbps（必填） |

**脚本样例**

```
ADD IPSQMSHAPING:IPTYPE=IPv4,PEERIPV4="192.168.1.235",CIR=100;
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：PEERIPV6（对端 IPv6 基站）/VPNINSTANCE（跨 VRF 整形）/DESCRIPTION（备注）均 optional，特性 md 未用，现网按需补绑。

**操作步骤定位**：紧接 SET LICENSESWITCH（开 License）之后、可选 SET IPSQMQDEPTH（0-00232，整形队列深度）/SET IPSQMADJUST 之前。前置由 feature-rule 0-00082（IPSQM 增强须先核心整形）表达。

## 约束

- 命令最大记录数 **4000**（命令 notes 强约束），超出被拒。
- 命令执行后**只对之后发生承载更新的用户或新激活用户生效**（存量用户不感知，需提前评估）。
- 配置整形速率会**消耗 CPU 影响数据转发性能**，开启前请联系华为技术人员确认。
- CIR 取值范围 1~10000 Mbps。
