# 配置IPFarm服务器（ADD IPFARMSERVER）

> 命令静态知识见 [ADD IPFARMSERVER](command/UDG/20.15.2/ADD-IPFARMSERVER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD IPFARMSERVER 是 IP Farm 基础设施的"server 实例"命令：配置 IP farm 中的 server 地址，作为重定向 Server 的目标地址和心跳检测地址。本命令的核心配置空间有 4 个维度：**IP 版本 IPVERSION**（决定 SERVERIPV4/SERVERIPV6 哪个出现）、**URL**（可选，缺省则 URL 值=IP 地址）、**重定向缺省动作 DEFAULTACT**（不满足重定向条件时的兜底）、**重定向携带信息 APPENDINFONAME**（可选，仅 REDIRECT）。

### 配置维度 1：IP 版本（参数 IPVERSION，必选）

| 取值 | 配套必选参数 | 备注 |
|---|---|---|
| **IPV4** | SERVERIPV4（点分十进制） | 主流 |
| **IPV6** | SERVERIPV6（冒号分十六进制） | IPv6 接入场景 |

> server IP 类型必须与所属 IPFARM 的虚拟 IP 类型一致（[ADD IPFARM IPVERSION](task/UDG/20.15.2/0-00087.md)）。

### 配置维度 2：URL（参数 URL，可选）

| 场景 | URL 取值 | 备注 |
|---|---|---|
| 不配 URL | 缺省 | URL 值即为对应 IP 地址 |
| 配 URL | 字符串 1~119 | URL 长度按转义后字符计算；特殊字符 ? → %3f、空格 → %20、逗号 → %2c、加号 → %2b、分号 → \;、双引号 → \"、反斜杠 → \\、% → %25 |

> URL 转义规则详见命令 md 注意事项段；不允许以"/"开头；非法 ASCII 0x00~0x1F 和 0x7F 禁止；缺省 scheme=http://。

### 配置维度 3：重定向缺省动作（参数 DEFAULTACT，可选/默认 BLOCK）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **BLOCK**（默认） | 不满足重定向条件时阻塞报文 | Captive Portal 重定向；proxy server 用于鉴权/分流 |
| **PASS** | 不满足重定向条件时放通报文 | proxy server 仅用于业务加速；用户Portal 不需严格拦截 |

### 配置维度 4：重定向携带信息名称（参数 APPENDINFONAME，可选）

字符串 1~31（不区分大小写、不支持空格），需先经 [ADD REDIRAPPENDINFO](task/UDG/20.15.2/0-00023.md) 配置生成；典型携带信息：REQURL/IMSI/IMEI 等用户上下文信息。

> **典型范式**：
> ```
> ADD IPFARMSERVER:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERIPV4="192.168.253.251";
> ADD IPFARMSERVER:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERIPV4="192.168.10.123",URL="www.example.com";
> ```

**步骤位置**：在所属 feature 编排中通常为"步骤 3：配置 IP Farm server"——ADD IPFARM 之后、ADD RULE（POLICYTYPE=SMARTREDIRECT/WEBPROXY 引 IPFARMNAME）之前。

## 决策点：URL 与携带信息（DP 0-00088）

问：ADD IPFARMSERVER 是否配 URL/APPENDINFONAME？

| 选项 | URL | APPENDINFONAME | 典型场景 |
|---|---|---|---|
| 仅 IP（默认） | 不配 | 不配 | WebProxy（无 URL）|
| 带 URL（REDIRECT + Portal） | 配 | 视场景 | 用户Portal（携带 REQURL/IMSI）|
| 带 URL + 携带信息 | 配 | 配（[0-00023](task/UDG/20.15.2/0-00023.md) REDIRAPPENDINFO 前置） | 用户Portal 全功能 |

## 约束

- **整机最大记录数为 7200**（warning，rule-0-00110 同族）：超 90% 上报 ALM-135602215 配置数量超阈值告警；阈值可 MOD CFGTHRESHOLD（★R1.5 atom 待补） 修改 — 后果：超规格新增失败
- **同一 IPFARM 中不允许重复 VIRTUALIP 的 server**（critical，隐含）：VIRTUALIP 唯一对应 IP farm — 后果：冲突拒收
- **server IP 类型须与 IPFARM 虚拟 IP 类型一致**（critical，隐含）：SERVERIPV4 仅 IPV4 IPFARM、SERVERIPV6 仅 IPV6 IPFARM — 后果：类型不一致拒收
- **DEFAULTACT 仅对 CaptivePortal 重定向生效**（warning，隐含）：WebProxy 等非 CaptivePortal 场景 DEFAULTACT 不生效 — 后果：配置不生效
- **P-CSCF server 须与 SMF 侧 ADD PCSCFIP（★R1.5 atom 待补） 一致**（critical，隐含）：P-CSCF 连接状态检测场景本命令绑定的 server 地址须与 SMF 上的 PCSCF IP 地址保持一致 — 后果：P-CSCF 故障时 IMS 业务恢复特性无法生效
- **VXLAN 隧道组 + 心跳检测链路最多 3200**（warning，隐含）：超出规格的链路心跳检测失效、自动断开 — 后果：链路断开、心跳失效
- **APPENDINFONAME 须已配置**（critical，隐含）：引用须先经 ADD REDIRAPPENDINFO（[0-00023](task/UDG/20.15.2/0-00023.md)）配置 — 后果：引用不存在拒收
- **立即生效**（info，隐含）：该命令执行后立即生效 — 后果：server 地址变更实时影响重定向选路
