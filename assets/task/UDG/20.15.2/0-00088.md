# 配置IPFarm服务器（ADD IPFARMSERVER）

> 命令静态知识见 [ADD IPFARMSERVER](command/UDG/20.15.2/ADD-IPFARMSERVER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

向已建好的 IPFarm（0-00087）里挂 server 地址。核心是 `IPFARMNAME`（引用 0-00087）+ `IPVERSION`（必选）+ `SERVERIPV4`（地址值）。**一个 server 一条命令**——多 server 负载均衡靠重复执行本命令挂多个地址实现，不是一条命令塞多个 IP。

**必配/可选**

| 参数 | 必选 | 取值 | 说明 |
|---|---|---|---|
| IPFARMNAME | required | 引用 0-00087 已建的 farm 名 | reference |
| IPVERSION | required | IPV4 | 三特性全用 IPv4 |
| SERVERIPV4 | conditional | 见下表（IPV4 时必选） | **取值来源跨特性分叉 → DP 0-00153** |
| URL | optional | www.example.com 等 | 仅用户Portal(110281)配；开启内嵌IPv4的IPv6业务处理时建议配 |
| DEFAULTACT | optional | BLOCK(默认)/PASS | 无法重定向时 BLOCK 阻塞 / PASS 放行 |

**SERVERIPV4 取值样例（跨特性源分叉）**

| 特性 | SERVERIPV4 样例 | 来源 | URL |
|---|---|---|---|
| 用户Portal(110281) | 192.168.10.123 / .124 | 与对端协商（Portal 地址由第三方规划） | www.example.com / www.vmall.com |
| Web Proxy(110282) | 192.168.253.251 / .253 | 与对端协商（Proxy 地址由第三方规划） | （未配） |
| P-CSCF 恢复(020253) farm1 | 192.168.253.251~.254 | 全网规划（P-CSCF 地址） | （未配） |
| P-CSCF 恢复(020253) farm2 | 192.168.254.251~.254 | 本端规划（P-CSCF 地址） | （未配） |

P-CSCF 场景一个 farm 挂 4 个 P-CSCF 地址做冗余；用户Portal/Web Proxy 的重定向 server 地址由对端规划，UDG 侧需与 Portal/业务平台/Proxy 协商获取。

**脚本（用户Portal，多 server 挂多个）**
```
ADD IPFARMSERVER:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERIPV4="192.168.10.123",URL="www.example.com";
ADD IPFARMSERVER:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERIPV4="192.168.10.124",URL="www.vmall.com";
```

**脚本（P-CSCF farm1，4 个 P-CSCF）**
```
ADD IPFARMSERVER:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERIPV4="192.168.253.251";
ADD IPFARMSERVER:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERIPV4="192.168.253.252";
ADD IPFARMSERVER:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERIPV4="192.168.253.253";
ADD IPFARMSERVER:IPFARMNAME="farm_test",IPVERSION=IPV4,SERVERIPV4="192.168.253.254";
```

## 决策点

**DP 0-00153：SERVERIPV4 地址来源**（single_choice，required）——跨特性源分叉

| 选项 | 值 | 适用 |
|---|---|---|
| 与对端协商(peer) | peer_planned | 用户Portal(110281)/Web Proxy(110282) 重定向类，server 由第三方规划 |
| 全网规划 | global_planned | P-CSCF 恢复(020253) farm1 |
| 本端规划 | local_planned | P-CSCF 恢复(020253) farm2 |

default=peer_planned（向后兼容重定向类业务）。

## 约束

- 最大记录数 7200；>90% 规格上报 ALM-135602215（阈值可 MOD CFGTHRESHOLD 修改）；立即生效。
- 同一 IPFarm 中不允许有相同虚拟 IP 地址的 IPFarm 服务器。
