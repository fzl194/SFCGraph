# 基于业务事件的计费（GWFD-020306）

> 特性静态知识见 [GWFD-020306](feature/UDG/20.15.2/GWFD-020306.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

基于业务事件的计费是**[内容计费基本功能](task/UDG/20.15.2/2-00003.md)（GWFD-020301）的增强形式**：核心差异是 URR 统计类型固定为 **`OFFMETERINGTYPE=EVENT`**（离线）/ **`ONLMETERINGTYPE=EVENT`**（在线）——按匹配业务的**触发次数**（serviceSpecificUnits / numberOfEvents）累计统计，而非字节数/时长。典型场景：按 HTTP Get/Post 请求次数、彩信收发次数计费。

配置结构与内容计费/流量计费完全相同：核心对象链 **URR**（费率，EVENT 统计）→ **URRGROUP**（费率组，同时绑在线+离线 URR）→ **PCCPOLICYGRP**（策略组，通过 `EVENTCHARGEFLAG=ENABLE` 使能事件计费 + `EVENTCHGPOINT=REQUEST/RESPONSE/FINISH` 配置事件计费点，参数证据见 [ADD PCCPOLICYGRP](command/UDG/20.15.2/ADD-PCCPOLICYGRP.md)）→ **RULE**（绑流过滤器+策略组）→ **USERPROFILE**（规则容器）。**复用通用 backbone compound**（与内容计费/流量计费/在线计费共享），仅 URR 统计类型差异化为 EVENT。

4 场景（URL/IMS/any/异常）由 DP（计费方式 + 业务过滤模式）驱动，参数差异化，不建多套流程。

## 配置流程（步骤编排，每步显示含命令）

1. **License**（单命令前置）：`SET LICENSESWITCH`（开启 `LKV3G5EBCS01`）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **计费三件套**（费率定义，URR 统计类型=EVENT）：`ADD URR`（`USAGERPTMODE`+`OFFMETERINGTYPE=EVENT`/`ONLMETERINGTYPE=EVENT`）+ `ADD URRGROUP`（同时绑在线+离线 URR）+ `ADD PCCPOLICYGRP`（`EVENTCHARGEFLAG=ENABLE` 使能事件计费 + `EVENTCHGPOINT=REQUEST/RESPONSE/FINISH` 配事件计费点，参数定义见 [ADD PCCPOLICYGRP](command/UDG/20.15.2/ADD-PCCPOLICYGRP.md)）→ [1-00010](task/UDG/20.15.2/1-00010.md)
   > **EVENT 取值证据来源说明**：本特性的 `OFFMETERINGTYPE/ONLMETERINGTYPE=EVENT` 取值及事件计费点配置，来自事件计费特性概述/参考信息 evidence（[GWFD-020306-支持事件计费特性概述](evidence/UDG/20.15.2/GWFD-020306-支持事件计费特性概述_70518084.md)、[GWFD-020306-支持事件计费参考信息](evidence/UDG/20.15.2/GWFD-020306-支持事件计费参考信息_70518687.md)）。**activation 模板未演示 EVENT 取值**——原始 activation（[部署UPF](evidence/UDG/20.15.2/部署UPF_28654078.md)）复用的是内容计费模板，其 URR 演示取值为 `VOLUME`；事件计费场景需将 `OFFMETERINGTYPE/ONLMETERINGTYPE` 改为 `EVENT`，并加 `EVENTCHARGEFLAG/EVENTCHGPOINT`。
3. **过滤链**（业务匹配，DP 选）：`ADD FILTER`/`ADD FILTERIPV6`/`ADD L7FILTER`/`ADD FLOWFILTER`/`ADD FLTBINDFLOWF`/`ADD PROTBINDFLOWF` → [1-00009](task/UDG/20.15.2/1-00009.md)
4. **规则与用户模板绑定**：`ADD RULE`（POLICYTYPE=PCC，绑 FLOWFILTER+PCCPOLICYGRP）+ `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
5. **计费收尾**（缺省费率+防欺诈+刷新）：`SET URRGRPBINDING`（缺省业务+信令费率）+ `ADD SPECURRGRPLIST`（可选防欺诈）+ `SET REFRESHSRV`（必须最后）→ [1-00012](task/UDG/20.15.2/1-00012.md)
6. **全局缺省 URR 组**（单命令，可选，承接原始文档步骤 10）：`SET SPECTRAFURRGRP`（处理特殊流量/未配 URR 的业务，配合 `GLBDFTURRGSW=ENABLE` 继承）→ [SET SPECTRAFURRGRP](task/UDG/20.15.2/0-00290.md)（atom 待建）

## 决策点

**DP（特性级）：计费方式（统计类型）+ 业务过滤场景 + 事件计费点**（影响 URR 费率 + PCCPOLICYGRP + 过滤链）。事件计费场景下统计类型固定 EVENT（异常场景用 FREE 兜底）：

| 场景 | 业务 | 过滤模式（[1-00009](task/UDG/20.15.2/1-00009.md) DP） | URR 统计类型（[1-00010](task/UDG/20.15.2/1-00010.md)） | 计费方式 | 事件计费点（PCCPOLICYGRP `EVENTCHARGEFLAG`+`EVENTCHGPOINT`） | 费率组 |
|---|---|---|---|---|---|---|
| 场景1 URL | www.huawei.com（HTTP Get/Post 次数） | L34 + L7（URL 匹配） | OFFMETERINGTYPE/ONLMETERINGTYPE=**EVENT** | 在线+离线 | EVENTCHARGEFLAG=ENABLE + EVENTCHGPOINT=REQUEST/RESPONSE/FINISH（DP 选） | UrrGp_URL |
| 场景2 IMS | 语音 | L34 IPv6（服务器 IP） | OFFMETERINGTYPE/ONLMETERINGTYPE=**EVENT** | 在线+离线 | EVENTCHARGEFLAG=ENABLE + EVENTCHGPOINT=REQUEST/RESPONSE/FINISH（DP 选） | UrrGp_IMS |
| 场景3 any | 基础业务（兜底） | L34 any to any（IPv4+IPv6） | OFFMETERINGTYPE/ONLMETERINGTYPE=**EVENT** | 在线+离线 | EVENTCHARGEFLAG=ENABLE + EVENTCHGPOINT=REQUEST/RESPONSE/FINISH（DP 选） | UrrGp_any |
| 场景4 异常 | 未匹配 | 不匹配（走缺省） | OFFMETERINGTYPE/ONLMETERINGTYPE=**FREE**（不计费） | FREE | 不适用（免费 RG 不支持事件计费） | UrrGp_abnormal |

**联动**：
- 统计类型 EVENT → URR（[1-00010](task/UDG/20.15.2/1-00010.md)）按业务事件次数累计，话单/CCR 上报 `numberOfEvents`+`eventTimeStamps`（离线）/ `serviceSpecificUnits`（在线/融合）。
- 事件计费点（`EVENTCHGPOINT=REQUEST/RESPONSE/FINISH`，须先 `EVENTCHARGEFLAG=ENABLE`）→ PCCPOLICYGRP（[1-00010](task/UDG/20.15.2/1-00010.md)）配置：REQUEST 直接占用配额；RESPONSE/FINISH 事件发生即**预占用**配额，识别到成功响应才**正式占用**，未识别则**返还**预占配额。
- 计费方式（在线/离线/融合）→ URR 的 `USAGERPTMODE` + URRGROUP 同时绑在线+离线 URR（`UPURRNAME1`/`DOWNURRNAME1`=离线，`UPURRNAME2`/`DOWNURRNAME2`=在线）。
- 过滤模式 → 过滤链（[1-00009](task/UDG/20.15.2/1-00009.md)）出现哪些过滤器。
- 缺省费率 → 收尾（[1-00012](task/UDG/20.15.2/1-00012.md)）的 `DFTURRGRPNAME`（通常 any 组）+ `DFTSIGURRGNAME`（异常组）；可选 `SET SPECTRAFURRGRP` 配全局缺省处理特殊流量。

## 约束（特性级 rule）

- **License 前置**（critical）：`SET LICENSESWITCH` 开启 `LKV3G5EBCS01`（82200CKE） — 未开 License 业务命令不生效。
- **依赖特性**（critical）：[SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，业务识别必选）+ [内容计费基本功能](feature/UDG/20.15.2/GWFD-020301.md)（GWFD-020301，事件计费是其增强） — 依赖链 SA-Basic → 内容计费 → 事件计费；按业务类型还需 SA-Web Browsing（GWFD-110103，HTTP）/ SA-Mobile（GWFD-110105，WAP/MMS）。
- **URRGROUP 必须同时配在线+离线 URR**（critical）：只配一种且与用户计费模式不匹配则无法计费 — 计费失败。
- **融合计费特殊**（critical）：SMF `RGAPPLIED=DEFAULT`（SET USRPROFCHARGE/SET APNCHARGECTRL/ADD CHARGEMETHOD）时 URRGROUP 不能同时配离线+在线 URR — 否则计费冲突。
- **SMF/UPF 一致性**（critical）：`URRID`/`USAGERPTMODE`/`OFFMETERINGTYPE`/`ONLMETERINGTYPE`/`RULENAME`/`POLICYTYPE`/`POLICYNAME`/`USERPROFILENAME` 在 SMF 与 UPF（及 PCF）必须一致 — 不一致则计费异常/策略不生效（告警 ALM-81054/ALM-81026）。
- **仅支持 SCUR 在线模式**（critical）：事件计费仅支持 SCUR（Session Charging with Unit Reservation）；ECUR 仅在线计费额外支持（需 PGW-C/UPF 开 BIT1250，彩信群发场景）。
- **免费 RG 不支持事件计费**（critical）：`OFFMETERINGTYPE=FREE`/`ONLMETERINGTYPE=FREE` 的 RG 不能使能事件计费。
- **不支持 Default Quota**（critical）：事件计费不支持 Default Quota 功能（与在线计费 `SET UPDEFAULTQUOTA` 不兼容）。
- **FLOWFILTER 必须至少绑一个过滤条件**（critical）：否则 Rule 无效。
- **REFRESHSRV 必须最后执行**（critical）：Filter 变更才生效。
- **RULE 优先级**（warning）：数字越小越高；只绑三四层的 RULE 优先级低于绑七层的；any 规则优先级最低（如 65000）。
- **每个 USERPROFILE 必绑缺省费率**（critical）：`DFTURRGRPNAME`（业务）+ `DFTSIGURRGNAME`（信令） — 否则未匹配费率的报文无法计费。
- **配额预占用**（warning）：RESPONSE/FINISH 计费点下，事件发生即预留配额，成功次数+预留次数≥门限或配额时即申请新配额。
- **RG 使能不可变**（warning）：整个 PDU 会话过程中某 RG 是否使能事件计费不会发生变化。
- **协议与加密限制**（warning）：仅支持 HTTP1.x，不支持 HTTP2.0；不支持加密场景基于 host 的计费；不支持 PCC 动态规则事件计费。
- **防欺诈绑定约束**（warning）：防欺诈开启时，RULE 绑 URL 类型七层 filter 必须同时绑三四层 filter；防欺诈未开启时可仅绑七层。
- **事件计费专属软参**（info，见 [GWFD-020306-支持事件计费参考信息](evidence/UDG/20.15.2/GWFD-020306-支持事件计费参考信息_70518687.md)）：
  - **BIT417**（warning）：控制事件计费 URLNode 节点耗尽后的动作——节点资源耗尽会影响事件计费持续处理，按容灾策略取值。
  - **BIT1250**（warning）：控制是否启用事件计费**群发彩信**功能——彩信 ECUR 事件计费场景 PGW-C 和 PGW-U **均需开启**（见约束"仅支持 SCUR 在线模式"中 ECUR 例外）。
  - **BIT419**（info）：事件计费是否使能**时间戳记录**功能的控制开关——影响话单/CCR 中 `eventTimeStamps` 是否记录（离线话单 `eventBasedChargingInformation.eventTimeStamps`）。
- **性能**（info）：七层解析+事件统计对吞吐量有影响；HTTP 下载的断点续传/多线程/多源下载会当作多个事件计费；HTTP connect 报文（建安全隧道）不触发事件计费。
