# 配置QoS接口信任（ADD QOSIFTRUST）

> 命令静态知识见 [ADD QOSIFTRUST](command/UDG/20.15.2/ADD-QOSIFTRUST.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD QOSIFTRUST 把物理接口绑定到指定 DS 域，使接口支持简单流分类并对报文优先级进行修改。DSNAME 引用 DS 域、IFNAME 引用物理接口，两参数均必填；FLAG8021P 仅在以太子接口/GE 子接口/Eth-Trunk 主接口场景可选开启。

**两种取值范式（从证据数据规划表归纳）**

| 场景 | DSNAME | IFNAME | FLAG8021P | 典型行 |
|---|---|---|---|---|
| QoS 基础（GWFD-010201，默认域） | default | Ethernet66/0/5（LST INTERFACE 查询） | 不配 | `ADD QOSIFTRUST:DSNAME="default",IFNAME="Ethernet66/0/5";` |
| QoS 简单流分类（IPFD-012001，多 DS 域） | default / 5p3d（按 VLAN→IP 映射分别绑定） | Ethernet64/0/2、Ethernet64/0/3 | 仅以太子接口/GE 子接口/Eth-Trunk 主接口支持，按需 TRUE | `ADD QOSIFTRUST:DSNAME="5p3d",IFNAME="Ethernet64/0/3";` |

**关键规则**
- **入口/出口两方向都要绑定一次**：同一接口的 ingress（SET QOSBA 配的入口 BA 映射）与 egress（SET QOSPHB 配的出口 PHB 映射）两条路径都需要本命令把 DS 域绑到接口上（DP 0-00016 选 opt-ingress/opt-egress 各执行一次）。
- **DSNAME** 引用 [配置DS域](task/UDG/20.15.2/0-00066.md)（ADD QOSDIFFERSERV）；同一接口可绑不同 DSNAME。
- **IFNAME** 引用 [配置接口](task/UDG/20.15.2/0-00056.md)（物理接口），可用 `LST INTERFACE` 查询。
- **FLAG8021P 接口类型约束**：只有以太子接口、GE 子接口、Eth-Trunk 主接口支持 802.1p 映射；其他接口类型置 TRUE 将被拒收。

## 约束

- **rule 0-00284**（capacity）系统最大记录数 65536 — 后果：超限 ADD 失败
- **rule 0-00284**（dependency）FLAG8021P 仅以太子接口/GE 子接口/Eth-Trunk 主接口支持；与 DP 0-00016 方向绑定，入口/出口两方向各自执行路径调用本 atom — 后果：接口不支持置 FLAG8021P=true 被拒收
