# 设置物联网能力上报（SET IOTCAPABILITY）

> 命令静态知识见 [SET IOTCAPABILITY](command/UDG/20.15.2/SET-IOTCAPABILITY.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET IOTCAPABILITY 是物联网接入能力上报的"全局开关"命令：通过 NBIOT 参数使能 UPF 的 NB-IoT 接入能力，使能后 UDG 在与 UNC 偶联建立/更新时，于消息中携带 UP Function Features 信元表示支持 NB-IoT，UNC 随后向 UDG 分配 NB-IoT 用户。本命令的核心配置空间是单一维度的使能开关 NBIOT，整机最多记录数 **1**（即全局唯一），**立即生效**，系统初始值即 ENABLE。

### 配置维度 1：NB-IoT 接入能力上报开关（参数 NBIOT，可选/默认 ENABLE）

| 取值 | 作用 | 适用 NF | 备注 |
|---|---|---|---|
| **ENABLE**（默认） | 使能 UPF 的 NB-IoT 接入能力；偶联消息携带 UP Function Features | UPF | 系统初始值；激活 NB-IoT 特性时显式置位确认 |
| **DISABLE** | 关闭 UPF 的 NB-IoT 接入能力；偶联消息不再上报该特性 | UPF | 拆除 NB-IoT 业务时配置 |

> **典型范式（GWFD-010296 NB-IoT终端标准接入）**：
> ```
> SET IOTCAPABILITY:NBIOT=ENABLE;
> ```

**步骤位置**：在所属 feature 编排中通常为"步骤 1：使能 NB-IoT 能力上报"——整机仅 1 条记录，独立于其它特性配置；NB-IoT 完整业务需 UDG（用户面）+ SGW-C/PGW-C（控制面）协同配置，本命令仅用户面侧开关。

## 决策点：无

## 约束

- **整机最大记录数为 1**（warning，隐含）：单板仅 1 条 SET IOTCAPABILITY 记录；后续 SET 覆盖前值；存在系统初始记录 NBIOT=ENABLE — 后果：重复执行仅覆盖初始值
- **立即生效**（info，隐含）：该命令执行后立即生效，无需重启 — 后果：能力上报变更影响后续偶联
- **NB-IoT 完整业务需用户面+控制面协同**（warning，隐含）：本命令仅 UDG 侧开关；SGW-C/PGW-C 上 NB-IoT 相关配置须同步生效 — 后果：仅 UDG 侧使能但控制面无配置，NB-IoT 业务无法正常建立
