# 增加IPsec策略绑定提议（ADD PROPATTACHIPSECPROPOSAL）

> 命令静态知识见 [ADD PROPATTACHIPSECPROPOSAL](command/UDG/20.15.2/ADD-PROPATTACHIPSECPROPOSAL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是 **IPsec SA 协商链**的关键一环：把 IPsec 安全提议按序列号挂到 IPsec 策略下，IKE 协商时按 SEQUENCENUMBER 顺序选用；POLICYNAME 由 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 产出，IPSECPROPNAME 由 [0-00212](task/UDG/20.15.2/0-00212.md) ADD IPSECPROPOSALIPSEC（或 [ADD IPSECPROPOSAL6（★R1.5 atom 待补）](command/UDG/20.15.2/ADD-IPSECPROPOSAL6.md)）产出。

## 配置方法（配置方法字典）

ADD PROPATTACHIPSECPROPOSAL 是"IPsec 策略绑定提议"命令。5 个必选参数（POLICYNAME + SEQUENCENUMBER + POLICYMODE + TEMPLATEMODE + IPSECPROPNAME），无任何可选参数。配置空间有 5 个维度：**策略引用**（POLICYNAME 引 IPSECPOLICY 产出名）、**序列号**（SEQUENCENUMBER 决定协商优先级）、**策略模式**（POLICYMODE=Isakmp 固定走 IKE）、**模板模式**（TEMPLATEMODE=None/PolicyTemplate 决定是否走策略模板）、**提议引用**（IPSECPROPNAME 引 IPSECPROPOSAL 产出名）。1 个特性命中（IPFD-015004 IPSec功能，普通 IPv4/IPv6/主备/GRE over IPsec/OSPF over IPsec 等 8 个 IPsec 子场景）共享同一命令空间，仅 SEQUENCENUMBER/IPSECPROPNAME 组合取值不同。

### 配置维度 1：策略引用（参数 POLICYNAME，必选）

POLICYNAME 引用 IPsec 策略名，本命令的所有 SEQUENCENUMBER 都挂在该策略下：

| 取值形式 | 范围约束 | 来源 | 适用场景 |
|---|---|---|---|
| **字符串 1~15（不区分大小写）** | 必填 | 引用 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 的 POLICYNAME | IPFD-015004 全部 8 子场景（10002/10003/10004/10005/10006/10007/10008/10009） |

> **链路族对齐**：POLICYNAME 链路族（IPv4/IPv6）须与下挂的 IPSECPROPNAME 链路族**一致**——IPv4 策略挂 IPv4 提议，IPv6 策略挂 IPv6 提议。

### 配置维度 2：序列号（参数 SEQUENCENUMBER，必选）

SEQUENCENUMBER 决定 IKE 协商时提议的优先级（数值越小越优先）：

| 取值 | 联动 | 适用场景 |
|---|---|---|
| **整数 1~10000** | 必填 | 同一 POLICYNAME 下可挂多条不同 SEQUENCENUMBER 的提议；IPFD-015004 10002~10007 范例常见 `SEQUENCENUMBER=10`（主提议）/ `SEQUENCENUMBER=20`（备提议），10008（多 Sequence 策略）则挂多条（10/20 等）形成优先级链 |

> **优先级语义**：同一策略下多条 SEQUENCENUMBER 不同 → IKE 协商按数值从小到大依次尝试，匹配成功即采用；故 SEQUENCENUMBER=10 的提议比 SEQUENCENUMBER=20 的优先。

### 配置维度 3：策略模式（参数 POLICYMODE，必选）

POLICYMODE 决定 SA 的建立方式：

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **Isakmp**（ISAKMP 模式） | 使用 IKE 协议自动协商建立 IPsec SA | IPFD-015004 全部 8 子场景（10002/10003/10004/10005/10006/10007/10008/10009）默认范式 |

> **唯一可用模式**：当前命令 POLICYMODE 仅支持 Isakmp——这意味着 IPsec SA 必须经 IKE 协议协商建立（不支持手工密钥 SA）。

### 配置维度 4：模板模式（参数 TEMPLATEMODE，必选）

TEMPLATEMODE 决定是否走策略模板建立 SA：

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **None**（无） | 不使用策略模板，直接按本命令绑定的 SEQUENCENUMBER/IPSECPROPNAME 进行 IKE 协商 | IPFD-015004 全部 8 子场景默认范式（10002~10009 全部用 `TEMPLATEMODE=None`） |
| **PolicyTemplate**（策略模板模式） | 绑定一个策略模板到 ISAKMP，使用策略模板建立 IPsec SA | 仅支持用户级 IPsec（系统级 IPsec 不支持模板模式） |

> **系统级硬约束**：本命令**不支持用户级 IPSEC**——系统级场景（IPFD-015004）TEMPLATEMODE 必须配 None；PolicyTemplate 仅用于用户级场景。

### 配置维度 5：提议引用（参数 IPSECPROPNAME，必选）

IPSECPROPNAME 引用 IPsec 安全提议名：

| 取值形式 | 范围约束 | 来源 | 适用场景 |
|---|---|---|---|
| **字符串 1~15（不区分大小写）** | 必填 | 引用 [0-00212](task/UDG/20.15.2/0-00212.md) ADD IPSECPROPOSALIPSEC 的 IPSECPROPNAME（IPv4）；或 [ADD IPSECPROPOSAL6（★R1.5 atom 待补）](command/UDG/20.15.2/ADD-IPSECPROPOSAL6.md) 的 IPSECPROPNAME（IPv6） | IPFD-015004 10002 范例 `tran1`；10008 范例 `Proposal1`；OSPF/GRE over IPsec 范例 `1` |

> **链路族对齐铁律**：IPSECPROPNAME 链路族须与 POLICYNAME 链路族**一致**——IPv4 策略只能挂 IPv4 提议（[0-00212](task/UDG/20.15.2/0-00212.md) IPSECPROPOSALIPSEC 产出），IPv6 策略只能挂 IPv6 提议（IPSECPROPOSAL6 产出）。链路族错配则 IKE 协商时提议匹配失败，SA 无法建立。

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，POLICYNAME=map1 + SEQUENCENUMBER=10 + Isakmp + None + tran1）**：
```
ADD PROPATTACHIPSECPROPOSAL:POLICYNAME="map1",SEQUENCENUMBER=10,POLICYMODE=Isakmp,TEMPLATEMODE=None,IPSECPROPNAME="tran1";
```

**典型脚本（IPFD-015004 多 Sequence IPsec 策略，单策略下挂两条不同优先级提议）**：
```
ADD PROPATTACHIPSECPROPOSAL:POLICYNAME="map1",SEQUENCENUMBER=10,POLICYMODE=Isakmp,TEMPLATEMODE=None,IPSECPROPNAME="tran1";
ADD PROPATTACHIPSECPROPOSAL:POLICYNAME="map1",SEQUENCENUMBER=20,POLICYMODE=Isakmp,TEMPLATEMODE=None,IPSECPROPNAME="tran1";
```

**典型脚本（IPFD-015004 IPv6 IPsec 隧道，POLICYNAME=policy1 + IPv6 链路）**：
```
ADD PROPATTACHIPSECPROPOSAL:POLICYNAME="policy1",SEQUENCENUMBER=1,POLICYMODE=Isakmp,TEMPLATEMODE=None,IPSECPROPNAME="Proposal1";
```

> 命令 notes：**最大记录数 2000**；**执行后立即生效**；**不支持用户级 IPSEC**——本命令仅作用于系统级 IPsec 策略；**权限**：G_1 管理员 / G_2 操作员命令组。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：IPsec 协商链中段——策略绑提议"——前置 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY（产出 POLICYNAME）+ [0-00212](task/UDG/20.15.2/0-00212.md) ADD IPSECPROPOSALIPSEC（产出 IPSECPROPNAME）；下游 [0-00307](task/UDG/20.15.2/0-00307.md) ADD IPSECINTFCFGIPSEC（策略挂接口）+ **IKE 协商生效**（IKE SA 建立后按 SEQUENCENUMBER 选用提议建立 IPsec SA）。

## 决策点：IPsec 提议链路族选择（DP 0-00314）

问：本 PROPATTACH 绑定的是 IPv4 链路还是 IPv6 链路？（决定 POLICYNAME + IPSECPROPNAME 链路族）

| 选项 | POLICYNAME 链路族 | IPSECPROPNAME 链路族 | TEMPLATEMODE | 典型特征 |
|---|---|---|---|---|
| IPv4 IPsec 链路（默认） | IPv4 策略（[0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 产出） | IPv4 提议（[0-00212](task/UDG/20.15.2/0-00212.md) ADD IPSECPROPOSALIPSEC 产出） | None | IPFD-015004 10002/10004/10006/10007/10008/10009 默认范式 |
| IPv6 IPsec 链路 | IPv6 策略（[ADD IPSECPOLICY6（★R1.5 atom 待补）](command/UDG/20.15.2/ADD-IPSECPOLICY6.md) 产出） | IPv6 提议（[ADD IPSECPROPOSAL6（★R1.5 atom 待补）](command/UDG/20.15.2/ADD-IPSECPROPOSAL6.md) 产出） | None | IPFD-015004 10003/10005 IPv6 范式 |

> **链路族铁律贯穿**：POLICYNAME 链路族 = IPSECPROPNAME 链路族 = IKE 协商的实际链路族——三方对齐才能完成协商；IPv4/IPv6 错配 IKE 提议匹配失败。

## 约束

- **最大记录数 2000**（warning，rule-0-01314）：系统最大记录数 2000 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-01314）：命令执行后立即生效 — 后果：误操作即刻生效影响 IKE 协商
- **POLICYNAME 须先建**（critical，rule-0-01314）：POLICYNAME 必须先经 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 或 [ADD IPSECPOLICY6（★R1.5 atom 待补）](command/UDG/20.15.2/ADD-IPSECPOLICY6.md) 创建 — 后果：引用未建策略命令拒收
- **IPSECPROPNAME 须先建**（critical，rule-0-01314）：IPSECPROPNAME 必须先经 [0-00212](task/UDG/20.15.2/0-00212.md) ADD IPSECPROPOSALIPSEC 或 [ADD IPSECPROPOSAL6（★R1.5 atom 待补）](command/UDG/20.15.2/ADD-IPSECPROPOSAL6.md) 创建 — 后果：引用未建提议命令拒收
- **链路族对齐铁律**（critical，rule-0-01314）：POLICYNAME 链路族必须与 IPSECPROPNAME 链路族一致（IPv4 配 IPv4 / IPv6 配 IPv6） — 后果：链路族错配 IKE 提议匹配失败，SA 无法建立
- **POLICYMODE 固定 Isakmp**（info，rule-0-01314）：当前仅支持 ISAKMP 模式（不支持手工密钥 SA） — 后果：配非 Isakmp 命令拒收
- **TEMPLATEMODE 系统级硬约束 None**（warning，rule-0-01314）：本命令不支持用户级 IPSEC；系统级场景 TEMPLATEMODE 必须 None；PolicyTemplate 仅用于用户级 — 后果：系统级误配 PolicyTemplate 命令拒收
- **POLICYNAME 长度**（info，rule-0-01314）：字符串 1~15（不区分大小写） — 后果：超长命令拒收
- **IPSECPROPNAME 长度**（info，rule-0-01314）：字符串 1~15（不区分大小写） — 后果：超长命令拒收
- **SEQUENCENUMBER 范围**（info，rule-0-01314）：整数 1~10000 — 后果：超范围命令拒收
- **同策略多 SEQUENCENUMBER**（info，隐含）：同一 POLICYNAME 下可挂多条不同 SEQUENCENUMBER 的提议，IKE 按数值从小到大依次尝试 — 后果：SEQUENCENUMBER=10 比 SEQUENCENUMBER=20 优先
- **权限要求**（info，rule-0-01314）：需 G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
