# 修改L3VPN实例地址族(路由交叉)（MOD VPNINSTAF）

> 命令静态知识见 [MOD VPNINSTAF](command/UDG/20.15.2/MOD-VPNINSTAF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

MOD VPNINSTAF 用于修改 L3VPN 实例的地址族配置，是 **GWFD-020423（MPLS OptionB 路由交叉）前置**：为两个 VPN 实例分别设置路由标识 RD 和标签分配模式。**MPLS OptionB 必须配 VRFRD，不同 VPN 实例 RD 必须不同**。VRFLABELMODE 固定取 `perInstance`（每实例一标签，比 `perRoute` 省标签资源）。高危命令，立即生效。1 个特性命中（020423 路由交叉，对 2 个 VRF 各执行一次）。

### 配置维度 1：地址族（参数 AFTYPE，必选）

| 取值 | 作用 |
|---|---|
| **ipv4uni**（020423 固定） | IPv4 单播地址族；当前路由交叉唯一场景 |
| 其他 | 见命令真相表枚举（020423 未用） |

### 配置维度 2：路由标识 RD（参数 VRFRD，必选）

| 取值 | 约束 |
|---|---|
| **AS:NN 格式**（如 `1:1`、`2:2`） | **不同 VPN 实例 RD 必须不同**；本端规划 |

> RD 是路由交叉的关键标识：发送 VPNv4 路由携带 RD；接收端按 RD+IPv4 前缀唯一标识路由来源。

### 配置维度 3：标签分配模式（参数 VRFLABELMODE，必选）

| 取值 | 作用 |
|---|---|
| **perInstance**（020423 固定） | 每实例一标签，**省标签资源**（比 perRoute 模式） |
| perRoute | 每路由一标签（020423 未用） |

**任务示例（020423 范式，VRF_networkA/B 各执行一次）**：

```
MOD VPNINSTAF: VRFNAME="VRF_networkA", AFTYPE=ipv4uni, VRFRD="1:1", VRFLABELMODE=perInstance;
MOD VPNINSTAF: VRFNAME="VRF_networkB", AFTYPE=ipv4uni, VRFRD="2:2", VRFLABELMODE=perInstance;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：改 VPN 地址族配 RD"——前置链：ADD L3VPNINST(0-00071) → ADD VPNINSTAF(0-00072) → MOD VPNINSTAF(本任务)。

## 决策点

（本命令在路由交叉场景下 RD 与 VRFLABELMODE 均固定取值，无独立决策分支。）

## 约束

- **高危命令**（critical，rule-0-00145）：删除 VRFRD 会级联删除该 VPN 地址族下配置 + BGP 配置 — 后果：删 RD 级联影响 BGP，运维窗口慎用
- **不可修改系统保留实例**（critical，rule-0-00145）：不能修改 `__mpp_vpn_inner__` / `__mpp_vpn_inner_server__` — 后果：改保留实例命令拒收
- **VPN 地址族前置必备**（critical，rule-0-00145）：前置 VPN 地址族必须已由 ADD VPNINSTAF(0-00072) 创建 — 后果：地址族未建命令拒收
- **不同 VPN RD 必须不同**（critical，rule-0-00145）：不同 VPN 实例的 VRFRD 值不可相同 — 后果：RD 重复命令拒收或路由冲突
- **立即生效**（warning，rule-0-00145）：执行后立即生效 — 后果：误改 RD 即刻影响 VPNv4 路由收发
- **7 个 optional 未绑**（info，rule-0-00145）：IMPOLICYNAME/EXPOLICYNAME 路由策略、VRFLABEL、TNLPOLICYNAME、FRRENABLE、LOCALCROSSNHPMOD、EXPLYADDERTFIRST 在 020423 路由交叉场景未用
