# 修改L3VPN实例地址族(路由交叉)（MOD VPNINSTAF）

> 命令静态知识见 [MOD VPNINSTAF](command/UDG/20.15.2/MOD-VPNINSTAF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是 **GWFD-020423（MPLS OptionB 路由交叉）前置**：为两个 VPN 实例分别设置路由标识 RD 和标签分配模式。MPLS OptionB 必须配 VRFRD，不同 VPN 实例 RD 必须不同。VRFLABELMODE 固定取 `perInstance`（每实例一标签，比 `perRoute` 省标签）。高危命令，立即生效。

**配置范式（对每个 VRF 实例各执行一次，共两行）**

| 参数 | VRF_networkA 取值 | VRF_networkB 取值 | 来源 | 说明 |
|---|---|---|---|---|
| VRFNAME | `VRF_networkA` | `VRF_networkB` | 引用 [0-00071](task/UDG/20.15.2/0-00071.md).VRFNAME | VPN 实例名，ADD L3VPNINST 先建 |
| AFTYPE | `ipv4uni` | `ipv4uni` | 本端规划（固定） | 地址族类型 |
| VRFRD | `1:1` | `2:2` | 本端规划 | **不同 VPN 必须 RD 不同** |
| VRFLABELMODE | `perInstance` | `perInstance` | 本端规划（固定） | 每实例一标签，省标签资源 |

**任务示例**

```
MOD VPNINSTAF: VRFNAME="VRF_networkA", AFTYPE=ipv4uni, VRFRD="1:1", VRFLABELMODE=perInstance;
MOD VPNINSTAF: VRFNAME="VRF_networkB", AFTYPE=ipv4uni, VRFRD="2:2", VRFLABELMODE=perInstance;
```

**前置链**：ADD L3VPNINST(0-00071) → ADD VPNINSTAF(0-00072) → MOD VPNINSTAF(0-00145)。

7 个 optional（IMPOLICYNAME/EXPOLICYNAME 路由策略、VRFLABEL、TNLPOLICYNAME、FRRENABLE、LOCALCROSSNHPMOD、EXPLYADDERTFIRST）路由交叉当前未用，不绑。

## 约束

- **rule-0-00233**（error）高危命令；删除 VRFRD 会级联删除该 VPN 地址族下配置 + BGP 配置；不能修改系统保留实例 `__mpp_vpn_inner__`/`__mpp_vpn_inner_server__`；前置 VPN 地址族必须已创建（0-00072 ADD 族）；立即生效
