# 计费收尾（缺省费率+防欺诈+刷新）

> **跨域通用 backbone**（对齐 ConfigTask 1-00004）：计费流程收尾。每个 USERPROFILE 必绑缺省费率；REFRESHSRV 必须最后执行。被引用于 [2-00003](task/UDG/20.15.2/2-00003.md)（内容计费）。

## 配置方法

三步：缺省费率（每用户模板）→ 防欺诈（可选）→ 刷新生效（必须最后）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 缺省费率 | SET URRGRPBINDING | USERPROFILENAME + DFTURRGRPNAME(缺省业务费率) + DFTSIGURRGNAME(缺省信令费率) |
| 防欺诈（可选） | ADD SPECURRGRPLIST | URRGROUPNAME |
| 刷新生效 | SET REFRESHSRV | REFRESHTYPE（DP 取值，见下） |

**典型脚本**：
```
SET URRGRPBINDING:USERPROFILENAME="userprofile001",DFTURRGRPNAME="UrrGp_any",DFTSIGURRGNAME="UrrGp_abnormal";
ADD SPECURRGRPLIST:URRGROUPNAME="UrrGp_URL";
SET REFRESHSRV:REFRESHTYPE=ALL;  # REFRESHTYPE 按特性取值（见下 DP），不固定 ALL
```

**步骤位置**：计费流程**最后**（承接 [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定，USERPROFILE 已建）；REFRESHSRV 必须最后执行。

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| 启用防欺诈 | 将有欺诈风险的费率加入 SPECURRGRPLIST（ADD SPECURRGRPLIST） |
| 不启用防欺诈（可选，默认） | 省略 ADD SPECURRGRPLIST |
| **REFRESHTYPE 取值** | 见下表（按各特性原始 activation 文档取值） |

**REFRESHTYPE 取值**（按特性原始 activation 文档）：

| 选项 | 影响 | 用于 |
|---|---|---|
| USERPROFILE | 按用户模板刷新（在线/离线原始文档取值） | [2-00004](task/UDG/20.15.2/2-00004.md) 在线 / [2-00005](task/UDG/20.15.2/2-00005.md) 离线 |
| ALL | 全量刷新（内容/融合原始文档取值） | [2-00003](task/UDG/20.15.2/2-00003.md) 内容 / [2-00006](task/UDG/20.15.2/2-00006.md) 融合 / [2-00007](task/UDG/20.15.2/2-00007.md) 时长 / [2-00008](task/UDG/20.15.2/2-00008.md) 流量 / [2-00009](task/UDG/20.15.2/2-00009.md) 事件 |

## 约束

- **每个 USERPROFILE 必绑缺省费率**（critical）：DFTURRGRPNAME + DFTSIGURRGNAME — 否则未匹配费率的报文无法计费。
- **SET REFRESHSRV 必须最后执行**（critical）：在 RULE 配置完成后执行 — 否则 FILTER 配置变更不生效。
- **REFRESHTYPE 取值**（warning）：须与各特性原始 activation 文档一致（在线/离线=USERPROFILE，内容/融合=ALL），不能一律 ALL。
- **缺省费率须已配置**（critical）：DFTURRGRPNAME/DFTSIGURRGNAME 须已通过 ADD URRGROUP 配置 — 否则绑定引用空对象。
