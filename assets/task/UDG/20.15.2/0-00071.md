# 配置L3VPN实例（ADD L3VPNINST）

> 命令静态知识见 [ADD L3VPNINST](command/UDG/20.15.2/ADD-L3VPNINST.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD L3VPNINST 创建一个 L3VPN 实例，只需 VRFNAME（VRFDESCRIPTION 是描述字段，6 个特性数据规划表均未引用，按通用门槛跳过）。VRFNAME 是后续 [配置L3VPN实例地址族](task/UDG/20.15.2/0-00072.md)（ADD VPNINSTAF）、ADD VPNINST、ADD OSPF、ADD LOGICINF 等命令的 VRFNAME 引用源。命令本体只录主体，地址族（AFTYPE/VRFRD）的 IPv4/IPv6 分叉由 DP 0-00017 在配套的 VPNINSTAF 表达。

**取值范式（从证据数据规划表归纳，VRFNAME 全网统一规划）**

| 场景 | VRFNAME 典型值 | 配套地址族 | 典型行 |
|---|---|---|---|
| Untrusted Non-3GPP / ePDG 互通（GWFD-010155，主场景） | vpn-s2b | ipv4uni（DP 0-00017 ip-version-v4） | `ADD L3VPNINST:VRFNAME="vpn-s2b";` |
| Untrusted Non-3GPP IPv6（GWFD-010155 OSPFv3+BFD） | vpn-paif | ipv6uni（DP 0-00017 ip-version-v6） | `ADD L3VPNINST:VRFNAME="vpn-paif";` |
| IPsec 隧道（IPFD-015004 各组网） | vrf1 | ipv4uni / ipv6uni | `ADD L3VPNINST:VRFNAME="vrf1";` |
| 地址分配（GWFD-010104/010105） | vpn1 | ipv4uni | `ADD L3VPNINST:VRFNAME="vpn1";` |
| IPv6 承载 / N6 Gi SGi 组网（GWFD-020401/020402） | VPN_Internet / VPN_CN | ipv6uni | `ADD L3VPNINST:VRFNAME="VPN_Internet";` |
| 路由交叉（GWFD-020423） | VRF_networkB | ipv4uni（按业务 IP 类型规划） | `ADD L3VPNINST:VRFNAME="VRF_networkB";` |

**关键规则**
- **VRFNAME 全网统一规划**：一条通信路径上所有节点绑定的 VPN 实例必须一致才能正常通信。
- **系统保留名禁用**：`_public_`、`__mpp_vpn_inner__`、`__mpp_vpn_inner_server__` 不能增加。
- **配套必加地址族**：ADD L3VPNINST 后须紧跟 [配置L3VPN实例地址族](task/UDG/20.15.2/0-00072.md)（ADD VPNINSTAF）使能对应地址族，否则 VPN 相关配置无法继续。

## 约束

- **rule 0-00162**（capacity）系统最大记录数 4096 — 后果：超规格新增失败
- **rule 0-00163**（value/critical）VRFNAME 不能取系统保留名 `_public_`、`__mpp_vpn_inner__`、`__mpp_vpn_inner_server__` — 后果：新增失败 / 系统内置 VPN 被污染
