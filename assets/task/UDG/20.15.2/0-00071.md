# 配置L3VPN实例（ADD L3VPNINST）

> 命令静态知识见 [ADD L3VPNINST](command/UDG/20.15.2/ADD-L3VPNINST.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD L3VPNINST 是 VPN 体系的"实例创建层"——创建一个 L3VPN 实例，产出 VRFNAME 作为后续 [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF、ADD VPNINST、ADD OSPF、ADD LOGICINF 等命令的引用键。本命令的核心配置空间有 2 个维度：**VPN 命名**（VRFNAME 字符串 1~31）、**地址族使能**（需后续 ADD VPNINSTAF 联动）。

### 配置维度 1：VPN 命名（参数 VRFNAME，必选）

| 取值 | 作用 | 配套地址族 | 典型场景 |
|---|---|---|---|
| **vpn-s2b** | ePDG/S2b 接口 VPN（Untrusted Non-3GPP） | ipv4uni（+VRFRD=100:1） | 010155 Untrusted Non-3GPP 互通 |
| **vpn-paif** | PAIF 接口 VPN（Non-3GPP IPv6） | ipv6uni | 010155 OSPFv3 + IPv6 BFD |
| **vpn1** | 通用业务 VPN | ipv4uni | 010104/010105 地址分配 |
| **VPN_Internet / VPN_CN** | 业务 VPN（IPv6 承载/N6 组网） | ipv6uni | 020401/020402 IPv6 业务 |
| **vrf1** | IPSec 隧道 VRF | ipv4uni/ipv6uni | IPFD-015004 IPSec 双栈 |
| **VRF_networkB** | 路由交叉业务 VRF | ipv4uni | 020423 路由交叉 |
| **`_public_`/`__mpp_vpn_inner__`/`__mpp_vpn_inner_server__`** | **系统保留名**（禁用） | — | 不可用（rule-0-00163） |

### 配置维度 2：地址族使能（下游联动，非本命令必填）

| 后续动作 | 联动命令 | 说明 |
|---|---|---|
| 使能 IPv4 地址族 | [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF AFTYPE=ipv4uni + VRFRD=100:1 | 标准 IPv4 VPN |
| 使能 IPv6 地址族 | [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF AFTYPE=ipv6uni | IPv6 承载 |
| 使能 IPv4+IPv6 双栈 | 同上两条 | 单 L3VPN 实例可配双地址族（规格高于 L3VPNINST 的 4096） |

| 参数 | 必选性 | 取值 | 作用 |
|---|---|---|---|
| `VRFNAME` | 必选 | 字符串 1~31 | VPN 实例名称（**全网统一规划**：一条通信路径上所有节点绑定的 VPN 实例必须一致） |
| `VRFDESCRIPTION` | 可选 | 字符串 1~242 | VPN 实例描述 |

**典型脚本（010155 Untrusted Non-3GPP ePDG 互通）**：
```
ADD L3VPNINST:VRFNAME="vpn-s2b";
```

**典型脚本（020401/020402 IPv6 业务 VPN）**：
```
ADD L3VPNINST:VRFNAME="VPN_Internet";
ADD L3VPNINST:VRFNAME="VPN_CN";
```

**步骤位置**：在所属 feature 编排中位于"步骤 1：创建 L3VPN 实例"——紧跟 [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF 使能地址族；后续接 [0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE / [0-00063](task/UDG/20.15.2/0-00063.md) ADD SRROUTE6 静态路由或 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 动态路由。

## 决策点

（无。本命令仅 VRFNAME 一个必选参数，命名按业务/全网统一规划决定。）

## 约束

- **最大记录数 4096**（critical，rule-0-00162）：系统最大记录数 4096 — 后果：超规格新增失败
- **系统保留名禁用**（critical，rule-0-00163）：VRFNAME 不能取 `_public_`、`__mpp_vpn_inner__`、`__mpp_vpn_inner_server__` — 后果：新增失败 / 系统内置 VPN 被污染
- **VRFNAME 全网统一规划**（critical，隐含）：一条通信路径上所有节点绑定的 VPN 实例必须一致才能正常通信 — 后果：VRFNAME 不一致致通信中断
- **配套必加地址族**（warning，隐含）：ADD L3VPNINST 后须紧跟 [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF 使能对应地址族，否则 VPN 相关配置无法继续 — 后果：地址族未使能→VPN 配置失败
