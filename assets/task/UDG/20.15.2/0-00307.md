# 应用IPsec安全策略到IPsec隧道接口（ADD IPSECINTFCFGIPSEC）

> 命令静态知识见 [ADD IPSECINTFCFGIPSEC](command/UDG/20.15.2/ADD-IPSECINTFCFGIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是 VNRS+IPSEC 双配的**最后生效点**：VNRS 侧 [0-00216](task/UDG/20.15.2/0-00216.md) ADD IPSECINTFCFG 把 Tunnel 接口标记为 IPsec 隧道协议（TNLTYPE=IPSEC/IPSEC6），IPsec 侧 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 产出 POLICYNAME，再由本命令把策略挂到该 Tunnel 接口上。**双配铁律**：INTERFACENAME + POLICYNAME + TNLTYPE 三者必须一一对应（VNRS 侧 TNLTYPE 与 IPsec 侧 INTERFACENAME 同 Tunnel，且 POLICYNAME 是该 Tunnel 上的 IPsec 策略）。

## 配置方法（配置方法字典）

ADD IPSECINTFCFGIPSEC 是"IPsec 策略→Tunnel 接口绑定"命令。3 个必选参数（INTERFACENAME + TNLTYPE + POLICYNAME）+ 1 个可选参数（SRCIFNAME）。配置空间有 4 个维度：**Tunnel 接口**（INTERFACENAME 引用 VNRS 侧 Tunnel IFNAME）、**隧道协议**（TNLTYPE 决定 IPv4/IPv6 链路）、**IPsec 策略**（POLICYNAME 引用 IPSECPOLICY 产出）、**源接口**（SRCIFNAME 可选，仅指定本端源场景）。8 个 IPsec 子场景命中（IPFD-015004 10002 普通 IPv4 / 10003 普通 IPv6 / 10004 IPv4 主备 / 10005 IPv6 主备 / 10006 GRE over IPsec / 10007 OSPF over IPsec / 10008 多 Sequence / 10009 指定本端接口）共享同一配置空间，仅 INTERFACENAME/POLICYNAME 取值不同。

### 配置维度 1：Tunnel 接口（参数 INTERFACENAME，必选）

| 取值形式 | 来源 | 双配对应 |
|---|---|---|
| **字符串 1~63** | 引用 [0-00216](task/UDG/20.15.2/0-00216.md) ADD IPSECINTFCFG:INTERFACENAME（VNRS 侧 Tunnel 接口名） | 须与 VNRS 侧 Tunnel 接口一一对应 |

> **接口双配铁律**：本参数必须与 [0-00216](task/UDG/20.15.2/0-00216.md) VNRS 侧产出的 INTERFACENAME **完全相同**（同一 Tunnel），否则 IPsec 协商无法找到对应接口。

### 配置维度 2：隧道协议（参数 TNLTYPE，必选，二选一固定）

| 取值 | 作用 | 配套 POLICYNAME | 典型场景 |
|---|---|---|---|
| **IPSEC** | IPv4 IP 安全隧道 | 引用 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 的 IPv4 POLICYNAME | IPFD-015004 10002/10004/10006/10007/10008/10009 默认范式 |
| **IPSEC6** | IPv6 IP 安全隧道 | 引用 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 的 IPv6 POLICYNAME | IPFD-015004 10003/10005 IPv6 范式 |

> **协议对齐铁律**：TNLTYPE 决定链路层是 IPv4 还是 IPv6；与之挂接的 POLICYNAME 必须使用**同链路族**的 IPsec 策略（IPv4 策略配 IPSEC，IPv6 策略配 IPSEC6）；VNRS 侧 [0-00216](task/UDG/20.15.2/0-00216.md) 的 TNLTYPE 也必须与本参数**完全相同**——三方对齐（VNRS 侧 TNLTYPE + IPsec 侧 TNLTYPE + POLICYNAME 链路族）。

### 配置维度 3：IPsec 策略（参数 POLICYNAME，必选）

| 取值形式 | 来源 | 适用场景 |
|---|---|---|
| **字符串 1~15（不区分大小写）** | 引用 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 的 POLICYNAME | IPFD-015004 10002 范例 `map1`；10009 范例 `policy1` |

> **空策略黑名单**：POLICYNAME 若引用**空安全策略**（未配置真实有效策略），IPsec 隧道接口**不会生成**——这是硬约束：只有真实有效的安全策略才生成 IPsec 隧道接口。POLICYNAME 链路族必须与 TNLTYPE 一致（IPv4 配 IPSEC，IPv6 配 IPSEC6）。

### 配置维度 4：源接口（参数 SRCIFNAME，可选）

| 取值形式 | 范围约束 | 适用场景 |
|---|---|---|
| **字符串 1~63** | 只支持**环回口**（LoopBack） | IPFD-015004 10009（指定本端接口建立 IPsec 隧道）场景显式配，如 `LoopBack1` / `loopback1` |

> **环回口铁律**：SRCIFNAME 仅支持环回口；若配非环回口，命令拒收。多数场景（10002/10003/10004/10005/10006/10007/10008）不显式传 SRCIFNAME——本端源 IP 由 IPsec 协商自动从接口地址借出。

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，仅 INTERFACENAME+TNLTYPE+POLICYNAME 三必选）**：
```
ADD IPSECINTFCFGIPSEC:INTERFACENAME="Tunnel10",TNLTYPE=IPSEC,POLICYNAME="map1";
```

**典型脚本（IPFD-015004 普通 IPv6 IPsec 隧道，TNLTYPE 改 IPSEC6）**：
```
ADD IPSECINTFCFGIPSEC:INTERFACENAME="Tunnel27",TNLTYPE=IPSEC6,POLICYNAME="map1";
```

**典型脚本（IPFD-015004 IPv4 IPsec 主备隧道，每条 Tunnel 各挂一条策略）**：
```
ADD IPSECINTFCFGIPSEC:INTERFACENAME="Tunnel17",TNLTYPE=IPSEC,POLICYNAME="map1";
// 主 Tunnel 10 同样挂一条 map1（详见数据规划表）
```

**典型脚本（IPFD-015004 指定本端接口建立 IPsec 隧道，10009 范例，加 SRCIFNAME）**：
```
ADD IPSECINTFCFGIPSEC:INTERFACENAME="Tunnel2",TNLTYPE=IPSEC,POLICYNAME="policy1",SRCIFNAME="LoopBack1";
```

> 命令 notes：**最大记录数 512**；**执行后立即生效**；**权限**：G_1 管理员 / G_2 操作员命令组；**空策略陷阱**：POLICYNAME 引用空安全策略不会生成 IPsec 隧道接口；**配置原则**：POLICYNAME 须与 [ADD IPSECPOLICY](command/UDG/20.15.2/ADD-IPSECPOLICY.md) / [ADD IPSECPOLICY6](command/UDG/20.15.2/ADD-IPSECPOLICY6.md) 产出的策略名称**保持一致**。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：IPsec 协商链末尾——策略挂接口"——前置 [0-00216](task/UDG/20.15.2/0-00216.md) ADD IPSECINTFCFG（VNRS 侧 Tunnel 标记 IPSEC）+ [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY（IPsec 策略产出 POLICYNAME）；下游 **IPsec 隧道协商生效**（IPsec SA 建立 / IKE 协商 / 业务流加密承载）。

## 决策点：IPsec 子场景选择（DP 0-00307）

问：本 IPsec 隧道接口属于哪种 IPsec 子场景？（决定 TNLTYPE 选 IPSEC/IPSEC6 + POLICYNAME 取 IPv4/IPv6 策略 + 是否配 SRCIFNAME）

| 选项 | TNLTYPE | POLICYNAME | SRCIFNAME | 典型特征 |
|---|---|---|---|---|
| 普通 IPv4 IPsec 隧道（10002） | IPSEC | IPv4 策略（如 `map1`） | 不配 | 单 Tunnel 单策略 |
| 普通 IPv6 IPsec 隧道（10003） | IPSEC6 | IPv6 策略（如 `map1`） | 不配 | 单 Tunnel 单策略，链路 IPv6 |
| IPv4 IPsec 主备隧道（10004） | IPSEC | IPv4 主/备策略 | 不配 | 双 Tunnel 双策略，主备协商 |
| IPv6 IPsec 隧道主备方式（10005） | IPSEC6 | IPv6 主/备策略 | 不配 | 双 Tunnel 双策略，主备协商 |
| GRE over IPsec（10006） | IPSEC | IPv4 策略 | 不配 | 内层 GRE 隧道，外层 IPsec 加密 |
| OSPF over IPsec（10007） | IPSEC | IPv4 策略 | 不配 | 动态路由协议 over IPsec 加密 |
| 多 Sequence IPsec 策略（10008） | IPSEC | IPv4 策略（多条 SEQUENCENUMBER） | 不配 | 单 Tunnel 多 Sequence 策略 |
| 指定本端接口建立 IPsec 隧道（10009） | IPSEC | IPv4 策略 | **必填 LoopBack1** | 本端源固定为环回口 |

> 8 个 IPsec 子场景（IPFD-015004 10002~10009）共享同一命令空间，差异在 TNLTYPE 二选一 + POLICYNAME 取对应链路族策略 + 是否传 SRCIFNAME；**VNRS+IPSEC 双配铁律贯穿所有场景**——INTERFACENAME 必须与 [0-00216](task/UDG/20.15.2/0-00216.md) 一致。

## 约束

- **最大记录数 512**（warning，rule-0-00307）：系统最大记录数 512 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00307）：命令执行后立即生效 — 后果：误操作即刻生效
- **空策略不生成隧道接口**（critical，rule-0-00307）：POLICYNAME 引用空安全策略时不会生成 IPsec 隧道接口 — 后果：空策略导致接口不生效，业务流无法加密承载
- **VNRS+IPSEC 双配铁律：INTERFACENAME 一致**（critical，rule-0-00307）：INTERFACENAME 必须与 [0-00216](task/UDG/20.15.2/0-00216.md) ADD IPSECINTFCFG:INTERFACENAME 完全相同 — 后果：接口不对应 IPsec 协商找不到 Tunnel
- **VNRS+IPSEC 双配铁律：TNLTYPE 一致**（critical，rule-0-00307）：本命令 TNLTYPE 必须与 [0-00216](task/UDG/20.15.2/0-00216.md) VNRS 侧 TNLTYPE 完全相同 — 后果：协议不对应 IPsec 隧道不通
- **POLICYNAME 链路族对齐 TNLTYPE**（critical，rule-0-00307）：IPv4 策略配 IPSEC；IPv6 策略配 IPSEC6 — 后果：链路族错配策略无法应用
- **POLICYNAME 须先建**（critical，rule-0-00307）：POLICYNAME 必须先经 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 或 [ADD IPSECPOLICY6](command/UDG/20.15.2/ADD-IPSECPOLICY6.md) 创建 — 后果：引用未建策略命令拒收
- **POLICYNAME 长度**（info，rule-0-00307）：字符串 1~15（不区分大小写） — 后果：超长命令拒收
- **INTERFACENAME 长度**（info，rule-0-00307）：字符串 1~63 — 后果：超长命令拒收
- **SRCIFNAME 仅支持环回口**（critical，rule-0-00307）：SRCIFNAME 只能配 LoopBack 接口 — 后果：配非环回口命令拒收
- **SRCIFNAME 长度**（info，rule-0-00307）：字符串 1~63 — 后果：超长命令拒收
- **权限要求**（info，rule-0-00307）：需 G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
