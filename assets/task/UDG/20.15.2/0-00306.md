# IPsec微服务接口绑VPN（ADD IPBINDVPNIPSEC）

> 命令静态知识见 [ADD IPBINDVPNIPSEC](command/UDG/20.15.2/ADD-IPBINDVPNIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是 IPsec 微服务侧"VPN + 隧道接口"双配的"接口绑 VPN"步骤——前置 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC（建 VPN）+ [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC（建接口），后置 [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC（配 IPv4 地址）。**关键铁律**：执行本命令将**清除该接口下所有的 IP 配置**——因此**必须在 [0-00304](task/UDG/20.15.2/0-00304.md) 之前下发**；**需要在 [0-00307](task/UDG/20.15.2/0-00307.md) ADD IPSECINTFCFGIPSEC 之前执行**，绑定的 VPN 才生效。

## 配置方法（配置方法字典）

ADD IPBINDVPNIPSEC 是"IPsec 微服务侧接口绑定 VPN"命令。7 个参数中 2 必选（IFNAME + VRFNAME）+ 5 条件可选（IFIPADDR/SUBNETMASK 联动 IPTYPECHECK=Ipv4，IPV6ADDR/SUBNETMASKIPV6 联动 IPTYPECHECK=Ipv6，IPTYPECHECK 默认 Ipv4）。配置空间有 4 个维度：**接口**（IFNAME 引用已建 Tunnel/LoopBack）、**VPN 实例**（VRFNAME 引用已建 VPN）、**地址族**（IPTYPECHECK Ipv4/Ipv6 二选一）、**IPv4/IPv6 地址+掩码**（条件可选）。1 个特性命中（IPFD-015004 普通 IPv4/IPv6 IPsec 隧道等 8 子场景）覆盖典型范式——所有命中脚本均显式传 `IPTYPECHECK=Ipv4`（IPv4 场景默认）。

### 配置维度 1：接口（参数 IFNAME，必选，引用上游）

| 取值形式 | 来源 | 双配对应 |
|---|---|---|
| **字符串 1~63（接口类型+接口编号）** | 引用 [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC:IFNAME（Tunnel / LoopBack） | 须与 VNRS 侧 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 的 IFNAME 一致 |

> **接口双配铁律**：本参数必须与 VNRS 侧 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE:IFNAME **完全相同**（同一 Tunnel/LoopBack 编号）。
>
> **IPsec Tunnel/LoopBack 范式**：IPFD-015004 命中脚本均为 `Tunnel10` / `Tunnel2` / `Tunnel17` / `Tunnel27` / `LoopBack1`——典型的 Tunnel 接口（隧道协商）+ LoopBack 接口（本端源）。

### 配置维度 2：VPN 实例（参数 VRFNAME，必选，引用上游）

| 取值形式 | 范围 | 来源 |
|---|---|---|
| **字符串 1~31** | 任意 VPN 实例名（不含保留名约束——`_public_` / `__mpp_vpn_inner__` 由上游 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 校验） | 引用 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC:VRFNAME |

> **VRFNAME 双配铁律**：本参数必须与 [0-00210](task/UDG/20.15.2/0-00210.md) IPsec 侧 ADD L3VPNINSTIPSEC:VRFNAME 一致（同一 VPN 实例）。
>
> **必须先建 VPN**：VRFNAME 必须先经 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 创建。
>
> **典型 VRFNAME**：IPFD-015004 命中脚本均为 `vrf1`。

### 配置维度 3：地址族（参数 IPTYPECHECK，可选，默认 Ipv4）

| 取值 | 联动参数 | 适用场景 |
|---|---|---|
| **Ipv4**（默认） | `IFIPADDR` + `SUBNETMASK`（条件可选） | IPFD-015004 范式默认（所有命中脚本显式传 `IPTYPECHECK=Ipv4`） |
| **Ipv6** | `IPV6ADDR` + `SUBNETMASKIPV6`（条件可选） | IPv6 IPsec 场景（如 10003 普通 IPv6 IPsec 隧道子场景） |

> **默认值**：未传 IPTYPECHECK 时取 Ipv4。

### 配置维度 4：IPv4/IPv6 地址+掩码（参数 IFIPADDR/SUBNETMASK 或 IPV6ADDR/SUBNETMASKIPV6，条件可选）

| 子参数组 | 条件 | 范围 | 作用 |
|---|---|---|---|
| **IFIPADDR** | IPTYPECHECK=Ipv4 时条件可选 | IPv4 地址（0.0.0.0~255.255.255.255） | 指定 IPv4 地址（IPFD-015004 命中脚本均不传，由后续 [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC 配） |
| **SUBNETMASK** | IPTYPECHECK=Ipv4 时条件可选 | IPv4 地址掩码（0.0.0.0~255.255.255.255） | 指定 IPv4 掩码（IPFD-015004 命中脚本均不传） |
| **IPV6ADDR** | IPTYPECHECK=Ipv6 时条件可选 | IPv6 地址 | 指定 IPv6 地址（IPFD-015004 IPv6 子场景不传） |
| **SUBNETMASKIPV6** | IPTYPECHECK=Ipv6 时条件可选 | IPv6 地址掩码 | 指定 IPv6 掩码（IPFD-015004 IPv6 子场景不传） |

> **范式观察**：IPFD-015004 命中脚本**均不传 IFIPADDR/SUBNETMASK/IPV6ADDR/SUBNETMASKIPV6**——即典型 IPsec 编排中本命令**仅传 3 参**（IFNAME + VRFNAME + IPTYPECHECK），接口 IP 由后续 [0-00304](task/UDG/20.15.2/0-00304.md) 单独配。

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，IPv4 Tunnel 接口绑 vrf1）**：
```
ADD IPBINDVPNIPSEC:IFNAME="Tunnel10",VRFNAME="vrf1",IPTYPECHECK=Ipv4;
```

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，LoopBack 接口绑 vrf1）**：
```
ADD IPBINDVPNIPSEC:IFNAME="LoopBack1",VRFNAME="vrf1",IPTYPECHECK=Ipv4;
```

**典型脚本（IPFD-015004 IPv4 主备隧道，双 Tunnel 各绑 vrf1）**：
```
ADD IPBINDVPNIPSEC:IFNAME="Tunnel10",VRFNAME="vrf1";
ADD IPBINDVPNIPSEC:IFNAME="Tunnel17",VRFNAME="vrf1";
```

**典型脚本（IPFD-015004 GRE over IPsec，Tunnel2 + LoopBack1 各绑 vrf1）**：
```
ADD IPBINDVPNIPSEC:IFNAME="Tunnel2",VRFNAME="vrf1";
ADD IPBINDVPNIPSEC:IFNAME="LoopBack1",VRFNAME="vrf1";
```

> 命令 notes：**执行后立即生效**；**关键风险**：执行该命令**将清除该接口下所有的 IP 配置**——必须在 [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC 之前执行；**需要在 [0-00307](task/UDG/20.15.2/0-00307.md) ADD IPSECINTFCFGIPSEC 之前执行**，绑定的 VPN 才生效；**该配置影响该隧道下的地址信息，有业务影响**；**最多可输入 65535 条记录**；**权限**：G_1 管理员 / G_2 操作员命令组。

**步骤位置**：在所属 feature 编排中位于"步骤 4：IPsec 微服务五件套——接口绑 VPN"——前置 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC（建 VPN）+ [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC（建接口）；后置 [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC（配 IPv4 地址）；下游 IPsec 协商链：[0-00307](task/UDG/20.15.2/0-00307.md) IPsec Policy→Tunnel 接口绑定。

## 决策点：IPsec 子场景选择（DP 0-00306）

问：本接口绑定属于哪种 IPsec 子场景？（决定 IFNAME 类型 Tunnel/LoopBack + IPTYPECHECK Ipv4/Ipv6）

| 选项 | IFNAME | VRFNAME | IPTYPECHECK | 典型特征 |
|---|---|---|---|---|
| 普通 IPv4 IPsec 隧道（10002） | `Tunnel10` / `Tunnel2` | `vrf1`（本端规划） | Ipv4 | 单 Tunnel 单绑 IPv4 |
| 普通 IPv6 IPsec 隧道（10003） | `Tunnel27` | `vrf1` | Ipv6 | 单 Tunnel 单绑 IPv6 |
| LoopBack 本端源（10009） | `LoopBack1` | `vrf1` | Ipv4 | LoopBack 绑 IPv4 |
| IPv4/IPv6 主备隧道（10004/10005） | 双 Tunnel 各一条 | `vrf1` | Ipv4 / Ipv6 | 双 Tunnel 双绑 |
| GRE over IPsec（10006） | `Tunnel2` + `LoopBack1` | `vrf1` | Ipv4 | Tunnel 绑 GRE + LoopBack 绑本端源 |
| OSPF over IPsec（10007） | `Tunnel2` | `vrf1` | Ipv4 | 动态路由协议 over IPsec |
| 指定本端接口（10009） | `LoopBack1` | `vrf1` | Ipv4 | 本端源固定 |

## 约束

- **最多 65535 条记录**（warning，rule-0-01306）：系统最多可输入 65535 条接口绑 VPN 记录 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-01306）：命令执行后立即生效 — 后果：误操作即刻生效
- **执行清除接口所有 IP**（critical，rule-0-01306）：执行该命令将清除该接口下所有的 IP 配置 — 后果：误操作导致已有 IP 配置丢失
- **该配置影响隧道地址信息**（critical，rule-0-01306）：该配置影响该隧道下的地址信息，有业务影响 — 后果：执行时业务流中断
- **必须在 IPSECINTFCFGIPSEC 之前执行**（critical，rule-0-01306）：需要在执行 [0-00307](task/UDG/20.15.2/0-00307.md) ADD IPSECINTFCFGIPSEC 之前执行本命令，绑定的 VPN 才生效 — 后果：顺序错配 VPN 不生效
- **必须在 IFIPV4ADDRESSIPSEC 之前执行**（critical，rule-0-01306）：必须在 [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC 之前执行本命令（否则 IP 配置被清空） — 后果：顺序错配接口 IP 丢失
- **VRFNAME 必须先建**（critical，rule-0-01306）：VRFNAME 必须先经 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 创建 — 后果：引用未建 VPN 命令拒收
- **IFNAME 必须先建**（critical，rule-0-01306）：IFNAME 必须先经 [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC 创建 — 后果：引用未建接口命令拒收
- **IFNAME 长度**（info，rule-0-01306）：字符串 1~63 — 后果：超长命令拒收
- **VRFNAME 长度**（info，rule-0-01306）：字符串 1~31 — 后果：超长命令拒收
- **IPTYPECHECK 二选一**（critical，rule-0-01306）：IPTYPECHECK 仅 Ipv4 / Ipv6 二选一 — 后果：传其它取值命令拒收
- **IPTYPECHECK 默认值**（info，rule-0-01306）：未传 IPTYPECHECK 时默认 Ipv4 — 后果：IPv6 场景未显式传会落 Ipv4
- **IPTYPECHECK 与地址联动**（critical，rule-0-01306）：IPTYPECHECK=Ipv4 时 IFIPADDR+SUBNETMASK 条件可选；=Ipv6 时 IPV6ADDR+SUBNETMASKIPV6 条件可选 — 后果：地址族与地址参数不匹配命令拒收
- **双配铁律：IFNAME 与 VNRS 侧一致**（critical，rule-0-01306）：IFNAME 必须与 [0-00056](task/UDG/20.15.2/0-00056.md) VNRS 侧 ADD INTERFACE 的 IFNAME 完全相同 — 后果：IFNAME 不一致 IPsec 隧道不通
- **双配铁律：VRFNAME 与 VNRS 侧一致**（critical，rule-0-01306）：VRFNAME 必须与 [0-00071](task/UDG/20.15.2/0-00071.md) VNRS 侧 ADD L3VPNINST 的 VRFNAME 一致 — 后果：VRFNAME 不一致 IPsec 隧道不通
- **权限要求**（info，rule-0-01306）：需 G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
