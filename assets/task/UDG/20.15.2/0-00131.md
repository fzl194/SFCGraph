# 设置L2TP缺省配置（SET GLOBALL2TP）

> 命令静态知识见 [SET GLOBALL2TP](command/UDG/20.15.2/SET-GLOBALL2TP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET GLOBALL2TP 设置 L2TP 隧道**全局唯一**的缺省属性（本命令为 SET，全网仅一组缺省）。它为 [AAA 下发](task/UDG/20.15.2/0-00132.md) 与 [本地配置](task/UDG/20.15.2/0-00133.md) 两种 L2TP 激活方式**共享**——AAA 下发场景 LAC 名称缺省取本命令的 LOCALNAME、隧道保活 HELLO 取本命令的间隔；本地配置场景缺省 HELLO/Magic/窗口集中在 ADD L2TPGROUP，SET GLOBALL2TP 仅设 LOCALNAME+HELLO+RETRYTIMES。

**两种激活方式配法对比**

| 参数 | AAA 下发方式取值 | 本地配置方式取值 | 来源 |
|---|---|---|---|
| LOCALNAME | `huawei` | `huawei` | local_planned |
| HELLOINTERVALSW | `ENABLE` | `ENABLE` | local_planned |
| HELLOINTERVAL | `60` | `60` | local_planned |
| RETRYTIMES | `3` | `3` | local_planned |
| PPPMAGICNUMBER | `DISABLE` | —（走 ADD L2TPGROUP） | peer_planned（与对端协商） |
| MAXSENDWINSIZE | `64` | —（走 ADD L2TPGROUP） | peer_planned |
| INITTUNNELNUM | `1` | —（走 ADD L2TPGROUP） | local_planned |
| MAXSESSIONNUM | `32767` | —（走 ADD L2TPGROUP） | local_planned |

> AAA 方式不建 L2TP 组，靠缺省会话上限控制隧道；本地方式把这些参数下放到 ADD L2TPGROUP（逐组覆盖），SET GLOBALL2TP 只留本端名 + HELLO + 重发。HELLOINTERVAL 仅在 HELLOINTERVALSW=ENABLE 时必填。

**AAA 方式典型脚本（全参）**：

```
SET GLOBALL2TP:LOCALNAME="huawei",HELLOINTERVALSW=ENABLE,HELLOINTERVAL=60,RETRYTIMES=3,PPPMAGICNUMBER=DISABLE,MAXSENDWINSIZE=64,INITTUNNELNUM=1,MAXSESSIONNUM=32767;
```

**本地方式典型脚本（仅缺省本端名 + HELLO + 重发）**：

```
SET GLOBALL2TP:LOCALNAME="huawei",HELLOINTERVALSW=ENABLE,HELLOINTERVAL=60,RETRYTIMES=3;
```

## 约束

- **rule-0-00365**（warning）命令最大记录数 **1**（全局唯一实例）；执行后**立即生效**（AAA 下发或本地建隧道时立即读新值）；存在系统初始记录（LOCALNAME=空 / HELLOINTERVALSW=DISABLE / HELLOINTERVAL=无 / RETRYTIMES=无 / PPPMAGICNUMBER=DISABLE / MAXSENDWINSIZE=64 / INITTUNNELNUM=1 / MAXSESSIONNUM=32767 …）；适用 NF：PGW-U、UPF — 后果：超规格（多实例）新建失败；非立即生效会导致隧道参数未刷新
- HELLOINTERVAL 仅在 HELLOINTERVALSW=ENABLE 时必填（取值 60~10000 秒）
