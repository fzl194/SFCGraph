# 配置接口绑定VPN（ADD IPBINDVPN）

> 命令静态知识见 [ADD IPBINDVPN](command/UDG/20.15.2/ADD-IPBINDVPN.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

把已建好的逻辑接口（来自 ADD INTERFACE）绑定到 VPN 实例（来自 ADD L3VPNINST），两个必选参数 IFNAME/VRFNAME 都是引用来源——IFNAME 引用 0-00056 的 IFNAME，VRFNAME 引用 0-00041 的 VPNINSTANCE。核心规则：**LoopBack 与 Tunnel 绑不同 VPN 实例**，完成物理与业务划分；IPv6 组网 / IPsec 场景的以太网子接口与 Tunnel 可同绑一个 VPN。

**绑定模式归纳（3 特性数据规划表）**

| 场景 | IFNAME 样例 | VRFNAME 样例 | 绑定要点 |
|---|---|---|---|
| GWFD-010107（静态路由冗余） | LoopBack1→vpn2；Tunnel1→vpn1 | vpn1/vpn2 | LoopBack 与 Tunnel **绑不同 VPN**（物理/业务划分） |
| GWFD-020402（IPv6 组网） | Ethernet70/0/4.1 等 | VPN_CN | 以太网子接口绑同一 VPN |
| IPFD-015004（IPsec） | Ethernet66/0/5、LoopBack1、Tunnel10/17/27 | vrf1 | 接口与 Tunnel 同绑 vrf1 |

**典型脚本**
```
ADD IPBINDVPN:IFNAME="LoopBack1",VRFNAME="vpn2";
ADD IPBINDVPN:IFNAME="Tunnel1",VRFNAME="vpn1";
ADD IPBINDVPN:IFNAME="Ethernet70/0/4.1",VRFNAME="VPN_CN";
```

**引用链**
- IFNAME ← [0-00056](task/UDG/20.15.2/0-00056.md)（ADD INTERFACE）的 IFNAME
- VRFNAME ← [0-00041](task/UDG/20.15.2/0-00041.md)（ADD L3VPNINST）的 VPNINSTANCE

**未绑参数**：IFIPADDR / SUBNETMASK（命令真相 optional，UDG 场景 IP 由 ADD IFIPV4ADDRESS 单独配，本命令不传）。

**高危行为**：当 IFIPADDR 不指定时（UDG 场景默认不传），执行本命令会**清空该接口下原 IP 地址**——重绑 VPN 前需先备份/确认接口地址。

## 约束

- **rule-0-00171**（warning）最大记录数 65535；立即生效；版本升级过程中禁止执行；IFIPADDR 不指定时清空接口原 IP、指定时替换（每主用接口最多 1 主 + 255 从 = 256 个 IP） — 违反后果：超规格新增失败 / 重绑 VPN 不传 IFIPADDR 会清空接口 IP（高危）
