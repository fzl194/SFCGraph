# 配置接口绑定VPN（ADD IPBINDVPN）

> 命令静态知识见 [ADD IPBINDVPN](command/UDG/20.15.2/ADD-IPBINDVPN.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**配置 VPN 实例后必须将接口与 VPN 实例绑定**，接口才会变为私网接口、才能配置私网地址/运行私网路由协议。

## 配置方法（配置方法字典）

ADD IPBINDVPN 是"接口→VPN 实例绑定"命令：两个必选参数都是引用来源——IFNAME 引用 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 的 IFNAME，VRFNAME 引用 [0-00041](task/UDG/20.15.2/0-00041.md) ADD L3VPNINST 的 VPNINSTANCE。核心配置空间有 3 个维度：**接口类型**（决定绑哪类接口）、**VPN 划分**（同/异 VPN，决定物理/业务划分模式）、**接口 IP 行为**（IFIPADDR 指定与否，决定是否清空/替换原 IP）。3 个特性命中（010107、020402、IPFD-015004）覆盖三种 VPN 划分范式。

### 配置维度 1：接口类型（参数 IFNAME，必选）

IFNAME 引用已配的逻辑接口，不同接口类型决定其在 VPN 中的角色：

| 接口类型 | IFNAME 引用源 | 典型角色 |
|---|---|---|
| **LoopBack** | [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 产出的 LoopBack<n> | 本端源接口（GRE/IPsec），绑 VPN 作为本端识别 |
| **Tunnel**（GRE/IPsec） | [0-00056](task/UDG/20.15.2/0-00056.md) + [0-00059](task/UDG/20.15.2/0-00059.md) 产出的 Tunnel<n> | 隧道接口，绑 VPN 隔离私网/公网业务 |
| **以太网子接口** | [0-00056](task/UDG/20.15.2/0-00056.md) + [0-00116](task/UDG/20.15.2/0-00116.md) ADD ETHSUBIF 产出的 Ethernet<slot>/<card>/<port>.<vlan> | N6/Gi/SGi 等物理接入子接口，绑 VPN 隔离私网/公网 |

> IFNAME 字符串长度 1~63；可用 `LST INTERFACE` 查询可用接口。

### 配置维度 2：VPN 划分（参数 VRFNAME，必选 + 同/异 VPN 选择）

VRFNAME 引用已配 VPN 实例，**多个接口绑同一 VPN 还是不同 VPN** 决定物理/业务划分：

| 划分模式 | VRFNAME 引用 | 适用场景 |
|---|---|---|
| **同 VPN 绑定** | 所有接口绑同一 VRF（如 `vrf1` / `VPN_CN`） | IPsec/IPv6 组网：接口与 Tunnel 绑同一 VPN 走同转发（IPFD-015004、020402 范式） |
| **异 VPN 绑定** | 不同接口绑不同 VRF（如 LoopBack→vpn2、Tunnel→vpn1） | 静态路由冗余：LoopBack 与 Tunnel 绑不同 VPN 完成物理/业务划分（010107 范式） |

> VRFNAME 字符串长度 1~31；可用 `LST L3VPNINST` 查询可用 VPN 实例。

### 配置维度 3：接口 IP 行为（参数 IFIPADDR/SUBNETMASK，可选）

IFIPADDR/SUBNETMASK 决定本命令对接口原 IP 的处理——**高危行为**，需谨慎：

| IFIPADDR 取值 | 行为 | 后果 |
|---|---|---|
| **不指定**（UDG 默认场景） | **清空该接口下原来的 IP 地址** | 高危：重绑 VPN 时丢失原 IP 配置 |
| **指定 + SUBNETMASK** | 用指定地址**替换**原 IP 地址 | 重绑 VPN 时保留/覆盖 IP |

> **高危铁律**：当 IFIPADDR 不指定时，执行本命令会**清空接口原 IP**——重绑 VPN 前需先备份/确认接口地址（UDG 默认不传 IP，IP 由 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS / [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS 单独配）。

**典型脚本（010107 静态路由冗余，LoopBack 与 Tunnel 绑不同 VPN）**：
```
ADD IPBINDVPN: IFNAME="LoopBack1", VRFNAME="vpn2";
ADD IPBINDVPN: IFNAME="Tunnel1", VRFNAME="vpn1";
```

**典型脚本（020402 IPv6 组网，以太网子接口绑同一 VPN）**：
```
ADD IPBINDVPN:IFNAME="Ethernet70/0/4.1",VRFNAME="VPN_CN";
ADD IPBINDVPN:IFNAME="Ethernet70/0/5.1",VRFNAME="VPN_CN";
```

**典型脚本（IPFD-015004 IPsec，多接口绑同一 vrf1）**：
```
ADD IPBINDVPN:IFNAME="Ethernet66/0/5", VRFNAME="vrf1";
ADD IPBINDVPN:IFNAME="LoopBack1", VRFNAME="vrf1";
ADD IPBINDVPN:IFNAME="Tunnel10", VRFNAME="vrf1";
```

> 命令 notes：**最大记录数 65535**；**执行后立即生效**；**版本升级过程中禁止执行**；每个主用接口最大 1 主 + 255 从 = **256 个 IP**；本命令参与自动化配置——下发前需 `DSP OPSASSISTSTATE（★R1.5 atom 待补）` 查询 autoscaling_autoconfig.py 是否在执行；配置私网 IPv6 时需先配 VPN 再用 [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS。

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：接口绑 VPN"——前置 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE（建接口）、[0-00041](task/UDG/20.15.2/0-00041.md) ADD L3VPNINST（建 VPN 实例），下游 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS / [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS（配 IP）。

## 决策点：VPN 划分模式（DP 0-00057）

问：多个接口在 ADD IPBINDVPN 阶段绑同一 VPN 还是不同 VPN？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 同 VPN 绑定（IPsec/IPv6） | 所有接口绑同一 VRFNAME | IPsec（IPFD-015004：vrf1）、IPv6 组网（020402：VPN_CN）——多接口走同转发 |
| 异 VPN 绑定（路由冗余） | 不同接口绑不同 VRFNAME | 静态路由冗余（010107：LoopBack→vpn2、Tunnel→vpn1）——物理/业务划分 |

> 3 个特性命中（010107/020402/IPFD-015004）覆盖三种范式；本决策只影响 VPN 划分模式，不影响 IP 配置。

## 约束

- **最大记录数 65535**（warning，rule-0-00457）：接口-VPN 绑定最大 65535 条 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00457）：命令执行后立即生效 — 后果：误操作即刻生效
- **版本升级过程禁止执行**（critical，rule-0-00457）：版本升级过程中禁止执行本命令 — 后果：升级期间执行中断升级流程
- **参与自动化配置**（critical，rule-0-00457）：IFIPADDR/SUBNETMASK/VRFNAME 三个参数参与自动化配置——下发前需 `DSP OPSASSISTSTATE（★R1.5 atom 待补）` 查询 autoscaling_autoconfig.py — 后果：自动化配置过程中执行致最终生效配置不可预期
- **IFIPADDR 不指定清空原 IP（高危）**（critical，rule-0-00457）：当 IFIPADDR 不指定时会清空该接口下原来的 IP 地址（指定时替换） — 后果：重绑 VPN 不传 IFIPADDR 致接口 IP 丢失
- **每主用接口最多 256 个 IP**（warning，rule-0-00457）：1 主 + 255 从 = 256 个 IP — 后果：超额命令拒收
- **接口需支持配 IP**（warning，隐含）：接口必须支持配置 IP 地址 — 后果：Loopback/Null0 等不可配 IP 的接口命令拒收
