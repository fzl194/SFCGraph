# 地址分配方式（GWFD-010104）

> 特性静态知识见 [GWFD-010104](feature/UDG/20.15.2/GWFD-010104.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-010104-地址分配方式.md)。
>
> **本页定位**：配置生成实例化视角——本特性 = "外部地址分配方式" 配置 + 用户面地址分配总览入口；详细子方式（基于 APN/DNN / SMF / SMF+APN / RADIUS / 位置）的配置链路详见 [GWFD-010105](task/UDG/20.15.2/2-00034.md) + [GWFD-020421](feature/UDG/20.15.2/GWFD-020421.md)。

## 配置概览

本特性按"两种分配方式 × 通用骨架"组织配置：

**对象链**（外部地址分配方式）：
```
SECTION（地址段，IPv4/IPv6）
   → POOL（地址池，POOLTYPE=EXTERNAL，CHECKIPVALID=ENABLE 开启白名单检测）
       → POOLGROUP（地址池组）
           → APN（绑定 VPN）+ SET APNADDRESSATTR（HOSTROUTEIP=ENABLE）
           → POOLGRPMAP（建立 APN → POOLGROUP 映射）
   → 下行路由 OSPF（PROTOCOL=wlr）发布到骨干网
```

**2 步通用骨架（外部地址分配方式）**：

| 步骤 | 干啥 | 主要命令 |
|---|---|---|
| 1. 地址池体系 + APN 映射 | EXTERNAL 地址池 + 白名单检测 + 地址段 + 池组 + 绑定 + APN + HOSTROUTEIP + 映射 | `ADD VPNINST` / `ADD POOLGROUP` / `ADD POOL` / `ADD SECTION` / `ADD POOLBINDGROUP` / `ADD APN` / `SET APNADDRESSATTR` / `ADD POOLGRPMAP` |
| 2. 下行路由发布 | OSPF 引入 WLR 路由发布到骨干网 | `ADD L3VPNINST` / `ADD VPNINSTAF` / `ADD OSPF` / `ADD OSPFAREA` / `ADD OSPFNETWORK` / `ADD OSPFIMPORTROUTE` |

**关键约束**：
- **本特性无需 License**
- C 面网元**必须为华为配套设备**（N4/Sxb PFCP 协议私有扩展）
- VPN 三者一致：地址池 / APN / Gi/SGi/N6 外联口
- POOLTYPE 固定 EXTERNAL（LOCAL 类型见 010105）
- HOSTROUTEIP=ENABLE 外部地址分配场景专用（区别于 010105 的 SUPPORTIPV4/V6）

## 配置流程

> 每步显示"干啥 + 含哪些命令"。本特性主承载外部地址分配；用户面地址分配详见 010105。

1. **APN 接入域基础设施**（VPN/APN 实例 + 主机地址接入使能）：
   - → 复用 [1-00029](task/UDG/20.15.2/1-00029.md) APN 接入域基础设施（**HOSTROUTEIP=ENABLE 外部地址分配变种**，区别于 010105 的 SUPPORTIPV4/V6）

2. **地址池绑定到地址池组**（EXTERNAL 地址池体系）：
   - `ADD POOLGROUP` → [0-00042](task/UDG/20.15.2/0-00042.md)（POOLGRPNAME）
   - `ADD POOL` → [0-00043](task/UDG/20.15.2/0-00043.md)（POOLNAME, **POOLTYPE=EXTERNAL**, HASVPN=ENABLE, VPNINSTANCE, **CHECKIPVALID=ENABLE**）
   - `ADD SECTION` → [0-00044](task/UDG/20.15.2/0-00044.md)（POOLNAME, IPVERSION=IPV4/IPV6, V4STARTIP/V4ENDIP）
   - `ADD POOLBINDGROUP` → [0-00045](task/UDG/20.15.2/0-00045.md)（POOLGROUPNAME, POOLNAME）
   - 复用 → [1-00022](task/UDG/20.15.2/1-00022.md) 地址分配体系骨架（仅步骤 1-4，外部 POOLTYPE 替换 LOCAL）

3. **配置 APN-池组映射关系**：
   - `ADD POOLGRPMAP` → [0-00047](task/UDG/20.15.2/0-00047.md)（MAPPINGNAME, APN=, POOLGROUPNAME=）

4. **下行路由发布**（OSPF 引入 wlr 发布到骨干网）：
   - `ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFNETWORK` + `ADD OSPFIMPORTROUTE PROTOCOL=wlr`
   - 复用 → [1-00024](task/UDG/20.15.2/1-00024.md) 下行路由发布（与 010105 共用）

| 分配方式 | 子方式 | 走法（compound/atom 编排） |

| 用户面地址分配 | 基于位置 | [GWFD-020421](feature/UDG/20.15.2/GWFD-020421.md) 详见（需 License LKV3G5LBAA01） |
| 用户面地址分配 | 基于 SMF | [1-00022](task/UDG/20.15.2/1-00022.md)（POOLTYPE=LOCAL，1 池）+ [1-00023](task/UDG/20.15.2/1-00023.md)（CPNODEID + SMF 全局开关）+ [1-00021](task/UDG/20.15.2/1-00021.md)（单级 SMF） + [1-00024](task/UDG/20.15.2/1-00024.md) |
| 用户面地址分配 | 基于 SMF+APN | [1-00022](task/UDG/20.15.2/1-00022.md)（双池双映射）+ [1-00023](task/UDG/20.15.2/1-00023.md) + [1-00021](task/UDG/20.15.2/1-00021.md)（两级规则）+ [1-00024](task/UDG/20.15.2/1-00024.md) |
| 用户面地址分配 | 基于 RADIUS 下发地址池名 | [1-00022](task/UDG/20.15.2/1-00022.md)（POOLTYPE=LOCAL，1 池）+ [0-00046](task/UDG/20.15.2/0-00046.md)（IGNORE*=DISABLE）+ [0-00318](task/UDG/20.15.2/0-00318.md)（WILDCARDSW=ENABLE）+ [1-00021](task/UDG/20.15.2/1-00021.md)（SET APNIPALLOCRULE 覆盖全局）+ [1-00024](task/UDG/20.15.2/1-00024.md) |

> **关键规则**：
> - 本特性独立承载**外部地址分配**（POOLTYPE=EXTERNAL + CHECKIPVALID + HOSTROUTEIP）
> - **IP 类型差异**：外部地址分配仅生成 wlr_ud 单路由（无 wlr_bh 黑洞路由）；用户面地址分配生成 wlr_bh + wlr_ud 双路由
> - **APNADDRESSATTR 参数语义**：HOSTROUTEIP=ENABLE（外部地址分配专用） vs SUPPORTIPV4/IPV6=ENABLE（用户面地址分配）
> - 所有分配方式共享 → [1-00024](task/UDG/20.15.2/1-00024.md) 下行路由发布

## 约束

- **无需 License**（info）：本特性无需获得 License 许可即可获得该特性的服务（特性概述明确声明）。
- **C 面网元必须为华为配套设备**（critical，rule-0-00497）：因 N4/Sxb 对 PFCP 协议有私有扩展 — 后果：非华为 C 面激活失败
- **VPN 三者一致**（critical，rule-0-00498）：地址池 / APN / Gi/SGi/N6 外联口绑定的 VPN 必须一致 — 后果：VPN 不一致致地址分配或路由异常
- **APN/DNN 全网规划一致**（critical，rule-0-00499）：UDG 本端 APN/DNN 实例应与 C 面配置的 APN/DNN 实例一致 — 后果：PFCP 信令失配
- **POOLTYPE 与 CHECKIPVALID 严格匹配**（critical，rule-0-00509）：CHECKIPVALID=ENABLE **仅对 EXTERNAL 类型有效**；POOLTYPE=LOCAL + CHECKIPVALID=ENABLE 无意义且可能致未定义行为 — 后果：白名单检测失效或异常
- **每 APN 绑池数 ≤16**（warning，rule-0-00500）：每 APN/DNN 支持绑定的地址池最多 16 个 — 后果：超规格新增失败
- **每 POOL 绑段数 ≤64**（warning，rule-0-00501）：每本地地址池支持绑定的地址段最多 64 个 — 后果：超规格新增失败
- **主机路由规格 40 万**（warning，rule-0-00502）：每 UDG 支持的主机路由规格为 40 万 — 后果：超规格下行路由发布异常
- **地址池优先级仅同类型地址池间**（warning，rule-0-00503）：IPv4 vs IPv6 地址池独立优先级 — 后果：跨类型无优先级比较
- **强制回收风险**（critical，rule-0-00505）：地址强制回收前必须先锁定地址池/段 — 后果：强制回收在线用户地址致强制下线
- **真值表未列场景**（critical，rule-0-00507）：PFCP Session Establishment Request 中 BIT 组合未在真值表的场景，PGW-U/UPF 均以激活失败处理 — 后果：地址分配失败
- **ROUTER ID 全网唯一**（critical，rule-0-00508）：OSPF Router ID（SCHEMAROUID）必须全网唯一 — 后果：OSPF 邻接建立失败
- **WLR 路由发布约束**（critical，rule-0-00510）：采用路由策略方式发布路由时，必须选择 **WLR 方式发布所有路由**，不能只选 WLR_SP 或 WLR_UD — 后果：下行路由发布不完整
- **整段配置建议**（info，rule-0-00511）：ADD SECTION 命令中配置的 IP 地址段建议整段配置，可减少对外发布的路由条数 — 后果：路由表项过多
- **外部地址白名单要求**（critical，rule-0-00512）：当 PGW-C/SMF 下发UE地址时，CHECKIPVALID=ENABLE 必须检查 PGW-C/SMF 下发UE地址是否在 EXTERNAL POOL 内 — 后果：不在则会话建立失败
- **地址冲突检测 BYTE44**（warning，rule-0-00513）：当 PGW-C/SMF 下发UE地址但未下发掩码时，根据 BYTE44 软参配置决定去活原用户或拒绝新用户接入 — 后果：BYTE44 配置不当致误去活
