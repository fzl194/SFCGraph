# SMF子方式（基于SMF/SMF+APN）

> 归属 [2-00034](task/UDG/20.15.2/2-00034.md) 用户面地址分配（GWFD-010105）。本 compound 是**基于 SMF 子方式专属**——配置 SMF NodeID + SMF 开关两级控制（全局 + 指定 SMF 覆盖）。被引用于 {2-00034}。

## 配置方法

本 compound 由 3 条命令组成（多命令可复用模块，建 compound 承载）：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 添加 SMF 的 NodeID（IPv4/IPv6/FQDN 三类型） | [ADD CPNODEID](task/UDG/20.15.2/0-00303.md) | `CPNAME` / `IPV4NODEID` / `IPV6NODEID` / `FQDNNODEID`（三选一） |
| 2. 设置基于 SMF 分配地址的全局开关 | [SET IPALLOCBYSMFGLBSW](task/UDG/20.15.2/0-00323.md) | `SWITCH` / `IPV6SWITCH` |
| 3. （可选）设置指定 SMF 的开关（覆盖全局） | [SET IPALLOCBYSMFSW](task/UDG/20.15.2/0-00324.md) | `SMF` / `SWITCH` / `IPV6SWITCH` |

**典型脚本（010105 场景二：基于 SMF 单级优先级 + SMF 全局开关）**：
```
ADD CPNODEID: CPNAME="smfnode1", IPV4NODEID="10.0.0.1", IPV6NODEID="FC00:1111:1001:0001:0100:1100:0000:0001", FQDNNODEID="consumer.huawei.com";
SET IPALLOCBYSMFGLBSW: SWITCH=ENABLE;
```

**典型脚本（010105 场景三：基于 SMF+APN 双映射）**：
```
ADD CPNODEID: CPNAME="smfnode1", IPV4NODEID="10.0.0.1", IPV6NODEID="FC00:1111:1001:0001:0100:1100:0000:0001", FQDNNODEID="consumer.huawei.com";
SET IPALLOCBYSMFGLBSW: SWITCH=ENABLE;
// SMF 级开关可选（SWITCH=INHERIT 继承全局值）
```

## 场景差异

| 场景 | CPNODEID | SMF 全局开关 | SMF 级开关 | 备注 |
|---|---|---|---|---|
| 基于 SMF（场景二） | 必配 | 必配（ENABLE） | 可选 | 010105 范式 |
| 基于 SMF+APN（场景三） | 必配 | 必配（ENABLE） | 可选 | 010105 范式 |
| 基于 APN/DNN（场景一） | 不配 | 不配 | 不配 | 非 SMF 子方式 |
| 基于 RADIUS（场景四） | 不配 | 不配 | 不配 | 非 SMF 子方式 |
| 基于位置（参见 GWFD-020421） | 不配 | 不配 | 不配 | 走位置链路 |

## 决策点：SMF 子方式开关两级控制（DP 0-00323）

问：SMF 开关怎么组合？

| 选项 | 全局 SWITCH | 指定 SMF SWITCH | 联动行为 |
|---|---|---|---|
| 全局使能 + SMF 继承 | ENABLE | INHERIT（默认） | SMF 开关继承全局值 |
| 全局使能 + SMF 覆盖 | ENABLE | DISABLE | 该 SMF 不参与基于 SMF 分配 |
| 全局使能 + SMF 强使能 | ENABLE | ENABLE | 冗余配置，与 INHERIT 等效 |
| 全局不使能 + SMF 强使能 | DISABLE | ENABLE | 该 SMF 参与，全局其他 SMF 不参与 |

## 约束

- **CPNODEID 三类型三选一**（critical，rule-0-00495）：IPV4NODEID / IPV6NODEID / FQDNNODEID 至少配一项；多类型可同时配，但需与对端 SMF 一致 — 后果：NodeID 不匹配致 PFCP 信令失败
- **不同 SMF 实例 NodeID 不可相同**（critical，rule-0-00495）：不同 CPNAME 对应的 IPV4/IPV6/FQDN NodeID 不可重复 — 后果：NodeID 冲突致 SMF 识别错乱
- **SET IPALLOCBYSMFGLBSW 整机唯一**（warning，rule-0-00495）：max_records=1，本命令整机唯一 — 后果：多次执行覆盖
- **SET IPALLOCBYSMFSW SMF 必配**（critical，rule-0-00495）：SMF 必须通过 ADD CPNODEID 预建（CPNAME）— 后果：SMF 未建命令拒收
- **SET IPALLOCBYSMFSW 默认 SWITCH=INHERIT**（info，rule-0-00495）：SMF 级开关默认继承全局 — 后果：未配时按全局值生效
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
