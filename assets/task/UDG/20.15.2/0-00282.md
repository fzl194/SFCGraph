# 设置IMS Bypass功能参数（SET IMSBYPASS）

> 命令静态知识见 [SET IMSBYPASS](command/UDG/20.15.2/SET-IMSBYPASS.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务在 PGW-U/UPF 上开启 PCF 双故障 Bypass，降低 PCF 双故障对 IMS 语音业务的影响。`IMSBYPASSSW` 为总开关；开启后由 `QOSURRFLOWRPT` 决定 QoS URR Stop 上报时流信息的填充方式，`QOSURRHYSTIMER` 决定 QoS 类型 URR Stop 上报的迟滞时间。

整机老化时间 = APN 老化时长 + `QOSURRHYSTIMER`（APN 老化时长通过 `LST APNFLOWAGETIME` 或 `LST FLOWAGETIME` 查询）。

**取值样例（数据规划表，主特性 GWFD-112000 双故障Bypass）**

| 参数 | 含义 | 样例值 | 获取方法 |
|---|---|---|---|
| IMSBYPASSSW | IMS Bypass 开关 | ENABLE | 固定取值（本特性必须开启） |
| QOSURRFLOWRPT | QoSURR 上报流信息方式 | FLOW | 本端规划（FLOW/FILTER；FLOW=流级别，FILTER=规则级别） |
| QOSURRHYSTIMER | QoSURR 迟滞时间（秒） | 0 | 本端规划（0～3600，整机老化=APN老化+本值） |

**主场景脚本样例**

```
SET IMSBYPASS: IMSBYPASSSW=ENABLE, QOSURRFLOWRPT=FLOW, QOSURRHYSTIMER=0;
```

`QOSURRFLOWRPT`/`QOSURRHYSTIMER` 取值随网元老化策略规划而定（获取方法多值，但跨场景不构成 role 级分叉，按 `local_planned` 变量处理）。

## 决策点

（本命令的核心维度是 IMSBYPASSSW 总开关（DISABLE/ENABLE 二态）；本特性场景固定 ENABLE，QOSURRFLOWRPT / QOSURRHYSTIMER 取值随网元老化策略规划而定，不构成 role 级分叉，未建 DP。）

## 约束

- **命令记录上限**（critical，rule-0-00282-01）：命令最大记录数 **1**，超出被拒。
- **存量会话不感知**（warning，rule-0-00282-02）：执行后**对新数据流生效**，存量会话不感知 — 切换 IMSBYPASSSW 后老用户需重新激活才能享受 Bypass。
- **测试特性限定**（critical，rule-0-00282-03）：IMS Bypass 在当前版本为**测试特性**，仅用于测试场景，**不能用于现网业务部署**。
- **系统初始记录默认值**（info，rule-0-00282-04）：`IMSBYPASSSW=DISABLE`、`QOSURRFLOWRPT=FLOW`、`QOSURRHYSTIMER=0`。
- **前置 SET REFRESHSRV**（warning，rule-0-00282-05）：前置需在编排步骤中执行 [0-00015](task/UDG/20.15.2/0-00015.md) SET REFRESHSRV 业务刷新。
