# 配置用户模板（ADD USERPROFILE）

> 命令静态知识见 [ADD USERPROFILE](command/UDG/20.15.2/ADD-USERPROFILE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

USERPROFILE 是规则组容器（RULE 的归属）。配置生成时通常**只填 USERPROFILENAME**——名称来源随场景分叉（见决策点），其余参数为按特性可选的能力开关。

**典型范式（计费场景，本端规划，2-00001 部署 UPF 等）**：
```
ADD USERPROFILE: USERPROFILENAME="userprofile001";
ADD USERPROFILE: USERPROFILENAME="userprofile002";
```
按业务语义规划多个 UP（如 any 兜底 / IMS 专网），后续 [ADD RULEBINDING](0-00012.md) 各绑一组 RULE。

**PCC 场景（动态 PCC / 本地 PCC，GWFD-020351）**：
```
ADD USERPROFILE: USERPROFILENAME="up_test";
```
名称须与 SMF 上一致——本地 PCC 时此 UP 作为预定义规则组标识，由 SMF 下发激活 UPF 本地静态规则。

**可选能力开关（按特性启用，③实证）**：

| 参数 | 启用场景（证据③） | 典型取值 |
|---|---|---|
| ALIASMARKFLAG | 会话 QoS / 双故障 Bypass / ProxyUPF | ENABLE |
| DDOSCHECK | 防 DDoS / 双故障 Bypass | ENABLE |
| CAPMODETHRES | 用户 Portal / Web Proxy | 0~60（分钟） |

## 决策点

**DP 0-00006**（single_choice, required, always）用户模板名称来源？

| 选项 | USERPROFILENAME 来源 | 适用 |
|---|---|---|
| opt-up-local（默认） | 本端规划（计费向后兼容） | 计费场景 |
| opt-up-peer | 与 SMF 协商一致；本地 PCC 时作规则组激活标识由 SMF 下发 | PCC / 本地 PCC 场景（GWFD-020351） |

## 约束

- **rule-0-00132**（warning）规格：UPSPECTYPE=DEFAULT（或未指定）最大记录数 **5000**；SPECIFICATION_LIMITED 最大 **100000**；两类合计规格 105000，超 90% 上报 ALM-135602215（阈值 `MOD CFGTHRESHOLD` 修）。名称一致性：USERPROFILENAME 须在 PCF/SMF/UPF（或 PCRF/PGW-C/PGW-U）间一致，不一致则 PCF 下发的策略无法生效。**立即生效**。
