# 配置用户模板（ADD USERPROFILE）

> 命令静态知识见 [ADD USERPROFILE](command/UDG/20.15.2/ADD-USERPROFILE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

USERPROFILE 是过滤链 backbone 的"RULE 归属容器"：把若干 RULE 装入同一个 UP，由 [ADD RULEBINDING](0-00012.md) 把 RULE 绑到 UP，再由 [SET URRGRPBINDING](0-00013.md) 给该 UP 配缺省 URR 组。本命令的核心配置空间有 3 个维度：**名称来源**（本端规划或与对端协商，决定跨网元一致性）、**能力开关**（按特性启用，控制 ALIASMARKFLAG / DDOSCHECK / CAPMODETHRES）、**规格类型**（DEFAULT / SPECIFICATION_LIMITED，决定单板最大记录数）。

### 配置维度 1：名称来源（参数 USERPROFILENAME，必选）

USERPROFILENAME 是该命令的核心键，后续 RULEBINDING / URRGRPBINDING / 业务特性配置都要引用此键。

| 取值来源 | USERPROFILENAME 取值约定 | 跨网元一致性要求 |
|---|---|---|
| **本端规划**（默认） | 由本端按业务语义独立规划，典型 `userprofile001` / `userprofile002` / `up_test` | 仅本端自洽 |
| **与对端协商** | 与 SMF/PCF/PGW-C 保持一致（用于 PCC/本地PCC 激活时作为预定义规则组标识） | SMF/UPF/PCF/PGW-C 一致 |

> 名称长度 1~32，由字母/数字/`_`/`.`/连字符组成（见命令 wiki）。**必须**与下游 [0-00012](0-00012.md) / [0-00013](0-00013.md) / 特性配置引用一致。

### 配置维度 2：能力开关（可选，按特性启用）

按业务需要使能不同能力开关，缺省全部关闭：

| 参数 | 取值 | 作用 | 联动必选参数 | 典型场景 |
|---|---|---|---|---|
| `ALIASMARKFLAG` | ENABLE / DISABLE | 别名/标记功能开关 | — | 会话 QoS、双故障 Bypass、ProxyUPF |
| `DDOSCHECK` | ENABLE / DISABLE | DDoS 检测开关 | — | 防 DDoS、双故障 Bypass |
| `CAPMODETHRES` | 0~60（分钟） | 容量模式阈值时间 | — | 用户 Portal、Web Proxy 超时控制 |
| `UPSPECTYPE` | DEFAULT / SPECIFICATION_LIMITED | 用户模板规格类型 | — | 见维度 3 |

> 条件可选——绝大多数特性不配置这四个参数；仅 Portal / DDoS / Bypass / ProxyUPF 场景需开启。

### 配置维度 3：规格类型（参数 UPSPECTYPE）

决定该 USERPROFILE 所受单板最大记录数约束，对应命令 notes 两条规则：

| 取值 | 单板最大记录数 | 触发条件 | 典型场景 |
|---|---|---|---|
| `DEFAULT`（或不配） | 5000 | 默认 | 一般计费/PCC 场景 |
| `SPECIFICATION_LIMITED` | 100000 | 大量用户模板池 | 大规模 ISP/集团客户场景 |

> 两类合计规格 105000——超 90% 上报 ALM-135602215，阈值可 `MOD CFGTHRESHOLD（★R1.5 atom 待补）` 修改。

**步骤位置**：在所属 feature 编排中通常位于"步骤 5：配置 USERPROFILE"——ADD RULE 之后，ADD RULEBINDING + SET URRGRPBINDING 之前。

## 决策点：用户模板名称来源（DP 0-00006）

问：USERPROFILENAME 取值来源（决定跨网元一致性范围）？

| 选项 | USERPROFILENAME 来源 | 联动要求 | 适用场景 |
|---|---|---|---|
| `local_planned`（默认） | 本端规划 | 仅本端自洽 | 计费场景（绝大多数） |
| `peer_aligned` | 与 SMF/PCF 一致；本地 PCC 时作为预定义规则组标识 | SMF/UPF/PCF/PGW-C 一致 | PCC/本地 PCC（GWFD-020351）|

## 约束

- **USERPROFILE 规格与告警**（warning，rule-0-00132）：UPSPECTYPE=DEFAULT（或未指定）最大记录数 5000；SPECIFICATION_LIMITED 最大 100000；两类合计 105000；超 90% 上报 ALM-135602215（阈值可 `MOD CFGTHRESHOLD（★R1.5 atom 待补）` 修改） — 后果：超规格新增失败；接近阈值时告警
- **USERPROFILENAME 跨网元一致**（critical，隐含）：与 PCF/SMF/UPF（或 PCRF/PGW-C/PGW-U）必须一致；本地 PCC 时作为预定义规则组标识由 SMF 下发激活 — 后果：PCF 下发的策略无法生效；本地PCC 规则组无法被激活
- **能力开关联动**（info，隐含）：CAPMODETHRES 仅在容量模式相关特性下生效；ALIASMARKFLAG/DDOSCHECK 开启需配套特性 License 已开 — 后果：误配开关能力不生效
