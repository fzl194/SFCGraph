# 配置带宽管理控制器业务等级策略（ADD BCSRVLEVELPLY）

> 命令静态知识见 [ADD BCSRVLEVELPLY](command/UDG/20.15.2/ADD-BCSRVLEVELPLY.md)

## 配置方法

为指定 BWM 控制器配置各 ServiceLevel 的业务等级策略，实现同一控制器下不同业务流间的**优先级公平调度**。BWMCNAME 引用 ADD BWMCONTROLLER 已配置的控制器名；SERVICELEVEL 越小优先级越高。

**两特性范式对比**（GWFD-110311 分级带宽管理 / GWFD-110313 智能Shaping组级）：

| 范式 | 核心参数 | 脚本样例 | 适用 |
|---|---|---|---|
| 多等级 CIR/PIR 分配（110311） | BWMCNAME / SERVICELEVEL / CIRRATE / PIRRATE | `ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=1,CIRRATE=50,PIRRATE=70;`<br>`ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=2,CIRRATE=30,PIRRATE=50;`<br>`ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=3,CIRRATE=20,PIRRATE=100;` | 用户组分级，按 CIRRATE 比例分配保证带宽 |
| Shaping 整形速率（110313） | BWMCNAME / SERVICELEVEL / SHAPRATE / MAXPKTLOSTRATE / MAXINCRDELAY | `ADD BCSRVLEVELPLY:BWMCNAME="bc1",SERVICELEVEL=2,SHAPRATE=100,MAXPKTLOSTRATE=10000,MAXINCRDELAY=100;` | 智能组级，实际 CIR = SHAPRATE × BWMCONTROLLER.RATE |

**关键推导**：
- CIRRATE/PIRRATE（万分比，0~10000）：承诺/峰值速率比例，同控制器下 CIRRATE 之和 ≤ 100%。
- SHAPRATE（万分比）：整形峰值带宽比例，实际承诺速率 = SHAPRATE × 控制器 RATE。
- MAXPKTLOSTRATE：该等级允许的最大丢包比率。
- MAXINCRDELAY：该等级因 Shaping 队列缓存的最大增加时延（毫秒）。

**步骤位置**：ADD BWMCONTROLLER 之后、配置 BWM 用户组之前——先定等级策略，规则再按等级调度。

## 约束

- 最大记录数 30000 · warning · 超规格新增失败
- 同一 BWMCNAME 下 CIRRATE 之和 ≤ 100%（万分比） · warning · 超限失败
- 此命令 CIR×CIRRate ≤ PIR×PIRRate（CIR/PIR 来自 BWMCONTROLLER） · warning · 跨命令强约束
- 同一 BWMCNAME 下 SERVICELEVEL 唯一（1~30，值越小优先级越高） · warning · 重复失败
- 命令执行后立即生效 · info
