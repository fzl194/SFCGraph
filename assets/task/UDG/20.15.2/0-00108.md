# 配置带宽管理控制器业务等级策略（ADD BCSRVLEVELPLY）

> 命令静态知识见 [ADD BCSRVLEVELPLY](command/UDG/20.15.2/ADD-BCSRVLEVELPLY.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD BCSRVLEVELPLY 用于在指定 BWMCONTROLLER 下为某一 ServiceLevel 配置保证/峰值/整形带宽比例与丢包/时延保护。**BWMCNAME + SERVICELEVEL 联合唯一**（同一 BWMC 下同一 LEVEL 不可重复），SERVICELEVEL 值越小优先级越高。核心配置空间有 4 个维度：**控制器绑定**（BWMCNAME）、**业务级别**（SERVICELEVEL）、**带宽比例**（CIRRATE/PIRRATE/SHAPRATE）、**业务保护**（MAXPKTLOSTRATE/MAXINCRDELAY）。2 个特性命中（GWFD-110311 / GWFD-110313）覆盖普通分级带宽与 Shaping 组级带宽两套典型场景。

### 配置维度 1：控制器绑定（参数 BWMCNAME，必选）

| 取值 | 作用 |
|---|---|
| **已建 BWMC 名称**（ref [0-00036](task/UDG/20.15.2/0-00036.md) ADD BWMCONTROLLER） | 该策略归属于哪个 BWMC 控制器 |

> BWMCNAME 必须是已通过 `ADD BWMCONTROLLER` 创建的控制器名；1~31 字符。

### 配置维度 2：业务级别（参数 SERVICELEVEL，必选）

| 取值 | 作用 |
|---|---|
| **0~N 整数** | 同一控制器内不同业务流的优先级，**值越小优先级越高**；系统优先保证高优先级带宽 |

> SERVICELEVEL 在同一 BWMC 下需唯一；不同 BWMC 可独立编号。

### 配置维度 3：带宽比例（参数 CIRRATE / PIRRATE / SHAPRATE，可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **CIRRATE** | 0~100 | 保证带宽比例；实际 CIR = CIRRATE × BwmController.CIR |
| **PIRRATE** | 0~100 | 峰值带宽比例；实际 PIR = PIRRATE × BwmController.PIR |
| **SHAPRATE** | 0~100 | Shaping 最高峰值带宽比例；实际 ShapRate × BwmController.RATE（GWFD-110313 范式） |

> 110311 范式按级别配 CIRRATE/PIRRATE（CIR/PIR 双轴）；110313 范式按级别配 SHAPRATE（Shaping 单轴）。

### 配置维度 4：业务保护（参数 MAXPKTLOSTRATE / MAXINCRDELAY，可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **MAXPKTLOSTRATE** | 0~10000 | 该业务等级的最大丢包比率（万分之） |
| **MAXINCRDELAY** | 0~N ms | 因 Shaping 队列缓存导致的最大增加时延（110313 范式） |

> 110313 范式联合配置：MAXPKTLOSTRATE=10000 + MAXINCRDELAY=100 + SHAPRATE=100。

**典型脚本（GWFD-110311 三级分级带宽）**：
```
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=1,CIRRATE=50,PIRRATE=70;
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=2,CIRRATE=30,PIRRATE=50;
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=3,CIRRATE=20,PIRRATE=100;
```

**典型脚本（GWFD-110313 Shaping 组级带宽控制）**：
```
ADD BCSRVLEVELPLY:BWMCNAME="bc1",SERVICELEVEL=2,SHAPRATE=100,MAXPKTLOSTRATE=10000,MAXINCRDELAY=100;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：为 BWMC 配置业务级别策略"——前置 [0-00036](task/UDG/20.15.2/0-00036.md) ADD BWMCONTROLLER 建好控制器；后续 [0-00034](task/UDG/20.15.2/0-00034.md) ADD BWMSERVICE 引用 SERVICELEVEL。

## 决策点：业务级别策略轴选择（DP 0-00108）

问：本策略采用 CIR/PIR 双轴还是 Shaping 单轴？

| 选项 | CIRRATE | PIRRATE | SHAPRATE | 业务保护 | 适用场景 |
|---|---|---|---|---|---|
| **CIR/PIR 双轴分级**（110311） | 按级别配置（50/30/20） | 按级别配置（70/50/100） | 不配 | 不配 | 普通分级带宽、业务优先级模型 |
| **Shaping 单轴**（110313） | 不配 | 不配 | 100 | MAXPKTLOSTRATE=10000,MAXINCRDELAY=100 | 智能 Shaping 组级带宽、依赖 Shaping 队列 |

> 2 个特性命中（110311/110313）；同一 BWMC 下同一 SERVICELEVEL 唯一；SERVICELEVEL 值越小优先级越高。

## 约束

- **BWMCNAME 必须已建**（critical，rule-0-00108）：BWMCNAME 必须是已通过 `ADD BWMCONTROLLER` 创建的控制器名 — 后果：BWMC 不存在命令拒收
- **(BWMCNAME,SERVICELEVEL) 联合唯一**（critical，rule-0-00108）：同一 BWMC 下同一 SERVICELEVEL 不可重复添加 — 后果：重复添加命令拒收
- **SERVICELEVEL 越小优先级越高**（info，rule-0-00108）：SERVICELEVEL 值越小业务优先级越高 — 后果：值配反导致业务优先级倒置
- **比例值是相对值**（info，rule-0-00108）：CIRRATE/PIRRATE/SHAPRATE 是相对 BwmController CIR/PIR/RATE 的比例 — 后果：忽略 BwmController 基准值导致实际带宽与预期不符
- **Shaping 依赖 GRPBWMSHAPSW**（warning，rule-0-00108）：使用 SHAPRATE 时 [0-00106](task/UDG/20.15.2/0-00106.md) SET BANDWIDTHMNG.GRPBWMSHAPSW=ENABLE 与 BWMC.SHAPINGSWITCH=ENABLE 须先开 — 后果：未开 Shaping 时 SHAPRATE 不生效
- **执行后立即生效**（warning，rule-0-00108）：本命令执行后立即生效 — 后果：误配即刻影响该 BWMC 下该级别业务带宽
