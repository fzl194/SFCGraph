# 基于业务流标识的无线资源优化（GWFD-110331）

> 特性静态知识见 [GWFD-110331](feature/UDG/20.15.2/GWFD-110331.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

基于业务流标识的无线资源优化（FPI）：UDG 开启 FPI 差异化控制开关后，将业务流标识（FPI）映射到业务数据报文的 DSCP 字段或 GTP-U 扩展头传递到无线侧，无线侧按 FPI 值进行 QoS 差异化调度，实现高价值业务/用户的资源倾斜。

**核心机制（activation 演示场景，DSCP 方式）**：`SET FPIFUNC` 全局使能（FPISWITCH=ENABLE, FPITRANSMETHOD=DSCP）→ 三四层过滤链（ADD FILTER/FLOWFILTER/FLTBINDFLOWF + SET REFRESHSRV）→ PCC 动作属性（ADD PCCACTIONPROP，UPINITDNGATE=PASS 下行透传）→ PCC 策略组（ADD PCCPOLICYGRP）→ 业务规则（ADD RULE，**两条**：POLICYTYPE=PCC 触发 PCC 策略 + POLICYTYPE=REMARK_FPI 标 FPI 值，REMARKFPISEL=FPI, FPIVALUE=63）。

**两条 RULE 的语义**：RULE1（POLICYTYPE=PCC + POLICYNAME=ppg_test）将匹配流绑定到 PCC 策略组（含动作属性 / 可含计费 URRGROUP）；RULE2（POLICYTYPE=REMARK_FPI + REMARKFPISEL=FPI + FPIVALUE=63）给匹配流打 FPI 标记，触发无线侧差异化调度。两条 RULE 共用同一 FLOWFILTER（同业务流），绑定同一 USERPROFILE。依赖 SA-Basic（GWFD-110101，SA 业务识别基础）+ PCC 基本功能（GWFD-020351，PCC 规则体系，activation 的 POLICYTYPE=PCC RULE 依赖）。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开本特性 License 开关，单命令）：`SET LICENSESWITCH`（LICITEM=LKV3G5WOFR01, SWITCH=ENABLE）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **FPI 全局使能**（特性专属首步，选 FPI 传递方式，单命令）：`SET FPIFUNC`（FPISWITCH=ENABLE, FPITRANSMETHOD=DSCP）→ [0-00109](task/UDG/20.15.2/0-00109.md)
3. **三四层过滤链**（业务流匹配，多命令可复用模块，**含刷新**）：`ADD FILTER`（FILTERNAME=filter_test, L34PROTTYPE=STRING, L34PROTOCOL=TCP）+ `ADD FLOWFILTER`（FLOWFILTERNAME=ff_test）+ `ADD FLTBINDFLOWF`（FILTER→FLOWFILTER）+ `SET REFRESHSRV`（REFRESHTYPE=ALL）→ [1-00009](task/UDG/20.15.2/1-00009.md)
   - activation 数据规划：三四层 = **TCP**（FPI 规则针对 TCP 业务流；非 IM 场景的 ANY 兜底，按目标业务规划）
4. **PCC 动作属性**（下行透传，单命令）：`ADD PCCACTIONPROP`（PCCACTPROPNAME=pap_test, UPINITDNGATE=PASS）→ [0-00028](task/UDG/20.15.2/0-00028.md)
   - FPI 信息携带在**下行**数据报文中（经 DSCP/GTP-U 扩展头传无线侧），下行需直接通过：UPINITDNGATE=PASS
5. **PCC 策略组**（绑动作属性 + 可选计费 URR 组，单命令）：`ADD PCCPOLICYGRP`（PCCPOLICYGRPNM=ppg_test, PCCACTPROPNAME=pap_test, URRGROUPNAME=urrgp_test）→ [0-00003](task/UDG/20.15.2/0-00003.md)
   - activation 注："当需要计费时，请配置参数 URRGROUPNAME"——本特性 activation 演示携带 URRGROUPNAME=urrgp_test（URR 组由 [计费三件套](task/UDG/20.15.2/1-00010.md) 预配，本特性非计费特性，URR 组**从已配置数据获取**）
6. **业务规则**（双 RULE + 用户模板 + 规则绑定，多命令可复用模块）：`ADD RULE`（**两条**：rule_test1 POLICYTYPE=PCC + rule_test2 POLICYTYPE=REMARK_FPI REMARKFPISEL=FPI FPIVALUE=63）+ `ADD USERPROFILE`（USERPROFILENAME）+ `ADD RULEBINDING`（RULE→USERPROFILE）→ [1-00011](task/UDG/20.15.2/1-00011.md)
   - rule_test1（PCC 触发）：`ADD RULE:RULENAME="rule_test1",POLICYTYPE=PCC,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="ff_test",PRIORITY=1,POLICYNAME="ppg_test";`
   - rule_test2（FPI 标记）：`ADD RULE:RULENAME="rule_test2",POLICYTYPE=REMARK_FPI,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="ff_test",PRIORITY=2,REMARKFPISEL=FPI,FPIVALUE=63;`

> **参数证据链**（R1.5，配置流程脚注）：以上命令+参数均来自 activation §数据规划表 + §操作步骤 + §任务示例脚本。
> - SET LICENSESWITCH（LICITEM=LKV3G5WOFR01, SWITCH=ENABLE）：activation §任务示例脚本步骤1（原样）
> - SET FPIFUNC（FPISWITCH=ENABLE, FPITRANSMETHOD=DSCP）：activation §数据规划表 + §任务示例脚本步骤2（原样）
> - ADD FILTER（FILTERNAME=filter_test, L34PROTTYPE=STRING, L34PROTOCOL=TCP）：activation §数据规划表 + §任务示例脚本步骤3a（原样，三四层 TCP）
> - ADD FLOWFILTER（FLOWFILTERNAME=ff_test）：activation §数据规划表 + §任务示例脚本步骤3b（原样）
> - ADD FLTBINDFLOWF（FLOWFILTERNAME=ff_test, FILTERNAME=filter_test）：activation §任务示例脚本步骤3c（原样）
> - SET REFRESHSRV（REFRESHTYPE=ALL）：activation §操作步骤3d + §任务示例脚本步骤3d（原样）
> - ADD PCCACTIONPROP（PCCACTPROPNAME=pap_test, UPINITDNGATE=PASS）：activation §数据规划表 + §任务示例脚本步骤4（原样，"FPI规则携带在下行数据报文中，因此下行数据需要配置为直接通过"）
> - ADD PCCPOLICYGRP（PCCPOLICYGRPNM=ppg_test, PCCACTPROPNAME=pap_test, URRGROUPNAME=urrgp_test）：activation §数据规划表 + §任务示例脚本步骤5（原样）
> - ADD RULE rule_test1（POLICYTYPE=PCC + FILTERINGMODE=FLOWFILTER + FLOWFILTERNAME=ff_test + PRIORITY=1 + POLICYNAME=ppg_test）：activation §数据规划表 + §任务示例脚本步骤6（原样）
> - ADD RULE rule_test2（POLICYTYPE=REMARK_FPI + FILTERINGMODE=FLOWFILTER + FLOWFILTERNAME=ff_test + PRIORITY=2 + REMARKFPISEL=FPI + FPIVALUE=63）：activation §数据规划表 + §任务示例脚本步骤6（原样，FPI 值 63 取全网规划）

## 决策点

**DP（特性级）：FPI 传递方式 + 三四层协议 + FPI 值**（影响 SET FPIFUNC + ADD FILTER + ADD RULE rule_test2）。维度按 activation §数据规划表 + [0-00109](task/UDG/20.15.2/0-00109.md) DP 0-00139 全列。

| 场景 | FPITRANSMETHOD | L34PROTOCOL | REMARKFPISEL | FPIVALUE | 走法 |
|---|---|---|---|---|---|
| **DSCP 方式·TCP 业务（activation 演示）** | DSCP | TCP | FPI | 63 | 骨架（步骤1~5）+ 直接双 RULE（步骤6）|
| GTP-U 方式（FPI 取本地规则，FPIPOLICYORIGIN=INTERNAL）| GTPU | 按业务规划 | FPI | 全网规划（0~255，按 FPI 值与调度权重对应表）| 同上 |
| FPI 取用户报文 DSCP（FPIPOLICYORIGIN=EXTERNAL）| GTPU | 按业务规划 | （下行报文 DSCP 映射 GTP-U 扩展头，REMARKFPISEL 见命令 wiki）| 见命令 wiki | 同上 |
| 其他三四层协议（UDP/ANY 等） | DSCP（默认）| UDP / ANY | FPI | 全网规划 | 同上 |

**联动**：
- FPITRANSMETHOD → [0-00109](task/UDG/20.15.2/0-00109.md) SET FPIFUNC（DSCP/GTPU 按 RAN 能力选；GTPU 方案需整网规划 MTU）
- L34PROTOCOL → [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链 ADD FILTER（按目标业务流规划，本特性 activation 取 TCP）
- REMARKFPISEL + FPIVALUE → [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定 ADD RULE rule_test2（REMARK_FPI 变体专属，FPI 值与 RAN 侧调度权重对应表见特性 wiki 表1）

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5WOFR01`（控制项 82200DHE） — 未开则本特性不生效。
- **强依赖 SA-Basic**（critical）：[GWFD-110101](feature/UDG/20.15.2/GWFD-110101.md)（License LKV3G5SABS01，控制项 82209749）—— UDG 基于 SA 执行业务识别，本特性开通必须同时开启 SA 功能（特性 wiki 明示）。
- **依赖 PCC 基本功能**（warning）：[GWFD-020351](feature/UDG/20.15.2/GWFD-020351.md)（控制项 82209825）—— activation 演示 POLICYTYPE=PCC 的 RULE1（POLICYNAME=PCCPOLICYGRPNM），PCC 规则体系为 FPI 规则链的载体（特性 wiki「与其他特性的交互」仅显式列 SA-Basic，但 activation 的 PCC RULE 隐含 PCC 基本功能依赖）。
- **解决方案特性·无线侧必须支持**（critical）：本特性要求无线侧网元同时支持基于业务流标识的无线资源优化功能 — 否则 FPI 信息到无线侧不触发调度。
- **GTP-U 方案影响 MTU 规划**（warning）：GTP-U 扩展头（最长 12 字节）影响网络 MTU 规划，需整网规划 MTU 值；DSCP 方案对性能影响可忽略。
- **双 RULE 共用 FLOWFILTER**（warning）：rule_test1（PCC）与 rule_test2（REMARK_FPI）绑定同一 FLOWFILTERNAME + PRIORITY 区分（activation：1/2） — PRIORITY 不能重复。
- **FPIPOLICYORIGIN 取值**（info）：默认 NULL（主特性场景），DSCP 传递方式；INTERNAL=本地规则取 FPI 值；EXTERNAL=用户下行报文 DSCP 映射 GTP-U 扩展头。
