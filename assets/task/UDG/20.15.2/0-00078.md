# 创建OSPFv3进程(IPv6)（ADD OSPFV3）

> 命令静态知识见 [ADD OSPFV3](command/UDG/20.15.2/ADD-OSPFV3.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令是 OSPFv3 (IPv6) backbone 根**——后续 [0-00079](task/UDG/20.15.2/0-00079.md) ADD OSPFV3AREA / [0-00117](task/UDG/20.15.2/0-00117.md) ADD OSPFV3INTERFACE / [0-00080](task/UDG/20.15.2/0-00080.md) ADD OSPFV3IMPORTROUTE 都必须在已有 PROCID 进程的前提下执行。

## 配置方法（配置方法字典）

ADD OSPFV3 是"**创建 IPv6 OSPFv3 动态路由协议进程**"命令。PROCID 必选；**ROUTERID 必配**（与 ADD OSPF 不同——不配则进程无法生成 LSA）；VRFNAME 决定进程归属（首次创建时确定，后续不可更改）。核心配置空间有 3 个维度：**进程号**（PROCID，1~4294967295）、**Router ID**（ROUTERID，IPv4 点分十进制，必配）、**VPN 归属**（VRFNAME，默认公网 `_public_`）、**协议进阶参数**（BFD/SPF/Stub/Domain/SA 等 28 项可选）。4 个特性命中（GWFD-010155/020401/020403/020406）覆盖 3 种范式：VPN 多实例+共网段、Internet 接入纯三层、Prefix Delegation。

### 配置维度 1：进程号（参数 PROCID，必选）

| 参数 | 取值范围 | 数据来源 | 备注 |
|---|---|---|---|
| **PROCID** | 1~4294967295 | 本端规划 / 全网规划 | 新增 OSPFv3 进程号不能与现网重复 |

### 配置维度 2：Router ID（参数 ROUTERID，**必选**）

| 参数 | 取值范围 | 数据来源 | 备注 |
|---|---|---|---|
| **ROUTERID** | IPv4 地址类型（点分十进制） | 全网规划 | **必选且全网唯一**——不配进程无法生成 LSA |

> **关键约束**：**如果没有指定 Router ID，则 OSPFv3 进程无法运行**（只能看到该进程，无法生成 LSA）。通过 [MOD OSPFV3](command/UDG/20.15.2/MOD-OSPFV3.md) 可重新配置 Router ID 但需重启进程。

### 配置维度 3：VPN 归属（参数 VRFNAME，可选，默认 _public_）

| 取值 | 默认值 | 备注 |
|---|---|---|
| `_public_`（默认） | _public_ | 进程属于全局实例 |
| ref [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST 已建的 VPN 名 | — | 进程属于该实例 |

> **关键约束**：**进程实例不可更改 VRFNAME**——只能第一次使能时指定；**设备的一个接口只能属于某一个 OSPFv3 进程**。**VPN 多实例场景须配 `VPNINSCAPSIMFLG=TRUE`** 去使能 VPN 路由环路检测，否则路由引入失败。

### 配置维度 4：协议进阶参数（可选，按需细化）

| 子维度 | 关键参数 | 默认值 | 典型场景 |
|---|---|---|---|
| **BFD 联动** | BFDALLINTFFLG（全局使能 BFD）+ BFDRXCFGFLAG+BFDMINRXINTV + BFDTXCFGFLAG+BFDMINTXINTV + DETECTMULINTV（3~50）+ FRRBINDINGFLAG | FALSE / 200ms / 200ms / 3 / FALSE | OSPFv3 BFD 加速收敛（010155/020406 范式） |
| **接口抑制** | SILENTALLFLAG | FALSE | 静默进程下所有接口 |
| **接口虚拟系统/共网段** | VIRTUALSYSFLAG（进程级，默认 TRUE）+ 配套 [0-00117](task/UDG/20.15.2/0-00117.md) ADD OSPFV3INTERFACE 接口级 VIRTUALSYSFLAG | TRUE | 共网段生效后 DRPRI 自动变 0；进程级+接口级须同时配 |
| **路由标签** | ROUTAGFLAG/ROUTAG | FALSE / — | VPN 引入路由 tag 值 |
| **最大重传** | RXMTLIMITENABLE/RETRANSLIMIT | FALSE / 30 | OSPFv3 报文重传限制 |
| **Stub 路由器** | STUBROUFLG（NoConfig/StubOnHand/StubOnBoot）+ STUBROUSTUPINTV | NoConfig / 500s | 倒换期抑制 |
| **环路检测** | VPNINSCAPSIMFLG（MCE 多实例去使能 VPN 环路检测）+ RTTAGDISFLAG | FALSE / FALSE | VPN 多实例必配 |
| **IPsec SA** | SANAME（1~15 字符） | — | IPsec 安全联盟（不能与认证同时配） |
| **SPF 智能定时器** | SPFMAXINTERVAL/SPFSTARTINTV/SPFHOLDINTERVAL/SPFDELAYINTV/SPFHOLDTIME | 5000ms/50ms/200ms/—/— | 自适应 SPF 抑制 |
| **共网段** | VIRTUALSYSFLAG（默认 TRUE）+ 配套 [0-00117](task/UDG/20.15.2/0-00117.md) ADD OSPFV3INTERFACE | TRUE | 进程级+接口级同时配 |
| **Domain ID** | DOMAINNULLFLAG | FALSE | 配置 Domain ID 为空 |
| **自描述** | DESCRIPTIONTEXT（1~80 字符） | — | 进程描述 |
| **协议安全** | 配套 [ADD OSPFV3AUTH](command/UDG/20.15.2/ADD-OSPFV3AUTH.md) / [ADD OSPFV3AREAAUTH](command/UDG/20.15.2/ADD-OSPFV3AREAAUTH.md) / [ADD OSPFV3INTERFACEAUTH](command/UDG/20.15.2/ADD-OSPFV3INTERFACEAUTH.md) 认证 | — | 建议配置协议认证 |

**典型脚本（010155 PGW-U→ePDG IPv6 动态路由，BFD+VPN 多实例+共网段全配）**：
```
ADD OSPFV3: PROCID=6, VRFNAME="vpn-paif", ROUTERID="10.8.20.1", BFDALLINTFFLG=TRUE, VPNINSCAPSIMFLG=TRUE, VIRTUALSYSFLAG=TRUE;
```

**典型脚本（020401 IPv6 承载上下文，纯三层）**：
```
ADD OSPFV3: PROCID=100, ROUTERID="10.10.10.1", VRFNAME="VPN_Internet";
```

**典型脚本（020406 IPv6 Prefix Delegation，BFD+VPN 多实例+共网段全配）**：
```
ADD OSPFV3: PROCID=6, VRFNAME="vpn1", ROUTERID="10.8.25.1", BFDALLINTFFLG=TRUE, VPNINSCAPSIMFLG=TRUE, VIRTUALSYSFLAG=TRUE;
```

> 命令 notes：**最大记录数 8016**；**执行后立即生效**；设备的一个接口只能属于某一个 OSPFv3 进程；**进程实例 VRFNAME 不可更改**（仅第一次使能时指定 VPN/全局归属）；**整机进程实例数规格默认 2000**；**ROUTERID 必配**否则进程无法生成 LSA；**建议配置协议认证**（ADD OSPFV3AUTH/AREAAUTH/INTERFACEAUTH）增强协议安全性。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：创建 OSPFv3 进程"——前置 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST（VRFNAME 引用），下游 [0-00079](task/UDG/20.15.2/0-00079.md) ADD OSPFV3AREA / [0-00117](task/UDG/20.15.2/0-00117.md) ADD OSPFV3INTERFACE / [0-00080](task/UDG/20.15.2/0-00080.md) ADD OSPFV3IMPORTROUTE。

## 决策点：OSPFv3 VPN 归属与 Router ID 策略（DP 0-00078）

问：ADD OSPFV3 的 ROUTERID / VRFNAME / VPNINSCAPSIMFLG / VIRTUALSYSFLAG 取何策略？

| 选项 | ROUTERID | VRFNAME | VPNINSCAPSIMFLG | VIRTUALSYSFLAG | 适用场景 |
|---|---|---|---|---|---|
| **PGW-U→ePDG（vpn-paif）** | 全网规划 IPv4 | ref `vpn-paif`（公网 VPN） | TRUE | TRUE | GWFD-010155 含 BFD+VPN 多实例+共网段 |
| **Internet 接入** | 全网规划 IPv4 | ref `VPN_Internet` | FALSE | TRUE（默认） | GWFD-020401 纯 IPv6 接入 |
| **Prefix Delegation** | 全网规划 IPv4 | ref `vpn1` | TRUE | TRUE | GWFD-020406 IPv6-PD 引入 |

> **关键规则**：ROUTERID **必配且全网唯一**（否则进程无法生成 LSA）；VRFNAME 须在首次创建时确定，后续不可修改；**VPN 多实例场景 VPNINSCAPSIMFLG=TRUE 必配**去使能 VPN 路由环路检测；**共网段须进程级 + 接口级 VIRTUALSYSFLAG 同时配**（进程级单独配不生效）。

## 约束

- **最大记录数 8016**（warning，rule-0-00478）：本命令最大记录数为 8016 — 后果：超规格新增失败
- **整机进程实例数默认 2000**（warning，rule-0-00478）：OSPFv3 进程实例数规格默认为整机 2000，实际以服务规格文件为准 — 后果：整机超过规格新增失败
- **执行后立即生效**（warning，rule-0-00478）：命令执行后立即生效 — 后果：误操作即刻生效
- **进程实例 VRFNAME 不可更改**（critical，rule-0-00478）：进程实例不可更改，只能在第一次使能该进程时指定 VPN 归属 — 后果：错填 VRFNAME 必须删除进程并重建
- **接口只能属一个 OSPFv3 进程**（critical，rule-0-00478）：设备的一个接口只能属于某一个 OSPFv3 进程 — 后果：接口归错进程需先 RMV 后重新绑定
- **ROUTERID 必配**（critical，rule-0-00478）：只有在 OSPFv3 进程下配置了 Router ID，OSPFv3 进程才能正常运行，否则只能看到该进程，但无法生成 LSA — 后果：未配 ROUTERID 进程无法生成 LSA
- **Router ID 必须全网唯一**（critical，rule-0-00478）：ROUTERID 全网唯一 — 后果：冲突导致 OSPFv3 邻居无法建立
- **VPN 多实例须去环路检测**（critical，rule-0-00478）：VPN 多实例场景须配 VPNINSCAPSIMFLG=TRUE 去使能 VPN 路由环路检测 — 后果：未去使能导致 OSPFv3 路由引入失败
- **共网段须进程级+接口级同时配**（critical，rule-0-00478）：进程下 VIRTUALSYSFLAG 和接口下 VIRTUALSYSFLAG 同时配置，共网段才能生效 — 后果：单配进程级共网段不生效
- **共网段后 DRPRI 自动变 0**（warning，rule-0-00478）：共网段生效后 DRPRI 生效值自动变为 0；网络中需要有至少一个非共网段接口 — 后果：违反约束导致 DR 选举异常
- **SANAME 不能与认证同时配**（critical，rule-0-00478）：SANAME 不能和认证同时配置 — 后果：违反致命令拒收
- **建议配置协议认证**（warning，rule-0-00478）：建议配置协议相关认证，增强协议安全性 — 后果：未认证可能在协议层引入攻击风险
