# 配置防欺诈URR列表（ADD SPECURRGRPLIST）

> 命令静态知识见 [ADD SPECURRGRPLIST](command/UDG/20.15.2/ADD-SPECURRGRPLIST.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

ADD SPECURRGRPLIST 是计费 backbone 的"防欺诈标记位"——整体可选，由 DP 0-00003（防欺诈开关）决定是否启用。本命令的核心配置空间有 3 个维度：**是否启用**（由防欺诈 DP 决定，本命令本身不开 DP）、**唯一有效参数**（URRGROUPNAME reference）、**重复检测**（同 URR 组是否可重复加入）。

### 配置维度 1：是否启用（受 DP 0-00003 外部控制）

本 atom 本身不开 DP——防欺诈需不需要走 ADD SPECURRGRPLIST 由 DP 0-00003（外部 DP）决定：

| 上游决策 | 是否启用本命令 | 适用场景 |
|---|---|---|
| 未启用防欺诈（默认） | **不配** | 一般计费场景 |
| 启用防欺诈 + 模式 A：报表功能 | **配** | 内容计费等开启报表功能的特性——将有欺诈风险的费率对应的 URR 组加入防欺诈列表 |
| 启用防欺诈 + 模式 B：纯 N40/SID 级计费 | **不配** | N40 融合计费且采用 SID 级计费——**避免**将 RG 对应的 URR Group 加入本列表（否则重复统计） |

### 配置维度 2：唯一有效参数（URRGROUPNAME reference）

| 参数 | 必填 | 取值 | 来源 |
|---|---|---|---|
| `URRGROUPNAME` | ✓ | 字符串 1~31 字符，引 [ADD URRGROUP](task/UDG/20.15.2/0-00002.md) 产出 | reference |

> 命令真相对该参数描述极简——本命令只承担"对哪些费率组做防欺诈检测"这一开关标记。

### 配置维度 3：防欺诈统计行为（隐含铁律）

虽非参数，但**必须**理解的统计行为：

| 触发条件 | 统计口径 |
|---|---|
| 用户同时使能离线 + 在线计费，且 URR Group 中绑定了离线计费 URR | 欺诈流量**只统计绑定的离线计费 URR 流量** |
| 未绑离线计费 URR | 欺诈流量**不统计** |
| 已绑在线计费 URR | 欺诈流量**不统计在线 URR 流量** |
| N40 融合计费 + SID 级，且 URR Group 加入本列表 | RG 对应流量被重复统计（应避免） |

**步骤位置**：在所属 feature 编排中通常位于"步骤 9：防欺诈列表"——SET URRGRPBINDING 之后，SET REFRESHSRV 之前。

## 决策点：无（本命令仅一个有效参数 URRGROUPNAME，配置用法一致，不开 DP）

> 是否启用本命令由外部 DP 0-00003（防欺诈开关）决定；本 atom 仅承载执行细节，不重复开 DP。

## 约束

- **SPECURRGRPLIST 最大记录数**（warning，rule-0-00114）：最大记录数 40000；超规格新增失败；**立即生效** — 后果：超规格新增失败
- **欺诈流量只统计离线计费 URR**（warning，rule-0-00115）：用于防欺诈检测时，用户同时使能离线+在线计费，欺诈流量**只统计 URR Group 中绑定的离线计费 URR 流量**（未绑则不统计），**不统计在线计费 URR 流量** — 后果：流量只对离线 URR 生效，报表数据视角片面
- **N40 SID 级计费避免双重统计**（warning，隐含）：N40 融合计费 + SID 级计费场景，**不要将 RG 对应的 URR Group 加入本列表** — 后果：RG 对应流量重复统计
- **URRGROUPNAME 一致性**（critical，隐含）：URRGROUPNAME 必须与 [ADD URRGROUP](task/UDG/20.15.2/0-00002.md) 已配 URRGROUPNAME 完全一致 — 后果：引用错位，防欺诈漏报或错报
- **重复加入行为**（info，隐含）：同 URRGROUPNAME 不可重复加入（40000 上限按去重后累计计） — 后果：重复 ADD 失败 / 占用名额
