# 配置防欺诈URR列表（ADD SPECURRGRPLIST）

> 命令静态知识见 [ADD SPECURRGRPLIST](command/UDG/20.15.2/ADD-SPECURRGRPLIST.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

整个 atom 可选，由 DP 0-00003（防欺诈开关）决定是否启用。仅 URRGROUPNAME 一个有效参数（reference），把上游产出、有欺诈风险的 URR 组加入防欺诈列表。所有特性（2-00001/04/09/10/11）配置范式完全一致：

```
ADD SPECURRGRPLIST: URRGROUPNAME="UrrGp_URL";
```

URRGROUPNAME 引 [0-00002](0-00002.md) ADD URRGROUP 产出的、对应有欺诈风险费率的 URR 组。配置位置在 SET URRGRPBINDING 之后、SET REFRESHSRV 之前。

## 约束

- **rule-0-00114**（warning）规格：最大记录数 **40000**，超规格新增失败。**立即生效**。
- **rule-0-00115**（warning）流量统计行为：用于防欺诈检测时，若用户同时使能离线 + 在线计费，欺诈流量**只统计 URR Group 中绑定的离线计费 URR 流量**（未绑则不统计），不统计在线计费 URR 流量。使用 N40 融合计费且采用 SID 级计费时，为避免重复统计，不要将 RG 对应的 URR Group 加入本列表。
