# 配置IPv4静态路由（ADD SRROUTE）

> 命令静态知识见 [ADD SRROUTE](command/UDG/20.15.2/ADD-SRROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。静态路由是手工配置的特殊路由——当网络结构简单、设备不能用动态路由协议或无法建立到目的网络的动态路由时使用；合理的静态路由可改进网络性能并为重要业务保证带宽。

## 配置方法（配置方法字典）

ADD SRROUTE 是"IPv4 单播静态路由添加"命令。**AFTYPE 固定 ipv4unicast**；**PREFIX/MASKLENGTH 必填**（构成目的网段）；**IFNAME/NEXTHOP/VRFNAME/BFDENABLE/SESSIONNAME** 随场景分叉。核心配置空间有 6 个维度：**地址族**（AFTYPE 固定 ipv4unicast）、**路由目的**（PREFIX + MASKLENGTH 三元组之一）、**出接口**（IFNAME Tunnel/Invalid0）、**下一跳**（NEXTHOP IPv4 地址）、**VPN 划分**（VRFNAME + DESTVRFNAME）、**BFD 联动**（BFDENABLE + SESSIONNAME）。4 个特性命中（010107、010155、IPFD-015002、IPFD-015004）覆盖五种典型场景。

### 配置维度 1：地址族（参数 AFTYPE，必选，固定 ipv4unicast）

| 取值 | 作用 |
|---|---|
| **ipv4unicast**（固定） | IPv4 单播静态路由 |

### 配置维度 2：路由目的（参数 PREFIX + MASKLENGTH，必填三元组之一）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **PREFIX** | IPv4 地址 | 路由前缀（目的网段/IP） |
| **MASKLENGTH** | 0~32 | 路由掩码长度（0 = 默认路由） |

> PREFIX 与 MASKLENGTH 共同构成"目的网段"。**添加该静态路由时要保证该路由未添加过**（避免重复）。**不能在 VPN 实例 `__mpp_vpn_inner__` 和 `__mpp_vpn_inner_server__` 下添加默认静态路由**（MASKLENGTH=0）。

### 配置维度 3：出接口（参数 IFNAME，可选）

| 取值 | 默认值 | 适用场景 |
|---|---|---|
| **Tunnel<n>** | — | GRE/IPsec 隧道承载路由（010107、IPFD-015002、IPFD-015004 范式） |
| **Invalid0** | Invalid0（不配时默认） | 逻辑接口出接口，依赖 NEXTHOP 解析（010155 VPN 缺省、IPsec 明细路由） |
| **NULL0** | — | 黑洞路由（命令使用实例：丢弃到特定 IP 的流量） |
| **Ethernet<n>** | — | 以太网外联口（IPFD-015004 引流明细） |

> IFNAME 不配时**默认 Invalid0**，不区分大小写。

### 配置维度 4：下一跳（参数 NEXTHOP，可选）

| 场景 | 是否必填 | 备注 |
|---|---|---|
| **广播接口类型**（如 Ethernet） | 必填 | NEXTHOP 必填 |
| **非广播接口类型**（如 Tunnel） | 可不填 | 引 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS 配的 Tunnel IP |
| **Invalid0 出接口** | 必填 | NEXTHOP 必填（010155 范式） |

### 配置维度 5：VPN 划分（参数 VRFNAME + DESTVRFNAME，可选）

| 参数 | 取值 | 默认值 | 作用 |
|---|---|---|---|
| **VRFNAME** | 1~31 字符串 | `_public_` | 路由所属 VPN 实例；不配默认公网 |
| **DESTVRFNAME** | 1~31 字符串 | 与 VRFNAME 或出接口 VPN 一致 | 路由下一跳所属 VPN 实例 |

> **联动规则**：DESTVRFNAME 不输入时——若未配出接口，默认与 VRFNAME 一致；若配了出接口，默认与出接口所属 VPN 一致。

### 配置维度 6：BFD 联动（参数 BFDENABLE + SESSIONNAME，可选）

| 参数 | 取值 | 默认值 | 联动约束 |
|---|---|---|---|
| **BFDENABLE** | TRUE / FALSE | FALSE | TRUE 时必须通过 `ADD SRBFDTEMPLET` 配置动态 BFD 模板 |
| **SESSIONNAME** | 1~64 字符串 | 无 | 静态 BFD 会话名；必须通过 [0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION 建立联动 BFD；与动态 BFD 互斥 |

> 静态 BFD 联动场景（010155 BFD 组网）：**BFDENABLE=TRUE + SESSIONNAME 必填**（前置全局使能 BFD）。

### 典型脚本矩阵

| 场景 | AFTYPE/PREFIX/MASKLENGTH | VRFNAME/DESTVRFNAME | IFNAME | NEXTHOP | BFD/SESSION |
|---|---|---|---|---|---|
| **冗余备份重定向**（010107） | ipv4unicast / `192.168.0.1` / 32 | `vpn1` / — | `Tunnel1` | 不配 | 不配 |
| **GRE 隧道间**（IPFD-015002） | ipv4unicast / `10.10.1.202` / 24 | 不配（公网） | `Tunnel1` | 不配 | 不配 |
| **IPsec 引流**（IPFD-015004） | ipv4unicast / `10.1.2.0` / 24 | `vrf1` / `vrf1` | `Tunnel10` | 必填 peer | FALSE + PREFERENCE=60 |
| **VPN 缺省**（010155 缺省组网） | ipv4unicast / `172.16.47.0` / 24 | `vpn-s2b` / `_public_` | `Invalid0` | 必填 peer | 不配 |
| **静态+BFD**（010155 BFD 组网） | ipv4unicast / `192.168.47.4` / 24 | `vpn-s2b` / `_public_` | 不显式（默认 Invalid0） | 必填 peer | TRUE + SESSIONNAME 必填 |

**典型脚本（010107 冗余备份，PREFIX=重定向 IP /32）**：
```
ADD SRROUTE: AFTYPE=ipv4unicast, PREFIX="192.168.0.1", MASKLENGTH=32, VRFNAME="vpn1", IFNAME="Tunnel1";
```

**典型脚本（010107 可选下行低优先级路由，PREFERENCE=60）**：
```
ADD SRROUTE: AFTYPE=ipv4unicast, PREFIX="10.1.1.0", MASKLENGTH=24, VRFNAME="vpn1", IFNAME="Tunnel1", BFDENABLE=FALSE, DHCPENABLE=FALSE;
```

**典型脚本（010155 VPN 缺省组网）**：
```
ADD SRROUTE:VRFNAME="vpn-s2b",AFTYPE=ipv4unicast,PREFIX="172.16.47.0",MASKLENGTH=24,DESTVRFNAME="_public_",NEXTHOP="192.168.94.1",IFNAME="Invalid0";
```

**典型脚本（010155 静态路由+BFD 组网）**：
```
ADD SRROUTE:AFTYPE=ipv4unicast,PREFIX="192.168.47.4",MASKLENGTH=24,VRFNAME="vpn-s2b",DESTVRFNAME="_public_",NEXTHOP="192.168.94.1",BFDENABLE=TRUE,SESSIONNAME="BfdName1";
```

> 命令 notes：**最大记录数 100000**；**执行后立即生效**；**添加时保证未添加过**（避免重复）；**不能在 VPN 实例 `__mpp_vpn_inner__` 和 `__mpp_vpn_inner_server__` 下添加默认路由**（MASKLENGTH=0）；PREFERENCE 默认 60（值越小优先级越高）；VRFNAME 默认 `_public_`（公网）；IFNAME 不配默认 `Invalid0`；DHCPENABLE 仅在通过 `ADD DHCPCLIENT` 使能接口 DHCP 时静态路由联动 DHCP 才生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：配置 IPv4 静态路由"——前置 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE（建出接口）+ [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS（配 IP）+ [0-00059](task/UDG/20.15.2/0-00059.md) ADD GRETUNNEL（建 GRE 隧道）/[0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION（BFD 联动前置）。

## 决策点：ADD SRROUTE 场景选择（DP 0-00062）

问：该 IPv4 静态路由用于哪种场景？（决定 PREFIX/VRFNAME/IFNAME/BFDENABLE/NEXTHOP/DESTVRFNAME/SESSIONNAME 的取源与是否必填）

| 选项 | PREFIX 来源 | VRFNAME | IFNAME | NEXTHOP | BFD |
|---|---|---|---|---|---|
| **冗余备份重定向**（010107，默认） | ref [0-00055](task/UDG/20.15.2/0-00055.md) 重定向 IP /32 | ref [0-00041](task/UDG/20.15.2/0-00041.md) VPNINST | ref [0-00056](task/UDG/20.15.2/0-00056.md) GRE Tunnel | 不配 | FALSE |
| **GRE 隧道间**（IPFD-015002） | peer_planned（对端网段） | 不配（公网） | ref [0-00056](task/UDG/20.15.2/0-00056.md) GRE Tunnel | 不配 | 不配 |
| **IPsec 引流**（IPFD-015004） | peer_planned | `vrf1` | `Tunnel10`/`Invalid0` | 必填 peer | FALSE，PREFERENCE=60 |
| **VPN 缺省**（010155 缺省组网） | peer_planned（明细或 0.0.0.0/0） | ref L3VPNINST | fixed `Invalid0` | 必填 peer | 不配 |
| **静态+BFD**（010155 BFD 组网） | peer_planned | ref VPNINST | fixed `Invalid0` | 必填 peer | TRUE + SESSIONNAME 必填（ref [0-00082](task/UDG/20.15.2/0-00082.md)） |

> 4 个特性命中（010107/010155/IPFD-015002/IPFD-015004）覆盖五种范式；默认冗余备份重定向（与现有 binding 默认值一致）；PREFIX 必须与重定向 IP 一致（010107 范式硬约束）。

## 约束

- **最大记录数 100000**（warning，rule-0-00462）：IPv4 静态路由最大 100000 条 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00462）：命令执行后立即生效 — 后果：误操作即刻生效
- **添加前保证未添加过**（critical，rule-0-00462）：添加该静态路由时要保证该路由未添加过 — 后果：重复添加命令拒收/异常
- **MPP 内部 VPN 禁加默认路由**（critical，rule-0-00462）：不能在 VPN 实例 `__mpp_vpn_inner__` 和 `__mpp_vpn_inner_server__` 下添加默认静态路由（MASKLENGTH=0） — 后果：受保护 VPN 实例下加默认路由将被拒绝/路由表异常
- **广播接口类型 NEXTHOP 必填**（critical，rule-0-00462）：广播接口类型（如 Ethernet）的 NEXTHOP 必须要输入；非广播接口类型（如 Tunnel）可不输入 — 后果：广播接口缺 NEXTHOP 命令拒收
- **DESTVRFNAME 联动默认值**（info，隐含）：DESTVRFNAME 不输入时——若未配出接口默认与 VRFNAME 一致；若配了出接口默认与出接口所属 VPN 一致 — 后果：DESTVRFNAME 不显式配时取隐含值
- **VRFNAME 不配默认公网**（info，隐含）：VRFNAME 不配置时默认为 `_public_`（公网）；`_public_` 表示公网 — 后果：私网路由忘配 VRFNAME 落入公网表
- **PREFERENCE 优先级方向**（info，隐含）：PREFERENCE 值越小优先级越高、越大越低；默认 60 — 后果：值配反致路由不优选
- **静态+BFD 前置约束**（critical，rule-0-00462）：BFDENABLE=TRUE 时必须通过 `ADD SRBFDTEMPLET` 配置动态 BFD 模板；SESSIONNAME 必须通过 [0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION 建立 — 后果：BFD 联动未前置建立则不生效
