# 配置IPv4静态路由（ADD SRROUTE）

> 命令静态知识见 [ADD SRROUTE](command/UDG/20.15.2/ADD-SRROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD SRROUTE 加 IPv4 静态路由。AFTYPE 固定 ipv4unicast，PREFIX/MASKLENGTH 必填。出接口 IFNAME、下一跳 NEXTHOP、VPN 实例 VRFNAME、BFD 等随场景分叉。

**5 种场景范式（从证据数据规划表归纳）**

| 场景 | VRFNAME | IFNAME | NEXTHOP | BFD/SESSION | 典型行 |
|---|---|---|---|---|---|
| 冗余备份重定向（010107） | vpn1（引用 VPNINST） | Tunnel1/Tunnel2（GRE） | 不配 | 不配 | `ADD SRROUTE:AFTYPE=ipv4unicast,PREFIX="192.168.0.1",MASKLENGTH=32,VRFNAME="vpn1",IFNAME="Tunnel1";` |
| GRE 隧道间（015002） | 不配（公网） | Tunnel1 | 不配 | 不配 | `ADD SRROUTE:AFTYPE=ipv4unicast,PREFIX="10.10.1.202",MASKLENGTH=24,IFNAME="Tunnel1";` |
| IPsec 引流（015004） | vrf1 | Tunnel10/Invalid0 | 必填 peer | BFDENABLE=FALSE | `ADD SRROUTE:VRFNAME="vrf1",AFTYPE=ipv4unicast,PREFIX="10.1.2.0",MASKLENGTH=24,DESTVRFNAME="vrf1",NEXTHOP="192.168.1.2",IFNAME="Tunnel10";` |
| VPN 缺省（010155 缺省组网） | vpn-s2b | Invalid0 | 必填 peer | 不配 | `ADD SRROUTE:VRFNAME="vpn-s2b",AFTYPE=ipv4unicast,PREFIX="172.16.47.0",MASKLENGTH=24,DESTVRFNAME="_public_",NEXTHOP="192.168.94.1",IFNAME="Invalid0";` |
| 静态路由+BFD（010155 BFD 组网） | vpn-s2b | Invalid0 | 必填 peer | BFDENABLE=TRUE + SESSIONNAME 必填 | `ADD SRROUTE:AFTYPE=ipv4unicast,PREFIX="192.168.47.4",MASKLENGTH=24,VRFNAME="vpn-s2b",NEXTHOP="192.168.94.1",BFDENABLE=TRUE,SESSIONNAME="BfdName1";` |

**AFTYPE 固定 ipv4unicast**。出接口为"无效/逻辑"时用 `Invalid0`（VPN 缺省、IPsec 明细路由）。DHCPENABLE 固定 false（010107 可选下行缺省低优先级路由）。

**冗余备份场景**：PREFIX = 冗余备份隧道虚拟重定向 IP（/32），须与 [SET REDUNDUSER] 配置的重定向 IP 一致；可选追加下行缺省低优先级用户网段路由（PREFERENCE=60，目的=用户地址池网段，如 `PREFIX="10.1.1.0",MASKLENGTH=24`）。

**静态路由+BFD**：必须**先全局使能 BFD**（ADD BFDSESSION），再 ADD SRROUTE 绑 SESSIONNAME。

## 决策点：ADD SRROUTE 场景选择（DP 0-00075）

问：该 IPv4 静态路由用于哪种场景？（决定 PREFIX/VRFNAME/IFNAME/BFDENABLE/NEXTHOP/DESTVRFNAME/SESSIONNAME 的取源与是否必填）

| 选项 | PREFIX | VRFNAME | IFNAME | NEXTHOP | BFDENABLE |
|---|---|---|---|---|---|
| 冗余备份重定向（010107，默认） | ref 0-00055 重定向 IP/32 | ref 0-00041 VPNINST | ref 0-00056 GRE Tunnel | 不配 | FALSE |
| GRE 隧道间（015002） | peer_planned（对端网段） | 不配（公网） | ref 0-00056 GRE Tunnel | 不配 | 不配 |
| IPsec 引流（015004） | peer_planned | vrf1 | Tunnel10/Invalid0 | 必填 peer | FALSE，PREFERENCE=60 |
| VPN 缺省（010155 缺省组网） | peer_planned（0.0.0.0/明细） | ref L3VPNINST | fixed Invalid0 | 必填 peer | 不配 |
| 静态路由+BFD（010155 BFD 组网） | peer_planned | ref VPNINST | fixed Invalid0 | 必填 peer | TRUE + SESSIONNAME 必填（ref 0-00082） |

默认冗余备份重定向（与现有 binding 默认值一致）。010155 静态+BFD 场景亦由 feature 级 DP 协同选定。

## 约束

- **rule 0-00181**（capacity）命令最大记录数 100000 条 IPv4 静态路由 — 后果：超 100000 新增失败
- **rule 0-00182**（constraint）PREFIX 为默认路由（MASKLENGTH=0）时，**不能**在 VPN 实例 `__mpp_vpn_inner__` 和 `__mpp_vpn_inner_server__` 下添加 — 后果：受保护 VPN 实例下加默认路由将被拒绝/路由表异常
