# 配置静态地址用户路由冗余全局开关（SET REDUNDUSER）

> 命令静态知识见 [SET REDUNDUSER](command/UDG/20.15.2/SET-REDUNDUSER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET REDUNDUSER 是"静态地址用户路由冗余"特性的**全局总开关**——一条全局唯一记录，控制整台设备静态地址用户路由冗余功能是否使能。本命令的核心配置空间仅 1 个维度：**开关取值**（SWITCH 枚举二值）。本开关须与地址池 `REDUNDFUNC=ENABLE`（[0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL 参数）**同时打开**才生效。本 atom 属特性链路**最后一步**——所有冗余相关配置完成后才使能。

### 配置维度 1：开关取值（参数 SWITCH，必选）

| 取值 | 作用 | 联动 | 典型场景 |
|---|---|---|---|
| **DISABLE** | 不使能全局冗余开关 | 系统初始值 | 系统初始化默认 |
| **ENABLE** | 使能全局冗余开关（高危命令） | 须 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL 同时配 REDUNDFUNC=ENABLE | 010107 静态地址冗余激活 |

> 适用 NF 限定：PGW-U、UPF；最大记录数为 1（全局唯一）；执行后立即生效；系统初始记录 SWITCH=DISABLE。

**典型脚本（010107 静态地址冗余激活）**：
```
SET REDUNDUSER: SWITCH=ENABLE;
```

**步骤位置**：在所属 feature 编排中位于"步骤 N：总开关使能"——前置 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL（地址池 REDUNDFUNC=ENABLE）+ [0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE（IPv4 静态路由）+ [0-00063](task/UDG/20.15.2/0-00063.md) ADD SRROUTE6（IPv6 静态路由）已配置完成。

## 决策点

（无。本命令仅 SWITCH 一个枚举值，无多维分支。）

## 约束

- **高危命令**（critical，rule-0-00330）：改前必须确认 APN 下**无正在使用或准备使用静态路由冗余的用户**，否则此部分用户业务不通 — 后果：高危配置错误致用户业务不通
- **最大记录数为 1**（critical，rule-0-00330）：全局唯一；多次 SET 后值覆盖 — 后果：违反唯一性命令拒收
- **与地址池 REDUNDFUNC 联动**（critical，隐含）：本开关须与 [0-00043](task/UDG/20.15.2/0-00043.md) ADD POOL 的 REDUNDFUNC=ENABLE **同时打开**才生效 — 后果：REDUNDFUNC 未打开时使能本开关不生效
- **主备 UDG 均需配置**（critical，隐含）：备用 UDG 须重复主用的全部步骤（包括本命令）— 后果：主备不一致致切换异常
- **执行后立即生效**（warning，隐含）：无 REFRESHSRV 缓冲 — 后果：配置错误立即影响业务
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
