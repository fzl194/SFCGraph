# 定义IPsec高级ACL规则(IPv4)（ADD ACLRULEADV4IPSEC）

> 命令静态知识见 [ADD ACLRULEADV4IPSEC](command/UDG/20.15.2/ADD-ACLRULEADV4IPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典",**不**逐特性罗列取值。ACLNAME 必须先由 [0-00211](task/UDG/20.15.2/0-00211.md) ADD ACLGROUPIPSEC 建组(数字段 3000~3999);VRFNAME 引用 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 的 IPsec 微服务 VPN;下游 IPsec 安全策略 [0-00215](task/UDG/20.15.2/0-00215.md) 经 ACLNUMBER/ACLNAME 引用本规则组。

## 配置方法(配置方法字典)

ADD ACLRULEADV4IPSEC 是"IPsec 专用高级 IPv4 ACL 规则创建"命令。高级 ACL 基于五元组(源/目的 IP、源/目的端口号、协议号)匹配报文,**IPsec 子场景下 ACLACTION 固定 Permit**(标记感兴趣流),ACLNAME 沿用数字段 3000~3999(高段 IPsec 专用),VRFNAME 引用 IPsec 微服务 VPN。1 个特性命中(IPFD-015004 各激活 md 10002/10006/10008 等子场景)覆盖普通 IPv4 IPsec 隧道、GRE over IPsec、多 Sequence IPsec 策略三种典型场景。

### 配置维度 1:匹配维度(五元组 + 协议类 + 分片类)

| 子维度 | 必选/可选 | 取值 | 联动参数 |
|---|---|---|---|
| **规则组(ACLNAME)** | 必选 | 字符串/数字 1~32,数字段 3000~3999(IPsec 专用高段) | 引 [0-00211](task/UDG/20.15.2/0-00211.md) ACLGROUPIPSEC |
| **规则名(ACLRULENAME)** | 必选 | 字符串 1~32,不支持空格,区分大小写 | — |
| **规则 ID(ACLRULEID)** | 可选 | 整数 0~4294967294(不配则按组步长增长) | — |
| **协议类型(ACLPROTOCOL)** | 必选 | 整数 0~255(IP=0 任意;ICMP=1;IGMP=2;TCP=6;UDP=17) | 协议类决定端口/ICMP/SYN 是否生效 |
| **源 IP(ACLSOURCEIP + ACLSRCWILD)** | 可选 | IPv4 + 反掩码(默认 0.0.0.0 + 255.255.255.255) | 全网规划 |
| **目的 IP(ACLDESTIP + ACLDESTWILD)** | 可选 | IPv4 + 反掩码 | 全网规划 |
| **源端口范围(ACLSRCPORTOP/BEGIN/END)** | ACLPROTOCOL=TCP/UDP 条件可选 | OP=Lt/Eq/Gt/Range/Neq + 整数 0~65535 | OP 决定 BEGIN/END 输入约束(Lt 仅 END,E/Gt/Neq 仅 BEGIN,Range 都必填) |
| **目的端口范围(ACLDESTPORTOP/BEGIN/END)** | 同上 | 同上 | 同上 |

**ACLACTION 维度**(必选):

| 取值 | 作用 | 说明 |
|---|---|---|
| **Permit** | 标记匹配流为"需要 IPsec 保护"(感兴趣流) | **IPsec 子场景下固定 Permit**;Deny 无业务含义(命令允许但与 IPsec 用途不符) |
| Deny | 禁止对匹配条件访问 | 通用 ACL 场景用;IPsec 子场景不推荐 |

**分片类维度**(ACLF RAGTYPE,可选):

| 取值 | 作用 |
|---|---|
| FragmentSubseq | 检查分片报文中的非首片 |
| Fragment | 检查报文是否为分片报文 |
| NonFragment | 检查非分片报文 |
| NonSubseq | 检查分片报文中的首分片或非分片报文 |
| FragmentSpeFirst | 检查分片报文中的首片 |
| 不配 | 不校验报文的分片信息 |

### 配置维度 2:QoS/优先级扩展(可选)

| 参数 | 取值范围 | 约束 |
|---|---|---|
| **ACLPRECEDENCE**(报文优先级) | 整数 0~7 | **不允许和 ACLDSCP 同时配** |
| **ACLTOS**(服务优先级) | 整数 0~15 | **不允许和 ACLDSCP 同时配** |
| **ACLDSCP**(差分服务码) | 整数 0~63 | **不允许和 ACLPRECEDENCE 或 ACLTOS 同时配** |

> **互斥原则**:ACLPRECEDENCE/ACLTOS/ACLDSCP 三者只能选其一,不可混用(IPsec 子场景通常不配,基本 ACL 通用场景按需选)。

### 配置维度 3:协议类扩展(ICMP/SYN,条件可选)

| 参数 | 触发条件 | 取值 | 说明 |
|---|---|---|---|
| **ACLICMPNAME** | ACLPROTOCOL=ICMP(1) | 22 个知名 ICMP 名称(Echo/EchoReply/HostUnreachable 等)或 Custom(自定义) | 配 Custom 时需补 ACLICMPTYPE + ACLICMPCODE |
| **ACLICMPTYPE** | ACLICMPNAME=Custom | 整数 0~255 | 网络控制管理协议报文类型 |
| **ACLICMPCODE** | ACLICMPNAME=Custom | 整数 0~255 | 网络控制管理协议消息码 |
| **ACLSYNFLAG** | ACLPROTOCOL=TCP(6) | 整数 0~63(TCP-Flag) | TCP-Flag 校验 |

> **TTL 维度**(注:命令 wiki 当前版本 ACLTTLOP/ACLTTL/ACLTTLEND 字段无显式参数列表;若场景启用须查命令 wiki 当前版本):Lt 仅 ACLTTLEND 可输入值;Gt/Eq/Neq 仅 ACLTTL 可输入值;Range 两者都必填。

### 配置维度 4:VPN 上下文(VRFNAME,可选)

| 取值形式 | 来源 | 适用场景 |
|---|---|---|
| **字符串 1~31** | 引用 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC:VRFNAME | 限定 IPsec ACL 生效 VPN 实例(IPsec 微服务 VPN,非通用 VPN) |

**典型脚本(IPFD-015004 普通 IPv4 IPsec 隧道,保护子网对子网)**:
```
ADD ACLRULEADV4IPSEC: ACLNAME="3000", ACLRULENAME="rule_1", ACLACTION=Permit, ACLPROTOCOL=0,
                     ACLSOURCEIP="10.102.101.0", ACLSRCWILD="0.0.0.255",
                     ACLDESTIP="10.102.102.0", ACLDESTWILD="0.0.0.255", VRFNAME="vrf1";
```

**典型脚本(IPFD-015004 多 Sequence IPsec 策略,两 Peer 双向流各一条)**:
```
ADD ACLRULEADV4IPSEC: ACLNAME="3001", ACLRULENAME="rule_5", ACLRULEID=5, ACLACTION=Permit,
                     VRFNAME="vrf1", ACLSOURCEIP="10.1.1.2", ACLSRCWILD="0.0.0.0",
                     ACLPROTOCOL=0, ACLDESTIP="10.1.2.2", ACLDESTWILD="0.0.0.0";
ADD ACLRULEADV4IPSEC: ACLNAME="3001", ACLRULENAME="rule_5", ACLRULEID=5, ACLACTION=Permit,
                     VRFNAME="vrf1", ACLSOURCEIP="10.1.2.2", ACLSRCWILD="0.0.0.0",
                     ACLPROTOCOL=0, ACLDESTIP="10.1.1.2", ACLDESTWILD="0.0.0.0";
```

**典型脚本(IPFD-015004 协议类扩展,OSPF over IPsec 用 OSPF 协议号 89)**:
```
ADD ACLRULEADV4IPSEC: ACLNAME="3001", ACLRULENAME="rule_1", ACLRULEID=1, ACLACTION=Permit,
                     VRFNAME="vrf1", ACLPROTOCOL=0;
ADD ACLRULEADV4IPSEC: ACLNAME="3001", ACLRULENAME="rule_2", ACLRULEID=1, ACLACTION=Permit,
                     VRFNAME="vrf1", ACLPROTOCOL=89;
```

> 命令 notes:**最大记录数 65535**(整机 ACL 规则);**执行后立即生效**;**ACL 规则不可含重复或具有包含关系的源/目的地址**;**IKEv1 场景下不支持 narrow down**——一端 ACL 范围被另一端 ACL 范围包含时仍能正常协商 IPSEC SA;端口 OP 选项输入限制(Lt/Eq/Gt/Neq/Range)严格按上表。

**步骤位置**:在所属 feature 编排中位于"IPsec 协商链第 2 步(组内下规则)"——前置 [0-00211](task/UDG/20.15.2/0-00211.md) ACLGROUPIPSEC(建组) + [0-00210](task/UDG/20.15.2/0-00210.md) L3VPNINSTIPSEC(VRFNAME 引用源);下游 [0-00215](task/UDG/20.15.2/0-00215.md) IPSECPOLICY(经 ACLNUMBER/ACLNAME 引用本规则组)。

## 决策点:ACLNAME 范围段选择(DP 0-00298)

问:本规则组 ACLNAME 用数字 3000~3999 高段还是其他段?

| 选项 | 联动 | 适用场景 |
|---|---|---|
| **数字 3000~3999(IPsec 专用高段)** | 下游 IPSECPOLICY 引用 ACLNUMBER | IPsec 子场景默认(IPFD-015004 各激活 md 10002/10006/10008 等子场景) |
| 数字 3000 以下 | 下游 IPSECPOLICY 引用 ACLNUMBER | 与通用 ACL 段冲突,不推荐 IPsec 子场景使用 |
| 命名字符串 1~32 | 下游 IPSECPOLICY 引用 ACLNAME | 命名 IPsec 规则组(等效于数字段) |

> 1 个特性命中(IPFD-015004)覆盖数字 3000/3001/3017 多种组号(多 Sequence IPsec 策略场景分多组);IPsec 子场景下 ACLACTION 固定 Permit。

## 约束

- **最大记录数 65535**(critical,rule-0-00298):整机 ACL 规则最大 65535 条 — 后果:超规格新增失败
- **前置必须先 ADD ACLGROUPIPSEC**(critical,rule-0-00298):ACLNAME 必须在 ACLGROUPIPSEC 已建组下 — 后果:未建组新增规则失败
- **执行后立即生效**(warning,rule-0-00298):执行后立即生效 — 后果:误操作即刻生效
- **ACLACTION 在 IPsec 子场景固定 Permit**(critical,rule-0-00298):IPsec 保护流必须 Permit(Deny 无业务含义) — 后果:配 Deny 致规则无业务含义
- **ACL 规则无重复或包含关系**(critical,rule-0-00298):规则源/目的地址不可重复或具有包含关系 — 后果:违反命令拒收
- **IKEv1 不支持 narrow down**(warning,rule-0-00298):IKEv1 场景下一端 ACL 范围被另一端 ACL 范围包含时仍能正常协商(不报错但不推荐) — 后果:隐式接受但不规范
- **端口 OP 选项输入限制**(warning,rule-0-00298):Lt 仅 END 可输入;E/Gt/Neq 仅 BEGIN 可输入;Range 两者都必填 — 后果:违反命令拒收
- **ACLPRECEDENCE/ACLTOS/ACLDSCP 互斥**(warning,rule-0-00298):三者只能选其一不可混用 — 后果:同时配命令拒收
- **ACLICMPNAME/ACLICMPTYPE/ACLCMPCODE 仅 ICMP 协议生效**(warning,rule-0-00298):仅 ACLPROTOCOL=ICMP(1) 时这三个参数才生效 — 后果:其他协议下配无意义
- **ACLSYNFLAG 仅 TCP 协议生效**(warning,rule-0-00298):仅 ACLPROTOCOL=TCP(6) 时该参数才生效 — 后果:其他协议下配无意义
- **ACLNAME 长度/字符约束**(info,rule-0-00298):字符串 1~32(数字形式 3000~3999),不支持空格,区分大小写 — 后果:超长/含空格命令拒收
- **VRFNAME 引用 IPsec VPN**(info,rule-0-00298):VRFNAME 应引用 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC:VRFNAME — 后果:引用非 IPsec VPN 致流定义与协商不一致
- **VRFNAME 长度**(info,rule-0-00298):字符串 1~31 — 后果:超长命令拒收
- **IPsec 协商链前置**(critical,rule-0-00298):本规则组位于"ACL→Proposal→Peer→Policy"协商链第 2 步,须先于 [0-00212](task/UDG/20.15.2/0-00212.md) IPSECPROPOSALIPSEC / [0-00213](task/UDG/20.15.2/0-00213.md) IKEPROPOSAL / [0-00214](task/UDG/20.15.2/0-00214.md) IKEPEER / [0-00215](task/UDG/20.15.2/0-00215.md) IPSECPOLICY 建 — 后果:未建组下游策略引用失败
