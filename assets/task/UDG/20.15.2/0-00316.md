# 添加一个新的TAC组（ADD TACGROUP）

> 命令静态知识见 [ADD TACGROUP](command/UDG/20.15.2/ADD-TACGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典",**不**逐特性罗列取值。TACGROUPNAME 是 TAC 号段聚合容器,被 [0-00315](task/UDG/20.15.2/0-00315.md) ADD S1TACID(4G) 与 [0-00312](task/UDG/20.15.2/0-00312.md) ADD N2TACID(5G) 共同引用;TACTYPE 决定该组承载 S1TAC 还是 N2TAC 号段。本命令 SOP §2.3 path 2 走法(无汇总 md)——基于原始命令 md + 命令 wiki 兜底构建。

## 配置方法(配置方法字典)

ADD TACGROUP 是"新建 TAC 组"命令。1 个必选参数(TACGROUPNAME)+ 1 个可选参数(TACTYPE,默认 S1TAC)。核心配置空间有 2 个维度:**TAC 组名**(TACGROUPNAME 字符串规划)、**TAC 类型**(TACTYPE=S1TAC/N2TAC 决定容器承载 4G/5G 哪种号段)。命令本身配置空间极简——建组是链路起点,号段绑定由下游号段命令在已建组内完成。

### 配置维度 1:TAC 组名(参数 TACGROUPNAME,必选)

| 取值形式 | 约束 | 引用关系 |
|---|---|---|
| **字符串 1~32** | 不支持空格;不区分大小写;本端规划 | 被 [0-00315](task/UDG/20.15.2/0-00315.md) ADD S1TACID(TACTYPE=S1TAC 时)与 [0-00312](task/UDG/20.15.2/0-00312.md) ADD N2TACID(TACTYPE=N2TAC 时) 的 TACGROUPNAME 引用,作为 TAC 号段聚合容器 |

> **TAC 组定位**:TACGROUPNAME 是 TAC 号段聚合的命名容器,本身不绑定 TAC 号段——号段绑定由 [0-00315](task/UDG/20.15.2/0-00315.md) ADD S1TACID / [0-00312](task/UDG/20.15.2/0-00312.md) ADD N2TACID 在已建 TAC 组内完成。本命令仅做"建组 + 类型声明"动作。

### 配置维度 2:TAC 类型(参数 TACTYPE,可选)

| 取值 | 含义 | 下游号段命令 | 适用场景 |
|---|---|---|---|
| **S1TAC**(默认) | 4G S1 接口 TAC(Tracking Area Code) | [0-00315](task/UDG/20.15.2/0-00315.md) ADD S1TACID(4/6 位十六进制号段) | 4G 位置→地址分配(默认范式,PGW-U) |
| **N2TAC** | 5G N2 接口 TAC | [0-00312](task/UDG/20.15.2/0-00312.md) ADD N2TACID(6/8 位十六进制号段) | 5G N2 位置→地址分配(UPF) |

> **类型与下游号段命令铁律**:TACTYPE 决定本组能承载哪种号段命令——S1TAC 组只能挂 S1TAC 号段,N2TAC 组只能挂 N2TAC 号段;混挂(在 S1TAC 组内 ADD N2TACID 或反之)命令拒收。若不显式传 TACTYPE,默认 S1TAC(4G 范式)。

**典型脚本(命令 wiki 范例,4G S1TAC 组 beijing)**:
```
ADD TACGROUP:TACGROUPNAME="beijing";
```

**典型脚本(5G N2TAC 组,显式传 TACTYPE)**:
```
ADD TACGROUP:TACGROUPNAME="beijing_5g",TACTYPE=N2TAC;
```

> 命令 notes:**最大记录数 6000**;**只对新激活用户生效**(已激活用户不感知);**适用 NF: PGW-U、UPF**(PGW-U 配 S1TAC,UPF 可配 S1TAC/N2TAC);**G_1 管理员/G_2 操作员**。

**步骤位置**:在所属 feature 编排中通常位于"步骤 1:建 TAC 组容器"——前置 **无**(建组是链路起点);下游 [0-00315](task/UDG/20.15.2/0-00315.md) ADD S1TACID / [0-00312](task/UDG/20.15.2/0-00312.md) ADD N2TACID(在已建 TAC 组内按 TACTYPE 绑定对应号段)。

## 决策点:TAC 类型选择(DP 0-00316)

问:本 TAC 组承载 4G S1TAC 号段还是 5G N2TAC 号段?(决定 TACTYPE 取值 + 下游号段命令)

| 选项 | TACTYPE | 下游号段命令 | 号段格式 | 适用场景 |
|---|---|---|---|---|
| 4G S1TAC 组(默认) | S1TAC | [0-00315](task/UDG/20.15.2/0-00315.md) ADD S1TACID | 4/6 位十六进制号段 | 4G 位置→地址分配(PGW-U),默认范式 |
| 5G N2TAC 组 | N2TAC | [0-00312](task/UDG/20.15.2/0-00312.md) ADD N2TACID | 6/8 位十六进制号段 | 5G N2 位置→地址分配(UPF) |

> **类型与下游号段命令铁律**:TACTYPE 决定本组能承载哪种号段命令——S1TAC 组只能挂 S1TAC 号段,N2TAC 组只能挂 N2TAC 号段;混挂(在 S1TAC 组内 ADD N2TACID 或反之)命令拒收。

## 约束

- **最大记录数 6000**(warning,rule-0-00316):系统最大记录数 6000 条 — 后果:超规格新增失败
- **只对新用户生效**(warning,rule-0-00316):该命令执行后只对新激活用户生效 — 后果:已激活用户不感知,需考虑存量用户重激活时机
- **TACTYPE 与下游号段命令对齐**(critical,rule-0-00316):TACTYPE=S1TAC 组只能挂 S1TAC 号段([0-00315](task/UDG/20.15.2/0-00315.md) ADD S1TACID);TACTYPE=N2TAC 组只能挂 N2TAC 号段([0-00312](task/UDG/20.15.2/0-00312.md) ADD N2TACID) — 后果:混挂号段命令拒收
- **TACTYPE 默认值 S1TAC**(info,rule-0-00316):不显式传 TACTYPE 时默认为 S1TAC(4G 范式) — 后果:UPF 配 5G 不显式传 N2TAC 会落入 S1TAC 组被拒
- **TACGROUPNAME 长度 1~32**(info,rule-0-00316):字符串类型,不支持空格,不区分大小写 — 后果:超长或含空格命令拒收
- **适用 NF: PGW-U、UPF**(info,rule-0-00316):本命令适用 PGW-U(S1TAC 范式)与 UPF(S1TAC/N2TAC 范式),非上述节点上下发命令拒收 — 后果:跨网元下发失败
- **权限要求**(info,rule-0-00316):需 G_1 管理员/G_2 操作员级别 — 后果:低权限用户命令拒收
