# MPLS VPN 基础设施骨架

> **MPLS VPN 共享基础设施**：VPN 实例（VRF）+ 地址族使能 + 分标签配置 + PE-CE 接口绑 VPN。被引用于 [2-00043](task/UDG/20.15.2/2-00043.md)（GWFD-020411 MPLS VPN 特性）。

## 配置方法

**统一步骤表**（按 6 步骨架组织）：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 创建 VPN 实例（VRF） | `ADD VPNINSTANCE`（待 Stage 3 建 atom） | NAME="vpn1" + RD="100:1"（8 字节，全局唯一） |
| 2. 配置 VPN Target ERT（发布） | `ADD VPNINSTANCE_ERT`（待 Stage 3 建） | VPNINSTANCE="vpn1" + ERTVALUE="100:1"（32 位扩展团体属性） |
| 3. 配置 VPN Target IRT（接收） | `ADD VPNINSTANCE_IRT`（待 Stage 3 建） | VPNINSTANCE="vpn1" + IRTVALUE="100:1" |
| 4. 地址族使能 | `ADD VPNINSTAF` → [0-00072](task/UDG/20.15.2/0-00072.md) | VRFNAME="vpn1" + AFTYPE=Ipv4uni（IPv6 VPN 为 Ipv6uni） |
| 5. 分标签方式配置（★关键节约） | `MOD VPNINSTANCE_AF_PERINSTANCE` 或 `SET VPNINSTANCE_PERINSTANCE`（待 Stage 3 建） | VPNINSTANCE="vpn1" + AFTYPE=Ipv4uni + LABELMODE=perInstance |
| 6. PE-CE 接口绑 VPN | `ADD IPBINDVPN` → [0-00057](task/UDG/20.15.2/0-00057.md) | IFNAME="GE0/1/0/1" + VRFNAME="vpn1" + IPTYPE=Ipv4 |

**典型脚本（Intranet，伪 MML，待 Stage 3 补全）**：

```
// 步骤1：创建 VPN 实例 vpn1
ADD VPNINSTANCE: NAME="vpn1", RD="100:1";

// 步骤2-3：配置 VPN Target（Intranet 同 RT）
ADD VPNINSTANCE_ERT: VPNINSTANCE="vpn1", ERTVALUE="100:1";
ADD VPNINSTANCE_IRT: VPNINSTANCE="vpn1", IRTVALUE="100:1";

// 步骤4：使能 IPv4 地址族
ADD VPNINSTAF: VRFNAME="vpn1", AFTYPE=Ipv4uni, VRFRD="100:1";

// 步骤5：配置分标签方式为 perInstance（★关键节约 299008 标签）
MOD VPNINSTANCE_AF_PERINSTANCE: VPNINSTANCE="vpn1", AFTYPE=Ipv4uni, LABELMODE=perInstance;

// 步骤6：PE-CE 接口绑到 VPN 实例
ADD IPBINDVPN: IFNAME="GE0/1/0/1", VRFNAME="vpn1", IPTYPE=Ipv4;
```

## 场景差异

| 场景 | VPN Target 配置 | 地址族 | 分标签 | 关键差异 |
|---|---|---|---|---|
| **Intranet** | 同 RT（如 RT=100:1，所有 PE 共享） | IPv4 | perInstance | 最简单，闭合用户群 |
| **Extranet** | 多 RT 匹配（如 Site1+Site2 都 RT=200:1 可访问 Site3 RT=300:1） | IPv4 | perInstance | 多个 RT 列表 |
| **Hub&Spoke** | 双 RT 控制——Spoke-PE: ERT=Spoke/IRT=Hub；Hub-PE: 接收接口 IRT=Spoke / 发布接口 ERT=Hub | IPv4 | perInstance | Hub-PE 双接口 |
| **跨域 OptionA** | 各 VPN 实例独立 RT（ASBR 互视为 CE） | IPv4 | perInstance | ASBR 需为每个 VPN 创建实例 |
| **跨域 OptionB** | 同 Intranet | IPv4 | **perInstance（强制）** | ASBR perNexthop 配套 |
| **MCE** | 同 Intranet（按业务隔离） | IPv4 | perInstance | CE 侧多 VPN 实例 |
| **6VPE** | 同 IPv4 VPN | **Ipv6uni**（IPv6 地址族） | perInstance | IPv4 骨干承 IPv6 VPN 业务 |

## 决策点

**DP 0-00350：组网模型选择**

| 选项 | 影响（参数/命令/联动） |
|---|---|
| Intranet（最简） | 单 RT，闭合用户群，PE-PE VPNv4 对等体；走法 = 骨架 6 步 + MP-BGP PE-PE |
| Extranet | 多 RT 匹配，需 VPN-INSTANCE 多 IRT/ERT 列表；走法 = 骨架 + MP-BGP |
| Hub&Spoke | 双 RT + Hub-PE 双接口 + EBGP AS-Loop 允许本地 AS 重复；走法 = 骨架 + MP-BGP + Hub-PE 双接口配置 |
| 跨域 OptionA | ASBR 互视为 CE，普通 IPv4 EBGP，ASBR 每 VPN 实例独立；走法 = 骨架 + EBGP ASBR-to-ASBR |
| 跨域 OptionB | ASBR MP-EBGP 标签 VPN-IPv4 + perNexthop；走法 = 骨架 + MP-EBGP ASBR-to-ASBR + perNexthop |
| MCE（多实例 CE） | CE 侧多 VPN 实例；走法 = 骨架 + MCE 侧多 VRF 配置 |
| IPv6 VPN（6VPE） | AFTYPE=Ipv6uni，IPv4 骨干；走法 = 骨架 + IPv6 地址族 + VPNv6 对等体 |

## 约束

- **VPN Target RT 匹配（critical，rule-0-01753）**：本端 IRT 需匹配对端 ERT 才注入路由表 — 后果：路由被丢弃
- **Hub&Spoke RT 约束（critical，rule-0-01754）**：任意 Spoke-PE 的 IRT 不与其它 Spoke-PE 的 ERT 相同（保证 Spoke 间不直通） — 后果：Spoke 站点间不直接通信
- **RD 全局唯一（critical，rule-0-01755）**：RD 8 字节，全局唯一，与 IPv4 前缀组合形成 12 字节 VPN-IPv4 地址 — 后果：地址重叠致路由错乱
- **perInstance 强制（critical，rule-0-01756）**：必须配置分标签方式为 perInstance 以节约 299008 标签资源 — 后果：标签资源耗尽
- **地址族匹配（critical，rule-0-01757）**：VPN 实例地址族必须与 PE-CE 路由协议 + MP-BGP 地址族一致（IPv4 场景 AFTYPE=Ipv4uni；IPv6 VPN 为 Ipv6uni） — 后果：路由无法迭代
- **PE-CE 接口手工绑定（critical，rule-0-01758）**：路由协议自身不能区分 VPN 实例，PE-CE 接口绑 VRF 必须手工 — 后果：路由错乱
