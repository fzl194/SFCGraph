# 配置路由策略节点（ADD ROUTEPOLICYNODE）

> 命令静态知识见 [ADD ROUTEPOLICYNODE](command/UDG/20.15.2/ADD-ROUTEPOLICYNODE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD ROUTEPOLICYNODE 在已建路由策略下挂一个节点，定义节点序号与匹配模式（permit/deny）。POLICYNAME 引用上游 [配置路由策略](task/UDG/20.15.2/0-00111.md)；(POLICYNAME,NODESEQUENCE) 构成复合键，供下游 [配置匹配路由类型](task/UDG/20.15.2/0-00113.md) 再引用。

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| POLICYNAME | `policy1` | reference（← 0-00111） | 引用上游 ADD ROUTEPOLICY |
| NODESEQUENCE | `10` | 本端规划 | 整数 0~65535，**同策略下唯一** |
| MATCHMODE | `permit` | 本端规划 | 枚举，permit=放行 / deny=拒绝 |

**典型脚本**（GWFD-020401 IPv6承载上下文）：

```
ADD ROUTEPOLICYNODE:POLICYNAME="policy1",NODESEQUENCE=10,MATCHMODE=permit;
```

**省略字段**：DESCRIPTION（optional 描述，冷门无特性用到，不绑）。

**复合键含义**：节点按 (策略名+节点号) 唯一。一个策略可挂多个节点（不同 NODESEQUENCE），每节点独立 MATCHMODE + 一组匹配条件（路由类型等由 MATCHROUTETYPE 承载）。

## 约束

- **rule-0-00241**（warning）(POLICYNAME,NODESEQUENCE) **同策略下节点号唯一**；**前置**必须先建 ROUTEPOLICY（POLICYNAME 引用）；命令最大记录数 65535；执行后立即生效 — 后果：同策略同节点号重复添加失败 / 未先建 ROUTEPOLICY 致 POLICYNAME 引用悬空
