# 配置IPSQM整形队列深度（SET IPSQMQDEPTH）

> 命令静态知识见 [SET IPSQMQDEPTH](command/UDG/20.15.2/SET-IPSQMQDEPTH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110941（基于基站粒度的 IPSQM）**可选增强**第 3 步——配置整形队列深度（缓存报文个数），缺省可用系统初始值，现网按需调整。适用 NF：SGW-U、UPF。后置于核心整形（ADD IPSQMSHAPING，0-00231）。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| DEPTH | 队列深度（报文个数） | `256` | 本端规划，整数 1~1024（必填） |

**脚本样例**

```
SET IPSQMQDEPTH:DEPTH=256;
```

**推荐算法（命令 notes）**：DEPTH 推荐按 `QUEDEPTH = rate * RTT / pktlen` 估算——rate 为 RATE 转换字节/秒，RTT 依网络估算（秒），pktlen 为报文平均长度（字节）。缺省用系统初始值。

**操作步骤定位**：核心 ADD IPSQMSHAPING 之后、可选 SET IPSQMADJUST（最低保护带宽/突发尺寸）之前。后置依赖由 feature-rule 0-00082 表达。

## 约束

- 命令最大记录数 **1**（命令 notes 强约束），超出被拒。
- DEPTH 推荐按公式 `QUEDEPTH = rate*RTT/pktlen` 估算，算错将导致缓存溢出或资源浪费。
- 命令执行后立即生效。
- DEPTH 取值范围 1~1024。
