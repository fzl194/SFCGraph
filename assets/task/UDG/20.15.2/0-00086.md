# 配置IPFarm全局参数（SET IPFARMGLOBAL）

> 命令静态知识见 [SET IPFARMGLOBAL](command/UDG/20.15.2/SET-IPFARMGLOBAL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET IPFARMGLOBAL 是 IP Farm 基础设施的"整机全局参数"命令：通过对每个 server 的网络状态进行检查（ICMP 心跳），以及配置 IP farm 配置下重定向选 server 的负荷分担方式，保证业务报文能到达可用 server。本命令的核心配置空间有 3 个维度：**服务器类型 SERVERTYPE**（决定命令作用域）、**心跳检测参数**（PCSCF/REDIRECT/IPMS 时可选）、**负荷分担模式 LBMETHOD**（仅 REDIRECT 时可选）。整机最多记录数 **3**（与 SERVERTYPE 取值数量一致），**立即生效**。

### 配置维度 1：服务器类型（参数 SERVERTYPE，必选）

| 取值 | 命令作用域 | LBMETHOD 可选 | 典型场景 |
|---|---|---|---|
| **REDIRECT** | 重定向场景（captive portal / WebProxy） | 是（ROUND_ROBIN/LEAST_LOAD/LEAST_RECENTLY_USED） | [2-00014](task/UDG/20.15.2/2-00014.md) 用户Portal、[2-00015](task/UDG/20.15.2/2-00015.md) Web Proxy |
| **PCSCF** | P-CSCF 连接状态检测 | 否 | [2-00013](task/UDG/20.15.2/2-00013.md) P-CSCF故障时IMS业务恢复 |
| **IPMS** | IPMS Server 连接状态检测 | 否 | 业务保持 |

> 3 种 SERVERTYPE 各自对应一条记录（系统初始 3 条）；同 SERVERTYPE 重复 SET 覆盖前值；不同 SERVERTYPE 共存。

### 配置维度 2：心跳检测参数（参数 TIMETHRESHOLD/HEALTHSUCCLIMIT/HEALTHFAILLIMIT/TIMEOUT，PCSCF/REDIRECT/IPMS 时可选）

| 参数 | 含义 | 取值范围 | 默认 |
|---|---|---|---|
| **TIMETHRESHOLD** | 两次心跳检测间的时间阈值（秒） | 整数 6~60 | 6 |
| **HEALTHSUCCLIMIT** | server 状态置为 up 所需连续心跳成功次数 | 整数 1~10 | 3 |
| **HEALTHFAILLIMIT** | server 状态置为 down 所需连续心跳失败次数 | 整数 1~10 | 3 |
| **TIMEOUT** | 心跳检测超时时长（秒） | 整数 1~59 | 2 |

> **关系约束**：TIMEOUT 必须 < TIMETHRESHOLD；HEALTHSUCCLIMIT/HEALTHFAILLIMIT 决定 server up/down 翻转灵敏度（up 需连续成功 N 次、down 需连续失败 M 次）。

### 配置维度 3：负荷分担模式（参数 LBMETHOD，仅 SERVERTYPE=REDIRECT 时可选）

| 取值 | 策略 | 适用场景 |
|---|---|---|
| **ROUND_ROBIN** | 轮询（跳过状态为 down 的 server） | 标准重定向（用户Portal/WebProxy 通用） |
| **LEAST_LOAD** | 选择最小负荷 server | 业务负荷波动大的场景 |
| **LEAST_RECENTLY_USED** | 选择最久未被使用的 server | 缓存友好场景 |

> **典型范式**：
> ```
> SET IPFARMGLOBAL:SERVERTYPE=PCSCF,HEALTHSUCCLIMIT=4,HEALTHFAILLIMIT=4,TIMETHRESHOLD=6;
> SET IPFARMGLOBAL:SERVERTYPE=REDIRECT,TIMETHRESHOLD=10,HEALTHSUCCLIMIT=2,HEALTHFAILLIMIT=4,LBMETHOD=LEAST_LOAD;
> ```

**步骤位置**：在所属 feature 编排中通常为"步骤 1：配置 IP Farm 全局参数"——位于 ADD IPFARM / ADD IPFARMSERVER 之前。

## 决策点：服务器类型与负荷分担模式（DP 0-00086）

问：SERVERTYPE 取何值、是否配 LBMETHOD？

| 选项 | SERVERTYPE | LBMETHOD | 适用场景 |
|---|---|---|---|
| 重定向（REDIRECT，default） | REDIRECT | 必选（3 选 1） | [2-00014](task/UDG/20.15.2/2-00014.md) 用户Portal、[2-00015](task/UDG/20.15.2/2-00015.md) WebProxy |
| P-CSCF 检测（PCSCF） | PCSCF | 不配 | [2-00013](task/UDG/20.15.2/2-00013.md) P-CSCF故障时IMS业务恢复 |
| IPMS Server 检测（IPMS） | IPMS | 不配 | 业务保持 |

## 约束

- **整机最大记录数为 3**（warning，隐含）：与 SERVERTYPE 取值数对应（REDIRECT/PCSCF/IPMS 各 1 条）；存在系统初始 3 条记录 — 后果：同 SERVERTYPE 重复 SET 仅覆盖；不同 SERVERTYPE 共存
- **立即生效**（info，隐含）：该命令执行后立即生效 — 后果：阈值/负荷分担模式变更实时影响 server 选路
- **TIMEOUT 必须小于 TIMETHRESHOLD**（warning，隐含）：心跳超时时长 < 心跳间隔 — 后果：TIMEOUT ≥ TIMETHRESHOLD 时心跳检测逻辑异常
- **LBMETHOD 仅 SERVERTYPE=REDIRECT 时可选**（warning，隐含）：PCSCF/IPMS 场景 LBMETHOD 不出现 — 后果：其它 SERVERTYPE 配置 LBMETHOD 被拒收
- **LBMETHOD 默认 ROUND_ROBIN（REDIRECT）**（info，隐含）：系统初始 REDIRECT 记录 LBMETHOD=ROUND_ROBIN — 后果：未显式配置按 ROUND_ROBIN 生效
