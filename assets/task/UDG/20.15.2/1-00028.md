# IPv6PD-V6PREFIXLENGTH<64 PD标志

> 归属 [2-00041](task/UDG/20.15.2/2-00041.md) IPv6 Prefix Delegation（GWFD-020406）。本 compound 是**PD 特性专属标志步骤**——通过 ADD SECTION 的 V6PREFIXLENGTH=49~63（<64）触发 PD 方式分配，并附带 CONFLICTIPV6 + IPALLOCRULE + License 开关共 4 条命令。被引用于 {2-00041}。

## 配置方法

本 compound 由 4 条命令组成（多命令可复用模块，建 compound 承载）：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 打开 IPv6 PD 特性 License 配置开关 | [SET LICENSESWITCH](task/UDG/20.15.2/0-00019.md) | `LICITEM="LKV3G5P6PD01"` / `SWITCH=ENABLE` |
| 2. 配置 PD 地址池地址段（★★ PD 标志） | [ADD SECTION](task/UDG/20.15.2/0-00044.md) | `V6PREFIXSTART` / `V6PREFIXEND` / **`V6PREFIXLENGTH=63`（49~63，<64 即 PD）** |
| 3. （可选）添加 PD 冲突前缀 | [ADD CONFLICTIPV6](task/UDG/20.15.2/0-00302.md) | `POOLNAME` / `V6PREFIX` / `V6PREFIXLENGTH=63` |
| 4. 配置地址分配规则（使能 PD 维度的规则字符串） | [SET IPALLOCRULE](task/UDG/20.15.2/0-00250.md) | `FIRSTRULESW=ENABLE` / `FIRSTRULE=APN-{0\|1}&LOCATION-{0\|1}&SMF-{0\|1}` |

> ★ 步骤 2 是本 compound 的**核心步骤**——`V6PREFIXLENGTH<64` 是 PD 方式与普通 IPv6 单栈（V6PREFIXLENGTH=64）的**唯一配置分水岭**。文档原文："前缀长度范围为 49~64，当前缀长度小于 64 时，则表示该地址池采用 IPv6 Prefix Delegation 方式分配"。
>
> 步骤 4 的 FIRSTRULE 规则字符串是**变体维度的配置接口**——5 种激活变体（外部/APN/SMF/SMF+APN/位置）通过不同规则字符串区分，详见 [2-00041](task/UDG/20.15.2/2-00041.md) 决策点表。

**典型脚本（变体2：本地地址池-基于 APN/DNN）**：
```
SET LICENSESWITCH:LICITEM="LKV3G5P6PD01",SWITCH=ENABLE;
// 步骤 2（核心 PD 标志）—— 在 [1-00022](task/UDG/20.15.2/1-00022.md) 地址池体系骨架的 ADD SECTION 中
ADD SECTION:POOLNAME="testpool",SECTIONNUM=1,IPVERSION=IPV6,V6PREFIXSTART="fc00:0000:0000:fcee:0000:0000:0000:0000",V6PREFIXEND="fc00:0000:0000:fcef:ffff:ffff:ffff:ffff", V6PREFIXLENGTH=63;
// 步骤 3（可选）
ADD CONFLICTIPV6:POOLNAME="testpool",V6PREFIX="fc00:0000:0000:fcee:0000:0000:0000:0001", V6PREFIXLENGTH=63;
// 步骤 4
SET IPALLOCRULE: FIRSTRULESW=ENABLE, FIRSTRULE=APN-1&LOCATION-0&SMF-0, SECONDRULESW=DISABLE, THIRDRULESW=DISABLE;
```

**典型脚本（变体5：本地地址池-基于位置，License 改 LKV3G5LBAA01）**：
```
// 变体5 License 实际使用 LKV3G5LBAA01（基于位置的地址分配 License），非 LKV3G5P6PD01
SET LICENSESWITCH:LICITEM="LKV3G5LBAA01",SWITCH=ENABLE;
ADD SECTION:POOLNAME="testpool",SECTIONNUM=1,IPVERSION=IPV6,V6PREFIXSTART="fc00:0000:0000:fcee:0000:0000:0000:0000",V6PREFIXEND="fc00:0000:0000:fcef:ffff:ffff:ffff:ffff", V6PREFIXLENGTH=63;
SET IPALLOCRULE: FIRSTRULESW=ENABLE, FIRSTRULE=APN-0&LOCATION-1&SMF-0, SECONDRULESW=DISABLE, THIRDRULESW=DISABLE;
```

## 场景差异

| 变体 | License | 步骤 2 V6PREFIXLENGTH | 步骤 3 CONFLICTIPV6 | 步骤 4 FIRSTRULE | 是否执行本 compound |
|---|---|---|---|---|---|
| 变体1（外部网元 SMF/RADIUS） | LKV3G5P6PD01 | 必配（=63） | 可选 | 不配（外部分配无规则） | **执行** |
| 变体2（本地-基于 APN/DNN） | LKV3G5P6PD01 | 必配（=63） | 可选 | `APN-1&LOCATION-0&SMF-0` | **执行** |
| 变体3（本地-基于 SMF） | LKV3G5P6PD01 | 必配（=63） | 可选 | `APN-0&LOCATION-0&SMF-1` | **执行** |
| 变体4（本地-基于 SMF+APN） | LKV3G5P6PD01 | 必配（=63） | 可选 | `APN-1&LOCATION-0&SMF-1` + `SECONDRULE=APN-0&LOCATION-0&SMF-1` | **执行** |
| 变体5（本地-基于位置） | **LKV3G5LBAA01**（★变体5 License 差异） | 必配（=63） | 可选 | `APN-0&LOCATION-1&SMF-0` | **执行**（License 替换 + 规则改 LOCATION 维度） |

> ★ **变体5 License 不一致说明**：变体5 激活文档实际使用 `SET LICENSESWITCH:LICITEM="LKV3G5LBAA01"`（GWFD-020421 基于位置的地址分配 License），非 LKV3G5P6PD01（PD 特性 License）。这是产品文档的混合场景设计——变体5 既是 PD 又是基于位置。feature 编排时**必须按变体5 实际 License 配**，勿按 LKV3G5P6PD01 盲目套用。

## 决策点：PD 标志与变体维度（DP 0-00347）

问：V6PREFIXLENGTH 与变体如何组合？

| 选项 | V6PREFIXLENGTH 取值 | License | FIRSTRULE | 适用变体 |
|---|---|---|---|---|
| 外部网元 | 63（=49~63 任意） | LKV3G5P6PD01 | 不配 | 变体1 |
| 本地-APN | 63 | LKV3G5P6PD01 | `APN-1&LOCATION-0&SMF-0` | 变体2 |
| 本地-SMF | 63 | LKV3G5P6PD01 | `APN-0&LOCATION-0&SMF-1` | 变体3 |
| 本地-SMF+APN | 63 | LKV3G5P6PD01 | `APN-1&LOCATION-0&SMF-1` | 变体4 |
| 本地-位置 | 63 | **LKV3G5LBAA01** | `APN-0&LOCATION-1&SMF-0` | 变体5 |

## 约束

- **V6PREFIXLENGTH 范围 49~63**（critical，rule-0-00521）：PD 标志的前缀长度必须 49~63（<64）；=64 为普通 IPv6 单栈，非 PD — 后果：=64 致 PD 流程不触发，无 delegated-prefix 下发
- **本地池同 APN 前缀长度一致**（critical，rule-0-00522）：本地地址池方式分配时，同一 APN 下的 IPv6 delegated-prefix 长度必须相同 — 后果：长度不一致致用户激活失败
- **外部分配同 APN 前缀长度可异**（info，rule-0-00523）：外部网元（SMF/RADIUS）分配时，同一 APN 下的前缀长度可以不同 — 后果：正常变体，无影响
- **License 必配**（critical，rule-0-00524）：变体1-4 必须加载 LKV3G5P6PD01 + SET LICENSESWITCH；变体5 必须加载 LKV3G5LBAA01 + SET LICENSESWITCH — 后果：License 未使能致特性不工作
- **V6PREFIXLENGTH 与 CONFLICTIPV6 一致**（warning，rule-0-00525）：CONFLICTIPV6 的 V6PREFIXLENGTH 必须与对应 POOL 的 V6PREFIXLENGTH 一致 — 后果：长度不一致致冲突检测失效
- **DHCP Server 不支持**（info，rule-0-00526）：IPv6 PD 功能不支持从 DHCP Server 获取并分配 IPv6 delegated-prefix — 后果：依赖外部 DHCP 不可行
- **单 UE 单 delegated-prefix**（info，rule-0-00527）：DHCPv6 Solicit 多 IA_PD option 时，仅第一个分配，其余返回 NoPrefixAvail — 后果：单 UE 多网段申请仅首个成功
