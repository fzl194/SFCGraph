# 配置地址池组（ADD POOLGROUP）
> 命令静态知识见 [ADD POOLGROUP](command/UDG/20.15.2/ADD-POOLGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

命名一个地址池组对象，作为地址池与 APN 映射的中间聚合锚点（POOLGRPNAME）——后续 ADD POOLBINDGROUP 把地址池绑进组、ADD POOLGRPMAP 把组映射到 APN。双栈/优先级场景再使能 IPv4/IPv6 地址池优先级分配算法。

| 参数 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| POOLGRPNAME（地址池组名） | poolgroup1 / poolgroup2 / poolgroup3 | 本端规划 | 1~79 字符串，禁空格及 # $ & 等特殊字符 |
| IPV4ALLOCPRIALG（IPv4 优先级分配算法） | DISABLE（默认）/ ENABLE | 本端规划 | 双栈/静态路由冗余/优先级功能场景置 ENABLE |
| IPV6ALLOCPRIALG（IPv6 优先级分配算法） | DISABLE（默认）/ ENABLE | 本端规划 | 双栈/优先级功能场景置 ENABLE |

典型脚本样例（按是否启用优先级算法）：
```
# 基础（单栈地址分配，GWFD-010104/020406/020421）
ADD POOLGROUP:POOLGRPNAME="poolgroup1";

# IPv4 静态地址路由冗余（GWFD-010107）
ADD POOLGROUP:POOLGRPNAME="poolgroup1", IPV4ALLOCPRIALG=ENABLE;

# IPv4v6 双栈（GWFD-020403/020406/020482）
ADD POOLGROUP:POOLGRPNAME="poolgroup1", IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE;
```

配置要点：双 ALG 默认 DISABLE；只在用到地址池优先级功能（双栈、静态路由冗余、入不转板）时置 ENABLE，置位后须配套建多个地址池（不同优先级）并 ADD POOLBINDGROUP 绑入。POOLGRPNAME 为下游 [0-00045](task/UDG/20.15.2/0-00045.md)（POOLBINDGROUP）、[0-00047](task/UDG/20.15.2/0-00047.md)（POOLGRPMAP）的引用源。注意：命令执行后只对新激活用户生效，已激活用户需重激活/刷新才命中新配置。
