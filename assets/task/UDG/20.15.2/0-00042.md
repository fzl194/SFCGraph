# 配置地址池组（ADD POOLGROUP）

> 命令静态知识见 [ADD POOLGROUP](command/UDG/20.15.2/ADD-POOLGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD POOLGROUP 是会话管理的"地址池组定义层"：把若干个 IPv4/IPv6 地址池按"组"组织，产出 POOLGRPNAME 供 ADD POOLBINDGROUP(0-00045) 引用做 APN 与地址池组的绑定。命令核心配置空间有 3 个维度：**地址池组命名**（POOLGRPNAME 必选 + 唯一）、**IPv4 优先级分配算法**（IPV4ALLOCPRIALG 可选）、**IPv6 优先级分配算法**（IPV6ALLOCPRIALG 可选）。两个算法开关完全独立，可单开/双开/全闭。7 个特性命中覆盖地址分配、IPv4v6 双栈、IPv6 PD、入不转板等多个场景，普遍在双栈/PD 场景开启两个算法。

### 配置维度 1：地址池组命名（参数 POOLGRPNAME，必选 + 全网唯一）

| 参数 | 取值范围 | 备注 |
|---|---|---|
| **POOLGRPNAME**（地址池组名称） | 字符串 1~79，不支持空格及某些特殊字符 | 本端规划 / 全网唯一；引用键 |

### 配置维度 2：IPv4 优先级分配算法（参数 IPV4ALLOCPRIALG，可选）

| 取值 | 作用 |
|---|---|
| **ENABLE** | 开启 IPv4 按地址池优先级算法分配地址（从该地址池组中选取 IPv4 地址时按优先级） |
| **DISABLE**（默认） | 关闭 IPv4 优先级算法，按地址池加入顺序/容量分配 |

### 配置维度 3：IPv6 优先级分配算法（参数 IPV6ALLOCPRIALG，可选）

| 取值 | 作用 |
|---|---|
| **ENABLE** | 开启 IPv6 按地址池优先级算法分配地址（从该地址池组中选取 IPv6 地址时按优先级） |
| **DISABLE**（默认） | 关闭 IPv6 优先级算法 |

**典型脚本（010104 地址分配方式，最小配置）**：
```
ADD POOLGROUP: POOLGRPNAME="poolgroup1";
```

**典型脚本（010105 SMF 分配地址 + 双算法开启）**：
```
ADD POOLGROUP: POOLGRPNAME="poolgroup2", IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE;
```

**典型脚本（010107 静态地址用户路由冗余，仅 IPv4 算法）**：
```
ADD POOLGROUP: POOLGRPNAME="poolgroup1", IPV4ALLOCPRIALG=ENABLE;
```

**典型脚本（020403 IPv4v6 双栈接入，双算法开启）**：
```
ADD POOLGROUP: POOLGRPNAME="poolgroup1", IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE;
```

**典型脚本（020482 入不转板功能，双算法开启）**：
```
ADD POOLGROUP: POOLGRPNAME="poolgroup1", IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE;
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；IPV4ALLOCPRIALG/IPV6ALLOCPRIALG 独立可配；最大记录数未明示。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：地址池组定义"——前置无；产出 POOLGRPNAME 被 ADD POOLBINDGROUP(0-00045) 引用做 APN 与地址池组的绑定。

## 决策点：双栈算法开关组合（DP 0-00042）

问：ADD POOLGROUP 选哪个 IPV4/IPV6 算法开关组合？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅 IPv4 优先 | IPV4ALLOCPRIALG=ENABLE | 静态地址用户路由冗余（010107 范式） |
| 仅 IPv6 优先 | IPV6ALLOCPRIALG=ENABLE | 命令参数齐全（无特性演示） |
| 双栈双开 | IPV4ALLOCPRIALG=ENABLE + IPV6ALLOCPRIALG=ENABLE | 双栈接入（020403）/ SMF 分配（010105）/ IPv6 PD（020406）/ 入不转板（020482） |
| 双栈全闭 | 无 | 仅做组定义，按容量/顺序分配（010104 范式） |

> 选项可叠加：同一系统可建多个不同算法组合的 POOLGROUP 供不同 APN/位置使用。

## 约束

- **POOLGRPNAME 全网唯一**（critical，rule-0-00342）：不同地址池组之间 POOLGRPNAME 不能重复 — 后果：重名 ADD 失败
- **POOLBINDGROUP 前置依赖**（critical，rule-0-00342）：POOLGRPNAME 被 [0-00045](task/UDG/20.15.2/0-00045.md) 引用前须先建 — 后果：未建引用失败
- **算法开关完全独立**（warning，rule-0-00342）：IPV4ALLOCPRIALG 与 IPV6ALLOCPRIALG 互不依赖，可任一独立开启 — 后果：单开 IPv4 不影响 IPv6 行为
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
- **算法优先级与地址池自身优先级配合**（warning，隐含）：开启 IPV*ALLOCPRIALG 时地址池自身需配优先级字段才生效 — 后果：未配优先级的地址池仍按顺序分配
