# HTTPS头增强（GWFD-110263）

> 特性静态知识见 [GWFD-110263](feature/UDG/20.15.2/GWFD-110263.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性是**头增强协议族（HTTP/HTTPS/RTSP/NSH）的 HTTPS 协议变种**，复用族范本 [2-00010](task/UDG/20.15.2/2-00010.md)（HTTP头增强）的 compound 骨架，差异集中在协议命令（ADD TLSHEADEN）+ TLS Extension 字段 TLV 格式 + 加密算法收敛 + HTTPS 独有依赖。

## 配置概览

HTTPS 头增强将用户信息（MSISDN/IMSI/APN/MSIP 等）以 **TLS 协议 TLV（Type-Length-Value）格式**插入 HTTPS **Client Hello 报文头的 Extension 字段**（TLS 握手阶段），传给 Web Server。**核心机制与 HTTP 头增强一致：RULE 经 `POLICYTYPE=HEADEN` 直接引用 HEADEN 对象，独立于 PCC/SMARTREDIRECT 的第 3 种 POLICYTYPE，不经 PCCPOLICYGRP**。

**与 HTTP 头增强（[2-00010](task/UDG/20.15.2/2-00010.md)）的本质差异**：HTTPS 报文是 TLS 加密的，HTTP 头增强（ADD HEADEN）无法直接处理；HTTPS 头增强需先 SA 解密识别（依赖 HTTP2.0 Host 识别），再用 **ADD TLSHEADEN** 在 TLS Client Hello Extension 字段按 TLV 格式插入字段——**没有 PREFIXNAME 参数**（字段名不进报文，靠 TLSTYPEVAL/SUBTLSTYPEVAL 在 Extension 内 TLV 编码）。

头增强与计费可绑定同一 UserProfile（activation 演示双 RULE：HEADEN 规则 + PCC 计费规则共用 FLOWFILTER）。多场景差异由 DP 驱动（触发方式/插入字段/加密/编码/防欺诈/灰名单）。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开 HTTPS 头增强 License 开关，单命令）：`SET LICENSESWITCH`（LKV3G5HTSE01）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **配置流过滤器**（流匹配容器，单命令）：`ADD FLOWFILTER`（FLOWFILTERNAME）→ [0-00007](task/UDG/20.15.2/0-00007.md)
3. **配置过滤条件 + HTTPS 协议识别**（按触发方式选，三四层/七层/协议三选一）：
   - 过滤条件：`ADD FILTER`+`ADD FLTBINDFLOWF` 或 `ADD L7FILTER`+`ADD PROTBINDFLOWF`（**PROTOCOLNAME=https**）+ `SET REFRESHSRV` → [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链 compound，仅含 FILTER/FILTERIPV6/L7FILTER/FLOWFILTER/FLTBINDFLOWF/PROTBINDFLOWF）
   - 协议识别（直引 atom，**非 1-00009 compound 成员**）：`ADD WELLKNOWNPORT`（**端口=443，PROTOCOLNAME=https**）→ [0-00039](task/UDG/20.15.2/0-00039.md) + `LOD SIGNATUREDB` → [0-00027](task/UDG/20.15.2/0-00027.md) + `LOD PARSERDB` → [0-00152](task/UDG/20.15.2/0-00152.md)
   - 基于特定 **SNI**（HTTPS 主流，activation 演示）：`ADD L7FILTER`（URL 实为 SNI，如 www.huawei.com/*）+ `ADD PROTBINDFLOWF`（PROTOCOLNAME=https, L7FILTERNAME）
   - 基于特定 IP：`ADD FILTER`（SVRIP/端口=443）+ `ADD FLTBINDFLOWF`（**不配协议**，同时影响 HTTP/RTSP/HTTPS）
   - 基于 HTTPS 协议：`ADD PROTBINDFLOWF`（PROTOCOLNAME=https，**不配 L7FILTER**）
4. **配置 HTTPS 头增强对象**（HTTPS 头增强核心，**ADD TLSHEADEN** 多字段 TLV 插入 + 加密 + 防欺诈）：`ADD TLSHEADEN`（HEADERENNAME+DATATYPE+**TLSTYPEVAL+SUBTLSTYPEVAL**+ENCRYALGORI+ANTIFRAUD+GRAYLIST，**多字段多次执行**）→ [1-00013](task/UDG/20.15.2/1-00013.md)（HTTPS 场景走 [0-00292](task/UDG/20.15.2/0-00292.md) ADD TLSHEADEN，非 0-00291 ADD HEADEN）
5. **配置计费属性和策略**（因头增强仅针对上行报文，需上行计费；可选）：`ADD URR`+`ADD URRGROUP`+`ADD PCCPOLICYGRP` → [1-00010](task/UDG/20.15.2/1-00010.md)
6. **配置头增强规则**（POLICYTYPE=HEADEN 触发，规则绑定收尾）：`ADD RULE`（POLICYTYPE=HEADEN + POLICYNAME=HEADERENNAME）+ `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
   - 计费规则（可选，与头增强规则共用同一 FLOWFILTER）：`ADD RULE`（POLICYTYPE=PCC + POLICYNAME=PCCPOLICYGRPNM）

## 决策点

**DP（特性级）：触发方式 + 插入字段类型 + TLS Type 规划 + 加密算法 + 编码方式 + 防欺诈 + 灰名单**（影响过滤链走法 + ADD TLSHEADEN 参数）。维度按 activation §表4/§表5 + 原理文档 §触发条件 + §安全机制 全列。

| 场景 | 触发方式 | 过滤链走法 | 插入字段示例 | TLS Type 规划 | 加密 | 防欺诈 | 灰名单 |
|---|---|---|---|---|---|---|---|
| **基于 SNI 触发（HTTPS 主流，activation 演示）** | 基于 SNI | L7FILTER（URL=SNI）+ PROTBINDFLOWF（**https**+L7FILTER）| APN/IMSI/MSIP/MSISDN | 同 TLSTYPEVAL=1000 + 不同 SUB=10/20/30/40 | NONE | DISABLE | DISABLE |
| 基于特定 IP 触发 | 基于特定 IP | FILTER（SVRIP+端口=**443**，不配协议）+ FLTBINDFLOWF | 任意 | 任意 | 任意 | 任意 | 任意 |
| 基于 HTTPS 协议触发 | 基于 HTTPS 协议 | PROTBINDFLOWF（**https**，无 L7FILTER）| 任意 | 任意 | 任意 | 任意 | 任意 |
| 加密变体（安全推荐）| 任意 | 同上 | 任意 | 任意 | **AES128_GCM/AES256_GCM**（+PSWDKEY+SET AES，**无 RSA**）| DISABLE | DISABLE |
| 加密变体（哈希）| 任意 | 同上 | 任意 | 任意 | **SHA256**（+PSWDKEY+SALTSW，RANDNUM 可作盐）| DISABLE | DISABLE |
| **十六进制编码变体（HTTPS 独有）** | 任意 | 同上 | MSISDN/IMSI/SGSNIP/IMEI/MSIP/GGSNIP/TIMESTAMP | 任意 | **NONE + 软参 Byte837 Bit5**（HTTPS 独有；证据：软参文档 `BYTE837 控制HTTPS头增强相关功能_08922628.md`，`SET SOFTPARA:DT=BYTE,BYTENUM=837,BYTEVALUE=16`）| DISABLE | DISABLE |
| 防欺诈变体（HTTPS 支持，与 HTTP 一致）| 任意 | 同上 | MSISDN/IMSI 等 | 任意 | AES128_GCM（命令 wiki 实例）| **ENABLE** | DISABLE |
| 灰名单变体（只防欺诈不插入）| 任意 | 同上 | MSISDN 等 | 任意 | NONE | **ENABLE** | **ENABLE** |

**联动**：
- 触发方式 → [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链走法（L7FILTER(SNI)+PROTBINDFLOWF(https) / FILTER+FLTBINDFLOWF / 仅 PROTBINDFLOWF(https)）
- 协议识别 → WELLKNOWNPORT 端口=**443** + PROTOCOLNAME=**https**（[0-00039](task/UDG/20.15.2/0-00039.md)，关键差异）；SNI 由 HTTP2.0 Host 识别（GWFD-110201）解析
- 协议命令变种 → HTTPS 用 **ADD TLSHEADEN**（[0-00292](task/UDG/20.15.2/0-00292.md)，TLV 格式，无 PREFIXNAME）；HTTP/RTSP 用 ADD HEADEN（[0-00291](task/UDG/20.15.2/0-00291.md)，`<前缀: 值>` 文本格式）
- 加密算法 → [0-00292](task/UDG/20.15.2/0-00292.md) 决策点（**HTTPS 不支持 RSA**，仅 10 种；联动 SET AES 加密配置命令）
- 编码方式 → [0-00292](task/UDG/20.15.2/0-00292.md) 决策点（INHERIT/BASE64/ASCII + **HTTPS 独有 Byte837 十六进制**；Byte837 证据：[GWFD-110263](feature/UDG/20.15.2/GWFD-110263.md) §安全机制 + 软参文档 `BYTE837 控制HTTPS头增强相关功能_08922628.md`）
- 防欺诈 → [1-00013](task/UDG/20.15.2/1-00013.md) 场景差异 §4（**HTTPS 支持**，内嵌 ADD TLSHEADEN 的 ANTIFRAUD 参数，与 HTTP 一致；RTSP 不支持）

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5HTSE01`（HTTPS 头增强 License，与 HTTP 头增强 LKV3G5HTHE01 不同）— 未开则 HTTPS 头增强不生效。
- **强依赖 SA-Basic**（critical）：GWFD-110101（License LKV3G5SABS01）—— HTTPS 报文解析基础，未开则无法识别报文。
- **协议依赖 SA-Web Browsing**（critical）：GWFD-110103（License LKV3G5SAWB01）—— HTTPS 解析必需；HTTPS 承载业务（如 MMS）还需 SA-Mobile（GWFD-110105）。
- **HTTPS 独有强依赖 HTTP2.0 Host 识别**（critical）：GWFD-110201（License LKV3G5HSHA01）—— 解析 HTTPS 的 SNI/证书必须开启，未开则无法解析 HTTPS（**HTTP/RTSP 头增强无此依赖，HTTPS 独有**）。
- **POLICYTYPE=HEADEN 触发条件**（critical）：RULE.POLICYTYPE=HEADEN + POLICYNAME=HEADERENNAME（直接引用，PCC 策略组名称=NULL）— 指错则头增强不触发。
- **必须用 ADD TLSHEADEN（不可用 ADD HEADEN）**（critical）：HTTPS 头增强必须用 ADD TLSHEADEN（含 TLSTYPEVAL 参数，TLV 格式插入 TLS Extension）；ADD HEADEN 只处理 HTTP/RTSP — 混用则 HTTPS 报文不做头增强。
- **仅支持 TLS1.0/1.1/1.2/1.3**（warning）：HTTPS 头增强只支持 TLS1.0/1.1/1.2/1.3 协议（遵循 IETF 5246 TLS 1.2）— 其他 TLS 版本报文不做头增强。
- **Client Hello 报文分片不支持**（warning）：HTTPS 的 Client Hello 报文分片场景下不支持头增强功能（对应 HTTP 的"HTTP 头域分片"限制）— 后果：分片报文不做头增强。
- **基于 IP 触发同时影响 HTTP/RTSP/HTTPS**（warning）：不区分协议，对用户的 HTTP/RTSP/HTTPS 报文都做头增强（设计如此）— 后果：HTTP/RTSP 报文意外被头增强；改用基于 SNI 触发可规避。
- **整机规格**（warning）：≤100 HTTPS 头增强动作；≤500 TLSHeadEn 前缀字段；≤32 TLS Type 启防欺诈；单报文 ≤16 前缀开防欺诈；新增防欺诈项 30 秒内生效；头增强仅针对上行报文（计费属性须含上行）。
- **所有报文走 SA 性能影响**（warning）：启用 HTTPS 头增强后匹配用户的所有报文都需 SA 处理，影响 SA 处理性能和报文转发吞吐 — 后果：系统性能下降。
- **IPv6 分片风险**（warning）：头增强字段过长致报文超 PMTU 将导致 IPv6 业务无法访问；建议 TCP MSS ≤ PMTU - 60B - 头增强字段长度。
- **外置 PCEF 计费误差**（warning）：头增强改变报文长度，外置 PCEF 计费有误差；建议头增强与计费部署在同一网元，或内置 PCEF 计费 + 外置 PCEF 部署头增强。
- **不支持 MULTIPARA 拼接**（info）：HTTPS 头增强不支持 MULTIPARA1/2/3 多参数拼接（HTTP/RTSP 20.10.0+ 支持）— 后果：需拼接场景改用 HTTP 头增强。
- **不支持 RSA 加密**（info）：HTTPS 头增强 ENCRYALGORI 不支持 RSA*（HTTP/RTSP 支持）— 后果：需 RSA 加密改用 HTTP 头增强。
- **法律声明**（info）：HTTPS 头增强在 HTTPS 报文头插入用户标识（IMSI/MSISDN/IP）有泄露个人身份信息风险，须遵循当地法律启用。
- **计费与话单**（info）：先计费再头增强（与规则优先级无关），头增强插入信息不计入业务量计费统计。
