# 下行路由发布（OSPF引入用户路由）

> 归属 [2-00034](task/UDG/20.15.2/2-00034.md) 用户面地址分配（GWFD-010105）。本 compound 配置**下行路由发布层**——通过 OSPF 进程/区域/网络/引入用户路由配置，使本地地址池生成的用户下行子段路由发布到骨干网。被引用于 {2-00034}。

## 配置方法

本 compound 由 4 条命令组成（多命令可复用模块，建 compound 承载）：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 创建 OSPF 进程（IPv4） | [ADD OSPF](task/UDG/20.15.2/0-00074.md) | `PROCID` / `VRFNAME` / `SCHEMAROUID`（Router ID，全网唯一）/ `LSAARRMAXINTV` / `LSAARRSTARINTV` / `LSAARRHLDINTV` |
| 2. 创建 OSPF 区域 | [ADD OSPFAREA](task/UDG/20.15.2/0-00075.md) | `PROCID` / `AREAID` |
| 3. 指定 OSPF 运行的接口网段 | [ADD OSPFNETWORK](task/UDG/20.15.2/0-00076.md) | `PROCID` / `AREAID` / `IPADDRESS` / `WILDCARDMASK` |
| 4. 配置 OSPF 引入外部路由（PROTOCOL=wlr 引入用户路由） | [ADD OSPFIMPORTROUTE](task/UDG/20.15.2/0-00077.md) | `PROCID` / `TOPOID` / `PROTOCOL=wlr` |

**典型脚本（010105 场景一：基于 APN/DNN，下行路由发布到骨干网）**：
```
ADD OSPF: PROCID=100, VRFNAME="vpn1", SCHEMAROUID="10.10.10.1", LSAARRMAXINTV=1000, LSAARRSTARINTV=500, LSAARRHLDINTV=500;
ADD OSPFAREA:PROCID=100,AREAID="0.0.0.0";
ADD OSPFNETWORK:PROCID=100,AREAID="0.0.0.0", IPADDRESS="10.13.21.0", WILDCARDMASK="0.0.0.255";
ADD OSPFIMPORTROUTE:PROCID=100,TOPOID=0,PROTOCOL=wlr;
```

## 场景差异

| 场景 | OSPF 命令集 | 备注 |
|---|---|---|
| 基于 APN/DNN（场景一） | 4 条全配 | 010105 典型 |
| 基于 SMF（场景二） | 同上 | 010105 范式 |
| 基于 SMF+APN（场景三） | 同上 | 010105 范式 |
| 基于 RADIUS（场景四） | 同上 | 010105 范式 |
| 基于位置（参见 GWFD-020421） | 同上 | 020421 范式 |

> 5 个子方式共享同一套 OSPF 路由发布配置（VRFNAME 与 APN 绑定的 VPN 一致）。

## 决策点

（无。本 compound 在所有场景下命令集一致，无分支。）

## 约束

- **ROUTER ID 全网唯一**（critical，rule-0-00496）：OSPF Router ID（SCHEMAROUID）必须全网唯一 — 后果：重复 Router ID 致 OSPF 邻接关系建立失败
- **PROTOCOL=wlr 必须**（critical，rule-0-00496）：OSPFIMPORTROUTE 的 PROTOCOL 必须为 `wlr`（用户路由） — 后果：未配 wlr 致下行路由不发布
- **OSPFNETWORK 必须配运行网段**（warning，rule-0-00496）：ADD OSPFNETWORK 必须配外联口所在网段，否则 OSPF 不通过该接口建邻接 — 后果：邻接失败致路由无法发布
- **VRFNAME 与 APN VPN 一致**（critical，rule-0-00496）：OSPF 进程的 VRFNAME 与 APN/地址池绑定的 VPN 必须一致 — 后果：VPN 不一致致下行路由不在目标 VRF 中
- **PROCID 唯一**（warning，rule-0-00496）：同一 VRF 内 PROCID 必须唯一 — 后果：重复 PROCID 致 OSPF 配置冲突
