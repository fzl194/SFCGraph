# 启动PDN侧路由探测（STR PDNROUTETST）

> 命令静态知识见 [STR PDNROUTETST](command/UDG/20.15.2/STR-PDNROUTETST.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**这是运行态探测命令**（STR=Start），不是配置态 ADD/SET。

## 配置方法（配置方法字典）

STR PDNROUTETST 触发**一次性** PDN/DN 侧路由连通性探测，控制 UPF 向 PDN Server 发送探测消息，检查路由可达性。命令核心配置空间有 8 个维度：**地址池引用**（IPPOOL 必选 + 必须已建）、**IP 版本**（IPVERSION 决定 V4* 或 V6PREFIX* 参数组）、**探测方式**（TESTMETHOD 三选一：PING/DNS/TRACERT 决定必选参数集）、**探测源**（SRCIPV4ADDR 可选，未配时 UDG 随机选）、**探测目的**（DSTIPV4ADDR 必选 + DOMAINVALUE 仅 DNS）、**全量探测**（WHOLEDETECT 仅 PING/DNS 支持，TRACERT 不支持）、**报文净荷**（LENGTH 仅 PING 可选）、**报文 QoS**（DSCPV 仅 IPv4，TCV 仅 IPv6）。1 个特性命中（GWFD-010108）覆盖 PING/DNS 全量+PING 单点+DNS 单点+TRACERT 5 个脚本。

### 配置维度 1：地址池引用（参数 IPPOOL，必选）

| 参数 | 范围 | 备注 |
|---|---|---|
| **IPPOOL** | 字符串 1~79 | 引用已配的 [ADD POOL](task/UDG/20.15.2/0-00043.md) POOLNAME；前置依赖：POOL 必须先建 |

### 配置维度 2：IP 版本（参数 IPVERSION，必选）

| 取值 | 配套必选地址参数 | 适用场景 |
|---|---|---|
| **IPV4** | DSTIPV4ADDR | IPv4 探测（010108 范式仅 IPv4） |
| **IPV6** | DSTIPV6PREFIX | IPv6 探测（命令支持，010108 未演示） |

### 配置维度 3：探测方式（参数 TESTMETHOD，可选）

| 取值 | 必带参数 | 排除参数 | 全量探测支持 | 典型场景 |
|---|---|---|---|---|
| **PING**（默认） | DSTIPV4ADDR | DOMAINVALUE | 支持 WHOLEDETECT | 通用连通性探测 |
| **DNS** | DSTIPV4ADDR（DNS Server IP） + DOMAINVALUE | — | 支持 WHOLEDETECT | 网络中存在防火墙禁 PING 报文时 |
| **TRACERT** | DSTIPV4ADDR | DOMAINVALUE | **不支持** WHOLEDETECT | 故障定位，查看网络故障位置（约 450s 返回） |

### 配置维度 4：探测源（参数 SRCIPV4ADDR/SRCIPV6PREFIX，可选）

| 取值 | 行为 |
|---|---|
| 指定空闲 IP | 探测源 IP 用此参数（**建议**——避免在线用户被去活） |
| 不配 + PING/DNS | UDG 从每个路由网段随机选一个 IP 作探测源 |
| 不配 + TRACERT | UDG 随机选一个地址网段的 IP，不遍历所有网段 |

### 配置维度 5：探测目的（参数 DSTIPV4ADDR/DSTIPV6PREFIX/DOMAINVALUE，必选/条件必选）

| 参数 | 必选性 | 备注 |
|---|---|---|
| **DSTIPV4ADDR** | IPV4 时必选 | 探测目的 PDN Server 的 IPv4 地址（PING/DNS/TRACERT 均必选） |
| **DSTIPV6PREFIX** | IPV6 时必选 | 探测目的 PDN Server 的 IPv6 前缀 |
| **DOMAINVALUE** | DNS 时必选 | 待解析域名（如 www.example.com） |

### 配置维度 6：全量探测（参数 WHOLEDETECT/SECTIONNUM，PING/DNS 时可选）

| WHOLEDETECT | SECTIONNUM | 行为 |
|---|---|---|
| ENABLE | 0~63 | 针对此地址段下所有路由网段触发探测 |
| ENABLE | 不配 | 针对 POOL 内所有地址段下所有路由网段触发探测 |
| 不配 / DISABLE | — | 仅探测单点 |

> TRACERT 不支持 WHOLEDETECT（命令条件约束）。

### 配置维度 7：报文净荷（参数 LENGTH，PING 时可选）

| 参数 | 范围 | 作用 |
|---|---|---|
| **LENGTH** | 整数 | PING 探测报文净荷长度（字节） |

### 配置维度 8：报文 QoS（参数 DSCPV/TCV，按 IPVERSION 条件可选）

| 参数 | IPVERSION | 作用 |
|---|---|---|
| **DSCPV** | IPV4 | IPv4 探测报文的 DSCP 值 |
| **TCV** | IPV6 | IPv6 探测报文的 traffic-class 值 |

**典型脚本（010108 PING 全量探测指定地址段 0）**：
```
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=PING, WHOLEDETECT=ENABLE, SECTIONNUM=0;
```

**典型脚本（010108 PING 非全量指定源 IP）**：
```
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, SRCIPV4ADDR="10.10.1.1", DSTIPV4ADDR="10.1.1.1", TESTMETHOD=PING;
```

**典型脚本（010108 DNS 探测全量，目的 IP 为 DNS Server）**：
```
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=DNS, DOMAINVALUE="www.example.com", WHOLEDETECT=ENABLE, SECTIONNUM=0;
```

**典型脚本（010108 DNS 探测非全量）**：
```
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=DNS, DOMAINVALUE="www.example.com";
```

**典型脚本（010108 TRACERT 探测，约 450s 返回）**：
```
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=TRACERT;
```

> 命令 notes：适用 NF：PGW-U、UPF；本命令属高危命令——执行可能导致在线用户被去激活；执行后立即探测；多次执行只保留最后一次结果；同一时间只能执行一个探测（并发后者失败）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：路由探测"——前置依赖 [0-00043](task/UDG/20.15.2/0-00043.md)（POOLNAME）+ [0-00044](task/UDG/20.15.2/0-00044.md)（SECTION）；用于业务调测/版本升级/地址扩容/异常投诉场景。

## 决策点：探测方式选择（DP 0-00048）

问：该次探测采用哪种方式（决定 TESTMETHOD 取值及关联参数）？

| 选项 | TESTMETHOD | 联动参数 | 适用场景 |
|---|---|---|---|
| PING 探测（默认） | PING | DSTIPV4ADDR；支持 WHOLEDETECT | 通用连通性探测 |
| DNS 探测 | DNS | DSTIPV4ADDR（DNS Server IP）+ DOMAINVALUE；支持 WHOLEDETECT | 网络中存在防火墙禁 PING 报文时 |
| TRACERT 探测 | TRACERT | DSTIPV4ADDR；**不支持** WHOLEDETECT | 故障定位，查看网络故障位置（约 450s 返回） |

> 1 个特性命中（010108）覆盖三方式 + 全量/非全量 + 指定源/随机源 5 个脚本。

## 约束

- **高危命令**（critical，rule-0-00448）：SRCIPV4ADDR 若填已在线用户 IP，执行会导致该在线用户被去激活 — 后果：在线用户被去激活；**建议 SRCIPV4ADDR 取空闲 IP**
- **POOLNAME 前置依赖**（critical，rule-0-00448）：IPPOOL 必须先由 [0-00043](task/UDG/20.15.2/0-00043.md) 建好 — 后果：未建引用命令拒收
- **DSTIPV4ADDR 有效目的 IP**（critical，rule-0-00448）：必须为有效目的 IP（非广播/非全 0） — 后果：无效地址探测失败
- **同时间只能执行一个探测**（critical，rule-0-00448）：并发后者失败 — 后果：并发探测后者被拒
- **多次执行只保留最后一次结果**（warning，rule-0-00448）：结果非累计 — 后果：先前结果丢失
- **TRACERT 不支持 WHOLEDETECT**（warning，rule-0-00448）：TRACERT 配 WHOLEDETECT 命令拒收 — 后果：TRACERT 全量探测失败
- **TRACERT 约 450s 返回**（warning，rule-0-00448）：长时间占用命令通道 — 后果：探测期间其他动作阻塞
- **LENGTH 仅 PING 可配**（warning，rule-0-00448）：LENGTH 是条件可选 — 后果：非 PING 配 LENGTH 命令拒收
- **DOMAINVALUE 仅 DNS 必填**（warning，rule-0-00448）：DOMAINVALUE 是条件必选 — 后果：DNS 缺 DOMAINVALUE 命令拒收
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
- **DSCPV/TCV 按 IPVERSION 互斥**（warning，rule-0-00448）：DSCPV 仅 IPV4，TCV 仅 IPV6 — 后果：跨版本配错命令拒收
