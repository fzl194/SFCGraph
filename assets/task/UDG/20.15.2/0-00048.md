# 启动PDN侧路由探测（STR PDNROUTETST）

> 命令静态知识见 [STR PDNROUTETST](command/UDG/20.15.2/STR-PDNROUTETST.md)。本页只讲配置生成实例化时这条命令**怎么配**。这是**运行态探测命令**（STR=Start），不是配置态 ADD/SET。

## 配置方法

本命令触发一次性路由连通性探测，参数随探测方式（TESTMETHOD）取值不同而组合。核心是按"探测方式 → 决定参数集"组织，IPPOOL/DSTIPV4ADDR 必填，其余按方式条件出现。

**三方式参数组合**

| 探测方式 TESTMETHOD | 必带参数 | 排除参数 | 全量探测 WHOLEDETECT | 典型场景 |
|---|---|---|---|---|
| PING（默认） | DSTIPV4ADDR（PDN/DN 服务器 IP） | DOMAINVALUE | 支持 ENABLE | 通用连通性探测 |
| DNS | DSTIPV4ADDR=**DNS Server IP** + DOMAINVALUE | — | 支持 ENABLE | 网络中存在防火墙禁 PING 报文时 |
| TRACERT | DSTIPV4ADDR | DOMAINVALUE | **不支持**（不可配 WHOLEDETECT） | 故障定位，查看网络故障位置（约 450s 返回） |

**源 IP（SRCIPV4ADDR）取值约定**

| 取值 | 行为 |
|---|---|
| 指定空闲 IP | 探测源 IP 用此参数（建议） |
| 不配 + PING/DNS | UDG 从每个路由网段随机选一个 IP 作探测源 |
| 不配 + TRACERT | UDG 随机选一个地址网段的 IP，不遍历所有网段 |

**全量探测（WHOLEDETECT=ENABLE）**

| SECTIONNUM | 行为 |
|---|---|
| 指定（0~63） | 针对此地址段下所有路由网段触发探测 |
| 不指定 | 针对 POOL 内所有地址段下所有路由网段触发探测 |

**典型行（GWFD-010108 实例）**
```
// PING 全量探测（指定地址段 0）
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=PING, WHOLEDETECT=ENABLE, SECTIONNUM=0;
// PING 非全量（指定源 IP）
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, SRCIPV4ADDR="10.10.1.1", DSTIPV4ADDR="10.1.1.1", TESTMETHOD=PING;
// DNS 探测（目的 IP 必须为 DNS Server）
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=DNS, DOMAINVALUE="www.example.com", WHOLEDETECT=ENABLE, SECTIONNUM=0;
// TRacert 探测（约 450s 返回，不支持全量）
STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=TRACERT;
```

**IPVERSION 固定 IPV4**（本特性场景仅 IPv4 探测，IPv6 路径 SRCIPV6PREFIX/DSTIPV6PREFIX 未走）。

## 决策点：PDN侧路由探测方式（DP 0-00014）

问：该次探测采用哪种方式？（决定 TESTMETHOD 取值及关联参数）

| 选项 | TESTMETHOD | 联动 |
|---|---|---|
| PING 探测（默认） | PING | 不需 DOMAINVALUE；支持 WHOLEDETECT |
| DNS 探测 | DNS | 必填 DOMAINVALUE；DSTIPV4ADDR 必须为 DNS Server IP；支持 WHOLEDETECT |
| Tracert 探测 | TRACERT | 不需 DOMAINVALUE；**不支持** WHOLEDETECT |

## 约束

- **rule-0-00166**（critical）高危命令：SRCIPV4ADDR 若填已在线用户 IP，执行会导致该在线用户被去激活 — 建议 SRCIPV4ADDR 取空闲 IP
- **rule-0-00167**（warning）执行约束：同一时间只能执行一个探测（并发后者失败）；探测前需确认 IPPOOL 下有可探测地址；DSTIPV4ADDR 须为有效目的 IP（非广播/非全 0）；多次执行只保留最后一次结果；Tracert 约 450s 返回且不支持全量探测 — 后果：探测失败 / 结果丢失 / Tracert 长时间占用
