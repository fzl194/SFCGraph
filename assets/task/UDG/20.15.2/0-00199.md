# 增加流行为（ADD MQCBEHAVIOR）

> 命令静态知识见 [ADD MQCBEHAVIOR](command/UDG/20.15.2/ADD-MQCBEHAVIOR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。MQC 复杂流分类三件套（Class-Behavior-Policy）骨架：**本命令建 Behavior 空容器**，挂流动作由后续子命令挂载。

## 配置方法（配置方法字典）

ADD MQCBEHAVIOR 是"增加流行为"命令。**仅 1 参必选**（BEHAVIORNAME）。Behavior 是**空容器**，本命令只命名；流动作（permit/deny/重定向/重标记/级联/URPF/CAR 等）全部由后续子命令挂载。1 个特性命中（IPFD-012001 QoS 复杂流分类）覆盖唯一典型范式。

### 配置维度 1：行为名称（参数 BEHAVIORNAME，必选）

| 取值 | 约束 | 适用场景 |
|---|---|---|
| **字符串 1~31** | 不允许为系统预定义的 `be` | 本端规划；IPFD-012001 范例 `behavior1` |

> **空容器特性**：本命令不挂任何流动作，仅创建 Behavior 对象；后续在 CB 对绑定时由 [0-00201](task/UDG/20.15.2/0-00201.md) ADD MQCCLASSIFIER + 本命令共同构成"分类 → 行为"映射。

**典型脚本（IPFD-012001 QoS 复杂流分类，建空 behavior1）**：
```
ADD MQCBEHAVIOR: BEHAVIORNAME="behavior1";
ADD QOSACTRDRNHP: BEHAVIORNAME="behavior1", IPVERSION=IPv4, NEXTHOP="192.168.2.3";
```

> 命令 notes：**最大记录数 8192**；**执行后立即生效**；BEHAVIORNAME 不允许 `be`（系统预定义）；流动作由后续子命令挂载：重定向→[0-00200](task/UDG/20.15.2/0-00200.md) ADD QOSACTRDRNHP；permit/deny→SET QOSACTFILTER（★R1.5 atom 待补）；重标记→ADD SQOSREMARK（★R1.5 atom 待补）；级联→ADD QOSACTRDRPOLICY（★R1.5 atom 待补）；URPF→ADD SQOSURPF（★R1.5 atom 待补）；CAR→ADD SQOSCAR（★R1.5 atom 待补）。

**步骤位置**：在所属 feature 编排中位于"步骤 N：建 Behavior 容器"——前置 无；下游 [0-00200](task/UDG/20.15.2/0-00200.md) ADD QOSACTRDRNHP（挂重定向动作）；[0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY（CB 节点绑定 BEHAVIORNAME）。

## 决策点

（无。本命令仅 1 个特性命中且用法单一——"建空 Behavior 容器"；无 DP 分叉。）

## 约束

- **最大记录数 8192**（critical，rule-0-00199）：整机 Behavior 最大 8192 条 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00199）：执行后立即生效 — 后果：误操作即刻生效
- **BEHAVIORNAME 不能为 `be`**（critical，rule-0-00199）：系统预定义行为名 `be` 不允许重复定义 — 后果：命名冲突命令拒收
- **空行为容器**（info，rule-0-00199）：本命令仅创建空 Behavior；流动作须由后续子命令挂载 — 后果：不挂动作的 Behavior 在 CB 绑定时无效果
- **B 侧须在 CB 节点绑定（0-00203）前先建**（warning，rule-0-00199）：BEHAVIORNAME 必须在 [0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY 引用前先建 — 后果：未建引用失败
- **BEHAVIORNAME 长度**（info，rule-0-00199）：字符串 1~31 — 后果：超长命令拒收
