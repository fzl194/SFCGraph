# 头增强对象配置

> **头增强族通用 compound**（HTTP/RTSP/HTTPS/防欺诈 共用骨架）：定义报文头扩展字段插入规则。被引用于 [2-00010](task/UDG/20.15.2/2-00010.md)（HTTP头增强）/[2-00011](task/UDG/20.15.2/2-00011.md)（RTSP头增强）/[2-00012](task/UDG/20.15.2/2-00012.md)（HTTPS头增强）/[2-00013](task/UDG/20.15.2/2-00013.md)（HTTP头防欺诈）。

## 配置方法

新增头增强对象（HEADERENNAME），**多次执行 ADD HEADEN 为同一对象配置多个插入字段**，按需配加密算法/编码/防欺诈/灰名单。HTTPS 场景用 ADD TLSHEADEN（场景差异见下）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 头增强对象（HTTP/RTSP） | ADD HEADEN | HEADERENNAME + DATATYPE + PREFIXNAME + ENCRYALGORI（+ PSWDKEY + CIPHERTXTENCODE + ANTIFRAUD + GRAYLIST）|
| 头增强对象（HTTPS，场景差异） | ADD TLSHEADEN | TLSHEADENNAME + DATATYPE + PREFIXNAME + ENCRYALGORI（TLV 格式扩展字段）|
| AES IV 值（加密联动，可选；atom 待建 `[[SET AES128]]` / `[[SET AES256]]`）| SET AES128 / SET AES256 | IV 值；HEADENRANDOMIV=ENABLE 支持随机 IV |
| RSA 公钥（加密联动，可选；atom 待建 `[[ADD RSA2048]]` / `[[ADD RSA3072]]` / `[[ADD RSA1024]]`）| ADD RSA2048 / ADD RSA3072 / ADD RSA1024 | 公钥文件（RSA*NAME 供 ADD HEADEN 引用）|
| RC4 密钥 MD5 开关（可选；atom 待建 `[[SET HEADENGLBPARA]]`）| SET HEADENGLBPARA | RC4KEYMD5EN=ENABLE（ADD HEADEN 的 RC4KEYMD5EN=INHERIT 时生效）|
| 编码开关（CIPHERTXTENCODE=INHERIT 时，可选；atom 待建 `[[SET BASE64]]`）| SET BASE64 | base64 编码规则；ASCII 由软参 BIT475 控 |

**典型脚本（基于 URL 触发的标准部署：4 字段不加密，activation 原样）**：
```
ADD HEADEN:HEADERENNAME="header_test",DATATYPE=APN,PREFIXNAME="pre-apn",ENCRYALGORI=NONE;
ADD HEADEN:HEADERENNAME="header_test",DATATYPE=IMSI1,PREFIXNAME="pre-imsi",ENCRYALGORI=NONE;
ADD HEADEN:HEADERENNAME="header_test",DATATYPE=MSIP1,PREFIXNAME="pre-msip",ENCRYALGORI=NONE;
ADD HEADEN:HEADERENNAME="header_test",DATATYPE=MSISDN1,PREFIXNAME="pre-msisdn",ENCRYALGORI=NONE;
```

**步骤位置**：承接 [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链，产出 FLOWFILTERNAME）+ 协议识别（WELLKNOWNPORT/SIGNATUREDB/PARSERDB）；产出 HEADERENNAME 供下游 [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定中 RULE 经 `POLICYTYPE=HEADEN, POLICYNAME=<HEADERENNAME>` 直接引用（**关键差异：不经 PCCPOLICYGRP，见场景差异 §1**）。

## 场景差异

> 头增强族（HTTP/RTSP/HTTPS）共用本 compound，差异集中在协议命令、加密算法、编码方式、防欺诈支持、灰名单模式五类。HTTP 与 RTSP 同用 ADD HEADEN（字段格式一致），HTTPS 用 ADD TLSHEADEN（TLV 格式）。（NSH 头增强不在本轮三场景构建清单。）

### 1. 协议命令变种（HTTP/RTSP vs HTTPS）

| 协议 | 命令 | 字段格式 | 防欺诈支持 | 对应 feature |
|---|---|---|---|---|
| **HTTP**（1.x）| ADD HEADEN | `<前缀: 值>` 标准头扩展 | 支持（ANTIFRAUD）| [2-00010](task/UDG/20.15.2/2-00010.md) HTTP头增强 |
| RTSP | ADD HEADEN | 同 HTTP | 不支持（官方差异表声明；证据：`头增强功能之间的差异_10706790.md` §头防欺诈行）| [2-00011](task/UDG/20.15.2/2-00011.md) RTSP头增强 |
| **HTTPS** | ADD TLSHEADEN | TLV（Type-Length-Value）| 支持 | [2-00012](task/UDG/20.15.2/2-00012.md) HTTPS头增强 |

> 防欺诈参数 ANTIFRAUD 复用本 compound：[2-00013](task/UDG/20.15.2/2-00013.md) HTTP头防欺诈（内嵌 ADD HEADEN 的 ANTIFRAUD，不独立配策略对象）。

> 关键约束：HTTP头增强**不支持 HTTPS 协议**（应用限制），HTTPS 需 ADD TLSHEADEN；**不支持 HTTP2.0**（仅 HTTP1.x）。

### 2. 加密算法（ENCRYALGORI，7+ 种）

| 取值 | 安全性 | 联动命令 | 适用 DATATYPE 限制 |
|---|---|---|---|
| NONE | 有风险 | — | 全部 |
| MD5 | 有风险（不建议）| PSWDKEY + SALTSW | 全部 |
| RC4 | 有风险（不建议）| PSWDKEY + RC4KEYMD5EN | 全部 |
| AES128（CBC）/ AES256（CBC）| 有风险（不建议）| PSWDKEY + SET AES128/SET AES256 | 全部 |
| **AES128_GCM / AES256_GCM**（推荐）| 安全 | PSWDKEY + SET AES128/SET AES256 | 全部 |
| AES128_CTR / AES256_CTR | 安全 | PSWDKEY + SET AES128/SET AES256 | 全部 |
| SHA256 | 安全 | PSWDKEY + SALTSW | 全部 |
| RSA1024 | 有风险（不建议）| ADD RSA1024 公钥 | 仅 MSISDN/IMSI/IMEI/MSIP |
| RSA2048 / RSA3072 | 安全 | ADD RSA2048/RSA3072 公钥 | 仅 MSISDN/IMSI/IMEI/MSIP |

> 默认 AES128_GCM。MD5/RC4/RSA1024/AES128(CBC)/AES256(CBC)/NONE 有安全风险不建议。

### 3. 编码方式（CIPHERTXTENCODE）

| 取值 | 含义 | 联动 |
|---|---|---|
| INHERIT（默认）| 继承全局 | base64 由 SET BASE64 控；ASCII 由软参 BIT475 控 |
| NONE | 不编码 | — |
| BASE64 | 命令内显式开 base64 | 可配等号/斜杠/加号替换字符（HEADENEQ* / HEADENSL* / HEADENPL*）|
| ASCII | 命令内显式开 ASCII | — |

### 4. 防欺诈（ANTIFRAUD，强耦合 GWFD-110401）

| ANTIFRAUD | 含义 | 联动 |
|---|---|---|
| DISABLE（默认）| 不检测已有字段 | — |
| ENABLE | 使能头防欺诈：插入前先检测 HTTP 头是否已有该字段，有则纠正错误值/删冗余 | 需配套 GWFD-110401 License LKV3G5HHAS01；执行顺序由软参 BIT1030 控（0=中执行/1=前执行，初始值 0；证据：软件参数文档 `BIT1030 控制HTTP头防欺诈功能是否在头增强业务前执行_08922610.md`；防欺诈 → 字段纠正 → 插入）|

> 防欺诈**内嵌于 ADD HEADEN 的 ANTIFRAUD 参数**，GWFD-110401 不独立配策略对象。

### 5. 灰名单模式（GRAYLIST）

| GRAYLIST | 含义 | 适用场景 |
|---|---|---|
| DISABLE（默认）| 正常头增强（防欺诈+插入）| 需要插入用户信息 |
| ENABLE | **只防欺诈不插入头增强字段** | 只需检测纠正已有错误字段，不做插入；须配合 ANTIFRAUD=ENABLE |

### 6. 多参数拼接（DATATYPE=MULTIPARA1/2/3）

- 每个头增强对象最多 **3 个拼接字段**（MULTIPARA1/2/3），每个拼接字段最多 **4 参数**（PARADATATYPE1/2/3）。
- 拼接参数**仅支持 MSISDN、SGSNIP、MSIP**。
- PARADATATYPE1/2/3 不可同时 NONE，任意两值不可相同。
- 分隔符 SEPARATOR 默认空格（不支持双引号/单引号/逗号）。
- 版本：20.10.0+ 支持。

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| 协议 HTTP/RTSP（联动特性级）| ADD HEADEN；HTTP 支持防欺诈，RTSP 按官方差异表不支持（证据：`头增强功能之间的差异_10706790.md`）|
| 协议 HTTPS（联动特性级）| ADD TLSHEADEN（TLV 格式）；需 GWFD-110263 |
| 加密 NONE（联动特性级）| 无 PSWDKEY；无编码（activation 演示）|
| 加密 AES128_GCM/AES256_GCM（推荐）| PSWDKEY + SET AES128/SET AES256（IV 值；atom 待建 `[[SET AES128]]`/`[[SET AES256]]`）|
| 加密 RSA2048/RSA3072 | 公钥文件（ADD RSA2048/RSA3072；atom 待建）；仅 MSISDN/IMSI/IMEI/MSIP |
| 编码继承 INHERIT | SET BASE64（atom 待建）+ 软参 BIT475 |
| 编码显式 BASE64/ASCII | 命令内 CIPHERTXTENCODE |
| 防欺诈 ANTIFRAUD=ENABLE | 强耦合 GWFD-110401（License LKV3G5HHAS01）；执行顺序 BIT1030（证据：`BIT1030 控制HTTP头防欺诈功能是否在头增强业务前执行_08922610.md`）|
| 灰名单 GRAYLIST=ENABLE | 只防欺诈不插入；须 ANTIFRAUD=ENABLE |

## 约束

- **PREFIXNAME 不可为 HTTP 标准头域名**（critical）：如 host、accept 会导致防欺诈失效 — 后果：防欺诈无法识别已有字段。
- **POLICYTYPE=HEADEN 直接引用 HEADERENNAME**（critical）：RULE 的 POLICYNAME 指向 HEADERENNAME（非 PCCPOLICYGRPNM）— 指错则头增强不触发。
- **HTTPS 不走 ADD HEADEN**（critical）：HTTPS 需 ADD TLSHEADEN；ADD HEADEN 只处理 HTTP/RTSP — 混用则 HTTPS 报文不做头增强。
- **不支持 HTTP2.0**（warning）：仅 HTTP1.x — 后果：HTTP2.0 报文不做头增强。
- **整机规格**（warning）：≤100 头增强实例 / ≤500 前缀字段 / ≤32 前缀开防欺诈 / 单报文 ≤16 前缀开防欺诈。
- **加密仅作用字段取值**（info）：前缀名不加密（预期设计）。
- **加密联动 atom 待建**（info）：SET AES128/AES256/RSA2048/RSA3072/RSA1024/HEADENGLBPARA/BASE64 等联动命令 atom 未建（`[[SET AES128]]` 等占位），当前 activation 演示 ENCRYALGORI=NONE 不加密，加密联动为可选；需加密场景补建对应 atom。
