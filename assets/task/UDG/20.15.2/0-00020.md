# 配置APN（ADD APN）

> 命令静态知识见 [ADD APN](command/UDG/20.15.2/ADD-APN.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

ADD APN 是多特性共享的**基础对象**（21+ 特性引用，最大记录数 10000）。APN 标识系统接入的数据网络——是地址分配 / 接入控制 / 过载控制 / 7 层统计 / 双栈 / NB-IoT 速率控制等特性的共同前置。本命令的核心配置空间有 4 个维度：**名称来源**（local 或 peer，决定是否需与控制面一致）、**流量统计维度**（`APPLAYERVOLUME` 控制 L7 / 全量口径）、**VPN 绑定**（`HASVPN` / `HASVPNIPV6` 分别绑 IPv4 / IPv6 VPN）、**其他能力开关**（PSEUDOACTSWITCH / RESTORPGWSWITCH / PDTNSWITCH / HEADENGRAYLIST / CFGDOMAINNAME 等冷门参数）。

### 配置维度 1：APN 名称（参数 APN，必选）

| 参数 | 取值约定 | 说明 |
|---|---|---|
| `APN` | 字符串 1~63 字符；合法字符为 `-`、数字、字母、`.`；**不以 `.` 开头** | APN 名称（核心键） |

**名称来源决策**：由 DP 0-00065 决策——见决策点段。

### 配置维度 2：流量统计维度（参数 APPLAYERVOLUME）

| 取值 | 统计口径 | 计费影响 |
|---|---|---|
| `ENABLE` | **只统计 L7 应用层流量** | 计费流量只算应用层（L7） |
| `DISABLE`（默认） | 统计 L3/L4 + L7 全部流量 | 计费流量含全协议层（默认） |

> **典型场景**：仅启用 7 层流量统计（GWFD-020308）时配 `APPLAYERVOLUME=ENABLE`；否则按默认 DISABLE 走全量统计。

### 配置维度 3：VPN 绑定（参数 HASVPN / HASVPNIPV6 + VPNINSTANCE / VPNINSTANCEIPV6）

IPv4 / IPv6 VPN 可独立绑定：

| 参数 | 取值 | 联动必传 | 引用对象 | 典型场景 |
|---|---|---|---|---|
| `HASVPN` | `ENABLE` / `DISABLE`（默认） | `VPNINSTANCE` | [ADD VPNINST](task/UDG/20.15.2/0-00041.md) | IPv4 VPN 绑定（GWFD-010104） |
| `VPNINSTANCE` | 字符串 1~31 | 前置 `HASVPN=ENABLE` | VPN 实例名 | — |
| `HASVPNIPV6` | `ENABLE` / `DISABLE`（默认） | `VPNINSTANCEIPV6` | [ADD VPNINST](task/UDG/20.15.2/0-00041.md) | IPv6 VPN 绑定（GWFD-020403） |
| `VPNINSTANCEIPV6` | 字符串 1~31 | 前置 `HASVPNIPV6=ENABLE` | VPN 实例名 | — |

> IPv4 和 IPv6 绑定**独立**——单 APN 可仅绑 IPv4（DISABLE IPv6）、仅绑 IPv6、或两者都绑。

### 配置维度 4：其他能力开关（可选、按通用配置门槛不绑）

| 参数 | 作用 | 说明 |
|---|---|---|
| `PSEUDOACTSWITCH` | 伪激活 / 试运行开关 | 冷门，需要时查命令 wiki |
| `RESTORPGWSWITCH` | 恢复源 GW 开关 | 冷门，需要时查命令 wiki |
| `PDTNSWITCH` | PDN 切换开关 | 冷门，需要时查命令 wiki |
| `HEADENGRAYLIST` | 头增强灰度名单 | 冷门，需要时查命令 wiki |
| `CFGDOMAINNAME` | 配置域名 | 冷门，需要时查命令 wiki |

> **未绑按默认门槛不配**——绝大多数特性用不到这五个参数。如确有需要（如头增强灰度场景），回查命令 wiki / 特性 md。

**步骤位置**：在所属 feature 编排中属于"基础对象配置"步骤——通常在业务特性配置**前**完成（地址分配 / 双栈 / NB-IoT 等）。

## 决策点：APN 名称来源（DP 0-00065）

问：ADD APN 的 APN 名称取值来源（决定是否需与控制面保持一致）？

| 选项 | APN variable_source | 适用场景 |
|---|---|---|
| `local_planned`（默认） | UDG 侧独立规划，无需与控制面同步 | 7 层统计 / 地址分配 / 接入控制 / 过载控制 / 双栈 / QoS / eDRX / Non-IP / URL 过滤等（绝大多数） |
| `peer_aligned` | 与控制面（MME / SGW-C / PGW-C）保持一致 | NB-IoT / eMTC 终端接入速率控制（110611/110612/110646/110647）——控制面同步配 CCR/APN Ratecontrol |

## 约束

- **最大记录数与生效时机**（warning，rule-0-00150）：最大记录数 10000；超规格新增失败；命令执行后**立即生效** — 后果：超规格新增失败
- **APN 命名约束**（warning，隐含）：APN 字符串 1~63，由 `-` / 数字 / 字母 / `.` 组成，**不以 `.` 开头** — 后果：非法命名致 ADD APN 失败
- **VPN 绑定前置**（critical，隐含）：`VPNINSTANCE` / `VPNINSTANCEIPV6` 必须在 [ADD VPNINST](task/UDG/20.15.2/0-00041.md) 中已创建；`HASVPN` / `HASVPNIPV6` 必须 `=ENABLE` 才传 VPN 实例名 — 后果：VPN 实例不存在致 ADD APN 失败
- **APN 名称跨网元一致**（critical，隐含）：NB-IoT / eMTC 场景下 APN 名称须与控制面（MME/SGW-C/PGW-C）一致 — 后果：控制面未配该 APN 致终端接入失败
