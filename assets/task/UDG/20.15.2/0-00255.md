# 增加内容过滤策略绑定关系（ADD CFPROFBINDCFT）

> 命令静态知识见 [ADD CFPROFBINDCFT](command/UDG/20.15.2/ADD-CFPROFBINDCFT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务把已建的内容过滤策略（CFPROFILE）绑定到已建的内容过滤模板（CFTEMPLATE），完成"模板←策略"的挂载。两个参数均来自已配置数据。适用 NF：PGW-U、UPF。

**取值样例（数据规划表，GWFD-110471）**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| CFTEMPLATENAME | `icaptemplate` | 引用 ADD CFTEMPLATE | 已通过 ADD CFTEMPLATE 配置 |
| CFPROFILENAME | `cf1` | 引用 ADD CFPROFILE | 已通过 ADD CFPROFILE 配置 |

**脚本样例**

```
ADD CFPROFBINDCFT: CFTEMPLATENAME="icaptemplate", CFPROFILENAME="cf1";
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：CFGDOMAINNAME（配置域名称，optional）主特性场景不配。本命令把策略容器（0-00252）挂到执行模板（0-00253）上，是 URL 过滤配置链路的收尾绑定步。

## 约束

- **rule-0-00389**（warning）命令最大记录数 **1000**，超限 ADD 失败；**每个模板最多绑定 10 个策略**；**只对之后发生承载更新的用户或新激活用户生效**（存量在线用户不感知）；CFTEMPLATENAME/CFPROFILENAME 均为引用，模板与策略必须先建。
