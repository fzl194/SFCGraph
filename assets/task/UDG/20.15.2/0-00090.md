# 配置DNS重写动作（ADD DNSOVERWRITING）

> 命令静态知识见 [ADD DNSOVERWRITING](command/UDG/20.15.2/ADD-DNSOVERWRITING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD DNSOVERWRITING 是 DNS 纠错的"动作策略定义"命令：当用户访问错误域名（DNS 响应返回 NXDOMAIN）时，将 DNS 响应报文中的 IP 地址改写为第三方 Platform IP，引导用户访问第三方 Platform。本命令的核心配置空间有 5 个维度：**动作名 DNSOVERWRTNAME**（被 RULE.POLICYNAME 经 POLICYTYPE=SMARTREDIRECT 引用）、**扩展过滤器组合**（EXTFLTTYPE1-5 + EXTFLTNAME1-5，最多 5 组 AND/OR/NOT 逻辑）、**目标服务器 IP**（IPv4/IPv6 各最多 5 个）、**绑定错误码 BINDERRCODENAME**（必前置 [0-00089](task/UDG/20.15.2/0-00089.md)）、**配置域 CFGDOMAINNAME**。

### 配置维度 1：动作名（参数 DNSOVERWRTNAME，必选，本端规划）

字符串 1~31（不支持空格、不区分大小写）；产出的 DNSOVERWRTNAME 对象被 ADD RULE 的 POLICYNAME 经 POLICYTYPE=SMARTREDIRECT 引用。

### 配置维度 2：扩展过滤器组合（参数 EXTFLTTYPE1-5/EXTFLTNAME1-5，可选，最多 5 组）

| 参数 | 取值 | 配套参数 |
|---|---|---|
| **EXTFLTTYPE1~5** | AND / OR / NOT（枚举） | EXTFLTNAME1~5（条件必选） |
| **EXTFLTNAME1~5** | 字符串 1~31 | 引用 [ADD EXTENDEDFILTER](task/UDG/20.15.2/0-00022.md) 产出的扩展过滤器名 |

> 多过滤器按 EXTFLTTYPE 逻辑组合（AND/OR/NOT），最多 5 组；AND 组个数 ≤3。

### 配置维度 3：目标服务器 IP（参数 SERVERIP1-5/SERVERIPV6_1-5，可选，最多各 5 个）

| 参数 | 取值类型 | 备注 |
|---|---|---|
| **SERVERIP1~5** | IPv4 点分十进制 | DNS 改写后的 IPv4 目标地址 |
| **SERVERIPV6_1~5** | IPv6 冒号分十六进制 | DNS 改写后的 IPv6 目标地址 |

### 配置维度 4：绑定错误码（参数 BINDERRCODENAME，可选）

字符串 1~31（不区分大小写、不支持空格）；引用 [ADD ERRORCODE](task/UDG/20.15.2/0-00089.md) 产出的 ERRORCODENAME；典型场景 dnserrorcode（NXDOMAIN=3 EQUAL）。

### 配置维度 5：配置域（参数 CFGDOMAINNAME，可选）

字符串 1~31；命令所属公共配置域的名称。

> **典型范式（DNS纠错，典型 NXDOMAIN → Platform IP）**：
> ```
> ADD DNSOVERWRITING:DNSOVERWRTNAME="dec_1",EXTFLTTYPE1=AND,EXTFLTNAME1="l7-filter1",SERVERIP1="192.168.1.1",BINDERRCODENAME="dnserrorcode";
> ```

**步骤位置**：在所属 feature 编排中通常为"步骤 3：配置 DNS 重写动作"——ADD EXTENDEDFILTER + ADD ERRORCODE 之后、ADD RULE（POLICYTYPE=SMARTREDIRECT 引 DNSOVERWRTNAME）之前。

## 约束

- **整机最大记录数为 2000**（warning，rule-0-00296）：系统上限 — 后果：超规格新增失败
- **AND 组个数 ≤3**（warning，隐含）：一条 DNS 重写配置中 AND 类型的过滤器组数量上限 — 后果：超 3 拒收
- **EXTFLTNAME 引用须已配置**（critical，隐含）：EXTFLTNAME1/2/3/4/5 须先经 ADD EXTENDEDFILTER（[0-00022](task/UDG/20.15.2/0-00022.md)）配置 — 后果：引用不存在拒收
- **BINDERRCODENAME 引用须已配置**（critical，隐含）：BINDERRCODENAME 须先经 ADD ERRORCODE（[0-00089](task/UDG/20.15.2/0-00089.md)）配置 — 后果：引用不存在拒收
- **DNSOVERWRTNAME 全局唯一**（critical，隐含）：动作名全局唯一 — 后果：重名拒收
- **立即生效**（info，隐含）：该命令执行后立即生效 — 后果：动作策略变更实时影响 DNS 改写行为
- **DNSOVERWRTNAME 须先于 RULE 配置**（critical，隐含）：被 RULE.POLICYTYPE=SMARTREDIRECT.POLICYNAME 引用，须先建 — 后果：未先建导致 RULE 引用失败
