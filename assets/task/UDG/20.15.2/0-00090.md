# 配置DNS重写动作（ADD DNSOVERWRITING）

> 命令静态知识见 [ADD DNSOVERWRITING](command/UDG/20.15.2/ADD-DNSOVERWRITING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD DNSOVERWRITING 是 GWFD-110283 DNS 纠错专属命令，将命中的 DNS 响应报文 IP 改写为第三方 Platform IP。主场景 1 组 AND 过滤器 + 1 个 IPv4 SERVERIP 已足，命令真相的 EXTFLTTYPE2-5/EXTFLTNAME2-5/SERVERIP2-5/SERVERIPV6_1-5/CFGDOMAINNAME 共 18 个多组参数按多组过滤器/IPv6 场景补充，主特性不补。EXTFLTNAME1 引用 EXTENDEDFILTER，BINDERRCODENAME 引用 ERRORCODE，DNSOVERWRTNAME 产出对象被 RULE(POLICYTYPE=SMARTREDIRECT) 的 POLICYNAME 引用。

**主场景范式（从证据数据规划表归纳）**

| 字段 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| DNSOVERWRTNAME | dec_1 | 本端规划 | DNS 重写动作名，1~31 字符 |
| EXTFLTTYPE1 | AND | 本端规划 | 扩展过滤器类型，AND/OR/NOT |
| EXTFLTNAME1 | l7-filter1 | 已配置数据中获取 | 引用 0-00022 产出的 EXTENDEDFILTER，可 LST EXTENDEDFILTER 查询 |
| SERVERIP1 | 192.168.1.1 | 全网规划 | 第三方 Platform IP，IPv4 单播 |
| BINDERRCODENAME | dnserrorcode | 已配置数据中获取 | 引用 0-00089 产出的 ERRORCODE，可 LST ERRORCODE 查询 |

**脚本样例**
```
ADD DNSOVERWRITING:DNSOVERWRTNAME="dec_1",EXTFLTTYPE1=AND,EXTFLTNAME1="l7-filter1",SERVERIP1="192.168.1.1",BINDERRCODENAME="dnserrorcode";
```

**配置链路**（GWFD-110283 步骤 5→6→7）：先 ADD EXTENDEDFILTER(0-00022) → ADD ERRORCODE(0-00089) → **ADD DNSOVERWRITING** → ADD RULE(POLICYTYPE=SMARTREDIRECT)。

## 约束

- **rule 0-00296**（requires/warning）系统最大记录数 2000；立即生效；一条 DNS 重写配置最多 5 组扩展过滤器，AND 组个数≤3；最多 5 组错误码范围；过滤器匹配逻辑（匹配 AND 组所有成功 OR 匹配 OR 组任一成功，且匹配 NOT 组所有失败；未配置走默认）；EXTFLTNAME1 引用 0-00022、BINDERRCODENAME 引用 0-00089；DNSOVERWRTNAME 须先于 RULE(POLICYTYPE=SMARTREDIRECT) 配置 — 后果：上限超限 ADD 失败；AND 组超 3 拒收；引用 EXTENDEDFILTER/ERRORCODE 不存在拒收；未先配 DNSOVERWRTNAME 导致 RULE 引用失败
