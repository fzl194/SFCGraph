# 刷新生效（SET REFRESHSRV）

> 命令静态知识见 [SET REFRESHSRV](command/UDG/20.15.2/SET-REFRESHSRV.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

SET REFRESHSRV 是过滤链 + 计费 backbone 的"高危收尾命令"——不执行则 Filter / Rule / RuleBinding 等配置变更不生效。仅 `REFRESHTYPE` 一个参数，按"本次改动对象范围"分叉。本命令的核心配置空间有 3 个维度：**刷新范围**（`ALL` / `USERPROFILE` 二选一）、**适用范围**（本次配置改动了什么对象决定）、**时序铁律**（必须作为最后一步）。

### 配置维度 1：刷新范围（参数 REFRESHTYPE，必选）

| 取值 | 作用 | 性能影响 | 适用场景 |
|---|---|---|---|
| `ALL` | 刷新 Filter / FilterGroup / Acl / AclNode 全集 | 重——大表全量重载 | 改动了过滤条件（Acl / FlowFilter / 七层规则等）所有特性 |
| `USERPROFILE` | 仅刷新 UserProfile / Rule / RuleBinding 层 | 轻——只动 UP/Rule 索引 | 仅新增/修改 UP / RULE / RULEBINDING（未动 Filter）的特性 |

> `ALL` 是**安全超集**（包含 USERPROFILE 范围）——不确定时选 `ALL`。

### 配置维度 2：适用范围（按本次改动对象决定）

由"本次配置动了哪些对象类型"决定 REFRESHTYPE 取值：

| 改动范围（按对象类型） | REFRESHTYPE 取值 | 典型场景 |
|---|---|---|
| 改动了 Filter / FilterGroup / Acl / AclNode | `ALL` | 内容计费 / 融合计费 / 时长流量事件计费 / 动态PCC / 本地PCC / ADC / Shaping / 双故障 / ProxyUPF 等 |
| 仅改动 UserProfile / Rule / RuleBinding | `USERPROFILE`（轻量） | 在线 / 离线计费 / 计费提醒 / URL 核减 / 用户Portal / DNS纠错 / HTTP智能重定向 / 视频承载 / FUP / Host识别 / URL过滤 等 |
| 全集安全超集 | `ALL`（默认） | 任何不确定的情况 |

### 配置维度 3：时序铁律

| 规则 | 说明 | 后果 |
|---|---|---|
| **必须最后执行** | SET REFRESHSRV 必须作为整段配置的**最后一步** | 提前执行 → 后置配置未刷新致部分生效 |
| **必须收敛于一次** | 完整配置生成后只调用一次（可用 `ALL` 覆盖）；多次配置可分批刷新 | 重复刷新降低性能 / 不报错但多余 |
| **不可对存量立即生效** | `REFRESHTYPE=USERPROFILE` 时新 RULEBINDING 只对新激活用户生效（见 [ADD RULEBINDING](0-00012.md) 约束） | 存量用户不生效需用户承载更新或新激活 |

**步骤位置**：在所属 feature 编排中始终是**最后一步**——所有 Filter / Rule / RuleBinding / URR 组绑定 / 防欺诈列表完成后执行。

## 决策点：业务刷新范围（DP 0-00060）

问：本次配置改动涉及的对象范围 → REFRESHTYPE 取值？

| 选项 | REFRESHTYPE | 适用范围 | 适用场景 |
|---|---|---|---|
| `refresh_all`（默认） | `ALL` | 刷新 Filter / FilterGroup / Acl / AclNode 全集（含 USERPROFILE/Rule 子集）| 改动了过滤条件的所有特性（内容计费 / 融合计费 / 时长流量事件计费 / 动态PCC / 本地PCC / ADC / Shaping / 双故障 / ProxyUPF 等）|
| `refresh_userprofile_only` | `USERPROFILE` | 仅刷新 UserProfile / Rule / RuleBinding 层 | 未改 Filter、仅新增 RULE / USERPROFILE / RULEBINDING 的特性（在线 / 离线计费 / 计费提醒 / URL 核减 / 用户Portal / DNS 纠错 / HTTP 重定向 / 视频承载 / FUP / Host 识别 / URL 过滤 等）|

## 约束

- **时序铁律**（critical，rule-0-00134）：必须作为收尾**最后一步**执行——在所有 Filter / FilterGroup / Acl / AclNode 及关联 RULE / USERPROFILE / RULEBINDING 配置完成后；高危——**立即生效** — 后果：时序错乱（先刷新后改配置）导致配置未生效或部分生效
- **覆盖与未覆盖区分**（critical，隐含）：`ALL` 不一定能完整覆盖本次改动（取决于底层刷新索引设计）——若不放心选 `ALL` 重刷保证安全 — 后果：选 `USERPROFILE` 但实际有 Filter 改动 → 部分过滤条件未生效
- **高危命令操作风险**（warning，隐含）：SET REFRESHSRV 是高危命令——错误刷新可能在配置变更窗口期造成瞬时业务中断 — 后果：业务瞬时中断
