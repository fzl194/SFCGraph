# 刷新生效（SET REFRESHSRV）

> 命令静态知识见 [SET REFRESHSRV](command/UDG/20.15.2/SET-REFRESHSRV.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

高危收尾命令——不执行则 Filter 等配置变更不生效。仅 REFRESHTYPE 一个参数，刷新范围按本次改动的对象分叉。

**改动了 Filter/FilterGroup/Acl/AclNode（全集刷新，内容计费/融合计费/时长流量事件计费/动态 PCC/本地 PCC）**：
```
SET REFRESHSRV: REFRESHTYPE=ALL;
```

**仅改动了 UserProfile/Rule/RULEBINDING（在线/离线计费等）**：
```
SET REFRESHSRV: REFRESHTYPE=USERPROFILE;
```

ALL 为安全超集（刷新全集），作为默认；多次配置可分批刷新。

## 决策点

**DP 0-00060**（single_choice, required, always）业务刷新范围？REFRESHTYPE 取何值。

| 选项 | REFRESHTYPE | 作用域 | 适用 |
|---|---|---|---|
| opt-refresh-all（默认） | ALL | 刷新 Filter/FilterGroup/Acl/AclNode 全集 | 改动了过滤条件的特性（内容计费/融合计费/时长流量事件计费/动态 PCC/本地 PCC/ADC/Shaping/双故障/ProxyUPF 等） |
| opt-refresh-userprofile | USERPROFILE | 仅刷新 UserProfile/Rule 层 | 未改 Filter、仅新增 RULE/USERPROFILE/RULEBINDING 的特性（在线/离线计费/计费提醒/URL 核减/用户 Portal/DNS 纠错/HTTP 重定向/视频承载/FUP/Host 识别/URL 过滤等） |

## 约束

- **rule-0-00134**（error）时序：必须作为收尾**最后一步**执行——在所有 Filter/FilterGroup/Acl/AclNode 及关联 RULE/USERPROFILE/RULEBINDING 配置完成后。高危，**立即生效**。时序错乱（先刷新后改配置）会导致配置未生效或部分生效。
