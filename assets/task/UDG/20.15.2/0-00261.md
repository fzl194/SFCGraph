# 增加ICAP服务器绑定关系（ADD ICAPSVRBINDISG）

> 命令静态知识见 [ADD ICAPSVRBINDISG](command/UDG/20.15.2/ADD-ICAPSVRBINDISG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务把 ICAP 服务器绑定进 ICAP 服务器组——形成"服务器组 → 服务器"多对一容器关系，URL 过滤模板最终通过组名落到具体 ICAP Server。两个必填参数均为引用型（已配置数据中获取），自身不规划新值。适用 NF：PGW-U、UPF。

**取值样例（数据规划表，主特性 GWFD-110471）**

| 参数 | 含义 | 样例值 | 获取方法 |
|---|---|---|---|
| ICAPSVRGRPNAME | ICAP服务器组名称 | icapsvrgrp1 | 引用 [0-00260](task/UDG/20.15.2/0-00260.md) 已配数据 |
| ICAPSERVERNAME | ICAP服务器名称 | icapserver1 | 引用 [0-00259](task/UDG/20.15.2/0-00259.md) 已配数据 |

**主场景脚本样例**

```
ADD ICAPSVRBINDISG: ICAPSVRGRPNAME="icapsvrgrp1", ICAPSERVERNAME="icapserver1";
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：CFGDOMAINNAME（配置域名称，optional，主场景不配）。

## 决策点

（本命令在所有引用特性下用法一致——组/服务器两必选，没有跨特性分叉，未建 DP。）

## 约束

- **命令记录上限**（critical，rule-0-00261-01）：命令最大记录数 **300**；单个 ICAP Server Group 实例最多绑定 **10 个** ICAP Server — 超规格被拒。
- **前置依赖 1**（critical，rule-0-00261-02）：ICAPSVRGRPNAME 必须已通过 [0-00260](task/UDG/20.15.2/0-00260.md) ADD ICAPSVRGRP 创建，否则报组不存在。
- **前置依赖 2**（critical，rule-0-00261-03）：ICAPSERVERNAME 必须已通过 [0-00259](task/UDG/20.15.2/0-00259.md) ADD ICAPSERVER 创建，否则报服务器不存在。
- **字符串约束**（critical，rule-0-00261-04）：ICAPSVRGRPNAME / ICAPSERVERNAME 长度 1～31 字符，**不支持空格**、不区分大小写。
- **生效时机**（warning，rule-0-00261-05）：命令执行后**立即生效**，新增的绑定关系立即纳入 URL 过滤模板的服务器选择范围。
