# 配置逻辑接口（ADD INTERFACE）

> 命令静态知识见 [ADD INTERFACE](command/UDG/20.15.2/ADD-INTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。逻辑接口是物理上不存在、需配置建立的接口（Loopback/Tunnel/以太网子接口等），承载 GRE/IPsec 隧道、IPv6 组网等场景。

## 配置方法（配置方法字典）

ADD INTERFACE 是"逻辑接口创建+初始化"命令。同一命令承载多类逻辑接口（LoopBack/Tunnel/以太网子接口），按接口类型决定参数出现与否。核心配置空间有 5 个维度：**接口类型**（决定哪些参数出现）、**管理状态**（IFADMINSTATUS）、**MTU**（IFMTU）、**不分片标记**（IFDF）、**统计/Trap**（IFSTATIENABLE/IFTRAPENABLE/IFSTATITVL）。4 个特性命中（010107 静态路由冗余、020402 IPv6 组网、IPFD-015002 GRE、IPFD-015004 IPsec）覆盖 LoopBack/Tunnel/Ethernet 三类接口范式。

### 配置维度 1：接口类型（参数 IFNAME，必选）

**IFNAME 决定接口类型**，不同接口类型支持的可选参数不同：

| 接口类型 | IFNAME 形态 | 典型场景 | 支持的可选参数 |
|---|---|---|---|
| **LoopBack** | `LoopBack<n>` | GRE/IPsec 本端源接口、OSPF loopback | IFADMINSTATUS、IFDESCR、IFMTU、IFDF |
| **Tunnel**（GRE/IPsec） | `Tunnel<n>` | 静态路由冗余 GRE 隧道、IPsec 隧道 | IFADMINSTATUS、IFDESCR、IFMTU、IFDF、IFSTATIENABLE、IFTRAPENABLE、IFSTATITVL |
| **以太网子接口** | `Ethernet<slot>/<card>/<port>.<vlan>` | N6/Gi/SGi IPv6 组网（VLAN 子接口） | 仅 IFNAME（IP/VLAN 由 [0-00116](task/UDG/20.15.2/0-00116.md) ADD ETHSUBIF 单独配） |

> **命名约束**：IFNAME 字符串长度 1~63；接口名由"接口类型+接口编号"组成，编号为十进制自然数；编号为 0 时填 "0"，其他情况不得以 "0" 开头。

### 配置维度 2：管理状态（参数 IFADMINSTATUS，可选）

| 取值 | 作用 | 默认值 |
|---|---|---|
| **up** | 接口 Up，立即生效 | LoopBack/Tunnel 默认 up；**Ethernet 接口默认 down** |
| **down** | 接口 Down（暂不启用） | — |

### 配置维度 3：MTU（参数 IFMTU，可选）

| 取值范围 | 默认值 | 约束 |
|---|---|---|
| 46~9600 字节 | 1500 | Loopback/Null0/GigabitEthernet0/0/1/Fabric-tunnel **MTU 不允许修改** |

> **MTU 优先级**：配置值 > DHCP 分配值 > 默认 1500；建议手工配置为规划值，避免被 DHCP 覆盖为非规划值致丢包；建议 MTU 配 1500~9000。GRE 隧道封装后出外联口不再分片，故 GRE Tunnel 取 **MTU=1500**（010107 范式）。

### 配置维度 4：不分片标记（参数 IFDF，可选）

| 取值 | 作用 | 默认值 |
|---|---|---|
| **FALSE**（默认） | 取消 IP 报文强制分片（允许分片） | FALSE |
| **TRUE** | 启用 IP 报文强制分片（DF=1，>MTU 时丢弃并发 ICMP） | — |

> IPsec/GRE Tunnel 场景常配 `IFDF=FALSE`（IPFD-015004 范式），避免协议封装导致报文超 MTU 丢弃。

### 配置维度 5：统计/Trap（参数 IFSTATIENABLE/IFTRAPENABLE/IFSTATITVL，可选）

| 参数 | 取值范围 | 默认值 | 作用 |
|---|---|---|---|
| **IFSTATIENABLE** | `NONE` / `TRUE` / `FALSE` | Ethernet/400GE/200GE/100GE/25GE/Eth-Trunk/Loopback/Fabric-Tunnel/GigabitEthernet0/0/1/NULL0 默认一直开启 | 接口接收/发送报文的统计使能 |
| **IFTRAPENABLE** | `TRUE` / `FALSE` | 默认 TRUE | 接口 Down 的告警上报使能 |
| **IFSTATITVL** | 5~600 秒（5 的倍数） | 全局流量统计间隔 | 接口流量统计时间间隔（接口配置 > 全局配置） |

**典型脚本（010107 静态路由冗余，LoopBack + Tunnel 组合）**：
```
ADD INTERFACE: IFNAME="LoopBack1", IFADMINSTATUS=up;
ADD INTERFACE: IFNAME="Tunnel1", IFMTU=1500;
```

**典型脚本（020402 IPv6 组网，以太网子接口）**：
```
ADD INTERFACE:IFNAME="Ethernet70/0/4.1";
ADD INTERFACE:IFNAME="Ethernet70/0/5.1";
```

**典型脚本（IPFD-015004 IPsec，Tunnel 全参数）**：
```
ADD INTERFACE:IFNAME="Tunnel2",IFADMINSTATUS=up,IFMTU=1500,IFDF=FALSE,IFSTATITVL=20;
```

> 命令 notes：**最大记录数 65535**；**执行后立即生效**；**版本升级过程中禁止执行**（中断升级流程）；**参与自动化配置**——下发前需 `DSP OPSASSISTSTATE` 查询 autoscaling_autoconfig.py 是否在执行，若是则禁止本业务增删改。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：创建逻辑接口"——前置无；下游由 [0-00057](task/UDG/20.15.2/0-00057.md)（ADD IPBINDVPN）/[0-00058](task/UDG/20.15.2/0-00058.md)（ADD IFIPV4ADDRESS）/[0-00060](task/UDG/20.15.2/0-00060.md)（SET IFIPV6ENABLE）/[0-00061](task/UDG/20.15.2/0-00061.md)（ADD IFIPV6ADDRESS）/[0-00059](task/UDG/20.15.2/0-00059.md)（ADD GRETUNNEL）/[0-00116](task/UDG/20.15.2/0-00116.md)（ADD ETHSUBIF）通过 source_ref 引用 IFNAME。

## 决策点：接口类型选择（DP 0-00056）

问：ADD INTERFACE 创建哪种接口？

| 选项 | IFNAME 形态 | 典型场景 |
|---|---|---|
| LoopBack 逻辑接口 | `LoopBack<n>` | GRE/IPsec 本端源接口、OSPF loopback（IPFD-015002、IPFD-015004） |
| Tunnel 逻辑接口 | `Tunnel<n>` | 静态路由冗余 GRE 隧道、IPsec 隧道（010107、IPFD-015004） |
| 以太网子接口 | `Ethernet<slot>/<card>/<port>.<vlan>` | N6/Gi/SGi IPv6 组网（020402） |

> 4 个特性命中（010107/020402/IPFD-015002/IPFD-015004）覆盖三类接口范式；本决策只分流接口类型，下游命令（IP/VPN/GRE 隧道）通过 source_ref 引用 IFNAME。

## 约束

- **最大记录数 65535**（warning，rule-0-00456）：逻辑接口最大 65535 条 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00456）：命令执行后立即生效 — 后果：误操作即刻生效
- **版本升级过程禁止执行**（critical，rule-0-00456）：版本升级过程中禁止执行本命令 — 后果：版本升级期间执行中断升级流程
- **参与自动化配置**（critical，rule-0-00456）：本命令参与自动化配置——下发前需 `DSP OPSASSISTSTATE` 查询 autoscaling_autoconfig.py — 后果：自动化配置过程中执行致最终生效配置不可预期
- **IFMTU 不可改接口**（warning，隐含）：Loopback/Null0/GigabitEthernet0/0/1/Fabric-tunnel 接口 MTU 不允许修改 — 后果：配 IFMTU 命令拒收
- **MTU 优先级**（warning，隐含）：配置值 > DHCP 分配值 > 默认 1500；建议手工配置为规划值 — 后果：MTU 被 DHCP 覆盖为非规划值致丢包
- **IFADMINSTATUS 默认值因接口类型异**（warning，隐含）：Ethernet 接口默认 down，其他支持 IFADMINSTATUS 的接口默认 up — 后果：误判接口状态
- **IFDF 默认 FALSE**（info，隐含）：IFDF 默认 FALSE（取消 IP 报文强制分片） — 后果：TRUE 时>MTU 丢弃并发 ICMP
- **IFSTATITVL 必须 5 的倍数**（warning，隐含）：统计间隔必须是 5 的倍数 — 后果：非 5 倍数命令拒收
