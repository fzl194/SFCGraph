# 配置逻辑接口（ADD INTERFACE）

> 命令静态知识见 [ADD INTERFACE](command/UDG/20.15.2/ADD-INTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

同一 ADD INTERFACE 命令承载多类逻辑接口，按接口类型决定哪些参数出现（IFADMINSTATUS/IFMTU/IFDF 互斥，由 IFNAME 类型决定）：

| 接口类型 | 必出参数 | 可选参数 | 典型场景 |
|---|---|---|---|
| LoopBack | IFNAME | IFADMINSTATUS=up | GRE/IPsec 本端源接口、OSPF loopback |
| Tunnel（GRE/IPsec） | IFNAME | IFADMINSTATUS=up, IFMTU=1500, IFDF=FALSE | 静态路由冗余 GRE 隧道、IPsec 隧道 |
| 以太网子接口 | IFNAME | —（VLAN 由 ADD ETHSUBIF 单独配） | N6/Gi/SGi IPv6 组网 |

**典型脚本（按接口类型）**
```
ADD INTERFACE:IFNAME="LoopBack1",IFADMINSTATUS=up;
ADD INTERFACE:IFNAME="Tunnel2",IFADMINSTATUS=up,IFMTU=1500,IFDF=FALSE;
ADD INTERFACE:IFNAME="Ethernet70/0/4.1";
```

**取值样例归纳（4 特性数据规划表）**

| 特性 | 接口名样例 | 联动参数 |
|---|---|---|
| GWFD-010107（静态路由冗余） | LoopBack1/2, Tunnel1/2 | IFADMINSTATUS=up（LoopBack）；IFMTU=1500（Tunnel，原则：封装 GRE 出外联口不再分片） |
| GWFD-020402（N6/Gi/SGi IPv6 组网） | Ethernet70/0/4.1, Ethernet71/0/5.1 | 仅 IFNAME（IP/VLAN 由 ADD IFIPV6ADDRESS/ADD ETHSUBIF 配） |
| IPFD-015002（GRE） | LoopBack1 | 仅 IFNAME |
| IPFD-015004（IPsec GRE/OSPF） | LoopBack1, Tunnel1/2/10/17/27 | IFADMINSTATUS=up, IFMTU=1500, IFDF=FALSE |

**IFMTU 配置原则**（GWFD-010107）：封装 GRE 隧道出外联口不会再次分片，故 GRE Tunnel 取 1500。

**下游引用**：本 atom 的 IFNAME 被 [0-00057](task/UDG/20.15.2/0-00057.md)（ADD IPBINDVPN）/[0-00058](task/UDG/20.15.2/0-00058.md)（ADD IFIPV4ADDRESS）/[0-00060](task/UDG/20.15.2/0-00060.md)（SET IFIPV6ENABLE）/[0-00061](task/UDG/20.15.2/0-00061.md)（ADD IFIPV6ADDRESS）/[0-00059](task/UDG/20.15.2/0-00059.md)（ADD GRETUNNEL）通过 source_ref 引用。

**未绑参数**：IFDESCR / IFSTATIENABLE / IFTRAPENABLE / IFSTATITVL（命令真相 optional，多数特性未用）。

## 决策点：接口名（IFNAME）规划来源（DP 0-00070）

问：ADD INTERFACE 的 IFNAME 接口名采用何种规划来源？（接口编号语义因接口类型/场景而异。）

| 选项 | 取值来源 | 适用场景 |
|---|---|---|
| 全局规划（默认） | variable_source=global_planned | LoopBack/Tunnel 逻辑接口名跨隧道两端/网元协调（010107 静态路由冗余） |
| 本端规划 | variable_source=local_planned | 以太网子接口/物理衍生接口由本端板位/VLAN 决定（020402 N6/Gi、IPFD LoopBack） |
| 引用已配置数据 | variable_source=reference | 物理接口名经 LST INTERFACE 查询后引用（020402 部分 Ethernet） |

注：该决策仅分流取值来源口径，不影响下游命令（下游通过 source_ref 引用本 atom 的 IFNAME，与来源口径无关）。
