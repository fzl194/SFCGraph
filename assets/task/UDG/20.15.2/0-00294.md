# 配置流过滤器组（ADD FLOWFILTERGRP）

> 命令静态知识见 [ADD FLOWFILTERGRP](command/UDG/20.15.2/ADD-FLOWFILTERGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD FLOWFILTERGRP 是 ADD FLOWFILTER 的可选升级形态：当需要**多个流过滤条件同时存在**时，把若干 FLOWFILTER 聚合进一个流过滤器组，按逻辑分组（AND/OR/NOT）组合匹配。RULE 的 `FILTERINGMODE=FLOWFILTERGRP` 引用流过滤器组（替代默认 `FILTERINGMODE=FLOWFILTER`）。**activation 默认演示 FILTERINGMODE=FLOWFILTER（单流过滤器），流过滤器组为可选变体**（重定向族 activation §操作步骤 RULE 说明、计费族/头增强族同此）。

**主场景范式（命令 wiki 实例：组内增加一个流过滤器）**

| 字段 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| FLWFLTRGRPNAME | testflowfiltergrp | 本端规划 | 流过滤器组名，1~63 字符，不区分大小写；产出对象被 RULE/BLACKLISTRULE 的 FLWFLTRGRPNAME 引用 |
| FLOWFILTERNAME | testflowfilter | 已配置数据中获取 | 引用 [0-00007](task/UDG/20.15.2/0-00007.md) FLOWFILTER；多次执行加入多个流过滤器 |
| FLWFLTRLOGICGRP | OR | 本端规划 | 逻辑分组 AND/OR/NOT，默认 OR；外置规则库属性流过滤器只能 OR |

**脚本样例**
```
ADD FLOWFILTERGRP:FLWFLTRGRPNAME="testflowfiltergrp",FLOWFILTERNAME="testflowfilter";
```

**逻辑分组语义**：一条流**同时命中组内 AND/OR/NOT 三个逻辑分组**才算命中该流过滤器组——
- AND：命中与组内所有流过滤器 → 命中与逻辑分组
- OR：命中或组内任一流过滤器 → 命中或逻辑分组（默认）
- NOT：未命中非组内任何流过滤器 → 命中非逻辑分组

**配置链路**：先配 FLOWFILTER（[0-00007](task/UDG/20.15.2/0-00007.md)）→ **ADD FLOWFILTERGRP**（聚合多 FLOWFILTER）→ RULE `FILTERINGMODE=FLOWFILTERGRP, FLWFLTRGRPNAME=<组名>`。

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| FILTERINGMODE=FLOWFILTER（默认） | 单流过滤器，RULE.FLOWFILTERNAME 引用 [0-00007](task/UDG/20.15.2/0-00007.md)（activation 演示） |
| FILTERINGMODE=FLOWFILTERGRP（可选） | 流过滤器组，RULE.FLWFLTRGRPNAME 引用本 atom；支持多条件 AND/OR/NOT 组合 |
| 逻辑分组 AND | 命中组内所有流过滤器 |
| 逻辑分组 OR（默认） | 命中组内任一流过滤器 |
| 逻辑分组 NOT | 未命中组内任何流过滤器 |

## 约束

- **仅对新激活用户生效**（warning）：命令执行后只对新激活用户生效，不影响已在线用户 — 后果：已在线用户不立即应用新组。
- **空组无流命中**（critical）：流过滤器组未配置任何逻辑分组的流过滤器，则没有流能命中该组 — 后果：规则匹配失效。
- **逻辑分组缺省按默认组**（warning）：某逻辑分组未配置流过滤器时，按该分组默认语义处理（AND/OR 默认所有流命中，NOT 默认所有流命中）。
- **性能影响**（info）：配置流过滤器组数量较多时对性能有一定影响，建议共用已存在配置。
- **规格**（info）：整机支持 100000 个流过滤器组；单组最多 128 个流过滤器；整机最多 200000 条组-流过滤器绑定关系。
