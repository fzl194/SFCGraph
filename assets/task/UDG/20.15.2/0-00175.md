# 设置基于APN的下行数据长时间缓存配置（SET APNDLLTBUFFER）

> 命令静态知识见 [SET APNDLLTBUFFER](command/UDG/20.15.2/SET-APNDLLTBUFFER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。属 eDRX（NB-IoT/eMTC/5G）的 APN 级下行长时间缓存，**优先级高于全局** [0-00173](task/UDG/20.15.2/0-00173.md) SET GLBDLLTBUFFER；APN 引用 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 已建实例。

## 配置方法（配置方法字典）

SET APNDLLTBUFFER 是"基于 APN 的下行数据长时间缓存配置"命令。3 参（APN/MAXBUFFERNUM/BUFFERMODE）必填，配置空间有 3 个维度：**储存方式**（BUFFERMODE，决定满规格时的丢弃策略）、**最大缓存个数**（MAXBUFFERNUM，受 License 限制）、**APN 实例**（引 ADD APN 已建）。3 个特性命中（GWFD-110601 NB-IoT eDRX / GWFD-110641 eMTC eDRX / GWFD-110661 5G eDRX）共用本 atom。

### 配置维度 1：储存方式（参数 BUFFERMODE，必选）

| 取值 | 作用 | 适用特性 |
|---|---|---|
| **BUFF_MODE_RING**（默认） | 环形存储：满规格时丢弃最先存储的报文，缓存当前报文 | NB-IoT eDRX（GWFD-110601）、5G eDRX（GWFD-110661） |
| **BUFF_MODE_CHAIN** | 链式存储：满规格时丢弃当前收到的报文（按到达顺序存放） | 仅 eMTC eDRX（GWFD-110641） |

### 配置维度 2：最大缓存个数（参数 MAXBUFFERNUM，必选）

| 取值范围 | 默认 | License 约束 | 实际生效值 |
|---|---|---|---|
| 整数 1~64 | 1（系统初始值，UDG 不配时） | 受"82200CKC LKV3..."License 控制项限制 | min(本参数, SMF/N4 下发 DL Buffering Packet Count) |

> 3 个特性命中（110601/110641/110661）均取 MAXBUFFERNUM=4。**License 控制项未打开时，配置会失败**。

### 配置维度 3：APN 实例（参数 APN，必选）

| 取值 | 来源 |
|---|---|
| 字符串 1~63 | 引用 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 已配置的 APN 实例名（可用 LST APN 查询） |

> APN 必须先经 ADD APN 配置——未配的 APN 命令拒收。

**典型脚本（GWFD-110601 NB-IoT eDRX，RING）**：
```
SET APNDLLTBUFFER: APN="apnnb", MAXBUFFERNUM=4, BUFFERMODE=BUFF_MODE_RING;
```

**典型脚本（GWFD-110641 eMTC eDRX，CHAIN）**：
```
SET APNDLLTBUFFER: APN="apnemtc", MAXBUFFERNUM=4, BUFFERMODE=BUFF_MODE_CHAIN;
```

**典型脚本（GWFD-110661 5G eDRX，RING）**：
```
SET APNDLLTBUFFER: APN="apn-test", MAXBUFFERNUM=4, BUFFERMODE=BUFF_MODE_RING;
```

> 命令 notes：**单命令最大记录数 10000**；**执行后立即生效**（对新数据流生效，不影响已下行数据）；**前置**下行缓存时长（[0-00174](task/UDG/20.15.2/0-00174.md) SET APNDLBUFTIME）必须 >15s 才生效（<15s 默认缓存 128 个，本参数不生效）；本 APN 级配置**优先级高于全局** [0-00173](task/UDG/20.15.2/0-00173.md) SET GLBDLLTBUFFER。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：配置 APN 级下行数据长时间缓存"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN（建 APN）+ [0-00174](task/UDG/20.15.2/0-00174.md) SET APNDLBUFTIME（缓存时长 >15s）；与 [0-00173](task/UDG/20.15.2/0-00173.md) SET GLBDLLTBUFFER（全局级，优先级低）共同形成两层覆盖。

## 决策点：APN级长时间缓存储存方式（DP 0-00175）

问：SET APNDLLTBUFFER 的 BUFFERMODE 选哪种？（3 个 eDRX 特性跨特性分叉）

| 选项 | BUFFERMODE | 适用特性 | 行为 |
|---|---|---|---|
| **环形 RING**（默认） | BUFF_MODE_RING | GWFD-110601 NB-IoT eDRX、GWFD-110661 5G eDRX | 满规格时丢弃最旧报文，缓存当前报文 |
| **链式 CHAIN** | BUFF_MODE_CHAIN | GWFD-110641 eMTC eDRX | 满规格时丢弃当前报文，按到达顺序存放 |

> 3 个特性命中（110601= RING / 110641= CHAIN / 110661= RING）。系统初始值 RING。

## 约束

- **APN 必须先经 ADD APN 配置**（critical，rule-0-00175）：APN 实例必须先由 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 定义 — 后果：未配 APN 命令拒收
- **前置缓存时长必须 >15s**（critical，rule-0-00175）：执行 SET APNDLBUFTIME 使缓存时长 >15s（<15s 默认 128 个，本参数不生效） — 后果：时长不足本参数配置无效
- **APN 优先级高于全局**（critical，rule-0-00175）：APN 级配置覆盖全局 [0-00173](task/UDG/20.15.2/0-00173.md) SET GLBDLLTBUFFER — 后果：APN 配置会屏蔽全局同 APN 配置
- **License 控制项约束**（critical，rule-0-00175）：MAXBUFFERNUM 受"82200CKC LKV3..."License 控制项限制（License 项未打开时配置失败） — 后果：License 不足配置失败
- **执行后立即生效**（warning，rule-0-00175）：执行后立即生效，但仅对新数据流生效 — 后果：已下行数据不受影响，误操作即刻生效
- **单命令最大记录数 10000**（info，rule-0-00175）：单命令最大记录数 10000 — 后果：超规格新增失败
- **MAXBUFFERNUM 取值范围**（warning，rule-0-00175）：MAXBUFFERNUM 取值 1~64 — 后果：超出范围命令拒收
- **实际生效值取较小值**（info，rule-0-00175）：实际缓存个数 = min(MAXBUFFERNUM, SMF/N4 下发 DL Buffering Packet Count)；SMF 未下发时用本参数 — 后果：实际生效受 SMF 影响
