# ICAP Server 互通配置

> URL/内容过滤的**前置**（多命令可复用模块，内容过滤域通用）。UDG 经 ICAP 协议把 URL 发给外部 ICAP Server 分析，返回分类/动作。被引用于 [2-00002](task/UDG/20.15.2/2-00002.md)（URL过滤）。

## 配置方法

ICAP 互通六步链：VPN 实例 → 逻辑接口 → ICAP 服务器 → 本端信息 → 服务器组 → 绑定。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| VPN 实例 | ADD VPNINST | VPNINSTANCE（ICAP 互通专网） |
| 逻辑接口 | ADD LOGICINF | NAME + IPVERSION/IPV4ADDRESS + VPNINSTANCE（绑上一步） |
| ICAP 服务器 | ADD ICAPSERVER | ICAPSERVERNAME + ICAPSERVERTYPE=URL_FILTERING + ICAPSERVERIPV4 + VPNINSTANCE |
| 本端信息 | ADD ICAPLOCALINFO | ICAPSERVERTYPE=URL_FILTERING + USERAGENT |
| 服务器组 | ADD ICAPSVRGRP | ICAPSVRGRPNAME + ICAPSERVERTYPE=URL_FILTERING |
| 绑定 | ADD ICAPSVRBINDISG | ICAPSVRGRPNAME + ICAPSERVERNAME |

**典型脚本**：
```
ADD VPNINST:VPNINSTANCE="vpn-gcfif";
ADD LOGICINF:NAME="gcfif1/0/0",IPVERSION=IPV4,IPV4ADDRESS1="192.168.1.1",IPV4MASK1="255.255.255.255",VPNINSTANCE="vpn-gcfif";
ADD ICAPSERVER:ICAPSERVERNAME="icapserver1",ICAPSERVERTYPE=URL_FILTERING,ICAPSVRIPTYPE=IPV4,ICAPSERVERIPV4="172.16.39.136",VPNINSTANCE="vpn-gcfif";
ADD ICAPLOCALINFO:ICAPSERVERTYPE=URL_FILTERING,USERAGENT="test";
ADD ICAPSVRGRP:ICAPSVRGRPNAME="icapsvrgrp1",ICAPSERVERTYPE=URL_FILTERING;
ADD ICAPSVRBINDISG:ICAPSVRGRPNAME="icapsvrgrp1",ICAPSERVERNAME="icapserver1";
```

**步骤位置**：URL/内容过滤的**最前置**（先于 License + 业务配置）；产出的 ICAPSVRGRPNAME 供下游 CFTEMPLATE 引用。

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| ICAP 部署模式—单机 | 1 个 ICAPSERVER 绑 1 个 ICAPSVRGRP |
| ICAP 部署模式—主备高可用 | 多个 ICAPSERVER 绑同一 ICAPSVRGRP |

## 约束

- **ICAPSERVERTYPE 恒定**（info）：恒为 `URL_FILTERING`（本特性的 ICAP 类型） — 改为其他类型则不属于本特性。
- **VPNINSTANCE 一致**（critical）：LOGICINF/ICAPSERVER 的 VPNINSTANCE 须与 VPNINST 一致 — 不一致则互通失败。
- **规格上限**（warning）：整机 ≤30 ICAP Server，每组 ≤10 — 超规格则配置失败。
