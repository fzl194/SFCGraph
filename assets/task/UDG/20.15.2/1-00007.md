# 分级带宽管理

> BWM 配置的场景特化阶段（仅分级带宽场景，增强版 20.9.0+）。归属特性 [2-00001](task/UDG/20.15.2/2-00001.md)。

## 配置方法

三级联动：控制器设业务等级规格 → 业务级别策略分配比例 → 规则指定级别 + 用户组启用组级规则。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 控制器等级规格 | ADD BWMCONTROLLER | SRVLEVELSPEC=1~15（见 [1-00002](task/UDG/20.15.2/1-00002.md)） |
| 业务级别策略 | ADD BCSRVLEVELPLY | BWMCNAME + SERVICELEVEL + CIRRATE + PIRRATE |
| 用户组启用 | ADD BWMUSERGROUP | USERGLEVRULESW=ENABLE（见 [1-00003](task/UDG/20.15.2/1-00003.md)） |
| 规则指定级别 | ADD BWMRULE | SERVICELEVEL=1~15 |

**典型脚本**：
```
ADD BWMCONTROLLER:BWMCNAME="bc_test",CTRLTYPE=CAR,CIR=300,CBS=60000,PIR=500,PBS=100000,GREEN=PASS,YELLOW=PASS,RED=DISCARD,SRVLEVELSPEC=3;
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=1,CIRRATE=50,PIRRATE=70;
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=2,CIRRATE=30,PIRRATE=50;
ADD BCSRVLEVELPLY:BWMCNAME="bc_test",SERVICELEVEL=3,CIRRATE=20,PIRRATE=100;
```

**步骤位置**：控制器等级规格在 [1-00002](task/UDG/20.15.2/1-00002.md)（加 SRVLEVELSPEC）；BCSRVLEVELPLY 在控制器之后、规则之前；规则 SERVICELEVEL 在 [1-00003](task/UDG/20.15.2/1-00003.md)。

## 决策点

**DP：ServiceLevel 分配** —— 各级别 CIRRATE/PIRRATE 比例规划（高优→低优递减，如 50/30/20）。
**DP（联动）：业务级别数** —— 控制器 `SRVLEVELSPEC` 须 ≥ 用到的 SERVICELEVEL 数。

## 约束

- **CIRRATE 之和 ≤ 100%（critical）**：所有 ServiceLevel 的 CIRRATE 之和不能超 100%，否则配置失败。
- SRVLEVELSPEC 最大 15 级。
- 高优先级业务优先获带宽，需求低时剩余带宽共享给低优先级。
