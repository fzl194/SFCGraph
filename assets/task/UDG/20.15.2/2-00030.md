# 基于上下行解耦的视频承载信令控制（GWFD-110302）

> 特性静态知识见 [GWFD-110302](feature/UDG/20.15.2/GWFD-110302.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性复用 [1-00017](task/UDG/20.15.2/1-00017.md) QoS专有承载链，差异在 `QOSPROP.DECOUPLINGSW=ENABLE`（上下行解耦开关），与 [2-00029](task/UDG/20.15.2/2-00029.md) 业务触发QoS保证同族。

## 配置概览

基于上下行解耦的视频承载信令控制：UDG 识别特定视频业务，对其**下行链路**建立专有承载/专有 QoS Flow。业务上行报文使用缺省承载/缺省 QoS Flow 转发，下行报文使用专有承载/专有 QoS Flow 转发（无线侧基于 QoS 参数提供差异化调度）。

与 [2-00029](task/UDG/20.15.2/2-00029.md) 业务触发QoS保证的核心差异：**QOSPROP 打开 DECOUPLINGSW=ENABLE（上下行解耦开关）**——这是本特性的硬性要求（activation §数据规划表明示"必须配置为ENABLE"）。专有承载触发不依赖 ADD SADEDICBEARER，而是由 DECOUPLINGSW + 规则匹配驱动。

核心对象链：`URR(QOS) + QOSPROP(QOSURRNAME + DECOUPLINGSW=ENABLE)` + `PCCPOLICYGRP(URRGROUPNAME+QOSPROPNAME)` → 过滤链（三四层 + PROTBINDFLOWF）→ `RULE(PCC) + USERPROFILE + RULEBINDING`。

## 配置流程（步骤编排，每步显示含命令）

> activation 共 7 步（[激活基于上下行解耦的视频承载信令控制](evidence/UDG/20.15.2/激活基于上下行解耦的视频承载信令控制_62146691.md)）。

1. **License**（单命令）：`SET LICENSESWITCH`（`LICITEM=LKV3G5SCUD01,SWITCH=ENABLE`）→ [0-00019](task/UDG/20.15.2/0-00019.md)
   - 注：License 控制项为 `LKV3G5SCUD01`（与 [2-00029](task/UDG/20.15.2/2-00029.md) 的 `LKV3G5STQE01` 不同）
2. **过滤链**（三四层 + 七层协议绑定）：`ADD FILTER`（三四层 any）+ `ADD FLOWFILTER` + `ADD FLTBINDFLOWF`（三四层绑定）+ `ADD PROTBINDFLOWF`（**PROTOCOLNAME=example_video，无 L7FILTERNAME**）→ [1-00009](task/UDG/20.15.2/1-00009.md)
   - **差异（待回填 [1-00009](task/UDG/20.15.2/1-00009.md) 场景差异）**：activation 的 PROTBINDFLOWF **未配 L7FILTERNAME**（仅 FLOWFILTERNAME + PROTOCOLNAME=example_video）。与 020358 七层触发（PROTBINDFLOWF 含 L7FILTERNAME=l7_test）不同——本特性按"协议名"绑定，未显式建 L7FILTER 对象。此差异待主流程统一回填 1-00009 场景差异（本任务不改 compound）。
3. **刷新生效**（单命令，承接过滤链置生效）：`SET REFRESHSRV`（`REFRESHTYPE=USERPROFILE`）→ [0-00015](task/UDG/20.15.2/0-00015.md)
4. **QoS 属性**（绑 QoS URR + **DECOUPLINGSW=ENABLE**）：`ADD URR`（`URRNAME=urr_1 + URRID=1 + USAGERPTMODE=QOS`）+ `ADD QOSPROP`（`QOSPROPNAME=qos-property1 + QOSURRNAME=urr_1 + DECOUPLINGSW=ENABLE`）→ [1-00017](task/UDG/20.15.2/1-00017.md)（QOSPROP 段·视频解耦变体）
   - **★与 1-00017 的关键差异（待回填 compound 场景差异）**：QOSPROP **不含 GBR/MBR 四档比特率**，仅 `QOSURRNAME + DECOUPLINGSW=ENABLE`；activation §数据规划表 QOSPROP 段仅有 QOSPROPNAME/QOSURRNAME/DECOUPLINGSW 三参数，无 GBRUPLKVALUE/GBRDNLKVALUE/MBRUPLKVALUE/MBRDNLKVALUE。此差异待主流程统一回填 1-00017 场景差异。
5. **PCC 策略组**（计费三件套 QoS 变体，绑 URRGROUP+QOSPROPNAME）：`ADD PCCPOLICYGRP`（`PCCPOLICYGRPNM=ppg1 + QOSPROPNAME=qos-property1 + URRGROUPNAME=urrgp_test`）→ [1-00010](task/UDG/20.15.2/1-00010.md)
   - 注：activation 脚本注释"当需要计费时，请配置参数 URRGROUPNAME"——URRGROUPNAME 引用已配的 URR 组（activation 数据规划表标"已配置数据中获取"，脚本无显式 ADD URRGROUP，为继承配置）。
6. **规则与用户模板绑定**：`ADD RULE`（`RULENAME=rule_test + POLICYTYPE=PCC + FILTERINGMODE=FLOWFILTER + FLOWFILTERNAME=flowfilter_test1 + POLICYNAME=ppg1 + PRIORITY=10`）+ `ADD USERPROFILE`（`USERPROFILENAME=up_test`）+ `ADD RULEBINDING`（`USERPROFILENAME=up_test + RULENAME=rule_test`）→ [1-00011](task/UDG/20.15.2/1-00011.md)

> **与 1-00017 QoS专有承载链命令集差异**（本任务不改 compound，仅标注待回填）：
> 1. **无 ADD SADEDICBEARER**：1-00017 含 SADEDICBEARER（TRIGGERMODE），本特性 activation 无此命令——专有承载触发由 QOSPROP.DECOUPLINGSW=ENABLE + 规则匹配驱动，不经 SA 专有承载命令。
> 2. **QOSPROP 参数集不同**：1-00017 典型脚本 QOSPROP 含 GBR/MBR 四档；本特性 QOSPROP 仅 QOSURRNAME + DECOUPLINGSW=ENABLE（无 GBR/MBR）。
> 3. **无 ADD WELLKNOWNPORT**：1-00017 场景差异列 WELLKNOWNPORT（020358 七层用）；本特性 activation 无。
> 4. **PROTBINDFLOWF 无 L7FILTERNAME**：1-00009 过滤链七层绑定典型含 L7FILTERNAME；本特性 PROTBINDFLOWF 仅 PROTOCOLNAME（无 L7FILTERNAME）。
>
> 以上差异属 1-00017/1-00009 场景差异的"待补条目"，主流程统一回填（本任务不改 compound，仅在本 feature 标注）。

## 决策点

**DP-1（特性级）：是否启用计费 + URRGROUPNAME 绑定**（影响 PCCPOLICYGRP 是否绑 URRGROUPNAME）。

| 场景 | PCCPOLICYGRP.URRGROUPNAME | 走法 |
|---|---|---|
| **纯 QoS 保证（不计费，activation 演示注释）** | 可省（activation 脚本注释"当需要计费时，请配置参数 URRGROUPNAME"）| 骨架 + [1-00010](task/UDG/20.15.2/1-00010.md)（PCCPOLICYGRP 仅绑 QOSPROPNAME，URRGROUPNAME 可选）|
| 计费变体 | 绑 URRGROUPNAME（引用已配 URRGROUP）| 骨架 + [1-00010](task/UDG/20.15.2/1-00010.md)（PCCPOLICYGRP 绑 QOSPROPNAME + URRGROUPNAME）|

> 注：activation 数据规划表 PCCPOLICYGRP 段含 URRGROUPNAME=urrgp_test（"已配置数据中获取"），脚本含 URRGROUPNAME="urrgp_test"——两处一致，实际演示为"配 URRGROUPNAME"。脚本注释"当需要计费时"为通用说明（PCCPOLICYGRP 不计费时可省 URRGROUPNAME）。

**联动**：
- DECOUPLINGSW=ENABLE → QOSPROP 硬性要求（本特性固定 ENABLE，非选项）。
- URRGROUPNAME → 仅计费时绑（activation 演示绑定，注释明示可选）。

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5SCUD01` — 未开则视频承载信令控制功能不生效。
- **依赖 SA-Basic + PCC**（critical）：[GWFD-110101](feature/UDG/20.15.2/GWFD-110101.md) SA-Basic（识别视频业务）+ [GWFD-020351](feature/UDG/20.15.2/GWFD-020351.md) PCC基本功能（从 PCRF/PCF 获取规则）— 缺一则不工作。
- **DECOUPLINGSW 必须 ENABLE**（critical）：ADD QOSPROP 须 `DECOUPLINGSW=ENABLE`（activation §数据规划表明示"基于上下行解耦的视频差异化调度功能开关，必须配置为ENABLE"）— 非 ENABLE 则上下行解耦不生效，特性功能失效。
- **QoS URR 上报模式固定 QOS**（critical）：ADD URR 须 `USAGERPTMODE=QOS`（固定取值）— 非 QOS 则 QoS 事件上报失效。
- **QOSPROP.QOSURRNAME 引用一致**（critical）：QOSURRNAME 须引用已配的 QoS URR（USAGERPTMODE=QOS）— 不一致则 QoS 属性引用断链。
- **PCCPOLICYGRP 绑 QOSPROPNAME**（critical）：PCCPOLICYGRP 须绑 QOSPROPNAME（指向含 DECOUPLINGSW=ENABLE 的 QOSPROP）— 漏绑则上下行解耦不生效。
- **PROTBINDFLOWF.PROTOCOLNAME 须为可识别视频协议**（warning）：PROTBINDFLOWF 绑定的 PROTOCOLNAME（activation 演示 example_video）须为 SA 可识别的视频协议 — 未识别则规则匹配失效。
- **并发链路流 ≤ 8**（info）：为避免信令风暴，只有并发链路流小于 8 个协议或协议组才适合配置（特性 wiki §应用限制）。
- **仅支持 HTTP1.x**（info）：本特性不支持 HTTP2.0/HTTP3.0 协议（特性 wiki §应用限制）。
- **互斥特性**（info）：与 [GWFD-110201](feature/UDG/20.15.2/GWFD-110201.md) HTTP2.0 Host识别、[GWFD-110203](feature/UDG/20.15.2/GWFD-110203.md) HTTPS业务地址识别、[GWFD-110251](feature/UDG/20.15.2/GWFD-110251.md)、[GWFD-110252](feature/UDG/20.15.2/GWFD-110252.md) 不能同时开启（HTTP2.0 加密报文不支持）。
