# 配置IPsec微服务接口IPv4地址（ADD IFIPV4ADDRESSIPSEC）

> 命令静态知识见 [ADD IFIPV4ADDRESSIPSEC](command/UDG/20.15.2/ADD-IFIPV4ADDRESSIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是 IPsec 微服务侧"VPN + 隧道接口"双配的最后生效点——必须先由 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 建 VPN → [0-00212](#) ADD VPNINSTAFIPSEC 加地址族 → [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC 建接口 → [0-00306](task/UDG/20.15.2/0-00306.md) ADD IPBINDVPNIPSEC 接口绑 VPN，再下发本命令给接口配 IPv4 地址。**双配铁律**：IFNAME/IFIPADDR/SUBNETMASK 必须与 VNRS 侧 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE / [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS **一对一一致**，否则 IPsec 隧道不通。

## 配置方法（配置方法字典）

ADD IFIPV4ADDRESSIPSEC 是"IPsec 微服务侧接口 IPv4 地址配置"命令。4 个参数中 3 必选（IFNAME + SUBNETMASK + IFIPADDR）+ 1 可选（ADDRTYPE）。配置空间有 4 个维度：**地址类型**（ADDRTYPE Main/Sub/Brwd/Brw）、**接口**（IFNAME 引用已建 Tunnel/LoopBack）、**掩码**（SUBNETMASK 几乎固定 255.255.255.255）、**IPv4 地址**（IFIPADDR 与 VNRS 侧一致）。1 个特性命中（IPFD-015004 普通 IPv4 IPsec 隧道，含 GRE over IPsec、IPv4 主备、OSPF over IPsec、指定本端接口等子场景）覆盖典型范式。

### 配置维度 1：地址类型（参数 ADDRTYPE，可选，默认 Main）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **Main**（默认） | 主地址——每个主用接口 1 个主地址 + 最多 255 个从地址 | IPFD-015004 范式默认（所有命中脚本均显式或隐式传 ADDRTYPE=Main） |
| **Sub** | 从地址 | 多 IP 场景（实际范例未见） |
| **Brwd**（被借用地址） | 被其他接口借用 | LoopBack 借用场景（实际范例未见） |
| **Brw**（借用地址） | 借用其他接口 | Tunnel 接口借用 LoopBack IP（实际范例未见） |

> **数量铁律**：每个主用接口**最大允许配置 256 个接口 IP**——1 个主地址 + 255 个从地址。

### 配置维度 2：接口（参数 IFNAME，必选，引用上游）

| 取值形式 | 来源 | 双配对应 |
|---|---|---|
| **字符串 1~63（接口类型+接口编号）** | 引用 [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC:IFNAME（Tunnel / LoopBack） | 须与 VNRS 侧 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 的 IFNAME 一致 |

> **接口类型说明**：Tunnel 接口（`Tunnel10` / `Tunnel2` / `Tunnel17`）用于 IPsec 隧道；LoopBack 接口（`LoopBack1`）用于本端源 IP 标识。两种接口均可在 IPsec 场景下配 IPv4 地址——Tunnel 接口配隧道 IP，LoopBack 接口配本端源 IP。
>
> **接口铁律**：必须先经 [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC 建好接口；可由 [LST INTERFACEIPSEC](command/UDG/20.15.2/LST-INTERFACEIPSEC.md) 查询可用接口。

### 配置维度 3：掩码（参数 SUBNETMASK，必选）

| 取值形式 | 范围 | 适用场景 |
|---|---|---|
| **255.255.255.255** | IPv4 子网掩码 | IPFD-015004 范式几乎全配 /32（所有命中脚本均为 255.255.255.255） |
| **A/B/C 类地址掩码** | 0.0.0.0～255.255.255.255 | 网段化地址（非 IPsec 范式主用） |

> **掩码铁律**：除 Loopback 接口外，**其它接口不能配置 255.255.255.255 的掩码**——但 IPsec 微服务侧 Tunnel 接口配 /32 在 IPFD-015004 范式中为合法（与 Loopback 接口同等对待）。

### 配置维度 4：IPv4 地址（参数 IFIPADDR，必选）

| 取值形式 | 范围 | 双配对应 |
|---|---|---|
| **IPv4 地址** | 0.0.0.0～255.255.255.255 | 与 VNRS 侧 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS:IFIPADDR 一致 |

> **地址合法性铁律**：IPv4 地址**不能配 0.0.0.0 / 255.255.255.255 / 环回地址 127.0.0.1**；IPv4 地址必须是 **A/B/C 类地址**；IPv4 地址在系统内**必须唯一**——三铁律同时满足命令才接收。

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，Tunnel 接口配主地址）**：
```
ADD IFIPV4ADDRESSIPSEC:IFNAME="Tunnel2",IFIPADDR="10.2.2.1",SUBNETMASK="255.255.255.255",ADDRTYPE=Main;
```

**典型脚本（IPFD-015004 LoopBack 配主地址，本端源 IP）**：
```
ADD IFIPV4ADDRESSIPSEC:IFNAME="LoopBack1",IFIPADDR="10.3.3.3",SUBNETMASK="255.255.255.255",ADDRTYPE=Main;
```

**典型脚本（IPFD-015004 不带 ADDRTYPE 参数走默认 Main）**：
```
ADD IFIPV4ADDRESSIPSEC:IFNAME="Tunnel10", IFIPADDR="192.168.1.1", SUBNETMASK="255.255.255.255";
```

> 命令 notes：**执行后立即生效**；**该命令在版本升级过程中禁止执行**；**最多可输入 65535 条记录**；**权限**：G_1 管理员 / G_2 操作员命令组；**该接口需要支持配置 IP 地址**——前置需 [0-00305](task/UDG/20.15.2/0-00305.md) 建接口，[0-00306](task/UDG/20.15.2/0-00306.md) 绑 VPN。

**步骤位置**：在所属 feature 编排中位于"步骤 5：IPsec 微服务侧五件套最后生效——给接口配 IPv4 地址"——前置 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC（建 VPN）+ [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC（建接口）+ [0-00306](task/UDG/20.15.2/0-00306.md) ADD IPBINDVPNIPSEC（接口绑 VPN）；下游 IPsec 协商链：[0-00211](task/UDG/20.15.2/0-00211.md) ACL → [0-00212](task/UDG/20.15.2/0-00212.md) Proposal → [0-00213](task/UDG/20.15.2/0-00213.md) IKE Peer → [0-00215](task/UDG/20.15.2/0-00215.md) IPsec Policy → [0-00307](task/UDG/20.15.2/0-00307.md) IPsec Policy→Tunnel 接口绑定。

## 决策点：IPsec 子场景选择（DP 0-00304）

问：本 IPsec 隧道接口属于哪种 IPsec 子场景？（决定 IFNAME 类型 Tunnel/LoopBack + IFIPADDR 取值 + 是否显式传 ADDRTYPE）

| 选项 | IFNAME | SUBNETMASK | ADDRTYPE | 典型特征 |
|---|---|---|---|---|
| 普通 IPv4 IPsec 隧道（10002） | `Tunnel10` 等 | 255.255.255.255 | Main（默认） | 单 Tunnel 单 IP，Tunnel 接口配隧道 IP |
| LoopBack 本端源（10009 指定本端接口） | `LoopBack1` | 255.255.255.255 | Main（默认） | LoopBack 接口配本端源 IP |
| IPv4 IPsec 主备隧道（10004） | `Tunnel10` + `Tunnel17` 各配一条 | 255.255.255.255 | Main（默认） | 双 Tunnel 双 IP，主备协商 |
| GRE over IPsec（10006） | `Tunnel2`（GRE 隧道）+ `LoopBack1` | 255.255.255.255 | Main（默认） | 内层 GRE 隧道 + 外层 IPsec 加密 |
| OSPF over IPsec（10007） | `Tunnel2` | 255.255.255.255 | Main（默认） | 动态路由协议 over IPsec |

> **单 IP 子场景（如 10003 IPv6）不适用**——IPv6 隧道接口由 ADD IFIPV6ADDRESSIPSEC（未纳入本批）配；本命令仅服务 IPv4 链路。

## 约束

- **最多 65535 条记录**（warning，rule-0-01304）：系统最多可输入 65535 条接口 IPv4 地址记录 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-01304）：命令执行后立即生效 — 后果：误操作即刻生效
- **版本升级过程中禁止执行**（critical，rule-0-01304）：版本升级过程中禁止执行本命令 — 后果：升级过程中执行可能破坏系统状态
- **每个主用接口最大 256 个 IP**（warning，rule-0-01304）：每个主用接口最大允许 1 个主地址 + 255 个从地址，共 256 — 后果：超数新增失败
- **非 Loopback 不能配 /32**（critical，rule-0-01304）：除 Loopback 接口外，其它接口不能配置 255.255.255.255 掩码（IPsec Tunnel 接口按 Loopback 同等对待） — 后果：非 Loopback 接口配 /32 命令拒收
- **接口必须支持配置 IP**（critical，rule-0-01304）：目标接口必须支持配置 IP 地址 — 后果：接口不支持命令拒收
- **IPv4 地址合法性**（critical，rule-0-01304）：IFIPADDR 不能为 0.0.0.0 / 255.255.255.255 / 环回地址 127.0.0.1，必须是 A/B/C 类地址 — 后果：非法地址命令拒收
- **IPv4 地址系统内唯一**（critical，rule-0-01304）：IPv4 地址在系统内必须唯一 — 后果：地址重复命令拒收
- **IFNAME 长度**（info，rule-0-01304）：字符串 1~63（接口类型+接口编号） — 后果：超长命令拒收
- **双配铁律：IFNAME 与 VNRS 侧一致**（critical，rule-0-01304）：IFNAME 必须与 [0-00056](task/UDG/20.15.2/0-00056.md) VNRS 侧 ADD INTERFACE 的 IFNAME 一致 — 后果：IFNAME 不一致 IPsec 隧道不通
- **双配铁律：IFIPADDR 与 VNRS 侧一致**（critical，rule-0-01304）：IFIPADDR 必须与 [0-00058](task/UDG/20.15.2/0-00058.md) VNRS 侧 ADD IFIPV4ADDRESS 的 IFIPADDR 一致 — 后果：IFIPADDR 不一致 IPsec 隧道不通
- **必须先建接口**（critical，rule-0-01304）：IFNAME 必须先经 [0-00305](task/UDG/20.15.2/0-00305.md) ADD INTERFACEIPSEC 创建 — 后果：未建接口命令拒收
- **建议先绑 VPN**（warning，rule-0-01304）：典型 IPsec 微服务编排要求先 [0-00306](task/UDG/20.15.2/0-00306.md) ADD IPBINDVPNIPSEC 再下发本命令 — 后果：未绑 VPN 时 IP 不在 VRF 路由表中
- **权限要求**（info，rule-0-01304）：需 G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
