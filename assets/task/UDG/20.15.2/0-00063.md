# 配置IPv6静态路由（ADD SRROUTE6）

> 命令静态知识见 [ADD SRROUTE6](command/UDG/20.15.2/ADD-SRROUTE6.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD SRROUTE6 是 IPv6 路由管理的"静态条目登记"层：在指定 VRF（VPN 实例）下登记一条 IPv6 静态路由条目，把"目的前缀 + 掩码 + 出接口/下一跳"用一条命名记录描述。3 个特性命中覆盖 010107 静态地址冗余（VRFNAME=vpn1 + IFNAME=Tunnel 备份隧道）、020402 N6/Gi/SGi IPv6 组网（VRFNAME=VPN_CN + DESTVRFNAME 跨 VRF + NEXTHOP）、IPFD-015004 IPSec（VRFNAME=vrf1 + DESTVRFNAME+NEXTHOP+Tunnel 接口）。

### 配置维度 1：地址族（参数 AFTYPE，必选）

| 取值 | 作用 | 配套参数 | 典型场景 |
|---|---|---|---|
| **ipv6unicast** | IPv6 单播地址族 | PREFIX（IPv6 地址）+ MASKLENGTH（0~128） | 全部 3 个特性命中（IPSec 演示 ipv4uni/ipv6uni 两套） |

> 当前特性命中均使用 `ipv6unicast`；AFTYPE 另有 ipv4unicast 等枚举，本命令本版特性未演示。

### 配置维度 2：下一跳方式（参数 IFNAME/NEXTHOP/DESTVRFNAME，组合必选）

| 模式 | 必选字段 | 适用场景 |
|---|---|---|
| **本端出接口** | IFNAME（引 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE） | 010107 静态地址冗余（IFNAME=Tunnel1/2）、IPSec 隧道下一跳（IFNAME=Tunnel27） |
| **VRF 内下一跳** | NEXTHOP（IPv6 地址） | N6/Gi/SGi IPv6 组网（同 VRF 内通过 NEXTHOP 转发） |
| **跨 VRF 下一跳** | DESTVRFNAME + NEXTHOP | N6/Gi/SGi IPv6 组网（VRFNAME=VPN_CN 但 NEXTHOP 走 DESTVRFNAME=VPN_CN）、IPSec VPN 间互通 |

| 参数 | 范围 | 约束 |
|---|---|---|
| `PREFIX` | IPv6 地址 | 必选 |
| `MASKLENGTH` | 整数 0~128 | 必选 |
| `VRFNAME` | 字符串 1~31 | 可选；不配时落入公网 |
| `IFNAME` | 字符串 1~63 | 引用 [0-00056](task/UDG/20.15.2/0-00056.md) INTERFACE 名称 |
| `NEXTHOP` | IPv6 地址 | 与 IFNAME 二选一或配合 DESTVRFNAME |
| `DESTVRFNAME` | 字符串 1~31 | 条件必选（指定跨 VRF 下一跳时） |

### 配置维度 3：联动增强（参数 BFDENABLE/DHCPENABLE/PREFERENCE/SESSIONNAME/TAG/DESCRIPTION，可选）

| 参数 | 范围 | 联动 |
|---|---|---|
| **BFDENABLE** | TRUE / FALSE | TRUE 时联动 BFD 会话检测链路（010107 演示 BFDENABLE=FALSE） |
| **SESSIONNAME** | 字符串 | 联动静态 BFD 会话（引 [0-00082](task/UDG/20.15.2/0-00082.md) BFDSESSION） |
| **DHCPENABLE** | TRUE / FALSE | DHCP 用户路由冗余触发（010107 演示 DHCPENABLE=FALSE） |
| **PREFERENCE** | 整数 1~255（默认 60） | 路由优先级（值越小越优） |
| **TAG** | 整数 0~4294967295 | 路由 Tag |
| **DESCRIPTION** | 字符串 1~35 | 描述 |

**典型脚本（010107 静态地址冗余，IFNAME=Tunnel 备份隧道）**：
```
ADD SRROUTE6: AFTYPE=ipv6unicast, PREFIX="fc00:3::1", MASKLENGTH=128, VRFNAME="vpn1", IFNAME="Tunnel1";
ADD SRROUTE6: AFTYPE=ipv6unicast, PREFIX="fc00:1::1", MASKLENGTH=64, VRFNAME="vpn1", IFNAME="Tunnel1", BFDENABLE=FALSE, DHCPENABLE=FALSE;
```

**典型脚本（020402 N6/Gi/SGi IPv6 组网，跨 VRF+NEXTHOP）**：
```
ADD SRROUTE6:VRFNAME="VPN_CN",AFTYPE=ipv6unicast,PREFIX="fc00:1000::1",MASKLENGTH=64,DESTVRFNAME="VPN_CN",NEXTHOP="fc00:1000::3",IFNAME="Ethernet70/0/4.1";
```

**典型脚本（IPFD-015004 IPSec，VRF 内 Tunnel 接口下一跳）**：
```
ADD SRROUTE6:VRFNAME="vrf1",AFTYPE=ipv6unicast,PREFIX="2001:db8:4::2",MASKLENGTH=128,DESTVRFNAME="vrf1",NEXTHOP="2001:db8:4::1:0:1",IFNAME="Tunnel27";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：路由规划"——前置 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE（IFNAME 来源）+ [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST + [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF（VRFNAME 来源）。

## 决策点：下一跳方式选择（DP 0-00063）

问：ADD SRROUTE6 选哪种下一跳方式？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 本端出接口（IFNAME_ONLY） | IFNAME 必选 | 点对点链路（GRE Tunnel、IPSec Tunnel 出接口固定） |
| VRF 内下一跳（NEXTHOP_LOCAL） | NEXTHOP 必选 | 同 VRF 内通过下一跳 IP 转发 |
| 跨 VRF 下一跳（NEXTHOP_CROSSVRF） | DESTVRFNAME + NEXTHOP | VPN 间互通、N6 跨 VRF 业务 |

> 默认按接口/VRF 形态决定：Tunnel 类接口通常配 IFNAME；同 VRF 多条等价路径通常配 NEXTHOP；跨 VRF 必须配 DESTVRFNAME。

## 约束

- **IFNAME / NEXTHOP 至少必选其一**（critical，隐含）：纯目的前缀无出口时不可达；IFNAME 与 NEXTHOP 可共存（接口+下一跳 IP 双保险） — 后果：缺出口参数命令拒收
- **VRFNAME 前置依赖**（critical，隐含）：VRFNAME 须先建（[0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST + [0-00072](task/UDG/20.15.2/0-00072.md) ADD VPNINSTAF）；DESTVRFNAME 同 — 后果：未建 VRF 引用失败
- **IFNAME 前置依赖**（critical，隐含）：IFNAME 须先建（[0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 或 IPSec 隧道产出） — 后果：未建接口引用失败
- **MASKLENGTH 范围 0~128**（critical，隐含）：越界命令拒收 — 后果：超 IPv6 掩码范围配置失败
- **BFDENABLE=TRUE 时联动 BFD 会话**（warning，隐含）：建议同时配置 SESSIONNAME 或动态 BFD 联动 — 后果：BFD 链路检测失效
- **默认路由受保护 VRF 限制**（critical，rule-0-00184）：PREFIX 为默认路由（MASKLENGTH=0）时，**不能**在 VPN 实例 `__mpp_vpn_inner__` 和 `__mpp_vpn_inner_server__` 下添加 — 后果：受保护 VPN 实例下加默认路由被拒
