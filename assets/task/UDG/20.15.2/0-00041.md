# 配置VPN实例（ADD VPNINST）

> 命令静态知识见 [ADD VPNINST](command/UDG/20.15.2/ADD-VPNINST.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD VPNINST 是 DN 管理的"VPN 实例创建层"：用一条命名记录创建 VPN 实例，产出 VPNINSTANCE 供接口绑定（ADD IPBINDVPN）、地址池绑定（ADD POOLBINDGROUP）、路由协议（OSPF/BGP）等下游配置引用。命令核心配置空间仅 1 个维度：**VPN 实例命名唯一性**（VPNINSTANCE 全网唯一，需与平台 ADD L3VPNINST 的 VPN 命名一致）。无其他参数——命令仅 1 参。10 个特性命中覆盖地址分配、IPv4v6 双栈、L2TP VPN、IMS 恢复、URL 过滤、Proxy UPF 漫游等基础接入/IPv6/组网/语音/业务感知多个场景。

### 配置维度 1：VPN 实例命名（参数 VPNINSTANCE，必选 + 全网唯一）

| 参数 | 取值范围 | 备注 |
|---|---|---|
| **VPNINSTANCE**（VPN 实例名） | 字符串（命令 wiki 给定范围） | 全网规划 / 全网唯一；与平台 ADD L3VPNINST 命名一致，否则业务不通 |

**典型脚本（010104 地址分配方式，010105 SMF 分配地址共用）**：
```
ADD VPNINST:VPNINSTANCE="vpn1";
```

**典型脚本（010155 Untrusted Non-3GPP 双 VPN）**：
```
ADD VPNINST: VPNINSTANCE ="vpn-s2b";
ADD VPNINST: VPNINSTANCE ="vpn-paif";
```

**典型脚本（020401 IPv6 承载上下文）**：
```
ADD VPNINST:VPNINSTANCE="VPN_Internet";
```

**典型脚本（020253 P-CSCF 故障 IMS 恢复）**：
```
ADD VPNINST:VPNINSTANCE="ims_vpn";
```

**典型脚本（020412 L2TP VPN）**：
```
ADD VPNINST:VPNINSTANCE="vpn_l2tp";
```

**典型脚本（110471 URL 过滤到 ICAP Server）**：
```
ADD VPNINST:VPNINSTANCE="vpn-gcfif";
```

**典型脚本（112001 Proxy UPF 漫游镜像 SIP 信令，★数据规划表唯一命中）**：
```
ADD VPNINST:VPNINSTANCE="SIP_OM";
```

> 命令 notes：适用 NF：SGW-U、PGW-U、UPF（本批次 10 条 atom 中唯一适用 SGW-U 的命令）；执行后立即生效；与平台 ADD L3VPNINST 的 VPN 命名一致（命名联动约束）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：VPN 实例创建"——前置 平台 ADD L3VPNINST（命名联动）；产出 VPNINSTANCE 被 ADD IPBINDVPN(0-00067) / ADD POOLBINDGROUP(0-00045) / OSPF/BGP 等下游引用。

## 决策点

（无。本命令仅 1 必选参数 + 唯一性约束，无可选分支、无 DP；命名一致性是配置生成自检的核心，详见约束段。）

## 约束

- **VPNINSTANCE 全网唯一**（critical，rule-0-00341）：不同 VPN 实例之间 VPNINSTANCE 不能重复 — 后果：重名 ADD 失败
- **VPNINSTANCE 与平台 L3VPNINST 命名一致**（critical，rule-0-00341）：本命令创建的 VPN 命名必须与平台 ADD L3VPNINST 的 VPN 命名一致 — 后果：命名不一致业务不通
- **适用 NF 限定**（warning，隐含）：SGW-U、PGW-U、UPF — 后果：仅 M2000-B 单板不适用（本批次唯一含 SGW-U 的命令）
- **执行后立即生效**（warning，rule-0-00341）：本命令立即生效无延时 — 后果：与其他延迟生效命令混排时需注意顺序
