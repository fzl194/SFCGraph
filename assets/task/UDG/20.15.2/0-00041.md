# 配置VPN实例（ADD VPNINST）
> 命令静态知识见 [ADD VPNINST](command/UDG/20.15.2/ADD-VPNINST.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

命名一个用户面（DN 侧）VPN 实例，作为后续 ADD POOL / ADD APN 的 VPNINSTANCE 引用源——把地址池、APN 绑到这个 VPN 网络上做私网转发。该命令本质是命名锚点，真正的接口绑定由 ADD LOGICINF 完成。

| 参数 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| VPNINSTANCE（VPN 实例名） | vpn1 / vpn_l2tp / ims_vpn / vpn-s2b / SIP_OM / VPN_Internet | 全网规划（主流） | 1~31 字符串，区分大小写，全网唯一 |

典型脚本样例（按业务域命名）：
```
# 地址分配 / APN 绑定（GWFD-010104/010105/020406/020421）
ADD VPNINST:VPNINSTANCE="vpn1";

# L2TP VPN（GWFD-020412）
ADD VPNINST:VPNINSTANCE="vpn_l2tp";

# Untrusted Non-3GPP 接入（GWFD-010155）
ADD VPNINST:VPNINSTANCE="vpn-s2b";

# 镜像 SIP 信令 OM 网络（GWFD-112001）
ADD VPNINST:VPNINSTANCE="SIP_OM";
```

配置要点：VPNINSTANCE 命名跨 10+ 特性复用（地址分配/ePDG/IPv6 承载/L2TP/IMS 恢复/URL 过滤等），是共享基础对象。主流规划口径为全网规划（命名全网唯一语义），个别特性局部规划（GWFD-020401 IPv6 承载/GWFD-110471 URL 过滤）按本端规划，未达 decision_driven 阈值。本 pass 仅录 VPNINSTANCE 单参数（命令真相仅此 1 项，无缺口），ADD VPNINST 其余参数待后续特性揭示时增量补全。

## 约束
- VPNINSTANCE 须与 VNRS 上 ADD L3VPNINST 创建的同名 VPN 一致 · critical · 否则 VPN 业务不通（见 [[UDG@20.15.2@TaskRule@0-00161]]）
- 承载该 VPN 的逻辑接口与其外联口 VPN 实例必须相同 · critical · 否则报文查路由失败
- 最大记录数 4096 / 立即生效 · warning · 超规格新增失败（见 [[UDG@20.15.2@TaskRule@0-00160]]）
- 适用 NF：SGW-U、PGW-U、UPF
