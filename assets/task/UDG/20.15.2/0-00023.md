# 配置重定向携带信息（ADD REDIRAPPENDINFO）

> 命令静态知识见 [ADD REDIRAPPENDINFO](command/UDG/20.15.2/ADD-REDIRAPPENDINFO.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD REDIRAPPENDINFO 是"重定向携带什么"的总控命令：定义一次重定向时，目标 URL 中可附带哪些用户标识/请求信息字段，并可对敏感字段做 AES/blowfish 加密。命令核心配置空间有 3 个维度：**携带字段组合**（6 个 FLAG，决定哪些信息进入重定向 URL）、**字段自定义名称**（各 NAME，缺省取默认名）、**加密**（每个被携带字段可选 AES128/AES256/AES256_GCM/BLOWFISH128 等 + SECRETKEY）。产出对象 APPENDINFONAME 被 SET GYONESHOT(0-00024) 与 ADD RULE 引用，立即生效不需 REFRESHSRV。

### 配置维度 1：携带字段组合（6 个 FLAG，独立开关）

| 字段 | FLAG 参数 | 默认名（不配 NAME 时） | 适用场景 |
|---|---|---|---|
| 原始请求 URL | `REQURLFLAG` | `dest_url` | 计费信息实时提醒、HTTP 智能重定向（默认必带） |
| MSISDN | `MSISDNFLAG` | `msisdn` | 计费信息实时提醒（020356 默认 DISABLE） |
| IMSI | `IMSIFLAG` | `imsi` | 计费信息实时提醒（020356 必带 + 加密）、HTTP 智能重定向（精简带） |
| IMEI | `IMEIFLAG` | `imei` | 计费信息实时提醒（020356 必带） |
| MS-IP | `MSIPFLAG` | `msip` | 计费信息实时提醒（020356 必带） |
| 时间戳 | `TIMEFLAG` | `u` | 时间戳维度（020356 默认不配） |

> 每个 FLAG 取值 DISABLE/ENABLE；不配即视为 DISABLE。020356 启用 REQURL+IMSI+IMEI+MSIP，MSISDN/TIME 不配；110284 仅启用 REQURL+IMSI。

### 配置维度 2：字段自定义名称（参数 *NAME，条件可选）

| 参数 | 格式 | 何时启用 |
|---|---|---|
| `REQURLNAME` / `MSISDNNAME` / `IMSINAME` / `IMEINAME` / `MSIPNAME` / `TIMENAME` | 字符串 | 对应 FLAG=ENABLE 时可配（缺省取上表默认名） |

### 配置维度 3：加密算法与密钥（条件可选，仅 FLAG=ENABLE 时存在）

| 字段 | 加密算法参数 | 配套密钥参数 | 备注 |
|---|---|---|---|
| MSISDN | `MSISDNENCRYALGORI` (AES128/AES256/AES256_GCM/BLOWFISH128/BLOWFISH256) | `MSISDNSECRETKEY` + `MSISDNSECRETKEYCONFIRM` | 020356 默认不配此字段 |
| IMSI | `IMSIENCRYALGORI` | `IMSISECRETKEY` + `IMSISECRETKEYCONFIRM` | 020356 实操配 AES256 |
| IMEI | `IMEIENCRYALGORI` | `IMEISECRETKEY` + `IMEISECRETKEYCONFIRM` | 020356 默认不加密 |
| MS-IP | `MSIPENCRYALGORI` | `MSIPSECRETKEY` + `MSIPSECRETKEYCONFIRM` | 020356 默认不加密 |
| 时间戳 | `TIMEENCRYALGORI` | `TIMESECRETKEY` + `TIMESECRETKEYCONFIRM` | 默认不启用 |

> 加密配套：ENCRYALGORI 选配时必填对应 SECRETKEY + SECRETKEYCONFIRM；不配 ENCRYALGORI 即明文携带。

**典型脚本（020356 多字段 + IMSI 加密）**：
```
ADD REDIRAPPENDINFO:APPENDINFONAME="testredirappendinfo",REQURLFLAG=ENABLE,MSISDNFLAG=DISABLE,IMSIFLAG=ENABLE,IMSIENCRYALGORI=AES256,IMEIFLAG=ENABLE,MSIPFLAG=ENABLE;
```

**典型脚本（110284 精简：URL+IMSI）**：
```
ADD REDIRAPPENDINFO:APPENDINFONAME="testredirappendinfo",REQURLFLAG=ENABLE,IMSIFLAG=ENABLE;
```

> 命令 notes：最大记录数 1000；适用 NF：PGW-U、UPF；执行后立即生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：重定向携带信息定义"——前置无；产出 APPENDINFONAME 被 SET GYONESHOT(0-00024) 或 ADD RULE 引用。

## 决策点：重定向携带字段组合（DP 0-00023）

问：ADD REDIRAPPENDINFO 的字段携带组合取哪个分支？

| 选项 | 携带字段 | 加密 | 适用特性 |
|---|---|---|---|
| 计费信息实时提醒（多字段 + IMSI 加密） | REQURL=ENABLE, IMSI=ENABLE, IMEI=ENABLE, MSIP=ENABLE（MSISDN/TIME=DISABLE） | IMSIENCRYALGORI=AES256 | 020356（计费信息实时提醒） |
| HTTP 智能重定向（精简：URL+IMSI 明文） | REQURL=ENABLE, IMSI=ENABLE（其余 FLAG 缺省 DISABLE） | 不配 ENCRYALGORI | 110284（HTTP 智能重定向） |
| 默认（仅 REQURL） | 仅 REQURL=ENABLE | 不加密 | 通用兜底 |

## 约束

- **加密算法配套必填**（critical，rule-0-00295）：任一 ENCRYALGORI 配值时，对应 SECRETKEY + SECRETKEYCONFIRM 必填 — 后果：ENCRYALGORI 选配但 SECRETKEY 漏填→ADD 拒收
- **APPENDINFONAME 全网唯一**（critical，隐含）：不同 APPENDINFONAME 之间名称不能重复 — 后果：重名 ADD 失败
- **APPENDINFONAME 须先于 SET GYONESHOT 配置**（critical，rule-0-00295）：SET GYONESHOT(0-00024) 引用本命令产出 APPENDINFONAME，本命令未配或失败→GYONESHOT 引用失败 — 后果：一次重定向不生效
- **最大记录数 1000**（warning，rule-0-00295）：单板最大 1000 条 — 后果：超规格 ADD 失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
