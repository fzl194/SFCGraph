# 配置重定向携带信息（ADD REDIRAPPENDINFO）

> 命令静态知识见 [ADD REDIRAPPENDINFO](command/UDG/20.15.2/ADD-REDIRAPPENDINFO.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是**跨 2 特性共享**：020356 计费信息实时提醒（多字段携带 + IMSI 加密）vs 110284 HTTP 智能重定向（精简：仅 URL+IMSI）。决定**重定向报文中携带哪些用户信息字段**（原始请求 URL / MSISDN / IMSI / IMEI / MS-IP / 时间戳），可对携带字段做 AES/blowfish 加密。产出对象 APPENDINFONAME 被 [0-00024](task/UDG/20.15.2/0-00024.md)（SET GYONESHOT）引用。立即生效，不需 REFRESHSRV。

**配置范式（按决策点选字段组合）**

| 场景 | APPENDINFONAME | 携带的 FLAG | 加密 | 典型行 |
|---|---|---|---|---|
| 110284 HTTP 智能重定向（精简） | `testredirappendinfo` | REQURLFLAG=ENABLE, IMSIFLAG=ENABLE | 无 | `ADD REDIRAPPENDINFO:APPENDINFONAME="testredirappendinfo",REQURLFLAG=ENABLE,IMSIFLAG=ENABLE;` |
| 020356 计费信息实时提醒（多字段+IMSI加密） | `testredirappendinfo` | REQURLFLAG=ENABLE, MSISDNFLAG=DISABLE, IMSIFLAG=ENABLE, IMEIFLAG=ENABLE, MSIPFLAG=ENABLE | IMSIENCRYALGORI=AES256 | `ADD REDIRAPPENDINFO:APPENDINFONAME="testredirappendinfo",REQURLFLAG=ENABLE,MSISDNFLAG=DISABLE,IMSIFLAG=ENABLE,IMSIENCRYALGORI=AES256,IMEIFLAG=ENABLE,MSIPFLAG=ENABLE;` |

**字段默认名**：REQURL/MSISDN/IMSI/IMEI/MSIP/TIME 字段若不配 NAME 则采用默认（`dest_url`/`msisdn`/`imsi`/`imei`/`msip`/`u`）。主场景两特性均用默认名，REQURLNAME/MSISDNNAME 等 NAME 参数不绑。

**加密配套（conditional）**：MSISDN/IMSI/IMEI/MSIP/TIME 的 ENCRYALGORI 选配时必填对应 SECRETKEY + SECRETKEYCONFIRM。主场景仅 020356 用 IMSIENCRYALGORI=AES256（注意：命令真相默认 AES256_GCM，020356 实操用 AES256——按任务示例采纳）；其余加密算法及 SECRETKEY 按定制场景补，主场景不绑。

## 决策点：REDIRAPPENDINFO 字段携带组合（DP 0-00110）

问：ADD REDIRAPPENDINFO 的字段携带组合取哪个分支？

| 选项 | 适用特性 | 携带字段 | 加密 | 不传字段 |
|---|---|---|---|---|
| opt-redirappendinfo-billing-prompt（默认） | 020356 计费信息实时提醒 | REQURL=ENABLE, MSISDN=DISABLE, IMSI=ENABLE, IMEI=ENABLE, MSIP=ENABLE | IMSIENCRYALGORI=AES256（required） | TIMEFLAG（absent） |
| opt-redirappendinfo-smart-http | 110284 HTTP 智能重定向 | REQURL=ENABLE, IMSI=ENABLE | 无（IMSIENCRYALGORI absent） | MSISDNFLAG/IMEIFLAG/MSIPFLAG/TIMEFLAG（absent，走默认 DISABLE） |

## 约束

- **rule-0-00295**（warning）最大记录数 1000；立即生效；字段默认名（不配 NAME 取 dest_url/msisdn/imsi/imei/msip/u）；加密配套必填（ENCRYALGORI 选配时必填对应 SECRETKEY+SECRETKEYCONFIRM）；APPENDINFONAME 须先配方可被 SET GYONESHOT(0-00024) 引用 — 后果：超上限 ADD 失败 / ENCRYALGORI 选配但 SECRETKEY 漏填→ADD 拒收 / APPENDINFONAME 未先配→SET GYONESHOT 引用失败
