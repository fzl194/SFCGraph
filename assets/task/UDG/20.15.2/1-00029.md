# APN 接入域基础设施（L3VPN+APN+地址分配属性）

> APN 域地址分配类特性的**接入域底层**（所有地址分配特性第一步）。被引用于 6 个 feature。
> 2026-07-11 从 6 feature 步骤 1 平铺中抽取（防平铺，SOP §3.5 跨特性复用预判）。

## 配置方法

本 compound 建 VPN/APN 实例 + 使能地址分配属性，为后续地址池体系（[1-00022](task/UDG/20.15.2/1-00022.md)）/地址分配规则（[1-00021](task/UDG/20.15.2/1-00021.md)）提供接入域基础。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. L3VPN 实例 | `ADD L3VPNINST` | VRFNAME=vpn 名（与 Gi/SGi/N6 外联口、地址池三者一致）|
| 2. L3VPN 地址族 | `ADD VPNINSTAF` | VRFNAME, AFTYPE=ipv4uni/ipv6uni, VRFRD=100:1 |
| 3. 用户面 VPN 实例 | `ADD VPNINST` | VPNINSTANCE=vpn 名 |
| 4. APN/DNN 实例（绑 VPN）| `ADD APN` | APN=apn 名, HASVPN=ENABLE, VPNINSTANCE=vpn 名（双栈加 HASVPNIPV6=ENABLE, VPNINSTANCEIPV6=）|
| 5. 使能地址分配属性 | `SET APNADDRESSATTR` | APN=, SUPPORTIPV4/SUPPORTIPV6=ENABLE（按 IP 类型）；HOSTROUTEIP（外部地址分配）；IGNOREV4/V6POOLID=DISABLE（RADIUS 子方式）|

**典型脚本（IPv4/IPv6 本地地址分配，2-00034 GWFD-010105 基线）**：
```
ADD L3VPNINST:VRFNAME="vpn1";
ADD VPNINSTAF:VRFNAME="vpn1",AFTYPE=ipv4uni,VRFRD=100:1;
ADD VPNINST:VPNINSTANCE="vpn1";
ADD APN:APN="apn-test1",HASVPN=ENABLE,VPNINSTANCE="vpn1";
SET APNADDRESSATTR:APN="apn-test1",SUPPORTIPV4=ENABLE,SUPPORTIPV6=ENABLE;
```

**步骤位置**：所有 APN 域地址分配特性的**第一步**（License 前置之后）；下游 [1-00022](task/UDG/20.15.2/1-00022.md) 地址池体系 / [1-00021](task/UDG/20.15.2/1-00021.md) 地址分配规则。

## 决策点

| 选项（IP 类型/子方式） | 影响（SET APNADDRESSATTR + ADD APN 参数） |
|---|---|
| IPv4 本地分配 | SUPPORTIPV4=ENABLE |
| IPv6 本地分配 | SUPPORTIPV6=ENABLE |
| IPv4v6 双栈 | SUPPORTIPV4+IPV6=ENABLE + ADD APN 加 HASVPNIPV6=ENABLE+VPNINSTANCEIPV6= |
| 外部地址分配（010104）| HOSTROUTEIP=ENABLE（主机地址接入）|
| RADIUS 下发地址池名子方式 | IGNOREV4/V6POOLID=DISABLE（必配，支持 SMF 下发地址池名）|

> 联动特性级 DP：IP 类型/子方式由各 feature 场景 DP 决定。

## 约束

- **VPN 三者一致**（critical，rule-0-00498）：L3VPN/VPNINST 的 VRFNAME、ADD APN 的 VPNINSTANCE、Gi/SGi/N6 外联口绑定的 VPN 三者必须一致 — 后果：VPN 不一致致地址分配或路由异常。
- **APN/DNN 全网规划一致**（critical，rule-0-00499）：UDG 本端 APN/DNN 与 C 面（SMF/SGW-C/PGW-C）配置一致 — 后果：PFCP 信令失配。
- **AFTYPE 与 IP 类型匹配**（warning）：IPv4 场景 AFTYPE=ipv4uni；IPv6 场景 AFTYPE=ipv6uni；双栈两者都配 — 后果：地址族不匹配致 VPN 实例失效。
- **RADIUS 子方式必配 IGNORE*=DISABLE**（critical）：RADIUS 下发地址池名子方式必须 IGNOREV4/V6POOLID=DISABLE，否则 SMF 下发的地址池名被忽略。
- **License 前置**（critical，按 feature）：020421/020403/020401/020406 各需对应 License（见场景差异表）—— 本 compound 前各 feature 必须先开 License。
