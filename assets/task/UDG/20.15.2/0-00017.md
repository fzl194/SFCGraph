# 设置全局计费参数（SET UPGLBCHGPARA）

> 命令静态知识见 [SET UPGLBCHGPARA](command/UDG/20.15.2/SET-UPGLBCHGPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是**在线计费的全局参数开关集合**（最大记录数 1，全局唯一）。仅 domain DP 选"在线计费"时配。各参数独立可选，按需开启：

**配置范式（全局开关，按特性需要选用）**

| 参数 | 用途 | 取值约定 / 注意 |
|---|---|---|
| CREDITPOOLSW | Credit Pooling 功能开关 | ENABLE / DISABLE。**与 DefaultQuota 互斥**（见 [0-00016](task/UDG/20.15.2/0-00016.md)），二者只能开一个 |
| QUOTAAPPACTTYPE | 配额申请期间报文动作 | BUFFER（缓存，默认体验）/ DROP（丢弃，**影响用户体验**，新业务申请配额期间报文丢） |
| VOLTHMININTVAL | 流量阈值上报最小时长间隔 | 单位 500ms，范围 1~1200。**建议 < 60（即 30 秒）**——过大导致流量阈值控制不准（如 SMF 下发 100MB 阈值，实际可能用到 200MB 才上报） |
| DFTOFLEVTTHD | 离线事件计费上报阈值 | 0~10000，0=关闭。SMF 未下发事件计费阈值时 UPF 不做阈值控制 |
| EVENTCHGMOD | 事件计费方式 | 枚举 |
| CHGCHECKSW | 计费核查开关 | ENABLE / DISABLE |
| VOLUMEBKPSW | 流量备份开关 | ENABLE / DISABLE（配套 VOLUMEBKPTHD/VOLUMEBKPINTV 未绑，冷门） |

**核心决策——Credit Pooling vs DefaultQuota**：在线计费选其一，由 domain DP 决定。两者都是"申请配额期间的容灾策略"：
- DefaultQuota：用预设默认配额放通（见 [0-00016](task/UDG/20.15.2/0-00016.md)）
- Credit Pooling：多承载共享信用池

**与 SET UPDEFAULTQUOTA 的配合**：当 DefaultQuota 不使能时，新业务申请配额期间的报文动作（缓存/丢弃）由本命令 QUOTAAPPACTTYPE 决定。

## 约束

- **rule-0-00118**（warning）最大记录数 1（全局参数）；VOLTHMININTVAL 不为 0 时限制流量阈值上报频率致控制不准，建议 < 60（30 秒）；QUOTAAPPACTTYPE=DROP 时新业务申请配额期间报文丢弃影响体验；CREDITPOOLSW 使能与 DefaultQuota 功能不能同时使能 — 后果：误配致流量阈值控制不准/新业务报文丢弃/Credit Pooling 与 DefaultQuota 冲突
