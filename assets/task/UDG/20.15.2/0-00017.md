# 设置全局计费参数（SET UPGLBCHGPARA）

> 命令静态知识见 [SET UPGLBCHGPARA](command/UDG/20.15.2/SET-UPGLBCHGPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

SET UPGLBCHGPARA 是在线计费（Gy/N40 ONLINE 模式）的**全局参数开关集合**——最大记录数 1（全局唯一），各参数独立可选。本命令的核心配置空间有 4 个维度：**开关集合**（CREDITPOOLSW / CHGCHECKSW / VOLUMEBKPSW 等多开关并存）、**申请配额动作**（QUOTAAPPACTTYPE：BUFFER 或 DROP）、**阈值上报**（VOLTHMININTVAL / DFTOFLEVTTHD 控制上报频率和事件计费阈值）、**互斥与配合**（与 [SET UPDEFAULTQUOTA](0-00016.md) 的 Credit Pooling vs DefaultQuota 互斥；与 [SET UPDEFAULTQUOTA](0-00016.md) 配合兜底）。

### 配置维度 1：开关集合（多个全局开关并存）

每个开关独立可选，按需使能（最大记录数 1 = 全局仅有这一组）：

| 参数 | 取值 | 作用 | 典型场景 |
|---|---|---|---|
| `CREDITPOOLSW` | `ENABLE` / `DISABLE`（默认） | Credit Pooling 功能开关——多承载共享信用池 | N40 融合 / 共享配额场景 |
| `CHGCHECKSW` | `ENABLE` / `DISABLE`（默认） | 计费核查开关 | 计费稽核补报 |
| `VOLUMEBKPSW` | `ENABLE` / `DISABLE`（默认） | 流量备份开关 | 冷门，配套 `VOLUMEBKPTHD` / `VOLUMEBKPINTV` |

### 配置维度 2：申请配额期间报文动作（参数 QUOTAAPPACTTYPE）

| 取值 | 行为 | 用户体验影响 |
|---|---|---|
| `BUFFER`（默认） | 申请配额期间**缓存**报文 | 不影响用户体验——首包不丢 |
| `DROP` | 申请配额期间**丢弃**报文 | **影响用户体验**——新业务首包直接丢 |

> 该参数仅在 DefaultQuota 未使能（且 Credit Pooling 未使能）时真正生效；DefaultQuota 使能后配额由默认配额放通，无须兜底。

### 配置维度 3：阈值上报与事件计费（参数 VOLTHMININTVAL / DFTOFLEVTTHD / EVENTCHGMOD）

| 参数 | 范围 / 取值 | 含义 | 默认 / 建议 |
|---|---|---|---|
| `VOLTHMININTVAL` | 整数 1~1200（单位 500ms，即 0.5s~600s） | 流量阈值上报最小时长间隔 | **建议 < 60（即 30 秒）**——过大导致阈值控制不准（如下发 100MB 阈值实际可能用到 200MB 才上报） |
| `DFTOFLEVTTHD` | 0~10000 | 离线事件计费上报阈值；0=关闭 | SMF 未下发事件计费阈值时 UPF 不做阈值控制 |
| `EVENTCHGMOD` | 枚举 | 事件计费方式 | 见命令 wiki |

### 配置维度 4：互斥与配合（与 [SET UPDEFAULTQUOTA](0-00016.md)）

在线计费"申请配额期间的容灾策略"只能开一个：

| DefaultQuota（SET UPDEFAULTQUOTA） | Credit Pooling（SET UPGLBCHGPARA） | 报文动作（QUOTAAPPACTTYPE） |
|---|---|---|
| `DFTQTSWITCH=ENABLE` | `CREDITPOOLSW=DISABLE` | DefaultQuota 兜底 |
| `DFTQTSWITCH=DISABLE` | `CREDITPOOLSW=ENABLE` | Credit Pooling 兜底 |
| `DFTQTSWITCH=DISABLE` | `CREDITPOOLSW=DISABLE` | 按 `QUOTAAPPACTTYPE` 决定缓存 / 丢弃 |
| `DFTQTSWITCH=ENABLE` | `CREDITPOOLSW=ENABLE` | **✗ 冲突** |

**步骤位置**：在所属 feature 编排中与 [SET UPDEFAULTQUOTA](0-00016.md) 同源——按 domain DP 决策结果择一使能。

## 决策点：申请配额期间策略（DP 0-00061）

问：在 DefaultQuota 与 Credit Pooling 之间选哪个？

| 选项 | SET UPDEFAULTQUOTA | SET UPGLBCHGPARA（关键参数） | 适用 |
|---|---|---|---|
| `default_quota` | `DFTQTSWITCH=ENABLE` | `QUOTAAPPACTTYPE` 不参与兜底 | 用预设默认配额放通首包 |
| `credit_pooling` | `DFTQTSWITCH=DISABLE` | `CREDITPOOLSW=ENABLE` | 多承载共享信用池 |
| `quota_buffer_drop`（默认） | `DFTQTSWITCH=DISABLE` | `CREDITPOOLSW=DISABLE` + `QUOTAAPPACTTYPE=BUFFER/DROP` | 默认行为——BUFFER 缓存首包或 DROP 丢弃首包 |

## 约束

- **最大记录数与全局唯一**（warning，rule-0-00118）：最大记录数 1（全局参数）；VOLTHMININTVAL 不为 0 时限制流量阈值上报频率致控制不准，建议 < 60（30 秒）；QUOTAAPPACTTYPE=DROP 时新业务申请配额期间报文丢弃影响体验；CREDITPOOLSW 使能与 [SET UPDEFAULTQUOTA](0-00016.md) 的 DefaultQuota 功能**不能同时使能** — 后果：误配致流量阈值控制不准 / 新业务报文丢弃 / Credit Pooling 与 DefaultQuota 冲突
- **冷门参数前置**（info，隐含）：`VOLUMEBKPTHD` / `VOLUMEBKPINTV`（流量备份阈值 / 间隔）需配合 `VOLUMEBKPSW=ENABLE` 才有意义 — 后果：单独配阈值 / 间隔无效
