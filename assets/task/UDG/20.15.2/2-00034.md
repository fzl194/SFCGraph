# 用户面地址分配（GWFD-010105）

> 特性静态知识见 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-010105-用户面地址分配.md)。
>
> **本页定位**：配置生成实例化视角——GWFD-010105 用户面地址分配的**完整配置体系**（4 子方式 + 6 步通用骨架 + 9 调测命令）。

## 配置概览

本特性按 **4 种子方式 × 5/6 步通用骨架 × 9 调测命令** 三维度组织配置：

**对象链**：
```
SECTION（地址段，IPv4/IPv6）→ POOL（地址池，LOCAL 类型）→ POOLGROUP（地址池组）→ POOLGRPMAP（映射）
                                                                                  ↓
                                              APN / CPNODEID / LOCATION / RADIUS
                                                                                  ↓
                                              IPALLOCRULE（全局/APN 两级，三级优先级）
                                                                                  ↓
                                              下行路由 OSPF（PROTOCOL=wlr）发布到骨干网
```

**5/6 步通用骨架**（所有 4 子方式共享）：

| 步骤 | 干啥 | 主要命令 |
|---|---|---|
| 1. VPN+APN 准备 | 建 VPN/APN 实例并使能地址分配属性 | `ADD L3VPNINST` / `ADD VPNINSTAF` / `ADD VPNINST` / `ADD APN` / `SET APNADDRESSATTR` |
| 2. 地址池绑定到地址池组 | 建 LOCAL 地址池 + 地址段 + 池组 + 绑定 | `ADD POOL` / `ADD SECTION` / `ADD POOLGROUP` / `ADD POOLBINDGROUP` / `ADD CONFLICTIP`（可选）|
| 3. 配置映射关系（按子方式） | 建立 APN/SMF/APN+SMF/LOCATION 到 POOLGROUP 的映射 | `ADD POOLGRPMAP` / `ADD CPNODEID`（SMF 子方式） |
| 4. 配置地址分配规则 | 全局/APN 两级规则 + 三级优先级 | `SET IPALLOCRULE` / `SET APNIPALLOCRULE` |
| 5. SMF 子方式开关（仅 SMF 相关） | SMF 全局开关 + 可选 SMF 级覆盖 | `SET IPALLOCBYSMFGLBSW` / `SET IPALLOCBYSMFSW` |
| 6. 下行路由发布 | OSPF 引入用户路由（wlr）发布到骨干网 | `ADD OSPF` / `ADD OSPFAREA` / `ADD OSPFNETWORK` / `ADD OSPFIMPORTROUTE` |
| 调测 | 验证地址分配是否生效 | `DSP SESSIONINFO` / `DSP POOLUSAGE` / `DSP SECTIONUSAGE` / `EXP MML` / `LST *` |

**关键约束**：
- **本特性无需 License**
- C 面网元**必须为华为配套设备**（N4/Sxb PFCP 协议私有扩展）
- VPN 三者一致：地址池 / APN / Gi/SGi/N6 外联口
- 每 APN 绑池数 ≤16，每 POOL 绑段数 ≤64

## 配置流程

> 每步显示"干啥 + 含哪些命令"。可按 4 子方式选择不同参数变种。

1. **VPN+APN 准备**（VPN/APN 实例 + 地址分配属性使能）：
   - `ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` → [0-00071](task/UDG/20.15.2/0-00071.md) + [0-00072](task/UDG/20.15.2/0-00072.md) + [0-00041](task/UDG/20.15.2/0-00041.md)
   - `ADD APN` → [0-00020](task/UDG/20.15.2/0-00020.md)（APN, HASVPN=ENABLE, VPNINSTANCE）
   - `SET APNADDRESSATTR` → [0-00046](task/UDG/20.15.2/0-00046.md)（SUPPORTIPV4/IPV6=ENABLE；**RADIUS 子方式必配 IGNOREV4/V6POOLID=DISABLE**）

2. **地址池绑定到地址池组**（LOCAL 地址池体系）：
   - `ADD POOL` + `ADD SECTION` + `ADD POOLGROUP` + `ADD POOLBINDGROUP` + `ADD CONFLICTIP`（可选）
   - → 复用 [1-00022](task/UDG/20.15.2/1-00022.md) 地址分配体系（地址池三级+映射）

3. **配置映射关系（按子方式）**：
   - **基于 APN/DNN 子方式**：仅 `ADD POOLGRPMAP` → [0-00047](task/UDG/20.15.2/0-00047.md)（APN=apn-test1）
   - **基于 SMF 子方式**：先 `ADD CPNODEID` → [0-00303](task/UDG/20.15.2/0-00303.md)，再 `ADD POOLGRPMAP`（SMF=smfnode1）
   - **基于 SMF+APN 子方式**：先 `ADD CPNODEID`，再**双 `ADD POOLGRPMAP`**（mapping1=APN+SMF / mapping2=SMF）+ 复用 [1-00023](task/UDG/20.15.2/1-00023.md) SMF 子方式开关
   - **基于 RADIUS 子方式**：仅 `ADD POOLGRPMAP`（APN=），外加 `SET ADDRESSATTR V4/V6POOLWILDCARDSW=ENABLE` → [0-00318](task/UDG/20.15.2/0-00318.md)
   - 步骤 2+3 合并复用 → [1-00022](task/UDG/20.15.2/1-00022.md)

4. **配置地址分配规则**（全局/APN 两级 + 三级优先级）：
   - `SET IPALLOCRULE` → [0-00250](task/UDG/20.15.2/0-00250.md)（全局规则，整机默认）
   - `SET APNIPALLOCRULE` → [0-00319](task/UDG/20.15.2/0-00319.md)（**RADIUS 子方式必配** APN 级规则覆盖全局，ALLOCATTR=LOCAL）
   - 复用 → [1-00021](task/UDG/20.15.2/1-00021.md) 地址分配规则

5. **SMF 子方式开关**（仅 SMF/SMF+APN 子方式执行）：
   - `SET IPALLOCBYSMFGLBSW` → [0-00323](task/UDG/20.15.2/0-00323.md)（全局开关 ENABLE）
   - `SET IPALLOCBYSMFSW`（可选）→ [0-00324](task/UDG/20.15.2/0-00324.md)（指定 SMF 覆盖）
   - 复用 → [1-00023](task/UDG/20.15.2/1-00023.md) SMF 子方式

6. **下行路由发布**（OSPF 引入 wlr 发布到骨干网）：
   - `ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFNETWORK` + `ADD OSPFIMPORTROUTE PROTOCOL=wlr`
   - → 复用 [1-00024](task/UDG/20.15.2/1-00024.md) 下行路由发布

7. **调测**（验证地址分配是否生效）：
   - `DSP SESSIONINFO` → [0-00327](task/UDG/20.15.2/0-00327.md)（验证 IPv4 PDP address + IPv4 Address type）
   - `DSP POOLUSAGE` → [0-00325](task/UDG/20.15.2/0-00325.md)（地址池占用）
   - `DSP SECTIONUSAGE` → [0-00326](task/UDG/20.15.2/0-00326.md)（地址段占用）
   - `LST POOL` / `LST POOLGROUP` / `LST APN` / `LST VPNINST` / `LST CPNODEID` → [0-00328](task/UDG/20.15.2/0-00328.md) / [0-00329](task/UDG/20.15.2/0-00329.md) / [0-00330](task/UDG/20.15.2/0-00330.md) / [0-00331](task/UDG/20.15.2/0-00331.md) / [0-00332](task/UDG/20.15.2/0-00332.md)（配置核查）
   - `EXP MML` → [0-00333](task/UDG/20.15.2/0-00333.md)（配置导出/故障定位）

## 决策点

**DP（特性级）：接入制式 × IP 类型 × 子方式 × 走法（★一张表覆盖全场景）**

| 接入制式 | IP 类型 | 子方式 | 走法（compound/atom 编排） |
|---|---|---|---|
| 5G | IPv4 | 基于 APN/DNN | [1-00022](task/UDG/20.15.2/1-00022.md)（1 池，APN=） + [1-00021](task/UDG/20.15.2/1-00021.md)（单级 APN-1&LOCATION-0&SMF-0） + [1-00024](task/UDG/20.15.2/1-00024.md)（OSPF） |
| 5G | IPv6 | 基于 APN/DNN | [1-00022](task/UDG/20.15.2/1-00022.md)（IPV6 SECTION） + [1-00021](task/UDG/20.15.2/1-00021.md)（同 IPv4，规则字符串相同） + [1-00024](task/UDG/20.15.2/1-00024.md)（OSPF，IPv6 用 OSPFv3） |
| 5G | IPv4v6 双栈 | 基于 APN/DNN | [1-00022](task/UDG/20.15.2/1-00022.md)（双 SECTION/IPV4+IPV6，各一池） + [1-00021](task/UDG/20.15.2/1-00021.md) + [1-00024](task/UDG/20.15.2/1-00024.md)（双协议：OSPF+OSPFv3） |
| 5G | IPv4 | 基于 SMF | [1-00022](task/UDG/20.15.2/1-00022.md)（1 池，SMF=） + [1-00023](task/UDG/20.15.2/1-00023.md)（CPNODEID + 全局开关） + [1-00021](task/UDG/20.15.2/1-00021.md)（单级 APN-0&LOCATION-0&SMF-1） + [1-00024](task/UDG/20.15.2/1-00024.md) |
| 5G | IPv6 | 基于 SMF | 同 IPv4，加 IPV6SWITCH |
| 5G | IPv4v6 双栈 | 基于 SMF | 同 IPv4，双 SECTION/IPV4+IPV6 |
| 5G | IPv4 | 基于 SMF+APN | [1-00022](task/UDG/20.15.2/1-00022.md)（2 池，mapping1=APN+SMF / mapping2=SMF） + [1-00023](task/UDG/20.15.2/1-00023.md)（CPNODEID + 全局开关） + [1-00021](task/UDG/20.15.2/1-00021.md)（两级：第一级 APN+SMF / 第二级 SMF） + [1-00024](task/UDG/20.15.2/1-00024.md) |
| 5G | IPv4v6 双栈 | 基于 SMF+APN | 同 IPv4，双 SECTION/IPV4+IPV6 + 优先级算法 |
| 5G | IPv4 | 基于 RADIUS 下发地址池名 | [1-00022](task/UDG/20.15.2/1-00022.md)（1 池，APN=） + [0-00046](task/UDG/20.15.2/0-00046.md)（IGNOREV4/V6POOLID=DISABLE）+ [0-00318](task/UDG/20.15.2/0-00318.md)（V4/V6POOLWILDCARDSW=ENABLE）+ [1-00021](task/UDG/20.15.2/1-00021.md)（**用 SET APNIPALLOCRULE 覆盖全局**） + [1-00024](task/UDG/20.15.2/1-00024.md) |
| 4G | IPv4/IPv6/IPv4v6 | 同 5G（共用 4 子方式） | 同 5G 走法（4G Sxb 接口替代 5G N4 接口） |
| 2/3G | IPv4/IPv6/IPv4v6 | 同 5G（共用 4 子方式） | 同 5G 走法（2/3G Gn/Gp 接口替代 5G N4 接口） |

> **关键规则**：
> - SMF 相关子方式（基于 SMF / SMF+APN）必配 [1-00023](task/UDG/20.15.2/1-00023.md) SMF 开关
> - RADIUS 子方式**必配** [0-00046](task/UDG/20.15.2/0-00046.md)（IGNORE*=DISABLE）+ [0-00318](task/UDG/20.15.2/0-00318.md)（WILDCARDSW=ENABLE）+ [1-00021](task/UDG/20.15.2/1-00021.md) APN 级规则
> - 所有 4 子方式都需配 [1-00024](task/UDG/20.15.2/1-00024.md) 下行路由发布

## 约束

- **无需 License**（info）：本特性无需获得 License 许可即可获得该特性的服务（特性概述明确声明）。
- **C 面网元必须为华为配套设备**（critical，rule-0-00497）：因 N4/Sxb 对 PFCP 协议有私有扩展 — 后果：非华为 C 面激活失败
- **VPN 三者一致**（critical，rule-0-00498）：地址池 / APN / Gi/SGi/N6 外联口绑定的 VPN 必须一致 — 后果：VPN 不一致致地址分配或路由异常
- **APN/DNN 全网规划一致**（critical，rule-0-00499）：UDG 本端配置的 APN/DNN 实例应与 C 面（SMF/SGW-C/PGW-C）配置的 APN/DNN 实例一致 — 后果：PFCP 信令失配
- **每 APN 绑池数 ≤16**（warning，rule-0-00500）：每 APN/DNN 支持绑定的地址池最多 16 个 — 后果：超规格新增失败
- **每 POOL 绑段数 ≤64**（warning，rule-0-00501）：每本地地址池支持绑定的地址段最多 64 个 — 后果：超规格新增失败
- **主机路由规格 40 万**（warning，rule-0-00502）：每 UDG 支持的主机路由规格为 40 万 — 后果：超规格下行路由发布异常
- **地址池优先级仅同类地址池间比较**（warning，rule-0-00503）：不同类型地址池（IPv4 vs IPv6）无优先级比较 — 后果：IPv4/IPv6 地址池独立优先级
- **冲突 IP 生效时机**（warning，rule-0-00504）：若某 IPv4 已分配给用户，配冲突 IP 命令成功但下次分配该地址时生效 — 后果：已在线用户不受影响
- **强制回收风险**（critical，rule-0-00505）：执行地址强制回收前必须先锁定地址池或地址段 — 后果：强制回收在线用户地址致强制下线
- **三级规则匹配失败则地址分配失败**（critical，rule-0-00506）：三级优先级规则都匹配失败 → 地址分配失败 — 后果：用户激活失败
- **真值表未列场景**（critical，rule-0-00507）：PFCP Session Establishment Request 中 BIT 组合未在真值表列出的场景，PGW-U/UPF 均以激活失败处理 — 后果：地址分配失败
- **ROUTER ID 全网唯一**（critical，rule-0-00508）：OSPF Router ID（SCHEMAROUID）必须全网唯一 — 后果：OSPF 邻接建立失败
- **基于位置的地址分配需参见 GWFD-020421**（info）：本特性实现位置分配的"地址池+映射+规则"链路，完整位置链路（位置号段/LACGROUP/TACGROUP）由 GWFD-020421 承载 — 后果：位置链路配置不完整
- **首次发布版本**（warning）：5G N4 接口首次发布版本为 UDG 20.0.0；4G Sxb 接口首次发布版本为 UDG 20.0.0；2/3G Gn/Gp 接口首次发布版本为 UDG 20.3.2 — 后果：低版本网元不支持
