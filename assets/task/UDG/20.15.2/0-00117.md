# 配置OSPFv3接口(IPv6)（ADD OSPFV3INTERFACE）

> 命令静态知识见 [ADD OSPFV3INTERFACE](command/UDG/20.15.2/ADD-OSPFV3INTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD OSPFV3INTERFACE 把接口加入 OSPFv3 进程的某个区域，使能 IPv6 动态路由。PROCID 引用上游 [OSPFv3 进程](task/UDG/20.15.2/0-00078.md)（ADD OSPFV3），AREAID 引用上游 [OSPFv3 区域](task/UDG/20.15.2/0-00079.md)（ADD OSPFV3AREA）。与 IPv4 [ADD OSPFNETWORK](task/UDG/20.15.2/0-00076.md) 平行（OSPFv3 二合一替代了 IPv4 的 OSPFNETWORK+OSPFINTERFACE）。

**IPv6 Prefix Delegation 单平面典型配法**（GWFD-020406，共网段虚拟系统）

| PROCID | AREAID | IFNAME | DRPRI | VIRTUALSYSFLAG | CFGROUTERIDFLAG |
|---|---|---|---|---|---|
| `6` | `0.0.0.5` | `Eth-trunk1.1` | `0` | `TRUE` | `FALSE` |

**参数取值样例**

| 参数 | 取值样例 | 来源 | 必选 | 说明 |
|---|---|---|---|---|
| PROCID | `6` | reference（← 0-00078 ADD OSPFV3） | required | OSPFv3 进程号 |
| AREAID | `0.0.0.5` | reference（← 0-00079 ADD OSPFV3AREA，与对端协商） | required | OSPFv3 区域号，点分十进制 |
| IFNAME | `Eth-trunk1.1` | global_planned（全网规划，接口名跨网元一致） | required | 接口名，1~63 字符 |
| DRPRI | `0` | fixed | optional | DR/BDR 优先级；**VNF 建议配 0** 避免被选为 DR/BDR（对端配非 0）；共网段生效时命令自动取 0 |
| VIRTUALSYSFLAG | `TRUE` | global_planned | conditional | OSPFv3 共网段虚拟系统使能；**共网段须与进程级 ADD OSPFV3 同时配 TRUE**；Tunnel 接口不支持 |
| CFGROUTERIDFLAG | `FALSE` | fixed | conditional | 不使能接口级 Router ID |

**典型脚本**：

```
ADD OSPFV3INTERFACE:PROCID=6,AREAID="0.0.0.5",IFNAME="Eth-trunk1.1",DRPRI=0,VIRTUALSYSFLAG=TRUE,CFGROUTERIDFLAG=FALSE;
```

**多接口同区域**：把多个接口配到同一区域时重复执行本命令，每接口一条。

**部署时序**：① ADD OSPFV3 建进程（0-00078）→ ② ADD OSPFV3AREA 建区域（0-00079）→ ③ SET IFIPV6ENABLE 接口使能 IPv6 → ④ ADD OSPFV3INTERFACE 接口入区域（本 atom）→ ⑤ ADD OSPFV3IMPORTROUTE 引入路由。

## 约束

- **rule-0-00203**（warning）**前置**：必须先执行 ADD OSPFV3（进程）与 ADD OSPFV3AREA（区域）；**接口上需先使能 IPv6**（SET IFIPV6ENABLE）才能加入 OSPFv3；OSPFv3 接口规格默认单进程 **1200**（实际以服务规格文件为准），命令最大记录数 **8000**；可把多个接口配到同一区域；**DRPRI=0** 表示不参与 DR/BDR 选举，建议 VNF 配 0；OSPFv3 共网段须**进程级与接口级 VIRTUALSYSFLAG 同时配 TRUE**，Tunnel 接口不支持 — 后果：接口使能失败 / 规格超限 / DR/BDR 误选 / 共网段配置不一致
