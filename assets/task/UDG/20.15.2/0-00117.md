# 配置OSPFv3接口(IPv6)（ADD OSPFV3INTERFACE）

> 命令静态知识见 [ADD OSPFV3INTERFACE](command/UDG/20.15.2/ADD-OSPFV3INTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD OSPFV3INTERFACE 把接口加入 OSPFv3 进程的某个区域，使能 IPv6 动态路由。**PROCID 必选**（1~4294967295 整数）、**AREAID 必选**（点分十进制 IPv4 格式）、**IFNAME 必选**（1~63 字符）。核心配置空间有 4 个维度：**进程区域绑定**（PROCID+AREAID）、**接口定位**（IFNAME）、**接口网络行为**（NETWORKTYPE/TIMER*/CONFIGCOST/BFD）、**共网段虚拟系统**（VIRTUALSYSFLAG/CFGROUTERIDFLAG）。1 个特性命中（GWFD-020406 IPv6 Prefix Delegation）使用 PROCID=6、AREAID=0.0.0.5、IFNAME=Eth-trunk1.1、DRPRI=0、VIRTUALSYSFLAG=TRUE、CFGROUTERIDFLAG=FALSE。

### 配置维度 1：进程区域绑定（参数 PROCID+AREAID，复合必选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **PROCID** | ref [0-00078](task/UDG/20.15.2/0-00078.md) ADD OSPFV3 | OSPFv3 进程号（020406 范式 `6`） |
| **AREAID** | ref [0-00079](task/UDG/20.15.2/0-00079.md) ADD OSPFV3AREA | OSPFv3 区域号，点分十进制 IPv4 格式（020406 范式 `0.0.0.5`） |

> (PROCID, AREAID) 必须对应已建 OSPFv3 进程+区域；同一接口可绑定多个 OSPFv3 进程（用 INSTANCEID 区分）。

### 配置维度 2：接口定位与实例（参数 IFNAME，必选 / INSTANCEID，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **IFNAME** | 字符串 1~63 字符 | 关联到本端的 IPv6 接口（ref [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS，020406 范式 `Eth-trunk1.1`）；Tunnel 接口不支持 VIRTUALSYSFLAG |
| **INSTANCEID** | 整数 | 同一接口下绑定多个 OSPFv3 进程时，用实例号区分不同进程 |

> IFNAME 必须在 `SET IFIPV6ENABLE` 启了 IPv6 后才能被 OSPFv3 接管。

### 配置维度 3：接口网络行为（参数 NETWORKTYPE / TIMER* / CONFIGCOST / BFD* / DRPRI / SILENTENABLE，可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **NETWORKTYPE** | BROADCAST/POINT_TO_POINT/NBMA/POINT_TO_MULTIPOINT | 接口网络类型（默认 BROADCAST） |
| **TIMERHELLOINTV** | 1~65535 秒 | Hello 报文间隔 |
| **TIMERDEADINTV** | 1~65535 秒 | 邻居失效时间；TIMERCONSERFLAG=TRUE 时实际配值小于 10s 仍按实际判断 |
| **TIMERRETRANINTV** | 1~65535 秒 | 重发 LSA 间隔 |
| **TIMERPOLLINTV** | 1~65535 秒 | NBMA 邻居失效后轮询 Hello 间隔 |
| **TIMERWAITINTV** | 1~65535 秒 | 超时时间 |
| **TRANSDELAY** | 1~800 秒 | LSA 传输延迟 |
| **CONFIGCOST** | 0~65535 | 接口开销值 |
| **MTUENABLE** | TRUE/FALSE | 使能 MTU |
| **BFDENABLE** | TRUE/FALSE | 使能 BFD（联动 MINRXRECVINTV/MINTXTRANINTV/DETECTMULINTV） |
| **MINRXRECVINTV / MINTXTRANINTV** | 30~1000 ms | 接收/发送 BFD 包最小间隔 |
| **DETECTMULINTV** | 3~50 | 本地检测倍数（默认 3） |
| **FRRBINDINGFLAG** | TRUE/FALSE | BFD 会话与接口链路状态绑定标志 |
| **FRRBLOCKFLAG** | TRUE/FALSE | 阻止指定 OSPFv3 接口的 FRR 能力 |
| **BFDBLOCK** | TRUE/FALSE | 阻塞接口动态创建 BFD 特性 |
| **DRPRI** | 0~255（默认 1） | DR/BDR 优先级；**VNF 建议配 0 避免被选为 DR/BDR** |
| **SILENTENABLE** | TRUE/FALSE | 禁止该接口收发 OSPFv3 报文 |

> NETWORKTYPE、计时器、COST 等参数独立组合；命令行不指定走进程级默认值。

### 配置维度 4：共网段虚拟系统（参数 VIRTUALSYSFLAG+CFGROUTERIDFLAG，条件可选）

| 参数 | 取值 | 前提 | 作用 |
|---|---|---|---|
| **VIRTUALSYSFLAG** | TRUE/FALSE | — | OSPFv3 共网段虚拟系统使能（020406 范式 TRUE；Tunnel 接口不支持） |
| **CFGROUTERIDFLAG** | TRUE/FALSE | VIRTUALSYSFLAG=TRUE 时 | 接口级虚拟 Router ID 使能标识（020406 范式 FALSE） |
| **CFGROUTERID** | 字符串 | CFGROUTERIDFLAG=TRUE 时 | 接口级虚拟 Router ID |

> 共网段须**进程级与接口级 VIRTUALSYSFLAG 同时配 TRUE**（[0-00078](task/UDG/20.15.2/0-00078.md) ADD OSPFV3）。

**典型脚本（GWFD-020406 IPv6 Prefix Delegation，共网段虚拟系统）**：
```
ADD OSPFV3INTERFACE:PROCID=6,AREAID="0.0.0.5",IFNAME="Eth-trunk1.1",DRPRI=0,VIRTUALSYSFLAG=TRUE,CFGROUTERIDFLAG=FALSE;
```

**多接口同区域**：把多个接口配到同一区域时重复执行本命令，每接口一条；进程下接口规格默认 1200。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：将接口入 OSPFv3 区域"——前置 [0-00078](task/UDG/20.15.2/0-00078.md) ADD OSPFV3 建进程 + [0-00079](task/UDG/20.15.2/0-00079.md) ADD OSPFV3AREA 建区域 + [0-00060](task/UDG/20.15.2/0-00060.md) SET IFIPV6ENABLE 启 IPv6；后续 [0-00080](task/UDG/20.15.2/0-00080.md) ADD OSPFV3IMPORTROUTE 引入路由。

## 决策点：共网段虚拟系统是否启用（DP 0-00117）

问：本接口是否启用 OSPFv3 共网段虚拟系统？

| 选项 | VIRTUALSYSFLAG | CFGROUTERIDFLAG | IFNAME 限制 | 适用场景 |
|---|---|---|---|---|
| **启用共网段**（020406 范式） | TRUE | FALSE（020406 范式） | **非 Tunnel 接口** | UPF 共网段组网、UPF 间虚拟系统区分 |
| **不启用**（普通 OSPFv3 接口） | FALSE | 不配 | 无 | 普通 IPv6 路由组网 |

> 1 个特性命中（020406）；启用共网段须**接口级 + 进程级同时配 TRUE**；Tunnel 接口不支持共网段。

## 约束

- **(PROCID, AREAID) 必须已建**（critical，rule-0-00117）：PROCID 须已通过 ADD OSPFV3 创建、AREAID 须已通过 ADD OSPFV3AREA 创建 — 后果：进程或区域不存在命令拒收
- **IFNAME 须先使能 IPv6**（critical，rule-0-00117）：IFNAME 必须已通过 `SET IFIPV6ENABLE` 启用 IPv6 — 后果：未启 IPv6 时 OSPFv3 接管该接口失败
- **PROCID + AREAID 必选**（critical，rule-0-00117）：PROCID 与 AREAID 是必选参数 — 后果：缺一命令拒收
- **Tunnel 接口不支持共网段**（warning，rule-0-00117）：IFNAME 为 Tunnel 类型时 VIRTUALSYSFLAG=TRUE 命令拒收 — 后果：Tunnel 接口误配导致命令失败
- **共网段须双级一致**（warning，rule-0-00117）：共网段使能时 [0-00078](task/UDG/20.15.2/0-00078.md) ADD OSPFV3 进程级与本接口 VIRTUALSYSFLAG 须同时为 TRUE — 后果：进程级未配时接口级失效
- **DRPRI=0 不参与 DR/BDR**（info，rule-0-00117）：DRPRI=0 表示该接口不参与 DR/BDR 选举 — 后果：VNF 建议配 0 时被对端配非 0 抢占 DR/BDR
- **OSPFv3 接口规格**（info，rule-0-00117）：单进程下默认 OSPFv3 接口规格 1200、命令最大记录数 8000 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00117）：本命令执行后立即生效 — 后果：误配即影响 IPv6 路由
