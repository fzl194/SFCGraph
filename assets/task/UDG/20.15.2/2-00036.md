# 基于位置的地址分配（GWFD-020421）

> 特性静态知识见 [GWFD-020421](feature/UDG/20.15.2/GWFD-020421.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-020421-基于位置的地址分配.md)。
>
> **本页定位**：配置生成实例化视角——基于位置的地址分配的完整配置体系。母特性（GWFD-010105 用户面地址分配）的"基于位置"子方式扩展为独立特性，**依赖**母特性的地址池体系，**新增** 4 项独有配置：License 开关 / 位置区组 / 位置分配开关 / 位置分配白名单。

## 配置概览

本特性按 **5 种组合维度 × 10 步通用骨架** 组织配置：

**对象链**：
```
位置区体系（LAC/TAC）                       地址池体系（继承母特性）
┌─────────────────────┐                ┌─────────────────────────┐
│ LACGROUP / TACGROUP │ → ADD POOLGRPMAP│ SECTION → POOL(LOCAL)   │
│ +  LACID / S1TACID  │   (LOCATION=  )│   → POOLGROUP           │
│     / N2TACID       │   可叠加APN/SMF │   → POOLBINDGROUP       │
└─────────────────────┘                └─────────────────────────┘
                                                  ↓
                              地址分配规则（SET IPALLOCRULE）
                                  + 基于位置的全局开关（IPALLOCBYLOCGLBSW）
                                  + 白名单豁免（ADRLOCWHITELST）
                                                  ↓
                              OSPF 引入 wlr 发布下行子段路由
```

**10 步通用骨架（仅位置 / 位置+APN 两场景共用，差异在步骤 6/7）**：

| 步骤 | 干啥 | 主要命令 |
|---|---|---|
| 1. License 开关 | 打开 LKV3G5LBAA01 配置开关 | `SET LICENSESWITCH` |
| 2. APN 实例 + 地址分配属性 | VPN/APN + SUPPORTIPV4/V6 | `ADD VPNINST` / `ADD APN` / `SET APNADDRESSATTR` |
| 3. 地址池三级 | LOCAL 地址池 + 段 + 组 + 绑定 | `ADD POOL` / `ADD SECTION` / `ADD POOLGROUP` / `ADD POOLBINDGROUP` |
| 4. TAC 组（可选） | 5G 场景 TAC 位置区组 | `ADD TACGROUP` / `ADD S1TACID`（4G）/ `ADD N2TACID`（5G） |
| 5. LAC 组（可选） | 2G/3G/4G 场景 LAC 位置区组 | `ADD LACGROUP` / `ADD LACID` |
| 6. POOLGRPMAP 映射 | 位置区组（+APN/SMF） → POOLGROUP | `ADD POOLGRPMAP`（LOCATIONGRPTYPE=LAC/TAC） |
| 7. 地址分配规则 | 全局三级规则（可启用回退） | `SET IPALLOCRULE`（FIRSTRULE/SECONDRULE） |
| 8. 位置分配开关 | 全局开关（IPv4/IPv6 分别）+ 指定位置区覆盖 | `SET IPALLOCBYLOCGLBSW` / `SET IPALLOCBYLOCSW`（可选） |
| 9. 下行路由 | OSPF 引入 wlr 子段路由发布到骨干网 | `ADD OSPF` / `ADD OSPFAREA` / `ADD OSPFNETWORK` / `ADD OSPFIMPORTROUTE PROTOCOL=wlr` |
| 10. 位置白名单 | 白名单 MSISDN 不参与位置分配 | `ADD ADRLOCWHITELST` |

**关键约束**：
- **必须加载 License LKV3G5LBAA01**（控制项 82200BAK）并打开 SET LICENSESWITCH
- 必须依赖 GWFD-010105（母特性）
- 仅支持 LOCAL 类型地址池（不支持 EXTERNAL/RADIUS 等）
- 对接的控制面必须为华为配套设备（PFCP 私有扩展）
- **与 L2TP VPN（GWFD-020412 / LKV3G5L2TP01）互斥**

## 配置流程

> 每步显示"干啥 + 含哪些命令"。两场景（仅位置 / 位置+APN）仅在步骤 6 映射和步骤 7 规则字符串有差异。

1. **License 开关**（LKV3G5LBAA01）：
   - `SET LICENSESWITCH` → [0-00019](task/UDG/20.15.2/0-00019.md)（LICITEM="LKV3G5LBAA01", SWITCH=ENABLE）

2. **APN 接入域基础设施**（VPN/APN 实例 + 地址分配属性使能）：
   - → 复用 [1-00029](task/UDG/20.15.2/1-00029.md) APN 接入域基础设施（**SUPPORTIPV4=ENABLE + SUPPORTIPV6=ENABLE 双栈变种**；步骤 1 License LKV3G5LBAA01 前置）

3. **LOCAL 地址池体系**：
   - `ADD POOL` → [0-00043](task/UDG/20.15.2/0-00043.md)（POOLNAME, **POOLTYPE=LOCAL**, HASVPN=ENABLE, VPNINSTANCE）
   - `ADD SECTION` → [0-00044](task/UDG/20.15.2/0-00044.md)（V4STARTIP/V4ENDIP 或 V6PREFIX*）
   - `ADD POOLGROUP` → [0-00042](task/UDG/20.15.2/0-00042.md)（**组合场景配 IPV4ALLOCPRIALG=ENABLE**）
   - `ADD POOLBINDGROUP` → [0-00045](task/UDG/20.15.2/0-00045.md)（**组合场景可配 PRIORITY**）
   - 复用 → [1-00022](task/UDG/20.15.2/1-00022.md) 地址分配体系骨架（POOLTYPE=LOCAL）

4. **TAC 位置区组（5G/4G 场景，按位置类型二选一）**：
   - `ADD TACGROUP` → [0-00316](task/UDG/20.15.2/0-00316.md)（TACGROUPNAME）
   - `ADD S1TACID`（4G）→ [0-00315](task/UDG/20.15.2/0-00315.md)（S1TAC 号段）
   - `ADD N2TACID`（5G）→ [0-00312](task/UDG/20.15.2/0-00312.md)（N2TAC 号段）

5. **LAC 位置区组（2G/3G/4G 场景，按位置类型二选一）**：
   - `ADD LACGROUP` → [0-00310](task/UDG/20.15.2/0-00310.md)（LACGROUPNAME）
   - `ADD LACID` → [0-00311](task/UDG/20.15.2/0-00311.md)（LACSECNUM, LACSTARTID 如 "0x000a", LACENDID 如 "0x000b"）

6. **POOLGRPMAP 位置映射**（按子方式差异）：
   - **仅位置场景**：1 条映射，`ADD POOLGRPMAP` → [0-00047](task/UDG/20.15.2/0-00047.md)（LOCATIONGRPTYPE=LAC/TAC, LOCATIONGRPNAME=lac1/tac1, POOLGROUPNAME=poolgroup1）
   - **位置+APN 场景**：2 条映射（mapping1=LOCATION+APN，mapping2=LOCATION 单维度作回退）

7. **地址分配规则**（全局三级，按组合维度写规则字符串）：
   - `SET IPALLOCRULE` → [0-00250](task/UDG/20.15.2/0-00250.md)
     - 仅位置：`FIRSTRULE=APN-0&LOCATION-1&SMF-0`
     - 位置+APN：`FIRSTRULE=APN-1&LOCATION-1&SMF-0`, `SECONDRULE=APN-0&LOCATION-1&SMF-0`（两级回退）
   - 复用 → [1-00021](task/UDG/20.15.2/1-00021.md) 地址分配规则层

8. **基于位置的开关**（两级控制 + 全局使能）：
   - `SET IPALLOCBYLOCGLBSW` → [0-00321](task/UDG/20.15.2/0-00321.md)（全局开关，IPv4/IPv6 分别）
   - `SET IPALLOCBYLOCSW`（可选）→ [0-00322](task/UDG/20.15.2/0-00322.md)（指定位置区覆盖全局）

9. **下行路由发布**（OSPF 引入 wlr 子段路由）：
   - `ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFNETWORK` + `ADD OSPFIMPORTROUTE PROTOCOL=wlr`
   - 复用 → [1-00024](task/UDG/20.15.2/1-00024.md) 下行路由发布（与 010105/010104 共用）

10. **位置分配白名单**（白名单用户不参与位置分配）：
    - `ADD ADRLOCWHITELST` → [0-00299](task/UDG/20.15.2/0-00299.md)（MSISDN="123456"）

## 决策点

**DP（特性级）：场景 × 位置类型 × 规则级数 × 走法**

| 场景 | 位置类型 | 规则级数 | 走法（compound/atom 编排） |
|---|---|---|---|
| 仅基于位置（LAC） | LACGROUP+LACID | 单级 | [1-00022](task/UDG/20.15.2/1-00022.md)（1 池，POOLTYPE=LOCAL）+ 步骤 5 LAC 组 + [0-00047](task/UDG/20.15.2/0-00047.md)（ADD POOLGRPMAP LOCATION=LAC）+ [1-00021](task/UDG/20.15.2/1-00021.md)（`APN-0&LOCATION-1&SMF-0`）+ [0-00321](task/UDG/20.15.2/0-00321.md)（全局开关 ENABLE）+ [1-00024](task/UDG/20.15.2/1-00024.md) |
| 仅基于位置（TAC 5G） | TACGROUP+N2TACID | 单级 | [1-00022](task/UDG/20.15.2/1-00022.md)（1 池）+ 步骤 4 TAC 组 + [0-00047](task/UDG/20.15.2/0-00047.md)（LOCATION=TAC）+ [1-00021](task/UDG/20.15.2/1-00021.md) + [0-00321](task/UDG/20.15.2/0-00321.md) + [1-00024](task/UDG/20.15.2/1-00024.md) |
| 仅基于位置（TAC 4G） | TACGROUP+S1TACID | 单级 | 同上 TAC 5G，换 S1TACID |
| 位置 + APN | LACGROUP | 两级 | [1-00022](task/UDG/20.15.2/1-00022.md)（1 池，POOLGROUP 配 IPV4ALLOCPRIALG=ENABLE）+ 步骤 5 LAC 组 + [0-00047](task/UDG/20.15.2/0-00047.md)（**2 条 POOLGRPMAP：mapping1=LOC+APN / mapping2=LOC**）+ [1-00021](task/UDG/20.15.2/1-00021.md)（两级规则）+ [0-00321](task/UDG/20.15.2/0-00321.md) + [1-00024](task/UDG/20.15.2/1-00024.md) |
| 位置 + SMF | LACGROUP | 单级 | [1-00022](task/UDG/20.15.2/1-00022.md)（1 池）+ 步骤 5 LAC 组 + [0-00047](task/UDG/20.15.2/0-00047.md)（LOCATION=LAC + SMF=）+ [1-00023](task/UDG/20.15.2/1-00023.md)（CPNODEID + SMF 开关）+ [1-00021](task/UDG/20.15.2/1-00021.md)（`APN-0&LOCATION-1&SMF-1`）+ [0-00321](task/UDG/20.15.2/0-00321.md) + [1-00024](task/UDG/20.15.2/1-00024.md) |
| 位置 + APN + SMF | LACGROUP | 单级 | [1-00022](task/UDG/20.15.2/1-00022.md)（1 池，IPV4ALLOCPRIALG=ENABLE）+ 步骤 5 LAC 组 + [0-00047](task/UDG/20.15.2/0-00047.md)（LOCATION=LAC + APN + SMF）+ [1-00023](task/UDG/20.15.2/1-00023.md) + [1-00021](task/UDG/20.15.2/1-00021.md)（`APN-1&LOCATION-1&SMF-1`）+ [0-00321](task/UDG/20.15.2/0-00321.md) + [1-00024](task/UDG/20.15.2/1-00024.md) |

> **关键规则**：
> - 必须首先 SET LICENSESWITCH（步骤 1），否则特性不工作
> - 必须配 SET IPALLOCBYLOCGLBSW 步骤 8 全局开关 ENABLE，否则即使规则字符串 LOCATION=1 也不触发
> - POOLGRPMAP 的 LOCATIONGRPTYPE 必须与实际网络制式匹配：4G → LAC 或 S1TAC；5G → N2TAC
> - 规则字符串与地址池映射条件必须匹配（`APN-1&LOCATION-1&SMF-0` 则 POOLGRPMAP 必须含 LOCATION+APN）
> - 所有组合共享 [1-00024](task/UDG/20.15.2/1-00024.md) 下行路由发布

## 约束

- **License 必需**（critical，rule-0-00514）：必须加载 License 控制项 LKV3G5LBAA01（控制项编号 82200BAK）并通过 SET LICENSESWITCH 打开开关 — 后果：特性不工作
- **依赖母特性 GWFD-010105**（critical，rule-0-00515）：使用本特性需依赖 GWFD-010105 用户面地址分配 — 后果：地址池体系不可用
- **L2TP VPN 互斥**（critical，rule-0-00516）：不支持与 GWFD-020412 L2TP VPN（License LKV3G5L2TP01 / 控制项 82200BVC）同时应用 — 后果：双向互斥
- **仅 LOCAL 地址池**（critical，rule-0-00517）：基于位置的地址分配只支持 LOCAL 类型地址池（不支持 EXTERNAL/RADIUS 等方式叠加位置维度） — 后果：EXTERNAL POOL + 位置维度映射无效
- **C 面网元限制**（critical）：对接控制面必须为华为解决方案配套版本（PFCP 私有扩展） — 后果：非华为 C 面位置协商失败
- **位置区唯一归属**（critical，rule-0-00518）：一个 LAC、TAC 只能属于一个位置区 — 后果：冲突归属致激活失败
- **规则与映射一致**（warning，rule-0-00519）：规则字符串需与地址池映射条件一致（如 LOCATION-1 则映射必须含 LOCATION） — 后果：规则匹配失败
- **C 面网元必须为华为配套设备**（critical，rule-0-00497）：因 N4/Sxb 对 PFCP 协议有私有扩展 — 后果：非华为 C 面激活失败
- **VPN 三者一致**（critical，rule-0-00498）：地址池 / APN / Gi/SGi/N6 外联口 VPN 一致 — 后果：VPN 不一致致地址分配或路由异常
- **APN/DNN 全网规划一致**（critical，rule-0-00499）：UDG 本端 APN/DNN 实例应与 C 面配置一致 — 后果：PFCP 信令失配
- **每 APN 绑池数 ≤16**（warning，rule-0-00500）：后果：超规格新增失败
- **每 POOL 绑段数 ≤64**（warning，rule-0-00501）：后果：超规格新增失败
- **主机路由规格 40 万**（warning，rule-0-00502）：后果：超规格下行路由发布异常
- **三级规则匹配失败则地址分配失败**（critical，rule-0-00506）：后果：用户激活失败
- **ROUTER ID 全网唯一**（critical，rule-0-00508）：后果：OSPF 邻接建立失败
- **UDG 用户面起始版本 20.2.0**（warning）：晚于母特性 GWFD-010105 的 20.0.0 — 后果：低版本网元不支持
- **白名单豁免**（info）：白名单中 MSISDN 用户走普通非位置敏感的地址分配逻辑 — 后果：白名单用户不受位置切换影响
