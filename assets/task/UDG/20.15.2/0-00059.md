# 配置GRE隧道（ADD GRETUNNEL）

> 命令静态知识见 [ADD GRETUNNEL](command/UDG/20.15.2/ADD-GRETUNNEL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。TNLNAME 引用 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 的 Tunnel IFNAME；SRCIFNAME 引用本端源 LoopBack；DSTIPADDR/DSTIPV6ADDR 引用对端 LoopBack 的 IP（来自 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS）。

## 配置方法（配置方法字典）

ADD GRETUNNEL 是"GRE 隧道创建"命令。通用路由封装 GRE（Generic Routing Encapsulation）是对某些网络层协议报文封装，使其能在另一网络层协议（如 IP）中传输——可作为 VPN 第三层隧道。核心配置空间有 4 个维度：**隧道类型**（TNLTYPE 决定 IPv4/IPv6 链路）、**源类型**（SRCTYPE 决定源 IP 来源方式）、**目的地址**（DSTIPADDR/DSTIPV6ADDR 引对端 LoopBack IP）、**业务增强**（Keepalive/Redundancy/GREKey/Checksum/State 5 类开关）。3 个特性命中（010107 静态路由冗余、IPFD-015002 GRE、IPFD-015004 IPsec over GRE）覆盖三种典型场景。

### 配置维度 1：隧道类型（参数 TNLTYPE，必选）

TNLTYPE 决定后续参数走 IPv4 链路还是 IPv6 链路：

| 取值 | 联动必选参数 | 适用场景 |
|---|---|---|
| **gre**（默认） | SRCTYPE + SRCIPADDR/SRCIFNAME + DSTIPADDR | IPv4 GRE 隧道（010107、IPFD-015002、IPFD-015004 默认范式） |
| **gre6** | SRCTYPE6 + SRCIPV6ADDR/SRCIFNAME + DSTIPV6ADDR | IPv6 GRE 隧道 |

### 配置维度 2：源类型（参数 SRCTYPE/SRCTYPE6，TNLTYPE 决定条件必选）

源类型决定封装后报文的源 IP 设置方式：

| 取值 | 联动必选参数 | 适用场景 |
|---|---|---|
| **if_name**（默认） | `SRCIFNAME` | 源用接口名（本端源 LoopBack 的 IFNAME）；SRCIFNAME 接口的 IP 作为封装源 IP（绝大多数部署） |
| **ip_address** | `SRCIPADDR` | 源用 IP 地址（直接指定封装源 IP） |
| **no_type** | — | 不配置源类型 |

> **关键铁律**：本端配置的源 IP **必须与隧道对端配置的目的 IP 相同**，隧道才能正常建立；不能对两个或两个以上使用同种封装协议的 Tunnel 接口配置完全相同的源 IP 和目的地址。

### 配置维度 3：目的地址（参数 DSTIPADDR/DSTIPV6ADDR + DSTVPNNAME）

| 参数 | 必选性 | 取值 | 联动 |
|---|---|---|---|
| **DSTIPADDR** | TNLTYPE=gre 时条件可选 | 对端 LoopBack 的 IFIPADDR | 引 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS |
| **DSTIPV6ADDR** | TNLTYPE=gre6 时条件可选 | 对端 LoopBack 的 IFIPV6ADDRESS | 引 [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS |
| **DSTVPNNAME** | 可选 | 1~31 字符串（VPN 实例名） | 引 [0-00041](task/UDG/20.15.2/0-00041.md) ADD L3VPNINST 的 VPNINSTANCE；SRCTYPE=ip_address 时源 IP 自动使用该 VPN；=if_name 时源接口不受影响 |

### 配置维度 4：业务增强（5 类开关，可选）

| 开关参数 | 取值/范围 | 默认 | 作用 |
|---|---|---|---|
| **KEEPALVEN** | TRUE / FALSE | FALSE | 周期性发送 Keepalive，对端不可达则 Tunnel 接口状态置 Down |
| **KEEPALVPERIOD** | 1~32767 秒（仅 KEEPALVEN=TRUE 时） | 5 | Keepalive 报文发送周期 |
| **KEEPALVRETRYCNT** | 1~255 次（仅 KEEPALVEN=TRUE 时） | 3 | 不可达计数器参数；到达此值仍无响应判定对端不可达 |
| **GREKEYEN** + **GREKEY** | TRUE+0~4294967295 / FALSE | FALSE | 识别关键字使能；使能后两端 KEY 必须相同（010107 备份隧道场景通常不配，由 [MOD GRETUNNEL](command/UDG/20.15.2/MOD-GRETUNNEL.md) 后续使能） |
| **CHECKSUMEN** | TRUE / FALSE | FALSE | 端到端校验；使能后对端计算校验和，不一致丢弃 |
| **STATENABLE** | TRUE / FALSE | FALSE | GRE 隧道接口流量统计 |
| **REDUNDANCYEN** | TRUE / FALSE（仅 TNLTYPE=gre/gre6 时可选） | FALSE | 备份静态地址用户流量功能（010107 备份隧道场景必配 TRUE） |

> **Keepalive 单向性**：GRE 隧道 Keepalive 是单向的，要使两端都具备 Keepalive 功能需在两端都使能；只在业务模块选择承载隧道时可防止选择对端不可达的 GRE 隧道。

**典型脚本（010107 静态路由冗余备份，GRE + Keepalive + Redundancy + DSTVPNNAME）**：
```
ADD GRETUNNEL: TNLNAME="Tunnel1", SRCTYPE=if_name, DSTIPADDR="10.10.0.1", TNLTYPE=gre, SRCIFNAME="LoopBack1", DSTVPNNAME="vpn2", REDUNDANCYEN=TRUE, KEEPALVEN=TRUE;
```

**典型脚本（IPFD-015002 基本 GRE，仅必填参数）**：
```
ADD GRETUNNEL:TNLNAME="Tunnel1", TNLTYPE=gre, SRCTYPE=if_name, SRCIFNAME="LoopBack1", DSTIPADDR="10.4.4.11";
```

**典型脚本（IPFD-015004 GRE over IPsec，校验类全 FALSE）**：
```
ADD GRETUNNEL:TNLNAME="Tunnel1",SRCTYPE=if_name,DSTIPADDR="10.102.105.224",KEEPALVEN=FALSE,TNLTYPE=gre,SRCIFNAME="LoopBack1",GREKEYEN=FALSE,CHECKSUMEN=FALSE,STATENABLE=FALSE;
```

> 命令 notes：**最大记录数 4096**；**执行后立即生效**；**TNLNAME 命名约束**：字符串形式为 `Tunnel+接口编号`，接口编号为一维或三维（`X/Y/Z`）；本端配置的源 IP **必须与隧道对端配置的目的 IP 相同**——这是隧道建立的硬约束。

**步骤位置**：在所属 feature 编排中通常位于"步骤 4：建 GRE 隧道"——前置 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE（建 Tunnel 接口 + LoopBack）+ [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS（配 Tunnel/LoopBack IP）+ [0-00041](task/UDG/20.15.2/0-00041.md) ADD L3VPNINST（DSTVPNNAME 引用）；下游 [MOD GRETUNNEL](command/UDG/20.15.2/MOD-GRETUNNEL.md)（后续使能校验/Key）；[0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE（备份路由）。

## 决策点：GRE 隧道场景选择（DP 0-00059）

问：该 GRE 隧道用于哪种场景？（决定 REDUNDANCYEN/KEEPALVEN/DSTVPNNAME/校验类开关取值）

| 选项 | REDUNDANCYEN | KEEPALVEN | DSTVPNNAME | CHECKSUMEN/GREKEYEN | STATENABLE | 典型场景 |
|---|---|---|---|---|---|---|
| 静态地址用户路由冗余备份（010107） | TRUE | TRUE | 必填 | 不配 | 不配 | 备份静态地址用户流量 |
| 基本 GRE 隧道（IPFD-015002） | FALSE（默认） | FALSE（默认） | 不配 | 可选 TRUE（MOD 后续使能） | 不配 | 普通 GRE 隧道 |
| GRE over IPsec（IPFD-015004） | FALSE | FALSE | 不配 | FALSE | FALSE | IPsec 加密承载 GRE |

> 3 个特性命中（010107/IPFD-015002/IPFD-015004）覆盖三种范式；校验/Key 类通常由 [MOD GRETUNNEL](command/UDG/20.15.2/MOD-GRETUNNEL.md) 后续使能；两端 GREKEY 必须一致。

## 约束

- **最大记录数 4096**（warning，rule-0-00459）：GRE 隧道最大 4096 条 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00459）：命令执行后立即生效 — 后果：误操作即刻生效
- **本端源 IP = 对端目的 IP（硬约束）**（critical，rule-0-00459）：本端配置的源 IP 必须与隧道对端配置的目的 IP 相同，隧道才能正常建立 — 后果：源/目的 IP 不匹配隧道无法建立
- **同封装协议不可重源+目的**（critical，rule-0-00459）：不能对两个或两个以上使用同种封装协议的 Tunnel 接口配置完全相同的源 IP 和目的地址 — 后果：配置冲突
- **DSTVPNNAME 必先建 VPN**（critical，rule-0-00459）：DSTVPNNAME 引用的 VPN 必须先由 ADD L3VPNINST 命令定义 — 后果：VPN 未建引用失败
- **GREKEY 两端必须一致**（critical，rule-0-00459）：GREKEYEN=TRUE 时两端 GREKEY 必须完全一致（明文仅数字 0~4294967295） — 后果：KEY 不一致报文丢弃
- **Keepalive 单向性**（info，隐含）：GRE 隧道 Keepalive 是单向的，要两端都具备需两端都使能 — 后果：仅一端使能时另一端 Tunnel 状态不对齐
- **TNLNAME 命名约束**（warning，隐含）：TNLNAME 字符串形式为 Tunnel+接口编号（一维或三维 X/Y/Z） — 后果：不符合命名规范命令拒收
