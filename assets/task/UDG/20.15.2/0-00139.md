# 设置APN PPP配置（SET APNPPPACCESS）

> 命令静态知识见 [SET APNPPPACCESS](command/UDG/20.15.2/SET-APNPPPACCESS.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**适用 NF：PGW-U、UPF**。

## 配置方法

SET APNPPPACCESS 为指定 APN 配置 PPP 接入相关参数。**APN 必选**（1~63 字符）、**AUTHENTICATION 必选**（DISABLE/ENABLE）；**MRU 可选**（64~1500，默认走系统 [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG 缺省）。核心配置空间有 3 个维度：**APN 定位**（APN）、**PPP 鉴权使能**（AUTHENTICATION）、**APN 级 MRU**（MRU）。1 个特性命中（GWFD-020412 L2TP VPN）仅配 AUTHENTICATION=ENABLE，使用系统 MRU。

### 配置维度 1：APN 定位（参数 APN，必选）

| 取值 | 作用 |
|---|---|
| **已建 APN 实例名**（ref [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN） | 本 PPP 配置归属的 APN（020412 范式 `apn-l2tp`） |

> APN 必须已通过 ADD APN 创建；同 APN 重复执行本命令则覆盖前次配置。

### 配置维度 2：PPP 鉴权使能（参数 AUTHENTICATION，必选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **DISABLE** | APN 不支持 PPP 鉴权 | 不需要 PPP 接入的 APN |
| **ENABLE** | APN 支持 PPP 鉴权（020412 范式） | PPP/L2TP 用户接入 APN |

> ENABLE 后用户接入走 PPP + LCP + 鉴权；DISABLE 时不发起 PPP 鉴权流程。

### 配置维度 3：APN 级 MRU（参数 MRU，可选）

| 取值/范围 | 作用 |
|---|---|
| **64~1500 整数** | APN 内 PPP 协商的最大接收单元（覆盖 [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG 的全局 MRU） |
| **省略** | 使用 SET PPPCFG 的全局 MRU（020412 范式省略） |

> APN 级 MRU 与全局 MRU 取**小者**；MRU 仅设置上限。

**典型脚本（020412 L2TP VPN，APN apn-l2tp 仅开 PPP 鉴权）**：
```
SET APNPPPACCESS:APN="apn-l2tp",AUTHENTICATION=ENABLE;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：APN PPP 接入"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 建 APN + [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR（本地方式下含 L2TPSWITCH=ENABLE）；AAA 方式下与 [0-00134](task/UDG/20.15.2/0-00134.md) ADD L2TPRDSCLIENT + [0-00131](task/UDG/20.15.2/0-00131.md) SET GLOBALL2TP + [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG 配合；本地方式下与 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP + [0-00135](task/UDG/20.15.2/0-00135.md) ADD L2TPCLIENTIP + [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG 配合。

## 决策点：APN PPP 鉴权与 MRU 范围（DP 0-00139）

问：本 APN 是否启用 PPP 鉴权？是否需要 APN 级 MRU 覆盖？

| 选项 | AUTHENTICATION | MRU | 适用场景 |
|---|---|---|---|
| **开 PPP 鉴权 + 用全局 MRU**（020412 范式） | ENABLE | 省略（走 SET PPPCFG 全局 MRU） | L2TP 用户 PPP 接入，使用全局 MRU |
| **关 PPP 鉴权** | DISABLE | 省略 | 不需要 PPP 接入的 APN |
| **开 PPP 鉴权 + 自配 MRU** | ENABLE | 64~1500 | 该 APN 需特定 MRU 覆盖全局 MRU |

> 1 个特性命中（020412）；020412 范式仅配 AUTHENTICATION=ENABLE，MRU 省略走全局。

## 约束

- **APN 必须已建**（critical，rule-0-00139）：APN 必须已通过 `ADD APN` 创建 — 后果：APN 不存在命令拒收
- **AUTHENTICATION 必选**（critical，rule-0-00139）：AUTHENTICATION 是必选参数 — 后果：缺 AUTHENTICATION 命令拒收
- **MRU 范围**（warning，rule-0-00139）：MRU 取值 64~1500；省略走 [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG 全局 MRU — 后果：超范围命令拒收
- **APN 名称约束**（info，rule-0-00139）：APN 1~63 字符，不支持空格与特殊字符 — 后果：超长或含特殊字符命令拒收
- **适用 NF**（info，rule-0-00139）：本命令仅适用 PGW-U、UPF — 后果：误在 SGW-U 上执行命令拒收
- **与 SET PPPCFG 协同**（warning，rule-0-00139）：本命令 APN MRU 与 [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG 全局 MRU 取小者 — 后果：APN MRU 与全局 MRU 配合不当导致 PPP 协商失败
- **执行后立即生效**（warning，rule-0-00139）：本命令执行后立即生效 — 后果：误配即刻影响该 APN 下用户 PPP 接入
