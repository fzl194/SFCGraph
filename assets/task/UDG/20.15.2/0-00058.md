# 配置接口IPv4地址（ADD IFIPV4ADDRESS）

> 命令静态知识见 [ADD IFIPV4ADDRESS](command/UDG/20.15.2/ADD-IFIPV4ADDRESS.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。IFNAME 引用 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 已建的接口名。

## 配置方法（配置方法字典）

ADD IFIPV4ADDRESS 是"接口 IPv4 地址配置"命令：给逻辑接口（LoopBack/Tunnel）和物理接口（Ethernet）配 IPv4 地址 + 掩码。核心配置空间有 3 个维度：**接口类型**（决定掩码形态）、**地址类型**（ADDRTYPE main/sub）、**掩码形态**（/32 主机路由 vs /24 网段）。3 个特性命中（010107、IPFD-015002、IPFD-015004）覆盖 LoopBack/Tunnel/Ethernet 三类接口范式。

### 配置维度 1：接口类型（参数 IFNAME，必选）

IFNAME 引用已配的逻辑/物理接口，**不同接口类型决定掩码形态与地址角色**：

| 接口类型 | 典型 IFNAME | 掩码形态 | 地址角色 | 典型场景 |
|---|---|---|---|---|
| **LoopBack** | `LoopBack<n>` | `/32`（255.255.255.255，主机路由） | 路由标识，本端源地址 | GRE/IPsec 本端源、OSPF router-id、010107 静态路由冗余 |
| **Tunnel** | `Tunnel<n>` | `/24`（网段）或 `/32`（主机路由） | 隧道本端/对端 IP | GRE/IPsec 隧道承载（IPFD-015002/015004） |
| **以太网外联口** | `Ethernet<slot>/<card>/<port>` | `/24`（网段） | 外联口 IP | N6/Gi/SGi 外联（IPFD-015004 范式） |

> **掩码硬约束**：**除 Loopback 接口、Tunnel 接口外**，其它接口**不能**配置 255.255.255.255 的掩码（即 LoopBack/Tunnel 可 /32，其他接口必须 < /32）。

### 配置维度 2：地址类型（参数 ADDRTYPE，可选，默认 main）

| 取值 | 作用 | 配对约束 |
|---|---|---|
| **main**（默认） | 主地址——每个主用接口只能有 1 个 | 与 sub 互斥（主从可同时存在，主 1 + 从最多 255） |
| **sub** | 从地址——可配置多个 | 最多 255 个从地址 |

> 3 个特性命中脚本**均显式 ADDRTYPE=main**（主地址），从地址在 UDG 场景实例化不用。

### 配置维度 3：掩码形态（参数 SUBNETMASK，必选）

| 掩码形态 | 取值 | 适用接口 |
|---|---|---|
| **/32 主机路由** | `255.255.255.255` | Loopback、Tunnel（**仅这两类允许**） |
| **/24 网段** | `255.255.255.0` | Tunnel、以太网外联口 |
| **其他网段** | 0.0.0.0~255.255.255.255（除 0.0.0.0） | 按规划网段取值（需本端/全局规划） |

> IPv4 地址不能在 0.0.0.0、255.255.255.255、127.0.0.1；必须是 A/B/C 类地址；**系统内必须唯一**。

**典型脚本（010107 静态路由冗余，LoopBack + Tunnel 组合）**：
```
ADD IFIPV4ADDRESS: IFNAME="LoopBack1", IFIPADDR="10.11.1.1", SUBNETMASK="255.255.255.255";
ADD IFIPV4ADDRESS: IFNAME="Tunnel1", IFIPADDR="10.11.1.3", SUBNETMASK="255.255.255.0";
```

**典型脚本（IPFD-015002 GRE，LoopBack /32 + Tunnel /24）**：
```
ADD IFIPV4ADDRESS:IFNAME="LoopBack1", IFIPADDR="10.3.3.11", SUBNETMASK="255.255.255.255", ADDRTYPE=main;
ADD IFIPV4ADDRESS:IFNAME="Tunnel1", IFIPADDR="10.10.1.201", SUBNETMASK="255.255.255.0", ADDRTYPE=main;
```

**典型脚本（IPFD-015004 IPsec，以太网外联 + Tunnel）**：
```
ADD IFIPV4ADDRESS:IFNAME="Ethernet66/0/5", IFIPADDR="10.1.1.1", SUBNETMASK="255.255.255.0";
ADD IFIPV4ADDRESS:IFNAME="Tunnel10", IFIPADDR="192.168.1.1", SUBNETMASK="255.255.255.255";
```

> 命令 notes：**最大记录数 65535**；**执行后立即生效**；**版本升级过程中禁止执行**；每主用接口最多 256 个 IP（1 主 + 255 从）；本命令参与自动化配置——下发前需 `DSP OPSASSISTSTATE` 查询 autoscaling_autoconfig.py；与 [0-00059](task/UDG/20.15.2/0-00059.md) ADD GRETUNNEL 协同——DSTIPADDR 引用对端 LoopBack 的 IFIPADDR。

**步骤位置**：在所属 feature 编排中通常位于"步骤 3：配置接口 IPv4 地址"——前置 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE（建接口）+ [0-00057](task/UDG/20.15.2/0-00057.md) ADD IPBINDVPN（绑 VPN），下游 [0-00059](task/UDG/20.15.2/0-00059.md) ADD GRETUNNEL（DSTIPADDR 引对端 LoopBack IP）。

## 决策点：接口 IP 地址与掩码规划来源（DP 0-00058）

问：ADD IFIPV4ADDRESS 的 IFIPADDR/SUBNETMASK 采用何种规划来源？（两者联动，由同一决策决定）

| 选项 | 联动 | 适用场景 |
|---|---|---|
| 全局规划（全网协调） | IFIPADDR/SUBNETMASK 全网唯一 | 承载路由协议的 LoopBack/Tunnel IP（010107 静态路由冗余——对端需知悉以建 GRE/静态路由） |
| 本端规划 | IFIPADDR/SUBNETMASK 本端分配 | GRE/IPsec 隧道本端 Tunnel/以太网外联口（IPFD-015002/015004 范式） |

> 3 个特性命中（010107/IPFD-015002/IPFD-015004）覆盖全局规划+本端规划两种范式；默认全局规划（路由协调场景口径）。

## 约束

- **最大记录数 65535**（warning，rule-0-00458）：接口 IPv4 地址最大 65535 条 — 后果：超规格新增失败
- **每主用接口最多 256 个 IP**（warning，rule-0-00458）：1 主 + 255 从 = 256 个 IP — 后果：超额命令拒收
- **执行后立即生效**（warning，rule-0-00458）：命令执行后立即生效 — 后果：误操作即刻生效
- **版本升级过程禁止执行**（critical，rule-0-00458）：版本升级过程中禁止执行本命令 — 后果：升级期间执行中断升级流程
- **参与自动化配置**（critical，rule-0-00458）：IFIPADDR/SUBNETMASK 参与自动化配置——下发前需 `DSP OPSASSISTSTATE` 查询 autoscaling_autoconfig.py — 后果：自动化配置过程中执行致最终生效配置不可预期
- **/32 掩码接口限定**（critical，rule-0-00458）：255.255.255.255 掩码仅允许 Loopback/Tunnel 接口配置，其他接口不能 — 后果：非 Loopback/Tunnel 配 /32 命令拒收
- **IPv4 地址不可取值**（critical，rule-0-00458）：不能配置为 0.0.0.0、255.255.255.255、127.0.0.1；必须是 A/B/C 类地址；系统内必须唯一 — 后果：非法地址命令拒收/地址冲突
- **接口需支持配 IP**（warning，隐含）：接口必须支持配置 IP 地址 — 后果：Loopback/Null0 等不可配 IP 的接口命令拒收
