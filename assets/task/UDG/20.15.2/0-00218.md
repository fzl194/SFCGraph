# 配置PFCP负荷上报（SET PFCPLOADRPT）

> 命令静态知识见 [SET PFCPLOADRPT](command/UDG/20.15.2/SET-PFCPLOADRPT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

单命令原子，全局唯一记录（SET 修改）。激活 GWFD-020154 第二步：开负荷上报开关并设阈值。UDG 将负荷封装进 LCI（Load Control Indication），随 PFCP Session Establishment/Modification/Deletion Response 及 Session Report Request 上报控制面。适用 SGW-U/PGW-U/UPF。

**数据规划取值样例**（证据③，GWFD-020154/激活用户面负载信息上报_88713326.md 数据规划表）：

| 参数 | 含义 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|---|
| LOADRPTSW | 负荷上报开关 | ENABLE | 固定取值 | 本特性目的即开启上报，固定 ENABLE |
| RPTTHD | 上报阈值 (%) | 50 | 本端规划 | 负荷达到时开始上报 |
| STOPTHD | 停止上报阈值 (%) | 45 | 本端规划 | 负荷回落到此值时停止上报 |
| RANGETHD | 变化范围阈值 | 5 | 本端规划 | 负荷变化幅度门限 |

**示例脚本**（取自激活 md）：

```
SET PFCPLOADRPT:LOADRPTSW=ENABLE,RPTTHD=50,STOPTHD=45,RANGETHD=5;
```

**配置要点**：
- 系统初始记录为 `LOADRPTSW=DISABLE/RPTTHD=50/STOPTHD=45/RANGETHD=5`——本特性只需把开关从 DISABLE 翻成 ENABLE，三个阈值常与初始值一致。
- 阈值设计铁律：STOPTHD（45）必须 < RPTTHD（50），否则上报/恢复机制失效；RANGETHD（5）独立配套，表示波动幅度。
- 前置：先开 License（SET LICENSESWITCH），再执行本命令。
- 上报不单独触发 PFCP Session Report，只随既有 PFCP 消息捎带 LCI。

## 约束

- 最大记录数 1（全局唯一）；该命令执行后立即生效。
- STOPTHD 必须 < RPTTHD（典型 45/50），RANGETHD 独立（典型 5）。
- 系统初始 LOADRPTSW=DISABLE，需显式置 ENABLE 才生效。
- LOADRPTSW 未启用 → 控制面无法获知网元负荷，无法据此选路。
