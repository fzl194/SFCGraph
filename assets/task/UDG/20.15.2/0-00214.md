# 配置IKE对等体（ADD IKEPEER）

> 命令静态知识见 [ADD IKEPEER](command/UDG/20.15.2/ADD-IKEPEER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

定义 IKE 对等体：标识、PSK 密钥、引用 IKE 提议、本端/远端地址、SA/出向 VPN 绑定、交换模式、NAT 穿越。下游 IPsec 策略经 [ADD ATTACHIKEPEER](command/UDG/20.15.2/ADD-ATTACHIKEPEER.md) 绑定本对等体。

**数据规划取值样例**（证据③，IPFD-015004 各激活 md 数据规划表）：

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| PEERNAME（对等体名称） | `a`、`b`、`c`、`1`（主备/多 Peer 各建一个） | 对端规划/本端规划 | 标识，1~15 字符不区分大小写 |
| PRESHAREDKEY（预共享密钥） | `abcde` | 对端协商 | PSK 认证必填，**两端须相同** |
| PROPOSAL（安全提议） | `1`、`10` | 引用（[0-00213](0-00213.md)） | 引用 IKE 安全提议号 |
| LOWREMOTEADDR（远程地址） | `192.168.1.2`、`10.102.101.21`；OSPF 场景用 loopback 口 | 全网规划 | 不能为地址段 |
| INVRFNAME（SA 绑定 VPN 名称） | `vrf1` | 引用（[0-00210](0-00210.md)） | SA 绑定的 IPsec 微服务 VPN |
| OUTVRFNAME（远程地址 VPN 名称） | `vrf1` | 引用（[0-00210](0-00210.md)） | 远程地址所在 VPN |
| EXCHANGEMODE（交换模式） | `Main`（主模式） | 对端协商 | optional，国密 IKEv1 仅支持主模式 |
| NATTRAVERSAL（NAT 穿越） | `TRUE`（两 IPsec 设备间存在 NAT 设备时） | 本端规划 | NAT 场景必须 TRUE |
| LOCALIDTYPE（本地 ID 类型） | `Ip` | 本端规划 | optional，缺省 Ip |

> REMOTEID/VERSION1/VERSION2/AUTHADDRESS/CERTLOCALFILE/ENCCERTLOCFILE 等 13 个冷门参数仅在 RSA 证书认证/DPD/IKEv1 显式/地址段认证等场景用，PSK 主场景不绑（VERSION1/VERSION2 + 证书仅国密 IKEv1 场景显式带）。

**典型范式**（证据③任务示例脚本，PSK + NAT 穿越，普通 IPv4 IPsec 隧道）：
```
ADD IKEPEER: PEERNAME="b", PRESHAREDKEY="abcde", NATTRAVERSAL=TRUE, PROPOSAL=10,
    LOWREMOTEADDR="192.168.1.2", INVRFNAME="vrf1", OUTVRFNAME="vrf1";
```

**典型范式**（GRE/OSPF 场景，带交换模式 + 本地 ID）：
```
ADD IKEPEER: PEERNAME="b", PRESHAREDKEY="abcde", EXCHANGEMODE=Main, PROPOSAL=1, LOCALIDTYPE=Ip,
    LOWREMOTEADDR="10.102.101.21", INVRFNAME="vrf1", OUTVRFNAME="vrf1";
```

> 主备隧道场景（10004）Device A 侧建两个对等体（`b`+`c`），分别对应主备隧道；下游 IPSECPOLICY 经 ATTACHIKEPEER 配 PEERPRIORITY 区分主备。

## 约束

- **rule-0-00338**（critical）命令立即生效；PSK 认证必须配 PRESHAREDKEY 且与对端相同；IKE 对等体被 IPsec 安全策略引用时必须配对端地址（LOWREMOTEADDR），**对端地址不能配置为地址段**；PROPOSAL 引用前置 [ADD IKEPROPOSAL](0-00213.md)（0-00213:PROPOSALNUMBER）；INVRFNAME/OUTVRFNAME 引用 IPsec 微服务 VPN（[0-00210](0-00210.md):VRFNAME）；NAT 设备场景必须 NATTRAVERSAL=TRUE；IKEv1 国密 IPSEC 仅支持主模式、不支持 NAT 穿越、不支持主备隧道。
