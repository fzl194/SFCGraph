# IPv6承载-OSPFv3+WLR（IPv6下行路由发布）

> 归属 [2-00040](task/UDG/20.15.2/2-00040.md) IPv6承载上下文（GWFD-020401）。本 compound 配置**IPv6 下行路由发布层**——通过 OSPFv3 进程+区域+引入用户路由（PROTOCOL=wlr），将 UDG 内部生成的 IPv6 WLR 路由（用户面）发布到骨干网。被引用于 {2-00040}。

> **★ 复用警示**：本骨架**仅适用于 IPv6 场景**（OSPFv3，命令族为 OSPFV3/OSPFV3AREA/OSPFV3IMPORTROUTE）。IPv4 场景下行路由发布走 [1-00024](task/UDG/20.15.2/1-00024.md) 下行路由发布（OSPF/OSPFAREA/OSPFNETWORK/OSPFIMPORTROUTE 4 步），**两套命令族勿混用**。

> **★ 与 GWFD-020403（IPv4v6 双栈）的关键差异**：GWFD-020403 激活脚本**不配 OSPF/OSPFv3 路由发布**——IPv4v6 双栈特性不解决下行路由发布问题，**下行路由发布由本骨架承载**。仅 IPv4v6 双栈场景下，本骨架与 020403 联动使用。

> **★ 与 GWFD-020406（IPv6 Prefix Delegation）的强依赖**：GWFD-020406（PD 前缀代理）依赖 GWFD-020401 提供的 IPv6 承载基础设施——本骨架是 020406 文档显式声明的"强制前置"——020406 必须在本骨架基础上叠加 PD 专用地址池（< 64 位前缀）。

## 配置方法

本 compound 由 3 条命令组成（多命令可复用模块，建 compound 承载）：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 创建 OSPFv3 进程（IPv6） | [ADD OSPFV3](task/UDG/20.15.2/0-00078.md) | `PROCID` / `VRFNAME` / `ROUTERID`（全网唯一 IPv4 点分十进制） |
| 2. 创建 OSPFv3 区域 | [ADD OSPFV3AREA](task/UDG/20.15.2/0-00079.md) | `PROCID` / `AREAID` |
| 3. 配置 OSPFv3 引入外部路由（PROTOCOL=wlr 引入 WLR 用户路由） | [ADD OSPFV3IMPORTROUTE](task/UDG/20.15.2/0-00080.md) | `PROCID` / `TOPOID` / `PROTOCOL=wlr` / `ROUPOLINAME`（可选，引用路由策略） |

**典型脚本（020401 场景：IPv6 承载上下文 WLR 路由发布）**：
```
ADD OSPFV3: PROCID=100, ROUTERID="10.10.10.1", VRFNAME="VPN_Internet";
ADD OSPFV3AREA: PROCID=100, AREAID="0.0.0.1";
ADD OSPFV3IMPORTROUTE: PROCID=100, TOPOID=0, PROTOCOL=wlr, ROUPOLINAME="policy1";
```

**典型脚本（020406 IPv6 PD 场景：PD 路由发布沿用本骨架）**：
```
ADD OSPFV3: PROCID=6, ROUTERID="10.8.25.1", VRFNAME="vpn1", BFDALLINTFFLG=TRUE, VPNINSCAPSIMFLG=TRUE, VIRTUALSYSFLAG=TRUE;
ADD OSPFV3AREA: PROCID=6, AREAID="0.0.0.1";
ADD OSPFV3IMPORTROUTE: PROCID=6, TOPOID=0, PROTOCOL=wlr, ROUPOLINAME="policy_pd";
```

## 场景差异

| 场景 | OSPFv3 命令集 | BFD/VPN 多实例 | ROUTERID 规划 | 备注 |
|---|---|---|---|---|
| IPv6 单栈承载（020401 范式） | 3 条全配 | 纯三层接入（不配 BFD/VPNINSCAPS） | 全网规划 10.10.10.1 范式 | 020401 典型 |
| IPv6 + IPv6 PD 联合部署（020401 + 020406） | 3 条全配 | **配 BFDALLINTFFLG=TRUE + VPNINSCAPSIMFLG=TRUE + VIRTUALSYSFLAG=TRUE**（VPN 多实例+共网段） | 全网规划（020406 范式 10.8.25.1） | **★ 共网段须进程级+接口级（[0-00117](task/UDG/20.15.2/0-00117.md) OSPFV3INTERFACE）同时配 VIRTUALSYSFLAG** |
| IPv4v6 双栈（020401 + 020403） | 3 条全配 | 纯三层 | 全网规划 | 020403 不解决下行路由，本骨架补充 |

> **★ 复用差异双向回填**：
> - **020406 共网段场景必须配 BFD/VPN 多实例/共网段三件套**——文档未单列但 020406 范式必须；本骨架"场景差异"已显式记录
> - **020406 共网段生效**须进程级（ADD OSPFV3 VIRTUALSYSFLAG）+ 接口级（ADD OSPFV3INTERFACE VIRTUALSYSFLAG）**同时配**，单配进程级不生效（[0-00078](task/UDG/20.15.2/0-00078.md) 约束）
> - **020406 PD 场景**的 ROUTERID 沿用全网规划（与 020401 范式同源），但 PROCID 取 6（PD 范式与单栈范式 PROCID 可不同）
> - **ROUPOOLINAME**（路由策略）020401 场景配 policy1 匹配 nssaExternal1；020406 场景配 policy_pd 匹配 PD 专用路由类型

## 决策点：OSPFv3 进程配置策略（DP 0-00340）

问：ADD OSPFV3 的 ROUTERID / VRFNAME / VPNINSCAPSIMFLG / VIRTUALSYSFLAG 取何策略？

| 选项 | ROUTERID | VRFNAME | VPNINSCAPSIMFLG | VIRTUALSYSFLAG | 适用场景 |
|---|---|---|---|---|---|
| **Internet 接入（020401 范式）** | 全网规划 IPv4 | ref `VPN_Internet` | FALSE | TRUE（默认） | GWFD-020401 纯 IPv6 接入 |
| **PD VPN 多实例（020406 范式）** | 全网规划 IPv4 | ref `vpn1` | TRUE | TRUE | GWFD-020406 IPv6-PD 引入 + BFD 加速 |
| **PD 共网段（020406 深度范式）** | 全网规划 IPv4 | ref `vpn1` | TRUE | TRUE（进程级+接口级同时配） | GWFD-020406 + VPN 多实例+共网段 |

> **关键规则**：ROUTERID **必配且全网唯一**（否则进程无法生成 LSA）；VRFNAME 须在首次创建时确定，后续不可修改；**VPN 多实例场景 VPNINSCAPSIMFLG=TRUE 必配**去使能 VPN 路由环路检测；**共网段须进程级 + 接口级 VIRTUALSYSFLAG 同时配**（进程级单独配不生效）。

## 约束

- **ROUTER ID 全网唯一**（critical，rule-0-01600）：OSPFv3 ROUTERID 必须全网唯一 — 后果：OSPFv3 邻接关系建立失败
- **ROUTERID 必配**（critical，rule-0-01600）：OSPFv3 进程下未配 ROUTERID 则无法生成 LSA — 后果：进程能建但无 LSA
- **PROTOCOL=wlr 必须**（critical，rule-0-01600）：OSPFV3IMPORTROUTE 的 PROTOCOL 必须为 `wlr`（用户路由） — 后果：未配 wlr 致下行路由不发布
- **WLR 路由发布完整性**（critical，rule-0-01600）：采用路由策略方式发布路由时，**不能只选 WLR_SP 或 WLR_UD**，必须选 WLR 方式发布所有路由（文档操作步骤9"说明"原文强制约束） — 后果：下行路由只发布部分用户
- **VRFNAME 与外联口 VPN 一致**（critical，rule-0-01600）：OSPFv3 进程的 VRFNAME 与外联口 L3VPNINST 的 VRFNAME 必须一致 — 后果：VPN 不一致致下行路由不在目标 VRF 中
- **进程 VRFNAME 不可更改**（critical，rule-0-01600）：进程实例 VRFNAME 仅第一次使能时指定，后续不可修改 — 后果：错填 VRFNAME 必须删除进程并重建
- **共网段须进程级+接口级同时配**（critical，rule-0-01600）：进程下 VIRTUALSYSFLAG 和接口下 VIRTUALSYSFLAG 同时配置，共网段才能生效（仅 020406 场景） — 后果：单配进程级共网段不生效
- **VPN 多实例须去环路检测**（critical，rule-0-01600）：VPN 多实例场景须配 VPNINSCAPSIMFLG=TRUE 去使能 VPN 路由环路检测（仅 020406 场景） — 后果：未去使能导致 OSPFv3 路由引入失败
- **PROCID 唯一**（warning，rule-0-01600）：同一 VRF 内 PROCID 必须唯一 — 后果：重复 PROCID 致 OSPFv3 配置冲突
- **共网段后 DRPRI 自动变 0**（warning，rule-0-01600）：共网段生效后 DRPRI 生效值自动变为 0；网络中需要有至少一个非共网段接口（仅 020406 场景） — 后果：违反约束导致 DR 选举异常
