# 增加用户关联识别协议（ADD USRRELATEIDEN）

> 命令静态知识见 [ADD USRRELATEIDEN](command/UDG/20.15.2/ADD-USRRELATEIDEN.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD USRRELATEIDEN 用于新增指定协议的用户关联识别配置，**对识别率诉求特别高的加密类应用通过该功能提升识别准确率**。**PROTOCOLNAME 与 USRRLTIDENSW 必选**；**CFGDOMAINNAME 可选**。核心配置空间有 3 个维度：**协议名**（PROTOCOLNAME）、**识别开关**（USRRLTIDENSW）、**配置域**（CFGDOMAINNAME）。1 个特性命中（GWFD-110203 HTTPS 业务地址识别）使用 `facebook` + `ENABLE` 范式，针对加密类应用增强识别。

### 配置维度 1：协议名（参数 PROTOCOLNAME，必选）

| 取值 | 作用 |
|---|---|
| **字符串，1~31 字符**（不支持空格） | 对应的协议名称；同一 PROTOCOLNAME 唯一标识一条协议配置；须是支持用户关联识别的协议 |

### 配置维度 2：用户关联识别开关（参数 USRRLTIDENSW，必选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 关闭用户关联识别功能（默认） |
| **ENABLE** | 开启用户关联识别功能（典型加密类应用 110203 范式） |

### 配置维度 3：配置域（参数 CFGDOMAINNAME，可选）

| 取值 | 作用 |
|---|---|
| **字符串，1~31 字符** | 命令所属公共配置域的名称 |

**典型脚本（GWFD-110203 HTTPS 业务地址识别，加密类协议 facebook 增强识别）**：
```
ADD USRRELATEIDEN:PROTOCOLNAME="facebook",USRRLTIDENSW=ENABLE;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：开用户关联识别"——配合 [0-00151](task/UDG/20.15.2/0-00151.md) ADD HOST（建 Host 域名条目）+ [0-00152](task/UDG/20.15.2/0-00152.md) LOD PARSERDB（加载协议解析库）使用，针对特定加密协议提升识别准确率。

## 决策点：USRRLTIDENSW 用户关联识别开关（DP 0-00153）

问：本协议是否启用用户关联识别增强？

| 选项 | USRRLTIDENSW | 适用场景 |
|---|---|---|
| **关闭** | DISABLE | 不需要用户关联识别增强；默认 |
| **开启**（110203） | ENABLE | 加密类应用需提升识别准确率（facebook 等典型） |

> 1 个特性命中（110203）使用 ENABLE；其余协议默认 DISABLE；按需开启，避免协议过多影响系统性能。

## 约束

- **PROTOCOLNAME+USRRLTIDENSW 联合必选**（critical，rule-0-00153）：PROTOCOLNAME 与 USRRLTIDENSW 是必选参数 — 后果：缺一命令拒收
- **PROTOCOLNAME 长度与字符**（warning，rule-0-00153）：PROTOCOLNAME 字符串长度 1~31，不支持空格 — 后果：超长/含空格命令拒收
- **USRRLTIDENSW 枚举**（warning，rule-0-00153）：USRRLTIDENSW 取值 DISABLE/ENABLE — 后果：其他取值命令拒收
- **单命令总记录数**（warning，rule-0-00153）：单命令总记录数最大 64 — 后果：超限 ADD 失败
- **执行后 60s 生效**（warning，rule-0-00153）：本命令执行后 60s 生效 — 后果：下发后立即跑业务流不生效，须等 60s 再验证
- **依赖协议解析库**（info，rule-0-00153）：用户关联识别依赖 [0-00152](task/UDG/20.15.2/0-00152.md) LOD PARSERDB 已加载的协议解析库 — 后果：未加载解析库时识别功能失效
