# 配置用户关联识别（ADD USRRELATEIDEN）

> 命令静态知识见 [ADD USRRELATEIDEN](command/UDG/20.15.2/ADD-USRRELATEIDEN.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

用户关联识别功能根据用户在特定时间段内的业务访问情况，关联识别出无明显特征的业务流，**提升加密类应用（如 facebook）的识别准确率**。本步骤在 HTTPS 业务地址识别特性中**标为可选**，仅对识别率诉求特别高的加密类应用才配。

**参数规划**

| 参数 | 含义 | 取值样例 | 说明 |
|---|---|---|---|
| PROTOCOLNAME | 协议名称 | `facebook` | 1~31 字符，不支持空格；须是支持用户关联识别的协议（LST SUPPUSRRLTIDEN 查支持列表） |
| USRRLTIDENSW | 用户关联识别开关 | `ENABLE`（fixed） | 固定开启；本 atom 单值 |

**典型脚本**

```
ADD USRRELATEIDEN:PROTOCOLNAME="facebook",USRRLTIDENSW=ENABLE;
```

**配置位置**（GWFD-110203 标准路径）
1. 加载特征库 / 解析库（LOD SIGNATUREDB / LOD PARSERDB）。
2. **可选：ADD USRRELATEIDEN**（本命令，仅高识别率诉求加密类应用）。
3. 配 HTTPS 报文 / DNS 报文业务策略组合。

**决策建议**：若无内容计费等特别高的计费诉求，**不建议开启**（协议过多影响系统性能）。

**冷门参数**：`CFGDOMAINNAME`（配置域名称，optional 不配）。

## 约束

- **rule-0-00351**（warning）单命令总记录数最大 **64**，超限 ADD 失败。
- **rule-0-00351**（warning）执行后 **60s 生效**，须等待 60s 再验证。
- **rule-0-00351**（warning）只能对支持该功能的协议配置；不支持时不报错但识别准确率不提升，用 `LST SUPPUSRRLTIDEN` 查支持列表。
- **rule-0-00351**（warning）协议过多影响系统性能（下降比例与话务模型相关）；按需开启软参 BIT387。
