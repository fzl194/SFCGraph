# 基于小区负荷上报的无线资源优化（GWFD-110332）

> 特性静态知识见 [GWFD-110332](feature/UDG/20.15.2/GWFD-110332.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

基于小区负荷上报的无线资源优化：RAN 侧监测小区负荷状态（Cell Load Level）变化，通过数据报文 GTP-U 扩展头实时上报给 PGW-U；PGW-U 经 N4 接口上报 PGW-C，PGW-C 触发 CELL_CONGESTION_CHANGE trigger 上报 PCRF，由 PCRF 下发策略控制指定用户/业务，解除/减缓小区拥塞。

**UDG 侧配置极简（仅 2 步）**：打开 License 开关 + 对指定 APN 开启拥塞上报（`SET APNREPORTATTR`，CONGESTIONRPT=ENABLE）。**UDG 侧不配任何过滤链/规则/PCC 策略组**——策略由 PCRF 动态下发（特性 wiki 明示"PCRF 下发 CELL_CONGESTION_CHANGE trigger + 下发策略"），UDG 仅作透传 + 上报。依赖 PCC 基本功能（GWFD-020351，UDG 从 PCRF 获取策略）+ RAN/PGW-C/PCRF 协同（非 UDG 配置）。

> **⚠ 证据冲突说明（critical，R1.5）**：部分任务分解曾将本特性关联 IPSQM（IPSQMSHAPING/IPSQMQDEPTH/IPSQMADJUST/IPSQMVIPLIST）整形命令。**本特性 activation（[激活基于小区负荷上报的无线资源优化](evidence/UDG/20.15.2/激活基于小区负荷上报的无线资源优化_30927860.md)）未演示任何 IPSQM 命令**——仅 `SET LICENSESWITCH` + `SET APNREPORTATTR` 两步。IPSQM 命令（atom [0-00231](task/UDG/20.15.2/0-00231.md)~[0-00234](task/UDG/20.15.2/0-00234.md)）归属于**另一特性 GWFD-110941（基于基站粒度的 IPSQM）**，证据为 `激活基于基站粒度的IPSQM_77772003.md`，与本特性机制不同（IPSQM=基于基站 IP 的静态整形；本特性=基于 APN 的拥塞上报+PCRF 动态策略）。**配置流程绝不编入 IPSQM 命令**（R1.5：无 activation 证据的命令编入=编造=critical 阻塞）。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开本特性 License 开关，单命令）：`SET LICENSESWITCH`（LICITEM=LKV3G5WOCR01, SWITCH=ENABLE）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **开启指定 APN 的小区负荷上报**（特性专属核心步，单命令）：`SET APNREPORTATTR`（APN=apn-test, CONGESTIONRPT=ENABLE）→ [0-00110](task/UDG/20.15.2/0-00110.md)
   - APN 取值：引用上游 [ADD APN](task/UDG/20.15.2/0-00020.md) 配置的 APN 名（本端规划）
   - CONGESTIONRPT=ENABLE：开启后 UDG 经 GTP-U 扩展头接收 RAN 上报的小区负荷，N4 上报 PGW-C
   - 其余上报字段（HEADERENRPT/PERFORMANCE/SERVICEREPORT）保持默认 SERVICE（本特性不涉及）

> **参数证据链**（R1.5，配置流程脚注）：以上 2 条命令均来自 activation §数据规划表 + §操作步骤 + §任务示例脚本（**全量覆盖，activation 仅 2 步**）。
> - SET LICENSESWITCH（LICITEM=LKV3G5WOCR01, SWITCH=ENABLE）：activation §任务示例脚本步骤1（原样）
> - SET APNREPORTATTR（APN=apn-test, CONGESTIONRPT=ENABLE）：activation §数据规划表（仅 2 行：APN + CONGESTIONRPT）+ §任务示例脚本步骤2（原样）
>
> **activation 步骤数**：activation §操作步骤仅 **2 步**（License + APN 上报属性），任务示例脚本仅 2 条命令——本特性 UDG 侧配置至此完成。RAN/PGW-C/PCRF 侧配置（CELL_CONGESTION_CHANGE trigger 下发、小区拥塞上报开关/周期、策略决策）非 UDG 配置范围（特性 wiki §配置 列出但属其他网元）。

## 决策点

**DP（特性级）：APN 选择 + 上报字段**（影响 SET APNREPORTATTR 参数）。维度按 activation §数据规划表 + [0-00110](task/UDG/20.15.2/0-00110.md) DP 0-00140 全列。

| 场景 | APN | CONGESTIONRPT | HEADERENRPT/PERFORMANCE/SERVICEREPORT | 走法 |
|---|---|---|---|---|
| **单 APN 开启小区负荷上报（activation 演示）** | apn-test（本端规划）| ENABLE | 默认 SERVICE | 骨架（步骤1~2）|
| 多 APN 分别开启 | 逐 APN 配置（每 APN 一条 SET APNREPORTATTR）| ENABLE | 默认 SERVICE | 同上（步骤2 按规划逐 APN 循环）|
| 上报字段取 REQUESTED（别名 APN 场景，见 [0-00110](task/UDG/20.15.2/0-00110.md) DP 0-00140）| 按规划 | ENABLE | REQUESTED | 同上 + 别名 APN 解析 |

**联动**：
- APN → 上游 [ADD APN](task/UDG/20.15.2/0-00020.md)（APN 必须先配置，atom 0-00110 约束"APN 必须先经 ADD APN 配置"）
- CONGESTIONRPT=ENABLE → 要求 RAN 侧和 PCRF 同时支持（atom 0-00110 约束，特性 wiki §涉及网元）

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5WOCR01`（控制项 82200DHW） — 未开则本特性不生效。
- **依赖 PCC 基本功能**（critical）：[GWFD-020351](feature/UDG/20.15.2/GWFD-020351.md)（控制项 82209825）—— 若需要 PCRF 进行策略决策，UDG 侧需开启 PCC 基本功能以从 PCRF 获取策略（特性 wiki 明示）。
- **解决方案特性·多网元协同**（critical）：要求无线侧（RAN V100R017C13+）+ PGW-C（UNC 20.7.0+）+ PCRF（V100R017C13+）+ PGW-U（UDG 20.8.2+）均支持本特性 — 单开 UDG 侧不生效。
- **仅 PGW-U 适用**（warning）：特性 wiki 明示适用 NF 仅 PGW-U（UPF 代际矩阵标 M 但 activation 仅演示 PGW-U 场景）。
- **仅 PCC 本地用户**（warning）：SGW-U 透传扩展 GTP-U 报文时，要求用户为 PCC 本地用户 — 不支持拜访用户和漫游用户。
- **报文单通风险**（warning）：无线侧累计发 100 次上行 GTP-U 报文无下行响应时不再携带扩展头，会导致 UDG 上报的小区负荷状态可能与实际不符。
- **MTU 规划**（warning）：GTP-U 扩展头（最长 12 字节）影响网络 MTU 规划，需整网规划。
- **GTP-U 处理消耗 CPU**（info）：UDG 处理 GTP-U 扩展头 + 实时向 PGW-C 上报小区负荷变化，增加 CPU 消耗与信令开销（特性 wiki §对系统的影响）。
- **SET APNREPORTATTR 单 APN 单条**（info）：每个 APN 支持一条配置（atom 0-00110 约束），多 APN 场景逐 APN 配置；整机最大记录数 10000。
