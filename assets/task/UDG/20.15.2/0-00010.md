# 配置规则（ADD RULE）

> 命令静态知识见 [ADD RULE](command/UDG/20.15.2/ADD-RULE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD RULE 是过滤链+计费三件套 backbone 的"规则装配点"：把 FLOWFILTER（匹配条件）与 POLICYNAME（动作策略）通过 PRIORITY（优先级）组装成一条命中文档的规则，供 ADD RULEBINDING 绑到 USERPROFILE。本命令的核心配置空间有 4 个维度：**策略类型 POLICYTYPE**（决定 POLICYNAME 引用对象）、**规则名称来源**（RULENAME 与对端一致性）、**流过滤器选择**（FLOWFILTER 或 FLOWFILTERGRP）、**优先级策略**（七层/三四层/any 兜底的相对优先级）。

### 配置维度 1：策略类型（参数 POLICYTYPE，必选）

POLICYTYPE 决定 POLICYNAME 引用何种动作对象——是本命令最关键的横向分叉：

| 取值 | POLICYNAME 引用对象 | 是否经 PCCPOLICYGRP | 典型特性 |
|---|---|---|---|
| **PCC**（计费/带宽/URL过滤/PCC触发） | PCCPOLICYGRPNM（PCC 策略组） | 是（PCCPOLICYGRP 绑 URRGROUP） | [2-00001](task/UDG/20.15.2/2-00001.md) 部署 UPF、[2-00002](task/UDG/20.15.2/2-00002.md) URL过滤、[2-00003](task/UDG/20.15.2/2-00003.md)~[2-00009](task/UDG/20.15.2/2-00009.md) 计费族、[2-00018](task/UDG/20.15.2/2-00018.md) PCC基本功能、[2-00028](task/UDG/20.15.2/2-00028.md) ADC（+ADCMUTEFLAG=DISABLE）、[2-00031](task/UDG/20.15.2/2-00031.md) FPI |
| **HEADEN**（头增强触发） | HEADERENNAME（HEADEN 对象） | 否（直接引用，PCC 策略组名称=NULL） | [2-00010](task/UDG/20.15.2/2-00010.md) HTTP头增强、[2-00011](task/UDG/20.15.2/2-00011.md) RTSP头增强、[2-00012](task/UDG/20.15.2/2-00012.md) HTTPS头增强、[2-00013](task/UDG/20.15.2/2-00013.md) HTTP头防欺诈 |
| **SMARTREDIRECT**（重定向触发） | 重定向动作对象名（IPFarm名 / DNS重写动作名 / HTTP智能重定向名） | 否（直接引用） | [2-00014](task/UDG/20.15.2/2-00014.md) 用户Portal（POLICYNAME=IPFARMNAME）、[2-00016](task/UDG/20.15.2/2-00016.md) DNS纠错（POLICYNAME=DNSOVERWRTNAME）、[2-00017](task/UDG/20.15.2/2-00017.md) HTTP智能重定向（POLICYNAME=SMTHTTPREDINAME）|
| **WEBPROXY**（Web代理触发） | IPFARMNAME（IP Farm，L3 IP NAT 重定向） | 否（直接引用） | [2-00015](task/UDG/20.15.2/2-00015.md) WebProxy |
| **REMARK_FPI**（DSCP/FPI 标记触发） | 无独立策略对象（REMARK 参数内嵌 RULE：REMARKFPISEL=REMARK/REMARKCFGTYPE/REMARKCLASS；FPI 变体 REMARKFPISEL=FPI+FPIVALUE） | 否（直接引用，不经 PCCPOLICYGRP） | [2-00022](task/UDG/20.15.2/2-00022.md) IM 无线资源管控（DSCP 标记，需 MOD USERPROFILE ALIASMARKFLAG=DISABLE）、[2-00031](task/UDG/20.15.2/2-00031.md) FPI 无线资源优化（REMARKFPISEL=FPI）|
| **ADC**（应用检测触发） | PCCPOLICYGRPNM（经 PCCPOLICYGRP） | 是（+ RULE/PCCPOLICYGRP 须 `ADCMUTEFLAG=DISABLE`） | [2-00028](task/UDG/20.15.2/2-00028.md) 增强的ADC（ADC 规则可与 PCC 计费规则共用 RULENAME+FLOWFILTERNAME）|

> **关键**：HEADEN/SMARTREDIRECT/WEBPROXY 均为独立于 PCC 的 POLICYTYPE，RULE 经 POLICYNAME 直接引用对应动作对象（HEADERENNAME/IPFARMNAME/SMTHTTPREDINAME/DNSOVERWRTNAME），**不经 PCCPOLICYGRP**（调测输出 PCC 策略组名称=NULL）。重定向动作特性（用户Portal/WebProxy）可与 PCC 计费规则**绑定同一 USERPROFILE + 共用同一 FLOWFILTER**（双 RULE 模式）。

### 配置维度 2：规则名称来源（参数 RULENAME，必选）

| 取值 | 来源 | 适用场景 |
|---|---|---|
| **本端规划**（default） | RULENAME 与计费策略组自洽 | 计费规则（绝大多数场景） |
| **与对端协商** | RULENAME 须与 SMF/PCF 一致 | 动态PCC（GWFD-020351）/本地PCC 由 SMF 下发激活 |

> RULENAME 与 POLICYTYPE 组合唯一——存在多条数据时，RULENAME+策略类型不能完全相同。

### 配置维度 3：流过滤器选择（参数 FILTERINGMODE，必选）

| 取值 | 含义 | 配套参数 |
|---|---|---|
| **FLOWFILTER**（默认） | 单流过滤器 | FLOWFILTERNAME 引 0-00007 产出的流过滤器 |
| **FLOWFILTERGRP** | 流过滤器组 | FLWFLTRGRPNAME 引流过滤器组（冷门，无 8 特性用到） |

### 配置维度 4：优先级策略（参数 PRIORITY，必选）

**值越小优先级越高**（1~4294967295 整数）。基于规则匹配粒度的相对约束：

| 规则类型 | 典型 PRIORITY | 说明 |
|---|---|---|
| 绑七层 filter 的规则 | 100 / 110 / 120 | 精确业务规则，优先匹配 |
| 绑三四层 filter 的规则 | < 七层规则值 | 三四层规则优先级应低于七层规则 |
| any 兜底规则（IPv4） | 65000 | 最低优先级，捕获未规划业务 |
| any 兜底规则（IPv6） | 65010 | 略高于 IPv4 any |
| PCC any-to-any | 65535 | PCC 场景兜底值 |

> 相对约束详见 selection_rule 0-00006。**铁律**：过滤条件为 any 的 RULE 优先级应设为最低；三四层 filter 的 RULE 优先级应低于七层 filter 的 RULE。

**典型范式（计费 4 场景，2-00001 部署 UPF）**：
```
ADD RULE: RULENAME="Rule001_dns", POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="ff_L7",     POLICYNAME="PPG_URL", PRIORITY=100;
ADD RULE: RULENAME="Rule002_IMS", POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="ff_IMS",    POLICYNAME="PPG_IMS", PRIORITY=110;
ADD RULE: RULENAME="Rule003_any", POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="ff_L34any", POLICYNAME="PPG_any", PRIORITY=65000;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 6：添加规则"——ADD FLOWFILTER + ADD PCCPOLICYGRP/HEADEN/IPFARM 等策略对象之后，ADD USERPROFILE + ADD RULEBINDING 之前。

## 决策点：规则名称来源（DP 0-00005）

问：RULE.RULENAME 的取值来源？

| 选项 | RULENAME 来源 | 适用场景 |
|---|---|---|
| 计费规则（LOCAL，默认） | 本端规划（与计费策略组自洽） | 计费规则（绝大多数场景） |
| PCC 规则（PEER） | 与 SMF/PCF 协商一致（动态PCC）/SMF 下发激活（本地PCC） | PCC 规则（GWFD-020351） |

## 决策点：规则策略类型（DP 0-00018）

问：ADD RULE 的 POLICYTYPE 取何值？

| 选项 | POLICYTYPE | POLICYNAME 引用对象 | 典型特性 |
|---|---|---|---|
| PCC 计费（PCC，默认） | PCC | ADD PCCPOLICYGRP 产出的 PCC 策略组 | 计费/PCC/OTT/ADC |
| SMARTREDIRECT 智能重定向 | SMARTREDIRECT | IPFarm 名 / DNS 重写动作名 / HTTP 智能重定向名 | 用户Portal / DNS纠错 / HTTP智能重定向 |
| WEBPROXY 代理重定向 | WEBPROXY | ADD IPFARM 产出的 IPFarm 名 | Web Proxy |

> 演进：HEADEN/REMARK_FPI/ADC 三类由 [1-00011](task/UDG/20.15.2/1-00011.md) backbone 描述，本 atom POLICYTYPE binding 暂保持 fixed:PCC（按"演进只记录不执行"边界，详见 [1-00011](task/UDG/20.15.2/1-00011.md) 场景差异 POLICYTYPE 变体表）。

## 约束

- **RULE 配置规格与优先级约束**（warning，rule-0-00131）：RULESPECTYPE=DEFAULT（或未指定）最大记录数 **8000**；SPECIFICATION_LIMITED 最大 **100000**；两类合计规格 108000；超 90% 上报 ALM-135602215；阈值可 MOD CFGTHRESHOLD 修改；唯一性 RULENAME+POLICYTYPE 组合；PRIORITY 1~4294967295；**立即生效** — 后果：超规格新增失败 / any 规则优先级过高误匹配业务 / 三四层七层优先级倒置
- **POLICYTYPE 决定业务动作**（critical，隐含）：POLICYTYPE=PCC→计费/带宽/URL过滤；HEADEN→头增强；SMARTREDIRECT→重定向；选错策略类型则对应业务不触发 — 后果：业务不触发
- **POLICYNAME 引用一致**（critical，隐含）：POLICYNAME 须与策略对象一致（PCC→PCCPOLICYGRPNM；HEADEN→HEADERENNAME；SMARTREDIRECT→重定向对象名）；FLOWFILTERNAME 须与 FLOWFILTER 一致 — 后果：规则链断开
- **RULENAME 跨网元一致**（critical，隐含）：RULENAME 在 SMF/UPF/PCF 间必须一致（计费本端自洽，PCC 与对端协商） — 后果：跨网元规则不一致
