# 配置规则（ADD RULE）

> 命令静态知识见 [ADD RULE](command/UDG/20.15.2/ADD-RULE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD RULE 是业务策略规则的装配点：把 FLOWFILTER（匹配条件）与 POLICYNAME（动作策略）通过 PRIORITY（优先级）组装成一条命中文档转发/计费动作的规则。证据③中 8 个特性的核心配置范式：

| 参数 | 取值约定 | 来源 |
|---|---|---|
| RULENAME | 业务语义命名（计费本端规划 / PCC 与 SMF 协商）；与 POLICYTYPE 组合唯一 | decision_driven（DP 0-00005） |
| POLICYTYPE | 计费/PCC/OTT=`PCC`；智能重定向=`SMARTREDIRECT`；代理重定向=`WEBPROXY` | decision_driven（DP 0-00018） |
| FILTERINGMODE | 固定 `FLOWFILTER`（证据③全部用流过滤器，非流过滤器组） | fixed |
| FLOWFILTERNAME | 引 0-00007 产出的流过滤器 | reference |
| POLICYNAME | 引 0-00003 产出的 PCC 策略组（POLICYTYPE=PCC 时）；或重定向/IPFarm 对象 | reference |
| PRIORITY | 见下表优先级约定 | 本端规划 |

**典型范式（计费 4 场景，2-00001 等部署 UPF）**：
```
ADD RULE: RULENAME="Rule001_dns", POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="ff_L7",     POLICYNAME="PPG_URL", PRIORITY=100;
ADD RULE: RULENAME="Rule002_IMS", POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="ff_IMS",    POLICYNAME="PPG_IMS", PRIORITY=110;
ADD RULE: RULENAME="Rule003_any", POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="ff_L34any", POLICYNAME="PPG_any", PRIORITY=65000;
```

**PRIORITY 取值约定**（值越小优先级越高，证据③）：

| 规则类型 | 典型 PRIORITY | 说明 |
|---|---|---|
| 绑七层 filter 的规则 | 100 / 110 / 120 | 精确业务规则，优先匹配 |
| 绑三四层 filter 的规则 | < 七层规则值 | 三四层规则优先级应低于七层规则 |
| any 兜底规则（IPv4） | 65000 | 最低优先级，捕获未规划业务 |
| any 兜底规则（IPv6） | 65010 | 略高于 IPv4 any |
| PCC any-to-any | 65535 | PCC 场景兜底值 |

## 决策点

**DP 0-00005**（single_choice, required, always）规则名称来源？

| 选项 | RULENAME 来源 | 适用 |
|---|---|---|
| opt-rule-local（默认） | 本端规划（与计费策略组自洽） | 计费规则 |
| opt-rule-peer | 与 SMF/PCF 协商一致（动态 PCC）/SMF 下发激活（本地 PCC） | PCC 规则（GWFD-020351） |

**DP 0-00018**（single_choice, required, always）规则策略类型？决定 POLICYNAME 引哪类对象。

| 选项 | POLICYTYPE | POLICYNAME 引用对象 | 典型特性 |
|---|---|---|---|
| opt-policytype-pcc（默认） | PCC | ADD PCCPOLICYGRP 产出的 PCC 策略组（PPG_xxx / cg_xxx / pg_test） | 计费/PCC/OTT |
| opt-policytype-smartredirect | SMARTREDIRECT | IPFarm 名 / DNS 重写动作名 / HTTP 智能重定向名 | 用户Portal/DNS纠错/HTTP智能重定向 |
| opt-policytype-webproxy | WEBPROXY | ADD IPFARM 产出的 IPFarm 名（代理 server 池） | Web Proxy |

## 约束

- **rule-0-00131**（warning）规格：RULESPECTYPE=DEFAULT（或未指定）最大记录数 **8000**；SPECIFICATION_LIMITED 最大 **100000**；两类合计规格 108000，超 90% 上报 ALM-135602215（阈值 `MOD CFGTHRESHOLD` 修）。唯一性：RULENAME + POLICYTYPE 组合多条数据不能完全相同。PRIORITY 取值 1~4294967295。**立即生效**。
