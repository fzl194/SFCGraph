# WebProxy（GWFD-110282）

> 特性静态知识见 [GWFD-110282](feature/UDG/20.15.2/GWFD-110282.md)。本页讲配置生成实例化时怎么配。本特性与用户Portal(2-00014)同属重定向族，共享 IP Farm 基础设施 compound [1-00014](task/UDG/20.15.2/1-00014.md)；差异：POLICYTYPE=WEBPROXY（L3 IP NAT重定向，非L7 URL重写）、仅L34过滤（无L7）、无captive模式。

## 配置概览

WebProxy 将用户业务流重定向到 Proxy Server。**核心机制：RULE 经 `POLICYTYPE=WEBPROXY` 直接引用 IP Farm（ADD IPFARM 定义），独立于 PCC/SMARTREDIRECT 的 POLICYTYPE，不经 PCCPOLICYGRP**。IP Farm 是 Proxy Server 的负荷分担集合 + ICMP 心跳检测基础设施（LBMETHOD 选 Proxy Server）。WebProxy 的重定向是 **L3 IP NAT 级别**（基于目的 IP 做代理转发），区别于用户Portal 的 SMARTREDIRECT **L7 URL 级别**重定向。

WebProxy 仅基于 **L34 匹配**（FILTER 的 SVRIP 服务器IP匹配），不需 L7 过滤器/特征库/LOD SIGNATUREDB。activation 演示双 RULE（rule_test1=WEBPROXY 重定向到 IPFarm farm_test + rule_test2=PCC 计费），两条规则共用同一 FLOWFILTER `flow-l34`，即重定向与计费可绑定同一 USERPROFILE。activation 虽在 ADD USERPROFILE 配 CAPMODETHRES=6，但 WebProxy 无 captive/non-captive 交替语义（该值继承自 UP 模板默认，无业务效果）。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开 WebProxy License 开关，单命令）：`SET LICENSESWITCH`（LKV3G5WEBP01）→ [0-00019](task/UDG/20.15.2/0-00019.md)

5. **计费属性**（activation 演示，rule_test2=PCC 用，可选）：`ADD URR`（cbb_test/URRID=1000/ONLINE）+ `ADD URRGROUP`（cp_test）+ `ADD PCCPOLICYGRP`（pcc-pg_test）→ [1-00010](task/UDG/20.15.2/1-00010.md)

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5WEBP01` — 未开则 WebProxy 不生效。
- **强依赖 SA-Basic**（critical）：GWFD-110101（License LKV3G5SABS01）—— 报文解析基础，未开则无法识别报文。
- **POLICYTYPE=WEBPROXY 触发条件**（critical）：RULE.POLICYTYPE=WEBPROXY + POLICYNAME=IPFARMNAME（直接引用 IP Farm，PCC 策略组名称=NULL）— 指错则重定向不触发。
- **IP 版本全链路一致**（critical）：IPFARM/SERVER/LOGICINF/Filter/接口须同 v4 或同 v6 — `DSP RULECHECK` 检查一致性；不一致则重定向失效。
- **仅 L34 匹配，无 L7**（warning）：WebProxy 仅基于 FILTER 的 SVRIP 服务器IP匹配，不需 L7FILTER/LOD SIGNATUREDB — 若配 L7 过滤条件则 WebProxy 语义偏离。
- **若流过滤器绑定 DNS 外七层协议不能用 WebProxy 策略**（warning）：WebProxy 仅适配 L34 TCP 业务流，绑定 DNS 等非 TCP/七层协议的流过滤器不能触发 WebProxy — 配错则规则不命中。
- **至少一 Server 状态 up**（warning）：`LST IPFARMSERVER` 查询须保证至少一个 server 状态 up — 否则重定向无可用目标。
- **不同 IP Farm 需不同心跳接口**（warning）：心跳检测接口不共享 — 复用则心跳冲突。
- **WebProxy 重定向为 L3 IP NAT 级别**（info）：区别于用户Portal(2-00014) 的 SMARTREDIRECT L7 URL 级别重定向；WebProxy 基于目的 IP 做 IP NAT 代理转发，不构造 HTTP 301/302/303 重定向响应。
- **CAPMODETHRES 无 captive 语义**（info）：activation 虽在 ADD USERPROFILE 配 CAPMODETHRES=6，但 WebProxy 无 captive/non-captive 交替模式，该值继承 UP 模板默认，无业务效果（对比 Portal 的 captive 交替需该参数）。
- **REFRESHTYPE=ALL**（info）：WebProxy 改动到 Filter（含 SVRIP），故 REFRESHSRV 用 ALL（全集刷新），区别于 Portal 的 USERPROFILE。
- **规格**（info）：整机 ≤64 IP Farm；每个 IP Farm ≤512 Server。
