# 流分类绑定ACL规则（ADD QOSRULEACL）

> 命令静态知识见 [ADD QOSRULEACL](command/UDG/20.15.2/ADD-QOSRULEACL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

把 ACL 规则组接到 [ADD MQCCLASSIFIER](0-00201.md) 建好的流分类上，让流分类具备报文匹配能力。CLASSIFIERNAME 引上游分类，ACLNAME 引上游 ACL 规则组（ADD ACLGROUP），ACLTYPE 本特性主场景固定按名称。

**数据规划取值样例**（证据③，IPFD-012001 数据规划表）：

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| CLASSIFIERNAME | `class1` | 引用（0-00201） | 引上游流分类 |
| IPVERSION | `IPv4` | 本端规划 | IPv4 引 IPv4 ACL，IPv6 引 IPv6 ACL |
| ACLTYPE | `acl-name` | 本端规划（主场景固定） | 另可选 `acl-number`（按编号） |
| ACLNAME | `acl1` | 引用（ADD ACLGROUP） | 引上游 ACL 规则组名 |

**典型范式**（证据③任务示例脚本）：
```
ADD QOSRULEACL: CLASSIFIERNAME="class1", IPVERSION=IPv4, ACLTYPE=acl-name, ACLNAME="acl1";
```

> 缺口标注：ACLTYPE=acl-number 分支需配 ACLNUM（编号 2000~4999）替代 ACLNAME，主场景未用，待编号绑定场景揭示后补绑。

## 约束

- **rule-0-00289**（warning）最大记录数 **65536**；ACLTYPE 与 ACLNAME/ACLNUM 配套（acl-name→ACLNAME 必选，acl-number→ACLNUM 必选）；**ACL 计数不参与**（QoS 流分类匹配不影响 ACL 统计计数）；IPVERSION 配套（IPv4→IPv4 ACL，IPv6→IPv6 ACL）；执行后立即生效。
