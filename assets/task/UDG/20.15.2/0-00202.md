# 流分类绑定ACL规则（ADD QOSRULEACL）

> 命令静态知识见 [ADD QOSRULEACL](command/UDG/20.15.2/ADD-QOSRULEACL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。把 ACL 规则组接入 [0-00201](task/UDG/20.15.2/0-00201.md) ADD MQCCLASSIFIER 已建的流分类；CLASSIFIERNAME 引上游流分类、ACLNAME/ACLNUM 引 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP。

## 配置方法（配置方法字典）

ADD QOSRULEACL 是"流分类下挂载 ACL 规则组"命令。5 参必选（CLASSIFIERNAME/IPVERSION/ACLTYPE/ACLNUM 或 ACLNAME）。配置空间有 2 个维度：**IP 版本**（IPVERSION 决定 IPv4/IPv6 ACL 配套）、**ACL 引用方式**（ACLTYPE 决定按 acl-name/acl-number）。1 个特性命中（IPFD-012001 QoS 复杂流分类）覆盖按名称引用 IPv4 ACL 范式。

### 配置维度 1：IP 版本（参数 IPVERSION，必选）

| IPVERSION | 联动 ACL | 适用场景 |
|---|---|---|
| **IPv4** | ACLNUM 2000~4999 / ACLNAME 命名表 | IPFD-012001 范例（IPv4 流重定向） |
| **IPv6** | IPv6 ACL 编号 / IPv6 ACL 命名 | IPv6 流匹配（本 atom 未涉及，按需取） |

> **关键约束**：IPv4 引用 IPv4 ACL；IPv6 引用 IPv6 ACL——两者不可混用。

### 配置维度 2：ACL 引用方式（参数 ACLTYPE 必选 + ACLNAME/ACLTYPE 条件必选）

| ACLTYPE | 联动必选 | 取值 | 适用场景 |
|---|---|---|---|
| **acl-name** | ACLNAME | 字符串（命名 ACL） | IPFD-012001 范例（命名 ACL `acl1`） |
| **acl-number** | ACLNUM | 数字 2000~4999 | 按编号引用高级/基本/以太 ACL |

> **ACLTYPE 与 ACLNAME/ACLNUM 互斥必选**：选 acl-name 则 ACLNAME 必填、ACLNUM 不填；选 acl-number 则 ACLNUM 必填、ACLNAME 不填。

**典型脚本（IPFD-012001 QoS 复杂流分类，按名称绑 IPv4 ACL）**：
```
ADD QOSRULEACL: CLASSIFIERNAME="class1", IPVERSION=IPv4, ACLTYPE=acl-name, ACLNAME="acl1";
```

> 命令 notes：**最大记录数 65536**；**执行后立即生效**；ACL 计数不参与（QoS 流分类匹配不影响 ACL 统计计数）；IPVERSION 配套约束（IPv4→IPv4 ACL，IPv6→IPv6 ACL）。

**步骤位置**：在所属 feature 编排中位于"步骤 N：Classifier 挂 ACL"——前置 [0-00201](task/UDG/20.15.2/0-00201.md) ADD MQCCLASSIFIER（建 Classifier）+ [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP（建 ACL 规则组）+ [0-00198](task/UDG/20.15.2/0-00198.md) ADD ACLRULEADV4（建 ACL 规则内容，按需）；下游 [0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY（CB 节点绑定 CLASSIFIERNAME）。

## 决策点：ACL 引用方式选择（DP 0-00202）

问：ADD QOSRULEACL 用 ACLNAME 还是 ACLNUM 引用 ACL 规则组？

| 选项 | ACLTYPE | 配套 | 适用场景 |
|---|---|---|---|
| **按名称引用**（IPFD-012001 范例） | acl-name | ACLNAME="..." | IPFD-012001 范式（命名 ACL） |
| **按编号引用** | acl-number | ACLNUM=2000~4999 | 按编号引用高级 ACL（3000+）/基本 ACL（2000+） |

> 1 个特性命中（IPFD-012001）按名称引用；按编号待编号 ACL 场景揭示后补。

## 约束

- **最大记录数 65536**（critical，rule-0-00289）：整机 QoS-ACL 关联最大 65536 条 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00289）：执行后立即生效 — 后果：误操作即刻生效
- **IPVERSION 与 ACL 类型配套**（critical，rule-0-00202）：IPv4 必须引 IPv4 ACL；IPv6 必须引 IPv6 ACL——不配套命令拒收 — 后果：跨版本引用规则不生效
- **ACLTYPE 与 ACLNAME/ACLNUM 配套**（critical，rule-0-00202）：acl-name 配 ACLNAME（不配 ACLNUM）；acl-number 配 ACLNUM（不配 ACLNAME）——混配命令拒收 — 后果：配套不一致命令拒收
- **CLASSIFIERNAME 必须先经 ADD MQCCLASSIFIER 建**（critical，rule-0-00202）：CLASSIFIERNAME 引用的 Classifier 必须先由 [0-00201](task/UDG/20.15.2/0-00201.md) ADD MQCCLASSIFIER 定义 — 后果：未建引用失败
- **ACLNAME/ACLNUM 必须先经 ADD ACLGROUP 建**（critical，rule-0-00202）：ACL 引用必须先由 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP 定义 — 后果：未建引用失败
- **ACLNUM 范围**（info，rule-0-00202）：ACLNUM 整数 2000~4999（基本/高级/以太） — 后果：跨范围命令拒收
- **ACL 计数不参与**（info，rule-0-00289）：QoS 流分类匹配不影响 ACL 统计计数（统计独立） — 后果：误以为 ACL 计数下降
- **CLASSIFIERNAME 长度**（info，rule-0-00202）：字符串 1~31 — 后果：超长命令拒收
