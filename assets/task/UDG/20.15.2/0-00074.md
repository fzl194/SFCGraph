# 创建OSPF进程(IPv4)（ADD OSPF）

> 命令静态知识见 [ADD OSPF](command/UDG/20.15.2/ADD-OSPF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD OSPF 创建 IPv4 OSPF 进程。三个必选参数中：`PROCID` 进程号由全网规划；`VRFNAME` 引用上游 ADD L3VPNINST 创建的 VPN 实例（binding=reference→0-00071）；`SCHEMAROUID` Router ID 全网唯一。OSPF 协议进阶参数（BFD/LSA/SPF/Stub/MPLS 等 44 项）在 7 个特性数据规划表+脚本中均未用到，按通用配置筛一律不绑。

**取值范式（从证据数据规划表归纳）**

| 特性 / 场景 | PROCID | VRFNAME | SCHEMAROUID | 脚本样例 |
|---|---|---|---|---|
| GWFD-010155 PGW-U 到 ePDG（VPN+OSPF 动态路由组网） | 6（全网规划） | vpn-s2b（引用 0-00071） | 10.8.20.1（全网规划，Router ID 全网唯一） | `ADD OSPF:PROCID=6,VRFNAME="vpn-s2b",SCHEMAROUID="10.8.20.1";` |
| GWFD-010104/010105/020406/020421 地址分配类场景（IPv4 下行路由） | 100（本端/已配置） | vpn1（引用 0-00071） | 10.10.10.1（全网规划） | `ADD OSPF:PROCID=100,VRFNAME="vpn1",SCHEMAROUID="10.10.10.1";` |
| IPFD-015004 OSPF over IPsec | 20214（本端规划） | vrf1 | Device A: 10.102.105.238 / Device B: 10.102.105.224 | `ADD OSPF:PROCID=20214,VRFNAME="vrf1",SCHEMAROUID="10.102.105.238";` |

> PROCID variable_source 跨特性分叉（010104 本端规划 / 010155 全网规划），但取值解析同构无差异，保 global_planned。

**关键规则**
- **VRFNAME 必须引用已创建的 VPN 实例**：binding=reference→[创建L3VPN实例](task/UDG/20.15.2/0-00071.md)，引用一致由 binding 保证。
- **SCHEMAROUID 全网唯一**：优选用户配置值，未配则选 VRF/公网内最大接口 IP（md 说明）。
- **VPN 多实例须取消环路检查**：md 说明强调 UDG 支持 VPN 多实例时须配 `VPNINSCAPSIMFLG=TRUE` 去使能 VPN 路由环路检测，否则 OSPF 路由引入失败。该参数仅在 md"说明"非数据规划表行，两特性脚本走默认 FALSE，记 rule 0-00190 投影条件要求而非绑参。
- **BFD 会话使能**：若需为 OSPF 动态路由绑定 BFD 会话，经 MOD OSPF `BFDALLINTFFLG=TRUE` 使能（脚本体现），视为修饰命令不单建 atom。
- **共网段**：当 OSPF 路由配置共网段时，需 MOD OSPFAREA `VIRTUALSYSFLAG=TRUE`（修饰命令，不单建 atom）。

## 约束

- **rule 0-00190**（constraint_rule/execution_constraint）ADD OSPF 须满足：(1) OSPF 进程实例数规格默认整机 2000、命令最大记录数 8016；(2) 设备的一个接口只能属于某一个 OSPF 进程，进程实例不可更改（仅第一次使能时指定 VPN/全局归属）；(3) SCHEMAROUID（Router ID）必须全网唯一；(4) VPN 多实例场景须配 `VPNINSCAPSIMFLG=TRUE` 去使能 VPN 路由环路检测，否则 OSPF 路由引入失败；(5) OSPF 共网段须 MOD OSPFAREA `VIRTUALSYSFLAG=TRUE`；(6) 建议配置协议认证（ADD OSPFAREAUTH）增强协议安全性 — 后果：路由引入失败 / Router ID 冲突 / 进程实例超规格 / 接口归属冲突（critical）。
