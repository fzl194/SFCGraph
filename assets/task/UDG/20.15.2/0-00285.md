# 配置切片与N3逻辑接口绑定（ADD SNSSAIUPINTF）

> 命令静态知识见 [ADD SNSSAIUPINTF](command/UDG/20.15.2/ADD-SNSSAIUPINTF.md)

## 配置方法

把 ADD SNSSAI 建好的切片（SST+SD）绑到一个已通过 ADD LOGICINF 配置的 N3 逻辑接口上。SST 必选、SD 可选（无 SD 默认全 F）、N3LOGICINF 必选。绑定后该切片的 PFCP 会话按此接口转发。

| 参数 | 取值样例 | 说明 |
|---|---|---|
| SST | `1` | 必选，与上游 ADD SNSSAI 一致 |
| SD | `123456` | 可选，与上游一致；不传时系统按全 F 处理 |
| N3LOGICINF | `n3if1/1/1` | 必选，1~63 字符；逻辑接口名须已由 ADD LOGICINF 建好 |

**N3LOGICINF 命名规则**（命令手册约束）：类型（n3if/saif）+ ISU 组号（1）+ ISU 实例类型（1=Instance 级）+ 逻辑接口号（0~31），形如 `n3if1/1/0`。默认接口 `n3if1/1/0` / `Saif1/1/0` 不可绑切片（留给无切片会话兜底）。

**典型脚本**：

```
// 把 SST=1、SD=123456 的切片绑到逻辑接口 n3if1/1/1
ADD SNSSAIUPINTF: SST=1,SD="123456",N3LOGICINF="n3if1/1/1";

// SD 为空的单字段切片绑接口（实际设置 SD=ffffff）
ADD SNSSAIUPINTF: SST=1,N3LOGICINF="n3if1/1/1";
```

**步骤位置**：切片带宽控制链路的第 2 步——在 ADD SNSSAI（0-00284）之后、ADD SNSSAIBWMUSRG（0-00286）之前；先定切片的转发接口，再把切片绑进带宽用户组。

## 决策点

**DP 0-00285**：切片绑定的目标逻辑接口选择——N3LOGICINF 必须指向一个已建好的非默认 N3 逻辑接口。

| 选项 | N3LOGICINF 取值 | 说明 |
|---|---|---|
| 绑定普通接口 | `n3if1/1/1` 等非默认接口 | 推荐切片专用，按接口规划选 |
| 不绑（用默认） | `n3if1/1/0` / `Saif1/1/0` | **禁止**：默认接口不可绑切片，留给无切片会话兜底 |

本命令为**高危命令**：切换切片绑定的逻辑接口会导致已激活切片用户的 PFCP 路径变化，可能造成业务链路不通，执行前需评估影响。

## 约束

- 最大记录数 8192 · warning · 超规格新增失败
- 命令执行后对新的 PFCP 消息生效 · warning · 不影响已建立会话
- 增加配置会导致切片用户所选逻辑接口变化，可能业务链路不通 · error · 高危
- 默认接口 `n3if1/1/0` / `Saif1/1/0` 不能绑切片 · error · 留给无切片会话兜底
- 删除逻辑接口时，切片与该接口的绑定关系自动解绑 · info · 联动删除
- 数据面接口模式为 INSTANCE 时不支持切片与逻辑接口绑定 · error · 模式限制
- MBS 会话激活时 n3mbif 接口不能绑切片 · error · 场景限制
- 允许配置融合接口 Saif · info · 融合组网支持
