# 配置切片与N3逻辑接口绑定（ADD SNSSAIUPINTF）

> 命令静态知识见 [ADD SNSSAIUPINTF](command/UDG/20.15.2/ADD-SNSSAIUPINTF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

把 ADD SNSSAI（0-00284）建好的切片（SST+SD）绑到一个已通过 ADD LOGICINF 配置的 N3 逻辑接口上。SST 必选、SD 可选（无 SD 默认全 F）、N3LOGICINF 必选。绑定后该切片的 PFCP 会话按此接口转发。**本命令为高危命令**：切换切片绑定的逻辑接口会导致已激活切片用户的 PFCP 路径变化，可能造成业务链路不通。适用 NF：UPF。

### 配置维度 1：切片与接口绑定（参数 `SST` + `SD` + `N3LOGICINF`）

| 参数 | 取值 | 作用 | 配套必选/条件 |
|---|---|---|---|
| SST | 整数 0～255 | 切片/服务类型，必须与上游 [0-00284](task/UDG/20.15.2/0-00284.md) ADD SNSSAI 已配 SST 一致 | 必选 |
| SD | 6 位十六进制 / `ffffff` | 切片区分码；不传时系统按全 F 处理 | 可选；与上游一致 |
| N3LOGICINF | 字符串 1～63 | N3 逻辑接口名称；必须已由 ADD LOGICINF 创建 | 必选 |

### 配置维度 2：N3 逻辑接口命名规则

`n3if{ISU组号}/{ISU实例类型}/{逻辑接口号}`，默认接口 `n3if1/1/0` / `Saif1/1/0` **不可绑切片**（留给无切片会话兜底）。

**典型脚本**

```
// 把 SST=1、SD=123456 的切片绑到逻辑接口 n3if1/1/1
ADD SNSSAIUPINTF: SST=1,SD="123456",N3LOGICINF="n3if1/1/1";

// SD 为空的单字段切片绑接口（系统按 SD=ffffff 处理）
ADD SNSSAIUPINTF: SST=1,N3LOGICINF="n3if1/1/1";
```

**步骤位置**：切片带宽控制链路的第 2 步——在 ADD SNSSAI（0-00284）之后、ADD SNSSAIBWMUSRG（0-00286）之前；先定切片的转发接口，再把切片绑进带宽用户组。

## 决策点：切片绑定的目标逻辑接口（DP 0-00285）

**[[UDG@20.15.2@DecisionPoint@0-00285]]**（single_choice，N3LOGICINF 是否默认接口）

| 选项 | N3LOGICINF 取值 | 说明 |
|---|---|---|
| 绑定普通接口（推荐） | `n3if1/1/1` 等非默认接口 | 按接口规划选，切片专用 |
| 不绑（用默认） | `n3if1/1/0` / `Saif1/1/0` | **禁止**：默认接口不可绑切片，留给无切片会话兜底 |

## 约束

- **命令记录上限**（warning，rule-0-00285-01）：最大记录数 8192，超规格新增失败。
- **新 PFCP 生效**（warning，rule-0-00285-02）：命令执行后对新的 PFCP 消息生效，不影响已建立会话。
- **高危操作**（critical，rule-0-00285-03）：增加配置会导致切片用户所选逻辑接口变化，可能业务链路不通——执行前需评估影响。
- **默认接口不可绑切片**（critical，rule-0-00285-04）：默认接口 `n3if1/1/0` / `Saif1/1/0` 不能绑切片（留给无切片会话兜底）。
- **删除联动**（info，rule-0-00285-05）：删除逻辑接口时，切片与该接口的绑定关系自动解绑。
- **数据面接口模式限制**（critical，rule-0-00285-06）：数据面接口模式为 INSTANCE 时不支持切片与逻辑接口绑定。
- **MBS 会话限制**（critical，rule-0-00285-07）：MBS 会话激活时 n3mbif 接口不能绑切片。
- **融合组网支持**（info，rule-0-00285-08）：允许配置融合接口 Saif。
