# 基于业务累计流量的策略控制（GWFD-110312）

> 特性静态知识见 [GWFD-110312](feature/UDG/20.15.2/GWFD-110312.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

本特性是 **FUP 业务级**（Fair Usage Policy，区分业务类型的累计流量策略控制）。针对**特定业务**（activation 演示 TCP/80 HTTP 流量）做累计流量监控：采用 3GPP 标准 Create URR 机制，对满足过滤条件的业务实现在线流量计费 + 累计流量上报。达阈值时由 PCRF/PCF 下发降速/改费率等策略，达到网络公平使用原则。

**核心机制：双轨 URR 同组**——URRGROUP 同时绑定两条 URR：① `urr_online`（`USAGERPTMODE=ONLINE`，在线流量计费）② `urr_mk`（`USAGERPTMODE=MONITORINGKEY`，累计流量上报）。PCCPOLICYGRP 绑该 URRGROUP（activation 演示绑 2 个 URRGROUP：`cp_red` + `urrgroup1`），RULE 以 POLICYTYPE=PCC 触发。

**与 [2-00026](task/UDG/20.15.2/2-00026.md)（GWFD-020353 会话级 FUP）的区别**：会话级 FUP 以整个 PDU 会话为粒度（不区分业务），License 门控 + 复用 PCC backbone，UDG 侧无独立配置；本特性（业务级）区分业务类型，需配业务过滤链 + 双轨 URR + PCC 策略组，activation 有完整配置脚本。

**复用通用 backbone**（计费族共享）：[1-00009](task/UDG/20.15.2/1-00009.md)（过滤链）+ [1-00010](task/UDG/20.15.2/1-00010.md)（计费三件套·双轨 URR）+ [1-00011](task/UDG/20.15.2/1-00011.md)（规则绑定）。差异在 URR 双轨组装方式（ONLINE + MONITORINGKEY）。

## 配置流程（步骤编排，每步显示含命令）

1. **License 前置**（单命令）：`SET LICENSESWITCH`（`LKV3G5FPBS01`）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **过滤链**（业务匹配，三四层）：`ADD FILTER`（TCP/80）+ `ADD FLOWFILTER` + `ADD FLTBINDFLOWF` → [1-00009](task/UDG/20.15.2/1-00009.md)
   - 刷新生效（三四层 Filter 变更）：`SET REFRESHSRV`（`REFRESHTYPE=USERPROFILE`）→ [0-00015](task/UDG/20.15.2/0-00015.md)
3. **计费三件套·双轨 URR**（费率定义，双 URR 同组）：`ADD URR`（urr_online：ONLINE+VOLUME）+ `ADD URR`（urr_mk：MONITORINGKEY）+ `ADD URRGROUP`（双轨：UP/DOWNURRNAME1=urr_online + UP/DOWNURRNAME2=urr_mk）+ `ADD PCCPOLICYGRP`（绑 URRGROUP，activation 演示绑 2 个）→ [1-00010](task/UDG/20.15.2/1-00010.md)
4. **规则与用户模板绑定**：`ADD RULE`（POLICYTYPE=PCC + FILTERINGMODE=FLOWFILTER + FLOWFILTERNAME + POLICYNAME=PCCPOLICYGRPNM）+ `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)

## 决策点

**DP（特性级）：业务匹配过滤 + 双轨 URR 统计维度**（影响过滤链 FILTER 参数 + URR 的 ONLMETERINGTYPE）。activation 演示单场景（TCP/80 + ONLINE VOLUME + MONITORINGKEY），其他业务/统计维度为变体。

| 场景 | 业务过滤（[1-00009](task/UDG/20.15.2/1-00009.md) FILTER） | 在线 URR 统计（ONLMETERINGTYPE） | 累计 URR（USAGERPTMODE） | 走法 |
|---|---|---|---|---|
| **HTTP 业务累计流量（activation 演示）** | TCP/80（L34PROTOCOL=TCP + SVRSTARTPORT/SVRENDPORT=80）| VOLUME（字节数）| MONITORINGKEY | 骨架 + [1-00009](task/UDG/20.15.2/1-00009.md) + [1-00010](task/UDG/20.15.2/1-00010.md) 双轨 + [1-00011](task/UDG/20.15.2/1-00011.md) |
| 时长统计变体 | TCP/80（同上）| DURATION（持续时间）| MONITORINGKEY | 同上，ONLMETERINGTYPE=DURATION（取值来自 [1-00010](task/UDG/20.15.2/1-00010.md) 场景差异，activation 未演示）|
| 其他业务变体 | 七层 L7FILTER+PROTBINDFLOWF（URL/协议匹配）| VOLUME | MONITORINGKEY | [1-00009](task/UDG/20.15.2/1-00009.md) 加 L7 段 |

**联动**：
- 业务过滤 → [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链：activation 演示三四层 TCP/80；七层匹配待补业务感知配置。
- 双轨 URR → [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套：URRGROUP 双轨组装（UP/DOWNURRNAME1=online + UP/DOWNURRNAME2=mk），区别于单轨在线/离线/融合（融合是 offline+online，本特性是 online+monitoringkey）。
- 阈值策略 → PCRF/PCF 侧（UDG 不配）：累计流量达阈值后的降速/改费率动作由 PCRF/PCF 下发，UDG 仅承载双轨上报。

## 约束（特性级 rule）

- **License 前置**（critical）：`SET LICENSESWITCH` 开启 `LKV3G5FPBS01` — 未开则本特性不生效。
- **依赖特性**（critical）：[SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，业务识别）+ [PCC 基本功能](feature/UDG/20.15.2/GWFD-020351.md)（GWFD-020351，PCC 触发链）+ [基于业务感知的带宽控制](feature/UDG/20.15.2/GWFD-110311.md)（GWFD-110311，activation §必备事项明列依赖）。
- **URRID 全网唯一**（critical）：activation 演示 urr_online=1000、urr_mk=1002 — URRID 在 UDG 与 UNC 两端须一致。
- **双轨 URR 同组**（warning）：URRGROUP 须同时绑 online URR（UP/DOWNURRNAME1）+ monitoring URR（UP/DOWNURRNAME2）— 缺一则功能不完整（在线计费或累计上报缺失）。
- **URRNAME/URRID 跨网元一致**（critical）：要求 UDG 和 UNC 两端的 URR 标识取值一致（activation §数据规划表明示）。
- **REFRESHSRV 必须执行**（critical）：三四层 Filter 变更后须 `SET REFRESHSRV`（REFRESHTYPE=USERPROFILE）才生效（activation 步骤 2d 明列）。
- **策略在 PCRF/PCF 侧**（info）：累计流量达阈值后的降速/改费率动作由 PCRF/PCF 下发，UDG 侧仅承载双轨上报与策略执行。
