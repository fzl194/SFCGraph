# 绑定IKE对等体到IPsec安全策略（ADD ATTACHIKEPEER）

> 命令静态知识见 [ADD ATTACHIKEPEER](command/UDG/20.15.2/ADD-ATTACHIKEPEER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典",**不**逐特性罗列取值。POLICYNAME 引用 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 的策略名;IKEPEERNAME 引用 [0-00214](task/UDG/20.15.2/0-00214.md) ADD IKEPEER 的对等体名;PEERPRIORITY 区分主备/多 Peer(1=最高,3=最低)。

## 配置方法(配置方法字典)

ADD ATTACHIKEPEER 是"IKE 对等体与 IPsec 安全策略的绑定"命令,把 IKE Peer 绑定到 IPsec 安全策略的特定序列号上,支持主备/多 Peer 场景。**关键约束**:**不支持用户级 IPsec**(TEMPLATEMODE=PolicyTemplate 仅支持用户级 IPsec),网际级 IPsec 场景下 TEMPLATEMODE 固定 None。核心配置空间有 4 个维度:**策略标识**、**模式**、**IKE 对等体**、**主备优先级**。1 个特性命中(IPFD-015004 各激活 md 10002/10003/10004/10005 等子场景)覆盖普通 IPv4/IPv6 IPsec 隧道、主备隧道两种典型场景。

### 配置维度 1:策略标识(参数 POLICYNAME + SEQUENCENUMBER,均必选)

| 参数 | 取值 | 联动 |
|---|---|---|
| **POLICYNAME** | 字符串 1~15 字符,不区分大小写 | 引用 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 的策略名 |
| **SEQUENCENUMBER** | 整数 1~10000 | 引用 IPSECPOLICY 的特定序列号(同一策略下不同序列号表示不同感兴趣流/不同 SA) |

> **多 Sequence IPsec 策略场景**(IPFD-015004 10008):同一 POLICYNAME 下不同 SEQUENCENUMBER 可绑定不同 IKE Peer(每条感兴趣流用不同对端)。

### 配置维度 2:策略模式(参数 POLICYMODE + TEMPLATEMODE,均必选)

| 子参数 | 取值 | 说明 |
|---|---|---|
| **POLICYMODE** | **Isakmp**(ISAKMP 模式,使用 IKE 建立 IPsec 安全联盟) | 唯一取值;IPsec 主流模式 |
| **TEMPLATEMODE** | **None**(无,网际级 IPsec 默认) | 网际级 IPsec 固定 None |
| TEMPLATEMODE | PolicyTemplate(策略模板模式,**仅支持用户级 IPsec**) | 用户级 IPsec 场景专用;**本命令不支持用户级 IPsec,实际不适用** |

> **互斥原则**:TEMPLATEMODE=PolicyTemplate 仅用户级 IPsec;本命令不支持用户级 IPsec,**实际 TEMPLATEMODE 固定 None**。

### 配置维度 3:IKE 对等体(参数 IKEPEERNAME,必选)

| 取值形式 | 来源 | 典型场景 |
|---|---|---|
| **字符串 1~15 字符,不区分大小写** | 引用 [0-00214](task/UDG/20.15.2/0-00214.md) ADD IKEPEER:PEERNAME | 普通场景 `IKEPEERNAME="a"`;主备场景多 Peer `IKEPEERNAME="b"`/`IKEPEERNAME="c"` |

> **前置约束**:IKEPEERNAME 必须先由 ADD IKEPEER 建对等体,且 PROPOSAL 引用 [0-00213](task/UDG/20.15.2/0-00213.md) ADD IKEPROPOSAL,LOWREMOTEADDR/INVRFNAME/OUTVRFNAME 引 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC。

### 配置维度 4:主备优先级(参数 PEERPRIORITY,必选)

| 取值 | 作用 | 典型场景 |
|---|---|---|
| **1**(最高优先级) | 主 Peer | IPFD-015004 10002 普通 IPv4 IPsec 隧道 / 10003 普通 IPv6 IPsec 隧道(`PEERPRIORITY=1`) |
| **2** | 备 Peer | IPFD-015004 10004 IPv4 IPsec 主备隧道(`PEERPRIORITY=2` 为备) |
| **3**(最低优先级) | 兜底 Peer | 多 Peer 场景,极少用 |

> **主备场景**(10004):Device A 侧配两条 ATTACHIKEPEER——主 Peer(b,PEERPRIORITY=1) + 备 Peer(c,PEERPRIORITY=2)——主链路故障时切换到备 Peer。

**典型脚本(IPFD-015004 普通 IPv4/IPv6 IPsec 隧道,单 Peer)**:
```
ADD ATTACHIKEPEER: POLICYNAME="map1", SEQUENCENUMBER=10, POLICYMODE=Isakmp, TEMPLATEMODE=None,
                   IKEPEERNAME="b", PEERPRIORITY=1;
```

**典型脚本(IPFD-015004 IPv4 IPsec 主备隧道,多 Peer + 备 Peer)**:
```
ADD ATTACHIKEPEER: POLICYNAME="map1", SEQUENCENUMBER=10, POLICYMODE=Isakmp, TEMPLATEMODE=None,
                   IKEPEERNAME="b", PEERPRIORITY=1;
ADD ATTACHIKEPEER: POLICYNAME="map1", SEQUENCENUMBER=20, POLICYMODE=Isakmp, TEMPLATEMODE=None,
                   IKEPEERNAME="c", PEERPRIORITY=1;
ADD ATTACHIKEPEER: POLICYNAME="map1", SEQUENCENUMBER=10, POLICYMODE=Isakmp, TEMPLATEMODE=None,
                   IKEPEERNAME="c", PEERPRIORITY=2;
```

> 命令 notes:**最大记录数 2000**;**执行后立即生效**;**不支持用户级 IPsec**(TEMPLATEMODE=PolicyTemplate 实际不可用);POLICYMODE 当前唯一取值为 Isakmp(IPsec 主流模式)。

**步骤位置**:在所属 feature 编排中位于"IPsec 协商链末段绑定"——前置 [0-00213](task/UDG/20.15.2/0-00213.md) IKEPROPOSAL(IKE 提议)+ [0-00214](task/UDG/20.15.2/0-00214.md) IKEPEER(IKE 对等体)+ [0-00215](task/UDG/20.15.2/0-00215.md) IPSECPOLICY(IPsec 安全策略);下游 IPsec SA 协商建立([0-00216](task/UDG/20.15.2/0-00216.md) ADD IPSECINTFCFG 后触发)。

## 决策点:主备/多 Peer 场景选择(DP 0-00300)

问:IPsec 安全策略是否需要绑定多个 IKE Peer(主备/多 Peer)?

| 选项 | 绑定条数 | 典型场景 |
|---|---|---|
| **单 Peer**(普通隧道) | 1 条 ATTACHIKEPEER(`PEERPRIORITY=1`) | IPFD-015004 10002 普通 IPv4 IPsec 隧道 / 10003 普通 IPv6 IPsec 隧道 |
| **主备 Peer**(备份切换) | 2 条 ATTACHIKEPEER——主 Peer(`PEERPRIORITY=1`)+ 备 Peer(`PEERPRIORITY=2`) | IPFD-015004 10004 IPv4 IPsec 主备隧道;主链路故障切备 |
| **多 Sequence Peer** | 同 POLICYNAME 下不同 SEQUENCENUMBER 各绑 1 个 Peer | IPFD-015004 10008 多 Sequence IPsec 策略 |

> 1 个特性命中(IPFD-015004)覆盖普通隧道 + 主备 + 多 Sequence 三种范式;TEMPLATEMODE 在本命令下固定 None(用户级 IPsec 不可用)。

## 约束

- **最大记录数 2000**(critical,rule-0-00300):整机绑定关系最大 2000 条 — 后果:超规格新增失败
- **不支持用户级 IPsec**(critical,rule-0-00300):本命令不支持用户级 IPSEC,TEMPLATEMODE=PolicyTemplate 不可用 — 后果:配 PolicyTemplate 命令拒收
- **执行后立即生效**(warning,rule-0-00300):执行后立即生效 — 后果:误操作即刻生效
- **POLICYMODE 当前唯一取值 Isakmp**(warning,rule-0-00300):POLICYMODE 枚举当前仅 Isakmp(IPsec 主流模式) — 后果:其他取值命令拒收
- **TEMPLATEMODE 网际级 IPsec 固定 None**(warning,rule-0-00300):本命令不支持用户级 IPsec,实际 TEMPLATEMODE 固定 None — 后果:配 PolicyTemplate 命令拒收
- **POLICYNAME 必须先建 IPsec 安全策略**(critical,rule-0-00300):POLICYNAME 必须在 [0-00215](task/UDG/20.15.2/0-00215.md) ADD IPSECPOLICY 已建策略下 — 后果:未建策略绑定失败
- **IKEPEERNAME 必须先建 IKE 对等体**(critical,rule-0-00300):IKEPEERNAME 必须在 [0-00214](task/UDG/20.15.2/0-00214.md) ADD IKEPEER 已建对等体下 — 后果:未建对等体绑定失败
- **SEQUENCENUMBER 范围 1~10000**(warning,rule-0-00300):整数 1~10000,超出范围命令拒收 — 后果:违反范围命令拒收
- **PEERPRIORITY 范围 1~3**(warning,rule-0-00300):整数 1~3,1=最高/3=最低 — 后果:违反范围命令拒收
- **POLICYNAME 长度 1~15**(info,rule-0-00300):字符串 1~15 字符,不区分大小写 — 后果:超长命令拒收
- **IKEPEERNAME 长度 1~15**(info,rule-0-00300):字符串 1~15 字符,不区分大小写 — 后果:超长命令拒收
- **主备 Peer 同 POLICYNAME**(info,rule-0-00300):主备 Peer 必须绑定同一 POLICYNAME(同一策略) — 后果:不同 POLICYNAME 下主备无意义
