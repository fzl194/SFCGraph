# 建IPsec微服务逻辑接口（ADD INTERFACEIPSEC）

> 命令静态知识见 [ADD INTERFACEIPSEC](command/UDG/20.15.2/ADD-INTERFACEIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。本命令是 IPsec 微服务侧"VPN + 隧道接口"双配的"建接口"步骤——在 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC 建好 VPN + 地址族之后下发，建 Tunnel 或 LoopBack 逻辑接口；后置 [0-00306](task/UDG/20.15.2/0-00306.md) ADD IPBINDVPNIPSEC 接口绑 VPN + [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC 接口配 IPv4 地址。**双配铁律**：IFNAME 必须与 VNRS 侧 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE **完全相同**，否则 IPsec 微服务侧找不到对应接口。

## 配置方法（配置方法字典）

ADD INTERFACEIPSEC 是"IPsec 微服务侧逻辑接口创建"命令。8 个参数中 1 必选（IFNAME）+ 7 可选（IFDESCR/IFADMINSTATUS/IFMTU/IFDF/IFSTATIENABLE/IFTRAPENABLE/IFSTATITVL）。配置空间有 5 个维度：**接口名**（IFNAME 引用命名规则）、**管理状态**（IFADMINSTATUS Up/Down）、**MTU**（IFMTU 46~9600 默认 1500）、**分片标记**（IFDF TRUE/FALSE 默认 FALSE）、**统计**（IFSTATIENABLE + IFSTATITVL 联动）。1 个特性命中（IPFD-015004 普通 IPv4/IPv6 IPsec 隧道等 8 子场景）覆盖典型范式。

### 配置维度 1：接口名（参数 IFNAME，必选）

| 取值形式 | 命名规则 | 来源 |
|---|---|---|
| **字符串 1~63（接口类型+接口编号）** | 类型 + 编号；Tunnel 接口如 `Tunnel10` / `Tunnel2` / `Tunnel17` / `Tunnel27`；LoopBack 接口如 `LoopBack1` | 本端规划，须与 VNRS 侧 [0-00056](task/UDG/20.15.2/0-00056.md) 一致 |

> **接口双配铁律**：本参数必须与 VNRS 侧 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE:IFNAME **完全相同**（同一 Tunnel/LoopBack 编号），否则 IPsec 微服务侧找不到对应接口。
>
> **必先建 VPN**：本命令通常在 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC + 地址族（VPNINSTAFIPSEC）之后下发；裸建接口（无 VPN 上下文）在 IPsec 微服务范式中无业务意义。

### 配置维度 2：管理状态（参数 IFADMINSTATUS，可选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **Up** | 接口 Up（IPFD-015004 范式默认） | 所有 IPsec 隧道接口必须 Up，协商前需接口可达 |
| **Down** | 接口 Down | 预配置但暂时禁用场景 |

> **隐式默认**：未传 IFADMINSTATUS 时接口默认 Up 状态（多数命中脚本显式传 `IFADMINSTATUS=Up`，如 Tunnel10/17/27；LoopBack1 多数不传 IFADMINSTATUS）。

### 配置维度 3：MTU（参数 IFMTU，可选，默认 1500）

| 取值形式 | 范围 | 适用场景 |
|---|---|---|
| **1500**（默认） | 整数 46~9600 字节 | IPFD-015004 范式默认；正常通信建议采用系统默认 |
| **小于 1500** | 整数 46~9600 字节 | 协议对最小报文有限制时调整（可能导致协议邻居无法建立） |
| **大于 1500** | 整数 46~9600 字节 | 大帧场景（如 jumbo frame） |

> **MTU 调整铁律**：MTU 默认 1500，**正常通信不建议随意更改**；某些协议对最小报文有限制，MTU 配置过小会导致协议邻居无法建立。

### 配置维度 4：分片标记（参数 IFDF，可选，默认 FALSE）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **TRUE** | IP 报文头 DF 标志位置 1，强制不允许分片 | 与对端协商不分片，避免分片丢失场景 |
| **FALSE**（默认） | 允许分片 | IPFD-015004 范式几乎全配 FALSE（所有命中 Tunnel 脚本显式或隐式传 IFDF=FALSE） |

> **DF=TRUE 风险**：DF=1 时如果 IP 报文长度大于接口 MTU（例如协议封装导致长度增加），报文将被丢弃并发送 ICMP 差错报文，从而引起数据丢失——IPsec 加密承载场景务必谨慎。

### 配置维度 5：统计（参数 IFSTATIENABLE + IFSTATITVL，可选，联动）

| 子参数 | 取值 | 默认 | 范围 | 作用 |
|---|---|---|---|---|
| **IFSTATIENABLE** | TRUE / FALSE | FALSE | — | 是否使能接口流量统计（接收/发送报文计数） |
| **IFSTATITVL** | 整数（秒） | — | 5~600（10 的倍数） | 流量统计时间间隔 |

> **联动铁律**：IFSTATITVL 仅在 IFSTATIENABLE=TRUE 时有效；统计间隔必须**是 10 的倍数**。

### 维度 6（辅助）：描述/Trap（参数 IFDESCR/IFTRAPENABLE，可选）

| 子参数 | 取值 | 范围 | 作用 |
|---|---|---|---|
| **IFDESCR** | 字符串 | 1~242 | 接口描述，标识接口用途（如 `huawei`） |
| **IFTRAPENABLE** | TRUE / FALSE | — | 是否使能接口 Down 告警上报 |

**典型脚本（IPFD-015004 普通 IPv6 IPsec 隧道，IPv6 Tunnel 接口 Up）**：
```
ADD INTERFACEIPSEC:IFNAME="Tunnel27", IFADMINSTATUS=Up;
```

**典型脚本（IPFD-015004 普通 IPv4 IPsec 隧道，IPv4 Tunnel 接口 Up）**：
```
ADD INTERFACEIPSEC:IFNAME="Tunnel10", IFADMINSTATUS=Up;
```

**典型脚本（IPFD-015004 GRE over IPsec，MTU+IFDF+统计间隔三参）**：
```
ADD INTERFACEIPSEC:IFNAME="Tunnel2",IFADMINSTATUS=Up,IFMTU=1500,IFDF=FALSE,IFSTATITVL=20;
```

**典型脚本（IPFD-015004 IPv4 主备隧道 LoopBack 配 Up）**：
```
ADD INTERFACEIPSEC:IFNAME="LoopBack1",IFADMINSTATUS=Up,IFMTU=1500,IFDF=FALSE;
```

**典型脚本（IPFD-015004 LoopBack 仅必选 IFNAME，隐式默认 Up）**：
```
ADD INTERFACEIPSEC:IFNAME="LoopBack1";
```

> 命令 notes：**执行后立即生效**；**该命令在版本升级过程中禁止执行**；**最多可输入 65535 条记录**；**权限**：G_1 管理员 / G_2 操作员命令组；本命令用于建**逻辑接口**（物理上不存在，需通过配置建立），物理接口不在此命令范围。

**步骤位置**：在所属 feature 编排中位于"步骤 3：IPsec 微服务五件套——建逻辑接口"——前置 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC（建 VPN）+ VPNINSTAFIPSEC（加地址族）；后置 [0-00306](task/UDG/20.15.2/0-00306.md) ADD IPBINDVPNIPSEC（接口绑 VPN）+ [0-00304](task/UDG/20.15.2/0-00304.md) ADD IFIPV4ADDRESSIPSEC（接口配 IPv4 地址）；下游 IPsec 协商链：[0-00307](task/UDG/20.15.2/0-00307.md) IPsec Policy→Tunnel 接口绑定。

## 决策点：IPsec 子场景选择（DP 0-00305）

问：本接口属于哪种 IPsec 子场景？（决定 IFNAME 类型 Tunnel/LoopBack + 是否显式传 IFADMINSTATUS/IFMTU/IFDF）

| 选项 | IFNAME | IFADMINSTATUS | IFMTU | IFDF | 典型特征 |
|---|---|---|---|---|---|
| 普通 IPv4/IPv6 IPsec 隧道（10002/10003） | `Tunnel10` / `Tunnel27` | Up | 默认 1500 | FALSE | Tunnel 接口，必须 Up |
| LoopBack 本端源（10009） | `LoopBack1` | Up（部分隐式） | 默认 1500 | FALSE | LoopBack 接口，本端源 IP 标识 |
| IPv4/IPv6 主备隧道（10004/10005） | 双 Tunnel 各一条 | Up | 默认 1500 | FALSE | 主备 Tunnel 协商 |
| GRE over IPsec（10006） | `Tunnel2` | Up | 1500 | FALSE | 内层 GRE 隧道 |
| OSPF over IPsec（10007） | `Tunnel2` | Up | 1500 | FALSE | 动态路由协议 over IPsec |
| 指定本端接口（10009） | `LoopBack1` 显式 Up+IFMTU+IFDF | Up | 1500 | FALSE | 本端源固定 |

> **隐式默认 vs 显式传**：多数 Tunnel 命中脚本显式传 `IFADMINSTATUS=Up`；LoopBack1 部分显式传 Up+MTU+IFDF，部分仅传 IFNAME（隐式默认）。

## 约束

- **最多 65535 条记录**（warning，rule-0-01305）：系统最多可输入 65535 条接口记录 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-01305）：命令执行后立即生效 — 后果：误操作即刻生效
- **版本升级过程中禁止执行**（critical，rule-0-01305）：版本升级过程中禁止执行本命令 — 后果：升级过程中执行可能破坏系统状态
- **IFNAME 长度**（info，rule-0-01305）：字符串 1~63（接口类型+接口编号） — 后果：超长命令拒收
- **IFMTU 范围**（critical，rule-0-01305）：整数 46~9600 字节 — 后果：超范围命令拒收
- **MTU 默认值**（info，rule-0-01305）：默认 1500 字节 — 后果：未配 IFMTU 时取 1500
- **MTU 调整风险**（warning，rule-0-01305）：正常通信不建议随意更改 MTU；过小会导致协议邻居无法建立 — 后果：MTU 过小协议邻居建立失败
- **IFDF 风险**（warning，rule-0-01305）：DF=TRUE 时如果 IP 报文长度大于接口 MTU（例如协议封装导致长度增加），报文将被丢弃并发送 ICMP 差错报文 — 后果：分片丢失引发数据丢失
- **IFSTATITVL 范围**（critical，rule-0-01305）：整数 5~600 秒 — 后果：超范围命令拒收
- **IFSTATITVL 必须是 10 的倍数**（critical，rule-0-01305）：统计间隔必须是 10 的倍数 — 后果：非 10 倍数命令拒收
- **IFSTATITVL 仅在 IFSTATIENABLE=TRUE 时有效**（info，rule-0-01305）：IFSTATITVL 需 IFSTATIENABLE 使能才有意义 — 后果：未使能统计时统计间隔配置无效
- **IFDESCR 长度**（info，rule-0-01305）：字符串 1~242 — 后果：超长命令拒收
- **双配铁律：IFNAME 与 VNRS 侧一致**（critical，rule-0-01305）：IFNAME 必须与 [0-00056](task/UDG/20.15.2/0-00056.md) VNRS 侧 ADD INTERFACE 的 IFNAME 完全相同 — 后果：IFNAME 不一致 IPsec 隧道不通
- **必先建 VPN**（warning，rule-0-01305）：典型 IPsec 微服务编排要求先 [0-00210](task/UDG/20.15.2/0-00210.md) ADD L3VPNINSTIPSEC + VPNINSTAFIPSEC 再下发本命令 — 后果：无 VPN 上下文时接口无业务意义
- **逻辑接口范围**（info，rule-0-01305）：本命令仅用于建逻辑接口（Tunnel/LoopBack），物理接口不在此范围 — 后果：物理接口由专用命令管理
- **权限要求**（info，rule-0-01305）：需 G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
