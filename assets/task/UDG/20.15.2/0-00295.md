# 设置异常流量检测报文阈值（SET ABNTRADTTHR）

> 命令静态知识见 [SET ABNTRADTTHR](command/UDG/20.15.2/SET-ABNTRADTTHR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET ABNTRADTTHR 设置整机终端异常下行流量检测的报文阈值个数。**系统存在初始记录（DETECTTHR 初始值=20），命令最大记录数 1（全局唯一）**。仅 GWFD-020305 终端异常下行流量检测的**方案二（基于报文行为检测）**需配置；方案一（基于业务感知，依赖 SA）通过 ADD ABNTRAFFICDT 的检测开关实现，不需本命令。

**主场景范式（方案二：调整检测报文阈值）**

| 字段 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| DETECTTHR | 20 | 本端规划 | 异常流量检测报文阈值个数，1-255；建议根据现网异常下行流量值配置 |

**脚本样例**
```
SET ABNTRADTTHR:DETECTTHR=20;
```

**配置链路**（GWFD-020305 方案二）：SET SRVCOMMONPARA（SIGDEFERREDTIME≥60s）→ SET LICENSESWITCH → **SET ABNTRADTTHR**（可选，方案二）→ ADD ABNTRAFFICDT（APN 级检测开关）。

## 约束

- **仅对新激活用户生效**（warning）：命令执行后只对新激活用户生效，不影响已在线用户。
- **全局唯一**（info）：最大记录数 1，系统存在初始记录（DETECTTHR=20）— 无需 ADD，直接 SET 修改。
