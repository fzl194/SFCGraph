# 设置异常流量检测报文阈值（SET ABNTRADTTHR）

> 命令静态知识见 [SET ABNTRADTTHR](command/UDG/20.15.2/SET-ABNTRADTTHR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`SET ABNTRADTTHR` 设置整机终端异常下行流量检测的报文阈值个数。**仅 SGW-U/PGW-U/UPF 适用**。**系统存在初始记录（DETECTTHR 初始值=20），命令最大记录数 1（全局唯一）**。仅 GWFD-020305 终端异常下行流量检测的**方案二（基于报文行为检测）**需配置；方案一（基于业务感知，依赖 SA）通过 `ADD ABNTRAFFICDT` 的检测开关实现，不需本命令。

### 配置维度 1：检测报文阈值（参数 DETECTTHR）
| 取值 | 作用 | 约束 |
|---|---|---|
| 整数 1~255（系统初始 20） | 异常流量检测触发所需的报文数 | 可选；建议根据现网异常下行流量值配置 |

**典型脚本**：

```
SET ABNTRADTTHR:DETECTTHR=20;
```

**配置链路**（GWFD-020305 方案二）：`SET SRVCOMMONPARA`（SIGDEFERREDTIME≥60s）→ `SET LICENSESWITCH` → **SET ABNTRADTTHR**（可选，方案二）→ `ADD ABNTRAFFICDT`（APN 级检测开关）。

## 决策点

（无。本命令在所有特性下用法一致：单条全局配置，DETECTTHR 一个可选参数。无 DP 分支；"是否开启异常流量检测"由 `ADD ABNTRAFFICDT` 开关控制，不属本命令维度。）

## 约束

- **rule-0-00295-1**（warning）：命令执行后只对新激活用户生效 — 不影响已在线用户
- **rule-0-00295-2**（info）：最大记录数 1，系统存在初始记录（DETECTTHR=20）— 无需 ADD，直接 SET 修改
- **rule-0-00295-3**（info）：本命令仅方案二（基于报文行为检测）需要；方案一（基于业务感知）通过 `ADD ABNTRAFFICDT` 检测开关实现，不需本命令
