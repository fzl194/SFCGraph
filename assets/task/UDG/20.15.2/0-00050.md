# 配置APN下用户接入速率（SET APNACCESSWAL）

> 命令静态知识见 [SET APNACCESSWAL](command/UDG/20.15.2/SET-APNACCESSWAL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令为某 APN 设置用户接入速率上限，用于过载控制时防止该 APN 下接入速率过快冲击周边网元。只有两个参数：APN 名 + 速率阈值。

**WALNUMBER 取值约定**

| 取值 | 含义 |
|---|---|
| 0 | 取消对该 APN 的接入速率限制 |
| 30~1000000（个/s） | 限制该 APN 下用户接入速率为此值 |

**典型行（GWFD-010251 实例）**
```
SET APNACCESSWAL: APN="apn1", WALNUMBER=30;
```

**前置依赖**：APN 需先由 ADD APN（0-00020）配置。

**同特性相关任务（超出本批 atom 预算，待后续补建）**：SET DEACTIVERATE（去活速率）、SET SOFTPARA 同属 GWFD-010251 系统过载控制特性，已在 feature task（2-00023）挂 dependency_rule 标记待补。

## 约束

- **rule-0-00154**（warning）高危命令：最大记录数 10000；命令执行后立即生效；流控失败将给用户返回**无资源可用**原因值；需配置合理的接入速率取值（取值不合理无法保护网元或导致用户激活失败） — 后果：超规格新增失败；流控触发时用户收到无资源可用原因值
