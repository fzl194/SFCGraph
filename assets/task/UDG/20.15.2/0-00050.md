# 配置APN下用户接入速率（SET APNACCESSWAL）

> 命令静态知识见 [SET APNACCESSWAL](command/UDG/20.15.2/SET-APNACCESSWAL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

SET APNACCESSWAL 是 APN 级"接入流控限速"：针对指定 APN 设置用户接入速率上限（个/s），用于过载控制时防止该 APN 下接入速率过快冲击周边网元。命令核心配置空间有 2 个维度：**APN 引用**（APN 必选 + 必须已建）、**接入速率**（WALNUMBER 必选，整数）。1 个特性命中（GWFD-010251 系统过载控制）取 WALNUMBER=30 范式。

### 配置维度 1：APN 引用（参数 APN，必选）

| 参数 | 范围 | 备注 |
|---|---|---|
| **APN** | 字符串 1~63 | 引用已配的 [ADD APN](task/UDG/20.15.2/0-00020.md) APN；前置依赖：APN 必须先建；每个 APN 仅一条本配置 |

### 配置维度 2：接入速率（参数 WALNUMBER，必选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **WALNUMBER** | **0** | 取消对该 APN 的接入速率限制（不限速） |
| **WALNUMBER** | **30~1000000**（个/s） | 限制该 APN 下用户接入速率为此值 |
| **WALNUMBER** | 其他值（1~29 或 >1000000） | 非法范围，命令拒收 |

> 010251 范式：WALNUMBER=30（典型下限值）。

**典型脚本（010251 系统过载控制）**：
```
SET APNACCESSWAL:APN="apn1",WALNUMBER=30;
```

> 命令 notes：适用 NF：SGW-U、PGW-U、UPF；本命令属高危命令（配置不合理可能无法保护网元或导致用户激活失败）；执行后立即生效；流控触发时用户返回"无资源可用"原因值。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：APN 接入流控配置"——前置依赖 [0-00020](task/UDG/20.15.2/0-00020.md)（ADD APN）；同 backbone 系统过载控制：[SET DEACTIVERATE（★R1.5 atom 待补）](task/UDG/20.15.2/0-00XXX.md)、[SET SOFTPARA（★R1.5 atom 待补）](task/UDG/20.15.2/0-00XXX.md)。

## 决策点：是否限速（DP 0-00050）

问：是否对指定 APN 设置接入速率限制？

| 选项 | WALNUMBER | 适用场景 |
|---|---|---|
| 不限速（取消限制） | 0 | 取消已有限制（仅 0 触发取消语义） |
| 限速 | 30~1000000 | 系统过载控制（010251 范式：WALNUMBER=30） |

> 1 个特性命中（010251）取 WALNUMBER=30；本命令仅 2 个参数，DP 实质为"是否限速 + 限速值"的二选一。

## 约束

- **APN 前置依赖**（critical，rule-0-00450）：APN 必须先由 [0-00020](task/UDG/20.15.2/0-00020.md)（ADD APN）建好 — 后果：未建 APN 命令拒收
- **APN 唯一**（critical，rule-0-00450）：每个 APN 仅一条接入速率配置 — 后果：同 APN 重复配置命令拒收
- **WALNUMBER 范围约束**（critical，rule-0-00450）：仅 0 或 30~1000000 合法 — 后果：1~29 或 >1000000 命令拒收
- **高危命令**（warning，rule-0-00450）：配置不合理可能无法保护网元或导致用户激活失败 — 后果：流控触发时用户收到"无资源可用"原因值
- **最大记录数 10000**（warning，rule-0-00450）：超规格新增失败 — 后果：超 10000 拒收
- **适用 NF 限定**（warning，隐含）：SGW-U、PGW-U、UPF — 后果：其他 NF 不适用
- **执行后立即生效**（warning，rule-0-00450）：不需重启 — 后果：执行后立即对后续接入生效
