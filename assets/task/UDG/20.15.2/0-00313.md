# 创建OSPF接口配置（ADD OSPFINTERFACE）

> 命令静态知识见 [ADD OSPFINTERFACE](command/UDG/20.15.2/ADD-OSPFINTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典",**不**逐特性罗列取值。PROCID 引用 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 的 PROCID;AREAID 引用 [0-00075](task/UDG/20.15.2/0-00075.md) ADD OSPFAREA 的 AREAID;IFNAME 引用 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 的 IFNAME;本命令优先级**大于** [0-00076](task/UDG/20.15.2/0-00076.md) ADD OSPFNETWORK(命令 notes 硬约束)。

## 配置方法(配置方法字典)

ADD OSPFINTERFACE 是"使能接口到 OSPF 进程/区域下"命令。OSPF 是基于接口的链路状态路由协议,本命令把已建的三层接口绑定到 OSPF 进程与区域上,使该接口参与 OSPF 邻居/路由计算。核心配置空间有 4 个维度:**区域归属**(PROCID+AREAID 三层定位)、**接口绑定**(IFNAME 引用 INTERFACE)、**接口网络类型**(NETWORKTYPE 4 选 1)、**接口增强**(12 类开关:DR 优先级/Hello/Dead/Cost/MTU/Smart-Discover/静默/BFD/FRR/MPLS-LDP/LDP 同步/共网段/最大开销抑制)。1 个特性命中(010107 静态地址用户路由冗余)覆盖 LoopBack 接口使能 OSPF 场景。

### 配置维度 1:区域归属(参数 PROCID + AREAID,必选)

| 参数 | 取值 | 联动必选 | 来源 |
|---|---|---|---|
| **PROCID**(进程号) | 整数 1~4294967295 | AREAID | 必先经 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 创建 |
| **AREAID**(区域标识) | IPv4 点分十进制(如 `0.0.0.0`、`0.0.0.5`) | — | 必先经 [0-00075](task/UDG/20.15.2/0-00075.md) ADD OSPFAREA 创建 |

> **三层拓扑铁律**:**进程 → 区域 → 接口** 必须三段都已建——PROCID(进程)、AREAID(区域)未建则命令拒收。**自动化配置**:`PROCID/AREAID/TIMERDEADINTV/TIMERHELLOINTV/CONFIGCOST` 参与自动化配置,下发前需用 `DSP OPSASSISTSTATE（★R1.5 atom 待补）` 查询 autoscaling_autoconfig.py 状态,确保为"准备好"或不存在;否则最终生效配置不可预期。

### 配置维度 2:接口绑定(参数 IFNAME,必选)

| 取值 | 范围约束 | 联动 |
|---|---|---|
| **字符串 1~63** | 必先经 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 创建(可用 `LST INTERFACE` 查) | 010107 范例取 `"LoopBack1"` / `"LoopBack2"` |

> **优先级铁律**:ADD OSPFINTERFACE 命令接口使能的优先级**大于** [0-00076](task/UDG/20.15.2/0-00076.md) ADD OSPFNETWORK(命令 notes 硬约束)——同一接口若既配 OSPFINTERFACE 又配 OSPFNETWORK,以前者为准。

### 配置维度 3:接口网络类型(参数 NETWORKTYPE,可选 4 选 1)

NETWORKTYPE 决定 OSPF 在该接口上的邻居发现与 DR 选举行为:

| 取值 | 作用 | 配套必选参数 | 典型场景 |
|---|---|---|---|
| **broadcast**(默认多数接口) | 广播网;自动发现邻居,DR/BDR 选举 | DR 选举类(DRPRI/DR election 在 broadcast 生效) | Ethernet 接口使能 OSPF(默认范式) |
| **p2p** | 点到点;无 DR 选举,自动发现邻居 | — | PPP/HDLC/Tunnel 点到点链路 |
| **nbma** | 非广播多路访问;**手动配 OSPFPEER** | 必加 ADD OSPFPEER（★R1.5 atom 待补） 命令配邻接点(命令 notes 硬约束) | 帧中继/ATM NBMA 网络 |
| **p2mp** | 点到多点;按需配 P2MPMASKIGN 忽略掩码检查 | 可选 P2MPMASKIGN | 部分 NBMA/Hub-Spoke |

> **链路两端必须一致**:一般情况下,链路两端的 OSPF 接口网络类型必须一致,否则双方无法建立邻居关系(命令 notes 硬约束)。**默认推断**:不输 NETWORKTYPE 则按接口类型推断——大部分接口为 broadcast,LoopBack 接口为 p2p。

### 配置维度 4:接口增强(12 类开关,可选)

| 开关参数 | 取值/范围 | 默认 | 作用 |
|---|---|---|---|
| **DRPRI**(指定路由器优先级) | 0~255 | 1 | 0=不参与 DR 选举;改 DRPRI 不会重选,需重启进程(不推荐) |
| **TIMERHELLOINTV**(Hello 间隔,s) | 1~65535 | P2P/Broadcast=10;P2MP/NBMA=30 | 发送 Hello 报文周期;两端必须一致 |
| **TIMERDEADINTV**(邻居失效时间,s) | 1~235926000 | 4 × TIMERHELLOINTV(补偿:小于 10s 取 max(2×配置,10)) | 邻居超时时间;必 > TIMERHELLOINTV |
| **TIMERCONSERFLAG**(邻居失效保守模式) | TRUE/FALSE | FALSE | TRUE 时 Dead < 10s 也按实际值判断失效 |
| **TIMERRETRANINTV**(LSA 重传间隔,s) | 1~3600 | 5 | LSA 重传等待时间;不要设太小 |
| **TIMERPOLLINTV**(NBMA 轮询 Hello 间隔,s) | 1~3600 | 4 × TIMERHELLOINTV | NBMA 失效后轮询 Hello;至少 4×Hello |
| **TIMERWAITINTV**(等待超时,s) | 1~235926000 | Broadcast=40;NBMA=120 | 仅 Broadcast/NBMA 生效;推荐 ≤ Dead |
| **CONFIGCOST**(接口开销值) | 0~65535 | 由公式 `带宽参考值/接口带宽` 计算(参考值 100Mbit/s) | 0=恢复公式计算 |
| **MTUENABLE**(使能接口 MTU) | TRUE/FALSE | FALSE | 启用接口 MTU |
| **SMARTDISCOVER**(Smart-Discover) | TRUE/FALSE | FALSE | 加快邻居发现 |
| **SILENTENABLE**(静默接口) | TRUE/FALSE | 未配 | 禁止该接口收发 OSPF 报文(受进程 SILENTALLFLAG 联动) |
| **P2MPMASKIGN**(P2MP 忽略掩码) | TRUE/FALSE | FALSE | P2MP 掩码不一致时忽略 Hello 掩码检查 |
| **SUPPEREACH**(可达性抑制) | Disable/Enable/Not-Configured | Not-Configured | 抑制接口地址发布 |
| **TRANSDELAY**(LSA 传输延迟,s) | 1~500 | 1 | LSA 在发送前的老化时间增量(低速网重要) |
| **LINKCOST**(低链路质量 Cost) | 1~65534 | 未配 | 链路质量下降时提高 Cost |
| **VIRTUALSYSFLAG**(共网段虚拟系统) | TRUE/FALSE | FALSE | 进程+接口同时配才能共网段;生效后 DRPRI 自动=0;Tunnel 接口不支持 |
| **CFGROUTERIDFLAG** + **CFGROUTERID**(虚 Router-id) | TRUE+IPv4 | FALSE | 仅 VIRTUALSYSFLAG=TRUE 时可选/必选;启用 OSPF 虚 Router-id |
| **MAXCOSTFLAG** + **MAXCOSTITVL**(保持最大开销) | TRUE+0~65535 | FALSE | LDP 重建前 OSPF 保持最大开销时长;ITVL 仅 FLAG=TRUE 时必选 |
| **INFINITEFLAG**(永久保持最大开销) | TRUE/FALSE | FALSE | 与 MAXCOSTFLAG 互斥,不可同时 TRUE |
| **HOLDDOWNITVL**(抑制邻居时间,s) | 0~65535 | 10 | 不建立邻居等待 LDP 会话建立的时间 |

**BFD/FRR/LDP 联动类**(5 个开关):

| 开关参数 | 取值/范围 | 默认 | 作用 |
|---|---|---|---|
| **BFDENABLE**(使能 BFD) | TRUE/FALSE | FALSE | 与 BFDBLOCK 互斥,不可同时 TRUE |
| **MINRXRECVINTV**(BFD 接收间隔,ms) | 30~1000 | 200 | 期望对端 BFD 报文最小接收间隔 |
| **MINTXTRANINTV**(BFD 发送间隔,ms) | 30~1000 | 200 | 向对端发送 BFD 报文最小发送间隔 |
| **DETECTMULINTV**(BFD 检测倍数) | 3~50 | 3 | BFD 本地检测倍数 |
| **FRRBINDINGFLAG**(BFD 绑链路状态) | TRUE/FALSE | FALSE | BFD Down 触发 OSPF Auto FRR 快速重路由 |
| **FRRBLOCKFLAG**(阻止 FRR) | TRUE/FALSE | FALSE | 阻止该接口 FRR 能力 |
| **BFDBLOCK**(阻塞动态 BFD) | TRUE/FALSE | FALSE | 与 BFDENABLE 互斥;阻塞动态创建 BFD |
| **MPLSLDPAUTOCFG**(禁自动 MPLS LDP) | None/Disable | None | None=恢复缺省;Disable=禁止自动使能 MPLS LDP |
| **LDPSYNCENABLE**(LDP-OSPF 同步) | TRUE/FALSE | FALSE | 启用 LDP 与 OSPF 同步 |
| **LDPSYNCBLOCK**(阻止 LDP-OSPF 同步) | TRUE/FALSE | FALSE | 阻止该接口运行 LDP-OSPF 同步 |

**典型脚本(命令 wiki 范例,Ethernet64/0/5 使能到进程 1 区域 0.0.0.0,仅三必选)**:
```
ADD OSPFINTERFACE:PROCID=1, AREAID="0.0.0.0", IFNAME="Ethernet64/0/5";
```

**典型脚本(010107 静态地址用户路由冗余,LoopBack1/LoopBack2 使能到进程 6 区域 0.0.0.5)**:
```
ADD OSPFINTERFACE: PROCID=6, AREAID="0.0.0.5", IFNAME="LoopBack1";
ADD OSPFINTERFACE: PROCID=6, AREAID="0.0.0.5", IFNAME="LoopBack2";
```

> 命令 notes:**最大记录数 65535**(单进程 2000,以服务规格文件为准);**执行后立即生效**;**PROCID/AREAID/IFNAME 须先建**;**优先级大于 OSPFNETWORK**;**OSPF 不支持 NULL 接口**;**链路两端 NETWORKTYPE 必须一致**;**NBMA 类型须配 OSPFPEER**;**BFDENABLE 与 BFDBLOCK 互斥**;**INFINITEFLAG 与 MAXCOSTFLAG 互斥**;**改 DRPRI 不重选**(需重启进程);**G_1 管理员/G_2 操作员**。

**步骤位置**:在所属 feature 编排中通常位于"步骤 N:OSPF 接口落地"——前置 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF(进程)+ [0-00075](task/UDG/20.15.2/0-00075.md) ADD OSPFAREA(区域)+ [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE(接口);可选 [0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION(BFD 联动);NBMA 类型须加 ADD OSPFPEER（★R1.5 atom 待补）;下游 OSPF 邻居建立 + 路由计算生效(立即)。

## 决策点:OSPF 接口网络类型(DP 0-00313)

问:本 OSPF 接口属于哪种网络类型?(决定 NETWORKTYPE 取值 + 是否需 OSPFPEER + DR 选举行为 + 计时器默认值)

| 选项 | NETWORKTYPE | OSPFPEER | DR 选举 | Hello/Dead 默认 | 典型场景 |
|---|---|---|---|---|---|
| 广播网(默认) | broadcast | 不需 | 有(DRPRI 生效) | Hello=10s,Dead=40s | Ethernet 接口使能 OSPF |
| 点到点 | p2p | 不需 | 无 | Hello=10s,Dead=40s | PPP/HDLC/Tunnel 点到点 |
| 非广播多路访问 | nbma | **必配** | 有(DRPRI 生效) | Hello=30s,Dead=120s | 帧中继/ATM NBMA |
| 点到多点 | p2mp | 不需 | 无 | Hello=30s,Dead=120s | NBMA/Hub-Spoke,可配 P2MPMASKIGN |

> 1 个特性命中(010107 LoopBack 接口)对应 p2p(LoopBack 默认推断为 p2p,即便不显式传 NETWORKTYPE);**链路两端 NETWORKTYPE 必须一致**——错配邻居无法建立。**改 NETWORKTYPE 不显式重配 DR 选举类参数(DRPRI)**——broadcast 才有 DR 选举。

## 约束

- **最大记录数 65535**(warning,rule-0-00313):系统最大记录数 65535 条;单进程默认 2000,以服务规格文件为准 — 后果:超规格新增失败
- **执行后立即生效**(warning,rule-0-00313):命令执行后立即生效,误操作即刻影响 OSPF 邻居/路由 — 后果:误配立即产生网络震荡
- **PROCID 须先建**(critical,rule-0-00313):PROCID 必须先经 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 创建 — 后果:进程不存在命令拒收
- **AREAID 须先建**(critical,rule-0-00313):AREAID 必须先经 [0-00075](task/UDG/20.15.2/0-00075.md) ADD OSPFAREA 创建 — 后果:区域不存在命令拒收
- **IFNAME 须先建**(critical,rule-0-00313):IFNAME 必须先经 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 创建 — 后果:接口不存在命令拒收
- **三层拓扑铁律**(critical,rule-0-00313):进程→区域→接口三段必须都已建(OSPF → OSPFAREA → INTERFACE) — 后果:任一未建命令拒收
- **OSPFINTERFACE 优先级 > OSPFNETWORK**(critical,rule-0-00313):同一接口既配 OSPFINTERFACE 又配 OSPFNETWORK,以前者为准(命令 notes 硬约束) — 后果:NETWORK 配置被 INTERFACE 覆盖
- **OSPF 不支持 NULL 接口**(critical,rule-0-00313):NULL 接口不支持 OSPF 配置(TIMER*/NETWORKTYPE/MTU 等多数参数均 NULL 不支持) — 后果:NULL 接口上配命令拒收
- **链路两端 NETWORKTYPE 一致**(critical,rule-0-00313):一般情况下,链路两端的 OSPF 接口网络类型必须一致 — 后果:错配邻居无法建立
- **NBMA 必配 OSPFPEER**(critical,rule-0-00313):NETWORKTYPE=nbma 或手工改 nbma 时,必须用 ADD OSPFPEER（★R1.5 atom 待补） 命令配邻接点 — 后果:未配 OSPFPEER 邻居无法建立
- **Tunnel 不支持共网段**(critical,rule-0-00313):Tunnel 接口不支持 OSPF 共网段配置(VIRTUALSYSFLAG=TRUE) — 后果:Tunnel 配共网段命令拒收
- **BFDENABLE 与 BFDBLOCK 互斥**(critical,rule-0-00313):BFDENABLE=TRUE 与 BFDBLOCK=TRUE 不可同时配置 — 后果:同时 TRUE 命令拒收
- **INFINITEFLAG 与 MAXCOSTFLAG 互斥**(critical,rule-0-00313):INFINITEFLAG=TRUE 与 MAXCOSTFLAG=TRUE 不可同时配置 — 后果:同时 TRUE 命令拒收
- **TIMERDEADINTV > TIMERHELLOINTV**(critical,rule-0-00313):Dead 时间必须大于 Hello 时间 — 后果:违例命令拒收
- **TIMERDEADINTV < 10s 补偿**(warning,rule-0-00313):若 TIMERCONSERFLAG!=TRUE 且 Dead<10s,生效值为 max(2×Dead,10) — 后果:实际生效非配置值
- **改 DRPRI 不重选**(info,rule-0-00313):重配 DR 优先级后不会改变网络中的 DR/BDR;需重启所有设备 OSPF 进程(不推荐) — 后果:配置生效但选举不变
- **SILENTENABLE 受进程 SILENTALLFLAG 联动**(info,rule-0-00313):接口 SILENTENABLE 实际生效值受本参数与进程 SILENTALLFLAG 联动,后配的为实际生效 — 后果:后配覆盖
- **共网段生效后 DRPRI 自动=0**(warning,rule-0-00313):VIRTUALSYSFLAG=TRUE 共网段生效后,DRPRI 生效值自动变为 0;网络中需至少一个非共网段接口 — 后果:DR 选举自动调整
- **配置 0 恢复 Cost 公式**(info,rule-0-00313):CONFIGCOST=0 表示不手动配,实际恢复公式计算(`带宽参考值/接口带宽`,参考值 100Mbit/s,<1 取 1) — 后果:误以为 Cost=0
- **IFNAME 长度 1~63**(info,rule-0-00313):字符串类型 — 后果:超长命令拒收
- **PROCID 范围 1~4294967295**(info,rule-0-00313):整数类型 — 后果:超界命令拒收
- **权限要求**(info,rule-0-00313):需 G_1 管理员/G_2 操作员级别 — 后果:低权限用户命令拒收
- **自动化配置护栏**(warning,rule-0-00313):PROCID/AREAID/TIMERDEADINTV/TIMERHELLOINTV/CONFIGCOST 参与自动化配置,下发前须用 `DSP OPSASSISTSTATE（★R1.5 atom 待补）` 查询 autoscaling_autoconfig.py 状态为"准备好"或不存在 — 后果:自动化配置过程中下发最终生效配置不可预期
