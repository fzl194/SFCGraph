# 配置L3VPN实例地址族（ADD VPNINSTAF）

> 命令静态知识见 [ADD VPNINSTAF](command/UDG/20.15.2/ADD-VPNINSTAF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD VPNINSTAF 给已创建的 L3VPN 实例使能地址族。VRFNAME 引用上游 [配置L3VPN实例](task/UDG/20.15.2/0-00071.md)；AFTYPE 随 IP 版本分叉（IPv4=ipv4uni、IPv6=ipv6uni），由 DP 0-00017 驱动；VRFRD 仅 IPv4 场景配（100:1），IPv6 场景不配。高级参数（路由策略/标签/FRR 等 8 项）属跨 VRF 路由交叉/隧道策略进阶场景，6 个特性数据规划表均未引用，按通用门槛跳过（演进债）。

**两种 IP 版本范式（从证据数据规划表归纳）**

| 场景 | VRFNAME | AFTYPE | VRFRD | 典型行 |
|---|---|---|---|---|
| IPv4 地址分配 / ePDG IPv4 / IPsec IPv4（GWFD-010104/010105/010155 IPv4、IPFD-015004 IPv4 组网） | vpn1 / vpn-s2b / vrf1 | ipv4uni | 100:1 | `ADD VPNINSTAF:VRFNAME="vpn1",AFTYPE=ipv4uni,VRFRD=100:1;` |
| IPv6 承载 / N6 Gi SGi / IPsec IPv6（GWFD-020401/020402、GWFD-010155 OSPFv3、IPFD-015004 IPv6 组网） | VPN_Internet / VPN_CN / vpn-paif / vrf1 | ipv6uni | 不配 | `ADD VPNINSTAF:VRFNAME="vpn-paif",AFTYPE=ipv6uni;` |

**关键规则**
- **必须先有 L3VPNINST**：VRFNAME 指向的 VPN 实例必须已通过 ADD L3VPNINST 创建（VRFNAME binding 为 reference→0-00071 保证引用一致）。
- **AFTYPE 取值由 DP 0-00017 决定**：option ip-version-v4 → ipv4uni；option ip-version-v6 → ipv6uni。
- **VRFRD conditional**：truth=optional，IPv4 场景配 100:1，IPv6 场景不用。
- **不能给系统保留实例** `__mpp_vpn_inner__`、`__mpp_vpn_inner_server__` 添加地址族。
- 单 L3VPN 实例可配 ipv4uni/ipv6uni 双栈（故规格高于 L3VPNINST 的 4096）。

## 约束

- **rule 0-00164**（capacity）系统最大记录数 16378 — 后果：超规格新增失败
- **rule 0-00165**（dependency/critical）VRFNAME 指向的 VPN 实例必须已通过 ADD L3VPNINST 创建；不能给系统保留实例 `__mpp_vpn_inner__`、`__mpp_vpn_inner_server__` 添加地址族 — 后果：地址族新增失败（实例不存在）/ 保留实例被污染
