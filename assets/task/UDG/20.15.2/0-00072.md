# 配置L3VPN实例地址族（ADD VPNINSTAF）

> 命令静态知识见 [ADD VPNINSTAF](command/UDG/20.15.2/ADD-VPNINSTAF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD VPNINSTAF 是 VPN 体系的"地址族使能层"——给已创建的 L3VPN 实例使能地址族，是后续路由协议/OSPF/静态路由/IPSec 等命令在指定 VPN 下运行的**前置开关**。本命令的核心配置空间有 5 个维度：**地址族类型**（AFTYPE 二选一）、**路由标识**（VRFRD 仅 IPv4 配）、**路由策略**（引入/发布策略，跨 VRF 路由交叉场景）、**标签分配**（VRFLABELMODE × VRFLABEL）、**隧道策略与 FRR**（TNLPOLICYNAME × FRRENABLE）。6 个特性命中覆盖 010104/010105/010155/020401/020402/IPFD-015004。

### 配置维度 1：地址族类型（参数 AFTYPE，必选）

| 取值 | 作用 | 配套参数 | 典型场景 |
|---|---|---|---|
| **ipv4uni** | IPv4 单播地址族 | VRFRD=100:1（必选） | 010104/010105 地址分配、010155 ePDG IPv4、IPFD-015004 IPv4 IPSec |
| **ipv6uni** | IPv6 单播地址族 | VRFRD 不配 | 020401 IPv6 承载、020402 N6/Gi/SGi、010155 OSPFv3、IPFD-015004 IPv6 IPSec |

### 配置维度 2：路由标识（参数 VRFRD，条件可选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **100:1**（示例） | VPN 路由标识符（RD）；不同 VPN 使用相同网段时 BGP 根据 VRFRD 区分 | IPv4 场景（010105/010155） |
| **不配** | — | IPv6 场景 |

### 配置维度 3：路由策略（参数 IMPOLICYNAME/EXPOLICYNAME/LOCALCROSSNHPMOD，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **IMPOLICYNAME** | 字符串 1~200 | 路由交叉入该 VPN 时的入口策略名 |
| **EXPOLICYNAME** | 字符串 1~200 | 路由发布时的出口策略名 |
| **EXPLYADDERTFIRST** | TRUE / FALSE | 路由上送时是否在过策略前添加 ERT |
| **LOCALCROSSNHPMOD** | TRUE / FALSE | 是否使能本地交叉修改下一跳功能 |

### 配置维度 4：标签分配（参数 VRFLABELMODE/VRFLABEL，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **VRFLABELMODE** | perRoute / perInstance / perInstanceStatic | VPN 实例标签分配模式 |
| **VRFLABEL** | 整数（VRFLABELMODE=perInstanceStatic 时条件可选） | VPN 实例静态标签值 |

### 配置维度 5：隧道策略与 FRR（参数 TNLPOLICYNAME/FRRENABLE，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **TNLPOLICYNAME** | 字符串 1~39 | 隧道策略名 |
| **FRRENABLE** | TRUE / FALSE | 是否允许 FRR 使能 |

| 参数 | 必选性 | 取值 | 作用 |
|---|---|---|---|
| `VRFNAME` | 必选 | 字符串 1~31 | 引用 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST |
| `AFTYPE` | 必选 | ipv4uni/ipv6uni | 地址族类型 |
| `VRFRD` | IPv4 条件必选 | 100:1 | 路由标识符（IPv6 不用） |

**典型脚本（010105 用户面地址分配，IPv4）**：
```
ADD VPNINSTAF:VRFNAME="vpn1",AFTYPE=ipv4uni,VRFRD=100:1;
```

**典型脚本（020401 IPv6 承载，IPv6 不配 VRFRD）**：
```
ADD VPNINSTAF:VRFNAME="VPN_Internet",AFTYPE=ipv6uni;
```

**典型脚本（010155 ePDG 双 VPN，IPv4+IPv6）**：
```
ADD VPNINSTAF:VRFNAME="vpn-s2b",AFTYPE=ipv4uni,VRFRD=100:1;
ADD VPNINSTAF:VRFNAME="vpn-paif",AFTYPE=ipv6uni;
```

**典型脚本（IPFD-015004 IPSec 双栈）**：
```
ADD VPNINSTAF:VRFNAME="vrf1", AFTYPE=ipv6uni;
ADD VPNINSTAF:VRFNAME="vrf1", AFTYPE=ipv4uni;
```

**步骤位置**：在所属 feature 编排中位于"步骤 2：使能地址族"——前置 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST（VRFNAME 来源）；后续接 [0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE / [0-00063](task/UDG/20.15.2/0-00063.md) ADD SRROUTE6 静态路由或 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 动态路由。

## 决策点：地址族类型选择（DP 0-00072）

问：ADD VPNINSTAF 的 AFTYPE 取 ipv4uni 还是 ipv6uni？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| ipv4uni | VRFRD=100:1（必选） | 010104/010105 地址分配、010155 ePDG IPv4、IPFD-015004 IPv4 IPSec |
| ipv6uni | VRFRD 不配 | 020401 IPv6 承载、020402 N6/Gi/SGi、010155 OSPFv3、IPFD-015004 IPv6 IPSec |

> 单 L3VPN 实例可同时配 ipv4uni/ipv6uni 双栈（故规格 16378 高于 L3VPNINST 的 4096）。

## 约束

- **最大记录数 16378**（critical，rule-0-00164）：系统最大记录数 16378（单 L3VPN 可配双栈） — 后果：超规格新增失败
- **VRFNAME 前置依赖**（critical，rule-0-00165）：VRFNAME 指向的 VPN 实例必须已通过 [0-00071](task/UDG/20.15.2/0-00071.md) ADD L3VPNINST 创建 — 后果：实例不存在→新增失败
- **系统保留实例保护**（critical，rule-0-00165）：不能给系统保留实例 `__mpp_vpn_inner__`、`__mpp_vpn_inner_server__` 添加地址族 — 后果：保留实例被污染
- **VRFRD 仅 IPv4 配**（warning，隐含）：VRFRD 在 IPv6 场景不配 — 后果：IPv6 误配 VRFRD 命令拒收
