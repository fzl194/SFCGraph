# 配置三四层过滤器-IPv6（ADD FILTERIPV6）

> 命令静态知识见 [ADD FILTERIPV6](command/UDG/20.15.2/ADD-FILTERIPV6.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD FILTERIPV6 配 IPv6 三四层过滤条件。与 [ADD FILTER](task/UDG/20.15.2/0-00004.md) 平行，多一个"服务器 IPv6 组"（IMS 语音场景必配，与对端协商）。

**2 种匹配范式（从证据数据规划表归纳）**

| 范式 | 典型场景 | 典型行 |
|---|---|---|
| any 通配 | any-to-any 基础业务 | `ADD FILTERIPV6:FILTERNAME="L34_any_ipv6",L34PROTTYPEV6=STRING,L34PROTOCOL=ANY,MSSTARTPORT=0,MSENDPORT=65535,SVRSTARTPORT=0,SVRENDPORT=65535;` |
| 服务器 IPv6（IMS 语音） | IMS 语音业务 | `ADD FILTERIPV6:FILTERNAME="ipv6_ims",L34PROTTYPEV6=STRING,L34PROTOCOL=ANY,SVRIPMODE=IP,SVRIPV6="fc00:8028:5002:10D::181",SVRIPV6MASKTYPE=LENGTHTYPE,SVRIPV6MASKLEN=128,MSSTARTPORT=0,MSENDPORT=65535,SVRSTARTPORT=0,SVRENDPORT=65535;` |

**服务器 IPv6 组**：SVRIPMODE=IP + SVRIPV6 + SVRIPV6MASKTYPE=LENGTHTYPE + SVRIPV6MASKLEN=128，**始终与对端协商（peer_planned）**（IMS 语音业务服务器、Bypass SBC-U 媒体面）。

**端口来源随场景分叉**：IMS 场景端口=与对端协商；any 场景端口=本端规划。

**L34PROTTYPEV6 固定 STRING**，L34PROTOCOL 选 ANY；L34PROTNUM（数字协议）实例化不用。

## 决策点：IPv6 过滤条件匹配模式（DP 0-00056）

问：过滤条件采用何种匹配模式？

| 选项 | 联动 | 来源 |
|---|---|---|
| any 通配 | 不配服务器 IPv6/端口，L34PROTOCOL=ANY | any 场景端口=本端规划 |
| 服务器 IPv6（IMS/与对端协商） | SVRIPMODE=IP + SVRIPV6/SVRIPV6MASKTYPE/SVRIPV6MASKLEN | 服务器 IPv6=peer；端口 IMS 场景=peer |

## 约束

- **rule-0-00121**（critical）配置后**必须** `SET REFRESHSRV`（REFRESHTYPE=USERPROFILE 或 ALL）刷新才生效；Filter+FilterIPv6 总记录数上限 150000，超 **98%** 上报 ALM-135602215、恢复阈值 **95%**（阈值可 MOD CFGTHRESHOLD 修改） — 后果：不刷新配置不生效 / 容量超限告警

> 注：IPv6 阈值（98%/95%）与 IPv4 的（85%）不同，容量规划需留意。
