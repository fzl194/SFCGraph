# 配置三四层过滤器-IPv6（ADD FILTERIPV6）

> 命令静态知识见 [ADD FILTERIPV6](command/UDG/20.15.2/ADD-FILTERIPV6.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD FILTERIPV6 是 [ADD FILTER](task/UDG/20.15.2/0-00004.md) 的 IPv6 对位命令：把"L3/L4 协议 + 端口 + IPv6 地址"用一条记录描述。多出"服务器 IPv6 组"（IMS 语音场景必配，与对端协商）。本命令的核心配置空间有 3 个维度：**过滤模式**（any 通配 vs 服务器 IPv6）、**协议输入方式**（STRING vs NUM）、**端口来源**（IMS 场景=peer、any 场景=本端）。

### 配置维度 1：过滤模式（决定配 IPv6/端口与否）

| 模式 | 必选字段 | 联动可选字段 | 典型场景 |
|---|---|---|---|
| **any 通配过滤** | FILTERNAME + L34PROTTYPEV6=STRING + L34PROTOCOL=ANY | 端口默认 0/65535 | any-to-any 基础业务（与 IPv4 FILTER 配对，如 L34_any/L34_any_ipv6） |
| **服务器 IPv6 过滤（IMS 语音）** | FILTERNAME + L34PROTTYPEV6=STRING + L34PROTOCOL=ANY + SVRIPMODE=IP + SVRIPV6 + SVRIPV6MASKTYPE=LENGTHTYPE + SVRIPV6MASKLEN=128 | 端口默认 0/65535 | IMS 语音业务（服务器 IPv6 与对端协商） |

### 配置维度 2：协议输入方式（参数 L34PROTTYPEV6，必选）

| 取值 | 协议表示 | 配套参数 | 备注 |
|---|---|---|---|
| **STRING** | 协议名字符串 | L34PROTOCOL（ANY/TCP/UDP 等） | 默认且高频（8 特性全部用 STRING） |
| **NUM** | 协议号数字（0~255） | L34PROTNUM（数字） | 冷门，无特性使用 |

### 配置维度 3：端口来源（参数 MSSTARTPORT/MSENDPORT/SVRSTARTPORT/SVRENDPORT）

四端口参数与 ADD FILTER 同源（0~65535 整数），但**来源按场景分叉**：

| 场景 | 端口来源 | 说明 |
|---|---|---|
| **IMS 语音（ipv6_ims）** | 与对端协商（peer_planned） | 端口默认 0/65535 |
| **any 基础业务（L34_any_ipv6）** | 本端规划 | 端口默认 0/65535 |
| **其他场景** | 跟随所属业务上下文 | 由 DP 0-00056 切换 |

**典型脚本（any 通配）**：
```
ADD FILTERIPV6:FILTERNAME="L34_any_ipv6", L34PROTTYPEV6=STRING, L34PROTOCOL=ANY, MSSTARTPORT=0, MSENDPORT=65535, SVRSTARTPORT=0, SVRENDPORT=65535;
```

**典型脚本（IMS 语音）**：
```
ADD FILTERIPV6:FILTERNAME="ipv6_ims", L34PROTTYPEV6=STRING, L34PROTOCOL=ANY, SVRIPMODE=IP, MSSTARTPORT=0, MSENDPORT=65535, SVRSTARTPORT=0, SVRENDPORT=65535, SVRIPV6="fc00:8028:5002:10D::181", SVRIPV6MASKTYPE=LENGTHTYPE, SVRIPV6MASKLEN=128;
```

> 服务器 IPv6 组（SVRIPMODE/SVRIPV6/SVRIPV6MASKTYPE/SVRIPV6MASKLEN）在 8 特性中**始终与对端协商（peer_planned）**——IMS 语音业务服务器 IPv6、Bypass SBC-U 媒体面 IPv6。手机 IPv6 组（MSIPV6/MSIPV6MASKTYPE 等）冷门，无特性使用。

**步骤位置**：在所属 feature 编排中通常位于"步骤 3：配置三四层过滤条件"——与 ADD FILTER 并列，ADD FLOWFILTER 之前，ADD FLTBINDFLOWF 引用本命令产出的 FILTERNAME（与 IPv4 FILTER 共用同一个 FLOWFILTER）。

## 决策点：IPv6 过滤条件匹配模式（DP 0-00056）

问：ADD FILTERIPV6 的过滤条件采用何种匹配模式？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| any 通配（ANY） | 仅 FILTERNAME+L34PROTTYPEV6+L34PROTOCOL；端口=本端规划 | any-to-any 基础业务场景 |
| 服务器 IPv6（IMS/对端协商，SVRIPV6_PEER） | SVRIPMODE=IP+SVRIPV6/SVRIPV6MASKTYPE/SVRIPV6MASKLEN，端口 IMS=peer/any=本端 | IMS 语音业务（ipv6_ims）、Bypass SBC-U 媒体面 IPv6 |

默认 ANY（绝大多数基础业务场景）；IMS 语音特性必选 SVRIPV6_PEER。

## 约束

- **ADD FILTERIPV6 刷新生效与记录数上限**（critical，rule-0-00121）：执行后需 SET REFRESHSRV（REFRESHTYPE=USERPROFILE 或 ALL）刷新生效；Filter 与 FilterIPv6 总记录数上限 150000；超规格 **98%** 上报 ALM-135602215、恢复阈值 **95%**（阈值可 MOD CFGTHRESHOLD（★R1.5 atom 待补） 修改） — 后果：未刷新不生效 / 容量超限告警
- **FILTERNAME 全局唯一**（critical，隐含）：不同过滤器之间 FILTERNAME 不能重复 — 后果：重复新增失败
- **协议输入方式二选一**（warning，隐含）：L34PROTTYPEV6=STRING 时配 L34PROTOCOL；L34PROTTYPEV6=NUM 时配 L34PROTNUM；不能混用 — 后果：参数缺失则校验失败
- **IPv6 容量阈值与 IPv4 不同**（warning，隐含）：IPv4 阈值 85%、IPv6 阈值 98%/95%；容量规划需分别核算 — 后果：阈值混淆导致告警/超限误判
