# 设置全局默认配额开关（SET UPDEFAULTQUOTA）

> 命令静态知识见 [SET UPDEFAULTQUOTA](command/UDG/20.15.2/SET-UPDEFAULTQUOTA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

SET UPDEFAULTQUOTA 是在线计费（Gy/N40 ONLINE 模式）的可选容灾开关——决定"配额申请期间是否用默认配额放通首包"。仅在 domain DP 选 ONLINE 时配置。本命令的核心配置空间有 4 个维度：**总开关**（DFTQTSWITCH 决定本功能使能与否）、**子开关**（DFTQTNEWSER/DFTQTOTHER 决定何时放通）、**按用户类型**（USERTYPE 决定按用户分类各配一条）、**互斥约束**（与 [SET UPGLBCHGPARA](0-00017.md) 的 CREDITPOOLSW 不能同时使能）。

### 配置维度 1：总开关（参数 DFTQTSWITCH）

| 取值 | 作用 | 后续子开关配置 |
|---|---|---|
| `ENABLE` | 使能 DefaultQuota——申请配额期间可用预设默认配额放通首包 | DFTQTNEWSER + DFTQTOTHER **required** |
| `DISABLE`（默认） | 关闭 DefaultQuota——申请配额期间报文按 [SET UPGLBCHGPARA](0-00017.md) 决定缓存或丢弃 | 不配 |

### 配置维度 2：子开关（参数 DFTQTNEWSER / DFTQTOTHER，前提 DFTQTSWITCH=ENABLE）

| 参数 | 语义 | 取值 | 行为 |
|---|---|---|---|
| `DFTQTNEWSER` | 新业务首包申请配额期间 | `ENABLE` | 用默认配额放通首包 |
| | | `DISABLE` | 不放通 |
| `DFTQTOTHER` | 非新业务场景（配额耗尽后续申请） | `ENABLE` | 用默认配额放通 |
| | | `DISABLE` | 不放通 |

> 两个子开关独立——可只开 `DFTQTNEWSER`、可全开、也可全关（**前提必须总开关为 ENABLE 才需配置子开关**）。

### 配置维度 3：按用户类型（参数 USERTYPE，必填）

| 取值 | 含义 | 条数 |
|---|---|---|
| `HOME`（本地用户） | 本网用户 | 1 条 |
| `ROAMING`（漫游用户） | 漫游用户 | 1 条 |
| `VISITING`（拜访用户） | 拜访用户 | 1 条 |

> **最大记录数 3**——按 USERTYPE 每类一条完整配置。可组合——`HOME` 与 `ROAMING` 可同时配。

### 配置维度 4：互斥约束（与 [SET UPGLBCHGPARA](0-00017.md) 配合）

DefaultQuota 功能与 `SET UPGLBCHGPARA: CREDITPOOLSW=ENABLE` **不能同时使能**——两者都是"申请配额期间的容灾策略"：

| DefaultQuota（SET UPDEFAULTQUOTA） | Credit Pooling（SET UPGLBCHGPARA） | 可否同使能 |
|---|---|---|
| DFTQTSWITCH=ENABLE | CREDITPOOLSW=DISABLE | ✓ |
| DFTQTSWITCH=DISABLE | CREDITPOOLSW=ENABLE | ✓（推荐 Credit Pooling 时）|
| DFTQTSWITCH=ENABLE | CREDITPOOLSW=ENABLE | **✗ 冲突**——二选一 |
| DFTQTSWITCH=DISABLE | CREDITPOOLSW=DISABLE | ✓（按 SET UPGLBCHGPARA 的 QUOTAAPPACTTYPE 决定缓存/丢弃）|

**前置依赖（必做）**：使能 DefaultQuota 前必须用 [ADD URR](0-00001.md) 的 `DFTQUOTAVOLUME` / `DFTQUOTATIME`（默认配额大小）+ `SET SRVCOMMONPARA` 设置全局默认配额——否则开关使能后无可用配额。

**生效时机**：仅对执行后**新激活用户或发生承载更新的用户**生效——存量用户不回溯。

**步骤位置**：在所属 feature 编排中属于"在线计费全局参数"步骤——通常与 [SET UPGLBCHGPARA](0-00017.md) 顺序无关、按需择一使能。

## 决策点：默认配额使能开关取值（DP 0-00050）

问：DFTQTSWITCH 是否使能？

| 选项 | 子开关配置 | 行为 |
|---|---|---|
| `default_quota_on` | `DFTQTNEWSER` + `DFTQTOTHER` 升为 required | 申请配额期间用默认配额放通首包 |
| `default_quota_off`（默认） | 排除（子开关不配） | 默认配额关闭，申请配额期间报文按 [SET UPGLBCHGPARA](0-00017.md) 的缓存或丢弃策略 |

## 约束

- **最大记录数与生效时机**（warning，rule-0-00116）：最大记录数 3（按 USERTYPE 每类一条）；高危命令——修改开关影响用户业务访问时延（配额耗尽申请期间若关闭，报文阻塞）；仅对承载更新 / 新激活用户生效 — 后果：误改开关致时延变大 / 存量用户不回溯
- **DFTQTSWITCH 前置依赖**（warning，rule-0-00117）：`DFTQTSWITCH=ENABLE` 前必须先用 [ADD URR](0-00001.md) 的 `DFTQUOTAVOLUME` / `DFTQUOTATIME`（或同义 `DFTQUOTAPARA`）+ `SET SRVCOMMONPARA` 设置默认配额 — 后果：开关使能但无配额，新业务首包无法用默认配额
- **与 Credit Pooling 互斥**（critical，隐含）：DefaultQuota 与 [SET UPGLBCHGPARA](0-00017.md) 的 `CREDITPOOLSW` **不能同时使能** — 后果：互斥冲突导致容灾策略失效
