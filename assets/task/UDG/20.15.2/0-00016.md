# 设置全局默认配额开关（SET UPDEFAULTQUOTA）

> 命令静态知识见 [SET UPDEFAULTQUOTA](command/UDG/20.15.2/SET-UPDEFAULTQUOTA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是**在线计费的可选容灾开关**：当用户新业务触发申请配额期间，决定是否放通默认配额（避免首包阻塞）。仅在 domain DP 选"在线计费"时才配。

**配置范式（按用户漫游类型逐条配，最多 3 条）**

| 步骤 | 参数 | 取值约定 | 说明 |
|---|---|---|---|
| 1 | USERTYPE | 漫游类型枚举 | 按本网用户/漫游用户等分类，每类一条 |
| 2 | DFTQTSWITCH | ENABLE / DISABLE | 总开关。DISABLE 时下面两个子开关不配 |
| 3 | DFTQTNEWSER | ENABLE / DISABLE | 新业务首包申请配额期间是否用默认配额（前提 DFTQTSWITCH=ENABLE） |
| 3 | DFTQTOTHER | ENABLE / DISABLE | 非新业务场景（如配额耗尽后续申请）是否用默认配额 |

**前置依赖（必做）**：使能前先用 [ADD URR](command/UDG/20.15.2/ADD-URR.md) 配 `DFTQUOTAVOLUME`/`DFTQUOTATIME`（默认配额大小），并用 SET SRVCOMMONPARA 设置默认配额——否则开关使能后无可用配额，新业务首包仍阻塞。

**与 Credit Pooling 互斥**：本命令使能的 DefaultQuota 功能与 [SET UPGLBCHGPARA](command/UDG/20.15.2/SET-UPGLBCHGPARA.md) 的 CREDITPOOLSW 不能同时使能（见关联 atom [0-00017](task/UDG/20.15.2/0-00017.md)）。

**生效时机**：仅对执行后**新激活用户或发生承载更新的用户**生效，存量用户不回溯。

## 决策点：默认配额使能开关取值（DP 0-00050）

问：DFTQTSWITCH 是否使能？

| 选项 | DFTQTNEWSER / DFTQTOTHER | 行为 |
|---|---|---|
| 使能 ENABLE | 升为 required（两个子开关必配） | 申请配额期间用默认配额放通首包 |
| 不使能 DISABLE | 排除（不配） | 默认配额关闭，申请配额期间报文按 UPGLBCHGPARA 缓存或丢弃 |

## 约束

- **rule-0-00116**（warning）最大记录数 3（按 USERTYPE 每类一条）；高危命令——修改开关影响用户业务访问时延（配额耗尽申请期间若关闭，报文阻塞）；仅对承载更新/新激活用户生效 — 后果：误改开关致时延变大
- **rule-0-00117**（warning）DFTQTSWITCH=ENABLE 前置依赖已配默认配额大小（ADD/MOD URR 的 DFTQUOTAVOLUME/DFTQUOTATIME + SET SRVCOMMONPARA） — 后果：开关使能但无配额，新业务首包无法用默认配额
