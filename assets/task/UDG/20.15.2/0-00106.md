# 设置带宽管理生效范围（SET BANDWIDTHMNG）

> 命令静态知识见 [SET BANDWIDTHMNG](command/UDG/20.15.2/SET-BANDWIDTHMNG.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。属高危命令：RANGE 变更后其他带宽管理命令（BWMCONTROLLER/BWMSERVICE 等）须同步修改。

## 配置方法

SET BANDWIDTHMNG 是带宽管理全局参数开关，**单参数修改即整机生效**。**RANGE 可选/条件必选**（LOCAL/SYSTEM 决定整机/单处理单元粒度），**PERIOD/CIRMINTHRES/PIRMINTHRES/ADJUSTVALUE 仅当 RANGE=SYSTEM 时可选**，**BWMSUBSCRCTRL 与 GRPBWMSHAPSW 不受 RANGE 约束**。核心配置空间有 4 个维度：**生效范围**（RANGE）、**系统级调参**（PERIOD+CIRMINTHRES+PIRMINTHRES+ADJUSTVALUE）、**用户级策略**（BWMSUBSCRCTRL）、**Shaping 开关**（GRPBWMSHAPSW）。2 个特性命中（GWFD-110301 / GWFD-110311）覆盖两套典型场景：仅设 SYSTEM 范围 / SYSTEM+CIR/PIR 联合。

### 配置维度 1：生效范围（参数 RANGE，可选/条件必选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **LOCAL** | 单处理单元内生效；用户组级带宽和连接数仅作用于本处理单元 | 默认；单板/单 PU 自治场景 |
| **SYSTEM** | 整机生效；用户组级带宽和连接数为整机配置，跨处理单元动态统计与调整 | 整机带宽保护、跨 PU 资源协调（110301/110311 范式） |

> **高危变更点**：RANGE 由 LOCAL→SYSTEM 或反向切换时，已配置的 BWMCONTROLLER/BWMSERVICE 等须同步修改，否则资源调度失效。

### 配置维度 2：系统级调参（参数 PERIOD/CIRMINTHRES/PIRMINTHRES/ADJUSTVALUE，RANGE=SYSTEM 时可选）

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **PERIOD**（秒） | 1~60 | 带宽资源动态统计与调整周期；默认 5 |
| **CIRMINTHRES**（kbps） | 0~整机 PIR | 整机 CIR 最低保护值；系统据此计算各业务 CIR 下限 |
| **PIRMINTHRES**（kbps） | 0~整机 PIR | 整机 PIR 最低保护值；系统据此计算各业务 PIR 下限 |
| **ADJUSTVALUE** | 整数 | 整机带宽控制流量占比调整系数 |

> 110311 范式联合配置：CIRMINTHRES=300000 + PIRMINTHRES=500000（300M/500M 整机保护门限），可选配 BWMSUBSCRCTRL=PCC_RULE_IGNORE。

### 配置维度 3：用户级业务带宽控制选项（参数 BWMSUBSCRCTRL，可选）

| 取值 | 作用 |
|---|---|
| **PCC_RULE_USE** | 用户级业务带宽控制器使用 PCC Rule 下发的 QoS 参数（默认） |
| **PCC_RULE_IGNORE** | 用户级业务带宽控制器忽略 PCC Rule 下发的 QoS 参数 |

> 110311 在设 CIR/PIR 最低保护时常配 PCC_RULE_IGNORE，保证 PCC 动态 QoS 不冲击整机保护门限。

### 配置维度 4：Shaping 开关（参数 GRPBWMSHAPSW，可选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 不使能基于 Shaping 的组级带宽控制（默认） |
| **ENABLE** | 使能基于 Shaping 的组级带宽控制；联动 BWMCONTROLLER.SHAPINGSWITCH |

**典型脚本（GWFD-110311 整机分级带宽保护，CIR/PIR 保护 + PCC 忽略）**：
```
SET BANDWIDTHMNG:RANGE=SYSTEM,CIRMINTHRES=300000,PIRMINTHRES=500000,BWMSUBSCRCTRL=PCC_RULE_IGNORE;
```

**典型脚本（GWFD-110301 码率差异化，仅切 SYSTEM）**：
```
SET BANDWIDTHMNG:RANGE=SYSTEM;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：设置带宽管理全局参数"——前置 [0-00036](task/UDG/20.15.2/0-00036.md) ADD BWMCONTROLLER / [0-00034](task/UDG/20.15.2/0-00034.md) ADD BWMSERVICE 之前，决定 BWMCONTROLLER 的 RANGE 取值。

## 决策点：RANGE 粒度选择（DP 0-00106）

问：本端带宽控制需要整机协同还是单板自治？

| 选项 | RANGE | 配套参数 | 适用场景 |
|---|---|---|---|
| **单处理单元自治**（默认） | LOCAL | 不配 PERIOD/CIR/PIR | 单 PU 资源充裕、跨 PU 不需协同（场景少） |
| **整机分级保护**（110311） | SYSTEM | CIRMINTHRES=300000,PIRMINTHRES=500000,BWMSUBSCRCTRL=PCC_RULE_IGNORE | 整机 CIR/PIR 门限保护、防止单业务抢占 |
| **仅切 SYSTEM 范围**（110301） | SYSTEM | 不配 PERIOD/CIR/PIR | 仅需切换粒度，配合 BWMSERVICE 码率策略 |

> 2 个特性命中（110301/110311）；默认 LOCAL，跨 PU 协同场景必选 SYSTEM；切换 SYSTEM 属高危变更，后续 BWMCONTROLLER/BWMSERVICE 须同步调整。

## 约束

- **高危命令**（critical，rule-0-00106）：本命令属高危命令，RANGE 变化后其他带宽管理命令（BWMCONTROLLER/BWMSERVICE 等）须同步修改 — 后果：误切换 RANGE 导致资源调度失效/整机流量异常
- **系统级参数生效条件**（critical，rule-0-00106）：PERIOD/CIRMINTHRES/PIRMINTHRES/ADJUSTVALUE 仅当 RANGE=SYSTEM 时可选 — 后果：RANGE=LOCAL 时配 SYSTEM 参数命令拒收
- **整机保护门限方向**（warning，rule-0-00106）：CIRMINTHRES 与 PIRMINTHRES 数值越小越严格，保护性越强但业务可用带宽越少 — 后果：门限值过小导致业务可用带宽不足
- **PCC_RULE_IGNORE 影响**（warning，rule-0-00106）：BWMSUBSCRCTRL=PCC_RULE_IGNORE 时用户级带宽控制器忽略 PCC Rule 的 QoS 参数 — 后果：与 PCC 动态 QoS 协同失效
- **执行后立即生效**（warning，rule-0-00106）：本命令执行后立即生效 — 后果：误操作即刻影响整机带宽调度
- **GRPBWMSHAPSW 与 BWMCONTROLLER.SHAPINGSWITCH 联动**（info，rule-0-00106）：GRPBWMSHAPSW=ENABLE 时 BWMCONTROLLER 下 SHAPINGSWITCH 才生效 — 后果：GRPBWMSHAPSW 未开时 SHAPINGSWITCH 不生效
