# 配置黑名单规则（ADD BLACKLISTRULE）

> 命令静态知识见 [ADD BLACKLISTRULE](command/UDG/20.15.2/ADD-BLACKLISTRULE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD BLACKLISTRULE 与 ADD RULE 同属 Rule 对象（共享配置记录空间/MOD/RMV/LST），但**黑名单规则不执行任何策略**——命中的业务流跳过对应 POLICYTYPE 的动作。重定向族（用户Portal/智能重定向）典型用途：UDG 不支持 HTTPS 报文重定向，当重定向 URL 采用 HTTPS 且命中重定向规则时，配一条 POLICYTYPE=SMARTREDIRECT 的黑名单规则**放行重定向报文**，保证用户重定向后的业务可正常处理（activation §操作步骤4 说明，证据：[配置用户Portal](evidence/UDG/20.15.2/配置用户Portal_66620114.md)）。

**主场景范式（重定向 HTTPS 旁路）**

| 字段 | 取值样例 | 数据来源 | 说明 |
|---|---|---|---|
| RULENAME | TESTBLACKLISTRULE | 本端规划 | 规则名，1~63 字符，不支持空格；与策略类型共同唯一标识（不能与同名同 POLICYTYPE 的 RULE 重复） |
| POLICYTYPE | SMARTREDIRECT | 本端规划 | 黑名单规则旁路的策略类型；重定向族取 SMARTREDIRECT（WEBPROXY 同理） |
| FILTERINGMODE | FLOWFILTER | 本端规划 | FLOWFILTER（单流过滤器）/ FLOWFILTERGRP（流过滤器组） |
| FLOWFILTERNAME | TESTFLOWFILTER | 已配置数据中获取 | FILTERINGMODE=FLOWFILTER 时必选，引用 [0-00007](task/UDG/20.15.2/0-00007.md) FLOWFILTER |
| PRIORITY | 50 | 本端规划 | 全局优先级，仅对 PCC 用户生效；值越小优先级越高，建议按 10 倍数规划 |

**脚本样例（命令 wiki 实例：头增强黑名单；重定向族改 POLICYTYPE=SMARTREDIRECT）**
```
ADD BLACKLISTRULE:RULENAME="TESTBLACKLISTRULE",POLICYTYPE=HEADEN,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="TESTFLOWFILTER",PRIORITY=50;
```

**可选字段**：`FLWFLTRGRPNAME`（FILTERINGMODE=FLOWFILTERGRP 时引用 [0-00294](task/UDG/20.15.2/0-00294.md) 流过滤器组）、`TIMERANGENAME`（控制生效时间段，引用 [0-00287](task/UDG/20.15.2/0-00287.md)）、`RULESPECTYPE`（DEFAULT/SPECIFICATION_LIMITED）、`BLWORKERNAME`（POLICYTYPE=WORKER 时）。

**配置链路**：先配 FLOWFILTER（[0-00007](task/UDG/20.15.2/0-00007.md)）→ **ADD BLACKLISTRULE**（与重定向 RULE 并列，命中黑名单的业务流跳过重定向动作）。

## 决策点

| 选项 | 影响（参数/命令/联动） |
|---|---|
| POLICYTYPE=SMARTREDIRECT（重定向族） | 旁路智能重定向/Captive Portal 动作（HTTPS 重定向 URL 放行） |
| POLICYTYPE=WEBPROXY | 旁路 Web Proxy 动作 |
| POLICYTYPE=HEADEN | 旁路头增强动作（命令 wiki 实例） |
| FILTERINGMODE=FLOWFILTERGRP | 改用流过滤器组，FLWFLTRGRPNAME 引用 [0-00294](task/UDG/20.15.2/0-00294.md) |

## 约束

- **不支持 POLICYTYPE=PCC 的黑名单规则**（critical）：命令 wiki 明确不支持 PCC 策略类型黑名单 — 配置则拒收。
- **RULENAME+POLICYTYPE 与 RULE 不可重复**（critical）：BLACKLISTRULE 与 RULE 共享配置记录空间，规则名+策略类型共同唯一 — 重复则新增失败。
- **POLICYTYPE=IPREDIR/SRV_TRIGGER 不允许绑七层协议**（warning）：当策略类型为 IPREDIR 或 SRV_TRIGGER 时，不允许配置绑定了七层协议和协议组的流过滤器 — 配置则拒收。
- **立即生效**（info）：命令执行后立即生效（无需 SET REFRESHSRV）；若未配生效时间段则默认一直生效。
- **规格**（info）：RULESPECTYPE=DEFAULT 时最大 8000 条；SPECIFICATION_LIMITED 时最大 100000 条。
