# 配置黑名单规则（ADD BLACKLISTRULE）

> 命令静态知识见 [ADD BLACKLISTRULE](command/UDG/20.15.2/ADD-BLACKLISTRULE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

`ADD BLACKLISTRULE` 与 `ADD RULE` 同属 Rule 对象（共享配置记录空间/MOD/RMV/LST），但**黑名单规则不执行任何策略**——命中的业务流跳过对应 POLICYTYPE 的动作。**仅 PGW-U/UPF 适用**。重定向族（用户 Portal/智能重定向）典型用途：UDG 不支持 HTTPS 报文重定向，当重定向 URL 采用 HTTPS 且命中重定向规则时，配一条 POLICYTYPE=SMARTREDIRECT 的黑名单规则**放行重定向报文**。

### 配置维度 1：基础三件套（必选）
| 参数 | 取值 | 作用 | 约束 |
|---|---|---|---|
| RULENAME | 字符串 1~63 字符，无空格 | 规则名；与 POLICYTYPE 共同唯一标识 | 不能与同名同 POLICYTYPE 的 RULE 重复 |
| POLICYTYPE | 枚举 | 旁路的策略类型（不配置执行策略，只跳过动作） | 不支持 PCC |
| FILTERINGMODE | FLOWFILTER / FLOWFILTERGRP | 选单流过滤器或流过滤器组 | 必选 |

### 配置维度 2：策略类型（参数 POLICYTYPE）
| 取值 | 旁路动作 | 典型用途 |
|---|---|---|
| SMARTREDIRECT | 智能重定向/Captive Portal | **HTTPS 重定向 URL 放行**（UDG 不支持 HTTPS 重定向，需黑名单旁路） |
| WEBPROXY | Web Proxy | HTTPS 流量走 WebProxy 旁路 |
| HEADEN | 头增强 | 头增强黑名单（命令 wiki 实例） |
| IPREDIR | IP 重定向 | IP 重定向族（不允许绑七层协议） |
| SRV_TRIGGER | Service Trigger Radius | 业务触发 Radius（不允许绑七层协议） |
| BWM / ADC / QOS / REMARK_FPI / FIREWALL | 对应策略 | — |
| WORKER | 通用 Rule（必配 BLWORKERNAME） | worker 类型最多 5 种 |

### 配置维度 3：过滤对象（参数 FLOWFILTERNAME / FLWFLTRGRPNAME）
| 参数 | 配套 | 作用 |
|---|---|---|
| FLOWFILTERNAME | FILTERINGMODE=FLOWFILTER 时必选 | 引 `ADD FLOWFILTER`（0-00007） |
| FLWFLTRGRPNAME | FILTERINGMODE=FLOWFILTERGRP 时必选 | 引 `ADD FLOWFILTERGRP`（0-00294） |

### 配置维度 4：时间段（参数 TIMERANGENAME，可选）
| 取值 | 作用 |
|---|---|
| 引 `ADD TIMERANGE`（0-00287） | 控制生效时间；不配则默认一直生效 |

### 配置维度 5：规则规格类型（参数 RULESPECTYPE，可选）
| 取值 | 作用 | 规格上限 |
|---|---|---|
| DEFAULT | 默认（不配默认） | 8000 条 |
| SPECIFICATION_LIMITED | 规格受限 | 100000 条（配置后不允许修改） |

### 配置维度 6：优先级（参数 PRIORITY，可选）
| 取值 | 作用 |
|---|---|
| 整数 1~4294967295（默认 4294967295） | 全局优先级，仅 PCC 用户生效；值越小越高；建议按 10 倍数规划 |

### 配置维度 7：WORKER 子策略（参数 BLWORKERNAME）
| 取值 | 作用 |
|---|---|
| 字符串 1~31 | POLICYTYPE=WORKER 时必选；通用 Rule 子策略名称 |

**典型脚本**：

```
// 头增强黑名单
ADD BLACKLISTRULE:RULENAME="TESTBLACKLISTRULE",POLICYTYPE=HEADEN,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="TESTFLOWFILTER",PRIORITY=50;
```

**步骤位置**：先配 FLOWFILTER（0-00007）→ ADD BLACKLISTRULE（与重定向 RULE 并列，命中黑名单的业务流跳过重定向动作）。

## 决策点：旁路策略类型（DP 0-00293）

| 选项 | POLICYTYPE | 典型场景 |
|---|---|---|
| HTTPS 重定向旁路 | SMARTREDIRECT | UDG 不支持 HTTPS 报文重定向，放行 HTTPS 重定向 URL |
| WebProxy 旁路 | WEBPROXY | HTTPS 流量走 WebProxy 路径放行 |
| 头增强旁路 | HEADEN | 命令 wiki 实例；跳过头增强插入 |
| 通用 Rule 旁路 | WORKER | 通用子策略（worker 最多 5 种） |

## 约束

- **rule-0-00293-1**（critical）：不支持 POLICYTYPE=PCC 的黑名单规则 — 配置则拒收
- **rule-0-00293-2**（critical）：RULENAME+POLICYTYPE 与 RULE 不可重复（共享 Rule 对象配置记录空间）— 重复则新增失败
- **rule-0-00293-3**（critical）：POLICYTYPE=IPREDIR/SRV_TRIGGER 时不允许配置绑定了七层协议/协议组的流过滤器或流过滤器组 — 配置则拒收
- **rule-0-00293-4**（warning）：RULESPECTYPE=DEFAULT 时最大 8000 条；SPECIFICATION_LIMITED 时最大 100000 条；DEFAULT 类型的 UserProfile 只能绑 DEFAULT 类型的 Rule
- **rule-0-00293-5**（info）：命令执行后立即生效（无需 SET REFRESHSRV）；若未配生效时间段则默认一直生效
- **rule-0-00293-6**（info）：当流过滤器绑定了除 DNS 外的其他七层协议时不能使用 WebProxy 策略
