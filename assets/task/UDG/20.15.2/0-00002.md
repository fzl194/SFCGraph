# 配置URR组（ADD URRGROUP）

> 命令静态知识见 [ADD URRGROUP](command/UDG/20.15.2/ADD-URRGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

URRGROUP 把成对的离线+在线 URR 聚合成"费率组"，1 号位绑离线 URR、2 号位绑在线 URR，上下行分别绑定（通常上下行绑同一条 URR）。一个业务一个 URRGROUP。

**位约定（从证据数据规划表归纳）**

| 参数位 | 含义 | 典型取值 | 必要性 |
|---|---|---|---|
| UPURRNAME1 / DOWNURRNAME1 | 1 号位（离线 URR） | `URR_URL_offline` | required（基础位，所有特性都配） |
| UPURRNAME2 / DOWNURRNAME2 | 2 号位（在线 URR） | `URR_URL_online` | conditional（融合/混合计费配，纯单一计费模式不配） |
| UPURRNAME3 / DOWNURRNAME3 | 3 号位 | — | 实例化时不配（冷门位） |

**典型行（融合计费场景，离线+在线共存）**
```
ADD URRGROUP:URRGROUPNAME="UrrGp_URL",UPURRNAME1="URR_URL_offline",DOWNURRNAME1="URR_URL_offline",UPURRNAME2="URR_URL_online",DOWNURRNAME2="URR_URL_online";
```

**单一计费模式（纯在线/纯离线）**：只配 1 号位。
```
ADD URRGROUP:URRGROUPNAME="urrgroup001",UPURRNAME1="onlineURR",DOWNURRNAME1="onlineURR";
```

**URRGROUPNAME 命名**：按业务语义命名（`UrrGp_URL` / `UrrGp_IMS` / `UrrGp_any` / `UrrGp_abnormal`），与 PCCPOLICYGRP 引用对齐。

> UPURRNAME1/2/3 只是编号，无优先级语义——同号位的上下行通常绑同一条 URR。

## 约束

- **rule-0-00112**（warning）最大记录数 40000，超 90% 上报 ALM-135602215（阈值可 MOD CFGTHRESHOLD 修改）；每个组支持在线+离线混合计费 — 后果：超规格新增失败
- **隐含铁律**：URRGROUP 下必须同时有在线 URR 和离线 URR，只配一种且与用户实际计费模式不匹配时无法计费。**唯一例外**：SMF 融合计费且 `SET USRPROFCHARGE`/`SET APNCHARGECTRL`/`ADD CHARGEMETHOD` 的 `RGAPPLIED=DEFAULT` 时，URRGROUP 下**不能**同时配离线和在线 URR（会冲突）。
