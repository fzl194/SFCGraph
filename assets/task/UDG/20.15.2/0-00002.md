# 配置URR组（ADD URRGROUP）

> 命令静态知识见 [ADD URRGROUP](command/UDG/20.15.2/ADD-URRGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

URRGROUP 是计费三件套的"费率组聚类层"：把已经配好的 URR（按业务 × 计费方式成对出现）按上行/下行分别绑定到 1~3 号位，形成一个可被 PCCPOLICYGRP 引用的"费率组"。本命令的核心配置空间有 3 个维度：**位号约定**（UP/DOWNURRNAME1~3，3 个编号槽）、**组装方式**（纯单一计费 vs 混合计费，决定配几位）、**命名约定**（URRGROUPNAME 跨命令引用锚点）。一个业务一个 URRGROUP。

### 配置维度 1：位号约定（参数 UPURRNAME1~3 / DOWNURRNAME1~3）

URRGROUP 提供 **3 组上下行位号**，每组独立绑定一条 URR。位号仅是编号槽，**无优先级语义**：

| 位号 | 通用语义 | 典型绑定 | 必要性 |
|---|---|---|---|
| `UPURRNAME1` / `DOWNURRNAME1` | 1 号位（融合场景下为离线 URR） | `URR_URL_offline` 等 | 基础位，必选 |
| `UPURRNAME2` / `DOWNURRNAME2` | 2 号位（融合场景下为在线 URR） | `URR_URL_online` 等 | 条件可选（混合计费配，纯单一计费不配） |
| `UPURRNAME3` / `DOWNURRNAME3` | 3 号位 | — | 实例化不配（冷门位） |

> **同号位的上下行通常绑同一条 URR**——计费是按业务计费，不区分上下行不同费率。命名约束：UPURRNAME1~3 / DOWNURRNAME1~3 必须引用已配的 URRNAME。

### 配置维度 2：组装方式（决定配几位号）

按业务的"计费模式组合"决定 URRGROUP 内绑定几位 URR：

| 组装方式 | 位号配置 | 适用场景 |
|---|---|---|
| **纯单一计费** | 仅配 1 号位 | 纯在线（Gy）或纯离线（Ga）特性，如基于业务时长的计费、事件计费、纯离线 |
| **混合计费**（离线 + 在线） | 1 号位绑离线 URR + 2 号位绑在线 URR | 融合计费、内容计费——离线兜底 + 在线重定向 |
| **复杂多 URR 组** | 1 号位离线 + 2 号位在线，3 号位空闲 | 预留扩展位——实例化通常不配 |

**典型脚本（纯单一计费，纯离线场景）**：
```
ADD URRGROUP:URRGROUPNAME="urrgroup1",UPURRNAME1="offlineURR",DOWNURRNAME1="offlineURR";
```

**典型脚本（纯单一计费，纯在线场景）**：
```
ADD URRGROUP:URRGROUPNAME="urrgroup001",UPURRNAME1="onlineURR",DOWNURRNAME1="onlineURR";
```

**典型脚本（混合计费，内容计费场景）**：
```
ADD URRGROUP:URRGROUPNAME="UrrGp_URL",UPURRNAME1="URR_URL_offline",DOWNURRNAME1="URR_URL_offline",UPURRNAME2="URR_URL_online",DOWNURRNAME2="URR_URL_online";
```

### 配置维度 3：命名约定（参数 URRGROUPNAME）

URRGROUPNAME 是 PCCPOLICYGRP.URRGROUPNAME 的引用锚点，**命名必须与下游 PCCPOLICYGRP 一致**：

| 命名风格 | 典型取值 | 适用场景 |
|---|---|---|
| 按业务语义 | `UrrGp_URL` / `UrrGp_IMS` / `UrrGp_any` / `UrrGp_abnormal` | 内容计费（按业务分组的费率组） |
| 通用默认 | `urrgroup1` / `urrgroup001` / `urrgp_test` | 纯在线 / 纯离线 / 调试用单一费率组 |

> PCCPOLICYGRP.URRGROUPNAME 引用本命令产出的 URRGROUPNAME——必须在 PCCPOLICYGRP 配置前完成。

**步骤位置**：在所属 feature 编排中位于"步骤 2：URR 组绑定"——紧随 ADD URR 之后，PCCPOLICYGRP 之前。

## 约束

- **ADD URRGROUP 最大记录数规格**（warning，rule-0-00112）：单板最大 40000；超 90% 上报 ALM-135602215（阈值可 `MOD CFGTHRESHOLD` 修改）；每个组支持在线+离线混合计费 — 后果：超规格新增失败
- **隐含铁律：URRGROUP 组装与用户计费模式匹配**（critical，隐含）：URRGROUP 下必须同时有在线 URR 和离线 URR，只配一种且与用户实际计费模式不匹配时无法计费；唯一例外——SMF 融合计费且 `SET USRPROFCHARGE` / `SET APNCHARGECTRL` / `ADD CHARGEMETHOD` 的 `RGAPPLIED=DEFAULT` 时，URRGROUP 下**不能**同时配离线和在线 URR（会冲突） — 后果：计费丢失 / 模式冲突
- **URRGROUPNAME 全网唯一**（critical，隐含）：同 nf@version 下 URRGROUPNAME 不重复 — 后果：PCCPOLICYGRP 引用错位
