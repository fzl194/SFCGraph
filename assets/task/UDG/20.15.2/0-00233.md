# 配置IPSQM调整参数（SET IPSQMADJUST）

> 命令静态知识见 [SET IPSQMADJUST](command/UDG/20.15.2/SET-IPSQMADJUST.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110941（基于基站粒度的 IPSQM）**可选增强**第 4 步——为整形资源调整的业务处理节点（POD）配置最低保护带宽及突发尺寸，区分高/低带宽域，保障低传输能力下的最低保证带宽。适用 NF：SGW-U、UPF。缺省时用系统初始记录，本步骤按数据规划表覆盖取值。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| THRESHOLD | 阈值（兆比特/秒） | `30` | 本端规划，整数 1~50 Mbps（**必填**） |
| LOWBANDWIDTH | 低带宽域的最低保护值（千比特/秒） | `300` | 全网规划，整数 100~100000 kbps（可选） |
| LOWCBS | 低带宽域的突发尺寸（字节） | `2000` | 全网规划，整数 500~100000000 字节（可选） |
| HIGHBANDWIDTH | 高带宽域的最低保护值（千比特/秒） | `1000` | 全网规划，整数 100~100000 kbps（可选） |
| HIGHCBS | 高带宽域的突发尺寸（字节） | `10000` | 全网规划，整数 500~100000000 字节（可选） |

**操作步骤定位**：在核心整形 [配置IPSQM静态整形带宽](task/UDG/20.15.2/0-00231.md)（ADD IPSQMSHAPING）、可选 [配置IPSQM整形队列深度](task/UDG/20.15.2/0-00232.md)（SET IPSQMQDEPTH）之后执行，属 IPSQM 增强步骤。后置依赖由 feature-rule 0-00082（IPSQM 增强须先核心整形）表达。

## 约束

- 命令最大记录数 **1**（命令 notes 强约束），全局唯一实例，超出被拒。
- 配置过小会导致用户丢包过多（命令 notes 风险提示），需结合现网带宽规划取值。
- 命令执行后**立即生效**。
- THRESHOLD 整数 1~50 Mbps（必填）；LOWBANDWIDTH/HIGHBANDWIDTH 整数 100~100000 kbps；LOWCBS/HIGHCBS 整数 500~100000000 字节。
