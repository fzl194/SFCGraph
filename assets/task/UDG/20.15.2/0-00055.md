# 配置冗余备份重定向IP（ADD REDUNDRDTIP）

> 命令静态知识见 [ADD REDUNDRDTIP](command/UDG/20.15.2/ADD-REDUNDRDTIP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。用于"静态地址用户路由冗余"场景（GWFD-010107）。

## 配置方法（配置方法字典）

ADD REDUNDRDTIP 是"冗余备份隧道虚拟重定向 IP 登记"命令：主 UPF 故障时，备 UPF 将下行业务流通过 GRE Tunnel 重定向到该虚拟 IP。**最大记录数 2**（IPv4 一条 + IPv6 一条，或主备各一条）。核心配置空间有 2 个维度：**IP 类型**（IPVERSION 二选一门控）、**重定向地址**（IPV4ADDRESS/IPV6ADDRESS）。1 个特性命中（GWFD-010107）演示 IPv4 + IPv6 各一条范式（主备一致）。

### 配置维度 1：IP 类型（参数 IPVERSION，必选）

IPVERSION 是门控开关，决定后续地址参数走 IPv4 链路还是 IPv6 链路：

| 取值 | 联动必选参数 | 适用场景 |
|---|---|---|
| **IPv4** | `IPV4ADDRESS` | 静态地址用户路由冗余 IPv4 重定向（010107 默认范式） |
| **IPv6** | `IPV6ADDRESS` | 静态地址用户路由冗余 IPv6 重定向（双栈部署） |

> 重定向 IP **全局规划**，仅内部重定向转发、**不对外发布**，只要不与其他路由冲突即可——主备 UDG 可规划相同 IP，也可规划不同 IP。

### 配置维度 2：重定向地址（参数 IPV4ADDRESS 或 IPV6ADDRESS，条件必选）

**IPv4 路径**（IPVERSION=IPv4 时）：

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **IPV4ADDRESS** | 合法 IPv4（点分十进制 "X.X.X.X"） | 将业务流重定向到 GRE Tunnel 的虚拟 IPv4 地址 |

**IPv6 路径**（IPVERSION=IPv6 时）：

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **IPV6ADDRESS** | 合法 IPv6（"X:X::X:X" 形式） | 将业务流重定向到 GRE Tunnel 的虚拟 IPv6 地址 |

> IPV4ADDRESS/IPV6ADDRESS 两组参数**互斥**：选 IPv4 时不能配 IPV6ADDRESS，反之亦然。

**典型脚本（010107 主备一致，IPv4 + IPv6 各一条）**：
```
ADD REDUNDRDTIP: IPVERSION=IPv4, IPV4ADDRESS="192.168.0.1";
ADD REDUNDRDTIP: IPVERSION=IPv6, IPV6ADDRESS="fc00:3::1";
```

**典型脚本（单条 IPv4 精简）**：
```
ADD REDUNDRDTIP: IPVERSION=IPv4, IPV4ADDRESS="192.168.0.1";
```

> 命令 notes：适用 NF：PGW-U、UPF；**最大记录数 2**（主备各一条 / IP 版本各一条）；**执行后立即生效**；重定向 IP 必须有对应静态路由存在（[0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE / [0-00063](task/UDG/20.15.2/0-00063.md) ADD SRROUTE6 的 PREFIX 必须与重定向 IP 一致），否则主 UPF 恢复后在备上激活的用户下行业务不能发往备 UPF。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：冗余备份重定向 IP"——前置 [0-00059](task/UDG/20.15.2/0-00059.md) ADD GRETUNNEL（GRE 隧道就绪），配套 [0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE（静态路由 PREFIX 与重定向 IP 一致）。

## 决策点：IP 版本选择（DP 0-00055）

问：ADD REDUNDRDTIP 选哪个 IP 版本？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPv4 单地址 | IPVERSION=IPv4 + IPV4ADDRESS | 静态地址用户路由冗余 IPv4 场景（010107 默认范式） |
| IPv6 单地址 | IPVERSION=IPv6 + IPV6ADDRESS | 静态地址用户路由冗余 IPv6 场景（双栈部署） |
| IPv4 + IPv6 各一条 | 两条独立 ADD | 主备双栈（010107 演示：IPv4 一条 + IPv6 一条） |

> 1 个特性命中（010107）演示主备一致 IPv4+IPv6 双条范式；累积记录 ≤2；主备 UDG 可配相同 IP 或不同 IP。

## 约束

- **最大记录数 2**（warning，rule-0-00455）：主备各一条 / IP 版本各一条，累积不超过 2 — 后果：超 2 条新增命令拒收
- **执行后立即生效**（warning，rule-0-00455）：命令执行后立即生效，无需刷新 — 后果：误操作即刻生效
- **IPVERSION 门控联动参数**（critical，rule-0-00455）：IPVERSION=IPv4 时必须配 IPV4ADDRESS；IPVERSION=IPv6 时必须配 IPV6ADDRESS；两组互斥 — 后果：参数配错或缺失命令拒收
- **重定向 IP 路由必存**（critical，rule-0-00455）：配 IPV4ADDRESS/IPV6ADDRESS 时必须确保对应静态路由存在（[0-00062](task/UDG/20.15.2/0-00062.md) ADD SRROUTE / [0-00063](task/UDG/20.15.2/0-00063.md) ADD SRROUTE6 的 PREFIX 与重定向 IP 一致） — 后果：路由缺失致主 UPF 恢复后备上激活的用户下行业务不能发往备 UPF
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：其他 NF 不适用
- **删除配套 RMV REDUNDRDTIP**（info，隐含）：本命令只能增，删除需 RMV REDUNDRDTIP，修改需 MOD REDUNDRDTIP — 后果：误增需 RMV 删除
