# 配置冗余备份重定向IP（ADD REDUNDRDTIP）

> 命令静态知识见 [ADD REDUNDRDTIP](command/UDG/20.15.2/ADD-REDUNDRDTIP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

冗余备份虚拟重定向 IP 用于静态地址用户路由冗余场景（GWFD-010107）：主 UPF 故障时，备 UPF 将下行业务流通过 GRE Tunnel 重定向到该虚拟 IP。最大记录数 2（IPv4 一条 + IPv6 一条，或主备各一条）。重定向 IP 全局规划，仅内部重定向转发、不对外发布，只要不与其他路由冲突即可——主备 UDG 可规划相同 IP，也可规划不同 IP。

**参数规划表（取值样例来自 GWFD-010107，主备两套）**

| IPVERSION | 配对参数 | 样例值（主/备一致） | 规划口径 |
|---|---|---|---|
| IPv4 | IPV4ADDRESS | 192.168.0.1 | 全网规划 |
| IPv6 | IPV6ADDRESS | fc00:3::1 | 全网规划 |

**典型脚本（IPv4 + IPv6 各一条）**
```
ADD REDUNDRDTIP:IPVERSION=IPv4,IPV4ADDRESS="192.168.0.1";
ADD REDUNDRDTIP:IPVERSION=IPv6,IPV6ADDRESS="fc00:3::1";
```

**关键联动**
- 重定向 IP 必须有对应静态路由存在：[0-00061](task/UDG/20.15.2/0-00061.md)（ADD SRROUTE）/[0-00062](task/UDG/20.15.2/0-00062.md)（ADD SRROUTE6）的 PREFIX 必须与本命令的重定向 IP 一致，否则主 UPF 恢复后在备上激活的用户下行业务不能发往备 UPF。
- IPVERSION=IPv4 配 IPV4ADDRESS，=IPv6 配 IPV6ADDRESS（二选一）。

## 约束

- **rule-0-00329**（critical）最大记录数 2（主备各一条 / IP 版本各一条）；立即生效；配置 IPV4ADDRESS/IPV6ADDRESS 时需确保对应路由存在（ADD SRROUTE/SRROUTE6 的 PREFIX 与重定向 IP 一致） — 违反后果：重定向 IP 路由缺失致主备切换后下行业务不能发往备 UPF
