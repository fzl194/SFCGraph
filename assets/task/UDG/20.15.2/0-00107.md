# 设置APN操作系统级带宽管理开关（SET APNOSLELBWMSW）

> 命令静态知识见 [SET APNOSLELBWMSW](command/UDG/20.15.2/SET-APNOSLELBWMSW.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。属"修改"语义，对已存在的 APN 设置操作系统级带宽管理开关。

## 配置方法

SET APNOSLELBWMSW 是 APN 级操作系统级带宽管理开关命令。**APNNAME 必选**（指定要操作的 APN）；**ISENABLE 必选**（DISABLE/ENABLE 切换）。核心配置空间有 2 个维度：**APN 范围**（APNNAME）、**使能开关**（ISENABLE）。1 个特性命中（GWFD-110301）覆盖 APN 级操作系统级带宽管理使能场景。

### 配置维度 1：APN 范围（参数 APNNAME，必选）

| 取值 | 作用 |
|---|---|
| **已建 APN 名**（ref [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN） | 对该 APN 下所有用户生效操作系统级带宽管理 |

> APNNAME 必须是已通过 `ADD APN` 创建的 APN；不支持空格，1~63 字符。

### 配置维度 2：使能开关（参数 ISENABLE，必选）

| 取值 | 作用 |
|---|---|
| **DISABLE** | 不使能该 APN 的操作系统级带宽管理 |
| **ENABLE** | 使能该 APN 的操作系统级带宽管理（GWFD-110301 范式） |

> 命令语义为"修改"——仅对已存在的 APN 设置；如需新建 APN，使用 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN。

**典型脚本（GWFD-110301 码率差异化，APN 级 OS 级带宽管理使能）**：
```
SET APNOSLELBWMSW:APNNAME="apn-test",ISENABLE=ENABLE;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：设置 APN OS 级带宽管理开关"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 建好目标 APN；可选与 [0-00106](task/UDG/20.15.2/0-00106.md) SET BANDWIDTHMNG（RANGE=SYSTEM）配合。

## 决策点：APN OS 级带宽管理使能（DP 0-00107）

（本命令无独立决策分支，仅有 ISENABLE 枚举。）

| 选项 | APNNAME | ISENABLE | 适用场景 |
|---|---|---|---|
| **启用**（110301 默认） | ref APNNAME | ENABLE | GWFD-110301 码率差异化控制使能 APN OS 级带宽 |
| **禁用** | ref APNNAME | DISABLE | 关闭已使能的 APN OS 级带宽管理 |

## 约束

- **APNNAME 必须已建**（critical，rule-0-00107）：APNNAME 必须是已通过 `ADD APN` 创建的 APN — 后果：APN 不存在命令拒收
- **修改语义**（info，rule-0-00107）：本命令语义为"修改"，不创建/删除 APN；新建 APN 使用 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN — 后果：用于不存在的 APN 无效
- **执行后立即生效**（warning，rule-0-00107）：本命令执行后立即生效 — 后果：误操作即刻影响该 APN 下所有用户带宽调度
- **ISENABLE 必选**（critical，rule-0-00107）：ISENABLE 是必选参数，不配命令拒收 — 后果：缺 ISENABLE 命令拒收
