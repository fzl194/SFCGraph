# 增加VPN Target(路由交叉RT发布/引入)（ADD VPNTARGET）

> 命令静态知识见 [ADD VPNTARGET](command/UDG/20.15.2/ADD-VPNTARGET.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是 **GWFD-020423（MPLS OptionB 路由交叉）RT 收发核心**：通过 export/import 两类 RT 控制跨 VPN 路由发布与引入。关键是**配对**——VRF_networkA 发布 RT、VRF_networkB 引入同一 RT，B 路由表即复制 A 的 BGP 引入路由。VRFRTTYPE 由 VRF 角色决定（固定），VRFRTVALUE 必须**两边相同**（全网规划）。立即生效。

**配置范式（两个 VRF 各执行一次，RT 值相同才能交叉）**

| 参数 | VRF_networkA | VRF_networkB | 来源 | 说明 |
|---|---|---|---|---|
| VRFNAME | `VRF_networkA` | `VRF_networkB` | 引用 [0-00071](task/UDG/20.15.2/0-00071.md).VRFNAME | VPN 实例名 |
| AFTYPE | `ipv4uni` | `ipv4uni` | 本端规划（固定） | 地址族类型 |
| VRFRTTYPE | `export_extcommunity` | `import_extcommunity` | 固定取值（由 VRF 角色决定） | A 发布 RT / B 引入 RT |
| VRFRTVALUE | `1:1` | `1:1` | 全网规划 | **两边必须相同**，否则路由交叉失败 |

**任务示例**

```
ADD VPNTARGET: VRFNAME="VRF_networkA", AFTYPE=ipv4uni, VRFRTTYPE=export_extcommunity, VRFRTVALUE="1:1";
ADD VPNTARGET: VRFNAME="VRF_networkB", AFTYPE=ipv4uni, VRFRTTYPE=import_extcommunity, VRFRTVALUE="1:1";
```

**前置链**：ADD L3VPNINST(0-00071) → MOD VPNINSTAF(0-00145) 配 RD → ADD VPNTARGET(0-00146) 配 RT。

参数 4/4 全绑定，0 缺口。RT 匹配规则（A export = B import = 同 RT 值）属 feature 级语义，不投影独立 rule。
