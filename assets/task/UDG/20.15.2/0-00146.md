# 增加VPN Target(路由交叉RT发布/引入)（ADD VPNTARGET）

> 命令静态知识见 [ADD VPNTARGET](command/UDG/20.15.2/ADD-VPNTARGET.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD VPNTARGET 用于增加 VPN Target（RT），是 **GWFD-020423（MPLS OptionB 路由交叉）RT 收发核心**。通过 export/import 两类 RT 控制跨 VPN 路由发布与引入。**关键配对**：VRF_networkA 发布 RT、VRF_networkB 引入同一 RT，B 路由表即复制 A 的 BGP 引入路由。VRFRTTYPE 由 VRF 角色决定（固定），**VRFRTVALUE 必须两边相同**（全网规划）。立即生效。1 个特性命中（020423 路由交叉，对 2 个 VRF 各执行一次）。

### 配置维度 1：地址族（参数 AFTYPE，必选）

| 取值 | 作用 |
|---|---|
| **ipv4uni**（020423 固定） | IPv4 单播地址族；当前路由交叉唯一场景 |
| 其他 | 见命令真相表枚举（020423 未用） |

### 配置维度 2：RT 方向（参数 VRFRTTYPE，必选）

| 取值 | 作用 | 适用 VRF 角色 |
|---|---|---|
| **export_extcommunity** | 发布 RT 值：发送路由时携带该 RT | VRF_networkA（路由发布侧） |
| **import_extcommunity** | 引入 RT 值：仅接收携带该 RT 的路由 | VRF_networkB（路由引入侧） |
| both | 同时发布 + 引入 | 双向 RT（020423 未用） |

> VRFRTTYPE 由 VRF 在交叉中的角色决定：发布侧取 export，引入侧取 import。

### 配置维度 3：RT 值（参数 VRFRTVALUE，必选）

| 取值 | 约束 |
|---|---|
| **AS:NN 格式**（如 `1:1`） | **A 发布 RT 值 = B 引入 RT 值**（两边必须相同），否则路由交叉失败；全网规划 |

> RT 值是路由交叉匹配的键：A export = B import = 同 RT 值才交叉。

**任务示例（020423 范式，VRF_networkA/B 各执行一次）**：

```
ADD VPNTARGET: VRFNAME="VRF_networkA", AFTYPE=ipv4uni, VRFRTTYPE=export_extcommunity, VRFRTVALUE="1:1";
ADD VPNTARGET: VRFNAME="VRF_networkB", AFTYPE=ipv4uni, VRFRTTYPE=import_extcommunity, VRFRTVALUE="1:1";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：配 RT 发布/引入"——前置链：ADD L3VPNINST(0-00071) → MOD VPNINSTAF(0-00145) 配 RD → ADD VPNTARGET(本任务) 配 RT。

## 决策点：RT 配对（DP 0-00146）

| 选项 | VRF_networkA 配法 | VRF_networkB 配法 | 适用场景 |
|---|---|---|---|
| **opt-rt-cross-1to1（1:1 配对）** | export_extcommunity + VRFRTVALUE=1:1 | import_extcommunity + VRFRTVALUE=1:1 | 020423 范式（一对 RT） |
| **opt-rt-cross-multi** | export + 多个 VRFRTVALUE | import + 多个 VRFRTVALUE | 多 RT 交叉（020423 未用） |

## 约束

- **RT 两边必须相同**（critical，rule-0-00146）：A 发布 RT 值必须等于 B 引入 RT 值 — 后果：RT 不匹配则路由交叉失败
- **VRFRTTYPE 由 VRF 角色决定**（warning，rule-0-00146）：A 侧固定 export，B 侧固定 import — 后果：角色反置（export 侧放错 VRF）路由无法交叉
- **VRFNAME 前置必备**（critical，rule-0-00146）：VRFNAME 须已由 ADD L3VPNINST(0-00071) 建出 — 后果：VRF 未建命令拒收
- **RD 前置必备**（warning，rule-0-00146）：前置 MOD VPNINSTAF(0-00145) 须已为该 VRF 配 RD — 后果：未配 RD 时 VPNv4 路由无法标识
- **立即生效**（warning，rule-0-00146）：执行后立即生效 — 后果：误配 RT 即刻影响跨 VPN 路由收发
- **4/4 参数全绑定**（info，rule-0-00146）：VRFNAME/AFTYPE/VRFRTTYPE/VRFRTVALUE 全部必填，0 缺口
