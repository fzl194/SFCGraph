# HTTP智能重定向（GWFD-110284）

> 特性静态知识见 [GWFD-110284](feature/UDG/20.15.2/GWFD-110284.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性是**智能重定向子族（HTTP智能重定向/DNS纠错）的族范本**，DNS纠错（[2-00016](task/UDG/20.15.2/2-00016.md)）复用本族 compound [1-00015](task/UDG/20.15.2/1-00015.md)。

## 配置概览

HTTP 智能重定向根据 HTTP 服务器返回的错误码（如 `>400`）触发 URL 纠错，将用户访问的 HTTP 页面重定向到第三方 server。**核心机制：RULE 经 `POLICYTYPE=SMARTREDIRECT` 直接引用 SMTHTTPREDINAME（ADD SMARTHTTPREDIR 定义的动作对象），独立于 PCC/HEADEN 的第 3 种 POLICYTYPE，不经 PCCPOLICYGRP、不经 IPFarm**，SERVERURL 在动作命令直配。

动作命令 ADD SMARTHTTPREDIR 绑定扩展过滤器（URL/USERAGENT 组合匹配）+ 错误码范围（GT 400）+ 重定向携带信息（REQURLFLAG+IMSIFLAG，可携带原始请求 URL+IMSI）。重定向之后 URL 为第三方 server（activation 演示 `www.example.com`）。多场景差异由 DP 驱动（触发匹配/错误码操作/携带信息/规则下发方式/FILTERINGMODE）。

> **284 activation 无计费/无 USERPROFILE/RULEBINDING**：activation 采用 SMF 下发预定义规则方式，无 URR/URRGROUP/PCCPOLICYGRP、无 USERPROFILE、无 RULEBINDING（采用 PCRF/SMF 下发预定义规则方式）。配置流程忠实反映 activation，不编造计费/用户模板步骤。如需本地绑定规则，可复用 [1-00011](task/UDG/20.15.2/1-00011.md) 的 USERPROFILE+RULEBINDING。

## 决策点

**DP（特性级）：触发匹配 + 错误码操作 + 重定向携带信息 + 规则下发方式 + FILTERINGMODE**（影响扩展过滤器组合 + ADD ERRORCODE 参数 + ADD REDIRAPPENDINFO 字段 + ADD RULE 走法）。维度按 activation §数据规划表 + §操作步骤 全列。

| 场景 | 触发匹配 | 错误码操作 | 携带信息 | 规则下发方式 | FILTERINGMODE |
|---|---|---|---|---|---|
| **标准场景（activation 演示）** | URL+USERAGENT（2 扩展过滤器 AND 组合）| GT, ERRORCODESTART=400 | REQURLFLAG+IMSIFLAG | SMF 下发预定义规则（无 USERPROFILE/RULEBINDING）| FLOWFILTER |
| 仅 URL 触发 | URL（1 扩展过滤器）| GT, 400 | 任意 | 任意 | FLOWFILTER |
| 仅 USERAGENT 触发 | USERAGENT（1 扩展过滤器）| GT, 400 | 任意 | 任意 | FLOWFILTER |
| 错误码精确匹配变体 | 同上 | EQUAL, ERRORCODESTART=404 | 任意 | 任意 | FLOWFILTER |
| 错误码范围变体 | 同上 | GE/LE/LT, START+END | 任意 | 任意 | FLOWFILTER |
| 携带信息增强变体 | 同上 | GT, 400 | REQURL+MSISDN+IMSI+IMEI+MSIP（+IMSI AES256 加密）| 任意 | FLOWFILTER |
| 本地绑定规则变体 | 同上 | GT, 400 | 任意 | 本地 USERPROFILE+RULEBINDING（复用 [1-00011](task/UDG/20.15.2/1-00011.md) 全套）| FLOWFILTER |
| 流过滤器组变体 | 同上 | GT, 400 | 任意 | 任意 | FLOWFILTERGRP（+[0-00294](task/UDG/20.15.2/0-00294.md)）|

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5SHPR01` — 未开则 HTTP 智能重定向不生效。
- **强依赖 SA-Basic**（critical）：GWFD-110101（License LKV3G5SABS01）—— HTTP 报文解析基础，未开则无法识别报文。
- **协议依赖 SA-Web Browsing**（critical）：GWFD-110103（License LKV3G5SAWB01）—— HTTP/WAP2.0 解析必需。
- **POLICYTYPE=SMARTREDIRECT 触发条件**（critical）：RULE.POLICYTYPE=SMARTREDIRECT + POLICYNAME=SMTHTTPREDINAME（直接引用动作对象，PCC 策略组名称=NULL）— 指错则智能重定向不触发。
- **EXTFLTNAME 引用须已配置**（critical）：ADD SMARTHTTPREDIR 的 EXTFLTNAME1/2 须先经 ADD EXTENDEDFILTER 配置 — 后果：引用不存在拒收。
- **BINDERRCODENAME 引用须已配置**（critical）：ADD SMARTHTTPREDIR 的 BINDERRCODENAME 须先经 ADD ERRORCODE 配置 — 后果：引用不存在拒收。
- **动作对象须先于 RULE 配置**（critical）：SMTHTTPREDINAME 须先于 `RULE(POLICYTYPE=SMARTREDIRECT)` 配置 — 后果：RULE 引用失败。
- **仅 HTTP，不支持 HTTPS/HTTP2.0**（warning）：HTTPS 需其他特性（ADD TLSHEADEN 体系）；HTTP2.0 不支持 — 后果：对应协议报文不做智能重定向。
- **扩展过滤器可重复执行配多个**（info）：ADD EXTENDEDFILTER 重复执行增加多个不同过滤条件（activation §操作步骤 说明）。
- **规格**（info）：SMARTHTTPREDIR 整机 ≤50（明显小于 DNSOVERWRITING ≤2000 / REDIRAPPENDINFO ≤1000）；动作命令最多 5 组扩展过滤器。
