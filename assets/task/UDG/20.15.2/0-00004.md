# 配置三四层过滤器-IPv4（ADD FILTER）

> 命令静态知识见 [ADD FILTER](command/UDG/20.15.2/ADD-FILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD FILTER 是过滤链 backbone 的"三四层 L34 原子"：把"L3/L4 协议 + 端口 + IPv4 地址"用一条记录描述，作为 FILTER+FILTERIPV6+L7FILTER 集合的最底层，供 ADD FLTBINDFLOWF 引用。本命令的核心配置空间有 4 个维度：**过滤模式**（决定配 IP/端口与否）、**协议输入方式**（STRING vs NUM 二选一）、**端口范围**（默认 0~65535 全匹配，按场景收紧）、**IPv4 地址过滤**（服务器 IP 与手机 IP，可选）。

### 配置维度 1：过滤模式（决定配 IP/端口与否）

按业务"匹配粒度"决定本条 FILTER 的内容密度：

| 模式 | 必选字段 | 联动可选字段 | 典型场景 |
|---|---|---|---|
| **any 通配过滤** | FILTERNAME + L34PROTTYPE=STRING + L34PROTOCOL=ANY | — | 内容/在线/离线/融合计费、URL过滤、PCC 等大多数场景（默认匹配） |
| **协议+端口过滤** | FILTERNAME + L34PROTTYPE=STRING + L34PROTOCOL=TCP/UDP + SVRSTARTPORT/SVRENDPORT | MSSTARTPORT/MSENDPORT | 计费 TCP:80、DNS:53、HTTPS:443 等固定端口业务 |
| **服务器 IPv4 过滤** | FILTERNAME + L34PROTTYPE=STRING + L34PROTOCOL=TCP + SVRIPMODE=IP + SVRIP + SVRIPMASKTYPE(+SVRIPMASK 或 SVRIPMASKLEN) | — | WebProxy（服务器 IP 全网规划）、Bypass（服务器 IP 本端规划）、Shaping（服务器 IP 与对端协商） |
| **手机 IPv4 过滤** | FILTERNAME + L34PROTTYPE=STRING + L34PROTOCOL + MSIPMODE=IP + MSIP + MSIPMASKTYPE/MSIPMASK | — | Shaping 业务（用户 IP 段匹配） |

### 配置维度 2：协议输入方式（参数 L34PROTTYPE，必选）

| 取值 | 协议表示 | 配套参数 | 备注 |
|---|---|---|---|
| **STRING** | 协议名字符串 | L34PROTOCOL（ANY/TCP/UDP 等） | 默认且高频（8 特性全部用 STRING） |
| **NUM** | 协议号数字（0~255） | L34PROTNUM（数字） | 冷门，无特性使用 |

### 配置维度 3：端口范围（参数 MSSTARTPORT/MSENDPORT/SVRSTARTPORT/SVRENDPORT）

四端口参数决定端口匹配范围；默认全匹配（0/65535），按场景收紧：

| 参数 | 默认 | 收紧场景 |
|---|---|---|
| `MSSTARTPORT` / `MSENDPORT` | 0 / 65535 | MS 起始端口（任何业务下都可不配） |
| `SVRSTARTPORT` / `SVRENDPORT` | 0 / 65535 | 服务器端口（协议+端口/服务器 IP 模式下必配或可选） |

**典型脚本（any 通配）**：
```
ADD FILTER: FILTERNAME="L34_any", L34PROTTYPE=STRING, L34PROTOCOL=ANY, MSSTARTPORT=0, MSENDPORT=65535, SVRSTARTPORT=0, SVRENDPORT=65535;
```

**典型脚本（TCP:80 端口过滤）**：
```
ADD FILTER:FILTERNAME="filter_red",L34PROTTYPE=STRING,L34PROTOCOL=TCP,SVRSTARTPORT=80,SVRENDPORT=80;
```

### 配置维度 4：IPv4 地址过滤（参数 MSIPMODE/MSIP/SVRIPMODE/SVRIP）

服务器 IP 组（SVRIPMODE/SVRIP/SVRIPMASKTYPE/SVRIPMASK/SVRIPMASKLEN）和手机 IP 组（MSIPMODE/MSIP/MSIPMASKTYPE/MSIPMASK）独立分支：

| 模式 | 参数 | 取值范围 | 来源分叉 |
|---|---|---|---|
| `SVRIPMODE` | 服务器 IP 模式 | NULL / IP / LISTNAME | — |
| `SVRIP` | 服务器 IPv4 | IPv4 地址 | Shaping=peer、WebProxy=全网、Bypass=local |
| `SVRIPMASKTYPE` | 服务器 IP 掩码类型 | LENGTHTYPE / 其他 | — |
| `SVRIPMASKLEN` | 服务器 IP 掩码长度 | 0~32 | — |
| `MSIPMODE` | 手机 IP 模式 | NULL / IP / LISTNAME | 仅 Shaping 用 |
| `MSIP` | 手机 IPv4 | IPv4 地址 | 仅 Shaping 用，peer |
| `MSIPMASKTYPE` | 手机 IP 掩码类型 | LENGTHTYPE / 其他 | — |

> HOSTNAME / DOMAINVALUE / HOSTPRIORITY（域名类过滤）通过 HTTPS 业务地址识别特性（GWFD-110203）使用 HOSTNAME；其余冷门参数（L34PROTNUM/MSIPLISTNAME/SVRIPLISTNAME/TOS*/CFGDOMAINNAME）实例化不展开。

**步骤位置**：在所属 feature 编排中通常位于"步骤 3：配置三四层过滤条件"——ADD FLOWFILTER 之前，ADD FLTBINDFLOWF 引用本命令产出的 FILTERNAME。

## 决策点：IPv4 过滤条件匹配模式（DP 0-00055）

问：ADD FILTER 的过滤条件采用何种匹配模式？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| any 通配（ANY） | 仅 FILTERNAME+L34PROTTYPE+L34PROTOCOL；端口默认 0/65535 | 内容计费/PCC/URL过滤等 any-to-any 场景 |
| 协议+端口（PROTO_PORT） | SVRSTARTPORT/SVRENDPORT 必配（如 TCP:80、DNS:53） | 计费 HTTP 固定端口、DNS 等 |
| 服务器 IP·对端协商（SVRIP_PEER） | SVRIPMODE=IP+SVRIP/SVRIPMASK，来源=peer | Shaping（业务服务器 IP） |
| 服务器 IP·全网规划（SVRIP_GLOBAL） | SVRIPMODE=IP+SVRIP/SVRIPMASK，来源=全网 | WebProxy（业务服务器 IP） |
| 服务器 IP·本端规划（SVRIP_LOCAL） | SVRIPMODE=IP+SVRIPMASKTYPE=LENGTHTYPE+SVRIPMASKLEN | Bypass SBC-U 媒体面 IP |
| 手机 IP（MSIP） | MSIPMODE=IP+MSIP/MSIPMASKTYPE/MSIPMASK | 仅 Shaping |

默认 ANY（绝大多数场景）；服务器 IP 来源按特性分叉（Shaping=peer、WebProxy=全网、Bypass=local）。

## 约束

- **ADD FILTER 刷新生效与记录数上限**（critical，rule-0-00120）：执行后需 SET REFRESHSRV（REFRESHTYPE=USERPROFILE 或 ALL）刷新生效；Filter 与 FilterIPv6 总记录数上限 150000；超规格 85% 上报 ALM-135602215；阈值可 MOD CFGTHRESHOLD 修改 — 后果：未刷新不生效 / 容量超限告警
- **FILTERNAME 全局唯一**（critical，隐含）：不同过滤器之间 FILTERNAME 不能重复 — 后果：重复新增失败
- **协议输入方式二选一**（warning，隐含）：L34PROTTYPE=STRING 时配 L34PROTOCOL；L34PROTTYPE=NUM 时配 L34PROTNUM；不能混用 — 后果：参数缺失则校验失败
