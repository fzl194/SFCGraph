# 配置三四层过滤器-IPv4（ADD FILTER）

> 命令静态知识见 [ADD FILTER](command/UDG/20.15.2/ADD-FILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD FILTER 配 IPv4 三四层过滤条件，匹配模式随业务分叉（见决策点）。FILTERNAME 本端规划、不重复，后续 ADD FLTBINDFLOWF 把它绑进 FLOWFILTER。

**4 种匹配范式（从证据数据规划表归纳）**

| 范式 | 典型场景 | 典型行 |
|---|---|---|
| any 通配 | 内容计费/PCC/URL 过滤的兜底 filter | `ADD FILTER:FILTERNAME="L34_any",L34PROTTYPE=STRING,L34PROTOCOL=ANY,MSSTARTPORT=0,MSENDPORT=65535,SVRSTARTPORT=0,SVRENDPORT=65535;` |
| 协议+端口 | 在线/离线计费按业务端口（如 TCP:80） | `ADD FILTER:FILTERNAME="filter_red",L34PROTTYPE=STRING,L34PROTOCOL=TCP,SVRSTARTPORT=80,SVRENDPORT=80;` |
| 协议+端口(DNS) | URL 过滤配 DNS 53 | `ADD FILTER:FILTERNAME="L34_dns",L34PROTTYPE=STRING,L34PROTOCOL=ANY,SVRSTARTPORT=53,SVRENDPORT=53;`（手机侧 0/65535 通配） |
| 服务器 IP | Shaping/WebProxy/Bypass | 配 SVRIPMODE=IP + SVRIP/SVRIPMASKTYPE(+SVRIPMASK 或 SVRIPMASKLEN)，来源分叉见下 |

**any 通配的端口默认值**：MSSTARTPORT=0/MSENDPORT=65535/SVRSTARTPORT=0/SVRENDPORT=65535（全通配）；端口四参数命令真相 optional，默认 0/65535，可省略。

**L34PROTTYPE 固定 STRING**（8 特性均用），L34PROTOCOL 选 ANY/TCP/UDP；L34PROTNUM（数字协议）实例化不用。

## 决策点：IPv4 过滤条件匹配模式（DP 0-00055）

问：过滤条件采用何种匹配模式？决定 IP/端口参数是否配及其来源（本端/对端/全网）。

| 选项 | 联动 | 来源 |
|---|---|---|
| any 通配 | 不配 IP/端口，L34PROTOCOL=ANY | — |
| 协议+端口 | 配 SVRSTARTPORT/SVRENDPORT（如 TCP:80、DNS:53） | 本端规划 |
| 服务器 IP（与对端协商） | SVRIPMODE=IP + SVRIP/SVRIPMASK | Shaping=peer |
| 服务器 IP（全网规划） | 同上 | WebProxy=全网 |
| 服务器 IP（本端规划） | SVRIPMASKTYPE=LENGTHTYPE + SVRIPMASKLEN | Bypass SBC-U 媒体面=local |

## 约束

- **rule-0-00120**（critical）配置后**必须** `SET REFRESHSRV`（REFRESHTYPE=USERPROFILE 或 ALL）刷新才生效；Filter+FilterIPv6 总记录数上限 150000，超 85% 上报 ALM-135602215（阈值可 MOD CFGTHRESHOLD 修改） — 后果：不刷新配置不生效 / 容量超限告警
