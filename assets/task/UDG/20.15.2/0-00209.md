# 增加IPv6 ACL规则组（ADD ACLGROUP6）

> 命令静态知识见 [ADD ACLGROUP6](command/UDG/20.15.2/ADD-ACLGROUP6.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。ACL 是一种 IP 包过滤技术，通过对 IPv6 报文的源地址/通配位、报文目的地址/通配位、协议号、源端口号、目的端口号（五元组，端口仅 TCP/UDP 有效）进行匹配；**使用 ACL 前必须先建组**。本命令与 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP（IPv4 ACL 组）的核心差异：范围段语义按 ACLNAME 数字段区分（IPv4:2000~2999 基本 / 3000~3999 高级 / 4000~4999 以太 / 1000~1999 接口；IPv6:2000~2999 基本 / 3000~3999 高级）。

## 配置方法（配置方法字典）

ADD ACLGROUP6 是"增加 IPv6 ACL 规则组"命令。**1 参必选**（ACLNAME）+ 4 参可选（ACLSTEP/ACLTYPE/ACLMATCHORDER/ACLDESCRIPTION）。配置空间有 4 个维度：**规则组标识**（ACLNAME 数字段 vs 字符串名）、**规则组类型**（ACLTYPE Basic/Advance）、**步长**（ACLSTEP 默认 5）、**匹配顺序**（ACLMATCHORDER Config/Auto）。1 个特性命中（IPFD-012002 IPv6 ACL）覆盖命名段 Basic 范式。

### 配置维度 1：规则组标识（参数 ACLNAME，必选）

| 取值形式 | 适用范围 | 典型场景 |
|---|---|---|
| **数字字符串 2000~2999（基本 IPv6）** | 基本 IPv6 ACL | IPFD-012002 范例可走数字段 `2000` |
| **数字字符串 3000~3999（高级 IPv6）** | 高级 IPv6 ACL | 高级 IPv6 五元组匹配 |
| **字母开头的字符串（如 "group_basic"）** | 命名 IPv6 ACL | IPFD-012002 范例 `group_basic`，长度 1~32 |

> **取值细节**：ACLNAME 以英文字母 a~z 或 A~Z 开始，可含字母/数字/连字符`-`/下划线`_`/中文字符，**不支持空格**、区分大小写。

### 配置维度 2：规则组类型（参数 ACLTYPE，可选）

| 取值 | 适用 ACLNAME 形式 | 默认行为 |
|---|---|---|
| **Basic** | 数字 2000~2999 或命名 | 基本 IPv6 ACL |
| **Advance** | 数字 3000~3999 或命名 | 高级 IPv6 ACL（五元组） |
| **不配** | 非数字 ACLNAME | **默认 Advanced** |
| **不配** | 数字 ACLNAME | 数字段推断类型 |

> **默认行为关键**：非数字 ACLNAME → 默认 Advanced；数字 ACLNAME → 可省 ACLTYPE；**ACLTYPE 与 ACLNAME 必须语义一致**（如数字 3001 不可配 Basic）。

### 配置维度 3：步长（参数 ACLSTEP，可选）

| 取值范围 | 默认 | 作用 |
|---|---|---|
| 整数 1~20 | 5 | 规则 ID 增长步长；仅当不指定规则 ID 时生效 |

### 配置维度 4：匹配顺序（参数 ACLMATCHORDER，可选）

| 取值 | 默认 | 作用 |
|---|---|---|
| **Config**（默认） | ✓ | 配置优先（按配置顺序） |
| **Auto** | — | 深度优先（按规则精确度匹配） |

**典型脚本（IPFD-012002 命名基本组 group_basic）**：
```
ADD ACLGROUP6: ACLNAME="group_basic", ACLTYPE=Basic;
```

**典型脚本（IPv6 高级组 数字段）**：
```
ADD ACLGROUP6: ACLNAME="3000";
```

> 命令 notes：**最大记录数 65535**（整机 ACL 组总数）；**单组最多 16384 条规则**；**执行后立即生效**；ACLDESCRIPTION（字符串 1~127，支持空格）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：建 IPv6 ACL 规则组"——前置 无；下游 ADD ACLRULEBAS6（★R1.5 atom 待补） / ADD ACLRULEADV6（★R1.5 atom 待补）（本组 ACLNAME 引用）。

## 决策点

（无。本命令在所有特性下用法一致——仅建空 IPv6 ACL 规则组容器，规则类型 Basic/Advance 二选一，无特性分叉。）

## 约束

- **最大记录数 65535**（critical，rule-0-00209）：整机 IPv6 ACL 规则组最大 65535 条 — 后果：超规格新增失败
- **单组最多 16384 条规则**（critical，rule-0-00209）：单个 ACL 规则组下最多 16384 条规则 — 后果：超限新增失败
- **执行后立即生效**（warning，rule-0-00209）：执行后立即生效 — 后果：误操作即刻生效
- **ACLTYPE 默认推断规则**（warning，rule-0-00209）：非数字 ACLNAME → 默认 Advanced；数字 ACLNAME → 可省 — 后果：ACLTYPE 与数字段不一致命令拒收
- **ACLNAME 范围段语义**（info，rule-0-00209）：数字字符串须落在 2000-2999（基本 IPv6）/ 3000-3999（高级 IPv6） — 后果：跨段数字命令拒收
- **ACLNAME 长度约束**（warning，rule-0-00209）：字符串 1~32（不支持特殊字符，以英文字母开始） — 后果：超长或格式错命令拒收
- **ACLDESCRIPTION 长度**（info，rule-0-00209）：字符串 1~127（支持空格，区分大小写） — 后果：超长命令拒收
- **ACLSTEP 范围**（info，rule-0-00209）：整数 1~20（默认 5） — 后果：超范围命令拒收
