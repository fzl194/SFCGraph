# 配置周期时间段（ADD PERITIMERANGE）

> 命令静态知识见 [ADD PERITIMERANGE](command/UDG/20.15.2/ADD-PERITIMERANGE.md)

## 配置方法

在一个已建的 TIMERANGE 容器下，按 PERITIMETYPE（DAYSOFWEEK 周内某几天 / STARTTOEND 跨天）填一条周期时段。容器名+序号唯一标识一条子记录；起始/结束时间必配。

**DAYSOFWEEK 模式（周内某几天，同一天起止）**：

| 参数 | 取值样例 | 说明 |
|---|---|---|
| TIMERANGENAME | `timerange1` / `timerange2` | 必选，引用 ADD TIMERANGE（0-00287） |
| PERITMRANGESEQ | `1` | 必选，1~32，容器内唯一 |
| PERITIMETYPE | `DAYSOFWEEK` | 必选，周内某几天类型 |
| DAYOFWEEK | `DAILY` / `WORKINGDAY` / `OFFDAY` / `MONDAY`... | DAYSOFWEEK 时必选 |
| STARTTIME | `8&30` / `22&00` | 必选，格式 HH&MM |
| ENDTIME | `10&00` / `23&00` | 必选，格式 HH&MM |

**STARTTOEND 模式（跨天，周一起周三止）**：

| 参数 | 取值样例 | 说明 |
|---|---|---|
| STARTDAY | `TUESDAY` | STARTTOEND 时必选 |
| ENDDAY | `WEDNESDAY` | STARTTOEND 时必选 |
| STARTTIME / ENDTIME | `10&00` / `11&00` | 必选 |

**典型脚本**：

```
// 每天 8:30~10:00 生效（白天时段）
ADD PERITIMERANGE:TIMERANGENAME="timerange1",PERITMRANGESEQ=1,PERITIMETYPE=DAYSOFWEEK,DAYOFWEEK=DAILY,STARTTIME=8&30,ENDTIME=10&00;

// 每天 22:00~23:00 生效（夜间时段）
ADD PERITIMERANGE:TIMERANGENAME="timerange2",PERITMRANGESEQ=1,PERITIMETYPE=DAYSOFWEEK,DAYOFWEEK=DAILY,STARTTIME=22&00,ENDTIME=23&00;

// 跨天：周二 10:00 到周三 11:00（STARTTOEND 模式）
ADD PERITIMERANGE:TIMERANGENAME="timerange1",PERITMRANGESEQ=2,PERITIMETYPE=STARTTOEND,STARTTIME=10&00,STARTDAY=TUESDAY,ENDTIME=11&00,ENDDAY=WEDNESDAY;
```

**步骤位置**：时段带宽控制链路的第 2 步——在 ADD TIMERANGE（0-00287）之后、ADD BWMRULE（0-00037）之前；容器建好后填充周期时段，再把容器挂到规则。

## 决策点

**DP 0-00288**：周期时段配置方式（PERITIMETYPE）——按"周内某几天同时段"还是"跨天连续时段"建模。

| 选项 | PERITIMETYPE | 必配联动参数 | 适用 |
|---|---|---|---|
| 周内某几天同时段 | `DAYSOFWEEK` | DAYOFWEEK（SUNDAY~SATURDAY/WORKINGDAY/OFFDAY/DAILY）+ STARTTIME + ENDTIME | 每天或工作日固定时段，同一天内起止 |
| 跨天连续时段 | `STARTTOEND` | STARTDAY + ENDDAY + STARTTIME + ENDTIME | 需跨天（如周二晚到周三晨），可超过单日边界 |

DAYOFWEEK 9 种枚举：SUNDAY/MONDAY/TUESDAY/WEDNESDAY/THURSDAY/FRIDAY/SATURDAY 单日，WORKINGDAY（周一~周五）、OFFDAY（周六~周日）、DAILY（周日~周六每天）批量。STARTTIME/ENDTIME 均相对周日 00:00 计算。

## 约束

- 最大记录数 6400 · warning · 超规格新增失败
- 命令执行后立即生效 · info · 无需刷新
- 每个时间段下最多 32 条周期时段 · error · 子记录上限（与 ADD TIMERANGE 容器配额联动）
- 周期时段开始时间不能大于等于结束时间 · error · 时序校验
- 被 BWMRULE 绑定时，新增周期时段不能与该规则已绑其它时段下的周期时段重叠 · error · 重叠校验
- 被 BWMRULE 绑定时，新增周期时段时间间隔不能小于 30 分钟 · error · 最小粒度
- PCC 用户遇夏令时切换/系统时间修改时，按 TimeRange 开始时间 + 时长（不考虑夏令时）计算结束 · info · 夏令时处理
