# 配置业务公共参数（SET SRVCOMMONPARA）

> 命令静态知识见 [SET SRVCOMMONPARA](command/UDG/20.15.2/SET-SRVCOMMONPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET SRVCOMMONPARA 承载**多个独立全局开关/参数**（命令真相 43 项），各特性按需设其一（requiredness=conditional）。本 atom 收敛 010201/110202/110910/110281 四特性用到的 5 个参数，38 个冷门参数按 §4.1"通用配置"门槛不绑。

**5 个绑定参数的配置范式（从证据数据规划表归纳）**

| 参数 | 取值 | 来源 | 用途特性 | 典型行 |
|---|---|---|---|---|
| TUNNELMARKINGID | ENABLE | 本端规划 | GWFD-010201 QoS 优先级映射使能 | `SET SRVCOMMONPARA:TUNNELMARKINGID=ENABLE;` |
| HTTP2DEGRADESW | ENABLE/DISABLE | 本端规划 | GWFD-110202 全局 HTTP2.0 协议回落 | `SET SRVCOMMONPARA:HTTP2DEGRADESW=ENABLE;` |
| FDMAXNUM | 800（默认 600） | 本端规划 | GWFD-110910 全局单用户五元组上限（超默认 600 才配） | `SET SRVCOMMONPARA:FDMAXNUM=800,USERFLOWADJSW=ENABLE;` |
| USERFLOWADJSW | ENABLE | 固定取值 | GWFD-110910 单用户最大流表数动态调整开关 | 同上（与 FDMAXNUM 配对）|
| **HTTPREDIRCODE** | **RSP_CODE_301/RSP_CODE_302/RSP_CODE_303**（枚举）| 本端规划 | GWFD-110281 用户Portal HTTP 重定向响应码（301永久/302临时/303 See Other）| `SET SRVCOMMONPARA:HTTPREDIRCODE=RSP_CODE_302;` |

**QoS 优先级映射（010201）**：入口/出口两方向各执行一次 SET SRVCOMMONPARA 使能 TunnelMarking，随后 [ADD QOSDIFFERSERV](task/UDG/20.15.2/0-00066.md) 建 DS 域、[SET QOSBA](task/UDG/20.15.2/0-00067.md)/[SET QOSPHB](task/UDG/20.15.2/0-00069.md) 配映射。若同时配 CoS→ToS 与 3GPP QoS→DSCP/ToS，前者被后者覆盖。

**HTTP2.0 回落（110202）**：回落优先级 rule > APN（[SET APNHTTP2DGRD]）> 全局；关则 DISABLE。

**Routing Behind MS（110910）**：FDMAXNUM 全局单用户五元组上限，须**大于等于** APN 级 [SET APNFLOWMAXNUM]；仅当现网超默认 600 才配；USERFLOWADJSW=ENABLE 与之配对开启动态调整。

**用户Portal HTTP 重定向响应码（110281）**：UDG 构造的 HTTP 重定向回复报文返回码全局配置，取枚举值 `RSP_CODE_301`（永久重定向）/`RSP_CODE_302`（临时重定向，推荐）/`RSP_CODE_303`（See Other，以 GET 访问）——**非裸数字 301/302/303**。用户 Portal 主 activation 未演示此参数，证据来自独立文档 [修改HTTP重定向响应码_66620118](evidence/UDG/20.15.2/修改HTTP重定向响应码_66620118.md)。

## 约束

- **rule 0-00331**（cardinality，critical）最大记录数为 1（全局唯一）；**高危命令**（可能影响业务）；执行后立即生效 — 后果：高危配置错误影响全局业务
