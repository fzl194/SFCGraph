# 配置带宽管理规则（ADD BWMRULE）

> 命令静态知识见 [ADD BWMRULE](command/UDG/20.15.2/ADD-BWMRULE.md)

## 配置方法

把已建的 BWMUSERGROUP + BWMSERVICE + BWMCONTROLLER（上下行）组装成一条规则，由 BWMRULETYPE 决定是用户级还是用户组级规则。规则是 BWM 三件套的汇聚点。

**固定骨架参数**（跨特性一致）：

| 参数 | 取值样例 | 来源 |
|---|---|---|
| USERGROUPTYPE | `SPECIFIC_GROUP` | 与 BWMUSERGROUP 一致 |
| USERGROUPNAME | `ug_test` | 引用 BWMUSERGROUP（0-00033） |
| BWMSERVICENAME | `bs_test` | 引用 BWMSERVICE（0-00034），GROUP_DEFAULT 类型可不传 |
| UPBWMCTRLNAME1 | `bc-up` | 引用 BWMCONTROLLER（0-00036），上行 |
| DNBWMCTRLNAME1 | `bc-down` | 引用 BWMCONTROLLER（0-00036），下行 |
| BWMRULEPRI | `1` | 1~32768，同级别下唯一 |

**BWMRULETYPE 分叉**：

| 规则类型 | 含义 | BWMRULENAME | 典型场景 |
|---|---|---|---|
| SUBSCRIBER_SPECIFIC | 用户级特定规则 | 必配 | 020354 用户级整形、110311 用户级带宽 |
| GROUP_SPECIFIC | 用户组级特定规则 | 必配 | 110301/110311 用户组级带宽、切片、时间段 |
| GROUP_DEFAULT | 用户组级默认规则 | 不配 | 110311 APN/DNN 用户组级（无需区分业务类型，不绑 BWMSERVICE） |

**典型脚本**：

```
# 用户级规则（020354）
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug-bwm",BWMRULETYPE=SUBSCRIBER_SPECIFIC,BWMRULENAME="br-p2p",BWMSERVICENAME="my-bwm-service1",UPBWMCTRLNAME1="bc-up-p2p",DNBWMCTRLNAME1="bc-dn-p2p",BWMRULEPRI=1;

# 用户组级默认规则（110311 APN/DNN，不绑 Service）
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_DEFAULT,UPBWMCTRLNAME1="bc-up",DNBWMCTRLNAME1="bc-down";
```

**多时段扩展**：基于时间段的场景（110311）启用 TIMERANGENAME1~4 + UPBWMCTRLNAME2~4/DNBWMCTRLNAME2~4，一个规则绑多个时段控制器。

**步骤位置**：BWM 三件套最后一步——所有被引用对象（用户组/业务/控制器）必须先建好。

## 决策点

**DP 0-00101**：BWMRULE 规则类型分叉（SUBSCRIBER_SPECIFIC vs GROUP_SPECIFIC），互斥二选一，BWMRULENAME 随之必配。GROUP_DEFAULT 走默认分支不参与本 DP。

| 选项 | BWMRULETYPE | BWMRULENAME |
|---|---|---|
| 用户级规则（默认，020354） | SUBSCRIBER_SPECIFIC | required |
| 用户组级规则（110301） | GROUP_SPECIFIC | required |

## 约束

- 最大记录数 5000（与 ADD BWMRULEGLOBAL 共享 BwmRule 对象配额） · warning · 超规格新增失败；90% 阈值告警 ALM-135602215
- 命令执行后立即生效 · info · 无需刷新
- 同一用户级/用户组级下 BWMRULEPRI 不能相同 · warning · 优先级冲突
- ADD BWMRULE 与 ADD BWMRULEGLOBAL 共享 5000 上限 · warning · 跨命令联合计数
