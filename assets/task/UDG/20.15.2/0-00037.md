# 配置带宽管理规则（ADD BWMRULE）

> 命令静态知识见 [ADD BWMRULE](command/UDG/20.15.2/ADD-BWMRULE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD BWMRULE 是带宽控制骨架的"规则装配层"：把"哪一用户组/用户的哪一业务、按什么规则、绑定哪个上/下行控制器"用一条命名记录描述，产出 BWMRULENAME 供 BWM 评估阶段按用户/用户组 + 业务匹配命中后下发对应控制器。命令核心配置空间有 3 个维度：**规则类型**（BWMRULETYPE 4 选 1：用户组默认/用户组特定/用户默认/用户特定）、**业务匹配**（BWMSERVICENAME 必选 + 可选 RATTYPE/ROAMINGTYPE 过滤）、**控制器挂接**（UPBWMCTRLNAME1~4 / DNBWMCTRLNAME1~4 + 可选 TIMERANGENAME1~4 时段区分 + SERVICELEVEL 业务级别）。3 个特性命中覆盖 020354 SUBSCRIBER_SPECIFIC 基础、110301 GROUP_SPECIFIC、110311 SUBSCRIBER_SPECIFIC + GROUP_DEFAULT 兜底 + 110311 三档 GROUP_SPECIFIC 多 SERVICELEVEL。

### 配置维度 1：规则类型（参数 BWMRULETYPE，条件必选 — USERGROUPTYPE=DEFAULT_GROUP/SPECIFIC_GROUP 时）

| 取值 | 作用 | 必选联动 |
|---|---|---|
| **GROUP_DEFAULT** | 用户组级默认规则（兜底） | USERGROUPTYPE + UPBWMCTRLNAME1/DNBWMCTRLNAME1 |
| **GROUP_SPECIFIC** | 用户组级特定规则（按业务匹配） | + BWMRULENAME + BWMSERVICENAME + BWMRULEPRI |
| **SUBSCRIBER_DEFAULT** | 用户级默认规则 | UPBWMCTRLNAME1/DNBWMCTRLNAME1 |
| **SUBSCRIBER_SPECIFIC** | 用户级特定规则（按业务匹配，最精细） | + BWMRULENAME + BWMSERVICENAME + BWMRULEPRI |

### 配置维度 2：业务匹配（参数 BWMSERVICENAME + RATTYPE/ROAMINGTYPE，特定规则时必选）

| 参数 | 必选性 | 取值 | 作用 |
|---|---|---|---|
| **BWMSERVICENAME** | GROUP/SPECIFIC 时必选 | 引用 [0-00034](task/UDG/20.15.2/0-00034.md) 的 BWMSERVICENAME | 业务范围匹配锚点 |
| **RATTYPE** | 可选 | NR / LTE / UMTS / GSM / ... | 按接入技术过滤 |
| **ROAMINGTYPE** | 可选 | HOME / ROAMING / ... | 按漫游状态过滤 |

### 配置维度 3：规则优先级（参数 BWMRULEPRI，特定规则时必选）

| 取值范围 | 作用 |
|---|---|
| 整数 | 规则匹配优先级，值越小优先级越高（命令 notes：值越小越优先） |

### 配置维度 4：控制器挂接（参数 UPBWMCTRLNAME1~4 / DNBWMCTRLNAME1~4 + 可选时段）

| 参数 | 必选性 | 数量 | 作用 |
|---|---|---|---|
| **UPBWMCTRLNAME1 / DNBWMCTRLNAME1** | DEFAULT 时必选；SPECIFIC 时条件可选 | 1~4 对 | 第 1~4 时段对应的上/下行控制器，引用 [0-00036](task/UDG/20.15.2/0-00036.md) 的 BWMCNAME |
| **TIMERANGENAME1~4** | 可选 | 1~4 | 第 1~4 时段对应的时间段名，引用 ADD TIMERANGE 配置（[0-00287](task/UDG/20.15.2/0-00287.md)） |

### 可选：业务级别（参数 SERVICELEVEL，仅特定规则时）

| 取值 | 作用 |
|---|---|
| 整数 | 同一控制器的不同业务流间的优先级，值越小优先级越高（110311 演示 1/2/3 三档） |

**典型脚本（020354 基于业务的 Shaping，SUBSCRIBER_SPECIFIC）**：
```
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="usergroup_test",BWMRULETYPE=SUBSCRIBER_SPECIFIC,BWMRULENAME="bwmrule_test",BWMSERVICENAME="bwmservice_test",UPBWMCTRLNAME1="bwmc_test",DNBWMCTRLNAME1="bwmc_test",BWMRULEPRI=1;
```

**典型脚本（110301 码率差异化，GROUP_SPECIFIC）**：
```
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="br_test",BWMSERVICENAME="bs_test",UPBWMCTRLNAME1="bc-up",DNBWMCTRLNAME1="bc-down",BWMRULEPRI=1;
```

**典型脚本（110311 业务感知带宽控制，3 档 GROUP_SPECIFIC + SERVICELEVEL）**：
```
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="br_test1",BWMSERVICENAME="bs_test1",UPBWMCTRLNAME1="bc_test",DNBWMCTRLNAME1="bc_test",BWMRULEPRI=10,SERVICELEVEL=1;
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_SPECIFIC,BWMRULENAME="br_test2",BWMSERVICENAME="bs_test2",UPBWMCTRLNAME1="bc_test",DNBWMCTRLNAME1="bc_test",BWMRULEPRI=15,SERVICELEVEL=2;
```

**典型脚本（110311 兜底，GROUP_DEFAULT）**：
```
ADD BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug_test",BWMRULETYPE=GROUP_DEFAULT,UPBWMCTRLNAME1="bc-up",DNBWMCTRLNAME1="bc-down";
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；4 对控制器+4 时段支持精细的"分时段分业务"控制；UPBWMCTRLNAME1+DNBWMCTRLNAME1 总是必选（DEFAULT 直接必选、SPECIFIC 条件必选）。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：规则绑定"——前置 ADD BWMUSERGROUP(0-00033) + ADD BWMSERVICE(0-00034) + ADD BWMCONTROLLER(0-00036) + ADD TIMERANGE(0-00287)（如配时段）；产出 BWMRULENAME 供 BWM 评估使用。

## 决策点：规则类型档位选择（DP 0-00037）

问：ADD BWMRULE 选哪个规则类型档位？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| GROUP_DEFAULT（兜底） | USERGROUPTYPE + UP/DNBWMCTRLNAME1 | 用户组级兜底（无业务匹配）（110311 兜底范式） |
| GROUP_SPECIFIC | + BWMRULENAME + BWMSERVICENAME + BWMRULEPRI [+ SERVICELEVEL] [+ 时段] | 用户组级精细化（110301/110311 范式） |
| SUBSCRIBER_DEFAULT | UP/DNBWMCTRLNAME1 | 用户级兜底（命令参数齐全） |
| SUBSCRIBER_SPECIFIC | + BWMRULENAME + BWMSERVICENAME + BWMRULEPRI [+ 时段] | 用户级精细化（020354/110311 范式） |

## 约束

- **USERGROUPTYPE + BWMRULETYPE 联动必选**（critical，rule-0-00337）：USERGROUPTYPE=DEFAULT_GROUP 时 BWMRULETYPE 必选；USERGROUPTYPE=SPECIFIC_GROUP 时 BWMRULETYPE 必选 — 后果：缺 BWMRULETYPE 命令拒收
- **特定规则必配业务+名称+优先级**（critical，rule-0-00337）：BWMRULETYPE=*_SPECIFIC 时 BWMSERVICENAME + BWMRULENAME + BWMRULEPRI 必选 — 后果：缺任一命令拒收
- **UPBWMCTRLNAME1 + DNBWMCTRLNAME1 总必选**（critical，rule-0-00337）：DEFAULT 直接必选；SPECIFIC 条件必选；其他 1~4 时段条件可选 — 后果：缺主控制器命令拒收
- **BWMRULENAME 全网唯一**（critical，隐含）：不同规则之间 BWMRULENAME 不能重复 — 后果：重名 ADD 失败
- **控制器引用前置依赖**（critical，rule-0-00337）：UP/DNBWMCTRLNAME1~4 须先建（[0-00036](task/UDG/20.15.2/0-00036.md)）；时段引用须先建（[0-00287](task/UDG/20.15.2/0-00287.md)） — 后果：未建引用失败
- **业务引用前置依赖**（critical，rule-0-00337）：BWMSERVICENAME 须先建（[0-00034](task/UDG/20.15.2/0-00034.md)） — 后果：未建引用失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
