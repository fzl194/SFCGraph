# 终端异常下行流量检测（GWFD-020305）

> 特性静态知识见 [GWFD-020305](feature/UDG/20.15.2/GWFD-020305.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

终端异常下行流量检测在指定 APN 下开启异常下行流量检测开关，识别终端异常下行流量（DDoS 攻击/端口扫描等）。**核心机制：APN 级 ADD ABNTRAFFICDT 开关（SWITCH）+ 基于流行为的检测开关（TRAFFICBEHAVIORSWITCH）**。activation 极简（2 步：License + APN 级检测开关）；可选 SET ABNTRADTTHR 调整检测报文阈值（方案二）。

## 配置流程（步骤编排，每步显示含命令）

1. **License 前置**（单命令）：`SET LICENSESWITCH`（LKV3G5ADTD01）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **APN 前置**（activation 必备事项要求"完成配置 APN"，单命令）：`ADD APN` → [0-00020](task/UDG/20.15.2/0-00020.md)
3. **APN 级异常流量检测开关**（核心，单命令）：`ADD ABNTRAFFICDT`（APN + SWITCH=ENABLE + TRAFFICBEHAVIORSWITCH=ENABLE 基于流行为检测）→ [0-00021](task/UDG/20.15.2/0-00021.md)
   - 可选：检测报文阈值调整（方案二，基于报文行为检测时）：`SET ABNTRADTTHR`（DETECTTHR）→ [0-00295](task/UDG/20.15.2/0-00295.md)

> **参数证据链**（R1.5）：SET LICENSESWITCH（LKV3G5ADTD01）/ ADD ABNTRAFFICDT（APN=apn1.com, SWITCH=ENABLE, TRAFFICBEHAVIORSWITCH=ENABLE）均来自 activation §数据规划表 + §任务示例脚本（原样）。
>
> **activation 模板复用说明**（R1.4）：activation 主脚本演示标准场景（APN 级开关 + 流行为检测开关）。SET ABNTRADTTHR 阈值调整为方案二可选参数（activation 未演示主脚本，证据来自命令 wiki + 特性概述）。

## 决策点

**DP（特性级）：检测方式 + 阈值**（影响 ADD ABNTRAFFICDT 参数 + 可选阈值命令）：

| 场景 | TRAFFICBEHAVIORSWITCH | SET ABNTRADTTHR | 走法 |
|---|---|---|---|
| **标准检测（activation 演示）** | ENABLE（基于流行为检测）| 不配（用初始阈值 20）| ADD ABNTRAFFICDT |
| 仅基础检测 | DISABLE | 不配 | ADD ABNTRAFFICDT（仅 SWITCH=ENABLE）|
| 方案二阈值调整 | ENABLE | **配置 DETECTTHR** | ADD ABNTRAFFICDT + SET ABNTRADTTHR（[0-00295](task/UDG/20.15.2/0-00295.md)）|

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5ADTD01` — 未开则检测不生效。
- **APN 前置**（critical）：ADD ABNTRAFFICDT 的 APN 须已通过 ADD APN 配置 — 未配则开关绑定失败。
- **APN 级粒度**（info）：检测开关为 APN 维度，逐 APN 配置。
- **阈值全局唯一**（info）：SET ABNTRADTTHR 全局唯一（最大记录数 1，初始值 20），仅对新激活用户生效。
