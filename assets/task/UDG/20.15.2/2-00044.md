# IPSec（IPFD-015004，UDG 侧）

> 特性静态知识见 [IPFD-015004](feature/UDG/20.15.2/IPFD-015004.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/IPFD-015004-IPSec功能.md)。
>
> **本页定位**：UDG 侧 IPSec 隧道配置生成——9 步建立隧道（VNRS+IPsec 双配 VPN/Tunnel + ACL 数据流 + IPsec/IKE 提议 + IKE Peer + 安全策略 + 应用 + 可选 DPD）。调测命令见 [调测 IPSec](evidence/UDG/20.15.2/IPFD-015004-IPSec功能/调测IPsec功能_61317192.md)，不在配置流程展开。

## 配置概览

本特性按 **9 场景 × 1 compound 编排** 组织。IPSec 隧道建立是核心（VNRS+IPsec 微服务双配），9 场景共享同一 9 步骨架，差异在协议/算法/IKE 版本/主备/NAT 穿越/前置隧道。

**对象链**：
```
VNRS 微服务（VPN+Tunnel+IP+静态路由）
    ↓ 双配（接口名/IP/VPN 必一致）
IPsec 微服务（VPN+Tunnel+IP）
    ↓
ACL 数据流（源/目的 IP）→ IPsec 安全提议（ESP/AH + 算法）
                        ↓
IKE 安全提议（PSK + DH 组）→ IKE 对等体（远程地址 + PSK + 交换模式）
                        ↓
IPsec 安全策略（ACL + 提议 + IKE Peer 绑定）→ 应用到 Tunnel 接口
                        ↓（可选）
SET IKEGLOBALCONFIG（DPD + NAT 保活）
```

**1 compound 编排**（9 场景共享）：

| 步骤 | 干啥 | 编排 compound |
|---|---|---|
| 1. IPSec 隧道建立 | VNRS+IPsec 双配 + ACL + 提议 + IKE Peer + 策略 + 应用 + DPD（9 步骨架）| [1-00033](task/UDG/20.15.2/1-00033.md) |
| 0.（VPN 组网场景前置）| APN 接入域基础设施（VPN 实例）| [1-00029](task/UDG/20.15.2/1-00029.md)（仅 VPN 组网场景）|
| 0'。（GRE over IPSec 场景前置）| GRE 隧道封装组播/广播 | [1-00032](task/UDG/20.15.2/1-00032.md)（仅 GRE over IPSec 场景）|

**关键约束**：
- **无需 License**
- **C-U 对称**：UDG/UNC 两侧配置同构（IPFD-015004 ↔ IPFD-016000）
- **UDG 独有 VNRS+IPsec 微服务双配架构**（硬约束）
- **源地址互斥 GRE**：IPSec 隧道源地址不能与 GRE 隧道源地址相同
- **隧道两端路由可达**（IKE 协商前提）
- **DH 组不可为 None**；**PSK 两端必一致**；**ACL 仅源/目的 IP**

## 配置流程

> 每步编排 compound；按场景选变种（变种见决策点）。调测见 [调测 IPSec](evidence/UDG/20.15.2/IPFD-015004-IPSec功能/调测IPsec功能_61317192.md)。

1. **（仅 VPN 组网场景）APN 接入域基础设施**：
   - → 复用 [1-00029](task/UDG/20.15.2/1-00029.md)（VPN 实例；公网场景跳过本步）

2. **（仅 GRE over IPSec 场景）GRE 隧道封装**：
   - → 复用 [1-00032](task/UDG/20.15.2/1-00032.md)（先建 GRE 隧道 Tunnel1 封装组播/广播，IPSec 用 Tunnel2；其他场景跳过本步）
   - 注意：GRE 与 IPSec **源地址必须不同**（LoopBack1 给 GRE，物理接口给 IPSec）

3. **IPSec 隧道建立**（9 步骨架 + 可选 DPD）：
   - → 复用 [1-00033](task/UDG/20.15.2/1-00033.md)（VNRS+IPsec 双配 + ACL + IPsec/IKE 提议 + IKE Peer + IPsec 策略 + 应用 + DPD）
   - 场景变种（按 DP 选）：
     - **普通 IPv4**：ESP+IKEv2+DH group19；步骤 7 NATTRAVERSAL 按需
     - **普通 IPv6（v02+）**：步骤 1/3 改 IPv6 地址族 + ADD IFIPV6ADDRESSIPSEC；UDG 20.8.0+
     - **IPv4 主备**：步骤 7/8 多 IKE Peer + PEERPRIORITY（热备一主一备 / 冷备最多一主两备）
     - **GRE over IPSec**：步骤 2 前置 GRE 隧道；IPSec Tunnel 用 Tunnel2；步骤 4 ACL 匹配 GRE 源/目的（LoopBack IP）
     - **OSPF over IPSec**：OSPF 引流到 IPSec；步骤 4 ACL 匹配 OSPF 组播（需 GRE 封装）
     - **多 Sequence 策略**：步骤 8 同 POLICYNAME 多 SEQUENCENUMBER + 多 ACL
     - **指定本端接口建立**：步骤 7 IKE Peer 指定非 Tunnel 接口为 IKE 协商 IP
     - **国密 IKEv1**：步骤 5/6 算法走国密（SM3/SM4/SM2）+ 步骤 7 IKEv1 + 国密证书

## 决策点

**DP（特性级）：场景 × 协议 × IKE 版本 × 主备 × NAT 穿越 × 走法**

| 场景 | 协议 | IKE 版本 | 主备 | NAT 穿越 | 走法（compound 编排） |
|---|---|---|---|---|---|
| 普通 IPv4 IPSec | ESP | IKEv2（缺省）| 无 | 按需 | [1-00033](task/UDG/20.15.2/1-00033.md) |
| 普通 IPv6 IPSec（v02+）| ESP | IKEv2 | 无 | 不支持 | [1-00033](task/UDG/20.15.2/1-00033.md) IPv6 变种；UDG 20.8.0+ |
| IPv4 IPSec 主备（热备）| ESP | IKEv2 | 热备（一主一备）| 按需 | [1-00033](task/UDG/20.15.2/1-00033.md) + 多 ATTACHIKEPEER + PEERPRIORITY |
| IPv4 IPSec 主备（冷备）| ESP | IKEv2 | 冷备（最多一主两备）| 按需 | [1-00033](task/UDG/20.15.2/1-00033.md) + 多 ATTACHIKEPEER（备用不协商） |
| IPv6 IPSec 主备（v02+）| ESP | IKEv2 | 热/冷备 | 不支持 | IPv6 变种 + 主备 |
| **GRE over IPSec** | ESP | IKEv2 | 无 | 按需 | [1-00032](task/UDG/20.15.2/1-00032.md) GRE 前置 + [1-00033](task/UDG/20.15.2/1-00033.md) IPSec（Tunnel2 + ACL 匹配 LoopBack IP）|
| OSPF over IPSec | ESP | IKEv2 | 无 | 按需 | [1-00033](task/UDG/20.15.2/1-00033.md) + OSPF 引流（OSPF 报文需 GRE 封装）|
| 多 Sequence IPSec 策略 | ESP | IKEv2 | 无 | 按需 | [1-00033](task/UDG/20.15.2/1-00033.md) + 步骤 8 多 SEQUENCENUMBER |
| 指定本端接口建立 IPSec | ESP | IKEv2 | 无 | 按需 | [1-00033](task/UDG/20.15.2/1-00033.md) + 步骤 7 IKE Peer 指定本端接口 |
| 国密 IPSec（IKEv1）| ESP | IKEv1 + 国密 | 无 | 按需 | [1-00033](task/UDG/20.15.2/1-00033.md) 国密变种（SM3/SM4/SM2 + IKEv1 + 国密证书）|
| VPN 组网 IPSec | ESP | IKEv2 | 无 | 按需 | [1-00029](task/UDG/20.15.2/1-00029.md) + [1-00033](task/UDG/20.15.2/1-00033.md) |
| 仅 AH 认证（冷门）| AH | IKEv2 | 无 | 不支持 | [1-00033](task/UDG/20.15.2/1-00033.md) Ah 变种（IPSECPROTOCOL=Ah）|

> **关键规则**：
> - **源地址不能与 GRE 隧道源地址相同**（应用限制硬约束；GRE over IPSec 场景用不同接口规避）
> - **DHGROUP 不能为 None**（建议 Dh_group19）
> - **PSK 两端必须相同**（SA 协商前提）
> - **远程地址不能配置为地址段**
> - **ACL 仅支持源/目的 IP**（端口配置不生效）
> - **NAT 穿越仅 ESP 隧道模式**（不支持 IPv6）
> - **AH+ESP 须同封装模式**
> - **国密算法集**：SM4 加密 / SM3 完整性 / SM2 非对称（国密 IKEv1 场景）

## 约束

- **无需 License**（info）：本特性无需 License。
- **C-U 对称**（info）：UDG/UNC 两侧配置同构；UNC 对应 IPFD-016000（UNC 20.6.0+）。
- **VNRS+IPsec 微服务双配**（critical）：两微服务 Tunnel 接口名/IP/类型/VPN 必须一对一配置 — 后果：业务不通。
- **源地址互斥 GRE**（critical）：IPSec 隧道源地址不能与 GRE 隧道源地址相同 — 后果：应用限制违反。
- **隧道两端路由可达**（critical）：IKE 协商前提（PING 验证）— 后果：隧道 DOWN。
- **DH 组不可为 None**（critical）：DHGROUP 不能为 None 或不配置，建议 Dh_group19 — 后果：SA 协商失败。
- **PSK 两端必一致**（critical）：PRESHAREDKEY 两端相同 — 后果：SA 协商失败。
- **远程地址不可为地址段**（critical）：LOWREMOTEADDR 必须具体 IP — 后果：命令拒收。
- **ACL 仅源/目的 IP**（critical）：端口配置不生效 — 后果：数据流不匹配，隧道不传输。
- **算法两端须一致**（critical）：协议/算法/封装模式两端协商一致 — 后果：SA 协商失败。
- **NAT 穿越约束**（warning）：仅 ESP 隧道模式；不支持 IPv6 组网 — 后果：NAT 场景隧道建立失败。
- **AH+ESP 须同封装模式**（warning）：同时采用时必须相同 ENCAPMOD — 后果：协商失败。
- **IKEv1 安全风险**（warning）：IKEv1 存在密码学漏洞；UDG 缺省 IKEv2 — 后果：弱安全。
- **对系统性能影响**（info）：IPSec 封装+加密运算开销；ACL 优化减少不必要加密。
- **不支持场景**（warning）：GREv6 over IPSecv6、OSPFv3 over IPSecv6、IPSecv6 地址借用、IPV4 报文入 IPSecv6 隧道、IPV6 报文入 IPSecv4 隧道 — 后果：不支持。
- **首次发布版本**（info）：UDG 20.5.0；IPv6/NAT 穿越/主备需 UDG 20.8.0+（v02）。
