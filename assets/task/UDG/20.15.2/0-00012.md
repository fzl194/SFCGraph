# 绑定规则到用户模板（ADD RULEBINDING）

> 命令静态知识见 [ADD RULEBINDING](command/UDG/20.15.2/ADD-RULEBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

ADD RULEBINDING 是过滤链 backbone 的"RULE→USERPROFILE 绑定点"——本命令不做规划决策，只把 [ADD RULE](0-00010.md) 产出的 RULE 与 [ADD USERPROFILE](0-00011.md) 产出的 USERPROFILE 关联起来。本命令的核心配置空间有 3 个维度：**绑定三参数**（三个 reference 引用源）、**POLICYTYPE 取舍**（可选省略条件）、**生效行为**（对新激活用户生效的特征）。

### 配置维度 1：绑定三参数

三个参数全部 reference——本命令不产生独立规划决策：

| 参数 | 取值来源 | 引用源 atom | 备注 |
|---|---|---|---|
| `USERPROFILENAME` | 引 [0-00011](0-00011.md) ADD USERPROFILE 产出 | task/UDG/20.15.2/0-00011.md | 必选 |
| `RULENAME` | 引 [0-00010](0-00010.md) ADD RULE 产出 | task/UDG/20.15.2/0-00010.md | 必选；RULENAME+POLICYTYPE 组合唯一 |
| `POLICYTYPE` | 继承 [0-00010](0-00010.md) ADD RULE.POLICYTYPE | task/UDG/20.15.2/0-00010.md | 条件可选 |

> **POLICYTYPE 可省略场景**：仅修改 UP/RULE 的场景（如用户 Portal、Web Proxy 单独加规则）下，ADD RULEBINDING 可只传 RULENAME，由系统按 RULE 自身策略类型自动匹配。

### 配置维度 2：POLICYTYPE 取舍

| 场景 | 是否必传 POLICYTYPE | 说明 |
|---|---|---|
| 计费族、PCC、ADC 等多策略类型共用 UP | **必传** | RULE 与策略类型组合唯一，避免歧义 |
| 单一策略类型 UP（仅 PCC 或仅 HEADEN） | 可省 | 系统按 RULE 自身类型自动匹配 |
| Portal/Web Proxy 单加规则 | 可省 | 见证据（Portal/Web Proxy 的 RULEBINDING 仅传 RULENAME） |

### 配置维度 3：生效行为

**配置生效特征**：本命令**只对新激活用户生效**——已激活用户不受影响：

| 影响范围 | 行为 |
|---|---|
| 新激活用户 | 绑定的 RULE 立即对该用户生效 |
| 已有用户 | **不受影响**——必须配合 [SET REFRESHSRV](0-00015.md) 刷新或用户承载更新 |

> 故配置流程必须：先 ADD/DELETE RULE + ADD RULEBINDING 修改完成，最后 `SET REFRESHSRV: REFRESHTYPE=USERPROFILE;`（或 ALL）使新增绑定对存量用户生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 7：绑定规则到用户模板"——ADD USERPROFILE + ADD RULE 之后，SET REFRESHSRV 之前。

## 决策点：POLICYTYPE 是否显式传入（DP 0-00007）

问：ADD RULEBINDING 是否显式传 POLICYTYPE？

| 选项 | POLICYTYPE 必填性 | 适用场景 |
|---|---|---|
| `explicit_politype`（默认） | 必填 | 多策略类型共用 UP 的场景（计费族 + PCC + ADC 共存），按 RULE+POLICYTYPE 唯一索引避免歧义 |
| `omit_politype` | 可省 | 单一策略类型 UP；仅修改 UP/RULE 单独加规则场景（用户 Portal/Web Proxy 单加规则的证据场景） |

## 约束在所属 feature 编排中通常位于"步骤 7：绑定规则到用户模板"——ADD USERPROFILE + ADD RULE 之后，SET REFRESHSRV 之前。

## 约束

- **RULEBINDING 最大记录数规格**（warning，rule-0-00133）：最大记录数 200000；超 90% 上报 ALM-135602215（阈值 `MOD CFGTHRESHOLD（★R1.5 atom 待补）` 修） — 后果：超规格新增失败；接近阈值时告警
- **单 USERPROFILE 绑定上限**（warning，隐含）：默认规则 UP（UPSPECTYPE=DEFAULT）单 UP 最多 8000 个规则；规格受限 UP（SPECIFICATION_LIMITED）上限更高（见命令 wiki） — 后果：超规格绑定失败
- **只对新激活用户生效**（critical，隐含）：本命令绑定后**已激活用户不受影响**；需配合 [SET REFRESHSRV](0-00015.md) 刷新 — 后果：不刷新致存量用户绑定未生效
- **一致性要求**（critical，隐含）：USERPROFILENAME/RULENAME/POLICYTYPE 在 SMF/UPF（或 PGW-C/PGW-U）必须保持一致 — 后果：策略无法按预期生效
