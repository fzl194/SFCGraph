# 绑定规则到用户模板（ADD RULEBINDING）

> 命令静态知识见 [ADD RULEBINDING](command/UDG/20.15.2/ADD-RULEBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

三个参数全部 reference——本命令不做规划决策，只把上游产出的 USERPROFILE 与 RULE 关联。POLICYTYPE 有条件可省。

**典型范式（计费 4 场景 + 内容计费，带 POLICYTYPE）**：
```
ADD RULEBINDING: USERPROFILENAME="userprofile001", RULENAME="Rule001_dns", POLICYTYPE=PCC;
ADD RULEBINDING: USERPROFILENAME="userprofile001", RULENAME="Rule003_any", POLICYTYPE=PCC;
ADD RULEBINDING: USERPROFILENAME="userprofile002", RULENAME="Rule002_IMS";
```

**PCC 场景（动态 / 本地 PCC，仅 USERPROFILENAME + RULENAME）**：
```
ADD RULEBINDING: USERPROFILENAME="up_test", RULENAME="rule_test";
```

**取值约定（全 reference，非本命令规划）**：

| 参数 | 来源 | 省略条件 |
|---|---|---|
| USERPROFILENAME | 引 [0-00011](0-00011.md) ADD USERPROFILE 产出 | — |
| RULENAME | 引 [0-00010](0-00010.md) ADD RULE 产出 | — |
| POLICYTYPE | 继承 [0-00010](0-00010.md) POLICYTYPE | 用户 Portal / Web Proxy 等仅改 UP/Rule 的场景可省（证据③此类 RULEBINDING 仅传 RULENAME） |

**一致性要求**：USERPROFILE 下绑定的 RULENAME/POLICYTYPE 须与 SMF/UPF（或 PGW-C/PGW-U）保持一致，否则策略无法按预期生效。

## 约束

- **rule-0-00133**（warning）规格：最大记录数 **200000**，超 90% 上报 ALM-135602215（阈值 `MOD CFGTHRESHOLD` 修）。单 UP 绑定上限：默认规则 UP（UPSPECTYPE=DEFAULT）最多 **8000** 个规则；规格受限 UP（SPECIFICATION_LIMITED）上限更高（见命令 wiki）。生效行为：**只对新激活用户生效**——已激活用户不受影响，故须配合 [SET REFRESHSRV](0-00015.md) 刷新并对新用户生效。
