# 配置TWAMP发送端Light模式（ADD TWAMPSENDER）

> 命令静态知识见 [ADD TWAMPSENDER](command/UDG/20.15.2/ADD-TWAMPSENDER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110921（TWAMP）**Light 模式独有**的第 5 步——配置探测路径对应的报文参数。Light 模式下 UDG 作发送端，依附于已建客户端（ADD TWAMPCLIENT）。Full 模式不走此命令（Full 时 UDG 作响应端，由对端发起探测）。CLIENTID 引用上游 ADD TWAMPCLIENT（0-00227）已建客户端。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| CLIENTID | 客户端 ID | `1` | **引用**上游 [配置TWAMP客户端Light模式](task/UDG/20.15.2/0-00227.md) 的 CLIENTID（必填，1~12000） |
| TIMEOUT | 探测超时秒 | `5` | 全网规划，整数 1~10 秒，默认 5（可选） |

**脚本样例**

```
ADD TWAMPSENDER:CLIENTID=1,TIMEOUT=5;
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：MODE/DSCP/PKTSIZETYPE/PKTSIZE/PADDINGTYPE/PERIOD 六参均为 optional/conditional，命令真相均有默认值（MODE=CONTINUAL、DSCP=0、PKTSIZETYPE=FIXED、PKTSIZE=128、PADDINGTYPE=PADDING_RANDOM、PERIOD=100），特性数据规划表与示例脚本均未涉及，现网按需后续补绑。

**操作步骤定位**：紧接 ADD TWAMPCLIENT 之后、SET LINKALMCFG（0-00230，链路丢包告警阈值）之前。Light 模式互斥由 feature-DP 0-00036（TWAMP 模式 LIGHT）承担。
