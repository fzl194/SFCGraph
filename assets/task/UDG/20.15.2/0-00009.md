# 绑定七层过滤条件到流过滤器（ADD PROTBINDFLOWF）

> 命令静态知识见 [ADD PROTBINDFLOWF](command/UDG/20.15.2/ADD-PROTBINDFLOWF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

把七层 L7FILTER 连同协议名绑到 [ADD FLOWFILTER](0-00007.md) 上，完成"七层维度"的过滤绑定（与 [ADD FLTBINDFLOWF](0-00008.md) 的三四层绑定互补，一个 FLOWFILTER 通常两者都配）。

**典型范式**（证据③，PROTOCOLNAME 与 SUBL7FLTNAME 命名呼应）：
```
ADD PROTBINDFLOWF: FLOWFILTERNAME="ff_L7", PROTOCOLNAME="dns",  L7FILTERNAME="L7_URL";  # DNS 计费
ADD PROTBINDFLOWF: FLOWFILTERNAME="ff_L7", PROTOCOLNAME="http", L7FILTERNAME="L7_URL";  # HTTP 流量/时长计费
```

**对接约定**：
- FLOWFILTERNAME 引 0-00007 产出的流过滤器
- L7FILTERNAME 引 0-00006 产出的七层过滤器（统一 `L7_URL`）
- PROTOCOLNAME 本端规划，常见取值 `dns`/`http`；个别特性取 `icloudprivaterelay`（HTTP3.0，2-00051）

## 约束

- **rule-0-00125**（critical）执行后 **60s 固定延时生效**（非 REFRESHSRV 刷新）；最大记录数 **100000**，超 90% 上报 ALM-135602215（阈值 `MOD CFGTHRESHOLD` 修）；单 FlowFilter 最多 **32 个 Protocol**（PROTOCOLNAME）。
