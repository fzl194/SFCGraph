# 绑定七层过滤条件到流过滤器（ADD PROTBINDFLOWF）

> 命令静态知识见 [ADD PROTBINDFLOWF](command/UDG/20.15.2/ADD-PROTBINDFLOWF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD PROTBINDFLOWF 是过滤链 backbone 的"七层绑定原子"：把七层 L7FILTER 连同协议名（PROTOCOLNAME）绑到 ADD FLOWFILTER 产出的流过滤器上，完成"七层维度"的过滤绑定。与 [ADD FLTBINDFLOWF](task/UDG/20.15.2/0-00008.md) 的三四层绑定互补——一个 FLOWFILTER 通常两者都配。本命令的核心配置空间有 3 个维度：**协议名取值**（按协议语义命名 PROTOCOLNAME）、**L7FILTER 引用维度**（引上游已配置数据）、**一对多绑定关系**（单 FlowFilter 可绑 32 个 Protocol）。

### 配置维度 1：协议名取值（参数 PROTOCOLNAME，本端规划）

PROTOCOLNAME 与上游 [ADD L7FILTER](task/UDG/20.15.2/0-00006.md) 的 SUBL7FLTNAME 命名呼应：

| 命名风格 | 典型取值 | 适用场景 |
|---|---|---|
| 按协议（默认/高频） | `dns`（DNS）、`http`（HTTP） | 计费 DNS/HTTP 时长/流量（绝大多数场景） |
| 边缘协议名 | `icloudprivaterelay`（HTTP3.0） | 个别特性（2-00051），属协议名取值而非来源分叉 |

### 配置维度 2：L7FILTER 引用维度（参数 L7FILTERNAME，引上游已配置数据）

| 取值 | 引用的源 atom | 典型七层过滤器名 |
|---|---|---|
| **统一 L7_URL**（默认） | [ADD L7FILTER](task/UDG/20.15.2/0-00006.md) | `L7_URL`（统一命名） |

### 配置维度 3：一对多绑定关系（参数 FLOWFILTERNAME，引上游已配置数据）

| 关系 | 数量约束 |
|---|---|
| **单 Protocol → 单 FLOWFILTER** | 单 FlowFilter 最多 **32 个** Protocol（PROTOCOLNAME） |
| **多 Protocol → 同一条 FLOWFILTER** | 按需绑定 1~32 个 PROTOCOLNAME |

**典型范式**（PROTOCOLNAME 与 SUBL7FLTNAME 命名呼应）：
```
ADD PROTBINDFLOWF: FLOWFILTERNAME="ff_L7", PROTOCOLNAME="dns",  L7FILTERNAME="L7_URL";  # DNS 计费
ADD PROTBINDFLOWF: FLOWFILTERNAME="ff_L7", PROTOCOLNAME="http", L7FILTERNAME="L7_URL";  # HTTP 流量/时长计费
```

**对接约定**：
- `FLOWFILTERNAME` → 引 [ADD FLOWFILTER](task/UDG/20.15.2/0-00007.md) 产出的流过滤器名
- `L7FILTERNAME` → 引 [ADD L7FILTER](task/UDG/20.15.2/0-00006.md) 产出的七层过滤器（统一 `L7_URL`）
- `PROTOCOLNAME` → 本端规划，常见取值 `dns`/`http`

**步骤位置**：在所属 feature 编排中通常位于"步骤 5：流过滤器与过滤条件绑定"——ADD FILTER/L7FILTER + ADD FLOWFILTER 之后，与 ADD FLTBINDFLOWF 互补。一个 FlowFilter 可多次执行本命令绑不同 PROTOCOLNAME。

## 决策点

无。本命令在所有特性下用法一致，无分支（PROTOCOLNAME 协议名取值不同属取值差异，非来源分叉；L7FILTERNAME 统一引用上游 L7_URL）。

## 约束

- **ADD PROTBINDFLOWF 延时生效与记录数上限**（critical，rule-0-00125）：执行后 **60s 固定延时生效**（非 SET REFRESHSRV 刷新）；PROTBINDFLOWF 最大记录数 **100000**，超规格 90% 上报 ALM-135602215；单个 FlowFilter 最多配置 **32 个** Protocol（PROTOCOLNAME） — 后果：配置延时生效 / 容量超限告警 / 协议数超限
- **FLOWFILTERNAME 与 L7FILTERNAME 必须已配置**（critical，隐含）：本命令仅做引用对接——FLOWFILTERNAME 必须已 ADD FLOWFILTER 配置；L7FILTERNAME 必须已 ADD L7FILTER 配置 — 后果：未配置则绑定失败
