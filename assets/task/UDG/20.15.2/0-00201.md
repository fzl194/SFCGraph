# 增加流分类(MQC)（ADD MQCCLASSIFIER）

> 命令静态知识见 [ADD MQCCLASSIFIER](command/UDG/20.15.2/ADD-MQCCLASSIFIER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

MQCCLASSIFIER 是 CB 对的 **C 侧**——本命令建分类容器，ACL 匹配规则由后续 [ADD QOSRULEACL](0-00202.md) 绑入。

**数据规划取值样例**（证据③，IPFD-012001 数据规划表）：

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| CLASSIFIERNAME（分类名称） | `class1` | 本端规划 | 唯一标识流分类 |
| OPERATOR（关系类型） | `or` | 本端规划 | optional，默认 `or`（脚本未显式传参取默认） |

**典型范式**（证据③任务示例脚本，脚本只传名称，OPERATOR 取默认 or）：
```
ADD MQCCLASSIFIER: CLASSIFIERNAME="class1";
```

> OPERATOR 决定多条 ACL 规则匹配时的逻辑关系（or=或，and=与）。主场景单 ACL 规则组，逻辑关系不关键，取默认即可。

## 约束

- **rule-0-00403**（warning）最大记录数 **10239**；执行后立即生效；为空分类容器，ACL 匹配规则经 [ADD QOSRULEACL](0-00202.md) 绑入。
