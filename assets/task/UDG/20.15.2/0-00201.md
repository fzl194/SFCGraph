# 增加流分类（ADD MQCCLASSIFIER）

> 命令静态知识见 [ADD MQCCLASSIFIER](command/UDG/20.15.2/ADD-MQCCLASSIFIER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。MQC 复杂流分类三件套（Class-Behavior-Policy）骨架：**本命令建 Classifier 空容器**，ACL 匹配规则由后续子命令绑入。

## 配置方法（配置方法字典）

ADD MQCCLASSIFIER 是"增加流分类"命令。**仅 1 参必选**（CLASSIFIERNAME）+ 1 参可选（OPERATOR）。Classifier 是**空容器**，本命令只命名（OPERATOR 决定多条规则间逻辑）。1 个特性命中（IPFD-012001 QoS 复杂流分类）覆盖唯一典型范式（OPERATOR 隐式默认 or）。

### 配置维度 1：分类名称（参数 CLASSIFIERNAME，必选）

| 取值 | 约束 | 适用场景 |
|---|---|---|
| **字符串 1~31** | 不允许为系统预定义的 `default-class` | 本端规划；IPFD-012001 范例 `class1` |

### 配置维度 2：规则关系类型（参数 OPERATOR，可选）

| 取值 | 默认 | 作用 |
|---|---|---|
| **or**（默认） | ✓ | 多条 ACL 规则匹配时取"或"（任一规则匹配即命中分类） |
| **and** | — | 多条 ACL 规则匹配时取"与"（全部规则匹配才命中分类） |

> **OPERATOR 仅在 Classifier 下挂多条规则时有意义**；主场景单 ACL 规则组（IPFD-012001 范例），逻辑关系不关键，取默认即可。

**典型脚本（IPFD-012001 QoS 复杂流分类，建空 class1）**：
```
ADD MQCCLASSIFIER: CLASSIFIERNAME="class1";
ADD QOSRULEACL: CLASSIFIERNAME="class1", IPVERSION=IPv4, ACLTYPE=acl-name, ACLNAME="acl1";
```

> 命令 notes：**最大记录数 10239**；**执行后立即生效**；CLASSIFIERNAME 不允许 `default-class`（系统预定义）；ACL 匹配规则经 [0-00202](task/UDG/20.15.2/0-00202.md) ADD QOSRULEACL 绑入；ACL 规则组由 [0-00197](task/UDG/20.15.2/0-00197.md) ADD ACLGROUP + [0-00198](task/UDG/20.15.2/0-00198.md) ADD ACLRULEADV4 配套。

**步骤位置**：在所属 feature 编排中位于"步骤 N：建 Classifier 容器"——前置 无；下游 [0-00202](task/UDG/20.15.2/0-00202.md) ADD QOSRULEACL（绑 ACL）；[0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY（CB 节点绑定 CLASSIFIERNAME）。

## 决策点：OPERATOR 选择（DP 0-00201）

问：当 Classifier 下挂多条 ACL 规则时，规则间逻辑关系用 or 还是 and？

| 选项 | OPERATOR | 命中条件 | 适用场景 |
|---|---|---|---|
| **或关系（默认）** | or | 任一规则匹配即命中分类 | 多选一流量分类（IPFD-012001 范式） |
| **与关系** | and | 全部规则匹配才命中分类 | 多维精确匹配（如 IP+端口双重过滤） |

> 1 个特性命中（IPFD-012001）取默认 or；and 仅在多维匹配场景下使用。

## 约束

- **最大记录数 10239**（critical，rule-0-00403）：整机 Classifier 最大 10239 条 — 后果：超规格新增失败
- **执行后立即生效**（warning，rule-0-00403）：执行后立即生效 — 后果：误操作即刻生效
- **CLASSIFIERNAME 不能为 `default-class`**（critical，rule-0-00403）：系统预定义分类名 `default-class` 不允许重复定义 — 后果：命名冲突命令拒收
- **空分类容器**（info，rule-0-00403）：本命令仅创建空 Classifier；ACL 匹配规则须由 [0-00202](task/UDG/20.15.2/0-00202.md) ADD QOSRULEACL 绑入 — 后果：不绑规则的 Classifier 无匹配效果
- **C 侧须在 CB 节点绑定（0-00203）前先建**（warning，rule-0-00403）：CLASSIFIERNAME 必须在 [0-00203](task/UDG/20.15.2/0-00203.md) ADD MQCPOLICY 引用前先建 — 后果：未建引用失败
- **CLASSIFIERNAME 长度**（info，rule-0-00403）：字符串 1~31 — 后果：超长命令拒收
