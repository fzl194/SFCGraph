# 添加一个新的LAC组（ADD LACGROUP）

> 命令静态知识见 [ADD LACGROUP](command/UDG/20.15.2/ADD-LACGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。LACGROUPNAME 是 LAC 号段聚合容器，引用关系：被 [0-00311](task/UDG/20.15.2/0-00311.md) ADD LACID 的 LACGROUPNAME 引用；被 [0-00047](task/UDG/20.15.2/0-00047.md) ADD POOLGRPMAP 经 POOLGRPNAME→地址池组映射间接关联。

## 配置方法（配置方法字典）

ADD LACGROUP 是"新建 LAC 组"命令。1 个必选参数（LACGROUPNAME），命令本身配置空间极简——核心是**LAC 组名**（LACGROUPNAME 字符串规划）。仅 1 个配置维度：**LAC 组名**（1~32 字符，不支持空格，不区分大小写，本端规划）。

### 配置维度 1：LAC 组名（参数 LACGROUPNAME，必选）

| 取值形式 | 约束 | 引用关系 |
|---|---|---|
| **字符串 1~32** | 不支持空格；不区分大小写；本端规划 | 被 [0-00311](task/UDG/20.15.2/0-00311.md) ADD LACID 的 LACGROUPNAME 引用，作为 LAC 号段聚合容器 |

> **LAC 组定位**：LACGROUPNAME 是 LAC 号段聚合的命名容器，本身不绑定 LAC 号段——号段绑定由 [0-00311](task/UDG/20.15.2/0-00311.md) ADD LACID 在已建 LAC 组内完成。本命令仅做"建组"动作，命令配置空间极简，仅 1 个必选参数。

**典型脚本（GWFD-020406 IPv6 Prefix Delegation 基于位置分配地址，范例 lac1）**：
```
ADD LACGROUP:LACGROUPNAME="lac1";
```

**典型脚本（GWFD-020421 基于位置的地址分配，范例 lac1）**：
```
ADD LACGROUP:LACGROUPNAME="lac1";
```

> 命令 notes：**对新激活用户生效**：该命令执行后只对新激活用户生效，老用户不感知；**最大记录数 3000**：单网元 LAC 组上限 3000 条；**生效方式**：执行后不实时刷新所有会话，仅作用于后续激活。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：建 LAC 组容器"——前置 **无**（建组是链路起点）；下游 [0-00311](task/UDG/20.15.2/0-00311.md) ADD LACID（在已建 LAC 组内绑定 LAC 号段）；关联 [0-00047](task/UDG/20.15.2/0-00047.md) ADD POOLGRPMAP（地址池组映射，将 LAC 组→APN→地址池关联起来）。

## 决策点

（无。本命令在所有特性下用法一致——仅 1 个必选参数 LACGROUPNAME，无分支决策。）

> GWFD-020406 IPv6 Prefix Delegation 与 GWFD-020421 基于位置的地址分配两个特性均直接传 `LACGROUPNAME="lac1"` 即可，命令本身无维度差异；特性差异由下游 [0-00311](task/UDG/20.15.2/0-00311.md) ADD LACID 的 LAC 号段绑定体现。

## 约束

- **最大记录数 3000**（warning，rule-0-00310）：单网元 LAC 组最大记录数 3000 条 — 后果：超规格新增失败
- **对新激活用户生效**（warning，rule-0-00310）：该命令执行后只对新激活用户生效 — 后果：老用户不感知，仅后续激活用户受影响
- **LACGROUPNAME 长度**（info，rule-0-00310）：字符串 1~32 — 后果：超长命令拒收
- **LACGROUPNAME 不支持空格**（warning，rule-0-00310）：字符串形式不支持空格 — 后果：含空格命令拒收
- **LACGROUPNAME 不区分大小写**（info，rule-0-00310）：大小写不敏感，写入统一处理 — 后果：大小写不同的同名组视为同一组
- **权限要求**（info，rule-0-00310）：需 G_1 管理员 / G_2 操作员级别 — 后果：低权限用户命令拒收
