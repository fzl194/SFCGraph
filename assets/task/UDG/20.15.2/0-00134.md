# 绑定APN到L2TP源端接口(AAA下发方式专属)（ADD L2TPRDSCLIENT）

> 命令静态知识见 [ADD L2TPRDSCLIENT](command/UDG/20.15.2/ADD-L2TPRDSCLIENT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令用于 **AAA 下发方式激活 L2TP VPN** 场景（GWFD-020412）：把 APN 绑到指定源端 Gi 接口，AAA 返回 L2TP 属性时系统以此接口作源端与 LNS 交互。是 GWFD-020412-AAA 流程步骤 7。本地配置方式用 [0-00135](task/UDG/20.15.2/0-00135.md) 绑 L2TP 组，不用本命令。

**配置范式**

| 场景 | 参数 | 典型行 |
|---|---|---|
| AAA 下发方式绑源端接口（GWFD-020412） | APN + INTERFACENAME（均引用） | `ADD L2TPRDSCLIENT:APN="apn-l2tp",INTERFACENAME="giif1/0/1";` |

**参数取值约定**

| 参数 | 取值来源 | 说明 |
|---|---|---|
| APN | 引用 [0-00020](task/UDG/20.15.2/0-00020.md) 已建 APN 名称（字符串 1~63） | 每 APN 只能绑 1 个源端接口（唯一约束） |
| INTERFACENAME | 引用 ADD LOGICINF.NAME 的 Giif 接口名（字符串 1~31） | 源端 Gi 接口，须先 [[UDG@20.15.2@Task@0-00136]] 建好 |

两个参数均 `reference` 绑定，取值来自已配置数据，本命令本身无本端规划值。

## 约束

- **rule-0-00368**（warning）系统最多 10000 条 L2tpRdsClient；每 APN 唯一绑 1 源端接口；立即生效；前置须先 ADD APN + ADD LOGICINF；适用 PGW-U/UPF — 后果：超限或重复绑定新增失败、缺前置则无引用源
