# 增加APN绑定的L2TP接口(AAA下发方式专属)（ADD L2TPRDSCLIENT）

> 命令静态知识见 [ADD L2TPRDSCLIENT](command/UDG/20.15.2/ADD-L2TPRDSCLIENT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**适用 NF：PGW-U、UPF**。

## 配置方法

ADD L2TPRDSCLIENT 在指定 APN 下绑定一个源端 Gi 接口，作为 AAA 下发方式下系统与 LNS 交互时的本地源端。**APN 必选**（1~63 字符）、**INTERFACENAME 必选**（1~31 字符）。本命令是 **AAA 下发方式**专属源端接口绑命令——本地配置方式下应在 [0-00133](task/UDG/20.15.2/0-00133.md) ADD L2TPGROUP 上用 [0-00135](task/UDG/20.15.2/0-00135.md) ADD L2TPCLIENTIP 绑源端。核心配置空间有 2 个维度：**APN 定位**（APN）、**源端接口定位**（INTERFACENAME）。1 个特性命中（GWFD-020412 L2TP VPN AAA 方式）APN=apn-l2tp + INTERFACENAME=giif1/0/1。

### 配置维度 1：APN 定位（参数 APN，必选）

| 取值 | 作用 |
|---|---|
| **已建 APN 实例名**（ref [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN + [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR 中 L2TPSWITCH=ENABLE） | 源端接口归属的 APN（020412 AAA 范式 `apn-l2tp`） |

> APN 必须已配置 L2TPSWITCH=ENABLE；同 APN 下可绑多个源端 Gi 接口；该 APN 必须是 PPP 接入 APN。

### 配置维度 2：源端接口定位（参数 INTERFACENAME，必选）

| 取值 | 作用 |
|---|---|
| **已建的源端 Gi 接口名**（ref [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE） | 系统与 LNS 交互时的源端；020412 AAA 范式 `giif1/0/1` |

> 接口名 1~31 字符；同一 APN 下可绑多个 Gi 接口；同一 (APN, INTERFACENAME) 不可重复添加。

**典型脚本（020412 L2TP VPN AAA 下发方式，apn-l2tp 绑 giif1/0/1）**：
```
ADD L2TPRDSCLIENT:APN="apn-l2tp",INTERFACENAME="giif1/0/1";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：APN 下绑源端接口"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN + [0-00132](task/UDG/20.15.2/0-00132.md) SET APNL2TPATTR（含 L2TPSWITCH=ENABLE）+ [0-00131](task/UDG/20.15.2/0-00131.md) SET GLOBALL2TP + [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 建源端接口；后续 [0-00139](task/UDG/20.15.2/0-00139.md) SET APNPPPACCESS + [0-00138](task/UDG/20.15.2/0-00138.md) SET PPPCFG。

## 决策点：L2TP 源端接口绑定粒度（DP 0-00134）

问：本 L2TP 源端接口是绑在 APN 还是绑在 L2TP 组上？

| 选项 | 命令 | 绑定粒度 | 适用场景 |
|---|---|---|---|
| **APN 绑定**（020412 AAA 范式） | ADD L2TPRDSCLIENT（本 atom） | APN | AAA 下发方式，源端由 APN 决定 |
| **L2TP 组绑定** | ADD L2TPCLIENTIP（[0-00135](task/UDG/20.15.2/0-00135.md)） | GROUPID | 本地配置方式，源端由组决定 |

> 1 个特性命中（020412 AAA 范式）；两种命令互斥，不可在同一 L2TP 实例中混用。

## 约束

- **APN 必须已建且 L2TP 已使能**（critical，rule-0-00134）：APN 必须已通过 ADD APN 创建，并已通过 SET APNL2TPATTR L2TPSWITCH=ENABLE 使能 L2TP — 后果：APN 不存在或 L2TP 未使能时命令拒收
- **INTERFACENAME 必须已建**（critical，rule-0-00134）：INTERFACENAME 必须已通过 ADD INTERFACE 创建 — 后果：接口不存在命令拒收
- **(APN, INTERFACENAME) 联合唯一**（critical，rule-0-00134）：同一 APN 下同一 Gi 接口不可重复添加 — 后果：重复添加命令拒收
- **APN 名称约束**（info，rule-0-00134）：APN 1~63 字符，不支持空格与特殊字符 — 后果：超长或含特殊字符命令拒收
- **适用 NF**（info，rule-0-00134）：本命令仅适用 PGW-U、UPF — 后果：误在 SGW-U 上执行命令拒收
- **与本地方式命令互斥**（warning，rule-0-00134）：同一 L2TP 实例下不应同时使用本命令与 ADD L2TPCLIENTIP — 后果：源端绑定混乱
- **执行后立即生效**（warning，rule-0-00134）：本命令执行后立即生效 — 后果：误配即刻影响该 APN 下用户 L2TP 源端
