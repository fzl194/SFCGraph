# 增加ICAP本端信息（ADD ICAPLOCALINFO）

> 命令静态知识见 [ADD ICAPLOCALINFO](command/UDG/20.15.2/ADD-ICAPLOCALINFO.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务配置 UDG 本端与 ICAP Server 交互时的标识信息（User Agent）——告诉 ICAP Server 是谁在发起 URL 分类查询。`ICAPSERVERTYPE` 引用上游 ICAP 服务器类型，与 [0-00259](task/UDG/20.15.2/0-00259.md) ADD ICAPSERVER 保持一致（本特性统一为 URL_FILTERING）；`USERAGENT` 为本端规划字符串。适用 NF：PGW-U、UPF。

**取值样例（数据规划表，主特性 GWFD-110471）**

| 参数 | 含义 | 样例值 | 获取方法 |
|---|---|---|---|
| ICAPSERVERTYPE | ICAP服务器类型 | URL_FILTERING | 引用 [0-00259](task/UDG/20.15.2/0-00259.md) 已配数据（跨 atom 一致性：URL_FILTERING） |
| USERAGENT | User Agent | test | 本端规划（字符串 1～63，区分大小写） |

**主场景脚本样例**

```
ADD ICAPLOCALINFO:ICAPSERVERTYPE=URL_FILTERING, USERAGENT="test";
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：CFGDOMAINNAME（配置域名称，optional，主场景不配）。

## 决策点

（本命令在所有引用特性下用法一致——ICAPSERVERTYPE + USERAGENT 两必选，本端标识全局唯一，没有跨特性分叉，未建 DP。）

## 约束

- **命令记录上限**（critical，rule-0-00262-01）：命令最大记录数 **1**（全局唯一本端标识），超出被拒。
- **前置依赖**（critical，rule-0-00262-02）：必须先配置 [0-00259](task/UDG/20.15.2/0-00259.md) ADD ICAPSERVER（ICAPSERVERTYPE 引用源），否则报服务器类型不存在。
- **跨 atom 一致性**（critical，rule-0-00262-03）：ICAPSERVERTYPE 取值必须与 [0-00259](task/UDG/20.15.2/0-00259.md) ADD ICAPSERVER 已配的 ICAPSERVERTYPE 完全一致（URL_FILTERING）。
- **字符串约束**（critical，rule-0-00262-04）：USERAGENT 字符串 1～63 字符，**区分大小写**。
- **生效时机**（warning，rule-0-00262-05）：命令执行后**立即生效**。
