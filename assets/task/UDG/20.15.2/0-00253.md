# 增加内容过滤模板（ADD CFTEMPLATE）

> 命令静态知识见 [ADD CFTEMPLATE](command/UDG/20.15.2/ADD-CFTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务创建一个内容过滤模板，把模板绑定到主用 CONTENT_FILTERING 类型的 ICAP 服务器组并指定缺省动作（ICAP 不可达时的兜底行为）。模板随后供 SET APNCFTEMPLATE（APN 维度引用）和 ADD CFPROFBINDCFT（绑定策略）使用。适用 NF：PGW-U、UPF。

**取值样例（数据规划表，GWFD-110471）**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| CFTEMPLATENAME | `icaptemplate` | 本端规划 | 模板名，供 SET APNCFTEMPLATE / ADD CFPROFBINDCFT 引用 |
| ICAPSRVGMNAME | `icapsvrgrp1` | 引用 ADD ICAPSVRGRP | 主用 ICAP 服务器组（必选，"已配置数据中获取"） |
| ACTION | `BLOCK` | 本端规划 | ICAP 不可达时丢弃报文（主特性样例） |

**脚本样例（主特性 BLOCK 路径）**

```
ADD CFTEMPLATE: CFTEMPLATENAME="icaptemplate",ICAPSRVGMNAME="icapsvrgrp1", ACTION=BLOCK;
```

**未绑参数说明（按 baseline §4.1/§4.3 冷门不绑）**：ICAPSRVGSNAME（备用 ICAP 服务器组，optional，主特性不配主备切换）、TIMELIMITNUM（响应超时限制次数，默认 100）、RESPTIMEOUT（响应超时 ms，默认 300）、CFGDOMAINNAME（配置域，optional）——主特性场景采用系统默认。DEFREDNAME 见决策点。

## 决策点：CFTEMPLATE 缺省动作分叉 PASS/BLOCK/REDIRECT（DP 0-00118）

问：ADD CFTEMPLATE 的 ACTION 取哪个分支？三路互斥（命令真相枚举，默认 PASS）。

| 选项 | ACTION | DEFREDNAME 联动 | 说明 |
|---|---|---|---|
| 缺省动作 PASS（默认） | `PASS`（不显式传） | absent | ICAP 不可达时转发报文 |
| 缺省动作 BLOCK（主特性样例） | `BLOCK` | absent | ICAP 不可达时丢弃报文 |
| 缺省动作 REDIRECT | `REDIRECT` | **required**（指向 ADD REDIRECT 已建实例） | ICAP 不可达时重定向 |

REDIRECT 路径需先建 ADD REDIRECT 实例（atom 待建），DEFREDNAME 引用其重定向名。

## 约束

- **rule-0-00315**（warning）单命令最大记录数 **100** + 整机最多 **100** 个内容过滤模板实例（两上限等价同源），超限 ADD 失败；执行后立即生效；ICAPSRVGMNAME 引用 ADD ICAPSVRGRP（atom 0-00260），必选前置；ICAPSRVGSNAME 可选（主用故障切换备用）；DEFREDNAME 仅 ACTION=REDIRECT 时必选，其余路径不配。
