# 配置外联口自动部署服务模板（ADD AUTOSCALINGSERVICE）

> 命令静态知识见 [ADD AUTOSCALINGSERVICE](command/UDG/20.15.2/ADD-AUTOSCALINGSERVICE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是路由交叉功能（GWFD-020423）网络 B 侧外联口自动化部署的基础模板，被后续静态路由（[0-00142](task/UDG/20.15.2/0-00142.md)）、BFD 单臂（[0-00143](task/UDG/20.15.2/0-00143.md)）/双向（[0-00144](task/UDG/20.15.2/0-00144.md)）检测模板的 SERVICENAME 引用。一个服务模板按地址族独立配置——IPv4v6 双栈需分别执行该命令两次（各配 IPv4 / IPv6 一次）。接口类型由组网方式决定。

**配置范式（按组网方式 / 地址族）**

| 维度 | 取值 | 说明 |
|---|---|---|
| SERVICENAME | 如 `ServName_networkB` | 本端规划，1~31 位字符串，供下游 SRROUTE/BFD 引用 |
| VPNNAME | `VRF_networkB`（引用 [0-00071](task/UDG/20.15.2/0-00071.md).VRFNAME） | 网络侧 VRF 实例，已规划数据获取 |
| AFTYPE | `IPv4`（双栈时再配一次 `IPv6`） | 地址族，决定用 IPv4/IPv6 参数族 |
| IPALLOCTYPE4 | `USER_CONFIG` | IPv4 地址分配方式（③脚本 SR-IOV bonding 示例显式值） |
| AUTOCFGIFTYPE | `ETHTRUNK`（SR-IOV bonding）/ `VNIC`（非 bonding） | 接口类型，决定下游 ETHTRUNKTMPID 或 VNICID |
| ETHTRUNKTMPID | 如 `6` | 仅 AUTOCFGIFTYPE=ETHTRUNK 时填，`LST AUTOSCALINGETHTRUNK` 查询获取 |
| VLANID | 如 `20` | 全网规划，0~4094 |
| BEGINADDR4 / ENDADDR4 | 如 `192.168.20.2` / `192.168.20.126` | 全网规划（参见 5G Core LLD 业务IP对接设计起始/结束IP），与网关互通的外联口IP池 |
| MASKLEN | 如 `25` | 全网规划，按 ISU VM 数量规划 |

**示例脚本（SR-IOV bonding 组网 IPv4 单栈）**：
```
ADD AUTOSCALINGSERVICE: SERVICENAME="ServName_networkB", VPNNAME="VRF_networkB",
  AFTYPE=IPv4, IPALLOCTYPE4=USER_CONFIG, AUTOCFGIFTYPE=ETHTRUNK, ETHTRUNKTMPID=6,
  VLANID=20, BEGINADDR4="192.168.20.2", ENDADDR4="192.168.20.126", MASKLEN=25;
```

非 SR-IOV bonding 组网时，AUTOCFGIFTYPE=VNIC（改配 VNICID，不填 ETHTRUNKTMPID）。

## 约束

- **rule-0-00230**（warning）最大记录数 4096；地址池范围（BEGINADDR*/ENDADDR*）不超过 2048；VNICID 取值须在设备物理主接口号有效范围；立即生效；自动化配置开关须先关闭（SET AUTOCONFIG 关闭 → 执行 → 开启） — 后果：超规格/超地址池上限/VNICID 越界新增失败
