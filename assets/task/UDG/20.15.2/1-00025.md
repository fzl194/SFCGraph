# 双栈 APN + 双栈 POOLGROUP（IPv4v6 对称配置）

> 归属 [2-00039](task/UDG/20.15.2/2-00039.md) IPv4v6 双栈接入（GWFD-020403）。本 compound 配置**双栈对称配置层**——在 [1-00022](task/UDG/20.15.2/1-00022.md) 地址池体系骨架基础上叠加双栈特有差异：APN 对称绑定 IPv4/IPv6 双 VPN、POOLGROUP 对称使能双优先级算法、双池（IPv4/IPv6）双段、单条 POOLGRPMAP 映射。被引用于 {2-00039}。

## 配置方法

本 compound 由 7 条命令组成（多命令可复用模块，建 compound 承载）：

| 步骤 | 命令 | 关键参数 | 与 1-00022 的差异 |
|---|---|---|---|
| 1. 配置双栈 APN（双 VPN 对称使能） | [ADD APN](task/UDG/20.15.2/0-00020.md) | `APN` / `HASVPN=ENABLE` / `VPNINSTANCE` / **`HASVPNIPV6=ENABLE`** / **`VPNINSTANCEIPV6`** | 新增 HASVPNIPV6 + VPNINSTANCEIPV6 |
| 2. 创建双栈地址池组（双优先级算法） | [ADD POOLGROUP](task/UDG/20.15.2/0-00042.md) | `POOLGRPNAME` / **`IPV4ALLOCPRIALG=ENABLE`** / **`IPV6ALLOCPRIALG=ENABLE`** | 双栈必配双算法（1-00022 范本未配算法） |
| 3. 创建 IPv4 地址池（LOCAL 类型，绑定 VPN） | [ADD POOL](task/UDG/20.15.2/0-00043.md) | `POOLNAME` / **`IPVERSION=IPV4`** / `POOLTYPE=LOCAL` / `HASVPN=ENABLE` / `VPNINSTANCE` | 新增 IPVERSION=IPV4 显式指定（双池并存第一池） |
| 4. 创建 IPv6 地址池（LOCAL 类型，绑定 VPN） | [ADD POOL](task/UDG/20.15.2/0-00043.md) | `POOLNAME` / **`IPVERSION=IPV6`** / `POOLTYPE=LOCAL` / `HASVPN=ENABLE` / `VPNINSTANCE` | 新增 IPVERSION=IPV6（双池并存第二池，1-00022 范本无） |
| 5. 配置 IPv4 地址段 | [ADD SECTION](task/UDG/20.15.2/0-00044.md) | `POOLNAME` / `SECTIONNUM` / **`IPVERSION=IPV4`** / `V4STARTIP` / `V4ENDIP` | 1-00022 共用，但双栈场景双段并存 |
| 6. 配置 IPv6 地址段（必填 IPv6 三参） | [ADD SECTION](task/UDG/20.15.2/0-00044.md) | `POOLNAME` / `SECTIONNUM` / **`IPVERSION=IPV6`** / **`V6PREFIXSTART`** / **`V6PREFIXEND`** / **`V6PREFIXLENGTH`**（49~64） | 1-00022 范本无；V6PREFIXLENGTH < 64 时切换为 IPv6 PD 模式（与 GWFD-020406 协同） |
| 7. 绑定双池到同一地址池组（双池双绑定） | [ADD POOLBINDGROUP](task/UDG/20.15.2/0-00045.md) ×2 | `POOLGROUPNAME` / `POOLNAME` / `PRIORITY` | 1-00022 共用；双栈需 2 次（IPv4 池 + IPv6 池各一） |
| 8. 配置 APN↔POOLGROUP 单映射（双栈统一） | [ADD POOLGRPMAP](task/UDG/20.15.2/0-00047.md) | `MAPPINGNAME` / `APN` / `POOLGROUPNAME` | 1-00022 共用；双栈仅 1 条映射（双池共用一组） |

**典型脚本（GWFD-020403 任务示例）：**
```
// 双栈 APN（双 VPN 对称使能）
ADD APN:APN="apn-test", HASVPN=ENABLE, VPNINSTANCE="VRF_Internet",
            HASVPNIPV6=ENABLE, VPNINSTANCEIPV6="VRF_Internet";

// 双栈地址池组（双优先级算法）
ADD POOLGROUP: POOLGRPNAME="poolgroup1",
                IPV4ALLOCPRIALG=ENABLE, IPV6ALLOCPRIALG=ENABLE;

// 双池（IPv4 + IPv6）
ADD POOL:POOLNAME="testpool",   POOLTYPE=LOCAL, IPVERSION=IPV4, HASVPN=ENABLE, VPNINSTANCE="VRF_Internet";
ADD POOL:POOLNAME="testpoolv6", POOLTYPE=LOCAL, IPVERSION=IPV6, HASVPN=ENABLE, VPNINSTANCE="VRF_Internet";

// 双段（IPv4 V4START/V4END + IPv6 V6PREFIXSTART/END/LENGTH）
ADD SECTION:POOLNAME="testpool",   SECTIONNUM=1, IPVERSION=IPV4,
            V4STARTIP="10.0.0.0", V4ENDIP="10.0.127.255";
ADD SECTION:POOLNAME="testpoolv6", SECTIONNUM=1, IPVERSION=IPV6,
            V6PREFIXSTART="fc00:0000:0000:fcee:0000:0000:0000:0000",
            V6PREFIXEND="fc00:0000:0000:fcef:ffff:ffff:ffff:ffff",
            V6PREFIXLENGTH=64;

// 双池绑到同一组（PRIORITY=10 双池各一）
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpool",   PRIORITY=10;
ADD POOLBINDGROUP: POOLGROUPNAME="poolgroup1", POOLNAME="testpoolv6", PRIORITY=10;

// APN↔POOLGROUP 单映射（双栈统一）
ADD POOLGRPMAP: MAPPINGNAME="mapping1", APN="apn-test", POOLGROUPNAME="poolgroup1";
```

## 场景差异

| 场景 | 池数量 | SECTION 数量 | POOLBINDGROUP 数量 | POOLGRPMAP 数量 | 关键参数差异 | 走法 |
|---|---|---|---|---|---|---|
| 双栈 IPv4v6（典型） | 2 | 2 | 2 | 1（APN=） | `IPVERSION=IPV4/IPV6` + `IPV4ALLOCPRIALG=ENABLE` + `IPV6ALLOCPRIALG=ENABLE` + `HASVPNIPV6=ENABLE` | 本 compound 全步骤 |
| 双栈 IPv6 PD（V6PREFIXLENGTH < 64） | 2 | 2 | 2 | 1（APN=） | `V6PREFIXLENGTH=60`（或 49~63 任一） | 本 compound + 与 GWFD-020406 IPv6 PD 协同 |
| 双栈 VPN 隔离（IPv4/IPv6 分属不同 VPN） | 2 | 2 | 2 | 1（APN=） | `VPNINSTANCE="VRF_V4"` 与 `VPNINSTANCEIPV6="VRF_V6"` 不同名 | 本 compound（仅 APN/POOL 的 VPN 参数差异） |
| 仅 IPv4 单栈（退化） | 1 | 1 | 1 | 1（APN=） | `HASVPNIPV6=DISABLE`；不配 testpoolv6 + IPv6 SECTION + IPv6 POOLBINDGROUP | **不执行本 compound 的 IPv6 段**（步骤 4/6/7-IPv6 跳过）—— 复用 [1-00022](task/UDG/20.15.2/1-00022.md) 1 池范式 |
| 仅 IPv6 单栈（退化） | 1 | 1 | 1 | 1（APN=） | `HASVPN=DISABLE`；不配 testpool + IPv4 SECTION + IPv4 POOLBINDGROUP | **不执行本 compound 的 IPv4 段**（步骤 3/5/7-IPv4 跳过）—— 复用 [1-00022](task/UDG/20.15.2/1-00022.md) 1 池范式 |

> ★ **复用警示**：本 compound 步骤 1（APN）必配；步骤 2-8（地址池体系）单栈场景不执行——按场景差异表跳过非执行段。单栈场景（仅 IPv4 / 仅 IPv6）应直接复用 [1-00022](task/UDG/20.15.2/1-00022.md) + 单池命令，不复用本 compound。

## 决策点：双栈 APN 与地址池形态选择（DP 0-00344）

问：本特性如何选择双栈 APN + POOLGROUP 配置形态？

| 选项 | 关键参数 | 必备命令 | 适用场景 |
|---|---|---|---|
| 双栈典型 | HASVPN=ENABLE + HASVPNIPV6=ENABLE；IPV4/IPV6ALLOCPRIALG=ENABLE；V6PREFIXLENGTH=64 | 本 compound 全 7 步 | 双栈使能典型场景（PDN 类型=IPv4v6，UDG 双栈上下文创建） |
| 双栈 IPv6 PD | V6PREFIXLENGTH=49~63 | 本 compound + 复用 GWFD-020406 IPv6 PD 能力 | UE IPv6 地址采用 Prefix Delegation 方式分配 |
| 双栈 VPN 隔离 | VPNINSTANCE ≠ VPNINSTANCEIPV6 | 本 compound | IPv4/IPv6 业务走不同 VPN 实例（IPv4/IPv6 隔离的园区/家庭宽带场景） |
| 仅 IPv4 单栈（退化） | HASVPNIPV6=DISABLE；不配 IPv6 池 | [1-00022](task/UDG/20.15.2/1-00022.md) 单池范式 | UDG 部署为 IPv4 单栈场景（本特性 License 仍需加载但不发挥双栈作用） |
| 仅 IPv6 单栈（退化） | HASVPN=DISABLE；不配 IPv4 池 | [1-00022](task/UDG/20.15.2/1-00022.md) 单池范式 | UDG 部署为 IPv6 单栈场景 |

## 约束

- **License 强制前置**（critical，rule-0-01610）：必须先 `SET LICENSESWITCH:LICITEM="LKV3G5VDSA01",SWITCH=ENABLE` 打开双栈 License，否则本 compound 步骤 1-8 业务命令配齐也无法让 UDG 创建双栈上下文 — 后果：双栈上下文无法建立，UE 退化为 IPv4 单栈
- **双栈对称配置**（critical，rule-0-01611）：APN 必须同时使能 HASVPN 与 HASVPNIPV6；POOLGROUP 必须同时绑定 IPv4 池与 IPv6 池；POOLGROUP 必须同时使能 IPV4ALLOCPRIALG 与 IPV6ALLOCPRIALG — 后果：不对称配置致双栈退化（如 HASVPNIPV6=DISABLE → PDP Type 退化为 IPv4）
- **VPN 三者一致（IPv4 与 IPv6 链各自）**（critical，rule-0-01612）：IPv4 链：APN 绑定的 VPN / testpool 绑定的 VPN / Gi/SGi/N6 外联口绑定的 VPN 必须一致；IPv6 链同理 — 后果：VPN 不一致致地址分配或路由异常
- **V6PREFIXLENGTH 范围 49~64**（critical，rule-0-01613）：V6PREFIXLENGTH 必须为 49~64 之间 — 后果：超出范围命令拒收
- **V6PREFIXLENGTH<64 切换 PD 模式**（warning，rule-0-01614）：当前缀长度 < 64 时，地址池采用 IPv6 Prefix Delegation 方式分配（与 GWFD-020406 IPv6 PD 协同）— 后果：UE 获 IPv6 前缀而非完整 IPv6 地址；本特性激活文档明确此切换条件
- **每 APN 绑池数 ≤16**（warning，rule-0-01615）：每 APN 支持绑定的地址池最多 16 个；双栈场景下 IPv4 池 + IPv6 池合计计入 — 后果：超规格新增失败
- **每 POOL 绑段数 ≤64**（warning，rule-0-01616）：每本地地址池支持绑定的地址段最多 64 个；双栈场景下 IPv4 池与 IPv6 池各自独立 — 后果：超规格新增失败
- **弱覆盖场景掉话率上升**（warning，rule-0-01617）：IPv4/v6 双栈改造后，UE 需通过 UDG 下发 Router Advertisement（RA）获取 IPv6 地址；弱覆盖场景下 eNodeB 无法下发该报文给终端时**掉话率会增加** — 后果：弱覆盖区域用户体验降级
- **仅 IPv4/IPv6 单栈退化**（info，rule-0-01618）：HASVPNIPV6=DISABLE 或 HASVPN=DISABLE 时本特性 License 仍需加载，但双栈能力未发挥作用；UE 退化为单栈接入 — 后果：双栈能力闲置
