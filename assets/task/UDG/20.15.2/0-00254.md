# 设置APN内容过滤模板（SET APNCFTEMPLATE）

> 命令静态知识见 [SET APNCFTEMPLATE](command/UDG/20.15.2/SET-APNCFTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务把已建的内容过滤模板绑定到指定 APN，使该 APN 维度的 URL 过滤按此模板执行。两个参数均来自已配置数据：APN 名引用 ADD APN，模板名引用 ADD CFTEMPLATE。适用 NF：PGW-U、UPF。

**取值样例（数据规划表，GWFD-110471）**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| APNNAME | `apn-test` | 引用 ADD APN | 与 ADD APN 配置的 APN 名保持一致 |
| CFTEMPLATENAME | `icaptemplate` | 引用 ADD CFTEMPLATE | 已通过 ADD CFTEMPLATE 配置 |

**脚本样例**

```
SET APNCFTEMPLATE: APNNAME="apn-test", CFTEMPLATENAME="icaptemplate";
```

**说明**：本命令两参数均 required，无缺口。这是 APN 维度的执行模板挂载点，把 APN（来自 0-00020）与模板（来自 0-00253）连接起来。

## 决策点：无（命令用法单一，跨特性一致）

SET APNCFTEMPLATE 仅指定 APNNAME + CFTEMPLATENAME 两个引用键，无配套必选/可选参数。无决策分支。

## 约束

- **rule-0-00316**（warning）**只对之后发生承载更新的用户或新激活用户生效**（存量在线用户不感知）；系统最多支持 **10000** 个 APN 内容过滤模板，超限 SET 失败；**每个 APN 仅可绑定一个内容过滤模板**（1:1，重复 SET 同一 APN 会覆盖前次模板）；APNNAME/CFTEMPLATENAME 均为引用，APN 与模板必须先建。
