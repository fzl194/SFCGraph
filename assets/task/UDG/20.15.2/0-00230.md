# 配置链路丢包告警阈值Light模式（SET LINKALMCFG）

> 命令静态知识见 [SET LINKALMCFG](command/UDG/20.15.2/SET-LINKALMCFG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110921（TWAMP）**Light 模式独有**的第 6 步——配置 ALM-100395（链路丢包率过高告警）的阈值。仅 Light 模式（UDG 作发送端可测丢包率/时延/抖动）可触发该告警；Full 模式（UDG 作响应端）不测。三参均有系统初始记录（10/5/30），但特性示例按本场景告警需求调整（30/10/40）。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| THRESHOLD | 丢包率过高告警阈值 | `30` | 全网规划，整数 1~100%（系统初始记录 10） |
| CLEARTHRESHOLD | 恢复阈值 | `10` | 全网规划，整数 0~99%，须小于 THRESHOLD（系统初始记录 5） |
| PERIOD | 检测周期 | `40` | 全网规划，整数 10~180 秒（系统初始记录 30） |

> 三参均 optional，未输入保持当前配置不变（可用 LST LINKALMCFG 查询）。特性脚本显式传 30/10/40（非初始值，按本场景调整）。

**脚本样例**

```
SET LINKALMCFG:THRESHOLD=30,CLEARTHRESHOLD=10,PERIOD=40;
```

**操作步骤定位**：Light 模式激活流程末尾，紧接 ADD TWAMPSENDER（0-00228）之后。命令执行后立即生效。

## 约束

- CLEARTHRESHOLD（恢复阈值）须**小于** THRESHOLD（告警阈值）。
- 系统部署后已存在初始记录（THRESHOLD=10/CLEARTHRESHOLD=5/PERIOD=30），三参 optional 未输入保持当前配置不变。
- 取值超范围新增失败。
