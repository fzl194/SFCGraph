# 全局激活BFD（SET BFD）

> 命令静态知识见 [SET BFD](command/UDG/20.15.2/SET-BFD.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET BFD 是 BFD 功能的全局开关，只有 `BFDENABLE` 一个关键参数。GWFD-010155 的 4 种组网脚本（VPN+OSPF 动态路由 / VPN 缺省路由 / 静态路由+BFD / OSPFv3 动态路由）取值完全一致——固定 `BFDENABLE=TRUE`。可选参数 `DELAYUPTIMER`（会话延迟Up时间）、`TOSEXP`（动态会话报文优先级）在 md 说明中明示"可不输，使用默认值"，4 个脚本均未显式传入，按通用配置筛不绑。

**取值范式（从证据数据规划表归纳）**

| 场景 | BFDENABLE | 脚本样例 |
|---|---|---|
| GWFD-010155 全部 4 种组网（VPN+OSPF / VPN 缺省路由 / 静态路由+BFD / OSPFv3） | TRUE（本端规划，固定） | `SET BFD:BFDENABLE=TRUE;` |

**关键规则**
- **固定取值无分叉**：单特性（GWFD-010155）4 个组网脚本均为 `BFDENABLE=TRUE`，binding=fixed。
- **DELAYUPTIMER/TOSEXP/MPLSPASSIVE 不绑**：md 明示可不输用默认值，当前特性不揭示，待后续特性揭示时增量补全。
- **执行位置**：在 ADD VPNINST/ADD VPNINSTAF 之后、ADD OSPF/ADD OSPFV3 之前全局使能（参考 GWFD-010155 操作步骤顺序）。

## 约束

- **rule 0-00185**（safety_rule）本命令属高危命令，全局去使能 BFD（`BFDENABLE=FALSE`）后 BFD 所有历史配置全部删除且无法恢复；若有业务在使用 BFD 功能可能导致去使能失败 — 后果：误去使能导致全部 BFD 配置丢失、链路检测中断、业务流量震荡（critical）。本 atom 固定 BFDENABLE=TRUE 不触发去使能，但 rule 守护防误用。
