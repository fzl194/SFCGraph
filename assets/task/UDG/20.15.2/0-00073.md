# 设置BFD全局属性（SET BFD）

> 命令静态知识见 [SET BFD](command/UDG/20.15.2/SET-BFD.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令是全局唯一的高危命令**——BFDENABLE=FALSE 会去使能 BFD 并清空所有历史 BFD 配置；同时是 [0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION 等 BFD 子命令的全局前置。

## 配置方法（配置方法字典）

SET BFD 是"**BFD 全局属性 + 全局使能门控**"的全局唯一配置命令（**最大记录数 1**）。核心配置空间有 4 个维度：**全局使能开关**（BFDENABLE，门控所有 BFD 行为）、**会话延迟 Up 时间**（DELAYUPTIMER，与对端厂商互通场景）、**报文优先级**（TOSEXP，系统堵塞时优先发送高优先级 BFD 报文）、**MPLS 被动会话**（MPLSPASSIVE，启用 MPLS LSP BFD TLV 触发反向 BFD 会话）。所有 BFD 子命令（[0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION / RMV BFDSESSION（★R1.5 atom 待补） / SET BFD 等）的生效前提是 SET BFD:BFDENABLE=TRUE。

### 配置维度 1：全局使能开关（参数 BFDENABLE，必选）

BFDENABLE 是门控开关，控制 BFD 全局使能或全局去使能：

| 取值 | 行为 | 联动 |
|---|---|---|
| **TRUE** | 全局使能 BFD — 开启所有 BFD 链路检测能力 | [0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION 等 BFD 子命令生效 |
| **FALSE** | 全局去使能 BFD — **清空所有历史 BFD 配置**（高危，不可恢复） | 业务已用 BFD 的话可能失败（参考提示信息） |

> **命令使用实例仅配 `BFDENABLE=TRUE`**；**全局唯一最大记录数 1**；系统初始记录参数初始值 `BFDENABLE=FALSE / DELAYUPTIMER=0 / TOSEXP=7 / MPLSPASSIVE=FALSE`。

### 配置维度 2：会话延迟 Up 时间（参数 DELAYUPTIMER，可选）

DELAYUPTIMER 用于"整机重启 / 资源重启后切换流量回主用链路"场景下的延迟保护：

| 参数 | 取值范围 | 默认值 | 作用 |
|---|---|---|---|
| **DELAYUPTIMER** | 0~600 秒 | 0 | BFD 会话建立协商 Up 之前延迟时间；避免重启初期对端路由未生成致流量丢失 |

> **典型场景**：与其它厂商互通时，对方设备切换链路依赖 BFD Up 状态但路由可能未生成。延迟 DELAYUPTIMER 让 BFD Up 等路由生成后再触发。

### 配置维度 3：报文优先级（参数 TOSEXP，可选）

| 参数 | 取值范围 | 默认值 | 作用 |
|---|---|---|---|
| **TOSEXP** | 0~7 | 7 | BFD 报文优先级；系统堵塞时优先发送优先级高的 BFD |

> **取值越大优先级越高**（与常规 IP TOS 语义一致）；**初始值 7（最高）**。

### 配置维度 4：MPLS 被动会话（参数 MPLSPASSIVE，可选）

| 参数 | 取值范围 | 默认值 | 作用 |
|---|---|---|---|
| **MPLSPASSIVE** | TRUE / FALSE | FALSE | TRUE=启用 MPLS 被动动态创建 BFD 会话（收到携带 BFD TLV 的 LSP 请求报文后建立反向 BFD 会话） |

**典型脚本（全局使能 BFD）**：
```
SET BFD: BFDENABLE=TRUE;
```

> 命令 notes：适用 NF：SGW-U、PGW-U、UPF；**高危命令**——BFDENABLE=FALSE 会清空所有 BFD 配置；**执行后立即生效**；**全局唯一最大记录数 1**；系统初始记录初始值见命令 wiki。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：BFD 全局使能"——是 [0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION / MOD BFDSESSION（★R1.5 atom 待补） / RMV BFDSESSION（★R1.5 atom 待补） 等 BFD 子命令的全局前置条件。

## 决策点：BFDENABLE 开关策略（DP 0-00073）

问：SET BFD 的 BFDENABLE 取何值？

| 选项 | BFDENABLE | 联动 |
|---|---|---|
| **全局使能（默认）** | TRUE | [0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION 等子命令生效 |
| **全局去使能** | FALSE | 清空所有历史 BFD 配置（不可恢复） |

> BFDENABLE 是所有 BFD 子命令生效的全局前提；使用实例演示全量脚本仅配 `BFDENABLE=TRUE`，其余参数按需（DELAYUPTIMER/TOSEXP/MPLSPASSIVE 多数场景使用系统初始值）。

## 约束

- **BFDENABLE=FALSE 清空所有 BFD 配置**（critical，rule-0-00473）：全局去使能 BFD 后，BFD 所有配置无法恢复 — 后果：业务已使用 BFD 时可能导致去使能失败，或清空所有历史 BFD 配置后无法恢复
- **高危命令**（critical，rule-0-00473）：本命令属于高危命令，操作不当会去使能 BFD，导致 BFD 功能不可用 — 后果：建议谨慎使用并联系华为技术支持协助操作
- **全局唯一最大记录数 1**（critical，rule-0-00473）：本命令全局唯一，记录数 1 — 后果：重复配置命令拒收
- **执行后立即生效**（warning，rule-0-00473）：命令执行后立即生效 — 后果：误操作即刻生效（BFDENABLE=FALSE 会清空所有 BFD 配置）
- **业务已使用时去使能可能失败**（warning，rule-0-00473）：如果有业务在使用 BFD 功能，有可能导致去使能 BFD 失败 — 后果：去使能不生效，需先解除业务 BFD 引用
- **前置必须使能 BFD 后才能 ADD BFDSESSION**（critical，rule-0-00473）：使用 [0-00082](task/UDG/20.15.2/0-00082.md) ADD BFDSESSION 前必须通过 SET BFD 命令全局使能 BFD 功能 — 后果：未使能则 ADD BFDSESSION 失效
- **适用 NF 限定**（warning，隐含）：SGW-U、PGW-U、UPF — 后果：其他 NF 不适用
- **TOSEXP 取值大小语义**（info，隐含）：TOSEXP 取值越大 BFD 报文优先级越高 — 后果：值配反致报文优先级不符预期
