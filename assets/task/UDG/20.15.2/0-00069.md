# 配置下行PHB映射（SET QOSPHB）

> 命令静态知识见 [SET QOSPHB](command/UDG/20.15.2/SET-QOSPHB.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。出口方向，与入口方向 [SET QOSBA](task/UDG/20.15.2/0-00067.md) 对偶。

## 配置方法

SET QOSPHB 是 QoS 5 件套的"出口映射层"——在接口下行（egress）方向，把 UDG 内部的服务等级 + 报文颜色映射成外部 VLAN 报文的 802.1p 值（或 IP DSCP 值）（内部优先级→外部优先级）。本命令的核心配置空间有 4 个维度：**映射类型**（PHBTYPE 二选一）、**内部服务等级**（SERVICECLASS 枚举）、**报文颜色**（COLOR 枚举）、**外部优先级值**（PHBVALUE 取值范围由 PHBTYPE 决定）。

### 配置维度 1：映射类型（参数 PHBTYPE，必选）

| 取值 | 作用 | PHBVALUE 范围 |
|---|---|---|
| **8021p** | VLAN 报文 802.1p 优先级映射（ToS→CoS） | 0~7 |
| **DSCP** | IP 报文 DSCP 优先级映射（ToS→DSCP） | 0~63 |

### 配置维度 2：内部服务等级（参数 SERVICECLASS，必选）

| 取值 | 含义 |
|---|---|
| **be** | 尽力而为（最低） |
| **af1 / af2 / af3 / af4** | 确保转发 4 档 |
| **ef** | 加速转发 |
| **cs1 ~ cs7** | 服务类别 7 档 |

### 配置维度 3：报文颜色（参数 COLOR，必选）

| 取值 | 含义 |
|---|---|
| **green** | 报文颜色（合规） |
| **yellow** | 报文颜色（超 CIR） |
| **red** | 报文颜色（超 PIR） |

### 配置维度 4：外部优先级值（参数 PHBVALUE，必选）

| 取值范围 | 含义 |
|---|---|
| 整数 0~7（PHBTYPE=8021p 时） | 802.1p 优先级值（取值越大优先级越高） |
| 整数 0~63（PHBTYPE=DSCP 时） | DSCP 优先级值 |

| 参数 | 必选性 | 取值 | 作用 |
|---|---|---|---|
| `DSNAME` | 必选 | 字符串 | 引用 [0-00066](task/UDG/20.15.2/0-00066.md) ADD QOSDIFFERSERV |
| `PHBTYPE` | 必选 | 8021p / DSCP | PHB 类型 |
| `PHBVALUE` | 必选 | 0~7 或 0~63 | 外部优先级值 |
| `SERVICECLASS` | 必选 | be/af1~4/ef/cs1~7 | 内部服务等级 |
| `COLOR` | 必选 | green/yellow/red | 报文颜色 |

**典型脚本（010201 QoS 出口映射）**：
```
SET QOSPHB:DSNAME="default",PHBTYPE=8021p,PHBVALUE=3,SERVICECLASS=be,COLOR=green;
```

**步骤位置**：在所属 feature 编排中位于"步骤 3b：出口映射"——前置 [0-00066](task/UDG/20.15.2/0-00066.md) ADD QOSDIFFERSERV（DSNAME 来源）+ [0-00065](task/UDG/20.15.2/0-00065.md) SET SRVCOMMONPARA（TunnelMarking）；后续 [0-00068](task/UDG/20.15.2/0-00068.md) ADD QOSIFTRUST（接口绑定）。

## 决策点：PHBTYPE 映射类型选择（DP 0-00069）

问：SET QOSPHB 的 PHBTYPE 取何值？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 8021p（VLAN 报文 CoS） | PHBVALUE 0~7 | 出口方向 ToS→VLAN 802.1p 映射（010201 主场景） |
| DSCP（IP 报文） | PHBVALUE 0~63 | 出口方向 ToS→IP DSCP 映射 |

## 约束

- **BATYPE/PHBTYPE 跨方向互斥**（critical，rule-0-00283）：同一 DSNAME 下入口 BA + 出口 PHB 配套取值 8021p 或 DSCP — 后果：PHBTYPE 错配→取值越界被拒
- **入口+出口方向各执行一次**（critical，rule-0-00283）：同一 DSNAME 入口 BA + 出口 PHB 须成对配置 — 后果：方向未配→DSCP 映射缺失
- **PHBVALUE 取值范围**（critical，隐含）：8021p 时 0~7，DSCP 时 0~63 — 后果：超范围命令拒收
- **BATYPE=DSCP 与 3GPP QoS-DSCP 并存时被后者覆盖**（warning，rule-0-00283）：若同时配 ToS→DSCP 与 3GPP QoS→DSCP/ToS，前者被后者覆盖 — 后果：PHBTYPE=DSCP 配置被覆盖
- **DSNAME 前置依赖**（critical，隐含）：DSNAME 须先建（[0-00066](task/UDG/20.15.2/0-00066.md)） — 后果：未建域命令拒收
