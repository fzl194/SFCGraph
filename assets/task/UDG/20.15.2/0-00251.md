# 设置APN内容过滤开关（SET APNCFFUNC）

> 命令静态知识见 [SET APNCFFUNC](command/UDG/20.15.2/SET-APNCFFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务基于 APN 粒度开启/关闭 URL 内容过滤功能。每个 APN 需单独配置开关，APN 名必须来自 ADD APN 已建实例。适用 NF：PGW-U、UPF。

**取值样例（数据规划表，GWFD-110471）**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| APNNAME | `apn-test` | 引用 ADD APN | 与 ADD APN 配置的 APN 名保持一致 |
| CFSWITCHVALUE | `ENABLE` | 本端规划 | 基于该 APN 开启 URL 过滤；关闭用 `DISABLE` |

**脚本样例**

```
SET APNCFFUNC: APNNAME="apn-test", CFSWITCHVALUE=ENABLE;
```

**说明**：CFSWITCHVALUE 命令真相标 global_planned，本端规划口径表达规划层权责（特性 md 标注）；枚举 DISABLE/ENABLE 二选一。开启 URL 过滤为整个 URL 过滤配置链路的第一步（APN 级总开关）。

## 约束

- **rule-0-00387**（warning）系统最多支持 **10000** 个 APN 内容过滤开关，超限 SET 失败；命令会导致用户匹配数变化、可能性能下降，执行前需评估性能影响；**只对之后发生承载更新的用户或新激活用户生效**（存量在线用户不感知）。APNNAME 仅由 `-`、数字、大小写字母、`.` 组成，不以 `.` 开头且无连续两个 `.`。
