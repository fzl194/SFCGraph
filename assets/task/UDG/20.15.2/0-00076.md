# 增加OSPF运行接口网段(IPv4)（ADD OSPFNETWORK）

> 命令静态知识见 [ADD OSPFNETWORK](command/UDG/20.15.2/ADD-OSPFNETWORK.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD OSPFNETWORK 在 OSPF 进程的区域下增加运行 OSPF 协议的接口网段。4 个必选参数：`PROCID` 引用上游 ADD OSPF（0-00074），`AREAID` 引用上游 ADD OSPFAREA（0-00075），`IPADDRESS`+`WILDCARDMASK` 全网规划。一个进程区域下可有多个网段——脚本对多网段重复执行（每网段一条命令）。可选 `DESCRIPTION` 描述参数在 7 个特性数据规划表+脚本中均未用到，不绑。

**取值范式（从证据数据规划表归纳）**

| 特性 / 场景 | PROCID | AREAID | IPADDRESS | WILDCARDMASK | 脚本样例 |
|---|---|---|---|---|---|
| GWFD-010104/010105/020406/020421 地址分配类 | 100（引用 0-00074） | 0.0.0.0（引用 0-00075） | 10.13.21.0（全网规划） | 0.0.0.255 | `ADD OSPFNETWORK:PROCID=100,AREAID="0.0.0.0",IPADDRESS="10.13.21.0",WILDCARDMASK="0.0.0.255";` |
| GWFD-010155 PGW-U 到 ePDG（多网段重复） | 6（引用 0-00074） | 0.0.0.1（引用 0-00075） | 10.8.7.0 / 10.8.8.0（全网规划，每网段一条） | 0.0.0.15 | `ADD OSPFNETWORK:PROCID=6,AREAID="0.0.0.1",IPADDRESS="10.8.7.0",WILDCARDMASK="0.0.0.15";`<br>`ADD OSPFNETWORK:PROCID=6,AREAID="0.0.0.1",IPADDRESS="10.8.8.0",WILDCARDMASK="0.0.0.15";` |
| IPFD-015004 OSPF over IPsec | 20214（引用 0-00074） | 10.102.105.0（引用 0-00075） | 10.102.105.0 | 0.0.0.255 | `ADD OSPFNETWORK:PROCID=20214,AREAID="10.102.105.0",IPADDRESS="10.102.105.0",WILDCARDMASK="0.0.0.255";` |

**关键规则**
- **PROCID+AREAID 必须引用已创建的进程和区域**：先 ADD OSPF → ADD OSPFAREA → ADD OSPFNETWORK（命令前置约束）。
- **多网段重复执行**：一个进程区域下可配多个接口网段（GWFD-010155 对 10.8.7.0/0.0.0.15 与 10.8.8.0/0.0.0.15 各执行一条），属正常迭代，非单参多值。
- **可选 ADD OSPFINTERFACE**：若需配接口属性（如优先级=0 避免 VNF 被选 DR/BDR），用 ADD OSPFINTERFACE（优先级大于 ADD OSPFNETWORK）。GWFD-010107 主备路由冗余场景亦提及可用 ADD OSPFINTERFACE 或 ADD OSPFNETWORK 打通主备 LoopBack 路由。视为可选修饰命令不单建 atom。

## 约束

- **rule 0-00192**（constraint_rule/execution_constraint）ADD OSPFNETWORK 须满足：(1) 前置——必须先执行 ADD OSPF（进程）与 ADD OSPFAREA（区域）后才能使用此命令；(2) OSPF 网段规格默认单进程 500、命令最大记录数 8016；(3) 可把多个接口配置到一个区域中（多网段重复执行，每网段一条命令）；(4) ADD OSPFINTERFACE 命令接口使能的优先级大于 ADD OSPFNETWORK（可选 ADD OSPFINTERFACE 配接口属性，优先级=0 避免 VNF 被选 DR/BDR）；(5) OSPF 不允许以从地址建网段 — 后果：网段创建失败 / 网段规格超限 / DR/BDR 误选 / 接口使能冲突（warning）。
