# 增加OSPF运行接口网段(IPv4)（ADD OSPFNETWORK）

> 命令静态知识见 [ADD OSPFNETWORK](command/UDG/20.15.2/ADD-OSPFNETWORK.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令是 OSPF backbone 4 件套之三**——必须在 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF（建进程）+ [0-00075](task/UDG/20.15.2/0-00075.md) ADD OSPFAREA（建区域）之后执行；接口 IP 落在 IPADDRESS 网段内时运行 OSPF 协议。

## 配置方法（配置方法字典）

ADD OSPFNETWORK 是"**在 OSPF 进程区域下增加运行 OSPF 协议的接口网段**"命令。PROCID/AREAID/IPADDRESS/WILDCARDMASK 四参必选；前两参引用上游已建，后两参构成"网段 + 反掩码"对。核心配置空间有 3 个维度：**进程号**（PROCID 引用上游）、**区域号**（AREAID 引用上游）、**接口网段配对**（IPADDRESS + WILDCARDMASK 全网规划，配合后接口自动落入 OSPF 协议）、**可选描述**（DESCRIPTION 1~80 字符）。N 个特性命中（GWFD-010104/010105/020406/020421 / GWFD-010155 / IPFD-015004）覆盖单网段和多网段两种范式。

### 配置维度 1：进程号（参数 PROCID，必选，引用上游）

| 参数 | 取值范围 | 数据来源 | 关联 |
|---|---|---|---|
| **PROCID** | 1~4294967295 | ref [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF 已建进程 | 必须已存在，使用 LST OSPF 查看 |

> **关键约束**：**必须先 ADD OSPF 配置进程和 ADD OSPFAREA 配置区域后才能使用本命令**。

### 配置维度 2：区域号（参数 AREAID，必选，引用上游）

| 参数 | 取值范围 | 数据来源 | 关联 |
|---|---|---|---|
| **AREAID** | IPv4 地址类型（点分十进制） | ref [0-00075](task/UDG/20.15.2/0-00075.md) ADD OSPFAREA 已建区域 | 必须已存在，使用 LST OSPFAREA 查看 |

> **关键约束**：OSPF 区域号必须已经存在。

### 配置维度 3：接口网段配对（参数 IPADDRESS + WILDCARDMASK，必选）

| 参数 | 取值范围 | 数据来源 | 备注 |
|---|---|---|---|
| **IPADDRESS** | IPv4 地址类型（点分十进制） | 全网规划 | 接口所在的网段地址 |
| **WILDCARDMASK** | IPv4 地址类型（点分十进制反掩码） | 全网规划 | IP 反掩码（0 变 1，1 变 0），"1"=忽略位/"0"=保留位 |

> **核心机制**：**接口 IP 落在 IPADDRESS 网段内时自动运行 OSPF 协议**，无需逐接口配（这也是 NETWORK 命令比 INTERFACE 命令粒度粗的原因）。**多网段重复执行**：一个进程区域可配多个接口网段，每网段一条命令（GWFD-010155 范式演示 10.8.7.0/0.0.0.15 + 10.8.8.0/0.0.0.15 各一条）。

### 配置维度 4：可选描述（参数 DESCRIPTION，可选）

| 参数 | 取值范围 | 备注 |
|---|---|---|
| **DESCRIPTION** | 字符串 1~80 字符 | 对网段进行描述信息（命令当前 UDG 场景实例化通常不填） |

> **接口类型支持**：**接口可以是电口以太网或以太网 Trunk 接口**；**ADD OSPFINTERFACE 命令接口使能的优先级大于 ADD OSPFNETWORK**（按需用 ADD OSPFINTERFACE 配接口属性如优先级=0 避免 VNF 被选 DR/BDR）。

**典型脚本（010104 单网段）**：
```
ADD OSPFNETWORK: PROCID=100, AREAID="0.0.0.0", IPADDRESS="10.13.21.0", WILDCARDMASK="0.0.0.255";
```

**典型脚本（010155 多网段重复执行）**：
```
ADD OSPFNETWORK: PROCID=6, AREAID="0.0.0.1", IPADDRESS="10.8.7.0", WILDCARDMASK="0.0.0.15";
ADD OSPFNETWORK: PROCID=6, AREAID="0.0.0.1", IPADDRESS="10.8.8.0", WILDCARDMASK="0.0.0.15";
```

**典型脚本（IPFD-015004 OSPF over IPsec）**：
```
ADD OSPFNETWORK: PROCID=20214, AREAID="10.102.105.0", IPADDRESS="10.102.105.0", WILDCARDMASK="0.0.0.255";
```

> 命令 notes：**最大记录数 8016**；**执行后立即生效**；**前置约束**：必须 ADD OSPF + ADD OSPFAREA 后才能使用；**单进程网段规格默认 500**（实际以服务规格文件为准）；**多接口可配到同一区域**；**OSPF 不允许以从地址建网段**；**接口可以是电口以太网或以太网 Trunk 接口**。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N+2：在区域下加网段"——前置 [0-00074](task/UDG/20.15.2/0-00074.md) ADD OSPF + [0-00075](task/UDG/20.15.2/0-00075.md) ADD OSPFAREA。

## 决策点

（无。本命令在所有特性下用法一致——PROCID/AREAID 引用上游已建进程区域，IPADDRESS/WILDCARDMASK 全网规划，无特性分叉。）

## 约束

- **最大记录数 8016**（warning，rule-0-00476）：本命令最大记录数为 8016 — 后果：超规格新增失败
- **单进程网段规格默认 500**（warning，rule-0-00476）：OSPF 网段规格默认为单进程 500，实际以服务规格文件为准 — 后果：单进程超过规格新增失败
- **执行后立即生效**（warning，rule-0-00476）：命令执行后立即生效 — 后果：误操作即刻生效
- **必须先 ADD OSPF + ADD OSPFAREA**（critical，rule-0-00476）：只有执行 ADD OSPF 配置了 OSPF 进程和 ADD OSPFAREA 配置了 OSPF 区域后才能使用此命令 — 后果：未先建进程/区域直接配网段命令拒收
- **PROCID 必须引用已建进程**（critical，rule-0-00476）：OSPF 进程号必须已经存在 — 后果：引用不存在的 PROCID 命令拒收
- **AREAID 必须引用已建区域**（critical，rule-0-00476）：OSPF 区域号必须已经存在 — 后果：引用不存在的 AREAID 命令拒收
- **不允许从地址建网段**（critical，rule-0-00476）：OSPF 不允许以从地址建立 OSPF 邻居关系，而只能以接口的主地址建立邻接关系 — 后果：网段配从地址致邻居建立失败
- **ADD OSPFINTERFACE 优先级大于 ADD OSPFNETWORK**（warning，rule-0-00476）：ADD OSPFINTERFACE 命令接口使能的优先级大于 ADD OSPFNETWORK — 后果：同一接口同时配 NETWORK 和 INTERFACE 时 INTERFACE 生效
- **多接口可配到同一区域**（info，隐含）：可以把多个接口配置到一个区域中（多网段重复执行） — 后果：每网段一条命令是正确的使用方式
