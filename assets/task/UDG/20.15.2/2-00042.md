# GRE（IPFD-015002，UDG 侧）

> 特性静态知识见 [IPFD-015002](feature/UDG/20.15.2/IPFD-015002.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/IPFD-015002-GRE.md)。
>
> **本页定位**：UDG 侧 GRE 隧道配置生成——5 步建立隧道（LoopBack 源 + GRETUNNEL + Tunnel 接口 + 静态路由 + 可选增强）。调测命令见原始特性信息（[调测 GRE](evidence/UDG/20.15.2/IPFD-015002-调测GRE_06422611.md)），不在配置流程展开。

## 配置概览

本特性按 **5 场景 × 1 compound 编排** 组织。GRE 隧道建立是核心，5 场景共享同一 5 步骨架，差异在源类型/地址族/VPN/增强项。

**对象链**：
```
LoopBack 接口（推荐源）→ ADD GRETUNNEL（gre 隧道对象）→ Tunnel 接口 IP → ADD SRROUTE（引导流量）
                                                                ↓（可选增强）
                                            MOD GRETUNNEL：Checksum / GREKey / Keepalive
```

**1 compound 编排**（5 场景共享）：

| 步骤 | 干啥 | 编排 compound |
|---|---|---|
| 1. GRE 隧道建立 | LoopBack 源 + GRETUNNEL + Tunnel 接口 + 静态路由 + 可选增强 | [1-00032](task/UDG/20.15.2/1-00032.md) |
| 0.（VPN 组网场景前置）| APN 接入域基础设施（VPN 实例）| [1-00029](task/UDG/20.15.2/1-00029.md)（仅 VPN 组网场景）|

**关键约束**：
- **无需 License**
- **C-U 对称**：UDG/UNC 两侧配置同构
- **源地址互斥 IPSec**：GRE 隧道源地址不能与 IPSec 隧道源地址相同
- 隧道源/目的路由可达（PING 验证）
- GRE 最多两层嵌套

## 配置流程

> 每步编排 compound；按场景选变种（变种见决策点）。调测见 [调测 GRE](evidence/UDG/20.15.2/IPFD-015002-调测GRE_06422611.md)。

1. **（仅 VPN 组网场景）APN 接入域基础设施**：
   - → 复用 [1-00029](task/UDG/20.15.2/1-00029.md)（VPN 实例；公网场景跳过本步）

2. **GRE 隧道建立**（5 步骨架 + 可选增强）：
   - → 复用 [1-00032](task/UDG/20.15.2/1-00032.md)（LoopBack 源 + ADD GRETUNNEL TNLTYPE=gre + Tunnel 接口 IP + ADD SRROUTE）
   - 场景变种（按 DP 选）：
     - **基础公网**：DSTVPNNAME 不配
     - **VPN 组网**：DSTVPNNAME=vpn 名（叠加步骤 1）
     - **IPv6（v02+）**：LoopBack/Tunnel 改 IPv6 地址（SET IFIPV6ENABLE+ADD IFIPV6ADDRESS），需 UDG 20.9.0+
     - **多租户共享**：MOD GRETUNNEL GREKEYEN=TRUE+GREKEY（同源同宿多隧道以 Key 区分）
     - **GRE over IPSec**：先建本隧道，再叠加 IPSec 加密（复用 [1-00033](task/UDG/20.15.2/1-00033.md) IPSec 隧道）

## 决策点

**DP（特性级）：场景 × 源类型 × 地址族 × 走法**

| 场景 | 源类型 | 地址族 | 走法（compound 编排） |
|---|---|---|---|
| 基础公网 GRE | if_name（LoopBack）| IPv4 | [1-00032](task/UDG/20.15.2/1-00032.md)（DSTVPNNAME 不配 + Keepalive 推荐）|
| VPN 组网 GRE | if_name | IPv4 | [1-00029](task/UDG/20.15.2/1-00029.md) + [1-00032](task/UDG/20.15.2/1-00032.md)（DSTVPNNAME=vpn）|
| IPv6 GRE（v02+）| if_name | IPv6 | [1-00032](task/UDG/20.15.2/1-00032.md) IPv6 变种（IFIPV6ENABLE+IFIPV6ADDRESS）；UDG 20.9.0+ |
| 多租户共享 GRE | if_name | IPv4 | [1-00032](task/UDG/20.15.2/1-00032.md) + 必配 GREKEYEN=TRUE+GREKEY（同源同宿多隧道）|
| GRE over IPSec | if_name | IPv4 | [1-00032](task/UDG/20.15.2/1-00032.md) + [1-00033](task/UDG/20.15.2/1-00033.md) IPSec 叠加加密 |
| 两层嵌套 GRE | if_name | IPv4 | [1-00032](task/UDG/20.15.2/1-00032.md) 执行两遍（外层+内层）|

> **关键规则**：
> - 源地址**不能与 IPSec 隧道源地址相同**（应用限制硬约束）
> - GREKey 两端必须一致（或两端都不配）
> - Tunnel MTU ≤ 出接口 MTU - GRE 头长度（避免分片）

## 约束

- **无需 License**（info）：本特性无需 License。
- **C-U 对称**（info）：UDG/UNC 两侧配置同构，ADD GRETUNNEL 对称。
- **源地址互斥 IPSec**（critical）：GRE 隧道源地址不能与 IPSec 隧道源地址相同 — 后果：应用限制违反。
- **隧道源/目的路由可达**（critical）：隧道源端口和目的端口已路由可达（PING 验证）— 后果：隧道 DOWN。
- **GREKey 两端一致**（warning）：GREKEYEN=TRUE 时两端 GREKey 必须相同 — 后果：报文丢弃。
- **Tunnel MTU**（warning）：建议 MOD INTERFACE 配置 MTU ≤ 出接口 MTU - GRE 头长度 — 后果：分片。
- **最多两层嵌套**（warning）：GRE 隧道嵌套层数最多 2 层 — 后果：超规格失败。
- **Keepalive 单向**（info）：KEEPALVEN 源端周期发送，对端不可达时本端 Tunnel 置 Down；两端都启用才能双向检测。
- **GRE 不加密**（warning）：GRE 只封装不加密不鉴权；需加密时走 GRE over IPSec 或直接用 IPSec。
- **首次发布版本**（info）：UDG 20.0.0；IPv6 GRE 需 UDG 20.9.0+（v02）。
