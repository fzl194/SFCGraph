# 设置APN上报属性（SET APNREPORTATTR）

> 命令静态知识见 [SET APNREPORTATTR](command/UDG/20.15.2/SET-APNREPORTATTR.md)

## 配置方法

配置指定 APN 的上报属性——性能统计/头增强/业务报表使用的 APN 类型，以及拥塞控制（小区负荷上报）。GWFD-110332 小区负荷上报无线资源优化场景仅开 CONGESTIONRPT=ENABLE，其余上报字段保持默认 SERVICE。

**配置范式**（110332）：

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| APN | apn-test | 本端规划（ADD APN） | 引用 0-00020 配置的 APN 名 |
| CONGESTIONRPT | ENABLE | 全网规划 | 开启小区负荷上报（需 RAN/PCRF 协同） |
| HEADERENRICH | SERVICE（默认） | 本端规划 | 上报给头增强的 APN 名 |
| PERFORMANCE | SERVICE（默认） | 本端规划 | 上报给话统的 APN 名 |
| SERVICEREPORT | SERVICE（默认） | 本端规划 | 业务报表使用的 APN 名 |

```
SET APNREPORTATTR:APN="apn-test",CONGESTIONRPT=ENABLE;
```

**REQUESTED 类型的别名 APN 解析**：当 HEADERENRICH/PERFORMANCE/SERVICEREPORT 任一取 REQUESTED 时，若用户激活请求里的 APN 是别名 APN，则上报到别名 APN 对应的真实 APN（UPF 支持虚拟/别名/普通 APN，别名 APN 非真实 APN）。主特性场景取 SERVICE，无别名解析需求。

**步骤位置**：特性首步（在 SET LICENSESWITCH 打开 license 之后）。

## 决策点

**DP 0-00140**（单选）：上报字段取 REQUESTED 时的别名 APN 解析编排（触发条件：HEADERENRICH/PERFORMANCE/SERVICEREPORT 任一=REQUESTED）。

| 选项 | 行为 | 适用 |
|---|---|---|
| 默认 SERVICE（本特性默认） | 上报字段取 SERVICE，无别名 APN 解析需求 | 主特性场景 |
| REQUESTED 解析到真实 APN | 上报到别名 APN 对应的真实 APN | 别名 APN 业务标识场景 |
| REQUESTED 直接使用别名 APN | 直接上报别名 APN 本身，无映射 | 别名 APN 即业务标识 |

## 约束

- 最大记录数 10000（整机 ApnReportAttr 条数） · warning · 超限 SET 失败
- 每个 APN 支持一条配置 · warning · 重复 SET 同 APN 覆盖
- 修改 PERF 统计 APN 前需先去活原 APN 下所有用户 · warning · 否则 perf 统计错误
- APN 必须先经 ADD APN 配置 · error · 否则引用失败
- CONGESTIONRPT=ENABLE 要求 RAN 侧和 PCRF 同时支持 · info
- 命令执行后立即生效 · info
