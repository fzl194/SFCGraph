# 设置APN上报属性（SET APNREPORTATTR）

> 命令静态知识见 [SET APNREPORTATTR](command/UDG/20.15.2/SET-APNREPORTATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。属"修改"语义。修改 PERFORMANCE 之前必须先去活该 APN 下所有用户。

## 配置方法

SET APNREPORTATTR 是 APN 级上报属性配置命令。**APN 必选**（指定要操作的 APN）；**HEADERENRICH/PERFORMANCE/SERVICEREPORT/CONGESTIONRPT 均为可选**，分别对应"使用请求 APN 还是用户真实使用 APN"四类上报语义。核心配置空间有 2 个维度：**APN 范围**（APN）、**上报维度**（HEADERENRICH/PERFORMANCE/SERVICEREPORT/CONGESTIONRPT）。1 个特性命中（GWFD-110332）覆盖拥塞控制使能范式。

### 配置维度 1：APN 范围（参数 APN，必选）

| 取值 | 作用 |
|---|---|
| **已建 APN 名**（ref [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN） | 对该 APN 设置上报属性 |

> APN 必须是已通过 `ADD APN` 创建的 APN；1~63 字符。

### 配置维度 2：上报维度（参数 HEADERENRICH/PERFORMANCE/SERVICEREPORT/CONGESTIONRPT，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **HEADERENRICH** | REQUEST_APN / ACTUAL_APN | 头增强使用的 APN 是用户请求的 APN 还是用户真正使用的 APN |
| **PERFORMANCE** | REQUEST_APN / ACTUAL_APN | 基于 APN 的性能统计是统计在请求 APN 还是真实使用 APN |
| **SERVICEREPORT** | REQUEST_APN / ACTUAL_APN | 业务报表使用的 APN 是请求 APN 还是真实使用 APN |
| **CONGESTIONRPT** | DISABLE / ENABLE | 当前 APN 下用户是否使能小区拥塞上报（GWFD-110332 范式） |

> **修改 PERFORMANCE 前必须先去活该 APN 下所有用户**（命令 notes 硬约束）。

**典型脚本（GWFD-110332 小区负荷上报无线资源优化，APN 拥塞上报使能）**：
```
SET APNREPORTATTR:APN="apn-test",CONGESTIONRPT=ENABLE;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：设置 APN 上报属性"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 建好目标 APN；CONGESTIONRPT=ENABLE 需配合无线侧上报链路。

## 决策点：上报维度选择（DP 0-00110）

（本命令无独立决策分支，上报维度按特性可选配。）

| 选项 | CONGESTIONRPT | 其他维度 | 适用场景 |
|---|---|---|---|
| **拥塞上报使能**（110332） | ENABLE | 不配 | GWFD-110332 小区负荷上报，无线资源优化 |
| **头增强 APN 选择** | 不配 | HEADERENRITH=ACTUAL_APN | 头增强按真实使用 APN 上报 |
| **性能统计 APN 选择** | 不配 | PERFORMANCE=ACTUAL_APN | 性能统计按真实使用 APN 统计 |
| **报表 APN 选择** | 不配 | SERVICEREPORT=ACTUAL_APN | 业务报表按真实使用 APN 报 |

> 1 个特性命中（110332）仅使用 CONGESTIONRPT=ENABLE；其他维度按需启用。

## 约束

- **APN 必须已建**（critical，rule-0-00110）：APN 必须是已通过 `ADD APN` 创建的 APN — 后果：APN 不存在命令拒收
- **修改 PERFORMANCE 前去活用户**（critical，rule-0-00110）：修改 PERFORMANCE 之前必须先去活原 APN 下所有用户 — 后果：未去活修改 PERFORMANCE 致统计异常
- **修改语义**（info，rule-0-00110）：本命令语义为"修改"，不创建/删除 APN；新建 APN 使用 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN — 后果：用于不存在的 APN 无效
- **执行后立即生效**（warning，rule-0-00110）：本命令执行后立即生效 — 后果：误操作即刻影响该 APN 下所有用户的上报链路
- **CONGESTIONRPT 与无线协同**（info，rule-0-00110）：CONGESTIONRPT=ENABLE 时需要无线侧支持并上报小区拥塞 — 后果：无线侧未支持时上报为空
