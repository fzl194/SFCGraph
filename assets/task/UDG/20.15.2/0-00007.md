# 配置流过滤器（ADD FLOWFILTER）

> 命令静态知识见 [ADD FLOWFILTER](command/UDG/20.15.2/ADD-FLOWFILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

FLOWFILTER 是聚合三四层（FILTER/FILTERIPV6）与七层（L7FILTER）过滤条件的容器。证据③中 8 个特性实例化时**只配 FLOWFILTERNAME**，过滤条件由后续 [ADD FLTBINDFLOWF](0-00008.md)/[ADD PROTBINDFLOWF](0-00009.md) 绑入。

**命名约定**（从证据③归纳，反映该流过滤器承载的过滤类型）：

| 命名 | 承载内容 | 典型场景 |
|---|---|---|
| `ff_L7` | 三四层 + 七层 URL | 内容计费 URL/HTTP 场景 |
| `ff_IMS` | IPv6 三四层 | 语音 IMS 场景 |
| `ff_L34any` | IPv4 全协议三四层 | 兜底/any 规则 |
| `ff_L34any_IPv6` | IPv6 全协议三四层 | 兜底/any 规则（IPv6） |
| `fg_red` / `flowfilter_test` | 单一三四层 | 在线/离线/PCC 单条规则 |

**铁律**：FLOWFILTER 必须至少绑一个 filter（FLTBINDFLOWF）或一个 l7filter（PROTBINDFLOWF），否则该流过滤器匹配不上任何报文，对应 RULE 无效。

**典型范式**：
```
ADD FLOWFILTER: FLOWFILTERNAME="ff_L7";
ADD FLTBINDFLOWF: FLOWFILTERNAME="ff_L7", FILTERNAME="L34_dns";
ADD PROTBINDFLOWF: FLOWFILTERNAME="ff_L7", PROTOCOLNAME="dns", L7FILTERNAME="L7_URL";
```

## 决策点

**DP 0-00057**（boolean, optional, always）是否配置 Tethering 检测？

| 选项 | 联动 | 适用 |
|---|---|---|
| opt-tether-default（默认） | 不配 TETHERDETFLAG | 多数计费/PCC/URL 过滤/重定向场景（27/29 特性） |
| opt-tether-none | TETHERDETFLAG=NONE | Proxy UPF 漫游（GWFD-112001）/双故障 Bypass（GWFD-112000）确保业务透传 |

> 另有 DP 0-00019（FLOWFILTERTYPE 普通/EXTEROTTDB）为演进建议，按"只记录不执行"边界，当前未补 FLOWFILTERTYPE binding；仅 GWFD-110321 选 EXTEROTTDB 需先 LOD EXTERNALDB 加载 OTT 规则库 + SET EXTOTTMATCHSW 使能。

## 约束

- **rule-0-00123**（critical）执行后**立即生效**；最大记录数 **100000**，超 90% 上报 ALM-135602215（阈值 `MOD CFGTHRESHOLD` 修）；必须至少绑定一个 filter 或 l7filter。
