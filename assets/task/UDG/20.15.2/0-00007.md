# 配置流过滤器（ADD FLOWFILTER）

> 命令静态知识见 [ADD FLOWFILTER](command/UDG/20.15.2/ADD-FLOWFILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD FLOWFILTER 是过滤链 backbone 的"流过滤器容器"：把已经配好的三四层 FILTER/FILTERIPV6 与七层 L7FILTER 通过 ADD FLTBINDFLOWF/ADD PROTBINDFLOWF 绑入一个 FLOWFILTER，供下游 ADD RULE 引用。本命令的核心配置空间有 3 个维度：**流过滤器类型**（FLOWFILTERTYPE，普通/外部 OTT 规则库）、**Tethering 检测开关**（TETHERDETFLAG）、**命名约定**（承载内容决定命名）。

### 配置维度 1：流过滤器类型（参数 FLOWFILTERTYPE）

| 取值 | 含义 | 配套操作 |
|---|---|---|
| **NORMAL**（默认） | 普通流过滤器，匹配三四层(FILTER/FLTBINDFLOWF)+七层(L7FILTER/PROTBINDFLOWF) 过滤条件；不需 OTT 规则库开关 | — |
| **EXTEROTTDB**（演进，2/29 特性用） | 匹配 LOD EXTERNALDB 加载的外部 OTT 业务规则库 | 须先 LOD EXTERNALDB 加载 OTT 规则库 + SET EXTOTTMATCHSW 全局开关使能 |

> 17/29 引用特性选 NORMAL（默认）；仅 GWFD-110321 选 EXTEROTTDB（按"演进只记录不执行"边界，本 atom 当前未补 FLOWFILTERTYPE binding，仅在 DP 0-00019 标注）。

### 配置维度 2：Tethering 检测开关（参数 TETHERDETFLAG，可选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **不配置**（默认） | 普通业务流过滤器，不关注 Tethering（终端共享上网）检测 | 多数计费/PCC/URL过滤/重定向场景（27/29 特性） |
| **NONE** | 明确不做 Tethering 检测 | Proxy UPF 漫游（GWFD-112001）/双故障 Bypass（GWFD-112000）确保业务透传 |

### 配置维度 3：命名约定（参数 FLOWFILTERNAME）

FLOWFILTERNAME 是 RULE.FLOWFILTERNAME 的引用锚点，命名反映该流过滤器承载的内容：

| 命名风格 | 典型取值 | 适用场景 |
|---|---|---|
| 按承载内容+协议族 | `ff_L7`（三四层+七层 URL）、`ff_IMS`（IPv6 IMS 语音）、`ff_L34any`（IPv4 any）、`ff_L34any_IPv6`（IPv6 any） | 部署 UPF 演示的全量脚本 |
| 通用默认 | `fg_red` / `flowfilter_test` / `flowfilter_test1` | 在线/离线/PCC 单条规则、URL过滤测试 |

**典型范式（FLOWFILTER + 绑定组合）**：
```
ADD FLOWFILTER: FLOWFILTERNAME="ff_L7";
ADD FLTBINDFLOWF: FLOWFILTERNAME="ff_L7", FILTERNAME="L34_dns";
ADD PROTBINDFLOWF: FLOWFILTERNAME="ff_L7", PROTOCOLNAME="dns", L7FILTERNAME="L7_URL";
```

**铁律**：FLOWFILTER 必须至少绑一个 filter（ADD FLTBINDFLOWF）或一个 l7filter（ADD PROTBINDFLOWF），否则所有报文都匹配不上该流过滤器，对应 RULE 无效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 4：配置流过滤器"——ADD FILTER/L7FILTER 之后，ADD FLTBINDFLOWF/ADD PROTBINDFLOWF 之前。

## 决策点：Tethering 检测开关（DP 0-00057）

问：是否对该流过滤器配置 Tethering 检测标识？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不配置（DEFAULT，默认） | 不配置 TETHERDETFLAG（缺省） | 普通业务流过滤器（27/29 特性） |
| 显式关闭 Tethering 检测（NONE） | TETHERDETFLAG=NONE | Proxy UPF 漫游（GWFD-112001）/双故障 Bypass（GWFD-112000）确保业务透传 |

> 另存演进 DP 0-00019（FLOWFILTERTYPE 普通/EXTEROTTDB）：遵循"演进只记录不执行"边界，本 atom 当前不补 FLOWFILTERTYPE binding；GWFD-110321 选 EXTEROTTDB 需先 LOD EXTERNALDB 加载规则库 + SET EXTOTTMATCHSW 全局开关使能。

## 约束

- **ADD FLOWFILTER 立即生效与记录数上限**（critical，rule-0-00123）：执行后**立即生效**；FlowFilter 最大记录数 **100000**，超规格 90% 上报 ALM-135602215；FLOWFILTER 必须至少绑一个 filter 或 l7filter — 后果：容量超限告警 / Rule 无效配置（未绑定过滤条件）
- **FLOWFILTERNAME 全网唯一**（critical，隐含）：不同流过滤器之间 FLOWFILTERNAME 不能重复 — 后果：重复新增失败
