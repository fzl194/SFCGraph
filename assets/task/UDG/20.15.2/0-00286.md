# 配置带宽用户组切片绑定（ADD SNSSAIBWMUSRG）

> 命令静态知识见 [ADD SNSSAIBWMUSRG](command/UDG/20.15.2/ADD-SNSSAIBWMUSRG.md)

## 配置方法

把 ADD SNSSAI 建好的切片（SST+SD）绑到一个已建好的带宽管理用户组（USERGROUPNAME）下，使该切片的用户受该组的 BWMRULE 管控。USERGROUPNAME / SST 必选、SD 可选。

| 参数 | 取值样例 | 说明 |
|---|---|---|
| USERGROUPNAME | `ug_test` | 必选，1~31 字符，由 ADD BWMUSERGROUP 建好；不能是默认/全局用户组 |
| SST | `1` | 必选，与上游 ADD SNSSAI 一致 |
| SD | `123456` | 可选，与上游一致；无 SD 时填全 F |

**典型脚本**：

```
// 把 SST=1、SD=123456 的切片绑到用户组 ug_test
ADD SNSSAIBWMUSRG:USERGROUPNAME="ug_test",SST=1,SD="123456";
```

**步骤位置**：切片带宽控制链路的第 3 步（终点）——在 ADD SNSSAI（0-00284）和 ADD SNSSAIUPINTF（0-00285）之后、BWM 三件套（BWMSERVICE/BWMCONTROLLER/BWMUSERGROUP/BWMRULE）已就绪后执行；本命令把切片这一维度接入已建好的用户组带宽规则。

## 决策点

**DP 0-00286**：切片绑定的目标带宽用户组选择——USERGROUPNAME 必须是一个已建好的具体用户组（SPECIFIC_GROUP），不能是默认/全局组。

| 选项 | USERGROUPNAME 取值 | 联动 |
|---|---|---|
| 绑到具体用户组 | `ug_test` 等 SPECIFIC_GROUP | 该切片用户受该组的 BWMRULE 管控 |
| 绑到默认/全局组 | DEFAULT_GROUP / 全局组 | **禁止**：手册约束，默认/全局组不能绑切片 |

**多组优先级**：若一个切片绑到多个 BwmUserGroup，只有优先级（USERGROUPPRI）最高的用户组生效——绑定时需与现有用户组优先级规划一致，避免被高优先级组"覆盖"导致规则不生效。

本命令为**高危命令**：操作不当会导致性能下降明显，执行前需评估对性能的影响，无法评估时联系华为技术支持。

## 约束

- 最大记录数 31984 · warning · 超规格新增失败
- 一个用户组下 APN + UserProfile + 切片的数量总和最大 16 · error · 维度总数上限
- 绑定切片的用户组不能是默认或全局用户组 · error · 仅 SPECIFIC_GROUP 可绑
- 被绑切片必须已通过 ADD SNSSAI 配置 · error · 前置依赖
- 切片绑多个 BwmUserGroup 时仅最高优先级组生效 · warning · 多组覆盖
- 用户更新可触发该配置对已在线用户生效，否则仅对新用户生效 · warning · 在线用户需触发更新
- 高危命令，可能导致性能下降 · error · 执行前评估性能影响
