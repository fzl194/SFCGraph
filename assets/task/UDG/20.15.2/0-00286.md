# 配置带宽用户组切片绑定（ADD SNSSAIBWMUSRG）

> 命令静态知识见 [ADD SNSSAIBWMUSRG](command/UDG/20.15.2/ADD-SNSSAIBWMUSRG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

把 ADD SNSSAI（0-00284）建好的切片（SST+SD）绑到一个已建好的带宽管理用户组（USERGROUPNAME）下，使该切片的用户受该组的 BWMRULE 管控。USERGROUPNAME / SST 必选、SD 可选。**本命令为高危命令**：操作不当会导致性能下降明显，执行前需评估对性能的影响，无法评估时联系华为技术支持。适用 NF：PGW-U、UPF。

### 配置维度 1：用户组与切片绑定（参数 `USERGROUPNAME` + `SST` + `SD`）

| 参数 | 取值 | 作用 | 配套必选/条件 |
|---|---|---|---|
| USERGROUPNAME | 字符串 1～31 | 已建好的带宽管理用户组名；不能是默认/全局用户组 | 必选；必须先建（参 [0-00033](task/UDG/20.15.2/0-00033.md) ADD BWMUSERGROUP） |
| SST | 整数 0～255 | 切片/服务类型，必须与上游 [0-00284](task/UDG/20.15.2/0-00284.md) ADD SNSSAI 一致 | 必选 |
| SD | 6 位十六进制 / `ffffff` | 切片区分码；与上游一致；不传时填全 F | 可选 |

**典型脚本**

```
// 把 SST=1、SD=123456 的切片绑到用户组 ug_test
ADD SNSSAIBWMUSRG:USERGROUPNAME="ug_test",SST=1,SD="123456";
```

**步骤位置**：切片带宽控制链路的第 3 步（终点）——在 ADD SNSSAI（0-00284）和 ADD SNSSAIUPINTF（0-00285）之后、BWM 三件套（BWMSERVICE/BWMCONTROLLER/BWMUSERGROUP/BWMRULE）已就绪后执行；本命令把切片这一维度接入已建好的用户组带宽规则。

## 决策点：切片绑定的目标带宽用户组（DP 0-00286）

**[[UDG@20.15.2@DecisionPoint@0-00286]]**（single_choice，USERGROUPNAME 是否默认/全局组）

| 选项 | USERGROUPNAME 取值 | 联动 |
|---|---|---|
| 绑到具体用户组 | `ug_test` 等 SPECIFIC_GROUP | 该切片用户受该组的 BWMRULE 管控 |
| 绑到默认/全局组 | DEFAULT_GROUP / 全局组 | **禁止**：命令约束，默认/全局组不能绑切片 |

**多组优先级**：若一个切片绑到多个 BwmUserGroup，只有优先级（USERGROUPPRI）最高的用户组生效——绑定时需与现有用户组优先级规划一致，避免被高优先级组"覆盖"导致规则不生效。

## 约束

- **命令记录上限**（warning，rule-0-00286-01）：最大记录数 31984，超规格新增失败。
- **维度上限**（critical，rule-0-00286-02）：一个用户组下 APN + UserProfile + 切片的数量总和最大 16，超出报错。
- **禁止绑默认/全局组**（critical，rule-0-00286-03）：绑定切片的用户组不能是默认或全局用户组，仅 SPECIFIC_GROUP 可绑。
- **前置依赖**（critical，rule-0-00286-04）：被绑切片必须已通过 [0-00284](task/UDG/20.15.2/0-00284.md) ADD SNSSAI 配置；USERGROUPNAME 必须已通过 [0-00033](task/UDG/20.15.2/0-00033.md) ADD BWMUSERGROUP 配置。
- **多组覆盖**（warning，rule-0-00286-05）：切片绑多个 BwmUserGroup 时仅最高优先级组生效。
- **在线用户生效条件**（warning，rule-0-00286-06）：用户更新可触发该配置对已在线用户生效，否则仅对新用户生效。
- **高危命令性能影响**（critical，rule-0-00286-07）：可能导致性能下降，执行前评估性能影响。
