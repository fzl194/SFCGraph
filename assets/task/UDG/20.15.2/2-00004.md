# 在线计费（GWFD-020300）

> 特性静态知识见 [GWFD-020300](feature/UDG/20.15.2/GWFD-020300.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

在线计费通过 OCS 实时配额管理：UDG 统计流量/时长上报 PGW-C→OCS，配额耗尽终止业务。架构：OCS ↔(Gy/DCC)↔ PGW-C ↔(N4/PFCP)↔ PGW-U(UDG)。

URR 取 `USAGERPTMODE=ONLINE` + `ONLMETERINGTYPE`（VOLUME/DURATION/EVENT/组合）。**复用内容计费的通用 backbone**（计费三件套/过滤链/规则绑定/收尾），差异在 URR 在线参数 + 在线专属（默认配额/失败动作）。

## 配置流程（步骤编排，每步显示含命令）

1. **License**（单命令）：`SET LICENSESWITCH`（`LKV3G5OLCH01`）
2. **计费三件套**（费率定义，URR=ONLINE）：`ADD URR`+`ADD URRGROUP`+`ADD PCCPOLICYGRP`（URR 的 `USAGERPTMODE=ONLINE`+`ONLMETERINGTYPE`+`DFTQUOTAVOLUME`/`DFTQUOTATIME` 默认配额）→ [1-00010](task/UDG/20.15.2/1-00010.md)
3. **过滤链**（业务匹配）：`ADD FILTER`+`ADD FLOWFILTER`+`ADD FLTBINDFLOWF` → [1-00009](task/UDG/20.15.2/1-00009.md)
4. **规则与用户模板绑定**：`ADD RULE`+`ADD USERPROFILE`+`ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
5. **计费收尾**（缺省费率+刷新）：`SET URRGRPBINDING`+`SET REFRESHSRV`（`REFRESHTYPE=USERPROFILE`，原始在线文档取值，非 ALL，见下"REFRESHTYPE 取值"脚注）→ [1-00012](task/UDG/20.15.2/1-00012.md)
6. **在线专属**（单命令，可选，对齐原始 activation 步骤 6/7/8）：
   - `SET UPDEFAULTQUOTA`（新业务默认配额开关，配合 DFTQUOTA 避免申请期间阻塞）→ [0-00016](task/UDG/20.15.2/0-00016.md)
   - `SET UPGLBCHGPARA`（Credit Pooling 全局开关，atom [0-00017](task/UDG/20.15.2/0-00017.md)，**可选，与 UPDEFAULTQUOTA 互斥**——activation 步骤 7 明列，两者只能开一个）
   - `SET URRFAILACTION`（URR 上报失败动作）→ [0-00018](task/UDG/20.15.2/0-00018.md)

> **REFRESHTYPE 取值**：本特性原始 activation（[配置Gy接口在线计费](evidence/UDG/20.15.2/配置Gy接口在线计费_83167737.md)）取 `REFRESHTYPE=USERPROFILE`（非 `ALL`）——在线计费流程在 `ADD RULE` 后即刷新 USERPROFILE 使规则生效，再建 `ADD USERPROFILE`+`ADD RULEBINDING`。具体取值以目标特性 activation 为准（[1-00012](task/UDG/20.15.2/1-00012.md) 的 `REFRESHTYPE` 由 DP 决定）。

## 决策点

**DP（特性级）：在线统计类型 + 默认配额 + 失败动作**（影响 URR + 在线专属）：

| 场景 | ONLMETERINGTYPE | 默认配额（DFTQUOTA + SET UPDEFAULTQUOTA） | SET URRFAILACTION |
|---|---|---|---|
| 流量计费 | VOLUME | 可选（申请配额期间用默认配额放通） | 配额超时阻塞/放通 |
| 时长计费 | DURATION | 同上 | 同上 |
| 事件计费 | EVENT | 同上 | 同上 |
| 综合 | EVENT_VOLUME_TIME | 同上 | 同上 |

**联动**：
- ONLMETERINGTYPE → URR（[1-00010](task/UDG/20.15.2/1-00010.md)）的在线统计维度。
- 默认配额 → URR 的 `DFTQUOTAVOLUME`/`DFTQUOTATIME` + `SET UPDEFAULTQUOTA` 使能开关；申请配额期间用默认配额放通，避免阻塞。
- URRFAILACTION → 配额耗尽申请新配额超时（`T3RESPONSE×N3REQUEST+4s`，SET UPPFCPPATH 配）后的动作。

## 约束（特性级 rule）

- **License 前置**：`SET LICENSESWITCH` 开启 `LKV3G5OLCH01`。
- **依赖特性**：[内容计费基本功能](feature/UDG/20.15.2/GWFD-020301.md)（GWFD-020301，基于业务粒度在线计费时需同开）+ [SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，业务识别）。
- **URR 在线一致性**（critical）：`URRID`/`USAGERPTMODE=ONLINE`/`ONLMETERINGTYPE` 在 PGW-C 与 PGW-U 必须一致。
- **USERPROFILE/RULE 一致性**：`USERPROFILENAME`/`RULENAME`/`POLICYTYPE` 在 PCRF/PGW-C/PGW-U 一致。
- **配额超时阻塞**（warning）：配额耗尽申请超时（`T3RESPONSE×N3REQUEST+4s`，SET UPPFCPPATH 配）→ PGW-U 阻塞业务。
- Credit Pooling（20.15.2+）：仅 Gy 接口、仅流量计费、不支持权重；**与 DefaultQuota（SET UPDEFAULTQUOTA）互斥**——两者只能开一个（见 atom [0-00017](task/UDG/20.15.2/0-00017.md)），由 domain DP 决定选哪个。
- 不支持 enhanced-ECUR/IEC 事件计费、紧耦合。
