# 会话管理（GWFD-010101）

> **能力型底座特性**——本特性是 UDG 的基本功能，**无需 License、无需 MML 配置、无激活步骤**。特性静态知识见 [GWFD-010101](feature/UDG/20.15.2/GWFD-010101.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-010101-会话管理.md)。
>
> **本页定位**：作为"底座骨架"显式记录**本特性是其他特性的被依赖锚点**——后续构建地址分配/PCC/接入控制等特性时，会话级 PFCP 信令的承载能力都依赖本特性的能力。本骨架 wiki 解决"会话承载"的可追溯性问题，不重复描述被依赖特性的配置。
>
> 构建依据：[特性步骤级构建SOP](task/特性步骤级构建SOP.md) §3.7 能力型底座特性范式。

## 配置概览

本特性作为 **UPF/PGW-U/SGW-U 用户面网元** 的底座能力，覆盖三种接入制式：

- **5G PDU 会话**（UDG 20.0.0+，PFCP/N4 接口）
- **4G EPS 承载**（UDG 20.0.0+，PFCP/Sxb 接口）
- **2/3G PDP 上下文**（UDG 20.3.2+，Gn/Gp 接口）

**核心动作**：UDG 侧**被动接收**控制面（SMF/SGW-C/PGW-C/SGSN）下发的 PFCP/GTP-C 信令，建立/修改/释放用户面会话上下文。本特性**不引入新命令、不引入新对象**，"配置生成"在本特性上**无内容可生成**。

**被依赖关系（本骨架显式声明）**：

| 会话建立依赖维度 | 被依赖特性 | 说明 |
|---|---|---|
| 地址分配（PDR UE IP Address/Pool Identity） | [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md) | UPF 侧地址池/分配规则配置 |
| PCC 策略（PDR/FAR/URR/QER 衍生） | [GWFD-020351](feature/UDG/20.15.2/GWFD-020351.md) | PCC 规则配置 |
| N4 接口（5G PFCP 信令通道） | [GWFD-010224](feature/UDG/20.15.2/GWFD-010224.md) | N4 接口偶联 |

| APN/DNN（会话维度的标识） | `ADD APN`（[0-00020](task/UDG/20.15.2/0-00020.md)） | APN 实例配置 |

## 配置流程（骨架——3 个被依赖维度的空骨架步骤）

> **每步不显示具体命令**——本骨架的所有"步骤"都**直接跳转到被依赖特性**生成配置。本骨架本身无配置步骤。

1. **会话建立依赖-地址分配**（地址池/地址分配规则）：无本骨架配置 → [1-00018](task/UDG/20.15.2/1-00018.md)（指向 [GWFD-010105](feature/UDG/20.15.2/GWFD-010105.md)）
2. **会话建立依赖-PCC 策略**（PCC 规则/动作/QoS）：无本骨架配置 → [1-00019](task/UDG/20.15.2/1-00019.md)（指向 [GWFD-020351](feature/UDG/20.15.2/GWFD-020351.md)）
3. **会话建立依赖-N4/Sxb/Gn 接口**（信令通道偶联）：无本骨架配置 → [1-00020](task/UDG/20.15.2/1-00020.md)（指向 [GWFD-010224](feature/UDG/20.15.2/GWFD-010224.md) / [GWFD-010233](feature/UDG/20.15.2/GWFD-010233.md) / 2-3G Gn 接口）
4. **会话建立触发**（运行时事件，非配置）：UE/SMF 触发 PFCP Session Establishment Request → UDG 被动响应；无需 UDG 配置
5. **会话修改/释放**（运行时事件）：SMF 触发 PFCP Session Modification/Deletion → UDG 被动响应；无需 UDG 配置

## 约束（特性级 rule）

- **无需 License**（info）：本特性无需获得 License 许可，无需配置即可使用。特性概述文档"License支持"章节明确"本特性无需配置即可使用"。
- **C-U 面偶联必须就绪**（critical）：所有会话信令交互的前提是 N4/Sxa/Sxb/Gn/Gp 偶联已建立 — 偶联配置归各接口特性。
- **UDG 网元基础配置完成**（critical）：UDG 已完成用户面逻辑接口配置。
- **控制面（UNC）已部署**（critical）：AMF/SMF、MME/SGW-C/PGW-C、SGSN/PGW-C 已部署并完成与 UDG 的偶联。
- **用户签约数据就绪**（critical）：UDM/HSS/HLR 已配置用户签约（DNN/APN、QoS、SSC mode、静态地址等）。
- **接入网就绪**（info）：(R)AN/eNodeB/RAN 已部署并提供空口承载。
- **承载数限制（4G/2-3G）**（warning）：每个用户可以激活的最大承载数为 **11**（无线侧限制，4G/2-3G 特性规格）。
- **PDN 连接-承载关系（4G）**（warning）：同一 PDN 连接必须且只能建立一个缺省承载；缺省承载删除则 PDN 连接释放；最后一个缺省承载只能在去附着流程中删除。
- **APN-PDN 一一对应（4G）**（warning）：一个 PDN 由一个 APN 代表；不同 APN 连接不同 PDN。
- **首次发布版本**（warning）：4G/5G N4/Sxb 接口首次发布版本为 UDG 20.0.0；2/3G Gn/Gp 接口首次发布版本为 UDG **20.3.2**（晚于 4G/5G）—— 配置 2/3G PDP 上下文会话需确认网元版本 ≥ 20.3.2。
