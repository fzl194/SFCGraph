# 增强的ADC基本功能（GWFD-020357）

> 特性静态知识见 [GWFD-020357](feature/UDG/20.15.2/GWFD-020357.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

增强的 ADC（Application Detection and Control）基本功能：UDG 检测应用（L7 协议特征库识别），通过 PGW-C/SMF 上报给 PCRF/PCF，PCRF/PCF 据此为应用数据流提供 QoS 保障。核心是**应用识别 + 流信息上报 + PCC 策略触发**。

activation 演示两种下发方式（两份任务实例）：
- **任务实例一·预定义规则**（PCRF/PCF 下发 APPLICATION_START/STOP Event-Trigger）：UDG 本地配全量（三四层 + 七层 URL + ADCPARA + 计费属性 + PCC 策略组 + ADC/PCC 规则 + 用户模板绑定），完整 9 步。
- **任务实例二·动态规则**（PCRF/PCF 下发携带 Application ID 的动态规则）：UDG 本地仅配三四层 + 七层 URL 识别（流信息上报开关以 SMF 下发为准），无 ADCPARA/RULE/USERPROFILE。

**核心机制**：应用识别依赖 SA-Basic（特征库加载）+ 七层过滤链（L7FILTER URL 匹配 + PROTBINDFLOWF 协议绑定）；ADC 专属参数 ADCPARA 控制流信息上报开关；PCCPOLICYGRP 与 RULE 的 `ADCMUTEFLAG=DISABLE`（检测到满足条件事件时 UPF 进行 ADC 上报，activation §数据规划表明示）。复用通用 backbone（[1-00009](task/UDG/20.15.2/1-00009.md) 过滤链含七层 + [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套 + [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定）。

> **关于 PCC 策略组业务属性链（PCCACTIONPROP/SERVICEPROP/SRVPBINDPCCPG）**：本特性 activation **未演示**这 3 条命令（[R1.5 子规则·可选命令证据链](task/特性task_wiki审视流程.md)）。ADC activation 演示的 PCC 触发走标准 PCCPOLICYGRP（绑 URRGROUP）+ RULE(PCC) 骨架。PCCACTIONPROP/SERVICEPROP/SRVPBINDPCCPG 是通用的 PCC 非默认业务属性组合机制（atom [0-00028](task/UDG/20.15.2/0-00028.md)/[0-00029](task/UDG/20.15.2/0-00029.md)/[0-00030](task/UDG/20.15.2/0-00030.md)，命令 wiki 有定义），用于按业务属性下发门控/转发等非默认动作（SA/QoS 等场景）。ADC 场景按 activation 演示走标准 PCCPOLICYGRP；若现网需按业务属性组合下发非默认动作，配置生成**默认不产出**这 3 条命令（activation 无证据），待补证据。

## 决策点

**DP（特性级）：规则下发方式 + 规则类型 + 是否需计费**（影响是否配 ADCPARA / RULE 走 ADC 还是 PCC / 是否配计费三件套）。维度按 activation §操作步骤4 注 + 两份任务实例全列。

| 场景 | 规则下发方式 | ADCPARA | 规则类型（[1-00011](task/UDG/20.15.2/1-00011.md) POLICYTYPE） | 计费三件套 | 走法 |
|---|---|---|---|---|---|
| **预定义规则·ADC 专属规则（activation 实例一部分演示）** | PCRF/PCF 下发 APPLICATION_START/STOP | 配（FLOWINFORPT=ENABLE）| **ADC**（ADCMUTEFLAG=DISABLE，单独 ADC 规则）| 配（URR+URRGROUP，PCCPOLICYGRP ADCMUTEFLAG=DISABLE）| 骨架 + [1-00009](task/UDG/20.15.2/1-00009.md) L34+L7 + [0-00031](task/UDG/20.15.2/0-00031.md) ADCPARA + [1-00010](task/UDG/20.15.2/1-00010.md) + [1-00011](task/UDG/20.15.2/1-00011.md) |
| **预定义规则·复用 PCC 规则（activation 实例一部分演示）** | PCRF/PCF 下发 APPLICATION_START/STOP | 配 | **PCC**（ADCMUTEFLAG=DISABLE，复用 PCC 规则带 ADC 属性）| 配 | 同上，RULE 走 POLICYTYPE=PCC |
| **动态规则（activation 实例二演示）** | PCRF/PCF 下发携带 Application ID 的动态规则 | **不配**（流信息上报开关以 SMF 下发为准）| **不配**（规则由 PCRF/PCF 动态下发）| **不配**（UDG 侧仅识别）| 仅步骤 1~3（License + 三四层 + 七层 URL 识别）|
| 仅策略不计费变体 | 预定义/动态任一 | 按场景 | ADC/PCC | **不配**（PCCPOLICYGRP 不配 URRGROUPNAME）| 省 [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套 |

**联动**：
- 规则下发方式 → 是否配 ADCPARA / RULE / USERPROFILE：预定义规则场景 UDG 全量配；动态规则场景 UDG 仅配识别（三四层 + 七层 URL）。
- 规则类型 → [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定 POLICYTYPE：ADC 专属规则（POLICYTYPE=ADC）与 PCC 计费规则（POLICYTYPE=PCC）可**共用同一 RULENAME + FLOWFILTERNAME**（activation 实例一演示双 RULE 同名 rule_test2），PCC 规则的 POLICYNAME 指向 PCCPOLICYGRPNM。
- ADCMUTEFLAG=DISABLE → RULE 与 PCCPOLICYGRP 均须设 DISABLE（检测到事件时 UPF 进行 ADC 上报；activation §操作步骤6 强制）。
- 是否需计费 → [1-00010](task/UDG/20.15.2/1-00010.md) 计费三件套：计费时 PCCPOLICYGRP 绑 URRGROUPNAME；纯识别不计费可省。

## 约束（特性级 rule）

- **License 前置**（critical）：`SET LICENSESWITCH` 开启 `LKV3G5ADCF01` — 未开则 ADC 功能不生效。
- **依赖特性**（critical）：[SA-Basic](feature/UDG/20.15.2/GWFD-110101.md)（GWFD-110101，协议特征库加载 + 七层应用识别基础）+ [PCC 基本功能](feature/UDG/20.15.2/GWFD-020351.md)（GWFD-020351，PCC 策略触发链，activation §必备条件明列"完成激活PCC基本功能"）。
- **ADCMUTEFLAG=DISABLE 强制**（critical）：配置增强的 ADC 基本功能时，RULE 与 PCCPOLICYGRP 的 `ADCMUTEFLAG` 必须设置为 `DISABLE`（检测到满足条件事件时 UPF 进行 ADC 上报；activation §操作步骤6 注明示）— 设为 ENABLE 则不上报。
- **ADCPARA 仅预定义规则场景**（warning）：PCRF/PCF 经预定义规则下发 APPLICATION_START/STOP Event-Trigger 时才配 ADCPARA（开流信息上报开关）；动态规则场景流信息上报开关以 SMF 下发为准，UDG 不配。
- **七层过滤依赖 SA 特征库**（critical）：L7FILTER URL 匹配 + PROTBINDFLOWF 协议绑定依赖 SA-Basic 加载的特征库（LOD SIGNATUREDB + LOD PARSERDB）— 未加载则应用识别失败。
- **FLOWFILTER 至少绑一个过滤条件**（critical）：三四层或七层至少其一 — activation 演示三四层 any + 七层 URL 双绑。
- **REFRESHSRV 必须执行**（critical）：三四层/七层 Filter 变更后须 `SET REFRESHSRV`（REFRESHTYPE=ALL）才生效（activation 步骤 2d/3c 明列）。
- **跨网元 FLOWFILTERNAME 一致**（warning）：PCRF/PCF 下发动态规则携带 appid 时，PGW-C/SMF、PGW-U/UPF、PCRF/PCF 三个网元上的 FLOWFILTERNAME 必须一致（activation §表1 注）。
- **PCCACTIONPROP/SERVICEPROP/SRVPBINDPCCPG activation 未演示**（info）：ADC activation 走标准 PCCPOLICYGRP 骨架；业务属性组合链（atom [0-00028](task/UDG/20.15.2/0-00028.md)/[0-00029](task/UDG/20.15.2/0-00029.md)/[0-00030](task/UDG/20.15.2/0-00030.md) 命令 wiki 有定义，SA/QoS 等场景用），ADC 场景配置生成默认不产出，待补证据。
