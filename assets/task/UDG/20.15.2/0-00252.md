# 增加内容过滤策略（ADD CFPROFILE）

> 命令静态知识见 [ADD CFPROFILE](command/UDG/20.15.2/ADD-CFPROFILE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务创建一个 URL 内容过滤策略（策略容器），供后续 ADD CFPROFBINDCFT 将其绑定到内容过滤模板。本特性场景下仅需指定全网规划的策略名。适用 NF：PGW-U、UPF。

**取值样例（数据规划表，GWFD-110471）**

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| CFPROFILENAME | `cf1` | 全网规划 | 跨网元一致的策略名，供 ADD CFPROFBINDCFT 引用 |

**脚本样例**

```
ADD CFPROFILE: CFPROFILENAME="cf1";
```

**未绑参数说明（按 baseline §4.1 冷门不绑）**：PRIORITY（优先级，optional 默认 65535）、CFGDOMAINNAME（配置域名称，optional）主特性场景不配，采用命令默认。策略本身为空容器，实际过滤动作由其绑定的模板/分类组决定。

## 约束

- **rule-0-00388**（warning）命令最大记录数 **500**，超限 ADD 失败；执行后立即生效；CFPROFILENAME 取值**不建议与 RULENAME/USERPROFILENAME 重复**，否则可能导致内容过滤功能失效；CFPROFILENAME 字符串 1～63 字符、不支持空格、不区分大小写。
