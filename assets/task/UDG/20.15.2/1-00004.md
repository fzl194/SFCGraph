# PCC 预定义规则链

> BWM 配置的场景特化阶段（仅 PCC 预定义规则场景）。把 BWM 策略挂到 PCC 规则体系，使 PCF/PCRF 能经 N4 动态安装/删除预定义规则触发限速。归属特性 [2-00001](task/UDG/20.15.2/2-00001.md)。

## 配置方法

核心是**命名一致性铁律**：`BWMSERVICE.CATEPROPNAME` 必须 == `RULE.POLICYNAME`；`RULENAME` 须与 PCF/PCRF + UNC 三方规划一致。

命令序列：
1. **ADD CATEGORYPROP** —— 定义分类属性（如 `cp_700_600`）
2. **ADD BWMSERVICE** —— NONTOSSRVTYPE=CATEGORY_PROP，CATEPROPNAME 指向上一步（控制器在 [1-00002](task/UDG/20.15.2/1-00002.md)）
3. **过滤链** —— ADD L7FILTER + ADD FLOWFILTER + ADD PROTBINDFLOWF（识别触发流）
4. **ADD RULE** —— POLICYTYPE=BWM，POLICYNAME=CATEPROPNAME，FLOWFILTERNAME 指向过滤链
5. **SET REFRESHSRV** —— 涉及 Rule/L7Filter 必须刷新

**典型脚本**：
```
ADD CATEGORYPROP:CATEPROPNAME="cp_700_600";
ADD BWMSERVICE:BWMSERVICENAME="my-bwm-service1",BWMSERVICETYPE=NONTOS,NONTOSSRVTYPE=CATEGORY_PROP,CATEPROPNAME="cp_700_600";
ADD L7FILTER:L7FILTERNAME="l7-p2p",SUBL7FLTNAME="sl7-p2p",URL="www.example.com*/*";
ADD FLOWFILTER:FLOWFILTERNAME="flowfilter_p2p";
ADD PROTBINDFLOWF:FLOWFILTERNAME="flowfilter_p2p",PROTOCOLNAME="http",L7FILTERNAME="l7-p2p";
SET REFRESHSRV:REFRESHTYPE=USERPROFILE;
ADD RULE:RULENAME="rule_p2pbwm",POLICYTYPE=BWM,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="flowfilter_p2p",PRIORITY=1,POLICYNAME="cp_700_600";
```

**步骤位置**：在 [1-00002](task/UDG/20.15.2/1-00002.md)（业务与控制器）之后、[1-00003](task/UDG/20.15.2/1-00003.md)（用户组与规则）之前。承接 BWM 控制器（`UPBWMCTRLNAME1`/`DNBWMCTRLNAME1`）。

## 决策点

**DP：预定义规则触发流** —— L7FILTER 的 URL/协议匹配范围（决定哪些流量触发 FUP 限速）。
**DP（联动）：RULENAME 三方对齐** —— 须与 PCF/PCRF + UNC 预定义规则名一致，否则 PCF 安装时匹配不上。

## 约束

- **命名一致性（critical）**：`CATEPROPNAME` == `RULE.POLICYNAME`；`RULENAME` 三方一致。违反→规则不生效。
- **必须刷新**：涉及 Rule/L7Filter 变更，`SET REFRESHSRV:REFRESHTYPE=USERPROFILE`。
- **典型用途**：FUP（套餐耗尽降速）、PRA 区域限速。
