# 配置TWAMP响应端Full模式（ADD TWAMPRESPONDER）

> 命令静态知识见 [ADD TWAMPRESPONDER](command/UDG/20.15.2/ADD-TWAMPRESPONDER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110921（TWAMP）**Full 模式独有**的第 4 步——UDG 作响应端被动响应 eNodeB/gNodeB 发起的探测（走 TCP TWAMP-Control 协商 + UDP TWAMP-Test）。Light 模式不用此命令。Full 模式因走 TCP 控制连接，后续须配套 SET TCPKEEPALIVECFG（0-00229）。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| RESPONDERID | 响应端索引 | `1` | 全网规划，1~32 |
| AFTYPE | 地址族类型 | `IPV4` | 固定 IPV4 |
| TWAMPARCH | TWAMP 架构 | `FULL` | 固定 FULL（本任务核心标识） |
| LOCALIPV4 | 本端 IP | `10.1.1.10` | 引用 [创建TWAMP逻辑接口](task/UDG/20.15.2/0-00225.md) 的 IPV4ADDRESS（条件必填，AF=IPV4 时） |
| SERVWAIT | 协商等待时间（秒） | `900` | 本端规划，10~1800，默认 900（可选） |
| REFWAIT | 测试等待时间（秒） | `900` | 本端规划，10~1800，默认 900（可选） |

**脚本样例**

```
ADD TWAMPRESPONDER:RESPONDERID=1,AFTYPE=IPV4,TWAMPARCH=FULL,LOCALIPV4="10.1.1.10",SERVWAIT=900,REFWAIT=900;
```

**操作步骤定位**：紧接 ADD TWAMPLOGICINF 之后、SET TCPKEEPALIVECFG 之前。LOCALIPV4 必须与上游逻辑接口 IP 一致（feature-rule 0-00081 TWAMP 一致性约束）。模式互斥由 feature-DP 0-00036 承担（FULL/LIGHT 二选一）。
