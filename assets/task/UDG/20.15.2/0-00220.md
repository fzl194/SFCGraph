# 配置会话核查功能（SET SESSCHKFUNC）

> 命令静态知识见 [SET SESSCHKFUNC](command/UDG/20.15.2/SET-SESSCHKFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

单命令原子，全局唯一记录（SET 修改）。激活 GWFD-020162 唯一一步：设置会话核查配置。控制 SGW-U/PGW-U/UPF 在 6 种触发条件下向 SGW-C/PGW-C/SMF 发 Report 消息核查会话，清理残留 EPS 承载/PDU 会话。系统默认全使能，现网多为确认或微调阈值。适用 SGW-U/PGW-U/UPF。

**数据规划取值样例**（证据③，GWFD-020162/配置用户面会话过载控制_43675114.md 数据规划表）：

| 参数 | 含义 | 取值样例 | 获取方法 | 触发条件 |
|---|---|---|---|---|
| ESTABLISHCHK | 会话创建核查开关 | ENABLE | 本端规划 | 新建会话时核查同 IMSSI 的存量会话 |
| SESSNUMTHD | 会话创建核查阈值 | 4 | 本端规划 | 同 IMSSI 会话数达此值触发核查（仅 ESTABLISHCHK=ENABLE 生效，范围 1~21） |
| PATHDWNCHK | 路径断核查开关 | ENABLE | 本端规划 | N4 链路从断恢复后核查 |
| DAYCHK | 每日核查开关 | ENABLE | 本端规划 | 每天凌晨四点核查 |
| SGRECOVERYCHK | SG 恢复核查开关 | ENABLE | 本端规划 | 进程故障恢复后核查 |
| SESSLIMITCHK | 会话规格限制核查开关 | ENABLE | 本端规划 | 会话资源达 90% 时核查 |

**示例脚本**（取自激活 md）：

```
SET SESSCHKFUNC:ESTABLISHCHK=ENABLE,SESSNUMTHD=4,PATHDWNCHK=ENABLE,DAYCHK=ENABLE,SGRECOVERYCHK=ENABLE,SESSLIMITCHK=ENABLE;
```

**配置要点**：
- 6 参数配合实现 6 种核查场景，系统初始记录默认全使能（SESSNUMTHD=4）——本特性取值与初始值完全一致，多为显式确认。
- SESSNUMTHD 仅在 ESTABLISHCHK=ENABLE 时生效，取值范围 1~21；超范围被拒。
- 关闭全部核查 → UDG 上残留 EPS 承载/PDU 会话无法清理，逐渐堆积至过载。
- Report 消息走 N4/Sx 接口，核查不阻断新会话创建流程。

## 约束

- 最大记录数 1（全局唯一）；该命令执行后立即生效。
- SESSNUMTHD 取值范围 1~21，且仅 ESTABLISHCHK=ENABLE 时生效。
- 存在系统初始记录，默认全使能、SESSNUMTHD=4。
- 全部关闭导致残留会话无法清理，引发过载。
