# 设置License开关（SET LICENSESWITCH）

> 命令静态知识见 [SET LICENSESWITCH](command/UDG/20.15.2/SET-LICENSESWITCH.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

SET LICENSESWITCH 是特性激活的**通用前置开关**——跨 56+ 特性复用。每个特性有自己的 License 控制项码（LICITEM），配本命令打开该特性的 License 后才能配该特性的业务命令。本命令的核心配置空间有 3 个维度：**LICITEM 来源**（每特性固定取一个码，由 feature 决定）、**开/关**（ENABLE / DISABLE 二选一，决定特性激活还是去激活）、**前置**（License 文件必须已激活）。

### 配置维度 1：LICITEM 来源（每特性固定取一个）

每特性有固定的 License 控制项码（字符串 0~255 字符），格式 `LKV3G5xxxxxx`：

| 特性 / 域 | LICITEM | 配套业务命令 | 说明 |
|---|---|---|---|
| 动态 PCC / 本地 PCC（GWFD-020351） | `LKV3G5PCCB01` | [ADD PCCPOLICYGRP](0-00003.md) | PCC 策略组前置 |
| TCP 重传识别（GWFD-020307） | `LKV3G5TCPR01` | [SET URRGRPBINDING](0-00013.md) TCP 子集 | TCP 重传计费前置 |
| 终端异常下行流量检测（GWFD-020305） | `LKV3G5ADTD01` | 见特性 md | APN 下异常流量检测前置 |
| HTTP2.0 协议回落（GWFD-110202） | 特性对应码 | 见特性 md | 见特性 md |
| Routing Behind MS（GWFD-110910） | 特性对应码 | 见特性 md | 见特性 md |
| IPv6 承载上下文（GWFD-020401） | 特性对应码 | 见特性 md | 见特性 md |
| IPv4v6 双栈接入（GWFD-020403） | 特性对应码 | 见特性 md | 见特性 md |
| 其他 50+ 特性 | 各特性对应码 | 见各 feature | 见各 feature wiki |

> LICITEM 与特性 ID 是 1:1 关系——本字段不在本 atom 开 DP（由 feature-level decision 决定）。具体取值由所属 feature md 提供。

### 配置维度 2：开 / 关（参数 SWITCH）

| 取值 | 作用 | 适用 |
|---|---|---|
| `ENABLE` | 打开特性 License——该特性的业务命令可被允许 | 激活特性 / 配置该特性 |
| `DISABLE` | 关闭特性 License——该特性的业务命令将被拒绝 | 去激活特性 / 临时禁用 |

> 同一 LICITEM 多次开关不会出错；但建议同一 nf@version 中**特性 ID 与 LICITEM 码保持 1:1**——避免多 LICITEM 误绑。

### 配置维度 3：前置约束（License 文件已激活）

| 前置条件 | 含义 | 后果 |
|---|---|---|
| License 文件已激活 | 必须先激活 License 文件（本命令外的"加载 License"操作） | 未激活时 SET LICENSESWITCH 失败 |
| 已购买且支持配置开关 | 仅对**已购买且支持配置开关**的 License 项生效 | 未购买项设开关无效（命令注：依据 License 文件实际内容生效） |

**步骤位置**：在所属 feature 编排中是**该特性的第 1 步**——打开 License 后才配该特性的业务命令。

## 决策点：无（本命令仅两个参数，每特性 LICITEM 来源由 feature wiki 决定）

> 是否调用本命令、按哪个 LICITEM 调用由 feature-level decision 决定——本 atom 作为**通用前置命令**开放给所有 feature 引用，不在本 atom 内重复开特性级 DP。

## 约束

- **前置：License 文件必须已激活**（critical，隐含）：License 文件未激活时 `SET LICENSESWITCH` 失败 — 后果：未激活文件下打开所有开关均失败
- **前置：仅已购买且支持配置开关**（warning，隐含）：激活后只能设置已购买且支持配置开关的 License 项 — 后果：未购买项设开关不报错也不生效
- **执行顺序铁律**（critical，隐含）：本命令是特性配置的**第 1 步**（打开 License），之后才配该特性的业务命令 — 后果：未开 License 直接配业务命令失败
