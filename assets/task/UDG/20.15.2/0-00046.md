# 配置APN地址分配属性（SET APNADDRESSATTR）

> 命令静态知识见 [SET APNADDRESSATTR](command/UDG/20.15.2/SET-APNADDRESSATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET APNADDRESSATTR 给指定 APN 配地址分配属性开关。**APN 引用上游 ADD APN 的 APN（0-00020，`LST APN` 可查）**，每个 APN 仅一条本配置。本命令属高危命令，按场景设相关 flag（任一特性按需设其一）。

**按场景的开关范式（数据规划表归纳）**

| 场景 | 开关参数 | 取值 | 典型特性 |
|---|---|---|---|
| 主机地址路由接入 | HOSTROUTEIP=ENABLE | 使能该 APN 支持主机地址接入 | 地址分配方式（010104） |
| 手机后路由接入 | FRAMEDROUTE=ENABLE | 使能 Routing Behind MS（手机后路由用户接入） | 支持 Routing Behind MS（110910） |
| 指定双栈地址支持 | SUPPORTIPV4=ENABLE + SUPPORTIPV6=ENABLE | 该 APN 支持 IPv4/IPv6 地址 | 多地址分配子场景（010105/020406/020421） |
| 支持 SMF 下发地址池名 | IGNOREV4POOLID=DISABLE + IGNOREV6POOLID=DISABLE | 该 APN 支持 SMF 下发的地址池名（不忽略） | 基于 RADIUS 下发地址池名分配（010105） |

**典型行**

主机地址路由接入（010104）：
```
SET APNADDRESSATTR: APN="apn-test", HOSTROUTEIP=ENABLE;
```
Routing Behind MS（110910）：
```
SET APNADDRESSATTR:APN="apn1",FRAMEDROUTE=ENABLE;
```
指定双栈地址支持（多数地址分配子场景，配完 ADD APN 后紧跟此步）：
```
SET APNADDRESSATTR:APN="apn-test1",SUPPORTIPV4=ENABLE,SUPPORTIPV6=ENABLE;
```
支持 SMF 下发地址池名（010105 RADIUS 场景补 IGNORE* 两个 DISABLE）：
```
SET APNADDRESSATTR:APN="apn-test1",SUPPORTIPV4=ENABLE,SUPPORTIPV6=ENABLE,IGNOREV4POOLID=DISABLE,IGNOREV6POOLID=DISABLE;
```

**步骤位置**：本命令通常紧跟在 ADD APN 之后、ADD POOL/SECTION/POOLBINDGROUP/POOLGRPMAP 链路之前——先使能 APN 地址属性，再建地址池族并映射。

**未绑冷门可选**：ANTISPOOFINGDL/UL、IPTYPEFORDUALIP、SINGLEADDRCAUSE、CPCTRL、IPV6RAMTUSW、IPV6MTU、IPV6RALIFETIME 等 12 个命令真相 optional/conditional 参数，2 特性数据规划表均未用到，按通用配置门槛不绑。

## 约束

- **rule-0-00151**（warning）最大记录数 10000；本命令属高危命令；执行后立即生效 — 后果：超规格新增失败 / 高危操作可能影响用户激活
- **rule-0-00152**（warning）每个 APN 仅一条地址属性配置（APN 唯一）；前置必须先 ADD APN — 后果：同 APN 重复配置失败 / 未先建 APN 无目标可挂
