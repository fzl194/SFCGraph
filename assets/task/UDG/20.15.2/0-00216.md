# 创建VNRS侧IPsec隧道接口并应用安全策略（ADD IPSECINTFCFG）

> 命令静态知识见 [ADD IPSECINTFCFG](command/UDG/20.15.2/ADD-IPSECINTFCFG.md)、[ADD IPSECINTFCFGIPSEC](command/UDG/20.15.2/ADD-IPSECINTFCFGIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

聚合 atom：先 [ADD IPSECINTFCFG](command/UDG/20.15.2/ADD-IPSECINTFCFG.md) 在 VNRS 侧把 Tunnel 接口标记为 IPsec 隧道协议（TNLTYPE=IPSEC/IPSEC6），再到 IPSEC 侧用 [ADD IPSECINTFCFGIPSEC](command/UDG/20.15.2/ADD-IPSECINTFCFGIPSEC.md) 把上一步建好的 IPsec 安全策略挂到该隧道接口上。这两步是 VNRS+IPSEC 双配原则的最终生效点，缺一不可。

**数据规划取值样例**（证据③，IPFD-015004 各激活 md 数据规划表）：

| 场景 | INTERFACENAME | TNLTYPE | 后续 IPSEC 侧 POLICYNAME |
|---|---|---|---|
| 普通 IPv4 IPsec 隧道 | Tunnel10 | IPSEC | 引用 0-00215 产出的 POLICYNAME |
| IPv4 IPsec 主备隧道 | Tunnel17 | IPSEC | 每条隧道各引用一条 POLICYNAME |
| GRE over IPsec / OSPF over IPsec / 多 Sequence / 指定本端接口 | Tunnel2 / Tunnel10 | IPSEC | 同上 |
| 普通 IPv6 IPsec 隧道 / IPv6 主备 | Tunnel27 | IPSEC6 | IPv6 策略同链路 |

**示例脚本**（取自激活 md，IPv4/IPv6 仅 TNLTYPE 与接口名不同）：

```
ADD IPSECINTFCFG:INTERFACENAME="Tunnel10",TNLTYPE=IPSEC;      // IPv4 隧道
ADD IPSECINTFCFG:INTERFACENAME="Tunnel27",TNLTYPE=IPSEC6;     // IPv6 隧道
// 紧接在 IPSEC 微服务侧补一条（atom 0-00216 的第二半）：
// ADD IPSECINTFCFGIPSEC:INTERFACENAME="Tunnel10",POLICYNAME=<0-00215 产出>;
```

**配置要点**：
- TNLTYPE 固定取 `IPSEC`（IPv4 隧道）或 `IPSEC6`（IPv6 隧道），不取其它枚举值——这正是该接口成为 IPsec 隧道的关键。
- INTERFACENAME 引用前置 [ADD INTERFACE](task/UDG/20.15.2/0-00056.md) 产出的 IFNAME（同一 Tunnel 口）。
- POLICYNAME 引用前置 [配置并组装IPsec安全策略](task/UDG/20.15.2/0-00215.md) 产出的策略名。
- 多隧道（主备/多感兴趣流）场景：每个 Tunnel 接口重复一遍 VNRS 侧建口 + IPSEC 侧挂策略两步。

## 约束

- 最大记录数 1024；该命令执行后立即生效。
- INTERFACENAME 必须先经 [ADD INTERFACE](task/UDG/20.15.2/0-00056.md) 创建（前置 IFNAME）。
- TNLTYPE 必须 = IPSEC/IPSEC6，否则该接口不作为 IPsec 隧道生效。
- POLICYNAME 必须先经 [ADD IPSECPOLICY](task/UDG/20.15.2/0-00215.md) 创建。
- VNRS+IPSEC 双配：必须同时执行 ADD IPSECINTFCFG 与 ADD IPSECINTFCFGIPSEC，单配 VNRS 侧隧道不生效。
