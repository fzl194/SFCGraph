# DNS纠错（GWFD-110283）

> 特性静态知识见 [GWFD-110283](feature/UDG/20.15.2/GWFD-110283.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性与 HTTP智能重定向([2-00017](task/UDG/20.15.2/2-00017.md))同属**智能重定向族**，共享动作策略 compound [1-00015](task/UDG/20.15.2/1-00015.md)；差异：**DNS层IP改写**（DNSOVERWRITING→SERVERIP1）vs **HTTP层URL重定向**（SMARTHTTPREDIR→SERVERURL）、**错误码 EQUAL=3** vs **GT400**、**无重定向携带信息**（DNS 不配 REDIRAPPENDINFO）。

## 配置概览

DNS 纠错在用户访问错误域名（DNS 响应返回 NXDOMAIN，错误码=3）时，将 DNS 响应报文中的 IP 地址改写为第三方 Platform IP，引导用户访问第三方 Platform。**核心机制：RULE 经 `POLICYTYPE=SMARTREDIRECT` 直接引用 DNSOVERWRTNAME（ADD DNSOVERWRITING 定义的动作对象），与 HTTP 智能重定向同属 SMARTREDIRECT 体系，独立于 PCC/HEADEN 的第 3 种 POLICYTYPE，不经 PCCPOLICYGRP、不经 IPFarm**，SERVERIP1 在动作命令直配。

动作命令 ADD DNSOVERWRITING 绑定扩展过滤器（URL 匹配用户输错的域名）+ 错误码（EQUAL=3 NXDOMAIN）→ 改写 DNS 响应 IP 为 SERVERIP1（activation 演示 `192.168.1.1`）。**与 HTTP 智能重定向对比：DNS 纠错不配重定向携带信息（无 REDIRAPPENDINFO），只改写 IP；HTTP 智能重定向改写 URL 且可携带 REQURL/IMSI 等**。多场景差异由 DP 驱动（触发匹配/错误码操作/改写层/规则下发方式/FILTERINGMODE）。

> **283 activation 无计费/无 USERPROFILE/RULEBINDING**：activation §任务描述明示"本例采用 PCRF 下发预定义规则的方式进行说明"，无 URR/URRGROUP/PCCPOLICYGRP、无 USERPROFILE、无 RULEBINDING（采用 PCRF 下发预定义规则方式）。配置流程忠实反映 activation，不编造计费/用户模板步骤。如需本地绑定规则，可复用 [1-00011](task/UDG/20.15.2/1-00011.md) 的 USERPROFILE+RULEBINDING。

> **前置依赖 PCC 基本功能**（critical）：activation §必备事项明示"完成激活 PCC 基本功能 [GWFD-020351]"——DNS 纠错建立在 PCC 基本功能之上，未完成 PCC 基本功能则 DNS 纠错不生效。

## 决策点

**DP（特性级）：触发匹配 + 错误码操作 + 改写层 + 规则下发方式 + FILTERINGMODE**（影响扩展过滤器组合 + ADD ERRORCODE 参数 + 动作命令选择 + ADD RULE 走法）。维度按 activation §数据规划表 + §操作步骤 全列。DNS 纠错与 HTTP 智能重定向的关键差异维度（改写层/错误码操作）在表中显式对比。

| 场景 | 触发匹配 | 错误码操作 | 改写层 | 规则下发方式 | FILTERINGMODE | 走法 |
|---|---|---|---|---|---|---|
| **标准场景（activation 演示）** | URL（1 扩展过滤器，www.example.com/*）| EQUAL, ERRORCODESTART=3（NXDOMAIN）| **DNS 层 IP 改写**（DNSOVERWRITING→SERVERIP1）| PCRF 下发预定义规则（无 USERPROFILE/RULEBINDING）| FLOWFILTER | 骨架[1-00009]+族 compound[1-00015]+atom RULE |
| 仅 URL 触发变体 | URL（1 扩展过滤器）| EQUAL, 3 | DNS 层 IP 改写 | 任意 | FLOWFILTER | 同上 |
| 错误码范围变体 | URL | GT/GE/LE/LT, START+END | DNS 层 IP 改写 | 任意 | FLOWFILTER | 同上（[1-00015](task/UDG/20.15.2/1-00015.md) §场景差异 §2 ERRORCODEOP 变体）|
| 本地绑定规则变体 | URL | EQUAL, 3 | DNS 层 IP 改写 | **本地 USERPROFILE+RULEBINDING**（复用 [1-00011](task/UDG/20.15.2/1-00011.md) 全套）| FLOWFILTER | 骨架[1-00009]+族compound[1-00015]+全套[1-00011] |
| 流过滤器组变体 | URL | EQUAL, 3 | DNS 层 IP 改写 | 任意 | **FLOWFILTERGRP**（+[0-00294](task/UDG/20.15.2/0-00294.md)）| 骨架[1-00009]+族compound[1-00015]+atom RULE+[0-00294] |

> **★ 与 HTTP 智能重定向（[2-00017](task/UDG/20.15.2/2-00017.md)）的关键差异**（三处体现：配置概览 + 本决策点 + 约束）：
> - **改写层**：DNS 纠错 = **DNS 层 IP 改写**（DNSOVERWRITING→SERVERIP1）；HTTP 智能重定向 = **HTTP 层 URL 重定向**（SMARTHTTPREDIR→SERVERURL）
> - **错误码操作**：DNS 纠错默认 **EQUAL=3**（NXDOMAIN）；HTTP 智能重定向默认 **GT 400**（HTTP 4xx/5xx）
> - **携带信息**：DNS 纠错**无**（不配 REDIRAPPENDINFO）；HTTP 智能重定向可携带 REQURL/IMSI 等
> - **触发匹配**：DNS 纠错用 **L34 UDP 端口 53** 匹配 DNS 流量；HTTP 智能重定向用 **L34 ANY** 匹配 HTTP 流量
> - **规则下发**：两者 activation 均采用 PCRF/SMF 下发预定义规则方式（无 USERPROFILE/RULEBINDING）

**联动**：
- 触发匹配 → [1-00015](task/UDG/20.15.2/1-00015.md) 场景差异 §3（扩展过滤器逻辑 EXTFLTTYPE）
- 错误码操作 → [1-00015](task/UDG/20.15.2/1-00015.md) 场景差异 §2（ERRORCODEOP EQUAL/GT/GE/LE/LT）
- 改写层（动作命令）→ [1-00015](task/UDG/20.15.2/1-00015.md) 场景差异 §1（DNSOVERWRITING vs SMARTHTTPREDIR）
- 规则下发方式 → [1-00011](task/UDG/20.15.2/1-00011.md)（PCRF 预定义 vs 本地 USERPROFILE+RULEBINDING）
- FILTERINGMODE → [1-00011](task/UDG/20.15.2/1-00011.md) 场景差异 §FILTERINGMODE 变体（FLOWFILTER vs FLOWFILTERGRP）

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5DNSO01` — 未开则 DNS 纠错不生效。
- **前置依赖 PCC 基本功能**（critical）：GWFD-020351（activation §必备事项明示"完成激活 PCC 基本功能"）—— DNS 纠错建立在 PCC 基本功能之上，未完成则 DNS 纠错不生效。
- **POLICYTYPE=SMARTREDIRECT 触发条件**（critical）：RULE.POLICYTYPE=SMARTREDIRECT + POLICYNAME=DNSOVERWRTNAME（直接引用 DNS 重写动作对象，PCC 策略组名称=NULL）— 指错则 DNS 纠错不触发。
- **EXTFLTNAME 引用须已配置**（critical）：ADD DNSOVERWRITING 的 EXTFLTNAME1/2 须先经 ADD EXTENDEDFILTER 配置 — 后果：引用不存在拒收。
- **BINDERRCODENAME 引用须已配置**（critical）：ADD DNSOVERWRITING 的 BINDERRCODENAME 须先经 ADD ERRORCODE 配置 — 后果：引用不存在拒收。
- **动作对象须先于 RULE 配置**（critical）：DNSOVERWRTNAME 须先于 `RULE(POLICYTYPE=SMARTREDIRECT)` 配置 — 后果：RULE 引用失败。
- **L34 用 UDP 端口 53 匹配 DNS 流量**（warning）：ADD FILTER 用 L34PROTOCOL=UDP + SVRSTARTPORT/SVRENDPORT=53（53=DNS Server 端口），区别 HTTP 智能重定向的 L34PROTOCOL=ANY — 用错协议/端口则 DNS 流量匹配失效。
- **扩展过滤器可重复执行配多个**（info）：ADD EXTENDEDFILTER 重复执行增加多个不同过滤条件（activation §操作步骤 步骤4 说明）；activation 标准场景演示 1 个 URL 扩展过滤器。
- **规格**（info）：DNSOVERWRITING 整机 ≤2000（明显大于 SMARTHTTPREDIR ≤50 / REDIRAPPENDINFO ≤1000）；动作命令最多 5 组扩展过滤器，AND 组个数≤3。
