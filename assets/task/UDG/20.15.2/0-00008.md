# 绑定三四层过滤条件到流过滤器（ADD FLTBINDFLOWF）

> 命令静态知识见 [ADD FLTBINDFLOWF](command/UDG/20.15.2/ADD-FLTBINDFLOWF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

把上游 ADD FILTER（IPv4）/ADD FILTERIPV6（IPv6）产出的三四层过滤器，绑到 [ADD FLOWFILTER](0-00007.md) 产出的流过滤器上。两参数都来自"已配置数据"，不本端重新规划——只做引用对接。

**典型范式**（证据③，一个 FLOWFILTER 可同时绑 IPv4 + IPv6）：
```
ADD FLTBINDFLOWF: FLOWFILTERNAME="ff_L7", FILTERNAME="L34_dns";       # IPv4 dns
ADD FLTBINDFLOWF: FLOWFILTERNAME="ff_IMS", FILTERNAME="ipv6_ims";     # IPv6 IMS
ADD FLTBINDFLOWF: FLOWFILTERNAME="ff_L34any", FILTERNAME="L34_any";   # IPv4 兜底
ADD FLTBINDFLOWF: FLOWFILTERNAME="ff_L34any_IPv6", FILTERNAME="L34_any_ipv6";  # IPv6 兜底
```

**对接约定**：FLOWFILTERNAME 引 0-00007；FILTERNAME 引 0-00004(IPv4) 或 0-00005(IPv6)，按 DP 0-00058 选。

## 决策点

**DP 0-00058**（single_choice, required, always）FILTERNAME 引用何种协议族的三四层过滤器？

| 选项 | 联动 FILTERNAME 来源 | 典型过滤器名 |
|---|---|---|
| opt-filter-ipv4（默认） | 引 ADD FILTER(0-00004) IPv4 | L34_dns / L34_any / filter_red |
| opt-filter-ipv6 | 引 ADD FILTERIPV6(0-00005) IPv6 | ipv6_ims / L34_any_ipv6 |

> 一个 FlowFilter 可同时绑 IPv4 和 IPv6 过滤器（多次执行本命令），选项不是互斥，是"这条绑定引哪个源"。

## 约束

- **rule-0-00124**（critical）执行后需 **SET REFRESHSRV 刷新**后对新数据流生效（非立即，非固定延时）；最大记录数 **300000**，超 90% 上报 ALM-135602215（阈值 `MOD CFGTHRESHOLD` 修）；单 FlowFilter 可绑 Filter 数量有规格上限。
