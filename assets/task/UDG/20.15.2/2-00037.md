# 用户面地址自动检测（GWFD-010108）

> **运维型特性骨架**（SOP §3.7）——本特性是**诊断旁路工具**，零 ADD/MOD/SET 配置命令，**配置生成无内容**。
>
> 特性静态知识见 [GWFD-010108](feature/UDG/20.15.2/GWFD-010108.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-010108-用户面地址自动检测.md)。探测触发与结果查询命令见原始特性信息：[激活用户面地址自动检测](evidence/UDG/20.15.2/GWFD-010108-激活用户面地址自动检测_04636908.md) + [调测用户面地址自动检测](evidence/UDG/20.15.2/GWFD-010108-调测用户面地址自动检测_04636931.md)。

## 配置概览

本特性是**运维触发型诊断工具**——配置生成器遇到本特性时**直接跳过**（无配置产出）：

- **零配置命令**：无 ADD/MOD/SET 持久化对象
- **依赖前置**：GWFD-010105 用户面地址分配（[2-00034](task/UDG/20.15.2/2-00034.md)）已配置 LOCAL 地址池/段（探测目标范围）
- **运维触发**：运维人员按需执行 STR/STP/DSP 三类命令（非配置生成产出）

**对象链**（无新对象，全部复用 GWFD-010105）：
```
[GWFD-010105 已配置的 LOCAL 地址池 / SECTION]  ← 探测目标范围
       ↓ 运维触发（非配置生成）
[STR PDNROUTETST 启动 / STP PDNROUTETST 终止 / DSP PDNTSTRESULT 查询结果]
```

## 配置流程

> **本特性无配置流程**（运维型骨架，零配置命令）。配置生成器跳过本特性。
>
> 运维触发流程（探测启停与结果查询三类运维命令的 PING/DNS/Tracert 探测方式选择）见原始特性信息：[激活用户面地址自动检测](evidence/UDG/20.15.2/GWFD-010108-激活用户面地址自动检测_04636908.md)。

## 决策点

> 本特性无配置决策，仅有运维决策（探测方式选择）。下表为运维参考，**非配置生成路径**。

| 探测方式 | 全量探测 | 关键参数 | 适用场景 |
|---|---|---|---|
| PING 全量 | ENABLE | IPPOOL, DSTIPV4ADDR, SECTIONNUM=0 | 全网段路由连通性扫描 |
| PING 单点 | DISABLE | + SRCIPV4ADDR | 指定源 IP 探测 |
| DNS | ENABLE/DISABLE | + DOMAINVALUE | 防火墙禁 ICMP 场景 |
| Tracert | **不支持全量** | IPPOOL, DSTIPV4ADDR | 故障定位（450s 耗时） |

> 探测命令实现：[0-00048](task/UDG/20.15.2/0-00048.md) STR PDNROUTETST（START/STOP/DISP 三态共用一 atom）。

## 约束

- **无需 License**（info）：本特性无需 License。
- **依赖 GWFD-010105**（critical）：仅在 UDG 采用**本地地址池**（POOLTYPE=LOCAL）分配 IP 场景下使用 — 后果：外部地址池/RADIUS/静态地址场景探测失败。
- **互斥 4 特性**（critical）：GWFD-010107 静态地址用户路由冗余 / GWFD-110910 Routing Behind MS（LKV3G5RBMS01）/ GWFD-020406 IPv6 Prefix Delegation（LKV3G5P6PD01）/ GWFD-020412 L2TP VPN（LKV3G5L2TP01）—— 双向互斥。
- **不支持 NAT**（critical）：本特性不支持 NAT 地址转换。
- **探测任务串行**（critical）：同一时间仅一个探测任务 + 单 APN 串行。
- **源 IP 占用风险**（critical）：探测源 IP 建议选空闲地址；用在线用户 IP 会导致在线用户被去激活。
- **Tracert 耗时**（warning）：约 450 秒返回结果；全量探测最多 5 分钟；失败记录上限 128 个。
