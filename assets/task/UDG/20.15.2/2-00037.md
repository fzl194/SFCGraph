# 用户面地址自动检测（GWFD-010108）

> 特性静态知识见 [GWFD-010108](feature/UDG/20.15.2/GWFD-010108.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-010108-用户面地址自动检测.md)。
>
> **本页定位**：配置生成实例化视角——本特性是**诊断旁路工具**（无持久化配置对象）。仅 "前提条件"（依赖 GWFD-010105 本地地址池已配置）+ "运维触发"（STR/STP/DSP 三条命令）。特性本身零 ADD/MOD/SET 命令。

## 配置概览

本特性是**运维触发型诊断工具**，无 ADD 类配置命令。配置 = "0 配置" + "3 条运维命令（STR/STP/DSP）" + "复用 GWFD-010105 地址池"：

**对象链**（无新对象，复用 GWFD-010105）：
```
[GWFD-010105 已配置的 LOCAL 地址池 / SECTION]
       ↓
[STR PDNROUTETST 触发探测（瞬时态，仅本地地址池范围）]
       ↓（UDG 本地构造探测报文）
Gi/SGi/N6 → PDN/DN 侧服务器
       ↓
[探测结果存入失败结果表（最多 128 条）+ DSP PDNTSTRESULT 查询]
```

**5 步运维流程**：

| 步骤 | 干啥 | 主要命令 |
|---|---|---|
| 0. 前置 | GWFD-010105 已配置 LOCAL 地址池/段（POOLTYPE=LOCAL） | 详见 [2-00034](task/UDG/20.15.2/2-00034.md) 配置流程 |
| 1. 查询可用地址池与地址段 | 确认 IPPOOL/SECTIONNUM | `LST POOL` / `LST SECTION` / `LST POOLGROUP` |
| 2. 查询当前路由表（探测前后对比） | IPv4/IPv6 路由表对比 | `DSP ROUTE` / `DSP ROUTE6` |
| 3. 启动 PDN/DN 侧路由探测 | PING/DNS/Tracert 三选一 | `STR PDNROUTETST` |
| 4. 查询探测结果 | 全部/成功/失败 | `DSP PDNTSTRESULT` |
| 5. 强制终止（如需） | 误操作/超时处理 | `STP PDNROUTETST` |

**关键约束**：
- **本特性无需 License**
- **必须依赖 GWFD-010105**（仅本地地址池场景可用；外部地址池/RADIUS/静态地址场景不可用）
- **互斥 4 个**：GWFD-010107 静态地址用户路由冗余 / GWFD-110910 Routing Behind MS / GWFD-020406 IPv6 Prefix Delegation / GWFD-020412 L2TP VPN
- 不支持 NAT 地址转换
- 探测任务串行（同一时间仅一个任务）
- 单 APN 串行（同一时间仅一个 APN 探测）
- 源 IP 建议选空闲地址（避免误触发用户去激活）
- Tracert 不支持全量探测
- 全量探测 5 分钟超时

## 配置流程

> 每步显示"干啥 + 含哪些命令"。本特性无 ADD/MOD/SET 类配置命令，全为运维命令。

1. **前置条件**（依赖 GWFD-010105 已配置完成）：
   - 必须先完成 GWFD-010105 步骤 1-3（VPN + APN + LOCAL 地址池 + 段 + 池组 + 绑定）
   - 复用 → [1-00022](task/UDG/20.15.2/1-00022.md) 地址分配体系骨架（POOLTYPE=LOCAL）

2. **查询可用地址池与地址段**（可选，构造 IPPOOL/SECTIONNUM 参数）：
   - `LST POOL` → [0-00328](task/UDG/20.15.2/0-00328.md)（查 POOLNAME）
   - `LST SECTION` / `LST POOLGROUP` → 确认 SECTIONNUM
   - → 复用既有查询 atom，无新建

3. **查询当前路由表**（可选，建议探测前后对比）：
   - `DSP ROUTE`（IPv4 路由表） / `DSP ROUTE6`（IPv6 路由表）
   - → 已知调试命令，无需新建 atom

4. **启动 PDN/DN 侧路由探测**（三种方式三选一）：
   - `STR PDNROUTETST` → [0-00048](task/UDG/20.15.2/0-00048.md)
     - PING 全量：`TESTMETHOD=PING, WHOLEDETECT=ENABLE, SECTIONNUM=0`
     - PING 单点：`TESTMETHOD=PING, SRCIPV4ADDR=10.10.1.1`（指定源 IP）
     - DNS 全量：`TESTMETHOD=DNS, DOMAINVALUE=www.example.com, WHOLEDETECT=ENABLE`
     - Tracert：`TESTMETHOD=TRACERT`（**不支持全量探测**）

5. **查询探测结果**：
   - `DSP PDNTSTRESULT` → 全部（不输 DISPLAYMODE）
   - `DSP PDNTSTRESULT: DISPLAYMODE=SUCCESS` → 探测成功记录
   - `DSP PDNTSTRESULT: DISPLAYMODE=FAIL` → 探测失败记录（最多 128 条）

6. **强制终止**（如异常处理）：
   - `STP PDNROUTETST` → 强制终止当前探测任务

## 决策点

**DP（特性级）：探测方式 × 全量探测 × 走法**

| 探测方式 | 全量探测 | 关键参数 | 走法（atom 编排） |
|---|---|---|---|
| PING 默认 | ENABLE（推荐全网段扫描）| IPPOOL, DSTIPV4ADDR | [0-00048](task/UDG/20.15.2/0-00048.md)（STR PDNROUTETST, TESTMETHOD=PING, WHOLEDETECT=ENABLE, SECTIONNUM=0） |
| PING 单点 | DISABLE（默认）| + SRCIPV4ADDR | [0-00048](task/UDG/20.15.2/0-00048.md)（STR PDNROUTETST, TESTMETHOD=PING, SRCIPV4ADDR=10.10.1.1） |
| DNS 防火墙禁 ICMP | ENABLE | + DOMAINVALUE | [0-00048](task/UDG/20.15.2/0-00048.md)（STR PDNROUTETST, TESTMETHOD=DNS, DOMAINVALUE=www.example.com, WHOLEDETECT=ENABLE） |
| DNS 单点 | DISABLE | + DOMAINVALUE | [0-00048](task/UDG/20.15.2/0-00048.md)（TESTMETHOD=DNS, DOMAINVALUE=...） |
| Tracert 故障定位 | **不支持全量** | IPPOOL, DSTIPV4ADDR | [0-00048](task/UDG/20.15.2/0-00048.md)（STR PDNROUTETST, TESTMETHOD=TRACERT） |
| 查询结果 | — | DISPLAYMODE | [0-00048](task/UDG/20.15.2/0-00048.md)（DSP PDNTSTRESULT 查询，DISPLAYMODE=SUCCESS/FAIL/空） |
| 强制终止 | — | — | [0-00048](task/UDG/20.15.2/0-00048.md)（STP PDNROUTETST 立即终止） |

> **关键规则**：
> - 本特性无 ADD 配置命令；走法完全由 STR/STP/DSP 三条命令参数驱动
> - 探测源 IP 不指定时：PING/DNS 从**每个路由网段随机选一个**；Tracert 从**随机一个地址网段选**（不遍历）
> - 防火墙禁 ICMP → 用 DNS 探测；故障定位 → Tracert
> - DNS 探测方式目的 IP 必须为 DNS Server IP

## 约束

- **无需 License**（info）：本特性无需获得 License 许可即可获得该特性的服务
- **依赖 GWFD-010105**（critical）：本特性仅在 UDG 采用本地地址池分配 IP 地址的场景下使用 — 后果：外部地址池场景探测失败
- **本地地址池必配段**（critical，rule-0-00520）：需通过 GWFD-010105 的 ADD POOL / ADD SECTION 已配置完成 — 后果：探测时无可用地址段
- **PDN/DN 可达**（critical，rule-0-00521）：探测目的 IP 地址需为 PDN/DN 侧的目的服务器 IP（DNS 探测必须为 DNS Server IP） — 后果：探测失败误判
- **互斥 GWFD-010107 静态地址用户路由冗余**（critical，rule-0-00522）：不支持与 GWFD-010107 同时应用 — 后果：双向互斥
- **互斥 GWFD-110910 Routing Behind MS**（critical，rule-0-00523）：不支持与 Routing Behind MS（License LKV3G5RBMS01 / 控制项 82200CKP）同时应用 — 后果：双向互斥
- **互斥 GWFD-020406 IPv6 Prefix Delegation**（critical，rule-0-00524）：不支持与 IPv6 Prefix Delegation（License LKV3G5P6PD01 / 控制项 82200CKF）同时应用 — 后果：双向互斥
- **互斥 GWFD-020412 L2TP VPN**（critical，rule-0-00525）：不支持与 L2TP VPN（License LKV3G5L2TP01 / 控制项 82200BVC）同时应用 — 后果：双向互斥
- **不支持外部地址池**（critical）：不支持在外部地址池（SMF、RADIUS、静态地址）分配 UE IP 场景下使用 — 后果：探测无效
- **不支持 NAT**（critical，rule-0-00526）：本特性不支持 NAT 地址转换 — 后果：探测失败
- **内置 NAT 探测失败风险**（warning）：UDG 部署内置 NAT 后，PGW-U/UPF 无法感知外部服务器 IP 公网私网类型，可能出现探测失败（联系华为技术支持） — 后果：探测失败误判
- **探测任务串行**（critical，rule-0-00527）：同一时间只能执行一个探测命令 — 后果：同时执行多个时后执行的会失败
- **单 APN 串行**（critical，rule-0-00528）：同一时间只支持一个 APN 下的探测 — 后果：未完成当前探测时不允许再下发
- **源 IP 占用风险**（critical，rule-0-00529）：探测源 IP 建议选空闲地址；执行在线用户 IP 会导致在线用户被去激活 — 后果：误触发用户去激活
- **全量探测占用风险**（warning）：全量探测时若网段全占用，可能导致极少数（几个或几十个）在线用户被去激活 — 后果：谨慎使用
- **Tracert 耗时长**（warning，rule-0-00530）：TRACERT 在未回全部探测响应的场景下，约需 **450 秒**左右才返回结果 — 后果：超时误判
- **全量探测时长**（warning，rule-0-00531）：全量探测时探测任务最多探测 5 分钟 — 后果：未覆盖的子段需指定 SECTION 再探测
- **失败记录上限**（info，rule-0-00532）：支持记录的最大探测失败结果个数为 **128 个** — 后果：超规格溢出
- **单网段探测时长**（info，rule-0-00533）：每个路由网段最长探测时间 **2 秒** — 后果：超时判失败
- **探测源 IP 选取**（warning）：PING/DNS 从每个路由网段随机；Tracert 随机一个地址网段（不遍历） — 后果：Tracert 探测效率低
