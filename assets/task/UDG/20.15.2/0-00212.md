# 配置IPsec安全提议（ADD IPSECPROPOSALIPSEC）

> 命令静态知识见 [ADD IPSECPROPOSALIPSEC](command/UDG/20.15.2/ADD-IPSECPROPOSALIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

定义 IPsec 数据面安全协议（ESP/AH/Ah_esp）、配套认证/加密算法、封装模式。算法取值由 [DP 0-00033](0-00033.md)（IPsec/IKE 算法集与封装模式分叉）编排，**两端协商一致**。提议由下游 IPsec 安全策略引用。

**数据规划取值样例**（证据③，IPFD-015004 各激活 md 数据规划表，ESP 主流 + Ah_esp 场景）：

| 参数 | 取值样例（ESP 主流） | 取值样例（Ah_esp 场景） | 获取方法 | 说明 |
|---|---|---|---|---|
| PROPOSALNAME（Proposal 名称） | `tran1`、`1` | `Proposal1` | 本端规划 | 提议标识，格式 proposal+数字或自定义 |
| IPSECPROTOCOL（IPsec 安全协议） | `Esp` | `Ah_esp` | 对端协商（DP 0-00033） | Esp 配套 ESP 算法；Ah_esp 额外配 AHAUTHALGO |
| ESPAUTHALGO（ESP 认证算法） | `Sha2_256` | `Sha2_512` | 对端协商（DP 0-00033） | IPSECPROTOCOL=Esp/Ah_esp 时生效 |
| ESPENCRYPTALGO（ESP 加密算法） | `Aes_256` | `Aes_256` | 对端协商（DP 0-00033） | 同上 |
| ENCAPMODE（封装模式） | `Tunnel`（隧道模式） | `Tunnel` | 对端协商（DP 0-00033） | 主场景全 Tunnel |
| AHAUTHALGO（AH 认证算法） | （不绑） | `Sha2_512` | 对端协商 | 仅 IPSECPROTOCOL=Ah/Ah_esp 时必填 |

> 国密场景算法集分叉见 [DP 0-00033](0-00033.md) option opt-ipsec-smcrypto。

**典型范式**（证据③任务示例脚本，ESP 场景）：
```
ADD IPSECPROPOSALIPSEC: PROPOSALNAME="tran1", IPSECPROTOCOL=Esp,
    ESPAUTHALGO=Sha2_256, ESPENCRYPTALGO=Aes_256, ENCAPMODE=Tunnel;
```

**典型范式**（Ah_esp 场景，10009/03567841）：
```
ADD IPSECPROPOSALIPSEC: PROPOSALNAME="Proposal1", IPSECPROTOCOL=Ah_esp,
    ESPAUTHALGO=Sha2_512, ESPENCRYPTALGO=Aes_256, AHAUTHALGO=Sha2_512, ENCAPMODE=Tunnel;
```

## 决策点

- [DP 0-00033](0-00033.md) 编排 IPSECPROTOCOL/ESPAUTHALGO/ESPENCRYPTALGO/ENCAPMODE（及 AHAUTHALGO）的分叉：ESP 主流、Ah_esp 双认证、国密算法集（opt-ipsec-smcrypto）。

## 约束

- **rule-0-00336**（critical）命令立即生效；部分配置含不安全算法（强度需符合安全规范）；PROPOSALNAME 格式 proposal+数字；IPsec 协议/算法（IPSECPROTOCOL/ESPAUTHALGO/ESPENCRYPTALGO/AHAUTHALGO/ENCAPMODE）与对端协商**两端须一致**；IPSECPROTOCOL=Esp 配套 ESPAUTHALGO+ESPENCRYPTALGO，=Ah 配套 AHAUTHALGO（二选一由 DP 0-00033 编排）。
