# HTTP头增强（GWFD-110261）

> 特性静态知识见 [GWFD-110261](feature/UDG/20.15.2/GWFD-110261.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性是**头增强协议族（HTTP/HTTPS/RTSP/NSH）的族范本**，后续 262/263/401 复用本族 compound。

## 配置概览

HTTP 头增强将用户信息（MSISDN/IMSI/APN/MSIP 等）以 `<字段前缀: 字段值>` 格式插入 HTTP 请求报文头扩展字段，传给 Web Server。**核心机制：RULE 经 `POLICYTYPE=HEADEN` 直接引用 HEADEN 对象（ADD HEADEN 定义），独立于 PCC/SMARTREDIRECT 的第 3 种 POLICYTYPE，不经 PCCPOLICYGRP**。

头增强与计费可绑定同一 UserProfile（activation 演示双 RULE：HEADEN 规则 + PCC 计费规则共用 FLOWFILTER）。多场景差异由 DP 驱动（触发方式/插入字段/加密/编码/防欺诈/灰名单）。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开头增强 License 开关，单命令）：`SET LICENSESWITCH`（LKV3G5HTHE01）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **配置流过滤器**（流匹配容器，单命令）：`ADD FLOWFILTER`（FLOWFILTERNAME）→ [0-00007](task/UDG/20.15.2/0-00007.md)
3. **配置过滤条件 + 协议识别**（按触发方式选，三四层/七层/协议三选一）：
   - 过滤条件：`ADD FILTER`+`ADD FLTBINDFLOWF` 或 `ADD L7FILTER`+`ADD PROTBINDFLOWF`（PROTOCOLNAME=http）+ `SET REFRESHSRV` → [1-00009](task/UDG/20.15.2/1-00009.md)（过滤链 compound，仅含 FILTER/FILTERIPV6/L7FILTER/FLOWFILTER/FLTBINDFLOWF/PROTBINDFLOWF）
   - 协议识别（直引 atom，**非 1-00009 compound 成员**）：`ADD WELLKNOWNPORT` → [0-00039](task/UDG/20.15.2/0-00039.md) + `LOD SIGNATUREDB` → [0-00027](task/UDG/20.15.2/0-00027.md) + `LOD PARSERDB` → [0-00152](task/UDG/20.15.2/0-00152.md)
   - 基于特定 URL（主流）：`ADD L7FILTER`（URL）+ `ADD PROTBINDFLOWF`（PROTOCOLNAME=http, L7FILTERNAME）
   - 基于特定 IP：`ADD FILTER`（SVRIP/端口）+ `ADD FLTBINDFLOWF`（**不配协议**，同时影响 HTTP/RTSP/HTTPS）
   - 基于 HTTP 协议：`ADD PROTBINDFLOWF`（PROTOCOLNAME=http，**不配 L7FILTER**）
4. **配置头增强对象**（HTTP 头增强核心，ADD HEADEN 多字段插入+加密+防欺诈）：`ADD HEADEN`（HEADERENNAME+DATATYPE+PREFIXNAME+ENCRYALGORI+ANTIFRAUD+GRAYLIST，**多字段多次执行**）→ [1-00013](task/UDG/20.15.2/1-00013.md)
5. **配置计费属性和策略**（因头增强仅针对上行报文，需上行计费；可选）：`ADD URR`+`ADD URRGROUP`+`ADD PCCPOLICYGRP` → [1-00010](task/UDG/20.15.2/1-00010.md)
6. **配置头增强规则**（POLICYTYPE=HEADEN 触发，规则绑定收尾）：`ADD RULE`（POLICYTYPE=HEADEN + POLICYNAME=HEADERENNAME）+ `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
   - 计费规则（可选，与头增强规则共用同一 FLOWFILTER）：`ADD RULE`（POLICYTYPE=PCC + POLICYNAME=PCCPOLICYGRPNM）

## 决策点

| 基于特定 IP 触发 | 基于特定 IP | FILTER（SVRIP+端口，不配协议）+ FLTBINDFLOWF | 任意 | 任意 | 任意 | 任意 |
| 基于 HTTP 协议触发 | 基于 HTTP 协议 | PROTBINDFLOWF（http，无 L7FILTER）| 任意 | 任意 | 任意 | 任意 |
| 加密变体（安全推荐）| 任意 | 同上 | 任意 | **AES128_GCM/AES256_GCM**（+PSWDKEY+SET AES）| DISABLE | DISABLE |

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5HTHE01` — 未开则头增强不生效。
- **强依赖 SA-Basic**（critical）：GWFD-110101（License LKV3G5SABS01）—— HTTP 报文解析基础，未开则无法识别报文。
- **协议依赖 SA-Web Browsing**（critical）：GWFD-110103（License LKV3G5SAWB01）—— HTTP/WAP2.0 解析必需；HTTP 承载业务（如 MMS）还需 SA-Mobile（GWFD-110105）。
- **POLICYTYPE=HEADEN 触发条件**（critical）：RULE.POLICYTYPE=HEADEN + POLICYNAME=HEADERENNAME（直接引用，PCC 策略组名称=NULL）— 指错则头增强不触发。
- **防欺诈强耦合 GWFD-110401**（critical）：ANTIFRAUD=ENABLE 需配套 License LKV3G5HHAS01 — 未开则防欺诈不生效。
- **仅 HTTP1.x，不支持 HTTP2.0/HTTPS**（warning）：HTTPS 需 GWFD-110263（ADD TLSHEADEN）— 混用则对应协议报文不做头增强。
- **基于 IP 触发同时影响 HTTP/RTSP/HTTPS**（warning）：不区分协议，对用户的 HTTP/RTSP/HTTPS 报文都做头增强（设计如此）— 后果：RTSP/HTTPS 报文意外被头增强；改用基于 URL 触发可规避。
- **IPv6 分片风险**（warning）：头增强字段过长致报文超 PMTU 将导致 IPv6 业务无法访问；建议 TCP MSS ≤ PMTU - 60B - 头增强字段长度。
- **分片报文不做头增强**（warning）：MTU 过小致用户报文到达 UDG 前被分片，UDG 不处理分片报文。
- **外置 PCEF 计费误差**（warning）：头增强改变报文长度，外置 PCEF 计费有误差；建议头增强与计费部署在同一网元，或内置 PCEF 计费 + 外置 PCEF 部署头增强。
- **METHODTYPE 限制**（info）：未配 ADD L7FILTER 的 METHODTYPE 时仅支持默认（GET/POST/CONNECT）；软参 BIT316 控特殊 method。
- **规格**（info）：整机 ≤100 头增强动作；头增强仅针对上行报文（计费属性须含上行）。
- **计费与话单**（info）：先计费再头增强（与规则优先级无关），头增强插入信息不计入业务量计费统计。
