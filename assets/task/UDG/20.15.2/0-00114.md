# 修改接口管理状态（MOD INTERFACE）

> 命令静态知识见 [MOD INTERFACE](command/UDG/20.15.2/MOD-INTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。属"修改"语义，对已通过 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 创建的接口生效。

## 配置方法

MOD INTERFACE 用于修改已创建接口的相关配置，可作用于物理接口与逻辑接口。**IFNAME 必选**，其余维度（IFADMINSTATUS/IFMTU/IFDF/IFSTATIENABLE/IFTRAPENABLE/IFSTATITVL/IFDESCR）按需可选。核心配置空间有 7 个维度：**接口定位**（IFNAME）、**管理状态**（IFADMINSTATUS）、**MTU**（IFMTU）、**分片**（IFDF）、**统计**（IFSTATIENABLE/IFSTATITVL）、**Trap**（IFTRAPENABLE）、**描述**（IFDESCR）。2 个特性命中（GWFD-020402 N6/Gi/SGi IPv6 组网 / IPFD-015004 IPSec）覆盖 Ethernet/LoopBack 接口的 up/MTU/统计多种典型场景。

### 配置维度 1：接口定位（参数 IFNAME，必选）

| 取值 | 作用 |
|---|---|
| **已建接口名**（ref [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE） | 指定要修改的接口 |

> 接口名格式：`Ethernet<n>/<m>/<p>`、`LoopBack<n>`、`Tunnel<n>`、`Invalid0` 等；1~63 字符。

### 配置维度 2：管理状态（参数 IFADMINSTATUS，可选）

| 取值 | 作用 |
|---|---|
| **up** | 接口管理状态 up（启动收发）（020402/IPFD-015004 范式） |
| **down** | 接口管理状态 down（停止收发） |

> 仅修改管理状态：MOD INTERFACE:IFNAME="...",IFADMINSTATUS=up；是最常见用法。

### 配置维度 3：MTU（参数 IFMTU，可选）

| 取值 | 作用 |
|---|---|
| **整数（byte）** | 接口最大传输单元；10M Ethernet 接口线性转发测试时按 14880pps 校准 |

> 同一接口 MTU 须上下游一致；IPsec/IPv6 组网常配 1500。

### 配置维度 4：分片（参数 IFDF，可选）

| 取值 | 作用 |
|---|---|
| **TRUE** | 启用 IP 报文强制分片 |
| **FALSE** | 不启用强制分片（IPFD-015004 范式） |

> IPsec/IPv6 链路建议 FALSE，由端到端 PMTU 协商处理。

### 配置维度 5：接口统计（参数 IFSTATIENABLE + IFSTATITVL，可选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **IFSTATIENABLE** | TRUE / FALSE | 是否使能接口流量统计（IPFD-015004 范式 TRUE） |
| **IFSTATITVL** | 5~600 秒 | 流量统计时间间隔（IPFD-015004 范式 20） |

> 查看接口收发报文统计计数前必须先使能 IFSTATIENABLE。

### 配置维度 6：Trap 告警（参数 IFTRAPENABLE，可选）

| 取值 | 作用 |
|---|---|
| **TRUE / FALSE** | 是否使能接口 Down 时的 Trap 告警上报 |

### 配置维度 7：接口描述（参数 IFDESCR，可选）

| 取值 | 作用 |
|---|---|
| **1~241 字符** | 接口用途描述 |

**典型脚本（GWFD-020402 N6/Gi/SGi IPv6 组网，仅启 up）**：
```
MOD INTERFACE:IFNAME="Ethernet70/0/4",IFADMINSTATUS=up;
```

**典型脚本（IPFD-015004 IPsec，up + MTU + 统计）**：
```
MOD INTERFACE:IFNAME="Ethernet66/0/3",IFADMINSTATUS=up,IFMTU=1500,IFDF=FALSE,IFSTATIENABLE=TRUE,IFSTATITVL=20;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：激活接口"——前置 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE 建接口；后续按需 [0-00058](task/UDG/20.15.2/0-00058.md) ADD IFIPV4ADDRESS / [0-00061](task/UDG/20.15.2/0-00061.md) ADD IFIPV6ADDRESS 配 IP。

## 决策点：接口修改范围选择（DP 0-00114）

问：本接口修改主要改哪些属性？

| 选项 | IFADMINSTATUS | IFMTU | IFSTATIENABLE | 其他 | 适用场景 |
|---|---|---|---|---|---|
| **仅启 up**（020402 默认） | up | 不配 | 不配 | 不配 | 普通 N6/Gi/SGi 接口启用 |
| **up + MTU + 统计**（IPsec 范式） | up | 1500 | TRUE + IFSTATITVL=20 | IFDF=FALSE | IPsec/IPv6 接口启用、需统计 |
| **LoopBack 启用**（IPsec） | up | 1500 | TRUE | IFDF=FALSE | LoopBack 接口启用 |

> 2 个特性命中（020402/IPFD-015004）；IFNAME 必选；其余参数按需配。

## 约束

- **IFNAME 必须已建**（critical，rule-0-00114）：IFNAME 必须是已通过 `ADD INTERFACE` 创建的接口 — 后果：接口不存在命令拒收
- **修改语义**（info，rule-0-00114）：本命令语义为"修改"，不创建/删除接口；新建接口使用 [0-00056](task/UDG/20.15.2/0-00056.md) ADD INTERFACE — 后果：用于不存在的接口无效
- **IFSTATITVL 范围**（warning，rule-0-00114）：IFSTATITVL 取值范围 5~600 秒 — 后果：超范围命令拒收
- **IFMTU 与上下游一致**（warning，rule-0-00114）：接口 MTU 应与上下游接口一致 — 后果：MTU 不一致导致分片/丢包
- **执行后立即生效**（warning，rule-0-00114）：本命令执行后立即生效 — 后果：误改 IFADMINSTATUS=down 即刻中断接口流量
- **IFSTATIENABLE 前置 IFSTATITVL**（info，rule-0-00114）：IFSTATITVL 仅在 IFSTATIENABLE=TRUE 时生效 — 后果：IFSTATIENABLE=FALSE 时设 IFSTATITVL 无效
