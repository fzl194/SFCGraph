# 配置用户模板URR组绑定（SET URRGRPBINDING）

> 命令静态知识见 [SET URRGRPBINDING](command/UDG/20.15.2/SET-URRGRPBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"**配置方法字典**"，**不**逐特性罗列取值。

## 配置方法

SET URRGRPBINDING 是一条命令三种用途——给同一个 USERPROFILE 配置不同场景的 URR 组绑定。本命令的核心配置空间有 3 个维度：**用途分叉**（缺省费率 / 继承全局缺省 / TCP 重传计费，决定参数子集）、**参数子集**（按用途选 5 类参数组合）、**URR 组来源**（各 URR 组均引用 [ADD URRGROUP](0-00002.md) 已配的 URRGROUPNAME）。

### 配置维度 1：用途分叉（决定参数子集）

同一条命令承载 3 种业务用途：

| 用途 | 必传参数 | 典型场景 |
|---|---|---|
| **缺省费率绑定**（业务 + 信令 URR 组） | USERPROFILENAME + DFTURRGRPNAME + DFTSIGURRGNAME | 计费（2-00001 内容计费等）|
| **继承全局缺省 URR 组**（追加全局开关） | 同上 + GLBDFTURRGSW + GLBDFTSIGURRGSW | 基于时长/流量/事件计费（2-00009/10/11）|
| **TCP 重传计费** | USERPROFILENAME + TCPCHGPROFLAG + TCPURRGRPNAME | GWFD-020307 激活 TCP 重传识别后启用 |

### 配置维度 2：参数子集（按用途挑选）

| 参数 | 缺省费率绑定 | 继承全局缺省 | TCP 重传计费 | 取值约束 / 来源 |
|---|---|---|---|---|
| `USERPROFILENAME` | ✓ | ✓ | ✓ | reference：引 [0-00011](0-00011.md) |
| `DFTURRGRPNAME` | ✓ | ✓ | — | reference：引 [0-00002](0-00002.md) 产出的 URRGROUPNAME |
| `DFTSIGURRGNAME` | ✓ | ✓ | — | reference：引 [0-00002](0-00002.md) — 信令面 URR 组（异常事件计费组）|
| `GLBDFTURRGSW` | — | ENABLE | — | ENABLE / DISABLE |
| `GLBDFTSIGURRGSW` | — | ENABLE | — | ENABLE / DISABLE |
| `TCPCHGPROFLAG` | — | — | CHARGING | 固定 |
| `TCPURRGRPNAME` | — | — | ✓ | reference：引 [0-00002](0-00002.md) |

### 配置维度 3：URR 组来源

| URR 组参数 | 用途 | 典型命名 | 来源 |
|---|---|---|---|
| `DFTURRGRPNAME`（业务面） | 普通业务流量的缺省费率组 | `UrrGp_any` / `UrrGp_URL` / `UrrGp_IMS` | [ADD URRGROUP](0-00002.md) 已配 |
| `DFTSIGURRGNAME`（信令面） | 异常/信令流量的费率组 | `UrrGp_abnormal` | [ADD URRGROUP](0-00002.md) 已配 |
| `TCPURRGRPNAME`（TCP 重传） | TCP 重传计费专用费率组 | `cp_tcp` | [ADD URRGROUP](0-00002.md) 已配 |

**步骤位置**：在所属 feature 编排中通常位于"步骤 8：URR 组绑定"——ADD URRGROUP 之后（3 类 URR 组均已配），SET REFRESHSRV 之前。

## 决策点：URR 组绑定用途分叉（DP 0-00008）

问：SET URRGRPBINDING 选哪种绑定用途（决定参数子集与该次配置语义）？

| 选项 | 必传参数 | 适用场景 |
|---|---|---|
| `default_charging`（默认） | `USERPROFILENAME` + `DFTURRGRPNAME` + `DFTSIGURRGNAME` | 计费场景（业务面 + 信令面 URR 组同时绑）|
| `inherit_global_default` | 同上 + `GLBDFTURRGSW=ENABLE` + `GLBDFTSIGURRGSW=ENABLE` | 基于时长/流量/事件计费（继承全局缺省 URR 组）|
| `tcp_retransmission` | `USERPROFILENAME` + `TCPCHGPROFLAG=CHARGING` + `TCPURRGRPNAME` | GWFD-020307 激活 TCP 重传识别后启用 TCP 重传计费 |

> 同一 USERPROFILE 可多次使用本命令（如同时配缺省费率 + TCP 重传），各次互不冲突。

## 约束

- **业务/信令 URR 组需同时配**（warning，rule-0-00113）：建议同时配 `DFTURRGRPNAME` + `DFTSIGURRGNAME`，否则使用该 UP 的用户部分流量无法计费 — 后果：异常/信令流量漏计费
- **例外：仅在线计费且 UP 未绑 Rule**（info，隐含）：仅在线计费且 UP 未绑任何 Rule 时，未匹配 RULE 的数据包按全局 URR 组计费，否则不计费
- **TCPCHGPROFLAG=CHARGING 前提**（warning，隐含）：配 TCP 重传计费子集前必须保证 GWFD-020307 已激活（`SET LICENSESWITCH:LICITEM="LKV3G5TCPR01",SWITCH=ENABLE;`） — 后果：未激活 TCP 重传识别，配 TCPURRGRPNAME 不生效
- **生效行为**（critical，隐含）：**立即生效**（个别 binding 在特定场景需刷新，参见 [SET REFRESHSRV](0-00015.md)） — 后果：误配致部分流量无法计费
- **URRGROUPNAME 一致性**（critical，隐含）：上述 3 类 URR 组名必须与 [ADD URRGROUP](0-00002.md) 已配的 URRGROUPNAME 一致 — 后果：URR 组引用错位
