# 添加APN的IMS信令分类器（ADD APNIMSSIGFLTR）

> 命令静态知识见 [ADD APNIMSSIGFLTR](command/UDG/20.15.2/ADD-APNIMSSIGFLTR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令属高危命令**——配置不当将影响 IMS 语音信令转发、造成语音不通。

## 配置方法

ADD APNIMSSIGFLTR 配置 APN 下 IMS 信令专用上下文的 Filter，用于 VoLTE 语音业务识别 IMS 信令流。**APN/PRIORITY/IPVERSION 必选**；**SRC/DSTIPV4* 与 SRC/DSTIPV6* 互斥**（IPVERSION 决定走 IPv4 还是 IPv6 路径）；**SRC/DSTPORT* 与 PROTOCOL 条件必选**（由 `*SPECTYPE` 决定 ANY/SPECIFY）。**核心配置空间有 6 个维度**：**IP 版本**（IPVERSION）、**源地址**（SRCIPV4* 或 SRCIPV6*）、**目的地址**（DSTIPV4* 或 DSTIPV6*）、**端口**（SRCPORT* / DSTPORT*）、**协议**（PROTOCOL）、**优先级**（PRIORITY）。1 个特性命中（GWFD-020251 VoLTE 基础语音业务）使用 TCP+UDP 双分类器范式。

### 配置维度 1：IP 版本（参数 IPVERSION，必选）

| 取值 | 作用 | 配套参数路径 |
|---|---|---|
| **IPV4** | IPv4 类型过滤规则（020251 范式） | SRCIPV4ADDR/MASKLEN + DSTIPV4ADDR/MASKLEN |
| **IPV6** | IPv6 类型过滤规则 | SRCIPV6ADDR/PREFLEN + DSTIPV6ADDR/PREFLEN |

### 配置维度 2：源 IP 地址（参数 SRCIPV4*/SRCIPV6*，条件必选）

| 取值 | 作用 |
|---|---|
| **SRCIPV4SPECTYPE=ANY**（默认） | 任意源 IPv4 |
| **SRCIPV4SPECTYPE=SPECIFY + SRCIPV4ADDR + SRCIPV4MASKLEN** | 指定源 IPv4 地址 + 掩码长度（0~32，020251 范式 10.1.1.1/32） |
| **SRCIPV6SPECTYPE=ANY**（默认） | 任意源 IPv6 |
| **SRCIPV6SPECTYPE=SPECIFY + SRCIPV6ADDR + SRCIPV6PREFLEN** | 指定源 IPv6 地址 + 前缀长度（0~128） |

### 配置维度 3：目的 IP 地址（参数 DSTIPV4*/DSTIPV6*，条件可选）

| 取值 | 作用 |
|---|---|
| **DSTIPV4SPECTYPE=ANY**（默认） | 任意目的 IPv4 |
| **DSTIPV4SPECTYPE=SPECIFY + DSTIPV4ADDR + DSTIPV4MASKLEN** | 指定目的 IPv4 地址 + 掩码长度 |
| **DSTIPV6SPECTYPE=ANY**（默认） | 任意目的 IPv6 |
| **DSTIPV6SPECTYPE=SPECIFY + DSTIPV6ADDR + DSTIPV6PREFLEN** | 指定目的 IPv6 地址 + 前缀长度 |

### 配置维度 4：源端口（参数 SRCPORT*，可选）

| 取值 | 作用 |
|---|---|
| **SRCPORTSPECTYPE=ANY**（默认） | 任意源端口 |
| **SRCPORTSPECTYPE=SPECIFY + SRCPORTSTART + SRCPORTEND** | 指定源端口范围 1~65535（020251 范式 1~2） |

### 配置维度 5：目的端口（参数 DSTPORT*，可选）

| 取值 | 作用 |
|---|---|
| **DSTPORTSPECTYPE=ANY**（默认） | 任意目的端口 |
| **DSTPORTSPECTYPE=SPECIFY + DSTPORTSTART + DSTPORTEND** | 指定目的端口范围 1~65535 |

### 配置维度 6：协议（参数 PROTOCOL，条件必选）

| 取值 | 作用 |
|---|---|
| **PROSPECTYPE=ANY**（默认） | 任意协议 |
| **PROSPECTYPE=SPECIFY + PROTOCOL=6** | TCP（020251 范式第一条） |
| **PROSPECTYPE=SPECIFY + PROTOCOL=17** | UDP（020251 范式第二条） |

> **优先级（参数 PRIORITY，必选，1~64，同一 APN 内唯一）**：值越小越优先；TCP/UDP 双分类器分别用 PRIORITY=1/2。

**典型脚本（GWFD-020251 VoLTE 基础语音业务，TCP+UDP 双分类器）**：
```
ADD APNIMSSIGFLTR:APN="ims",PRIORITY=1,IPVERSION=IPV4,SRCIPV4SPECTYPE=SPECIFY,SRCIPV4ADDR="10.1.1.1",SRCIPV4MASKLEN=32,SRCPORTSPECTYPE=SPECIFY,SRCPORTSTART=1,SRCPORTEND=2,PROSPECTYPE=SPECIFY,PROTOCOL=6;
ADD APNIMSSIGFLTR:APN="ims",PRIORITY=2,IPVERSION=IPV4,SRCIPV4SPECTYPE=SPECIFY,SRCIPV4ADDR="10.1.1.1",SRCIPV4MASKLEN=32,SRCPORTSPECTYPE=SPECIFY,SRCPORTSTART=1,SRCPORTEND=2,PROSPECTYPE=SPECIFY,PROTOCOL=17;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：建 IMS 信令分类器"——前置 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN（建 APN 实例）+ SET LICENSESWITCH 开 License；后续 [0-00156](task/UDG/20.15.2/0-00156.md) SET APNIMSATTR 配 IMS 使能开关。

## 决策点：协议组合（DP 0-00155）

问：IMS 信令需识别哪些协议？

| 选项 | PROTOCOL 取值 | PRIORITY | 适用场景 |
|---|---|---|---|
| **TCP 信令** | 6 | 1（最小） | IMS TCP 信令（020251 范式第一条） |
| **UDP 信令** | 17 | 2 | IMS UDP 信令（020251 范式第二条） |

> 1 个特性命中（020251）使用 TCP+UDP 双分类器；IMS 信令通常双协议并存，须**多次执行**本命令；高危命令，须确认合法报文都命中其中一条。

## 约束

- **APN/PRIORITY/IPVERSION 联合必选**（critical，rule-0-00155）：APN、PRIORITY、IPVERSION 是必选参数 — 后果：缺一命令拒收
- **高危命令**（critical，rule-0-00155）：本命令属高危命令，须确认添加规则后合法 IMS 信令报文都能命中其中一条规则 — 后果：配置不当造成语音不通
- **仅对新激活用户生效**（warning，rule-0-00155）：本命令仅对新激活用户生效，已激活用户不受影响 — 后果：已激活用户 IMS 信令仍按旧规则转发
- **APN 仅限 IMS APN**（warning，rule-0-00155）：APN 参数只能配做 IMS 业务的 APN，非 IMS APN 配置无意义 — 后果：非 IMS APN 干扰识别
- **PRIORITY 同 APN 内唯一**（warning，rule-0-00155）：PRIORITY 取值 1~64，同一 APN 内唯一 — 后果：重复 PRIORITY 命令拒收
- **单 APN 最多 64000 条**（warning，rule-0-00155）：单命令单 APN 最大记录数 64000 — 后果：超限 ADD 失败
- **APN 前置依赖**（warning，rule-0-00155）：APN 必须先经 [0-00020](task/UDG/20.15.2/0-00020.md) ADD APN 配置 — 后果：APN 不存在命令拒收
