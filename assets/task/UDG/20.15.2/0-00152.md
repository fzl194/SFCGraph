# 加载协议解析库（LOD PARSERDB）

> 命令静态知识见 [LOD PARSERDB](command/UDG/20.15.2/LOD-PARSERDB.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

LOD PARSERDB 用于加载/更新协议解析数据库，指定每个协议需解析的字段；该过程**不中断业务**。系统启动时自动加载，本命令用于版本升级或 HTTPS/IM 业务识别引导场景。**幂等**：已加载最新版则跳过该步骤。核心配置空间有 1 个维度：**加载模式**（LOADMODE）。2 个特性命中（GWFD-020359 IM 类业务无线资源管控 + GWFD-110203 HTTPS 业务地址识别）均使用 `LATEST` 范式。

### 配置维度 1：加载模式（参数 LOADMODE，必选）

| 取值 | 作用 |
|---|---|
| **LATEST** | 指定加载最新版本；典型场景（020359 IM 类业务 / 110203 HTTPS 业务均使用） |

> 单值；库文件由网管/MAE"网元文件传输"或 OM Portal"文件传输"上传；加载期间保持协议解析功能不中断业务。

**典型脚本（GWFD-020359 / GWFD-110203 标准路径）**：
```
LOD PARSERDB:LOADMODE=LATEST;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：加载最新协议解析库"——先 `DSP PARSERDB（★R1.5 atom 待补）` 查询当前版本，与规划一致则**跳过**，不一致才执行；与 [0-00027](task/UDG/20.15.2/0-00027.md) LOD SIGNATUREDB 配套（特征库识别协议类型，解析库解析协议字段），HTTP/IM 七层识别依赖两者共同生效。

## 决策点

（本命令无独立决策分支，LOADMODE 单值 LATEST，无分支选项。）

> 2 个特性命中（020359/110203）均使用 LOADMODE=LATEST；执行前建议先 `DSP PARSERDB（★R1.5 atom 待补）` 核对当前版本。

## 约束

- **LOADMODE 必选**（critical，rule-0-00152）：LOADMODE 是必选参数 — 后果：缺省命令拒收
- **执行后立即生效**（warning，rule-0-00152）：本命令执行后立即生效 — 后果：误加载错误版本会导致协议解析失败
- **库文件传输方式**（warning，rule-0-00152）：库文件经网管/MAE"网元文件传输"或 OM Portal"文件传输"上传；更新需一段时间 — 后果：期间须保持协议解析功能不中断业务
- **非例行操作需经确认**（warning，rule-0-00152）：系统启动自动加载，非例行操作需经华为技术支持确认后执行 — 后果：避免解析库文件错误导致系统异常
- **幂等性**（info，rule-0-00152）：已加载最新版则跳过该步骤 — 后果：避免重复加载浪费带宽
