# 加载协议解析库（LOD PARSERDB）

> 命令静态知识见 [LOD PARSERDB](command/UDG/20.15.2/LOD-PARSERDB.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

LOD PARSERDB 加载协议解析数据库（指定每个协议需解析的字段）。系统启动时**自动加载**，一般不需手工更新；本命令用于版本升级或 HTTPS/IM 业务识别引导场景。**幂等**：已加载则跳过该步骤。

**参数规划**

| 参数 | 含义 | 取值 | 说明 |
|---|---|---|---|
| LOADMODE | 解析库加载模式 | `LATEST`（fixed） | 加载最新版本；单值，全场景一致 |

**典型脚本**

```
LOD PARSERDB:LOADMODE=LATEST;
```

**前置 / 配套（HTTPS 业务地址识别特性 GWFD-110203 标准路径）**
1. 先 `DSP PARSERDB` 查询当前解析库版本；与规划值一致则**跳过本步骤**，不一致才执行 LOD PARSERDB。
2. 与 [0-00027](task/UDG/20.15.2/0-00027.md)（LOD SIGNATUREDB 加载协议特征库）**配套**：特征库识别协议类型，解析库解析协议字段，HTTP/IM 七层识别依赖两者共同生效。
3. 加载期间需保持协议解析功能，不中断业务。

**调测确认**（非本 atom 命令域，但配置工程师应知）：`LST USRRELATEIDEN` 查用户关联识别是否开启。

## 约束

- **rule-0-00308**（warning）**立即生效**。
- **rule-0-00308**（warning）系统启动自动加载，**非例行操作需经华为技术支持确认**后执行，避免解析库文件错误导致系统异常。
- **rule-0-00308**（warning）库文件经网管/MAE "网元文件传输" 或 OM Portal "文件传输" 上传；更新需一段时间，期间保持协议解析功能。
