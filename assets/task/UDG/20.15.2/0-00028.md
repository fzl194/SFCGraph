# 配置PCC动作属性（ADD PCCACTIONPROP）

> 命令静态知识见 [ADD PCCACTIONPROP](command/UDG/20.15.2/ADD-PCCACTIONPROP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD PCCACTIONPROP 是 PCC 策略组的"动作定义层"：把"哪种业务发起 + 上下行门控/重定向动作"用一条记录描述，产出对象 PCCACTPROPNAME 供 ADD SRVPBINDPCCPG(0-00030) 引用。命令核心配置空间有 2 个维度：**动作类型**（门控/重定向二选一为主，门控高频）+ **上下行门控组合**（UPINIT/UP×DN INIT×DN × GATE，4 个独立开关）。4 个门控均 optional，缺省=PASS；按目标语义补齐需改 DISCARD 的方向即可，无须显式写 PASS。

### 配置维度 1：动作类型（门控 / 重定向）

| 类型 | 联动参数 | 备注 |
|---|---|---|
| **门控（Gating）** | `UPINITUPGATE` / `UPINITDNGATE` / `DNINITUPGATE` / `DNINITDNGATE`（PASS/DISCARD） | 高频，3 特性全部命中 |
| **URL 重定向（Redirect）** | `UPINITREDIRNM` / `DNINITREDIRNM`（引用重定向名称） | 命令参数齐全，3 特性场景均未用 |

### 配置维度 2：上下行门控组合（参数 *GATE，4 个独立 PASS/DISCARD）

| 门控 | 含义 | 缺省值 |
|---|---|---|
| **UPINITUPGATE** | 上行发起的上行门控（手机发起 → 手机上行） | PASS |
| **UPINITDNGATE** | 上行发起的下行门控（手机发起 → 手机下行/网络下行） | PASS |
| **DNINITUPGATE** | 下行发起的上行门控（网络发起 → 手机上行） | PASS |
| **DNINITDNGATE** | 下行发起的下行门控（网络发起 → 手机下行） | PASS |

**典型脚本（112001 Proxy UPF 漫游 pcc_pass，正常转发）**：
```
ADD PCCACTIONPROP:PCCACTPROPNAME="pcc_pass",UPINITUPGATE=PASS,UPINITDNGATE=PASS,DNINITUPGATE=PASS,DNINITDNGATE=PASS;
```

**典型脚本（112001 Proxy UPF 漫游 pcc_drop，全阻断）**：
```
ADD PCCACTIONPROP:PCCACTPROPNAME="pcc_drop",UPINITUPGATE=DISCARD,UPINITDNGATE=DISCARD,DNINITUPGATE=DISCARD,DNINITDNGATE=DISCARD;
```

**典型脚本（020352 IPv6 SA 阻塞业务）**：
```
ADD PCCACTIONPROP:PCCACTPROPNAME="ap_test",UPINITUPGATE=DISCARD,UPINITDNGATE=DISCARD;
```

**典型脚本（110331 FPI 下行透传）**：
```
ADD PCCACTIONPROP:PCCACTPROPNAME="pap_test",UPINITDNGATE=PASS;
```

> 命令 notes：最大记录数 200；适用 NF：PGW-U、UPF；执行后立即生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：PCC 动作属性定义"——前置无；产出 PCCACTPROPNAME 被 ADD SRVPBINDPCCPG(0-00030) 的 PCCACTPROPNAME 引用。

## 决策点：PCC 动作语义（DP 0-00028）

问：ADD PCCACTIONPROP 的门控组合表达哪种动作语义？

| 选项 | 门控组合 | 适用场景 |
|---|---|---|
| 正常转发（pcc_pass，四门控全 PASS） | UPINIT/UP×DN INIT/UP×DN INIT/DN 全部 PASS | 112001 Proxy UPF 漫游（透传业务） |
| 全阻断（pcc_drop，四门控全 DISCARD） | UPINIT/UP×DN INIT/UP×DN INIT/DN 全部 DISCARD | 112001 Proxy UPF 漫游（屏蔽业务） |
| 阻塞上行（IPv6 SA 上行屏蔽） | UPINITUP=DISCARD, UPINITDN=DISCARD（其余默认 PASS） | 020352 IPv6 SA 阻塞业务 |
| 下行透传（FPI 默认其余门控 PASS） | 仅显式 UPINITDNGATE=PASS（其余默认 PASS） | 110331 基于业务流标识的无线资源优化 |

## 约束

- **PCCACTPROPNAME 全网唯一**（critical，rule-0-00328）：不同动作属性之间 PCCACTPROPNAME 不能重复 — 后果：重名 ADD 失败
- **须先于 SRVPBINDPCCPG 配置**（critical，rule-0-00328）：PCCACTPROPNAME 必须先于 ADD SRVPBINDPCCPG(0-00030) 建成 — 后果：未建引用失败
- **上下行门控缺省语义**（warning，rule-0-00328）：4 个门控均 optional，缺省=PASS；按目标语义补齐 DISCARD 的方向即可，无须显式写 PASS — 后果：仅写 DISCARD 时未显式方向默认 PASS 衔接，业务流可能被放通
- **最大记录数 200**（warning，rule-0-00328）：单板最大 200 条 — 后果：超规格 ADD 失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
