# 配置PCC动作属性（ADD PCCACTIONPROP）
> 命令静态知识见 [ADD PCCACTIONPROP](command/UDG/20.15.2/ADD-PCCACTIONPROP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

通过 4 个方向门控（PASS/DISCARD）组合表达"正常转发 / 阻塞上行 / 阻塞下行 / 全阻断"语义。`PCCACTPROPNAME` 全局唯一命名动作属性对象，产出对象供 [绑定PCC策略组业务属性](0-00030.md) 的 `PCCACTPROPNAME` 引用。

| 场景 | UPINITUPGATE | UPINITDNGATE | DNINITUPGATE | DNINITDNGATE | 脚本样例 |
|---|---|---|---|---|---|
| 正常转发（Proxy UPF 漫游 pcc_pass） | PASS | PASS | PASS | PASS | `ADD PCCACTIONPROP:PCCACTPROPNAME="pcc_pass",UPINITUPGATE=PASS,UPINITDNGATE=PASS,DNINITUPGATE=PASS,DNINITDNGATE=PASS;` |
| 全阻断（Proxy UPF 漫游 pcc_drop） | DISCARD | DISCARD | DISCARD | DISCARD | `ADD PCCACTIONPROP:PCCACTPROPNAME="pcc_drop",UPINITUPGATE=DISCARD,UPINITDNGATE=DISCARD,DNINITUPGATE=DISCARD,DNINITDNGATE=DISCARD;` |
| 阻塞上行（IPv6 SA 阻塞业务） | DISCARD | DISCARD | PASS | PASS | `ADD PCCACTIONPROP:PCCACTPROPNAME="ap_test",UPINITUPGATE=DISCARD,UPINITDNGATE=DISCARD;` |
| FPI 下行透传（业务流标识无线资源优化） | （默认 PASS） | PASS | （默认 PASS） | （默认 PASS） | `ADD PCCACTIONPROP:PCCACTPROPNAME="pap_test",UPINITDNGATE=PASS;` |

配置要点：
- 命名（PCCACTPROPNAME）：本端规划，1~31 字符串，全局唯一。
- 4 个门控均 optional，缺省 = PASS；按目标语义补齐需要改为 DISCARD 的方向即可，无须显式写 PASS。
- 重定向参数（UPINITREDIRNM/DNINITREDIRNM）与配置域（CFGDOMAINNAME）三特性场景均未用，属冷门不配。

## 约束
