# 用户Portal（GWFD-110281）

> 特性静态知识见 [GWFD-110281](feature/UDG/20.15.2/GWFD-110281.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性是**重定向子族（用户Portal/WebProxy）的族范本**，后续 WebProxy(2-00015) 复用本族 compound 1-00014。

## 配置概览

用户 Portal 将用户上线后首次访问的 HTTP 请求自动重定向到个人门户网站（Portal Server）。**核心机制：RULE 经 `POLICYTYPE=SMARTREDIRECT` 直接引用 IP Farm（ADD IPFARM 定义），独立于 PCC/HEADEN 的第 3 种 POLICYTYPE，不经 PCCPOLICYGRP**。IP Farm 是重定向 Server 的负荷分担集合 + ICMP 心跳检测基础设施（LBMETHOD 选 Portal Server）。

用户Portal 支持 captive/non-captive 双模式：用户首次访问触发 captive 重定向到 Portal；USERPROFILE.CAPMODETHRES 定时器到期前为 non-captive（不再强制重定向），实现交替。activation 演示双 RULE（rule_test1=SMARTREDIRECT 重定向 + rule_test2=PCC 计费共用同一 FLOWFILTER ff_test1），即重定向与计费可绑定同一 USERPROFILE。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开用户Portal License 开关，单命令）：`SET LICENSESWITCH`（LKV3G5CPPT01）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **IP Farm 重定向基础设施**（整机全局参数 + 心跳检测接口 + 重定向 IP Farm + Portal Server，多命令可复用模块）：`SET IPFARMGLOBAL`（SERVERTYPE=REDIRECT/LBMETHOD）+ `ADD LOGICINF`（Phif 心跳接口）+ `ADD IPFARM`（绑心跳接口）+ `ADD IPFARMSERVER`（Portal Server IP+URL）→ [1-00014](task/UDG/20.15.2/1-00014.md)
3. **三四层过滤条件**（匹配 TCP 报文，过滤链 L34 部分）：`ADD FILTER`（TCP）+ `ADD FLOWFILTER`（容器）+ `ADD FLTBINDFLOWF`（FILTER→FLOWFILTER）→ [1-00009](task/UDG/20.15.2/1-00009.md)
4. **协议识别**（识别 HTTP 协议，直引 atom，**非 1-00009 compound 成员**）：`LOD SIGNATUREDB` → [0-00027](task/UDG/20.15.2/0-00027.md)
5. **七层过滤条件**（URL 匹配，过滤链 L7 部分）：`ADD L7FILTER`（URL）+ `ADD PROTBINDFLOWF`（http+L7FILTERNAME）→ [1-00009](task/UDG/20.15.2/1-00009.md)
6. **刷新生效**（可选，立即生效，单命令）：`SET REFRESHSRV`（REFRESHTYPE=USERPROFILE）→ [0-00015](task/UDG/20.15.2/0-00015.md)
7. **计费属性**（activation 演示，rule_test2=PCC 用，可选）：`ADD URR`+`ADD URRGROUP`+`ADD PCCPOLICYGRP` → [1-00010](task/UDG/20.15.2/1-00010.md)
8. **业务规则**（SMARTREDIRECT 重定向 + PCC 计费双规则 + captive 模板 + 绑定 + APN）：`ADD RULE` rule_test1（POLICYTYPE=SMARTREDIRECT, POLICYNAME=farm_test→IPFarm）+ rule_test2（POLICYTYPE=PCC, POLICYNAME=pcc-pg_test）+ `ADD USERPROFILE`（CAPMODETHRES=6）+ `ADD RULEBINDING` + `ADD APN` → [1-00011](task/UDG/20.15.2/1-00011.md)
   - 可选分支（HTTPS 重定向旁路）：`ADD BLACKLISTRULE`（重定向 URL 为 HTTPS 时放行）→ [0-00293](task/UDG/20.15.2/0-00293.md)
   - 可选分支（HTTP 重定向响应码，activation 主脚本未演示）：`SET SRVCOMMONPARA:HTTPREDIRCODE=RSP_CODE_302`（枚举 RSP_CODE_301永久/RSP_CODE_302临时/RSP_CODE_303，**非裸数字**）→ [0-00065](task/UDG/20.15.2/0-00065.md)；证据 [修改HTTP重定向响应码_66620118](evidence/UDG/20.15.2/修改HTTP重定向响应码_66620118.md)（独立参数调整文档）

## 决策点

**DP（特性级）：负荷分担方式 + 触发匹配层 + IP 版本 + 无法重定向缺省动作 + HTTPS 重定向 URL 处理 + HTTP 响应码 + 计费组合**（影响 IP Farm 参数 + 过滤链走法 + RULE 组合）。维度按 activation §数据规划表 + §操作步骤说明 + §任务示例全列。

| 场景 | 负荷分担 | IP版本 | 触发匹配 | 无法重定向缺省 | HTTPS重定向URL | HTTP响应码 | 计费组合 | 走法 |
|---|---|---|---|---|---|---|---|---|
| **标准部署（activation 演示）** | LEAST_LOAD | IPv4 | L7 URL（www.huawei.com/*）+ L34 TCP | BLOCK | 不涉及（URL 为 HTTP）| 默认 | SMARTREDIRECT + PCC 双规则 | 骨架 + [1-00014](task/UDG/20.15.2/1-00014.md) + [1-00009](task/UDG/20.15.2/1-00009.md) L34+L7 + [1-00011](task/UDG/20.15.2/1-00011.md) |
| 负荷分担轮询变体 | ROUND_ROBIN | IPv4 | L7 URL + L34 | BLOCK | 不涉及 | 默认 | 任意 | SET IPFARMGLOBAL LBMETHOD=ROUND_ROBIN |
| 负荷分担最久未用变体 | LEAST_RECENTLY_USED | IPv4 | L7 URL + L34 | BLOCK | 不涉及 | 默认 | 任意 | SET IPFARMGLOBAL LBMETHOD=LEAST_RECENTLY_USED |
| IPv6 全链路变体 | LEAST_LOAD | IPv6 | L7 URL + L34（IPv6）| BLOCK | 不涉及 | 默认 | 任意 | IPFARM/SERVER/LOGICINF/Filter 同 v6（DSP RULECHECK 验）|
| 无法重定向放行变体 | LEAST_LOAD | IPv4 | L7 URL + L34 | **PASS** | 不涉及 | 默认 | 任意 | ADD/MOD IPFARMSERVER DEFAULTACT=PASS |
| HTTPS 重定向旁路变体 | LEAST_LOAD | IPv4 | L7 URL + L34 | BLOCK | **BLACKLISTRULE 放行** | 默认 | 任意 | + ADD BLACKLISTRULE（[0-00293](task/UDG/20.15.2/0-00293.md)）|
| HTTP 响应码变体 | LEAST_LOAD | IPv4 | L7 URL + L34 | BLOCK | 不涉及 | **HTTPREDIRCODE=RSP_CODE_301/302/303**（枚举，非裸数字）| 任意 | SET SRVCOMMONPARA:HTTPREDIRCODE=RSP_CODE_302（[0-00065](task/UDG/20.15.2/0-00065.md)）|
| 仅重定向不计费变体 | LEAST_LOAD | IPv4 | L7 URL + L34 | BLOCK | 不涉及 | 默认 | **SMARTREDIRECT 单规则**（无 PCC）| [1-00010](task/UDG/20.15.2/1-00010.md) 可省 |

**联动**：
- 负荷分担 → [1-00014](task/UDG/20.15.2/1-00014.md) 场景差异 §5（SET IPFARMGLOBAL LBMETHOD）
- IP 版本 → [1-00014](task/UDG/20.15.2/1-00014.md) 场景差异 §6（全链路一致，DSP RULECHECK 验）
- 触发匹配（L7 URL + L34）→ [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链（L34 FILTER+FLTBINDFLOWF + L7 L7FILTER+PROTBINDFLOWF，两段都用）
- 无法重定向 → [1-00014](task/UDG/20.15.2/1-00014.md) 决策点 DEFAULTACT（ADD/MOD IPFARMSERVER）
- HTTPS 重定向 → [0-00293](task/UDG/20.15.2/0-00293.md) BLACKLISTRULE（旁路放行）
- HTTP 响应码 → [0-00065](task/UDG/20.15.2/0-00065.md) SRVCOMMONPARA 的 HTTPREDIRCODE 参数（RSP_CODE_301/302/303 枚举）
- 计费组合 → [1-00010](task/UDG/20.15.2/1-00010.md)（双规则时用，单规则可省）

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5CPPT01` — 未开则用户Portal 不生效。
- **强依赖 SA-Basic**（critical）：GWFD-110101（License LKV3G5SABS01）—— HTTP 报文解析基础，未开则无法识别报文。
- **协议依赖 SA-Web Browsing**（critical）：GWFD-110103（License LKV3G5SAWB01）—— HTTP/WAP2.0 解析必需；WAP1.X 承载业务还需 SA-Mobile（GWFD-110105）。
- **POLICYTYPE=SMARTREDIRECT 触发条件**（critical）：RULE.POLICYTYPE=SMARTREDIRECT + POLICYNAME=IPFARMNAME（直接引用 IP Farm，PCC 策略组名称=NULL）— 指错则重定向不触发。
- **IP 版本全链路一致**（critical）：IPFARM/SERVER/LOGICINF/Filter/接口须同 v4 或同 v6 — `DSP RULECHECK` 检查一致性；不一致则重定向失效。
- **仅 HTTP/WAP1.X/WAP2.0，不支持 HTTPS/HTTP2.0**（warning）：HTTPS 需 BLACKLISTRULE 旁路放行（[0-00293](task/UDG/20.15.2/0-00293.md)）；HTTP2.0 不支持 — 混用则对应协议报文不做重定向。
- **HTTPS 重定向 URL 需 BLACKLISTRULE 放行**（warning）：UDG 不支持 HTTPS 协议报文的重定向处理；重定向 URL 为 HTTPS 时命中规则会阻塞，需配 ADD BLACKLISTRULE 放行 — 不配则重定向后业务无法处理。
- **至少一 Server 状态 up**（warning）：`LST IPFARMSERVER` 查询须保证至少一个 server 状态 up — 否则重定向无可用目标。
- **不同 IP Farm 需不同心跳接口**（warning）：心跳检测接口不共享 — 复用则心跳冲突。
- **REFRESHSRV 60s 自动生效**（info）：系统自动在 60s 后将新配置的 L7Filter 置为生效；SET REFRESHSRV（REFRESHTYPE=USERPROFILE）可立即生效。
- **规格**（info）：整机 ≤64 IP Farm；每个 IP Farm ≤512 Server。
