# IPv4v6双栈接入（GWFD-020403）

> 特性静态知识见 [GWFD-020403](feature/UDG/20.15.2/GWFD-020403.md) + [特性知识](业务图谱体系/APN业务域/feature-knowledge/GWFD-020403-IPv4v6双栈接入.md)。
>
> **本页定位**：配置生成实例化视角——GWFD-020403 IPv4v6 双栈接入的完整配置体系（1 个 License 前置 + 1 个双栈特化 compound + 6 步通用骨架 + 2 条调测命令）。
>
> **★ 本特性与 GWFD-010105 的关系**：GWFD-020403 是**双栈能力使能层**（License + 双栈对称配置 + RA 通告机制）；GWFD-010105 是**地址分配机制层**（4 子方式 + 规则 + 地址池体系）。本特性的激活脚本复用 GWFD-010105 的命令体系，但叠加 4 项双栈对称差异（详见 [1-00025](task/UDG/20.15.2/1-00025.md)）。

## 配置概览

本特性按 **1 前置 License + 1 双栈特化 compound + 6 步通用骨架 + 2 调测命令** 组织配置：

**对象链（双栈对称模型）：**
```
License 使能（前置开关）
    ↓
双栈 APN（IPv4 VPN + IPv6 VPN 对称使能）
    ↓
双栈 POOLGROUP（IPV4/IPV6ALLOCPRIALG 双算法使能）
    ↓
双池（IPv4 POOL + IPv6 POOL）+ 双段（V4START/V4END + V6PREFIXSTART/V6PREFIXEND/V6PREFIXLENGTH）
    ↓
双 POOLBINDGROUP（双池绑到同一组）
    ↓
APN↔POOLGROUP 单映射（双栈统一）
    ↓
Router Advertisement 主动下发（★本特性核心机制，由双栈 APN+IPv6 地址段自动触发）
    ↓
下行路由发布（OSPF+OSPFv3 双协议，wlr 引入用户子段路由）
```

**6 步通用骨架**（双栈典型场景）：

| 步骤 | 干啥 | 主要命令 |
|---|---|---|
| 1. License 开关 | 打开 LKV3G5VDSA01 配置开关 | `SET LICENSESWITCH` |
| 2. 双栈 APN（双 VPN 对称使能） | APN 绑 IPv4 VPN + IPv6 VPN | `ADD APN`（HASVPN+HASVPNIPV6 双开关） |
| 3. 双栈 POOLGROUP + 双池 | POOLGROUP + IPv4 池 + IPv6 池 | `ADD POOLGROUP` + `ADD POOL` ×2 |
| 4. 双段（IPv4 + IPv6） | IPv4 段 + IPv6 段 | `ADD SECTION` ×2 |
| 5. 双池绑组 + APN 映射 | 双 POOLBINDGROUP + POOLGRPMAP | `ADD POOLBINDGROUP` ×2 + `ADD POOLGRPMAP` |
| 6. 下行路由发布（OSPF+OSPFv3 双协议） | IPv4 走 OSPF / IPv6 走 OSPFv3 引入 wlr | `ADD OSPF/OSPFIMPORTROUTE` + `ADD OSPFV3/OSPFV3IMPORTROUTE` |
| 调测 | 验证双栈上下文与 RA 通告 | `DSP SESSIONINFO` + `EXP MML` |

**关键约束**：
- **必须加载 License LKV3G5VDSA01**（控制项 82209829）并 SET LICENSESWITCH 打开
- **双栈对称配置**：APN 双 VPN、POOLGROUP 双算法、双池双段同时配齐
- V6PREFIXLENGTH 范围 49~64；<64 时切换为 IPv6 PD 模式（与 GWFD-020406 协同）
- 双栈 VPN 三者一致：IPv4 链 / IPv6 链各自需满足"APN 绑 VPN / 池绑 VPN / Gi/SGi/N6 外联口绑 VPN"三者一致
- 弱覆盖场景掉话率增加（RA 主动下发机制副作用）

## 配置流程

> 每步显示"干啥 + 含哪些命令"。双栈典型场景执行全部步骤；单栈退化场景（仅 IPv4 / 仅 IPv6）跳过本特性复用 [1-00022](task/UDG/20.15.2/1-00022.md) 单池范式。

1. **License 开关**（LKV3G5VDSA01，双栈能力使能前置）：
   - `SET LICENSESWITCH` → [0-00019](task/UDG/20.15.2/0-00019.md)（`LICITEM="LKV3G5VDSA01", SWITCH=ENABLE`）

2. **双栈 APN**（双 VPN 对称使能）：
   - `ADD APN` → [0-00020](task/UDG/20.15.2/0-00020.md)（`APN="apn-test"`, `HASVPN=ENABLE`, `VPNINSTANCE="VRF_Internet"`, **`HASVPNIPV6=ENABLE`**, **`VPNINSTANCEIPV6="VRF_Internet"`**）

3. **双栈地址池体系**（POOLGROUP + IPv4/IPv6 双池 + 双段 + 双绑 + 单映射）：
   - 复用 → [1-00025](task/UDG/20.15.2/1-00025.md) 双栈 APN + 双栈 POOLGROUP（IPv4v6 对称配置）
   - 步骤 2-7：`ADD POOLGROUP` + `ADD POOL` ×2（IPVERSION=IPV4/IPV6）+ `ADD SECTION` ×2（V4START/V4END + V6PREFIXSTART/V6PREFIXEND/V6PREFIXLENGTH）+ `ADD POOLBINDGROUP` ×2 + `ADD POOLGRPMAP`

4. **下行路由发布**（OSPF+OSPFv3 双协议引入 wlr）：
   - IPv4：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFNETWORK` + `ADD OSPFIMPORTROUTE PROTOCOL=wlr` → 复用 [1-00024](task/UDG/20.15.2/1-00024.md)（IPv4 段）
   - IPv6：`ADD OSPFV3` + `ADD OSPFV3AREA` + `ADD OSPFV3IMPORTROUTE` → 复用 [0-00078](task/UDG/20.15.2/0-00078.md) + [0-00079](task/UDG/20.15.2/0-00079.md) + [0-00080](task/UDG/20.15.2/0-00080.md)（IPv6 段）
   - **说明**：参考信息章节列出 7 条路由命令（ADD ROUTEPOLICY / ROUTEPOLICYNODE / MATCHROUTETYPE / OSPF / OSPFV3 / OSPFIMPORTROUTE / OSPFV3IMPORTROUTE），其中路由策略族（ROUTEPOLICY×3）为可选用法，本特性激活任务示例未演示；OSPF 系列（4 条 IPv4 + 3 条 IPv6）为必配基础。

5. **Router Advertisement 主动下发**（★本特性核心机制，无需 MML）：
   - 由双栈 APN + IPv6 地址段配置**自动触发**：UE 接入后无论是否发送 Router Solicitation，只要 PDP Type=IPv4v6，UDG 主动向 UE 发送 RA 消息，通告 IPv6 路由与前缀
   - **无需额外 MML 命令**——属平台能力，依赖步骤 1 License + 步骤 2-3 双栈配置使能

6. **调测**（验证双栈上下文与三元组）：
   - `DSP SESSIONINFO` → [0-00327](task/UDG/20.15.2/0-00327.md)（验证 `PDP Type=IPv4v6` + `IPv4 PDP address` + `IPv6 PDP address` 三元组）
   - `EXP MML` → [0-00333](task/UDG/20.15.2/0-00333.md)（配置导出/故障定位）

## 决策点

**DP（特性级）：场景 × PDN 类型 × 双栈 VPN 隔离 × 走法**

| 场景 | PDN 类型 | 双栈 VPN 隔离 | 走法（compound/atom 编排） |
|---|---|---|---|
| 双栈典型（PDP Type=IPv4v6） | IPv4v6 | 否（IPv4/IPv6 同 VPN） | [1-00025](task/UDG/20.15.2/1-00025.md)（全 7 步）+ [1-00024](task/UDG/20.15.2/1-00024.md)（OSPF）+ OSPFV3（[0-00078](task/UDG/20.15.2/0-00078.md) + [0-00080](task/UDG/20.15.2/0-00080.md)） |
| 双栈 IPv6 PD（V6PREFIXLENGTH<64） | IPv4v6 | 否 | [1-00025](task/UDG/20.15.2/1-00025.md)（步骤 6 `V6PREFIXLENGTH=49~63`）+ 与 GWFD-020406 IPv6 PD 协同 |
| 双栈 VPN 隔离 | IPv4v6 | 是（IPv4/IPv6 不同 VPN） | [1-00025](task/UDG/20.15.2/1-00025.md)（步骤 1 `VPNINSTANCE≠VPNINSTANCEIPV6` + 步骤 3-7 IPv6 池/段绑 IPv6 VPN） |
| 仅 IPv4 单栈（退化） | IPv4 | — | **不执行 [1-00025](task/UDG/20.15.2/1-00025.md)**；复用 [1-00022](task/UDG/20.15.2/1-00022.md) 单池范式（步骤 1 APN 配 `HASVPNIPV6=DISABLE`）+ [1-00024](task/UDG/20.15.2/1-00024.md) |
| 仅 IPv6 单栈（退化） | IPv6 | — | **不执行 [1-00025](task/UDG/20.15.2/1-00025.md)**；复用 [1-00022](task/UDG/20.15.2/1-00022.md) 单池范式（步骤 1 APN 配 `HASVPN=DISABLE`）+ OSPFV3 |

> **关键规则**：
> - 必须首先执行步骤 1 License 开关（`SET LICENSESWITCH:LICITEM="LKV3G5VDSA01",SWITCH=ENABLE`），否则步骤 2-6 业务命令配齐也无法创建双栈上下文
> - 步骤 3 复用 [1-00025](task/UDG/20.15.2/1-00025.md) 时，**双栈典型场景执行全部 7 步**；单栈场景不执行本特性，复用 [1-00022](task/UDG/20.15.2/1-00022.md) 1 池范式
> - 双栈典型场景步骤 4 必须双配 OSPF（IPv4）+ OSPFV3（IPv6）双协议引入 wlr；任一缺失致 IPv4 或 IPv6 子段路由不发布
> - 单栈退化场景 License 仍需加载（步骤 1）但不发挥双栈作用

## 约束

- **License 强制前置**（critical，rule-0-01610）：必须加载 License 控制项 82209829 LKV3G5VDSA01 并通过 SET LICENSESWITCH 打开开关 — 后果：未加载则双栈能力不可用，UE 退化为 IPv4 单栈
- **双栈对称配置**（critical，rule-0-01611）：APN 必须同时使能 HASVPN 与 HASVPNIPV6；POOLGROUP 必须同时绑定 IPv4 池与 IPv6 池；POOLGROUP 必须同时使能 IPV4ALLOCPRIALG 与 IPV6ALLOCPRIALG — 后果：不对称配置致双栈退化
- **VPN 三者一致（IPv4 与 IPv6 链各自）**（critical，rule-0-01612）：IPv4 链：APN/地址池/Gi 外联口 绑定的 VPN 必须一致；IPv6 链同理 — 后果：VPN 不一致致地址分配或路由异常
- **V6PREFIXLENGTH 范围 49~64**（critical，rule-0-01613）：V6PREFIXLENGTH 必须为 49~64 之间 — 后果：超出范围命令拒收
- **V6PREFIXLENGTH<64 切换 PD 模式**（warning，rule-0-01614）：当前缀长度 < 64 时，地址池采用 IPv6 Prefix Delegation 方式分配（与 GWFD-020406 IPv6 PD 协同） — 后果：UE 获 IPv6 前缀而非完整 IPv6 地址
- **C 面网元必须为华为配套设备**（critical，rule-0-00497）：因 N4/Sxb 对 PFCP 协议有私有扩展（双栈 CHV4=1&CHV6=1 协商） — 后果：非华为 C 面激活失败
- **APN/DNN 全网规划一致**（critical，rule-0-00499）：UDG 本端 APN/DNN 实例应与 C 面（SMF/SGW-C/PGW-C）配置一致 — 后果：PFCP 信令失配致双栈上下文无法建立
- **ROUTER ID 全网唯一**（critical，rule-0-00508）：OSPF/OSPFv3 Router ID 必须全网唯一 — 后果：邻接建立失败
- **每 APN 绑池数 ≤16**（warning，rule-0-01615）：每 APN 支持绑定的地址池最多 16 个；双栈场景下 IPv4 池 + IPv6 池合计计入 — 后果：超规格新增失败
- **每 POOL 绑段数 ≤64**（warning，rule-0-01616）：每本地地址池支持绑定的地址段最多 64 个；双栈场景下 IPv4 池与 IPv6 池各自独立 — 后果：超规格新增失败
- **主机路由规格 40 万**（warning，rule-0-00502）：每 UDG 支持的主机路由规格为 40 万；双栈场景双协议路由条目均计入 — 后果：超规格下行路由发布异常
- **弱覆盖场景掉话率上升**（warning，rule-0-01617）：UE 需通过 UDG 下发 RA 获取 IPv6 地址；弱覆盖场景下 eNodeB 无法下发该报文给终端时**掉话率会增加** — 后果：弱覆盖区域用户体验降级
- **首次发布版本**（warning）：UDG 20.0.0；UNC 20.0.0（控制面侧 WSFD-104002） — 后果：低版本网元不支持
- **参考信息命令（路由策略族）非必配**（info，rule-0-01619）：参考信息列出 7 条路由命令，其中 ADD ROUTEPOLICY / ROUTEPOLICYNODE / MATCHROUTETYPE 为可选用法，本特性激活任务示例未演示；OSPF 系列（4 条 IPv4 + 3 条 IPv6）为必配基础 — 后果：未配路由策略族不影响双栈上下文创建
