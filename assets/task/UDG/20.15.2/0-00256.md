# 增加内容分类组（ADD CONTCATEGROUP）

> 命令静态知识见 [ADD CONTCATEGROUP](command/UDG/20.15.2/ADD-CONTCATEGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务定义内容分类组（CONTCATEGROUP）——URL 过滤的分类匹配维度容器。命令按 `CATEGORYTYPE` 四路互斥编排，每个分支决定哪组分类维度参数必选、哪组必不传。适用 NF：PGW-U、UPF。

`CONTCATEGNAME` 是分类组名，供后续 [ADD CONTCATEGBIND](task/UDG/20.15.2/0-00257.md) 引用绑定到内容过滤策略。

**四路取值样例（数据规划表，主特性 GWFD-110471）**

| CATEGORYTYPE 分支 | 必选参数 | 必不传参数 | 样例值 |
|---|---|---|---|
| SPECIFIC（特定分类ID，默认） | CATEGORYID | STARTID/ENDID/PRIORITY/CATEGORYNAME | CATEGORYID=1 |
| RANGE（分类ID范围） | STARTID、ENDID（且 STARTID≤ENDID，区间不可重叠） | CATEGORYID/PRIORITY/CATEGORYNAME | STARTID=10, ENDID=20 |
| PRIORITY（优先级，全局唯一） | PRIORITY（整机最多 1 条，默认 65535） | STARTID/ENDID/CATEGORYID/CATEGORYNAME | PRIORITY=65535 |
| NAME（分类名称） | CATEGORYID、CATEGORYNAME | STARTID/ENDID/PRIORITY | CATEGORYID=1, CATEGORYNAME="cate1" |

**主场景脚本样例**（SPECIFIC 路径，ICAP Server 已导入 URL 分类数据，CATEGORYID 对应其分类值）

```
ADD CONTCATEGROUP:CONTCATEGNAME="ccgp1", CATEGORYTYPE=SPECIFIC, CATEGORYID=1;
```

**未绑参数说明**：CFGDOMAINNAME（配置域名称）optional，主场景不配。

## 决策点

**DP [UDG@20.15.2@DecisionPoint@0-00119] —— CATEGORYTYPE 四路分叉（SPECIFIC/RANGE/PRIORITY/NAME）**
决定本任务的必选/必不传参数集合（见上表）。默认 SPECIFIC 路径。本 atom 仅 1 特性使用（GWFD-110471），不存在跨特性分叉。

**DP [UDG@20.15.2@DecisionPoint@0-00121] —— 整机优先级分类组自动添加**
系统首次添加且 CATEGORYTYPE≠PRIORITY 时，自动追加一条 PRIORITY=65535 默认记录（命令 notes 强约束）。该自动追加不可见但占整机 200 个内容分类组限额（隐性消耗，配置工程师易遗漏）。本 DP 把"系统自动追加"显化为决策点。

## 约束

- 命令最大记录数 **20200**；整机支持 **200 个内容分类组**；每个组最多 **101 条**分类记录，其中 RANGE/SPECIFIC/NAME 合计 ≤100 条，PRIORITY 类型限 **1 条**。
- CATEGORYTYPE=PRIORITY 的分类组，其 PRIORITY **全局唯一**，重复→ADD 失败。
- RANGE 类型须 STARTID≤ENDID，且同一组内 RANGE 记录区间不可重叠。
- 命令执行后**立即生效**。
