# 配置匹配路由类型（ADD MATCHROUTETYPE）

> 命令静态知识见 [ADD MATCHROUTETYPE](command/UDG/20.15.2/ADD-MATCHROUTETYPE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD MATCHROUTETYPE 在已建路由策略节点下挂一条"按路由类型匹配"的条件。**(POLICYNAME,NODESEQUENCE) 联合引用上游节点**，**ROUTETYPE 必选**（决定匹配哪类动态路由协议学到的路由）。核心配置空间有 2 个维度：**策略节点绑定**（POLICYNAME+NODESEQUENCE）、**路由类型**（ROUTETYPE）。1 个特性命中（GWFD-020401 IPv6 承载上下文）使用 nssaExternal1。

### 配置维度 1：策略节点绑定（参数 POLICYNAME+NODESEQUENCE，复合必选）

| 参数 | 取值 | 作用 |
|---|---|---|
| **POLICYNAME** | ref [0-00111](task/UDG/20.15.2/0-00111.md) ADD ROUTEPOLICY | 路由策略名 |
| **NODESEQUENCE** | ref [0-00112](task/UDG/20.15.2/0-00112.md) ADD ROUTEPOLICYNODE | 节点序号 |

> (POLICYNAME,NODESEQUENCE) 必须对应已通过 `ADD ROUTEPOLICYNODE` 创建的节点；一个节点可挂多条匹配条件（同条件类型）。

### 配置维度 2：路由类型（参数 ROUTETYPE，必选）

| 取值 | 作用 |
|---|---|
| **external1** | OSPF 外部路由类型 1 |
| **external2** | OSPF 外部路由类型 2 |
| **nssaExternal1** | OSPF NSSA 外部路由类型 1（GWFD-020401 范式） |
| **internal** | OSPF 内部路由 |
| **其他** | 见命令真相表枚举 |

> 路由类型枚举见命令真相表；不同动态路由协议学到的路由类型不同，匹配 ROUTETYPE 即匹配该类型的路由。

**典型脚本（GWFD-020401 IPv6 承载上下文，匹配 OSPF NSSA 外部路由类型 1）**：
```
ADD MATCHROUTETYPE:POLICYNAME="policy1",NODESEQUENCE=10,ROUTETYPE=nssaExternal1;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：挂路由类型匹配条件"——前置 [0-00111](task/UDG/20.15.2/0-00111.md) ADD ROUTEPOLICY 建策略 + [0-00112](task/UDG/20.15.2/0-00112.md) ADD ROUTEPOLICYNODE 挂节点。

## 决策点：路由类型匹配选择（DP 0-00113）

（本命令无独立决策分支，仅有 ROUTETYPE 枚举选择。）

| 选项 | ROUTETYPE | 适用场景 |
|---|---|---|
| **NSSA 外部路由类型 1**（020401） | nssaExternal1 | OSPF NSSA 区域重发布到 IPv6 路由表的路由 |
| **外部路由类型 1** | external1 | OSPF 外部类型 1（E1）路由 |
| **外部路由类型 2** | external2 | OSPF 外部类型 2（E2）路由 |
| **内部路由** | internal | OSPF 内部区域间/区域内路由 |

> 1 个特性命中（020401）；ROUTETYPE 必选；同一节点可挂多条 ROUTETYPE 匹配。

## 约束

- **(POLICYNAME,NODESEQUENCE) 必须已建**（critical，rule-0-00113）：POLICYNAME 必须已通过 ADD ROUTEPOLICY 创建，NODESEQUENCE 必须已通过 ADD ROUTEPOLICYNODE 创建 — 后果：节点不存在命令拒收
- **ROUTETYPE 必选**（critical，rule-0-00113）：ROUTETYPE 是必选参数 — 后果：缺 ROUTETYPE 命令拒收
- **节点遍历顺序**（info，rule-0-00113）：路由策略按节点 NODESEQUENCE 升序遍历匹配，匹配条件决定命中范围 — 后果：节点号/匹配条件编排不当导致预期外路由被匹配
- **执行后立即生效**（warning，rule-0-00113）：本命令执行后立即生效 — 后果：误配匹配条件即刻影响路由策略生效范围
