# 配置匹配路由类型（ADD MATCHROUTETYPE）

> 命令静态知识见 [ADD MATCHROUTETYPE](command/UDG/20.15.2/ADD-MATCHROUTETYPE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD MATCHROUTETYPE 为已建路由策略节点指定要匹配的路由类型（ROUTETYPE），决定哪些外部路由类型被该节点放行/拒绝。POLICYNAME 与 NODESEQUENCE **双引用**上游两个 atom（[配置路由策略](task/UDG/20.15.2/0-00111.md) + [配置路由策略节点](task/UDG/20.15.2/0-00112.md)），构成复合键。

| 参数 | 取值样例 | 来源 | 说明 |
|---|---|---|---|
| POLICYNAME | `policy1` | reference（← 0-00111） | 引用上游 ROUTEPOLICY |
| NODESEQUENCE | `10` | reference（← 0-00112） | 引用上游 ROUTEPOLICYNODE |
| ROUTETYPE | `nssaExternal1` | 本端规划 | 枚举，OSPF/NSSA 外部路由类型 |

**典型脚本**（GWFD-020401 IPv6承载上下文，OSPFv3 引入 WLR 路由类型过滤）：

```
ADD MATCHROUTETYPE:POLICYNAME="policy1",NODESEQUENCE=10,ROUTETYPE=nssaExternal1;
```

**三件套时序**：① ADD ROUTEPOLICY（建策略名）→ ② ADD ROUTEPOLICYNODE（挂节点）→ ③ ADD MATCHROUTETYPE（节点匹配路由类型）→ ④ ADD OSPFV3IMPORTROUTE（引入时按策略过滤，ROUPOLINAME 再引用 POLICYNAME）。

**ROUTETYPE 控制语义**：本 atom 决定哪些外部路由类型（如 nssaExternal1）被该节点按 MATCHMODE 放行；是 OSPFv3 引入 WLR 路由时的类型过滤开关。

## 约束

- **rule-0-00242**（warning）**前置双依赖**：必须先建 ROUTEPOLICY（POLICYNAME）+ ROUTEPOLICYNODE（NODESEQUENCE），即 (POLICYNAME,NODESEQUENCE) 复合键引用必须落地；命令最大记录数 65535；执行后立即生效 — 后果：复合键引用悬空
