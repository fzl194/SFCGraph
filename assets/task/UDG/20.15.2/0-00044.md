# 配置地址池IP地址段（ADD SECTION）

> 命令静态知识见 [ADD SECTION](command/UDG/20.15.2/ADD-SECTION.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法（配置方法字典）

ADD SECTION 是地址池族链路的"地址段填充层"：引用已建的 [ADD POOL](task/UDG/20.15.2/0-00043.md) POOLNAME，为该池绑定一段连续 IPv4/IPv6 地址区间（Section），供 LOCAL 池分配地址或 EXTERNAL 池发布路由用。命令核心配置空间有 5 个维度：**地址池引用**（POOLNAME 必选 + 必须已建）、**地址段号**（SECTIONNUM 0~63，同一池内唯一）、**IP 版本**（IPVERSION 决定 V4* 或 V6PREFIX* 参数组）、**地址范围**（V4 起止 IP 或 V6 起止前缀+长度）、**路由掩码**（MASK 控制系统发布用户 wlr_ud 网段路由掩码长度）。6 个特性命中覆盖 IPV4 段（010104/010105/010107/020421）、IPV6 双栈段（020403）、IPV6 Prefix Delegation 段（020406）。

### 配置维度 1：地址池引用（参数 POOLNAME，必选）

| 参数 | 范围 | 备注 |
|---|---|---|
| **POOLNAME** | 字符串 1~79 | 引用已配的 [ADD POOL](task/UDG/20.15.2/0-00043.md) POOLNAME；前置依赖：POOL 必须先建 |

### 配置维度 2：地址段号（参数 SECTIONNUM，必选）

| 参数 | 范围 | 备注 |
|---|---|---|
| **SECTIONNUM** | 整数 0~63 | 同一池内唯一；同 IPVERSION 下多段用不同 SECTIONNUM 区分（020406 多段范式：testpool1/testpool3/testpool 共享 SECTIONNUM=1，因属不同 POOLNAME） |

### 配置维度 3：IP 版本（参数 IPVERSION，必选）

| 取值 | 配套必选地址范围参数 | 适用场景 |
|---|---|---|
| **IPV4** | V4STARTIP + V4ENDIP | IPv4 段、IPV4 双栈 v4 端、PDN 路由探测 v4 |
| **IPV6** | V6PREFIXSTART + V6PREFIXEND + V6PREFIXLENGTH | IPv6 双栈 v6 端、Prefix Delegation v6 段 |

### 配置维度 4：地址范围（参数 V4*/V6PREFIX*，按 IPVERSION 条件必选）

**IPV4 分支**：

| 参数 | 取值 | 备注 |
|---|---|---|
| **V4STARTIP** | IPv4 点分十进制 | 段起始 IP |
| **V4ENDIP** | IPv4 点分十进制 | 段终止 IP（必须 ≥ V4STARTIP） |

**IPV6 分支**：

| 参数 | 取值 | 备注 |
|---|---|---|
| **V6PREFIXSTART** | IPv6 前缀 | 段起始前缀 |
| **V6PREFIXEND** | IPv6 前缀 | 段终止前缀（必须 ≥ V6PREFIXSTART） |
| **V6PREFIXLENGTH** | 整数（位） | 前缀长度；Prefix Delegation 典型 63，双栈典型 64 |

### 配置维度 5：路由掩码（参数 MASK，可选）

| 参数 | 范围 | 作用 |
|---|---|---|
| **MASK** | 整数（位） | 控制系统发布用户 wlr_ud 网段路由掩码长度；配置为 N 代表发布路由覆盖地址个数为 2^N |

**典型脚本（010104 IPV4 段）**：
```
ADD SECTION:POOLNAME="testpool",SECTIONNUM=1,IPVERSION=IPV4,V4STARTIP="10.10.1.1",V4ENDIP="10.10.1.10";
```

**典型脚本（010107 主备 4 段，IPV4+IPV6 各 2 段）**：
```
ADD SECTION: POOLNAME="pool1", SECTIONNUM=1, IPVERSION=IPV4, V4STARTIP="10.1.1.0", V4ENDIP="10.1.1.225";
ADD SECTION: POOLNAME="pool2", SECTIONNUM=2, IPVERSION=IPV6, V6PREFIXSTART="fc00:1::1", V6PREFIXEND="fc00:1::1111", V6PREFIXLENGTH=64;
ADD SECTION: POOLNAME="pool3", SECTIONNUM=3, IPVERSION=IPV4, V4STARTIP="10.1.1.0", V4ENDIP="10.1.1.225";
ADD SECTION: POOLNAME="pool4", SECTIONNUM=4, IPVERSION=IPV6, V6PREFIXSTART="fc00:1::1", V6PREFIXEND="fc00:1::1111", V6PREFIXLENGTH=64;
```

**典型脚本（020403 双栈）**：
```
ADD SECTION:POOLNAME="testpool", SECTIONNUM=1, IPVERSION=IPV4, V4STARTIP="10.0.0.0", V4ENDIP="10.0.127.255";
ADD SECTION:POOLNAME="testpoolv6", SECTIONNUM=1, IPVERSION=IPV6, V6PREFIXSTART="fc00:0000:0000:fcee:0000:0000:0000:0000", V6PREFIXEND="fc00:0000:0000:fcef:ffff:ffff:ffff:ffff", V6PREFIXLENGTH=64;
```

**典型脚本（020406 Prefix Delegation，多段）**：
```
ADD SECTION:POOLNAME="testpool1",SECTIONNUM=1,IPVERSION=IPV6,V6PREFIXSTART="fc00:0000:0000:fcee:0000:0000:0000:0000",V6PREFIXEND="fc00:0000:0000:fcef:ffff:ffff:ffff:ffff", V6PREFIXLENGTH=63;
```

> 命令 notes：适用 NF：PGW-U、UPF；执行后立即生效；SECTIONNUM 同 POOLNAME 内唯一；V4ENDIP/V6PREFIXEND 必须 ≥ 起始。

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：地址段填充"——前置依赖 [0-00043](task/UDG/20.15.2/0-00043.md)（POOL 必须先建）；与 [0-00045](task/UDG/20.15.2/0-00045.md)（POOLBINDGROUP 绑组）、[0-00047](task/UDG/20.15.2/0-00047.md)（POOLGRPMAP 建 APN 映射）协同完成地址分配全链路。

## 决策点：IP 版本与地址范围组合（DP 0-00044）

问：ADD SECTION 选哪个 IPVERSION + 地址范围参数组合？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPV4 单段 | IPVERSION=IPV4 + V4STARTIP + V4ENDIP | 010104 地址分配（最小 IPv4 段）、010105 多段 IPv4、010107 路由冗余 IPV4 主备、020421 基于位置 |
| IPV6 双栈段 | IPVERSION=IPV6 + V6PREFIXSTART + V6PREFIXEND + V6PREFIXLENGTH=64 | 020403 IPv4v6 双栈接入 |
| IPV6 Prefix Delegation | IPVERSION=IPV6 + V6PREFIXSTART + V6PREFIXEND + V6PREFIXLENGTH=63 | 020406 IPv6 Prefix Delegation（前缀长度典型 63） |
| IPV4/IPV6 主备成对 | 同上 IPV4+IPV6 各一段 | 010107 静态地址路由冗余（pool1+pool2 主用 / pool3+pool4 备用） |

> 6 个特性命中以 IPV4 段为主（010104/010105/010107/020421），双栈/PD 场景为 IPV6 段（020403/020406）。

## 约束

- **POOLNAME 前置依赖**（critical，rule-0-00444）：POOLNAME 必须先由 [0-00043](task/UDG/20.15.2/0-00043.md) 建好 — 后果：未建引用命令拒收
- **IPVERSION 决定 V4*/V6PREFIX* 参数集**（critical，rule-0-00444）：IPVERSION=IPV4 必配 V4STARTIP+V4ENDIP；IPVERSION=IPV6 必配 V6PREFIXSTART+V6PREFIXEND+V6PREFIXLENGTH — 后果：缺配套参数命令拒收
- **SECTIONNUM 同 POOLNAME 内唯一**（critical，rule-0-00444）：同一 POOLNAME 下 SECTIONNUM 0~63 不能重复 — 后果：重复命令拒收
- **V4ENDIP/V6PREFIXEND ≥ 起始**（critical，rule-0-00444）：终止值必须 ≥ 起始值 — 后果：范围非法命令拒收
- **最大记录数 40000**（warning，rule-0-00444）：>98% 上报 ALM-135602215（≤95% 恢复，阈值可 `MOD CFGTHRESHOLD` 改）；单地址池最多 64 段（SECTIONNUM 0~63） — 后果：超规格/超 64 段新增失败
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
- **IPVERSION 与 POOL 同步**（warning，隐含）：SECTION 的 IPVERSION 应与对应 [POOL](task/UDG/20.15.2/0-00043.md) 的 IPVERSION 一致 — 后果：跨版本段配置异常
