# 配置GTP路径管理IP地址（ADD ECHOIPLIST）

> 命令静态知识见 [ADD ECHOIPLIST](command/UDG/20.15.2/ADD-ECHOIPLIST.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。**本命令是 SET UPGTPPATH 的从属命令**（前置 [0-00051](task/UDG/20.15.2/0-00051.md) SET UPGTPPATH.ECHOLISTSWITCH=ENABLE）。

## 配置方法（配置方法字典）

ADD ECHOIPLIST 是"GTP 路径管理黑白名单成员登记"命令：按"每对端 IP 地址段一条 ADD"逐条下发（**最大 200 条**），决定哪些对端 IP 段纳入名单管理。核心配置空间有 3 个维度：**IP 类型**（IPVERSION 二选一门控）、**地址段**（IPV4/IPV6 配对应 MASK）、**名单性质**（黑/白名单由 SET UPGTPPATH 决定，本命令只填成员）。1 个特性命中（GWFD-010102 路径管理）覆盖 3 条 IPv4 + 1 条 IPv6 范式。

### 配置维度 1：IP 类型（参数 IPVERSION，必选）

IPVERSION 是门控开关，决定后续地址参数走 IPv4 链路还是 IPv6 链路：

| 取值 | 联动必选参数 | 适用场景 |
|---|---|---|
| **IPV4**（默认） | `IPV4ADDR` + `IPV4MASK` | 对端为 IPv4 地址段（绝大多数部署） |
| **IPV6** | `IPV6ADDR` + `IPV6MASK` | 对端为 IPv6 地址段（双栈部署） |

### 配置维度 2：地址段（参数 IPV4ADDR/IPV4MASK 或 IPV6ADDR/IPV6MASK，条件必选）

**IPv4 路径**（IPVERSION=IPV4 时）：

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **IPV4ADDR** | IPv4 地址类型 | 名单中的 IPv4 地址段 |
| **IPV4MASK** | 整数 1~32 | IPv4 掩码长度 |

**IPv6 路径**（IPVERSION=IPV6 时）：

| 参数 | 取值范围 | 作用 |
|---|---|---|
| **IPV6ADDR** | IPv6 地址类型 | 名单中的 IPv6 地址段 |
| **IPV6MASK** | 整数 1~128 | IPv6 掩码长度 |

> IPV4/IPV6 两组参数**互斥**：选 IPV4 时不能配 IPV6ADDR/IPV6MASK，反之亦然。

### 配置维度 3：名单性质（由 SET UPGTPPATH 决定，本命令不体现）

名单性质（黑名单/白名单）由前置 SET UPGTPPATH.ECHOLISTSWITCH 决定，本命令只填成员：

| 名单类型 | 名单内 IP 段 | 名单外 IP 段 |
|---|---|---|
| **白名单**（ECHOIPLSTTYPE=WHITELIST） | 系统**发送** GTP 信令/数据路径 Echo 探测 | 系统**不发送** Echo 探测 |
| **黑名单**（ECHOIPLSTTYPE=BLACKLIST） | 系统**不发送** Echo 探测 | 系统**发送** Echo 探测 |

> 用 `LST UPGTPPATH` 查看名单类型；用 `RMV ECHOIPLIST（★R1.5 atom 待补）` 删除成员。

**典型脚本（010102 多条逐行下发，IPv4/IPv6 混合）**：
```
ADD ECHOIPLIST:IPVERSION=IPV4,IPV4ADDR="192.168.1.10",IPV4MASK=24;
ADD ECHOIPLIST:IPVERSION=IPV4,IPV4ADDR="192.168.10.10",IPV4MASK=24;
ADD ECHOIPLIST:IPVERSION=IPV4,IPV4ADDR="192.168.100.10",IPV4MASK=24;
ADD ECHOIPLIST:IPVERSION=IPV6,IPV6ADDR="fc00::1",IPV6MASK=64;
```

**典型脚本（单条 IPv4 精简）**：
```
ADD ECHOIPLIST: IPVERSION=IPV4, IPV4ADDR="10.36.0.2", IPV4MASK=27;
```

> 命令 notes：适用 NF：SGW-U、PGW-U、UPF；**最大记录数 200**（一条对端 IP 段一条记录）；**执行后立即生效**；配合 SET UPGTPPATH 使用——名单性质（黑/白）由 SET UPGTPPATH 决定，本命令仅填成员；前置 SET UPGTPPATH.ECHOLISTSWITCH=ENABLE。

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：GTP 路径黑白名单成员"——前置 [0-00051](task/UDG/20.15.2/0-00051.md) SET UPGTPPATH（开 ECHOLISTSWITCH + 选名单类型），本命令逐条下发名单成员，配套 RMV ECHOIPLIST（★R1.5 atom 待补） 删除。

## 决策点：IP 地址类型选择（DP 0-00054）

问：ADD ECHOIPLIST 选哪个 IP 类型？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPv4 单地址段 | IPVERSION=IPV4 + IPV4ADDR + IPV4MASK | 对端为 IPv4 地址段（绝大多数部署，010102 默认范式） |
| IPv6 单地址段 | IPVERSION=IPV6 + IPV6ADDR + IPV6MASK | 对端为 IPv6 地址段（双栈部署，010102 演示） |
| 多地址段循环下发 | 按地址段循环 ADD，每条独立 | 对端有多个 IP 段需管理（010102 演示：3 条 IPv4 + 1 条 IPv6） |

> 1 个特性命中（010102）覆盖 IPv4 多条+IPv6 单条范式；累积记录 ≤200；名单类型由前置 SET UPGTPPATH 决定。

## 约束

- **最大记录数 200**（warning，rule-0-00454）：一条对端 IP 段一条记录，累积不超过 200 — 后果：超 200 条新增命令拒收
- **执行后立即生效**（warning，rule-0-00454）：命令执行后立即生效，无需刷新 — 后果：误操作即刻生效
- **IPVERSION 门控联动参数**（critical，rule-0-00454）：IPVERSION=IPV4 时必须配 IPV4ADDR+IPV4MASK；IPVERSION=IPV6 时必须配 IPV6ADDR+IPV6MASK；两组互斥 — 后果：参数配错或缺失命令拒收
- **前置 SET UPGTPPATH.ECHOLISTSWITCH=ENABLE**（critical，rule-0-00454）：本命令必须配合 SET UPGTPPATH 使用，名单性质（黑/白）由 SET UPGTPPATH 决定 — 后果：未开 ECHOLISTSWITCH 时本命令虽可下发但不生效
- **名单内/外行为倒置**（info，隐含）：白名单=名单内探测/名单外不探测；黑名单=名单内不探测/名单外探测（010102 演示黑名单） — 后果：名单类型选错致探测行为相反
- **适用 NF 限定**（warning，隐含）：SGW-U、PGW-U、UPF — 后果：其他 NF 不适用
- **删除配套 RMV ECHOIPLIST（★R1.5 atom 待补）**（info，隐含）：名单成员变更需用 RMV ECHOIPLIST（★R1.5 atom 待补） 删除，本命令只能增 — 后果：误增需 RMV 删除
