# 配置DS域（ADD QOSDIFFERSERV）

> 命令静态知识见 [ADD QOSDIFFERSERV](command/UDG/20.15.2/ADD-QOSDIFFERSERV.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD QOSDIFFERSERV 是 QoS 5 件套的"DS 域定义层"——建一个 Diff-Serv 域（DSNAME），作为后续 SET QOSBA（入口映射）、SET QOSPHB（出口映射）、ADD QOSIFTRUST（接口绑定）三命令的**命名容器**，DSNAME 在链路中传递。本命令的核心配置空间有 2 个维度：**DSNAME 取值来源**（系统预置 vs 自建）、**系统预置保护**（哪些 DSNAME 不可修改）。

### 配置维度 1：DSNAME 命名来源（参数 DSNAME，必选）

| 取值 | 作用 | 配套引用 | 典型场景 |
|---|---|---|---|
| **default** | 系统预置 DS 域（VLAN 到 IP 网络的优先级映射容器） | SET QOSBA / SET QOSPHB / ADD QOSIFTRUST 三命令的 DSNAME 引用 | 010201 QoS 流量管理（主场景两方向均用预置域 default） |
| **5p3d** | 系统预置 DS 域 | — | 不允许修改（只读） |
| **自定义** | 自建 DS 域（避开 5p3d） | 同上 | 特殊业务场景 |

### 配置维度 2：系统预置保护（隐含）

| DSNAME | 是否可建/可改 | 说明 |
|---|---|---|
| `default` | 可建（系统默认存在） | 两方向主场景使用 |
| `5p3d` | **不可修改** | 系统预置，不允许 MOD |
| 其他自定义 | 可建/可改 | 自建域须避开 `5p3d` |

**典型脚本（010201 QoS 两方向均用预置域）**：
```
ADD QOSDIFFERSERV:DSNAME="default";
```

**步骤位置**：在所属 feature 编排中位于"步骤 2：建 DS 域"——前置 [0-00065](task/UDG/20.15.2/0-00065.md) SET SRVCOMMONPARA:TUNNELMARKINGID=ENABLE → 本命令建 DS 域 → SET QOSBA / SET QOSPHB 配映射 → ADD QOSIFTRUST 接口绑定。

## 决策点

（无。本命令仅 DSNAME 一个必选参数，命名按预置/自建即可。）

## 约束

- **最大记录数 8**（critical，rule-0-00282）：DS 域总数上限 8 — 后果：超限 ADD 失败
- **DSNAME=5p3d 不可修改**（critical，rule-0-00282）：系统预置域不允许 MOD — 后果：MOD 命令拒收
- **DSNAME 是链路引用键**（critical，隐含）：DSNAME 是 SET QOSBA / SET QOSPHB / ADD QOSIFTRUST 三命令的引用键，改名导致下游断链 — 后果：下游三命令找不到域失败
- **执行后立即生效**（warning，隐含）：无 REFRESHSRV 缓冲 — 后果：配置错误立即生效
- **自建域避开 `5p3d`**（warning，隐含）：自建 DSNAME 须避开系统预置 `5p3d` — 后果：与预置域重名 ADD 失败
