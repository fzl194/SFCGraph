# 配置DS域（ADD QOSDIFFERSERV）

> 命令静态知识见 [ADD QOSDIFFERSERV](command/UDG/20.15.2/ADD-QOSDIFFERSERV.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD QOSDIFFERSERV 建一个 Diff-Serv 域（DSNAME），作为后续 [SET QOSBA](task/UDG/20.15.2/0-00067.md)（入口）、[SET QOSPHB](task/UDG/20.15.2/0-00069.md)（出口）、[ADD QOSIFTRUST](task/UDG/20.15.2/0-00068.md)（接口绑定）三命令的**命名容器**，DSNAME 在链路中传递。

**配置范式**

| 参数 | 取值 | 来源 | 说明 |
|---|---|---|---|
| DSNAME | default | 本端规划 | DS 域名，VLAN 到 IP 网络的优先级映射容器 |

```
ADD QOSDIFFERSERV:DSNAME="default";
```

主场景两方向均用预置域 `default`。**自建域须避开 `5p3d`**（系统预置，不允许修改）。DSNAME 是下游三 atom 的链路引用键，改名会断链。

执行顺序（010201 QoS 流量管理）：先 [SET SRVCOMMONPARA](task/UDG/20.15.2/0-00065.md) 使能 TunnelMarking → 本命令建 DS 域 → SET QOSBA / SET QOSPHB 配映射 → ADD QOSIFTRUST 接口绑定。

## 约束

- **rule 0-00282**（dependency）最大记录数 8；系统预置两个 DiffServ 域 `default` 与 `5p3d`，其中 `5p3d` 不允许修改；DSNAME 是 SET QOSBA/SET QOSPHB/ADD QOSIFTRUST 的链路引用键，改名导致下游断链；执行后立即生效 — 后果：超限 ADD 失败 / DSNAME 取 5p3d 被 MOD 拒收 / 改名致下游找不到域
