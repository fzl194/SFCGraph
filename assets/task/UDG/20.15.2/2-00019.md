# SA-Basic（GWFD-110101）

> 特性静态知识见 [GWFD-110101](feature/UDG/20.15.2/GWFD-110101.md)。本页讲配置生成实例化时这个特性**怎么配**。本特性是**业务感知族 backbone**（七层协议识别基础），计费/BWM/头增强/重定向/IM管控等 SA 依赖特性的前置。

## 配置概览

SA-Basic（Service Awareness Basic）是业务感知的基本功能，负责对用户报文逐层处理：三四层解析（IP/端口/协议）→ 协议识别（知名端口优先 + 特征字兜底）→ 七层解析（URL/Method/响应码），把"无差别的 IP 报文流"转化为"有业务标签的报文流"供规则匹配引擎消费。

**核心机制**：建立七层协议识别能力（`LOD SIGNATUREDB` 特征字识别 + `LOD PARSERDB` 深度解析 + `ADD WELLKNOWNPORT` 知名端口加速），再配预定义规则匹配业务流（`POLICYTYPE=PCC` 走 PCC 触发链：过滤链 + RULE + USERPROFILE），为业务感知族提供协议解析基础。SA-Basic 是其他高级 SA 特性（计费/BWM/头增强/重定向/IM管控）的基础依赖。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开 SA-Basic License 开关，单命令）：`SET LICENSESWITCH`（LKV3G5SABS01）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **SA 协议识别链**（加载特征库+解析库+知名端口，建立七层协议识别能力，多命令可复用模块）：`LOD SIGNATUREDB`（LOADMODE=LATEST 特征字识别）+ `LOD PARSERDB`（FILENAME 七层解析）+ `ADD WELLKNOWNPORT`（PROTNAME+PORT 知名端口加速，可选）→ [1-00016](task/UDG/20.15.2/1-00016.md)
3. **过滤链**（三四层 + 七层流匹配容器）：`ADD FILTER` + `ADD FLOWFILTER` + `ADD FLTBINDFLOWF`（三四层）+ `ADD L7FILTER` + `ADD PROTBINDFLOWF`（七层，PROTOCOLNAME 引用识别出的协议）→ [1-00009](task/UDG/20.15.2/1-00009.md)
4. **业务规则**（POLICYTYPE=PCC 触发，规则绑流过滤器）：`ADD RULE`（POLICYTYPE=PCC + FLOWFILTERNAME + POLICYNAME=PCCPOLICYGRPNM）→ [1-00011](task/UDG/20.15.2/1-00011.md)
5. **用户模板 + 绑定**（规则绑用户模板，PCC 触发链收尾）：`ADD USERPROFILE` + `ADD RULEBINDING` → [1-00011](task/UDG/20.15.2/1-00011.md)
6. **刷新生效**（立即生效，单命令）：`SET REFRESHSRV`（REFRESHTYPE=USERPROFILE）→ [0-00015](task/UDG/20.15.2/0-00015.md)
   - 可选分支（SA 公共参数）：`SET SRVCOMMONPARA`（SAURLLEN=511/1023 最大 URL 解析长度，REDWITHPREURL 前缀 URL 控制等）→ [0-00065](task/UDG/20.15.2/0-00065.md)

## 决策点

**DP（特性级）：特征库加载模式 + 知名端口 + 规则下发方式 + 过滤匹配层**（影响 SA 协议识别链参数 + 过滤链走法 + 规则绑定走法）。维度按参考信息 MML 命令清单 + 特性知识 §4.1 激活步骤 + §3.1 识别机制 全列。

| 场景 | 特征库加载 | 知名端口 | 规则下发 | 过滤匹配 | 走法 |
|---|---|---|---|---|---|
| **标准部署（特性知识 §5.1 演示）** | LATEST | 配加速（HTTP=80）| 本地预定义 | 三四层 + 七层 | 骨架 + [1-00016](task/UDG/20.15.2/1-00016.md) + [1-00009](task/UDG/20.15.2/1-00009.md) L34+L7 + [1-00011](task/UDG/20.15.2/1-00011.md) |
| 特征库指定版本变体 | **SPECIFIC + VERSION** | 配加速 | 本地预定义 | 三四层 + 七层 | LOD SIGNATUREDB LOADMODE=SPECIFIC+VERSION（DNS 纠错等需特定版本）|
| 不配知名端口变体 | LATEST | **不配** | 本地预定义 | 三四层 + 七层 | 依赖特征库自动识别（性能略低，[1-00016](task/UDG/20.15.2/1-00016.md) WELLKNOWNPORT 可选）|
| 动态规则下发变体 | LATEST | 配加速 | **PCRF/PCF 动态下发** | 由 UNC 生成 PDR | SA-Basic 仅配识别能力，规则由控制面下发（特性知识 §3.2 业务流程）|
| 仅三四层匹配变体 | LATEST | 配加速 | 本地预定义 | **仅三四层** | [1-00009](task/UDG/20.15.2/1-00009.md) 仅 FILTER+FLTBINDFLOWF，无 L7FILTER/PROTBINDFLOWF |

**联动**：
- 特征库加载模式 → [1-00016](task/UDG/20.15.2/1-00016.md) 决策点（LATEST vs SPECIFIC+VERSION）
- 知名端口 → [1-00016](task/UDG/20.15.2/1-00016.md) 决策点（配加速 vs 不配）
- 规则下发方式 → 特性知识 §3.2（本地预定义 ADD RULE+USERPROFILE vs PCRF/PCF 动态 PDR）
- 过滤匹配层 → [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链（L34+L7 vs 仅 L34）
- POLICYTYPE=PCC → [1-00011](task/UDG/20.15.2/1-00011.md) 场景差异（SA-Basic 主场景 PCC，POLICYNAME=PCCPOLICYGRPNM）

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5SABS01` — 未开则 SA-Basic 不生效（特性知识 §1.4 License）。
- **特征库 + 解析库须加载**（critical）：LOD SIGNATUREDB（特征字识别）+ LOD PARSERDB（深度解析）— 未加载则协议识别失败、ALM-81030 协议解析数据库加载失败告警（参考信息告警段 + 特性知识 §1.5 前置条件）。
- **WELLKNOWNPORT 可选加速**（info）：仅加速知名端口协议识别（HTTP=80/RTSP=554/DNS=53/FTP=20,21/SMTP=25/POP3=110/IMAP4=143 等），不配则依赖特征库自动识别（特性知识 §3.1.3 知名端口映射表 + §3.1.3 识别策略）。
- **软参 BIT414/BIT469 控制识别**（info）：BIT414 控制承载层协议选择知名端口还是协议识别组件；BIT469 控制知名端口识别成功后是否还需特征库再识别（参考信息软参段 + 特性知识 §3.1.3 识别策略）。
- **性能影响**（warning）：开启 SA-Basic 后，匹配到规则的报文都需业务感知，系统处理负荷增加、报文转发性能和吞吐量下降；增大 SAURLLEN 会进一步增加系统消耗（特性知识 §1.7 对系统的影响）。
- **CP-UP 配置一致性**（warning）：Rule/UserProfile/URR/CbbID 等须 UNC 与 UDG 合一规划一致；UDG 每 30 分钟扫描，不一致产生 ALM-81054 告警；控制面优先（特性知识 §4.4 约束条件）。
- **URL 长度截断**（warning）：SAURLLEN=511/1023 字节，超过将截断，可能导致业务策略匹配不准确（特性知识 §4.3.1 参数说明）。
- **HTTP Method 匹配限制**（info）：HTTP 支持 GET/POST/CONNECT/PUT/DELETE/OPTIONS/HEAD/TRACE；WAP1.x 仅支持 GET/POST/CONNECT（特性知识 §4.4 约束条件）。
