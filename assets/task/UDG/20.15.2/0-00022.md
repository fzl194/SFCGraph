# 配置扩展过滤器（ADD EXTENDEDFILTER）

> 命令静态知识见 [ADD EXTENDEDFILTER](command/UDG/20.15.2/ADD-EXTENDEDFILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD EXTENDEDFILTER 是七层过滤的"扩展条件定义层"：把七层协议之上的业务维度匹配（URL 路径、Content-Type、客户端 UserAgent、URL 后缀）用一条记录描述，供 ADD FLOWFILTER / ADD FLOWFILTERGRP / ADD RULE(EXTFLTNAME1/2) / SET GYONESHOT 引用。命令核心配置空间有 3 个维度：**过滤条件维度**（URL/CONTENTTYPE/USERAGENT/URLPOSTFIX 四类，至少输入其一）、**后缀匹配模式**（POSTFIXMUST，仅 URLPOSTFIX 配时生效）、**分组匹配**（GROUPEDFLAG，多扩展过滤器聚合时）。同一特性下可建多条不同过滤条件的扩展过滤器，**重复执行可叠加**。

### 配置维度 1：过滤条件维度（四类必选其一至四，可同时配多类）

| 条件 | 参数 | 格式 | 典型场景 |
|---|---|---|---|
| **URL 路径** | `URL` | 字符串 1~119，不区分大小写，支持 `*` 通配 | URL 白名单、HTTP 智能重定向、DNS 纠错 |
| **ContentType** | `CONTENTTYPE` | 字符串 1~31 | MIME 过滤 |
| **客户端类型** | `USERAGENT` | 字符串 1~119，不区分大小写，支持 `*` 通配 | UA 维度重定向（浏览器 vs App） |
| **URL 后缀** | `URLPOSTFIX` | 字符串 1~7 | 文件后缀过滤（.apk/.mp4 等） |

> 4 个过滤条件至少输入其一，否则命令拒收。可同时配多个条件构成 AND 关系。

### 配置维度 2：URL 后缀匹配模式（参数 POSTFIXMUST，条件可选）

| 取值 | 作用 | 前提 |
|---|---|---|
| **DISABLE**（默认） | 仅匹配 URL 后缀包含关系 | URLPOSTFIX 已配 |
| **ENABLE** | 强制"必须为该后缀"匹配 | URLPOSTFIX 已配 |

### 配置维度 3：分组匹配标识（参数 GROUPEDFLAG）

| 取值 | 作用 |
|---|---|
| **DISABLE**（默认） | 单一扩展过滤器独立匹配 |
| **ENABLE** | 多扩展过滤器聚合匹配（同一组内任一匹配即命中） |

**典型脚本（020356 一次重定向 URL 白名单）**：
```
ADD EXTENDEDFILTER:EXTFLTNAME="gy-oneshot-url",URL="www.huawei.com";
```

**典型脚本（110283 DNS 纠错，URL 通配）**：
```
ADD EXTENDEDFILTER:EXTFLTNAME="l7-filter1",URL="www.example.com/*";
```

**典型脚本（110284 HTTP 智能重定向，双维度各一条）**：
```
ADD EXTENDEDFILTER:EXTFLTNAME="filter_http-redirect1",URL="www.huawei.com/*";
ADD EXTENDEDFILTER:EXTFLTNAME="filter_http-redirect2",USERAGENT="Mozilla/*";
```

> 命令 notes：最大记录数 1000；每 filter 最多 20000 条过滤条件；整机最多 20000 条过滤条件；执行后 60s 生效。

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：扩展过滤器定义"——前置无；产出 EXTFLTNAME 被 ADD RULE/ADD FLOWFILTER/SET GYONESHOT 引用。

## 决策点：扩展过滤器过滤条件组合（DP 0-00022）

问：ADD EXTENDEDFILTER 选哪些过滤条件维度？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| URL 维度（默认） | URL（可配通配 `*`） | URL 白名单（020356）、DNS 纠错（110283）、HTTP 智能重定向（110284） |
| UA 维度 | USERAGENT（可配通配 `*`） | HTTP 智能重定向按客户端区分（110284） |
| URL 后缀维度 | URLPOSTFIX + POSTFIXMUST（DISABLE/ENABLE） | 文件后缀过滤场景（命令参数齐全，无特性命中演示） |
| ContentType 维度 | CONTENTTYPE | MIME 过滤（命令参数齐全，无特性命中演示） |

> 选项不互斥：同一特性下可建多条不同过滤条件的扩展过滤器（重复执行可叠加）。

## 约束

- **过滤条件至少输入其一**（critical，rule-0-00333）：URL/CONTENTTYPE/USERAGENT/URLPOSTFIX 四过滤条件至少输入其一 — 后果：四条件均不配时命令拒收
- **最大记录数与生效延时**（warning，rule-0-00333）：最大记录数 1000；每 filter 最多 20000 条过滤条件；整机最多 20000 条过滤条件；执行后 **60s** 生效 — 后果：超规格被拒 / 新建后未等 60s 即生效判定失败
- **EXTFLTNAME 全网唯一**（critical，隐含）：不同过滤器之间 EXTFLTNAME 不能重复 — 后果：重名 ADD 失败
- **POSTFIXMUST 仅 URLPOSTFIX 配时生效**（warning，隐含）：未配 URLPOSTFIX 时 POSTFIXMUST 不允许 ENABLE — 后果：参数误用拒收
- **适用 NF 限定**（warning，隐含）：PGW-U、UPF — 后果：SGW-U 等不适用
