# 配置扩展过滤器（ADD EXTENDEDFILTER）

> 命令静态知识见 [ADD EXTENDEDFILTER](command/UDG/20.15.2/ADD-EXTENDEDFILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是**跨 3 特性共享**的七层过滤条件配置：URL 白名单（020356 计费信息实时提醒）、DNS 纠错（110283）、HTTP 智能重定向（110284）。产出对象 EXTFLTNAME 被 [0-00024](task/UDG/20.15.2/0-00024.md)（SET GYONESHOT）引用。核心规则：**URL / CONTENTTYPE / USERAGENT / URLPOSTFIX 四个过滤条件至少输入其一**，其余可选参数按场景补。

**配置范式（按特性选过滤条件维度）**

| 特性场景 | EXTFLTNAME | 选用的过滤条件 | 典型行 |
|---|---|---|---|
| 020356 计费信息实时提醒（URL 白名单） | `gy-oneshot-url` | URL=`www.huawei.com` | `ADD EXTENDEDFILTER:EXTFLTNAME="gy-oneshot-url",URL="www.huawei.com";` |
| 110283 DNS 纠错 | `l7-filter1` | URL=`www.example.com/*`（通配） | `ADD EXTENDEDFILTER:EXTFLTNAME="l7-filter1",URL="www.example.com/*";` |
| 110284 HTTP 智能重定向（URL 维度） | `filter_http-redirect1` | URL=`www.huawei.com/*` | `ADD EXTENDEDFILTER:EXTFLTNAME="filter_http-redirect1",URL="www.huawei.com/*";` |
| 110284 HTTP 智能重定向（UA 维度） | `filter_http-redirect2` | USERAGENT=`Mozilla/*` | `ADD EXTENDEDFILTER:EXTFLTNAME="filter_http-redirect2",USERAGENT="Mozilla/*";` |

**重复执行可叠加**：同一特性下可建多个不同过滤条件的扩展过滤器（如 110284 的 URL 维度 + UA 维度各一条）。

**未绑冷门参数**（命令真相 optional 且 3 特性数据规划表均未用到）：URLPOSTFIX / POSTFIXMUST / GROUPEDFLAG / CFGDOMAINNAME——按通用配置门槛不绑，需要时查命令 wiki。

## 约束

- **rule-0-00333**（warning）最大记录数 1000（扩展过滤器数）；每 filter 最多 20000 条过滤条件；整机最多 20000 条过滤条件；执行后 60s 生效；URL/CONTENTTYPE/USERAGENT/URLPOSTFIX 四过滤条件至少输入其一 — 后果：超规格被拒 / 四条件均不配时命令拒收
