# 配置TCP保活参数Full模式（SET TCPKEEPALIVECFG）

> 命令静态知识见 [SET TCPKEEPALIVECFG](command/UDG/20.15.2/SET-TCPKEEPALIVECFG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本任务是 GWFD-110921（TWAMP）**Full 模式独有**的第 5 步——配置 TWAMP-Control 连接的 TCP 保活参数。Full 模式走 TCP 控制连接（建立会话前需握手协商），故需 TCP 保活；Light 模式无 TCP 控制连接，不走此命令。三参均有系统初始记录（90/30/9），缺省即用初始值。

**参数取值规划（数据规划表样例）**

| 参数 | 含义 | 取值样例 | 获取方法 / 取值规律 |
|---|---|---|---|
| KEEPALIVETIME | 保活时间 | `90` | 全网规划，整数 30~7200 秒（系统初始记录 90） |
| INTERVAL | 保活间隔 | `30` | 全网规划，整数 10~300 秒（系统初始记录 30） |
| RETRY | 保活重试 | `9` | 全网规划，整数 3~20 次（系统初始记录 9） |

> 三参均 optional，未输入保持当前配置不变（可用 LST TCPKEEPALIVECFG 查询）。特性脚本显式传 90/30/9=初始记录值。

**脚本样例**

```
SET TCPKEEPALIVECFG:KEEPALIVETIME=90,INTERVAL=30,RETRY=9;
```

**操作步骤定位**：Full 模式激活流程中，紧接 ADD TWAMPRESPONDER 之后。生效时机见约束（非立即生效）。

## 约束

- 系统部署后已存在初始记录（KEEPALIVETIME=90/INTERVAL=30/RETRY=9），三参 optional 未输入保持当前配置不变。
- 命令执行后**在 TCP 链路重建时生效**（非立即生效），改动需等待链路重建。
- 取值超范围新增失败。
