# IM类业务无线资源管控（GWFD-020359）

> 特性静态知识见 [GWFD-020359](feature/UDG/20.15.2/GWFD-020359.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

IM 类业务无线资源管控用于无线资源有限的场景。**核心机制：SA 识别 IM 业务（QQ/MSN/微信等协议）→ RULE 经 `POLICYTYPE=REMARK_FPI` 将 IM 业务映射为 DSCP 值（如 AF12）标记业务报文 → 含 DSCP 标记的报文透传到 BSC/RNC，无线侧按 DSCP 综合调度优先级对 IM 报文做无线资源优化**。

三四层过滤条件配置为 ANY（全匹配兜底），IM 业务识别主要靠七层协议（`ADD PROTBINDFLOWF` 绑定 IM 协议名，如 `qq_im`）。**DSCP 标记方式（华为私有，仅 UDG+BSC/RNC 均为华为设备）需 MOD USERPROFILE 禁用 Alias Marking（ALIASMARKFLAG=DISABLE），否则系统不改写报文 ToS 域、DSCP 标记不生效**。依赖 SA-Basic（协议识别基础）+ SA-IM（IM 协议特征）+ PCC基本功能（PCRF 下发 PCC 规则）。

## 配置流程（步骤编排，每步显示含命令；每参数挂证据）

1. **License 前置**（打开本特性 License 开关，单命令）：`SET LICENSESWITCH`（LKV3G5ITSR01）→ [0-00019](task/UDG/20.15.2/0-00019.md)
2. **三四层过滤条件**（ANY 兜底，三四层全匹配；多命令可复用模块）：`ADD FILTER`（L34PROTTYPE=STRING, L34PROTOCOL=ANY）+ `ADD FLOWFILTER`（FLOWFILTERNAME）+ `ADD FLTBINDFLOWF`（FILTER→FLOWFILTER）+ `SET REFRESHSRV`（REFRESHTYPE=ALL）→ [1-00009](task/UDG/20.15.2/1-00009.md)
   - activation 数据规划：三四层 = ANY（IM 业务识别靠七层协议，三四层仅作兜底全匹配）
3. **SA 协议识别**（可选，加载特征库识别 IM 协议；如已加载可跳过）：`LOD SIGNATUREDB`（LOADMODE=LATEST）+ `LOD PARSERDB`（LOADMODE=LATEST）→ [1-00016](task/UDG/20.15.2/1-00016.md)
4. **七层过滤条件**（IM 协议绑定，业务识别核心；多命令可复用模块）：`ADD PROTBINDFLOWF`（PROTOCOLNAME=qq_im 等 IM 协议名）+ `SET REFRESHSRV`（REFRESHTYPE=ALL）→ [1-00009](task/UDG/20.15.2/1-00009.md)
   - activation 数据规划：`PROTOCOLNAME` 按实际 IM 业务规划（QQ=qq_im，其他 IM 类业务改对应协议名）
5. **业务规则**（POLICYTYPE=REMARK_FPI 标记 DSCP，规则绑定收尾）：`ADD RULE`（POLICYTYPE=REMARK_FPI + REMARKFPISEL=REMARK + REMARKCFGTYPE=CLASS + REMARKCLASS=AF12 + FILTERINGMODE=FLOWFILTER + FLOWFILTERNAME + PRIORITY）+ `ADD USERPROFILE`（USERPROFILENAME）+ `ADD RULEBINDING`（RULE→USERPROFILE）→ [1-00011](task/UDG/20.15.2/1-00011.md)
6. **禁用 Alias Marking**（DSCP 方式必需，单命令）：`MOD USERPROFILE`（ALIASMARKFLAG=DISABLE）→ [0-00296](task/UDG/20.15.2/0-00296.md)
   - DSCP 方式必需：ALIASMARKFLAG=ENABLE 时系统不改写报文 ToS 域，DSCP 标记失效

> **参数证据链**（R1.5，配置流程脚注）：以上命令+参数均来自 activation §表1~表4 + §操作步骤 + §任务示例脚本。

> - ADD PROTBINDFLOWF（FLOWFILTERNAME + PROTOCOLNAME=qq_im）：activation §表2 + §任务示例脚本（原样，"此处根据qq_im识别IM类业务QQ，其他IM类业务时PROTOCOLNAME需要根据实际业务规划"）

## 决策点

**DP（特性级）：标记方式 + REMARK 分类值 + IM 业务协议**（影响 RULE 参数 + 过滤链协议名 + 是否禁用 Alias Marking）。维度按 activation §表2/§表3 + §操作步骤7c 全列。

| 场景 | 标记方式 | REMARKCFGTYPE | REMARKCLASS | IM 协议名 | Alias Marking | 走法 |
|---|---|---|---|---|---|---|
| **DSCP 方式·QQ（主流，activation 演示）** | DSCP 标记 | CLASS | **AF12** | qq_im | **DISABLE** | 骨架 + 直接 atom（MOD USERPROFILE）|
| DSCP 方式·其他 IM 业务 | DSCP 标记 | CLASS | 按 IM 业务规划（AF11/AF13/AF21/...）| 对应 IM 协议名（msn_im/wechat_im 等）| DISABLE | 同上 |
| 非 DSCP 方式（FPI 方式，理论可选）| FPI 标记 | 见命令 wiki | 见命令 wiki | 同上 | ENABLE（不禁用）| 同上（无 MOD USERPROFILE 步骤）|

**联动**：
- 标记方式 → 是否执行步骤6（MOD USERPROFILE）：DSCP 方式需 ALIASMARKFLAG=DISABLE（critical，不禁用则 DSCP 标记不生效）
- REMARKCLASS → [1-00011](task/UDG/20.15.2/1-00011.md) 规则绑定（ADD RULE 参数，REMARK_FPI 变体专属）
- IM 协议名 → [1-00009](task/UDG/20.15.2/1-00009.md) 过滤链 PROTBINDFLOWF.PROTOCOLNAME（依赖 [1-00016](task/UDG/20.15.2/1-00016.md) SA 协议识别链加载的特征库已含该协议）

## 约束（特性级 rule）

- **License 前置**（critical）：SET LICENSESWITCH 开启 `LKV3G5ITSR01`（控制项 82200BLD） — 未开则本特性不生效。
- **强依赖 SA-Basic**（critical）：GWFD-110101（License LKV3G5SABS01，控制项 82209749）—— SA 协议解析基础，IM 业务识别依赖其加载的特征库与解析库，未开则无法识别 IM 协议。
- **强依赖 SA-IM**（critical）：控制项 82209763（SA-IM）—— IM 协议特征依赖，未开则 IM 类业务识别失效。
- **强依赖 PCC 基本功能**（critical）：GWFD-020351（控制项 82209825）—— PCRF 下发 IM 类业务的 PCC 规则给 UDG，未开则无法接收规则。
- **DSCP 标记方式须 MOD USERPROFILE ALIASMARKFLAG=DISABLE**（critical）：ALIASMARKFLAG=ENABLE 时系统不改写报文 ToS 域，DSCP 标记不生效 — DSCP 方式下必须禁用 Alias Marking。
- **POLICYTYPE=REMARK_FPI 触发条件**（critical）：RULE.POLICYTYPE=REMARK_FPI + REMARKFPISEL=REMARK + REMARKCFGTYPE=CLASS + REMARKCLASS — 指错或漏配则 DSCP 标记不触发。
- **DSCP 方式为华为私有**（warning）：仅适用于 UDG 和 BSC/RNC 均为华为设备的网络 — 混网则 DSCP 透传链断。
- **仅优化下行无线信道资源**（warning）：功能先识别 IM 协议再传 BSC/RNC 控制下行，对上行无线信道资源不作改变。
- **仅支持 2/3G 用户接入**（warning）：IM 类业务无线资源管控当前仅支持 2/3G 接入 — 4G/5G 用户不生效。
- **特征库需定期升级**（warning）：IM 协议类型不断更新增加，为保证网络优化效果需定期升级 UDG 协议特征库。
- **三四层兜底 + 七层识别**（info）：activation 数据规划明确"IM类业务识别一般使用协议识别，三四层过滤条件配置为 any"。
