# 设置全局下行数据长时间缓存个数（SET GLBDLLTBUFFER）

> 命令静态知识见 [SET GLBDLLTBUFFER](command/UDG/20.15.2/SET-GLBDLLTBUFFER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET GLBDLLTBUFFER 用于配置 eDRX 模式下整机级下行报文长时间缓存的最大缓存个数与存储方式（仅在不配 APN 场景生效；APN 级配置由 SET APNDLLTBUFFER 覆盖，**优先级 APN>全局**）。**MAXBUFFERNUM 与 BUFFERMODE 必选**（整机级全局配置，**最大记录数 1**）。核心配置空间有 2 个维度：**最大缓存个数**（MAXBUFFERNUM）、**存储方式**（BUFFERMODE）。3 个特性命中（110601 NB-IoT eDRX / 110641 eMTC eDRX / 110661 5G eDRX）跨三特性取值一致。

### 配置维度 1：最大缓存个数（参数 MAXBUFFERNUM，必选）

| 取值 | 作用 |
|---|---|
| **整数 1~64（包）**（系统初始值 1） | 全局下行报文长时间缓存最大缓存个数；当 SMF/N4 接口下发 DL Buffering Packet Count 时，取下发值与本参数**最小值**作为系统实际缓存个数 |

### 配置维度 2：存储方式（参数 BUFFERMODE，必选）

| 取值 | 作用 |
|---|---|
| **BUFF_MODE_RING** | 环形存储——丢弃最先存储的报文，缓存当前报文（110601/110641/110661 范式） |
| **BUFF_MODE_LINK** | 链式存储——丢弃当前收到的报文，保留最先存储的报文 |

**典型脚本（GWFD-110601/110641/110661 三特性共用）**：
```
SET GLBDLLTBUFFER:MAXBUFFERNUM=16,BUFFERMODE=BUFF_MODE_RING;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 N：设全局下行长时间缓存个数"——配合 [0-00172](task/UDG/20.15.2/0-00172.md) SET GLBDLBUFTIME（全局下行缓存时长）+ [0-00174](task/UDG/20.15.2/0-00174.md) SET APNDLBUFTIME（APN 级缓存时长，覆盖全局）使用。

## 决策点：存储方式（DP 0-00173）

问：缓存报文达到规格时采用哪种存储方式？

| 选项 | BUFFERMODE | 适用场景 |
|---|---|---|
| **环形存储**（110601/110641/110661） | BUFF_MODE_RING | 优先保留最新报文；典型 eDRX 场景 |
| **链式存储** | BUFF_MODE_LINK | 优先保留最先报文 |

> 3 个特性命中（110601/110641/110661）均使用 BUFF_MODE_RING；MAXBUFFERNUM 与 DDN ACK 携带缓存个数取最小值。

## 约束

- **MAXBUFFERNUM 与 BUFFERMODE 联合必选**（critical，rule-0-00173）：MAXBUFFERNUM 与 BUFFERMODE 是必选参数 — 后果：缺一命令拒收
- **整机最大记录数 1**（warning，rule-0-00173）：本命令整机级全局配置，最大记录数 1 — 后果：超限 SET 失败
- **执行后对新数据流生效**（warning，rule-0-00173）：本命令对新数据流生效 — 后果：已存在的下行报文按旧规则缓存
- **APN 级覆盖优先**（warning，rule-0-00173）：APN 级 SET APNDLLTBUFFER 优先级 APN>全局 — 后果：配 APN 后全局设置仅对该 APN 外用户生效
- **DDN ACK 协同**（warning，rule-0-00173）：MAXBUFFERNUM 与 DDN ACK 携带缓存个数取最小值作为系统实际缓存个数 — 后果：未与 DDN ACK 协同时错取缓存个数
- **缓存个数与速率限制**（warning，rule-0-00173）：缓存个数超过 Serving PLMN Downlink Rate Limit 会触发丢包 — 后果：须按无线侧能力合理取值
