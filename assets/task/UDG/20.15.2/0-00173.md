# 设置全局下行数据长时间缓存个数（SET GLBDLLTBUFFER）

> 命令静态知识见 [SET GLBDLLTBUFFER](command/UDG/20.15.2/SET-GLBDLLTBUFFER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令配置 **eDRX 模式下整机级下行数据长时间缓存的最大个数与存储方式**（仅在不配 APN 场景生效；APN 级配置由 [0-00175](task/UDG/20.15.2/0-00175.md) SET APNDLLTBUFFER 覆盖，优先级 APN>全局）。整机级全局参数，最大记录数为 1。NB-IoT/eMTC/5G 三特性共用同一组取值。

**配置范式（两参数同条命令，跨三特性取值一致）**

| 参数 | 取值样例 | 获取方法 | 取值范围 | 说明 |
|---|---|---|---|---|
| MAXBUFFERNUM | `16` | 本端规划 | 1~64 包 | 最大缓存个数；与 DDN Ack 携带缓存个数取最小值作为系统实际缓存个数；未修改时为初始值 1 |
| BUFFERMODE | `BUFF_MODE_RING` | 本端规划 | 枚举 | 存储方式；三特性取值一致 |

> MAXBUFFERNUM 协同规则：当 SMF/N4 接口下发 DL Buffering Packet Count 时，取下发值与本参数的**最小值**；未下发则用本参数。设置缓存个数须同时考虑无线侧能力，过大超过 Serving PLMN Downlink Rate Limit 会触发丢包。

**配置步骤**

1. 全网规划确认最大缓存个数（按无线侧 Downlink Rate Limit 能力，过大可能丢包）。
2. 执行全局长时间缓存配置：
   ```
   SET GLBDLLTBUFFER: MAXBUFFERNUM=16, BUFFERMODE=BUFF_MODE_RING;
   ```
3. 如需 APN 级差异化，改用 [0-00175](task/UDG/20.15.2/0-00175.md) 配置（优先级 APN>全局）。

**示例脚本**：`SET GLBDLLTBUFFER: MAXBUFFERNUM=16, BUFFERMODE=BUFF_MODE_RING;`

## 约束

- **rule-0-00346**（warning）最大记录数 1（整机级），超限 SET 失败；命令对新数据流生效；仅在不配 APN 场景生效，APN 级配置（[0-00175](task/UDG/20.15.2/0-00175.md)）优先级 APN>全局；缓存个数超过 Serving PLMN Downlink Rate Limit 会触发丢包，须同时考虑无线侧能力；系统初始 MAXBUFFERNUM=1、BUFFERMODE=BUFF_MODE_RING；MAXBUFFERNUM 与 DDN ACK 携带缓存个数取最小值 — 后果：超限 SET 失败 / 在 APN 场景外执行不生效 / 超速率限制丢包 / 未与 DDN ACK 协同错取缓存个数
