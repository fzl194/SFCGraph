# 控制面地址分配方式（WSFD-010504）

> 特性静态知识见 [WSFD-010504](feature/UNC/20.15.2/WSFD-010504.md+ [特性知识](业务图谱体系/APN业务域/feature-knowledge/WSFD-010504-控制面地址分配方式.md（★业务图谱未建，归 R1.5 后续）。本页讲配置生成实例化时这个特性**怎么配**——本特性是 WSFD-010502 地址分配主特性的"控制面视角补充"，仅当 SMF 对接的 UPF 不支持地址分配时启用。

## 配置概览

核心对象链：**SET APNADDRESSATTR（IPV4ALLOCTYPE=Local 强制控制面分配）** + **ADDRPOOL（POOLTYPE=Local）** + **SECTION（地址段）** + **ADDRPOOLGRP/POOLBINDGRP**（按需）；由 SMF 控制面本地地址池分配地址。

本特性与 [WSFD-010502](feature/UNC/20.15.2/WSFD-010502.md) 的 Local 子方式**配置结构相同**（共享 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 地址池体系），差异仅在**强制走控制面分配路径**（UPF 不分配时，由 SMF 本端分配）。本特性额外涉及 `SET IPALLOCRULE`（地址分配规则全局开关）/ `SET IPALLOCBYLOCGLBSW`（基于位置区分配的全局开关）/ `ADD CONFLICTIP`（冲突地址管理）—— 这些是控制面地址分配专属。

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（L3VPN + VPN 实例 + APN + 使能地址分配）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR`（`IPV4ALLOCTYPE=Local`） → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **SMF 本地地址池**（控制面本地地址池 + 段 + 池组）：`ADD ADDRPOOL`（POOLTYPE=Local） + `ADD SECTION` + `ADD ADDRPOOLGRP` + `ADD POOLBINDGRP` → [UNC 1-00023](task/UNC/20.15.2/1-00023.md)
3. **地址分配规则**（按需，全局开关）：`SET IPALLOCRULE`（UNC pre-build 阶段 atom 未建，归 P1 后续补建）
   > **★ R1.5 待补 atom**：`SET IPALLOCRULE` 命令真实存在 UNC OM，evidence = [WSFD-010504-参考信息_77813716.md](evidence/UNC/20.15.2/WSFD-010504/WSFD-010504-参考信息_77813716.md) 命令清单 + ALM-100297 告警；配置生成暂不产出。
4. **基于位置区的地址分配开关**（按需）：`SET IPALLOCBYLOCGLBSW`（UNC pre-build 阶段 atom 未建，归 P1 后续补建）
   > **★ R1.5 待补 atom**：evidence 同上。
5. **冲突地址管理**（按需，处理地址冲突）：`ADD CONFLICTIP`（UNC pre-build 阶段 atom 未建，归 P1 后续补建）
   > **★ R1.5 待补 atom**：evidence 同上。
6. **下行路由发布**（让骨干网可达分配的地址）：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFINTERFACE` + `ADD OSPFIMPORTROUTE` → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)
   - 关键参数：PROCID, AREAID, IFNAME（UNC 真实命令名 **OSPFINTERFACE**——v1.0 写的 `OSPFNETWORK` 是错的，UNC 无此命令）

> **★ 与 [WSFD-010502](feature/UNC/20.15.2/WSFD-010502.md) 的关系**：本特性是 010502 Local 子方式的**控制面强制版**—— 步骤 1-2 与 010502 的 Local 走法**结构相同**（共享 compound），差异在 `SET APNADDRESSATTR:IPV4ALLOCTYPE=Local` 是本特性的强制值 + 步骤 3-5 是控制面专属规则（010502 不含）。

## 决策点（特性级：地址分配控制面开关）

| 选项 × IP 类型 × 控制面规则 | 走法（复用 compound）| 联动 / 规则 |
|---|---|---|
| **Local × IPv4 × 默认全局规则** | 骨架 [1-00024](task/UNC/20.15.2/1-00024.md) + 地址池 [1-00023](task/UNC/20.15.2/1-00023.md) + 路由 [1-00029](task/UNC/20.15.2/1-00029.md) | 无附加规则 |
| **Local × IPv4 × 基于位置区分配** | 骨架 1-00024 + 地址池 1-00023 + `SET IPALLOCBYLOCGLBSW`（开） + 路由 1-00029 | 需配置位置区 → 地址池映射 |
| **Local × IPv4 × 自定义全局分配规则** | 骨架 1-00024 + 地址池 1-00023 + `SET IPALLOCRULE`（按规则） + 路由 1-00029 | 需配置分配规则参数 |
| **Local × IPv4 × 含冲突地址管理** | 骨架 1-00024 + 地址池 1-00023 + `ADD CONFLICTIP` + 路由 1-00029 | 需配置冲突地址段 |

> **走法列**：本特性 4 个选项**共享骨架 + 地址池 + 路由 3 骨架 compound**；差异在控制面专属规则（步骤 3-5）的附加命令。

## 约束（特性级 rule）

- **IPV4ALLOCTYPE=Local 强制**（critical）：本特性场景下 `SET APNADDRESSATTR` 的 `IPV4ALLOCTYPE` 必须为 `Local`（控制面本地分配），否则走 UPF 分配路径——与本特性"UPF 不支持分配"的前提冲突。
- **UPF 不支持 IP 地址分配**（critical）：本特性触发前提是 SMF 对接的 UPF **不支持** IP 地址分配功能——否则应直接走 [WSFD-010502](feature/UNC/20.15.2/WSFD-010502.md) 的 Local 子方式（UPF 分配更高效）。
- **地址段必配**（critical）：本地地址池建好后**必须** `ADD SECTION` 添加地址段，否则用户激活失败。
- **POOLNAME 命名约束**（warning）：字符串 1~79，由 `-/_`/数字/字母/`.` 组成，不能以 `.` 开头 — 后果：违规命名新增失败。
- **最大记录数 80000**（warning）：每 ADDRPOOL 最多 80000 记录 — 后果：超规格新增失败。
- **基于位置区分配需位置数据**（warning）：`SET IPALLOCBYLOCGLBSW` 开启后需保证位置区→地址池映射已配置 — 后果：位置区未映射时降级为默认分配。
- **冲突地址管理需配合告警**（info）：`ADD CONFLICTIP` 配置后产生冲突时上报 `ALM-100297`（地址池组使用率超阈值）。
- **首次发布版本**（info）：本特性首次发布版本为 UNC **20.8.0**。
