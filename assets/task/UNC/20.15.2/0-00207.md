# SET USRPROFCHARGE 原子任务
> 命令静态知识见 [SET USRPROFCHARGE](command/UNC/20.15.2/SET-USRPROFCHARGE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令承担 UserProfile 计费的"多职责收口"：计费方式开关（ONLINESW/OFFLINESW/CONVERGEDSW）、离线模板（PGW/SGW/GGSN/SMF OFCTEMPLATE）、在线 DCC 模板、融合 CCT 模板、费率切换组（TARIFFGRPNAME）、计费属性（CCNAME）、上报模式（RGAPPLIED）、QBC 开关。一条命令可同时携带多个职责参数。

## 配置方法

| 职责 | 参数 | 配置方法 | 取值样例 |
|---|---|---|---|
| 计费方式-离线 | OFFLINESW | 本端规划 | `ENABLE` |
| 计费方式-在线 | ONLINESW | 本端规划 | `ENABLE` |
| 计费方式-融合 | CONVERGEDSW | 本端规划 | `ENABLE` |
| 业务申请上报模式 | RGAPPLIED | 全网规划（仅融合场景） | `ONLINERGONLY` / `OFFLINERGONLY` / `DEFAULT` |
| 离线模板-PGW | PGWOFCTEMPLATE | 已配置数据中获取（引用 ADD OFCTEMPLATE） | `offlinecharge-test` |
| 离线模板-SGW | SGWOFCTEMPLATE | 已配置数据中获取（引用 ADD OFCTEMPLATE） | `offlinecharge-test` |
| 离线模板-GGSN | GGSNOFCTEMPLATE | 已配置数据中获取（引用 ADD OFCTEMPLATE） | — |
| 离线模板-SMF | SMFOFCTEMPLATE | 已配置数据中获取（引用 ADD OFCTEMPLATE） | — |
| 在线模板 | DCCTEMPLATE | 已配置数据中获取（引用 ADD DCCTEMPLATE） | `dcc-test` |
| 融合模板 | CCTEMPLATE | 已配置数据中获取（引用 ADD CCT） | `cct_test` |
| 费率切换组 | TARIFFGRPNAME | 已配置数据中获取（引用 ADD TARIFFGROUP） | `testtarif` / `tar_nb` |
| 计费属性 | CCNAME | 已配置数据中获取（引用 ADD CHARGECHAR） | `testchgcha` |
| QBC 开关 | QBCSW | 全网规划（条件可选） | — |
| USRPROFNAME（键） | USRPROFNAME | 已配置数据中获取（ADD USERPROFILE 创建） | `up-test` |

典型脚本（按职责拆）：
```
SET USRPROFCHARGE:USRPROFNAME="up-test",OFFLINESW=ENABLE;
SET USRPROFCHARGE:USRPROFNAME="up-test",ONLINESW=ENABLE;
SET USRPROFCHARGE:USRPROFNAME="up-test",CONVERGEDSW=ENABLE,RGAPPLIED=ONLINERGONLY;
SET USRPROFCHARGE:USRPROFNAME="up-test",PGWOFCTEMPLATE="offlinecharge-test",SGWOFCTEMPLATE="offlinecharge-test";
SET USRPROFCHARGE:USRPROFNAME="up-test",DCCTEMPLATE="dcc-test";
SET USRPROFCHARGE:USRPROFNAME="up-test",CCTEMPLATE="cct_test";
SET USRPROFCHARGE:USRPROFNAME="up-test",TARIFFGRPNAME="testtarif";
SET USRPROFCHARGE:USRPROFNAME="up-test",CCNAME="testchgcha";
```

## 决策点

### 计费方式选择（在线/离线/融合）
- 问什么：该 UserProfile 采用哪种计费方式。
- 取值由本端规划决定，但必须与 APN 级（SET APNCHARGECTRL）、用户属性级（SET CHARGECTRL）、CC 属性级（ADD/MOD CHARGEMETHOD）规划的计费方式一致。
- 继承语义：ONLINESW/OFFLINESW/CONVERGEDSW 取 `INHERIT` 时继承基于 SET APNCHARGECTRL 的配置。
- 一致性硬约束：同一用户不同属性下计费方式必须一致，否则计费出错（如本地用户既配离线又配在线）。

### RGAPPLIED（融合场景上报模式）
- 问什么：融合计费时业务申请（RG）上报到在线侧、离线侧还是默认。
- `DEFAULT` → URRGROUP 下同时配离线和在线 URR 时优先选在线 URR。
- `ONLINERGONLY` / `OFFLINERGONLY` → URRGROUP 下建议同时配离线和在线 URR。
- 仅在融合计费（CONVERGEDSW=ENABLE）语境生效。

### 模板绑定（OFC/DCC/CCT）取值来源
- 问什么：各模板名引用哪个已建模板实例。
- 全部为"已配置数据中获取"——引用 ADD OFCTEMPLATE / ADD DCCTEMPLATE / ADD CCT 产物，非自由命名。
- 网元匹配：按部署 NF 选填对应 OFC 模板（PGW-C 填 PGWOFCTEMPLATE，SGW-C 填 SGWOFCTEMPLATE，GGSN 填 GGSNOFCTEMPLATE，SMF 填 SMFOFCTEMPLATE）。

### 费率切换组 / CC 单绑定语义
- 问什么：每个 UP 只能绑一个费率切换组、一个 CC。
- 改绑必须再次调用本命令覆盖（无单独"解绑"命令，靠覆盖修改）。
- TARIFFGRPNAME 引用 ADD TARIFFGROUP 产物；CCNAME 引用 ADD CHARGECHAR 产物。

## 约束
- 生效时机：执行后只对**新激活用户**生效。`severity=medium`
- 容量上限：最大记录数 105000。`severity=low`
- 全局模板禁绑：全局在线模板（DccTemplate 名为 `global`）不允许绑在 UserProfile 下。`severity=medium`
- 模板名格式：不支持包含空格，但支持仅输入一个空格（表示删除该 UP 与对应模板的绑定）。`severity=medium`
- 一致性：与 SET APNCHARGECTRL / SET CHARGECTRL / CHARGEMETHOD 规划的计费方式必须一致，否则计费出错。`severity=high`
- 单绑定：TARIFFGRPNAME、CCNAME 每 UP 单绑，改绑用本命令覆盖。`severity=medium`
- 前置依赖：USRPROFNAME（[ADD USERPROFILE](task/UNC/20.15.2/0-00094.md)）；模板/组/CC 引用对应 ADD 命令产物。`severity=high`
