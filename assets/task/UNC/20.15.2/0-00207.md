# 配置UserProfile计费方式与模板绑定（SET USRPROFCHARGE）

> 命令静态知识见 [SET USRPROFCHARGE](command/UNC/20.15.2/SET-USRPROFCHARGE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET USRPROFCHARGE 是 UserProfile 计费配置的"多职责收口"命令。14 个参数按职责分为 5 组：计费方式开关（3）+ RG 上报模式（1）+ OFC/DCC/CCT 模板绑定（5）+ 费率切换组/CC（2）+ QBC（1）+ 键（1）。一条命令可同时携带多个职责参数，典型骨架为 `ADD USERPROFILE` → **SET USRPROFCHARGE**（按职责拆多次调用）→ `ADD USRPROFGROUP`。命令最大记录数 105000（限 UP 实例数），仅对新激活用户生效。

### 配置维度 1：计费方式开关（参数 ONLINESW/OFFLINESW/CONVERGEDSW）

| 取值 | 作用 | 联动参数 | 典型场景 |
|---|---|---|---|
| `OFFLINESW=ENABLE` | 使能离线计费 | `PGWOFCTEMPLATE`/`SGWOFCTEMPLATE`/`GGSNOFCTEMPLATE`/`SMFOFCTEMPLATE`（按 NF 选填）；`SET APNCHARGECTRL` | WSFD-011201 离线计费、WSFD-109001 在线（同时使能 ONLINE+OFFLINE） |
| `ONLINESW=ENABLE` | 使能在线计费 | `DCCTEMPLATE`；`SET APNCHARGECTRL` | WSFD-109001 Gy_Diameter 在线计费 |
| `CONVERGEDSW=ENABLE` | 使能融合计费 | `CCTEMPLATE`；`RGAPPLIED`（条件必选）；`QBCSW`（条件可选） | WSFD-011206 融合计费 |
| `INHERIT` | 继承 [SET APNCHARGECTRL](task/UNC/20.15.2/0-00149.md) 的配置 | 与 APN 级一致 | 未独立规划的 UP 粒度 |

> 三开关取 `INHERIT` 时继承基于 SET APNCHARGECTRL 的配置；与 [SET APNCHARGECTRL](task/UNC/20.15.2/0-00149.md) / [SET CHARGECTRL](task/UNC/20.15.2/0-00157.md) / [ADD CHARGEMETHOD](task/UNC/20.15.2/0-00015.md) / [MOD CHARGEMETHOD](task/UNC/20.15.2/0-00116.md) 规划的计费方式必须一致（同用户不同属性下计费方式必须一致）。

### 配置维度 2：RG 上报模式与 QBC（参数 RGAPPLIED/QBCSW）

| 取值 | 作用 | 联动参数 | 典型场景 |
|---|---|---|---|
| `RGAPPLIED=DEFAULT` | URRGROUP 下同时配离线和在线 URR 时优先选在线 URR | `CONVERGEDSW=ENABLE` | 融合计费默认 |
| `RGAPPLIED=ONLINERGONLY` | 仅上报在线 RG；URRGROUP 下建议同时配离线和在线 URR | `CONVERGEDSW=ENABLE` | 融合计费 WSFD-011206（典型取 ONLINERGONLY） |
| `RGAPPLIED=OFFLINERGONLY` | 仅上报离线 RG；URRGROUP 下建议同时配离线和在线 URR | `CONVERGEDSW=ENABLE` | 融合计费 |
| `QBCSW=ENABLE/DISABLE` | QBC 计费场景开关（条件可选，需 CONVERGEDSW=ENABLE） | `CONVERGEDSW=ENABLE` | QBC 计费场景 |

> RGAPPLIED/QBCSW 仅在 `CONVERGEDSW=ENABLE` 语境生效；条件可选参数。

### 配置维度 3：模板绑定（参数 PGW/SGW/GGSN/SMF OFCTEMPLATE + DCCTEMPLATE + CCTEMPLATE）

| 参数 | 取值 | 作用 | 适用 NF | 联动 |
|---|---|---|---|---|
| `PGWOFCTEMPLATE` | OFC 模板名（1~63 字符） | UP 绑 PGW 离线计费模板 | PGW-C | `ADD OFCTEMPLATE` 产物 |
| `SGWOFCTEMPLATE` | OFC 模板名（1~63 字符） | UP 绑 SGW 离线计费模板 | SGW-C | `ADD OFCTEMPLATE` 产物 |
| `GGSNOFCTEMPLATE` | OFC 模板名（1~63 字符） | UP 绑 GGSN 离线计费模板 | GGSN | `ADD OFCTEMPLATE` 产物 |
| `SMFOFCTEMPLATE` | OFC 模板名（1~63 字符） | UP 绑 SMF 离线计费模板 | SMF | `ADD OFCTEMPLATE` 产物 |
| `DCCTEMPLATE` | DCC 模板名（1~63 字符） | UP 绑 DCC 在线计费模板 | 通用 | `ADD DCCTEMPLATE` 产物（**全局在线模板 `global` 禁绑**） |
| `CCTEMPLATE` | CCT 模板名（1~63 字符） | UP 绑融合计费模板 | 通用 | `ADD CCT` 产物 |

> 模板名不支持空格，但支持**仅输入一个空格**——表示删除该 UP 与对应模板的绑定（解绑语义）。

### 配置维度 4：费率切换组与计费属性（参数 TARIFFGRPNAME/CCNAME）

| 参数 | 取值 | 作用 | 联动 |
|---|---|---|---|
| `TARIFFGRPNAME` | 费率切换组名（1~31 字符） | UP 绑费率切换组（**每 UP 单绑**，改绑用本命令覆盖） | `ADD TARIFFGROUP` 产物 |
| `CCNAME` | 计费属性名称（1~31 字符） | UP 绑计费属性 CC（**每 UP 单绑**，改绑用本命令覆盖） | `ADD CHARGECHAR` 产物 |

> 每个 UP 只能绑 1 个费率切换组、1 个 CC；无单独"解绑"命令，靠本命令覆盖修改。

### 配置维度 5：键参数（参数 USRPROFNAME）

| 参数 | 取值 | 作用 | 联动 |
|---|---|---|---|
| `USRPROFNAME`（必选） | UserProfile 实例名（1~63 字符） | 本命令操作的 UP 键 | `ADD USERPROFILE` 产物 |

**典型脚本**（按职责拆）：
```
SET USRPROFCHARGE:USRPROFNAME="up-test",OFFLINESW=ENABLE;
SET USRPROFCHARGE:USRPROFNAME="up-test",ONLINESW=ENABLE;
SET USRPROFCHARGE:USRPROFNAME="up-test",CONVERGEDSW=ENABLE,RGAPPLIED=ONLINERGONLY;
SET USRPROFCHARGE:USRPROFNAME="up-test",PGWOFCTEMPLATE="offlinecharge-test",SGWOFCTEMPLATE="offlinecharge-test";
SET USRPROFCHARGE:USRPROFNAME="up-test",DCCTEMPLATE="dcc-test";
SET USRPROFCHARGE:USRPROFNAME="up-test",CCTEMPLATE="cct_test";
SET USRPROFCHARGE:USRPROFNAME="up-test",TARIFFGRPNAME="testtarif";
SET USRPROFCHARGE:USRPROFNAME="up-test",CCNAME="testchgcha";
```

**步骤位置**：UserProfile 创建后的紧接步骤。`ADD USERPROFILE` → **SET USRPROFCHARGE**（按职责多次调用）→ `ADD USRPROFGROUP` → `ADD UPBINDUPG`。在计费方式规划上，本命令 UP 粒度优先级低于 [SET APNCHARGECTRL](task/UNC/20.15.2/0-00149.md) APN 粒度、CC 级 [ADD/MOD CHARGEMETHOD](task/UNC/20.15.2/0-00015.md) 显式规划；UP 取 `INHERIT` 时继承 APN 级。

## 决策点

### 决策点：UP 粒度计费方式选择（在线/离线/融合/继承）（DP 0-04005）

问：该 UserProfile 采用哪种计费方式？与上下层（APN/USER/CC 级）如何一致？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 离线计费 | `OFFLINESW=ENABLE` + `PGWOFCTEMPLATE`/`SGWOFCTEMPLATE`（按 NF） | WSFD-011201 离线计费场景 |
| 在线计费 | `ONLINESW=ENABLE` + `DCCTEMPLATE` | WSFD-109001 Gy_Diameter 在线计费场景 |
| 融合计费 | `CONVERGEDSW=ENABLE` + `CCTEMPLATE` + `RGAPPLIED` | WSFD-011206 融合计费场景 |
| 继承 APN 级 | `OFFLINESW/ONLINESW/CONVERGEDSW=INHERIT` | UP 不独立规划，继承 [SET APNCHARGECTRL](task/UNC/20.15.2/0-00149.md) |

> **一致性硬约束**：同一用户不同属性下计费方式必须一致——UP 级（SET USRPROFCHARGE）与 APN 级（[SET APNCHARGECTRL](task/UNC/20.15.2/0-00149.md)）、用户属性级（[SET CHARGECTRL](task/UNC/20.15.2/0-00157.md)）、CC 属性级（[ADD/MOD CHARGEMETHOD](task/UNC/20.15.2/0-00015.md)）规划的计费方式须一致，否则用户计费出错。

### 决策点：模板/组/CC 绑定取值来源（DP 0-04006）

问：OFC/DCC/CCT/TARIFF/CC 各模板名引用哪个已建模板实例？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 引用已建 OFCTEMPLATE（按 NF 选） | `PGWOFCTEMPLATE`/`SGWOFCTEMPLATE`/`GGSNOFCTEMPLATE`/`SMFOFCTEMPLATE`="已建 OFC 名" | 离线计费场景 |
| 引用已建 DCCTEMPLATE | `DCCTEMPLATE`="已建 DCC 名"（**禁 `global`**） | 在线计费场景 |
| 引用已建 CCT | `CCTEMPLATE`="已建 CCT 名" | 融合计费场景 |
| 引用已建 TARIFFGROUP | `TARIFFGRPNAME`="已建 TARIFF 组名" | 费率切换场景 |
| 引用已建 CHARGECHAR | `CCNAME`="已建 CC 名" | 计费属性绑定场景 |

> 全部为"已配置数据中获取"（个别样本存在本端规划取名，但引用对象仍须已建）；按部署 NF 选填对应 OFC 模板（PGW-C → PGWOFCTEMPLATE，SGW-C → SGWOFCTEMPLATE，GGSN → GGSNOFCTEMPLATE，SMF → SMFOFCTEMPLATE）。

## 约束

- **只对新激活用户生效**（critical，rule-0-04011）：本命令执行后**只对新激活用户生效**；已激活用户不受影响（区别于 SET URRGRPBINDING 立即生效） — 后果：现网存量用户不感知新配置，配置变更需用户重激活
- **容量上限与全局模板禁绑**（critical，rule-0-04012）：本命令最大记录数为 105000；全局在线模板（DCC 模板名 `global`）**不允许绑在 UserProfile 下** — 后果：超出容量或绑 global 致命令失败
- **模板名格式与解绑语义**（warning，rule-0-04013）：模板名不支持包含空格，但**支持仅输入一个空格**（表示删除该 UP 与对应模板的绑定）；TARIFFGRPNAME/CCNAME 每 UP 单绑，改绑用本命令覆盖（无单独"解绑"命令） — 后果：单空格语义模糊，需明确"删除绑定"意图时显式输入
- **UP/APN/CC 级计费方式一致性**（critical，rule-0-04014）：UP 级（SET USRPROFCHARGE）必须与 [SET APNCHARGECTRL](task/UNC/20.15.2/0-00149.md) APN 级 / [SET CHARGECTRL](task/UNC/20.15.2/0-00157.md) 用户属性级 / [ADD/MOD CHARGEMETHOD](task/UNC/20.15.2/0-00015.md) CC 属性级规划的计费方式一致 — 后果：同用户不同属性下计费方式不一致导致计费出错
- **前置依赖：USRPROFNAME + 模板/组/CC 须先建**（critical，rule-0-04015）：USRPROFNAME 引用 [ADD USERPROFILE](task/UNC/20.15.2/0-00094.md) 产物；模板/组/CC 引用对应 ADD 命令产物（`ADD OFCTEMPLATE`/`ADD DCCTEMPLATE`/`ADD CCT`/`ADD TARIFFGROUP`/`ADD CHARGECHAR`） — 后果：引用对象不存在致绑定无效
