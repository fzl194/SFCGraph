# MOD RGTRIGGER 原子任务
> 命令静态知识见 [MOD RGTRIGGER](command/UNC/20.15.2/MOD-RGTRIGGER.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：PGW-C、SMF。本命令按融合计费模板（CCTMPLTNAME）定位已存在的 RG 级 trigger 集合，修改各类 trigger（QoS/位置/PLMN/配额阈值等）的上报动作及生效原则，是 RG 级 trigger 上报功能调测的核心修改入口。

## 配置方法
按融合计费模板名称定位，按规划逐 trigger 类型修改上报动作。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTMPLTNAME | 融合计费模板名称（必选，1~63，不区分大小写） | 按规划模板名 |
| QOSCHG | QoS 更新 trigger 动作 | `NONREPORT`（不上报） |
| ULCHG / SRVNDCHG / PRACHG / PSDATAOFFCHG / UETZCHG / PLMNCHG / RATCHG / TAICHG / CGISAICHG / RAICHG / SERVPLMNRTCTCHG / APNRATECTRLCHG | 各类事件/位置/速率 trigger 动作 | `NONREPORT` |
| TIMELIMIT / VOLUMELIMIT / EVENTLIMIT / QUOTATHRESHOLD | 时间/流量/事件/配额阈值 trigger 动作 | `NONREPORT` |
| VT | 配额有效时长 trigger 动作 | `NONREPORT` |
| QHT | 配额保持时长 trigger 动作 | `NONREPORT` |

调测 RG 级 trigger 上报（WSFD-011206）典型流程：先 `LST RGTRIGGER` 查本地是否配置对应 trigger 类型与生效原则，未配置或错误时 `MOD RGTRIGGER` 按规划修正，再回到计费验证步骤核对流量一致性。

## 决策点

### Trigger 类型上报动作（上报 / 不上报）
每个 trigger 参数（QOSCHG、ULCHG、PLMNCHG、QUOTATHRESHOLD …）取上报动作，常用 `NONREPORT`（事件发生不生成容器、不上报 CHF）。是否上报取决于业务对计费粒度的要求，取值随规划分叉；本命令不暴露具体上报取值（证据中仅见 NONREPORT）。

## 约束
- **只对新激活用户生效**：修改后不立即影响已激活用户，需重新激活用户后核对计费流量。
- **CCTMPLTNAME 存在性前置**：融合计费模板需已 ADD，且 RG 级 trigger 已通过 ADD RGTRIGGER 添加；MOD 仅修改动作/原则，可用 `LST RGTRIGGER` 校验当前配置。
- **CCTMPLTNAME 不区分大小写**：长度 1~63。
- **与 PDU 级 trigger 维度区分**：RG 级（本命令）作用于 Rating Group 维度，PDU 会话级用 MOD PDUTRIGGER，二者不可混用。
