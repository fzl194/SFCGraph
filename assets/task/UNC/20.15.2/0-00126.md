# 修改RG级Trigger参数（MOD RGTRIGGER）

> 命令静态知识见 [MOD RGTRIGGER](command/UNC/20.15.2/MOD-RGTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按融合计费模板名称定位，按规划逐 trigger 类型修改上报动作。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 定位 | CCTMPLTNAME | 融合计费模板名（1~63，不区分大小写） | 无 | 指定目标 CCT | 所有场景必选 |
| QoS/位置 | QOSCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | QoS 更新 trigger | — |
| QoS/位置 | ULCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 用户位置更新 trigger | RAT=UTRAN 不支持 |
| QoS/位置 | SRVNDCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 服务节点更新 trigger | — |
| QoS/位置 | PRACHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 区域用户位置上报更新 | — |
| 状态更新 | PSDATAOFFCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | PS 数据关闭状态更新 | — |
| 状态更新 | UETZCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 用户时区更新 | — |
| 状态更新 | PLMNCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | PLMN 更新 | — |
| 状态更新 | RATCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | RAT 更新 | — |
| 阈值 | TIMELIMIT | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 时间阈值 trigger | — |
| 阈值 | VOLUMELIMIT | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 流量阈值 trigger | — |
| 阈值 | EVENTLIMIT | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 事件阈值 trigger | 当前版本暂不支持 |
| 阈值 | QUOTATHRESHOLD | `NONREPORT` / `IMMEDIATE` | 无 | 配额阈值 trigger | — |
| 阈值 | VT | `NONREPORT` / `IMMEDIATE` | 无 | 配额有效时长 trigger | — |
| 阈值 | QHT | `NONREPORT` / `IMMEDIATE` | 无 | 配额保持时长 trigger | — |
| 速率控制 | SERVPLMNRTCTCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 服务 PLMN 速率控制更新 | — |

| 位置标识 | TAICHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 跟踪区标识更新 | — |
| 位置标识 | CGISAICHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | CGISAI 更新 | — |
| 位置标识 | RAICHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | RAI 更新 | — |

上报方式说明：

| 上报方式 | 行为 |
|---|---|
| NONREPORT | 不生成容器，不上报 CHF |
| IMMEDIATE | 立即触发 Charging Data Request Update，携带所有已生成用量容器 |
| DEFERRED | 生成用量容器但不立即发消息，在下一条 Update 中携带 |

典型流程：先 `LST RGTRIGGER` 查本地是否配置对应 trigger 类型与生效原则，未配置或错误时 `MOD RGTRIGGER` 按规划修正，再回到计费验证步骤核对流量一致性。

## 决策点

### 决策点：Trigger 类型上报动作（上报 / 不上报）（DP 0-01284）

每个 trigger 参数（QOSCHG、ULCHG、PLMNCHG、QUOTATHRESHOLD …）取上报动作，常用 `NONREPORT`（事件发生不生成容器、不上报 CHF）。是否上报取决于业务对计费粒度的要求，取值随规划分叉；本命令不暴露具体上报取值（证据中仅见 NONREPORT）。

### 决策点：Session 级 vs RG 级 Trigger 选择（DP 0-01285）

| 选项 | 作用域 | 适用用户 |
|---|---|---|
| PDU 会话级（MOD PDUTRIGGER） | Session 维度 | Session 用户 |
| RG 级（MOD RGTRIGGER，本命令） | Rating Group 维度 | RG 用户 |

订阅存在冲突时，以 Session 级 Trigger 的规则优先（Session 级立即上报 > RG 级延迟上报时，按 Session 级立即上报处理）。具体走哪一级由用户计费模式决定。

## 约束

- **仅对新激活用户生效**（critical，rule-0-01662）：仅对新激活用户生效：执行后只对新激活用户生效，不追溯已激活用户，需重新激活用户后核对计费流量

- **前置存在性**（critical，rule-0-01663）：前置存在性：融合计费模板需已 ADD，且 RG 级 trigger 已通过 ADD RGTRIGGER 添加；MOD 仅修改动作/原则，可用 LST RGTRIGGER 校验当前配置

- **CCTMPLTNAME 规格**（warning，rule-0-01664）：CCTMPLTNAME 规格：长度 1~63，不区分大小写

- **与 PDU 级 Trigger 维度区分**（warning，rule-0-01665）：与 PDU 级 Trigger 维度区分：RG 级（本命令）作用于 Rating Group 维度，PDU 会话级用 MOD PDUTRIGGER，二者不可混用

- **生效原则校验**（info，rule-0-01666）：生效原则校验：需校验本地配置的 Session 级或 RG 级对应 Trigger 类型的生效原则是否正确，若错误用本命令重配
