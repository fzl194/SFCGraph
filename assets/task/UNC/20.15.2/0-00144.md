# 删除用户模板和规则的绑定关系（RMV RULEBINDING）（RMV RULEBINDING）

> 命令静态知识见 [RMV RULEBINDING](command/UNC/20.15.2/RMV-RULEBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

删除对象是 `RULEBINDING`（用户模板-规则绑定关系）记录。按用户模板名 + 规则名 + 策略类型组合定位删除；**不输入规则名 = 删除该用户模板下全部绑定关系**（高危）。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 用户模板定位 | USERPROFILENAME | 1~63，不支持空格，不区分大小写，本端规划 | — | 定位用户模板，必选 | 必选 |
| 规则定位 | RULENAME | 1~63，不支持空格，不区分大小写，本端规划 | — | 缩小到单条规则绑定 | 精确删单条 |
| 策略类型缩范围 | POLICYTYPE | 枚举（PCC/BWM/HEADEN/WEBPROXY/IPREDIR/SMARTREDIRECT/REMARK_FPI/SRV_TRIGGER/TRAFFICCLASS/LBO/FIREWALL/ULCL/ADC/QOS/FUP/NON_SPECIFIC_TYPE），本端规划 | — | 同名规则多策略类型时进一步锁定 | 同名多策略类型规则 |

典型脚本（删某用户模板下单条规则绑定）：
```
RMV RULEBINDING: USERPROFILENAME=up-default, RULENAME=app_sta_rule;
```

全删某用户模板下绑定（高危，生产慎用）：
```
RMV RULEBINDING: USERPROFILENAME=up-default;
```

步骤位置：调测 ADC（WSFD-109102）/ IPv6 SA（WSFD-109103）标识不一致时，按"先 RMV RULEBINDING 解绑 →（必要时 RMV RULE 删规则）→ ADD RULE 重建规则 → ADD RULEBINDING 重建绑定"顺序执行。

## 决策点

### 决策点：删除范围选择（DP 0-03027）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 精确删单条绑定 | USERPROFILENAME + RULENAME（+ POLICYTYPE） | 调测场景修正某条规则绑定标识 |
| 模板下全删 | 仅 USERPROFILENAME，不输入 RULENAME | 清理整个用户模板的绑定关系（高危，生产慎用） |

### 决策点：是否连带重建 RULE（DP 0-03028）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅解绑重建 | RMV RULEBINDING → ADD RULEBINDING | 规则本身正确，仅绑定标识不一致（ADC/IPv6 SA 调测常见） |
| 规则+绑定一起重建 | RMV RULEBINDING → RMV RULE → ADD RULE → ADD RULEBINDING | PCF 预定义规则 APP_STA/APP_STO 标识在 UPF/SMF/PCF 间不一致，规则定义本身错误 |

## 约束

- **高危批量禁用**（critical，rule-0-03066）：高危批量禁用：本命令属高危命令，不允许批量删除操作；如确需批量，须将软参 BYTE976 设为 169，否则裸批量会触发高危拦截

- **模板下全删风险**（warning，rule-0-03067）：模板下全删风险：仅给 USERPROFILENAME 不输入 RULENAME 会删该模板下全部绑定——使用该模板的用户可能因无法命中规则导致业务受损，删前必须确认模板未被大量在用会话引用并联系华为支持协助

- **用户模板必填**（warning，rule-0-03068）：用户模板必填：USERPROFILENAME 为必选参数不可省略；RULENAME/POLICYTYPE 为可选缩小范围参数

- **生效时机**（info，rule-0-03069）：生效时机：命令执行后立即生效，无延迟窗口，删后该模板下用户立即受影响

- **名称格式**（info，rule-0-03070）：名称格式：USERPROFILENAME/RULENAME 长度 1～63、不支持空格、不区分大小写
