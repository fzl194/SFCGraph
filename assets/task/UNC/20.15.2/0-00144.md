# RMV RULEBINDING 原子任务
> 命令静态知识见 [RMV RULEBINDING](command/UNC/20.15.2/RMV-RULEBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

**高危命令**。删除对象是 `RULEBINDING`（用户模板-规则绑定关系）记录。`USERPROFILENAME` 必填定位用户模板，`RULENAME/POLICYTYPE` 可选缩范围；**不输入规则名 = 删该用户模板下全部绑定关系**（高危）。

## 配置方法
按用户模板名 + 规则名 + 策略类型组合定位删除。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| USERPROFILENAME | 用户模板名称（local_planned，required） | `up-default` |
| RULENAME | 规则名称（local_planned，optional） | `app_sta_rule` |
| POLICYTYPE | 策略类型（local_planned，optional，枚举） | `Charging` |

典型脚本（删某用户模板下单条规则绑定）：
```
RMV RULEBINDING: USERPROFILENAME=up-default, RULENAME=app_sta_rule;
```

删某用户模板下全部绑定（高危，生产慎用）：
```
RMV RULEBINDING: USERPROFILENAME=up-default;
```

## 决策点（若有）

### 删除范围（单条绑定 vs 模板下全删）
- 问什么：删一条规则绑定，还是删该用户模板下所有绑定关系。
- 精确删：给 `USERPROFILENAME + RULENAME`（+ `POLICYTYPE`）锁定单条绑定。
- 模板下全删：仅给 `USERPROFILENAME`、不输入 `RULENAME` = 删除该用户模板下全部绑定——高危，使用该模板的用户可能因无法命中规则导致业务受损。

### 是否连带重建 RULE（规则标识不一致场景）
- 调测 ADC（WSFD-109102）/ IPv6 SA（WSFD-109103）时，若仅绑定标识不一致：先 RMV RULEBINDING，再 ADD RULEBINDING 重建绑定即可（规则本身正确）。
- 若规则本身标识也错（PCF 预定义规则 APP_STA/APP_STO 标识在 UPF/SMF/PCF 间不一致）：须**先 RMV RULEBINDING，再 RMV RULE**，然后 ADD RULE + ADD RULEBINDING 重建。
- 即：判断"只解绑重建"还是"规则+绑定一起重建"取决于规则定义本身是否正确。

## 约束（若有）
- 高危：删除绑定可能改变业务匹配结果、影响策略获取、导致用户业务受损，删前必须确认模板未被大量在用会话引用，并联系华为支持协助。`severity=high`
- 禁止批量：本命令不允许批量删除操作；如确需批量，须将软参 BYTE976 设为 169。`severity=high`
- 模板下全删风险：仅给 USERPROFILENAME 不输入 RULENAME 会删该模板下全部绑定——高危，谨慎使用。`severity=high`
- 用户模板必填：USERPROFILENAME 为 required，不可省略。`severity=medium`
- 生效时机：立即生效。`severity=medium`
- 名称格式：USERPROFILENAME/RULENAME 长度 1～63、不支持空格、不区分大小写。`severity=medium`
