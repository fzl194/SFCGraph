# 设置APN鉴权属性（SET APNAUTHATTR）

> 命令静态知识见 [SET APNAUTHATTR](command/UNC/20.15.2/SET-APNAUTHATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令配置指定 APN 的鉴权属性（用于 RADIUS 鉴权或本地鉴权场景）。配置空间主要由**接入模式**这一核心维度决定（接入模式决定后续鉴权模式/密码/AAA 处理等参数的可选范围）。

### 配置维度1：接入模式（参数 ACCESSMODE）

| 取值 | 作用 | 配套后续参数 | 典型场景 |
|---|---|---|---|
| TRANS_NON_AUTH（透明不鉴权，初始默认） | 透明接入，不鉴权 | 不需 RADIUS | 默认场景；不需鉴权的 APN |
| TRANS_AUTH | 透明鉴权 | 公共用户名/密码（COMMONUSERNAME/COMMONUSERPASS）或 PCO | 需 RADIUS 鉴权 |
| NON_TRANS | 不透明接入（需鉴权） | AUTHMODE + PASSWORD + 可选 APNEAP | RADIUS 二次鉴权；EAP 鉴权 |
| LOC_AUTH | 本地鉴权（不去 RADIUS） | 公共用户名/密码；地址分配不能是 RADIUS | 本地鉴权场景 |

### 配置维度2：鉴权模式（参数 AUTHMODE，仅 NON_TRANS 有效）

| 取值 | 作用 | 配套参数 |
|---|---|---|
| PCO（初始默认） | 用 PCO 中携带的用户名/密码 | — |
| APN | 用 APN 作用户名 | PASSWORD（必填） |
| MSISDN | 用 MSISDN 作用户名 | PASSWORD（必填）；可选 USRNMTYPE |

### 配置维度3：AAA 无响应/拒绝处理（参数 AAANORSPCTRL / AAAREJECTCTRL）

| 参数 | 取值 | 作用 |
|---|---|---|
| AAANORSPCTRL | DEACTIVE/CONTINUE | AAA 无响应时去活/继续（CONTINUE 时可配 HOLDINGTIME/ADJUSTRANGE/NORSPBYPASSULCL） |
| AAAREJECTCTRL | DEACTIVE/CONTINUE | AAA 拒绝时去活/继续（CONTINUE 时可配 REJBYPASSULCL） |

### 配置维度4：UDM/2G3G4G 鉴权联动（参数 UDMSUBAUTHSW / GULAUTHSW，仅 TRANS_AUTH/NON_TRANS/LOC_AUTH 显示）

| 参数 | 取值 | 作用 |
|---|---|---|
| UDMSUBAUTHSW | DISABLE/ENABLE | 是否使用 UDM 签约数据参与 AAA 鉴权（ENABLE 时可配 USEUDMAAAIP/UDMSUBNOSECSW） |
| GULAUTHSW | DISABLE/ENABLE | 2/3/4G 场景下是否鉴权 |

### 配置维度5：COA/DM 开关（参数 COASWITCH / DISCONNECT）

| 参数 | 取值 | 作用 |
|---|---|---|
| COASWITCH | DISABLE/ENABLE | COA 功能（未使能时收到 COA Request 直接回 NAK） |
| DISCONNECT | DISABLE/ENABLE（初始 ENABLE） | DM/Disconnect 消息处理开关 |

> 每次 `ADD APN` 时系统自动为本命令增加一条初始记录（ACCESSMODE=TRANS_NON_AUTH 等缺省值）。

**步骤位置**：在所属 feature 编排中，于 `ADD APN`（建实例）之后、RADIUS 对接配置（`ADD APNRDSSVRGRP` 等）配合执行。

## 决策点

**接入模式选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 透明不鉴权 | ACCESSMODE=TRANS_NON_AUTH | 默认场景（初始值） |
| 透明鉴权 | ACCESSMODE=TRANS_AUTH, COMMONUSERNAME/COMMONUSERPASS | 需 RADIUS 鉴权 |
| 不透明（PCO） | ACCESSMODE=NON_TRANS, AUTHMODE=PCO | RADIUS 用 PCO 用户名密码 |
| 不透明（APN/MSISDN） | ACCESSMODE=NON_TRANS, AUTHMODE=APN/MSISDN, PASSWORD | RADIUS 用 APN 或 MSISDN |
| 本地鉴权 | ACCESSMODE=LOC_AUTH, COMMONUSERNAME/COMMONUSERPASS | 本地鉴权（无 RADIUS） |

## 约束

- **只对新激活用户生效**（info）：命令执行后只对新激活用户生效，已在线用户不受影响 — 后果：需用户重激活
- **配置须与网络规划一致**（critical）：必须和网络规划一致，否则用户鉴权/激活失败 — 后果：用户激活失败
- **NON_TRANS+APN/MSISDN 首次须配 PASSWORD**（critical）：AUTHMODE=APN/MSISDN 首次设置时必须配 PASSWORD，否则用默认值导致鉴权失败 — 后果：用户鉴权失败
- **LOC_AUTH/TRANS_NON_AUTH 不能用 RADIUS 分配地址**（critical）：LOC_AUTH/TRANS_NON_AUTH 地址分配模式不能为 RADIUS 分配 — 后果：配置冲突
- **LOC_AUTH/TRANS_NON_AUTH 的 IPv6 IID 不能是 RADIUS**（critical）：LOC_AUTH/TRANS_NON_AUTH 时 ADDRESSATTR 的 IPv6 Interface Identifier 生成方式不能是 RADIUS — 后果：配置冲突
- **AAANORSPCTRL=CONTINUE+ACCTTOAUTH=ENABLE 可能导致激活失败**（warning）：AAANORSPCTRL=CONTINUE 且 RDSSVRGRP 中 ACCTTOAUTH=ENABLE 时，鉴权服务器不回复会导致激活失败 — 后果：用户激活失败
- **2B2C DNN 限制**（warning）：2B2C 专用 DNN 不支持 NON_TRANS 的 PCO 方式；不支持 LOC_AUTH — 后果：配置无效
- **密码确认字段必须一致**（warning）：CFMCOMMUSERPASS 与 COMMONUSERPASS、CFMPASSWORD 与 PASSWORD 必须相同 — 后果：配置失败
