# ADD GLBOFCTEMPLATE 原子任务
> 命令静态知识见 [ADD GLBOFCTEMPLATE](command/UNC/20.15.2/ADD-GLBOFCTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
GLOBALFLAG 决定模板形态——`GLOBAL`（全局兜底）与 `CHARGE_CHARACT`（按 CC 值匹配）两种范式参数集不同，需分别建条目。整机=1 条 GLOBAL + 至多 16 条 CHARGE_CHARACT。

| 形态 | 必填参数 | 配置方法 |
|---|---|---|
| GLOBAL（全局兜底） | GLOBALFLAG=GLOBAL，PGWTEMPLATE，SGWTEMPLATE | 模板名取自已配置的离线计费模板 |
| CHARGE_CHARACT（按 CC 值匹配） | GLOBALFLAG=CHARGE_CHARACT，CCVALUE，CCMASK，CCPRIORITY，PGWTEMPLATE，SGWTEMPLATE | CCVALUE/CCMASK 本端规划，模板名取自已配置数据 |

典型脚本：
```
// Charge Characteristic 类型（按 CC 值精确匹配）
ADD GLBOFCTEMPLATE:GLOBALFLAG=CHARGE_CHARACT,CCVALUE="0x3",CCMASK="0x3",CCPRIORITY=1,PGWTEMPLATE="offlinecharge-test",SGWTEMPLATE="offlinecharge-test";
// 全局兜底
ADD GLBOFCTEMPLATE:GLOBALFLAG=GLOBAL,PGWTEMPLATE="offlinecharge-test",SGWTEMPLATE="offlinecharge-test";
```

## 决策点

### GLOBALFLAG 形态选择
按是否需要按 Charge Characteristic 值区分计费模板决定：
- 全局统一一套离线计费模板 → 仅配 `GLOBAL`（1 条）。
- 不同 CC 值走不同模板 → 配 `CHARGE_CHARACT`（按需多条），通常另配 1 条 `GLOBAL` 作兜底。

### CCVALUE / CCMASK / CCPRIORITY 的来源
自动比对显示这三个参数"本端规划"，而 PGWTEMPLATE/SGWTEMPLATE"已配置数据中获取"——口径分叉：
- CCVALUE / CCMASK：本端规划，十六进制（`0x` 前缀，长度 1~6），由计费策略定义。
- CCPRIORITY：本端规划，整数 0~65535，多条 CHARGE_CHARACT 间用优先级消歧匹配冲突。
- PGWTEMPLATE / SGWTEMPLATE（/ GGSNTEMPLATE）：引用已通过 `ADD OFCTEMPLATE` 配置存在的模板名。

## 约束
- 必填强约束：配置全局离线计费模板时，必须配置离线模板（PGWTEMPLATE/SGWTEMPLATE 至少其一非空引用真实模板）。`severity=high` · 后果：模板绑定失败。
- 前置依赖：PGWTEMPLATE/SGWTEMPLATE/GGSNTEMPLATE 必须已在离线计费模板配置中存在。`severity=high` · 后果：引用失败。
- 规格上限：最大记录数 17（整机至多 16 条 CHARGE_CHARACT + 1 条 GLOBAL）。`severity=high` · 后果：超出无法新增。
- 唯一性：CCVALUE + CCMASK 组合在 CHARGE_CHARACT 记录间不重复（由 CCPRIORITY 消歧匹配冲突，但同值多条无意义）。`severity=medium`
- 生效时机：执行后立即生效。`severity=low`
