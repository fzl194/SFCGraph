# 配置全局离线计费模板（ADD GLBOFCTEMPLATE）

> 命令静态知识见 [ADD GLBOFCTEMPLATE](command/UNC/20.15.2/ADD-GLBOFCTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD GLBOFCTEMPLATE 按 GLOBALFLAG 两种形态参数集不同，整机 = 1 条 GLOBAL + 至多 16 条 CHARGE_CHARACT。3 个核心维度：**绑定形态**（GLOBALFLAG 必选，决定后续参数集）、**CC 匹配三件套**（CHARGE_CHARACT 时条件必选）、**离线模板名**（引用前置 OFCTEMPLATE）。

### 配置维度 1：绑定形态（参数 GLOBALFLAG，必选）

| 取值 | 作用 | 联动条件必选参数 | 适用场景 |
|---|---|---|---|
| `GLOBAL` | 全局兜底——未配 CC 级模板的用户回退到此 | PGWTEMPLATE/SGWTEMPLATE/GGSNTEMPLATE（至少其一） | 整机统一一套离线计费模板 |
| `CHARGE_CHARACT` | 按 CC 值精确匹配——按用户计费属性差异化 | CCVALUE + PGWTEMPLATE/SGWTEMPLATE/GGSNTEMPLATE（+可选 CCMASK/CCPRIORITY） | 不同 CC 用户走不同离线模板 |

> UNC 获取离线配置的优先级由高到低：User Profile > APN > Charge Characteristic > Global。高级别未绑定时逐级回退，全未绑定取系统缺省值。

### 配置维度 2：CC 匹配三件套（GLOBALFLAG=CHARGE_CHARACT 时条件必选/可选）

| 参数 | 取值范围 | 必选性 | 含义 |
|---|---|---|---|
| `CCVALUE` | 十六进制 0x0000~0xFFFF，长度 1~6 | CHARGE_CHARACT 时必选 | Charge Characteristic 值 |
| `CCMASK` | 十六进制 0x0000~0xFFFF，长度 1~6 | CHARGE_CHARACT 时可选 | CC 掩码；全匹配或部分 bit 匹配 |
| `CCPRIORITY` | 整数 0~65535 | 配 CCMASK 时必选 | CC 优先级，多条 CHARGE_CHARACT 间用优先级消歧匹配冲突 |

### 配置维度 3：离线模板名（PGWTEMPLATE/SGWTEMPLATE/GGSNTEMPLATE，条件可选）

| 参数 | 取值范围 | 来源 | 含义 |
|---|---|---|---|
| `PGWTEMPLATE` | 字符串 1~63，不支持空格 | 已配置数据（ADD OFCTEMPLATE） | PGW 形态离线计费模板名 |
| `SGWTEMPLATE` | 字符串 1~63，不支持空格 | 已配置数据（ADD OFCTEMPLATE） | SGW 形态离线计费模板名 |
| `GGSNTEMPLATE` | 字符串 1~63，不支持空格 | 已配置数据（ADD OFCTEMPLATE） | GGSN 形态离线计费模板名 |

> 必须配置离线模板（PGWTEMPLATE/SGWTEMPLATE/GGSNTEMPLATE 至少其一非空引用真实模板）。对应 OFCTEMPLATE 中 THRESHINHERIT=INHERIT 时继承下一级 SET OFCTHRESHOLD 配置。

**典型脚本**：
```
// Charge Characteristic 类型（按 CC 值精确匹配）
ADD GLBOFCTEMPLATE:GLOBALFLAG=CHARGE_CHARACT,CCVALUE="0x3",CCMASK="0x3",CCPRIORITY=1,PGWTEMPLATE="offlinecharge-test",SGWTEMPLATE="offlinecharge-test";
// 全局兜底
ADD GLBOFCTEMPLATE:GLOBALFLAG=GLOBAL,PGWTEMPLATE="offlinecharge-test",SGWTEMPLATE="offlinecharge-test";
```

**步骤位置**：在离线计费编排中位于"全局模板绑定"——前置 [ADD OFCTEMPLATE](command/UNC/20.15.2/ADD-OFCTEMPLATE.md)（模板实体）之后，被 User Profile/APN 级绑定作为兜底层引用。

## 决策点：GLOBALFLAG 形态选择（DP 0-02045）

问：是否需要按 Charge Characteristic 值区分离线计费模板？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅 GLOBAL（1 条） | PGWTEMPLATE/SGWTEMPLATE/GGSNTEMPLATE | 全局统一一套离线计费模板 |
| CHARGE_CHARACT（多条）+ GLOBAL 兜底（1 条） | CCVALUE + CCMASK + CCPRIORITY + 模板名 | 不同 CC 值走不同模板，另配 1 条 GLOBAL 兜底 |

### CCVALUE / CCMASK / CCPRIORITY 来源（DP 0-02046）

问：CC 三件套参数从哪来？

| 选项 | 来源 | 含义 |
|---|---|---|
| CCVALUE / CCMASK | 本端规划（计费策略定义） | 十六进制 0x 前缀，长度 1~6 |
| CCPRIORITY | 本端规划 | 整数 0~65535；多条 CHARGE_CHARACT 间消歧 |
| PGWTEMPLATE / SGWTEMPLATE / GGSNTEMPLATE | 已配置数据中获取 | 引用已通过 ADD OFCTEMPLATE 配置存在的模板名 |

> 三件套"本端规划" vs 模板名"已配置数据中获取"——口径分叉，需分别取值。

## 约束

- rule-0-02111 · severity=critical · 必填强约束：配置全局离线计费模板时，必须配置离线模板（PGWTEMPLATE/SGWTEMPLATE/GGSNTEMPLATE 至少其一非空引用真实模板），否则模板绑定失败
- rule-0-02112 · severity=critical · 前置依赖：PGWTEMPLATE/SGWTEMPLATE/GGSNTEMPLATE 必须已在离线计费模板配置（ADD OFCTEMPLATE）中存在，否则引用失败
- rule-0-02113 · severity=high · 规格上限：最大记录数 17（整机至多 16 条 CHARGE_CHARACT + 1 条 GLOBAL），超出无法新增
- rule-0-02114 · severity=warning · 唯一性：CCVALUE + CCMASK 组合在 CHARGE_CHARACT 记录间不重复（同值多条无意义，由 CCPRIORITY 消歧）
- rule-0-02115 · severity=info · 生效时机：执行后立即生效
