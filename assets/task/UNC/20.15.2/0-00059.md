# ADD PCRFGROUP 原子任务
> 命令静态知识见 [ADD PCRFGROUP](command/UNC/20.15.2/ADD-PCRFGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令创建 PCRF 组容器，决定组内 PCRF 的工作模式与故障倒换行为。建组后由 [ADD PCRFBINDGRP](task/UNC/20.15.2/0-00058.md) 把具体 PCRF 加入，再由 [ADD PCRFGRPBNDAPN](task/UNC/20.15.2/0-00060.md) 绑定到 APN。LOADBALANCEMODE 是组行为的核心开关，直接决定下游 PCRFBINDGRP 是否需要配 WEIGHT。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PCRFGRPNAME | PCRF 组名称，本端规划，1~128 字符 | `pcrf_group_1` |
| LOADBALANCEMODE | 负荷分担模式，全网规划，默认 `MASTER_SLAVE` | `LOAD_BALANCE` / `MASTER_SLAVE` |
| FAILOVERSW | 宕机备份开关（Gx Failover），本端规划，默认 DISABLE | `ENABLE` / `DISABLE` |

典型脚本：
```
ADD PCRFGROUP:PCRFGRPNAME="pcrf_group_1",LOADBALANCEMODE=LOAD_BALANCE;
ADD PCRFGROUP:PCRFGRPNAME="pcrf_group_2",LOADBALANCEMODE=MASTER_SLAVE;
ADD PCRFGROUP:PCRFGRPNAME="pcrf.group.1",LOADBALANCEMODE=MASTER_SLAVE,FAILOVERSW=ENABLE;
```

## 决策点

### PCRF 组工作模式选择
- 问什么：该 PCRF 组内多台 PCRF 是按权重分担用户会话，还是主备倒换。
| 模式 | 行为 | 下游影响 |
|---|---|---|
| LOAD_BALANCE | 组内 PCRF 按 WEIGHT 百分比负荷分担 | [ADD PCRFBINDGRP](task/UNC/20.15.2/0-00058.md) 需为每个 PCRF 配 WEIGHT |
| MASTER_SLAVE（默认） | 主用 PCRF 服务，主用故障时倒换备 PCRF | 第一个加入组的 PCRF 自动成为缺省 Master，WEIGHT 不配 |

### 是否开启 Gx Failover（FAILOVERSW）
- 问什么：当 PCRF 在重传间隔内未响应（状态异常）时，是否按 Failover 逻辑向备 PCRF 重发；若不开则按 SET PCCFAILACTION/ADD PCCTEMPLATE 动作处理。
| FAILOVERSW | 行为 |
|---|---|
| ENABLE | 主备组下向备 PCRF 重发；备也不可用再按 PCCFAILACTION 处理 |
| DISABLE（默认） | 直接按 [SET PCCFAILACTION](task/UNC/20.15.2/0-00192.md) 与 [ADD PCCTEMPLATE](task/UNC/20.15.2/0-00053.md) 配置的去活/转本地 PCC 动作处理 |

## 约束
- 生效时机：命令执行后**立即生效**。`severity=medium`
- 记录上限：最大记录数 **100**。`severity=medium`
- 缺省 Master 规则：主备模式下，第一个加入某 PCRF 组的 PCRF 自动成为该组缺省 Master PCRF，后续加入者为 Slave，该状态持续至 SET MASTERPCRF 改 Master 或 Master 被删除。`severity=high`
- Failover 分支：FAILOVERSW=DISABLE 时故障处理交给 SET PCCFAILACTION + ADD PCCTEMPLATE，须确保二者已配置。`severity=high`
- 前置依赖：建组后须由 [ADD PCRF](task/UNC/20.15.2/0-00057.md) 配 PCRF、[ADD PCRFBINDGRP](task/UNC/20.15.2/0-00058.md) 加入组才能实际生效。`severity=high`
