# 配置用户 TAI 区域（ADD USRTAIRANGE）

> 命令静态知识见 [ADD USRTAIRANGE](command/UNC/20.15.2/ADD-USRTAIRANGE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"区域名 + PLMN(MCC/MNC) + TAC 范围"四元组配置，每个 TAI 区域一条。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 区域标识 | RANGENAME | 1~32 字符，字符串 | 无 | 指定区域名称，不区分大小写，转小写存储 | 所有场景 |
| PLMN | MCC | 3 位数字（十进制） | 无 | 移动国家码，全网规划 | 所有场景 |
| PLMN | MNC | 2~3 位数字（十进制） | 无 | 移动网络码，全网规划 | 所有场景 |
| TAC 范围 | TACSTART | 4~6 位十六进制 | 无 | TAC 区域起点 | 所有场景 |
| TAC 范围 | TACEND | 4~6 位十六进制，需与 TACSTART 位数一致且 >= TACSTART | 无 | TAC 区域终点 | 所有场景 |

典型脚本（单区域）：
```
ADD USRTAIRANGE:RANGENAME="tai1",MCC="460",MNC="02",TACSTART="1234",TACEND="1239";
```

步骤位置：5G PCC "PCF 发现和选择"链路第三环——`ADD PCFSSCOPE`（建 PCF 业务服务区）→ 本命令（定义用户 TAI 区域）→ `ADD PCFSSCOPEBIND`（绑定 TAI 区域到 PCF 业务服务区）。语音用户建 N7 会话时，其 TAI 匹配本配置记录后进而选到对应 PCF。

## 决策点

### 决策点：区域内多 TAC 段是否拆多条（DP 0-01232）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 拆多条 | RANGENAME 区分命名（如 tai1、tai2） | 一个逻辑区域含多个不连续 TAC 段 |

每条记录只能表示一个 `[TACSTART, TACEND]` 闭区间；不连续段需拆多条，同 RANGENAME 不能重复。

### 决策点：TAC 范围重叠策略（DP 0-01233）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 允许重叠（本命令层） | 无 | 录入时不做校验 |
| 禁止重叠（绑定层） | `ADD PCFSSCOPEBIND` 绑定时限制 | 不同 PCF 业务服务区对应的用户 TAI 区域不能重叠 |

本命令层允许不同 USRTAIRANGE 记录的 `[TACSTART, TACEND]` 重叠；但下游 `ADD PCFSSCOPEBIND` 会限制不同 PCF 业务服务区对应的用户 TAI 区域不能重叠。

## 约束

- **下游一致性**（critical，rule-0-01516）：下游一致性：RANGENAME 须被 `ADD PCFSSCOPEBIND` 正确引用，否则业务选路失败
- **最大记录数 5000**（warning，rule-0-01517）：最大记录数 5000：全网 TAI 区域条数上限
- **RANGENAME 唯一**（warning，rule-0-01518）：RANGENAME 唯一：存在多条记录时该参数不能重复
- **前置依赖**（warning，rule-0-01519）：前置依赖：SMF 需按 SERVINGSCOPE 选 PCF 才需规划本命令；纯本地 PCC 场景不强求
- **立即生效**（warning，rule-0-01520）：立即生效：命令执行后无需等待
