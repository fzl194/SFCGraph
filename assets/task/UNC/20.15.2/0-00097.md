# ADD USRTAIRANGE 原子任务
> 命令静态知识见 [ADD USRTAIRANGE](command/UNC/20.15.2/ADD-USRTAIRANGE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是 5G PCC "PCF 发现和选择"链路的地域划分环：`ADD PCFSSCOPE`（建 PCF 业务服务区）→ **本命令**（定义用户 TAI 区域）→ `ADD PCFSSCOPEBIND`（把 TAI 区域绑到 PCF 业务服务区）。语音用户建 N7 会话时，其 TAI 匹配到本配置某条记录，进而选到对应 PCF。

## 配置方法
按"区域名 + PLMN(MCC/MNC) + TAC 范围"四元组配置，每个 TAI 区域一条。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| RANGENAME | 本端规划命名（区域语义名） | `tai1` |
| MCC | 全网规划（移动国家码，固定 3 位） | `460` |
| MNC | 全网规划（移动网络码，2~3 位） | `02` |
| TACSTART | 全网规划（TAC 起点，4~6 位十六进制） | `1234` |
| TACEND | 全网规划（TAC 终点，4~6 位十六进制） | `1239` |

典型脚本（单区域）：
```
ADD USRTAIRANGE:RANGENAME="tai1",MCC="460",MNC="02",TACSTART="1234",TACEND="1239";
```

## 决策点

### 区域内多 TAC 段是否拆多条
- 问什么：一个逻辑区域含多个不连续 TAC 段时如何配。
- 每条记录只能表示一个 `[TACSTART, TACEND]` 闭区间；不连续段需拆多条，RANGENAME 区分（如 `tai1`、`tai2`）。同 RANGENAME 不能重复。

### RANGENAME 命名来源
- 问什么：RANGENAME 取自哪里。
- 标"本端规划"，由配置生成按区域划分产出。不区分大小写，转小写存储——上游引用时需按小写匹配。

### TAC 范围重叠策略
- 问什么：不同记录的 TAC 范围能否重叠。
- 本命令层允许不同 USRTAIRANGE 记录的 `[TACSTART,TACEND]` 重叠；但下游 `ADD PCFSSCOPEBIND` 会限制"不同 PCF 业务服务区对应的用户 TAI 区域不能重叠"。即：重叠在录入层合法，在绑定层才被约束。

## 约束

- **立即生效** · severity=LOW。
- **最大记录数 5000**：全网 TAI 区域条数上限 · severity=MEDIUM。
- **RANGENAME 唯一**：存在多条记录时该参数不能重复 · severity=MEDIUM。
- **RANGENAME 格式**：1~32 字符，不区分大小写，转小写存储 · severity=LOW。
- **PLMN 长度**：MCC 固定 3 位，MNC 2~3 位 · severity=LOW。
- **TAC 长度**：TACSTART/TACEND 均 4~6 位十六进制 · severity=LOW。
- **前置依赖**：SMF 需按 SERVINGSCOPE 选 PCF 才需规划本命令；纯本地 PCC 场景不强求 · severity=MEDIUM。
- **下游一致性**：RANGENAME 须被 `ADD PCFSSCOPEBIND` 正确引用 · severity=HIGH。
