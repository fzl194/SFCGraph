# SET SMFSOFTPARA 原子任务

> 命令静态知识见 [SET SMFSOFTPARA](command/UNC/20.15.2/SET-SMFSOFTPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET SMFSOFTPARA 是 SMF 软件参数设置命令，按 `(DT 数据类型, 索引, 值)` 三元组定位一个软参位并写入值。同一命令在不同特性里复用，**唯一区别是索引号和取值**——每个特性在文档里指定了自己需要打开的软参位（如 BIT717、BIT719、BIT1202、BYTE863 等）。配置生成时只需把特性声明的那个软参位翻成参数行即可。

**参数骨架（按 DT 选择索引族）**

| DT | 索引参数 | 值参数 | 索引范围 | 值范围 |
|---|---|---|---|---|
| Bit | BITNUM | BITVALUE | 1~4000 | 0~1 |
| Byte | BYTENUM | BYTEVALUE | 1~1200 | 0~255 |
| Dw（双字） | DWORDNUM | DWORDVALUE | 1~1500 | 0~4294967295 |
| String | STRINGNUM | STRINGVALUE | 1~30 | 字符串，1~64 字符 |

**特性→软参位映射（实测范式）**

| 特性 | 软参位 | 行样例 | 用途 |
|---|---|---|---|
| Category NB2 接入（WSFD-215501） | BIT717 | `DT=Bit, BITNUM=717, BITVALUE=1` | 打开 NB2 接入软参开关 |
| E-UTRAN 和 WLAN 互操作（WSFD-201301） | BIT719 | `DT=Bit, BITNUM=719, BITVALUE=1` | 允许 non-3GPP/3GPP 切换时双栈与单栈切换 |
| Gx Failover 增强（WSFD-109101） | BIT1202 | `DT=BIT, BITNUM=1202, BITVALUE=1` | 开启 Gx Failover 增强功能（PCRF 不支持热备时） |
| Gx 接口 WAL 流控（WSFD-010802） | BYTE863 | `DT=BYTE, BYTENUM=863, BYTEVALUE=40` | Gx 集中点 WAL 流控开关与阈值 |

**典型行（开 NB2 软参）**

```
SET SMFSOFTPARA: DT=Bit, BITNUM=717, BITVALUE=1;
```

## 决策点

### 开哪个软参位（特性驱动，单特性单软参）

本命令的参数绑定完全由"当前激活哪个特性"决定——每个特性在文档里声明了它依赖的特定软参位（BIT/BYTE 索引+语义化取值）。配置生成时按特性查表，无需在运行期多值决策。

| 待激活特性 | 应配置的软参位 |
|---|---|
| Category NB2 接入 | BIT717=1 |
| E-UTRAN/WLAN 互操作 | BIT719=1 |
| Gx Failover 增强 | BIT1202=1 |
| Gx WAL 流控 | BYTE863=<阈值> |

### BYTE863 取值来源（与对端协商）

WSFD-010802 中 BYTE863 的取值标记为"与对端协商"——其值（如 40）并非全网统一规划，而是取决于与对端 PCF/PCRF 的协商结果，需运行期获取。

### Gx Failover 增强是否需要开（PCRF 热备决定）

BIT1202 仅在"PCRF 不支持热备份功能"时才开。若 PCRF 支持热备则跳过此软参，避免误开 failover 增强。

| 选项 | 条件 | 动作 |
|---|---|---|
| 开 Failover 增强 | PCRF 不支持热备 | BIT1202=1 + 配合 SET PCCMSGATTR 上报参数 |
| 跳过 | PCRF 支持热备 | 不配 |

## 约束

- **立即生效** · severity=LOW。
- **存在系统初始记录** · severity=LOW · 部署后已有初始值，可通过 LST SMFSOFTPARA 查询全部。
- **未输入的参数保持系统当前配置不变** · severity=MEDIUM · 非覆盖语义，按需显式传入；改前先查。
- **软参位语义化取值** · severity=MEDIUM · 每个索引号对应特定语义（见软件参数手册），取值必须按特性文档指定，不能臆造。
- **License 前置**（NB2 等）：部分软参开关需 SET LICENSESWITCH 先打开 License 项 · severity=MEDIUM。
- **Gx Failover 增强**开 BIT1202 后通常配合 SET PCCMSGATTR 配置 CCR-U/CCR-T 携带信元 · severity=MEDIUM。
