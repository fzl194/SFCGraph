# 修改目标NF实例绑定组（MOD TNFBINDGRP）

> 命令静态知识见 [MOD TNFBINDGRP](command/UNC/20.15.2/MOD-TNFBINDGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按目标 NF 组索引 + 实例索引定位已存在的绑定，按规划修改优先级/权重。

### 参数维度表

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|------|------|------|------|------|----------|
| 组索引 | TNFGRPINDEX | 0~511 | — | 目标 NF 组索引，必选，需与 ADD TNFGRP 一致 | 所有场景 |
| 实例索引 | TNFINSINDEX | 0~2047 | — | 目标 NF 实例索引，必选，需与 ADD TNFINS 一致 | 所有场景 |
| 优先级 | PRIORITY | 1~255，越小优先级越高 | — | NF 选择时优先选择高优先级 NF | 主备链路中主实例置小、备实例置大 |
| 权重 | WEIGHT | 1~65535，越大权重越大 | — | 同优先级 NF 间的相对权重（负荷分担） | 负荷分担或多实例权重调优 |

### 典型脚本

将索引 0 的目标 NF 组中索引 1 的 NF 实例优先级改为 10、权重改为 20：

```
MOD TNFBINDGRP: TNFGRPINDEX=0, TNFINSINDEX=1, PRIORITY=10, WEIGHT=20;
```

### 步骤位置

主备 CHF 可靠性调测链路（WSFD-011206）中的位置：
1. `ADD TNFINSIP` — 配置 IP 地址
2. `MOD TNFINS` — 补全协议/域名
3. `ADD TNFGRP` — 建目标 NF 组
4. `MOD TNFBINDGRP` — 绑定实例到组并配优先级/权重（本案）
5. `MOD SELECTCHFGBYCC` — 基于 CC 选择 CHF 组

## 决策点

### 决策点：优先级调优（DP 0-01296）

主备链路中主实例 PRIORITY 置小、备实例置大，是否调优、调到何值随主备规划分叉。

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 主实例置小 PRIORITY | PRIORITY 取较小值（如 1~10） | 主备 CHF 部署中的主 CHF 实例 |
| 备实例置大 PRIORITY | PRIORITY 取较大值（如 100+） | 主备 CHF 部署中的备 CHF 实例 |
| 不配置 PRIORITY | 留空，按默认选择顺序 | 不需要显式优先级控制的场景 |

属全网规划决策（主备角色与优先级数值由网络拓扑规划决定）。

### 决策点：权重调优（DP 0-01297）

同优先级多实例间的负荷分担通过 WEIGHT 调节；是否调优、调到何值按负载规划分叉。

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 按负载配权重 | WEIGHT 按实例处理能力比例取值 | 同优先级多实例的负荷分担场景 |
| 不配置 WEIGHT | 留空，按默认权重 | 单实例或不需要权重细分的场景 |

属全网规划决策（实例间的负载比例由容量规划决定）。

## 约束

- **立即生效**（critical，rule-0-01677）：立即生效：修改后立即影响 NRF 服务发现对该实例的选择结果

- **绑定存在性前置**（warning，rule-0-01678）：绑定存在性前置：TNFGRPINDEX + TNFINSINDEX 组合需已通过 ADD TNFBINDGRP 添加（即目标 NF 组与目标 NF 实例均已存在且已绑定）；MOD 仅改优先级/权重，可用 LST TNFBINDGRP 校验

- **实例需有 IP 才有意义**（warning，rule-0-01679）：实例需有 IP 才有意义：目标 NF 实例需先 ADD TNFINS + ADD TNFINSIP 配置 IP，绑定才有意义

- **优先级/权重方向相反**（warning，rule-0-01680）：优先级/权重方向相反：PRIORITY 越小越优先，WEIGHT 越大权重越大（与 RULE.PRIORITY 单维度语义不同，避免混淆）

- **索引范围**（info，rule-0-01681）：索引范围：TNFGRPINDEX 0~511，TNFINSINDEX 0~2047，需与 ADD TNFGRP / ADD TNFINS 中使用的索引一致
