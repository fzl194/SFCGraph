# MOD TNFBINDGRP 原子任务
> 命令静态知识见 [MOD TNFBINDGRP](command/UNC/20.15.2/MOD-TNFBINDGRP.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：SMF、NCG、SMSF。本命令按目标 NF 组索引 + 目标 NF 实例索引定位已存在的绑定，修改优先级与权重。是 NRF 服务发现侧"把目标 NF 实例绑进目标 NF 组"的修改入口，常作为基于 CC 选择 CHF 主备链路配置的一环（实例入组后才能被 SELECTCHFGBYCC 选中的 CHF 组引用）。

## 配置方法
按目标 NF 组索引 + 实例索引定位绑定，按规划修改优先级/权重。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| TNFGRPINDEX | 目标 NF 组索引（必选，0~511） | 按规划 |
| TNFINSINDEX | 目标 NF 实例索引（必选，0~2047） | 按规划 |
| PRIORITY | 优先级（可选，1~255，越小优先级越高） | 主实例置小、备实例置大 |
| WEIGHT | 权重（可选，1~65535，越大权重越大） | 按负载规划 |

主备 CHF 可靠性调测（WSFD-011206）：链路 `ADD TNFINSIP → MOD TNFINS → ADD TNFGRP → MOD TNFBINDGRP → MOD SELECTCHFGBYCC` 中，MOD TNFBINDGRP 负责把备 CHF 实例绑进目标 NF 组并配优先级/权重，配合 LST TNFBINDGRP 校验。

## 决策点

### 优先级与权重调优（PRIORITY / WEIGHT）
绑定关系同时承载优先级（越小越优先）和权重（越大权重越大）两个选择调优维度。主备链路中：主 CHF 实例 PRIORITY 小、WEIGHT 大；备 CHF 反之。是否调优、调到何值随主备规划分叉。

## 约束
- **立即生效**：修改后立即影响 NRF 服务发现对该实例的选择结果。
- **绑定存在性前置**：TNFGRPINDEX + TNFINSINDEX 组合需已通过 ADD TNFBINDGRP 添加（即目标 NF 组与目标 NF 实例均已存在且已绑定）；MOD 仅改优先级/权重，可用 LST TNFBINDGRP 校验。
- **索引范围**：TNFGRPINDEX 0~511，TNFINSINDEX 0~2047。
- **优先级/权重方向相反**：PRIORITY 越小越优先，WEIGHT 越大权重越大（与 RULE.PRIORITY 单维度语义不同，避免混淆）。
- **配合 TNFINSIP/TNFINS 才完整**：目标 NF 实例需先 ADD TNFINS + ADD TNFINSIP 配置 IP，绑定才有意义。
