# MOD SELECTCHFGBYCC 原子任务
> 命令静态知识见 [MOD SELECTCHFGBYCC](command/UNC/20.15.2/MOD-SELECTCHFGBYCC.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：PGW-C、SMF。本命令按 Charge Characteristic（CC）类型+值+掩码定位已存在的 CHF 选择规则，修改主/备 CHF 组绑定与优先级。是融合计费"基于 CC 选择 CHF"本地配置链的核心修改步骤，用于主备 CHF 可靠性、计费会话创建（无可用 CHF 时配置可选 CHF）两类调测。

## 配置方法
按 CC 类型 + CC 值 + 掩码定位规则，按规划修改主备 CHF 组与优先级。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTYPE | Charge Characteristic 类型（必选） | `DEFAULT`（未指定 CC 值） |
| CCVALUE | CC 值（conditional，十六进制 1~6，0x/X+a-f/A-F+0~9） | 按规划 |
| MASK | CC 特定值掩码（conditional，十六进制 1~6） | 按规划 |
| PRIORITY | CC 优先级（conditional，1~65535） | 按规划 |
| PRIMARYCHFGRP | 主 CHF 组（可选，1~63，输空格/null 清空） | 按规划 CHF 组名 |
| SECONDARYCHFGRP | 备 CHF 组（可选，1~63，输空格/null 清空） | 按规划 CHF 组名 |

主备 CHF 可靠性调测（WSFD-011206）：先用 LST SELECTCHFGBYCC / LST TNFBINDGRP / LST TNFGRP / LST TNFINSIP / LST TNFINS 查本地备 CHF 组及 IP，未配置时按链路 `ADD TNFINSIP → MOD TNFINS → ADD TNFGRP → MOD TNFBINDGRP → MOD SELECTCHFGBYCC` 配置基于 CC 的主备 CHF 组绑定。

计费会话创建调测（WSFD-011206）：当无可用 CHF 时，与 SET GLBDFTCHFGROUP 一起执行 MOD SELECTCHFGBYCC，按规划配置当前用户可选 CHF。

## 决策点

### CC 维度组合（CCTYPE / CCVALUE / MASK）
CC 选择规则由 CCTYPE + CCVALUE + MASK 共同定位。CCTYPE 取 `DEFAULT` 表示未指定 CC 值；其余类型需配 CCVALUE/MASK（十六进制）按业务分叉。取值/掩码随计费业务规划分叉，是本命令的核心条件定位维度。

### 主备 CHF 组选择（PRIMARYCHFGRP / SECONDARYCHFGRP）
主备 CHF 组名随网络规划（哪些 CHF 组作主、哪些作备）分叉；输空格或 null（不区分大小写）可清空已绑定值。是否配备组取决于是否启用 CHF failover。

### 优先级（PRIORITY）
多条 CC 规则的匹配优先级（1~65535），随 CC 规则集规划分叉。

## 约束
- **立即生效**：修改后立即影响 CHF 选择结果。
- **规则存在性前置**：CCTYPE+CCVALUE+MASK 组合需已通过 ADD SELECTCHFGBYCC 添加，MOD 仅改主备组/优先级；可用 LST SELECTCHFGBYCC 校验。
- **CCVALUE/MASK 为十六进制**：长度 1~6，仅支持 0x/X、a-f/A-F、0~9。
- **PRIMARYCHFGRP/SECONDARYCHFGRP 清空语义**：输空格或 null（不区分大小写）即清空参数值，用于解绑 CHF 组。
- **备 CHF 组需有 IP 与绑定**：备 CHF 组需经 ADD TNFGRP + ADD TNFINSIP + MOD TNFINS + MOD TNFBINDGRP 完整配置，否则 failover 不可用；主备 CHF 的 failover 能力还受 CHF 下发的 sessionFailover 信元（FAILOVER_NOT_SUPPORTED）约束。
