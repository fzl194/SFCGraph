# 增加目标NF实例IP地址（ADD TNFINSIP）

> 命令静态知识见 [ADD TNFINSIP](command/UNC/20.15.2/ADD-TNFINSIP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"主备 CHF 各一条 IP"配置，索引回指 TNFINS，PORT 需在同一 TNFINSINDEX 下保持一致。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 索引 | TNFINSINDEX | 0~2047 整数，本端规划 | 无 | 回指 TNFINS 的实例索引，必须与 ADD TNFINS 的 TNFINSINDEX 一致 | 主备 CHF 各取对应索引（1 主、2 备） |
| IP 类型 | IPTYPE | IPV4 / IPV6 / NONEIP | 无 | 指定地址族；IPTYPE 为 NONEIP 时最多只能配一条 | 全网规划，由对端 CHF 实际地址族决定 |
| IPv4 地址 | IPV4ADDR | 1.0.0.0~255.255.255.254，IPv4 单播地址 | 无（IPTYPE=IPV4 时必选） | CHF 实例 IPv4 地址 | IPTYPE=IPV4 时填写，不可为组播/环回地址 |
| IPv6 地址 | IPV6ADDR | 全球单播 IPv6 地址 | 无（IPTYPE=IPV6 时必选） | CHF 实例 IPv6 地址 | IPTYPE=IPV6 时填写，不可为链路本地/组播/环回地址 |
| 端口号 | PORT | 1~65535 整数 | 无（IPTYPE=NONEIP 时必选） | CHF 服务端口；同一 TNFINSINDEX 下多条记录时必须一致 | 全网规划；IPTYPE 为 IPV4/IPV6 时可选，为 NONEIP 时必选 |

典型脚本（主备 CHF）：
```
ADD TNFINSIP: TNFINSINDEX=1, IPTYPE=IPV4, IPV4ADDR="192.168.94.2", PORT=3220;
ADD TNFINSIP: TNFINSINDEX=2, IPTYPE=IPV4, IPV4ADDR="192.168.104.2", PORT=3230;
```

双栈脚本（同一 CHF 实例需分别建两条不同 IPTYPE 的记录）：
```
ADD TNFINSIP: TNFINSINDEX=1, IPTYPE=IPV4, IPV4ADDR="192.168.94.2", PORT=3220;
ADD TNFINSIP: TNFINSINDEX=1, IPTYPE=IPV6, IPV6ADDR="2001:db8::1", PORT=3220;
```

步骤位置：全局 NF 实例配置 → NRF 基础数据 → 目标 NF 实例管理 → IP 地址配置。

## 决策点

### 决策点：IPTYPE 与地址字段选择（DP 0-01212）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPV4 | 填 IPV4ADDR，PORT 填写 | 对端 CHF 为 IPv4 地址 |
| IPV6 | 填 IPV6ADDR，PORT 填写 | 对端 CHF 为 IPv6 地址 |
| 双栈 | 为同一 TNFINSINDEX 分别建两条 IPTYPE 不同的记录（IPV4 + IPV6），PORT 各自填写且需一致 | 对端 CHF 同时支持 IPv4/IPv6 |
| NONEIP | 不配地址，只配 PORT；最多只能配一条 | 特殊场景无需 IP 只需端口 |

### 决策点：PORT 一致性（DP 0-01213）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 同一 TNFINSINDEX 下所有 TNFINSIP 记录 PORT 一致 | 多条 TNFINSIP 时 PORT 相同，地址可不同 | 标准主备 CHF 配置 |
| 同一 TNFINSINDEX 下 PORT 不一致（禁止） | 配不同 PORT 会覆盖对端 NF 实例配置中的 PORT，导致 IP/PORT 不匹配，业务失败 | — |

## 约束

- **该命令执行后立即生效**（info，rule-0-01466）：该命令执行后立即生效
- **PORT 一致性**（warning，rule-0-01467）：PORT 一致性：相同 TNFINSINDEX 下 PORT 必须一致，否则覆盖对端 PORT 导致 IP/PORT 不匹配，业务失败
- **IP 归属一致性**（warning，rule-0-01468）：IP 归属一致性：若关联的目标 NF 实例名称能匹配到对端 NF 概述信息（大小写不敏感），则本 IP 必须属于该对端 NF 概述的 IP 集合，否则配置失败
- **CHF IP 包含关系**（warning，rule-0-01469）：CHF IP 包含关系：若 IP 包含在 NfCHF 类型对端 NF 实例概述信息中，需确认关联的目标 NF 实例索引一致（大小写不敏感），否则配置失败
- **记录上限**（info，rule-0-01470）：记录上限：最多 4096 条；IPTYPE 为 NONEIP 的记录最多只能配置一条
