# ADD TNFINSIP 原子任务
> 命令静态知识见 [ADD TNFINSIP](command/UNC/20.15.2/ADD-TNFINSIP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令为 [ADD TNFINS](task/UNC/20.15.2/0-00086.md) 注册的实例补全 IP/端口，是 CHF 可达性的最后一环。TNFINSINDEX 必须与 TNFINS 一致；同一实例下多条记录的 PORT 必须一致。

## 配置方法
按"主备 CHF 各一条 IP"配置，索引回指 TNFINS。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| TNFINSINDEX | 本端规划，回指 TNFINS 的实例索引 | `1`（主）、`2`（备） |
| IPTYPE | 全网规划，IP 地址类型 | `IPV4` |
| IPV4ADDR | 全网规划，CHF 实例 IPv4 地址 | `192.168.94.2` |
| PORT | 全网规划，CHF 服务端口 | `3220` |

典型脚本（主备 CHF）：
```
ADD TNFINSIP: TNFINSINDEX=1, IPTYPE=IPV4, IPV4ADDR="192.168.94.2", PORT=3220;
ADD TNFINSIP: TNFINSINDEX=2, IPTYPE=IPV4, IPV4ADDR="192.168.104.2", PORT=3230;
```

## 决策点

### IPTYPE 与地址字段选择
- 问什么：IPv4 / IPv6 / 双栈。
- IPV4 → 填 IPV4ADDR；IPV6 → 填 IPV6ADDR；双栈需为同一实例分别建两条不同 IPTYPE 的记录。
- 取值由对端 CHF 实际地址族决定。

### PORT 一致性
- 问什么：同一 TNFINSINDEX 下若配置多条 TNFINSIP，PORT 必须一致。
- 配不同 PORT 会覆盖对端 NF 实例配置中的 PORT，导致对端 NF 实例 IP 使用不匹配的 PORT，业务失败。

## 约束

- **立即生效** · severity=LOW。
- **PORT 一致性**：相同 TNFINSINDEX 下 PORT 必须一致，否则覆盖对端 PORT 导致 IP/PORT 不匹配 · severity=HIGH。
- **IP 归属一致性**：若关联的目标 NF 实例名称能匹配到对端 NF 概述信息，则本 IP 必须属于该对端 NF 概述的 IP 集合，否则配置失败 · severity=HIGH。
- **CHF IP 包含关系**：若 IP 包含在 NfCHF 类型对端 NF 实例概述信息中，需确认关联的目标 NF 实例索引一致 · severity=HIGH。
- **记录上限**：最多 4096 条 · severity=MEDIUM。
