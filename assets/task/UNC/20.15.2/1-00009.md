# 配置融合计费费率标识链

> 融合计费内容计费费率标识配置——URR(费率组RG/费率组+业务ID(RG,SID))→URRGROUP(建议同时配在线+离线URR)→PCCPOLICYGRP→RULE→RULEBINDING→UserProfile + 缺省 URR 组绑定。被引用于 [2-00003 融合计费](task/UNC/20.15.2/2-00003.md) · [2-00004 内容计费](task/UNC/20.15.2/2-00004.md) · [2-00011 业务级FUP](task/UNC/20.15.2/2-00011.md)。域通用（计费族费率标识链结构跨在线/离线/融合/内容计费/业务级FUP，但融合的"双 URR（在线+离线）"、FUP 的"MONITORINGKEY 变种"是特性差异，见场景差异）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 配 URR（在线） | `ADD URR` | URRNAME；URRID（SMF/UPF 一致）；USAGERPTMODE=ONLINE；ONLCOMPOUNDTYPE=ONLYRG/RGSID；ONLINERG（+ONLINESID 若 RGSID）；ONLMETERINGTYPE=VOLUME/DURATION/EVENT/FREE |
| 配 URR（离线） | `ADD URR` | URRNAME；URRID；USAGERPTMODE=OFFLINE；OFFCOMPOUNDTYPE=ONLYRG/RGSID；RG；OFFMETERINGTYPE |
| 配 URR 组（建议同时配在线+离线） | `ADD URRGROUP` | URRGROUPNAME；UPURRNAME1/DOWNURRNAME1（在线）；UPURRNAME2/DOWNURRNAME2（离线） |
| 配 PCC 策略组（绑 URR 组） | `ADD PCCPOLICYGRP` | PCCPOLICYGRPNM；URRGROUPNAME |
| 配 Rule（绑 PCC 策略组） | `ADD RULE` | RULENAME；POLICYTYPE=PCC；PRIORITY；POLICYNAME |
| 绑 Rule 到 UserProfile | `ADD RULEBINDING` | USERPROFILENAME；RULENAME |
| 缺省 URR 组绑定 | `SET URRGRPBINDING` | USERPROFILENAME；DFTURRGRPNAME（异常场景 URR 组） |

**典型脚本（内容计费：URL/IMS/any 三业务 + 异常 URR，每业务配在线+离线双 URR）**：
```
ADD URR: URRNAME="urr_online_URL", URRID=2300, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=100, ONLMETERINGTYPE=VOLUME;
ADD URR: URRNAME="urr_offline_URL", URRID=12500, USAGERPTMODE=OFFLINE, OFFCOMPOUNDTYPE=ONLYRG, RG=1100, OFFMETERINGTYPE=VOLUME;
ADD URRGROUP: URRGROUPNAME="UrrGp_URL", UPURRNAME1="urr_online_URL", DOWNURRNAME1="urr_online_URL", UPURRNAME2="urr_offline_URL", DOWNURRNAME2="urr_offline_URL";
ADD PCCPOLICYGRP: PCCPOLICYGRPNM="pccpolicy_URL", URRGROUPNAME="UrrGp_URL";
ADD RULE: RULENAME="Rule001_dns", POLICYTYPE=PCC, PRIORITY=100, POLICYNAME="pccpolicy_URL";
ADD USERPROFILE: USERPROFILENAME="userprofile001", UPTYPE=PCC;
ADD RULEBINDING: USERPROFILENAME="userprofile001", RULENAME="Rule001_dns";
SET URRGRPBINDING: USERPROFILENAME="userprofile001", DFTURRGRPNAME="UrrGp_abnormal";
```

**步骤位置**：CCT 模板（[1-00005](task/UNC/20.15.2/1-00005.md)）之后或并行；费率标识链是内容计费的核心，UserProfile 层绑定（[1-00002](task/UNC/20.15.2/1-00002.md)）共享 UserProfile 子 atom。

## 场景差异

- **融合计费（WSFD-011206，本特性）**：URRGROUP 建议同时配在线+离线 URR（UPURRNAME1/DOWNURRNAME1=在线，UPURRNAME2/DOWNURRNAME2=离线）——因融合计费同一 RG 可在线/离线，由 `RGAPPLIED`（[1-00002](task/UNC/20.15.2/1-00002.md)/[1-00003](task/UNC/20.15.2/1-00003.md) 的 SET USRPROFCHARGE/SET APNCHARGECTRL 参数）决定走在线还是离线 URR。`RGAPPLIED=DEFAULT` 时 URRGROUP 不能同时配在线+离线（优先选在线）；`ONLINERGONLY`/`OFFLINERGONLY` 时建议同时配。ONLCOMPOUNDTYPE=ONLYRG 用 RG，=RGSID 用 (RG,SID) 做更细粒度配额管理。ONLMETERINGTYPE=VOLUME/DURATION/EVENT/FREE 四种统计类型。被引用于 [2-00003 融合计费](task/UNC/20.15.2/2-00003.md)。
- **内容计费（WSFD-109002，参数变种）**：内容计费**无专属命令**，完全复用本费率标识链——差异化体现在 RG/SID 标识不同业务费率（如 RG=100=URL业务、RG=200=IMS业务、RG=300=any业务），每条业务配一组在线+离线双 URR，异常场景配 FREE URR 作缺省（DFTURRGRPNAME）。activation 原文"操作同融合计费费时识别"。License 前置（LKV3W9BCC12）在费率标识链外（[2-00004](task/UNC/20.15.2/2-00004.md) 步骤 1）。UNC 侧 RULE 不配 FILTERINGMODE/FLOWFILTERNAME（过滤条件在 UDG 侧或 PCF 动态下发）。被引用于 [2-00004 内容计费](task/UNC/20.15.2/2-00004.md)。
- **业务级 FUP（WSFD-211009，MONITORINGKEY 变种）**：业务级累计流量策略控制复用本费率标识链，但 **URR 取 MONITORINGKEY 变种**——`ADD URR:USAGERPTMODE=MONITORINGKEY+MONITORINGKEY=<业务标识>`（非 ONLINE/OFFLINE 计费轨道，MONITORINGKEY 标识一种业务，与 PCRF/PCF 一致），**不配 RG/ONLINERG/OFFMETERINGTYPE 等费率参数**；`ADD PCCPOLICYGRP` 带 `FUPSESSIONEXC=ENABLE`（该业务不计入 Session 级流量，避免与会话级 FUP 重复累计）。多业务独立累计=每业务配独立 URR（不同 MONITORINGKEY）+ 独立 PCCPOLICYGRP，多 RULEBINDING 绑同一 USERPROFILE。N7 L7 预定义规则场景 URRID 高位 bit=1。区别于会话级 FUP [2-00007](task/UNC/20.15.2/2-00007.md)（会话级不配 URR 标识链，仅 3 参数增量）。被引用于 [2-00011 业务级FUP](task/UNC/20.15.2/2-00011.md)。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| ONLCOMPOUNDTYPE=ONLYRG | 用 RG 做费率标识（粒度粗） |
| ONLCOMPOUNDTYPE=RGSID | 用 (RG,SID) 做费率标识（更细粒度配额管理） |
| ONLMETERINGTYPE=VOLUME/DURATION/EVENT/FREE | 流量/时长/事件/免费统计类型 |
| RGAPPLIED=DEFAULT | URRGROUP 不能同时配在线+离线 URR；优先选在线 URR |
| RGAPPLIED=ONLINERGONLY/OFFLINERGONLY | URRGROUP 建议同时配在线+离线 URR |

> 联动特性级 DP：费率标识的 RG/RGAPPLIED 与 [2-00003](task/UNC/20.15.2/2-00003.md) 使能开关 DP（RGAPPLIED）联动；同一计费会话内同一 RG 的在线/离线属性须唯一。

## 约束

- **URRID 跨网元一致**（critical）：URRID/USAGERPTMODE/ONLMETERINGTYPE/OFFMETERINGTYPE 在 SMF 和 UPF 上须一致 — 否则计费统计不一致。
- **同一 RG 在线/离线属性唯一**（critical）：同一计费会话内，同一 RG 要么在线要么离线，不能同时 — 否则计费出错。
- **RGAPPLIED 与 URRGROUP 配合**（warning）：DEFAULT 不能同时配在线+离线 URR；ONLINERGONLY/OFFLINERGONLY 建议同时配。
- **RULENAME 跨网元一致**（warning）：SMF/UPF 的 RULENAME 及 RULE 下 POLICYTYPE/POLICYNAME 须一致 — 否则 PCF 下发策略无法生效。
- **建议绑缺省 URR 组**（info）：每个 UserProfile 绑定 DFTURRGRPNAME（异常场景 URR 组），未匹配规则的流量按缺省组计费。
