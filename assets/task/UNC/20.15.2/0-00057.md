# ADD PCRF 原子任务
> 命令静态知识见 [ADD PCRF](command/UNC/20.15.2/ADD-PCRF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令为 PCC 策略控制的核心配置（适用 NF：PGW-C、GGSN）。同一组网中可重复执行配置多个 PCRF 设备标识（主备/负荷分担），后续由 `ADD PCRFGROUP`/`ADD PCRFBINDGRP` 组织成 PCRF 组。HOSTNAME/REALMNAME 需与对端 `ADD DIAMPEERADDR` 配置一致。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| HOSTNAME | PCRF 主机名，本端规划，1~127 字符可见 ASCII（与对端协商/全网规划） | `pcrf1` |
| REALMNAME | PCRF 域名，本端规划，1~127 字符可见 ASCII（全网规划） | `pcrf.huawei.com` |
| VPNINSTANCE | VPN 实例，从已配置的 `ADD VPNINST` 数据中获取（可选） | `vpn_gxif` |
| SUPFEANEGOSW | Supported-Features 动态协商开关，对端协商，ENABLE/DISABLE | `ENABLE` |
| CARRYSUPFEASW | Supported-Feature AVP 携带开关，全网规划，ENABLE/DISABLE（conditional） | `ENABLE` |
| FEATURELIST | Feature 列表，与对端协商（位域） | `RELEASE8-1&RELEASE9-1&RELEASE10-1` |
| DSCPV | DSCP 值，全网规划，0~63、255（可选） | `255` |
| WALVALUE | WAL 值，本端规划，限制 UNC 每秒发往该 PCRF 的最大 CCR 消息数（可选） | `1000` |

典型脚本：
```
ADD PCRF:HOSTNAME="pcrf1",REALMNAME="pcrf.huawei.com",VPNINSTANCE="vpn_gxif",SUPFEANEGOSW=ENABLE,FEATURELIST=RELEASE8-1&RELEASE9-1&RELEASE10-1;
ADD PCRF:HOSTNAME="pcrf1",REALMNAME="www.huawei.com",WALVALUE=1000;
```

## 决策点

### 是否启用 Supported-Features 动态协商及协商的 Feature 能力集
- 问什么：是否需要与对端 PCRF 在 IP-CAN Session 建立过程中协商协议版本和 Feature 能力；协商哪些 Feature（RELEASE8/9/10、EXTENDED_BW_NR、UMCH 等）。
| 场景 | SUPFEANEGOSW | FEATURELIST |
|---|---|---|
| 不启用动态协商 | DISABLE | 不配置 |
| NSA 用户 QoS 管理 | ENABLE | `EXTENDED_BW_NR-1&RELEASE8-1` |
| 标准动态 PCC | ENABLE | `RELEASE8-1&RELEASE9-1&RELEASE10-1` |
| Gx Usage Monitoring 拥塞处理（UMCH） | ENABLE | FEATURELIST 须包含 `UMCH` |

### 是否需要按 PCRF 接收能力做流控（WAL）
- 问什么：是否根据 PCRF 接收能力限制 UNC 每秒发往该 PCRF 的最大 CCR 消息数。
| 是否流控 | WALVALUE | 典型场景 |
|---|---|---|
| 不限流 | 不配置（默认 0） | PCRF 处理能力充足 |
| 限流 | 本端规划具体值（如 1000） | 周边 Gx/Gy/Ga/Gi 接口流控、PCRF 过载保护 |

## 约束
- 生效时机：命令执行后**立即生效**。`severity=medium`
- 记录上限：最大记录数 **100**。`severity=medium`
- VPN 一致性：PCRF 绑定的 VPN 实例必须与 Gx 逻辑接口绑定的 VPN 实例一致。`severity=high`
- 主机名一致性：本命令 HOSTNAME/REALMNAME 必须与 [ADD DIAMPEERADDR](task/UNC/20.15.2/0-00026.md) 中对端地址配置的主机名/域名一致。`severity=high`
- 组内 Feature 一致性：加入同一 PCRF Group（[ADD PCRFGROUP](task/UNC/20.15.2/0-00059.md)）的各 PCRF 支持的 Feature 能力必须一致。`severity=high`
- 多链路组网：GGSN/PGW-C 与 PCRF 间若采用 TCP 传输多 Diameter 链路组网，需配置多个 PCRF IP/端口并设置链路选择模式 SELECTMODE。`severity=medium`
- 修改路径：变更 PCRF 标识需先 `RMV PCRF` 删除再 `ADD PCRF` 重配（不支持原位改 HOSTNAME/REALMNAME）。`severity=medium`
