# 增加PCRF（ADD PCRF）

> 命令静态知识见 [ADD PCRF](command/UNC/20.15.2/ADD-PCRF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"主机名 + 域名 + VPN 实例 + 动态协商开关 + Feature 列表 + DSCP/WAL"组合配置一个 PCRF 设备标识。同一组网中可重复执行配置多个 PCRF（主备/负荷分担），后续由 `ADD PCRFGROUP`/`ADD PCRFBINDGRP` 组织成 PCRF 组。HOSTNAME/REALMNAME 需与对端 `ADD DIAMPEERADDR` 配置一致。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| PCRF 标识（必选） | HOSTNAME, REALMNAME | HOSTNAME/REALMNAME 本端规划（1~127 可见 ASCII，不支持空格，BIT150 控制大小写）；HOSTNAME 不与 DRA 主机名重名，需与对端一致 | — | 唯一标识一台 PCRF 设备 | 所有 PCRF 对接场景必配 |
| VPN 实例绑定（可选） | VPNINSTANCE | 已配置数据获取（由 `ADD VPNINST` 建好，1~31，区分大小写） | 无 | 把 PCRF 归属到指定 VPN 实例 | Gx 接口走 VPN 实例 |
| 动态协商（可选） | SUPFEANEGOSW, CARRYSUPFEASW | SUPFEANEGOSW 对端协商（ENABLE/DISABLE）；CARRYSUPFEASW 全网规划（ENABLE/DISABLE，仅 SUPFEANEGOSW=DISABLE 时有意义） | DISABLE / ENABLE | 控制是否在 IP-CAN Session 建立时与 PCRF 协商协议版本与 Feature 能力 | NSA QoS、Usage Monitoring 等需 Feature 协商的场景 |
| Feature 列表（可选） | FEATURELIST | 对端协商，位域（RELEASE8/9/10 必至少选一；可选 ADC/NETLOC/PROVAFSIGNALFL/UMCH/SGW_REST/CNO_ULI/PENDINGTRANSACTION/PCSCF_RESTORATION_ENHANCEMENT/EXTENDED_BW_NR） | 无（动态协商关闭时默认 RELEASE8+RELEASE9） | 指定要协商的 Gx Feature 能力集 | 动态 PCC、NSA 扩展带宽、Usage Monitoring 拥塞处理 |
| DSCP（可选） | DSCPV | 全网规划，0~63、255 | 255（继承 `SET LOGIFDSCP（★R1.5 atom 待补）` 全局值） | PCC 信令报文 DSCP 标记 | QoS 标记 |
| WAL 流控（可选） | WALVALUE | 本端规划，0~4294967295 | 0（不限流） | 限制 UNC 每秒发往该 PCRF 的最大 CCR 消息数（CCR-T 不受限） | PCRF 过载保护、周边接口流控 |

典型脚本（动态协商 + Feature 列表 + VPN 实例）：
```
ADD PCRF:HOSTNAME="pcrf1",REALMNAME="www.huawei.com",VPNINSTANCE="vpn1",SUPFEANEGOSW=ENABLE,FEATURELIST=RELEASE8-1;
```
典型脚本（主备组网下，只配主机名/域名 + WAL 流控）：
```
ADD PCRF:HOSTNAME="pcrf1",REALMNAME="host1.example.com";
ADD PCRF:HOSTNAME="pcrf2",REALMNAME="host2.example.com",WALVALUE=1000;
```

步骤位置：建 VPN 实例（`ADD VPNINST`）→ 建 Diameter 对端地址（`ADD DIAMPEERADDR`，HOSTNAME 须一致）→ 本任务 → 下游建 PCRF 组（`ADD PCRFGROUP`）/ 加入组（`ADD PCRFBINDGRP`）

## 决策点

### 决策点：是否启用 Supported-Features 动态协商及协商的 Feature 能力集（DP 0-02093）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不启用动态协商 | SUPFEANEGOSW=DISABLE（默认），FEATURELIST 不配或按实际配置交互 | PCRF 处理能力固定、无需 Feature 协商 |
| 标准动态 PCC | SUPFEANEGOSW=ENABLE, FEATURELIST=`RELEASE8-1&RELEASE9-1&RELEASE10-1` | 3GPP Rel-8/9/10 标准 Gx 功能 |
| NSA 用户 QoS 管理 | SUPFEANEGOSW=ENABLE, FEATURELIST 须含 `EXTENDED_BW_NR` | NSA NSA(Opt.3) 扩展带宽特性 |
| Gx Usage Monitoring 拥塞处理 | SUPFEANEGOSW=ENABLE, FEATURELIST 须含 `UMCH` | Usage Monitoring 拥塞处理 |

> 动态协商开关不依赖 FEATURELIST 是否配置；但开启后若 PCRF 不下发或下发错误 FEATURELIST，可能导致用户激活失败或 Feature 不可用，需谨慎开启。RELEASE8/9/10 为必选项，至少设置一个。

### 决策点：是否按 PCRF 接收能力做 Gx 流控（WAL）（DP 0-02094）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不限流（默认） | WALVALUE 不配（默认 0） | PCRF 处理能力充足 |
| 限流 | WALVALUE=本端规划具体值（如 1000） | 周边接口流控、PCRF 过载保护（CCR-T 不受窗口限制） |

## 约束

- **一致性**（warning，rule-0-02231）：一致性：HOSTNAME/REALMNAME 必须与对端 `ADD DIAMPEERADDR` 中配置的主机名/域名一致；该参数错误会导致 PCRF 连接不能建立、PCC 策略控制特性无法使用
- **VPN 一致性**（warning，rule-0-02232）：VPN 一致性：PCRF 绑定的 VPN 实例必须与 Gx 逻辑接口绑定的 VPN 实例一致
- **组内 Feature 一致性**（warning，rule-0-02233）：组内 Feature 一致性：加入同一 PCRF Group（`ADD PCRFGROUP`）的各 PCRF 支持的 Feature 能力必须一致
- **规格上限**（warning，rule-0-02234）：规格上限：最大记录数 100
- **生效时机与修改路径**（info，rule-0-02235）：生效时机与修改路径：命令执行后立即生效；变更 PCRF 标识需先 `RMV PCRF` 删除再 `ADD PCRF` 重配（不支持原位改 HOSTNAME/REALMNAME）
