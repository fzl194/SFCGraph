# 配置用户位置组（ADD USRLOCATIONGRP）

> 命令静态知识见 [ADD USRLOCATIONGRP](command/UNC/20.15.2/ADD-USRLOCATIONGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD USRLOCATIONGRP 是 PGW-C/SMF 上"用户位置组"的原子配置入口，逻辑为"建组 + 逐条位置绑到组"：先建位置组（LOCGROUPNAME），再对组内每个位置（[ADD USRLOCATION](task/UNC/20.15.2/0-00210.md) 已建）逐条执行 ADD USRLOCATIONGRP 绑定。一个位置组最多 20 条位置，整机最多 1000 条位置组。该命令的参数只有 LOCGROUPNAME（必选，建组主键）+ LOCATIONNAME（可选，每次绑一条位置），结构相对简单但调用次数随位置数线性增长。

### 配置维度 1：位置组名称（参数 LOCGROUPNAME）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| 字符串 1~63 字符，不支持空格，不区分大小写 | 位置组主键（记录唯一标识） | 通用；下游 MOD/ADD UPBINDUPG 的 LOCGROUPNAME 引用本值 |

### 配置维度 2：位置绑定（参数 LOCATIONNAME）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| 字符串 1~63 字符，不支持空格，不区分大小写 | 单条位置名称（引用 [ADD USRLOCATION](task/UNC/20.15.2/0-00210.md)） | 每位置执行一次本命令，逐条追加到同一 LOCGROUPNAME |

### 配置维度 3：位置组的引用面（参数联动维度，非本命令字段）

| 引用方 | 引用参数 | 作用 |
|---|---|---|
| [MOD UPBINDUPG](task/UNC/20.15.2/0-00212.md) / [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) | `LOCGROUPNAME`（UPBINDTYPE=SPECIFIC 时可选） | 把位置组作为 UsrProfGroup 下 UserProfile 的精匹配条件之一（按位置策略选择） |
| [SET PCCFUNC](task/UNC/20.15.2/0-00193.md) 等 PCC 体系 | 间接通过 UPBINDUPG | 位置策略控制的基础数据源 |

> 步骤位置：位置策略控制链的第二步（[1-00019 配置用户位置与模板绑定](task/UNC/20.15.2/1-00019.md)），先建位置（[0-00210 ADD USRLOCATION](task/UNC/20.15.2/0-00210.md)），再逐位置绑到组，最后 [MOD UPBINDUPG](task/UNC/20.15.2/0-00212.md) / [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) 引用位置组。

## 决策点

### 决策点：建组 + 单条 vs 多条绑定（DP 0-04031）

问：一个位置组内要绑几条位置？是否首次建组？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅建组（不绑位置） | 仅 `LOCGROUPNAME`，不设 `LOCATIONNAME` | 先建占位组，后续按位置逐条 ADD USRLOCATIONGRP 绑入 |
| 单条位置绑入 | `LOCGROUPNAME` + `LOCATIONNAME` | 单位置组（如单 CGI 试点） |
| 多条位置逐条绑入 | 同一 `LOCGROUPNAME` + 不同 `LOCATIONNAME` 重复执行 N 次 | 多位置混合组（典型：2/3G CGI + 4G ECGI + 5G NCGI 三条） |

> 每条 ADD USRLOCATIONGRP 只绑一条位置；多位置场景下同 LOCGROUPNAME 重复执行 N 次，每次换 LOCATIONNAME。

### 决策点：位置组规格与规模（DP 0-04032）

问：当前 PLMN 内位置组与单组位置数量上限是多少？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单组内 ≤ 20 条位置 | 每次 ADD USRLOCATIONGRP 绑 1 条，组内累计 ≤ 20 | 单 UsrLocationGrp 规格上限 |
| 整机 ≤ 1000 条位置组 | 全网所有 UsrLocationGrp 累计 ≤ 1000 | 全局规模上限 |

> 单组 20 + 整机 1000 是硬上限；超出需拆分多组或评估必要性。

## 约束

- **位置名须先建**（critical，rule-0-04076）：LOCATIONNAME 必须是已通过 [ADD USRLOCATION](task/UNC/20.15.2/0-00210.md) 建好的位置记录，否则命令失败 — 后果：缺前置位置则执行报错，绑定失败
- **单组最多 20 条位置**（critical，rule-0-04077）：每个 UsrLocationGrp 最多支持 20 个 UserLocation，超过则命令执行失败 — 后果：超过 20 条需拆分组
- **整机最多 1000 条位置组**（critical，rule-0-04078）：系统最多支持配置 1000 条 UsrLocationGrp，超过则命令执行失败 — 后果：超过 1000 条需清理/合并历史组
- **位置名称不区分大小写且不支持空格**（warning，rule-0-04079）：LOCGROUPNAME/LOCATIONNAME 字符串 1~63 字符，不支持空格，不区分大小写 — 后果：含空格或超长直接报错；引用方（UPBINDUPG）按同一名称匹配
- **逐位置一条**（info，rule-0-04080）：单条 ADD USRLOCATIONGRP 仅绑 1 个 LOCATIONNAME，多位置场景需重复执行 — 后果：同 LOCGROUPNAME 多次调用，调用次数 = 位置数
