# DSP SMPDPNUM 原子任务
> 命令静态知识见 [DSP SMPDPNUM](command/UNC/20.15.2/DSP-SMPDPNUM.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：SGW-C、PGW-C、SMF、GGSN。本命令是容量/在线用户数统计入口——验证当前在线 PDP 上下文数与规划一致，常用于 FWA 接入、离线计费、Gy 在线计费场景的容量核对。

## 配置方法
按"查询范围 + 查询分类 + 过滤条件"组合统计。QRY_CLASS 省略表示查汇总；指定分类（如 APN/RAT/FWA 等）时按维度细分。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| QRY_SCOPE | 查询范围（默认 SUMMARY） | `SUMMARY`（汇总）/ `ALL_POD_INFO`、`SPECIFIED_POD_INFO`（按 POD） |
| QRY_CLASS | 查询分类（默认 SUMMARY，省略=汇总） | `APN`、`FWA`、`RAT` 等 |
| APN | 当按 APN 分类时 | `apn-test` |
| RAT | 无线接入类型 | `UTRAN` 等 |
| UPF_NAME | UPF 名称 | `upf-1` |
| SST | 切片/服务类型 | `0~255` |
| SD | 切片区分码 | `010203` |
| POD_ID | POD 名称（QRY_SCOPE=SPECIFIED_POD_INFO 时） | `pod-1` |
| ALIASAPN | 别名 APN | 按规划 |

典型脚本（FWA 场景核对当前在线 FWA 用户 PDP 上下文数）：
```
DSP SMPDPNUM: QRY_SCOPE=SUMMARY, QRY_CLASS=FWA;
```

调测语义：
- FWA 接入调测：核对当前在线 FWA 用户的 PDP 上下文数与实际规划一致。
- 离线计费（CG/抄送 CG 组/CG 绑定）与 Gy Diameter 在线计费调测：作为容量与在线用户数旁路核对命令，配合业务验证使用。

## 决策点（若有）

### 查询范围选择（QRY_SCOPE）
- 问什么：查汇总还是按 POD 维度。
- SUMMARY（默认）→ 汇总信息，最常用。
- ALL_POD_INFO → 查询所有 POD 的明细。
- SPECIFIED_POD_INFO → 需配合 POD_ID 查指定 POD。

### 查询分类选择（QRY_CLASS）
- 问什么：按哪个维度切分上下文数统计。
- 省略（默认 SUMMARY）→ 查汇总总数。
- FWA → 仅统计 FWA 用户（FWA 接入调测场景必选）。
- APN → 按 APN 统计，需配合 APN 参数。
- RAT → 按无线接入类型统计，需配合 RAT 参数。
- 切片维度 → 需配合 SST/SD。

## 约束（若有）
- QRY_CLASS 不输入时表示查询汇总信息（与 SUMMARY 等效为汇总呈现）。`severity=medium`
- "指定查询条件"指的是指定查询范围（ALL_POD_INFO、SPECIFIED_POD_INFO）。例如"指定查询条件的 PGW-C 上激活的 5G 承载上下文数"实际指的是该查询范围内的统计。`severity=medium`
- APN/UPF_NAME/SST/SD/RAT/ALIASAPN 等过滤参数仅在对应 QRY_CLASS 下生效，按数据规划取值。`severity=low`
