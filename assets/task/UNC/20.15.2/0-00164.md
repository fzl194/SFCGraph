# 配置按星期的费率类型（SET CHGWKDY）

> 命令静态知识见 [SET CHGWKDY](command/UNC/20.15.2/SET-CHGWKDY.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CHGWKDY 是 SGSN 离线计费的"费率类型层"：把"哪种计费属性用户 × 星期几"映射为当天按工作日还是休息日计费，供下游费率时段配置引用实现 S-CDR 话单的灵活费率切换。本命令的核心配置空间有 3 个维度：**星期**（必选，定位一周的某一天）、**计费属性**（必选，区分用户计费类别）、**费率类型**（必选，决定当天套哪条费率时段）。每种计费属性周记录固定 7 条，全表固定 28 条（4 计费属性 × 7 天）。

### 配置维度 1：星期（参数 WKDAY，必选）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| `MON` / `TUE` / `WED` / `THU` / `FRI` | 工作日定位（周一~周五） | 标准工作日费率 |
| `SAT` / `SUN` | 休息日定位（周六/周日） | 标准周末费率 |

> 每条命令只配一天；同一计费属性的周记录固定 7 条，逐天下发。

### 配置维度 2：计费属性（参数 CC，必选）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| `NORMAL` | 普通计费用户 | 默认离线计费 |
| `PREPAID` | 预付费用户 | 预付费场景 |
| `FLATRATE` | 包月制用户 | 包月套餐 |
| `HOTBILLING` | 实时计费用户 | 热计费场景 |

### 配置维度 3：费率类型（参数 TT，必选）

| 取值 | 作用 | 联动配置 |
|---|---|---|
| `WORK` | 当天按工作日费率计 | 对应 [ADD CHGTARI](task/UNC/20.15.2/0-00022.md) 工作日时段定义 |
| `WEEKEND` | 当天按休息日费率计 | 对应 [ADD CHGTARI](task/UNC/20.15.2/0-00022.md) 休息日时段定义 |

> 费率切换场景（如周五提前进入周末费率、周日切换为周一费率准备）通过把某天 TT 设为非默认值实现。

**典型脚本**：
```
SET CHGWKDY:WKDAY=FRI,CC=HOTBILLING,TT=WEEKEND;
SET CHGWKDY:WKDAY=MON,CC=NORMAL,TT=WORK;
SET CHGWKDY:WKDAY=SAT,CC=FLATRATE,TT=WEEKEND;
```

**步骤位置**：在所属 feature 编排中通常位于"离线计费参数"段——紧随费率时段配置（ADD CHGTARI）之后，可选与节假日配置（ADD CHGHOLI）联动。

## 决策点

### 决策点：每日费率类型（DP 0-03067）

问：某计费属性用户在某天按工作日还是休息日费率计费？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 工作日费率 | `TT=WORK` | 该天按工作日时段计费（默认工作日） |
| 休息日费率 | `TT=WEEKEND` | 该天按休息日时段计费（费率切换，如周五提前进入周末） |

> 取值由整网费率规划决定，必须与 [ADD CHGTARI](task/UNC/20.15.2/0-00022.md) 的时段定义配套；可选叠加 [ADD CHGHOLI](task/UNC/20.15.2/0-00019.md) 覆盖法定节假日。

### 决策点：计费属性覆盖范围（DP 0-03068）

问：本套星期费率需要覆盖哪些计费属性类别的用户？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单一类别 | 仅对一种 CC 配 7 条（如只配 NORMAL） | 场景仅涉及一种计费类别 |
| 多类别全配 | 对 NORMAL/PREPAID/FLATRATE/HOTBILLING 分别配 7 条 × N 类 | 多种计费属性并存的全套费率切换 |

## 约束

- **"星期(WKDAY) + 计费属性(CC)"组合唯一确定一条记录**（critical，rule-0-03166）："星期(WKDAY) + 计费属性(CC)"组合唯一确定一条记录，同一组合不能重复配置 — 后果：重复下发被拒绝，费率表无法建立
- **同一计费属性的周记录数固定为 7 条（周一~周日全配）**（warning，rule-0-03167）：同一计费属性的周记录数固定为 7 条（周一~周日全配），全表总记录数固定为 28 条（4 计费属性 × 7 天） — 后果：少配某天会导致该天费率类型未定义，话单费率异常
- **必须与 [ADD CHGTARI](task/UNC/20.15**（warning，rule-0-03168）：必须与 [ADD CHGTARI](task/UNC/20.15.2/0-00022.md) 费率时段配置联动才能实现灵活费率切换；可选与 [ADD CHGHOLI](task/UNC/20.15.2/0-00019.md) 节假日配置联动覆盖节假日费率 — 后果：缺时段定义则 TT 取值无意义
- **系统初次运行执行系统初始设置值**（info，rule-0-03169）：系统初次运行执行系统初始设置值，部署后存在初始记录 — 后果：仅按需修改偏离默认值参数
- **该命令执行后立即生效**（info，rule-0-03170）：该命令执行后立即生效 — 后果：无需重启，配置变更即时影响后续 S-CDR 话单费率
