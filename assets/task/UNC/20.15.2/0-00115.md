# MOD CHARGECHAR 原子任务
> 命令静态知识见 [MOD CHARGECHAR](command/UNC/20.15.2/MOD-CHARGECHAR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
按"用户归属维度（本地/漫游/拜访）"分别填计费属性。每个 `CCNAME` 下，HOME/ROAM/VISIT 三组参数各自独立——可只配关心的维度，也可叠加 Serving Node 维度（`HOMESGSN/ROAMSGSN/VISITSGSN`）。热计费（WSFD-011202）等场景用它把不同归属用户的话单生成方式区分开。

| 配置目标 | 关键参数 | 典型脚本 |
|---|---|---|
| 仅区分本地/漫游/拜访计费属性 | `CCNAME` + `HOME`/`ROAM`/`VISIT`（十六进制 bitmask） | `MOD CHARGECHAR:CCNAME="cc1",HOME="0x01",ROAM="0x02",VISIT="0x04";` |
| 叠加 Serving Node 计费属性 | 上述 + `HOMESGSN`/`ROAMSGSN`/`VISITSGSN`（枚举） | `MOD CHARGECHAR:CCNAME="cc1",HOME="0x01",HOMESGSN="...";` |
| 只改某一类用户（如仅漫游） | `CCNAME` + 单一维度 | `MOD CHARGECHAR:CCNAME="cc1",ROAM="0x02";` |

## 决策点

### 计费属性来源（UNC 本地 vs Serving Node 透传）
问：用户计费属性遵从哪一端配置？
- 遵从 UNC 本地配置 → 填 `HOME/ROAM/VISIT`（十六进制 bitmask，编码计费类型/话单生成方式）。
- 遵从 SGSN/SGW 上属性 → 填 `HOMESGSN/ROAMSGSN/VISITSGSN`（枚举，指明是否使用 Serving Node 计费属性）。
- 两者可并存：本端属性 + Serving Node 属性组合决定最终话单生成方式。

### 用户归属维度选定
问：本次修改覆盖哪些归属维度？
- 本地用户（HOME）+ 漫游用户（ROAM）= 本 PLMN 归属用户。
- 拜访用户（VISIT）= 拜访到本 PLMN 的用户。
- 三类可独立修改，按数据规划分别赋十六进制值（1~6 字符，`0x`/`0X` 前缀 + a-f/A-F/0-9）。

## 约束
- `CCNAME` 必选，1~31 字符。`severity=critical` 后果：属性集不存在则修改失败。
- `HOME/ROAM/VISIT` 为十六进制数据类型，1~6 字符，仅 `0x`/`0X`/`a-f`/`A-F`/`0-9`。`severity=high` 后果：非法格式被拒。
- `HOMESGSN/ROAMSGSN/VISITSGSN` 为枚举类型，取值需符合命令静态定义。`severity=medium`
- 本地用户和漫游用户统称本 PLMN 归属用户；三类用户属性语义不可混用。`severity=medium`
- 适用 NF：SGW-C、PGW-C、SMF。`severity=medium` 后果：非适用 NF 执行无效。
- 执行后立即生效。`severity=low`
