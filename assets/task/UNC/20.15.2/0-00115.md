# 修改计费属性区分本地/漫游/拜访用户（MOD CHARGECHAR）

> 命令静态知识见 [MOD CHARGECHAR](command/UNC/20.15.2/MOD-CHARGECHAR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"用户归属维度（本地/漫游/拜访）"分别填计费属性。每个 `CCNAME` 下，HOME/ROAM/VISIT 三组参数各自独立——可只配关心的维度，也可叠加 Serving Node 维度（`HOMESGSN/ROAMSGSN/VISITSGSN`）。热计费等场景用它把不同归属用户的话单生成方式区分开。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 仅区分本地/漫游/拜访计费属性 | `CCNAME` + `HOME`/`ROAM`/`VISIT` | 十六进制 bitmask（`0x`/`0X` 前缀 + a-f/A-F/0-9，1~6 字符） | 无 | 编码计费类型/话单生成方式 | 基础归属区分 |
| 叠加 Serving Node 计费属性 | 上述 + `HOMESGSN`/`ROAMSGSN`/`VISITSGSN` | 枚举类型 | 无 | 指明是否使用 Serving Node 计费属性 | 精细化归属区分 |
| 只改某一类用户（如仅漫游） | `CCNAME` + 单一维度 | 十六进制 bitmask | 无 | 独立修改某一类用户属性 | 增量修改 |

典型脚本：

```
MOD CHARGECHAR:CCNAME="cc1",HOME="0x01",ROAM="0x02",VISIT="0x04";
MOD CHARGECHAR:CCNAME="cc1",HOME="0x01",HOMESGSN="...";
MOD CHARGECHAR:CCNAME="cc1",ROAM="0x02";
```

## 决策点

### 决策点：计费属性来源（UNC 本地 vs Serving Node 透传）（DP 0-01268）

问：用户计费属性遵从哪一端配置？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 遵从 UNC 本地配置 | `HOME/ROAM/VISIT`（十六进制 bitmask） | 本端规划话单生成方式 |
| 遵从 SGSN/SGW 上属性 | `HOMESGSN/ROAMSGSN/VISITSGSN`（枚举） | Serving Node 透传场景 |
| 两者并存 | 本端 `HOME/ROAM/VISIT` + Serving Node 属性组合 | 混合场景 |

### 决策点：用户归属维度选定（DP 0-01269）

问：本次修改覆盖哪些归属维度？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 本地用户（HOME） | `HOME` 十六进制值 | 本 PLMN 归属用户 |
| 漫游用户（ROAM） | `ROAM` 十六进制值 | 本 PLMN 归属用户（出差/漫游） |
| 拜访用户（VISIT） | `VISIT` 十六进制值 | 拜访到本 PLMN 的用户 |
| 三类可独立修改 | 分别赋十六进制值（1~6 字符，`0x`/`0X` 前缀） | 按数据规划分别赋值 |

## 约束

· rule-0-01606 · severity=critical · `CCNAME` 必选，1~31 字符；属性集不存在则修改失败。

· rule-0-01607 · severity=high · `HOME/ROAM/VISIT` 为十六进制数据类型，1~6 字符，仅 `0x`/`0X`/`a-f`/`A-F`/`0-9`；非法格式被拒。

· rule-0-01608 · severity=medium · `HOMESGSN/ROAMSGSN/VISITSGSN` 为枚举类型，取值需符合命令静态定义。

· rule-0-01609 · severity=medium · 本地用户和漫游用户统称本 PLMN 归属用户；三类用户属性语义不可混用。

· rule-0-01610 · severity=medium · 适用 NF：SGW-C、PGW-C、SMF；非适用 NF 执行无效。
