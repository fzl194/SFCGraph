# 配置N40去活原因映射（ADD N40DIAGTRIGGER）

> 命令静态知识见 [ADD N40DIAGTRIGGER](command/UNC/20.15.2/ADD-N40DIAGTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD N40DIAGTRIGGER 在融合计费（Nchf N40 接口）场景中，把 SMF 内部诊断字段（DIAGNOSTICS，0~1000）映射到一个去活原因（RELEASETRIGGER）。该映射决定异常去活时 N40 `ChargingDataRelease Request` 消息里 `triggerType` 信元填 `FINAL` 还是 `ABNORMALRELEASE`。命令极简，只有 2 个维度且都必选：诊断值（键）、去活原因（值）。**本命令与 [SET CNVRGDCHGPARA](task/UNC/20.15.2/0-00165.md) 的 SPTABNTRIGGER 联合决定 triggerType，且本命令优先级更高。**

### 配置维度 1：诊断值（参数 DIAGNOSTICS，必选）

| 取值 | 作用 | 说明 |
|---|---|---|
| `0~1000` 的某整数 | 命中 SMF 内部去活诊断字段原因值，触发本命令的去活原因映射 | 整数类型；超出 0~1000 范围下发失败 |

> 数据来源：本端规划。典型取值如 `259`（CHF Notify 通知去活时的内部诊断值，见证据实例）。

### 配置维度 2：去活原因（参数 RELEASETRIGGER，必选）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| **FINAL**（正常去活） | 把该诊断值"提升"为正常去活处理，triggerType 填 FINAL | 把某些异常诊断值归类为正常去活（如 CHF 主动 Notify） |
| **ABNORMALRELEASE**（异常去活） | triggerType 填 AbnormalRelease | 保持异常去活语义并下发到 CHF |

> 命令 wiki 取值范围含 FINAL 与 ABNORMALRELEASE 两值；证据使用实例只见 FINAL。

**典型脚本（把诊断值 259 映射为 FINAL）**
```
ADD N40DIAGTRIGGER: DIAGNOSTICS=259, RELEASETRIGGER=FINAL;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤：配置 N40 去活原因映射"——融合计费 N40 接口场景的精细化补充，紧随 [SET CNVRGDCHGPARA](task/UNC/20.15.2/0-00165.md)（兜底开关）之后或并列。

## 决策点：诊断值命中后的去活原因（DP 0-02069）

问：命中本命令配置的某诊断值时，triggerType 填 FINAL 还是 ABNORMALRELEASE？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 正常去活 | `RELEASETRIGGER=FINAL` | 把某些异常诊断值（如 CHF 主动 Notify 的 259）归类为正常去活 |
| 异常去活 | `RELEASETRIGGER=ABNORMALRELEASE` | 保持异常去活语义并下发 CHF |

### 决策点：异常去活 triggerType 两层控制（DP 0-02070）

问：诊断值未命中本命令时，triggerType 由谁兜底决定？

| 选项 | 生效规则 | triggerType |
|---|---|---|
| 本命令命中 | 完全覆盖 SPTABNTRIGGER 行为 | 由 RELEASETRIGGER 决定（FINAL/ABNORMALRELEASE） |
| 本命令未命中 → 走 SET CNVRGDCHGPARA 兜底 | 看 SPTABNTRIGGER 开关 | SPTABNTRIGGER=ENABLE → ABNORMALRELEASE；DISABLE → FINAL |

> 规划时需明确两层职责：本命令做"精确诊断值映射"，SET CNVRGDCHGPARA 做"未命中兜底"，避免误把本应用于异常的场景全标成 FINAL。

## 约束

- ****优先级高于 SET CNVRGDCHGPARA****（critical，rule-0-02171）：**优先级高于 SET CNVRGDCHGPARA**：本命令命中诊断值时完全覆盖 SPTABNTRIGGER 的行为，规划时需明确两层职责，避免误把异常场景全标 FINAL
- ****取值范围****（warning，rule-0-02172）：**取值范围**：DIAGNOSTICS 取值 0~1000，超出范围下发失败
- ****RELEASETRIGGER 取值****（warning，rule-0-02173）：**RELEASETRIGGER 取值**：仅支持 FINAL 或 ABNORMALRELEASE，其他值下发失败
- ****最多 1001 条记录****（warning，rule-0-02174）：**最多 1001 条记录**：超过上限新增失败
- ****NF 适用****（info，rule-0-02175）：**NF 适用**：PGW-C、SMF（仅融合计费 N40 接口场景）；UDG 与非融合计费场景不用本命令
