# ADD N40DIAGTRIGGER 原子任务

> 命令静态知识见 [ADD N40DIAGTRIGGER](command/UNC/20.15.2/ADD-N40DIAGTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD N40DIAGTRIGGER 在融合计费（Nchf N40 接口）场景中，把 SMF 内部诊断字段（DIAGNOSTICS，0~1000）映射到一个去活原因（RELEASETRIGGER）。该映射决定异常去活时 N40 `ChargingDataRelease Request` 消息里 `triggerType` 信元填 `FINAL` 还是 `AbnormalRelease`。

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| DIAGNOSTICS | `0~1000` 的某整数 | 本端规划 | 内部诊断原因值，必填 |
| RELEASETRIGGER | `FINAL` | 本端规划 | 去活原因；当前证据仅见 FINAL（正常去活） |

**典型行（把诊断值 X 映射为正常去活）**

```
ADD N40DIAGTRIGGER: DIAGNOSTICS=xxx, RELEASETRIGGER=FINAL;
```

## 决策点

### 异常去活时 triggerType 由谁决定

异常去活的 `triggerType`（FINAL/AbnormalRelease）由两个命令联合控制，**ADD N40DIAGTRIGGER 优先级更高**：

| 命中情况 | 生效规则 | triggerType |
|---|---|---|
| 诊断值命中本命令配置 | 按 RELEASETRIGGER 配置 | 由本命令决定 |
| 诊断值未命中本命令 | 看 SET CNVRGDCHGPARA 的 SPTABNTRIGGER | SPTABNTRIGGER=ENABLE → AbnormalRelease；DISABLE → FINAL |

### 命中诊断值时的去活原因选择

RELEASETRIGGER 当前证据只见 FINAL 取值（即把某个异常诊断值"提升"为正常去活处理）。是否支持 AbnormalRelease 等其他取值需以命令 wiki 取值范围为准。

## 约束

- **立即生效** · severity=LOW。
- **最多 1001 条记录** · severity=MEDIUM · 超过上限新增失败。
- **优先级高于 SET CNVRGDCHGPARA**：本命令命中诊断值时完全覆盖 SPTABNTRIGGER 的行为 · severity=HIGH · 规划时需明确两层职责，避免误把本应用于异常的场景全标成 FINAL。
- **取值范围**：DIAGNOSTICS 0~1000，超出范围下发失败 · severity=MEDIUM。
- **适用 NF**：PGW-C、SMF（融合计费 N40 接口场景） · severity=MEDIUM。
