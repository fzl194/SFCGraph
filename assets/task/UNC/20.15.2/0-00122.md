# 修改 PCRF 基本信息（MOD PCRF）

> 命令静态知识见 [MOD PCRF](command/UNC/20.15.2/MOD-PCRF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按 HOSTNAME 定位 PCRF，按规划修改对端协商参数与传输属性。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|------|------|------|------|------|----------|
| PCRF 标识 | HOSTNAME | 字符串，1~127（已存在，必选） | — | 唯一标识目标 PCRF | 所有场景 |
| 对端协商 | SUPFEANEGOSW | `ENABLE` / `DISABLE` | `DISABLE` | Supported-Features 动态协商开关，与对端协商 | 需要动态协商 Feature 的场景 |
| Feature 列表 | FEATURELIST | 位域（如 `UMCH`） | 空 | 与对端协商使能的 Feature 位域 | 流量监控拥塞处理（UMCH）等场景 |
| AVP 携带开关 | CARRYSUPFEASW | 枚举（按 Gx 协议规划） | — | Supported-Feature AVP 携带开关 | 按 Gx 协议规划 conditional 配置 |
| VPN 实例 | VPNINSTANCE | 字符串，1~31（区分大小写） | 空 | PCRF 所属 VPN 实例 | 多 VPN 场景 |
| DSCP 值 | DSCPV | 0~63 或 255 | 0 | DSCP 标记值 | QoS 差异化场景 |
| WAL 值 | WALVALUE | 0~4294967295 | 0 | WAL 值，按链路规划 | 链路优先级配置 |

**典型脚本：**

```
MOD PCRF:HOSTNAME="pcrf_01",SUPFEANEGOSW=ENABLE,FEATURELIST="UMCH",CARRYSUPFEASW=ENABLE;
```

**步骤位置：** 前置需先 [ADD PCRF](task/UNC/20.15.2/0-00057.md) 建对端；本命令属**高危命令**——修改对端信息可能触发 Diameter 链路重建/中断。

## 决策点

### 决策点：Supported-Features 协商策略（DP 0-01282）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| SUPFEANEGOSW=`ENABLE` + FEATURELIST 指定位域 | 与对端动态协商，Feature 列表由对端能力决定 | 需要激活 UMCH 等特定 Feature（WSFD-109104 累计流量策略控制） |
| SUPFEANEGOSW=`DISABLE` + FEATURELIST 固定 | 禁用动态协商，使用本地静态配置 | 对端不支持协商或稳定运行场景 |

> notes：SUPFEANEGOSW（动态协商开关）+ FEATURELIST（位域，如 UMCH 流量监控拥塞处理）需与对端 PCRF 协商一致。这两个参数"获取方法"标注为"与对端协商"——具体使能哪些 Feature 取决于对端能力，存在跨场景分叉。

### 决策点：HOSTNAME 格式规范（DP 0-01283）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 按软参 BIT150=ON（区分大小写） | HOSTNAME 严格区分大小写 | 严格主机名区分场景 |
| 按软参 BIT150=OFF（不区分大小写） | HOSTNAME 不区分大小写 | 宽松匹配场景 |

> notes：HOSTNAME 字符串长度 1~127，必须是可见 ASCII 码，是否区分大小写由软参 BIT150 控制。同一物理 PCRF 在不同 NF 上的命名规则存在分叉。

## 约束

- **高危命令**（critical，rule-0-01642）：高危命令，执行后立即生效；可能导致 PCRF 链路闪断，影响 PCC 策略控制（用户可能被去活），操作前需评估影响

- **修改特定 PCRF 时必须指定 HOSTNAME**（critical，rule-0-01643）：修改特定 PCRF 时必须指定 HOSTNAME，不支持通配；必须确认目标 PCRF 存在

- **PCRF 绑定到 PCRFGroup 时**（critical，rule-0-01644）：PCRF 绑定到 PCRFGroup 时，修改该 PCRF 的动态协商参数会同时修改该 PCRFGROUP 下所有其他 PCRF 的协商参数；若该组下某个 PCRF 同时被其他 PCRFGROUP 绑定，那些组的 PCRF 协商参数也会被刷新——操作前需确认所有受影响 PCRF

- **只能修改已 ADD 成功的 PCRF**（warning，rule-0-01645）：只能修改已 ADD 成功的 PCRF；目标不存在则报错，建议操作前用 LST PCRF 校验

- **HOSTNAME 1~127（可见 ASCII**（warning，rule-0-01646）：HOSTNAME 1~127（可见 ASCII，BIT150 控制大小写）；VPNINSTANCE 1~31 区分大小写；DSCPV 0~63 或 255；WALVALUE 0~4294967295；名称均不支持空格
