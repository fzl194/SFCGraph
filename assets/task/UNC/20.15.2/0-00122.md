# MOD PCRF 原子任务
> 命令静态知识见 [MOD PCRF](command/UNC/20.15.2/MOD-PCRF.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用网元：PGW-C、GGSN。本命令修改特定 PCRF 的基本信息，属**高危命令**——修改对端信息可能触发 Diameter 链路重建/中断，进而影响用户使用业务（如用户被去活）。前置必须先 [ADD PCRF](task/UNC/20.15.2/0-00057.md) 建对端。

## 配置方法
按 HOSTNAME 定位 PCRF，按规划修改对端协商参数与传输属性。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| HOSTNAME | PCRF 主机名（必选，1~127，已存在） | `pcrf_01` |
| SUPFEANEGOSW | Supported-Features 动态协商开关（可选，与对端协商） | `ENABLE` |
| FEATURELIST | Feature 列表（位域，与对端协商） | `UMCH`（PCRF 需支持 UMCH） |
| CARRYSUPFEASW | Supported-Feature AVP 携带开关（conditional，枚举） | 按 Gx 协议规划 |
| VPNINSTANCE | VPN 实例（可选，1~31，区分大小写） | 已规划 VPN 名 |
| DSCPV | DSCP 值（可选，0~63 或 255） | `46` |
| WALVALUE | wal 值（可选，0~4294967295） | 按链路规划 |

## 决策点

### Supported-Features 协商开关 + Feature 列表（与对端协商）
SUPFEANEGOSW（动态协商开关）+ FEATURELIST（位域，如 UMCH 流量监控拥塞处理）需与对端 PCRF 协商一致。这两个参数"获取方法"标注为"与对端协商"——具体使能哪些 Feature 取决于对端能力，存在跨场景分叉（典型用于 WSFD-109104 累计流量策略控制激活 UMCH）。

### HOSTNAME 由软参 BIT150 控制格式
notes：HOSTNAME 字符串长度 1~127，必须是可见 ASCII 码，是否区分大小写由软参 BIT150 控制。同一物理 PCRF 在不同 NF 上的命名规则存在分叉。

## 约束
- **高危 + 立即生效**：执行后立即生效；可能导致 PCRF 链路闪断，影响 PCC 策略控制（用户可能被去活）。
- **必须指定 HOSTNAME**：修改特定 PCRF 时一定要输入指定 PCRF 主机名，不支持通配。
- **PCRFGROUP 联动刷新**：PCRF 绑定到 PCRFGroup 时，修改该 PCRF 的动态协商参数会**同时修改该 PCRFGROUP 下所有其他 PCRF 的协商参数**；若该组下某个 PCRF 同时被其他 PCRFGROUP 绑定，那些组的 PCRF 协商参数也会被刷新。
- **前置存在性**：只能修改已 ADD 成功的 PCRF；不存在则报错。
- **名称规格**：HOSTNAME 1~127（可见 ASCII，BIT150 控制大小写）；VPNINSTANCE 1~31 区分大小写；DSCPV 0~63 或 255；WALVALUE 0~4294967295。
