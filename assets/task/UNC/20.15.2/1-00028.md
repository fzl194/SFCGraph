# UNC DHCP/DHCPv6 服务器组链路

> **UNC 域 DHCP 族共享底座**（SOP §3.5 域通用）。被引用于 **WSFD-104413**（DHCPv4）+ **WSFD-104005**（DHCPv6）2 个特性。
>
> **★ R1.5 待补 atom（critical，SOP §6）**：本骨架 3 条命令（`ADD DHCPSERVERGRP` / `ADD DHCPSERVER` / `ADD DHCPBINDPOOLGRP`）的 atom 在 UNC pre-build 阶段未建（_numbering.json 未收录）。配置生成遇 WSFD-104413/WSFD-104005 这些步骤时**暂不产出**，等 atom 补齐后回填。Evidence = WSFD-104413 参考信息命令清单。

## 配置方法

UNC 控制面作为 DHCP Client 代理从外部 DHCP Server 拉地址——主链路 3 步骤。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 建 DHCP/DHCPv6 服务器组 | `ADD DHCPSERVERGRP` | DHCPSVRGRPNAME（全网规划）；IPVERSION=IPv4/DHCPv4、IPv6/DHCPv6 |
| 2. 添加 DHCP/DHCPv6 服务器 | `ADD DHCPSERVER` | DHCPSVRGRPNAME；SERVERIPV4=DHCP Server IP / SERVERIPV6=DHCPv6 Server IPv6 地址；VPNINSTANCE |
| 3. 绑地址池组到服务器组 | `ADD DHCPBINDPOOLGRP` | DHCPSVRGRPNAME；POOLGRPNAME（已在 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 步骤 3 配）|

**步骤位置**：在 [UNC 1-00023](task/UNC/20.15.2/1-00023.md)（地址池体系）步骤 4 POOLBINDGRP 建好池组之后；[WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md) / [WSFD-104005](feature/UNC/20.15.2/WSFD-104005.md) 配置流程步骤 4。

## 场景差异

- **[WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md) DHCP 功能**（IPv4 单栈场景）：步骤 1 `IPVERSION=IPv4`；步骤 2 `SERVERIPV4`；步骤 3 绑 IPv4 池组（POOLTYPE=DHCP 的 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) IPv4 池组）
- **[WSFD-104005](feature/UNC/20.15.2/WSFD-104005.md) DHCPv6 地址分配**（IPv6 单栈场景）：步骤 1 `IPVERSION=IPv6`；步骤 2 `SERVERIPV6`；步骤 3 绑 IPv6 池组（[UNC 1-00023](task/UNC/20.15.2/1-00023.md) IPv6 池组）

## 决策点

| 选项 | 影响（参数） |
|---|---|
| DHCPv4 子方式 | 步骤 1 IPVERSION=IPv4；步骤 2 SERVERIPV4 |
| DHCPv6 子方式 | 步骤 1 IPVERSION=IPv6；步骤 2 SERVERIPV6 |

## 约束

- **本骨架不含独立命令外的命令**：DHCP 客户端代理 IP（`ADD AGENTIP`）归 WSFD-104413/WSFD-104005 步骤 3；DHCP 请求信元参数（`SET DHCPPARAREQ`）归步骤 6（可选）；地址池组归 [UNC 1-00023](task/UNC/20.15.2/1-00023.md)
- **DHCP 服务器可达性**（critical）：DHCP 服务器必须可达——UNC 通过 `ADD AGENTIP` 代理 IP 在地址池网段内与外部 DHCP Server 通信，需保证 `ALM-100466` 不告警
- **ADDRESS POOLGROUP 必须已建**（critical）：步骤 3 `ADD DHCPBINDPOOLGRP` 的 POOLGRPNAME 必须已在 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 配好 POOLTYPE=DHCP 的池组
