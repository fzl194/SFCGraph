# 配置流过滤器（ADD FLOWFILTER）

> 命令静态知识见 [ADD FLOWFILTER](command/UNC/20.15.2/ADD-FLOWFILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD FLOWFILTER 是单参数命令，只起"命名一个流过滤器对象"的作用，下游 `ADD RULE`（POLICYTYPE=ULCL 等）和 `ADD ADCPARA` 通过 FLOWFILTERNAME 反向引用它。1 个核心维度：**流过滤器名称口径**（场景驱动）。

### 配置维度 1：流过滤器名称（参数 FLOWFILTERNAME，必选）

| 场景类型 | 取值口径 | 一致性要求 | 典型取值样例 |
|---|---|---|---|
| ULCL 分流（WSFD-108002 / 223001 / 223003 / 223101） | 与对端协商 | 须与 UPF/PGW-U 上规划的流过滤器一致 | `ulcl` |
| ADC 应用检测（WSFD-109102） | 本端规划 | SMF、PGW-U/UPF、PCRF/PCF 三网元必须一致 | `flowfilter_ADC01` |
| 业务触发 RADIUS（WSFD-011306） | 与对端协商 | 须与 UPF/PGW-U 规划一致 | `flowfilter_1` |

> FLOWFILTERNAME 字符串 1~63，不支持空格，不区分大小写。此参数对应 UPF 上 ADD FLOWFILTERGRP 的"流过滤器组名称"或 ADD FLOWFILTER 的"流过滤器名称"，SMF 与 UPF 须协商一致。

**典型脚本**（按场景）：
```
// ULCL 分流场景
ADD FLOWFILTER: FLOWFILTERNAME="ulcl";
// ADC 应用检测场景
ADD FLOWFILTER: FLOWFILTERNAME="flowfilter_ADC01";
// 业务触发 RADIUS 场景
ADD FLOWFILTER: FLOWFILTERNAME="flowfilter_1";
```

**步骤位置**：在分流/ADC/RADIUS 编排中位于"命名锚点"——先于 `ADD RULE`（POLICYTYPE=ULCL）和 `ADD ADCPARA`，是规则反向引用的前置对象。

## 决策点：流过滤器名称取值口径（DP 0-02043）

问：FLOWFILTERNAME 由"本端规划"还是"与对端协商"决定？

| 选项 | 联动一致性要求 | 适用场景 |
|---|---|---|
| 与对端协商 | 须与 UPF/PGW-U 上规划的流过滤器一致 | ULCL 分流、业务触发 RADIUS |
| 本端规划 | SMF、PGW-U/UPF、PCRF/PCF 三网元必须一致 | ADC 应用检测 |

### 业务场景识别（DP 0-02044）

问：本流过滤器服务哪类业务？决定下游引用方式。

| 选项 | 下游引用 | 适用场景 |
|---|---|---|
| ULCL 分流 | 被 ADD RULE（POLICYTYPE=ULCL）引用 | 上行分类 ULCL、位置分流、漫游地分流 |
| ADC 应用检测 | 被 ADD ADCPARA 引用，匹配 PCF 下发的 ADC 规则 AppId | ADC 基本功能（WSFD-109102） |
| 业务触发 RADIUS | 被 ADD RULE 引用 | 业务触发 RADIUS 上报（WSFD-011306） |

## 约束

- rule-0-02106 · severity=critical · 前置依赖：先建本流过滤器，下游 ADD RULE / ADD ADCPARA 才能引用 FLOWFILTERNAME，否则引用失败
- rule-0-02107 · severity=high · 跨网元一致性：FLOWFILTERNAME 对应 UPF 上 ADD FLOWFILTERGRP/FLOWFILTER 的名称，SMF 与 UPF 须协商一致，否则流匹配失败
- rule-0-02108 · severity=high · 唯一性：不同流过滤器之间 FLOWFILTERNAME 不能重复，否则引用歧义、规则匹配错乱
- rule-0-02109 · severity=warning · 规格上限：最大记录数 100000；达 98% 上报 `ALM-135602215 配置数量超阈值`，95% 恢复（阈值可由 MOD CFGTHRESHOLD 修改）
- rule-0-02110 · severity=info · 生效时机：执行后立即生效
