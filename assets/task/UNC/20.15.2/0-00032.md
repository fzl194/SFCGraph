# ADD FLOWFILTER 原子任务
> 命令静态知识见 [ADD FLOWFILTER](command/UNC/20.15.2/ADD-FLOWFILTER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令只起"命名一个流过滤器对象"的作用，下游 `ADD RULE`（POLICYTYPE=ULCL 等）和 `ADD ADCPARA` 通过 FLOWFILTERNAME 反向引用它。单参数命令。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| FLOWFILTERNAME | 流过滤器名称。各场景取值口径不同，见下表 | `ulcl`（分流）/ `flowfilter_ADC01`（ADC）/ `flowfilter_1`（RADIUS） |

典型脚本（按场景）：
```
// 分流场景（ULCL/位置/漫游地）
ADD FLOWFILTER: FLOWFILTERNAME="ulcl";
// ADC 场景
ADD FLOWFILTER: FLOWFILTERNAME="flowfilter_ADC01";
// RADIUS 业务触发
ADD FLOWFILTER: FLOWFILTERNAME="flowfilter_1";
```

## 决策点

### 流过滤器名称的取值口径
FLOWFILTERNAME 的来源在不同业务特性下不一致（自动比对："与对端协商"5 处 vs "本端规划"1 处），需按场景决定：

| 业务场景 | 取值口径 | 一致性要求 |
|---|---|---|
| ULCL 分流（WSFD-108002 / 223001 / 223003 / 223101） | 与对端协商 | 须与 UPF/PGW-U 上规划的流过滤器一致 |
| ADC 应用检测（WSFD-109102） | 本端规划 | GGSN-C/PGW-C/SMF、PGW-U/UPF、PCRF/PCF 三网元必须一致 |
| 业务触发 RADIUS（WSFD-011306） | 与对端协商 | 须与 UPF/PGW-U 规划一致 |

## 约束
- 唯一性：不同流过滤器之间 FLOWFILTERNAME 不能重复。`severity=high` · 后果：引用歧义，规则匹配错乱。
- 前置/后置依赖：先建本流过滤器，下游 `ADD RULE`、`ADD ADCPARA` 才能引用其名称。`severity=high` · 后果：引用失败。
- 规格上限：最大记录数 100000；达 98% 上报 `ALM-135602215 配置数量超阈值`，95% 恢复（阈值可由 `MOD CFGTHRESHOLD` 修改）。`severity=medium`
- 生效时机：执行后立即生效。`severity=low`
