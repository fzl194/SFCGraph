# 离线计费（WSFD-011201）

> 特性静态知识见 [WSFD-011201](feature/UNC/20.15.2/WSFD-011201.md)。本页讲配置生成实例化时这个特性**怎么配**——编排哪些步骤、4 种配置层次如何选。

## 配置概览

核心对象链：**OFCTemplate**（离线计费模板：话单版本+计费方式+阈值+话单触发）→ 绑定到 `{UserProfile | APN | 计费属性CC | 全局}` 之一。

4 种配置方法（层次）**共享同一套 OFCTemplate 参数骨架**（[1-00001](task/UNC/20.15.2/1-00001.md)），差异仅在"绑到哪个对象"——**不建 4 套流程**，由特性级 DP（配置层次选择）驱动走哪个绑定 compound。

## 配置流程（步骤编排，每步显示含命令）

1. **配置模板参数**（话单版本+计费方式+阈值+话单触发，通用骨架）：`ADD OFCTEMPLATE` + `SET OFCTHRESHOLD` + `SET CDRTRIGGER` + `SET CONTAINERTRIGGER` → [1-00001](task/UNC/20.15.2/1-00001.md)
2. **绑到配置层次**（按 DP 选，四选一）：
   - UserProfile 层（用户级参数）：`ADD USERPROFILE` + `SET USRPROFCHARGE` + `ADD USRPROFGROUP` + `ADD UPBINDUPG` + `ADD APNUSRPROFG` → [1-00002](task/UNC/20.15.2/1-00002.md)
   - APN 层（APN 级参数）：`ADD APN` + `SET APNCHARGECTRL` → [1-00003](task/UNC/20.15.2/1-00003.md)
   - 计费属性 CC 层（CC 级参数）：`ADD CHARGECHAR` + `ADD GLBOFCTEMPLATE` → [1-00004](task/UNC/20.15.2/1-00004.md)
   - 全局层（兜底，单命令）：`ADD GLBOFCTEMPLATE` → [0-00033](task/UNC/20.15.2/0-00033.md)

## 决策点（特性级：配置层次选择）

| 配置层次 | 走法 | 优先级 | 关键联动 |
|---|---|---|---|
| UserProfile | 骨架 [1-00001](task/UNC/20.15.2/1-00001.md) + 绑定 [1-00002](task/UNC/20.15.2/1-00002.md) | 最高 | 用户级离线参数；未配则降级取 APN |
| APN | 骨架 1-00001 + 绑定 [1-00003](task/UNC/20.15.2/1-00003.md) | 次高 | APN 级；未配则降级取 CC |
| 计费属性 CC | 骨架 1-00001 + 绑定 [1-00004](task/UNC/20.15.2/1-00004.md) | 次低 | CC 级；未配则降级取全局 |
| 全局 | 骨架 1-00001 + 直接 atom [0-00033](task/UNC/20.15.2/0-00033.md) | 最低 | 兜底；所有未配层次的最终取值 |

> **走法列**：UserProfile/APN/CC 三层走多命令绑定 compound；全局层单命令直接 atom（§3.5）。4 层次是**降级链**而非并列。

## 约束（特性级 rule）

- **模板名是引用键**（critical）：OFCTEMPLATENAME 须先建（[1-00001](task/UNC/20.15.2/1-00001.md)）再绑定 — 否则绑定命令找不到模板。
- **4 层次降级链**（warning）：UserProfile > APN > CC > 全局，高一级未配取下一级。
- **话单版本跨网元一致**（warning）：全网规划；GUL 切换网络须 R8+。
- **CG 对接前置**（info）：须先完成"配置到 CG 的数据"（见特性 wiki）。
