# UNC MPLS VPN 基础设施（VPNINST+VPNINSTAF+IPBINDVPN+LOGICIP/LOGICINF）

> **UNC 域 MPLS VPN 接入方式共享底座**（SOP §3.5 域通用）。被引用于 **WSFD-104411**（MPLS VPN 接入方式特性，能力型底座特性）。
>
> **能力型底座特性（§3.7 范式）说明**：WSFD-104411 MPLS VPN 原始产品文档**仅有基本概念和实现原理文档（9 个 md），无「激活」/「特性配置」子目录**——MPLS VPN 的实际配置承载在 UNC 各 L3VPN 接入特性中（典型：L3VPN 实例 + VPN 实例 + 逻辑接口绑 VPN + 接口绑 VPN），由用户按实际组网（Hub&Spoke / Multi-VPN-Instance CE / 跨域 VPN / BGP_MPLS IPv6 VPN / VPN GR / VPN NSR / MPLS VPN 的分标签方式）选用本骨架的命令族配置。
>
> **C-U 对应**：UNC 控制面 MPLS VPN，UDG 用户面 MPLS VPN 端**结构同构但命令族独立维护**。UNC 与 UDG MPLS VPN 是**两个独立产品的资产**，不复用 md。

## 配置方法

### 步骤维度：VPN 实例 → 地址族 → 逻辑接口绑 VPN → 接口绑 VPN

本 compound 建立完整的 MPLS VPN 基础设施骨架：

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. L3VPN 实例 | `ADD L3VPNINST` → [0-00040](task/UNC/20.15.2/0-00040.md) | VRFNAME=vpn 名 |
| 2. L3VPN 地址族 | `ADD VPNINSTAF` → [0-00099](task/UNC/20.15.2/0-00099.md) | VRFNAME, AFTYPE=ipv4uni/ipv6uni, VRFRD=100:1（RT 路由标识）|
| 3. VPN 实例属性扩展 | `MOD VPNINSTAF` → [0-00133](task/UNC/20.15.2/0-00133.md) | VRFNAME, AFTYPE（用于变体场景如 Hub&Spoke 的 RT 调整、IPv6 VPN 的特殊属性）|
| 4. 接口绑 VPN | `ADD IPBINDVPN` → [0-00237](task/UNC/20.15.2/0-00237.md) | IFNAME=外联口, VRFNAME |
| 5. 逻辑接口 + 逻辑地址（可选）| `ADD LOGICINF` → [0-00043](task/UNC/20.15.2/0-00043.md) + `ADD LOGICIP` → [0-00044](task/UNC/20.15.2/0-00044.md) | 逻辑接口名 + IP（按需）|

**典型脚本（基本 MPLS VPN）**：
```
ADD L3VPNINST: VRFNAME="vrf1";
ADD VPNINSTAF: VRFNAME="vrf1", AFTYPE=ipv4uni, VRFRD="100:1";
ADD IPBINDVPN: IFNAME="Ethernet1/0/0", VRFNAME="vrf1";
ADD LOGICINF: LOGICIFNAME="logif1", VRFNAME="vrf1";
ADD LOGICIP: LOGICIFNAME="logif1", IPADDR="10.1.1.1", MASKLEN=24;
```

**步骤位置**：MPLS VPN 接入方式的**核心骨架**——前置 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施（VPN+APN），后续由 feature 级 DP（组网变体）驱动 Hub&Spoke / Multi-VPN-Instance CE / 跨域 VPN 等变体。

## 约束

- **VPN 三者一致**（critical）：L3VPN/VPNINST 的 VRFNAME、ADD APN 的 VPNINSTANCE、外联口绑定的 VPN 三者必须一致 — 后果：VPN 不一致致地址分配或路由异常。
- **VRFRD 全网规划**（critical）：VRFRD（RT 路由标识）全网规划统一 — 后果：VRF 路由无法跨设备互通。
- **AFTYPE 与 IP 类型匹配**（warning）：IPv4 场景 AFTYPE=ipv4uni；IPv6 场景 AFTYPE=ipv6uni；双栈两者都配 — 后果：地址族不匹配致 VPN 实例失效。
- **本 compound 不含独立命令外的命令**：完整 MPLS VPN 接入域特性配置由 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施 + [UNC 1-00029](task/UNC/20.15.2/1-00029.md) OSPF 路由发布 + MPLS/LDP/BGP 等命令族共同承担。
- **能力型底座特性（§3.7）**：本骨架的"配置方法"段仅承载**命令族与典型脚本**，**不内嵌具体组网变体的命令序列**——变体由 [2-00031](task/UNC/20.15.2/2-00031.md) 通过 DP 驱动。
