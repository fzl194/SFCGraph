# SET POLICYMODE 原子任务
> 命令静态知识见 [SET POLICYMODE](command/UNC/20.15.2/SET-POLICYMODE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是策略接口（Gx/Npcf）选择的**全局层**。优先级：用户漫游属性（ADD ROAMCHGMODE）> APN/DNN（ADD APNPOLICYMODE）> 全局（本命令）。用于在 EPC+5GC 融合组网下，按"终端接入类型"选择走 Gx（PCRF）还是 N7/Npcf（PCF）。

## 配置方法

按"终端接入类型 + 策略接口 + 互操作指示"三元组配置。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| TMACCTYPE | 指定终端和接入类型（必选，键值） | `UE5G_RAT4G`（5G 终端 4G 接入）等 |
| FORCED | 指定该终端接入类型使用的策略接口 | `Npcf`（N7/PCF） / `Gx`（PCRF） |
| BY5GSIWKI | 是否按 5GS 互操作指示选择策略接口 | `DISABLE` |
| BYIWKUEN5GR4G | 非 5G 终端 4G 接入时是否按 5GS 互操作指示选择 | `DISABLE`（与 FORCED 联动同步） |
| BY5GCNRI | 是否按 5GC 无限制接入标识选择策略接口 | `DISABLE` |
| PCFRESELBYPCFID | 是否基于 PCF 实例标识决策策略接口类型 | `TRUE` / `FALSE` |

典型脚本（5G 终端 4G 接入走 Npcf，融合组网）：
```
SET POLICYMODE: TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=DISABLE, FORCED=Npcf;
```
典型脚本（5G 终端 4G 接入仍走 Gx，传统 EPC）：
```
SET POLICYMODE: TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=DISABLE, FORCED=Gx;
```

## 决策点

### 该终端接入类型走 Gx 还是 Npcf
- 问什么：在融合组网下，指定 TMACCTYPE 对应的策略接口。
- 5G 终端在 5G 接入 → 必须使用 Npcf 接口。
- 5G 终端 4G 接入（UE5G_RAT4G）→ 视组网选 Npcf（对接 PCF）或 Gx（对接 PCRF）。
- 2G/3G 接入使用 Npcf → 需提前和对端 PCF 确认是否支持，否则激活失败。
- 配置不当会选错 PCRF/PCF，用户无法正常激活。

### 互操作指示二次决策（BY5GSIWKI / BYIWKUEN5GR4G）
- 问什么：是否在 FORCED 之外，再按 5GS 互操作指示或非 5G 终端 4G 接入场景二次决策接口。
- 默认 DISABLE；使能后按用户实际的 5GS 互操作指示决定。
- 联动：非 5G 终端 4G 接入下修改 FORCED，系统会同步修改 BYIWKUEN5GR4G 取值。

### 策略接口优先级回落
- 当未配指定 APN 的策略接口、未配基于 PCF 实例标识的策略接口时，回落到本命令全局配置；系统初始值为 Gx。

## 约束
- 生效时机：执行后只对新激活用户生效，不影响在线用户。`severity=medium`
- 高危：配置不当导致选错 PCRF/PCF 接口，用户无法正常激活。`severity=high`
- 互操作不变：4G/5G 互操作流程的策略接口不变；2/3/4G 互操作流程的策略接口不变。`severity=medium`
- 2G/3G 用 Npcf 前置确认：必须提前和对端 PCF 确认支持。`severity=high`
- 联动同步：非 5G 终端 4G 接入下改 FORCED，BYIWKUEN5GR4G 被同步修改。`severity=medium`
- 优先级：低于 ADD ROAMCHGMODE（用户漫游属性），高于全局默认；同级可被 ADD APNPOLICYMODE（APN 级）覆盖。`severity=medium`
- TMACCTYPE 必选、FORCED/BY5GSIWKI/BYIWKUEN5GR4G/BY5GCNRI 为 conditional（不输入则保持现状）。`severity=low`
