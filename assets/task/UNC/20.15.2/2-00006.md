# ADC基本功能（WSFD-109102）

> 特性静态知识见 [WSFD-109102](feature/UNC/20.15.2/WSFD-109102.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系**：带宽控制族首个（第 6 批）。ADC 基本功能是**应用级策略触发通道**——UNC 侧承担控制面中继角色（转发检测信息 + 承载/QoS Flow 管理），与 PCC 基本功能（[2-00005](task/UNC/20.15.2/2-00005.md)）正交：ADC 提供"应用检测上报"维度，PCC 提供"策略获取通道"。本特性**依赖** PCC 基本功能（License 82207979），须先完成 PCC 激活。用户面检测由 UDG 侧 GWFD-020357（增强的 ADC 基本功能）执行，本特性仅配 UNC 侧。带宽控制族后续特性（BWM/FUP 等）可复用本特性的 ADC 检测上报链路触发策略变更。

## 配置概览

核心对象链（按规则下发方式分两条路径，二选一）：

**路径 A：PCRF/PCF 动态规则下发 appid（任务示例一）**：
**License（LKV2BADCF01）** → **FLOWFILTER（appid 锚点）** → **ADCPARA（流信息上报开关）** → PCRF/PCF 动态下发携带 appid 的 PCC 规则 + APPLICATION_START/STOP（APP_STA/APP_STO）Event-Trigger → UNC 透传 UPF → UPF 检测应用并上报。

**路径 B：PCRF/PCF 预定义规则下发 Event-Trigger（任务示例二）**：
**License（LKV2BADCF01）** → **RULE（POLICYTYPE=ADC）** → **USERPROFILE（UPTYPE=PCC）** → **RULEBINDING** → PCRF/PCF 下发预定义规则 APP_STA/APP_STO Event-Trigger → SMF 本地 RULENAME 匹配 → UNC 透传 UPF → UPF 检测应用并上报。

关键差异维度：① 规则下发方式（动态 appid vs 预定义规则 Event-Trigger）② 流信息上报（仅动态路径的 ADCPARA.FLOWINFORPT 控制）③ 接口代际（Gx 4G/5G 融合 vs N7 纯 5G，仅影响信元命名，不影响 UNC 配置命令）。

## 配置流程（步骤编排，每步显示含命令）

1. **激活 ADC License**（前置）：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)
   - 关键参数：LICITEM="LKV2BADCF01", SWITCH=ENABLE
   - **依赖**：须先完成 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md) 激活（License 82207979）— ADC 规则无 PCC 通道则无法承载
2. **配置应用检测参数**（路径 A，PCRF/PCF 动态规则下发 appid 时执行）：`ADD FLOWFILTER` + `ADD ADCPARA` → [1-00015](task/UNC/20.15.2/1-00015.md)
   - 关键参数：FLOWFILTERNAME（三网元一致）；FLOWINFORPT=ENABLE（需流信息）/DISABLE（默认）
3. **配置预定义规则与用户模板绑定**（路径 B，PCRF/PCF 预定义规则下发 Event-Trigger 时执行，可选）：`ADD RULE` + `ADD USERPROFILE` + `ADD RULEBINDING` → [1-00016](task/UNC/20.15.2/1-00016.md)
   - 关键参数：RULENAME（三网元一致）+ POLICYTYPE=ADC；USERPROFILENAME + UPTYPE=PCC；RULEBINDING 绑定

> **路径选择**：步骤 2 与步骤 3 **二选一**，由 DP1（规则下发方式）驱动。PCRF/PCF 通过动态规则下发 appid → 走步骤 2（[1-00015](task/UNC/20.15.2/1-00015.md)）；PCRF/PCF 通过预定义规则下发 Event-Trigger → 走步骤 3（[1-00016](task/UNC/20.15.2/1-00016.md)）。activation 标注步骤 3 为"可选"是相对路径选择而言（路径 B 内三命令必选）。

> **activation 演示范围脚注（R1.4）**：WSFD-109102 仅 1 份 activation（`激活ADC基本功能_92582136.md`），含两个任务示例——示例一演示路径 A（FLOWFILTER+ADCPARA，FLOWINFORPT=ENABLE），示例二演示路径 B（RULE+USERPROFILE+RULEBINDING）。两路径均完整演示，**无零证据差异参数**。ADC activation 与计费族/PCC activation 雷同度低（ADC 专属命令 ADCPARA + License LKV2BADCF01 + POLICYTYPE=ADC），非模板复用。

## 决策点（特性级）

### DP1：规则下发方式（决定走路径 A 还是路径 B）

| 规则下发方式 | 走法 | UNC 侧配置 | appid 来源 | 流信息上报 | activation 演示 |
|---|---|---|---|---|---|
| 动态规则下发 appid | 步骤 1+2（[1-00015](task/UNC/20.15.2/1-00015.md)） | FLOWFILTER + ADCPARA | PCRF/PCF 动态下发 | ADCPARA.FLOWINFORPT 控制 | ✓（任务示例一） |
| 预定义规则下发 Event-Trigger | 步骤 1+3（[1-00016](task/UNC/20.15.2/1-00016.md)） | RULE(POLICYTYPE=ADC) + USERPROFILE + RULEBINDING | 预定义规则指定 | UPF 决定，UNC 透传 | ✓（任务示例二） |

### DP2：流信息上报开关（仅路径 A，DP1=动态规则时）

| FLOWINFORPT | 适用业务 | 影响 |
|---|---|---|
| ENABLE | 可推论业务（FTP/RTSP 等，持续时间长、并发流少） | 上报 Flow-Information（Gx）/sdfDescriptions（N7），PCRF/PCF 据此生成动态 QoS 规则 |
| DISABLE（默认） | 不可推论业务（P2P/HTTP 等，五元组老化快、并发流多） | 不上报流信息，减少 N4/N7 信令负荷 |

### DP3：接口代际（仅影响信元命名，不影响 UNC 配置命令）

| 代际 | 接口 | 应用开始触发器 | 应用停止触发器 | 检测信息信元 | UNC 配置影响 |
|---|---|---|---|---|---|
| 4G/5G 融合 | Gx（Diameter） | APPLICATION_START | APPLICATION_STOP | Application-Detection-Information（TDF-Application-Identifier/Instance-Identifier/Flow-Information） | 无（命令集相同） |
| 纯 5G | N7（服务化 HTTP/JSON） | APP_STA | APP_STO | AppDetectionInfo（appId/instanceId/sdfDescriptions） | 无（命令集相同） |

> 代际选择由 PCC 基本功能（[2-00005](task/UNC/20.15.2/2-00005.md)）的代际 DP 决定，本特性不单独配置接口——ADC 规则经已建立的 Gx/N7 通道传递。

## 约束（特性级 rule）

- **License 前置**（critical）：须激活 LKV2BADCF01 License — 否则 ADC 功能不生效。
- **依赖 PCC 基本功能**（critical）：须先完成 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md) 激活（License 82207979）— PCRF/PCF 通过 Gx/N7 下发携带 Application ID 的 PCC 规则，PGW-C/SMF 通过 N4 传递，开通 ADC 必须同时开启 PCC。
- **三网元 FLOWFILTERNAME 一致**（critical，路径 A）：GGSN-C/PGW-C/SMF、PGW-U/UPF、PCRF/PCF 上的 FLOWFILTERNAME 必须一致 — 否则 PCRF 下发的 appid 无法匹配 UPF 本地配置。
- **三网元 RULENAME 一致**（critical，路径 B）：预定义规则场景下三网元规则标识必须一致 — 否则 PCRF/PCF 下发的预定义规则名无法本地匹配。
- **ADCPARA 引用 FLOWFILTER 须先建**（high，路径 A）：ADCPARA 的 FLOWFILTERNAME 必须已通过 ADD FLOWFILTER 配置存在。
- **PCC 规则删除后停止 STOP 上报**（warning）：携带 Application ID 的 PCC 规则被删除或失效后，业务停止不再上报 APPLICATION_STOP 事件 — 影响带宽控制恢复链路。
- **信令流控容错**（info）：N4/N7 接口信令流控或链路故障时，PCRF/PCF 可能收到重复 Start 事件或仅收到 Stop 事件，PCRF/PCF 须能正确处理。
- **缺省规则建议**（info）：建议在 PGW-U/UPF 上配一个三四层为 any、七层协议为 any 的 rule 作为缺省配置（UDG 侧配置，非 UNC）。
- **软件参数 BIT1916**（info）：控制 Gx 接口 ADC 上报时，同一应用已产生但尚未上报的 start/stop 事件是否仍然上报（信令流控场景可靠性）。
