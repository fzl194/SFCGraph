# OFCTemplate 绑定 APN

> 离线计费参数绑到 APN 层（优先级次于 UserProfile）。被引用于 [2-00001 离线计费](task/UNC/20.15.2/2-00001.md)。域通用（在线/融合计费模板绑 APN 同此结构）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 建 APN（已配可跳过） | `ADD APN` | APN（全网规划） |
| 模板绑 APN | `SET APNCHARGECTRL` | APN；PGWOFCTEMPLATE/SGWOFCTEMPLATE=模板名 |

**典型脚本**：
```
ADD APN: APN="apn-test";
SET APNCHARGECTRL: APN="apn-test", PGWOFCTEMPLATE="offlinecharge-test", SGWOFCTEMPLATE="offlinecharge-test";
```

**步骤位置**：紧跟 [1-00001](task/UNC/20.15.2/1-00001.md) 之后；仅当特性级 DP 选 APN 层时编排。

## 场景差异

- **热计费（WSFD-011202）**：`SET APNCHARGECTRL` 参数增量——activation 演示带 `ONLINESW=DISABLE, OFFLINESW=ENABLE`（显式关在线、开离线）、`GGSNOFCTEMPLATE=模板名`（GGSN 模板，离线计费 activation 未带此参数）、`CCNAME=热计费CC名`（携带计费属性名）。模板名引用热计费模板（阈值更小）。被引用于 [2-00002 热计费](task/UNC/20.15.2/2-00002.md)。
- **融合计费（WSFD-011206）**：`SET APNCHARGECTRL` 参数差异——用 `CCTEMPLATE`（融合计费模板名，非 PGWOFCTEMPLATE/SGWOFCTEMPLATE）+ `CONVERGEDSW=ENABLE`（融合开关）+ `RGAPPLIED`（业务申请上报模式：DEFAULT/ONLINERGONLY/OFFLINERGONLY）。上游模板是 CCT（[1-00005](task/UNC/20.15.2/1-00005.md)）非 OFCTEMPLATE。activation 演示（配置融合计费模板任务三）。被引用于 [2-00003 融合计费](task/UNC/20.15.2/2-00003.md)。
  > **★ 组合参数脚注（R1.4）**：`CCTEMPLATE` 与 `CONVERGEDSW/RGAPPLIED` 分属**不同 activation** 演示——`CCTEMPLATE` 来自 [配置融合计费模板_93400212](evidence/UNC/20.15.2/配置融合计费模板_93400212.md)（任务三），`CONVERGEDSW/RGAPPLIED` 来自 [使能融合计费功能_77691175](evidence/UNC/20.15.2/使能融合计费功能_77691175.md)。两份 activation **未在单一脚本中合并演示三参数同条**。配置生成合并产出 `SET APNCHARGECTRL: CCTEMPLATE=x, CONVERGEDSW=ENABLE, RGAPPLIED=y` 时，须核实 [SET APNCHARGECTRL 命令 wiki](command/UNC/20.15.2/SET-APNCHARGECTRL.md) 支持同条带这三参数。

## 约束

- **前置模板**（critical）：模板名须已在 [1-00001](task/UNC/20.15.2/1-00001.md)（离线 OFCTEMPLATE）或 [1-00005](task/UNC/20.15.2/1-00005.md)（融合 CCT）建立。
- **APN 须存在**（warning）：`LST APN` 可查；未建则先 ADD APN。
