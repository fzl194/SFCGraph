# 配置PLMN计费（ADD CHGPLMNCFG）

> 命令静态知识见 [ADD CHGPLMNCFG](command/UNC/20.15.2/ADD-CHGPLMNCFG.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD CHGPLMNCFG 是 UNC 三种 S-CDR 话单生成策略之一（另两种为 [ADD CHGIMSICFG](task/UNC/20.15.2/0-00020.md) 基于 IMSI 前缀、[SET CHGCHAR](command/UNC/20.15.2/SET-CHGCHAR.md) 全局基于计费属性）。本命令用"PLMN(MCC+MNC) + APN NI"定位一组用户，定制其 S-CDR 生成行为。维度比 CHGIMSICFG 粗（不区分 IMSI 前缀和拜访类型），适合按归属网络运营商整体差异化。激活前提：SET CHGCDR 的 PLMNCTRL=LOCINFO（取位置信息中的 PLMN ID）时本策略生效；若 PLMNCTRL=IMSI 则走 ADD CHGIMSICFG。

核心配置空间有 4 个维度：**用户范围**（必选，决定是否需要 MCC/MNC）、**触发开关组**（5 个枚举开关，决定是否生成各类 S-CDR）、**触发阈值**（3 个数值，配合开关生效）、**策略层级选择**（由 SET CHGCDR 闸门决定本命令是否激活）。

### 配置维度 1：用户范围（参数 SUBRANGE，必选）

| 取值 | 作用 | 联动必选参数 | 典型场景 |
|---|---|---|---|
| **ALL_USER** | 覆盖所有用户（PLMN 无关） | — | 整网统一策略，APNNI 不允许配 `*` |
| **CELL_PLMNID** | 指定 PLMN 的用户 | `MCC` + `MNC` + `APNNI` | 按归属运营商差异化话单 |

| 参数 | 范围 | 约束 |
|---|---|---|
| `MCC` | 3 位十进制数字 | SUBRANGE 非 ALL_USER 时必填；整网规划 |
| `MNC` | 2 或 3 位十进制数字 | SUBRANGE 非 ALL_USER 时必填；整网规划 |
| `APNNI` | 1~62 字符串 | 必填；LABEL 用 `.` 间隔；ALL_USER 时不允许 `*`；其他情况只允许单个 `*` |

### 配置维度 2：触发开关组（参数 SP / SPP / SVP / SCCP / SLCP）

**5 个开关默认均为 `DFT`（缺省策略，即无效值）**，需显式填有效值才生效：

| 参数 | 含义 | 取值集 | 默认 |
|---|---|---|---|
| `SP` | 是否生成 S-CDR | `NO` / `YES` / `DFT` | DFT |
| `SPP` | 周期生成 S-CDR | `FORBID` / `PERMIT` / `VOLUMN` / `DFT` | DFT |
| `SVP` | 流量生成 S-CDR | `NO` / `YES` / `DFT` | DFT |
| `SCCP` | 计费条件变更生成 S-CDR | `NO` / `YES` / `DFT` | DFT |
| `SLCP` | 位置更新生成 S-CDR | `NO` / `YES` / `DFT` | DFT |

> `DFT` 表示"走缺省策略"=无效值；只有显式配 `YES`/`PERMIT`/`VOLUMN` 才让该类触发生效，配 `NO`/`FORBID` 则显式抑制。与 ADD CHGIMSICFG 字段语义完全一致。

### 配置维度 3：触发阈值（参数 SPL / SVL / SCCL）

**3 个数值阈值需配合对应开关启用后才生效**：

| 参数 | 含义 | 取值范围 | 默认 | 配置原则 |
|---|---|---|---|---|
| `SPL` | S-CDR 生成周期（分钟） | 0~1440 | 0（无效值） | 建议不小于 15 分钟；配合 SPP 启用 |
| `SVL` | 流量生成 S-CDR 阈值（KB） | 0~1000000 | 0（无效值） | 建议不小于 100 KB；配合 SVP=YES |
| `SCCL` | 最大计费条件变更次数 | 0~10 | 0（无效值） | 需与客户计费中心确认能否处理多流量节点话单 |

> 配 SCCL 前必须与客户计费中心确认，避免多流量节点话单被丢弃。

### 配置维度 4：策略层级选择（由 SET CHGCDR 闸门决定）

| 策略层 | 定位维度 | 命令 | 生效条件 |
|---|---|---|---|
| PLMN 细化 | PLMN(MCC+MNC) + APN NI | ADD CHGPLMNCFG | SET CHGCDR 的 PLMNCTRL=LOCINFO |
| IMSI 细化 | IMSI 前缀 + APN NI + 拜访类型 + 计费属性 | ADD CHGIMSICFG | SET CHGCDR 的 PLMNCTRL=IMSI |
| 全局 | 计费属性 | SET CHGCHAR | 始终（无 PLMN 闸门） |

> 同一特性的策略优先级：基于 PLMN+APNNI 共同配置 > 仅基于 PLMN > 仅基于 APNNI。

**典型脚本（为 CELL_PLMNID 用户配置周期+计费条件变更触发）**

```
ADD CHGPLMNCFG: SUBRANGE=CELL_PLMNID, MCC="123", MNC="001", APNNI="huawei", SPL=30, SCCP=YES, SCCL=5, SLCP=YES;
```

**步骤位置**：在"配置离线计费参数（SGSN）"流程中作为第 3 步（基于 PLMN 和 APN NI 共同配置生成 S-CDR 策略），可选地再叠加 ADD CHGIMSICFG 做更细的 IMSI 级定制。前置 SET CHGCDR 已选 PLMNCTRL=LOCINFO，后置 ADD CHGIMSICFG（可选更细粒度）。

## 决策点

### 决策点：话单生成策略层级选择（DP 0-02021）

问：该特性采用哪一层话单生成策略（决定走本命令还是 ADD CHGIMSICFG）？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| PLMN 细化 | SET CHGCDR 的 `PLMNCTRL=LOCINFO` + 本命令 | 按归属运营商整体差异化；MCC/MNC 来自位置信息 |
| IMSI 细化 | SET CHGCDR 的 `PLMNCTRL=IMSI` + ADD CHGIMSICFG | 按 IMSI 前缀+拜访类型更细粒度差异化 |
| 全局兜底 | SET CHGCHAR | 不需 PLMN/IMSI 区分，按计费属性全局生效 |

> PLMNCTRL 配错会导致策略完全不生效——这是本命令的硬前置。

### 决策点：触发开关与阈值组合（DP 0-02022）

问：该 PLMN+APNNI 策略启用哪些 S-CDR 触发方式（决定 SP/SPP/SVP/SCCP/SLCP 与 SPL/SVL/SCCL 组合）？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 全周期触发 | `SPP=PERMIT/VOLUMN` + `SPL=周期` | 按固定时间窗生成话单（典型 30 分钟） |
| 流量阈值触发 | `SVP=YES` + `SVL=阈值` | 累计流量达阈值生成话单（典型 1000 KB） |
| 计费条件变更触发 | `SCCP=YES` + `SCCL=次数` | QoS/费率时段变更时生成话单（需与计费中心确认） |
| 位置更新触发 | `SLCP=YES` | inter-RAU 时生成话单 |
| 显式抑制 | `SP=NO` / `SPP=FORBID` | 对该 PLMN 用户不生成某类话单 |
| 走缺省 | 各开关留 `DFT` | 按全局 SET CHGCHAR 决定 |

> 5 个开关可组合启用；阈值参数仅在对应开关启用时才生效。同一 PLMN 用户的多类触发可并存。

## 约束

- rule-0-02051 · severity=critical · 生效前置：仅当 SET CHGCDR 的 PLMNCTRL=LOCINFO 时本策略生效；PLMNCTRL=IMSI 时走 ADD CHGIMSICFG，配错前置会导致策略不生效
- rule-0-02052 · severity=high · 上层闸门优先：若 SET CHGPLMNCHAR 禁止生成某种话单，则无论本命令如何配置都不生成；允许生成时最终是否生成由本命令、ADD CHGIMSICFG、SET CHGCHAR 共同决定
- rule-0-02053 · severity=warning · 本表最大 1500 条记录，超过上限新增失败
- rule-0-02054 · severity=warning · 立即生效但只对配置生效后激活的用户有效，已激活用户不立即受影响
- rule-0-02055 · severity=info · 各字段默认值均为 DFT（无效值），未显式填的字段按缺省策略走；SCCL 需与客户计费中心确认能否处理多流量节点话单
