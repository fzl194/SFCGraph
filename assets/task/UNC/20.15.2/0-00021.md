# ADD CHGPLMNCFG 原子任务

> 命令静态知识见 [ADD CHGPLMNCFG](command/UNC/20.15.2/ADD-CHGPLMNCFG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CHGPLMNCFG 是 UNC 三种 S-CDR 话单生成策略之一（另两种为 ADD CHGIMSICFG 基于 IMSI 前缀、SET CHGCHAR 全局基于计费属性）。本命令用"PLMN(MCC+MNC) + APN NI"定位一组用户，定制其 S-CDR 生成行为。维度比 CHGIMSICFG 粗（不区分 IMSI 前缀和拜访类型），适合按归属网络运营商整体差异化。

激活前提：SET CHGCDR 的 PLMNCTRL=LOCINFO（取位置信息中的 PLMN ID）时本策略生效；若 PLMNCTRL=IMSI 则走 ADD CHGIMSICFG。

**参数取值样例（PLMN 细化话单策略范式）**

| 参数 | 取值样例 | 默认 | 说明 |
|---|---|---|---|
| SUBRANGE | `ALL_USER` | — | 用户范围（必填） |
| MCC | `460` | — | 移动国家码，3 位；SUBRANGE 非 ALL_USER 时必填 |
| MNC | `00` / `001` | — | 移动网号，2 或 3 位；SUBRANGE 非 ALL_USER 时必填 |
| APNNI | `huawei.com` | — | APN NI，1~62 字符（必填） |
| SP | `NO` / `DFT` | DFT | 生成 S-CDR；NO=不生成，DFT=缺省策略 |
| SPP | `FORBID` / `DFT` | DFT | 周期生成 S-CDR |
| SPL | `30` | 0 | S-CDR 生成周期(min)，0~1440 |
| SVP | `NO` / `DFT` | DFT | 流量生成 S-CDR |
| SVL | `1000` | 0 | 生成 S-CDR 流量阈值(KB)，0~1000000 |
| SCCP | `NO` / `DFT` | DFT | 计费条件变更生成 S-CDR |
| SCCL | `3` | 0 | 最大计费条件变更次数，0~10 |
| SLCP | `NO` / `DFT` | DFT | 位置更新生成 S-CDR |

**配置位置**：在"配置离线计费参数（SGSN）"流程中作为第 3 步（基于 PLMN 和 APN NI 共同配置生成 S-CDR 策略），可选地再叠加 ADD CHGIMSICFG 做更细的 IMSI 级定制。

## 决策点

### 话单生成策略选哪一层

与 ADD CHGIMSICFG 共享同一组定位维度决策，由 SET CHGCDR 的 PLMNCTRL 决定走哪条：

| 策略层 | 定位维度 | 命令 | 生效条件 |
|---|---|---|---|
| PLMN 细化 | PLMN(MCC+MNC) + APN NI | ADD CHGPLMNCFG | PLMNCTRL=LOCINFO |
| IMSI 细化 | IMSI 前缀 + APN NI + 拜访类型 + 计费属性 | ADD CHGIMSICFG | PLMNCTRL=IMSI |
| 全局 | 计费属性 | SET CHGCHAR | 始终 |

### 各触发开关填 DFT 还是显式 NO/YES

SP/SPP/SVP/SCCP/SLCP 默认 DFT（走缺省策略），显式 NO/FORBID 表示抑制。与 ADD CHGIMSICFG 一致。

## 约束

- **立即生效，但只对配置生效后激活的用户有效** · severity=MEDIUM · 已激活用户不立即受影响。
- **本表最大 1500 条记录** · severity=MEDIUM · 超过上限新增失败。
- **生效前置**：仅当 SET CHGCDR 的 PLMNCTRL=LOCINFO 时本策略生效；PLMNCTRL=IMSI 时走 ADD CHGIMSICFG · severity=HIGH · 配错前置会导致策略不生效。
- **字段默认值均为"无效值"**：与 ADD CHGIMSICFG 一致，未显式填的字段按 DFT 走缺省策略 · severity=MEDIUM。
- **上层闸门**：若 SET CHGPLMNCHAR 禁止生成某种话单，则无论本命令如何配置都不生成；允许生成时最终是否生成由本命令、ADD CHGIMSICFG、SET CHGCHAR 共同决定 · severity=HIGH。
