# 配置N40接口配额控制（SET N40QUOTACTRL）

> 命令静态知识见 [SET N40QUOTACTRL](command/UNC/20.15.2/SET-N40QUOTACTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET N40QUOTACTRL 是 N40 接口配额控制的全局参数命令。系统部署后已存在初始记录（`ZEROQUOTAACT=NORMAL, ZEROTIMEQTACT=TERMINATE, BLKTIMER=0, ONLMETERINGTYPE=DEFAULT, TCMODE=DEFAULT`），所有参数均为可选，不输入则保持系统当前值不变（可通过 `LST N40QUOTACTRL` 查看）。证据中本命令的典型用法是**可选**配置费率切换模式 `TCMODE` 或零配额处理动作。本命令有 2 个配置维度：**零配额处理动作组**、**费率切换与上报参数**。

### 配置维度 1：零配额处理动作组（参数 ZEROQUOTAACT/ZEROTIMEQTACT/BLKTIMER，均可选）

| 维度 | 参数 | 取值 | 默认（初始值） | 必填条件 | 作用 | 适用场景 |
|---|---|---|---|---|---|---|
| 零配额动作 | `ZEROQUOTAACT` | `NORMAL`/`BLCK_TRG_RPT`/`BLCK_IMMED_RPT`/`BLCK_NO_RPT` | NORMAL | 可选 | CHF 下发零配额时的处理动作 | 按业务对零配额的容忍度决定 |
| 零时长配额动作 | `ZEROTIMEQTACT` | `TERMINATE`/`ACCEPT`/`NOTCTRL` | TERMINATE | 可选（仅 ZEROQUOTAACT=NORMAL 时生效） | CHF 响应 GrantedUnit 携带零时长配额的动作 | 按业务对零时长配额的策略决定 |
| RG 级阻塞时间 | `BLKTIMER` | 整数 0~1440 分钟 | 0 | ZEROQUOTAACT=BLCK_IMMED_RPT/BLCK_NO_RPT 时条件可选 | RG 级阻塞动作的业务阻塞时间，经过配置时长后业务可重新触发配额申请 | 阻塞类零配额动作配套 |

> 数据来源：ZEROQUOTAACT/ZEROTIMEQTACT 本端规划；BLKTIMER 全网规划。`ZEROTIMEQTACT` 仅在 `ZEROQUOTAACT=NORMAL` 时生效，与软参 BYTE504 BIT1 冲突时按软参生效。

### 配置维度 2：费率切换与上报参数（参数 TCMODE/ONLMETERINGTYPE，均可选）

| 维度 | 参数 | 取值 | 默认（初始值） | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 费率切换模式 | `TCMODE` | `DEFAULT`/`DAILY`/`MONTHLY` | DEFAULT | 指定零点是否强制发生费率切换：DEFAULT 按费率切换组配置，DAILY 每天 00:00 强制切换，MONTHLY 月初第一天 00:00 强制切换 | 需在费率切换组配置基础上叠加强制切换时 |
| 在线计费上报类型 | `ONLMETERINGTYPE` | `DEFAULT`/`TIME_VOLUME` | DEFAULT | CHF 仅下发在线计费 RG 时长或流量配额时，是否同时上报时长和流量 | DEFAULT 按 GrantedUnit 类型上报，TIME_VOLUME 同时上报时长和流量 |

> 数据来源：TCMODE 本端规划；ONLMETERINGTYPE 全网规划。`TCMODE=DEFAULT` 时费率切换发生的时间点完全由 [ADD TARIFFGROUP](task/UNC/20.15.2/0-00083.md) 配置决定。

**典型脚本**

```
# 证据使用实例：设置零配额动作为阻塞不上报
SET N40QUOTACTRL: ZEROQUOTAACT=BLCK_NO_RPT;

# 典型：配置费率切换模式（DEFAULT，按费率切换组生效）
SET N40QUOTACTRL: TCMODE=DEFAULT;
```

**步骤位置**：位于融合计费/费率切换特性激活流程的"全局配额策略调优"步骤——通常在 [ADD TARIFFGROUP](task/UNC/20.15.2/0-00083.md)（费率切换组）配置之后，作为全局费率切换行为的覆盖层。

## 决策点

### 决策点：零配额处理动作选择（DP 0-03109）

问：CHF 下发零配额时如何处理？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 正常处理（系统默认） | `ZEROQUOTAACT=NORMAL`（初始默认） | 按正常配额处理，下发零配额给 UPF；此时 ZEROTIMEQTACT 生效 |
| 阻塞 + Trigger 上报 | `ZEROQUOTAACT=BLCK_TRG_RPT` | 阻塞业务，后续 Trigger 触发时上报 Charging Data Request Update |
| 阻塞 + 立即上报 | `ZEROQUOTAACT=BLCK_IMMED_RPT`（联动 `BLKTIMER`） | 阻塞业务，立即触发原因为 Final 的 Charging Data Request Update |
| 阻塞不上报 | `ZEROQUOTAACT=BLCK_NO_RPT`（联动 `BLKTIMER`） | 阻塞业务（证据使用实例） |

> 取值由本端对零配额的容忍度决定：NORMAL 倾向放通，BLCK_* 系列倾向阻塞保账；阻塞类动作需联动 `BLKTIMER` 配置阻塞时长。

### 决策点：费率切换模式选择（DP 0-03110）

问：费率切换发生的时间点按什么逻辑？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 按费率切换组配置（系统默认） | `TCMODE=DEFAULT`（初始默认，证据唯一取值） | 完全由 [ADD TARIFFGROUP](task/UNC/20.15.2/0-00083.md) 配置决定切换时间点 |
| 每天零点强制切换 | `TCMODE=DAILY` | 在费率切换组配置基础上，每天 00:00 再强制切换，生效范围为所有用户 |
| 月初第一天零点强制切换 | `TCMODE=MONTHLY` | 在费率切换组配置基础上，月初第一天 00:00 再强制切换，生效范围为所有用户 |

> 该参数控制全局费率切换行为，需与费率切换组配置联动；`TCMODE=DEFAULT` 时若费率切换组未配会导致切换不生效。

## 约束

- rule-0-03271 · severity=critical · 联动缺失（TCMODE=DEFAULT）：`TCMODE=DEFAULT` 时费率切换发生的时间点完全由 [ADD TARIFFGROUP](task/UNC/20.15.2/0-00083.md) 配置决定，费率切换组未配导致切换不生效 — 后果：费率切换组未配导致费率切换不生效
- rule-0-03272 · severity=high · 条件生效（ZEROTIMEQTACT）：`ZEROTIMEQTACT` 仅在 `ZEROQUOTAACT=NORMAL` 时生效；与软参 BYTE504 BIT1 冲突时按软参配置生效 — 后果：ZEROQUOTAACT 非 NORMAL 时 ZEROTIMEQTACT 配置无效
- rule-0-03273 · severity=warning · 条件可选（BLKTIMER）：`BLKTIMER` 仅在 `ZEROQUOTAACT=BLCK_IMMED_RPT/BLCK_NO_RPT` 时为条件可选，取值 0~1440 分钟 — 后果：非阻塞类零配额动作配置 BLKTIMER 无意义
- rule-0-03274 · severity=info · 唯一性与生效：全局一份（N40 接口配额控制唯一记录），执行后立即生效；系统部署后已存在初始记录（ZEROQUOTAACT=NORMAL/ZEROTIMEQTACT=TERMINATE/BLKTIMER=0/ONLMETERINGTYPE=DEFAULT/TCMODE=DEFAULT），未输入参数保持当前值；适用 NF 为 PGW-C、SMF — 后果：覆盖既有全局配置，跨 NF 各网元独立配置
- rule-0-03275 · severity=info · 上报类型影响：`ONLMETERINGTYPE=TIME_VOLUME` 时同时上报时长和流量使用情况给 CHF，DEFAULT 仅按 GrantedUnit 下发类型上报 — 后果：上报类型影响 CHF 配额管理精度
