# SET N40QUOTACTRL 原子任务
> 命令静态知识见 [SET N40QUOTACTRL](command/UNC/20.15.2/SET-N40QUOTACTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
系统部署后已存在初始记录（`ZEROQUOTAACT=NORMAL, ZEROTIMEQTACT=TERMINATE, BLKTIMER=0, ONLMETERINGTYPE=DEFAULT, TCMODE=DEFAULT`）。证据中本命令的典型用法是**可选**配置费率切换模式 `TCMODE`，其余参数按需覆盖。本命令无必选参数，不输入的参数保持当前值。

| 配置目标 | 参数 | 典型脚本 |
|---|---|---|
| 配置费率切换模式（典型） | `TCMODE=DEFAULT` | `SET N40QUOTACTRL:TCMODE=DEFAULT;` |
| 覆盖零配额/零时长动作 | `ZEROQUOTAACT` + `ZEROTIMEQTACT` | 按业务对零配额的容忍度规划 |
| 调整阻塞时间 | `BLKTIMER`(0~1440 分钟) | 按 RG 级阻塞需求规划 |

**取值范围**：`ZEROQUOTAACT`=`NORMAL`（按正常配额处理，下发零配额给 UPF）；`ZEROTIMEQTACT`=`TERMINATE`（去活 PDU 会话）；`ONLMETERINGTYPE`=`DEFAULT`（按 GrantedUnit 下发的配额类型上报）；`TCMODE`=`DEFAULT`（按 [ADD TARIFFGROUP](task/UNC/20.15.2/0-00083.md) 配置的费率切换生效）；`BLKTIMER`=0~1440。

## 决策点

### 费率切换模式（TCMODE）
问：费率切换发生的时间点按什么逻辑？
- `DEFAULT`（证据唯一取值，本端规划）→ 费率切换发生的时间点按 [ADD TARIFFGROUP](task/UNC/20.15.2/0-00083.md) 配置的费率切换生效。
- 该参数控制全局费率切换行为，需与费率切换组配置联动。

### CHF 下发零配额的动作（ZEROQUOTAACT）
问：CHF 下发零配额时如何处理？
- `NORMAL`（初始默认）→ 按正常配额处理，下发零配额给 UPF。
- 本端规划，按业务是否容忍零配额继续使用决定。

### CHF 下发零时长配额的动作（ZEROTIMEQTACT）
问：CHF 下发零时长配额时如何处理？
- `TERMINATE`（初始默认）→ 去活 PDU 会话。
- 本端规划，按业务对零时长配额的处理策略决定。

### 在线计费上报统计类型（ONLMETERINGTYPE）
问：在线计费上报什么统计类型给 CHF？
- `DEFAULT`（初始默认）→ 按 GrantedUnit 下发的配额类型上报。

### RG 级阻塞处理时间间隔（BLKTIMER）
问：RG 级阻塞多久？
- 0~1440 分钟，global_planned，不输入则保持当前值。

## 约束
- 执行后立即生效。`severity=low`
- 系统部署后已存在初始记录，不输入的参数保持当前值。`severity=low`
- `TCMODE=DEFAULT` 时，费率切换发生的时间点完全由 [ADD TARIFFGROUP](task/UNC/20.15.2/0-00083.md) 配置决定。`severity=medium` 后果：费率切换组未配导致切换不生效。
- `BLKTIMER` 取值 0~1440，global_planned。`severity=low`
