# ADD LOCALHOSTBIND 原子任务
> 命令静态知识见 [ADD LOCALHOSTBIND](command/UNC/20.15.2/ADD-LOCALHOSTBIND.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
在 PCC 对接（PCRF/PCF）场景下，本命令把已通过 `ADD LOCALHOSTGRP` 建好的本端主机逐一绑定到对应本端主机组中。一对主机组与主机名通常需建立一一对应（多 PGW-C 实例时按实例分组绑定）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| LOCGRPNAME | 引用已通过 `ADD LOCALHOSTGRP` 创建的本端信息组名（先建组，后绑定） | `pgwc_host_group_01` |
| LOCHOSTNAME | 引用已配置的 Diameter 本端主机名，按 PGW-C/SMF 实例规划，从已配置数据中取 | `pgwc_1` |

典型脚本（按实例数循环生成多条）：
```
ADD LOCALHOSTBIND: LOCGRPNAME="pgwc_host_group_01", LOCHOSTNAME="pgwc_1";
ADD LOCALHOSTBIND: LOCGRPNAME="pgwc_host_group_02", LOCHOSTNAME="pgwc_2";
```

## 决策点（若有）

### 主机组与主机名的对应关系
- 问什么：每个 PGW-C/SMF 实例的本端主机归属到哪个本端主机组；多实例时是否一组多主机还是一实例一组。
- 影响绑定方式：证据范例为"一实例一组、单主机绑定"，亦可在单个组内绑定多达 64 个本端主机实现负载分担。
- 取决于 Diameter 路由与 PCRF/PCF 对接拓扑规划，属**全网规划/对端协商**层面。

## 约束（若有）
- 前置依赖：LOCGRPNAME 必须已通过 `ADD LOCALHOSTGRP` 存在，LOCHOSTNAME 必须已配置存在，否则引用失败。`severity=high`
- 规格上限：单个 Diameter 本端主机分组内最多 64 个本端主机；命令最大记录数 1000。`severity=medium`
- 生效时机：执行后立即生效。`severity=low`
