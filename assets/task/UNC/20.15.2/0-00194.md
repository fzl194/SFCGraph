# SET PCCMSGATTR 原子任务

> 命令静态知识见 [SET PCCMSGATTR](command/UNC/20.15.2/SET-PCCMSGATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET PCCMSGATTR 按 MSGTYPE（消息类型）逐条配置 Gx 接口消息中可选 AVP 的携带开关。每条记录绑定一个 MSGTYPE，AVP 开关取 ENABLE/DISABLE。命令存在系统初始记录（4 条，对应 CCR-I/CCR-U/CCR-T/RAA 的默认 AVP 携带方式），用户配置覆盖默认。证据中唯一范例为 Gx Failover 功能：在 CCR-U 与 CCR-T 中开启特定信元携带。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| MSGTYPE | `CCRU` / `CCRT` / `CCRI` / `RAA` | 对端协商 | 消息类型（必选，每条记录的主键） |
| MSISDN | `ENABLE` | 对端协商 | Subscription-Id 携带 MSISDN 开关 |
| CALLEDSTATIONID | `ENABLE` | 对端协商 | Called-Station-ID AVP 开关 |
| USREQUIPINFO | `ENABLE` | 对端协商 | User-Equipment-Info AVP 开关 |
| FRAMEDIPADDRESS | `ENABLE` | 对端协商 | Framed-IP-Address AVP 开关 |
| FRAMEDIPV6PREF | `ENABLE` | 对端协商 | Framed-IPv6-Prefix AVP 开关 |
| 其余 AVP 开关 | `ENABLE` / `DISABLE` | 对端协商 | 28 个可选 AVP 开关，详见命令 wiki |

**特性场景取值对比**

| 特性场景 | MSGTYPE | 开启的 AVP | 说明 |
|---|---|---|---|
| Gx Failover（WSFD-109101） | CCRU | MSISDN, CALLEDSTATIONID, USREQUIPINFO, FRAMEDIPADDRESS, FRAMEDIPV6PREF | CCR-U 携带用户/位置/地址信元 |
| Gx Failover（WSFD-109101） | CCRT | MSISDN, CALLEDSTATIONID | CCR-T 仅携核心标识 |

## 决策点

### 各 AVP 携带开关（须与 PCRF 协商）

所有 AVP 开关取值取决于对端 PCRF 的能力与需求——PCRF 需要哪些信元做策略决策，GGSN/PGW-C 就在对应消息类型中开启携带。证据中所有参数数据来源均为"对端协商"。配置生成时按对端 PCRF 的信元需求清单逐 AVP 决策。

### 消息类型分条配置（4 条记录对应 4 种消息）

MSGTYPE 是每条记录的主键，CCR-I/CCR-U/CCR-T/RAA 各一条记录，AVP 携带方式按消息类型独立配置。不可在一条命令中混合多种 MSGTYPE。

## 约束

- **立即生效** · 执行后立即生效。
- **最大 4 条记录** · 对应 4 种消息类型。
- **系统初始记录** · severity=MEDIUM · 命令存在 4 条系统初始记录（MSGTYPE × 默认 AVP 设置），用户配置覆盖默认值，未覆盖的 AVP 沿用初始值。
- **AVP 携带须与 PCRF 协商一致** · severity=HIGH · 开启 PCRF 不识别的 AVP 可能致消息处理异常；关闭 PCRF 需要的 AVP 致策略决策信息缺失。
- **配套软件参数** · severity=MEDIUM · Gx Failover 增强功能需先 SET SMFSOFTPARA 设 BIT1202=1 才生效。
