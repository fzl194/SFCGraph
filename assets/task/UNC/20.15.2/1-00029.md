# UNC 接入域路由发布（OSPF 引入用户路由）

> **UNC 域接入域 OSPF 路由共享底座**。被引用于 8 个地址分配特性。
>
> 设计参考 UDG 域 `1-00024` 下行路由发布结构（结构对齐，UNC 命令族独立维护）。

## 配置方法

本 compound 在 OSPF 进程下引入用户路由（wlr），把 UPF 分配的地址发布到骨干网，让 N4/S1/S5 等其他网元可达。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 创建 OSPF 进程 | `ADD OSPF` → [0-00246](task/UNC/20.15.2/0-00246.md) | PROCID（IPv4）/ ROUTERID 全网唯一 |
| 2. 创建 OSPF 区域 | `ADD OSPFAREA` → [0-00247](task/UNC/20.15.2/0-00247.md) | PROCID, AREAID |
| 3. 创建 OSPF 接口（绑定网段）| `ADD OSPFINTERFACE` → [0-00249](task/UNC/20.15.2/0-00249.md) | PROCID, AREAID, IFNAME（**UNC 真实命令名**——UNC `ADD OSPFNETWORK` 不存在，是 `ADD OSPFINTERFACE`，命令族归 `平台服务管理/VNRS功能管理/IP服务/路由管理/OSPF管理/OSPF接口配置/`）|
| 4. 引入用户路由（wlr） | `ADD OSPFIMPORTROUTE` → [0-00248](task/UNC/20.15.2/0-00248.md) | PROCID, AREAID, PROTOCOL=wlr（用户本地路由）|

**典型脚本（IPv4 单栈基线，**命令名已按 UNC 真实命令修正**）**：
```
ADD OSPF: PROCID=1, ROUTERID="10.1.1.1";
ADD OSPFAREA: PROCID=1, AREAID="0.0.0.0";
ADD OSPFINTERFACE: PROCID=1, AREAID="0.0.0.0", IFNAME="接口名";
ADD OSPFIMPORTROUTE: PROCID=1, AREAID="0.0.0.0", PROTOCOL=wlr;
```

> **★ 重要命令名修正（2026-07-12，R1.5 脚注）**：v1.0 写的是 `ADD OSPFNETWORK`，**UNC 命令文档无此命令**——UNC 真实命令名是 `ADD OSPFINTERFACE`（路径：`平台服务管理/VNRS功能管理/IP服务/路由管理/OSPF管理/OSPF接口配置/创建OSPF接口配置（ADD OSPFINTERFACE）_00866569.md`）。本次 v1.0→v1.1 一并修正。OSPFv3 同理——UNC 真实命令名是 `ADD OSPFV3INTERFACE`（全大写，非小驼峰），归 [1-00032](task/UNC/20.15.2/1-00032.md) UNC OSPFv3 路由发布骨架。

**步骤位置**：在地址池体系（[UNC 1-00023](task/UNC/20.15.2/1-00023.md)）建立后、骨干网互通之前执行。

## 约束

- **ROUTER ID 全网唯一**（critical）：OSPF Router ID 必须全网唯一 — 后果：OSPF 邻接建立失败。
- **VPN 三者一致**（critical）：OSPF 进程应与对应 VPN 绑定 — 后果：路由无法迭代。
- **本 compound 不含独立命令外的命令**：完整 IPv6/双栈需评估是否新建 OSPFv3 atom（UNC 0-00246 等仅覆盖 IPv4）。
- **静态地址冗余场景**（critical）：主备 UPF 需 IMPORTROUTE COST 区分（主 100/备 200），避免流量黑洞 — 归 [WSFD-107021](feature/UNC/20.15.2/WSFD-107021.md)。
