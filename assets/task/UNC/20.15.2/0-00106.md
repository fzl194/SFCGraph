# DSP PCCSESSINFO 原子任务

> 命令静态知识见 [DSP PCCSESSINFO](command/UNC/20.15.2/DSP-PCCSESSINFO.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。


适用 NF：PGW-C、SMF。本命令是 PCC 用户调测的核对入口——验证 PCRF/PCF 下发的规则是否在 PGW-C/SMF 上安装成功、是否生效。

## 配置方法

按"用户标识类型 + 标识值 + 显示方式"三元组查询。三选一传入用户标识（IMSI/MSISDN/IMEI）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| USERIDTYPE | 必选，用户标识类型（枚举） | `IMSI` |
| IMSI | 当 USERIDTYPE=IMSI 时的标识值 | `460000123456789` |
| MSISDN | 当 USERIDTYPE=MSISDN 时 | `13800138000` |
| IMEI | 当 USERIDTYPE=IMEI 时 | `1234567890123456` |
| DISPLAYMODE | 可选，显示方式（默认 SIMPLE） | `SIMPLE` / `VERBOSE` / `USG_MONITORING` |

**DISPLAYMODE 三种模式**：
- `SIMPLE`：显示用户的会话信息（默认）
- `VERBOSE`：显示用户的会话详细信息
- `USG_MONITORING`：显示用户的 Usage Monitoring 信息

**典型脚本**（按 IMSI 查 PCC 会话，核对规则安装）：
```
DSP PCCSESSINFO:USERIDTYPE=IMSI,IMSI="460000123456789";
```

**典型脚本**（VERBOSE 模式查详细信息）：
```
DSP PCCSESSINFO:USERIDTYPE=IMSI,IMSI="123011223344551",DISPLAYMODE=VERBOSE;
```

**调测语义**：在"缺省承载 GBR 保障"和"PCC 业务/基本功能调测"中，本命令用于确认 PCRF 下发的 default-gbr 承载规则或全部规则是否安装成功——存在则调测通过；不存在或安装失败则回查 PCRF 状态、Gx 接口消息或 rule-failure-code。

## 决策点

### 决策点：用户标识类型选择（USERIDTYPE）（DP 0-01250）

问：用哪种标识定位目标 PCC 用户？

| 选项 | `USERIDTYPE` | 联动字段 | 适用场景 |
|---|---|---|---|
| IMSI（最常用） | `IMSI` | IMSI（1~15 位） | 调测脚本默认，按用户身份定位 |
| MSISDN | `MSISDN` | MSISDN（1~15 位） | 按号码定位（终端侧定位） |
| IMEI | `IMEI` | IMEI（1~16 位） | 按终端设备定位 |

> 三者互斥，按现场可获取的标识选择。

### 决策点：DISPLAYMODE 显示方式选择（DP 0-01251）

| 选项 | `DISPLAYMODE` | 适用场景 |
|---|---|---|
| SIMPLE（默认） | `SIMPLE` | 快速核对会话是否存在、规则是否安装 |
| VERBOSE | `VERBOSE` | 详细查看 QoS 流的 FiveQI、ARP、动态规则信息 |
| USG_MONITORING | `USG_MONITORING` | 查看用户级流量监控配额信息 |

## 约束

· rule-0-01561 · severity=medium · 适用范围：仅适用于 PCC 用户；非 PCC 用户查询无 PCC 规则信息
· rule-0-01562 · severity=low · 命令 notes 无额外规格上限；USERIDTYPE 为 required，三种标识类型互斥
· rule-0-01563 · severity=info · 本命令为只读诊断，适用于 PGW-C、SMF 网元
