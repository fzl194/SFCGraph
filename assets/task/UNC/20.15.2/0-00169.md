# SET DFTGLBPCRFGRP 原子任务
> 命令静态知识见 [SET DFTGLBPCRFGRP](command/UNC/20.15.2/SET-DFTGLBPCRFGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

PGW-C/GGSN 侧**全局级** PCRF 选择兜底：当会话无法通过 APN 级绑定（[ADD PCRFGRPBNDAPN](task/UNC/20.15.2/0-00060.md)）或用户级绑定找到 PCRF 组时，使用本命令配置的缺省 PCRF 组。属于动态 PCC（Policy and Charging Control）功能的全局缺省配置之一。

## 配置方法
单参数命令，指定一个已建的 PCRF 组名作为全局缺省。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PCRFGRPNAME | PCRF 组名（前置 ADD GLBPCRFGROUP 已建） | `"pcrf_group_1"` |

典型脚本（开启动态 PCC 全局缺省，PCRF 组名 pcrf_group_1）：
```
SET DFTGLBPCRFGRP: PCRFGRPNAME="pcrf_group_1";
```

## 决策点

### 缺省 PCRF 组选择
- 问什么：哪个 PCRF 组作为全局缺省。
- 取值由整网 PCRF 部署规划决定，PCRF 组来自 [ADD GLBPCRFGROUP](task/UNC/20.15.2/0-00034.md)。
- 全局缺省优先级最低：APN 级绑定（[ADD PCRFGRPBNDAPN](task/UNC/20.15.2/0-00060.md)）> 全局缺省（本命令）。

## 约束
- 前置：PCRF 组名必须已在 [ADD GLBPCRFGROUP](task/UNC/20.15.2/0-00034.md) 中创建。`severity=high`
- 唯一性：最大记录数 1（全网一份缺省）。`severity=medium`
- 生效时机：立即生效。`severity=low`
- 优先级：本命令为全局缺省兜底，APN 级/用户级绑定优先匹配。`severity=medium`
