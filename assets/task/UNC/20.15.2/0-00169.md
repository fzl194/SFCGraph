# 配置全局缺省 PCRF 组（SET DFTGLBPCRFGRP）

> 命令静态知识见 [SET DFTGLBPCRFGRP](command/UNC/20.15.2/SET-DFTGLBPCRFGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

PGW-C/GGSN 侧**全局级** PCRF 选择兜底：当会话无法通过 APN 级绑定（[ADD PCRFGRPBNDAPN](task/UNC/20.15.2/0-00061.md)）或用户级绑定找到 PCRF 组时，使用本命令配置的缺省 PCRF 组。属于动态 PCC（Policy and Charging Control）功能的全局缺省配置之一。核心配置空间仅 1 个维度：**缺省 PCRF 组选择**（单参数命令，指定一个已建的 PCRF 组名）。

### 配置维度 1：缺省 PCRF 组（参数 PCRFGRPNAME）

| 参数 | 取值 | 作用 | 备注 |
|---|---|---|---|
| `PCRFGRPNAME` | 字符串 1~128 | PCRF 组名（前置 ADD GLBPCRFGROUP 已建） | 由 [ADD GLBPCRFGROUP](task/UNC/20.15.2/0-00034.md) 生成；输入单空格删除已有配置 |

> 全局缺省优先级最低：APN 级绑定（[ADD PCRFGRPBNDAPN](task/UNC/20.15.2/0-00061.md)）> 用户级绑定 > 全局缺省（本命令）。

典型脚本（开启动态 PCC 全局缺省，PCRF 组名 pcrf_group_1）：
```
SET DFTGLBPCRFGRP: PCRFGRPNAME="pcrf_group_1";
```

**步骤位置**：在动态 PCC 全局基础配置中，位于 ADD GLBPCRFGROUP 之后，与 SET MASTERPCRF、SET PCCFUNC 同层级。

## 决策点

### 决策点：缺省 PCRF 组选择（DP 0-03077）

问：哪个 PCRF 组作为全局缺省？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 指定已建 PCRF 组 | `PCRFGRPNAME="已建组名"` | 整网规划了兜底 PCRF 组，未匹配 APN/用户级绑定的会话走此组 |
| 清除缺省 | `PCRFGRPNAME=" "`（单空格） | 不需要全局兜底，未匹配会话按系统默认行为处理 |

> 取值由整网 PCRF 部署规划决定，PCRF 组来自 ADD GLBPCRFGROUP。

### 决策点：优先级层级（DP 0-03078）

问：本命令配置的缺省组在 PCRF 选择链中的位置？

| 选项 | 说明 |
|---|---|
| APN 级绑定优先 | ADD PCRFGRPBNDAPN 配置的 APN 级绑定最先匹配 |
| 用户级绑定次之 | 用户级 PCRF 绑定其次匹配 |
| 全局缺省兜底 | 本命令配置的全局缺省最后兜底，优先级最低 |

## 约束

- **前置**（warning，rule-0-03191）：前置：PCRF 组名必须已在 ADD GLBPCRFGROUP 中创建
- **唯一性**（warning，rule-0-03192）：唯一性：最大记录数 1（全网一份缺省）
- **生效时机**（info，rule-0-03193）：生效时机：立即生效
- **优先级**（warning，rule-0-03194）：优先级：本命令为全局缺省兜底，APN 级/用户级绑定优先匹配
- **清除语义**（info，rule-0-03195）：清除语义：输入单空格将删除该参数已有配置项
