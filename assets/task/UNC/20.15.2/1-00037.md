# UNC 接入控制族

> **UNC 域接入控制族骨架**（SOP §3.5 特性专属）。被引用于 **WSFD-106003**（用户接入控制）。
>
> **按制式分代际**：SGSN（2G/3G）+ MME（4G）+ AMF（5G）—— 命令族按 NF 类型不同，差异化配置。

## 配置方法

| 步骤 | 命令（按代际） | 关键参数 |
|---|---|---|
| 1a. SGSN 接入限制（2G/3G）| `ADD SGSNARD`（族，命令名待核）| 接入拒绝条件（IMSI/IMEI/区域）|
| 1b. MME 接入限制（4G）| `ADD MMEARD`（族，命令名待核）| 接入拒绝条件（IMSI/IMEI/区域）|
| 1c. AMF 接入限制（5G）| `ADD AMFARD`（族，命令名待核）+ `SET NGMMPROCTRL` → [0-00279](task/UNC/20.15.2/0-00279.md) | 接入拒绝条件（SUPI/区域） |
| 2. 拒绝原因值配置（仅 AMF 必配）| `SET NGMMPROCTRL`（[0-00279](task/UNC/20.15.2/0-00279.md)）| PROT=OTHER_PROC, ACCRST=0 |

**典型脚本（仅 AMF 制式）**：
```
SET LICENSESWITCH: LICITEM=LKV2ARD02, SWITCH=ENABLE;
ADD AMFARD: 接入拒绝条件参数;
SET NGMMPROCTRL: PROT=OTHER_PROC, ACCRST=0;
```

**步骤位置**：所有接入控制特性的核心——前置 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施（VPN+APN）。

## 场景差异

- **[WSFD-106003](feature/UNC/20.15.2/WSFD-106003.md) 用户接入控制**：按 NF 类型选步骤 1a/1b/1c；AMF 必配步骤 2

## 决策点

| NF 类型 | 步骤 | License |
|---|---|---|
| SGSN（2G/3G）| 步骤 1a | LKV2ARD02 |
| MME（4G）| 步骤 1b | LKV2ARD02 |
| AMF（5G）| 步骤 1c + 步骤 2 | LKV2ARD02 |

## 约束

- **License 前置**（critical）：`SET LICENSESWITCH LICITEM=LKV2ARD02` 必须先使能 — 后果：未开 License → 接入控制不生效
- **★ 本骨架命令名多为族名推测**：SGSNARD/MMEARD/AMFARD 的 UNC 真实命令名需根据 UNC OM 文档核对——具体命令族路径在 `业务服务管理/Pre 5G接入业务管理/控制面管理/接入控制管理/`（待核）
- **本骨架大部分命令 atom 待补**
