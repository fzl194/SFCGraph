# UPF选择（WSFD-107010）

> 特性静态知识见 [WSFD-107010](feature/UNC/20.15.2/WSFD-107010.md)。本页讲配置生成实例化时这个特性**怎么配**——编排哪些步骤、7 类选择原则如何选。

## 配置概览

核心对象链：**对端 NF Profile**（`ADD PNFPROFILE`）→ **切片/DNN/DNAI/UPF INFO/TAI/SMF 服务区**（`ADD PNFNS`+`ADD PNFDNN`+`ADD PNFDNAI`+`ADD PNFUPFINFO`+`ADD PNFTAI`+`ADD PNFTAIRANGE`+`ADD PNFSMFSERAREA`）→ **本地 UP 节点**（`ADD UPNODE`）→ **UP 区域与位置绑定**（`ADD UPAREA`+`ADD UPAREABINDN2TAI`+`ADD LOCBINDAREA`）→ **接口绑定**（`ADD UPBINDS11`+`ADD UPBINDGNGP`）→ **UPF 选择策略开关**（`SET UPSELECTFLAG`+`SET UPSELECTPRI`+`SET APNUPSELPLY`+`SET UPLOADBALANCE`）。

7 类原则**共享接入域骨架**（[UNC 1-00024](task/UNC/20.15.2/1-00024.md)）+ **SMF 地址池体系**（[UNC 1-00023](task/UNC/20.15.2/1-00023.md)）+ **OSPF 路由发布**（[UNC 1-00029](task/UNC/20.15.2/1-00029.md)），差异在选择条件（DNN/切片/DNAI/位置区/接口/优先级/权重/负载）的启用开关组合。由特性级 DP（UPF 角色 × 选择条件）驱动。

**License 信息**：需 [SET LICENSESWITCH](task/UNC/20.15.2/0-00180.md) 前置 — 配 `LKV2USBL01`（UPF 选择）+ 可选 `LKV2GWUS01`（GW-U 选择）。

## 决策点（特性级：UPF 角色 × 选择条件）

| UPF 角色 × 选择条件 | 走法（复用 compound）| 关键联动参数 |
|---|---|---|
| **主锚点 UPF × DNN**（基线）| 骨架 1-00024 + License LKV2USBL01 + PNF 族（PNFPROFILE + PNFDNN）+ UPNODE + UPSELECTFLAG | DNN 必选 |
| **I-UPF × 切片（S-NSSAI）**| 骨架 1-00024 + License LKV2USBL01 + PNF 族（PNFPROFILE + PNFNS）+ UPNODE + UPSELECTFLAG(N3UPFAPNFLAG) | 切片必选；I-UPF 不需 DNN |
| **PSA UPF × DNAI**（分流）| 骨架 1-00024 + License LKV2USBL01 + PNF 族（PNFPROFILE + PNFDNAI）+ UPNODE + UPSELECTFLAG | DNAI 必选 |
| **SGW-U × S11 接口**（4G/2-3G 切换）| 骨架 1-00024 + License LKV2USBL01 + UPNODE + ADD UPBINDS11 + UPSELECTFLAG(ULISGWFLAG) | S11 必选；ULI 为空时按 S11 选 |
| **PGW-U/GGSN-U × Gn/Gp 或 S5/S8 接口**| 骨架 1-00024 + License LKV2USBL01 + UPNODE + ADD UPBINDGNGP | 接口必选 |
| **合一 UPF（SGW-U/PGW-U 合一）**| 骨架 1-00024 + License LKV2USBL01 + PNF 族 + UPNODE + UPSELECTPRI（第一优先级=COMBINED）+ UPSELECTFLAG(PRIORITYFLAG=DISABLE) | 合一优选 |
| **按权重 + 负载**（多 UPF 并存）| 骨架 1-00024 + License LKV2USBL01 + PNFPROFILE(权重) + UPLOADBALANCE（负载均衡开关）| 权重 + 负载 |
| **按位置区**（贴近用户）| 骨架 1-00024 + License LKV2USBL01 + UPAREA + UPAREABINDN2TAI + LOCBINDAREA + UPSELECTPRI(第二优先级=LOCS11PRIORITY) | 位置区 |
| **优先级排序**（PRIORITYFLAG=ENABLE）| 骨架 1-00024 + License LKV2USBL01 + UPSELECTFLAG(PRIORITYFLAG=ENABLE) + APNUPSELPLY(COMBINEPRISTG) | 启用优先级开关 |

## 约束（特性级 rule）

- **PNFPROFILE 必须先建**（critical）：`ADD UPNODE` 时 NFINSTANCENAME 须引用已 `ADD PNFPROFILE` 注册的实例 — 后果：UPNODE 配置失败。
- **SMF/UPF 必须同厂商**（critical）：SMF 与 UPF 必须同时为 HUAWEI 设备 — 后果：跨厂商 UPF 互通失败。
- **PNFNSINDEX 默认值建议 0**（warning，4G 场景）：4G 用户接入 EPC 网络时 SGW-C/PGW-C 选 SGW-U/PGW-U 不带切片信息，建议 PNFDNN 的 PNFNSINDEX 保持默认 0 — 后果：规划不当致 4G UPF 选择错乱。
- **第一轮必选无优先级**（warning）：必选条件（DNN/切片/DNAI/接口）均满足才进入第二轮优选；任一必选失败则 UPF 选择失败。
- **PRIORITYFLAG 初始 DISABLE**（warning）：系统默认 UPSELECTFLAG 的 PRIORITYFLAG=DISABLE，UPF 选择不考虑优先级 — 后果：未启用优先级开关时按必选 + 合一 + 位置区选 UPF。
- **UPF 权重越大选中概率越大**（info）：权重代表 UPF 容量；多 UPF 满足条件时按权重选。
- **UP 负载来自偶联上报**（info）：SMF 通过 PFCP 偶联消息获取 UPF 负载信息（受 UPLOADBALANCE 控制）。
- **PNF* 族原子债务**（critical，配置生成阻塞）：本特性 17+ 条 PNF/UPAREA/UPSELECT* 命令**当前 numbering.json 未登记**，配置生成前**必须**先按命令名排序接续补建 atom（详见附录），否则无法生成可执行脚本。
