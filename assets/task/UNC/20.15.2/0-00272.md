# 设置APN L2TP控制属性（SET APNL2TPCTRL）

> 命令静态知识见 [SET APNL2TPCTRL](command/UNC/20.15.2/SET-APNL2TPCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令以 APN 为粒度控制 L2TP VPN 接入属性，是 APN 级 L2TP 总开关。配置空间核心维度为 **L2TP 总开关**（L2TPSWITCH）和 **配套鉴权/专有承载子参数**（L2TPSWITCH=ENABLE 时生效）。

### 配置维度1：L2TP 总开关（参数 L2TPSWITCH）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| DISABLE（默认） | 该 APN 不支持 L2TP 功能 | 非 L2TP APN（默认） |
| ENABLE | 该 APN 支持 L2TP 功能（UPF 充当 LAC 与 LNS 对接） | L2TP VPN（WSFD-104410：L2TPSWITCH=ENABLE） |

### 配置维度2：配套鉴权子参数（L2TPSWITCH=ENABLE 时条件可选）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| ICCN_PROXYAUTH=ENABLE | MSISDN 作为 ICCN 代理认证用户名带给 UPF/LNS（PAP 鉴权） | 需 MSISDN 作用户名的 L2TP |
| COMMONUSERUSED=ENABLE | 用 `SET APNAUTHATTR` 配置的 CommonUserName/Pass 作 LNS 用户鉴权（PAP） | 公用账号 L2TP |
| DEDICATEDBEARSW=ENABLE | 支持 L2TP 专有承载 | L2TP 专有承载场景 |
| PASSWORD + CFMPASSWORD | L2TP 用户鉴权密码（两参数必须一致） | 静态密码鉴权 |

**步骤位置**：在 L2TP VPN（WSFD-104410）编排中，于 `ADD APN`（建 APN）之后执行——`ADD APN` 自动创建一条默认记录（全 DISABLE），本命令把 L2TPSWITCH 改为 ENABLE。常与 UPF 侧 L2TP 隧道配置（`ADD L2TPGROUP` 等）配套。

## 决策点

**L2TP 鉴权模式选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| L2TPSWITCH=ENABLE（最小） | 仅总开关 | 默认 L2TP 接入（WSFD-104410） |
| + ICCN_PROXYAUTH=ENABLE | + 可选 PASSWORD | MSISDN 作 ICCN 代理用户名 |
| + COMMONUSERUSED=ENABLE | + 配套 SET APNAUTHATTR CommonUserName/Pass | 公用账号 L2TP |
| + DEDICATEDBEARSW=ENABLE | + 专有承载 | 专有承载 L2TP |

> ICCN_PROXYAUTH 与 COMMONUSERUSED 决定 LNS 用户鉴权的用户名/密码来源；两者未配则 ICCN 消息不带用户名密码。

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无回滚窗口
- **APN 必须先建**（critical）：必须先 `ADD APN` 配置 APN；`ADD APN` 时自动创建一条默认记录 — 后果：APN 不存在则配置失败
- **子参数前提**（warning）：ICCN_PROXYAUTH/COMMONUSERUSED/DEDICATEDBEARSW/PASSWORD/CFMPASSWORD 仅在 L2TPSWITCH=ENABLE 时可选 — 后果：DISABLE 时配置无效
- **PASSWORD 与 CFMPASSWORD 一致**（critical）：PASSWORD 与 CFMPASSWORD 必须同时输入且相同 — 后果：不一致新增失败
- **PASSWORD 长度**（warning）：密码 1~128 — 后果：超长配置失败
- **APN 命名约束**（warning）：字符串 1~63，仅 `-`/数字/字母/`.`，不能以 `.` 开头或连续 `.`，不区分大小写 — 后果：违规命名配置失败
