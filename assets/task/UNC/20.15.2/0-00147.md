# SET AMFPEERSELFUNC 原子任务
> 命令静态知识见 [SET AMFPEERSELFUNC](command/UNC/20.15.2/SET-AMFPEERSELFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：AMF。系统部署后存在初始记录（SCPSEPPRESEL=OFF, NFLOCATION=UDM-1&SMF-1, SCPRESELEN=OFF, AUSFRESELEN=ON），本命令是覆盖性修改。证据显示同一 NFLOCATION 在不同特性下取不同值（PCF-1 / SMSF-1），获取方法多值（本端规划+全网规划），属 decision_driven。

## 配置方法
按"对端 NF 类型 + 选择功能开关"配置；NFLOCATION 是 NF 类型集合，可重复配置不同对端。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| NFLOCATION | 支持使用 Location 信元携带地址通信的对端 NF 类型（全网规划） | `PCF-1`（PCC 基本功能）/ `SMSF-1`（短消息） |
| AUSFRESELEN | AUSF 重选增强开关（本端规划） | `ON`（UDM 全故障保活） |
| SCPRESELEN | SCP 重选增强开关（全网规划） | `OFF` / `ON` |
| SCPSEPPRESEL | SCP/SEPP 重选开关（本端规划） | `OFF` |

典型脚本（UDM 全故障保活场景）：
```
SET AMFPEERSELFUNC: AUSFRESELEN=ON;
```
（PCC 基本功能场景）：
```
SET AMFPEERSELFUNC: NFLOCATION=PCF-1;
```

## 决策点

### NFLOCATION 取值随对端业务类型而变
- 问什么：AMF 与哪类对端 NF（PCF/SMSF/UDM/SMF）通过 Location 信元携带的地址通信。
- PCC 基本功能（5G）→ `PCF-1`：配置 AMF 支持使用 PCF 通过 Location 携带的地址通信。
- 短消息业务 → `SMSF-1`：对端 SMSF 支持多 IP 地址对接时，AMF 使用 Location 地址后续交互。
- 取值错配会导致 AMF 不识别对端多地址，信令交互失败。

### AUSF/SCP 重选增强（AUSFRESELEN / SCPRESELEN）
- 问什么：对端 AUSF/SCP 返回 5xx 故障码、ProblemDetails 校验失败后，AMF 是否重选并决定是否进入 UDM Bypass 状态。
- UDM 全故障业务保活 → `AUSFRESELEN=ON`：控制重选后再次校验失败是否进入 UDM Bypass。
- 默认 `OFF`/初始 `AUSFRESELEN=ON`，按局点是否部署 UDM Bypass 场景决策。
- 依赖前置：AUSF 重选增强需配合 [ADD BYPASSFAULTCODE](command/UNC/20.15.2/ADD-BYPASSFAULTCODE.md) 故障状态码配置与软参 DWORD68 BIT5 / DWORD17 BIT21。

### SCPSEPPRESEL SCP/SEPP 重选
- 漫游或 SCP/SEPP 代理场景才启用，常规部署保持 `OFF`。

## 约束
- 立即生效：参数修改后立即作用于后续 NF 选择，不影响已建立的偶联。`severity=medium`
- conditional 参数：SCPRESELEN 为 conditional，按 SCP 部署形态决定是否必填。`severity=low`
- 初始值不变语义：未输入的参数保持系统当前配置不变，可经 LST AMFPEERSELFUNC 查询。`severity=low`
- AUSF 重选增强链路一致性：开启 AUSFRESELEN 需同步配置 BYPASSFAULTCODE 与对应软参，否则重选后处理策略不完整。`severity=high`
