# 设置AMF对端选择功能参数（SET AMFPEERSELFUNC）

> 命令静态知识见 [SET AMFPEERSELFUNC](command/UNC/20.15.2/SET-AMFPEERSELFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET AMFPEERSELFUNC 控制不同场景下 AMF 选择对端 NF 的功能。系统部署后已存在初始记录（SCPRESELEN=OFF, NFLOCATION=UDM-1&SMF-1, SCPSEPPRESEL=OFF, AUSFRESELEN=ON），本命令是覆盖性修改。配置空间有 4 个维度：**对端 NF 类型**（决定 AMF 是否使用 Location 信元携带的地址通信）、**AUSF 重选增强**（UDM 全故障保活）、**SCP 重选增强**（Model C 故障处理）、**SCP/SEPP 重选**（漫游代理场景）。

### 配置维度 1：对端 NF 类型（参数 NFLOCATION，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 对端 NF 类型 | `NFLOCATION` | 全网规划；枚举：UDM/SMF/AMF/SMSF/PCF/AUSF/NSSF，可叠加 `-{1..n}` 表示多实例 | UDM-1&SMF-1 | AMF 支持使用该类对端 NF 通过 Location 信元携带的地址进行后续消息交互（对端支持多 IP 对接场景） | PCC 基本功能（PCF-1）、短消息业务（SMSF-1）；对端为 UDM/SMF 时需软参 Dword71 Bit26=1 才生效；Model D 模式下不生效 |

### 配置维度 2：AUSF 重选增强（参数 AUSFRESELEN，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| AUSF 重选增强开关 | `AUSFRESELEN` | 本端规划；枚举：ON/OFF | ON（初始） | 对端 AUSF 返回 5xx 且 ProblemDetails 校验失败后，是否重选 AUSF；重选后再次校验失败且开启 UDM 全故障保活时允许进入 UDM Bypass 状态 | UDM 全故障业务保活特性 |

### 配置维度 3：SCP 重选增强（参数 SCPRESELEN，conditional）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| SCP 重选增强开关 | `SCPRESELEN` | 全网规划；枚举：ON/OFF；条件可选（SCPSEPPRESEL=ON 时） | OFF | Model C 场景下 SCP 返回 504 且 ProblemDetails=TARGET_NF_NOT_REACHABLE 时，ON=只重选对端 NF 不重选 SCP；OFF=只重选 SCP | 当前仅对 LMF 网元生效；SCP/SEPP 代理部署场景 |

### 配置维度 4：SCP/SEPP 重选（参数 SCPSEPPRESEL，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| SCP/SEPP 重选开关 | `SCPSEPPRESEL` | 本端规划；枚举：ON/OFF | OFF | Model C/D 近端 SCP 故障或漫游场景 SEPP 故障时是否重选 SCP/SEPP | 漫游场景、SCP/SEPP 代理部署；常规部署保持 OFF |

**典型脚本（UDM 全故障保活场景）**
```
SET AMFPEERSELFUNC: AUSFRESELEN=ON;
```

**典型脚本（PCC 基本功能：AMF 使用 PCF Location 地址）**
```
SET AMFPEERSELFUNC: NFLOCATION=PCF-1;
```

**步骤位置**：AMF 策略与对接 NF 选择阶段，UDM 全故障保活特性中 AUSFRESELEN=ON 需配合 [ADD BYPASSFAULTCODE](command/UNC/20.15.2/ADD-BYPASSFAULTCODE.md) 故障状态码配置与软参 DWORD68 BIT5 / DWORD17 BIT21。

## 决策点

### 决策点：NFLOCATION 取值随对端业务类型而变（DP 0-03033）

问：AMF 与哪类对端 NF（PCF/SMSF/UDM/SMF）通过 Location 信元携带的地址通信？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| PCF-1（5G PCC 基本功能） | `NFLOCATION=PCF-1` | AMF 支持使用 PCF 通过 Location 携带的地址通信；取值错配会导致 AMF 不识别对端多地址，信令交互失败 |
| SMSF-1（短消息业务） | `NFLOCATION=SMSF-1` | 对端 SMSF 支持多 IP 地址对接时，AMF 使用 Location 地址后续交互 |
| UDM/SMF（需软参生效） | `NFLOCATION=UDM-1` 或 `SMF-1` | 需软参 Dword71 Bit26=1 才生效；AMF/NSSF 为预留字段当前不支持 |

### 决策点：AUSF/SCP 重选增强（AUSFRESELEN / SCPRESELEN）（DP 0-03034）

问：对端 AUSF/SCP 返回 5xx 故障码、ProblemDetails 校验失败后，AMF 是否重选并决定是否进入 UDM Bypass 状态？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| UDM 全故障业务保活 | `AUSFRESELEN=ON` | 重选后再次校验失败且开启 UDM Bypass 特性时，允许用户进入 UDM Bypass 状态；需同步配置 BYPASSFAULTCODE 与软参 DWORD68 BIT5 / DWORD17 BIT21 |
| 默认（不重选 AUSF） | `AUSFRESELEN=OFF` | 对端 AUSF 返回 5xx 且 ProblemDetails 校验失败时不进行 AUSF 重选 |
| SCP 重选增强（Model C） | `SCPRESELEN=ON`（前提 `SCPSEPPRESEL=ON`） | Model C 场景下 504+TARGET_NF_NOT_REACHABLE 时只重选对端 NF 不重选 SCP；当前仅对 LMF 生效 |
| SCP 重选（默认） | `SCPRESELEN=OFF` | 上述场景下只重选 SCP |

## 约束

- **AUSF 重选增强链路一致性**（critical，rule-0-03081）：AUSF 重选增强链路一致性：开启 AUSFRESELEN 需同步配置 [ADD BYPASSFAULTCODE](command/UNC/20.15.2/ADD-BYPASSFAULTCODE.md) 故障状态码与软参 DWORD68 BIT5 / DWORD17 BIT21，否则重选后处理策略不完整，UDM Bypass 状态进入逻辑失效
- **NFLOCATION 取值错配后果**（warning，rule-0-03082）：NFLOCATION 取值错配后果：取值错配会导致 AMF 不识别对端多地址，信令交互失败；对端为 UDM/SMF 时需软参 Dword71 Bit26=1，AMF/NSSF 为预留字段当前不支持；Model D 模式下 NFLOCATION 不生效
- **立即生效与初始值**（warning，rule-0-03083）：立即生效与初始值：参数修改后立即作用于后续 NF 选择，不影响已建立的偶联；系统初始值 SCPSEPPRESEL=OFF, NFLOCATION=UDM-1&SMF-1, SCPRESELEN=OFF, AUSFRESELEN=ON，未输入参数保持系统当前配置不变，可经 LST AMFPEERSELFUNC 查询
- **SCPRESELEN conditional 前提**（warning，rule-0-03084）：SCPRESELEN conditional 前提：SCPRESELEN 为 conditional，`SCPSEPPRESEL=ON` 时才生效；当前仅对 LMF 网元生效，对其他网元不生效
- **SCPSEPPRESEL 适用范围**（info，rule-0-03085）：SCPSEPPRESEL 适用范围：SCP/SEPP 重选仅漫游或 SCP/SEPP 代理场景才启用，常规部署保持 OFF
