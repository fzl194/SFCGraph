# 配置用户位置（ADD USRLOCATION）

> 命令静态知识见 [ADD USRLOCATION](command/UNC/20.15.2/ADD-USRLOCATION.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD USRLOCATION 是 PGW-C/SMF 上"用户位置"的原子配置入口，单命令一条位置记录，按 `LOCATIONTYPE` 枚举（PLMN/LAI/RAI/CGI/SAI/ECGI/TAI/NCGI）二选一确定代际维度并联动对应的位置码字段。位置记录既可被 USRLOCATIONGRP 引用，也可直接用于带宽管理或 TCP 小区拥塞检测。本命令参数按"位置类型枚举 → 条件必选位置码"的代际分支树组织。

### 配置维度 1：位置类型枚举（参数 LOCATIONTYPE）

| 取值 | 作用 | 联动位置码参数 | 代际 | 典型场景 |
|---|---|---|---|---|
| `PLMN` | 公共陆地移动网络标识（最粗粒度） | MCC + MNC | 跨代际 | 全网策略，位置服务兜底 |
| `LAI` | 位置区域标识 | MCC + MNC + LAC | 2/3G 粗粒度 | 位置区域维度的带宽/策略控制 |
| `RAI` | 路由区域标识 | MCC + MNC + LAC + RAC | 2/3G | 路由区域维度的带宽管理 |
| `CGI` | 小区全局标识 | MCC + MNC + LAC + CI | 2/3G 细粒度 | 单小区维度的 TCP 拥塞检测 / 带宽控制 |
| `SAI` | 服务区域标识 | MCC + MNC + LAC + SAC | 2/3G | 服务区维度策略 |
| `ECGI` | E-UTRAN 小区全局标识 | MCC + MNC + ECI | 4G 细粒度 | 4G 单小区维度的位置策略 |
| `TAI` | 跟踪区域标识 | MCC + MNC + TAC | 4G/5G | 跟踪区域维度的带宽/策略控制 |
| `NCGI` | NR 小区标识 | MCC + MNC + NCI | 5G 细粒度 | 5G 单小区维度的位置策略 |

> 8 种取值对应不同位置粒度。MCC/MNC 是 PLMN/LAI/RAI/CGI/SAI/ECGI/NCGI/TAI 八取值的通用前置；其余位置码（LAC/CI/RAC/SAC/ECI/TAC/NCI）按 LOCATIONTYPE 取值条件必选。

### 配置维度 2：PLMN 标识（参数 MCC/MNC）

| 参数 | 取值 | 作用 | 适用场景 |
|---|---|---|---|
| `MCC` | 字符串 3 位数字，000~999 | 移动国家码 | 所有位置类型前置条件 |
| `MNC` | 字符串 2 或 3 位数字，00~99 或 000~999 | 移动网络码 | 所有位置类型前置条件 |

### 配置维度 3：2/3G 位置码（参数 LAC/CI/RAC/SAC）

| 参数 | 取值 | 作用 | LOCATIONTYPE 取值 |
|---|---|---|---|
| `LAC` | 1~6 字符十六进制，0x1~0xfffd 或 0xffff | 位置区码 | LAI/RAI/CGI/SAI 必选 |
| `CI` | 1~6 字符十六进制，0x0~0xffff | 小区标识 | CGI 必选 |
| `RAC` | 1~4 字符十六进制，0x0~0xff | 路由区码 | RAI 必选 |
| `SAC` | 1~6 字符十六进制，0x0~0xffff | 服务区码 | SAI 必选 |

### 配置维度 4：4G/5G 位置码（参数 ECI/TAC/NCI）

| 参数 | 取值 | 作用 | LOCATIONTYPE 取值 |
|---|---|---|---|
| `ECI` | 1~9 字符十六进制，0x0~0xfffffff | E-UTRAN 小区标识 | ECGI 必选 |
| `TAC` | 1~8 字符十六进制，0x1~0xffffff | 跟踪区码 | TAI 必选 |
| `NCI` | 1~11 字符十六进制，0x1~0xfffffffff | NR 小区标识 | NCGI 必选 |

### 配置维度 5：位置名称（参数 LOCATIONNAME）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| 字符串 1~63 字符，不支持空格，不区分大小写 | 本端规划的位置名称（记录主键） | 通用；下游 USRLOCATIONGRP 引用 |

> 步骤位置：位置策略控制链的第一步（[1-00019 配置用户位置与模板绑定](task/UNC/20.15.2/1-00019.md)），先于 ADD USRLOCATIONGRP 与 MOD/ADD UPBINDUPG。带宽管理与 TCP 小区拥塞检测场景下也直接被业务特性引用。

## 决策点

### 决策点：位置类型代际分叉（DP 0-04029）

问：按接入代际选择 LOCATIONTYPE 取值（CGI/ECGI/NCGI 等），位置码参数如何联动？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 2/3G 位置（CGI） | `MCC`、`MNC`、`LAC`、`CI` | 2/3G 单小区粒度的带宽/拥塞检测 |
| 4G 位置（ECGI） | `MCC`、`MNC`、`ECI` | 4G 单小区粒度的位置策略控制 |
| 5G 位置（NCGI） | `MCC`、`MNC`、`NCI` | 5G 单小区粒度的位置策略控制 |
| 2/3G 区域粒度 | `LOCATIONTYPE=LAI/RAI/SAI`，对应 `LAC`/`RAC`/`SAC` | 区域级带宽管理 |
| 4G/5G 跟踪区 | `LOCATIONTYPE=TAI`，对应 `TAC` | 跟踪区域级带宽/策略控制 |
| 全网粒度 | `LOCATIONTYPE=PLMN`，仅 `MCC`/`MNC` | 全网级策略兜底 |

> 取值由用户接入代际决定；同一 UsrLocationGrp 可同时包含不同代际的位置记录（如 CGI + ECGI + NCGI 混合）。

### 决策点：单/多位置规模（DP 0-04030）

问：当前 PLMN 内需要配置多少条位置记录？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单位置 | 单条 ADD USRLOCATION | 单小区试点 |
| 多位置（同一组） | 多条 ADD USRLOCATION（不同 LOCATIONNAME），后续 ADD USRLOCATIONGRP 逐位置绑到同一 LOCGROUPNAME | 多小区/多代际混合组（典型初始接入位置策略） |

> 单命令上限 2000 条；多位置场景需逐条 ADD USRLOCATION 后由 ADD USRLOCATIONGRP 绑定。

## 约束

- **最大记录数 2000**（critical，rule-0-04071）：本命令最大记录数为 2000，超过则命令执行失败 — 后果：单命令的硬上限，超出需拆分多条命令或清理历史
- **立即生效**（info，rule-0-04072）：命令执行后配置立即生效，无需重启或用户重激活 — 后果：在线生效，对在线用户立即可见
- **位置码条件必选**（critical，rule-0-04073）：LAC/CI/RAC/SAC/ECI/TAC/NCI 按 LOCATIONTYPE 取值条件必选，缺配则命令失败 — 后果：缺位置码执行报错，需按 LOCATIONTYPE 补齐对应字段
- **位置名称不区分大小写且不支持空格**（warning，rule-0-04074）：LOCATIONNAME 字符串 1~63 字符，不支持空格，不区分大小写 — 后果：含空格或超长直接报错，引用方（USRLOCATIONGRP）需按同一名称匹配
- **多条记录允许相同/包含关系**（info，rule-0-04075）：配置的多条用户位置信息之间允许存在相同或者包含关系（如某 CGI 与某 LAI 区域重叠），不强制互斥 — 后果：重叠区域匹配按最细粒度优先（CGI > LAI > PLMN）
