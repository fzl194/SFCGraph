# ADD PDUTRIGGER 原子任务
> 命令静态知识见 [ADD PDUTRIGGER](command/UNC/20.15.2/ADD-PDUTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
Session 级 Trigger 按"事件类型 × 上报时机"规划：以 CCT 模板为载体，每个 `*CHG` 参数对应一类会话级事件，取值三选一（`IMMEDIATE` 立即上报 / `DEFERRED` 延后合并上报 / `NONREPORT` 不上报）。系统部署后存在初始缺省值表，本命令按业务对其覆盖。Session 级与 RG 级（ADD RGTRIGGER）同时生效冲突时以 Session 级优先；CHF 下发 Trigger 优先级高于本地配置。

| 上报时机 | 含义 | 何时选 |
|---|---|---|
| `IMMEDIATE` | 事件发生立即生成容器上报 CHF | QoS 变更、RAT 切换、UPF 添加、AMBR 变更等需即时计费的迁移事件 |
| `DEFERRED` | 事件发生延后到下次计费容器一并上报 | 服务节点变更、位置上报、I-SMF 插删、RAI/CGISAI 变更等非即时事件 |
| `NONREPORT` | 不生成容器不上报 | 无需计费的辅助事件（如 TAICHG 默认） |

**典型组合脚本**（证据 WSFD-011206）：
```
ADD PDUTRIGGER: CCTMPLTNAME="cct-test", QOSCHG=IMMEDIATE, SRVNDCHG=DEFERRED, RATCHG=IMMEDIATE, PRACHG=DEFERRED, ADDUPF=IMMEDIATE, INSERTISMF=DEFERRED, CGISAICHG=DEFERRED, RAICHG=DEFERRED;
```

**参数分组**（24 个 `*CHG` 参数）：
- 移动性/位置类：`ULCHG`/`PRACHG`/`TAICHG`/`CGISAICHG`/`RAICHG`/`PLMNCHG`/`RATCHG`
- 拓扑变更类：`SRVNDCHG`/`ADDUPF`/`INSERTISMF`/`REMOVALISMF`/`CHANGEISMF`
- QoS/速率类：`QOSCHG`/`SESSAMBRCHG`/`SERVPLMNRTCTCHG`/`APNRATECTRLCHG`/`PSDATAOFFCHG`
- 配额/阈值类：`TIMELIMIT`/`VOLUMELIMIT`/`EVENTLIMIT`/`MAXNUMCCC`/`UCITIMER`
- 用户属性类：`UETZCHG`

## 决策点

### 上报时机选择
问：某类会话事件发生时计费容器如何生成？
- `IMMEDIATE`：事件即触发独立容器上报，计费精度高但信令开销大（证据中 QOSCHG/RATCHG/ADDUPF 采用）。
- `DEFERRED`：事件记入延后桶，与下次计费容器合并上报，平衡精度与开销（证据中 SRVNDCHG/PRACHG/INSERTISMF/CGISAICHG/RAICHG 采用）。
- `NONREPORT`：不参与计费（默认 TAICHG/CGISAICHG/RAICHG 倾向此类，但证据中 CGISAICHG/RAICHG 改为 DEFERRED）。

### Trigger 来源优先级
问：CHF 下发 Trigger 与本地 ADD PDUTRIGGER 冲突时谁优先？
- CHF 下发优先级更高；CHF 只下发 Trigger 不下发计费参数时，SMF 按本地配置计费参数实现。
- SET CNVRGDCHGPARA 的 `IGNORETRIGGER`（位域）可指定忽略 CHF 下发的某些 Trigger（计费条件改变阈值/会话级时间阈值/会话级流量阈值/RG 级时间阈值/RG 级流量阈值/配额保持时长），改用本地配置。文字提及 SET CNVRGDCHGPARA。

## 约束
- `CCTMPLTNAME` 必须指向已存在的 CCT 模板。`severity=critical`
- 最多 101 条记录；系统部署后已存在初始缺省记录，本命令对其覆盖。`severity=medium`
- **只对新激活用户生效**——已激活用户的 Trigger 不变更。`severity=high` 后果：现网用户需等待重 attach 才应用新 Trigger。
- Session 级与 RG 级 Trigger 同时生效冲突时 Session 级优先。`severity=medium`
- `MAXNUMCCC`/`TIMELIMIT`/`VOLUMELIMIT`/`EVENTLIMIT` 等阈值类参数的 `IMMEDIATE` 含义是"达到阈值立即上报"，与事件类 `IMMEDIATE`（事件触发上报）语义不同。`severity=medium`
- 执行后立即生效（对后续新激活用户）。`severity=low`
