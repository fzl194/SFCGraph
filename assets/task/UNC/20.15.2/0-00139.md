# RMV DIAMCONNECTION 原子任务
> 命令静态知识见 [RMV DIAMCONNECTION](command/UNC/20.15.2/RMV-DIAMCONNECTION.md)。本页只讲配置生成实例化时这条命令**怎么配**。

**高危命令**。删除对象是 `DIAMCONNECTION`（Diameter 链路）记录。通过 `DIAMCONNGRP/LOCINTERFACE/对端地址端口/LOCALPORT` 等参数组合定位；全部参数都不指定会**删除所有记录**。

## 配置方法
按链路组/本端接口/对端地址端口定位删除；LOCALPORT 三态语义不同。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| DIAMCONNGRP | Diameter 链路组名（local_planned） | `gx-grp` |
| LOCINTERFACE | 本端接口名（local_planned） | `loop1` |
| PEERADDRTYPE | 对端地址类型（枚举，对端协商） | `IPv4` |
| PEERIPV4ADDR | 对端 IPv4（conditional，对端协商） | `10.1.1.2` |
| PEERIPV6ADDR | 对端 IPv6（conditional，对端协商） | `2001:db8::2` |
| PEERPORT | 对端端口号（conditional，对端协商） | `3868` |
| PEERSCTPENDPT | 对端 SCTP 端点名（conditional，对端协商） | `peer-sctp-1` |
| LOCALPORT | 本端端口（local_planned，三态） | `0` / `13200` / 省略 |

典型脚本（删指定链路组下一条 Gx 链路）：
```
RMV DIAMCONNECTION: DIAMCONNGRP=gx-grp, PEERIPV4ADDR=10.1.1.2, PEERPORT=3868;
```

## 决策点（若有）

### 删除范围（精确单条 vs 全删）
- 问什么：删一条链路，还是删全部链路。
- 精确删：给 `DIAMCONNGRP + 对端地址端口` 等组合锁定单条。
- 全删：**所有参数都不指定 = 删除所有记录**——高危，禁止在生产直接用，需软参 BYTE976=169 才允许某些极简删除。

### LOCALPORT 三态语义
- 问什么：本端端口怎么过滤。
- 省略 LOCALPORT → 删除由其他输入参数锁定的所有 CONNECTION 记录（含系统指定端口和用户指定端口）。
- `LOCALPORT=0` → 只删系统指定端口的 CONNECTION 记录。
- `LOCALPORT=<具体值>` → 删该具体端口（取值 0，13200～13263，16400～16463，19765～19784）的记录。

### 对端地址类型与地址匹配
- `PEERADDRTYPE` 决定用 IPv4/IPv6/SCTP 端点哪种对端标识匹配，需与对端协商一致；地址端口类参数标 conditional，按所选 ADDRTYPE 提供对应值。

## 约束（若有）
- 高危：删除 Diameter 链路可能导致链路中断、用户被去活等业务影响，删前必须确认链路未承载在用会话。`severity=high`
- 全删禁止：所有参数都不指定会删全部记录，生产环境禁止裸执行。`severity=high`
- 链路组名前置：未指定 `DIAMCONNGRP` 时禁止执行（除非软参 BYTE976=169）。`severity=high`
- 生效时机：立即生效。`severity=medium`
- 配套删除：调测场景常成对使用本命令 + RMV DIAMLOCINFO 删除原配置，再 ADD DIAMLOCINFO + ADD DIAMCONNECTION 重建。`severity=medium`
