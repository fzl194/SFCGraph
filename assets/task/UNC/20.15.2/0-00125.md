# 修改PDU会话级Trigger参数（MOD PDUTRIGGER）

> 命令静态知识见 [MOD PDUTRIGGER](command/UNC/20.15.2/MOD-PDUTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按 CCTMPLTNAME 定位 PDU 会话级 Trigger 集，逐项修改各事件 Trigger 的上报策略与生效原则。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 定位 | CCTMPLTNAME | 融合计费模板名（1~63，不区分大小写） | 无 | 指定目标 CCT | 所有场景必选 |
| QoS/位置 | QOSCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | QoS 更新 trigger 上报方式 | 按计费粒度分叉 |
| QoS/位置 | ULCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 用户位置更新 trigger | RAT=UTRAN 不支持 ULCHG |
| QoS/位置 | SRVNDCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 服务节点更新 trigger | — |
| QoS/位置 | PRACHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 区域用户位置上报更新 | — |
| 状态更新 | PSDATAOFFCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | PS 数据关闭状态更新 | — |
| 状态更新 | UETZCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 用户时区更新 | — |
| 状态更新 | PLMNCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | PLMN 更新 | — |
| 状态更新 | RATCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | RAT 更新 | — |
| UPF | ADDUPF | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 添加 UPF trigger | IUPF/SGW-U 不计费不控 |
| 阈值 | TIMELIMIT | `NONREPORT` / `IMMEDIATE` | 无 | 数据时间阈值 trigger | 不支持 DEFERRED |
| 阈值 | VOLUMELIMIT | `NONREPORT` / `IMMEDIATE` | 无 | 数据流量阈值 trigger | 不支持 DEFERRED |
| 阈值 | EVENTLIMIT | `NONREPORT` / `IMMEDIATE` | 无 | 事件阈值 trigger | 当前版本暂不支持 |
| 阈值 | MAXNUMCCC | `NONREPORT` / `IMMEDIATE` | 无 | 计费条件改变阈值 | — |
| 阈值 | UCITIMER | `NONREPORT` / `IMMEDIATE` | 无 | 业务停止时长超时 | 当前版本暂不支持 |
| I-SMF | INSERTISMF | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 插入 I-SMF trigger | 依赖 SET N40APIVER 使能 |
| I-SMF | REMOVALISMF | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 删除 I-SMF trigger | 依赖 SET N40APIVER 使能 |
| I-SMF | CHANGEISMF | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 更新 I-SMF trigger | 依赖 SET N40APIVER 使能 |
| AMBR/速率 | SESSAMBRCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | Session AMBR 更新 | 未使能时按 QOSCHG 处理 |
| AMBR/速率 | SERVPLMNRTCTCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 服务 PLMN 速率控制更新 | — |
| AMBR/速率 | APNRATECTRLCHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | APN 速率控制更新 | — |
| 位置标识 | TAICHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | 跟踪区标识更新 | — |
| 位置标识 | CGISAICHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | CGISAI 更新 | — |
| 位置标识 | RAICHG | `NONREPORT` / `IMMEDIATE` / `DEFERRED` | 无 | RAI 更新 | — |

典型脚本（QoS 更新延迟上报）：

```
MOD PDUTRIGGER: CCTMPLTNAME="test", QOSCHG=DEFERRED;
```

上报方式说明：

| 上报方式 | 行为 |
|---|---|
| NONREPORT | 不生成容器，不上报 CHF |
| IMMEDIATE | 立即触发 Charging Data Request Update，携带所有已生成用量容器 |
| DEFERRED | 生成用量容器但不立即发消息，在下一条 Update 中携带 |

步骤位置：调测融合计费 Trigger 上报功能流程步骤 6/7（LST 校验 → MOD 重配 → 重新触发验证）。

## 决策点

### 决策点：各类 Trigger 是否上报——事件维度的开关矩阵（DP 0-01282）

每个 Trigger 参数决定一类事件发生时的上报策略，取值 NONREPORT / IMMEDIATE / DEFERRED。具体启用哪些 Trigger 由融合计费话单规划决定，存在跨场景取值分叉。调测时需先 LST 查本地是否已配、生效原则是否正确，再决定是否 MOD。

### 决策点：Session 级 vs RG 级 Trigger 选择（DP 0-01283）

| 选项 | 作用域 | 适用用户 |
|---|---|---|
| PDU 会话级（MOD PDUTRIGGER，本命令） | Session 维度 | Session 用户 |
| RG 级（MOD RGTRIGGER） | Rating Group 维度 | RG 用户 |

二者可并存但作用域不同；订阅冲突时以 Session 级 Trigger 优先。具体走哪一级由用户计费模式决定。

## 约束

- **仅对新激活用户生效**（critical，rule-0-01657）：仅对新激活用户生效：执行后只对新激活用户生效，不追溯已激活用户

- **前置存在性**（critical，rule-0-01658）：前置存在性：只能修改已 ADD 成功的 PDU Trigger 参数集；可用 LST PDUTRIGGER 校验

- **CCT 模板前置**（critical，rule-0-01659）：CCT 模板前置：CCTMPLTNAME 必须是已存在的融合计费模板

- **调测闭环**（warning，rule-0-01660）：调测闭环：配后需重新执行调测流程步骤 3（重新激活/触发）验证 Trigger 上报是否生效

- **名称规格**（warning，rule-0-01661）：名称规格：CCTMPLTNAME 1~63 不区分大小写
