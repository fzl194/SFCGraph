# MOD PDUTRIGGER 原子任务
> 命令静态知识见 [MOD PDUTRIGGER](command/UNC/20.15.2/MOD-PDUTRIGGER.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用网元：PGW-C、SMF。本命令修改融合计费模板（CCT）下 PDU 会话级（Session 级）Trigger 参数，定义各类事件触发时是否生成容器并上报 CHF。被融合计费场景（WSFD-011206）用于调测计费 Trigger 上报功能。与 RG 级 Trigger [MOD RGTRIGGER](task/UNC/20.15.2/0-00126.md) 平行（Session 级 vs RG 级）。前置需先 [ADD PDUTRIGGER](task/UNC/20.15.2/0-00062.md) 建参数集。

## 配置方法
按 CCTMPLTNAME 定位 PDU 会话级 Trigger 集，逐项修改各事件 Trigger 的上报策略与生效原则。

| 参数（按语义分组） | 配置方法 | 取值样例 |
|---|---|---|
| CCTMPLTNAME | 融合计费模板名（必选，1~63） | `cct-test` |
| QOSCHG / ULCHG / SRVNDCHG / PRACHG | QoS/用户位置/服务节点/区域用户位置更新 | `NONREPORT`（不上报）等 |
| PSDATAOFFCHG / UETZCHG / PLMNCHG / RATCHG | PS 数据关闭/时区/PLMN/RAT 更新 | `NONREPORT` 等 |
| ADDUPF | 添加 UPF | `NONREPORT` 等 |
| TIMELIMIT / VOLUMELIMIT / EVENTLIMIT / MAXNUMCCC | 时间/流量/事件/计费条件改变阈值 | `NONREPORT` 等 |
| UCITIMER | 业务停止时长 | `NONREPORT` 等 |
| INSERTISMF / REMOVALISMF / CHANGEISMF | 插入/删除/更新 I-SMF | `NONREPORT` 等 |
| SESSAMBRCHG / SERVPLMNRTCTCHG / APNRATECTRLCHG | Session AMBR/服务 PLMN 速率/APN 速率控制更新 | `NONREPORT` 等 |
| TAICHG / CGISAICHG / RAICHG | 跟踪区/CGISAI/RAI 更新 | `NONREPORT` 等 |

## 决策点

### 各类 Trigger 是否上报（事件维度的开关矩阵）
每个 Trigger 参数决定一类事件发生时"是否生成容器并上报 CHF"——NONREPORT（不上报）/各类上报策略。具体启用哪些 Trigger 由融合计费话单规划决定，存在跨场景取值分叉。调测时需先 LST 查本地是否已配、生效原则是否正确，再决定是否 MOD（见调测流程步骤 6/7）。

### Session 级 vs RG 级 Trigger 选择
PDU 会话级（MOD PDUTRIGGER，本命令）与 RG 级（[MOD RGTRIGGER](task/UNC/20.15.2/0-00126.md)）平行。按用户类型（Session 用户 vs RG 用户）走对应层级，二者可并存但作用域不同。具体走哪一级由用户计费模式决定。

### 生效原则（Trigger 类型的生效原则）
调测流程步骤 7 指出：需校验"本地配置的 Session 级或 RG 级对应 Trigger 类型的**生效原则是否正确**"，若错误用本命令/MOD RGTRIGGER 重配。生效原则（何种条件下该 Trigger 真正触发上报）存在跨场景取值分叉。

## 约束
- **仅对新激活用户生效**：执行后只对新激活用户生效（不追溯已激活用户）。
- **前置存在性**：只能修改已 ADD 成功的 PDU Trigger 参数集；可用 LST PDUTRIGGER 校验本地配置与生效原则。
- **CCT 模板前置**：CCTMPLTNAME 必须是已存在的融合计费模板。
- **调测闭环**：配后需重新执行调测流程步骤 3（重新激活/触发）验证 Trigger 上报是否生效。
- **名称规格**：CCTMPLTNAME 1~63 不区分大小写。
