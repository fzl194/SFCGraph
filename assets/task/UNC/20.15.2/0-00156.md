# 配置离线计费话单产生开关（SET CDRTRIGGER）

> 命令静态知识见 [SET CDRTRIGGER](command/UNC/20.15.2/SET-CDRTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

绑定到已建离线计费模板（OFCTEMPLATENAME），按 6 类触发条件勾选开关 + 可选优先级。开关按"是否触发"成对出现（开关参数 + 配套优先级参数），优先级取 1~100，数字越小越优先。存在系统初始记录（全部 ENABLE，优先级 255）。

| 维度 | 参数 | 取值 | 默认（初始值） | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 绑定模板 | OFCTEMPLATENAME | 模板名，来自已建 OFCTEMPLATE，字符串 1~63（不支持空格） | — | 指定本组开关所属模板 | 必须先 `ADD OFCTEMPLATE` |
| RAT 更新触发 | CDRTRIGRATCHNG / CDRTRIGRATPRIOR | ENABLE/DISABLE；优先级 1~100 | ENABLE / 255 | RAT 切换时是否产生话单（话单版本 ≥ r6v660gcdr） | 关注 RAT 切换 |
| Serving Node 更新触发 | CDRTRIGSNCHNG / CDRTRIGSNPRIOR | ENABLE/DISABLE；优先级 1~100 | ENABLE / 255 | SGSN/SGW 更新时是否触发产生话单（话单版本 ≥ r98cmccv130） | 关注用户位置/节点迁移 |
| MS 时区更新触发 | CDRTRIGTIMEZONE / CDRTRIGTZPRIOR | ENABLE/DISABLE；优先级 1~100 | ENABLE / 255 | MS time zone 更新时是否触发产生话单（话单版本 ≥ r6v660gcdr） | 关注跨时区漫游 |
| 计费条件改变触发 | CDRTRIGMAXCHNG / CDRTRIMAXCHGPRI | ENABLE/DISABLE；优先级 1~100 | ENABLE / 255 | 计费条件改变次数达上限时是否触发产生话单（QoS/ULI/RAI 改变；话单版本 ≥ r98cmccv130） | 关注 QoS/位置变更 |
| PLMN 标识更新触发 | CDRTRIGPLMNCHNG / CDRTRIGPLMNPRIO | ENABLE/DISABLE；优先级 1~100 | ENABLE / 255 | SGSN/SGW PLMN 标识改变时是否触发产生话单（话单版本 ≥ r99v3a0） | 关注 PLMN 归属变化 |
| MO Exception 触发 | CDRTRIGMOEXC / CDRTRIGMOEXCPRI | ENABLE/DISABLE；优先级 1~100 | ENABLE / 255 | MO Exception Data Counter 更新时是否产生话单 | 关注 MO Exception 计数 |

典型脚本（仅使能 Serving Node、计费条件、PLMN 三类触发）：
```
SET CDRTRIGGER: OFCTEMPLATENAME="offlinecharge-test", CDRTRIGSNCHNG=ENABLE, CDRTRIGMAXCHNG=ENABLE, CDRTRIGPLMNCHNG=ENABLE;
```

步骤位置：建离线计费模板（`ADD OFCTEMPLATE`，[0-00049](task/UNC/20.15.2/0-00049.md)）→ 本任务（按模板勾选触发开关）→ 配套阈值（`SET OFCTHRESHOLD`）+ 容器产生开关（`SET CONTAINERTRIGGER`）

## 决策点

### 决策点：触发类别使能选择（DP 0-03051）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| RAT 更新触发 | CDRTRIGRATCHNG=ENABLE（+ 可选 CDRTRIGRATPRIOR） | 关注无线接入技术切换产生话单 |
| Serving Node 更新触发 | CDRTRIGSNCHNG=ENABLE（+ 可选 CDRTRIGSNPRIOR） | 关注 SGSN/SGW 节点迁移产生话单 |
| MS 时区更新触发 | CDRTRIGTIMEZONE=ENABLE（+ 可选 CDRTRIGTZPRIOR） | 关注跨时区漫游产生话单 |
| 计费条件改变触发 | CDRTRIGMAXCHNG=ENABLE（+ 可选 CDRTRIMAXCHGPRI） | 关注 QoS/ULI/RAI 改变次数达上限产生话单 |
| PLMN 标识更新触发 | CDRTRIGPLMNCHNG=ENABLE（+ 可选 CDRTRIGPLMNPRIO） | 关注 PLMN 归属变化产生话单 |
| MO Exception 触发 | CDRTRIGMOEXC=ENABLE（+ 可选 CDRTRIGMOEXCPRI） | 关注 MO Exception Data Counter 更新产生话单 |

取值来源为本端规划（业务规划决定哪些变更要落话单），跨场景取值不同 → 按模板逐一定制。

### 决策点：触发优先级配置（DP 0-03052）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 使用默认优先级（不配 PRIOR） | 各 PRIOR 省略，等同 255 | 不需要自定义优先顺序，采用默认优先级顺序 |
| 用户自定义优先级 | 各 PRIOR=1~100（值越小越优先） | 多个触发条件同时满足时，需指定哪个优先产生话单 |

用户配置优先级的开关比使用默认优先级（255）的开关优先级高；多个用户配置优先级之间取值越小越优先，且取值不能相同。

## 约束

· rule-0-03126 · severity=critical · 前置：只有执行 `ADD OFCTEMPLATE`（[0-00049](task/UNC/20.15.2/0-00049.md)）后才能配置本命令
· rule-0-03127 · severity=high · 唯一性：用户配置的优先级（1~100）不能相同；同值会冲突
· rule-0-03128 · severity=medium · 规格上限：最大记录数 100
· rule-0-03129 · severity=info · 默认优先级：未配置 PRIOR 的开关默认值 255（采用默认优先级），默认顺序由高到低为：计费条件改变 > Serving Node 更新 > MS 时区更新 > CDR RAT 更新 > Serving Node PLMN 标识更新
· rule-0-03130 · severity=info · 查看方式：本命令设定数据需通过 `LST OFCTEMPLATE` 查看（非 LST CDRTRIGGER）
