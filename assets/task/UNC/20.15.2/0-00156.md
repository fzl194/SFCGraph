# SET CDRTRIGGER 原子任务
> 命令静态知识见 [SET CDRTRIGGER](command/UNC/20.15.2/SET-CDRTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

配置离线计费模板后，按"哪些条件变更就产生一张话单"勾选开关；开关按"是否触发"成对出现，每对可选配优先级（1~100，数字越小越优先）。

## 配置方法
绑定到已建离线计费模板（OFCTEMPLATE），按 6 类触发条件勾选 + 可选优先级。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| OFCTEMPLATENAME | 离线计费模板名（来自已建 OFCTEMPLATE） | `offlinecharge-test` |
| CDRTRIGRATCHNG | CDR RAT 更新触发开关 | `ENABLE` / `DISABLE` |
| CDRTRIGRATPRIOR | RAT 更新优先级（开关使能时可选） | `1`~`100` |
| CDRTRIGSNCHNG | Serving Node 更新开关 | `ENABLE` |
| CDRTRIGSNPRIOR | Serving Node 更新优先级 | `1`~`100` |
| CDRTRIGTIMEZONE | MS 时区更新开关 | `DISABLE` |
| CDRTRIGTZPRIOR | MS 时区更新优先级 | `1`~`100` |
| CDRTRIGMAXCHNG | 计费条件改变开关 | `ENABLE` |
| CDRTRIMAXCHGPRI | 计费条件改变优先级 | `1`~`100` |
| CDRTRIGPLMNCHNG | Serving Node PLMN 标识更新开关 | `ENABLE` |
| CDRTRIGPLMNPRIO | Serving Node PLMN 标识优先级 | `1`~`100` |
| CDRTRIGMOEXC | MO Exception Data Counter 更新开关 | `DISABLE` |
| CDRTRIGMOEXCPRI | MO Exception Data Counter 更新优先级 | `1`~`100` |

典型脚本（仅使能 Serving Node、计费条件、PLMN 三类触发）：
```
SET CDRTRIGGER: OFCTEMPLATENAME="offlinecharge-test",CDRTRIGRATCHNG=DISABLE,CDRTRIGTIMEZONE=DISABLE,CDRTRIGPLMNCHNG=ENABLE;
```

## 决策点

### 触发开关的使能/禁止（每类条件）
- 问什么：该模板下，RAT/Serving Node/MS 时区/计费条件/PLMN/MO Exception 这 6 类变更中，哪几类变更触发产生话单。
- 视运营需求勾选：关注 RAT 切换→使能 RAT；关注用户位置/节点迁移→使能 Serving Node；关注 PLMN 归属变化→使能 PLMN。
- 取值来源为本端规划（业务规划决定哪些变更要落话单），跨场景取值不同→按模板逐一定制。

### 触发优先级配置
- 问什么：当多个触发条件同时满足时，哪个优先产生话单。
- 未配置优先级的开关默认值 255（采用默认优先级，默认顺序由高到低：计费条件改变 > Serving Node 更新 > MS 时区更新 > CDR RAT 更新 > Serving Node PLMN 更新）。
- 用户配置优先级的开关比使用默认优先级的开关优先级高；同配置优先级取值越小越优先。

## 约束
- 前置：只有执行 `ADD OFCTEMPLATE`（[0-00049](task/UNC/20.15.2/0-00049.md)）后才能配置本命令。`severity=high`
- 生效时机：执行后立即生效。`severity=low`
- 规格上限：最大记录数 100。`severity=medium`
- 优先级默认：未配置优先级的开关默认值 255，采用默认优先级顺序。`severity=medium`
