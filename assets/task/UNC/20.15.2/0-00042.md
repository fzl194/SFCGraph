# 创建Diameter本端主机组（ADD LOCALHOSTGRP）

> 命令静态知识见 [ADD LOCALHOSTGRP](command/UNC/20.15.2/ADD-LOCALHOSTGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD LOCALHOSTGRP 创建一个 Diameter 本端主机组（仅组名 `LOCGRPNAME`，不含主机成员）。主机成员由后续 [ADD LOCALHOSTBIND](task/UNC/20.15.2/0-00041.md) 关联进组。该组供同一 PGW-C 对接多个 PGW-U、且不同 PGW-U 下用户 IP 可能重叠时，做 Diameter 本端多主机归组。

核心配置空间有 1 个维度：**组名**（必选，本端规划命名）。

### 配置维度 1：组名（参数 LOCGRPNAME，必选）

| 取值样例 | 数据来源 | 约束 |
|---|---|---|
| `pgwc_host_group_01` / `pgwc_host_group_02` / `localgroup` | 本端规划（运营商规划） | 1~63 字符，不支持空格，不区分大小写；要求系统内唯一 |

> LOCGRPNAME 为 local_planned（本端规划），命名以便运维识别；证据样例为 `localgroup`。

**典型脚本（PCRF 对接场景，建两个组）**

```
ADD LOCALHOSTGRP: LOCGRPNAME="pgwc_host_group_01";
ADD LOCALHOSTGRP: LOCGRPNAME="pgwc_host_group_02";
```

**步骤位置**：在"PCC 基本功能（5G）"对接流程中，位于 ADD DIAMLOCINFO（建主机）之前或之后、ADD LOCALHOSTBIND（绑定）之前。典型顺序：ADD LOCALHOSTGRP（建组）→ ADD DIAMLOCINFO（建主机）→ ADD LOCALHOSTBIND（组下绑主机）。

## 决策点

### 决策点：是否需要建本端主机组（DP 0-02063）

问：什么场景下需要建 Diameter 本端主机组（决定本步是否执行）？

| 场景条件 | 动作 | 说明 |
|---|---|---|
| 多 PGW-U + 用户 IP 重叠 | 必须建组并做主机绑定 | Diameter 本端多主机归组，解决 IP 重叠 |
| 多 PGW-U + 用户 IP 不重叠 | 可不建组 | 本步可省略 |
| 单 PGW-U | 一般不建组 | 无多主机归组需求 |

> 是否建组由 PGW-C 对接 PGW-U 拓扑与用户 IP 规划决定（global_planned / 对端协商）；无 IP 重叠时可省略本步。

### 决策点：建几个组（DP 0-02064）

问：确定要建组时，建几个 Diameter 本端主机组（决定 ADD LOCALHOSTGRP 执行次数与命名）？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 一主机名一组（常见范式） | `pgwc_1` 占 `pgwc_host_group_01`、`pgwc_2` 占 `pgwc_host_group_02` | 证据范例范式，每实例独占一组 |
| 多主机名合一组（负载分担） | 多个主机名绑同一 `LOCGRPNAME`（最多 64 个，由 ADD LOCALHOSTBIND 实现） | 单组内多主机归组 |

> 组数与命名由 PGW-C 本端主机规划决定（local_planned）；建组本身是空壳，成员靠 ADD LOCALHOSTBIND 关联。

## 约束

- rule-0-02156 · severity=critical · 唯一性：LOCGRPNAME 要求系统内唯一（1~63 字符，不支持空格，不区分大小写），重复命名新增失败
- rule-0-02157 · severity=high · 前置依赖（反向）：组本身是空壳，建组后须由 ADD LOCALHOSTBIND 关联本端主机才产生实际效用，未绑定主机则组无实际意义
- rule-0-02158 · severity=warning · 规格上限：本命令最大记录数为 100，超过上限新增失败
- rule-0-02159 · severity=warning · 命名约束：LOCGRPNAME 1~63 字符、不支持空格、不区分大小写，违规被拒
- rule-0-02160 · severity=info · 立即生效：命令执行后立即生效，无需重启
