# ADD LOCALHOSTGRP 原子任务

> 命令静态知识见 [ADD LOCALHOSTGRP](command/UNC/20.15.2/ADD-LOCALHOSTGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD LOCALHOSTGRP 只创建一个"主机组名"容器（唯一参数 `LOCGRPNAME`），不含主机成员。主机成员由后续 [ADD LOCALHOSTBIND](command/UNC/20.15.2/ADD-LOCALHOSTBIND.md) 关联进组。该组供同一 PGW-C 对接多个 PGW-U、且不同 PGW-U 下用户 IP 可能重叠时，做 Diameter 本端多主机归组。

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| LOCGRPNAME | `pgwc_host_group_01`<br>`pgwc_host_group_02` | 本端规划 | 主机组名，1~63 字符，不支持空格，不区分大小写 |

**典型行（PCRF 对接场景，建两个组）**

```
ADD LOCALHOSTGRP: LOCGRPNAME="pgwc_host_group_01";
ADD LOCALHOSTGRP: LOCGRPNAME="pgwc_host_group_02";
ADD LOCALHOSTBIND: LOCGRPNAME="pgwc_host_group_01", LOCHOSTNAME="pgwc_1";
ADD LOCALHOSTBIND: LOCGRPNAME="pgwc_host_group_02", LOCHOSTNAME="pgwc_2";
```

**何时建组（决策依据）**：同一 PGW-C 对接多个 PGW-U，且不同 PGW-U 下用户 IP 存在相同时**必须**配置；IP 不同时可省略本步。

## 决策点

### 是否需要建本端主机组

| 条件 | 动作 |
|---|---|
| 多 PGW-U + 用户 IP 重叠 | 必须建组并做主机绑定 |
| 多 PGW-U + 用户 IP 不重叠 | 可不建组（本步可省） |
| 单 PGW-U | 一般不建组 |

### 建几个组

由 PGW-C 本端主机规划决定，一个本端主机名对应一个组是常见范式（见上 `pgwc_1`/`pgwc_2` 各占一组）。

## 约束

- **立即生效** · severity=LOW · 配置后无需重启。
- **最大 100 条记录** · severity=MEDIUM · 超过上限新增失败。
- **命名约束**：LOCGRPNAME 1~63 字符、不支持空格、不区分大小写 · severity=MEDIUM。
- **组本身是空壳**：成员靠 ADD LOCALHOSTBIND 关联，建组后未绑定主机则组无实际效用 · severity=MEDIUM。
