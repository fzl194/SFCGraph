## 配置方法

查询范式分两步：先用 SIMPLE + IMSI 查到 PDU 会话 ID，再用 DETAILED + 网络类型 + PDU 会话 ID + QFI 查 QoS Flow 级详情。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 查询方式 | QUERYTYPE | IMSI / MSISDN / IMEI / IPV4ADDR / IPV6ADDR（必选） | 无 | 指定按哪种用户标识定位会话 | 调测脚本默认 IMSI；IP 地址仅支持 SIMPLE |
| 用户标识 | IMSI | 字符串，5~15位 | 条件必选（QUERYTYPE=IMSI 时） | 5G 对应输出为 SUPI | IMSI 查询 |
| 用户标识 | MSISDN | 字符串，1~15位 | 条件必选（QUERYTYPE=MSISDN 时） | 按手机号查 | MSISDN 查询 |
| 用户标识 | MEI | 字符串，1~16位 | 条件必选（QUERYTYPE=IMEI 时） | 仅在 Emergency Request 且 IMSI 非法时支持 DETAILED | IMEI 查询 |
| 信息呈现 | DSPINFOTYPE | SIMPLE / DETAILED | SIMPLE | SIMPLE 查会话概要；DETAILED 查 QoS Flow 级详情 | 默认 SIMPLE；深度核对用 DETAILED |
| 无线网络 | WLNETTWTYPE | NW5G / NW2G3G4G | 条件必选（DETAILED 时） | 5G 用 NW5G；2/3/4G 用 NW2G3G4G | DETAILED 必带 |
| 会话定位 | PDUSESSIONID | 整数，1~255 | 条件必选（NW5G 时） | 定位具体 PDU 会话 | NW5G DETAILED 必带 |
| QoS流定位 | QFI | 整数，1~63 | 条件必选（NW5G 时） | 定位具体 QoS Flow | NW5G DETAILED 必带 |
| 承载定位 | EBIORNSAPI | 整数，1~15 | 条件必选（NW2G3G4G 时） | 指定 EPS 承载标识；EBI=0 表示未分配 | NW2G3G4G DETAILED 必带 |
| 接入方式 | ACCESSTYPE234G | AT_3GPP_ACCESS / AT_UNTRUSTED_NON_3GPP_ACCESS / AT_TRUSTED_NON_3GPP_ACCESS | 条件必选（NW2G3G4G 时） | 3GPP 接入 vs 非可信非 3GPP 接入 | VoLTE 紧急呼叫用 AT_3GPP_ACCESS；VoWIFI 紧急呼叫用 AT_UNTRUSTED_NON_3GPP_ACCESS |
| IP地址查 | UEIPV4ADDR | IPv4 地址 | 条件必选（QUERYTYPE=IPV4ADDR 时） | 仅支持 SIMPLE 方式 | IP 地址查会话（仅 SIMPLE） |
| IP地址查 | UEIPV6ADDR | IPv6 地址 | 条件必选（QUERYTYPE=IPV6ADDR 时） | 仅支持 SIMPLE 方式 | IPv6 地址查会话（仅 SIMPLE） |
| IP域 | IPDOMAIN | 字符串，0~255 | 条件可选（QUERYTYPE=IPV4ADDR 时） | 按 IPv4 地址段归属的域过滤 | 已划分域的场景 |

典型脚本一（SIMPLE 查会话 ID，5G 接入管理与分流调测入口）：
```
DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=SIMPLE;
```

典型脚本二（DETAILED 查 QoS Flow 级详情，分流/MEC/语音专载核对）：
```
DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=DETAILED, WLNETTWTYPE=NW5G, PDUSESSIONID=5, QFI=1;
```

典型脚本三（4G 紧急呼叫/互操作场景，按 EBI + 接入方式查 PDN 连接）：
```
DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=DETAILED, WLNETTWTYPE=NW2G3G4G, EBIORNSAPI=5, ACCESSTYPE234G=AT_3GPP_ACCESS;
```

调测语义（按场景）：
- 激活成功核对：SIMPLE 查到会话即激活成功；核对"用户 IP 地址 / UP 的 PFCP IPv4 地址 / APN 或 DNN / PCF ID"与规划一致。
- 分流策略（ULCL/I-UPF）：DETAILED 核对是否选到预期 ULCL UPF + 辅锚点 UPF，会话是否插入正确 UPF。
- MEC 冗余/单点故障保护：故障前后各查一次，比对 UPF 是否按预期主备切换。
- 语音专载（VoNR/VoLTE/EPS Fallback）：核对是否存在 IMS 的 PDU 会话（APN=ims）、5QI=1 / QCI=5 的语音专载 QoS Flow。
- 惯性运行（用户面业务通道保活）：核对用户是否进入惯性运行状态（输出项"惯性运行状态"）。
- ARP 差异化/PDP 数门限：核对低优先级用户是否被拒（上下文不存在）、高优先级用户是否接入成功。

## 决策点

### 决策点：查询方式与标识选择（DP 0-01256）
| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IMSI（调测脚本默认） | QUERYTYPE=IMSI + IMSI=xxx | 5G/4G/3G/2G 通用入口 |
| MSISDN | QUERYTYPE=MSISDN + MSISDN=xxx | 按手机号查 |
| MEI | QUERYTYPE=IMEI + MEI=xxx | 设备标识查（仅 Emergency Request+IMSI 非法时支持 DETAILED） |
| IPV4ADDR/IPV6ADDR（仅 SIMPLE） | QUERYTYPE=IPV4ADDR/IPV6ADDR + UEIPV4ADDR/UEIPV6ADDR | I-SMF/V-SMF 不支持按 IP 地址查询，需改用 IMSI/MSISDN |

### 决策点：信息呈现方式（DP 0-01257）
| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| SIMPLE（默认） | 无需额外参数 | 看会话是否存在、用户 IP、UPF、APN，获取 PDUSESSIONID 供下一步 |
| DETAILED | 必带 WLNETTWTYPE + PDUSESSIONID + QFI（NW5G）或 EBIORNSAPI + ACCESSTYPE234G（NW2G3G4G） | 查 QoS Flow / 专载 / 用户面深度信息 |

## 约束

· rule-0-01576 · severity=high · I-SMF/V-SMF 不支持按 IP 地址（QUERYTYPE=IPV4ADDR/IPV6ADDR）查询会话信息，需改用 IMSI/MSISDN
· rule-0-01577 · severity=high · 按 IP 地址（UEIPV4ADDR/UEIPV6ADDR）查询仅支持 SIMPLE 方式，DETAILED 必须用 IMSI/MSISDN/IMEI
· rule-0-01578 · severity=medium · 2B2C 漫游双 DNN 用户（通用 DNN 会话 4G + 专用 DNN 会话 5G）在 SIMPLE 查询结果中，专用 DNN 会话记录的 EBI 字段表示该专用 DNN 会话的 PDU Session ID（语义特殊，需注意）
· rule-0-01579 · severity=low · QoS 详细信息请用 DSP SESSIONQOSINFO，签约详细信息请用 DSP SESSIONSUBDATA（本命令不含）
· rule-0-01580 · severity=info · DETAILED 方式下 WLNETTWTYPE 与实际网络不符会导致查询无果，需先确认用户所在网络制式
