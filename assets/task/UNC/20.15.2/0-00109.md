# DSP PDUSESSION 原子任务
> 命令静态知识见 [DSP PDUSESSION](command/UNC/20.15.2/DSP-PDUSESSION.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：SMF、PGW-C、SGW-C、GGSN。本命令是会话层调测的通用核对入口——验证用户是否激活成功、UPF/IP/P-CSCF 选择是否符合规划、专载与 QoS Flow 是否建立、故障场景下 UPF 是否按预期重选。QoS 详情用 DSP SESSIONQOSINFO；签约详情用 DSP SESSIONSUBDATA。

## 配置方法
查询范式分两步：先用 SIMPLE + IMSI 查到 PDU 会话 ID，再用 DETAILED + 网络类型 + PDU 会话 ID + QFI 查 QoS Flow 级详情。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| QUERYTYPE | 查询方式（必选） | `IMSI`（按用户 IMSI 查） |
| IMSI | 当 QUERYTYPE=IMSI 时的标识 | `123000123456789` |
| MSISDN | 当 QUERYTYPE=MSISDN 时 | `13800138000` |
| MEI | 当 QUERYTYPE=MEI 时（IMEI） | `1234567890123456` |
| UEIPV4ADDR | 按 IPv4 地址查（仅 SIMPLE） | `192.168.1.24` |
| UEIPV6ADDR | 按 IPv6 地址查（仅 SIMPLE） | `2001:db8::1` |
| DSPINFOTYPE | 信息呈现方式（默认 SIMPLE） | `SIMPLE` / `DETAILED` |
| WLNETTWTYPE | 无线网络类型（DETAILED 必带） | `NW5G` / `NW2G3G4G` |
| PDUSESSIONID | PDU 会话 ID（DETAILED 定位用） | `5` |
| QFI | QoS 流 ID（DETAILED 定位用） | `1` |
| EBIORNSAPI | 4G EBI 或 NSAPI | `5` |
| ACCESSTYPE234G | 4G 接入方式 | `AT_3GPP_ACCESS` / `AT_UNTRUSTED_NON_3GPP_ACCESS` |
| IPDOMAIN | 用户 IP 域 | 按规划 |

典型脚本一（SIMPLE 查会话 ID，5G 接入管理与分流调测入口）：
```
DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=SIMPLE;
```

典型脚本二（DETAILED 查 QoS Flow 级详情，分流/MEC/语音专载核对）：
```
DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=DETAILED, WLNETTWTYPE=NW5G, PDUSESSIONID=5, QFI=1;
```

典型脚本三（4G 紧急呼叫/互操作场景，按 EBI + 接入方式查 PDN 连接）：
```
DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=DETAILED, WLNETTWTYPE=NW2G3G4G, EBIORNSAPI=5, ACCESSTYPE234G=AT_3GPP_ACCESS;
```

调测语义（按场景）：
- 激活成功核对：SIMPLE 查到会话即激活成功；核对"用户 IP 地址 / UP 的 PFCP IPv4 地址 / APN 或 DNN / PCF ID"与规划一致。
- 分流策略（ULCL/I-UPF）：DETAILED 核对是否选到预期 ULCL UPF + 辅锚点 UPF，会话是否插入正确 UPF。
- MEC 冗余/单点故障保护：故障前后各查一次，比对 UPF 是否按预期主备切换。
- 语音专载（VoNR/VoLTE/EPS Fallback）：核对是否存在 IMS 的 PDU 会话（APN=ims）、5QI=1 / QCI=5 的语音专载 QoS Flow。
- 惯性运行（用户面业务通道保活）：核对用户是否进入惯性运行状态。
- ARP 差异化/PDP 数门限：核对低优先级用户是否被拒（上下文不存在）、高优先级用户是否接入成功。

## 决策点（若有）

### 查询方式与标识选择（QUERYTYPE）
- 问什么：用哪种标识定位会话。
- IMSI（调测脚本默认）→ QUERYTYPE=IMSI 配 IMSI。
- MSISDN / MEI / IP（IPV4ADDR/IPV6ADDR）→ 仅支持 SIMPLE 方式。
- I-SMF/V-SMF 不支持按 IP 地址（IPV4ADDR/IPV6ADDR）查询，需改用 IMSI/MSISDN。

### 信息呈现方式（DSPINFOTYPE）
- 问什么：要看会话概要还是 QoS Flow 级详情。
- SIMPLE（默认）→ 看会话是否存在、用户 IP、UPF、APN，并获取 PDUSESSIONID 供下一步。
- DETAILED → 必带 WLNETTWTYPE + PDUSESSIONID + QFI，查 QoS Flow / 专载 / 用户面深度信息。

### 无线网络类型（WLNETTWTYPE，DETAILED 必填）
- 问什么：会话在 5G 还是 2/3/4G 网络。
- NW5G → 5G PDU 会话（分流、VoNR、5G LAN 等场景）。
- NW2G3G4G → 2/3/4G PDN 连接（VoLTE、EPS Fallback、SGSN 黑白名单等场景）。

### 4G 接入方式（ACCESSTYPE234G）
- 问什么：4G 侧是 3GPP 接入还是非可信非 3GPP 接入。
- AT_3GPP_ACCESS → VoLTE 紧急呼叫、常规 3GPP 接入。
- AT_UNTRUSTED_NON_3GPP_ACCESS → VoWIFI 紧急呼叫等非可信非 3GPP 接入。

## 约束（若有）
- I-SMF/V-SMF 不支持按 IP 地址（QUERYTYPE 为 IPV4ADDR/IPV6ADDR）查询会话信息。`severity=high`
- 按 IP 地址（UEIPV4ADDR/UEIPV6ADDR）查询仅支持 SIMPLE 方式。`severity=high`
- SIMPLE 方式下，2B2C 漫游双 DNN 用户（通用 DNN 会话 4G + 专用 DNN 会话 5G）查询结果中专用 DNN 会话记录的 EBI 字段表示该专用 DNN 会话的 PDU Session ID（语义特殊，需注意）。`severity=medium`
- QoS 详细信息请用 DSP SESSIONQOSINFO，签约详细信息请用 DSP SESSIONSUBDATA（本命令不含）。`severity=low`
