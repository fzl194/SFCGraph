# OFCTemplate 绑定计费属性 CC

> 阶段定位：离线计费参数绑到 Charge Characteristic 层（优先级次于 APN）。被引用于：[2-00001 离线计费](task/UNC/20.15.2/2-00001.md)。
> 域通用 compound（在线/融合计费模板绑 CC 同此结构）。

## 配置方法

命令序列（建 CC → CC 绑模板）：

1. [ADD CHARGECHAR](task/UNC/20.15.2/0-00014.md) — 计费属性（CCNAME 本端规划）；已配则跳过
2. [ADD GLBOFCTEMPLATE](task/UNC/20.15.2/0-00033.md) — CC 绑 OFCTemplate（GLOBALFLAG=CHARGE_CHARACT + CCVALUE + PGWTEMPLATE/SGWTEMPLATE）

**典型脚本**
```
ADD CHARGECHAR: CCNAME="testchgcha";
ADD GLBOFCTEMPLATE: GLOBALFLAG=CHARGE_CHARACT, CCVALUE=0x0003, PGWTEMPLATE="offlinecharge-test", SGWTEMPLATE="offlinecharge-test";
```

## 约束
- 前置：[1-00001 模板参数](task/UNC/20.15.2/1-00001.md)
- GLOBALFLAG=CHARGE_CHARACT（区别于全局的 GLOBAL）；CCVALUE 与 ADD CHARGECHAR 的 CC 对应
