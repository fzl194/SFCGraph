# SET N7RCVATTRCTRL 原子任务
> 命令静态知识见 [SET N7RCVATTRCTRL](command/UNC/20.15.2/SET-N7RCVATTRCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

该命令是 N7 接口接收侧信元处理的细粒度开关集合（6 参数），被 WSFD-109101（激活 PCC 基本功能）按对接 PCF 能力分场景可选配置。各参数独立含义，激活 md 中以单参数配置为主。系统部署后已存在初始记录。

## 配置方法
按 PCF 对接能力逐项配置需要的信元处理参数；未输入参数保持系统当前值不变（可通过 LST N7RCVATTRCTRL 查看）。激活 md 实际显式配置的典型参数如下。

| 参数 | 描述 | 配置方法 | 典型场景 | 取值样例 |
|---|---|---|---|---|
| NOPKTFLTUSAGE | 无 PacketFilterUsage 处理 | 本端规划 | PCF 不下发 packetFilterUsage 时，将 FlowInformation 映射的 Filter 推送给 UE | `SEND_PKTFLT_TO_UE` / `NOT_SEND_PKTFLT_TO_UE` |
| SCELLCHNR | SCELL_CH 是否允许上报 NCGI | 对端协商 | PCF 订阅 SCELL_CH 场景，SMF 向 PCF 上报 NCGI | `ENABLE` / `DISABLE` |
| MODQOSKEYATTR | 修改 QoSData 关键属性 | 本端规划 | 按 PCF 下发的 QoS 关键属性修改 QoSFlow | `MOD_QOSFLOW` 等 |
| DFTQOSRULEGEN | 缺省 QosRule 生成方法 | 本端规划 | 优先选择安装到缺省 qosflow 的 match-all PCC 规则 | `PCC_RULE` 等 |
| NTFRSRURI | 基于 Notify 消息 ResourceURI 触发 PCF 重选 | 本端规划 | 是否忽略 PCF UpdateNotify 携带的 ResourceURI | `DISABLE` / `ENABLE` |
| RSPLOCURI | 是否使用 PCF 返回的 URI 转发 | global_planned | 向新 URI 标识的 PCF 转发 | 开关枚举 |

典型脚本（WSFD-109101，PCF 不支持 packetFilterUsage）：
```
SET N7RCVATTRCTRL: NOPKTFLTUSAGE=SEND_PKTFLT_TO_UE;
```
典型脚本（WSFD-109101，PCF 订阅 SCELL_CH 上报 NCGI）：
```
SET N7RCVATTRCTRL: SCELLCHNR=ENABLE;
```

## 决策点

### NOPKTFLTUSAGE 取值（PCF 能力分叉）
- 问什么：PCF 不下发 packetFilterUsage 时，SMF 是否将映射后的 Filter 推送给 UE。
- 取值由对端 PCF 是否支持下发 packetFilterUsage 属性决定：不支持时配 SEND_PKTFLT_TO_UE，跨场景枚举（②conditional 分叉，按 PCF 能力二选一）。

### SCELLCHNR 取值（PCF 订阅分叉）
- 问什么：PCF 是否订阅 SCELL_CH，SMF 是否上报 NCGI。
- 取值由对端 PCF 是否订阅 SCELL_CH 决定；对端协商来源。注意：SMF 向 PCF 上报 ECGI 无开关控制，默认支持。

### 参数组合（按场景可选）
- 问什么：6 个参数中哪些需要配置。
- 取值由 PCC 对接清单驱动：激活 md 将每项标注为"（可选）"，配置生成需逐条对照对接 PCF 能力勾选子集；同一命令在一次配置中可只配 1 个参数。

## 约束
- 唯一性：全局一份（系统部署后已存在初始记录）。`severity=medium`
- 生效时机：执行后立即生效。`severity=high`
- 对端协商：SCELLCHNR 取值依赖对端 PCF 能力，需与对端确认是否订阅 SCELL_CH。`severity=high`
- 默认值：未输入参数保持系统当前值不变；初始记录 NOPKTFLTUSAGE=SEND_PKTFLT_TO_UE 等。`severity=low`
- 适用 NF：SMF。`severity=low`
