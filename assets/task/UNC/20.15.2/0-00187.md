# 设置N7接收信元处理控制（SET N7RCVATTRCTRL）（SET N7RCVATTRCTRL）

> 命令静态知识见 [SET N7RCVATTRCTRL](command/UNC/20.15.2/SET-N7RCVATTRCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按 PCF 对接能力逐项配置需要的信元处理参数。系统部署后已存在初始记录（NOPKTFLTUSAGE=SEND_PKTFLT_TO_UE、MODQOSKEYATTR=MOD_QOSFLOW、DFTQOSRULEGEN=PCC_RULE、NTFRSRURI=DISABLE、SCELLCHNR=DISABLE、RSPLOCURI=DISABLE）。未输入参数保持系统当前值不变（可通过 LST N7RCVATTRCTRL 查看）。激活 md 中以单参数配置为主。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| PacketFilterUsage 处理 | `NOPKTFLTUSAGE` | `NOT_SEND_PKTFLT_TO_UE` / `SEND_PKTFLT_TO_UE` | SEND_PKTFLT_TO_UE | PCF 未下发 packetFilterUsage 时是否把映射 Filter 推送给 UE | PCF 不遵循 3GPP 协议不携带 PacketUsageFilter |
| QoS 关键属性修改 | `MODQOSKEYATTR` | `MOD_QOSFLOW` / `DEL_ADD_QOSFLOW` | MOD_QOSFLOW | 收到 PCF 改 QoS 关键属性时改 QoSFlow 还是重建 | 5QI/Arp/QNC 等 6 个关键属性变化 |
| 缺省 QosRule 生成 | `DFTQOSRULEGEN` | `PCC_RULE` / `MATCH_ALL_RULE` | PCC_RULE | 缺省 QosRule 选择策略 | 缺省 qosflow 规则选择 |
| PCF 重选触发 | `NTFRSRURI` | `DISABLE` / `ENABLE` | DISABLE | UpdateNotify 携带不同 ResourceURI 是否触发 PCF 重选 | PCF 重选组网 |
| SCELL_CH 上报 NCGI | `SCELLCHNR` | `DISABLE` / `ENABLE` | DISABLE | PCF 下发 SCELL_CH/USER_LOCATION_CH 时是否上报 NCGI | PCF 订阅 SCELL_CH 场景（对端协商） |
| 响应 URI 转发 | `RSPLOCURI` | `DISABLE` / `ENABLE` | DISABLE | PCF 激活响应消息头 URI 变化时是否改用新 URI | 全网规划：向新 URI 标识的 PCF 转发 |

典型脚本（PCF 不支持 packetFilterUsage，向 UE 推送 Filter）：
```
SET N7RCVATTRCTRL: NOPKTFLTUSAGE=SEND_PKTFLT_TO_UE;
```

典型脚本（PCF 订阅 SCELL_CH 上报 NCGI）：
```
SET N7RCVATTRCTRL: SCELLCHNR=ENABLE;
```

步骤位置：激活 PCC 基本功能（WSFD-109101）"配置 N7 接收侧信元处理"步骤——按对接 PCF 能力清单逐项勾选子集，紧随 SET PCCFUNC 前置能力开关之后。

## 决策点

### 决策点：NOPKTFLTUSAGE 取值（DP 0-03113）

PCF 不下发 packetFilterUsage 时，SMF 是否将映射后的 Filter 推送给 UE；取值由对端 PCF 是否支持下发 packetFilterUsage 决定。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 推送给 UE | `NOPKTFLTUSAGE=SEND_PKTFLT_TO_UE` | PCF 不支持 packetFilterUsage，需 UE 侧感知 Filter |
| 不推送 | `NOPKTFLTUSAGE=NOT_SEND_PKTFLT_TO_UE` | PCF 自管 Filter 激活/去活时间，规则所有 Filter 不推 UE |

> 关键约束：当规则上的 Filter 均未推送给 UE 时，才允许 PCF 为该规则指定激活或去活时间，否则规则安装失败。

### 决策点：参数子集勾选（DP 0-03114）

6 个参数中哪些需要配置；取值由 PCC 对接能力清单驱动——激活 md 将每项标注为"（可选）"，配置生成需逐条对照对接 PCF 能力勾选子集。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单参数配置（常见） | 仅配 1 个参数 | 按对接清单仅一项能力分叉 |
| 多参数组合 | 配 2 个及以上参数 | 多项能力同时分叉（NOPKTFLTUSAGE + SCELLCHNR 等） |

> 同一命令在一次配置中可只配 1 个参数；未输入参数保持系统当前值不变。

## 约束

- **该命令执行后立即生效**（warning，rule-0-03281）：该命令执行后立即生效，无延迟窗口 — 后果：误以为有窗口期导致操作时序混乱
- **SCELLCHNR 取值依赖对端 PCF 能力**（warning，rule-0-03282）：SCELLCHNR 取值依赖对端 PCF 能力，需与对端确认是否订阅 SCELL_CH — 后果：PCF 未订阅时 ENABLE 上报会被对端丢弃或报错
- **RSPLOCURI 使能时更新或去活失败不进行 Failover**（warning，rule-0-03283）：RSPLOCURI 使能时更新或去活失败不进行 Failover；非直连组网下仅控制 Model C 模式，Model D 由软参 DWORD548 BIT4 控制 — 后果：误以为全部模式生效导致 Failover 行为不符预期
- **全局唯一一份（系统部署后已存在初始记录）**（info，rule-0-03284）：全局唯一一份（系统部署后已存在初始记录），未输入参数保持系统当前值不变 — 后果：漏填参数沿用初始值可能与规划不符
- **适用 NF 仅 SMF**（info，rule-0-03285）：适用 NF 仅 SMF；SMF 向 PCF 上报 ECGI 无开关控制，默认支持 — 后果：误以为 ECGI 也有开关导致配置遗漏
