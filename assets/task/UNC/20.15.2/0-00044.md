# 配置逻辑IP地址（ADD LOGICIP）

> 命令静态知识见 [ADD LOGICIP](command/UNC/20.15.2/ADD-LOGICIP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD LOGICIP 是平台层的"逻辑 IP 池"配置，先于 [ADD LOGICINF](command/UNC/20.15.2/ADD-LOGICINF.md) 配置——逻辑接口引用这里的 IP + VPN。该命令被 7 个特性共用，覆盖计费/策略接口（Gx/Gy/Ga）、AAA 对接（Gi）、SBI 加密、QoS 能力开放（Restif IPv6）等。核心范式：**与 ADD LOGICINF 成对配置，"IP + VPN"必须一致**。配置空间有 3 个维度：IP 版本与地址（必选）、VPN 实例（可选，决定隔离域）、MTU/描述（可选，调优）。

### 配置维度 1：IP 版本与地址（参数 IPVERSION + LOGICIPV4/LOGICIPV6，必选）

| 取值 | 地址参数 | 取值约束 | 典型场景 |
|---|---|---|---|
| **IPv4**（常用） | `LOGICIPV4` | 不能是 0.0.0.0、255.255.255.255、环回（127.x）；必须 A/B/C 类合法地址 | Gx/Gy/Ga/Gi/S6b 本端业务 IP |
| **IPv6** | `LOGICIPV6` | 必须全球单播；不能为 ::、FFFF:FFFF:FFFF:FFFF、环回 ::1、链路本地 FE80::/10、组播 FF00::/8 | QoS 能力开放 Restif（WSFD-109101） |

### 配置维度 2：VPN 实例（参数 VPNINSTNAME，可选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| 已有 VPN 实例名（如 `vpn_gxif`、`vpn_gy`、`vpn_aaa`、`vpn_s6bif`、`vpn_restif`） | IP 归属该 L3VPN 实例 | 接口走 L3VPN 隔离（多数对接面） |
| `_public_` | 归属公网 VPN | SBI 加密场景（WSFD-010308） |
| 不填 | 默认 `_public_` | 公网直通 |

> 前置依赖：VPNINSTNAME 需先由 [ADD VPNINST](command/UNC/20.15.2/ADD-VPNINST.md) 创建（可用 [LST VPNINST](command/UNC/20.15.2/LST-VPNINST.md) 查询）。**与 ADD LOGICINF 的 VPNINSTANCE 必须一致**——见约束 rule-0-02168。

### 配置维度 3：MTU 与描述（参数 IPV4MTU/IPV6MTU/DESC，可选）

| 参数 | 取值范围 | 默认 | 适用场景 |
|---|---|---|---|
| `IPV4MTU` | 328~9600（存在 NP121 卡时上限 3600） | 1500 | 链路 MTU 受限时显式改 |
| `IPV6MTU` | 1280~9600（存在 NP121 卡时上限 3600） | 1500 | 同上 |
| `DESC` | 0~255 字符 | 无 | 备注用途（如 `Restful_IPv6_Client`） |

> 是否配 NP121 卡可用 DSP NPNODE 查询。

**典型脚本（IPv4 + VPN，Gx 接口底层 IP）**
```
ADD LOGICIP: IPVERSION=IPv4, LOGICIPV4="10.8.10.1", VPNINSTNAME="vpn_gxif";
ADD LOGICINF: NAME="gxif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.10.1", IPV4MASK1="255.255.255.255", VPNINSTANCE="vpn_gxif";
```

**典型脚本（IPv6，QoS 能力开放 Restif）**
```
ADD LOGICIP: IPVERSION=IPv6, LOGICIPV6="fe80:8029:5008:6B::247", VPNINSTNAME="vpn_restif", DESC="Restful_IPv6_Client";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：配置逻辑 IP"——是最早的平台层 IP 配置，先于 [ADD LOGICINF](task/UNC/20.15.2/0-00043.md)，更早于对接命令（[ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md) 等）。

## 决策点：IP 版本选择（DP 0-02067）

问：该逻辑 IP 走 IPv4 还是 IPv6？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPv4 | `IPVERSION=IPv4` + `LOGICIPV4` +（可选 `IPV4MTU`） | 多数计费/策略接口（Gx/Gy/Ga/Gi/S6b）、SBI 加密 |
| IPv6 | `IPVERSION=IPv6` + `LOGICIPV6` +（可选 `IPV6MTU`） | QoS 能力开放 Restif 接口（WSFD-109101） |

### 决策点：VPN 实例归属（DP 0-02068）

问：该逻辑 IP 归属哪个 VPN 实例？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 接口所属 VPN（如 `vpn_gxif`/`vpn_gy`/`vpn_ga`/`vpn_aaa`/`vpn_s6bif`/`vpn_restif`） | `VPNINSTNAME=<接口VPN>` | L3VPN 隔离的对接面（主流） |
| 公网 VPN | `VPNINSTNAME=_public_` | SBI 加密（WSFD-010308） |
| 不填（默认 `_public_`） | 省略 VPNINSTNAME | 公网直通（罕见） |

> Gi 场景证据明确：ADD LOGICIP 用 `vpn_pdn`，但 ADD LOGICINF 的 VPNINSTANCE 取 AAA 服务器所属 VPN（vpn_aaa/vpn_enterprise）——这是证据中的真实差异，配置时以数据规划表为准。

## 约束

- ****只在 UNC 网元下生效****（critical，rule-0-02166）：**只在 UNC 网元下生效**：UDG 不用本命令；跨产品规划时注意适用范围
- ****IP 地址合法性****（critical，rule-0-02167）：**IP 地址合法性**：IPv4 不能是 0.0.0.0、255.255.255.255、环回（127.x）；IPv6 必须全球单播，不能为 ::、FFFF:FFFF:FFFF:FFFF、环回 ::1、链路本地 FE80::/10、组播 FF00::/8，否则下发失败
- ****与 ADD LOGICINF 成对一致****（critical，rule-0-02168）：**与 ADD LOGICINF 成对一致**：本命令的 "LOGICIPV4+VPNINSTNAME" 必须与 ADD LOGICINF 的 "IPV4ADDRESS1+VPNINSTANCE" 完全一致，否则接口绑定失败
- ****VPN 前置依赖****（warning，rule-0-02169）：**VPN 前置依赖**：VPNINSTNAME 需先由 ADD VPNINST 创建，否则下发失败
- ****MTU 范围与上限****（warning，rule-0-02170）：**MTU 范围与上限**：IPV4MTU 328~9600、IPV6MTU 1280~9600；存在 NP121 NP 卡时上限收窄至 3600，超范围下发失败
