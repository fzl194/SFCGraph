# ADD LOGICIP 原子任务

> 命令静态知识见 [ADD LOGICIP](command/UNC/20.15.2/ADD-LOGICIP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD LOGICIP 是平台层的"逻辑 IP 池"配置，先于 [ADD LOGICINF](command/UNC/20.15.2/ADD-LOGICINF.md) 配置——逻辑接口引用这里的 IP + VPN。该命令被 7 个特性共用，覆盖计费/策略接口（Gx/Gy/Ga）、AAA 对接（Gi）、SBI 加密、QoS 能力开放（Restif IPv6）等。

**核心范式：与 ADD LOGICINF 成对配置，"IP + VPN"必须一致**

证据反复强调：ADD LOGICIP 的 `LOGICIPV4 + VPNINSTNAME` 必须与后续 ADD LOGICINF 的 `IPV4ADDRESS1 + VPNINSTANCE` 完全一致。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| IPVERSION | `IPv4` / `IPv6` | 全网规划/本端规划 | 必填 |
| LOGICIPV4 | `10.8.10.1`<br>`192.168.10.63` | 全网规划/本端规划 | IPVERSION=IPv4 时填；不能是 0.0.0.0、255.255.255.255、环回 |
| LOGICIPV6 | `fe80:8029:5008:6B::247` | 全网规划 | IPv6 必须全球单播，不能为 ::、FFFF:FFFF:FFFF:FFFF |
| VPNINSTNAME | `vpn_gxif`<br>`vpn_gy`<br>`_public_` | 已配置数据中获取/全网规划 | 需先 [ADD VPNINST](command/UNC/20.15.2/ADD-VPNINST.md)；不填下发缺省 VPN |
| IPV4MTU | `1500`（默认） | 全网规划 | 328~9600 |
| IPV6MTU | `1500`（默认） | 全网规划 | 1280~9600 |
| DESC | `Restful_IPv6_Client`<br>`test` | 全网规划 | 0~255 字符，可选 |

**典型行（IPv4 + VPN，Gx 接口底层 IP）**

```
ADD LOGICIP: IPVERSION=IPv4, LOGICIPV4="10.8.10.1", VPNINSTNAME="vpn_gxif";
ADD LOGICINF: NAME="gxif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.10.1", IPV4MASK1="255.255.255.255", VPNINSTANCE="vpn_gxif";
```

**典型行（IPv6，QoS 能力开放 Restif）**

```
ADD LOGICIP: IPVERSION=IPv6, LOGICIPV6="fe80:8029:5008:6B::247", VPNINSTNAME="vpn_restif", DESC="Restful_IPv6_Client";
```

**典型行（_public_ VPN，SBI 加密）**

```
ADD LOGICIP: IPVERSION=IPv4, LOGICIPV4="10.16.1.5", VPNINSTNAME="_public_", DESC="test";
```

## 决策点

### IP 版本（IPv4 vs IPv6）

由接口组网决定。多数计费/策略接口（Gx/Gy/Ga/Gi/S6b）走 IPv4；QoS 能力开放的 Restif 接口走 IPv6（证据 WSFD-109101）。

### VPN 实例填什么

| 接口 | 典型 VPN | 来源 |
|---|---|---|
| Gx | `vpn_gxif` | 已配置数据中获取（ADD VPNINST 先建） |
| Gy | `vpn_gy` / `vpn_gyif` | 同上 |
| Ga | `vpn_ga` | 同上 |
| Gi (AAA) | `vpn_aaa` / `vpn_pdn` / `vpn_enterprise` | 同上 |
| S6b | `vpn_s6bif` | 同上 |
| SBI | `_public_` | 全网规划 |
| Restif (QoS 开放) | `vpn_restif` | 全网规划 |

不填 VPNINSTNAME 时下发缺省 VPN。

### MTU 是否显式配置

默认 1500 满足绝大多数场景。仅在链路 MTU 受限时（如特定隧道/封装）显式改 IPV4MTU/IPV6MTU。

## 约束

- **立即生效** · severity=LOW。
- **最多 10240 条逻辑 IP** · severity=MEDIUM · 上限较高，一般不触顶。
- **只在 UNC 网元下生效** · severity=MEDIUM · UDG 不用本命令。
- **IP 地址合法性**：IPv4 不能是 0.0.0.0、255.255.255.255、环回（127.x）；IPv6 必须全球单播 · severity=HIGH。
- **MTU 范围**：IPV4MTU 328~9600，IPV6MTU 1280~9600 · severity=MEDIUM。
- **与 ADD LOGICINF 成对一致**：本命令的"IP+VPN"必须与 ADD LOGICINF 的"IPV4ADDRESS1+VPNINSTANCE"完全一致，否则接口绑定失败 · severity=HIGH。
- **前置依赖**：VPNINSTNAME 需先由 ADD VPNINST 创建（可用 [LST VPNINST](command/UNC/20.15.2/LST-VPNINST.md) 查询），否则下发失败 · severity=HIGH。
- **NF 适用**：SGW-C、PGW-C、SMF（证据标注）；SBI 加密场景见 WSFD-010308。
