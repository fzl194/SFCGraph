# 用户数据管理（WSFD-010400）

> 特性静态知识见 [WSFD-010400](feature/UNC/20.15.2/WSFD-010400.md)。本页讲配置生成实例化时这个特性**怎么配**——4 制式签约数据管理，**核心对象已由网元协议栈内置**，UNC 域**无独立配置命令**（仅 2 条 DSP 调试命令）。

## 配置概览

核心对象链：**签约数据**（UDR/UDSF 存储的 HLR/HSS/UDM 签约：IMSI/SUPI/MSISDN/PDN-PGW 关系/APN-OI-Replacement/ODB/3GPP-Charging-Characteristics/UE-AMBR/RFSP Index 等）→ **移动性上下文**（MM/EPS/PDU 上下文：MM State/GUTI/TAlist/PDN Address 等）。

**4 制式 × 3 套接口协议**：2G/3G（SGSN-HLR Gr/S6d 接口，MAP/Diameter） + 4G（MME-HSS S6a 接口，Diameter） + 5G（AMF/SMF-UDM Nudm 服务化接口，HTTP/JSON）。

**License 信息**：本特性**无需 License**（特性概述明示，4 制式一致）。

**★ 本特性为「能力型+配置型混合」特性**：
- **能力层**：4 制式签约数据管理由 AMF/SMF/MME/SGSN 网元协议栈内置（首次注册/TAU/RAU 自动触发获取流程 + UDM/HSS 推送通知自动接收）— **无 UNC 配置命令**
- **可配置项**：① `SET SDBTMR`（用户分离后签约数据保留时间 `PURGTMR`，归 2G/3G SGSN）② 签约数据存储方式 `SET SYS` 参数 `SUBSTORAG`（GPRS/EPS 公有字段分开存储控制）③ `SET NGMMPROCTRL` 等 NGMM 子开关 — **1~2 条 SET 命令**
- **DSP 调试**：`DSP COMMMCTX`（用户 MM 上下文详情） + `DSP COMUSRPDPNUM`（整系统 MM/PDN 用户数统计）— 仅作运维查询证据，**不进配置流程**

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（仅在签约数据驱动 PDN/PDU 会话建立时需要，纯签约查询时不需）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR` → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **（按需）签约数据保留时间设置**（2G/3G SGSN 必配，控制 PURGTMR）：`SET SDBTMR`
   > **★ R1.5 待补 atom + 命令名核实**：`SET SDBTMR` 命令名真实存在 UNC OM（路径：`业务服务管理/Pre 5G接入业务管理/控制面管理/用户数据管理/签约保留时间/`），evidence = [WSFD-010400 2G&3G 特性概述](evidence/UNC/20.15.2/WSFD-010400/WSFD-010400-特性概述_2G3G_85436067.md)；UNC pre-build 阶段 atom 未建，归 P1 后续补建。
   - 5G/4G 网元内部默认保留时间不同，UNC 域**通常不需要配置**；2G/3G SGSN **必配**（同上特性概述明示"可由维护人员设置 PURGTMR"）
3. **（按需）签约数据存储方式**（GPRS/EPS 共有字段分开存储，2G/3G+4G 融合部署时）：`SET SYS: SUBSTORAG=SEPARATE`
   > **★ R1.5 子规则·激活未演示 + 命令名待核**：2G/3G 特性概述仅说"GPRS/EPS 共有字段分开存储控制"，未明示命令名 `SET SYS`。evidence 待核 UNC OM 中签约数据存储方式真实命令名（可能为 `SET SGSNCOMMONCTRL` 或 `SET MMECOMMONCTRL` 等同义命令）。**配置生成对 2G/3G+4G 融合部署场景默认不产出**，待命令名核实后回填。UNC pre-build 阶段 atom 未建。
   - 仅在 SGSN/MME 同时部署（GPRS+EPS 融合）的场景下生效；纯 5G 不需此参数

> **★ 单命令步骤直接引 atom（§3.5）**：本特性每步均为单命令 SET，**不建 compound**——直接复用 atom 或标 R1.5 待补 atom。
>
> **★ 4 制式能力由协议栈内置**：5G AMF/SMF 的 Nudm_SubscriberDataManagement_Get/Subscribe/Notification 流程、4G MME 的 S6a Insert Subscriber Data/Update Location/Cancel Location/Notification 流程、2G/3G SGSN 的 Gr/S6d Insert Subscriber Data/Update Location 流程，**均由网元协议栈自动处理**，UNC 无显式配置命令触发。

## 约束（特性级 rule）

- **本特性无需 License**（info）：适用 NF 为 AMF/SMF/MME/SGSN。
- **2G/3G SGSN PURGTMR 必设**（critical）：2G/3G SGSN 默认保留时间不可满足所有场景，需维护人员 `SET SDBTMR PURGTMR=...` — 后果：未设致用户分离后数据留存过久占用资源 / 过短致频繁拉签约数据增加 HLR 信令负荷。
- **签约数据存储方式变更生效时机**（critical）：`SET SYS SUBSTORAG` 配置**仅在用户发生移动性管理流程时生效**（[WSFD-010400 2G&3G 特性概述](evidence/UNC/20.15.2/WSFD-010400/WSFD-010400-特性概述_2G3G_85436067.md) 明示）— 后果：配置后未触发移动性管理流程的存量用户仍按旧存储方式生效。
- **AMF/SMF 接入控制检查**（warning）：AMF/SMF 接收到 UDM 推送的签约特性后做接入控制检查，不支持的特性**由 UDM 本地存储**并据此决定是否允许用户接入本 AMF/SMF — 归 [WSFD-010400 5G 特性概述](evidence/UNC/20.15.2/WSFD-010400/WSFD-010400-特性概述_5G_60374923.md) 「用户数据管理功能」段。
- **4G MME Notification 仅适用 MME**（info）：Notification 流程（MME 通知 HSS）**仅适用于 MME，不适用于 SGSN** — 归 [WSFD-010400 4G 实现原理](evidence/UNC/20.15.2/WSFD-010400/WSFD-010400-实现原理_4G_70018152.md)「Notification 流程」段。
- **Super-Charger 特性**（info，2G/3G）：HLR/HSS 与 SGSN 同时支持 Super-Charger 功能时，HLR/HSS 漫游时**不**通知原 SGSN 删除用户数据 — 后果：原 SGSN 持续保存数据（详见 2G&3G 特性概述原理段）。
- **★ 待补 atom**（critical）：`SET SDBTMR`（2G/3G PURGTMR） / `SET SYS`（SUBSTORAG 存储方式）UNC pre-build 阶段**未建 atom**，**待补**。
- **首次发布版本**（info）：2G/3G 首次发布 UNC **20.3.0**；4G/5G 首次发布 UNC **20.0.0**。
