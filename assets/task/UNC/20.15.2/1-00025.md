# UNC Radius 对接链

> **UNC 域鉴权族共享底座**（SOP §3.5 域通用）。被引用于 5 个 WSFD 鉴权族特性。
>
> 设计参考：本骨架的 **APNRDSSVRGRP/APNRDSCLIENTIP/RDSSVR/RDSSVRGRP** 4 条是 UDG 域没有的（UDG 域 Radius 由 UPF 触发），UNC 是**控制面发起** RADIUS 鉴权；**UPFRDSSVR/UPFRDSCLIENTIP/RDSUPFCTRL/UPLIST4RDS** 4 条是 UPF 中转模式独有（WSFD-108007 终端二次鉴权使用）。UDG 域无对应，UNC 域独立建。

## 配置方法

本骨架建 Radius 鉴权/计费链路的接入域基础设施——AAA 服务器组 + APN/UPF 绑定。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. AAA 服务器组 | `ADD RDSSVRGRP` → [0-00255](task/UNC/20.15.2/0-00255.md) | RDSSVRGRPNAME, MODE=MASTER_SLAVE/LOAD_BALANCE, ACCTTIMEOUT/AUTHTIMEOUT, ACCTRETRANSMIT/AUTHRETRANSMIT |
| 2. 鉴权 AAA 服务器 | `ADD RDSSVR` → [0-00254](task/UNC/20.15.2/0-00254.md) | RDSSVRGRPNAME, SERVERTYPE=AUTHENTICATION, SERVERIPV4, VPNINSTANCE, PORT=1812, CIPHERKEY, PRIFLAG=PRIMARY/SECONDARY/CARBON_COPY, PRIORITY |
| 3. 计费 AAA 服务器 | `ADD RDSSVR` → [0-00254](task/UNC/20.15.2/0-00254.md) | RDSSVRGRPNAME, SERVERTYPE=ACCOUNTING, PORT=1813, 其他同鉴权 |
| 4. APN 绑 Client IP（鉴权/计费各 1 条）| `ADD APNRDSCLIENTIP` → [0-00221](task/UNC/20.15.2/0-00221.md) | APN, AUTHORACCT=AUTHENTICATION/ACCOUNTING, INTFNAME（Gi 接口名）|
| 5. APN 绑 AAA 服务器组 | `ADD APNRDSSVRGRP` → [0-00222](task/UNC/20.15.2/0-00222.md) | APN, RDSSVRGRPNAME |
| 6.（WSFD-108007 专用·UPF 中转链）UP 列表 | `ADD UPLIST4RDS` → [0-00260](task/UNC/20.15.2/0-00260.md) | UPLISTNAME, UPINSTANCEID |
| 7.（WSFD-108007 专用·UPF 中转链）UP 列表 UPF 属性 | `ADD RDSUPFCTRL` → [0-00256](task/UNC/20.15.2/0-00256.md) | UPLISTNAME, UPINSTANCEID, PREFERENCE, LOCKED |
| 8.（WSFD-108007 专用·UPF 中转链）UPF 绑 Radius 服务器 | `ADD UPFRDSSVR` → [0-00262](task/UNC/20.15.2/0-00262.md) | SERVERTYPE, SERVERIPV4（DN-AAA IP）, UPLISTNAME |
| 9.（WSFD-108007 专用·UPF 中转链）UPF 绑 Radius 客户端 IP | `ADD UPFRDSCLIENTIP` → [0-00261](task/UNC/20.15.2/0-00261.md) | CLIENTYPE, VPNINSTANCE, CLIENTIPV4, UPLISTNAME（**最后一步**）|

**典型脚本（WSFD-011306 主链路，IPv4 + 带 VPN）**：
```
ADD RDSSVRGRP: RDSSVRGRPNAME="isprg", MODE=MASTER_SLAVE, SIGOPTACCTMSG=DISABLE, ACCTTIMEOUT=10, ACCTRETRANSMIT=5, AUTHTIMEOUT=10, AUTHRETRANSMIT=5, ACCTSVRTOUT=12, ACCTSVRDTIME=180, AUTHSVRTOUT=12, AUTHSVRDTIME=180;
ADD RDSSVR: RDSSVRGRPNAME="isprg", SERVERTYPE=AUTHENTICATION, IPVERSION=IPV4, SERVERIPV4="10.168.10.1", PORT=1812, VPNINSTANCE="vpn1", CIPHERKEY="*****", PRIFLAG=PRIMARY, PRIORITY=1, CIPHERKEYCNFM="*****";
ADD RDSSVR: RDSSVRGRPNAME="isprg", SERVERTYPE=ACCOUNTING, IPVERSION=IPV4, SERVERIPV4="10.168.10.1", PORT=1813, VPNINSTANCE="vpn1", CIPHERKEY="*****", PRIFLAG=PRIMARY, PRIORITY=1, CIPHERKEYCNFM="*****";
ADD APNRDSCLIENTIP: APN="apn-test", AUTHORACCT=AUTHENTICATION, INTFNAME="giif1/0/1";
ADD APNRDSCLIENTIP: APN="apn-test", AUTHORACCT=ACCOUNTING, INTFNAME="giif1/0/1";
ADD APNRDSSVRGRP: APN="apn-test", RDSSVRGRPNAME="isprg";
```

**典型脚本（WSFD-108007 UPF 中转链）**：
```
ADD CPGTPUADDR: IPVERSION=IPV4, IPV4ADDR="192.168.0.20";  // N4 GTP-U 地址
ADD UPLIST4RDS: UPLISTNAME="uplist1", UPINSTANCEID="up1";
ADD RDSUPFCTRL: UPLISTNAME="uplist1", UPINSTANCEID="up1", PREFERENCE=1, LOCKED=ENABLE;
ADD UPFRDSSVR: SERVERTYPE=AUTHENTICATION, IPVERSION=IPV4, SERVERIPV4="192.168.10.2", UPLISTNAME="uplist1";
ADD UPFRDSCLIENTIP: CLIENTYPE=AUTHENTICATION, IPVERSION=IPV4, UPLISTNAME="uplist1", VPNINSTANCE="vpnInstance1", CLIENTIPV4="192.168.10.1";  // 最后一步
```

**步骤位置**：在 WSFD-011305（仅本骨架步骤 1 部分·4 接入模式配置）/[UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施（必先建）之后；WSFD-108007 需先建 CP GTP-U 地址。

## 场景差异（**SOP §3.5 必填**：逐 feature 双向回填）

| feature | 用到的步骤 | 关键差异 |
|---|---|---|
| **WSFD-011305** Radius 鉴权接入 | 间接引用（前置依赖 WSFD-011306 走完 1-00025 步骤 1-5）| 1 条 `SET APNAUTHATTR`（[0-00271](task/UNC/20.15.2/0-00271.md)），属特性级配置，本骨架未覆盖；**WSFD-011305 本身不直接配 RDSSVR 服务器**——通过 `SET APNAUTHATTR` 配 ACCESSMODE + 必先激活 [WSFD-011306](task/UNC/20.15.2/2-00025.md) |
| **WSFD-011306** Radius 功能 | 步骤 1-5（全 5 条标准链路 RDSSVRGRP+RDSSVR+APNRDSCLIENTIP×2+APNRDSSVRGRP）| **不执行 UPF 中转链（步骤 6-9）**——WSFD-011306 不涉及 108007 场景；必加 `SET RDSACCTREQVSA`（3GPP VSA）+ `SET APNRDSACCTCTRL`（计费阈值）→ [UNC 1-00026](task/UNC/20.15.2/1-00026.md) |
| **WSFD-011307** Radius 抄送 | 步骤 1+3+5（精简版，仅 RDSSVRGRP + 计费 RDSSVR + APNRDSSVRGRP）| **不执行步骤 4（APNRDSCLIENTIP×2）**——WSFD-011307 激活不配 APN Client IP（参考信息未列）；鉴权可选；抄送靠 `PRIFLAG=CARBON_COPY` 标识同 RDSSVR 命令；额外走 [UNC 1-00026](task/UNC/20.15.2/1-00026.md) `SET APNRDSACCTCTRL` + `SET APNRADIUSATTR` |
| **WSFD-010301** 鉴权功能 | 不用本骨架 | 5G `ADD NGUSRSECPARA` / 2G `MOD GBAUTHCIPH` / 4G `MOD S1USRSECPARA`（atom 待补）/ 3G `MOD IUAUTHCIPH`（atom 待补）— 属控制面 NAS 信令级鉴权，与 AAA 对接无关 |
| **WSFD-108007** 终端二次鉴权 | 仅步骤 6-9（UPF 中转链，4 条 UPLIST4RDS+RDSUPFCTRL+UPFRDSSVR+UPFRDSCLIENTIP）+ N4 GTP-U 前置（CPGTPUADDR）| **不走标准主链路步骤 1-5**（RDSSVRGRP+RDSSVR+APNRDSCLIENTIP+APNRDSSVRGRP 不在 108007 激活档）；UPF 中转专属：依赖 `ADD CPGTPUADDR`（[0-00226](task/UNC/20.15.2/0-00226.md)）建 N4 GTP-U 隧道；**本特性依赖 [WSFD-011305](feature/UNC/20.15.2/WSFD-011305.md) + [WSFD-011306](feature/UNC/20.15.2/WSFD-011306.md) 已激活** |

## 决策点（UNC pre-build 内嵌，不编号）

| 选项（场景）| 影响（走哪些步骤）|
|---|---|
| 标准 Radius 鉴权链路（WSFD-011306）| 步骤 1-5 全 5 条 |
| 鉴权+计费+抄送（WSFD-011307 抄送场景）| 步骤 1+3+5 + 1 条 `PRIFLAG=CARBON_COPY` RDSSVR + [UNC 1-00026](task/UNC/20.15.2/1-00026.md) SET APNRDSACCTCTRL + SET APNRADIUSATTR |
| UPF 中转鉴权（WSFD-108007）| 步骤 6-9 + 必加 `ADD CPGTPUADDR`（[0-00226](task/UNC/20.15.2/0-00226.md)）|
| 接入模式（WSFD-011305 4 制式）| 不直接走本骨架；本特性是 `SET APNAUTHATTR` 一步配置 ACCESSMODE（4 取值 TRANS_AUTH/TRANS_NON_AUTH/NON_TRANS/LOC_AUTH）+ 可选公用用户名/密码 |
| NAS 信令级鉴权（WSFD-010301 2G/3G/4G/5G）| 不走本骨架（属控制面 NAS 信令，不经 RADIUS 服务器）|

> 联动特性级场景 DP：走哪些步骤由各 feature 场景 DP 决定。

## 约束

- **APNRDSCLIENTIP 必配**（critical）：`ADD APN` 之后**必须**配 `ADD APNRDSCLIENTIP`（AUTHENTICATION+ACCOUNTING 各 1 条），否则 APN 用户激活失败 — 约束归 [WSFD-011306](feature/UNC/20.15.2/WSFD-011306.md) 主链路。
- **鉴权+计费 IP/密钥可不同**（info）：同一 IP+VPN 的 Server 在不同组里密钥不限制；鉴权+计费 AAA 服务器密钥可不同。
- **UPFRDSCLIENTIP 必须最后执行**（critical）：该命令执行后 SMF 立即触发 N4-U 会话建立，需所有其他 UPF 中转配置完成后才执行 — 中途修改需先 `RMV UPFRDSCLIENTIP` 再重 `ADD`，否则 SMF 尝试建链失败（延迟 15 分钟重试）。
- **UPFRDSSVR → UPFRDSCLIENTIP 顺序**（critical）：必须先 `ADD UPFRDSSVR` 再 `ADD UPFRDSCLIENTIP`，否则 AAA 计费业务无法正常进行。
- **UPLIST4RDS 命名约束**（warning）：UPLISTNAME 建议统一用小写（WSFD-108007 激活文档明示）。
- **每个 Radius Server Group 最多 16 主+16 备**（warning）：鉴权/计费服务器主用+备用总和上限 16+16；超出新增失败。
- **整机最大支持 4000 个 AAA Server**（info）：WSFD-011306 特性规格。
- **本骨架不含 3GPP 私有扩展 / 计费控制 / 域名剥离 / PoD** 等 RADIUS 私有扩展配置 — 归 [UNC 1-00026](task/UNC/20.15.2/1-00026.md)。
