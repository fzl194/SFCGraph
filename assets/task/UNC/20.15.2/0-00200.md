# SET SGWCHARGECFG 原子任务
> 命令静态知识见 [SET SGWCHARGECFG](command/UNC/20.15.2/SET-SGWCHARGECFG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令为 SGW-C 全局单例（最大记录数 1），按"用户归属属性（本地/漫游/拜访）"分别使能离线计费，并决定 SGW CG IP 用本地配置还是 PGW 携带的 CG。仅适用于 UNC 用作 S-GW。

## 配置方法

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| HOMEOFFLINE | 本地用户离线计费开关（本端规划） | `DISABLE`（PGW-C 计费时 SGW-C 不使能） |
| ROAMOFFLINE | 漫游用户离线计费开关（本端规划） | `ENABLE`（运营商间结算） |
| VISITOFFLINE | 拜访用户离线计费开关（本端规划） | `ENABLE`（运营商间结算） |
| SGWCGFLAG | SGW CG IP 选择来源（本端规划） | `USE_LOCAL`（用本地 CG）/ 使用 PGW 携带 CG |

典型脚本（本地用户不产生 SGW 话单，漫游/拜访用户产生，CG 用本地）：
```
SET SGWCHARGECFG:HOMEOFFLINE=DISABLE,ROAMOFFLINE=ENABLE,VISITOFFLINE=ENABLE,SGWCGFLAG=USE_LOCAL;
```

## 决策点

### 各归属属性用户是否在 SGW-C 产生离线话单
- 问什么：本地/漫游/拜访三类用户各自的离线计费开关。
- 常规部署：本地用户由 PGW-C 计费，SGW-C 关闭（HOMEOFFLINE=DISABLE）；漫游/拜访用户为运营商间结算，SGW-C 使能（ROAMOFFLINE/VISITOFFLINE=ENABLE）。
- 一致性约束：针对同一用户，不同属性下的计费方式必须一致，否则计费出错。

### SGW CG IP 来源
- 问什么：SGW-C 发送话单的 CG 地址用本地配置还是 PGW 携带。
- USE_LOCAL → 使用本地配置的 CG（ADD CGGROUP/ADD CGBINDING）。
- 使用 PGW 携带 CG → 沿用 PGW 侧 CG。
- 高危：选错或无可用 CG，SGW 话单无法正常处理，用户无法计费。

## 约束
- 生效时机：执行后只对新激活用户生效。`severity=medium`
- 高危：配置不当导致无可用 CG，SGW 话单无法处理，用户无法计费。`severity=high`
- 一致性：同一用户不同属性下的计费方式必须一致，否则计费出错（如某用户既配离线又配在线）。`severity=high`
- NF 限制：仅适用于 UNC 用作 S-GW。`severity=medium`
- 单例：最大记录数 1，全局唯一。`severity=low`
- 系统初始值：HOMEOFFLINE=DISABLE, ROAMOFFLINE=ENABLE, VISITOFFLINE=ENABLE, SGWCGFLAG=USE_LOCAL。`severity=low`
- 优先级：被 APN 级 SET SGWAPNCHGMETH（OFFLINE）覆盖。`severity=medium`
