# MOD PDUSCACT 原子任务
> 命令静态知识见 [MOD PDUSCACT](command/UNC/20.15.2/MOD-PDUSCACT.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用网元：PGW-C、SMF。本命令修改融合计费模板（CCT）下"PDU 异常返回码动作"条目，定义当收到特定异常返回码时的处理动作。被融合计费场景（WSFD-011206）用于"配置计费消息缓存"——开启缓存的前提是 SCACT 配置为 FAILOVER 或 CONTINUE。前置需先 [ADD PDUSCACT](task/UNC/20.15.2/0-00061.md) 建条目。

## 配置方法
按 CCTMPLTNAME + CODETYPE（+ STATUSCODE）定位条目，修改处理动作及相关条件参数。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTMPLTNAME | 融合计费模板名（必选，1~63） | `cct-test` |
| CODETYPE | 返回码类型（必选） | `VALUE`、`DEFAULT`（针对未指定的异常返回码） |
| STATUSCODE | 状态码（conditional，300~999 或 65535） | `502` |
| SCACT | 处理动作（必选） | `FAILOVER` / `CONTINUE` 等 |
| RDVIRTIP | 重定向 IPv4 地址（conditional） | 规划重定向 IP |
| RDVIRTIPV6 | 重定向 IPv6 地址（conditional） | 规划重定向 IPv6 |
| BLKTIMER | 阻塞处理时间间隔（conditional，0~1440 min） | `60` |
| GTPV01CAUSE | 原因值 GtpV0-1（conditional，0~255） | 按规划 |
| GTPV2CAUSE | 原因值 GtpV2（conditional，0~255） | 按规划 |
| REACTREQ | 重新激活请求（conditional） | `DISABLE` |

## 决策点

### 处理动作 SCACT 选择（FAILOVER / CONTINUE / 重定向 / 阻塞）
SCACT 是条目的核心动作开关。典型取值 FAILOVER（进行 Failover 处理，优先按 CCT 规则切换）或 CONTINUE（继续处理）。开启计费消息缓存的前提是 SCACT 必须为 FAILOVER 或 CONTINUE。具体选哪种动作由融合计费可靠性策略（缓存/重定向/阻塞）决定，存在跨场景分叉。

### CODETYPE：精准匹配 vs 兜底
CODETYPE=VALUE 表示按 STATUSCODE 精准匹配某一具体返回码；CODETYPE=DEFAULT 表示针对未指定的异常返回码统一兜底。两者可并存（一个模板下既配精准又配兜底），如何分配由返回码规划决定。

### 条件参数联动（随 SCACT 触发）
重定向类动作需配 RDVIRTIP/RDVIRTIPV6；阻塞类动作需配 BLKTIMER；GTP 原因值类需配 GTPV01CAUSE/GTPV2CAUSE；重激活类需配 REACTREQ。这些 conditional 参数是否必填取决于 SCACT 取值。

## 约束
- **立即生效**：执行后立即生效。
- **前置存在性**：只能修改已 ADD 成功的 PDUSCACT 条目；条目不存在则报错。可用 LST PDUSCACT 校验。
- **CCT 模板前置**：CCTMPLTNAME 必须是已存在的融合计费模板。
- **计费消息缓存前提**：开启缓存要求 SCACT ∈ {FAILOVER, CONTINUE}。
- **取值范围**：STATUSCODE 300~999 或 65535；BLKTIMER 0~1440 min；GTPV01CAUSE/GTPV2CAUSE 0~255。
