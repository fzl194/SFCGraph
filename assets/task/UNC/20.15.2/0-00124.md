# 修改PDU异常返回码动作（MOD PDUSCACT）

> 命令静态知识见 [MOD PDUSCACT](command/UNC/20.15.2/MOD-PDUSCACT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按 CCTMPLTNAME + CODETYPE（+ STATUSCODE）定位条目，修改处理动作及相关条件参数。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 定位 | CCTMPLTNAME | 融合计费模板名（1~63，不区分大小写） | 无 | 指定目标 CCT | 所有场景必选 |
| 定位 | CODETYPE | `VALUE`（精准匹配）/ `DEFAULT`（兜底） | 无 | 指定返回码匹配方式 | 按规划分叉 |
| 定位 | STATUSCODE | 300~999 或 65535（条件必选，CODETYPE=VALUE 时） | 无 | 精准匹配特定返回码 | 按规划分叉 |
| 动作 | SCACT | `FAILOVER` / `CONTINUE` / `TERM_WITH_REL` / `TERM_NO_REL` / `BLOCK` / `REDIRECT` | 无 | 核心处理动作开关 | 按可靠性策略分叉 |
| 条件-重定向 | RDVIRTIP | IPv4 地址（SCACT=REDIRECT 时条件可选） | 0.0.0.0 | 重定向目标 IPv4 | SCACT=REDIRECT |
| 条件-重定向 | RDVIRTIPV6 | IPv6 地址（SCACT=REDIRECT 时条件可选） | :: | 重定向目标 IPv6 | SCACT=REDIRECT |
| 条件-阻塞 | BLKTIMER | 0~1440 分钟（SCACT=BLOCK/REDIRECT 时条件可选） | 0 | 阻塞/重定向持续时长 | SCACT=BLOCK/REDIRECT |
| 条件-去活 | GTPV01CAUSE | 0~255（SCACT=TERM_WITH_REL/TERM_NO_REL 时条件可选） | 0 | GTPv0/v1 原因值 | SCACT=TERM_WITH_REL/TERM_NO_REL |
| 条件-去活 | GTPV2CAUSE | 0~255（SCACT=TERM_WITH_REL/TERM_NO_REL 时条件可选） | 0 | GTPv2 原因值 | SCACT=TERM_WITH_REL/TERM_NO_REL |
| 条件-去活 | REACTREQ | `DISABLE` / `ENABLE`（SCACT=TERM_WITH_REL/TERM_NO_REL 时条件可选） | DISABLE | 是否通知用户重新激活 | SCACT=TERM_WITH_REL/TERM_NO_REL |

典型脚本（502 返回码 FAILOVER）：

```
MOD PDUSCACT: CCTMPLTNAME="cct-test", CODETYPE=VALUE, STATUSCODE=502, SCACT=FAILOVER;
```

步骤位置：配置计费消息缓存流程步骤 2（前置：ADD PDUSCACT 建条目）。

## 决策点

### 决策点：处理动作 SCACT 选择（DP 0-01280）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| FAILOVER | — | 进行 Failover 处理，优先按 CHF 下发的 sessionFailover 与 failureHandling 信元处理；CHF 无下发则按 SET FAILHANDLING 动作处理 |
| CONTINUE | — | 允许业务继续进行，不再进行配额管理；计费消息缓存前提条件之一 |
| TERM_WITH_REL | GTPV01CAUSE / GTPV2CAUSE / REACTREQ | 去活 PDU 会话，发送 Charging Data Request Release 消息 |
| TERM_NO_REL | GTPV01CAUSE / GTPV2CAUSE / REACTREQ | 去活 PDU 会话，不发送 Release 消息 |
| BLOCK | BLKTIMER | 阻塞业务，使业务不能继续进行 |
| REDIRECT | RDVIRTIP / RDVIRTIPV6 / BLKTIMER | 将当前业务重定向到指定地址 |

计费消息缓存开启的前提：SCACT ∈ {FAILOVER, CONTINUE}。

### 决策点：返回码匹配方式 CODETYPE（DP 0-01281）

| 选项 | 适用场景 |
|---|---|
| VALUE | 按 STATUSCODE 精准匹配某一具体返回码（如 502） |
| DEFAULT | 针对未指定的异常返回码统一兜底处理 |

两者可并存（同一模板下既配精准又配兜底），如何分配由返回码规划决定。

## 约束

- **立即生效**（critical，rule-0-01652）：立即生效：执行后立即生效，不追溯已激活用户

- **前置存在性**（critical，rule-0-01653）：前置存在性：只能修改已 ADD 成功的 PDUSCACT 条目，条目不存在则报错；可用 LST PDUSCACT 校验

- **CCT 模板前置**（critical，rule-0-01654）：CCT 模板前置：CCTMPLTNAME 必须是已存在的融合计费模板

- **计费消息缓存前提**（warning，rule-0-01655）：计费消息缓存前提：开启缓存要求 SCACT ∈ {FAILOVER, CONTINUE}

- **取值范围**（warning，rule-0-01656）：取值范围：STATUSCODE 300~999 或 65535；BLKTIMER 0~1440 min；GTPV01CAUSE/GTPV2CAUSE 0~255
