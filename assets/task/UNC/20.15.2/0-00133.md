# MOD VPNINSTAF 原子任务
> 命令静态知识见 [MOD VPNINSTAF](command/UNC/20.15.2/MOD-VPNINSTAF.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

本命令属于高危命令（操作不当会删除 VPN 实例地址族下的配置及 BGP 相关配置）。用于修改设备上 VPN 实例下配置的地址族参数，例如修改 RD、应用路由策略等。是 L3VPN 地址族建好后调整 RD 的主要入口。

## 配置方法
按 VRFNAME + AFTYPE 定位地址族，按场景修改 RD 与路由/隧道策略参数。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| VRFNAME | VPN 实例名称（必选，已配置数据中获取，1~31 字符） | `vpn1`、`vpn_ga`、`vpn_gy` |
| AFTYPE | 地址族类型（必选，已配置数据中获取，枚举） | `ipv4uni`、`ipv6uni` |
| VRFRD | 路由标识（可选，全网规划/本端规划，3~21 字符） | `65512:2`、`1050:1`、`2050:1` |
| IMPOLICYNAME | 引入路由策略名称（可选，1~200 字符） | `impolicy1` |
| EXPOLICYNAME | 发布路由策略名称（可选，1~200 字符） | `expolicy1` |
| EXPLYADDERTFIRST | 路由上送时首先添加 ERT（可选，TRUE/FALSE） | `TRUE` |
| LOCALCROSSNHPMOD | 可更改本地交叉下一跳（可选，TRUE/FALSE） | `FALSE` |
| VRFLABELMODE | 实例标签分配模式（可选，枚举） | 按产品定义 |
| VRFLABEL | 实例标签值（条件，16~32767） | `1000` |
| TNLPOLICYNAME | 隧道策略名称（可选，1~39 字符） | `tnlpolicy1` |
| FRRENABLE | FRR 使能（可选，TRUE/FALSE） | `TRUE` |

典型用法：ADD L3VPNINST 建实例 → ADD VPNINSTAF 建地址族 → MOD VPNINSTAF 配 RD；IPv6 前缀代理（WSFD-104004）配 `ipv6uni` RD，对接 CG（WSFD-011201）/OCS（WSFD-109001）配 `ipv4uni` RD。

## 决策点

### VRFRD 取值来源多值（全网规划/本端规划/已配置获取）
VRFRD 在不同场景取值来源不同：IPv6 前缀代理为本端规划（`65512:2`），对接 CG 为全网规划（`1050:1`），对接 OCS 为已配置数据中获取（`2050:1`）。取值口径由 VPN 组网归属（全局统一规划 vs 网元本端规划）决定，数据规划阶段按对接系统选定。

### AFTYPE 跨场景枚举（ipv4uni / ipv6uni）
地址族类型按业务地址族选定：IPv4 业务用 `ipv4uni`，IPv6 前缀代理用 `ipv6uni`。与 ADD VPNINSTAF 创建时的地址族保持一致。

## 约束
- **立即生效**：命令执行后立即生效。
- **高危：删除 RD 连锁影响**：删除 RD 会导致此 VPN 实例地址族下的配置和此 VPN 实例在 BGP 中的相关配置被删除，谨慎操作并联系华为技术支持。
- **地址族存在性前置**：配置前需对应 VPN 实例地址族已创建（先 ADD VPNINSTAF），可用 LST VPNINSTAF 校验。
- **保留实例不可改**：不能修改 VPN 实例 `__mpp_vpn_inner__`、`__mpp_vpn_inner_server__` 的配置。
- **VRFRD 格式**：3~21 字符，RD 唯一标识一个 VPN。
