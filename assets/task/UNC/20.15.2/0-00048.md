# ADD NGSGWPLMN 原子任务
> 命令静态知识见 [ADD NGSGWPLMN](command/UNC/20.15.2/ADD-NGSGWPLMN.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令为**可选**命令，仅当 PGW-C 与 SGW-C 部署在同一台设备、且期望两者对同一用户按**不同**漫游属性计费时才配置；若期望两者漫游属性一致，则**不配置**本命令，SGW-C 直接复用 `ADD NGPLMN` 的 Home PLMN 判定。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| NOID | 运营商标识，当前版本固定取值 `0` | `0` |
| MCC | 移动国家码，全网规划，3 位数字 | `123` |
| MNC | 移动网号，全网规划，2~3 位数字 | `68` |
| DESC | 描述信息，本端规划，可选，0~32 字符 | `sgw homeplmn1` |

典型脚本：
```
ADD NGSGWPLMN: NOID=0, MCC="123", MNC="68", DESC="sgw homeplmn1";
```

匹配规则：仅对 IMSI 已与 `ADD NGPLMN` 匹配判定为**本地用户**的用户生效；SGW-C 用户 IMSI 的 PLMN 未匹配到本命令配置时判定为漫游用户。

## 决策点（若有）

### 是否配置 SGW-C 独立 Home PLMN
- 问什么：SGW-C 与 PGW-C 合设时，两者对同一用户的漫游属性判定是否需要区分。
| 场景 | 是否配置 | 后果 |
|---|---|---|
| 期望漫游属性一致 | 不配置 | SGW-C 复用 `ADD NGPLMN` 判定，与 PGW-C 一致 |
| 期望漫游属性不同 | 配置本命令 | SGW-C 按本命令独立判定，可对同一用户产生不同于 PGW-C 的漫游计费属性 |

## 约束（若有）
- MNC 长度互斥：若 MCC 相同，MNC 有效长度为 2 位与 3 位的记录，前两位不允许相同。`severity=high`（违反将导致漫游属性判定歧义）
- 联动依赖：本命令仅对与 `ADD NGPLMN` 匹配为本地用户的用户生效，须先配置 `ADD NGPLMN`。`severity=high`
- 规格上限：最多 512 条记录；NOID 当前固定取值 0。`severity=medium`
- 计费一致性：同一用户不同属性下规划的计费方式必须一致，否则计费出错。`severity=high`
- 生效时机：执行后立即生效。`severity=low`
