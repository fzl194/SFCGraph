# 配置SGW-C Home PLMN（ADD NGSGWPLMN）

> 命令静态知识见 [ADD NGSGWPLMN](command/UNC/20.15.2/ADD-NGSGWPLMN.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD NGSGWPLMN 为 SGW-C 单独配置 Home PLMN。本命令为**可选**命令，仅当 PGW-C 与 SGW-C 部署在同一台设备、且期望两者对同一用户按**不同**漫游属性计费时才配置；若期望两者漫游属性一致，则**不配置**本命令，SGW-C 直接复用 `ADD NGPLMN` 的 Home PLMN 判定。配置空间有 2 个维度：**运营商标识**（必选，当前版本固定）、**PLMN 编码**（必选，MCC+MNC）。

### 配置维度 1：运营商标识（参数 NOID，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 运营商标识 | `NOID` | 整数，当前版本固定取值 `0` | 无 | 在 UNC 上唯一标识运营商 | 当前版本仅支持 0 |

### 配置维度 2：PLMN 编码（参数 MCC + MNC，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 移动国家码 | `MCC` | 字符串，3 位数字（0-9） | 无 | PLMN 的国家码 | 全网规划 |
| 移动网号 | `MNC` | 字符串，2~3 位数字（0-9） | 无 | PLMN 的网络码 | 全网规划；同 MCC 下 2 位与 3 位记录前两位不允许相同 |
| 描述 | `DESC` | 字符串 0~32 字符 | 无 | 助记描述 | 可选；输入单空格删除已有配置 |

> 匹配规则：仅对 IMSI 已与 `ADD NGPLMN` 匹配判定为**本地用户**的用户生效；SGW-C 用户 IMSI 的 PLMN 未匹配到本命令配置时判定为漫游用户。

**典型脚本（为标识 0 的运营商新增 SGW-C Home PLMN 12368）**
```
ADD NGSGWPLMN: NOID=0, MCC="123", MNC="68", DESC="sgw homeplmn1";
```

**步骤位置**：在所属 feature 编排中通常位于"配置 SGW-C Home PLMN"步骤——网元基础数据准备阶段，紧随 `ADD NGPLMN`（Home PLMN 基准）之后，与计费场景配置并行。

## 决策点

### 决策点：是否配置 SGW-C 独立 Home PLMN（DP 0-02075）

问：SGW-C 与 PGW-C 合设时，两者对同一用户的漫游属性判定是否需要区分？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不配置（复用 ADD NGPLMN） | 省略本命令 | 期望漫游属性一致，SGW-C 复用 `ADD NGPLMN` 判定 |
| 配置独立 Home PLMN | `NOID=0, MCC, MNC`（可选 `DESC`） | 期望漫游属性不同，SGW-C 按本命令独立判定，对同一用户产生不同于 PGW-C 的漫游计费属性 |

> 本命令仅在与 `ADD NGPLMN` 匹配为本地用户的用户上生效；配置前须先建 `ADD NGPLMN`。

### 决策点：PLMN 编码规划与 MNC 长度互斥（DP 0-02076）

问：同 MCC 下 MNC 有效长度为 2 位与 3 位的记录如何共存？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单一 MNC 长度 | `MNC` 统一 2 位或统一 3 位 | 无互斥风险 |
| 混合 MNC 长度（受约束） | 同 MCC 下 2 位与 3 位记录前两位不允许相同 | 需差异化区分运营商，但须避免前缀冲突 |

> 违反 MNC 长度互斥约束将导致漫游属性判定歧义；同 MCC 下 2 位与 3 位记录前两位若相同，系统无法区分。

## 约束

- rule-0-02186 · severity=critical · 若 MCC 相同，MNC 有效长度为 2 位与 3 位的记录，前两位不允许相同 — 后果：漫游属性判定歧义
- rule-0-02187 · severity=high · 本命令仅对与 ADD NGPLMN 匹配为本地用户的用户生效，须先配置 ADD NGPLMN — 后果：未建 NGPLMN 时本地用户判定链断裂
- rule-0-02188 · severity=high · 同一用户不同属性下规划的计费方式必须一致，否则计费出错 — 后果：计费错误
- rule-0-02189 · severity=warning · 最多可输入 512 条记录；NOID 当前版本固定取值 0 — 后果：超规格新增失败
- rule-0-02190 · severity=info · 该命令执行后立即生效；适用 NF：SGW-C — 后果：无延迟
