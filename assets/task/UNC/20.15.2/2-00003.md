# 融合计费（WSFD-011206）

> 特性静态知识见 [WSFD-011206](feature/UNC/20.15.2/WSFD-011206.md)。本页讲配置生成实例化时这个特性**怎么配**——编排哪些步骤、CCT 模板 4 层次如何选、CHF 对接/费率标识/异常处理/缓存如何配。
>
> **特性关系**：本特性是计费族第 3 批，与离线（[2-00001](task/UNC/20.15.2/2-00001.md)）/热（[2-00002](task/UNC/20.15.2/2-00002.md)）的差异是**命令集不同**——融合用 CCT 模板（`ADD CCT`/`MOD CCT`）+ CHF 对接（TNF 族）+ Nchf 交互（Trigger/故障处理/缓存），非 OFCTEMPLATE 参数变种。仅 UserProfile/APN 绑定**结构**可复用 1-00002/03（命令集同，差异回填）。

## 配置概览

核心对象链：**CCT 模板**（融合计费模板：配额阈值+Trigger容器+终结方式）→ 绑定到 `{UserProfile | APN | 计费属性CC | 全局}` 之一；并配套 **CHF 选择**（TNF 族 + 选择方式）、**费率标识链**（URR→URRGROUP→PCCPOLICYGRP→RULE→RULEBINDING）、**SMF-CHF 交互**（Trigger+RG老化）、**异常处理**（故障切换）、**消息缓存**（可靠性）。

融合计费配置分两条主线：
1. **使能与接口选择**（前置）：融合开关 + Nchf 计费模式 + N40 接口版本 + CHF 选择
2. **计费参数**（核心）：CCT 模板 → 绑定层次 + 费率标识链 + 交互 Trigger + 异常处理 + 缓存（可选）

CCT 模板 4 种配置层次**共享同一套 CCT 参数骨架**（[1-00005](task/UNC/20.15.2/1-00005.md)），差异仅在"绑到哪个对象"——由特性级 DP（配置层次选择）驱动走哪个绑定 compound。

## 配置流程（步骤编排，每步显示含命令）

1. **使能融合计费开关**（总开关，前置）：`SET CHARGECTRL` → [0-00157](task/UNC/20.15.2/0-00157.md)
   - 关键参数：HOMECONVERGED=ENABLE, VISITCONVERGED=ENABLE, ROAMCONVERGED=ENABLE
   - 粒度使能开关（按 DP 选一，与总开关配合——**注意：此为使能开关命令，非模板绑定**）：UserProfile 级 `SET USRPROFCHARGE`(CONVERGEDSW/RGAPPLIED) · APN 级 `SET APNCHARGECTRL`(CONVERGEDSW/RGAPPLIED) · CC 级 `ADD CHARGEMETHOD`(CONVERGED/RGAPPLIED) → [0-00015](task/UNC/20.15.2/0-00015.md)
   > **★ DP 分列规则（R1.5）**：CC 粒度涉及**两个不同命令**——① **使能开关**走 `ADD CHARGEMETHOD`（本步骤，[0-00015](task/UNC/20.15.2/0-00015.md)，参数 CONVERGED/RGAPPLIED）；② **CCT 模板绑定**走 `ADD SELECTCCTBYCC`（步骤 6，[1-00006](task/UNC/20.15.2/1-00006.md)/[0-00077](task/UNC/20.15.2/0-00077.md)，参数 CCVALUE→CCTMPLTNAME）。两者不可混写为"走法=ADD CHARGEMETHOD + ADD SELECTCCTBYCC"——前者是 CC 是否使能融合（开关），后者是 CC 选哪个 CCT 模板（绑定）。命令 wiki 佐证：[ADD CHARGEMETHOD](command/UNC/20.15.2/ADD-CHARGEMETHOD.md)（CONVERGED/RGAPPLIED/QBCSW 开关参数组）vs [ADD SELECTCCTBYCC](command/UNC/20.15.2/ADD-SELECTCCTBYCC.md)（CCVALUE/CCTMPLTNAME 绑定参数组），参数集正交。
2. **配计费和策略接口模式**（选 Nchf 模式，前置）：`SET CHGMODE` + `SET POLICYMODE` → [0-00162](task/UNC/20.15.2/0-00162.md) · [0-00197](task/UNC/20.15.2/0-00197.md)
   - 关键参数：SET CHGMODE FORCED=NchfMode；SET POLICYMODE FORCED=Npcf
   - 可选粒度（按 DP 选）：APN 级 `ADD APNCHGMODE` + `ADD APNPOLICYMODE` · 漫游级 `ADD ROAMCHGMODE` · 基于 PCF 实例 `ADD PCCCHGMODEBYPCFID` → [0-00005](task/UNC/20.15.2/0-00005.md) · [0-00007](task/UNC/20.15.2/0-00007.md) · [0-00070](task/UNC/20.15.2/0-00070.md) · [0-00050](task/UNC/20.15.2/0-00050.md)
3. **配 N40 接口 API 版本**（Nchf 协议版本，前置）：`SET N40APIVER` → [0-00183](task/UNC/20.15.2/0-00183.md)
   - 关键参数：APIVER（如 F30）；FEATURE（位域，如 NODEFUNC-1）
4. **配置 CHF 选择**（建 CHF 实例/组 + 选 CHF 方式）：`ADD TNFINS` + `ADD TNFINSIP` + `ADD TNFGRP` + `ADD TNFBINDGRP` + 选择命令 → [1-00007](task/UNC/20.15.2/1-00007.md)
   - 选择方式按 DP 选（IMSI/号段/CC/全局默认/NRF）
5. **配置 CCT 模板参数**（配额/阈值/终结方式，通用骨架）：`ADD CCT` 或 `MOD CCT` → [1-00005](task/UNC/20.15.2/1-00005.md)
6. **绑到配置层次**（按 DP 选，四选一）：
   - UserProfile 层（用户级）：`ADD USERPROFILE` + `SET USRPROFCHARGE` + `ADD USRPROFGROUP` + `ADD UPBINDUPG` + `ADD APNUSRPROFG` → [1-00002](task/UNC/20.15.2/1-00002.md)
     - 融合差异：SET USRPROFCHARGE 用 CCTEMPLATE（非 PGWOFCTEMPLATE）+ CONVERGEDSW=ENABLE + RGAPPLIED
   - APN 层（APN 级）：`ADD APN` + `SET APNCHARGECTRL` → [1-00003](task/UNC/20.15.2/1-00003.md)
     - 融合差异：SET APNCHARGECTRL 用 CCTEMPLATE + CONVERGEDSW=ENABLE + RGAPPLIED
   - 计费属性 CC 层：`ADD CHARGECHAR`（建 CC）+ `ADD SELECTCCTBYCC`（CC 绑 CCT 模板，**非使能开关**） → [1-00006](task/UNC/20.15.2/1-00006.md)
     - 注意：CC 粒度的**融合使能开关**（CONVERGED/RGAPPLIED）已在步骤 1 经 `ADD CHARGEMETHOD` 配置；本步骤只做 CCT 模板绑定
   - 全局层（兜底，单命令）：`MOD CCT`（CCTMPLTNAME=global）→ [0-00114](task/UNC/20.15.2/0-00114.md)
7. **配置费率标识链**（内容计费 URR→URRGROUP→PCCPOLICYGRP→RULE→RULEBINDING→UserProfile）：`ADD URR` + `ADD URRGROUP` + `ADD PCCPOLICYGRP` + `ADD RULE` + `ADD RULEBINDING` + `SET URRGRPBINDING` → [1-00009](task/UNC/20.15.2/1-00009.md)
   - 融合特点：URRGROUP 建议同时配在线+离线 URR（RGAPPLIED=ONLINERGONLY/OFFLINERGONLY 时）
8. **配置 SMF-CHF 交互 Trigger 与 RG 老化**：`SET CHFINIT` + `ADD PDUTRIGGER` + `ADD RGTRIGGER` + `SET RGRESCTRL` → [1-00008](task/UNC/20.15.2/1-00008.md)
9. **配置异常处理**（故障切换，可选但推荐）：`SET FAILHANDLING` + `ADD PDUSCACT` + `ADD RGRCACT` + `SET CNVRGDCHGPARA` → [1-00010](task/UNC/20.15.2/1-00010.md)
10. **配置计费消息缓存**（可靠性，可选）：`SET N40MSGSTG` + `SET STGTRIGGER` + `SET CNVRGDCHGPARA` + `SET GLBDFTCHFGROUP` + `SET CDRSTORAGECTRL` + `SET STGALARMCTRL` + `SET N4CHGMSGCTRL` → [1-00011](task/UNC/20.15.2/1-00011.md)
    - 前提：步骤 9 FHACTION=CONTINUE + SCACT=FAILOVER/CONTINUE

> **可选增强**：费率切换（节假日/星期/费率时段）——`ADD FESTIVAL` + `ADD WEEKDAY` + `ADD TARIFFGROUP` + `SET N40QUOTACTRL` + `SET GLBTARIFFGROUP` → [0-00031](task/UNC/20.15.2/0-00031.md) · [0-00100](task/UNC/20.15.2/0-00100.md) · [0-00083](task/UNC/20.15.2/0-00083.md) · [0-00185](task/UNC/20.15.2/0-00185.md) · [0-00177](task/UNC/20.15.2/0-00177.md)。优先级：UserProfile > APN > CC > 整机。

## 决策点（特性级）

### DP1：CCT 模板配置层次选择

| 配置层次 | 走法 | 优先级 | 关键联动 | activation 演示 |
|---|---|---|---|---|
| UserProfile | 骨架 [1-00005](task/UNC/20.15.2/1-00005.md) + 绑定 [1-00002](task/UNC/20.15.2/1-00002.md) | 最高 | SET USRPROFCHARGE 带 CCTEMPLATE+CONVERGEDSW+RGAPPLIED | ✓ 演示（任务四） |
| APN | 骨架 1-00005 + 绑定 [1-00003](task/UNC/20.15.2/1-00003.md) | 次高 | SET APNCHARGECTRL 带 CCTEMPLATE+CONVERGEDSW+RGAPPLIED | ✓ 演示（任务三） |
| 计费属性 CC | 骨架 1-00005 + 使能开关 [0-00015](task/UNC/20.15.2/0-00015.md)(ADD CHARGEMETHOD) + 绑定 [1-00006](task/UNC/20.15.2/1-00006.md)(ADD SELECTCCTBYCC) | 次低 | **分两步**：① ADD CHARGEMETHOD 配 CONVERGED/RGAPPLIED（使能）② ADD SELECTCCTBYCC 绑 CCVALUE→CCTMPLTNAME（模板绑定） | ✓ 演示（任务二，CCT 绑定部分） |
| 全局 | 骨架 1-00005（MOD CCT，单命令 [0-00114](task/UNC/20.15.2/0-00114.md)） | 最低 | 兜底；所有未配层次的最终取值 | ✓ 演示（任务一/全局实例） |

> **走法列**：UserProfile/APN/CC 三层走多命令绑定 compound；全局层单命令直接 atom（§3.5）。4 层次是**降级链**而非并列。**全部 4 层 activation 均有演示**（配置融合计费模板_93400212 的任务一~四 + 使用全局CCT模板实例）。

### DP2：CHF 选择方式（联动 [1-00007](task/UNC/20.15.2/1-00007.md)）

| CHF 选择方式 | 走法 | 优先级 | 适用场景 | activation 演示 |
|---|---|---|---|---|
| 基于 IMSI | ADD SELCHFGBYIMSI | 最高 | 拨测 | ✓ 任务一 |
| 基于 IMSI 号段 | ADD SELCHFGBIMSISEG | 次高 | 多 CHF 负载均衡 | ✓ 任务二 |
| PCF 下发 FQDN | ADD TNFINS 带 FQDN | 中 | PCF 携带 chargingInfo | ✓ 任务三 |
| 本地 CC 选 CHF | ADD SELECTCHFGBYCC | 低 | 本地 CC 绑 CHF 组 | ✓ 任务四 |
| 全局默认 CHF 组 | SET GLBDFTCHFGROUP | 最低 | 兜底；缓存回放前提 | ✓ 任务六 |

### DP3：融合计费方式（在线/离线）

| 方式 | 影响 | 配置差异 |
|---|---|---|
| 在线计费 | SMF 向 CHF 申请配额，配额不足可停业务 | URR USAGERPTMODE=ONLINE；CHFINIT=SENDREQ；配额管理信元 |
| 离线计费 | SMF 不申请配额，仅上报用量 | URR USAGERPTMODE=OFFLINE；Create Request 不含 requestedUnit |

> 由 RGAPPLIED（DEFAULT/ONLINERGONLY/OFFLINERGONLY）+ URR 的 USAGERPTMODE 联动决定。

## 约束（特性级 rule）

- **总开关前置**（critical）：`SET CHARGECTRL` 须先使能融合计费（HOMECONVERGED/VISITCONVERGED/ROAMCONVERGED=ENABLE）— 否则粒度开关不生效。
- **CHGMODE 选 Nchf**（critical）：`SET CHGMODE` FORCED=NchfMode（或 APN/漫游级）— 否则不走融合计费接口。
- **同一 RG 在线/离线属性唯一**（critical）：同一计费会话内，同一 RG 要么在线要么离线，不能同时 — 否则计费出错（特性 wiki 应用限制）。
- **不支持多 UPF 共享在线 RG 配额**（warning）：多 UPF 场景在线计费 RG 配额不共享（特性 wiki 应用限制）。
- **模板名是引用键**（critical）：CCTMPLTNAME 须先建（[1-00005](task/UNC/20.15.2/1-00005.md)）再绑定 — 否则绑定命令找不到模板。
- **4 层次降级链**（warning）：UserProfile > APN > CC > 全局，高一级未配取下一级。
- **CHF 交互使能**（warning）：`SET CHFINIT` CHFINIT=SENDREQ 时 PDU 会话建立即与 CHF 交互；在线计费建议开启。
- **缓存前提**（critical）：开启消息缓存（[1-00011](task/UNC/20.15.2/1-00011.md)）必须 FHACTION=CONTINUE + SCACT=FAILOVER/CONTINUE + STGSWITCH=ENABLE + CHGDATAREFGEN=SMF + 配全局默认 CHF 组。
- **RGAPPLIED 与 URRGROUP 配合**（warning）：DEFAULT 不能同时配在线+离线 URR；ONLINERGONLY/OFFLINERGONLY 建议同时配。
- **URRID/RULENAME 跨网元一致**（critical）：SMF 与 UPF 的 URRID/USAGERPTMODE/METERINGTYPE/RULENAME 须一致 — 否则计费统计不一致。
- **N40APIVER 与 CHF 一致**（warning）：APIVER 须与 CHF 使用相同协议版本 — 否则接口信元不一致（ALM-81026）。
