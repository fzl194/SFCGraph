# 配置 ADC 预定义规则与用户模板绑定

> ADC 预定义规则路径的规则承载层——本地建 ADC 规则（POLICYTYPE=ADC，无 POLICYNAME）+ 用户模板（UPTYPE=PCC）+ 绑定关系，供 PCRF/PCF 下发预定义规则 APP_STA/APP_STO（APPLICATION_START/STOP）Event-Trigger 时本地 Rule 名匹配。被引用于 [2-00006 ADC基本功能](task/UNC/20.15.2/2-00006.md)。ADC 域专属（POLICYTYPE=ADC 是 ADC 独有变种，RULE/USERPROFILE/RULEBINDING 命令跨 PCC 族共享但 ADC 场景参数集不同，见场景差异）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 配本地 ADC 规则 | `ADD RULE` | RULENAME（三网元一致）；POLICYTYPE=ADC（固定）；无 POLICYNAME |
| 配用户模板 | `ADD USERPROFILE` | USERPROFILENAME（全网规划）；UPTYPE=PCC（固定） |
| 绑规则到用户模板 | `ADD RULEBINDING` | USERPROFILENAME（引用上一步）；RULENAME（引用第一步） |

**典型脚本（PCRF/PCF 预定义规则下发 Event-Trigger）**：
```
ADD RULE: RULENAME="rule_test", POLICYTYPE=ADC;
ADD USERPROFILE: USERPROFILENAME="up_test", UPTYPE=PCC;
ADD RULEBINDING: USERPROFILENAME="up_test", RULENAME="rule_test";
```

**步骤位置**：在 License 前置（[SET LICENSESWITCH](task/UNC/20.15.2/0-00180.md) 开 LKV2BADCF01）之后；本 compound 完成本地规则承载，之后由 PCRF/PCF 下发 APP_STA/APP_STO（APPLICATION_START/STOP）Event-Trigger，SMF 按本地 RULENAME 匹配后透传 UPF 执行检测。

## 场景差异

- **ADC 规则无 POLICYNAME（与计费/PCC QoS 规则不同）**：POLICYTYPE=ADC 的规则不绑 PCCPOLICYGRP/QOSPROP（ADC 仅做应用检测上报，策略由 PCRF/PCF 动态下发），故无 POLICYNAME 参数。对比：内容计费 POLICYTYPE=PCC 绑 PCCPOLICYGRPNM，QoS 保证 POLICYTYPE=QOS 绑 QOSPROPNAME，ULCL POLICYTYPE=ULCL 绑 ULCL 属性 + FLOWFILTERNAME。
- **UPTYPE=PCC（与计费族共享）**：ADC 的 USERPROFILE 复用 PCC 类型模板（UPTYPE=PCC），与 PCC 基本功能/内容计费一致。USERPROFILE 是跨特性共享容器，ADC 通过 RULEBINDING 挂 ADC 规则。
- **三网元 RULENAME 一致（预定义规则场景）**：PCRF/PCF 下发预定义规则名时，GGSN-C/PGW-C/SMF、PGW-U/UPF、PCRF/PCF 上的规则标识必须一致。对比动态规则路径（[1-00015](task/UNC/20.15.2/1-00015.md)）三网元一致的是 FLOWFILTERNAME。
- **Gx（4G/5G 融合）vs N7（纯 5G）**：本 compound 命令集相同，仅 Event-Trigger 命名不同——Gx 为 APPLICATION_START/APPLICATION_STOP，N7 为 APP_STA/APP_STO。差异在接口层，不影响本 compound 配置。
- **可选步骤说明**：activation 将本路径标记为"可选"（PCRF/PCF 不通过动态规则下发 appid 时执行）。但预定义规则路径下本 compound 的三命令是必选组合（RULE+USERPROFILE+RULEBINDING 缺一不可），非内部可选。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| POLICYTYPE=ADC | ADC 规则，无 POLICYNAME；仅做应用检测上报，策略由 PCRF/PCF 动态下发 |
| UPTYPE=PCC | PCC 类型用户模板（ADC 共享 PCC 模板体系） |

> 联动特性级 DP：预定义规则路径选择由 [2-00006](task/UNC/20.15.2/2-00006.md) DP1（规则下发方式=预定义规则）驱动；本 compound 仅在 DP1=预定义规则时编排。

## 约束

- **三网元 RULENAME 一致**（critical）：预定义规则场景下，GGSN-C/PGW-C/SMF、PGW-U/UPF、PCRF/PCF 上的 RULENAME 必须一致 — 否则 PCRF/PCF 下发的预定义规则名无法本地匹配。
- **RULENAME+POLICYTYPE 唯一**（high）：多条记录时 RULENAME+POLICYTYPE 组合不能完全相同。`severity=high`
- **License 前置**（critical）：SET LICENSESWITCH 激活 LKV2BADCF01 须先于本 compound — 否则 ADC 功能不生效。
- **依赖 PCC 基本功能**（critical）：须已完成 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md) 激活 — 否则 ADC 规则无 PCC 通道承载。
- **USERPROFILE 须先建再绑**（high）：RULEBINDING 的 USERPROFILENAME 与 RULENAME 均须已配置存在 — 否则引用失败。
- **PCC 规则删除后停止 STOP 上报**（warning）：若携带 Application ID 的 PCC 规则被删除或失效，后续业务停止后不再向 PCRF/PCF 上报 APPLICATION_STOP 事件 — 影响带宽控制恢复链路。
- **生效时机**（low）：执行后立即生效。
