# 复位PCC定时器（RST PCCTIMER）（RST PCCTIMER）

> 命令静态知识见 [RST PCCTIMER](command/UNC/20.15.2/RST-PCCTIMER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"要复位的定时器类型"单参数配置，无需 APN/用户绑定。当前证据支持的 TIMERTYPE 取值为 `SUPFEA_NEGO_TMR`（动态协商定时器），复位后清除 UNC 本地缓存的所有 PCRF 实际 feature 支持能力。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 定时器类型 | TIMERTYPE | 枚举（SUPFEA_NEGO_TMR），本端规划 | 无 | 指定要复位的 PCC 定时器类型，复位后清除对应缓存 | 必选 |

典型脚本（PCRF 升级后清除本地能力缓存，强制新会话重新协商）：
```
RST PCCTIMER: TIMERTYPE=SUPFEA_NEGO_TMR;
```

步骤位置：作为 [SET PCCTIMER](command/UNC/20.15.2/SET-PCCTIMER.md) 定时器机制的运维补充 —— SET 配置本地缓存的定时器时长（到期自动清除对端能力），RST 用于 PCRF 升级、对端能力变更等需要立即清缓存的运维场景，应在确认本地配置 PCRF feature 能力与 PCRF 实际能力一致后执行。

## 决策点

### 决策点：定时器类型选择（DP 0-03031）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| SUPFEA_NEGO_TMR（动态协商定时器） | TIMERTYPE=SUPFEA_NEGO_TMR | supported-features 动态协商使能时，PCRF 升级/对端能力变更后需立即清除本地缓存的所有 PCRF feature 支持能力 |

### 决策点：复位时机判断（DP 0-03032）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| PCRF 升级/对端能力变更 | 先对齐本地 PCRF feature 能力配置再 RST | 运维场景需立即清缓存，新接入用户发起全新协商 |
| 等待定时器自然到期 | 不执行 RST，由 SET PCCTIMER 配置的时长到期后自动清除 | 非紧急场景，避免触发二次协商信令开销 |

## 约束

- **立即生效**（warning，rule-0-03076）：立即生效：执行后立即清除本地缓存的对端能力，新接入用户发起全新 Supported-Features 协商流程；在线用户不受影响（保留已学到的能力）

- **二次协商代价**（warning，rule-0-03077）：二次协商代价：清缓存后首个会话触发二次协商（Gx 信令增加），若本地配置的 PCRF feature 能力与 PCRF 实际能力不一致，建议先对齐配置再复位，减少协商失败

- **TIMERTYPE 取值范围**（warning，rule-0-03078）：TIMERTYPE 取值范围：当前证据仅暴露 SUPFEA_NEGO_TMR 一个枚举值，选错类型会复位不相关的定时器无法达成清缓存目的，取值需对照产品枚举定义

- **在线用户不重协商**（warning，rule-0-03079）：在线用户不重协商：复位只影响后续新接入用户，已激活在线用户保留已学到的 PCRF 能力，不会重新发起协商——若需对在线用户生效须先去活再重接入

- **与 SET PCCTIMER 的关系**（info，rule-0-03080）：与 SET PCCTIMER 的关系：定时器时长配置用 SET PCCTIMER，本命令仅做立即复位，不修改时长；两者配合构成 PCC 定时器完整管理
