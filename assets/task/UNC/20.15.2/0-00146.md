# RST PCCTIMER 原子任务
> 命令静态知识见 [RST PCCTIMER](command/UNC/20.15.2/RST-PCCTIMER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

该命令是对 [SET PCCTIMER](command/UNC/20.15.2/SET-PCCTIMER.md) 定时器机制的补充：SET 配置本地缓存的定时器时长（到期自动清除对端能力），RST 用于 PCRF 升级、对端能力变更等需要立即清缓存的运维场景。适用 NF：SGW-C、PGW-C、SMF。

## 配置方法
按"要复位的定时器类型"单参数配置，无需 APN/用户绑定。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| TIMERTYPE | 选择要复位的 PCC 定时器类型（枚举） | 缓存对端 PCRF 能力的定时器类型 |

典型脚本（PCRF 升级后清除本地能力缓存）：
```
RST PCCTIMER: TIMERTYPE=<对应枚举>;
```

## 决策点

### 定时器类型选择（TIMERTYPE）
- 问什么：当前要复位哪一类 PCC 定时器。
- 证据中 UNC 通过该命令学习并存储对端 PCRF 实际能力（首次会话协商结果），相关场景聚焦于"清除对端 PCRF 能力缓存"这一类定时器。
- 选错类型会复位不相关的定时器，无法达成清缓存目的；TIMERTYPE 取值需对照产品枚举定义。

## 约束
- 立即生效：执行后立即清除本地缓存的对端能力，新接入用户发起全新协商流程；在线用户不受影响（保留已学到的能力）。`severity=medium`
- 二次协商代价：清缓存后首个会话会触发二次协商（Gx 信令增加），若本地配置的 PCRF feature 能力与 PCRF 实际能力不一致，建议先对齐再复位，减少协商失败。`severity=medium`
- 与 SET PCCTIMER 的关系：定时器时长配置用 SET PCCTIMER；本命令仅做立即复位，不修改时长。`severity=low`
