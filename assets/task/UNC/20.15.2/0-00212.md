# 修改用户模板组绑定（MOD UPBINDUPG）

> 命令静态知识见 [MOD UPBINDUPG](command/UNC/20.15.2/MOD-UPBINDUPG.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

MOD UPBINDUPG 是 PGW-C/SMF 上"用户模板组绑定"的修改入口，对应 [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) 的增量修改版本：用于**已存在**的 UsrProfGroup↔UserProfile 绑定关系，调整 PRIORITY 或追加精匹配条件（IMSI/MSISDN 号段、IMEISV 号段、RAT、Roaming、ChargeChar、位置组等）。UPBINDTYPE 仍取 SPECIFIC（精匹配）/ DEFAULT（默认）/ ULCL（分流专用），SPECIFIC 类型时上述精匹配参数至少要设置一个。命令立即生效。

### 配置维度 1：定位主键（参数 USERPROFGNAME/UPBINDTYPE）

| 参数 | 取值 | 作用 | 适用场景 |
|---|---|---|---|
| `USERPROFGNAME` | 字符串 1~63 字符，不支持空格，不区分大小写 | 必选；指定已建 UsrProfGroup（[ADD USRPROFGROUP](task/UNC/20.15.2/0-00096.md) 产物） | 通用 |
| `UPBINDTYPE` | `SPECIFIC` / `DEFAULT` / `ULCL` | 必选；绑定类型，决定其它参数生效范围 | 通用 |

> USERPROFGNAME + UPBINDTYPE 是定位主键：定位到 UsrProfGroup 下某一类型的某条绑定，对其增量修改。

### 配置维度 2：用户模板定位（参数 USERPROFILENAME）

| 参数 | 取值 | 作用 | 适用场景 |
|---|---|---|---|
| `USERPROFILENAME` | 字符串 1~63 字符，不支持空格，不区分大小写 | UPBINDTYPE=SPECIFIC/DEFAULT 时必选；指定已建 UserProfile（[ADD USERPROFILE](task/UNC/20.15.2/0-00094.md) 产物） | 通用 |

### 配置维度 3：精匹配条件（参数集合，仅 UPBINDTYPE=SPECIFIC 时生效）

| 参数 | 取值 | 作用 | 适用场景 |
|---|---|---|---|
| `PRIORITY` | 整数 1~65534（越小越高；65535 保留给 DEFAULT） | UPBINDTYPE=SPECIFIC 时必选；UserProfile 在组内匹配优先级 | 多 SPECIFIC 模板场景 |
| `RAT` | NONE/UTRAN/GERAN/WLAN/GAN/HSPA/EUTRAN/NR/NBIOT/EMTC/REDCAP | 可选；按无线接入技术精匹配 | 异构接入场景 |
| `ROAMING` | NONE/HOME/ROAMING/VISITING | 可选；按漫游属性精匹配 | HOME/ROAM/VISIT 差异化策略 |
| `CCCONFIGMODE` | NORMAL/HOTBILLING/PREPAID/FLATBILLING/SPECIFIC | 可选；按计费属性配置模式精匹配 | 计费模式区分 |
| `CHARGECHAR` | 1~6 字符十六进制，0x0~0xffff | CCCONFIGMODE=SPECIFIC 时条件必选；计费属性值 | 计费属性精匹配 |
| `CCMASK` | 1~6 字符十六进制，0x1~0xFFFF（默认 0xFFFF） | 可选；计费属性掩码 | 计费属性精匹配 |
| `IMSIMSSEGNAME` | 字符串 1~31 字符，不支持空格，不区分大小写 | 可选；引用 IMSI/MSISDN 号段（[ADD IMSIMSISDNSEG](task/UNC/20.15.2/0-00195.md) 产物） | 按号段区分 |
| `IMEISVSEGNAME` | 字符串 1~31 字符，不支持空格，不区分大小写 | 可选；引用 IMEISV 号段（[ADD IMEISVSEG] 产物） | 按终端类型区分 |
| `LOCGROUPNAME` | 字符串 1~63 字符，不支持空格，不区分大小写 | 可选；引用位置组（[ADD USRLOCATIONGRP](task/UNC/20.15.2/0-00211.md) 产物） | 按位置区域区分 |
| `SEGGROUPNAME` | 字符串 1~31 字符，不支持空格，不区分大小写 | 可选；引用号段组（ADD SUBSCRIBERIDSEGGRP 产物） | 按号段组区分 |

> UPBINDTYPE=SPECIFIC 时，IMSIMSSEGNAME / IMEISVSEGNAME / RAT / ROAMING / CHARGECHAR / LOCGROUPNAME **至少要设置其中任意一个**（PRIORITY 是必选但不算"匹配条件"，仅是优先级）。

### 配置维度 4：本地 PCC 策略选择（参数 LOCALPCCSELECT）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| `INHERIT`（默认） | 继承 [SET PCCFUNC](task/UNC/20.15.2/0-00193.md) 全局 LOCALPCCSELECT | 通用 |
| `LOCAL_PCC_DEACTIVE` | 本地 PCC 策略不激活 | 关闭本地 PCC |
| `LOCAL_PCC_ACTIVE` | 本地 PCC 策略激活 | 强制本地 PCC |
| `LOCAL_PCC_NO_CRBN_ACT` | 未下发计费规则组时本地 PCC 策略激活 | 部分本地 PCC 场景 |
| `LOCAL_PCC_NO_RULE_ACT` | 未下发任意计费规则时本地 PCC 策略激活 | 兜底本地 PCC |

### 配置维度 5：与 ADD UPBINDUPG 的分叉选择（参数联动维度，非本命令字段）

| 场景 | 选择命令 | 原因 |
|---|---|---|
| UsrProfGroup 与 UserProfile 已有绑定，需要改优先级/追加匹配条件 | 本命令（MOD UPBINDUPG） | 增量修改已存在的绑定 |
| UsrProfGroup 与 UserProfile 尚未绑定，需要一次性建绑定（含位置组） | [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) | 一次性绑定模板与匹配条件 |

> 步骤位置：位置策略控制链的第三步（[1-00019 配置用户位置与模板绑定](task/UNC/20.15.2/1-00019.md)），在 [ADD USRLOCATION](task/UNC/20.15.2/0-00210.md) + [ADD USRLOCATIONGRP](task/UNC/20.15.2/0-00211.md) 之后；已绑定模板组的场景用本命令追加 LOCGROUPNAME 等精匹配条件，未绑定则用 [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) 一次性绑定。

## 决策点

### 决策点：绑定类型选择（DP 0-04033）

问：当前需要修改的是哪种绑定类型？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| `SPECIFIC` | `PRIORITY` 必选 + 至少 1 个精匹配条件 | 多模板精匹配（如按位置/RAT/漫游/计费区分） |
| `DEFAULT` | 仅 `USERPROFILENAME` | 默认兜底模板调整 |
| `ULCL` | 仅 `USERPROFILENAME` | ULCL 分流策略专用模板调整 |

> UPBINDTYPE 决定可修改的参数集合：SPECIFIC 可改全部精匹配条件；DEFAULT/ULCL 仅定位 USERPROFILENAME。

### 决策点：MOD vs ADD（DP 0-04034）

问：UsrProfGroup 与目标 UserProfile 之间是否已存在绑定？

| 选项 | 联动命令 | 适用场景 |
|---|---|---|
| 已存在绑定 | 本命令（MOD UPBINDUPG）增量修改 | 已通过 [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) 建过绑定，需追加位置组/调整 PRIORITY |
| 不存在绑定 | [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) 一次建绑定 | 首建绑定（含模板 + 位置组 + 优先级 + 匹配条件） |

> 选错会导致命令执行失败或产生重复绑定；优先按 `LST UPBINDUPG` 查询当前绑定关系再选命令。

## 约束

- **立即生效**（info，rule-0-04081）：命令执行后配置立即生效，无需重启或用户重激活 — 后果：在线生效，对在线用户的下一次匹配立即可见
- **SPECIFIC 至少 1 个精匹配条件**（critical，rule-0-04082）：UPBINDTYPE=SPECIFIC 时，IMSIMSSEGNAME/IMEISVSEGNAME/RAT/ROAMING/CHARGECHAR/LOCGROUPNAME 至少要设置其中任意一个 — 后果：无精匹配条件命令执行失败，需补充至少 1 项
- **前置绑定必须存在**（critical，rule-0-04083）：本命令定位主键 USERPROFGNAME+UPBINDTYPE(+USERPROFILENAME) 对应的绑定关系必须已通过 [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) 建好，否则命令失败 — 后果：未建绑定执行报错，需先 ADD 后 MOD
- **名称字符规则**（warning，rule-0-04084）：USERPROFGNAME/USERPROFILENAME 字符串 1~63 字符，不支持空格，不区分大小写；IMSIMSSEGNAME/IMEISVSEGNAME/SEGGROUPNAME 字符串 1~31 字符，同规则 — 后果：含空格或超长直接报错；引用对象（USRLOCATIONGRP/USERPROFILE/USRPROFGROUP/IMSIMSISDNSEG/IMEISVSEG/SUBSCRIBERIDSEGGRP）按同一名称匹配
- **PRIORITY 唯一性**（warning，rule-0-04085）：UPBINDTYPE=SPECIFIC 时 PRIORITY 同组内唯一，值越小优先级越高；65535 保留给 UPBINDTYPE=DEFAULT — 后果：同组内 PRIORITY 冲突会导致匹配顺序混乱或修改失败
