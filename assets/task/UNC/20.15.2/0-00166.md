# SET CONCENPOINT 原子任务
> 命令静态知识见 [SET CONCENPOINT](command/UNC/20.15.2/SET-CONCENPOINT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

PGW-C/SMF 侧**接口级**集中点模式设置：Gx（PCRF）、Gy（OCS）、Ga（CG）、S6b（3GPP AAA）四类 Diameter 接口各一个独立参数，按接口分别配置。模式决定后续 [ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md) 建链时本端端口号是否可配。

## 配置方法
按对接接口（Gx/Gy/Ga/S6b）分别选模式，一个接口配一行；多接口可一条命令同时下。

| 接口 | 参数 | 配置方法 | 取值样例 |
|---|---|---|---|
| Gx（对接 PCRF） | GXCONCENMODE | 本端规划 | `LOCALPORT`（本端端口可配）/ `LOCALIP_PEER`（不可配，系统自动分配） |
| Gy（对接 OCS） | GYCONCENMODE | 本端规划 | `LOCALPORT` |
| Ga（对接 CG） | GACONCENMODE | 本端规划 | `SINGLE_CONNECT`（单连接）/ `MULTI_PORT`（多端口，需配 GAPORTPERPROC） |
| S6b（对接 3GPP AAA） | S6BCONCENMODE | 本端规划 | `LOCALPORT` |
| Ga 每进程端口数 | GAPORTPERPROC | 本端规划，仅 GACONCENMODE=MULTI_PORT 时配 | `1`~`5` |

典型脚本（Gx 接口对接 PCRF，本端端口可配）：
```
SET CONCENPOINT: GXCONCENMODE=LOCALPORT;
```
Ga 接口对接 CG（链路数多，配多端口，每进程 3 个端口）：
```
SET CONCENPOINT: GACONCENMODE=MULTI_PORT, GAPORTPERPROC=3;
```
S6b 接口对接 3GPP AAA Server：
```
SET CONCENPOINT: S6BCONCENMODE=LOCALPORT;
```

## 决策点

### 集中点模式选择（LOCALPORT vs LOCALIP_PEER）
- 问什么：本端 Diameter 端口号是否需要显式配置。
- `LOCALPORT`：本端端口可配，建链时用 [ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md) 设置 LocalPort；缺省（LOCALIP_PEER/系统自动分配）时端口不可配。
- 取值由本端网络规划决定：是否需要对端 CG/OCS/PCRF 指定固定本端端口、是否走 DRA（路由代理）等。

### Ga 链路数策略（SINGLE_CONNECT vs MULTI_PORT）
- 问什么：Ga 接口（对接 CG）需要多少条链路。
- 对端 CG 需要足够多 Ga 链路 → `MULTI_PORT` + 配 GAPORTPERPROC（1~5）调整每进程端口数；否则 → `SINGLE_CONNECT`。

## 约束
- 唯一性：最大记录数 1（全网一份）。`severity=medium`
- 生效时机：立即生效。`severity=low`
- 高危：修改集中点模式可能导致 Diameter 链路重建或中断（Gx/Gy/S6b），影响用户业务（如被去活）。`severity=high`
- LOCALIP_PEER 模式下，[ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md) 添加的链路仅 LocalPort 未配或为 0 的生效；非 0 的允许下发但不建链。`severity=medium`
- LOCALPORT 模式下，仅 LocalPort 非 0 的链路生效。`severity=medium`
- GAPORTPERPROC 为 conditional，仅 GACONCENMODE=MULTI_PORT 时必填。`severity=low`
