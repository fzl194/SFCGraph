# 配置信令集中点部署模式（SET CONCENPOINT）

> 命令静态知识见 [SET CONCENPOINT](command/UNC/20.15.2/SET-CONCENPOINT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CONCENPOINT 是 PGW-C/SMF 的"接口级集中点模式层"：对 Gx（对接 PCRF）、Gy（对接 OCS）、Ga（对接 CG）、S6b（对接 3GPP AAA）四类 Diameter 接口分别设置部署模式，模式决定后续 [ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md) 建链时本端端口号是否可配及 Ga 链路数策略。单命令覆盖四接口的集中点模式 + Ga 每进程端口数。本命令的核心配置空间有 3 个维度：**Gx/Gy/S6b 集中点模式**（LOCALPORT vs LOCALIP_PEER）、**Ga 链路数策略**（SINGLE_CONNECT vs MULTI_PORT）、**Ga 每进程端口数**（conditional）。

### 配置维度 1：Gx/Gy/S6b 集中点模式（参数 GXCONCENMODE / GYCONCENMODE / S6BCONCENMODE）

| 取值 | 作用 | 下游 [ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md) 行为 |
|---|---|---|
| `LOCALPORT` | 按本端端口绑定 Diameter 集中点 | 仅 LocalPort 非 0 的链路生效 |
| `LOCALIP_PEER` | 按本端源 IP + 对端主机名绑定集中点 | 仅 LocalPort 未配或为 0 的链路生效，端口不可配 |

> 初始值默认 `LOCALIP_PEER`（系统自动分配端口）。

### 配置维度 2：Ga 链路数策略（参数 GACONCENMODE）

| 取值 | 作用 | 配套参数 |
|---|---|---|
| `SINGLE_CONNECT` | 一个对端应用服务器和一个逻辑端口只有一条链路（初始默认） | — |
| `MULTI_PORT` | 使用 group 级接口，和对端建立多条连接，多条连接使用组级 IP 作源 IP、源端口号不同 | `GAPORTPERPROC=<1~5>`（每进程端口数） |

### 配置维度 3：Ga 每进程端口数（参数 GAPORTPERPROC，conditional）

| 取值 | 作用 | 前提条件 |
|---|---|---|
| `1`~`5` | 指定 Ga 集中点模式为 MULTI_PORT 时每个 Ga 进程提供的端口数 | 仅 `GACONCENMODE=MULTI_PORT` 时必填 |

> **当前版本不支持 GAPORTPERPROC 参数超过 1**（命令 notes 明确）。

**典型脚本**：
```
SET CONCENPOINT:GXCONCENMODE=LOCALPORT;
SET CONCENPOINT:GYCONCENMODE=LOCALPORT;
SET CONCENPOINT:GACONCENMODE=MULTI_PORT,GAPORTPERPROC=1;
SET CONCENPOINT:S6BCONCENMODE=LOCALPORT;
```

**步骤位置**：在所属 feature 编排中通常位于"接口基础参数"段——**必先于** [ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md) 建链；模式切换需用 RMV SCTPASSOC 删除原模式的 SCTP 偶联测量对象并重新配置。

## 决策点

### 决策点：Gx/Gy/S6b 集中点模式选择（DP 0-03071）

问：本端 Diameter 端口号（Gx/Gy/S6b）是否需要显式配置？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 本端端口可配 | `GXCONCENMODE/GYCONCENMODE/S6BCONCENMODE=LOCALPORT` | 需要对端 CG/OCS/PCRF 指定固定本端端口；不走 DRA 路由代理 |
| 本端端口不可配（系统自动分配） | `GXCONCENMODE/GYCONCENMODE/S6BCONCENMODE=LOCALIP_PEER` | 不需要固定本端端口；走 DRA 路由代理 |

> 取值由本端网络规划决定。模式在 LOCALIP_PEER 和 LOCALPORT 之间切换时，需用 `RMV SCTPASSOC` 删除原模式下的 SCTP 偶联测量对象并重新配置。

### 决策点：Ga 链路数策略（DP 0-03072）

问：Ga 接口（对接 CG）需要多少条链路？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单连接 | `GACONCENMODE=SINGLE_CONNECT` | CG 链路需求少（初始默认） |
| 多端口多连接 | `GACONCENMODE=MULTI_PORT` + `GAPORTPERPROC=1` | CG 链路需求多，需多端口分发；当前版本 GAPORTPERPROC 上限 1 |

## 约束

- **本命令为高危命令**（critical，rule-0-03176）：本命令为高危命令：修改集中点模式可能导致 Gx/Gy/S6b 的 Diameter 链路重建或中断，进而影响用户业务（如被去活） — 后果：现网变更需在维护窗口执行
- **Gx/Gy/S6b 集中点模式在 LOCALIP_PEER 和**（warning，rule-0-03177）：Gx/Gy/S6b 集中点模式在 LOCALIP_PEER 和 LOCALPORT 之间切换时，需使用 `RMV SCTPASSOC` 删除原模式下的 SCTP 偶联测量对象并重新配置 — 后果：未清理旧偶联会导致模式切换后建链异常
- **唯一性**（warning，rule-0-03178）：唯一性：本命令最大记录数为 1（全网一份） — 后果：重复下发被拒绝
- **LOCALIP_PEER 模式下 [ADD**（warning，rule-0-03179）：LOCALIP_PEER 模式下 [ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md) 添加的链路仅 LocalPort 未配或为 0 的生效；LOCALPORT 模式下仅 LocalPort 非 0 的链路生效 — 后果：模式与 LocalPort 配置不符的链路允许下发但不建链
- **GAPORTPERPROC 为 conditional**（info，rule-0-03180）：GAPORTPERPROC 为 conditional，仅 GACONCENMODE=MULTI_PORT 时必填；当前版本不支持该参数超过 1；该命令执行后立即生效 — 后果：超 1 的取值下发失败
