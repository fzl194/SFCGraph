# 创建OSPF接口配置（ADD OSPFINTERFACE）

> 命令静态知识见 [ADD OSPFINTERFACE](command/UNC/20.15.2/ADD-OSPFINTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**。
>
> **★ 2026-07-12 修正**：UNC 域真实命令名为 `ADD OSPFINTERFACE`（v1.0 写的 `ADD OSPFNETWORK` 是历史错误——UNC 命令族采用 `interface` 子命令形式而非标准 `network` 子命令形式）。命令文档路径：`平台服务管理/VNRS功能管理/IP服务/路由管理/OSPF管理/OSPF接口配置/`。

## 配置方法

该命令将指定接口绑定到 OSPF 区域并配置接口级 OSPF 参数，是 OSPF 族的网络承载对象。配置空间核心维度为 **接口绑定**（PROCID + AREAID + IFNAME，决定接口属哪 OSPF 进程 + 哪区域）和 **接口级参数**（认证方式/Hello-Dead 间隔/网络类型/路由过滤等）。

### 配置维度1：接口绑定（参数 PROCID + AREAID + IFNAME）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| PROCID（必填） | OSPF 进程号（须先 `ADD OSPF`）| 1~4294967295 |
| AREAID（必填） | OSPF 区域号（须先 `ADD OSPFAREA`）| 0~4294967295（0 = 骨干区域）|
| IFNAME（必填）| 接口名（物理口/逻辑口/LoopBack）| Ethernet/Xgigabit/LoopBack/LOGICINF 等 |

### 配置维度2：接口级 OSPF 参数（可选）

| 参数组 | 作用 | 典型场景 |
|---|---|---|
| AUTHTYPE=AUTHNOTHING/AUTHMD5/AUTHPASSWORD | 接口认证方式 | 安全敏感场景 |
| AUTHKEYID/AUTHKEY | 认证密钥（MD5 或明文）| 配合 AUTHTYPE |
| HELLOINTV/DEADINTV | Hello/Dead 间隔（默认 10s/40s）| 大型网络调优 |
| NETTYPE=BROADCAST/POINT_TO_POINT | 接口网络类型 | 帧中继/PPP 链路 |
| PRIORITY（0~255）| DR 优先级（默认 1，0 = 不参与 DR 选举）| 影响广播网络 DR 选举 |
| COST | 接口开销（1~65535）| 影响路由选路 |
| PASSIVEINTVFLAG=YES | 被动接口（不收发 Hello，仅宣告网络）| 接入设备/Stub 区域 |

**步骤位置**：在 OSPF 族编排中——于 `ADD OSPF`（建进程）+ `ADD OSPFAREA`（建区域）之后、`ADD OSPFIMPORTROUTE`（引入外部路由）之前执行。是 OSPF 进程把接口"挂"上区域的关键命令。

## 决策点

**接口类型 × 认证方式**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 物理口/逻辑口（基线）| PROCID + AREAID + IFNAME + COST | 公网/私网 OSPF 接口 |
| LoopBack | 同上 + NETTYPE=POINT_TO_POINT | 网络稳定锚点（Router ID 来源）|
| Stub 区域 | + PASSIVEINTVFLAG=YES | 边界路由器 |
| 认证接口 | + AUTHTYPE + AUTHKEY | 安全敏感 |

> 接口绑定必须在 OSPF 进程 + 区域已建的基础上；同一接口在同一区域只能绑定一次（重复 `ADD OSPFINTERFACE` 同一 PROCID+AREAID+IFNAME 失败）。

## 约束

- **PROCID/AREAID/IFNAME 三元组唯一性**（critical）：同一进程同一区域同一接口只能绑定一次 — 后果：重复新增失败
- **PROCID 必须先建**（critical）：须由 `ADD OSPF` 已配置 — 后果：进程不存在则新增失败
- **AREAID 必须先建**（critical）：须由 `ADD OSPFAREA` 已配置 — 后果：区域不存在则新增失败
- **IFNAME 必须存在**（critical）：接口必须已由 `ADD INTERFACE`/`ADD LOGICINF` 等建好 — 后果：接口不存在则新增失败
- **HELLOINTV < DEADINTV**（warning）：Hello 间隔必须小于 Dead 间隔，Dead 默认 = 4 × Hello — 后果：错配导致邻接频繁 Down
- **PRIORITY=0 不参与 DR 选举**（warning）：在广播网络上 PRIORITY=0 接口永久不参与 DR 选举 — 后果：DR 由其他路由器选举
- **PASSIVE 接口不发 Hello**（warning）：被动接口不建立邻接但仍宣告网络 — 后果：误配致该接口下挂设备无法建立 OSPF 邻接
