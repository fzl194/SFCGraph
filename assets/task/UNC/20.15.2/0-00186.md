# SET N4CHGMSGCTRL 原子任务
> 命令静态知识见 [SET N4CHGMSGCTRL](command/UNC/20.15.2/SET-N4CHGMSGCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

该命令控制单会话 N4 计费消息缓存池的扩展与满池动作。系统部署后已存在初始记录（MSGCACHEEPDSW=DISABLE、CACHEFULLACT=TERMINATE，即默认不扩展且满池去活）。仅 WSFD-011206（配置计费消息缓存）引用，用于高带宽/CHF 慢响应场景下保活用户。

## 配置方法
按数据规划表两参数同时配置：开启缓存池扩展（ENABLE）并选择满池动作（CONTINUE 允许业务继续）。未输入参数保持系统当前值不变（可通过 LST N4CHGMSGCTRL 查看）。

| 参数 | 描述 | 配置方法 | 取值样例 |
|---|---|---|---|
| MSGCACHEEPDSW | 消息缓存池扩展开关 | 本端规划 | `ENABLE` / `DISABLE` |
| CACHEFULLACT | 缓存池满的动作 | 本端规划 | `CONTINUE`（允许业务继续，不再配额管理） / `TERMINATE` |

典型脚本（WSFD-011206，开启缓存池扩展 + 满池允许业务继续）：
```
SET N4CHGMSGCTRL: MSGCACHEEPDSW=ENABLE, CACHEFULLACT=CONTINUE;
```

扩展规格说明：单个会话 N4 计费消息缓存池满（20 条）时，扩展到 40 条，最多支持 100 个会话进行扩展；若该会话已缓存消息中存在丢中间消息，则不可扩展。

## 决策点

### MSGCACHEEPDSW 与 CACHEFULLACT 组合选择
- 问什么：缓存池是否扩展、满池后是去活还是继续业务。
- 取值由组网对 CHF 响应时延的容忍度决定：默认 DISABLE+TERMINATE 倾向去活保账；高带宽/CHF 慢响应场景选 ENABLE+CONTINUE 保活用户但放弃配额管理。
- ②conditional 分叉：两参数为可选联动——扩展开关开启后满池动作才有实际意义，二者需协同决策。

## 约束
- 唯一性：全局一份（系统部署后已存在初始记录）。`severity=medium`
- 生效时机：执行后立即生效。`severity=high`
- 规格上限：单个会话缓存池扩展上限 40 条消息，最多 100 个会话可扩展；存在丢中间消息的会话不可扩展。`severity=high`
- 行为变更（CONTINUE）：满池动作=CONTINUE 时允许业务继续但不再进行配额管理，影响计费准确性。`severity=high`
- 默认值：MSGCACHEEPDSW=DISABLE、CACHEFULLACT=TERMINATE（满池默认去活）。`severity=low`
- 前置（告警监控软参）：MONITORTIME=0 时告警监控周期受软参 DWORD507 控制（扫描硬盘缓存文件周期），间接影响缓存监控。`severity=medium`
- 适用 NF：SMF、PGW-C、GGSN。`severity=low`
