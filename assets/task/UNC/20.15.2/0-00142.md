# RMV PCRF 原子任务
> 命令静态知识见 [RMV PCRF](command/UNC/20.15.2/RMV-PCRF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

**高危命令**。删除对象是 `PCRF`（PCRF 主机记录）。通过 `HOSTNAME` 定位，`FORCED` 控制是否强制删除，二者组合决定级联删除行为。

## 配置方法
按 PCRF 主机名定位删除，配合是否强制。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| HOSTNAME | PCRF 主机名（local_planned，required） | `pcrf-001` |
| FORCED | 是否强制删除（global_planned，optional，默认 False） | `TRUE` / `FALSE` |

典型脚本（普通删除单台 PCRF）：
```
RMV PCRF: HOSTNAME=pcrf-001;
```

强制删除：
```
RMV PCRF: HOSTNAME=pcrf-001, FORCED=TRUE;
```

## 决策点（若有）

### FORCED 强制删除开关
- 问什么：是否绕过级联约束强制删除。
- `FORCED=FALSE`（默认）：删除时按规则级联处理 PCRF 组/Diameter 对端地址。
- `FORCED=TRUE`：强制删除，用于难以解除绑定的清理场景。

### PCRF 组成员数决定的级联行为
- 问什么：PCRF 在 PCRF 组中是唯一成员还是多成员之一。
- 唯一成员（组内仅此一个 PCRF）：删除 PCRF 同时**删除整个 PCRF 组**。
- 多成员（组内还有其他 PCRF）：删除 PCRF 仅从组中移除该成员，组保留。
- 此级联由系统按组内成员数自动判断，非用户参数控制。

### Diameter 对端地址级联
- 若本设备是最后一个使用该 HOSTNAME 的 Diameter 应用设备，删除时**同时删除该 HostName 对应的 Diameter 对端地址**；否则保留。系统按引用计数自动判断。

### 直连路径改造场景的前置用户去活
- 直连路径改经 DRA 非直连路径、或直连/非直连共存切换场景，删除直连路径前必须**先锁定对应 APN 并去活用户**，用户去活完成后才能执行 RMV PCRF，删后（如需 DRA 路径）执行 ADD REALMBINDAPN 绑定 DRA 到 APN，再解锁 APN。
- 此为跨场景安全前置，非命令参数。

## 约束（若有）
- 高危：删除 PCRF 可能导致 Diameter 链路中断、用户被去活等业务影响，删前必须确认链路未承载在用会话。`severity=high`
- 直连路径改造前置：直连路径删除前必须先锁 APN + 去活用户，再 RMV PCRF，再绑 DRA + 解锁 APN，严禁跳步。`severity=high`
- 级联删除：组内唯一成员时连带删 PCRF 组；末位引用时连带删 Diameter 对端地址——执行前需预期级联影响。`severity=high`
- 生效时机：立即生效。`severity=medium`
- 主机名格式：HOSTNAME 不支持空格、须可见 ASCII，长度 1～127，是否支持中文由软参 BIT150 控制。`severity=medium`
