# 删除PCRF（RMV PCRF）

> 命令静态知识见 [RMV PCRF](command/UNC/20.15.2/RMV-PCRF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按 PCRF 主机名（HOSTNAME）定位删除，配合 FORCED 控制是否强制删除；二者组合决定级联删除行为。删除对象是 `PCRF`（PCRF 主机记录），属 PCC 策略控制的核心配置。

### 参数维度表

| 维度 | 参数 | 数据来源 | 取值 | 默认 | 作用 | 适用场景 |
|------|------|----------|------|------|------|----------|
| PCRF 主机名 | HOSTNAME | 本端规划 | 字符串 1～127，不支持空格，须可见 ASCII，是否区分大小写由软参 BIT150 控制 | — | 定位待删 PCRF，必选参数 | 精确删指定 PCRF |
| 是否强制删除 | FORCED | 全网规划 | DISABLE / ENABLE | False（不使能） | True 时无论是否存在用户都可强制删；False 时若存在用户则拒绝 | 清理难解绑场景 |

> 说明：高危命令。HOSTNAME 必选。FORCED=True 仅豁免"有用户"约束，仍受绑定条件约束（如 PCRF 组、Diameter 链路组、Diameter 对端地址引用）。删前必须确认链路未承载在用会话。

### 典型脚本

不再使用 PCRF 实例 pcrf1，普通删除：
```
RMV PCRF: HOSTNAME="pcrf1";
```

强制删除（清理场景）：
```
RMV PCRF: HOSTNAME="pcrf1", FORCED=ENABLE;
```

### 步骤位置

直连路径改造场景的标准动作：
1. `LST PCRF` — 核对目标 PCRF 是否承载在用会话（未建）
2. 锁定对应 APN + 去活用户（直连路径改造前置）
3. `RMV PCRF` — 删除直连路径 PCRF（本案）
4. 如需 DRA 路径：`ADD REALMBINDAPN` 绑定 DRA 到 APN（文字提及，未在编号映射）
5. 解锁 APN

## 决策点

### 决策点：FORCED 强制删除开关（DP 0-03023）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| FORCED=DISABLE/省略 | 按默认 False 执行；若 PCRF 上存在用户则拒绝删除 | 常规回收，期望系统保守拦截 |
| FORCED=ENABLE | 无论 PCRF 上是否存在用户都强制执行删除（仍受绑定条件约束） | 清理难解绑、用户残留无法去活的场景 |

> 注意：FORCED=True 仅豁免"有用户"约束，PCRF 组/Diameter 链路组/Diameter 对端地址等绑定级联仍按系统规则处理（见 DP 0-03024）。

### 决策点：PCRF 组成员数与引用计数决定的级联行为（DP 0-03024）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 组内唯一成员 | 连带删除整个 PCRF 组 | 删除后该 PCRF 组无成员 |
| 组内多成员之一 | 仅从组中移除该 PCRF，组保留 | 组内还有其他 PCRF |
| 末位 Diameter 引用 | 连带删除该 HOSTNAME 对应的 Diameter 对端地址 | 本设备是最后一个使用该 HOSTNAME 的 Diameter 应用设备 |
| Diameter 链路组引用 | 连带删除该 Diameter 链路组配置 | HOSTNAME 已被 Diameter 链路组引用 |

> 注意：此级联由系统按组内成员数与引用计数自动判断，非用户参数控制；执行前需预期级联影响范围。直连路径改造（改经 DRA 或直连/非直连共存切换）删除直连路径前必须先锁 APN + 去活用户，删后再处理 DRA 绑定与解锁 APN，严禁跳步。15 分钟内 RMV PCRF 再 ADD PCRF 可能导致该 HOSTNAME 局向指标无法上报，需在网管上重新订阅。

## 约束

- **高危业务影响**（critical，rule-0-03056）：高危业务影响：删除 PCRF 可能导致 Diameter 链路中断、用户被去活等业务影响，删前必须确认链路未承载在用会话
- **直连路径改造前置**（critical，rule-0-03057）：直连路径改造前置：直连路径删除前必须先锁 APN + 去活用户，再 RMV PCRF，再（如需）绑 DRA + 解锁 APN，严禁跳步
- **级联删除**（warning，rule-0-03058）：级联删除：组内唯一成员连带删 PCRF 组；末位引用连带删 Diameter 对端地址；Diameter 链路组引用连带删链路组——执行前需预期级联影响
- **指标订阅重订**（warning，rule-0-03059）：指标订阅重订：15 分钟内 RMV PCRF 再 ADD PCRF 可能导致该 HOSTNAME 局向指标无法上报，需在网管上重新订阅相关指标
- **主机名格式**（info，rule-0-03060）：主机名格式：HOSTNAME 不支持空格、须可见 ASCII，长度 1～127，是否区分大小写由软参 BIT150 控制；执行后立即生效
