# 基于接入点策略控制（WSFD-109108）

> 特性静态知识见 [WSFD-109108](feature/UNC/20.15.2/WSFD-109108.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系（必读）**：本特性是带宽族第 4 批。**纯被动响应型特性**——参考信息原文「本特性无相关 MML 命令 / 无相关告警 / 无相关软参 / 无相关测量指标」。功能实现完全依赖：① License 加载（LKV3WPWULI11）后自动生效；② PCC 基本功能（[WSFD-109101](task/UNC/20.15.2/2-00005.md)）的 Gx 接口通道承载 Event-Trigger 上报。配置生成仅编 1 步 License 开关 + 引用 PCC 基本功能前置；话单扩展字段与 Radius 消息携带位置为运行时自动行为，无 PGW-C 侧配置命令。
>
> 与 UDG 侧无对应特性（本特性是 UNC/PGW-C 侧 Non-3GPP 接入专属，UDG/UPF 侧仅执行 PGW-C 下发的策略动作）。

## 配置概览

核心对象链（无本特性专属对象，全部复用 PCC 基本功能基础设施）：

**License 前置**（LKV3WPWULI11）→ **PCC 基本功能 Gx 路径**（VPN→逻辑接口→Diameter 本端→PCRF→链路组→PCRF 分组→PCC 开关）→ 运行时 PGW-C 被动检测 UE Local IP/Port 变化 → Gx 接口上报 Event-Trigger(43) → PCRF 下发差异化策略 → PGW-C 执行。

本特性的"接入点"特指**非可信 Non-3GPP（WiFi）接入场景下，WiFi 用户所在的网络位置**，通过 UE Local IP + UE UDP Port 组合标识——与传统 APN/DNN（接入点名称/数据网络名称）维度的策略控制不同，后者不在本特性范围。

两条可选联动（均为运行时行为，无配置命令）：
- **离线话单扩展**：依赖 [2-00001 离线计费](task/UNC/20.15.2/2-00001.md) 已激活；PGW-CDR 扩展字段携带用户 IP/Port，IP/Port 变化不触发新容器/话单。
- **Radius 消息携带**：依赖 Radius 功能（WSFD-011306）/ Radius 抄送（WSFD-011307）已激活；Accounting Start/Stop/Update 自动携带位置信息。

关键差异维度：① 接入类型（仅 Non-3GPP WiFi，3GPP 接入不适用）② 是否需话单记录位置（联动离线计费）③ 是否需 Radius 上报位置（联动 Radius）。

## 配置流程（步骤编排，每步显示含命令）

1. **激活本特性 License**（前置，本特性唯一专属步骤）：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)
   - 关键参数：LICITEM="LKV3WPWULI11", SWITCH=ENABLE（控制项 82209475 基于接入点策略控制）
   - **生效方式**：License 加载后即自动生效，无需额外配置开关（特性概述原文）
2. **完成 PCC 基本功能前置**（必前置，本特性依赖其 Gx 通道）：完整参见 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md)
   - 关键：须完成 Gx 接口路径（PCRF 对接链 [1-00012](task/UNC/20.15.2/1-00012.md) + PCRF 选择 [1-00013](task/UNC/20.15.2/1-00013.md) + PCC 开关与模板 [1-00014](task/UNC/20.15.2/1-00014.md)）
   - License 前置：PCC 基本功能双 License（LKV2PCCBF01 + LKV3W9SPCC11）须已激活
   - **无本特性专属增量参数**：不像 FUP（[2-00007](task/UNC/20.15.2/2-00007.md)）需在 SET PCCFUNC/MOD PCRF/MOD PCCPOLICYGRP 叠加增量——本特性 Event-Trigger(43) 上报是 Gx 标准信元，PCC 基本功能激活后即支持
3. **（可选，话单记录位置）完成离线计费前置**：参见 [2-00001 离线计费](task/UNC/20.15.2/2-00001.md)
   - 仅当需在 PGW-CDR 中记录 WiFi 用户 IP/Port 时前置；运行时话单扩展字段自动携带，无配置命令
   - **CG 侧要求**：CG 须支持解析处理话单中 WiFi 用户 IP/Port 扩展字段（应用限制，非 PGW-C 配置）
4. **（可选，Radius 上报位置）完成 Radius 功能前置**：依赖 WSFD-011306 Radius 功能 / WSFD-011307 Radius 抄送功能（UNC 侧特性 wiki 待建）
   - 仅当需在 Radius 消息中携带 WiFi 用户位置时前置；运行时 Accounting Start/Stop/Update 自动携带，无配置命令

> **activation 演示范围脚注（R1.4）**：WSFD-109108 仅 1 份 activation（[调测基于接入点策略控制_79943605](evidence/UNC/20.15.2/调测基于接入点策略控制_79943605.md)），**无配置脚本**——全部步骤为调测验证（LST LICENSESWITCH 查询 / 创建跟踪任务 / DSP PDUSESSION 查询 / 用户关机开机触发 / 检查 Gx 报文 Event-Trigger）。这印证参考信息「本特性无相关 MML 命令」：唯一配置动作是 `SET LICENSESWITCH`（License 开关，通用命令），其余为查询/验证。**无零证据差异参数**——本特性无差异化配置参数，Event-Trigger(43) 取值与 UE Local IP/Port 信元均为 Gx 标准信元（特性概述 [79067211](evidence/UNC/20.15.2/特性概述_79067211.md) 原文），非 PGW-C 侧配置项。
>
> **配置生成处置**：配置生成按本特性产出 = `SET LICENSESWITCH:LICITEM="LKV3WPWULI11", SWITCH=ENABLE;` 1 条 + 引用 PCC 基本功能（2-00005）的完整配置流程。不产出 LST/DSP/EXP 等查询验证命令（非配置命令，属调测运维）。

## 决策点（特性级）

### DP1：接入类型（决定本特性是否适用）

| 接入类型 | 走法 | 适用性 |
|---|---|---|
| 非可信 Non-3GPP（WiFi，经 ePDG S2b 接入） | 步骤 1（License）+ 步骤 2（PCC 基本功能前置） | ✓ 适用——PGW-C 检测 UE Local IP/Port 变化上报 Event-Trigger(43) |
| 3GPP 接入（2G/3G/4G/5G） | 不适用 | ✗ 本特性仅针对 Non-3GPP WiFi 接入场景，3GPP 接入无 UE Local IP/Port 维度 |

### DP2：是否需话单记录 WiFi 用户位置（联动离线计费）

| 选项 | 影响 | 前置 |
|---|---|---|
| 需要（话单扩展字段携带 IP/Port） | 步骤 3 前置离线计费；运行时 PGW-CDR 扩展字段自动携带；CG 须支持解析扩展字段 | [2-00001 离线计费](task/UNC/20.15.2/2-00001.md) 已激活 |
| 不需要 | 步骤 3 跳过；话单不携带位置字段 | 无 |

> **话单行为约束**：WiFi 用户 IP/Port 变化**不触发**新容器或话单生成，不涉及计费和话单结构变更（特性概述原文）。

### DP3：是否需 Radius 上报 WiFi 用户位置（联动 Radius）

| 选项 | 影响 | 前置 |
|---|---|---|
| 需要（Radius 消息携带位置） | 步骤 4 前置 Radius 功能；运行时 Accounting Start/Stop/Update 自动携带 IP/Port | WSFD-011306 / WSFD-011307 已激活 |
| 不需要 | 步骤 4 跳过；Radius 消息不携带位置 | 无 |

## 约束（特性级 rule）

- **依赖 PCC 基本功能**（critical）：本特性须先激活 [WSFD-109101 PCC基本功能](task/UNC/20.15.2/2-00005.md)（License LKV2PCCBF01+LKV3W9SPCC11 + Gx 接口 PCRF 对接 + PCC 开关） — 否则无 Gx 通道承载 Event-Trigger 上报，特性无法工作。
- **License 前置**（critical）：须激活 LKV3WPWULI11（控制项 82209475） — 否则基于接入点策略控制功能不生效。
- **仅 Non-3GPP WiFi 接入适用**（critical）：本特性仅适用于非可信 Non-3GPP（WiFi，经 ePDG S2b）接入场景 — 3GPP 接入用户无 UE Local IP/Port 维度，不触发 Event-Trigger(43)。
- **CG 须支持扩展字段解析**（warning）：若开启话单记录位置功能，CG 须支持解析处理 PGW-CDR 中 WiFi 用户 IP/Port 扩展字段 — 否则话单字段解析失败。
- **CPU 占用随位置变化次数升高**（warning）：PCRF 或 Radius Server 订阅 WiFi 用户位置变化 trigger 时，CPU 占用率随位置变化次数升高 — 大规模 WiFi 漫游场景须评估订阅粒度。
- **无专属配置命令**（info）：参考信息原文「本特性无相关 MML 命令」——配置生成仅 License 开关 + PCC 基本功能前置，无本特性专属参数化配置。
- **Event-Trigger(43) 为 Gx 标准信元**（info）：UE_LOCAL_IP_ADDRESS_CHANGE(43) 是 Gx 接口标准 Event-Trigger 取值，PCC 基本功能激活后即支持上报，无需额外配置。
