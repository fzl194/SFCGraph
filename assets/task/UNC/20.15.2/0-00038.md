# ADD HTTPLEGRP 原子任务
> 命令静态知识见 [ADD HTTPLEGRP](command/UNC/20.15.2/ADD-HTTPLEGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令极简：每激活一个 NF（AMF/NRF/BSF）的服务化角色，建一个组，`INDEX` 本端编号、`DESC` 用 NF 名标识。组号随后被 [ADD HTTPLE](task/UNC/20.15.2/0-00037.md) 的 `HTTPLEGRPIDX` 引用。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| INDEX | 本端规划，按 NF 分配（AMF=1, NRF=2, BSF=3） | `1` / `2` / `3` |
| DESC | 标识该组归属的 NF 名（可选但建议配） | `"AMF"` / `"NRF"` / `"BSF"` |

典型脚本：
```
ADD HTTPLEGRP: INDEX=1, DESC="AMF";
ADD HTTPLEGRP: INDEX=2, DESC="NRF";
ADD HTTPLEGRP: INDEX=3, DESC="BSF";
```

## 决策点（若有）
无。INDEX 本端规划、DESC 全网规划，均无取值分叉。

## 约束（若有）
- 规格上限：INDEX 取值 1~64；一个组下最多 32 个 HTTPLE 成员。`severity=medium` 后果：超规格新增失败。
- 前置依赖：本组须先于 [ADD HTTPLE](task/UNC/20.15.2/0-00037.md) 建立，否则 HTTPLE 的 HTTPLEGRPIDX 引用失败。`severity=high`
- 执行后立即生效。`severity=low`
