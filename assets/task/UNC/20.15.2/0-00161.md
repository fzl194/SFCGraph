# 配置计费Ga接口参数（SET CHGGA）

> 命令静态知识见 [SET CHGGA](command/UNC/20.15.2/SET-CHGGA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CHGGA 是 SGSN 侧**计费 Ga 接口全局参数命令**：一组参数决定 SGSN 生成话单遵循的协议版本、话单重发策略、硬盘空间保护、UDP 校验及 IP 地址选择策略。本命令影响整个 SGSN 的 Ga 接口行为，参数错误可能导致计费中心无法正确计费。本命令参数众多，核心配置空间有 4 个维度：**话单协议版本**（GCR/UCR/CHGVER98~9/QOSVER）、**CDR 重发策略**（ITVRES/RSNUM）、**硬盘空间保护**（HDDETH/HDDFSL/REDICTFRMMAXOCCRATE/CDROVERWRITE）、**网络层参数**（GASENDUDPCHECK/GAREVUDPCHECK/IPPLCY）。

### 配置维度 1：话单协议版本（参数 GCR/UCR + CHGVER98~9 + QOSVER）

GCR 决定 2G GPRS 话单遵循的协议大版本，UCR 决定 3G UMTS 话单遵循的协议大版本；CHGVER98~9 在对应大版本生效时进一步指定具体遵循的规范版本。数据来源整网规划：

| 参数 | 控制对象 | 取值 | 系统初始值 |
|---|---|---|---|
| `GCR` | GPRS CDR 协议版本（2G） | `R98`/`R99`/`R4`/`R5`/`R6`/`R7`/`R9` | `R98` |
| `UCR` | UMTS CDR 协议版本（3G） | `R99`/`R4`/`R5`/`R6`/`R7`/`R9` | `R99` |
| `CHGVER98` | R98 话单规范版本（GCR=R98 时生效） | `CMCCV130`/`GSM1215V760`/`USI1215V760` | `CMCCV130` |
| `CHGVER99` | R99 话单规范版本（GCR/UCR=R99 时生效） | `GPP32015V360`/`GPP32015V3A0`/`AIS32015V360` | `GPP32015V360` |
| `CHGVER4` | R4 话单规范版本（GCR/UCR=R4 时生效） | `GPP32215V440`/`VOX32215V440` | `GPP32215V440` |
| `CHGVER5` | R5 话单规范版本（GCR/UCR=R5 时生效） | `GPP32215V560` | `GPP32215V560` |
| `CHGVER6` | R6 话单规范版本（GCR/UCR=R6 时生效） | `GPP32251V660` | `GPP32251V660` |
| `CHGVER7` | R7 话单规范版本（GCR/UCR=R7 时生效） | `GPP32251V740` | `GPP32251V740` |
| `CHGVER9` | R9 话单规范版本（GCR/UCR=R9 时生效） | `GPP32251V940` | `GPP32251V940` |
| `QOSVER` | 话单 QoS 最高版本 | `R5`/`R7` | `R5` |

> 各话单协议版本必须与对接 CG 的协议版本一致（联动 [ADD CHGCG](task/UNC/20.15.2/0-00017.md)）。CHGVER99 的 `AIS32015V360` 在 S-CDR/S-SMO-CDR/S-SMT-CDR 中增加 PLMN ID 字段，CHGVER4 的 `VOX32215V440` 对部分信元做了扩展重定义。

### 配置维度 2：CDR 重发策略（参数 ITVRES + RSNUM）

| 参数 | 作用 | 取值范围 | 系统初始值 |
|---|---|---|---|
| `ITVRES` | CDR 重发间隔（s），CDP 发送话单后未收 CG 应答的重发间隔 | `1`~`10` | `3` |
| `RSNUM` | CDR 重发次数，CDP 发送话单的最大重发次数 | `1`~`10` | `3` |

### 配置维度 3：硬盘空间保护（参数 HDDETH/HDDFSL/REDICTFRMMAXOCCRATE + CDROVERWRITE）

| 参数 | 作用 | 取值范围 | 系统初始值 |
|---|---|---|---|
| `HDDETH` | 硬盘操作失败告警次数门限，CDP 连续硬盘操作失败超门限触发告警 | `1`~`10` | `5` |
| `HDDFSL` | 硬盘空间不足门限（%），已用/总量超门限在主备硬盘间切换存储 | `50`~`90` | `70` |
| `REDICTFRMMAXOCCRATE` | 重定向帧最大占用率（%），CG 故障重指 CG 时发出帧的最大占用率 | `20`~`80` | `80` |
| `CDROVERWRITE` | 覆盖硬盘上话单文件，两盘均不足时是否删旧存新 | `NO`（不覆盖，话单丢失）/ `YES`（覆盖） | `NO` |

> 开启 `CDROVERWRITE=YES` 前应通过 DSP CHGFILE 确认硬盘不存在话单文件，否则删除的文件不能保证为最旧；SFTP 上传场景系统只有最后修改时间，不保证删最旧。

### 配置维度 4：网络层参数（参数 GASENDUDPCHECK/GAREVUDPCHECK + IPPLCY）

| 参数 | 作用 | 取值 | 系统初始值 |
|---|---|---|---|
| `GASENDUDPCHECK` | 发送消息 UDP 校验（Ga 口 UDP 连接） | `DISABLE`/`ENABLE` | `ENABLE` |
| `GAREVUDPCHECK` | 接收消息 UDP 校验 | `DISABLE`/`ENABLE` | `ENABLE` |
| `IPPLCY` | Ga 接口 IP 地址选择策略 | `IPV4`（仅 IPv4）/ `IPV6`（仅 IPv6）/ `IPV6PRE`（优先 IPv6） | `IPV4` |

> `IPPLCY` 按 IPv4/IPv6 改造阶段选：纯 IPv4 → `IPV4`；改造过程中 → `IPV6PRE`；IPv6 改造完成 → `IPV6`。

典型脚本（话单可靠性保证：硬盘话单覆盖）：
```
SET CHGGA: HDDFSL=90, CDROVERWRITE=YES;
```
离线话单版本与 CG 对齐（UMTS R99 + 3GPP 32.015 V3.6.0）：
```
SET CHGGA: UCR=R99, CHGVER99=GPP32015V360;
```
热计费 UMTS 话单版本对齐（R99 + 3GPP 32.015 V3.a.0）：
```
SET CHGGA: UCR=R99, CHGVER99=GPP32015V3A0;
```

**步骤位置**：在所属 feature 编排中通常位于"配置 Ga 接口全局参数"步骤——[ADD CHGCG](task/UNC/20.15.2/0-00017.md) 配置 CG 之后，全局话单生成策略 [SET CHGCDR](task/UNC/20.15.2/0-00159.md) / CC 级 [SET CHGCHAR](task/UNC/20.15.2/0-00160.md) 之前或同级（Ga 接口参数是话单发送通道层，独立于话单生成策略层）。

## 决策点

### 决策点：话单协议版本选择（DP 0-03061）

问：GPRS / UMTS 用户分别走哪个 CDR 协议版本，且该版本对应遵循哪个具体规范？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 2G/3G 分开指定 | `GCR=R98` + `UCR=R99`（SGSN 同时支持 2G/3G） | SGSN 同时为 2G/3G 用户指定不同话单格式 |
| R99 离线计费 | `UCR=R99` + `CHGVER99=GPP32015V360` | 遵循 3GPP 32.015 V3.6.0，与对接 CG 协议版本一致 |
| R99 热计费 | `UCR=R99` + `CHGVER99=GPP32015V3A0` | 遵循 3GPP 32.015 V3.a.0（2003 年 1 月发布） |
| R99 带 PLMN ID 增强 | `CHGVER99=AIS32015V360` | 在 S-CDR/S-SMO-CDR/S-SMT-CDR 中增加 PLMN ID 字段 |
| R4 增强规范 | `CHGVER4=VOX32215V440` | 基于 3GPP 32.215 V4.4.0 对部分信元扩展重定义 |

> CHGVER98/99/4/5/6/7/9 必须与对接 CG 的协议版本一致（证据 WSFD-011202）；错误版本可致计费中心无法正确计费。

### 决策点：IP 地址选择策略（DP 0-03062）

问：IPv4/IPv6 改造阶段如何选 Ga 接口 IP 地址策略？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 纯 IPv4 网络 | `IPPLCY=IPV4` | 仅使用 IPv4 地址（初始值） |
| IPv4 向 IPv6 改造过渡 | `IPPLCY=IPV6PRE` | 优先使用 IPv6 地址，改造过程中 |
| IPv6 网络改造完成 | `IPPLCY=IPV6` | 仅使用 IPv6 地址 |

> 改造过程的具体行为见命令联机帮助；纯 IPv4 网络必选 `IPV4`。

## 约束

- **计费参数错误修改风险**（critical，rule-0-03151）：计费参数错误修改可能导致计费中心无法正确计费，需谨慎修改 — 后果：错误修改致计费失败
- **协议版本必须与 CG 一致**（high，rule-0-03152）：GCR/UCR/CHGVER98/99/4/5/6/7/9 各话单协议版本必须与对接 CG 协议版本一致（联动 [ADD CHGCG](task/UNC/20.15.2/0-00017.md)） — 后果：不一致致 CG 无法解析话单
- **部分参数复位生效**（high，rule-0-03153）：`ITVRES`/`HDDETH`/`HDDFSL`/`REDICTFRMMAXOCCRATE` 需复位 CDP 进程生效；`GCR`/`UCR`/`CHGVER98`/`CHGVER99`/`CHGVER4`~`CHGVER9`（协议版本类）需重启 SPP 和 UPP 进程生效 — 后果：未复位参数不生效
- **硬盘话单覆盖依赖**（warning，rule-0-03154）：硬盘话单覆盖功能依赖 `CDROVERWRITE=YES` 且硬盘使用达 `HDDFSL` 门限；SFTP 上传场景下删除最旧文件不保证（系统只有最后修改时间） — 后果：覆盖时不能保证删最旧
- **系统初始设置值**（info，rule-0-03155）：系统初次运行执行系统初始设置值；`RSNUM`/`CDROVERWRITE` 配置后立即生效 — 后果：未显式配置时走初始值
