# 修改GRE隧道（MOD GRETUNNEL）

> 命令静态知识见 [MOD GRETUNNEL](command/UNC/20.15.2/MOD-GRETUNNEL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令修改已存在的 GRE 隧道（由 `ADD GRETUNNEL` 建立）的配置，常用于网络重新规划时调整源/目的 IP 或安全性/可靠性功能。可修改维度与 ADD GRETUNNEL 一致。

### 配置维度1：可修改的字段族

| 字段族 | 参数 | 作用 |
|---|---|---|
| 源/目的 IP | SRCTYPE/SRCIPADDR/SRCIFNAME/DSTIPADDR（IPv4）；SRCTYPE6/SRCIPV6ADDR/DSTIPV6ADDR（IPv6） | 重新规划隧道端点 |
| 目的 VPN | DSTVPNNAME | 修改目的地址所属 VPN |
| Keepalive | KEEPALVEN/KEEPALVPERIOD/KEEPALVRETRYCNT | 调整可达性探测参数 |
| 识别关键字 | GREKEYEN/GREKEY | 启用/修改通道识别关键字 |
| 端到端校验 | CHECKSUMEN | 启用/去使能校验和 |
| 报文统计 | STATENABLE | 启用/去使能流量统计 |
| 备份隧道 | REDUNDANCYEN | 启用/去使能备份 |

### 配置维度2：常见修改场景

| 场景 | 典型参数组合 |
|---|---|
| 启用 Keepalive | TNLNAME, TNLTYPE, KEEPALVEN=TRUE, KEEPALVPERIOD=5, KEEPALVRETRYCNT=3 |
| 启用识别关键字 | TNLNAME, TNLTYPE, GREKEYEN=TRUE, GREKEY="123" |
| 启用校验和 | TNLNAME, TNLTYPE, CHECKSUMEN=TRUE |

**步骤位置**：在所属 feature（IPFD-015002 GRE）编排中，于 `ADD GRETUNNEL` 之后、按需调整时执行。

## 决策点

（无。本命令为按需修改，无特性级分支。）

## 约束

- **定位参数必填**（critical）：TNLNAME + TNLTYPE 是必选参数，用于定位要修改的隧道 — 后果：缺一无法定位
- **源类型配套**（warning）：SRCTYPE=ip_address 时 SRCIPADDR 必填；SRCTYPE=if_name 时 SRCIFNAME 必填 — 后果：错配失败
- **本端源 IP = 对端目的 IP**（critical）：修改后的源 IP 必须与对端目的 IP 相同 — 后果：隧道中断
- **识别关键字两端一致**（warning）：GREKEYEN=TRUE 时两端 GREKEY 必须一致 — 后果：报文丢弃
