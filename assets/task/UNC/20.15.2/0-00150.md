# 设置APN空闲上下文定时器（SET APNIDLETIME）

> 命令静态知识见 [SET APNIDLETIME](command/UNC/20.15.2/SET-APNIDLETIME.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET APNIDLETIME 配置指定 APN 的空闲上下文时长（无数据传输超时删会话/承载/QoS Flow）和不活动上下文时长（释放用户面资源，仅 5G）。功能防垃圾上下文与无线侧资源浪费。APN 粒度优先级 > 全局粒度（[SET DFTIDLETIME](command/UNC/20.15.2/SET-DFTIDLETIME.md)）。每次执行 ADD APN 时会自动为本命令增加一条初始记录（INHERIT=YES 等）。配置空间有 4 个维度：**继承开关**（INHERIT=NO 才能独立配置）、**核查开关组**（各 NF 的空闲/不活动核查开关）、**回收级别**（4G 承载级 / 5G QoS Flow 级 / SMF 会话级）、**定时器时长**。

### 配置维度 1：APN 标识与继承开关（参数 APN + INHERIT，必选 + 可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| APN 标识 | `APN` | 本端规划；字符串 1~63，由 `-`/数字/字母/`.` 构成，不区分大小写；必选 | 无 | APN 实例名，由 [ADD APN](command/UNC/20.15.2/ADD-APN.md) 配置生成 | 全部场景 |
| 继承默认开关 | `INHERIT` | 本端规划；枚举：YES/NO | YES | YES=继承 [SET DFTIDLETIME](command/UNC/20.15.2/SET-DFTIDLETIME.md) 的开关状态及时长；NO=不继承，未设置的可选参数取 ADD APN 自动生成记录的初始值 | 独立配置 APN 级定时器时置 NO |

### 配置维度 2：核查开关组（参数 SCTXCHKSW/PCTXCHKSW/GCTXCHKSW/HCTXCHKSW/ICTXCHKSW/HCTXINACHKSW/ICTXINACHKSW/PROS8CTXCHKSW/PROXYSMFCHKSW，conditional，INHERIT=NO 时生效）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| SGW-C 空闲核查 | `SCTXCHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | SGW-C 空闲上下文使能状态 | 4G/2G/3G；开关由 DISABLE 改 ENABLE 对新创建上下文检查 |
| PGW-C/合一核查 | `PCTXCHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | PGW-C 和 SGW-C/PGW-C 合一部署时空闲核查 | 4G/2G/3G |
| GGSN 空闲核查 | `GCTXCHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | GGSN 空闲核查 | 2G/3G |
| H-SMF 空闲核查 | `HCTXCHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | H-SMF 空闲核查 | 5G |
| I-SMF/V-SMF 空闲核查 | `ICTXCHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | I-SMF/V-SMF 空闲核查；I-SMF/V-SMF 下无辅锚点 UPF 时只支持会话级核查 | 5G 漫游 |
| H-SMF 不活动核查 | `HCTXINACHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | H-SMF 不活动上下文核查 | 5G |
| I-SMF/V-SMF 不活动核查 | `ICTXINACHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | I-SMF/V-SMF 不活动核查 | 5G 漫游 |
| Proxy-SMF S8 核查 | `PROS8CTXCHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | Proxy-SMF S8 空闲核查；改 ENABLE 后 SMF 会话空闲定时器生效 | Proxy-SMF S8 场景 |
| Proxy-SMF 核查 | `PROXYSMFCHKSW` | 本端规划；枚举：ENABLE/DISABLE | ENABLE | Proxy-SMF 空闲核查 | Proxy-SMF 场景 |

### 配置维度 3：回收级别（参数 GULTIMERLEVEL + HSMFTIMERLEVEL，conditional，INHERIT=NO 时生效）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| GUL 承载级别 | `GULTIMERLEVEL` | 本端规划；枚举：SESSION/BEARER；仅 2G/3G/4G | SESSION | SESSION=会话级（SESSIONTIMER 生效）；BEARER=承载级（DFTBEARPOLICY/BEARERTIMER 生效） | 4G 紧急呼叫/IMS 用 BEARER；2G/3G 二次上下文用 BEARER |
| H-SMF 核查级别 | `HSMFTIMERLEVEL` | 本端规划；枚举：SESSION/QOSFLOW；仅 5G | SESSION | SESSION=会话级（SMFSESIDLETIMER 生效，DED/DFT QF 定时器不生效）；QOSFLOW=QoS Flow 级（DEDQFIDLETIMER/DFTQFIDLETIMER 生效，SMF 会话空闲不生效） | 5G QoS 保证用 QOSFLOW；会话级回收用 SESSION |

### 配置维度 4：定时器时长组（参数 BEARERTIMER/SESSIONTIMER/DEDQFIDLETIMER/DFTQFIDLETIMER/SMFSESIDLETIMER/SMFSESINATIMER/DFTBEARPOLICY/DFTBEARERTIMER/DFTBRPLCYGGSN/IDLEUPDATEMSG，conditional）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 承载定时器 | `BEARERTIMER` | 本端规划；整数 5~12000（min） | 1440 | 2G/3G/4G 承载定时器时长；GULTIMERLEVEL=BEARER 时生效 | 紧急呼叫建议 60；IMS 建议 5 |
| 会话定时器 | `SESSIONTIMER` | 本端规划；整数 5~12000（min） | 1440 | 2G/3G/4G 会话定时器时长；GULTIMERLEVEL=SESSION 时生效 | 2G/3G/4G 会话级 |
| 专有 QoS Flow 定时器 | `DEDQFIDLETIMER` | 本端规划；整数 5~12000（min）；HSMFTIMERLEVEL=QOSFLOW 时生效 | 1440 | 5G 专有 QoS Flow 空闲定时器 | 业务触发 QoS 保证（如 600） |
| 缺省 QoS Flow 定时器 | `DFTQFIDLETIMER` | 本端规划；整数 0 或 5~12000（min）；HSMFTIMERLEVEL=QOSFLOW 时生效 | 1440 | 5G 缺省 QoS Flow 空闲定时器；0=不开启缺省 QoS Flow 空闲核查 | 5G |
| SMF 会话空闲定时器 | `SMFSESIDLETIMER` | 本端规划；整数 5~12000（min） | 1440 | 5G SMF PDU 会话空闲定时器；HSMFTIMERLEVEL=SESSION 时生效 | 5G 会话级 |
| SMF 会话不活动定时器 | `SMFSESINATIMER` | 本端规划；整数 5~12000（min） | 40 | 5G SMF PDU 会话不活动定时器（释放用户面资源） | 5G；须 ≤ SMFSESIDLETIMER |
| 缺省承载定时器策略 | `DFTBEARPOLICY` | 本端规划；枚举：ONEDAY/OFF/USEBEARERTIMER/USESPECIFICVAL；仅 4G | ONEDAY | 缺省承载和默认 GBR 的定时器策略 | 紧急呼叫 ONEDAY；IMS OFF |
| 缺省承载定时器时长 | `DFTBEARERTIMER` | 本端规划；整数 0~255（min）；DFTBEARPOLICY/DFTBRPLCYGGSN=USESPECIFICVAL 时生效 | 0 | 缺省承载定时器时长；0 时以 SESSIONTIMER 取值为准 | 4G |
| GGSN 一次激活定时器 | `DFTBRPLCYGGSN` | 本端规划；枚举：ONEDAY/OFF/USEBEARERTIMER/USESPECIFICVAL；仅 2G/3G | ONEDAY | GGSN 一次激活上下文定时器 | 2G/3G |
| 空闲超时更新消息 | `IDLEUPDATEMSG` | 本端规划；枚举：ENABLE/DISABLE；仅 2G/3G/4G | DISABLE | 持续无流量超 APN 配置时长后是否发送更新消息 | 2G/3G/4G |

**典型脚本（5G QoS Flow 级，业务触发 QoS 保证）**
```
SET APNIDLETIME: APN="huawei.com", INHERIT=NO, HSMFTIMERLEVEL=QOSFLOW, DEDQFIDLETIMER=600;
```

**典型脚本（4G 紧急呼叫缺省承载回收）**
```
SET APNIDLETIME: APN="sos", INHERIT=NO, SCTXCHKSW=ENABLE, PCTXCHKSW=ENABLE, GCTXCHKSW=ENABLE, HCTXCHKSW=ENABLE, ICTXCHKSW=ENABLE, GULTIMERLEVEL=BEARER, DFTBEARPOLICY=ONEDAY, BEARERTIMER=60;
```

**步骤位置**：在会话管理/资源回收特性编排中位于"APN 空闲定时器配置"步骤——业务策略阶段，前置依赖 [ADD APN](command/UNC/20.15.2/ADD-APN.md)（APN 实例源），APN 粒度配置覆盖 [SET DFTIDLETIME](command/UNC/20.15.2/SET-DFTIDLETIME.md) 全局默认。

## 决策点

### 决策点：回收粒度选择（GULTIMERLEVEL / HSMFTIMERLEVEL）（DP 0-03039）

问：按承载级、会话级还是 QoS Flow 级回收空闲上下文？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 4G 承载级 | `GULTIMERLEVEL=BEARER` + `DFTBEARPOLICY=<ONEDAY/OFF/USEBEARERTIMER/USESPECIFICVAL>` + `BEARERTIMER=5~12000` | 4G 紧急呼叫（DFTBEARPOLICY=ONEDAY + BEARERTIMER=60）、IMS 语音（DFTBEARPOLICY=OFF + BEARERTIMER=5） |
| 4G 会话级 | `GULTIMERLEVEL=SESSION` + `SESSIONTIMER=5~12000` | 2G/3G/4G 会话级回收；与承载级且缺省承载/默认 GBR 定时器指定使用承载定时器的效果相同 |
| 5G QoS Flow 级 | `HSMFTIMERLEVEL=QOSFLOW` + `DEDQFIDLETIMER=5~12000` + `DFTQFIDLETIMER=0/5~12000` | 5G 专有 QoS Flow 回收（业务触发 QoS 保证）；此级 SMFSESIDLETIMER 不生效 |
| 5G 会话级 | `HSMFTIMERLEVEL=SESSION` + `SMFSESIDLETIMER=5~12000` + `SMFSESINATIMER=5~12000` | 5G 会话级回收；此级 DED/DFT QoS Flow 定时器不生效 |

### 决策点：时长取值与策略（BEARERTIMER / DEDQFIDLETIMER / SESSIONTIMER / DFTBEARPOLICY 等）（DP 0-03040）

问：各定时器具体时长与缺省承载/默认 GBR 的定时器策略？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 紧急呼叫缺省承载策略 | `DFTBEARPOLICY=ONEDAY` + `BEARERTIMER=60`（不能为 0，防回呼场景缺省承载已被删） | 4G 紧急呼叫；保留缺省承载一天 |
| IMS 语音缺省承载策略 | `DFTBEARPOLICY=OFF` + `BEARERTIMER=5` | IMS 语音；呼叫结束即按 BEARERTIMER 回收 |
| 业务触发 QoS 保证 | `HSMFTIMERLEVEL=QOSFLOW` + `DEDQFIDLETIMER=600`（局点规划） | 5G 专有 QoS Flow 回收；过大长时间占用资源，过小强制拆话后重建增加信令开销 |
| SMF 会话级 | `SMFSESIDLETIMER=1440` + `SMFSESINATIMER=40`（须 ≤ IDLETIMER） | 5G 会话级；不活动先于空闲触发会逻辑冲突 |

## 约束

- **SMF 会话定时器大小关系**（critical，rule-0-03096）：SMF 会话定时器大小关系：`SMFSESINATIMER` 配置值须 ≤ `SMFSESIDLETIMER`，否则不活动先于空闲触发，逻辑冲突；两者取值范围均为 5~12000（min）
- **APN 粒度优先级与继承**（warning，rule-0-03097）：APN 粒度优先级与继承：APN 级配置（INHERIT=NO）覆盖 [SET DFTIDLETIME](command/UNC/20.15.2/SET-DFTIDLETIME.md) 全局默认；INHERIT=YES 时继承全局；INHERIT 由 YES 改 NO 时，未设置的可选参数取 ADD APN 自动生成记录的初始值
- **生效范围与 NB-IoT**（warning，rule-0-03098）：生效范围与 NB-IoT：本命令只对新激活用户生效，不立即作用于在线用户；不适用于 NB-IoT 终端（长时间才交互，NB-IoT 空闲上下文功能参考软参 BYTE801）
- **HSMFTIMERLEVEL 与 GULTIMERLEVEL**（warning，rule-0-03099）：HSMFTIMERLEVEL 与 GULTIMERLEVEL 联动：HSMFTIMERLEVEL=SESSION 时 DED/DFT QoS Flow 定时器不生效，仅 SMF 会话空闲定时器生效；=QOSFLOW 时反之；GULTIMERLEVEL=BEARER 时 DFTBEARPOLICY/BEARERTIMER 生效，=SESSION 时 SESSIONTIMER 生效；I-SMF/V-SMF 下无辅锚点 UPF 时只支持会话级核查
- **取值范围与外部优先级**（info，rule-0-03100）：取值范围与外部优先级：BEARERTIMER/SESSIONTIMER/DEDQFIDLETIMER/SMFSESIDLETIMER 取值 5~12000（min），DFTQFIDLETIMER 取值 0 或 5~12000（0=不开启缺省 QoS Flow 核查），DFTBEARERTIMER 取值 0~255；当 [SET RDSAUTHRSPATTR](command/UNC/20.15.2/SET-RDSAUTHRSPATTR.md) 的 SESSIDLETIMESW=ENABLE 时，AAA 下发的最大会话在线时长和空闲上下文时长比本端配置（APNIDLETIME 或 DFTIDLETIME）优先级高
