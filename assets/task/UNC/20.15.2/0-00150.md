# SET APNIDLETIME 原子任务
> 命令静态知识见 [SET APNIDLETIME](command/UNC/20.15.2/SET-APNIDLETIME.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：SGW-C、PGW-C、SMF、GGSN。功能包括空闲上下文定时器（无数据传输超时删会话/承载/QoS Flow）和不活动上下文定时器。证据显示获取方法多值（本端规划 27 + 已配置数据中获取 3 + 固定取值 2），属 decision_driven。APN 粒度优先级 > 全局粒度（[SET DFTIDLETIME](command/UNC/20.15.2/SET-DFTIDLETIME.md)）。

## 配置方法
按"接入制式 + 回收粒度"分组建模：①4G/2G/3G 承载级（GULTIMERLEVEL=BEARER）②5G QoS Flow 级（HSMFTIMERLEVEL=QOSFLOW）③SMF 会话级。INHERIT=NO 表示不继承全局默认。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | APN 名称（本端规划/已配置） | `huawei.com` / `sos` / `ims` |
| INHERIT | 继承默认开关 | `NO`（按 APN 独立配置） |
| SCTXCHKSW | SGW-C 空闲上下文核查开关 | `ENABLE` |
| PCTXCHKSW | PGW-C/SGW-C&PGW-C 空闲核查开关 | `ENABLE` |
| GCTXCHKSW | GGSN 空闲核查开关 | `ENABLE` |
| HCTXCHKSW / ICTXCHKSW | H-SMF / I-SMF、V-SMF 空闲核查开关 | `ENABLE` |
| HCTXINACHKSW / ICTXINACHKSW | H-SMF / I-SMF 不活动上下文核查开关 | `ENABLE` |
| GULTIMERLEVEL | GUL 承载级别参数 | `BEARER`（承载级） / `SESSION`（会话级） |
| DFTBEARPOLICY | 缺省承载和默认 GBR 的定时器 | `ONEDAY` / `OFF` |
| BEARERTIMER | 承载定时器(min) | `60`（紧急呼叫默认）/ `5`（IMS） |
| SESSIONTIMER | 会话定时器时长(min) | `120` |
| HSMFTIMERLEVEL | H-SMF 空闲上下文核查级别 | `QOSFLOW`（固定取值） |
| DFTQFIDLETIMER | 缺省 QoS Flow 空闲定时器时长(min) | `1440` |
| DEDQFIDLETIMER | 专有 QoS Flow 空闲定时器时长(min) | `1440` / `600` / `5` |
| SMFSESIDLETIMER | SMF 会话空闲定时器时长(min) | — |
| SMFSESINATIMER | SMF 会话不活动定时器时长(min) | — |
| DFTBEARERTIMER | 缺省承载定时器时长(min) | `0~255` |
| IDLEUPDATEMSG | 空闲超时发送更新消息 | `ENABLE` |
| DFTBRPLCYGGSN | GGSN 一次激活上下文定时器 | `ONEDAY` |
| PROS8CTXCHKSW / PROXYSMFCHKSW | Proxy-SMF S8 / Proxy-SMF 空闲核查开关 | `ENABLE` |

典型脚本（5G QoS Flow 级，业务触发 QoS 保证）：
```
SET APNIDLETIME: APN="huawei.com", INHERIT=NO, HSMFTIMERLEVEL=QOSFLOW, DEDQFIDLETIMER=600;
```
（4G 紧急呼叫缺省承载回收）：
```
SET APNIDLETIME: APN="sos", INHERIT=NO, SCTXCHKSW=ENABLE, PCTXCHKSW=ENABLE, GCTXCHKSW=ENABLE, HCTXCHKSW=ENABLE, ICTXCHKSW=ENABLE, GULTIMERLEVEL=BEARER, DFTBEARPOLICY=ONEDAY, BEARERTIMER=60;
```

## 决策点

### 回收粒度选择（GULTIMERLEVEL / HSMFTIMERLEVEL）
- 问什么：按承载级、会话级还是 QoS Flow 级回收空闲上下文。
- 4G/2G/3G 承载级（紧急呼叫/IMS 语音）→ `GULTIMERLEVEL=BEARER`，配 BEARERTIMER。
- 5G QoS Flow 级（专有 QoS Flow 回收）→ `HSMFTIMERLEVEL=QOSFLOW`（固定取值），配 DEDQFIDLETIMER/DFTQFIDLETIMER。
- 会话级 → `GULTIMERLEVEL=SESSION`，配 SESSIONTIMER。
- 选错粒度会导致回收对象不匹配（如想回收专有 QoS Flow 却配了承载级）。

### 时长取值（BEARERTIMER / DEDQFIDLETIMER / SESSIONTIMER 等）
- 问什么：各定时器具体时长。
- 5~12000 分钟整数；DFTBEARERTIMER 0~255；DFTQFIDLETIMER 取值 0 或 5~12000。
- 设置过大长时间占用上下文资源；过小强制拆话/删 QoS Flow 后需重建，增加信令开销。以局点规划为准。
- 紧急呼叫 BEARERTIMER 不能为 0（防回呼场景缺省承载已被删），建议默认 60 分钟。

### DFTBEARPOLICY 缺省承载定时器策略
- 问什么：缺省承载和默认 GBR 的定时器如何生效。
- `ONEDAY`：紧急呼叫场景（保留缺省承载一天）。
- `OFF`：IMS 语音场景（呼叫结束后即按 BEARERTIMER 回收）。

## 约束
- 仅对新激活用户生效：不立即作用于在线用户。`severity=medium`
- SMF 会话定时器大小关系：SMFSESINATIMER 配置值须 ≤ SMFSESIDLETIMER，否则不活动先于空闲触发逻辑冲突。`severity=high`
- 不适用 NB-IoT：NB-IoT 终端长时间才交互，空闲上下文功能参考软参 BYTE801，本命令不适用。`severity=medium`
- HSMFTIMERLEVEL 固定取值：设 QoS Flow 级 SMF(N11) 上下文空闲定时时，必须用 QOSFLOW（固定取值）。`severity=medium`
- 会话级与承载级等价：GUL 承载级别参数指定为会话级时，对 4G 用户与"承载级且缺省承载/默认 GBR 定时器指定使用承载定时器"效果相同。`severity=low`
- APN 粒度优先级 > 全局粒度：APN 级配置覆盖 SET DFTIDLETIME 全局默认；APN INHERIT=YES 时继承全局。`severity=medium`
- 取值范围：定时器时长参数多为 5~12000 分钟整数，超出无效。`severity=medium`
