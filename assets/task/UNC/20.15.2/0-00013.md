# ADD CGGRPBINDING 原子任务

> 命令静态知识见 [ADD CGGRPBINDING](command/UNC/20.15.2/ADD-CGGRPBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CGGRPBINDING 把"一个离线计费模板"和"一个 CG 组"挂上，并可选地用 IMSI/MSISDN 号段组细化（按用户号段选不同的 CG 组发话单）。它是离线计费 CG 组链路的最后一步，位置在 `ADD CGGROUP` 建组、`ADD CGBINDING` 装 CG 之后。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| OFCTEMPLATENAME | 已建的离线计费模板名 | 已配置数据中获取（ADD GLBOFCTEMPLATE） | 离线计费模板，须已建 |
| CGGRPID | `1~32` | 已配置数据中获取（ADD CGGROUP） | 目标 CG 组，须已建 |
| SEGGROUPNAME | IMSI/MSISDN 号段组名 | 已配置数据中获取（ADD IMSIMSISDNSEG / ADD SUBSCRIBERIDSEGGRP） | 可选，按号段细分；不配则该 CG 组作为本模板的兜底 |
| IMSIPRIORITY | `1~65535` | 本端规划 | 可选，号段组优先级，值越小越高且全局唯一 |

**配置链路（本命令位置）**

```
ADD IMSIMSISDNSEG / ADD SUBSCRIBERIDSEGGRP   # （可选）建号段组
ADD GLBOFCTEMPLATE                            # 建离线计费模板
ADD CGGROUP → ADD CGBINDING                   # 建 CG 组并装 CG
ADD CGGRPBINDING     ← 本命令                # 把模板/号段组/CG 组三者绑在一起
```

## 决策点

### IMSIPRIORITY 号段组优先级语义

证据 notes 明确 IMSIPRIORITY "值越小优先级越高，优先级唯一"。一个模板下配多个号段组对应不同 CG 组时，按号段匹配顺序由此优先级决定。

| 场景 | 配法 | 含义 |
|---|---|---|
| 不细分号段（兜底 CG 组） | 不配 SEGGROUPNAME | 该 CG 组作本模板唯一/兜底出口 |
| 按号段细分 | 配 SEGGROUPNAME + IMSIPRIORITY | 多组共存时按优先级匹配号段选 CG |

> 一个模板下"未配号段组"的 CG 组最多只能配 1 个（见约束），其余必须带号段组。

## 约束

- **只对新激活用户生效** · severity=MEDIUM · 已激活用户仍走原绑定，增删改后需用户重激活才反映。
- **最大 2000 条记录** · severity=MEDIUM。
- **一个离线模板下最多 32 个 CG 组** · severity=MEDIUM · 超出无法继续绑定。
- **一个模板下未配号段组的 CG 组最多 1 个** · severity=HIGH · 想配多个 CG 组必须各自带 SEGGROUPNAME。
- **号段组匹配风险** · 引用了 IMSI/MSISDN 号段组，若号段未覆盖到，会在全局范围内选 CG 发话单（绕过号段匹配）· severity=MEDIUM · 规划号段时要全覆盖，否则可能话单走非预期 CG。
- **IMSIPRIORITY 全局唯一且值越小优先级越高** · severity=MEDIUM。
- **前置依赖**：OFCTEMPLATENAME 须已建（`ADD GLBOFCTEMPLATE`）；CGGRPID 须已建（`ADD CGGROUP`）；若用 SEGGROUPNAME 则号段组须已建 · severity=HIGH。
