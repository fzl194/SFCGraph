# 增加IPsec安全提议（ADD IPSECPROPOSALIPSEC）

> 命令静态知识见 [ADD IPSECPROPOSALIPSEC](command/UNC/20.15.2/ADD-IPSECPROPOSALIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令增加一条 IPSec 安全提议，为 IPSec 协商（阶段 2）提供算法套件。配置空间主要由**安全协议**（AH/ESP/AH-ESP）和**封装模式**两个维度决定。

### 配置维度1：安全协议（参数 IPSECPROTOCOL）

| 取值 | 作用 | 配套算法参数 | 典型场景 |
|---|---|---|---|
| Esp（ESP协议，默认） | 提供加密+认证 | ESPAUTHALGO + ESPENCRYPTALGO | 常规 IPSec（加密+认证） |
| Ah（AH协议） | 仅提供认证（不加密） | AHAUTHALGO | 仅需认证场景 |
| Ah_esp（AH-ESP） | AH+ESP 双协议 | AHAUTHALGO + ESPAUTHALGO + ESPENCRYPTALGO | 高安全场景 |

### 配置维度2：ESP 加密算法（参数 ESPENCRYPTALGO，仅 ESP/AH-ESP）

| 取值 | 作用 | 配套约束 |
|---|---|---|
| Aes_gcm_128（默认） | 128 位 AES-GCM | ESP 认证算法必须为空 |
| Aes_gcm_256 / Aes_gmac_128 | GCM/GMAC 系列 | ESP 认证算法必须为空 |
| Aes_128/Aes_192/Aes_256 | AES 系列（不安全） | ESP 认证算法必须不为空 |
| Sm4 | 国密 SM4 | 需先打开国密软参开关 |
| Des / A3des | DES/3DES（不安全） | ESP 认证算法必须不为空 |
| Null | 空加密 | — |

> 当 ESP 加密算法配置 AES_GCM 或 AES_GMAC 时，ESP 认证算法必须为空；反之认证算法必须不为空。

### 配置维度3：ESP/AH 认证算法（参数 ESPAUTHALGO / AHAUTHALGO）

| 取值 | 作用 |
|---|---|
| Sha2_256（推荐） | 推荐 SHA2-256 |
| Md5 / Sha1 | 不安全 |
| Sha2_384 / Sha2_512 | 高强度 SHA2 |
| Sm3 | 国密 SM3（需打开国密软参开关） |
| Null（空）/ Unconfigure（未配置） | 空（AH 不能为空，Unconfigure 时下发 Sha2_256） |

### 配置维度4：封装模式（参数 ENCAPMODE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| Tunnel（隧道模式，默认） | 封装整个原始 IP 包 | 隧道场景（典型） |
| Transport（传输模式） | 仅封装 payload | 传输场景 |

**步骤位置**：在所属 feature（IPFD-016000 IPSec功能）编排中，于 `ADD IPSECPOLICY`/`ADD IPSECPOLICY6` 之后、`ADD PROPATTACHIPSECPROPOSAL`（将本提议绑定到策略 sequence）之前执行。

## 决策点

**安全协议与算法组合选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| ESP + AES-GCM-128 | IPSECPROTOCOL=Esp, ESPENCRYPTALGO=Aes_gcm_128, ESPAUTHALGO=Null | 推荐 IPv4/IPv6 IPSec |
| ESP + 国密 SM4/SM3 | IPSECPROTOCOL=Esp, ESPENCRYPTALGO=Sm4, ESPAUTHALGO=Sm3；需先 SET FWSOFTPARA 打开国密软参 | 国密 IPsec 支持 IKEv1（IPSec 提议唯一配置） |
| AH + SHA2-256 | IPSECPROTOCOL=Ah, AHAUTHALGO=Sha2_256 | 仅认证场景 |

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无延迟
- **部分算法不安全**（warning）：部分参数含不安全算法（Md5/Sha1/DES/3DES/AES_128/192/256），影响加解密安全性 — 后果：安全性弱
- **GCM/GMAC 下 ESP 认证必须空**（critical）：ESP 加密算法配 AES_GCM/AES_GMAC 时，ESP 认证算法必须为空；反之必须不为空 — 后果：配置失败
- **GCM/GMAC 不能用软参配**（critical）：ESP 加密算法配 AES_GCM/AES_GMAC 时不能用软参配置算法 — 后果：配置无效
- **国密算法需软参开关**（critical）：国密 Sm3/Sm4 需要先执行 `SET FWSOFTPARA:PARAMETERSTYPE=DWORD,DWORDINDEX=1401,DWORDVALUE=1;` — 后果：未打开无法使用国密算法
- **IKEv1 ESP 加密限制**（warning）：以下 ESP 加密算法不支持 IKEv1：AES-GCM-128/256、AES-GMAC-128、NULL — 后果：IKEv1 下配置无效
- **IKEv1 国密唯一配置**（critical）：IKEv1 国密 IPSec 提议仅支持 ESP+SM3+SM4+隧道模式 — 后果：配置不符会协商失败
- **AH 认证不能为空**（warning）：AH 认证算法不能为空；选 Unconfigure 时下发 Sha2_256 — 后果：实际下发的算法可能与预期不符
- **提议名 80~91 可配不安全算法**（info）：proposal80~91 可通过软参配置 MD5/DES/3DES — 后果：仅这几个名能配旧算法
- **老旧 V3 单板不支持国密加速**（warning）：老旧 V3 单板不支持开启国密算法加速功能 — 后果：国密加速不可用
- **最多 100 条记录**（warning）：本命令最多可输入 100 条记录 — 后果：超规格新增失败
