# RMV DIAMLOCINFO 原子任务
> 命令静态知识见 [RMV DIAMLOCINFO](command/UNC/20.15.2/RMV-DIAMLOCINFO.md)。本页只讲配置生成实例化时这条命令**怎么配**。

**高危命令**。删除对象是 `DIAMLOCINFO`（Diameter 本端信息，即 Gx/IP-CAN Session 接口的设备标识）。支持批量删除（不给 HOSTNAME 即删全部）；删除在用标识对已建立 IP-CAN Session 影响较大，且会导致所有 Group 下使用该 LocalInfo 的链路断开。

## 配置方法
按本端主机名定位删除；不给 HOSTNAME = 批量删全部。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| HOSTNAME | 本端主机名（local_planned，1～127 字符，不含空格） | `gx-local-1` |

典型脚本（精确删一个本端标识）：
```
RMV DIAMLOCINFO: HOSTNAME=gx-local-1;
```

## 决策点（若有）

### 删除范围（单条 vs 批量）
- 问什么：删一个本端标识，还是删全部。
- 精确删：给 `HOSTNAME` 删指定一条。
- 批量删：不给 `HOSTNAME` = 删全部本端信息——高危，禁止在生产裸执行（需软参 BYTE976=169 才允许）。

### 在用标识删除影响
- 问什么：被删标识是否在用。
- 删在用的 Gx/IP-CAN Session 标识 → 对已建立的 IP-CAN Session 影响较大。
- 删除后 → 所有 Group 下使用该 LocalInfo 的链路断开（已建立会话状态不受影响，但新会话无链路）。
- 删前应确认无在用会话或可接受影响。

## 约束（若有）
- 高危：删除本端信息可能导致 Diameter 链路中断、用户被去活，删前必须确认影响。`severity=high`
- 批量删禁止：不给 HOSTNAME 会删全部，生产禁止裸执行（需 BYTE976=169）。`severity=high`
- HOSTNAME 前置：未指定本端主机名时禁止执行（除非 BYTE976=169）。`severity=high`
- 链路断开级联：删除后所有 Group 下使用该 LocalInfo 的链路断开。`severity=high`
- 生效时机：立即生效。`severity=medium`
- 配套重建：调测场景标准动作 = RMV DIAMLOCINFO（删原配置）→ ADD DIAMLOCINFO（按规划重建），常配合 RMV/ADD DIAMCONNECTION 调 Gx 口设备标识。`severity=medium`
