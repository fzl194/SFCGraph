# 增加PCC策略组（ADD PCCPOLICYGRP）

> 命令静态知识见 [ADD PCCPOLICYGRP](command/UNC/20.15.2/ADD-PCCPOLICYGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

PCC 策略组是**计费属性 + QoS 属性 + FUP 累计标识**的聚合容器，被 RULE 以 `POLICYTYPE=PCC, POLICYNAME=<本组名>` 引用。所有属性参数可选，全不选即为空策略组。配置空间有 4 个维度：**策略组名**（必选，本端规划唯一）、**计费属性 URR 组**（可选，引用 URRGROUP）、**QoS 属性**（可选，引用 QOSPROP）、**FUP 累计标识**（可选，控制是否计入 Session 级流量）。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 策略组名 | `PCCPOLICYGRPNM` | 本端规划，1~63 字符，不支持空格、不区分大小写 | 无 | PCC 策略组唯一标识，被 RULE.POLICYNAME 引用 | 所有记录必填 |
| 计费属性 | `URRGROUPNAME` | 本端规划，1~31 字符，不支持空格、不区分大小写；引用 `ADD URRGROUP` 已配置 URR 组 | 无 | 承载计费策略（离线/在线/融合/事件） | 内容计费、融合计费、FUP 累计 |
| QoS 属性 | `QOSPROPNAME` | 本端规划，1~31 字符，不支持空格、不区分大小写；引用 `ADD QOSPROP` 已配置 QoS 属性 | 无 | 承载 QoS 保证策略 | 业务触发 QoS、语音 Bypass |
| FUP 累计标识 | `FUPSESSIONEXC` | 枚举：`DISABLE`（计入 Session 级流量）/ `ENABLE`（不计入 Session 级流量） | DISABLE | 控制该策略组业务流量是否累计到 Session 级 FUP | 免流业务、FUP 排除场景 |

> 一个 PccPolicyGrp 最多可配置 **10 个**（URRGroupName, FupSessionExc）组合：1 个默认组合（不含 ServiceProp）+ 9 个非默认组合（以 ServiceProp 标识，通过 `ADD SRVPBINDPCCPG（★R1.5 atom 待补）` 添加）。多组合时匹配优先级取决于 ServiceProp 优先级，默认组合优先级最低。

典型脚本（内容计费：策略组 + URR 组）：
```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="cg-test",URRGROUPNAME="urrgroup1";
```

典型脚本（QoS 保证：策略组 + QoS 属性）：
```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicy_group",QOSPROPNAME="qosprop_501";
```

典型脚本（FUP 累计排除：策略组 + URR 组 + 不计入 Session 级）：
```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicygrp_01",URRGROUPNAME="urrgrp_01",FUPSESSIONEXC=ENABLE;
```

典型脚本（计费 + QoS 叠加，如语音 Bypass）：
```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicygrp_voice",URRGROUPNAME="urrg_voice_01",QOSPROPNAME="qosprop_voice";
```

**步骤位置**：建 URR 组（`ADD URRGROUP`）→ 建 QoS 属性（`ADD QOSPROP`）→ 本任务（组装策略组）→ `ADD RULE`（POLICYTYPE=PCC 引用本组）。

## 决策点

### 决策点：策略组装哪些属性（DP 0-02083）

问：本 PCC 策略组承载哪类业务属性？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅计费 | `PCCPOLICYGRPNM` + `URRGROUPNAME` | 内容计费、融合计费、事件计费、在线计费（证据主流） |
| 仅 QoS | `PCCPOLICYGRPNM` + `QOSPROPNAME` | 业务触发 QoS 保证，规则通过 PCC 策组间接绑 QoS |
| 计费 + QoS 叠加 | `PCCPOLICYGRPNM` + `URRGROUPNAME` + `QOSPROPNAME` | 语音 Bypass 等需同时承载计费与 QoS 的场景 |
| FUP 累计排除 | `PCCPOLICYGRPNM` + `URRGROUPNAME` + `FUPSESSIONEXC=ENABLE` | 基于业务累计流量的策略控制，特定业务不计入 Session 级 |
| 空策略组 | 仅 `PCCPOLICYGRPNM` | 占位或仅做引用锚点 |

> 规则直接绑 QoS（RULE.POLICYTYPE=QOS）时跳过本步；仅当 RULE.POLICYTYPE=PCC 时才用本组间接承载属性。

### 决策点：URR 组合与 ServiceProp 扩展（DP 0-02084）

问：本策略组是否需要多套（URRGroupName, FupSessionExc）组合按业务属性分流？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单默认组合 | `URRGROUPNAME` + `FUPSESSIONEXC`（不含 ServiceProp） | 全部业务统一计费/FUP 策略 |
| 默认 + 非默认多组合 | 默认组合 + 通过 `ADD SRVPBINDPCCPG（★R1.5 atom 待补）` 添加最多 9 个非默认组合（以 ServiceProp 标识） | 按业务属性（如 APN/ Rating Group）差异化计费策略 |
| 无默认仅非默认 | 不配 URRGROUPNAME，仅通过 `ADD SRVPBINDPCCPG（★R1.5 atom 待补）` 添加最多 10 个非默认组合 | 全部按 ServiceProp 匹配，无兜底 |

> 多组合时匹配优先级取决于 SRVPBINDPCCPG 下 ServiceProp 优先级，默认组合优先级最低；被绑定的 PccPolicyGrp 不允许删除，须先解除 RULE 绑定。

## 约束

- **唯一性**（critical，rule-0-02206）：唯一性：PCCPOLICYGRPNM 在多条记录中不能重复，否则 RULE.POLICYNAME 引用歧义 — 后果：策略组引用混乱，计费/QoS 策略错配
- **联动一致性**（warning，rule-0-02207）：联动一致性：RULE 的 `POLICYTYPE=PCC` 时 `POLICYNAME` 必须等于本命令配的 PCCPOLICYGRPNM；PCC 类型的 RULENAME 不能与 QOS 类型的 RULENAME 相同 — 后果：规则引用断链
- **组合规格**（warning，rule-0-02208）：组合规格：一个 PccPolicyGrp 最多 10 个（URRGroupName, FupSessionExc）组合（1 默认 + 9 非默认，或 10 非默认） — 后果：超规格新增失败
- **长度规格**（warning，rule-0-02209）：长度规格：URRGROUPNAME 长度 1~31、QOSPROPNAME 长度 1~31、PCCPOLICYGRPNM 长度 1~63（均不支持空格、不区分大小写）；最大记录数 40000 — 后果：超规格或非法字符新增失败
- **生效时机**（info，rule-0-02210）：生效时机：执行后立即生效；适用 NF：PGW-C、SMF — 后果：无延迟
