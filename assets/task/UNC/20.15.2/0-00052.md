# ADD PCCPOLICYGRP 原子任务
> 命令静态知识见 [ADD PCCPOLICYGRP](command/UNC/20.15.2/ADD-PCCPOLICYGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
PCC 策略组是**计费属性 + QoS 属性 + FUP 累计标识**的聚合容器，被 RULE 以 `POLICYTYPE=PCC, POLICYNAME=<本组名>` 引用。按业务用途决定装哪类属性：计费装 URRGROUPNAME，QoS 保证装 QOSPROPNAME，FUP/累计流量装 URRGROUPNAME+FUPSESSIONEXC。所有参数可选，全不选即为空策略组。

| 业务用途 | 必备参数 | 典型脚本 |
|---|---|---|
| 内容计费（离线/在线/融合/事件） | `PCCPOLICYGRPNM` + `URRGROUPNAME`（已配 URRGROUP） | `ADD PCCPOLICYGRP:PCCPOLICYGRPNM="cg-test",URRGROUPNAME="urrgroup1";` |
| QoS 保证（业务触发） | `PCCPOLICYGRPNM` + `QOSPROPNAME`（已配 QOSPROP） | `ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicy_group",QOSPROPNAME="qosprop_501";` |
| FUP / 累计流量策略控制 | `PCCPOLICYGRPNM` + `URRGROUPNAME` + `FUPSESSIONEXC=ENABLE` | `ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicygrp_01",URRGROUPNAME="urrgrp_01",FUPSESSIONEXC=ENABLE;` |
| 仅 QoS 类型（本地 PCC） | `PCCPOLICYGRPNM` + `QOSPROPNAME` | `ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicy_group",QOSPROPNAME="qos_property1";` |
| 语音 Bypass（信令+语音双组） | 每个会话角色一条：信令组装 URRGROUPNAME，语音组装 URRGROUPNAME+QOSPROPNAME | `ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicygrp_signaling",URRGROUPNAME="urrg_signaling_01";` + `ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicygrp_voice",URRGROUPNAME="urrg_voice_01",QOSPROPNAME="qosprop_voice";` |

**FUPSESSIONEXC 语义**：`ENABLE`=该 PCC 策组对应业务**不计入** Session 级流量（默认 `DISABLE` 计入）。用于把特定业务（如免流）排除在 FUP 累计之外。

## 决策点

### PCC 策略组装哪些属性
问：本 PCC 策略组承载哪类业务属性？
- **计费**：装 URRGROUPNAME（引用 ADD URRGROUP 创建的 URR 组名）。证据中内容计费/融合计费/事件计费/在线计费普遍采用。
- **QoS 保证**：装 QOSPROPNAME（引用 ADD QOSPROP 创建的 QoS 属性名）。规则通过 PCC 策组间接绑 QoS 时用；规则直接绑 QoS（POLICYTYPE=QOS）时跳过本步。
- **FUP 累计**：URRGROUPNAME + FUPSESSIONEXC 组合，用于基于业务累计流量的策略控制。
- **可叠加**：语音 Bypass 场景同一策略组可同时装 URRGROUPNAME + QOSPROPNAME。

### URRGROUPNAME 值来源
问：URR 组名取已配数据还是本端新规划？
- 取**已配置数据中获取**：引用 [0-00093](task/UNC/20.15.2/0-00093.md)（ADD URRGROUP）定义的 URRGROUPNAME。证据中绝大多数场景如此。
- 多业务套餐需多个策略组：每个费率/业务套餐一条 PCCPOLICYGRP，各自引用不同 URR 组（如 URL/IMS/any 三组）。

## 约束
- PCCPOLICYGRPNM 存在多条记录时不能重复。`severity=critical`
- PCCPOLICYGRPNM 长度 1~63，不支持空格，不区分大小写。`severity=medium`
- URRGROUPNAME 长度 1~31，QOSPROPNAME 长度 1~31。`severity=medium`
- 一个 PccPolicyGrp 最多可配置 **10 个** (URRGroupName, FupSessionExc) 组合。`severity=medium`
- 最大记录数 40000。`severity=medium`
- 与 RULE 联动：RULE 的 `POLICYTYPE=PCC` 时 `POLICYNAME` 必须等于本命令配的 PCCPOLICYGRPNM；PCC 类型的 RULENAME 不能与 QOS 类型的 RULENAME 相同。`severity=high`
- 执行后立即生效。`severity=low`
