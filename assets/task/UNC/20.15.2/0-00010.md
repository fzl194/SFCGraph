# ADD CDRFIELDTEMP 原子任务

> 命令静态知识见 [ADD CDRFIELDTEMP](command/UNC/20.15.2/ADD-CDRFIELDTEMP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

话单字段模板是一个"字段开关大表"：给模板起名，然后把需要的字段设 `ENABLE`、不需要的设 `DISABLE`。绝大多数字段默认 `ENABLE`，少数（IMSIUNAUTHFLAG/PDNCONNECTIONID/PGWADDRESSUSED/PGWPLMNID/RECORDEXTENTION/STARTTIME/STOPTIME 等）默认 `DISABLE`。实际配置时**只显式改与默认不同的字段**，其余走默认即可。模板建好后通常还要经 `SET GLBCDRFLDTEMP`（全局模板）或 `SET OFCCDRPARA` 落到话单生成路径。

**典型配置范式（按特性分组，证据归纳）**

| 特性场景 | 显式开关的字段 | 典型行 |
|---|---|---|
| 用户属性反射 | RATTYPE/ULI/MSTIMEZONE = ENABLE | `ADD CDRFIELDTEMP:TEMPLATENAME="cdr-field-test",RATTYPE=ENABLE,ULI=ENABLE,MSTIMEZONE=ENABLE;` |
| 信令面数据传输（SGW-C） | CPEPSOINDICATOR/UNIPDUCPONLY = ENABLE | `ADD CDRFIELDTEMP:TEMPLATENAME="cdrfieldtemp",CPEPSOINDICATOR=ENABLE,UNIPDUCPONLY=ENABLE;` |
| Non-IP 数据传输 | PDPTYPEEXT/SGIPTPTUNMETH = ENABLE | `ADD CDRFIELDTEMP:TEMPLATENAME="cdrfieldtemp",PDPTYPEEXT=ENABLE,SGIPTPTUNMETH=ENABLE;` |
| 离线计费（关默认字段） | APNNETWORKID/MSTIMEZONE = DISABLE | `ADD CDRFIELDTEMP:TEMPLATENAME="cdrfield-test",APNNETWORKID=DISABLE,MSTIMEZONE=DISABLE,ULI=ENABLE,RECORDEXTENTION=DISABLE;` |
| eMTC/NB-IoT 速率控制 | APNRATECTRL / SRVPLMNRATECTRL = ENABLE | （配合 `SET OFCCDRPARA` 在 PGW-CDR 携带） |
| NSA 用户数据话单 | SECRATUSAGE = ENABLE | 使话单支持携带 RAN-SecondaryRAT-Usage-Report |

**字段语义分类（便于按需开关）**

- **用户/接入标识**：MSISDN、SERVEDIMEISV、APNNETWORKID、APNSELECTMODE、PDPTYPE、SGSNSGWPLMNID、SERVEDPDPADDR、DYNADDRFLAG
- **位置/接入类型**：ULI、RATTYPE、MSTIMEZONE、THREEGPP2USRLOC、UWANULI
- **PGW/节点信息**：NODEID、PGWADDRESSUSED、PGWADDRESSIPV6、PGWPLMNID、SGWADDRESSIPV6
- **序列/时间**：RECSEQNUMBER、LOCALRECSEQNUM、STARTTIME、STOPTIME、HPTIME、PDNCONNECTIONID
- **流量/业务**：LOTV、LOSD、IMSSIGCONTEXT、PSFCI、DIAGNOSTICS、NETINITPDPCONT
- **扩展/特殊**：RECORDEXTENTION+RECORDEXTTYPE、CCBURL/CCBUSGDURATION、DTRECORD、PACKETNUM
- **CIoT/NB-IoT 专用**：CPEPSOINDICATOR、UNIPDUCPONLY、PDPTYPEEXT、SGIPTPTUNMETH、APNRATECTRL、SRVPLMNRATECTRL、LOWPRIIND、MOEXCPTDATA
- **5G/NSA**：SNSSAI、SSCMODE、SECRATUSAGE

## 决策点

### RATTYPE/ULI/MSTIMEZONE 等"双端支持"字段是否开启

证据中这些字段的"获取方法"为 **与对端协商**：只有 SGSN/S-GW 和 GGSN/PGW-C/SMF **都支持**该字段时，在模板里开启才生效，否则配置无效。

| 选项 | 取值 | 前提 |
|---|---|---|
| 双端都支持，话单需要 | ENABLE | 已与对端确认支持该字段 |
| 对端不支持或不需要 | DISABLE | 避免无效配置 |

### RECORDEXTTYPE 与运营商扩展

当 `RECORDEXTENTION=ENABLE` 时，RECORDEXTTYPE（默认 CCB）决定扩展字段内容来源（GLB/DT 等），影响 PGW 话单扩展字段携带什么。

## 约束

- **立即生效** · severity=LOW。
- **最大 100 条记录** · severity=MEDIUM。
- **EXTCHARGEID/THREEGPP2USRLOC/ULIWLAN 当前版本不支持**：配置无效 · severity=HIGH。
- **双端支持才生效**：RATTYPE/ULI/MSTIMEZONE 等需 SGSN/S-GW 与 GGSN/PGW-C/SMF 双端支持，否则开了也不携带 · severity=HIGH · 易被误判为配置失效。
- **PACKETNUM=ENABLE 需先配费率**：仅当配置了费率时 PGW 话单扩展字段才携带 IP 层报文包数 · severity=MEDIUM。
- **RECORDEXTTYPE=GLB/DT 时 PacketNum 不受控**：此时 PGW 扩展字段是否携带 IP 层报文包数不受 PacketNum 控制 · severity=MEDIUM。
- **字段名大小写注意**：模板内参数名以命令真相表为准（如 CCSELETIONMODE，非 CCSelectionMode）· severity=LOW。
