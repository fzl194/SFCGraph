# 增加IPsec隧道接口（绑定策略）（ADD IPSECINTFCFGIPSEC）

> 命令静态知识见 [ADD IPSECINTFCFGIPSEC](command/UNC/20.15.2/ADD-IPSECINTFCFGIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令增加一个 IPsec 隧道接口，并将一个安全策略（由 `ADD IPSECPOLICY`/`ADD IPSECPOLICY6` 建立）绑定到该接口；可选指定源接口地址借用（SRCIFNAME）。配置空间主要由**隧道协议类型**和**是否地址借用**两个维度决定。

### 配置维度1：隧道协议类型（参数 TNLTYPE）

| 取值 | 作用 | 配套策略 | 典型场景 |
|---|---|---|---|
| IPSEC | IPv4 IP 安全隧道 | POLICYNAME 引用 `ADD IPSECPOLICY` 建立的策略 | 普通 IPv4 IPSec 隧道 |
| IPSEC6 | IPv6 IP 安全隧道 | POLICYNAME 引用 `ADD IPSECPOLICY6` 建立的策略 | IPv6 IPSec 隧道 |

### 配置维度2：地址借用（参数 SRCIFNAME）

| 取值 | 作用 | 约束 |
|---|---|---|
| 省略 | 不借用地址，接口用自身 IP | — |
| LoopBack 接口名（如 LoopBack1） | 借用指定环回口的 IP 作为本接口地址 | 只支持环回口 |

### 配置维度3：策略绑定（参数 POLICYNAME）

| 取值 | 作用 | 约束 |
|---|---|---|
| 真实有效的策略名 | 将该接口与策略绑定，生成 IPsec 隧道 | 必须是 `ADD IPSECPOLICY`/`ADD IPSECPOLICY6` 已建策略；空策略不生成隧道 |

> 若配置了空的安全策略，不会进行记录；只有配置真实有效的安全策略才会生成 IPSec 隧道接口。

**步骤位置**：在所属 feature（IPFD-016000 IPSec功能）编排中，于 `ADD IPSECPOLICY`/`ADD IPSECPOLICY6`（建策略）之后执行；是 IPSec 隧道建立的最后一步（接口 + 策略绑定）。

## 决策点

**隧道协议类型与地址借用选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPv4 + 不借用 | TNLTYPE=IPSEC, POLICYNAME | 普通 IPv4 IPSec 隧道 |
| IPv4 + 借用环回口 | TNLTYPE=IPSEC, POLICYNAME, SRCIFNAME=LoopBackX | 需要地址借用场景 |
| IPv6 + 不借用/借用 | TNLTYPE=IPSEC6, POLICYNAME, [SRCIFNAME] | IPv6 IPSec 隧道 |

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无延迟
- **空策略不生成隧道**（critical）：配置空的安全策略不会进行记录；只有真实有效的策略才生成 IPSec 隧道接口 — 后果：策略名必须真实存在
- **POLICYNAME 必须与策略命令一致**（critical）：POLICYNAME 必须与 `ADD IPSECPOLICY`/`ADD IPSECPOLICY6` 配置的策略名一致 — 后果：绑定失败
- **SRCIFNAME 只支持环回口**（warning）：SRCIFNAME 只支持配置环回口（LoopBack） — 后果：非环回口配置失败
- **最多 512 条记录**（warning）：本命令最多可输入 512 条记录 — 后果：超规格新增失败
