# 增加接口IPv4地址（ADD IFIPV4ADDRESS）

> 命令静态知识见 [ADD IFIPV4ADDRESS](command/UNC/20.15.2/ADD-IFIPV4ADDRESS.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令用于配置接口的 IPv4 地址，接口可以是物理接口（真实存在，如 Ethernet66/0/5）或逻辑接口（需通过配置建立，如 LoopBack0/1、Tunnel1/2/10/17）。配置空间核心维度为 **接口类型**（决定地址用于 LoopBack/Tunnel/物理口）、**地址类型**（ADDRTYPE：主地址 main / 子地址）。

### 配置维度1：接口类型（参数 IFNAME，承载接口实例名）

| 接口类型 | 作用 | 典型掩码 | 典型场景 |
|---|---|---|---|
| LoopBack0/1 | 配置本端 LoopBack 接口 IPv4（业务/管理/对接地址） | 255.255.255.255（32 位主机） | RADIUS NEXTHOP、IPSec 本端、GRE 源（IPFD-015002 LoopBack1=10.3.3.11、IPFD-016000 LoopBack1=10.3.3.3、WSFD-011306 LoopBack0=192.168.100.105） |
| Tunnel1/2/10/17 | 配置隧道接口 IPv4（GRE/IPSec 隧道对端可达地址） | 255.255.255.0 或 255.255.255.255 | GRE 隧道（IPFD-015002 Tunnel1=10.10.1.201）、IPSec 隧道（IPFD-016000 Tunnel10/2/17） |
| Ethernet66/0/x | 配置物理接口 IPv4（与对端物理对接） | 255.255.255.0 | IPSec 物理出接口（IPFD-016000 Ethernet66/0/5=10.1.1.1、Ethernet66/0/6=172.16.163.1） |

### 配置维度2：地址类型（参数 ADDRTYPE，可选）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| main（显式声明） | 主地址 | 与接口绑定主要业务地址（多数特性显式 ADDRTYPE=main） |
| 子地址（缺省时按系统规则） | 接口次地址/别名 | 同接口配多地址时除 main 外的其他地址 |

**步骤位置**：在地址分配/接口管理族（GRE/IPSec/RADIUS）编排中，于 `ADD INTERFACE`（建逻辑接口）之后、依赖该 IP 的命令（`ADD IPBINDVPN`、`ADD APNRDSCLIENTIP`、`ADD CPGTPUADDR`、`ADD ATTACHIKEPEER` 等）之前执行。

## 决策点

**接口与地址类型选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| LoopBack + main | IFNAME=LoopBackx, ADDRTYPE=main | 业务/对接固定地址（GRE 源、IPSec 本端、RADIUS NEXTHOP） |
| Tunnel + main/缺省 | IFNAME=Tunnelx | GRE/IPSec 隧道接口地址 |
| Ethernet + main/缺省 | IFNAME=Ethernetxx/x/x | 物理接口主地址 |

> 实战中 IFNAME 已规划好的接口名直接复用；ADDRTYPE 在主从地址场景下显式区分，单地址场景可不填（系统默认 main）。

## 约束

- **接口必须先建**（critical）：IFNAME 必须是 `ADD INTERFACE` 已配置的接口（逻辑接口需先建） — 后果：接口不存在则配置失败
- **IFIPADDR 合法性**（critical）：IPv4 地址范围 0.0.0.0~255.255.255.255，需为合法 A/B/C 类地址，掩码合法 — 后果：非法地址新增失败
- **IFNAME 命名约束**（warning）：字符串 1~63，不支持空格及部分特殊字符 — 后果：违规命名配置失败
- **主地址唯一性**（warning）：同一接口 main 地址唯一 — 后果：重复配置 main 失败
