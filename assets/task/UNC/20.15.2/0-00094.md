# ADD USERPROFILE 原子任务
> 命令静态知识见 [ADD USERPROFILE](command/UNC/20.15.2/ADD-USERPROFILE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
User Profile 是策略/计费绑定的载体。`USERPROFILENAME` 全网规划（PCF/SMF/UPF 间名称必须一致，否则 PCF 下发策略失效）；`UPTYPE` 决定业务类别。配置后由后续命令往这个模板上挂 RULE（[ADD RULEBINDING](task/UNC/20.15.2/0-00073.md)）/ URR 组（[0-00206](task/UNC/20.15.2/0-00206.md) SET URRGRPBINDING）/ 计费属性（[0-00207](task/UNC/20.15.2/0-00207.md) SET USRPROFCHARGE）/ DNAI（[ADD USRPROBINDDNAI](task/UNC/20.15.2/0-00095.md)）。

| 业务类别（UPTYPE） | 必备/常用参数 | 典型脚本 |
|---|---|---|
| PCC（计费/QoS/ADC/Radius 等通用） | `USERPROFILENAME` + `UPTYPE=PCC`（+ 可选 `PROFILERANGE`） | `ADD USERPROFILE:USERPROFILENAME="up_test",UPTYPE=PCC,PROFILERANGE=ALL;` |
| ULCL（分流策略控制） | `USERPROFILENAME` + `UPTYPE=ULCL` | `ADD USERPROFILE:USERPROFILENAME="testuserprofilename", UPTYPE=ULCL;` |
| 免费业务计费控制 | `USERPROFILENAME` + `UPTYPE=PCC` + `FREESERONLCHG/FREESEROFFCHG/FREESERCVGCHG`（ENABLE/DISABLE） | 默认 ENABLE；需在话单/配额上报观察免费业务计费信息时显式使能 |

**PROFILERANGE 取值**：`ALL`（同时下发主+辅锚点 UPF，默认）/ `CENTRAL`（仅主锚点）/ `LOCAL`（仅辅锚点）。SMF/PGW-C 将 UserProfile 下发到哪些 UPF 由它决定。

**UPSPECTYPE 取值**：`DEFAULT`（规格 5000）/ `SPECIFICATION_LIMITED`（规格 100000，超出 98% 上报告警）。

## 决策点

### UPTYPE 业务类别选择（跨场景枚举）
问：该 UserProfile 承载哪类业务？
- `PCC`：通用计费/QoS/ADC/Radius 策略承载（证据中绝大多数场景）。
- `ULCL`：ULCL 分流策略承载（[WSFD-108002](evidence/UNC/20.15.2/WSFD-108002-基于预定义规则的分流策略控制参考信息_28860592.md) / 223001 / 223003 / 223101）。

### PROFILERANGE 下发范围选择（conditional 分叉）
问：UserProfile 下发给哪些 UPF？
- `ALL`：主+辅锚点都下发（默认）。
- `CENTRAL`：仅主锚点 UPF。
- `LOCAL`：仅辅锚点 UPF——只有此场景才需 [ADD USRPROBINDDNAI](task/UNC/20.15.2/0-00095.md) 绑定边缘 UPF 的 DNAI。
- 条件：与上游 [ADD USERPROFILE](task/UNC/20.15.2/0-00094.md) 配置一致；CENTRAL 时不能绑 DNAI。

### 免费业务计费开关（获取方法多值）
问：免费业务是否产出在线/离线/融合计费信息？
- `FREESERONLCHG` / `FREESEROFFCHG` / `FREESERCVGCHG` 默认 `ENABLE`。
- 想在 CG 离线话单或在线配额上报消息中观察到免费业务计费信息，必须在所在 UserProfile 下显式使能 offline 或 online（系统缺省已使能）。

### 跨网元命名一致性
问：USERPROFILENAME 如何取值？
- 全网规划，PCF/SMF/UPF 间名称必须一致；不一致时 PCF 下发的策略无法生效（[WSFD-011206 配置融合计费费率标识](evidence/UNC/20.15.2/配置融合计费费率标识_93360308.md)）。
- 个别场景（如 211009 累计流量策略控制）取值与 UPF 协商一致。

## 约束
- `USERPROFILENAME` 全网唯一、跨 PCF/SMF/UPF 名称一致。`severity=critical` 后果：PCF 下发策略失效、跨网元不一致。
- `USERPROFILENAME` 字符串 1~63，不支持空格，不区分大小写。
- `UPTYPE` 必选、固定取值（PCC/ULCL 等）。`severity=high`
- 记录数上限：`UPSPECTYPE=DEFAULT` 时 5000；`=SPECIFICATION_LIMITED` 时 100000；>98% 触发 ALM-135602215 告警，≤95% 恢复。`severity=medium`
- `PROFILERANGE=CENTRAL` 时该模板不能绑定任何边缘 UPF 的 DNAI。`severity=high`
- 执行后立即生效。`severity=low`
