# DHCP 功能（WSFD-104413）

> 特性静态知识见 [WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md+ [特性知识](业务图谱体系/APN业务域/feature-knowledge/WSFD-104413-DHCP功能.md（★业务图谱未建，归 R1.5 后续）。本页讲配置生成实例化时这个特性**怎么配**——DHCPv4 子方式下的 DHCP 服务器组链路 + 地址池 + APN 维度使能。

## 配置概览

核心对象链：**DHCPSERVERGRP（DHCP 服务器组）** → **DHCPSERVER（单台 DHCP 服务器 IP）** → **DHCPBINDPOOLGRP（DHCP 服务器组绑地址池组）** + **ADDRPOOL（POOLTYPE=DHCP）** + **AGENTIP（UNC 代理 IP）** → **SET APNADDRESSATTR（IPV4ALLOCTYPE=DHCP）** + **SET DHCPPARAREQ（DNS/P-CSCF 请求信元开关）** + **SET IPALLOCRULE（地址分配规则）**。

本特性是 [WSFD-010502](feature/UNC/20.15.2/WSFD-010502.md) 地址分配方式中 **DHCP 子方式**的完整展开——含 DHCP 服务器组链路（DHCP 服务器组 + 服务器 + 地址池组绑定）+ UNC 代理 IP + APN 维度使能。由特性级 DP（DNS/P-CSCF 请求维度）驱动走哪些步骤。

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（L3VPN + VPN 实例 + APN + 使能 DHCP 分配）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR`（`IPV4ALLOCTYPE=DHCP`） → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **SMF 地址池（DHCP 子方式）**（建地址池+段+池组，POOLTYPE=DHCP）：`ADD ADDRPOOL`（POOLTYPE=DHCP）+ `ADD SECTION` + `ADD ADDRPOOLGRP` + `ADD POOLBINDGRP` + `ADD ADDRUPGROUP` → [UNC 1-00023](task/UNC/20.15.2/1-00023.md)
3. **UNC 代理 IP**（DHCP 客户端在地址池网段内的代理 IP）：`ADD AGENTIP`
   > **★ R1.5 待补 atom**：`ADD AGENTIP` 命令真实存在 UNC OM（路径：`业务服务管理/5G接入业务管理/控制面管理/会话管理/DHCP管理/`），evidence = [WSFD-104413-参考信息](evidence/UNC/20.15.2/WSFD-104413/) 命令清单；UNC pre-build 阶段 atom 未建，归 P1 后续补建。**AGENTIP 必须在地址段内**（critical 约束）。
4. **DHCP 服务器组链路**（DHCP 子方式专属）：
   - `ADD DHCPSERVERGRP`
   - `ADD DHCPSERVER`
   - `ADD DHCPBINDPOOLGRP`
   > **★ R1.5 待补 atom**（3 条命令）：evidence 同上，UNC pre-build 阶段未建，归 P1 后续补建。
5. **地址分配规则**（按需，全局开关）：`SET IPALLOCRULE`
   > **★ R1.5 待补 atom**：evidence = [WSFD-010504-参考信息_77813716.md](evidence/UNC/20.15.2/WSFD-010504/WSFD-010504-参考信息_77813716.md) 命令清单 + ALM-100297 告警；UNC pre-build 阶段未建。
6. **（按需）DHCP 请求信元参数**（DNS/P-CSCF 请求选项）：`SET DHCPPARAREQ`
   > **★ R1.5 待补 atom**：evidence 同步骤 4；UNC pre-build 阶段未建。
7. **下行路由发布**（让骨干网可达分配的地址）：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFINTERFACE` + `ADD OSPFIMPORTROUTE` → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)
   - 关键参数：PROCID, AREAID, IFNAME（UNC 真实命令名 **OSPFINTERFACE**——v1.0 写的 `OSPFNETWORK` 是错的）

> **★ 与 [WSFD-010502](feature/UNC/20.15.2/WSFD-010502.md) 的关系**：本特性是 010502 DHCP 子方式的**完整实现**——010502 仅建 POOLTYPE=DHCP 的地址池；本特性额外含 DHCP 服务器组链路（步骤 4）+ 代理 IP（步骤 3）+ DHCP 请求信元（步骤 6）。

## 决策点（特性级：DHCPv4 请求维度）

| 选项 × DNS/P-CSCF 请求 × 地址分配规则 | 走法（复用 compound + 步骤）| 联动 / 规则 |
|---|---|---|
| **DHCPv4 × 仅地址**（默认） | 骨架 [1-00024](task/UNC/20.15.2/1-00024.md) + 地址池 [1-00023](task/UNC/20.15.2/1-00023.md)（POOLTYPE=DHCP）+ 步骤 3-4（DHCP 服务器组）+ 路由 [1-00029](task/UNC/20.15.2/1-00029.md) | 无附加 |
| **DHCPv4 × DNS+P-CSCF** | 骨架 1-00024 + 地址池 1-00023 + 步骤 3-4 + 步骤 6（`SET DHCPPARAREQ` 开启 DNS/P-CSCF 请求信元）+ 路由 1-00029 | DHCP 服务器需支持 DNS/P-CSCF 选项 |
| **DHCPv4 × 自定义分配规则** | 骨架 1-00024 + 地址池 1-00023 + 步骤 3-4 + 步骤 5（`SET IPALLOCRULE` 全局规则）+ 路由 1-00029 | 需配置分配规则参数 |

> **走法列**：3 选项**共享骨架 + 地址池 + DHCP 服务器组链路 + 路由 4 骨架 compound**；差异在步骤 5-6 的附加命令（按需）。

## 约束（特性级 rule）

- **License 前置**（critical）：本特性需 License `LKV3W9DHCP12`（控制项 82209430）授权——`SET LICENSESWITCH` 先开 — 后果：无 License 时功能不可用。
- **DHCP 服务器可达性**（critical）：UNC 通过 `ADD AGENTIP` 配置的代理 IP 与外部 DHCP Server 通信，需保证 `ALM-100466`（DHCP 服务器无响应）不告警 — 后果：用户激活失败。
- **DHCPv4 请求信元须 DHCP Server 支持**（warning）：步骤 6（`SET DHCPPARAREQ` 开启 DNS/P-CSCF 选项）需 DHCP Server 支持对应 Option — 后果：Server 不支持时分配不出 DNS/P-CSCF。
- **P-CSCF 地址上限 2 个**（warning）：UNC 分配给用户的 P-CSCF 地址最大支持 2 个（含本地 + DHCP）— 后果：超出时 DHCP 返回的 P-CSCF 不生效。
- **仅地址不支持域名**（warning）：仅支持在 DHCP 过程中获取 P-CSCF **服务器地址**，不支持获取 P-CSCF **服务器域名** — 后果：域名场景需改用其他方式。
- **POOLNAME 命名约束**（warning）：字符串 1~79，由 `-/_`/数字/字母/`.` 组成，不能以 `.` 开头 — 后果：违规命名新增失败。
- **地址段必配**（critical）：本地地址池（POOLTYPE=DHCP）建好后**必须** `ADD SECTION` 添加地址段，否则 DHCP Discover 失败。
- **AGENTIP 必须在地址段内**（critical）：`ADD AGENTIP` 代理 IP 须位于 DHCP 地址池网段内 — 后果：代理 IP 不在网段内致 DHCP Server 找不到对应地址池。
- **首次发布版本**（info）：本特性首次发布版本为 UNC **20.8.0**；GGSN/SGW-C/PGW-C/SMF 均支持。
