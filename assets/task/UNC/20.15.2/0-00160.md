# SET CHGCHAR 原子任务
> 命令静态知识见 [SET CHGCHAR](command/UNC/20.15.2/SET-CHGCHAR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

SGSN 侧**按计费属性（CC）粒度**的话单参数命令：每个 CC 一条记录，决定该 CC 用户的各类话单怎么生成。话单优先级：HLR 签约 < 全局 `SET CHGCDR`（[0-00159](task/UNC/20.15.2/0-00159.md)）< 本命令（CC 级）。

## 配置方法
以 CC 为主键，按业务场景勾选话单类型 + 配套周期/阈值/CG。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CC | 计费属性（主键） | `NORMAL`（普通）/ `HOTBILLING`（热计费） |
| MP / MPCP / MPL | 生成 M-CDR / 周期生成 / 周期（min） | `YES`·`NO` / `1`~`1440` |
| MLCP / MLCL | 位置更新生成 M-CDR / 最大次数 | `YES`·`NO` / `1`~`10` |
| RPCMCDR | 精简 M-CDR | `YES` / `NO` |
| SP / SPP / SPL | 生成 S-CDR / 周期生成 / 周期（min） | `YES`·`NO`·`PERMIT`·`FORBID` / `1`~`1440` |
| SVP / SVL | 流量生成 S-CDR / 流量阈值（KB） | `YES`·`NO` / `1`~`1000000` |
| SCCP / SCCL | 计费条件变更生成 S-CDR / 最大次数 | `YES`·`NO` / `1`~`10` |
| SLCP | 位置更新生成 S-CDR | `YES`·`NO` |
| PCC | PLMN 变化生成特定原因值 S-CDR | `YES`·`NO` |
| RPCSCDR | 精简 S-CDR | `YES` / `NO` |
| SMOP / SMTP | 生成 S-SMO-CDR / S-SMT-CDR | `YES` / `NO` |
| LCSMOP / LCSMTP / LCSNIP | 生成 LCS-MO/MT/NI-CDR | `YES` / `NO` |
| SR | 计费限呼标志（缓存满时策略） | `UNRESTRICT`（不限制）等 |
| CGIP / CGIPV6 | CG 的 IPv4 / IPv6 地址 | `10.10.171.9` / IPv6 |
| TARIACCFLG | 费率变更允许的用户类型 | `HOME_USER` 等 |

典型脚本（SMS over GPRS 计费，NORMAL 用户开 SMS 计费）：
```
SET CHGCHAR: CC=NORMAL, SMOP=YES, SMTP=YES, CGIP="10.10.171.9";
```
热计费（HOTBILLING 用户周期生成 S-CDR，周期 2 min）：
```
SET CHGCHAR: CC=HOTBILLING, SP=YES, SPP=PERMIT, SPL=2;
```
费率切换（NORMAL 用户 S-CDR + 最大计费条件变更 3 次）：
```
SET CHGCHAR: CC=NORMAL, SP=YES, SCCL=3;
```

## 决策点

### 计费属性（CC）选择
- 问什么：为哪个计费属性配置话单规则。
- 取值由整网规划（如 `NORMAL` 普通计费、`HOTBILLING` 热计费、`REFUND` 等），不同 CC 对应不同话单策略。
- 同一 CC 一条记录；按业务运营的用户分群决定配哪些 CC。

### 话单类型与触发方式（周期/流量/位置/计费条件）
- 问什么：该 CC 用户下，各类话单（M/S/S-SMO/S-SMT/LCS）是否生成，以及 S-CDR 按什么阈值触发。
- 周期触发 → 配 SPL（周期 min）；流量触发 → 配 SVL（KB 阈值）；位置更新 → 配 SLCP/MLCP；计费条件变更 → 配 SCCL（次数）。
- 短消息场景 → SMOP/SMTP=YES；热计费场景 → SPP=PERMIT + SPL 短周期。
- 视运营计费颗粒度决定（细颗粒多触发，粗颗粒仅周期）。

## 约束
- 前置：只有 `ADD CHGCG`（[0-00017](task/UNC/20.15.2/0-00017.md)）命令配置的 CG，才能在本命令中生效（CGIP/CGIPV6 指向的 CG 必须先建）。`severity=high`
- 生效时机：执行后立即生效。`severity=low`
- 初始值：系统初次运行执行初始设置值。`severity=low`
- 计费属性优先级：用户计费属性优先级由低到高为 `SET CHGCDR`（全局缺省）< 本命令（CC 级）< HLR 签约。`severity=medium`
- QoS 变更与话单：PDP 上下文 QoS 改变后是否重新生成话单由本命令配置决定。`severity=medium`
