# 配置计费属性话单参数（SET CHGCHAR）

> 命令静态知识见 [SET CHGCHAR](command/UNC/20.15.2/SET-CHGCHAR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CHGCHAR 是 SGSN 侧**按计费属性（CC）粒度**的话单参数命令：每个 CC 一条记录，决定该 CC 用户的各类话单怎么生成。话单优先级：HLR 签约 < 全局 [SET CHGCDR](task/UNC/20.15.2/0-00159.md) < 本命令（CC 级）。本命令的核心配置空间有 2 个维度：**CC 主键 + 话单类型开关**（必选，决定各类话单是否生成）、**触发阈值与周期**（可选，决定 S-CDR 按什么阈值触发）。

### 配置维度 1：CC 主键 + 话单类型开关（参数 CC + MP/SP/SMOP/SMTP/LCS*）

以 CC 为主键，按业务场景勾选话单类型。数据来源整网规划：

| 参数 | 控制 CDR 类型 | 取值 | 系统初始值 |
|---|---|---|---|
| `CC`（必选） | 计费属性（主键） | `NORMAL`（普通）/ `PREPAID`（预付费）/ `FLATRATE`（包月制）/ `HOTBILLING`（实时计费） | 系统为每种计费方式均提供初始设置 |
| `MP` | 生成 M-CDR | `YES` / `NO` | `NO` |
| `SP` | 生成 S-CDR | `YES` / `NO` | `YES` |
| `SMOP` | 生成 S-SMO-CDR（短消息主叫） | `YES` / `NO` | `YES` |
| `SMTP` | 生成 S-SMT-CDR（短消息被叫） | `YES` / `NO` | `YES` |
| `LCSMOP` | 生成 LCS-MO-CDR（定位主叫） | `YES` / `NO` | `YES` |
| `LCSMTP` | 生成 LCS-MT-CDR（定位被叫） | `YES` / `NO` | `YES` |
| `LCSNIP` | 生成 LCS-NI-CDR（定位网络触发） | `YES` / `NO` | `YES` |

> 同一 CC 一条记录；按业务运营的用户分群决定配哪些 CC。每个 CC 对应不同话单策略（NORMAL 普通计费、HOTBILLING 热计费短周期、PREPAID 预付费等）。

### 配置维度 2：触发阈值与周期（参数 MPL/SPL/SVL/SCCL/MLCP/SLCP + CG/SR/TARIACCFLG）

| 参数 | 作用 | 取值范围 | 系统初始值 |
|---|---|---|---|
| `MPCP` / `MPL` | 周期生成 M-CDR / M-CDR 生成周期（min） | `YES`/`NO` / `1`~`1440` | `YES` / `60` |
| `MLCP` / `MLCL` | 位置更新生成 M-CDR / 最大次数 | `YES`/`NO` / `1`~`10` | `YES` / `5` |
| `RPCMCDR` | 精简 M-CDR（仅含必选域+变化域） | `YES` / `NO` | `NO` |
| `SPP` / `SPL` | 周期生成 S-CDR / S-CDR 生成周期（min） | `FORBID`/`PERMIT`/`VOLUMN` / `1`~`1440` | `VOLUMN` / `60` |
| `SVP` / `SVL` | 流量生成 S-CDR / 流量阈值（KB） | `YES`/`NO` / `1`~`1000000` | `YES` / `1000` |
| `SCCP` / `SCCL` | 计费条件变更生成 S-CDR（QoS/费率时段变更）/ 最大次数 | `YES`/`NO` / `1`~`10` | `YES` / `3` |
| `SLCP` | 位置更新生成 S-CDR | `YES` / `NO` | `NO` |
| `PCC` | PLMN 变化生成原因值 101 的 S-CDR | `YES` / `NO` | `NO` |
| `RPCSCDR` | 精简 S-CDR（仅含必选域+变化域） | `YES` / `NO` | `NO` |
| `SR` | 计费限呼标志（话单池缓存满时策略） | `UNRESTRICT`（不限制，丢话单）/ `RESTRICT`（禁新用户，丢话单）/ `INTERRUPT`（中止业务，不丢话单） | `UNRESTRICT` |
| `CGIP` / `CGIPV6` | CG 的 IPv4 / IPv6 地址（优先于 GGSN/其他 CG 推荐，低于计费行为指定 CG） | `0.0.0.0`~`255.255.255.255` / IPv6 单播 | `0.0.0.0` / `::` |
| `TARIACCFLG` | 费率变更允许的用户类型 | `HOME_USER` / `ROAM_USER` / `VISIT_USER`（可多选） | 全部选中 |

> S-CDR 触发方式：周期触发→配 SPL；流量触发→配 SVL；位置更新→配 SLCP/MLCP；计费条件变更→配 SCCL。短消息场景→SMOP/SMTP=YES；热计费场景→SPP=PERMIT + SPL 短周期。

典型脚本（SMS over GPRS 计费，NORMAL 用户开 SMS 计费）：
```
SET CHGCHAR: CC=NORMAL, SMOP=YES, SMTP=YES, CGIP="10.10.171.9";
```
热计费（HOTBILLING 用户周期生成 S-CDR，周期 2 min）：
```
SET CHGCHAR: CC=HOTBILLING, SP=YES, SPP=PERMIT, SPL=2;
```
费率切换（NORMAL 用户 S-CDR + 最大计费条件变更 3 次）：
```
SET CHGCHAR: CC=NORMAL, SP=YES, SCCL=3;
```
实时计费属性 M-CDR 周期 120 min，关闭 S-SMO-CDR：
```
SET CHGCHAR: CC=HOTBILLING, MPL=120, SMOP=NO;
```

**步骤位置**：在所属 feature 编排中通常位于"配置 CC 级话单参数"步骤——[ADD CHGCG](task/UNC/20.15.2/0-00017.md) 配置 CG 之后，全局 [SET CHGCDR](task/UNC/20.15.2/0-00159.md) 之后（CC 级覆盖全局缺省）。

## 决策点

### 决策点：计费属性（CC）选择（DP 0-03059）

问：为哪个计费属性配置话单规则？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| NORMAL（普通计费） | CC=NORMAL + 按常规话单策略 | 普通用户，按常规方式支付费用 |
| PREPAID（预付费） | CC=PREPAID + 预付费话单策略 | 预付费用户，获取服务前预支付 |
| FLATRATE（包月制） | CC=FLATRATE + 包月话单策略 | 包月用户，月内收费固定 |
| HOTBILLING（实时计费） | CC=HOTBILLING + 短周期/流量阈值触发 | 热计费用户，短时间/流量达阈值及时生成话单 |

> 取值由整网规划，不同 CC 对应不同话单策略。同一 CC 一条记录；按业务运营的用户分群决定配哪些 CC。

### 决策点：话单类型与触发方式（DP 0-03060）

问：该 CC 用户下，各类话单（M/S/S-SMO/S-SMT/LCS）是否生成，以及 S-CDR 按什么阈值触发？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 周期触发 S-CDR | SPP=PERMIT/VOLUMN + SPL（周期 min） | 按固定周期生成话单 |
| 流量触发 S-CDR | SVP=YES + SVL（KB 阈值） | 按流量阈值生成话单（注意需 ≥ SET PFRES 流量上报门限） |
| 位置更新触发 | SLCP=YES（S-CDR）/ MLCP=YES + MLCL（M-CDR） | 位置变化生成话单 |
| 计费条件变更触发 | SCCP=YES + SCCL（最大次数） | QoS/费率时段变更生成话单 |
| 短消息计费 | SMOP=YES / SMTP=YES | SMS over GPRS 计费 |
| 热计费短周期 | SPP=PERMIT + SPL 短周期（如 2 min） | HOTBILLING 及时收费 |
| 精简话单 | RPCSCDR=YES / RPCMCDR=YES | 仅含必选域+变化域，减少话单冗余 |

> 视运营计费颗粒度决定（细颗粒多触发，粗颗粒仅周期）。设置最大计费条件变更次数（SCCL）前要与客户计费中心确认是否能够正常处理多流量节点话单，避免多流量节点话单被丢弃。

## 约束

- **CG 前置（ADD CHGCG）**（high，rule-0-03146）：只有 [ADD CHGCG](task/UNC/20.15.2/0-00017.md) 命令配置的 CG，才能在本命令中生效（CGIP/CGIPV6 指向的 CG 必须先建） — 后果：引用未建 CG 配置无效
- **立即生效**（info，rule-0-03147）：执行后立即生效 — 后果：无需重启
- **系统初始设置值**（info，rule-0-03148）：系统初次运行执行初始设置值，为每种计费方式（NORMAL/PREPAID/FLATRATE/HOTBILLING）均提供计费相关参数的初始设置 — 后果：未显式配置时走初始值
- **话单优先级 CC 级 > 全局 > HLR 签约**（warning，rule-0-03149）：用户计费属性优先级由低到高为 HLR 签约 < 全局 [SET CHGCDR](task/UNC/20.15.2/0-00159.md)（缺省）< 本命令（CC 级） — 后果：本命令覆盖全局缺省；不同 CG 配置优先级从高到低：ADD CHGPLMNCG > ADD CHGBEHA > 本命令 > ADD CHGCG
- **SVL 需 ≥ SET PFRES 流量上报门限**（warning，rule-0-03150）：SVL（生成 S-CDR 流量阈值）取值需大于等于 [SET PFRES] 配置的流量上报门限（2G 对应 RPTF，3G 对应 RPTF3G） — 后果：否则基于流量生成的话单可能不按实际配置的流量阈值生效
