# ADD GLBPCRFGROUP 原子任务
> 命令静态知识见 [ADD GLBPCRFGROUP](command/UNC/20.15.2/ADD-GLBPCRFGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
一条记录=（号段 × PCRF 组 × 优先级）三元组。多号段分别绑不同 PCRF 组时，建多条记录，PRIORITY 体现选择顺序。业务处理中当 APN 下 PCC 使能=INHERIT 且全局 PCC 使能=ENABLE 时，按本绑定关系选 PCRF 组，全不匹配才回落系统缺省 PCRF 组。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| IMSIMSISDNSEG | 号段名称，引用已配置号段 | `msisdnseg1` / `msisdnseg2` |
| PCRFGRPNAME | PCRF 组名，引用已配置 PCRF 组 | `pcrf_group_1` / `pcrf_group_2` |
| PRIORITY | 优先级（本端规划，整数 1~65535） | `1` / `2` |
| DESCRIPTION | 描述（可选） | — |

典型脚本：
```
ADD GLBPCRFGROUP:IMSIMSISDNSEG="msisdnseg1",PCRFGRPNAME="pcrf_group_1",PRIORITY=1;
ADD GLBPCRFGROUP:IMSIMSISDNSEG="msisdnseg2",PCRFGRPNAME="pcrf_group_2",PRIORITY=2;
```

## 决策点

### 号段 × PCRF 组的绑定拓扑
PCC 基本功能（WSFD-109101）将本命令定位为"**可选**：基于 IMSI 号段选择 PCRF 分组"。是否启用、绑几对、优先级如何排，取决于号段运营划分：
- 不按号段细分 PCRF → 不配本命令，直接用 `SET DFTGLBPCRFGRP` 设系统缺省组。
- 多号段分流的 PCRF 组 → 每号段一条记录，PRIORITY 数值越小越优先（业务按号段匹配，全不命中才走缺省）。

### PRIORITY 的来源
自动比对：PRIORITY "本端规划"，而 IMSIMSISDNSEG / PCRFGRPNAME "已配置数据中获取"——PRIORITY 是需规划的新增信息（号段、PCRF 组都是引用既存对象）。多条记录间 PRIORITY 应形成明确的优先序，避免无序兜底。

## 约束
- 唯一性：一个号段最多只能绑定一个指定的 PCRF 组。`severity=high` · 后果：一号段多绑则选择歧义，业务行为不可预期。
- 前置依赖：IMSIMSISDNSEG 必须已通过号段配置命令存在；PCRFGRPNAME 必须已通过 `ADD PCRFBINDGRP` / PCRF 组配置存在。`severity=high` · 后果：引用失败。
- 规格上限：最大记录数 4096。`severity=medium`
- 生效时机：执行后**只对新激活用户生效**，已激活用户不受影响。`severity=medium` · 后果：现网用户需重激活或等待自然流转才应用新绑定。
