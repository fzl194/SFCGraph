# 增加全局PCRF组绑定关系（ADD GLBPCRFGROUP）

> 命令静态知识见 [ADD GLBPCRFGROUP](command/UNC/20.15.2/ADD-GLBPCRFGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

一条记录 = （号段 × PCRF 组 × 优先级）三元组。多号段分别绑不同 PCRF 组时建多条记录，PRIORITY 体现选择顺序。业务处理中当 APN 下 PCC 使能 = INHERIT 且全局 PCC 使能 = ENABLE 时，按本绑定关系选 PCRF 组，全不匹配才回落系统缺省 PCRF 组。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 号段绑定 | IMSIMSISDNSEG | 本端规划，1~31 字符，引用 `ADD IMSIMSISDNSEG` 已配置号段 | 无 | 指定本绑定关系适用的 IMSI/MSISDN 号段 | 所有记录必填 |
| PCRF 组绑定 | PCRFGRPNAME | 本端规划，1~128 字符，引用 `ADD PCRFGROUP` 已配置 PCRF 组 | 无 | 指定号段命中的 PCRF 组 | 所有记录必填 |
| 优先级 | PRIORITY | 本端规划，整数 1~65535，数值越小越优先 | 无 | 多条记录命中时的选择顺序 | 所有记录必填 |
| 描述 | DESCRIPTION | 本端规划，1~63 字符 | 无 | 号段绑定 PCRF 组的描述 | 可选 |

典型脚本：
```
ADD GLBPCRFGROUP:IMSIMSISDNSEG="msisdnseg1",PCRFGRPNAME="pcrf_group_1",PRIORITY=1;
ADD GLBPCRFGROUP:IMSIMSISDNSEG="msisdnseg2",PCRFGRPNAME="pcrf_group_2",PRIORITY=2;
```

步骤位置：建号段（`ADD IMSIMSISDNSEG`）→ 建 PCRF 组（`ADD PCRFGROUP`）→ 本任务 → 配套 `SET DFTGLBPCRFGRP`（系统缺省 PCRF 组兜底）

## 决策点

### 决策点：号段 × PCRF 组的绑定拓扑（DP 0-02047）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不按号段细分 PCRF | 不配本命令，直接用 `SET DFTGLBPCRFGRP` 设系统缺省组 | 全网统一 PCRF 选路 |
| 单号段绑单 PCRF 组 | 1 条记录，PRIORITY 任取 | 号段级 PCRF 分流 |
| 多号段绑多 PCRF 组 | 多条记录，PRIORITY 数值越小越优先（全不命中才走缺省） | 多号段运营划分的 PCRF 分流 |

### 决策点：PRIORITY 的来源（DP 0-02048）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单一规划来源 | PRIORITY 本端规划（号段、PCRF 组是引用既存对象，PRIORITY 是新增规划信息） | 需明确多条记录间优先序，避免无序兜底 |
| 多条记录优先序编排 | 多条记录 PRIORITY 形成明确优先序（数值越小越优先） | 多号段分流且可能多命中场景 |

## 约束

· rule-0-02116 · severity=critical · 唯一性：一个号段最多只能绑定一个指定的 PCRF 组，一号段多绑则选择歧义、业务行为不可预期
· rule-0-02117 · severity=high · 前置依赖：IMSIMSISDNSEG 必须已通过 `ADD IMSIMSISDNSEG` 存在；PCRFGRPNAME 必须已通过 `ADD PCRFGROUP` 存在
· rule-0-02118 · severity=warning · 规格上限：最大记录数 4096
· rule-0-02119 · severity=warning · 生效时机：命令执行后只对新激活用户生效，已激活用户不受影响（需重激活或等待自然流转）
· rule-0-02120 · severity=info · 适用 NF：PGW-C、GGSN
