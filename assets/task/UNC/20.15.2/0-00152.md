# SET APNREPORTATTR 原子任务
> 命令静态知识见 [SET APNREPORTATTR](command/UNC/20.15.2/SET-APNREPORTATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：SMF、SGW-C、PGW-C、GGSN。用于修改 SMF 与 PCRF/PCF、AAA、OCS/CHF、CG、话统、用户面交互时使用的 APN 类型。每次 ADD APN 自动建一条记录（CONGESTIONRPT=DISABLE, INTELLIGENTSEL=DISABLE, LOCATIONREPORT=DISABLE, MAPTRANSDATA=DISABLE, 其余 REQUESTED）。证据显示获取方法多值（本端规划 4 + 全网规划 2），属 decision_driven。

## 配置方法
按"对接网元维度 × APN 类型映射"建模。APN 类型取值常见 `REQUESTED`（请求的，用终端上报的 APN）、`SERVICE`（业务的，用真实 APN）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | APN 名称（本端规划/全网规划，须先 [ADD APN](task/UNC/20.15.2/0-00004.md)） | `apn5` / `apn1` / `ims.real` |
| PCF | 上报给 PCF 的 APN 名 | `SERVICE` / `REQUESTED` |
| CHF | 上报给 CHF 的 APN 名 | `SERVICE` / `REQUESTED` |
| PCRF | 上报给 PCRF 的 APN 名 | `REQUESTED` |
| OCS | 上报给 OCS 的 APN 名 | `REQUESTED` |
| CG | 上报给 CG 的 APN 名 | `REQUESTED` |
| AAAACCT | 上报给 AAA 计费的 APN 名 | `REQUESTED` |
| AAAAUTH | 上报给 AAA 鉴权的 APN 名 | `REQUESTED` |
| DIAMETERAAA | 上报给 Diameter AAA 的 APN 名 | `REQUESTED` |
| PERFORMANCE | 上报给话统的 APN 名 | `REQUESTED` |
| UPF | 上报给用户面的 APN 名 | `REQUESTED` |
| CONGESTIONRPT | 拥塞控制（小区拥塞上报开关） | `ENABLE` / `DISABLE` |
| INTELLIGENTSEL | 智能网关选择 | `ENABLE` / `DISABLE` |
| LOCATIONREPORT | 实时位置 | `ENABLE` / `DISABLE` |
| MAPTRANSDATA | 通过本地配置获取 VLR ID/Global Title | `ENABLE` / `DISABLE` |

典型脚本（激活别名 APN：真实 APN 的 PCF/CHF 配为 SERVICE）：
```
SET APNREPORTATTR: APN="apn5", PCF=SERVICE, CHF=SERVICE;
```
（基于小区负荷上报：开 CONGESTIONRPT）：
```
SET APNREPORTATTR: APN="apn1", CONGESTIONRPT=ENABLE;
```
（VoLTE 快速恢复：PCRF=REQUESTED 映射真实 APN）：
```
SET APNREPORTATTR: APN="ims.real", PCRF=REQUESTED;
```

## 决策点

### 各交互网元 APN 类型映射
- 问什么：上报给 PCF/CHF/PCRF/OCS/CG/AAA/话统/用户面时用哪种 APN 类型。
- `REQUESTED`：终端激活请求里的 APN（别名 APN 时用别名）；`SERVICE`：真实/业务 APN。
- 别名 APN 场景：需将真实 APN 的 PCF 和 CHF 配为 SERVICE，使对端按真实 APN 计费/管控。
- 各网元取值独立，获取方法分叉（本端规划/全网规划/与对端协商），属 decision_driven。

### 周边上报开关组（CONGESTIONRPT / INTELLIGENTSEL / LOCATIONREPORT / MAPTRANSDATA）
- 问什么：是否使能基于 APN 的小区拥塞上报、智能网关选择、实时位置上报、本地获取 VLR ID。
- WSFD-211101 基于小区负荷上报须将 CONGESTIONRPT 设为 ENABLE；License 前置需开 [SET LICENSESWITCH](task/UNC/20.15.2/0-00180.md)（若在映射）。
- 未配时保持当前值不变，可查 LST APNREPORTATTR 确认。

## 约束
- 只对新激活用户生效：不立即作用于在线用户。`severity=medium`
- 每个 APN 一条配置：不允许多条同 APN 记录。`severity=medium`
- ADD APN 自动建记录：每次 [ADD APN](task/UNC/20.15.2/0-00004.md) 自动增加一条初始记录，随后用本命令改写。`severity=medium`
- 可选参数不输入则保持当前值不变：可通过 LST APNREPORTATTR 查询当前值。`severity=medium`
- REQUESTED 遇别名 APN：终端激活请求里是别名 APN 时，对该网元按别名 APN 处理（计费/统计口径需评估）。`severity=medium`
