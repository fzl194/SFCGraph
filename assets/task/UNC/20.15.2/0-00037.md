# ADD HTTPLE 原子任务
> 命令静态知识见 [ADD HTTPLE](command/UNC/20.15.2/ADD-HTTPLE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
HTTP 本端实体按"角色 × IP 类型"规划：`LETYPE` 决定是服务端（需 `PORT`）还是客户端；同组内服务端+客户端成对出现，组号由 [ADD HTTPLEGRP](task/UNC/20.15.2/0-00038.md) 预先建立，`HTTPLEGRPIDX` 引用之。启 TLS 时 `TLSIDX` 必须与 `ADD TLSPARA` 的索引匹配。NF（AMF/NRF/BSF）每激活一个服务化角色，通常配一对（SERVER+CLIENT）记录。

| 角色 | 必备参数 | 典型脚本 |
|---|---|---|
| 服务端 SERVER | `INDEX/HTTPLEGRPIDX/LETYPE=SERVER/PORT/IPTYPE/IPV4或IPV6`(+启 TLS 时 `TLSFLG=YES/TLSIDX`) | `ADD HTTPLE:INDEX=1,HTTPLEGRPIDX=1,LETYPE=SERVER,PORT=30116,TLSFLG=YES,TLSIDX=1,IPTYPE=IPv4,IPV4="10.16.1.5",VPNNAME="_public_";` |
| 客户端 CLIENT | `INDEX/HTTPLEGRPIDX/LETYPE=CLIENT/IPTYPE/IPV4或IPV6`(+启 TLS 时 `TLSFLG=YES/TLSIDX`，无 PORT) | `ADD HTTPLE:INDEX=2,HTTPLEGRPIDX=1,LETYPE=CLIENT,TLSIDX=2,IPTYPE=IPv4,IPV4="10.16.1.6",VPNNAME="_public_";` |

**IP 类型**：`IPv4` → 配 `IPV4`；`IPv6` → 配 `IPV6`（QoS 能开放 BSF 场景见 IPv6 用例）。

**典型组合**：SBI 接口加密（CERT/PSK）下，AMF 与 NRF 各配一对（SERVER+CLIENT）；QoS 能力开放的 BSF 单独一组（INDEX=3，DESC="BSF"）。

## 决策点（若有）

### 是否启用 TLS
问：该本端实体链路是否走 HTTPS？
- `TLSFLG=YES` + `TLSIDX=<TLSPARA 索引>`：SBI 接口加密特性（CERT/PSK）必选。
- `TLSFLG=NO`（或留空）：QoS 能力开放 BSF 等非加密场景。
- 注意：CLIENT 若未配 TLSIDX 而实际建的是 HTTPS 链路，使用系统缺省 TLS 参数；若配了 TLSIDX 而实际建 HTTP 链路，TLSIDX 不生效。

### IP 类型与地址
问：链路走 IPv4 还是 IPv6？
- `IPTYPE=IPv4` + `IPV4=<地址>`：IPv4 组网。
- `IPTYPE=IPv6` + `IPV6=<地址>`：IPv6 组网（如 QoS 能力开放 BSF 场景 `fe80:8029:5008:6B::247`）。

## 约束（若有）
- 前置依赖：`HTTPLEGRPIDX` 必须已通过 [ADD HTTPLEGRP](task/UNC/20.15.2/0-00038.md) 配置存在。`severity=high` 后果：引用失败。
- 前置依赖：启 TLS 时 `TLSIDX` 必须与 `ADD TLSPARA` 的索引匹配。`severity=high`
- 唯一性：CLIENT 类型同一 IP 地址只能配一条记录。`severity=high`
- 端口冲突：SERVER 与 CLIENT 相同 IP 时，SERVER 端口号不能落在 [24576,32767] 范围内。`severity=high`
- 规格上限：INDEX 取值 1~128，HTTPLEGRPIDX 取值 1~64。`severity=medium`
- 执行后立即生效。`severity=low`
