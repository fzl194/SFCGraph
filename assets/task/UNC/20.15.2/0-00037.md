# 配置HTTP本端实体（ADD HTTPLE）

> 命令静态知识见 [ADD HTTPLE](command/UNC/20.15.2/ADD-HTTPLE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

HTTP 本端实体是服务化接口（SBI/BSF）的本端地址承载层：每激活一个 NF（AMF/NRF/BSF）的服务化角色，通常配一对（SERVER + CLIENT）记录，组号引用 [ADD HTTPLEGRP](task/UNC/20.15.2/0-00038.md)。本命令的核心配置空间有 3 个维度：**本端实体类型**（必选，决定是否需要 PORT）、**IP 类型**（必选，决定走 IPv4 还是 IPv6 地址）、**是否启用 TLS**（必选，决定是否需要 TLSIDX 关联）。同组内 SERVER+CLIENT 成对出现，IP 地址须已通过 [ADD LOGICIP](task/UNC/20.15.2/0-00044.md) 配置存在。

### 配置维度 1：本端实体类型（参数 LETYPE，必选）

| 取值 | 作用 | 联动必选参数 | 联动可选参数 | 典型场景 |
|---|---|---|---|---|
| **SERVER**（服务端） | 监听并对端提供 HTTP/HTTPS 服务 | `PORT`（条件必选，不配则默认 80/443）、`TLSFLG`（条件必选） | — | NF 作为服务端接收 SBI 请求（AMF/NRF/BSF 各一对） |
| **CLIENT**（客户端） | 主动向对端发起 HTTP/HTTPS 请求 | 无 PORT（CLIENT 端口号由系统动态生成或静态配置） | `TLSIDX`（条件可选） | NF 作为客户端向 NRF/BSF 发起 SBI 调用 |

> 同一组内 SERVER 与 CLIENT 成对出现；同一 IP 只能配一条 CLIENT 记录。

### 配置维度 2：IP 类型（参数 IPTYPE，必选）

| 取值 | 作用 | 联动必选参数 | 约束 |
|---|---|---|---|
| **IPv4** | 使用 IPv4 地址 | `IPV4` | IPV4 必须已通过 `ADD LOGICIP` 配置存在 |
| **IPv6** | 使用 IPv6 地址 | `IPV6` | IPV6 必须已通过 `ADD LOGICIP` 配置存在；QoS 能力开放 BSF 场景见 IPv6 用例 |

> IP 地址相同的记录，HTTPLEGRPIDX、VPNNAME、RFLAG 必须相同。

### 配置维度 3：是否启用 TLS（参数 TLSFLG，必选，SERVER；CLIENT 通过 TLSIDX 隐式）

| 取值 | 作用 | 联动必选参数 | 适用场景 |
|---|---|---|---|
| **YES**（默认） | 启用 TLS 传输加密，HTTP 层报文基于 TLS 加密 | `TLSIDX`（关联 [ADD TLSPARA](command/UNC/20.15.2/ADD-TLSPARA.md) 的索引） | SBI 接口加密特性（CERT/PSK）必选 |
| **NO** | 不启用 TLS，明文 HTTP | 无 TLSIDX | QoS 能力开放 BSF 等非加密场景 |

> CLIENT 若未配 TLSIDX 而实际建的是 HTTPS 链路，使用系统缺省 TLS 参数；若配了 TLSIDX 而实际建 HTTP 链路，TLSIDX 不生效。同一本端实体组内所有 CLIENT 的 TLSIDX 值必须相同。

典型脚本：
```
ADD HTTPLE:INDEX=1,HTTPLEGRPIDX=1,LETYPE=SERVER,PORT=30116,TLSFLG=YES,TLSIDX=1,IPTYPE=IPv4,IPV4="10.16.1.5",VPNNAME="_public_";
ADD HTTPLE:INDEX=2,HTTPLEGRPIDX=1,LETYPE=CLIENT,TLSFLG=YES,TLSIDX=1,IPTYPE=IPv4,IPV4="10.16.1.6",VPNNAME="_public_";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：配置 HTTP 本端实体"——先 ADD HTTPLEGRP 建组，再 ADD HTTPLE 配 SERVER+CLIENT 成员，最后被 ADD SBIAPLE 等上层引用。

## 决策点：是否启用 TLS（DP 0-02053）

问：该本端实体链路是否走 HTTPS？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 启用 TLS（YES） | `TLSFLG=YES` + `TLSIDX=<TLSPARA 索引>` | SBI 接口加密特性（CERT/PSK）必选；AMF 与 NRF 通信加密 |
| 不启用 TLS（NO） | `TLSFLG=NO`（或不配 TLSIDX） | QoS 能力开放 BSF 等非加密场景 |

### 决策点：IP 类型与地址（DP 0-02054）

问：链路走 IPv4 还是 IPv6？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPv4 组网 | `IPTYPE=IPv4` + `IPV4=<地址>` | 主流 IPv4 组网（如 `10.16.1.5`） |
| IPv6 组网 | `IPTYPE=IPv6` + `IPV6=<地址>` | IPv6 组网（如 QoS 能力开放 BSF 场景 `fe80:8029:5008:6B::247`） |

## 约束

- **HTTPLEGRPIDX 必须已通过 ADD**（critical，rule-0-02131）：HTTPLEGRPIDX 必须已通过 ADD HTTPLEGRP 配置存在，否则引用失败 — 后果：本端实体无法关联到组，服务化链路建链失败
- **CLIENT 类型同一 IP 地址只能配一条记录**（warning，rule-0-02132）：CLIENT 类型同一 IP 地址只能配一条记录；SERVER 类型同一 IP+端口号只能配一条记录 — 后果：配置冲突，链路标识歧义
- **SERVER 与 CLIENT 相同 IP 时**（warning，rule-0-02133）：SERVER 与 CLIENT 相同 IP 时，SERVER 端口号不能落在 [24576,32767] 范围内 — 后果：端口冲突导致服务端无法监听
- **启 TLS 时 TLSIDX 必须与 ADD TLSPARA**（warning，rule-0-02134）：启 TLS 时 TLSIDX 必须与 ADD TLSPARA 的索引匹配；同一本端实体组内所有 CLIENT 的 TLSIDX 值必须相同 — 后果：TLS 握手失败或组内链路参数不一致
- **INDEX 取值 1~128**（warning，rule-0-02135）：INDEX 取值 1~128、HTTPLEGRPIDX 取值 1~64、单板最多 128 条记录 — 后果：超规格新增失败
