# 增加用户模板和规则的绑定关系（ADD RULEBINDING）

> 命令静态知识见 [ADD RULEBINDING](command/UNC/20.15.2/ADD-RULEBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD RULEBINDING 建立 (USERPROFILENAME, RULENAME[, POLICYTYPE]) 绑定关系：Rule 本身不能直接下发 UPF，必须先挂到 UserProfile 上，SMF 下发 UserProfile 即激活其下全部 Rule。当用户报文匹配多条规则时，把多条规则逐条绑到同一 UserProfile 即可。配置空间有 3 个维度：**用户模板名称**（必选，引用源）、**规则名称**（必选，引用源）、**策略类型**（可选，同名跨类型时需显式指定）。

### 配置维度 1：用户模板名称（参数 USERPROFILENAME，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 用户模板名称 | `USERPROFILENAME` | 本端规划；字符串 1~63，不支持空格，不区分大小写；由 ADD USERPROFILE 配置生成 | 无 | 指定规则绑定的目标用户模板；ULCL 类型模板只能绑 ULCL 规则，PCC 类型模板可绑除 ULCL 外的规则 | 全部 RULEBINDING 场景 |

### 配置维度 2：规则名称（参数 RULENAME，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 规则名称 | `RULENAME` | 本端规划；字符串 1~63，不支持空格，不区分大小写；由 ADD RULE 配置生成 | 无 | 指定要绑定的规则；一条命令绑一个 RULENAME，多规则场景连续执行多条 | 单规则 / 多规则挂同一模板场景 |

### 配置维度 3：策略类型（参数 POLICYTYPE，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 策略类型 | `POLICYTYPE` | 本端规划；枚举：PCC/BWM/HEADEN/WEBPROXY/IPREDIR/SMARTREDIRECT/REMARK_FPI/SRV_TRIGGER/TRAFFICCLASS/LBO/FIREWALL/ULCL/ADC/QOS/FUP/NON_SPECIFIC_TYPE | 不配时取 RULENAME 对应 rule 的策略类型 | 指定绑定的策略类型；PCC 类型模板不指定时把 RULENAME 相同的所有非 ULCL 类型规则都绑上 | 同名跨类型场景显式指定；唯一场景可省略 |

> POLICYTYPE 省略语义：不指定时把 RULENAME 相同的所有非 ULCL 策略类型的规则都绑到模板（仅 PCC 类型模板生效）。建议同名跨类型时显式指定以避免误绑。

**前置依赖**：必须先执行 [ADD USERPROFILE](task/UNC/20.15.2/0-00094.md) 和 [ADD RULE](task/UNC/20.15.2/0-00071.md)，顺序不能反。

**典型脚本（PCC 类型，融合计费单规则）**
```
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="rule_test";
```

**典型脚本（QOS 显式指定）**
```
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="rule_test",POLICYTYPE=QOS;
```

**典型脚本（ULCL 分流场景）**
```
ADD RULEBINDING:USERPROFILENAME="testuserprofilename",RULENAME="testrule",POLICYTYPE=ULCL;
```

**典型脚本（多规则挂同一模板，融合计费连续执行）**
```
ADD RULEBINDING:USERPROFILENAME="userprofile001",RULENAME="Rule001_dns";
ADD RULEBINDING:USERPROFILENAME="userprofile001",RULENAME="Rule001_http";
```

**步骤位置**：在 PCC/计费/QoS/分流特性编排中通常位于"绑定规则到模板"步骤——规则下发阶段，前置依赖 [ADD USERPROFILE](task/UNC/20.15.2/0-00094.md)（USERPROFILENAME 引用源）和 [ADD RULE](task/UNC/20.15.2/0-00071.md)（RULENAME 引用源），后续可由 [ADD UPBINDUPG](task/UNC/20.15.2/0-00089.md) 把模板聚合到模板组（[ADD USRPROFGROUP](task/UNC/20.15.2/0-00096.md)）。

## 决策点

### 决策点：POLICYTYPE 是否显式指定（DP 0-02125）

问：同一 RULENAME 是否存在多个 POLICYTYPE 的规则？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 同名跨类型，必须显式指定 | `USERPROFILENAME` + `RULENAME` + `POLICYTYPE=<具体类型>` | 同一 RULENAME 存在 PCC/QOS/BWM 等多个策略类型版本；省略会把所有同名规则都绑上模板导致误绑 |
| 唯一规则，可省略 | `USERPROFILENAME` + `RULENAME`（不配 POLICYTYPE，默认取 RULENAME 对应 rule 的策略类型） | ULCL/QOS/BWM/PCC 命名不跨类型复用时；证据中多数场景省略 |

### 决策点：单模板挂多少规则及模板类型约束（DP 0-02126）

问：一个 UserProfile 绑几条 Rule？模板类型与规则类型如何匹配？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单规则 | 一条 ADD RULEBINDING | ADC、业务触发 QoS 保证等单业务场景 |
| 多规则挂同一模板 | 同一 USERPROFILENAME 连续执行多条 ADD RULEBINDING，每条绑一个 RULENAME | 融合计费（多业务规则挂一模板）、语音 PCF/PCRF 故障 Bypass（信令组/语音组规则挂一模板）；受单模板上限约束 |
| ULCL 类型模板 | `USERPROFILENAME`（UPTYPE=ULCL）+ `RULENAME`（POLICYTYPE=ULCL） | ULCL 分流场景；ULCL 类型模板只能绑 ULCL 类型规则，最多 512 个 |
| PCC 类型模板 | `USERPROFILENAME`（UPTYPE=PCC）+ `RULENAME`（POLICYTYPE 非 ULCL） | 计费/QoS/ADC/BWM 等场景；PCC 类型模板可绑除 ULCL 外的规则 |

## 约束

- rule-0-02311 · severity=critical · 前置依赖：必须先执行 [ADD USERPROFILE](task/UNC/20.15.2/0-00094.md) 和 [ADD RULE](task/UNC/20.15.2/0-00071.md)，否则本命令失败
- rule-0-02312 · severity=critical · 模板类型与规则类型匹配：ULCL 类型用户模板只能绑 ULCL 类型规则；PCC 类型用户模板可绑除 ULCL 外的规则，不指定 POLICYTYPE 时把 RULENAME 相同的所有非 ULCL 策略类型规则绑定到模板
- rule-0-02313 · severity=high · 跨网元一致性：SMF、UPF 上 USERPROFILE 下绑定的 RULENAME、POLICYTYPE 必须一致；PCF 通过预定义规则组下发 APP_STA/APP_STO 时，UPF/SMF/PCF 上的规则（组）标识必须一致
- rule-0-02314 · severity=warning · 单模板规则数上限：同一 ULCL 类型用户模板下最多绑 512 个规则；UPSPECTYPE=SPECIFICATION_LIMITED 时单模板最多 100 个规则；UPSPECTYPE=DEFAULT（或不指定）时单模板最多 8000 个规则（其中 RULETYPE=SPECIFICATION_LIMITED 的规则最多 100 个）
- rule-0-02315 · severity=info · 命名规格与生效：USERPROFILENAME、RULENAME 长度均 1~63，不支持空格，不区分大小写；命令最大记录数 200000；执行后立即生效；调测建议——配错时先 RMV RULEBINDING 再重新 ADD，不要直接覆盖；适用 NF：PGW-C、SMF
