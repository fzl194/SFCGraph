# 配置N40接口协议版本与增强功能集（SET N40APIVER）

> 命令静态知识见 [SET N40APIVER](command/UNC/20.15.2/SET-N40APIVER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET N40APIVER 是 N40 接口（SMF↔CHF 融合计费接口）协议版本与增强功能集的设置命令。系统部署后已存在初始记录（`APIVER=F30, FEATURE=NODEFUNC-1`）。本命令有 2 个配置维度：**API 接口版本（APIVER）**、**增强功能集组合（FEATURE，位域叠加型）**。

### 配置维度 1：API 接口版本（参数 APIVER，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| API 版本 | `APIVER` | `F30`/`F40` | 无（初始记录 F30） | 指定 SMF 支持的 N40 接口 API 版本（3GPP TS 32.291） | 全网规划统一，须与 CHF 一致 |

> 数据来源：全网规划。`F30` 遵循 3GPP TS 32.291 F30 定义的 Nchf_ConvergedCharging；`F40` 在 F30 基础上变更 usedUnitContainer 信元名并新增 dnnSelectionMode。SMF 和 CHF 对接须使用相同协议版本。

### 配置维度 2：增强功能集组合（参数 FEATURE，可选，位域叠加型）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 功能集 | `FEATURE` | 位域叠加：`NODEFUNC`/`ISMF`/`SESSIONAMBRCHG`/`UPFRGLEVEL`/`GERANCHG`/`WLANCHG`/`NBIOTCHG`/`LTEMCHG`/`UTRANCHG`/`UTRANCHG3GPP`/`TRIGGER23G`/`AFCHARGINGIDSTR`/`VSMF` 等，使能后缀 `-1`、不使能 `-0`、不携带=不修改；多值用 `&` 拼接 | 无（初始记录 `NODEFUNC-1`） | 指定 N40 接口增强的功能集 | 由本局点接入的 RAT 类型集合决定，须与 CHF 能力对齐 |

> 数据来源：全网规划。FEATURE 为位域叠加型——支持多值用 `&` 拼接，使能时后缀 `-1`、不使能后缀 `-0`，不修改的功能不携带。代表性组合：

| 启用能力组合 | FEATURE 取值 | 适用场景 |
|---|---|---|
| 仅基础 NodeFunction（初始默认） | `NODEFUNC-1` | 无特殊接入类型 |
| NB-IoT + LTE-M 计费增强 | `NBIOTCHG-1&LTEMCHG-1` | NB-IoT/LTE-M 接入融合计费，联动 [ADD N40MSGTEMP](task/UNC/20.15.2/0-00046.md) `NBEXTENDATTR=ENABLE` |
| UTRAN 3GPP 计费增强 | `UTRANCHG3GPP-1` | 3G 用户接入，非漫游场景携带位置信息+ULI 变化事件 |
| Trigger 2/3G 计费增强 | `TRIGGER23G-1` | 2/3G 用户接入，处理 CHF 下发的 CGI_SAI_CHANGE/RAI_CHANGE |
| GERAN 计费增强 | `GERANCHG-1` | 2G 用户接入，携带 geraLocation/GERA/SGSN |
| WLAN 计费增强 | `WLANCHG-1` | 非可信 Non-3GPP 用户接入，服务节点类型携带 EPDG |
| 多能力叠加 | `NODEFUNC-1&GERANCHG-1&WLANCHG-1` | 多种接入类型并存 |

**典型脚本**

```
# 场景1：仅基础 NodeFunction（初始默认）
SET N40APIVER: APIVER=F30, FEATURE=NODEFUNC-1;

# 场景2：NB-IoT + LTE-M 计费增强（联动 N40MSGTEMP）
SET N40APIVER: APIVER=F30, FEATURE=NBIOTCHG-1&LTEMCHG-1;
ADD N40MSGTEMP: TEMPLATENAME="n40attr", NBEXTENDATTR=ENABLE;

# 场景3：多能力叠加（基础 + GERAN + WLAN）
SET N40APIVER: APIVER=F30, FEATURE=NODEFUNC-1&GERANCHG-1&WLANCHG-1;
```

**步骤位置**：[配置N40接口的API版本和增强的功能集](evidence/UNC/20.15.2/配置N40接口的API版本和增强的功能集_31702747.md) 流程的核心步骤（第 2 步），在 License 加载完成后执行；如启用 NB-IoT/LTE-M 计费增强，需联动执行 [ADD N40MSGTEMP](task/UNC/20.15.2/0-00046.md)（配置 NBEXTENDATTR=ENABLE）。

## 决策点

### 决策点：API 接口版本选择（DP 0-03105）

问：APIVER 取哪个值？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| F30 | `APIVER=F30` | 遵循 3GPP TS 32.291 F30，现网主流；证据中全网规划统一为 F30 |
| F40 | `APIVER=F40` | 遵循 F40，变更 usedUnitContainer 信元名并新增 dnnSelectionMode；建议联动 DWORD513 BIT5 |

> 全网规划项，须与 CHF 协议版本完全一致；跨版本升级由全网规划决定。

### 决策点：增强功能集组合（DP 0-03106）

问：本局点启用哪些 N40 增强计费能力？

| 选项 | 联动 | 适用场景 |
|---|---|---|
| `NODEFUNC-1` | 仅基础 NodeFunction（Serving Node 支持 SGW/AMF） | 系统初始默认，无特殊接入类型 |
| `NBIOTCHG-1` | 联动 [ADD N40MSGTEMP](task/UNC/20.15.2/0-00046.md) `NBEXTENDATTR=ENABLE` | NB-IoT 接入融合计费，携带 NB-IoT 接入类型+扩展字段，PDU 会话支持 NONIP |
| `LTEMCHG-1` | 常与 `NBIOTCHG-1` 同启 | LTE-M 接入融合计费，携带 LTE-M 接入类型+扩展字段 |
| `UTRANCHG3GPP-1` | 独立 | 3G 用户接入，非漫游场景携带位置信息+支持 ULI 变化事件 |
| `GERANCHG-1`/`WLANCHG-1`/`TRIGGER23G-1` | 各自独立 | 对应 GERAN/WLAN/2-3G Trigger 计费场景 |

> 决策依据：本局点实际接入的 RAT 类型集合（NB-IoT / LTE-M / UTRAN / GERAN / WLAN 等），与 CHF 能力对齐；多能力用 `&` 拼接，不修改的功能不携带。

## 约束

- **CHF 能力一致**（critical，rule-0-03261）：CHF 能力一致：`APIVER`/`FEATURE` 需与 CHF 协议版本和接口能力完全一致（全网规划），否则 CHF 无法处理 SMF 发送的消息 — 后果：N40 接口对接失败，CHF 无法处理 SMF 消息
- **功能影响范围**（critical，rule-0-03262）：功能影响范围：执行后立即生效，且会修改 N40 接口功能和信元携带能力；如果 CHF 不支持对应功能或信元，将无法正常处理 SMF 发送的消息 — 后果：CHF 不支持时无法处理 SMF 消息
- **NB-IoT 联动 N40MSGTEMP**（warning，rule-0-03263）：NB-IoT 联动 N40MSGTEMP：NB-IoT 计费增强（`NBIOTCHG-1`）开启后，需同步在 [ADD N40MSGTEMP](task/UNC/20.15.2/0-00046.md) 配 `NBEXTENDATTR=ENABLE` 才能生效 — 后果：未配 N40MSGTEMP 导致 NB-IoT 扩展属性不携带
- **FEATURE 位域拼接**（warning，rule-0-03264）：FEATURE 位域拼接：FEATURE 为位域叠加型，使能带 `-1`、不使能带 `-0`、不修改不携带；多值用 `&` 拼接 — 后果：拼接错误导致能力误开/误关
- **初始记录与查询**（info，rule-0-03265）：初始记录与查询：系统部署后已存在初始记录（`F30 + NODEFUNC-1`）；不输入 `FEATURE` 时保持当前值不变，可通过 `LST N40APIVER` 查询 — 后果：覆盖既有记录，未输入参数保持不变
