# 增加IPv6静态路由（ADD SRROUTE6）

> 命令静态知识见 [ADD SRROUTE6](command/UNC/20.15.2/ADD-SRROUTE6.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令手工添加 IPv6 静态路由，用于简单网络或动态路由不可达场景。配置空间核心维度为 **地址族**（AFTYPE）、**VPN 实例**（VRFNAME/DESTVRFNAME）和 **下一跳出接口组合**（IFNAME/NEXTHOP）。

### 配置维度1：地址族与 VPN 实例（参数 AFTYPE / VRFNAME / DESTVRFNAME）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| AFTYPE=ipv6unicast | IPv6 单播地址族 | 所有 IPv6 静态路由 |
| + VRFNAME | 路由属指定 VPN 实例（私网 IPv6 路由） | VPN 内静态路由（IPFD-016000 IPv6 IPSec：vrf1） |
| + DESTVRFNAME | 下一跳属另一 VPN 实例（跨 VPN 路由泄漏） | VPN 间互访 |

### 配置维度2：目的前缀 + 下一跳/出接口（参数 PREFIX / MASKLENGTH / NEXTHOP / IFNAME）

| 组合 | 作用 | 典型场景 |
|---|---|---|
| PREFIX + MASKLENGTH + NEXTHOP + IFNAME | 经指定接口到下一跳（点对点链路） | IPSec 隧道（IPFD-016000：PREFIX="2001:db8:4::2"/128 + NEXTHOP + IFNAME=Tunnel27） |
| PREFIX + MASKLENGTH + NEXTHOP（无 IFNAME） | 仅按下一跳寻址（广播网段） | 共享网段场景 |
| PREFIX + MASKLENGTH + IFNAME（无 NEXTHOP，如 Invalid0） | 仅出接口（点对点/P2P 接口） | P2P 隧道/丢弃接口 |

> MASKLENGTH：IPv6 单条 0~128；实战多用 /128（主机路由）或 /24、/32（网段）。

### 配置维度3：可选调优（PREFERENCE / BFD / TAG / DESCRIPTION）

| 参数 | 作用 | 典型值 |
|---|---|---|
| PREFERENCE | 路由优先级（值小优先） | 60（默认）~255 |
| BFDENABLE + SESSIONNAME | 绑定 BFD（动态/静态）做链路检测 | TRUE + 会话名 |
| TAG | 路由 tag | 0~4294967295 |
| DESCRIPTION | 路由描述 | 字符串 1~35 |

**步骤位置**：在 IPv6 路由族（IPv6 IPSec 等）编排中，于 `ADD VPNINST`（VPN 实例）、`ADD INTERFACE`（出接口如 Tunnel27）、`ADD IFIPV6ADDRESS`（接口 IPv6）之后执行。

## 决策点

**下一跳表达方式**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IFNAME + NEXTHOP | 指定出接口 + 下一跳 IPv6 | 点对点隧道（IPFD-016000 Tunnel27 + 下一跳） |
| 仅 NEXTHOP | 仅下一跳（无显式接口） | 共享/广播网段 |
| 仅 IFNAME（含 Invalid0） | 仅出接口 | P2P 接口或丢弃（Invalid0） |

> VRFNAME 与 DESTVRFNAME 决定路由的 VPN 归属与跨 VPN 引用；IPFD-016000 中常见 VRFNAME=DESTVRFNAME=vrf1（同 VPN 内）。

## 约束

- **AFTYPE 必填**（critical）：必须指定地址族（ipv6unicast） — 后果：缺则新增失败
- **PREFIX/MASKLENGTH 必填且合法**（critical）：PREFIX 为合法 IPv6，MASKLENGTH 0~128 — 后果：非法新增失败
- **VRFNAME 必须先建**（critical）：VRFNAME 若填必须由 `ADD VPNINST` 已配置 — 后果：VPN 不存在新增失败
- **DESTVRFNAME 必须先建**（critical）：DESTVRFNAME 若填必须由 `ADD VPNINST` 已配置 — 后果：VPN 不存在新增失败
- **IFNAME 必须存在**（warning）：IFNAME 若填必须为已建接口（含 Invalid0） — 后果：接口不存在新增失败
- **NEXTHOP 合法性**（warning）：合法 IPv6 地址 — 后果：非法新增失败
- **PREFERENCE 范围**（warning）：整数 1~255 — 后果：超范围新增失败
- **DESCRIPTION 长度**（warning）：字符串 1~35 — 后果：超长新增失败
