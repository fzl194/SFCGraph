# 增加预定义规则关联的DNAI（ADD RULEBINDDNAI）

> 命令静态知识见 [ADD RULEBINDDNAI](command/UNC/20.15.2/ADD-RULEBINDDNAI.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD RULEBINDDNAI 把一条已配的预定义规则（RULENAME）与一个边缘 UPF 的 DNAI 绑定，绑定后该规则**只**下发给该 DNAI 对应的辅锚点 UPF。本命令是动态/本地 PCC 的可选步骤，仅当规则需下发指定辅锚点时配置。配置空间有 2 个维度：**预定义规则名称**（必选，引用源且决定 RULERANGE 前提）、**数据网络访问标识符**（必选，全网规划）。

### 配置维度 1：预定义规则名称（参数 RULENAME，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 预定义规则名称 | `RULENAME` | 全网规划；字符串 1~63，不支持空格，不区分大小写；由 ADD RULE 配置生成 | 无 | 指定要绑 DNAI 的预定义规则；该规则的 RULERANGE 必须为 LOCAL | 动态 PCC（5G 辅锚点 UPF）、本地 PCC（4G 辅锚点 PGW-U） |

### 配置维度 2：数据网络访问标识符（参数 DNAI，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 数据网络访问标识符 | `DNAI` | 全网规划；字符串 1~63，不支持空格，不区分大小写；需与 ADD PNFDNAI 的 DNAI 保持一致 | 无 | 指定规则下发的目标边缘 UPF；绑定后规则只下发该 DNAI 对应辅锚点 | 全部 RULEBINDDNAI 场景 |

> 前置：必须先用 [ADD RULE](task/UNC/20.15.2/0-00071.md) 创建规则，且该规则的 `RULERANGE=LOCAL`。绑定关系由本命令独立建立，RULE 本身不感知 DNAI。

**典型脚本（动态 PCC，5G 辅锚点 UPF）**
```
ADD RULEBINDDNAI:RULENAME="rule_test",DNAI="huawei.com";
```

**典型脚本（本地 PCC，4G 辅锚点 PGW-U）**
```
ADD RULEBINDDNAI:RULENAME="rule_test",DNAI="huawei.com";
```

**步骤位置**：在动态/本地 PCC 特性编排中通常位于"配置规则 DNAI 绑定"步骤——边缘分流规则精准下发阶段，紧随 [ADD RULE](task/UNC/20.15.2/0-00071.md)（RULENAME 来源且决定 RULERANGE 前提）和 ADD PNFDNAI（DNAI 来源）之后，与 [ADD USRPROBINDDNAI](task/UNC/20.15.2/0-00095.md)（用户模板级 DNAI 绑定）互补。

## 决策点

### 决策点：是否配置本命令（DP 0-02123）

问：预定义规则是否需要下发到指定辅锚点 UPF/PGW-U？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 需要绑定 DNAI | `RULENAME` + `DNAI`（前提：[ADD RULE](task/UNC/20.15.2/0-00071.md) 的 `RULERANGE=LOCAL`） | 动态 PCC（5G 辅锚点 UPF）、本地 PCC（4G 辅锚点 PGW-U）——规则精准下发指定边缘 UPF |
| 跳过本命令（RULERANGE=ALL） | 不执行本命令 | SMF 自动下发中心和所有边缘 UPF，绑 DNAI 无意义 |
| 跳过本命令（RULERANGE=CENTRAL） | 不执行本命令 | 规则只下主锚点，不能绑任何边缘 DNAI |

### 决策点：DNAI 取值来源与全网一致性（DP 0-02124）

问：DNAI 取值如何确定？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 全网规划（与 PNFDNAI 一致） | `DNAI=<全网规划标识>`（如 `huawei.com`），需与 ADD PNFDNAI 的"数据网络访问标识"参数保持一致 | 证据中所有场景均标"全网规划"；DNAI 是网络拓扑标识，需与对端 UPF/PGW-U 的 DNAI 规划一致 |

## 约束

- **RULERANGE 前提**（critical，rule-0-02306）：RULERANGE 前提：仅当 [ADD RULE](task/UNC/20.15.2/0-00071.md) 的 `RULERANGE=LOCAL` 时可绑定 DNAI；`RULERANGE=ALL` 绑定无意义（SMF 会同时下发中心和所有边缘 UPF）；`RULERANGE=CENTRAL` 不能绑定任何边缘 DNAI
- **绑定后行为**（warning，rule-0-02307）：绑定后行为：当一个预定义规则绑定到某边缘 UPF 所对应的 DNAI 时，该预定义相关规则就只下发给该边缘 UPF（不再下发给其他锚点）
- **RULENAME 前置**（warning，rule-0-02308）：RULENAME 前置：`RULENAME` 长度 1~63，不支持空格，不区分大小写；必须已由 [ADD RULE](task/UNC/20.15.2/0-00071.md) 配置生成
- **DNAI 一致性**（warning，rule-0-02309）：DNAI 一致性：`DNAI` 长度 1~63，不支持空格，不区分大小写；需与 ADD PNFDNAI 命令中的"数据网络访问标识"参数保持一致
- **规格与生效**（info，rule-0-02310）：规格与生效：最多可输入 10000 条记录；命令执行后立即生效；适用 NF：SMF、PGW-C、GGSN
