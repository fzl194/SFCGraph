# ADD RULEBINDDNAI 原子任务
> 命令静态知识见 [ADD RULEBINDDNAI](command/UNC/20.15.2/ADD-RULEBINDDNAI.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令把一条已配的预定义规则（RULENAME）与一个边缘 UPF 的 DNAI 绑定。绑定后该规则**只**下发给该 DNAI 对应的辅锚点 UPF。属于动态/本地 PCC 的可选步骤，仅当规则需下发指定辅锚点时配置。

| 业务场景 | RULENAME 来源 | DNAI 来源 | 典型脚本 |
|---|---|---|---|
| 动态 PCC（5G，辅锚点 UPF） | ADD RULE 配的 RULENAME，且 RULERANGE=LOCAL | 全网规划的 DNAI（如 huawei.com） | `ADD RULEBINDDNAI:RULENAME="rule_test",DNAI="huawei.com";` |
| 本地 PCC（4G，辅锚点 PGW-U） | ADD RULE 配的 RULENAME，且 RULERANGE=LOCAL | 全网规划的 DNAI | `ADD RULEBINDDNAI:RULENAME="rule_test",DNAI="huawei.com";` |

**前置**：必须先用 [ADD RULE](task/UNC/20.15.2/0-00071.md) 创建规则，且该规则的 RULERANGE 必须为 LOCAL。绑定关系由本命令独立建立，RULE 本身不感知 DNAI。

## 决策点

### 是否配置本命令
问：预定义规则是否需要下发到指定辅锚点 UPF/PGW-U？
- **需要**：执行本命令绑定 RULE↔DNAI。前提是 ADD RULE 的 RULERANGE=LOCAL。
- **不需要**：跳过。RULERANGE=ALL 时 SMF 自动下发中心和所有边缘 UPF，绑 DNAI 无意义；RULERANGE=CENTRAL 时规则只下主锚点，不能绑任何边缘 DNAI。

### DNAI 取值来源
问：DNAI 取值如何确定？
- **全网规划**：DNAI 是网络拓扑标识（如 huawei.com），需与对端 UPF/PGW-U 的 DNAI 规划一致。证据中所有场景均标"全网规划"。

## 约束
- RULENAME 长度 1~63，不支持空格，不区分大小写。`severity=medium`
- DNAI 长度 1~63，不支持空格，不区分大小写。`severity=medium`
- 仅当 RULE 的 RULERANGE=LOCAL 时可绑定 DNAI；RULERANGE=ALL 绑定无意义；RULERANGE=CENTRAL 不能绑定。`severity=critical`
- 绑定后该预定义规则只下发给该 DNAI 对应的边缘 UPF。`severity=high`
- 最大记录数 10000。`severity=medium`
- 执行后立即生效。`severity=low`
