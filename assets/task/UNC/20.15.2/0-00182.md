# SET MASTERPCRF 原子任务
> 命令静态知识见 [SET MASTERPCRF](command/UNC/20.15.2/SET-MASTERPCRF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令为**可选**配置，仅当 PCRF 组工作模式为主备模式时才需修改缺省主用 PCRF。前置：PCRF 组与 PCRF-组绑定关系已通过 [ADD PCRFGROUP](task/UNC/20.15.2/0-00059.md)、[ADD PCRFBINDGRP](task/UNC/20.15.2/0-00058.md) 建好。两参均为必选，且 `MASTERPCRF` 必须是该组已绑定的 PCRF 之一。

| 配置目标 | 必备参数 | 典型脚本 |
|---|---|---|
| 修改某组缺省主用 PCRF | `PCRFGRPNAME`(已存在的组名) + `MASTERPCRF`(该组已绑定的 PCRF 主机名) | `SET MASTERPCRF:PCRFGRPNAME="pcrf_group_2",MASTERPCRF="pcrf_3";` |

**参数来源**：`PCRFGRPNAME` 取自已配置数据（必须先建 PCRFGROUP），`MASTERPCRF` 取自本端规划（必须是该组已通过 PCRFBINDGRP 绑定的成员）。

## 决策点

### PCRF 组工作模式是否为主备模式
问：是否需要执行本命令？
- 主备模式 → 需配置缺省主用 PCRF，本命令生效。
- 负载分担模式 → 无主备概念，本命令无意义，不执行。
- 工作模式由 [ADD PCRFGROUP](task/UNC/20.15.2/0-00059.md) 决定，本命令不自带该开关。

### 主用 PCRF 选择
问：该 PCRF 组内选哪台作主用？
- `MASTERPCRF` 必须是该组已通过 [ADD PCRFBINDGRP](task/UNC/20.15.2/0-00058.md) 绑定的 PCRF 成员之一，否则配置失败。
- 取值为本端规划主机名（如 `pcrf_3`）。

## 约束
- 执行后立即生效。`severity=low`
- 最大记录数 100。`severity=medium` 后果：超规格新增失败。
- `MASTERPCRF` 必须已通过 [ADD PCRFBINDGRP](task/UNC/20.15.2/0-00058.md) 绑定到该 `PCRFGRPNAME` 组。`severity=critical` 后果：配置失败或主用指向不存在的主机。
- 设定结果需通过 `LST PCRFGROUP` 查看（本命令无独立查询）。`severity=low`
- `PCRFGRPNAME`/`MASTERPCRF` 均不支持空格，不区分大小写。`severity=low`
