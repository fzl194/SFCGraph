# ADD CCT 原子任务

> 命令静态知识见 [ADD CCT](command/UNC/20.15.2/ADD-CCT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

CCT 是融合计费（Nchf）的参数模板，一台 SMF/PGW-C 上可建多条（最多 101，含初始记录）。配置时先用一个语义名建模板，再把流量/时长/事件三类阈值、配额有效时长、终结方式等填全。模板建好后通过 **3 种粒度**之一绑定生效：CC 粒度（`ADD SELECTCCTBYCC`）、DNN 粒度（`ADD APN`）、UserProfile 粒度（`SET USRPROFCHARGE`）。事件计费场景下可只给名字（其余走默认）。

**参数取值样例（融合计费特性配置范式）**

| 参数 | 取值样例 | 默认 | 说明 |
|---|---|---|---|
| CCTMPLTNAME | `cct_test` / `cct-test` | — | 模板名，1~63 字符，不区分大小写 |
| QHT | `0` | 0 | 配额空闲时间门限(秒)，0=不基于空闲触发上报 |
| VQT / TQT / UQT | `20` / `20` / `0` | 20/20/20 | 流量/时间/事件阈值触发百分比(%) |
| VT | `30` | 30 | RG 级在线配额有效时长(分)，超时 SMF 主动上报 CHF 申请新配额 |
| MAXNUMOFCCC | `3` | 3 | 计费条件改变几次触发 ChargingDataRequestUpdate |
| PDUVOLUMELIMIT / PDUTIMELIMIT | `500` / `30` | 500/30 | PDU 级流量阈值(MB) / 时长阈值(分钟) |
| RGVOLUMELIMIT / RGTIMELIMIT | `500` / `30` | 500/30 | RG（业务）级流量/时长阈值 |
| FUATERMINATE | `TERM_SERVICE` | TERM_SERVICE | 最终配额动作：阻塞业务 |
| TIMEFORMAT | `LocalTime` | LocalTime | 发给 CHF 消息的时间格式 |
| MAXSVCCONTAINER | `100` | 100 | N40 消息最大业务容器数，0=不限 |
| SECRUTHRESHOLD | `100` | 100 | RAN-SecondaryRAT-Usage-Report 上报 CHF 阈值 |

**典型行（完整融合计费模板）**

```
ADD CCT: CCTMPLTNAME="cct_test", QHT=0, VQT=20, TQT=20, UQT=0, VT=30, MAXNUMOFCCC=3,
  PDUVOLUMELIMIT=500, PDUTIMELIMIT=30, RGVOLUMELIMIT=500, RGTIMELIMIT=30,
  FUATERMINATE=TERM_SERVICE, TIMEFORMAT=LocalTime, MAXSVCCONTAINER=100, SECRUTHRESHOLD=100;
```

**事件计费极简行（仅命名，走默认）**

```
ADD CCT: CCTMPLTNAME="cct-test";
SET USRPROFCHARGE: USRPROFNAME="up-test", CCTEMPLATE="cct-test";
```

## 决策点

### 三类阈值的使能与取值

VQT/TQT/UQT 三个阈值百分比决定了用量上报的及时性，证据样例中 UQT 常取 `0`（关事件阈值）而 VQT/TQT 取 `20`。

| 选项 | VQT/TQT | UQT | 适用场景 |
|---|---|---|---|
| 流量+时间驱动上报 | 20 | 0 | 多数融合计费场景，按流量/时长门限触发 |
| 事件驱动上报 | 0 或小值 | 20 | 基于业务事件的计费（事件型业务） |

### 绑定粒度选择（建完模板后）

CCT 建好后选哪种粒度绑定，决定模板作用范围，由特性计费模型决定：

| 粒度 | 后续命令 | 适用 |
|---|---|---|
| CC 粒度 | `ADD SELECTCCTBYCC` | 按计费属性(CC)选不同模板 |
| DNN 粒度 | `ADD APN` | 一个 DNN 一套融合计费 |
| UserProfile 粒度 | `SET USRPROFCHARGE` | 按用户模板绑模板（事件计费常用此路） |

## 约束

- **立即生效** · severity=LOW。
- **最多 101 条记录**（含系统初始记录） · severity=MEDIUM · 超过上限新增失败。
- **VT 是 RG 级配额有效时长**：SMF 收 CHF 配额后按此启动定时器，超时即上报申请新配额 · severity=HIGH · 设过大会延迟用量上报，设过小会增加 N40 信令。
- **MAXSVCCONTAINER=0 表示不限**：延时上报场景下容器数可能膨胀，建议按消息大小设上限 · severity=MEDIUM。
- **SECRUTHRESHOLD 依赖前置开关**：仅当 `ADD N40MSGTEMP` 的 `SECRATUSAGE=ENABLE` 时才生效 · severity=MEDIUM。
- **UCITIMER 来自对端协商**：业务停止时长需与 CHF 侧约定 · severity=MEDIUM。
