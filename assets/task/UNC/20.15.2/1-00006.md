# CCT 绑定计费属性 CC

> 融合计费模板绑到 Charge Characteristic 层（优先级次于 APN）。被引用于 [2-00003 融合计费](task/UNC/20.15.2/2-00003.md)。特性专属（离线计费用 `ADD GLBOFCTEMPLATE` 绑 OFCTEMPLATE，见 [1-00004](task/UNC/20.15.2/1-00004.md)；融合用 `ADD SELECTCCTBYCC` 绑 CCT，绑定命令不同故新建）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 建计费属性（已配可跳过） | `ADD CHARGECHAR` | CCNAME（本端规划）；HOME/ROAM/VISIT（CC 取值，如 0x0800 normal/0x0400 prepaid/0x100 hotbilling）；HOMESGSN/ROAMSGSN/VISITSGSN（是否用 UDM 下发，DISABLE=用本地） |
| CC 绑 CCT 模板 | `ADD SELECTCCTBYCC` | CCTYPE=VALUE；CCVALUE（与 CHARGECHAR 取值对应，带掩码）；CCTMPLTNAME=模板名 |

**典型脚本**：
```
ADD CHARGECHAR: CCNAME="testchgcha", HOME="0x0800", ROAM="0x0800", VISIT="0x0800", HOMESGSN=DISABLE, ROAMSGSN=DISABLE, VISITSGSN=DISABLE;
ADD SELECTCCTBYCC: CCTYPE=VALUE, CCVALUE="0x0800", CCTMPLTNAME="cct_test";
```

**步骤位置**：紧跟 [1-00005](task/UNC/20.15.2/1-00005.md)（CCT 模板参数）之后；仅当特性级 DP 选 CC 层时编排。

## 约束

- **前置模板**（critical）：CCTMPLTNAME 须已在 [1-00005](task/UNC/20.15.2/1-00005.md) 建立 — 否则 `ADD SELECTCCTBYCC` 绑定失败。
- **CC 取值与 CHARGECHAR 一致**（critical）：`ADD SELECTCCTBYCC` 的 CCVALUE 须与 `ADD CHARGECHAR` 的 HOME/ROAM/VISIT 取值对应（按掩码匹配）— 否则 CC 选不到 CCT 模板。
- **HOMESGSN/ROAMSGSN/VISITSGSN**（warning）：DISABLE=用本地配置 CC；ENABLE=用 UDM 下发 CC（此时本地 CC 配置不生效）。
