# 配置PLMN计费属性参数（SET CHGPLMNCHAR）

> 命令静态知识见 [SET CHGPLMNCHAR](command/UNC/20.15.2/SET-CHGPLMNCHAR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CHGPLMNCHAR 是 SGSN 侧**按 PLMN 类型粒度的话单生成全局总闸**：以 `PLMN` 类型（HPLMN 本网 / VPLMN 拜访）为主键，全局开关 7 类话单（M-CDR/S-CDR/S-SMO-CDR/S-SMT-CDR/LCS-MO-CDR/LCS-MT-CDR/LCS-NI-CDR）。本命令是话单抑制的**最高优先级总闸**：一旦本命令禁止生成某种话单，无论 [ADD CHGPLMNCFG](task/UNC/20.15.2/0-00021.md) / [ADD CHGIMSICFG](task/UNC/20.15.2/0-00020.md) / [SET CHGCHAR](task/UNC/20.15.2/0-00160.md) 如何配置，该话单都不生成。本命令核心配置空间有 2 个维度：**PLMN 类型主键**（PLMN 必选）、**7 类话单生成开关**（MP/SP/SMOP/SMTP/LCSMOP/LCSMTP/LCSNIP）。

### 配置维度 1：PLMN 类型主键（参数 PLMN）

`PLMN` 为必选主键，决定本次配置作用于本网用户还是漫游用户。数据来源整网规划：

| 参数 | 控制对象 | 取值 | 系统初始值 |
|---|---|---|---|
| `PLMN` | PLMN 类型（主键） | `HPLMN`（本地 PLMN，本网用户）/ `VPLMN`（拜访 PLMN，漫游用户） | — |

> 同一 PLMN 类型一条记录；非本网用户即拜访用户。漫游控制场景必选 `VPLMN`（证据 WSFD-011001）。

### 配置维度 2：7 类话单生成开关（参数 MP/SP/SMOP/SMTP/LCSMOP/LCSMTP/LCSNIP）

| 参数 | 控制 CDR 类型 | 取值 | 系统初始值（HPLMN/VPLMN） |
|---|---|---|---|
| `MP` | 生成 M-CDR（移动管理话单） | `YES`/`NO` | `YES` / `YES` |
| `SP` | 生成 S-CDR（会话话单） | `YES`/`NO` | `YES` / `YES` |
| `SMOP` | 生成 S-SMO-CDR（短消息主叫） | `YES`/`NO` | `YES` / `YES` |
| `SMTP` | 生成 S-SMT-CDR（短消息被叫） | `YES`/`NO` | `YES` / `YES` |
| `LCSMOP` | 生成 LCS-MO-CDR（定位主叫） | `YES`/`NO` | `YES` / `YES` |
| `LCSMTP` | 生成 LCS-MT-CDR（定位被叫） | `YES`/`NO` | `YES` / `YES` |
| `LCSNIP` | 生成 LCS-NI-CDR（定位网络触发） | `YES`/`NO` | `YES` / `YES` |

> 话单抑制优先级（仅对 S-CDR 举例）：本命令（全局总闸）> [ADD CHGPLMNCFG](task/UNC/20.15.2/0-00021.md)（PLMN + APN NI）> [ADD CHGIMSICFG](task/UNC/20.15.2/0-00020.md)（IMSI 前缀 + APN NI + 拜访类型 + 计费属性）。[SET CHGCDR](task/UNC/20.15.2/0-00159.md) 的 `PLMNCTRL` 控制话单生成策略基于哪种 PLMN 来源（IMSI/LOCINFO）。

典型脚本（漫游 VPLMN 用户抑制 LCS-NI 定位话单，其余默认 YES）：
```
SET CHGPLMNCHAR: PLMN=VPLMN, LCSNIP=NO;
```
漫游 VPLMN 用户开启全部 7 类话单：
```
SET CHGPLMNCHAR: PLMN=VPLMN, MP=YES, SP=YES, SMOP=YES, SMTP=YES, LCSMOP=YES, LCSMTP=YES, LCSNIP=YES;
```
全局 S-CDR 生成（供下层 PLMN+APN NI / IMSI 精细策略生效）：
```
SET CHGPLMNCHAR: PLMN=VPLMN, SP=YES;
```
本网用户抑制 S-SMO-CDR（短消息主叫）：
```
SET CHGPLMNCHAR: PLMN=HPLMN, SMOP=NO;
```

**步骤位置**：在所属 feature 编排中通常位于"配置全局 PLMN 话单总闸"步骤——细粒度 PLMN/IMSI 话单策略 [ADD CHGPLMNCFG](task/UNC/20.15.2/0-00021.md) / [ADD CHGIMSICFG](task/UNC/20.15.2/0-00020.md) 之前（总闸先开，精细策略后配）；与全局 CDR 参数 [SET CHGCDR](task/UNC/20.15.2/0-00159.md) / CC 级 [SET CHGCHAR](task/UNC/20.15.2/0-00160.md) 平级或先于它们。

## 决策点

### 决策点：PLMN 类型选择（DP 0-03065）

问：配置本网用户（HPLMN）还是漫游用户（VPLMN）的话单生成属性？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 本地 PLMN（本网用户） | `PLMN=HPLMN` + 按本网运营策略开关 7 类话单 | 本网用户计费属性（通常全部 YES） |
| 拜访 PLMN（漫游用户） | `PLMN=VPLMN` + 按漫游运营策略开关 7 类话单 | 漫游用户计费属性（常抑制个别话单如 LCS-NI）；漫游控制场景必选 |

> 取值由整网规划决定；同一 PLMN 类型一条记录。漫游控制场景（证据 WSFD-011001）必选 `VPLMN`。

### 决策点：7 类话单生成开关（DP 0-03066）

问：该 PLMN 下 7 类话单（M-CDR/S-CDR/S-SMO/S-SMT/LCS-MO/MT/NI）分别生成还是抑制？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 默认全开（7 类全 YES） | MP/SP/SMOP/SMTP/LCSMOP/LCSMTP/LCSNIP=YES | 初始设置，全部话单正常生成 |
| 抑制定位话单（LCS-NI） | `LCSNIP=NO`（其余 YES） | 漫游用户不生成网络触发定位话单 |
| 抑制短消息话单 | `SMOP=NO` 或 `SMTP=NO` | 不计短消息主叫/被叫话单 |
| 全局开 S-CDR（精细策略前置） | `SP=YES` | 让下层 PLMN+APN NI / IMSI 精细 S-CDR 策略可生效 |

> 本命令是总闸：抑制后无法被下层细粒度命令（[ADD CHGPLMNCFG](task/UNC/20.15.2/0-00021.md) / [ADD CHGIMSICFG](task/UNC/20.15.2/0-00020.md) / [SET CHGCHAR](task/UNC/20.15.2/0-00160.md)）覆盖。全局 S-CDR 策略生效前提是本命令 `SP=YES`，否则三种 S-CDR 细分策略都不生效。

## 约束

- **总闸优先：抑制后下层无效**（critical，rule-0-03161）：本命令禁止生成某种话单后，无论 [ADD CHGPLMNCFG](task/UNC/20.15.2/0-00021.md) / [ADD CHGIMSICFG](task/UNC/20.15.2/0-00020.md) / [SET CHGCHAR](task/UNC/20.15.2/0-00160.md) 如何配置都不生成 — 后果：总闸抑制不可被下层覆盖
- **全局 S-CDR 前置**（high，rule-0-03162）：全局 S-CDR 策略生效前提是本命令开启 `SP=YES`，否则三种 S-CDR 细分策略（PLMN+APN NI / IMSI 前缀+APN NI+拜访类型+计费属性）都不生效 — 后果：SP=NO 时 S-CDR 细分策略失效
- **话单抑制优先级链**（warning，rule-0-03163）：话单优先级（S-CDR 举例）：本命令（全局总闸）> [ADD CHGPLMNCFG](task/UNC/20.15.2/0-00021.md) > [ADD CHGIMSICFG](task/UNC/20.15.2/0-00020.md)；[SET CHGCDR](task/UNC/20.15.2/0-00159.md) 的 `PLMNCTRL` 决定话单策略基于哪种 PLMN 来源 — 后果：本命令优先级最高
- **系统初始设置值**（info，rule-0-03164）：系统初次运行执行系统初始设置值，HPLMN/VPLMN 初始 7 类话单均为 `YES` — 后果：未显式配置时全部话单生成
- **立即生效**（info，rule-0-03165）：本命令执行后立即生效 — 后果：无需重启
