# 配置全局计费CDR参数（SET CHGCDR）

> 命令静态知识见 [SET CHGCDR](command/UNC/20.15.2/SET-CHGCDR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CHGCDR 是 SGSN 侧**全局 CDR 配置命令**，影响 S-CDR/M-CDR/S-SMO/S-SMT/LCS 各类话单的字段产生、缺省计费属性、PLMN ID 来源、强制话单生成等。计费 CDR 参数由系统缺省增加（初次运行有初始设置值）。话单字段优先级：CC 级（[SET CHGCHAR](task/UNC/20.15.2/0-00160.md)）> 全局级（本命令）> HLR 签约。本命令参数众多，核心配置空间有 3 个维度：**话单内容定制选项**（5 类 CDR 字段勾选）、**PLMN 控制策略**（话单/CG PLMN 来源）、**缺省计费属性与强制话单**（本地/拜访/漫游缺省 CC + 强制生成）。

### 配置维度 1：话单内容定制选项（参数 ML / SMOL / SMTL / SL / LCS*）

每类 CDR 通过选项列表勾选携带字段，数据来源整网规划：

| 参数 | 控制 CDR 类型 | 取值示例 | 系统初始值 |
|---|---|---|---|
| `ML` | M-CDR（移动管理话单） | `SERVED_IMEI` / `SGSN_ADDR` / `MS_NETWORK_CAPABILITY` / `RAC` / `LAC` / `CI` / `CHANGE_OF_LOCATION` / `DURATION` / `DIAGNOSTICS` / `NODE_ID` / `SERVED_MSISDN` / `SYSTEM_TYPE` / `CC_SELECT_MODE` / `CELL_PLMN_ID` | 全部选中 |
| `SMOL` | S-SMO-CDR（短消息主叫） | `SERVED_IMEI` / `SERVED_MSISDN` / `SC` / `RECORDING_ENTITY` / `LAC` / `RAC` / `NODE_ID` / `DEST_NUM` / `RECORD_EXTENTIONS` 等 | 全部选中，除 `RECORD_EXTENTIONS` |
| `SMTL` | S-SMT-CDR（短消息被叫） | 同 SMOL | 全部选中，除 `RECORD_EXTENTIONS` |
| `SL` | S-CDR（会话话单） | `NETWORK_INITIATED_PDP` / `SERVED_IMEI` / `SGSN_ADDR` / `APNNI` / `PDP_TYPE` / `SERVED_PDP_ADDR` / `LIST_OF_TDV` / `DIAGNOSTICS` / `NODE_ID` / `APN_SELECT_MODE` / `APNOI` / `SERVED_MSISDN` / `RNC_UDV` / `CC_SELECT_MODE` / `CELL_PLMN_ID` / `RECORD_EXTENTIONS` 等 | 全部选中，除 `CELL_PLMN_ID` |
| `LCSMTL` / `LCSMOL` / `LCSNIL` | LCS-MT/MO/NI-CDR（定位话单） | `SERVED_MSISDN` / `SGSN_ADDR` / `MEASUREMENT_DURATION` / `LOCATION` / `LOCATION_ESTIMATE` / `LCS_CAUSE` 等 | 全部选中 |

> `NODEID` 参数（SGSN 节点标识，≤19 字符串）控制上述各 CDR 中 `NODE_ID` 字段是否出现；初始值 `InvalidNodeID`。`SERVED_IMEI` 字段携带时还需 [ADD GBIMEICFG（★R1.5 atom 待补）] / [ADD IUIMEICFG（★R1.5 atom 待补）] 配置 IMEI 获取策略不为 NO，话单才含 IMEI。

### 配置维度 2：PLMN 控制策略（参数 PLMNCTRL / CDRPLMNSELECT / CGPLMNSELECT / NONSUPPUEPLMN）

4 个参数控制 PLMN ID 的不同获取来源，数据来源整网规划：

| 参数 | 控制对象 | 取值 | 系统初始值 | 联动配置 |
|---|---|---|---|---|
| `PLMNCTRL` | PLMN ID 控制策略（话单生成维度） | `LOCINFO`（位置信息 PLMN）/ `IMSI`（IMSI 中 PLMN） | `IMSI` | LOCINFO 时 [ADD CHGPLMNCFG] 生效；IMSI 时 [ADD CHGIMSICFG] 生效 |
| `CDRPLMNSELECT` | 话单填写 PLMN ID 选择策略（S-CDR） | `LOCINFO` / `IMSI` | `LOCINFO` | 同上 |
| `CGPLMNSELECT` | 话单发送 PLMN ID 选择策略（选 CG） | `LOCINFO` / `IMSI` | `LOCINFO` | 同上 |
| `NONSUPPUEPLMN` | MOCN 中 Non-supporting UE 的 PLMN 获取策略 | `COMMON_PLMN`（公共 PLMN）/ `SRV_PLMN`（服务 PLMN） | `SRV_PLMN` | 仅 PLMNCTRL/CDRPLMNSELECT/CGPLMNSELECT=LOCINFO 时遵从；需 WSFD-207001 MOCN 特性启动 |

> `EXTRANGE`（PLMNCG 范围扩展，YES 允许非 PLMNCG / NO 禁止，初始 YES）：控制 PLMN CG 全故障时 S-CDR 是否发往非 PLMN CG。

### 配置维度 3：缺省计费属性与强制话单（参数 HCC/VCC/RCC/VP + FRCGEN/FRCTIME + 其他）

| 参数 | 作用 | 取值 | 系统初始值 |
|---|---|---|---|
| `HCC` / `VCC` / `RCC` | 本地/拜访/漫游用户缺省计费属性 | `NORMAL` / `PREPAID` / `FLATRATE` / `HOTBILLING` | 均为 `NORMAL` |
| `VP` | 缺省非本地用户类型 | `ROAMING`（漫游用户）/ `VISITING`（拜访用户） | `ROAMING` |
| `IGNOREFLG` | 忽略签约计费属性 | `HOME_SUBSCRIBER` / `ROAM_SUBSCRIBER` / `VISIT_SUBSCRIBER` / `HOME_APN` / `ROAM_APN` / `VISIT_APN`（可多选） | 全部清空 |
| `LRSNP` | 本地话单序列号 | `PRESENT`（有）/ `ABSENT`（无） | `PRESENT` |
| `CRC` | 话单关闭原因值 | `NORMAL`（正常释放）/ `ABNORMAL`（异常释放） | `NORMAL` |
| `CRQOS` | 保持请求 QoS（S-CDR 中 Request QoS 填充） | `YES`（保持原始值）/ `NO`（用签约值替代 0 值） | `NO` |
| `FRCGEN` | 强制生成话单方式 | `SCDRONLY` / `MCDRONLY` / `ALL` / `NONE` | `NONE` |
| `FRCTIME` | 强制生成话单时间（FRCGEN 配套，条件可选） | `00&00&00`~`23&59&59`（小时&分钟&秒） | 无 |

> FRCGEN 配置为 MCDRONLY/SCDRONLY/ALL 时 FRCTIME 才生效。

典型脚本（M-CDR 选项选 SERVED_IMEI，本地用户缺省计费属性选 HOTBILLING）：
```
SET CHGCDR: ML=SERVED_IMEI-1, HCC=HOTBILLING;
```
MOCN 网络共享场景，Non-supporting UE 用 Serving PLMN：
```
SET CHGCDR: NONSUPPUEPLMN=SRV_PLMN;
```
话单可靠性（生成本地话单序列号）：
```
SET CHGCDR: LRSNP=PRESENT;
```

**步骤位置**：在所属 feature 编排中通常位于"配置全局 CDR 参数"步骤——CC 级 [SET CHGCHAR](task/UNC/20.15.2/0-00160.md) 之前（全局缺省先于 CC 级覆盖），PLMN 计费属性策略 [SET CHGPLMNCHAR] 之后。

## 决策点

### 决策点：PLMN ID 控制策略（DP 0-03057）

问：话单/CG 发送时 PLMN ID 从哪里取，决定 S-CDR 话单生成策略的匹配维度？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 位置信息 PLMN（LOCINFO） | PLMNCTRL/CDRPLMNSELECT/CGPLMNSELECT=LOCINFO | 话单策略基于 PLMN + APN NI 共同配置；[ADD CHGPLMNCFG] 生效 |
| IMSI 前 PLMN（IMSI） | PLMNCTRL/CDRPLMNSELECT/CGPLMNSELECT=IMSI | 话单策略基于 IMSI 前缀 + APN NI + 拜访类型 + 计费属性共同配置，可精细抑制特定号段漫游用户；[ADD CHGIMSICFG] 生效 |

> 仅当全局策略 [SET CHGPLMNCHAR] 设为生成 S-CDR 时，上述策略才生效（见约束 rule-0-03143）。

### 决策点：MOCN Non-supporting UE PLMN 获取（DP 0-03058）

问：网络共享（MOCN）下，不能解析共享信息的 Non-supporting UE，话单/Gn 用哪个 PLMN？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| Serving PLMN（SRV_PLMN） | NONSUPPUEPLMN=SRV_PLMN + 配合 `ADD HNOSRVPLMN（★R1.5 atom 待补）` 指定 Serving PLMN | 配合归属网络 Serving PLMN 管理，替换 Common PLMN 填入话单，解决 Billing Center 无法识别话单问题；一般运营商期望基于 IMSI 为 Non-supporting UE 指定 Serving PLMN，便于网间计费结算 |
| Common PLMN（COMMON_PLMN） | NONSUPPUEPLMN=COMMON_PLMN | 直接用广播的 Common PLMN |

> 视 Billing Center 是否归属本网决定。本参数需 WSFD-207001 MOCN 特性启动时生效；若 Byte_Ex63 Bit1 软参值为 1，则 SRV_PLMN 不生效。

## 约束

- **系统初始设置值**（info，rule-0-03141）：系统初次运行即有初始设置值；CDR 参数由系统缺省增加 — 后果：未显式配置时走初始值
- **立即生效**（info，rule-0-03142）：执行后立即生效 — 后果：无需重启
- **PLMN 策略前置依赖**（warning，rule-0-03143）：PLMNCTRL 相关话单生成策略仅在 [SET CHGPLMNCHAR](task/UNC/20.15.2/0-00163.md) 全局策略设为生成 S-CDR 时生效 — 后果：全局策略未配 S-CDR 时本命令 PLMN 策略无效
- **计费属性优先级链**（warning，rule-0-03144）：S-CDR 计费属性选择优先级为 IMSI 匹配的 SPEC 配置 > APN 签约 > 用户签约 > [ADD CHGDCHAR] 缺省 > 本命令（全局缺省）；CC 级 [SET CHGCHAR](task/UNC/20.15.2/0-00160.md) 配置优先级高于本命令 — 后果：本命令仅作缺省兜底
- **话单序列号固定值**（info，rule-0-03145）：话单可靠性场景 LRSNP 固定选 `PRESENT` — 后果：`ABSENT` 导致话单无本地序列号，影响话单排序与去重
