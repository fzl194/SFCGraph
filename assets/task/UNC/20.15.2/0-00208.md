# SET ZEROCHGSKIPSW 原子任务
> 命令静态知识见 [SET ZEROCHGSKIPSW](command/UNC/20.15.2/SET-ZEROCHGSKIPSW.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令为全局单例（最大记录数 1），用一个总开关 + 13 个细分子开关控制"零流量计费事件是否忽略"。属计费精细化调优项，开启忽略会减少冗余话单，但可能丢失 RAT/ULI 等伴随信息，需确认影响。

## 决策点

### 总开关与子开关组合
- 问什么：总开关打开后，哪些子事件保留话单、哪些忽略。
- 总开关 `ENABLE` 后，各 `SKIP*` 子开关逐项决定：`ENABLE`=忽略该类事件（不出话单）、`DISABLE`=保留该类事件（出话单）。
- 影响权衡：忽略零流量计费事件会导致部分伴随信息缺失（RAT 更新、ULI 改变等），需确认业务对位置/QoS 变更等信息的依赖后再裁剪。

### SKIPULIBASEDCHG 无效场景
- 问什么：基于位置的计费订阅和取消是否忽略。
- UNC 当前**暂不支持**基于位置的计费订阅和取消，故 `SKIPULIBASEDCHG` 无实际生效场景。
- 建议：建 atom 时标注无效，避免误配。

## 约束
- 生效前置：本命令仅在软参 **DWORD519 BIT4=1** 时生效，否则不生效。`severity=high`
- 生效时机：执行后立即生效。`severity=low`
- 单例：最大记录数 1，全局唯一。`severity=low`
- 信息缺失风险：开启忽略会丢失 RAT/ULI 等伴随计费信息，需确认影响后使能。`severity=high`
- 无效子开关：SKIPULIBASEDCHG 在 UNC 无实际生效场景。`severity=medium`
- 存在系统初始记录，参数有初始默认值。`severity=low`
