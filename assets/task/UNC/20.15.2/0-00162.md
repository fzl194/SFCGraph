# SET CHGMODE 原子任务
> 命令静态知识见 [SET CHGMODE](command/UNC/20.15.2/SET-CHGMODE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
SET CHGMODE 是计费接口选择三级优先级中的**最低级全局兜底**：优先级 `ADD ROAMCHGMODE`（基于用户漫游属性）> `ADD APNCHGMODE`（基于 APN/DNN）> `SET CHGMODE`（全局）。以"终端类型 × 接入类型"（`TMACCTYPE`）为粒度配置计费接口，按终端/接入组合分别下发命令。`TMACCTYPE` 为必选且取值由本端规划（如 `UE5G_RAT4G`/`UE5G_RAT5G`），其余 `FORCED`/`FORSGWONLY`/`FORVSMFONLY`/`BY5GSIWKI`/`BYIWKUEN5GR4G`/`BY5GCNRI`/`ISMFCHGSW` 为 conditional（按场景出现）。

| 终端/接入组合 | 关键参数 | 典型脚本 |
|---|---|---|
| 5G 终端 4G 接入（融合计费） | `TMACCTYPE=UE5G_RAT4G` + `FORCED=NchfMode` | `SET CHGMODE:TMACCTYPE=UE5G_RAT4G,BY5GSIWKI=False,FORCED=NchfMode;` |
| 5G 终端 4G 接入（GaGy 计费） | `TMACCTYPE=UE5G_RAT4G` + `FORCED=GaGyMode` | `SET CHGMODE:TMACCTYPE=UE5G_RAT4G,BY5GSIWKI=False,FORCED=GaGyMode;` |
| 5G 终端 5G 接入（V-SMF 漫游） | `TMACCTYPE=UE5G_RAT5G` + `FORVSMFONLY=GaGyMode` | `SET CHGMODE:TMACCTYPE=UE5G_RAT5G,FORVSMFONLY=GaGyMode;` |
| SGW 独立部署（指定 SGW 计费接口） | `FORSGWONLY=GaGyMode`（与上述组合联用） | 见组合脚本 |

**计费接口选择三级优先级**：`ADD ROAMCHGMODE` > `ADD APNCHGMODE` > `SET CHGMODE`。SGW 独立部署场景通过三级命令中任一的 `FORSGWONLY` 指定 SGW 发起计费会话时的接口。

## 决策点

### 计费接口选择（GaGy vs Nchf）
问：该终端/接入组合走 GaGy 接口还是 N40/Nchf 接口？
- `FORCED=GaGyMode`：传统离线(Ga)+在线(Gy)计费。
- `FORCED=NchfMode`：5G 融合计费 N40 接口（需先与对端 CHF 确认支持）——配置后需进一步用 [LST CHARGECTRL](task/UNC/20.15.2/0-00160.md) 等检查用户是否使能 N40 融合计费（证据 WSFD-011206）。
- 初选结果可被 [ADD PCCCHGMODEBYPCFID](task/UNC/20.15.2/0-00050.md) 基于 PCF 实例标识二次调整（N40↔GaGy 回落/升级）。

### 是否按互操作标识选择接口
问：跨制式互操作时是否按互操作指示选接口？
- `BY5GSIWKI`：按 S5/S8 接口的 "5GS Interworking Indication" 取值选择（5G 终端跨制式场景）。
- `BYIWKUEN5GR4G`：非 5G 终端 4G 接入时按 5GS 互操作指示选择。
- `BY5GCNRI`：按 5GC 无限制接入标识选择（local_planned）。

## 约束
- 配置计费接口选择方式不当可能选错计费策略接口，相应接口未配置会导致计费流程失败（如用户无法正常激活）。`severity=critical`
- `TMACCTYPE` 为 required，每次执行必填；其余 conditional 参数不输入时保持当前值不变（可通过 LST CHGMODE 查询）。`severity=medium`
- 2G 接入使用 Nchf 接口需提前与对端 CHF 确认是否支持。`severity=high`
- 5G 终端 5G 接入配置 `gagy` 模式需开启特定 License（提前联系华为技术支持确认）。`severity=high`
- 4G/5G 互操作流程的计费接口不变；2G/3G/4G 互操作流程的计费接口不变。`severity=medium`
- 该命令执行后立即生效。`severity=low`
