# 修改接口（MOD INTERFACE）

> 命令静态知识见 [MOD INTERFACE](command/UNC/20.15.2/MOD-INTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令修改已创建接口（逻辑或物理）的属性。配置空间核心维度为 **管理状态**（IFADMINSTATUS）和 **MTU/DF/统计组合**（IFMTU/IFDF/IFSTATIENABLE/IFSTATITVL）。

### 配置维度1：管理状态切换（参数 IFADMINSTATUS）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| up | 接口管理 Up（启用） | 接口投入使用（IPFD-016000 大量 MOD 把物理口/LoopBack 切 up） |
| down | 接口管理 Down（禁用） | 维护/故障屏蔽 |

### 配置维度2：MTU/DF/统计组合（参数 IFMTU / IFDF / IFSTATIENABLE / IFSTATITVL / IFTRAPENABLE / IFDESCR）

| 组合 | 作用 | 典型场景 |
|---|---|---|
| 仅 IFADMINSTATUS=up | 仅启用接口，其他保持默认 | 简单启用（IPFD-016000 Ethernet66/0/5/6 仅切 up） |
| + IFMTU=1500 | 设 MTU 1500 | 标准 MTU（IPFD-016000 部分场景） |
| + IFMTU + IFDF=FALSE + IFSTATIENABLE=TRUE + IFSTATITVL=20 | MTU + 允许分片 + 启用统计 + 20s 周期 | 完整调优（IPFD-016000 Ethernet66/0/3/5） |
| + IFDESCR | 加接口描述 | 标识用途 |

**步骤位置**：在地址分配/接口管理族编排中，于 `ADD INTERFACE`（建逻辑接口）之后执行，用于把新建接口或物理口切到 Up 并调优参数；通常在配 IP（`ADD IFIPV4ADDRESS`/`ADD IFIPV6ADDRESS`）前后均可。

## 决策点

**修改场景选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅启用 | IFADMINSTATUS=up | 接口投入使用（最常见） |
| 启用 + MTU 调优 | + IFMTU=1500 | 需统一 MTU |
| 启用 + 完整调优 | + IFMTU + IFDF=FALSE + IFSTATIENABLE=TRUE + IFSTATITVL | 需统计监控（IPFD-016000 IPSec 物理口） |

> IFADMINSTATUS=down 用于维护时禁用接口。

## 约束

- **接口必须先建**（critical）：IFNAME 必须由 `ADD INTERFACE` 已配置（逻辑接口）或为已有物理接口 — 后果：接口不存在则修改失败
- **IFNAME 命名约束**（warning）：字符串 1~63 — 后果：违规命名修改失败
- **IFMTU 范围**（warning）：取值由设备规格决定 — 后果：超规格修改失败
- **IFSTATITVL 范围**（warning）：整数 5~600，单位秒 — 后果：超范围修改失败
- **IFDESCR 长度**（warning）：字符串 1~24 — 后果：超长修改失败
