# SET LICENSESWITCH 原子任务
> 命令静态知识见 [SET LICENSESWITCH](command/UNC/20.15.2/SET-LICENSESWITCH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

SET LICENSESWITCH 是平台级的 License 控制项开关命令。当 License 文件中含有某功能的许可时，通过此命令设置该功能是否开通。该命令被 **213 个特性**引用——几乎所有特性激活流程的第一步都是"打开本特性的 License 配置开关"。系统部署后已存在大量初始记录（仅前 20 条展示）。

## 配置方法
每个特性对应一个特定的 License 控制项（LICITEM），激活时一律 SWITCH=ENABLE。LICITEM 取值（License 控制项编码，如 LKV2xxxxxx）由上游特性决定，不在本命令内部能推导。

| 参数 | 描述 | 配置方法 | 取值样例 |
|---|---|---|---|
| LICITEM | License 项 | 本端规划（特性→LICITEM 映射，见激活 md） | `LKV2IPV601` / `LKV2M2MS01` / `LKV2CELP02` 等 |
| SWITCH | 开关 | 本端规划（激活=ENABLE） | `ENABLE` / `DISABLE` |

典型脚本（激活 IPv6 承载上下文特性，含 AMF/SMF 两侧）：
```
SET LICENSESWITCH: LICITEM="LKV2IPV6AM01",SWITCH=ENABLE;
SET LICENSESWITCH: LICITEM="LKV2IPV6SM01",SWITCH=ENABLE;
```
典型脚本（激活 eMTC 终端省电 PSM 模式）：
```
SET LICENSESWITCH: LICITEM="LKV2M2MS01", SWITCH=ENABLE;
```
典型脚本（激活 5G LAN 跨 SMF 群组管理）：
```
SET LICENSESWITCH: LICITEM="LKV2CUFMN01",SWITCH=ENABLE;
```

## 决策点

### LICITEM 取值（特性→License 控制项映射）
- 问什么：当前要激活的特性对应哪个（或哪几个）LICITEM。
- 取值完全由上游特性决定，跨场景枚举（213 个特性各有自己的 LICITEM）；同一特性在 AMF/SMF/MME/SGW-C/PGW-C 等不同网元角色上可能需开多个 LICITEM（如 WSFD-104001 在 AMF/SMF 侧开 LKV2IPV6AM01+LKV2IPV6SM01）。
- 数据规划表"获取方法"两值并存（本端规划 / 全网规划），按是否首次部署分叉。

### SWITCH 取值
- 问什么：开通还是关闭。
- 激活流程一律 ENABLE；关闭特性时 DISABLE。多数激活 md 默认 ENABLE。

## 约束
- 唯一性：每个 LICITEM 一条记录（系统部署后已存在初始记录）。`severity=low`
- 生效时机：执行后立即生效。`severity=high`
- 前置（License 文件激活）：License 文件未激活时无法设置任何 license 项；激活后只能设置**已购买且支持配置开关**的 license 项。`severity=high`
- 前置（购买）：LICITEM 必须在当前 License 文件中已购买且支持配置开关，否则设置失败。`severity=high`
- 验证惯例：激活前一般先 LST LICENSESWITCH 查 SWITCH 是否已 ENABLE，已开则跳过。`severity=low`
- 适用 NF：平台级（UNC 通用）。`severity=low`
