# 配置License项开关（SET LICENSESWITCH）

> 命令静态知识见 [SET LICENSESWITCH](command/UNC/20.15.2/SET-LICENSESWITCH.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET LICENSESWITCH 是平台级的 License 控制项开关命令。当 License 文件中含有某功能的许可时，通过此命令设置该功能是否开通。该命令被 **213 个特性**引用——几乎所有特性激活流程的第一步都是"打开本特性的 License 配置开关"。系统部署后已存在大量初始记录（多数 DISABLE，仅前 20 条展示）。本命令有 2 个配置维度：**LICITEM 取值（特性→License 控制项映射）**、**SWITCH 开关取值**。

### 配置维度 1：LICITEM 取值（参数 LICITEM，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| License 项 | `LICITEM` | 字符串 0~255 字符（如 `LKV2IPV601`/`LKV2M2MS01`/`LKV2CELP02`） | 无 | 指定要开通的 License 控制项 | 由上游特性决定，非命令内部能推导 |

> 数据来源：本端规划。LICITEM 取值（License 控制项编码）由上游特性决定，每个特性对应一个或多个 LICITEM；同一特性在 AMF/SMF/MME/SGW-C/PGW-C 等不同网元角色上可能需开多个 LICITEM。代表性映射（共 213 特性，下表为样例）：

| 特性 | LICITEM |
|---|---|
| WSFD-225002 5G LAN 跨 SMF 群组管理 | LKV2CUFMN01 |
| WSFD-216001 eMTC 终端省电 PSM 模式 | LKV2M2MS02 |
| WSFD-216002 eMTC eDRX 模式（MME） | LKV2EDRX02 |
| WSFD-216102 eMTC 基于 eNodeB 覆盖等级寻呼 | LKV2CELP02 |
| WSFD-104001 IPv6 承载上下文（AMF/SMF） | LKV2IPV6AM01 / LKV2IPV6SM01 |
| WSFD-011601 NB-IoT 终端标准接入（MME） | LKV2NBPS01 / LKV2NBFQ01 / LKV2NBFQ02 |
| WSFD-215001 终端省电 PSM 模式 | LKV2M2MS01 |
| WSFD-108002 基于预定义规则的分流策略控制 | LKV2PCLBO01 |
| WSFD-108004 双主负荷分担部署模式 | LKV2DMECFP01 |
| WSFD-108005 MEC 单点模式故障保护 | LKV2SMECFP01 |

> 完整 213 特性 → LICITEM 映射见各特性激活 md（证据包已抽取）。

### 配置维度 2：SWITCH 开关取值（参数 SWITCH，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 开关 | `SWITCH` | `ENABLE`/`DISABLE` | 无 | License 控制项是否开通 | 激活流程一律 ENABLE；关闭特性时 DISABLE |

> 数据来源：本端规划。多数激活 md 默认 ENABLE。

**典型脚本**

```
# 激活 IPv6 承载上下文特性（AMF/SMF 两侧）
SET LICENSESWITCH: LICITEM="LKV2IPV6AM01", SWITCH=ENABLE;
SET LICENSESWITCH: LICITEM="LKV2IPV6SM01", SWITCH=ENABLE;

# 激活 eMTC 终端省电 PSM 模式
SET LICENSESWITCH: LICITEM="LKV2M2MS01", SWITCH=ENABLE;

# 激活 5G LAN 跨 SMF 群组管理
SET LICENSESWITCH: LICITEM="LKV2CUFMN01", SWITCH=ENABLE;
```

**步骤位置**：几乎每个特性激活 md 的第 1 步都引用本命令——是 213 个特性激活流程的**首步前置**（开通本特性的 License 控制项），位于具体特性配置（如 PCC 基本功能、FUP、接入点策略等）之前。

## 决策点

### 决策点：LICITEM 取值选择（DP 0-03099）

问：当前要激活的特性对应哪个（或哪几个）LICITEM？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单 LICITEM 单网元 | `LICITEM` 取该特性的单个控制项编码 | 特性仅需一个 License 项（如 WSFD-215001） |
| 多 LICITEM 多网元 | 多条命令，分别取 AMF/SMF/MME/SGW-C/PGW-C 各角色的控制项 | 同一特性跨网元角色需开多个（如 WSFD-104001 在 AMF/SMF 侧开 LKV2IPV6AM01+LKV2IPV6SM01） |

> 取值完全由上游特性决定，跨场景枚举（213 个特性各有自己的 LICITEM）；数据规划表"获取方法"两值并存（本端规划 / 全网规划），按是否首次部署分叉。

### 决策点：SWITCH 取值（DP 0-03100）

问：开通还是关闭？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 开通（激活） | `SWITCH=ENABLE` | 特性激活流程（多数激活 md 默认） |
| 关闭（去激活） | `SWITCH=DISABLE` | 关闭/回退特性 |

> 激活流程一律 ENABLE；激活前一般先 `LST LICENSESWITCH` 查 SWITCH 是否已 ENABLE，已开则跳过。

## 约束

- rule-0-03246 · severity=critical · 前置（License 文件激活）：License 文件未激活时无法设置任何 license 项；激活后只能设置已购买且支持配置开关的 license 项 — 后果：License 文件未激活导致命令失败
- rule-0-03247 · severity=critical · 前置（购买）：LICITEM 必须在当前 License 文件中已购买且支持配置开关，否则设置失败 — 后果：LICITEM 未购买或不支持配置开关导致命令失败
- rule-0-03248 · severity=high · 多网元一致性：同一特性在 AMF/SMF/MME/SGW-C/PGW-C 等不同网元角色上需开多个 LICITEM 时，须全部 ENABLE，遗漏任一导致特性不生效 — 后果：跨网元 LICITEM 遗漏导致特性部分不生效
- rule-0-03249 · severity=warning · 验证惯例：激活前一般先 LST LICENSESWITCH 查 SWITCH 是否已 ENABLE，已开则跳过 — 后果：重复执行无副作用但浪费操作
- rule-0-03250 · severity=info · 唯一性与生效：每个 LICITEM 一条记录（系统部署后已存在初始记录，多数 DISABLE），执行后立即生效；适用 NF 为平台级（UNC 通用） — 后果：覆盖既有记录，跨 NF 各网元独立配置
