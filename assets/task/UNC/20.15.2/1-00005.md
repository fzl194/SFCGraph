# 配置融合计费模板参数

> 融合计费模板（CCT, Converged Charging Template）参数配置——配额阈值+Trigger容器+终结方式，融合计费所有绑定层次的共享骨架。被引用于 [2-00003 融合计费](task/UNC/20.15.2/2-00003.md)。特性专属（离线计费用 OFCTEMPLATE，见 [1-00001](task/UNC/20.15.2/1-00001.md)，命令集不同不可复用）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 全局模板（修改默认 global） | `MOD CCT` | CCTMPLTNAME=global；QHT（配额空闲时间门限秒） |
| 普通模板（新增，承载完整参数） | `ADD CCT` | CCTMPLTNAME；QHT；VQT/TQT/UQT（流量/时间/事件阈值百分比）；VT（在线配额有效时长分）；MAXNUMOFCCC；PDUVOLUMELIMIT/PDUTIMELIMIT；RGVOLUMELIMIT/RGTIMELIMIT；FUATERMINATE；TIMEFORMAT；MAXSVCCONTAINER；SECRUTHRESHOLD |

**典型脚本（新增普通 CCT 模板，承载数据规划表完整参数）**：
```
ADD CCT: CCTMPLTNAME="cct_test", QHT=0, VQT=20, TQT=20, UQT=0, VT=30, MAXNUMOFCCC=3, PDUVOLUMELIMIT=500, PDUTIMELIMIT=30, RGVOLUMELIMIT=500, RGTIMELIMIT=30, FUATERMINATE=TERM_SERVICE, TIMEFORMAT=LocalTime, MAXSVCCONTAINER=100, SECRUTHRESHOLD=100;
```

**典型脚本（修改全局 global 模板，仅 QHT）**：
```
MOD CCT: CCTMPLTNAME="global", QHT=5;
```

**步骤位置**：融合计费配置第一步（使能开关之后）；下游是绑定 compound（[1-00002](task/UNC/20.15.2/1-00002.md) 绑 UserProfile / [1-00003](task/UNC/20.15.2/1-00003.md) 绑 APN / [1-00006](task/UNC/20.15.2/1-00006.md) 绑 CC）或全局直接 `MOD CCT`。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| 全局模板（global） | `MOD CCT` 修改系统默认模板；仅配 QHT 等少量全局参数；下游无需绑定（整机通用） |
| 普通模板（新增） | `ADD CCT` 新增；承载完整配额/阈值参数；下游须绑定到 UserProfile/APN/CC 之一 |

> 联动特性级 DP：模板建好后，由 [2-00003](task/UNC/20.15.2/2-00003.md) 配置层次 DP 决定绑到哪个对象（UserProfile/APN/CC/全局）。全局模板走"全局"选项，下游无绑定步骤。

## 约束

- **模板名是引用键**（critical）：CCTMPLTNAME 须先建（本步），下游所有绑定（[1-00002](task/UNC/20.15.2/1-00002.md)/[1-00003](task/UNC/20.15.2/1-00003.md)/[1-00006](task/UNC/20.15.2/1-00006.md)）都引用此名 — 否则绑定命令找不到模板。
- **global 模板系统默认存在**（info）：`MOD CCT` 仅修改默认全局模板的参数；新增普通模板用 `ADD CCT`。
- **SECRUTHRESHOLD 依赖 N40MSGTEMP**（info）：该参数在 `ADD N40MSGTEMP` 的 SECRATUSAGE=ENABLE 时才生效。
- **VT 与 RG 老化配合**（warning）：VT（在线配额有效时长）超时后 SMF 立即向 CHF 上报申请新配额；与 [1-00008](task/UNC/20.15.2/1-00008.md) 的 RG 老化时长协同规划。
