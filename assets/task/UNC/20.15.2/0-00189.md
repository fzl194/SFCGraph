# 设置NB-IoT用户的RAT值（SET NBIOTRATVALUE）（SET NBIOTRATVALUE）

> 命令静态知识见 [SET NBIOTRATVALUE](command/UNC/20.15.2/SET-NBIOTRATVALUE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按对接网元是否支持 EUTRAN-NB-IoT 类型，逐个网元参数配置对应 RAT 值。系统部署后已存在初始记录（全部 9 参数均为 EUTRAN_NB_IOT）。未输入参数保持系统当前值不变（可通过 LST NBIOTRATVALUE 查看）。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| OCS RAT 值 | `OCS` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | 和 OCS 交互使用的 RAT 值 | 对端协商：OCS 不支持 NB-IoT 时配 EUTRAN |
| CG RAT 值 | `CG` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | 和 CG 交互使用的 RAT 值 | 对端协商 |
| AAA 计费 RAT 值 | `AAAACCT` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | 和 AAA 计费服务器交互使用的 RAT 值 | 对端协商 |
| AAA 鉴权 RAT 值 | `AAAAUTH` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | 和 AAA 鉴权服务器交互使用的 RAT 值 | 对端协商 |
| PCRF RAT 值 | `PCRF` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | 和 PCRF 交互使用的 RAT 值 | PCRF 免升级（WSFD-215302）：PCRF 不支持时配 EUTRAN，需开 License |
| PGW RAT 值 | `PGW` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | SGW 发送给 PGW 使用的 RAT 值 | 对端协商 |
| CHF RAT 值 | `CHF` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | 和 CHF 交互使用的 RAT 值 | 对端协商 |
| PCF RAT 值 | `PCF` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | 和 PCF 交互使用的 RAT 值 | PCC 基本功能（WSFD-109101）：PCF 不支持时配 EUTRAN |
| UPF RAT 值 | `UPF` | `EUTRAN` / `EUTRAN_NB_IOT` | EUTRAN_NB_IOT | 和 UPF 交互使用的 RAT 值 | 对端协商 |

典型脚本（PCRF 免升级，PCRF 不支持 EUTRAN-NB-IoT）：
```
SET NBIOTRATVALUE: PCRF=EUTRAN;
```

典型脚本（PGW-C 对接，PCRF/PCF 均支持 EUTRAN-NB-IoT）：
```
SET NBIOTRATVALUE: PCRF=EUTRAN_NB_IOT, PCF=EUTRAN_NB_IOT;
```

步骤位置：NB-IoT 终端标准接入（WSFD-011601）/ PCRF 免升级（WSFD-215302）/ 激活 PCC 基本功能（WSFD-109101）"配置 NB-IoT RAT 值"步骤——按对接网元清单逐项决策；PCRF=EUTRAN 时需叠加 SET LICENSESWITCH 开启 NB-IoT 升级优化 License。

## 决策点

### 决策点：各网元 RAT 值（PCRF/PCF 等）——对端能力二选一（DP 0-03117）

每个对接网元的 RAT 信元填 EUTRAN_NB_IOT 还是 EUTRAN；取值由对端网元是否支持 EUTRAN-NB-IoT 类型决定。支持→EUTRAN_NB_IOT（系统初始值，透传）；不支持→EUTRAN（替换，需配合 License）。这是本命令核心决策。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 透传（EUTRAN_NB_IOT） | 对应参数=EUTRAN_NB_IOT | 对端网元支持 EUTRAN-NB-IoT 类型（系统初始值） |
| 替换（EUTRAN） | 对应参数=EUTRAN | 对端网元不支持 EUTRAN-NB-IoT；PCRF 取 EUTRAN 时还需开 PCRF 免升级 License |

> 三特性（WSFD-011601/WSFD-215302/WSFD-109101）均围绕 PCRF/PCF 这一对参数决策。

### 决策点：PCRF=EUTRAN 的 License 连锁（DP 0-03118）

PCRF 设为 EUTRAN 时还需开启 NB-IoT 升级优化功能 License（对应 WSFD-215302 激活流程），跨命令前置依赖。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| EUTRAN + License 联动 | `PCRF=EUTRAN`；SET LICENSESWITCH 开对应 LICITEM | PCRF 免升级完整配置 |
| 仅 EUTRAN（缺 License） | `PCRF=EUTRAN`，License 未开 | 配置无效，PCRF=EUTRAN 不生效 |

> 关键认知：PCRF 配置原则明确说明——当 NB-IoT 升级优化功能 License 使能时，PCRF=EUTRAN 才能生效。

## 约束

- **PCRF=EUTRAN 时需开启 NB-IoT 升级优化功能**（critical，rule-0-03291）：PCRF=EUTRAN 时需开启 NB-IoT 升级优化功能 License（SET LICENSESWITCH，WSFD-215302） — 后果：缺 License 导致 PCRF=EUTRAN 配置不生效
- **OCS/CG/AAAACCT/AAAAUTH/PGW/CHF/U**（warning，rule-0-03292）：OCS/CG/AAAACCT/AAAAUTH/PGW/CHF/UPF 等 6 个参数取值依赖对端能力，需逐网元确认 — 后果：误配导致对端网元收到未识别 RAT 类型
- **该命令执行后立即生效**（warning，rule-0-03293）：该命令执行后立即生效 — 后果：误以为有窗口期
- **CHF/PCF 升级场景默认值规则**（warning，rule-0-03294）：CHF/PCF 升级场景默认值规则：升级前版本早于 UNC20.6.1 时，升级场景默认值与升级前 OCS/PCRF 参数取值保持一致；新建场景默认 EUTRAN_NB_IOT — 后果：升级时误用新建默认值导致行为变化
- **最大记录数为 1（全局唯一一份）**（info，rule-0-03295）：最大记录数为 1（全局唯一一份），未输入参数保持系统当前值不变 — 后果：漏填参数沿用 EUTRAN_NB_IOT 初始值
