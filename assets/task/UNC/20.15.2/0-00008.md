# 绑定APN用户模板组（ADD APNUSRPROFG）

> 命令静态知识见 [ADD APNUSRPROFG](command/UNC/20.15.2/ADD-APNUSRPROFG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令是**纯绑定命令**：两个参数均引用已存在对象，建立"APN → 用户模板组"的归属关系。绑定的语义由 `USERPROFGNAME` 内部装了什么 UserProfile（通过 `ADD UPBINDUPG` 加入）决定——分流、QoS、计费等策略都挂在那组模板上。

配置链路（本命令在链路中的位置）：

```
ADD APN                          # 建 APN/DNN
ADD USRPROFGROUP                 # 建用户模板组（空壳）
ADD UPBINDUPG                    # 把 UserProfile 装进组（UPBINDTYPE=ULCL/QoS 等）
ADD APNUSRPROFG   ← 本命令       # 把组装到 APN 上
```

### 配置维度 1：APN 来源（参数 APN）

| 取值要求 | 说明 |
|---|---|
| 字符串 1~63 字符 | 只能由 `-`、数字、大小写字母和 `.` 组成，不能以 `.` 开头，不能出现连续两个 `.` |
| 须先由 `ADD APN` 创建 | 本命令仅做绑定，不创建 APN |

### 配置维度 2：用户模板组来源（参数 USERPROFGNAME）

| 取值要求 | 说明 |
|---|---|
| 字符串 1~63 字符 | 不支持空格，不区分大小写 |
| 须先由 `ADD USRPROFGROUP` 创建 | 组内成员（UserProfile）由 `ADD UPBINDUPG` 装入 |

**步骤位置**：在所属 feature 编排中通常位于"步骤 4~5：绑定 APN 用户模板组"——先完成 APN 实例创建 + 用户模板组组装，本命令做最终挂接。

## 决策点：USERPROFGNAME 的来源归属（DP 0-00006）

问：该 APN 绑定的用户模板组由谁规划？

| 选项 | 获取方法 | 含义 |
|---|---|---|
| 全网统一策略（如 QoS baseline、融合计费） | 全网规划 | 组名/内容由全网规划统一，多网元一致 |
| 本网元本地策略（如本端 UL CL 分流） | 本端规划 | 组名/策略为本端独立设计 |

> 同一个 APN 只能绑定**一个** USERPROFGROUP，所以选哪一组直接决定该 APN 下用户的策略集合。若需同时承载分流与 QoS，需把相关 UserProfile 都装进**同一个组**，而非建多个组分别绑。

## 约束

- **每个 APN 只能绑定一个 USRPROFGROUP**（critical，rule-0-00006）：重复绑定会冲突 — 后果：新增失败或覆盖原有绑定；若一个 APN 要同时承载分流与 QoS，需把相关 UserProfile 都装进**同一个组**
- **前置依赖（硬性）**（critical，rule-0-00007）：必须先执行 `ADD APN` 和 `ADD USRPROFGROUP`（组内成员由 `ADD UPBINDUPG` 装好） — 后果：引用对象不存在而失败
- **立即生效**（low，rule-0-00008）：执行后立即生效 — 后果：无延迟
- **最大 10000 条记录**（warning，rule-0-00009）：超过上限新增失败 — 后果：新增失败
- **APN 取值规则**（info，rule-0-00010）：1~63 字符，仅 `-`/数字/字母/`.`，不能以 `.` 开头，不支持空格及 `_`/`#`/`$` 等特殊字符 — 后果：校验拒绝
