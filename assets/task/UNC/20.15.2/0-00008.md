# ADD APNUSRPROFG 原子任务

> 命令静态知识见 [ADD APNUSRPROFG](command/UNC/20.15.2/ADD-APNUSRPROFG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

这是一个**纯绑定**命令：两个参数都是已存在对象名，本命令建立"APN → 用户模板组"的归属关系。绑定的语义由 `USRPROFGROUP` 内部装了什么 UserProfile（通过 `ADD UPBINDUPG` 加入）决定——分流、QoS、计费等策略都挂在那组模板上。本命令在配置链路中处于靠后位置：先建 `ADD APN` 和 `ADD USRPROFGROUP`/`ADD UPBINDUPG`，再用本命令挂到 APN。

**参数取值样例（跨 17 个特性高度一致）**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| APN | `Huawei.com` / `dnn-ar` / `apn1` | 全网规划 | 该 APN/DNN 的全网唯一名，须 ADD APN 已建 |
| USERPROFGNAME | `ulcl_test` / `usrg` | 本端规划或全网规划 | 用户模板组名，须 ADD USRPROFGROUP 已建 |

**典型行**

```
ADD APNUSRPROFG:APN="Huawei.com",USERPROFGNAME="ulcl_test";
ADD APNUSRPROFG:APN="apn1",USERPROFGNAME="usrg";
```

**配置链路（本命令的位置）**

```
ADD APN                          # 建 APN/DNN
ADD USRPROFGROUP                 # 建用户模板组（空壳）
ADD UPBINDUPG                    # 把 UserProfile 装进组（UPBINDTYPE=ULCL/QoS 等）
ADD APNUSRPROFG   ← 本命令       # 把组装到 APN 上
```

## 决策点

### USERPROFGNAME 的来源归属

证据中 USERPROFGNAME 的"获取方法"在不同特性下出现 **全网规划**（如 QoS 特性样例 `usrg`）与 **本端规划**（如 UL CL 分流特性样例 `ulcl_test`）两种。

| 情形 | 获取方法 | 含义 |
|---|---|---|
| 该组承载全网统一策略（如 QoS baseline） | 全网规划 | 组名/内容由全网规划统一，多网元一致 |
| 该组承载本网元本地策略（如本端 UL CL 分流） | 本端规划 | 组名/策略为本端独立设计 |

> 同一个 APN 只能绑定**一个** USERPROFGROUP，所以选哪一组直接决定该 APN 下用户的策略集合。

## 约束

- **每个 APN 只能绑定一个 USRPROFGROUP**：重复绑定会冲突 · severity=HIGH · 若一个 APN 要同时承载分流与 QoS，需把相关 UserProfile 都装进**同一个组**，而非建多个组分别绑。
- **前置依赖（硬性）**：必须先执行 `ADD APN` 和 `ADD USRPROFGROUP`（组内成员由 `ADD UPBINDUPG` 装好）· severity=HIGH · 否则本命令引用对象不存在而失败。
- **立即生效** · severity=LOW。
- **最大 10000 条记录** · severity=MEDIUM。
- **APN 取值规则**：1~63 字符，仅 `-`/数字/字母/`.`，不能以 `.` 开头 · severity=LOW。
