# 增加中转UPF与RADIUS服务器绑定（ADD UPFRDSSVR）

> 命令静态知识见 [ADD UPFRDSSVR](command/UNC/20.15.2/ADD-UPFRDSSVR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令配置经 UPF 中转的 RADIUS 服务器绑定关系（SMF 不能直连 AAA 时，由 UPF 中转鉴权/计费）。配置空间核心维度为 **服务器类型**（SERVERTYPE）和 **IP 版本**（IPVERSION），绑定到指定 UP 列表。

### 配置维度1：服务器类型（参数 SERVERTYPE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| AUTHENTICATION | 鉴权服务器（经 UPF 中转鉴权 RADIUS） | WSFD-108007 终端二次鉴权（SERVERTYPE=AUTHENTICATION，DN-AAA=192.168.10.2） |
| ACCOUNTING | 计费服务器（经 UPF 中转计费 RADIUS） | 中转计费 |

### 配置维度2：IP 版本（参数 IPVERSION）

| 取值 | 作用 | 配套必选参数 | 典型场景 |
|---|---|---|---|
| IPV4 | 服务器 IPv4 | SERVERIPV4 | WSFD-108007：SERVERIPV4="192.168.10.2" |
| IPV6 | 服务器 IPv6 | SERVERIPV6 | IPv6 中转 |

### 配置维度3：UP 列表绑定（参数 UPLISTNAME）

| 取值关系 | 作用 | 典型场景 |
|---|---|---|
| UPLISTNAME（已有 `ADD UPLIST4RDS`） | 该中转服务器绑定到 UP 列表（决定哪些 UPF 中转） | WSFD-108007：UPLISTNAME="uplist1" |

**步骤位置**：在 WSFD-108007 终端二次鉴权编排中，于 `ADD RDSSVR`（建直连 RADIUS 服务器，含 DN-AAA IP）、`ADD UPLIST4RDS`（建 UP 列表）之后执行，常与 `ADD UPFRDSCLIENTIP`（中转客户端 IP）配套。

## 决策点

**直连 vs 中转选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 直连（用 `ADD RDSSVR`） | RDSSVRGRPNAME + SERVERIPV4 | SMF 与 AAA 直连 |
| 中转（用 `ADD UPFRDSSVR`）| SERVERTYPE + SERVERIPV4 + UPLISTNAME | SMF 不能直连 AAA，经 UPF 中转（WSFD-108007） |

> 直连 RADIUS 与中转 RADIUS 的 Server IP **不能相同**，否则直连场景业务受损。中转场景下，中转 UPF 绑定的 RADIUS Server IP 必须与直连的不同。

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无回滚窗口
- **最大记录数8000**（warning）：最多可输入 8000 条记录（超 98% 告警，≤95% 恢复） — 后果：超规格新增失败
- **UPLISTNAME 必须先建**（critical）：UPLISTNAME 必须由 `ADD UPLIST4RDS` 已配置 — 后果：列表不存在则新增失败
- **SERVERIPV4/V6 条件必选**（critical）：IPVERSION=IPV4 必填 SERVERIPV4；IPV6 必填 SERVERIPV6 — 后果：缺则新增失败
- **SERVERIPV4/V6 应已规划**（critical）：SERVERIPV4/V6 应对应 `ADD RDSSVR` 已配置的 RADIUS 服务器 IP — 后果：未规划影响中转链路
- **中转与直连 IP 不可相同**（critical）：中转 RADIUS Server IP 与直连 RADIUS Server IP 不能相同 — 后果：直连场景业务受损
- **先配 Client IP**（critical）：新增中转服务器前需先用 `ADD APNRDSCLIENTIP`（或 RADIUSCLIENTIP）为该服务器组配置 Client IP 接口 — 后果：未配 Client IP 中转链路不通
