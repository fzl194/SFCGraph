# L2TP VPN（WSFD-104410）

> 特性静态知识见 [WSFD-104410](feature/UNC/20.15.2/WSFD-104410.md)。本页讲配置生成实例化时这个特性**怎么配**——APN 级 L2TP 总开关 + 鉴权子参数 + 可选加密开关 + PFCP 私有信元兼容性配置。

## 配置概览

核心对象链：**APN 实例**（ADD APN 已建）→ **APN L2TP CTRL**（SET APNL2TPCTRL 控制 L2TP 总开关）→ **UPF/PGW-U** 作为 LAC 与企业网 LNS 设备建立 L2TP 隧道 → 鉴权由 AAA Server 返回 LNS 参数（UNC 通过 N4/Sx 接口下发到 UPF）。

**核心配置空间**：本特性**无需 License**（特性概述明确"本特性无需获得 License 许可即可获得该特性的服务"），配置集中在 APN 维度 + 跨产品 PFCP 协同。

| 步骤 | 命令 | 关键参数 | 备注 |
|---|---|---|---|
| 1. APN L2TP 总开关 | SET APNL2TPCTRL | APN=apn 名, L2TPSWITCH=ENABLE | APN 必须先 ADD APN 建 |
| 2. (可选) L2TP 加密开关 | SET L2TPKEY | ENSWITCH=ENABLE/DISABLE（默认 DISABLE）| 对端 UPF 必须同步开启 |
| 3. (可选) UP 节点协议兼容性 | ADD UPCMPT | L2TPTNL=INHERIT, L2TPPCO=INHERIT | 全网规划 |
| 4. (可选) PFCP 私有信元携带 | SET PFCPPVTEXT | FEATURE=ENABLE | 全网规划 |

> **控制面下发模式**：本特性是 UNC 控制面中转型特性——UNC 仅 `SET APNL2TPCTRL` 控制面下发 L2TP 参数到 UPF/PGW-U，**真正的 L2TP 隧道在 UPF/PGW-U 与 LNS 之间建立**（UDG 用户面执行）。**L2TP 隧道建立归 UDG 域特性**，UNC 域不涉及 `ADD L2TPGROUP`/`ADD L2TPCLIENTIP` 等命令。
>
> **L2TP 鉴权路径**：MS/UE → GnGp SGSN/MME/SMF → AAA Server（access-accept 返回 LNS 参数）→ GGSN/PGW-C/SMF → 通过 N4/Sx 接口下发 LNS 参数到 PGW-U/UPF。AAA 鉴权由 [WSFD-011306 Radius 功能](feature/UNC/20.15.2/WSFD-011306.md) 承担，UNC 域不重复配鉴权。

> **本特性 = 几乎单命令特性**：原始激活 md（46559215）操作步骤仅 2 步（SET APNL2TPCTRL + 可选 SET L2TPKEY），参考信息 46559217 列 1 条命令。**L2TP VPN 是 UNC 域配置量最少的地址分配族特性之一**。

## 配置流程（步骤编排，每步显示含命令）

1. **APN 接入域基础设施**（APN 实例前置）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN`（APN=apn 名, HASVPN=ENABLE, VPNINSTANCE=vpn 名）+ `SET APNADDRESSATTR`（SUPPORTIPV4=ENABLE, IPV4ALLOCTYPE=LNS）→ [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **APN 维度 L2TP 总开关**（本特性核心）：`SET APNL2TPCTRL` → [0-00272](task/UNC/20.15.2/0-00272.md)（APN=apn 名, L2TPSWITCH=ENABLE）
3. **(可选) L2TP 加密开关**：`SET L2TPKEY`（ENSWITCH=DISABLE 默认/ENABLE 增强）
   - 关键参数：ENSWITCH=DISABLE/ENABLE
   - **★ R1.5 待补 atom（critical，SOP §6）**：`SET L2TPKEY` 命令真实存在 UNC OM 文档（`业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/`）—— UNC pre-build 阶段**未建 atom**；配置生成时**待 atom 补齐后**才产出。归 [1-00035](task/UNC/20.15.2/1-00035.md) UNC L2TP 控制族骨架。
4. **(可选) UP 节点协议兼容性**：`ADD UPCMPT`（L2TPTNL=INHERIT, L2TPPCO=INHERIT）
   - 关键参数：L2TPTNL=INHERIT/INCOMPATIBLE, L2TPPCO=INHERIT/INCOMPATIBLE
   - **★ R1.5 待补 atom**：`ADD UPCMPT` 命令真实存在 UNC OM（`业务服务管理/Pre 5G接入业务管理/`），atom 未建。归 [1-00035](task/UNC/20.15.2/1-00035.md)。
5. **(可选) PFCP 私有信元携带配置**：`SET PFCPPVTEXT`（FEATURE=ENABLE）
   - 关键参数：FEATURE=ENABLE
   - **★ R1.5 待补 atom**：`SET PFCPPVTEXT` 命令真实存在 UNC OM（`业务服务管理/Pre 5G接入业务管理/控制面管理/`），atom 未建。归 [1-00035](task/UNC/20.15.2/1-00035.md)。
6. **地址分配**（LNS 侧）：L2TP 用户 IP 地址由企业网 LNS 设备分配（非 UNC 配置范围）→ APN 上 IPV4ALLOCTYPE=LNS 在步骤 1 标记，由 LNS 实际分配

> **步骤 1 编排共享骨架**：APN 实例由 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施承担（**IPV4ALLOCTYPE=LNS** 已在 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) 场景差异中记录）；步骤 2 是本特性唯一核心 atom（**已建 [0-00272](task/UNC/20.15.2/0-00272.md)**）；步骤 3-5 是 PFCP 协同可选步骤。

> **本特性不建独立骨架**：所有命令均可在 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) 共享骨架 + [0-00272](task/UNC/20.15.2/0-00272.md) 单命令 atom 中覆盖；本特性**不**新建 compound（骨架已存在 + 核心命令已是 atom）。

## 决策点（特性级：L2TP 鉴权模式 × 加密开关）

| 选项（鉴权模式 × 加密）| 走法 | 关键联动 |
|---|---|---|
| 最小 L2TP（仅总开关）| 步骤 1（接入基础设施）+ 步骤 2（SET APNL2TPCTRL L2TPSWITCH=ENABLE）| 默认 L2TP 接入；最常用 |
| L2TP + ICCN 代理鉴权（MSISDN 用户名）| + `SET APNL2TPCTRL` 配 ICCN_PROXYAUTH=ENABLE（可选 PASSWORD）| MSISDN 作 ICCN 代理用户名（PAP 鉴权）|
| L2TP + 公用账号 | + `SET APNL2TPCTRL` 配 COMMONUSERUSED=ENABLE | 需配套 `SET APNAUTHATTR` 配置 CommonUserName/Pass（**待补 atom**）|
| L2TP + 专有承载 | + `SET APNL2TPCTRL` 配 DEDICATEDBEARSW=ENABLE | 4G/5G 专有承载 L2TP 场景 |
| L2TP + 静态密码鉴权 | + `SET APNL2TPCTRL` 配 PASSWORD + CFMPASSWORD（两参数必须一致）| 静态密码 L2TP 鉴权 |
| L2TP + 加密 | 步骤 1-3 加密开关 ENSWITCH=ENABLE | 增强 N4 接口传输安全（**对端 UPF 必须同步开启**）|

> **走法列**：所有鉴权子参数由 `SET APNL2TPCTRL` 单命令承载（**单命令 + 多参数组合**），**不**拆为多 compound。加密/UP 协议/PFCP 私有信元为独立可选步骤。

> **★ R1.5 子规则·激活未演示**（critical）：决策点表 5-8 行的 L2TP 鉴权子参数 `PASSWORD` / `CFMPASSWORD` / `COMMONUSERUSED` / `ICCN_PROXYAUTH` / `DEDICATEDBEARSW` 激活档 [WSFD-104410-激活_46559215.md](evidence/UNC/20.15.2/WSFD-104410/WSFD-104410-激活_46559215.md) 未直接演示（该激活仅演示最小 L2TP 开关 + PFCP 协同 3 命令），参数取值来自 [WSFD-104410-特性概述](evidence/UNC/20.15.2/WSFD-104410/WSFD-104410-特性概述_46559214.md) 鉴权模式描述 + 命令 wiki `SET APNL2TPCTRL` 参数表。**配置生成合并产出时须核实命令 wiki 支持同条带多参数（COMMONUSERUSED+SET APNAUTHATTR 链）**，未核实即合并 = 隐含编造同条支持 = critical 阻塞。

## 约束

- **本特性无需 License**（critical）：特性概述明确"本特性无需获得 License 许可即可获得该特性的服务" — 后果：配置生成**不**生成 License 命令。
- **APN 必须先建**（critical）：`SET APNL2TPCTRL` 必须在 `ADD APN` 之后执行（`ADD APN` 自动创建默认记录，全 DISABLE）— 后果：APN 不存在 → SET 失败。
- **子参数前提**（warning）：ICCN_PROXYAUTH/COMMONUSERUSED/DEDICATEDBEARSW/PASSWORD/CFMPASSWORD 仅在 `L2TPSWITCH=ENABLE` 时可选；DISABLE 时配置无效 — 后果：DISABLE 时配子参数无效。
- **PASSWORD 与 CFMPASSWORD 一致**（critical）：PASSWORD 与 CFMPASSWORD 必须同时输入且相同 — 后果：不一致 SET 失败。
- **PASSWORD 长度**（warning）：密码 1~128 — 后果：超长配置失败。
- **APN 命名约束**（warning）：字符串 1~63，仅 `-`/数字/字母/`.`，不能以 `.` 开头或连续 `.`，不区分大小写 — 后果：违规命名配置失败。
- **加密开关同步**（critical，启用加密时）：加密开关 ENSWITCH=ENABLE 时，**对端 UPF 必须同时打开该功能并且配置相同密钥** — 后果：任一端未同步 → L2TP 业务失败。
- **依赖 Radius 功能**（critical）：特性概述明确 L2TP 鉴权由 AAA Server 完成，需 [WSFD-011306 Radius 功能](feature/UNC/20.15.2/WSFD-011306.md) 前置 — 后果：未配 Radius → AAA 鉴权失败 → L2NS 参数无法下发 → L2TP 业务失败。
- **LNS 设备必需**（critical）：应用限制明确"企业网侧必须部署 LNS 设备" — 后果：企业网侧无 LNS 设备 → L2TP 隧道建立失败。
- **L2TP 与 Routing Behind MS 互斥（同 APN）**（warning）：特性概述明确"在同一 APN 下，L2TP VPN 和 Routing Behind MS 无需同时部署"（WSFD-205101）— 后果：同 APN 同时配致资源浪费。
- **L2TP 与 IPv6 前缀代理互斥（同 APN）**（warning）：[WSFD-104004 IPv6 前缀代理](task/UNC/20.15.2/2-00019.md激活 md 明确"如果 APN 使能了 L2TP 功能，则无需开启 IPv6 Prefix Delegation 功能" — 后果：同 APN 同时配 L2TP + IPv6 PD 致资源浪费。
- **特性规格上限**（warning）：L2TP 隧道信息（个）上限 **20**（R17 UNC 20.11.2 SMF 支持）— 后果：超过 20 个 L2TP 隧道 → 后续建立失败。
- **首次发布版本**（warning）：本特性首次发布版本为 **UNC 20.3.2**；IPv6 用户接入支持为 **UNC 20.6.0**；R17 标准支持为 **UNC 20.9.0**；20 个隧道上限为 **UNC 20.11.2** — 后果：网元版本 < 20.3.2 配置失败。
- **适用 NF 范围**（warning）：适用 NF 为 **GGSN、PGW-C、SMF**（**不含 SGSN/MME/SGW-C**）；UDG 侧 PGW-U/UPF 充当 LAC（**UDG 域特性**）— 后果：SGSN/MME 配置本特性命令被拒绝；UPF 侧 L2TP 隧道归 UDG 域。
- **C-U 协同**（warning）：UPF/PGW-U 的 L2TP 隧道配置归 UDG 域（如 `ADD L2TPGROUP`/`ADD L2TPCLIENTIP`），UNC 域仅控制面下发 L2NS 参数 — 后果：UDG 侧 L2TP 隧道未配 → 即使 UNC 控制面下发成功，业务也失败。
