# IPSec（IPFD-016000）

> 特性静态知识见 [IPFD-016000](feature/UNC/20.15.2/IPFD-016000.md)。本页讲配置生成实例化时这个特性**怎么配**——编排哪些步骤、9 场景变体如何选。

## 配置概览

核心对象链：**VNRS 微服务**（VPN + Tunnel 接口 + 静态路由，引流表）→ **IPsec 微服务**（双配 VPN + Tunnel 接口，IKE 协商）→ **ACL 数据流**（保护流量定义）→ **IPsec 安全提议 + IKE 协商**（Proposal/IKEProposal/IKEPeer）→ **IPsec 安全策略 + 应用**（Policy/PropAttach/AttachIkePeer/IPsecIntfCfg）→ **可选全局**（IKEGlobalConfig DPD/NAT 保活 + CERTSCENE 证书）。

9 场景变体**共享接入域骨架**（[UNC 1-00024](task/UNC/20.15.2/1-00024.md)）+ **IPSec 五件套**（[UNC 1-00030](task/UNC/20.15.2/1-00030.md)），差异在 IP 类型（IPv4/IPv6）+ 主备（单隧道/IPv4 主备/IPv6 主备）+ 嵌套（单层 Tunnel/GRE over IPsec/OSPF over IPsec）+ 本端接口（默认 Tunnel 接口/指定本端接口）+ 国密（PSK 认证/RSA/国密）。由特性级 DP（场景选择）驱动。

**License 信息**：本特性**无需 License**（特性概述明示；VNRS+IPsec 微服务安装需 IPSec 可选服务安装）。

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（L3VPN + VPN 实例 + APN + 使能地址分配）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR` → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **IPSec 五件套**（VNRS+IPsec 双配 22+ 步）：VNRS 微服务建 VPN + Tunnel + 路由 + IPsec 微服务双配 + ACL + Proposal + IKE + Policy + 应用 + 可选 DPD/NAT/证书 → [UNC 1-00030](task/UNC/20.15.2/1-00030.md)
   - **普通 IPv4 IPsec 隧道（基线）**：按 [UNC 1-00030](task/UNC/20.15.2/1-00030.md) 完整 27 步执行（VNRS 10 步 + IPsec 5 步 + ACL 2 步 + Proposal 3 步 + Policy 4 步 + DPD 1 步 + IPv6/证书 2 步可选）
   - **普通 IPv6 IPsec 隧道**：AFTYPE=Ipv6uni + ACLGroup6IPSEC/ACLRuleAdv6IPsec + IPSecPolicy6 + IKEPeer6 + IFIPV6ADDRESS/IFIPV6ADDRESSIPSEC + SET IFIPV6ENABLEIPSEC
   - **IPv4 IPsec 主备隧道**：增加 BACKUP_TUNNEL_INTERFACE 双 Tunnel 实例，IPsecPolicy 引用主备 Tunnel
   - **IPv6 IPsec 主备隧道**：IPv6 主备 Tunnel 双实例
   - **多 Sequence IPsec 策略**：同一 IPSecPolicy 名下多 SEQUENCENUMBER（10/20/...），每 Sequence 独立 ACL + Proposal + Peer
   - **指定本端接口建立 IPsec 隧道**：IKEPeer 增加 LOCALIFNAME=本端接口（非 Tunnel 接口）
   - **GRE over IPsec**：外层 IPSec 隧道 + 内层 GRE 隧道（嵌套）→ 先建 [UNC 1-00027](task/UNC/20.15.2/1-00027.md) GRE 隧道族，再将 GRE Tunnel 走 IPSec 保护
   - **OSPF over IPsec**：IPSec 隧道承载 OSPF 邻居 → 走 [UNC 1-00029](task/UNC/20.15.2/1-00029.md) OSPF 路由发布（OSPF NETWORK 网段经 IPSec Tunnel 接口互通）
   - **国密 IPsec 支持 IKEv1**：ADD CERTSCENE 上传国密证书；IKEPeer 改证书认证；PROPOSAL 算法替换 SM 系列
3. **下行路由发布**（让骨干网可达 IPSec 隧道封装流量）：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFINTERFACE` + `ADD OSPFIMPORTROUTE` → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)
   - 关键参数：PROCID, AREAID, IFNAME

## 约束（特性级 rule）

- **VNRS+IPsec 双配**（critical）：VNRS 微服务和 IPsec 微服务**必须一一对应**——IPsec 协商用到的隧道接口、隧道接口 IP、隧道类型、VPN 以及指定本端接口建立 IPsec 隧道的接口需要在 VNRS 微服务和 IPsec 微服务上一对一配置 — 后果：业务不通，隧道无法建立。
- **DHGROUP 不能 None**（critical）：IKEProposal 中 DHGROUP 参数不能配置为 None 或不配置，建议 Dh_group19 — 后果：SA 建立不成功。
- **预共享密钥两端一致**（critical）：PRESHAREDKEY 配置必须与对端设备相同 — 后果：IKE 协商失败。
- **对端地址不能是地址段**（critical）：IKEPeer LOWREMOTEADDR 不能配置为地址段，必须为具体 IP — 后果：IKE 对等体引用失败。
- **NAT 穿越配置一致性**（critical）：若两 IPsec 设备间存在 NAT 设备，IKEPeer NATTRAVERSAL 必须 TRUE，且 IKEGLOBALCONFIG NATKLI 必须配置 — 后果：NAT 设备不识别 ESP 报文，业务不通。
- **PROPOSALNUMBER 范围**（warning）：IKEProposal PROPOSALNUMBER 取值 1~100（101 为系统默认，仅查询），与对端无需保持一致 — 后果：超出范围配置失败。
- **IKEv1 vs IKEv2 协商**（warning）：设备同时开启 IKEv1 和 IKEv2，缺省 IKEv2；若对端不支持 IKEv2，需禁用 IKEv2 采用 IKEv1 — 后果：协商失败。
- **IPsec 双微服务独立删除**（warning）：删除 VPN/Tunnel/IPsec 配置需同时删除 VNRS 微服务和 IPsec 微服务的对应数据 — 后果：残留数据导致后续配置异常。
- **不跨 IP 类型组合**（critical）：不支持场景包括 GREv6 over IPSecv6 / OSPFv3 over IPSecv6 / IPSecv6 地址借用 / IPv4 报文入 IPSecv6 隧道 / IPv6 报文入 IPSecv4 隧道 — 后果：业务无法建立。
- **ACL 仅源/目的 IP 生效**（warning）：ACL 只支持源 IP 和目的 IP 配置，若配置端口则不生效 — 后果：ACL 规则无效。
- **首次发布版本**（info）：本特性首次发布版本为 UNC **20.5.0**（IPv4 IPsec）；**20.8.0** 第二次发布，支持 IPv6 IPsec、IPsec NAT 穿越、IPsec 主备隧道。
- **IPSec 服务前置**（critical）：IPSec 功能由 IPSec 服务承载，需完成 IPSec 安装（软件安装 > 安装可选服务 > 安装 IPSec 服务）—— 后果：未安装 IPSec 服务，业务不通。
