# SET AMFPLCYFUNC 原子任务
> 命令静态知识见 [SET AMFPLCYFUNC](command/UNC/20.15.2/SET-AMFPLCYFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：AMF。14 个参数全部 global_planned，部署后存在初始记录。证据显示不同特性取用不同参数子集：动态 UE Logo 下发关注 FULLNAME/SHORTNAME；RFSP 选频选网关注 DYNRFSPSW/RFSPCLRMODE/DFTRFSP；PCC 基本功能关注 AMNOTIFYSW/UENOTIFYSW。

## 配置方法
按"策略主题"分组建模，一次只配置与该主题相关的参数子集，未涉及参数保持初始值不变。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| AMNOTIFYSW | AM 策略关闭实时通知 PCF 开关 | `YES`（实时通知删除 AM 偶联） |
| UENOTIFYSW | UE 策略关闭实时通知 PCF 开关 | `YES`（实时通知删除 UE 偶联） |
| OLDAMPLCY | N14 接口是否传递 AM 策略 | `NO` / `YES` |
| OLDUEPLCY | N14 接口是否传递 UE 策略 | `NO` / `YES` |
| DYNRFSPSW | 灵活选频（RFSP）功能开关 | `YES`（动态 UE Logo） |
| RFSPCLRMODE | RFSP 清除方式 | `WITHOUT_RFSP` 等 |
| DFTRFSP | 默认 RFSP | `0~256` 整数 |
| DYNNISW | 是否开启动态 NI 功能 | `NO` / `YES` |
| FULLNAME | 默认运营商全称 | 字符串（0~255） |
| SHORTNAME | 默认运营商简称 | 字符串（0~255） |
| OVERLAPAREAPLCY | 重叠区域生效策略 | `PUBLIC_AREA` 等 |
| NEIGHBORLOCSW | 相邻位置区域处理开关 | `NO` / `YES` |
| HRAMAGINGSW | 高铁 AM 偶联老化开关 | `NO` / `YES` |
| HRAMAGINGTMR | 高铁 AM 偶联老化时长(min) | `1~1440` 整数 |

典型脚本（PCC 基本功能：实时通知 PCF 删除 AM/UE 偶联）：
```
SET AMFPLCYFUNC: AMNOTIFYSW=YES, UENOTIFYSW=YES;
```

## 决策点

### AM/UE 策略通知开关（AMNOTIFYSW / UENOTIFYSW）
- 问什么：用户去活时是否实时通知 PCF 同步删除 AM/UE 偶联及网络侧下发的策略。
- `YES`：实时通知，AMF 同步删除偶联与 PCF 下发的策略。
- `NO`（默认）：用户下次激活时使用本地配置的 AM/UE 策略。
- 影响 PCF 与 AMF 策略一致性，需与 [ADD AMUEPLCYCTRL](command/UNC/20.15.2/ADD-AMUEPLCYCTRL.md) 配置的 AM/UE 策略控制范围对齐。

### 灵活选频 RFSP 组（DYNRFSPSW / RFSPCLRMODE / DFTRFSP）
- 问什么：是否启用 RFSP 动态选频选网、清除方式与默认 RFSP 值。
- 动态 UE Logo 下发特性必配；常规网络保持默认。
- RFSPCLRMODE 为 conditional，DYNRFSPSW=YES 时必填；DFTRFSP 取值范围 0~256。

### 运营商名称下发（FULLNAME / SHORTNAME）
- 问什么：默认下发给 UE 的运营商全称与简称。
- 动态 UE Logo 下发特性必配，字符串 0~255 字符。

### 高铁 AM 偶联老化（HRAMAGINGSW / HRAMAGINGTMR）
- 问什么：是否开启高铁场景 AM 偶联老化及时长。
- 高铁场景（频繁切换导致偶联堆积）才开启；时长取值 1~1440 分钟。
- HRAMAGINGSW=YES 时 HRAMAGINGTMR 必填。

## 约束
- 立即生效：执行后立即作用于后续策略处理。`severity=medium`
- conditional 参数组：RFSPCLRMODE/DFTRFSP/FULLNAME/SHORTNAME/HRAMAGINGTMR 为 conditional，对应开关开启时必填。`severity=medium`
- DFTRFSP 取值范围：0~256 整数，超出无效。`severity=medium`
- HRAMAGINGTMR 取值范围：1~1440 分钟。`severity=low`
- N14 策略传递一致性：OLDAMPLCY/OLDUEPLCY 决定 N14 接口是否传递 AM/UE 策略，需与对端 AMF 版本及策略一致。`severity=medium`
