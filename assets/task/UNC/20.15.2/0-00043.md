# 配置逻辑接口（ADD LOGICINF）

> 命令静态知识见 [ADD LOGICINF](command/UNC/20.15.2/ADD-LOGICINF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD LOGICINF 创建一个"逻辑接口"对象，把接口名与本端 IP（主/从）、VPN 实例绑定。该命令被 9 个特性共用，是 Gx/Gy/Ga/Gi/S6b/SCTP/DRA 等接口的本端端点基础。核心范式：**与 [ADD LOGICIP](command/UNC/20.15.2/ADD-LOGICIP.md) 成对配置，"IP + VPN"必须完全一致**。配置空间有 3 个维度：接口类型/命名（必选）、IP 版本与主从（必选，决定地址参数集）、VPN 实例绑定（可选）。

### 配置维度 1：接口类型与命名（参数 NAME，必选）

NAME 字符串编码接口类型 + 组号/实例类型/接口号，按对接面选型：

| 取值模式 | 接口类型 | 接口号范围 | 典型用途 |
|---|---|---|---|
| `gxif1/0/N` | Gx | 0~63 | 与 PCRF 对接（策略） |
| `gyif1/0/N` | Gy | 0~15 | 与 OCS 对接（在线计费） |
| `gaif1/0/N` | Ga | 0~4 | 与 CG 对接（离线计费） |
| `giif1/0/N` | Gi | 0~1023 | 与 AAA 对接（鉴权/计费 AAA） |
| `s6bif1/0/N` | S6b | 0~1 | 与 3GPP AAA / DRA 对接 |

> 命名约定来自证据：组号=1，实例类型=0（组级），接口号按对接面取。

### 配置维度 2：IP 版本与主从（参数 IPVERSION + 地址/掩码组）

| 取值 | 主 IP 参数（条件必选） | 从 IP 参数（可选） | 联动说明 |
|---|---|---|---|
| **IPV4**（常用） | `IPV4ADDRESS1` + `IPV4MASK1`（掩码固定 32 位） | `IPV4ADDRESS2` + `IPV4MASK2`（仅 gxif/gyif/s6bif 用主从） | 地址必须由 ADD LOGICIP 先建 |
| **IPV6** | `IPV6ADDRESS1` + `IPV6PREFIXLEN1`（=128） | `IPV6ADDRESS2` + `IPV6PREFIXLEN2`（=128） | 地址必须全球单播 |
| **IPVER_ALL** | 两套均必选 | — | 双栈场景 |

> 掩码约束：IPV4MASK 只能是 255.255.255.255（即长度 32）；IPv6 前缀长度固定 128。

### 配置维度 3：VPN 实例绑定（参数 VPNINSTANCE，可选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| 已有 VPN 实例名（如 `vpn_gxif`、`vpn_gy`、`vpn_aaa`） | 将逻辑接口与 VPN 绑定 | 接口走 L3VPN 隔离（绝大多数对接面） |
| 不填 | 系统下发缺省 VPN | 公网直通场景（罕见） |

> 前置依赖：填 VPNINSTANCE 需先 [ADD VPNINST](command/UNC/20.15.2/ADD-VPNINST.md) 创建（可 [LST VPNINST](command/UNC/20.15.2/LST-VPNINST.md) 查询）。

**典型脚本（单 IP，Gx 接口）**
```
ADD LOGICIP: IPVERSION=IPv4, LOGICIPV4="10.8.10.1", VPNINSTNAME="vpn_gxif";
ADD LOGICINF: NAME="gxif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.10.1", IPV4MASK1="255.255.255.255", VPNINSTANCE="vpn_gxif";
```

**典型脚本（主从 IP，SCTP/DRA over Gx）**
```
ADD LOGICINF: NAME="gxif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.20.1", IPV4MASK1="255.255.255.255", IPV4ADDRESS2="10.8.20.2", IPV4MASK2="255.255.255.255", VPNINSTANCE="vpn_gxif";
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：配置逻辑接口"——先 [ADD LOGICIP](task/UNC/20.15.2/0-00044.md) 配 IP，再本命令绑接口，最后才由对接命令（如 [ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md)）引用。

## 决策点：接口命名与对接面（DP 0-02065）

问：该逻辑接口对接哪类网元，从而决定 NAME 接口类型与接口号段？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| Gx 对接 PCRF | `NAME=gxif1/0/N`（N: 0~63） | Gx 策略接口（PCC） |
| Gy 对接 OCS | `NAME=gyif1/0/N`（N: 0~15） | Gy 在线计费 |
| Ga 对接 CG | `NAME=gaif1/0/N`（N: 0~4） | Ga 离线计费话单 |
| Gi 对接 AAA | `NAME=giif1/0/N`（N: 0~1023） | Gi 鉴权/计费 AAA |
| S6b 对接 3GPP AAA/DRA | `NAME=s6bif1/0/N`（N: 0~1） | S6b Diameter |

### 决策点：是否配主从 IP（DP 0-02066）

问：是否需要在一对偶联中配主备本端 IP（IPV4ADDRESS2）？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单 IP（仅主） | 只配 `IPV4ADDRESS1`+`IPV4MASK1` | 普通 IPv4 单平面（Gx/Gy/Ga/Gi 直连） |
| 主从 IP | 额外配 `IPV4ADDRESS2`+`IPV4MASK2` | SCTP over Gx/Gy 偶联主备（WSFD-104508）、DRA 多平面（WSFD-011132/011133/011134） |

> 对端端口号为奇数时，可用 [MOD DIAMCONNECTION](command/UNC/20.15.2/MOD-DIAMCONNECTION.md) 设 REVERSEIP=ENABLE 颠倒本端主从 IP 与对端建偶联。

## 约束

- rule-0-02161 · severity=critical · **与 ADD LOGICIP 成对一致**：本命令的 "IPV4ADDRESS1+VPNINSTANCE" 必须与 ADD LOGICIP 的 "LOGICIPV4+VPNINSTNAME" 完全一致，否则接口绑定失败
- rule-0-02162 · severity=critical · **IP 来源前置**：IPV4ADDRESS1/IPV4ADDRESS2/IPV6ADDRESS1/IPV6ADDRESS2 必须由 ADD LOGICIP 先创建，不能凭空填
- rule-0-02163 · severity=high · **掩码固定 32 位**：IPV4MASK 只能是 255.255.255.255 或长度 32，IPv6 前缀长度固定 128，配错接口不生效
- rule-0-02164 · severity=high · **主备 SMF 不同步**：逻辑接口配置不在主备 SMF 间自动同步，需在主备 SMF 分别配置相同数据，漏配备机会导致倒换后接口缺失
- rule-0-02165 · severity=warning · **最大 4096 条记录**：超过上限新增失败；NAME 在所有逻辑接口内唯一
