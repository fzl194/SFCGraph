# ADD LOGICINF 原子任务

> 命令静态知识见 [ADD LOGICINF](command/UNC/20.15.2/ADD-LOGICINF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD LOGICINF 创建一个"逻辑接口"对象，把接口名与本端 IP（主/从）、VPN 实例绑定。该命令被 9 个特性共用，是 Gx/Gy/Ga/Gi/S6b/SCTP/DRA 等接口的本端端点基础。**必须先 [ADD LOGICIP](command/UNC/20.15.2/ADD-LOGICIP.md) 配好逻辑 IP**，且两者的"IP+VPN"完全一致。

**核心范式：与 ADD LOGICIP 成对，IP+VPN 一致；主备 SMF 需分别配置**

证据反复强调：ADD LOGICINF 的 `IPV4ADDRESS1 + VPNINSTANCE` 必须与 ADD LOGICIP 的 `LOGICIPV4 + VPNINSTNAME` 完全一致。且本配置不在主备 SMF 间自动同步。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| NAME | `gxif1/0/0`<br>`gyif1/0/1`<br>`gaif1/0/0`<br>`giif1/0/0`<br>`s6bif1/0/0` | 本端规划/全网规划 | 接口名，1~63 字符，必填 |
| IPVERSION | `IPV4` / `IPv4` / `IPv6` | 本端规划 | 必填（大小写均见） |
| IPV4ADDRESS1 | `10.8.10.1` | 全网规划 | 主 IPv4 |
| IPV4MASK1 | `255.255.255.255` | 固定取值 | 掩码**只能** 255.255.255.255 或长度 32 |
| IPV4ADDRESS2 | `10.8.10.2` | 全网规划 | 从 IPv4（SCTP/DRA 主从 IP 场景） |
| IPV4MASK2 | `255.255.255.255` | 固定取值 | 同上 |
| IPV6ADDRESS1 | — | 全网规划 | 主 IPv6（冒号十六进制） |
| IPV6PREFIXLEN1 | — | 全网规划 | 0~128 |
| IPV6ADDRESS2 | — | 全网规划 | 从 IPv6 |
| IPV6PREFIXLEN2 | — | 全网规划 | 0~128 |
| VPNINSTANCE | `vpn_gxif`<br>`vpn_gy`<br>`vpn_ga`<br>`vpn_aaa`<br>`vpn_s6bif` | 已配置数据中获取 | 需先 ADD VPNINST；不填下发缺省 VPN |

**典型行（单 IP，Gx 接口）**

```
ADD LOGICINF: NAME="gxif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.10.1", IPV4MASK1="255.255.255.255", VPNINSTANCE="vpn_gxif";
```

**典型行（主从 IP，SCTP/DRA over SCTP）**

```
ADD LOGICINF: NAME="gxif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.20.1", IPV4MASK1="255.255.255.255", IPV4ADDRESS2="10.8.20.2", IPV4MASK2="255.255.255.255", VPNINSTANCE="vpn_gxif";
```

**典型行（多接口多平面，Gy 双接口）**

```
ADD LOGICINF: NAME="Gyif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.20.1", IPV4MASK1="255.255.255.255", VPNINSTANCE="vpn_gy";
ADD LOGICINF: NAME="Gyif1/0/1", IPVERSION=IPV4, IPV4ADDRESS1="10.8.20.2", IPV4MASK1="255.255.255.255", VPNINSTANCE="vpn_gy";
```

## 决策点

### 接口名命名约定

按接口类型 + 序号规划，证据中一致约定：

| 接口类型 | 典型 NAME | 用途 |
|---|---|---|
| Gx | `gxif1/0/0`、`gxif1/0/1` | 与 PCRF 对接（策略） |
| Gy | `Gyif1/0/0`、`gyif1/0/1` | 与 OCS 对接（在线计费） |
| Ga | `gaif1/0/0` | 与 CG 对接（离线计费） |
| Gi | `giif1/0/0`、`giif1/0/1` | 与 AAA 对接（鉴权/计费 AAA 分开） |
| S6b | `s6bif1/0/0`、`s6bif1/0/1` | 与 3GPP AAA / DRA 对接 |
| S6b（3GPP AAA 直连） | `s6bif1/0/0` | 单接口 |

### 是否配主从 IP（IPV4ADDRESS2）

| 场景 | 是否配从 IP |
|---|---|
| SCTP over Gx/Gy（WSFD-104508） | 是，配主从 IP 实现偶联主备 |
| DRA 组网（WSFD-011132/011133/011134） | 是，DRA 多平面需主从 IP |
| 普通 IPv4 单平面（Gx/Gy/Ga/Gi 直连） | 否，仅 IPV4ADDRESS1 |

> 对端端口号为奇数时，可用 [MOD DIAMCONNECTION](command/UNC/20.15.2/MOD-DIAMCONNECTION.md) 设 REVERSEIP=ENABLE 颠倒本端主从 IP 与对端建偶联。

### VPN 实例填什么（跨场景差异）

| 接口 | 典型 VPN | 跨场景差异 |
|---|---|---|
| Gx | `vpn_gxif` | 一致 |
| Gy | `vpn_gy` / `vpn_gyif` | 直连 OCS 用 vpn_gy；DRA 用 vpn_gyif |
| Ga | `vpn_ga` | 一致 |
| Gi (AAA) | `vpn_aaa` / `vpn_pdn` / `vpn_enterprise` | 单平面静态路由=vpn_aaa；双平面 OSPF=vpn_pdn；带内 GRE=vpn_enterprise；带外 GRE=vpn_aaa（AAA 服务器侧） |
| S6b | `vpn_s6bif` | 一致 |

> Gi 场景证据明确：ADD LOGICIP 用 `vpn_pdn`，但 ADD LOGICINF 的 VPNINSTANCE 取 AAA 服务器所属 VPN（vpn_aaa/vpn_enterprise）——这是证据中的真实差异，配置时以数据规划表为准。

### 主备 SMF 配置

逻辑接口配置不在主备 SMF 间自动同步，需在主备 SMF 上分别配置（相同数据）。

## 约束

- **立即生效** · severity=LOW。
- **最大 4096 条记录** · severity=MEDIUM · 上限较高。
- **掩码约束**：IPV4MASK 只能是 255.255.255.255 或掩码长度 32 · severity=HIGH · 配错掩码接口不生效。
- **VPN 前置依赖**：填 VPNINSTANCE 需先 [ADD VPNINST](command/UNC/20.15.2/ADD-VPNINST.md) 创建（可 [LST VPNINST](command/UNC/20.15.2/LST-VPNINST.md) 查询），否则下发失败 · severity=HIGH。
- **不填 VPN 时下发缺省 VPN** · severity=MEDIUM。
- **与 ADD LOGICIP 成对一致**：本命令的"IPV4ADDRESS1+VPNINSTANCE"必须与 ADD LOGICIP 的"LOGICIPV4+VPNINSTNAME"完全一致 · severity=HIGH。
- **主备 SMF 不同步**：需在主备 SMF 分别配置 · severity=HIGH · 漏配备机会导致倒换后接口缺失。
- **IPv6 前缀长度**：IPV6PREFIXLEN 0~128 · severity=MEDIUM。
- **NF 适用**：SGW-C、PGW-C、SMF。
- **修改需先删再建**：Gx 接口 IP 变更时，先 [RMV LOGICINF](command/UNC/20.15.2/RMV-LOGICINF.md) 再 ADD（证据 WSFD-109101 调测流程）。
