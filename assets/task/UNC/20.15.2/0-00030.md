# 配置Diameter本端信息（ADD DIAMLOCINFO）

> 命令静态知识见 [ADD DIAMLOCINFO](command/UNC/20.15.2/ADD-DIAMLOCINFO.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD DIAMLOCINFO 配置本端在 Diameter 协议中的身份三件套（Origin-Host / Origin-Realm / Product-Name），HOSTNAME 即下游 [ADD DIAMCONNGRP](command/UNC/20.15.2/ADD-DIAMCONNGRP.md) 的 LOCALHOSTNAME 来源。3 个核心维度：**身份三元组**（必选）、**AVP 可选项**（可选开关）、**协商一致性口径**（跨网元）。

### 配置维度 1：身份三元组（必选参数）

| 参数 | 取值范围 | 作用 | 典型取值样例 |
|---|---|---|---|
| `HOSTNAME` | 字符串 1~127，不支持空格 | Origin-Host AVP；被 DIAMCONNGRP.LOCALHOSTNAME 引用 | `unc_1` / `pgw_1` / `pgwc_1` |
| `REALMNAME` | 字符串 1~127，不支持空格 | Origin-Realm AVP；须与对端协商一致 | `huawei.com` / `example.com` / `apn2` |
| `PRODUCTNAME` | 字符串 1~63，不支持空格 | Product-Name AVP；本端产品标识 | `unc` / `unc@huawei.com` |

> HOSTNAME 是下游链路组的引用锚点——配好后被 [ADD DIAMCONNGRP](command/UNC/20.15.2/ADD-DIAMCONNGRP.md) 的 LOCALHOSTNAME 字段反查。host/realm 须与对端（PCRF/OCS/AAA）协商一致。

### 配置维度 2：AVP 可选项（可选参数）

| 参数 | 取值范围 | 默认值 | 作用 |
|---|---|---|---|
| `VENDORID` | 整数 0~4294967295 | `10415` | Vendor-Id AVP |
| `FIRMWAREREV` | 整数 0~4294967295 | `0` | Firmware-Revision AVP；配 4294967295=消息不携带该 AVP |
| `ORIGINSTATEID` | `DISABLE` / `ENABLE` | `DISABLE` | Origin-State-Id AVP 使能开关 |
| `SUPPORTVENDORID` | `DISABLE` / `ENABLE` | `DISABLE` | Supported-Vendor-Id AVP 使能开关 |

> 不配时按上表默认值生效；多数场景保持默认即可。

### 配置维度 3：协商一致性口径（场景驱动）

| 场景类型 | HOSTNAME 来源 | 一致性要求 |
|---|---|---|
| 直连 PCRF / OCS / 3GPP AAA | 全网规划或与对端协商 | origin-host/realm 须匹配对端 CEA |
| over DRA（Gx/Gy/S6b） | 本端规划或全网规划 | DRA 不感知本端主机 |
| CU Full Mesh 多 PGW-U 用户 IP 重叠 | 每 PGW-U 一个 HOSTNAME | 供 PCRF 用 UE IP + GxLocalHost 唯一确定用户 |

**典型脚本**：
```
// 标准本端身份（3 必选参数）
ADD DIAMLOCINFO: HOSTNAME="unc_1", REALMNAME="example.com", PRODUCTNAME="unc";
// 带可选 AVP
ADD DIAMLOCINFO: HOSTNAME="test", REALMNAME="test", PRODUCTNAME="product", VENDORID=10415, FIRMWAREREV=0, ORIGINSTATEID=DISABLE, SUPPORTVENDORID=DISABLE;
```

**步骤位置**：在 Diameter 对接链路编排中位于"步骤 1：本端身份"——先于 [ADD DIAMCONNGRP](task/UNC/20.15.2/0-00027.md)、[ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md)，是整条 Diameter 链路编排的起点。

## 决策点：HOSTNAME 取值来源（DP 0-02039）

问：HOSTNAME 由"全网规划"还是"与对端协商"决定？

| 选项 | 联动 | 适用场景 |
|---|---|---|
| 全网规划（本端自主） | 单 HOSTNAME 即可 | over DRA（DRA 不感知本端主机）；本端独立命名 |
| 与对端协商一致 | 直连场景 origin-host/realm 须匹配对端 CEA | 直连 PCRF / OCS / 3GPP AAA |
| 每 PGW-U 一个 HOSTNAME | 多个 DIAMLOCINFO 记录（如 `pgwc_1`/`pgwc_2`） | CU Full Mesh 多 PGW-U 用户 IP 重叠 |

### Gx 与 Gy 本端主机名一致性（DP 0-02040）

问：Gx 应用与 Gy 应用是否共用同一本端主机名？

| 选项 | 联动约束 | 适用场景 |
|---|---|---|
| Gx/Gy 同 HOSTNAME | REALMNAME + PRODUCTNAME 必须相同 | 单本端身份对接双应用 |
| Gx/Gy 不同 HOSTNAME | 各自独立配 DIAMLOCINFO 记录 | 多本端身份分应用隔离 |

> 同 HOSTNAME 时 REALMNAME/PRODUCTNAME 必须同源——否则协议栈行为异常。

## 约束

- rule-0-02096 · severity=critical · 前置依赖：必须先于 ADD DIAMCONNGRP 配置，否则链路组 LOCALHOSTNAME 引用失败
- rule-0-02097 · severity=high · 对端协商一致性：直连场景 origin-host/realm 须匹配对端 CEA，否则对端拒绝；调测发现不一致时须 RMV DIAMLOCINFO + ADD 重配
- rule-0-02098 · severity=high · Gx/Gy 同源约束：Gx 与 Gy 共用同一 HOSTNAME 时，REALMNAME 和 PRODUCTNAME 必须相同
- rule-0-02099 · severity=warning · 修改须先 RMV 关联链路/链路组：调测时改 Gx 口设备标识须先 RMV DIAMCONNECTION + RMV DIAMLOCINFO 再 ADD
- rule-0-02100 · severity=info · 规格上限：最大记录数 128；执行后立即生效
