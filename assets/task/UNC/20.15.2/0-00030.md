# ADD DIAMLOCINFO 原子任务

> 命令静态知识见 [ADD DIAMLOCINFO](command/UNC/20.15.2/ADD-DIAMLOCINFO.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD DIAMLOCINFO 配置本端在 Diameter 协议中的身份（Origin-Host = HOSTNAME，Origin-Realm = REALMNAME，Product-Name = PRODUCTNAME），以及可选的 Vendor-Id / Firmware-Revision / Origin-State-Id / Supported-Vendor-Id 等 AVP。HOSTNAME 即 ADD DIAMCONNGRP 的 LOCALHOSTNAME 来源，配好后被链路组命令引用。host/realm 须与对端（PCRF/OCS/AAA）协商一致。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| HOSTNAME | `unc_1` / `pgw_1` / `pgwc_1` / `pgwc_2` | 全网规划或本端规划 | Origin-Host，1~127 字符 |
| REALMNAME | `huawei.com` / `example.com` / `example1.com` / `apn2` | 全网规划或与对端协商 | Origin-Realm，须与对端一致 |
| PRODUCTNAME | `unc` / `unc@huawei.com` | 本端规划或与对端协商 | Product-Name，1~63 字符 |
| VENDORID | `10415`（缺省） | 本端规划 | Vendor-Id AVP，0~4294967295 |
| FIRMWAREREV | `0`（缺省） | 本端规划 | Firmware-Revision AVP |
| ORIGINSTATEID | `DISABLE`（缺省） | 本端规划 | Origin-State-Id AVP 使能开关 |
| SUPPORTVENDORID | `DISABLE`（缺省） | 本端规划 | Supported-Vendor-Id AVP 使能开关 |

**特性场景取值对比**

| 特性场景 | HOSTNAME | REALMNAME | PRODUCTNAME | 说明 |
|---|---|---|---|---|
| Gx 直连 PCRF（WSFD-109101） | pgw_1 / pgwc_1 / pgwc_2 | huawei.com | unc@huawei.com | 多 PGW-U 用户 IP 重叠时配多 HOSTNAME |
| Gx over SCTP（WSFD-104508） | unc_1 | example.com | unc | host/realm 与对端 PCRF 协商 |
| Gy over SCTP（WSFD-104508） | unc_1 | example.com | unc | host/realm 与 OCS 一致 |
| Gy 直连 OCS（WSFD-109001） | pgw1 | huawei.com | unc@huawei.com | 配置后调测发现 origin-host 不符时 RMV+ADD 重配 |
| Gx over DRA（WSFD-011132） | unc_1 | example1.com | unc | — |
| Gy over DRA（WSFD-011133） | unc_1 | example1.com | unc | — |
| S6b over DRA（WSFD-011134） | pgw_1 / pgw_2 | example1.com | unc | 多本端主机 |
| S6b 直连 3GPP AAA（WSFD-010102） | unc_1 | apn2 | unc | 端口号固定由系统生成 |

## 决策点

### HOSTNAME 取值来源（全网规划 vs 与对端协商）

HOSTNAME 在不同场景下来源不同，影响后续 LOCALHOSTNAME 引用：

| 场景 | HOSTNAME 来源 |
|---|---|
| 直连 PCRF / OCS / 3GPP AAA（TCP 或 SCTP） | 多为全网规划，或与对端协商（须与 OCS/PCRF 上配置一致） |
| over DRA（Gx/Gy/S6b） | 本端规划或全网规划，DRA 不感知本端主机 |
| CU Full Mesh 多 PGW-U 用户 IP 重叠 | 每 PGW-U 一个 HOSTNAME（pgwc_1/pgwc_2），供 PCRF 用 UE IP + GxLocalHost 唯一确定用户 |

### Gx 与 Gy 应用本端主机名一致性

UNC 的 Gx 应用和 Gy 应用本端主机名**可以相同**，但若相同则要求 REALMNAME 和 PRODUCTNAME 也必须相同（同源本端信息）。

## 约束

- **立即生效** · severity=LOW。
- **最大 128 条记录** · severity=LOW。
- **HOSTNAME 须与对端协商一致** · severity=HIGH · 直连场景 origin-host/realm 须匹配对端 CEA，否则对端拒绝；调测发现不一致时须 RMV DIAMLOCINFO + ADD 重配。
- **被 DIAMCONNGRP.LOCALHOSTNAME 引用前置依赖** · severity=HIGH · 必须先于 ADD DIAMCONNGRP 配置，否则链路组配置失败。
- **Gx/Gy 同 HOSTNAME 时 REALMNAME+PRODUCTNAME 必须相同** · severity=MEDIUM · 否则协议栈行为异常。
- **修改须先 RMV 关联链路/链路组** · severity=MEDIUM · 调测时改 Gx 口设备标识须先 RMV DIAMCONNECTION + RMV DIAMLOCINFO 再 ADD。
