# SET N7SNDATTRCTRL 原子任务
> 命令静态知识见 [SET N7SNDATTRCTRL](command/UNC/20.15.2/SET-N7SNDATTRCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

该命令控制 N7 接口发送侧是否携带 GERAN（2G）和 UTRAN（3G）类信元，仅 WSFD-109101（激活 PCC 基本功能）引用，按 2G/3G 用户接入场景可选配置。系统部署后已存在初始记录（GERAN=FALSE、UTRAN=FALSE，即默认不携带 2G/3G 信元）。

## 配置方法
按接入场景（2G / 3G）逐项打开对应 RAT 信元开关；未输入参数保持系统当前值不变（可通过 LST N7SNDATTRCTRL 查看）。

| 参数 | 描述 | 配置方法 | 典型场景 | 取值样例 |
|---|---|---|---|---|
| GERAN | 携带 GERAN 类信元 | 本端规划 | 2G 用户接入场景，N7 携带 2G 相关接入类型/位置/服务节点类型 | `TRUE` / `FALSE` |
| UTRAN | 携带 UTRAN 类信元 | 本端规划 | 3G 用户接入场景，N7 携带 3G 相关接入类型/位置/服务节点类型 | `TRUE` / `FALSE` |

典型脚本（WSFD-109101，2G 用户接入）：
```
SET N7SNDATTRCTRL: GERAN=TRUE;
```
典型脚本（WSFD-109101，3G 用户接入）：
```
SET N7SNDATTRCTRL: UTRAN=TRUE;
```

## 决策点

### GERAN / UTRAN 取值（对端 PCF 能力 + 接入场景分叉）
- 问什么：N7 发送消息是否携带 GERAN/UTRAN 类信元。
- 取值由两点驱动：①本端是否部署 2G/3G 用户接入（场景分叉）；②对端 PCF 是否支持 23G 接入（能力分叉）。
- ②conditional 分叉：对端 PCF 不支持时配 FALSE；此时 N7 不携带 RAT-Type，RAT 切换不会触发 N7 更新流程。

### UTRAN 与 SET PCCFUNC 协同
- 问什么：UTRAN=TRUE 是否还需配置其他命令。
- 跨命令一致性：3G 用户接入场景下，UTRAN=TRUE 需同时保证 SET PCCFUNC 的 N7FEATURELIST 取值含 "UtranSupport"（前置能力开关，跨命令依赖）。

## 约束
- 唯一性：全局一份（系统部署后已存在初始记录）。`severity=medium`
- 生效时机：执行后立即生效。`severity=high`
- 前置（对端确认）：修改 GERAN/UTRAN 为 TRUE 前需提前和对端 PCF 确认是否支持 23G 接入。`severity=high`
- 互斥（FALSE 行为）：GERAN/UTRAN=FALSE 时，RatType 和位置相关的 Trigger 不生效（N7 不携带 RAT-Type，RAT 切换不触发 N7 更新）。`severity=high`
- 一致性：UTRAN=TRUE 需 SET PCCFUNC 的 N7FEATURELIST 含 "UtranSupport"。`severity=high`
- 默认值：GERAN=FALSE、UTRAN=FALSE（默认不携带 2G/3G 信元）。`severity=low`
- 适用 NF：GGSN（命令适用），激活 md 中用于 PCC 基本功能对接（SMF 场景）。`severity=low`
