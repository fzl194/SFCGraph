# 设置N7发送信元处理控制（SET N7SNDATTRCTRL）（SET N7SNDATTRCTRL）

> 命令静态知识见 [SET N7SNDATTRCTRL](command/UNC/20.15.2/SET-N7SNDATTRCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按接入场景（2G / 3G）逐项打开对应 RAT 信元开关。系统部署后已存在初始记录（GERAN=FALSE、UTRAN=FALSE，即默认不携带 2G/3G 信元）。未输入参数保持系统当前值不变（可通过 LST N7SNDATTRCTRL 查看）。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| GERAN 类信元 | `GERAN` | `TRUE` / `FALSE` | FALSE | 2G 接入时 N7 是否携带 sgsnAddr、geraLocation、rattype 信元 | 2G 用户接入场景（本端规划） |
| UTRAN 类信元 | `UTRAN` | `TRUE` / `FALSE` | FALSE | 3G 接入时 N7 是否携带 sgsnAddr、utraLocation、rattype 信元 | 3G 用户接入场景（本端规划） |

典型脚本（2G 用户接入）：
```
SET N7SNDATTRCTRL: GERAN=TRUE;
```

典型脚本（3G 用户接入）：
```
SET N7SNDATTRCTRL: UTRAN=TRUE;
```

步骤位置：激活 PCC 基本功能（WSFD-109101）"配置 N7 发送侧信元处理"步骤——紧随 SET PCCFUNC 前置能力开关之后；UTRAN=TRUE 时需同步确保 SET PCCFUNC 的 N7FEATURELIST 含 "UtranSupport"。

## 决策点

### 决策点：GERAN/UTRAN 取值（DP 0-03115）

N7 发送消息是否携带 GERAN/UTRAN 类信元；取值由两点驱动——①本端是否部署 2G/3G 用户接入（场景分叉）；②对端 PCF 是否支持 23G 接入（能力分叉）。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 携带（TRUE） | `GERAN=TRUE` / `UTRAN=TRUE` | 本端部署 2G/3G 接入且对端 PCF 支持 23G 接入 |
| 不携带（FALSE） | `GERAN=FALSE` / `UTRAN=FALSE` | 本端无 2G/3G 接入 或 对端 PCF 不支持 23G 接入 |

> 关键约束：FALSE 时 RatType 和位置相关的 Trigger 不生效（N7 不携带 RAT-Type，RAT 切换不触发 N7 更新流程）。

### 决策点：UTRAN=TRUE 的跨命令协同（DP 0-03116）

UTRAN=TRUE 是否还需配置其他命令；跨命令一致性——需同时保证 SET PCCFUNC 的 N7FEATURELIST 取值含 "UtranSupport"。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| UTRAN=TRUE + PCCFUNC 联动 | `UTRAN=TRUE`；SET PCCFUNC N7FEATURELIST 含 "UtranSupport" | 3G 用户接入场景完整配置 |
| 仅 UTRAN=TRUE（缺前置） | `UTRAN=TRUE`，PCCFUNC 未配 UtranSupport | 配置不完整，UtranSupport 能力未开导致行为异常 |

> 关键认知：UtranSupport 是前置能力开关，SET N7SNDATTRCTRL 的 UTRAN=TRUE 必须叠加 SET PCCFUNC 才能完整生效。

## 约束

- rule-0-03286 · severity=critical · 修改 GERAN/UTRAN 为 TRUE 前需提前和对端 PCF 确认是否支持 23G 接入 — 后果：对端不支持时强行 TRUE 导致对接失败或信元被丢弃
- rule-0-03287 · severity=high · GERAN/UTRAN=FALSE 时 RatType 和位置相关的 Trigger 不生效，N7 不携带 RAT-Type，RAT 切换不触发 N7 更新流程 — 后果：误以为 FALSE 也能触发 RAT 切换通知
- rule-0-03288 · severity=high · UTRAN=TRUE 需叠加 SET PCCFUNC 的 N7FEATURELIST 含 "UtranSupport" 才能完整生效 — 后果：仅配 UTRAN=TRUE 缺前置能力开关导致 3G 信元发送异常
- rule-0-03289 · severity=info · 全局唯一一份（系统部署后已存在初始记录），未输入参数保持系统当前值不变 — 后果：漏填参数沿用 FALSE 初始值
- rule-0-03290 · severity=info · 该命令执行后立即生效 — 后果：误以为有窗口期
