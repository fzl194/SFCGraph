# 配置离线计费容器产生开关（SET CONTAINERTRIGGER）

> 命令静态知识见 [SET CONTAINERTRIGGER](command/UNC/20.15.2/SET-CONTAINERTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SGW-C/PGW-C/SMF 侧**离线计费模板粒度**的容器产生开关：每个离线计费模板（OFCTEMPLATE）一条记录，勾选哪些"计费条件变更"会触发容器产生。容器概念：一张话单内含多个容器，分别记录不同计费条件下的流量，避免话单量暴增。核心配置空间有 2 个维度：**位置/QoS/SN 类触发条件集合**（按运营颗粒度批量勾选）和**速率控制类触发条件**（仅终端接入速率控制特性场景开启）。

### 配置维度 1：基础计费条件触发（位置/QoS/SN/费率/用户面）

| 参数 | 取值 | 作用 | 备注 |
|---|---|---|---|
| `OFCTEMPLATENAME` | 字符串 1~63 | 离线计费模板名（主键，前置已建） | 由 [ADD OFCTEMPLATE](task/UNC/20.15.2/0-00049.md) 生成 |
| `CONTTRIGRATCHNG` | DISABLE / ENABLE | RAT 更新触发新容器 | 默认 ENABLE |
| `CONTTRIGQOSCHNG` | DISABLE / ENABLE | QoS 更新触发新容器 | 默认 ENABLE；ENABLE 时可配 PCCINIT |
| `PCCINIT` | DISABLE / ENABLE | PCRF 发起的 QoS 更新触发（conditional，CONTTRIGQOSCHNG=ENABLE 时配） | 仅 Gx 接口部署场景 |
| `CONTTRIGSNCHNG` | DISABLE / ENABLE | Serving Node 更新触发 | 默认 ENABLE |
| `CONTTRIGCGISAI` | DISABLE / ENABLE | CGI/SAI 更新触发 | 2G/3G 制式 |
| `CONTTRIGECGI` | DISABLE / ENABLE | ECGI 更新触发 | 4G 制式 |
| `CONTTRIGTAI` | DISABLE / ENABLE | TAI 更新触发 | 4G/5G 制式 |
| `CONTTRIGULI` | DISABLE / ENABLE | ULI 更新触发 | 用户位置更新 |
| `CONTTRIGPLMNID` | DISABLE / ENABLE | Serving Node PLMN 标识更新触发 | 默认 ENABLE |
| `CONTTRIGRAI` | DISABLE / ENABLE | RAI 更新触发 | 路由区更新 |
| `CONTTRIGTARRIF` | DISABLE / ENABLE | 费率切换触发 | 配合费率切换特性 |
| `CONTTRIGENB` | DISABLE / ENABLE | eNodeB 更新触发 | **当前版本不支持** |
| `USERPLANECHNG` | DISABLE / ENABLE | 用户面更新触发 | 用户面切换场景 |

### 配置维度 2：速率控制类触发（终端接入速率控制特性场景）

| 参数 | 取值 | 作用 | 备注 |
|---|---|---|---|
| `SPLMNRATECHNG` | DISABLE / ENABLE | 服务 PLMN 控制速率改变触发 | 仅 WSFD-216104/216105(eMTC)/WSFD-215204/215205(NB-IoT) 等终端接入速率控制特性场景 |
| `APNRATECHNG` | DISABLE / ENABLE | APN 控制速率改变触发 | 同上特性场景 |
| `APNRATECHNGCLOSECAUSE` | `SERVINGPLMN_RATE_CONTROL_CHANGE` / `APN_RATE_CONTROL_CHANGE` | 容器关闭原因值（conditional，APNRATECHNG=ENABLE 时配，对端协商） | 决定话单填充字段 |

典型脚本（基础离线计费，仅模板名，全部 DISABLE，覆盖系统初始值）：
```
SET CONTAINERTRIGGER: OFCTEMPLATENAME="offlinecharge-test", CONTTRIGRATCHNG=DISABLE, CONTTRIGQOSCHNG=DISABLE;
```

**步骤位置**：在所属 feature 编排中位于离线计费模板建立之后、离线计费阈值/话单产生开关之前。

## 决策点

### 决策点：容器触发条件勾选集合（DP 0-03073）

问：该离线计费模板下，哪些计费条件变化需要单独记一个容器？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 细颗粒（多开） | 位置类（CGI/SAI/ECGI/TAI/ULI/RAI）+ QoS + 费率切换 多个=ENABLE | 需要按位置/费率/QoS 分别统计流量；按 2G/3G/4G/5G 制式选配位置类参数 |
| 粗颗粒（全 DISABLE） | 全部 DISABLE | 仅周期/流量阈值统计，不按条件分容器 |

> 取值由运营计费颗粒度决定。位置类参数按制式选配；费率切换配合费率切换特性。

### 决策点：APN/服务 PLMN 速率控制场景（DP 0-03074）

问：是否启用基于 APN 或服务 PLMN 的终端接入速率控制触发的容器？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 启用 | `APNRATECHNG=ENABLE` / `SPLMNRATECHNG=ENABLE` + `APNRATECHNGCLOSECAUSE` | WSFD-216104/216105(eMTC)、WSFD-215204/215205(NB-IoT) 等终端接入速率控制特性 |
| 不启用 | 保持 DISABLE | 非终端接入速率控制特性场景 |

## 约束

- **前置**（warning，rule-0-03181）：前置：必须先配置离线计费模板（ADD OFCTEMPLATE），才能配本命令
- **唯一性**（warning，rule-0-03182）：唯一性：最大记录数 100
- **不支持项**（warning，rule-0-03183）：不支持项：当前版本不支持 CONTTRIGENB 参数
- **系统存在初始记录**（info，rule-0-03184）：系统存在初始记录，参数默认值见命令静态表（多数默认 ENABLE）
- **conditional**（warning，rule-0-03185）：conditional：PCCINIT 在 CONTTRIGQOSCHNG=ENABLE 时配；APNRATECHNGCLOSECAUSE 在 APNRATECHNG=ENABLE 时配
