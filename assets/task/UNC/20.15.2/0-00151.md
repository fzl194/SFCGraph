# 设置APN-PCC功能（SET APNPCCFUNC）

> 命令静态知识见 [SET APNPCCFUNC](command/UNC/20.15.2/SET-APNPCCFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

每次 ADD APN 自动生成一条记录（初值 HOMEPCCSWITCH=INHERIT, ROAMPCCSWITCH=INHERIT, VISITPCCSWITCH=INHERIT, PCCTEMPLATE=NULL, PCFSELECTMODE=NULL）。典型按"开关 + 模板绑定 +（可选）PCF 选择方式"三段建模。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 用户类别开关 | HOMEPCCSWITCH, ROAMPCCSWITCH, VISITPCCSWITCH | ENABLE/DISABLE/INHERIT（对端协商） | INHERIT（ADD APN 自动建记录初值） | 本地/漫游/拜访用户是否走动态 PCC | 运营商按用户类别差异化 PCC 策略 |
| PCC 模板绑定 | PCCTEMPLATE | 本端规划（1~63，不区分大小写，无空格，须先 ADD PCCTEMPLATE） | NULL | APN 绑定的 PCC 模板 | 动态 PCC 全开时绑模板 |
| PCF 选择方式 | PCFSELECTMODE | 位域复选：DNN/IMSI/GPSI/SNSSAIS/PLMN/NFLOC/SERVINGSCOPE（全网规划，`-1`选中/`-0`未选拼接） | 继承 SET PCCFUNC | 5G 场景 PCF 选择策略 | 仅 5G/PCF 场景 |
| PCF 状态过滤 | DISCCUSTOM | MUTIL_SORTED_RES/MUTIL_SORTED_RES_IGN_NF_ALL_STATUS_IGN_LINK_STAT/INHERIT（本端规划） | — | 语音 PCF/PCRF 故障 Bypass 时 PCF 状态过滤 | Gx 接口场景不配 |

典型脚本（5G PCC 基本功能，动态 PCC 全开 + 绑模板）：
```
SET APNPCCFUNC:APN="huawei.com",HOMEPCCSWITCH=ENABLE,ROAMPCCSWITCH=ENABLE,VISITPCCSWITCH=ENABLE,PCCTEMPLATE="pcctemplate";
```
（5G PCF 选择方式复选 DNN+SERVINGSCOPE）：
```
SET APNPCCFUNC:APN="apn-test1",HOMEPCCSWITCH=ENABLE,ROAMPCCSWITCH=ENABLE,VISITPCCSWITCH=ENABLE,PCFSELECTMODE=DNN-1&SERVINGSCOPE-1;
```
（语音 PCF/PCRF 故障 Bypass，绑模板 + PCF 状态过滤）：
```
SET APNPCCFUNC:APN="ims", PCCTEMPLATE="pcctemplate", DISCCUSTOM=MUTIL_SORTED_RES_IGN_NF_ALL_STATUS_IGN_LINK_STAT;
```

步骤位置：建 APN（`ADD APN`）→ 建 PCC 模板（`ADD PCCTEMPLATE`）→ 本任务 → （可选）配 NF 概述位置信息（`MOD NFPROFILE`）

## 决策点

### 决策点：动态 PCC 开关取值（DP 0-03041）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 开启动态 PCC | HOMEPCCSWITCH/ROAMPCCSWITCH/VISITPCCSWITCH=ENABLE，从 PCRF/PCF 取策略 | 需动态计费/策略控制 |
| 本地 PCC | =DISABLE，不从 PCRF/PCF 取策略 | 无动态 PCC 部署 |
| 继承全局 | =INHERIT，采用 SET PCCFUNC 全局值 | 回落全局默认 |
| 不对称组合（紧急呼叫 WSFD-102101） | 本地 ENABLE，漫游/拜访 INHERIT | 紧急呼叫本地 PCC |
| 全开（VoNR 紧急呼叫 WSFD-102706） | 三者全 ENABLE | VoNR 紧急呼叫 |

### 决策点：PCF 选择方式与状态过滤（DP 0-03042）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| DNN+SERVINGSCOPE | PCFSELECTMODE=DNN-1&SERVINGSCOPE-1 | 5G 常规 PCF 选择 |
| 含 NFLOC | + 需 MOD NFPROFILE 增位置信息 | 按优选区域选 PCF |
| PCF 状态过滤 Bypass | DISCCUSTOM=MUTIL_SORTED_RES_IGN_NF_ALL_STATUS_IGN_LINK_STAT | 语音 PCF/PCRF 故障 Bypass（N7） |
| Gx 接口 | 不配 DISCCUSTOM | Gx 接口场景 |

## 约束

- **一致性**（warning，rule-0-03101）：一致性：同一 APN 只能配 TCP 或 SCTP 一种场景（PCC 对接），不允许两种同时配同一 APN
- **生效时机**（warning，rule-0-03102）：生效时机：只对新激活用户生效，不立即作用于在线用户，已有用户保持原状态
- **一致性（前置）**（warning，rule-0-03103）：一致性（前置）：PCCTEMPLATE 须先通过 `ADD PCCTEMPLATE` 建好才能绑模板
- **规格上限**（info，rule-0-03104）：规格上限：最大记录数 10000，超出上限无效
- **依赖**（warning，rule-0-03105）：依赖：PCFSELECTMODE 含 NFLOC 时需配套 `MOD NFPROFILE` 增加位置信息
