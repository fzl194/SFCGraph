# SET APNPCCFUNC 原子任务
> 命令静态知识见 [SET APNPCCFUNC](command/UNC/20.15.2/SET-APNPCCFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：PGW-C、SMF。用于使能/关闭指定 APN/DNN 用户的动态 PCC 功能（ENABLE=动态 PCC，从 PCRF/PCF 获取策略；DISABLE=本地 PCC，不从 PCRF/PCF 取策略；INHERIT=继承 [SET PCCFUNC](command/UNC/20.15.2/SET-PCCFUNC.md) 全局默认）。证据显示获取方法多值（本端规划 15 + 全网规划 8 + 对端协商/与对端协商 12 + 已配置数据中获取 7），属 decision_driven。

## 配置方法
每次 ADD APN 自动生成一条记录（初值 HOMEPCCSWITCH=INHERIT, ROAMPCCSWITCH=INHERIT, VISITPCCSWITCH=INHERIT, PCCTEMPLATE=NULL, PCFSELECTMODE=NULL）。典型按"开关 + 模板绑定 +（可选）PCF 选择方式"三段建模。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | APN/DNN 名称（已配置/全网规划，须先 [ADD APN](task/UNC/20.15.2/0-00004.md)） | `huawei.com` / `apn-test1` / `ims` / `sos` |
| HOMEPCCSWITCH | 本地用户动态 PCC 功能（对端协商） | `ENABLE` / `DISABLE` / `INHERIT` |
| ROAMPCCSWITCH | 漫游用户动态 PCC 功能（对端协商） | `ENABLE` / `DISABLE` / `INHERIT` |
| VISITPCCSWITCH | 拜访用户动态 PCC 功能（对端协商） | `ENABLE` / `DISABLE` / `INHERIT` |
| PCCTEMPLATE | PCC 模板名称（本端规划，须先 [ADD PCCTEMPLATE](task/UNC/20.15.2/0-00053.md)） | `pcctemplate` / `test_template` |
| PCFSELECTMODE | 选择 PCF 方式（位域，全网规划，复选） | `DNN-1&SERVINGSCOPE-1` |
| DISCCUSTOM | PCF 状态过滤参数（本端规划） | `MUTIL_SORTED_RES_IGN_NF_ALL_STATUS_IGN_LINK_STAT` |

典型脚本（5G PCC 基本功能，动态 PCC 全开 + 绑模板）：
```
SET APNPCCFUNC:APN="huawei.com",HOMEPCCSWITCH=ENABLE,ROAMPCCSWITCH=ENABLE,VISITPCCSWITCH=ENABLE,PCCTEMPLATE="pcctemplate";
```
（5G PCF 选择方式复选 DNN+SERVINGSCOPE）：
```
SET APNPCCFUNC:APN="apn-test1",HOMEPCCSWITCH=ENABLE,ROAMPCCSWITCH=ENABLE,VISITPCCSWITCH=ENABLE,PCFSELECTMODE=DNN-1&SERVINGSCOPE-1;
```
（语音 PCF/PCRF 故障 Bypass，绑模板 + PCF 状态过滤）：
```
SET APNPCCFUNC:APN="ims", PCCTEMPLATE="pcctemplate", DISCCUSTOM=MUTIL_SORTED_RES_IGN_NF_ALL_STATUS_IGN_LINK_STAT;
```

## 决策点

### 动态 PCC 开关取值（HOMEPCCSWITCH / ROAMPCCSWITCH / VISITPCCSWITCH）
- 问什么：按用户类别决定走动态 PCC、本地 PCC 还是继承全局。
- `ENABLE` 开启动态 PCC（从 PCRF/PCF 取策略）；`DISABLE` 本地 PCC（不从 PCRF/PCF 取）；`INHERIT` 采用 [SET PCCFUNC](command/UNC/20.15.2/SET-PCCFUNC.md) 全局值。
- APN 粒度优先级 > 全局粒度：本命令覆盖全局默认；INHERIT 时回落全局。
- 紧急呼叫（WSFD-102101）常见本地开 ENABLE、漫游/拜访 INHERIT 的不对称组合；VoNR 紧急呼叫（WSFD-102706）三者全 ENABLE。

### PCF 选择方式（PCFSELECTMODE）
- 问什么：按哪些维度（DNN/GPSI/IMSI/NFLOC/PLMN/SERVINGSCOPE/SNSSAIS）选择 PCF，位域复选。
- 例如 DNN+NFLOC、DNN+SUPI、DNN+SERVINGSCOPE；维度用 `-1`（选中）/`-0`（未选）位域拼接，如 `DNN-1&SERVINGSCOPE-1`。
- 含 NFLOC 时需在 SMF 的 NF 概述信息中增加位置信息（[MOD NFPROFILE](task/UNC/20.15.2/0-00120.md)）。仅 5G/PCF 场景适用。

### PCF 状态过滤参数（DISCCUSTOM）
- 问什么：语音 PCF/PCRF 故障 Bypass（N7 接口）场景下 PCF 状态过滤策略。
- 取值如 `MUTIL_SORTED_RES_IGN_NF_ALL_STATUS_IGN_LINK_STAT`（本端规划）。Gx 接口场景不配此参数。

## 约束
- 只对新激活用户生效：不立即作用于在线用户，已有用户保持原状态。`severity=medium`
- 最大记录数 10000：超出上限无效。`severity=medium`
- ADD APN 自动建记录：每次 [ADD APN](task/UNC/20.15.2/0-00004.md) 会自动为本命令增加一条 INHERIT 初始记录，随后用本命令改写。`severity=medium`
- APN 粒度 > 全局粒度：本命令优先于 [SET PCCFUNC](command/UNC/20.15.2/SET-PCCFUNC.md)。`severity=medium`
- PCCTEMPLATE 须先存在：绑模板前需先 [ADD PCCTEMPLATE](task/UNC/20.15.2/0-00053.md)。`severity=medium`
- 同一 APN 只能配 TCP 或 SCTP 一种场景（PCC 对接），不允许两种同时配同一 APN。`severity=high`
- PCFSELECTMODE 含 NFLOC 需配套 MOD NFPROFILE 增位置信息。`severity=medium`
