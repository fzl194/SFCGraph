# ADD PNFGPSI 原子任务
> 命令静态知识见 [ADD PNFGPSI](command/UNC/20.15.2/ADD-PNFGPSI.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
GPSI 号段路由按"对端 NF 实例 × 号段区间"规划：`NFINSTANCEID` 指向已配置的对端 NF（CHF_Instance_0 / LMF_Instance_1 等），`QUERYTYPE` 决定匹配方式，`RANGESTART/RANGEEND` 或 `PATTERN` 描述该 NF 接纳的 GPSI 号段集合。与 ADD PNFSUPI（按 SUPI 号段）是孪生命令，常成对出现；与 ADD PNFPROFILE/ADD PNFWILDCARD 共同构成"无 NRF/本地策略"对端选择链路。

| 查询方式 | 必备参数 | 典型脚本 |
|---|---|---|
| 号段区间 START_END | `INDEX` + `NFINSTANCEID` + `QUERYTYPE=START_END` + `RANGESTART` + `RANGEEND` | `ADD PNFGPSI: INDEX=1, NFINSTANCEID="CHF_Instance_0", QUERYTYPE=START_END, RANGESTART="138100000000000", RANGEEND="138100000000001";` |
| 模式匹配 PATTERN | `INDEX` + `NFINSTANCEID` + `QUERYTYPE=PATTERN` + `PATTERN` | 按正则/通配模式匹配 GPSI 号段，`RANGESTART/RANGEEND` 不配。 |

**号段规划要点**：GPSI 为 5G 用户永久标识（MSISDN 形态），号段长度 5~15 位；同一 NF 可配多条不重叠号段，跨 NF 间号段不可重叠。

## 决策点

### 选择维度：GPSI 还是 SUPI？
问：本地按号段选对端 NF 时，用 GPSI（用户号码）还是 SUPI（IMSI）维度？
- 用 GPSI：本命令（ADD PNFGPSI），适用于按用户号码（MSISDN 段）路由的业务（如 CHF 选择、LMF 选择）。
- 用 SUPI：[0-00064](task/UNC/20.15.2/0-00064.md)（ADD PNFSUPI），适用于按 IMSI 段路由。证据中两者常成对配置覆盖同批用户。

### 匹配方式
问：号段用区间还是模式描述？
- `QUERYTYPE=START_END`：明确定长号段 `[RANGESTART, RANGEEND]`，适合规整连续号段（证据中两例均为此）。
- `QUERYTYPE=PATTERN`：模式串（0~64 字符），适合离散/不规则号段集合。

## 约束
- 除 `INDEX` 外，其它参数组合输入必须唯一——同一 `(NFINSTANCEID, QUERYTYPE, RANGESTART/RANGEEND 或 PATTERN)` 不可重复。`severity=critical`
- `INDEX` 取值 0~4294967295；输入已使用索引时按规则处理（指定索引优先）。`severity=medium`
- `NFINSTANCEID` 长度 0~50，需满足约束规则（通常指向 ADD PNFPROFILE 已建实例）。`severity=high`
- `RANGESTART/RANGEEND` 长度 5~15，需成对出现且 START ≤ END。`severity=critical`
- 当 ADD PNFWILDCARD 的 `GPSICFGSWITCH=ON` 且本地未配 GPSI 时，代表允许所有 GPSI 号段——此时本命令可省略。文字提及 ADD PNFWILDCARD。`severity=medium`
- `NFINSTANCEID="sbidialtest"` 时用于 model-D/model-C 间接路由拨测，最多 80 条。`severity=low`
- 执行后立即生效。`severity=low`
