# MOD CCT 原子任务
> 命令静态知识见 [MOD CCT](command/UNC/20.15.2/MOD-CCT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
先 `ADD CCT` 建模板（或复用系统默认 `global` 模板），再用 `MOD CCT` 按业务维度填参。模板分两层：①全局通用模板（名称 `global`，配 `QHT` 等基础参）；②业务级 CCT（如时长计费 `cct-test` 配 `QHT/TQT/VT`，流量计费配 `VQT`，可叠加 PDU/RG/QF 各级流量/时长阈值与漫游阈值）。

| 模板类型 | 关键参数 | 典型脚本 |
|---|---|---|
| 全局通用模板（基础配额空闲时间） | `CCTMPLTNAME=global` + `QHT` | `MOD CCT:CCTMPLTNAME="global",QHT=5;` |
| 业务时长计费模板 | `CCTMPLTNAME` + `QHT/TQT/VT` | `MOD CCT:CCTMPLTNAME="cct-test",QHT=40,TQT=30,VT=40;` |
| 业务流量计费模板 | `CCTMPLTNAME` + `VQT` | `MOD CCT:CCTMPLTNAME="cct-test",VQT=30;` |
| 业务事件计费模板 | `CCTMPLTNAME` + `UQT`（事件阈值触发百分比） | `MOD CCT:CCTMPLTNAME="cct-event",UQT=20;` |
| 漫游用户独立阈值 | `CCTMPLTNAME` + `ROAMVOLUMELIMIT`/`ROAMFUATERMACT` | `MOD CCT:CCTMPLTNAME="cct-roam",ROAMVOLUMELIMIT=1024,ROAMFUATERMACT="TERM_SERVICE";` |

## 决策点

### 阈值触发百分比维度（流量/时长/事件多值）
问：模板按哪种计量维度触发配额更新？
- 流量 → `VQT`（0~50%）。
- 时长 → `TQT`（0~50%）。
- 事件 → `UQT`（0~50%）。
- 三者可同模板并存（融合计费普遍同时配流量+时长）。

### 配额到期/耗尽终结方式
问：最终配额动作指示终结方式（`FUATERMINATE`）怎么处理？
- `TERM_SERVICE`（阻塞业务）：配额耗尽阻塞用户业务。
- 漫游用户独立设置 `ROAMFUATERMACT`（conditional，漫游场景必填）。

### 阈值粒度选择（PDU / RG / QF 三级）
问：在哪一级设流量/时长阈值？
- PDU 会话级 → `PDUVOLUMELIMIT`（MB）/`PDUTIMELIMIT`（分钟）。
- 业务级（Rating Group）→ `RGVOLUMELIMIT`/`RGTIMELIMIT`。
- QF 级（Quota Holding Key）→ `QFVOLUMELIMIT`/`QFTIMELIMIT`。
- 按计费颗粒度需求选其一或多级组合。

### QHT 超时触发的上报内容
问：配额空闲时间门限（QHT）超时触发什么？
- `QHTRPTTRIGGER=QHT`：Nchf 消息容器 Trigger 填 QHT。
- `QHTEXPIREDRSU=WITH_RSU`：N40 请求 MUU 携带 RSU（否则不携带）。
- `NOQUOTATRIGGER=ENABLE`：允许接入侧更新或 PDU 阈值到触发时发起配额更新。

### 会话层异常返回码对免费业务的处理
问：会话层异常返回码动作为 Block 时是否阻塞免费业务？
- `BLKFREESRV=PASS`：不阻塞免费业务。
- 其他取值：阻塞。

## 约束
- `CCTMPLTNAME` 必选，1~63 字符，不区分大小写。`severity=critical` 后果：模板不存在则修改失败。
- 阈值取值范围：`QHT` 0 或 5~3600（秒）；`VQT/TQT/UQT` 0~50（%）；`VT` 0 或 5~1440（分）；`MAXNUMOFCCC` 1~10；`PDUVOLUMELIMIT/RGVOLUMELIMIT/QFVOLUMELIMIT/ROAMVOLUMELIMIT` 0~2147483647（MB，1MB=1048576 字节）；`PDUTIMELIMIT/RGTIMELIMIT/QFTIMELIMIT` 0~1440（分）；`UCITIMER` 0~1440（分，对端协商）；`MAXSVCCONTAINER/MAXUSEDSRVNUM` 0~100；`SECRUTHRESHOLD` 0~100。`severity=high` 后果：超规格参数被拒。
- 漫游用户相关参数（`ROAMFUATERMACT`/`ROAMVOLUMELIMIT`）为 conditional，漫游场景必填。`severity=high`
- 适用 NF：PGW-C、SMF。`severity=medium` 后果：非适用 NF 执行无效。
- 配套前置：DCC 在线计费模板/离线计费模板需先建好（参见激活 Gy 在线计费、激活离线计费、激活融合计费），本命令在其上修改融合相关参数。`severity=medium`
- 执行后立即生效。`severity=low`
