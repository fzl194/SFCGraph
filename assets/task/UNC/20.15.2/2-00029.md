# GRE（IPFD-015002）

> 特性静态知识见 [IPFD-015002](feature/UNC/20.15.2/IPFD-015002.md)。本页讲配置生成实例化时这个特性**怎么配**——编排哪些步骤、5 场景变体如何选。

## 配置概览

核心对象链：**LoopBack 接口**（隧道源物理出接口）→ **Tunnel 接口**（GRETUNNEL 隧道逻辑接口）→ **隧道静态路由**（流量引入 GRE 隧道）→ **可选增强**（端到端校验 / GRE Key / Keepalive）。

5 场景变体**共享接入域骨架**（[UNC 1-00024](task/UNC/20.15.2/1-00024.md)）+ **GRE 隧道族**（[UNC 1-00027](task/UNC/20.15.2/1-00027.md)），差异在 IP 类型（IPv4/IPv6）+ 增强功能（端到端校验 / GRE Key / Keepalive）。由特性级 DP（场景选择）驱动。

**License 信息**：本特性**无需 License**（特性概述明示）。

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（L3VPN + VPN 实例 + APN + 使能地址分配）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR` → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **GRE 隧道族**（本端 LoopBack 接口 → Tunnel 接口 → Tunnel IP → 隧道静态路由）：`ADD INTERFACE` + `ADD IFIPV4ADDRESS` + `ADD GRETUNNEL` + `ADD IFIPV4ADDRESS` + `ADD SRROUTE` → [UNC 1-00027](task/UNC/20.15.2/1-00027.md)
   - **IPv4 GRE（基线）**：SRCTYPE=if_name + DSTIPADDR + ADD SRROUTE AFTYPE=ipv4unicast
   - **IPv6 GRE**：SRCTYPE6 + DSTIPV6ADDR + ADD SRROUTE6（[0-00258](task/UNC/20.15.2/0-00258.md)）
3. **（按需）GRE 增强**（三选 N，可叠加）：
   - 端到端校验：`MOD GRETUNNEL:CHECKSUMEN=TRUE` → [0-00267](task/UNC/20.15.2/0-00267.md)
   - GRE 识别关键字：`MOD GRETUNNEL:GREKEYEN=TRUE, GREKEY="key"` → [0-00267](task/UNC/20.15.2/0-00267.md)
   - Keepalive 检测：`MOD GRETUNNEL:KEEPALVEN=TRUE, KEEPALVPERIOD=5, KEEPALVRETRYCNT=3` → [0-00267](task/UNC/20.15.2/0-00267.md)
4. **Tunnel MTU 优化（按需）**：`MOD INTERFACE:MTU ≤ (出接口 MTU - GRE 头长度)` → [0-00268](task/UNC/20.15.2/0-00268.md)
5. **下行路由发布**（让骨干网可达 GRE 隧道封装流量）：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFINTERFACE` + `ADD OSPFIMPORTROUTE` → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)
   - 关键参数：PROCID, AREAID, IFNAME（UNC 真实命令名 **OSPFINTERFACE**——v1.0 写的 `OSPFNETWORK` 是错的）

## 决策点（特性级：场景变体选择）

| 场景变体 × IP 类型 × 增强 | 走法（复用 compound）| 关键联动参数 |
|---|---|---|
| **普通 IPv4 GRE（基线）**| 骨架 [1-00024](task/UNC/20.15.2/1-00024.md) + GRE 族 [1-00027](task/UNC/20.15.2/1-00027.md) + 路由 [1-00029](task/UNC/20.15.2/1-00029.md) | SRCTYPE=if_name + DSTIPADDR |
| **IPv4 GRE + 端到端校验** | 骨架 1-00024 + GRE 族 1-00027 + MOD GRETUNNEL 增强 + 路由 1-00029 | CHECKSUMEN=TRUE |
| **IPv4 GRE + GRE Key** | 骨架 1-00024 + GRE 族 1-00027 + MOD GRETUNNEL 增强 + 路由 1-00029 | GREKEYEN=TRUE, GREKEY="key"（两端必须相同）|
| **IPv4 GRE + Keepalive 检测** | 骨架 1-00024 + GRE 族 1-00027 + MOD GRETUNNEL 增强 + 路由 1-00029 | KEEPALVEN=TRUE, KEEPALVPERIOD=5（默认）, KEEPALVRETRYCNT=3（默认）|
| **多租户共享 IPv4 GRE 隧道** | 骨架 1-00024 + GRE 族 1-00027（多 TNLNAME 实例共用源/目端口）+ 路由 1-00029 | 多 TNLNAME 共享同一源/目的地址 |

> **走法列**：5 场景**共享 3 骨架 compound**（[1-00024](task/UNC/20.15.2/1-00024.md)/[1-00027](task/UNC/20.15.2/1-00027.md)/[1-00029](task/UNC/20.15.2/1-00029.md)）；差异仅增强功能参数 + IP 类型参数。

## 约束（特性级 rule）

- **GRE 源地址与 IPSec 源地址不能相同**（critical）：同一网元上 GRE 隧道源地址不能与 IPSec 隧道源地址相同 — 后果：报文转发冲突。
- **GRE 嵌套不超过 2 层**（critical）：GRE 隧道可嵌套但最多 2 层，3 层嵌套时第三层状态会变 Down — 后果：嵌套过深报文丢弃。
- **GRE 识别关键字两端一致**（critical）：若两端设置识别关键字，必须相同；若一端设置、另一端不设置，则 GRE 隧道无法建立。
- **GRE Key/Keepalive 三选 N 可叠加**（warning）：端到端校验 / GRE Key / Keepalive 检测**任一或多个**可同时配置；Keepalive 不填参数默认 KEEPALVPERIOD=5s, KEEPALVRETRYCNT=3 次。
- **Tunnel MTU 配置**（warning）：建议 `MOD INTERFACE` ([0-00268](task/UNC/20.15.2/0-00268.md)) 配置 Tunnel MTU ≤（出接口 MTU - GRE 头长度），GRE 头 = GRE header + 外层 IP header — 后果：MTU 过大致分片丢性能。
- **源/目的端路由可达**（critical）：隧道源端口和目的端口必须路由可达 — 后果：GRE 报文无法封装转发。
- **最大 4k GRE Tunnel**（warning）：整个系统最大支持 4k GRE Tunnel 接口 — 后果：超出规格新增失败。
- **本端 GRE 配置双侧对称**（warning）：GRE 隧道两端（本端/对端）必须配置相同 — 后果：隧道无法建立。
- **首次发布版本**（info）：本特性首次发布版本为 UNC **20.0.0**。
