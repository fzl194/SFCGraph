# RMV CHGPLMNCFG 原子任务
> 命令静态知识见 [RMV CHGPLMNCFG](command/UNC/20.15.2/RMV-CHGPLMNCFG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

删除对象是 `CHGPLMNCFG`（PLMN 计费配置）的记录。匹配键为 `SUBRANGE + APNNI`（均必选），可选用 `MCC/MNC`（conditional）进一步定位 PLMN；不给 MCC/MNC 时按 SUBRANGE+APNNI 维度匹配删除。

## 配置方法
按"用户范围 + APNNI"主键删除，可选 `MCC/MNC` 收窄到具体 PLMN。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| SUBRANGE | 用户范围（必选） | `ALL_USER`（所有用户） |
| APNNI | APNNI（必选，1～62 位字符串） | `cmnet` |
| MCC | 移动国家码（conditional，3 位十进制） | `460` |
| MNC | 移动网号（conditional，2 或 3 位十进制） | `00` |

典型脚本（精确删 460/00 PLMN 下 cmnet 的策略）：
```
RMV CHGPLMNCFG: SUBRANGE=ALL_USER, APNNI=cmnet, MCC=460, MNC=00;
```

## 决策点（若有）

### 删除范围（按 PLMN 精确 vs 按 APN 维度）
- 问什么：删某条具体 PLMN 策略，还是按 APN 维度清理。
- 精确：给 `MCC/MNC` 收窄到具体 PLMN。
- 维度清理：省略 `MCC/MNC`，按 `SUBRANGE+APNNI` 匹配（匹配范围更宽），删前需 LST CHGPLMNCFG 核对命中。

### MCC/MNC conditional 语义
- `MCC/MNC` 标 conditional，通常成对出现以定位 PLMN；只给一个会缩小但可能不唯一定位，建议成对提供。

## 约束（若有）
- 唯一键：`SUBRANGE + APNNI (+ MCC/MNC)` 定位记录；建议 MCC/MNC 成对提供以精确锁定。`severity=medium`
- 前置：被删记录应存在，否则报错；删前建议 LST 核对。`severity=medium`
- 生效时机：立即生效，仅影响之后新激活用户的话单生成策略（不影响在线用户）。`severity=medium`
