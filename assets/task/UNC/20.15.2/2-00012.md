# 基于小区负荷上报的无线资源优化（WSFD-211101）

> 特性静态知识见 [WSFD-211101](feature/UNC/20.15.2/WSFD-211101.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系（必读）**：本特性是带宽族第 7 批（最后）。**控制面中继·被动响应型特性**——UNC（GGSN-C/PGW-C）不执行限速，仅做：① License 前置；② 使能 APN 级拥塞上报开关；③ 经 Gx/N4 中继小区负荷信息（运行时自动行为，无配置命令）。策略决策在 PCRF，限速执行在 PGW-U。本特性强依赖 PCC 基本功能（[2-00005](task/UNC/20.15.2/2-00005.md)）的 Gx 通道承载 CELL-CONGESTION-CHANGE(1003) trigger。
>
> 与 UDG 侧对应特性 GWFD-110332（[2-00032](task/UDG/20.15.2/2-00032.md)）的关系：**UNC=控制面中继（Gx 上报 PCRF + N4 透传 PGW-U），UDG=用户面感知（GTP-U 扩展头收 RAN 上报 + N4 透传 PGW-C）**。两侧均不配限速过滤链/规则——限速策略由 PCRF 动态下发，执行在 PGW-U。两侧 activation 均仅 2 步（License + SET APNREPORTATTR），但 License 控制项不同（UNC=LKV3W9WOCR11 / UDG=LKV3G5WOCR01）。

## 配置概览

核心对象链（UNC 侧无本特性专属对象，全部复用 PCC 基本功能基础设施 + 一个 APN 级开关）：

**License 前置**（LKV3W9WOCR11）→ **PCC 基本功能 Gx 路径**（VPN→逻辑接口→Diameter 本端→PCRF→链路组→PCRF 分组→PCC 开关）→ **使能 APN 级小区拥塞上报**（SET APNREPORTATTR CONGESTIONRPT=ENABLE）→ 运行时 PGW-C 中继小区负荷信息 → PCRF 决策 → N4 透传 PGW-U 执行。

关键差异维度：① APN 选择（决定对哪个 APN 生效）② 是否需联动 BWM（业务无法三四层识别时）③ 代际（2G/3G/4G Gx vs 5G Npcf，决定 PCRF 对接方式，由 PCC 基本功能承载）。

## 配置流程（步骤编排，每步显示含命令）

1. **激活本特性 License**（前置，本特性专属）：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)
   - 关键参数：LICITEM="LKV3W9WOCR11", SWITCH=ENABLE（控制项 82209457 基于小区负荷上报的无线资源优化）
   - 生效方式：License 加载后允许使用本特性
2. **使能指定 APN 的小区拥塞上报**（本特性核心，单命令）：`SET APNREPORTATTR` → [0-00152](task/UNC/20.15.2/0-00152.md)
   - 关键参数：APN=apn1（本端规划，须先 [ADD APN](task/UNC/20.15.2/0-00004.md)）, CONGESTIONRPT=ENABLE
   - 生效方式：开启后 PGW-C 对该 APN 接入的用户经 Gx 上报 CELL-CONGESTION-CHANGE(1003) trigger + 3GPP-User-Location-Info 给 PCRF
   - 多 APN 场景：逐 APN 配置（每个 APN 一条，atom 0-00152 约束）
3. **完成 PCC 基本功能前置**（必前置，本特性依赖其 Gx 通道）：完整参见 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md)
   - 关键：须完成 Gx 接口路径（PCRF 对接链 [1-00012](task/UNC/20.15.2/1-00012.md) + PCRF 选择 [1-00013](task/UNC/20.15.2/1-00013.md) + PCC 开关与模板 [1-00014](task/UNC/20.15.2/1-00014.md)）
   - License 前置：PCC 基本功能双 License（LKV2PCCBF01 + LKV3W9SPCC11）须已激活
   - **无本特性专属增量参数**：不像 FUP（[2-00007](task/UNC/20.15.2/2-00007.md)）需在 SET PCCFUNC 叠加增量——CELL-CONGESTION-CHANGE(1003) trigger 是 Gx 标准 Event-Trigger，PCC 基本功能激活后即支持承载

> **activation 演示范围脚注（R1.4）**：WSFD-211101 仅 1 份 activation（[激活基于小区负荷上报的无线资源优化_80495435](evidence/UNC/20.15.2/激活基于小区负荷上报的无线资源优化_80495435.md)），演示单一场景：对 APN "apn1" 使能小区拥塞上报。activation §操作步骤仅 **2 步**（License + SET APNREPORTATTR），§任务示例脚本仅 2 条命令——本特性 UNC 侧配置至此完成。**无零证据差异参数**——CONGESTIONRPT=ENABLE 与 LICITEM=LKV3W9WOCR11 均在 activation §数据规划表 + §任务示例脚本原样演示。
>
> **命令名差异说明（info）**：参考信息（[WSFD-211101-基于小区负荷上报的无线资源优化参考信息_34615790](evidence/UNC/20.15.2/WSFD-211101-基于小区负荷上报的无线资源优化参考信息_34615790.md)）列出的相关命令为 `ADD APNREPORTATTR`，但 activation §操作步骤实际使用 `SET APNREPORTATTR`——以 activation 为准（[0-00152](task/UNC/20.15.2/0-00152.md) atom 即 SET APNREPORTATTR）。
>
> **配置生成处置**：配置生成按本特性产出 = `SET LICENSESWITCH:LICITEM="LKV3W9WOCR11", SWITCH=ENABLE;` + `SET APNREPORTATTR:APN="apn1", CONGESTIONRPT=ENABLE;` 2 条 + 引用 PCC 基本功能（2-00005）完整配置流程。不产出 LST/EXP 等查询验证命令（属调测运维）。PCRF/RAN/PGW-U 侧配置非 UNC 范围。

## 决策点（特性级）

### DP1：APN 选择（决定 SET APNREPORTATTR 参数）

| 场景 | APN | CONGESTIONRPT | 走法 | activation 演示 |
|---|---|---|---|---|
| 单 APN 开启小区拥塞上报 | apn1（本端规划，须先 ADD APN）| ENABLE | 步骤 1-3 | ✓（activation §任务示例脚本原样演示） |
| 多 APN 分别开启 | 逐 APN 配置（每 APN 一条 SET APNREPORTATTR）| ENABLE | 同上（步骤 2 按规划逐 APN 循环） | ✗（activation 仅演示单 APN，命令集相同，逐 APN 重复） |
| 显式关闭某 APN 拥塞上报 | 指定 APN | DISABLE | 步骤 2（SWITCH=DISABLE） | ✗（activation 未演示，参数取值对称） |

**联动**：
- APN → 上游 [ADD APN](task/UNC/20.15.2/0-00004.md)（APN 必须先配置，atom 0-00152 约束"每次 ADD APN 自动建一条记录"）
- CONGESTIONRPT=ENABLE → 要求 RAN 侧 + PCRF 同时支持（特性 wiki §应用限制）

### DP2：是否需联动 BWM（业务无法三四层识别时）

| 选项 | 影响 | 前置 |
|---|---|---|
| 需要（PCRF 下发策略涉及非三四层可识别业务） | 前置 BWM 特性，PGW-C 侧配 BWM 本地规则承载链承载规则名 | [2-00010 基于业务感知的带宽控制](task/UNC/20.15.2/2-00010.md) 已激活 |
| 不需要（策略仅限低优先级用户/三四层可识别业务） | 步骤跳过；PCRF 下发策略直接经 N4 透传 PGW-U 执行 | 无 |

> 特性 wiki §特性交互明示：若需要对特性业务进行限制，且该业务无法通过三四层特征识别时，需部署 BWM（WSFD-211005）。是否联动由 PCRF 策略规划决定，非 UNC 侧配置参数。

### DP3：代际（决定 PCRF 对接方式，联动 [2-00005](task/UNC/20.15.2/2-00005.md) DP1）

| 代际 | PCRF 对接 | PCRF 选择 | 拥塞上报 |
|---|---|---|---|
| 2G/3G/4G（Gx 接口） | [1-00012](task/UNC/20.15.2/1-00012.md) Diameter 对接链 | [1-00013](task/UNC/20.15.2/1-00013.md) PCRF 分组+号段/APN/缺省 | SET APNREPORTATTR 不变 |
| 5G（Npcf 接口） | 无 Diameter（PCF 经 NRF 服务化发现） | [ADD PCFSELPLCY](task/UNC/20.15.2/0-00054.md)（AMF）+ SET APNPCCFUNC PCFSELECTMODE（SMF） | SET APNREPORTATTR 不变 |

> 小区拥塞上报开关（SET APNREPORTATTR）跨代际不变，代际差异完全由 PCC 基本功能承载。特性 wiki §适用性矩阵显示本特性仅 2.5&3G/4G/5G NSA/5G SA 的 PGW-C/SMF 适用（SGSN/AMF/NRF 不适用）。

## 约束（特性级 rule）

- **License 前置**（critical）：须激活 LKV3W9WOCR11（控制项 82209457） — 否则基于小区负荷上报的无线资源优化功能不生效。
- **依赖 PCC 基本功能**（critical）：须先激活 [WSFD-109101 PCC基本功能](task/UNC/20.15.2/2-00005.md)（License LKV2PCCBF01+LKV3W9SPCC11 + Gx 接口 PCRF 对接 + PCC 开关） — 否则无 Gx 通道承载 CELL-CONGESTION-CHANGE(1003) trigger 上报与策略下发。
- **解决方案特性·多网元协同**（critical）：端到端网元须均为 HUAWEI 设备——RAN（UTRAN/eNodeB V100R017C13+）+ PGW-U（UDG 20.7.0+，对应 [2-00032 GWFD-110332](task/UDG/20.15.2/2-00032.md)）+ PCRF（V100R017C13+）须同时支持 — 单开 UNC 侧不生效。
- **仅 PCC 本地用户**（warning）：要求用户为 PCC 本地用户 — 不支持拜访用户和漫游用户（特性 wiki §应用限制）。
- **SGSN/SGW-U 须透传扩展 GTP-U**（warning）：SGSN/SGW-U 须能透传携带小区负荷信息的扩展 GTP-U 上下行报文 — 否则负荷信息无法从 RAN 到达 PGW-U。
- **MTU 规划**（warning）：RAN 侧与核心网之间传输携带 GTP-U 扩展头的 IP 报文，可能影响网络 MTU 值 — 需整网规划（特性 wiki §对系统的影响）。
- **CPU 与信令开销**（warning）：PGW-C 实时向 PCRF 上报小区负荷变化事件，增加 CPU 资源消耗和信令开销 — 须按当前话务模型评估（特性 wiki §对系统的影响）。
- **SET APNREPORTATTR 单 APN 单条**（info）：每个 APN 一条配置（atom 0-00152 约束），多 APN 场景逐 APN 配置。
- **可选软参 DWORD519 Bit14**（info）：控制开启本特性后小区拥塞上报位置时 Gx 的 CCR-U 中是否产生 2 个 ULI AVP（参考信息 [34615790](evidence/UNC/20.15.2/WSFD-211101-基于小区负荷上报的无线资源优化参考信息_34615790.md) 软参段，非激活配置项，按需由软件参数文档规划）。
- **UNC 侧不配限速参数**（info）：限速策略由 PCRF 动态下发，执行在 PGW-U/UPF 侧。UDG 2-00032 仅做 GTP-U 收 RAN 上报 + N4 透传 PGW-C（不配过滤链/规则），UNC 仅做 Gx 上报 PCRF + N4 透传 PGW-U — 两侧均无带宽/整形参数。
- **不涉及计费与话单**（info）：特性 wiki §计费与话单明示本特性不涉及计费与话单。
