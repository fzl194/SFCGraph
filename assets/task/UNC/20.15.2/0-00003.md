# 配置AM策略与UE策略控制参数（ADD AMUEPLCYCTRL）

> 命令静态知识见 [ADD AMUEPLCYCTRL](command/UNC/20.15.2/ADD-AMUEPLCYCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

本命令为指定用户（群）或网络切片配置 AM 策略与 UE 策略的控制参数。核心配置空间有 3 个维度：**用户范围**（决定哪些用户受控）、**偶联开关**（决定是否建立 AMF↔PCF 偶联）、**失败/终止处理策略**（决定异常情况下的兜底行为）。偶联开关决定 AMF 是否走网络侧规划策略。

### 配置维度 1：用户范围（参数 SUBRANGE，必选）

| 取值 | 作用 | 说明 |
|---|---|---|
| **ALL_USER** | 所有用户 | 基于网络切片控制方式仅此可用；优先级最低 |
| **HOME_USER** | 本网用户 | 最常用；按用户属性匹配 |
| **FOREIGN_USER** | 外网用户 | 按用户属性匹配 |
| **IMSI_PREFIX** | 按 IMSI 前缀圈定 | 需配 IMSIPRE；优先级高于 HOME_USER |
| **MSISDN_PREFIX** | 按 MSISDN 前缀圈定 | 需配 MSISDNPRE；优先级最高 |

> 策略匹配优先级（高→低）：MSISDN_PREFIX > IMSI_PREFIX > HOME_USER/FOREIGN_USER > ALL_USER

### 配置维度 2：偶联开关（参数 ISAMASSOC / ISUEASSOC）

**两个独立开关**，可按需组合：

| 参数 | 取值 | 作用 | 说明 |
|---|---|---|---|
| **ISAMASSOC** | `YES` | AMF 向 PCF 建立 AM 策略偶联 | 应用网络侧 AM 策略 |
| **ISAMASSOC** | `NO`（默认） | 不建立 AM 策略偶联 | 使用本地/签约策略 |
| **ISUEASSOC** | `YES` | AMF 向 PCF 建立 UE 策略偶联 | 应用网络侧 UE 策略 |
| **ISUEASSOC** | `NO`（默认） | 不建立 UE 策略偶联 | 使用本地/签约策略 |

> 常见组合：`ISAMASSOC=YES, ISUEASSOC=YES`（全建偶联，PCC 基本功能场景）

### 配置维度 3：失败/终止处理策略（可选，默认为宽松兜底）

| 参数 | 取值 | 作用 |
|---|---|---|
| **AMFAILPLCY**（AM 建立/更新失败） | `LOCAL_OR_SUB_AMPLCY`（默认） | 回退本地或签约 AM 策略 |
| **AMFAILPLCY** | `DEREGIST` | 注册流程拒绝注册；其他流程发起去注册 |
| **UEFAILPLCY**（UE 建立/更新失败） | `NOPROCESS`（默认） | 不处理异常，用户移动性流程正常进行 |
| **UEFAILPLCY** | `DEREGIST` | 对用户发起去注册 |
| **AMTERPLCY**（PCF 终止 AM 策略） | `LOCAL_OR_SUB_AMPLCY` | 回退本地或签约 AM 策略 |
| **AMTERPLCY** | `DEREGIST` | 立即发起去注册 |
| **UETERPLCY**（PCF 终止 UE 策略） | `NOPROCESS`（默认） | 不处理异常 |
| **UETERPLCY** | `DEREGIST` | 立即发起去注册 |

> 常规场景用默认即可；仅在需要定制失败/终止兜底行为时显式指定。

### 配置维度 4：网络切片群组（参数 NSGRPID）

| 取值 | 作用 | 说明 |
|---|---|---|
| `0`（默认） | 不按网络切片控制 | 常规场景 |
| `1~16` | 按网络切片群组控制 | 仅适用于 ALL_USER；需先将用户可用切片通过 `ADD PLCYNSGRPMEM` 配置在同一群组下 |

> 基于网络切片群组（NSGRPID≠0）的控制方式**仅适用于整网用户**，且优先级**低于**基于用户属性的控制。

典型脚本（本网用户建 AM+UE 偶联）：
```
ADD AMUEPLCYCTRL: SUBRANGE=HOME_USER, NSGRPID=0, ISAMASSOC=YES, ISUEASSOC=YES, DESC="Enable UE Policy for Home Subscribers";
```
按号段建 UE 策略偶联：
```
ADD AMUEPLCYCTRL: SUBRANGE=IMSI_PREFIX, IMSIPRE="123456", ISUEASSOC=YES;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 3：配置 AM/UE 策略控制参数"——用户策略参数配置阶段。

## 决策点：用户范围选择（DP 0-01005）

问：按哪种维度指定要应用网络侧策略的用户群？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| ALL_USER | `SUBRANGE=ALL_USER` + `NSGRPID`（条件必选） | 整网用户（切片方式仅此可用）；优先级最低 |
| HOME_USER | `SUBRANGE=HOME_USER` | 本网用户（最常用） |
| IMSI_PREFIX / MSISDN_PREFIX | `SUBRANGE=IMSI_PREFIX/MSISDN_PREFIX` + `IMSIPRE/MSISDNPRE` | 按号段精细化圈定 |

### AM 策略与 UE 策略是否各自建立偶联（DP 0-01006）

问：AMF 是否分别向 PCF 建立 AM 策略偶联、UE 策略偶联？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 两者均建立 | `ISAMASSOC=YES` + `ISUEASSOC=YES` | PCC 基本功能（最常用） |
| 仅建 AM 偶联 | `ISAMASSOC=YES` + `ISUEASSOC=NO` | 只需 AM 策略控制 |
| 仅建 UE 偶联 | `ISAMASSOC=NO` + `ISUEASSOC=YES` | 只需 UE 策略控制（如按号段建 UE 策略） |
| 两者均不建 | `ISAMASSOC=NO` + `ISUEASSOC=NO` | 本地/签约策略兜底 |

## 约束

- **切片控制优先级**（high，rule-0-01009）：基于网络切片群组（NSGRPID≠0）的控制方式**仅适用于整网用户**，且优先级**低于**基于用户属性的控制 — 后果：切片控制对非整网用户不生效
- **切片成员前置依赖**（high，rule-0-01010）：当基于 NSGRPID 建 AM/UE 策略时，必须将用户所有可用切片（Allowed NSSAI）通过 `ADD PLCYNSGRPMEM` 配置在同一群组下 — 后果：切片策略匹配失败
- **最大记录数规格**（warning，rule-0-01011）：最多可输入 128 条记录 — 后果：超规格新增失败
- **执行后立即生效**（info，rule-0-01012）：无延迟，配置即时生效
