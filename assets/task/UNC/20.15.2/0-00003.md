# ADD AMUEPLCYCTRL 原子任务
> 命令静态知识见 [ADD AMUEPLCYCTRL](command/UNC/20.15.2/ADD-AMUEPLCYCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令按"用户范围 + 偶联开关 + 描述"三段式配置；偶联开关决定 AMF 是否走网络侧规划策略。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| SUBRANGE | 指定用户范围 | `HOME_USER`（本网用户）/ `ALL_USER`（所有用户）/ `IMSI_PREFIX`（按号段） |
| IMSIPRE / MSISDNPRE | 当 SUBRANGE=IMSI_PREFIX/MSISDN_PREFIX 时填前缀 | `123456` |
| NSGRPID | 网络切片群组标识，0=不按切片控制 | `0` |
| ISAMASSOC | 是否建立 AM 策略偶联 | `YES`（应用网络侧 AM 策略）/ `NO` |
| ISUEASSOC | 是否建立 UE 策略偶联 | `YES`（应用网络侧 UE 策略）/ `NO` |
| DESC | 描述，便于记忆 | `Enable UE Policy for Home Subscribers` |

典型脚本（PCC 基本功能/默认场景，本网用户建 AM+UE 偶联）：
```
ADD AMUEPLCYCTRL: SUBRANGE=HOME_USER, NSGRPID=0, ISAMASSOC=YES, ISUEASSOC=YES, DESC="Enable UE Policy for Home Subscribers";
```
按号段建 UE 策略偶联：
```
ADD AMUEPLCYCTRL: SUBRANGE=IMSI_PREFIX, IMSIPRE="123456", ISUEASSOC=YES;
```

## 决策点（若有）

### 用户范围选择
- 问什么：按哪种维度指定要应用网络侧策略的用户群。
- ALL_USER → 整网用户（切片方式仅此可用，且优先级低于用户属性）。
- HOME_USER → 本网用户（最常用）。
- IMSI_PREFIX / MSISDN_PREFIX → 按号段精细化圈定，配 IMSIPRE/MSISDNPRE。

### AM 策略与 UE 策略是否各自建立偶联
- 问什么：AMF 是否分别向 PCF 建立 AM 策略偶联、UE 策略偶联。
- 应用网络侧 AM 策略 → ISAMASSOC=YES；不应用（用本地/签约策略）→ NO。
- 应用网络侧 UE 策略 → ISUEASSOC=YES；不应用 → NO。
- 两者独立开关，常见组合为 AM=YES, UE=YES。

### 失败处理策略（AMFAILPLCY/UEFAILPLCY）与终止处理策略（AMTERPLCY/UETERPLCY）
- 默认 AMFAILPLCY=LOCAL_OR_SUB_AMPLCY（回退本地或签约 AM 策略），UEFAILPLCY=NOPROCESS。
- 仅在需要定制失败/终止兜底行为时显式指定，常规场景用默认即可。

## 约束（若有）
- 切片控制优先级：基于网络切片群组（NSGRPID≠0）的控制方式**仅适用于整网用户**，且优先级**低于**基于用户属性的控制。`severity=high`
- 切片成员前置：当基于 NSGRPID 建 AM/UE 策略时，必须将用户所有可用切片（Allowed NSSAI）通过 `ADD PLCYNSGRPMEM` 配置在同一群组下。`severity=high`
- 规格上限：最多可输入 128 条记录。`severity=medium`
- 生效时机：执行后立即生效。`severity=low`
