# ADD SGWSEGGCHGMETH 原子任务
> 命令静态知识见 [ADD SGWSEGGCHGMETH](command/UNC/20.15.2/ADD-SGWSEGGCHGMETH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是 SGW-C 离线计费链路的一环：先建号段组（ADD SUBSCRIBERIDSEGGRP），再用本命令对号段组开/关离线计费并定优先级。

## 配置方法
按"号段组名 + 优先级 + 离线计费开关"三元组配置，一条记录绑一个号段组的计费方式。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| SEGGROUPNAME | IMSI/MSISDN 号段组名（从已配置数据获取，1~31，不支持空格） | `"testmsisdngr1"` |
| PRIORITY | 号段组优先级（本端规划，1~65535，越小越高，须唯一） | `2` |
| OFFLINEFLAG | 离线计费开关（本端规划，枚举 ENABLE/DISABLE） | `ENABLE` |

典型脚本（号段组 testmsisdngr1，优先级 2，开离线计费）：
```
ADD SGWSEGGCHGMETH:SEGGROUPNAME="testmsisdngr1", PRIORITY=2, OFFLINEFLAG=ENABLE;
```

## 决策点

### 离线计费开关取值
- 问什么：该号段组是否启用离线计费。
- 启用 → OFFLINEFLAG=ENABLE（生成 SGW 离线话单）。
- 禁用 → OFFLINEFLAG=DISABLE（号段组不计费或不走离线）。
- 属本端规划决策（按号段组业务属性定）。

## 约束
- 唯一性：PRIORITY 在表内唯一（不可重复）。`severity=medium`
- 取值范围：SEGGROUPNAME 1~31 不支持空格；PRIORITY 1~65535。`severity=low`
- 规格上限：最大记录数 10。`severity=low`
- 生效时机（重要）：执行后**只对新激活用户生效**（不影响已在线用户）。`severity=medium`
- 前置：SEGGROUPNAME 需已通过 `ADD SUBSCRIBERIDSEGGRP` 建好（号段组引用一致性）。`severity=medium`
