# ADD SPECIFICAPNVAL 原子任务
> 命令静态知识见 [ADD SPECIFICAPNVAL](command/UNC/20.15.2/ADD-SPECIFICAPNVAL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是 APN 呈现策略的"可选"增强：在动态 PCC、RADIUS 等场景下，把用户使用的 APN 改写为指定 APN 上报给各对端。

## 配置方法
按"用户 APN（键）+ 各上报目标 APN（值）"配置，一条记录覆盖一个用户 APN 在多类信令中的映射。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| SUBSCRIBERAPN | 用户 APN（键，1~63，须已配置） | `"internet"` |
| ONLYROAMSWITCH | 是否仅对漫游用户生效（optional，默认 DISABLE） | `DISABLE` / `ENABLE` |
| PCRFAPN | PCRF 交互上报 APN（optional，0~63） | `"internet"` |
| PCFAPN | PCF 交互上报 APN（optional，0~63） | `"internet"` |
| CGAPN | CG 话单上报 APN（optional，0~63） | `"internet"` |
| OCSAPN | OCS 交互上报 APN（optional，0~63） | `"internet"` |
| CHFAPN | CHF 交互上报 APN（optional，0~63） | `"internet"` |
| AAAAUTHAPN | AAA 鉴权消息上报 APN（optional，0~63） | `"internet"` |
| AAAACCTAPN | AAA 计费消息上报 APN（optional，0~63） | `"internet"` |
| AAAAUTHAPNCASE | AAA 鉴权上报 APN 大小写敏感（optional，默认不敏感） | `CASE_INSENSITIVE` |

典型用法：用户 APN 为键，按需填某一类或几类上报 APN；未填则该类信令不改写。

## 决策点

### 上报 APN 的改写范围（按对端类型选择性改写）
- 问什么：需要改写哪些信令交互中的 APN（PCRF/PCF/CG/OCS/AAA/CHF）。
- 全部对端统一改写 → 所有 *APN 参数填同一目标。
- 仅特定对端差异化（如仅 CG 话单改、信令不改） → 只填对应 *APN，其余留空。
- 由业务呈现策略决定，属本端规划。

### 是否限定漫游用户生效
- 问什么：映射是否仅对漫游用户生效。
- 全用户生效 → ONLYROAMSWITCH=DISABLE（默认）。
- 仅漫游用户 → ONLYROAMSWITCH=ENABLE。
- 证据中 RADIUS 场景特别提到 ONLYROAMSWITCH 控制生效范围。

## 约束
- 前置（强约束）：SUBSCRIBERAPN 必须是 `ADD APN` 或 `ADD APNALIAS` 已配置过的 APN 或别名；上报 APN 不需预先配置。`severity=high`
- 一致性：执行 `RMV APN`/`RMV APNALIAS` 删除 APN/别名时，需同步删除本命令中对应映射关系。`severity=medium`
- 删除规则：若一条记录中所有映射 APN 均为 NULL，该记录被删除。`severity=low`
- 取值范围：APN 字符串 1~63（SUBSCRIBERAPN）/0~63（上报 APN），只能由"-"、数字、大小写字母、"."组成，不能以"."开头。`severity=low`
- 生效时机（重要）：执行后**只对新激活用户生效**。`severity=medium`
