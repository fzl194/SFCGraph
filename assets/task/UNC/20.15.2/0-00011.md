# 配置CG绑定关系（ADD CGBINDING）

> 命令静态知识见 [ADD CGBINDING](command/UNC/20.15.2/ADD-CGBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

CGBINDING 把"一个 CG 实例（IP+端口）"挂到"一个 CG 组"下并赋优先级。它是离线计费 CG 组管理三件套（`ADD CG（★R1.5 atom 待补）` 建 CG → `ADD CGGROUP` 建组 → `ADD CGBINDING` 把 CG 装进组）的第三步。本命令本身不含业务策略，纯粹建立"CG ↔ CG组"归属关系，再由 [ADD CGGRPBINDING](task/UNC/20.15.2/0-00013.md) 把组绑到离线计费模板。配置空间有 3 个维度：**CG 组归属**（必选，决定挂到哪个组）、**IP 版本与地址**（必选二选一，决定走 v4 还是 v6 栈）、**优先级**（必选，决定组内多 CG 的选择顺序）。

### 配置维度 1：CG 组归属（参数 CGGRPID，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| CG 组 ID | `CGGRPID` | 整数 1~32 | 无 | 指定本 CG 挂入的目标 CG 组 | 须先由 [ADD CGGROUP](task/UNC/20.15.2/0-00012.md) 建立 |

> CGGRPID 必须引用已建的 CG 组；组 ID 全局唯一，范围即最大 32 条记录。

### 配置维度 2：IP 版本与地址（参数 CGIPVERSION + CGIPV4ADDR/CGIPV6ADDR，条件必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| IP 版本 | `CGIPVERSION` | `IPV4` / `IPV6` | `IPV4` | 决定填哪个地址字段 | CG 支持 v4/v6 双栈时按网络规划选 |
| IPv4 地址 | `CGIPV4ADDR` | IPv4 地址 | 无 | CG 服务器的 IPv4 地址 | `CGIPVERSION=IPV4` 时必填（与 v6 二选一） |
| IPv6 地址 | `CGIPV6ADDR` | IPv6 地址 | 无 | CG 服务器的 IPv6 地址 | `CGIPVERSION=IPV6` 时必填 |
| 端口号 | `CGPORT` | 整数 1024~65535 | 无 | CG 的 GTPP 信令端口 | 须与 CG 侧实际监听端口一致 |

> CGIPVERSION 选 IPV4 必填 CGIPV4ADDR、选 IPV6 必填 CGIPV6ADDR；CG 数据来源是全网规划（CG 实例由 `ADD CG（★R1.5 atom 待补）` 配置）。

### 配置维度 3：组内优先级（参数 PRIORITY，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 优先级 | `PRIORITY` | 整数 0~100 | 无 | 本 CG 在组内的选择优先级，**值越小优先级越高**（0 最高、100 最低） | 同组多 CG 主备/负载分流 |

> 见决策点 DP 0-02001：实际生效的优先级是**本命令**绑定时指定的值，**不使用** `ADD CG（★R1.5 atom 待补）` 时指定的同名参数。

**典型脚本**

```
ADD CGBINDING: CGGRPID=1, CGIPVERSION=IPV4, CGIPV4ADDR="192.168.0.2", CGPORT=25009, PRIORITY=1;
```

**配置链路（本命令位置）**

```
ADD CG（★R1.5 atom 待补）              # 建一个 CG 实例（IP/端口/协议等）
ADD CGGROUP         # 建 CG 组（CGGRPID）
ADD CGBINDING  ← 本命令   # 把 CG 绑进组，给优先级
ADD CGGRPBINDING    # （可选）把组绑到离线计费模板
```

**步骤位置**：在所属 feature 编排中通常位于"配置 CG 组成员关系"步骤——紧随 `ADD CGGROUP` 建组之后，`ADD CGGRPBINDING` 绑定模板之前。

## 决策点

### 决策点：组内 CG 优先级取值（DP 0-02001）

问：同组多 CG 时，本 CG 应排第几优先级？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 主 CG（优先选用） | `PRIORITY` 取较小值（如 0 或 1） | 同组多 CG 时优先选此 CG 收话单 |
| 备 CG（兜底） | `PRIORITY` 取较大值（如 100） | 主 CG 不可用时切到此 CG |

> 关键认知：选定 CG Group 后，CG 优先级**使用本命令绑定时指定的 PRIORITY**，**不使用** `ADD CG（★R1.5 atom 待补）` 时指定的优先级——两处都配了优先级时易混淆，导致 CG 选择行为与预期不符。

### 决策点：IP 版本选择（DP 0-02002）

问：CG 走 IPv4 还是 IPv6 栈？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPv4 | `CGIPVERSION=IPV4` + `CGIPV4ADDR` | CG 与本端走 IPv4 网络互通（默认） |
| IPv6 | `CGIPVERSION=IPV6` + `CGIPV6ADDR` | CG 与本端走 IPv6 网络互通 |

## 约束

- **CGGRPID 须先由 ADD CGGROUP 建立**（critical，rule-0-02001）：CGGRPID 须先由 ADD CGGROUP 建立；CG 实例（IP+端口）须先由 ADD CG（★R1.5 atom 待补） 配置 — 后果：前置缺失导致绑定失败
- **选定 CG Group 后实际生效的优先级以本命令的**（warning，rule-0-02002）：选定 CG Group 后实际生效的优先级以本命令的 PRIORITY 为准，不使用 ADD CG（★R1.5 atom 待补） 的同名参数 — 后果：两处都配优先级时 CG 选择行为与预期不符
- **CGPORT 取值范围 1024~65535 且须与 CG**（warning，rule-0-02003）：CGPORT 取值范围 1024~65535 且须与 CG 侧实际监听端口一致 — 后果：端口不匹配导致话单通道不通
- **CGGRPID 取值范围 1~32**（warning，rule-0-02004）：CGGRPID 取值范围 1~32；PRIORITY 取值范围 0~100，值越小优先级越高（0 最高、100 最低） — 后果：超范围新增失败
- **该命令执行后立即生效**（info，rule-0-02005）：该命令执行后立即生效；最大记录数 1024 — 后果：超规格新增失败
