# ADD CGBINDING 原子任务

> 命令静态知识见 [ADD CGBINDING](command/UNC/20.15.2/ADD-CGBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

CGBINDING 把"一个 CG 实例（IP+端口）"挂到"一个 CG 组"下，并赋优先级。它是离线计费 CG 组管理三件套（`ADD CG` 建 CG → `ADD CGGROUP` 建组 → `ADD CGBINDING` 把 CG 装进组）的第三步。本命令本身不含业务策略，纯粹建立"CG ↔ CG组"的归属关系，再由 `ADD CGGRPBINDING` 把组绑到计费模板。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| CGGRPID | `1~32` 范围内某值 | 本端规划 | 目标 CG 组 ID，须 ADD CGGROUP 已建 |
| CGIPVERSION | `IPV4` | 全局规划 | CG 的 IP 版本 |
| CGIPV4ADDR | CG 的 IPv4 地址 | 全局规划 | CGIPVERSION=IPV4 时必填（与 v6 二选一） |
| CGIPV6ADDR | CG 的 IPv6 地址 | 全局规划 | CGIPVERSION=IPV6 时必填 |
| CGPORT | `1024~65535` | 全局规划 | CG GTPP 信令端口号，须与 CG 侧一致 |
| PRIORITY | `0~100` | 本端规划 | 本 CG 在组内的优先级（数值语义见下方决策点） |

**配置链路（本命令位置）**

```
ADD CG              # 建一个 CG 实例（IP/端口/协议等）
ADD CGGROUP         # 建 CG 组（CGGRPID）
ADD CGBINDING  ← 本命令   # 把 CG 绑进组，给优先级
ADD CGGRPBINDING    # （可选）把组绑到计费模板
SET SGWCHARGECFG    # SGW 计费基础参数
```

## 决策点

### PRIORITY 优先级语义

证据 notes 明确：用户选定 CG Group 后，**CG 优先级使用"绑定 CG 到 CG Group 时指定的优先级"**（即本命令的 PRIORITY），而**不使用** `ADD CG` 时指定的优先级。因此本命令的 PRIORITY 才是实际生效值。

| 选项 | PRIORITY 取值 | 含义 |
|---|---|---|
| 主 CG（优先选用） | 较小/较高（按现场约定） | 同组多 CG 时优先级决定选哪个 CG 收话单 |
| 备 CG（兜底） | 次级值 | 主 CG 不可用时切到备 CG |

> 具体数值大小与优先级高低的对应关系以现场/规划约定为准；关键认知是"以本命令的值为准，ADD CG 的同名参数在本上下文被覆盖"。

### IPv4 / IPv6 二选一

CGIPVERSION 决定填哪个地址字段：选 IPV4 必填 CGIPV4ADDR，选 IPV6 必填 CGIPV6ADDR。

## 约束

- **立即生效** · severity=LOW。
- **最大 1024 条记录** · severity=MEDIUM。
- **优先级以本命令为准**：选定 CG Group 后用本命令绑定时指定的 PRIORITY，不用 `ADD CG` 的 PRIORITY · severity=HIGH · 在两处都配了优先级时易混淆，导致 CG 选择行为与预期不符。
- **前置依赖**：CGGRPID 须先由 `ADD CGGROUP` 建立；CG 实例（IP+端口）须先由 `ADD CG` 配置 · severity=HIGH。
- **CGPORT 范围 1024~65535** · severity=MEDIUM；CGGRPID 范围 1~32 · severity=LOW；PRIORITY 范围 0~100 · severity=LOW。
