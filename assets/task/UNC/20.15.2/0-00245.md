# 增加5G用户安全配置（ADD NGUSRSECPARA）

> 命令静态知识见 [ADD NGUSRSECPARA](command/UNC/20.15.2/ADD-NGUSRSECPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令为指定号段用户增加 5G 安全配置（鉴权事件、加密算法、完整性算法）。配置空间主要由三个维度决定：**用户范围**、**鉴权事件**、**加密/完整性算法**。

### 配置维度1：用户范围（参数 SUBRANGE）

| 取值 | 作用 | 配套参数 | 典型场景 |
|---|---|---|---|
| ALL_USER | 对所有用户生效 | — | 全局默认安全策略（系统已缺省生成一条） |
| IMSI_PREFIX | 对指定 IMSI 前缀用户生效 | IMSIPRE（必填，5~15 位数字） | 按用户群差异化安全策略 |

### 配置维度2：鉴权事件（参数 AUTHEVENT，多选）

| 取值 | 作用 |
|---|---|
| INIT_REG | 初始注册强制鉴权（建议开启） |
| MOBL_INTER_REG | 移动性 INTER 注册强制鉴权（建议开启） |
| IDLE_SYSCHG_REG | 空闲态 EPS 到 5GS 注册强制鉴权（建议开启） |
| SERVICE_REQUEST | 服务请求强制鉴权 |
| MOBL_INTRA_REG | 移动性 INTRA 注册 |
| PROD_REG | 周期性注册 |
| REG_AFT_INTRAHO | INTRA 切换后注册（建议关闭，否则时延增大） |
| REG_AFT_INTERHO | INTER 切换后注册（建议关闭） |
| CONN_SYSCHG_REG | 连接态 EPS 到 5GS 切换后注册（建议关闭） |
| DEREGISTRATION | 去注册 |
| CONNECT_MOD | GUTI 重分配 |

> 建议在初始注册/移动性 Inter 注册/空闲态 EPS 到 5GS 注册中开启强制鉴权；切换类注册建议关闭。

### 配置维度3：加密/完整性算法（参数 ENCRYALG / INTEGALG，多选）

| 参数 | 可选取值 | 作用 |
|---|---|---|
| ENCRYALG | NEA0（空加密）/ NEA1（SNOW 3G）/ NEA2（AES）/ NEA3（ZUC） | 加密算法组合 |
| INTEGALG | NIA1（SNOW 3G）/ NIA2（AES）/ NIA3（ZUC） | 完整性算法组合（NIA0 仅紧急业务，不支持配置） |

> 算法格式为 `NXA{1|2|3}-{优先级}`，多算法用 `&` 连接，如 `NIA1-1&NIA2-1&NIA3-1`。

### 配置维度4：鉴权周期（参数 AUTHPERIOD）

| 取值 | 作用 |
|---|---|
| 0 | 不启用周期鉴权 |
| 1~24（小时，默认 24） | 用户去注册态/空闲态发起注册或服务请求时，距上次鉴权超此周期则强制鉴权 |

**步骤位置**：在所属 feature 编排中，通常在基础网络配置之后执行（AMF 安全策略配置）。

## 决策点

**用户范围与鉴权事件选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| ALL_USER + SERVICE_REQUEST 强制鉴权 | SUBRANGE=ALL_USER, AUTHEVENT=SERVICE_REQUEST, AUTHPERIOD=24 | WSFD-010301 鉴权功能（5G 服务请求强制鉴权） |
| ALL_USER + 算法配置 | SUBRANGE=ALL_USER, INTEGALG=..., ENCRYALG=... | WSFD-010303 NAS 信令加密与完整性保护 |
| IMSI_PREFIX + 算法配置 | SUBRANGE=IMSI_PREFIX, IMSIPRE=..., INTEGALG/ENCRYALG | 按用户群差异化安全策略 |

## 约束

- **下次安全管理流程生效**（info）：命令执行后在用户发起下一次涉及安全管理流程时生效 — 后果：非立即生效
- **算法配置有误导致接入异常**（critical）：加密性算法或完整性算法配置有误可能导致终端接入异常 — 后果：终端无法接入
- **NIA0 仅紧急业务**（warning）：根据 33.501 协议，NIA0 仅用于紧急业务，系统只对紧急业务支持 NIA0，不支持配置 — 后果：配置 NIA0 无效
- **指定号段优先**（info）：指定号段的用户采用对应的安全配置，未指定的用户采用默认安全配置 — 后果：未指定号段用户走缺省
- **系统缺省记录**（info）：系统部署完成后已存在一条 ALL_USER 缺省记录 — 后果：无需为全用户场景重复配置
- **最多 1000 条记录**（warning）：本命令最多可输入 1000 条记录 — 后果：超规格新增失败
