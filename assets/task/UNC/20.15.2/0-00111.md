# EXP MML 原子任务
> 命令静态知识见 [EXP MML](command/UNC/20.15.2/EXP-MML.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
EXP MML 是只读的配置导出工具（不改变运行配置），跨 80+ 特性的调测文档里使用方式完全一致——把它放在"收集信息寻求技术支持"的最后一步，把当前配置全量导出供分析。导出范围由 `MEID` 决定，文件落地由 `FNM` + SFTP 上传组决定。

| 用途 | 关键参数 | 典型脚本 |
|---|---|---|
| 全网元配置备份（默认本地存储） | 不带参数，或仅 `FNM` | `EXP MML:FNM="cfg_backup_20260709.mml";` |
| 导出指定网元 | `MEID` | `EXP MML:MEID=1,FNM="ne1_cfg.mml";` |
| 自动上传到 SFTP 服务器 | `ISULDFTP=TRUE` + `BKSERVERIPTYPE/IPV4/FTPUSER/FTPPSSWD/DSTF/FTPPORT` | `EXP MML:FNM="cfg.mml",ISULDFTP=TRUE,BKSERVERIPTYPE=IPV4,IPV4="10.1.1.100",FTPUSER="ftpuser",FTPPSSWD="***",DSTF="/backup/",FTPPORT=22;` |
| 移交/批量开局导出（密码匿名化） | `PWDENCRPT=TRUE`（默认） | `EXP MML:FNM="handover.mml",PWDENCRPT=TRUE;` |

## 决策点

### 导出范围
问：导出哪些网元的配置？
- 全部网元 → 不输入 `MEID`。
- 单个网元 → `MEID=网元ID`（0~65535）。

### 上传方式（本地存储 vs SFTP）
问：导出文件落到本端还是远端 SFTP 服务器？
- 本地默认存储 → 不配 SFTP 组，执行后及时下载文件链接。
- 远端 SFTP → `ISULDFTP=TRUE` 并补齐 `BKSERVERIPTYPE` + 对应 IP（`IPV4`/`IPV6`）+ `FTPUSER/FTPPSSWD/DSTF/FTPPORT`(默认 22)。
  - `BKSERVERIPTYPE=IPV4` 时必须配 `IPV4`；`IPV6` 时必须配 `IPV6`。

### 密码字段匿名化
问：导出文件是否保留密码明文？
- 默认 `PWDENCRPT=TRUE`（是，匿名化）——移交/批量开局/对外发送时保持默认。
- `PWDENCRPT=FALSE`（否）——仅在受控内部恢复场景按需关闭。

## 约束
- 配置导出文件系统默认保留 7 天：每天凌晨 2 点定时删除超 7 天文件，执行 EXP MML 时也会清理超期文件。`severity=medium` 后果：旧文件被自动清理。
- 文件系统最大存储 500M、最多保留 500 个文件：超限时按产生时间删最老文件直到本次可存。`severity=medium` 后果：老导出被覆盖丢失。
- 执行后请及时下载，文件链接失效需重新执行本命令。`severity=low`
- `MEID` 取值 0~65535；`FNM` 长度≤128 英文字符。`severity=low`
- 执行后立即生效（只读导出，无配置驻留）。`severity=low`
