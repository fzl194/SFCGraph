# 配置离线计费模板参数

> 离线计费参数模板（OFCTemplate）配置——话单版本+计费方式+阈值+话单触发，所有绑定层次的共享骨架。被引用于 [2-00001 离线计费](task/UNC/20.15.2/2-00001.md)。域通用（在线/融合计费也用 OFCTEMPLATE，后续特性可复用）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 话单版本+计费方式+定时器 | `ADD OFCTEMPLATE` | OFCTEMPLATENAME（本端规划）；GCDRVERSION/PGWCDRVERSION/SGWCDRVERSION（全网规划，GUL切换须 R8+）；TQM=QCT；QHTVALUE |
| 阈值 | `SET OFCTHRESHOLD` | THRESHINHERIT=NON_INHERIT；TIMETHRESHOLD/VOLUMETHRESHOLD/CONDITIONCHANGE/SERVINGNODECHNG |
| 话单产生条件 | `SET CDRTRIGGER` | CDRTRIGRATCHNG=DISABLE；CDRTRIGSNCHNG/CDRTRIGMAXCHNG=ENABLE；CDRTRIGTIMEZONE=DISABLE |
| 话单容器产生条件 | `SET CONTAINERTRIGGER` | CONTTRIGRATCHNG/CONTTRIGQOSCHNG |

**典型脚本**：
```
ADD OFCTEMPLATE: OFCTEMPLATENAME="offlinecharge-test",TQM=QCT,QCTVALUE=0;
SET OFCTHRESHOLD: OFCTEMPLATENAME="offlinecharge-test",THRESHINHERIT=NON_INHERIT,TIMETHRESHOLD=30,VOLUMETHRESHOLD=20,CONDITIONCHANGE=3,SERVINGNODECHNG=5;
SET CDRTRIGGER: OFCTEMPLATENAME="offlinecharge-test",CDRTRIGRATCHNG=DISABLE,CDRTRIGTIMEZONE=DISABLE,CDRTRIGPLMNCHNG=ENABLE;
SET CONTAINERTRIGGER: OFCTEMPLATENAME="offlinecharge-test";
```

**步骤位置**：离线计费配置第一步；下游是绑定 compound（[1-00002](task/UNC/20.15.2/1-00002.md)/[1-00003](task/UNC/20.15.2/1-00003.md)/[1-00004](task/UNC/20.15.2/1-00004.md)）或全局 atom（[0-00033](task/UNC/20.15.2/0-00033.md)）。

## 场景差异

- **热计费（WSFD-011202）**：阈值取更小值——activation 演示 `VOLUMETHRESHOLD=1000`（普通离线计费演示 20）、`TIMETHRESHOLD=30`、`CONDITIONCHANGE=3`、`SERVINGNODECHNG=3`。话单产生更快是热计费本质（CG 优先处理带 0x100 标志话单）。模板名/话单版本/计费方式 TQM 等其余参数同离线计费。被引用于 [2-00002 热计费](task/UNC/20.15.2/2-00002.md)。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| 计费方式 TQM=QCT | 按时长切话单，QCTVALUE=时长秒（0 表示用 QHT） |

> 联动特性级 DP：模板建好后，由 [2-00001](task/UNC/20.15.2/2-00001.md) 配置层次 DP 决定绑到哪个对象。

## 约束

- **模板名是引用键**（critical）：OFCTEMPLATENAME 须先建，下游所有绑定都引用此名 — 否则绑定命令找不到模板。
- **话单版本跨网元一致**（warning）：全网规划；GUL 切换网络须 R8+，否则话单不兼容。
