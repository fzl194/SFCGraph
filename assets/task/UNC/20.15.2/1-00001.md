# 配置离线计费模板参数

> 阶段定位：离线计费的参数模板（OFCTemplate）——话单版本+计费方式+阈值+话单触发，是所有绑定层次的共享骨架。
> 被引用于：[2-00001 离线计费](task/UNC/20.15.2/2-00001.md)。域通用 compound（在线/融合计费也用 OFCTemplate，后续特性可复用）。

## 配置方法

命令序列（建模板 → 阈值 → 话单触发 → 容器触发）：

1. [ADD OFCTEMPLATE](task/UNC/20.15.2/0-00049.md) — 话单版本（G/PGW/SGW-CDR）+ 计费方式 TQM + QHT 定时器 + Record Sequence Number
2. [SET OFCTHRESHOLD](task/UNC/20.15.2/0-00191.md) — 时长/流量/计费条件改变/ServingNode 改变 阈值 + THRESHINHERIT
3. [SET CDRTRIGGER](task/UNC/20.15.2/0-00156.md) — 话单产生条件（RAT/ServingNode/计费条件/MS时区/PLMN 更新）
4. [SET CONTAINERTRIGGER](task/UNC/20.15.2/0-00167.md) — 话单容器产生条件（RAT/QoS 更新）

**典型脚本**
```
ADD OFCTEMPLATE: OFCTEMPLATENAME="offlinecharge-test",TQM=QCT,QCTVALUE=0;
SET OFCTHRESHOLD: OFCTEMPLATENAME="offlinecharge-test",THRESHINHERIT=NON_INHERIT,TIMETHRESHOLD=30,VOLUMETHRESHOLD=20,CONDITIONCHANGE=3,SERVINGNODECHNG=5;
SET CDRTRIGGER: OFCTEMPLATENAME="offlinecharge-test",CDRTRIGRATCHNG=DISABLE,CDRTRIGTIMEZONE=DISABLE,CDRTRIGPLMNCHNG=ENABLE;
SET CONTAINERTRIGGER: OFCTEMPLATENAME="offlinecharge-test";
```

**取值要点**：OFCTEMPLATENAME 本端规划（后续所有绑定引用此名）；话单版本 GCDRVERSION/PGWCDRVERSION/SGWCDRVERSION 全网规划，GUL 切换网络须 R8+；TQM=QCT 按时长切话单。

## 约束
- OFCTEMPLATENAME 是后续绑定 compound 的引用键，**必须先建模板再绑定**
- 话单版本跨网元一致（全网规划）
