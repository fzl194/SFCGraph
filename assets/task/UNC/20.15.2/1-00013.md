# 配置 PCRF 选择与分组

> 把已建 PCRF（[1-00012](task/UNC/20.15.2/1-00012.md)）组成 PCRF 分组（主备/负荷分担），并按号段/APN/全局缺省绑定选择方式。被引用于 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md)。特性专属（PCRF 分组用 PCRF 族命令；融合计费 CHF 选择用 TNF 族命令 [1-00007](task/UNC/20.15.2/1-00007.md)，结构相似但命令集不同）。仅 2G/3G/4G 代际用 Gx 接口；5G 代际 PCF 经 NRF 发现，无需建 PCRF 分组。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| （可选）配 IMSI/MSISDN 号段 | `ADD IMSIMSISDNSEG` | SEGMENTNAME；SEGMENTTYPE=MSISDN/IMSI；SEGSTART；SEGEND（全网规划） |
| 建 PCRF 分组 | `ADD PCRFGROUP` | PCRFGRPNAME；LOADBALANCEMODE=LOAD_BALANCE/MASTER_SLAVE |
| PCRF 加入分组 | `ADD PCRFBINDGRP` | PCRFGRPNAME（须与 ADD PCRFGROUP 一致）；PCRFHOSTNAME（须已建，[1-00012](task/UNC/20.15.2/1-00012.md)）；WEIGHT（负荷分担时权重） |
| （可选）主备模式设主用 PCRF | `SET MASTERPCRF` | PCRFGRPNAME；MASTERPCRF（主机名）；仅 LOADBALANCEMODE=MASTER_SLAVE 时配 |
| （可选）全局号段绑 PCRF 组 | `ADD GLBPCRFGROUP` | IMSIMSISDNSEG（号段名）；PCRFGRPNAME；PRIORITY（多号段时优先级，值小优先） |
| 设全局缺省 PCRF 组 | `SET DFTGLBPCRFGRP` | PCRFGRPNAME（号段/APN 都未匹配时的兜底组） |
| （可选）APN 绑 PCRF 组 | `ADD PCRFGRPBNDAPN` | APN；DEFAULTFLAG=DEFAULT/IMSI_MSISDN_SEG；PCRFGRPNAME；（DEFAULTFLAG=IMSI_MSISDN_SEG 时）IMSIMSISDNSEG |

**典型脚本（负荷分担组 pcrf_group_1 + 主备组 pcrf_group_2 + 号段选择 + 全局缺省）**：
```
ADD IMSIMSISDNSEG: SEGMENTNAME="msisdnseg1", SEGMENTTYPE=MSISDN, SEGSTART="18900000000", SEGEND="18949999999";
ADD IMSIMSISDNSEG: SEGMENTNAME="msisdnseg2", SEGMENTTYPE=MSISDN, SEGSTART="18950000000", SEGEND="18999999999";
ADD PCRFGROUP: PCRFGRPNAME="pcrf_group_1", LOADBALANCEMODE=LOAD_BALANCE;
ADD PCRFBINDGRP: PCRFGRPNAME="pcrf_group_1", PCRFHOSTNAME="pcrf_1", WEIGHT=40;
ADD PCRFBINDGRP: PCRFGRPNAME="pcrf_group_1", PCRFHOSTNAME="pcrf_2", WEIGHT=60;
ADD PCRFGROUP: PCRFGRPNAME="pcrf_group_2", LOADBALANCEMODE=MASTER_SLAVE;
ADD PCRFBINDGRP: PCRFGRPNAME="pcrf_group_2", PCRFHOSTNAME="pcrf_3";
ADD PCRFBINDGRP: PCRFGRPNAME="pcrf_group_2", PCRFHOSTNAME="pcrf_4";
SET MASTERPCRF: PCRFGRPNAME="pcrf_group_2", MASTERPCRF="pcrf_3";
ADD GLBPCRFGROUP: IMSIMSISDNSEG="msisdnseg1", PCRFGRPNAME="pcrf_group_1", PRIORITY=1;
ADD GLBPCRFGROUP: IMSIMSISDNSEG="msisdnseg2", PCRFGRPNAME="pcrf_group_2", PRIORITY=2;
SET DFTGLBPCRFGRP: PCRFGRPNAME="pcrf_group_1";
ADD PCRFGRPBNDAPN: APN="apn-test1", DEFAULTFLAG=DEFAULT, PCRFGRPNAME="pcrf_group_2";
```

**步骤位置**：在 [1-00012](task/UNC/20.15.2/1-00012.md)（PCRF 对接链）之后、[1-00014](task/UNC/20.15.2/1-00014.md)（PCC 开关）之前或并行；PCC 用户激活选 PCRF 组时生效。

## 场景差异

- **负荷分担（LOAD_BALANCE）**：`ADD PCRFBINDGRP` 配 WEIGHT 权重，组内 PCRF 按权重分担用户。被引用于 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md)。
- **主备（MASTER_SLAVE）**：`SET MASTERPCRF` 设主用 PCRF；主用故障切备用。无需 WEIGHT。被引用于 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md)。
- **选择方式优先级**：APN 下绑定的 PCRF 组（`ADD PCRFGRPBNDAPN`）> 全局号段绑定的 PCRF 组（`ADD GLBPCRFGROUP`）> 全局缺省 PCRF 组（`SET DFTGLBPCRFGRP`）。
- **5G 代际（Npcf 接口）**：不建 PCRF 分组——PCF 经 NRF 服务化发现。5G 场景的 PCF 选择走 [ADD PCFSELPLCY](task/UNC/20.15.2/0-00054.md)（AMF 选 PCF）+ `SET APNPCCFUNC` 的 PCFSELECTMODE（SMF 按 DNN/SERVINGSCOPE 选 PCF），属单命令步骤不进本 compound。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| LOADBALANCEMODE=LOAD_BALANCE | ADD PCRFBINDGRP 配 WEIGHT 权重；不配 SET MASTERPCRF |
| LOADBALANCEMODE=MASTER_SLAVE | SET MASTERPCRF 设主用；不配 WEIGHT |
| DEFAULTFLAG=DEFAULT | `ADD PCRFGRPBNDAPN` 该 APN 下不区分号段统一选此 PCRF 组；APN 下只允许一个不带号段的 PcrfGroup |
| DEFAULTFLAG=IMSI_MSISDN_SEG | `ADD PCRFGRPBNDAPN` 该 APN 下按号段选不同 PCRF 组；须配 IMSIMSISDNSEG |

> 联动特性级 DP：PCRF 分组的工作模式（主备/负荷分担）与选择粒度（全局号段/APN/全局缺省）由 [2-00005](task/UNC/20.15.2/2-00005.md) 的 PCRF 选择 DP 驱动。

## 约束

- **PCRF 须先建**（critical）：`ADD PCRFBINDGRP` 的 PCRFHOSTNAME 须已在 [1-00012](task/UNC/20.15.2/1-00012.md) `ADD PCRF` 建立 — 否则加入分组失败。
- **PCRFGRPNAME 跨命令一致**（critical）：ADD PCRFGROUP / ADD PCRFBINDGRP / SET MASTERPCRF / ADD GLBPCRFGROUP / SET DFTGLBPCRFGRP / ADD PCRFGRPBNDAPN 的 PCRFGRPNAME 须完全一致。
- **APN 下不带号段的 PcrfGroup 唯一**（warning）：一个 APN 下只允许一个 DEFAULTFLAG=DEFAULT 的 PcrfGroup。
- **全局缺省 PCRF 组建议配**（info）：作为号段/APN 都未匹配时的兜底，避免用户无法选到 PCRF。
