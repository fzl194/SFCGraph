# SET PCCFUNC 原子任务

> 命令静态知识见 [SET PCCFUNC](command/UNC/20.15.2/SET-PCCFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET PCCFUNC 是 PCC 体系的"总开关 + 公共参数"命令。最常见用法是使能/关闭全局用户的动态 PCC 功能（HOME/ROAM/VISIT PCCSWITCH）。本地 PCC 场景下这三开关置 DISABLE（再由 SET APNPCCFUNC 开 APN 粒度）。其余参数按场景按需配置：COMPOLICY、MKPARSEFORMAT、PCFLBPARA、N7FAILOVERSW、LOCSLCTMODE/LOCALHOSTNAME/UPFGLOCGBNDGNAME 等。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| HOMEPCCSWITCH | `ENABLE` / `DISABLE` | 本端规划 | 本地用户动态 PCC 功能 |
| ROAMPCCSWITCH | `ENABLE` / `DISABLE` | 本端规划 | 漫游用户动态 PCC 功能 |
| VISITPCCSWITCH | `ENABLE` / `DISABLE` | 本端规划 | 拜访用户动态 PCC 功能 |
| COMPOLICY | `LOCALAAA` | 对端协商/与对端协商 | 公共策略 |
| MKPARSEFORMAT | `UNSIGNED32` | 与对端协商 | Monitoring-Key 解析方式，须与 PCRF/PCF 一致 |
| PCFLBPARA | `GROUPID` | 与对端协商 | PCF 负荷分担参数 |
| N7FAILOVERSW | `ENABLE` | 与对端协商 | N7 Failover 开关（PCF 主备容灾时） |
| LOCSLCTMODE | `UPFGRP` | 固定取值 | PCC 本端主机名选择模式 |
| UPFGLOCGBNDGNAME | `test` | 已配置数据 | UPF 组与本端主机组绑定关系组名 |
| LOCALHOSTNAME | （1~127 字符） | 本端规划 | 本端主机名（LOCSLCTMODE 条件依赖） |

**特性场景取值对比**

| 特性场景 | HOME/ROAM/VISIT | 其他关键参数 | 说明 |
|---|---|---|---|
| PCC 基本功能 5G（WSFD-109101） | ENABLE/ENABLE/ENABLE | COMPOLICY=LOCALAAA, N7FAILOVERSW=ENABLE, PCFLBPARA=GROUPID | 5G 全开 |
| PCC 基本功能 2/3/4G（WSFD-109101） | ENABLE/ENABLE/ENABLE | COMPOLICY=LOCALAAA, LOCSLCTMODE=UPFGRP | 2/3/4G 全开 |
| 本地 PCC 功能（WSFD-109101） | DISABLE/DISABLE/DISABLE | — | 关全局动态 PCC，开本地 PCC |
| VoLTE 基础语音（WSFD-102001） | ENABLE/—/— | — | 仅开本地用户 |
| Gx over DRA（WSFD-011132） | ENABLE/ENABLE/ENABLE | — | 经 DRA 接 PCRF |
| 累计流量策略控制（WSFD-109104/211009） | — | MKPARSEFORMAT=UNSIGNED32 | 仅配 MK 解析 |
| N7 PCF Group（WSFD-213002） | — | PCFLBPARA=GROUPID | 仅配负荷分担 |
| 语音 PCF Bypass N7（WSFD-201207） | — | N7FAILOVERSW=ENABLE | 仅配 Failover |

## 决策点

### 动态 PCC 开关取值（ENABLE 全开 vs DISABLE 转本地 PCC）

HOME/ROAM/VISIT 三开关的取值取决于网络是否部署 PCRF/PCF：
- 部署 PCRF/PCF → 三开关 ENABLE（动态 PCC）。
- 未部署 PCRF/PCF → 三开关 DISABLE，转为全局本地 PCC 功能，再由 SET APNPCCFUNC 开 APN 粒度。

证据显示该参数获取方法为本端规划，属决策驱动。

### Monitoring-Key 解析方式（须与 PCRF/PCF 协商）

MKPARSEFORMAT 取值需与 PCRF/PCF 保持一致（证据中 WSFD-109104/211009 均取 UNSIGNED32）。不一致会导致累计流量监控失败。获取方法为"与对端协商"。

### PCF 负荷分担参数（GROUPID vs PRIORITY）

PCFLBPARA 决定 PCF 主备容灾的负荷分担依据：
- `GROUPID` → 主备 PCF 的 GroupID 必须相同。
- `PRIORITY` → 主备 PCF 的 PRIORITY 必须不同。

证据中取 GROUPID。属对端协商。

### N7 Failover 开关（PCF 主备容灾组网）

N7FAILOVERSW 仅 PCF 采用主备容灾组网时置 ENABLE；PCF 故障且无可用备 PCF 时置 DISABLE。支持 APN 粒度（ADD PCCTEMPLATE）和全局粒度（本命令），需按实际 PCF 能力规划。

### 本端主机名选择模式（LOCSLCTMODE 及其条件依赖）

LOCSLCTMODE=UPFGRP 时，需配套 UPFGLOCGBNDGNAME（已配置数据中获取，来自 ADD PCCPBINDUPG 之类绑定关系）。LOCSLCTMODE/LOCALHOSTNAME 仅对 Gx 接口生效，修改后只对新激活用户生效。

## 约束

- **只对新激活用户生效** · severity=HIGH · 命令执行后只对新激活用户生效；修改 LOCSLCTMODE/LOCALHOSTNAME 后亦只对新激活用户生效，已激活用户不受影响。
- **最大 1 条记录** · 全局唯一配置。
- **ARPEXTAVPSW/LOCSLCTMODE/LOCALHOSTNAME 仅 Gx 接口生效** · severity=MEDIUM · N7 接口承载绑定默认使用 ARP 信元所有参数。
- **APN 粒度优先级高于全局** · severity=MEDIUM · SET APNPCCFUNC 优先于本命令；APN 未配时继承全局。
- **MKPARSEFORMAT 须与对端一致** · severity=HIGH · 不一致致累计流量监控失效。
- **PCF 主备容灾约束** · severity=MEDIUM · PCFLBPARA=GROUPID 时主备 PCF GroupID 须同；=PRIORITY 时 PRIORITY 须不同。
- **License 前置** · severity=MEDIUM · 需 SET LICENSESWITCH 开 PCC 基本 License 后 PCC 功能才生效。
- **系统初始记录** · 命令存在系统初始记录。
