# 设置话单发送控制参数（SET CDRTRANSFER）

> 命令静态知识见 [SET CDRTRANSFER](command/UNC/20.15.2/SET-CDRTRANSFER.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

全局单记录参数，按"报文长度 + 重传控制 + CG 选择 + Checksum"四类规划。存在系统初始记录（1400/3/3/5/PDP_BASED_LB/ENABLE），五字段全可选，不输入不改、输入则改对应字段。

| 维度 | 参数 | 取值 | 默认（初始值） | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 报文长度 | GTPPMAXPAYLOAD | 1200~7180，整数，本端规划 | 1400 | 每个 GTP' 消息携带话单的最大字节数（仅对新消息生效） | 调优避免 IP 分片 |
| 重传次数 | RETRANSTIMES | 1~3，整数，本端规划 | 3 | Echo/Data Record Transfer Request 重发次数（达上限未收到 CG 响应则判 CG 异常） | 控制 CG 链路探测灵敏度 |
| 重传间隔（话单） | RETRANSINTERVAL | 1~10 秒，整数，本端规划 | 3 | Data Record Transfer Request 重发时间间隔 | 话单重发节奏 |
| 重传间隔（Node Alive） | NARESTRANSINTVL | 5~600 秒，整数，本端规划 | 5 | Node Alive 消息超时重发时间 | 链路活性探测节奏 |
| CG 选择模式 | CGSELECTIONMODE | 枚举：MSG_BASED_LB / PDP_BASED_LB，本端规划 | PDP_BASED_LB | 发送话单时如何选择 CG | 多 CG 负荷分担策略 |
| Checksum 开关 | UDPCHECKSUM | 枚举：ENABLE / DISABLE，本端规划 | ENABLE | GTP' 报文是否携带 Checksum | 协议合规开关 |

典型脚本（CG 负荷分担 + GTP' 重传控制）：
```
SET CDRTRANSFER: GTPPMAXPAYLOAD=1400, RETRANSTIMES=3, RETRANSINTERVAL=2, NARESTRANSINTVL=20, CGSELECTIONMODE=MSG_BASED_LB;
```

步骤位置：建立 CG 对接（`ADD CG（★R1.5 atom 待补）`、`ADD CGGRPBINDING`，不在本任务范围）→ 本任务（控制链路探测与话单重发参数）

## 决策点

### 决策点：CG 选择模式（DP 0-03049）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 基于消息的负荷分担（MSG_BASED_LB） | CGSELECTIONMODE=MSG_BASED_LB | 从状态正常、优先级最高的 CG 中选当前发送窗口负载最少的 CG；适合多 CG 性能不均的场景 |
| 基于 PDP/Bearer 的负荷分担（PDP_BASED_LB） | CGSELECTIONMODE=PDP_BASED_LB | 激活时获取的 CG 状态正常则固定向此 CG 发送；状态异常则按优先级轮选；适合会话亲和场景。注意：此模式下 PCRF 下发的备 CG 不生效 |

### 决策点：GTP' 消息最大字节数调优（DP 0-03050）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 小于网络 MTU（避免分片） | GTPPMAXPAYLOAD=1200~MTU 上限 | CG 性能不足或 IP 分片导致 CG 响应不及时、CG 闪断时建议调小 |
| 调大（单消息多话单） | GTPPMAXPAYLOAD 接近 7180 | CG 处理能力强、链路 MTU 充足时，提升话单传输吞吐；但对端 CG 处理能力弱时可能触发 ALM-81021 CG 无响应 |

## 约束

- **规格上限**（info，rule-0-03121）：规格上限：最大记录数 1（全局唯一记录，重复配即覆盖）
- **增量修改**（info，rule-0-03122）：增量修改：五字段全可选，参数未输入不改、输入则改对应字段
- **副作用**（warning，rule-0-03123）：副作用：GTPPMAXPAYLOAD 调大可能导致对端 CG 处理不及时，触发 ALM-81021 CG 无响应告警；建议修改前联系华为技术工程师
- **失效场景**（warning，rule-0-03124）：失效场景：CGSELECTIONMODE=PDP_BASED_LB 时，PCRF 下发的备 CG 不生效
- **行为说明**（info，rule-0-03125）：行为说明：CG 故障或负载过重时，话单重传实际间隔可能大于配置值
