# SET CDRTRANSFER 原子任务
> 命令静态知识见 [SET CDRTRANSFER](command/UNC/20.15.2/SET-CDRTRANSFER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：SGW-C、PGW-C、SMF。设置话单发送控制参数。Echo/Data Record Transfer Request 重传次数默认 3，超过则判 CG 状态异常并触发告警 ALM-81021 CG 无响应。CG 故障或负载过重时话单重传间隔会延长。存在系统初始记录。五个字段全可选，不输入不改、输入则改对应字段。证据显示获取方法单一（本端规划 5），属 global_planned。

## 配置方法
全局单记录参数，按"CG 链路探测/话单重发/负荷分担/报文长度"四类规划。

| 参数 | 配置方法 | 取值范围 / 样例 |
|---|---|---|
| GTPPMAXPAYLOAD | GTP' 消息最大可携带话单字节数（本端规划） | 1200～7180，样例 `1400` |
| RETRANSTIMES | Echo and Data Record Transfer Request 重传次数（本端规划） | 1～3，样例 `3` |
| RETRANSINTERVAL | Data Record Transfer Request 重传时间间隔(秒)（本端规划） | 1～10，样例 `2` |
| NARESTRANSINTVL | Node Alive 消息重传时间间隔(秒)（本端规划） | 5～600，样例 `20` |
| CGSELECTIONMODE | CG 选择模式（本端规划，枚举） | `MSG_BASED_LB`（基于消息的负荷分担） |
| UDPCHECKSUM | GTP' 报文 CheckSum 开关（本端规划，枚举） | — |

典型脚本（CG 负荷分担 + GTP' 重传控制）：
```
SET CDRTRANSFER:GTPPMAXPAYLOAD=1400,RETRANSTIMES=3,RETRANSINTERVAL=2,NARESTRANSINTVL=20,CGSELECTIONMODE=MSG_BASED_LB;
```

## 决策点

### CG 选择模式（CGSELECTIONMODE）
- 问什么：多 CG 场景下 CG 的选择/负荷分担算法。
- 取值如 `MSG_BASED_LB`（基于消息的负荷分担），枚举。按局点 CG 组网与负荷分担策略规划（与 ADD CG / ADD CGGRPBINDING 配套）。
- 仅当多 CG / CG 组场景需规划；单 CG 可不配或沿用初始值。

### 重传次数与间隔（RETRANSTIMES / RETRANSINTERVAL / NARESTRANSINTVL）
- 问什么：Echo/Data Record 与 Node Alive 消息的重发次数和间隔。
- RETRANSTIMES ∈ 1～3（默认 3），RETRANSINTERVAL ∈ 1～10s，NARESTRANSINTVL ∈ 5～600s。
- 设大值加重 CG 负担、判异常更慢；设小值易误判 CG 故障（CG 响应不及时触发 ALM-81021 CG 无响应 / CG 闪断）。CG 性能不足或 IP 分片时建议调小 GTPPMAXPAYLOAD。

### GTP' 消息最大字节数（GTPPMAXPAYLOAD）
- 问什么：每条 GTP' 话单消息最大携带字节数。
- 1200～7180，建议小于网络 MTU 避免分片；过大易 IP 分片导致 CG 响应不及时、CG 闪断。

## 约束
- 立即生效。`severity=low`
- 最大记录数 1：全局唯一记录，重复配即覆盖。`severity=medium`
- 五字段全可选：不输入不改、输入则改对应字段（增量修改）。`severity=medium`
- CG 故障/负载过重时话单重传间隔会延长：实际重传间隔可能大于配置值。`severity=medium`
- 重传超阈值触发告警 ALM-81021 CG 无响应：判 CG 链路异常。`severity=high`
- GTPPMAXPAYLOAD 建议小于 MTU：避免 IP 分片导致 CG 响应不及时。`severity=medium`
