# 配置ADC参数（ADD ADCPARA）

> 命令静态知识见 [ADD ADCPARA](command/UNC/20.15.2/ADD-ADCPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADC（应用检测与控制）特性激活时，本命令绑定一条已存在的流过滤器并控制流信息上报开关。核心配置空间仅 2 个维度：**引用哪个流过滤器**（由 `ADD FLOWFILTER` 先建）和**是否向 PCRF/PCF 上报流信息**。

### 配置维度 1：流过滤器引用（参数 FLOWFILTERNAME，必选）

| 取值 | 作用 | 说明 |
|---|---|---|
| 已有的 FLOWFILTERNAME | 引用已通过 `ADD FLOWFILTER` 配置的流过滤器 | 先执行 `ADD FLOWFILTER` 建立流过滤器，否则引用失败 |

> 前置依赖：FLOWFILTERNAME 必须已通过 `ADD FLOWFILTER` 配置存在。

### 配置维度 2：流信息上报开关（参数 FLOWINFORPT）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **DISABLE**（默认） | 不向 PCRF/PCF 上报流信息 | 仅做 ADC 计费/统计（PCRF 不参与） |
| **ENABLE** | 在向 PCRF/PCF 上报应用开始/结束时，同时上报流信息 | PCRF/PCF 需基于流信息下发精确策略 |

典型脚本：
```
ADD ADCPARA: FLOWFILTERNAME="flowfilter_ADC01", FLOWINFORPT=DISABLE;
ADD ADCPARA: FLOWFILTERNAME="flowfilter_ADC01", FLOWINFORPT=ENABLE;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：配置 ADC 参数"——先 ADD FLOWFILTER，再 ADD ADCPARA 绑定。

## 决策点：流信息上报开关（DP 0-01004）

问：是否需要 PCRF/PCF 实时获取应用与流的对应关系以做动态策略下发？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不上报（DISABLE） | `FLOWINFORPT=DISABLE` | 仅做 ADC 计费/统计（PCRF 不参与） |
| 上报（ENABLE） | `FLOWINFORPT=ENABLE` | PCRF/PCF 需基于流信息下发精确策略 |

## 约束

- **FLOWFILTERNAME 前置依赖**（critical，rule-0-01007）：FLOWFILTERNAME 必须已通过 `ADD FLOWFILTER` 配置存在，否则引用失败 — 后果：ADC 参数绑定失败，业务检测不生效
- **ADD ADCPARA 最大记录数规格**（warning，rule-0-01008）：单板最大 8000 — 后果：超规格新增失败
