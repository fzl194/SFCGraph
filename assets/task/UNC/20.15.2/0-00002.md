# ADD ADCPARA 原子任务
> 命令静态知识见 [ADD ADCPARA](command/UNC/20.15.2/ADD-ADCPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
ADC（应用检测与控制）特性激活时，本命令绑定一条已存在的流过滤器并打开流信息上报开关。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| FLOWFILTERNAME | 引用已配置的流过滤器名（先执行 `ADD FLOWFILTER`） | `flowfilter_ADC01` |
| FLOWINFORPT | 是否向 PCRF/PCF 上报流信息 | `ENABLE`（需精确策略时）/ `DISABLE`（默认） |

典型脚本：
```
ADD ADCPARA: FLOWFILTERNAME="flowfilter_ADC01", FLOWINFORPT=ENABLE;
```

## 决策点（若有）

### 流信息上报开关
- 问什么：是否需要 PCRF/PCF 实时获取应用与流的对应关系以做动态策略下发。
- 仅做 ADC 计费/统计（PCRF 不参与）→ `FLOWINFORPT=DISABLE`。
- PCRF/PCF 需基于流信息下发精确策略 → `FLOWINFORPT=ENABLE`。

## 约束（若有）
- 前置依赖：FLOWFILTERNAME 必须已通过 `ADD FLOWFILTER` 配置存在，否则引用失败。`severity=high`
- 规格上限：最大记录数 8000。`severity=medium`
- 生效时机：执行后立即生效。`severity=low`
