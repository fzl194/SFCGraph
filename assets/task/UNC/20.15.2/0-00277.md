# 设置IPSEC接口IPv6使能（SET IFIPV6ENABLEIPSEC）

> 命令静态知识见 [SET IFIPV6ENABLEIPSEC](command/UNC/20.15.2/SET-IFIPV6ENABLEIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令修改 IPSec 域接口（Ethernet/Eth-Trunk/Loopback/Tunnel 等）的 IPv6 使能标志，并可选修改 IPv6 MTU。配置空间主要由**使能标志**这一维度决定。

### 配置维度1：IPv6 使能标志（参数 ENABLEFLAG）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| TRUE | 使能接口 IPv6（缺省不使能） | IPv6 IPSec 隧道场景（接口需先使能 IPv6 才能配 IPv6 地址） |
| FALSE | 去使能接口 IPv6 | 关闭接口 IPv6 |

### 配置维度2：IPv6 MTU（参数 IFMTU6，仅 ENABLEFLAG=TRUE 时可选）

| 取值 | 作用 |
|---|---|
| 1280~9600（缺省 1500） | 接口 IPv6 最大传输单元 |

> 建议 MTU 配置在 1500~9000 之间；若 IPv4 MTU 与 IPv6 MTU 都配置，实际取两者较大值配置至 vNic。

**步骤位置**：在所属 feature（IPFD-016000 IPSec功能）编排中，于 `ADD INTERFACEIPSEC`（建接口）之后、`ADD IFIPV6ADDRESSIPSEC`（配 IPv6 地址）之前执行。

（无。本命令用法由 IPv6 规划决定，无特性级分支。）

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无延迟
- **适用接口范围**（info）：在 VNRS_VNFC 的 Ethernet/Ethernet 子接口/Eth-Trunk/Eth-Trunk 子接口/Loopback/Tunnel 口上配置 — 后果：范围外接口配置失败
- **缺省不使能 IPv6**（info）：缺省情况下接口上不使能 IPv6 — 后果：需显式使能才能配 IPv6 地址
- **MTU 取 IPv4/IPv6 较大值**（warning）：若 IPv4 MTU 与 IPv6 MTU 都配置，实际取两者较大值 — 后果：实际 MTU 可能与预期不符
- **MTU 配置过大或过小可能失败**（warning）：不同网卡支持的 MTU 范围不一样，建议配置 1500~9000 — 后果：配置可能不成功
- **自动化配置期间禁止操作**（critical）：本参数参与自动化配置，下发前需用 `DSP OPSASSISTSTATE` 查询 autoscaling_autoconfig.py 是否在自动化配置过程中；如在自动化配置过程中不要做增删改 — 后果：生效配置不可预期
