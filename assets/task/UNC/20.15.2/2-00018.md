# IPv6 承载上下文（WSFD-104001）

> 特性静态知识见 [WSFD-104001](feature/UNC/20.15.2/WSFD-104001.md)。本页讲配置生成实例化时这个特性**怎么配**——按 5G/4G/3G/2G 制式 + License 套数编排，本特性**无独立 MML 配置命令**（参考信息明确"本特性无相关命令"），仅由 License 开关 + 上游地址分配特性共同承载。

## 配置概览

核心对象链：**(5G) AMF/SMF** 或 **(4G) MME/SGW-C/PGW-C** 或 **(2-3G) SGSN/GGSN** → PDU 会话/PDN 连接/PDP 上下文建立 → 控制面分配 IPv6 地址 → 通过 N4/S1/S5/S8/Gn 接口下发 IPv6 路由广播（IPv6 RA 前缀）→ UPF/SGW-U/PGW-U 透传给 UE。

**核心配置空间**：本特性**仅有 License 前置**（3 套 License：5G AMF+SMF / 4G MME+SMF / 2-3G SGSN+GGSN），**无独立 MML 配置命令**（参考信息 48043348/48043353 确认"本特性无相关命令"）。所有 IPv6 地址分配 + 路由发布 + APN 实例配置由被依赖特性（[UNC 1-00024](task/UNC/20.15.2/1-00024.md) 接入基础设施 + [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 地址池 + [UNC 1-00029](task/UNC/20.15.2/1-00029.md) OSPF 路由）共同承担。

**License 矩阵**（按 5G/4G/2-3G 拆解）：

| 制式 | NF | License 控制项 |
|---|---|---|
| 5G | AMF | `LKV2IPV6AM01` IPv6 承载上下文-UAM |
| 5G | SMF | `LKV2IPV6SM01` IPv6 承载上下文-USM |
| 4G | MME、SGW-C、PGW-C | `LKV2IPV6SM01` IPv6 承载上下文-USM（注：4G 激活示例使用 `LKV2IPV601`，**与 5G 名称不同**，全网规划需明确） |
| 2-3G | SGSN、GGSN | `LKV2IPV601`（需核实，参考信息 48043353 仅列 License 测量单元）|

> **5G vs 4G/2-3G License 名称差异**：5G 激活示例（AMF/SMF 48043375）使用 `LKV2IPV6AM01` + `LKV2IPV6SM01`；4G 激活示例（MME/SGW-C/PGW-C 48043351）使用 `LKV2IPV601` + `LKV2IPV6SM01`。**`LKV2IPV601` 与 `LKV2IPV6AM01` 关系待核**（可能分别对应不同代际产品）。

> **本特性 = License 前置特性（License-only feature）**：所有 3 份参考信息 md 明确"本特性无相关命令"，因此本特性 wiki **不建独立配置流程步骤 1-N**，配置流程直接复用被依赖的 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) + [UNC 1-00023](task/UNC/20.15.2/1-00023.md) + [UNC 1-00029](task/UNC/20.15.2/1-00029.md) 骨架，本特性在骨架之上叠加 License 前置步骤。

## 配置流程（步骤编排，每步显示含命令）

> **配置生成原则**：本特性无独立 MML 命令，配置生成器遇到本特性时应：(1) 先按制式确定 License 套数并生成 License 前置；(2) 直接跳转到被依赖的地址分配骨架（[UNC 1-00024](task/UNC/20.15.2/1-00024.md) → [UNC 1-00023](task/UNC/20.15.2/1-00023.md) → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)）生成 IPv6 单栈地址分配配置；(3) IPv6 RA 前缀广播由控制面生成后经 N4/S1/S5/S8/Gn 接口下发 UPF/SGW-U/PGW-U，UPF 侧无需 UNC 配。

1. **License 前置**（按制式选一）：
   - 5G AMF/SMF：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)（LICITEM=LKV2IPV6AM01,SWITCH=ENABLE；LICITEM=LKV2IPV6SM01,SWITCH=ENABLE）
   - 4G MME/SGW-C/PGW-C：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)（LICITEM=LKV2IPV601,SWITCH=ENABLE；LICITEM=LKV2IPV6SM01,SWITCH=ENABLE）
   - 2-3G SGSN/GGSN：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)（License 项名称待核）
2. **APN 接入域基础设施**（IPv6 单栈 VPN/APN 实例）：`ADD L3VPNINST` + `ADD VPNINSTAF`（AFTYPE=ipv6uni）+ `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR`（SUPPORTIPV6=ENABLE）→ [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
3. **IPv6 地址池**（UNC ADDRPOOL 系）：`ADD ADDRPOOL`（IPVERSION=IPv6、POOLTYPE=Local）+ `ADD SECTION`（IPv6 段）→ [UNC 1-00023](task/UNC/20.15.2/1-00023.md)
4. **接入域路由发布**（OSPFv3 单栈发布 IPv6 用户路由）：`ADD OSPFV3` 进程（UNC OSPFv3 真实命令族，全大写） + `ADD OSPFV3AREA` + `ADD OSPFV3INTERFACE` + `ADD OSPFV3IMPORTROUTE`（PROTOCOL=wlr） → [1-00032](task/UNC/20.15.2/1-00032.md) UNC OSPFv3 路由发布
   - 关键参数：PROCID（IPv6 OSPFv3 进程）, ROUTERID 全网唯一；AREAID；IFNAME；PROTOCOL=wlr（用户本地路由）
   - **★ 命令名修正（R1.5 必加脚注，2026-07-12 v1.0→v1.1）**：v1.0 写的 `ADD OSPFv3` + `ADD OSPFv3AREA` + `ADD OSPFv3NETWORK` 是错的——UNC 真实命令名是 **`ADD OSPFV3` + `ADD OSPFV3AREA` + `ADD OSPFV3INTERFACE`**（全大写，非小驼峰；UNC 无 OSPFv3NETWORK 命令族）。命令文档路径：`平台服务管理/VNRS功能管理/IP服务/路由管理/OSPFv3管理/`。
   - **★ OSPFv3 atom 待补**：本步骤命令的 atom 在 UNC pre-build 阶段未建（_numbering.json 未收录 0-00249 之外的 OSPFV3* 系列），须按命令级 atom SOP §5 补 0-00281 起接续入 _numbering.json；本骨架归 [1-00032](task/UNC/20.15.2/1-00032.md) UNC OSPFv3 路由发布，配置生成遇 OSPFv3 步骤时**待 atom 补齐后**才产出。
5. **IPv6 RA 前缀广播**（运行时行为，UNC 不配）：控制面（SMF/MME/SGSN）生成 IPv6 RA 前缀 → 经 N4/S1/S5/S8/Gn 接口下发到 UPF/SGW-U/PGW-U → UPF 透传到 UE；本步骤**无 UNC MML 命令**

> **共享骨架 + License 前置**模式：本特性与 [WSFD-104002 IPv4v6 双栈接入](task/UNC/20.15.2/2-00017.md)/[WSFD-104004 IPv6 前缀代理](feature/UNC/20.15.2/WSFD-104004.md)/[WSFD-010502 地址分配方式](feature/UNC/20.15.2/WSFD-010502.md) 共享 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) + [UNC 1-00023](task/UNC/20.15.2/1-00023.md) + [UNC 1-00029](task/UNC/20.15.2/1-00029.md) 三骨架，差异在 License 前置（5G vs 4G/2-3G 套数不同）。

## 决策点（特性级：制式 × License 套数）

| 接入制式 | 走法 | License 套数 | 关键差异 |
|---|---|---|---|
| 5G（AMF/SMF）| 步骤 1（5G License 2 项）+ 步骤 2-4（IPv6 单栈骨架）| 2 项（LKV2IPV6AM01 + LKV2IPV6SM01）| 5G 特有 `LKV2IPV6AM01` |
| 4G MME | 步骤 1（4G License 2 项）+ 步骤 2-4 | 2 项（LKV2IPV601 + LKV2IPV6SM01）| 4G 特有 `LKV2IPV601` |
| 4G SGW-C/PGW-C | 步骤 1（4G License 1 项）+ 步骤 2-4 | 1 项（LKV2IPV6SM01 共享 SMF）| 同 SMF |
| 2-3G SGSN/GGSN | 步骤 1（2-3G License 1 项）+ 步骤 2-4 | 1 项（**待核**）| 2-3G License 项名待核 |

> **★ R1.5 子规则·激活未演示**：2-3G SGSN/GGSN License 项名在 [WSFD-104001-参考信息_SGSN_GGSN_48043353.md](evidence/UNC/20.15.2/WSFD-104001/WSFD-104001-参考信息_SGSN_GGSN_48043353.md) 仅列 License 测量单元，未明文列 License 控制项名。证据来自 4G 48043353.md 的 License 测量单元描述 + SGSN/GGSN 同源推导。**配置生成对 2-3G 制式默认不产出 License 项**，待主 Agent 复核 UNC License 控制项字典后补全。

> **走法列**：所有制式**前 3 步走共享骨架 compound**（[UNC 1-00024](task/UNC/20.15.2/1-00024.md) → [UNC 1-00023](task/UNC/20.15.2/1-00023.md) → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)），5G/4G 多 1-2 步 License 前置。

## 约束

- **本特性无独立 MML 配置命令**（critical）：3 份参考信息 md（AMF/SMF、MME/SGW-C/PGW-C）明确"本特性无相关命令" — 后果：配置生成时不应产出 MML 配置命令，只生成 License 前置 + 复用被依赖骨架。
- **依赖 IPv6 地址分配特性**（critical）：IPv6 承载上下文特性**不**分配 IPv6 地址，仅承载 IPv6 RA 前缀广播；IPv6 地址分配由 [WSFD-010502](feature/UNC/20.15.2/WSFD-010502.md)（地址分配方式）/ [WSFD-104002](feature/UNC/20.15.2/WSFD-104002.md)（IPv4v6 双栈）/ [WSFD-104004](feature/UNC/20.15.2/WSFD-104004.md)（IPv6 前缀代理）承担 — 后果：被依赖特性未配 → 本特性"无 IPv6 地址可承载"。
- **License 名称跨代际差异**（warning）：5G `LKV2IPV6AM01`（UAM 后缀） vs 4G `LKV2IPV601`（无后缀） — 后果：License 名称错配 → 特性未生效。**配置生成时按接入制式严格区分**。
- **特性无相关告警/软参**（info）：3 份参考信息 md 均明确"本特性无相关告警/软参"，因此本特性 wiki 无软参/告警相关约束。
- **5G 4G S1 模式 PDN 连接失败原因值 #51**（warning）：IPv6 only 场景 PDN 连接失败原因值 **#51 PDN type IPv6 only allowed** — 后果：IPv6 only 接入失败原因值与 3GPP TS 24.301 一致，配置生成无需特殊处理。
- **依赖 WSFD-104002 IPv4v6 双栈特性**（warning）：双栈特性依赖本特性提供 IPv6 路由广播（特性概述明确）— 后果：本特性未开 → 双栈特性 IPv6 部分不可用。
- **C-U 对应**（info）：UDG 域独立提供 IPv6 RA 前缀广播功能，本特性与之解耦。
