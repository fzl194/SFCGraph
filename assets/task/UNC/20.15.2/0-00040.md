# 创建L3VPN实例（ADD L3VPNINST）

> 命令静态知识见 [ADD L3VPNINST](command/UNC/20.15.2/ADD-L3VPNINST.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD L3VPNINST 创建一个 L3VPN 实例（VRF），为 IPsec/AAA/CG/OCS/3GPP AAA/Radius 等组网提供 VPN 隔离承载。本命令极简：仅给出本端唯一的 `VRFNAME` + 可选 `VRFDESCRIPTION`；建后由 [ADD VPNINSTAF](task/UNC/20.15.2/0-00099.md) 启用地址族、[MOD VPNINSTAF](task/UNC/20.15.2/0-00133.md) 配 RD，再绑接口/隧道。

核心配置空间有 2 个维度：**VRF 名称**（必选，按业务语义命名）、**描述**（可选，运维说明）。

### 配置维度 1：VRF 名称（参数 VRFNAME，必选）

| 业务承载 | 推荐命名 | 证据来源 |
|---|---|---|
| IPsec 隧道通用承载（GRE over IPsec / IPv4/IPv6 主备 / OSPF over IPsec / 多 Sequence / 国密 IKEv1） | `vrf1` | IPfd-016000 全部 IPsec 变体 |
| 到 AAA Server（Radius 场景） | `vpn_pdn` / `vpn_aaa` | WSFD-011306 Radius |
| 到 OCS（Gy 在线计费） | `vpn_gy` | WSFD-109001 Gy |
| 到 CG（Ga 离线计费） | `vpn_ga` | WSFD-011201 离线计费 |
| 到 3GPP AAA | `vpn_3gpp_aaa` | WSFD-011306 Radius |
| 带内/带外 GRE VPN | `vpn_enterprise` / `vpn_tunnel` | 多 VPN 并存场景 |

> VRFNAME 数据来源为 local_planned（本端规划），按业务接口语义命名以便运维识别；命名需本端唯一（1~31 字符）。

### 配置维度 2：描述（参数 VRFDESCRIPTION，可选）

| 取值 | 作用 | 说明 |
|---|---|---|
| 不填 | 最小化配置 | 仅建 VRF、无运维说明（证据样例均省略） |
| `<字符串>` | VRF 可读描述 | 1~242 字符，可带空格 |

**典型脚本（IPsec 隧道承载）**

```
ADD L3VPNINST: VRFNAME="vrf1";
```

**典型脚本（OCS/CG 承载）**

```
ADD L3VPNINST: VRFNAME="vpn_gy";
ADD L3VPNINST: VRFNAME="vpn_ga";
```

**典型脚本（AAA 承载，带描述）**

```
ADD L3VPNINST: VRFNAME="vpn_aaa", VRFDESCRIPTION="vpn for test";
```

**步骤位置**：在各组网流程（IPsec/AAA/OCS/CG）中作为 VRF 隔离域的第 1 步。典型顺序：ADD L3VPNINST（建 VRF）→ ADD VPNINSTAF（启用地址族）→ MOD VPNINSTAF（配 RD）→ 绑接口/隧道。

## 决策点

### 决策点：VRF 命名语义选择（DP 0-02059）

问：本 VRF 承载哪种业务（决定 VRFNAME 取值）？

| 选项 | 取值 | 适用场景 |
|---|---|---|
| IPsec 隧道承载 | `vrf1` | GRE over IPsec / IPv4/IPv6 主备 / OSPF over IPsec / 多 Sequence / 国密 IKEv1 |
| AAA/Radius 承载 | `vpn_aaa` / `vpn_pdn` | 到 AAA Server（Radius 鉴权） |
| OCS 承载（Gy） | `vpn_gy` | 在线计费对接 OCS |
| CG 承载（Ga） | `vpn_ga` | 离线计费对接 CG |
| 3GPP AAA 承载 | `vpn_3gpp_aaa` | 3GPP AAA Server 对接 |
| 多 VPN 并存（带内/带外 GRE） | `vpn_enterprise` / `vpn_tunnel` | 多 VPN 隔离场景 |

> VRFNAME 命名由整网规划决定（local_planned），按业务接口语义命名以便运维识别；不同业务命名不同以避免冲突。

### 决策点：是否填写 VRF 描述（DP 0-02060）

问：建 VRF 时是否需要填写 VRFDESCRIPTION？

| 选项 | 适用 |
|---|---|
| 仅配 VRFNAME | 无需描述，最小化配置（证据样例均省略） |
| 配 VRFNAME + VRFDESCRIPTION | 需要可读的 VRF 说明（运维识别） |

> VRFDESCRIPTION 为可选参数（1~242 字符，可带空格）；本参数为 local_planned，无取值分叉决策。

## 约束

- rule-0-02146 · severity=critical · 唯一性：VRFNAME 本端唯一（1~31 字符），重复命名新增失败
- rule-0-02147 · severity=high · 禁用名：不能增加 `_public_`、`__mpp_vpn_inner__`、`__mpp_vpn_inner_server__`（系统保留），新增失败
- rule-0-02148 · severity=warning · 规格上限：本命令最大记录数为 4096，超过上限新增失败
- rule-0-02149 · severity=high · 前置依赖（反向）：建后须配合 ADD VPNINSTAF（启用地址族）+ MOD VPNINSTAF（配 VRFRD）才完整可用，部分场景还须先 ADD VPNINST，单独建 VRF 不产生实际隔离效果
- rule-0-02150 · severity=info · 立即生效：命令执行后立即生效，无需重启
