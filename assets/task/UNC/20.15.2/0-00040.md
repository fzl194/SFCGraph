# ADD L3VPNINST 原子任务
> 命令静态知识见 [ADD L3VPNINST](command/UNC/20.15.2/ADD-L3VPNINST.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令极简：每建一个 VPN 隔离域，给一个本端唯一的 `VRFNAME`（语义化命名按业务接口定），`VRFDESCRIPTION` 可选。建后由 [ADD VPNINSTAF](task/UNC/20.15.2/0-00099.md) 启用地址族、[MOD VPNINSTAF](task/UNC/20.15.2/0-00133.md) 配 RD，再绑接口/隧道。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| VRFNAME | 本端规划，按业务接口语义命名 | IPsec: `vrf1` / AAA: `vpn_aaa` / OCS(Gy): `vpn_gy` / CG(Ga): `vpn_ga` / 3GPP AAA: `vpn_3gpp_aaa` / IPv6 前缀代理: `vpn1` / Radius: `vpn_pdn` |
| VRFDESCRIPTION | 可选，描述用途 | （证据样例均省略） |

典型脚本：
```
ADD L3VPNINST:VRFNAME="vrf1";
ADD L3VPNINST:VRFNAME="vpn_gy";
ADD L3VPNINST:VRFNAME="vpn_aaa";
```

**典型组合**：IPsec 隧道（GRE over IPsec / IPv4/IPv6 主备 / OSPF over IPsec / 多 Sequence / 国密 IKEv1）均先建 `vrf1`；到 AAA Server（单/双平面、带内/带外 GRE VPN）建 `vpn_pdn`/`vpn_aaa`/`vpn_enterprise`/`vpn_tunnel`；到 OCS 建 `vpn_gy`；到 CG 建 `vpn_ga`。

## 决策点（若有）

### VPN 实例命名
问：该 VPN 承载哪种业务？`VRFNAME` 按接口语义命名以便运维识别。
- IPsec 隧道通用承载 → `vrf1`（证据 IPfd-016000 全部 IPsec 变体）。
- 到 AAA Server → `vpn_aaa`/`vpn_pdn`（Radius 场景）。
- 到 OCS（Gy）→ `vpn_gy`。
- 到 CG（Ga）→ `vpn_ga`。
- 到 3GPP AAA → `vpn_3gpp_aaa`。
- 带内/带外 GRE VPN → `vpn_enterprise`/`vpn_tunnel`（多 VPN 并存）。

## 约束（若有）
- 唯一性：`VRFNAME` 本端唯一（1~31 字符）。`severity=high`
- 禁用名：不能增加 `_public_`、`__mpp_vpn_inner__`、`__mpp_vpn_inner_server__`（系统保留）。`severity=high` 后果：新增失败。
- 规格上限：最大记录数 4096。`severity=medium` 后果：超规格新增失败。
- 前置依赖：建后须配合 [ADD VPNINSTAF](task/UNC/20.15.2/0-00099.md)（启用地址族）+ [MOD VPNINSTAF](task/UNC/20.15.2/0-00133.md)（配 VRFRD）才完整可用；部分场景还须先 [ADD VPNINST](task/UNC/20.15.2/0-00098.md)。`severity=high`
- 执行后立即生效。`severity=low`
