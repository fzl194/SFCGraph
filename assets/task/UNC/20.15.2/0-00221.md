# 增加APN Radius客户端IP接口（ADD APNRDSCLIENTIP）

> 命令静态知识见 [ADD APNRDSCLIENTIP](command/UNC/20.15.2/ADD-APNRDSCLIENTIP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令将一个 Gi 接口绑定到 APN，使得发往 RADIUS 服务器的鉴权/计费请求消息源 IP 取自该 Gi 接口。配置空间核心维度是**请求类型**（AUTHORACCT，决定 Gi 接口 IP 充当鉴权源、计费源还是两者），结合 **APN × 接口** 决定绑定关系。

### 配置维度1：请求类型（参数 AUTHORACCT）

| 取值 | 作用 | 配套必选参数 | 典型场景 |
|---|---|---|---|
| AUTHENTICATION | 指定 Gi 接口 IP 仅作鉴权时的 RADIUS client IP | APN, INTFNAME | 仅鉴权场景 |
| ACCOUNTING | 指定 Gi 接口 IP 仅作计费时的 RADIUS client IP | APN, INTFNAME | 仅计费场景 |
| ACCT_AND_AUTH | Gi 接口 IP 同时作鉴权与计费的 RADIUS client IP | APN, INTFNAME | 鉴权计费共用同接口 |

> 实战中常见**鉴权/计费分接口**（AUTHENTICATION + giif1/0/0，ACCOUNTING + giif1/0/1），也见**共用接口**（apn-test 鉴权计费都配 giif1/0/1）。

### 配置维度2：APN × 接口绑定（参数 APN / INTFNAME）

| 取值关系 | 作用 | 约束 |
|---|---|---|
| APN（已有 `ADD APN`）+ INTFNAME（已有 Gi 逻辑接口） | 该 APN 的指定请求类型 RADIUS 消息从该 Gi 接口发源 | 同 APN 下 (INTFNAME, AUTHORACCT) 组合不重复，最多 6 条 |

**步骤位置**：在 RADIUS 功能（WSFD-011306）编排中，于 `ADD APN`（建 APN）、`ADD INTERFACE`（建 Gi 接口）之后、`SET APNRADIUSATTR`（APN 级 RADIUS 属性总开关）之前/同时执行；通常与 `ADD APNRDSSVRGRP`（APN 绑 RADIUS 服务器组）配合使用。

## 决策点

**请求类型与接口划分**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 鉴权/计费分接口 | AUTHENTICATION→giifA，ACCOUNTING→giifB | 鉴权计费流量隔离（典型 WSFD-011306 场景） |
| 鉴权计费共用接口 | 两条都配同一 INTFNAME，或用 ACCT_AND_AUTH | 鉴权计费共链路（简化场景） |

> APN 下配置的 RADIUS client IP 优先级高于 RADIUS server group 下配置的 client IP（同 VPN 情况下）；APN 与 server group 都未配或都配错时，鉴权/计费请求消息发送失败。

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无回滚窗口
- **最大记录数6000**（warning）：本命令最大记录数为 6000 — 后果：超规格新增失败
- **单 APN 组合上限6**（warning）：对象 ApnRdsClientIp 中 APN 对应的 (INTFNAME, AUTHORACCT) 组合记录数不大于 6，且不能重复 — 后果：超规格或重复新增失败
- **APN 必须先建**（critical）：APN 参数必须已通过 `ADD APN` 命令配置 — 后果：APN 不存在则配置失败
- **接口 IP 版本一致**（warning）：RADIUS server 仅选择相同 IP 版本的逻辑接口通信；server group 内同时配 IPv4/IPv6 server 时，client IP 对应接口需同时配 IPv4/IPv6 地址 — 后果：版本不匹配导致通信失败
- **client IP 缺失致请求失败**（critical）：APN 与 server group 都未配 client IP、或未配相关 Giif、或都配错时，鉴权/计费请求消息发送失败 — 后果：用户激活/计费失败
