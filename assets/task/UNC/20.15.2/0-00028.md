# ADD DCCTEMPLATE 原子任务

> 命令静态知识见 [ADD DCCTEMPLATE](command/UNC/20.15.2/ADD-DCCTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**。该命令有 61 个参数、被 10 个特性共用，是 Gy/Diameter 在线计费的核心载体。

## 配置方法

ADD DCCTEMPLATE 新建一个 DCC 模板（DCCTMPLTNAME 命名），模板内聚在线计费的全部行为参数。配置按"建空模板 → 配主从 OCS 组 → 配配额门限/触发场景 → 配 Tx 超时处理与 failover → 配自动回切"分步累加。全局默认模板用 [MOD DCCTEMPLATE](command/UNC/20.15.2/MOD-DCCTEMPLATE.md)，按粒度（UserProfile/APN/计费属性）的模板用 ADD DCCTEMPLATE 后再绑定（[SET USRPROFCHARGE](command/UNC/20.15.2/SET-USRPROFCHARGE.md) 等）。

**范式一：建空模板（最小可用）**

| 参数 | 取值样例 | 说明 |
|---|---|---|
| DCCTMPLTNAME | `dcc-test` | DCC 模板名称（必填），1~63 字符 |

```
ADD DCCTEMPLATE:DCCTMPLTNAME="dcc-test";
```

**范式二：配主从 OCS 组 + failover（WSFD-109001）**

| 参数 | 取值样例 | 说明 |
|---|---|---|
| DCCTMPLTNAME | `dcc-test` | 模板名 |
| PRIOCSGRPNAME | `ocs-gra` | 主 OCS 组名（已配置数据获取） |
| SECOCSGRPNAME | `ocs-grb` | 备 OCS 组名（已配置数据获取） |
| CCFH | `RETRY_AND_TERM` | Tx 超时处理动作 |
| FAILOVERSUP | `ENABLE` | 支持 failover |
| AUTOFAILBACK | `ENABLE` | 支持 failback（可选） |

```
ADD DCCTEMPLATE: DCCTMPLTNAME="dcc-test", PRIOCSGRPNAME="ocs-gra", SECOCSGRPNAME="ocs-grb", CCFH=RETRY_AND_TERM, AUTOFAILBACK=ENABLE, FAILOVERSUP=ENABLE;
```

**范式三：配 CCR 触发场景 + 配额门限（WSFD-109001）**

| 参数 | 取值样例 | 说明 |
|---|---|---|
| QCT | `0` | 配额空耗时间门限（秒），0=连续统计业务时长 |
| QHT | `0` | 配额空闲时间门限（秒），0=不生效（OCS 支持 QHT 超时 CCR 时建议 ≥60） |
| VQT | `20` | 流量阈值触发百分比，剩余流量配额达此值触发 CCR |
| TQT | `20` | 时间阈值触发百分比 |
| CCRINITRGNUM | `5` | CCR-I 携带 RG 个数（需 ≥ SET CTXSTARTRATING 指定的 RG 数） |
| VALIDTIME | `30` | 在线配额有效时长（分），超时触发 CCR |
| SERVICETRIGGER | `ENABLE` | 应用新业务是否发 CCR |
| CCFH | `TERMINATE` | Tx 超时动作 |
| RATTRIGGER / SGSNTRIGGER / QOSTRIGGER / ULITRIGGER / MCCTRIGGER / MNCTRIGGER / LACTRIGGER / UETZTRIGGER | `ENABLE` | 各类位置/QoS/PLMN 变化是否触发重授权 CCR |

```
ADD DCCTEMPLATE: DCCTMPLTNAME="dcc-test", QCT=0, QHT=0, VQT=20, TQT=20, CCRINITRGNUM=5, CCFH=TERMINATE, SGSNTRIGGER=ENABLE, SERVICETRIGGER=ENABLE;
```

**范式四：配 CCR 消息携带参数（WSFD-109001）**

| 参数 | 取值样例 | 说明 |
|---|---|---|
| CCRTARIFFSW | `ENABLE` | CCR 携带费率切换点 |
| PRIVATEATTR | `ENABLE` | 携带运营商私有扩展属性 |
| QUOTATOTAL | `ENABLE` | 按累加模式上报计费信息（默认增量） |

```
ADD DCCTEMPLATE:DCCTMPLTNAME="dcc-test", CCRTARIFFSW=ENABLE, PRIVATEATTR=ENABLE, QUOTATOTAL=ENABLE;
```

**范式五：配 RAR 无 RG 上报动作（WSFD-109001）**

| 参数 | 取值样例 | 说明 |
|---|---|---|
| RARNORGACTION | `REPORT_ALL_RG` | RAR 中无 RG 时上报所有 RG |
| CCFH | `CONTINUE` | Tx 超时转备 OCS，仍无响应转离线 |
| CCFHOFFLINE | `ENABLE` | CCFH 离线标志位 |
| FAILOVERSUP | `ENABLE` | 支持 failover |

**范式六：配在线计费自动回切（WSFD-109001）**

| 参数 | 取值样例 | 说明 |
|---|---|---|
| ONLCHGRECOVER | `ENABLE` | 自动恢复开关 |
| RECOVERINTERVAL | `10` | 恢复间隔（分），1~60，需 < SET HOLDINGTIME |

```
ADD DCCTEMPLATE:DCCTMPLTNAME="dcc-test",ONLCHGRECOVER=ENABLE,RECOVERINTERVAL=10;
```

**范式七：基于 CC+用户号段选 OCS（WSFD-109001，多模板反向主备）**

为不同号段建两个模板，主备 OCS 组对调：`dcctemp01→(ocs-gra,ocs-grb)`，`dcctemp02→(ocs-grb,ocs-gra)`。

**配置位置**：在"激活 Gy/Diameter 在线计费（WSFD-109001）"流程中，ADD DCCTEMPLATE 位于 ADD OCSBINDING 之后，常跟 SET OCSINIT（OCS 初始化）、SET TXTIMER（Tx 定时器）。全局默认模板用 MOD DCCTEMPLATE，粒度模板用 ADD DCCTEMPLATE + 绑定。

## 决策点

### 全局默认模板 vs 粒度模板

| 选项 | 命令 | 适用 |
|---|---|---|
| 全局默认模板（DCCTMPLTNAME=GLOBAL） | MOD DCCTEMPLATE | 全网统一在线计费行为 |
| 粒度模板（UserProfile/APN/计费属性） | ADD DCCTEMPLATE + 绑定 | 不同用户/APN 差异化在线计费 |

> CCRSPESMFLAG/EVENTCHGMOD/QHTEXPIREDRSU/BLKFREESRV 仅在模板名为 GLOBAL 时生效；SERIALCCR/LOCSLCTMODE/SERVICETRIGGER/BLKFREESRV/LOCALHOSTNAME 仅对新上线用户生效。

### Tx 超时处理动作 CCFH（决定 failover 行为）

| 取值 | 行为 | 是否触发 failover |
|---|---|---|
| `TERMINATE` | 终结业务 | 否 |
| `RETRY_AND_TERM` | 向备 OCS 重发，仍无响应则终结 | 是（需 FAILOVERSUP=ENABLE） |
| `CONTINUE` | 向备 OCS 重发，仍无响应转离线继续业务 | 是（需主备 OCS 组 + FAILOVERSUP=ENABLE） |

### 主备 OCS 组配置方式

| 选项 | 适用 |
|---|---|
| 单主 OCS 组（仅 PRIOCSGRPNAME） | 无容灾，OCS Down 即按 CCFH 处理 |
| 主 + 备 OCS 组（PRIOCSGRPNAME + SECOCSGRPNAME） | 支持 failover/failback，CCFH=RETRY_AND_TERM 或 CONTINUE |

### 单 RG 单 DCC 会话（SESSIONMODE）

| 选项 | 适用 |
|---|---|
| `SINGLE`（默认） | 一个 Gy 会话处理多个 RG（OCS 支持多 RG） |
| `MULTIPLE` | 每个 RG 独立 DCC 会话（OCS 不支持单会话多 RG） |

> ③ 数据规划表"获取方法"列分布为 `本端规划(44)` + `已配置数据中获取(7)` 多值 → OCS 组名类参数（PRIOCSGRPNAME/SECOCSGRPNAME）取自分叉（依赖已建 OCS 组），绑定类型 decision_driven/global_planned；其余本地规划参数 local_planned。

## 约束

- **立即生效（部分例外）** · severity=LOW · SERIALCCR/LOCSLCTMODE/SERVICETRIGGER/BLKFREESRV/LOCALHOSTNAME 仅对新上线用户生效，存量用户不回溯。
- **GLOBAL 专属生效**：CCRSPESMFLAG/EVENTCHGMOD/QHTEXPIREDRSU/BLKFREESRV 仅在 DCCTMPLTNAME=GLOBAL 时生效 · severity=HIGH · 粒度模板配这些参数无效。
- **记录上限**：最大记录数 101 · severity=MEDIUM · 超过上限新增失败。
- **RECOVERINTERVAL 条件**：ONLCHGRECOVER=ENABLE 时必填，1~60 分钟，需 < SET HOLDINGTIME 的 HOLDINGTIME · severity=HIGH · 不满足则自动回切功能异常。
- **LOCALHOSTNAME 条件**：LOCSLCTMODE 选定本端主机名选择模式时必填 · severity=MEDIUM。
- **CCRINITRGNUM 联动**：需 ≥ SET CTXSTARTRATING 指定的 RG 数，否则部分 RG 无法在 CCR-I 申请配额 · severity=HIGH。
- **failover 前置**：CCFH=RETRY_AND_TERM/CONTINUE 时需 FAILOVERSUP=ENABLE + 主备 OCS 组 + （CONTINUE 时）OCS 侧 CC-Session-Failover 支持 failover · severity=HIGH · 缺一则 failover 不生效。
- **本端主机名类参数**：LOCALHOSTNAME 数据来源 local_planned，长度 1~127 · severity=LOW。
