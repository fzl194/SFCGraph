# 配置DCC在线计费模板（ADD DCCTEMPLATE）

> 命令静态知识见 [ADD DCCTEMPLATE](command/UNC/20.15.2/ADD-DCCTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。该命令有 61 个参数、被 10 个特性共用，是 Gy/Diameter 在线计费的核心载体。全局默认模板（DCCTMPLTNAME=GLOBAL）用 [MOD DCCTEMPLATE](command/UNC/20.15.2/MOD-DCCTEMPLATE.md)，粒度模板（UserProfile/APN/计费属性）用 ADD DCCTEMPLATE + 绑定（[SET USRPROFCHARGE](command/UNC/20.15.2/SET-USRPROFCHARGE.md) 等）。

## 配置方法

ADD DCCTEMPLATE 新建一个 DCC 模板（DCCTMPLTNAME 命名），模板内聚在线计费的全部行为参数。配置按"建空模板 → 配主备 OCS 组 → 配 Tx 超时处理 → 配 Gy 会话模式 → 配 CCR 触发与配额门限"分步累加。核心配置维度有 4 个：**模板粒度**（全局默认 vs 粒度差异化）、**Tx 超时处理动作**（CCFH 决定 failover 与离线回切行为）、**Gy 会话模式**（SESSIONMODE 决定单会话多 RG vs 每 RG 独立会话）、**主备 OCS 组绑定**（PRIOCSGRPNAME/SECOCSGRPNAME 决定容灾能力）。

### 配置维度 1：Tx 超时处理动作（参数 CCFH，可选，缺省 TERMINATE）

CCFH 决定 Tx 定时器超时（OCS 无响应）时的处理动作，直接影响 failover 与离线回切行为：

| 取值 | 行为 | 是否触发 failover | 联动要求 |
|---|---|---|---|
| **TERMINATE**（缺省） | Tx 超时后终结业务（去激活用户） | 否 | 无需主备 OCS 组 |
| **RETRY_AND_TERM** | 向备 OCS 重发，仍无响应则去激活 | 是 | 需 FAILOVERSUP=ENABLE + 主备 OCS 组 |
| **CONTINUE** | 向备 OCS 重发，仍无响应转离线继续业务 | 是 | 需 FAILOVERSUP=ENABLE + 主备 OCS 组 + OCS 侧 CC-Session-Failover 支持 |
| **BLOCK** | 阻塞业务一段时间后去活（BLOCKTIMER 控制） | 否 | 联动 BLOCKTIMER |

> CCFH 转离线场景若需强制产生话单，须同时使能 CCFHOFFLINE 和 [SET OFCCDRPARA](command/UNC/20.15.2/SET-OFCCDRPARA.md) 的 CCFHCDRSW 或 R6EGCDRCCFHSW。

### 配置维度 2：Gy 会话模式（参数 SESSIONMODE，可选，缺省 SINGLE）

SESSIONMODE 决定 Gy 接口会话与 RG（Rating Group）的对应关系：

| 取值 | 含义 | 适用场景 | 联动约束 |
|---|---|---|---|
| **SINGLE**（缺省） | 每个承载使用单一 Gy 会话（一个 Gy 会话处理多个 RG） | OCS 支持多 RG；EVENTCHGMOD=SCUR 时支持在线自动恢复 | OCSINIT=ENABLE 时不可配 MULTIPLE |
| **MULTIPLE** | 承载内每个业务使用独立 Gy 会话（每 RG 独立 DCC 会话） | OCS 不支持单会话多 RG | CCFH=TERMINATE/RETRY_AND_TERM 时去活该业务 Gy 会话；命令层 VT 不生效 |

### 配置维度 3：主备 OCS 组绑定（参数 PRIOCSGRPNAME/SECOCSGRPNAME，可选）

| 配置 | 含义 | 容灾能力 |
|---|---|---|
| 仅 PRIOCSGRPNAME | 单主 OCS 组 | 无容灾，OCS Down 即按 CCFH 处理 |
| PRIOCSGRPNAME + SECOCSGRPNAME | 主 + 备 OCS 组 | 支持 failover/failback（需 CCFH=RETRY_AND_TERM/CONTINUE + FAILOVERSUP=ENABLE） |

> OCS 组名须已用 [ADD OCSGROUP](command/UNC/20.15.2/ADD-OCSGROUP.md) 建好；不配 PRIOCSGRPNAME 时使用表名为 global 的全局模板主 OCS 组。基于 CC+用户号段选 OCS 场景可为不同号段建两个模板，主备 OCS 组对调。

### 配置维度 4：CCR 触发与配额门限（参数组 QCT/QHT/VQT/TQT/CCRINITRGNUM/VALIDTIME + 各 Trigger 开关）

| 参数 | 作用 | 典型取值 |
|---|---|---|
| **QCT** | 配额空耗时间门限（秒），0=无空耗 | 0 或 5~200 |
| **QHT** | 配额空闲时间门限（秒），0=不生效 | 0 或 5~3600（OCS 支持 QHT 超时 CCR 时建议 ≥60） |
| **VQT/TQT** | 流量/时间阈值触发百分比 | 0~50 |
| **CCRINITRGNUM** | CCR-I 携带 RG 个数 | 0~10，需 ≥ [SET CTXSTARTRATING](command/UNC/20.15.2/SET-CTXSTARTRATING.md) 指定的 RG 数 |
| **VALIDTIME** | 在线配额有效时长（分） | 0 或 5~1440 |
| **SERVICETRIGGER** | 应用新业务是否发 CCR | ENABLE/DISABLE |
| **RATTRIGGER/SGSNTRIGGER/QOSTRIGGER/ULITRIGGER/MCCTRIGGER/MNCTRIGGER/LACTRIGGER/UETZTRIGGER** | 各类位置/QoS/PLMN 变化是否触发重授权 CCR | ENABLE/DISABLE |

### 参数取值样例

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| DCCTMPLTNAME | `dcc-test` / `DccTemplate` | 本端规划 | 模板名，1~63 字符；GLOBAL 为全局默认模板保留名 |
| PRIOCSGRPNAME | `ocsgroup` / `ocs-gra` | 本端规划（依赖已建 OCS 组） | 已 ADD OCSGROUP 建好 |
| SECOCSGRPNAME | `ocsgroup1` / `ocs-grb` | 本端规划（依赖已建 OCS 组） | 已 ADD OCSGROUP 建好 |
| CCFH | `TERMINATE` / `RETRY_AND_TERM` / `CONTINUE` | 本端规划 | Tx 超时处理动作 |
| SESSIONMODE | `SINGLE` / `MULTIPLE` | 全网规划 | Gy 会话模式 |
| FAILOVERSUP | `ENABLE` / `DISABLE` | 本端规划 | 支持 failover 使能 |
| AUTOFAILBACK | `ENABLE` / `DISABLE` | 本端规划 | 支持 failback（可选） |
| RECOVERINTERVAL | `10` | 本端规划 | ONLCHGRECOVER=ENABLE 时填，1~60 分钟 |

**典型脚本（主备 OCS 组 + failover + 自动回切）**：
```
ADD DCCTEMPLATE: DCCTMPLTNAME="dcc-test", PRIOCSGRPNAME="ocs-gra", SECOCSGRPNAME="ocs-grb", CCFH=RETRY_AND_TERM, AUTOFAILBACK=ENABLE, FAILOVERSUP=ENABLE, ONLCHGRECOVER=ENABLE, RECOVERINTERVAL=10;
```

**典型脚本（CCR 触发 + 配额门限）**：
```
ADD DCCTEMPLATE: DCCTMPLTNAME="dcc-test", QCT=0, QHT=0, VQT=20, TQT=20, CCRINITRGNUM=5, CCFH=TERMINATE, SGSNTRIGGER=ENABLE, SERVICETRIGGER=ENABLE;
```

**步骤位置**：在"激活 Gy/Diameter 在线计费"backbone 编排中，ADD DCCTEMPLATE 位于 [ADD OCSBINDING](command/UNC/20.15.2/ADD-OCSBINDING.md) 之后，常跟 [SET OCSINIT](command/UNC/20.15.2/SET-OCSINIT.md)（OCS 初始化）、[SET TXTIMER](command/UNC/20.15.2/SET-TXTIMER.md)（Tx 定时器）。全局默认模板用 MOD DCCTEMPLATE，粒度模板用 ADD DCCTEMPLATE 后再绑定。

## 决策点

### 决策点：全局默认模板 vs 粒度模板（DP 0-02035）

问：该 DCC 模板是全网统一还是针对特定 UserProfile/APN/计费属性差异化？（决定用 ADD 还是 MOD，以及部分参数生效条件）

| 选项 | 命令 | 适用场景 | 生效约束 |
|---|---|---|---|
| 全局默认模板（DCCTMPLTNAME=GLOBAL） | [MOD DCCTEMPLATE](command/UNC/20.15.2/MOD-DCCTEMPLATE.md) | 全网统一在线计费行为 | CCRSPESMFLAG/EVENTCHGMOD/QHTEXPIREDRSU/BLKFREESRV 仅在此模板名下生效 |
| 粒度模板（UserProfile/APN/计费属性） | ADD DCCTEMPLATE + 绑定（[SET USRPROFCHARGE](command/UNC/20.15.2/SET-USRPROFCHARGE.md) 等） | 不同用户/APN 差异化在线计费 | 粒度模板配 CCRSPESMFLAG/EVENTCHGMOD/QHTEXPIREDRSU/BLKFREESRV 无效 |

> SERIALCCR/LOCSLCTMODE/SERVICETRIGGER/BLKFREESRV/LOCALHOSTNAME 参数值配置后只对新上线用户生效，存量用户不回溯。

### 决策点：CCFH Tx 超时处理动作与 failover（DP 0-02036）

问：Tx 定时器超时（OCS 无响应）时采用何种处理动作？（决定是否触发 failover、是否转离线、对端冗余要求）

| 选项 | 行为 | 是否触发 failover | 联动要求 |
|---|---|---|---|
| TERMINATE（缺省） | 终结业务 | 否 | 无需主备 OCS 组 |
| RETRY_AND_TERM | 向备 OCS 重发，仍无响应去激活 | 是 | FAILOVERSUP=ENABLE + 主备 OCS 组 |
| CONTINUE | 向备 OCS 重发，仍无响应转离线继续业务 | 是 | FAILOVERSUP=ENABLE + 主备 OCS 组 + OCS 侧 CC-Session-Failover 支持 |
| BLOCK | 阻塞业务一段时间后去活 | 否 | 联动 BLOCKTIMER |

## 约束

- rule-0-02086 · severity=high · GLOBAL 专属生效：CCRSPESMFLAG/EVENTCHGMOD/QHTEXPIREDRSU/BLKFREESRV 仅在 DCCTMPLTNAME=GLOBAL 时生效 — 后果：粒度模板配这些参数无效
- rule-0-02087 · severity=high · failover 前置：CCFH=RETRY_AND_TERM/CONTINUE 时需 FAILOVERSUP=ENABLE + 主备 OCS 组 + （CONTINUE 时）OCS 侧 CC-Session-Failover 支持 — 后果：缺一则 failover 不生效
- rule-0-02088 · severity=high · RECOVERINTERVAL 条件：ONLCHGRECOVER=ENABLE 时必填，1~60 分钟，需 < [SET HOLDINGTIME](command/UNC/20.15.2/SET-HOLDINGTIME.md) 的 HOLDINGTIME — 后果：不满足则自动回切功能异常
- rule-0-02089 · severity=high · CCRINITRGNUM 联动：需 ≥ [SET CTXSTARTRATING](command/UNC/20.15.2/SET-CTXSTARTRATING.md) 指定的 RG 数 — 后果：否则部分 RG 无法在 CCR-I 申请配额
- rule-0-02090 · severity=warning · 最大记录数 101；SERIALCCR/LOCSLCTMODE/SERVICETRIGGER/BLKFREESRV/LOCALHOSTNAME 仅对新上线用户生效；OCSINIT=ENABLE 时 SESSIONMODE 不能配 MULTIPLE — 后果：超规格新增失败；存量用户参数不回溯；会话模式冲突激活失败
