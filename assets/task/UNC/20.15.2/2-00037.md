# 基于位置区域的对等网元选择（WSFD-010202）

> 特性静态知识见 [WSFD-010202](feature/UNC/20.15.2/WSFD-010202.md)。本页讲配置生成实例化时这个特性**怎么配**——3 网元类型（SGSN/MME/MSC）× 2 域名定制方法（起始 LAI/RAI/TAI / 区域名称 ZONE）。

## 配置概览

核心对象链：**AREADNS**（位置区域 DNS 域名策略：起始 LAI/RAI/TAI 或区域范围 → 区域名称 ZONE 映射）+ **DNSN**（NAPTR 解析记录：FQDN → 主机名 → 网元类型/接口类型）+ **IPV4DNSH**（Hostfile A 解析记录：主机名 → IPv4 地址）。

本特性**3 网元类型**（SGSN/MME/MSC）**共享同一套命令族**（ADD IPV4DNSH + ADD DNSN + ADD AREADNS），差异在 DNTYPE 参数（SGSN_RAI/SGSN_LAI/MME_TAI/MSC_LAI/MSC_RAI）+ 是否启用 ZONESW（区域名称模式）+ MSC 选择策略（`SET MSCSELPLCY`）。

**License 信息**：本特性**无需 License**（特性概述明示）。

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（对等网元选择不依赖 APN 接入域；如本特性与 APN 域特性同场景，则按需先建）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR` → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)（按需）
2. **（可选）Hostfile A 解析记录**（如运营商 DNS Server 已配置对应关系则不需）：`ADD IPV4DNSH`
   - 主机名格式：`ZONE.MNC123.MCC123.GPRS`（2G/3G SGSN 区域名）/ `TOPON.MME-S10.MMEC02.MMEGI8001.MME.EPC.MNC000.MCC123.3GPPNETWORK.ORG`（4G MME S10 接口）
   - 关键参数：HOSTNAME（主机名格式见上），IPV4ADDR（IPv4 地址）
   - **★ R1.5 待补 atom（critical，SOP §6）**：`ADD IPV4DNSH` 命令真实存在 UNC 文档（`业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS Hostfile管理/增加IPV4 DNS Hostfile记录(ADD IPV4DNSH)_26145884.md`），但 UNC pre-build 阶段 atom 未建；配置生成遇本步骤时**待 atom 补齐后**才产出。归 [1-00038](task/UNC/20.15.2/1-00038.md) UNC 位置区域 DNS 族骨架。
3. **（可选）NAPTR 解析记录**（如运营商 DNS Server 已配置对应关系则不需）：`ADD DNSN`
   - 4 网元类型 + 多接口（S10/Sv 等）组合，每条对应一个 NAPTR 记录
   - 关键参数：HOSTNAME, PRIORITY, WEIGHT, REPLACEMENT
   - **★ R1.5 待补 atom**：`ADD DNSN` 真实存在 UNC 文档（`业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS NAPTR管理/增加DNS NAPTR记录(ADD DNSN)_72225569.md`），atom 未建。归 [1-00038](task/UNC/20.15.2/1-00038.md)。
4. **位置区域 DNS 域名策略**（核心单命令步骤，按网元类型 + 区域类型定制）：`ADD AREADNS`
   - 关键参数：DNTYPE=SGSN_RAI/SGSN_LAI/MME_TAI/MSC_LAI/MSC_RAI（网元类型 × 区域类型）；LAC+RACRANGE / TAC+TACRANGE（范围）；ZONESW=YES+ZONENAME=ZONE（区域名称模式）
   - **★ R1.5 待补 atom**：`ADD AREADNS` 真实存在（`业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-SGSN_S10_S16_S3接口管理/位置域名管理/增加位置区域DNS域名(ADD AREADNS)_72345559.md`），atom 未建。归 [1-00038](task/UNC/20.15.2/1-00038.md)。
   - **SGSN × RAI × 起始 RAC**：`ADD AREADNS: DNTYPE=SGSN_RAI, LAC="0x0001", RAC="0x01", RACRANGE="0x05"`
   - **SGSN × RAI × 区域名称 ZONE**：`ADD AREADNS: DNTYPE=SGSN_RAI, LAC="0x0001", RAC="0x01", RACRANGE="0x05", ZONESW=YES, ZONENAME="ZONE"`
   - **MME × TAI × 起始 TAC**：`ADD AREADNS: DNTYPE=MME_TAI, TAC="0x0001", TACRANGE="0x0005"`
   - **MSC × LAI × 区域名称 ZONE**：`ADD AREADNS: DNTYPE=MSC_LAI, LAC="0x0001", LACRANGE="0x0005", ZONESW=YES, ZONENAME="ZONE"`
5. **（按需）MSC 选择策略**（SRVCC 流程：基于 RAI FQDN 或 LAI FQDN 解析 MSC）：`SET MSCSELPLCY`

## 约束（特性级 rule）

- **本特性无需 License**（info）：适用 NF 为 SGSN/MME。
- **AREADNS 必先建**（critical）：网元发起对等网元选择前必须先 `ADD AREADNS` 配置位置区域 DNS 域名策略 — 后果：未配置致对等网元查询失败，附着/RAU/TAU/Handover/SRVCC 流程异常。
- **区域范围必填**（critical）：区域类型（LAI/RAI/TAI）必填**起始码 + 范围**（LAC+RACRANGE / TAC+TACRANGE） — 后果：范围未填致全网范围 DNS 查询无法收敛。
- **ZONESW 与 ZONENAME 联动**（critical）：启用 ZONESW=YES 时**强制必填** ZONENAME — 后果：未填致命令失败。
- **Hostfile/NAPTR 与 DNS Server 二选一**（warning）：步骤 2/3（Hostfile + NAPTR）与运营商 DNS Server 配置**功能等效**，二者择一即可；本特性**真正生效**靠步骤 4 AREADNS。
- **ZONE 域名格式与现网对齐**（warning）：ZONE.MNCxxx.MCCxxx.GPRS（SGSN）/ ZONE.EPC.MNCxxx.MCCxxx.3GPPNETWORK.ORG（MME/MSC）的 ZONE 名前缀必须与现网 DNS 后缀格式一致 — 后果：ZONE 命名错致 DNS 解析失败。
- **MSC 选择策略 RAI/LAI 二选一**（warning）：`SET MSCSELPLCY` 控制 MSC 解析基于 RAI FQDN 或 LAI FQDN，与 AREADNS DNTYPE=MSC_RAI/MSC_LAI 对应 — 后果：策略与 AREADNS 类型不匹配致 MSC 解析失败。
- **DNS 查询总次数下降**（info）：启用本特性后，DNS 服务器查询总次数测量指标 `117470813` 统计值会降低 — 归 [WSFD-010202 参考信息](evidence/UNC/20.15.2/WSFD-010202/WSFD-010202-参考信息_66344717.md)「测量指标」段。
- **★ 待补 atom**（critical）：`ADD IPV4DNSH` / `ADD DNSN` / `ADD AREADNS` / `SET MSCSELPLCY` / `ADD SGSNDNS` UNC pre-build 阶段**未建 atom**，**待补**。配置生成时若无 atom 不可引用，需先补建 atom 编号未分配。
- **首次发布版本**（info）：本特性首次发布版本为 UNC **20.0.0**。
