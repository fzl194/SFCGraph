# 增加接口IPv6地址（ADD IFIPV6ADDRESS）

> 命令静态知识见 [ADD IFIPV6ADDRESS](command/UNC/20.15.2/ADD-IFIPV6ADDRESS.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令手工配置接口的 IPv6 地址。配置空间核心维度为 **接口类型**（LoopBack/Tunnel/Ethernet）、**前缀长度**（PREFIXLEN，1~128）和 **地址类型/计算类型**（ADDRTYPE：global 等；IDGENTYPE：none 等）。

### 配置维度1：接口类型（参数 IFNAME）

| 接口类型 | 作用 | 典型前缀 | 典型场景 |
|---|---|---|---|
| Ethernet66/0/x | 配置物理接口 IPv6 | 96 或 64 | IPv6 IPSec 物理对接（IPFD-016000 Ethernet66/0/5=2001:db8:1::1/96） |
| Tunnel27 | 配置隧道接口 IPv6 | 96 或 64 | IPv6 IPSec 隧道（IPFD-016000 Tunnel27=2001:db8:1::1:0:1） |
| LoopBack | 配置 LoopBack 接口 IPv6 | 128（通常） | 业务/对接固定地址 |

### 配置维度2：前缀长度与地址类型（参数 PREFIXLEN / ADDRTYPE / IDGENTYPE）

| 取值组合 | 作用 | 典型场景 |
|---|---|---|
| PREFIXLEN=96 + ADDRTYPE=global | IPv6 全球单播地址 /96 网段 | IPSec 对接网段（IPFD-016000） |
| PREFIXLEN=64 + ADDRTYPE=global | IPv6 全球单播 /64 网段 | IPSec 对接备选网段 |
| PREFIXLEN=128 | 主机位 | LoopBack 固定地址 |
| IDGENTYPE=none（默认） | 不使用地址计算 | 手工指定地址场景 |

> 修改已配置的 IPv6 地址需先用 `RMV IFIPV6ADDRESS（★R1.5 atom 待补）` 删除后再重新配置。

**步骤位置**：在地址分配/接口管理族（IPv6 IPSec 等）编排中，于 `ADD INTERFACE`（建逻辑接口）、`SET IFIPV6ENABLE`（使能接口 IPv6）之后执行，先于依赖该 IPv6 地址的命令（如 IPSec 对端配置）。

## 决策点

**前缀长度选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 96 | + ADDRTYPE=global | 与对端对接的 IPv6 网段（IPSec 物理口/隧道） |
| 64 | + ADDRTYPE=global | 较大网段对接 |
| 128 | （主机位） | LoopBack 类固定主机地址 |

> 前缀长度需与对端网段规划一致；地址类型在标准全球单播场景下默认 global，无需额外配置。

## 约束

- **接口必须先建并使能 IPv6**（critical）：IFNAME 必须是 `ADD INTERFACE` 已配置的接口，且 IPv6 必须先通过 `SET IFIPV6ENABLE` 使能 — 后果：未建/未使能则配置失败
- **地址修改需先 RMV**（warning）：修改接口 IPv6 地址需先 `RMV IFIPV6ADDRESS（★R1.5 atom 待补）` 删除原地址 — 后果：直接重复配置失败
- **PREFIXLEN 范围**（warning）：整数 1~128 — 后果：超范围配置失败
- **IFNAME 命名约束**（warning）：字符串 1~63，由接口类型决定命名规则 — 后果：违规命名配置失败
- **地址合法性**（critical）：IPV6ADDRESS 必须为合法 IPv6 地址（全球单播场景不能为 ::/环回 ::1/链路本地 FE80::/10/组播 FF00::/8） — 后果：非法地址配置失败
