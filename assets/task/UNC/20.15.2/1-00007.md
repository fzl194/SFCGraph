# 配置 CHF 选择

> 融合计费对接 CHF（Charging Function）的实例/组配置 + 选择方式绑定。被引用于 [2-00003 融合计费](task/UNC/20.15.2/2-00003.md)。特性专属（离线计费对接 CG 用 `ADD CGGROUP`/`CGBINDING`，融合对接 CHF 用 TNF 族命令，命令集完全不同）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 建 CHF 实例 | `ADD TNFINS` | TNFINSINDEX；TNFTYPE=CHF；TNFINSNAME；SCHEMA（http/https）；FQDN（可选，PCF 下发 FQDN 场景） |
| 配 CHF 实例 IP | `ADD TNFINSIP` | TNFINSINDEX；IPTYPE=IPV4；IPV4ADDR；PORT |
| 建 CHF 组 | `ADD TNFGRP` | TNFGRPINDEX；TNFTYPE=CHF；TNFGRPNAME |
| CHF 实例绑到组 | `ADD TNFBINDGRP` | TNFGRPINDEX；TNFINSINDEX |
| 选 CHF 方式（按 DP 选一） | `ADD SELCHFGBYIMSI` / `ADD SELCHFGBIMSISEG` / `ADD SELECTCHFGBYCC` / `SET GLBDFTCHFGROUP` | 见决策点 |

**典型脚本（主备 CHF + 基于 CC 选择）**：
```
ADD TNFINS: TNFINSINDEX=1, TNFTYPE=CHF, TNFINSNAME="target_chf_0", SCHEMA=http;
ADD TNFINS: TNFINSINDEX=2, TNFTYPE=CHF, TNFINSNAME="target_chf_1", SCHEMA=http;
ADD TNFINSIP: TNFINSINDEX=1, IPTYPE=IPV4, IPV4ADDR="192.168.94.2", PORT=3220;
ADD TNFINSIP: TNFINSINDEX=2, IPTYPE=IPV4, IPV4ADDR="192.168.104.2", PORT=3230;
ADD TNFGRP: TNFGRPINDEX=1, TNFTYPE=CHF, TNFGRPNAME="ChfGroup1";
ADD TNFGRP: TNFGRPINDEX=2, TNFTYPE=CHF, TNFGRPNAME="ChfGroup2";
ADD TNFBINDGRP: TNFGRPINDEX=1, TNFINSINDEX=1;
ADD TNFBINDGRP: TNFGRPINDEX=2, TNFINSINDEX=2;
ADD SELECTCHFGBYCC: CCTYPE=VALUE, CCVALUE="0x1111", MASK="0xffff", PRIORITY=2, PRIMARYCHFGRP="ChfGroup1", SECONDARYCHFGRP="ChfGroup2";
```

**步骤位置**：融合计费配置早期（使能开关 + 计费模式之后，CCT 模板之前或之后均可）；是主备 CHF 故障处理（[1-00010](task/UNC/20.15.2/1-00010.md)）与消息缓存（[1-00011](task/UNC/20.15.2/1-00011.md)）的前提。

## 决策点

| CHF 选择方式 | 命令 | 优先级 | 适用场景 |
|---|---|---|---|
| 基于 IMSI | `ADD SELCHFGBYIMSI` | 最高 | 拨测场景，指定 IMSI 发到指定 CHF |
| 基于 IMSI 号段 | `ADD SELCHFGBIMSISEG` | 次高 | 多 CHF 主备组负载均衡，按号段分流 |
| 基于 PCF 下发 FQDN | （仅 `ADD TNFINS` 带 FQDN，无专有绑定命令） | 中 | PCF 在 Npcf 响应携带 chargingInfo FQDN |
| 基于 UDM 签约 CC | （无专有配置，靠 CC 选择） | 中 | UDM 下发 CC，配合 `ADD SELECTCHFGBYCC` |
| 基于 NRF 发现 | （无专有配置） | 次低 | 标准化服务发现，不支持主备仅支持优先级/权重 |
| 基于 SMF 本地 CC | `ADD SELECTCHFGBYCC` | 低 | 本地配置 CC 选 CHF 组（需先 [1-00006](task/UNC/20.15.2/1-00006.md) 配 CC） |
| 全局默认 CHF 组 | `SET GLBDFTCHFGROUP` | 最低 | 兜底；消息缓存回放必须有全局默认组 |

> 联动特性级 DP：CHF 选择方式由 [2-00003](task/UNC/20.15.2/2-00003.md) 的 CHF 选择 DP 驱动。全局默认 CHF 组（`SET GLBDFTCHFGROUP`）是消息缓存（[1-00011](task/UNC/20.15.2/1-00011.md)）的硬前提。

## 约束

- **TNFINSINDEX 唯一**（critical）：CHF 实例索引全局唯一，IP 与组绑定都引用此索引。
- **主备 CHF 组须不同**（warning）：PRIMARYCHFGRP 与 SECONDARYCHFGRP 应指向不同 CHF 组（不同 TNFGRPINDEX），否则主备无意义。
- **全局默认组是缓存前提**（critical）：开启消息缓存（[1-00011](task/UNC/20.15.2/1-00011.md)）必须配 `SET GLBDFTCHFGROUP` — 否则 CHF 故障时缓存消息无处回放。
- **NRF 发现不支持主备**（info）：基于 NRF 发现 CHF 仅支持优先级/权重方式，不支持主备。
