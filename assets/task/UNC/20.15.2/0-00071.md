# 增加规则（ADD RULE）

> 命令静态知识见 [ADD RULE](command/UNC/20.15.2/ADD-RULE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD RULE 是 PCC 体系的枢纽命令：SMF 收到 PCRF/PCF 下发的预定义规则名后，按本地 Rule 匹配并下发 UPF。`POLICYTYPE` 是总开关，决定 Rule 挂哪类策略对象，并决定 `POLICYNAME` 是否必填以及引用对象类型；`RULERANGE` 决定规则下发给哪些锚点 UPF；`PRIORITY` 仅对 PCC 用户生效。配置空间有 4 个维度：**规则名称**（必选，跨网元一致）、**策略类型与策略名称**（必选总开关 + 条件必选引用对象）、**全局优先级**（可选，仅 PCC 生效）、**规则生效范围**（可选，4G/5G 主辅锚点分发控制）。

### 配置维度 1：规则名称（参数 RULENAME，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 规则名称 | `RULENAME` | 对端协商；字符串 1~63，不支持空格，不区分大小写 | 无 | 规则全局唯一标识；预定义规则场景下 SMF/UPF/PCF 上 RULENAME 必须一致 | 全部 Rule 场景 |

### 配置维度 2：策略类型与策略名称（参数 POLICYTYPE + POLICYNAME + FLOWFILTERNAME，必选总开关 + 条件必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 策略类型 | `POLICYTYPE` | 对端协商；枚举：PCC/BWM/HEADEN/WEBPROXY/IPREDIR/SMARTREDIRECT/REMARK_FPI/SRV_TRIGGER/TRAFFICCLASS/LBO/FIREWALL/ULCL/ADC/QOS/FUP/NON_SPECIFIC_TYPE | 无 | 总开关：决定 POLICYNAME 引用对象类型及是否必填 | 全部 Rule 场景 |
| 策略名称 | `POLICYNAME` | 对端协商；字符串 1~63，不支持空格，不区分大小写；POLICYTYPE=PCC/ULCL/QOS 时必填 | 无 | 引用已配策略对象：PCC→PCCPOLICYGRPNM（ADD PCCPOLICYGRP），ULCL→ULCL 属性名（ADD ULCLPROP），QOS→QOSPROPNAME（ADD QOSPROP） | PCC / ULCL / QOS 场景 |
| 流过滤器名称 | `FLOWFILTERNAME` | 对端协商；字符串 1~63；POLICYTYPE=ULCL 时必填 | 无 | ULCL 规则的流过滤器（APPID），由 ADD FLOWFILTER 配置生成 | ULCL 分流场景 |

> POLICYTYPE 为 BWM/ADC/SRV_TRIGGER/HEADEN/WEBPROXY/IPREDIR/SMARTREDIRECT/REMARK_FPI/TRAFFICCLASS/LBO/FIREWALL/FUP/NON_SPECIFIC_TYPE 时**不配 POLICYNAME**。PCC 策略类型与 QOS 策略类型的规则名称不能相同。

### 配置维度 3：全局优先级（参数 PRIORITY，可选，仅 PCC 生效）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 全局优先级 | `PRIORITY` | 对端协商；整数 1~4294967295，值越小优先级越高 | 4294967295（最低优先级） | 仅对 PCC 用户生效；同一 SDF 满足多规则时取高优先级，同值时动态规则 > 预定义规则 | PCC 场景；建议按 10 的倍数规划便于后续插入 |

### 配置维度 4：规则生效范围与规格类型（参数 RULERANGE + RULESPECTYPE，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 规则生效范围 | `RULERANGE` | 全网规划；枚举：ALL（主+辅锚点）/ CENTRAL（仅主锚点）/ LOCAL（仅辅锚点） | ALL | 4G/5G 主辅锚点 UPF 分发控制；仅 LOCAL 才能再由 ADD RULEBINDDNAI 绑 DNAI | 多锚点分流场景 |
| 规则规格类型 | `RULESPECTYPE` | 全网规划；枚举：DEFAULT / SPECIFICATION_LIMITED | DEFAULT | 决定最大记录数与单模板绑定规则数上限；配置后不允许修改；同名 Rule 的 RULESPECTYPE 必须一致 | DEFAULT 通用；SPECIFICATION_LIMITED 配合相应特性使用 |

**典型脚本（PCC 类型，内容计费场景）**
```
ADD RULE:RULENAME="Rule001",POLICYTYPE=PCC,PRIORITY=110,POLICYNAME="cg-test";
```

**典型脚本（QOS 直绑场景）**
```
ADD RULE:RULENAME="rule_test",POLICYTYPE=QOS,POLICYNAME="qosprop_501";
```

**典型脚本（ULCL 分流场景，含 FLOWFILTERNAME）**
```
ADD RULE:RULENAME="testrule",POLICYTYPE=ULCL,POLICYNAME="testulclpropname",FLOWFILTERNAME="ulcl";
```

**典型脚本（BWM 场景，无 POLICYNAME）**
```
ADD RULE:RULENAME="rule_test",POLICYTYPE=BWM,PRIORITY=10;
```

**步骤位置**：在 PCC/计费/QoS/分流特性编排中通常位于"配置规则"步骤——业务策略阶段，前置依赖 [ADD PCCPOLICYGRP](task/UNC/20.15.2/0-00052.md)（POLICYNAME 引用源，PCC 场景）、[ADD QOSPROP](task/UNC/20.15.2/0-00065.md)（POLICYNAME 引用源，QOS 场景）、[ADD FLOWFILTER](task/UNC/20.15.2/0-00032.md)（FLOWFILTERNAME 引用源，ULCL 场景），后续被 [ADD RULEBINDING](task/UNC/20.15.2/0-00073.md)（绑 USERPROFILE）和 [ADD RULEBINDDNAI](task/UNC/20.15.2/0-00072.md)（绑 DNAI）引用。

## 决策点

### 决策点：POLICYTYPE 选型与 POLICYNAME 引用对象（DP 0-02121）

问：本 Rule 服务于哪类业务控制？POLICYNAME 引用哪类已配对象？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| PCC（经 PCC 策略组聚合） | `POLICYTYPE=PCC` + `POLICYNAME=<PCCPOLICYGRPNM>`（引用 [ADD PCCPOLICYGRP](task/UNC/20.15.2/0-00052.md)） | 内容计费（离线/在线/融合/事件/IPv6 在线）、业务触发 QoS 保证经 PCC 策略组、语音 PCF/PCRF 故障 Bypass（信令组/语音组各一） |
| QOS（直绑 QoS 属性） | `POLICYTYPE=QOS` + `POLICYNAME=<QOSPROPNAME>`（引用 [ADD QOSPROP](task/UNC/20.15.2/0-00065.md)） | 业务触发 QoS 保证的直绑场景；后果：PCC 与 QOS 类型的 RULENAME 不能相同 |
| ULCL（UL CL 分流） | `POLICYTYPE=ULCL` + `POLICYNAME=<ULCL 属性名>` + `FLOWFILTERNAME=<APPID>`（引用 ADD FLOWFILTER） | UL CL 分流场景 |
| BWM / ADC / SRV_TRIGGER / 其他 | `POLICYTYPE=BWM` / `ADC` / `SRV_TRIGGER` 等，不配 `POLICYNAME` | 基于业务感知的带宽控制（BWM）、应用检测与控制（ADC）、业务触发 RADIUS（SRV_TRIGGER）等 |

### 决策点：RULERANGE 与 PRIORITY 取值来源（DP 0-02122）

问：规则下发给哪些锚点 UPF？PRIORITY 取本端规划还是与对端协商？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 主+辅锚点都下发 | `RULERANGE=ALL`（默认） | 4G/5G PCC 基本功能常见 |
| 仅主锚点（中心 UPF/PGW-U/UPF） | `RULERANGE=CENTRAL` | 规则只下主锚点；不能绑任何边缘 DNAI |
| 仅辅锚点（边缘 UPF） | `RULERANGE=LOCAL` | 规则只下辅锚点；必须配 LOCAL 才能由 [ADD RULEBINDDNAI](task/UNC/20.15.2/0-00072.md) 绑定 DNAI |
| PRIORITY 本端规划 | `PRIORITY=<整数>`（值越小优先级越高，建议按 10 倍数规划） | 多数 PCC 场景 |
| PRIORITY 与对端协商 | `PRIORITY=<协商值>`（如语音 Bypass 65535） | 语音 PCF/PCRF 故障 Bypass 场景：需保证 PCF 下发语音专有承载规则优先级 > 本地语音专有承载规则 > PCF 下发 any-to-any 动态规则 > 本地 any-to-any 预定义规则 |

## 约束

- rule-0-02301 · severity=critical · RULENAME 跨网元一致性：预定义规则场景下 SMF/UPF/PCF 上的 RULENAME 必须一致（PCF 下发规则名，SMF 本地匹配）；多条记录时 RULENAME+POLICYTYPE 组合不能完全相同（唯一性）
- rule-0-02302 · severity=critical · POLICYTYPE↔POLICYNAME 绑定：`POLICYTYPE=PCC` 时 `POLICYNAME` 必须是已配 PCCPOLICYGRPNM；`=QOS` 时必须是已配 QOSPROPNAME；`=ULCL` 时必须是已配 ULCL 属性名且必配 `FLOWFILTERNAME`；PCC 与 QOS 类型的 RULENAME 不能相同
- rule-0-02303 · severity=high · RULERANGE 语义与 DNAI 绑定前提：`ALL`=主+辅，`CENTRAL`=仅主，`LOCAL`=仅辅；仅 `RULERANGE=LOCAL` 才能由 [ADD RULEBINDDNAI](task/UNC/20.15.2/0-00072.md) 绑定边缘 UPF 的 DNAI；`CENTRAL` 不能绑任何边缘 DNAI
- rule-0-02304 · severity=warning · PRIORITY 仅对 PCC 用户生效：取值 1~4294967295，默认 4294967295（最低优先级），值越小优先级越高；同值时动态规则 > 预定义规则；建议按 10 的倍数规划便于后续插入新规则
- rule-0-02305 · severity=info · 规格与生效：RULESPECTYPE=DEFAULT（或不指定）时最大记录数 8000，SPECIFICATION_LIMITED 时 100000，配置达 98% 上报 ALM-135602215 告警、≤95% 恢复；RULESPECTYPE 配置后不允许修改，同名 Rule 必须保持一致；命令执行后立即生效；适用 NF：PGW-C、SMF
