# RMV CHGDCHAR 原子任务
> 命令静态知识见 [RMV CHGDCHAR](command/UNC/20.15.2/RMV-CHGDCHAR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

删除对象是 `CHGDCHAR`（缺省计费属性参数）的一条记录，定位键为三元组 `ST + MCC + MNC`，三个均必选、无通配，精确删一条。典型用于回退/重建默认计费属性。

## 配置方法
按"用户类型 + MCC + MNC"三元组定位删除。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| ST | 用户类型 | `ROAMING`（漫游用户） |
| MCC | 移动国家码（3 位十进制） | `460` |
| MNC | 移动网号（2 或 3 位十进制） | `00` |

典型脚本（删除 460/00 漫游用户的缺省属性）：
```
RMV CHGDCHAR: ST=ROAMING, MCC=460, MNC=00;
```

## 决策点（若有）

### 删除范围定位
- 问什么：要删哪一条缺省计费属性记录。
- 三个必选参数 `ST/MCC/MNC` 共同唯一定位一条记录，无通配、无批量删，配错三元组会误删他人默认属性。
- 删前建议先用 LST CHGDCHAR 核对存在性，避免误删。

## 约束（若有）
- 唯一键：`ST + MCC + MNC` 三元组唯一定位，不可只给部分。`severity=high`
- 前置：被删记录应存在，否则报错；建议删前 LST 核对。`severity=medium`
- 生效时机：立即生效，仅影响之后新激活用户的话单生成策略（不影响在线用户）。`severity=medium`
- 高危语义：属回退/清理类操作，删除会改变默认计费属性，应与对端计费核对。`severity=medium`
