# 增加基于CC配置融合计费模板处理（ADD SELECTCCTBYCC）

> 命令静态知识见 [ADD SELECTCCTBYCC](command/UNC/20.15.2/ADD-SELECTCCTBYCC.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

本命令是"基于 CC 配置融合计费模板"链路的一环：先建 CCT 模板（`ADD CCT`），再用本命令把 CCT 绑到具体 CC 值上。按"CCTYPE + CCVALUE（+ 可选 MASK/PRIORITY）→ CCTMPLTNAME"绑定，CCTMPLTNAME 由 [ADD CCT](task/UNC/20.15.2/0-00009.md) 建好。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| CC 类型 | CCTYPE | DEFAULT / VALUE（全网规划） | — | DEFAULT=未指定 CC 值；VALUE=指定 CC 值 | 全部场景（必选） |
| CC 特定值 | CCVALUE | 十六进制 0x0000~0xFFFF，1~6 位，允许不带 0x 前缀（全网规划） | — | 指定特殊 CC 值 | CCTYPE=VALUE 时条件必选 |
| CC 掩码 | MASK | 十六进制 0x0001~0xFFFF，1~6 位（全网规划） | — | 按位匹配 CC（全匹配或部分 bit 匹配） | CCTYPE=VALUE 时条件可选；配 MASK 必须同时配 PRIORITY |
| CC 优先级 | PRIORITY | 整数 1~65535，越小越高，须唯一（全网规划） | 0（不配时） | 多 CC 记录定序 | 配 MASK 时必选 |
| 融合计费模板名 | CCTMPLTNAME | 字符串 1~63，不区分大小写（全网规划，由 ADD CCT 建好） | — | 本 CC 命中的计费模板 | 全部场景 |
| I-SMF/SGW 用模板名 | SERVINGCCT | 字符串 0~63，不区分大小写（全网规划，由 ADD CCT 建好） | — | I-SMF/SGW 使用的融合计费模板 | 可选（I-SMF/SGW 场景） |

典型脚本（VALUE 型 CC 值 1234，绑定到模板 test）：
```
ADD SELECTCCTBYCC: CCTYPE=VALUE, CCVALUE="1234", CCTMPLTNAME="test";
```
典型脚本（带掩码 + 优先级精细区分）：
```
ADD SELECTCCTBYCC: CCTYPE=VALUE, CCVALUE="0x0800", MASK="0xffff", PRIORITY=2, CCTMPLTNAME="cct_test";
```

步骤位置：建 CCT 模板（`ADD CCT`）→ 本任务（按 CC 绑模板）→ 关联规则绑定（`ADD RULEBINDING`，用户模板与规则绑定，同属融合计费模板链路）

## 决策点

### 决策点：CC 值的来源（DP 0-02133）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| DEFAULT 未指定 CC | CCTYPE=DEFAULT（不带 CCVALUE） | 不区分 CC 值的通用模板绑定 |
| 本地/全网规划 CC | CCTYPE=VALUE, CCVALUE=本端或全网规划值 | SMF 本地配 CC，直接取规划值 |
| UDM 签约 CC | CCTYPE=VALUE, CCVALUE=从对端 UDM 获取 | CC 值随签约下发，按对端取值 |

### 决策点：是否需要掩码/优先级（DP 0-02134）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 简单单值绑定 | 仅 CCTYPE+CCVALUE+CCTMPLTNAME（不带 MASK/PRIORITY） | 单一 CC 值对应一个模板（证据范式） |
| 多 CC 精细区分/排序 | 加 MASK（按位匹配）+ PRIORITY（定序） | 多 CC 记录需按位匹配或排序 |

## 约束

- **一致性（强约束）**（warning，rule-0-02331）：一致性（强约束）：CCVALUE 与 MASK 按位与后的值必须等于 CCVALUE（即掩码须覆盖 CC 值的所有有效位），否则不允许配置 — 违反则配置失败
- **唯一性**（warning，rule-0-02332）：唯一性：新增 CCVALUE&MASK 与已有记录的 CCVALUE&MASK 按位与结果不可相等 — 避免按位匹配冲突
- **前置**（critical，rule-0-02333）：前置：CCTMPLTNAME 指向的融合计费模板需已存在（由 ADD CCT 先行建好） — 否则引用悬空，计费模板不可解析
- **取值范围**（warning，rule-0-02334）：取值范围：CCVALUE/MASK 十六进制 1~6 位（0x/X、a-f/A-F、0-9，允许不带 0x 前缀，0x0000~0xFFFF；MASK 0x0001~0xFFFF）；PRIORITY 1~65535（不配时默认 0，不允许重复）；CCTMPLTNAME 1~63 不区分大小写；SERVINGCCT 0~63 — 越界或格式不符则配置失败
- **规格与生效**（info，rule-0-02335）：规格与生效：最多可输入 101 条记录；命令执行后立即生效 — 超出上限拒绝写入
