# ADD SELECTCCTBYCC 原子任务
> 命令静态知识见 [ADD SELECTCCTBYCC](command/UNC/20.15.2/ADD-SELECTCCTBYCC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是"基于 CC 配置融合计费模板"链路的一环：先建 CCT 模板（ADD CCT），再用本命令把 CCT 绑到具体 CC 值上。

## 配置方法
按"CCTYPE + CCVALUE（+ 可选 MASK/PRIORITY）→ CCTMPLTNAME"绑定。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTYPE | CC 类型（固定取值 DEFAULT 或 VALUE） | `VALUE` |
| CCVALUE | Charge Characteristic 值（十六进制，1~6 位，全网规划） | `"0x0800"` |
| MASK | CC 特定值掩码（conditional，十六进制） | `"0xffff"` |
| PRIORITY | CC 优先级（conditional，1~65535，越小越高） | `2` |
| CCTMPLTNAME | 融合计费模板名（从已配置 CCT 中获取，1~63，不区分大小写） | `"cct_test"` |
| SERVINGCCT | I-SMF/SGW 使用的 CCT 名（optional） | — |

典型脚本（VALUE 型 CC，绑定到 cct_test）：
```
ADD SELECTCCTBYCC: CCTYPE=VALUE, CCVALUE="0x0800", CCTMPLTNAME="cct_test";
```

## 决策点

### CC 值的来源（本地 CC vs UDM 签约 CC）
- 问什么：CCVALUE 是本端规划、全网规划，还是从对端/已配置数据获取。
- 本地 CC（SMF 本地配）→ 从已配置数据获取。
- UDM 签约 CC → 从对端获取。
- 数据规划表"获取方法"分布多值（本端规划/全网规划/已配置数据获取），属 decision_driven。

### 是否需要掩码/优先级
- 问什么：是否同时配置 MASK 和 PRIORITY。
- 简单单值绑定 → 仅 CCTYPE+CCVALUE+CCTMPLTNAME（证据范式一）。
- 多 CC 精细区分/排序 → 加 MASK（按位匹配）+ PRIORITY（定序）。
- CCVALUE 和 MASK 均为 conditional，是否填取决于是否做掩码匹配。

## 约束
- 一致性（强约束）：CCVALUE 与 MASK 按位与后的值必须**等于 CCVALUE**（即掩码须覆盖 CC 值的所有有效位），否则不允许配置。`severity=high`
- 唯一性：新增 CCVALUE&MASK 与已有记录的 CCVALUE&MASK 按位与结果不可相等（避免匹配冲突）。`severity=high`
- 取值范围：CCVALUE/MASK 十六进制 1~6 位（0x/X、a-f/A-F、0-9）；PRIORITY 1~65535；CCTMPLTNAME 1~63 不区分大小写。`severity=low`
- 规格上限：最多 101 条记录。`severity=low`
- 生效时机：执行后立即生效。`severity=low`
- 前置：CCTMPLTNAME 指向的融合计费模板需已存在（ADD CCT 先行）。`severity=medium`
