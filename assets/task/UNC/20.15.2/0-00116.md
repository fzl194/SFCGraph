# MOD CHARGEMETHOD 原子任务
> 命令静态知识见 [MOD CHARGEMETHOD](command/UNC/20.15.2/MOD-CHARGEMETHOD.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：PGW-C、SMF。本命令按计费属性维度切换计费方式，是融合计费会话创建场景的配套命令——与 [SET USRPROFCHARGE](task/UNC/20.15.2/0-00207.md)、[SET APNCHARGECTRL](task/UNC/20.15.2/0-00149.md)、[SET CHARGECTRL](task/UNC/20.15.2/0-00157.md) 配合，按 User Profile / DNN / CC 三个维度规划使能融合计费。

## 配置方法
先规划计费属性 + 掩码 + 优先级（定位一条属性），再设置该属性对应的在线/离线/融合/QBC 计费开关与上报模式。融合计费场景必置 CONVERGED。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CHARGECHARACT | 计费属性（必选，枚举，规划） | 按整网规划 |
| CCVALUE | 计费属性值（十六进制，1~6 位） | `0xXX` 形式 |
| MASK | 计费属性掩码（十六进制，1~6 位，与 CCVALUE 配套） | `0xFF` |
| PRIORITY | 计费属性优先级（1~65535） | 按规划 |
| ONLINE | 在线计费开关 | `是`/`否` |
| OFFLINE | 离线计费开关 | `是`/`否` |
| CONVERGED | 融合计费开关（融合计费场景必置） | `是`/`否` |
| RGAPPLIED | 业务申请上报模式 | 按规划 |
| QBCSW | QBC 计费开关 | 按规划 |

融合计费会话创建典型用法（WSFD-011206）：配合 SET USRPROFCHARGE / SET APNCHARGECTRL / SET CHARGECTRL 使能 N40 接口融合计费后，本命令按 CC 维度落实 CONVERGED 开关，随后用 LST CCT 校验"CHF 交互使能开关"是否为"激活发送"。

## 决策点

### 计费维度选择（User Profile / DNN / CC）
使能融合计费有三条并列路径：按 User Profile（SET USRPROFCHARGE）、按 DNN/APN（SET APNCHARGECTRL）、按计费属性 CC（MOD CHARGEMETHOD）。具体走哪条由数据规划决定，三条路径在不同网元/不同规划下可并存。获取方法"global_planned"稳定，但取值随规划分叉。

### 计费方式开关组合（ONLINE / OFFLINE / CONVERGED / QBCSW）
四个开关按业务场景组合：纯在线、纯离线、融合（CONVERGED=是）。融合计费会话创建场景 CONVERGED 必置"是"，其余按是否需要 QBC/在线/离线叠加规划。取值随场景枚举分叉。

## 约束
- **仅对新激活用户生效**：执行后只对之后激活的用户生效，存量用户不回溯。
- **CCVALUE/MASK 十六进制格式**：仅支持 0x/X、a-f/A-F、0~9，长度 1~6。
- **PRIORITY 取值范围**：1~65535。
- **融合计费前置**：融合计费开关生效需先经 SET CHARGECTRL/SET USRPROFCHARGE/SET APNCHARGECTRL 使能 N40 接口融合计费，且融合计费模板的"CHF 交互使能开关"为"激活发送"（用 LST CCT 校验）。
