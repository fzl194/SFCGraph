# 基于计费属性切换在线/离线/融合计费方式（MOD CHARGEMETHOD）

> 命令静态知识见 [MOD CHARGEMETHOD](command/UNC/20.15.2/MOD-CHARGEMETHOD.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

适用 NF：PGW-C、SMF。本命令按计费属性维度切换计费方式，是融合计费会话创建场景的配套命令——与 [SET USRPROFCHARGE](task/UNC/20.15.2/0-00207.md)、[SET APNCHARGECTRL](task/UNC/20.15.2/0-00149.md)、[SET CHARGECTRL](task/UNC/20.15.2/0-00157.md) 配合，按 User Profile / DNN / CC 三个维度规划使能融合计费。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 计费属性规划 | `CHARGECHARACT` | 枚举（按整网规划） | 无 | 计费属性标识（必选） | 融合计费会话创建 |
| 计费属性值 | `CCVALUE` | 十六进制 1~6 位（`0xXX` 形式） | 无 | 计费属性具体值 | 按规划 |
| 计费属性掩码 | `MASK` | 十六进制 1~6 位（与 CCVALUE 配套） | 无 | 匹配掩码，`0xFF` | 按规划 |
| 优先级 | `PRIORITY` | 1~65535 | 无 | 同类属性优先级（唯一） | 按规划 |
| 在线计费开关 | `ONLINE` | 是/否 | 否 | 是否使能在线计费 | 按业务需求 |
| 离线计费开关 | `OFFLINE` | 是/否 | 否 | 是否使能离线计费 | 按业务需求 |
| 融合计费开关 | `CONVERGED` | 是/否 | 否 | 融合计费场景必置"是" | 融合计费会话创建场景 |
| 业务申请上报模式 | `RGAPPLIED` | 按规划 | 无 | RG 申请上报模式 | 按规划 |
| QBC 计费开关 | `QBCSW` | 按规划 | 无 | QBC 计费使能 | 按规划 |

融合计费会话创建典型用法（WSFD-011206）：配合 SET USRPROFCHARGE / SET APNCHARGECTRL / SET CHARGECTRL 使能 N40 接口融合计费后，本命令按 CC 维度落实 CONVERGED 开关，随后用 LST CCT 校验"CHF 交互使能开关"是否为"激活发送"。

典型脚本：

```
MOD CHARGEMETHOD:CHARGECHARACT="xxx",CCVALUE="0xXX",MASK="0xFF",PRIORITY=10,ONLINE="是",OFFLINE="否",CONVERGED="是";
```

## 决策点

### 决策点：计费维度选择（User Profile / DNN / CC）（DP 0-01270）

使能融合计费有三条并列路径：按 User Profile（SET USRPROFCHARGE）、按 DNN/APN（SET APNCHARGECTRL）、按计费属性 CC（MOD CHARGEMETHOD）。具体走哪条由数据规划决定，三条路径在不同网元/不同规划下可并存。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 按 User Profile 使能 | `SET USRPROFCHARGE` | 用户级融合计费使能 |
| 按 DNN/APN 使能 | `SET APNCHARGECTRL` | APN 级融合计费使能 |
| 按计费属性 CC 使能 | `MOD CHARGEMETHOD`（`CONVERGED`） | CC 维度融合计费使能 |
| 三条路径并存 | 上述组合 | 多网元/多规划场景 |

获取方法"global_planned"稳定，但取值随规划分叉。

### 决策点：计费方式开关组合（ONLINE / OFFLINE / CONVERGED / QBCSW）（DP 0-01271）

四个开关按业务场景组合：纯在线、纯离线、融合（CONVERGED=是）。融合计费会话创建场景 CONVERGED 必置"是"，其余按是否需要 QBC/在线/离线叠加规划。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 纯在线 | `ONLINE=是`，`OFFLINE=否`，`CONVERGED=否` | 在线计费场景 |
| 纯离线 | `ONLINE=否`，`OFFLINE=是`，`CONVERGED=否` | 离线计费场景 |
| 融合（融合计费会话创建） | `CONVERGED=是`，其余按需叠加 | PGW-C/SMF 融合计费会话创建场景 |
| QBC 叠加 | `QBCSW=是`，其余按需叠加 | QBC 业务场景 |

取值随场景枚举分叉。

## 约束

- **仅对新激活用户生效**（critical，rule-0-01611）：仅对新激活用户生效：执行后只对之后激活的用户生效，存量用户不回溯。

- **CCVALUE/MASK 十六进制格式**（warning，rule-0-01612）：CCVALUE/MASK 十六进制格式：仅支持 `0x`/`0X`/`a-f`/`A-F`/`0-9`，长度 1~6；非法格式被拒。

- **PRIORITY 取值范围 1~65535**（warning，rule-0-01613）：PRIORITY 取值范围 1~65535，超出被拒。

- **融合计费前置条件**（critical，rule-0-01614）：融合计费前置条件：融合计费开关生效需先经 SET CHARGECTRL/SET USRPROFCHARGE/SET APNCHARGECTRL 使能 N40 接口融合计费，且融合计费模板的"CHF 交互使能开关"为"激活发送"（用 LST CCT 校验）；前置不满足则融合计费不生效。

- **适用 NF**（info，rule-0-01615）：适用 NF：PGW-C、SMF；非适用 NF 执行无效。
