# ADD PCCCHGMODEBYPCFID 原子任务
> 命令静态知识见 [ADD PCCCHGMODEBYPCFID](command/UNC/20.15.2/ADD-PCCCHGMODEBYPCFID.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令为**可选**命令，仅在需要基于 PCF 实例标识精细化调整接口类型时配置。用户接口类型的决策链是分层的：先由 `ADD APNPOLICYMODE`/`SET POLICYMODE`（策略）与 `ADD APNCHGMODE`/`SET CHGMODE`（计费）做初选，再由本命令按 PCF 实例标识做最终覆盖（回落或升级）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PCFINSID | PCF 实例标识，全网规划，1~63 字符，作为匹配键 | `pcf1` |
| POLICYINTF | 策略接口类型：`N7`/`Gx`/`INHERIT`（继承初选） | `N7` |
| CHGINTF | 计费接口类型：`N40`/`GaGy`/`INHERIT`（继承初选） | `N40` |

典型脚本：
```
ADD PCCCHGMODEBYPCFID: PCFINSID="pcf1", POLICYINTF=N7, CHGINTF=N40;
```

匹配规则：PCFINSID 在本命令未配置时，用户接口类型以初选结果为准；配置后按 PCF 实例标识匹配覆盖。

## 决策点（若有）

### 是否需要按 PCF 实例细化接口类型
- 问什么：是否需要在初选结果之上，按具体 PCF 实例做 N7↔Gx、N40↔GaGy 的回落/升级。
| 是否配置 | 行为 |
|---|---|
| 不配置 | 完全沿用 `ADD APNPOLICYMODE`/`SET POLICYMODE` 与 `ADD APNCHGMODE`/`SET CHGMODE` 初选结果 |
| 配置 | 对指定 PCF 实例，POLICYINTF/CHGINTF 可强制为 N7/Gx、N40/GaGy 或 INHERIT（继承初选） |

### 策略接口与计费接口的回落/升级方向
- 问什么：策略接口选 N7 时，是否需要本命令最终决策执行面用 Gx；计费接口是否由 N40 回落 GaGy 或反向升级。
| 初选 | 本命令可决策 |
|---|---|
| 策略选 N7 | 决策策略执行是否用 Gx |
| 计费初选 N40/GaGy | 决策是否由 N40 回落 GaGy，或由 GaGy 升级 N40 |

## 约束（若有）
- 生效时机：执行后**只对新激活用户生效**，已激活用户不回溯。`severity=high`（变更后需用户重激活才生效）
- 配置风险：接口类型配置不当可能导致 PCC 用户选择错误接口的 PCRF/PCF，影响业务（如无法正常激活）。`severity=high`
- 决策层级：本命令是对初选结果的二次覆盖，须在 `ADD APNPOLICYMODE`/`ADD APNCHGMODE`（或 SET 变体）已配置的前提下才有意义。`severity=high`
- 跨特性一致性：同一 PCFINSID 在不同特性（PCC 基本功能、融合计费、Gy 在线计费）下的接口类型规划须一致。`severity=medium`
