# 增加基于PCF的计费策略接口类型（ADD PCCCHGMODEBYPCFID）

> 命令静态知识见 [ADD PCCCHGMODEBYPCFID](command/UNC/20.15.2/ADD-PCCCHGMODEBYPCFID.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

本命令为**可选**命令，仅在需要基于 PCF 实例标识精细化调整接口类型时配置。用户接口类型的决策链是分层的：先由 `ADD APNPOLICYMODE`/`SET POLICYMODE`（策略）与 `ADD APNCHGMODE`/`SET CHGMODE`（计费）做初选，再由本命令按 PCF 实例标识做最终覆盖（回落或升级）。PCFINSID 在本命令未配置时，用户接口类型以初选结果为准。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| PCF 实例匹配键 | PCFINSID | 全网规划（1~63，不支持空格，不区分大小写）；可由 NRF 返回或本地配置（`LST PNFPROFILE` 查询） | — | 作为匹配键，按 PCF 实例差异化覆盖 | 多 PCF 实例差异化接口类型 |
| 策略接口类型 | POLICYINTF | 全网规划（N7/Gx/INHERIT） | — | 决定策略执行面接口；INHERIT 继承初选 | 在初选基础上回落 Gx 或升级 N7 |
| 计费接口类型 | CHGINTF | 全网规划（N40/GaGy/INHERIT） | — | 决定计费面接口；INHERIT 继承初选 | 在初选基础上回落 GaGy 或升级 N40 |

典型脚本（PCF 实例 pcf1，策略 N7、计费 N40）：
```
ADD PCCCHGMODEBYPCFID:PCFINSID="pcf1",POLICYINTF=N7,CHGINTF=N40;
```

典型脚本（继承计费初选，仅调整策略接口）：
```
ADD PCCCHGMODEBYPCFID:PCFINSID="pcf1",POLICYINTF=Gx,CHGINTF=INHERIT;
```

步骤位置：初选层（`ADD APNPOLICYMODE`/`SET POLICYMODE` + `ADD APNCHGMODE`/`SET CHGMODE`）→ 本任务（按 PCF 实例覆盖）

## 决策点

### 决策点：是否按 PCF 实例细化接口类型（DP 0-02079）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不配置本命令 | — | 完全沿用初选结果（APNPOLICYMODE/POLICYMODE + APNCHGMODE/CHGMODE） |
| 配置并强制指定 | PCFINSID + POLICYINTF=N7/Gx + CHGINTF=N40/GaGy | 对指定 PCF 实例强制策略/计费接口类型 |
| 配置并继承初选 | PCFINSID + POLICYINTF=INHERIT + CHGINTF=INHERIT | 占位或仅调整单一接口，另一接口继承初选 |

### 决策点：策略与计费接口的回落/升级方向（DP 0-02080）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 策略 N7 → Gx 回落 | POLICYINTF=Gx（初选为 N7） | 5G→4G 接口回落 |
| 策略 Gx → N7 升级 | POLICYINTF=N7（初选为 Gx） | 4G→5G 接口升级 |
| 计费 N40 → GaGy 回落 | CHGINTF=GaGy（初选为 N40） | 5G→4G 计费回落 |
| 计费 GaGy → N40 升级 | CHGINTF=N40（初选为 GaGy） | 4G→5G 计费升级 |

## 约束

- **配置风险**（critical，rule-0-02196）：配置风险：接口类型配置不当可能导致 PCC 用户选择错误接口的 PCRF/PCF，影响业务（如用户无法正常激活）
- **生效时机**（warning，rule-0-02197）：生效时机：执行后**只对新激活用户生效**，已激活用户不回溯（变更后需用户重激活才生效）
- **决策层级**（warning，rule-0-02198）：决策层级：本命令是对初选结果的二次覆盖，须在 `ADD APNPOLICYMODE`/`ADD APNCHGMODE`（或 SET 变体）已配置的前提下才有意义
- **跨特性一致性**（warning，rule-0-02199）：跨特性一致性：同一 PCFINSID 在不同特性（PCC 基本功能、融合计费、Gy 在线计费）下的接口类型规划须一致
- **规格上限**（info，rule-0-02200）：规格上限：最多可输入 1000 条记录
