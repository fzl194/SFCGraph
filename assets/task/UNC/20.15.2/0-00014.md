# 配置计费属性CC实例（ADD CHARGECHAR）

> 命令静态知识见 [ADD CHARGECHAR](command/UNC/20.15.2/ADD-CHARGECHAR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD CHARGECHAR 建一个计费属性（Charging Characteristics, CC）实例，对本地/漫游/拜访三类用户分别指定计费类型，并决定各类用户是否遵从 Serving Node（SGSN/SGW 或 UDM）下发的签约 CC。CC 实例建好后须绑到 UserProfile / APN / 全局三处之一才生效。本命令的核心配置空间有 2 个维度：**用户计费类型**（本地/漫游/拜访各自取值）、**签约遵从开关**（是否使用 Serving Node 下发的 CC）。

### 配置维度 1：用户计费类型（参数 HOME/ROAM/VISIT）

HOME/ROAM/VISIT 三个参数分别配置本地用户、漫游用户、拜访用户的计费类型，取值空间一致（十六进制 0x0000~0xFFFF）。3GPP 标准定义了 4 种语义化取值，现场多按语义对齐：

| 取值 | 计费类型 | 含义 | 适用场景 |
|---|---|---|---|
| **0x0800** | normal 普通计费 | 按流量/时长计费，不区分业务种类（3GPP 标准） | 默认值；最常用 |
| **0x0400** | prepaid 预付费 | 预付余额不足即中止服务 | 预付费用户 |
| **0x0200** | flat-billing 统一费率 | 包月制，周期内费用固定 | 套餐包月 |
| **0x0100** | hotbilling 热计费 | 与普通计费同，话单产生更快（近实时出账） | 热计费场景 |

> 三参默认值均为 0x0800（normal）。允许不输入 0x 前缀，字母不区分大小写。HOME 和 ROAM 用户统称本 PLMN 归属用户，VISIT 为外 PLMN 拜访用户。

### 配置维度 2：签约遵从开关（参数 HOMESGSN/ROAMSGSN/VISITSGSN）

当左侧网元（SGSN/SGW/MME/UDM）携带签约 CC 时，本地/漫游/拜访用户最终 CC 由本组开关控制：

| 取值 | CC 来源 | 适用场景 |
|---|---|---|
| **ENABLE**（默认） | 使用 Serving Node 下发的签约 CC（SGSN 从 HLR、MME 从 HSS、UDM 下发） | 签约 CC 优先，本地配置作兜底 |
| **DISABLE** | 使用本命令 HOME/ROAM/VISIT 的本地配置值 | 强制本地 CC，不接网元下发 |

> 若 ENABLE 时网元未携带 CC 或携带错误，则回退到本地配置。同样的开关逻辑在全局 CC 场景由 `SET GLBCHARGECHAR` 承担；本命令仅对 UserProfile/DNN/APN 级 CC 场景生效。

### 典型脚本

```
# 普通离线计费（遵从签约 CC，本地 0x0800 兜底）
ADD CHARGECHAR:CCNAME="cc",HOME="0x800",ROAM="0x800",VISIT="0x800",
  HOMESGSN=ENABLE,ROAMSGSN=ENABLE,VISITSGSN=ENABLE;

# 强制本地热计费（不接 SGSN 下发）
ADD CHARGECHAR:CCNAME="hotcc",HOME="0x100",ROAM="0x100",VISIT="0x100",
  HOMESGSN=DISABLE,ROAMSGSN=DISABLE,VISITSGSN=DISABLE;
```

**步骤位置**：在计费特性配置流程中，位于 License 前置之后、`SET USRPROFCHARGE` / `SET APNCHARGECTRL` 绑定 CC 之前——本命令产出的 CCNAME 供下游绑定命令引用。

## 决策点

### 决策点：计费类型选择（DP 0-02007）

问：本地/漫游/拜访三类用户各走哪种计费类型？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 普通计费 | HOME/ROAM/VISIT=0x0800 | 默认；按流量/时长计费，3GPP 标准 |
| 预付费 | HOME/ROAM/VISIT=0x0400 | 预付余额不足即中止服务 |
| 统一费率 | HOME/ROAM/VISIT=0x0200 | 包月制套餐 |
| 热计费 | HOME/ROAM/VISIT=0x0100 | 近实时出账 |
| 分类型差异化 | HOME/ROAM/VISIT 取不同值 | 本地走普通、漫游走热计费等差异化策略 |

> 计费类型决定话单生成方式，须与现场业务策略及 `ADD CHARGEMETHOD` 的在线/离线/融合开关一致。

### 决策点：签约CC遵从开关（DP 0-02008）

问：当 Serving Node 携带签约 CC 时，用签约值还是本地配置值？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 遵从签约 CC | HOMESGSN/ROAMSGSN/VISITSGSN=ENABLE | SGSN/MME/UDM 下发的签约 CC 优先，本地作兜底（默认） |
| 用本地配置 CC | HOMESGSN/ROAMSGSN/VISITSGSN=DISABLE | 显式不接网元下发，强制走本地 HOME/ROAM/VISIT 值（热计费强制场景） |

## 约束

- **生效前置**（critical，rule-0-02016）：生效前置：CC 实例须绑到 UserProfile（`SET USRPROFCHARGE`）或 APN（`SET APNCHARGECTRL`）或全局（`SET GLBCHARGECHAR`）才生效，仅建 CCNAME 实例不绑定则计费属性不生效
- **一致性**（warning，rule-0-02017）：一致性：针对同一用户，本命令（设 CC 计费类型）与 `ADD CHARGEMETHOD`（基于 CC 设在线/离线/融合开关）结果须一致（同为 ONLINE / 同为 OFFLINE / 或同时使能），否则用户计费方式错误
- **CC 选择优先级**（warning，rule-0-02018）：CC 选择优先级：RADIUS 下发 CC > UNC 本地；UNC 内 UserProfile 配置 > APN 配置 > 全局配置 > normal 缺省，高级别未配则逐级回退
- **规格上限**（warning，rule-0-02019）：规格上限：本表最大记录数 1000，超出新增失败
- **生效时机**（info，rule-0-02020）：生效时机：命令执行后立即生效，但仅对之后激活的用户有效，已激活用户不受影响
