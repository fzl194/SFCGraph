# ADD CHARGECHAR 原子任务

> 命令静态知识见 [ADD CHARGECHAR](command/UNC/20.15.2/ADD-CHARGECHAR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CHARGECHAR 建一个计费属性（Charging Characteristics, CC）实例，对本地/漫游/拜访三类用户分别指定计费类型（如普通计费 0x0800、预付费 0x0400、统一费率 0x0200、热计费 0x0100），并决定各类用户是否遵从 Serving Node（SGSN/SGW 或 UDM）下发的签约 CC。CC 实例建好后须绑到 UserProfile / APN / 全局三处之一才生效。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| CCNAME | 计费属性名（如 `testchgcha`、`HOTBILLING`） | 本端规划 | CC 实例名，全局唯一 |
| HOME / ROAM / VISIT | 十六进制计费类型（如 `0x0800` 普通 / `0x100` 热计费 / `0x0400` 预付费） | 全网规划或本端规划 | 本地/漫游/拜访用户各自的计费类型 |
| HOMESGSN / ROAMSGSN / VISITSGSN | `ENABLE` / `DISABLE` | 全网规划或本端规划 | ENABLE=使用 Serving Node 签约下发的 CC；DISABLE=使用本命令本地配的 CC |

**取值样例对比**

| 特性场景 | CCNAME | HOME/ROAM/VISIT | *SGSN 三参 | 说明 |
|---|---|---|---|---|
| 离线计费（WSFD-011201，签约 CC 生效） | testchgcha | 0x0800 | ENABLE | 本地配值仅作兜底，优先用 SGSN 下发 |
| 热计费（WSFD-011202，强制本地热计费） | testchgcha / HOTBILLING | 0x100 | DISABLE | 显式不接 SGSN CC，用本地热计费 |
| 融合计费（WSFD-011206，UDM 下发 CC） | testchgcha | 0x0800 | DISABLE | 用本地 CC；UDM 场景 ENABLE 时 SMF 动态获取与本地无关 |
| 在线计费（WSFD-109001） | testchgcha | 0x0800 | ENABLE | 配合在线计费方式 |

## 决策点

### HOME/ROAM/VISIT 计费类型选择

计费类型决定话单生成方式，须与现场业务策略一致。证据 notes 给出标准取值：

| 取值 | 计费类型 | 含义 |
|---|---|---|
| 0x0800 | normal 普通计费 | 按流量/时长计费，不区分业务种类（3GPP 标准，默认值） |
| 0x0400 | prepaid 预付费 | 预付余额不足即中止服务 |
| 0x0200 | flat-billing 统一费率 | — |
| 0x0100 / 0x100 | hot billing 热计费 | 近实时出账 |

### HOMESGSN/ROAMSGSN/VISITSGSN 签约遵从开关

证据明确：当左侧网元（SGSN/SGW/UDM）携带签约 CC 时，本地/漫游/拜访用户最终 CC 由本组参数控制：

| 选项 | 三参取值 | CC 来源 |
|---|---|---|
| 遵从签约 CC | ENABLE | 使用 SGSN/UDM 下发的 CC |
| 用本地配置 CC | DISABLE | 使用本命令 HOME/ROAM/VISIT 的值 |

> 同样的开关逻辑在全局 CC 场景由 `SET GLBCHARGECHAR` 承担；本命令仅对 UserProfile/DNN/APN 级 CC 场景生效。

## 约束

- **立即生效但仅对之后激活的用户有效** · severity=MEDIUM · 已激活用户不受影响。
- **最大 1000 条记录** · severity=LOW。
- **CC 生效前置依赖**：CC 实例须绑到 APN 或 UserProfile 实例才生效，仅建实例不绑定则计费属性不生效 · severity=HIGH · 易漏绑导致 CC 无效。
- **与 ADD CHARGEMETHOD 一致性**：针对同一个用户，本命令（设 CC 计费类型）和 `ADD CHARGEMETHOD`（基于 CC 设在线/离线/融合开关）的结果要保持一致（同为 ONLINE / 同为 OFFLINE / 或同时使能），否则用户计费方式错误 · severity=HIGH · 两条命令交叉配置时最易出错。
- **CC 选择优先级（强约束认知）**：RADIUS 下发 CC > UNC 本地；UNC 内 UserProfile 配置 > APN 配置 > 全局配置 > normal 缺省 · severity=HIGH · 高级别未配则逐级回退。
