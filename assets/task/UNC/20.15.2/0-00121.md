# MOD PCCPOLICYGRP 原子任务
> 命令静态知识见 [MOD PCCPOLICYGRP](command/UNC/20.15.2/MOD-PCCPOLICYGRP.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用网元：PGW-C、SMF。本命令修改已存在的 PccPolicyGrp，调整计费属性、QoS 属性、FupSessionExc 等构成策略集。被 5 个特性共用（WSFD-109103/109104/211009/011206/201207），是 PCC 策略控制链上"策略组承载层"的修改入口。前置必须先 [ADD PCCPOLICYGRP](task/UNC/20.15.2/0-00052.md) 建组。

## 配置方法
按 PCCPOLICYGRPNM 定位组，按规划修改各属性槽（全部可选，未指定即留空槽）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PCCPOLICYGRPNM | PCC 策略组名（必选，1~63，已存在） | `pccpolicygrp_01` |
| URRGROUPNAME | 使用量上报规则组名（可选，1~31） | 已规划 URRGroup 名 |
| FUPSESSIONEXC | Session 级 FUP 累计标识（可选，枚举） | `ENABLE` / `DISABLE` |
| QOSPROPNAME | QoS 属性名（可选，1~31） | 已规划 QosProp 名 |

## 决策点

### 默认组合 vs 非默认组合（最多十个）
notes：PccPolicyGrp 最多配十个 URRGroupName + FupSessionExc 组合——一个默认、九个非默认，或无默认。是否设置默认组合、如何分配 10 个槽位由 ServiceProp 规划决定，存在跨场景取值分叉。

### FUPSESSIONEXC：是否将该策略组业务计入 Session 级流量
FUPSESSIONEXC=DISABLE（默认）表示该 PCC 策略组对应的业务**计入** Session 级流量；FUPSESSIONEXC=ENABLE 表示**不计入**。是否剥离某类业务/应用流量（典型用于累计流量策略控制 WSFD-109104/211009）由流量统计口径规划决定。

### 属性全空组合
notes：所有参数可选，若一个都不选则配置一个空的 PccPolicyGrp。是否保留空策略组（仅作占位/ServiceProp 选择点）由上层设计决定。

## 约束
- **立即生效**：执行后立即生效。
- **前置存在性**：只能修改已 ADD 成功的 PccPolicyGrp；不存在则报错。可用 LST PCCPOLICYGRP 校验。
- **组上限**：最多 10 个 URRGroupName + FupSessionExc 组合（1 默认 + 9 非默认，或无默认）。
- **名称规格**：PCCPOLICYGRPNM 1~63，URRGROUPNAME/QOSPROPNAME 1~31，均不支持空格、不区分大小写。
