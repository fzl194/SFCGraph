# 设置APN计费配置（SET APNCHARGECTRL）

> 命令静态知识见 [SET APNCHARGECTRL](command/UNC/20.15.2/SET-APNCHARGECTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET APNCHARGECTRL 是 APN/DNN 粒度计费配置的核心命令，6 个特性共用。系统初始记录三套计费开关均为 INHERIT。配置空间有 4 个维度：**计费方式开关**（在线/离线/融合，INHERIT 时继承基于计费属性 CC 的配置）、**模板绑定**（OFC/CDR/DCC/CCT 四类模板，模板须先在对应命令配置完成，本命令做绑定）、**CC 绑定**（计费属性名）、**费率切换组绑定**。

### 配置维度 1：计费方式开关（参数 ONLINESW + OFFLINESW + CONVERGEDSW，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| APN 标识 | `APN` | 本端规划；字符串 1~63；必选 | 无 | APN/DNN 实例名，由 [ADD APN](command/UNC/20.15.2/ADD-APN.md) 配置生成 | 全部场景 |
| 在线计费开关 | `ONLINESW` | 本端规划；枚举：DISABLE/ENABLE/INHERIT | INHERIT | ENABLE=执行在线计费；DISABLE=不执行；INHERIT=继承 ADD/MOD CHARGEMETHOD 配置 | 在线计费（Gy 接口） |
| 离线计费开关 | `OFFLINESW` | 本端规划；枚举：DISABLE/ENABLE/INHERIT | INHERIT | ENABLE=执行离线计费；DISABLE=不执行；INHERIT=继承 CHARGEMETHOD 配置 | 离线计费（GGSN/SGW-C/PGW-C） |
| 融合计费开关 | `CONVERGEDSW` | 本端规划；枚举：DISABLE/ENABLE/INHERIT | INHERIT | ENABLE=开启融合计费；DISABLE=不开启；INHERIT=继承 CHARGEMETHOD 配置 | 融合计费（5GC Nchf） |

### 配置维度 2：模板绑定（参数 PGWOFCTEMPLATE/SGWOFCTEMPLATE/GGSNOFCTEMPLATE/PGWCDRTEMPLATE/SGWCDRTEMPLATE/GCDRTEMPLATE/DCCTEMPLATE/CCTEMPLATE，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| PGW 离线计费模板 | `PGWOFCTTEMPLATE` | 本端规划；字符串 1~63，不支持空格 | NULL | 为 APN 绑定 PGW 形态离线计费模板；优先级 User Profile > APN > Charge Characteristic > Global | 离线计费；由 [ADD OFCTEMPLATE](command/UNC/20.15.2/ADD-OFCTEMPLATE.md) 配置生成 |
| SGW 离线计费模板 | `SGWOFCTTEMPLATE` | 本端规划；字符串 1~63，不支持空格 | NULL | 为 APN 绑定 SGW 形态离线计费模板 | 离线计费；由 ADD OFCTEMPLATE 配置生成 |
| GGSN 离线计费模板 | `GGSNOFCTEMPLATE` | 本端规划；字符串 1~63，不支持空格 | NULL | 为 APN 绑定 GGSN 形态离线计费模板 | 离线计费；由 ADD OFCTEMPLATE 配置生成 |
| CDR 话单字段模板 | `PGWCDRTEMPLATE`/`SGWCDRTEMPLATE`/`GCDRTEMPLATE` | 本端规划；字符串 1~63 | NULL | 为 PGW-CDR/SGW-CDR/G-CDR 绑定话单字段模板 | 话单定制；由 [ADD CDRFIELDTEMP](command/UNC/20.15.2/ADD-CDRFIELDTEMP.md) 配置生成 |
| DCC 模板 | `DCCTEMPLATE` | 本端规划；字符串 1~63 | NULL | 为 APN 绑定在线计费模板 | 在线计费；由 [ADD DCCTEMPLATE](command/UNC/20.15.2/ADD-DCCTEMPLATE.md) 配置生成 |
| 融合计费模板 | `CCTEMPLATE` | 本端规划；字符串 1~63 | NULL | 为 APN 绑定融合计费模板 | 融合计费；由 [ADD CCT](command/UNC/20.15.2/ADD-CCT.md) 配置生成 |
| I-SMF/SGW 融合模板 | `SERVINGCCT` | 本端规划；字符串 1~63 | NULL | 为 APN 绑定 I-SMF/SGW 使用的融合计费模板 | I-SMF/SGW 融合场景；由 ADD CCT 配置生成 |

### 配置维度 3：CC 与费率切换组绑定（参数 CCNAME + TARIFFGRPNAME，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 计费属性名 | `CCNAME` | 本端规划；字符串 1~31 | NULL | 为 APN 绑定计费属性；每个 APN 只能绑定一个 CC | 基于计费属性的差异化计费；由 [ADD CHARGECHAR](command/UNC/20.15.2/ADD-CHARGECHAR.md) 配置生成 |
| 费率切换组名 | `TARIFFGRPNAME` | 本端规划；字符串 1~31 | NULL | 为 APN 绑定费率切换组；每个 APN 只能绑定一个 | 费率切换；由 [ADD TARIFFGROUP](command/UNC/20.15.2/ADD-TARIFFGROUP.md) 配置生成 |

### 配置维度 4：业务申请上报模式（参数 RGAPPLIED + QBCSW，conditional）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 业务申请上报模式 | `RGAPPLIED` | 全网规划；枚举：DEFAULT/ONLINERGONLY/OFFLINERGONLY/NOCHG；CONVERGEDSW=ENABLE 时生效 | DEFAULT | DEFAULT=在线 RG 与离线 RG 均可向 CHF 业务申请和上报；ONLINERGONLY=仅在线 RG；OFFLINERGONLY=仅离线 RG；NOCHG=非 QBC 场景不使能融合计费 | 融合计费场景，影响 URR 选择路径 |
| QBC 计费开关 | `QBCSW` | 全网规划；枚举：ENABLE/DISABLE；CONVERGEDSW=ENABLE 时生效 | ENABLE | QBC 计费场景下是否使能计费功能 | QBC 计费场景 |

**典型脚本（融合计费 + 业务申请上报模式）**
```
SET APNCHARGECTRL: APN="apn-test", CONVERGEDSW=ENABLE, RGAPPLIED=ONLINERGONLY;
```

**典型脚本（在线计费 + DCC 模板 + CC 绑定）**
```
SET APNCHARGECTRL: APN="apn-test", ONLINESW=ENABLE, DCCTEMPLATE="dcc-test", CCNAME="testchgcha";
```

**步骤位置**：在计费特性编排中位于"APN 计费配置"步骤——业务策略阶段，前置依赖模板配置命令（ADD OFCTEMPLATE/ADD CDRFIELDTEMP/ADD DCCTEMPLATE/ADD CCT/ADD CHARGECHAR/ADD TARIFFGROUP），后续被融合计费/在线计费/离线计费 compound 引用。

## 决策点

### 决策点：计费方式选择（ONLINESW / OFFLINESW / CONVERGEDSW）（DP 0-03037）

问：APN/DNN 实例启用哪种计费方式？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 离线计费 | `OFFLINESW=ENABLE` | GGSN/SGW-C/PGW-C 形态；需配 PGW/SGW/GGSN OFCTEMPLATE + CDRTEMPLATE |
| 在线计费 | `ONLINESW=ENABLE` + `DCCTEMPLATE=<已配>` | Gy 接口在线计费；需配 DCCTEMPLATE |
| 融合计费 | `CONVERGEDSW=ENABLE` + `CCTEMPLATE=<已配>` + `RGAPPLIED=<DEFAULT/ONLINERGONLY/OFFLINERGONLY>` | 5GC Nchf 融合计费；需配 CCTEMPLATE；RGAPPLIED 仅此时生效 |
| 继承（默认） | `ONLINESW/OFFLINESW/CONVERGEDSW=INHERIT` | 继承基于计费属性 CC 的配置（ADD/MOD CHARGEMETHOD 结果）；须与用户级 CHARGECHAR/CHARGEMETHOD 结果同为 ONLINE/OFFLINE/同时使能 |

### 决策点：CC/DCC/CCT/费率切换组绑定（CCNAME / DCCTEMPLATE / CCTEMPLATE / TARIFFGRPNAME）（DP 0-03038）

问：APN/DNN 绑定哪个计费属性、模板或费率切换组？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 绑定 CC | `CCNAME=<已配计费属性>` | 基于计费属性的差异化计费；每 APN 只能绑一个 CC；由 [ADD CHARGECHAR](command/UNC/20.15.2/ADD-CHARGECHAR.md) 生成 |
| 绑定费率切换组 | `TARIFFGRPNAME=<已配费率切换组>` | 费率切换；每 APN 只能绑一个；由 [ADD TARIFFGROUP](command/UNC/20.15.2/ADD-TARIFFGROUP.md) 生成 |
| 绑定在线模板 | `DCCTEMPLATE=<已配 DCC>` | 在线计费；由 [ADD DCCTEMPLATE](command/UNC/20.15.2/ADD-DCCTEMPLATE.md) 生成；全局模板（名称=global）不允许绑定 APN |
| 绑定融合模板 | `CCTEMPLATE=<已配 CCT>` + `SERVINGCCT=<I-SMF/SGW 用 CCT>` | 融合计费；由 [ADD CCT](command/UNC/20.15.2/ADD-CCT.md) 生成 |
| 修改绑定 | 本命令改写 | 不能重复 ADD；模板名仅输入一个空格表示删除该 APN 与对应模板的绑定关系 |

## 约束

- **计费属性一致性**（critical，rule-0-03091）：计费属性一致性：APN 级计费方式（INHERIT 时继承）须与基于用户计费属性的配置一致——同一用户 ADD CHARGECHAR 与 ADD/MOD CHARGEMETHOD 结果需同为 ONLINE / OFFLINE / 同时使能，否则用户计费方式错误
- **模板绑定前置**（critical，rule-0-03092）：模板绑定前置：绑定 OFC/CDR/DCC/CCT 模板前，对应模板实例必须已配置（[ADD OFCTEMPLATE](command/UNC/20.15.2/ADD-OFCTEMPLATE.md) / [ADD CDRFIELDTEMP](command/UNC/20.15.2/ADD-CDRFIELDTEMP.md) / [ADD DCCTEMPLATE](command/UNC/20.15.2/ADD-DCCTEMPLATE.md) / [ADD CCT](command/UNC/20.15.2/ADD-CCT.md)）；模板名不支持空格；离线模板优先级 User Profile > APN > Charge Characteristic > Global
- **全局在线模板不可绑定 APN**（warning，rule-0-03093）：全局在线模板不可绑定 APN：DccTemplate 名称为 `global` 的全局在线模板不允许绑定在 APN 下；强行绑定会导致在线计费路由错误
- **立即生效与规格**（warning，rule-0-03094）：立即生效与规格：执行后立即作用于后续激活用户；最大记录数 20000；SMFOFCTEMPLATE 当前版本不支持，建任务时不应配置该参数
- **RGAPPLIED/QBCSW conditional**（info，rule-0-03095）：RGAPPLIED/QBCSW conditional：`RGAPPLIED` 与 `QBCSW` 仅在 `CONVERGEDSW=ENABLE` 时生效；模板名仅输入一个空格表示删除该 APN 与对应模板的绑定关系；CCNAME/TARIFFGRPNAME 每 APN 只能各绑一个，修改绑定用本命令改写
