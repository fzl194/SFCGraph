# SET APNCHARGECTRL 原子任务
> 命令静态知识见 [SET APNCHARGECTRL](command/UNC/20.15.2/SET-APNCHARGECTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：SGW-C、PGW-C、SMF。该命令是 APN/DNN 粒度计费配置的核心，6 个特性共用，证据显示获取方法多值（已配置数据中获取 33 + 全网规划 6 + 本端规划 3），属 decision_driven。计费方式开关默认 INHERIT，配置 INHERIT 时继承基于计费属性（CC）的配置。

## 配置方法
按"计费任务"分组建模：①计费方式开关（在线/离线/融合）②模板绑定（OFC/CDR/DCC/CCT）③CC 绑定 ④费率切换组绑定。每条以 APN 实例为锚点。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | APN/DNN 名称（已配置） | `apn-test` |
| ONLINESW | 在线计费开关 | `ENABLE` / `DISABLE` / `INHERIT` |
| OFFLINESW | 离线计费开关 | `ENABLE` / `DISABLE` / `INHERIT` |
| CONVERGEDSW | 融合计费开关 | `ENABLE`（5GC 融合） |
| RGAPPLIED | 业务申请上报模式 | `ONLINERGONLY` / `OFFLINERGONLY` / `DEFAULT` |
| PGWOFCTEMPLATE | PGW 离线计费模板名（已配置） | `offlinecharge-test` |
| SGWOFCTTEMPLATE | SGW 离线计费模板名（已配置） | `offlinecharge-test` |
| GGSNOFCTEMPLATE | GGSN 离线计费模板名（已配置） | — |
| PGWCDRTEMPLATE / SGWCDRTEMPLATE / GCDRTEMPLATE | CDR 话单字段模板名 | `cdrfield-test` |
| DCCTEMPLATE | DCC 模板名（已配置） | `dcc-test` |
| CCTEMPLATE | 融合计费模板（已配置） | `cct_test` |
| CCNAME | 计费属性名（已配置） | `testchgcha` |
| TARIFFGRPNAME | 费率切换组名（已配置） | `testtariff` |
| N40MSGTEMP | N40 消息属性模板 | — |
| QBCSW | QBC 计费开关 | — |
| SERVINGCCT | I-SMF/SGW 使用的融合计费模板名 | — |
| SMFOFCTEMPLATE | SMF 离线计费模板名（当前版本不支持） | — |

典型脚本（融合计费 + 业务申请上报模式）：
```
SET APNCHARGECTRL: APN="apn-test", CONVERGEDSW=ENABLE, RGAPPLIED=ONLINERGONLY;
```
（在线计费 + DCC 模板 + CC 绑定）：
```
SET APNCHARGECTRL: APN="apn-test", ONLINESW=ENABLE, DCCTEMPLATE="dcc-test", CCNAME="testchgcha";
```

## 决策点

### 计费方式选择（ONLINESW / OFFLINESW / CONVERGEDSW）
- 问什么：APN/DNN 实例启用哪种计费方式。
- 离线计费（GGSN/SGW-C/PGW-C）→ `OFFLINESW=ENABLE`。
- 在线计费（Gy 接口）→ `ONLINESW=ENABLE`，需配 DCCTEMPLATE。
- 融合计费（5GC Nchf）→ `CONVERGEDSW=ENABLE`，需配 CCTEMPLATE。
- `INHERIT`：继承基于计费属性（CC）的配置（默认值）。
- 与基于用户计费属性的配置须一致：同一用户 ADD CHARGECHAR 与 ADD/MOD CHARGEMETHOD 结果需同为 ONLINE / OFFLINE / 同时使能，否则用户计费方式错误。`severity=high`

### 业务申请上报模式（RGAPPLIED）
- 问什么：URRGROUP 下同时配置在线/离线 URR 时，业务申请（RG）按什么模式上报。
- `DEFAULT`：URRGROUP 下同时配离线和在线 URR 时优先选在线 URR。
- `ONLINERGONLY` / `OFFLINERGONLY`：URRGROUP 下建议同时配离线和在线 URR。
- 仅融合计费场景生效，影响计费会话的 URR 选择路径。

### CC/DCC/CCT/费率切换组绑定（唯一性）
- 问什么：APN/DNN 绑定哪个计费属性、模板或费率切换组。
- 每个 APN/DNN 只能绑定一个 CC（CCNAME）、一个费率切换组（TARIFFGRPNAME）；修改绑定需用本命令改写，不能重复 ADD。
- 模板（OFC/CDR/DCC/CCT）须先在对应命令（ADD OFCTEMPLATE / ADD DCCTEMPLATE / ADD CCT 等）配置完成，本命令做绑定。
- 取值来自"已配置数据中获取"，模板名不支持空格。

## 约束
- 立即生效：执行后立即作用于后续激活用户。`severity=medium`
- 记录数上限：最多 20000 条记录。`severity=medium`
- 全局在线模板不可绑定 APN：DccTemplate 名称为 `global` 的全局在线模板不允许绑定在 APN 下。`severity=high`
- 模板名空格：模板名不支持包含空格；仅输入一个空格表示删除该 APN 与对应模板的绑定关系。`severity=medium`
- SMFOFCTEMPLATE 当前版本不支持：建任务时不应配置该参数。`severity=high`
- 模板绑定前置：绑定 OFC/CDR/DCC/CCT 模板前，对应模板实例必须已配置。`severity=high`
- 计费属性一致性：CHARGECHAR 与 CHARGEMETHOD 配置结果须一致（同为 ONLINE/OFFLINE/两者使能），否则计费方式错误。`severity=high`
