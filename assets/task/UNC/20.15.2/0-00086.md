# ADD TNFINS 原子任务
> 命令静态知识见 [ADD TNFINS](command/UNC/20.15.2/ADD-TNFINS.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令注册一个目标 NF 实例（融合计费场景下为 CHF）。一个实例由"索引 + 类型 + 名称"标识，配套 [ADD TNFINSIP](task/UNC/20.15.2/0-00087.md) 提供可达 IP，再由 `ADD TNFGRP` 组成 CHF 组供选择。主备 CHF 需注册两个实例（两个 TNFINSINDEX）。

## 配置方法
按"主备 CHF 各注册一个实例"配置，固定类型 CHF。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| TNFINSINDEX | 本端规划，主备实例各取一个唯一索引（0~2047） | `1`（主）、`2`（备） |
| TNFTYPE | 固定取值，融合计费目标 NF 为 CHF | `CHF` |
| TNFINSNAME | 从对端获取，与 PCF 下发的 FQDN 名称保持一致 | `target_chf_0` |
| SCHEMA | 全网规划，协议模式 | `http` |
| FQDN | 从对端获取，CHF 域名 | `chf1.hw.com` |

典型脚本（主备 CHF，带 FQDN）：
```
ADD TNFINS: TNFINSINDEX=1, TNFTYPE=CHF, TNFINSNAME="target_chf_0", SCHEMA=http, FQDN="chf1.hw.com";
ADD TNFINS: TNFINSINDEX=2, TNFTYPE=CHF, TNFINSNAME="target_chf_1", SCHEMA=http, FQDN="chf2.hw.com";
```

## 决策点

### TNFINSNAME 与 ADD PNFPROFILE 的关系
- 问什么：本端 TNFINSNAME 与 `ADD PNFPROFILE` 的 NFINSTANCEID 取值相同时（即对端同一 CHF），本命令关联的 [ADD TNFINSIP](task/UNC/20.15.2/0-00087.md) 下所有 IP 必须被 PNFPROFILE 中该 CHF 的 IP 包含。
- 不一致会导致 NRF 发现与对端 NF 概述信息冲突，配置失败或选不到实例。

### SRVINSID 与 SRVNAME 的取舍
- 问什么：是否需要注册服务实例标识。SRVNAME 默认 NrfNfm（NRF 提供的 Nnrf_NFManagement 服务）；SRVINSID 为空时按 NF 实例级匹配。
- 常规融合计费 CHF 注册 → 不填 SRVINSID（按实例级），仅当需要服务级精细化发现时填写。

## 约束

- **立即生效** · severity=LOW。
- **与 ADD PNFPROFILE 的 IP 包含关系**：当 TNFINSNAME 与 PNFPROFILE 的 NFINSTANCEID 相同时，本实例关联的 TNFINSIP 全部 IP 必须被 PNFPROFILE 中该 CHF 的 IP 包含 · severity=HIGH。
- **TNFINSINDEX 唯一性**：同一索引标识一个实例，跨 TNFINS/TNFINSIP 引用需一致 · severity=HIGH。
- **TNFINSNAME 长度**：0~63 字符 · severity=LOW。
