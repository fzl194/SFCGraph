# 配置离线计费话单参数（SET OFCCDRPARA）（SET OFCCDRPARA）

> 命令静态知识见 [SET OFCCDRPARA](command/UNC/20.15.2/SET-OFCCDRPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按场景选少数相关参数子集配置。系统部署后已存在初始记录（SPCDRCONTROL=PGW_CDR、PGWCDRLOTVSW=DISABLE、CCFHCDRSW=DISABLE、SGWLOTVQOSEXT=DISABLE 等）。未输入参数保持系统当前值不变（可通过 LST OFCCDRPARA 查看）。本命令属高危命令——SPCDRCONTROL=PGW_SGW_CDR 后需检查流量容器占用率防资源耗尽。下表仅列实际激活 md 中显式配置的参数。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| SGW-CDR 扩展 QoS | `SGWLOTVQOSEXT` | `DISABLE` / `ENABLE` | DISABLE | SGW-CDR 流量容器 EPC Qos 携带扩展 Qos 参数 | NSA 用户数据话单（与对端协商） |
| PGW-CDR 流量扩展 QoS | `PGWLOTVQOSEXT` | `DISABLE` / `ENABLE` | DISABLE | PGW-CDR 流量容器 EPC Qos 携带扩展 Qos 参数 | NSA 用户数据话单（与对端协商） |
| PGW-CDR 业务扩展 QoS | `PGWLOSDQOSEXT` | `DISABLE` / `ENABLE` | DISABLE | PGW-CDR 业务容器 EPC Qos 携带扩展 Qos 参数 | NSA 用户 QoS 管理（与对端协商） |
| SP 合一话单控制 | `SPCDRCONTROL` | `PGW_CDR` / `PGW_SGW_CDR` | PGW_CDR | SP 合一用户只产生 PGW-CDR 或同时产生 SGW+PGW-CDR | SP 合一组网（本端规划，高危） |
| CCFH 强制话单开关 | `CCFHCDRSW` | `DISABLE` / `ENABLE` | DISABLE | CCFH 是否强制产生话单 | 在线计费 CCFH 处理（本端规划） |
| CCFH 话单 CC 值 | `CCFHCDRCC` | 十六进制 1~6 位（0x0000~0xFFFF） | 无 | CCFH 强制话单填充的 Charging Characteristics | CCFHCDRSW=ENABLE 时条件必填 |
| PGW-CDR LoTV 开关 | `PGWCDRLOTVSW` | `DISABLE` / `ENABLE` | DISABLE | PGW-CDR 是否携带 listOfTrafficVolume | 话单字段控制（本端规划） |
| PGW-CDR LoSD RAT | `PGWCDRLOSDRATSW` | `DISABLE` / `ENABLE` | DISABLE | PGW-CDR 业务容器是否携带 RAT-Type | 话单字段控制（本端规划） |
| PGW-CDR LoSD AMBR | `PGWCDRLOSDAMBR` | `DISABLE` / `ENABLE` | DISABLE | PGW-CDR 业务容器是否携带 APN-AMBR | 话单字段控制（本端规划） |
| PGW-CDR LoSD APN Rate | `PGWLOSDAPNRATE` | `DISABLE` / `ENABLE` | DISABLE | PGW-CDR LoSD 是否携带 APN Rate Control | eMTC/NB-IoT 接入速率控制（WSFD-216104/215204） |
| PGW-CDR LoSD SP Rate | `PGWLOSDSPRATE` | `DISABLE` / `ENABLE` | DISABLE | PGW-CDR LoSD 是否携带 Serving PLMN Rate Control | eMTC/NB-IoT 接入速率控制（WSFD-216105/215205） |
| SGW-CDR LoTV SP Rate | `SGWLOTVSPRATE` | `DISABLE` / `ENABLE` | DISABLE | SGW-CDR LoTV 是否携带 Serving PLMN Rate Control | NB-IoT 接入速率控制（WSFD-215205） |
| 费率切换强制话单 | `CDRAFTERTCSW` | `DISABLE` / `ENABLE` | ENABLE | 费率切换后用量是否强制产生话单 | 防止大量用户时间阈值同时触发冲击 CG |
| 费率切换关闭原因 | `CDRAFTERTCCAUSE` | `MANAGEMENT_INTERVENTION` / `MAX_CHANGE_COND` | MANAGEMENT_INTERVENTION | 费率切换强制话单的关闭原因值 | CDRAFTERTCSW=ENABLE 时条件必填 |
| 话单缓存加密 | `STGENCRYPT` | `DISABLE` / `ENABLE` | DISABLE | 是否对 4G 缓存话单加密 | 开启前需联系华为技术服务评估（本端规划） |

典型脚本（NSA 用户话单扩展 QoS）：
```
SET OFCCDRPARA: SGWLOTVQOSEXT=ENABLE, PGWLOTVQOSEXT=ENABLE, PGWLOSDQOSEXT=ENABLE;
```

典型脚本（SP 合一产生 SGW 话单 + CCFH 强制话单）：
```
SET OFCCDRPARA: SPCDRCONTROL=PGW_SGW_CDR, CCFHCDRSW=ENABLE, CCFHCDRCC="0x1000";
```

步骤位置：按引用特性编排——NSA 用户（WSFD-011511/011521）配扩展 QoS 三参数；离线计费（WSFD-011201）配 SP 合一/CCFH/字段开关；接入速率控制（WSFD-216104/216105/215204/215205）配 Rate Control 子集；紧随 ADD CDRFIELDTEMP 等前置模板之后。

## 决策点

### 决策点：扩展 QoS 三参数组（DP 0-03119）

SGW-CDR/PGW-CDR 是否携带扩展 Qos 参数；取值由对端（话单接收/计费系统）能力协商决定。NSA 用户场景下三者通常同 ENABLE。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| NSA 联动 ENABLE | SGWLOTVQOSEXT=ENABLE、PGWLOTVQOSEXT=ENABLE、PGWLOSDQOSEXT=ENABLE | NSA 用户数据话单/QoS 管理（WSFD-011511/011521） |
| 全 DISABLE（默认） | 三参数保持 DISABLE | 非 NSA 场景 |

> 三参数为联动集合——NSA 场景需三者协同开启，单开一个会留下话单字段缺口。

### 决策点：SPCDRCONTROL 高危（DP 0-03120）

SP 合一网关是否产生 SGW 话单；取值由是否 SP 合一组网决定。属高危——配置 PGW_SGW_CDR 后需检查流量容器占用率。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅 PGW-CDR | `SPCDRCONTROL=PGW_CDR` | SP 合一组网，仅产生 PGW 话单（默认） |
| PGW+SGW-CDR | `SPCDRCONTROL=PGW_SGW_CDR` | SP 合一组网需同时产生 SGW 话单（WSFD-011201 SGW-C，高危） |

> 关键约束：配置 PGW_SGW_CDR 后需立即检查流量容器占用率，防止资源耗尽引发用户激活成功率下降。

## 约束

- rule-0-03296 · severity=critical · 本命令属高危命令：配置 SPCDRCONTROL=PGW_SGW_CDR 后需检查流量容器占用率，防止资源耗尽引发用户激活成功率下降 — 后果：资源耗尽导致用户激活失败
- rule-0-03297 · severity=high · CCFHCDRCC 仅在 CCFHCDRSW=ENABLE 时条件必填，十六进制 1~6 位（0x0000~0xFFFF），允许不输入 0x 前缀 — 后果：缺填或超范围导致 CCFH 强制话单 CC 字段异常
- rule-0-03298 · severity=high · CDRAFTERTCCAUSE 仅在 CDRAFTERTCSW=ENABLE 时条件必填 — 后果：缺填导致费率切换强制话单关闭原因缺失
- rule-0-03299 · severity=warning · 扩展 QoS 三参数（SGWLOTVQOSEXT/PGWLOTVQOSEXT/PGWLOSDQOSEXT）NSA 场景下通常同 ENABLE，单开一个会留字段缺口 — 后果：话单字段不完整影响计费准确性
- rule-0-03300 · severity=info · 最大记录数为 1（全局唯一一份），系统部署后已存在初始记录；该命令执行后立即生效；STGENCRYPT 开启前需联系华为技术服务评估性能开销 — 后果：漏填参数沿用初始值，加密开启导致额外性能开销
