# SET OFCCDRPARA 原子任务
> 命令静态知识见 [SET OFCCDRPARA](command/UNC/20.15.2/SET-OFCCDRPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

该命令是离线计费话单参数的总开关集（24 参数），被 8 个特性引用，**属高危命令**（SPCDRCONTROL=PGW_SGW_CDR 后需检查流量容器占用率防资源耗尽）。系统部署后已存在初始记录。各特性按需配置少数相关参数子集。

## 配置方法
按场景选少数相关参数子集配置；未输入参数保持系统当前值不变（可通过 LST OFCCDRPARA 查看）。下表仅列实际激活 md 中显式配置的参数与场景；完整 24 参数见命令 wiki。

| 参数 | 描述 | 配置方法 | 取值样例 | 典型场景 |
|---|---|---|---|---|
| SGWLOTVQOSEXT | SGW-CDR 流量容器携带扩展 Qos 参数开关 | 与对端协商 | `ENABLE` | NSA 用户数据话单/QoS 管理（WSFD-011511/011521） |
| PGWLOTVQOSEXT | PGW-CDR 流量容器携带扩展 Qos 参数开关 | 与对端协商 | `ENABLE` | NSA 用户数据话单/QoS 管理（WSFD-011511/011521） |
| PGWLOSDQOSEXT | PGW-CDR 业务容器携带扩展 Qos 参数开关 | 与对端协商 | `ENABLE` | NSA 用户数据话单/QoS 管理（WSFD-011511/011521） |
| SPCDRCONTROL | SP 合一网关话单控制 | 本端规划 | `PGW_SGW_CDR` | SP 合一用户产生 SGW 话单（WSFD-011201 SGW-C） |
| CCFHCDRSW | CCFH 强制产生话单开关 | 本端规划 | `ENABLE` | 在线计费 CCFH 处理后强制产生话单（WSFD-011201） |
| CCFHCDRCC | CCFH 强制产生话单填充的 Charging Characteristics 值 | 本端规划 | `0x1000` | CCFH 话单 charging characteristic 字段（WSFD-011201） |
| PGWCDRLOTVSW | PGW-CDR 携带 List of Traffic Volume | 本端规划 | 枚举 | 控制是否携带流量容器（WSFD-011201） |
| PGWCDRLOSDRATSW | PGW-CDR 业务容器携带 RAT-Type | 本端规划 | 枚举 | List of Service Data 是否携带 RAT Type（WSFD-011201） |
| PGWCDRLOSDAMBR | PGW-CDR 业务容器携带 APN-AMBR | 本端规划 | 枚举 | List of Service Data 是否携带 APN-AMBR（WSFD-011201） |
| PGWLOSDAPNRATE | PGW-CDR 话单 LoSD 携带 APN Rate Control | 本端规划 | 枚举 | 基于 APN 的 eMTC/NB-IoT 终端接入速率控制（WSFD-216104/215204） |
| PGWLOSDSPRATE | PGW-CDR 话单 LoSD 携带 Serving PLMN Rate Control | 本端规划 | 枚举 | 基于服务 PLMN 的 eMTC/NB-IoT 终端接入速率控制（WSFD-216105/215205） |
| SGWLOTVSPRATE | SGW-CDR 话单 LoTV 携带 Serving PLMN Rate Control | 本端规划 | 枚举 | 基于服务 PLMN 的 NB-IoT 接入速率控制（WSFD-215205） |

典型脚本（WSFD-011511/011521 NSA 用户话单扩展 QoS）：
```
SET OFCCDRPARA:SGWLOTVQOSEXT=ENABLE,PGWLOTVQOSEXT=ENABLE,PGWLOSDQOSEXT=ENABLE;
```
典型脚本（WSFD-011201 在线计费 CCFH 强制话单）：
```
SET OFCCDRPARA: CCFHCDRSW=ENABLE,CCFHCDRCC="0x1000";
```
典型脚本（WSFD-011201 SP 合一产生 SGW 话单）：
```
SET OFCCDRPARA: SPCDRCONTROL=PGW_SGW_CDR;
```

## 决策点

### 扩展 QoS 三参数组（SGWLOTVQOSEXT/PGWLOTVQOSEXT/PGWLOSDQOSEXT）
- 问什么：SGW-CDR/PGW-CDR 是否携带扩展 Qos 参数。
- 取值由对端（话单接收/计费系统）能力协商决定（与对端协商来源）；NSA 用户场景下三者通常同 ENABLE。
- 三参数为联动集合（数据规划表"获取方法"=与对端协商占 6 条，多值→决策驱动）。

### CCFH 强制话单（CCFHCDRSW + CCFHCDRCC 联动）
- 问什么：在线计费 CCFH 处理后是否强制产生话单及话单 charging characteristic 值。
- ②conditional：CCFHCDRCC 仅在 CCFHCDRSW=ENABLE 时必填（conditional 来源）；CC 值为十六进制字符串（1～6 位，0x/X+a-f/A-F+0-9）。

### SPCDRCONTROL（高危）
- 问什么：SP 合一网关是否产生 SGW 话单。
- 取值由是否 SP 合一组网决定（本端规划）；**属高危**，配置 PGW_SGW_CDR 后需检查流量容器占用率防资源耗尽、用户激活成功率下降。

### 字段携带开关子集（PGWCDRLOTVSW/LOSDRATSW/LOSDAMBR/Rate Control 等）
- 问什么：PGW-CDR/SGW-CDR 携带哪些可选字段。
- 取值由上游特性（NB-IoT/eMTC 接入速率控制、字段控制）驱动，每特性开启自己子集，跨场景枚举。

### CDRAFTERTCCAUSE（conditional）
- 问什么：开启 CDRAFTERTCSW 强制费率切换话单后，关闭原因填什么。
- ②conditional：仅 CDRAFTERTCSW=ENABLE 时必填。

## 约束
- 唯一性：全局一份（最大记录数为 1，系统部署后已存在初始记录）。`severity=high`
- 生效时机：执行后立即生效。`severity=high`
- **高危（SPCDRCONTROL）**：配置 SPCDRCONTROL=PGW_SGW_CDR 后需检查流量容器占用率，防止资源耗尽引发用户激活成功率下降。`severity=high`（CRITICAL）
- conditional：CCFHCDRCC 仅在 CCFHCDRSW=ENABLE 时必填（十六进制 1～6 位）。`severity=medium`
- conditional：CDRAFTERTCCAUSE 仅在 CDRAFTERTCSW=ENABLE 时必填。`severity=medium`
- 长度规格：GSNNODEIDPREFIX 1～17 字符；GNIDELIMITER 1 字符；CCFHCDRCC 1～6 位十六进制。`severity=medium`
- 一致性：扩展 QoS 三参数（SGWLOTVQOSEXT/PGWLOTVQOSEXT/PGWLOSDQOSEXT）NSA 场景下通常同 ENABLE。`severity=medium`
- 默认值：系统部署后已存在初始记录，未输入参数保持当前值。`severity=low`
- 适用 NF：SGW-C、PGW-C、SMF。`severity=low`
