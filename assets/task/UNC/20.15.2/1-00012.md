# 配置 PCRF 对接链（Diameter）

> UNC（GGSN-C/PGW-C）到 PCRF 的 Gx 接口 Diameter 对接——建 VPN、配 Gx 逻辑接口、配本端/对端设备标识、建链路组与链路。被引用于 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md)。特性专属（对接 PCRF 用 Diameter 命令族；融合计费对接 CHF 用 TNF 服务化命令族 [1-00007](task/UNC/20.15.2/1-00007.md)，命令集完全不同，不可复用）。仅 2G/3G/4G 代际用 Gx 接口；5G 代际用 Npcf 服务化接口（PCF 经 NRF 发现，不建 Diameter 链路）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 建 VPN 实例 | `ADD VPNINST` | VPNINSTANCE（全网规划）；PCRF 绑的 VPN 须与 Gx 逻辑接口绑的 VPN 一致 |
| （可选）配 Gx 集中点模式 | `SET CONCENPOINT` | GXCONCENMODE=LOCALPORT（本端端口可配；缺省本端端口不可配，用系统分配） |
| 配 Gx 逻辑 IP | `ADD LOGICIP` | IPVERSION=IPv4；LOGICIPV4（全网规划）；VPNINSTNAME |
| 配 Gx 逻辑接口 | `ADD LOGICINF` | NAME（如 gxif1/0/0）；IPV4ADDRESS1（须与 LOGICIP 一致）；IPV4MASK1=255.255.255.255；VPNINSTANCE |
| 配本端设备标识 | `ADD DIAMLOCINFO` | HOSTNAME（如 pgwc_1）；REALMNAME（如 huawei.com）；PRODUCTNAME（如 unc@huawei.com） |
| 配 PCRF 设备标识 | `ADD PCRF` | HOSTNAME（如 pcrf_1）；REALMNAME；VPNINSTANCE（须与 Gx 逻辑接口同 VPN）；SUPFEANEGOSW（动态协商开关）；FEATURELIST（如 RELEASE8-1&RELEASE9-1&RELEASE10-1） |
| 配 PCRF 对端地址 | `ADD DIAMPEERADDR` | HOSTNAME（须与 ADD PCRF 一致）；ADDRTYPE=IPv4；IPV4ADDRESS（全网规划） |
| 配到 PCRF 链路组 | `ADD DIAMCONNGRP` | CONNGROUPNAME；LOCALHOSTNAME（本端 DIAMLOCINFO 主机名）；APPLICATION=GX；PEERHOSTNAME；SELECTMODE=SESSION_ID/ROUND-ROBIN |
| 配到 PCRF 链路 | `ADD DIAMCONNECTION` | DIAMCONNGRP（链路组名）；LOCINTERFACE（Gx 逻辑接口名）；PEERADDRTYPE=IPv4；PEERIPV4ADDR；LOCALPORT（GXCONCENMODE=LOCALPORT 时必配非 0） |

**典型脚本（单本端 + 单 PCRF，Gx 链路）**：
```
ADD VPNINST: VPNINSTANCE="vpn_gxif";
SET CONCENPOINT: GXCONCENMODE=LOCALPORT;
ADD LOGICIP: IPVERSION=IPv4, LOGICIPV4="10.8.10.1", VPNINSTNAME="vpn_gxif";
ADD LOGICINF: NAME="gxif1/0/0", IPVERSION=IPV4, IPV4ADDRESS1="10.8.10.1", IPV4MASK1="255.255.255.255", VPNINSTANCE="vpn_gxif";
ADD DIAMLOCINFO: HOSTNAME="pgwc_1", REALMNAME="huawei.com", PRODUCTNAME="unc@huawei.com";
ADD PCRF: HOSTNAME="pcrf_1", REALMNAME="example.com", VPNINSTANCE="vpn_gxif", SUPFEANEGOSW=ENABLE, FEATURELIST=RELEASE8-1&RELEASE9-1&RELEASE10-1;
ADD DIAMPEERADDR: HOSTNAME="pcrf_1", ADDRTYPE=IPv4, IPV4ADDRESS="10.11.21.59";
ADD DIAMCONNGRP: CONNGROUPNAME="dconn_pcrf1", LOCALHOSTNAME="pgwc_1", APPLICATION=GX, PEERHOSTNAME="pcrf_1", SELECTMODE=SESSION_ID;
ADD DIAMCONNECTION: DIAMCONNGRP="dconn_pcrf1", LOCINTERFACE="gxif1/0/0", PEERADDRTYPE=IPv4, PEERIPV4ADDR="10.11.21.59", LOCALPORT=16450;
```

**步骤位置**：PCC 基本功能配置的第一阶段（链路前提）；下游是 [1-00013](task/UNC/20.15.2/1-00013.md)（PCRF 分组与选择）和 [1-00014](task/UNC/20.15.2/1-00014.md)（PCC 开关）。仅 2G/3G/4G 代际需配；5G 代际走 Npcf 服务化接口（PCF 经 NRF 发现），不建 Diameter 链路。

## 场景差异

- **单本端 + 单 PCRF（基本）**：1 个 DIAMLOCINFO + 1 个 PCRF + 1 条链路组 + N 条链路（PCRF 多 IP 则多链路）。被引用于 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md)。
- **多本端（CU Full Mesh，不同 PGW-U 用户 IP 重叠）**：配多个 DIAMLOCINFO（pgwc_1/pgwc_2）+ LOCALHOSTGRP + LOCALHOSTBIND + GXUPFGROUP + UPFBINDGXUPFGRP + UPFGLOCGBNDGRP + UPFGBINDLOCG（UPF 组绑不同本端主机组），PCC 模板 LOCSLCTMODE=UPFGRP。仅当"同一 PGW-C 对接多个 PGW-U 且不同 PGW-U 用户 IP 存在相同/重叠，且 PCRF 要求 UE IP + GxLocalHost 唯一确定用户"时配。对应 atom：[ADD LOCALHOSTGRP](task/UNC/20.15.2/0-00042.md) · [ADD LOCALHOSTBIND](task/UNC/20.15.2/0-00041.md) · [ADD GXUPFGROUP](task/UNC/20.15.2/0-00036.md) · [ADD UPFBINDGXUPFGRP](task/UNC/20.15.2/0-00090.md) · [ADD UPFGLOCGBNDGRP](task/UNC/20.15.2/0-00092.md) · [ADD UPFGBINDLOCG](task/UNC/20.15.2/0-00091.md)。
- **Supported-Features 动态协商**：`ADD PCRF` 设 SUPFEANEGOSW=ENABLE + FEATURELIST；加 [SET PCCTIMER](task/UNC/20.15.2/0-00196.md) 配动态协商定时器（缺省用默认值）。同 PCRF Group 内 PCRF 的 feature 能力须一致。
- **5G 代际（Npcf 接口）**：不建 Gx Diameter 链路——PCF 经 NRF 服务化发现（本端 [ADD SBIAPLE](task/UNC/20.15.2/0-00074.md) 注册 + 对端 PCF 注册到 NRF）。本 compound 不适用 5G。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| GXCONCENMODE=LOCALPORT | `ADD DIAMCONNECTION` 必配 LOCALPORT 非 0；本端端口可配 |
| GXCONCENMODE=缺省 | 本端端口不可配，用系统自动分配；ADD DIAMCONNECTION 不配或配 LOCALPORT=0 |
| SELECTMODE=SESSION_ID | 同一用户会话固定走一条链路 |
| SELECTMODE=ROUND_ROBIN | 不同会话轮询分担到多条链路 |
| SUPFEANEGOSW=ENABLE | 启用 Supported-Features 动态协商；须配 FEATURELIST；同 PCRF Group 内 PCRF feature 须一致 |

> 联动特性级 DP：是否配本 compound 由 [2-00005](task/UNC/20.15.2/2-00005.md) 的"代际"DP 驱动（5G 不配；2G/3G/4G 配）。

## 约束

- **PCRF VPN 须与 Gx 逻辑接口同 VPN**（critical）：`ADD PCRF` 的 VPNINSTANCE 必须与 `ADD LOGICINF` 的 VPNINSTANCE 一致 — 否则链路不通。
- **LOGICINF 与 LOGICIP 的 IP+VPN 须一致**（critical）：逻辑接口的 IPV4ADDRESS1+VPNINSTANCE 须与 ADD LOGICIP 的 LOGICIPV4+VPNINSTNAME 完全一致。
- **PCRF HOSTNAME 跨命令一致**（warning）：`ADD PCRF` 的 HOSTNAME 须与 `ADD DIAMPEERADDR`/`ADD DIAMCONNGRP` 的 PEERHOSTNAME 一致。
- **本端端口与集中点模式联动**（warning）：GXCONCENMODE=LOCALPORT 时 ADD DIAMCONNECTION 必配 LOCALPORT 非 0；否则不配或配 0。
- **同 PCRF Group feature 须一致**（info）：SUPFEANEGOSW=ENABLE 时，加入同一 PCRFGROUP 的 PCRF 支持的 feature 能力须一致。
