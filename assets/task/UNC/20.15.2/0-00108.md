# DSP PCRFSTATUS 原子任务
> 命令静态知识见 [DSP PCRFSTATUS](command/UNC/20.15.2/DSP-PCRFSTATUS.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：PGW-C、GGSN。本命令是 PCRF 链路就绪性核对入口——PCC 调测前先确认 PCRF 状态正常，再做规则下发与负荷分担验证。

## 配置方法
支持两种查询粒度：查全部 PCRF（不传参）或查指定 PCRF（传 PCRFNAME）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PCRFNAME | PCRF 主机名（可选，省略则查全部） | `pcrf_1`、`pcrf_2` |

典型脚本（负荷分担验证场景，逐个核对组内各 PCRF 状态）：
```
DSP PCRFSTATUS:PCRFNAME="pcrf_1";
DSP PCRFSTATUS:PCRFNAME="pcrf_2";
```

调测语义：
- 缺省承载 GBR 保障调测：在用户激活前先核对 PCRF 状态是否正常；未部署 PCRF 或状态异常则不走后续承载规则下发核对。
- PCC 负荷分担调测：核对组内各 PCRF 状态均正常后，再结合消息跟踪比对各 PCRF 的 CCR-I 消息数比例与配置的负荷分担比例是否一致。
- Gx over SCTP / 到 PCRF 的数据调测：License 开关 ENABLE 后，先核对 PCRF 状态，再做后续数据对接验证。

## 决策点（若有）

### 查询粒度选择（PCRFNAME 省略与否）
- 问什么：查全部 PCRF 还是单个指定 PCRF。
- 全局巡检 → 省略 PCRFNAME，查所有 PCRF 连接状态。
- 定位单点 / 负荷分担逐个核对 → 传具体 PCRFNAME（如 pcrf_1、pcrf_2）。
- PCRFNAME 数据来源为对端协商（Diameter 主机名），取值范围 1~127 字符。

### PCRF 状态结果判读
- Ready → PCRF 链路正常，可继续后续 PCC 调测。
- 仅一条 "Not Ready" 记录 → 此 PCRF 未配置 Diameter 链路，需先补链路配置。
- 非直连（通过 DRA 连接）的 PCRF → 状态不显示，属预期。

## 约束（若有）
- 非直连（通过 DRA 连接）的 PCRF 状态不显示。`severity=medium`
- 若只显示一条 "Not Ready" 记录，则此 PCRF 未配置 Diameter 链路。`severity=high`
- PCRFNAME 大小写敏感性由软参 BIT 150 控制，不支持空格。`severity=low`
