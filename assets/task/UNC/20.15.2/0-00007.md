# ADD APNPOLICYMODE 原子任务

> 命令静态知识见 [ADD APNPOLICYMODE](command/UNC/20.15.2/ADD-APNPOLICYMODE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

逐 APN 增加一条"基于 APN 的策略接口选择方式"记录，核心是锁定 `TMACCTYPE=UE5G_RAT4G`（5G 终端 4G 接入）时该 APN 走的策略接口。其余条件开关（BY5GSIWKI / BY5GCNRI / PCFRESELBYPCFID）按互操作场景决定是否打开。该命令通常跟在 `SET POLICYMODE`（全局策略接口选择方式）之后，作为**按 APN 维度的细化/覆盖**。

**主参数取值样例（证据归纳）**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| APN | `apn-test` / `HUAWEI.COM` | 已配置数据中获取（ADD APN 已建） | 指定要细化策略接口的 APN/DNN，须先存在 |
| TMACCTYPE | `UE5G_RAT4G` | 本端规划 | 固定为"5G 终端 4G 接入"，本命令的语义前提 |
| BY5GSIWKI | `DISABLE` | 本端规划 | 按 5GS 互操作指示选策略接口的开关，证据样例均 DISABLE |
| FORCED | `Gx` 或 `Npcf` | 本端规划 | **强制指定的策略接口**，是本命令的关键产出 |
| PCFRESELBYPCFID | （可选，未在样例中出现） | 本端规划 | 是否按 PCF 实例标识决策，开启另一条选择路径 |
| BY5GCNRI | `DISABLE` | 本端规划 | 按 5GC 无限制接入标识选择的开关 |

**典型行**

```
ADD APNPOLICYMODE: APN="HUAWEI.COM",TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=DISABLE, FORCED=Npcf;
ADD APNPOLICYMODE: APN="HUAWEI.COM",TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=DISABLE, FORCED=Gx;
```

## 决策点

### 策略接口强制选择（FORCED）

证据中 FORCED 出现 `Gx`（4G/传统 PCRF）与 `Npcf`（5G PCF）两种取值，分叉由"该 APN 下 5G 终端 4G 接入用户应被哪个策略网元服务"决定。

| 选项 | FORCED 取值 | 适用场景 |
|---|---|---|
| 走 Npcf 接口 | `Npcf` | 部署 5G PCF，希望 5G 终端即使 4G 接入也由 PCF 下发策略 |
| 走 Gx 接口 | `Gx` | 沿用 PCRF，或 PCF 不支持该互操作场景 |

> 选 `Npcf` 用于 2G/3G 接入走 Npcf 时，须**提前和对端 PCF 确认是否支持**。

## 约束

- **仅对新激活用户生效**：本命令执行后，已激活用户不受影响，新激活才按此选择 · severity=HIGH · 否则现网存量用户行为不变，易被误判为配置失效。
- **最多 20000 条记录**：超过上限新增失败 · severity=MEDIUM。
- **2G/3G 接入走 Npcf 需对端支持**：用本命令给 2G/3G 接入配 Npcf 前，和对端 PCF 确认 · severity=HIGH · 不确认可能导致 PCF 选择失败、用户无法激活。
- **前置依赖**：APN 须先由 ADD APN 建好；策略接口选择通常先做全局 `SET POLICYMODE`，再按 APN 细化 · severity=MEDIUM。
