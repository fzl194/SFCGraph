# SET GLBTARIFFGROUP 原子任务
> 命令静态知识见 [SET GLBTARIFFGROUP](command/UNC/20.15.2/SET-GLBTARIFFGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

UNC 选择费率切换组的优先级：user-profile 实例绑定 > APN 实例绑定 > 全局配置（本命令）。即 user-profile / APN 都未绑定时，回退到本命令设置的全局组。用于离线计费费率切换（WSFD-011201）与融合/在线计费费率切换（WSFD-011206、WSFD-104003、WSFD-109001）。

## 配置方法
填一个由 ADD TARIFFGROUP 预先创建的费率切换组名。仅一个参数，全局唯一。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| GLBTARIFFGRP（费率切换组名） | 本端规划（须先 ADD TARIFFGROUP 建组） | `testtarif` / `tar_nb` |

典型脚本：
```
SET GLBTARIFFGROUP: GLBTARIFFGRP="testtarif";
```

## 决策点

### 全局费率切换组名选择
- 问什么：全局兜底用哪个费率切换组。
- 取值依赖：组名须先经 ADD TARIFFGROUP 创建（含工作日表 ADD WEEKDAY、费率切换配置 ADD TARIFFGROUP 内的 TCMODE 等）；本命令只做绑定引用。
- 多场景差异：不同特性（离线 / 融合 / 在线）示例组名不同（testtarif / tar_nb），由运营商费率计划决定，非命令内部能推导。

### 是否覆盖到 APN/UserProfile 粒度
- 问什么：全局组之外，是否还需在 APN / user-profile 实例上单独绑定费率切换组。
- 本命令仅设全局；APN 粒度需后续在 APN 实例上配置（覆盖全局），优先级更高。

## 约束
- 唯一性：全局一份（最大记录数为 1）。`severity=medium`
- 生效时机：执行后立即生效。`severity=high`
- 前置：GLBTARIFFGRP 指向的组名必须先经 ADD TARIFFGROUP 添加，否则命令失败。`severity=high`
- 适用 NF：SGW-C、PGW-C、SMF。`severity=low`
- 优先级：user-profile > APN > 全局（本命令）；细粒度覆盖全局。`severity=medium`
