# 配置全局费率切换组（SET GLBTARIFFGROUP）

> 命令静态知识见 [SET GLBTARIFFGROUP](command/UNC/20.15.2/SET-GLBTARIFFGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET GLBTARIFFGROUP 把一个由 `ADD TARIFFGROUP` 预先创建的费率切换组名绑定为**全局兜底费率切换组**。UNC 选择费率切换组的优先级是：user-profile 实例 > APN 实例 > 全局（本命令）。即 user-profile / APN 都未绑定时，回退到本命令设置的全局组。用于离线计费费率切换（WSFD-011201）与融合/在线计费费率切换（WSFD-011206、WSFD-104003、WSFD-109001）。本命令仅 1 个配置维度：**全局费率切换组名绑定**。

### 配置维度 1：全局费率切换组名绑定（参数 GLBTARIFFGRP，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 费率切换组名 | `GLBTARIFFGRP` | 字符串 1~31 字符 | 无 | 指定全局兜底费率切换组 | 须先由 `ADD TARIFFGROUP` 创建 |

> 数据来源：本端规划。组名必须先经 `ADD TARIFFGROUP` 添加（含工作日表 `ADD WEEKDAY`、费率切换配置 TCMODE 等），本命令只做绑定引用；输入单空格将删除已有配置项。

**典型脚本**

```
SET GLBTARIFFGROUP: GLBTARIFFGRP="testtarif";
```

**配置链路（本命令位置）**

```
ADD WEEKDAY            # 建工作日表（费率切换组依赖）
ADD TARIFFGROUP        # 建费率切换组（组名 + 时段 + 费率类型）
SET GLBTARIFFGROUP  ← 本命令   # 绑定为全局兜底组
```

**步骤位置**：在费率切换类特性编排中位于"全局费率切换组绑定"步骤——紧随 `ADD TARIFFGROUP` 建组之后，是费率切换链路的最后一步全局绑定；APN / user-profile 粒度的覆盖绑定（优先级更高）在各自实例配置中独立完成。

## 决策点

### 决策点：全局费率切换组名选择（DP 0-03093）

问：全局兜底用哪个费率切换组？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 离线计费费率组 | `GLBTARIFFGRP` 取离线场景组名（如 `testtarif`） | 离线计费费率切换（WSFD-011201） |
| 融合/在线计费费率组 | `GLBTARIFFGRP` 取融合场景组名（如 `tar_nb`） | 融合/在线计费费率切换（WSFD-011206/104003/109001） |

> 组名取值由运营商费率计划决定，非命令内部能推导；不同特性示例组名不同。

### 决策点：是否覆盖到 APN/UserProfile 粒度（DP 0-03094）

问：全局组之外，是否还需在 APN / user-profile 实例上单独绑定费率切换组？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅全局兜底 | 只配本命令，APN/user-profile 不绑 | 所有用户统一费率切换 |
| APN 粒度覆盖 | 本命令 + APN 实例绑定（优先级更高） | 按 APN 差异化费率切换 |
| user-profile 粒度覆盖 | 本命令 + user-profile 实例绑定（优先级最高） | 按用户模板精细化费率切换 |

> 优先级：user-profile > APN > 全局（本命令）；细粒度覆盖全局。

## 约束

- rule-0-03231 · severity=critical · 前置：GLBTARIFFGRP 指向的组名必须先经 ADD TARIFFGROUP 添加（含工作日表 ADD WEEKDAY、TCMODE 等依赖），否则命令失败 — 后果：前置缺失导致绑定失败
- rule-0-03232 · severity=high · 优先级：user-profile > APN > 全局（本命令）；细粒度覆盖全局，规划全局组时须确认上层无更细绑定 — 后果：误判生效组导致费率切换不符合预期
- rule-0-03233 · severity=high · 生效时机：执行后立即生效 — 后果：变更后无延迟，回滚须立即同步
- rule-0-03234 · severity=warning · 唯一性：全局一份（最大记录数 1），不可重复配置 — 后果：重复配置覆盖既有全局组
- rule-0-03235 · severity=info · 适用 NF：SGW-C、PGW-C、SMF — 后果：跨 NF 部署时各网元独立配置
