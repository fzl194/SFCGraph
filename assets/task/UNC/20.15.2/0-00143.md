# RMV RULE 原子任务
> 命令静态知识见 [RMV RULE](command/UNC/20.15.2/RMV-RULE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

**高危命令**。删除对象是 `RULE`（规则）记录。通过 `RULENAME/POLICYTYPE/RULESPECTYPE` 三参数组合定位；**不输入任何参数 = 删除系统中所有规则**（高危批量）。

## 配置方法
按规则名 + 策略类型 + 规格类型组合定位删除。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| RULENAME | 规则名称（对端协商，optional） | `app_sta_rule` |
| POLICYTYPE | 策略类型（对端协商，optional，枚举） | `Charging` / `QoS` |
| RULESPECTYPE | 规则规格类型（global_planned，optional，枚举） | `预定义` |

典型脚本（删单条命名规则）：
```
RMV RULE: RULENAME=app_sta_rule;
```

全删（高危，生产慎用）：
```
RMV RULE:;
```

## 决策点（若有）

### 删除范围（单条 vs 全删）
- 问什么：删一条命名规则，还是删系统中所有规则。
- 精确删：给 `RULENAME`（必要时 + `POLICYTYPE` + `RULESPECTYPE`）锁定单条。
- 全删：**所有参数都不输入 = 删除全部规则**——高危，配置量大时单次执行可能删不完，需多次执行。

### 是否先解绑再删（绑定类型决定级联）
- 问什么：待删 Rule 是否与 UserProfile / DNAI 有绑定关系。
- 与 UserProfile 绑定：删 Rule 时**自动解除**与该用户模板的绑定（系统级联，无需手动 RMV RULEBINDING）。
- 与 DNAI 绑定：删 Rule 时**自动解除**与 DNAI 的绑定。
- 被业务在用：**不允许删除**，须先停业务/解除在用状态。

### 规则配置错误的重建顺序（ADC/SA 调测场景）
- 调测 ADC（WSFD-109102）或 IPv6 SA（WSFD-109103）时，若 PCF 预定义规则标识不一致：须先 RMV RULEBINDING，再 RMV RULE，再 ADD RULE + ADD RULEBINDING 重建，然后重跑校验步骤。
- 即：规则绑定关系未解除前，直接 RMV RULE 会触发自动级联，但调测流程要求显式按序重建以保证标识一致。

## 约束（若有）
- 高危：删除规则会连带删除规则下所有绑定关系，可能导致在用用户规则匹配错误、计费错误，删前必须确认规则未被业务在用。`severity=high`
- 全删禁止：所有参数都不输入 = 删全部规则，生产环境禁止裸执行；配置量大时单次删不完需多次执行。`severity=high`
- 在用阻删：规则正被业务使用时禁止删除，须先停业务。`severity=high`
- 自动解绑级联：删 Rule 自动解绑 UserProfile 与 DNAI 绑定——执行前需预期级联影响（调测场景仍建议先显式 RMV RULEBINDING 再 RMV RULE）。`severity=high`
- 规则名格式：RULENAME 长度 1～63、不支持空格、不区分大小写。`severity=medium`
- 生效时机：立即生效。`severity=medium`
