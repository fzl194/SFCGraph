# 删除规则（RMV RULE）

> 命令静态知识见 [RMV RULE](command/UNC/20.15.2/RMV-RULE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按规则名 + 策略类型 + 规则规格类型组合定位删除；三个参数全部可选，全部不输入 = 删除系统中所有规则（高危批量）。删除对象是 `RULE`（规则）记录。

### 参数维度表

| 维度 | 参数 | 数据来源 | 取值 | 作用 | 适用场景 |
|------|------|----------|------|------|----------|
| 规则名称 | RULENAME | 对端协商 | 字符串 1～63，不支持空格、不区分大小写 | 定位待删规则；未设置 = 删全部 | 精确删指定规则名 |
| 策略类型 | POLICYTYPE | 对端协商 | 枚举：PCC / BWM / HEADEN / WEBPROXY / IPREDIR / SMARTREDIRECT / REMARK_FPI / SRV_TRIGGER / TRAFFICCLASS / LBO / FIREWALL / ULCL / ADC / QOS / FUP / NON_SPECIFIC_TYPE | 收窄到指定策略类型规则 | 按策略类型过滤 |
| 规则规格类型 | RULESPECTYPE | 全网规划 | 枚举：DEFAULT（默认）/ SPECIFICATION_LIMITED（规格受限） | 收窄到指定规格类型 | 规格受限规则场景 |

> 说明：高危命令。不输入任何参数 = 删除系统中所有规则；配置量较大时单次执行可能无法删除全部记录，需要执行多次。删除会连带解除 Rule 与 UserProfile / DNAI 的绑定关系。

### 典型脚本

删名为 testrule 的所有规则：
```
RMV RULE: RULENAME="testrule";
```

按规则名 + 策略类型精确删：
```
RMV RULE: RULENAME="app_sta_rule", POLICYTYPE=ADC;
```

全删（高危，生产慎用）：
```
RMV RULE:;
```

### 步骤位置

规则配置错误（如 PCF 预定义规则标识不一致）的重建顺序（ADC/SA 调测场景）：
1. `LST RULE` — 核对目标规则是否被业务在用（未建）
2. `RMV RULEBINDING` — 先显式解除与 UserProfile 的绑定（0-00144）
3. `RMV RULE` — 删除原规则（本案）
4. `ADD RULE` — 按正确标识重建规则（0-00071）
5. `ADD RULEBINDING` — 重建绑定关系

## 决策点

### 决策点：删除范围——单条 vs 全删（DP 0-03025）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 精确删单条 | RULENAME（必要时 + POLICYTYPE + RULESPECTYPE）锁定 | 已知待删规则名，常规回收 |
| 按策略类型过滤 | POLICYTYPE（+ 可选 RULENAME/RULESPECTYPE） | 删某类策略规则（如清空 ADC 规则） |
| 全删 | 所有参数都不输入 | 清空重建；高危，生产禁止裸执行；配置量大需多次执行 |

> 注意：不输入任何参数 = 删除系统中所有规则——高危；配置量大时单次执行可能删不完，需多次执行；删前必须确认规则未被业务在用。

### 决策点：删除在用规则的绑定级联处理（DP 0-03026）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 先解绑再删 | 删前显式 RMV RULEBINDING 解除 UserProfile 绑定 | 调测场景要求显式按序重建（ADC/SA 标识一致），保证标识一致 |
| 直接删除（自动级联） | 直接 RMV RULE，系统自动解除 UserProfile / DNAI 绑定 | 非调测场景，可接受系统自动级联 |
| 暂不删除 | — | Rule 正被业务使用，系统拒绝删除；须先停业务/解除在用状态 |

> 注意：删除 Rule 会自动解除与 UserProfile、DNAI 的绑定（系统级联）；但调测 ADC（WSFD-109102）或 IPv6 SA（WSFD-109103）场景若 PCF 预定义规则标识不一致，须先 RMV RULEBINDING → 再 RMV RULE → 再 ADD RULE + ADD RULEBINDING 重建，显式按序保证标识一致。Rule 正被业务使用时系统拒绝删除。

## 约束

· rule-0-03061 · severity=critical · 全删禁止：所有参数都不输入 = 删除系统中所有规则，生产环境禁止裸执行；配置量大时单次执行可能删不完需多次执行
· rule-0-03062 · severity=critical · 在用阻删：Rule 正被业务使用时不允许删除，须先停业务/解除在用状态
· rule-0-03063 · severity=high · 高危业务影响：删除规则会连带删除规则下所有绑定关系，可能导致在用用户规则匹配错误、计费错误，删前必须确认规则未被业务在用
· rule-0-03064 · severity=high · 自动解绑级联：删 Rule 自动解除与 UserProfile、DNAI 的绑定关系——执行前需预期级联影响；调测场景仍建议先显式 RMV RULEBINDING 再 RMV RULE
· rule-0-03065 · severity=info · 规则名格式：RULENAME 长度 1～63、不支持空格、不区分大小写；执行后立即生效
