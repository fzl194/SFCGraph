# MOD URRGROUP 原子任务
> 命令静态知识见 [MOD URRGROUP](command/UNC/20.15.2/MOD-URRGROUP.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：PGW-C、SMF。本命令修改使用量上报规则组，指定上下行发起使用的 URR 名称（即上下行报文如何计费），是计费三件套（URR→URRGROUP→PCCPOLICYGRP）的规则组绑定环节。

## 配置方法
按 URRGROUPNAME 定位条目，配置上行/下行发起的 URR 名称对与不计费标记。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| URRGROUPNAME | 使用量上报规则组名称（必选，本地规划，1~31 字符） | `urrgroup1` |
| UPURRNAME1/2/3 | 上行发起 URR 名称 1/2/3（可选，1~31 字符，指向已建 URR） | `upurr1` |
| DOWNURRNAME1/2/3 | 下行发起 URR 名称 1/2/3（可选，1~31 字符，指向已建 URR） | `downurr1` |
| NOCHARGINGFLAG | 不计费标记（可选，枚举） | 按产品定义 |

典型用法：先 ADD URR 建好成员 URR，再用本命令把上下行 URR 名称绑定进规则组；调测融合计费流量计费（WSFD-011206）时与 MOD URR、MOD PCCPOLICYGRP、MOD RULE 联动按规划值重新绑定 RG。

## 决策点

### 上下行 URR 名称组合（UPURRNAME/DOWNURRNAME 跨场景枚举）
UPURRNAME1/2/3 与 DOWNURRNAME1/2/3 三档为编号位（无优先级语义），实际选用哪几档、上下行是否对称取决于业务流量模型（对称计费/非对称计费/多 URR 分流）。组合方式由数据规划按场景确定，至少保留任意一个有效名称。

## 约束
- **立即生效**：命令执行后立即生效。
- **覆盖重写模式（关键）**：本命令采取覆盖重写，即先删除 URRGROUPNAME 的原配置后新建，非增量修改；修改前需备齐完整目标配置。
- **被上下文激活引用时禁止修改**：若引用该 URRGROUP 的上下文激活时存在特定 URR 记录，则不允许修改；需先去活相关上下文。
- **至少一个 URR 名称**：修改后 UPURRNAME1/DOWNURRNAME1/UPURRNAME2/DOWNURRNAME2/UPURRNAME3/DOWNURRNAME3 至少存在任意一个。
- **成员 URR 前置存在**：UPURRNAME*/DOWNURRNAME* 指向的 URR 应已由 ADD URR/MOD URR 建好。
- **同 URRGROUP 内成员唯一性**：同一 URRGROUP 内不能配置重复的 URR 名称引用。
