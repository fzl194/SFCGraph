# 设置话单存储控制参数（SET CDRSTORAGECTRL）

> 命令静态知识见 [SET CDRSTORAGECTRL](command/UNC/20.15.2/SET-CDRSTORAGECTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

全局单记录参数，按局点缓存保留时长规划。超期总天数 = CDREXPIREDAY + CDREXPIREWEEK × 7。存在系统初始记录（DAY=2, WEEK=4，即总 30 天）。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 话单缓存超期天数 | CDREXPIREDAY | 0~6 天整数（本端规划） | 系统初始 2 | 配置缓存话单超期天数 | 局点规划 |
| 话单缓存超期周数 | CDREXPIREWEEK | 0~52 整数（本端规划） | 系统初始 4 | 配置缓存话单超期周数 | 局点规划 |

超期总天数 = DAY + WEEK × 7。初始值 DAY=2/WEEK=4 → 总 30 天；WSFD-011201 配 DAY=3/WEEK=3 → 总 24 天；WSFD-011206 配 DAY=2/WEEK=4 → 总 30 天。

典型脚本（总 24 天）：
```
SET CDRSTORAGECTRL: CDREXPIREDAY=3,CDREXPIREWEEK=3;
```
（DAY=2/WEEK=4，维持初始总 30 天）：
```
SET CDRSTORAGECTRL:CDREXPIREDAY=2,CDREXPIREWEEK=4;
```

步骤位置：本任务（全局唯一记录）→ 与 `SET CDRSTRGSTATUS`（缓存目录锁定/解锁）配合：锁定目录 → 操作话单 → 配超期时间 → 解锁目录

## 决策点

### 决策点：超期总天数取值（DP 0-03045）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 正常超期检测 | DAY=1~6, WEEK=0~52（总天数 = DAY + WEEK×7） | 局点按磁盘容量规划保留时长 |
| 设为 0（不检测不上报） | DAY=0, WEEK=0 | 所有缓存文件不超期检测，CG/CHF 恢复正常后全部回放（话单长期留存，有计费信息丢失/磁盘占用风险） |
| 维持初始值 | DAY=2, WEEK=4（总 30 天） | 沿用系统初始记录 |

### 决策点：超期后果与告警（DP 0-03046）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 超期触发告警 | 总天数 > 0 时，话单超期触发 ALM-81059 超期话单缓存告警 | 需运维及时处理，否则用户计费信息丢失 |
| 超期不再发往 CG/CHF | 判超期后即使 CG/CHF 恢复正常也不再回放该文件 | 计费信息丢失风险 |
| 设 0 不上报告警 | DAY=0, WEEK=0 时不上报告警 | 无告警但话单长期占磁盘 |

## 约束

- **数据安全**（critical，rule-0-03111）：数据安全：超期话单不再发往 CG/CHF，判超期后即使 CG/CHF 恢复正常也不再回放该文件，导致用户计费信息丢失
- **告警**（warning，rule-0-03112）：告警：超期触发告警 ALM-81059 超期话单缓存，话单长期超期未处理会导致用户计费信息丢失
- **数据安全**（warning，rule-0-03113）：数据安全：设为 0（DAY=0,WEEK=0）时所有缓存文件不超期检测不上报告警，CG/CHF 恢复正常后全部回放，有计费信息丢失/磁盘占用风险
- **规格上限**（info，rule-0-03114）：规格上限：最大记录数 1，全局唯一记录，重复配即覆盖
- **生效时机**（info，rule-0-03115）：生效时机：命令执行后立即生效，无需刷新
