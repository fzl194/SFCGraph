# SET CDRSTORAGECTRL 原子任务
> 命令静态知识见 [SET CDRSTORAGECTRL](command/UNC/20.15.2/SET-CDRSTORAGECTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：SGW-C、PGW-C、SMF。配置缓存话单超期总天数 = CDREXPIREDAY + CDREXPIREWEEK × 7。超期话单不再发往 CG/CHF；超期触发告警 ALM-81059 超期话单缓存。设 0 时所有缓存文件不超期检测且不上报告警，CG/CHF 恢复正常后全部回放。存在系统初始记录（DAY=2, WEEK=4，即总 30 天）。证据显示获取方法单一（本端规划 4），属 global_planned。

## 配置方法
全局单记录参数，按局点缓存保留时长规划。

| 参数 | 配置方法 | 取值范围 / 样例 |
|---|---|---|
| CDREXPIREDAY | 话单缓存超期天数（本端规划） | 0～6 天，样例 `3` / `2` |
| CDREXPIREWEEK | 话单缓存超期周数（本端规划） | 0～52，样例 `3` / `4` |

超期总天数 = DAY + WEEK × 7。初始值 DAY=2/WEEK=4 → 总 30 天；WSFD-011201 配 DAY=3/WEEK=3 → 总 24 天；WSFD-011206 配 DAY=2/WEEK=4 → 总 30 天。

典型脚本：
```
SET CDRSTORAGECTRL: CDREXPIREDAY=3,CDREXPIREWEEK=3;
```
（DAY=2/WEEK=4，维持初始）：
```
SET CDRSTORAGECTRL:CDREXPIREDAY=2,CDREXPIREWEEK=4;
```

## 决策点

### 超期总天数取值（DAY + WEEK × 7）
- 问什么：话单缓存保留多久后判超期。
- DAY ∈ 0～6，WEEK ∈ 0～52，总天数 = DAY + WEEK × 7。
- 设为 0（DAY=0,WEEK=0）表示不检测、不上报告警，CG/CHF 恢复正常后全部回放（话单长期留存，有计费信息丢失/磁盘占用风险）。
- 设大值长期占缓存、占磁盘；设小值正常流量文件可能被误判超期不发往 CG/CHF。以局点规划和磁盘容量为准。

## 约束
- 立即生效：无需刷新。`severity=low`
- 最大记录数 1：全局唯一记录，重复配即覆盖。`severity=medium`
- 存在系统初始记录：DAY=2/WEEK=4（总 30 天），未配时沿用初始值。`severity=low`
- 超期触发告警 ALM-81059：话单长期超期未处理会导致用户计费信息丢失。`severity=high`
- 超期话单不再发往 CG/CHF：判超期后即使 CG/CHF 恢复正常也不再回放该文件。`severity=high`
- 设为 0 不检测不上报：所有缓存文件不超期检测，CG/CHF 恢复正常后全部回放。`severity=medium`
