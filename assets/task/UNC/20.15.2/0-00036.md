# ADD GXUPFGROUP 原子任务
> 命令静态知识见 [ADD GXUPFGROUP](command/UNC/20.15.2/ADD-GXUPFGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令仅创建 UPF 组命名对象，下游 `ADD UPFBINDGXUPFGRP` 通过 UPFGRPNAME 引用本组并把具体 UPF HostName 绑进来。单参数命令，本端规划命名。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| UPFGRPNAME | Gx UPF 组名称（本端规划，1~32 字符，不支持空格及 `#` `$` `&` 等特殊字符） | `upfgrp_01` / `upfgrp_02` |

典型脚本：
```
ADD GXUPFGROUP: UPFGRPNAME="upfgrp_01";
ADD GXUPFGROUP: UPFGRPNAME="upfgrp_02";
```

## 决策点

### UPF 组的划分粒度
PCC 对接（WSFD-109101，配置与 PCRF 对接数据）中本命令为"配置 UPF 组"步骤，划分多少个组、每组承载哪些 UPF 由网络规划决定：
- 单组承载所有 UPF → 配 1 条，所有 UPF 绑入此组。
- 按 PCRF 实例 / 地域 / 容量分组 → 配多条（样例配了 `upfgrp_01` + `upfgrp_02` 两组），各组后续分别绑不同 UPF 实例。

> UPFGRPNAME 仅本端规划口径，无取值分叉（自动比对：单一"本端规划"来源）。

## 约束
- 命名规范：1~32 字符，不支持空格及特殊字符 `#`、`$`、`&` 等，由 `_`、`-` 等组成。`severity=high` · 后果：命令解析失败。
- 前置/后置依赖：先建本 UPF 组，下游 `ADD UPFBINDGXUPFGRP` 才能把 UPF 实例绑入组。`severity=high` · 后果：绑定失败。
- 规格上限：最大记录数 300。`severity=medium`
- 生效时机：执行后立即生效。`severity=low`
