# ADD N40MSGTEMP 原子任务

> 命令静态知识见 [ADD N40MSGTEMP](command/UNC/20.15.2/ADD-N40MSGTEMP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD N40MSGTEMP 建一个"模板名 + 7 个字段开关"的模板，控制 N40 消息里携带哪些信元。模板建好后由 N40 接口相关流程引用。系统部署后已有初始记录（各字段默认值见下）。

**字段开关与默认值**

| 参数 | 信元 | 默认 | 取值 | 典型用途 |
|---|---|---|---|---|
| TEMPLATENAME | 模板名 | — | 1~63 字符，不区分大小写 | 必填，命名 |
| SECRATUSAGE | RANSecondaryRATUsageReport | ENABLE | ENABLE/DISABLE | 是否携带 |
| NBEXTENDATTR | NB 扩展属性 | ENABLE | ENABLE/DISABLE | NB-IOT/LTEM 接入时携带 APN Rate Control、Serving PLMN Rate Control、CP only indication、PtP 隧道 |
| QFIDOWNLINK | QFI 容器 DownLinkVolume | ENABLE | ENABLE/DISABLE | 是否携带 |
| HOMECHGID | HomeProvidedChargingID | ENABLE | ENABLE/DISABLE | 是否携带 |
| HWQBCIDCT | huaweiQBCIndication | DISABLE | ENABLE/DISABLE | 默认不携带 |
| APPSVCPROVID | applicationServiceProviderIdentity | DISABLE | ENABLE/DISABLE | 默认不携带 |
| RCP | roamingChargingProfile | ENABLE | ENABLE/DISABLE | 是否携带 |

**典型行（NB-IOT/LTEM 场景，开 NB 扩展属性）**

```
ADD N40MSGTEMP: TEMPLATENAME="n40attr", NBEXTENDATTR=ENABLE;
```

**典型场景**：与 SET N40APIVER 配合——先 SET N40APIVER 设 API 版本与 FeatureList（如 NBIOTCHG-1&LTEMCHG-1），再 ADD N40MSGTEMP 控制对应字段开关。

## 决策点

### 哪些字段开关需要显式配置

默认值已覆盖大多数场景（多数字段默认 ENABLE）。需要偏离默认时才显式配置：

| 场景 | 需显式改的字段 |
|---|---|
| NB-IOT/LTEM 计费 | NBEXTENDATTR（确保 ENABLE，且需 SET N40APIVER 开 FeatureList） |
| 不携带某信元 | 把对应字段置 DISABLE |
| 华为特有 QBC 指示 | HWQBCIDCT 置 ENABLE（默认 DISABLE） |
| 应用服务商标识 | APPSVCPROVID 置 ENABLE（默认 DISABLE） |

### 是否新建模板还是改初始记录

系统部署后已存在初始记录。新增场景才用 ADD；调整已有模板用 MOD N40MSGTEMP。

## 约束

- **立即生效** · severity=LOW。
- **最多 101 条记录** · severity=MEDIUM · 超过上限新增失败。
- **初始记录已存在**：系统部署完成后默认有初始记录，新增前确认是否已有所需模板 · severity=MEDIUM。
- **与 SET N40APIVER 联动**：NB-IOT/LTEM 扩展属性需 SET N40APIVER 开对应 FeatureList（NBIOTCHG-1&LTEMCHG-1）才真正生效，单开 NBEXTENDATTR 不足 · severity=HIGH。
- **适用 NF**：PGW-C、SMF（融合计费 N40 接口） · severity=MEDIUM。
