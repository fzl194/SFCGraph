# 配置N40消息属性模板（ADD N40MSGTEMP）

> 命令静态知识见 [ADD N40MSGTEMP](command/UNC/20.15.2/ADD-N40MSGTEMP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD N40MSGTEMP 建一个"模板名 + 7 个字段开关"的模板，控制 N40 消息里携带哪些信元。模板建好后由 N40 接口相关流程引用。系统部署后已有初始记录（TEMPLATENAME=`global`，各字段默认值见下）。配置空间有 2 个维度：**模板名**（必选，命名）、**字段开关**（可选，按信元控制是否携带）。

### 配置维度 1：模板名（参数 TEMPLATENAME，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 模板名 | `TEMPLATENAME` | 字符串 1~63，不区分大小写 | 无 | 命名本模板，供后续 N40 接口引用 | 必填；系统初始已有 `global` 记录 |

### 配置维度 2：字段开关（参数 SECRATUSAGE / NBEXTENDATTR / QFIDOWNLINK / HOMECHGID / HWQBCIDCT / APPSVCPROVID / RCP，可选）

| 参数 | 信元 | 默认 | 取值 | 典型用途 |
|---|---|---|---|---|
| `SECRATUSAGE` | RANSecondaryRATUsageReport | ENABLE | ENABLE/DISABLE | 是否携带辅助 RAT 用量报告；DISABLE 时 ADD CCT 中 SECRUTHRESHOLD 功能不生效 |
| `NBEXTENDATTR` | NB 扩展属性（APN RateControl/Serving PLMN RateControl/CP only/PtP 隧道） | ENABLE | ENABLE/DISABLE | NB-IOT/LTEM 接入时携带扩展属性；仅在 SET N40APIVER 使能 NBIOTCHG/LTEMCHG 时生效 |
| `QFIDOWNLINK` | MultipleQFIcontainer 中的 DownLinkVolume | ENABLE | ENABLE/DISABLE | 是否携带 QFI 容器下行流量 |
| `HOMECHGID` | HomeProvidedChargingID | ENABLE | ENABLE/DISABLE | 是否携带归属网络计费 ID |
| `HWQBCIDCT` | huaweiQBCIndication | DISABLE | ENABLE/DISABLE | QBC 计费时是否显式指示；开启前须确认对端 CHF 支持 |
| `APPSVCPROVID` | applicationServiceProviderIdentity | DISABLE | ENABLE/DISABLE | 是否携带应用服务商标识 |
| `RCP` | roamingChargingProfile（在 RoamingQBCInformation 信元内） | ENABLE | ENABLE/DISABLE | RCP 不协商场景是否携带；仅在 SET HVNEGRCPSW 的 VSMFSW/HSMFSW=DISABLE 时生效 |

> 默认值覆盖大多数场景：SECRATUSAGE/NBEXTENDATTR/QFIDOWNLINK/HOMECHGID/RCP 默认 ENABLE，HWQBCIDCT/APPSVCPROVID 默认 DISABLE。需要偏离默认时才显式配置。

**典型脚本（NB-IOT/LTEM 场景，确保 NB 扩展属性）**
```
ADD N40MSGTEMP: TEMPLATENAME="n40attr", NBEXTENDATTR=ENABLE;
```

**配置链路（本命令位置）**
```
SET N40APIVER        # 先设 N40 API 版本与 FeatureList（如 NBIOTCHG-1&LTEMCHG-1）
ADD N40MSGTEMP       ← 本命令    # 建模板控制字段开关
```

**步骤位置**：在所属 feature 编排中通常位于"配置 N40 消息属性模板"步骤——紧随 SET N40APIVER 设 API 版本之后。

## 决策点

### 决策点：是否新建模板还是复用初始记录（DP 0-02071）

问：系统部署后已有名为 `global` 的初始记录，本场景是新建独立模板还是直接复用/调整初始记录？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 新建独立模板 | `TEMPLATENAME=<自定义名>` | 需要与默认 `global` 隔离的多模板策略 |
| 复用初始记录（用 MOD 调整） | 省略 ADD，改用 `MOD N40MSGTEMP` 调 `global` | 单模板策略，仅需调整少量字段 |

> 系统部署完成后默认存在初始记录；新增前确认是否已有所需模板，避免重复。

### 决策点：字段开关是否需要显式偏离默认（DP 0-02072）

问：哪些字段开关需要显式改默认值？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 全部保持默认 | 省略各开关参数 | 常规场景（多数字段默认 ENABLE） |
| NB-IOT/LTEM 扩展 | `NBEXTENDATTR=ENABLE`（且 SET N40APIVER 开 NBIOTCHG/LTEMCHG） | NB-IOT/LTEM 计费 |
| 华为 QBC 指示 | `HWQBCIDCT=ENABLE` | 需 CHF 显式识别 QBC 计费消息 |
| 应用服务商标识 | `APPSVCPROVID=ENABLE` | 需携带应用服务商标识 |
| 关闭某信元 | 对应字段置 `DISABLE` | 按对端 CHF 能力裁剪字段 |

## 约束

- rule-0-02176 · severity=critical · NBEXTENDATTR=ENABLE 时必须在 SET N40APIVER 中使能 NBIOTCHG-1 或 LTEMCHG-1，单开 NBEXTENDATTR 不足 — 后果：NB 扩展属性实际不携带
- rule-0-02177 · severity=high · HWQBCIDCT=ENABLE 前须确认对端 CHF 支持 huaweiQBCIndication 信元 — 后果：CHF 解析失败
- rule-0-02178 · severity=warning · 最多可输入 101 条记录；系统部署后已有初始记录，新增前确认是否已有所需模板 — 后果：超规格或重复新增失败
- rule-0-02179 · severity=info · SECRATUSAGE=DISABLE 时 ADD CCT 中 SECRUTHRESHOLD 功能不生效；RCP 仅在 SET HVNEGRCPSW 的 VSMFSW/HSMFSW=DISABLE 场景生效 — 后果：联动功能静默失效
- rule-0-02180 · severity=info · 该命令执行后立即生效；适用 NF：PGW-C、SMF（融合计费 N40 接口） — 后果：无延迟
