# 设置失败旁路处理（SET FHBYPASS）（SET FHBYPASS）

> 命令静态知识见 [SET FHBYPASS](command/UNC/20.15.2/SET-FHBYPASS.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET FHBYPASS 是**高危紧急故障恢复命令**，仅用于 PCRF/OCS/Radius 无响应或异常返回码时的一键放通。全局唯一一条记录（最大记录数=1），系统初始记录所有开关均为 `DISABLE`（HOLDINGTIME=`10`、RANGETIME=`30`）。按受影响的接口域分四套独立开关：**在线计费 Gy（ONLCHARGE）**、**策略与计费控制 Gx/PCC（PCC）**、**Radius 鉴权（RDSAUTH）**、**Radius 计费（RDSACCT）**，可按需单独使能。

每套开关的主参数控制是否旁路放通；conditional 参数（GYERRRC/GXERRRC/CCFHOFFLINE）仅在前置主参数使能时生效，控制异常结果码场景与话单标识。本命令的核心配置空间有 2 个维度：**接口域放通开关集合**（按故障的周边网元选 Gy/Gx/Radius 中哪些域放通）、**异常结果码与话单标识**（conditional，仅在对应主开关使能时考虑）。

### 配置维度 1：接口域放通开关集合（参数 ONLCHARGE/PCC/RDSAUTH/RDSACCT）

每个接口域独立决定是否对故障网元放通。证据中三套范式分别对应：Gy（WSFD-109001 DCC）、Gx/PCC（WSFD-109101）、Radius（WSFD-011306）。

| 主参数 | 控制接口域 | 故障网元 | 使能后行为 | 系统初始值 |
|---|---|---|---|---|
| `ONLCHARGE` | 在线计费 Gy | OCS | 用户转离线计费（不旁路则按 DCC 模板配置处理） | `DISABLE` |
| `PCC` | 策略与计费控制 Gx | PCRF | 受影响 PCC 用户回滚为本地 PCC 用户（不旁路则按 [SET PCCFAILACTION] 处理） | `DISABLE` |
| `RDSAUTH` | Radius 鉴权 | 鉴权 AAA | 鉴权失败仍允许用户继续激活 | `DISABLE` |
| `RDSACCT` | Radius 计费 | 计费 AAA | accounting start 超时无响应仍允许用户继续激活 | `DISABLE` |

> 四套开关相互独立，可单使能或多使能；使能后本命令优先级高于各域的细粒度失败处理命令（[SET PCCFAILACTION]、ADD RESULTCODECTRL、SET APNAUTHATTR、SET APNRDSACCTCTRL、DCC 模板下 SET OCSDOWNACTION 等），将覆盖其策略。

### 配置维度 2：异常结果码与话单标识（参数 GYERRRC/GXERRRC/CCFHOFFLINE + HOLDINGTIME/RANGETIME）

| 参数 | 作用 | 前提条件 | 取值 | 系统初始值 |
|---|---|---|---|---|
| `GYERRRC` | Gy 接口命令层异常结果码是否也旁路（不旁路则仅超时无响应旁路） | ONLCHARGE=ENABLE | `DISABLE` / `ENABLE` | `DISABLE` |
| `GXERRRC` | Gx 接口命令层异常结果码是否旁路（不旁路则按 ADD RESULTCODECTRL 处理） | PCC=ENABLE | `DISABLE` / `ENABLE` | `DISABLE` |
| `CCFHOFFLINE` | OCS 故障旁路转离线计费时，话单容器是否携带 failureHandlingContinue 字段 | ONLCHARGE=ENABLE | `DISABLE` / `ENABLE` | `DISABLE` |
| `HOLDINGTIME` | 用户保持时长(分)，超出去活；0=永久保持在线 | 至少一个主开关使能才生效 | `0`~`1440` | `10` |
| `RANGETIME` | HoldingTime 超时后的随机调整范围(分) | HOLDINGTIME 非 0 才生效 | `1`~`60` | `30` |

> HOLDINGTIME/RANGETIME 联动：若配置了 HOLDINGTIME 但四个主开关均未使能，HOLDINGTIME 不生效、恢复系统初始；若 HOLDINGTIME=0 则 RANGETIME 不生效。

典型脚本（Gy 在线计费一键放通，OCS 故障转离线计费，话单记 failureHandlingContinue）：
```
SET FHBYPASS: ONLCHARGE=ENABLE, GYERRRC=ENABLE, CCFHOFFLINE=ENABLE;
```
典型脚本（Gx/PCC 域放通，PCRF 故障 PCC 用户回滚为本地 PCC 用户，含 Gx 异常结果码）：
```
SET FHBYPASS: PCC=ENABLE, GXERRRC=ENABLE;
```
典型脚本（Radius 鉴权域放通，鉴权 AAA 故障用户仍可激活）：
```
SET FHBYPASS: RDSAUTH=ENABLE;
```

**步骤位置**：本命令不属于常规配置流程，仅在故障应急时执行（证据：[配置异常场景数据](evidence/UNC/20.15.2/配置异常场景数据_31422947.md)）。恢复后用 `RTR FHBYPASS` 恢复初始配置、`DSP CPPDPCHGINFO` 查询被放通用户归属后定向处理。

## 决策点

### 决策点：各接口域是否使能旁路放通（DP 0-03083）

问：哪些接口域（Gy/Gx/Radius 鉴权/Radius 计费）的周边网元故障，需要对本命令使能放通？每个域独立决定。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| Gy 在线计费放通 | ONLCHARGE=ENABLE（+ GYERRRC/CCFHOFFLINE 可选） | OCS 故障/无响应，紧急维持用户在线（转离线计费） |
| Gx 策略计费放通 | PCC=ENABLE（+ GXERRRC 可选） | PCRF 故障，PCC 用户回滚为本地 PCC 用户 |
| Radius 鉴权放通 | RDSAUTH=ENABLE | 鉴权 AAA 故障，鉴权失败仍允许激活 |
| Radius 计费放通 | RDSACCT=ENABLE | 计费 AAA 故障，accounting start 超时仍允许激活 |
| 全部 DISABLE（恢复） | 四个主开关均 DISABLE | 正常运行，或故障恢复后关闭旁路 |

> 取值由当前故障的周边网元决定。本命令为高危操作，执行前需客户书面认可。

### 决策点：异常结果码与话单标识（DP 0-03084）

问：对应主开关使能后，是否进一步放通异常返回码（非仅无响应），以及话单是否记 failureHandlingContinue？这些是 conditional 参数，仅在对应主开关使能时生效。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 异常结果码也放通 | GYERRRC=ENABLE / GXERRRC=ENABLE | 收到 Gy/Gx 命令层异常结果码也旁路（可能导致计费损失，谨慎） |
| 仅无响应放通 | GYERRRC=DISABLE / GXERRRC=DISABLE | 只处理超时无响应，异常码走原逻辑（DCC 模板/ADD RESULTCODECTRL） |
| 话单记 failureHandlingContinue | CCFHOFFLINE=ENABLE | 计费中心支持该标识，便于后续对账（推荐配合 ONLCHARGE 使能） |
| 话单不记标识 | CCFHOFFLINE=DISABLE | 计费中心不支持该标识 |

> GYERRRC/GXERRRC 使能可能导致计费策略改变、造成计费损失，需根据计费中心能力谨慎决定；CCFHOFFLINE 需根据计费中心能力确认，否则可能导致计费损失。

## 约束

· rule-0-03206 · severity=critical · 高危命令：仅用于紧急故障恢复，执行前需客户书面认可，可能造成计费误差/计费策略改变；本命令优先级最高，将覆盖 [SET PCCFAILACTION]、ADD RESULTCODECTRL、SET APNAUTHATTR(AAANORSPCTRL)、SET APNRDSACCTCTRL(DEACTIVE)、DCC 模板下 SET OCSDOWNACTION/SET CMDLEVDFTBEH/ADD CMDRCACT 等各域细粒度失败处理策略

· rule-0-03207 · severity=high · 全局唯一一条记录（最大记录数=1），重复执行为改写而非新增；执行后立即生效

· rule-0-03208 · severity=high · RDSAUTH 使能时地址分配约束：用户鉴权失败仍可激活，需保证本地地址分配方式不是 Radius 分配、且本地有可用地址池，否则激活失败；若 ADD RDSSVRGRP 的 ACCTTOAUTH=enable，需同时使能 RDSACCT 否则用户无法正常使用业务

· rule-0-03209 · severity=high · ONLCHARGE 使能时话单标识：需根据计费中心能力确认是否使能 CCFHOFFLINE（话单记 failureHandlingContinue），否则可能导致计费损失；PCC 使能时需保证本地计费策略可用否则回滚失败用户激活失败

· rule-0-03210 · severity=warning · HOLDINGTIME/RANGETIME 联动：HOLDINGTIME 取值 0~1440 分钟（0=永久保持），RANGETIME 取值 1~60 分钟；若四个主开关均未使能 HOLDINGTIME 不生效恢复初始，若 HOLDINGTIME=0 则 RANGETIME 不生效
