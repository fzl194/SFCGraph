# SET FHBYPASS 原子任务

> 命令静态知识见 [SET FHBYPASS](command/UNC/20.15.2/SET-FHBYPASS.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET FHBYPASS 是**高危紧急故障恢复命令**，仅用于 PCRF/OCS/Radius 无响应或异常返回码时的一键放通。全局唯一一条记录（最大记录数=1）。按受影响的接口域分三套独立开关：**在线计费（Gy）**、**策略与计费控制（Gx/PCC）**、**Radius（鉴权/计费）**，可按需单独使能。

每套开关的主参数（ONLCHARGE/PCC/RDSAUTH/RDSACCT）控制是否旁路放通；conditional 参数（GYERRRC/GXERRRC/CCFHOFFLINE）仅在主参数使能时生效，控制异常结果码场景与话单标识。

**参数取值样例（按接口域）**

| 接口域 | 主参数 | conditional 参数 | HOLDINGTIME/RANGETIME | 典型行 |
|---|---|---|---|---|
| 在线计费 Gy | `ONLCHARGE=ENABLE` | `GYERRRC=ENABLE`（异常结果码也放通）、`CCFHOFFLINE=ENABLE`（话单记 failureHandlingContinue） | 0~1440 / 1~60 分 | `SET FHBYPASS: ONLCHARGE=ENABLE, GYERRRC=ENABLE;` |
| 策略计费 Gx | `PCC=ENABLE` | `GXERRRC=ENABLE` | 同上 | `SET FHBYPASS: PCC=ENABLE, GXERRRC=ENABLE;` |
| Radius 鉴权 | `RDSAUTH=ENABLE` | — | 同上 | `SET FHBYPASS: RDSAUTH=ENABLE;` |
| Radius 计费 | `RDSACCT=ENABLE` | — | 同上 | `SET FHBYPASS: RDSACCT=ENABLE;` |

**典型行（Gy 在线计费一键放通，含异常结果码）**

```
SET FHBYPASS: ONLCHARGE=ENABLE, GYERRRC=ENABLE;
```

## 决策点

### 各接口域是否使能旁路放通

每个接口域独立决定。证据中三套范式分别对应：Gy（WSFD-109001 DCC）、Gx/PCC（WSFD-109101）、Radius（WSFD-011306）。

| 选项 | 主参数 | 适用 |
|---|---|---|
| 使能放通 | `ENABLE` | PCRF/OCS/Radius 故障，紧急维持业务 |
| 不使能 | `DISABLE` | 正常运行，或已恢复后关闭旁路 |

### 是否放通异常返回码场景

GYERRRC/GXERRRC 是 conditional 参数，仅当对应主参数（ONLCHARGE/PCC）使能时才生效，决定"无响应"与"异常返回码"两类故障是否都放通。

| 选项 | 取值 | 适用 |
|---|---|---|
| 无响应 + 异常返回码都放通 | `ENABLE` | 收到 command/CCA 异常结果码也旁路 |
| 仅无响应放通 | `DISABLE` | 只处理超时无响应，异常码走原逻辑 |

### 故障恢复后对放通用户的处理方式

| 选项 | 方式 | 说明 |
|---|---|---|
| 方式1（全部用户，可能恢复不及时） | `HOLDINGTIME` 控制放通用户后续（永久在线/去激活/转离线） | 针对全部放通用户 |
| 方式2（推荐，精准） | `DSP CPPDPCHGINFO` 查询被放通用户归属后定向处理 | 恢复更及时 |

## 约束

- **高危命令** · severity=CRITICAL · 仅用于紧急故障恢复，执行前需客户书面认可，可能造成计费误差。
- **立即生效** · severity=LOW。
- **最大记录数=1**（全局唯一） · severity=HIGH · 重复执行为改写。
- **优先级最高**：本命令优先级高于 `SET PCCFAILACTION`、`ADD RESULTCODECTRL`（PCC 域）、`SET APNAUTHATTR` 的 AAANORSPCTRL、`SET APNRDSACCTCTRL` 的 DEACTIVE（Radius 域）、DCC 模板下 `SET OCSDOWNACTION`/`SET CMDLEVDFTBEH`/`ADD CMDRCACT`（Gy 域） · severity=HIGH · 配置后将覆盖上述命令的策略。
- **影响用户激活状态/计费方式/控制策略** · severity=HIGH · 须确认客户书面认可后方可使用。
- **RDSAUTH 使能时地址分配约束**：用户鉴权失败仍可激活，需保证本地地址分配方式不是 Radius 分配，且本地有可用地址池，否则激活失败 · severity=HIGH。
- **ONLCHARGE 使能时话单标识**：需根据计费中心能力确认是否使能 CCFHOFFLINE（话单记 failureHandlingContinue），否则可能导致计费损失 · severity=HIGH。
- **HOLDINGTIME 范围**：0~1440 分钟；RANGETIME 1~60 分钟 · severity=MEDIUM。
