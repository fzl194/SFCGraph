# RMV TSTPCFBINDING 原子任务
> 命令静态知识见 [RMV TSTPCFBINDING](command/UNC/20.15.2/RMV-TSTPCFBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

删除对象是 `TSTPCFBINDING`（拨测用户-PCF 绑定关系）记录。三参数全部 optional，可单独或组合作为查询条件；**不输入任何参数 = 删除所有配置**。

## 配置方法
按 APN / IMSI / PCF 实例标识单独或组合定位删除。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | APN 名称（local_planned，optional） | `apn-test` |
| IMSI | IMSI（local_planned，optional） | `460000123456789` |
| PCFINSTANCEID | PCF 实例标识（global_planned，optional） | `testPcfInstanceID` |

典型脚本（调测 PCC 基本功能收尾清理，三参数全指定精确删一条）：
```
RMV TSTPCFBINDING: APN="apn-test",IMSI="460000123456789",PCFINSTANCEID="testPcfInstanceID";
```

全删（不输入任何参数，高危）：
```
RMV TSTPCFBINDING:;
```

## 决策点（若有）

### 删除范围（精确组合 vs 全删）
- 问什么：删一条拨测绑定，还是删全部拨测绑定。
- 精确删：给 `APN + IMSI + PCFINSTANCEID` 任意组合锁定记录。
- 全删：**不输入任何参数 = 删除所有配置**——调测收尾清场时可用，但生产慎用。

### 查询条件组合策略
- 三参数无 required，支持以 APN、IMSI、PCFINSTANCEID **及其组合**作为查询条件。
- 调测 PCC 基本功能（WSFD-109101）场景的标准清理范式：三参数全指定，精确删掉本批次拨测创建的那条绑定。

## 约束（若有）
- 生效范围：本命令**只对新激活用户生效**，已激活用户不受影响——清理后仍有在用拨测会话时需先去活。`severity=medium`
- 全删风险：不输入任何参数删除所有配置，生产慎用。`severity=medium`
- APN 格式：长度 1～63、大小写不敏感。`severity=low`
- IMSI 格式：长度 6～15、每字符为十进制数字。`severity=low`
- PCFINSTANCEID 格式：长度 1～50、构成字符限字母（A～Z/a～z）、数字（0～9）、中划线 `-`、下划线 `_`。`severity=low`
