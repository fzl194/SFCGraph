# ADD PCCTEMPLATE 原子任务
> 命令静态知识见 [ADD PCCTEMPLATE](command/UNC/20.15.2/ADD-PCCTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
PCC 模板按**功能场景**组合参数：本地 PCC / 动态 PCC 开关类、PCRF/PCF 故障处理类、Gx over DRA 重选类、本端主机名选择类、5G PCF 容灾类。模板配好后通过 SET APNPCCFUNC 绑定到 APN/DNN，**仅对新激活用户生效**。IMS 语音建议单独用一个模板。

| 场景 | 必备参数 | 典型脚本 |
|---|---|---|
| 本地 PCC 开关 | `PCCTEMPNAME` + `SELPEERFAILACT` + `INITIALFAILACT=FORBIDDEN` + `UPDATEFAILACT` | `ADD PCCTEMPLATE:PCCTEMPNAME="pcctemplate",INITIALFAILACT=FORBIDDEN,UPDATEFAILACT=INHERIT;` |
| 异常处理（PCRF 故障动作+回滚） | `PCCTEMPNAME` + `SELPEERFAILACT=DEFAULT` + `INITIALFAILACT=ROLLBACK` + `INITFAILRDSPCC=LOCAL_PCC` + `UPDATEFAILACT=ROLLBACK` + `UPDFAILLOCPCC=LOCAL_PCC` | `ADD PCCTEMPLATE:PCCTEMPNAME="templatetest",SELPEERFAILACT=DEFAULT,INITIALFAILACT=ROLLBACK,INITFAILRDSPCC=LOCAL_PCC,UPDATEFAILACT=ROLLBACK,UPDFAILLOCPCC=LOCAL_PCC;` |
| 本端主机名选择（基于 UPF 组） | `PCCTEMPNAME` + `LOCSLCTMODE=UPFGRP` + `UPFGLOCGBNDGNAME`（已配绑定组） | `ADD PCCTEMPLATE:PCCTEMPNAME="pcctemplate_01",LOCSLCTMODE=UPFGRP,UPFGLOCGBNDGNAME="test";` |
| Gx over DRA（PCRF 重选） | `PCCTEMPNAME` + `ORGHOSTCCAI=ENABLE` + `ORGHOSTCCAU=ENABLE` + `ORGHOSTRAR=ENABLE` | `ADD PCCTEMPLATE:PCCTEMPNAME="test_template",ORGHOSTCCAI=ENABLE,ORGHOSTCCAU=ENABLE,ORGHOSTRAR=ENABLE;` |
| 5G PCF 容灾（N7 Failover+Holding Time） | `PCCTEMPNAME` + `SELPEERFAILACT=ROLLBACK` + `INITIALFAILACT=ROLLBACK` + `UPDATEFAILACT=ROLLBACK` + `UPDFAILLOCPCC=INHERIT_PCC` + `HOLDINGTIME` + `ADJUSTRANGE` + `N7FAILOVERSW=ENABLE` | `ADD PCCTEMPLATE:PCCTEMPNAME="pcctemplate",SELPEERFAILACT=ROLLBACK,INITIALFAILACT=ROLLBACK,UPDATEFAILACT=ROLLBACK,UPDFAILLOCPCC=INHERIT_PCC,HOLDINGTIME=30,ADJUSTRANGE=10,N7FAILOVERSW=ENABLE;` |
| 5G PCF 容灾（Notify 重选+负荷分担） | 上一行 + `NTFRSRURI=ENABLE` + `PCFLBPARA=GROUPID/PRIORITY` | `ADD PCCTEMPLATE:PCCTEMPNAME="pcctemplate",SELPEERFAILACT=DEFAULT,INITIALFAILACT=FORBIDDEN,UPDATEFAILACT=INHERIT,NTFRSRURI=ENABLE,N7FAILOVERSW=ENABLE,PCFLBPARA=GROUPID;` |

## 决策点

### PCC 模板用途定位
问：本模板主要解决哪个问题（决定参数组合）？
- **本地 PCC**：`INITIALFAILACT=FORBIDDEN`（禁止回滚为动态 PCC），适合不依赖 PCRF 的本地策略场景。
- **动态 PCC + PCRF 故障回滚**：`SELPEERFAILACT/INITIALFAILACT/UPDATEFAILACT=ROLLBACK` + 回滚用户类型（`INITFAILRDSPCC/UPDFAILLOCPCC=LOCAL_PCC`），PCRF 故障时降级为本地 PCC。
- **PCRF 重选（Gx over DRA）**：`ORGHOSTCCAI/CCAU/RAR=ENABLE`，根据 CCA-I/CCA-U/RAR 的 Origin-Host AVP 触发 PCRF 重选。
- **5G PCF 容灾（N7）**：`N7FAILOVERSW` + `NTFRSRURI` + `HOLDINGTIME` + `PCFLBPARA`，PCF 主备容灾/快速回滚。

### Update 流程回滚后是否继续用 PCF 策略
问（仅 5G 语音 Bypass）：`UPDFAILLOCPCC` 取 `LOCAL_PCC` 还是 `INHERIT_PCC`？
- `LOCAL_PCC`：通知 UPF 删动态 PCC 策略、装本地静态 PCC 策略（用于语音专有 QoS Flow/承载）。
- `INHERIT_PCC`：继续使用激活时安装的 PCF 下发策略（语音 Bypass 推荐值）。

### PCF 负荷分担参数
问（仅 5G）：`PCFLBPARA` 取 `GROUPID` 还是 `PRIORITY`？
- `GROUPID`：SMF 要求主备 PCF 的 GroupID **相同**。
- `PRIORITY`：SMF 要求主备 PCF 的 PRIORITY **不同**。

### Holding Time 是否启用 + 取值
问：PCF/PCRF 故障回滚后用户在线保持时长？
- 启用：配 `HOLDINGTIME`（0~1440 或 65535=不限制）+ 可选 `ADJUSTRANGE`（0~60 分钟随机延迟，避免回滚风暴）。
- 软参 DWORD519 Bit9 控制激活流程 PCF 失败回滚时 Holding Time 是否生效。

## 约束
- PCCTEMPNAME 长度 1~63，不支持空格，不区分大小写，**不允许命名为 "global"**。`severity=high`
- LOCSLCTMODE=UPFGRP 时 UPFGLOCGBNDGNAME 必填（conditional）。`severity=high`
- LOCALHOSTNAME 在 LOCSLCTMODE 启用本端主机名时必填（conditional）。`severity=medium`
- 回滚用户类型参数（SELPEERLOCPCC/SELPEERRDSPCC/INITFAILLOCPCC/INITFAILRDSPCC/UPDFAILLOCPCC）为 conditional，仅对应故障动作触发回滚时生效。`severity=medium`
- PCFLBPARA=GROUPID 时主备 PCF GroupID 须相同；=PRIORITY 时须不同。`severity=high`
- NTFRSRURI/N7FAILOVERSW 支持 APN 粒度（本命令）与全局粒度（SET PCCFUNC / SET N7RCVATTRCTRL），按 PCF 实际能力规划。`severity=medium`
- 最大记录数 100。`severity=medium`
- 仅对新激活用户生效，已激活用户不受影响。`severity=high`
- 执行后立即生效（对新激活用户）。`severity=low`
