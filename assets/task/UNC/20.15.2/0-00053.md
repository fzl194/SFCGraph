# 增加PCC模板（ADD PCCTEMPLATE）

> 命令静态知识见 [ADD PCCTEMPLATE](command/UNC/20.15.2/ADD-PCCTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

PCC 模板按**功能场景**组合参数：本地 PCC / 动态 PCC 开关类、PCRF/PCF 故障处理类、Gx over DRA 重选类、本端主机名选择类、5G PCF 容灾类。模板配好后通过 `SET APNPCCFUNC` 绑定到 APN/DNN，**仅对新激活用户生效**。配置空间有 5 个维度：**模板名**（必选）、**故障处理动作**（可选，选择失败/Initial/Update 三阶段）、**回滚用户类型**（条件可选，随故障动作 ROLLBACK 出现）、**PCRF/PCF 重选与容灾**（可选，Origin-Host/Notify/N7 Failover）、**本端主机名选择与保持时长**（可选）。

### 配置维度 1：模板标识（参数 PCCTEMPNAME，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 模板名 | `PCCTEMPNAME` | 本端规划，1~63 字符，不支持空格、不区分大小写，**不允许命名为 "global"** | 无 | PCC 模板唯一标识，被 SET APNPCCFUNC 引用 | 所有记录必填 |

### 配置维度 2：故障处理动作（可选，三阶段独立配置）

| 阶段 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 选择 PCF/PCRF 失败 | `SELPEERFAILACT` | DEFAULT（回滚本地 PCC）/ ROLLBACK（回滚，有缺省则用缺省 PCRF 组）/ INHERIT（继承全局）/ FORBIDDEN（激活失败） | INHERIT | 号段匹配选 PCF/PCRF 失败时的动作 | 动态 PCC 选路失败兜底 |
| Initial 流程故障 | `INITIALFAILACT` | FORBIDDEN（激活失败）/ ROLLBACK（回滚）/ INHERIT（继承全局） | INHERIT | 激活过程中 Gx/N7 链路故障、流控丢消息或响应超时动作 | 本地 PCC（FORBIDDEN）/ 动态 PCC 回滚（ROLLBACK） |
| Update 流程故障 | `UPDATEFAILACT` | ROLLBACK（回滚）/ TERMINATE（去活）/ CONTINUE（继续交互）/ INHERIT（继承全局） | INHERIT | 更新过程中 Gx/N7 链路故障动作 | 语音 Bypass（ROLLBACK）/ 普通动态 PCC（CONTINUE） |

### 配置维度 3：回滚用户类型（条件可选，随 ROLLBACK 出现）

| 触发阶段 | 参数 | 取值 | 默认 | 作用 |
|---|---|---|---|---|
| 选择失败回滚 | `SELPEERLOCPCC` / `SELPEERRDSPCC` | LOCAL_PCC | LOCAL_PCC | 回滚为本地 PCC 用户（Local UserProfile / RADIUS UserProfile） |
| Initial 回滚 | `INITFAILLOCPCC` / `INITFAILRDSPCC` | LOCAL_PCC | LOCAL_PCC | 同上 |
| Update 回滚 | `UPDFAILLOCPCC` | LOCAL_PCC（删动态策略装本地静态）/ INHERIT_PCC（继续用 PCF 策略） | LOCAL_PCC | 5G 语音 Bypass 关键分叉 |

### 配置维度 4：PCRF/PCF 重选与容灾（可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| Gx over DRA 重选 | `ORGHOSTCCAI` / `ORGHOSTCCAU` / `ORGHOSTRAR` | DISABLE / ENABLE | DISABLE | 按 CCA-I/CCA-U/RAR 的 Origin-Host AVP 触发 PCRF 重选 | Gx over DRA（4G） |
| 5G PCF Notify 重选 | `NTFRSRURI` | INHERIT / DISABLE / ENABLE | INHERIT | 按 UpdateNotify 携带的 ResourceURI 触发 PCF 重选 | 5G PCF 容灾 |
| 5G N7 Failover | `N7FAILOVERSW` | DISABLE / ENABLE / INHERIT | INHERIT | 主 PCF 交互失败时切换到备 PCF | 5G PCF 主备容灾 |
| PCF 负荷分担 | `PCFLBPARA` | INHERIT / GROUPID / PRIORITY | INHERIT | 主备 PCF 负荷分担依据 | 5G PCF 负荷分担 |

### 配置维度 5：本端主机名选择与保持时长（可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 本端主机名选择 | `LOCSLCTMODE` | LOADSHARING / SPECIFIC / INHERIT / UPFGRP | INHERIT | PCC 本端主机名选择模式 | 基于 UPF 组选择（5G） |
| 本端主机名（条件必选） | `LOCALHOSTNAME` | 1~127 字符，引用 `ADD DIAMLOCINFO` | 无 | LOCSLCTMODE=SPECIFIC 时必选 | 指定链路交互 |
| UPF 组绑定（条件必选） | `UPFGLOCGBNDGNAME` | 1~63 字符，引用 `ADD UPFGLOCGBNDGRP` | 无 | LOCSLCTMODE=UPFGRP 时必选 | 按 UPF Group 粒度选本端主机名 |
| 回滚保持时长 | `HOLDINGTIME` | 0~1440 或 65535（分钟） | 65535（继承全局 SET PCCTIMER） | 回滚后用户在线保持时长 | 5G PCF 容灾快速回滚 |
| 随机延迟范围 | `ADJUSTRANGE` | 0~60（分钟） | 0 | HOLDINGTIME 超时后增加随机延迟，避免回滚风暴 | 大批量回滚场景 |

典型脚本（本地 PCC：禁止回滚为动态 PCC）：
```
ADD PCCTEMPLATE:PCCTEMPNAME="pcctemplate_local",INITIALFAILACT=FORBIDDEN,UPDATEFAILACT=INHERIT;
```

典型脚本（PCRF 故障回滚为本地 PCC）：
```
ADD PCCTEMPLATE:PCCTEMPNAME="templatetest",SELPEERFAILACT=DEFAULT,INITIALFAILACT=ROLLBACK,INITFAILRDSPCC=LOCAL_PCC,UPDATEFAILACT=ROLLBACK,UPDFAILLOCPCC=LOCAL_PCC;
```

典型脚本（Gx over DRA：按 Origin-Host 重选 PCRF）：
```
ADD PCCTEMPLATE:PCCTEMPNAME="test_template",ORGHOSTCCAI=ENABLE,ORGHOSTCCAU=ENABLE,ORGHOSTRAR=ENABLE;
```

典型脚本（5G PCF 容灾：N7 Failover + Holding Time + 继续用 PCF 策略）：
```
ADD PCCTEMPLATE:PCCTEMPNAME="pcctemplate_5g",SELPEERFAILACT=ROLLBACK,INITIALFAILACT=ROLLBACK,UPDATEFAILACT=ROLLBACK,UPDFAILLOCPCC=INHERIT_PCC,HOLDINGTIME=30,ADJUSTRANGE=10,N7FAILOVERSW=ENABLE;
```

**步骤位置**：建 Diameter 本端信息（`ADD DIAMLOCINFO`）/ 建 UPF 组绑定（`ADD UPFGLOCGBNDGRP`）/ 建 PCRF 组（`ADD PCRFGROUP`，FAILOVERSW 总开关）→ 本任务（配 PCC 模板）→ `SET APNPCCFUNC`（模板绑定到 APN/DNN）。

## 决策点

### 决策点：PCC 模板用途定位（DP 0-02085）

问：本模板主要解决哪个问题（决定参数组合）？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 本地 PCC | `INITIALFAILACT=FORBIDDEN`（禁止回滚为动态 PCC） | 不依赖 PCRF 的本地策略场景 |
| 动态 PCC + PCRF 故障回滚 | `SELPEERFAILACT/INITIALFAILACT/UPDATEFAILACT=ROLLBACK` + 回滚用户类型（`*RDSPCC/*LOCPCC=LOCAL_PCC`） | PCRF 故障时降级为本地 PCC |
| PCRF 重选（Gx over DRA） | `ORGHOSTCCAI/CCAU/RAR=ENABLE` | 按 CCA-I/CCA-U/RAR 的 Origin-Host 触发 PCRF 重选（4G） |
| 5G PCF 容灾（N7） | `N7FAILOVERSW=ENABLE` + `NTFRSRURI=ENABLE` + `HOLDINGTIME` + `PCFLBPARA` | PCF 主备容灾/快速回滚（5G） |
| 本端主机名选择 | `LOCSLCTMODE=UPFGRP/SPECIFIC/LOADSHARING` + 条件必选参数 | 基于 UPF 组或指定链路选本端主机名 |

### 决策点：Update 回滚后是否继续用 PCF 策略（DP 0-02086）

问（仅 5G 语音 Bypass）：`UPDFAILLOCPCC` 取 LOCAL_PCC 还是 INHERIT_PCC？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| LOCAL_PCC | `UPDFAILLOCPCC=LOCAL_PCC` | 通知 UPF 删动态 PCC 策略、装本地静态 PCC 策略（用于语音专有 QoS Flow/承载） |
| INHERIT_PCC | `UPDFAILLOCPCC=INHERIT_PCC` | 继续使用激活时安装的 PCF 下发策略（语音 Bypass 推荐值） |

> PCFLBPARA=GROUPID 时主备 PCF GroupID 须相同；=PRIORITY 时须不同。HOLDINGTIME=0 表示回滚后保持在线直到用户主动下线，=65535 表示继承全局 SET PCCTIMER。

## 约束

- **模板名约束**（critical，rule-0-02211）：模板名约束：PCCTEMPNAME 长度 1~63、不支持空格、不区分大小写，**不允许命名为 "global"**（系统保留） — 后果：命名冲突导致配置失败或与全局模板歧义
- **条件必选**（warning，rule-0-02212）：条件必选：LOCSLCTMODE=SPECIFIC 时 LOCALHOSTNAME 必填；LOCSLCTMODE=UPFGRP 时 UPFGLOCGBNDGNAME 必填 — 后果：本端主机名选择模式无法生效
- **PCF 负荷分担一致性**（warning，rule-0-02213）：PCF 负荷分担一致性：PCFLBPARA=GROUPID 时主备 PCF GroupID 须相同；=PRIORITY 时须不同 — 后果：主备 PCF 选择歧义，Failover 失败
- **生效时机**（warning，rule-0-02214）：生效时机：仅对新激活用户生效，已激活用户不受影响（需重激活或等待自然流转）；最大记录数 100；适用 NF：PGW-C、SMF — 后果：模板变更不回溯存量用户
- **APN 粒度与全局粒度互补**（info，rule-0-02215）：APN 粒度与全局粒度互补：NTFRSRURI/N7FAILOVERSW 支持 APN 粒度（本命令）与全局粒度（SET PCCFUNC / SET N7RCVATTRCTRL），按 PCF 实际能力规划 — 后果：双层配置需保持一致，否则继承歧义
