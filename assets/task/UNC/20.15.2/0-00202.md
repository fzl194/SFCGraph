# SET SMCOMMFUNC 原子任务

> 命令静态知识见 [SET SMCOMMFUNC](command/UNC/20.15.2/SET-SMCOMMFUNC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET SMCOMMFUNC 是一组"通用会话管理扩展功能"开关的总闸，共 12 个开关参数。每个特性按需打开对应那一个开关；不输入的参数保持系统当前配置不变。实际激活脚本几乎总是单参数行——只翻待开特性需要的那一个开关。本命令在不同特性里复用，区别仅在"打开哪个开关"。

**开关→特性映射（实际开哪个参数取决于特性）**

| 开关参数 | 含义 | 典型取值 | 触发特性 |
|---|---|---|---|
| PCRFQRYSW | 查询 PCRF 地址开关 | ENABLE | QoS 能力开放（WSFD-109101） |
| RTBEHINDMSSW | 手机后路由功能开关 | ENABLE | 支持 Routing Behind MS（WSFD-205101） |
| GAGYCHGSW | GaGy 计费功能开关 | ENABLE/DISABLE | Ga/Gy 计费（默认使能，谨慎修改） |
| RESTFULQRYSW | Restful 查询 PCF/PCRF 地址开关 | ENABLE | Restful 接口查询 |
| LOCREPORT | 位置上报 | NotSupport/支持值 | 位置上报 |
| MULTIDNNSW | 2B2C 漫游双 DNN 开关 | ENABLE/DISABLE | 双 DNN 漫游 |
| UDMDEREGINSID | 携带实例 ID 向 UDM 去注册 | DISABLE/ENABLE | UDM 去注册 |
| PGWRNSSWITCH | PGW 重定向功能开关 | DISABLE/ENABLE | PGW 重定向 |
| QOSANASW | QoS 质差分析开关 | NotSupport/支持值 | QoS 质差分析 |
| QOSEXPSW | 体验感知信息采集订阅开关 | NotSupport/支持值 | 体验感知采集 |
| LOCATIONCHSW | 位置变更订阅开关 | NotSupport/支持值 | 位置变更订阅 |
| EXPOSURELOCRPT | 能力开放位置上报 | NotSupport/支持值 | 能力开放位置上报 |

**典型行（QoS 能力开放场景）**

```
SET SMCOMMFUNC: PCRFQRYSW=ENABLE;
```

**典型行（Routing Behind MS 场景）**

```
SET SMCOMMFUNC: RTBEHINDMSSW=ENABLE;
```

## 决策点

### 开哪个开关（特性驱动）

本命令在不同特性里复用，参数绑定完全由"当前要开哪个特性"决定。同一原子任务里通常只开一个开关；多特性叠加时各自独立调用或单次调用多个开关均可。

| 待开特性 | 应置 ENABLE 的开关 |
|---|---|
| QoS 能力开放（WSFD-109101） | PCRFQRYSW |
| 支持 Routing Behind MS（WSFD-205101） | RTBEHINDMSSW |
| Ga/Gy 计费 | GAGYCHGSW（默认已使能） |

### PCRFQRYSW 是否依赖 RESTFULQRYSW

PCRFQRYSW 仅在 RESTFULQRYSW 开启时生效。若需支持通过 Restful 查询"非异网漫游/非国漫"场景下 PCRF 的地址，必须先开 RESTFULQRYSW，再开 PCRFQRYSW。

| 选项 | 取值 | 前置 |
|---|---|---|
| 仅本地查询 PCRF | PCRFQRYSW=ENABLE | 无 |
| Restful 查询 PCRF | RESTFULQRYSW=ENABLE 然后 PCRFQRYSW=ENABLE | RESTFULQRYSW 先开 |

## 约束

- **立即生效** · severity=LOW。
- **GAGYCHGSW 改成不使能会导致 GaGy 计费功能不可用** · severity=HIGH · 谨慎修改该参数取值。
- **PCRFQRYSW 仅在 RESTFULQRYSW 开启时生效** · severity=HIGH · 顺序约束：先 RESTFULQRYSW 后 PCRFQRYSW。
- **未输入的参数保持系统当前配置不变** · severity=MEDIUM · 非覆盖语义，按需显式传入。
- **存在系统初始记录** · severity=LOW · 部署后已有初始值，覆盖前用 LST SMCOMMFUNC 查当前状态。
- **Routing Behind MS 后续步骤**：开 RTBEHINDMSSW 后通常需 ADD ROUTINGBEHINDMS 配置本地后路由信息 · severity=MEDIUM。
