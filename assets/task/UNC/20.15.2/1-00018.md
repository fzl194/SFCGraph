# 配置 BWM 本地规则与用户模板绑定

> BWM 特性的本地规则承载层——本地建 BWM 规则（POLICYTYPE=BWM，无 POLICYNAME）+ 用户模板（UPTYPE=PCC）+ 绑定关系 + 用户模板组挂接到 APN/DNN。承接 License 前置（[SET LICENSESWITCH](task/UNC/20.15.2/0-00180.md) 开 LKV3TCBSA01）与 PCC 基本功能（[2-00005](task/UNC/20.15.2/2-00005.md)）。被引用于 [2-00010 BWM](task/UNC/20.15.2/2-00010.md)。BWM 域专属（POLICYTYPE=BWM 是 BWM 独有变种；RULE/USERPROFILE/RULEBINDING/USRPROFGROUP/UPBINDUPG/APNUSRPROFG 命令跨 PCC 族共享，但 BWM 场景参数集不同，见场景差异）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 配本地 BWM 规则 | `ADD RULE` | RULENAME（三网元一致）；POLICYTYPE=BWM（固定）；PRIORITY=10（值越小优先级越高）；无 POLICYNAME（BWM 规则不绑 PCCPOLICYGRP/QOSPROP） |
| 配用户模板 | `ADD USERPROFILE` | USERPROFILENAME（全网规划）；UPTYPE=PCC（固定） |
| 绑规则到用户模板 | `ADD RULEBINDING` | USERPROFILENAME（引用上一步）；RULENAME（引用第一步）；POLICYTYPE=BWM（activation 演示带此参数，可只按 RULENAME 绑同名的所有策略类型规则） |
| 配用户模板组 | `ADD USRPROFGROUP` | USERPROFGNAME（全网规划） |
| 绑用户模板到模板组 | `ADD UPBINDUPG` | USERPROFGNAME；UPBINDTYPE=DEFAULT；USERPROFILENAME |
| 模板组挂接到 DNN | `ADD APNUSRPROFG` | APN（DNN 名，全网规划）；USERPROFGNAME |

**典型脚本（activation 任务示例：指定 P2P 业务带宽控制规则 rule_test，优先级 10）**：
```
//配置带宽控制规则。
ADD RULE: RULENAME="rule_test", POLICYTYPE=BWM, PRIORITY=10;
//配置业务策略组合。
ADD USERPROFILE:USERPROFILENAME="up_test",UPTYPE=PCC;
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="rule_test";
ADD USRPROFGROUP:USERPROFGNAME="upg_test";
ADD UPBINDUPG:USERPROFGNAME="upg_test",UPBINDTYPE=DEFAULT,USERPROFILENAME="up_test";
ADD APNUSRPROFG:APN="huawei.com",USERPROFGNAME="upg_test";
```

**步骤位置**：在 License 前置（[SET LICENSESWITCH](task/UNC/20.15.2/0-00180.md) 开 LKV3TCBSA01）+ PCC 基本功能（[2-00005](task/UNC/20.15.2/2-00005.md)）之后；本 compound 完成本地 BWM 规则承载并挂接到 DNN，之后由 PCRF/PCF 下发预定义规则或本地 PCC 静态规则经 N4 透传 UPF 执行限速。

## 场景差异

- **BWM 规则无 POLICYNAME（与计费/PCC QoS 规则不同）**：POLICYTYPE=BWM 的规则不绑 PCCPOLICYGRP/QOSPROP/URRGROUP（BWM 仅在 UNC 侧做规则名承载，实际限速参数 BWMSERVICE/BWMCONTROLLER/BWMUSERGROUP/BWMRULE 在 UDG/UPF 侧配置），故无 POLICYNAME 参数。对比：内容计费 POLICYTYPE=PCC 绑 PCCPOLICYGRPNM，QoS 保证 POLICYTYPE=QOS 绑 QOSPROPNAME，ADC POLICYTYPE=ADC 无 POLICYNAME。
- **UPTYPE=PCC（与计费族/ADC 共享）**：BWM 的 USERPROFILE 复用 PCC 类型模板（UPTYPE=PCC），与 PCC 基本功能/内容计费/ADC 一致。USERPROFILE 是跨特性共享容器，BWM 通过 RULEBINDING 挂 BWM 规则。
- **完整挂接到 APN/DNN（与 ADC 预定义规则链 1-00016 不同）**：ADC 预定义规则链（[1-00016](task/UNC/20.15.2/1-00016.md)）仅含 RULE+USERPROFILE+RULEBINDING 三命令（规则承载层），UserProfile 组挂接 APN 在 ADC 特性层编排；BWM activation **完整演示六命令**（含 USRPROFGROUP+UPBINDUPG+APNUSRPROFG），故本 compound 含完整挂接链。挂接三命令与 [1-00002](task/UNC/20.15.2/1-00002.md) UserProfile 绑定 compound 共享 atom（ADD USRPROFGROUP/UPBINDUPG/APNUSRPROFG）。
- **三网元 RULENAME 一致（预定义规则场景）**：PCRF/PCF 下发预定义规则名时，GGSN-C/PGW-C/SMF、PGW-U/UPF、PCRF/PCF 上的规则标识必须一致。本地 PCC 静态规则场景同样要求 UNC 与 UDG 侧 RULENAME 一致。
- **动态 PCC vs 本地 PCC（命令集相同）**：动态 PCC 时 PCRF/PCF 下发预定义规则名，本地 RULENAME 匹配后透传 UPF；本地 PCC（无 PCRF）时按 PRIORITY 全局优先级匹配本地静态规则。两路径本 compound 命令集相同，仅规则来源不同。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| POLICYTYPE=BWM | BWM 规则，无 POLICYNAME；UNC 侧仅规则名承载，限速参数在 UDG/UPF 侧 |
| UPTYPE=PCC | PCC 类型用户模板（BWM 共享 PCC 模板体系） |
| UPBINDTYPE=DEFAULT | 默认绑定类型（activation 演示取值） |
| 动态 PCC（PCRF/PCF 下发预定义规则） | 本 compound 提供本地 RULENAME 匹配；须完成 [1-00012](task/UNC/20.15.2/1-00012.md)/[1-00013](task/UNC/20.15.2/1-00013.md)/[1-00014](task/UNC/20.15.2/1-00014.md) PCRF 对接+选择+开关 |
| 本地 PCC（无 PCRF） | 本 compound 提供本地静态规则；须 [1-00014](task/UNC/20.15.2/1-00014.md) SET PCCFUNC HOMEPCCSWITCH=DISABLE + 本 compound 必配 |

> 联动特性级 DP：PCC 模式（动态 vs 本地）由 [2-00005](task/UNC/20.15.2/2-00005.md) DP2 决定；本 compound 在两种模式下均编排（动态=可选承载预定义规则，本地=必配静态规则）。

## 约束

- **三网元 RULENAME 一致**（critical）：GGSN-C/PGW-C/SMF、PGW-U/UPF、PCRF/PCF 上的 RULENAME 必须一致 — 否则 PCRF/PCF 下发的预定义规则名无法本地匹配，或本地静态规则与 UPF 限速参数脱节。
- **RULENAME+POLICYTYPE 唯一**（high）：多条记录时 RULENAME+POLICYTYPE 组合不能完全相同 — 重复则规则歧义。
- **License 前置**（critical）：SET LICENSESWITCH 激活 LKV3TCBSA01 须先于本 compound — 否则 BWM 功能不生效。
- **依赖 PCC 基本功能**（critical）：须已完成 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md) 激活（License LKV2PCCBF01+LKV3W9SPCC11 + PCC 开关） — 否则 BWM 规则无 PCC 通道承载，无法经 N4 透传 UPF。
- **USERPROFILE/USRPROFGROUP 须先建再绑**（high）：RULEBINDING 的 USERPROFILENAME 与 RULENAME、UPBINDUPG 的 USERPROFGNAME 与 USERPROFILENAME 均须已配置存在 — 否则引用失败。
- **加密协议不可 BWM**（warning）：对 SSL/FTPS 等加密协议报文不能应用带宽控制策略（特性 wiki 应用限制） — 仅 P2P/VoIP 等可识别协议。
- **UNC 侧不配限速参数**（info）：BWMSERVICE/BWMCONTROLLER/BWMUSERGROUP/BWMRULE/APNBINDBWMUSRG/BANDWIDTHMNG 等限速对象在 UDG/UPF 侧配置（对应 UDG 特性 [2-00001 GWFD-110311](task/UDG/20.15.2/2-00001.md)） — UNC 侧仅做规则名承载与 N4 透传。
- **生效时机**（low）：执行后立即生效；新增用户组对在线用户不生效，需重激活。
