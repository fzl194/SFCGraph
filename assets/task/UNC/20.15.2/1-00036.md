# UNC UPF 选择族

> **UNC 域 UPF 选择族骨架**（SOP §3.5 域通用 + 特性专属）。被引用于 **WSFD-107010**（UPF 选择）+ **WSFD-010202**（部分）+ **WSFD-106003**（部分 UPF 维度）。
>
> **UNC 域独有**：UDG 是用户面执行，UPF 选择由 SMF/MME 等控制面触发——本骨架命令族仅 UNC 控制面有。

## 配置方法

本骨架在 UNC 域选择 UPF 实例 + 配置 UP 节点 + 绑定 UP 区域位置——按"对端 NF 描述 / 本端 UP 节点 / UP 选择条件 / UP 区域位置"4 组建设。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 对端 NF 实例 Profile | `ADD PNFPROFILE` | NFINSTANCENAME（UPF 实例名，全网规划）|
| 2. UPF 选择条件（按切片/DNN/DNAI/UPF INFO/TAI/SMF 服务区）| `ADD UPFSELECTOR`（族，按选择条件配；命令名待核）| 切片/DNN/DNAI/UPF INFO/TAI/SMF 服务区等选择维度 |
| 3. 本地 UP 节点 | `ADD UPNODE` | UPNODEID, NFINSTANCENAME（已 `ADD PNFPROFILE` 注册的 UPF）|
| 4. UP 区域与位置绑定 | `ADD UPAREA`（族，命令名待核）+ `ADD UPBIND`（族，命令名待核）| UP 区域 + 位置（TAI/RAI 等）|
| 5. UPF 选择策略开关 | `ADD UPSELPLCY`（族，命令名待核）| UPF 角色 × 选择条件 |
| 6. （可选）UPF 主锚点标注 | `SET UPFMASTER`（族，命令名待核）| 主备 UPF 标识 |

**典型脚本（基础 UPF 选择，WSFD-107010 场景）**：
```
ADD PNFPROFILE: NFINSTANCENAME="upf-site1";
ADD UPNODE: UPNODEID="upf-01", NFINSTANCENAME="upf-site1", PFINSTANCEID="pf-01";
ADD UPSELPLCY: PREFERENCE=1;
```

**步骤位置**：所有 UPF 选择族特性的核心——前置 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施（VPN+APN）+ [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 地址池体系（地址池必须先建）。

## 场景差异

### [WSFD-107010](task/UNC/20.15.2/2-00033.md) UPF 选择（9 行场景差异）

按 UPF 角色 × 选择条件展开：

| UPF 角色 × 选择条件 | 步骤命令子集 | 关键联动 |
|---|---|---|
| **主锚点 UPF × DNN**（基线）| 步骤 1（PNFPROFILE + PNFDNN）+ 步骤 2-3（UPNODE + UPSELECTFLAG）| DNN 必选 |
| **I-UPF × 切片（S-NSSAI）**| 步骤 1（PNFPROFILE + PNFNS）+ 步骤 2-3（UPNODE + UPSELECTFLAG(N3UPFAPNFLAG)）| 切片必选；I-UPF 不需 DNN |
| **PSA UPF × DNAI**（分流）| 步骤 1（PNFPROFILE + PNFDNAI）+ 步骤 2-3 | DNAI 必选 |
| **SGW-U × S11 接口**（4G/2-3G 切换）| 步骤 2-3（UPNODE + UPBINDS11 + UPSELECTFLAG(ULISGWFLAG)）| S11 必选；ULI 为空时按 S11 选 |
| **PGW-U/GGSN-U × Gn/Gp 或 S5/S8 接口**| 步骤 2-3（UPNODE + UPBINDGNGP）| 接口必选 |
| **合一 UPF（SGW-U/PGW-U 合一）**| 步骤 1 + 2 + 3（UPSELECTPRI 第一优先级=COMBINED + UPSELECTFLAG(PRIORITYFLAG=DISABLE)）| 合一优选 |
| **按权重 + 负载**（多 UPF 并存）| 步骤 1（PNFPROFILE 权重）+ 步骤 3（UPLOADBALANCE）| 权重 + 负载 |
| **按位置区**（贴近用户）| 步骤 1-2（UPAREA + UPAREABINDN2TAI + LOCBINDAREA）+ 步骤 3（UPSELECTPRI 第二优先级=LOCS11PRIORITY）| 位置区 |
| **优先级排序**（PRIORITYFLAG=ENABLE）| 步骤 3（UPSELECTFLAG(PRIORITYFLAG=ENABLE) + APNUPSELPLY(COMBINEPRISTG)）| 启用优先级开关 |

> **★ 命令子集警示**（SOP §3.5 R1.2 第 5 条命令多带型假通用）：本骨架命令集是 9 场景并集（17+ 条命令），个别 feature **仅用子集**。配置生成若按骨架典型脚本盲目套用某一具体 feature，可能**误产出**该 feature 不执行的命令（如按 9 场景并集配置"按权重+负载"场景会漏 PNFPROFILE 的 PFTYPE=UPF 强制要求）。**特征差异维度**（7 选择原则 + 合一/分立 + 优先级 + 权重）必须按上表子集选配。

## 约束

- **UPF 节点须 PNFPROFILE 先建**（critical）：`ADD UPNODE` 的 NFINSTANCENAME 必须已在 `ADD PNFPROFILE` 注册 — 后果：UPF 节点配置失败
- **本骨架命令 atom 多待补**：除 [0-00263](task/UNC/20.15.2/0-00263.md) `ADD UPNODE` 外，UPF 选择/区域/选择条件等命令的 atom UNC pre-build 阶段未建
