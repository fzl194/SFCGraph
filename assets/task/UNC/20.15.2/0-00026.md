# 配置Diameter对端地址（ADD DIAMPEERADDR）

> 命令静态知识见 [ADD DIAMPEERADDR](command/UNC/20.15.2/ADD-DIAMPEERADDR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD DIAMPEERADDR 指定对端主机的地址类型与地址值，HOSTNAME 必须已用对应服务器命令（[ADD PCRF](command/UNC/20.15.2/ADD-PCRF.md) / [ADD OCS](command/UNC/20.15.2/ADD-OCS.md) / [ADD DRA](command/UNC/20.15.2/ADD-DRA.md) / ADD DIAMETERAAA）建好。地址类型三选一：**IPv4**（TCP，可选 PORT，缺省 3868）、**IPv6**（TCP，可选 PORT）、**SCTP**（SCTP，端点须已用 [ADD SCTPENDPOINT](command/UNC/20.15.2/ADD-SCTPENDPOINT.md) 建好）。同一主机可重复本命令加多个地址/端点构成多链路（负荷分担）。

### 配置维度 1：地址类型（参数 ADDRTYPE）

| 取值 | 承载 | 后续参数 | 前置依赖 |
|---|---|---|---|
| `IPv4` | TCP | IPV4ADDRESS（+可选 PORT，缺省 3868） | 对端主机已建 |
| `IPv6` | TCP | IPV6ADDRESS（+可选 PORT） | 对端主机已建 |
| `SCTP` | SCTP | SCTPENDPOINT | 已 ADD SCTPENDPOINT 建好对端端点 |

> 同一主机**不允许** IPv4/IPv6/SCTP 多种类型地址混合配置。

### 配置维度 2：IPv4 地址（参数 IPV4ADDRESS，ADDRTYPE=IPv4 时必填）

| 取值 | 获取方法 | 说明 |
|---|---|---|
| `<ipv4>` | 与对端协商或全网规划 | 禁 0.0.0.0 / 255.255.255.255；多 IP 负荷分担 |

### 配置维度 3：IPv6 地址（参数 IPV6ADDRESS，ADDRTYPE=IPv6 时必填）

| 取值 | 获取方法 | 说明 |
|---|---|---|
| `<ipv6>` | 与对端协商 | 禁全 F |

### 配置维度 4：端口号（参数 PORT，ADDRTYPE=IPv4/IPv6 时可选）

| 取值 | 默认 | 说明 |
|---|---|---|
| `1~65534` | `3868` | 需与对端设备配置一致；错误会导致连接建立失败、服务器状态异常 |

### 配置维度 5：SCTP 端点（参数 SCTPENDPOINT，ADDRTYPE=SCTP 时必填）

| 取值 | 获取方法 | 说明 |
|---|---|---|
| `<端点名>` | 本端规划（已 ADD SCTPENDPOINT） | 1~63 字符，不支持空格，不区分大小写 |

### 典型脚本

```
ADD DIAMPEERADDR:HOSTNAME="ocs1",ADDRTYPE=IPv4,IPV4ADDRESS="10.10.10.10";
ADD DIAMPEERADDR:HOSTNAME="ocs1",ADDRTYPE=SCTP,SCTPENDPOINT="end1";
```

**步骤位置**：在 Gx/Gy/S6b 接口建链流程中，位于 ADD PCRF/ADD OCS/ADD DRA（建对端主机）之后、[ADD DIAMCONNECTION](task/UNC/20.15.2/0-00029.md)（建链）之前。

## 决策点

### 决策点：ADDRTYPE 对端地址类型选择（DP 0-02031）

问：对端 Diameter 主机走哪种承载协议（决定后续参数集与前置依赖）？

| 选项 | 承载 | 后续参数 | 前置依赖 |
|---|---|---|---|
| IPv4 | TCP | IPV4ADDRESS（+可选 PORT，缺省 3868） | 对端主机已建（ADD PCRF/OCS/DRA/AAA） |
| IPv6 | TCP | IPV6ADDRESS（+可选 PORT） | 对端主机已建 |
| SCTP | SCTP | SCTPENDPOINT | 已 ADD SCTPENDPOINT 建好对端端点 |

> over-DRA 场景下同一 DRA 主机**强制二选一**（IP 与 SCTP 端点不可并存）；PCRF/OCS 直连场景无此互斥限制。

### 决策点：直连 OCS vs 非直连 OCS（Gy over DRA 场景）（DP 0-02032）

问：Gy 接口 OCS 是直连还是经 DRA 转发？

| 选项 | 配置 | 路由方式 |
|---|---|---|
| 直连 OCS | ADD OCS（Realm）+ ADD DIAMPEERADDR（IP） | 直连链路 |
| 非直连 OCS | ADD OCS（Realm），**不配** ADD DIAMPEERADDR | 基于 Realm 查 Diameter 路由经 DRA 转发 |

## 约束

- **HOSTNAME 前置依赖**（critical，rule-0-02076）：HOSTNAME 前置依赖：要配的 Diameter 主机不存在则不能配对端地址信息，配置失败 — 先建对应类型对端主机（ADD PCRF/OCS/DRA/AAA）
- **类型不可混合**（critical，rule-0-02077）：类型不可混合：同一主机不允许 IPv4/IPv6/SCTP 多种类型地址混合配置；同一主机的 IP 或 SCTP 端点不可重复 — 一台主机只能选一种承载类型
- **单主机地址数上限**（warning，rule-0-02078）：单主机地址数上限：PCRF/OCS/DRA 主机最多 20 个 IP 或 20 个 SCTP 端点；3GPP AAA 主机最多 2 个 IP 或 2 个 SCTP 端点，超规格配置失败 — 规划前核对主机配额
- **PORT 须与对端一致**（warning，rule-0-02079）：PORT 须与对端一致：PORT 默认 3868，取值 1~65534，与对端不一致会导致连接不能建立、服务器状态异常 — 配置前与对端核对端口
- **立即生效与记录上限**（info，rule-0-02080）：立即生效与记录上限：命令执行后立即生效，最大记录数 510；整机上限 OCS 100/PCRF 200/DRA 80/AAA 20 条地址 — 规划前核对整机配额
