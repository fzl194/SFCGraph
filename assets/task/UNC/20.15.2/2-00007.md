# 基于累计流量的策略控制（WSFD-109104）

> 特性静态知识见 [WSFD-109104](feature/UNC/20.15.2/WSFD-109104.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系**：PCC 族增量特性（第 7 批，带宽族第 2 批）。本特性 = [PCC基本功能 WSFD-109101](task/UNC/20.15.2/2-00005.md) 的会话级累计流量增量——PCC 基础设施（PCRF 对接链/PCRF 选择/PCC 开关模板）全部复用 2-00005，本特性仅叠加 3 个 FUP 专属参数增量：① `SET PCCFUNC` 加 `MKPARSEFORMAT`（Monitoring-Key 解析方式，与 PCRF 一致）② `MOD PCRF` 加 `SUPFEANEGOSW`+`FEATURELIST=UMCH-1`（使能流量监控拥塞处理）③ `MOD PCCPOLICYGRP` 加 `FUPSESSIONEXC=ENABLE`（该策略组业务不计入 Session 级流量）。N7 接口路径本端无需专有配置（仅 License）。
>
> 与 UDG 侧 [GWFD-020353](feature/UDG/20.15.2/GWFD-020353.md) 对应：UDG 侧 UPF 执行策略动作（URR 阈值/上报），UNC 侧仅做策略接收与转换。

## 配置概览

核心增量链（叠在 PCC 基本功能之上）：**License 前置** → **SET PCCFUNC 设 MKPARSEFORMAT**（Gx 接口 Monitoring-Key 解析方式，与 PCRF 一致）→ **MOD PCRF 使能 UMCH**（流量监控拥塞处理，仅 Gx）→ **MOD PCCPOLICYGRP 设 FUPSESSIONEXC**（该策略组业务不计入 Session 级流量，可选）。

两条接口路径差异：
- **Gx 接口（2G/3G/4G，与 PCRF 交互）**：需本端配 MKPARSEFORMAT + UMCH + FUPSESSIONEXC 三增量，前置完成 PCC 基本功能 Gx 路径（PCRF 对接链 + PCRF 选择 + PCC 开关）。
- **N7 接口（5G，与 PCF 交互）**：会话级累计流量仅在 PCF 上配置，本端 PGW-C/SMF **不需要专有配置**（仅 License 前置）；SMF 内部由 N7 umDecs 自动生成 N4 URR 规则。

关键差异维度：① 接口路径（Gx 本端配 vs N7 仅 License）② 是否使能 UMCH（仅 Gx，避免月末配额重置信令拥塞）③ 是否排除某类业务流量（FUPSESSIONEXC）④ MKPARSEFORMAT 与 PCRF 一致性。

## 配置流程（步骤编排，每步显示含命令）

1. **激活 FUP License**（前置）：`SET LICENSESWITCH` LICITEM="LKV3W9PCBT12" → [0-00180](task/UNC/20.15.2/0-00180.md)
   - 关键参数：LICITEM="LKV3W9PCBT12"（基于累计流量的策略控制，控制项 82207980），SWITCH=ENABLE
   - **前置依赖**：须先完成 [PCC基本功能 WSFD-109101](task/UNC/20.15.2/2-00005.md) 激活（License LKV2PCCBF01 + LKV3W9SPCC11 + PCRF 对接 + PCC 开关模板）
2. **设置累计流量上报方式（仅 Gx 路径）**：`SET PCCFUNC` MKPARSEFORMAT=UNSIGNED32 → [0-00193](task/UNC/20.15.2/0-00193.md)
   - 关键参数：MKPARSEFORMAT=UNSIGNED32（Monitoring-Key 解析方式，**必须与 PCRF 一致**）
   - 复用 [1-00014](task/UNC/20.15.2/1-00014.md) PCC开关与模板的 SET PCCFUNC atom；本特性只增 MKPARSEFORMAT，不重配 HOMEPCCSWITCH（已在 PCC 基本功能配过）
   - N7 路径跳过此步（本端不配）
3. **使能流量监控拥塞处理 UMCH（仅 Gx 路径）**：`MOD PCRF` HOSTNAME + SUPFEANEGOSW=ENABLE + FEATURELIST=UMCH-1 → [0-00122](task/UNC/20.15.2/0-00122.md)
   - 关键参数：HOSTNAME=已 ADD PCRF 定义的主机名；SUPFEANEGOSW=ENABLE（动态协商开关）；FEATURELIST=UMCH-1（使能 UMCH）
   - 前置：HOSTNAME 须已在 PCC 基本功能 [1-00012](task/UNC/20.15.2/1-00012.md) 的 ADD PCRF 中定义
   - N7 路径跳过此步（UMCH 是 Gx 拥塞处理机制，N7 无对应）
4. **配置 FUP 累计排除（可选，Gx/N7 均可）**：`MOD PCCPOLICYGRP` FUPSESSIONEXC=ENABLE → [0-00121](task/UNC/20.15.2/0-00121.md)
   - 关键参数：PCCPOLICYGRPNM=已 ADD PCCPOLICYGRP 定义的策略组名；FUPSESSIONEXC=ENABLE（该策略组业务**不计入** Session 级流量）/ DISABLE（默认，计入）
   - 前置：PCCPOLICYGRPNM 须已在 PCC 基本功能（2-00005 步骤 8）或费率标识链 [1-00009](task/UNC/20.15.2/1-00009.md) 的 ADD PCCPOLICYGRP 中定义
   - 仅当需将某类业务/应用流量排除在 FUP 统计之外时配置；默认 DISABLE 不需此步

> **activation 演示范围脚注（R1.4）**：WSFD-109104 仅 1 份 activation（`激活基于累计流量的策略控制_29056190.md`），演示 4 条命令完整脚本（SET LICENSESWITCH + SET PCCFUNC:MKPARSEFORMAT + MOD PCRF:SUPFEANEGOSW/FEATURELIST=UMCH-1 + ADD PCCPOLICYGRP:FUPSESSIONEXC）。**无零证据差异参数**——MKPARSEFORMAT/SUPFEANEGOSW/FEATURELIST/FUPSESSIONEXC 均有 activation 数据规划表 + 脚本双重证据。
>
> **★ activation 脚本与配置流程的差异（R1.4 子规则，warning）**：activation 步骤 4 写的是 `ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicygrp_01", FUPSESSIONEXC=ENABLE;`（ADD 新建），而本特性配置流程写 `MOD PCCPOLICYGRP`（修改已存在）。差异原因：activation 假设首次部署（PCCPOLICYGRP 未建），本特性作为 PCC 基本功能增量，PCCPOLICYGRP 已在 PCC 基本功能步骤 8 建过——故实际配置生成走 MOD 修改增量参数 FUPSESSIONEXC。**配置生成处置**：若 PCCPOLICYGRP 已存在（依赖 PCC 基本功能已激活）→ 用 MOD PCCPOLICYGRP；若首次部署未建 → 用 ADD PCCPOLICYGRP 并带 FUPSESSIONEXC。两命令 wiki 均有 FUPSESSIONEXC 参数（[MOD PCCPOLICYGRP](command/UNC/20.15.2/MOD-PCCPOLICYGRP.md) / ADD PCCPOLICYGRP），证据来自 activation 脚本。

## 决策点（特性级）

### DP1：接口路径（决定本端配置量）

| 接口路径 | 走法 | 本端配置 | 前置 |
|---|---|---|---|
| Gx（2G/3G/4G，与 PCRF 交互） | 步骤 1+2+3+4 | MKPARSEFORMAT + UMCH + FUPSESSIONEXC 三增量 | PCC 基本功能 Gx 路径（[2-00005](task/UNC/20.15.2/2-00005.md) DP1=2G/3G/4G）已激活 |
| N7（5G，与 PCF 交互） | 步骤 1（仅 License） | 本端**无专有配置**（会话级累计流量仅 PCF 配） | PCC 基本功能 N7 路径（[2-00005](task/UNC/20.15.2/2-00005.md) DP1=5G）已激活 |

### DP2：是否使能 UMCH（仅 Gx，避免配额重置信令拥塞）

| 选项 | 影响 | 适用 |
|---|---|---|
| UMCH=ENABLE（FEATURELIST=UMCH-1 + SUPFEANEGOSW=ENABLE） | 步骤 3 必配；PCRF 通过 Monitoring-Time 指定配额重置点，UNC 分统计重置前后流量合并上报 | 包月用户月末配额重置场景（典型 FUP） |
| UMCH 不使能 | 步骤 3 跳过；PCRF 同时要求所有在线用户上报→Gx 信令拥塞风险 | 无配额重置需求或用户数少 |

### DP3：是否排除某类业务流量（FUPSESSIONEXC，Gx/N7 均可）

| 选项 | 影响 | 适用 |
|---|---|---|
| FUPSESSIONEXC=ENABLE（步骤 4） | 该 PCCPOLICYGRPNM 对应业务**不计入** Session 级流量 | 免流应用/特定业务排除 FUP 统计 |
| FUPSESSIONEXC=DISABLE（默认，步骤 4 跳过） | 该策略组业务**计入** Session 级流量 | 默认 FUP 统计口径 |

## 约束（特性级 rule）

- **依赖 PCC 基本功能**（critical）：本特性须先激活 [WSFD-109101 PCC基本功能](task/UNC/20.15.2/2-00005.md)（License LKV2PCCBF01+LKV3W9SPCC11 + PCRF 对接 + PCC 开关） — 否则无 PCC 基础设施，FUP 增量无处叠加。
- **License 前置**（critical）：须激活 LKV3W9PCBT12（控制项 82207980） — 否则 FUP 功能不生效。
- **MKPARSEFORMAT 须与 PCRF 一致**（critical，仅 Gx）：SET PCCFUNC 的 MKPARSEFORMAT 必须与对端 PCRF 协商一致 — 否则 Monitoring-Key 解析不匹配，流量累计错误。
- **UMCH 仅支持总流量阈值**（warning，仅 Gx）：UMCH 机制不支持上下行独立阈值，仅总流量阈值 — 规划时不可设分上下行阈值。
- **UE Category 用户阈值下限**（warning）：PCRF/PCF 下发的会话级流量阈值须不低于 1GB — 否则影响 UE Category 用户流量累计准确性。
- **MOD PCRF 联动刷新同组**（warning）：MOD PCRF 修改 SUPFEANEGOSW/FEATURELIST 会同时刷新该 PCRFGROUP 下所有 PCRF 的协商参数 — 修改前确认组内影响范围。
- **FUPSESSIONEXC 语义**（info）：ENABLE=不计入 Session 级流量（排除该业务），DISABLE（默认）=计入。
- **N7 路径本端免配**（info）：N7 接口会话级累计流量仅在 PCF 配置，本端 SMF 内部自动由 N7 umDecs 生成 N4 URR 规则，无需 MML。
