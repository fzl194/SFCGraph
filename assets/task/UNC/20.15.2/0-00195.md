# SET PCCPCRFMSGATTR 原子任务

> 命令静态知识见 [SET PCCPCRFMSGATTR](command/UNC/20.15.2/SET-PCCPCRFMSGATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET PCCPCRFMSGATTR 控制激活和更新流程中 UNC 是否根据 DRA 或 PCRF 消息触发的 PCRF 重选。三个参数分别对应三种触发消息（CCA-I/CCA-U/RAR 的 Origin-Host AVP）。命令存在系统初始记录，三参数初始值均为 DISABLE（缺省不支持重选）。需启用时置 ENABLE。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| ORGHOSTCCAI | `ENABLE` / `DISABLE`（初始） | 本端规划 | 基于CCA-I Origin-Host AVP 触发 PCRF 重选 |
| ORGHOSTCCAU | `ENABLE` / `DISABLE`（初始） | 本端规划 | 基于CCA-U Origin-Host AVP 触发 PCRF 重选 |
| ORGHOSTRAR | `ENABLE` / `DISABLE`（初始） | 本端规划 | 基于RAR Origin-Host AVP 触发 PCRF 重选 |

**特性场景取值对比**

| 特性场景 | ORGHOSTCCAI | ORGHOSTCCAU | ORGHOSTRAR | 说明 |
|---|---|---|---|---|
| PCC 基本功能（WSFD-109101） | DISABLE（缺省） | DISABLE（缺省） | DISABLE（缺省） | 缺省不支持，按需使能 |

证据中 WSFD-109101 将此命令列为"可选配置"，用于整机粒度控制在激活过程中根据 PCRF 返回的重定向指示重选 PCRF。

## 决策点

### 是否启用 PCRF 重选（缺省 DISABLE，按需 ENABLE）

三参数缺省均为 DISABLE（不支持重选）。是否启用取决于网络是否需要 UNC 根据 DRA/PCRF 的 Origin-Host 重定向指示动态重选 PCRF。属本端规划决策，需结合组网（是否经 DRA、PCRF 是否支持重选指示）决定。

### 触发消息类型选择（CCA-I/CCA-U/RAR）

三种触发消息对应不同流程阶段：
- `ORGHOSTCCAI` → 激活流程（CCA-I 为 Initial CCA）。
- `ORGHOSTCCAU` → 更新流程（CCA-U 为 Update CCA）。
- `ORGHOSTRAR` → PCRF 主动推送（RAR 为 Re-Auth-Request）。

可按需单独或组合使能。

## 约束

- **立即生效** · 执行后立即生效。
- **最大 1 条记录** · 全局唯一配置。
- **系统初始记录** · severity=MEDIUM · 三参数初始值均为 DISABLE；未配置时缺省不支持 PCRF 重选。
- **整机粒度** · severity=LOW · 本命令为整机粒度控制，非 APN/用户粒度。
- **可选配置** · severity=LOW · 证据中标记为"可选配置"，缺省不支持时按常规 PCRF 选择流程处理。
