# SET FAILHANDLING 原子任务

> 命令静态知识见 [SET FAILHANDLING](command/UNC/20.15.2/SET-FAILHANDLING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET FAILHANDLING 给一个已建的融合计费模板（CCT）挂上"故障处理策略"。它本身不是建模板，而是**改写模板的故障行为**，因此前置必须先 [ADD CCT](task/UNC/20.15.2/0-00009.md)。一条模板一条记录，参数全部可选（不输则保持现值，可 `LST CCT` 查）。

按"故障恢复策略"分两大范式：**主备 CHF 切换（可靠性）** 与 **业务放通+消息缓存（容灾）**。两者 FHACTION 取值不同，配套参数也不同。

**参数取值样例（融合计费特性配置范式）**

| 参数 | 主备切换范式 | 业务放通+缓存范式 | 说明 |
|---|---|---|---|
| CCTMPLTNAME | `cct-test` / `global` | `cct-test` | 目标模板名，前置由 ADD CCT 建 |
| FHACTION | `RETRY_AND_TERM` / `CONTINUE` | `CONTINUE`（固定） | FH 处理动作；缓存场景必须 CONTINUE 才能放通 |
| FAILOVERSUP | `ENABLE` | （可不设） | 主 CHF 故障时向备 CHF 发消息 |
| AUTOFAILBACK | `ENABLE` | （可不设） | 主 CHF 恢复后回切 |
| TXTIMER | `5` | `5` | Tx 定时器(秒)，超时即认定 CHF 异常，0~20 |
| HOLDINGTIME | `10` | — | 用户保持时长(分)，0~2880 |
| UPDCONTTERM | — | `ENABLE` | Update 消息触发缓存场景发 Terminate 结束计费会话 |
| CNVCHGRECOVER | — | `ENABLE` | CHF 故障恢复后重建计费会话 |
| RECOVERINTERVAL | — | `20` / `5` | 自动恢复间隔(分)，0~60 |
| SCPFAILOVERSW | `SCP_NOT_FAILOVER` | — | SCP 故障重选开关 |

**典型行（主备 CHF 切换/可靠性）**

```
SET FAILHANDLING: CCTMPLTNAME="global", FHACTION=RETRY_AND_TERM, FAILOVERSUP=ENABLE,
  AUTOFAILBACK=ENABLE, TXTIMER=5, HOLDINGTIME=10;
```

**典型行（业务放通 + 消息缓存）**

```
SET FAILHANDLING: CCTMPLTNAME="cct-test", FHACTION=CONTINUE, TXTIMER=5,
  UPDCONTTERM=ENABLE, CNVCHGRECOVER=ENABLE, RECOVERINTERVAL=5;
```

## 决策点

### FHACTION 故障处理动作选择

FHACTION 决定 SMF 超时未收到 ChargingDataResponse 后做什么，是可靠性范式与容灾范式的分水岭。

| 选项 | 含义 | 适用场景 | 配套 |
|---|---|---|---|
| `RETRY_AND_TERM` | 重试后去活 PDU 会话 | 主备 CHF 可靠性，要求计费严格 | 配 FAILOVERSUP/AUTOFAILBACK |
| `CONTINUE` | 放通业务、不进行配额管理/上报 | CHF 故障容灾，开启消息缓存的前提 | 配 SET N40MSGSTG（[0-00184](task/UNC/20.15.2/0-00184.md)）开启缓存 |
| `TERMINATE` | 直接去活 PDU 会话 | 故障即断网（少见，合规要求严） | — |

### 是否开启主备 CHF 切换

| 选项 | FAILOVERSUP / AUTOFAILBACK | 适用 |
|---|---|---|
| 主备切换 + 自动回切 | `ENABLE` / `ENABLE` | 部署主备 CHF，要求故障自动恢复 |
| 仅切换不回切 | `ENABLE` / `DISABLE` | 主恢复后人工确认再回切 |
| 不切换（单 CHF） | `DISABLE` / — | 无备 CHF 或纯容灾放通 |

### 是否启用融合计费自动恢复

CNVCHGRECOVER + RECOVERINTERVAL 决定 CHF 恢复后是否自动重建计费会话。

| 选项 | CNVCHGRECOVER / RECOVERINTERVAL | 适用 |
|---|---|---|
| 自动恢复 | `ENABLE` / `5~20` | CHF 恢复后无需人工干预即重建会话 |
| 不自动恢复 | `DISABLE` / — | 需人工触发恢复 |

## 约束

- **立即生效** · severity=LOW。
- **TXTIMER 与 T3N3 协调**：建议 TXTIMER（等待 CHF 最大时长）小于本产品 `ADD GTPCT3N3` 配置的 T3N3 时长，及左侧（MME/SGSN/AMF）T3N3 时长，否则激活响应总时长过长 · severity=HIGH。
- **CCTMPLTNAME 前置依赖**：必须由 [ADD CCT](task/UNC/20.15.2/0-00009.md) 先建模板 · severity=HIGH。
- **系统初始记录存在**：部署后已有初始值，`LST CCT` 可查，本命令为改写 · severity=LOW。
- **缓存范式前置**：FHACTION=CONTINUE 是开启计费消息缓存（[SET N40MSGSTG](task/UNC/20.15.2/0-00184.md) STGSWITCH=ENABLE）的前提，且需 [ADD PDUSCACT](task/UNC/20.15.2/0-00061.md) SCACT=FAILOVER/CONTINUE 配合 · severity=HIGH。
- **TXTIMER 与配额耗尽联动**：申请配额后 Total Volume 到达仍未收到 CHF 响应、且 TXTIMER 未超时，则 RG 配额耗尽动作走 [ADD QUOTAEXHAUSTACT](task/UNC/20.15.2/0-00068.md)（注：此处指 ADD RGRCACT/QUOTAEXHAUSTACT 类配额耗尽动作命令）的 ACTION · severity=MEDIUM。
