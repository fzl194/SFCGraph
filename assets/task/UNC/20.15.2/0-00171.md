# 设置融合计费模板故障处理动作（SET FAILHANDLING）（SET FAILHANDLING）

> 命令静态知识见 [SET FAILHANDLING](command/UNC/20.15.2/SET-FAILHANDLING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET FAILHANDLING 给一个已建的融合计费模板（CCT）挂上"故障处理策略"。它本身不是建模板，而是**改写模板的故障行为**，因此前置必须先 [ADD CCT](task/UNC/20.15.2/0-00009.md)。一条模板一条记录，参数全部可选（不输则保持现值，可通过 `LST CCT` 查）。系统部署完成后已存在初始记录（CCTMPLTNAME=`global`、FHACTION=`TERMINATE`、TXTIMER=`3`、HOLDINGTIME=`30` 等），本命令为改写。

按"故障恢复策略"分两大范式：**主备 CHF 切换（可靠性）** 与 **业务放通+消息缓存（容灾）**。两者 FHACTION 取值不同，配套参数也不同。本命令的核心配置空间有 2 个维度：**故障处理动作（FHACTION）**（决定 CHF 故障时是去活、重试去活还是放通）、**故障恢复策略组合**（主备切换 vs 自动恢复，二选一或共存）。

### 配置维度 1：故障处理动作 FHACTION（参数 FHACTION）

FHACTION 决定 Tx 定时器超时或 CHF 返回某些错误码后的处理动作，是可靠性范式与容灾范式的分水岭。

| FHACTION 取值 | 含义 | 适用范式 | 配套参数 |
|---|---|---|---|
| `RETRY_AND_TERM` | 尝试向备用 CHF 重发消息，重发失败则去活用户 | 主备 CHF 可靠性，要求计费严格 | 配 FAILOVERSUP=ENABLE / AUTOFAILBACK |
| `CONTINUE` | 尝试向备用 CHF 重发，失败则放通业务、不再进行配额管理/上报 | CHF 故障容灾，开启消息缓存的前提 | 配 [SET N40MSGSTG](task/UNC/20.15.2/0-00184.md) 开启缓存、UPDCONTTERM、CNVCHGRECOVER |
| `TERMINATE` | 直接去活 PDU 会话（系统初始值） | 故障即断网（少见，合规要求严） | — |

### 配置维度 2：故障恢复策略组合（参数 FAILOVERSUP/AUTOFAILBACK + CNVCHGRECOVER/RECOVERINTERVAL）

| 参数 | 作用 | 取值 | 系统初始值 |
|---|---|---|---|
| `CCTMPLTNAME`（必选） | 目标模板名，前置由 ADD CCT 建 | 字符串 1~63，不区分大小写 | `global`（初始记录） |
| `FAILOVERSUP` | 主 CHF 故障时向备 CHF 发消息（failover 开关） | `DISABLE` / `ENABLE` | `DISABLE` |
| `AUTOFAILBACK` | 主 CHF 恢复后自动回切（failback 开关） | `DISABLE` / `ENABLE` | `ENABLE` |
| `TXTIMER` | Tx 定时器(秒)：向 TOPO 查询 CHF 与等待 CHF 响应的时长，超时即认定 CHF 异常 | `0`~`20` | `3` |
| `HOLDINGTIME` | CHF 故障后用户保持时长(分)，超出去活；0=不主动去活 | `0`~`2880` | `30` |
| `RANGETIME` | HoldingTime 超时后的随机调整范围(分) | `0`~`60` | `30` |
| `CNVCHGRECOVER` | 融合计费自动恢复开关：CHF 故障保持期间是否重建会话恢复配额管理 | `DISABLE` / `ENABLE` | `DISABLE` |
| `RECOVERINTERVAL` | 自动恢复间隔(分) | `0`~`60` | `20` |
| `UPDCONTTERM` | Update 消息触发业务放通时，SMF 向 CHF 发 Terminate 结束计费会话的开关（需 CHF 支持） | `DISABLE` / `ENABLE` | `DISABLE` |
| `SCPFAILOVERSW` | Model C/D 通信模式下 SCP 故障是否重选 | `SCP_NOT_FAILOVER` / `SCP_FAILOVER` | `SCP_NOT_FAILOVER` |

> 范式选择：主备 CHF 部署 → 配 FAILOVERSUP=ENABLE + AUTOFAILBACK=ENABLE + FHACTION=RETRY_AND_TERM；CHF 故障容灾 → 配 FHACTION=CONTINUE + CNVCHGRECOVER=ENABLE + RECOVERINTERVAL + 开启 [SET N40MSGSTG](task/UNC/20.15.2/0-00184.md) 消息缓存。

典型脚本（主备 CHF 切换/可靠性，对 `global` 模板开 failover+failback，Tx 定时器 5 秒，保持 10 分钟）：
```
SET FAILHANDLING: CCTMPLTNAME="global", FHACTION=RETRY_AND_TERM, FAILOVERSUP=ENABLE, AUTOFAILBACK=ENABLE, TXTIMER=5, HOLDINGTIME=10;
```
典型脚本（业务放通 + 消息缓存/容灾，对 `cct-test` 模板放通业务，CHF 恢复后自动重建会话，恢复间隔 5 分钟）：
```
SET FAILHANDLING: CCTMPLTNAME="cct-test", FHACTION=CONTINUE, TXTIMER=5, UPDCONTTERM=ENABLE, CNVCHGRECOVER=ENABLE, RECOVERINTERVAL=5;
```

**步骤位置**：在所属 feature 编排中位于"配置融合计费故障处理"步骤——[ADD CCT](task/UNC/20.15.2/0-00009.md)（建模板）之后；缓存范式还需配合 [ADD PDUSCACT](task/UNC/20.15.2/0-00061.md)（PDU 异常返回码动作）与 [SET N40MSGSTG](task/UNC/20.15.2/0-00184.md)（消息缓存开关）。

## 决策点

### 决策点：FHACTION 故障处理动作选择（DP 0-03081）

问：SMF 超时未收到 ChargingDataResponse（Tx 定时器超时）或 CHF 返回错误码后做什么？这是可靠性范式与容灾范式的分水岭。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| RETRY_AND_TERM（重试后去活） | FHACTION=RETRY_AND_TERM + FAILOVERSUP/AUTOFAILBACK | 主备 CHF 可靠性，要求计费严格，重试失败即去活 |
| CONTINUE（放通业务、不再配额管理） | FHACTION=CONTINUE + [SET N40MSGSTG](task/UNC/20.15.2/0-00184.md) 开启消息缓存 | CHF 故障容灾，放通业务维持用户在线，是开启计费消息缓存的前提 |
| TERMINATE（直接去活，系统初始值） | FHACTION=TERMINATE | 故障即断网（少见，合规要求严） |

> 取值由故障恢复策略决定：有主备 CHF 且要求计费严格 → RETRY_AND_TERM；单 CHF 或需容灾放通 → CONTINUE。

### 决策点：故障恢复策略组合（DP 0-03082）

问：是开主备 CHF 切换（failover/failback），还是开融合计费自动恢复（CNVCHGRECOVER）？两者可按部署形态共存或择一。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 主备切换 + 自动回切 | FAILOVERSUP=ENABLE + AUTOFAILBACK=ENABLE | 部署主备 CHF，要求故障自动切换、主恢复后自动回切 |
| 仅切换不回切 | FAILOVERSUP=ENABLE + AUTOFAILBACK=DISABLE | 主恢复后需人工确认再回切 |
| 不切换（单 CHF）+ 自动恢复 | FAILOVERSUP=DISABLE + CNVCHGRECOVER=ENABLE + RECOVERINTERVAL=`5~20` | 无备 CHF，靠 CHF 恢复后自动重建计费会话 |
| 不切换、不自动恢复 | FAILOVERSUP=DISABLE + CNVCHGRECOVER=DISABLE | 需人工触发恢复 |

> 主备切换依赖部署主备 CHF；自动恢复（CNVCHGRECOVER）在 CHF 恢复后无需人工干预即重建会话，RECOVERINTERVAL 建议 5~20 分钟。

## 约束

· rule-0-03201 · severity=critical · CCTMPLTNAME 前置依赖：必须由 [ADD CCT](task/UNC/20.15.2/0-00009.md) 先建模板，CCTMPLTNAME 指向的模板名必须已存在，否则改写无效

· rule-0-03202 · severity=high · TXTIMER 与 T3N3 协调：建议 TXTIMER（等待 CHF 最大时长）小于本产品 `ADD GTPCT3N3` 配置的 T3N3 时长及左侧（MME/SGSN/AMF）T3N3 时长，否则激活响应总时长过长

· rule-0-03203 · severity=high · 缓存范式前置：FHACTION=CONTINUE 是开启计费消息缓存（[SET N40MSGSTG](task/UNC/20.15.2/0-00184.md) STGSWITCH=ENABLE）的前提，且需 [ADD PDUSCACT](task/UNC/20.15.2/0-00061.md) SCACT=FAILOVER/CONTINUE 配合

· rule-0-03204 · severity=warning · UPDCONTTERM 使能前提：只有在 CHF 支持该场景下处理 SMF 发送的 Terminate 消息时，才可使能 UPDCONTTERM=ENABLE

· rule-0-03205 · severity=info · 系统初始记录存在：部署后已有初始值（CCTMPLTNAME=global、FHACTION=TERMINATE、TXTIMER=3、HOLDINGTIME=30 等），`LST CCT` 可查，本命令为改写；执行后立即生效
