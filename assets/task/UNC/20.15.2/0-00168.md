# 配置上下文激活初始计费属性（SET CTXSTARTRATING）

> 命令静态知识见 [SET CTXSTARTRATING](command/UNC/20.15.2/SET-CTXSTARTRATING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

PGW-C/SMF 侧**用户模板粒度**的初始计费属性绑定：把 UserProfile 与最多 10 个 URR 组（费率组）绑定，用户激活/会话建立时，UNC 在发给 OCS/CHF 的 CCR-Initial 或 Nchf ChargingDataRequest[Initial] 中携带这些 RG，OCS/CHF 据此下发配额。仅对**内容计费用户**生效（PDP 级别计费用户不适用）。核心配置空间有 2 个维度：**URR 组数量与选择**（决定 CCR-I 携带哪几个 RG）和**适用范围**（内容计费 vs PDP 级计费）。

### 配置维度 1：URR 组绑定（参数 USERPROFILENAME + CTXRURRGRPNAME1~10）

| 参数 | 取值 | 作用 | 备注 |
|---|---|---|---|
| `USERPROFILENAME` | 字符串 1~63，不区分大小写 | 用户模板名（主键，前置已建） | 由 [ADD USERPROFILE](task/UNC/20.15.2/0-00094.md) 生成 |
| `CTXRURRGRPNAME1~10` | 字符串 1~31，不区分大小写 | 初始请求 URR 组名称 1~10（前置 ADD URRGROUP 已建） | 由 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md) 生成；最多 10 个 |

> URR 组数量 N 必须 ≤ [ADD DCCTEMPLATE](task/UNC/20.15.2/0-00028.md) 配置的 CCR-I 允许携带 RG 最大个数，否则 N 个 RG 不能全部带上。

典型脚本（融合计费，UserProfile 绑定 1 个 URR 组，激活即申请配额）：
```
SET CTXSTARTRATING: USERPROFILENAME="up_test", CTXRURRGRPNAME1="urrgroup_test";
```
多 RG（绑 3 个费率组，CCR-Initial 同时为 3 个 RG 申请配额）：
```
SET CTXSTARTRATING: USERPROFILENAME="up-test", CTXRURRGRPNAME1="urrgroup1", CTXRURRGRPNAME2="urg2", CTXRURRGRPNAME3="urg3";
```

**步骤位置**：在所属 feature 编排中位于 UserProfile 与 URRGROUP 建立之后，融合计费消息策略配置之前。

## 决策点

### 决策点：URR 组数量与选择（DP 0-03075）

问：该用户模板的用户激活时，CCR-Initial 携带哪几个 RG？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 单 RG | `CTXRURRGRPNAME1` | 单业务场景，1 个费率组 |
| 多 RG（2~10） | `CTXRURRGRPNAME1~N` | 多业务（多费率）场景，最多 10 个 |

> 取值由业务内容计费策略决定。必须配合 ADD DCCTEMPLATE：CCR-I 允许携带的 RG 最大个数 ≥ 本命令指定的 RG 数 N。URR 组来自 ADD URRGROUP，UPURRNAME/DOWNURRNAME 来自 ADD URR。

### 决策点：适用范围（DP 0-03076）

问：该用户模板是否适用本命令？

| 选项 | 适用场景 |
|---|---|
| 内容计费用户 | 本命令生效，CCR-I 携带指定 RG |
| PDP 级别计费用户 | 本命令不生效；CCR-I 携带 UserProfile 或全局下的默认计费属性（UserProfile 优先 > 全局 > 不携带） |

## 约束

- rule-0-03188 · severity=high · 一致性：RG 个数 N 必须 ≤ ADD DCCTEMPLATE 配置的 CCR-I 允许携带 RG 最大个数
- rule-0-03189 · severity=warning · 适用限制：仅对内容计费用户生效，PDP 级别计费用户不适用
