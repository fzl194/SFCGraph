# SET CTXSTARTRATING 原子任务
> 命令静态知识见 [SET CTXSTARTRATING](command/UNC/20.15.2/SET-CTXSTARTRATING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

PGW-C/SMF 侧**用户模板粒度**的初始计费属性绑定：把 UserProfile 与最多 10 个 URR 组（费率组）绑定，用户激活/会话建立时，UNC 在发给 OCS/CHF 的 CCR-Initial 或 Nchf ChargingDataRequest[Initial] 中携带这些 RG，OCS/CHF 据此下发配额。仅对**内容计费用户**生效（PDP 级别计费用户不适用）。

## 配置方法
以 UserProfile 为主键，按业务需要挂 1~10 个 URR 组（每个 URR 组对应一个费率组/RG）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| USERPROFILENAME | 用户模板名（主键，前置已建） | `"up_test"` |
| CTXRURRGRPNAME1~10 | 初始请求 URR 组名称 1~10（前置 ADD URRGROUP 已建） | `"urrgroup_test"` |

典型脚本（融合计费，UserProfile 绑定 1 个 URR 组，激活即申请配额）：
```
SET CTXSTARTRATING: USERPROFILENAME="up_test", CTXRURRGRPNAME1="urrgroup_test";
```
多 RG（绑 3 个费率组，CCR-Initial 同时为 3 个 RG 申请配额）：
```
SET CTXSTARTRATING: USERPROFILENAME="up-test", CTXRURRGRPNAME1="urrgroup1", CTXRURRGRPNAME2="urg2", CTXRURRGRPNAME3="urg3";
```

## 决策点

### URR 组数量与选择
- 问什么：该用户模板的用户激活时，CCR-Initial 携带哪几个 RG。
- 取值由业务内容计费策略决定：单业务 → 1 个 URR 组；多业务（多费率）→ 多个 URR 组（最多 10 个）。
- 必须配合 [ADD DCCTEMPLATE](task/UNC/20.15.2/0-00028.md)：CCR-I 允许携带的 RG 最大个数 ≥ 本命令指定的 RG 数 N，否则 N 个 RG 不能全部带上。
- URR 组来自 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md)，UPURRNAME/DOWNURRNAME 来自 [ADD URR](task/UNC/20.15.2/0-00001.md)。

### 适用范围（内容计费 vs PDP 级计费）
- 问什么：该用户模板是否适用本命令。
- 仅内容计费用户生效；PDP 级别计费用户 CCR-I 携带 UserProfile 或全局下的默认计费属性（UserProfile 优先 > 全局 > 不携带）。

## 约束
- 前置：必须先 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md) 添加计费属性（URR 组），并 [ADD USERPROFILE](task/UNC/20.15.2/0-00094.md) 建用户模板。`severity=high`
- 唯一性：最大记录数 5000。`severity=medium`
- 生效时机：立即生效。`severity=low`
- 查询：配置后需通过 LST USERPROFILE 命令查看（本命令无独立 LST）。`severity=low`
- 一致性：RG 个数 N 必须 ≤ [ADD DCCTEMPLATE](task/UNC/20.15.2/0-00028.md) 配置的 CCR-I 允许携带 RG 最大个数。`severity=high`
- 适用限制：仅对内容计费用户生效，PDP 级别计费用户不适用。`severity=medium`
