# SET SGWCHGMETH 原子任务

> 命令静态知识见 [SET SGWCHGMETH](command/UNC/20.15.2/SET-SGWCHGMETH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET SGWCHGMETH 是 SGW-C 离线计费的两条接入路径之一：本命令按"计费属性（CHARGECHAR）"打开离线计费开关；姐妹命令 SET SGWAPNCHGMETH 则按 APN 打开。两者上层都依赖 SET SGWCHARGECFG（SGW-C 计费基础参数）。本命令的典型行极其简单——固定填 `CHARGECHAR=DEFAULT, OFFLINEFLAG=ENABLE` 即"对 DEFAULT 计费属性使能离线计费"。

**参数取值样例（SGW-C 离线计费使能范式）**

| 参数 | 取值样例 | 默认 | 必选 | 说明 |
|---|---|---|---|---|
| CHARGECHAR | `DEFAULT` | — | required | 计费属性；DEFAULT 表示全网规划统一属性 |
| OFFLINEFLAG | `ENABLE` | — | required | 是否离线计费；ENABLE=使能 |
| CHARGECHARVALUE | — | 无 | conditional | 计费属性值（十六进制，1~6 位），按属性细化定位时填 |
| CCMASK | — | 无 | conditional | 计费属性掩码（十六进制，1~6 位），与 CHARGECHARVALUE 配合做位与匹配 |
| CCPRIORITY | — | 无 | conditional | 计费属性优先级（1~65535），多条记录冲突时排序 |

**典型行（全网规划使能 DEFAULT 属性离线计费）**

```
SET SGWCHGMETH: CHARGECHAR=DEFAULT, OFFLINEFLAG=ENABLE;
```

**典型流程（离线计费使能）**：SET SGWCHARGECFG（基础参数）→ SET SGWCHGMETH（按属性使能，本命令）和/或 SET SGWAPNCHGMETH（按 APN 使能）→ ADD OFCTEMPLATE + SET OFCTHRESHOLD（离线计费模板与阈值）。

## 决策点

### 按计费属性使能 还是 按 APN 使能

SGW-C 离线计费有两条并列的使能路径，由实际业务维度选择，可并存：

| 选项 | 命令 | 适用维度 |
|---|---|---|
| 按计费属性 | SET SGWCHGMETH | 全网 DEFAULT 属性统一开关，或按特定 CHARGECHAR 细化 |
| 按 APN | [SET SGWAPNCHGMETH](task/UNC/20.15.2/0-00199.md) | 针对特定 APN 单独开关 |

### 是否用 CHARGECHARVALUE+CCMASK 做位级细化

仅全网统一开关时填 `CHARGECHAR=DEFAULT, OFFLINEFLAG=ENABLE` 即可，无需 CHARGECHARVALUE/CCMASK。当需要对"某一计费属性下特定属性值子集"精细化使能时，才用 `CHARGECHAR + CHARGECHARVALUE + CCMASK` 做位与匹配定位，并用 CCPRIORITY 排序。

## 约束

- **立即生效，但只对新激活用户生效** · severity=MEDIUM · 已激活用户不立即受影响。
- **最大记录数 17** · severity=MEDIUM · 超过上限新增失败。
- **CHARGECHARVALUE 与掩码位与后必须等于 CHARGECHARVALUE 自身** · severity=HIGH · 不允许配置"掩码位与结果不等于原值"的自相矛盾记录。
- **新记录不得与已有记录位与重叠** · severity=HIGH · 新记录的 `(CHARGECHARVALUE & CCMASK)` 不得等于任一已有记录的 `(CHARGECHARVALUE & CCMASK)`，否则定位冲突。
- **存在系统初始记录** · severity=LOW · 部署后已有初始值，按需覆盖而非重复添加。
- **生效前置**：依赖 SET SGWCHARGECFG 基础参数已配置 · severity=MEDIUM。
