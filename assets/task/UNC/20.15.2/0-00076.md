# ADD SELCHFGBYIMSI 原子任务
> 命令静态知识见 [ADD SELCHFGBYIMSI](command/UNC/20.15.2/ADD-SELCHFGBYIMSI.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是 CHF 选择六种方式中优先级最高者（基于 IMSI），仅用于拨测定位；拨测完应立即删除，避免长期驻留。

## 配置方法
按"目标 IMSI + 主 CHF 组 + 备 CHF 组"配置，一条记录绑一个 IMSI。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| IMSI | 拨测用户的 IMSI（本端规划，14~15 位） | `123456789012345` |
| PRIMARYCHFGRP | 主 CHF 组名（全网规划，需已存在） | `ChfGroup1` |
| SECONDARYCHFGRP | 备 CHF 组名（全网规划，需已存在） | `ChfGroup2` |

典型脚本（拨测指定用户到 ChfGroup1 主备）：
```
ADD SELCHFGBYIMSI: IMSI="123456789012345", PRIMARYCHFGRP="ChfGroup1", SECONDARYCHFGRP="ChfGroup2";
```

## 决策点

### 主备 CHF 组的来源
- 问什么：PRIMARYCHFGRP/SECONDARYCHFGRP 是本端规划还是全网规划（数据规划表取值方法为"全网规划"，但不同部署可能差异）。
- 全网统一规划 → global_planned（证据范式：两个 CHF 组均为全网规划）。
- 本端独立配置 → local_planned。
- 证据默认倾向 global_planned，需按部署确认。

## 约束
- 唯一性：一个 IMSI 对应一条记录，不可重复。`severity=medium`
- 取值范围：IMSI 长度 14~15；CHF 组名长度 0~63。`severity=low`
- 规格上限：最多 30 条记录。`severity=low`
- 生效时机：执行后立即生效。`severity=low`
- 生命周期：拨测完成后建议立即 `RMV` 删除，避免长期影响选 CHF 逻辑。`severity=medium`
- 前置：主备 CHF 组需已通过 ADD TNFGRP/ADD TNFBINDGRP 之类预先建好（CHF 组引用一致性）。`severity=medium`
