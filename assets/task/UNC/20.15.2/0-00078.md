# ADD SELECTCHFGBYCC 原子任务
> 命令静态知识见 [ADD SELECTCHFGBYCC](command/UNC/20.15.2/ADD-SELECTCHFGBYCC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是 CHF 选择链路之一（基于本地配置 CC 选 CHF），优先级低于基于 IMSI/IMSI 号段选择；与 PCF 下发 CHF 信息、NRF 服务发现并存。

## 配置方法
按"CCTYPE + CCVALUE + MASK + PRIORITY + 主备 CHF 组"五元组配置。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTYPE | CC 类型（固定取值） | `VALUE` |
| CCVALUE | Charge Characteristic 值（本端规划或从对端获取，十六进制 1~6 位） | `"0x1111"` |
| MASK | CC 掩码（本端规划，十六进制，按位匹配） | `"0xffff"` |
| PRIORITY | CC 优先级（全网规划，1~65535，越小越高） | `2` |
| PRIMARYCHFGRP | 主 CHF 组（已配置数据中获取，1~63） | `"ChfGroup1"` |
| SECONDARYCHFGRP | 备 CHF 组（已配置数据中获取，1~63） | `"ChfGroup2"` |

典型脚本（VALUE 型 CC 0x1111，主备 ChfGroup1/2）：
```
ADD SELECTCHFGBYCC: CCTYPE=VALUE, CCVALUE="0x1111", MASK="0xffff", PRIORITY=2, PRIMARYCHFGRP="ChfGroup1", SECONDARYCHFGRP="ChfGroup2";
```

## 决策点

### CC 值的来源
- 问什么：CCVALUE 取值方法在"本端规划 / 已配置数据中获取 / 从对端获取 / 全网规划"之间分叉。
- 本地 CC（SMF 本地配）→ 从已配置数据获取。
- UDM 签约 CC → 从对端获取。
- 数据规划表获取方法分布多值，属 decision_driven。

### CHF 组主备来源
- 问什么：主备 CHF 组是本端规划还是引用已存在的 NF 组。
- 证据范式：PRIMARYCHFGRP/SECONDARYCHFGRP 均"已配置数据中获取"（引用 TNFGRP 定义的 CHF 组）。
- 注意：可输入空格或 null 清空参数值（不区分大小写）。

## 约束
- 一致性（强约束）：CCVALUE & MASK 按位与后必须**等于 CCVALUE**，否则不允许配置。`severity=high`
- 唯一性：新增记录的 (CCVALUE & MASK) 不可与已有记录的 (CCVALUE & MASK) 相等（避免按位匹配冲突）。`severity=high`
- 取值范围：CCVALUE/MASK 十六进制 1~6 位；PRIORITY 1~65535；CHF 组名 1~63（可空格/null 清空）。`severity=low`
- 规格上限：最多 20 条记录。`severity=low`
- 生效时机：执行后立即生效。`severity=low`
- 前置：主备 CHF 组需已建好（CHF 组引用一致性）。`severity=medium`
