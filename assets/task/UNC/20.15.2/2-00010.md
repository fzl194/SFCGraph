# 基于业务感知的带宽控制（WSFD-211005）

> 特性静态知识见 [WSFD-211005](feature/UNC/20.15.2/WSFD-211005.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系（必读）**：本特性是带宽族第 5 批。**控制面中转型特性**——UNC（GGSN-C/PGW-C/SMF）不执行实际限速，仅做：① License 前置；② 本地 BWM 规则承载（规则名匹配）；③ 经 N4 接口将带宽控制策略透传 PGW-U/UPF。实际限速对象（BWMSERVICE/BWMCONTROLLER/BWMUSERGROUP/BWMRULE）在 UDG/UPF 侧配置（对应 UDG 特性 [2-00001 GWFD-110311](task/UDG/20.15.2/2-00001.md)）。本特性依赖 PCC 基本功能（[2-00005](task/UNC/20.15.2/2-00005.md)）的 Gx/Npcf 通道获取策略，配置生成 = License 开关 + PCC 基本功能前置 + BWM 本地规则承载链。
>
> 与 UDG 侧对应特性 GWFD-110311（[2-00001](task/UDG/20.15.2/2-00001.md)）的关系：**UNC=控制面（规则承载+N4透传），UDG=用户面（限速执行）**。两侧 RULENAME 须一致；UNC 配 POLICYTYPE=BWM 规则名，UDG 配 BWMSERVICE/BWMCONTROLLER 等限速参数。

## 配置概览

核心对象链（UNC 侧无 BWM 专属对象，全部复用 PCC 族基础设施 + 一条 BWM 本地规则承载链）：

**License 前置**（LKV3TCBSA01）→ **PCC 基本功能 Gx/Npcf 路径**（VPN→逻辑接口→Diameter 本端→PCRF→链路组→PCRF 分组→PCC 开关，或 5G PCF 经 NRF 发现）→ **BWM 本地规则承载链**（RULE(POLICYTYPE=BWM)→USERPROFILE→RULEBINDING→USRPROFGROUP→UPBINDUPG→APNUSRPROFG）→ N4 透传 UPF 执行限速。

关键差异维度：① PCC 模式（动态 vs 本地，决定规则来源）② 代际（2G/3G/4G Gx vs 5G Npcf，决定 PCRF 对接方式）③ 业务类型（P2P/VoIP，决定 RULENAME 规划，非配置参数）。

## 配置流程（步骤编排，每步显示含命令）

1. **激活 BWM License**（前置，本特性专属）：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)
   - 关键参数：LICITEM="LKV3TCBSA01", SWITCH=ENABLE（控制项 82200CQU 基于业务感知的带宽控制）
   - 生效方式：License 加载后允许激活 PDP/PDU 上下文后使用 BWM 功能；关闭时不允许
2. **完成 PCC 基本功能前置**（必前置，本特性依赖其 Gx/Npcf 通道）：完整参见 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md)
   - 关键：须完成 PCC 开关与模板（[1-00014](task/UNC/20.15.2/1-00014.md)）；动态 PCC 须完成 PCRF 对接链（[1-00012](task/UNC/20.15.2/1-00012.md)，仅 2G/3G/4G）+ PCRF 选择（[1-00013](task/UNC/20.15.2/1-00013.md)）
   - License 前置：PCC 基本功能双 License（LKV2PCCBF01 + LKV3W9SPCC11）须已激活
   - **无本特性专属增量参数**：不像 FUP（[2-00007](task/UNC/20.15.2/2-00007.md)）需在 SET PCCFUNC 叠加增量——BWM 的差异化完全由 RULE(POLICYTYPE=BWM) 规则名承载
3. **配置 BWM 本地规则承载链**（本特性核心）：`ADD RULE(POLICYTYPE=BWM)` + `ADD USERPROFILE` + `ADD RULEBINDING` + `ADD USRPROFGROUP` + `ADD UPBINDUPG` + `ADD APNUSRPROFG` → [1-00018](task/UNC/20.15.2/1-00018.md)
   - 关键参数：RULENAME（三网元一致，如 rule_test）；POLICYTYPE=BWM（固定）；PRIORITY=10；UPTYPE=PCC；UPBINDTYPE=DEFAULT；APN=DNN 名
   - 动态 PCC：PCRF/PCF 下发预定义规则名时本地 RULENAME 匹配
   - 本地 PCC：按 PRIORITY 全局优先级匹配本地静态规则（须 SET PCCFUNC HOMEPCCSWITCH=DISABLE）

> **activation 演示范围脚注（R1.4）**：WSFD-211005 仅 1 份 activation（[激活基于业务感知的带宽控制_79619226](evidence/UNC/20.15.2/激活基于业务感知的带宽控制_79619226.md)），演示单一场景：配置 P2P 业务带宽控制规则 rule_test（PRIORITY=10）+ UserProfile up_test + RuleBinding + UsrProfGroup upg_test + 挂接 APN huawei.com。**activation 未演示 License 开关脚本**（步骤前提"完成加载 License"为文字描述，无 `SET LICENSESWITCH` 脚本）——License 开关参数取值来自 [LKV3TCBSA01 License wiki](license/UNC/20.15.2/LKV3TCBSA01.md)（控制项 82200CQU）+ [SET LICENSESWITCH 命令 wiki](command/UNC/20.15.2/SET-LICENSESWITCH.md)，证据充分。**activation 未演示 PCRF 对接/PCC 开关**——前提"完成激活PCC基本功能"指向 [WSFD-109101](task/UNC/20.15.2/2-00005.md) 的 activation（[激活PCC基本功能_72467890](evidence/UNC/20.15.2/激活PCC基本功能_72467890.md) 等），PCC 族配置步骤有独立证据。
>
> **无零证据差异参数**：BWM 的唯一差异参数 POLICYTYPE=BWM 在 activation 脚本首行明确演示；PRIORITY=10、UPTYPE=PCC、UPBINDTYPE=DEFAULT 均有 activation 脚本取值。配置生成按 [1-00018](task/UNC/20.15.2/1-00018.md) 典型脚本产出为有证据基准。

## 决策点（特性级）

| PCC 模式 | 走法 | PCC 开关 | BWM 本地规则 | activation 演示 |
|---|---|---|---|---|
| 动态 PCC（有 PCRF/PCF） | 步骤 1-3（步骤 2 含 [1-00012](task/UNC/20.15.2/1-00012.md)/[1-00013](task/UNC/20.15.2/1-00013.md)/[1-00014](task/UNC/20.15.2/1-00014.md)） | SET PCCFUNC HOMEPCCSWITCH=ENABLE | PCRF/PCF 下发预定义规则名时本地 RULENAME 匹配 | ✓（activation 前提"完成激活PCC基本功能"，规则链脚本演示） |

### DP3：业务类型（决定 RULENAME 规划，非配置参数）

## 约束（特性级 rule）

- **License 前置**（critical）：须激活 LKV3TCBSA01（控制项 82200CQU） — 否则 BWM 功能不生效，PDP/PDU 上下文激活后不能使用带宽控制。
- **依赖 PCC 基本功能**（critical）：须先激活 [WSFD-109101 PCC基本功能](task/UNC/20.15.2/2-00005.md)（License LKV2PCCBF01+LKV3W9SPCC11 + Gx/Npcf 接口对接 + PCC 开关） — 否则 BWM 规则无 PCC 通道承载，PCRF/PCF 策略无法获取与透传。
- **依赖 N4 接口**（critical）：GGSN-C/PGW-C/SMF 通过 N4 接口将带宽控制策略传递给 UPF（[WSFD-011136](feature/UNC/20.15.2/WSFD-011136.md)） — 否则 UNC 侧规则无法透传 UPF 执行。
- **三网元 RULENAME 一致**（critical）：GGSN-C/PGW-C/SMF、PGW-U/UPF、PCRF/PCF 上的 RULENAME 必须一致 — 否则 PCRF/PCF 下发的预定义规则名无法本地匹配，或与 UDG 侧 BWM 限速参数脱节。
- **UNC 侧不配限速参数**（info）：BWMSERVICE/BWMCONTROLLER/BWMUSERGROUP/BWMRULE/APNBINDBWMUSRG/BANDWIDTHMNG 等限速对象在 UDG/UPF 侧配置（[2-00001 GWFD-110311](task/UDG/20.15.2/2-00001.md)） — UNC 侧仅做规则名承载与 N4 透传。
- **加密协议不可 BWM**（warning）：对 SSL/FTPS 等加密协议报文不能应用带宽控制策略 — 仅 P2P/VoIP 等可识别协议（特性 wiki 应用限制）。
- **业务类型识别需 SA**（info）：是否支持非 P2P/VoIP 协议需联系营销支持部确认 — UPF 侧 SA（业务感知）能力决定。
- **对系统性能影响大**（warning）：启用 BWM 需 GGSN-C/PGW-C/SMF 与 PCRF/PCF 持续交互 + 动态处理下发策略，规划时需考虑启用用户的比例（特性 wiki 对系统影响）。
