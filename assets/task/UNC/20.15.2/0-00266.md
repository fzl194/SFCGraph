# 修改Gb模式用户安全参数（MOD GBAUTHCIPH）

> 命令静态知识见 [MOD GBAUTHCIPH](command/UNC/20.15.2/MOD-GBAUTHCIPH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令修改 SGSN 上 2G/3G Gb 模式用户的鉴权加密配置。配置空间主要由两个核心维度决定：**用户范围**（ALL_USER vs SPECIAL_USER）和**安全策略**（NEVER/AUTHONLY/AUTHANDPROTECTED）。

### 配置维度1：用户范围（参数 SUBRANGE）

| 取值 | 作用 | 配套参数 | 典型场景 |
|---|---|---|---|
| ALL_USER | 对所有用户生效 | — | 全局默认安全策略 |
| SPECIAL_USER | 对指定 IMSI 前缀用户生效 | IMSIPRE（必填，5~15 位数字） | 按用户群差异化安全策略 |

> IMSI 前缀按最长匹配查询；相同 IMSI 前缀只能配一条记录。

### 配置维度2：安全策略（参数 SECPLC）

| 取值 | 作用 | 后续参数生效条件 |
|---|---|---|
| NEVER | 不鉴权不保护 | 其余安全参数无效 |
| AUTHONLY | 只鉴权 | 鉴权相关参数生效；加密参数无效 |
| AUTHANDPROTECTED（建议值） | 鉴权并保护 | 所有安全参数均可生效 |

### 配置维度3：鉴权事件（参数 AUTHEVENT，多选，仅 AUTHONLY/AUTHANDPROTECTED 有效）

| 建议开启鉴权的事件 | 说明 |
|---|---|
| INTER_RAU | INTER 位置更新 |
| PTMSI_ATTACH | 使用 PTMSI 的附着 |
| IMSI_ATTACH | 使用 IMSI 的附着 |
| PTMSI_SIG_NOMATCH | PTMSI 签名不匹配 |
| PTMSI_SIG_NOSIG | 不带 PTMSI 签名 |
| SYSTEM_CHANGE_INTRA_RAU | 系统间切换类型的 INTRA 位置更新 |

> 其他可选事件：PROD_RAU（周期 RAU）、SMS_MT/SMS_MO（短消息）、DETACH（分离）、INTRA_RAU、SM_PDP/SM_DEA_PDP（PDP 激活/去激活）、LCS（位置业务）。

### 配置维度4：加密算法（参数 CIPHALG，仅 AUTHANDPROTECTED 有效，多选）

| 取值 | 作用 | License 要求 |
|---|---|---|
| GEA_1 | GEA-1 加密（不安全） | WSFD-103001 / LKV2GEA102 |
| GEA_2 | GEA-2 加密 | WSFD-103002 / LKV2GEA202 |
| GEA_3 | GEA-3 加密 | WSFD-103003 / LKV2GEA302 |

> 算法格式 `GEA_{1|2|3}-{优先级}`，多算法用 `&` 连接；建议配置所有加密算法。加密算法变更后若用户原算法不在配置集合内，会触发重新鉴权。

### 配置维度5：鉴权集管理（参数 AUTHSETSNUMBER / PREGETAUTHSETS / GETAUTHADVTH / CAFTIMES）

| 参数 | 取值 | 建议值 |
|---|---|---|
| RNUM（鉴权集重用次数） | 0~255 | 0 |
| AUTHPERIOD（鉴权周期，受 License 控制） | 0~24 小时 | 24（0 表示不启用周期鉴权） |
| AUTHEVENTTHRESHOLD（鉴权事件上限，受 License 控制） | 0~1023 | 1 |
| AUTHSETSNUMBER（每次取鉴权集数） | 1~5 | 5 |
| PREGETAUTHSETS | NO/YES | YES |
| GETAUTHADVTH（预先取鉴权集门限值） | 0~4 | 0 |
| CAFTIMES（鉴权失败次数临界值） | 3~20 | 3 |

**步骤位置**：在所属 feature（WSFD-103001/103002/103003 GEA 加密、灵活鉴权等）编排中，按需修改。

## 决策点

**用户范围与安全策略选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| ALL_USER + AUTHANDPROTECTED + GEA 组合 | SUBRANGE=ALL_USER, SECPLC=AUTHANDPROTECTED, CIPHALG=GEA_3-0&GEA_2-0&GEA_1-1 | 全局 2G 加密策略 |
| SPECIAL_USER + IMSI 前缀差异化 | SUBRANGE=SPECIAL_USER, IMSIPRE=..., ... | 按用户群差异化 |
| 灵活鉴权（License 控制） | AUTHPERIOD + AUTHEVENTTHRESHOLD | 基于用户群的灵活鉴权特性 |

## 约束

- **非立即生效**（info）：此命令对已签约用户不立即生效，在用户再次触发安全流程后生效 — 后果：需用户重触发
- **部分参数受 License 控制**（critical）：AUTHPERIOD/AUTHEVENTTHRESHOLD 受"基于用户群的灵活鉴权"License 控制；GEA_1/2/3 分别受 WSFD-103001/103002/103003 License 控制 — 后果：未授权配置不生效
- **GEA_1 不安全**（warning）：GEA_1 为不安全加密算法，存在安全风险，请谨慎设置 — 后果：安全性弱
- **加密算法变更触发重鉴权**（info）：加密算法变更后且用户原算法不在配置集合内，会触发新流程的鉴权 — 后果：用户被重新鉴权
- **加密版本协商失败的处理**（warning）：NCIPH 控制加密版本协商失败后是否允许用户不加密接入；LICENSE 不支持加密时 NCIPH 无效 — 后果：可能允许不加密接入
- **建议值偏离可能导致问题**（warning）：各建议值（SECPLC=AUTHANDPROTECTED、RNUM=0、AUTHPERIOD=24 等）偏离可能影响安全/性能 — 后果：安全或性能受损
