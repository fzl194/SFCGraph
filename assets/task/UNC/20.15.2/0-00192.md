# SET PCCFAILACTION 原子任务

> 命令静态知识见 [SET PCCFAILACTION](command/UNC/20.15.2/SET-PCCFAILACTION.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET PCCFAILACTION 配置全局 PCC 故障处理动作，覆盖"选择 PCRF/PCF 失败""Initial 流程故障""Update 流程故障""流量上报失败"四类故障，以及 SCP 故障重选开关。ROLLBACK 类动作需配套指明回滚为 Local PCC（LOCAL_PCC）还是 RADIUS PCC（LOCAL_PCC）或继续用激活时 PCC 策略（INHERIT_PCC）。证据中所有 XFAILACT 均取 ROLLBACK，差异在回滚用户类型。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| SELPEERFAILACT | `DEFAULT` / `ROLLBACK` | 全网规划/本端规划 | 选择 PCRF/PCF 失败动作 |
| SELPEERLOCPCC | `LOCAL_PCC` | 全网规划 | 选择失败回滚为 Local PCC 类型 |
| SELPEERRDSPCC | `LOCAL_PCC` | 全网规划 | 选择失败回滚为 RADIUS PCC 类型 |
| INITIALFAILACT | `ROLLBACK` | 全网规划/本端规划 | Initial 流程故障动作 |
| INITFAILLOCPCC | `LOCAL_PCC` | 全网规划 | Initial 故障回滚为 Local PCC |
| INITFAILRDSPCC | `LOCAL_PCC` | 全网规划 | Initial 故障回滚为 RADIUS PCC |
| UPDATEFAILACT | `ROLLBACK` | 全网规划/本端规划 | Update 流程故障动作 |

## 决策点

### Update 流程故障回滚类型（LOCAL_PCC vs INHERIT_PCC）

UPDFAILLOCPCC 是核心决策点：
- `LOCAL_PCC`（PCC 基本功能场景）→ 通知 UPF 删除激活时安装的动态 PCC 策略，安装本地静态 PCC 策略。
- `INHERIT_PCC`（语音 PCF/PCRF Bypass 场景）→ 继续使用激活时安装的 PCC 策略，保证语音专有 QoS Flow/专有承载不丢策略。

需根据是否需要保留语音专有 QoS 来决定。证据中获取方法分布全网规划/本端规划皆有。

### SCP 故障重选开关是否启用（SCPFAILOVERSW）

仅当 SMF 与 PCF 采用 Model C/D 通信模式（经 SCP 中转）时，需配 SCPFAILOVERSW=SCP_FAILOVER 启用 SCP 故障重选。直连模式不配。

### LOCAL_PCC 用户类型含义

LOCAL_PCC 代表本地 PCC 用户：有 PCC 功能、占用 PCC 基本功能 License，但不与 PCRF/PCF 交互。EPS 场景下支持业务触发的 QoS 保证。回滚决策即是否让故障用户降级为本地 PCC 继续业务。

## 约束

- **立即生效** · 执行后立即生效。
- **最大 1 条记录** · 全局唯一配置。
- **优先级低于 SET FHBYPASS** · severity=HIGH · Gx 接口回滚策略中，当 SET FHBYPASS 的 GXERRRC=ENABLE 时，SET FHBYPASS 命令优先级高于本命令。FHBYPASS 为故障紧急处理，须客户书面认可后方可使用。
- **优先级低于 ADD RESULTCODECTRL** · severity=MEDIUM · 异常返回码控制优先于本命令。
- **配套 ADD PCRFGROUP.FAILOVERSW** · severity=MEDIUM · failover 功能需 PCRFGROUP 中 FAILOVERSW=ENABLE 才生效；否则按本命令和 ADD PCCTEMPLATE 配置处理。
- **与 SET PCCTIMER 配套** · severity=LOW · 回滚后的在线保持时长由 SET PCCTIMER 配置。
- **系统初始记录** · 命令存在系统初始记录，缺省使用默认值。
