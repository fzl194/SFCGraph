# 配置PCC故障处理（SET PCCFAILACTION）

> 命令静态知识见 [SET PCCFAILACTION](command/UNC/20.15.2/SET-PCCFAILACTION.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET PCCFAILACTION 配置全局 PCC 故障处理动作，覆盖"选择 PCRF/PCF 失败""Initial 流程故障""Update 流程故障""流量上报失败"四类故障，以及 SCP 故障重选开关。全局唯一一条记录（最大记录数 1）。ROLLBACK 类动作需配套指明回滚为 Local PCC（LOCAL_PCC）还是 RADIUS PCC（LOCAL_PCC）或继续用激活时 PCC 策略（INHERIT_PCC）。证据中所有 XFAILACT 均取 ROLLBACK，差异在回滚用户类型。核心配置空间 3 个维度：**故障阶段动作**（按阶段分支）、**回滚用户类型**（条件联动）、**SCP 重选开关**（独立分支）。

### 配置维度 1：故障阶段动作（按阶段分支）

| 阶段 | 参数 | 取值 | 作用 | 典型场景 |
|---|---|---|---|---|
| 选择 PCRF/PCF 失败 | `SELPEERFAILACT` | `DEFAULT` / `ROLLBACK` / `FORBIDDEN` | DEFAULT=选缺省 PCRF/PCF 或降级本地，ROLLBACK=回滚，FORBIDDEN=激活失败 | PCC 基本功能取 DEFAULT；语音 Bypass 取 ROLLBACK |
| Initial 流程故障 | `INITIALFAILACT` | `ROLLBACK` / `FORBIDDEN` | ROLLBACK=回滚，FORBIDDEN=按激活失败处理 | PCC 基本功能/语音 Bypass 均取 ROLLBACK |
| Update 流程故障 | `UPDATEFAILACT` | `ROLLBACK` / `TERMINATE` / `CONTINUE` | ROLLBACK=回滚，TERMINATE=去活，CONTINUE=继续交互 | 全场景普遍取 ROLLBACK |
| 流量上报失败 | `UMRFAILACT` | `INHERIT_UPDATEFAILACT` / `TERMINATE` | 继承 Update 动作 / 去活整个用户 | 按是否需严格去活规划 |

> `ROLLBACK`（回滚）= 通知 UPF 删除激活时安装的动态 PCC 策略，改装本地静态 PCC 策略；`FORBIDDEN`/`TERMINATE`=更激进，按激活失败或去活处理。

### 配置维度 2：回滚用户类型（条件可选，ROLLBACK 时联动）

| 参数 | 取值 | 作用 | 典型场景 |
|---|---|---|---|
| `SELPEERLOCPCC` | `LOCAL_PCC` | 选择失败回滚为 Local PCC 用户 | 证据统一取 LOCAL_PCC |
| `SELPEERRDSPCC` | `LOCAL_PCC` | 选择失败回滚为 RADIUS PCC 用户 | 证据统一取 LOCAL_PCC（RADIUS PCC 也降级为本地 PCC） |
| `INITFAILLOCPCC` | `LOCAL_PCC` | Initial 故障回滚为 Local PCC | 证据统一取 LOCAL_PCC |
| `INITFAILRDSPCC` | `LOCAL_PCC` | Initial 故障回滚为 RADIUS PCC | 证据统一取 LOCAL_PCC |
| `UPDFAILLOCPCC` | `LOCAL_PCC` / `INHERIT_PCC` | Update 故障回滚类型 | PCC 基本功能取 LOCAL_PCC；语音 Bypass 取 INHERIT_PCC 保语音专有 QoS |

> LOCAL_PCC 代表本地 PCC 用户：有 PCC 功能、占用 PCC 基本功能 License，但不与 PCRF/PCF 交互，EPS 场景下支持业务触发的 QoS 保证。INHERIT_PCC 表示 Update 故障时继续使用激活时安装的 PCC 策略，保证语音专有 QoS Flow/专有承载不丢策略——见决策点 DP 0-03123。

### 配置维度 3：SCP 故障重选开关（独立分支）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **SCP_NOT_FAILOVER**（默认） | SCP 故障时不进行重选 | SMF 与 PCF 直连，或 Model A/B 通信 |
| **SCP_FAILOVER** | SCP 故障时进行 SCP 重选 | SMF 与 PCF 采用 Model C/D（经 SCP 中转） |

典型脚本：
```
SET PCCFAILACTION:SELPEERFAILACT=ROLLBACK,INITIALFAILACT=ROLLBACK,UPDATEFAILACT=ROLLBACK;
SET PCCFAILACTION:SELPEERFAILACT=ROLLBACK,INITIALFAILACT=ROLLBACK,UPDATEFAILACT=ROLLBACK,SCPFAILOVERSW=SCP_FAILOVER;
```

**步骤位置**：在所属 feature 编排中通常位于"PCC 故障处理配置"——紧随 `SET PCCFUNC`（PCC 总开关）和 `ADD PCRFGROUP`（PCRF 组/Faiolover）之后，`SET PCCTIMER`（Holding Time）之前。

## 决策点：Update 流程故障回滚类型（DP 0-03123）

问：Update 故障时回滚为本地 PCC（删动态策略装静态），还是继承 PCC（保留激活时策略）？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| LOCAL_PCC | `UPDATEFAILACT=ROLLBACK` + `UPDFAILLOCPCC=LOCAL_PCC` | PCC 基本功能场景，通知 UPF 删动态策略装本地静态 PCC |
| INHERIT_PCC | `UPDATEFAILACT=ROLLBACK` + `UPDFAILLOCPCC=INHERIT_PCC` | 语音 PCF/PCRF Bypass 场景，保留语音专有 QoS Flow/专有承载不丢策略 |

### SCP 故障重选开关（DP 0-03124）

问：SMF 与 PCF 通信模式是否经 SCP（Model C/D），需启 SCP 重选？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不启 SCP 重选 | 不配或 `SCPFAILOVERSW=SCP_NOT_FAILOVER` | SMF 与 PCF 直连，或 Model A/B |
| 启 SCP 重选 | `SCPFAILOVERSW=SCP_FAILOVER` | Model C/D 经 SCP 中转，语音 PCF Bypass N7 场景 |

## 约束

- **SET PCCFAILACTION 最大记录数为 1**（critical，rule-0-03306）：SET PCCFAILACTION 最大记录数为 1，全局唯一配置，超规格新增失败
- **Gx 接口优先级低于 SET FHBYPASS**（warning，rule-0-03307）：Gx 接口优先级低于 SET FHBYPASS：当 `SET FHBYPASS.GXERRRC=ENABLE` 时，SET FHBYPASS 命令优先级高于本命令；FHBYPASS 为故障紧急处理，须客户书面认可后方可使用
- **failover 功能需配套 ADD**（warning，rule-0-03308）：failover 功能需配套 `ADD PCRFGROUP.FAILOVERSW=ENABLE` 才生效，否则按本命令和 ADD PCCTEMPLATE 配置处理
- **异常返回码控制优先于本命令**（warning，rule-0-03309）：异常返回码控制优先于本命令：`ADD RESULTCODECTRL` 配置的返回码控制先于本命令生效
- **回滚后的在线保持时长由 SET PCCTIMER 配置配套**（info，rule-0-03310）：回滚后的在线保持时长由 `SET PCCTIMER` 配置配套，本命令不控制保持时长
