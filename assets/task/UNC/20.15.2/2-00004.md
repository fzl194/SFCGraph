# 内容计费基本功能（WSFD-109002）

> 特性静态知识见 [WSFD-109002](feature/UNC/20.15.2/WSFD-109002.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系（必读）**：本特性是计费族第 4 批。内容计费**本身无专属配置命令**——activation 文档原文"内容计费基于离线计费/在线计费/融合计费的基础，激活内容计费 License，配置内容计费使用的 URR，关联到对应 Rule 绑定，操作同融合计费费时识别"。即：内容计费 = License 前置 + 复用承载计费方式（在线/离线/融合）的费率标识链。配置生成仅编 1 步 License + 引用承载计费方式的费率标识链 compound。

## 配置概览

核心对象链（与融合计费费率标识链完全一致，无内容计费专属对象）：

**URR**（费率规则：RG/SID + 计费模式 ONLINE/OFFLINE + 统计类型 VOLUME/DURATION/EVENT/FREE）→ **URRGROUP**（上下行 + 在线/离线组合）→ **PCCPOLICYGRP**（绑 URR 组）→ **RULE**（POLICYTYPE=PCC，绑 PCC 策略组）→ **RULEBINDING**（绑 Rule 到 UserProfile）→ **USERPROFILE** + **SET URRGRPBINDING**（缺省 URR 组）。

内容计费与"纯离线/在线/融合计费"的差异**不在命令层**，而在**语义层**：
1. **业务识别在用户面**：UPF/PGW-U 通过 SA（Service Awareness）解析业务流，匹配计费规则后标记 URR ID。UNC（SMF/PGW-C）只做计费规则管理与计费信息上报——**不配 FILTER/L7FILTER/FLOWFILTER**（这些在 UDG 侧或由 PCF 动态下发）。
2. **License 前置**：内容计费费率标识的配置不依赖 License 即可执行，但**费率标识需在打开内容计费 License 后生效**（特性概述原文）。激活内容计费 = `SET LICENSESWITCH`（LKV3W9BCC12）。
3. **承载计费方式复用**：内容计费通过在线（OCS）/离线（CG）/融合（CHF）之一实现，费率标识链结构复用承载计费方式的 compound（典型 = 融合 [1-00009](task/UNC/20.15.2/1-00009.md)）。

## 配置流程（步骤编排，每步显示含命令）

1. **激活内容计费 License**（前置，本特性唯一专属步骤）：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)
   - 关键参数：LICITEM="LKV3W9BCC12", SWITCH=ENABLE
   - 前提：License 文件已购买并加载该项；费率标识配置本身不依赖此开关，但**生效依赖**此开关
2. **完成承载计费方式的基础配置**（按 DP1 选一，内容计费建立在此基础上）：
   - 在线计费基础：参见 [2-00001 离线计费](task/UNC/20.15.2/2-00001.md) 同族的在线计费特性（WSFD-109001，UNC 侧待建）— `SET CHARGECTRL` 使能在线开关 + Gy 接口对接
   - 离线计费基础：[2-00001 离线计费](task/UNC/20.15.2/2-00001.md) — OFCTemplate 模板 + CG 对接
   - 融合计费基础：[2-00003 融合计费](task/UNC/20.15.2/2-00003.md) — CCT 模板 + CHF 对接
3. **配置费率标识链**（内容计费核心，复用承载计费方式的费率标识链 compound）：`ADD URR` + `ADD URRGROUP` + `ADD PCCPOLICYGRP` + `ADD RULE` + `ADD RULEBINDING` + `SET URRGRPBINDING` + `ADD USERPROFILE` → [1-00009](task/UNC/20.15.2/1-00009.md)
   - 内容计费语义：每条业务（如 URL/IMS/any）配一组在线+离线双 URR，RG 区分业务费率；异常场景配 FREE URR 作缺省
4. **UserProfile 组与 APN 绑定**（用户级挂接，复用承载计费方式的 UserProfile 绑定结构）：`ADD USRPROFGROUP` + `ADD UPBINDUPG` + `ADD APN` + `ADD APNUSRPROFG` → [1-00002](task/UNC/20.15.2/1-00002.md)（结构复用，无内容计费专属差异）
5. **配置配额耗尽动作**（在线 RG 可选）：`ADD QUOTAEXHAUSTACT` → [0-00066](task/UNC/20.15.2/0-00066.md)
   - 在线 RG 配额耗尽后动作（BLOCK/REDIRECT/FORWARD），仅在线/融合计费场景

> **activation 演示范围脚注（R1.4）**：WSFD-109002 activation 文档（`激活内容计费_74013177.md`）**本身无配置步骤脚本**——原文步骤 2"参见 [离线计费的费时识别](WSFD-109001) 或 [融合计费费时识别](WSFD-011206)"，数据示例与脚本均指向 WSFD-011206 的"配置融合计费费率标识"证据（[93360308](evidence/UNC/20.15.2/配置融合计费费率标识_93360308.md)）。**内容计费无零证据差异参数**——其差异化（业务类型识别）完全由费率标识链的 RG/SID 体现，已在 [1-00009](task/UNC/20.15.2/1-00009.md) 演示。配置生成按 [1-00009](task/UNC/20.15.2/1-00009.md) 典型脚本产出为有证据基准。

| 配置粒度 | 走法 | 说明 |

## 约束（特性级 rule）

- **License 生效前置**（critical）：内容计费费率标识配置本身不依赖 License，但**费率标识需在打开内容计费 License（LKV3W9BCC12）后生效** — 否则配了费率规则也不产生内容计费话单。
- **依赖承载计费方式**（critical）：内容计费必须建立在在线/离线/融合计费之一的基础上 — 须先完成对应承载计费特性的配置（使能开关 + 接口对接 + 模板）。
- **加密协议不可内容计费**（warning）：对 SSL/FTPS 等加密协议报文不能应用内容计费特性 — 特性 wiki 应用限制。
- **URRID/RULENAME 跨网元一致**（critical）：UNC（SMF/PGW-C）与 UDG（UPF/PGW-U）的 URRID/USAGERPTMODE/METERINGTYPE/RULENAME/POLICYTYPE/POLICYNAME 须一致 — 否则计费统计不一致。
- **USERPROFILENAME 跨 PCF/SMF/UPF 一致**（critical）：不一致时 PCF 下发的策略无法生效。
- **UNC 侧不配过滤条件**（info）：FILTER/L7FILTER/FLOWFILTER/FLTBINDFLOWF/PROTBINDFLOWF 由 UDG 侧或 PCF 动态下发 — UNC 侧 RULE 不配 FILTERINGMODE/FLOWFILTERNAME。
- **建议绑缺省 URR 组**（info）：每个 UserProfile 绑 DFTURRGRPNAME（异常场景 URR 组），未匹配规则的流量按缺省组计费。
- **规格限制**（info）：整机内容计费 rule ≤ 5000，URR ≤ 10000。
