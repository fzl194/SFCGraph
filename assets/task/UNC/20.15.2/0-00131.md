# MOD URR 原子任务
> 命令静态知识见 [MOD URR](command/UNC/20.15.2/MOD-URR.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：PGW-C、SMF。本命令修改已存在的使用量上报规则条目，是计费三件套（URR→URRGROUP→PCCPOLICYGRP）调整环节，与 ADD/RMV/LST URR 构成完整 CRUD。

## 配置方法
按 URRNAME 定位条目，按计费场景（在线/离线、流量/时间/事件配额）修改对应参数子集。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| URRNAME | 使用量上报规则名称（必选，本地规划，1~31 字符） | `urr1` |
| URRID | URR 标识（可选，0~2147483646） | `1` |
| USAGERPTMODE | 使用量上报模式（可选，枚举：ONLINE/OFFLINE） | `ONLINE` |
| RG / SID / UPSID / DOWNSID | 离线计费组/业务标识（条件，0~4294967294） | `RG=100` |
| ONLINERG / ONLINESID | 在线计费组/业务标识（条件） | `ONLINERG=200` |
| OFFMETERINGTYPE / ONLMETERINGTYPE | 离线/在线计费统计类型（条件，枚举） | 按产品定义 |
| OFFCOMPOUNDTYPE / ONLCOMPOUNDTYPE | 离线/在线计费标识组成类型（条件，枚举） | 按产品定义 |
| TIMERSUVALUE | 时间配额值秒（条件，对端协商，0~3600000） | `3600` |
| VOLUMERSUVALUE | 流量配额值字节（条件，对端协商，0~4294967295） | `1048576` |
| EVENTRSUVALUE | 事件配额值次（条件，对端协商，0~5000） | `100` |
| OFCSRVTMPLNAME | 离线业务模板名（条件，1~63 字符） | `svctmpl1` |
| MONITORINGKEY | 监控属性值（条件，1~31 字符） | `mk1` |
| REDIRENDTOKEN | Gy 重定向结束 Token（条件，1~31 字符） | `token1` |
| N40AGESW | N40 接口是否允许老化（条件，global_planned，枚举） | `YES` |

典型用法：ADD URR 建条目后，按数据规划修改计费 RG/统计类型/配额；调测融合计费流量计费（WSFD-011206）时与 MOD URRGROUP、MOD PCCPOLICYGRP、MOD RULE 联动按规划值重新绑定 RG。

## 决策点

### 计费模式决定可改字段子集（USAGERPTMODE 多值）
USAGERPTMODE 取 ONLINE / OFFLINE 时，被计费属性记录引用后可改字段不同：ONLINE 仅允许改 ONLMETERINGTYPE，OFFLINE 仅允许改 OFFMETERINGTYPE。取值由业务在线/离线计费方式决定，数据规划阶段按场景（融合计费 vs 纯离线）选定。

### 配额类型多值（流量/时间/事件）
TIMERSUVALUE / VOLUMERSUVALUE / EVENTRSUVALUE 三类配额按业务计费维度（按流量、按时长、按次数）选用其一或组合，数据来源为对端协商，取值依赖对接 OCS/在线计费系统的配额协商结果。

## 约束
- **立即生效**：命令执行后立即生效，无延迟窗口。
- **被引用时受限修改**：若引用该 URR 的计费属性记录已存在，USAGERPTMODE=ONLINE 时仅可改 ONLMETERINGTYPE，OFFLINE 时仅可改 OFFMETERINGTYPE。
- **条目存在性前置**：MOD 前需先 ADD URR 建立条目，否则修改失败。
- **MONITORINGKEY 无效值**：4294967295 为无效值，初始化时 MonitoringKey 不能取该值。
- **N40AGESW 取自 global_planned**：非本端规划，按整网口径取值。
