# SET RGRESCTRL 原子任务
> 命令静态知识见 [SET RGRESCTRL](command/UNC/20.15.2/SET-RGRESCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令为全局单例（仅 1 条记录），用于在线/离线计费两条线分别使能 RG 老化并配老化时长。背景：用户访问业务匹配某 RG 后，即使后续不再访问，长时间在线用户在每次上报用量时仍会携带该 RG 用量（为零），导致 N40 接口消息膨胀与 RG/URR 资源浪费。RG 老化可在 QHT 未使能或 QHT=0 时保证已停用 RG 老化、携带 Trigger=Final。

## 配置方法

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| ONLRGAGESW | 在线计费 RG 老化开关（本端规划） | `ENABLE` |
| ONLAGETIMER | 在线计费 RG 老化时长（分钟） | `20` |
| ONLBLKTIMER | 在线计费全局业务阻塞/重定向处理时间间隔（分钟），0 表示阻塞时长无效 | `0` |
| OFLRGAGESW | 离线计费 RG 老化开关（本端规划） | `ENABLE` |
| OFLAGETIMER | 离线计费 RG 老化时长（分钟） | `20` |
| EXCSRVLMTACT | RG 超出业务最大规格处理动作 | `BLOCK`（阻塞对应业务） |

典型脚本（在线/离线均使能老化，时长 20 分钟，阻塞时长无效）：
```
SET RGRESCTRL: ONLRGAGESW=ENABLE,ONLAGETIMER=20,ONLBLKTIMER=0,OFLRGAGESW=ENABLE,OFLAGETIMER=20;
```

## 决策点

### 在线/离线老化是否分别使能
- 问什么：在线计费和离线计费两条线是否各自使能 RG 老化。
- 融合计费场景下两条线通常都配（如样例均 ENABLE）。
- 老化开关与老化时长为 conditional 联动：使能老化需同时配老化时长。

### 阻塞时长是否生效
- 问什么：ONLBLKTIMER 是否需要全局业务阻塞/重定向时长。
- 配 0 → 阻塞时长无效，直接按 ONLAGETIMER 老化。
- 配非 0 → 从阻塞/重定向开始经过该时长后才进行 RG 老化。

### 与 QHT、流量计费信息的互斥
- RG 老化时长通过业务级 QHT 下发到 UDG，UDG 上报业务 QHT 时决策是否老化。
- 当 ADD DCCTEMPLATE 使能 QUOTATOTAL 或 PRIVATEATTR（Gy 接口携带流量计费信息）时，RG 老化功能不生效。

## 约束
- 立即生效（命令执行后立即生效）。`severity=medium`
- 老化时长取值：0 或 10~1440 分钟（ONLAGETIMER/OFLAGETIMER/ONLBLKTIMER）。`severity=low`
- QHT 依赖：RG 老化时长经业务级 QHT 下发 UDG，依赖 UDG 上报。`severity=medium`
- 互斥-流量计费信息：Gy 接口使能 QUOTATOTAL 或 PRIVATEATTR（ADD DCCTEMPLATE）时，RG 老化不生效。`severity=high`
- conditional：ONLRGAGESW/OFLRGAGESW 使能时，对应 AGETIMER 必须配置。`severity=medium`
- 单例：全局唯一记录。`severity=low`
