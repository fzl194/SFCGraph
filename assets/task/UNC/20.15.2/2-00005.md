# PCC基本功能（WSFD-109101）

> 特性静态知识见 [WSFD-109101](feature/UNC/20.15.2/WSFD-109101.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系**：计费族第 5 批（最后）。PCC 基本功能是**策略控制基础设施**——与计费族（离线/在线/融合/内容计费）正交：PCC 提供"从 PCRF/PCF 获取策略"的通道，计费特性提供"费率标识与上报"的规则。内容计费（[2-00004](task/UNC/20.15.2/2-00004.md)）的 RULE/PCCPOLICYGRP/USERPROFILE 复用本特性的业务策略组合 atom 子集（共享命令，非复用计费族 compound）。本特性与计费族复用 [1-00009](task/UNC/20.15.2/1-00009.md) 费率标识链的**部分 atom**（ADD PCCPOLICYGRP/ADD RULE/ADD RULEBINDING/ADD USERPROFILE），但 PCCPOLICYGRP 的语义不同（计费族绑 URRGROUP，PCC 可绑 QOSPROP 或都不绑），故本特性直接引 atom 而非整体复用 1-00009。

## 配置概览

核心对象链（按代际与 PCC 模式分两条骨架）：

**2G/3G/4G 动态 PCC（Gx 接口对接 PCRF）**：
**VPN** → **Gx 逻辑接口** → **Diameter 本端信息** → **PCRF** → **Diameter 链路组/链路** → **PCRF 分组（主备/负荷分担）** → **PCRF 选择（号段/APN/全局缺省）** → **PCC 开关（全局+APN 级）** → **PCC 模板** → （可选）**PCCPOLICYGRP + RULE + USERPROFILE + RULEBINDING**（预定义规则/规则组的本地承载）。

**5G 动态 PCC（Npcf 接口，PCF 经 NRF 发现）**：
无需 Diameter 对接链——本端 SBIAPLE 注册到 NRF + 对端 PCF 注册到 NRF；**PCC 开关 + PCC 模板（5G 容灾参数）** → **PCF 选择（PCFSELPLCY for AMF / PCFSELECTMODE for SMF）** → （可选）**PCCPOLICYGRP + RULE + USERPROFILE + RULEBINDING**。

**本地 PCC（无 PCRF）**：仅 **PCC 开关（DISABLE）+ PCC 模板** + **本地静态规则 + USERPROFILE + RULEBINDING**，按全局优先级匹配。

关键差异维度：① 代际（2G/3G/4G Gx vs 5G Npcf）② PCC 模式（动态 vs 本地）③ PCRF 分组工作模式（主备 vs 负荷分担）④ 规则下发范围（RULERANGE/PROFILERANGE ALL/CENTRAL/LOCAL + DNAI 绑定）。

## 配置流程（步骤编排，每步显示含命令）

1. **激活 PCC License**（前置）：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)
   - 关键参数：LICITEM="LKV2PCCBF01", SWITCH=ENABLE + LICITEM="LKV3W9SPCC11", SWITCH=ENABLE（两项 License）
2. **配置 PCRF 对接链（仅 2G/3G/4G 动态 PCC；5G/本地 PCC 跳过）**：`ADD VPNINST` + `SET CONCENPOINT` + `ADD LOGICIP` + `ADD LOGICINF` + `ADD DIAMLOCINFO` + `ADD PCRF` + `ADD DIAMPEERADDR` + `ADD DIAMCONNGRP` + `ADD DIAMCONNECTION` → [1-00012](task/UNC/20.15.2/1-00012.md)
3. **配置 PCRF 选择与分组（仅 2G/3G/4G 动态 PCC）**：`ADD IMSIMSISDNSEG` + `ADD PCRFGROUP` + `ADD PCRFBINDGRP` + `SET MASTERPCRF` + `ADD GLBPCRFGROUP` + `SET DFTGLBPCRFGRP` + `ADD PCRFGRPBNDAPN` → [1-00013](task/UNC/20.15.2/1-00013.md)
4. **配置 PCC 开关与模板**：`SET PCCFUNC` + `ADD PCCTEMPLATE` + `SET APNPCCFUNC`（License 前置已在步骤 1）→ [1-00014](task/UNC/20.15.2/1-00014.md)
5. **（可选）配置策略接口选择方式（混合代际场景）**：`SET POLICYMODE` → [0-00197](task/UNC/20.15.2/0-00197.md)
   - 关键参数：POLICYMODE=Gx/Npcf（全局策略接口选择）；可选 `ADD APNPOLICYMODE`（APN 级）+ `ADD PCCCHGMODEBYPCFID`（按 PCF 实例标识）
6. **（可选，5G AMF 侧）配置 AMF 选 PCF + AM/UE 策略**：`ADD PCFSELPLCY` + `ADD AMUEPLCYCTRL` + `ADD NGMMSUBDATA` + `SET AMFPEERSELFUNC` → [0-00054](task/UNC/20.15.2/0-00054.md) · [0-00003](task/UNC/20.15.2/0-00003.md) · [0-00047](task/UNC/20.15.2/0-00047.md) · [0-00147](task/UNC/20.15.2/0-00147.md)
7. **（可选，5G PCF 服务区选择）配置 PCF 业务服务区**：`ADD PCFSSCOPE` + `ADD USRTAIRANGE` + `ADD PCFSSCOPEBIND` → [0-00055](task/UNC/20.15.2/0-00055.md) · [0-00097](task/UNC/20.15.2/0-00097.md) · [0-00056](task/UNC/20.15.2/0-00056.md)
8. **（可选）配置本地规则与业务策略组合**（PCRF 下发预定义规则/规则组时需本地承载；本地 PCC 必配）：
   - 配 PCC 策略组（绑 URR 组或 QoS 属性）：`ADD PCCPOLICYGRP` → [0-00052](task/UNC/20.15.2/0-00052.md)
   - 配本地 Rule（POLICYTYPE=PCC，绑 PCC 策略组）：`ADD RULE` + `ADD RULEBINDDNAI`（辅锚点 DNAI 绑定，可选）→ [0-00071](task/UNC/20.15.2/0-00071.md) · [0-00072](task/UNC/20.15.2/0-00072.md)
   - 配 UserProfile + 绑 Rule：`ADD USERPROFILE` + `ADD USRPROBINDDNAI`（辅锚点，可选）+ `ADD RULEBINDING` → [0-00094](task/UNC/20.15.2/0-00094.md) · [0-00095](task/UNC/20.15.2/0-00095.md) · [0-00073](task/UNC/20.15.2/0-00073.md)
9. **（可选）配置业务策略组合挂接到 APN**：`ADD USRPROFGROUP` + `ADD UPBINDUPG` + `ADD APNUSRPROFG` → [0-00096](task/UNC/20.15.2/0-00096.md) · [0-00089](task/UNC/20.15.2/0-00089.md) · [0-00008](task/UNC/20.15.2/0-00008.md)
10. **（可选，本地 PCC 多 UserProfile）** `ADD PCCPBINDUPG` → [0-00051](task/UNC/20.15.2/0-00051.md)

> **activation 演示范围脚注（R1.4）**：WSFD-109101 有两份 activation（5G 版 `激活PCC基本功能_72467890.md` + 2G/3G/4G 版 `配置动态PCC功能_30805098.md`/`配置本地PCC功能_30805097.md`/`配置与PCRF对接数据_30805096.md`）。5G 版演示三个任务（动态 PCC 全局+APN 级使能、主备容灾、基于 DNN+SERVINGSCOPE 选 PCF），**全部走 Npcf 接口，无 Gx Diameter 配置脚本**。2G/3G/4G 版演示 PCRF 分组（负荷分担 pcrf_group_1 + 主备 pcrf_group_2）、号段选择、APN 绑 PCRF 组。两份 activation 互补覆盖：5G 版覆盖 Npcf/PCF/AMF 策略侧，2G/3G/4G 版覆盖 Gx/PCRF Diameter 侧。**无零证据差异参数**——所有步骤均有对应 activation 脚本演示。

## 决策点（特性级）

### DP1：代际（决定接口类型与 PCRF 对接方式）

| 代际 | 走法 | PCRF 对接 | PCRF 选择 | License |
|---|---|---|---|---|
| 2G/3G/4G（Gx 接口） | 步骤 1-4（含 1-00012/1-00013）+ 可选 5-10 | [1-00012](task/UNC/20.15.2/1-00012.md) Diameter 对接链 | [1-00013](task/UNC/20.15.2/1-00013.md) PCRF 分组+号段/APN/缺省 | LKV2PCCBF01 + LKV3W9SPCC11 |
| 5G（Npcf 接口） | 步骤 1+4（跳过 1-00012/1-00013）+ 步骤 6-7（AMF/SMF PCF 选择）+ 可选 8-10 | 无 Diameter（PCF 经 NRF 服务化发现，[ADD SBIAPLE](task/UNC/20.15.2/0-00074.md) 本端注册） | [ADD PCFSELPLCY](task/UNC/20.15.2/0-00054.md)（AMF）+ SET APNPCCFUNC 的 PCFSELECTMODE（SMF，DNN/SERVINGSCOPE/NFLOC/SUPI） | 同上 |

### DP2：PCC 模式（动态 vs 本地）

| PCC 模式 | 走法 | PCC 开关 | 本地规则 | activation 演示 |
|---|---|---|---|---|
| 动态 PCC（有 PCRF/PCF） | 步骤 1-4（动态）+ 可选 8-10（PCRF 下发预定义规则/规则组时本地承载） | SET PCCFUNC HOMEPCCSWITCH=ENABLE | 仅 PCRF 下发预定义规则时配本地 RULE/USERPROFILE | ✓（5G 版任务一/三 + 2G/3G/4G 版配置动态PCC功能） |
| 本地 PCC（无 PCRF） | 步骤 1+4（本地，跳过 1-00012/1-00013）+ 步骤 8-10（必配本地静态规则） | SET PCCFUNC HOMEPCCSWITCH=DISABLE | 必配 ADD RULE/USERPROFILE/RULEBINDING，按全局优先级匹配 | ✓（2G/3G/4G 版配置本地PCC功能） |

### DP3：PCRF 分组工作模式（仅 2G/3G/4G 动态 PCC）

| 工作模式 | 走法 | 关键参数 |
|---|---|---|
| 负荷分担（LOAD_BALANCE） | [1-00013](task/UNC/20.15.2/1-00013.md) ADD PCRFBINDGRP 配 WEIGHT | ADD PCRFGROUP LOADBALANCEMODE=LOAD_BALANCE；ADD PCRFBINDGRP WEIGHT=权重 |
| 主备（MASTER_SLAVE） | [1-00013](task/UNC/20.15.2/1-00013.md) SET MASTERPCRF 设主用 | ADD PCRFGROUP LOADBALANCEMODE=MASTER_SLAVE；SET MASTERPCRF MASTERPCRF=主机名 |

### DP4：PCRF 选择粒度（仅 2G/3G/4G 动态 PCC）

| 选择粒度 | 命令 | 优先级 |
|---|---|---|
| APN 下绑 PCRF 组（不区分号段） | ADD PCRFGRPBNDAPN DEFAULTFLAG=DEFAULT | 最高 |
| APN 下按号段绑 PCRF 组 | ADD PCRFGRPBNDAPN DEFAULTFLAG=IMSI_MSISDN_SEG + IMSIMSISDNSEG | 次高 |
| 全局号段绑 PCRF 组 | ADD GLBPCRFGROUP + PRIORITY | 中 |
| 全局缺省 PCRF 组 | SET DFTGLBPCRFGRP | 兜底 |

### DP5：规则下发范围（5G/辅锚点场景）

| 下发范围 | 参数 | 适用 |
|---|---|---|
| 主+辅锚点 UPF | ADD RULE RULERANGE=ALL / ADD USERPROFILE PROFILERANGE=ALL | 默认 |
| 仅主锚点 UPF | RULERANGE=CENTRAL / PROFILERANGE=CENTRAL | 主锚点策略 |
| 仅辅锚点 UPF | RULERANGE=LOCAL / PROFILERANGE=LOCAL + ADD RULEBINDDNAI / ADD USRPROBINDDNAI 绑 DNAI | 辅锚点策略 |

## 约束（特性级 rule）

- **License 双项前置**（critical）：须激活 LKV2PCCBF01 + LKV3W9SPCC11 两项 License — 否则 PCC 功能不生效。
- **PCRF VPN 须与 Gx 逻辑接口同 VPN**（critical）：`ADD PCRF` 的 VPNINSTANCE 须与 `ADD LOGICINF` 的 VPNINSTANCE 一致 — 否则 Gx 链路不通（仅 2G/3G/4G）。
- **动态 PCC 须配 PCRF 对接**（critical）：动态 PCC 场景 SET PCCFUNC 使能前须完成 PCRF Diameter 对接（[1-00012](task/UNC/20.15.2/1-00012.md)）+ PCRF 分组选择（[1-00013](task/UNC/20.15.2/1-00013.md)）— 否则用户激活选不到 PCRF。
- **本地 PCC 须配本地规则**（critical）：本地 PCC（SET PCCFUNC HOMEPCCSWITCH=DISABLE）须配 ADD RULE + USERPROFILE + RULEBINDING — 否则无静态规则可匹配。
- **RULENAME/USERPROFILENAME 跨 PCF/SMF/UPF 一致**（critical）：不一致时 PCF/PCRF 下发的策略无法生效。
- **APN 级 PCC 开关优先于全局**（info）：SET APNPCCFUNC 优先级高于 SET PCCFUNC；APN 级 INHERIT 才继承全局。
- **策略接口优先级**（info，混合代际）：基于 PCF 实例标识 > 基于 APN > 全局（SET POLICYMODE）；支持 N7 回落 Gx。
- **规则下发范围跨网元一致**（warning）：RULERANGE/PROFILERANGE + DNAI 绑定在 PCF/SMF/UPF 间须一致 — 否则策略下发错位。
- **加密协议不可 PCC**（warning）：对 SSL/FTPS 等加密协议报文不能应用 PCC 策略（特性 wiki 应用限制）。
