# RMV LOGICINF 原子任务
> 命令静态知识见 [RMV LOGICINF](command/UNC/20.15.2/RMV-LOGICINF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

**高危命令**。删除对象是 `LOGICINF`（逻辑接口）记录。仅靠 `NAME` 单参数定位，不存在"不输参数全删"的批量语义。

## 配置方法
按逻辑接口名精确删除。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| NAME | 逻辑接口名称（global_planned，required） | `gx-logicinf` |

典型脚本（删 Gx 接口对应逻辑接口，便于重建）：
```
RMV LOGICINF: NAME=gx-logicinf;
```

## 决策点（若有）

### 是否需要重建（先删后建）
- 问什么：删除后是保持删除态，还是立即 ADD LOGICINF 重建。
- 调测到 PCRF 的数据场景中，本命令是**可选**步骤：仅当 Gx 接口 IP 与规划值不一致时才执行 RMV LOGICINF，紧接着 ADD LOGICINF 按规划数据重新配置 Gx 接口 IP。
- 配套前置：RMV LOGICINF 前通常已先执行 RMV DIAMCONNECTION + RMV DIAMLOCINFO 删原 Diameter 配置。

## 约束（若有）
- 高危：删除逻辑接口可能导致相关业务中断，删前必须确认接口未承载在用业务。`severity=high`
- 前置存在性：待删逻辑接口须为系统中已存在记录，否则命令执行失败。`severity=high`
- 绑定阻删：逻辑接口被其他对象绑定时禁止删除，需先解除绑定。`severity=high`
- 主备不同步：逻辑接口配置不在主备 SMF 之间自动同步，需在主备 SMF 上分别配置/删除。`severity=medium`
- 生效时机：立即生效。`severity=medium`
- 无批量删除：仅靠 NAME 定位单条，不存在"省略参数删全部"语义。`severity=low`
