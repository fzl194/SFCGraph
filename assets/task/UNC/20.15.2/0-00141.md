# 删除逻辑接口（RMV LOGICINF）

> 命令静态知识见 [RMV LOGICINF](command/UNC/20.15.2/RMV-LOGICINF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按逻辑接口名（NAME）精确删除单条记录；删除对象是 `LOGICINF`（逻辑接口），仅靠 NAME 单参数定位，不存在"省略参数删全部"的批量语义。调测到 PCRF 数据场景中 Gx 接口 IP 变更时常成对 ADD LOGICINF 重建。

### 参数维度表

| 维度 | 参数 | 数据来源 | 取值 | 作用 | 适用场景 |
|------|------|----------|------|------|----------|
| 逻辑接口名 | NAME | 全网规划 | 字符串 1～63，格式形如 `gaif1/0/0`（gaif=类型；1/0/0=组号/实例类型/逻辑接口号） | 定位待删逻辑接口，必选参数 | 精确删指定逻辑接口 |

> 说明：高危命令。NAME 为必选参数，缺失则命令执行失败；不存在"省略参数 = 删全部"语义。逻辑接口被绑定时禁止删除，需先解除绑定。

### 典型脚本

删 Gx 接口对应逻辑接口（便于重建）：
```
RMV LOGICINF: NAME="gxif1/0/1";
```

### 步骤位置

调测到 PCRF 数据场景的 Gx 接口 IP 变更重建链：
1. `LST LOGICINF` — 校验待删逻辑接口存在性与在用状态（未建）
2. 前置删除 Diameter 配置：`RMV DIAMCONNECTION`（0-00139）+ `RMV DIAMLOCINFO`（0-00140）
3. `RMV LOGICINF` — 删除原逻辑接口（本案）
4. `ADD LOGICINF` — 按规划 IP 重建逻辑接口（0-00043）

## 决策点

### 决策点：删除后是否立即重建（DP 0-03021）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 先删后建 | 删后紧跟 `ADD LOGICINF` 按规划 IP 重建 | 调测到 PCRF 场景中 Gx 接口 IP 与规划值不一致，需重设接口 IP |
| 仅删保持删除态 | — | 接口已废弃或由后续步骤另行处理 |

> 注意：调测到 PCRF 数据场景中本命令是**可选**步骤——仅当 Gx 接口 IP 与规划值不一致时才执行 RMV LOGICINF，紧接 ADD LOGICINF 重建；配套前置通常是 RMV DIAMCONNECTION + RMV DIAMLOCINFO 删原 Diameter 配置。

### 决策点：绑定阻删处理（DP 0-03022）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 先解绑再删 | 删前先解除引用本逻辑接口的对象绑定 | 逻辑接口被其他对象绑定（命令会拒绝执行） |
| 直接删除 | — | 逻辑接口未被任何对象绑定 |

> 注意：若逻辑接口被其他对象绑定则不允许删除——需先识别并解除所有引用关系。逻辑接口配置不在主备 SMF 之间自动同步，需在主备 SMF 上分别配置/删除。

## 约束

· rule-0-03051 · severity=critical · 高危业务影响：删除逻辑接口可能导致相关业务中断，删前必须确认接口未承载在用业务
· rule-0-03052 · severity=high · 前置存在性：待删逻辑接口须为系统中已存在记录，否则命令执行失败
· rule-0-03053 · severity=high · 绑定阻删：逻辑接口被其他对象绑定时禁止删除，需先解除绑定
· rule-0-03054 · severity=warning · 主备不同步：逻辑接口配置不在主备 SMF 之间自动同步，需在主备 SMF 上分别配置/删除
· rule-0-03055 · severity=info · 立即生效：命令执行后立即生效，不可回滚；无批量删除语义（仅靠 NAME 定位单条）
