# MOD TNFINS 原子任务
> 命令静态知识见 [MOD TNFINS](command/UNC/20.15.2/MOD-TNFINS.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：AMF、SMF、NSSF、SMSF、NCG。本命令按目标 NF 实例索引修改实例的协议模式（SCHEMA）、域名（FQDN）、描述名，是 NRF 服务发现侧目标 NF 实例属性的修改入口。常作为基于 CC 选择 CHF 主备链路配置的一环（为备 CHF 实例补全协议/域名后才能被访问与绑定）。

## 配置方法
按目标 NF 实例索引定位，按规划修改协议模式/域名/描述名。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| TNFINSINDEX | 目标 NF 实例索引（必选，0~2047） | 按规划 |
| SCHEMA | 协议模式（可选） | 按规划（`UriSchemeINVALID/SchemaInvalid` 已废弃，勿用） |
| FQDN | 域名（可选，0~127，不区分大小写） | 按规划 CHF/NF 域名 |
| NFDESCNAME | NF 描述名称（可选，0~255） | 按规划 |

主备 CHF 可靠性调测（WSFD-011206）：链路 `ADD TNFINSIP → MOD TNFINS → ADD TNFGRP → MOD TNFBINDGRP → MOD SELECTCHFGBYCC` 中，MOD TNFINS 在 ADD TNFINSIP 配 IP 之后执行，为备 CHF 实例补全协议/域名，再经绑定入组，最终被 SELECTCHFGBYCC 引用。

## 决策点

### 协议模式（SCHEMA）
SCHEMA 取值随目标 NF 暴露的服务协议分叉；其中 `UriSchemeINVALID/SchemaInvalid` 已废弃，配置时需规避该取值。

## 约束
- **立即生效**：修改后立即影响 NRF 对该实例的访问/发现结果。
- **实例存在性前置**：TNFINSINDEX 需已通过 ADD TNFINS 添加，MOD 仅改属性；可用 LST TNFINS 校验。
- **SCHEMA 废弃取值**：`UriSchemeINVALID（SchemaInvalid）` 已废弃，不能配置。
- **FQDN 不区分大小写**：长度 0~127；FQDN 需与 IP（ADD TNFINSIP）配套，仅配 FQDN 而无 IP 实例不可达。
- **索引范围**：TNFINSINDEX 0~2047，需与 ADD TNFINSIP/MOD TNFBINDGRP 中使用的索引一致。
