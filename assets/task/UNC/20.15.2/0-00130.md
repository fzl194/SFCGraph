# 修改目标NF实例配置（MOD TNFINS）

> 命令静态知识见 [MOD TNFINS](command/UNC/20.15.2/MOD-TNFINS.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按目标 NF 实例索引定位，按规划修改协议模式/域名/描述名。

### 参数维度表

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|------|------|------|------|------|----------|
| 实例索引 | TNFINSINDEX | 0~2047 | — | 目标 NF 实例唯一索引，必选 | 所有场景 |
| 协议模式 | SCHEMA | 按目标 NF 暴露的服务协议分叉 | — | 指定 NF 实例使用的协议类型 | 根据 NF 类型选择对应协议 |
| 域名 | FQDN | 0~127 字符，不区分大小写 | 空 | NF 实例的域名标识 | 主备 CHF 场景中配置备 CHF 域名 |
| 描述名 | NFDESCNAME | 0~255 字符 | 空 | NF 实例的描述名称 | 运维标识用途 |

> 说明：`UriSchemeINVALID/SchemaInvalid` 已废弃，勿使用该取值。

### 典型脚本

```
MOD TNFINS:TNFINSINDEX=128,SCHEMA="HTTP",FQDN="chf-backup.example.com",NFDESCNAME="备CHF-主备链路";
```

### 步骤位置

主备 CHF 可靠性调测链路中的位置：
1. `ADD TNFINSIP` — 配置 IP 地址
2. `MOD TNFINS` — 补全协议/域名（本案）
3. `ADD TNFGRP` — 加入目标 NF 组
4. `MOD TNFBINDGRP` — 绑定组配置
5. `MOD SELECTCHFGBYCC` — 基于 CC 选择 CHF

---

## 决策点

### 决策点：是否配置 FQDN（DP 0-01298）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| 配置 FQDN | 需先通过 ADD TNFINSIP 配置对应 IP | 主备 CHF 场景中为备 CHF 实例补全域名 |
| 不配置 FQDN | 仅依赖 IP 访问 | NF 实例仅通过 IP 寻址的场景 |

> 注意：仅配 FQDN 而无 IP 实例不可达，FQDN 需与 ADD TNFINSIP 配置的 IP 配套使用。

### 决策点：协议模式 SCHEMA 取值（DP 0-01299）

| 选项 | 联动参数 | 适用场景 |
|------|----------|----------|
| HTTP | NF 实例支持 HTTP 协议服务 | 标准 CHF/NRF 互通场景 |
| Diameter | NF 实例支持 Diameter 协议服务 | 传统计费接口场景 |
| PYE | NF 实例支持 PYE 协议服务 | 特定协议扩展场景 |

> 注意：`UriSchemeINVALID/SchemaInvalid` 已废弃，配置时需规避该取值。

## 约束

- **立即生效**（critical，rule-0-01682）：立即生效：修改后立即影响 NRF 对该实例的访问/发现结果
- **实例存在性前置**（critical，rule-0-01683）：实例存在性前置：TNFINSINDEX 需已通过 ADD TNFINS 添加，MOD 仅改属性；可用 LST TNFINS 校验
- **SCHEMA 废弃取值**（warning，rule-0-01684）：SCHEMA 废弃取值：UriSchemeINVALID（SchemaInvalid）已废弃，不能配置
- **FQDN 与 IP 配套**（warning，rule-0-01685）：FQDN 与 IP 配套：仅配 FQDN 而无 IP 实例不可达；FQDN 长度 0~127，不区分大小写
- **索引范围**（info，rule-0-01686）：索引范围：TNFINSINDEX 0~2047，需与 ADD TNFINSIP/MOD TNFBINDGRP 中使用的索引一致
