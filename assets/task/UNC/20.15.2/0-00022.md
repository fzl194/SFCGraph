# ADD CHGTARI 原子任务

> 命令静态知识见 [ADD CHGTARI](command/UNC/20.15.2/ADD-CHGTARI.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CHGTARI 登记一条"费率时段"。它定义某类计费属性用户在"某天"（工作日 WORK / 周休日 / 节假日）中一段连续时间 `[ST, ET]` 采用某个费率类型。"某天是工作日还是节假日"由 [SET CHGWKDY](command/UNC/20.15.2/SET-CHGWKDY.md)（星期表）+ [ADD CHGHOLI](task/UNC/20.15.2/0-00019.md)（节假日）共同识别。配置时按整网规划给出费率类型 TT、起止时间 ST/ET、计费属性 CC 四个必填项。

**参数取值样例（离线计费费率切换范式，WSFD-011201）**

| 参数 | 取值样例 | 默认 | 说明 |
|---|---|---|---|
| TT | `WORK` | — | 费率类型，WORK=工作日（必填） |
| ST | `18&00&00` | — | 起始时间，HH&MM&SS（必填），00&00&00~23&59&59 |
| ET | `21&30&00` | — | 结束时间，HH&MM&SS（必填），00&00&00~23&59&59 |
| CC | `PREPAID` | — | 计费属性（必填）；PREPAID=预付费计费 |
| TARIACCFLG | — | 无 | 费率变更允许的用户类型（可选），HOME_USER=本地用户 |

**典型行（为预付费用户配置工作日 18:00–21:30 费率时段）**

```
ADD CHGTARI: TT=WORK, ST=18&00&00, ET=21&30&00, CC=PREPAID;
```

**配置位置**：在"配置离线计费费率切换（SGSN）"流程中作为第 4 步可选项（可选地增加费率时段配置），通常先执行 SET CHGWKDY 设置星期表，再按需要的费率类型逐条 ADD CHGTARI。

## 决策点

### 费率类型 TT 与计费属性 CC 的组合

TT（哪类天）与 CC（哪类用户）共同决定一条费率时段记录的归属。数据规划阶段需对每个"TT × CC"组合给出起止时间。

| 选项维度 | 取值 | 适用 |
|---|---|---|
| 费率类型 TT | `WORK`（工作日）/ 周休日 / 节假日 | 与星期表、节假日配置联动识别当天归属 |
| 计费属性 CC | `NORMAL`（普通计费）/ `PREPAID`（预付费）/ 包月制 / 实时计费 | 按用户的计费属性差异化配费率时段 |

> ③ 各特性数据规划表"获取方法"列分布为单一值 `全网规划`（4 行均为本端/整网规划），TT 与 CC 的具体组合由整网规划决定 → 绑定类型倾向 global_planned，组合本身是规划侧决策。

### 是否配置费率变更允许的用户类型

| 选项 | 适用 |
|---|---|
| 不配 TARIACCFLG（缺省） | 默认行为 |
| 配 `HOME_USER` | 仅允许本地用户在费率变更时受影响 |

## 约束

- **立即生效** · severity=LOW。
- **唯一性**：费率类型 TT + 起始时间 ST + 结束时间 ET + 计费属性 CC 四者唯一确定一条记录 · severity=HIGH · 重复登记新增失败。
- **不重叠**：新增费率时段与相同 TT + CC 的已有费率时段不得相互重叠 · severity=HIGH · 重叠会导致费率归属歧义。
- **记录上限**：同一 CC + TT 的记录总数不超过 24，本表记录总数不超过 288 · severity=MEDIUM · 超过上限新增失败。
- **最小时长**：任一费率时段长度不得小于 10 分钟 · severity=MEDIUM。
- **对齐零点**：任一费率时段的 ST、ET 与零点的间隔需满足规格约束（见命令 notes） · severity=MEDIUM。
- **联动依赖**：费率时段需与 SET CHGWKDY（星期表）+ ADD CHGHOLI（节假日）配合才能识别"当天是工作日还是节假日"，单独配费率时段不产生差异化效果 · severity=MEDIUM。
