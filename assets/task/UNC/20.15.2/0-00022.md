# 配置费率时段（ADD CHGTARI）

> 命令静态知识见 [ADD CHGTARI](command/UNC/20.15.2/ADD-CHGTARI.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD CHGTARI 登记一条"费率时段"记录。它定义某类计费属性用户在"某天"（工作日 WORK / 周休日 / 节假日）中一段连续时间 `[ST, ET]` 采用某个费率类型。"某天是工作日还是节假日"由 [SET CHGWKDY](command/UNC/20.15.2/SET-CHGWKDY.md)（星期表）+ [ADD CHGHOLI](task/UNC/20.15.2/0-00019.md)（节假日）共同识别。配置时按整网规划给出费率类型 TT、起止时间 ST/ET、计费属性 CC 四个必填项。

核心配置空间有 4 个维度：**费率类型**（必选，决定天类）、**计费属性**（必选，决定用户类）、**时间窗**（必选，决定生效区间）、**费率变更用户类型**（可选，决定漫游控制）。

### 配置维度 1：费率类型（参数 TT，必选）

| 取值 | 含义 | 识别来源 |
|---|---|---|
| **WORK** | 工作日 | SET CHGWKDY 星期表 |
| **WEEKEND** | 周休日 | SET CHGWKDY 星期表 |
| **FESTIVAL** | 节假日 | ADD CHGHOLI 节假日表 |

> 三类天互斥；同一天的归属由星期表+节假日表共同识别。配置前需先完成 SET CHGWKDY 与 ADD CHGHOLI。

### 配置维度 2：计费属性（参数 CC，必选）

| 取值 | 含义 | 适用用户 |
|---|---|---|
| **NORMAL** | 普通计费 | 按常规方式支付费用 |
| **PREPAID** | 预付费 | 获取服务前预支付 |
| **FLATRATE** | 包月制 | 月内收费固定 |
| **HOTBILLING** | 实时计费 | 短时间/流量达阈值时及时生成话单 |

### 配置维度 3：时间窗（参数 ST / ET，必选）

| 参数 | 格式 | 取值范围 | 特殊语义 |
|---|---|---|---|
| `ST` | `HH&MM&SS` | 00&00&00~23&59&59 | 起始时间 |
| `ET` | `HH&MM&SS` | 00&00&00~23&59&59 | 结束时间；ST=ET=00&00&00 表示一整天 |

> 任一费率时段长度不得小于 10 分钟；ST/ET 与零点的间隔或等于 0，或大于等于 10 分钟；任两条费率时段的间隔或等于 0，或大于等于 10 分钟。

### 配置维度 4：费率变更用户类型（参数 TARIACCFLG，可选）

| 取值 | 含义 |
|---|---|
| `HOME_USER` | 仅本地用户在费率变更时受影响 |
| `ROAM_USER` | 仅漫游用户 |
| `VISIT_USER` | 仅拜访用户 |
| 不配 | 系统默认所有字段全选（HOME+ROAM+VISIT） |

**典型脚本（为预付费用户配置工作日 18:00–21:30 费率时段）**

```
ADD CHGTARI: TT=WORK, ST=18&00&00, ET=21&30&00, CC=PREPAID;
```

**典型脚本（为普通用户配置周休日全天费率时段）**

```
ADD CHGTARI: TT=WEEKEND, ST=00&00&00, ET=00&00&00, CC=NORMAL;
```

**步骤位置**：在"配置离线计费费率切换（SGSN）"流程中作为第 4 步可选项（可选地增加费率时段配置）。前置 SET CHGWKDY 设置星期表，再按需要的费率类型逐条 ADD CHGTARI；如需节假日费率切换，前置 ADD CHGHOLI。

## 决策点

### 决策点：费率类型与计费属性组合（DP 0-02023）

问：本条费率时段记录归属哪类天 × 哪类用户（决定 TT 与 CC 取值）？

| 选项维度 | 取值 | 适用 |
|---|---|---|
| 费率类型 TT | `WORK` / `WEEKEND` / `FESTIVAL` | 与星期表、节假日配置联动识别当天归属 |
| 计费属性 CC | `NORMAL` / `PREPAID` / `FLATRATE` / `HOTBILLING` | 按用户的计费属性差异化配费率时段 |

> 数据规划阶段需对每个"TT × CC"组合给出起止时间；TT 与 CC 的具体组合由整网规划决定（global_planned）。

### 决策点：费率变更用户类型与时间窗策略（DP 0-02024）

问：本条费率时段覆盖哪些用户类型 + 采用全天还是分时段（决定 TARIACCFLG 与 ST/ET）？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 全天 + 所有用户类型 | `ST=00&00&00, ET=00&00&00`，不配 TARIACCFLG | 一整天统一费率（默认全选用户类型） |
| 分时段 + 所有用户类型 | `ST/ET` 指定区间，不配 TARIACCFLG | 时段差异化费率（如 18:00–21:30 高峰） |
| 分时段 + 仅本地用户 | `ST/ET` 指定区间，`TARIACCFLG=HOME_USER` | 漫游/拜访用户走另一套费率 |
| 分时段 + 仅漫游用户 | `ST/ET` 指定区间，`TARIACCFLG=ROAM_USER` | 漫游用户差异化计费 |

> TARIACCFLG 不配时系统默认 HOME+ROAM+VISIT 全选；如需对漫游用户单独配费率，需显式指定 ROAM_USER。

## 约束

- **唯一性**（critical，rule-0-02056）：唯一性：费率类型 TT + 起始时间 ST + 结束时间 ET + 计费属性 CC 四者唯一确定一条记录，重复登记新增失败
- **不重叠**（warning，rule-0-02057）：不重叠：新增费率时段与相同 TT + CC 的已有费率时段不得相互重叠，重叠会导致费率归属歧义
- **记录上限**（warning，rule-0-02058）：记录上限：同一 CC + TT 的记录总数不超过 24，本表记录总数不超过 288，超过上限新增失败
- **时间约束**（warning，rule-0-02059）：时间约束：任一费率时段长度不得小于 10 分钟；ST/ET 与零点的间隔或等于 0 或大于等于 10 分钟；任两条费率时段的间隔或等于 0 或大于等于 10 分钟
- **联动依赖**（info，rule-0-02060）：联动依赖：费率时段需与 SET CHGWKDY（星期表）+ ADD CHGHOLI（节假日）配合才能识别"当天是工作日还是节假日"，单独配费率时段不产生差异化效果
