# ADD SRROUTE 原子任务
> 命令静态知识见 [ADD SRROUTE](command/UNC/20.15.2/ADD-SRROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是平台服务管理侧的 IPv4 静态路由建立任务，广泛用于 IPsec 引流（IPFD-016000）、GRE 隧道引流（IPFD-015002）、到 AAA Server / CG / OCS 的目的网络路由（WSFD-011306/WSFD-011201/WSFD-109001）以及 Direct Tunnel 控制面路由（WSFD-104506）。

## 配置方法
按"地址族 + 目的网络 + VPN 实例 + 出接口/下一跳"四元组配置一条静态路由。缺省路由用 `PREFIX=0.0.0.0, MASKLENGTH=0`。

| 配置形态 | 参数组合 | 获取方法 | 样例 |
|---|---|---|---|
| 普通 IPv4 路由（公网/接口出向） | AFTYPE=ipv4unicast, PREFIX, MASKLENGTH, IFNAME, NEXTHOP | AFTYPE 固定；PREFIX/MASKLENGTH 全网规划；IFNAME/NEXTHOP 已配置数据获取 | `ADD SRROUTE:AFTYPE=ipv4unicast,PREFIX="10.10.1.202",MASKLENGTH=24,IFNAME="Tunnel1";` |
| VPN 绑定路由（IPsec/GRE 引流） | + VRFNAME=vpn 名, DESTVRFNAME=下一跳 VPN 名 | VRFNAME/DESTVRFNAME 已配置数据获取（由 `ADD L3VPNINST` 建好） | `ADD SRROUTE:VRFNAME="vrf1",AFTYPE=ipv4unicast,PREFIX="10.1.2.0",MASKLENGTH=24,DESTVRFNAME="vrf1",NEXTHOP="192.168.1.2",IFNAME="Tunnel10";` |
| VPN 缺省路由（到 CG/OCS） | VRFNAME, PREFIX=0.0.0.0, MASKLENGTH=0, NEXTHOP | VRFNAME 已配置获取；NEXTHOP 全网规划 | `ADD SRROUTE:VRFNAME="vpn_gy",AFTYPE=ipv4unicast,PREFIX="0.0.0.0",MASKLENGTH=0,NEXTHOP="10.3.37.81";` |
| Direct Tunnel 控制面黑洞路由 | VRFNAME=_public_, PREFIX=对端IP, MASKLENGTH=32, IFNAME=NULL0/Invalid0 | VRFNAME 固定；PREFIX 全网规划；IFNAME 固定 NULL0 | `ADD SRROUTE:VRFNAME="_public_",AFTYPE=ipv4unicast,PREFIX="10.3.1.1",MASKLENGTH=32,DESTVRFNAME="_public_",NEXTHOP="10.4.1.1",IFNAME="NULL0";` |

## 决策点

### 路由形态选择（缺省路由 vs 主机路由 vs 网段路由）
- 问什么：本条路由服务于哪种业务意图，决定 PREFIX/MASKLENGTH 取法。
- 到 CG/OCS 的目的网络可达性 → VPN 缺省路由（PREFIX=0.0.0.0/MASKLENGTH=0）。
- IPsec 隧道对端 IP（隧道目的）→ 主机路由（MASKLENGTH=32，IFNAME=物理接口/Invalid0）。
- IPsec 引流到对端网段 → 网段路由（MASKLENGTH=24，IFNAME=Tunnel 隧道接口）。
- Direct Tunnel 控制面黑洞 → 主机路由 + IFNAME=NULL0。
- 属业务意图驱动决策（对应不同特性 WSFD）。

### 出接口与下一跳的填法
- 问什么：路由出向是用隧道接口还是物理接口，下一跳是同 VPN 还是跨 VPN。
- 同 VPN 内出向 → 仅 IFNAME（隧道/物理接口名，已配置获取）+ NEXTHOP。
- 跨 VPN 引流 → 加 DESTVRFNAME（下一跳所在 VPN），此时 IFNAME 常为 `Invalid0`/`NULL0`。
- IFNAME/NEXTHOP 来源为"已配置数据中获取"（来自 `ADD IFIPV4ADDRESS` / `ADD GRETUNNEL` / `ADD IPSECTUNNEL` 等已建对象）。

### VPN 实例是否绑定
- 问什么：路由属于公网表还是某 VPN 实例表。
- 公网/默认 → VRFNAME 取默认 `_public_`（可省略，默认值）。
- 业务隔离（IPsec/AAA/CG/OCS 专用 VPN）→ VRFNAME 显式指定（已通过 `ADD L3VPNINST` 建好的 VPN 名）。

## 约束
- 唯一性：同一静态路由不可重复添加（要保证该路由未添加过）。`severity=medium`
- 规格上限：最大记录数 100000。`severity=low`
- 取值范围：MASKLENGTH 0~32；VRFNAME/DESTVRFNAME 1~31；IFNAME 1~63；PREFERENCE 1~255（默认 60）；TAG 0~4294967295；DESCRIPTION 1~35；SESSIONNAME 1~64。`severity=low`
- 禁止：不能在 VPN 实例 `__mpp_vpn_inner__` 和 `__mpp_vpn_inner_server__` 下添加默认静态路由。`severity=medium`
- 生效时机：命令执行后立即生效。`severity=low`
- 一致性（前置）：VRFNAME 需已通过 `ADD L3VPNINST` 建好；IFNAME 需已通过 `ADD IFIPV4ADDRESS` / `ADD GRETUNNEL` 等建好（路由引用一致性）。`severity=medium`
- BFD 联动：BFDENABLE=TRUE 时必须配套指定已建好的静态 BFD 会话 SESSIONNAME；BFDENABLE=FALSE 不填 SESSIONNAME。`severity=medium`
