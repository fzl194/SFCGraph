# 增加地址池组映射关系（ADD POOLGRPMAP）

> 命令静态知识见 [ADD POOLGRPMAP](command/UNC/20.15.2/ADD-POOLGRPMAP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令建立"地址池组 ↔ UPF 组/位置区组/APN"的映射规则，是地址分配族的高层路由表——决定用户激活时按哪些维度匹配到地址池组。配置空间核心维度为 **映射维度组合**（UPF 组、位置区组、APN 中至少一项）和 **位置区组类型**（LOCATIONGRPTYPE）。

### 配置维度1：映射维度组合（参数 POOLGRPNAME × UPFGRPNAME / LOCATIONGRPNAME / APN）

| 组合 | 作用 | 典型场景 |
|---|---|---|
| POOLGRPNAME + UPFGRPNAME | 按用户锚点 UPF 组选地址池组 | WSFD-010502/WSFD-107021：POOLGRPNAME="poolgroup1" + UPFGRPNAME="upfgroup1"（UDM 静态地址，必配 UPF 组） |
| POOLGRPNAME + APN | 按 APN 选地址池组 | APN 维度分配 |
| POOLGRPNAME + UPFGRPNAME + APN | UPF 组 + APN 双维度 | 精细化分配 |
| POOLGRPNAME + LOCATIONGRPNAME（LAC/TAC）| 按位置区选地址池组 | 3G LAC / 4G/5G TAC 位置感知（当前版本不支持 LOCATIONGRPTYPE/LOCATIONGRPNAME） |

> APN、UPFGRPNAME、LOCATIONGRPTYPE 三者不能同时为空。

### 配置维度2：位置区组类型（参数 LOCATIONGRPTYPE，可选）

| 取值 | 作用 | 联动参数 |
|---|---|---|
| LAC | 使用 3G 用户 LAC 作位置区 | LOCATIONGRPNAME（由 ADD ADDRLACGROUP（★R1.5 atom 待补） 配置） |
| TAC | 使用 4G/5G 用户 TAC 作位置区 | LOCATIONGRPNAME（由 ADD ADDRTACGROUP（★R1.5 atom 待补） 配置） |
| None（默认） | 不使用位置区作匹配条件 | 无 |

> **当前版本不支持** LOCATIONGRPTYPE、LOCATIONGRPNAME 参数（按命令 md 注意事项），实战中映射维度主要是 UPFGRPNAME 和 APN。

**步骤位置**：在地址分配族编排中，于 `ADD ADDRPOOLGRP`（建池组）、`ADD ADDRUPGROUP`（建 UPF 组）、`ADD APN`（建 APN）之后执行；与 `SET APNIPALLOCRULE（★R1.5 atom 待补）`/`SET IPALLOCRULE（★R1.5 atom 待补）`（地址分配规则总开关）配套使用。

## 决策点

**映射维度与分配规则对齐**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| POOLGRPNAME + UPFGRPNAME | 仅 UPF 组维度 | UDM/Radius/DHCP 静态地址组（必配 UPF 组） |
| POOLGRPNAME + APN | 仅 APN 维度 | 按 APN 分配 |
| POOLGRPNAME + UPFGRPNAME + APN | 双维度 | 精细化 |

> 映射维度的选择必须与 `SET APNIPALLOCRULE（★R1.5 atom 待补）`/`SET IPALLOCRULE（★R1.5 atom 待补）` 的 IPV4FIRSTRULE/SECONDRULE/THIRDRULE 取值一致（如规则取 "APN-1&UPNODE-1" 则 APN+UPFGRPNAME 必须配置，LOCATIONGRPNAME 不能配置）。

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无回滚窗口
- **最大记录数40000**（warning）：最多可输入 40000 条记录 — 后果：超规格新增失败
- **POOLGRPNAME 必须先建**（critical）：POOLGRPNAME 必须由 `ADD ADDRPOOLGRP` 已配置 — 后果：组不存在则新增失败
- **UPFGRPNAME 必须先建**（critical）：UPFGRPNAME 必须由 `ADD ADDRUPGROUP` 已配置；UDM/Radius/DHCP 组类型时 UPF 组名不能为空 — 后果：组不存在/未配则新增失败
- **组类型一致**（critical）：地址池组类型（POOLGRPTYPE）与 UPF 组类型（UPFGRPTYPE）必须一致 — 后果：不一致则新增失败
- **映射唯一性**（critical）：相同 (UPF 组, 位置区组, APN) 组合只能对应一个地址池组 — 后果：重复映射失败
- **VPN 一致性**（critical）：若映射含 APN，则 APN 的 VPN 与地址池组下所有地址池的 IP 对应 VPN 必须一致 — 后果：VPN 不一致则新增失败
- **维度非空**（critical）：APN、UPFGRPNAME、LOCATIONGRPTYPE 不能同时为空 — 后果：全空则新增失败
- **多 UPF 组最后生效**（warning）：同一 POOLGRPNAME 对应多个 UPFGRPNAME 时，最后添加的 UPFGRPNAME 才生效 — 后果：旧配置被覆盖
- **当前版本不支持 LOCATIONGRPTYPE/LOCATIONGRPNAME**（warning）：当前版本不支持这两个参数 — 后果：配置无效
