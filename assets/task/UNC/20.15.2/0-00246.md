# 创建OSPF进程（ADD OSPF）

> 命令静态知识见 [ADD OSPF](command/UNC/20.15.2/ADD-OSPF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令创建 OSPF 进程实例，是 OSPF 族的根对象。配置空间核心维度为 **VPN 实例绑定**（VRFNAME，决定进程属公网还是某私网）和 **路由器标识**（SCHEMAROUID），其余为进程级调优参数（BFD/SPF/LSA/Stub 等）。

### 配置维度1：VPN 实例绑定（参数 VRFNAME）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| 不配 | OSPF 进程属公网 | 公网 OSPF |
| 指定 VRFNAME | OSPF 进程属指定 VPN 实例（私网 OSPF） | VPN 内 OSPF（IPFD-016000 OSPF over IPsec：vrf1） |

### 配置维度2：路由器标识（参数 SCHEMAROUID）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| LoopBack 接口 IPv4 | 用本端 LoopBack 地址作 Router ID（推荐稳定） | IPFD-016000：SCHEMAROUID="10.102.105.238" / "10.102.105.224" |
| 任一 IPv4 | 显式指定 Router ID | 临时或非 LoopBack 场景 |
| 不配 | 系统自动选举 | 简单场景 |

### 配置维度3：进程级调优（可选，BFD/SPF/LSA/Stub 等）

| 参数组 | 作用 | 典型场景 |
|---|---|---|
| BFDALLINTFFLG/BFDMINRXINTV/BFDMINTXINTV/SCHEMABFDDETMUL | 进程级 BFD（链路快速检测） | 需快速收敛 |
| SPFINTVSEC/SPFMAXINTERVAL/SPFSTARTINTV/SPFHOLDINTERVAL | SPF 智能定时器调优 | 抑制网络频繁变化 |
| LSAORIGINSTFLAG/LSAORIGMAXINTV 等 | LSA 更新间隔调优 | 链路不稳定场景 |
| STUBROUFLG=StubOnBoot + STUBROUSTUPINTV | 设备主备倒换后临时通告最大开销 | 平滑倒换 |
| SILENTALLFLAG=TRUE | 抑制所有接口收发 OSPF 报文 | 安全隔离 |

**步骤位置**：在 OSPF 族编排中，是 OSPF 系列的起点——于 `ADD VPNINST`（若属私网）之后、`ADD OSPFAREA`（建区域）之前执行。

## 决策点

**VPN 实例与 Router ID 选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 公网 OSPF | VRFNAME 不配 + SCHEMAROUID | 公网对接 |
| 私网 OSPF（VPN 内） | VRFNAME + SCHEMAROUID | IPFD-016000 OSPF over IPsec（vrf1 + 10.102.105.x） |

> Router ID 必须全网唯一（同进程内）；建议用 LoopBack IPv4 作 Router ID 以保证稳定性。

## 约束

- **PROCID 进程号范围**（warning）：整数 1~4294967295 — 后果：超范围新增失败
- **PROCID 唯一性**（critical）：进程号在本设备唯一 — 后果：重复新增失败
- **VRFNAME 必须先建**（critical）：VRFNAME 若填，必须由 `ADD VPNINST` 已配置 — 后果：VPN 不存在则新增失败
- **SCHEMAROUID 必须为合法 IPv4**（warning）：点分十进制 IPv4 格式 — 后果：非法格式新增失败
- **VRFNAME 命名约束**（warning）：字符串 1~31，区分大小写 — 后果：违规命名新增失败
