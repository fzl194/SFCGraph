# ADD UPFBINDGXUPFGRP 原子任务
> 命令静态知识见 [ADD UPFBINDGXUPFGRP](command/UNC/20.15.2/ADD-UPFBINDGXUPFGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是 PCRF 对接 UPF 组装链的中间环：`ADD GXUPFGROUP`（建 Gx UPF 组）→ **本命令**（把 UPF 实例装进组）→ `ADD UPFGLOCGBNDGRP` + `ADD UPFGBINDLOCG`（组与 Diameter 本端主机组绑定）。一个组可装多个 UPF，同一 UPF 可属于不同组。

## 配置方法
按"Gx UPF 组 + UPF 实例标识"二元组配置，每个 UPF 一条。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| UPFGRPNAME | 已配置数据中获取（来自 `ADD GXUPFGROUP`） | `upfgrp_01`、`upfgrp_02` |
| UPFINSTANCEID | 与对端协商，UPF 实例标识（UPF 主机名） | `upf_instance_01` |

典型脚本（两个 UPF 各入一个组）：
```
ADD UPFBINDGXUPFGRP: UPFGRPNAME="upfgrp_01", UPFINSTANCEID="upf_instance_01";
ADD UPFBINDGXUPFGRP: UPFGRPNAME="upfgrp_02", UPFINSTANCEID="upf_instance_02";
```

## 决策点

### UPFINSTANCEID 取值来源
- 问什么：UPF 实例标识从哪来。
- 取自对端 UPF 的 UpfHostName，需与对端协商一致；填写约束：1~36 字符，若以 `upf` 开头需满足特定格式规则。
- 与对端不一致会导致 PCRF/Gx 路径选错 UPF，策略下发不到正确网元。

### UPF 与组的归属关系
- 问什么：一个 UPF 是否可入多个 Gx UPF 组。
- 允许：同一 UPF 可绑到不同 Gx UPF 组下（跨组复用）。但单组上限 300 个 UPF。

## 约束

- **立即生效** · severity=LOW。
- **前置依赖**：必须先执行 `ADD GXUPFGROUP` 建 Gx UPF 组，本命令才能引用 UPFGRPNAME · severity=HIGH。
- **单组上限**：单个 Gx UPF 组最多 300 个 UPF；本命令最大记录数 300 · severity=MEDIUM。
- **UPFGRPNAME 字符规则**：1~32 字符，不支持空格及 `#`/`$`/`&` 等特殊字符 · severity=LOW。
- **跨组复用**：同一 UPF 可绑不同组，不冲突 · severity=LOW。
