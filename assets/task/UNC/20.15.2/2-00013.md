# 基于初始接入位置的策略控制（WSFD-211001）

> 特性静态知识见 [WSFD-211001](feature/UNC/20.15.2/WSFD-211001.md)。本页讲配置生成实例化时这个特性**怎么配**。

## 配置概览

核心对象链：**用户位置**（USRLOCATION，代际 CGI/ECGI/NCGI）→ **位置组**（USRLOCATIONGRP）→ 绑 **用户模板组**（UPBINDUPG）→ 绑 **APN**。策略动作（如带宽控制）在规则中配置，引用 [BWM](task/UNC/20.15.2/2-00010.md) / [PCC](task/UNC/20.15.2/2-00005.md)。

三种部署场景（DP）：有 PCRF 动态（UNC 仅 License）/ 有 PCRF 预定义 或 无 PCRF 本地（UNC 配位置链）。

## 配置流程（步骤编排，每步显示含命令）

1. **License 前置**（本特性专属）：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)
2. **配置用户位置与模板绑定**（位置链，代际分叉）：`ADD USRLOCATION` + `ADD USRLOCATIONGRP` + `MOD UPBINDUPG` → [1-00019](task/UNC/20.15.2/1-00019.md)
3. **绑 APN**（用户模板组绑 APN）：`ADD APNUSRPROFG` → [0-00008](task/UNC/20.15.2/0-00008.md)

> **activation 演示范围脚注（R1.4）**：WSFD-211001 仅 1 份 activation（[激活基于初始接入位置的策略控制_27915138](evidence/UNC/20.15.2/激活基于初始接入位置的策略控制_27915138.md)），演示 **5 命令**完整脚本：SET LICENSESWITCH + ADD USRLOCATION（CGI/ECGI/NCGI 三代际同脚本演示：loc_test1=CGI/LAC+CI、loc_test2=ECGI/ECI、loc_test3=NCGI/NCI）+ ADD USRLOCATIONGRP + MOD UPBINDUPG + ADD APNUSRPROFG。**无零证据差异参数**——步骤 1-3 全部命令 + 代际分叉参数（LOCATIONTYPE/位置码 LAC+CI vs ECI vs NCI）均在 activation §任务示例脚本原样演示。代际差异完全由 ADD USRLOCATION 的 LOCATIONTYPE + 位置码参数承载（数据规划表列示 3 行取值样例对应三代际），配置生成按本特性 DP1 代际分叉产出对应位置码参数即可。

## 决策点（特性级：部署场景选择）

| 场景 | 走法 | UNC 配置 |
|---|---|---|
| 有 PCRF/PCF 动态 | 仅 License（PCRF 决策，UNC 无新增） | 步骤 1 |
| 有 PCRF/PCF 预定义规则 | License + 位置链 + 绑 APN（本页配置） | 步骤 1-3 |
| 无 PCRF/PCF 或本地 PCC | License + 位置链 + 绑 APN + 本地规则（引用 BWM/PCC） | 步骤 1-3 + [2-00010 BWM](task/UNC/20.15.2/2-00010.md) / [2-00005 PCC](task/UNC/20.15.2/2-00005.md) |

## 约束（特性级 rule）

- **License 前置**（critical）：`SET LICENSESWITCH` 开启 `LKV2PCIAL01` 是第一步 — 否则特性不生效。
- **依赖 BWM**（info）：策略动作若为带宽控制，须先完成 [2-00010 基于业务感知的带宽控制](task/UNC/20.15.2/2-00010.md)。
- **代际分叉**（warning）：LOCATIONTYPE=CGI/ECGI/NCGI 决定位置码参数（2/3G LAC+CI / 4G ECI / 5G NCI）。
