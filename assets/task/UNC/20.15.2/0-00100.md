# 配置计费星期表（ADD WEEKDAY）

> 命令静态知识见 [ADD WEEKDAY](command/UNC/20.15.2/ADD-WEEKDAY.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"全局标志 + 计费属性 + 周日期 + 费率类型"配置，每周每天各一条（典型 7 条覆盖整周）。

**维度/参数/取值/默认/作用/适用场景对照表：**

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 全局标志 | `GLOBALFLG` | `GLOBAL` / `CHARGE_CHARACT` | 无 | 指定计费类型：全局统一 or 按计费属性（CC）区分 | 全网统一费率切换选 GLOBAL；分 CC（如语音 CC 与数据 CC 不同费率）选 `CHARGE_CHARACT` |
| 计费属性值 | `CCVALUE` | 1~6 位十六进制（`0x0000`~`0xFFFF`） | 无 | 配合 `GLOBALFLG=CHARGE_CHARACT`，指定特殊 CC 值 | 按 CC 区分时必填；例 `0x0800` |
| 计费属性掩码 | `CCMASK` | 1~6 位十六进制（`0x0001`~`0xFFFF`） | `0xFFFF` | 配合 `CCVALUE` 限定匹配范围，按 CC 配置时可选 | `CHARGE_CHARACT` 模式下需与 `CCVALUE` 联合规划 |
| 计费属性优先级 | `CCPRIORITY` | `0`~`65535` | `0` | 同周日期下不允许相同优先级；配 mask 时必填 | 多 CC 覆盖相同日期需优先级分叉 |
| 周日期 | `DAYOFWEEK` | `MONDAY` / `TUESDAY` / `WEDNESDAY` / `THURSDAY` / `FRIDAY` / `SATURDAY` / `SUNDAY` | 无 | 枚举固定七值，不支持范围缩写 | 每周每天独立一条；全周 7 条 |
| 费率类型 | `TARIFFTYPE` | `WORKDAY` / `WEEKEND` | 无 | 指定本日期归类为工作日或周末 | 典型：周一~周五 WORKDAY，周六~周日 WEEKEND；中东部分区域周五~周六为周末 |

**典型脚本**（按 CC=0x0800 配整周，周一~周五工作日，周六~周日周末）：

```
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",CCPRIORITY=0,DAYOFWEEK=MONDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",CCPRIORITY=0,DAYOFWEEK=TUESDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",CCPRIORITY=0,DAYOFWEEK=WEDNESDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",CCPRIORITY=0,DAYOFWEEK=THURSDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",CCPRIORITY=0,DAYOFWEEK=FRIDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",CCPRIORITY=0,DAYOFWEEK=SATURDAY,TARIFFTYPE=WEEKEND;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",CCPRIORITY=0,DAYOFWEEK=SUNDAY,TARIFFTYPE=WEEKEND;
```

**步骤位置：** `ADD FESTIVAL`（节假日）→ **本命令**（周一~周日的费率类型）→ `ADD TARIFFGROUP`（节假日/工作日/周末的时间段）→ `SET GLBTARIFFGROUP`（全局费率切换组）。

## 决策点

### 决策点：GLOBALFLG 取全局还是按计费属性（DP 0-01238）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| `GLOBALFLG=GLOBAL` | 无需 `CCVALUE`/`CCMASK`/`CCPRIORITY` | 所有用户同一套工作日/周末规则；运营商统一资费策略 |
| `GLOBALFLG=CHARGE_CHARACT` | 必填 `CCVALUE`；可选 `CCMASK`/`CCPRIORITY` | 分 CC 区分费率（如语音 CC 与数据 CC 不同费率）；多业务线独立切换规则 |

### 决策点：TARIFFTYPE 归类与 DAYOFWEEK 映射（DP 0-01239）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 周一~周五 WORKDAY，周六~周日 WEEKEND | `DAYOFWEEK` 对应 `TARIFFTYPE` | 标准工时制度运营商（默认） |
| 周五~周六 WEEKEND（中东部分区域） | `DAYOFWEEK` 对应 `TARIFFTYPE` | 中东运营商周末定义不同 |
| 多优先级 CC 独立星期表 | 每组 `CCVALUE` 独立配 7 条 | 同一网元多业务线需分别切换 |

## 约束

- **立即生效**（info，rule-0-01531）：立即生效——命令执行后无需等待，自动应用

- **最大记录数 119**（warning，rule-0-01532）：最大记录数 119——每天最多 17 条，全周 7×17=119 条上限；超出会导致新增失败

- **CC 配置有效性**（warning，rule-0-01533）：CC 配置有效性——`CHARGE_CHARACT` 模式下：`CCVALUE & CCMASK` 的值必须等于 `CCVALUE` 本身；`CCVALUE & CCMASK` 不得与已有配置的 `CCVALUE & CCMASK` 相同

- **枚举取值约束**（info，rule-0-01534）：枚举取值约束——`DAYOFWEEK` 仅限 `MONDAY`~`SUNDAY`；`TARIFFTYPE` 仅限 `WORKDAY`/`WEEKEND`（不含统一费率）；`CCVALUE`/`CCMASK` 格式为 1~6 位十六进制（`0x/`无前缀/`a-f`/`A-F`/`0-9`）

- **下游一致性**（warning，rule-0-01535）：下游一致性——本命令的 `TARIFFTYPE` 取值须与 `ADD TARIFFGROUP` 的费率类型、`SET GLBTARIFFGROUP` 的全局组引用保持一致，否则费率切换链路断路
