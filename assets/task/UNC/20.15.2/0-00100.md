# ADD WEEKDAY 原子任务
> 命令静态知识见 [ADD WEEKDAY](command/UNC/20.15.2/ADD-WEEKDAY.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是"费率切换"链路的工作日定义环：`ADD FESTIVAL`（节假日）→ **本命令**（周一~周日的费率类型）→ `ADD TARIFFGROUP`（节假日/工作日/周末的时间段）→ `SET GLBTARIFFGROUP`（全局费率切换组）。用户持续在线跨费率类型时，在 00:00:00 切换。统一费率计费不涉及费率切换，本命令不适用。

## 配置方法
按"全局标志 + 计费属性 + 周日期 + 费率类型"配置，每周每天各一条（典型 7 条覆盖整周）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| GLOBALFLG | 本端规划（全局/按计费属性） | `CHARGE_CHARACT`（按 CC 配置） |
| CCVALUE | 本端规划（计费属性值，十六进制） | `0x0800` |
| CCMASK | 本端规划（计费属性掩码，十六进制，按 CC 配置时填） | `0x0800` |
| CCPRIORITY | 本端规划（计费属性优先级，按 CC 配置时填） | `0`~`65535` |
| DAYOFWEEK | 全网规划（周日期枚举） | `MONDAY`~`SUNDAY` |
| TARIFFTYPE | 全网规划（费率类型） | `WORKDAY`、`WEEKEND` |

典型脚本（按 CC=0x0800 配整周，周一~周五工作日，周六~周日周末）：
```
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",DAYOFWEEK=MONDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",DAYOFWEEK=TUESDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",DAYOFWEEK=WEDNESDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",DAYOFWEEK=THURSDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",DAYOFWEEK=FRIDAY,TARIFFTYPE=WORKDAY;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",DAYOFWEEK=SATURDAY,TARIFFTYPE=WEEKEND;
ADD WEEKDAY: GLOBALFLG=CHARGE_CHARACT,CCVALUE="0x0800",DAYOFWEEK=SUNDAY,TARIFFTYPE=WEEKEND;
```

## 决策点

### GLOBALFLG：全局还是按计费属性
- 问什么：GLOBALFLG 取全局值还是 CHARGE_CHARACT。
- 全局模式为所有用户提供默认费率切换；CHARGE_CHARACT 模式按计费 CC 区分（不同 CC 走不同费率切换）。需按运营商资费策略选——若所有用户同一套工作日/周末规则用全局；若分 CC（如语音 CC 与数据 CC 不同费率）用 CHARGE_CHARACT 并填 CCVALUE/CCMASK。

### TARIFFTYPE 归类
- 问什么：周一~周日哪几天算 WORKDAY，哪几天算 WEEKEND。
- 典型：周一~周五 WORKDAY，周六~周日 WEEKEND。但部分地区/客户周末定义不同（如中东周末为周五~周六）。配置生成需按运营商资费日历产出，非硬编码。

### 是否需要为每个 CC 单独配
- 问什么：多个 CC 是否各配一套星期表。
- CHARGE_CHARACT 模式下，每个 CCVALUE 需独立配 7 天。CC 越多记录数增长快，受 119 条上限约束。

## 约束

- **立即生效** · severity=LOW。
- **最大记录数 119**：每天最多 17 条，全周 7×17=119 · severity=HIGH。
- **CC 自洽**：不允许配置 CCVALUE 和 CCMASK 按位与后的值不等于 CCVALUE · severity=HIGH。
- **CC 唯一性**：GLOBALFLG=CHARGE_CHARACT 时，不允许 CCVALUE&CCMASK 与已有配置的 CCVALUE&CCMASK 相等 · severity=HIGH。
- **DAYOFWEEK/TARIFFTYPE 枚举**：DAYOFWEEK 仅 MONDAY~SUNDAY，TARIFFTYPE 仅 WORKDAY/WEEKEND（不含统一费率） · severity=MEDIUM。
- **CCVALUE/CCMASK 格式**：1~6 字符十六进制（`0x/X`、`a-f/A-F`、`0-9`） · severity=LOW。
- **下游一致性**：本命令的 TARIFFTYPE 须与 `ADD TARIFFGROUP` 的费率类型、`SET GLBTARIFFGROUP` 的全局组引用一致 · severity=HIGH。
