# 增加RADIUS用UP列表成员（ADD UPLIST4RDS）

> 命令静态知识见 [ADD UPLIST4RDS](command/UNC/20.15.2/ADD-UPLIST4RDS.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令向 RADIUS 服务器使用的 UP 列表增加 UP 实例。该 UP 列表用于 SMF 根据 UP 选择 RADIUS 服务器并向其发送 RADIUS 消息。配置空间仅 **UPLISTNAME × UPINSTANCEID 绑定关系**（两必选参数）。

### 配置维度1：UP 列表 × UP 实例绑定（参数 UPLISTNAME / UPINSTANCEID）

| 取值关系 | 作用 | 典型场景 |
|---|---|---|
| UPLISTNAME（UP 列表名，需全网规划）+ UPINSTANCEID（UP 实例标识，需全网规划） | 将 UP 实例加入指定 UP 列表 | WSFD-011306：uplist1 + up1/up2（双 UP）；WSFD-108007：uplist1 + up1 |

> UP 列表通过多条 `ADD UPLIST4RDS` 累积成员；同一 UPLISTNAME 下可挂多个 UPINSTANCEID。

**步骤位置**：在 RADIUS UP 列表族（WSFD-011306/WSFD-108007）编排中，是 UP 列表的起点——于 `ADD UPNODE`（建 UP 实例）之后、`ADD RDSUPFCTRL`（为该 UP 设优先级/锁定）、`ADD RDSSVR`（服务器引用此 UPLISTNAME）之前执行。

## 决策点

（无。本命令在所有特性下用法一致，无分支——两必选参数 UPLISTNAME/UPINSTANCEID 决定唯一绑定关系，由 `ADD RDSUPFCTRL` 后续控制该 UP 的优先级与锁定。）

## 约束

- **UPLISTNAME 命名约束**（warning）：字符串 1~63，非空格字符 — 后果：违规命名新增失败
- **UPINSTANCEID 命名约束**（warning）：字符串 1~36 — 后果：违规命名新增失败
- **UPINSTANCEID 应已规划**（info）：UPINSTANCEID 应对应 `ADD UPNODE` 已建的 UP 实例标识 — 后果：未规划则后续 `ADD RDSUPFCTRL` 失败
- **(UPLISTNAME, UPINSTANCEID) 唯一**（critical）：同一 UP 列表下不能重复加同一 UP 实例 — 后果：重复新增失败
