# ADD PCFSSCOPE 原子任务
> 命令静态知识见 [ADD PCFSSCOPE](command/UNC/20.15.2/ADD-PCFSSCOPE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令建立"服务区标识（SSCOPEID，本端标识）→ 服务区名称（SSCOPENAME，与 PCF 向 NRF 注册时携带的服务区名称一致）"的映射。SMF 按语音用户 TAI 区域 → 查 PCFSSCOPEBIND 得 SSCOPEID → 查本表得 SSCOPENAME → 携 SSCOPENAME 向 NRF 查可用 PCF。

| 配置项 | 取值来源 | 典型脚本 |
|---|---|---|
| SSCOPEID（服务区标识） | 本端规划，多条记录时不重复；存储转小写 | `ADD PCFSSCOPE:SSCOPEID="citya",SSCOPENAME="City_A";` |
| SSCOPENAME（服务区名称） | 全网规划，须与 PCF 向 NRF 注册的服务区名称**完全一致**（区分大小写） | （见上行） |

**与三件套联动**：SMF 需按 SERVINGSCOPE 选 PCF 时，依次配 ADD PCFSSCOPE（本命令，建服务区）→ [0-00097](task/UNC/20.15.2/0-00097.md)（ADD USRTAIRANGE，建 TAI 区域）→ [0-00056](task/UNC/20.15.2/0-00056.md)（ADD PCFSSCOPEBIND，绑 TAI 区域到服务区）。

## 决策点

### SSCOPENAME 取值来源
问：服务区名称取本端规划还是全网规划？
- **全网规划**：SSCOPENAME 必须与 PCF 向 NRF 注册时携带的服务区名称一致（区分大小写），属于跨网元全网协同数据，非本端独立决定。
- SSCOPEID 是本端局部标识（不区分大小写，转小写存储），可本端自由规划。

### 是否需要配 PCFSSCOPE
问：SMF 是否需要根据 SERVINGSCOPE 选择 PCF？
- **需要**（如语音用户按 TAI 区域选 PCF）：配本命令 + USRTAIRANGE + PCFSSCOPEBIND。
- **不需要**：跳过本步。若 PCFSSCOPE 无记录，SMF 按 PNFSLCTSSCOP 软参逻辑选 PCF。

## 约束
- SSCOPEID 长度 1~32，不区分大小写，转小写存储，多条记录时不重复。`severity=high`
- SSCOPENAME 长度 1~50，**区分大小写**，须与 PCF 向 NRF 注册的服务区名称完全一致。`severity=critical`
- 兜底行为：若 PCFSSCOPE 有记录但激活请求未携带 TAI / TAI 在 PCFSSCOPEBIND 中无匹配 SSCOPEID → 从 PCFSSCOPE 记录中**随机选**一个 SSCOPENAME 用于 PCF 服务发现。`severity=medium`
- 若 PCFSSCOPE 无记录 → 按 PNFSLCTSSCOP 软参逻辑选 PCF。`severity=low`
- 适用 NF：SMF、PGW-C、GGSN。`severity=low`
- 执行后立即生效。`severity=low`
