# 配置 QoS 属性与规则绑定链

> 业务触发 QoS 保证的规则承载层——先配 QoS 使用量上报规则（USAGERPTMODE=QOS），再配 QoS 属性（GBR/MBR/5QI/QCI/ARP 抢占），经 PCC 策略组或直接绑到 RULE，最后 RULE 经 USERPROFILE+RULEBINDING 承载，供 UPF 业务识别后上报 QoS 事件触发专有承载/专有 QoS Flow 建立。被引用于 [2-00008 业务触发的QoS保证](task/UNC/20.15.2/2-00008.md)。域通用（QoS 保证族 RULE 绑定结构；RULE/USERPROFILE/RULEBINDING 命令跨 PCC 族共享但 QoS 场景参数集不同，见场景差异）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 配 QoS 使用量上报规则 | `ADD URR` | URRNAME（SMF/UPF 一致）；URRID（全局唯一，建议>1000）；**USAGERPTMODE=QOS**（固定，区别于计费的 ONLINE/OFFLINE） |
| 配 QoS 属性（5G） | `ADD QOSPROP` | QOSPROPNAME；GBRUPLKVALUE/GBRDNLKVALUE（保证比特率 kbps）；MBRUPLKVALUE/MBRDNLKVALUE（最大比特率）；**QOSTYPE=QOS_FLOW_PARA**；ARPVALUE（值越小优先级越高）；**FQI**（5QI）；QOSURRNAME（引用上步 URR）；EMPCAP/EMPVUL（抢占能力/被抢占） |
| 配 QoS 属性（2/3/4G） | `ADD QOSPROP` | 同上，但 **QOSTYPE=QOS_BEARER_PARA** + **QCIVALUE**（QCI）替代 FQI |
| 配 PCC 策略组（方式一用） | `ADD PCCPOLICYGRP` | PCCPOLICYGRPNM；QOSPROPNAME（引用上步 QOSPROP）—— 本步**方式一直接绑定用；方式二跳过** |
| 配 Rule | `ADD RULE` | RULENAME（三网元一致）；POLICYTYPE=PCC（绑 PCCPOLICYGRPNM）/ QOS（直接绑 QOSPROPNAME）；POLICYNAME |
| 配用户模板 | `ADD USERPROFILE` | USERPROFILENAME（全网规划）；UPTYPE=PCC（固定） |
| 绑 Rule 到用户模板 | `ADD RULEBINDING` | USERPROFILENAME；RULENAME；POLICYTYPE（与 ADD RULE 一致） |

**典型脚本（5G 用户，方式一：PCC 策略组绑定）**：
```
ADD URR: URRNAME="URR_qos_01", URRID=1005, USAGERPTMODE=QOS;
ADD QOSPROP: QOSPROPNAME="qosprop_501", GBRUPLKVALUE=600, GBRDNLKVALUE=6000, MBRUPLKVALUE=1000, MBRDNLKVALUE=10000, QOSTYPE=QOS_FLOW_PARA, ARPVALUE=5, FQI=2, QOSURRNAME="URR_qos_01", EMPCAP=DISABLE, EMPVUL=ENABLE;
ADD PCCPOLICYGRP: PCCPOLICYGRPNM="pccpolicy_group", QOSPROPNAME="qosprop_501";
ADD RULE: RULENAME="rule_test", POLICYTYPE=PCC, POLICYNAME="pccpolicy_group";
ADD USERPROFILE: USERPROFILENAME="up_test", UPTYPE=PCC;
ADD RULEBINDING: USERPROFILENAME="up_test", RULENAME="rule_test", POLICYTYPE=QOS;
```

**典型脚本（2/3/4G 用户，方式二：直接绑 QoS 属性，跳过 PCCPOLICYGRP）**：
```
ADD URR: URRNAME="URR_qos_02", URRID=1006, USAGERPTMODE=QOS;
ADD QOSPROP: QOSPROPNAME="qosprop_401", GBRUPLKVALUE=600, GBRDNLKVALUE=6000, MBRUPLKVALUE=1000, MBRDNLKVALUE=10000, QOSTYPE=QOS_BEARER_PARA, QCIVALUE=66, ARPVALUE=5, QOSURRNAME="URR_qos_02";
ADD RULE: RULENAME="rule_l34_test", POLICYTYPE=QOS, POLICYNAME="qosprop_401";
ADD USERPROFILE: USERPROFILENAME="up_l34_test", UPTYPE=PCC;
ADD RULEBINDING: USERPROFILENAME="up_l34_test", RULENAME="rule_l34_test", POLICYTYPE=QOS;
```

**步骤位置**：在 License 前置（[SET LICENSESWITCH](task/UNC/20.15.2/0-00180.md) 开 LKV3W9STQE11）之后；本 compound 完成规则承载，之后 UserProfile 组与 APN 绑定（[2-00008](task/UNC/20.15.2/2-00008.md) 步骤 4，复用 atom）+ 可选定时器/去活策略（步骤 5-6）。

## 场景差异

- **5G 用户（QOS_FLOW_PARA）**：ADD QOSPROP 用 QOSTYPE=QOS_FLOW_PARA + FQI（5QI，如 2=对话型语音）；触发**专有 QoS Flow** 建立；可选配 SET APNIDLETIME 的 DEDQFIDLETIMER（专有 QoS Flow 空闲定时器）+ ADD APNDEACTQFPLCY（UE 无线丢失延迟释放）。被引用于 [2-00008 业务触发的QoS保证](task/UNC/20.15.2/2-00008.md)。
- **2/3/4G 用户（QOS_BEARER_PARA）**：ADD QOSPROP 用 QOSTYPE=QOS_BEARER_PARA + QCIVALUE（QCI，如 66）；触发**专有承载**建立；无独立空闲定时器参数（2/3/4G 承载老化），无延迟释放策略。被引用于 [2-00008 业务触发的QoS保证](task/UNC/20.15.2/2-00008.md)。
- **规则绑定方式二选一（PCC 策略组 vs 直接绑 QoS 属性）**：方式一 POLICYTYPE=PCC 经 ADD PCCPOLICYGRP 间接绑 QOSPROP（多规则共享一组 QoS 属性时用）；方式二 POLICYTYPE=QOS 直接绑 QOSPROPNAME（单规则独占 QoS 属性时用，跳过 ADD PCCPOLICYGRP）。**PCC 类型的 RULENAME 值与 QOS 类型的 RULENAME 值不能相同**。
- **USAGERPTMODE=QOS 是 QoS 专属变种**（区别于计费费率标识链 [1-00009](task/UNC/20.15.2/1-00009.md) 的 ONLINE/OFFLINE）：URR 在此不承载费率，仅做 QoS 事件上报（UPF 业务识别后按 URRID 上报 QoS 事件）。故无 URRGROUP、无 SET URRGRPBINDING（计费专属），URR 与 QOSPROP 经 QOSURRNAME 一对一关联。
- **UPTYPE=PCC（与计费族/ADC 共享）**：USERPROFILE 复用 PCC 类型模板，QoS 保证通过 RULEBINDING 挂 QoS 规则（与 ADC [1-00016](task/UNC/20.15.2/1-00016.md) 挂 ADC 规则同构，区别在 POLICYTYPE=PCC/QOS vs ADC）。
- **三网元 RULENAME 一致**：网络中存在 PCRF/PCF 时，UPF、SMF、PCRF/PCF 上配置的预定义规则名称必须一致，否则不能达到预期效果。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| QOSTYPE=QOS_FLOW_PARA | 5G；配 FQI（5QI）；触发专有 QoS Flow 建立；可配 DEDQFIDLETIMER/APNDEACTQFPLCY |
| QOSTYPE=QOS_BEARER_PARA | 2/3/4G；配 QCIVALUE（QCI）；触发专有承载建立；无独立空闲定时器/延迟释放 |
| POLICYTYPE=PCC（方式一） | 经 ADD PCCPOLICYGRP 间接绑 QOSPROP；多规则共享 QoS 属性 |
| POLICYTYPE=QOS（方式二） | 直接绑 QOSPROPNAME；单规则独占 QoS 属性；跳过 ADD PCCPOLICYGRP |
| EMPCAP=ENABLE | 该业务可抢占已分配给低优先级业务流的资源 |
| EMPVUL=ENABLE | 本业务流资源可被高优先级业务抢占 |

> 联动特性级 DP：代际（5G/2-3-4G）与规则绑定方式（PCC/QOS）由 [2-00008](task/UNC/20.15.2/2-00008.md) DP1/DP2 驱动；本 compound 在 License 前置后编排。

> ★复用警示（命令多带陷阱，SOP §3.5）：本 compound 典型脚本演示方式一（POLICYTYPE=PCC 经 PCCPOLICYGRP 间接绑）。方式二（POLICYTYPE=QOS 直接绑 RULE）**跳过 ADD PCCPOLICYGRP**——配置生成按方式二产出时勿套用典型脚本的 ADD PCCPOLICYGRP 段。feature DP2 已驱动分支，本警示作 compound 层防御。

## 约束

- **USAGERPTMODE=QOS 固定**（critical）：QoS 保证的 URR 必须 USAGERPTMODE=QOS（区别于计费的 ONLINE/OFFLINE） — 否则 UPF 不上报 QoS 事件，不触发专有承载/QoS Flow 建立。
- **URRID/QOSPROPNAME/RULENAME 跨网元一致**（critical）：SMF（UNC）与 UPF（UDG）的 URRID/RULENAME/POLICYTYPE/POLICYNAME 须一致；存在 PCRF/PCF 时三网元 RULENAME 须一致 — 否则 QoS 事件无法匹配、策略无法生效。
- **PCC 类型与 QOS 类型 RULENAME 不可相同**（critical）：PCC 类型的 RULENAME 值与 QOS 类型的 RULENAME 值不能相同 — 否则规则冲突。
- **GBR/MBR 须满足 GBR≤MBR**（warning）：GBRUPLKVALUE/GBRDNLKVALUE 须不大于对应 MBRUPLKVALUE/MBRDNLKVALUE — 否则 QoS 保证无法兑现。
- **QOSURRNAME 须引用已建 URR**（critical）：ADD QOSPROP 的 QOSURRNAME 须为已 ADD URR 定义的 URRNAME — 否则 QoS 属性引用失败。
- **依赖 PCC 基本功能**（critical）：从 PCRF/PCF 获取预定义规则时须已完成 [2-00005 PCC基本功能](task/UNC/20.15.2/2-00005.md) 激活 — 否则无 PCC 通道承载。
- **License 前置**（critical）：SET LICENSESWITCH 激活 LKV3W9STQE11 须先于本 compound — 否则 QoS 保证功能不生效。
- **生效时机**（info）：执行后立即生效；DEDQFIDLETIMER 修改仅对新激活用户生效。
