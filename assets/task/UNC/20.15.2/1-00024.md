# UNC APN 接入域基础设施（L3VPN+VPN+APN+APNADDRESSATTR）

> **UNC 域 APN 接入域底座**。被引用于 21 个 APN 域特性。**注**：UNC 域特性建设时**总是先建本 compound 再走特性主骨架**。
>
> 设计参考 UDG 域 `1-00029` APN 接入基础设施结构（结构对齐，UNC 命令族独立维护）。

## 配置方法

本 compound 建 VPN/APN 实例 + 使能地址分配属性，为后续地址分配族（[UNC 1-00023](task/UNC/20.15.2/1-00023.md)）/Radius 族（UNC 1-00025）/L2TP/GRE/IPSec/MPLS 等特性提供接入域基础。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. L3VPN 实例 | `ADD L3VPNINST` → [0-00040](task/UNC/20.15.2/0-00040.md) | VRFNAME=vpn 名（与外联口、地址池三者一致）|
| 2. L3VPN 地址族 | `ADD VPNINSTAF` → [0-00099](task/UNC/20.15.2/0-00099.md) | VRFNAME, AFTYPE=ipv4uni/ipv6uni, VRFRD=100:1 |
| 3. 用户面 VPN 实例 | `ADD VPNINST` → [0-00098](task/UNC/20.15.2/0-00098.md) | VPNINSTANCE=vpn 名 |
| 4. APN/DNN 实例（绑 VPN）| `ADD APN` → [0-00004](task/UNC/20.15.2/0-00004.md) | APN=apn 名, HASVPN=ENABLE, VPNINSTANCE=vpn 名 |
| 5. 使能地址分配属性 | `SET APNADDRESSATTR` → [0-00270](task/UNC/20.15.2/0-00270.md) | APN=, SUPPORTIPV4/SUPPORTIPV6=ENABLE（IPV4/V6ALLOCTYPE 由 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 的 POOLTYPE 决定：Local/UDM/Radius/DHCP）|

**典型脚本（IPv4 Local 地址分配，2-00015 场景基线）**：
```
ADD L3VPNINST: VRFNAME="vpn1";
ADD VPNINSTAF: VRFNAME="vpn1", AFTYPE=ipv4uni, VRFRD="100:1";
ADD VPNINST: VPNINSTANCE="vpn1";
ADD APN: APN="apn-test1", HASVPN=ENABLE, VPNINSTANCE="vpn1";
SET APNADDRESSATTR: APN="apn-test1", SUPPORTIPV4=ENABLE, IPV4ALLOCTYPE=Local;
```

**步骤位置**：所有 APN 域特性的**第一步**（License 前置之后）；下游 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 地址池体系 / [UNC 1-00029](task/UNC/20.15.2/1-00029.md) OSPF 路由发布。

## 决策点（UNC pre-build 内嵌，不编号）

| 选项（IP 类型/子方式）| 影响（SET APNADDRESSATTR + ADD APN 参数）|
|---|---|
| IPv4 Local | SUPPORTIPV4=ENABLE；IPV4ALLOCTYPE=Local |
| IPv6 Local | SUPPORTIPV6=ENABLE；IPV6ALLOCTYPE=Local |
| IPv4v6 双栈 | SUPPORTIPV4+IPV6=ENABLE；ADD APN 加 HASVPNIPV6=ENABLE+VPNINSTANCEIPV6= |
| UDM 静态（地址分配特性 4 子方式之一）| IPV4ALLOCTYPE=UDM + CHECKIPVALID=ENABLE（由 1-00023 配合）|
| Radius 分配（4 子方式之一）| IPV4ALLOCTYPE=Radius |
| DHCP 分配（4 子方式之一）| IPV4ALLOCTYPE=DHCP |
| LNS 分配（L2TP）| 不通过本特性，走 WSFD-104410 L2TP VPN（SET APNL2TPCTRL）|

> 联动特性级 DP：IP 类型/子方式由各 feature 场景 DP 决定。

## 约束

- **VPN 三者一致**（critical）：L3VPN/VPNINST 的 VRFNAME、ADD APN 的 VPNINSTANCE、外联口绑定的 VPN 三者必须一致 — 后果：VPN 不一致致地址分配或路由异常。
- **APN/DNN 全网规划一致**（critical）：UNC 域配置 APN/DNN 实例应与 C 面/外部网元配置一致 — 后果：NAS 信令失配。
- **AFTYPE 与 IP 类型匹配**（warning）：IPv4 场景 AFTYPE=ipv4uni；IPv6 场景 AFTYPE=ipv6uni；双栈两者都配 — 后果：地址族不匹配致 VPN 实例失效。
- **本 compound 不含独立命令外的命令**：完整特性配置由各 feature 主骨架承担。
- **License 前置按 feature**：部分 feature（如 IPSec 国密）需 `SET LICENSESWITCH` 前置——本 compound 不含 License 命令。
