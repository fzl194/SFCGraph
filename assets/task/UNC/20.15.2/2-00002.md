# 热计费（WSFD-011202）

> 特性静态知识见 [WSFD-011202](feature/UNC/20.15.2/WSFD-011202.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系（必读）**：本特性 = [WSFD-011201 离线计费](feature/UNC/20.15.2/WSFD-011201.md) 的**参数变种**——activation 文档原文"热计费功能的配置与离线计费的配置类似，请参见 [激活离线计费]"，命令集完全复用离线计费，差异仅在参数取值（见决策点）。配置生成可复用离线计费的 compound 骨架，按本特性 DP 注入热计费参数。

## 配置概览

核心对象链：与离线计费**完全一致**——**OFCTemplate**（话单版本+计费方式+阈值+话单触发）→ 绑定到 `{UserProfile | APN | 计费属性CC | 全局}` 之一。

热计费与普通离线计费的差异**全部在参数层**（activation 文档明示）：
1. **计费属性 CC 取值**：热计费 `HOME/ROAM/VISIT=0x100`（hotbilling 标志），普通计费为 `0x0800`（normal）。这是热计费的本质差异——CG 优先处理带 0x100 标志的话单。
2. **阈值更小**：话单产生更快（如 VOLUMETHRESHOLD=1000 vs 普通 20，TIMETHRESHOLD=30），保证运营商及时收费。
3. **离线开关**：`SET CHARGECTRL` 显式使能离线计费（HOMEOFFLINE/VISITOFFLINE/ROAMOFFLINE=ENABLE）——热计费是离线计费增强，必须开离线开关。> **ROAMOFFLINE 标注（R1.5）**：activation 脚本仅演示 `HOMEOFFLINE=ENABLE,VISITOFFLINE=ENABLE`（[激活支持热计费功能_28072077](evidence/UNC/20.15.2/激活支持热计费功能_28072077.md) 脚本行），**ROAMOFFLINE=ENABLE 未在脚本中演示**（仅数据规划表列示）。配置生成产出脚本时：HOMEOFFLINE/VISITOFFLINE 按脚本产出（有证据）；**ROAMOFFLINE 须按全网规划确认后产出，不可默认编造为 ENABLE**。
4. **APN 绑定参数增量**：activation 演示的 `SET APNCHARGECTRL` 带 `ONLINESW=DISABLE, OFFLINESW=ENABLE, GGSNOFCTEMPLATE, CCNAME`（显式关闭在线、带上 GGSN 模板与计费属性名）。

4 种配置层次**共享离线计费同一套 compound 骨架**（[1-00001](task/UNC/20.15.2/1-00001.md) + 绑定 compound），不建新流程。

## 配置流程（步骤编排，每步显示含命令）

1. **使能离线计费开关**（热计费前置，离线计费增强特性）：`SET CHARGECTRL` → [0-00157](task/UNC/20.15.2/0-00157.md)
   - 关键参数：HOMEOFFLINE=ENABLE, VISITOFFLINE=ENABLE（activation 脚本演示）；ROAMOFFLINE activation **脚本未演示**（仅数据规划表列示 ENABLE），按全网规划确认后产出，不可默认 ENABLE
2. **配置模板参数**（话单版本+计费方式+阈值+话单触发，复用离线计费骨架）：`ADD OFCTEMPLATE` + `SET OFCTHRESHOLD` + `SET CDRTRIGGER` + `SET CONTAINERTRIGGER` → [1-00001](task/UNC/20.15.2/1-00001.md)
   - 热计费参数增量（见 DP）：阈值取更小值（VOLUMETHRESHOLD=1000 等）
3. **配热计费属性 CC**（建 CC 并打热计费标志 0x100）：`ADD CHARGECHAR` → [0-00014](task/UNC/20.15.2/0-00014.md)
   - 关键参数：CCNAME=HOTBILLING；HOME/ROAM/VISIT=0x100（热计费标志）；HOMESGSN/ROAMSGSN/VISITSGSN=DISABLE
4. **绑到配置层次**（按 DP 选，四选一，复用离线计费绑定结构）：
   - APN 层（activation 演示路径）：`ADD APN` + `SET APNCHARGECTRL` → [1-00003](task/UNC/20.15.2/1-00003.md)
     - 热计费参数增量：SET APNCHARGECTRL 带 `ONLINESW=DISABLE, OFFLINESW=ENABLE, GGSNOFCTEMPLATE=模板名, CCNAME=热计费CC名`
   - UserProfile 层（复用离线结构，activation 未演示）：`ADD USERPROFILE` + `SET USRPROFCHARGE` + `ADD USRPROFGROUP` + `ADD UPBINDUPG` + `ADD APNUSRPROFG` → [1-00002](task/UNC/20.15.2/1-00002.md)
   - 计费属性 CC 层（复用离线结构，activation 未演示）：`ADD GLBOFCTEMPLATE` → [1-00004](task/UNC/20.15.2/1-00004.md)
   - 全局层（兜底，单命令）：`ADD GLBOFCTEMPLATE` → [0-00033](task/UNC/20.15.2/0-00033.md)

> **activation 演示范围脚注（R1.4）**：热计费 activation 文档原文复用离线计费 activation，仅演示 **APN 层**绑定路径（步骤 4 第 1 项）。UserProfile/CC/全局三层**activation 未单独演示**，取值来自离线计费 activation 结构复用 + 本特性 DP 参数增量。配置生成按 APN 层路径产出脚本为有证据基准；其余层次走离线计费同构骨架。

## 约束（特性级 rule）

- **依赖离线计费**（critical）：热计费是离线计费增强特性，须先 `SET CHARGECTRL` 使能离线开关 — 否则热计费话单不产生。
- **CC 标志值**（critical）：`ADD CHARGECHAR` 的 HOME/ROAM/VISIT 须取 `0x100`（hotbilling）— 否则话单计费属性为 normal(0x0800)，CG 不优先处理。
- **阈值更小**（warning）：热计费话单产生速度要求更快，VOLUMETHRESHOLD/TIMETHRESHOLD 应小于普通离线计费 — 否则失去热计费意义。
- **模板名是引用键**（critical）：OFCTEMPLATENAME 须先建（[1-00001](task/UNC/20.15.2/1-00001.md)）再绑定。
- **4 层次降级链**（warning）：UserProfile > APN > CC > 全局，高一级未配取下一级（同离线计费）。
