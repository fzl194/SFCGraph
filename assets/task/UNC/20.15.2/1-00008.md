# 配置 SMF-CHF 交互 Trigger 与 RG 老化

> SMF 与 CHF 交互条件及内容配置——CHF 交互使能 + Session/RG 级 Trigger + RG 老化，融合计费交互的核心参数。被引用于 [2-00003 融合计费](task/UNC/20.15.2/2-00003.md)。特性专属（离线/热计费无 Nchf 交互，无此模块）。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| CHF 交互使能（PDU 会话建立时） | `SET CHFINIT` | CCTMPLTNAME；CHFINIT=SENDREQ（交互）/NOTSENDREQ（不交互）；CCRINITRGNUM（初始请求 RG 个数） |
| Session 级 Trigger | `ADD PDUTRIGGER` | CCTMPLTNAME；QOSCHG/SRVNDCHG/RATCHG/PRACHG/ADDUPF/INSERTISMF/CGISAICHG/RAICHG（IMMEDIATE/DEFERRED） |
| RG 级 Trigger | `ADD RGTRIGGER` | CCTMPLTNAME；ULCHG/PRACHG/PSDATAOFFCHG（IMMEDIATE/DEFERRED/NONREPORT） |
| RG 老化功能 | `SET RGRESCTRL` | ONLRGAGESW/ONLAGETIMER/ONLBLKTIMER（在线）；OFLRGAGESW/OFLAGETIMER（离线） |

**典型脚本**：
```
SET CHFINIT: CCTMPLTNAME="cct_test", CHFINIT=SENDREQ, CCRINITRGNUM=5;
ADD PDUTRIGGER: CCTMPLTNAME="cct_test", QOSCHG=IMMEDIATE, SRVNDCHG=DEFERRED, RATCHG=IMMEDIATE, PRACHG=DEFERRED, ADDUPF=IMMEDIATE, INSERTISMF=DEFERRED, CGISAICHG=DEFERRED, RAICHG=DEFERRED;
ADD RGTRIGGER: CCTMPLTNAME="cct_test", ULCHG=NONREPORT, PRACHG=IMMEDIATE, PSDATAOFFCHG=IMMEDIATE;
SET RGRESCTRL: ONLRGAGESW=ENABLE, ONLAGETIMER=20, ONLBLKTIMER=0, OFLRGAGESW=ENABLE, OFLAGETIMER=20;
```

**步骤位置**：紧跟 [1-00005](task/UNC/20.15.2/1-00005.md)（CCT 模板参数）之后；CCT 模板名是 Trigger 的引用键。

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| CHFINIT=SENDREQ | PDU 会话建立时发 ChargingDataRequest 与 CHF 交互（在线计费必须；离线计费也建会话） |
| CHFINIT=NOTSENDREQ | 不发交互请求（仅按 Trigger 上报） |
| Session 级 Trigger=IMMEDIATE | 触发时立即上报 CHF（如 QOSCHG/RATCHG/ADDUPF） |
| Session 级 Trigger=DEFERRED | 记录但不立即发送，下一条请求携带（如 SRVNDCHG/PRACHG/INSERTISMF） |
| RG 级 Trigger=NONREPORT | 不上报（如 ULCHG） |
| RG 老化开启 | 已停访问业务的 RG 超时老化，TriggerType=Final，避免 N40 消息膨胀 |

> 联动特性级 DP：Trigger 冲突时 Session 级优先于 RG 级；CHF 下发的 Trigger 优先级高于 SMF 本地配置（`ADD PDUTRIGGER`/`ADD RGTRIGGER`）。`SET CNVRGDCHGPARA`（见 [1-00010](task/UNC/20.15.2/1-00010.md)）的 IGNORETRIGGER 可忽略 CHF 下发的指定 Trigger。

## 约束

- **前置 CCT 模板**（critical）：CCTMPLTNAME 须已在 [1-00005](task/UNC/20.15.2/1-00005.md) 建立 — Trigger 命令引用模板名。
- **Trigger 冲突规则**（warning）：Session 级与 RG 级同时生效的 Trigger，以 Session 级优先；CHF 下发优先于本地配置。
- **RG 老化与 QHT 配合**（info）：QHT 依赖 CHF 下发 QHT Trigger；`SET RGRESCTRL` 的老化时长在 QHT 不使能或 QHT=0 时保证 RG 老化。
- **交互频率**（warning）：建议每用户上报 ChargingDataRequest 间隔 > 30 秒，避免 SMF-CHF 高负荷。
