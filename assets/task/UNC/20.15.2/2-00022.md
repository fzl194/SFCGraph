# DHCPv6 地址分配（WSFD-104005）

> 特性静态知识见 [WSFD-104005](feature/UNC/20.15.2/WSFD-104005.md+ [特性知识](业务图谱体系/APN业务域/feature-knowledge/WSFD-104005-DHCPv6地址分配.md（★业务图谱未建，归 R1.5 后续）。本页讲配置生成实例化时这个特性**怎么配**——DHCPv6 子方式下的 DHCPv6 服务器组链路 + 地址池 + APN 维度使能。

## 配置概览

核心对象链：**DHCPSERVERGRP（DHCPv6 服务器组）** → **DHCPSERVER（单台 DHCPv6 服务器 IPv6）** → **DHCPBINDPOOLGRP（DHCPv6 服务器组绑地址池组）** + **ADDRPOOL（POOLTYPE=DHCP，IPv6 SECTION）** + **AGENTIP（UNC 代理 IPv6）** → **SET APNADDRESSATTR（IPV6ALLOCTYPE=DHCP）** + **SET DHCPPARAREQ（DNS Recursive Name Server / SIP Servers IPv6 List）** + **SET IPALLOCRULE（地址分配规则）**。

本特性是 [WSFD-010502](feature/UNC/20.15.2/WSFD-010502.md) 地址分配方式中 **DHCPv6 子方式**的完整展开——结构与 [WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md)（DHCPv4）**对称**，差异在 IP 类型（IPv6 vs IPv4）+ 协议报文（DHCPv6 Solicit/Advertise/Request/Renew/Release/Decline vs DHCPv4 Discover/Offer/Request）。

**依赖前置特性**：本特性需先开 [WSFD-104001](feature/UNC/20.15.2/WSFD-104001.md)（IPv6 承载上下文，License `LKV2IPV601`/`LKV2IPV6SM01`）+ 若为 IPv4v6 双栈用户再加 [WSFD-104002](feature/UNC/20.15.2/WSFD-104002.md)（IPv4v6 双栈接入，License `LKV2DUSA02`/`LKV2IPDSSM01`）。

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（L3VPN + VPN 实例 + APN + 使能 IPv6 DHCPv6 分配）：`ADD L3VPNINST` + `ADD VPNINSTAF`（AFTYPE=ipv6uni）+ `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR`（`SUPPORTIPV6=ENABLE`，`IPV6ALLOCTYPE=DHCP`） → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **SMF 地址池（DHCPv6 子方式）**（建 IPv6 地址池+段+池组，POOLTYPE=DHCP）：`ADD ADDRPOOL`（POOLTYPE=DHCP）+ `ADD SECTION`（IPVERSION=IPv6）+ `ADD ADDRPOOLGRP` + `ADD POOLBINDGRP` + `ADD ADDRUPGROUP` → [UNC 1-00023](task/UNC/20.15.2/1-00023.md)
3. **UNC 代理 IPv6**（DHCPv6 客户端在地址池网段内的代理 IP）：`ADD AGENTIP`
   > **★ R1.5 待补 atom**：`ADD AGENTIP` 命令真实存在 UNC OM（路径：`业务服务管理/5G接入业务管理/控制面管理/会话管理/DHCPv6管理/`），evidence = [WSFD-104005-参考信息](evidence/UNC/20.15.2/WSFD-104005/) 命令清单；UNC pre-build 阶段 atom 未建，归 P1 后续补建。**AGENTIP 必须在地址段内**（critical 约束）。
4. **DHCPv6 服务器组链路**（DHCPv6 子方式专属）：
   - `ADD DHCPSERVERGRP`
   - `ADD DHCPSERVER`
   - `ADD DHCPBINDPOOLGRP`
   > **★ R1.5 待补 atom**（3 条命令）：evidence 同上，UNC pre-build 阶段未建，归 P1 后续补建。
5. **地址分配规则**（按需，全局开关）：`SET IPALLOCRULE`
   > **★ R1.5 待补 atom**：evidence = [WSFD-010504-参考信息_77813716.md](evidence/UNC/20.15.2/WSFD-010504/WSFD-010504-参考信息_77813716.md) 命令清单；UNC pre-build 阶段未建。
6. **（按需）DHCPv6 请求信元参数**（DNS Recursive Name Server / SIP Servers IPv6 List 选项）：`SET DHCPPARAREQ`
   > **★ R1.5 待补 atom**：evidence 同步骤 4；UNC pre-build 阶段未建。
7. **下行路由发布**（让骨干网可达分配的 IPv6 地址）：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFINTERFACE` + `ADD OSPFIMPORTROUTE` → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)
   - 关键参数：PROCID, AREAID, IFNAME（UNC 真实命令名 **OSPFINTERFACE**——v1.0 写的 `OSPFNETWORK` 是错的）
8. **（依赖前置特性）**：开启 [WSFD-104001](feature/UNC/20.15.2/WSFD-104001.md) IPv6 承载上下文（IPv6 RA 前缀广播源）；IPv4v6 双栈用户需叠加 [WSFD-104002](feature/UNC/20.15.2/WSFD-104002.md)

> **★ 与 [WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md) 的关系**：本特性是 104413（DHCPv4）的 IPv6 对称版本——结构对称（步骤 1-7），差异在 IP 类型（IPv6）+ 协议报文（DHCPv6）+ License 不同（`LKV3W9V6AA11` vs `LKV3W9DHCP12`）。

## 决策点（特性级：DHCPv6 承载类型 × 请求维度）

| 选项 × 承载类型 × DNS/P-CSCF 请求 × 地址分配规则 | 走法（复用 compound + 步骤）| 联动 / 规则 |
|---|---|---|
| **DHCPv6 × 纯 IPv6 承载 × 仅地址** | 骨架 [1-00024](task/UNC/20.15.2/1-00024.md)（AFTYPE=ipv6uni）+ 地址池 [1-00023](task/UNC/20.15.2/1-00023.md)（IPv6 SECTION）+ 步骤 3-4 + 路由 [1-00029](task/UNC/20.15.2/1-00029.md) | 依赖 [WSFD-104001](feature/UNC/20.15.2/WSFD-104001.md) |
| **DHCPv6 × 纯 IPv6 承载 × DNS+P-CSCF** | 骨架 1-00024（SUPPORTIPV6）+ 地址池 1-00023（IPv6 SECTION）+ 步骤 3-4 + 步骤 6（`SET DHCPPARAREQ` 开启 DNS Recursive Name Server + SIP Servers IPv6 List）+ 路由 1-00029 | DHCPv6 服务器需支持对应 Option |
| **DHCPv6 × IPv4v6 双栈 × DNS+P-CSCF** | 骨架 1-00024（双 VPN + SUPPORTIPV4+IPV6）+ 地址池 1-00023（IPv4+IPv6 双 SECTION）+ 步骤 3-4 + 步骤 6 + 路由 1-00029 | 依赖 [WSFD-104001](feature/UNC/20.15.2/WSFD-104001.md) + [WSFD-104002](feature/UNC/20.15.2/WSFD-104002.md) |
| **DHCPv6 × 纯 IPv6 承载 × 自定义分配规则** | 骨架 1-00024 + 地址池 1-00023 + 步骤 3-4 + 步骤 5（`SET IPALLOCRULE`）+ 路由 1-00029 | 需配置分配规则参数 |

> **走法列**：4 选项**共享骨架 + 地址池 + DHCPv6 服务器组链路 + 路由 4 骨架 compound**；差异在承载类型（AFTYPE）+ 请求维度（步骤 6）+ 联动特性。

## 约束（特性级 rule）

- **License 前置**（critical）：本特性需 License `LKV3W9V6AA11`（控制项 82209423）授权——`SET LICENSESWITCH` 先开 — 后果：无 License 时功能不可用。
- **依赖 IPv6 承载上下文特性**（critical）：需先开 [WSFD-104001](feature/UNC/20.15.2/WSFD-104001.md) IPv6 承载上下文（License `LKV2IPV601`/`LKV2IPV6SM01`）— 后果：未开 IPv6 承载时 PDU 会话无法建立。
- **IPv4v6 双栈需双特性**（critical）：IPv4v6 双栈用户 DHCPv6 分配须叠加 [WSFD-104002](feature/UNC/20.15.2/WSFD-104002.md) IPv4v6 双栈接入（License `LKV2DUSA02`/`LKV2IPDSSM01`）— 后果：未开双栈时用户 IPv6 部分无法激活。
- **DHCPv6 服务器可达性**（critical）：UNC 通过 `ADD AGENTIP` 配置的代理 IPv6 与外部 DHCPv6 Server 通信，需保证 `ALM-100466` 不告警 — 后果：用户激活失败。
- **DHCPv6 请求信元须 Server 支持**（warning）：步骤 6（`SET DHCPPARAREQ` 开启 DNS/P-CSCF 选项）需 DHCPv6 Server 支持对应 Option — 后果：Server 不支持时分配不出 DNS/P-CSCF。
- **P-CSCF 地址上限 2 个**（warning）：UNC 分配给用户的 P-CSCF 地址最大支持 2 个（含本地 + DHCPv6）— 后果：超出时 DHCPv6 返回的 P-CSCF 不生效。
- **仅地址不支持域名**（warning）：仅支持在 DHCPv6 过程中获取 P-CSCF **服务器地址**，不支持获取 P-CSCF **服务器域名** — 后果：域名场景需改用其他方式。
- **POOLNAME 命名约束**（warning）：字符串 1~79，由 `-/_`/数字/字母/`.` 组成，不能以 `.` 开头 — 后果：违规命名新增失败。
- **地址段必配**（critical）：本地地址池（POOLTYPE=DHCPv6）建好后**必须** `ADD SECTION` 添加 IPv6 地址段，否则 DHCPv6 Solicit 失败。
- **AGENTIP 必须在 IPv6 地址段内**（critical）：`ADD AGENTIP` 代理 IPv6 须位于 DHCPv6 地址池网段内 — 后果：代理 IP 不在网段内致 DHCPv6 Server 找不到对应地址池。
- **首次发布版本**（info）：本特性首次发布版本为 UNC **20.8.0**；GGSN/SGW-C/PGW-C/SMF 均支持。
