# 配置IMSI计费策略（ADD CHGIMSICFG）

> 命令静态知识见 [ADD CHGIMSICFG](command/UNC/20.15.2/ADD-CHGIMSICFG.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD CHGIMSICFG 是 UNC 三种 S-CDR 话单生成策略之一（另两种为 ADD CHGPLMNCFG 基于 PLMN+APN NI、SET CHGCHAR 全局基于计费属性）。本命令用"IMSI 前缀 + APN NI + 拜访类型 + 计费属性"四元组定位一组用户，定制其 S-CDR 生成行为（是否生成、周期、流量阈值、条件变更/位置更新触发）。典型用途是抑制特定号段漫游用户对特定 APN NI 的话单。

激活前提：SET CHGCDR 的 PLMNCTRL=IMSI（取 IMSI 中的 PLMN ID）时本策略才生效；若 PLMNCTRL=LOCINFO 则走 ADD CHGPLMNCFG。

### 配置维度 1：用户定位（SUBRANGE + IMSIPRE + VISITTYPE + APNNI + CC）

四元组定位被定制用户群，五个参数共同锁定一条策略记录。

| 参数 | 取值样例 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|
| SUBRANGE | `IMSI_PREFIX` / `ALL_USER` | — | 用户范围（必填） | 按前缀定位或全量用户 |
| IMSIPRE | `12303` | — | IMSI 前缀，5~15 位十进制 | SUBRANGE=IMSI_PREFIX 时必填 |
| VISITTYPE | `ROAMING-1` | ROAMING&VISITING | 拜访类型（使用归属地/拜访地 GGSN 的漫游用户） | 仅对漫游用户生效 |
| APNNI | `huawei.com` | — | APN 网络标识，1~62 字符（必填） | 按 APN 细化话单策略 |
| CC | `NORMAL_BILLING-1` | HOT_BILLING&FLAT_RATE&PREPAID_SERVICE&NORMAL_BILLING | 计费属性 | 按计费属性细化 |

> 注意：VISITTYPE 已设 ROAMING&VISITING 时，同一 APN NI + IMSI PREFIX 不允许再添加其他拜访类型记录；CC 对同一 APN NI+IMSI PRE 不允许交叉选择（如同时 HOT+FLAT 和 HOT+PREPAID）。

### 配置维度 2：话单生成开关与触发阈值

五个开关 + 三个阈值控制 S-CDR 的生成时机。默认 DFT（走缺省策略 = 沿用上层 SET CHGCHAR 配置），抑制场景需显式置 NO/FORBID。

| 参数 | 取值样例 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|
| SP | `NO` | DFT | 是否生成 S-CDR（YES/NO/DFT） | NO=抑制该类话单 |
| SPP | `FORBID` | DFT | 周期生成 S-CDR（PERMIT/FORBID/VOLUMN/DFT） | FORBID=不周期生成 |
| SPL | `30` | 0 | S-CDR 生成周期(min)，0~1440 | 周期触发，建议≥15min |
| SVP | `NO` | DFT | 流量生成 S-CDR（YES/NO/DFT） | NO=不按流量触发 |
| SVL | `1000` | 0 | 流量阈值(KB)，0~1000000 | 流量达到阈值触发 |
| SCCP | `NO` | DFT | 计费条件变更生成 S-CDR（YES/NO/DFT） | NO=QoS/费率变更不触发 |
| SCCL | `3` | 0 | 最大计费条件变更次数，0~10 | 变更次数达上限触发 |
| SLCP | `NO` | DFT | 位置更新生成 S-CDR（YES/NO/DFT） | NO=位置更新不生成（流量/时间达阈值仍生成） |

> 默认值均为"无效值"（DFT/0）：未显式填的字段走缺省策略而非覆盖。抑制话单范式需把相关开关显式置 NO/FORBID。

### 典型脚本（抑制 12303 前缀漫游用户对 huawei.com 的话单）

```
SET CHGPLMNCHAR: PLMN=VPLMN, SP=YES;    # 前置：允许 VPLMN 生成话单
SET CHGCDR: PLMNCTRL=IMSI;              # 前置：切到 IMSI 来源，本策略才生效
ADD CHGIMSICFG: SUBRANGE=IMSI_PREFIX, IMSIPRE="12303", VISITTYPE=ROAMING-1,
  APNNI="huawei.com", CC=NORMAL_BILLING-1, SP=NO;
```

**步骤位置**：在 SGSN 离线计费配置流程中，位于 SET CHGCDR（PLMNCTRL=IMSI）之后，作为"基于 IMSI 细化话单策略"步骤。

## 决策点

### 决策点：话单生成策略选哪一层（DP 0-02019）

UNC 三种 S-CDR 策略互斥于不同定位维度，由 SET CHGCDR 的 PLMNCTRL 决定走哪条：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IMSI 细化（ADD CHGIMSICFG） | PLMNCTRL=IMSI + IMSIPRE + APNNI + VISITTYPE + CC | 需按 IMSI 前缀精细化定制特定号段用户话单 |
| PLMN 细化（ADD CHGPLMNCFG） | PLMNCTRL=LOCINFO + PLMN(MCC+MNC) + APNNI | 按 PLMN 粒度定制漫游用户话单（不细分到号段） |
| 全局（SET CHGCHAR） | 仅 CC（计费属性） | 全局按计费属性设默认话单策略，作为上层缺省 |

### 决策点：各触发开关填 DFT 还是显式 NO/YES（DP 0-02020）

SP/SPP/SVP/SCCP/SLCP 五个开关默认 DFT（走缺省策略），显式填 NO/FORBID 表示抑制该类话单。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 走缺省策略 | 开关=DFT | 不在本层覆盖，沿用上层 SET CHGCHAR 配置 |
| 抑制生成 | SP=NO / SPP=FORBID / SVP=NO / SCCP=NO / SLCP=NO | 抑制特定号段用户对该 APN 的话单 |
| 显式触发 | SP=YES / SPP=PERMIT / SVP=YES / SCCP=YES / SLCP=YES + 对应阈值（SPL/SVL/SCCL） | 强制在该维度生成话单并设触发阈值 |

## 约束

- **生效前置**（critical，rule-0-02046）：生效前置：仅当 SET CHGCDR 的 PLMNCTRL=IMSI 时本策略生效；PLMNCTRL=LOCINFO 时走 ADD CHGPLMNCFG，配错前置会导致策略不生效
- **上层闸门**（warning，rule-0-02047）：上层闸门：若 SET CHGPLMNCHAR 禁止生成某种话单，则无论本命令如何配置都不生成，本命令无法突破上层禁止
- **拜访类型前提**（warning，rule-0-02048）：拜访类型前提：SUBRANGE=IMSI_PREFIX 且 VISITTYPE 单配 ROAMING 或 VISITING 时，IMSI 前缀指定用户必须是外网签约用户（通过 ADD CONNECTPLMN（★R1.5 atom 待补） 配置），否则此条配置无效
- **立即生效但只对配置生效后激活的用户有效**（warning，rule-0-02049）：立即生效但只对配置生效后激活的用户有效：已激活用户不立即受影响，变更不追溯已激活会话
- **本表最大 2048 条记录**（warning，rule-0-02050）：本表最大 2048 条记录：超过上限新增失败，规划前需核查现有记录数
