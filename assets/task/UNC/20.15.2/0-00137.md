# RMV CHGIMSICFG 原子任务
> 命令静态知识见 [RMV CHGIMSICFG](command/UNC/20.15.2/RMV-CHGIMSICFG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

删除对象是 `CHGIMSICFG`（IMSI 前缀计费配置）的记录。匹配键为 `SUBRANGE + IMSIPRE + APNNI`，可选限定 `VISITTYPE/CC`；当不指定可选限定字段且必选键同时匹配多条时，会**批量删全部匹配记录**。

## 配置方法
按"用户范围 + IMSI 前缀 + APNNI"主键删除，可选用 `VISITTYPE/CC` 收窄。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| SUBRANGE | 用户范围（必选） | `ALL_USER`（所有用户） |
| IMSIPRE | IMSI 前缀（conditional） | `46000`（5～15 位十进制） |
| APNNI | APNNI（必选） | `cmnet`（最长 62 字符） |
| VISITTYPE | 拜访类型（可选，收窄） | `ROAMING`（使用归属地 GGSN 的漫游用户） |
| CC | 计费属性（可选，收窄） | `HOT_BILLING`（实时计费） |

典型脚本（精确删一条：全用户/46000/cmnet/漫游/实时计费）：
```
RMV CHGIMSICFG: SUBRANGE=ALL_USER, IMSIPRE=46000, APNNI=cmnet, VISITTYPE=ROAMING, CC=HOT_BILLING;
```

## 决策点（若有）

### 删除范围（精确 vs 批量）
- 问什么：删一条还是删一批。
- 精确删一条：给全 `SUBRANGE/IMSIPRE/APNNI + VISITTYPE + CC`。
- 批量删：仅给 `SUBRANGE/IMSIPRE/APNNI`（不带 VISITTYPE/CC），匹配到的多条记录会被**全部删除**——属批量高危，删前必须 LST CHGIMSICFG 核对命中条数。

### 可选限定字段是否提供
- `IMSIPRE` 标 conditional：与 `APNNI` 等组合时承担匹配职责；不收窄则匹配更宽。
- `VISITTYPE/CC` 是可选收窄维度，省略=批量命中。

## 约束（若有）
- 批量删风险：不指定 `VISITTYPE/CC` 时，`SUBRANGE+IMSIPRE+APNNI` 同时匹配多条会全部删除，删前必须 LST 核对。`severity=high`
- 唯一键：完整五元组 `SUBRANGE+IMSIPRE+APNNI+VISITTYPE+CC` 定位单条。`severity=medium`
- 前置：被删记录应存在，否则报错。`severity=low`
- 生效时机：立即生效，仅影响之后新激活用户的话单生成策略（不影响在线用户）。`severity=medium`
