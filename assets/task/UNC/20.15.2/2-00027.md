# 终端二次鉴权（WSFD-108007）

> 特性静态知识见 [WSFD-108007](feature/UNC/20.15.2/WSFD-108007.md)。本页讲配置生成实例化时这个特性**怎么配**——UPF 中转 Radius 链路 4 步 + 必加 N4 GTP-U 地址前置。

## 配置概览

核心对象链：**N4 GTP-U 地址**（CPGTPUADDR，SMF 本端 IP）→ **UP 列表**（UPLIST4RDS）→ **UPF 属性**（RDSUPFCTRL，UPF 偏好+锁定）→ **UPF 绑 Radius 服务器**（UPFRDSSVR，DN-AAA IP）→ **UPF 绑 Radius 客户端 IP**（UPFRDSCLIENTIP，最后一步）。

本特性是 **WSFD-011305 + WSFD-011306 的 UPF 中转扩展**——同一套 RADIUS 鉴权协议但**走 UPF 中转**（SMF ↔ UPF 走 N4 GTP-U 隧道，UPF ↔ DN-AAA 走 Radius）。需先开 [WSFD-011305](feature/UNC/20.15.2/WSFD-011305.md)（ACCESSMODE 选合适接入方式）+ [WSFD-011306](feature/UNC/20.15.2/WSFD-011306.md)（标准 RADIUS 主链路 + 3GPP VSA）。

## 配置流程（步骤编排，每步显示含命令）

1. **前置依赖**（必先开）：[WSFD-011305](feature/UNC/20.15.2/WSFD-011305.md) Radius 鉴权接入 + [WSFD-011306](feature/UNC/20.15.2/WSFD-011306.md) Radius 功能（建 RDSSVRGRP+RDSSVR 等主链路）
2. **接入域基础设施**（建 APN 实例 + VPN）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR` → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
3. **N4 GTP-U 地址**（SMF 本端 IPv4，UPF 通过此地址识别 N4-U 会话）：`ADD CPGTPUADDR: IPVERSION=IPV4, IPV4ADDR="192.168.0.20"` → [0-00226](task/UNC/20.15.2/0-00226.md)
4. **UPF 中转 Radius 链**（CPGTPUADDR + UPLIST4RDS + RDSUPFCTRL + UPFRDSSVR + UPFRDSCLIENTIP）：引用 [UNC 1-00025](task/UNC/20.15.2/1-00025.md) 步骤 6-9（UPF 中转链 4 条）+ N4 GTP-U 前置
   - 步骤 6（UPF 中转链·步骤 1）：建 UP 列表 → `ADD UPLIST4RDS: UPLISTNAME="uplist1", UPINSTANCEID="up1"` → [0-00260](task/UNC/20.15.2/0-00260.md)
   - 步骤 7（UPF 中转链·步骤 2，可选）：UPF 偏好 + 锁定 → `ADD RDSUPFCTRL: UPLISTNAME="uplist1", UPINSTANCEID="up1", PREFERENCE=1, LOCKED=ENABLE` → [0-00256](task/UNC/20.15.2/0-00256.md)
   - 步骤 8（UPF 中转链·步骤 3）：UPF 绑 Radius 服务器 → `ADD UPFRDSSVR: SERVERTYPE=AUTHENTICATION, IPVERSION=IPV4, SERVERIPV4="192.168.10.2", UPLISTNAME="uplist1"` → [0-00262](task/UNC/20.15.2/0-00262.md)
   - 步骤 9（UPF 中转链·步骤 4·最后一步）：UPF↔DN-AAA VPN + Client IP → `ADD UPFRDSCLIENTIP: CLIENTYPE=AUTHENTICATION, IPVERSION=IPV4, UPLISTNAME="uplist1", VPNINSTANCE="vpnInstance1", CLIENTIPV4="192.168.10.1"` → [0-00261](task/UNC/20.15.2/0-00261.md)

> **★ 关键顺序约束**：
> - 步骤 6（`ADD UPFRDSSVR`）→ 步骤 7（`ADD UPFRDSCLIENTIP`）：必须**先配 SVR 后配 CLIENT**，否则 AAA 计费业务无法正常进行。
> - 步骤 7（`ADD UPFRDSCLIENTIP`）必须**最后执行**：该命令执行后 SMF 立即触发 N4-U 会话建立；中途修改需先 `RMV UPFRDSCLIENTIP` 再重 `ADD`，否则 SMF 尝试建链失败（延迟 15 分钟重试）。
> - 步骤 4-7 全部完成后，**先在 UPF 上执行 `ADD NETWORKINSTVPNMAP`** 才能让 SMF N4-U 会话建立成功。

## 决策点（特性级：场景影响表）

| 场景 × UPF 数量 × DN-AAA 类型 × 故障场景 | 走法（核心 5 步）| 联动 / 关键参数 |
|---|---|---|
| **标准单 UPF × 鉴权 DN-AAA** | 步骤 1-7 全配（步骤 5 可选）| PREFERENCE=1 优先 |
| **多 UPF 负载分担** | 步骤 1-7 全配 + 步骤 4 重复 `ADD UPLIST4RDS` 加多 UP | 步骤 5 PREFERENCE 区分主备 |
| **多 UPF 主备** | 步骤 1-7 全配 + 步骤 5 `PREFERENCE` 1/2 + `LOCKED=ENABLE` 锁定主 | 主 UPF 故障时切备 UPF |
| **鉴权+计费双 DN-AAA** | 步骤 6 重复 `ADD UPFRDSSVR` 加 ACCOUNTING 类型 + 步骤 7 重复 `ADD UPFRDSCLIENTIP` 加 ACCOUNTING 类型 | CLIENTYPE 区分鉴权/计费 |
| **DN-AAA 故障重选** | 步骤 6 配多 DN-AAA IP | SMF 故障自动重选（无须配置）|
| **边缘+中心 UPF 主备容灾** | 步骤 4 配 UPLIST 包含边缘+中心 UPF，步骤 5 PREFERENCE 区分 | 边缘 UPF 故障切中心 UPF |

> **走法列**：所有场景**共享步骤 1-3 前置**；差异在步骤 4-7 UPF 中转链的**重复/多 UPF/多 DN-AAA**配置。

## 约束（特性级 rule）

- **本特性需 License**（critical）：License 控制项 82200DSF `LKV2SECAA01` 终端二次鉴权 — 后果：无 License 时特性不可用。
- **必先开 WSFD-011305 + WSFD-011306**（critical）：本特性使用前要激活 Radius 鉴权接入 + Radius 功能 — 后果：未激活前置特性时 N4-U 会话建立失败。
- **UPFRDSSVR → UPFRDSCLIENTIP 顺序**（critical）：必须先 `ADD UPFRDSSVR` 再 `ADD UPFRDSCLIENTIP` — 后果：顺序错误致 AAA 计费业务无法正常进行。
- **UPFRDSCLIENTIP 最后执行**（critical）：该命令触发 SMF 立即建 N4-U 会话 — 后果：中途修改未删后重配致 SMF 失败（延迟 15 分钟重试）。
- **UPF 上需 `ADD NETWORKINSTVPNMAP`**（critical）：UPF 侧需先配该命令 — 后果：未配致 SMF 尝试建 N4-U 会话失败。
- **仅支持 PAP/CHAP 鉴权**（warning）：启用本特性后 SMF 不支持 EAP 鉴权方式，只支持 PAP、CHAP 鉴权。
- **仅支持 Radius 协议**（warning）：SMF↔DN-AAA 二次鉴权消息只支持 Radius 协议，不支持 Diameter 协议。
- **直连 AAA + UPF 中转 AAA 场景 IP 不可相同**（critical）：一个 SMF 网元上所有业务中，直连 AAA 和经 UPF 中转 AAA 场景中的 Radius Server IP **不可相同** — 后果：IP 相同致直连 AAA 场景业务受损。
- **每 RADIUS Server 最多配置 64 UPF Group**（warning）：本特性规格。
- **N4-U 会话不小于 6000 个**（info）：UPF 同时对接多个企业/园区时 UNC 同时支持不少于 6000 个 UPF 转接会话。
- **重选 UPF 转发**（info）：N4-U 会话建立后如 UPF 故障，SMF 会选择另外的 UPF 发送二次鉴权消息。
- **重选 DN-AAA**（info）：DN-AAA Server 故障时 SMF 发送二次鉴权消息无响应 → SMF 选另外 DN-AAA。
- **UPF 到 DN-AAA 故障 SMF 不感知**（warning）：本特性对系统的影响 — 后果：故障时 SMF 无法感知 DN-AAA 状态。
- **首次发布版本**（info）：**UNC 22.0**（首次发布）。
- **本特性适用 SMF**（info）：UPF/SMF 协同（UNC 20.8.0+ N4 链路基础）。
