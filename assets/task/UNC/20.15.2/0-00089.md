# ADD UPBINDUPG 原子任务
> 命令静态知识见 [ADD UPBINDUPG](command/UNC/20.15.2/ADD-UPBINDUPG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是模板体系的枢纽：`ADD USRPROFGROUP`（建组）→ **本命令**（把 UserProfile 装进组并设匹配条件）→ `ADD APNUSRPROFG`（组绑到 APN/DNN）。UPBINDTYPE 决定是"默认模板"还是"按条件精匹配"。

## 配置方法

### 默认模板（UPBINDTYPE=DEFAULT）
最常见：给组指定一个默认 UserProfile，不设任何匹配条件。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| USERPROFGNAME | 已配置数据中获取（来自 `ADD USRPROFGROUP`） | `upg_test` |
| UPBINDTYPE | 固定取值 | `DEFAULT` |
| USERPROFILENAME | 已配置数据中获取（来自 `ADD USERPROFILE`） | `up_test` |

```
ADD UPBINDUPG: USERPROFGNAME="upg_test", UPBINDTYPE=DEFAULT, USERPROFILENAME="up_test";
```

### 精匹配多模板（UPBINDTYPE=SPECIFIC）
为同一组建多条带优先级 + 匹配条件的绑定，按用户属性选不同 UserProfile（如不同费率/套餐/IMS 专用）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| USERPROFGNAME | 已配置数据中获取 | `upg_test` / `upg_ims` |
| UPBINDTYPE | 固定取值 | `SPECIFIC` |
| PRIORITY | 本端规划，取值越小优先级越高（1~65534） | `10`、`20` |
| USERPROFILENAME | 已配置数据中获取 | `userprofile001` |
| RAT/ROAMING/CHARGECHAR/IMSIMSSEGNAME/LOCGROUPNAME | 按业务匹配维度按需填写 | — |

```
ADD UPBINDUPG: USERPROFGNAME="upg_test", UPBINDTYPE=SPECIFIC, PRIORITY=10, USERPROFILENAME="userprofile001";
ADD UPBINDUPG: USERPROFGNAME="upg_test", UPBINDTYPE=SPECIFIC, PRIORITY=20, USERPROFILENAME="userprofile002";
```

### ULCL 分流专用（UPBINDTYPE=ULCL）
分流策略控制场景，把带 ULCL 属性的 UserProfile 绑到组。

```
ADD UPBINDUPG: USERPROFGNAME="ulcl_test", UPBINDTYPE=ULCL, USERPROFILENAME="testuserprofilename";
```

## 决策点

### UPBINDTYPE 绑定类型
- 问什么：该 UserProfile 在组内是默认兜底还是按条件精匹配。
- DEFAULT → 组的默认模板，不设匹配条件（最多 1 个）。
- SPECIFIC → 带优先级 + 匹配条件，多模板差异化（费率/套餐/IMS）。
- ULCL → 分流场景专用属性。
- 选错会导致用户匹配不到正确模板或优先级混乱。

### PRIORITY 优先级（SPECIFIC 时）
- 问什么：多个 SPECIFIC 模板的匹配顺序。
- 取值越小优先级越高；同一组内优先级必须唯一。65535 保留给 DEFAULT。

### 匹配条件维度（SPECIFIC 时）
- 问什么：用哪些用户属性区分模板。可选 IMSI/MSISDN 号段、IMEISV 号段、RAT、漫游属性、计费属性（CHARGECHAR+CCMASK）、位置组、号段组。
- 计费属性用法：CCCONFIGMODE 定模式，CHARGECHAR 为值，CCMASK（默认 0xFFFF）定掩码位匹配。
- 本地 PCC 多模板：用户激活后按 `ADD UPBINDUPG` 匹配条件最终只装一个 UserProfile；若需装多个，改用 `ADD PCCPBINDUPG`。

## 约束

- **立即生效** · severity=LOW。
- **前置依赖（硬性）**：必须先执行 `ADD USRPROFGROUP` 和 `ADD USERPROFILE`，否则引用对象不存在 · severity=HIGH。
- **组内上限**：一个 UserProfGroup 最多绑 500 个 UserProfile，优先级唯一；另可绑 1 个默认模板（不带任何匹配条件） · severity=MEDIUM。
- **PRIORITY 唯一性**：SPECIFIC 模板优先级组内唯一，65535 保留给 DEFAULT · severity=HIGH。
- **名称字符规则**：USERPROFGNAME/USERPROFILENAME 不支持空格，不区分大小写 · severity=LOW。
- **LOCALPCCSELECT**：本地 PCC 策略选择模式，默认 INHERIT · severity=LOW。
