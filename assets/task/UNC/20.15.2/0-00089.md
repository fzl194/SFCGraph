## 配置方法

本命令是模板体系的枢纽：`ADD USRPROFGROUP`（建组）→ **本命令**（把 UserProfile 装进组并设匹配条件）→ `ADD APNUSRPROFG`（组绑到 APN/DNN）。

### 默认模板（UPBINDTYPE=DEFAULT）

最常见：给组指定一个默认 UserProfile，不设任何匹配条件。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 组名称 | USERPROFGNAME | 1~63 字符，不支持空格，不区分大小写 | 无（必选） | 指定已建的 UserProfGroup | 通用 |
| 绑定类型 | UPBINDTYPE | `DEFAULT` | 无（必选） | 指定默认模板 | 默认模板 |
| 模板名称 | USERPROFILENAME | 1~63 字符，不支持空格，不区分大小写 | 无（必选） | 指定已建的 UserProfile | 通用 |

**典型脚本：**
```
ADD UPBINDUPG: USERPROFGNAME="upg_test", UPBINDTYPE=DEFAULT, USERPROFILENAME="up_test";
```

### 精匹配多模板（UPBINDTYPE=SPECIFIC）

为同一组建多条带优先级 + 匹配条件的绑定，按用户属性选不同 UserProfile（如不同费率/套餐/IMS 专用）。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 组名称 | USERPROFGNAME | 1~63 字符，不支持空格，不区分大小写 | 无（必选） | 指定目标组 | 精匹配 |
| 绑定类型 | UPBINDTYPE | `SPECIFIC` | 无（必选） | 指定精匹配模板 | 精匹配 |
| 优先级 | PRIORITY | 1~65534（越小越高；65535 保留给 DEFAULT） | 无（必选） | 多模板匹配顺序 | 精匹配 |
| 模板名称 | USERPROFILENAME | 1~63 字符，不支持空格，不区分大小写 | 无（必选） | 指定目标 UserProfile | 精匹配 |
| 接入类型 | RAT | NONE/UTRAN/GERAN/WLAN/GAN/HSPA/EUTRAN/NR/NBIOT/EMTC/REDCAP | 无（可选） | 按无线接入技术区分 | 精匹配 |
| 漫游属性 | ROAMING | NONE/HOME/ROAMING/VISITING | 无（可选） | 按漫游属性区分 | 精匹配 |
| 计费配置模式 | CCCONFIGMODE | NORMAL/HOTBILLING/PREPAID/FLATBILLING/SPECIFIC | 无（可选） | 按计费模式区分 | 精匹配 |
| 计费属性 | CHARGECHAR | 0x0~0xFFFF（条件必选，当 CCCONFIGMODE=SPECIFIC 时） | 无 | 计费属性值 | 精匹配 |
| 计费掩码 | CCMASK | 0x1~0xFFFF | 0xFFFF | 计费属性掩码 | 精匹配 |
| IMSI/MSISDN 号段 | IMSIMSSEGNAME | 1~31 字符，不支持空格，不区分大小写 | 无（可选） | 按号段区分 | 精匹配 |
| IMEISV 号段 | IMEISVSEGNAME | 1~31 字符，不支持空格，不区分大小写 | 无（可选） | 按终端类型区分 | 精匹配 |
| 位置组 | LOCGROUPNAME | 1~63 字符，不支持空格，不区分大小写 | 无（可选） | 按位置区域区分 | 精匹配 |
| 号段组 | SEGGROUPNAME | 1~31 字符，不支持空格，不区分大小写 | 无（可选） | 按号段组区分 | 精匹配 |
| 本地 PCC | LOCALPCCSELECT | INHERIT/LOCAL_PCC_DEACTIVE/LOCAL_PCC_ACTIVE/LOCAL_PCC_NO_CRBN_ACT/LOCAL_PCC_NO_RULE_ACT | INHERIT | 本地 PCC 策略选择模式 | 精匹配 |

> UPBINDTYPE=SPECIFIC 时，IMSI/MSISDN 号段名称、RAT、漫游属性、计费属性、位置组等**至少要设置其中一个**。

**典型脚本：**
```
ADD UPBINDUPG: USERPROFGNAME="upg_test", UPBINDTYPE=SPECIFIC, PRIORITY=10, USERPROFILENAME="userprofile001";
ADD UPBINDUPG: USERPROFGNAME="upg_test", UPBINDTYPE=SPECIFIC, PRIORITY=20, USERPROFILENAME="userprofile002";
```

### ULCL 分流专用（UPBINDTYPE=ULCL）

分流策略控制场景，把带 ULCL 属性的 UserProfile 绑到组。

**典型脚本：**
```
ADD UPBINDUPG: USERPROFGNAME="ulcl_test", UPBINDTYPE=ULCL, USERPROFILENAME="testuserprofilename";
```

**步骤位置：** 模板体系建立阶段，先执行 `ADD USRPROFGROUP`（建组）和 `ADD USERPROFILE`（建模板），再执行本命令绑定，最后执行 `ADD APNUSRPROFG` 将组绑到 APN/DNN。

## 决策点

### 决策点：UPBINDTYPE 绑定类型（DP 0-01216）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| `DEFAULT` | 不设任何匹配条件；一个组最多 1 个；优先级固定 65535 | 默认兜底模板 |
| `SPECIFIC` | 需设 PRIORITY；可选设 RAT/ROAMING/计费属性/号段/位置组等匹配条件 | 按用户属性精匹配多模板 |
| `ULCL` | 与 ULCL 分流策略配合 | 分流场景专用 |

> 问什么：该 UserProfile 在组内是默认兜底还是按条件精匹配。
> DEFAULT → 组的默认模板，不设匹配条件（最多 1 个）。
> SPECIFIC → 带优先级 + 匹配条件，多模板差异化（费率/套餐/IMS）。
> ULCL → 分流场景专用属性。
> 选错会导致用户匹配不到正确模板或优先级混乱。

### 决策点：PRIORITY 优先级分配（DP 0-01217）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 唯一优先级 | 同组内 PRIORITY 值唯一，值越小优先级越高 | 多 SPECIFIC 模板场景 |
| DEFAULT 保留 | 65535 保留给 DEFAULT 类型绑定 | DEFAULT 与 SPECIFIC 共存 |

> 问什么：多个 SPECIFIC 模板的匹配顺序。
> 取值越小优先级越高；同一组内优先级必须唯一。65535 保留给 DEFAULT。

## 约束

- rule-0-01476 · severity=low · 立即生效：命令执行后配置立即生效，无需重启
- rule-0-01477 · severity=high · 前置依赖：必须先执行 ADD USRPROFGROUP 和 ADD USERPROFILE，否则引用对象不存在
- rule-0-01478 · severity=medium · 组内上限：一个 UserProfGroup 最多绑 500 个 UserProfile（优先级唯一），另可绑 1 个默认模板（无匹配条件）；UNC 整机最大绑定规格 25000
- rule-0-01479 · severity=high · PRIORITY 唯一性：SPECIFIC 模板优先级组内唯一，65535 保留给 DEFAULT
- rule-0-01480 · severity=low · 名称字符规则：USERPROFGNAME/USERPROFILENAME 不支持空格，不区分大小写
