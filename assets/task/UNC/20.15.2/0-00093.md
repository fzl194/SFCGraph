# ADD URRGROUP 原子任务
> 命令静态知识见 [ADD URRGROUP](command/UNC/20.15.2/ADD-URRGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
URR 组 = "上下行方向 × URR 名称槽位"的绑定。`UPURRNAME1/2/3` 是上行三槽、`DOWNURRNAME1/2/3` 是下行三槽，编号 1/2/3 **只是槽位无优先级语义**。引用的 URR 名称必须先用 [ADD URR](task/UNC/20.15.2/0-00001.md) 建好。`URRGROUPNAME` 本端规划、跨记录不重复。支持上下行同 URR、上下行不同 URR、在线离线混合（仅融合计费场景）三种范式。

| 计费结构 | 必备参数 | 典型脚本 |
|---|---|---|
| 上下行同一 URR（单轨道：纯离线 / 纯在线 / 事件） | `URRGROUPNAME` + `UPURRNAME1` + `DOWNURRNAME1`（同值） | `ADD URRGROUP: URRGROUPNAME="urrgroup1", UPURRNAME1="offlineURR", DOWNURRNAME1="offlineURR";` |
| 在线+离线混合（融合计费，每方向两 URR） | `URRGROUPNAME` + `UPURRNAME1/DOWNURRNAME1`(在线) + `UPURRNAME2/DOWNURRNAME2`(离线) | `ADD URRGROUP: URRGROUPNAME="UrrGp_URL", UPURRNAME1="urr_online_URL", DOWNURRNAME1="urr_online_URL", UPURRNAME2="urr_offline_URL", DOWNURRNAME2="urr_offline_URL";` |
| 不配置下行（缺省沿用上行） | `URRGROUPNAME` + `UPURRNAME1`（不输 DOWNURRNAME1） | 缺省采用上行 URR |
| 不计费标记（可选） | 上述 + `NOCHARGINGFLAG` | 枚举，按业务套餐决定 |

**UAR/URRGROUP 命名语义后缀**便于多业务对齐：`UrrGp_URL` / `UrrGp_IMS` / `UrrGp_any` / `UrrGp_abnormal` 等按业务类型分组建。

## 决策点

### 上下行 URR 是否相同
问：上下行报文用同一个 URR 还是不同 URR？
- 相同：上下行计费规则一致（证据中离线/在线/事件计费普遍采用）。
- 不同：上下行用不同 URR，但**两者必须同为在线或同为离线**（非融合计费时）。

### URR 名称槽位编排
问：用几槽、在线离线怎么分槽？
- 单轨道（纯离线 / 纯在线 / QoS / 事件）：仅用槽位 1，上下行同 URR。
- 融合计费（RGAPPLIED=ONLINERGONLY/OFFLINERGONLY）：槽位 1=在线、槽位 2=离线，**建议同时配置在线离线 URR**。
- 默认 RGAPPLIED=DEFAULT：URRGROUP 下**不能**同时配离线和在线 URR。

### 融合计费 RGAPPLIED 联动（跨场景枚举）
问：URRGROUP 内是否可混配在线+离线 URR？
- 取决于上游 [ADD CHARGEMETHOD](task/UNC/20.15.2/0-00015.md) / SET USRPROFCHARGE / SET APNCHARGECTRL 的 `RGAPPLIED`：
  - `DEFAULT` → URRGROUP 内不能混在线离线。
  - `ONLINERGONLY` / `OFFLINERGONLY` → URRGROUP 内建议混配（在线+离线各一）。

## 约束
- `URRGROUPNAME` 全网唯一、跨记录不重复。`severity=critical` 后果：组名冲突导致计费规则错绑。
- `UPURRNAME1/2/3` 之间（上行槽位）不能重名；`DOWNURRNAME1/2/3` 之间（下行槽位）不能重名。`severity=critical`
- 引用的 URRNAME 必须已由 [ADD URR](task/UNC/20.15.2/0-00001.md) 预先配置；不输入下行 URR 时实际采用上行 URR。`severity=high` 后果：引用失败、组无效。
- 上下行使用不同 URR 时（非融合），两者必须同为在线或同为离线计费。`severity=critical`
- 每个 URRGROUP 支持在线和离线混合计费（融合场景）。`severity=low`
- 最大记录数 40000。`severity=medium`
- 槽位 1/2/3 仅是编号无优先级语义。`severity=low`
- 执行后立即生效。`severity=low`
