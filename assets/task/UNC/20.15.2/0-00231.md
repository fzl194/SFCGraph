# 增加IPSEC接口IPv6地址（ADD IFIPV6ADDRESSIPSEC）

> 命令静态知识见 [ADD IFIPV6ADDRESSIPSEC](command/UNC/20.15.2/ADD-IFIPV6ADDRESSIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令为 IPSec 域接口（Ethernet/Eth-Trunk/Loopback/Tunnel 等）手工配置 IPv6 地址。配置空间主要由**地址类型**这一维度决定。

### 配置维度1：地址类型（参数 ADDRTYPE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| Global | 全局地址（默认主用） | IPv6 隧道接口的全局地址 |
| LinkLocal | 链路本地地址 | 链路本地通信（fe80::） |
| Anycast | 任播地址 | 任播场景 |

### 配置维度2：地址与前缀（参数 IFIP6ADDR / ADDRPREFIXLEN）

| 参数 | 作用 |
|---|---|
| IFIP6ADDR | IPv6 地址 |
| ADDRPREFIXLEN | 前缀长度（1~128） |

**步骤位置**：在所属 feature（IPFD-016000 IPSec功能）编排中，于 `ADD INTERFACEIPSEC`（建接口）+ `SET IFIPV6ENABLEIPSEC`（使能 IPv6）之后执行。

## 决策点

（无。本命令用法由 IPv6 规划决定，无特性级分支。）

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无延迟
- **适用接口范围**（info）：可在 VNRS_VNFC 的 Ethernet/Ethernet 子接口/Eth-Trunk/Eth-Trunk 子接口/Loopback/Tunnel 口上配置 — 后果：范围外接口配置失败
- **IFNAME 必须已存在**（critical）：IFNAME 必须可通过 `LST INTERFACEIPSEC` 查询到 — 后果：索引失败
- **最多 65535 条记录**（warning）：本命令最多可输入 65535 条记录 — 后果：超规格新增失败
