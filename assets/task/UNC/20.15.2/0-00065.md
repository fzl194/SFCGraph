# ADD QOSPROP 原子任务
> 命令静态知识见 [ADD QOSPROP](command/UNC/20.15.2/ADD-QOSPROP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
QoS 属性按"接入代际 × 业务模型"规划：`QOSTYPE` 是总开关，决定走 5G QoS Flow（`QOS_FLOW_PARA`）还是 2/3/4G 承载（`QOS_BEARER_PARA`），二者必备 QoS 标识不同（5G 用 FQI+RQI，2/3/4G 用 QCIVALUE）。`QOSPROPNAME` 全局唯一，通过 ADD PCCPOLICYGRP 的 `QOSPROPNAME` 或 ADD RULE 的 `POLICYTYPE=QOS` 两种路径绑定到规则。

| QoS 属性类型 | 必备 QoS 标识 | 速率参数 | 典型脚本 |
|---|---|---|---|
| 5G QoS Flow `QOS_FLOW_PARA` | `FQI`(5QI 1~255) + `RQI`（可选）+ `ARPVALUE` | `GBRUPLK/DNLK` + `MBRUPLK/DNLK`（GBR 业务） | `ADD QOSPROP: QOSPROPNAME="qosprop_501", GBRUPLKVALUE=600, GBRDNLKVALUE=6000, MBRUPLKVALUE=1000, MBRDNLKVALUE=10000, QOSTYPE=QOS_FLOW_PARA, ARPVALUE=5, FQI=2, QOSURRNAME="URR_qos_01";` |
| 2/3/4G 承载 `QOS_BEARER_PARA` | `QCIVALUE`(1~255) + `ARPVALUE` | `GBRUPLK/DNLK` + `MBRUPLK/DNLK`（GBR 业务） | `ADD QOSPROP: QOSPROPNAME="qosprop_401", GBRUPLKVALUE=600, GBRDNLKVALUE=6000, MBRUPLKVALUE=1000, MBRDNLKVALUE=10000, QOSTYPE=QOS_BEARER_PARA, QCIVALUE=66, ARPVALUE=5, QOSURRNAME="URR_qos_01";` |
| 最简（本地 PCC 占位） | 仅 `QOSPROPNAME` + `QOSTYPE` | 无 | `ADD QOSPROP: QOSPROPNAME="qos_property1", QOSTYPE=QOS_BEARER_PARA;` |

**速率单位**：`GBRUPLKVALUE/GBRDNLKVALUE/MBRUPLKVALUE/MBRDNLKVALUE` 均为千比特每秒(kbit/s)，取值 0~20000000。

**抢占属性**：`EMPCAP`(抢占能力，默认 DISABLE) / `EMPVUL`(被抢占设置，默认 ENABLE)。证据中语音/一键通业务常 `EMPCAP=ENABLE, EMPVUL=DISABLE`（可抢占、不被抢占）；普通业务反之或不配。

**QoS 使用量上报**：`QOSURRNAME` 引用 ADD URR 已建的 URRNAME（QoS 保证场景必备），用于 QoS 用量统计上报。

## 决策点

### QoS 属性类型（总开关）
问：本业务用户走 5G QoS Flow 还是 2/3/4G 承载？
- `QOS_FLOW_PARA`：5G 用户，配 `FQI`(+`RQI`) 作 QoS 标识（证据 WSFD-109107 5G 用户、WSFD-201207 N7 接口 5G 场景）。
- `QOS_BEARER_PARA`：2/3/4G 用户，配 `QCIVALUE` 作 QoS 标识（证据 WSFD-109101、WSFD-102602、WSFD-201207 Gx 接口 2/3/4G 场景）。
- 跨代业务需建两条 QOSPROP（5G 一条 `QOS_FLOW_PARA`+FQI，2/3/4G 一条 `QOS_BEARER_PARA`+QCIVALUE），证据 WSFD-109107 显式给出双轨示例。

### 绑定路径
问：QoS 属性如何挂到规则？
- 经 PCC 策略组：ADD RULE 的 `POLICYTYPE=PCC` + `POLICYNAME=<PCCPOLICYGRPNM>`，QoS 属性通过 ADD PCCPOLICYGRP 的 `QOSPROPNAME` 绑定。
- 直接绑定规则：ADD RULE 的 `POLICYTYPE=QOS` + `POLICYNAME=<QOSPROPNAME>`。`severity=high` 后果：PCC 与 QOS 类型的 RULENAME 不能相同。
- ADD L2RULE 也可绑定一个 QOSPROP（L7 规则 QoS 路径，文字提及）。

### QoS 用量上报是否需要
问：是否引用 URR 做 QoS 用量统计？
- 需要（业务触发 QoS 保证、语音 Bypass）：配 `QOSURRNAME=<已建 URR 的 URRNAME>`，"已配置数据中获取"。
- 不需要（本地 PCC 占位、纯 QoS 描述）：省略 `QOSURRNAME`。

### 抢占能力组合
问：本业务流能否抢占/被抢占？
- `EMPCAP=ENABLE, EMPVUL=DISABLE`：高优先级业务（语音、一键通），可抢占低优先级且自身不被抢占。
- `EMPCAP=DISABLE, EMPVUL=ENABLE`：普通业务，不抢占且可被抢占（默认值组合）。
- 中间组合按业务模型规划，证据 WSFD-109107 以不抢占(EMPCAP=DISABLE)为例、WSFD-201207 以抢占(EMPCAP=ENABLE)为例。

## 约束
- `QOSPROPNAME` 长度 1~31，不支持空格，不区分大小写，存在多条记录时不可重复。`severity=critical`
- 一个 ADD PCCPOLICYGRP 可绑定一个 QOSPROP；一个 ADD L2RULE 可绑定一个 QOSPROP。`severity=high`
- 预定义规则的 QOSPROP 与 PCC 动态规则的 QoS-information 作用相同，**仅当数据流匹配到预定义规则才执行**该 QOSPROP。`severity=medium`
- `QOSTYPE=QOS_BEARER_PARA` 时需配 `QCIVALUE`；`=QOS_FLOW_PARA` 时需配 `FQI`（+可选 `RQI`）。`severity=critical` 后果：QoS 标识缺失导致规则失效。
- `QCIVALUE` 取值 1~255（标准 QCI 见 3GPP）；`FQI` 取值 1~255（标准 5QI）；`ARPVALUE` 取值 1~15，值越小优先级越高。`severity=medium`
- `GBR/MBR` 取值 0~20000000 kbit/s；GBR 业务需配全 4 项速率，非 GBR 业务可不配。`severity=medium`
- 最大记录数 500。`severity=medium` 后果：超规格新增失败。
- 与 BWM Rule 同时绑定时存在一致性约束（notes 提及"如果同时配置了 BWM Rule 和绑定…"）。`severity=medium`
- 执行后立即生效。`severity=low`
