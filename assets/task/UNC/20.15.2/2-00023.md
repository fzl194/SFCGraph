# 静态地址用户路由冗余（WSFD-107021）

> 特性静态知识见 [WSFD-107021](feature/UNC/20.15.2/WSFD-107021.md) + [特性知识](../业务图谱体系/APN业务域/feature-knowledge/WSFD-107021-静态地址用户路由冗余.md)（**★业务图谱未建，归 R1.5 后续**）。
>
> **本页定位**：配置生成实例化视角——本特性的完整配置体系（11 步 UPF 主备链路 + ADDRUPGROUP 多 UPF + UPFBINDGRP PRIORITY）。调测命令见原始特性信息，不在配置流程展开。

## 配置概览

WSFD-107021 是 UNC 域**可靠性类特性**，针对静态地址网段内用户实现"主备 UPF 自动切换"：

**对象链**：
```
ADDRPOOL（静态地址，POOLTYPE=UDM，CHECKIPVALID=ENABLE，HASVPN=ENABLE）
  ├─ POOLBINDAPN（地址池绑 APN，白名单路径）
  ├─ POOLBINDGRP（地址池绑地址池组）
  └─ ADDRPOOLGRP（地址池组，POOLGRPTYPE=UDM）
       └─ POOLGRPMAP（地址池组绑 UPF 组）
            └─ UPFGRPNAME（ADDRUPGROUP，UPFGRPTYPE=UDM）
                 ├─ UPNODE（UPF_1，主）
                 ├─ UPNODE（UPF_2，备）
                 └─ UPFBINDGRP（UPFID + PRIORITY=0 主 / PRIORITY=1 备）
```

**核心动作**：
1. SMF 基于配置为静态地址用户分配主用 UPF（PRIORITY=0）→ 上线用户走 UPF_1
2. UPF_1 故障 → SMF 自动选 UPF_2（PRIORITY=1）→ 用户走 UPF_2
3. UPF_1 恢复 → 高优先级，流量回切 UPF_1

## 配置流程

> 每步显示"干啥 + 命令 + 关联骨架"。本特性**独立**于 [2-00015](task/UNC/20.15.2/2-00015.md) WSFD-010502 主特性（虽然共用 ADDRPOOL 命令族，但本特性额外需要 `ADD POOLBINDAPN` 白名单绑定）。

### 步骤 0（前置）：VPN/APN 域基础设施
- → 复用 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施

### 步骤 1：Address Pool（静态地址网段 + HASVPN + CHECKIPVALID）
- `ADD ADDRPOOL`：→ [0-00217](task/UNC/20.15.2/0-00217.md)
- 参数：POOLNAME=pool01, IPVERSION=IPv4, POOLTYPE=UDM, CHECKIPVALID=Enable, HASVPN=Enable, VPNINSTANCE=VRF-A

### 步骤 2（白名单检查·本特性特有）：地址池绑 APN
- `ADD POOLBINDAPN`：→ [0-00250](task/UNC/20.15.2/0-00250.md)
- 参数：APN="huawei.com", POOLNAME="pool01"
- **本步骤是 WSFD-107021 与 WSFD-010502 主特性的关键差异**（主特性通过 POOLGRPMAP 绑 APN→PoolGroup，本特性直接绑 APN→Pool）

### 步骤 3：地址池地址段
- `ADD SECTION`：→ [0-00257](task/UNC/20.15.2/0-00257.md)
- 参数：POOLNAME=pool01, SECTIONNUM=1, IPVERSION=IPv4, V4STARTIP=10.0.0.0, V4ENDIP=10.1.2.3
- **约束**：本地地址池最多 64 段；配置池后必配段，否则用户激活失败

### 步骤 4：地址池组
- `ADD ADDRPOOLGRP`：→ [0-00218](task/UNC/20.15.2/0-00218.md)
- 参数：POOLGRPNAME=poolgroup1, POOLGRPTYPE=UDM

### 步骤 5：地址池绑地址池组
- `ADD POOLBINDGRP`：→ [0-00251](task/UNC/20.15.2/0-00251.md)
- 参数：POOLGRPNAME=poolgroup1, POOLNAME=pool01

### 步骤 6：UPF 节点（主+备）
- `ADD UPNODE`：→ [0-00263](task/UNC/20.15.2/0-00263.md)
- 主：`ADD UPNODE: NFINSTANCENAME="UPF_Instance_1", ADDRALLOCMODE=INHERIT`
- 备：`ADD UPNODE: NFINSTANCENAME="UPF_Instance_2", ADDRALLOCMODE=INHERIT`
- 前置：已通过 `ADD PNFPROFILE` 配置（不在本特性范围内）

### 步骤 7：UPF 组
- `ADD ADDRUPGROUP`：→ [0-00219](task/UNC/20.15.2/0-00219.md)
- 参数：UPFGRPNAME=upfgroup1, UPFGRPTYPE=UDM

### 步骤 8：UPF 绑 UPF 组（主+备 + 优先级）
- `ADD UPFBINDGRP`：→ [0-00259](task/UNC/20.15.2/0-00259.md)
- 主：`ADD UPFBINDGRP: UPFGRPNAME="upfgroup1", UPFID="UPF_Instance_1", PRIORITY=0`
- 备：`ADD UPFBINDGRP: UPFGRPNAME="upfgroup1", UPFID="UPF_Instance_2", PRIORITY=1`

### 步骤 9：地址池组绑 UPF 组
- `ADD POOLGRPMAP`：→ [0-00252](task/UNC/20.15.2/0-00252.md)
- 参数：MAPPINGNAME="mapping01", POOLGRPNAME="poolgroup1", UPFGRPNAME="upfgroup1"

### 步骤 10（UNC 域独有·5GC 强绑定）：路由发布
- → 复用 [UNC 1-00029](task/UNC/20.15.2/1-00029.md) OSPF 路由发布
- **本步配置主 UPF 故障时备用 UPF 的流量承载**——OSPF 需感知 UPF 组上下线（UNC ADDRUPGROUP 自动同步路由表）

## 决策点（场景影响表）

| 优先级策略 × 接入制式 × 网段类型 | 走法（编排骨架 + 自有命令族） |
|---|---|
| **5G PDU 会话 × UDM 静态地址** | [UNC 1-00024](task/UNC/20.15.2/1-00024.md) + [2-00015](task/UNC/20.15.2/2-00015.md) 主特性骨架 + **步骤 2 POOLBINDAPN**（白名单绑定）+ 步骤 3-9（本特性自建链路）+ [UNC 1-00029](task/UNC/20.15.2/1-00029.md) |
| **4G EPS × UDM 静态地址** | 同 5G（SGW-C/PGW-C 视角，Full-Mesh 组网）|
| **2/3G PDP × UDM 静态地址** | 同上（SGSN 视角，Full-Mesh）|
| **Radius 静态地址** | 同上（POOLTYPE=Radius；WSFD-011306 Radius 功能配套）|
| **PRIORITY=0 主 / PRIORITY=1 备**（默认策略） | 步骤 8 标准配置 |
| **PRIORITY=0 主+多备 PRTY=1/2/3**（多 UPF 负载分担） | 步骤 8 多个 UPFBINDGRP |

## 约束

- **POOLTYPE=UDM + CHECKIPVALID=Enable**（critical）：白名单校验必须 ENABLE — 后果：白名单不生效致非法静态地址用户激活。
- **HASVPN=ENABLE + VPN 三者一致**（critical）：VPNINSTANCE 必须与 UPF 侧配置一致 — 后果：VPN 不一致致地址分配/路由异常。
- **PRIORITY=0 主 / PRIORITY=1 备**（critical）：PRIORITY 唯一标识主备；同一 UPF 组内 PRIORITY 不重复 — 后果：主备识别失败致流量黑洞。
- **每本地地址池最多 64 段**（warning）：每 ADDRPOOL 最多 64 个 SECTION — 后果：超规格配置失败。
- **配置池后必配地址段**（critical）：配置 ADDRPOOL 后必须配置 ADD SECTION — 后果：不配置段致用户激活失败。
- **UPF_Instance_1/2 必须已配 ADD PNFPROFILE**（critical，前置）：UPNODE 注册前需对端 NF 实例描述信息已配 — 后果：PNFPROFILE 缺失致 UPNODE 创建失败。
- **本特性 UNC 域依赖 UDG 域**（critical）：UNC 侧网段信息必须包含在 UDG 侧配置地址池内（参见 UDG GWFD-010104）— 后果：UNC/UDG 不一致致用户上线失败。
- **本特性与 UPF 选择特性互依**（critical）：本特性在 WSFD-107010 UPF 选择基础上工作（用户先基于 UPF 选择选主 UPF，才可知该 UPF 故障后切换备用）— 后果：脱离 UPF 选择则主备切换不生效。
- **首次发布版本**（info）：GGSN/PGW-C/SMF 首次发布 UNC **20.5.0**。
- **本特性无需 License**（info）：适用NF 为 GGSN/PGW-C/SMF。
