# 增加IPSEC接口IPv4地址（ADD IFIPV4ADDRESSIPSEC）

> 命令静态知识见 [ADD IFIPV4ADDRESSIPSEC](command/UNC/20.15.2/ADD-IFIPV4ADDRESSIPSEC.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令为 IPSec 域接口（逻辑接口及物理接口）配置 IPv4 地址。配置空间主要由**地址类型**这一维度决定。

### 配置维度1：地址类型（参数 ADDRTYPE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| Main（主地址，默认） | 接口主 IPv4 地址 | 默认场景；Tunnel/LoopBack 接口的唯一地址 |
| Sub（从地址） | 接口从 IPv4 地址 | 一接口多地址场景 |
| Brwd（被借用地址） | 被借用的地址 | 地址借用（被借用方） |
| Brw（借用地址） | 借用他人的地址 | 地址借用（借用方） |

### 配置维度2：地址与掩码（参数 IFIPADDR / SUBNETMASK）

| 参数 | 作用 | 约束 |
|---|---|---|
| IFIPADDR | IPv4 地址 | 不能是 0.0.0.0/255.255.255.255/127.0.0.1；必须 A/B/C 类；系统内唯一 |
| SUBNETMASK | 掩码 | 除 Loopback 外，其他接口不能配 255.255.255.255 |

> IPSec 隧道场景下，Tunnel 接口与 LoopBack 接口的 IPv4 地址通常配 255.255.255.255 掩码（单点地址）。

**步骤位置**：在所属 feature（IPFD-016000 IPSec功能）编排中，于 `ADD INTERFACEIPSEC`（建接口）之后执行。

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无延迟
- **接口需支持配 IP**（warning）：该接口需要支持配置 IP 地址 — 后果：不支持的接口配置失败
- **版本升级过程中禁止执行**（critical）：该命令在版本升级过程中禁止执行 — 后果：升级中执行可能导致配置异常
- **每接口最多 256 个 IP**（warning）：每个主用接口最大允许 1 个主 IP + 255 个从 IP — 后果：超规格新增失败
- **IFNAME 必须已存在**（critical）：IFNAME 必须可通过 `LST INTERFACEIPSEC` 查询到（即先 `ADD INTERFACEIPSEC`） — 后果：索引失败
- **IPv4 地址系统内唯一**（critical）：IPv4 地址在系统内必须唯一 — 后果：重复配置失败
- **最多 65535 条记录**（warning）：本命令最多可输入 65535 条记录 — 后果：超规格新增失败
