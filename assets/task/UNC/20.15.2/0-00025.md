# ADD DEACTQFPLCY 原子任务

> 命令静态知识见 [ADD DEACTQFPLCY](command/UNC/20.15.2/ADD-DEACTQFPLCY.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD DEACTQFPLCY 配置**整系统（全局）粒度**的专有 GBR QoS Flow 去活策略。每条记录由 NgAp Cause（组 + 值）定位，决定 SMF 收到该 Cause 的去激活用户面请求时，对专有 GBR QoS Flow 的处理动作（立即释放 / 延迟释放）及延迟时长。系统部署后已存在初始记录（如 RADIO_NETWORK+20→NOT_RELEASE+60），本命令用于覆盖或新增。APN 粒度的同类策略见 [ADD APNDEACTQFPLCY](task/UNC/20.15.2/0-00006.md)。

**参数取值样例（VoNR 承载延时保活范式，WSFD-221001）**

| 参数 | 取值样例 | 默认 | 说明 |
|---|---|---|---|
| NGAPCAUSEGROUP | `RADIO_NETWORK` | — | NgAp Cause 组（必填），Radio Network Layer |
| NGAPCAUSEVALUE | `21` | — | NgAp Cause 值（必填），0~255；21=radio-connection-with-ue-lost（UE 进电梯/隧道等无信号区） |
| QFPLCY | `DELAY_RELEASE` | — | 专有 GBR QoS Flow 处理策略（必填）；DELAY_RELEASE=延迟释放 |
| DELAYTIMER | `60` | 60 | 延迟释放时长（秒），10~100；仅 QFPLCY=DELAY_RELEASE 时必填 |

**典型行（UE 无线连接丢失时延迟 60 秒释放专有 QoS Flow）**

```
ADD DEACTQFPLCY: NGAPCAUSEGROUP=RADIO_NETWORK, NGAPCAUSEVALUE=21, QFPLCY=DELAY_RELEASE, DELAYTIMER=60;
```

**配置位置**：在"激活 VoNR 承载延时保活（WSFD-221001）"流程中作为第 2 步可选项（可选地配整系统粒度策略），其后第 3 步可选配 APN 粒度策略（ADD APNDEACTQFPLCY）。也出现于"业务触发的 QoS 保证（WSFD-109107）"的命令清单中。

## 决策点

### 处理策略 QFPLCY（立即释放 vs 延迟释放）

| 取值 | 行为 | DELAYTIMER | 适用 |
|---|---|---|---|
| `RELEASE` | 立即释放专有 QoS Flow | 不填 | 无需保活，Cause 命中即释放 |
| `NOT_RELEASE` | 不释放 | 不填 | 初始记录默认值（如 Cause 20） |
| `DELAY_RELEASE` | 延迟释放 | 必填（10~100 秒） | 短暂无线丢失（如电梯/隧道）期间保活 VoNR 承载，等待 UE 重连 |

### 全局粒度 vs APN 粒度

| 选项 | 命令 | 优先级 |
|---|---|---|
| 整系统粒度 | ADD DEACTQFPLCY（本任务） | 低 |
| APN 粒度 | ADD APNDEACTQFPLCY | **高**（APN 粒度 > 全局粒度） |

> 同一 Cause 下 APN 粒度策略优先于全局策略；全局策略作为 APN 未配置时的兜底。

> ③ 数据规划表"获取方法"列分布为单一值 `本端规划(4)`，Cause 组合与策略由本端规划决定 → 绑定类型 local_planned。具体 NGAPCAUSEVALUE 的取值（哪些 Cause 需延迟释放、延迟多久）依赖业务保活需求 → 策略本身是规划侧决策。

## 约束

- **立即生效** · severity=LOW。
- **记录上限**：最多 255 条记录 · severity=MEDIUM · 超过上限新增失败。
- **初始记录**：系统部署后已存在初始记录（如 RADIO_NETWORK+20→NOT_RELEASE+60 等），ADD 会覆盖或新增，需避免与初始记录 Cause 重复 · severity=MEDIUM。
- **DELAYTIMER 条件**：仅 QFPLCY=DELAY_RELEASE 时必填，取值 10~100 秒 · severity=HIGH · QFPLCY 非 DELAY_RELEASE 时配 DELAYTIMER 无意义。
- **粒度优先级**：APN 粒度（ADD APNDEACTQFPLCY）> 全局粒度（本命令） · severity=MEDIUM · 同 Cause 下 APN 策略覆盖全局策略。
- **唯一性**：NGAPCAUSEGROUP + NGAPCAUSEVALUE 唯一确定一条全局记录 · severity=HIGH · 重复 Cause 新增失败。
