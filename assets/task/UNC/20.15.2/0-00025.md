# 配置去活用户面专有QoSFlow策略（ADD DEACTQFPLCY）

> 命令静态知识见 [ADD DEACTQFPLCY](command/UNC/20.15.2/ADD-DEACTQFPLCY.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD DEACTQFPLCY 配置**整系统（全局）粒度**的专有 GBR QoS Flow 去活策略。每条记录由 NgAp Cause（组 + 值）定位，决定 SMF 收到该 Cause 的去激活用户面请求时，对专有 GBR QoS Flow 的处理动作（立即释放 / 不释放 / 延迟释放）及延迟时长。系统部署后已存在初始记录（RADIO_NETWORK+20→NOT_RELEASE+60、RADIO_NETWORK+41→NOT_RELEASE+60），本命令用于覆盖或新增。APN 粒度的同类策略见 [ADD APNDEACTQFPLCY](task/UNC/20.15.2/0-00006.md)。

### 配置维度 1：NgAp Cause 组（参数 NGAPCAUSEGROUP）

| 取值 | 含义 | 说明 |
|---|---|---|
| `RADIO_NETWORK` | Radio Network Layer | 最常配（UE 无线连接丢失、用户不活动等） |
| `TRANSPORT` | Transport Layer | 传输资源不可用 |
| `NAS` | NAS | 正常释放/认证失败/注销 |
| `PROTOCOL` | Protocol | 语法/语义错误 |
| `MISC` | Miscellaneous | 过载/硬件故障/运维干预 |
| `INVALID` | 未携带 | 此时 NGAPCAUSEVALUE 只能填 0 |

### 配置维度 2：NgAp Cause 值（参数 NGAPCAUSEVALUE）

| 取值 | 含义 | 适用 |
|---|---|---|
| `0~255` | Cause 值整数 | 参照 3GPP TS 38.413；RADIO_NETWORK 常用 21=UE 无线连接丢失、19=降低服务小区负载、20=用户不活动 |
| `0` | 当 NGAPCAUSEGROUP=INVALID | 仅此值可用 |

### 配置维度 3：处理策略（参数 QFPLCY）

| 取值 | 行为 | DELAYTIMER |
|---|---|---|
| `RELEASE` | 立即释放专有 QoS Flow | 不填 |
| `NOT_RELEASE` | 不释放 | 不填 |
| `DELAY_RELEASE` | 延迟释放 | 必填（10~100 秒） |

### 配置维度 4：延迟释放时长（参数 DELAYTIMER，条件必选）

| 取值 | 作用 | 适用 |
|---|---|---|
| `10~100`（秒） | 延迟释放时长，默认 60 | 仅 QFPLCY=DELAY_RELEASE 时必填 |

### 典型脚本（VoNR 承载延时保活范式，WSFD-221001）

```
ADD DEACTQFPLCY: NGAPCAUSEGROUP=RADIO_NETWORK, NGAPCAUSEVALUE=21, QFPLCY=DELAY_RELEASE, DELAYTIMER=60;
```

**步骤位置**：在"激活 VoNR 承载延时保活（WSFD-221001）"流程中作为第 2 步可选项（可选地配整系统粒度策略），其后第 3 步可选配 APN 粒度策略（ADD APNDEACTQFPLCY）。也出现于"业务触发的 QoS 保证（WSFD-109107）"的命令清单中。

## 决策点

### 决策点：处理策略 QFPLCY 选择（DP 0-02029）

问：该 Cause 命中时对专有 GBR QoS Flow 做什么动作？

| 选项 | 行为 | DELAYTIMER | 适用 |
|---|---|---|---|
| `RELEASE` | 立即释放 | 不填 | 无需保活，Cause 命中即释放 |
| `NOT_RELEASE` | 不释放 | 不填 | 初始记录默认值（如 Cause 20/41） |
| `DELAY_RELEASE` | 延迟释放 | 必填（10~100 秒） | 短暂无线丢失（电梯/隧道）期间保活 VoNR 承载，等待 UE 重连 |

### 决策点：全局粒度 vs APN 粒度（DP 0-02030）

问：该 Cause 的去活策略应配在全局（本命令）还是 APN 粒度（ADD APNDEACTQFPLCY）？

| 选项 | 命令 | 优先级 |
|---|---|---|
| 整系统粒度 | ADD DEACTQFPLCY（本任务） | 低（兜底） |
| APN 粒度 | ADD APNDEACTQFPLCY | **高**（同 Cause 下 APN 策略覆盖全局） |

> 同一 Cause 下 APN 粒度策略优先于全局策略；全局策略作为 APN 未配置时的兜底。数据规划表"获取方法"列分布为单一值 `本端规划`，Cause 组合与策略由本端规划决定 → 绑定类型 local_planned。

## 约束

- rule-0-02071 · severity=high · DELAYTIMER 条件必选：仅 QFPLCY=DELAY_RELEASE 时必填，取值 10~100 秒；QFPLCY 非 DELAY_RELEASE 时配 DELAYTIMER 无意义 — 策略与延迟参数须匹配
- rule-0-02072 · severity=high · 唯一性：NGAPCAUSEGROUP + NGAPCAUSEVALUE 唯一确定一条全局记录，重复 Cause 新增失败 — 规划前查现有记录
- rule-0-02073 · severity=info · 立即生效：命令执行后立即生效，无需重启 — 配置即时生效
- rule-0-02074 · severity=warning · 记录上限：最多 255 条记录，超过上限新增失败 — 规划前统计现有记录数
- rule-0-02075 · severity=warning · 初始记录覆盖风险：系统部署后已存在初始记录（RADIO_NETWORK+20→NOT_RELEASE+60、+41→NOT_RELEASE+60），ADD 会覆盖或新增，需避免与初始记录 Cause 重复误覆盖 — 改写前核对初始记录
