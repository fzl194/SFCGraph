# MPLS VPN（WSFD-104411，能力型底座）

> **能力型底座特性（§3.7 范式）**：本特性原始产品文档**仅有 8 个概念/原理 md，无「激活」/「特性配置」子目录**——MPLS VPN 的实际配置承载在 UNC 各 L3VPN 接入特性中，由用户按实际组网（6 组网变体）选用 [UNC 1-00031](task/UNC/20.15.2/1-00031.md) MPLS VPN 基础设施的命令族配置。
>
> 本特性**提供"被依赖关系"的可追溯锚点**——业务图谱中 MPLS VPN 是其他特性的底座（L3VPN 实例 + VPN 实例 + 逻辑接口绑 VPN 是各接入特性的基础设施）；task wiki 让审查/配置生成 Agent 一眼看到"此特性为底座，配置查找请到被依赖特性"。
>
> **资源浪费很小**（1 个 feature + 1 个 compound ≈ 100 行）；信息密度高。

## 配置概览

MPLS VPN 在 UNC 域是**多协议族承载能力**——BGP/LDP/MPLS/IPv6 VPN 等命令族共同实现，**核心对象**：
- **VRF 实例**（`ADD L3VPNINST` + `ADD VPNINSTAF`）—— MPLS VPN 的核心标识
- **逻辑接口 + 逻辑 IP**（`ADD LOGICINF` + `ADD LOGICIP`）—— VRF 内逻辑寻址
- **物理接口绑 VPN**（`ADD IPBINDVPN`）—— VRF 流量入口
- **RT 路由标识**（`VRFRD=100:1`）—— 跨设备 VRF 路由互通标识

6 组网变体**共享 MPLS VPN 基础设施**（[UNC 1-00031](task/UNC/20.15.2/1-00031.md)），差异在组网拓扑（Hub&Spoke / Multi-VPN-Instance CE / 跨域 VPN / BGP_MPLS IPv6 VPN / VPN GR / VPN NSR / 分标签方式）。由特性级 DP（组网变体）驱动。

**License 信息**：本特性**无独立 License**——具体组网变体（如 VPN GR/NSR）可能涉及 [0-00180](task/UNC/20.15.2/0-00180.md) SET LICENSESWITCH 开关。

## 配置流程（步骤编排，每步显示含命令）

> **★ 能力型底座特性**：本特性**不内嵌具体组网变体的命令序列**——变体的命令差异由对应的 L3VPN/LDP/MPLS/BGP 命令族承担。用户实际配置时**先按接入域骨架 + MPLS VPN 基础设施骨架建立基础，再按组网变体启用对应命令族**。

1. **接入域基础设施**（L3VPN + VPN 实例 + APN + 使能地址分配）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR` → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **MPLS VPN 基础设施**（VRF 实例 + 地址族 + 逻辑接口绑 VPN + 物理接口绑 VPN）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `MOD VPNINSTAF` + `ADD IPBINDVPN` + `ADD LOGICINF` + `ADD LOGICIP` → [UNC 1-00031](task/UNC/20.15.2/1-00031.md)
3. **（按组网变体）扩展配置**（依赖变体命令族）：
   - **Hub&Spoke 拓扑**：多个 VRF 实例（Hub + Spoke）+ `MOD VPNINSTAF` 调整 RT 控制路由流向
   - **Multi-VPN-Instance CE**：CE 设备上多 VRF 实例与单 PE 多 VRF 互联
   - **跨域 VPN**：AS 间 Option A/B/C，命令族涉及 BGP peer/RR 跨域
   - **BGP_MPLS IPv6 VPN**：IPv6 地址族 + 6PE/6VPE 场景
   - **VPN GR / VPN NSR**：可靠性增强，由 `SET LICENSESWITCH` 等命令族承担
   - **MPLS 分标签方式**：标签分发协议由 LDP/MP-BGP 命令族承担
4. **下行路由发布**（让骨干网可达 VRF 路由）：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFINTERFACE` + `ADD OSPFIMPORTROUTE` → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)
   - 关键参数：PROCID, AREAID, IFNAME

## 约束（特性级 rule）

- **VPN 三者一致**（critical）：L3VPN/VPNINST 的 VRFNAME、ADD APN 的 VPNINSTANCE、外联口绑定的 VPN 三者必须一致 — 后果：VPN 不一致致地址分配或路由异常。
- **VRFRD 全网规划**（critical）：VRFRD（RT 路由标识）全网规划统一 — 后果：VRF 路由无法跨设备互通。
- **AFTYPE 与 IP 类型匹配**（warning）：IPv4 场景 AFTYPE=ipv4uni；IPv6 场景 AFTYPE=ipv6uni；双栈两者都配 — 后果：地址族不匹配致 VPN 实例失效。
- **能力型底座特性（§3.7）**：本特性**无独立配置流程**——实际配置由用户按组网变体选用 [UNC 1-00031](task/UNC/20.15.2/1-00031.md) 骨架 + 对应命令族。
- **跨域 VPN 三方案选型**（warning）：跨域 VPN Option A（VRF-to-VRF，最简单）/ Option B（单跳 MP-eBGP）/ Option C（多跳 MP-eBGP + 标签分发）—— 三方案对 ASBR 性能/扩展性影响不同，按全网规划选型。
- **BGP_MPLS IPv6 VPN IPv6 路由**（warning）：BGP_MPLS IPv6 VPN 需 IPv6 路由协议承载（OSPFv3），UNC 域 OSPFv3 系列 atom 待补。
- **VPN GR/NSR 需 License**（warning）：VPN GR（平滑重启）/ VPN NSR（不中断路由）需对应 License 支持，由 [0-00180](task/UNC/20.15.2/0-00180.md) SET LICENSESWITCH 等命令族承担。
- **首次发布版本**（info）：本特性首次发布版本为 UNC **20.0.0**（MPLS VPN 基础 + Hub&Spoke + 分标签方式等核心组网）。
