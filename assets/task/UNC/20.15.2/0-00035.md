# ADD GUAMI 原子任务
> 命令静态知识见 [ADD GUAMI](command/UNC/20.15.2/ADD-GUAMI.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
增加 GUAMI 在"初始配置 > 配置 AMF"阶段完成。PLMN/Region/Set 三段标识同一 AMF Set（对应一个 AMF Pool），Pointer 在 Set 内区分不同 AMF 实例。同一 AMF 实例可配多个 GUAMI（多 PLMN 场景）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| INDEX | GUAMI 索引（本端规划，0~255） | `0` |
| PLMNIDX | PLMN 索引（global_planned，0~127） | `0` |
| AMFREGIONID | AMF 区域标识，十六进制 2 位（无 `0x` 前缀） | `"2A"` |
| AMFSETID | AMF 集合标识，十六进制 3 位（无 `0x` 前缀） | `"1F4"` |
| AMFPOINTER | AMF 指示符，十六进制 2 位（无 `0x` 前缀） | `"00"` |
| BACKUPAMFNAME | 备用 AMF 名称（可选，0~150 字符） | — |

典型脚本（示意，实际值由网络规划给定）：
```
ADD GUAMI:INDEX=0,PLMNIDX=0,AMFREGIONID="2A",AMFSETID="1F4",AMFPOINTER="00";
```

## 决策点

### Pool 内多 AMF 的 GUAMI 划分
AMF Set 对应一个 AMF Pool，Pool 内所有 AMF 共享相同的 PLMN + AMF Region ID + AMF Set ID（这三段全球唯一标识一个 AMF Set），仅 AMF Pointer 区分实例：
- 同 Set 内多个 AMF 实例 → 各自配 GUAMI，三段标识相同，Pointer 不同。
- 多 PLMN 部署 → 同一 AMF 实例配多条 GUAMI，PLMNIDX 不同时 Pointer 可同可不同。

### BACKUPAMFNAME 是否指定
AMF 实例支持以 GUAMI 粒度指定备用 AMF。是否填 BACKUPAMFNAME 取决于 Pool 内是否做 1:1 备份规划：
- 启用 GUAMI 粒度备份 → 填备用 AMF 名（须为 Pool 内合法 AMF）。
- 不做粒度备份 / 走 Pool 级冗余 → 留空。

## 约束
- 同一性约束（同一 AMF 实例多条 GUAMI 间）：AMFREGIONID 和 AMFSETID 必须相同；当 PLMNIDX 相同时 AMFPOINTER 必须不同，PLMNIDX 不同时 AMFPOINTER 可同可不同。`severity=high` · 后果：违反则 GUAMI 配置非法，标识冲突。
- 唯一性：同一个 AMF Set 内，各 AMF 配置的 GUAMI 不能相同。`severity=high` · 后果：Pool 内实例标识冲突，用户路由错乱。
- 生效时机：执行后立即生效。`severity=low`
