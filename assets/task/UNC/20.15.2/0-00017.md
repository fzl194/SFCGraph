## 配置方法

ADD CHGCG 为 SGSN 离线计费链路登记一条 CG（计费网关）记录。一条记录由"GTP 承载协议（PRO）+ CG IP + CG 接收端口号（SPN）"三者唯一确定，登记后才能在 `SET CHGCHAR` / `ADD CHGBEHA` / `ADD CHGPLMNCG` 中被引用。配置时按整网规划填 CG 的 IP、端口、协议版本与优先级，并指定是否为缺省 CG。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| IP 地址族 | IPT | IPV4 / IPV6 | IPV4 | 决定填 IP 还是 IPV6 | 承载网络协议栈决定 |
| IPv4 地址 | IP | 0.0.0.0~255.255.255.255（不可为环回 127.x.y.z） | — | CG 的 IPv4 地址 | IPT=IPV4 时生效 |
| IPv6 地址 | IPV6 | :: ~ FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF（不可为环回/链路本地/组播） | — | CG 的 IPv6 地址 | IPT=IPV6 时生效 |
| 承载协议 | PRO | UDP / TCP | — | GTP' 承载协议（必填） | 现版本暂不支持 TCP，实测均填 UDP |
| CG 协议版本 | CGR | R98 / R99 / R4 / R5 / R6 / R7 / R9 | — | CG 支持的协议版本（必填） | 需与 CG 实际版本及 SET CHGGA 话单/CDR 版本一致 |
| 接收端口号 | SPN | 1024~65535 | 3386 | CG 接收端口 | 与对端 CG 设置一致；相同 IP 的 CG 间端口号不能相同 |
| 优先级 | GRD | 0~11 | 0 | CG 优先级，小值优先；同优先级同用户话单固定发、不同用户轮选 | DEFAULTCG=YES 时生效，多 CG 场景规划 |
| 缺省 CG | DEFAULTCG | YES / NO | YES | 当 PLMN CG / 计费行为 CG / 计费属性 CG 推荐均不可用时是否走缺省 CG | 兜底 CG 选路；EXTRANGE=NO 时 PLMN CG 的 CGIP 此处须为 NO |
| CG 名 | CGN | 1~32 位字符串 | noname | CG 名称标识 | 可读性命名 |

典型脚本：
```
ADD CHGCG: IPT=IPV4, IP="172.22.5.50", PRO=UDP, CGR=R98, SPN=3386, DEFAULTCG=YES, GRD=0, CGN="xx";
ADD CHGCG: IPT=IPV4, IP="10.10.171.9", PRO=UDP, CGR=R6, DEFAULTCG=YES, GRD=0, CGN="abc";
```

步骤位置：SGSN 离线计费 CG 对接的第 1 步（CG 记录登记）→ 其后 `SET CHGCHAR` / `ADD CHGBEHA` / `ADD CHGPLMNCG` 按需引用本 CG。

## 决策点

### 决策点：IP 地址族选 IPV4 还是 IPV6（DP 0-02013）

IPT 决定填 IP 还是 IPV6，由承载网络协议栈决定。IPV4 与 IPV6 二选一，不可同时填。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| IPv4 承载 | IPT=IPV4，填 IP（IPv4 地址） | 多数现网传统对接 |
| IPv6 承载 | IPT=IPV6，填 IPV6（IPv6 地址） | IPv6 组网 |

### 决策点：CG 协议版本对齐策略（DP 0-02014）

CGR 取 R98/R99/R4/R5/R6/R7/R9 之一，取决于 CG 设备实际支持版本。GPRS 可用 R98~R9；WCDMA 可用 R99~R9（不含 R98）。CGR 必须与 `SET CHGGA` 设置的 UMTS 话单版本 / CDR 协议版本一致，否则话单解析失败。

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| R98 | CGR=R98 | 仅 GPRS 场景的旧版 CG |
| R4/R5/R6/R7/R9 | CGR=R4 等 | GPRS 或 WCDMA 通用 |
| R99 | CGR=R99 | WCDMA 旧版 CG（GPRS 也可） |

## 约束

· rule-0-02031 · severity=critical · 前置依赖：只有在本命令登记过的 CG，才能在 SET CHGCHAR、ADD CHGBEHA、ADD CHGPLMNCG 中被引用生效；漏配 CHGCG 会导致计费属性/行为/PLMN-CG 配置无 CG 可挂
· rule-0-02032 · severity=high · 唯一性：PRO + IP（或 IPV6）+ SPN 三者唯一确定一条记录，重复登记新增失败
· rule-0-02033 · severity=high · 版本一致性：CGR 必须与 CG 实际协议版本及 SET CHGGA 的话单/CDR 版本一致，不一致导致 CG 解析话单失败
· rule-0-02034 · severity=warning · 规格上限：本表最大 16 条记录，超过上限新增失败
· rule-0-02035 · severity=info · 生效时机：命令执行后立即生效
