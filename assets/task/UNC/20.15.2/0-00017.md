# ADD CHGCG 原子任务

> 命令静态知识见 [ADD CHGCG](command/UNC/20.15.2/ADD-CHGCG.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CHGCG 为 SGSN 离线计费链路登记一条 CG（计费网关）记录。一条记录由"承载协议 + CG IP + 接收端口"唯一确定，登记后才能在 SET CHGCHAR / ADD CHGBEHA 中被引用。配置时按整网规划填 CG 的 IP、端口、协议版本与优先级，并指定是否为缺省 CG。

**参数取值样例（离线计费 CG 对接范式）**

| 参数 | 取值样例 | 默认 | 说明 |
|---|---|---|---|
| IPT | `IPV4` | IPV4 | IP 地址类型；选 IPV4 走 IP，选 IPV6 走 IPV6 |
| IP / IPV6 | `10.10.171.9` | — | CG 的 IP；IPT=IPV4 填 IP，IPT=IPV6 填 IPV6，二选一 |
| GRD | `0` | 0 | 优先级 0~11，多 CG 时小值优先 |
| PRO | `UDP` | — | GTP 承载协议（必填） |
| CGR | `R6` / `R98` | — | CG 协议版本（必填）；需与 CG 实际版本及 SET CHGGA 设置的话单/CDR 版本一致 |
| SPN | `3386` | 3386 | CG 接收端口号，1024~65535 |
| DEFAULTCG | `YES` | YES | 是否缺省 CG；YES/NO |
| CGN | `abc` | noname | CG 名，1~32 字符 |

**典型行（新增一个缺省 CG）**

```
ADD CHGCG: IP="10.10.171.9", PRO=UDP, CGR=R6, DEFAULTCG=YES, GRD=0, CGN="abc";
```

## 决策点

### IP 地址类型选 IPV4 还是 IPV6

IPT 决定填 IP 还是 IPV6，承载网络协议栈决定。

| 选项 | IPT | IP/IPV6 | 适用 |
|---|---|---|---|
| IPv4 承载 | IPV4 | 填 IP（IPv4 地址） | 多数现网传统对接 |
| IPv6 承载 | IPV6 | 填 IPV6（IPv6 地址） | IPv6 组网 |

### 协议版本需与 CG 实际版本对齐

CGR 取 R98 或 R6 等取决于 CG 设备实际支持版本；同时 SET CHGGA 设置的 UMTS 话单版本 / CDR 协议版本必须与 CGR 一致，否则话单解析失败。

## 约束

- **立即生效** · severity=LOW。
- **本表最大 16 条记录** · severity=MEDIUM · 超过上限新增失败。
- **唯一性**：PRO + IP + SPN 三者唯一确定一条记录 · severity=HIGH · 重复登记新增失败。
- **前置依赖**：只有在本命令登记过的 CG，才能在 SET CHGCHAR、ADD CHGBEHA 中被引用生效 · severity=HIGH · 漏配 CHGCG 会导致计费属性/行为配置无 CG 可挂。
- **版本一致性**：CGR 必须与 CG 实际协议版本及 SET CHGGA 的话单/CDR 版本一致 · severity=HIGH · 不一致导致 CG 解析话单失败。
