# 配置计费控制总开关（SET CHARGECTRL）

> 命令静态知识见 [SET CHARGECTRL](command/UNC/20.15.2/SET-CHARGECTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CHARGECTRL 是 PGW-C/SMF 侧计费方式的**全局总开关**：按"用户归属属性 × 计费方式"二维勾选要使能的总开关。只有总开关使能了对应计费方式，UserProfile、APN、CC、RADIUS 等其他计费开关配置才生效（PCRF 下发的计费方式不受此控制）。系统存在初始记录（默认 9 个计费开关全部 ENABLE + 2 个备份流量开关 ENABLE）。本命令的核心配置空间有 2 个维度：**归属属性 × 计费方式开关矩阵**（必选，9 个开关）、**备份流量处理**（可选，2 个开关）。

### 配置维度 1：归属属性 × 计费方式开关矩阵（9 个开关）

按"归属属性（本地 HOME / 拜访 VISIT / 漫游 ROAM）× 计费方式（在线 ONLINE / 离线 OFFLINE / 融合 CONVERGED）"组合成 9 个开关，每个开关取值 `ENABLE` / `DISABLE`：

| 归属属性 | 在线计费 | 离线计费 | 融合计费 |
|---|---|---|---|
| 本地（HOME） | `HOMEONLINE` | `HOMEOFFLINE` | `HOMECONVERGED` |
| 拜访（VISIT） | `VISITONLINE` | `VISITOFFLINE` | `VISITCONVERGED` |
| 漫游（ROAM） | `ROAMONLINE` | `ROAMOFFLINE` | `ROAMCONVERGED` |

| 计费方式 | 使能开关后缀 | 适用接口 | 典型场景 |
|---|---|---|---|
| **在线 ONLINE** | `*ONLINE` | Gy/DCC、Diameter在线 | Gy 在线计费、OCS 实时配额 |
| **离线 OFFLINE** | `*OFFLINE` | Ga 接口、CG话单 | 传统 CG 离线计费、话单结算 |
| **融合 CONVERGED** | `*CONVERGED` | Nchf/N40、5GC融合 | 5GC 融合计费、CHF 服务化 |

> 数据来源：全网规划。系统初始值全部 ENABLE（即默认三种计费方式对三类用户全开）。实际部署按网络实际部署的计费体系裁剪：只部署离线→只使能 OFFLINE 三项；只部署融合→只使能 CONVERGED 三项。

### 配置维度 2：备份流量处理（2 个开关，可选）

| 参数 | 控制对象 | 取值 | 适用场景 |
|---|---|---|---|
| `N40BKPTRAFFIC` | 融合计费是否处理 UPF 上报的备份流量（N40 接口） | `ENABLE` / `DISABLE` | 融合计费容灾/备份链路 |
| `GAGYBKPTRAFFIC` | 在/离线计费是否处理 UPF 上报的备份流量（GaGy 接口） | `ENABLE` / `DISABLE` | 在/离线计费容灾/备份链路 |

> 数据来源：本端规划。系统初始值均为 ENABLE。涉及计费容灾/备份链路时按规划使能，无容灾规划可 DISABLE。

典型脚本（离线场景，使能三类属性的离线开关）：
```
SET CHARGECTRL: HOMEOFFLINE=ENABLE,VISITOFFLINE=ENABLE,ROAMOFFLINE=ENABLE;
```
融合场景（使能三类属性的融合开关）：
```
SET CHARGECTRL: HOMECONVERGED=ENABLE,VISITCONVERGED=ENABLE,ROAMCONVERGED=ENABLE;
```
关闭归属地/拜访地融合（仅保留漫游融合，典型裁剪）：
```
SET CHARGECTRL: HOMECONVERGED=DISABLE, VISITCONVERGED=DISABLE;
```

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：配置计费总开关"——License 前置之后、UserProfile/APN/CC 等下游计费开关配置之前，是下游开关生效的前提。

## 决策点

### 决策点：计费方式使能（DP 0-03053）

问：网络部署了哪种计费体系，对应使能哪些总开关？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 部署离线计费（传统 CG） | 使能 `HOMEOFFLINE`/`VISITOFFLINE`/`ROAMOFFLINE` 三项 | Ga 接口话单计费、传统 PGW/GGSN |
| 部署在线计费（Gy/DCC） | 使能 `HOMEONLINE`/`VISITONLINE`/`ROAMONLINE` 三项 | Gy 在线计费、OCS 实时配额 |
| 部署融合计费（Nchf/N40） | 使能 `HOMECONVERGED`/`VISITCONVERGED`/`ROAMCONVERGED` 三项 | 5GC 融合计费、CHF 服务化 |
| 默认全开（系统初始） | 9 个开关全部 ENABLE | 未明确裁剪，三种方式均放通 |

> 默认同时使能在/离线/融合；按实际部署裁剪未部署的计费方式，避免无效开关。

### 决策点：备份流量处理（DP 0-03054）

问：融合（N40）/在线离线（GaGy）接口是否需要处理备份流量？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 使能备份流量 | `N40BKPTRAFFIC=ENABLE` 和/或 `GAGYBKPTRAFFIC=ENABLE` | 涉及计费容灾/备份链路，需处理 UPF 上报的备份流量 |
| 不使能（裁剪） | `N40BKPTRAFFIC=DISABLE` 和/或 `GAGYBKPTRAFFIC=DISABLE` | 无容灾规划，减少备份流量处理开销 |

> 取值来源本端规划，视容灾规划决定。系统初始值均为 ENABLE。

## 约束

- **只对新激活用户生效**（high，rule-0-03131）：执行后只对新激活用户生效，不立即作用于在线用户 — 后果：存量在线用户计费方式不变
- **最大记录数 1**（critical，rule-0-03132）：系统存在初始记录，本命令即修改该唯一记录，不可新增第二条 — 后果：重复新增失败
- **与 CHARGEMETHOD 一致性**（high，rule-0-03133）：本命令按漫游属性配计费方式，[ADD CHARGEMETHOD](task/UNC/20.15.2/0-00015.md) 按 CC 属性配计费方式，两者针对同一用户必须一致 — 后果：计费出错（例：本地用户设在线、NORMAL 设离线，冲突）
- **总开关优先级地位**（high，rule-0-03134）：本地其他计费开关（UserProfile/APN/CC/RADIUS 下发）的生效前提；本地开关优先级从高到低：PCRF > 本地总开关（本命令）> RADIUS > UserProfile > APN > 用户计费属性 — 后果：总开关 DISABLE 后下游开关配置无效
- **PCRF 下发不受控**（warning，rule-0-03135）：PCRF 下发的计费方式不受本命令控制 — 后果：PCRF 指定的计费方式即使总开关 DISABLE 仍生效
