# SET CHARGECTRL 原子任务
> 命令静态知识见 [SET CHARGECTRL](command/UNC/20.15.2/SET-CHARGECTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

**高危命令**。本命令是本地配置侧在线/离线/融合计费的总开关：只有总开关使能了对应计费方式，UserProfile、APN、CC、RADIUS 等其他计费开关配置才生效；PCRF 下发的计费方式不受此控制。系统存在初始记录（默认同时使能在/离线/融合）。

## 配置方法
按"归属属性 × 计费方式"二维勾选要使能的总开关；按场景成组配置。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| HOMEONLINE | 归属地在线计费开关 | `ENABLE` |
| HOMEOFFLINE | 归属地离线计费开关 | `ENABLE` |
| HOMECONVERGED | 归属地融合计费开关 | `ENABLE` |
| VISITONLINE | 拜访地在线计费开关 | `ENABLE` |
| VISITOFFLINE | 拜访地离线计费开关 | `ENABLE` |
| VISITCONVERGED | 拜访地融合计费开关 | `ENABLE` |
| ROAMONLINE | 漫游在线计费开关 | `ENABLE` |
| ROAMOFFLINE | 漫游离线计费开关 | `ENABLE` |
| ROAMCONVERGED | 漫游融合计费开关 | `ENABLE` |
| N40BKPTRAFFIC | 融合计费是否处理备份流量（N40 接口） | `DISABLE` |
| GAGYBKPTRAFFIC | 在/离线计费是否处理备份流量（GaGy 接口） | `DISABLE` |

典型脚本（离线场景，使能三类属性的离线开关）：
```
SET CHARGECTRL: HOMEOFFLINE=ENABLE,VISITOFFLINE=ENABLE,ROAMOFFLINE=ENABLE;
```
融合场景（使能三类属性的融合开关）：
```
SET CHARGECTRL: HOMECONVERGED=ENABLE,VISITCONVERGED=ENABLE,ROAMCONVERGED=ENABLE;
```

## 决策点

### 计费方式使能（在线/离线/融合）
- 问什么：网络部署了哪种计费体系，对应使能哪些总开关。
- 部署离线计费（传统 CG）→ 使能 OFFLINE 三项。
- 部署在线计费（Gy/DCC）→ 使能 ONLINE 三项。
- 部署融合计费（Nchf/N40）→ 使能 CONVERGED 三项。
- 默认同时使能在/离线/融合；按实际部署裁剪。

### 备份流量处理
- 问什么：融合（N40）/在线离线（GaGy）接口是否需要处理备份流量。
- 涉及计费容灾/备份链路时使能；默认 DISABLE（不使能）。
- 取值来源 local_planned，视容灾规划决定。

## 约束
- 生效时机：执行后只对新激活用户生效，不立即作用于在线用户。`severity=medium`
- 唯一性：最大记录数 1（系统存在初始记录，本命令即修改该唯一记录）。`severity=high`
- 一致性：本命令按漫游属性配计费方式，[ADD CHARGEMETHOD](task/UNC/20.15.2/0-00015.md) 按 CC 属性配计费方式，两者针对同一用户必须一致，否则计费出错（例：本地用户设在线、NORMAL 设离线，冲突）。`severity=high`
- 总开关地位：本地其他计费开关（UserProfile/APN/CC/RADIUS 下发）的生效前提；PCRF 下发不受此控制。本地开关优先级从高到低：PCRF > 本地总开关（本命令）> RADIUS > UserProfile > APN > 用户计费属性。`severity=high`
