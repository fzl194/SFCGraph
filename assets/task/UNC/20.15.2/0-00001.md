# ADD URR 原子任务
> 命令静态知识见 [ADD URR](command/UNC/20.15.2/ADD-URR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
URR 按"业务 × 计费方式"规划：`USAGERPTMODE` 决定计费轨道，离线走 `OFFCOMPOUNDTYPE/OFFMETERINGTYPE/RG(+SID)`，在线走 `ONLCOMPOUNDTYPE/ONLMETERINGTYPE/ONLINERG(+ONLINESID)`，QoS 保证走极简三参。URRID 全网唯一、URRNAME 带语义后缀，便于跨网元（SMF/UPF、PGW-C/PGW-U）对齐。成对出现的离线+在线 URR 通过 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md) 绑定。

| 计费轨道 | 必备参数 | 典型脚本 |
|---|---|---|
| 离线 OFFLINE | `USAGERPTMODE=OFFLINE` + `OFFCOMPOUNDTYPE` + `RG`(+可选 `SID`) + `OFFMETERINGTYPE` | `ADD URR:URRNAME="urr_offline_URL",URRID=12500,USAGERPTMODE=OFFLINE,OFFCOMPOUNDTYPE=ONLYRG,RG=1100,OFFMETERINGTYPE=VOLUME;` |
| 在线 ONLINE | `USAGERPTMODE=ONLINE` + `ONLCOMPOUNDTYPE` + `ONLINERG`(+可选 `ONLINESID`) + `ONLMETERINGTYPE` | `ADD URR:URRNAME="urr_online_URL",URRID=2300,USAGERPTMODE=ONLINE,ONLCOMPOUNDTYPE=ONLYRG,ONLINERG=100,ONLMETERINGTYPE=VOLUME;` |
| QoS 保证 | 仅 `URRNAME/URRID/USAGERPTMODE=QOS` | `ADD URR:URRNAME="URR_qos_01",URRID=1005,USAGERPTMODE=QOS;` |
| 监控（策略控制） | `USAGERPTMODE=MONITORINGKEY` + `MONITORINGKEY`（与 PCRF/PCF 一致） | `ADD URR:URRNAME="urr_01",URRID=2000,USAGERPTMODE=MONITORINGKEY,MONITORINGKEY="2001";` |
| 事件计费 | 在线/离线轨道 + `OFFMETERINGTYPE=EVENT`/`ONLMETERINGTYPE=EVENT`（+可选 `EVENTRSUVALUE` 预申请配额） | `ADD URR:URRNAME="urr2",URRID=1002,USAGERPTMODE=OFFLINE,OFFCOMPOUNDTYPE=ONLYRG,RG=102,OFFMETERINGTYPE=EVENT;` |

**统计类型取值**：`VOLUME`(流量) / `TIME`(时长) / `EVENT`(事件) / `EVENT_VOLUME_TIME`(流量+时长+事件) / `FREE`(免费业务)。

**费率标识组成类型**：`ONLYRG`(仅 RG) / `RGSID`(RG+SID) / `ONLYSID`(仅 SID)——按业务套餐决定用 RG 还是 (RG,SID) 作费率标识。

## 决策点（若有）

### 计费方式选择
问：该 URR 走哪条计费轨道？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 离线 OFFLINE | `OFFCOMPOUNDTYPE/OFFMETERINGTYPE/RG(+SID)` 出现 | Ga 接口话单计费 |
| 在线 ONLINE | `ONLCOMPOUNDTYPE/ONLMETERINGTYPE/ONLINERG(+ONLINESID)` 出现 | Gy/N40 接口 OCS/CHF 实时配额 |
| QoS | 仅三参，无计费标识 | 业务触发 QoS 保证 |
| 监控 MONITORINGKEY | `MONITORINGKEY` 出现 | 基于业务累计流量的策略控制 |

### 费率标识粒度
问：用 RG 还是 (RG,SID) 作为费率标识？
- `ONLYRG`：套餐按 RG 区分即可（证据中融合计费/事件计费普遍采用）。
- `RGSID`：需在同 RG 内用 SID 进一步区分业务（证据中 GGSN/PGW-C 经典离线/在线计费采用）。

### 在线配额申请方式（仅 ONLINE）
问：OCS/CHF 不下发配额数值时，是否本地预申请？
- OCS 下发配额 → 本地不配 `TIMERSUVALUE/VOLUMERSUVALUE`，OCS 优先级高。
- OCS 不支持 → 本地按 `ONLMETERINGTYPE=VOLUME/TIME` 配 `VOLUMERSUVALUE/TIMERSUVALUE` 预申请。
- 事件计费 → `EVENTRSUVALUE` 预申请事件配额（N40 接口通过 `serviceSpecificUnits` 携带；标准 ECUR 忽略此值）。

## 约束（若有）
- URRID 在所有 URR 内全网唯一、建议从 1000 起始（0~2147483646）。`severity=critical` 后果：URRID 重复导致计费规则冲突。
- MONITORINGKEY 在所有 URR 内唯一（4294967295 是无效值）。`severity=critical`
- `USAGERPTMODE=OFFLINE` 时 `(OFFCOMPOUNDTYPE, RG, SID)` 组合唯一；`=ONLINE` 时 `(ONLCOMPOUNDTYPE, ONLINERG, ONLINESID)` 组合唯一。`severity=critical`
- URRID/USAGERPTMODE/OFFMETERINGTYPE/ONLMETERINGTYPE 在 SMF 与 UPF（或 PGW-C/PGW-U）配置必须完全一致。`severity=critical` 后果：跨网元计费不一致、用量上报失败。
- 同时配 UPSID、DOWNSID 且不配离线 RG 时，离线话单 RG 取 DOWNSID 值。`severity=medium`
- 最大记录数 65000。`severity=medium` 后果：超规格新增失败。
- 执行后立即生效。`severity=low`
