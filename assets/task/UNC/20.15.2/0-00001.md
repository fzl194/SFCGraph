# 配置URR（ADD URR）

> 命令静态知识见 [ADD URR](command/UNC/20.15.2/ADD-URR.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

URR 是计费三件套的"费率定义层"：把"哪种业务 × 哪种计费方式 × 哪种统计口径"用一条记录描述，供下游 URRGROUP→PCCPOLICYGRP 引用。本命令的核心配置空间有 3 个维度：**上报模式**（必选，决定后续统计/费率参数是否出现）、**统计类型**（必选，按计费模式分支）、**费率标识组成**（条件可选，按上报模式分支）。每个特性选定一组取值后即得一条 URR；同一业务常以离线+在线成对出现，再被 URRGROUP 聚合成费率组。

### 配置维度 1：上报模式（参数 USAGERPTMODE，必选）

| 取值 | 作用 | 联动必选参数 | 联动可选参数 | 典型场景 |
|---|---|---|---|---|
| **OFFLINE** | 离线计费，话单上报 CG（Ga 接口） | `OFFMETERINGTYPE` | `OFFCOMPOUNDTYPE` + `RG`(+`SID`) | Ga 离线、内容计费（VOLUME/EVENT_VOLUME_TIME） |
| **ONLINE** | 在线计费，实时配额申请（Gy/N40 接口） | `ONLMETERINGTYPE` | `ONLCOMPOUNDTYPE` + `ONLINERG`(+`ONLINESID`) | Gy 在线、N40 融合（VOLUME/EVENT） |
| **MONITORINGKEY** | 流量累计监控（不产生费用，仅作 Monitoring Key 累计） | `MONITORINGKEY` | — | FUP 流量累计策略控制 |
| **QOS** | QoS 保证标记（不产生费用，仅触发 QoS 控制） | — | — | 业务触发 QoS |

> 4 种上报模式互斥；同一业务通常按"业务 × 计费模式"成对配 2 条 URR（离线一条 + 在线一条）。**CONVERGED 不是 URR 自身的属性**，而是 URRGROUP 内 UP/DOWNURRNAME1（离线）+ UP/DOWNURRNAME2（在线）共存的组装方式——见 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md)。

### 配置维度 2：统计类型（参数 OFFMETERINGTYPE / ONLMETERINGTYPE）

**两参数完全对称的取值集**，按上报模式分支出现：

| 取值 | 统计对象 | 适用上报模式 | 备注 |
|---|---|---|---|
| **VOLUME** | 按流量（默认） | OFFLINE / ONLINE | 默认值；最常用 |
| **TIME** | 按时长 | OFFLINE / ONLINE | 时长计费 |
| **EVENT** | 按事件 | OFFLINE / ONLINE | 防欺诈判断场景配 `FREE` 标志 |
| **FREE** | 免费标记 | OFFLINE / ONLINE | 计费防欺诈判断（不实际计费） |
| **VOLUME_TIME** | 流量 + 时长 | OFFLINE / ONLINE | 多维度累计 |
| **EVENT_VOLUME** | 事件 + 流量 | OFFLINE / ONLINE | 多维度累计 |
| **EVENT_TIME** | 事件 + 时长 | OFFLINE / ONLINE | 多维度累计 |
| **EVENT_VOLUME_TIME** | 事件 + 流量 + 时长 | OFFLINE / ONLINE | 内容计费多维累计（典型） |

> 选含 EVENT 的枚举前评估性能影响（命令 notes 明确提示）。OFFLINE 默认 VOLUME、ONLINE 默认 VOLUME；不配置时按默认值生效。

### 配置维度 3：费率标识组成（参数 OFFCOMPOUNDTYPE/ONLCOMPOUNDTYPE + RG/SID，条件可选）

**RG 与 SID 配对与否**取决于计费场景对"费率组 vs 业务标识"二级的需要：

| 参数 | 取值 | 作用 | 适用场景 |
|---|---|---|---|
| `OFFCOMPOUNDTYPE`（OFFLINE） | `ONLYRG` | 仅用 RG 作为费率标识（一级） | 融合计费/事件计费普遍采用 |
| `OFFCOMPOUNDTYPE`（OFFLINE） | `RGSID` | RG + SID 二级费率标识 | GGSN/PGW-C 经典离线/在线计费 |
| `OFFCOMPOUNDTYPE`（OFFLINE） | `ONLYSID` / `UPDOWNSID` | 仅 SID 或上下行 SID | 融合计费场景不支持（命令 notes 明确） |
| `ONLCOMPOUNDTYPE`（ONLINE） | `ONLYRG` | 仅用 ONLINERG 作为费率标识 | 标准在线计费 |
| `ONLCOMPOUNDTYPE`（ONLINE） | `RGSID` | ONLINERG + ONLINESID 二级费率标识 | 需在同 RG 内用 SID 细分业务 |

| 参数 | 范围 | 约束 |
|---|---|---|
| `RG` / `ONLINERG` | 整数 0~4294967294 | 4294967295=无效值；典型多档位：10/20/100/1100 |
| `SID` / `ONLINESID` | 整数 0~4294967294 | 4294967295=无效值 |

> RG/SID 在 SMF/UPF（或 PGW-C/PGW-U）跨网元必须一致——见约束 rule-0-01001。

**步骤位置**：在所属 feature 编排中通常位于"步骤 1：配置 URR"——紧随 License 前置之后，URRGROUP 绑定之前。

## 决策点：计费方式选择（DP 0-01001）

问：该 URR 走哪条计费轨道？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 离线 OFFLINE | `USAGERPTMODE=OFFLINE` + `OFFMETERINGTYPE` + `OFFCOMPOUNDTYPE/RG/SID` | Ga 接口话单计费、内容计费、融合计费（与 ONLINE 配对） |
| 在线 ONLINE | `USAGERPTMODE=ONLINE` + `ONLMETERINGTYPE` + `ONLCOMPOUNDTYPE/ONLINERG/ONLINESID` | Gy 在线、N40 融合、累计流量策略 |
| 监控 MONITORINGKEY | `USAGERPTMODE=MONITORINGKEY` + `MONITORINGKEY` | 基于业务累计流量的策略控制（FUP） |
| QoS 保证 | 仅 `USAGERPTMODE=QOS`（极简三参） | 业务触发 QoS 保证 |

### 费率标识粒度（DP 0-01002）

问：用 RG 还是 (RG, SID) 作为费率标识？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅 RG（`ONLYRG`） | `OFFCOMPOUNDTYPE=ONLYRG` 或 `ONLCOMPOUNDTYPE=ONLYRG` | 套餐按 RG 区分即可；融合计费/事件计费普遍采用 |
| RG + SID（`RGSID`） | `OFFCOMPOUNDTYPE=RGSID` 或 `ONLCOMPOUNDTYPE=RGSID` | 需在同 RG 内用 SID 进一步区分业务；GGSN/PGW-C 经典离线/在线计费采用 |

> 融合计费场景要求 URRGROUP 下绑定的在线和离线 URR 中的 RG 和 SID **完全一致**，否则离线话单无法正确关联 PS-Furnish-Charging-Information 字段（命令 notes）。

### 在线配额申请方式（DP 0-01003，仅 ONLINE）

问：OCS/CHF 不下发配额数值时，是否本地预申请？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| OCS 下发配额 → 本地不配 | 不配 `TIMERSUVALUE`/`VOLUMERSUVALUE` | OCS 优先级高，正常流程 |
| OCS 不支持 → 本地预申请 | `ONLMETERINGTYPE=VOLUME/TIME` + `VOLUMERSUVALUE`/`TIMERSUVALUE` | 配额耗尽前本地放通 |
| 事件计费预申请 | `EVENTRSUVALUE` 配事件配额 | N40 接口通过 `serviceSpecificUnits` 携带 |

## 约束

- **URR 关键参数跨网元一致**（critical，rule-0-01001）：URRID / USAGERPTMODE / OFFMETERINGTYPE / ONLMETERINGTYPE 在 SMF 与 UPF（或 PGW-C/PGW-U）必须保持一致 — 后果：跨网元计费不一致、用量上报失败
- **URRID 全网唯一且从 1000 起始分配**（critical，rule-0-01002）：URRID 在所有 URR 内唯一（取值 0~2147483646，从 1000 起始） — 后果：URRID 重复导致计费规则冲突/统计错配
- **MONITORINGKEY 全网唯一**（critical，rule-0-01003）：4294967295 是无效值 — 后果：监控策略失效
- **USAGERPTMODE=OFFLINE 时组合唯一**（critical，rule-0-01004）：`(OFFCOMPOUNDTYPE, RG, SID)` 组合唯一；`=ONLINE` 时 `(ONLCOMPOUNDTYPE, ONLINERG, ONLINESID)` 组合唯一 — 后果：话单路由错乱
- **ADD URR 最大记录数规格**（warning，rule-0-01005）：单板最大 65000 — 后果：超规格新增失败
- **同时配 UPSID/DOWNSID 且不配离线 RG 时**（info，rule-0-01006）：离线话单 RG 取 DOWNSID 值 — 后果：话单费率与预期不符
