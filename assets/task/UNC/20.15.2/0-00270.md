# 设置基于APN的地址分配属性（SET APNADDRESSATTR）

> 命令静态知识见 [SET APNADDRESSATTR](command/UNC/20.15.2/SET-APNADDRESSATTR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令以 APN 为粒度控制地址分配属性，是 APN 级地址策略的总开关。配置空间核心维度为 **IP 分配方式**（IPV4ALLOCTYPE/IPV6ALLOCTYPE）、**双栈支持组合**（SUPPORTIPV4/SUPPORTIPV6/SUPPORTPRIOR/IPTYPEFORDUALIP）和 **UE 地址分配主体**（ALLOCPRECEDENCE）。

### 配置维度1：IP 分配方式（参数 IPV4ALLOCTYPE / IPV6ALLOCTYPE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| LOCAL | 通过 5GC（SMF/UPF）分配地址（默认） | 5GC 本地地址池（多数场景默认） |
| RADIUS | 通过外部 AAA 服务器分配地址 | 外部 AAA 静态地址（需配套 `SET APNRADIUSATTR`/`ADD RDSSVRGRP`） |
| DHCP | 通过外部 DHCP 服务器分配地址 | 外部 DHCP 静态地址 |

> 可配套 IPV4RADIUSPRIOR/IPV6RADIUSPRIOR=ENABLE：在 LOCAL/DHCP 方式下优先使用 AAA 签约静态地址。

### 配置维度2：双栈支持组合（参数 SUPPORTIPV4 / SUPPORTIPV6 / SUPPORTPRIOR / IPTYPEFORDUALIP）

| 取值组合 | 作用 | 典型场景 |
|---|---|---|
| 双栈 ENABLE + SUPPORTPRIOR + IPV4V6（默认） | 双栈用户配 IPv4+IPv6 | 标准 5GC 双栈 |
| 仅 IPv4 | SUPPORTIPV4=ENABLE, SUPPORTIPV6=DISABLE | 纯 IPv4 APN |
| 仅 IPv6 | SUPPORTIPV6=ENABLE, SUPPORTIPV4=DISABLE | 纯 IPv6 APN（WSFD-104004 IPv6 前缀代理配 IPV6ALLOCTYPE=LOCAL） |

> 仅设 SUPPORTIPV4=DISABLE 会自动把 IPTYPEFORDUALIP 改为 IPV6；仅设 SUPPORTIPV6=DISABLE 会自动把 SUPPORTPRIOR/IPTYPEFORDUALIP 改为 IPV4。SUPPORTPRIOR/IPTYPEFORDUALIP 必须与对应 SUPPORT 开关自洽。

### 配置维度3：UE 地址分配主体（参数 ALLOCPRECEDENCE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| INHERIT（默认） | 继承全局 `SET ADDRESSATTR（★R1.5 atom 待补）` 配置 | 默认 |
| SMF_ALLOC | UE 地址由 SMF 分配 | SMF 集中分配 |
| UPF_FIRST | UE 地址优先由 UPF 分配（UPF 未分则 SMF 分） | UPF 本地分配 |

### 配置维度4：APN 级可选开关（节选）

| 参数 | 作用 | 默认 |
|---|---|---|
| FRAMEDROUTE | 手机后路由 | DISABLE（WSFD-205101 Routing Behind MS：ENABLE） |
| IGNORECONFLICT | 忽略静态地址冲突 | ENABLE |
| STATICIP | 静态地址接入 | ENABLE |
| IPV6RAINTV/IPV6RAOFLAG/IPV6RALIFETIME/IPV6RATIMES | IPv6 RA 消息参数 | 14400/ENABLE/65535/0 |
| IPV6RAMTUSW/IPV6MTU | IPv6 RA MTU 选项 | DISABLE/1800 |
| RDSSVRCTRL | Radius AAA 控制单双栈 | DISABLE |
| CONFLICTSTG | 地址冲突去激活策略 | INHERIT |

**步骤位置**：在地址分配族编排中，于 `ADD APN`（建 APN）之后执行——`ADD APN` 会自动为本命令创建一条默认记录，本命令覆盖默认值。被 `ADD UPNODE.ADDRALLOCMODE=INHERIT`（UPF 节点地址分配）作为全局引用源。

## 决策点

**IP 分配方式 + 双栈组合**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| LOCAL + 双栈 | IPV4/IPV6ALLOCTYPE=LOCAL + 双栈 ENABLE | 5GC 本地双栈（默认） |
| LOCAL + 仅 IPv6 | IPV6ALLOCTYPE=LOCAL + SUPPORTIPV6=ENABLE, SUPPORTIPV4=DISABLE | IPv6 前缀代理（WSFD-104004） |
| RADIUS + 双栈 | IPV4/IPV6ALLOCTYPE=RADIUS + 配套 RADIUS 服务器组 | 外部 AAA 静态地址 |
| DHCP | IPV4/IPV6ALLOCTYPE=DHCP | 外部 DHCP |
| + FRAMEDROUTE=ENABLE | + FRAMEDROUTE | Routing Behind MS（WSFD-205101） |
| + ALLOCPRECEDENCE | + SMF_ALLOC/UPF_FIRST | 指 SMF 或 UPF 分配 |

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无回滚窗口
- **APN 必须先建**（critical）：必须先 `ADD APN` 配置 APN；`ADD APN` 时自动创建一条默认记录 — 后果：APN 不存在则配置失败
- **双栈自洽**（critical）：SUPPORTPRIOR=IPv4 则 SUPPORTIPV4 必须 ENABLE；IPTYPEFORDUALIP=IPv4V6 则两者都 ENABLE；仅设 SUPPORTIPV4=DISABLE 会自动改 IPTYPEFORDUALIP=IPv6 — 后果：配置被自动调整或失败
- **当前版本不支持部分参数**（warning）：当前版本不支持 ANTISPOOFINGDI/ANTISPOOFINGUI/RADIUSIPROUTE/DHCPIPROUTE/STATICIPROUTE/IPV4ALLOCPRIALG/IPV6ALLOCPRIALG — 后果：配置这些参数无效
- **ACCESSMODE 与 RADIUS 分配互斥**（critical）：APNAUTHATTR 中 ACCESSMODE=LOC_AUTH/TRANS_NON_AUTH 时，IPVxALLOCTYPE=RADIUS 失效，按 ALLOCPRECEDENCE 判断 SMF/UPF 分配 — 后果：RADIUS 分配不生效
- **虚拟 APN 软参**（warning）：虚拟 APN 且本命令 SUPPORTIPV4/SUPPORTIPV6/IPTYPEFORDUALIP 非默认值时，需软参 DWORD1044 BIT24=1 才对 5G 用户生效 — 后果：未设软参则不生效
- **RDSSVRCTRL 自恢复**（info）：IPTYPEFORDUALIP≠IPv4v6 时 RDSSVRCTRL 自动恢复 DISABLE — 后果：非双栈下该开关无意义
- **IPV6RAINTV 范围**（warning）：整数 1800~65535 秒 — 后果：超范围配置失败
