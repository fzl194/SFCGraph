# UNC 控制面地址分配规则

> **UNC 域控制面地址分配规则骨架**（SOP §3.5 域通用）。被引用于 **WSFD-010504** + **WSFD-104413** + **WSFD-104005** 3 个特性。
>
> **触发条件**：当 SMF/SGW-C/PGW-C 对接的 UPF 不支持 IP 地址分配（WSFD-010504），或当 DHCP 子方式（WSFD-104413/104005）需要按自定义规则/位置区/冲突管理时启用。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 设置全局地址分配规则（按需）| `SET IPALLOCRULE` | IPRULE=全网规划（按业务类型分配规则）|
| 2. 设置基于位置区地址分配全局开关（按需）| `SET IPALLOCBYLOCGLBSW` | IPALLOCBYLOCSWITCH=ENABLE/DISABLE |
| 3. 增加冲突地址（按需，处理地址冲突）| `ADD CONFLICTIP` | IPADDR=冲突 IP |

**典型脚本（WSFD-010504 默认全局规则）**：
```
SET IPALLOCRULE: IPRULE=全网规划分配规则;
SET IPALLOCBYLOCGLBSW: IPALLOCBYLOCSWITCH=ENABLE;
ADD CONFLICTIP: IPADDR="10.1.1.200";
```

**步骤位置**：在 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) SMF 地址池体系（[UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施之后）之后；DHCP 子方式时与 [UNC 1-00028](task/UNC/20.15.2/1-00028.md) DHCP 服务器组链路并列。

## 场景差异

- **[WSFD-010504](task/UNC/20.15.2/2-00016.md) 控制面地址分配方式**：步骤 1-3 按需（默认全局规则不执行；位置区分配启用步骤 2；冲突地址管理启用步骤 3）
- **[WSFD-104413](task/UNC/20.15.2/2-00021.md) DHCPv4 功能**：可选启用步骤 1-3（DHCP 子方式下，可配置自定义分配规则/位置区/冲突地址）
- **[WSFD-104005](task/UNC/20.15.2/2-00022.md) DHCPv6 地址分配**：同 WSFD-104413

## 决策点

| 选项 | 影响 |
|---|---|
| 默认全局规则 | 不执行步骤 1（由系统默认规则）|
| 基于位置区分配 | 步骤 2 开启（位置区→地址池映射）|
| 自定义全局分配规则 | 步骤 1（按业务类型）|
| 含冲突地址管理 | 步骤 3 |

## 约束

- **IPV4ALLOCTYPE=Local 强制**（critical，WSFD-010504 场景）：本骨架场景下 `SET APNADDRESSATTR` 的 `IPV4ALLOCTYPE` 必须为 `Local`（控制面本地分配）—— 与 UPF 分配路径互斥
- **位置区→地址池映射配套**（warning）：`SET IPALLOCBYLOCGLBSW` 开启后需保证位置区→地址池映射已配 — 后果：位置区未映射时降级为默认分配
- **冲突地址管理需配合告警**（info）：`ADD CONFLICTIP` 配置后产生冲突时上报 `ALM-100297`（地址池组使用率超阈值）
