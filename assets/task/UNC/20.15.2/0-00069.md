# ADD RGTRIGGER 原子任务
> 命令静态知识见 [ADD RGTRIGGER](command/UNC/20.15.2/ADD-RGTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令把一组 RG 级计费 Trigger 的上报时机挂到一个融合计费模板（CCTMPLTNAME，来自 [ADD CCT](task/UNC/20.15.2/0-00009.md)）。每个 Trigger 参数描述一类事件（QoS 更新、用户位置更新、配额阈值到达等）发生时的上报策略：IMMEDIATE（立即上报）、DEFERRED（延迟上报）、NONREPORT（不上报）。系统部署后已存在初始记录（含各 Trigger 默认值），本命令按需覆盖。CHF 下发的 Trigger 优先级更高；可用 [SET CNVRGDCHGPARA](task/UNC/20.15.2/0-00165.md) 的 IGNORETRIGGER 忽略 CHF 下发的指定 Trigger。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTMPLTNAME | 融合计费模板名称，从已配置 CCT 数据获取，1~63 字符，不区分大小写 | `cct-test` / `global` |
| QOSCHG | QoS 更新，本端规划 | `IMMEDIATE` / `DEFERRED` / `NONREPORT` |
| ULCHG | 用户位置更新，本端规划 | `NONREPORT` / `IMMEDIATE` |
| SRVNDCHG | 服务节点更新，本端规划 | `DEFERRED` / `IMMEDIATE` / `NONREPORT` |
| PRACHG | 区域用户位置上报更新，本端规划 | `IMMEDIATE` / `DEFERRED` / `NONREPORT` |
| PSDATAOFFCHG | PS 数据关闭状态更新，本端规划 | `IMMEDIATE` / `DEFERRED` / `NONREPORT` |
| UETZCHG | 用户时区更新，本端规划 | `IMMEDIATE`（默认） |
| PLMNCHG | PLMN 更新，本端规划 | `IMMEDIATE`（默认） |
| RATCHG | RAT 更新，本端规划 | `IMMEDIATE`（默认） |
| TIMELIMIT | 时间阈值，本端规划 | `DEFERRED` / `IMMEDIATE` / `NONREPORT` |
| VOLUMELIMIT | 流量阈值，本端规划 | `DEFERRED` / `IMMEDIATE` / `NONREPORT` |
| EVENTLIMIT | 事件阈值，本端规划 | `DEFERRED` / `IMMEDIATE` / `NONREPORT` |
| QUOTATHRESHOLD | 配额阈值，本端规划 | `IMMEDIATE`（默认） |
| VT | 配额有效时长，本端规划 | `IMMEDIATE`（默认） |
| QHT | 配额保持时长，本端规划 | `IMMEDIATE`（默认） |
| SERVPLMNRTCTCHG | 服务 PLMN 速率控制更新，本端规划 | `DEFERRED` |
| APNRATECTRLCHG | APN 速率控制更新，本端规划 | `DEFERRED` |
| TAICHG / CGISAICHG / RAICHG | 跟踪区/CGISAI/RAI 标识更新，本端规划（默认 NONREPORT） | `NONREPORT` |

典型脚本（覆盖部分 RG 级 Trigger）：
```
ADD RGTRIGGER:CCTMPLTNAME="cct_test", ULCHG=NONREPORT, PRACHG=IMMEDIATE, PSDATAOFFCHG=IMMEDIATE;
```

## 决策点

### 各 Trigger 事件的上报时机
- 问什么：每一类 RG 级计费事件发生时，是立即上报、延迟上报还是不上报。
| 取值 | 含义 | 容器生成 / CHF 上报 |
|---|---|---|
| IMMEDIATE | 立即上报 | 事件发生立即生成容器上报 CHF |
| DEFERRED | 延迟上报 | 事件发生不立即上报，等待合并时机 |
| NONREPORT | 不上报 | 事件发生不生成容器，不上报 CHF |

### 是否忽略 CHF 下发的指定 Trigger（联动 SET CNVRGDCHGPARA）
- 问什么：当 CHF 在响应消息中下发 Trigger 列表时，是否用本命令本地配置覆盖部分 CHF Trigger。
| IGNORETRIGGER（SET CNVRGDCHGPARA）| 本命令角色 |
|---|---|
| 未忽略 | CHF 下发的 Trigger 优先级更高，本命令本地值仅在 CHF 未下发对应 Trigger 时生效 |
| 配置忽略（如 RG 级时间/流量阈值等） | 被忽略的 Trigger 采用本命令本地配置值 |

## 约束
- 生效时机：命令执行后**只对新激活用户生效**（已激活用户不重配）。`severity=high`
- 记录上限：每个 CCT 模板维度最多可输入 **101** 条记录。`severity=medium`
- 初始记录：系统部署完成后已存在初始记录（含各 Trigger 默认值，QOSCHG/ULCHG/SRVNDCHG/PRACHG/PSDATAOFFCHG/TIMELIMIT/VOLUMELIMIT/EVENTLIMIT 默认 DEFERRED，UETZCHG/PLMNCHG/RATCHG/QUOTATHRESHOLD/VT/QHT 默认 IMMEDIATE，TAICHG/CGISAICHG/RAICHG 默认 NONREPORT），本命令按需覆盖。`severity=low`
- 优先级规则：Session 级与 RG 级同时生效的 Trigger 冲突时，**Session 级 Trigger 规则优先**；CHF 下发的 Trigger 优先级高于本命令本地配置。`severity=high`
- 前置校验：CCTMPLTNAME 须已由 [ADD CCT](task/UNC/20.15.2/0-00009.md) 配置。`severity=high`
