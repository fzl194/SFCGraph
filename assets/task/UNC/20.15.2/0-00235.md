# 增加逻辑接口（ADD INTERFACE）

> 命令静态知识见 [ADD INTERFACE](command/UNC/20.15.2/ADD-INTERFACE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令创建逻辑接口（LoopBack、Tunnel、NULL0 等）并初始化其配置。配置空间核心维度为 **接口类型**（IFNAME 承载：LoopBack0/1、Tunnel1/2/10/17/27）和 **管理状态/链路参数**（IFADMINSTATUS / IFMTU / IFDF / IFSTATIENABLE / IFSTATITVL）。

### 配置维度1：接口类型（参数 IFNAME）

| 接口类型 | 作用 | 典型配置 | 典型场景 |
|---|---|---|---|
| LoopBack0/1 | 创建本端 LoopBack 接口（业务/管理/对接固定地址） | 仅 IFNAME 或 + IFADMINSTATUS=up + IFMTU=1500 | GRE 源（IPFD-015002 LoopBack1）、IPSec 本端（IPFD-016000 LoopBack1）、RADIUS NEXTHOP（WSFD-011306 LoopBack0） |
| Tunnel1/2/10/17/27 | 创建隧道接口（GRE/IPSec 隧道） | + IFADMINSTATUS=up + IFMTU=1500 + IFDF=FALSE（+ IFSTATIENABLE/IFSTATITVL） | GRE 隧道、IPv4/IPv6 IPSec 隧道（IPFD-016000） |
| NULL0 | 创建 NULL 接口（黑洞） | 仅 IFNAME | 黑洞路由场景 |

### 配置维度2：管理状态与链路参数（IFADMINSTATUS / IFMTU / IFDF / IFSTATIENABLE / IFSTATITVL）

| 取值组合 | 作用 | 典型场景 |
|---|---|---|
| IFADMINSTATUS=up + IFMTU=1500 + IFDF=FALSE | 隧道接口标准三件套：管理 Up、MTU 1500、允许分片 | IPSec 隧道（IPFD-016000 Tunnel 系列） |
| 上述 + IFSTATIENABLE=TRUE + IFSTATITVL=20/5 | 启用统计并按周期上报 | 需观测隧道流量 |
| 仅 IFNAME（其他默认） | 创建接口使用默认参数 | LoopBack 简单场景（IPFD-015002/WSFD-011306） |

**步骤位置**：在地址分配/接口管理族编排中，是 `ADD IFIPV4ADDRESS`/`ADD IFIPV6ADDRESS`/`SET IFIPV6ENABLE`/`ADD IPBINDVPN` 的**前置必建步骤**——逻辑接口必须先 `ADD INTERFACE` 才能配 IP。

## 决策点

**接口类型选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| LoopBack | 默认管理 Up | 业务/对接固定地址（GRE/IPSec/RADIUS 源） |
| Tunnel | + IFADMINSTATUS=up + IFMTU=1500 + IFDF=FALSE | GRE/IPSec 隧道接口 |
| NULL0 | 默认 | 黑洞路由 |

> 实战中 LoopBack 常仅填 IFNAME（默认状态即可）；Tunnel 多数要显式 IFADMINSTATUS=up 并配 MTU/DF。

## 约束

- **IFNAME 命名约束**（warning）：字符串 1~63，接口名称由接口类型+编号组成（如 LoopBack1、Tunnel27），不支持空格 — 后果：违规命名新增失败
- **IFNAME 全局唯一**（critical）：接口名全局唯一 — 后果：重名新增失败
- **IFMTU 范围**（warning）：取值由设备规格决定 — 后果：超规格配置失败
- **IFSTATITVL 范围**（warning）：整数 5~600，单位秒 — 后果：超范围配置失败
- **管理状态默认**（info）：IFADMINSTATUS 缺省按系统默认（一般 down），需 Up 时必须显式配置 — 后果：忘配导致接口不可用
