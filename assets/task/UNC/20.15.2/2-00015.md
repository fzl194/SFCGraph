# 地址分配方式（WSFD-010502）

> 特性静态知识见 [WSFD-010502](feature/UNC/20.15.2/WSFD-010502.md+ [特性知识](业务图谱体系/APN业务域/feature-knowledge/WSFD-010502-地址分配方式.md（★业务图谱未建，归 R1.5 后续）。本页讲配置生成实例化时这个特性**怎么配**——编排哪些步骤、4 子方式如何选。

## 配置概览

核心对象链：**ADDRPOOL**（POOLTYPE=Local/UDM/Radius/DHCP）→ `SECTION`（地址段）→ `ADDRPOOLGRP`（池组）→ `ADDRUPGROUP`+`UPNODE`+`UPFBINDGRP`（UPF 组绑定）→ `POOLGRPMAP`（池组→UPF 组映射）→ `SET APNADDRESSATTR`（使能 IPV4ALLOCTYPE）。

4 子方式**共享接入域骨架**（[UNC 1-00024](task/UNC/20.15.2/1-00024.md)）+ **地址池体系**（[UNC 1-00023](task/UNC/20.15.2/1-00023.md)），差异在 POOLTYPE + 配套特性（DHCP 子方式必走 [WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md)/[WSFD-104005](feature/UNC/20.15.2/WSFD-104005.md)；Radius 子方式必走 [WSFD-011306](feature/UNC/20.15.2/WSFD-011306.md)）。由特性级 DP（地址分配子方式）驱动。

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（L3VPN + VPN 实例 + APN + 使能地址分配）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` + `SET APNADDRESSATTR` → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **地址池体系**（按子方式建池/段/组/映射/UPF 绑定）：`ADD ADDRPOOL` + `ADD SECTION` + `ADD ADDRPOOLGRP` + `ADD POOLBINDGRP` + `ADD ADDRUPGROUP` + `ADD UPNODE` + `ADD UPFBINDGRP` + `ADD POOLGRPMAP` → [UNC 1-00023](task/UNC/20.15.2/1-00023.md)
   - **UDM 子方式**：POOLTYPE=UDM, CHECKIPVALID=Enable
   - **Radius 子方式**：POOLTYPE=Radius, CHECKIPVALID=Enable
   - **Local 子方式**（默认）：POOLTYPE=Local
   - **DHCP 子方式**：POOLTYPE=DHCP（DHCP 服务器组链路必配套，见 DP 表）
3. **下行路由发布**（让骨干网可达分配的地址）：`ADD OSPF` + `ADD OSPFAREA` + `ADD OSPFINTERFACE` + `ADD OSPFIMPORTROUTE` → [UNC 1-00029](task/UNC/20.15.2/1-00029.md)
   - 关键参数：PROCID, AREAID, IFNAME（UNC 真实命令名 **OSPFINTERFACE**——v1.0 写的 `OSPFNETWORK` 是错的，UNC 无此命令）
   - IPv6/双栈需叠加 [1-00032](task/UNC/20.15.2/1-00032.md) UNC OSPFv3 路由发布（OSPFV3/OSPFV3AREA/OSPFV3INTERFACE/OSPFV3IMPORTROUTE，**OSPFV3 全大写**，非小驼峰）
4. **（按需）静态地址黑名单**（防非法地址接入）：`ADD BLACKLIST` → [0-00224](task/UNC/20.15.2/0-00224.md)

> **★ 子方式联动配置（按 DP 选）**：
> - **DHCP 子方式**：本特性仅建 POOLTYPE=DHCP 地址池；DHCP 服务器组链路（`ADD DHCPSERVERGRP`+`ADD DHCPSERVER`+`ADD DHCPBINDPOOLGRP`）**必须**配套 [WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md)（DHCPv4）/ [WSFD-104005](feature/UNC/20.15.2/WSFD-104005.md)（DHCPv6）特性——本特性不内含。
> - **Radius 子方式**：必配套 [WSFD-011306](feature/UNC/20.15.2/WSFD-011306.md)（Radius 功能）+ [WSFD-011305](feature/UNC/20.15.2/WSFD-011305.md)（Radius 鉴权）。
> - **UDM 子方式**：必配套 [WSFD-010400](feature/UNC/20.15.2/WSFD-010400.md)（UDM 签约）。

## 决策点（特性级：地址分配子方式选择）

| 子方式 × IP 类型 × CHECKIPVALID | 走法（复用 compound）| 联动特性 |
|---|---|---|
| **UDM × IPv4 × Enable** | 骨架 [1-00024](task/UNC/20.15.2/1-00024.md) + 地址池 [1-00023](task/UNC/20.15.2/1-00023.md) + 路由 [1-00029](task/UNC/20.15.2/1-00029.md) | [WSFD-010400](feature/UNC/20.15.2/WSFD-010400.md)（UDM 签约）|
| **Radius × IPv4 × Enable** | 骨架 1-00024 + 地址池 1-00023 + 路由 1-00029 | [WSFD-011305](feature/UNC/20.15.2/WSFD-011305.md) + [WSFD-011306](feature/UNC/20.15.2/WSFD-011306.md) |
| **Local × IPv4 × Disable**（默认）| 骨架 1-00024 + 地址池 1-00023 + 路由 1-00029 | 无（普通移动用户，最常用）|
| **DHCP × IPv4 × Disable** | 骨架 1-00024 + 地址池 1-00023（POOLTYPE=DHCP）+ 路由 1-00029 | [WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md)（DHCPv4 服务器组链路）|
| **DHCPv6 × IPv6 × Disable** | 骨架 1-00024（SUPPORTIPV6）+ 地址池 1-00023（IPv6 SECTION）+ 路由 1-00029 | [WSFD-104005](feature/UNC/20.15.2/WSFD-104005.md)（DHCPv6 服务器组链路）|

> **走法列**：4 子方式**共享 3 骨架 compound**（[1-00024](task/UNC/20.15.2/1-00024.md)/[1-00023](task/UNC/20.15.2/1-00023.md)/[1-00029](task/UNC/20.15.2/1-00029.md)）；差异仅 POOLTYPE 参数 + 联动特性。

## 约束（特性级 rule）

- **POOLTYPE 与 CHECKIPVALID 联动**（critical）：UDM/Radius 子方式**必配** CHECKIPVALID=Enable（白名单保护 APN 静态地址），否则非法地址用户可激活 — 后果：白名单失效，非法接入。
- **UPF 节点须 PNFPROFILE 先建**（critical）：`ADD UPNODE` 时 NFINSTANCENAME 必须是已 `ADD PNFPROFILE` 注册的 UPF 实例 — 后果：UPF 节点配置失败，静态地址网段无法绑定 UPF。
- **地址段必配**（critical）：本地地址池建好后**必须** `ADD SECTION` 添加地址段，否则用户激活失败。
- **POOLNAME 命名约束**（warning）：字符串 1~79，由 `-/_`/数字/字母/`.` 组成，不能以 `.` 开头 — 后果：违规命名新增失败。
- **最大记录数 80000**（warning）：每 ADDRPOOL 最多 80000 记录 — 后果：超规格新增失败。
- **DHCP 服务器可达性**（critical）：DHCP 子方式下 UNC 通过代理 IP 与外部 DHCP Server 通信，需保证 `ALM-100466` 不告警 — 后果：用户激活失败。
- **DHCP 子方式必走配套特性**（critical）：本特性仅建 `POOLTYPE=DHCP` 的地址池；DHCP 服务器组链路（`ADD DHCPSERVERGRP`+`ADD DHCPSERVER`+`ADD DHCPBINDPOOLGRP`）必须配套 [WSFD-104413](feature/UNC/20.15.2/WSFD-104413.md) / [WSFD-104005](feature/UNC/20.15.2/WSFD-104005.md) 特性。
- **静态 IP 网段与 SMF 主锚点冲突**（warning）：当静态 IP 网段绑定的 UPF 与 SMF 选择的**主锚点 UPF**不一致时，优先用 SMF 选择的主锚点；不一致场景下 `SET STATICADDRPARA`（UNC pre-build 阶段，SET STATICADDRPARA 的 atom 待补，编号未分配）开启按时间扫描会话去活 — 后果：旧会话持续锚到错误 UPF。
- **首次发布版本**（info）：本特性首次发布版本为 UNC **20.0.0**；UDM 子方式绑定 UPF 网段为 20.3.0 性能增强版（POOLGRPTYPE=UDM 的 UPF 绑定）。
