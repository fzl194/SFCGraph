# ADD TSTPCFBINDING 原子任务
> 命令静态知识见 [ADD TSTPCFBINDING](command/UNC/20.15.2/ADD-TSTPCFBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是**拨测专用**的硬绑定：把 (APN, IMSI) 二元组直接钉到某个 PCF 实例，绕过正常 PCF 发现流程。仅用于测试，**拨测完成后必须删除**（`RMV TSTPCFBINDING`）。

## 配置方法
按"被测 APN + 测试 IMSI + 目标 PCF 实例标识"三元组配置。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | 拨测用 APN 名称（1~63 字符，大小写不敏感） | `apn-test` |
| IMSI | 测试用户的 IMSI（6~15 位数字） | `460000123456789` |
| PCFINSTANCEID | 目标 PCF 实例标识（1~50 字符，字母/数字/`-`/`_`） | `testPcfInstanceID` |

典型脚本：
```
ADD TSTPCFBINDING: APN="apn-test",IMSI="460000123456789",PCFINSTANCEID="testPcfInstanceID";
```

## 决策点

### 拨测目标 PCF 的选取
- 问什么：本次拨测要打到哪个 PCF 实例。
- PCFINSTANCEID 取自对端 PCF 实例概述信息中的实例标识，需与被测 PCF 一一对应。
- 错填会拨测到错误的 PCF，结论无效。

## 约束

- **仅对新激活用户生效**：执行后只影响后续激活的用户，不作用于在线用户 · severity=MEDIUM。
- **拨测后必须删除**：`RMV TSTPCFBINDING` 清理，否则残留绑定会干扰正常 PCF 发现 · severity=HIGH。
- **不支持 failover**：绑定的 PCF 故障则用户激活失败，不会切换到其他 PCF · severity=HIGH。
- **记录上限**：最多 100 条 · severity=MEDIUM。
- **IMSI 格式**：6~15 位十进制数字 · severity=LOW。
