# ADD NGMMSUBDATA 原子任务
> 命令静态知识见 [ADD NGMMSUBDATA](command/UNC/20.15.2/ADD-NGMMSUBDATA.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令在 AMF 本地为指定用户（群）配置移动性管理参数，是**一条命令承载多种意图**的多用途命令。不同特性激活时，关注的参数子集完全不同——配置生成的关键在于**按特性意图选取参数子集**，其余参数留默认。

| 特性意图 | 关键参数子集 | 取值样例 |
|---|---|---|
| RFSP 策略下发（基于 SPID 的 UE 驻留） | SUBRANGE、IMSIPRE、RFSPINDEX、RFSPPRI | `SUBRANGE=IMSI_PREFIX, IMSIPRE="30809", RFSPINDEX=1, RFSPPRI=CFG_FIRST` |
| 移动性接入限制（用户接入控制） | SUBRANGE、（IMSI 前缀时）IMSIPRE、RATRESTRICT、CORERESTRICT | `SUBRANGE=ALL_USER, CORERESTRICT=FIFTHGC-1` |
| 本地 RFSP 索引（PCC 基本功能） | SUBRANGE、RATRESTRICT、CORERESTRICT、RFSPINDEX、RFSPPRI | `SUBRANGE=HOME_USER, RATRESTRICT=EUTRA-0&NR-1, CORERESTRICT=FIFTHGC-1, RFSPPRI=CFG_FIRST, RFSPINDEX=5` |
| 网络切片包含模式下发 | SUBRANGE、NIM | `SUBRANGE=HOME_USER, NIM=MODE_A` |
| RedCap RAT 限制（NR_REDCAP 拒绝） | SUBRANGE、RATRESTRICT | `RATRESTRICT` 含 NR_REDCAP |

| 公共参数 | 配置方法 |
|---|---|
| SUBRANGE | 选择用户范围粒度：`ALL_USER`（所有用户）/`HOME_USER`（本网）/`EXTERN_USER`（外网）/`IMSI_PREFIX`（IMSI 前缀号段）。粒度从粗到细优先级递增 |
| IMSIPRE | SUBRANGE=IMSI_PREFIX 时必填，号段 5~15 位数字；其余 SUBRANGE 不填 |
| RFSPINDEX | 本端规划的 RFSP 索引（0~256），0 表示未配置本地 RFSP |
| RFSPPRI | 配置优先 `CFG_FIRST` / 签约优先 `SUB_FIRST`，决定本地与 UDM 签约 RFSP 谁优先 |

## 决策点（若有）

### RFSP 优先级（本地配置 vs UDM 签约）
- 问什么：AMF 本地配置的 RFSP 与 UDM 签约 RFSP 冲突时，谁优先发给 PCF/NG-RAN。
| RFSPPRI 取值 | 行为 | 适用场景 |
|---|---|---|
| `CFG_FIRST` | 无论是否签约，均用本地 RFSP；本地 RFSPINDEX 不能为 0 | 不信赖 UDM 签约（如漫游用户自定义驻留策略） |
| `SUB_FIRST` | 优先用签约 RFSP，未签约才用本地 | 信赖 UDM 签约，本地仅作兜底 |

### 用户范围匹配优先级
- 问什么：多条记录用户群重叠时，AMF 按号段最长匹配为准。
- 优先级从高到低：IMSI 前缀（含用户群组）> 外网用户 > 本网用户 > 所有用户。为指定用户群配置某签约数据时，**其它参数必须一并配置**，否则默认值会引起误用。

### RAT 限制 / 核心网类型限制的填写粒度
- 问什么：限制只对主接入还是含辅助接入，以及限制哪个 RAT/核心网。
- RATRESTRICT、CORERESTRICT 默认"全部灰化"（未生效）；配置时按主辅接入分别填 PRIMRATRESTRICT/SECRATRESTRICT 或合并填 RATRESTRICT，取决于限制策略范围（跨场景枚举差异）。

## 约束（若有）
- 规格上限：最多 1024 条记录。`severity=medium`
- 优先级高于签约：AMF 本地配置的接入和移动性签约数据优先级高于用户签约；用户匹配本地记录即按本地执行，未匹配才查签约。`severity=high`（误配会覆盖签约策略）
- 号段最长匹配：用户群重叠时以号段最长匹配为准，多条记录须参数齐全以防默认值误用。`severity=high`
- 生效时机：执行后立即生效。`severity=low`
