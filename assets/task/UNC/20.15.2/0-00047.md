# 配置用户移动性管理签约数据（ADD NGMMSUBDATA）

> 命令静态知识见 [ADD NGMMSUBDATA](command/UNC/20.15.2/ADD-NGMMSUBDATA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD NGMMSUBDATA 在 AMF 本地为指定用户（群）配置移动性管理参数，是**一条命令承载多种意图**的多用途命令。不同特性激活时关注的参数子集完全不同——配置生成的关键在于**按特性意图选取参数子集**，其余参数留默认。配置空间有 4 个维度：**用户范围**（必选，决定哪些用户受控）、**RFSP 策略**（可选，控制驻留策略）、**接入限制**（可选，控制 RAT/核心网类型）、**网络切片包含模式**（可选，控制切片下发）。

### 配置维度 1：用户范围（参数 SUBRANGE，必选）

| 取值 | 作用 | 联动参数 | 优先级 |
|---|---|---|---|
| `ALL_USER` | 所有用户 | 无 | 最低 |
| `HOME_USER` | 本网用户 | 无 | 中 |
| `FOREIGN_USER` | 外网用户 | 无 | 中 |
| `IMSI_PREFIX` | 按 IMSI 前缀号段 | `IMSIPRE`（条件必选，5~15 位数字） | 高 |
| `IMSI` | 指定 IMSI | `IMSI`（条件必选，14~15 位数字） | 最高（全匹配） |

> 策略匹配优先级（高→低）：IMSI > IMSI_PREFIX > HOME_USER/FOREIGN_USER > ALL_USER。用户群重叠时以号段最长匹配为准。

### 配置维度 2：RFSP 策略（参数 RFSPINDEX + RFSPPRI，可选）

| 参数 | 取值 | 默认 | 作用 |
|---|---|---|---|
| `RFSPINDEX` | 整数 0~256 | 0 | 本端规划的 RFSP 索引；0 表示直接使用签约 RFSP |
| `RFSPPRI` | `CFG_FIRST` / `SUB_FIRST` | `CFG_FIRST` | 配置优先（无论是否签约均用本地，此时 RFSPINDEX 不能为 0）/ 签约优先（未签约才用本地） |

### 配置维度 3：接入限制（参数 RATRESTRICT / CORERESTRICT / EXRATRESTRICT，可选）

| 参数 | 取值 | 默认 | 作用 |
|---|---|---|---|
| `RATRESTRICT` | EUTRA / NR / NR_REDCAP（多选） | 全部灰化（未生效） | 限制接入的 RAT 类型；NR_REDCAP 是 NR 子类型，限制它时普通 NR 仍可接入 |
| `CORERESTRICT` | EPC / FIFTHGC（多选） | 全部灰化（未生效） | 限制接入的核心网类型 |
| `EXRATRESTRICT` | `CFG_ONLY`（主/辅接入分别填 PRIMRATRESTRICT/SECRATRESTRICT） | NOT_SUPPORT | 扩展 RAT 限制（仅测试用） |

### 配置维度 4：网络切片包含模式（参数 NIM，可选）

| 取值 | 作用 | 默认 |
|---|---|---|
| `MODE_A` / `MODE_B` / `MODE_C` / `MODE_D` | 控制 UE 在 RRC 连接建立时携带网络切片信息的方式（3GPP 24.501） | `MODE_C` |

> 其余参数：`DRX`（寻呼 DRX，0/32/64/128/256，默认 0 无效）、`UEAMBRULK`/`UEAMBRDLK`（上下行 UE-AMBR 限制，0 不生效）、`MPSMCSPRI`/`MPS`/`MCS`（仅 IMSI 单用户拨测，依赖 ADD UEOVERLDCTRL（★R1.5 atom 待补） 的 MPSMCSSWITCH=ON）。

**典型脚本（按 IMSI 前缀下发 RFSP 策略）**
```
ADD NGMMSUBDATA: SUBRANGE=IMSI_PREFIX, IMSIPRE="30809", RFSPINDEX=1, RFSPPRI=CFG_FIRST;
```

**典型脚本（限制 LTE 接入）**
```
ADD NGMMSUBDATA: SUBRANGE=IMSI_PREFIX, IMSIPRE="123031234567890", RATRESTRICT=EUTRA-1;
```

**步骤位置**：在所属 feature 编排中通常位于"配置用户移动性管理参数"步骤——用户群圈定与策略下发阶段。

## 决策点

### 决策点：用户范围与匹配优先级（DP 0-02073）

问：按哪种维度指定要应用本地签约数据的用户群？多记录用户群重叠时按什么规则匹配？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| ALL_USER（兜底） | `SUBRANGE=ALL_USER` | 整网用户兜底策略 |
| HOME_USER / FOREIGN_USER | `SUBRANGE=HOME_USER/FOREIGN_USER` | 按本网/外网属性 |
| IMSI_PREFIX（号段） | `SUBRANGE=IMSI_PREFIX` + `IMSIPRE` | 按号段精细化圈定 |
| IMSI（单用户） | `SUBRANGE=IMSI` + `IMSI` | 单用户拨测场景 |

> 多记录用户群重叠时 AMF 以号段最长匹配为准；为指定用户群配置某签约数据时，其它参数必须一并配置，否则默认值会引起误用。

### 决策点：RFSP 优先级（本地配置 vs UDM 签约）（DP 0-02074）

问：AMF 本地配置的 RFSP 与 UDM 签约 RFSP 冲突时，谁优先发给 PCF/NG-RAN？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 配置优先 | `RFSPPRI=CFG_FIRST`（RFSPINDEX 不能为 0） | 不信赖 UDM 签约（如漫游用户自定义驻留策略） |
| 签约优先 | `RFSPPRI=SUB_FIRST`（未签约才用本地） | 信赖 UDM 签约，本地仅作兜底 |

> AMF 本地配置的接入和移动性签约数据优先级整体高于用户签约——用户匹配本地记录即按本地执行，未匹配才查签约。

## 约束

- **AMF 本地配置的接入和移动性签约数据优先级高于用户签约**（critical，rule-0-02181）：AMF 本地配置的接入和移动性签约数据优先级高于用户签约；用户匹配本地记录即按本地执行，未匹配才查签约 — 后果：误配覆盖签约策略
- **用户群重叠时以号段最长匹配为准**（warning，rule-0-02182）：用户群重叠时以号段最长匹配为准；为指定用户（群）配置某签约数据时，其它参数必须一并配置，防止默认值未改引起误用 — 后果：默认值误用导致策略非预期
- **RFSPPRI=CFG_FIRST 时 RFSPINDEX**（warning，rule-0-02183）：RFSPPRI=CFG_FIRST 时 RFSPINDEX 不能为 0；RFSPINDEX=0 表示直接使用签约 RFSP Index — 后果：配置优先模式下无本地 RFSP 可用
- **最多可输入 1024 条记录**（warning，rule-0-02184）：最多可输入 1024 条记录；NR_REDCAP 是 NR 的子类型，限制 NR 时普通 NR 和 NR_REDCAP 用户都不允许接入，限制 NR_REDCAP 时普通 NR 仍可接入 — 后果：超规格新增失败；接入限制范围误判
- **该命令执行后立即生效**（info，rule-0-02185）：该命令执行后立即生效；适用 NF：AMF — 后果：无延迟
