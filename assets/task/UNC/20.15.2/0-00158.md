# SET CHFINIT 原子任务
> 命令静态知识见 [SET CHFINIT](command/UNC/20.15.2/SET-CHFINIT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

绑定到已建融合计费模板（CCT，由 `ADD CCT` 创建），决定该模板在用户激活时与 CHF 的交互行为。系统存在初始记录（CCTMPLTNAME=`global`，CHFINIT=SENDREQ）。

## 配置方法
按 CCT 模板逐一配置；未输入的可选参数保持系统当前配置不变（可通过 `LST CCT` 查询）。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTMPLTNAME | 融合计费模板名（来自已建 CCT） | `cct-test` / `global` |
| CHFINIT | CHF 交互使能开关（激活时是否发 ChargingDataRequest） | `SENDREQ`（激活发送） |
| CCRINITRGNUM | 初始 RG 个数（激活时为哪些 RG 预申请配额） | `0`~`10`，如 `5`、`10` |
| WAITCHFRESP | CHF 交互是否等待 CHF 响应 | `DISABLE`（不使能） |
| RGSOURCE | RG 来源（仅 PCC 用户生效，使用 PCF 下发规则向 CHF 预申请） | `DEFAULT`（缺省配置） |

典型脚本（激活即与 CHF 交互，预申请 5 个 RG）：
```
SET CHFINIT: CCTMPLTNAME="cct_test", CHFINIT=SENDREQ, CCRINITRGNUM=5;
```

## 决策点

### 用户激活是否建计费会话（CHFINIT）
- 问什么：PDU 会话建立时是否立即向 CHF 发 ChargingDataRequest 协商 RoamingChargingProfile。
- 激活发送（SENDREQ）→ 用户激活即建计费会话；常见默认。
- 激活不发送 → 业务使用时才触发；漫游切换场景下，本地用户切漫游用户时不会触发计费会话创建，直接用 SMF 本地配置生成 RoamingChargingProfile（见 WSFD-224103 约束）。
- 取值来源本端规划，视网络是否要求激活即协商而定。

### 初始 RG 个数（CCRINITRGNUM）
- 问什么：建计费会话时一次性为多少个 RG 预申请配额。
- 0~10 之间；值取决于该模板下需预申请配额的 RG 数量，配合 URR 组合 URR 指定具体 RG。
- 是否申请配额还取决于 RG 的 Charging method 是否 online 类型（仅 online RG 才在 Multiple Unit Usage 中申请配额）。

## 约束
- 前置：CCTMPLTNAME 参数使用 `ADD CCT`（[0-00009](task/UNC/20.15.2/0-00009.md)）配置生成。`severity=high`
- 初始记录：系统部署后已存在 CCTMPLTNAME=`global` 的初始记录（CHFINIT=SENDREQ）。`severity=low`
- 生效时机：执行后立即生效。`severity=low`
- 漫游切换一致性：配置"激活不与 CHF 交互"时，本地用户切换为漫游用户不会触发计费会话创建请求，直接使用 SMF 本地 RoamingChargingProfile。`severity=medium`
- QBC 计费限制（WSFD-224103）：UNC 作为 V-SMF/H-SMF 时 QBC 计费不支持 GaGy 接口；用户激活场景使用 QBC 计费时 SMF 必须与 CHF 交互（不支持"激活不交互"）。`severity=medium`
