# 配置CHF激活交互参数（SET CHFINIT）

> 命令静态知识见 [SET CHFINIT](command/UNC/20.15.2/SET-CHFINIT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CHFINIT 绑定到已建融合计费模板（CCT，由 [ADD CCT](task/UNC/20.15.2/0-00009.md) 创建），决定该模板在用户激活时与 CHF 的交互行为。系统存在初始记录（CCTMPLTNAME=`global`，CHFINIT=SENDREQ，CCRINITRGNUM=5，WAITCHFRESP=ENABLE，RGSOURCE=DEFAULT）。本命令的核心配置空间有 2 个维度：**CHF 交互使能**（必选，决定激活是否建计费会话）、**初始 RG 预申请**（可选，决定预申请配额的 RG 个数与来源）。

### 配置维度 1：CHF 交互使能（参数 CHFINIT，必选）

| 取值 | 作用 | 联动参数 | 适用场景 |
|---|---|---|---|
| **SENDREQ**（激活发送） | PDU 会话建立即向 CHF 发 ChargingDataRequest 协商 RoamingChargingProfile | 配 CCRINITRGNUM 指定预申请 RG 个数 | 常见默认，要求激活即协商 |
| **NOTSENDREQ**（激活不发送） | 业务使用时才触发计费会话 | 漫游切换时本地用户切漫游不触发计费会话创建，直接用 SMF 本地配置生成 RoamingChargingProfile | 不要求激活即协商 |
| **INITRGTRIGR**（有预申请 RG 触发发送） | 仅当有预申请 RG 时触发向 CHF 交互 | 配合 CCRINITRGNUM/RGSOURCE | 按需触发场景 |

> 数据来源：本端规划。未输入该参数时保持系统当前配置不变（可通过 `LST CCT` 查询）。

### 配置维度 2：初始 RG 预申请（参数 CCRINITRGNUM / RGSOURCE / WAITCHFRESP，可选）

| 参数 | 作用 | 取值 | 适用场景 |
|---|---|---|---|
| `CCRINITRGNUM` | 初始请求消息中允许携带 RG 的最大个数 | 整数 `0`~`10` | 配合 CHFINIT=SENDREQ，决定激活时为哪些 RG 预申请配额 |
| `RGSOURCE` | RG 来源（仅 PCC 用户生效，使用 PCF 下发规则向 CHF 预申请） | `DEFAULT`（缺省，动态规则>预定义规则>预定义规则组 UserProfile 的 CtxStartRating）/ `CTXSTARTRATING`（用 UserProfile 的 CtxStartRating 配置） | PCC 用户预申请配额来源 |
| `WAITCHFRESP` | CHF 交互是否等待 CHF 响应 | `ENABLE` / `DISABLE` | **当前版本暂不支持配置此参数，默认等待 CHF 响应** |

> RGSOURCE=DEFAULT 时，在新增 UPF 与在线恢复场景下，仅使用 PCF 下发的动态规则向 CHF 预申请配额。是否实际申请配额还取决于 RG 的 Charging method 是否 online 类型（仅 online RG 才在 Multiple Unit Usage 中申请配额）。

典型脚本（激活即与 CHF 交互，预申请 5 个 RG）：
```
SET CHFINIT: CCTMPLTNAME="cct_test", CHFINIT=SENDREQ, CCRINITRGNUM=5;
```
仅配置交互使能（不传可选参数，保持系统当前值）：
```
SET CHFINIT: CCTMPLTNAME="test", CHFINIT=SENDREQ;
```

**步骤位置**：在所属 feature 编排中通常位于"配置 SMF-CHF 交互"步骤——[ADD CCT](task/UNC/20.15.2/0-00009.md) 创建模板之后，URR/URRGROUP 预申请配额配置之后。

## 决策点

### 决策点：用户激活是否建计费会话（DP 0-03055）

问：PDU 会话建立时是否立即向 CHF 发 ChargingDataRequest 协商 RoamingChargingProfile？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 激活发送（SENDREQ） | CHFINIT=SENDREQ + CCRINITRGNUM | 用户激活即建计费会话；常见默认 |
| 激活不发送（NOTSENDREQ） | CHFINIT=NOTSENDREQ | 业务使用时才触发；漫游切换场景下本地用户切漫游不触发计费会话创建，直接用 SMF 本地 RoamingChargingProfile |
| 有预申请 RG 触发（INITRGTRIGR） | CHFINIT=INITRGTRIGR + CCRINITRGNUM/RGSOURCE | 仅当有预申请 RG 时才触发交互 |

> 取值来源本端规划，视网络是否要求激活即协商而定。注意 QBC 计费场景下必须激活即交互（见约束 rule-0-03140）。

### 决策点：初始 RG 个数与来源（DP 0-03056）

问：建计费会话时一次性为多少个 RG 预申请配额，RG 从哪里来？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 指定 RG 个数（0~10） | CCRINITRGNUM=5 或 10 | 配合 URR 组合 URR 指定具体 RG；值取决于该模板下需预申请配额的 RG 数量 |
| RG 来源=DEFAULT（PCF 下发） | RGSOURCE=DEFAULT | PCC 用户，使用 PCF 下发规则（动态>预定义>预定义组 UserProfile）向 CHF 预申请 |
| RG 来源=CTXSTARTRATING | RGSOURCE=CTXSTARTRATING | 使用 UserProfile 的 CtxStartRating 配置向 CHF 申请配额 |

> 是否实际申请配额取决于 RG 的 Charging method 是否 online 类型（仅 online RG 才在 Multiple Unit Usage 中申请配额）。

## 约束

- **CCT 模板前置**（high，rule-0-03136）：CCTMPLTNAME 参数使用 [ADD CCT](task/UNC/20.15.2/0-00009.md) 配置生成，模板必须先建 — 后果：引用未建模板配置失败
- **系统初始记录 global**（info，rule-0-03137）：系统部署后已存在 CCTMPLTNAME=`global` 的初始记录（CHFINIT=SENDREQ，CCRINITRGNUM=5，WAITCHFRESP=ENABLE，RGSOURCE=DEFAULT） — 后果：未显式配置时走初始值
- **立即生效**（info，rule-0-03138）：执行后立即生效 — 后果：无需重启
- **漫游切换一致性**（warning，rule-0-03139）：配置"激活不与 CHF 交互"（NOTSENDREQ）时，本地用户切换为漫游用户不会触发计费会话创建请求，直接使用 SMF 本地 RoamingChargingProfile — 后果：漫游用户计费属性可能与本地配置不一致
- **QBC 计费必须交互（WSFD-224103）**（high，rule-0-03140）：UNC 作为 V-SMF/H-SMF 时 QBC 计费不支持 GaGy 接口；用户激活场景使用 QBC 计费时 SMF 必须与 CHF 交互（不支持"激活不交互"） — 后果：QBC 计费用户激活失败
