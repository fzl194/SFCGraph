# 增加对端NF的SUPI信息（ADD PNFSUPI）

> 命令静态知识见 [ADD PNFSUPI](command/UNC/20.15.2/ADD-PNFSUPI.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD PNFSUPI 为**可选**命令，仅当网络中无 NRF、对端 NF 未注册到 NRF、或虽有 NRF 但需基于运营商策略本地配置周边 NF 时才配置。本命令把"对端 NF 实例 × SUPI 号段区间"绑定起来，使 AMF/SMF 等网元能按 SUPI（IMSI）号段本地选择对端 NF（如 LMF、CHF、UDM、AUSF）。与孪生命令 [ADD PNFGPSI](task/UNC/20.15.2/0-00063.md)（按 GPSI 号段）常成对出现；与 ADD PNFPROFILE（对端 NF 实例概述）、ADD PNFWILDCARD（通配策略）共同构成"无 NRF/本地策略"对端选择链路。配置空间有 3 个维度：**NF 实例标识**（必选，指向 ADD PNFPROFILE 已建实例）、**查询方式**（必选，总开关）、**号段描述**（条件必选，由查询方式决定走区间或模式）。

### 配置维度 1：NF 实例标识（参数 NFINSTANCEID，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| NF 实例标识 | `NFINSTANCEID` | 全网规划；字符串 0~50 字符；uuid 格式或非 uuid 格式（非 uuid 长度不超 18，且不允许纯 IP/纯十六进制冒号串） | 无 | 指向已建对端 NF 实例；与 ADD PNFPROFILE 的 NFINSTANCEID 取值一致时关联关系生效 | 无 NRF 或需本地策略时按 SUPI 选对端 NF |

### 配置维度 2：查询方式（参数 QUERYTYPE，必选总开关）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 查询方式 | `QUERYTYPE` | 全网规划；`START_END`（号段区间）/ `PATTERN`（正则表达式） | 无 | 决定 SUPI 号段的匹配方式；推荐 START_END | START_END 适合规整连续号段；PATTERN 适合离散/不规则号段集合 |

### 配置维度 3：号段描述（参数 RANGESTART+RANGEEND 或 PATTERN，条件必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 起始/终止号段 | `RANGESTART` + `RANGEEND` | 全网规划；字符串 5~15 位；成对出现，END ≥ START，两者长度需相等 | 无 | QUERYTYPE=START_END 时描述号段区间 `[START, END]` | 明确定长连续号段（证据两例均为此） |
| 模式串 | `PATTERN` | 全网规划；正则表达式 0~50 字符（不需输入 `^`/`$`）；转化后字符串总长 10~20 | 无 | QUERYTYPE=PATTERN 时按正则匹配 SUPI 号段 | 离散/不规则号段集合 |

> SUPI 含 IMSI（15 位）或网络专用格式；号段长度 5~15 位；同一 NF 可配多条不重叠号段，跨 NF 间号段不可重叠。

**典型脚本（号段区间，LMF1 支持 SUPI 12345~12346）**
```
ADD PNFSUPI: INDEX=1, NFINSTANCEID="LMF_Instance_1", QUERYTYPE=START_END, RANGESTART="12345", RANGEEND="12346";
```

**典型脚本（正则模式，AUSF 按 imsi-12345678904 前缀匹配）**
```
ADD PNFSUPI: INDEX=1, NFINSTANCEID="AUSF_Instance_0", QUERYTYPE=PATTERN, PATTERN="imsi-12345678904[0-9]{4}";
```

**步骤位置**：在"基于号段选择 LMF/CHF"特性编排中通常位于"配置对端 NF 的 SUPI 号段"步骤——服务化接口管理/本地 NRF 功能管理阶段，紧随 ADD PNFPROFILE（对端 NF 实例概述）之后，与 ADD PNFGPSI（GPSI 号段孪生）并行，最终由 SET NGLCSPARA（LMF 选择开关）激活。

## 决策点

### 决策点：SUPI 还是 GPSI 维度（DP 0-02107）

问：本地按号段选对端 NF 时，用 SUPI（IMSI）还是 GPSI（用户号码）维度？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 用 SUPI（本命令） | `INDEX + NFINSTANCEID + QUERYTYPE + RANGESTART/RANGEEND` 或 `PATTERN` | 按 IMSI 号段路由（如 LMF 选择 SUPI 12345~12346、CHF 选择 SUPI 123031200100001、UDM/AUSF 实例） |
| 用 GPSI（孪生命令） | 省略本命令，改用 [ADD PNFGPSI](task/UNC/20.15.2/0-00063.md) | 按用户号码路由；证据中两者常成对配置覆盖同批用户 |

> 证据中 LMF 选择场景 SET NGLCSPARA 的 SUPISW=YES/GPSISW=NO 即选 SUPI 维度；CHF 选择场景任务五亦用 ADD PNFSUPI 绑定 CHF_Instance_0。

### 决策点：号段用区间还是模式描述（DP 0-02108）

问：SUPI 号段用区间还是模式描述？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 区间 START_END | `QUERYTYPE=START_END` + `RANGESTART` + `RANGEEND` | 明确定长连续号段；证据 LMF1（12345~12346）、CHF（123031200100001）均为此；推荐方式 |
| 模式 PATTERN | `QUERYTYPE=PATTERN` + `PATTERN` | 离散/不规则号段集合；证据 AUSF 以 `imsi-12345678904[0-9]{4}` 为例 |

## 约束

- **除 INDEX 外**（critical，rule-0-02266）：除 `INDEX` 外，其它参数组合输入必须唯一——同一 `(NFINSTANCEID, QUERYTYPE, RANGESTART/RANGEEND 或 PATTERN)` 不可重复
- **NFINSTANCEID 长度 0~50**（warning，rule-0-02267）：`NFINSTANCEID` 长度 0~50，须满足格式约束（uuid 或非纯 IP/纯十六进制冒号串），且与 ADD PNFPROFILE 的 NFINSTANCEID 取值一致时关联关系才生效
- **RANGESTART/RANGEEND 长度 5~15**（critical，rule-0-02268）：`RANGESTART`/`RANGEEND` 长度 5~15，需成对出现且 END ≥ START、两者长度相等；`PATTERN` 长度 0~50，转化后字符串总长 10~20
- **当 ADD PNFWILDCARD 的**（warning，rule-0-02269）：当 ADD PNFWILDCARD 的 `SUPICFGSWITCH=ON` 且本地未配 SUPI 时，代表允许所有 SUPI 号段——此时本命令可省略；`NFINSTANCEID="sbidialtest"` 时用于 model-D/model-C 间接路由拨测，最多 80 条且不支持 PATTERN 模式
- **INDEX 取值 0~4294967295（指定索引优先**（info，rule-0-02270）：`INDEX` 取值 0~4294967295（指定索引优先，已使用则分配未使用索引）；最多可输入 500000 条记录；执行后立即生效；适用 NF：AMF、SMF、NSSF、SMSF、NCG
