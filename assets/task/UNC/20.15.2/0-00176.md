# SET GLBDFTCHFGROUP 原子任务
> 命令静态知识见 [SET GLBDFTCHFGROUP](command/UNC/20.15.2/SET-GLBDFTCHFGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

融合计费（WSFD-011206）场景下，全局默认 CHF 组提供两层用途：① 作为基于 CC 选择 CHF（MOD SELECTCHFGBYCC）之外的兜底；② 当 CHF 的 FQDN / NFInstance ID / CHF Group 被配置修改或删除时，为在途计费消息提供可回放的 CHF。系统部署后已存在初始记录（PRIMARYCHFGRP=null、SECONDARYCHFGRP=null）。

## 配置方法
按全网规划填主/备 CHF 组名（与 ADD CHFGROUP 创建的组名一致）；只配一个时另一个保持当前值不变（可通过 LST GLBDFTCHFGROUP 查看）。输入空格或 null（不区分大小写）可清空对应参数值。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PRIMARYCHFGRP（主 CHF 组） | 全网规划 | `ChfGroup1` |
| SECONDARYCHFGRP（备 CHF 组） | 全网规划 | `ChfGroup2` |

典型脚本（主备 CHF 组）：
```
SET GLBDFTCHFGROUP: PRIMARYCHFGRP="ChfGroup1", SECONDARYCHFGRP="ChfGroup2";
```

## 决策点

### 主备 CHF 组选择
- 问什么：主、备 CHF 组分别选哪个（来自全网规划）。
- 取值依赖组网：CHF 组名需先经 ADD CHFGROUP / ADD TNFBINDGRP 建立，本命令只是绑定引用；规划口径由组网设计决定，非命令内部能推导。
- 是否配备用：备用 CHF 组用于主用不可达时回放，跨场景差异（强容灾必配，单 CHF 部署可不配）。

## 约束
- 唯一性：全局一份（隐含，最大记录数为 1）。`severity=medium`
- 生效时机：执行后立即生效。`severity=high`
- 前置：主/备 CHF 组名须先经 ADD CHFGROUP 创建并已绑定 TNF 实例（ADD TNFBINDGRP），否则引用无效。`severity=high`
- 清空语义：参数输入空格或 null（不区分大小写）清空当前值；用于解除绑定。`severity=medium`
- 适用 NF：PGW-C、SMF。`severity=low`
