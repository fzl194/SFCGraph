# ADD CPCGBINDING 原子任务

> 命令静态知识见 [ADD CPCGBINDING](command/UNC/20.15.2/ADD-CPCGBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CPCGBINDING 在一个已存在的抄送 CG 组（[ADD CPCGGRP](task/UNC/20.15.2/0-00024.md)）下增加一条 CG 绑定：指定 CG 的 IP 版本、IP 地址、端口号及优先级。它服务于离线计费场景下的话单抄送（WSFD-011201 离线计费参考信息），与抄送 CG 组配合使用。

> **重要**：命令真相 notes 明确标注"**当前版本不支持此命令**"，且 ③ 各特性配置范式无数据规划表样例（仅出现于特性参考信息的命令清单中）。下表取值样例由命令参数真相表推导，实际配置以整网 CG 规划为准。

**参数取值样例（抄送 CG 绑定范式，依参数真相推导）**

| 参数 | 取值样例 | 默认 | 说明 |
|---|---|---|---|
| CPCGGRPID | `1` | — | 抄送 CG 组 ID（必填），整数 1，需先由 ADD CPCGGRP 建组 |
| CGIPVERSION | `IPV4` | IPV4 | CG IP 版本（必填），枚举 |
| CGIPV4ADDR | `<ipv4>` | 无 | CG IPv4 地址（conditional，CGIPVERSION=IPV4 时必填） |
| CGIPV6ADDR | `<ipv6>` | 无 | CG IPv6 地址（conditional，CGIPVERSION=IPV6 时必填） |
| CGPORT | `1024~65535` | — | CG 端口号（必填），整数 1024~65535 |
| PRIORITY | `0~100` | — | 等级（必填），整数 0~100，数值越小优先级越高 |

**配置位置**：在"WSFD-011201 支持离线计费"流程的命令清单中，位于 ADD LOGICINF、ADD CPCGGRP 之后。典型顺序：ADD CPCGGRP（建组）→ ADD CPCGBINDING（组下绑 CG）。

## 决策点

### CG IP 版本与地址选择

CGIPVERSION 决定填写 CGIPV4ADDR 还是 CGIPV6ADDR（conditional 互斥）。

| 选项 | 适用 |
|---|---|
| `IPV4` + CGIPV4ADDR | CG 仅支持 IPv4 或组网为纯 IPv4 |
| `IPV6` + CGIPV6ADDR | CG 支持 IPv6 / IPv6 组网 |

> 参数真相中 CGIPVERSION/CGIPV4ADDR/CGIPV6ADDR/CGPORT 标注数据来源为 global_planned，CPCGGRPID/PRIORITY 为 local_planned → 绑定类型：CG 地址侧 global_planned，组 ID 与优先级 local_planned。

## 约束

- **立即生效** · severity=LOW。
- **记录上限**：本命令最大记录数为 32 · severity=MEDIUM · 超过上限新增失败。
- **版本不支持**：当前版本不支持此命令（见命令 notes） · severity=HIGH · 实际部署前需确认网元版本是否启用该功能。
- **前置依赖**：CPCGGRPID 必须先由 ADD CPCGGRP 创建（取值仅限整数 1） · severity=HIGH · 组不存在则绑定失败。
- **conditional 互斥**：CGIPV4ADDR 与 CGIPV6ADDR 按 CGIPVERSION 二选一 · severity=MEDIUM。
