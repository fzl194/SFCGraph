# 设置5G移动性管理流程控制参数（SET NGMMPROCTRL）

> 命令静态知识见 [SET NGMMPROCTRL](command/UNC/20.15.2/SET-NGMMPROCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令控制 AMF 在 5G 移动性管理流程异常场景下发给 UE 的原因值，满足运营商对现网设备兼容性特殊控制的需求。配置空间主要由**流程类型（PROT）**这一核心维度决定——每个 PROT 对应一组条件可选的拒绝原因值参数。

### 配置维度1：流程类型（参数 PROT）

| PROT 取值 | 作用 | 典型条件参数 |
|---|---|---|
| AUSF_DISCOVERY | AUSF 发现流程异常的下发原因值 | AUSFDISCNOFOUND/AUSFDISCFC/AUSFDISCOTHER |
| AIR | 获取鉴权集流程异常的下发原因值 | AIRUSERNOTFOUND/AIRAUTHREJECTED/AIRCTXNOTFOUND/AIRTIMEOUT 等 |
| AUTHENTICATION | 鉴权流程异常的下发原因值 | AUTHRESCHKFAIL/AUTHMACFAIL/AUTHSYNCHFAIL |
| CHECK_IMEI | IMEI 检查失败的下发原因值 | CHKIMEICHKFAIL |
| UDM_DISCOVERY | UDM 发现流程异常的下发原因值 | UDMDISCNOFOUND/UDMDISCOTHER |
| UDM_REGISTRATION | UDM 注册流程异常的下发原因值 | UDMREGUSRNOTFD/UDMREGUNKNOWSUB/UDMREGROAMNOTAL/UDMREGALWAYS 等 |
| UDM_GET_SUBSCRIBER_DATA | UDM 获取签约数据异常的下发原因值 | UDMGETSUBNOUSR/UDMGETSUBNODATA/UDMGETTOOMANY 等 |
| UDM_SUBSCIBE | UDM 订阅流程异常的下发原因值 | UDMSUBNOUSR/UDMSUBUNSPT/UDMSUBTIMEOUT 等 |
| PCF_DISCOVERY | PCF 发现流程异常的下发原因值 | — |
| PCF_POLICY | PCF 策略交互流程异常的下发原因值 | — |
| OTHER_PROC | 其他流程（漫游/接入限制）的下发原因值 | ROAMRST/ACCRST |
| CAG | CAG 限制 | — |
| SMC | 安全算法协商流程 | — |

### 配置维度2：拒绝原因值（每个条件参数）

| 取值 | 作用 |
|---|---|
| 0 | 不使用特殊指定原因值，按系统默认下发（如 #15/#22/#27/#111 视场景而定） |
| 1 | 非协议定义原因值（不建议配置） |
| 2~111 | 协议定义原因值（具体含义见 3GPP TS 24.501） |

> 系统初始记录：UDMREGALWAYS=NO，UDMDISCALWAYS=YES，其他原因值均为 0。

### 配置维度3：强制向 UDM 注册（参数 UDMREGALWAYS，仅 PROT=UDM_REGISTRATION）

| 取值 | 作用 |
|---|---|
| NO（初始） | 不强制，AMF 本地有上下文时不向 UDM 注册 |
| YES | 不论本地是否有上下文，每次初始注册都强制向 UDM 发起注册（用于 4G/5G/IWK+HSS 组网场景确保被叫域选正确） |

**步骤位置**：在所属 feature（鉴权/移动性管理类特性）编排中，按需修改对应流程的原因值。

## 决策点

**流程类型选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| AIR 鉴权集流程 | PROT=AIR, AIRxxx 原因值 | 鉴权集获取异常的兼容性控制 |
| UDM 注册流程 | PROT=UDM_REGISTRATION, UDMREGxxx/UDMREGALWAYS | UDM 注册异常；4G/5G 互通场景强制注册 |
| 漫游/接入限制 | PROT=OTHER_PROC, ROAMRST/ACCRST | 漫游受限/接入限制下发原因值 |

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无延迟
- **原因值不合理可能导致用户无法驻留**（critical）：执行该命令配置的原因值不合理可能导致用户无法驻留网络，影响用户业务 — 后果：用户业务受影响
- **原因值变化影响话统**（warning）：配置下发的拒绝原因值变化可能导致相应拒绝类话统指标变化，并对终端行为产生影响 — 后果：话统指标变化
- **ADD NGCAUSEMAP 优先级更高**（warning）：相同流程中，对于同一种原因值映射配置，`ADD NGCAUSEMAP` 命令优先级高于本命令 — 后果：本命令配置可能被覆盖
- **原因值 1 不建议配置**（warning）：拒绝原因值 1 表示非协议定义原因值，不建议配置 — 后果：协议未定义行为
- **原因值修改为非 0 可能改变拒绝行为**（warning）：拒绝原因值修改为非 0 可能改变异常场景中下发给终端的拒绝原因值 — 后果：终端行为变化
