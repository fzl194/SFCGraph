# 配置计费方式开关（ADD CHARGEMETHOD）

> 命令静态知识见 [ADD CHARGEMETHOD](command/UNC/20.15.2/ADD-CHARGEMETHOD.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD CHARGEMETHOD 基于一条计费属性（CC）规则，开启该 CC 对应的在线/离线/融合计费开关。它定位在 `ADD CHARGECHAR` 建 CC 实例、`SET APNCHARGECTRL` / `SET CHARGECTRL` 完成绑定之后的"计费方式使能"环节，是决定用户最终走在线还是离线出账的总开关。本命令的核心配置空间有 3 个维度：**计费属性定位方式**（关键字 vs 精确值）、**计费方式开关**（ONLINE/OFFLINE/CONVERGED）、**业务申请上报模式**（仅融合场景）。

### 配置维度 1：计费属性定位方式（参数 CHARGECHARACT）

CC 规则的定位方式有两条路径，决定后续是否带掩码和优先级：

| 定位方式 | CHARGECHARACT 取值 | 联动参数 | 默认值 | 适用场景 |
|---|---|---|---|---|
| **关键字** | NORMAL / HOT_BILLING / PREPAID / FLAT_BILLING / DEFAULT | 仅 ONLINE/OFFLINE/CONVERGED 开关 | — | 按语义化计费类型匹配，简洁 |
| **精确值** | SPECIFIC_VALUE | CCVALUE + MASK（可选）+ PRIORITY（可选） | — | 按具体十六进制 CC 值精确匹配 |

| 参数 | 取值范围 | 默认 | 作用 |
|---|---|---|---|
| CCVALUE | 十六进制 0x0000~0xFFFF（可省 0x 前缀） | — | SPECIFIC_VALUE 时必填，指定特殊 CC 值 |
| MASK | 十六进制 0x0001~0xFFFF | — | 掩码匹配，与 CCVALUE 按位与 |
| PRIORITY | 整数 1~65535 | 0 | 值越小优先级越高；配 MASK 时必填 |

> 系统初始缺省记录：CHARGECHARACT=DEFAULT, CCVALUE=0x0000, MASK=0xFFFF, PRIORITY=0, ONLINE=DISABLE, OFFLINE=ENABLE, CONVERGED=ENABLE, RGAPPLIED=DEFAULT, QBCSW=ENABLE。当用户 CC 值无法匹配到非缺省配置时按缺省配置选择计费方式。

### 配置维度 2：计费方式开关（参数 ONLINE/OFFLINE/CONVERGED）

三种计费方式开关独立配置，默认值不对称：

| 参数 | 默认 | 取值 | 作用 |
|---|---|---|---|
| ONLINE | DISABLE | ENABLE/DISABLE | 在线计费开关（Gy/N40 实时配额申请） |
| OFFLINE | ENABLE | ENABLE/DISABLE | 离线计费开关（Ga 话单上报 CG） |
| CONVERGED | ENABLE | ENABLE/DISABLE | 融合计费开关（SMF 与 CHF 交互） |

| 计费方式目标 | 开关组合 | 适用场景 |
|---|---|---|
| 仅离线计费 | OFFLINE=ENABLE, ONLINE=DISABLE | Ga 离线话单 |
| 仅在线计费 | ONLINE=ENABLE, OFFLINE=DISABLE | Gy/N40 在线配额 |
| 在线+离线同时使能 | ONLINE=ENABLE, OFFLINE=ENABLE | 双轨并行 |
| 融合计费 | CONVERGED=ENABLE（+RGAPPLIED） | SMF 与 CHF 融合交互 |

### 配置维度 3：业务申请上报模式（参数 RGAPPLIED/QBCSW，仅 CONVERGED=ENABLE 时）

融合计费下决定 SMF 与 CHF 交互时业务申请上报的走向：

| 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|
| RGAPPLIED | DEFAULT / ONLINERGONLY / OFFLINERGONLY / NOCHG | DEFAULT | 业务申请上报模式 | CONVERGED=ENABLE 时生效 |
| QBCSW | ENABLE / DISABLE | ENABLE | QBC 计费开关 | CONVERGED=ENABLE 时生效 |

| RGAPPLIED 取值 | 含义 |
|---|---|
| DEFAULT | 在线 RG 与离线 RG 均可向 CHF 上报 |
| ONLINERGONLY | 仅在线 RG 向 CHF 上报 |
| OFFLINERGONLY | 仅离线 RG 向 CHF 上报 |
| NOCHG | 非 QBC 场景下不使能融合计费 |

### 典型脚本

```
# 仅离线计费（精确值定位）
ADD CHARGEMETHOD:CHARGECHARACT=SPECIFIC_VALUE,CCVALUE="0x400",MASK="0x400",
  PRIORITY=2,ONLINE=DISABLE,OFFLINE=ENABLE;

# 融合计费（关键字定位 + 在线 RG 上报）
ADD CHARGEMETHOD:CHARGECHARACT=NORMAL,CONVERGED=ENABLE,RGAPPLIED=ONLINERGONLY;

# 在线+离线同时使能（关键字定位）
ADD CHARGEMETHOD:CHARGECHARACT=NORMAL,ONLINE=ENABLE,OFFLINE=ENABLE;
```

**步骤位置**：在计费特性配置流程中，位于 `ADD CHARGECHAR`（建 CC 实例）、`SET CHARGECTRL` / `SET APNCHARGECTRL`（绑定 CC）之后的"计费方式使能"环节。

## 决策点

### 决策点：计费属性定位方式（DP 0-02009）

问：用语义化关键字还是具体十六进制值定位 CC 规则？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 关键字（NORMAL 等） | CHARGECHARACT=NORMAL/HOT_BILLING/PREPAID/FLAT_BILLING/DEFAULT + 仅 ONLINE/OFFLINE/CONVERGED | 按计费类型语义匹配，简洁，不允许带 MASK/PRIORITY |
| 精确值 | CHARGECHARACT=SPECIFIC_VALUE + CCVALUE + MASK（可选）+ PRIORITY（可选） | 需按具体 CC 值精确匹配或多条规则用掩码+优先级区分 |

### 决策点：计费方式开关组合（DP 0-02010）

问：该 CC 规则走哪条计费轨道？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅离线 | OFFLINE=ENABLE, ONLINE=DISABLE（默认即此组合） | Ga 接口话单计费 |
| 仅在线 | ONLINE=ENABLE, OFFLINE=DISABLE | Gy/N40 实时配额申请 |
| 在线+离线双轨 | ONLINE=ENABLE, OFFLINE=ENABLE | 双轨并行出账 |
| 融合 | CONVERGED=ENABLE + RGAPPLIED（+ QBCSW） | SMF 与 CHF 融合交互，5G Converged Charging |

## 约束

- **关键字禁配掩码/优先级**（critical，rule-0-02021）：关键字禁配掩码/优先级：CHARGECHARACT 为 normal/hotbilling/prepaid/flat-billing 关键字时不允许配 MASK 和 PRIORITY，仅 SPECIFIC_VALUE（0x0800/0x0400/0x0200/0x0100）才允许带 CCVALUE+MASK+PRIORITY
- **生效时机**（warning，rule-0-02022）：生效时机：本命令增删改后只对新激活用户生效，已激活用户仍走原在线/离线计费方式，切换计费方式时易误判未生效，须用户重激活
- **一致性**（warning，rule-0-02023）：一致性：针对同一用户，本命令与 `ADD CHARGECHAR`（设 CC 计费类型）、`SET CHARGECTRL` 配置结果要一致（同为 ONLINE / OFFLINE / 同时使能），否则用户计费方式错误
- **规格上限**（warning，rule-0-02024）：规格上限：本表最大记录数 17，规划要精简
- **掩码匹配约束**（info，rule-0-02025）：掩码匹配约束：不允许配置 CCVALUE 和 MASK 取与操作后的值不等于 CCVALUE；不允许与已有配置的 CCVALUE&MASK 取与结果相等（除缺省配置外）
