# ADD CHARGEMETHOD 原子任务

> 命令静态知识见 [ADD CHARGEMETHOD](command/UNC/20.15.2/ADD-CHARGEMETHOD.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CHARGEMETHOD 基于一条计费属性（CC）规则，开启该 CC 对应的在线/离线/融合计费开关。它定位在 `ADD CHARGECHAR` 建 CC 实例、`SET APNCHARGECTRL` / `SET CHARGECTRL` 完成绑定之后的"计费方式使能"环节，是决定用户最终走在线还是离线出账的总开关。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| CHARGECHARACT | `SPECIFIC_VALUE` / `NORMAL` / `HOTBILLING` / `PREPAID` / `FLAT-BILLING` | 全网规划 | 计费属性关键字；SPECIFIC_VALUE 时按具体十六进制值匹配 |
| CCVALUE | 十六进制（如 `0x0400`、`0x400`） | 全网规划 | CHARGECHARACT=SPECIFIC_VALUE 时必填，可配掩码和优先级 |
| MASK | 十六进制 | 全网规划 | SPECIFIC_VALUE 时配合 CCVALUE 做掩码匹配 |
| PRIORITY | `1~65535` | 全网规划 | SPECIFIC_VALUE 时优先级 |
| ONLINE / OFFLINE / CONVERGED | `ENABLE` / `DISABLE` | 全网规划 | 三种计费方式开关；默认 ONLINE=DISABLE / OFFLINE=ENABLE / CONVERGED=ENABLE |
| RGAPPLIED | `DEFAULT` / `ONLINERGONLY` / `OFFLINERGONLY` | 全网规划 | 业务申请上报模式，CONVERGED 融合场景相关 |
| QBCSW | `ENABLE` / `DISABLE` | 全网规划 | QBC 计费开关 |

**典型配置范式（按特性场景）**

| 计费方式目标 | 脚本样例 |
|---|---|
| 仅离线计费 | `ADD CHARGEMETHOD: CHARGECHARACT=SPECIFIC_VALUE, CCVALUE="0x0400", MASK="0x0400", PRIORITY=2, OFFLINE=ENABLE;` |
| 仅在线计费 | 同上，末尾改 `ONLINE=ENABLE;` |
| 在线+离线同时使能 | 末尾 `ONLINE=ENABLE, OFFLINE=ENABLE;` |
| 融合计费 | `ADD CHARGEMETHOD: CHARGECHARACT=SPECIFIC_VALUE, CCVALUE="0x400", MASK="0x400", PRIORITY=2, CONVERGED=ENABLE, RGAPPLIED=ONLINERGONLY;` |

## 决策点

### CHARGECHARACT 关键字 vs SPECIFIC_VALUE

证据 notes 明确：用关键字（normal/hotbilling/prepaid/flat-billing）时**不允许配掩码和优先级**；只有 SPECIFIC_VALUE 才能带 CCVALUE+MASK+PRIORITY 做精确匹配。证据脚本里关键字与具体值的混用需注意此约束。

| 选项 | 配法 | 是否带 MASK/PRIORITY |
|---|---|---|
| 关键字（NORMAL 等） | `CHARGECHARACT=NORMAL, OFFLINE=ENABLE` | 不允许 |
| 精确值 | `CHARGECHARACT=SPECIFIC_VALUE, CCVALUE="0x0400", MASK="0x0400", PRIORITY=2` | 必须带 |

### RGAPPLIED 业务申请上报模式（融合场景）

融合计费下决定 URRGROUP 选离线还是在线 URR：

| 选项 | RGAPPLIED 取值 | 含义 |
|---|---|---|
| 自动选 RG | DEFAULT | URRGROUP 同时配离线+在线 URR 时优先在线 |
| 仅在线 RG | ONLINERGONLY | URRGROUP 下建议同时配离线+在线 URR |
| 仅离线 RG | OFFLINERGONLY | 同上 |

## 约束

- **只对新激活用户生效** · severity=HIGH · 增删改后已激活用户仍走原在线/离线计费方式，须用户重激活才反映，切换计费方式时易误判未生效。
- **最大 17 条记录** · severity=MEDIUM · 记录数受限，规划要精简。
- **关键字禁配掩码/优先级** · severity=MEDIUM · normal/hotbilling/prepaid/flat-billing 不允许带 MASK/PRIORITY；SPECIFIC_VALUE（0x0800/0x0400/0x0200/0x0100）才允许。
- **与 ADD CHARGECHAR 一致性** · severity=HIGH · 针对同一用户，本命令与 `ADD CHARGECHAR`、`SET CHARGECTRL` 配置结果要一致（同为 ONLINE / OFFLINE / 同时使能），否则用户计费方式错误。证据原文：例如 CC=normal 的本地用户，本地设在线又 normal 设离线，结果将出错。
- **前置依赖**：CC 实例须已由 `ADD CHARGECHAR` 建好；通常先 `SET CHARGECTRL` / `SET APNCHARGECTRL` 绑定 CC · severity=HIGH。
