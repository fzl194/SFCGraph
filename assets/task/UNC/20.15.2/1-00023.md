# UNC SMF 地址池体系（ADDRPOOL 系）

> **UNC 域地址池族共享底座**（SOP §3.5 域通用）。被引用于 8 个 APN 域特性。
>
> **命名空间独立**：UNC 用 `ADDRPOOL/ADDRPOOLGRP/SECTION/ADDRUPGROUP`（与 UDG 用户面 `POOL/POOLGROUP/SECTION` 同义但命名空间独立维护）。
>
> **设计参考**：UDG 域 `1-00022` 地址分配体系结构（结构对齐，UNC 命令族独立维护）。
>
> **重要修正（2026-07-12）**：UNC WSFD-010502 主特性命令清单（参考资料 72291445）仅含 8 条，**不含** `ADD POOLBINDAPN`（UNC 域与 UDG 域命名空间不同）；UNC 域地址池 POOLTYPE 取值 4 子方式（UDM/Radius/Local/DHCP），**不**含 LNS（LNS 归 WSFD-104410 L2TP VPN 独立特性 + SET APNL2TPCTRL 联动）。`ADD POOLBINDAPN` 仅 WSFD-107021 静态地址用户路由冗余专用。

## 配置方法

UNC 域地址池族核心骨架——按 4 子方式（UDM/Radius/Local/DHCP）建池/段/组/映射/UPF 绑定。服务于 [UNC 1-00024](task/UNC/20.15.2/1-00024.md)（APN 接入域基础设施）之后、特性级场景 DP（IP 类型 × 子方式 × CHECKIPVALID）驱动选哪些步骤。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 建地址池 | `ADD ADDRPOOL` → [0-00217](task/UNC/20.15.2/0-00217.md) | POOLNAME（全网规划）；POOLTYPE=Local/UDM/Radius/DHCP；IPVERSION=IPv4/IPv6/IPv4v6；HASVPN=Disable/Enable；VPNINSTANCE（HASVPN=Enable 时必配，与 APN.VPNINSTANCE 一致）；CHECKIPVALID=Enable（UDM/Radius 必配，白名单保护） |
| 2. 建地址段 | `ADD SECTION` → [0-00257](task/UNC/20.15.2/0-00257.md) | POOLNAME；IPVERSION=IPv4/IPv6；SECTIONNUM（段编号，1 起递增）；V4STARTIP/V4ENDIP（IPv4 段起止）；V6STARTIP/V6ENDIP/V6PREFIXLEN（IPv6 段起止 + 前缀长度） |
| 3. 建地址池组 | `ADD ADDRPOOLGRP` → [0-00218](task/UNC/20.15.2/0-00218.md) | POOLGRPNAME（全网规划）；POOLGRPTYPE=Default/UDM/Local/Radius/DHCP；IPVERSION |
| 4. 池绑池组 | `ADD POOLBINDGRP` → [0-00251](task/UNC/20.15.2/0-00251.md) | POOLNAME；POOLGRPNAME；POOLPRIORITY（数值小优先） |
| 5. 池绑 APN（白名单生效，**仅 WSFD-107021 静态地址用户路由冗余**） | `ADD POOLBINDAPN` | POOLNAME；APN |
| 6. 池组映射到 UPF 组 | `ADD POOLGRPMAP` → [0-00252](task/UNC/20.15.2/0-00252.md) | MAPPINGNAME（全网规划）；POOLGRPNAME；UPFGRPNAME（UPF 组名） |
| 7. 地址池绑 UPF 组 | `ADD ADDRUPGROUP` → [0-00219](task/UNC/20.15.2/0-00219.md) | UPFGRPNAME（UPF 组名）；UPFGRPTYPE=UDM/Local/Radius/DHCP；UPLISTNAME（已在 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) 步骤 8 配） |
| 8. 建 UP 节点 | `ADD UPNODE` → [0-00263](task/UNC/20.15.2/0-00263.md) | UPNODEID；NFINSTANCENAME（**须已 `ADD PNFPROFILE` 注册**，白名单保护）；PFINSTANCEID；IPADDRV4/IPADDRV6（UPF N4 GTP-U 地址，由 [UNC 1-00025](task/UNC/20.15.2/1-00025.md) ADD CPGTPUADDR 提供） |
| 9. UPF 绑 UPF 组 | `ADD UPFBINDGRP` → [0-00259](task/UNC/20.15.2/0-00259.md) | UPFGRPNAME；UPFID（UPF 实例 ID，同 UPNODE 的 NFINSTANCENAME）；UPLISTNAME；PRIORITY |

**典型脚本（WSFD-010502 IPv4 + UDM 子方式 + 带 VPN，2-00015 基线，按 [激活_67401512.md](evidence/UNC/20.15.2/WSFD-010502/WSFD-010502-配置_静态IP地址网段与UPF的绑定_67401512.md) 实证 + 0-00257/0-00219/0-00259/0-00252 atom 重写）**：
```
ADD ADDRPOOL: POOLNAME="pool01", POOLTYPE=UDM, IPVERSION=IPv4, HASVPN=Enable, VPNINSTANCE="vpn1", CHECKIPVALID=Enable;
ADD SECTION: POOLNAME="pool01", SECTIONNUM=1, IPVERSION=IPv4, V4STARTIP="10.0.0.0", V4ENDIP="10.1.2.3";
ADD ADDRPOOLGRP: POOLGRPNAME="poolgroup1", POOLGRPTYPE=UDM, IPVERSION=IPv4;
ADD POOLBINDGRP: POOLNAME="pool01", POOLGRPNAME="poolgroup1", POOLPRIORITY=1;
ADD POOLGRPMAP: MAPPINGNAME="mapping01", POOLGRPNAME="poolgroup1", UPFGRPNAME="upfgroup1";
ADD UPNODE: UPNODEID="upf-01", NFINSTANCENAME="upf-site1", PFINSTANCEID="pf-01";
ADD ADDRUPGROUP: UPFGRPNAME="upfgroup1", UPFGRPTYPE=UDM;
ADD UPFBINDGRP: UPFGRPNAME="upfgroup1", UPFID="UPF_Instance_1", UPLISTNAME="uplist-01", PRIORITY=0;
```

**步骤位置**：[UNC 1-00024](task/UNC/20.15.2/1-00024.md)（APN 接入域基础设施）之后，[UNC 1-00029](task/UNC/20.15.2/1-00029.md)（OSPF 路由发布）之前；WHITE 步骤 5（`ADD POOLBINDAPN`）**仅 WSFD-107021 执行**，不属通用骨架。

## 场景差异

- **WSFD-010502** [2-00015.md](task/UNC/20.15.2/2-00015.md) 地址分配方式：4 子方式（UDM/Radius/Local/DHCP）共用步骤 1-4 + 6-9；UDM/Radius 时 CHECKIPVALID=Enable；DHCP 子方式**不配套本骨架内 DHCP 服务器组链**（`ADD DHCPSERVERGRP/DHCPSERVER/DHCPBINDPOOLGRP` 归 WSFD-104413/WSFD-104005，详见 [特征决策表](task/UNC/20.15.2/2-00015.md)）。**★ WSFD-010502 + UDM 子方式 + CHECKIPVALID=Enable 场景下 `ADD POOLBINDAPN` 可选执行**（非必走）——activation 实证 [WSFD-010502-配置_静态IP地址网段与UPF的绑定_67401512.md](evidence/UNC/20.15.2/WSFD-010502/WSFD-010502-配置_静态IP地址网段与UPF的绑定_67401512.md) 步骤 3 演示该命令（开启白名单检查时），与 WSFD-107021 必走场景区分。
- **WSFD-010504** [2-00016.md](task/UNC/20.15.2/2-00016.md) 控制面地址分配方式：步骤 1-4（仅 Local 子方式，CHECKIPVALID=Disable）；步骤 8-9 不执行——控制面本地分配不绑 UPF
- **WSFD-104002** [2-00017.md](task/UNC/20.15.2/2-00017.md) IPv4v6 双栈：步骤 1-2 + 步骤 3-4 各跑 IPv4/IPv6 对称（POOLNAME/POOLGRPNAME 后缀区分）；步骤 6-9 共用 IPv4+IPv6 共享 UPFBINDGRP
- **WSFD-104001** [2-00018.md](task/UNC/20.15.2/2-00018.md) IPv6 承载上下文：步骤 1-2 + 3-4 仅 IPv6 单栈（IPVERSION=IPv6），CHECKIPVALID=Disable；步骤 8-9 必备——IPv6 单播路由依赖 UPF 实例透传
- **WSFD-104004** [2-00019.md](task/UNC/20.15.2/2-00019.md) IPv6 前缀代理：步骤 1-4 IPv6 单栈（V6PREFIXLENGTH<64 PD 模式，HASVPN=Enable + VPNINSTANCE=vpn1）；步骤 6-9 IPv6 单栈配套
- **WSFD-107021** [2-00023.md](task/UNC/20.15.2/2-00023.md) 静态地址用户路由冗余：步骤 1-9 全配，**+ 步骤 5（`ADD POOLBINDAPN` 白名单路径）必执行**——这是本骨架唯一一特性必走步骤 5
- **WSFD-104413** [2-00021.md](task/UNC/20.15.2/2-00021.md) DHCP 功能：POOLTYPE=DHCP + IPVERSION=IPv4；步骤 1-4 + 7-9；DHCP 服务器组链路（`ADD DHCPSERVERGRP/DHCPSERVER/DHCPBINDPOOLGRP` 3 命令，**UNC atom 待补**，见 R1.5 待补脚注）由本特征独立骨架承载，**不属本骨架**
- **WSFD-104005** [2-00022.md](task/UNC/20.15.2/2-00022.md) DHCPv6 地址分配：POOLTYPE=DHCP + IPVERSION=IPv6；同 WSFD-104413 步骤；DHCPv6 服务器组链路（同 3 命令待补）独立承载

## 决策点

| 选项 | 影响（参数/联动） |
|---|---|
| Local | POOLTYPE=Local；CHECKIPVALID=Disable（默认）；最常用 |
| UDM 静态 + 白名单 | POOLTYPE=UDM；CHECKIPVALID=Enable（必配）；企业固定 IP 场景 |
| Radius 静态/动态 + 白名单 | POOLTYPE=Radius；CHECKIPVALID=Enable（必配）；外部 AAA 场景 |
| DHCP | POOLTYPE=DHCP；CHECKIPVALID=Disable；DHCP 服务器组归 WSFD-104413/WSFD-104005 独立承载 |
| HASVPN=Disable | 地址池全局可用，不绑 VPN |
| HASVPN=Enable | 必填 VPNINSTANCE（与 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) 的 APN.VPNINSTANCE 一致） |
| POOLGRPTYPE=Default | 默认子方式（不区分 UDM/Local/Radius/DHCP） |
| POOLGRPTYPE=UDM/Local/Radius/DHCP | 区分子方式（WSFD-010502 4 子方式分别配） |

> 联动特性级 DP：POOLTYPE × HASVPN × CHECKIPVALID 由各 feature 场景 DP（IP 类型、子方式）决定。

## 约束

- **VPN 三者一致**（critical）：ADDRPOOL.VPNINSTANCE / APN.VPNINSTANCE / 外联口.VPNINSTANCE 三者必须一致 — 否则地址分配或路由异常
- **UDM/Radius 时必配 CHECKIPVALID**（critical）：CHECKIPVALID=Enable 是 APN 静态地址白名单保护前提 — 否则非法地址用户可激活
- **VPNINSTANCE 必须先建 VPN 实例**（critical）：ADD ADDRPOOL 时 HASVPN=Enable 时 VPNINSTANCE 必须已在 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) 步骤 3（`ADD VPNINST`）建好
- **POOLNAME 命名约束**（warning）：字符串 1~79，由 `-/_`/数字/字母/`.` 组成，不能以 `.` 开头
- **RELEASETIME=0 与 IPLEASE=Enable 互斥**（critical）：同条带配置时执行失败（命令 wiki 约束）
- **最大记录数 80000**（warning）：每 ADDRPOOL 最多 80000 记录
- **UPF 节点须 PNFPROFILE 先建**（critical）：`ADD UPNODE` 时 NFINSTANCENAME 必须是已 `ADD PNFPROFILE` 注册的 UPF 实例 — 否则配置失败
- **UNC 域 5GC N4 强绑定**（warning）：地址池必须绑 UPF（步骤 7 ADDRUPGROUP + 步骤 8 UPNODE + 步骤 9 UPFBINDGRP）以限定 UPF 实例可用地址池
- **本骨架不含独立命令外的命令**：DHCP 服务器组链路命令（WSFD-104413/WSFD-104005 专用）、OSPF 路由发布（[UNC 1-00029](task/UNC/20.15.2/1-00029.md)）、Radius 服务器组（[UNC 1-00025](task/UNC/20.15.2/1-00025.md)）等独立承载
