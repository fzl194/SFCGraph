# 配置APN计费接口选择（ADD APNCHGMODE）

> 命令静态知识见 [ADD APNCHGMODE](command/UNC/20.15.2/ADD-APNCHGMODE.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD APNCHGMODE 配置 APN/DNN 在不同终端接入类型下的计费接口选择方式，是三层优先级体系中的中间层（用户漫游属性 > APN/DNN 级 > 全局级）。本命令的核心配置空间有 3 个维度：**主计费接口**（FORCED，确定主流计费路径）、**SGW 独立部署计费模式**（FORSGWONLY，SGW 单独发起计费）、**V-SMF 漫游计费模式**（FORVSMFONLY，5G 国漫场景）。每条记录按"APN + 终端接入类型"定位一行。

### 配置维度 1：主计费接口（参数 FORCED）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **NchfMode** | Nchf 接口（5GC 融合计费，CHF/NChf 服务化接口） | 融合计费、5G 终端 4G 接入、Gy/Diameter 在线计费 |
| **GaGyMode** | GaGy 接口（传统离线计费，Gy + Ga 双接口） | 离线计费、传统 PGW/GGSN 场景 |
| **INVALID** | 无效值（当前版本实际选择 NchfMode） | 不推荐显式配置 |

> 选错计费接口且对应接口未配置时：**计费流程失败，用户无法激活**。

### 配置维度 2：终端接入类型（参数 TMACCTYPE）

| 取值 | 含义 | 典型场景 |
|---|---|---|
| **UE5G_RAT4G** | 5G 终端 4G 接入（PDN 激活 PCO 携带 5G PDU Session ID） | 5G 终端经 4G 接入 |
| **UE5G_RAT5G** | 5G 终端 5G 接入（5G PDU 激活流程） | 5G 终端 5G 接入 |
| **UENON5G_RAT4G** | 非 5G 终端 4G 接入 | 4G 终端 |
| **RAT2G** | 2G 接入（一次 PDP 激活） | 2G 接入（需提前确认对端 CHF 支持 Nchf） |
| **RAT3G** | 3G 接入（一次 PDP 激活） | 3G 接入 |
| **RATLTEM** | LTE-M 接入 | LTE-M 用户 |
| **RATNBIOT** | NB-IoT 接入 | NB-IoT 用户 |
| **NON_3GPP** | 非 3GPP 接入（如 WiFi 卸载） | 非 3GPP 接入 |

### 配置维度 3：SGW 独立部署 / V-SMF 漫游（参数 FORSGWONLY / FORVSMFONLY）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| FORSGWONLY=GaGyMode | SGW 独立部署时，SGW 自行发起计费会话走 GaGy 接口 | SGW 独立部署（与 PGW-C 分离） |
| FORSGWONLY=NchfMode | SGW 独立部署时走 Nchf 接口 | — |
| FORSGWONLY=Inherit（默认） | 继承其他选项，实际选 GaGyMode | 常规合一部署 |
| FORVSMFONLY=GaGyMode | V-SMF（国漫 H-SMF）漫游地接入 5GC 时计费走 GaGy | 5G 终端国漫漫游 |
| FORVSMFONLY=NchfMode | V-SMF 走 Nchf 接口 | — |

> 常规合一部署（SMF+SGW 合设）可不配置 FORSGWONLY 和 FORVSMFONLY。

### 互操作指示参数（BY5GSIWKI / BY5GCNRI）

| 参数 | 取值 | 作用 |
|---|---|---|
| **BY5GSIWKI** | False（默认）/ True | False：按 FORCED 选；True：根据 5GS 互操作指示值二次决策（1→Nchf，0/未携带→GaGy） |
| **BY5GCNRI** | False（默认）/ True | False：按 FORCED 选；True：根据 5GC 无限制接入标识二次决策（1→Nchf，0/未携带→GaGy） |

> 两者均为 False 时，按 FORCED 直接选接口；至少一个为 True 时，FORCED 作为兜底。

**步骤位置**：在所属 feature 编排中通常位于"步骤 2：配置计费接口选择"——APN 实例创建之后，URR/计费策略配置之前。

## 决策点：计费接口选择（DP 0-01005）

问：该 APN/DNN 在指定终端接入类型下用哪个计费接口？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| Nchf 接口（融合计费） | FORCED=NchfMode | 5GC 融合计费、Gy/Diameter 在线计费、5G 终端 4G 接入 |
| GaGy 接口（传统离线） | FORCED=GaGyMode | 离线计费、传统 PGW/GGSN 场景 |
| SGW 独立部署 | FORSGWONLY=GaGyMode | SGW 单独发起计费会话 |
| 5G 国漫漫游 | FORVSMFONLY=GaGyMode | V-SMF 漫游地接入 5GC |

> 优先级：ADD ROAMCHGMODE（用户漫游属性，高） > ADD APNCHGMODE（本命令，中） > SET CHGMODE（全局，低）。

## 约束

- **2G Nchf 前置确认**（critical，rule-0-01010）：当配置 2G 接入使用 Nchf 接口时，需提前和对端 CHF 确认是否支持 — 后果：计费流程失败
- **计费接口选错导致激活失败**（critical，rule-0-01011）：配置计费接口后相应接口未配置，计费流程失败，用户无法激活
- **互操作接口不变**（warning，rule-0-01012）：4G/5G 互操作流程的计费接口不变；2G/3G/4G 互操作流程的计费接口不变 — 后果：互操作切换时不能期望接口切换
- **APN 级优先级低于漫游属性**（warning，rule-0-01013）：本命令 APN/DNN 级配置优先级低于 `ADD ROAMCHGMODE`（用户漫游属性），高于 `SET CHGMODE`（全局） — 后果：与漫游属性冲突时以漫游属性为准
- **对新用户生效**（info，rule-0-01014）：执行后只对新激活用户生效，不立即对在线用户生效 — 后果：存量用户不变
- **规格上限**（info，rule-0-01015）：最多可输入 24000 条记录 — 后果：超规格新增失败
