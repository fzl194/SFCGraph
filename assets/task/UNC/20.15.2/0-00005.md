# ADD APNCHGMODE 原子任务
> 命令静态知识见 [ADD APNCHGMODE](command/UNC/20.15.2/ADD-APNCHGMODE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

计费接口选择有三层优先级（高到低）：用户漫游属性（`ADD ROAMCHGMODE`）> APN/DNN（本命令）> 全局（`SET CHGMODE`）。本命令是中间层。

## 配置方法
按"APN + 终端接入类型 + 计费接口"三元组配置。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | APN 名称（已配置或新建） | `apn-test` |
| TMACCTYPE | 终端和接入类型 | `UE5G_RAT4G`（5G 终端 4G 接入）等 |
| BY5GSIWKI | 是否按 5GS 互操作指示选接口 | `False` |
| FORCED | 全局计费接口 | `NchfMode` / `GaGyMode` |
| FORSGWONLY | 作为 SGW 时的计费模式 | `GaGyMode`（SGW 独立部署时用） |
| FORVSMFONLY | 作为 V-SMF 时的计费模式 | `GaGyMode`（国漫漫游场景） |

典型脚本（5G 终端 4G 接入走 Nchf）：
```
ADD APNCHGMODE: APN="apn-test", TMACCTYPE=UE5G_RAT4G, BY5GSIWKI=False, FORCED=NchfMode;
```

## 决策点（若有）

### 计费接口选择（FORCED）
- 问什么：该 APN/DNN 在指定终端接入类型下用哪个计费接口。
- Nchf 接口（5GC 融合计费）→ FORCED=NchfMode。
- GaGy 接口（传统离线计费）→ FORCED=GaGyMode。
- 配置不当会选错计费策略接口，相应接口未配时导致计费流程失败、用户无法激活。

### SGW 独立部署与 V-SMF 漫游场景
- 问什么：是否需要单独指定 SGW 或 V-SMF 发起计费会话时的接口。
- SGW 独立部署（SGW 需自己发起计费会话）→ 设 FORSGWONLY。
- 5G 终端国漫在漫游地接入 5GC → 设 FORVSMFONLY。
- 常规合一部署 → 两者可不设。

### BY5GSIWKI / BY5GCNRI 互操作指示
- 是否按 5GS 互操作指示 / 5GC 无限制接入标识二次决策接口，默认 False。

## 约束（若有）
- 互操作不变：4G/5G 互操作流程的计费接口不变；2G/3G/4G 互操作流程的计费接口不变。`severity=medium`
- 2G Nchf 前置确认：当用本命令配置 2G 接入使用 Nchf 接口时，需提前和对端 CHF 确认是否支持。`severity=high`
- 优先级：本命令的 APN/DNN 级配置优先级低于 `ADD ROAMCHGMODE`（用户漫游属性），高于 `SET CHGMODE`（全局）。`severity=medium`
- 生效时机：执行后只对新激活用户生效（不立即生效于在线用户）。`severity=medium`
- 规格上限：最多可输入 24000 条记录。`severity=low`
