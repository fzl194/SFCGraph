# 用户接入控制功能（WSFD-106003）

> 特性静态知识见 [WSFD-106003](feature/UNC/20.15.2/WSFD-106003.md)。本页讲配置生成实例化时这个特性**怎么配**——编排哪些步骤、AMF vs SGSN/MME 两代际如何选。

## 配置概览

核心对象链：**5G AMF（NAS 信令级）**：`ADD NGMMSUBDATA`（IMSI 段 + RAT/CORE 限制 + RFSP + NIM）→ `SET NGMMPROCTRL`（拒绝原因值）→ **4G SGSN/MME（NAS 信令级）**：`ADD GBARD`（Gb 模式 GERAN）+ `ADD IUARD`（Iu 模式 UTRAN）+ `ADD S1ARD`（S1 模式 E-UTRAN）（IMSI 段 + APNNI + ARD 签约 + CTRLTYPE + CAUSE + 可选 CARDTYPE）。

两代际**不共享命令族**（AMF 5G NAS vs SGSN/MME 2/3/4G NAS），均独立控制接入。共享接入域骨架（[UNC 1-00024](task/UNC/20.15.2/1-00024.md)）作为前置。由特性级 DP（NF 代际选择）驱动。

**License 信息**：AMF 免 License（特性概述明示）；SGSN/MME 需 [SET LICENSESWITCH](task/UNC/20.15.2/0-00180.md) 前置 — 配 `LKV2ARD02`。

## 配置流程（步骤编排，每步显示含命令）

1. **接入域基础设施**（L3VPN + VPN 实例 + APN 实例）：`ADD L3VPNINST` + `ADD VPNINSTAF` + `ADD VPNINST` + `ADD APN` → [UNC 1-00024](task/UNC/20.15.2/1-00024.md)
2. **License 前置**（仅 SGSN/MME 代际）：`SET LICENSESWITCH: LICITEM=LKV2ARD02, SWITCH=ENABLE` → [0-00180](task/UNC/20.15.2/0-00180.md)
3. **用户接入控制配置**（按 NF 代际选）：
   - **AMF（5G NAS）**：`ADD NGMMSUBDATA`（`SUBRANGE` + `IMSIPRE` + `RATRESTRICT`/`CORERESTRICT` + `RFSPINDEX` + `NIM`）→ [0-00047](task/UNC/20.15.2/0-00047.md)
   - **SGSN/MME（4G/3G/2G NAS）**：
     - `ADD GBARD`（Gb GERAN，`SUBRANGE`+`IMSIPRE`+`APNNI`+`CARDTYPE`+`ARD`+`CTRLTYPE`+`CAUSE`）
     - `ADD IUARD`（Iu UTRAN，同 GBARD 参数集）
     - `ADD S1ARD`（S1 E-UTRAN，无 `CARDTYPE`）
     > **★ R1.5 待补 atom**（3 条命令）：`ADD GBARD` / `ADD IUARD` / `ADD S1ARD` 真实存在 UNC OM（路径：`业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/接入控制/`），evidence = [WSFD-106003-激活_SGSN_MME_64082494.md](evidence/UNC/20.15.2/WSFD-106003/WSFD-106003-激活用户接入控制功能特性_适用于SGSN_MME_64082494.md) 7 任务示例脚本；UNC pre-build 阶段 atom 未建，归 P1 后续补建。
4. **（仅 AMF）拒绝原因值配置**：`SET NGMMPROCTRL: PROT=OTHER_PROC, ACCRST=0` → [0-00279](task/UNC/20.15.2/0-00279.md)

> **★ 代际差异驱动 DP（按 NF 选）**：
> - **AMF（5G NAS，UNC 20.3.2+）**：基于用户签约或 AMF 本地配置的 RAT 限制 / 核心网类型限制 控制接入；本地配置优先级 > 签约数据；紧急注册跳过接入控制。
> - **SGSN/MME（2/3G SGSN UNC 20.3.0+ / 4G MME UNC 20.0.0+）**：基于 ARD 签约/APNNI/卡类型 控制接入；按接入模式分 Gb/Iu/S1 三类 ARD 命令；CARDTYPE 仅 Gb/Iu 模式有效（S1 无卡类型）。

## 决策点（特性级：NF 代际 × 接入模式 × 限制维度）

| NF × 接入模式 × 限制维度 | 走法（复用 compound）| 关键差异 |
|---|---|---|
| **AMF × ALL_USER × 5GC 拒入**| 骨架 1-00024 + ADD NGMMSUBDATA(SUBRANGE=ALL_USER, CORERESTRICT=FIFTHGC-1) + SET NGMMPROCTRL(ACCRST=0) | 全部 5G 用户拒入 5GC，系统下 #27（N1 mode not allowed）|
| **AMF × IMSI 段 × NR 拒入**| 骨架 1-00024 + ADD NGMMSUBDATA(SUBRANGE=IMSI_PREFIX, IMSIPRE, RATRESTRICT=NR-1) + SET NGMMPROCTRL | NR 无线接入限制；指定 IMSI 号段 |
| **AMF × IMSI 段 × EUTRA 拒入**| 骨架 1-00024 + ADD NGMMSUBDATA(SUBRANGE=IMSI_PREFIX, IMSIPRE, RATRESTRICT=EUTRA-1) + SET NGMMPROCTRL | EUTRA 无线接入限制 |
| **AMF × IMSI 段 × 4G 互操作拒入**| 骨架 1-00024 + ADD NGMMSUBDATA(SUBRANGE=IMSI_PREFIX, IMSIPRE, CORERESTRICT=EPC-1) + SET NGMMPROCTRL | 5G→4G 互操作限制 |
| **SGSN/MME × ALL_USER × E-UTRAN 拒入**| 骨架 1-00024 + License LKV2ARD02 + ADD S1ARD(SUBRANGE=ALL_USER, APNNI="*", ARD=NO, CTRLTYPE=REJECT, CAUSE=EPS_SERVICE_NOT_ALLOWED) | 全部用户拒入 4G E-UTRAN |
| **SGSN/MME × IMSI 段 × UTRAN + SIM 卡拒入**| 骨架 1-00024 + License LKV2ARD02 + ADD IUARD(SUBRANGE=SPECIAL_IMSIPRE, IMSIPRE, APNNI="*", CARDTYPE=SIM-1, ARD=NO, CTRLTYPE=REJECT, CAUSE=GPRS_SERVICE_NOT_ALLOWED) | UTRAN 网络拒入 SIM 卡用户 |
| **SGSN/MME × IMSI 段 × GERAN + SIM 卡准入**| 骨架 1-00024 + License LKV2ARD02 + ADD GBARD(SUBRANGE=SPECIAL_IMSIPRE, IMSIPRE, APNNI="*", CARDTYPE=SIM-1, ARD=NO, CTRLTYPE=ALLOW) | GERAN 网络准入 SIM 卡用户 |
| **SGSN/MME × IMSI 段 × E-UTRAN + APNNI 准入**| 骨架 1-00024 + License LKV2ARD02 + ADD S1ARD(SUBRANGE=SPECIAL_IMSIPRE, IMSIPRE, APNNI="wcdma", ARD=NO, CTRLTYPE=ALLOW) | E-UTRAN 网络准入指定 APN 用户 |
| **SGSN/MME × IMSI 段 × ARD 签约准入**| 骨架 1-00024 + License LKV2ARD02 + ADD S1ARD(SUBRANGE=SPECIAL_IMSIPRE, IMSIPRE, APNNI="*", ARD=YES, CTRLTYPE=ALLOW) | 启用 ARD 签约校验 |

> **走法列**：AMF 代际仅 2 步（ADD NGMMSUBDATA + SET NGMMPROCTRL）；SGSN/MME 代际 3-5 步（GBARD+IUARD+S1ARD 选配 + License 前置）。**两代际不共享命令族**（AMF 5G NAS vs SGSN/MME 2/3/4G NAS）。

## 约束（特性级 rule）

- **紧急注册跳过接入控制**（critical，AMF）：紧急注册流程不执行接入控制检查 — 后果：紧急用户不受 ARD 限制。
- **本地配置 > 签约数据**（warning，AMF）：AMF 本地配置的 RAT/CORE 限制信息优先级高于签约数据；当签约或本地任一为空时表示不限制 — 后果：本地配置变更后仅对新注册用户生效，已注册存量用户不变。
- **ARD=NO + CTRLTYPE=ALLOW 时 APNNI 最好为 * **（info，SGSN/MME）：当只使用 ARD 签约控制接入时，"APNNI" 设置为 "*" 且 "CTRLTYPE" 最好选择 "ALLOW" — 后果：避免 APNNI 不匹配致用户激活失败。
- **CARDTYPE 仅 Gb/Iu 模式**（critical，SGSN/MME）：`ADD S1ARD` 无 `CARDTYPE` 参数（4G E-UTRAN 不区分卡类型）— 后果：S1ARD 配 CARDTYPE 参数执行失败。
- **License 控制项必配**（critical，SGSN/MME）：未 `SET LICENSESWITCH` 启用 LKV2ARD02 时 SGSN/MME 接入控制功能不可用 — 后果：ARD 参数配置无效。
- **CAUSE 原因值与接入模式匹配**（warning）：GPRS_SERVICE_NOT_ALLOWED 用于 2/3G（GPRS）场景；EPS_SERVICE_NOT_ALLOWED 用于 4G（EPS）场景 — 后果：CAUSE 不匹配致 UE 拒绝原因值不准确。
- **本特性不含独立命令外的命令**（info）：完整接入控制链路由各 feature 主骨架承担；本特性仅负责 NAS 信令级本地接入控制配置，不涉及 RADIUS AAA 鉴权（鉴权族特性 WSFD-011305/011306/010301）。
