# 配置全局使用量上报规则组（SET GLBURRGROUP）

> 命令静态知识见 [SET GLBURRGROUP](command/UNC/20.15.2/SET-GLBURRGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET GLBURRGROUP 为 PDP 用户指定上下行发起使用的 URR（即上下行报文如何计费），是**无 user-profile 时的全局兜底**。系统部署后已存在初始记录（NOCHARGINGFLAG=NONE）。**当前版本不支持此命令**（产品 notes 明示），用于离线/融合/在线计费的费率标识配置（WSFD-011201、WSFD-011206、WSFD-104003、WSFD-109001）。本命令有 2 个配置维度：**上下行 URR 绑定与计费模式**、**不计费标记**。

### 配置维度 1：上下行 URR 绑定与计费模式（参数 UPOFF/DOWNOFF/UPONL/DNONL URRNAME，可选）

按上下行 × 在线/离线 4 个 URR 槽位绑定 URR 名（URR 名须先经 `ADD URR` 创建并设定 USAGERPTMODE）。上下行可同 URR 也可不同，但必须同为在线或同为离线；不允许只配下行不配上行。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 上行离线 URR | `UPOFFURRNAME` | 字符串 1~31，不支持空格，不区分大小写 | 无 | 上行发起使用的离线计费 URR | 离线计费上行报文 |
| 下行离线 URR | `DOWNOFFURRNAME` | 字符串 1~31 | 无 | 下行发起使用的离线计费 URR | 离线计费下行报文 |
| 上行在线 URR | `UPONLURRNAME` | 字符串 1~31 | 无 | 上行发起使用的在线计费 URR | 在线计费上行报文 |
| 下行在线 URR | `DNONLURRNAME` | 字符串 1~31 | 无 | 下行发起使用的在线计费 URR | 在线计费下行报文 |

> 数据来源：本端规划。URR 名必须先经 `ADD URR` 创建；输入单空格删除已有配置项。

### 配置维度 2：不计费标记（参数 NOCHARGINGFLAG，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 不计费标记 | `NOCHARGINGFLAG` | `NONE`/`UPLINK`/`DOWNLINK`/`UPLINK_DOWNLINK` | NONE（初始值） | 指定方向流量不做计费 | 特定方向免计费 |

> NOCHARGINGFLAG 初始值为 NONE（正常计费）；UPLINK=上行不计费，DOWNLINK=下行不计费，UPLINK_DOWNLINK=上下行都不计费。

**典型脚本**

```
# 离线计费（上下行同 URR）
SET GLBURRGROUP: UPOFFURRNAME="offlineURR", DOWNOFFURRNAME="offlineURR";

# 在线计费（上下行同 URR）
SET GLBURRGROUP: UPONLURRNAME="urr1", DNONLURRNAME="urr1";
```

**步骤位置**：在费率标识类特性编排中位于"全局 URR 组绑定"步骤——紧随 `ADD URR` 建 URR（并设定 USAGERPTMODE）之后；细粒度的 `SET URRGRPBINDING`（UserProfile 计费属性绑定 URRGroup）优先级更高，按需独立配置。

## 决策点

### 决策点：上下行 URR 绑定与计费模式（DP 0-03095）

问：上行 / 下行分别用哪个 URR，走在线还是离线轨道？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 离线·上下行同 URR | `UPOFFURRNAME`=`DOWNOFFURRNAME`（同一离线 URR） | 上下行同费率，离线计费 |
| 离线·上下行不同 URR | `UPOFFURRNAME`≠`DOWNOFFURRNAME`（同为离线 URR） | 上下行差异化费率，离线计费 |
| 在线·上下行同 URR | `UPONLURRNAME`=`DNONLURRNAME`（同一在线 URR） | 上下行同费率，在线计费 |
| 在线·上下行不同 URR | `UPONLURRNAME`≠`DNONLURRNAME`（同为在线 URR） | 上下行差异化费率，在线计费 |

> 取值由运营商费率计划决定；USAGERPTMODE 驱动在线 vs 离线分叉，跨场景枚举；上下行必须同为在线或同为离线。

### 决策点：URR 名获取来源（DP 0-03096）

问：URR 名从哪里取？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 本端规划（新建） | URR 名为首次规划的全新名称，须先 `ADD URR` 创建 | 首次部署，无既有 URR |
| 已配置数据中获取（引用） | URR 名引用既有 URR（仍须 `ADD URR` 已存在） | 复用既有 URR，非首次配置 |

> 数据规划表"获取方法"两值并存，按是否首次配置分叉。

## 约束

- **互斥（在线/离线一致）**（critical，rule-0-03236）：互斥（在线/离线一致）：上下行 URR 必须同为在线或同为离线计费，不允许交叉（如上行在线 + 下行离线） — 后果：计费模式不一致导致报文计费归属错误
- **互斥（上下行完整性）**（critical，rule-0-03237）：互斥（上下行完整性）：不允许只配下行 URR 不配上行 URR — 后果：上行报文无计费规则导致漏计
- **前置**（warning，rule-0-03238）：前置：URR 名须先经 ADD URR 创建并设定 USAGERPTMODE；离线 URR 不允许同时绑定到 URRGroup（ADD URRGROUP）和 GlbURRGroup（本命令） — 后果：前置缺失或重复归属导致绑定失败/计费重复
- **版本限制**（warning，rule-0-03239）：版本限制：产品 notes 明示"当前版本不支持此命令"，配置前须确认本网元版本是否生效 — 后果：在受影响版本上命令不生效
- **唯一性**（info，rule-0-03240）：唯一性：全局一份（最大记录数 1），系统部署后已存在初始记录（NOCHARGINGFLAG=NONE） — 后果：超规格配置失败；适用 NF 为 PGW-C、SMF
