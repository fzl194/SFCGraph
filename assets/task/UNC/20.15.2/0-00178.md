# SET GLBURRGROUP 原子任务
> 命令静态知识见 [SET GLBURRGROUP](command/UNC/20.15.2/SET-GLBURRGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

全局 URR 组用于指定上下行报文各自发起的 URR（上行/下行 × 在线/离线 共 4 个槽位 + 不计费标记）。系统部署后已存在初始记录（NOCHARGINGFLAG=NONE）。**当前版本不支持此命令**（产品 notes 明示），用于离线/融合/在线计费的费率标识配置（WSFD-011201、WSFD-011206、WSFD-104003、WSFD-109001）。

## 配置方法
按上下行 × 在线/离线 4 个 URR 槽位绑定 URR 名（URR 名须先经 ADD URR 创建并设定 USAGERPTMODE）。上下行可同 URR 也可不同，但必须同为在线或同为离线；不允许只配下行不配上行。

| 参数 | 描述 | 配置方法 | 取值样例 |
|---|---|---|---|
| UPOFFURRNAME | 上行发起离线 URR 名 | 本端规划 / 已配置数据中获取 | `offlineURR` |
| DOWNOFFURRNAME | 下行发起离线 URR 名 | 本端规划 / 已配置数据中获取 | `offlineURR` |
| UPONLURRNAME | 上行发起在线 URR 名 | 本端规划 / 已配置数据中获取 | `onlineURR` / `urr1` |
| DNONLURRNAME | 下行发起在线 URR 名 | 本端规划 / 已配置数据中获取 | `onlineURR` / `urr1` |
| NOCHARGINGFLAG | 不计费标记 | 本端规划 | 枚举（初始 NONE） |

典型脚本（离线计费，上下行同 URR）：
```
SET GLBURRGROUP:UPOFFURRNAME="offlineURR", DOWNOFFURRNAME="offlineURR";
```
典型脚本（在线计费，上下行同 URR）：
```
SET GLBURRGROUP:UPONLURRNAME="urr1", DNONLURRNAME="urr1";
```

## 决策点

### 上下行 URR 绑定与计费模式
- 问什么：上行 / 下行分别用哪个 URR，是否同 URR。
- 取值由运营商费率计划决定：上下行同费率→同 URR；差异化→不同 URR，但必须同为在线或同为离线。
- 计费模式分叉（在线 vs 离线）：决定填 UPONL/DNONL 还是 UPOFF/DOWNOFF；由 USAGERPTMODE 驱动，跨场景枚举。

### URR 名获取来源
- 问什么：URR 名从哪里取。
- 数据规划表"获取方法"两值并存（本端规划 / 已配置数据中获取）：本端规划=新建 URR 名；已配置数据中获取=引用既有 URR；按是否首次配置分叉。

## 约束
- 唯一性：全局一份（最大记录数为 1）。`severity=medium`
- 生效时机：执行后立即生效。`severity=high`
- 当前版本：产品 notes 明示"当前版本不支持此命令"，配置前需确认本网元版本是否生效。`severity=high`
- 互斥（上下行完整性）：不允许只配下行 URR 不配上行 URR。`severity=high`
- 互斥（在线/离线一致）：上下行 URR 必须同为在线或同为离线计费。`severity=high`
- 互斥（URR 归属）：离线 URR 不允许同时绑定到 URRGroup（ADD URRGROUP）和 GlbURRGroup（本命令）。`severity=medium`
- 前置：URR 名须先经 ADD URR 创建。`severity=high`
- 适用 NF：PGW-C、SMF。`severity=low`
