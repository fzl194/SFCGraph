# 增加GRE隧道（ADD GRETUNNEL）

> 命令静态知识见 [ADD GRETUNNEL](command/UNC/20.15.2/ADD-GRETUNNEL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令创建一个 GRE 隧道接口，将某些网络层协议报文封装在 IP 中传输。配置空间主要由三个维度决定：**隧道类型**（IPv4 gre vs IPv6 gre6）、**源类型**（IP 地址 vs 接口名）、**可选功能**（Keepalive/识别关键字/校验和/统计/备份）。

### 配置维度1：隧道类型（参数 TNLTYPE）

| 取值 | 作用 | 配套源参数 | 典型场景 |
|---|---|---|---|
| gre | IPv4 GRE 隧道 | SRCTYPE + SRCIPADDR/SRCIFNAME + DSTIPADDR | 普通 IPv4 GRE 隧道（IPFD-015002 GRE / GRE over IPSec） |
| gre6 | IPv6 GRE 隧道 | SRCTYPE6 + SRCIPV6ADDR/SRCIFNAME + DSTIPV6ADDR | IPv6 GRE 隧道 |

### 配置维度2：源类型（参数 SRCTYPE / SRCTYPE6）

| 取值 | 作用 | 配套必选参数 |
|---|---|---|
| no_type（默认） | 不配置源类型 | — |
| ip_address | 用指定 IP 地址作封装源 | SRCIPADDR（IPv4）/ SRCIPV6ADDR（IPv6） |
| if_name | 用指定接口的 IP 作封装源 | SRCIFNAME |

> 本端配置的源 IP 必须与隧道对端配置的目的 IP 相同，隧道才能正常建立。

### 配置维度3：可选功能（参数 KEEPALVEN / GREKEYEN / CHECKSUMEN / STATENABLE / REDUNDANCYEN）

| 功能 | 使能参数 | 配套参数 | 作用 |
|---|---|---|---|
| Keepalive | KEEPALVEN=TRUE | KEEPALVPERIOD（默认5）、KEEPALVRETRYCNT（默认3） | 周期性探测对端可达性，不可达时 Tunnel 接口置 Down |
| 识别关键字 | GREKEYEN=TRUE | GREKEY（必填，0~4294967295） | 收发双方验证通道识别关键字 |
| 端到端校验 | CHECKSUMEN=TRUE | — | 对接收报文计算校验和校验 |
| 报文统计 | STATENABLE=TRUE | — | 隧道接口进出流量统计 |
| 备份隧道 | REDUNDANCYEN=TRUE | — | 备份静态地址用户流量 |

**步骤位置**：在所属 feature 编排中，于接口/LoopBack 与目的 IP 规划完成后执行；GRE over IPSec 场景下，GRE 隧道与 IPSec 策略配合。

## 决策点

**隧道类型与源类型选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| gre + if_name | TNLTYPE=gre, SRCTYPE=if_name, SRCIFNAME, DSTIPADDR | IPFD-015002 GRE / WSFD-011306 Radius功能（带外组网 GRE VPN） / GRE over IPSec |
| gre + ip_address | TNLTYPE=gre, SRCTYPE=ip_address, SRCIPADDR, DSTIPADDR | 用指定 IP 作源 |
| gre6 + if_name / ip_address | TNLTYPE=gre6, SRCTYPE6=... | IPv6 GRE 隧道 |

## 约束

- **立即生效**（info）：命令执行后立即生效 — 后果：无延迟
- **GRE 源地址不能与 IPSec 源地址相同**（critical）：GRE 隧道的源地址不能和 IPSec 隧道的源地址相同 — 后果：配置冲突
- **源目的地址对唯一**（warning）：不能对两个或两个以上使用同种封装协议的 Tunnel 接口配置完全相同的源 IP 地址和目的地址 — 后果：配置冲突
- **本端源 IP = 对端目的 IP**（critical）：本端配置的源 IP 必须与隧道对端配置的目的 IP 相同；本端目的 IP 必须与对端源 IP 相同 — 后果：隧道无法建立
- **DSTVPNNAME 需先建 L3VPNINST**（warning）：DSTVPNNAME 指定的 VPN 必须先由 `ADD L3VPNINST` 定义 — 后果：索引失败
- **识别关键字两端必须一致**（warning）：GREKEYEN=TRUE 时，两端必须设相同 GREKEY，或两端都不设 — 后果：验证失败报文丢弃
- **Keepalive 单向**（info）：Keepalive 功能是单向的，两端都使能才能双方都具备 — 后果：单端使能只本端能探测对端
- **最多 4096 条记录**（warning）：本命令最大记录数为 4096 — 后果：超规格新增失败
