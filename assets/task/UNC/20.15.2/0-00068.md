# 增加RG级异常返回码处理动作（ADD RGRCACT）

> 命令静态知识见 [ADD RGRCACT](command/UNC/20.15.2/ADD-RGRCACT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"融合计费模板 + RG 级异常返回码 + 处理动作"三元组配置一条记录。处理动作 RCACT 分阻塞类（BLCK_TRG_RPT/BLCK_IMMED_RPT/BLCK_NO_RPT，联动 BLKTIMER 阻塞时长）、重定向类（REDIRECT，联动 RDVIRTIP/RDVIRTIPV6 重定向地址）、去活类（TERMINATE，联动 REACTREQ 重新激活请求）、继续类（CONTINUE，联动 HOLDINGTIME 用户保持时长）。系统部署后已存在初始记录（global/DEFAULT/BLCK_TRG_RPT），本命令按需覆盖或新增。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 模板绑定 | CCTMPLTNAME | 已配置数据获取（由 ADD CCT 建好），1~63，不区分大小写 | — | 指定本条记录所属融合计费模板 | 全部场景 |
| 阻塞类动作 | RCCODE, RCACT=BLCK_TRG_RPT/BLCK_IMMED_RPT/BLCK_NO_RPT, BLKTIMER | RCCODE 全网规划（DEFAULT/SERVICEDENIED/QUOTAMNOTAPPL 等）；BLKTIMER 全网规划 0~1440 分钟 | BLKTIMER 默认 0 | 阻塞业务并按时机上报 CHF | 配额不足/拒绝服务等阻塞场景 |
| 重定向类动作 | RCACT=REDIRECT, RDVIRTIP, RDVIRTIPV6 | RDVIRTIP/RDVIRTIPV6 全网规划（有效 IPv4/IPv6，不允许全 0/全 F/0.0.0.0~0.255.255.255/255.255.255.255） | RDVIRTIP 默认 0.0.0.0；RDVIRTIPV6 默认 :: | 重定向业务到指定地址 | 重定向 Portal 场景 |
| 去活类动作 | RCACT=TERMINATE, REACTREQ | REACTREQ 全网规划 DISABLE/ENABLE | REACTREQ 默认 DISABLE | 去活 PDU 会话，可通知用户重新激活 | SERVICEDENIED 等终止场景 |
| 继续类动作 | RCACT=CONTINUE, HOLDINGTIME | HOLDINGTIME 全网规划 0~2880 分钟 | HOLDINGTIME 默认 30（CONTINUE 时） | 允许业务继续不再配额管理，超时去活 | 转离线计费场景 |

典型脚本（global 模板 SERVICEDENIED 去活 PDU 会话）：
```
ADD RGRCACT:CCTMPLTNAME="global",RCCODE=SERVICEDENIED,RCACT=TERMINATE;
```
典型脚本（阻塞类，配阻塞时长）：
```
ADD RGRCACT:CCTMPLTNAME="cct-test",RCCODE=QUOTAMNOTAPPL,RCACT=BLCK_IMMED_RPT,BLKTIMER=30;
```

步骤位置：建融合计费模板（`ADD CCT`）→ 本任务 → 可选联动忽略 CHF 响应（`SET CNVRGDCHGPARA` IGNORERSPAVP=FUI）

## 决策点

### 决策点：RG 级异常返回码的动作类型（DP 0-02115）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 阻塞类（BLCK_TRG_RPT/BLCK_IMMED_RPT/BLCK_NO_RPT） | BLKTIMER（阻塞时长 0~1440 分钟） | 阻塞业务并按 Trigger/立即/不上报时机上报 CHF |
| 重定向类（REDIRECT） | RDVIRTIP 和 RDVIRTIPV6 至少配置一个 | 重定向到指定 IPv4/IPv6 Portal 地址 |
| 去活类（TERMINATE） | REACTREQ（是否通知用户重新激活） | 直接去活 PDU 会话 |
| 继续类（CONTINUE） | HOLDINGTIME（用户保持时长 0~2880 分钟） | 允许业务继续不再配额管理，超时去活 |

### 决策点：是否覆盖 DEFAULT 缺省返回码（DP 0-02116）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 具体返回码（SERVICEDENIED/QUOTAMNOTAPPL/QUOTALIMITRCH/SERVICEREJECT/USERUNKNOWN/RATINGFAILED） | RCCODE=具体码 | 仅命中该码时执行动作 |
| DEFAULT | RCCODE=DEFAULT | 收到配置外结果码时执行的缺省动作（初始记录即 DEFAULT/BLCK_TRG_RPT） |

## 约束

· rule-0-02286 · severity=high · 重定向地址前置：当 RCACT=REDIRECT 时，RDVIRTIP 和 RDVIRTIPV6 至少配置一个；有效 IPv4 不允许 0.0.0.0~0.255.255.255 或 255.255.255.255，有效 IPv6 不允许全 0 或全 F
· rule-0-02287 · severity=high · 阻塞时长前置：当 RCACT 为阻塞类（BLCK_IMMED_RPT/BLCK_NO_RPT/REDIRECT）时，BLKTIMER 为条件可选（0~1440 分钟，默认 0）
· rule-0-02288 · severity=high · 前置校验：CCTMPLTNAME 须已由 ADD CCT 配置生成
· rule-0-02289 · severity=medium · 规格上限：最多可输入 707 条记录；系统部署后已存在初始记录（global/DEFAULT/BLCK_TRG_RPT）
· rule-0-02290 · severity=medium · 生效时机：命令执行后立即生效
