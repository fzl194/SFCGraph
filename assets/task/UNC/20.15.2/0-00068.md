# ADD RGRCACT 原子任务
> 命令静态知识见 [ADD RGRCACT](command/UNC/20.15.2/ADD-RGRCACT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令把"RG 级异常返回码 → 处理动作"映射挂到一个融合计费模板（CCTMPLTNAME，来自 [ADD CCT](task/UNC/20.15.2/0-00009.md)）。当 CHF 返回的 Charging Data Response 携带 RG 级异常返回码时，按此配置处理。动作含阻塞类（如 BLCK_IMMED_RPT / BLCK_TRG_RPT，需配 BLKTIMER 阻塞时长）、重定向类（需配 RDVIRTIP/RDVIRTIPV6 重定向地址）等；系统部署后已有初始记录，本命令按需覆盖或新增。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTMPLTNAME | 融合计费模板名称，全网规划（从已配置 CCT 数据获取），1~63 字符，不区分大小写 | `cct-test` / `global` |
| RCCODE | RG 级异常返回码，全网规划；支持 DEFAULT（收到配置外结果码时的缺省动作） | `QUOTAMNOTAPPL` / `DEFAULT` |
| RCACT | RG 级异常返回码动作，全网规划 | `BLCK_IMMED_RPT` / `BLCK_TRG_RPT` / 重定向类 |
| BLKTIMER | RG 级阻塞处理时间间隔（分钟），全网规划，0~1440（仅阻塞类动作时必配） | `30` |
| RDVIRTIP / RDVIRTIPV6 | RG 级重定向 IPv4/IPv6 地址，全网规划（仅重定向类动作时必配，至少配一个） | `10.1.1.1` |
| REACTREQ | RG 级重新激活请求，全网规划（默认 DISABLE） | `DISABLE` / `ENABLE` |
| HOLDINGTIME | 用户保持时长（分钟），全网规划，0~2880 | `60` |

典型脚本（阻塞类动作，配阻塞时长）：
```
ADD RGRCACT:CCTMPLTNAME="cct-test", RCCODE=QUOTAMNOTAPPL, RCACT=BLCK_IMMED_RPT, BLKTIMER=30;
```

## 决策点

### RG 级异常返回码的动作类型
- 问什么：收到指定 RG 级异常返回码时执行哪类动作；不同动作类别联动不同的必配参数。
| RCACT 类别 | 含义 | 必配联动参数 |
|---|---|---|
| 阻塞类（BLCK_IMMED_RPT 等） | 阻塞业务并立即/触发上报 | BLKTIMER（阻塞时长） |
| 阻塞类（BLCK_TRG_RPT 等） | 阻塞业务，后续 Trigger 触发时上报 | BLKTIMER |
| 重定向类 | 重定向到指定 IP/Portal | RDVIRTIP 与 RDVIRTIPV6 至少一个 |

### 是否覆盖 DEFAULT 缺省返回码
- 问什么：除了为指定返回码（如 QUOTAMNOTAPPL）配动作外，是否要为"配置外结果码"配一条缺省动作。
| RCCODE 取值 | 行为 |
|---|---|
| 具体返回码（如 QUOTAMNOTAPPL） | 仅命中该码时执行动作 |
| DEFAULT | 收到配置外结果码时执行的缺省动作 |

## 约束
- 生效时机：命令执行后**立即生效**。`severity=medium`
- 记录上限：最多可输入 **707** 条记录。`severity=medium`
- 重定向地址前置（条件必配）：当 RCACT 配置为重定向类动作时，RDVIRTIP 和 RDVIRTIPV6 **至少配置一个**。`severity=high`
- 阻塞时长前置（条件必配）：当 RCACT 为阻塞类动作时，BLKTIMER 必配（0~1440 分钟）。`severity=high`
- 初始记录：系统部署完成后已存在初始记录，本命令按需覆盖或新增。`severity=low`
- 前置校验：CCTMPLTNAME 须已由 [ADD CCT](task/UNC/20.15.2/0-00009.md) 配置。`severity=high`
- FUI 联动：当 [SET CNVRGDCHGPARA](task/UNC/20.15.2/0-00165.md) 的 IGNORERSPAVP=FUI 时，忽略 CHF 响应的 FinalUnitIndication，按本命令的 RCACT 执行；本命令未配置时执行默认动作。`severity=medium`
