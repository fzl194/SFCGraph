# 增加静态地址黑名单（ADD BLACKLIST）

> 命令静态知识见 [ADD BLACKLIST](command/UNC/20.15.2/ADD-BLACKLIST.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令将一个 IP 地址段设为静态地址黑名单，禁止携带该地址段内 IP 的用户入网（防止静态地址冲突/盗用）。配置空间主要由两个维度决定：**IP 版本**（IPv4/IPv6）和**是否绑定 VPN**。

### 配置维度1：IP 版本（参数 IPVERSION）

| 取值 | 作用 | 配套必选参数 | 典型场景 |
|---|---|---|---|
| IPv4（默认） | IPv4 黑名单 | V4STARTIP + V4ENDIP | IPv4 静态地址黑名单 |
| IPv6 | IPv6 黑名单 | V6PREFIXSTART + V6PREFIXEND | IPv6 静态地址黑名单 |

> IPv4 地址段每个最多 32768 个地址；IPv6 地址段每个最多 1048576 个地址（前缀默认 64 bit）。

### 配置维度2：是否绑定 VPN（参数 HASVPN / VPNINSTANCE）

| 取值 | 作用 | 配套参数 | 典型场景 |
|---|---|---|---|
| Disable（默认） | 全局黑名单（不绑 VPN） | — | 全局静态地址黑名单 |
| Enable | 绑定 VPN 实例 | VPNINSTANCE（必填） | VPN 内静态地址黑名单 |

> VPNINSTANCE 必须是 `ADD VPNINST` 已配置的 VPN 实例。

**步骤位置**：在所属 feature（WSFD-010502 地址分配方式）编排中，于 `ADD VPNINST`（若绑 VPN）之后执行。

## 约束

- **只对新激活用户生效**（info）：命令执行后只对新激活用户生效，已在线用户不受影响 — 后果：需用户重激活才应用黑名单
- **每 VPN 最多 20 个黑名单**（warning）：静态地址黑名单基于 VPN 实例配置，每个 VPN 实例最多可配置 20 个黑名单列表 — 后果：超规格新增失败
- **不支持 C 面分地址**（warning）：该命令不支持 C 面分地址 — 后果：C 面分地址场景不可用
- **地址段不可重叠**（critical）：同一 VPN 下，各个地址段里的地址不能相互重叠；起始 IP 必须 ≤ 结束 IP — 后果：配置冲突
- **IPv4 地址范围限制**（critical）：IPv4 地址不能为组播（224.x.y.z）和环回（127.x.y.z）；必须是 A/B/C 类 — 后果：配置失败
- **IPv6 地址范围限制**（critical）：IPv6 必须是全球单播地址；不能为::/FFFF:.../环回/链路本地/组播/IPv4 映射 — 后果：配置失败
- **最多 200 条记录**（warning）：本命令最多可输入 200 条记录 — 后果：超规格新增失败
