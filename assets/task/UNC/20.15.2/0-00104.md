# DSP CHGGA 原子任务

> 命令静态知识见 [DSP CHGGA](command/UNC/20.15.2/DSP-CHGGA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。


## 配置方法

本命令为**只读查询**，用于核对 `SET CHGGA` 修改后"数据配置表 vs 当前进程"是否一致。两种查询粒度：

| 查询粒度 | `RUNAME` | 典型脚本 | 用途 |
|---|---|---|---|
| 查询所有 CHGGA 状态 | 不填 | `DSP CHGGA:;` | 全局 Ga 接口参数巡检 |
| 查询指定 RU 上所有 CHGGA 状态 | 填 RU 名称 | `DSP CHGGA:RUNAME="ru1";` | 定位单 RU 的 Ga 参数 |

**一致性核对场景**：`SET CHGGA` 修改数据库后**不即时生效**，需重启 SPP/GTP-UPP/CDP 进程；本命令查的是"当前进程实际加载的配置"，可与 `LST CHGGA`（查数据库配置）对比，判断是否需要重启进程使新配置生效。

**典型脚本**（全局巡检）：
```
DSP CHGGA:;
```

**典型脚本**（指定 RU）：
```
DSP CHGGA:RUNAME="USN_SP_RU_0067";
```

**步骤位置**：SET CHGGA 修改后的一致性核对（SET CHGGA → 重启进程 → DSP CHGGA 核查）

## 决策点

### 决策点：查询范围选择（DP 0-01246）

问：查全系统所有 CHGGA 状态，还是指定 RU？

| 选项 | `RUNAME` | 适用场景 |
|---|---|---|
| 全系统查询（不填参数） | 不填 | 全局巡检 Ga 接口参数一致性 |
| 指定 RU 查询 | 填值（0~63位字符串） | 精准定位单台 RU 上的 Ga 参数 |

> 由运维意图决定（巡检 vs 精准定位），非配置型分叉，建议绑定为 decision_driven（intent 驱动）。

### 决策点：是否带 RUNAME 参数（DP 0-01247）

| 选项 | 参数填写 | 联动影响 |
|---|---|---|
| 不带 RUNAME（默认） | 留空 | 查所有 RU 上的 CHGGA 状态 |
| 带 RUNAME | 填入目标 RU 名称 | 只查该 RU 上的 CHGGA 状态 |

## 约束

- **执行后立即生效**（info，rule-0-01551）：执行后立即生效
- **只能查询 GTP/UPP**（warning，rule-0-01552）：只能查询 GTP/UPP、SPP、CDP 三类进程的 CHGGA 状态，其他进程不可查
- **不输入参数 = 查所有**（info，rule-0-01553）：不输入参数 = 查所有；输入 RUNAME = 查该 RU 全部
- **SET CHGGA 修改后不即时生效**（info，rule-0-01554）：SET CHGGA 修改后不即时生效，需重启对应进程；本命令查的是进程实际配置，与数据库配置可能不一致
- **RUNAME 为 0~63 位字符串**（info，rule-0-01555）：RUNAME 为 0~63 位字符串，optional
