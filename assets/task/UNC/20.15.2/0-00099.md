# ADD VPNINSTAF 原子任务
> 命令静态知识见 [ADD VPNINSTAF](command/UNC/20.15.2/ADD-VPNINSTAF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令在"平台服务管理/VNRS/L3VPN管理/L3VPN实例地址族"下，紧跟 `ADD L3VPNINST`（建 L3VPN 实例）之后执行——使能地址族后才能做该地址族下的 VPN 路由、IPsec/GRE 隧道等配置。注意 VRFNAME 引用的是 L3VPNINST 建的实例，不是业务层 `ADD VPNINST` 建的实例。

## 配置方法
按"VRF + 地址族"二元组配置，多数场景仅填两个必选参数；AAA/CG/OCS 等需跨 VPN 路由的场景才补 VRFRD/VRFLABELMODE。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| VRFNAME | 已配置数据中获取（来自 `ADD L3VPNINST`，可用 `LST L3VPNINST` 查） | `vrf1`、`vpn_pdn`、`vpn_aaa`、`vpn_ga`、`vpn_gy`、`vpn_3gpp_aaa`、`vpn1` |
| AFTYPE | 本端规划（按隧道地址族） | `ipv4uni`、`ipv6uni` |
| VRFRD | 本端规划（路由标识，跨 VPN 路由场景必填） | `100:1`、`200:1`、`65512:2`、`1050:1`、`2050:1` |
| VRFLABELMODE | 本端规划（标签分配模式，跨 VPN 路由场景常用） | `perRoute` |

典型脚本（最小两参，IPsec 隧道）：
```
ADD VPNINSTAF:VRFNAME="vrf1", AFTYPE=ipv4uni;
```

带 RD 的完整脚本（AAA 跨 VPN 路由）：
```
ADD VPNINSTAF: VRFNAME="vpn_pdn",AFTYPE=ipv4uni,VRFRD="100:1", VRFLABELMODE=perRoute;
```

VRFRD 常通过后续 `MOD VPNINSTAF` 单独设置（如 `vpn_ga` 先 ADD 再 MOD VRFRD="1050:1"），ADD 阶段可不填。

## 决策点

### 地址族类型（IPv4 / IPv6）
- 问什么：AFTYPE 选 ipv4uni 还是 ipv6uni。
- 由隧道地址族决定：IPv4 IPsec 隧道 → `ipv4uni`；IPv6 IPsec 隧道 → `ipv6uni`；IPv6 前缀代理 → `ipv6uni`。同一 VRF 若双栈需建两条（ipv4uni + ipv6uni）。

### 是否需要 VRFRD / VRFLABELMODE
- 问什么：ADD 阶段是否补 RD 和标签模式。
- 简单 IPsec/GRE 隧道（点对点，不跨 VPN 路由）只填 VRFNAME+AFTYPE；AAA/CG/OCS 等需 BGP/MPLS 跨 VPN 引入路由的场景需补 VRFRD（如 `100:1`）和 VRFLABELMODE（`perRoute`）。RD 也可后置用 `MOD VPNINSTAF` 补。

### VRFNAME 来源
- 问什么：VRFNAME 取自哪里。
- 标"已配置数据中获取"——必须来自前置 `ADD L3VPNINST` 已建的实例名，可用 `LST L3VPNINST` 查询。配置生成应从已建数据派生，不能凭空命名。

## 约束

- **立即生效** · severity=LOW。
- **最大记录数 16378** · severity=MEDIUM。
- **前置依赖**：VRFNAME 指定的 VPN 实例必须已通过 `ADD L3VPNINST` 创建 · severity=HIGH。
- **保留实例禁用**：不能给 `__mpp_vpn_inner__`、`__mpp_vpn_inner_server__` 添加地址族 · severity=HIGH。
- **VRFNAME 格式**：1~31 字符 · severity=LOW。
- **VRFRD 格式**：3~21 字符 · severity=LOW。
- **VRFLABEL conditional**：VRFLABEL 取值 16~32767，仅当 VRFLABELMODE 为特定模式时才需填 · severity=MEDIUM。
- **同一 VRF+AF 唯一**：同一 VRF 同一地址族不能重复 ADD · severity=MEDIUM。
