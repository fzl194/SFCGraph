## 配置方法

按"VRF + 地址族"二元组配置，多数场景仅填两个必选参数；AAA/CG/OCS 等需跨 VPN 路由的场景才补 VRFRD/VRFLABELMODE。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| L3VPN 实例 | VRFNAME | 1~31 字符，来自 `ADD L3VPNINST` 已建实例 | 无 | 指定 L3VPN 实例名称 | 所有场景 |
| 地址族 | AFTYPE | `ipv4uni`（IPv4 单播）/ `ipv6uni`（IPv6 单播） | 无 | 指定地址族类型 | 所有场景 |
| 路由标识 | VRFRD | 3~21 字符，格式：`X.X.X.X:number` 或 `number:number`；不可为 `0:0` 或 `0.0:0` | 无（可选） | 跨 VPN 路由场景区分相同网段的不同 VPN | AAA/CG/OCS 等需 BGP/MPLS 跨 VPN 引入路由的场景 |
| 标签分配模式 | VRFLABELMODE | `perRoute`（每路由每标签）/ `perInstance`（每实例每标签）/ `perInstanceStatic`（每实例每静态标签） | `perRoute` | 标签资源优化，跨 VPN 路由场景常配 `perInstance` | AAA/CG/OCS 等需跨 VPN 路由场景 |
| 静态标签值 | VRFLABEL | 16~32767 | 无（条件可选） | 仅当 `VRFLABELMODE=perInstanceStatic` 时需填 | 静态标签分配场景 |

典型脚本（最小两参，IPsec 隧道）：
```
ADD VPNINSTAF:VRFNAME="vrf1", AFTYPE=ipv4uni;
```

带 RD 的完整脚本（AAA 跨 VPN 路由）：
```
ADD VPNINSTAF: VRFNAME="vpn_pdn",AFTYPE=ipv4uni,VRFRD="100:1", VRFLABELMODE=perRoute;
```

步骤位置：在"平台服务管理/VNRS/L3VPN管理/L3VPN实例地址族"下，紧跟 `ADD L3VPNINST`（建 L3VPN 实例）之后执行——使能地址族后才能做该地址族下的 VPN 路由、IPsec/GRE 隧道等配置。注意 VRFNAME 引用的是 L3VPNINST 建的实例，不是业务层 `ADD VPNINST` 建的实例。

## 决策点

### 决策点：地址族类型（IPv4 / IPv6）（DP 0-01236）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| `ipv4uni` | IPv4 IPsec 隧道 / IPv4 GRE 隧道 | IPv4 隧道场景 |
| `ipv6uni` | IPv6 IPsec 隧道 / IPv6 前缀代理 | IPv6 隧道场景 |
| 双栈（两条） | `ipv4uni` + `ipv6uni` 同一 VRF | 同时支持 IPv4/IPv6 的双栈场景 |

### 决策点：是否需要 VRFRD / VRFLABELMODE（DP 0-01237）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅填 VRFNAME + AFTYPE | 无 RD/LABELMODE | 简单 IPsec/GRE 隧道（点对点，不跨 VPN 路由） |
| 补 VRFRD + VRFLABELMODE | VRFRD=`X:X` 或 `X.X.X.X:number`，VRFLABELMODE=`perInstance` | AAA/CG/OCS 等需 BGP/MPLS 跨 VPN 引入路由的场景 |

简单 IPsec/GRE 隧道（点对点，不跨 VPN 路由）只填 VRFNAME+AFTYPE；AAA/CG/OCS 等需 BGP/MPLS 跨 VPN 引入路由的场景需补 VRFRD（如 `100:1`）和 VRFLABELMODE（`perInstance`）。RD 也可后置用 `MOD VPNINSTAF` 补。

VRFNAME 来源：必须来自前置 `ADD L3VPNINST` 已建的实例名，可用 `LST L3VPNINST` 查询。配置生成应从已建数据派生，不能凭空命名。
