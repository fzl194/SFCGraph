# UNC 4G/2-3G 双栈全局开关

> **UNC 域 4G/2-3G IPv4v6 双栈专属骨架**（SOP §3.5 特性专属）。被引用于 **WSFD-104002**（IPv4v6 双栈）的 4G SGSN/MME + 2-3G SGSN/GGSN 制式。
>
> **5G 不使用**——5G AMF/SMF 由 SMF/UDM 签约驱动双栈，不需此骨架。
>
> **C-U 对应**：UDG 域用户面无对应控制面开关，UDG/UPF 接收 PFCP 信令透传双栈。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 开启 SMF 双栈全局开关（4G/2-3G SGSN/MME 必选） | `SET SMFUNC` | DUALFLAG=YES（IPv4v6 双栈使能）|
| 2. 签约数据纠正（4G/2-3G 可选，仅当 Gr/S6a 连接的 HLR/HSS 不支持 IPv4v6 双栈时） | `ADD SMSUBDATA` | PDPTYPE=IPV4, NEWPDPTYPE=IPV4V6, IPV4PDPADDRTYPE/NEWIPV4PDPADDRTYPE/NEWIPV6PDPADDRTYPE=DYNAMIC|

**典型脚本**：
```
SET SMFUNC: DUALFLAG=YES;
ADD SMSUBDATA: PDPTYPE=IPV4, NEWPDPTYPE=IPV4V6, IPV4PDPADDRTYPE=DYNAMIC, NEWIPV4PDPADDRTYPE=DYNAMIC, NEWIPV6PDPADDRTYPE=DYNAMIC;
```

**步骤位置**：在 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施（VPN+APN）+ [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 地址池 + [UNC 1-00029](task/UNC/20.15.2/1-00029.md)/[1-00032](task/UNC/20.15.2/1-00032.md) 路由发布之后；仅 4G/2-3G 制式执行。

## 场景差异

- **[WSFD-104002](task/UNC/20.15.2/2-00017.md) IPv4v6 双栈接入**：
  - 5G AMF/SMF：**不执行**本骨架（5G 由 SMF/UDM 签约驱动）
  - 4G SGSN/MME：步骤 1 必选 + 步骤 2 可选（HSS 不支持时）
  - 4G/5G PGW-C/SMF：不执行 SET SMFUNC（PGW-C 独立 License）
  - 2-3G SGSN：步骤 1 必选 + 步骤 2 可选（HLR 不支持时）

## 决策点

| 选项 | 影响 |
|---|---|
| 4G 静态双栈（HSS 不支持）| 步骤 1+2 全配 |
| 4G 动态双栈（HSS 支持）| 步骤 1（仅 DUALFLAG）|
| 2-3G 静态双栈（HLR 不支持）| 步骤 1+2 |
| 2-3G 动态双栈（HLR 支持）| 步骤 1 |
| 5G（AMF/SMF/PGW-C）| **不执行**本骨架 |

## 约束

- **5G 不执行本骨架**（critical）：`SET SMFUNC` 仅适用 4G/2-3G SGSN/MME — 5G 配置无效
- **签约数据纠正的 APNNI 范围**（critical）：`ADD SMSUBDATA` 的 APNNIRANGE 决定纠正生效范围；全网规划时取 `APNNI_ALL`（默认值）或精确匹配
- **签约数据纠正参数一致性**（warning）：`PDPTYPE=IPV4` + `NEWPDPTYPE=IPV4V6` 是 IPv4 单栈改双栈的标准组合；若原签约是 IPv6 则 `PDPTYPE=IPV6` + `NEWPDPTYPE=IPV4V6`
