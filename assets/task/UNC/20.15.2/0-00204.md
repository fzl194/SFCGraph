# SET STGALARMCTRL 原子任务

> 命令静态知识见 [SET STGALARMCTRL](command/UNC/20.15.2/SET-STGALARMCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET STGALARMCTRL 控制 SMF/PGW-C 上报 ALM-81025（话单缓存）告警的两个参数：占用空间门限（ALMTHD）和监控时长（MONITORTIME）。属于融合计费消息缓存配置链的一环（在 SET CDRSTORAGECTRL 之后、SET N40MSGSTG 与 SET STGTRIGGER 配套）。配置值通常本端规划，给出明确门限与时长即可。

**参数取值样例（融合计费缓存告警控制范式）**

| 参数 | 取值样例 | 默认（初始记录） | 范围 | 说明 |
|---|---|---|---|---|
| ALMTHD | `5120` | 5120 | 0~1048576 (KB) | 告警产生门限；缓存文件占用超此阈值上报 ALM-81025 |
| MONITORTIME | `60` | 60 | 0~65535 (分钟) | 告警监控时长；0 表示改由软参 DWORD507 控制扫描周期 |

**典型行（融合计费消息缓存场景，与系统初始值一致）**

```
SET STGALARMCTRL: ALMTHD=5120, MONITORTIME=60;
```

**典型流程（融合计费消息缓存配置链）**：SET CDRSTORAGECTRL → SET STGALARMCTRL（本命令）→ SET N40MSGSTG → SET STGTRIGGER → SET CNVRGDCHGPARA。

## 决策点

### MONITORTIME 配 0 还是显式值

| 选项 | 取值 | 含义 |
|---|---|---|
| 显式监控时长 | 60（推荐 ≥ N40MSGSTG 最小回放间隔×2） | SMF 检查到缓存存在且未超门限，超过此时长上报告警 |
| 改由软参控制 | 0 | 监控时长退回由软参 DWORD507（硬盘缓存文件扫描周期）控制 |

推荐显式值且配置为 SET N40MSGSTG 融合计费消息回放最小间隔的 2 倍以上。

### ALMTHD 门限高低权衡

门限过低会频繁告警；门限过高可能导致缓存堆积到危险水位才告警。需结合硬盘容量与回放速率本端规划。

## 约束

- **立即生效** · severity=LOW。
- **MONITORTIME 建议为 N40MSGSTG 最小回放间隔的 2 倍以上** · severity=MEDIUM · 配小会导致误告警。
- **MONITORTIME=0 时退回软参 DWORD507 控制** · severity=MEDIUM · DWORD507 控制扫描硬盘缓存文件的周期。
- **ALMTHD 控制的是 ALM-81025 的空间阈值** · severity=LOW · 缓存全部回放完毕后清除告警。
- **存在系统初始记录**（ALMTHD=5120, MONITORTIME=60） · severity=LOW · 按需覆盖。
- **未输入的参数保持系统当前配置不变** · severity=MEDIUM · 非覆盖语义。
