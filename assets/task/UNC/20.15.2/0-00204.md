# 配置融合计费话单缓存告警上报控制参数（SET STGALARMCTRL）

> 命令静态知识见 [SET STGALARMCTRL](command/UNC/20.15.2/SET-STGALARMCTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET STGALARMCTRL 是会话管理层计费缓存子模块的**告警上报控制**命令，单实例（全局一份），针对融合计费话单缓存告警（ALM-81025）。共 2 个参数：ALMTHD（告警产生门限 KB）与 MONITORTIME（告警监控时长 分钟），均为 optional，未输入保持当前值。系统初始 ALMTHD=5120 KB、MONITORTIME=60 分钟。本命令所有参数均为本端规划；执行后立即生效。文件占用超过 ALMTHD 阈值时上报告警，缓存文件全部回放后清除告警；缓存文件存在但不超过阈值时，超过 MONITORTIME 时长后才上报。本命令有 2 个配置维度：**告警产生门限**与**告警监控时长**。

### 配置维度 1：告警产生门限（参数 ALMTHD，optional）

| 取值 | 作用 | 联动参数 | 典型场景 |
|---|---|---|---|
| `ALMTHD=5120`（默认） | 缓存文件占用空间阈值（KB 单位），超过该阈值上报告警 | `MONITORTIME` | WSFD-011206 全网规划默认（5120 KB ≈ 5 MB） |
| `ALMTHD=<n>` | n 取 0~1048576 KB（整数，约 0~1 GB）；缓存文件占用超过此值时上报 ALM-81025 | — | 按业务容量调整缓存上限，阈值越低越早告警 |
| `ALMTHD=0` | 不基于占用空间阈值触发告警（仅 MONITORTIME 时长控制） | `MONITORTIME` | 关闭阈值告警，仅靠时长控制 |

> ALMTHD 单位为千字节（KB），取值 0~1048576；阈值越低，越早触发 ALM-81025 告警。

### 配置维度 2：告警监控时长（参数 MONITORTIME，optional）

| 取值 | 作用 | 联动参数 | 典型场景 |
|---|---|---|---|
| `MONITORTIME=60`（默认） | 缓存文件存在但不超过占用空间阈值时，超过 60 分钟（默认值）后上报告警 | `ALMTHD` | WSFD-011206 全网规划默认（60 分钟） |
| `MONITORTIME=<n>` | n 取 0~65535 分钟（整数，约 0~约 45.5 天），超过此时间后上报；建议为 [SET N40MSGSTG](task/UNC/20.15.2/0-00184.md) REPLAYINTERVAL 的 2 倍以上 | `ALMTHD` | 按回放间隔时长配告警监控时长 |
| `MONITORTIME=0` | 告警监控时长受软参 DWORD507 控制（DWORD507 控制 SMF 检查硬盘上缓存文件的扫描周期） | [SET SMFSOFTPARA](task/UNC/20.15.2/0-00203.md) `DT=Dw, DWORDNUM=507` | 用平台软参定时扫描代替本命令 MONITORTIME |

> MONITORTIME 单位为分钟，取值 0~65535；0 时由 SMFSOFTPARA DWORD507 控检查周期。建议为 N40MSGSTG REPLAYINTERVAL 的 2 倍以上以避免缓存未达阈值时漏报。

**步骤位置**：融合计费特性（WSFD-011206）激活流程中，作为"缓存告警配置"步骤——位于 [SET N40MSGSTG](task/UNC/20.15.2/0-00184.md)（缓存开关 + REPLAYINTERVAL）+ [SET STGTRIGGER](task/UNC/20.15.2/0-00205.md)（缓存期 Trigger）+ [SET CNVRGDCHGPARA](task/UNC/20.15.2/0-00165.md)（CHGDATAREFGEN=SMF）+ [SET GLBDFTCHFGROUP](task/UNC/20.15.2/0-00176.md)（全局默认 CHF 组）+ [SET CDRSTORAGECTRL](task/UNC/20.15.2/0-00153.md)（缓存文件超期）之后。

## 决策点

### 决策点：告警监控时长配置策略（DP 0-04021）

问：MONITORTIME 走默认值、回放间隔联动还是平台软参？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 默认值 | `ALMTHD=5120, MONITORTIME=60`（系统初始） | WSFD-011206 全网规划默认；通用部署 |
| 与回放间隔联动 | `MONITORTIME ≥ 2 × REPLAYINTERVAL`（`SET N40MSGSTG`） | 缓存回放间隔较长场景，避免缓存未达阈值漏报 |
| 软参定时控制 | `MONITORTIME=0` + [SET SMFSOFTPARA](task/UNC/20.15.2/0-00203.md) `DWORD507=<scan_cycle>` | 用平台软参 DWORD507 的扫描周期来控制告警监控时长 |

> 取值由缓存回放频率决定；MONITORTIME 单位为分钟，配置过大延后告警、过小则频繁告警。

### 决策点：告警阈值与时长组合（DP 0-04022）

问：ALMTHD 与 MONITORTIME 是否同时调整？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅配 ALMTHD | `ALMTHD=<n>, MONITORTIME=60`（默认） | 业务容量较大，需降低告警阈值；时长保留默认 |
| 仅配 MONITORTIME | `ALMTHD=5120`（默认）, `MONITORTIME=<n>` | 时长按回放间隔联动调整 |
| 双参配合 | `ALMTHD=<n>, MONITORTIME=<m>`（推荐 m ≥ 2 × REPLAYINTERVAL） | 容量与告警时机独立调整；典型 WSFD-011206 完整配置 |
| 关闭阈值告警 | `ALMTHD=0`，`MONITORTIME=<m>` 或 `MONITORTIME=0`+DWORD507 | 仅靠时长/扫描周期控制，关闭占用空间阈值 |

> 0 值含义不同：ALMTHD=0 表示"不基于占用空间阈值告警"；MONITORTIME=0 表示"由 DWORD507 软参控制检查周期"。

## 约束

- **立即生效**（critical，rule-0-04051）：本命令执行后立即生效（系统初始记录基础上覆盖），未输入参数保持系统当前值不变 — 后果：每次仅修改指定阈值/时长，未指定保留当前值
- **MONITORTIME 建议 ≥ 2×REPLAYINTERVAL**（critical，rule-0-04052）：MONITORTIME 建议为 [SET N40MSGSTG](task/UNC/20.15.2/0-00184.md) REPLAYINTERVAL 的 2 倍以上；低于该值时未达阈值的缓存文件未及时监控 — 后果：回放间隔覆盖下告警监控失效
- **MONITORTIME=0 转移 DWORD507**（warning，rule-0-04053）：MONITORTIME 配置为 0 时，告警监控时长受软参 DWORD507 控制（DWORD507 控制扫描硬盘上缓存文件的周期） — 后果：MONITORTIME=0 时实际扫描周期由 DWORD507 决定，须确保 DWORD507 已配置
- **ALMTHD 单位 KB**（warning，rule-0-04054）：ALMTHD 单位为千字节（KB），取值 0~1048576；配置时确认单位为 KB 而非字节 — 后果：单位混淆导致告警阈值偏差
- **缓存前提（继承 1-00011）**（warning，rule-0-04055）：缓存前提（与 [1-00011](task/UNC/20.15.2/1-00011.md) 同步）必须同时满足——SET FAILHANDLING FHACTION=CONTINUE + MOD PDUSCACT SCACT=FAILOVER/CONTINUE + SET N40MSGSTG STGSWITCH=ENABLE + SET CNVRGDCHGPARA CHGDATAREFGEN=SMF——任一不满足缓存不生效，本命令告警也无法触发 — 后果：缓存前提缺失时本命令配置存在但缓存不生效
