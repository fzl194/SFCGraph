# ADD DIAMCONNGRP 原子任务

> 命令静态知识见 [ADD DIAMCONNGRP](command/UNC/20.15.2/ADD-DIAMCONNGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD DIAMCONNGRP 用 `LOCALHOSTNAME + PEERHOSTNAME + APPLICATION` 唯一确定一组 Diameter 链路，并通过 `SELECTMODE` 决定组内多条链路（由 ADD DIAMCONNECTION 建）间的选择模式。建链路组前 LOCALHOSTNAME 须已用 [ADD DIAMLOCINFO](task/UNC/20.15.2/0-00030.md) 配好，PEERHOSTNAME 须已用对应 APPLICATION 类型的服务器命令（[ADD PCRF](command/UNC/20.15.2/ADD-PCRF.md) / [ADD OCS](command/UNC/20.15.2/ADD-OCS.md) / [ADD DRA](command/UNC/20.15.2/ADD-DRA.md) / ADD DIAMETERAAA）配好。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| CONNGROUPNAME | `dconn_gx` / `dconn_pcrf1` / `dia1` / `dconn_s6b1` | 本端规划 | 链路组名，全局唯一 |
| LOCALHOSTNAME | `unc_1` / `pgwc_1` / `pgw_1` | 全网规划 | 必须已 ADD DIAMLOCINFO 配好 |
| APPLICATION | `GX` / `GY` / `S6B` | 全网规划 | 与对端服务器 APPLICATION 类型必须一致 |
| PEERHOSTNAME | `pcrf_1` / `ocs` / `host1.example.com` / `3gppaaa` | 本端规划或与对端协商 | 必须已建对应类型对端服务器 |
| SELECTMODE | `SESSION_ID` / `ROUND_ROBIN` / `MASTER_SLAVE` | 全网规划 | 缺省 SESSION_ID |

**特性场景取值对比**

| 特性场景 | CONNGROUPNAME | LOCALHOSTNAME | APPLICATION | PEERHOSTNAME | SELECTMODE | 说明 |
|---|---|---|---|---|---|---|
| Gx 直连 PCRF（WSFD-109101） | dconn_pcrf1/dconn_pcrf2 | pgw_1/pgwc_1/pgwc_2 | GX | pcrf_1/pcrf_2 | SESSION_ID / ROUND_ROBIN | 多 PGW-U 用户 IP 重叠时建 4 个链路组 |
| Gx over SCTP（WSFD-104508） | dconn_gx | unc_1 | GX | pcrf | SESSION_ID | 本端 1 端点对对端 2 端点建 2 偶联 |
| Gy over SCTP（WSFD-104508） | dconn_gy | unc_1 | GY | ocs | SESSION_ID | — |
| Gy 直连 OCS（WSFD-109001） | dia1/dia2 | pgw1 | GY | ocs1 | MASTER_SLAVE | 主备模式 |
| Gx over DRA（WSFD-011132） | dconn_gx1/dconn_gx2 | unc_1 | GX | host1.example.com/host2.example.com | SESSION_ID | 多 DRA 多链路组 |
| Gy over DRA（WSFD-011133） | dconn_dra1/dconn_dra2 | unc_1 | GY | host1.example.com/host2.example.com | MASTER_SLAVE | — |
| S6b over DRA（WSFD-011134） | dconn_s6b1/dconn_s6b2 | pgw_1/pgw_2 | S6B | host1.example.com/host2.example.com | SESSION_ID（缺省） | — |
| S6b 直连 3GPP AAA（WSFD-010102） | dconn_s6b | unc_1 | S6b | 3gppaaa | — | — |

## 决策点

### SELECTMODE 链路选择模式

链路组内多条链路（由 ADD DIAMCONNECTION 建）间的选择模式，直接决定故障倒换行为：

| SELECTMODE | 含义 | 适用 |
|---|---|---|
| SESSION_ID | 基于 session-id 负荷分担（用户粒度） | 同一用户消息走同一链路；多用于直连 PCRF/OCS |
| ROUND_ROBIN | 基于 round-robin 负荷分担（消息粒度） | 轮选，逐消息分摊 |
| MASTER_SLAVE | 主备 | 第一条链路（ADD DIAMCONNECTION 首条）为主用，故障切备用 | 缺省 SESSION_ID |

> 同样的 SELECTMODE 在 [ADD DIAMRTREALM](command/UNC/20.15.2/ADD-DIAMRTREALM.md) 也用于 DRA 路由间选择；本命令仅控链路组内链路间选择。

### APPLICATION 与 PEERHOSTNAME 类型一致性

APPLICATION 必须与 PEERHOSTNAME 已建的对端服务器类型一一对应：

| APPLICATION | 对端类型 | 对端配置命令 |
|---|---|---|
| GX | PCRF | [ADD PCRF](command/UNC/20.15.2/ADD-PCRF.md) |
| GY | OCS | [ADD OCS](command/UNC/20.15.2/ADD-OCS.md) |
| S6B | 3GPP AAA / DRA | ADD DIAMETERAAA / [ADD DRA](command/UNC/20.15.2/ADD-DRA.md) |

## 约束

- **立即生效** · severity=LOW。
- **最大 1000 条记录** · severity=LOW。
- **LOCALHOSTNAME 前置依赖** · severity=HIGH · 必须先 ADD DIAMLOCINFO 配 Local Info。
- **PEERHOSTNAME 前置依赖** · severity=HIGH · 必须已建与 APPLICATION 对应类型的对端服务器，否则配置失败。
- **APPLICATION 与对端主机应用类型必须一致** · severity=HIGH · 类型不一致被拒。
- **PEERHOSTNAME 与 LOCALHOSTNAME 不能重复** · severity=HIGH · 同名配置失败。
- **DRA 与 PCRF/OCS/AAA 主机名不能重复** · severity=HIGH · over-DRA 场景特别注意。
- **每个 Peer 与 Gx/Gy 应用最多配有限链路组** · severity=MEDIUM · 超规格配置失败。
