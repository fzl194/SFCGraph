# 配置Diameter链路组（ADD DIAMCONNGRP）

> 命令静态知识见 [ADD DIAMCONNGRP](command/UNC/20.15.2/ADD-DIAMCONNGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。LOCALHOSTNAME 引 [0-00030](task/UNC/20.15.2/0-00030.md) ADD DIAMLOCINFO（本端主机）；PEERHOSTNAME 引对应 APPLICATION 类型的对端服务器命令（[ADD PCRF](command/UNC/20.15.2/ADD-PCRF.md) / [ADD OCS（★R1.5 atom 待补）](command/UNC/20.15.2/ADD-OCS.md) / [ADD DRA（★R1.5 atom 待补）](command/UNC/20.15.2/ADD-DRA.md) / ADD DIAMETERAAA（★R1.5 atom 待补））。

## 配置方法

ADD DIAMCONNGRP 用 `LOCALHOSTNAME + PEERHOSTNAME + APPLICATION` 三元组唯一确定一组 Diameter 链路，并通过 `SELECTMODE` 决定组内多条链路（由 [0-00029](task/UNC/20.15.2/0-00029.md) ADD DIAMCONNECTION 建）间的选择模式。建链路组前 LOCALHOSTNAME 须已配好（[0-00030](task/UNC/20.15.2/0-00030.md) ADD DIAMLOCINFO），PEERHOSTNAME 须已建对应 APPLICATION 类型的对端服务器。核心配置维度有 2 个：**Diameter 应用类型**（APPLICATION 决定走 Gx/Gy/S6b 接口及对端服务器类型）与**链路选择模式**（SELECTMODE 决定组内多链路的故障倒换/负荷分担行为）。

### 配置维度 1：Diameter 应用类型（参数 APPLICATION，必选）

APPLICATION 必须与 PEERHOSTNAME 已建的对端服务器类型一一对应：

| 取值 | 含义 | 对端类型 | 对端配置命令 |
|---|---|---|---|
| **GX** | Gx 接口应用（PCC 策略控制） | PCRF | [ADD PCRF](command/UNC/20.15.2/ADD-PCRF.md) |
| **GY** | Gy 接口应用（在线计费） | OCS | [ADD OCS（★R1.5 atom 待补）](command/UNC/20.15.2/ADD-OCS.md) |
| **S6B** | S6b 接口应用（漫游认证） | 3GPP AAA / DRA | ADD DIAMETERAAA（★R1.5 atom 待补） / [ADD DRA（★R1.5 atom 待补）](command/UNC/20.15.2/ADD-DRA.md) |

> over-DRA 场景下 PEERHOSTNAME 为 DRA 主机名（如 `host1.example.com`），DRA 与 PCRF/OCS/AAA 主机名不能重复。

### 配置维度 2：链路选择模式（参数 SELECTMODE，可选，缺省 SESSION_ID）

SELECTMODE 决定链路组内多条链路（ADD DIAMCONNECTION 建）间的选择模式，直接影响故障倒换行为：

| 取值 | 含义 | 适用场景 |
|---|---|---|
| **SESSION_ID**（缺省） | 基于 session-id 负荷分担（用户粒度） | 同一用户消息走同一链路；多用于直连 PCRF/OCS |
| **ROUND_ROBIN** | 基于 round-robin 负荷分担（消息粒度） | 逐消息轮选分摊 |
| **MASTER_SLAVE** | 主备模式 | 第一条链路（ADD DIAMCONNECTION 首条）为主用，故障切备用 |

> 同样的 SELECTMODE 在 [ADD DIAMRTREALM（★R1.5 atom 待补）](command/UNC/20.15.2/ADD-DIAMRTREALM.md) 也用于 DRA 路由间选择；本命令仅控链路组内链路间选择。MASTER_SLAVE 模式下删除首条链路或主链路对端地址会导致主备切换，当前不支持命令直接指定主链路——需按顺序重建。

### 参数取值样例

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| CONNGROUPNAME | `dconn_gx` / `dconn_pcrf1` / `dia1` / `dconn_s6b1` | 本端规划 | 链路组名，全局唯一，1~31 字符 |
| LOCALHOSTNAME | `unc_1` / `pgwc_1` / `pgw_1` | 全网规划 | 必须已 ADD DIAMLOCINFO 配好，1~127 字符 |
| APPLICATION | `GX` / `GY` / `S6B` | 全网规划 | 与对端服务器 APPLICATION 类型必须一致 |
| PEERHOSTNAME | `pcrf_1` / `ocs` / `host1.example.com` / `3gppaaa` | 全网规划 | 必须已建对应类型对端服务器，1~127 字符 |
| SELECTMODE | `SESSION_ID` / `ROUND_ROBIN` / `MASTER_SLAVE` | 全网规划 | 缺省 SESSION_ID |

**典型脚本（Gx 直连 PCRF，SESSION_ID 负荷分担）**：
```
ADD DIAMCONNGRP: CONNGROUPNAME="gxconngrp", LOCALHOSTNAME="gxlocalhost", APPLICATION=GX, PEERHOSTNAME="pcrfhost";
```

**典型脚本（Gy 直连 OCS，MASTER_SLAVE 主备）**：
```
ADD DIAMCONNGRP: CONNGROUPNAME="dia1", LOCALHOSTNAME="pgw1", APPLICATION=GY, PEERHOSTNAME="ocs1", SELECTMODE=MASTER_SLAVE;
```

**步骤位置**：在 Diameter 链路 backbone 编排中位于"建链路组"步骤——前置 [0-00030](task/UNC/20.15.2/0-00030.md) ADD DIAMLOCINFO（本端主机）+ [0-00026](task/UNC/20.15.2/0-00026.md) ADD DIAMPEERADDR（对端地址）+ 对应 APPLICATION 对端服务器命令；下游 [0-00029](task/UNC/20.15.2/0-00029.md) ADD DIAMCONNECTION（在组内建链路）。

## 决策点

### 决策点：SELECTMODE 链路选择模式（DP 0-02033）

问：链路组内多条链路（由 ADD DIAMCONNECTION 建）间采用何种选择模式？（决定故障倒换行为与对端冗余要求）

| 选项 | 含义 | 适用场景 | 联动要求 |
|---|---|---|---|
| SESSION_ID（缺省） | 基于 session-id 负荷分担（用户粒度） | 同一用户消息走同一链路；直连 PCRF/OCS | 链路组内多条链路（≥2 推荐） |
| ROUND_ROBIN | 基于 round-robin 负荷分担（消息粒度） | 逐消息轮选分摊 | 链路组内多条链路（≥2 推荐） |
| MASTER_SLAVE | 主备模式 | 第一条链路为主用，故障切备用 | ADD DIAMCONNECTION 首条即主链路；删首条触发主备切换 |

> 同样的 SELECTMODE 在 [ADD DIAMRTREALM（★R1.5 atom 待补）](command/UNC/20.15.2/ADD-DIAMRTREALM.md) 也用于 DRA 路由间选择；本命令仅控链路组内链路间选择。

### 决策点：APPLICATION 与 PEERHOSTNAME 类型一致性（DP 0-02034）

问：该链路组承载哪种 Diameter 应用？（决定 PEERHOSTNAME 的对端服务器类型与前置命令）

| 选项 | 对端类型 | 对端配置命令 | 典型场景 |
|---|---|---|---|
| GX | PCRF | [ADD PCRF](command/UNC/20.15.2/ADD-PCRF.md) | Gx 接口 PCC 策略控制 |
| GY | OCS | [ADD OCS（★R1.5 atom 待补）](command/UNC/20.15.2/ADD-OCS.md) | Gy 接口在线计费 |
| S6B | 3GPP AAA / DRA | ADD DIAMETERAAA（★R1.5 atom 待补） / [ADD DRA（★R1.5 atom 待补）](command/UNC/20.15.2/ADD-DRA.md) | S6b 接口漫游认证 |

> APPLICATION 与 PEERHOSTNAME 已建的对端服务器应用类型必须一一对应，类型不一致被拒。

## 约束

- **LOCALHOSTNAME 前置依赖**（warning，rule-0-02081）：LOCALHOSTNAME 前置依赖：必须先 ADD DIAMLOCINFO 配 Local Info — 后果：本端主机不存在配置失败
- **PEERHOSTNAME 前置依赖**（warning，rule-0-02082）：PEERHOSTNAME 前置依赖：必须已建与 APPLICATION 对应类型的对端服务器（ADD PCRF/OCS/DRA/DIAMETERAAA） — 后果：对端不存在配置失败
- **APPLICATION 与对端主机应用类型必须一致**（critical，rule-0-02083）：APPLICATION 与对端主机应用类型必须一致 — 后果：类型不一致被拒
- **PEERHOSTNAME 与 LOCALHOSTNAME**（warning，rule-0-02084）：PEERHOSTNAME 与 LOCALHOSTNAME 不能重复 — 后果：同名配置失败
- **每个 Peer 与 Gx/Gy 应用最多绑定 64 个**（info，rule-0-02085）：每个 Peer 与 Gx/Gy 应用最多绑定 64 个 Local Host；最大记录数 1000 — 后果：超规格配置失败
