# UNC GRE 隧道族（GRETUNNEL+IFIPV4ADDRESS+INTERFACE+SRROUTE）

> **UNC 域 GRE 接入方式共享底座**（SOP §3.5 域通用）。被引用于 **IPFD-015002**（GRE 接入方式特性）。
>
> **C-U 对应**：UNC 控制面承担 GRE 隧道建立与监控职责，与 UDG 用户面 GRE 端**结构同构但命令族独立维护**（UNC 无 `ADD GRETUNNEL` 复用 UDG atom，UNC atom 编号 0-00227/0-00228/0-00267/0-00269）。UDG 与 UNC GRE 是**两个独立产品的资产**，不复用 md。
>
> **设计参考**：UDG 1-00032 GRE 隧道族结构（结构对齐，UNC 命令族独立维护）。

## 配置方法

### 步骤维度：本端接口 → Tunnel 接口 → 路由 → 可选增强

本 compound 在 UNC 域建立一个完整的 GRE 隧道：

1. **本端 LoopBack 接口**（GRE 隧道源地址的物理出接口）：`ADD INTERFACE` → [0-00235](task/UNC/20.15.2/0-00235.md) + `ADD IFIPV4ADDRESS` → [0-00228](task/UNC/20.15.2/0-00228.md)
2. **Tunnel 接口**（GRE 隧道逻辑接口）：`ADD GRETUNNEL` → [0-00227](task/UNC/20.15.2/0-00227.md) + Tunnel 接口 IP 地址 `ADD IFIPV4ADDRESS` → [0-00228](task/UNC/20.15.2/0-00228.md)
3. **隧道静态路由**（流量引入 GRE 隧道）：`ADD SRROUTE` → [0-00081](task/UNC/20.15.2/0-00081.md)
4. **可选增强**（按需选，三选一）：
   - 端到端校验：`MOD GRETUNNEL` → [0-00267](task/UNC/20.15.2/0-00267.md) `CHECKSUMEN=TRUE`
   - GRE 识别关键字：`MOD GRETUNNEL` → [0-00267](task/UNC/20.15.2/0-00267.md) `GREKEYEN=TRUE, GREKEY="key"`
   - Keepalive 检测：`MOD GRETUNNEL` → [0-00267](task/UNC/20.15.2/0-00267.md) `KEEPALVEN=TRUE, KEEPALVPERIOD, KEEPALVRETRYCNT`

> **说明**：LoopBack 接口与 Tunnel 接口**使用相同的 IFIPV4ADDRESS 命令**（[0-00228](task/UNC/20.15.2/0-00228.md)），通过 IFNAME 区分（本端 LoopBack vs Tunnel）；Tunnel 接口 IP 通常为 /32（本端点），LoopBack 接口 IP 也为 /32（隧道源地址）。

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 本端 LoopBack 接口 | `ADD INTERFACE` → [0-00235](task/UNC/20.15.2/0-00235.md) | IFNAME="LoopBack1" |
| 1b. LoopBack IP 地址 | `ADD IFIPV4ADDRESS` → [0-00228](task/UNC/20.15.2/0-00228.md) | IFNAME, IFIPADDR, SUBNETMASK（/32）, ADDRTYPE=main |
| 2. Tunnel 接口 | `ADD GRETUNNEL` → [0-00227](task/UNC/20.15.2/0-00227.md) | TNLNAME="Tunnel1", TNLTYPE=gre, SRCTYPE=if_name, SRCIFNAME="LoopBack1", DSTIPADDR="对端 GRE 源 IP" |
| 2b. Tunnel IP 地址 | `ADD IFIPV4ADDRESS` → [0-00228](task/UNC/20.15.2/0-00228.md) | IFNAME="Tunnel1", IFIPADDR（本端 Tunnel IP）, SUBNETMASK, ADDRTYPE=main |
| 3. 隧道静态路由 | `ADD SRROUTE` → [0-00081](task/UNC/20.15.2/0-00081.md) | AFTYPE=ipv4unicast, PREFIX, MASKLENGTH, IFNAME="Tunnel1" |
| 4a. 端到端校验（可选） | `MOD GRETUNNEL` → [0-00267](task/UNC/20.15.2/0-00267.md) | TNLNAME, TNLTYPE=gre, CHECKSUMEN=TRUE |
| 4b. GRE 识别关键字（可选） | `MOD GRETUNNEL` → [0-00267](task/UNC/20.15.2/0-00267.md) | TNLNAME, TNLTYPE=gre, GREKEYEN=TRUE, GREKEY="key" |
| 4c. Keepalive 检测（可选） | `MOD GRETUNNEL` → [0-00267](task/UNC/20.15.2/0-00267.md) | TNLNAME, TNLTYPE=gre, KEEPALVEN=TRUE, KEEPALVPERIOD=5, KEEPALVRETRYCNT=3 |

**典型脚本（普通 IPv4 GRE 隧道，2-00029 场景基线）**：
```
ADD INTERFACE:IFNAME="LoopBack1";
ADD IFIPV4ADDRESS:IFNAME="LoopBack1", IFIPADDR="10.3.3.11", SUBNETMASK="255.255.255.255", ADDRTYPE=main;
ADD GRETUNNEL:TNLNAME="Tunnel1", TNLTYPE=gre, SRCTYPE=if_name, SRCIFNAME="LoopBack1", DSTIPADDR="10.4.4.11";
ADD IFIPV4ADDRESS:IFNAME="Tunnel1", IFIPADDR="10.10.1.201", SUBNETMASK="255.255.255.0", ADDRTYPE=main;
ADD SRROUTE:AFTYPE=ipv4unicast, PREFIX="10.10.1.202", MASKLENGTH=24, IFNAME="Tunnel1";
MOD GRETUNNEL:TNLNAME="Tunnel1", TNLTYPE=gre, CHECKSUMEN=TRUE;
MOD GRETUNNEL:TNLNAME="Tunnel1", TNLTYPE=gre, GREKEYEN=TRUE, GREKEY="123";
MOD GRETUNNEL:TNLNAME="Tunnel1", TNLTYPE=gre, KEEPALVEN=TRUE, KEEPALVPERIOD=5, KEEPALVRETRYCNT=3;
```

**步骤位置**：所有 GRE 接入方式的**核心骨架**——前置 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施（VPN+APN），下游业务流经 Tunnel 接口。

## 决策点（UNC pre-build 内嵌，不编号）

| 选项 | 影响（命令 + 参数） |
|---|---|
| IPv4 GRE | `ADD GRETUNNEL` 用 SRCTYPE=if_name + DSTIPADDR；[0-00081](task/UNC/20.15.2/0-00081.md) ADD SRROUTE 用 AFTYPE=ipv4unicast |
| IPv6 GRE | `ADD GRETUNNEL` 用 SRCTYPE6=if_name + DSTIPV6ADDR；需走 ADD SRROUTE6 ([0-00258](task/UNC/20.15.2/0-00258.md) UNC 域已建) |
| 端到端校验使能 | `MOD GRETUNNEL:CHECKSUMEN=TRUE` |
| GRE 识别关键字使能 | `MOD GRETUNNEL:GREKEYEN=TRUE, GREKEY="key"`（两端必须相同或都不设置）|
| Keepalive 检测使能 | `MOD GRETUNNEL:KEEPALVEN=TRUE, KEEPALVPERIOD=5, KEEPALVRETRYCNT=3`（默认周期 5s/重试 3 次）|

## 约束

- **源/目的端路由可达**（critical）：隧道源端口和目的端口必须路由可达 — 后果：GRE 报文无法封装转发。
- **GRE 源地址与 IPSec 源地址不能相同**（critical）：同一网元上 GRE 隧道源地址不能与 IPSec 隧道源地址相同 — 后果：报文转发冲突。
- **GRE 嵌套不超过 2 层**（critical）：GRE 隧道可嵌套但最多 2 层，3 层嵌套时第三层状态会变 Down — 后果：嵌套过深报文丢弃。
- **Tunnel MTU 配置**（warning）：建议 `MOD INTERFACE` ([0-00268](task/UNC/20.15.2/0-00268.md)) 配置 Tunnel MTU ≤（出接口 MTU - GRE 头长度），GRE 头 = GRE header + 外层 IP header — 后果：MTU 过大致分片丢性能。
- **GRE 识别关键字两端一致**（warning）：若两端设置识别关键字，必须相同；若一端设置、另一端不设置，则 GRE 隧道无法建立。
- **本端 GRE 配置双侧对称**（warning）：GRE 隧道两端（本端/对端）必须配置相同 — 后果：隧道无法建立。
- **最大 4k GRE Tunnel**（warning）：整个系统最大支持 4k GRE Tunnel 接口 — 后果：超出规格新增失败。
- **本 compound 不含独立命令外的命令**：完整 GRE 接入域特性配置由 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施（VPN+APN）+ [UNC 1-00029](task/UNC/20.15.2/1-00029.md) OSPF 路由发布（骨干网路由发布）共同承担。
