# 配置Diameter链路（ADD DIAMCONNECTION）

> 命令静态知识见 [ADD DIAMCONNECTION](command/UNC/20.15.2/ADD-DIAMCONNECTION.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。DIAMCONNGRP 引 [0-00027](task/UNC/20.15.2/0-00027.md) ADD DIAMCONNGRP（链路组）；LOCINTERFACE 引 [ADD LOGICINF](command/UNC/20.15.2/ADD-LOGICINF.md)（本端逻辑接口）；对端地址引 [0-00026](task/UNC/20.15.2/0-00026.md) ADD DIAMPEERADDR。

## 配置方法

ADD DIAMCONNECTION 在已建好的 [0-00027](task/UNC/20.15.2/0-00027.md) 链路组内增加一条 Diameter 链路，绑定本端逻辑接口（LOCINTERFACE）与对端地址。对端地址通过 `PEERADDRTYPE` 二选一：**TCP 模式**走 IPv4/IPv6 地址（可选 PEERPORT），**SCTP 模式**走已用 [ADD SCTPENDPOINT](command/UNC/20.15.2/ADD-SCTPENDPOINT.md) 建好的对端 SCTP 端点名（PEERSCTPENDPT）。LOCALPORT 是否必配取决于 [SET CONCENPOINT](command/UNC/20.15.2/SET-CONCENPOINT.md) 的集中点模式。核心配置维度有 2 个：**对端地址类型**（PEERADDRTYPE 决定承载协议与后续参数集）与**本端端口模式**（LOCALPORT 由集中点模式决定是否必配）。

### 配置维度 1：对端地址类型（参数 PEERADDRTYPE，可选）

PEERADDRTYPE 决定承载协议（TCP/SCTP）、后续参数集与对端前置准备，必须与对端协商一致：

| 取值 | 承载 | 后续必选参数 | 对端前置准备 |
|---|---|---|---|
| **IPv4** | TCP | PEERIPV4ADDR（+可选 PEERPORT，缺省 3868） | 对端主机 IP 可达，禁 0.0.0.0 / 255.255.255.255 |
| **IPv6** | TCP | PEERIPV6ADDR（+可选 PEERPORT） | 对端主机 IP 可达，禁全 F |
| **SCTP** | SCTP | PEERSCTPENDPT（+可选 REVERSEIP） | 已用 [ADD SCTPENDPOINT](command/UNC/20.15.2/ADD-SCTPENDPOINT.md) 建好对端端点 |
| 不选 | — | — | 本端接口与对端主机**全部地址**建立链接（须已用 [0-00026](task/UNC/20.15.2/0-00026.md) ADD DIAMPEERADDR 配好对端多地址） |

> REVERSEIP 仅 SCTP 模式可选（缺省 DISABLE）：ENABLE 时本端逻辑接口子 IP 作偶联主用 IP、主 IP 作从 IP（对端端口为奇数时使用）。

### 配置维度 2：本端端口模式（参数 LOCALPORT，可选，缺省 0）

LOCALPORT 是否必配取决于 [SET CONCENPOINT](command/UNC/20.15.2/SET-CONCENPOINT.md) 的集中点模式：

| SET CONCENPOINT 模式 | LOCALPORT 配置 | 端口范围（按应用区分） |
|---|---|---|
| GXCONCENMODE/GYCONCENMODE/S6BCONCENMODE = LOCALPORT | **必须配非 0 有效值** | Gx：16400~16463；Gy：13200~13263；S6b：19765~19784 |
| 其他模式 | 不配或配 0（系统自动分配） | LocalPort=0 只在 LOCAL_IP_PEER 模式下生效 |

> LOCALPORT 配非 0 但集中点模式非 LOCALPORT 时，配置可下发但**不会建链**。配置了相同 LOCALPORT 的 DIAMCONNECTION 记录，所生成链路会绑定在同一个集中点进程上。

### 参数取值样例

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| DIAMCONNGRP | `dconn_gx` / `dconn_pcrf1` / `dia1` | 本端规划（依赖已建链路组） | 已 ADD DIAMCONNGRP 建好，可 LST DIAMCONNGRP 查 |
| LOCINTERFACE | `gxif1/0/0` / `gyif1/0/0` / `s6bif1/0/0` | 本端规划（依赖已建逻辑接口） | 已 ADD LOGICINF 建好，可 LST LOGICINF 查 |
| LOCALPORT | `16400`（Gx SCTP）/ `13200`（Gy SCTP）/ `16450`（Gx LOCALPORT）/ `19766`（S6b）/ `0` | 本端规划 | LOCALPORT 模式必配非 0；否则可省或 0 |
| PEERADDRTYPE | `IPv4` / `IPv6` / `SCTP` | 与对端协商 | 不选则与对端全部地址建链 |
| PEERIPV4ADDR | `10.11.21.59` | 与对端协商 | PEERADDRTYPE=IPv4 时配 |
| PEERPORT | `3868` | 与对端协商 | TCP 模式可选 |
| PEERSCTPENDPT | `pcrfendb` / `ocsend` / `endpoint0` | 与对端协商（依赖已建 SCTP 端点） | PEERADDRTYPE=SCTP 时配，已 ADD SCTPENDPOINT |
| REVERSEIP | `ENABLE` / `DISABLE` | 与对端协商 | SCTP 模式下对端端口为奇数时可 ENABLE |

**典型脚本（Gx 直连 PCRF IPv4）**：
```
ADD DIAMCONNECTION: DIAMCONNGRP="Grp1", LOCINTERFACE="gxif1/0/0", PEERADDRTYPE=IPv4, PEERIPV4ADDR="10.10.10.11", PEERPORT=3868;
```

**典型脚本（Gx over SCTP）**：
```
ADD DIAMCONNECTION: DIAMCONNGRP="dconn_gx", LOCINTERFACE="gxif1/0/0", LOCALPORT=16400, PEERADDRTYPE=SCTP, PEERSCTPENDPT="pcrfendb";
```

**步骤位置**：在 Diameter 链路 backbone 编排中位于"建链路"步骤——前置 [0-00027](task/UNC/20.15.2/0-00027.md) ADD DIAMCONNGRP（链路组）+ [0-00030](task/UNC/20.15.2/0-00030.md) ADD DIAMLOCINFO（本端信息）+ [0-00026](task/UNC/20.15.2/0-00026.md) ADD DIAMPEERADDR（对端地址）+ [ADD LOGICINF](command/UNC/20.15.2/ADD-LOGICINF.md)（本端接口）+ [ADD SCTPENDPOINT](command/UNC/20.15.2/ADD-SCTPENDPOINT.md)（SCTP 端点，SCTP 模式）+ [SET CONCENPOINT](command/UNC/20.15.2/SET-CONCENPOINT.md)（集中点模式）。

## 决策点

### 决策点：PEERADDRTYPE 对端地址类型选择（DP 0-02037）

问：该 Diameter 链路采用何种承载协议？（决定后续参数集与对端准备物，必须与对端协商一致）

| 选项 | 承载 | 后续必选参数 | 对端前置准备 |
|---|---|---|---|
| IPv4 | TCP | PEERIPV4ADDR（+可选 PEERPORT，缺省 3868） | 对端主机 IP 可达 |
| IPv6 | TCP | PEERIPV6ADDR（+可选 PEERPORT） | 对端主机 IP 可达 |
| SCTP | SCTP | PEERSCTPENDPT（+可选 REVERSEIP） | 已用 [ADD SCTPENDPOINT](command/UNC/20.15.2/ADD-SCTPENDPOINT.md) 建好对端端点 |
| 不选 | — | — | 本端接口与对端主机全部地址建立链接（须已配 ADD DIAMPEERADDR 多地址） |

### 决策点：LOCALPORT 是否必配（DP 0-02038）

问：[SET CONCENPOINT](command/UNC/20.15.2/SET-CONCENPOINT.md) 集中点模式是否为 LOCALPORT？（决定 LOCALPORT 是否必配非 0 值）

| 选项 | LOCALPORT 配置 | 端口范围 | 说明 |
|---|---|---|---|
| GXCONCENMODE/GYCONCENMODE/S6BCONCENMODE = LOCALPORT | 必须配非 0 有效值 | Gx：16400~16463；Gy：13200~13263；S6b：19765~19784 | 相同 LOCALPORT 的链路绑定同一集中点进程 |
| 其他模式 | 不配或配 0（系统自动分配） | — | LocalPort=0 只在 LOCAL_IP_PEER 模式下生效；配非 0 但非 LOCALPORT 模式不会建链 |

> 多 PGW-U 用户 IP 重叠场景（CU Full Mesh 组网）：同一 PGW-C 对接多个 PGW-U 且不同 PGW-U 下用户 IP 重叠、PCRF 要求 UE IP + GxLocalHost 唯一确定用户时，每对接一个 PCRF 须规划 2 个 Diameter 链路组（dconn_pcrf1~4），本命令相应建多条链路。

## 约束

- **DIAMCONNGRP 前置依赖**（warning，rule-0-02091）：DIAMCONNGRP 前置依赖：DIAMCONNGRP 须已用 [0-00027](task/UNC/20.15.2/0-00027.md) ADD DIAMCONNGRP 建好 — 后果：链路组不存在配置失败
- **同 ConnGrp+LocInterface**（warning，rule-0-02092）：同 ConnGrp+LocInterface 允许多条记录但地址必须不同：指定地址重复则配置失败 — 后果：地址重复配置失败
- **PEERADDRTYPE 与 SCTPENDPOINT/IP**（warning，rule-0-02093）：PEERADDRTYPE 与 SCTPENDPOINT/IP 地址联动：SCTP 模式须先 [ADD SCTPENDPOINT](command/UNC/20.15.2/ADD-SCTPENDPOINT.md)；TCP 模式须对端 IP 可达 — 后果：前置缺失建链失败
- **VPN 一致性**（warning，rule-0-02094）：VPN 一致性：本端接口的 VPN 与 Diameter 链路组中 peer 绑定的 VPN 不一致则配置失败 — 后果：VPN 不匹配配置失败
- **最大记录数 3000**（warning，rule-0-02095）：最大记录数 3000；一个 DIAMCONNGRP 最多 32 个 DIAMCONNECTION 实例；Gx/Gy/S6b 单应用整机最大链接数 200（SCTP 60）；配 DiamConnection 后再增加 Addr 不会自动建链 — 后果：超规格或仅加地址不建链
