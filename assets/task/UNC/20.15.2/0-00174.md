# 设置全局CG组（SET GLBCGGROUP）（SET GLBCGGROUP）

> 命令静态知识见 [SET GLBCGGROUP](command/UNC/20.15.2/SET-GLBCGGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET GLBCGGROUP 设置全局兜底 CG 组。UNC 选 CG 组的优先级：**离线计费模板下绑定的 CG 组 > 按号段匹配的 CG 组 > 全局 CG 组**；当全局 CG 组也未配置时回退到全局 CG（未绑定任一 CG 组的 CG，优先级最低）。本命令配的是优先级最低的全局兜底。全局唯一一条记录（最大记录数=1），系统初始记录 CGGRPID=0（不指定全局 CG 组）。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 兜底组 | CGGRPID | 0~32 | 0（初始） | CG 组 ID；0=删除/不指定全局 CG 组 | 模板/号段未覆盖用户的兜底 |

**典型脚本（设置全局 CG 组兜底）**：当 CG/CG 组侧已通过 `ADD CG（★R1.5 atom 待补）`/`ADD CGGROUP` 建好、且离线计费模板或号段匹配无法覆盖全部用户时，执行 `SET GLBCGGROUP` 指定全局兜底组，确保未命中用户也有 CG 可回放话单。

**典型脚本（删除全局 CG 组绑定）**：当模板/号段已全覆盖或需要回退到全局 CG 时，输入 `CGGRPID=0` 解除全局组绑定，恢复系统初始值。

步骤位置（证据：[配置到CG的数据（GGSN_SGW-C_PGW-C）](evidence/UNC/20.15.2/配置到CG的数据（GGSN_SGW-C_PGW-C）_95923479.md)）：在 `ADD CG（★R1.5 atom 待补）` 之后、`SET CDRTRANSFER`（CG 负荷分担算法）之前的"可选"步骤——配置 CG 信息步骤的第 6.c 子步。

## 决策点

### 决策点：是否配置全局兜底 CG 组（DP 0-03087）

取决于离线计费模板/号段是否已覆盖所有用户。本命令是"可选项"。

| 选项 | 配置 | 适用场景 |
|---|---|---|
| 配置全局兜底 | `CGGRPID=1~32` | 模板/号段无法覆盖所有用户，需兜底 |
| 不配（用初始值） | `CGGRPID=0` | 模板/号段已全覆盖，或暂不需要全局 CG 组 |

### 决策点：CGGRPID 取值来源（DP 0-03088）

| 选项 | 来源 | 说明 |
|---|---|---|
| 已建 CG 组 ID | [ADD CGGROUP](task/UNC/20.15.2/0-00012.md) 建的 ID | 1~32，必须先建组 |
| 0 | 删除/不指定 | 系统初始值，回退到全局 CG |

## 约束

- **CGGRPID 非 0 时对应 CG 组必须由 ADD**（warning，rule-0-03216）：CGGRPID 非 0 时对应 CG 组必须由 ADD CGGROUP 先建，否则引用无效

- **全局唯一一条记录（最大记录数=1）**（warning，rule-0-03217）：全局唯一一条记录（最大记录数=1），重复执行为改写而非新增

- **CGGRPID=0 表示删除/不指定全局 CG 组（系统初始值）**（warning，rule-0-03218）：CGGRPID=0 表示删除/不指定全局 CG 组（系统初始值），输入 0 即解除全局 CG 组绑定

- **优先级最低**（warning，rule-0-03219）：优先级最低：UNC 优先选离线计费模板绑定/号段匹配的 CG 组，仅都未命中时才用全局 CG 组；再不命中回退到全局 CG

- **适用 NF**（info，rule-0-03220）：适用 NF：SGW-C、PGW-C、GGSN（离线计费场景）
