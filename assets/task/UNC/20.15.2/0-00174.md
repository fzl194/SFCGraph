# SET GLBCGGROUP 原子任务

> 命令静态知识见 [SET GLBCGGROUP](command/UNC/20.15.2/SET-GLBCGGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET GLBCGGROUP 设置全局兜底 CG 组。UNC 选 CG 组的优先级：**离线计费模板下绑定的 CG 组 > 按号段匹配的 CG 组 > 全局 CG 组**。本命令配的是最低优先级的兜底。全局唯一一条记录（最大记录数=1），系统初始记录 CGGRPID=0（不指定全局 CG 组）。

**参数取值样例（离线计费特性配置范式）**

| 参数 | 取值样例 | 默认 | 说明 |
|---|---|---|---|
| CGGRPID | `1`~`32` | 0（初始） | CG 组 ID，0~32，0=删除/不指定全局 CG 组 |

**典型行（设全局 CG 组）**

```
SET GLBCGGROUP: CGGRPID=1;
```

**典型行（删除全局 CG 组）**

```
SET GLBCGGROUP: CGGRPID=0;
```

## 决策点

### 是否需要配置全局兜底 CG 组

取决于离线计费模板/号段是否已覆盖所有用户。本命令是"可选项"。

| 选项 | 配置 | 适用 |
|---|---|---|
| 配全局兜底 | `CGGRPID=1~32` | 模板/号段无法覆盖所有用户，需兜底 |
| 不配（用初始值） | `CGGRPID=0` | 模板/号段已全覆盖，或暂不需要全局 CG |

### CGGRPID 取值来源

| 选项 | 来源 | 说明 |
|---|---|---|
| 已建 CG 组 ID | [ADD CGGROUP](task/UNC/20.15.2/0-00012.md) 建的 ID | 1~32，必须先建组 |
| 0 | 删除/不指定 | 系统初始值 |

## 约束

- **只对新激活用户生效** · severity=MEDIUM · 已在线用户不受影响，需重新激活才应用。
- **最大记录数=1 / 仅 1 条全局组** · severity=HIGH · 重复执行为改写。
- **前置依赖**：CGGRPID 非 0 时，对应 CG 组必须由 [ADD CGGROUP](task/UNC/20.15.2/0-00012.md) 先建 · severity=HIGH。
- **CGGRPID=0 表示删除**：输入 0 即解除全局 CG 组绑定（系统初始值） · severity=MEDIUM。
- **优先级最低**：UNC 优先选离线计费模板绑定/号段匹配的 CG 组，仅都没命中时才用全局 · severity=MEDIUM。
