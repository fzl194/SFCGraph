# MOD APNCHGMODE 原子任务
> 命令静态知识见 [MOD APNCHGMODE](command/UNC/20.15.2/MOD-APNCHGMODE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
按"APN × 终端接入类型"维度选定计费接口轨道：Ga/Gy 模式走离线/在线 Diameter 计费，Nchf 模式走 N40 融合计费。融合计费场景（WSFD-011206）的典型动作是把指定用户的 `FORCED` 改为 `NchfMode` 切到 N40 接口。修改方式可按是否互操作/漫游进一步细化（SGW-only、V-SMF-only、I-SMF 计费开关、按 5GS 互操作指示、按 5GC 无限制接入标识）。

| 计费接口轨道 | 必备参数 | 典型脚本 |
|---|---|---|
| 切到 N40 融合接口（融合计费主用） | `APN` + `TMACCTYPE` + `FORCED=NchfMode` | `MOD APNCHGMODE:APN="cmnet",TMACCTYPE="UE5G_RAT4G",FORCED="NchfMode";` |
| 保持 Ga/Gy 离线/在线模式 | `APN` + `TMACCTYPE` + `FORCED=GaGyMode` | `MOD APNCHGMODE:APN="cmnet",TMACCTYPE="UE5G_RAT4G",FORCED="GaGyMode";` |
| SGW 计费模式单独指定 | `APN`+`TMACCTYPE`+`FORSGWONLY=GaGyMode` | `MOD APNCHGMODE:APN="cmnet",TMACCTYPE="UE5G_RAT4G",FORSGWONLY="GaGyMode";` |
| V-SMF 计费模式单独指定 | `APN`+`TMACCTYPE`+`FORVSMFONLY=GaGyMode`/`NchfMode` | `MOD APNCHGMODE:APN="cmnet",TMACCTYPE="UE5G_RAT4G",FORVSMFONLY="NchfMode";` |

## 决策点

### 计费接口选择维度（核心多值决策）
问：用哪种方式决定该 APN 的计费接口？
- 强制指定 → `FORCED`（`GaGyMode` 或 `NchfMode`），对所有 Serving Node 生效。
- 按 5GS 互操作指示 → `BY5GSIWKI`（True/False），发生 5GS 互操作时按指示切换。
- 按 5GC 无限制接入标识 → `BY5GCNRI`（True/False）。
- 按网元角色细分 → `FORSGWONLY`（SGW 模式）、`FORVSMFONLY`（V-SMF 模式）、`ISMFCHGSW`（I-SMF 是否支持计费 ENABLE/DISABLE）。

### 终端与接入类型
问：该 APN 配置针对哪种终端/接入组合？（必选 `TMACCTYPE`）
- `UE5G_RAT4G`（5G 终端 4G 接入）等枚举值——决定 PDN 激活场景下的计费接口。

### 漫游 I-SMF 是否计费
问：I-SMF 变更场景下是否对该 APN 计费？（仅漫游 I-SMF 组网考虑）
- `ISMFCHGSW=ENABLE`：I-SMF 支持计费。
- `ISMFCHGSW=DISABLE`：不使能（默认）。

## 约束
- `APN` 必选，1~63 字符（`-`/数字/大小写字母，首尾仅字母数字）。`severity=critical`
- `TMACCTYPE` 必选（指定终端和接入类型枚举）。`severity=critical`
- 计费接口选择方式互斥：`FORCED`、`BY5GSIWKI`+对应模式、`BY5GCNRI`+对应模式为不同决策路径，按规划选其一，避免冲突。`severity=high` 后果：选错计费策略接口，未配置时计费流程失败，用户无法正常激活。
- 只对新激活用户生效（已激活用户不重选接口）。`severity=medium` 后果：变更需用户重激活或等待自然重连。
- 适用 NF：SGW-C、PGW-C、SMF、GGSN。`severity=medium`
- 执行后立即生效。`severity=low`
