# 配置HTTP属性（SET HTTPCONF）

> 命令静态知识见 [SET HTTPCONF](command/UNC/20.15.2/SET-HTTPCONF.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET HTTPCONF 是平台服务管理层的 HTTP 链路总开关与调参命令，参数众多（35 个）。系统部署后已存在初始记录，未输入参数保持系统当前值不变（可通过 `LST HTTPCONF` 查看）。实际激活 md 中只有极少数参数被显式配置，本页按维度组织实际激活 md 中出现配置的参数与典型场景；完整 35 参数见命令 wiki。本命令有 3 个配置维度：**功能开关集合**、**容灾探测参数**、**Keepalive 子参数组**。

### 配置维度 1：功能开关集合（参数 HTTPVERSIONSW/REQCBFCSW/PINGSWITCH/SIGNALROUTING 等，可选）

| 维度 | 参数 | 取值 | 默认（初始值） | 作用 | 适用场景 |
|---|---|---|---|---|---|
| HTTP1.1 开关 | `HTTPVERSIONSW` | TRUE/FALSE | FALSE | 使能服务端支持 HTTP1.1（默认仅 HTTP2.0） | QoS 能力开放（WSFD-109101）需使能 |
| ReqCB 流控开关 | `REQCBFCSW` | TRUE/FALSE | TRUE | 控制服务端和客户端 REQCB 流控 | 服务化接口流控（WSFD-010801） |
| HTTP PING 探测开关 | `PINGSWITCH` | TRUE/FALSE | TRUE | 客户端 HTTP PING 探测功能开关 | UDM 全故障业务保活（WSFD-206101、WSFD-130011） |
| 信令路由开关 | `SIGNALROUTING` | TRUE/FALSE | FALSE | HTTP 链路按信令路由方式处理 | **不建议开启**（性能降低） |
| 最大并发流 | `MAXCLIENTSTREAM` | 整数 0~32768（0 实际生效 4096） | 0 | 客户端最大并发流上限 | **建议直接使用默认** |

> 数据来源：HTTPVERSIONSW/PINGSWITCH 本端规划；REQCBFCSW 全网规划。SIGNALROUTING 开启会降低 HTTP 性能，可能影响正常业务；MAXCLIENTSTREAM 过大或过小均降低性能。

### 配置维度 2：容灾探测参数（参数 PINGINTVAL/PINGPROBES，可选）

| 维度 | 参数 | 取值 | 默认（初始值） | 作用 | 适用场景 |
|---|---|---|---|---|---|
| PING 探测间隔 | `PINGINTVAL` | 整数 10~600 秒（协议最小 60） | 60 | HTTP PING 报文发送时间间隔 | 容灾探测周期（默认探测窗口 PINGINTVAL×PINGPROBES） |
| PING 探测次数 | `PINGPROBES` | 整数 2~10 | 3 | 连续未收到响应的次数阈值，达到即判故障 | 容灾探测次数 |

> 数据来源：本端规划。PINGINTVAL 协议最小 60 秒，需设置低于 60 秒时必须先开对应软参且确认对端支持。

### 配置维度 3：Keepalive 子参数组（参数 KEEPALIVESW/KEEPALIVETIME/KEEPALIVEINTVL/KEEPALIVEPROBES，conditional）

| 维度 | 参数 | 取值 | 默认（初始值） | 必填条件 | 作用 |
|---|---|---|---|---|---|
| Keepalive 开关 | `KEEPALIVESW` | ON/OFF | ON | — | TCP Keepalive 机制总开关 |
| 空闲时间 | `KEEPALIVETIME` | 整数 60~65535 秒 | 1800 | `KEEPALIVESW=ON` 时必填 | 空闲后触发周期性保活探测 |
| 探测间隔 | `KEEPALIVEINTVL` | 整数 1~150 秒 | 5 | `KEEPALIVESW=ON` 时必填 | 无响应时下一次探测的间隔 |
| 探测次数 | `KEEPALIVEPROBES` | 整数 1~20 次 | 5 | `KEEPALIVESW=ON` 时必填 | 无响应可继续探测的次数 |

> 数据来源：本端规划。Keepalive 子参数建议使用默认值；过大/过小都会影响保活探测效果。

**典型脚本**

```
# QoS 能力开放前置——使能 HTTP1.1
SET HTTPCONF: HTTPVERSIONSW=TRUE;

# 服务化接口流控——开 ReqCB 流控开关
SET HTTPCONF: REQCBFCSW=TRUE;
```

**步骤位置**：平台级 HTTP 链路调优命令，在依赖 HTTP 链路的特性编排中位于"HTTP 属性前置使能"步骤——通常作为 QoS 能力开放（`ADD SBIAPLE` + `SET SMCOMMFUNC`）、服务化接口流控（`ADD HTTPRESCFG`）、容灾探测（UDM/SCP 故障保活）等特性的前置配置。

## 决策点

### 决策点：容灾探测参数取值（DP 0-03097）

问：UDM/SCP 故障探测周期与次数配多少？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 默认探测窗口（约 4 分钟） | `PINGINTVAL=60`、`PINGPROBES=3`（默认） | 一般容灾时延要求 |
| 缩短故障发现时延 | `PINGINTVAL`<60（须先开软参）+ `PINGPROBES` 调小 | 高实时性容灾要求，容忍探测开销增加 |
| 延长探测窗口 | `PINGINTVAL` 调大 + `PINGPROBES` 调大 | 降低探测开销，容忍更长故障发现时延 |

> 取值由容灾时延要求决定：默认探测窗口约 4 分钟（PINGINTVAL×PINGPROBES），AMF/SMF 在此期间用户注册/会话激活可能失败；调小缩短故障发现但增加探测开销。

### 决策点：功能开关集合选择（DP 0-03098）

问：哪些 HTTP 功能开关需打开？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| QoS 能力开放前置 | `HTTPVERSIONSW=TRUE`（使能 HTTP1.1） | WSFD-109101 QoS 能力开放 |
| 服务化接口流控 | `REQCBFCSW=TRUE`（开 ReqCB 流控） | WSFD-010801 服务化接口流控 |
| UDM 全故障保活 | `PINGSWITCH=TRUE`（开 PING 探测）+ 容灾探测参数 | WSFD-206101、WSFD-130011 容灾保活 |

> 取值由上游特性要求驱动；每个特性开启自己的子集，跨场景枚举；SIGNALROUTING 不建议开启（性能降低）。

## 约束

- **性能风险（SIGNALROUTING）**（critical，rule-0-03241）：性能风险（SIGNALROUTING）：信令路由开关开启会降低 HTTP 性能，可能影响正常业务，不建议开启 — 后果：误开启导致 HTTP 性能下降影响业务
- **性能风险（MAXCLIENTSTREAM/HTTPLOGLEVE**（warning，rule-0-03242）：性能风险（MAXCLIENTSTREAM/HTTPLOGLEVEL/TCPLOGLEVEL）：最大并发流上限过大或过小均降低 HTTP 性能（0 实际以 4096 生效）；HttpLogLevel/TcpLogLevel 配成 DEBUG/INFO/WARN 会造成性能下降，建议直接使用默认 — 后果：参数误配导致 HTTP 性能下降
- **conditional**（warning，rule-0-03243）：conditional：KEEPALIVETIME/KEEPALIVEINTVL/KEEPALIVEPROBES 仅在 KEEPALIVESW=ON 时必填，关闭时配置无效 — 后果：开关关闭时子参数不生效，规划须先确认开关状态
- **下限软参**（warning，rule-0-03244）：下限软参：PINGINTVAL 协议最小 60 秒，需设置低于 60 秒时必须先开对应软参且确认对端支持 — 后果：低于协议最小值导致探测异常
- **唯一性与生效**（info，rule-0-03245）：唯一性与生效：全局一份（HTTP 属性唯一记录），执行后立即生效；系统部署后已存在初始记录，未输入参数保持当前值；适用 NF 为平台级（UNC 通用） — 后果：覆盖既有全局配置，跨 NF 各网元独立配置
