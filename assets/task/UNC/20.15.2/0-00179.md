# SET HTTPCONF 原子任务
> 命令静态知识见 [SET HTTPCONF](command/UNC/20.15.2/SET-HTTPCONF.md)。本页只讲配置生成实例化时这条命令**怎么配**。

SET HTTPCONF 是平台服务管理层的 HTTP 链路总开关与调参命令，参数众多。实际激活 md 中只有极少数参数被显式配置（典型为 PINGINTVAL/PINGPROBES 容灾探测、HTTPVERSIONSW 使能 HTTP1.1、REQCBFCSW 资源流控），其余参数保持系统初始记录值。系统部署后已存在初始记录。

## 配置方法
按场景选少数相关参数配置；未输入参数保持系统当前值不变（可通过 LST HTTPCONF 查看）。下表仅列实际激活 md 中出现配置的参数与典型场景；完整 35 参数见命令 wiki。

| 参数 | 描述 | 配置方法 | 取值样例 | 典型场景 |
|---|---|---|---|---|
| PINGSWITCH | 客户端 HTTP PING 探测开关 | 本端规划 | `TRUE` | UDM 全故障业务保活（WSFD-206101、WSFD-130011） |
| PINGINTVAL | 客户端 HTTP PING 探测时间间隔(s) | 本端规划 | 默认 60；协议最小 60，低于 60 需先开软参 | 容灾探测周期（默认 4 分钟探测窗口） |
| PINGPROBES | 客户端 HTTP PING 探测次数 | 本端规划 | `2`~`10`（默认 3） | 容灾探测次数 |
| HTTPVERSIONSW | 使能 HTTP1.1 开关 | 本端规划 | `TRUE` | QoS 能力开放（WSFD-109101）需使能 HTTP1.1 |
| REQCBFCSW | Reqcb 流控开关 | 全网规划 | `TRUE` | 服务化接口流控（WSFD-010801） |

典型脚本（QoS 能力开放前置——使能 HTTP1.1）：
```
SET HTTPCONF: HTTPVERSIONSW=TRUE;
```
典型脚本（服务化接口流控——开 ReqCB 流控开关）：
```
SET HTTPCONF: REQCBFCSW=TRUE;
```

## 决策点

### 容灾探测参数（PINGINTVAL/PINGPROBES）
- 问什么：UDM/SCP 故障探测周期与次数。
- 取值由容灾时延要求决定：默认探测窗口约 4 分钟（PINGINTVAL×PINGPROBES），AMF/SMF 在此期间用户注册/会话激活可能失败；调小可缩短故障发现但增加探测开销。
- 跨场景差异：UDM 全故障保活（WSFD-206101）、SMSF 容灾（WSFD-130011）均引用，但取值由组网容灾策略决定。

### 功能开关集合（HTTPVERSIONSW/REQCBFCSW/PINGSWITCH 等）
- 问什么：哪些 HTTP 功能开关需打开。
- 取值由上游特性要求驱动：QoS 能力开放要求 HTTP1.1；服务化接口流控要求 ReqCB 流控；UDM 全故障保活要求 PING 探测。每个特性开启自己的子集，跨场景枚举。

### Keepalive 子参数组（conditional）
- 问什么：开启 KEEPALIVESW=ON 后，空闲/探测间隔/探测次数怎么配。
- conditional：KEEPALIVETIME / KEEPALIVEINTVL / KEEPALIVEPROBES 仅在 KEEPALIVESW=ON 时必填。

## 约束
- 唯一性：全局一份（HTTP 属性唯一记录）。`severity=medium`
- 生效时机：执行后立即生效。`severity=high`
- 性能风险（SIGNALROUTING）：信令路由开关开启会降低 HTTP 性能，可能影响正常业务，不建议开启。`severity=high`
- 性能风险（MAXCLIENTSTREAM）：最大并发流上限过大或过小均会降低 HTTP 性能，建议直接使用默认；配置为 0 时实际以 4096 的最小值生效。`severity=high`
- conditional：KEEPALIVETIME / KEEPALIVEINTVL / KEEPALIVEPROBES 仅在 KEEPALIVESW=ON 时必填。`severity=medium`
- 下限软参：PINGINTVAL 协议最小 60 秒，需设置低于 60 秒时必须先开对应软参。`severity=medium`
- 默认值：系统部署后已存在初始记录，未输入参数保持当前值。`severity=low`
- 适用 NF：平台级（UNC 通用）。`severity=low`
