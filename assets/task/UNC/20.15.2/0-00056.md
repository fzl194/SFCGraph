# 配置 PCF 业务服务区绑定（ADD PCFSSCOPEBIND）

> 命令静态知识见 [ADD PCFSSCOPEBIND](command/UNC/20.15.2/ADD-PCFSSCOPEBIND.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

本命令把"PCF 业务服务区（SSCOPEID）"与"用户 TAI 区域（USRTAIRANGENAME）"绑定起来，是 PCF 服务区选择三件套的"绑定层"。前置链：先 [ADD PCFSSCOPE](task/UNC/20.15.2/0-00055.md) 定义 PCF 业务服务区，再 [ADD USRTAIRANGE](task/UNC/20.15.2/0-00097.md) 定义用户 TAI 区域，最后用本命令把两者绑定。语音用户 N7 会话建立时，其 TAI 匹配本表 → 得 SSCOPEID → 查 PCFSSCOPE 得 SSCOPENAME → 向 NRF 查可用 PCF。配置空间有 2 个维度：**绑定键**（本端局部）和**两端引用**（须指向已配置记录）。

### 配置维度 1：绑定键（参数 BINDNAME + BINDTYPE，必选）

| 参数 | 取值来源 | 约束 | 作用 |
|---|---|---|---|
| **BINDNAME** | 本端规划 | 字符串 1~32，不区分大小写，转小写存储 | 绑定记录名称，本端局部标识 |
| **BINDTYPE** | 本端规划 | 固定取值 `USRTAIRANGE`（用户 TAI 范围） | 指定和服务区标识绑定的数据类型 |

### 配置维度 2：两端引用（参数 SSCOPEID + USRTAIRANGENAME，必选）

| 参数 | 取值来源 | 约束 | 作用 |
|---|---|---|---|
| **SSCOPEID** | 引用 ADD PCFSSCOPE 已配置值 | 字符串 1~32，不区分大小写；**必须是 ADD PCFSSCOPE 已配置的 SSCOPEID** | 指向 PCF 业务服务区记录 |
| **USRTAIRANGENAME** | 引用 ADD USRTAIRANGE 已配置值 | 字符串 1~32，不区分大小写；**必须是 ADD USRTAIRANGE 已配置的 RANGENAME** | 指向用户 TAI 区域记录 |

> BINDTYPE 当前仅支持 `USRTAIRANGE` 一种取值——本命令本质是"用户 TAI 区域 → PCF 业务服务区"的单向映射。

**典型脚本**（绑定 towna = 服务区 citya + TAI 区域 tai1）：
```
ADD PCFSSCOPEBIND:BINDNAME="towna",SSCOPEID="citya",BINDTYPE=USRTAIRANGE,USRTAIRANGENAME="tai1";
```

**步骤位置**：5G PCC "PCF 发现和选择"链路第三环（末环）——`ADD PCFSSCOPE`（建服务区）→ `ADD USRTAIRANGE`（建 TAI 区域）→ 本命令（绑定两者）。属可选步骤，仅 SMF/PGW-C/GGSN 需要配置，且仅当前两步均已配置时才有意义。

## 决策点

### 决策点：是否按用户位置映射 PCF 业务服务区（DP 0-02091）

问：组网是否要求 SMF 根据用户 TAI 区域选择对应的 PCF 业务服务区（即是否启用 5G PCC 基于 SERVINGSCOPE 的 PCF 选择）？

| 选项 | 行为 | 适用场景 |
|---|---|---|
| 不配置本命令 | N7 会话建立时不按用户位置映射 PCF 业务服务区 | 无区域差异化 PCF 需求 |
| 配置本命令 | 用户 TAI 区域命中后映射到指定 PCF 业务服务区，进而选择可用 PCF | 语音用户按地理位置选区域 PCF |

### 决策点：绑定两端取值来源（DP 0-02092）

问：SSCOPEID 与 USRTAIRANGENAME 从哪里取值？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 引用已配置记录（唯一正确选项） | SSCOPEID 须是 ADD PCFSSCOPE 已配置值；USRTAIRANGENAME 须是 ADD USRTAIRANGE 已配置值 | 正常配置（两端必须前置） |
| 本端自由规划（错误） | BINDNAME 本端局部标识，可自由规划 | 仅 BINDNAME 可自由规划；SSCOPEID/USRTAIRANGENAME 不可自由编造 |

> SSCOPEID 和 USRTAIRANGENAME 是引用键，必须指向已存在的记录；仅 BINDNAME 是本端自由规划的标识。BINDTYPE 固定为 `USRTAIRANGE`，无选择空间。

## 约束

- rule-0-02226 · severity=critical · 前置依赖：SSCOPEID 必须是 ADD PCFSSCOPE 已配置的服务区标识；当 BINDTYPE=USRTAIRANGE 时 USRTAIRANGENAME 必须是 ADD USRTAIRANGE 已配置的 RANGENAME — 后果：绑定关系无法建立、PCF 服务发现失败
- rule-0-02227 · severity=critical · 唯一性（重叠禁止）：BINDTYPE=USRTAIRANGE 时，系统中不同 PCF 业务服务区对应的用户 TAI 区域**不能重叠**；配置时有检测，重叠则下发失败 — 后果：配置下发失败（避免映射歧义）
- rule-0-02228 · severity=high · 唯一性（绑定名）：BINDNAME 长度 1~32，不区分大小写（转小写存储），多条记录间不可重复 — 后果：记录冲突、新增失败
- rule-0-02229 · severity=warning · 兜底行为：若 PCFSSCOPE 有记录但激活请求未携带 TAI、或根据 TAI 无法从本表匹配到合适 SSCOPEID → 从 PCFSSCOPE 记录中随机选一个 SSCOPENAME 用于 PCF 服务发现（非确定性选路） — 后果：选路结果不可预期
- rule-0-02230 · severity=info · 规格上限：最多可输入 5000 条记录；命令执行后立即生效；适用 NF：SMF、PGW-C、GGSN
