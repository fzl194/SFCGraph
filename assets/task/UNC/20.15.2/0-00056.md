# ADD PCFSSCOPEBIND 原子任务
> 命令静态知识见 [ADD PCFSSCOPEBIND](command/UNC/20.15.2/ADD-PCFSSCOPEBIND.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令为**必需**命令，仅在 SMF 需要按 SERVINGSCOPE 选择 PCF（5G 语音用户 N7 会话建立场景）时配置。前置链：先 `ADD USRTAIRANGE` 定义用户 TAI 区域，再 `ADD PCFSSCOPE` 定义 PCF 业务服务区，最后用本命令把两者绑定起来。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| BINDNAME | 绑定名称，本端规划，1~32 字符，全网唯一 | `towna` |
| SSCOPEID | 服务区标识，从已配置的 `ADD PCFSSCOPE` 数据中获取 | `citya` |
| BINDTYPE | 绑定类型，固定取值 `USRTAIRANGE`（用户 TAI 范围） | `USRTAIRANGE` |
| USRTAIRANGENAME | 用户 TAI 区域名称，从已配置的 `ADD USRTAIRANGE` 数据中获取 | `tai1` |

典型脚本：
```
ADD PCFSSCOPEBIND:BINDNAME="towna",SSCOPEID="citya",BINDTYPE=USRTAIRANGE,USRTAIRANGENAME="tai1";
```

## 决策点

### 是否需要按用户位置映射 PCF 业务服务区
- 问什么：组网是否要求 SMF 根据用户 TAI 区域选择对应的 PCF 业务服务区（即是否启用 5G PCC 基于 SERVINGSCOPE 的 PCF 选择）。
| 是否配置 | 行为 |
|---|---|
| 不配置 | N7 会话建立时不按用户位置映射 PCF 业务服务区 |
| 配置 | 用户 TAI 区域命中后映射到指定 PCF 业务服务区，进而选择可用 PCF |

## 约束
- 前置依赖：需先配置 [ADD PCFSSCOPE](task/UNC/20.15.2/0-00055.md)；当 BINDTYPE=USRTAIRANGE 时还需先配置 [ADD USRTAIRANGE](task/UNC/20.15.2/0-00097.md)。`severity=high`
- 唯一性：系统中不同 PCF 业务服务区对应的用户 TAI 区域**不能重叠**；配置时有检测，重叠则下发失败。`severity=high`
- 生效时机：命令执行后**立即生效**。`severity=medium`
