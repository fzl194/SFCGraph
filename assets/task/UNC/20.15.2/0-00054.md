# 配置 PCF 选择策略（ADD PCFSELPLCY）

> 命令静态知识见 [ADD PCFSELPLCY](command/UNC/20.15.2/ADD-PCFSELPLCY.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

本命令为 AMF 增加 PCF 选择策略，按**用户范围（SUBRANGE）**圈定用户群，再用一组开关（SUPI/GPSI/网络切片/目标 PLMN/重选/Serving Scope）决定匹配维度。系统部署后已有初始记录（`SUBRANGE=ALL_USER`，按 SUPI 选 PCF），本命令在其上增加差异化策略。配置空间分 3 个维度：**用户范围**（必选，决定后续条件必选参数）、**PCF 发现匹配键**（多开关组合）、**容灾重选/服务范围**（条件可选）。

### 配置维度 1：用户范围（参数 SUBRANGE，必选）

| 取值 | 作用 | 联动必选参数 | 数据来源 |
|---|---|---|---|
| **ALL_USER** | 所有用户（系统初始记录即此） | — | 全网规划 |
| **HOME_USER** | 本网用户 | 通常按 SUPI 选择（`SUPISW=YES`） | 全网规划 |
| **FOREIGN_USER** | 外网用户（漫游用户） | 通常 `TGTPLMNSW=YES` + GPSI/SUPI 组合 | 全网规划 |
| **USER_GROUP** | 指定用户群 | `USRGRPID`（须已用 ADD NGUSRGRP（★R1.5 atom 待补） 创建）+ `CTRLMODE`（白/黑名单） | 全网规划 |
| **IMSI_PREFIX** | 按 IMSI 前缀细分 | `IMSIPRE`（5~15 位数字）+ `CTRLMODE` — **预留参数，策略暂未实现** | 本端规划 |

> 匹配优先级（从高到低）：`USER_GROUP` > `FOREIGN_USER`/`HOME_USER` > `ALL_USER`。

### 配置维度 2：PCF 发现匹配键（多开关组合，可选）

| 开关参数 | 默认 | 作用 | 数据来源 |
|---|---|---|---|
| **SUPISW** | YES | 用 SUPI 作为 PCF 发现条件（默认发现键） | 全网规划 |
| **GPSISW** | NO | 用 GPSI 作为发现条件；=YES 时需公共软参 DWORD206 BIT3=0 | 全网规划 |
| **NSSW** | NO | 用可用网络切片作为发现条件（按切片规划 PCF 时打开） | 全网规划 |
| **TGTPLMNSW** | YES | 携带 target-plmn-list（漫游场景必选，AMF 与目标 PCF 跨网时必携） | 全网规划 |

### 配置维度 3：容灾重选与服务范围（条件可选）

| 参数 | 触发条件 | 取值 | 作用 |
|---|---|---|---|
| **RETRYSW** | 可选（默认 YES） | YES/NO | 初选 PCF 收 5xx 是否重选新 PCF 重试 AM/UE 策略偶联创建（仅重试一次） |
| **REDIRECTSW** | 可选 | YES/NO | 初选 PCF 收 308 永久重定向是否按指示重选新 PCF（仅重试一次） |
| **SSSW** | SUBRANGE=USER_GROUP/IMSI_PREFIX 时可选 | YES/NO | 是否携带 Serving Scope 信息（为指定用户群选特定区域服务的 PCF） |
| **SERVINGSCOPE** | SSSW=YES 时必填 | 字符串 0~64，多个用 `:` 分隔 | 目标 PCF 的服务范围；未填则用本 AMF 的服务范围（ADD NFSRVSCOPE（★R1.5 atom 待补）） |
| **CROSSPROVSW** | 可选 | YES/NO | 是否区分跨省漫游：=YES 时跨省用 Serving Scope 发现 PCF、本省用号段发现 |
| **NBEGIN/NEND** | CROSSPROVSW=YES 时必填 | 整数 1~12，NEND≥NBEGIN | NF Instance ID 的 node 部分第 N~M 字符用于跨省判定（UUID v4 格式） |

**典型脚本**（漫游用户按 GPSI 发现 V-PCF）：
```
ADD PCFSELPLCY:SUBRANGE=FOREIGN_USER,TGTPLMNSW=YES,SUPISW=NO,GPSISW=YES,NSSW=NO,DESC="for roaming user";
```

**典型脚本**（指定用户群按白名单 + Serving Scope 选特定区域 PCF）：
```
ADD PCFSELPLCY:SUBRANGE=USER_GROUP,USRGRPID=20,CTRLMODE=WHITELIST,SSSW=YES,SERVINGSCOPE="pudong:puxi",DESC="for vip user";
```

**步骤位置**：5G PCC 特性编排中的"AMF 侧 PCF 选择策略"步骤——属可选步骤，仅 AMF NF 需要配置（SMF 侧 PCF 选择走 SET APNPCCFUNC 的 PCFSELECTMODE，见 [0-00151](task/UNC/20.15.2/0-00151.md)）。

## 决策点

### 决策点：用户范围与匹配优先级（DP 0-02087）

问：本策略针对哪类用户？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 所有用户（`ALL_USER`） | 仅初始记录配此值；新增策略不应再用 | 兜底（系统已有初始记录） |
| 本网用户（`HOME_USER`） | 通常 `SUPISW=YES` | 归属用户按 SUPI 选 PCF（默认） |
| 外网用户（`FOREIGN_USER`） | 通常 `TGTPLMNSW=YES` + GPSI/SUPI 组合 | 漫游用户选 V-PCF（漫游地分流场景） |
| 指定用户群（`USER_GROUP`） | `USRGRPID` + `CTRLMODE` | VIP 用户群、白名单用户群选特定 PCF |
| IMSI 前缀（`IMSI_PREFIX`） | `IMSIPRE` + `CTRLMODE` | 预留参数，策略暂未实现 |

### 决策点：PCF 发现匹配键与服务范围（DP 0-02088）

问：用哪些标识匹配 PCF，是否按服务区域/跨省漫游细分？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 默认（SUPI + 目标 PLMN） | `SUPISW=YES` + `TGTPLMNSW=YES`（默认） | 常规归属/漫游用户 |
| GPSI 发现 | `GPSISW=YES`（+ 建议关 SUPISW、需 DWORD206 BIT3=0） | 运营商按 GPSI 号段规划 PCF |
| 网络切片发现 | `NSSW=YES` | 运营商按切片规划 PCF |
| 按服务范围细化（USER_GROUP） | `SSSW=YES` + `SERVINGSCOPE` | 为指定用户群选特定区域服务的 PCF |
| 区分跨省漫游 | `CROSSPROVSW=YES` + `NBEGIN/NEND` + `SERVINGSCOPE`（跨省）+ SUPI/GPSI（本省） | MEC To Mall、跨省漫游场景 |

## 约束

- **前置依赖**（critical，rule-0-02216）：前置依赖：SUBRANGE=USER_GROUP 时 USRGRPID 必填，且该用户群须已通过 ADD NGUSRGRP（★R1.5 atom 待补） 创建（可 LST NGUSRGRP 查询），群成员由 ADD NGUSRGRPMEM（★R1.5 atom 待补） 添加 — 后果：策略无法匹配到用户群
- **条件必选**（warning，rule-0-02217）：条件必选：SUBRANGE=USER_GROUP/IMSI_PREFIX 时 CTRLMODE 必填（WHITELIST 或 BLACKLIST） — 后果：命令下发失败
- **条件必选**（warning，rule-0-02218）：条件必选：SSSW=YES 时 SERVINGSCOPE 必填（0~64 字符，多个用 `:` 分隔）；CROSSPROVSW=YES 时 NBEGIN/NEND 必填且 1~12、NEND≥NBEGIN — 后果：服务范围/跨省判定失效
- **软参联动**（warning，rule-0-02219）：软参联动：GPSISW=YES 时需将公共软参 DWORD206 BIT3 置为 0，否则 GPSI 发现不生效；IMSIPRE 为预留参数，策略暂未实现，不建议配置
- **生效时机**（info，rule-0-02220）：生效时机：命令执行后立即生效；最多可输入记录数受规格约束；适用 NF 仅 AMF
