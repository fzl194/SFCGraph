# 基于业务累计流量的策略控制（WSFD-211009）

> 特性静态知识见 [WSFD-211009](feature/UNC/20.15.2/WSFD-211009.md)。本页讲配置生成实例化时这个特性**怎么配**。
>
> **特性关系**：PCC 族增量特性（带宽族第 6 批）。本特性 = [PCC基本功能 WSFD-109101](task/UNC/20.15.2/2-00005.md) 的**业务级**累计流量增量。与姊妹特性 [WSFD-109104 会话级FUP](task/UNC/20.15.2/2-00007.md) 同属 FUP 族，差异在累计粒度：
> - **WSFD-109104 会话级FUP**（2-00007）：按整会话累计，本端仅 3 参数增量（MKPARSEFORMAT/UMCH/FUPSESSIONEXC），**不配 URR 标识链**（会话级 UM 由 PCRF 在 SESSION_LEVEL 下发，本端无 URR 绑定）。
> - **WSFD-211009 业务级FUP**（本特性）：按业务（per SVC/per APP）累计，PCC_RULE_LEVEL，本端**须配 URR 标识链**——`ADD URR:USAGERPTMODE=MONITORINGKEY+MONITORINGKEY=<业务标识>` + URRGROUP + PCCPOLICYGRP(URRGROUPNAME+FUPSESSIONEXC) + RULE + USERPROFILE + RULEBINDING，即复用费率标识链 [1-00009](task/UNC/20.15.2/1-00009.md) 的 MONITORINGKEY 变种。
>
> 与 UDG 侧 [GWFD-110312](feature/UDG/20.15.2/GWFD-110312.md) 对应：UDG 侧 UPF 执行策略动作（业务数据检测/流量累计/降速阻塞），UNC 侧做策略接收转换 + URR 标识链配置。

## 配置概览

业务级 FUP 的核心是**按业务独立累计**：每个业务（如 P2P、视频、IM）配独立的 URR（不同 MONITORINGKEY 标识不同业务），通过 URRGROUP→PCCPOLICYGRP 挂到 RULE，再绑到 USERPROFILE。多个业务可绑同一 USERPROFILE（多 RULEBINDING），实现多业务同时 FUP、互不影响。

对象链（与费率标识链 1-00009 同构，URR 取 MONITORINGKEY 变种）：

```
SET LICENSESWITCH(LKV2FUPSAT01)
  └ SET PCCFUNC(MKPARSEFORMAT,仅Gx)                      ← 复用 1-00014 atom
  └ ADD URR(USAGERPTMODE=MONITORINGKEY,MONITORINGKEY)     ← 复用 1-00009
     └ ADD URRGROUP(UPURRNAME1/DOWNURRNAME1)              ← 复用 1-00009
        └ ADD PCCPOLICYGRP(URRGROUPNAME,FUPSESSIONEXC=ENABLE) ← 复用 1-00009
           └ ADD RULE(POLICYTYPE=PCC,POLICYNAME)          ← 复用 1-00009
              └ ADD USERPROFILE(UPTYPE=PCC)               ← 复用 1-00009
                 └ ADD RULEBINDING                        ← 复用 1-00009
  └ MOD PCRF(SUPFEANEGOSW+FEATURELIST=UMCH-1,仅Gx,可选)   ← 单命令 atom
```

两条接口路径 + 业务识别层差异：
- **Gx 接口（2G/3G/4G，与 PCRF）**：本端配完整 URR 标识链 + MKPARSEFORMAT + 可选 UMCH；Monitoring-Key 在 PCC 预定义规则用 ADD PCCPOLICYGRP 配（即本链），PCC 动态规则由 PCRF 在 Charging-Rule-Definition 下发。
- **N7 接口（5G，与 PCF）**：
  - **L3/L4 层**：PCF 动态下发 PccRule（含 flowInfos+refUmData），SMF 据此生成 N4 URR；本端 URR 标识链作为预定义规则承载（业务级 UM 仍需 URR MONITORINGKEY 标识）。
  - **L7 层**：使用预定义规则，PGW-C/SMF/PCF/PGW-U/UPF **四方预定义规则名 + URR ID + umID 须一致**；URR ID 高位 bit 为 1 区分预定义 URR。
  - N7 路径本端仍需 License + URR 标识链；MKPARSEFORMAT/UMCH 是 Gx 专属，N7 跳过。

## 配置流程（步骤编排，每步显示含命令）

1. **激活 FUP License**（前置）：`SET LICENSESWITCH` LICITEM="LKV2FUPSAT01" → [0-00180](task/UNC/20.15.2/0-00180.md)
   - 关键参数：LICITEM="LKV2FUPSAT01"（基于业务累计流量的策略控制，控制项 82200BNU），SWITCH=ENABLE
   - **前置依赖**：须先完成 [PCC基本功能 WSFD-109101](task/UNC/20.15.2/2-00005.md) 激活（License LKV2PCCBF01 + LKV3W9SPCC11 + PCRF 对接 + PCC 开关模板）；并依赖 [WSFD-211005 业务感知带宽控制](task/UNC/20.15.2/2-00010.md)（业务级带宽控制需 BWM 特性使能）
2. **设置 Monitoring-Key 解析方式（仅 Gx 路径）**：`SET PCCFUNC` MKPARSEFORMAT=UNSIGNED32 → [0-00193](task/UNC/20.15.2/0-00193.md)
   - 关键参数：MKPARSEFORMAT=UNSIGNED32（**必须与 PCRF 一致**）；复用 [1-00014](task/UNC/20.15.2/1-00014.md) 的 SET PCCFUNC atom，本特性只增 MKPARSEFORMAT，不重配 HOMEPCCSWITCH（已在 PCC 基本功能配过）
   - N7 路径跳过此步（MKPARSEFORMAT 是 Gx 接口 Monitoring-Key 解析方式，N7 无对应）
3. **配置业务级 URR 标识链**（核心，Gx/N7 均需）：`ADD URR`+`ADD URRGROUP`+`ADD PCCPOLICYGRP`+`ADD RULE`+`ADD USERPROFILE`+`ADD RULEBINDING` → [1-00009](task/UNC/20.15.2/1-00009.md)
   - 子步骤（每业务一组，多业务重复）：
     - `ADD URR` URRNAME + URRID（与 UPF 一致）+ **USAGERPTMODE=MONITORINGKEY** + **MONITORINGKEY**（与 PCRF/PCF 一致，标识一种业务） → [0-00001](task/UNC/20.15.2/0-00001.md)
     - `ADD URRGROUP` URRGROUPNAME + UPURRNAME1/DOWNURRNAME1（上下行均引用上步 URR） → [0-00093](task/UNC/20.15.2/0-00093.md)
     - `ADD PCCPOLICYGRP` PCCPOLICYGRPNM + URRGROUPNAME + **FUPSESSIONEXC=ENABLE**（该业务不计入 Session 级流量，避免与会话级 FUP 重复累计） → [0-00052](task/UNC/20.15.2/0-00052.md)
     - `ADD RULE` RULENAME + POLICYTYPE=PCC + PRIORITY + POLICYNAME=上步策略组名 → [0-00071](task/UNC/20.15.2/0-00071.md)
     - `ADD USERPROFILE` USERPROFILENAME（与 UPF 一致） + UPTYPE=PCC → [0-00094](task/UNC/20.15.2/0-00094.md)
     - `ADD RULEBINDING` USERPROFILENAME + RULENAME（多业务绑同一 USERPROFILE 用多条 RULEBINDING） → [0-00073](task/UNC/20.15.2/0-00073.md)
   - **复用说明**：本步完整复用费率标识链 [1-00009](task/UNC/20.15.2/1-00009.md)，差异是 URR 取 MONITORINGKEY 变种（非 ONLINE/OFFLINE 计费轨道）+ PCCPOLICYGRP 带 FUPSESSIONEXC。该差异已记入 1-00009 场景差异（FUP 业务级累计变种）。
4. **使能流量监控拥塞处理 UMCH（可选，仅 Gx 路径）**：`MOD PCRF` HOSTNAME + SUPFEANEGOSW=ENABLE + FEATURELIST=UMCH-1 → [0-00122](task/UNC/20.15.2/0-00122.md)
   - 关键参数：HOSTNAME=已 ADD PCRF 定义的主机名；SUPFEANEGOSW=ENABLE；FEATURELIST=UMCH-1（使能 UMCH）
   - 前置：HOSTNAME 须已在 PCC 基本功能 [1-00012](task/UNC/20.15.2/1-00012.md) 的 ADD PCRF 中定义
   - 适用：包月用户月末配额重置场景（PCRF 通过 Monitoring-Time 指定重置点，UNC 分统计重置前后流量合并上报，避免 Gx 信令拥塞）
   - N7 路径跳过此步（UMCH 是 Gx 拥塞处理机制）

> **activation 演示范围脚注（R1.4）**：WSFD-211009 仅 1 份 activation（`激活基于业务累计流量的策略控制_27915156.md`），演示完整 6 命令脚本：SET LICENSESWITCH + ADD URR(MONITORINGKEY) + ADD PCCPOLICYGRP(FUPSESSIONEXC) + ADD RULE + ADD USERPROFILE + ADD RULEBINDING。SET PCCFUNC:MKPARSEFORMAT 与 MOD PCRF:UMCH 在 activation **数据规划表有参数行**但**任务示例脚本未演示**（数据规划表列出 MKPARSEFORMAT=UNSIGNED32 取值样例）。
> - MKPARSEFORMAT：activation 数据规划表有参数行（取值 UNSIGNED32，与对端协商），证据来自 [SET PCCFUNC 命令 wiki](command/UNC/20.15.2/SET-PCCFUNC.md) + activation 数据规划表。配置生成按数据规划表产出。
> - MOD PCRF:UMCH：activation 未直接演示，UMCH 机制证据来自 [实现原理（Gx）](evidence/UNC/20.15.2/实现原理（Gx）_29039989.md) + WSFD-109104 activation（同族，UMCH 在 2-00007 已有 activation 脚本证据）。本特性步骤 4 标"可选"，配置生成对未规划月末重置的局点不产出。
> - **activation 脚本与配置流程的差异（R1.4 子规则，warning）**：activation 步骤 5 写 `ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicygrp_01",URRGROUPNAME="urr_01",FUPSESSIONEXC=ENABLE;`——**URRGROUPNAME 值="urr_01"是 URRNAME 不是 URRGROUPNAME**（activation 原文笔误，漏了 ADD URRGROUP 步骤）。配置生成须按对象链补 `ADD URRGROUP:URRGROUPNAME="urrgrp_01",UPURRNAME1="urr_01",DOWNURRNAME1="urr_01";` 再 ADD PCCPOLICYGRP:URRGROUPNAME="urrgrp_01"。此修正依据费率标识链 [1-00009](task/UNC/20.15.2/1-00009.md) 标准结构 + [ADD PCCPOLICYGRP 命令 wiki](command/UNC/20.15.2/ADD-PCCPOLICYGRP.md)（URRGROUPNAME 引用 URRGROUP 的 URRGROUPNAME）。

## 决策点（特性级）

### DP1：接口路径（决定本端配置量与 URR 生成方式）

| 接口路径 | 走法 | 本端配置 | URR 生成 | 前置 |
|---|---|---|---|---|
| Gx（2G/3G/4G，与 PCRF） | 步骤 1+2+3+4 | MKPARSEFORMAT + URR 标识链 + 可选 UMCH | PCEF 据 PCRF 下发 PCC 规则（含 Monitoring-Key）生成；预定义规则用本端 ADD PCCPOLICYGRP 配 | [2-00005](task/UNC/20.15.2/2-00005.md) DP1=2G/3G/4G 已激活 |
| N7 L3/L4（5G，PCF 动态） | 步骤 1+3 | License + URR 标识链（本端仍需配 URR MONITORINGKEY 标识业务） | SMF 据 PccRule(flowInfos+refUmData)+umDecs 动态生成 N4 URR，绑定指定业务流 PDRs | [2-00005](task/UNC/20.15.2/2-00005.md) DP1=5G 已激活 |
| N7 L7（5G，预定义规则） | 步骤 1+3（URRID 高位 bit=1） | License + URR 标识链（预定义 URR，URRID 高位 bit 为 1） | SMF 据预定义规则配置生成预定义 URR；四方（SMF/PCF/UPF）规则名+URRID+umID 须一致 | [2-00005](task/UNC/20.15.2/2-00005.md) DP1=5G 已激活 |

### DP2：业务识别层（决定 N7 路径 URR 生成方式与一致性要求）

| 选项 | 影响 | 一致性要求 |
|---|---|---|
| L3/L4 层（五元组/SDF Filter） | N7: PCF 动态下发 PccRule+flowInfos+refUmData；SMF 生成动态 URR 绑指定业务流 PDRs | PCF 动态下发，无需预配规则名 |
| L7 层（DPI 应用特征） | N7: 预定义规则；SMF/PCF/PGW-U/UPF 四方预定义规则名+URR ID+umID 须一致；URR ID 高位 bit=1 标预定义 | 四方一致性（critical） |

### DP3：是否排除 Session 级累计（FUPSESSIONEXC，Gx/N7 均可）

| 选项 | 影响 | 适用 |
|---|---|---|
| FUPSESSIONEXC=ENABLE（步骤 3 PCCPOLICYGRP 带） | 该 PCCPOLICYGRPNM 对应业务**不计入** Session 级流量，避免业务级流量同时被会话级 FUP 重复累计 | 业务级 FUP 典型场景（默认推荐） |
| FUPSESSIONEXC=DISABLE（默认，步骤 3 不带） | 该策略组业务**计入** Session 级流量 | 需业务级+会话级双重累计的特殊场景 |

### DP4：是否使能 UMCH（仅 Gx，避免配额重置信令拥塞）

| 选项 | 影响 | 适用 |
|---|---|---|
| UMCH=ENABLE（步骤 4，FEATURELIST=UMCH-1 + SUPFEANEGOSW=ENABLE） | PCRF 通过 Monitoring-Time 指定配额重置点，UNC 分统计重置前后流量合并上报 | 包月用户月末配额重置场景（典型 FUP） |
| UMCH 不使能（步骤 4 跳过） | PCRF 同时要求所有在线用户上报→Gx 信令拥塞风险 | 无配额重置需求或用户数少 |

## 约束（特性级 rule）

- **依赖 PCC 基本功能**（critical）：本特性须先激活 [WSFD-109101 PCC基本功能](task/UNC/20.15.2/2-00005.md)（License LKV2PCCBF01+LKV3W9SPCC11 + PCRF 对接 + PCC 开关） — 否则无 PCC 基础设施，URR 标识链无处挂接。
- **依赖业务感知带宽控制**（critical）：须激活 [WSFD-211005](task/UNC/20.15.2/2-00010.md)（License LKV3TCBSA01） — 按用户某类业务进行带宽控制需启用 BWM 特性。
- **License 前置**（critical）：须激活 LKV2FUPSAT01（控制项 82200BNU） — 否则业务级 FUP 功能不生效。
- **MKPARSEFORMAT 须与 PCRF 一致**（critical，仅 Gx）：SET PCCFUNC 的 MKPARSEFORMAT 必须与对端 PCRF 协商一致 — 否则 Monitoring-Key 解析不匹配，业务流量累计错误。
- **MONITORINGKEY 标识业务且与 PCRF/PCF 一致**（critical）：ADD URR 的 MONITORINGKEY 标识一种业务，须与 PCRF/PCF 的监控属性值保持一致 — 否则业务级流量无法正确关联累计。
- **URRID 跨网元一致**（critical）：ADD URR 的 URRID 须与 PGW-U/UPF 上的 URR 标识一致 — 否则 N4 URR 规则无法对应。
- **L7 预定义规则四方一致**（critical，仅 N7 L7）：PGW-C/SMF/PCF/PGW-U/UPF 的预定义规则名 + URR ID + umID 须一致；URR ID 高位 bit 为 1 标预定义 URR — 否则预定义规则不生效。
- **UMCH 仅支持总流量阈值**（warning，仅 Gx）：UMCH 机制不支持上下行独立阈值，仅总流量阈值 — 规划时不可设分上下行阈值。
- **UE Category 用户阈值下限**（warning）：PCRF/PCF 下发的业务级流量阈值须不低于 1GB — 否则影响 UE Category 用户流量累计准确性。
- **FUPSESSIONEXC 语义**（info）：ENABLE=不计入 Session 级流量（业务级 FUP 典型，避免重复累计），DISABLE（默认）=计入。
- **多业务独立累计**（info）：每业务配独立 URR（不同 URRID+MONITORINGKEY）+ 独立 PCCPOLICYGRP，多 RULEBINDING 绑同一 USERPROFILE 实现多业务同时 FUP。
- **N7 L3/L4 本端仍需 URR 标识链**（info）：N7 动态 PccRule 由 PCF 下发，但业务级 UM 的 URR MONITORINGKEY 标识仍需本端配置（预定义规则承载）。
