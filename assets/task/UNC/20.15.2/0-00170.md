# 配置全局默认空闲上下文定时器（SET DFTIDLETIME）

> 命令静态知识见 [SET DFTIDLETIME](command/UNC/20.15.2/SET-DFTIDLETIME.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SGW-C/PGW-C/SMF/GGSN 侧**全局级**空闲/不活动上下文定时器：会话无数据传输超时后，按角色（SGW-C/PGW-C/GGSN/H-SMF/I-SMF/Proxy-SMF）和级别（会话级/承载级/QoS Flow 级）决定删除会话/承载/QoS Flow（空闲定时器，2G/3G/4G/5G 通用）或仅释放用户面资源（不活动定时器，仅 5G）。APN 粒度由 SET APNIDLETIME 覆盖，全局为本命令。核心配置空间有 3 个维度：**网元角色开关集合**、**核查级别**、**定时器时长**。

### 配置维度 1：网元角色开关（参数 SCTXCHKSW/PCTXCHKSW/GCTXCHKSW/HCTXCHKSW/ICTXCHKSW/PROXYSMFCHKSW 等）

| 参数 | 取值 | 作用 | 备注 |
|---|---|---|---|
| `SCTXCHKSW` | DISABLE / ENABLE | SGW-C 空闲核查开关 | 由 DISABLE 改 ENABLE 时仅对新创建上下文检查 |
| `PCTXCHKSW` | DISABLE / ENABLE | PGW-C/SGW-C 合一部署空闲核查开关 | 同上 |
| `GCTXCHKSW` | DISABLE / ENABLE | GGSN 空闲核查开关 | 同上 |
| `HCTXCHKSW` | DISABLE / ENABLE | H-SMF 空闲核查开关 | 5G 用户 |
| `HCTXINACHKSW` | DISABLE / ENABLE | H-SMF 不活动核查开关 | 仅 5G 用户；默认 ENABLE |
| `ICTXCHKSW` | DISABLE / ENABLE | I-SMF/V-SMF 空闲核查开关 | 仅会话级核查 |
| `ICTXINACHKSW` | DISABLE / ENABLE | I-SMF/V-SMF 不活动核查开关 | 仅 5G 用户 |
| `PROS8CTXCHKSW` | DISABLE / ENABLE | Proxy-SMF S8 空闲核查开关 | 由 DISABLE 改 ENABLE 时仅对新创建上下文检查 |
| `PROXYSMFCHKSW` | DISABLE / ENABLE | Proxy-SMF 空闲核查开关 | 同上 |

### 配置维度 2：核查级别（参数 HSMFTIMERLEVEL / GULTIMERLEVEL）

| 参数 | 取值 | 作用 | 备注 |
|---|---|---|---|
| `HSMFTIMERLEVEL` | SESSION / QOSFLOW | H-SMF 核查级别（仅 5G） | SESSION 时 SMFSESIDLETIMER 生效，DEDQF/DFTQF 不生效；QOSFLOW 时反之 |
| `GULTIMERLEVEL` | SESSION / BEARER | GUL 承载级别（仅 2G/3G/4G） | SESSION 时 SESSIONTIMER 生效；BEARER 时 BEARERTIMER 生效 |

### 配置维度 3：定时器时长（分钟）

| 参数 | 范围 | 作用 | 备注 |
|---|---|---|---|
| `BEARERTIMER` | 5~12000 | 承载级空闲定时器（2G/3G/4G） | GULTIMERLEVEL=BEARER 时生效 |
| `SESSIONTIMER` | 5~12000 | 会话级空闲定时器（2G/3G/4G） | GULTIMERLEVEL=SESSION 时生效 |
| `SMFSESIDLETIMER` | 5~12000 | SMF 会话空闲定时器（5G） | HSMFTIMERLEVEL=SESSION 时生效 |
| `SMFSESINATIMER` | 5~12000 | SMF 会话不活动定时器（5G） | ≤ SMFSESIDLETIMER |
| `DEDQFIDLETIMER` | 5~12000 | 专有 QoS Flow 空闲定时器（conditional，HSMFTIMERLEVEL=QOSFLOW 时配） | 不支持 0 |
| `DFTQFIDLETIMER` | 0 / 5~12000 | 缺省 QoS Flow 空闲定时器（conditional，HSMFTIMERLEVEL=QOSFLOW 时配） | 0=不开启缺省 QoS Flow 核查 |
| `DFTBEARPOLICY` | ONEDAY / OFF / USEBEARERTIMER / USESPECIFICVAL | 缺省承载和默认 GBR 定时器（4G） | 不支持 USESPECIFICVAL |
| `DFTBRPLCYGGSN` | ONEDAY / OFF / USEBEARERTIMER / USESPECIFICVAL | GGSN 一次激活定时器（2G/3G） | 不支持 USESPECIFICVAL |
| `IDLEUPDATEMSG` | DISABLE / ENABLE | 空闲超时发送更新消息（2G/3G/4G） | — |

典型脚本（H-SMF 全局默认：开核查，会话级，空闲定时器 1000 分钟）：
```
SET DFTIDLETIME: HCTXCHKSW=ENABLE, HSMFTIMERLEVEL=SESSION, SMFSESIDLETIMER=1000;
```

**步骤位置**：在所属 feature 编排中位于全局基础配置阶段，与 SET APNIDLETIME（APN 粒度覆盖）配套。

## 决策点

### 决策点：网元角色开关集合（DP 0-03079）

问：哪些网元角色（SGW-C/PGW-C/GGSN/H-SMF/I-SMF/Proxy-SMF）启用空闲/不活动上下文核查？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 纯 5G 组网 | `HCTXCHKSW` + `ICTXCHKSW` + `HCTXINACHKSW`/`ICTXINACHKSW` | 仅配 H-SMF/I-SMF 相关开关 |
| 融合组网 | 多角色开关全配（SGW-C/PGW-C/GGSN + H-SMF/I-SMF） | 2G/3G/4G/5G 混合场景 |
| 不活动核查 | `HCTXINACHKSW=ENABLE` / `ICTXINACHKSW=ENABLE` | 仅 5G 用户；默认使能 |

> 取值由组网角色与本特性范围决定。不活动上下文定时器默认使能，仅对 5G 用户生效。

### 决策点：核查级别与定时器时长（DP 0-03080）

问：核查到会话级还是承载/QoS Flow 级，时长多少分钟？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 会话级（2G/3G/4G） | `GULTIMERLEVEL=SESSION` + `SESSIONTIMER` | 整会话统一空闲判断 |
| 承载级（2G/3G/4G） | `GULTIMERLEVEL=BEARER` + `BEARERTIMER` + `DFTBEARPOLICY`/`DFTBRPLCYGGSN` | 按承载/一次激活分别判断 |
| 会话级（5G） | `HSMFTIMERLEVEL=SESSION` + `SMFSESIDLETIMER` + `SMFSESINATIMER` | 5G 会话级核查 |
| QoS Flow 级（5G） | `HSMFTIMERLEVEL=QOSFLOW` + `DEDQFIDLETIMER` + `DFTQFIDLETIMER` | 业务触发 QoS 保证（WSFD-109107）等场景需单独配 DEDQFIDLETIMER |

> 时长取值范围 5~12000 分钟，由运营策略（用户活跃度、资源回收力度）决定。DEDQFIDLETIMER 不支持 0，DFTQFIDLETIMER 支持 0（表示不开启缺省 QoS Flow 核查）。

## 约束

- rule-0-03196 · severity=high · 生效时机：仅对新激活用户生效，已激活用户不受影响
- rule-0-03197 · severity=high · 一致性：SMF 会话不活动定时器时长 ≤ SMF 会话空闲定时器时长
- rule-0-03198 · severity=warning · 不适用：NB-IoT 终端不适用（NB-IoT 空闲上下文参考软参 BYTE801）
- rule-0-03199 · severity=warning · conditional：DEDQFIDLETIMER / DFTQFIDLETIMER 在 HSMFTIMERLEVEL=QOSFLOW 时必填；启用对应核查时必填
- rule-0-03200 · severity=info · 等价性：GULTIMERLEVEL=SESSION 时，对 4G 用户与"承载级 + 缺省承载和默认 GBR 定时器用承载定时器"效果相同
