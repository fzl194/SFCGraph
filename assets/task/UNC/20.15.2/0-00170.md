# SET DFTIDLETIME 原子任务
> 命令静态知识见 [SET DFTIDLETIME](command/UNC/20.15.2/SET-DFTIDLETIME.md)。本页只讲配置生成实例化时这条命令**怎么配**。

SGW-C/PGW-C/SMF/GGSN 侧**全局级**空闲/不活动上下文定时器：会话无数据传输超时后，按角色（SGW-C/PGW-C/GGSN/H-SMF/I-SMF/Proxy-SMF）和级别（会话级/承载级/QoS Flow 级）决定删除会话/承载/QoS Flow（空闲定时器，2G/3G/4G/5G 通用）或仅释放用户面资源（不活动定时器，仅 5G）。APN 粒度由 SET APNIDLETIME 覆盖，全局为本命令。

## 配置方法
按网元角色 + 级别组合选开关与时长；未输入的参数保持系统当前配置不变（可通过 LST DFTIDLETIME 查看）。

| 参数组 | 参数 | 配置方法 | 取值样例 |
|---|---|---|---|
| SGW-C 空闲 | SCTXCHKSW | 本端规划 | `ENABLE` / `DISABLE` |
| PGW-C/SGW-C 空闲 | PCTXCHKSW | 本端规划 | `ENABLE` / `DISABLE` |
| GGSN 空闲 | GCTXCHKSW | 本端规划 | `ENABLE` / `DISABLE` |
| H-SMF 空闲 | HCTXCHKSW | 本端规划 | `ENABLE` / `DISABLE` |
| H-SMF 不活动 | HCTXINACHKSW | 本端规划 | `ENABLE`（默认使能） / `DISABLE` |
| I-SMF/V-SMF 空闲 | ICTXCHKSW | 本端规划 | `ENABLE` / `DISABLE` |
| I-SMF/V-SMF 不活动 | ICTXINACHKSW | 本端规划 | `ENABLE` / `DISABLE` |
| H-SMF 核查级别 | HSMFTIMERLEVEL | 本端规划 | `SESSION`（会话级） |
| GUL 承载级别 | GULTIMERLEVEL | 本端规划 | `SESSION`（会话级）/ 承载级 |
| 缺省承载策略 | DFTBEARPOLICY | 本端规划 | `ONEDAY`（一天） |
| 承载定时器 | BEARERTIMER | 本端规划，分钟 | `5`~`12000` |
| 会话定时器 | SESSIONTIMER | 本端规划，分钟 | `5`~`12000` |
| SMF 会话空闲 | SMFSESIDLETIMER | 本端规划，分钟 | `5`~`12000` |
| SMF 会话不活动 | SMFSESINATIMER | 本端规划，分钟 | `5`~`12000` |
| 专有 QoS Flow 空闲 | DEDQFIDLETIMER | 本端规划，conditional，分钟 | `5`~`12000` |
| 缺省 QoS Flow 空闲 | DFTQFIDLETIMER | 本端规划，conditional，分钟 | `0` / `5`~`12000` |
| 空闲超时更新消息 | IDLEUPDATEMSG | 本端规划 | `ENABLE` / `DISABLE` |
| GGSN 一次激活定时器 | DFTBRPLCYGGSN | 本端规划 | `ONEDAY` |
| Proxy-SMF S8 空闲 | PROS8CTXCHKSW | 本端规划 | `ENABLE` / `DISABLE` |
| Proxy-SMF 空闲 | PROXYSMFCHKSW | 本端规划 | `ENABLE` / `DISABLE` |

典型脚本（H-SMF 全局默认：开核查，会话级，空闲定时器 1000 分钟）：
```
SET DFTIDLETIME: HCTXCHKSW=ENABLE, HSMFTIMERLEVEL=SESSION, SMFSESIDLETIMER=1000;
```

## 决策点

### 网元角色开关集合
- 问什么：哪些网元角色（SGW-C/PGW-C/GGSN/H-SMF/I-SMF/Proxy-SMF）启用空闲/不活动上下文核查。
- 取值由组网角色与本特性范围决定（如纯 5G 组网只需配 H-SMF/I-SMF 相关开关；融合组网需配多角色）。
- 不活动上下文定时器（HCTXINACHKSW/ICTXINACHKSW）默认使能，仅对 5G 用户生效。

### 核查级别与会话/承载/QoS Flow 定时器时长
- 问什么：核查到会话级还是承载/QoS Flow 级，时长多少分钟。
- H-SMF 核查级别 HSMFTIMERLEVEL / GUL 承载级别 GULTIMERLEVEL 决定粒度；时长取值范围 5~12000 分钟。
- 时长由运营策略（用户活跃度、资源回收力度）决定。

### QoS Flow 专用定时器（专有 vs 缺省）
- 问什么：专有 QoS Flow 与缺省 QoS Flow 分别配多长。
- 业务触发 QoS 保证（WSFD-109107）等场景需单独配 DEDQFIDLETIMER；DEDQFIDLETIMER 不支持 0，DFTQFIDLETIMER 支持 0。

## 约束
- 唯一性：全局一份（隐含，配合 SET APNIDLETIME 做 APN 粒度覆盖）。`severity=medium`
- 生效时机：仅对新激活用户生效（已激活用户不受影响）。`severity=high`
- 不适用：NB-IoT 终端不适用（NB-IoT 空闲上下文参考软参 BYTE801）。`severity=medium`
- 一致性：SMF 会话不活动定时器时长 ≤ SMF 会话空闲定时器时长。`severity=high`
- 等价性：GULTIMERLEVEL=SESSION 时，对 4G 用户与"承载级 + 缺省承载和默认 GBR 定时器用承载定时器"效果相同。`severity=low`
- conditional：DEDQFIDLETIMER / DFTQFIDLETIMER 在启用对应核查时必填。`severity=medium`
- Proxy-SMF：PROXYSMFCHKSW 由 DISABLE 改 ENABLE 时，仅对新创建上下文检查。`severity=medium`
