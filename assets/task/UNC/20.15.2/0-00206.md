# SET URRGRPBINDING 原子任务
> 命令静态知识见 [SET URRGRPBINDING](command/UNC/20.15.2/SET-URRGRPBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令只做一件事：把已存在的 URR 组绑到 UserProfile 上作为缺省计费属性。建议每个 UserProfile 都绑定缺省费率 URR 组，避免命中不到显式 Rule 的流量"不计费"。属计费三件套（URR → URRGROUP → 本绑定）的收尾一步。

## 配置方法

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| USERPROFILENAME | UserProfile 名称（已配置数据中获取，由 ADD USERPROFILE 创建） | `up-test` |
| DFTURRGRPNAME | 缺省 URR 组名称（已配置数据中获取，由 ADD URRGROUP 创建） | `urrgroup1` / `UrrGp_abnormal` |

典型脚本：
```
SET URRGRPBINDING:USERPROFILENAME="up-test",DFTURRGRPNAME="urrgroup1";
```

部署流程定位（费率标识场景）：ADD USERPROFILE → **SET URRGRPBINDING**（绑缺省 URR 组）→ ADD USRPROFGROUP → ADD UPBINDUPG。融合计费场景可再叠加 ADD RULE + ADD RULEBINDING 做显式 Rule 覆盖。

## 决策点

### DFTURRGRPNAME 取值来源
- 问什么：缺省 URR 组名来自哪里。
- 证据样本分布：`已配置数据中获取`（多数，引用 ADD URRGROUP 产物）/ `本端规划`（少数）。值本身是 URRGROUP 实例名，需引用既存 ADD URRGROUP 配置，非自由命名。
- 一致性：绑定的 URR 组必须已由 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md) 创建，否则绑定无效。

### 缺省计费兜底语义
- 问什么：不绑或绑哪个，决定未命中显式 Rule 的流量如何计费。
- 绑定缺省 URR 组 → 命中不到显式 Rule 的数据包按该 URR 组计费。
- 不绑缺省 URR 组 → 命中不到显式 Rule 的数据包**不计费**（漏计风险）。
- 推荐：每个 UserProfile 都绑缺省费率 URR 组做兜底。

## 约束
- 生效时机：执行后立即生效。`severity=low`
- 查看方式：本命令设定数据需通过 `LST USERPROFILE` 查看。`severity=low`
- 兜底语义：不绑缺省计费属性时未命中显式 Rule 的流量不计费，建议必绑。`severity=high`
- 前置依赖：USERPROFILE（[ADD USERPROFILE](task/UNC/20.15.2/0-00094.md)）、URRGROUP（[ADD URRGROUP](task/UNC/20.15.2/0-00093.md)）须先建。`severity=high`
