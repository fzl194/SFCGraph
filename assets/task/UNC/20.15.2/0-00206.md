# 绑定UserProfile缺省URR组（SET URRGRPBINDING）

> 命令静态知识见 [SET URRGRPBINDING](command/UNC/20.15.2/SET-URRGRPBINDING.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET URRGRPBINDING 只做一件事：把已存在的 URR 组绑到 UserProfile 上作为缺省计费属性。命令仅 2 个参数（USERPROFILENAME 必选 + DFTURRGRPNAME 可选），执行后立即生效。本命令是计费三件套（URR → URRGROUP → 本绑定）的收尾一步；典型骨架为 `ADD USERPROFILE` → **SET URRGRPBINDING** → `ADD USRPROFGROUP` → `ADD UPBINDUPG`。融合计费场景可再叠加 `ADD RULE` + `ADD RULEBINDING` 做显式 Rule 覆盖。

### 配置维度 1：缺省 URR 组绑定（参数 USERPROFILENAME/DFTURRGRPNAME）

| 取值 | 作用 | 联动参数 | 典型场景 |
|---|---|---|---|
| `USERPROFILENAME`（必选） | UserProfile 实例名（1~63 字符，不支持空格，不区分大小写） | 由 `ADD USERPROFILE` 创建 | 通用 |
| `DFTURRGRPNAME`（可选） | 缺省 URR 组名称（1~31 字符，不支持空格，不区分大小写）；留空表示不绑缺省 | 由 `ADD URRGROUP` 创建 | 在线/离线/融合计费场景通用 |

> DFTURRGRPNAME 取值必须引用既存 `ADD URRGROUP` 配置（典型实例名 `urrgroup1`/`urrgroup001`/`UrrGp_abnormal` 等），非自由命名；不绑缺省 URR 组时用户数据包按全局计费属性计费（**否则不计费**——漏计风险）。

### 配置维度 2：典型取值（按特性场景）

| 场景 | USERPROFILENAME | DFTURRGRPNAME | 适配特性 |
|---|---|---|---|
| 离线计费费率标识 | `up-test` | `urrgroup1` | WSFD-011201 支持离线计费 |
| 在线计费费率标识 | `up-test` | `urrgroup001` | WSFD-109001 Gy_Diameter在线计费 |
| 融合计费异常兜底 | `userprofile001`、`userprofile002` | `UrrGp_abnormal`（异常 URR 组） | WSFD-011206 支持融合计费 |
| IPv6 在线计费 | `up-test` | 任意已建 URR 组 | WSFD-104003 IPv6 在线计费 |

> **典型脚本**：
> ```
> SET URRGRPBINDING:USERPROFILENAME="up-test",DFTURRGRPNAME="urrgroup1";
> SET URRGRPBINDING:USERPROFILENAME="userprofile001",DFTURRGRPNAME="UrrGp_abnormal";
> ```

**步骤位置**：UserProfile 创建后的紧接步骤。`ADD USERPROFILE` → **SET URRGRPBINDING** → `ADD USRPROFGROUP` → `ADD UPBINDUPG`。融合计费场景下还可在 `SET URRGRPBINDING` 之后叠加 `ADD RULE` + `ADD RULEBINDING` 做显式 Rule 覆盖（Rule 的优先级高于缺省 URR 组兜底）。

## 决策点

### 决策点：是否绑定缺省 URR 组（DP 0-04003）

问：UserProfile 是否绑定 DFTURRGRPNAME？不绑会怎样？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 绑定缺省 URR 组（推荐） | `DFTURRGRPNAME="<已建 URR 组>"` | 所有场景；命中不到显式 Rule 的数据包按该 URR 组计费 |
| 不绑（漏计风险） | `DFTURRGRPNAME` 不填或留空 | **不推荐**；命中不到显式 Rule 的数据包**不计费**（按全局计费属性兜底可能为不计费） |

> 唯一推荐配置是"必绑"——所有特性（WSFD-011201/109001/011206/104003）均要求每个 UserProfile 绑缺省费率 URR 组做兜底。融合计费场景下建议绑异常 URR 组（如 `UrrGp_abnormal`）做容错。

### 决策点：缺省 URR 组名取值来源（DP 0-04004）

问：DFTURRGRPNAME 取值从哪来？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 引用已建 URR 组（"已配置数据中获取"） | `DFTURRGRPNAME="<ADD URRGROUP 实例名>"` | 主流场景；URR 组已通过 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md) 创建 |
| 本端规划新名（少数样本） | `DFTURRGRPNAME="<本端命名>"` | WSFD-011206 融合计费 activation 中存在 `USERPROFILENAME` 本端规划；但 DFTURRGRPNAME 仍是"已配置数据中获取" |

> DFTURRGRPNAME 引用 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md) 产物；绑定的 URR 组必须已由 ADD URRGROUP 创建，否则绑定无效（运行期无匹配规则）。

## 约束

- **缺省计费兜底语义**（critical，rule-0-04006）：UserProfile 不绑默认计费属性时，命中不到显式 Rule 的用户数据包**不计费**——必须为每个 UserProfile 绑定缺省 URR 组做兜底 — 后果：漏计费（按全局属性兜底可能为不计费），业务侧收入损失
- **前置依赖：USERPROFILE 与 URRGROUP 须先建**（critical，rule-0-04007）：USERPROFILENAME 引用 [ADD USERPROFILE](task/UNC/20.15.2/0-00094.md) 产物；DFTURRGRPNAME 引用 [ADD URRGROUP](task/UNC/20.15.2/0-00093.md) 产物；两者均未创建时绑定无效 — 后果：命令执行失败或绑定对象不存在
- **立即生效**（info，rule-0-04008）：本命令执行后**立即生效**，无"对新激活用户"限制（区别于 SET USRPROFCHARGE） — 后果：覆盖既有绑定立即影响后续数据包匹配
- **查询命令**（info，rule-0-04009）：本命令设定后的数据须通过 `LST USERPROFILE` 查看（本命令无独立查询命令） — 后果：必须用 USERPROFILE 查询命令确认绑定结果
- **适用 NF 与取值格式**（info，rule-0-04010）：适用 NF 为 PGW-C、SMF；USERPROFILENAME 长度 1~63 字符、不支持空格、不区分大小写；DFTURRGRPNAME 长度 1~31 字符、不支持空格、不区分大小写 — 后果：取值超限或含空格导致命令失败
