# SET CDRSTRGSTATUS 原子任务
> 命令静态知识见 [SET CDRSTRGSTATUS](command/UNC/20.15.2/SET-CDRSTRGSTATUS.md)。本页只讲配置生成实例化时这条命令**怎么配**。

适用 NF：SGW-C、PGW-C、SMF。用于对话单缓存目录 CHARGE1 或 CHARGE2 中的话单文件进行操作前的目录锁定/解锁。操作前必须 LOCK，操作后必须 UNLOCK。证据显示获取方法单一（本端规划 3），属 global_planned（按操作流程确定 POD/DIRECTORY/STATUS）。

## 配置方法
按"先锁定主操作目录、操作完成后解锁"的维护流程建模；常同时解锁另一目录保证只有一个被锁。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PODNAME | POD 名称（本端规划） | `pod1` |
| DIRECTORY | 话单缓存目录（本端规划，枚举） | `CHARGE1` / `CHARGE2` |
| STATUS | 目录状态（本端规划，枚举） | `LOCK` / `UNLOCK` |

典型脚本（手动操作话单缓存主目录 CHARGE1：先解锁 CHARGE2，再锁 CHARGE1，操作完后解锁 CHARGE1）：
```
SET CDRSTRGSTATUS: PODNAME="pod1",DIRECTORY=CHARGE2,STATUS=UNLOCK;
SET CDRSTRGSTATUS: PODNAME="pod1",DIRECTORY=CHARGE1,STATUS=LOCK;
SET CDRSTRGSTATUS: PODNAME="pod1",DIRECTORY=CHARGE1,STATUS=UNLOCK;
```

## 决策点

### 锁定对象选择（DIRECTORY）
- 问什么：本次操作针对 CHARGE1 还是 CHARGE2。
- 通常以 CHARGE1 为主操作目录；需保证另一目录处于 UNLOCK，避免两目录同时被锁。
- 获取方法本端规划，按局点主备/操作流程确定。

### 目录状态（STATUS）
- 问什么：LOCK（操作前锁定）还是 UNLOCK（操作后解锁）。
- 操作目录内话单文件前必须 LOCK；操作完毕必须 UNLOCK。
- 不可长期保留 LOCK 状态——锁定期间话单缓存周期扫描会导致告警被清除。

## 约束
- 立即生效。`severity=low`
- 最大记录数 2048。`severity=medium`
- CHARGE1 和 CHARGE2 不能同时锁定（互斥）。`severity=high`
- POD 缓存路径被锁定时不允许缩容。`severity=high`
- POD 缓存路径解锁动作处理完前不允许缩容。`severity=high`
- 锁定后遇话单缓存周期扫描会导致告警清除：需确认影响后使用本功能。`severity=medium`
- 操作前必须 LOCK、操作后必须 UNLOCK：不解锁会导致话单缓存无法正常运转。`severity=high`
