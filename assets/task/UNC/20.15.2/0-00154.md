# 设置话单缓存目录状态（SET CDRSTRGSTATUS）

> 命令静态知识见 [SET CDRSTRGSTATUS](command/UNC/20.15.2/SET-CDRSTRGSTATUS.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"POD + 缓存目录 + 锁定状态"三元组建模。维护流程上：先 LOCK 主操作目录 → 操作目录内话单文件 → 操作完毕 UNLOCK。常同步保证另一目录处于 UNLOCK，避免两目录同时被锁。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| POD 定位 | PODNAME | POD 名称，本端规划，字符串 1~63 | — | 指定本次操作的目标 POD | 所有 NF（SGW-C/PGW-C/SMF） |
| 缓存目录 | DIRECTORY | 枚举：CHARGE1（主）/ CHARGE2（备）/ CONVERGED1（融合主）/ CONVERGED2（融合备），本端规划 | — | 指定操作的话单缓存目录 | 离线话单 CHARGE1/2；融合话单 CONVERGED1/2 |
| 目录状态 | STATUS | 枚举：LOCK（锁定）/ UNLOCK（解锁），本端规划 | — | 控制目录锁定/解锁 | 操作前 LOCK，操作后 UNLOCK |

典型脚本（手动操作主目录 CHARGE1：先确保 CHARGE2 解锁，再锁 CHARGE1，操作完后解锁 CHARGE1）：
```
SET CDRSTRGSTATUS: PODNAME="uncpod-011-30-0-217", DIRECTORY=CHARGE2, STATUS=UNLOCK;
SET CDRSTRGSTATUS: PODNAME="uncpod-011-30-0-217", DIRECTORY=CHARGE1, STATUS=LOCK;
SET CDRSTRGSTATUS: PODNAME="uncpod-011-30-0-217", DIRECTORY=CHARGE1, STATUS=UNLOCK;
```

步骤位置：建立话单缓存（`ADD CDRSTORAGE` 系列，不在本任务范围）→ 本任务（锁定 → 操作 → 解锁）→ 配套设置缓存超期时间（`SET CDRSTORAGECTRL`，关联 [0-00153](task/UNC/20.15.2/0-00153.md)）

## 决策点

### 决策点：锁定对象选择（DP 0-03047）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| CHARGE1（离线主目录） | DIRECTORY=CHARGE1 | 操作离线话单主缓存目录 |
| CHARGE2（离线备目录） | DIRECTORY=CHARGE2 | 操作离线话单备缓存目录 |
| CONVERGED1（融合主目录） | DIRECTORY=CONVERGED1 | 操作融合计费话单主缓存目录 |
| CONVERGED2（融合备目录） | DIRECTORY=CONVERGED2 | 操作融合计费话单备缓存目录 |

通常以 CHARGE1/CONVERGED1 为主操作目录；选定任一目录锁定时，须同步保证另一目录处于 UNLOCK，避免两目录同时被锁。

### 决策点：目录状态选择（DP 0-03048）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 操作前锁定 | STATUS=LOCK | 准备操作目录内话单文件前 |
| 操作后解锁 | STATUS=UNLOCK | 话单文件操作完毕，恢复目录正常运转 |

LOCK 不可长期保留——锁定期间遇话单缓存周期扫描会导致告警被清除。

## 约束

· rule-0-03116 · severity=critical · 禁止：CHARGE1 与 CHARGE2 不能同时锁定（互斥）；CONVERGED1 与 CONVERGED2 同理
· rule-0-03117 · severity=critical · 流程约束：操作目录内话单文件前必须 LOCK，操作完毕必须 UNLOCK；不解锁会导致话单缓存无法正常运转
· rule-0-03118 · severity=high · 禁止：POD 缓存路径被锁定时不允许缩容；POD 缓存路径解锁动作处理完前也不允许缩容
· rule-0-03119 · severity=medium · 规格上限：最大记录数 2048
· rule-0-03120 · severity=warning · 副作用：锁定后遇话单缓存周期扫描会导致告警清除，需确认影响后使用本功能
