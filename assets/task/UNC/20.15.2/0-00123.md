# MOD PCRFBINDGRP 原子任务
> 命令静态知识见 [MOD PCRFBINDGRP](command/UNC/20.15.2/MOD-PCRFBINDGRP.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用网元：PGW-C、GGSN。本命令修改指定 PCRF 与 PCRF 分组的绑定关系，核心可调项为 WEIGHT（PCRF 权重），用于 PCRF 负荷分担场景下调权重以匹配规划值。前置需先 [ADD PCRFBINDGRP](task/UNC/20.15.2/0-00058.md) 建立绑定。

## 配置方法
按 PCRFGRPNAME + PCRFHOSTNAME 定位绑定关系，调整 WEIGHT。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PCRFGRPNAME | PCRF 组名（必选，1~128，已存在） | 已规划 PCRF 组名 |
| PCRFHOSTNAME | PCRF 主机名（必选，1~127，已存在） | 已规划 PCRF 主机名 |
| WEIGHT | PCRF 权重（可选，1~100） | 与各 PCRF 规划权重一致 |

## 决策点

### WEIGHT 权重值（负荷分担比例）
WEIGHT 决定该 PCRF 在 PCRFGroup 内承担业务流量的比例。调测 PCRF 负荷分担（WSFD-109101）时，需检查各 PCRF 上配置的权重值是否与规划值一致——若不一致则用本命令重新配置。具体权重分配由 PCRF 设备性能与运营商规划共同决定，存在跨场景取值分叉。

## 约束
- **立即生效**：执行后立即生效。
- **前置存在性**：PCRFGRPNAME + PCRFHOSTNAME 绑定关系必须已 ADD 成功；不存在则报错。可用 LST PCRFBINDGRP 校验。
- **WEIGHT 取值范围**：1~100。
- **名称规格**：PCRFGRPNAME 1~128 不区分大小写；PCRFHOSTNAME 1~127 不区分大小写，均不支持空格。
- **性能约束**：若 PCRF 设备性能不足以承载当前业务量，仅调 WEIGHT 无效，需联系运营商重新规划 PCRF（属规划层，非本命令职责）。
