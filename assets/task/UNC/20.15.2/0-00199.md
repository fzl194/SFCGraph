# SET SGWAPNCHGMETH 原子任务
> 命令静态知识见 [SET SGWAPNCHGMETH](command/UNC/20.15.2/SET-SGWAPNCHGMETH.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令用于 SGW-C 网元，按 APN 维度控制是否产生 S-GW 话单。优先级：APN 级（本命令）覆盖基于计费属性的全局配置（SET SGWCHGMETH）；未配时 OFFLINE 默认 INHERIT（继承计费属性配置）。仅适用于 UNC 用作 S-GW 场景。

## 配置方法

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | APN 名称（必须已在 ADD APN 中存在；已配置数据获取） | `apn-test` |
| OFFLINE | 该 APN 的 SGW 离线计费开关（本端规划） | `ENABLE`（产生 S-GW 话单）/ `INHERIT`（继承计费属性配置） |

典型脚本（指定 APN 使能 SGW 离线计费）：
```
SET SGWAPNCHGMETH: APN="apn-test",OFFLINE=ENABLE;
```

## 决策点

### 该 APN 是否在 SGW-C 产生话单
- 问什么：OFFLINE 取值。
- ENABLE → 该 APN 用户在 SGW-C 产生 S-GW 话单（用于漫游/拜访运营商间结算）。
- INHERIT → 继承 SET SGWCHGMETH 基于计费属性的配置（默认）。
- 高危：修改配置影响新激活 SGW 用户是否产生话单，配错可能导致用户无法计费。

## 约束
- 生效时机：执行后只对新激活用户生效，不影响在线用户。`severity=medium`
- 高危：配错导致 SGW 用户无法产生话单、无法计费。`severity=high`
- 前置：参数 APN 必须在 APN 表中已存在（ADD APN 已配置）才能配置成功。`severity=high`
- 规格：最大记录数 10000。`severity=low`
- 系统初始记录：OFFLINE 初始值 INHERIT。`severity=low`
- 优先级：APN 级（本命令）覆盖 SET SGWCHGMETH 计费属性级配置。`severity=medium`
- NF 限制：仅适用于 UNC 用作 S-GW。`severity=medium`
