# 配置融合计费全局参数（SET CNVRGDCHGPARA）

> 命令静态知识见 [SET CNVRGDCHGPARA](command/UNC/20.15.2/SET-CNVRGDCHGPARA.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET CNVRGDCHGPARA 是 PGW-C/SMF 融合计费（N40 接口）的"全局参数层"：单命令覆盖 25 个参数，按功能分为六组——CHF 响应异常处理、用量上报与配额管理、Trigger 控制与填写、ChargingDataRef 生成、位置信息与时间戳、消息缓存与放通。系统部署后存在初始记录，仅按需修改偏离默认值的参数。本命令的核心配置空间有 6 个维度：**CHF 响应异常动作**、**用量上报方式**、**Trigger 控制来源**、**ChargingDataRef 生成方**、**多 UPF 配额管理模式**、**位域类批量开关**。

### 配置维度 1：CHF 响应异常动作（参数 BADRSPACT + IGNORERSPAVP + CONTINUEALARM）

| 取值 | 作用 | 配套参数 | 典型场景 |
|---|---|---|---|
| `BADRSPACT=TERMINATE` | CHF 响应异常致无法计费时去活 PDU 会话 | （默认推荐） | 计费严谨场景 |
| `BADRSPACT=CONTINUE` | 放通用户业务继续，不再进行配额管理 | `CONTINUEALARM=ENABLE` 开启放通告警 | 放通不计费场景 |
| `IGNORERSPAVP=<位域>` | 忽略 CHF 响应中指定异常信元（INVOCATIONNUM/FUI/...） | 单独或组合配置 | 信元容错场景 |

### 配置维度 2：用量上报方式（参数 RPTBASEDONGU + MERGERGVTSW）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| `RPTBASEDONGU=DISABLE` | 按实际使用量上报（默认） | 标准在线计费 |
| `RPTBASEDONGU=ENABLE` | 基于 CHF 下发 GrantedUnit 做配额管理 | 配额严格管控 |
| `MERGERGVTSW=ENABLE` | RG VT 事件合并上报 | 减少 N40 信令量 |
| `MERGERGVTSW=DISABLE` | RG VT 事件单独上报（默认） | 事件粒度精细化 |

### 配置维度 3：Trigger 控制来源（参数 IGNORETRIGGER + IGNOREERRTRIGS）

| 取值 | 作用 | 配套本地配置 |
|---|---|---|
| 默认采用 CHF 响应携带的 Trigger | CHF 下发的 Trigger 列表生效（MAXNUMCCC/会话级时间/会话级流量/RG 级时间/RG 级流量/配额保持时长） | — |
| `IGNORETRIGGER=<位域>` | 忽略 CHF 指定 Trigger，采用本地配置生效 | [ADD PDUTRIGGER](task/UNC/20.15.2/0-00062.md)、[ADD RGTRIGGER](task/UNC/20.15.2/0-00069.md) |

> 若 CHF 响应未携带被忽略的 Trigger，该 Trigger 仍用本地值。

### 配置维度 4：ChargingDataRef 生成与格式（参数 CHGDATAREFGEN + CHGDATAREFFMT + CHGDATAREFUSE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| `CHGDATAREFGEN=SMF` | ChargingDataRef 由 SMF 生成（消息缓存回放功能要求） | 消息缓存回放开启 |
| `CHGDATAREFGEN=CHF` | ChargingDataRef 由 CHF 生成 | 标准流程 |
| `CHGDATAREFFMT=<格式>` | ChargingDataRef 格式（conditional） | 由整网规划 |
| `CHGDATAREFUSE=<使用方式>` | ChargingDataRef 使用方式（conditional） | 由整网规划 |

### 配置维度 5：多 UPF 配额管理（参数 MULTIUPFQUOTAM）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| `MULTIUPFQUOTAM=QUOTAINDEP` | 多 UPF 配额独立管理 | 多 UPF 部署 |

### 配置维度 6：位域类批量开关（IGNORERSPAVP / IGNORETRIGGER / IGNOREERRTRIGS）

位域类参数支持单独或组合配置（不互斥）：按位组合屏蔽 CHF 响应中不需要处理的信元或 Trigger 列表。

**典型脚本**：
```
SET CNVRGDCHGPARA:IGNORERSPAVP=FUI,BADRSPACT=TERMINATE;
SET CNVRGDCHGPARA:CHGDATAREFGEN=SMF;
SET CNVRGDCHGPARA:MULTIUPFQUOTAM=QUOTAINDEP;
SET CNVRGDCHGPARA:RPTBASEDONGU=DISABLE;
SET CNVRGDCHGPARA:MERGERGVTSW=ENABLE;
```

**步骤位置**：在所属 feature 编排中通常位于"融合计费全局参数"段——融合计费特性激活后、URR/URRGROUP/CCT 绑定之前；消息缓存回放开启前必先配 `CHGDATAREFGEN=SMF`。

## 决策点

### 决策点：CHF 响应异常处理动作（DP 0-03069）

问：CHF 响应携带异常信元致 SMF 无法计费时，去活还是放通？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 去活 PDU 会话 | `BADRSPACT=TERMINATE` | 计费严谨（默认推荐） |
| 业务继续不配额管理 | `BADRSPACT=CONTINUE` + `CONTINUEALARM=ENABLE` | 放通不计费，需开告警 |

### 决策点：用量上报与配额管理方式（DP 0-03070）

问：SMF 按实际用量还是按 CHF 下发 GrantedUnit 上报？多 UPF 配额是否独立？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 按实际用量上报 | `RPTBASEDONGU=DISABLE` | 标准在线计费（默认） |
| 按 GrantedUnit 配额管理 | `RPTBASEDONGU=ENABLE` | 配额严格管控 |
| 多 UPF 配额独立 | `MULTIUPFQUOTAM=QUOTAINDEP` | 多 UPF 部署 |
| RG VT 事件合并 | `MERGERGVTSW=ENABLE` | 减少 N40 信令量 |

## 约束

- **消息缓存回放功能需三参数联动**（warning，rule-0-03171）：消息缓存回放功能需三参数联动：`CHGDATAREFGEN=SMF`（本命令）+ [SET N40MSGSTG](task/UNC/20.15.2/0-00184.md) 的 `STGSWITCH=ENABLE` + [ADD PDUSCACT](task/UNC/20.15.2/0-00061.md) 的 `SCACT=FAILOVER/CONTINUE` — 后果：缺任一参数消息缓存回放失效
- **OFLRGTIMECALC 和 CHGDATAREFFMT**（warning，rule-0-03172）：`OFLRGTIMECALC` 和 `CHGDATAREFFMT` 功能仅针对新用户生效，已激活用户不追溯 — 后果：现网用户参数变更不立即生效，需用户重激活
- **IGNORERSPAVP/IGNORETRIGGER 为位域类型**（warning，rule-0-03173）：`IGNORERSPAVP`/`IGNORETRIGGER` 为位域类型，支持单独或组合配置，不互斥 — 后果：误用单值语义会导致部分信元未被屏蔽
- **BADRSPACT=CONTINUE 放通场景建议配合**（warning，rule-0-03174）：`BADRSPACT=CONTINUE` 放通场景建议配合 `CONTINUEALARM=ENABLE` 开启告警 — 后果：缺告警则放通状态无监控
- **系统部署完成后存在初始记录（各参数默认值见命令联机帮助表）**（info，rule-0-03175）：系统部署完成后存在初始记录（各参数默认值见命令联机帮助表），仅按需修改偏离默认值参数；该命令执行后立即生效 — 后果：变更即时影响后续融合计费行为
