# ADD SUBSCRIBERIDSEGGRP 原子任务
> 命令静态知识见 [ADD SUBSCRIBERIDSEGGRP](command/UNC/20.15.2/ADD-SUBSCRIBERIDSEGGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是计费和策略业务管理侧的号段组建模任务：把已建好的号段（`ADD IMSIMSISDNSEG`）组织成组，供下游 `ADD SGWSEGGCHGMETH`、`ADD OCSGROUP`（按号段选 OCS）、`ADD CGGRPBINDING`（按号段选 CG）等引用。

## 配置方法
按"号段组名 + 号段名 + 号段类型"三元组配置，一条记录把一个号段加入一个号段组。一个组要绑多个号段需多次执行。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| SEGGROUPNAME | IMSI/MSISDN/IMEISV 号段组名（本端规划，1~31，不支持空格，不区分大小写） | `"testmsisdngr1"` |
| SEGMENTNAME | 号段名（已配置数据获取，由 `ADD IMSIMSISDNSEG` 建好，1~31） | `"testmsisdn1"` |
| SEGMENTTYPE | 号段类型（已配置数据获取，枚举；计费场景固定取 IMSIMSISDN） | `IMSIMSISDN` |

典型脚本（组 testmsisdngr1，成员号段 testmsisdn1）：
```
ADD SUBSCRIBERIDSEGGRP: SEGGROUPNAME="testmsisdngr1", SEGMENTNAME="testmsisdn1", SEGMENTTYPE=IMSIMSISDN;
```
OCS 负荷分担场景需建多个组（testmsisdngr1/gr2），每个组分别绑各自号段，供 `ADD OCSGROUP` 按号段组负荷分担。

## 决策点

### 号段类型取值
- 问什么：号段组承载哪种用户标识。
- IMSI + MSISDN → SEGMENTTYPE=IMSIMSISDN（计费/策略场景的主流选择，所有证据固定取值）。
- IMEISV（设备标识）→ SEGMENTTYPE 取 IMEISV 对应枚举（本批证据未覆盖，按需规划）。
- 属已配置数据获取（由号段定义侧决定），跨场景枚举。

## 约束
- 规格上限：系统最大支持 128 条号段组；每条号段组最大绑定 12000 条号段；号段组单个 UNC 绑定规格 25000 个。`severity=medium`
- 唯一性（重复绑定）：同一号段可被绑入多个组；同一组内多次执行本命令追加成员号段，需保证 SEGMENTNAME 在该组内不重复。`severity=medium`
- 取值范围：SEGGROUPNAME/SEGMENTNAME 1~31，不支持空格，不区分大小写。`severity=low`
- 生效时机：命令执行后立即生效。`severity=low`
- 一致性（前置）：SEGMENTNAME 需已通过 `ADD IMSIMSISDNSEG` 建好（号段引用一致性，证据 notes 截断处即此前置说明）。`severity=medium`
