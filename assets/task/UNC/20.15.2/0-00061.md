# 增加PDU异常返回码动作（ADD PDUSCACT）

> 命令静态知识见 [ADD PDUSCACT](command/UNC/20.15.2/ADD-PDUSCACT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"返回码维度 × 处理动作"二维规划。`CCTMPLTNAME` 指向已建 CCT 模板（由 `ADD CCT` 生成），`CODETYPE` 划定匹配粒度（具体状态码 `VALUE` 或默认兜底 `DEFAULT`），`SCACT` 决定匹配到该码时的容灾轨道。`SCACT=REDIRECT` 时补重定向 IP；`SCACT=BLOCK/REDIRECT` 时补 `BLKTIMER`；`SCACT=TERM_WITH_REL/TERM_NO_REL` 时补 GTP 原因值与重新激活请求。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 具体状态码动作 | CCTMPLTNAME, CODETYPE=VALUE, STATUSCODE, SCACT | CCTMPLTNAME 本端规划（1~63，不区分大小写）；STATUSCODE 300~999 或 65535；SCACT 枚举 | — | 命中指定码时执行容灾动作 | 502/504 等 CHF 故障码逐码定义 |
| 默认兜底动作 | CCTMPLTNAME, CODETYPE=DEFAULT, SCACT | SCACT 枚举 | — | 对未单独配置的码统一动作 | 兜底容灾策略 |
| FAILOVER 联动 | SCACT=FAILOVER（按 CHF 下发或 SET FAILHANDLING） | — | — | 主备 CHF 切换 | 与 SET FAILHANDLING FHACTION 协同 |
| REDIRECT 联动 | SCACT=REDIRECT, RDVIRTIP/RDVIRTIPV6, BLKTIMER | RDVIRTIP/RDVIRTIPV6 全网规划（按地址族至少配一个）；BLKTIMER 0~1440 分钟 | RDVIRTIP=0.0.0.0/RDVIRTIPV6=::；BLKTIMER=0 | 业务重定向到指定地址 | 重定向容灾 |
| BLOCK 联动 | SCACT=BLOCK, BLKTIMER | BLKTIMER 本端规划 0~1440 分钟 | BLKTIMER=0 | 阻塞业务一段时间后可重新申请配额 | 阻塞类容灾 |
| 去活联动 | SCACT=TERM_WITH_REL/TERM_NO_REL, GTPV01CAUSE/GTPV2CAUSE, REACTREQ | GTP 原因值 0~255；REACTREQ=DISABLE/ENABLE | GTPV01CAUSE/GTPV2CAUSE=0；REACTREQ=DISABLE | 去活 PDU 会话并通知左侧网元 | 终止类容灾 |

典型脚本（证据 配置异常返回码和异常信元动作_93260161 任务示例）：
```
ADD PDUSCACT: CCTMPLTNAME="cct-test", CODETYPE=VALUE, STATUSCODE=502, SCACT=FAILOVER;
ADD PDUSCACT: CCTMPLTNAME="test", CODETYPE=VALUE, STATUSCODE=300, SCACT=BLOCK, BLKTIMER=30;
```

步骤位置：建 CCT 模板（`ADD CCT`，[0-00009](task/UNC/20.15.2/0-00009.md)）→ 本任务 → 配套 RG 级动作（`ADD RGRCACT`，[0-00068](task/UNC/20.15.2/0-00068.md)）/ 全局故障处理（`SET FAILHANDLING`）

## 决策点

### 决策点：返回码匹配粒度（DP 0-02101）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 具体状态码 | CODETYPE=VALUE, STATUSCODE（300~999 或 65535） | 502/504/601/602/603/605 等已知异常码逐码定义动作 |
| 默认兜底 | CODETYPE=DEFAULT（不配 STATUSCODE） | 对配置指定之外的异常返回码统一处理 |

动作优先级（证据 ADD-PDUSCACT 参数说明）：异常码 604/605 与内部异常码 606 的 `SCACT` 配置 > 默认动作（依 ADD CCT FHACTION / 主备 SCP 组关系）> `CODETYPE=DEFAULT` 配置 > 未配置的异常码默认动作；其他异常码的 `SCACT` 配置 > `CODETYPE=DEFAULT` 配置 > 未配置默认动作。

### 决策点：处理动作轨道（DP 0-02102）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| FAILOVER（主备切换） | SCACT=FAILOVER；优先按 CHF 下发 sessionFailover/failureHandling，无下发则按 SET FAILHANDLING | CHF 故障主备切换（证据 STATUSCODE=502） |
| CONTINUE（业务继续） | SCACT=CONTINUE | 允许业务继续，不再配额管理；去活时上报用量 |
| TERM_WITH_REL（去活并上报） | SCACT=TERM_WITH_REL, GTPV01CAUSE/GTPV2CAUSE, REACTREQ | 去活 PDU 会话并发送 Release 消息 |
| TERM_NO_REL（去活不发消息） | SCACT=TERM_NO_REL, GTPV01CAUSE/GTPV2CAUSE, REACTREQ | 去活 PDU 会话不发送 Release 消息 |
| BLOCK（阻塞业务） | SCACT=BLOCK, BLKTIMER（0~1440 分钟） | 阻塞业务一段时间（证据 STATUSCODE=300, BLKTIMER=30） |
| REDIRECT（重定向） | SCACT=REDIRECT, RDVIRTIP/RDVIRTIPV6（至少一个）, BLKTIMER | 将业务重定向到指定地址 |

## 约束

· rule-0-02251 · severity=critical · 一致性（前置）：CCTMPLTNAME 必须指向已通过 `ADD CCT` 建好的 CCT 模板，否则动作无法挂载
· rule-0-02252 · severity=critical · 取值范围：STATUSCODE 取值 300~999 或 65535；CODETYPE=VALUE 时 STATUSCODE 必选，CODETYPE=DEFAULT 时不可配 STATUSCODE
· rule-0-02253 · severity=high · 联动完整性：SCACT=REDIRECT 时 RDVIRTIP/RDVIRTIPV6 至少配一个（按用户业务地址族 IPv4/IPv6 对应配置）；SCACT=BLOCK/REDIRECT 时 BLKTIMER 控制阻塞/重定向时长（0~1440 分钟）
· rule-0-02254 · severity=medium · 规格上限：最多 2000 条记录；超规格新增失败
· rule-0-02255 · severity=low · 生效时机：命令执行后立即生效
