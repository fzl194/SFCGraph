# ADD PDUSCACT 原子任务
> 命令静态知识见 [ADD PDUSCACT](command/UNC/20.15.2/ADD-PDUSCACT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
PDU 异常返回码动作按"返回码维度 × 处理动作"规划：以 CCT 模板为载体，`CODETYPE` 划定匹配粒度（具体状态码 `VALUE` 或默认兜底 `DEFAULT`），`SCACT` 决定匹配到该码时的容灾轨道。`SCACT=FAILOVER` 时还需补重定向 IP 与 GTP 原因值；`SCACT=CONTINUE` 仅改路重发不重定向。本命令属融合计费可靠性容灾链路，常与 SET FAILHANDLING、ADD RGRCACT 配套。

| 处理动作 | 必备参数 | 典型脚本 |
|---|---|---|
| 容灾重发 FAILOVER | `CCTMPLTNAME` + `CODETYPE=VALUE` + `STATUSCODE` + `SCACT=FAILOVER` | `ADD PDUSCACT: CCTMPLTNAME="cct-test", CODETYPE=VALUE, STATUSCODE=502, SCACT=FAILOVER;` |
| 继续 CONTINUE | `CCTMPLTNAME` + `CODETYPE=VALUE` + `STATUSCODE` + `SCACT=CONTINUE` | `ADD PDUSCACT: CCTMPLTNAME="global", CODETYPE=VALUE, STATUSCODE=404, SCACT=CONTINUE;` |
| 默认兜底 | `CCTMPLTNAME` + `CODETYPE=DEFAULT` + `SCACT` | `CODETYPE=DEFAULT` 时 STATUSCODE 不需配，对未单独配置的码统一动作。 |

**FAILOVER 联动参数**（仅 `SCACT=FAILOVER` 出现）：`RDVIRTIP/RDVIRTIPV6`(重定向 IP，按主备 CHF 地址族选其一) + `BLKTIMER`(阻塞时长) + `GTPV01CAUSE/GTPV2CAUSE`(GTP 原因值) + `REACTREQ`(是否请求重新激活)。

## 决策点

### 返回码匹配粒度
问：针对具体状态码动作，还是对所有未指定码统一兜底？
- `CODETYPE=VALUE`：需配 `STATUSCODE`(300~999 或 65535)，逐码定义动作（证据中 404、502 均为此模式）。
- `CODETYPE=DEFAULT`：不配 `STATUSCODE`，对配置指定之外的异常返回码统一处理。

### 处理动作轨道
问：SMF 收到 CHF PDU 级异常码后计费会话如何处置？
- `CONTINUE`：向备 CHF 重发计费请求，不重定向用户业务（证据中 STATUSCODE=404 采用）。
- `FAILOVER`：重定向到备用 CHF，需补 `RDVIRTIP/RDVIRTIPV6`、`BLKTIMER`、GTP 原因值；与 SET FAILHANDLING FHACTION=CONTINUE/RETRY_AND_TERM 协同构成完整主备切换（证据中 STATUSCODE=502 采用）。
- 阻塞类：`BLKTIMER` 决定阻塞间隔（0~1440 分钟），用于限制重发频率。

## 约束
- 除 INDEX（无）外，`CCTMPLTNAME` 必须指向已存在的 CCT 模板。`severity=critical` 后果：动作无法挂载。
- `STATUSCODE` 取值 300~999 或 65535，`(CCTMPLTNAME, STATUSCODE)` 在同一 CCT 内唯一；同一码不可重复定义动作。`severity=critical`
- 最多 2000 条记录。`severity=medium` 后果：超规格新增失败。
- `SCACT=FAILOVER` 时 `RDVIRTIP/RDVIRTIPV6` 至少配一个；按用户业务地址族(IPv4/IPv6)对应配置。`severity=high`
- 执行后立即生效。`severity=low`
- 与 SET N40MSGSTG(STGSWITCH=ENABLE) 配套构成计费消息缓存回放；与 SET CNVRGDCHGPARA(CHGDATAREFGEN=SMF) 协同决定缓存回放数据生成端。`severity=medium`
