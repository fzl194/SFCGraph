# 增加RADIUS服务器（ADD RDSSVR）

> 命令静态知识见 [ADD RDSSVR](command/UNC/20.15.2/ADD-RDSSVR.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令在 RADIUS 服务器组下新增 RADIUS 服务器（鉴权/计费，可配主备抄送）。配置空间核心维度为 **服务器类型**（SERVERTYPE：鉴权/计费）、**主备抄送类型**（PRIFLAG）、**IP 版本 + VPN 实例** 和 **端口/密钥**。

### 配置维度1：服务器类型（参数 SERVERTYPE）

| 取值 | 作用 | 典型端口 | 典型场景 |
|---|---|---|---|
| AUTHENTICATION | 鉴权服务器（处理 Access-Request） | 1812 | 用户激活鉴权（WSFD-011306/WSFD-011308） |
| ACCOUNTING | 计费服务器（处理 Accounting-Request） | 1813 | 计费话单上报（WSFD-010802/WSFD-011306/WSFD-011307/WSFD-011308） |

### 配置维度2：主备抄送类型（参数 PRIFLAG）

| 取值 | 作用 | 联动参数 | 典型场景 |
|---|---|---|---|
| PRIMARY（默认/显式） | 主服务器 | 可选 PRIORITY（优先级）、WEIGHT（鉴权时权重） | 主鉴权/计费（WSFD-011306/WSFD-011308） |
| SECONDARY | 备服务器（主故障切换） | 无 PRIORITY | 主备冗余（WSFD-011307） |
| CARBON_COPY | 抄送服务器（同步抄一份） | 可选 ACCTATTTEMP（计费属性模板）、COPYSVRPRIORITY | RADIUS 抄送（WSFD-011307） |

### 配置维度3：IP 版本与 VPN 实例（参数 IPVERSION / SERVERIPV4 / SERVERIPV6 / VPNINSTANCE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| IPV4 + SERVERIPV4 | IPv4 地址对接 AAA | 多数场景（10.168.10.1、192.168.7.12、192.168.8.181） |
| IPV6 + SERVERIPV6 | IPv6 地址对接 AAA | IPv6 AAA |
| + VPNINSTANCE | 通过指定 VPN 实例对接 AAA | VPN 隔离（vpn_aaa/vpn_enterprise/vpn_pdn/vpn_Gi/vpntest/vpn1） |

> 主备冗余/抄送组上限：每组鉴权服务器最多 16 主 + 16 备 + 64 抄送；计费同上。

**步骤位置**：在 RADIUS 族编排中，于 `ADD RDSSVRGRP`（建服务器组）之后、`ADD APNRDSSVRGRP`（APN 绑服务器组）、`ADD UPLIST4RDS`（UP 列表）之前执行。

## 决策点

**服务器类型 + 主备抄送组合**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| AUTHENTICATION + PRIMARY | + PRIORITY（可选） | 主鉴权（WSFD-011306） |
| ACCOUNTING + PRIMARY | + PRIORITY | 主计费（WSFD-011306/WSFD-010802/WSFD-011308） |
| AUTHENTICATION/ACCOUNTING + SECONDARY | 无 PRIORITY | 主备冗余（WSFD-011307） |
| AUTHENTICATION/ACCOUNTING + CARBON_COPY | + ACCTATTTEMP + COPYSVRPRIORITY | RADIUS 抄送（WSFD-011307） |

> 主备冗余：主故障自动切备；抄送：主备之外再抄一份到第三服务器。三种模式可在同组共存（如 WSFD-011307 同时配 PRIMARY+SECONDARY+CARBON_COPY）。

## 约束

- **每组服务器数量上限**（warning）：每组鉴权/计费各最多 16 主 + 16 备 + 64 抄送 — 后果：超上限新增失败
- **RDSSVRGRPNAME 必须先建**（critical）：RDSSVRGRPNAME 必须由 `ADD RDSSVRGRP` 已配置 — 后果：组不存在则新增失败
- **CIPHERKEY/CIPHERKEYCNFM 必填且一致**（critical）：服务器密钥与确认密钥必须一致 — 后果：不一致新增失败
- **SERVERIPV4/SERVERIPV6 条件必选**（critical）：IPVERSION=IPV4 时必填 SERVERIPV4；IPV6 时必填 SERVERIPV6 — 后果：缺则新增失败
- **PORT 范围**（warning）：整数 1~65535 — 后果：超范围新增失败
- **VPNINSTANCE 必须先建**（critical）：VPNINSTANCE 若填必须由 `ADD VPNINST` 已配置 — 后果：VPN 不存在新增失败
- **WEIGHT 仅鉴权可用**（info）：WEIGHT 仅在 SERVERTYPE=AUTHENTICATION 时可选 — 后果：计费时配 WEIGHT 无效
