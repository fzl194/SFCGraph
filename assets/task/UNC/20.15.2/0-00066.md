# ADD QUOTAEXHAUSTACT 原子任务
> 命令静态知识见 [ADD QUOTAEXHAUSTACT](command/UNC/20.15.2/ADD-QUOTAEXHAUSTACT.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令把"在线 RG 配额耗尽后动作"绑定到一个融合计费模板（CCT，来自 [ADD CCT](task/UNC/20.15.2/0-00009.md)）。可选动作中，重定向类动作需要同步给出重定向 IP 地址（IPv4 / IPv6 至少一个）。该命令是融合计费模板的可选配置项，位于配额耗尽处理路径上。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| CCTMPLTNAME | 融合计费模板名称，从已配置的 `ADD CCT` 数据中获取，1~63 字符，不区分大小写 | `cct-test` |
| ACTION | 在线 RG 配额耗尽后动作，本端规划；不配时默认 0（不下发动作） | 重定向类动作值 |
| RDVIRTIP | 重定向 IPv4 地址，全网规划（仅 ACTION 为重定向类时必配） | `10.1.1.1` |
| RDVIRTIPV6 | 重定向 IPv6 地址，全网规划（仅 ACTION 为重定向类时必配） | `2001:db8::1` |

典型脚本（可选步骤，配置在线 RG 配额耗尽后的默认动作）：
```
ADD QUOTAEXHAUSTACT:CCTMPLTNAME="cct-test", ACTION=<重定向动作>, RDVIRTIP=10.1.1.1;
```

## 决策点

### 是否配置在线 RG 配额耗尽动作及动作类型
- 问什么：当 SMF 通过 [SET FAILHANDLING](task/UNC/20.15.2/0-00171.md) 配置的 TXTIMER 超时、Total Volume 到达后仍未收到 CHF 响应时，RG 配额耗尽后执行什么默认动作。
| 场景 | ACTION 配置 | 重定向地址 |
|---|---|---|
| 不下发耗尽动作 | 不配 ACTION（默认 0） | 不需要 |
| 重定向到指定 IP/Portal | 配置重定向类动作值 | RDVIRTIP 与 RDVIRTIPV6 至少配一个 |

## 约束
- 生效时机：命令执行后**只对新激活用户生效**（已激活用户不重配）。`severity=high`
- 记录上限：每个 CCT 模板维度最多可输入 **101** 条记录。`severity=medium`
- 重定向地址前置（条件必配）：当 ACTION 配置为重定向类动作时，RDVIRTIP 和 RDVIRTIPV6 **至少配置一个**。`severity=high`
- 前置校验：CCTMPLTNAME 须已由 [ADD CCT](task/UNC/20.15.2/0-00009.md) 配置。`severity=high`
- 时序联动：该命令的耗尽动作生效受 [SET FAILHANDLING](task/UNC/20.15.2/0-00171.md) 的 TXTIMER 超时时长约束（TXTIMER 未超时前不触发耗尽动作）。`severity=medium`
