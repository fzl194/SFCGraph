# 增加配额耗尽后的动作（ADD QUOTAEXHAUSTACT）

> 命令静态知识见 [ADD QUOTAEXHAUSTACT](command/UNC/20.15.2/ADD-QUOTAEXHAUSTACT.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD QUOTAEXHAUSTACT 把"在线 RG 配额耗尽后动作"绑定到一个融合计费模板（CCT，来自 [ADD CCT](task/UNC/20.15.2/0-00009.md)）。本命令为**可选**命令，是融合计费模板的可选配置项，位于配额耗尽处理路径上。当 SMF 通过 [SET FAILHANDLING](task/UNC/20.15.2/0-00171.md) 配置的 TXTIMER 超时、Total Volume 到达后仍未收到 CHF 响应时，按本命令配置的动作执行。配置空间有 2 个维度：**融合计费模板名称**（必选，引用源）、**耗尽动作**（可选，默认不下发动作；重定向类动作需联动重定向地址）。

### 配置维度 1：融合计费模板名称（参数 CCTMPLTNAME，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 融合计费模板名称 | `CCTMPLTNAME` | 本端规划；字符串 1~63，不区分大小写；由 ADD CCT 配置生成 | 无 | 指定本动作绑定的融合计费模板 | 所有在线计费配额耗尽处理场景 |

### 配置维度 2：耗尽动作与重定向地址（参数 ACTION + RDVIRTIP/RDVIRTIPV6，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 耗尽动作 | `ACTION` | 本端规划；`BLOCK`（阻塞业务）/ `REDIRECT`（重定向）/ `FORWARD`（继续转发） | 0（不下发动作） | 指定在线 RG 配额耗尽后动作 | 不下发/阻塞/重定向/继续转发 |
| 重定向 IPv4 地址 | `RDVIRTIP` | 全网规划；IPv4 地址类型 | 0.0.0.0 | ACTION=REDIRECT 时重定向 IPv4 地址 | 重定向动作（与 RDVIRTIPV6 至少配一个） |
| 重定向 IPv6 地址 | `RDVIRTIPV6` | 全网规划；IPv6 地址类型 | :: | ACTION=REDIRECT 时重定向 IPv6 地址 | 重定向动作（与 RDVIRTIP 至少配一个） |

> 当 ACTION 配置为 REDIRECT 时，RDVIRTIP 与 RDVIRTIPV6 **至少配置一个**；BLOCK/FORWARD 动作不需要重定向地址。

**典型脚本（阻塞业务）**
```
ADD QUOTAEXHAUSTACT: CCTMPLTNAME="global", ACTION=BLOCK;
```

**典型脚本（重定向到指定 IPv4 地址）**
```
ADD QUOTAEXHAUSTACT: CCTMPLTNAME="cct-test", ACTION=REDIRECT, RDVIRTIP=10.1.1.1;
```

**步骤位置**：在融合计费特性编排中通常位于"配置配额耗尽动作"步骤——在线计费 RG 配额耗尽处理阶段，紧随 [ADD CCT](task/UNC/20.15.2/0-00009.md)（CCT 模板）之后，与 [SET FAILHANDLING](task/UNC/20.15.2/0-00171.md)（TXTIMER 超时，耗尽动作触发前提）联动。

## 决策点

### 决策点：是否配置在线 RG 配额耗尽动作及动作类型（DP 0-02111）

问：当 SMF 通过 SET FAILHANDLING 配置的 TXTIMER 超时、Total Volume 到达后仍未收到 CHF 响应时，RG 配额耗尽后执行什么默认动作？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 不下发耗尽动作 | 省略本命令 或 不配 ACTION（默认 0） | 不需要配额耗尽后特殊处理 |
| 阻塞业务 | `CCTMPLTNAME` + `ACTION=BLOCK` | 配额耗尽后阻塞业务，使业务不能继续进行 |
| 重定向到指定 IP/Portal | `CCTMPLTNAME` + `ACTION=REDIRECT` + `RDVIRTIP` 和/或 `RDVIRTIPV6` | 配额耗尽后将当前业务重定向到指定地址（至少配一个重定向地址） |
| 继续转发业务 | `CCTMPLTNAME` + `ACTION=FORWARD` | 配额耗尽后允许继续转发业务 |

### 决策点：重定向地址族选择（DP 0-02112）

问：ACTION=REDIRECT 时，重定向地址用 IPv4、IPv6 还是两者？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅 IPv4 | `ACTION=REDIRECT` + `RDVIRTIP` | 用户面走 IPv4（默认 0.0.0.0） |
| 仅 IPv6 | `ACTION=REDIRECT` + `RDVIRTIPV6` | 用户面走 IPv6（默认 ::） |
| 双栈（两者） | `ACTION=REDIRECT` + `RDVIRTIP` + `RDVIRTIPV6` | 用户面双栈，按业务匹配地址族 |

> 约束：ACTION=REDIRECT 时 RDVIRTIP 与 RDVIRTIPV6 至少配置一个，否则重定向无目标地址。

## 约束

- **前置校验**（warning，rule-0-02276）：前置校验：`CCTMPLTNAME` 须已由 [ADD CCT](task/UNC/20.15.2/0-00009.md) 配置；该命令的耗尽动作生效受 [SET FAILHANDLING](task/UNC/20.15.2/0-00171.md) 的 TXTIMER 超时时长约束（TXTIMER 未超时前不触发耗尽动作）
- **生效时机**（warning，rule-0-02277）：生效时机：命令执行后**只对新激活用户生效**（已激活用户不回溯，变更后需用户重激活才生效）
- **重定向地址前置（条件必配）**（warning，rule-0-02278）：重定向地址前置（条件必配）：当 `ACTION` 配置为 `REDIRECT` 时，`RDVIRTIP` 和 `RDVIRTIPV6` **至少配置一个**（否则重定向无目标地址）
- **记录上限**（warning，rule-0-02279）：记录上限：每个 CCT 模板维度最多可输入 **101** 条记录；超规格新增失败
- **ACTION 不配时默认 0（不下发动作）**（info，rule-0-02280）：`ACTION` 不配时默认 0（不下发动作）；`RDVIRTIP` 默认 0.0.0.0、`RDVIRTIPV6` 默认 :: ；适用 NF：PGW-C、SMF；MOD/RMV/LST QUOTAEXHAUSTACT 为同对象增删改查（文字提及）
