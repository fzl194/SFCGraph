# SET STGTRIGGER 原子任务

> 命令静态知识见 [SET STGTRIGGER](command/UNC/20.15.2/SET-STGTRIGGER.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET STGTRIGGER 设置融合计费消息**缓存期间**仍需生成计费消息的触发条件。8 个 trigger 参数分别对应"时间/PDU 时长/流量/PDU 流量/RAT 更新/服务节点更新/QoS 更新/用户位置更新"8 个维度。每个 trigger 既有"用户面"阈值（TIMELIMIT/VOLUMELIMIT），也有"PDU 面"阈值（PDUTIMELIMIT/PDUVOLUMELIMIT）；4 个"变更类"trigger（RATCHG/SRVNDCHG/QOSCHG/ULCHG）是开关型取值。属于融合计费缓存配置链的一环（在 SET N40MSGSTG 之后、SET CNVRGDCHGPARA 之前）。

**参数取值样例（融合计费缓存期间 trigger 范式）**

| 参数 | 含义 | 取值样例 | 范围/枚举 | 说明 |
|---|---|---|---|---|
| TIMELIMIT | 时间阈值 | NONREPORT（默认不上报） | 枚举 | 用户面时间触发 |
| PDUTIMELIMIT | PDU 时长阈值(分钟) | — | 0~1440 | PDU 面时间触发 |
| VOLUMELIMIT | 流量阈值 | IMMEDIATE（立即） | 枚举 | 用户面流量触发 |
| PDUVOLUMELIMIT | PDU 流量阈值(MB) | `500` | 0~2147483647 | PDU 面流量触发 |
| RATCHG | RAT 更新 | NONREPORT | NONREPORT/上报值 | RAT 变更触发 |
| SRVNDCHG | 服务节点更新 | NONREPORT | NONREPORT/上报值 | 服务节点变更触发 |
| QOSCHG | QoS 更新 | NONREPORT | NONREPORT/上报值 | QoS 变更触发 |
| ULCHG | 用户位置更新 | NONREPORT | NONREPORT/上报值 | 用户位置变更触发 |

**典型行（融合计费消息缓存场景，全局 trigger）**

```
SET STGTRIGGER: VOLUMELIMIT=IMMEDIATE, PDUVOLUMELIMIT=500;
```

**典型流程（融合计费消息缓存配置链）**：SET CDRSTORAGECTRL → SET STGALARMCTRL → SET N40MSGSTG → SET STGTRIGGER（本命令）→ SET CNVRGDCHGPARA（由 SMF 生成 ChargingDataRef）。

## 决策点

### 用户面 trigger 与 PDU 面 trigger 是否成对配置

8 个 trigger 分为两组：用户面（TIMELIMIT/VOLUMELIMIT）和 PDU 面（PDUTIMELIMIT/PDUVOLUMELIMIT）。实测范式倾向"流量维度成对配"——VOLUMELIMIT=IMMEDIATE + PDUVOLUMELIMIT=500，即用户面流量达阈值立即上报、PDU 流量达 500MB 上报。是否启用 PDU 面取决于是否需要 PDU 粒度计费。

### 4 个变更类 trigger 开还是关

RATCHG/SRVNDCHG/QOSCHG/ULCHG 是"缓存期间是否因这些事件再生成计费消息"的开关。默认 NONREPORT（不上报）。是否开启取决于业务对缓存态下变更事件计费精度的要求。

| 选项 | 取值 | 含义 |
|---|---|---|
| 不上报 | NONREPORT（默认） | 缓存期间该类变更不触发新计费消息 |
| 上报 | 上报值 | 缓存期间该类变更触发新计费消息 |

### VOLUMELIMIT 取 IMMEDIATE 还是阈值枚举

实测范式用 IMMEDIATE（立即上报）。IMMEDIATE 意味着缓存期间只要有流量就立即触发，适合强调"不漏报"。其他枚举值（具体阈值类型）取决于产品枚举定义与业务容忍度。

## 约束

- **立即生效** · severity=LOW。
- **全局配置** · severity=MEDIUM · trigger 对所有融合计费会话生效，非按用户/PDN 粒度。
- **存在系统初始记录** · severity=LOW · 部署后已有初始值（TIMELIMIT 等默认 NONREPORT），覆盖前用 LST STGTRIGGER 查当前状态。
- **未输入的参数保持系统当前配置不变** · severity=MEDIUM · 非覆盖语义，按需显式传入。
- **建议与 SET N40MSGSTG 回放策略协同** · severity=MEDIUM · trigger 决定"缓存期间生成什么消息"，N40MSGSTG 决定"恢复后如何回放"，两者需一致避免缓存膨胀。
- **下游 SET CNVRGDCHGPARA 生成 ChargingDataRef** · severity=LOW · 本命令配置的 trigger 影响后续 ChargingDataRef 生成逻辑。
