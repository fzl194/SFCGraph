# UNC OSPFv3 路由发布

> **UNC 域 IPv6/双栈接入域 OSPFv3 路由骨架**（SOP §3.5 域通用）。被引用于 **WSFD-104001**（IPv6 承载）/ **WSFD-104002**（IPv4v6 双栈）/ **WSFD-104004**（IPv6 PD）3 个特性。
>
> **命令名规范（UNC 真实）**：OSPFv3 命令族**全大写** + `INTERFACE`（不叫 NETWORK）。路径：`平台服务管理/VNRS功能管理/IP服务/路由管理/OSPFv3管理/`。
>
> **C-U 对应**：UDG 域 OSPFv3 命令族独立维护（UDG 用 `OSPFv3INTERFACE` 而非 `OSPFV3INTERFACE`），与 UNC 同义但命名空间独立。
>
> **★ R1.5 待补 atom（critical，SOP §6）**：本骨架 4 条命令的 atom 在 UNC pre-build 阶段未建（_numbering.json 未收录）。配置生成遇 OSPFv3 步骤时**待 atom 补齐后**才产出。Evidence = UNC OM 命令文档路径。

## 配置方法

| 步骤 | 命令 | 关键参数 |
|---|---|---|
| 1. 创建 OSPFv3 进程 | `ADD OSPFV3` | PROCID（IPv6 进程 ID，唯一）；ROUTERID 全网唯一 |
| 2. 创建 OSPFV3 区域 | `ADD OSPFV3AREA` | PROCID, AREAID |
| 3. 创建 OSPFV3 接口（绑定 IPv6 网段） | `ADD OSPFV3INTERFACE` | PROCID, AREAID, IFNAME（**UNC 真实命令名，无 OSPFV3NETWORK 命令族**）|
| 4. 引入 IPv6 用户路由（wlr） | `ADD OSPFV3IMPORTROUTE` | PROCID, AREAID, PROTOCOL=wlr（用户本地路由）|

**典型脚本（IPv6 单栈基线）**：
```
ADD OSPFV3: PROCID=1, ROUTERID="10.1.1.1";
ADD OSPFV3AREA: PROCID=1, AREAID="0.0.0.0";
ADD OSPFV3INTERFACE: PROCID=1, AREAID="0.0.0.0", IFNAME="接口名";
ADD OSPFV3IMPORTROUTE: PROCID=1, AREAID="0.0.0.0", PROTOCOL=wlr;
```

**步骤位置**：在地址池体系（[UNC 1-00023](task/UNC/20.15.2/1-00023.md)）+ OSPFv2 路由发布（[UNC 1-00029](task/UNC/20.15.2/1-00029.md)）之后；典型用法 = IPv4 单栈只用 OSPFv2，IPv6 单栈/双栈叠加 OSPFv3。

## 场景差异

- **[WSFD-104001](feature/UNC/20.15.2/WSFD-104001.md) IPv6 承载**（IPv6 单栈）：步骤 1-4 全配（PROCID=1；PROTOCOL=wlr）
- **[WSFD-104002](feature/UNC/20.15.2/WSFD-104002.md) IPv4v6 双栈**：OSPFv2 ([UNC 1-00029](task/UNC/20.15.2/1-00029.md)) + 本骨架 OSPFv3 八件套（PROCID 与 OSPFv2 不同）
- **[WSFD-104004](feature/UNC/20.15.2/WSFD-104004.md) IPv6 PD**：步骤 1-4 全配

## 决策点

| 选项 | 影响 |
|---|---|
| IPv4 单栈 | 仅 [UNC 1-00029](task/UNC/20.15.2/1-00029.md) OSPFv2 四件套 |
| IPv6 单栈 | OSPFv2 不配，仅本骨架步骤 1-4 |
| IPv4v6 双栈 | OSPFv2 + OSPFv3 八件套 |

## 约束

- **ROUTERID 全网唯一**（critical）：OSPFv3 Router ID 必须全网唯一 — 后果：邻接建立失败
- **VPN 三者一致**（critical）：OSPFv3 进程应与对应 VPN 绑定 — 后果：路由无法迭代
- **本骨架不含独立命令外的命令**：OSPFv2 单栈路由归 [UNC 1-00029](task/UNC/20.15.2/1-00029.md)；区域认证、域标识、汇总等其他 OSPFv3 配置族归各自的 L3VPN/IP 相关骨架
- **★ 命令名修正（2026-07-12 v1.0→v1.1，R1.5）**：v1.0 写的 `ADD OSPFv3` / `ADD OSPFv3NETWORK` 是错的——UNC 真实命令族是 `ADD OSPFV3` / `ADD OSPFV3INTERFACE`（全大写 + INTERFACE 非 NETWORK）
