# 增加UPF节点（ADD UPNODE）

> 命令静态知识见 [ADD UPNODE](command/UNC/20.15.2/ADD-UPNODE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令增加指定实例名称的 UPF 节点特征标识，是网元选择族的根对象——UPF 实例的属性登记。配置空间核心维度为 **位置特征**（LOCATION）、**UPF 功能**（UPFUNCTION）、**地址分配属性**（ADDRALLOCMODE）和 **共享/锁定/辅锚点迁移开关族**。

### 配置维度1：位置特征（参数 LOCATION）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| Local（本地节点） | 只与本地 DN 连接，只能作辅锚点 | MEC 本地 UPF（WSFD-108002：UPF_Instance_0 LOCATION=Local） |
| Central（中心节点和本地节点，默认） | 既连本地 DN 也连中心 DN，既可作主锚点也可作辅锚点 | 多数场景（WSFD-213001/223002：LOCATION=Central） |
| CentralOnly（仅中心节点） | 只与中心 DN 连接，只能作主锚点 | 中心专用 UPF |

### 配置维度2：UPF 功能（参数 UPFUNCTION）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| None（默认） | 无特殊功能 | 普通锚点（WSFD-213001：UPFUNCTION=None） |
| UlClAndBp | 支持 UL CL 和 BP 分流方式 | 分流策略控制（WSFD-108002/WSFD-223002：UPFUNCTION=UlClAndBp） |

### 配置维度3：地址分配属性（参数 ADDRALLOCMODE）

| 取值 | 作用 | 典型场景 |
|---|---|---|
| INHERIT（默认） | 继承 `SET APNADDRESSATTR` 指定的全局分配属性 | 默认（WSFD-010502/107021：ADDRALLOCMODE=INHERIT） |
| SMF_ALLOC | UE 地址由 SMF/PGW-C/GGSN 分配 | SMF 集中分配 |
| UPF_FIRST | UE 地址优先由 UPF 分配，UPF 未分配则由 SMF 分配 | UPF 本地分配 |

### 配置维度4：开关族（可选，按场景组合）

| 参数 | 作用 | 默认 | 典型场景 |
|---|---|---|---|
| LOCK | UPF 锁定（SMF 不选） | FALSE | 故障/维护屏蔽（WSFD-213001 LOCK=FALSE） |
| SHAREDUPFSW | 共享 UPF 开关 | DISABLE | 主辅锚点合一时 ENABLE（WSFD-223002 SHAREDUPFSW=ENABLE，需 LOCATION=Central） |
| APSAMIGFUNC | 辅锚点故障迁移开关 | DISABLE | 辅锚点负荷分担部署时 ENABLE |
| SUPULCLBPONLY | 仅作分流 UPF | （Local 默认 Enable） | 分流专用 UPF |
| SUPATSSS | ATSSS UPF | Disable | ATSSS 场景 |
| PDUESTWITHOUTN9 | 无 N9 地址时作主锚点激活 | DISABLE | I-UPF/PSA-UPF 合一形态 |
| NGLANSW / MBSSW / SUPINTELLISRV / CAMPUSSW / UEIPNATFUNC / PROXYUPFS8USW / TOHPSASW / QOSANASW | 各类业务开关 | DISABLE | 各业务场景 |
| PATHMIGSW + TOPATHMIGSW + PATHMIGTIMER + PATHMIGEXT + SILENTSIGMIGSW | 路径迁移族 | DISABLE | 多路径容灾 |

**步骤位置**：在网元选择族编排中，是 UPF 族的起点——于 `ADD PNFPROFILE（★R1.5 atom 待补）`（建 NF Profile，关联 NFINSTANCENAME）之后、`ADD UPFBINDGRP`（UPF 入 UPF 组）、`ADD UPLIST4RDS`（UPF 入 RADIUS UP 列表）、`ADD CPGTPUADDR`（CP-UP 对接地址）之前执行。

## 决策点

**位置+功能+分配组合**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| Central + None + INHERIT | 默认 | 普通主锚点（多数场景） |
| Local + UlClAndBp | + SUPULCLBPONLY=Enable（Local 默认） | MEC 本地分流（WSFD-108002） |
| Central + UlClAndBp + SHAREDUPFSW=ENABLE | 主辅合一共享 UPF | 公私网协同（WSFD-223002） |
| CentralOnly + None | 仅中心主锚点 | 中心专用 |
| + ADDRALLOCMODE=SMF_ALLOC | SMF 分配 UE 地址 | SMF 集中地址 |
| + ADDRALLOCMODE=UPF_FIRST | UPF 优先分配 | UPF 本地地址 |

> SUPULCLBPONLY=Enable 或 LOCATION=Local 时，该 UPF 不能作 I-UPF；非 UL CL 场景需确保某位置区存在 LOCATION 非 Local 且 SUPULCLBPONLY=Disable 的 UPF。

## 约束

- **只对新激活用户生效**（info）：命令执行后只对新激活用户生效，已在线用户不受影响 — 后果：需用户重激活才应用新特征
- **最大记录数2048**（warning）：最多可输入 2048 条记录 — 后果：超规格新增失败
- **NFINSTANCENAME 与 PNFPROFILE 一致**（critical）：NFINSTANCENAME 取值需与 `ADD PNFPROFILE（★R1.5 atom 待补）` 的 NF 实例标识一致才生效 — 后果：不一致则关联关系不生效
- **NFINSTANCENAME 唯一**（critical）：UPF 实例名全局唯一 — 后果：重复新增失败
- **SHAREDUPFSW 生效条件**（warning）：SHAREDUPFSW=ENABLE 仅在 LOCATION=Central（或 `ADD APNUPFINFO（★R1.5 atom 待补）` 的位置特征=BOTH）时生效 — 后果：Local 下配置无效
- **SUPULCLBPONLY 条件可选**（warning）：仅 LOCATION=Central/CentralOnly 时可选；Local 默认 Enable — 后果：场景错配影响 I-UPF 选择
- **SUPATSSS 与 IMSIBINDUP 互斥**（critical）：NFINSTANCENAME 已绑在 IMSIBINDUP 时，SUPATSSS 不能设 ENABLE — 后果：配置失败
- **PDUESTWITHOUTN9 形态限制**（critical）：UPF 不配 N9 地址且 PDUESTWITHOUTN9=ENABLE 时，不支持 GGSN/PGW-C 形态、不支持 EPS Fallback；语音场景主锚点必须配 N9 — 后果：场景不兼容导致激活失败
- **PSACOMBINEULCL/SUPULCLBPONLY 条件可选**（warning）：仅在 LOCATION=Central/CentralOnly 时可选 — 后果：Local 下配置无效
- **PATHMIGEXT 对齐 UPG**（warning）：PATHMIGEXT 取值需与对接 UPG 一致（UPG 不做 SEID 映射→DISABLE；做映射→ENABLE） — 后果：不一致导致迁移失败
- **当前版本不支持 SILENTSIGMIGSW**（warning）：当前版本不支持此参数 — 后果：配置无效
