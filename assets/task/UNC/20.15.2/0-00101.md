# DEA SMCTX 原子任务
> 命令静态知识见 [DEA SMCTX](command/UNC/20.15.2/DEA-SMCTX.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
`ACTIONTYPE` 固定为 `START_DEA`；按 `DEATYPE` 选择去活维度（按用户身份 / 按网元节点 / 按 APN / 按失败类型 / 按比例），其余参数随维度联动。三类典型用法：

| 去活场景 | `DEATYPE` | 联动参数 | 典型脚本 |
|---|---|---|---|
| 按用户身份（IMSI/MSISDN/IMEI/号段） | `IMSI` | `IMSI`（或 `MSISDN`/`IMEI`/`SEGSTART+SEGEND`） | `DEA SMCTX:ACTIONTYPE=START_DEA,DEATYPE=IMSI,IMSI="460000123456789";` |
| 按别名 APN 批量 | `APN` | `APNTYPE=REQUESTED` + `APN=apn1`（须先 `LCK APNALIAS` 锁定） | `DEA SMCTX:ACTIONTYPE=START_DEA,DEATYPE=APN,APNTYPE=REQUESTED,APN="apn1";` |
| 回滚为本地 PCC 的会话 | `FAIL_HANDLE_TYPE` | `FAILHANDLETYPE=PCC_ROLLBACK` | `DEA SMCTX:ACTIONTYPE=START_DEA,DEATYPE=FAIL_HANDLE_TYPE,FAILHANDLETYPE=PCC_ROLLBACK;` |

**前置/配套**：按别名 APN 去活前**必须**先用 `LCK APNALIAS` 锁定该别名 APN；批量去活速率过高可配 `SET DEACTIVERATE` 限速，避免 CPU 飙升。

**生效语义**：执行后**立即生效**；只去活命令下发时**已存在**的用户，命令下发后新激活的用户不受影响。

## 决策点

### 去活维度选择
问：本次去活按什么维度定位目标用户？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 按用户身份（`IMSI`/`MSISDN`/`IMEI`/号段） | 对应身份字段 | 拨测前清场、指定用户异常会话回收 |
| 按 APN（`APN`） | `APNTYPE`+`APN`，前置 `LCK APNALIAS` | 别名 APN 批量切换/迁移 |
| 按失败类型（`FAIL_HANDLE_TYPE`） | `FAILHANDLETYPE` | PCF 回滚为本地 PCC 后的异常会话清理 |
| 按网元节点（`NODETYPE`/`NFINSTANCEID`） | 节点标识 | 指定 UPF/PCF 实例上的会话清理 |
| 按比例（`RATIOTYPE=SGWRATIO`） | `RATIOVALUE` | 灰度/分批去活，控制负荷 |

> 多维度（身份/节点/APN/失败/比例）互斥选择，本命令的 `DEATYPE` 是跨场景枚举分叉 → atom 绑定为 decision_driven。

### 用户标识字段选择
问：按用户身份去活时用哪种标识？
- `IMSI`：5~15 位数字，最常用（拨测清场标准用法）。
- `MSISDN`：1~15 位数字，按号码定位。
- `IMEI`：15~16 位，按终端定位。
- `SEGSTART`+`SEGEND`：号段范围去活。

## 约束
- 执行后**立即生效**。`severity=low`
- 只去活命令下发时**已存在**的用户；命令后新激活用户不受影响。`severity=medium` 后果：若去活与接入并发，新用户不会被清。
- 批量去活使系统负荷升高、CPU 上升，去活完成后恢复。`severity=medium` → 配 `SET DEACTIVERATE` 限速。
- 按别名 APN 去活**必须前置** `LCK APNALIAS` 锁定，否则去活不生效。`severity=high`
- `IMSI` 取值 5~15 位纯数字；`MSISDN` 1~15 位；`IMEI` 15~16 位；`APN` 1~63 位（`-`/数字/字母/`.`，不以此约束开头）。`severity=medium`
- `HOLDTIME`（0~1440 分钟）与 `HOLDPDPTYPE=VOLTE` 联用，超时前不删语音承载。`severity=low`
- 按 APN 去活且 Radius 支持可选计费时，单用户去活可不向 Radius 发 Accounting Stop Request。`severity=low`
