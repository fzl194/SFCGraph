## 配置方法

按配置维度组织如下「字典」：

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 操作类型 | `ACTIONTYPE` | `START_DEA`（开始去活）、`STOP_DEA`（停止去活） | 无 | 决定是去活还是停止去活 | 全部场景 |
| 去活方式 | `DEATYPE` | `IMSI`/`MSISDN`/`IMEI`/`IMSI_SEG`/`MSISDN_SEG`/`APN`/`PEER_NODE_TYPE`/`PCSCFGROUP`/`FAIL_HANDLE_TYPE`/`RAT_TYPE`/`TIMESTAMP`/`USER_PROFILE`/`CHARGING_TYPE`/`BEARER_STATE`/`LAC_GROUP`/`TAC_GROUP`/`NGLANGROUPID`/`RATIO`/`MULTIDNN_DEDDNN` | 无 | 定位去活目标的维度 | 全部场景 |
| 用户身份 | `IMSI`/`MSISDN`/`IMEI` | IMSI 5~15位纯数字/MSISDN 1~15位/IMEI 15~16位 | 无 | 按用户身份标识定位 | 拨测清场、异常会话回收 |
| 号段范围 | `SEGSTART`+`SEGEND` | 5~15位数字，不足补0/9 | 无 | 按号段批量去活 | 整系统去活（起始全0截止全9） |
| APN去活 | `APNTYPE`+`APN` | `REQUESTED`/`SERVICE` + APN名（1~63位） | `SERVICE` | 按APN名称去活 | 别名APN批量切换/迁移 |
| 网元节点 | `NODETYPE`+`NFINSTANCEID`/`NODEIPVERSION`+`NODEIPV4/V6` | UPF/PCF/PCRF/MME_IP/SGW_IP/PGW_IP/EPDG_IP/SGSN_IP/AMF/HSMF/ISMF/VSMF... | 无 | 按对端网元类型定位 | 指定UPF/PCF实例上的会话清理 |
| 失败类型 | `FAILHANDLETYPE` | `AUTHFAIL_BYPASS`/`PCC_ROLLBACK`/`ACCTFAIL_BYPASS`/`ONLINETOOFFLINE` | 无 | 按旁路处理失败类型去活 | PCF回滚为本地PCC后的异常会话清理 |
| 比例去活 | `RATIOTYPE`+`RATIOVALUE` | `SGWRATIO`/`PGWRATIO`/`SPGWRATIO`/`GGSNRATIO`/`N11SMFRATIO`/`HSMFRATIO`/`I_VSMFRATIO`/`PROXY_PGWRATIO`/`ALLRATIO`/`PROXY_GGSNRATIO`/`PROXY_SMFS8RATIO`/`PROXY_SMFRATIO` + 1~100% | `100` | 按比例分批去活 | 灰度/分批去活，控制负荷 |
| 承载保持 | `HOLDPDPTYPE`+`HOLDTIME` | `VOLTE`/`CONNECTION` + 0~1440分钟 | `0` | 超时前保留语音/连接态承载 | VoLTE用户保护 |
| 时间戳 | `STARTDATE`+`STARTTIME`/`ENDDATE`+`ENDTIME` | YYYY/MM/DD / HH:MM:SS | 无 | 按激活时间范围去活 | 定时清理历史会话 |
| 用户模板 | `USERPROFILE` | 字符串1~63 | 无 | 按用户模板去活 | 需先SET USERPROFILELOCK锁定 |

**典型脚本：**

```mml
# 按用户身份去活
DEA SMCTX:ACTIONTYPE=START_DEA,DEATYPE=IMSI,IMSI="460000123456789";

# 按APN批量去活（须先LCK APN锁定）
DEA SMCTX:ACTIONTYPE=START_DEA,DEATYPE=APN,APNTYPE=REQUESTED,APN="apn1";

# 按失败类型回滚为本地PCC
DEA SMCTX:ACTIONTYPE=START_DEA,DEATYPE=FAIL_HANDLE_TYPE,FAILHANDLETYPE=PCC_ROLLBACK;

# 停止去活操作
DEA SMCTX:ACTIONTYPE=STOP_DEA;
```

**前置/配套约束：**
- 按别名APN去活**必须前置** `LCK APN` 锁定该别名APN
- 按用户模板去活须先 `SET USERPROFILELOCK` 锁定用户模板
- 批量去活速率过高可配 `SET DEACTIVERATE` 限速（默认1000个/秒）
- 批量去活进行中及完成后10s内**禁止**执行APN相关配置命令
- 建议用 `DSP DEASMCTXSTATUS` 确认无正在进行的去活操作后再执行新去活

**生效语义：** 执行后**立即生效**；只去活命令下发时**已存在**的用户，命令下发后新激活的用户不受影响。

**步骤位置：** 本命令为运维诊断命令，通常在拨测前清场、异常会话回收、计费策略回滚等场景下手动执行，无固定配置顺序要求。

## 决策点

### 决策点：去活维度选择（DP 0-01240）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 按用户身份（IMSI/MSISDN/IMEI/号段） | 对应身份字段 | 拨测前清场、指定用户异常会话回收 |
| 按APN（APN） | `APNTYPE`+`APN`，前置 `LCK APN` | 别名APN批量切换/迁移 |
| 按网元节点（PEER_NODE_TYPE） | `NODETYPE`+节点标识 | 指定UPF/PCF实例上的会话清理 |
| 按失败类型（FAIL_HANDLE_TYPE） | `FAILHANDLETYPE` | PCF回滚为本地PCC后的异常会话清理 |
| 按比例（RATIO） | `RATIOTYPE`+`RATIOVALUE` | 灰度/分批去活，控制负荷 |
| 按时间戳（TIMESTAMP） | `STARTDATE/STARTTIME`+`ENDDATE/ENDTIME` | 定时清理历史激活用户 |
| 按用户模板（USER_PROFILE） | `USERPROFILE`，前置 `SET USERPROFILELOCK` | 批量模板用户去活 |
| 按承载状态（BEARER_STATE） | `BEARERSTATE` | 按连接态/空闲态去活 |
| 按计费类型（CHARGING_TYPE） | `CHARGINGTYPE` | 按在线/离线计费类型筛选 |
| 按RAT类型（RAT_TYPE） | `RATTYPE` | 按2G/3G/4G/5G网络类型筛选 |
| 按TAC/LAC组 | `TAC_GROUP`/`LAC_GROUP` | 按小区组批量去活 |
| 按5G LAN组ID | `NGLANGROUPID` | 按5G LAN组去活 |
| 按P-CSCF组 | `PCSCFGROUPNAME`+`PCSCFIPVERSION`+`PCSCFIPV4/V6` | 按P-CSCF组去活 |
| 按智能分流专用DNN | `MULTIDNN_DEDDNN`+`DEDDNN` | 拜访地智能分流场景 |

> 多维度互斥选择，`DEATYPE` 是跨场景枚举分叉 → atom 绑定为 **decision_driven**。

### 决策点：用户标识字段选择（DP 0-01241）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| `IMSI` | IMSI 5~15位纯数字 | 最常用，拨测清场标准用法 |
| `MSISDN` | MSISDN 1~15位 | 按手机号码定位 |
| `IMEI` | IMEI 15~16位 | 按终端设备定位 |
| `SEGSTART`+`SEGEND` | 号段范围5~15位 | 号段批量去活；全0~全9实现整系统去活 |

> 由运维意图决定，按用户身份去活时用哪种标识字段 → **decision_driven**。

## 约束

· rule-0-01536 · severity=critical · 按APN去活**必须前置** `LCK APN` 锁定该别名APN，否则去活不生效（`LOCKDEACTIVE=ENABLE`时若未锁定则拒绝执行）

· rule-0-01537 · severity=high · 批量去活进行中及完成后10s内**禁止**执行APN相关配置命令，否则可能导致去活失败或系统异常

· rule-0-01538 · severity=high · 按用户模板去活**必须前置** `SET USERPROFILELOCK` 锁定用户模板，否则无法按模板匹配去活

· rule-0-01539 · severity=medium · 执行后**立即生效**，且只去活命令下发时**已存在**的用户；命令后新激活用户不受影响。若去活与接入并发，新用户不会被清

· rule-0-01540 · severity=medium · 批量去活会使系统负荷升高、CPU上升；默认去活速率1000个/秒，可配 `SET DEACTIVERATE` 限速
