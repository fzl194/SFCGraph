# ADD TNFBINDGRP 原子任务
> 命令静态知识见 [ADD TNFBINDGRP](command/UNC/20.15.2/ADD-TNFBINDGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是服务化接口注册与服务发现（本地 NRF 功能）侧的成员绑定任务：在已建好的目标 NF 组（`ADD TNFGRP`）和目标 NF 实例（`ADD TNFINS` + `ADD TNFINSIP`）之间建立绑定关系。融合计费场景（WSFD-011206）用于把 CHF 实例绑进 CHF 组。

## 配置方法
按"目标 NF 组索引 + 目标 NF 实例索引"二元组配置一条绑定记录。一个组绑多个实例需多次执行。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| TNFGRPINDEX | 目标 NF 组索引（已配置数据获取，由 `ADD TNFGRP` 建好，0~511） | `1` |
| TNFINSINDEX | 目标 NF 实例索引（已配置数据获取，由 `ADD TNFINS` 建好，0~2047） | `1` |
| PRIORITY | 优先级（可选，全网规划，1~255，越小越高，默认 10）— 主备场景填，负荷分担可不填 | `10` |
| WEIGHT | 权重（可选，全网规划，1~65535，越大权重越大）— 负荷分担场景填 | `100` |

典型脚本（CHF 组 1 绑 CHF 实例 1，CHF 组 2 绑 CHF 实例 2）：
```
ADD TNFBINDGRP: TNFGRPINDEX=1, TNFINSINDEX=1;
ADD TNFBINDGRP: TNFGRPINDEX=2, TNFINSINDEX=2;
```

## 决策点

### 选择模式（负荷分担 vs 主备）
- 问什么：组内多实例是按权重分担流量，还是按优先级主备倒换。
- 负荷分担 → 填 WEIGHT（按容量比例分配），PRIORITY 取默认或一致。
- 主备 → 填 PRIORITY（主实例优先级数值小=高优先级），主故障自动切备。
- 属全网规划决策（取决于 CHF/PCF 等 NF 的部署拓扑）。

### 优先级与权重的填法
- 问什么：PRIORITY/WEIGHT 两个可选参数取何值。
- 主备：主实例 PRIORITY=较小值（如 1），备实例 PRIORITY=较大值（如 10）。
- 负荷分担：按实例容量比填 WEIGHT（如 100/100 等比，或 70/30 按容量）。
- 不填 → PRIORITY 默认 10，WEIGHT 无默认（负荷分担必填）。

## 约束
- 规格上限：最多 4096 条记录。`severity=low`
- 取值范围：TNFGRPINDEX 0~511；TNFINSINDEX 0~2047；PRIORITY 1~255（越小越高）；WEIGHT 1~65535（越大越大）。`severity=low`
- 生效时机：命令执行后立即生效。`severity=low`
- 一致性（前置）：TNFGRPINDEX 需已通过 `ADD TNFGRP` 建好（NF 组存在）；TNFINSINDEX 需已通过 `ADD TNFINS` 建好且 `ADD TNFINSIP` 配好 IP（实例与 IP 存在）。`severity=medium`
- 修改：已建绑定用 `MOD TNFBINDGRP` 改 PRIORITY/WEIGHT（不可改索引键）。`severity=low`
