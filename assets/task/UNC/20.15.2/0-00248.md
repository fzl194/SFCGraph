# 创建OSPF引入路由（ADD OSPFIMPORTROUTE）

> 命令静态知识见 [ADD OSPFIMPORTROUTE](command/UNC/20.15.2/ADD-OSPFIMPORTROUTE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

该命令将其他协议（直连/BGP/静态/OSPF 等）的外部路由引入 OSPF 域中发布。配置空间核心维度为 **引入协议类型**（PROTOCOL）和 **开销/标签/类型开关**（IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG 三组开关联动 COST/TAG/TYPE）。

### 配置维度1：引入协议类型（参数 PROTOCOL）

| 取值 | 作用 | 联动条件可选参数 | 典型场景 |
|---|---|---|---|
| Direct | 引入直连路由 | 无 | IPFD-016000 OSPF over IPsec：PROTOCOL=Direct |
| BGP | 引入 BGP 路由 | PROTOCOLPROCID（BGP 进程号）、PERMITIBGPCFG（允许引入 IBGP） | 与 BGP 互引 |
| OSPF | 引入其他 OSPF 进程路由 | PROTOCOLPROCID（其他 OSPF 进程号） | 进程间互引 |
| Static/ISIS/... | 引入其他协议路由 | 视协议而定 | 多协议互引 |

### 配置维度2：开销/标签/类型开关（IMPTCOSTCFG/IMPTTAGCFG/IMPTTYPECFG）

| 开关 | 联动必选参数 | 作用 |
|---|---|---|
| IMPTCOSTCFG=TRUE | COST | 指定引入路由开销 |
| IMPTTAGCFG=TRUE | TAG | 指定引入路由标签 |
| IMPTTYPECFG=TRUE | TYPE | 指定外部 LSA 度量类型（Type1/Type2） |

> 默认（开关 FALSE）按系统默认开销/标签/类型发布。

**步骤位置**：在 OSPF 族编排中，于 `ADD OSPF`（建进程）之后、按需在区域配置之后执行。常配 `ROUPOLINAME`（路由策略）做过滤。

## 决策点

**引入协议选择**（UNC pre-build 内嵌，不编号——按 `assets/CLAUDE.md` §5.7）：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| Direct | 默认开销/标签 | IPFD-016000 直连路由发布到 OSPF |
| BGP | + PROTOCOLPROCID + 可选 PERMITIBGPCFG | 与 BGP 互通 |
| OSPF | + PROTOCOLPROCID | 进程间互引 |

## 约束

- **PROCID 必须先建**（critical）：PROCID 必须由 `ADD OSPF` 已配置 — 后果：进程不存在则新增失败
- **PROTOCOL 必填**（critical）：必须指定引入协议类型 — 后果：缺则新增失败
- **条件必选参数**（warning）：IMPTCOSTCFG=TRUE 时必填 COST；IMPTTAGCFG=TRUE 时必填 TAG；IMPTTYPECFG=TRUE 时必填 TYPE；PROTOCOL=OSPF/BGP 时 PROTOCOLPROCID 视情况必填 — 后果：缺则新增失败
- **ROUPOLINAME 长度**（warning）：字符串 1~200 — 后果：超长新增失败
- **TOPOID 固定**（info）：目前只支持 0 — 后果：非 0 取值不被接受
