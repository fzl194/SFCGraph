# ADD RESULTCODECTRL 原子任务
> 命令静态知识见 [ADD RESULTCODECTRL](command/UNC/20.15.2/ADD-RESULTCODECTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令把"对端结果码 → PCC 流程处理动作"映射挂到一个 PCC 模板（PCCTEMPLATE）。结果码分两类来源：Gx 返回码（GXRESULTCODEVAL，1xxx-9xxx 格式）与 N7 返回码（N7RESULTCODEVAL，300-599 格式），由接口类型 INTFTYPE 决定取哪一类。每个 PCC 流程（Initial/Update/Terminate）有两组动作：标准组（INITACTION/UPDATEACTION/TERMINACTION）与直连对端组（DIRECTINITACT/DIRECTUPDACT/DIRECTTERMINACT，直连组网时生效）；动作回滚后可由对应 HOLDTMSW 开关使能 Holding-Time。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PCCTEMPLATE | PCC 模板名称，从已配置数据中获取，1~63 字符 | `pcctemplate` / `templatetest` |
| INTFTYPE | 接口类型，与对端协商（决定结果码字段） | `INTFTYPE_GX` / `INTFTYPE_N7` |
| VENDORID | 设备提供商标识，全网规划，0~65535 | `0` |
| GXRESULTCODEVAL | Gx 返回码，与对端 PCRF/DRA 协商（INTFTYPE_GX 时配），1xxx-9xxx 或单值 | `3xxx` / `3004` |
| N7RESULTCODEVAL | N7 返回码，与对端 PCF/SCP 协商（INTFTYPE_N7 时配），300-599 或 3xx-5xx | `504` / `500` |
| INITACTION / UPDATEACTION / TERMINACTION | 标准 Initial/Update/Terminate 流程处理动作，本端规划 | `FAILOVER` / `LOCALPCC` / `INHERIT_PCC` |
| DIRECTINITACT / DIRECTUPDACT / DIRECTTERMINACT | 直连对端 Initial/Update/Terminate 流程处理动作，本端规划（直连组网时配，默认 INVALID） | `FAILOVER` |
| INITHOLDTMSW / UPDHOLDTMSW / DIRECTINITHTSW / DIRECTUPDHTSW | 流程回滚后使能 Holding-Time，本端规划（默认 DISABLE） | `ENABLE` |
| REACTREQ / DIRECTREACTREQ | 重新激活请求，本端规划（默认 DEFAULT） | `DEFAULT` |

典型脚本（Gx 接口，标准流程动作）：
```
ADD RESULTCODECTRL:PCCTEMPLATE="templatetest", INTFTYPE=INTFTYPE_GX, VENDORID=0, GXRESULTCODEVAL="3xxx", INITACTION=FAILOVER, UPDATEACTION=FAILOVER, TERMINACTION=FAILOVER;
```
典型脚本（N7 接口，直连对端流程动作 + Holding-Time）：
```
ADD RESULTCODECTRL:N7RESULTCODEVAL="504", PCCTEMPLATE="pcctemplate", DIRECTINITACT=LOCALPCC, DIRECTINITHTSW=ENABLE, DIRECTUPDACT=INHERIT_PCC, DIRECTUPDHTSW=ENABLE, INTFTYPE=INTFTYPE_N7;
```

## 决策点

### 接口类型与对应结果码字段
- 问什么：本条记录针对哪个策略接口、用哪一类结果码匹配。
| INTFTYPE | 结果码字段 | 码值格式 | 协商对象 |
|---|---|---|---|
| INTFTYPE_GX | GXRESULTCODEVAL | 1000-9999 / 1xxx-9xxx | 对端 PCRF/DRA |
| INTFTYPE_N7 | N7RESULTCODEVAL | 300-599 / 3xx-5xx | 对端 PCF/SCP |

### 标准动作 vs 直连对端动作
- 问什么：是配置标准 PCC 流程动作，还是配置直连组网（DirectInitAct/DirectUpdAct/DirectTerminAct）下的对端流程动作。
| 组网模式 | 配置参数组 | 备注 |
|---|---|---|
| 标准组网 | INITACTION/UPDATEACTION/TERMINACTION | 通用 |
| 直连组网 | DIRECTINITACT/DIRECTUPDACT/DIRECTTERMINACT | 直连对端时生效；配置为 INVALID 时回退取标准动作 |

### 动作取值（回滚后是否使能 Holding-Time）
- 问什么：流程动作回滚后是否启用 Holding-Time 保留策略。
| 流程动作典型值 | 含义 | 是否常配 HOLDTMSW |
|---|---|---|
| FAILOVER | 回滚到备用 / 故障切换 | 常配 INITHOLDTMSW/UPDHOLDTMSW=ENABLE |
| LOCALPCC | 回滚到本地 PCC | 常配（N7 接口样例） |
| INHERIT_PCC | 继承 PCC 策略 | 常配（N7 接口样例） |

## 约束
- 生效时机：命令执行后**立即生效**。`severity=medium`
- 记录上限：最大记录数 **2100**。`severity=medium`
- 优先级被覆盖：Gx 接口回滚策略下，当 [SET FHBYPASS](task/UNC/20.15.2/0-00172.md) 的 GXERRRC=ENABLE 时，SET FHBYPASS 优先级**高于**本命令（紧急旁路优先）。`severity=high`
- 直连组网回退：直连组网模式下，当 DirectInitAct/DirectUpdAct/DirectTerminAct 配置为 INVALID 时，取对应 InitAction/UpdateAction/TerminAction。`severity=medium`
- 结果码须协商：GXRESULTCODEVAL / N7RESULTCODEVAL 的具体取值必须与对端 PCRF/DRA/PCF/SCP 确认，文档样例仅为举例。`severity=high`
- 前置校验：PCCTEMPLATE 须已配置（PCC 模板数据）。`severity=high`
- 书面认可约束：优先级更高的 [SET FHBYPASS](task/UNC/20.15.2/0-00172.md) 影响用户控制策略，须获得客户书面认可后方可使用（本命令本身无此约束，但旁路覆盖时受其影响）。`severity=medium`
