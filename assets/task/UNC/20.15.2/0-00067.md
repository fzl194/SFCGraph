# 增加返回码控制（ADD RESULTCODECTRL）

> 命令静态知识见 [ADD RESULTCODECTRL](command/UNC/20.15.2/ADD-RESULTCODECTRL.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

按"PCC 模板 + 接口类型 + 结果码 + 流程动作"四元组配置一条返回码控制记录。接口类型 INTFTYPE 决定结果码字段（Gx 用 GXRESULTCODEVAL，N7 用 N7RESULTCODEVAL）。每个 PCC 流程分标准动作组（INITACTION/UPDATEACTION/TERMINACTION）与直连对端动作组（DIRECTINITACT/DIRECTUPDACT/DIRECTTERMINACT，直连组网时生效）；回滚类动作（LOCALPCC/INHERIT_PCC）可联动 HOLDTMSW 使能 Holding-Time。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| Gx 接口结果码匹配 | PCCTEMPLATE, INTFTYPE=INTFTYPE_GX, VENDORID, GXRESULTCODEVAL | PCCTEMPLATE 已配置数据获取（global 或 ADD PCCTEMPLATE 建好的模板名）；VENDORID 全网规划 0~65535；GXRESULTCODEVAL 对端协商 1000-9999/1xxx-9xxx | — | 匹配 Gx 接口返回码 | PCRF/DRA 组网 |
| N7 接口结果码匹配 | PCCTEMPLATE, INTFTYPE=INTFTYPE_N7, N7RESULTCODEVAL | N7RESULTCODEVAL 对端协商 300-599/3xx-5xx | — | 匹配 N7 接口返回码 | PCF/SCP 组网 |
| 标准 PCC 流程动作 | INITACTION, UPDATEACTION, TERMINACTION | 枚举：DEFAULT/FAILOVER/TERMINATE/LOCALPCC/INHERIT_PCC/CONTINUE/INHERIT | global 默认 DEFAULT/CONTINUE/DEFAULT；指定模板默认 INHERIT | 控制创建/更新/删除响应消息中携带该返回码时的处理动作 | 通用组网 |
| 直连对端流程动作 | DIRECTINITACT, DIRECTUPDACT, DIRECTTERMINACT | 枚举：含 FAILOVERALL/INVALID 等 | INVALID | 直连组网时对端流程动作；INVALID 回退取标准动作 | 直连组网 |
| 回滚后 Holding-Time | INITHOLDTMSW, UPDHOLDTMSW, DIRECTINITHTSW, DIRECTUPDHTSW | ENABLE/DISABLE | DISABLE | 回滚为本地 PCC 后按 holding-time 保活，超时去活 | LOCALPCC/INHERIT_PCC 联动 |
| 重新激活请求 | REACTREQ, DIRECTREACTREQ | DISABLE/ENABLE/DEFAULT | DEFAULT | 异常结果码去活时是否通知用户重新激活（Gx 接口不支持此参数） | UPDATEACTION/DIRECTUPDACT=TERMINATE 联动 |

典型脚本（Gx 全局，更新响应 5xxx 执行 FAILOVER）：
```
ADD RESULTCODECTRL:PCCTEMPLATE="global",INTFTYPE=INTFTYPE_GX,VENDORID=0,GXRESULTCODEVAL="5xxx",INITACTION=DEFAULT,UPDATEACTION=FAILOVER;
```
典型脚本（Gx 指定模板，创建响应 5140 执行 LOCALPCC）：
```
ADD RESULTCODECTRL:PCCTEMPLATE="pcctemplate",INTFTYPE=INTFTYPE_GX,VENDORID=10415,GXRESULTCODEVAL="5140",INITACTION=LOCALPCC,UPDATEACTION=LOCALPCC;
```

步骤位置：建 PCC 模板（`ADD PCCTEMPLATE`）→ 本任务 → 关联失败旁路（`SET FHBYPASS`）/PCC 故障处理（`SET PCCFAILACTION`）

## 决策点

### 决策点：接口类型与对应结果码字段（DP 0-02113）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| INTFTYPE_GX | GXRESULTCODEVAL（1000-9999/1xxx-9xxx）+ VENDORID | 与 PCRF/DRA 对接的 Gx 接口 |
| INTFTYPE_N7 | N7RESULTCODEVAL（300-599/3xx-5xx） | 与 PCF/SCP 对接的 N7 接口 |

### 决策点：标准动作 vs 直连对端动作（DP 0-02114）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 标准组网 | INITACTION/UPDATEACTION/TERMINACTION | 通用组网 |
| 直连组网 | DIRECTINITACT/DIRECTUPDACT/DIRECTTERMINACT | 直连对端时生效；配置为 INVALID 时回退取标准动作 |

## 约束

- **优先级覆盖**（warning，rule-0-02281）：优先级覆盖：Gx 接口回滚策略下，当 SET FHBYPASS 的 GXERRRC=ENABLE 时，SET FHBYPASS 命令优先级高于本命令（紧急旁路优先）
- **前置校验**（warning，rule-0-02282）：前置校验：PCCTEMPLATE 须已配置（global 或已通过 ADD PCCTEMPLATE 建好的 PCC 模板名）
- **结果码须协商**（warning，rule-0-02283）：结果码须协商：GXRESULTCODEVAL/N7RESULTCODEVAL 的具体取值必须与对端 PCRF/DRA/PCF/SCP 确认，文档样例仅为举例
- **直连组网回退**（info，rule-0-02284）：直连组网回退：直连组网模式下，当 DirectInitAct/DirectUpdAct/DirectTerminAct 配置为 INVALID 时，取对应标准动作 InitAction/UpdateAction/TerminAction
- **规格上限**（info，rule-0-02285）：规格上限：最大记录数 2100；超时返回码 504 不受本配置控制（执行一次 failover）
