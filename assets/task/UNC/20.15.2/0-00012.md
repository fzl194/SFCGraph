# 配置CG组（ADD CGGROUP）

> 命令静态知识见 [ADD CGGROUP](command/UNC/20.15.2/ADD-CGGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD CGGROUP 用来建一个空的 CG 组（仅含组 ID 和描述），本身不含业务策略。它是离线计费 CG 组管理三件套（`ADD CG` → `ADD CGGROUP` → [ADD CGBINDING](task/UNC/20.15.2/0-00011.md)）的第二步——组创建后由 [ADD CGBINDING](task/UNC/20.15.2/0-00011.md) 装入 CG 实例，再由 [ADD CGGRPBINDING](task/UNC/20.15.2/0-00013.md) 或 `SET GLBCGGROUP` 把组绑到离线计费模板或设为全局 CG 组才会被实际使用。配置空间有 2 个维度：**组 ID**（必选，全局唯一标识）、**描述**（可选，运维识别用）。

### 配置维度 1：组 ID（参数 CGGRPID，必选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| CG 组 ID | `CGGRPID` | 整数 1~32 | 无 | CG 组的全局唯一标识 | 全局规划分配；后续 [ADD CGBINDING](task/UNC/20.15.2/0-00011.md) 与 [ADD CGGRPBINDING](task/UNC/20.15.2/0-00013.md) 均引用此 ID |

> 取值范围 1~32 即全系统最大 32 条记录；ID 全局唯一不可重复。

### 配置维度 2：描述（参数 DESCRIPTION，可选）

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| CG 组描述 | `DESCRIPTION` | 字符串 1~63 字符 | 无 | 便于运维识别组用途 | 建议填写（如"主用CG组"），便于多人协作时区分 |

> 描述字段为可选项，但强烈建议填写——仅建组不带描述在多组并存时难以识别。

**典型脚本**

```
ADD CGGROUP: CGGRPID=1, DESCRIPTION="CGGroup1";
```

**配置链路（本命令位置）**

```
ADD CG              # 建一个 CG 实例（IP/端口/协议）
ADD CGGROUP   ← 本命令    # 建 CG 组（CGGRPID）
ADD CGBINDING       # 把 CG 绑进组，给优先级
ADD CGGRPBINDING    # 把组绑到离线计费模板（或 SET GLBCGGROUP 设为全局）
```

**步骤位置**：在所属 feature 编排中通常位于"配置 CG 组"步骤——紧随 `ADD CG` 建 CG 实例之后，[ADD CGBINDING](task/UNC/20.15.2/0-00011.md) 装入成员之前。

## 决策点

### 决策点：CGGRPID 取值分配（DP 0-02003）

问：本 CG 组用哪个 ID？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 复用现有规划段 | `CGGRPID` 取 1~32 范围内某值，按本端规划分配 | 正常建组，ID 全局唯一 |
| 超出 32 上限 | 不支持 | 超出系统最大 32 条记录上限，须先 RMV 旧组释放 ID |

> CGGRPID 取值范围 1~32，超出无法配置；分配后即占用，删除（`RMV CGGROUP`）才释放。

### 决策点：是否填写描述（DP 0-02004）

问：DESCRIPTION 描述字段填不填？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 填写描述（推荐） | `DESCRIPTION="xxx"`（1~63 字符） | 多组并存或多人协作场景，便于识别组用途 |
| 不填 | 省略 `DESCRIPTION` | 临时测试组或单一 CG 组环境，无识别需求 |

> 描述为可选；但实际运维强烈建议填写，避免多组并存时混淆。

## 约束

- rule-0-02006 · severity=critical · 仅建组不绑定不会生效——CG 组必须绑定到离线计费模板下（ADD CGGRPBINDING）或通过 SET GLBCGGROUP 设为全局 CG 组才会被实际使用 — 后果：误以为建组即生效，导致话单不按预期发送
- rule-0-02007 · severity=high · CGGRPID 取值范围 1~32 且全局唯一 — 后果：ID 重复或超范围导致新增失败
- rule-0-02008 · severity=warning · 最大记录数 32（与 CGGRPID 取值范围一致） — 后果：达到上限后无法继续新增组
- rule-0-02009 · severity=info · 该命令执行后立即生效 — 后果：建组后即可被引用，但实际起作用仍需下游绑定
- rule-0-02010 · severity=info · DESCRIPTION 取值长度 1~63 字符 — 后果：超长截断或新增失败
