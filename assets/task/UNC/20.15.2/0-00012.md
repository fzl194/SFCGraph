# ADD CGGROUP 原子任务

> 命令静态知识见 [ADD CGGROUP](command/UNC/20.15.2/ADD-CGGROUP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CGGROUP 用来建一个空的 CG 组（仅含组 ID 和描述），本身不含业务策略。它是离线计费 CG 组管理三件套（`ADD CG` → `ADD CGGROUP` → `ADD CGBINDING`）的第二步，组创建后由 `ADD CGBINDING` 装入 CG 实例，再由 `ADD CGGRPBINDING` / `SET GLBCGGROUP` 把组绑到离线计费模板或设为全局 CG 组才能生效。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| CGGRPID | `1~32` 范围内某值 | 本端规划 | CG 组 ID，全局唯一 |
| DESCRIPTION | 1~63 字符描述 | 本端规划 | 可选，便于运维识别 |

**配置链路（本命令位置）**

```
ADD CG              # 建一个 CG 实例（IP/端口/协议）
ADD CGGROUP   ← 本命令    # 建 CG 组（CGGRPID）
ADD CGBINDING       # 把 CG 绑进组，给优先级
ADD CGGRPBINDING    # 把组绑到离线计费模板（或 SET GLBCGGROUP 设为全局）
```

## 约束

- **立即生效** · severity=LOW。
- **最大 32 条记录**（CGGRPID 取值范围即 1~32）· severity=MEDIUM。
- **生效前置依赖**：CG 组要实际起作用，必须绑定到离线计费模板下，或通过 `SET GLBCGGROUP` 设为全局 CG 组；仅建组不绑定则该 CG 组不会被使用 · severity=HIGH · 易误以为建组即生效，导致话单不按预期发送。
