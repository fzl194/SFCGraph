# SET OFCTHRESHOLD 原子任务

> 命令静态知识见 [SET OFCTHRESHOLD](command/UNC/20.15.2/SET-OFCTHRESHOLD.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

SET OFCTHRESHOLD 为已存在的离线计费模板（由 ADD OFCTEMPLATE 创建）配置话单生成阈值。OFCTEMPLATENAME 为前置依赖（来自已配置数据或本端规划），THRESHINHERIT 决定是否继承上级，其余阈值参数按场景按需配置。时长/流量阈值同时设置时，先达者先生成话单，此后两种阈值重置、不累计。

**参数取值样例**

| 参数 | 取值样例 | 获取方法 | 说明 |
|---|---|---|---|
| OFCTEMPLATENAME | `offlinecharge-test` / `offlinecharg-test` | 已配置数据（ADD OFCTEMPLATE） | 离线计费模板名，1~63 字符，不支持空格 |
| THRESHINHERIT | `NON_INHERIT` | 本端规划 | 是否继承上级配置 |
| TIMETHRESHOLD | `5` / `15` / `30` | 本端规划 | 时长阈值（分），0~1440 |
| VOLUMETHRESHOLD | `20` / `1000` | 本端规划 | 流量阈值（KB），0 或 20~9000000000 |
| CONDITIONCHANGE | `3` | 本端规划 | 计费条件改变阈值，1~10 |
| SERVINGNODECHNG | `3` / `5` | 本端规划 | Serving Node 改变次数，0~5 |
| MAXSVCCONTAINER | （规格 1~40） | 本端规划 | 最大业务容器数 |
| MINVOLTHRESHOLD | （规格 0~65535） | 本端规划 | 最小流量阈值（KB） |
| SECRUTHRESHOLD | `3` | 本端规划 | RAN-SecondaryRAT-Usage-Report 触发话单阈值，1~10 |
| TIMETHRESHCCFH | （规格 0~1440,4294967295） | 本端规划 | 在线转离线后时长阈值（分） |
| VOLTHRESHCCFH | （规格 0,20~9000000001） | 本端规划 | 在线转离线后流量阈值（KB） |

**特性场景取值对比**

| 特性场景 | THRESHINHERIT | 时长 | 流量 | CONDITIONCHANGE | SERVINGNODECHNG | SECRUTHRESHOLD | 说明 |
|---|---|---|---|---|---|---|---|
| 离线计费（WSFD-011201） | NON_INHERIT | 30 | 20 | 3 | 5 | — | 默认时长+流量双阈值 |
| 热计费（WSFD-011202） | NON_INHERIT | 30 | 1000 | 3 | 3 | — | 配套 CHARGECTRL |
| NSA 话单（WSFD-011511） | NON_INHERIT | — | — | — | — | 3 | 仅配 SecondaryRAT 阈值 |
| DCC 在线转离线（WSFD-109001） | — | — | — | — | — | — | 仅配 TIMETHRESHCCFH/VOLTHRESHCCFH |

## 决策点

### 时长阈值与流量阈值的取舍（单阈值生效 vs 双阈值）

离线计费默认同时使能时长计费和流量计费。调测单阈值功能时，将不生效侧的阈值置 0 或留空：仅时长阈值（如 5 分钟）则流量阈值不生效；仅流量阈值（如 20 KB）则时长阈值不生效。需根据话单切分粒度需求在本端规划层决定。

### OFCTEMPLATENAME 获取来源（已配置数据 vs 本端规划）

模板名通常由 ADD OFCTEMPLATE 预先创建，可用 LST OFCTEMPLATE 查询后引用；亦可本端规划命名后先 ADD OFCTEMPLATE 再引用。证据中两种获取方法并存（已配置数据中获取 / 本端规划），取决于工作流顺序。

### 在线转离线阈值是否单独配置

DCC 场景下（WSFD-109001），在线计费用户转为离线计费后可执行不同的时长/流量阈值（TIMETHRESHCCFH、VOLTHRESHCCFH）。未配置时默认沿用原有离线计费阈值；如需差异化切单则配置。

## 约束

- **立即生效** · 执行后立即生效。
- **最大 100 条记录** · 每模板一条阈值记录。
- **OFCTEMPLATENAME 前置依赖** · severity=HIGH · 必须先 ADD OFCTEMPLATE 创建模板，再 SET OFCTHRESHOLD 配阈值。
- **2/3G 与 4G 模板阈值差异影响切换** · severity=MEDIUM · GPRS/UMTS 模板与 PGW 模板的时间/流量/最小流量阈值不同时，2/3G↔4G 切换后新建话单受切换后 RAT 对应模板阈值控制。
- **阈值过小导致性能下降** · severity=MEDIUM · 时间阈值 <10 分钟或流量阈值 <10 MB 会致话单过多引起性能下降。
- **流量阈值上限约束** · severity=LOW · 配置流量阈值超过 0xC0000000（3G）时需注意。
- **THRESHINHERIT 一致性** · 各特性均取 NON_INHERIT，继承上级时需全网规划一致。
