# 配置离线计费阈值（SET OFCTHRESHOLD）

> 命令静态知识见 [SET OFCTHRESHOLD](command/UNC/20.15.2/SET-OFCTHRESHOLD.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

SET OFCTHRESHOLD 为已存在的离线计费模板（由 ADD OFCTEMPLATE 创建）配置话单生成阈值。OFCTEMPLATENAME 是前置依赖，THRESHINHERIT 决定是否继承上级，其余阈值参数按场景按需配置。时长/流量阈值同时设置时，先达者先生成话单，此后两种阈值重置、不累计。核心配置空间 3 个维度：**模板名引用**（必选）、**继承开关**（必选）、**阈值类型**（按场景分支）。

### 配置维度 1：模板名引用（参数 OFCTEMPLATENAME，必选）

| 取值 | 作用 | 说明 |
|---|---|---|
| 已有的 OFCTEMPLATENAME | 引用已通过 `ADD OFCTEMPLATE` 配置的离线计费模板 | 1~63 字符，不支持空格；先 `ADD OFCTEMPLATE` 建模板，可 `LST OFCTEMPLATE` 查名后引用 |

> 前置依赖：OFCTEMPLATENAME 必须已通过 `ADD OFCTEMPLATE` 配置存在——见约束 rule-0-03301。

### 配置维度 2：继承开关（参数 THRESHINHERIT，必选）

| 取值 | 作用 | 适用场景 |
|---|---|---|
| **NON_INHERIT** | 不继承上一级，本模板独立配阈值 | 各特性普遍采用（离线/热计费/NSA 话单/DCC） |
| **INHERIT** | 继承上一级离线计费模板的阈值 | UNC 基于 UserProfile/APN/ChargeCharacteristic/全局多级绑定时取下级继承 |

### 配置维度 3：阈值类型（条件可选，按场景分支）

| 阈值参数 | 范围 | 作用 | 典型场景 |
|---|---|---|---|
| `TIMETHRESHOLD`（分） | 0~1440 | 时长阈值，0=不生效 | 离线计费 30、热计费 30 |
| `VOLUMETHRESHOLD`（KB） | 0 或 20~9000000000 | 流量阈值，0=由软参 BYTE86 控制 | 离线计费 20、热计费 1000 |
| `MINVOLTHRESHOLD`（KB） | 0~65535 | 最小流量阈值，0=不生效；时阈值到且满足最小流量才切单 | 需避免零流量切单时配置 |
| `CONDITIONCHANGE` | 1~10 | 计费条件改变阈值（QoS/费率时段/在线转离线/ECGI 等） | 默认 3 |
| `SERVINGNODECHNG` | 0~5 | MME/SGSN/SGW/ePDG 地址改变次数，0=不生效 | 离线 5、热计费 3 |
| `MAXSVCCONTAINER` | 1~40 | 话单最多携带业务容器数 | 默认 20 |
| `SECRUTHRESHOLD` | 1~10 | RAN-SecondaryRAT-Usage-Report 触发话单阈值 | NSA 话单 3 |
| `TIMETHRESHCCFH`（分） | 0~1440, 4294967295 | 在线转离线后时长阈值；0=不生效；4294967295=继承转前 | DCC 差异化切单 |
| `VOLTHRESHCCFH`（KB） | 0, 20~9000000000, 9000000001 | 在线转离线后流量阈值；0=不生效；9000000001=继承转前 | DCC 差异化切单 |

> 时长/流量双阈值同时配时先达者切单，此后两阈值重置不累计。`TIMETHRESHOLD/VOLUMETHRESHOLD` 修改对新用户或在线自动恢复时生效；`MINVOLTHRESHOLD/MAXSVCCONTAINER` 仅对新入网用户生效；`TIMETHRESHCCFH/VOLTHRESHCCFH` 对新用户或在线转离线时生效。

典型脚本：
```
SET OFCTHRESHOLD:OFCTEMPLATENAME="offlinecharge-test",THRESHINHERIT=NON_INHERIT,TIMETHRESHOLD=30,VOLUMETHRESHOLD=20,CONDITIONCHANGE=3,SERVINGNODECHNG=5;
SET OFCTHRESHOLD:OFCTEMPLATENAME="OFCThreshold",THRESHINHERIT=NON_INHERIT,TIMETHRESHOLD=5,VOLUMETHRESHOLD=2000,MINVOLTHRESHOLD=100;
```

**步骤位置**：在所属 feature 编排中通常位于"配置离线计费阈值"——紧随 `ADD OFCTEMPLATE`（建模板）之后，`SET CDRTRIGGER`/`SET CONTAINERTRIGGER`（话单/容器产生开关）之前。

## 决策点：时长阈值与流量阈值的取舍（DP 0-03121）

问：该模板走单阈值（仅时长/仅流量）还是双阈值？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 仅时长阈值 | `TIMETHRESHOLD=非0` + `VOLUMETHRESHOLD=0` | 仅按时长切单 |
| 仅流量阈值 | `VOLUMETHRESHOLD=非0` + `TIMETHRESHOLD=0` | 仅按流量切单 |
| 双阈值（默认） | `TIMETHRESHOLD=非0` + `VOLUMETHRESHOLD=非0` | 先达者切单，最常用 |

### 模板名获取来源（DP 0-03122）

问：OFCTEMPLATENAME 从已配置数据取，还是本端规划命名后新建？

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| 引用已配置数据 | 先 `LST OFCTEMPLATE` 查名后引用 | 工作流中 ADD OFCTEMPLATE 已先执行 |
| 本端规划命名 | 本端规划名 → 先 `ADD OFCTEMPLATE` 再引用 | 从零规划，命名权在本端 |

## 约束

- rule-0-03301 · severity=critical · OFCTEMPLATENAME 前置依赖：必须先 `ADD OFCTEMPLATE` 创建模板，再 `SET OFCTHRESHOLD` 配阈值，否则引用失败
- rule-0-03302 · severity=high · 2/3G 与 4G 模板阈值差异影响切换：GPRS/UMTS 模板与 PGW 模板的时间/流量/最小流量阈值不同时，2/3G↔4G 切换后新建话单受切换后 RAT 对应模板阈值控制
- rule-0-03303 · severity=warning · 阈值过小导致性能下降：时间阈值 <10 分钟或流量阈值 <10 MB 会致话单过多引起性能下降
- rule-0-03304 · severity=warning · SET OFCTHRESHOLD 最大记录数为 100，每模板一条阈值记录，超规格新增失败
- rule-0-03305 · severity=info · 流量阈值上限约束：配置流量阈值超过 0xC0000000（3G）时须优先配置软参 BYTE86
