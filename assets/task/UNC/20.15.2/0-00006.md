# ADD APNDEACTQFPLCY 原子任务
> 命令静态知识见 [ADD APNDEACTQFPLCY](command/UNC/20.15.2/ADD-APNDEACTQFPLCY.md)。本页只讲配置生成实例化时这条命令**怎么配**。

本命令是 APN 粒度策略，优先级高于全局粒度的 `ADD DEACTQFPLCY`。典型用于语音等 GBR 业务在 UE 短暂无线丢失时延时保活，避免立即释放语音专载。

## 配置方法
按"APN + NgAp 原因码 + 处理策略 + 延迟时长"四元组配置。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| APN | APN 名称（语音场景通常 `ims`/`imsapn.com`） | `ims` |
| NGAPCAUSEGROUP | NgAp 原因码组 | `RADIO_NETWORK` |
| NGAPCAUSEVALUE | NgAp 原因码值（UE 无线丢失固定为 21） | `21` |
| QFPLCY | 专有 GBR QoS Flow 处理策略 | `DELAY_RELEASE`（延迟释放）/ `RELEASE`（立即释放） |
| DELAYTIMER | 延迟释放时长（秒，10~100） | `60` |

典型脚本（语音承载延时保活，UE 无线丢失 21 号原因）：
```
ADD APNDEACTQFPLCY: APN="ims", NGAPCAUSEGROUP=RADIO_NETWORK, NGAPCAUSEVALUE=21, QFPLCY=DELAY_RELEASE, DELAYTIMER=60;
```

## 决策点（若有）

### QoS Flow 去活处理策略
- 问什么：收到指定 NgAp 原因的去活请求时，专有 GBR QoS Flow 立即释放还是延迟释放。
- 立即释放 → QFPLCY=RELEASE，资源立即回收。
- 延迟释放 → QFPLCY=DELAY_RELEASE + DELAYTIMER（10~100 秒），等 UE 重新接入；语音等实时业务推荐，避免短暂丢失挂断通话。

### 粒度选择（APN 粒度 vs 全局粒度）
- 问什么：本策略是否只针对特定 APN（如 ims）生效。
- 仅特定 APN（如语音 APN）需要差异化保活 → 用本命令（APN 粒度，优先级高）。
- 全网统一兜底 → 用 `ADD DEACTQFPLCY`（全局粒度）。
- APN 粒度优先级 > 全局粒度。

## 约束（若有）
- 取值范围：DELAYTIMER 取值 10~100 秒。`severity=medium`
- 粒度优先级：APN 粒度（本命令）> 全局粒度（`ADD DEACTQFPLCY`）。`severity=medium`
- 规格上限：最多可输入 20000 条记录。`severity=low`
- 生效时机：执行后立即生效。`severity=low`
