# 配置抄送CG组（ADD CPCGGRP）

> 命令静态知识见 [ADD CPCGGRP](command/UNC/20.15.2/ADD-CPCGGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**——按配置维度组织成"字典"，**不**逐特性罗列取值。

## 配置方法

ADD CPCGGRP 创建一个抄送 CG 组（CPCGGRPID 固定为整数 1，全系统仅允许 1 个组），并可选填写组描述。建组后由 [ADD CPCGBINDING](task/UNC/20.15.2/0-00023.md) 在该组下绑定具体 CG 的 IP/端口/优先级。它服务于离线计费场景（WSFD-011201 离线计费参考信息）。

> **重要**：命令真相 notes 明确标注"**当前版本不支持此命令**"且"最大记录数为 1"，③ 各特性配置范式无数据规划表样例（仅出现于特性参考信息的命令清单中）。下表取值由命令参数真相表推导。

### 配置维度 1：组 ID（参数 CPCGGRPID）

| 取值 | 作用 | 说明 |
|---|---|---|
| `1` | 抄送 CG 组 ID（唯一取值） | 整数 1，全系统仅一个组 |

### 配置维度 2：组描述（参数 DESCRIPTION，可选）

| 取值 | 作用 | 说明 |
|---|---|---|
| 不填 | 最小化配置 | 仅建组、无运维说明 |
| `<字符串>` | 组可读描述 | 1~63 字符，便于运维识别 |

### 典型脚本

```
ADD CPCGGRP: CPCGGRPID=1, DESCRIPTION="CpCGGroup1";
```

**步骤位置**：在"WSFD-011201 支持离线计费"流程的命令清单中，位于 ADD LOGICINF 之后、ADD CPCGBINDING 之前。典型顺序：ADD CPCGGRP（建组）→ ADD CPCGBINDING（组下绑 CG）。

## 决策点

### 决策点：是否填写组描述（DP 0-02027）

问：建抄送 CG 组时是否需要填写 DESCRIPTION？

| 选项 | 适用 |
|---|---|
| 仅配 CPCGGRPID=1 | 无需描述，最小化配置 |
| 配 CPCGGRPID=1 + DESCRIPTION | 需要可读的组说明（运维识别） |

> 两个参数数据来源均为 local_planned，组 ID 取值唯一、描述可选 → 绑定类型 local_planned，无取值分叉决策。

### 决策点：组容器是否就绪以挂接 CG 绑定（DP 0-02028）

问：ADD CPCGGRP 建组时机与 ADD CPCGBINDING 挂接的关系？

| 选项 | 前置条件 |
|---|---|
| 先建组再挂 CG 绑定 | 本命令已执行成功（CPCGGRPID=1 存在） |
| 组未建直接配 CG 绑定 | 不允许——ADD CPCGBINDING 的 CPCGGRPID 必须先由本命令创建 |

## 约束

- rule-0-02066 · severity=high · 全系统仅一个抄送 CG 组：CPCGGRPID 仅能取整数 1，最大记录数为 1，重复建组失败 — 规划前确认组未建
- rule-0-02067 · severity=high · 当前版本不支持此命令（命令 notes 明确标注）：实际部署前需确认网元版本是否启用该功能 — 未启用版本配置无效
- rule-0-02068 · severity=info · 立即生效：命令执行后立即生效，无需重启 — 配置即时生效
- rule-0-02069 · severity=warning · DESCRIPTION 长度限制：输入长度 1~63 字符，超长被拒 — 描述须控制在 63 字符内
- rule-0-02070 · severity=high · 前置依赖（反向）：ADD CPCGBINDING 必须先有 CPCGGRPID=1 才能挂 CG 绑定 — 组未建则绑定失败
