# ADD CPCGGRP 原子任务

> 命令静态知识见 [ADD CPCGGRP](command/UNC/20.15.2/ADD-CPCGGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法

ADD CPCGGRP 创建一个抄送 CG 组（CPCGGRPID 固定为整数 1，全系统仅允许 1 个组），并可填写组描述。建组后由 [ADD CPCGBINDING](task/UNC/20.15.2/0-00023.md) 在该组下绑定具体 CG 的 IP/端口/优先级。它服务于离线计费场景（WSFD-011201 离线计费参考信息）。

> **重要**：命令真相 notes 明确标注"**当前版本不支持此命令**"且"最大记录数为 1"，③ 各特性配置范式无数据规划表样例（仅出现于特性参考信息的命令清单中）。下表取值由命令参数真相表推导。

**参数取值样例（建抄送 CG 组范式，依参数真相推导）**

| 参数 | 取值样例 | 默认 | 说明 |
|---|---|---|---|
| CPCGGRPID | `1` | — | 抄送 CG 组 ID（必填），整数 1（唯一取值） |
| DESCRIPTION | `<字符串>` | 无 | 抄送 CG 组描述（可选），1~63 字符 |

**配置位置**：在"WSFD-011201 支持离线计费"流程的命令清单中，位于 ADD LOGICINF 之后、ADD CPCGBINDING 之前。典型顺序：ADD CPCGGRP（建组）→ ADD CPCGBINDING（组下绑 CG）。

## 决策点

### 是否填写组描述

| 选项 | 适用 |
|---|---|
| 仅配 CPCGGRPID | 无需描述，最小化配置 |
| 配 CPCGGRPID + DESCRIPTION | 需要可读的组说明（运维识别） |

> 两个参数数据来源均为 local_planned，组 ID 取值唯一、描述可选 → 绑定类型 local_planned，无取值分叉决策。

## 约束

- **立即生效** · severity=LOW。
- **唯一组**：最大记录数为 1，CPCGGRPID 仅能取整数 1 · severity=HIGH · 全系统仅一个抄送 CG 组，重复建组失败。
- **版本不支持**：当前版本不支持此命令（见命令 notes） · severity=HIGH · 实际部署前需确认网元版本是否启用该功能。
- **长度限制**：DESCRIPTION 输入长度 1~63 字符 · severity=LOW。
