# IPv6 前缀代理（WSFD-104004）

> 特性静态知识见 [WSFD-104004](feature/UNC/20.15.2/WSFD-104004.md)。本页讲配置生成实例化时这个特性**怎么配**——按 Local/Radius 分配方式 + 接入制式 + VPN 绑定编排，共享地址分配族骨架 + APN/IPv6 地址族特殊配置。

## 配置概览

核心对象链：**VPN/IPv6 APN 实例** → **本地/Radius 地址池**（IPv6 delegated-prefix 段）→ **APN 地址分配属性**（IPV6ALLOCTYPE=LOCAL 或 Radius）+ **MOD APN** 启用 IPv6 VPN 绑定（HASVPNIPV6=ENABLE）→ 控制面将 IPv6 前缀经 N4/Sx/Gn 接口下发到 UPF/SGW-U/PGW-U → UPF 通过 DHCPv6 方式分配给 MS/UE → MS/UE 作为无线路由器为后端多台终端设备分配 IPv6 网段。

**核心配置空间**（与 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) APN 接入域基础设施 + [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 地址池体系有结构差异）：

| 步骤差异点 | 标准接入基础设施 ([UNC 1-00024](task/UNC/20.15.2/1-00024.md)) | IPv6 前缀代理（本特性） |
|---|---|---|
| L3VPN 地址族 | AFTYPE=ipv4uni / ipv6uni | 仅 AFTYPE=ipv6uni（IPv6 单地址族） |
| VRF RD | VRFRD=100:1（标准）| VRFRD=65512:2（**前缀代理特殊**）|
| APN 维度 | ADD APN + SET APNADDRESSATTR（SUPPORTIPV4/IPV6）| ADD APN + SET APNADDRESSATTR（IPV6ALLOCTYPE=LOCAL/Radius）+ **MOD APN**（HASVPNIPV6=ENABLE,VPNINSTANCEIPV6=）|
| 地址池 | ADDRPOOL POOLTYPE=Local/UDM/Radius/DHCP/LNS | ADDRPOOL POOLTYPE=Local/Radius + IPVERSION=IPv6 + HASVPN=Enable + VPNINSTANCE=vpn1 |

> **本特性专用 MOD APN 参数**：`MOD APN` 的 `HASVPNIPV6=ENABLE,VPNINSTANCEIPV6=` 是 IPv6 前缀代理特有——一般 APN 实例在 ADD 时通过 `HASVPN=ENABLE,VPNINSTANCE=` 绑 IPv4 VPN，IPv6 VPN 绑定需**后置 MOD**（因 ADD APN 流程中 IPV6 VPN 通常未先建）。**这一参数增量已在 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) 场景差异中记录**（WSFD-104004 行：ipv6uni、SUPPORTIPV6=ENABLE；APN 上启用 IPv6RA 前缀代理）。

> **VRFRD=65512:2 特例**：原始激活 md（76459527）步骤 3c 明确 `MOD VPNINSTAF: VRFNAME="vpn1", AFTYPE=ipv6uni, VRFRD="65512:2"` ——VRFRD **65512:2 是 IPv6 前缀代理场景默认值**，与其他 ipv6uni 场景（如 [WSFD-104001](task/UNC/20.15.2/2-00018.md) IPv6 单栈承载）不同。本特性**VRFRD 不走标准骨架的 100:1 默认值**。

## 配置流程（步骤编排，每步显示含命令）

1. **License 前置**：`SET LICENSESWITCH` → [0-00180](task/UNC/20.15.2/0-00180.md)（LICITEM=LKV3W9V6PD11,SWITCH=ENABLE）
2. **配置 IPv6 VPN**（IPv6 单地址族 + RD=65512:2）：
   - 2a. 创建 VPN 实例：`ADD VPNINST` → [0-00098](task/UNC/20.15.2/0-00098.md)（VPNINSTANCE=vpn1）
   - 2b. 创建 L3VPN 实例 + 地址族：`ADD L3VPNINST` → [0-00040](task/UNC/20.15.2/0-00040.md)（VRFNAME=vpn1）+ `ADD VPNINSTAF` → [0-00099](task/UNC/20.15.2/0-00099.md)（VRFNAME=vpn1, AFTYPE=ipv6uni）
   - 2c. **修改 VPN RD**：`MOD VPNINSTAF` → [0-00133](task/UNC/20.15.2/0-00133.md)（VRFNAME=vpn1, AFTYPE=ipv6uni, VRFRD="65512:2"）
3. **创建 APN 实例 + 使能地址分配属性**：
   - 3a. `ADD APN` → [0-00004](task/UNC/20.15.2/0-00004.md)（APN=huawei.com）
   - 3b. `SET APNADDRESSATTR` → [0-00270](task/UNC/20.15.2/0-00270.md)（APN=huawei.com, IPV6ALLOCTYPE=LOCAL 或 Radius）
4. **创建 IPv6 前缀地址池**（Local 或 Radius 方式）：`ADD ADDRPOOL` → [0-00217](task/UNC/20.15.2/0-00217.md)（POOLNAME=pool01, POOLTYPE=Local/Radius, IPVERSION=IPv6, HASVPN=Enable, VPNINSTANCE=vpn1, CHECKIPVALID=Enable）
5. **APN 绑定 IPv6 VPN**（**本特性专有**）：`MOD APN` → [0-00265](task/UNC/20.15.2/0-00265.md)（APN=huawei.com, HASVPNIPV6=ENABLE, VPNINSTANCEIPV6=vpn1）
6. **接入域路由发布**（OSPFv3 单栈发布 IPv6 用户路由）：`ADD OSPFV3` + `ADD OSPFV3AREA` + `ADD OSPFV3INTERFACE` + `ADD OSPFV3IMPORTROUTE`（PROTOCOL=wlr） → [1-00032](task/UNC/20.15.2/1-00032.md) UNC OSPFv3 路由发布
   - 关键参数：PROCID, AREAID, IFNAME, PROTOCOL=wlr
   - **命令名修正（R1.5）**：v1.0 小驼峰 `OSPFv3` 是错的——UNC 真实命令族是 **`OSPFV3` 全大写** + `OSPFV3INTERFACE`（无 NETWORK）。归 [1-00032](task/UNC/20.15.2/1-00032.md)。

> **步骤 2-5 编排共享骨架 + 2 个本特性专有步骤**：步骤 2-3 是 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) 的子集（仅 ipv6uni 路径）+ VRFRD 特例；步骤 4 走 [UNC 1-00023](task/UNC/20.15.2/1-00023.md)（HASVPN=Enable + VPNINSTANCE=vpn1 已记录为 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 场景差异 WSFD-104004 行）；步骤 5 `MOD APN HASVPNIPV6=ENABLE` 是**本特性专有**（[UNC 1-00024](task/UNC/20.15.2/1-00024.md) 场景差异表已记录但**未抽独立 compound**——属单命令参数增量）；步骤 6 走共享 OSPF 骨架。

> **本特性不新建骨架 compound**：所有命令均可在共享骨架 [UNC 1-00024](task/UNC/20.15.2/1-00024.md) + [UNC 1-00023](task/UNC/20.15.2/1-00023.md) + [UNC 1-00029](task/UNC/20.15.2/1-00029.md) 中覆盖；MOD APN 的 HASVPNIPV6/VRFRD 特殊值作为参数增量写入各骨架场景差异。

## 决策点（特性级：分配方式 × 接入制式 × Radius 依赖）

| 选项（分配方式 × 制式）| 走法 | 关键差异 |
|---|---|---|
| Local × 5G SMF | 步骤 1（License）+ 步骤 2-6（Local 地址池）| IPV6ALLOCTYPE=LOCAL；POOLTYPE=Local |
| Local × 4G SGW-C/PGW-C | 步骤 1 + 步骤 2-6 | 同上（4G 共享 License 仅 SGW-C+PGW-C，无 SMF 专属 License）|
| Local × 2-3G GGSN-C | 步骤 1 + 步骤 2-6 | 2-3G License 待核 |
| Radius × 5G/4G/2-3G | 步骤 1 + 步骤 2-3 + 步骤 5-6 + **前置 WSFD-011306 Radius 功能** | IPV6ALLOCTYPE=Radius；POOLTYPE=Radius；**前置 Radius 服务器组 + 鉴权配置** |
| Local + Radius 共存 × 任意 | 步骤 1 + 步骤 2-6 配两个 POOL（Local + Radius）| 多地址池共存；APN 上 SET APNADDRESSATTR 选主导分配方式 |

> **走法列**：所有 5G/4G/2-3G 制式**前 5 步走共享骨架+本特性专有参数增量**，Radius 方式前置 [WSFD-011306](feature/UNC/20.15.2/WSFD-011306.md) Radius 功能（**特性概述明确**："MS/UE 必须到 AAA Server 进行 RADIUS 鉴权"）。

## 约束

- **本特性依赖 IPv6 承载上下文**（critical）：特性概述明确"必须首先开启 IPv6 承载上下文特性才能使用本特性" — 后果：未先开 IPv6 承载上下文 → 本特性 IPv6 前缀无法下发。
- **本特性依赖 Radius 功能**（critical，Radius 方式）：特性概述明确"使用 IPv6 Prefix Delegation 功能的 MS/UE 在激活时需要为 MS/UE 后的多台设备获取一个或多个网段地址，该网段地址只能从 AAA Server 动态获取，因此 MS/UE 必须到 AAA Server 进行 RADIUS 鉴权" — 后果：未配 Radius 功能 → Radius 方式 IPv6 前缀无法下发。
- **本特性依赖 IPv4v6 双栈**（critical，IPv4v6 双栈用户场景）：特性概述明确"如果需要为双栈接入用户提供 IPv6 Prefix Delegation 功能，必须首先开启 IPv4v6 双栈接入特性" — 后果：双栈用户场景未开 → IPv6 前缀代理对双栈用户不可用。
- **本地池 IPv6 delegated-prefix 长度一致**（critical）：本地地址池方式为用户分配 IPv6 delegated-prefix 时，**同一 APN 下的 IPv6 delegated-prefix 长度必须相同** — 后果：长度不一致致地址分配异常。
- **Radius 池 delegated-prefix 长度可不同**（warning）：Radius 方式为用户分配 IPv6 delegated-prefix 时，**同一 APN 下的 IPv6 delegated-prefix 长度可以不同**（与本地池相反）— 后果：Radius 场景需明确单 APN 下不同用户 delegated-prefix 长度策略。
- **不支持 DHCP Server 下发 IPv6 delegated prefixes**（critical）：特性概述明确"IPv6 Prefix Delegation 功能不支持从 DHCP Server 获取并分配 IPv6 delegated-prefix" — 后果：DHCP 方式不可用作本特性前缀源。
- **AAA 服务器不支持固定 IPv6 delegated prefixes**（critical）：不支持 AAA 服务器下发固定的 IPv6 delegated prefixes；AAA 服务器动态下发 IPv6 delegated prefixes 时**必须在 UNC 上配置 AAA 服务器上规划的 IPv6 delegated prefixes Radius 地址池** — 后果：未配 Radius 地址池 → 动态前缀下发失败。
- **VRFRD=65512:2 特例**（warning）：IPv6 前缀代理 VPN 必须 VRFRD=65512:2，区别于标准骨架默认 100:1 — 后果：VRFRD 错配致路由发布失败。
- **HASVPN=Enable 必选**（critical）：本特性 IPv6 地址池必须 HASVPN=Enable + VPNINSTANCE=vpn1（已在 [UNC 1-00023](task/UNC/20.15.2/1-00023.md) 场景差异记录）；CHECKIPVALID=Enable 必配 — 后果：VPN 未绑定致前缀路由泄漏。
- **特性规格上限**（warning）：整机支持无线路由器同时在线最大数 **20 000**；单用户最大支持手机后路由网段数 **30** — 后果：超规格致后续路由器接入失败。
- **L2TP 与 IPv6 PD 互斥（同 APN）**（warning）：特性概述明确"如果 APN 使能了 L2TP 功能，则无需开启 IPv6 Prefix Delegation 功能" — 后果：同 APN 同时配 L2TP + IPv6 PD 致资源浪费，且 L2TP 用户 IPv6 前缀由 LNS 设备分配。
- **UPF/PGW-U/SGW-U 透传 RS 数据报文**（warning）：IPv4/v6 双栈改造后 UE 通过 UPF 透传 Router Solicitation 数据报文获取 IPv6 地址；UE 弱覆盖时 eNodeB 无法下发该报文，**掉话率会增加**（特性概述"对系统的影响"明确）— 后果：弱覆盖场景下 IPv6 用户掉话率上升，需无线优化。
- **首次发布版本**（warning）：本特性首次发布版本为 **UNC 20.5.0**，低于此版本不支持 — 后果：网元版本 < 20.5.0 配置失败。
- **适用 NF 范围**（warning）：适用 NF 为 **GGSN-C、SGW-C、PGW-C、SMF**（不含 SGSN/MME）— 后果：SGSN/MME 配置本特性命令被拒绝。
- **依赖 WSFD-104002**（critical，见上）
- **无 AAA 服务器配动态下发必须配 Radius 地址池**（critical，见上）
