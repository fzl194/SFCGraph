## 配置方法

按"模板名 + 时长配额机制 + 话单版本 + 话单时间格式"组合配置一个离线计费模板对象。模板名（OFCTEMPLATENAME）是引用键，创建后由 `SET OFCTHRESHOLD`、`SET CDRTRIGGER` 补充阈值与话单触发开关，最终被计费特性引用。

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 模板标识 | OFCTEMPLATENAME | 本端规划（1~63，不支持空格） | — | 模板引用键 | 所有离线计费场景 |
| 时长配额机制（QCT） | TQM=QCT, QCTVALUE | TQM 本端规划（QCT/CTP/OPERATOR_CTP，当前版本不支持 CTP）；QCTVALUE 本端规划（0~200 秒） | TQM=QCT, QCTVALUE=0 | 按 QCT 周期统计时长，精度最高 | QCT 时长计费主流场景 |
| G-CDR 版本 | GCDRVERSION | 对端协商（与 CG 协商），枚举 R98_CMCCV130~R10_PGW_CDR | R7V740_EGCDR | GGSN 形态产生的 G-CDR 版本 | UNC 做 GGSN 形态 |
| PGW-CDR 版本 | PGWCDRVERSION | 对端协商，枚举 R8V850_PGW_CDR~R10_PGW_CDR | R8V850_PGW_CDR | UNC 从 GGSN 切换到 PGW 时话单平滑切换 | PGW 形态/角色切换 |
| SGW-CDR 版本 | SGWCDRVERSION | 本端规划，枚举 R8V850_SGW_CDR~R10_SGW_CDR | R8V850_SGW_CDR | UNC 做 SGW 或合一形态时 SGW 话单版本 | SGW/合一形态 |
| 话单时间格式 | CDRTIMEFORMAT | 本端规划（LOCAL_TIME/UTC） | LOCAL_TIME | 话单时间格式；UTC 是 LastActivity 计费前提 | LastActivity 计费需 UTC |
| 辅助字段 | RECORDSEQNUMBER, QHTVALUE | 本端规划；RECORDSEQNUMBER 0~1；QHTVALUE 0~3600 秒 | RECORDSEQNUMBER=0, QHTVALUE=0 | 序列号起始值、QHT 配额保持时间 | 按需规划 |

典型脚本（基本离线计费）：
```
ADD OFCTEMPLATE:OFCTEMPLATENAME="ofc_test",TQM=QCT,QCTVALUE=197;
```

典型脚本（多话单版本 + QCT）：
```
ADD OFCTEMPLATE:OFCTEMPLATENAME="ofc_test",GCDRVERSION=R98V760,PGWCDRVERSION=R9V950_PGW_CDR,SGWCDRVERSION=R10_SGW_CDR,TQM=QCT,QCTVALUE=197;
```

步骤位置：本任务（建模板）→ `SET OFCTHRESHOLD`（补充阈值）→ `SET CDRTRIGGER`（话单触发开关）→ 计费特性引用

## 决策点

### 决策点：时长配额机制选择（DP 0-02077）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| QCT（QUOTA_CONSUME_TIME） | TQM=QCT, QCTVALUE=0~200（必填） | 主流场景，按 QCT 周期统计，精度最高 |
| CTP / OPERATOR_CTP | TQM=CTP/OPERATOR_CTP, BTIVALUE | **当前版本不支持**（见约束） |

### 决策点：话单版本与时间格式规划（DP 0-02078）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| GUL 切换网络 | GCDRVERSION/PGWCDRVERSION/SGWCDRVERSION 配为 R8 及以后 | UNC 角色从 GGSN 切换到 PGW 时话单平滑切换 |
| 继承默认话单版本 | 不显式配置 CDR 版本参数 | 默认 R7V740_EGCDR / R8V850_PGW_CDR / R8V850_SGW_CDR |
| UTC 时间格式 | CDRTIMEFORMAT=UTC | LastActivity 计费（依赖 UTC） |
| 本地时间格式 | CDRTIMEFORMAT=LOCAL_TIME | 默认，无 LastActivity 计费需求 |

## 约束

· rule-0-02191 · severity=high · 条件依赖：TQM=QCT 时 QCTVALUE 必填（0~200 秒）
· rule-0-02192 · severity=high · 引用前置：模板创建后才能被 `SET OFCTHRESHOLD`、`SET CDRTRIGGER` 及计费特性引用，模板名须全局一致
· rule-0-02193 · severity=warning · 版本限制：当前版本**不支持** BTIVALUE 参数（虽为条件必选，实际不可用；即 TQM 不支持 CTP/OPERATOR_CTP）
· rule-0-02194 · severity=warning · 规格上限：最大记录数 100
· rule-0-02195 · severity=info · 生效时机：命令执行后立即生效
