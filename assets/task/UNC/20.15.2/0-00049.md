# ADD OFCTEMPLATE 原子任务
> 命令静态知识见 [ADD OFCTEMPLATE](command/UNC/20.15.2/ADD-OFCTEMPLATE.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令创建一个离线计费模板对象（OFCTemplate），定义话单版本、计费时长机制、定时器等。模板由后续命令（如 `SET OFCTHRESHOLD`、`SET CDRTRIGGER`）补充阈值与话单触发开关后，被计费特性引用。模板名是引用键。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| OFCTEMPLATENAME | 模板名，本端规划，1~63 字符不含空格，作为引用键 | `offlinecharge-test` |
| TQM | 时长配额机制，本端规划，`QCT`（QCT 时长计费）/连续时长计费 | `QCT` |
| QCTVALUE | TQM=QCT 时必填，QCT 时长（秒），0~200 | `0` / `30` |
| QHTVALUE | QHT 时长（秒），0~3600，定时器信息 | `30` |
| GCDRVERSION | G-CDR 版本，对端协商（与 CG 协商） | `R7V740_GCDR` / `R7V740_EGCDR` |
| PGWCDRVERSION | PGW-CDR 版本，对端协商 | `R8V850_PGW_CDR` |
| SGWCDRVERSION | SGW-CDR 版本，本端规划 | `R8V850_SGW_CDR` |
| RECORDSEQNUMBER | Record Sequence Number 字段起始值，0/1 | `0` |
| CDRTIMEFORMAT | 话单时间格式，本端规划（`LOCAL_TIME`/`UTC`） | `UTC`（LastActivity 计费需 UTC） |

典型脚本（离线计费场景）：
```
ADD OFCTEMPLATE: OFCTEMPLATENAME="offlinecharge-test", TQM=QCT, QCTVALUE=0;
```
热计费场景（G-CDR 版本与 QCT 时长）：
```
ADD OFCTEMPLATE: OFCTEMPLATENAME="offlinecharg-test", GCDRVERSION=R7V740_EGCDR, TQM=QCT, QCTVALUE=30;
```

## 决策点（若有）

### 时长计费方式（TQM）
- 问什么：离线计费时长如何统计。
| TQM 取值 | 含义 | 精度 |
|---|---|---|
| `QCT`（QCT 时长计费） | 按 QCTVALUE 周期统计，QCTVALUE 必填 | 精度最高 |
| 连续时长计费 | 按实际业务时长连续统计 | 精度次之 |

### 话单版本（CDR Version）选择
- 问什么：与 CG 对接的话单协议版本，对端协商。
- 各 CDR 版本（G/PGW/SGW）独立配置；**若部署 GUL 切换网络，话单版本应配置为 R8 及以后版本**。
- 跨特性一致性：同一设备上离线计费、热计费场景引用同一套话单版本规划。

### 话单时间格式
- 问什么：是否启用 LastActivity 计费等依赖 UTC 的特性。
- LastActivity 计费只在 `CDRTIMEFORMAT=UTC` 下生效，需在创建模板时确定。

## 约束（若有）
- 规格上限：最大记录数 100。`severity=medium`
- 版本限制：当前版本**不支持** BTIVALUE 参数（虽为 conditional，实际不可用）。`severity=medium`
- 条件依赖：TQM=QCT 时 QCTVALUE 必填。`severity=high`
- 引用前置：模板创建后才能被 `SET OFCTHRESHOLD`、`SET CDRTRIGGER` 及计费特性引用，模板名须全局一致。`severity=high`
- 生效时机：执行后立即生效。`severity=low`
