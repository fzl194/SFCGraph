# MOD NFPROFILE 原子任务
> 命令静态知识见 [MOD NFPROFILE](command/UNC/20.15.2/MOD-NFPROFILE.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：AMF、SMF、NSSF、NRF、NCG、SMSF。本命令修改 NF 实例概述信息，修改后会触发该 NF 向 NRF 发起更新流程，是服务发现/选择相关调测（按优先级/容量/位置选择 NF）的核心入口。

## 配置方法
按 NF 实例名称定位，按场景修改容量/优先级/位置/负载等字段。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| NFINSTANCENAME | NF 实例名称（必选，0~50 字符） | `smf1`、`AMF_Instance_0` |
| NFSTATUS | NF 状态（可选） | `Invalid` 等（**不支持 Suspend**） |
| CAPACITY | 容量（0~65535，越大容量越大） | AMF 迁移时置 `0` |
| PRIORITY | 优先级（0~65535，越小优先级越高） | SMF 多服务区置 `1` |
| LOCALITY | 位置信息（0~150 字符） | `beijing` |
| LOAD | 负载（0~100） | 按规划 |
| FQDN | 域名（0~255，不区分大小写） | 按规划 |
| INTERPLMNFQDN | PLMN 间域名（0~255） | 按规划 |
| HTTPLEIDX | http 地址实例标识（0~255） | 按规划 |
| ALLOWEDNFTYPES | 支持的 NF类型 | 按规划 |

典型脚本（WSFD-205015 SMF 多服务区，确保 smf1/smf2 优先级低于 smfInfoList 中其他 smfInfo）：
```
MOD NFPROFILE: NFINSTANCENAME="smf1", PRIORITY=1;
MOD NFPROFILE: NFINSTANCENAME="smf2", PRIORITY=1;
```

## 决策点

### NF 选择调优维度（容量 / 优先级 / 位置 / 负载）
NF 概述信息同时承载四个影响 NF 选择结果的字段：CAPACITY（容量权重）、PRIORITY（优先级，越小越高）、LOCALITY（按位置选 PCF/NF）、LOAD（负载）。不同场景调优重点不同：AMF Pool 用户迁移调 CAPACITY（置 0 把目标 AMF 排到选择末位）；SMF 多服务区调 PRIORITY（确保服务注册消息中 smfInfo priority 满足约束）；PCC 基本功能（NFLOC 选 PCF）调 LOCALITY。获取方法"全网规划/本端规划/-"多值分布，取值随场景分叉。

### NF 状态取值（排除 Suspend）
NFSTATUS 可选值中**不支持 Suspend 状态**。需将 NF 标记为不可选时用 `Invalid` 等允许的取值，不能用 Suspend。

## 约束
- **立即生效并触发 NRF 更新**：修改后立即生效，并向 NRF 发起更新流程，影响 NF 服务发现结果——多实例并发修改需评估对在用业务的影响。
- **NFSTATUS 不支持 Suspend**：状态取值不能选择 Suspend。
- **NFINSTANCENAME 存在性前置**：NF 实例需已向 NRF 注册（或已 ADD NFPROFILE），MOD 才能成功；可用 LST NFPROFILE 校验。
- **CAPACITY/PRIORITY 语义**：CAPACITY 越大容量越大；PRIORITY 越小优先级越高（注意两者方向相反）。
- **PRIORITY 一致性（SMF 多服务区）**：服务注册消息中 smfInfo 的 priority 值需小于 smfInfoList 中其他 smfInfo 的 priority 值，修改前用 LST NFPROFILE 核对当前值。
- **FQDN/INTERPLMNFQDN 大小写**：不区分大小写。
