# MOD NFPROFILE 原子任务

> 命令静态知识见 [MOD NFPROFILE](command/UNC/20.15.2/MOD-NFPROFILE.md)。本页只讲调测/运维实例化时这条命令**怎么用**。

适用 NF：AMF、SMF、NSSF、NRF、NCG、SMSF。本命令修改 NF 实例概述信息，修改后会触发该 NF 向 NRF 发起更新流程，是服务发现/选择相关调测（按优先级/容量/位置选择 NF）的核心入口。

## 配置方法

### NF 实例定位

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 实例名称 | NFINSTANCENAME | 字符串 0~50，字母/数字/-/_ | 无（必选） | 唯一标识 NF 实例 | 所有场景，MOD 前置条件 |

### 容量与优先级

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 容量权重 | CAPACITY | 0~65535（越大容量越大） | 无 | 容量越大在 NF 选择时被选概率越高 | AMF Pool 用户迁移（置 0 把目标 AMF 排到选择末位） |
| 优先级 | PRIORITY | 0~65535（越小优先级越高） | 无 | 优先级越高在 NF 选择时被选概率越高 | SMF 多服务区（确保目标 SMF priority 值最小） |

> 注意：CAPACITY 和 PRIORITY 语义方向相反——CAPACITY 越大权重越高，PRIORITY 越小优先级越高。

### 位置信息

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 位置描述 | LOCALITY | 字符串 0~150，字母/数字/-/_ | 无 | 用于 NF 按地域选择（如 NFLOC 选 PCF） | PCC 基本功能（PCFSELECTMODE=NFLOC 时需配置 LOCALITY） |

### 负载与状态

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 负载 | LOAD | 0~100 | 无 | NRF 可能将此作为 NF 选择依据 | 按规划配置 |
| NF 状态 | NFSTATUS | Invalid / Registered / Undiscoverable（**不支持 Suspend**） | 无 | 标记 NF 是否可被服务发现 | 运维场景（迁移时置 Undiscoverable，迁移后改回 Registered） |

### 域名配置

| 维度 | 参数 | 取值 | 默认 | 作用 | 适用场景 |
|---|---|---|---|---|---|
| 域名 | FQDN | 字符串 0~255，不区分大小写 | 无 | 本网内域名查询 | 对 AMF/SMF 无效（以 ADD AMFINFO（★R1.5 atom 待补）/ADD SMFINFO（★R1.5 atom 待补） 为准） |
| PLMN 间域名 | INTERPLMNFQDN | 字符串 0~255，不区分大小写 | 无 | 跨 PLMN 的 NF 发现 | 跨国场景 |
| HTTP 地址标识 | HTTPLEIDX | 0~255 | 无 | http 地址实例标识 | 已废弃，请使用 ADD SBIAPLE |
| 支持的 NF 类型 | ALLOWEDNFTYPES | AllowNf... 多值 | 允许所有 | 控制哪些 NF 类型可访问本实例 | 按需配置 |

### 典型脚本

**WSFD-205015 SMF 多服务区调测**（确保 smf1/smf2 优先级低于 smfInfoList 中其他 smfInfo）：

```
MOD NFPROFILE: NFINSTANCENAME="smf1", PRIORITY=1;
MOD NFPROFILE: NFINSTANCENAME="smf2", PRIORITY=1;
```

**AMF Pool 用户迁移**（将目标 AMF 容量置 0 使其排到选择末位）：

```
MOD NFPROFILE: NFINSTANCENAME="amf_target", CAPACITY=0;
```

### 步骤位置

1. 按 NFINSTANCENAME 定位目标 NF 实例
2. 根据业务场景确定需调整的维度（CAPACITY/PRIORITY/LOCALITY/NFSTATUS 等）
3. 执行 MOD 命令
4. 命令立即生效并触发向 NRF 的更新流程

## 决策点

### 决策点：NF 选择调优维度（DP 0-01278）

NF 概述信息同时承载四个影响 NF 选择结果的字段，不同场景调优重点不同：

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| CAPACITY 置 0 | 将目标 AMF 排到选择末位 | AMF Pool 用户迁移 |
| PRIORITY 调低 | 确保目标 SMF priority 值最小 | SMF 多服务区优先级调测 |
| LOCALITY 配置 | PCFSELECTMODE=NFLOC 时按位置选 PCF | PCC 基本功能（NFLOC 选 PCF） |
| LOAD 调整 | 按规划值设置负载权重 | 一般场景 |

> 获取方法"全网规划/本端规划/-"多值分布，取值随场景分叉。

### 决策点：NF 状态取值（DP 0-01279）

| 选项 | 联动参数 | 适用场景 |
|---|---|---|
| Invalid | NF 注册时不携带状态，NRF 不会选择此 NF | 将 NF 标记为不可用 |
| Registered | NRF 只选择该状态下的 NF | 正常运营状态 |
| Undiscoverable | 保持心跳但不可被服务发现，适合运维场景 | 用户迁移等运维操作期间 |

> **注意：状态取值不支持 Suspend**。需将 NF 标记为不可选时用 Invalid 或 Undiscoverable，不能用 Suspend。

## 约束

- **立即生效并触发 NRF 更新**（critical，rule-0-01632）：立即生效并触发 NRF 更新：修改后立即生效，并向 NRF 发起更新流程，影响 NF 服务发现结果——多实例并发修改需评估对在用业务的影响

- **NFSTATUS 不支持 Suspend**（critical，rule-0-01633）：NFSTATUS 不支持 Suspend：状态取值不能选择 Suspend，需用 Invalid 或 Undiscoverable 代替

- **NFINSTANCENAME 存在性前置**（warning，rule-0-01634）：NFINSTANCENAME 存在性前置：NF 实例需已向 NRF 注册（或已 ADD NFPROFILE（★R1.5 atom 待补）），MOD 才能成功；可用 LST NFPROFILE 校验

- **CAPACITY/PRIORITY 语义方向相反**（warning，rule-0-01635）：CAPACITY/PRIORITY 语义方向相反：CAPACITY 越大容量越大；PRIORITY 越小优先级越高——注意两者方向相反，配置时不要混淆

- **PRIORITY 一致性（SMF 多服务区）**（warning，rule-0-01636）：PRIORITY 一致性（SMF 多服务区）：服务注册消息中 smfInfo 的 priority 值需小于 smfInfoList 中其他 smfInfo 的 priority 值，修改前用 LST NFPROFILE 核对当前值
