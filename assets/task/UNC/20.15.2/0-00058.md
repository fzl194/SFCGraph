# ADD PCRFBINDGRP 原子任务
> 命令静态知识见 [ADD PCRFBINDGRP](command/UNC/20.15.2/ADD-PCRFBINDGRP.md)。本页只讲配置生成实例化时这条命令**怎么配**。

## 配置方法
本命令把已配置的 PCRF（来自 [ADD PCRF](task/UNC/20.15.2/0-00057.md)）加入指定 PCRF 分组（来自 [ADD PCRFGROUP](task/UNC/20.15.2/0-00059.md)）。PCRFGRPNAME/PCRFHOSTNAME 均从已配置数据中获取。WEIGHT 仅在 PCRF 组为负荷分担模式（LOAD_BALANCE）时有意义。

| 参数 | 配置方法 | 取值样例 |
|---|---|---|
| PCRFGRPNAME | PCRF 组名称，从已配置的 `ADD PCRFGROUP` 数据中获取 | `pcrf_group_1` |
| PCRFHOSTNAME | PCRF 主机名，从已配置的 `ADD PCRF` 数据中获取 | `pcrf_1` |
| WEIGHT | PCRF 权重，本端规划，1~100（仅 LOAD_BALANCE 模式有意义，可选） | `40` |

典型脚本（负荷分担组）：
```
ADD PCRFBINDGRP:PCRFGRPNAME="pcrf_group_1",PCRFHOSTNAME="pcrf_1",WEIGHT=40;
ADD PCRFBINDGRP:PCRFGRPNAME="pcrf_group_1",PCRFHOSTNAME="pcrf_2",WEIGHT=60;
```
典型脚本（主备组，无需 WEIGHT）：
```
ADD PCRFBINDGRP:PCRFGRPNAME="pcrf_group_2",PCRFHOSTNAME="pcrf_3";
ADD PCRFBINDGRP:PCRFGRPNAME="pcrf_group_2",PCRFHOSTNAME="pcrf_4";
```

## 决策点

### 是否配置权重及权重分配比例
- 问什么：当所属 PCRF 组工作模式为 LOAD_BALANCE 时，是否配置各 PCRF 的流量分担权重。
| 组工作模式 | WEIGHT 配置 | 行为 |
|---|---|---|
| LOAD_BALANCE（负荷分担） | 本端规划，1~100 | 按权重百分比在组内 PCRF 间分担用户会话 |
| MASTER_SLAVE（主备） | 不配 | 先绑定的备 PCRF 优先级高（见约束） |

## 约束
- 生效时机：命令执行后**立即生效**。`severity=medium`
- 记录上限：最大记录数 **3200**；单个 PCRF 分组内最多 **32 个** PCRF。`severity=medium`
- 主备优先级：主备模式下绑定的多个备 PCRF，**先绑定者优先级高**（不执行删除操作时持续生效）。`severity=high`
- 导出导入排序：EXP 导出配置文件时，备 PCRF 绑定顺序按 PCRF 名称字典序排序，重新 import 时备 PCRF 选择顺序可能与原配置不同，需复核。`severity=medium`
- 前置依赖：PCRFGRPNAME 必须已由 [ADD PCRFGROUP](task/UNC/20.15.2/0-00059.md) 配置；PCRFHOSTNAME 必须已由 [ADD PCRF](task/UNC/20.15.2/0-00057.md) 配置。`severity=high`
