#### [适用NF](#ZH-CN_TOPIC_0250176278)

GGSN、PGW-C、SMF

#### [定义](#ZH-CN_TOPIC_0250176278)

UNC 支持多种接入方式将用户接入PDN：透明接入、非透明接入、透明鉴权接入和本地鉴权接入。

#### [客户价值](#ZH-CN_TOPIC_0250176278)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 针对不同的组网，可以选择不同的接入方式，从而限定为不同用户划分不同网络服务。使用公用的用户名和密码进行认证场景下，可以减少配置过程，提高易用性。 |
| 用户 | 使用公用的用户名和密码进行认证场景下，如果没有输入用户名和密码或者输入有误的情况下仍能接入，提升体验。 |

#### [应用场景](#ZH-CN_TOPIC_0250176278)

- 透明接入适用于用户访问运营商的网络，运营商不对用户权限和计费作要求的场景。
- 非透明接入适用于用户访问ISP（Internet service provider）或者企业网，ISP或者企业网要对用户权限有要求，并对用户进行计费。
- 透明鉴权接入适用于用户访问运营商的网络，运营商使用统一的用户名和密码为用户鉴权和计费。用户在没有输入用户名和密码或者输入有误的情况下仍能保证用户接入。
- 本地鉴权接入适用于用户访问ISP或者企业网，在没有部署AAA鉴权服务器的情况下，仍然可以提供简单鉴权功能，满足ISP或者企业网要对用户权限有要求。

#### [原理概述](#ZH-CN_TOPIC_0250176278)

**透明接入：** 用户在接入 UNC 时无需鉴权。

典型的透明接入方式是运营商充当Internet服务提供商ISP，直接向用户提供不同的业务，如电子邮件、主页浏览等。在运营商的IP网络中，可以提供WWW服务器、邮件服务器、DNS服务器等设备。在同外部网连接处，设置了一个防火墙，以对非法的访问进行屏蔽。这种场景下用户在PDP上下文/缺省承载/PDU会话创建过程中无需发送任何鉴权请求， UNC 也无需发起用户鉴权/授权过程。

这种方式下，用户在接入 UNC 时无需Radius鉴权，但是可能选择是否进行计费（请参考 [WSFD-011306 Radius功能](../WSFD-011306 Radius功能_14755851.md) ）。移动用户所分配的IP地址属于运营商的地址空间，该地址可以是在HLR/HSS/UDM签约时确定的IP地址（静态地址），也可以是在PDP上下文/缺省承载/PDU会话创建阶段获得的地址（动态地址）。

**非透明接入：** 用户在接入 UNC 时必须进行鉴权。

在一个非透明接入的PDP上下文/缺省承载/PDU会话创建过程中， UNC 提供到Radius服务器的用户身份认证。认证服务器属于用户所要连接的ISP或者企业网。

这种场景下用户需要在PDP上下文/缺省承载/PDU会话创建过程中发送鉴权请求， UNC 会向归属于ISP或者企业网的Server（如AAA服务器 （AAA服务器与客户端交互使用Radius协议，AAA服务器即Radius服务器） 等）请求用户鉴权。用户的鉴权请求信息在PCO（协议配置选项Protocol Configuration Option，该字段由用户在进行PDP上下文/缺省承载/PDU会话创建过程中携带到网络侧）中携带。 UNC 上获得Radius鉴权的结果和一些PCO申请的其他内容（如主备DNS等）都应该携带在PCO中作为响应，发送给用户。

这种方式下，用户在接入 UNC 时必须进行Radius鉴权， UNC 上支持PAP和CHAP两种Radius鉴权方式，Radius服务器同时可具有计费功能，可选择是否进行计费（请参考 [WSFD-011306 Radius功能](../WSFD-011306 Radius功能_14755851.md) ）。移动用户所分配的IP地址属于ISP或企业网的地址空间，该地址可以是在HLR/HSS/UDM签约时确定的IP地址（静态地址），也可以是在PDP上下文/缺省承载/PDU会话创建阶段获得的地址（动态地址）。

这种方式下， UNC 支持用户是否使用UDM的签约数据参与决策进行AAA鉴权，若用户使用UDM的签约数据进行AAA鉴权， UNC 支持配置用户是否使用UDM签约的AAA地址进行鉴权。

**透明鉴权接入：** 在用户接入 UNC 时， UNC 使用系统配置的用户名和密码去鉴权服务器为用户进行鉴权。

透明鉴权方式，又称为匿名接入，即透明鉴权方式。这种场景下，由 UNC 在网络侧使用该APN下配置的公用的用户名和密码发送鉴权请求到AAA Server，并从AAA Server获取控制用户的属性，以便对终端灵活控制。 UNC 支持基于APN配置默认的用户名和密码，当用户以该APN接入，同时APN配置的鉴权方式为透明鉴权时， UNC 直接使用该APN下配置的公用的用户名和密码为用户的用户名和密码，到Radius服务器上鉴权。

这种方式下，用户在接入 UNC 时使用系统配置的用户名和密码进行Radius鉴权， UNC 上支持PAP和CHAP两种Radius鉴权方式，Radius服务器同时可具有计费功能，可选择是否进行计费（请参考 [WSFD-011306 Radius功能](../WSFD-011306 Radius功能_14755851.md) ）。移动用户所分配的IP地址属于ISP或企业网的地址空间，该地址可以是在HLR/HSS/UDM签约时确定的IP地址（静态地址），也可以是在PDP上下文/缺省承载/PDU会话创建阶段获得的地址（动态地址）。

这种方式下， UNC 支持用户是否使用UDM的签约数据参与决策进行AAA鉴权，若用户使用UDM的签约数据进行AAA鉴权， UNC 支持配置用户是否使用UDM签约的AAA地址进行鉴权。

**本地鉴权接入：** 在用户接入 UNC 时， UNC 使用系统配置的用户名和密码与用户接入时携带的PCO的密码进行匹配为用户进行鉴权。

本地鉴权接入方式是在用户通过运营商网络访问ISP或者企业网时，运营商通过在 UNC 本地配置的公用的用户名和密码对用户进行鉴权的方式。在ISP网络或者企业网络没有部署AAA鉴权服务器不能提供鉴权功能的情况下，由运营商提供本地鉴权接入功能，可以保证ISP网络或者企业网络接入的安全性。这种场景下用户在PDP上下文/缺省承载/PDU会话创建过程中需发送携带的用户名和密码， UNC 根据本地的用户名和密码配置，对用户进行用户名和密码的匹配。

| 方式 | 相同点 | 不同点 |
| --- | --- | --- |
| 透明接入 | 作为用户接入网络的必要方式，均可使用在HLR/HSS/UDM签约时确定的IP地址（静态地址），也可使用在PDP上下文/缺省承载/PDU会话创建阶段获得的地址（动态地址）。 | 无需对用户进行鉴权和计费，用户不需要进行用户名和密码输入操作。 |
| 非透明接入 | 作为用户接入网络的必要方式，均可使用在HLR/HSS/UDM签约时确定的IP地址（静态地址），也可使用在PDP上下文/缺省承载/PDU会话创建阶段获得的地址（动态地址）。 | 需要对用户进行鉴权和计费（计费可选），用户进行用户名和密码输入操作。 |
| 透明鉴权接入 | 作为用户接入网络的必要方式，均可使用在HLR/HSS/UDM签约时确定的IP地址（静态地址），也可使用在PDP上下文/缺省承载/PDU会话创建阶段获得的地址（动态地址）。 | 需要对用户进行鉴权和计费（计费可选），用户不需要进行用户名和密码输入操作，<br>UNC<br>使用统一的用户名和密码为用户鉴权。 |
| 本地鉴权接入 | 作为用户接入网络的必要方式，均可使用在HLR/HSS/UDM签约时确定的IP地址（静态地址），也可使用在PDP上下文/缺省承载/PDU会话创建阶段获得的地址（动态地址）。 | 需要对用户进行鉴权，用户需要进行用户名和密码输入操作，<br>UNC<br>使用统一的用户名和密码为用户鉴权。 |
