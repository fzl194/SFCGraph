# 查询跟踪调试信息(DSP TRCDBGINFO)

#### [命令功能](#ZH-CN_CONCEPT_0129626899)

该命令用于显示VNFC跟踪任务的调试信息，此信息只用于跟踪问题定位，不涉及用户私有信息。

#### [注意事项](#ZH-CN_CONCEPT_0129626899)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0129626899)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0129626899)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROCID | 进程ID | 可选必选说明：可选参数<br>参数含义：该参数表示指定查询进程的ID。通过<br>**DSP PROCESSXXX**<br>（XXX表示CSLB等服务，例如CSLB的命令为DSP PROCESSCSLB，CSDB的命令为DSP PROCESSCSDB）命令可以查询进程ID。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1000～65535。<br>默认值：无 |
| SERVICEINSTANCE | 服务实例 | 可选必选说明：必选参数<br>参数含义：该参数表示大颗粒服务实例名称。<br>数据来源：本端规划<br>取值范围：字符串类型，通过LST VNFC命令获取。该参数不能取值为VNFP、ACS、VNRS_VNFC的服务实例名称。<br>默认值：无<br>配置原则：只能填写通过LST VNFC命令查询到的管理代理标识。 |

#### [使用实例](#ZH-CN_CONCEPT_0129626899)

查询1003号进程上的跟踪任务的调试信息：

```
DSP TRCDBGINFO:PROCID=1003
,SERVICEINSTANCE="vnfc"
;
```

```
RETCODE = 0  操作成功。

结果如下
--------
              进程ID  =  1003
            跟踪类型  =  4103
            跟踪句柄  =  1051648
      接收的消息总数  =  1
定位比较失败的消息数  =  0
过滤比较失败的消息数  =  0
    上报成功的消息数  =  1
    上报失败的消息数  =  0
(结果个数 = 1)
---    END
```

#### [输出结果说明](#ZH-CN_CONCEPT_0129626899)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 进程ID | 显示的进程ID。<br>取值范围：整数类型，取值范围为1000～65535。 |
| 跟踪类型 | 显示的跟踪类型。<br>取值范围：整数类型，取值范围为0～65535。 |
| 跟踪句柄 | 显示的跟踪句柄。<br>取值范围：整数类型，取值范围为0～4294967295。 |
| 接收的消息总数 | 进程上报的跟踪消息总数。<br>取值范围：整数类型，取值范围为0～4294967295。 |
| 定位比较失败的消息数 | 定位参数不一致引起的跟踪上报失败的消息数。<br>取值范围：整数类型，取值范围为0～4294967295。 |
| 过滤比较失败的消息数 | 过滤参数不一致引起的跟踪上报失败的消息数。<br>取值范围：整数类型，取值范围为0～4294967295。 |
| 上报成功的消息数 | 进程上报成功的跟踪消息数。<br>取值范围：整数类型，取值范围为0～4294967295。 |
| 上报失败的消息数 | 进程上报失败的跟踪消息数。<br>取值范围：整数类型，取值范围为0～4294967295。 |
