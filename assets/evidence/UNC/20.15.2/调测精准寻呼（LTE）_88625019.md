# 调测精准寻呼（LTE）

- [操作场景](#ZH-CN_OPI_0188625019__1.3.1)
- [必备事项](#ZH-CN_OPI_0188625019__1.3.2)
- [操作步骤](#ZH-CN_OPI_0188625019__1.3.3)

## [操作场景](#ZH-CN_OPI_0188625019)

随着PS寻呼量的快速增加，无线寻呼信道的拥塞越来越严重。 UNC 根据配置的寻呼规则、寻呼策略及移动性规则，依次在各个寻呼范围内进行智能寻呼，从而缩小寻呼范围，节省网络资源。本操作介绍了通过调测手段检查 UNC 是否正确配置该功能的过程。

## [必备事项](#ZH-CN_OPI_0188625019)

前提条件

- 请仔细阅读[WSFD-206001 LTE精准寻呼特性概述](特性概述_88625076.md)
- 完成[激活精准寻呼（LTE）](激活精准寻呼（LTE）_88625065.md)。

数据

该操作无需准备数据。

工具

- 测试终端
- OM Portal

## [操作步骤](#ZH-CN_OPI_0188625019)

- **检查是否激活精准寻呼。**
  进入 “MML命令行-UNC” 窗口。
  [**DSP LICENSE**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/显示License(DSP LICENSE)_00360098.md)
    - 预期结果：License资源项 “精准寻呼” 为实际购买的License数量。
    - 异常结果：License资源项 “精准寻呼” 为 “0” 。
      异常结果处理：激活 “精准寻呼” License。
- **检查LTE精准寻呼配置。**
    1. 检查寻呼规则配置。
      [**LST S1PAGINGCTRL**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼策略管理/查询S1寻呼策略控制表(LST S1PAGINGCTRL)_26146246.md)
      ```
      %%LST S1PAGINGCTRL:;%%
      RETCODE = 0  操作成功。

      输出结果如下
      ------------
               最近eNodeB开关  =  开
               邻接eNodeB开关  =  关
                   最近TA开关  =  关
               SUB_T3413（s）  =  3
             低移动性检查开关  =  开
        eNodeB粘性时长（min）  =  2
            TA粘性时长（min）  =  0
                  ECT学习开关  =  关
                 X2HO学习开关  =  关
                 S1HO学习开关  =  关
      邻接eNodeB老化时间（h）  =  360
                 全网寻呼开关  =  关
         SGs寻呼重发上限（s）  =  15
             寻呼汇聚功能开关  =  开
      (结果个数 = 1)
      ---    END
      ```
          - 预期结果：各项参数显示结果与配置结果一致。
          - 异常结果：各项参数显示结果与配置结果不一致或查看不到相关配置信息。
            异常结果处理：参考 [激活LTE精准寻呼](激活精准寻呼（LTE）_88625065.md) 并重新配置。
    2. 检查寻呼策略配置。
      [**LST S1PAGINGRULE**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼规则管理/查询S1寻呼规则(LST S1PAGINGRULE)_72225927.md)
      ```
      %%LST S1PAGINGRULE: RECID=1;%%
      RETCODE = 0  操作成功。

      输出结果如下
      ------------
             规则索引  =  1
           用户群类型  =  指定IMSI前缀
             IMSI前缀  =  12300000
           MSISDN前缀  =  NULL
             IMEI TAC  =  NULL
             业务类型  =  下行数据触发
              APN指示  =  所有APN
           移动国家码  =  NULL
             移动网号  =  NULL
               APN NI  =  NULL
                QCI值  =  NULL
            ARP优先级  =  NULL
      SGs接口业务指示  =  NULL
             消息类型  =  CBR/UBR & DDN
           匹配优先级  =  200
         寻呼动作组合  =  NULL
             规则描述  =  NULL
      (结果个数 = 1)
      ---    END
      ```
          - 预期结果：各项参数显示结果与配置结果一致。
          - 异常结果：各项参数显示结果与配置结果不一致或查看不到相关配置信息。
            异常结果处理：参考 [激活LTE精准寻呼](激活精准寻呼（LTE）_88625065.md) 并重新配置。
    3. 可选：检查eNodeB邻接关系。
      当 [**ADD S1PAGINGRULE**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼规则管理/增加S1寻呼规则(ADD S1PAGINGRULE)_26306058.md) 命令中配置的寻呼动作组合中包含邻接eNodeB，且 [**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md) 命令中 “邻接eNodeB开关” 已开启时，需要检查此项，否则可以跳过。
      [**DSP ENBNEIBS**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/eNodeB邻接关系管理/显示eNodeB邻接关系(DSP ENBNEIBS)_26146258.md)
          - 预期结果：能够查询到指定中心eNodeB的邻接eNodeB列表，eNodeB已经学习到邻接关系。
          - 异常结果：查询不到指定中心eNodeB的邻接eNodeB列表，eNodeB未完成邻接关系的学习。
            异常结果处理：请联系华为技术支持处理。
            > **说明**
            > MME学习eNodeB邻接关系需要约一周时间，如果配置完立即进行调测，存在查询不到学习结果的场景。
- **调测LTE精准寻呼** 。
  预置条件：
    - 寻呼范围组合为：“最近eNodeB->TA List”为例。
    - 拨测的用户在最近eNodeB附着时间大于2分钟（低移动性检查已开启且eNodeB粘性时长为2分钟）。
    - 已创建用户跟踪。IMSI：被跟踪用户的IMSI。其他参数：选择默认值。
    - 已创建用户所在最近eNodeB以及预计用户会移动到的eNodeB的S1接口跟踪。
    1. 用户附着接入MME，执行如下命令 **，** eNodeB释放S1链接，使UE处于ECM-IDLE状态。
      **[**RMV USRS1CONN**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1接口用户连接/删除S1接口用户连接(RMV USRS1CONN)_72225921.md) : RMVOPTION=BYIMSI, IMSI="<IMSI>"**
      查询MM上下文状态信息。
      [**DSP COMMMCTX**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/融合接入业务管理/融合用户数据库管理/显示移动性管理上下文的相关信息（DSP COMMMCTX）_58365337.md)
      ```
      %%DSP COMMMCTX: QUERYOPT=BYIMSI, IMSI="123036500000002";%%
      RETCODE = 0  操作成功

      MM上下文信息：
      ----------------
                 ............

                              移动管理状态  =  ECM-IDLE

                 ............

      ---    END
      ```
          - 预期结果： “移动管理状态” 为 “ECM-IDLE” 。
          - 异常结果： “移动管理状态” 不为 “ECM-IDLE” 。
    2. 从其他用户向用户发送数据业务（例如发起微信视频），触发被测试用户的下行数据业务，验证MME配置后，普通数据业务支持精准寻呼功能。
          - 预期结果：
                  a. 现象1：通过S1接口跟踪以及用户跟踪可观察到首次寻呼的发送范围为用户上次接入的eNodeB，如果寻呼成功，用户发送Service Request消息响应Paging消息，数据业务正常。
                  b. 现象2：首次寻呼最近eNodeB失败，终端不响应时，MME重发寻呼消息的发送范围为用户注册的TA List中所有TA区中的eNodeB。如果二级寻呼成功，用户发送Service Request消息响应Paging消息，数据业务正常。
          - 异常结果：不符合以上两种现象预期，请联系华为技术支持处理。
    3. 对用户做语音被叫，验证MME配置后，CSFB被叫语音业务不支持精准寻呼功能。
          - 预期结果：通过S1接口跟踪以及用户跟踪可观察到首次寻呼的发送范围为用户注册的TA List中所有TA区中的eNodeB。寻呼成功后，用户发送Extend Service Request消息响应寻呼消息，语音通话正常。
          - 异常结果：不符合预期时，请联系华为技术支持处理。
    4. 查询MM上下文状态消息。
      [**DSP COMMMCTX**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/融合接入业务管理/融合用户数据库管理/显示移动性管理上下文的相关信息（DSP COMMMCTX）_58365337.md)
          - 预期结果： “移动管理状态” 为 “ECM-CONNECTED” 。
          - 异常结果： “移动管理状态” 不为 “ECM-CONNECTED” 。
            异常结果请联系华为技术支持处理。
