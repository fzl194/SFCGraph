详细流程说明如下：

1. 当用户发起附着或路由区更新时，UNC首先比较用户的IMSI与配置的IMSI前缀是否匹配。
    - 是 => 步骤[2](#ZH-CN_CONCEPT_0168358178__step2)
    - 否 => 允许接入
2. UNC判断是否配置了接入控制属性列表。
    - 是 => 步骤[3](#ZH-CN_CONCEPT_0168358178__step3)
    - 否 => 步骤[4](#ZH-CN_CONCEPT_0168358178__step4)
3. UNC根据用户属性（签约ARD/APN/卡类型）判断是否至少有一个用户属性与配置的属性相匹配。
    - 是 => 步骤[4](#ZH-CN_CONCEPT_0168358178__step4)
    - 否 => 步骤[5](#ZH-CN_CONCEPT_0168358178__step5)
4. UNC采取和配置策略相同的接入控制策略。
5. UNC采取和配置策略相反的接入控制策略。
6. UNC判断配置的控制策略。
    - 拒绝 => 拒绝接入
    - 允许 => 允许接入

根据上述的接入控制策略，针对用户的典型配置策略如 [表1](#ZH-CN_CONCEPT_0168358178__table1) 所示。

**根据签约的ARD控制用户接入功能原理**

ARD：接入限制数据。3GPP R6协议中在HLR/HSS签约数据中加入了ARD信息，该信息中指示了允许用户使用的RAT（Radio Access Technologies），用于表明是否拒绝用户接入GERAN、UTRAN或E-UTRAN。

ARD信息在HLR/HSS和SGSN/MME之间通过Insert Subscriber Data Request消息或Update Location Answer消息中的AccessRestrictionData信元传送。AccessRestrictionData信元包括bit0~bit7八个bit位，bit0、bit1、bit4分别代表了是否允许用户接入UTRAN、GERAN或E-UTRAN，bit6代表是否允许NB-IoT用户接入网络。其中bit位取值为1表示拒绝接入，为0表示允许接入。

> **说明**
> 如果 UNC 支持ARD特性，但是没有从HLR/HSS收到RAT消息， UNC 将假定用户签约时不需要限制RAT。

若 UNC 启动了支持ARD功能，它就可以在接收到Insert Subscriber Data Request消息或Update Location Answer消息中的ARD信息后，结合当前用户请求登录的无线接入系统，来判断是允许还是拒绝该用户的接入请求。如果拒绝该用户的接入请求， UNC 根据命令 [**ADD GBARD**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/接入限制管理/Gb模式接入限制参数/增加Gb模式接入限制参数(ADD GBARD)_26305284.md) 、 [**ADD IUARD**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/接入限制管理/Iu模式接入限制参数/增加Iu模式接入限制参数(ADD IUARD)_72345073.md) 或 [**ADD S1ARD**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/接入限制管理/S1模式接入限制参数/增加S1模式接入限制参数(ADD S1ARD)_26145478.md) 下的参数 “原因值” 的设置，返回给用户合适的拒绝原因值。

例如：根据IMSI号段和签约的ARD信息拒绝用户接入到UTRAN网络，具体实现方式如下：

1. 当用户发起附着或路由区更新时，如果该用户的IMSI号码落在所设置的IMSI号段内，且UE是通过UTRAN接入。UNC根据接收到Insert Subscriber Data Request消息或Update Location Answer消息中的ARD信息为“禁止接入UMTS”，来拒绝用户接入。
2. 当HLR发起用户签约数据修改时，如果该用户的IMSI号码落在所设置的IMSI号段内，且UE是通过UTRAN接入。此时如果HLR返回消息中的ARD信息变更为“禁止接入UMTS”，则UNC会直接将该用户分离。

> **说明**
> 用户国际漫游出访时，在拜访地做位置更新，用户归属HLR/HSS不将ARD数据送往国际局，而是让用户遵从当地运营商管理策略。所以用户服务不会受到影响。

**根据签约的APN控制用户接入功能原理**

在HLR/HSS中为用户签约一个特殊的APNNI，此APNNI用来控制用户是否允许接入GERAN、UTRAN或E-UTRAN。用户附着时 UNC 可以据此来判断用户登录的无线接入系统是GERAN、UTRAN还是E-UTRAN，从而进行接入控制。

例如，HLR/HSS中用户的APN签约情况如下：

- 正常签约用户：Internet，WAP
- GERAN接入用户：Internet，WAP，GERAN_USER
- UTRAN接入用户：Internet，WAP，UTRAN_USER
- E-UTRAN接入用户：Internet， WAP， E_UTRAN_USER

> **说明**
> 其中，GERAN_USER是标识接入GERAN的用户的特殊APNNI，UTRAN_USER是标识接入UTRAN的用户的特殊APNNI，E_UTRAN_USER是标识接入E-UTRAN的用户的特殊APNNI。

UNC 通过配置上述特殊APNNI，来控制用户接入。

例如：根据IMSI号段和签约的APN只允许GERAN用户接入到GERAN网络，具体实现方式如下：

当用户请求接入时， UNC 首先根据用户IMSI号段中的IMSI信息找到与之对应的APNNI列表，然后比对该列表中的APNNI和用户在HLR中签约的特殊APNNI，比对一致，则 UNC 允许GERAN用户接入GERAN网络，否则 UNC 拒绝其接入GERAN网络。

> **说明**
> 这种接入控制方式可能导致电路域和分组域区分不同接入网用户的标准不同。

**根据卡类型控制用户接入功能原理（只适用于GERAN/UTRAN接入用户）**

UNC 通过分析用户签约数据中的鉴权信息是三元组还是五元组来确定是SIM卡还是USIM卡，从而区分GERAN用户和UTRAN用户，实现接入控制。具体有以下两种控制方式：

- 拒绝SIM卡用户接入UTRAN网络
  UNC 启用拒绝SIM卡用户接入UTRAN网络功能后，当用户请求附着或路由区更新时， UNC 向HLR请求取该用户的鉴权参数，取得鉴权参数后，判断其中是否带有鉴权三元组，若有，则拒绝该用户接入UTRAN网络，否则允许接入。
- 拒绝USIM卡用户接入GERAN网络
  UNC 启用拒绝USIM卡用户接入GERAN网络功能后，当用户请求附着或路由区更新时， UNC 向HLR请求取该用户的鉴权参数，取得鉴权参数后，判断其中是否带有鉴权五元组，若有，则拒绝该用户接入GERAN网络，否则允许接入。

> **说明**
> - SIM（Subscriber Identity Module）卡是用户识别模块，也称为智能卡、用户身份识别卡，卡中包括了用户接入所注册的网络的相关信息，用于进行用户身份识别。SIM卡在GSM网络/UMTS网络均可以使用。
> - USIM（UMTS Subscriber Identity Module）卡是UMTS用户识别模块。USIM卡除了包括用户接入所注册的网络的相关信息之外，还在安全性方面对算法进行了升级，并增加了卡对网络的认证功能，这种双向认证可以有效防止黑客对卡片的攻击。USIM卡在GSM网络/UMTS网络/LTE网络均可以使用。

例如：根据IMSI号段和卡类型拒绝USIM卡用户接入到GERAN网络的具体实现方式如下：

当用户请求附着或路由区更新时， UNC 首先比较发现该用户的IMSI号码落在所设置的IMSI号段内，再向HLR请求取该用户的鉴权参数，取得鉴权参数后，判断其中是否带有鉴权五元组，若有，则拒绝该用户接入GERAN网络。
