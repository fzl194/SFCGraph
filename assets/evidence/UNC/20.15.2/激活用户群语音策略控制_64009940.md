## [操作场景](#ZH-CN_OPI_0164009940)

本操作指导介绍在运行网络中激活用户群语音策略控制特性的操作过程。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0164009940)

前提条件

- 请仔细阅读[WSFD-201002 用户群语音策略控制特性概述](特性概述_70014709.md)。
- 已完成加载License。
- 运营商必须为IMS用户签约IMS APN，且IMS APN不能为“*”。
- （可选）启用 [WSFD-106203 别名APN](../../会话管理功能/WSFD-106203 别名APN_64082528.md) 特性。
  如果外网用户签约的IMS APN和本网运营商部署的IMS APN不同的情况下，例如外网用户的IMS APN为“IMS-X”，本网运营商部署的APN为“IMS”，建议开启 [WSFD-106203 别名APN](../../会话管理功能/WSFD-106203 别名APN_64082528.md) 特性，将外网用户的IMS APN转换成和本网的IMS APN相同，简化网络部署。

## [操作步骤](#ZH-CN_OPI_0164009940)

- 独立部署
  运营商可以根据实际部署情况，选择先部署本特性，再部署 [WSFD-201004 基于IMEI的语音策略控制](../WSFD-201004 基于IMEI的语音策略控制_70014720.md) 。
    1. 进入 “MML命令行-UNC” 窗口。
    2. 打开本特性的License配置开关。
      [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
      > **说明**
      > MME被共享的场景下，语音策略的配置过程和MME不被共享情况下的类似。 [步骤 4](#ZH-CN_OPI_0164009940__step2_Local) 到 [步骤 5](#ZH-CN_OPI_0164009940__step3_foreign) 为MME不被共享场景下的配置过程。
      >
      > 如果MME被共享，例如MME被运营商A和运营商B共享，重复执行 [步骤 4](#ZH-CN_OPI_0164009940__step2_Local) 到 [步骤 5](#ZH-CN_OPI_0164009940__step3_foreign) ，分别完成运营商A和运营商B的本网用户和外网用户的语音策略。
    3. **可选：**当需要配置禁止特征RFSP用户使用IMS VoPS业务时，需要执行此步骤，增加特征RFSP索引。
      [**ADD SPECRFSPVALUE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/RFSP管理/特征RFSP管理/特征RFSP取值范围管理/增加特征RFSP取值(ADD SPECRFSPVALUE)_26145534.md)
    4. 配置本网用户的语音策略。
      [**ADD VOICEDEPLOY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/语音业务管理/增加语音部署配置(ADD VOICEDEPLOY)_72345361.md)
    5. 配置外网用户的语音策略。
          a. 配置语音策略。
            [**ADD VOICEDEPLOY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/语音业务管理/增加语音部署配置(ADD VOICEDEPLOY)_72345361.md)
          b. 将外网用户的IMS APN转换成本网部署的IMS APN。
            [**ADD ALIASAPN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/别名APN管理/增加别名APN配置(ADD ALIASAPN)_72345281.md) :
            > **说明**
            > 如果外网用户签约的IMS APN和本网运营商部署的IMS APN不同的情况下，例如外网用户的IMS APN为“IMS-X”，本网运营商部署的APN为“IMS”，且运营商不需要在计费话单上区分不同运营商的IMS APN，则推荐使用该配置，简化网络部署复杂度。
          c. 配置外网用户采用Local Breakout进行VoLTE语音业务。
            [**ADD GWSELPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md)
            > **说明**
            > GSMA标准要求VoLTE业务漫游采用Local Breakout的方式。如果需要强制要求外网的IMS用户使用Local Breakout方式，执行本步骤。否则，不执行本步骤，系统根据用户的签约数据决定是否使用Local Breakout方式。
            >
            > - 如果没有进行别名转换，“APNNI（APNNI）”取值为[5.a](#ZH-CN_OPI_0164009940__step3a)中的APN。
            > - 如果进行了别名转换，“APNNI（APNNI）”取值为[5.b](#ZH-CN_OPI_0164009940__step3b)中转换后的APN。
    6. **可选：**配置用户使用IMS VoPS业务的匹配模式。
          - 若对所有用户系统的匹配模式均一致，则执行如下步骤。
            [**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md)
          - 若存在部分用户的匹配模式与其他用户不一致的情况，则执行如下步骤。
                  - [**LST DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/查询Diameter兼容配置(LST DMCMPT)_72345869.md)
                    查询系统中Diameter兼容性配置

                    ![](激活用户群语音策略控制_64009940.assets/notice_3.0-zh-cn_2.png)

                    命令 [**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md) 与 [**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 中除 “用户范围” 、 “运营商标识” 以及 “IMSI前缀” 外的其余参数功能均相同，且若系统中同时存在这两条命令的配置，系统会优先匹配命令 [**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md) 的配置。因此在配置 [**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md) 前需先执行 [**LST DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/查询Diameter兼容配置(LST DMCMPT)_72345869.md) 查询系统中 [**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 的配置记录，确保命令 [**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md) 的配置记录中所携带的参数默认值不影响系统已有配置。
                  - 配置用户使用IMS VoPS业务的匹配模式。
                    [**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md)

      [**SET IMSVOPS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/VoPS管理/设置VoPS配置(SET IMSVOPS)_72345711.md)

## [任务示例](#ZH-CN_OPI_0164009940)

任务描述

如 [表1](#ZH-CN_OPI_0164009940__plc) 所示，列举了3种典型的用户群语音策略场景。

- 场景一：运营商A（NO ID=0）允许本网用户使用VoLTE语音业务，且优先使用VoLTE语音业务，不允许外网用户使用VoLTE语音业务。对于Voice Centric类型的终端需要考虑UE的SRVCC能力。
- 场景二：运营商A（NO ID=0）允许本网用户使用VoLTE语音业务，仅允许来自运营商B和运营商C的外网用户使用VoLTE语音业务，其他外网用户不允许使用VoLTE语音业务。其中运营商C的用户签约的IMS APN和运营商A的不同，为“IMS-X”。对于Voice Centric类型的终端需要考虑UE的SRVCC能力。
- 场景三：运营商A（NO ID=0）允许Voice Centric类型的终端本网用户使用VoLTE语音业务，不允许外网用户使用VoLTE语音业务。对于Voice Centric类型的终端不需要考虑UE的SRVCC能力。

*表1 语音策略场景*

| 场景 | 用户类型 | 签约APN | VoLTE语音策略 |
| --- | --- | --- | --- |
| 场景一 | 本网用户 | IMS | 允许使用VoLTE，且指示用户优先使用VoLTE语音业务。 |
| 场景一 | 外网用户 | IMS | 不允许使用VoLTE。 |
| 场景二 | 本网用户 | IMS | 允许使用VoLTE，且不指示用户优先使用VoLTE语音业务。 |
| 场景二 | 外网用户（来自运营商B） | IMS | 允许使用VoLTE，且不指示用户优先使用VoLTE语音业务。 |
| 场景二 | 外网用户（来自运营商C） | IMS-X | 允许使用VoLTE，且不指示用户优先使用VoLTE语音业务。 |
| 场景二 | 外网用户（来自B和C之外的运营商） | - | 不允许使用VoLTE。 |
| 场景三 | 本网用户 | IMS | 允许Voice Centric类型的终端本网用户使用VoLTE语音业务，且不指示用户优先使用VoLTE语音业务。 |
| 场景三 | 外网用户 | IMS | 不允许使用VoLTE。 |

//打开本特性的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2VPCU01", SWITCH=ENABLE
```

//配置用户群语音策略。

- 场景一：运营商A允许本网用户使用VoLTE语音业务，且优先使用VoLTE语音业务，不允许外网用户使用VoLTE语音业务。
  //配置本网用户的语音策略。
  ```
  ADD VOICEDEPLOY: SUBRANGE=HOME_USER,
  NOID=0, IMSVOPS=YES, APNNI="IMS", UETYPE=VOICE_CENTRIC-1&DATA_CENTRIC-0, CSFBNOTPREIND=YES, IMEICTRL=NO; 
  ```
  //配置外网用户的语音策略。
  ```
  ADD VOICEDEPLOY: SUBRANGE=FOREIGN_USER, NOID=0, IMSVOPS= NO; 
  ```
  //可选，针对“IMS PS voice preferred，CS Voiceas secondary”的终端，配置不检查用户的签约SRVCC能力。
  ```
  SET IMSVOPS: VDPSRVCCPOLICY=PS_CS_IGNORE_STNSR-1; 
  ```
- 场景二：运营商A允许本网用户使用VoLTE语音业务，仅允许来自运营商B和运营商C的部分外网用户使用VoLTE语音业务，其他外网用户不允许使用VoLTE语音业务。其中运营商C的用户签约的IMS APN和运营商A的不同，为“IMS-X”。
  //配置本网用户的语音策略。
  ```
  ADD VOICEDEPLOY: SUBRANGE=HOME_USER, NOID=0, IMSVOPS=YES, APNNI="IMS", UETYPE=VOICE_CENTRIC-1&DATA_CENTRIC-0, CSFBNOTPREIND=NO, IMEICTRL=NO;
  ```
  //配置外网用户语音策略。
  ```
  ADD VOICEDEPLOY: SUBRANGE= FOREIGN_USER, NOID=0, IMSVOPS= NO; 
  ```
  //配置运营商B的用户语音策略。
  ```
  ADD VOICEDEPLOY: SUBRANGE=IMSI_PREFIX, IMSIPRE="46123", IMSVOPS=YES, APNNI="IMS", UETYPE=VOICE_CENTRIC-1&DATA_CENTRIC-0, CSFBNOTPREIND=NO, IMEICTRL=NO; 
  ADD GWSELPLCY: SUBRANGE=IMSI_PREFIX, IMSIPRE="46123", APNNI="IMS", VAA=YES;
  ```
  //配置运营商C的用户语音策略。
  ```
  ADD VOICEDEPLOY: SUBRANGE=IMSI_PREFIX, IMSIPRE="46321", IMSVOPS=YES, APNNI="IMS-X", UETYPE=VOICE_CENTRIC-1&DATA_CENTRIC-0, CSFBNOTPREIND=NO, IMEICTRL=NO; 

  ADD ALIASAPN: SUBRANGE=IMSI_PREFIX, IMSIPRE="46321", OLDAPN="IMS-X", NEWAPN="IMS"; 
 
  ADD GWSELPLCY: SUBRANGE=IMSI_PREFIX, IMSIPRE="46321", APNNI="IMSIMS-X", VAA=YES; 
  ```
  //可选，针对“IMS PS voice preferred，CS Voice as secondary”的终端，配置不检查用户的签约SRVCC能力。
  ```
  SET IMSVOPS: VDPSRVCCPOLICY=PS_CS_IGNORE_STNSR-1;
  ```

  ```
  SET DMCMPT: NORPARA=IMS_VOPS-1, HSIMSVOPS=BY_UE_CAP, UECAPMATCH=EXACT;  
  ```
- 场景三：运营商A允许Voice Centric类型的终端本网用户使用VoLTE语音业务，不允许外网用户使用VoLTE语音业务。对于Voice Centric类型的终端不需要考虑UE的SRVCC能力。
  //配置本网用户的语音策略。
  ```
  ADD VOICEDEPLOY: SUBRANGE=HOME_USER, NOID=0, IMSVOPS=YES, APNNI="IMS", UETYPE=DATA_CENTRIC-0&VOICE_CENTRIC-0, SUBRFSP=NO, IMEICTRL=NO, CSFBNOTPREIND=YES;
  ```
