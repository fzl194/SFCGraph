## [操作步骤](#ZH-CN_OPI_0248998097)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. 配置相同策略的SMF选择的DNN。
  **[ADD DNNGRP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/通用配置管理/DNN群组标识管理/增加DNN群组（ADD DNNGRP）_64343822.md)**

4. 针对DNN群组对应的本网用户进行特定服务范围的SMF选择。
  [**ADD SMFSELPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NF发现和选择管理/SMF选择策略管理/增加SMF选择策略（ADD SMFSELPLCY）_09653765.md)

## [任务示例](#ZH-CN_OPI_0248998097)

任务描述

大型企业业务，基于某些DNN在不同区域部署多个SMF共同服务于服务区域范围，其中，每个SMF分别就近服务于不同的区域范围，SMF支持的DNN和Snssais相同。

例如，企业业务部署了3个SMF（支持的DNN为Huawei1和Huawei2）共同服务于A、B、C、D、E、F、G、H、I、J 10个省份，其中：

A、B、C、D省之间距离较近，SMF_1部署位置在C省。

E、F、G省之间距离较近，SMF_2部署位置在F省。

H、I、J省之间距离较近，SMF_3部署位置在I省。

具体规划的SMF和服务范围关系如 [表1](#ZH-CN_OPI_0248998097__table1026775313139) 所示。

*表1 多点部署的SMF服务范围*

| SMF | SMF服务范围区域(SMF注册时携带此服务范围) |
| --- | --- |
| SMF_1 | A；B；C；D |
| SMF_2 | E；F；G |
| SMF_3 | H；I；J |

运营商规划DNN为Huawei1和Huawei2的本网用户进行业务时，可以根据接入地选择特定服务范围的SMF提供业务（就近选择SMF）。针对A省接入的用户，AMF选择服务范围包含A的SMF进行服务，对应的AMF配置如下，其他省份的用户接入配置类似。

脚本

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2SDSC01", SWITCH=ENABLE;
```

//对进行相同SMF选择策略的DNN进行组划分，便于统一管理SMF选择策略。

```
ADD DNNGRP: DNNGRPID="GRP_DNN1", DESC="for SMF Selection";
ADD DNNGRPMEM: DNNGRPID="GRP_DNN1", DNN="Huawei1";
ADD DNNGRPMEM: DNNGRPID="GRP_DNN1", DNN="Huawei2";
```

//规划DNN为Huawei本网用户的PDU会话采用相同的SMF选择策略，即就近选择本接入省A省服务范围的SMF。

```
ADD SMFSELPLCY:SUBRANGE=HOME_USER, DNNGRPID="GRP_DNN1", SSSW=YES, SERVINGSCOPE="A";
```
