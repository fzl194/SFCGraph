# 扩展QCI/5QI应用

*表1 不同场景中 UNC 对QCI/5QI的处理方式*

| 场景 | QCI来源 | 相关流程 |
| --- | --- | --- |
| [场景1](#ZH-CN_TOPIC_0311073829__fig133414419382) | 签约QoS | UNC<br>从UDM获取签约QoS |
| [场景2](#ZH-CN_TOPIC_0311073829__fig18526173196) | 动态PCC规则 | UNC<br>从PCF获取动态PCC规则（Npcf_SMPolicyControl_Create Response） |
| [场景3](#ZH-CN_TOPIC_0311073829__fig13794102919488) | 签约QoS | - 用户附着，UNC从MME收到创建会话的请求（Create Session Request）<br>- UNC从MME收到更新缺省承载的请求（Modify Bearer Command） |
| [场景4](#ZH-CN_TOPIC_0311073829__fig0801144618316) | 动态PCC规则 | - UNC从PCF/PCRF获取动态PCC创建缺省承载/专有承载<br>- 动态PCC下，PCF/PCRF主动更新用户缺省承载 |
| [场景5](#ZH-CN_TOPIC_0311073829__fig1163842727) | 动态PCC规则 | 对于GPRS/UMTS用户，一次/二次PDP上下文激活/更新流程中，<br>UNC<br>收到PCRF下发的更新PDP消息/CCA应答消息。 |

**图1** 场景1： UNC 从UDM获取签约QoS

<br>

![](扩展QCI_5QI应用_11073829.assets/zh-cn_image_0313094603_2.png)

1. 5QI如果不在1~255之间，会被 UNC 认定为不合法，则通过 **[SET 5GCQOSCORRT](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/5GC QoS配置/全局5GC QoS纠错/设置5GC QoS纠错配置（SET 5GCQOSCORRT）_09652438.md)** 进行纠错。
2. 当5QI为扩展5QI（即5QI值在 **[SET EXTENQCIATTR](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI应用属性/设置应用扩展QCI参数属性（SET EXTENQCIATTR）_09653201.md)** 中 **EXTENDQCIRANGE** 配置的范围内且不在标准5QI范围内）时，需要打开本特性的License，未开启License则使用 **[SET 5GCQOSCORRT](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/5GC QoS配置/全局5GC QoS纠错/设置5GC QoS纠错配置（SET 5GCQOSCORRT）_09652438.md)** 配置的5QI进行激活。
3. 如果是本地用户，则使用该扩展5QI进行接入。针对漫游用户，还需要开启 [**SET EXTENQCIATTR**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI应用属性/设置应用扩展QCI参数属性（SET EXTENQCIATTR）_09653201.md) 中的“QCICTRLFLAG（用户接入的扩展QCI控制开关）”。
    - 当开关开启，则根据[**ADD EXTENDQCIMAP**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI和标准QCI映射/增加扩展QCI和标准QCI的映射关系（ADD EXTENDQCIMAP）_09653762.md)配置扩展5QI和标准5QI的映射关系，将扩展5QI转换成标准5QI进行接入，未配置[**ADD EXTENDQCIMAP**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI和标准QCI映射/增加扩展QCI和标准QCI的映射关系（ADD EXTENDQCIMAP）_09653762.md)时使用[**SET EXTENQCIATTR**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI应用属性/设置应用扩展QCI参数属性（SET EXTENQCIATTR）_09653201.md)配置的缺省映射值。
      > **说明**
      > UNC 漫游用户使用扩展QCI接入时，还可以通过软参 [BIT_1858 控制漫游用户使用扩展QCI接入时UNC是否将扩展QCI按照本地配置映射成为标准的QCI处理](../../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（SMF_GGSN_SGW-C_PGW-C）/QoS管理/BIT_1858 控制漫游用户使用扩展QCI接入时UNC是否将扩展QCI按照本地配置映射成为标准的QCI处理_39462088.md) 。
      >
      > - 值为0时，按照本地配置映射成标准QCI处理。
      > - 值为1时，不按照本地配置映射。
    - 当开关关闭，则拒绝用户接入。

**图2** 场景2： UNC 从PCF获取动态PCC规则

    - 针对漫游用户，还需要开启[**SET EXTENQCIATTR**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI应用属性/设置应用扩展QCI参数属性（SET EXTENQCIATTR）_09653201.md)中的“QCICTRLFLAG（用户接入的扩展QCI控制开关）”。

            >
            > - 值为0时，按照本地配置映射成标准QCI处理。

          - 当开关关闭，则拒绝用户接入。

**图3** 场景3： UNC 收到MME创建会话的请求（Create Session Request）/更新缺省承载的请求（Modify Bearer Command）

<br>

![](扩展QCI_5QI应用_11073829.assets/zh-cn_image_0313664771_2.png)

1. QCI如果不在1~255之间，会被 UNC 认定为不合法，则通过 [**SET DEFEPSQOS**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/EPS QoS配置/全局EPS QoS纠错/设置EPS缺省QoS参数（SET DEFEPSQOS）_09653732.md) 进行纠错。
2. 根据QCI的类型进行判断：
    - 当QCI为标准QCI（即QCI值为1~9或在**[**ADD STDQOSID**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/标准QoS ID配置/增加标准QoS ID配置（ADD STDQOSID）_06399909.md)**配置范围内）时，如果QCI值为65/66/69/70时，需要开启[WSFD-102602 LTE一键通（适用于PGW-C/SGW-C）](../../../语音功能/WSFD-102600 Push to Talk（适用于PGW-C_SGW-C）/WSFD-102602 LTE一键通（适用于PGW-C_SGW-C）_10282603.md)功能，否则拒绝用户接入。
    - 当QCI为扩展QCI（即QCI值在**[**SET EXTENQCIATTR**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI应用属性/设置应用扩展QCI参数属性（SET EXTENQCIATTR）_09653201.md)**中**EXTENDQCIRANGE**配置的范围内且不在标准QCI范围内）时，需要打开本特性的License，未开启License则使用[**SET DEFEPSQOS**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/EPS QoS配置/全局EPS QoS纠错/设置EPS缺省QoS参数（SET DEFEPSQOS）_09653732.md)配置的QCI进行激活。
      > **说明**
      > SGW-C和PGW-C分离场景，可能存在PGW-C有扩展QCI的license但是SGW-C无扩展QCI的license的情况。此情况下SGW-C收到携带扩展QCI的专有承载创建请求时的处理受软参控制：
      >
      > [BIT844 控制SGW-C在无扩展QCI的license时，收到携带扩展QCI的专有承载激活请求是否允许激活成功](../../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（SMF_GGSN_SGW-C_PGW-C）/QoS管理/BIT844 控制SGW-C在无扩展QCI的license时，收到携带扩展QCI的专有承载激活请求是否允许激活成功_26951283.md) 。
      >
      > - 取值为0时，允许激活成功，使用标准QCI。
      > - 取值为1时，不允许激活。
3. 如果是本地用户，则使用该扩展QCI进行接入。针对漫游用户，还需要开启 [**SET EXTENQCIATTR**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI应用属性/设置应用扩展QCI参数属性（SET EXTENQCIATTR）_09653201.md) 中的“QCICTRLFLAG（用户接入的扩展QCI控制开关）”。

          - 当开关开启，则根据[**ADD EXTENDQCIMAP**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI和标准QCI映射/增加扩展QCI和标准QCI的映射关系（ADD EXTENDQCIMAP）_09653762.md)配置扩展QCI和标准QCI的映射关系，将扩展QCI映射为标准QCI进行接入，未配置[**ADD EXTENDQCIMAP**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI和标准QCI映射/增加扩展QCI和标准QCI的映射关系（ADD EXTENDQCIMAP）_09653762.md)时使用[**SET EXTENQCIATTR**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/QoS/扩展QCI功能/扩展QCI应用属性/设置应用扩展QCI参数属性（SET EXTENQCIATTR）_09653201.md)配置的缺省映射值。
            > **说明**
            > UNC 漫游用户使用扩展QCI接入时，还可以通过软参 [BIT_1858 控制漫游用户使用扩展QCI接入时UNC是否将扩展QCI按照本地配置映射成为标准的QCI处理](../../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（SMF_GGSN_SGW-C_PGW-C）/QoS管理/BIT_1858 控制漫游用户使用扩展QCI接入时UNC是否将扩展QCI按照本地配置映射成为标准的QCI处理_39462088.md) 。
            >
            > - 值为0时，按照本地配置映射成标准QCI处理。
            > - 值为1时，不按照本地配置映射。
          - 当开关关闭，则拒绝用户接入。
