#### [Iu接口控制面协议](#ZH-CN_CONCEPT_0185436077)

Iu接口控制面协议包括以下部分：

- RANAP
  RANAP协议（Radio Access Network Application Protocol）负责封装及承载高层信令，处理SGSN和UTRAN之间的信令，并负责管理Iu接口上的GTP连接。
- SCCP
  SCCP协议（Signalling Connection Control Part）提供两类服务：

  - 0类无连接服务，支持与特定UE无关的通用过程。
    - 2类有连接服务，支持与特定UE相关的信令过程。
- SIGTRAN
  SIGTRAN协议（Signaling Transport），即信令传输协议，它支持通过IP网络传输传统的电路交换网SCN（Switched Circuit Network）信令。M3UA和SCTP是SIGTRAN的一部分。
- M3UA
  M3UA是SS7 MTP3用户适配协议，它使用流控制传输协议（SCTP）通过IP传输SS7 MTP3层的用户信令消息（即SCCP消息），支持协议元素实现MTP3对等用户在SS7和IP域里的无缝操作。
  M3UA用于SGSN和STP/RNC之间的信令传输。在SIGTRAN协议栈中，M3UA运行在SCTP的上层，是SCTP用户，上层用户是MTP3层用户（SCCP），M3UA在SG端的上层用户则是NIF（Nodal Interworking Function）。
- SCTP
  流控制传输协议SCTP（Stream Control Transmission Protocol）是提供基于不可靠传输业务的协议（如IP）之上的可靠的数据报传输协议。
  SCTP的设计用于通过IP网传输SCN窄带信令消息。SCTP对TCP（Transfer Control Protocol）的缺陷进行了一些完善，使得信令传输具有更高的可靠性，SCTP的设计包括适当的拥塞控制、防止泛滥和伪装攻击、更优的实时性能和多归属性支持。

#### [Iu接口用户面协议](#ZH-CN_CONCEPT_0185436077)

Iu接口用户面协议包括以下部分：

- GTP-U
  GTP-U（GTP User Plane）协议利用隧道传输信令或用户数据。
- UDP
  UDP（User Datagram Protocol）协议用于提供无连接的数据包服务。
- IP
  IP（Internet Protocol）协议分为IPv4、IPv6。

#### [业务流程](#ZH-CN_CONCEPT_0185436077)

Iu接口的主要信令流程包括RAB指配、Iu信令连接、加密。

- Iu接口RAB指配流程（建立/释放）
  **图1** RAB指配流程（建立/释放）

  <br>

  ![](实现原理_85436077.assets/zh-cn_image_0000002523267478_2.png)
  Iu接口RAB指配流程包括建立和释放流程，如 [图1](#ZH-CN_CONCEPT_0185436077__zh-cn_concept_0130427556_fig04) 所示。
    1. SGSN向RNC发送RAB Assignment Request（Setup/Release）消息。
      RAB Assignment Request（Setup）消息中携带对于用户面资源的QoS要求、用户面地址/版本等信息。
      RAB Assignment Request（Release）消息中含有要释放的RAB ID及释放原因等信息。
    2. RNC向SGSN发送RAB Assignment Response（Setup/Release）响应消息。
- Iu信令连接建立/释放/直接传输流程
  **图2** Iu信令连接流程

  <br>

  ![](实现原理_85436077.assets/zh-cn_image_0000002523107484_2.png)
  Iu信令连接包括建立、释放和直接传输流程，如 [图2](#ZH-CN_CONCEPT_0185436077__zh-cn_concept_0130427556_fig05) 所示。
    1. RNC向SGSN发送Initial UE Message消息，建立Iu信令连接。
    2. SGSN和RNC之间发送双向Direct Transfer消息，进行信令消息传送。
    3. RNC向SGSN发送Iu Release Request消息。正常情况下都是由SGSN发起Iu释放请求，只有在接入侧发生异常情况时，RNC才发起Iu释放请求。
    4. SGSN向RNC下发Iu Release Command消息。
    5. Iu释放成功，RNC向SGSN发送Iu Release Complete消息。
- 加密流程
  **图3** 加密流程

  <br>

  ![](实现原理_85436077.assets/zh-cn_image_0000002523107482_2.png)
  加密流程用于SGSN向RNC传送进行加密和一致性保护的信息，如 [图3](#ZH-CN_CONCEPT_0185436077__zh-cn_concept_0130427556_fig06) 所示。加密流程可应用于位置更新、寻呼、迁移等各种面向连接的业务流程。
    1. SGSN向RNC发送Security Mode Command消息。
    2. 加密成功，则RNC向SGSN发送Security Mode Complete响应消息。
      若RNC返回的是Security Mode Reject响应消息，则SGSN需要终止当前的业务处理流程，发起Iu释放请求。

#### [支持Iu接口全系统资源共享](#ZH-CN_CONCEPT_0185436077)

支持Iu接口全系统资源共享是指每个RNC控制的用户可以在全系统的Iu接口处理子系统中进行负荷分担，不会由于一块物理板容量处理能力不足而导致系统性的容量处理能力下降。

- 资源共享
  Iu接口资源共享主要是RANAP层相关用户上下文资源的共享，使用负荷分担方式，让全系统的资源得到充分利用。
- 负荷分担
  负荷分担是一种设备运行方式，指两个或多个硬件单元在正常工作时，按照自己的处理能力，均匀的负担相同功能的处理。当其中一个硬件单元出现故障时，在保证一定性能指标（如呼损）的前提下，由其他硬件单元分担完成故障硬件单元的任务。
- 实现流程
  **图4** Iu全系统负荷分担

  <br>

  ![](实现原理_85436077.assets/zh-cn_image_0000002523267476_2.png)
  Iu接口的全系统负荷分担如 [图4](#ZH-CN_CONCEPT_0185436077__zh-cn_concept_0130427556_fig11) 所示，具体过程如下：
    1. 当有新用户发起接入请求时，Iu接口链路所在的SGP-1进程上的SCCP层通过轮选方式选择SGSN内其他SGP进程，在被选中的SGP-2进程上建立SCCP层连接和RANAP层连接，实现上行信令的传输。
    2. Iu接口的下行消息在Iu连接所在的SGP-2进程完成RANAP层处理后，发送至本进程的SCCP层。SCCP完成本层协议处理后，根据连接选择掩码选择到对应RNC的一条链路，判断链路所在的SGP进程，将下行消息转发至该SGP进程。如 [图4](#ZH-CN_CONCEPT_0185436077__zh-cn_concept_0130427556_fig11) 所示，Iu接口的下行消息转发给SGP-1进程的M3UA层处理。
  如果SGSN内所有SGP进程的用户数都已经达到拥塞门限，则整系统已经达到容量极限，SGSN将新的附着请求丢弃。
  SGP进程有进程CPU过载告警，告警阈值可以在后台设定；如果出现预警，则需要扩容或者优化组网方案。

SGSN内Iu接口信令面由SGP进程实现。当用户从Iu接口接入SGSN时，SGSN按照负荷分担的方式在整个系统中均匀选择SGP进程处理。
