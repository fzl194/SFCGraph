# 调测基于APN的信令拥塞控制

- [操作场景](#ZH-CN_OPI_0188625071__1.3.1)
- [必备事项](#ZH-CN_OPI_0188625071__1.3.2)
- [操作步骤](#ZH-CN_OPI_0188625071__1.3.3)

## [操作场景](#ZH-CN_OPI_0188625071)

在M2M终端反复发起附着请求，产生大量异常信令的场景下， UNC 通过配置APN信令拥塞控制功能减少网络的信令负荷。本操作介绍了通过调测手段检查 UNC 是否正确配置该功能的过程。

> **说明**
> 适用于 SGSN、 MME。

## [必备事项](#ZH-CN_OPI_0188625071)

前提条件

- 请仔细阅读[WSFD-206003 基于APN的信令拥塞控制特性概述](特性概述_88625086.md)
- 完成[激活基于APN的信令拥塞控制](激活基于APN的信令拥塞控制_88625063.md)。

数据

无。

## [操作步骤](#ZH-CN_OPI_0188625071)

- 进入 “MML命令行-UNC” 窗口。
- 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) 命令查询License配置开关是否已打开。
    - 如果“SWITCH”为“ENABLE”，请执行[3](#ZH-CN_OPI_0188625071__cmd1430237208180641)。
    - 如果“SWITCH”为“DISABLE”，请执行[**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)命令打开本特性对应的License配置开关。
- 检查APN拥塞控制功能配置。
  进入 “MML命令行-UNC” 窗口。
  [**LST APNCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/基于APN的MM拥塞控制_流控_寻呼优化配置/查询APN控制参数配置(LST APNCTRL)_72345069.md)
  ```
  %%LST APNCTRL: SUBSCRIBEDAPN="HUAWEI0.COM";%%
  RETCODE = 0  操作成功。

  操作结果如下
  ------------
                            签约APN  =  HUAWEI0.COM
                            APN类型  =  拥塞控制APN
                          APN优先级  =  低
              Backoff Timer分配开关  =  关
                     附着拒绝原因值  =  拥塞
  识别异常附着行为的门限（次/小时）  =  30
        Ready Timer定时器时长（秒）  =  NULL
              T3324定时器时长（秒）  =  NULL
              T3312定时器时长（分）  =  NULL
  (结果个数 = 1)
  ---    END
  ```
    - 预期结果：各项参数显示结果与配置结果一致。
    - 异常结果：各项参数显示结果与配置结果不一致或查看不到相关配置信息。
      参考 [激活基于APN的信令拥塞控制](激活基于APN的信令拥塞控制_88625063.md) 并重新配置。
- 调测APN拥塞控制业务。
    1. 创建用户跟踪。
      参数选择如下：
          - IMSI:被跟踪用户的IMSI。
          - 其它参数：选择默认值。
    2. 用户频繁发起附着请求，达到 [**ADD APNCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/基于APN的MM拥塞控制_流控_寻呼优化配置/增加APN控制参数配置(ADD APNCTRL)_26145470.md) 命令中 “ATTACHTHRESH” 参数设置的门限值。
    3. 查看跟踪结果。
          - 预期结果：用户附着拒绝，Attach Reject消息中携带MM backoff timer（即t3346）。
          - 异常结果：Attach Reject消息中未携带MM backoff timer。
            参考 [激活基于APN的信令拥塞控制](激活基于APN的信令拥塞控制_88625063.md) 并重新配置。
            > **说明**
            > 若Backoff timer开关未打开，则不会下发MM backoff timer。
