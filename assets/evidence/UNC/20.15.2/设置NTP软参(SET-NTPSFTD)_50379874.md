#### [命令功能](#ZH-CN_TOPIC_0000001150379874)

![](设置NTP软参(SET NTPSFTD)_50379874.assets/notice_3.0-zh-cn_2.png)

NTP软参配置属于系统关键配置，修改可能会影响系统时间的准确性，请谨慎操作。

该命令仅在 Full-stack 虚机场景下支持。

该命令为专家维护命令，用于设置NTP软参值。

该命令仅适用于开局阶段调测或者维护的操作场景，当需要设置NTP软参配置信息时，操作员可以使用该命令实现这一目的。

#### [注意事项](#ZH-CN_TOPIC_0000001150379874)

- 设置手工同步策略为优先从CSP侧外部时钟源同步时间，该设置会在48小时后自动恢复为优先从paas管理面外部时钟源同步时间。

- 当进行“**NTP同步策略**”软参设置（NM=4，CTRL=1）时：
    - 在设置PM0为1时，需要先登录paas管理面portal，确认不存在告警原因为“所配置的NTP服务器不可用或者key认证失败”的“ALM-135901 NTP连接状态异常”活动告警。
    - 在设置PM0为2时，需要先登录OM portal，确认不存在告警原因为“所配置的NTP服务器不可用或者key认证失败”的“ALM-135901 NTP连接状态异常”活动告警。
    - 在设置PM0为3时，需要先登录paas管理面portal，确认不存在告警原因为“所配置的NTP服务器不可用或者key认证失败”的“ALM-135901 NTP连接状态异常”活动告警。
- 该命令在进行“**NTP同步策略**”软参设置时，存在系统初始记录，参数的初始值如下：
  | 手工同步策略 | 自动同步策略 |
  | --- | --- |
  | priority_get_time_from_fst | auto_get_time_from_fst |

#### [使用实例](#ZH-CN_TOPIC_0000001150379874)

1. 设置时间同步策略，设置手工同步策略为优先从paas管理面外部时钟源同步时间。
  ```
  %%SET NTPSFTD: NM=4, CTRL=1, PM0=1;%%
  RETCODE = 0  操作成功

  ---    END
  ```
2. 设置时间同步策略，设置手工同步策略为优先从CSP侧外部时钟源同步时间。
  ```
  %%SET NTPSFTD: NM=4, CTRL=1, PM0=2;%%
  RETCODE = 0  操作成功

  ---    END
  ```
3. 设置时间同步策略，设置手工同步策略强制从paas管理面外部时钟源同步时间。
  ```
  %%SET NTPSFTD: NM=4, CTRL=1, PM0=3;%%
  RETCODE = 0  操作成功

  ---    END
  ```
