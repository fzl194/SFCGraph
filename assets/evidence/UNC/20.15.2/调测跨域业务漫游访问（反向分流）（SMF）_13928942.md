# 调测跨域业务漫游访问（反向分流）（SMF）

## [操作场景](#ZH-CN_OPI_0000001213928942)

本操作指导介绍在运行网络中调测基于反向分流的跨域业务漫游访问特性的操作过程。

## [必备事项](#ZH-CN_OPI_0000001213928942)

前提条件

- 请仔细阅读[WSFD-228001 跨域业务访问特性概述](特性概述_55914877.md)特性原理概述。
- 完成[部署跨域业务漫游访问（正向分流）（SMF）](部署跨域业务漫游访问（正向分流）（SMF）_11634940.md)。

数据

| 类别 | 参数名称 | 取值样例 | 说明 |
| --- | --- | --- | --- |
| 用户信息 | 国际移动用户识别码（IMSI） | 123000123456789 | - |
| 用户签约的DNN信息 | DNN名称（DNN） | Huawei.com | 为行业签约的专用DNN |
| DNAI | 数据网络访问标识符（DNAI） | testdnai | 位于中心侧辅锚点UPF的DNAI |

## [操作步骤](#ZH-CN_OPI_0000001213928942)

1. 用户进入MEC覆盖区域，发起PDU会话建立流程，创建PDU会话标识为5的PDU会话。
2. 创建用户跟踪
3. 用户发起PDU会话建立流程。
4. 查看用户跟踪。
    - SMF收到来自AMF的PDU Session Establishment Request消息，该消息中携带了AMF纠正后的用户签约的专用DNN信息“Huawei.com”。
    - SMF基于专用DNN “Huawei.com”选择园区主锚点和UL CL合设的UPF，以及位于中心侧的辅锚点UPF。
    - SMF发送向位于园区UL CL UPF（和主锚点UPF合设）发送PFCP Session Establishment Request消息，创建N4分流会话，携带分流策略（application ID）。
    - SMF发送向位于中心侧的辅锚点UPF发送PFCP Session Establishment Request消息，创建辅锚点N4会话，园区辅锚点UPF的DNAI “testdnai”。
    - UL CL UPF（和主锚点UPF合设）返回的PFCP Session Establishment Response消息中携带分配给RAN侧的上行隧道信息。
