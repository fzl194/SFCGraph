#### [命令功能](#ZH-CN_CONCEPT_0259103844)

该命令用于显示网元上跟踪任务信息。

#### [注意事项](#ZH-CN_CONCEPT_0259103844)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0259103844)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0259103844)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| TRACETASKID | 跟踪任务号 | 可选必选说明：可选参数<br>参数含义：跟踪任务ID。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～4294967294。<br>默认值：无 |
| SERVICEINSTANCE | 服务实例 | 可选必选说明：必选参数<br>参数含义：该参数表示大颗粒服务实例名称。<br>数据来源：本端规划<br>取值范围：字符串类型，通过LST VNFC命令获取。<br>默认值：无<br>配置原则：只能填写通过LST VNFC命令查询到的管理代理标识。 |

#### [使用实例](#ZH-CN_CONCEPT_0259103844)

显示跟踪任务号为1的跟踪任务相关信息，可通过如下命令显示：

```
DSP TRCTASK: TRACETASKID=1
,SERVICEINSTANCE="vnfc"
;
```

#### [输出结果说明](#ZH-CN_CONCEPT_0259103844)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 跟踪任务号 | 跟踪任务ID。 |
| 跟踪业务类型 | 跟踪业务类型。<br>- 0x00000000~0x0000FFFF：跟踪，从235～254留给全网跟踪用，其他产品自定义，512~2047留给VNF跟踪（比如染色跟踪）。<br>- 0x00000000~0x0001FFFF：监控，0x00010000~0x00010100公共监控，其他产品自定义，要求和跟踪不重复。<br>- 其他保留。 |
| 上报方式 | 上报方式，分为：<br>- 消息上报(Report Bin Message)：通过notification通知直接上报trace消息给网管。<br>- 文件上报(Record Trace File)：在网元本地生成跟踪文件，并上报文件生成通知发送给对应的网管。 |
| 生成的文件大小（字节） | 生成的文件大小，单位是字节，默认值2097152。 |
| 生成文件周期（秒） | 生成文件周期，单位是秒，默认值60。 |
| 握手周期（秒） | 握手周期，网管定期下发握手操作，网元三个周期内没有收到该消息删除跟踪， 单位是秒，默认值300。 |
| 句柄 | handle，跟踪创建时的handle，网元无需理解，但需要在上报消息中原样返回，如何使用由网管设计。 |
| 跟踪任务运行状态 | 跟踪任务运行状态（正常、流控、异常、删除）。 |
| Notification编号 | 最新跟踪通知序号，网管可通过此序号判断通知消息是否丢失。 |
| 跟踪ID | 跟踪ID，用于全网跟踪或者VNF跟踪，默认值4294967295。 |
| 连接句柄 | 连接句柄 ，同订阅时的connHandle，用于关联VNF跟踪上报。 |
