# 调测 基于信令面的数据传输 （SGW-C）

- [操作场景](#ZH-CN_OPI_0277260997__1.3.1)
- [必备事项](#ZH-CN_OPI_0277260997__1.3.2)
- [操作步骤](#ZH-CN_OPI_0277260997__1.3.3)

## [操作场景](#ZH-CN_OPI_0277260997)

当运营商部署 基于信令面的数据传输 功能时，需对该功能进行调测，确保该特性正常生效。

> **说明**
> 适用于SGW-C。

## [必备事项](#ZH-CN_OPI_0277260997)

前提条件

- 请仔细阅读[WSFD-215101 基于信令面的数据传输特性概述（SGW-C）](特性概述_77260995.md)。
- 完成[激活基于信令面的数据传输（SGW-C）](激活基于信令面的数据传输（SGW-C）_77260996.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 123000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN | test | 已配置数据中获取 | 取自<br>[激活基于信令面的数据传输（SGW-C）](激活基于信令面的数据传输（SGW-C）_77260996.md)<br>中配置的APN实例名。 |

工具

- 测试终端
- OM Portal跟踪

## [操作步骤](#ZH-CN_OPI_0277260997)

1. 进入 “MML命令行-UNC” 窗口。
2. 查询License中是否允许使用 基于信令面的数据传输 功能。
  [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) : LICITEM="LKV3WPSDTO01";
    - 如果“SWITCH”为“ENABLE”，请执行[3](#ZH-CN_OPI_0277260997__cmd1831301046184705)。
    - 如果“SWITCH”为“DISABLE”，则执行 [**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md) 命令打开本特性对应的License配置开关。
3. 在OM Portal上 [创建用户跟踪任务](../../../../../../网络运维/日常维护/UNC基础运维操作/创建消息跟踪/创建用户跟踪/用户跟踪_72467812.md) ，在“参数配置”栏输入用户IMSI，在“消息类型”栏选择GTPC、UP DATA和DOWN DATA消息类型。
4. 激活用户，并跟踪该用户消息。
5. 查看用户跟踪消息，打开MME发送给S-GW的Create Session Request消息，查看消息的“f-teid”信元是否携带“s11-mme-gtpu-ingterface”，如 [图1](#ZH-CN_OPI_0277260997__fig1) 所示。
  **图1** Create Session Request消息

  <br>

  ![](调测基于信令面的数据传输（SGW-C）_77260997.assets/zh-cn_image_0000002119473745_2.png)
    - 如果“f-teid”信元携带“s11-mme-gtpu-ingterface”，则请执行 [6](#ZH-CN_OPI_0277260997__cmd1618355034184705) 。
    - 如果“f-teid”信元没有携带“s11-mme-gtpu-ingterface”，请执行 [7](#ZH-CN_OPI_0277260997__cmd1951538945184705) 。
6. 查看用户跟踪消息，打开S-GW发送给MME的Create Session Response消息，查看消息的“f-teid”信元是否携带“s11-sgw-gtpu-ingterface”，如 [图2](#ZH-CN_OPI_0277260997__fig2) 所示。
  **图2** Create Session Response消息

  <br>

  ![](调测基于信令面的数据传输（SGW-C）_77260997.assets/zh-cn_image_0000002119393761_2.png)
    - 如果“f-teid”信元携带“s11-sgw-gtpu-ingterface”，则调测完成。
    - 如果“f-teid”信元没有携带“s11-sgw-gtpu-ingterface”，请执行 [7](#ZH-CN_OPI_0277260997__cmd1951538945184705) 。
7. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务 ，执行 [4](#ZH-CN_OPI_0277260997__step3) 并保存报文。
    b. 执行 **[EXP MML](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md)** 命令将当前配置数据导出为MML脚本文件并保存。
    c. 查看并收集对端设备配置及接口状态信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
