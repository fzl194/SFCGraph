## [操作场景](#ZH-CN_OPI_0185510498)

本操作指导介绍在运行网络中激活多信令点功能的操作过程。

多信令点功能，即在同一个SGSN上可以配置多个7号信令源信令点，使一个物理实体可以虚拟为多个逻辑信令点，每个逻辑信令点都可以作为独立信令点与其他信令点相连，通过上层程序对多个源信令点进行统一的管理和业务的负荷分担。

> **说明**
> 适用于 SGSN 。

## [任务示例](#ZH-CN_OPI_0185510498)

任务描述

在 UNC 上配置两个本局源信令点，两个本局信令点在国内网的编码分别为1234和5678，本局信令点和目的信令点之间使用M3UA承载信令。

脚本

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2MUSP01", SWITCH=ENABLE;
```

//添加两个本局源信令点。

```
ADD M3LE: LEX=1, NI=NAT, OPC="1234", LET=IPSP, LEN="to SS7 1";
ADD M3LE: LEX=2, NI=NAT, OPC="5678", LET=IPSP, LEN="to SS7 2";
```

//增加SCCP本局信令点。

```
ADD SCCPOPC: OPX=1, NI=NAT, OPC="1234", SPN=COREONLY, SGSNN="8613912345";
ADD SCCPOPC: OPX=2, NI=NAT, OPC="5678", SPN=COREONLY, SGSNN="8613912346";
```

//增加SCCP本局子系统。

```
ADD SCCPSSN: SSNX=0, SSN=SCMG, NI=NAT, DPC="1234", OPC="1234";
ADD SCCPSSN: SSNX=1, SSN=SGSN, NI=NAT, DPC="1234", OPC="1234";
ADD SCCPSSN: SSNX=2, SSN=CAP, NI=NAT, DPC="5678", OPC="5678";
ADD SCCPSSN: SSNX=3, SSN=BSSAP+, NI=NAT, DPC="5678", OPC="5678";
```

//增加本局GT翻译数据。

```
ADD SCCPGT: GTX=0, NI=NAT, RT=DPC, GTEXP=FOUR, NP=ISDNP, ADDREXP=INTDN, ADDR="8613912345", DPC="1234";
ADD SCCPGT: GTX=1, NI=NAT, RT=DPC, GTEXP=FOUR, NP=ISDNP, ADDREXP=INTDN, ADDR="8613912346", DPC="5678";
```
