# PDU会话创建使能专网业务流程

**图1** PDU会话创建使能专网业务UL CL流程

<br>

![](PDU会话创建使能专网业务流程_85233681.assets/zh-cn_image_0000001139790762_2.png)

1. UE发起PDU会话创建流程，携带TAI、SUPI以及Requesteddnn信息。
2. AMF根据用户信息及本地配置（**ADD PCFSELPLCY**命令中的“SERVINGSCOPE”参数）就近选择AM-PCF。
3. AMF向AM-PCF发送Npcf_AMPolicyControl_Create Request消息，携带用户TAI位置信息以及SUPI等信息。AM-PCF根据用户对专网业务的签约状态，以及用户当前所处位置的TAI信息决策是否下发专网DNN。
    - 若条件满足则将标识专网业务预定义UL CL分流规则的UserProfileName以selectedDnn（"smfSelInfo.dnn" = "dnn-ar"）的形式通过Npcf_AMPolicyControl_Create Response消息下发给AMF。
    - 若条件不满足则仅向AMF下发普通的AM策略（LOC_CH trigger），不做其他特殊处理。该场景不是本特性的核心功能点，下面不再展开介绍。
4. AMF从Npcf_AMPolicyControl_Create response消息中解析出AM-PCF指定的selectedDnn（dnn-ar）后匹配本地配置（**ADD ALLOWDNN**）的预设白名单。
    - 若匹配，则在向SMF发起N11会话创建请求中携带selectedDnn（dnn-ar）。
    - 若不匹配，则不携带selectedDnn（dnn-ar）。该场景不是本特性的核心功能点，下面不再展开介绍。
5. AMF通过Nsmf_PDUSession_CreateSMContext request消息将UE激活请求带上来的Requesteddnn和AM-PCF指定的selectedDnn（dnn-ar）携带给SMF。
  > **说明**
  > SMF从AMF收到selectedDnn（dnn-ar）后会基于本地配置（ **ADD APN** 中的 “TRAFFICDIST” ）以及License判定，如果匹配则继续会话创建流程，否则直接拒绝流程。
6. 会话注册。
7. SMF选择SM-PCF。
8. SMF通过Npcf_SMPolicyControl_Create Request消息向SM-PCF发起N7会话创建请求。这时SM-PCF通过Response消息下发普通PCC策略给SMF。分流策略SMF则将从Nsmf_PDUSession_CreateSMContext request消息中解析出的selectedDnn（dnn-ar）作为Userprofile名称来匹配并提取本地配置的ULCL专网分流规则。
9. SMF给AMF返回Nsmf_PDUSession_CreateSMContext response消息。
10. SMF基于用户TAI、Requesteddnn等信息选择PSA0 UPF，基于用户TAI、Requesteddnn、DNAI等选择UL CL UPF+PSA1 UPF。
11. SMF向PSA0发送PFCP Session创建流程，并携带N3下行隧道信息以及SM-PCF下发的普通PCC策略，PSA0 UPF返回N3上行端隧道信息。
12. SMF向UL CL UPF+PSA1 UPF发送PFCP Session创建流程，携带基于selectedDnn（dnn-ar）匹配的本地分流规则。UL CL UPF+PSA1在回应的Response消息中携带分配的N3上行隧道信息、UL CL UPF的N9下行隧道信息以及PSA1 UPF的N9隧道上行信息。
  > **说明**
  > 专网业务场景下辅锚点不做计费，因此SMF仅仅下发分流规则给UL CL UPF+PSA1 UPF，不下发计费相关的策略。
13. SMF向PSA0发送PFCP Session Modification Request消息，将其下行隧道端更新至N9接口。PSA0 UPF返回N9接口上行端隧道信息。
14. SMF向“UL CL UPF+PSA1”发送PFCP Session Modification Request消息，携带PSA0 UPF返回N9接口上行端隧道信息，将其上行隧道端更新至N9接口。
15. SMF发送Namf_Communication_N1N2MessageTransfer Request消息，携带N2 SM Information通知(R)AN侧，将基站侧N3隧道上行隧道信息更新为UL CL UPF。
16. 16~18，AMF通过(R)AN和UE完成信令交互发送PDU Session Resource Setup Request，请求中包含N2 SM Information，包含QFI、QoS Profile、CN Tunnel Info，发送给UE的N1 SM Container，包含PDU Session Establishment Accept、Allocated IPv4 Address。
  (R)AN向AMF发送PDU Session Resource Setup Response，建立AN隧道信息，响应消息中包含AN Tunnel Info、 List of Accepted/Rejected QFI等。
17. 19，SMF和AMF交互获取AN Tunnel Info等信息。
18. 20a~20b步，SMF向UL CL UPF+PSA1发送PFCP Session Modification Request，将(R)AN侧N3隧道下行端地址信息以及相应的PDR发送给UL CL UPF+PSA1，UL CL UPF+PSA1回应PFCP Session Modification Response。
