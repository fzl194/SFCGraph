## [操作场景](#ZH-CN_OPI_0163535130)

本操作指导介绍在运行网络中激活基于 位置区域 选择 网关 特性的操作过程。

基于 位置区域 选择 网关 ，是指 UNC 在使用 APN (Access Point Name)进行 DNS (Domain Name Service)解析获取网关的IP地址前，首先通过域名定制方法在原APN中加入用户位置信息(LAI/RAI/TAI)，然后向DNS服务器发送定制域名，获取GGSN/P-GW的IP地址，实现基于用户位置信息的网关选择。

> **说明**
> 适用于 SGSN、 MME。

## [对系统的影响](#ZH-CN_OPI_0163535130)

该操作对系统正常运行没有影响。

## [必备事项](#ZH-CN_OPI_0163535130)

前提条件

- 请仔细阅读[WSFD-205003 基于位置区域选择网关特性概述](特性概述_70166782.md)。
- DNS必须配置添加了用户位置信息的APN所对应的网关的IP地址，以便解析扩展后的域名。
  > **说明**
  > 用户也可以选择自行在 UNC 上本地配置添加了用户位置信息的定制域名所对应的网关的IP地址（见 [操作步骤2](#ZH-CN_OPI_0163535130__step3) ），若两者都没配置将导致基于位置信息选择网关失败。
- 完成加载License。

## [操作步骤](#ZH-CN_OPI_0163535130)

1. 进入 “MML命令行-UNC” 窗口。
2. 配置基于位置区域选择网关参数。确认是否使用基于LAI/RAI/TAI集合的APN定制方式。
  [**ADD GWSELPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md)
  > **说明**
  > - 参数“LABEL”（定制标识类型）选择“RAI”。
  > - 参数“SLCTM”（选择方法）选择“CUSTOMIZE_APN-OI_REPLACEMENT”。
  > - 在不同的位置区内存在多个GGSN/P-GW，且其中既有支持预付费的也有不支持预付费的，需选择与UE在同一位置区且支持预付费的GGSN/P-GW的场景下，启用基于位置区和计费属性选择网关，“label”选择“CC_RAI”。
  > - 如果需要配置基于RAI/TAI的域名定制，本步骤中参数“LOCINFO”需要选择为“RAI/TAI”；如果需要配置基于RAI/TAI集合的域名定制，需要配置[步骤 4](#ZH-CN_OPI_0163535130__step2)且本步骤中参数“LOCINFO”需要选择为“RAI/TAI范围”。
3. **可选：**本地配置基于位置信息(LAI/RAI/TAI)的定制域名与网关的对应关系，如果运营商在DNS Server上已配置该对应关系，则不需要配置本步骤。
    - 2/3G场景
      增加Hostfile的A解析记录，配置基于位置信息(LAI/RAI/TAI)的定制域名所对应的网关IP地址。
      > **说明**
      > 2/3G场景下，定制域名的位置信息(LAI/RAI/TAI)需要在命令 [**ADD IPV4DNSH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS Hostfile管理/增加IPV4 DNS Hostfile记录(ADD IPV4DNSH)_26145884.md) 的参数 “HOSTNAME” 中体现。
      [**ADD IPV4DNSH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS Hostfile管理/增加IPV4 DNS Hostfile记录(ADD IPV4DNSH)_26145884.md)
    - 4G场景
      > **说明**
      > 4G场景下，定制域名的位置信息(LAI/RAI/TAI)需要在命令 [**ADD DNSN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS NAPTR管理/增加DNS NAPTR记录(ADD DNSN)_72225569.md) 的参数 “FQDN” 中体现。
          a. 增加Hostfile的A解析记录。
            [**ADD IPV4DNSH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS Hostfile管理/增加IPV4 DNS Hostfile记录(ADD IPV4DNSH)_26145884.md)
          b. 增加Hostfile的NAPTR解析记录，配置基于位置信息(LAI/RAI/TAI)的定制域名与网关的对应关系。
            [**ADD DNSN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS NAPTR管理/增加DNS NAPTR记录(ADD DNSN)_72225569.md)
4. **可选：**增加基于LAI/RAI/TAI集合的映射。
  > **说明**
  > 如果 [2](#ZH-CN_OPI_0163535130__cmd1431525042180641) 配置 “LOCINFO” 为 “RAI/TAI范围” ，需要增加基于LAI/RAI/TAI集合的域名定制参数，请执行本步骤。
    - 开启基于LAI/RAI/TAI集合（映射到起始LAI/RAI/TAI）的域名定制方式
      [**ADD AREADNS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-SGSN_S10_S16_S3接口管理/位置域名管理/增加位置区域DNS域名(ADD AREADNS)_72345559.md)
    - 开启基于LAI/RAI/TAI集合（映射为区域名称ZONE）的域名定制方式
      当 [**ADD AREADNS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-SGSN_S10_S16_S3接口管理/位置域名管理/增加位置区域DNS域名(ADD AREADNS)_72345559.md) “域名类型” 配置为 “GGSN/PGW_RAI(根据路由区(RAI)查找GGSN/PGW)” 或 “GGSN/PGW_LAI(根据位置区(LAI)查找GGSN/PGW)” 、 “定制区域标识” 配置为 “YES(是)” 时，SGSN/MME会启用 LAI/RAI/TAI集合（映射为区域名称ZONE） 的域名定制方式。设置后可根据组合命名ZONE构造域名。
      [**ADD AREADNS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-SGSN_S10_S16_S3接口管理/位置域名管理/增加位置区域DNS域名(ADD AREADNS)_72345559.md)
5. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)

## [任务示例](#ZH-CN_OPI_0163535130)

任务描述

- 任务1：
  UNC 作为SGSN，配置基于RAI集合(映射到起始RAI)的位置信息定制域名的网关选择，其中LAC为"0x0001"，起始RAC为"0x01"，RACRANGE为"0x22"，APNNI为"HUAWEI132.com"。
- 任务2：
  UNC 作为SGSN，配置基于RAI集合(映射为区域名称ZONE)的位置信息定制域名的网关选择，其中LAC为"0x0001"，起始RAC为"0x01"，RACRANGE为"0x22"，ZONENAME为"ZONE1111"，APNNI为"HUAWEI132.com"。
- 任务3：
  UNC 作为SGSN，配置基于LAI集合(映射到起始LAI)的位置信息定制域名的网关选择，其中起始LAC为"0x0001"，LACRANGE为"0x0022"，APNNI为"HUAWEI132.com"。
- 任务4：
  UNC 作为SGSN，配置基于LAI集合(映射为区域名称ZONE)的位置信息定制域名的网关选择，其中起始LAC为"0x0001"，LACRANGE为"0x0022"，ZONENAME为"ZONE1111"，APNNI为"HUAWEI132.com"。
- 任务5：
  UNC 作为MME，配置基于TAI集合(映射到起始TAI)的位置信息定制域名的网关选择，其中起始TAC为"0x0000"，TACRANGE为"0x0022"，APNNI为"HUAWEI1.com"。

脚本

- 任务1
  //增加基于位置区域选择网关的策略。
  ```
  ADD GWSELPLCY: SUBRANGE=ALL_USER, APNNI="HUAWEI132.com", SLCTM=CUSTOMIZE_APN-OI_REPLACEMENT, LABEL=RAI, RAT=GERAN-1&UTRAN-1, LOCINFO=RAI/TAI_RANGE;
  ```
  //增加Hostfile的A解析记录，配置基于RAI集合(映射到起始RAI)的定制域名所对应的网关IP地址。
  ```
  ADD IPV4DNSH: HSINDEX=1, HOSTNAME="RAC0001.LAC0001.HUAWEI132.COM.MNC003.MCC123.GPRS", ADDRSECTION=SECTION1, IPV4ADDR1="10.2.3.10";
  ```
  //开启基于RAI集合(映射到起始RAI)的域名定制方式。
  ```
  ADD AREADNS: DNTYPE=GGSN/PGW_RAI, LAC="0x0001", RAC="0x01", RACRANGE="0x22", ZONESW=NO;
  ```
- 任务2
  //增加基于位置区域选择网关的策略。
  ```
  ADD GWSELPLCY: SUBRANGE=ALL_USER, APNNI="HUAWEI132.com", SLCTM=CUSTOMIZE_APN-OI_REPLACEMENT, LABEL=RAI, RAT=GERAN-1&UTRAN-1, LOCINFO=RAI/TAI_RANGE;
  ```
  //增加Hostfile的A解析记录，配置基于RAI集合(映射为区域名称ZONE)的定制域名所对应的网关IP地址。
  ```
  ADD IPV4DNSH:  HSINDEX=2, HOSTNAME="HUAWEI132.ZONE1111.MNC003.MCC123.GPRS", ADDRSECTION=SECTION1, IPV4ADDR1="10.2.3.11";
  ```
  //开启配置基于RAI集合(映射为区域名称ZONE)的域名定制方式。

  ```
  ADD AREADNS: DNTYPE=GGSN/PGW_RAI, LAC="0x0001", RAC="0x01", RACRANGE="0x22", ZONESW=YES, ZONENAME="ZONE1111";
  ```
- 任务3
  //增加基于位置区域选择网关的策略。
  ```
  ADD GWSELPLCY: SUBRANGE=ALL_USER, APNNI="HUAWEI132.com", SLCTM=CUSTOMIZE_APN-OI_REPLACEMENT, LABEL=RAI, RAT=GERAN-1&UTRAN-1, LOCINFO=RAI/TAI_RANGE;
  ```
  //增加Hostfile的A解析记录，配置基于LAI集合(映射到起始LAI)的定制域名所对应的网关IP地址。
