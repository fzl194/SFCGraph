#### [适用NF](#ZH-CN_TOPIC_0174013201)

GGSN、 SGW-C、PGW-C、SMF

#### [定义](#ZH-CN_TOPIC_0174013201)

基于业务流量的计费是在业务识别解析的基础上，按照访问业务的流量进行计费。

#### [客户价值](#ZH-CN_TOPIC_0174013201)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 可以开展基于流量的计费业务，和基于时长的计费策略共同构成灵活多样的计费方案。 |
| 用户 | - 提供基于流量的计费，满足用户多样性的需求。<br>- 相较于基于用户的流量计费，基于业务的流量计费可以提供更详细的业务消费信息。 |

#### [应用场景](#ZH-CN_TOPIC_0174013201)

当运营商希望区分用户数据流中的内容所属的业务类型，从而配以不同的资费标准，并按用户使用业务的流量向用户收取不同的资费时，可部署本特性。

#### [可获得性](#ZH-CN_TOPIC_0174013201)

**涉及网元**

| 涉及网元 | 支持版本 | 功能说明 |
| --- | --- | --- |
| GGSN-C/SGW-C/PGW-C | UNC 20.3.0及后续版本 | 将业务访问流量上报给CG或OCS。 |
| SMF | UNC 20.0.0及后续版本 | 将业务访问流量上报给CHF。 |
| GGSN-U/SGW-U/PGW-U | UDG 20.0.0及后续版本 | - 通过业务感知功能识别用户业务类型。<br>- 统计不同业务的访问流量，并上报给GGSN-C/SGW-C/PGW-C。 |
| UPF | UDG 20.0.0及后续版本 | - 对用户业务进行识别和解析。<br>- 统计不同业务的访问流量，并上报SMF。 |
| OCS | 无特殊要求 | 对在线计费用户进行鉴权、下发流量配额，基于业务类型对用户进行信用控制。 |
| CG | 无特殊要求 | 合并用户离线计费话单，发送给计费中心，最终生成基于业务流量的计费话单。 |
| CHF | 无特殊要求 | - 下发流量配额，基于业务类型对用户进行配额管理。<br>- 生成基于业务流量的计费话单。 |

#### [原理概述](#ZH-CN_TOPIC_0174013201)

基于业务流量的计费是在内容计费的基础上，对识别出的特定业务，按照业务的使用流量进行计费。业务使用流量统计由GGSN-U/SGW-U/PGW-U/UPF完成，支持统计业务粒度的上下行流量，如 [图1](#ZH-CN_TOPIC_0174013201__fig1) 所示。

**图1** 流量统计示意图

![](特性概述_74013201.assets/zh-cn_image_0179735197_2.png)

基于业务流量的计费的基础功能是离线计费、在线计费和融合计费。

- 离线计费：离线计费通过话单将用户计费信息上报给计费系统BS，话单中默认会记录用户级流量，是否根据流量向终端用户收费由计费中心决定。有关离线计费系统的详细描述请参考[WSFD-011201 支持离线计费](../WSFD-011201 支持离线计费_25745813.md)。
- 在线计费：在线计费通过信用控制会话实时扣费，当OCS在CCA消息中下发流量配额即可进行流量计费，是否根据流量向终端用户收费由OCS决定。有关在线计费系统的详细描述请参考[WSFD-109001 Gy/Diameter在线计费](../WSFD-109001 Gy_Diameter在线计费_27636706.md)。
- 融合计费：
    - 带配额管理（在线计费）：SMF根据CHF下发的Trigger与CHF进行交互，并通过下列信元完成业务总流量、业务上行流量、业务下行流量的配额管理，是否根据流量向终端用户收费由计费中心决定。
          - CHF下发业务的流量配额：Charging Data Request>MultipleUnitInformation>grantedUnit>totalVolume/uplinkVolume/downlinkVolume
          - SMF上报业务的流量用量：Charging Data Request>MultipleUnitUsage>UsedUnitContainertotalVolume/uplinkVolume/downlinkVolume
    - 不带配额管理（离线计费）：SMF通过Charging Data Request>MultipleUnitUsage>UsedUnitContainer>totalVolume/uplinkVolume/downlinkVolume信元将业务总流量、业务上行流量、业务下行流量的计费信息上报给计费系统，是否根据流量向终端用户收费由计费中心决定。
