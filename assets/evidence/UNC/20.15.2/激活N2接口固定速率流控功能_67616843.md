## [操作步骤](#ZH-CN_OPI_0167616843)

1. 进入 “MML命令行-UNC” 窗口。
2. 设置N2接口指定消息类型固定流控速率。
  **[SET N2FIXEDFCBYMSG](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N2接口管理/N2接口固定速率流控管理/设置指定消息类型固定速率流控信息（SET N2FIXEDFCBYMSG）_96805504.md)**
3. **可选：**设置软参DWORD14 BIT30值为0，开启寻呼范围为TA级别的寻呼消息固定速率流控功能。
  **[SET AMFSOFTPARAOFBIT](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/软件参数管理/设置AMF软件参数比特位（SET AMFSOFTPARAOFBIT）_09651487.md)**
4. **可选：**通过软参DWORD28设置寻呼范围为TA级别的寻呼消息固定速率流控阈值。
  **[SET AMFSOFTPARA](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/软件参数管理/设置AMF软参（SET AMFSOFTPARA）_09653628.md)**

## [任务示例](#ZH-CN_OPI_0167616843)

任务描述

在重大节日或大型活动时，某些TA下集中大量用户，可以预期大量语音呼叫同时发生，因此系统开启N2接口Paging消息和Service Request消息的固定速率流控功能，设置打开固定速率流控开关，并设置固定流控速率。AMF的N2接口上发送的Paging消息按照设定的固定流控速率发送，接收的Service Request消息按照设定的固定流控速率接入，对于超过门限的Service Request消息， UNC 丢弃，并上报N2固定速率流控起控告警。

脚本

//针对Paging和Registration Request开启固定速率流控，速率门限是单Pod 4000个/秒。执行如下命令：

```
SET N2FIXEDFCBYMSG:MSGTYPE=PAGING,FCSWITCH=On,THRESHOLD=4000;
SET N2FIXEDFCBYMSG:MSGTYPE=SERVICEREQUEST,FCSWITCH=On,THRESHOLD=4000;
```

//（可选）设置软参DWORD14 BIT30值为0，开启寻呼范围为TA级别的寻呼消息固定速率流控功能。

```
SET AMFSOFTPARAOFBIT: DT=Dw, DWORDNUM=14, VALUE=VALUE_0, POSITION=30;
```

//（可选）通过软参DWORD28设置寻呼范围为TA级别的寻呼消息固定速率流控阈值。

```
SET AMFSOFTPARA: DT=Dw, DWORDNUM=28, DWORDVALUE=80000;
```
