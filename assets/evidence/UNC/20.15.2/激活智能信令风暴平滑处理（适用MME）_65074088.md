## [操作场景](#ZH-CN_OPI_0165074088)

LTE网络中，当MME和SGW-C间S11接口故障导致流程失败时，MME会批量分离用户，引发短时间内大批量UE附着，进而引发信令风暴。引入本特性后，当部分流程的失败速率超过一定门限时，MME以较低的速率延迟分离受影响的UE，从而降低引发网络信令风暴的风险。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0165074088)

前提条件

- 请仔细阅读[WSFD-010804 智能信令风暴平滑处理特性概述](特性概述_62734662.md)。
- 建议同时开启统一HTR流控特性，控制向周边网元发送的信令数量，避免网络拥塞。

## [操作步骤](#ZH-CN_OPI_0165074088)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开智能信令风暴平滑处理特性开关，设置 “抑制用户数控制参考值” 和 “抑制时长（分）” 。
  **[**SET SIGSUPPRESS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/信令抑制管理/S11接口故障场景信令抑制管理/设置S11接口信令风暴抑制功能参数(SET SIGSUPPRESS)_72345767.md)**
3. 配置MME在GTP-C路径上向SGW-C发送Echo Request消息的间隔周期。
  **[**SET GTPPUB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C协议参数配置/设置GTP-C协议参数(SET GTPPUB)_26145920.md)**
  ![](激活智能信令风暴平滑处理（适用MME）_65074088.assets/notice_3.0-zh-cn_2.png)

  为快速感知MME与SGW-C之间链路状态的变化，建议将该参数值配置为最小值（ “60s” ）。

## [任务示例](#ZH-CN_OPI_0165074088)

任务描述

为减少现网S11接口故障场景下信令风暴的风险，开启智能信令风暴平滑处理特性并配置自定义信令抑制策略。运营商在系统中规划有30万4G用户数，并且希望在20分钟内完成对受影响UE的分离。为快速感知MME与SGW-C之间链路状态的变化，将MME在GTP-C路径上向SGW-C发送Echo Request消息的间隔周期配置为60秒。

脚本

//打开智能信令风暴平滑处理特性开关，并按上述任务配置本特性。

```
SET SIGSUPPRESS: SIGSUPPSWITCH=ON, SIGSUPPCFGPLCY=SPECIFIC, SUPPUSRNUM=300000, SUPPTIME=20;
```

//配置MME在GTP-C路径上向SGW-C发送Echo Request消息的间隔周期。

```
SET GTPPUB: V2EI=60;
```
