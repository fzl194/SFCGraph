#### [适用NF](#ZH-CN_CONCEPT_0184949357)

SGSN

#### [定义](#ZH-CN_CONCEPT_0184949357)

去激活空闲 PDP上下文 是指在一个可控的时间内，如果一个PDP上下文没有任何流量， SGSN 将会去激活这个空闲PDP上下文。

#### [客户价值](#ZH-CN_CONCEPT_0184949357)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 减少系统内的垃圾上下文，降低CPU负荷，提高<br>无线资源<br>和SGSN中用户资源的利用效率。 |
| 用户 | 享受更灵活的服务，节省资费，如在无业务的情况下自动下线等。 |

#### [应用场景](#ZH-CN_CONCEPT_0184949357)

现网存在大量空闲 PDP ，使得网络资源利用率低，需要开通去激活空闲PDP上下文。

#### [对系统的影响](#ZH-CN_CONCEPT_0184949357)

当较短时间内有大量PDP上下文需要去激活时，可能会影响SGSN系统对接入消息的处理速度，但是由于SGSN系统的平滑处理，该影响可以忽略。

#### [应用限制](#ZH-CN_CONCEPT_0184949357)

本特性在DT（Direct Tunnel）场景下无效。

#### [原理概述](#ZH-CN_CONCEPT_0184949357)

PDP上下文在 UNC 系统中激活后，SGSN检测PDP上下文是否有数据流量，如果没有数据流量的时间超过关闭空闲PDP定时器时长，就去激活该PDP上下文。SGSN去激活空闲PDP上下文的业务流程如 [图1](#ZH-CN_CONCEPT_0184949357__zh-cn_concept_0130429370_fig1) 所示。

处理过程如下：

1. 如果GGSN上有相应的PDP上下文信息且关闭空闲PDP定时器超时，SGSN向GGSN发送Delete PDP Context Request消息去激活指定PDP上下文。
2. GGSN向SGSN反馈Delete PDP Context Response消息。
3. SGSN向MS发送Deactivate PDP Context Request消息去激活指定PDP上下文。
4. MS向SGSN反馈Deactivate PDP Context Accept消息。SGSN删除指定PDP上下文。
