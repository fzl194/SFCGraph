  > **说明**
  > SGW-C和PGW-C分离场景，可能存在PGW-C有扩展QCI的license但是SGW-C无扩展QCI的license的情况。此情况下SGW-C收到携带扩展QCI的专有承载创建请求时的处理受软参控制：
  >
  > [BIT844 控制SGW-C在无扩展QCI的license时，收到携带扩展QCI的专有承载激活请求是否允许激活成功](../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（SMF_GGSN_SGW-C_PGW-C）/QoS管理/BIT844 控制SGW-C在无扩展QCI的license时，收到携带扩展QCI的专有承载激活请求是否允许激活成功_26951283.md) 。

## [任务示例](#ZH-CN_OPI_0310505693)

任务描述

运营商规划扩展QCI 64用于支付类业务，在 UNC 上配置：

- 允许漫游用户使用扩展QCI进行接入网络
- Ga接口话单消息用扩展QCI
- Gy接口CCR消息用扩展QCI
- Radius鉴权和计费请求消息用扩展QCI
- N40接口请求消息用扩展QCI
- 扩展QCI映射为标准QCI的缺省值为1
- 新增扩展QCI 64向标准QCI 8的映射规则

脚本

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2UNCQCI1", SWITCH=ENABLE;
```

//允许漫游用户使用扩展QCI进行接入网络，Ga接口话单消息/Gy接口CCR消息/Radius鉴权和计费请求消息/N40接口请求消息用扩展QCI，扩展QCI映射为标准QCI的缺省值为1。

```
SET EXTENQCIATTR:QCICTRLFLAG=ENABLE,GASHOWFLAG=ENABLE,GYSHOWFLAG=ENABLE,RADIUSSHOWFLAG=ENABLE,N40SHOWFLAG=ENABLE,EXTQCIMAPDEFQCI=1;
```

//新增扩展QCI 128向标准QCI 8的映射规则。

```
ADD EXTENDQCIMAP:EXTENDQCI=128,STANDARDQCI=8;
```

//（可选）当PCF向SMF发送的N7接口消息中未包含qosChars信元时，需配置扩展QCI的QoS特征。

```
ADD QOSCHARS: FQI=128, RESOURCETYPE=NON_CRITICAL_GBR, PRIORITYLEVEL=10, PACKETDELBUDGET=1, PACKETERRRATE="4E-6", AVERWINDOW=1;
```
