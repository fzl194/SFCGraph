#### [适用NF](#ZH-CN_CONCEPT_0248043355)

SGSN、 GGSN

#### [定义](#ZH-CN_CONCEPT_0248043355)

SGSN、GGSN支持IPv6 PDP上下文的激活、去激活和修改。IPv6 PDP上下文承载于GPRS和UMTS的IPv4/IPv6骨干网。

#### [客户价值](#ZH-CN_CONCEPT_0248043355)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 增加可用的IP地址空间，使网络同时附着更多的用户数，并以此开展更丰富的业务。 |
| 用户 | 网络具有更高的安全性，用户可使用更丰富的业务。 |

#### [应用场景](#ZH-CN_CONCEPT_0248043355)

3GPP在R5阶段引入了IPv6。在R5阶段，IMS采用IPv6承载，RNC、SGSN、GGSN之间采用IPv4或IPv6网络互连，用户终端具有IPv4/IPv6双协议栈，可以访问IPv4/IPv6业务。

#### [可获得性](#ZH-CN_CONCEPT_0248043355)

**涉及NF**

| **涉及NF** | 支持版本 | 功能 |
| --- | --- | --- |
| MS | 无特殊要求 | 将自身的IP能力指示给网络侧。 |
| RNC | 无特殊要求 | 支持IPv6类型数据或头压缩功能。 |
| SGSN | UNC 20.3.0及后续版本 | 为MS创建IPv6类型PDP上下文。 |
| GGSN | UNC 20.3.0及后续版本 | 为MS分配IPv6类型IP地址。 |

#### [应用限制](#ZH-CN_CONCEPT_0248043355)

华为 UNC 支持用户面IPv6承载，但不支持信令面的IPv6承载。即 UNC 仍然处于IPv4网络中，通过IPv4网络与GGSN/BSC/RNC/HLR等相连。

#### [原理概述](#ZH-CN_CONCEPT_0248043355)

随着Internet应用及规模的飞速发展，IPv4协议的可用地址空间越来越少，已经很难满足日益增长的需求，为此，IPv6协议被提出来解决IPv4协议遇到的问题。针对IPv4的不足，IPv6标准采取了一系列措施，增加了许多新功能，包括足够大的地址空间、更高的安全性、对移动性和QoS更好的支持等。IPv6为IP网的可持续发展奠定了良好的基础。

IPv6中的变化体现在以下五个重要方面：

- 扩展地址：地址由IPv4的32bit扩展为128bit，意味着IP可以继续增长而无需考虑资源的匮乏，该地址结构对于提高路由效率也有所帮助。
- 简化头格式：包头的简化减少了路由器上所需的处理过程，从而提高了选择路由的效率。
- 增强对于扩展和选项的支持：意味着可以在几乎不影响普通数据包和特殊包选路的前提下适应更多的特殊需求。
- 流标记：流标记方法为更加高效地处理包流提供了一种机制，这种方法对于实时应用尤其有用。
- 身份验证和保密：身份验证和保密方面的改进使得IPv6更加适用于那些要求对敏感信息和资源特别对待的商业应用。

**系统实现**

UNC 能够为用户创建IPv6类型的PDP上下文，为用户分配IPv6类型的IP地址，IPv6类型的数据包跟IPv4类型的数据包一样，在Gn、Gb或Iu接口上透明传输。

UNC 还能够为同一用户分配不同IP地址类型的PDP上下文，支持IPv4和IPv6的移动终端可以同时运行IPv4和IPv6的PDP上下文。

**业务流程**

如 [图1](#ZH-CN_CONCEPT_0248043355__fig1508155762610) 所示，为用户携带静态PDP Address发起激活流程，以此为例来说明IPv6 PDP上下文的激活流程。

**图1** IPv6 PDP上下文激活流程

<br>

![](特性概述_48043355.assets/zh-cn_image_0248043354_2.png)

1. MS发送Activate PDP Context Request消息给SGSN，消息中携带PDP Type为IPv6，PDP Address不为空。
2. SGSN发送Create PDP Context Request消息给GGSN，消息中携带PDP Type为IPv6，PDP Address不为空。
  GGSN根据请求消息中携带的静态PDP Address进行地址有效性检查，检查都通过后，GGSN反馈Create PDP Context Response消息给SGSN。
3. 执行RAB指配过程建立RAB。
4. SGSN发送Activate PDP Context Accept消息给MS，通知用户激活成功。
5. IPV6场景下，GGSN生成IPv6路由广播（前缀）并且经SGSN将其发送到MS。
