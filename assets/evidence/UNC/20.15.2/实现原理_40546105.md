# 实现原理

系统实现详细机制如 [图1](#ZH-CN_TOPIC_0000001340546105__fig02) 所示。

1. 判断终端是否支持长周期定时器
  若来自终端的Attach Request/RAU Request/TAU Request消息中MS network feature support信元的extended periodic timer比特位为1， UNC 则判定终端支持长周期定时器。
2.
  若终端支持长周期定时器，则判断本地是否配置或终端是否在Attach Request/TAU Request/RAU Request消息中携带或终端是否签约了长周期定时器并通过Attach Accept/TAU Accept/RAU Accept消息下发给终端。

  > **说明**
  > 对于周期性TAU，仅在长周期定时器发生变化时MME才会通过TAU Accept消息带给终端。
    - 从HSS获取长周期定时器
      若用户在HSS签约了长周期定时器， UNC 从Update Location Answer、Insert Subscriber Data Request消息中获取签约数据并解析出Subscribed-Periodic-RAU-TAU-Timer。
    - 本地配置长周期定时器
      UNC 支持基于用户范围（所有用户、本网用户、外网用户以及指定IMSI前缀用户）和APN（APN支持通配，如果配置为“*”，表示所配置的记录适用于所有的APN）配置长周期定时器（extended periodic timer），可通过命令 [**ADD M2MPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/M2M策略参数配置/增加M2M策略参数(ADD M2MPLCY)_72225443.md) 进行本地配置。
    - 终端在Attach Request/TAU Request/RAU Request消息中携带长周期定时器
      系统支持从Attach Request/TAU Request/RAU Request消息中识别T3312/T3412 extended value信元值，并以此信元值作为长周期定时器下发给终端。可通过命令 [**ADD M2MPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/M2M策略参数配置/增加M2M策略参数(ADD M2MPLCY)_72225443.md) 中的参数 “长周期RAU或TAU定时器来源” 进行配置。
