## [操作场景](#ZH-CN_OPI_0295923447)

在线计费会话过程中，对于各种正常和异常场景，可以在GGSN/PGW-C本地设置一些处理动作对DCC流程进行干预。这里的正常和异常场景，主要是指OCS响应GGSN/PGW-C请求消息的状况，及响应消息的返回码值。

> **说明**
> 适用于 GGSN、 PGW-C。

## [必备事项](#ZH-CN_OPI_0295923447)

前提条件

- 请仔细阅读[Ga/Gy接口离线/在线计费](../../../../../业务专题/5G Core 计费解决方案/计费解决方案概述/计费原理/Ga_Gy接口离线_在线计费_87165686.md)。
- 完成[配置到OCS的数据(静态路由+BFD组网)](配置到OCS的数据(静态路由+BFD组网)_95923542.md)。

## [操作步骤](#ZH-CN_OPI_0295923447)

- 配置GGSN/PGW-C收到OCS的请求消息RAR中没有携带RG的处理动作。
  **ADD DCCTEMPLATE**
- 配置用户激活后OCS Down的处理动作。
    1. 配置Tx定时器超时以后的处理方式。
      **ADD DCCTEMPLATE**
      > **说明**
      > 参数 “CCFH” 用于其他方式下在线计费Tx定时器超时后的动作处理。
      >
      > Tx定时器超时后有三种动作处理方式：
      >
      > - “TERMINATE”：表示去激活用户。
      > - “RETRY_AND_TERM”：表示GGSN/PGW-C会向备OCS重发一次请求消息，如果能收到响应，则与备OCS交互；如果Tx定时器超时仍然未得到响应，则去激活用户。选择该种方式时，需要一方面部署主备OCS，一方面主OCS组支持failover，即在CCA消息中CC-Session-Failover AVP支持failover，另一方面还需要在GGSN/PGW-C本地配置支持failover功能，即配置了[步骤 2](#ZH-CN_OPI_0295923447__step4)，该动作才能生效。
      > - “CONTINUE”：表示GGSN/PGW-C会向备OCS重发一次请求消息，如果能收到响应，则与备OCS交互；如果Tx定时器超时仍然未得到响应，则转离线计费，继续进行业务。选择该种方式时，同上需要一方面部署主备OCS组，一方面主OCS组支持failover，即在CCA消息中CC-Session-Failover AVP支持failover，另一方面还需要在GGSN/PGW-C本地配置支持failover功能，即配置[步骤 2](#ZH-CN_OPI_0295923447__step4)，[步骤 7](#ZH-CN_OPI_0295923447__step5)，该动作才能生效。
    2. 配置在线计费是否支持failover，配置 “FAILOVERSUP” 参数为 “ENABLE” 。当Tx定时器超时后的处理方式为 “RETRY_AND_TERM” 或者 “CONTINUE” 时，需要配置该步骤。
      **ADD DCCTEMPLATE**
    3. **可选：**配置在线计费是否支持自动failback。
      **ADD DCCTEMPLATE**
      > **说明**
      > “FAILOVERSUP” 和 “AUTOFAILBACK” 是协议定义的两种OCS故障容错方式， “FAILOVERSUP” 是指在主OCS故障的情况下，GGSN/PGW-C会发送DCC消息给备OCS服务器组进行处理。 “AUTOFAILBACK” 是指当主OCS链路故障恢复后，DCC业务切换回到主OCS服务器上。
    4. 配置在线计费和离线计费同时使能的用户进行在线计费时，发生CCFH之后是否在离线话单中添加ccfh标志位。
      **ADD DCCTEMPLATE**
      > **说明**
      > 本命令只针对在线计费用户生效，且在 [配置离线/在线计费方式（GGSN/PGW-C）](../../WSFD-011201 支持离线计费/激活离线计费/配置离线_在线计费方式（GGSN_PGW-C）_95923455.md) 中根据用户归属属性配置的离线计费开关 **SET CHARGECTRL** 必须使能。
    5. 配置Tx定时器的时长。
      **SET TXTIMER**
    6. 配置在线计费CCFH处理时是否强制产生话单，强制产生话单时话单中charging characteristic字段的取值；配置R6版本的eG-CDR是否支持CCFH处理。
      **SET OFCCDRPARA**
    7. 配置业务持续时长。
      **SET HOLDINGTIME**
      > **说明**
      > - 使用命令**SET DEACTIVERATE**可以去激活在线计费转为离线计费且Holding Time定时器值为0的用户。
      > - 当Tx定时器超时后的处理方式为“CONTINUE”时，允许用户保持业务的时长，超出该时长则去激活用户。当配置为0分钟表示允许用户永久保持在线进行业务。
    8. **可选：**配置在线计费用户转为离线计费用户后，是否对该用户执行不同的时长、流量阈值。
      **SET OFCTHRESHOLD**
      > **说明**
      > 本命令中 “TIMETHRESHCCFH” 、 “VOLTHRESHCCFH” 参数配置的时长、流量阈值只针对已转离线计费的在线计费用户生效。若参数未配置，则默认为使用原有的离线计费阈值。
- 配置用户激活时OCS Down的处理动作。
    1. **可选：**配置转离线后业务持续时长。
      **SET HOLDINGTIME**
      > **说明**
      > 当 **SET OCSDOWNACTION** 命令 **OCSDNACTION** 选择参数 “PERMIT” 时，需要首先配置 **HOLDINGTIME** 。
    2. 配置OCS Down时是否允许用户激活。
      **SET OCSDOWNACTION**
- 配置command层消息码为异常返回码时的处理动作。
    1. 配置command层消息码为异常返回码时的处理动作。
      **ADD CMDRCACT**
      > **说明**
      > - 返回码DIAMETER_SUCCESS (2001)，DIAMETER_LIMITED_SUCCESS (2002)的动作不可配置，属于正常返回码，业务动作取决于MSCC层的返回码动作控制。
      > - 当在线计费用户本地配置用户转离线时，GGSN/PGW-C支持开启Holding Time定时器，Holding Time定时器超时之后再进行用户去活。
      > - 如果某返回码没有单独配置对应的动作，则采用[步骤 2](#ZH-CN_OPI_0295923447__step30)的**SET CMDLEVDFTBEH**配置的动作进行处理。
      > - 部分协议规定了标准处理的特殊返回码缺省值为固定处理，如果这部分返回码对应的动作没有配置，缺省值为特殊的处理流程，不会采用**SET CMDLEVDFTBEH**配置的动作；如果配置了返回码对应的动作，则按照配置的动作处理。这部分返回码及缺省动作处理请参见命令帮助的说明。
    2. 配置command层消息码为异常返回码时的默认处理动作。
      **SET CMDLEVDFTBEH**
- 配置MSCC层消息码为异常返回码时的处理动作。
    1. 配置MSCC层消息码为异常返回码时的处理动作。
      **ADD MSCCRCACT**
      > **说明**
      > - 返回码DIAMETER_SUCCESS (2001)，DIAMETER_LIMITED_SUCCESS (2002)的动作不可配置，属于正常返回码，正常转发报文并计费。
      > - 如果某返回码没有单独配置对应的动作，则采用[步骤 2](#ZH-CN_OPI_0295923447__step3000)的**SET MSCCLEVDFTBEH**配置的动作进行处理。
      > - 部分协议规定了标准处理的特殊返回码缺省值为固定处理，如果这部分返回码对应的动作没有配置，缺省值为特殊的处理流程，不会采用**SET MSCCLEVDFTBEH**配置的动作；如果配置了返回码对应的动作，则按照配置的动作处理。这部分返回码及缺省动作处理请参见命令帮助的说明。
    2. 配置MSCC层消息码为异常返回码时的默认处理动作。
      **SET MSCCLEVDFTBEH**
  > **说明**
  > GGSN/PGW-C支持针对OCS返回的特殊返回码设置动作，包括重定向动作。当GGSN/PGW-C收到这些特殊返回码时，会根据配置的动作进行重定向处理。重定向处理动作可以由OCS下发，也可以在GGSN/PGW-C本地配置。其优先级关系是：
  >
  > - OCS下发的动作优先级最高。
  > - 其次是在GGSN/PGW-C本地为Command层特殊返回码配置的动作**ADD CMDRCACT**。
  > - 再者是在GGSN/PGW-C本地为MSCC层特殊返回码配置的动作**ADD MSCCRCACT**。
  > - 然后是GGSN/PGW-C本地默认的Command层特殊返回码处理动作**SET CMDLEVDFTBEH**。
  > - 优先级最低的是GGSN/PGW-C本地默认的MSCC层特殊返回码处理动作**SET MSCCLEVDFTBEH**。
- 配置MSCC层消息码为异常返回码且OCS下发FUI时的处理动作。
    1. 配置MSCC层消息码为异常返回码且OCS下发FUI时的处理动作。
      **ADD MSCCRCACT**
    2. 配置MSCC层消息码为异常返回码且OCS下发FUI时的默认处理动作。
      **SET MSCCLEVDFTBEH**
  > **说明**
  > - 当**BIT1087**值为0，**BIT1738**值为0时，如果MSCC层消息码为异常返回码为4012，以FUI的处理动作为准，如果MSCC层消息码为其他异常码执行**SET MSCCLEVDFTBEH**命令配置的动作，忽略FUI。
  > - 当**BIT1087**值为0，**BIT1738**值为1时，执行FUI动作。
  > - 当**BIT1087**值为1，执行**ADD MSCCRCACT**命令配置的动作，忽略FUI。
- 配置GGSN/PGW-C收到OCS的CCA消息中携带FUI Terminate时的处理动作，配置 “FUATERMINATE” 参数。
  **ADD DCCTEMPLATE**
- 配置GGSN/PGW-C与对端OCS重建TCP链接的间隔时长。
  **SET TCTIMER**
- 配置DWR消息响应超时次数达到上限后GGSN/PGW-C的处理动作。
  **SET TWTIMER**
- 故障场景紧急处理：配置OCS故障（非异常结果码场景）或收到command层异常结果码时的Gy接口的一键放通功能。
  **SET FHBYPASS**
  > **说明**
  > - 由于配置**SET FHBYPASS**将影响用户的计费方式，只有在取得了客户的书面认可后方可使用。
  > - **SET FHBYPASS**优先级高于DCC全局模板或DCC模板下的**SET OCSDOWNACTION**、**SET CMDLEVDFTBEH**、**ADD CMDRCACT**。
  > - GGSN/PGW-C支持如下方式配置故障恢复后的处理。方式1针对全部放通用户，可能存在恢复不及时的情况，推荐使用方式2。
  >     - 方式1：通过**SET FHBYPASS**命令“HOLDINGTIME”控制对放通用户的后续处理，即是否转离线。
  >     - 方式2：通过**DSP CPPDPCHGINFO**查询被放通用户的归属信息。

## [任务示例](#ZH-CN_OPI_0295923447)

任务描述

普通在线计费会话过程中的处理动作。

脚本

//配置DCC template模板（配置GGSN/PGW-C收到OCS的请求消息RAR中没有携带RG的处理动作、配置用户激活后OCS Down的处理动作）。

```
ADD DCCTEMPLATE:DCCTMPLTNAME="dcc-test", CCFH=CONTINUE, CCFHOFFLINE=ENABLE, RARNORGACTION=REPORT_ALL_RG, FAILOVERSUP=ENABLE;
```

//配置用户激活后OCS Down的处理动作。

```
SET TXTIMER:DCCTMPLTNAME="dcc-test",TXTIMERFLAG=TXTIMER_VALUE,TXTIMER=10;
```

```
SET HOLDINGTIME:DCCTMPLTNAME="dcc-test",HOLDINGTIME=30,RANGETIME=10;
```

//配置用户激活时OCS Down的处理动作。

```
SET HOLDINGTIME:DCCTMPLTNAME="dcc-test",HOLDINGTIME=30,RANGETIME=10;
```

```
SET OCSDOWNACTION:DCCTMPLTNAME="dcc-test",OCSDNACTION=PERMIT,HOLDINGTMFLAG=ENABLE;
```

//配置消息码为异常返回码时的处理动作。

```
ADD CMDRCACT:DCCTMPLTNAME="dcc-test",CMDRC="4012",CMDRCACT=TERM_WITH_CCRT;
```

```
SET CMDLEVDFTBEH:DCCTMPLTNAME="dcc-test",CMDLEVDFTBEH=TERM_WITH_CCRT;
```

```
ADD MSCCRCACT:DCCTMPLTNAME="dcc-test",MSCCRC="4012",MSCCRCACT=BLOCK;
```

```
SET MSCCLEVDFTBEH:DCCTMPLTNAME="dcc-test",MSCCLEVDFTBEH=BLOCK;
```

//配置GGSN/PGW-C与对端OCS重建TCP链接的间隔时长。

```
SET TCTIMER:TCTIMERLEN=1,TCMODE=MULTIPLY;
```

//配置DWR消息响应超时次数达到上限后GGSN/PGW-C的处理动作。

```
SET TWTIMER:TWTIMERLEN=30,TWNORESPTIMES=3,TWRSTTCP=DISABLE;
```

//配置OCS故障（非异常结果码场景）或收到command层异常结果码时的Gy接口的一键放通功能。

```
SET FHBYPASS: ONLCHARGE=ENABLE, GYERRRC=ENABLE;
```
