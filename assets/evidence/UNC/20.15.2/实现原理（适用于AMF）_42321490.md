# 实现原理（适用于AMF）

不论5GC网络小区广播服务是否部署PWS-IWF，AMF始终通过基于Namf服务的接口向CBCF或是PWS-IWF提供服务。AMF针对小区广播功能提供了如下服务和服务操作：

*表1 AMF针对小区广播功能提供的服务和服务操作*

| 服务名称 | 服务操作 | 操作语义 | 服务消费者 |
| --- | --- | --- | --- |
| Namf_Communication | NonUeN2MessageTransfer | Request/Response | CBCF |
| Namf_Communication | NonUeN2InfoSubscribe | Subscribe/Notify | CBCF |
| Namf_Communication | NonUeN2InfoUnSubscribe | Subscribe/Notify | CBCF |
| Namf_Communication | NonUeN2InfoNotify | Subscribe/Notify | CBCF |

相关的服务化流程如 [图1](#ZH-CN_CONCEPT_0000001142321490__fig2821134917597) 所示。

**图1** 小区广播服务化流程图

<br>

![](实现原理（适用于AMF）_42321490.assets/zh-cn_image_0000001188460079_2.png)

1. 如果CBCF支持接收Write-Replace-Warning notification和Stop-Warning notification，则CBCF使用Namf_Communication_NonUeInfoSubscribe服务操作订阅该服务。
2. CBCF通过Namf_Communication_NonUeN2Message传输服务操作向AMF发送Write-Replace-Warning Request消息或Stop-Warning-Request消息。AMF返回Namf_Communication_NonUeN2MessageTransfer Response响应消息。
3. AMF根据广播消息中携带的globalRanNodeList或者TAI List确定N2 Message Container应转发到NG-RAN节点。
4. AMF将N2 Message Container中包含的消息通过N2接口转发给选定的NG-RAN节点，并接收来自NG-RAN的响应。
5. 如果CBCF在步骤1中订阅了接收基站的notification响应，AMF使用Namf_Communication_NonUeN2InfoNotify服务操作将汇总的响应转发给CBCF。

**告警消息发送流程**

NF服务消费者（例如CBCF）应调用NonUeN2MessageTransfer服务操作，以启动警告消息广播。AMF应接受请求，并立即响应NF服务消费者。同时AMF需要将告警消息下发到多个NG-RAN节点。NG-RAN节点负责将告警消息广播给UE。

**图2** 小区广播服务发送流程图

<br>

![](实现原理（适用于AMF）_42321490.assets/zh-cn_image_0000001142161696_2.png)

1. CBE向CBCF发送告警信息。CBCF收到告警信息后，首先鉴权告警信息的有效性，然后CBCF将告警消息转换为标准的Write-Replace Warning Request信令消息(消息ID、序列号、跟踪区ID列表、告警区域等)，并发送消息到AMF。
2. CBCF发送Namf_Communication_NonUeN2MessageTransfer Request消息给AMF，Namf_Communication_NonUeN2Message主要包含N2 Message Container、globalRanNodeList IE、taiList IE、ratSelector IE。Write-Replace Warning Request NG-RAN消息通过Namf_Communication_NonUeN2Message中的N2 Message Container传输。
3. AMF发送Namf_Communication_NonUeN2MessageTransfer Response消息给CBCF，确认AMF开始了消息广播分发。
  如果消息中存在globalRanNodeList IE ，AMF应将N2 Message Container转发给globalRanNodeList IE中指示的gNB；
  如果消息中不存在globalRanNodeList IE，AMF根据消息中的ratSelector和taiList列表，选择需要广播的所有的gNB；如果消息中不包含tailist，则应该向符合ratSelector的所有gNB广播消息。
4. CBCF向CBE通知告警消息已经下发。
5. AMF发送Write-Replace Warning Request告警消息到选中的各个NG-RAN。
6. NG-RAN收到Write-Replace Warning Request告警消息后，会根据告警消息的消息标识、序列号来判断告警消息是否重复消息。如果非重复消息则进行消息广播处理。如果是重复消息则只响应Write-Replace Warning Response，只对第一个收到的消息进行广播。NG-RAN根据消息中的告警区域信元来确定待广播的小区区域。
7. UE通过声音、震动、或者是其他方式向用户告警。
8. **可选：**如果CBCF在步骤1中订阅了接收基站的notification响应，AMF使用Namf_Communication_NonUeN2InfoNotify服务操作将汇总的响应转发给CBCF。

**告警消息取消流程**

NF服务消费者（例如CBCF）应调用NonUeN2MessageTransfer服务操作，以取消告警消息广播。AMF应接受请求，并立即响应NF服务消费者。同时AMF需要将取消告警消息下发到多个NG-RAN节点。

告警消息取消流程如 [图3](#ZH-CN_CONCEPT_0000001142321490__fig3) 所示。

**图3** 小区广播服务取消流程图

<br>

![](实现原理（适用于AMF）_42321490.assets/zh-cn_image_0000001188361451_2.png)

1. CBE向CBCF发送Stop Emergency Broadcast Request发起告警消息取消流程。CBCF收到告警信息后，首先鉴权告警信息的有效性，CBCF确定需要向哪些AMF发送告警取消消息，并确定Warning Area Information中的信息。
2. CBCF向AMF发送Namf_Communication_NonUeN2MessageTransfer Request消息主要包含N2 Message Container、globalRanNodeList IE、 taiList IE 、ratSelector IE。
  Stop Warning Request NG-RAN消息通过N2 Message Container传输，其中包括消息ID，序列号，跟踪区ID列表，告警区域、OMC ID、发送停止警告指示。
  AMF收到Namf_Communication_NonUeN2MessageTransfer Request消息后，如果消息中存在globalRanNodeList IE ，AMF应将N2 Message Container转发给globalRanNodeList IE中指示的gNB；如果消息中不存在globalRanNodeList IE，AMF根据消息中的ratSelector和taiList列表，选择需要发送告警取消消息的所有的gNB；如果消息中不包含tailist，则应该向符合ratSelector的所有gNB广播消息。
3. AMF发送Stop Warning Confirm NG-RAN消息给CBCF，确认AMF已经开始向NG-RAN节点分发告警取消请求消息。
4. CBCF收到AMF发送的告警取消确认消息后，向CBE确认已经启动了警告消息取消流程。
5. AMF发送PWS Cancel Request消息到选中的各个NG-RAN。
6. NG-RAN收到PWS Cancel Request消息后，会根据告警消息的消息标识、序列号来判断告警消息是否重复消息。如果是重复消息则只对第一个AMF响应并停止小区广播流程。
7. **可选：**如果Stop Warning Request NG-RAN消息中包含Stop Warning Indication参数，则AMF可以根据本地配置策略决定是否将从NG-RAN获取到的告警消息取消区域的列表转发给CBCF。

**PWS重启指示流程**

NG-RAN重启后会删除所有告警消息数据，如果告警消息服务在NG-RAN的一个或多个小区中运行，NG-RAN应发送PWS重启指示消息，通知CBCF重新加载告警消息数据。

**图4** PWS重启指示流程

<br>

![](实现原理（适用于AMF）_42321490.assets/zh-cn_image_0000001142347578_2.png)

1. NG-RAN向AMF发送PWS Restart Indication消息，其中应包括NG-RAN的标识、重启小区的标识、以及配置重启cell(s)、TAI(s)和Emergency Area Id(s)，以请求CBCF重新加载其警告消息数据。
2. 如果存在多个CBCF，AMF应将PWS restart Indication转发给所有CBCF（AMF只需要透传消息即可）。

**PWS失败指示流程**

当NG-RAN的一个或多个小区中正在进行的PWS操作已失败时，NG-RAN向AMF发送PWS失败指示消息进行通知。

**图5** PWS失败指示流程

<br>

![](实现原理（适用于AMF）_42321490.assets/zh-cn_image_0000001849714241_2.png)

1. NG-RAN向AMF发送PWS Failure Indication消息，其中应包括NG-RAN的标识、PWS失败小区的标识。
2. 如果存在多个CBCF，AMF应将PWS Failure Indication转发给所有CBCF（AMF只需要透传消息即可）。
