#### [适用NF](#ZH-CN_CONCEPT_0248043368)

AMF、SMF

#### [定义](#ZH-CN_CONCEPT_0248043368)

本特性支持在5G网络的PDU会话建立流程中为UE分配IPv6前缀类型的地址，建立IPv6 PDU会话的用户上下文，并支持IPv6 PDU会话的创建、修改、释放。

#### [客户价值](#ZH-CN_CONCEPT_0248043368)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 可以向用户提供一个IPv6的终端地址，使网络同时附着更多的用户数。<br>- 支持开展丰富的IPv6业务。 |
| 用户 | 享受丰富的IPv6业务。 |

#### [应用场景](#ZH-CN_CONCEPT_0248043368)

用户终端具有IPv6协议栈，可以访问IPv6业务。

#### [可获得性](#ZH-CN_CONCEPT_0248043368)

**涉及NF**

| 涉及NF | 支持版本 | 功能说明 |
| --- | --- | --- |
| UE | 无特殊要求 | 支持IPv6。 |
| SMF | UNC 20.0.0及后续版本 | 支持从UDM获取并解析PDU会话类型、静态IP地址等签约信息，根据这些签约信息及UE请求的PDU会话类型，为PDU会话分配IP地址。 |
| UPF | 无特殊要求 | 传递上行、下行用户数据流。 |

**License支持**

#### [原理概述](#ZH-CN_CONCEPT_0248043368)

SMF在PDU会话建立流程中，能够为用户创建IPv6的PDU会话，为用户分配IPv6类型的IP地址，IPv6类型的数据包与IPv4类型的数据包一样，在接口上透明传输。

PDU会话建立流程中的IPv6 IP地址分配的流程如 [图1](#ZH-CN_CONCEPT_0248043368__fig733863531520) 所示。

> **说明**
> PDU会话建立详细流程可参见3GPP 23.502协议，本特性中仅描述与IPv6地址分配强相关的消息和信元。

**图1** IPv6的PDU会话建立流程

<br>

![](特性概述_48043368.assets/zh-cn_image_0248043382_2.png "点击放大")

1. UE向AMF发送PDU会话建立请求，消息中携带Requested PDU Session Type为**IPv6或者IPv4v6**。
2. AMF根据S-NSSAI、DNN信息等为PDU会话建立选择SMF。
3. AMF向SMF发送Nsmf_PDUSession_CreateSMContext Request请求创建SM上下文，带Requested PDU Session Type为**IPv6或者IPv4v6**。
4. **可选：**SMF向UDM注册PDU会话、获取UE签约数据、订阅UE签约数据变化通知。
5. SMF向AMF返回PDU会话响应，接受创建PDU会话。
6. **可选：**获取PCC规则。
7. SMF选择UPF，并根据会话请求消息中携带的Requested PDU Session Type类型，从UDM中存储的静态地址类型分配**I****Pv6的IP地址/前缀**。
8. **可选：**SMF发起SM Policy Association Modification流程向PCF报告信息，例如**分配的IPv6地址/前缀**。PCF可以向SMF提供更新的策略信息。
9. SMF发起N4 Session Establishment流程建立与UPF的连接，提供用于该PDU会话的数据监测，报告规则，CN隧道信息等。
10. SMF向AMF发送Namf_Communication_N1N2MessageTransfer消息，携带的信息包括分配的**IPv6地址/前缀**，通知RAN和UE需要建立PDU会话。
11. AMF向RAN发送N2 PDU Session Request，该消息中包含NAS message信元，该信元的N1 SM Container中包含PDU Session Establishment Accept及分配的**IPv6地址****/前缀**。
12. RAN与UE发起信令交互，将SMF需要发送给UE的N1 SM Container消息转发至UE，请求UE建立PDU会话，消息中包含PDU Session Establishment Accept及分配的**IPv6地址/前缀**。
13. **可选：**RAN向AMF发送N2 PDU Session Response，建立AN隧道信息。
14. AMF将从RAN接收到的N2 SM Information转发给SMF。
15. SMF发起N4 Session Modification流程，将AN隧道信息和相应的转发规则发送给UPF。
16. SMF向AMF发送Nsmf_PDUSession_UpdateSMContext Response。
17. IPV6场景下，SMF生成IPv6路由广播（前缀）并且经由N4会话和UPF将其发送到UE。
