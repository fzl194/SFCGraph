#### [命令功能](#ZH-CN_CONCEPT_0259036939)

该命令用于显示资源的基本信息。用户可用该命令查看资源的名称、类型、运行状况等信息。

#### [注意事项](#ZH-CN_CONCEPT_0259036939)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0259036939)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0259036939)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| RESNAME | 资源名称 | 可选必选说明：可选参数<br>参数含义：该参数表示资源名称，指节点或容器的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。不支持空格，区分大小写。<br>默认值：无 |

#### [输出结果说明](#ZH-CN_CONCEPT_0259036939)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 逻辑资源编号 | 用于表示逻辑资源编号，该编号由网元负责分配。 |
| 资源名称 | 表示资源名称，指节点或容器的名称。 |
| VNFM分配的资源编号 | 用于表示资源编号，该编号由VNFM层负责分配。 |
| VIM分配的资源编号 | 用于表示资源编号，该编号由VIM层负责分配。 |
| 资源实例类型 | 表示资源实例类型。 |
| 运行状态 | 表示资源的运行状态。<br>- 正常：资源运行状态正常。<br>- 不正常：资源运行状态异常。<br>- 停止：资源没有运行。 |
| 内存容量（MB） | 表示资源的内存容量（MB）。 |
| 存储容量（GB） | 表示资源的存储容量（GB）。 |
| CPU核数量 | 表示资源的CPU核个数。 |
| 内存利用率（%） | 表示OS的内存利用率。 用户可以设置内存过载阈值（缺省情况下为95%），如果超过该阈值就会触发告警。在没有大规模部署业务前，如果内存占用率持续偏高属不正常现象，此时应检查设备查找异常原因。 |
| CPU利用率（%） | 表示OS的CPU利用率。用户可以设置CPU过载阈值（缺省情况下为95%），如果超过该阈值就会触发告警，在没有大规模部署业务前，如果CPU占用率持续偏高属不正常现象，此时应检查设备查找异常原因。 |
| 磁盘利用率（%） | 表示资源的磁盘利用率，当前体现的是已分配的业务磁盘空间百分比。 |
| HA组 | 表示资源的HA组。在同一HA组的资源需要部署在不同的物理机上。显示为NULL时表示没有指定HA组，不受该限制。OMU天然具有反亲和属性，需要部署在不同的物理机上。 |
| 亲和性组 | 表示资源的亲和性组。如果两个资源在同一个亲和性组里，则这两个资源会尽量部署在同一个物理上。显示为NULL表示没有指定亲和性组，资源部署时不受该限制。 |
| 扩展组 | 表示资源的扩展组。每个扩展组会独立判断Scale out/in（扩容/减容）的条件是否满足。显示为NULL时表示没有指定扩展组。 |
| 主机位置 | 表示资源的主机位置。为NULL时表示物理机未知。 |
| 主机名称 | 表示资源的主机名称。为NULL时表示物理机未知。 |
| 资源类型 | 表示资源类型。<br>- 虚拟机：表示资源类型为虚拟机。<br>- 容器：表示资源类型为容器。说明：Full-stack裸机场景下，资源类型为<br>**容器**<br>。 |
| 父资源名称 | 表示父资源名称。为NULL时表示没有对应的父资源，虚拟机环境下为NULL，容器环境下为所在Pod的名称。 |
| 位置信息 | 表示资源的位置信息。 |
| CPU架构类型 | 该参数表示资源的CPU架构类型。<br>- X86：CPU架构类型是X86。<br>- ARM：CPU架构类型是ARM。 |
