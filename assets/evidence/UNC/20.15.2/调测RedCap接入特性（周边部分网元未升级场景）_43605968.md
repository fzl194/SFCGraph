## [操作场景](#ZH-CN_OPI_0000001543605968)

本操作指导用于调测gNodeB和部分网元（AMF和SMF）支持RedCap功能的场景下，RedCap用户是否接入。

## [对系统的影响](#ZH-CN_OPI_0000001543605968)

该操作对系统正常运行没有影响。

## [必备事项](#ZH-CN_OPI_0000001543605968)

前提条件

已完成特性激活配置 [激活RedCap接入（周边部分网元未升级场景）](激活RedCap接入（周边部分网元未升级场景）_43596544.md) 。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0000001543605968)

1. 在 UNC 上 [创建用户跟踪](../../../../../网络运维/日常维护/UNC基础运维操作/创建消息跟踪/创建用户跟踪/用户跟踪_72467812.md) 。
2. 用户开机发起注册，并且新建PDU会话。
  预期结果：通过跟踪看到，用户成功注册，并且成功建立PDU会话，但是在不支持RedCap功能的网元上，会作为普通NR用户处理。
  检查点：
    - AMF是否从gNodeB获取到RedCap Indication信元。
          - 若在消息"INITIAL UE MESSAGE"中，redCapIndication取值为redcap，则代表AMF从gNodeB获取到RedCap Indication信元。
            **图1** INITIAL UE MESSAGE中的RedCap指示位示意图

            <br>

            ![](调测RedCap接入特性（周边部分网元未升级场景）_43605968.assets/zh-cn_image_0000001544424226_2.png)
          - 若未发现此指示位，或者取值不为"redcap"，请检查：
                  - 终端是否为RedCap终端。
                  - gNodeB是否为3GPP协议定义的R17及以上版本。
                  - AMF是否为20.11.0及以上版本。
                  - 若以上条件均满足，请检查AMF上的RedCap功能开关是否开启：通过[**LST AMFREDCAPFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G M2M管理/RedCap功能管理/查询AMF RedCap功能（LST AMFREDCAPFUNC）_17306398.md)可查询。若未开启，执行激活章节的[步骤 2](激活RedCap接入（周边部分网元未升级场景）_43596544.md#ZH-CN_OPI_0000001543596544__step4441348236)。
    - AMF向UDM传递RatType取值为NR。
          - 若在消息"Nudm_UECM_RegistrationAMF3GppAccess Request"中，ratType取值为NR，则代表AMF向UDM传递了RatType取值为NR。
            **图2** Nudm_UECM_RegistrationAMF3GppAccess Request中的ratType

            <br>

            ![](调测RedCap接入特性（周边部分网元未升级场景）_43605968.assets/zh-cn_image_0000001543612540_2.png)
          - 若此消息中，ratType取值不为NR，请检查
                  - UDM版本是否支持RedCap。
                  - AMF上的N8接口兼容性策略是否配置为不支持RedCap接入：通过[**LST AMFN8CMPTPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/AMF N8接口兼容性参数管理/查询AMF N8接口兼容性策略（LST AMFN8CMPTPLCY）_89422861.md)可查询。若为配置为支持，可通过[**ADD AMFN8CMPTPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/AMF N8接口兼容性参数管理/增加AMF N8接口兼容性策略（ADD AMFN8CMPTPLCY）_38983042.md)的参数**REDCAPRAT**来配置。
                  - AMF上的RedCap关于UDM的兼容性开关是否关闭：通过[**LST AMFREDCAPFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G M2M管理/RedCap功能管理/查询AMF RedCap功能（LST AMFREDCAPFUNC）_17306398.md)可查询。若开启，可通过[**SET AMFREDCAPFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G M2M管理/RedCap功能管理/设置AMF RedCap功能（SET AMFREDCAPFUNC）_17306402.md)的参数**RATCMPTSW**取值设置为空来关闭。
    - AMF向SMF传递RatType取值为NR_REDCAP。
          - 若在消息"MsgBetweenHttpSmc"中，ratType取值为NR_REDCAP，则代表AMF向SMF传递了RatType取值为NR_REDCAP。
            **图3** MsgBetweenHttpSmc中的ratType

            <br>

            ![](调测RedCap接入特性（周边部分网元未升级场景）_43605968.assets/zh-cn_image_0000001594743997_2.png)
          - 若此消息中，ratType取值不为NR_REDCAP，请检查AMF上的RedCap兼容性开关是否开启：通过[**LST AMFREDCAPFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G M2M管理/RedCap功能管理/查询AMF RedCap功能（LST AMFREDCAPFUNC）_17306398.md)可查询。若未开启，请执行激活章节的[步骤 2](激活RedCap接入（周边部分网元未升级场景）_43596544.md#ZH-CN_OPI_0000001543596544__step4441348236)。
    - SMF向PCF传递RatType取值为NR。
          - 若在消息"Npcf_SMPolicyControlAPI_SmPolicyControlCreateRequest"中，ratType取值为NR，PCF不支持RedCap功能，将RedCap用户当作普通NR用户处理。
      **图4** Npcf_SMPolicyControlAPI_SmPolicyControlCreateRequest中的ratType

      ![](调测RedCap接入特性（周边部分网元未升级场景）_43605968.assets/zh-cn_image_0000002106798001_2.png "点击放大")
          - 若此消息中的ratType取值不为NR，请检查：
                  - PCF版本是否支持RedCap功能。
                  - SMF向周边网元传递RatType是否配置了PCF网元取值为"NR"：[**LST REDCAPRATVALUE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/RedCap用户RAT值/查询RedCap接入用户的RAT填写值（LST REDCAPRATVALUE）_68306125.md)。若不是，可通过[**SET REDCAPRATVALUE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/RedCap用户RAT值/设置RedCap终端接入RAT填写值（SET REDCAPRATVALUE）_68425885.md)的参数**PCF**取值设置为"NR"来关闭。
    - SMF向UPF传递RatType取值为NR。
          - 若在消息"PFCP Session Establishment Request"中，rat-type取值为nr，则代表SMF向UPF传递了RatType取值为NR。
      **图5** PFCP Session Establishment Request中的rat-type

      <br>

      ![](调测RedCap接入特性（周边部分网元未升级场景）_43605968.assets/zh-cn_image_0000001545100912_2.png)
          - 若此消息中的rat-type取值不为nr，请检查：
                  - UPF版本是否支持RedCap功能。
                  - SMF向周边网元传递RatType是否配置了UPF网元为NR：[**LST REDCAPRATVALUE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/RedCap用户RAT值/查询RedCap接入用户的RAT填写值（LST REDCAPRATVALUE）_68306125.md)。若不是，可通过[**SET REDCAPRATVALUE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/RedCap用户RAT值/设置RedCap终端接入RAT填写值（SET REDCAPRATVALUE）_68425885.md)的参数**UPF**取值设置为"NR"来关闭。

## [任务示例](#ZH-CN_OPI_0000001543605968)

任务描述

调测RedCap用户是否接入核心网。

脚本

//查询AMF上的RedCap功能开关，RatType所有周边网元兼容性开关，配置AMF通知空闲态会话SMF策略为立即通知。

```
LST AMFREDCAPFUNC:;
```

//关闭AMF上的关于UDM的兼容性开关。

```
SET AMFREDCAPFUNC:RATCMPTSW=UDM-0;
```

//查询SMF上的RedCap功能开关。

```
LST SMFFUNC:;
```

//查询SMF向所有周边网元传递RatType的值。

```
LST REDCAPRATVALUE:;
```
