## [操作场景](#ZH-CN_OPI_0000001304203170)

本操作指导介绍在运行网络中激活基于IMEI的NR差异化服务特性的操作过程。

本特性是指AMF在获取终端IMEISV后通过N2接口将IMEISV以“Masked IMEISV”信元形式发送给gNodeB，便于gNodeB基于终端的类型、软件版本进行无线侧网络质量分析，从而实现对不同终端的差异化控制。

> **说明**
> 适用于AMF。

## [操作步骤](#ZH-CN_OPI_0000001304203170)

1. 进入 “MML命令行-UNC” 窗口。
2. 针对一定范围的用户，开启AMF获取终端IMEISV功能。
  [**ADD NGPEIPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/业务安全管理/PEI策略管理/增加5G PEI策略（ADD NGPEIPLCY）_09651346.md)
3. 设置AMF在Initial Context Setup Request和Handover Request消息中携带Masked IMEISV。
  [**SET NGAPCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N2接口管理/NGAP兼容性参数管理/设置NGAP兼容性参数（SET NGAPCMPT）_09652644.md)

## [任务示例](#ZH-CN_OPI_0000001304203170)

任务描述

无线5G SA网络希望AMF配合，通过gNodeB实现在调度、状态控制、功率分配等方面对所有终端做差异化控制。

脚本

//针对所有用户，开启AMF获取终端IMEISV功能。

```
ADD NGPEIPLCY: SUBRANGE=ALL_USER, GETPEIPLCY=GETIMEISV;
```

//AMF给gNodeB下发的Initial Context Setup Request和Handover Request消息中携带Masked IMEISV信元。

```
SET NGAPCMPT: MSKIMEISV=YES, IMEISVCODING=FORWARD_ORDER;
```
