# 实现原理

**原因值下发流程**

**图1** 原因值下发流程

<br>

![](实现原理_66338518.assets/zh-cn_image_0166915166_2.png "点击放大")

1. 判断失败事件的类型。
    - 周边网元拒绝 => 步骤[2](#ZH-CN_CONCEPT_0166338518__step2)
    - 周边网元超时/周边网元返回响应消息错误/UNC内部错误 => 步骤[6](#ZH-CN_CONCEPT_0166338518__step6)
2. 查询MM/SM流程控制参数中是否配置了原因值映射组。
  > **说明**
  > 拒绝原因值非 “0” ，表示配置了原因值映射组。
    - 是 => 步骤[3](#ZH-CN_CONCEPT_0166338518__step3)
    - 否 => 步骤[9](#ZH-CN_CONCEPT_0166338518__step9)
3. 根据原因值映射组和“原因值范围”查找映射规则。
4. 查询映射规则中是否存在匹配记录。
    - 是 => 步骤[7](#ZH-CN_CONCEPT_0166338518__step7)
    - 否 => 步骤[5](#ZH-CN_CONCEPT_0166338518__step5)
5. 查询映射规则中是否存在“原因值范围”是“DEFAULT”的记录。
    - 是 => 步骤[7](#ZH-CN_CONCEPT_0166338518__step7)
    - 否 => 步骤[9](#ZH-CN_CONCEPT_0166338518__step9)
6. MM/SM流程控制参数中是否设置了拒绝原因值。
    - 是 => 步骤[8](#ZH-CN_CONCEPT_0166338518__step8)
    - 否 => 步骤[10](#ZH-CN_CONCEPT_0166338518__step10)
7. 使用[**ADD CAUSEMAP**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/原因值管理/原因值映射配置/增加原因值映射配置(ADD CAUSEMAP)_26145490.md)命令中配置的“目标原因值”。
8. 使用MM/SM流程控制参数中设置的拒绝原因值。
9. 使用按系统默认规则进行映射的原因值。
10. 使用系统默认原因值。
