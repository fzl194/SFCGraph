#### [命令功能](#ZH-CN_CONCEPT_0000001550121594)

该命令用于查询OSPFv3接口配置。

#### [注意事项](#ZH-CN_CONCEPT_0000001550121594)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001550121594)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001550121594)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROCID | OSPFv3进程号 | 可选必选说明：可选参数<br>参数含义：OSPFv3进程号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～4294967295。<br>默认值：无 |
| AREAID | OSPFv3区域号 | 可选必选说明：可选参数<br>参数含义：OSPFv3区域号。<br>数据来源：本端规划<br>取值范围：IPv4地址类型。点分十进制格式。<br>默认值：无 |
| IFNAME | 接口名称 | 可选必选说明：可选参数<br>参数含义：接口名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。<br>默认值：无<br>配置原则：请使用LST INTERFACE命令查看可用接口。 |

```
LST OSPFV3INTERFACE:;
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000001550121594)

| 输出项名称 | 输出项解释 |
| --- | --- |
| OSPFv3进程号 | OSPFv3进程号。 |
| OSPFv3区域号 | OSPFv3区域号。 |
| 接口名称 | 接口名称。 |
| 使能MTU | 是否使能MTU。 |
| 接口网络类型 | 接口网络类型，包含P2P、P2MP、广播、NBMA、P2MP-NONBROADCAST、LOOPBACK。 |
| 邻居失效的时间（s） | OSPFv3邻居失效的时间。OSPFv3邻居的失效时间是指：在该时间间隔内，若未收到邻居的Hello报文，就认为该邻居已失效。OSPFv3协议不支持NULL接口的配置。 |
| 发送Hello包的时间间隔（s） | 接口发送Hello报文的时间间隔。hello interval的值写入Hello报文中后随之发送。hello interval的值越小，发现网络拓扑改变的速度越快，路由开销也就越大。确定接口和邻接路由器的参数要保持一致。OSPFv3协议不支持NULL接口的配置。 |
| 是否使能邻居失效定时器保守模式 | 是否使能邻居失效定时器保守模式。使能后，即使邻居失效时间小于10s，仍按照实际配置值来判断邻居是否失效。 |
| 重发LSA的时间间隔（s） | 接口重传LSA的时间间隔。当一台路由器向它的邻居发送一条＂链路状态广播＂（LSA）后，需要等到对方的确认报文。若在该重传LSA的时间间隔内未收到对方的确认报文，就会重传这条LSA。相邻路由器重传LSA时间间隔的值不要配置得太小，否则将会引起不必要的重传。OSPFv3协议不支持NULL接口的配置。 |
| 发送Hello包的Poll时间间隔（s） | NBMA网络上发送轮询Hello报文的时间间隔。在NBMA网络上，当邻居失效后，路由器将按配置的轮询时间间隔定期地发送Hello报文。轮询时间间隔值至少应为Hello报文时间间隔的4倍。OSPFv3协议不支持NULL接口的配置。 |
| 超时时间（s） | 等待超时时间。 |
| 接口开销值 | 接口开销值。 |
| 使能BFD | 是否使能BFD。 |
| 接收BFD包的最小时间间隔（ms） | 接收BFD包的最小时间间隔。 |
| 发送BFD包的最小时间间隔（ms） | 发送BFD包的最小时间间隔。 |
| 本地检测倍数 | 本地检测倍数。 |
| 阻塞接口动态创建BFD特性 | 阻塞接口动态创建BFD特性。 |
| 接口对LSA的传输延迟时间（s） | 接口对LSA的传输延迟时间。 |
| 指定路由器优先级 | 指定路由器优先级。 |
| 禁止该接口接收和发送OSPFv3报文 | 是否禁止该接口接收和发送OSPFv3报文。 |
| 实例号 | 实例号。 |
| OSPFv3共网段虚拟系统使能标志 | 是否使能OSPFv3共网段虚拟系统标志。 |
| OSPFv3虚拟路由器标识使能标识 | 是否配置OSPFv3虚拟路由器标识。 |
| OSPFv3虚拟路由器标识 | OSPFv3虚拟路由器标识。 |
| BFD会话与接口链路状态绑定标志 | 是否将BFD会话状态与接口的链路状态进行绑定，配置此参数后，BFD会话状态与此接口链路状态绑定（当BFD会话状态变为Down时，接口的链路状态也会变为Down），从而达到快速感知故障的目的。 |
| 阻止指定OSPFv3接口的FRR能力 | 是否阻止指定OSPFv3接口的FRR能力。 |
