## [操作场景](#ZH-CN_OPI_0185511003)

本操作指导介绍在运行网络中激活固定终端寻呼优化特性的操作过程。

固定终端是指固定位置接入的M2M（Machine To Machine）终端（如电子水表）。固定终端寻呼优化特性通过调整READY定时器增加固定终端Ready状态时长，延缓由Ready状态转为Standby状态，从而减少固定终端状态的变迁，减少寻呼开销。

> **说明**
> 适用于 SGSN 。

## [操作步骤](#ZH-CN_OPI_0185511003)

1. 打开本特性的License配置开关。
  进入 “MML命令行-UNC” 窗口。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
2. 配置需要优化的签约APN及READY定时器时长。
  进入 “MML命令行-UNC” 窗口。
  [**ADD APNCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/基于APN的MM拥塞控制_流控_寻呼优化配置/增加APN控制参数配置(ADD APNCTRL)_26145470.md)
  > **说明**
  > APN类型选择 “固定终端寻呼优化APN” 或者 “Both” 。

## [任务示例](#ZH-CN_OPI_0185511003)

任务描述

对APN为“huawei.com”的终端启用固定终端寻呼优化功能，设置其READY定时器为60s。

脚本

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2POFT01", SWITCH=ENABLE;
```

//配置需要优化的签约APN“huawei.com”，其READY定时器时长为60s。

```
ADD APNCTRL: SUBSCRIBEDAPN="huawei.com", APNTYPE=CALL_OPTIMIZE, READYTIMER=60;
```
