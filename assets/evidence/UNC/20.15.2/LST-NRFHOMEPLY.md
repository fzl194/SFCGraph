## [命令功能](#ZH-CN_MMLREF_0000001171516437)

**适用NF：NRF**

此命令用于查询NRF作为归属地NRF时跨PLMN请求的默认策略。

## [注意事项](#ZH-CN_MMLREF_0000001171516437)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001171516437)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001171516437)

无

## [使用实例](#ZH-CN_MMLREF_0000001171516437)

查询漫出默认策略。

```
%%LST NRFHOMEPLY:;%%
RETCODE = 0  操作成功

结果如下
------------------------
是否支持跨PLMN订阅  =  打开
      信息精简策略  =  NULL
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001171516437)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 是否支持跨PLMN订阅 | 该参数用于表示本NRF作为归属地NRF时，是否处理其他拜访地PLMN的NF发起的对本PLMN的NF订阅以及订阅更新请求。开关为FUNC_ON表示处理，开关为FUNC_OFF表示不处理，并返回403错误码。订阅更新请求仅最终处理的NRF会按照此开关进行控制，中间层NRF仅做透传处理。 |
| 信息精简策略 | 该参数用于表示NRF作为归属地NRF时，对于收到跨PLMN服务发现和通知时，服务发现结果和订阅通知中精简的NF Profile的字段信息。<br>选择某一项表示该项被精简，不在消息中携带，不选择表示不精简。 |
