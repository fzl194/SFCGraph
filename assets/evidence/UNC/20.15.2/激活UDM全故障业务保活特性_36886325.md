# 激活UDM全故障业务保活特性

- [操作场景](#ZH-CN_OPI_0000001236886325__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001236886325__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001236886325__1.3.3)
- [任务示例](#ZH-CN_OPI_0000001236886325__1.3.4)

## [操作场景](#ZH-CN_OPI_0000001236886325)

为提升UDM全故障场景下的业务连续性，可以激活本特性。UDM全故障业务保活特性需要在AMF和SMF侧分别开启UDM全故障业务保活特性。

> **说明**
> 适用于AMF、SMF。

## [必备事项](#ZH-CN_OPI_0000001236886325)

前提条件

- 请仔细阅读[WSFD-206101 UDM全故障业务保活特性概述](WSFD-206101 UDM全故障业务保活特性概述_90966326.md)。
- 完成加载License。

- AMF侧配置
    1. 打开本特性的License配置开关。
      **[**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)**

      **[SET AMFUDMBYPASS](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/可靠性管理/AMF的UDM故障BYPASS功能/设置AMF的UDM故障BYPASS功能（SET AMFUDMBYPASS）_26366865.md)**
    3. 增加UDM Bypass后的最小签约数据配置。
      **[ADD UDMBYPASSSUB](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/可靠性管理/UDM故障BYPASS最小签约数据配置管理/增加UDM Bypass最小签约数据配置（ADD UDMBYPASSSUB）_86856256.md)**
    4. （可选）设置AMF在UDM Bypass用户本地上下文不存在MSISDN场景下不允许新建PDU会话，以保证话单正确性。
      **[SET AMFSOFTPARAOFBIT](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/软件参数管理/设置AMF软件参数比特位（SET AMFSOFTPARAOFBIT）_09651487.md)**
    5. （可选）设置AMF在匹配用户当前安全上下文时，忽略TSC标识，解决由于手机TSC携带错误导致用户安全检查失败而无法接入的问题。

## [任务示例](#ZH-CN_OPI_0000001236886325)

任务描述

- 任务1：开启基础的UDM全故障业务保活特性，在AMF侧增加UDM Bypass时使用的最小签约数据，在SMF侧增加UDM Bypass时指定APN的QoS属性。
- 任务2：如果已明确AUSF或UDM全故障，运营商期望UDM/AUSF网元返回除5XX之外的错误码时也允许用户进入UDM Bypass状态，或者应急场景下UDM/AUSF网元返回指定的故障码或者故障码和错误信息组合时允许用户进入UDM Bypass状态，在任务1的基础上增加BYPASS故障状态码配置。

- 任务3：手动触发AMF上用户退出UDM Bypass状态。
- 任务4：手动触发SMF上会话退出UDM Bypass状态。

- AMF侧配置
  //开启AMF的UDM全故障特性license。

  //（可选）设置AMF在匹配用户当前安全上下文时，忽略TSC标识，解决由于手机TSC携带错误导致用户安全检查失败而无法接入的问题。

  ```
  SET AMFSOFTPARAOFBIT: DT=Dw, DWORDNUM=68, VALUE=VALUE_0, POSITION=5;
  ```

  //开启SMF的UDM全故障特性license。
  ```
  SET LICENSESWITCH: LICITEM="LKV2UDMBPSM01",SWITCH=ENABLE;
  ```
