#### [适用NF](#ZH-CN_TOPIC_0175704843)

MME

#### [定义](#ZH-CN_TOPIC_0175704843)

NAS（Non-Access-Stratum）协议是UE-MME之间的协议，支持核心网和UE之间业务和信令消息的传输。

NAS信令加密和完整性保护可以对NAS信令进行加密及完整性保护，提高系统的安全性。

#### [客户价值](#ZH-CN_TOPIC_0175704843)

| --- | --- |
| 运营商 | 保证NAS信令的安全性和可靠性。 |
| 用户 | 用户不感知该特性。 |

#### [应用场景](#ZH-CN_TOPIC_0175704843)

用户和网络交互过程中，启用本特性保护NAS信令的安全性和可靠性。

#### [原理概述](#ZH-CN_TOPIC_0175704843)

本节包括以下内容：

**NAS信令加密和完整性保护的相关概念**

在EPS网络中，UE与网络之间存在两种不同级别的安全联盟：

- UE与E-UTRAN之间的RRC（Radio Resource Control）和UP（User Plane）安全联盟。RRC安全联盟对UE与E-UTRAN之间的RRC信令进行完整性保护和加密。UP安全联盟对UE与E-UTRAN之间的用户面数据进行加密。
- UE与MME之间的NAS安全联盟。它提供NAS信令的加密与完整性保护。

**NAS信令加密和完整性保护的原理**

EPS安全架构，如 [图1](#ZH-CN_TOPIC_0175704843__fig_1) 所示，有相互独立的分层安全：

- MME和UE执行NAS信令加密和完整性保护。
- eNodeB和UE执行RRC信令加密和完整性保护，UP加密。

**图1** EPS安全架构

![](特性概述_75704843.assets/zh-cn_image_0175704853_2.png)

<br>

UNC 支持的NAS信令加密和完整性算法包括：空加密/空完整性算法、AES（Advanced Encryption Standard）、SNOW 3G、ZUC。

- 空加密/空完整性算法：空加密算法中的算法流程和其他算法相同，但加密前后的明文和密文是完全相同的。空完整性算法采用全0的32位消息鉴权码（MAC），使用过程与其他完整性算法相同，空完整性算法仅用于紧急呼叫中未通过鉴权的UE。
- SNOW 3G算法：SNOW 3G是一种基本的3GPP加密算法和完整性算法，对应算法包括EEA1（EPS Encryption Algorithm1）和EIA1（EPS Integrity Algorithm1），密钥长度为128位。
- AES算法：AES是目前世界上应用最为广泛的加解密和完整性算法，对应算法包括EEA2（EPS Encryption Algorithm2）和EIA2（EPS Integrity Algorithm2），密钥长度为128位。
- ZUC算法：ZUC算法(也称为祖冲之算法)是一个面向硬件设计的序列密码算法。该算法根据128比特的初始密钥和128比特的初始向量，输出32比特的密钥序列，用以对数据进行加密和完整性保护。UNC支持ZUC加密算法。

上述四种算法的优先级可以通过命令 [**ADD S1ALGPRIORITY**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/S1模式用户安全参数/增加S1模式加密和完整性算法优先级配置信息(ADD S1ALGPRIORITY)_26305462.md) 配置，系统会按配置的算法优先级和UE进行协商，最终确定采用的加密/完整性算法。

> **说明**
> 在EPS网络中，至少需要开通AES或SNOW 3G中的一种，以保证网络中的消息都是经过加密和完整性保护的。

**NAS信令加密和完整性保护的流程**

[图2](#ZH-CN_TOPIC_0175704843__fig_2) 是NAS安全模式协商流程。

**图2** NAS信令加密和完整性保护的流程

<br>

![](特性概述_75704843.assets/zh-cn_image_0175704856_2.png)

过程描述如下:

1. MME根据手机Attach或TAU带上来的支持算法（即UE的安全能力）和MME自身配置的支持的算法进行算法协商。根据协商好的算法ID和算法类型进行加密密钥和完整性密钥计算。
2. MME向手机发送NAS Security Mode Command消息告知手机网络所选择的算法。消息中主要带有：Replayed UE Security Capability（返回给手机的安全能力，手机用来判断是否被攻击者篡改过）、ENEA（协商好了的加密算法ID），ENIA（协商好了的完整性保护算法ID），KSI。
  > **说明**
  > 该消息已经用选择了的完整性保护算法进行了完整性保护。
3. 手机收到NAS Security Mode Command消息以后，会先进行完整性检查。完整性检通过后，将比较返回的手机安全能力，如果安全能力匹配，手机返回NAS Security Mode Complete消息给MME。
  > **说明**
  > 该消息已经进行了加密和完整性保护了。
4. MME收到NAS Security Mode Complete消息以后进行完整性检查和解密，得知UE已经和MME达成一致。
5. NAS Security Mode Command流程成功之后对所有下发的NAS消息都将启动加密和完整性保护，对所有上来的NAS消息根据消息中的安全头进行完整性检查和解密。
