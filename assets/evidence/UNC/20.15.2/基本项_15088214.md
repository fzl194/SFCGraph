#### [82208006 LKV3W9V6PD11 IPv6 前缀代理](#ZH-CN_TOPIC_0000001615088214)

| **适用NF** | GGSN-C |
| --- | --- |
| **功能描述** | 支持为MS分配IPv6 Prefix Delegation地址段（前缀长度大于等于49位且小于64位），让IPv6 MS作为移动路由器为IPv6终端从Prefix Delegation地址段中分配64位前缀，并提供网络接入服务。 |
| **实现描述** | 系统中每激活一个IPv6 Prefix Delegation用户的PDP上下文，IPv6 Prefix Delegation的总数加一；每去激活一个IPv6 Prefix Delegation用户的上下文，IPv6 Prefix Delegation的总数减一。<br>如果系统中已接入的支持IPv6 Prefix Delegation功能的PDP上下文数达到License中“IPv6 Prefix Delegation”的最大值，新的IPv6 Prefix Delegation用户PDP上下文无法激活成功。 |
| **取值范围** | 0～16000000 PDP |
| **默认值** | 10 |
| **对应特性** | WSFD-104004 IPv6 前缀代理 |
| **相关控制项** | 依赖<br>WSFD-104001 IPv6承载上下文<br>WSFD-104002 IPv4v6双栈接入 |
| **应用场景** | 我们知道，IPv4用户支持手机后路由功能，即可以为MS分配一个网段，MS作为一个小的路由器，下面再挂多个终端设备。而IPv6 Prefix Delegation特性就是针对Ipv6 用户的这种需求：使Ipv6的MS既可以作为小型的路由器（比如家庭的路由器），又可以作为企业的路由器。这样就对IPv6的前缀长度要求是可变的。<br>因为Ipv6的地址资源较多，所以这个功能在实际应用中比较常用。 |
