#### [紧急号码和紧急呼叫能力下发流程](#ZH-CN_TOPIC_0000001089832510)

UE注册到5G网络时，UE通过标准注册流程接入后，5G核心网会提供当前网络的紧急呼叫能力以及支持的紧急呼叫号码。当UE需要拨打紧急呼叫时，通过紧急Service Request请求消息，要求核心网触发EPS Fallback流程将其回落到LTE网络。当UE成功回落到LTE网络后，会发起紧急PDN连接建立流程，并通过紧急PDN连接执行紧急语音通话。

在注册流程中，无论是初始注册、移动注册更新、周期性注册流程中（目前AMF拒绝任何紧急注册流程），AMF会根据本地配置策略决策紧急呼叫号码列表和紧急呼叫能力，并在Registration Accept中发送给UE。UE收到后需要将紧急呼叫号码列表保存在本地，以便发起紧急呼叫时能正确识别。AMF在Registration Accept中携带的紧急呼叫能力可以为：

- 不允许紧急呼叫业务，表示该AMF不支持紧急呼叫。
- NR接入的紧急呼叫，表示AMF支持NR接入5GC的紧急呼叫业务，并直接使用5G网络进行紧急呼叫。
- NR接入的紧急呼叫回落，表示AMF支持从NR接入5GC的紧急呼叫业务回落到EPS。UE在有紧急呼叫需求时，会发送Service Request触发紧急EPS FallBack流程，回落到LTE网络进行紧急呼叫。

> **说明**
> 周期性注册流程，AMF只下发紧急呼叫能力，不携带紧急呼叫号码。

#### [EPS Fallback紧急呼叫流程](#ZH-CN_TOPIC_0000001089832510)

**图3** 紧急呼叫EPS Fallback流程图

<br>

![](实现原理_89832510.assets/zh-cn_image_0000001089834290_2.png "点击放大")

UE注册到5G网络后已建立IMS PDU会话，并且已在IMS网络上注册成功。

AMF在Registration Accept中携带的紧急呼叫能力为“支持从NR接入5GC的紧急呼叫业务回落到EPS”。该紧急呼叫业务受 **[SET NGMMFUNC](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动性管理/设置5G移动性管理功能（SET NGMMFUNC）_09653748.md)** 参数 “EMG” 和 **[ADD NGCONNECTPLMN](../../../../../OM参考/命令/UNC MML命令/业务服务管理/运营商管理/互联PLMN管理/增加5G Connect PLMN（ADD NGCONNECTPLMN）_09651402.md)** 参数 “EMG” 控制，且同时开启才能生效。

1. 当UE拨打紧急呼叫电话时，触发紧急EPS FallBack流程，UE在Service Request消息中携带的service type值为emergency services fallback（普通语音的EPS Fallback是由基站发起的）。
2. AMF会根据UE当前的CM State（CM-IDLE或CM-CONNECTED）来决定使用Initial Context Setup Request还是UE Context Modification Request来通知NG RAN触发5G到4G的重选或者切换流程。
    - 如果UE当前的CM State是CM-IDLE，AMF使用Initial Context Setup Request来通知NG RAN触发5G到4G的重选流程，AMF会收到Initial Context Setup Response。 AMF继续当前的EPS FallBack流程，并等待MME发送的Context Request消息。
      > **说明**
      > AMF有可能在该步骤收到的是Initial Context Setup Failure，因为UE即将重选到LTE网络，此场景下的 **基站并不需要真正建立初始上下文** 。
    - 如果UE当前的CM State是CM-CONNECTED，AMF使用UE Context Modification Request来通知NG RAN触发5G到4G的切换流程，AMF会收到UE Context Modification Response消息。AMF继续当前的EPS FallBack流程，并等待基站发送的Handover Required消息。
      > **说明**
      > AMF有可能在该步骤收到的是UE Context Modification Failure，因为UE即将切换到LTE网络，此场景下的 **基站并不需要真正修改UE上下文** 。
3. UE根据基站的决策结果发起5G到4G的重选或者切换流程。
4. UE在4G网络建立到紧急APN的PDN链接，默认承载QCI=5。
  > **说明**
  > 如果命令 **[SET POLICYMODE](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/基本功能/策略模式选择/设置策略接口的选择方式（SET POLICYMODE）_09653658.md)** 的 “FORCED” 参数配置为 “Gx（Gx接口）” ，通过CCR/CCA进行策略更新。
  >
  > 如果命令 **[SET POLICYMODE](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/基本功能/策略模式选择/设置策略接口的选择方式（SET POLICYMODE）_09653658.md)** 的 “FORCED” 参数配置为 “Npcf（Npcf接口）” ，通过Npcf_SMPolicyControl_Create Request/Response进行策略更新。
  >
  > 系统初始设置值为 “Gx（Gx接口）” 。
5. INVITE信令通过紧急PDN的默认承载发送给IMS网络，触发4G紧急呼叫的语音专载建立。
6. SIP信令通过默认承载传输，语音数据包通过语音专载进行传输，从而进行语音通话。

#### [EPS Fallback和EPS Fallback 紧急呼叫差异点](#ZH-CN_TOPIC_0000001089832510)

*表1 EPS Fallback和EPS Fallback 紧急呼叫差异点*

| 差异点 | EPS Fallback 紧急呼叫流程 | EPS Fallback普通呼叫流程 |
| --- | --- | --- |
| 注册流程 | 注册流程中，AMF携带本地配置的紧急呼叫号码列表和紧急呼叫能力。 | 5G网络上完成正常的注册流程。 |
| 用户状态 | CM-CONNECTED、CM-IDLE | CM-CONNECTED |
| 回落方式 | 重选、切换 | 切换、重定向 |
| 拨号过程 | UE拨打紧急号码后，UE发消息给AMF，<br>**AMF通知NG RAN触发重选或者切换**<br>。 | UE拨打电话后，UE发消息给IMS，IMS发起语音会话建立，<br>**NG RAN拒绝语音会话建立后**<br>，再触发切换或者重定向。 |
| 承载 | QCI=1的语音专载，另外UE在4G网络建立到紧急APN的PDN链接，<br>**信令通过紧急PDN的默认承载**<br>发送给IMS网络。 | QCI=1的语音专载。 |
