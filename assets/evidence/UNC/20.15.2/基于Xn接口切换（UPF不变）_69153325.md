# 基于Xn接口切换（UPF不变）

- [业务场景](#ZH-CN_TOPIC_0000001469153325__1.3.1.1)
- [信令流程](#ZH-CN_TOPIC_0000001469153325__1.3.2.1)

#### [业务场景](#ZH-CN_TOPIC_0000001469153325)

Handover流程用于将UE从一个源NG-RAN节点切换到一个目标NG-RAN节点，切换过程中使用Xn接口，基于Xn接口的NG-RAN间的Handover流程只发生在当前业务所在AMF的管理范围内。新的无线条件、负载均衡或特定的服务都可能触发切换流程，例如UE在AMF的管理范围内并处于连接态时，移动出了原有NG-RAN的服务范围，导致UE需要新的NG-RAN提供服务，此时会触发Handover流程。

该流程用于源NG-RAN与目标NG-RAN存在有Xn接口，当AMF和UPF不改变时发起的切换场景。

1. 交互流程：切换准备&切换开始。
  相关机制：源NG-RAN会根据目标NG-RAN所需要的消息格式提供必要的信息，并传输给目标NG-RAN。目标NG-RAN准备无线侧资源。
2. 消息交互：Target NG-RAN发送[PATH SWITCH REQUEST](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PATH SWITCH REQUEST_76784993.md)给AMF。
  关键信元：
  | User Location Information | 该IE用于提供UE的位置信息。 |
  | --- | --- |
  | UE Security Capabilities | 该信元用于携带UE的5G安全能力。 |
  | **PDU Session Resource**<br>**to be Switched in Downlink List** | 该信元用于携带需要切换的PDU会话列表，每个PDU会话包含会话ID和pathSwitchRequestTransfer。<br>pathSwitchRequestTransfer中包括目标gNB分配的N3接口AN隧道信息。 |
  | **PDU Session Resource Failed to Setup List** | 该信元用于携带需要拒绝的PDU会话列表 |
  相关机制：目标NG-RAN通知AMF用户已经移动到新的区域，并且提供需要切换的PDU会话列表。如果PDU会话的QoS流不被目标NG-RAN所接受或者目标NG-RAN不支持相应网络切片情况下，消息中会携带需要拒绝的PDU会话列表。
3. 消息交互：AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  关键信元：
  | ueLocation | 如果有UE的位置信息，该IE应包含UE的位置信息。 |
  | --- | --- |
  | n2SmInfoType | N2 SM消息类型，取值为PATH_SWITCH_REQ表示路径切换请求传输。 |
  相关机制：AMF携带NG-RAN侧的N2消息，并传递AN隧道信息。
4. 消息交互：SMF发送[PFCP Session Modification Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)给UPF。
  关键信元：
  | Update FAR | 修改先前为PFCP会话创建的FAR，并传递AN隧道信息。 |
  | --- | --- |
  | PFCPSMReq-Flags | 取值为SNDEM表示发送结束标记报文。 |
  相关机制：SMF携带AN隧道信息给UPF，并通知UPF丢弃原始的Data Notification消息或者不再发送后续Data Notification消息。如果SMF分配了新的CN隧道信息，则提供给UPF。
5. 消息交互：UPF发送[PFCP Session Modification Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)给SMF。
  关键信元：
  | Cause | 该信元应指示接受或拒绝相应的请求消息。 |
  | --- | --- |
  | Create PDR | 该信元应包含与PFCP会话相关的PDR信息，可能存在多个该信元的实例。 |
  相关机制：UPF通知SMF PDU会话切换完成，并且将CN隧道信息携带给SMF。如果有PDU会话需要去激活，UPF释放掉N3接口AN隧道信息。为了保证目标NG-RAN数据包的顺序不混乱，在路径切换后，UPF立即通过老的路径发送一个或多个“End Marker”数据包给源侧NG-RAN，源侧NG-RAN再将数据包转发给目标NG-RAN。下行数据包可以直接通过目标NG-RAN发送给UE。
6. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给AMF。
  关键信元：
  | N2SmInfoType | N2 SM消息类型，取值为PATH_SWITCH_REQ_ACK表示路径切换请求确认传输。 |
  | --- | --- |
  相关机制：SMF将CN隧道信息传递给AMF。
7. 消息交互：AMF发送[PATH SWITCH REQUEST ACKNOWLEDGE](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PATH SWITCH REQUEST ACKNOWLEDGE_76785522.md)给Target NG-RAN。
  关键信元：
  | **PDU Session Resource**<br>**Switched List** | PDU会话资源切换列表。<br>包括PDU会话ID以及SMF通过AMF透传给NG-RAN的Path Switch Request Acknowledge Transfer IE。 |
  | --- | --- |
  | **PDU Session Resource**<br>**Released List** | PDU会话资源释放列表。<br>包括PDU会话ID以及SMF通过AMF透传给NG-RAN的Path Switch Request Unsuccessful Transfer IE。 |
  相关机制：携带N2 SM Information以及Failed PDU Sessions。
8. 交互流程：切换确认及资源释放。
  相关机制：目标NG-RAN通过发送Release Resources message给源NG-RAN，确认切换流程成功，触发源侧释放资源。
9. 子流程：[移动注册更新](../注册流程/移动注册更新_69433041.md#ZH-CN_TOPIC_0000001469433041)。
  相关机制：如果触发移动注册更新条件，则UE会发起移动性注册更新流程。
