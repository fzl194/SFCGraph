- [信令流程](#ZH-CN_TOPIC_0194229086__1.3.1.1)

#### [信令流程](#ZH-CN_TOPIC_0194229086)

PFC功能相关的流程有PFC创建、PFC修改、PFC删除流程。

- PFC创建流程
  **图1** PFC创建流程

  <br>

  ![](实现原理_94229086.assets/zh-cn_image_0194244337_2.png)
  PFC创建流程如 [图1 PFC创建流程](#ZH-CN_TOPIC_0194229086__fig02) 所示，流程详细介绍如下：
    1. 如果在上下行数据传输过程中，BSS上没有对应的PFC上下文，则BSS可以向SGSN发送Download BSS Packet Flow Context Request消息，请求下载一个BSS PFC，触发SGSN发起PFC创建流程。
    2. 如果BSS和MS都支持PFC，则SGSN在激活PDP的过程当中可以发起PFC创建流程，SGSN向BSS发送Create BSS Packet Flow Context Request消息。
    3. BSS创建一个PFC并将参数保存到BSS上下文后，返回Create BSS Packet Flow Context Accept消息给SGSN，消息中携带IMSI，Packet Flow Id，Aggregate BSS QoS Profile Negotiated等信元。
  > **说明**
  > UNC 通过Create BSS Packet Flow Context Request消息设置ARP值传递给BSS。针对ARP的传递策略， UNC 实现了可以为特定IMSI范围用户、全局用户配置差异化的ARP策略。当需要对特定IMSI范围用户配置ARP策略时，需要先购买 [WSFD-106303 基于ARP的差异化服务](../../差异化服务管理/WSFD-106303 基于ARP的差异化服务_84683751.md) 的License。
- PFC修改流程
  **图2** BSS发起的PFC修改流程

  <br>

  ![](实现原理_94229086.assets/zh-cn_image_0194244276_2.png)

  <br>
  PFC的修改流程，可以由BSS发起，也可以由SGSN发起。在PFC修改流程中可实现MS、BSS和SGSN上PFC信息的同步修改。BSS发起的PFC修改流程如 [图2 BSS发起的PFC修改流程](#ZH-CN_TOPIC_0194229086__fig03) 所示，流程详细介绍如下：
    1. BSS向SGSN发送Modify BSS Packet Flow Context Request消息，消息中携带IMSI，Packet Flow Id，Aggregate BSS QoS Profile Negotiated等信元。
    2. SGSN返回Modify BSS Packet Flow Context Accept消息给BSS，消息中携带IMSI，TLLI，Packet Flow Id，Aggregate BSS QoS Profile Negotiated，BSS Packet Flow Timer等信元，BSS将修改的信息插入BSS上下文。
- PFC删除流程
  **图3** SGSN发起的PFC删除流程

  <br>

  ![](实现原理_94229086.assets/zh-cn_image_0214961750_2.png)
  PFC删除流程，可以由BSS发起，也可以由SGSN发起。SGSN发起的PFC删除流程如 [图3 SGSN发起的PFC删除流程](#ZH-CN_TOPIC_0194229086__fig04) 所示，流程详细介绍如下：
    1. SGSN向BSS发送Delete BSS Packet Flow Context Request消息，消息中携带TLLI，Packet Flow Id信元，BSS删除BSS上下文中相应的BSS PFC信息。
    2. BSS返回Delete BSS Packet Flow Context Accept消息给SGSN，消息中携带TLLI，Packet Flow Id信元。
