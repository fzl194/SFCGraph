## [命令功能](#ZH-CN_MMLREF_0296805489)

**适用NF：NRF**

该命令用于查询NRF大小包协同返回功能相关参数。

## [注意事项](#ZH-CN_MMLREF_0296805489)

无

#### [操作用户权限](#ZH-CN_MMLREF_0296805489)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0296805489)

无

## [使用实例](#ZH-CN_MMLREF_0296805489)

当运营商希望查询NRF大小包协同返回功能相关参数时，执行此命令。

```
LST NRFFCPKGMIX:;
%%LST NRFFCPKGMIX:;%%
RETCODE = 0  操作成功
结果如下
-------------------------
    大小包协同返回功能开关  =  打开
     大包比例递减周期 (秒)  =  3
大包比例递减的公比的百分比  =  90
     大包比例抑制时长 (秒)  =  360
   恢复大包比例的周期 (秒)  =  6
  大包比例恢复的公差百分比  =  3
       带宽速率上限 (MB/s)  =  25
     解除带宽控制周期 (秒)  =  120
(结果个数 = 1)
```

## [输出结果说明](#ZH-CN_MMLREF_0296805489)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 大小包协同返回功能开关 | 该参数用于表示大小包协同返回功能开关。该参数为FUNC_ON时，表示该功能打开；该参数为FUNC_OFF时，表示该功能关闭。按InstanceID的发现不受此开关控制。 |
| 大包比例递减周期 (秒) | 该参数用于表示NRF流控期间大包比例递减周期。在流控期间，以该参数（秒）为周期，每次按公比RATIO计算出最新的大包比例。例如这个参数设置为2，RATIO设置为95，表示NRF服务发现返回的大包比例每2秒按95%递减。 |
| 大包比例递减的公比的百分比 | 该参数用于表示大包比例递减的公比的百分比，在流控期间，每个递减周期依据此参数对服务发现返回的大包比例等比递减。比例接近于0（小于1%）之后，就直接按100%的小包返回。例如这个参数设置为95，FCPERIOD设置为2，表示NRF服务发现返回的大包比例每2秒按95%递减。 |
| 大包比例抑制时长 (秒) | 该参数用于表示解控后服务发现返回的大包比例抑制时间，在此时间内大包比例不变，超过此时间后，大包比例按照配置恢复。 |
| 恢复大包比例的周期 (秒) | 该参数用于表示恢复服务发现返回的大包比例的周期，每过一个周期按公差增加大包比例。例如这个参数设置为5，TOLENRANCE设置为2，表示NRF服务发现返回的大包比例每5秒按2%递增。 |
| 大包比例恢复的公差百分比 | 该参数用于表示在解控期间，恢复服务发现返回的大包比例的公差百分比，每个恢复周期固定增加该参数的大包比例。例如这个参数设置为2，RECPERIOD设置为5，表示NRF服务发现返回的大包比例每5秒按2%递增。 |
| 带宽速率上限 (MB/s) | 该参数用于表示单进程的服务发现返回结果的带宽速率上限，带宽速率表示每秒服务发现返回的包长总和。带宽速率设置为0时表示带宽速率不受限制。 |
| 解除带宽控制周期 (秒) | 该参数用于表示解除带宽控制周期。在解除流控且大包的比例恢复为100%之后，此周期内的平均带宽速率（平均带宽速率为10s带宽的平均值）没有超过“带宽速率上限”，就解除带宽控制；如果超过，将重新监控一个周期。 |
