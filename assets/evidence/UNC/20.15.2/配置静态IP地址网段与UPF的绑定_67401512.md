# 配置静态IP地址网段与UPF的绑定

- [操作场景](#ZH-CN_OPI_0167401512__1.3.1)
- [必备事项](#ZH-CN_OPI_0167401512__1.3.2)

## [操作场景](#ZH-CN_OPI_0167401512)

可以在本地配置静态IP地址网段与UPF的绑定关系，当用户接入网络时，对静态IP地址用户、Radius分配IP地址用户可以基于配置的网段绑定关系为用户选择指定的UPF。

> **说明**
> 适用于 GGSN、PGW-C、SMF 。

## [必备事项](#ZH-CN_OPI_0167401512)

前提条件

- 请仔细阅读[WSFD-010502 地址分配方式特性概述](WSFD-010502 地址分配方式特性概述_64915075.md)。
- 本特性与UDG配合使用生效，要求UNC上配置的用户静态地址网段信息必须在包含在UDG上配置的地址池内，请参考UDG中的**GWFD-010104 地址分配方式**。

## [操作步骤](#ZH-CN_OPI_0167401512)

1. 进入 “MML命令行-UNC” 窗口。
2. 创建本地静态地址地址池。
  [**ADD ADDRPOOL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/UE地址池管理/地址池管理/增加地址池（ADD ADDRPOOL）_09653289.md)
3. 如果需要开启静态地址用户的白名单检查功能，需要配置地址池与APN的绑定关系。接入该APN的静态地址用户激活时，根据用户地址查找对应的地址池，然后根据地址池为用户选择对应的UPF。
  **[ADD POOLBINDAPN](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/UE地址池管理/地址池绑定APN/增加APN实例与地址池绑定关系（ADD POOLBINDAPN）_09653789.md)**
4. 配置本地地址池里的地址段。
  [**ADD SECTION**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/UE地址池管理/地址段管理/增加地址段（ADD SECTION）_09651691.md)
  ![](配置静态IP地址网段与UPF的绑定_67401512.assets/notice_3.0-zh-cn_2.png)

  - 本地地址池：本地负责分配地址的地址池。本地地址池下需要配置地址段，一个本地地址池最多可以配置64个地址段。
    - 配置地址池后，必须配置地址段。如果不配置地址段，将导致用户激活失败。
5. 配置地址池组名称。
  **[ADD ADDRPOOLGRP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/UE地址池管理/地址池组管理/增加地址池组（ADD ADDRPOOLGRP）_32232812.md)**
6. 配置地址池与地址池组绑定。
  **[ADD POOLBINDGRP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/UE地址池管理/地址池组管理/增加地址池和地址池组的绑定关系（ADD POOLBINDGRP）_32232813.md)**
7. 配置静态地址用户使用的UPF。
  **[ADD UPNODE](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/PFCP接口管理/UP管理/UP节点管理/增加UPF节点（ADD UPNODE）_09652571.md)**
8. 配置静态地址用户使用的UPF组。
  **[ADD ADDRUPGROUP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/UE地址池管理/地址分配UPF组管理/增加UPF组（ADD ADDRUPGROUP）_49644911.md)**
9. 配置UPF与UPF组绑定。
  **[ADD UPFBINDGRP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/UE地址池管理/地址分配UPF组管理/增加UPF和UPF组的绑定关系（ADD UPFBINDGRP）_32232815.md)**
10. 配置地址池组与UPF组绑定。
  **[ADD POOLGRPMAP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/UE地址池管理/地址池组映射配置/增加地址池组映射关系（ADD POOLGRPMAP）_32232814.md)**
  > **说明**
  > - 会话建立流程中，当静态IP地址网段绑定的UPF与SMF选择的主锚点UPF不一致时，优先使用SMF选择的主锚点UPF建立会话。
  > - 如果已有会话的主锚点UPF与静态IP地址绑定的UPF不一致时，可以根据**[SET STATICADDRPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UE地址管理/静态地址参数配置/设置静态地址参数（SET STATICADDRPARA）_32233051.md)**命令中的开关配置时间扫描会话去活。

## [任务示例](#ZH-CN_OPI_0167401512)

任务描述

某企业用户接入网络时，为用户分配的IP地址采用静态IP地址分配方式。开启静态地址用户的白名单检查功能，需要为10.0.0.0~10.1.2.3地址段内的用户选择到UPF名称为UPF_Instance_1的园区UPF。UPF_Instance_1上已经配置包含网段10.0.0.0~10.1.2.3的地址池信息，需要在 UNC 上配置静态IP地址段10.0.0.0~10.1.2.3到UPF_Instance_1的关系，为用户选择到指定UPF。

脚本

//创建本地地址池。

```
ADD ADDRPOOL: POOLNAME="pool01", IPVERSION=IPv4, POOLTYPE=UDM, CHECKIPVALID=Enable;
```

//配置地址池与APN的绑定关系。

```
ADD POOLBINDAPN:APN="huawei.com",POOLNAME="pool01";
```

//配置本地地址池里的IPv4地址段。

```
ADD SECTION: POOLNAME="pool01", SECTIONNUM=1, IPVERSION=IPv4, V4STARTIP="10.0.0.0", V4ENDIP="10.1.2.3";
```

//配置地址池组名称。

```
ADD ADDRPOOLGRP: POOLGRPNAME="poolgroup1", POOLGRPTYPE=UDM;
```

//配置地址池与地址池组绑定。

```
ADD POOLBINDGRP: POOLGRPNAME="poolgroup1", POOLNAME="pool01";
```

//配置静态地址用户使用的UPF。

```
ADD UPNODE: NFINSTANCENAME="UPF_Instance_1", ADDRALLOCMODE=INHERIT;
```

//配置静态地址用户使用的UPF组。

```
ADD ADDRUPGROUP: UPFGRPNAME="upfgroup1", UPFGRPTYPE=UDM;
```

//配置UPF与UPF组绑定。

```
ADD UPFBINDGRP: UPFGRPNAME="upfgroup1", UPFID="UPF_Instance_1";
```

//配置地址池组与UPF组绑定。

```
ADD POOLGRPMAP: MAPPINGNAME="mapping01", POOLGRPNAME="poolgroup1", UPFGRPNAME="upfgroup1";
```
