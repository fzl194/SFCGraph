# 实现原理

本特性主要实现了在切换等信令流程中MME根据Home eNB ID或TAI查询目标侧HeNB GW的功能。

在 **eNB status transfer** 、 **eNB direct information transfer** 、 **S1-based handover** 、 **UTRAN Iu mode to E-UTRAN Inter RAT handover** 、 **eNB configuration transfer** 、 **direct information transfer** 、 **RIM-PDU-TRANSFER.req** 7个流程的相关消息会携带目标侧Home eNB ID给MME，用于指示系统查找目标侧HeNB，各流程中携带Home eNB ID的消息如 [表1](#ZH-CN_CONCEPT_0166292249__table_henb) 所示，MME对HeNB GW的查询机制如 [图1](#ZH-CN_CONCEPT_0166292249__fig1) 所示。图中根据TAI查找20位HeNB的归属HeNB GW的功能可通过命令 [**SET MMFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/MM扩展功能管理/设置移动性管理扩展功能(SET MMFUNC)_26145512.md) 中的参数 “是否支持对20bit长HeNB的TAI寻址” 配置。

*表1 流程与携带Home eNB ID的消息对应关系*

| 流程 | 携带Home eNB ID的消息 |
| --- | --- |
| eNB status transfer | eNB status transfer |
| eNB direct information transfer | eNB direct information transfer |
| S1-based handover | Handover Required |
| UTRAN Iu mode to E-UTRAN Inter RAT handover | Relocation Required |
| eNB configuration transfer | eNB configuration transfer |
| direct information transfer | direct information transfer |
| RIM-PDU-TRANSFER.req | RIM-PDU-TRANSFER.req |

以 **direct information transfer** 流程为例，假设Direct Information Transfer消息指示的目标eNodeB为HeNB，且该HeNB通过HeNB GW接入，如 [图2](#ZH-CN_CONCEPT_0166292249__fig03) 所示。

1. 源侧从Direct Information Transfer消息中解析到的目的HeNB的ID。
2. 按照[图1](#ZH-CN_CONCEPT_0166292249__fig1)所示的策略查找目的HeNB GW。
    - 若查找成功，源侧向目的HeNB连接的HeNB GW发送MME Direct Information Transfer消息。
    - 若查找失败，源侧发起inter流程，通过RAN Information Relay消息将Home eNB ID携带给目标侧，若目标侧查找到目的HeNB GW，则向目的HeNB连接的HeNB GW发送MME Direct Information Transfer消息。
