# 调测固定终端寻呼优化特性

- [操作场景](#ZH-CN_OPI_0185511004__1.3.1)
- [必备事项](#ZH-CN_OPI_0185511004__1.3.2)
- [操作步骤](#ZH-CN_OPI_0185511004__1.3.3)

## [操作场景](#ZH-CN_OPI_0185511004)

UNC 通过固定终端寻呼优化机制减少固定终端频繁收发数据带来的寻呼消耗。本操作介绍了通过调测手段检查 UNC 是否正确配置该功能的过程。

> **说明**
> 适用于 SGSN 。

## [必备事项](#ZH-CN_OPI_0185511004)

前提条件

- 请仔细阅读[WSFD-206004 固定终端寻呼优化特性概述](特性概述_85511001.md)。
- 完成[激活固定终端寻呼优化特性](激活固定终端寻呼优化特性_85511003.md)。

数据

该操作无需准备数据。

工具

OM Portal跟踪。

## [操作步骤](#ZH-CN_OPI_0185511004)

- 检查固定终端寻呼优化功能配置。
  进入 “MML命令行-UNC” 窗口。
  [**LST APNCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/基于APN的MM拥塞控制_流控_寻呼优化配置/查询APN控制参数配置(LST APNCTRL)_72345069.md) :;
  ```
  %%LST APNCTRL: SUBSCRIBEDAPN="HUAWEI0.CN";%%
  RETCODE = 0  操作成功。

  操作结果如下
  ------------
                            签约APN  =  HUAWEI0.CN
                            APN类型  =  固定终端寻呼优化APN
                          APN优先级  =  NULL
              Backoff Timer分配开关  =  NULL
                     附着拒绝原因值  =  NULL
  识别异常附着行为的门限（次/小时）  =  NULL
        Ready Timer定时器时长（秒）  =  180
              T3324定时器时长（秒）  =  10800
              T3312定时器时长（分）  =  54
  (结果个数 = 1)
  ---    END
  ```
    - 预期结果：各项参数显示结果与配置结果一致。
    - 异常结果：各项参数显示结果与配置结果不一致或查看不到相关配置信息。
      参考 [激活固定终端寻呼优化特性](激活固定终端寻呼优化特性_85511003.md) ，查看配置是否正确。
- 调测固定终端寻呼优化业务。
    1. 创建用户跟踪。
      参数选择如下：
          - IMSI:被跟踪用户的IMSI。
          - 其它参数：选择默认值。
    2. 用户附着或RAU成功。
    3. 查看跟踪结果。
          - 预期结果：Attach accept或者RAU accept中能看到配置的READY定时器值，如 [图1](#ZH-CN_OPI_0185511004__zh-cn_opi_0130429012_fig1) 所示。
            **图1** 固定终端寻呼优化消息跟踪（样例）

            ![](调测固定终端寻呼优化特性_85511004.assets/zh-cn_image_0186524155_2.png "点击放大")

            <br>
          - 异常结果：Attach accept或者RAU accept中能看不到配置的READY定时器值。
            参考 [激活固定终端寻呼优化特性](激活固定终端寻呼优化特性_85511003.md) ，查看配置是否正确。
