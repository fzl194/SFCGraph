[**低速区域公网用户不占用专网技术实现**](#ZH-CN_CONCEPT_0168358250__ul1)

为了解决低速区域公网用户不占用专用网络资源问题，在低速区域建立了一个专网eNodeB群组，如果用户首次接入该群组内的某个eNodeB， UNC 就会为UE记录接入的时间。当UE发起业务流程时， UNC 计算当前时间与记录的时间的差值，作为UE在低速区域的驻留时长。

如果驻留时长超过 [**ADD ENBGP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/eNodeB管理/eNodeB群组管理/增加eNodeB群组(ADD ENBGP)_26145606.md) 命令设定的驻留时长（STICK_TIME）参数， UNC 将该UE标记为公网用户，并选择命令 [**ADD EXTRFSP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/RFSP管理/扩展RFSP策略管理/增加扩展RFSP策略组成员(ADD EXTRFSP)_26145536.md) 指定的RFSP ID（该RFSP ID代表公共网络频点有较高的优先级），下发给eNodeB，由eNodeB指示UE接入到公共网络，避免占用专用网络资源。

> **说明**
> 专网eNodeB群组由 UNC 通过命令 [**ADD ENBGP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/eNodeB管理/eNodeB群组管理/增加eNodeB群组(ADD ENBGP)_26145606.md) 和 [**ADD ENBGPMEM**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/eNodeB管理/eNodeB群组成员管理/增加eNodeB群组成员(ADD ENBGPMEM)_72225289.md) 定义。

上述处理过程中，存在三个问题：

1. 如果真正的高铁用户在低速区域停留时间超过设定的驻留时长，也会被迁出至公共网络，很难返回专用网络。
2. 由于无线网络覆盖问题，导致用户虽然长时间在低速区域内，但在同一指定专网区域和周边公网做乒乓切换，消耗网络资源。因为该类用户持续驻留时长无法达到阈值，MME不会下发SPID进行接入控制。
3. 用户被识别为公网用户后，UNC在下发RFSP ID指示eNodeB将公网用户从专用网络迁出至公共网络，但是由于无线网络覆盖原因，用户又重新接入到专用网络中，又因为该用户被标识为公网用户，UNC将再次下发RFSP ID将用户迁出至公共网络，造成用户在专用网络和公共网络之间乒乓切换。
  > **说明**
  > 称此问题中的公网用户为“乒乓低移动性用户”，其用户数可通过性能指标查看。

- 对于问题1， UNC 通过设置入口eNodeB群组识别出高铁用户，并延长高铁用户在低速区域专用网络的驻留时长，降低因为临时停车等场景迁出专用网络的风险。
  在MME覆盖边界专用网络区域和高铁站候车厅配置专网入口eNodeB群组，这两个区域是高铁用户进入低速区域的入口。如 [图1](#ZH-CN_CONCEPT_0168358250__fig_003) 所示。当用户经过该区域，通过 **Attach、TAU、Handover、Service Request流程** 从入口eNodeB群组接入时， UNC 将用户标记为高铁用户。该类用户进入到低速区域专网eNodeB群组时，其驻留时长的实际阈值延长为配置阈值的三倍。

- 对于问题2，将在公网和专网乒乓切换无法达到驻留时长阈值的用户标记为公网用户，并迁出至公共网络。此类用户识别条件为：如果在驻留时长（STICK_TIME）参数配置的时间里，用户接入同一低速检测区（专网eNodeB群组）的次数超过 [**ADD ENBGP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/eNodeB管理/eNodeB群组管理/增加eNodeB群组(ADD ENBGP)_26145606.md) 命令设定的接入次数（ACC_TIMES）参数， UNC 将该用户标识为公网用户，并下发RFSP ID指示eNodeB将用户迁出至公共网络。
- 对于问题3，为了避免已被标记的公网用户做乒乓切换，UNC需要进行RFSP ID下发抑制。在命令[**ADD EXTRFSP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/RFSP管理/扩展RFSP策略管理/增加扩展RFSP策略组成员(ADD EXTRFSP)_26145536.md)的设置的抑制周期（CTRL_PERIOD）里，UNC对某个用户下发RFSP ID的次数超过目标RFSP重发次数（RFSPSNDCUNT）参数时停止为该用户下发。

[**高速铁路用户驻留专用网络技术实现**](#ZH-CN_CONCEPT_0168358250__ul1)

为确保高铁用户驻留专用网络，在 UNC 中定义了一个专用网络小区所属的TA群组（该群组由命令 [**ADD TAGP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/跟踪区管理/跟踪区群组管理/增加TA群组(ADD TAGP)_26145580.md) 和 [**ADD TAGPMEM**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/跟踪区管理/跟踪区群组成员管理/增加TA群组成员(ADD TAGPMEM)_72225263.md) 定义）。在TAU流程触发时，在满足如下条件时， UNC 认为UE从专用网络TA群组中移出至公共网络：

- 上次接入TA为专用网络TA：TAU Request消息中last visited registered TAI信元携带的TA属于专用网络TA群组。
- 待接入TA为公共网络TA：AS（Access Stratum）消息Initial UE Message中携带的用户待接入的TA不属于专用网络TA群组。

如果 UNC 判断UE从专用网络中移出至公共网络，则为UE选择命令 [**ADD EXTRFSP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/RFSP管理/扩展RFSP策略管理/增加扩展RFSP策略组成员(ADD EXTRFSP)_26145536.md) 指定的RFSP ID（该RFSP ID代表了专用网络频点有较高的优先级），并下发给eNodeB，由eNodeB指示UE重新接入到专用网络。

> **说明**
> 有以下两种情况， UNC 不会为该用户下发RFSP ID指示其重新接入到专用网络。
>
> - 针对切换流程后伴随的TAU流程。原因是eNodeB通过移动速度识别该用户为公网用户，通过切换流程（伴随TAU）迁出专用网络，不应该再迁回专用网络。
> - 针对在低速区域驻留时长超过阈值而被迁出专网的TAU流程。原因是由于驻留时长被迁出的用户为公网用户，不应该再迁回专用网络。

开启该特性，需要确保用户的TA List中的所有的TA都属于专用网络或都属于公共网络，否则UE在由专用网络移出至公共网络时，不会触发TAU流程， UNC 无法重新指示其接入到专用网络。为解决上述问题， UNC 在Attach/TAU Accept流程中给UE下发TA List时，会过滤掉其中所有与UE当前所在TA归属不同网络的TA。

[**SPID传递过程**](#ZH-CN_CONCEPT_0168358250__ul1)

UNC 通过TAU和Service Request流程向eNodeB下发RFSP ID指示UE接入相应的网络。RFSP ID的传递场景如 [表1](#ZH-CN_CONCEPT_0168358250__table_010) 所示。

*表1 RFSP ID传递的场景*

| 场景 | 当...时 | 携带RFSP ID的消息为... |
| --- | --- | --- |
| 用户发起TAU流程且满足下述任何一个条件。<br>- 用户从专用网络中移出至公共网络。<br>- 用户在低速区域的驻留时长超过相应阈值。<br>- 用户在驻留时长参数时长内，连续接入同一eNodeB群组次数超过相应阈值。 | TAU request消息中“Active flag”信元值为“Bearer establishment requested”。 | Initial Context Setup Request |
| 用户发起TAU流程且满足下述任何一个条件。<br>- 用户从专用网络中移出至公共网络。<br>- 用户在低速区域的驻留时长超过相应阈值。<br>- 用户在驻留时长参数时长内，连续接入同一eNodeB群组次数超过相应阈值。 | TAU request消息中“Active flag”信元值为“No bearer establishment requested”，且<br>[**SET S1CMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1接口兼容性/设置S1接口兼容性(SET S1CMPT)_72345837.md)<br>中的“SPID”取值为“YES”。 | Downlink NAS Transport（TAU Accept） |
| 用户在低速区域发起Service Request流程且驻留时长超过相应阈值。 | 无。 | Initial Context Setup Request |
