## [操作场景](#ZH-CN_OPI_0000001111845318)

VoNR即Voice over NR，UE在NR覆盖下，采用VoNR技术直接在NR上拨打电话，专用QoS Flow也在NR上建立。通话过程中，如果UE移动到LTE网络覆盖下，EPC和5GC会负责把整个IMS DNN平滑到LTE中，并确保给终端分配的IP地址以及P-CSCF的地址不变，使得IMS并不感知。所以只要在NR上发起呼叫，无论通话中终端是否移动到LTE网络，均称为VoNR呼叫。

> **说明**
> 适用于AMF、SMF。

## [操作步骤](#ZH-CN_OPI_0000001111845318)

1. 验证UE语音的VoNR可实现。
    a. 在AMF的OM Portal上创建用户跟踪任务。5G用户开机注册。
    b. 查询用户上下文的用户状态信息。
      进入 “MML命令行-UNC” 窗口。 执行命令 **[DSP PDUSESSION](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/查询PDP上下文信息/显示PDU会话（DSP PDUSESSION）_09653215.md)** 查询用户在5G网络下的PDU会话。如果为IMS的PDU会话，则 “APN” 为 “ ims” 。
      预期结果：用户在5G网络有数据PDU会话和IMS的PDU会话。
    c. 查看AMF上用户跟踪。
      预期结果：
          - 用户附着到网络，AMF向UE返回Registration Accept消息中携带5GS network feature support信元，并对IMS voice PS session over 3GPP access indicator置位。
            ![](调测VoNR基础语音业务_11845318.assets/zh-cn_image_0000001158365293_2.png)
          - UE发起会话建立。PDU Session Establishment Accept消息中的Extended protocol configuration options信元包含P-CSCF地址或者地址列表。
            ![](调测VoNR基础语音业务_11845318.assets/zh-cn_image_0000001111845334_2.png)
          - 在5GC触发5QI=1的IMS语音数据流。AMF向NG RAN发送N2 PDUSession Resource Modification Request消息，通知NG RAN建立语音Qos Flow资源。
            fiveQI：指示业务质量的索引，5QI对应4G的QCI，此处取值为1。
            ![](调测VoNR基础语音业务_11845318.assets/zh-cn_image_0000001111685446_2.png)
          - 在呼叫中NG RAN与UE发起信令交互，将SMF需要发送给UE的PDU session ID、N1 SM Container（PDU Session Modification Command）消息转发至UE，请求UE更新QoS信息；同时NG RAN为语音QoS Flow分配空口资源。
            预期结果：NG RAN回复PDU Session Resource Modify Response携带成功建立QoS Flow的信息，同时通过Uplink NAS消息将UE返回的PDU Session Modification Complete消息透传给AMF。
    d. 语音业务执行成功。
      进入 “MML命令行-UNC” 窗口。 执行命令 **[DSP PDUSESSION](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/查询PDP上下文信息/显示PDU会话（DSP PDUSESSION）_09653215.md)** ，查询用户在5G网络下的PDU会话。
      预期结果：用户在5G网络有VoNR语音专载，存在5QI=1的QoS Flow。
    e. 若涉及4G/5G互操作流程，切换流程调测参见 [调测5G SA到LTE网络间切换](../../../../业务专题/5G Core 4_5G互操作解决方案/调测指导/调测5G SA到LTE网络间切换_01_10043.md) 。
2. 收集信息并寻求技术支持。
    a. 在 “MML命令行-UNC” 窗口上执行命令 将当前配置数据导出为MML脚本文件并保存。
    b. 收集并保存上述所有查询信息。
    c. 收集归纳所有信息并联系华为技术支持解决。
