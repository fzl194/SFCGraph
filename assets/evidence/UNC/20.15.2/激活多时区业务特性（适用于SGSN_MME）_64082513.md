## [操作场景](#ZH-CN_OPI_0164082513)

本操作指导介绍在运行网络中激活多时区功能的操作过程。

多时区业务是指随着用户的移动， UNC 可以根据其位置信息与时区/夏令时的映射表，将用户当前位置的时区和夏令时信息上报给UE以刷新UE系统时间，同时还将用户当前位置的时区和夏令时信息上报给用户业务承载连接的网关（ GGSN/ S-GW/SMF），以实现基于当前位置时区的差异化计费。

## [任务示例](#ZH-CN_OPI_0164082513)

任务描述

在 GPRS/UMTS/ EPC网络中启用多时区功能，且满足如下多时区需求。

- UNC需要在时区发生变化时向UE和GGSN/S-GW和UE上报实时时区信息。
- 在GPRS/UMTS网络中，LAC号码112~350的位置区的时区配置为E0100(GMT+01:00)，且存在夏令时。夏令时开始时间为每年三月最后一周周日的凌晨2点，结束时间为每年10月最后一周周日的凌晨3点，夏令时相对于标准时的时间偏移量为1小时。
- 在EPC网络中，TAC号码100~300的位置区的时区配置为E0100(GMT+01:00)，且存在夏令时。夏令时开始时间为每年三月最后一周周日的凌晨2点，结束时间为每年10月最后一周周日的凌晨3点，夏令时相对于标准时的时间偏移量为1小时。

脚本

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2TZSV01", SWITCH=ENABLE;
```

//设置 UNC 在时区发生变化时向UE和GGSN/S-GW上报实时时区信息。

//设置 UNC 可以向UE发送2G/3G/4G网络信息。

```
SET MMFUNC: MMINFO=Iu_MODE-1&Gb_MODE-1&S1_MODE-1,EMMINFOFIRST=S1_MODE-1;
```

//设置GGSN CDR/e-GCDR信息上报的内容包括用户时区和夏令时信息。

```
ADD GGSNCHARACT: RANGE=ALL_GGSN, EGCDR=MS_TIME_ZONE_DST-1;
```

//设置 UNC 在Update PDP Context Response/Delete PDP Context Response/Delete PDP Context Request消息中会向GGSN上报更新后的用户时区/夏令时信息。

```
ADD GTPCCMPT: IDX=100, MSGCLS=TM, INCLUDE=PRO_DEFINE, TMMSGTYPE=UPDATE_PDP_CONTEXT_RESPONSE, UPDPDPRES=TZ;
ADD GTPCCMPT: IDX=101, MSGCLS=TM, INCLUDE=PRO_DEFINE, TMMSGTYPE=DELETE_PDP_CONTEXT_RESPONSE, DELPDPRES=TZ;
ADD GTPCCMPT: IDX=102, MSGCLS=TM, INCLUDE=PRO_DEFINE, TMMSGTYPE=DELETE_PDP_CONTEXT_REQUEST, DELPDPREQ=TZ;
```

//设置位置区/跟踪区对应的时区和夏令时信息。

//设置时区标识与时区夏令时的映射关系。

```
ADD TZLST: TZID=1, TZDESC="E0100", TZONE=E0100, DST=YES, SM=WEEK, SMONTH=MAR, SWSEQ=LAST, SWEEK=SUN, ST=02&00&00, EM=WEEK, EMONTH=OCT, EWSEQ=LAST, EWEEK=SUN, ET=03&00&00, TO=1;
```

//设置位置区/跟踪区与时区标识的映射关系。

```
ADD AREATZ: AREA=LA, MCC="460", MNC="03", LAC="112", LACRANGE="350", TZID=1;
ADD AREATZ: AREA=TA, MCC="460", MNC="03", TAC="100", TACRANGE="300", TZID=1;
```
