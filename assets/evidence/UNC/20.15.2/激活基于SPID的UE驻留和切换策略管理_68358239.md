## [操作场景](#ZH-CN_OPI_0168358239)

本操作指导介绍在运行网络中激活基于SPID的UE驻留和切换策略管理特性的操作过程。

3GPP协议中定义了SPID（Subscriber Profile ID for RAT/Frequency Selection Priority），它定义了一组频段及其优先级。这个信元可以由用户签约信息指定，也可以由 UNC 指定，传递到无线侧之后，可以用于灵活控制终端的行为策略，例如指定终端驻留和切换频段的优先级等。

无线侧的SPID和核心网侧的RFSP ID为同一概念。

> **说明**
> 适用于 SGSN、 MME。

## [任务示例](#ZH-CN_OPI_0168358239)

任务描述

- 场景一： UNC 作为MME出S6a接口，或者作为SGSN出S6d接口，运营商规划完全信赖用户在HSS的签约信息，在HSS直接规划好RFSP ID， UNC 本地不做RFSP ID控制，将用户签约的RFSP ID发送到RNC和BSC。其中Gb接口未启用Gb接口动态自动配置。
- 场景二： UNC 作为MME出S6a接口，假设不信赖HSS中的用户签约信息（例如对于漫游用户）。为了实现自定义的驻留优先级控制， UNC 可以在本地针对用户进行RFSP策略的配置，并发送到eNodeB。
- 场景三：假设运营商需要某号段的用户优先使用签约的RFSP ID对应的频点优先级进行驻留控制（例如：4G用户回落3G后，优先使用签约RFSP ID对应的频点优先级进行驻留控制），如果没有签约RFSP ID，则按照配置的RFSP ID对应的频点优先级进行驻留控制。

脚本

- 场景一脚本
  //根据运营商规划，对端RNC支持SPID，并希望核心网将SPID带给RNC和BSC。
  ```
  ADD RNC: RNCX=1, RNCMCC="460", RNCMNC="00", RNCID=1, NI=INT, SPC="123", RNCVER=R8, SPID=YES;
  ADD NSE: OTHERNODE="1", NSEI=1, BSSID=1, PFC=YES, SPID=YES;
  ```
- 场景二脚本
  //开启基于SPID的UE驻留和切换策略管理功能开关。
  ```
  SET LICENSESWITCH: LICITEM="LKV2SPID01", SWITCH=ENABLE;
  ```
