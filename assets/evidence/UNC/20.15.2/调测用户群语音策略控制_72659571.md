# 调测用户群语音策略控制

- [操作场景](#ZH-CN_OPI_0172659571__1.3.1)
- [必备事项](#ZH-CN_OPI_0172659571__1.3.2)
- [操作步骤](#ZH-CN_OPI_0172659571__1.3.3)

## [操作场景](#ZH-CN_OPI_0172659571)

调测用户群语音策略控制特性。

## [必备事项](#ZH-CN_OPI_0172659571)

前提条件

- 请仔细阅读[WSFD-201002 用户群语音策略控制特性概述](特性概述_70014709.md)。
- 完成[激活用户群语音策略控制](激活用户群语音策略控制_64009940.md)。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0172659571)

- 运营商A部署了VoLTE语音业务，不允许外网用户使用VoLTE语音业务。用户X和用户Y为运营商A本网用户，签约了IMS APN，用户X支持SRVCC，用户Y不支持SRVCC。用户Z为运营商A的外网用户，且签约了IMS APN。
  在相关网元均完成本特性的配置后，可以采用以下步骤检查特性工作是否正常：
    1. 分别创建用户X、Y、Z的用户跟踪。
    2. 用户X、Y、Z开机在LTE网络附着。
    3. 观察MME向用户X、Y、Z发送的Attach Accept消息中EPS Network Feature Support信元的bit1的取值。验证结果如 [表1](#ZH-CN_OPI_0172659571__table_0228140D) 所示。
      *表1 验证结果*

      | 用户 | EPS Network Feature Support信元的bit1 |
      | --- | --- |
      | X | **预期结果**<br>：bit1取值为1。 |
      | X | **异常结果**<br>：bit1取值为0。 |
      | Y | **预期结果**<br>：bit1取值为0。 |
      | Y | **异常结果**<br>：bit1取值为1。 |
      | Z | **预期结果**<br>：bit1取值为0。 |
      | Z | **异常结果**<br>：bit1取值为1。 |
      > **说明**
      > 当EPS Network Feature Support信元的所有bit值均为0时，Attach Accept消息中不携带此信元。
