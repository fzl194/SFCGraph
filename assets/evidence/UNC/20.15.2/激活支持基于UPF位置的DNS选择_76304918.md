## [操作场景](#ZH-CN_OPI_0276304918)

当需要为用户选择地理位置贴近用户的DNS服务器时，激活UPF位置的DNS选择。

> **说明**
> 适用于GGSN、PGW-C、SMF。

## [对系统的影响](#ZH-CN_OPI_0276304918)

该操作对系统正常运行没有影响。

1. 进入 “MML命令行-UNC” 窗口。
2. 配置用户面节点实例。

  **[ADD UPAREA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/PFCP接口管理/UP管理/UP跟踪区管理/UP区域管理/增加UPF服务区（ADD UPAREA）_09652457.md)**

  **[ADD UEDNSUPGROUP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入控制/DN网络DNS_NBNS选择管理/DNS选择管理/DNS关联的UPF组管理/增加DNS关联的UPF组（ADD UEDNSUPGROUP）_73321231.md)**

## [任务示例](#ZH-CN_OPI_0276304918)

任务描述

按照网络规划，dnn1有City A和City B两个接入点，对端存在如下所示2个UPF，UPF1内APN为huawei1.com绑定DNS1，APN为huawei2.com绑定DNS2，UPF2绑定DNS3，在SMF上需要为UE1（huawei1.com，TAI=12303260006）选择地理位置贴近用户的DNS1，为UE2（TAI=12303260016）选择地理位置贴近用户的DNS3。

| UPF标识 | UPF地址信息 | UPF支持的覆盖区名称 | UPF支持的覆盖区绑定的N2TAI范围 | UPF绑定的DNS服务器 |
| --- | --- | --- | --- | --- |
| UPF_Instance_1 | 10.2.2.2 | UPAREA1 | 12303260001~12303260007 | APN=huawei1.com：DNS1<br>主：10.1.1.1<br>备：10.12.12.12<br>APN=huawei2.com：DNS2<br>主：10.15.15.15<br>备：10.16.16.16 |
| UPF_Instance_2 | 10.3.3.3 | UPAREA2 | 12303260011~12303260017 | DNS3<br>主：10.13.13.13<br>备：10.14.14.14 |

脚本

如果对端用户面信息中支持的位置区信息已经在初始配置中根据网络规划完成配置，本脚本中无需重复配置用户面信息。

//配置用户面节点实例。

```
ADD PNFPROFILE:NFINSTANCEID="UPF_Instance_1", NFTYPE=NfUPF, NFSTATUS=Registered, FQDN="
upf01.node.mnc003.mcc123.3gppnetwork.org
", IPADDRESSTYPE=IPTypeV4, IPV4ADDRESS1="
10.2.2.2
", PORT=
8080
;
ADD PNFPROFILE:NFINSTANCEID="UPF_Instance_2", NFTYPE=NfUPF, NFSTATUS=Registered, FQDN="
upf01.node.mnc003.mcc123.3gppnetwork.org
", IPADDRESSTYPE=IPTypeV4, IPV4ADDRESS1="
10.3.3.3
", PORT=
8080
;
```

//配置用户面节点的位置区信息，将位置区与TAI绑定。

```

", N2ENDTAI="

12303260011
", N2ENDTAI="
12303260017
";
```

//配置用户面节点支持的为SMF提供服务区域的信息。

```
ADD PNFSMFSERAREA:NFINSTANCEID="UPF_Instance_1", SMFSERVINGAREA="UPAREA1";
ADD PNFSMFSERAREA:NFINSTANCEID="UPF_Instance_2", SMFSERVINGAREA="UPAREA2";
```

//增加DNS关联的UPF组。

```
ADD UEDNSUPGROUP: UPFGRPNAME="UPFGROUP1";
ADD UEDNSUPGROUP: UPFGRPNAME="UPFGROUP2";
```
