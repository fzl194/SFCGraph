## [操作场景](#ZH-CN_OPI_0295923556)

LastActivity计费功能在话单中记录用户最后一个报文通过GGSN/SGW-U/PGW-U的时间点，据此可以根据用户使用业务的时间进行计费，或者核查话单。当运营商部署了LastActivity计费功能，需要对该功能进行调测，确保该功能可正常使用。

> **说明**
> 适用于SGW-C、PGW-C 、GGSN 。

## [必备事项](#ZH-CN_OPI_0295923556)

前提条件

- 请仔细阅读[Ga/Gy接口离线/在线计费](../../../../../业务专题/5G Core 计费解决方案/计费解决方案概述/计费原理/Ga_Gy接口离线_在线计费_87165686.md)。
- 完成[配置话单可选功能（GGSN/SGW-C/PGW-C）](../激活离线计费/配置话单可选功能（GGSN_SGW-C_PGW-C）_95923448.md)。

## [操作步骤](#ZH-CN_OPI_0295923556)

1. 在OM Portal上创建用户跟踪任务。
2. 激活用户，测试终端使用“apn-test”APN接入网络，访问业务。
    - 如果测试终端成功接入网络，请执行[步骤 3](#ZH-CN_OPI_0295923556__stp2)。
    - 如果测试终端无法接入网络，请调测UNC的接入功能。
3. 设置场景产生话单，以去激活用户产生话单为例，进行消息跟踪。比如，17:59:40激活用户，大约25秒后去激活用户，生成用户话单。检查话单中RecordExtensions字段记录的LastActivityTime是否与实际去激活规划值一致。
  如 [图1](#ZH-CN_OPI_0295923556__fig18902175318426) 所示是用户话单。从话单中看出，LastActivity字段已经记录用户最后一个报文通过的时间18:00:05，与设置场景吻合。
  **图1** 用户话单

  <br>

  ![](调测LastActivity计费_95923556.assets/zh-cn_image_0295923516_2.png "点击放大")
    - 如果一致，说明功能正常，本调测任务结束。
    - 如果不一致或字段缺失，请执行[步骤 4](#ZH-CN_OPI_0295923556__stp5)。
4. 执行 **LST OFCCDRPARA** 命令，查看LastActivity计费功能是否开启。
  ```
  LST OFCCDRPARA:;
  ```
  ```
  RETCODE = 0  操作成功。

  离线话单参数配置
  -------------------------
             gsn-node-id字段前缀  =  huawei.com
                                ......
           Last Activity功能开关  =  使能
  ```
    - 如果是，请执行[步骤 5](#ZH-CN_OPI_0295923556__step1829245425110)。
    - 如果否，请参考[配置话单可选功能（GGSN/SGW-C/PGW-C）](../激活离线计费/配置话单可选功能（GGSN_SGW-C_PGW-C）_95923448.md)重新配置，然后返回[步骤 2](#ZH-CN_OPI_0295923556__stp1)重新调测。
5. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务，执行 [步骤 3](#ZH-CN_OPI_0295923556__stp2) 并保存报文。
    b. 执行 **EXP MML** 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
