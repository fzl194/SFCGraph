## [操作场景](#ZH-CN_OPI_0258600013)

本操作指导介绍在运行网络中激活5G精准寻呼的操作过程。

UNC 基于用户类型、业务类型选择相应的寻呼策略，综合用户的移动性进行寻呼规则过滤，选择合适的范围对用户进行寻呼，从而有效地减少整个网络中gNodeB的寻呼负荷，节省网络资源。

> **说明**
> 适用于AMF。

## [必备事项](#ZH-CN_OPI_0258600013)

前提条件

- 请仔细阅读[WSFD-206001 5G精准寻呼特性概述](特性概述_58600011.md)。
- 建议由华为专业服务对本网进行评估后选定最优的精准寻呼参数。
- 已完成加载License。

## [操作步骤](#ZH-CN_OPI_0258600013)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. 配置寻呼规则，包括用户类型、业务类型和寻呼范围组合。
  [**ADD NGPAGINGRULE**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N2接口管理/NGAP接口寻呼管理/NG寻呼规则管理/增加5G寻呼规则（ADD NGPAGINGRULE）_09652969.md)

  > **说明**
  > - 在部署本特性时，建议针对时延比较敏感的语音业务使用TA List的寻呼范围，从而在不影响语音被叫时延的情况下节省网络资源。
  > - 针对数据业务和其他非语音类业务，“寻呼动作组合”推荐采用“最近访问gNodeB -> TA List”组合，即“ACTGRP（寻呼动作组合）”取值设置为“LAST_GNB”。这样能够有效减少寻呼范围的同时，明显减少对gNodeB侧的资源消耗。
4. 配置寻呼策略，包括寻呼范围开关、低移动性检查开关、粘性时长，进行寻呼规则过滤。配置寻呼定时器、重发次数、邻接关系学习开关、老化时间等。
  [**SET NGPAGINGCTRL**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N2接口管理/NGAP接口寻呼管理/NG寻呼策略管理/设置5G寻呼策略控制表（SET NGPAGINGCTRL）_09652258.md)
  > **说明**
  > 全局TA List的寻呼定时器、重发次数和重呼间隔时间由 [**SET NGMMPARA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动管理定时器/设置5G MM协议参数（SET NGMMPARA）_09653645.md) 配置。
5. 配置指定的5G TA List是否关闭精准寻呼功能。
  [**ADD NGTAPAGINGPLCY**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N2接口管理/NGAP接口寻呼管理/NG基于跟踪区列表的寻呼策略管理/增加5G跟踪区列表的寻呼策略配置（ADD NGTAPAGINGPLCY）_48290734.md)
  > **说明**
  > - TA List需要先通过[**ADD NGTALST**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/跟踪区管理/跟踪区列表管理/增加5G跟踪区列表（ADD NGTALST）_09651355.md)配置。
  > - 若针对指定的TA List关闭精准寻呼功能，则[3](#ZH-CN_OPI_0258600013__cmd36451038194417)设置的精准寻呼规则不生效。
6. 基于业务参数配置寻呼定时器、重发次数和重呼间隔时间。
  [**ADD NGPAGINGTMRPLCY**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N2接口管理/NGAP接口寻呼管理/NG寻呼定时器策略管理/增加5G寻呼定时器策略配置（ADD NGPAGINGTMRPLCY）_09652229.md)
  > **说明**
  > - 匹配成功，[**SET NGMMPARA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动管理定时器/设置5G MM协议参数（SET NGMMPARA）_09653645.md)命令设置的寻呼定时器、重发次数和重呼间隔时间将失效。
  > - 匹配失败，则使用全局的定时器时长、重发次数，即[**SET NGMMPARA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动管理定时器/设置5G MM协议参数（SET NGMMPARA）_09653645.md)命令设置的寻呼定时器、重发次数和重呼间隔时间。
7. 配置DNN、ARP映射寻呼优先级的规则。
  [**ADD NGPAGINGPRIPLCY**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N2接口管理/NGAP接口寻呼管理/寻呼优先级策略参数配置/增加5G寻呼优先级策略参数配置（ADD NGPAGINGPRIPLCY）_09652363.md)

## [任务示例](#ZH-CN_OPI_0258600013)

任务描述

任务1：

针对其他业务，采用TA List寻呼范围组合，不启用5G精准寻呼。

任务2：

针对“N1N2TRANS消息”类型的业务，配置寻呼规则，使用“最近gNodeB -> TA List”寻呼范围组合。

1. 配置寻呼策略：
    - 打开最近gNodeB寻呼范围开关，打开低移动性检查开关，配置GNB粘性时长为2分钟，过滤寻呼策略中的寻呼范围。
    - 配置的寻呼定时器时长为4秒、寻呼重发次数为1次。
    - 开启最近驻留位置更新和gNodeB邻接关系学习，邻接gNodeB老化时间为360小时。
2. 指定TA List标识为1的5G TA List关闭精准寻呼功能。
3. 基于业务的DNN为“huawei.com”配置寻呼定时器设置寻呼定时器为5秒、重发次数为1和重呼间隔时间为默认的0。
4. 配置业务的DNN为“huawei.com”的用户寻呼优先级为2，使业务在系统拥塞时，可以被优先处理。

脚本

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2PRPG02", SWITCH=ENABLE;
```

任务1：

//配置其他业务不启用5G精准寻呼。

```
ADD NGPAGINGRULE: RULEIDX=1, GRPTYPE=ALL_USER, SVRTYPE=OTHER, PRIORITY=100, ACTGRP=LAST_GNB-0&NEIGH_GNB-0&LAST_TA-0;
```

任务2：

//配置“N1N2TRANS消息”类型的业务启用精准寻呼，使用“最近gNodeB -> TA List”寻呼范围组合。

```
ADD NGPAGINGRULE: RULEIDX=2, GRPTYPE=ALL_USER, SVRTYPE=SMF_N1N2, DNNIND=ALL_DNN, FQI=5, PRIORITY=50, ACTGRP=LAST_GNB-1&NEIGH_GNB-0&LAST_TA-0;
```

//打开最近gNodeB寻呼范围开关，打开低移动性检查开关，配置GNB粘性时长为2分钟，过滤寻呼策略中的寻呼范围。

//配置的寻呼定时器时长为4秒、寻呼重发次数为1次。

//开启最近驻留位置更新和gNodeB邻接关系学习，邻接gNodeB老化时间为360小时。

```
SET NGPAGINGCTRL: LASTGNBSW=ON, LASTGNBT3513=4, LASTGNBN3513=1, LOWMOBICHKSW=ON, GNBSTICKTIME=2, HOLRNSW=ON, SONLRNSW=ON, PSLRNSW=ON, GNBEXPIRETIME=360;
```

//指定标识为1的5G TA List关闭精准寻呼功能。

```
ADD NGTAPAGINGPLCY: TALISTID=1, CLSPRECISEPAG=YES;
```

//基于业务的DNN为“huawei.com”配置寻呼定时器设置寻呼定时器为5秒、重发次数为1和重呼间隔时间为默认的0。

```
ADD NGPAGINGTMRPLCY: PLCYIDX=1, DNN="huawei.com", T3513=5, N3513=1;
```

//配置业务的DNN为“huawei.com”的用户寻呼优先级为2。

```
ADD NGPAGINGPRIPLCY: PLCYIDX=1, DNN="huawei.com", PAGINGPRIORITY=2;
```
