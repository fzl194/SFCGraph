# 5GC业务流程

5G网络中，IPv6 Prefix Delegation的业务流程如 [图1](#ZH-CN_TOPIC_0000001259334341__fig243115811463) 所示。

**图1** IPv6 Prefix Delegation业务流程图

<br>

![](5GC业务流程_59334341.assets/zh-cn_image_0000001258934385_2.png)

**UE先启动获取IPv6前缀流程。**

1. UE发起PDU会话请求。

2. AMF向SMF发送Nsmf_PDUSession_CreateSMContext Request消息。

3. SMF向AAA Server发送鉴权请求。

P1：AAA Server在发送鉴权响应时，AAA Server支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

4. AAA Server将鉴权响应消息发送给SMF。

5. SMF发送PFCP Session Establishment Request消息给UPF。

P2：SMF支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

6. UPF返回PFCP Session Establishment Response消息给SMF。

P3：UPF支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

7. SMF返回Nsmf_PDUSession_CreateSMContext Response消息给AMF。

8. AMF与SMF完成Nsmf_PDUSession_N1N2MessageTransfer消息给交互。

9. UE发送ICMPv6 Router Solicitation消息给UPF，发起IPv6地址自动配置协商。

10. UPF通过PFCP Session Report Request消息将IPv6地址自动协商请求发给SMF。

11. SMF返回PFCP Session Report Response消息给UPF。

12. UPF返回ICMPv6 Router Advertisement消息给UE，携带分配给UE的IPv6地址前缀。同时将Router Advertisement消息中的O-flag置位为1，表示后续可以通过DHCPv6方式获取其他的相关参数。

![](5GC业务流程_59334341.assets/notice_3.0-zh-cn_2.png)

IPv4/v6双栈改造后，UE需通过UPF透传Router Solicitation数据报文至SMF，SMF将v6地址前缀通过Router Advertisement数据报文回传至UE。UE获取IPv6地址。该数据报文会在一个周期内重复下发，该周期内如果UE处于弱覆盖的场景，则eNodeB无法下发该报文给终端，掉话率会增加。

**UE获取IPv6地址前缀后，发起DHCPv6地址分配流程。**

13. UE发送DHCPv6 Solicit请求消息给UPF，消息中携带IA_PD option、Rapid Commit option和Prefix Exclude Option。

14~15. UPF通过PFCP Session Report Request消息将UE进行DHCPv6地址分配信息发给SMF，SMF返回响应。

16. （快速流程不包含此消息）UPF返回DHCPv6 Advertise消息给UE。

17. （快速流程不包含此消息）UE发送DHCPv6 Request消息给UPF。

18~19. （快速流程不包含此消息）UPF通过PFCP Session Report Request消息将UE协商信息发给SMF，SMF返回响应。

20. UPF返回DHCPv6 Reply消息给UE，携带为其分配的IPv6 delegated-prefix和Prefix Exclude Option。

21. UE下的终端设备申请接入网络时，UE为其从IPv6 delegated-prefix段中分配64位前缀。
