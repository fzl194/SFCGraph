# GPRS/UMTS业务流程

GPRS/UMTS网络中，Routing Behind MS的业务流程如 [图1](#ZH-CN_TOPIC_0276458441__fig243115811463) 所示。

**图1** Routing Behind MS业务流程图

<br>

![](GPRS_UMTS业务流程_76458441.assets/zh-cn_image_0000002560620233_2.png)

1. 无线设备（MS/UE）发起激活请求。
2. SGSN向PGW-C发送Create PDP Context Request消息。
3. PGW-C获取到MS的激活请求后，向AAA Server发送鉴权请求。
4. AAA Server检测MS是Routing Behind MS用户，将MS对应的PC下行路由封装在鉴权响应消息中发送给PGW-C。
5. PGW-C获取PC的下行路由，将该路由插入到PGW-C路由表中，并通过PFCP Session Establishment Request消息发送给PGW-U。后续，PGW-U会根据该手机后网段路由更新本地路由表，以便能找到对应的终端设备，进行下行转发，如步骤[12](#ZH-CN_TOPIC_0276458441__li258439340175820)所示。
6. PGW-U给PGW-C响应PFCP Session Establishment Response消息。
7. PGW-C向SGSN发送Create PDP Context Response消息。
8. MS完成一次PDP的建立。
9. PC通过MS发起上行业务。
10. PGW-U发现该报文源IP地址与MS的主机地址不一致，根据用户属性自动关闭Anti-spoofing功能。终端设备上行数据正常转发。
11. 网络侧发送到MS下某PC的下行数据。
12. PGW-U发现该报文目的IP地址与MS的主机地址不一致，根据网段路径将该报文关联到MS的PDP上下文，再通过该PDP发送到MS。MS将下行数据报文转发到目的PC。

MS发起去激活请求时，PGW-C可以感知MS下线，删除其对应路由信息并通知PGW-U。
