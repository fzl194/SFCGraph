## [操作步骤](#ZH-CN_OPI_0000001376983597)

1. 进入 “MML命令行-UNC” 窗口。
2. 设置服务类型为USM_0的微服务的Token分配权重管理策略为动态策略。
  **[SET TOKENALLOCWT](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/权重分配管理/权重分配策略/设置服务类型分配权重的管理策略（SET TOKENALLOCWT）_23736568.md)**
3. 设置服务类型为USM_0的微服务的动态权重调整相关参数。
  **[SET DYNAMICPOLICY](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/权重分配管理/动态权重策略/设置服务类型分配动态权重的管理策略（SET DYNAMICPOLICY）_24015952.md)**
4. 增加不同CPU类型的能力系数和基础消耗。
  **[ADD CPUABILITYCFG](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/权重分配管理/CPU能力系数策略/增加不同CPU类型的能力系数和基础消耗。（ADD CPUABILITYCFG）_23736532.md)**

## [任务示例](#ZH-CN_OPI_0000001376983597)

任务描述

设置服务类型为USM_0的微服务的Token分配权重管理策略为动态策略和动态权重调整相关参数，使USM的Pod实例间CPU负荷差异超过阈值时，系统进行动态负荷调整。

脚本

//设置服务类型为USM_0的微服务的Token分配权重管理策略为动态策略。

```
SET TOKENALLOCWT:SERVICETYPE=USM_0,WEIGHTPOLICY=DYNAMIC;
```

//设置服务类型为USM_0的微服务的动态权重调整相关参数。

```
SET DYNAMICPOLICY: SERVICETYPE=USM_0, ADJUSTTHRESHOLD=75, GAPTHRESHOLD=6, GLBBASEOST=5, GAPBUSY=10, CONFIRM=Y;
```

//（可选）增加不同CPU类型的能力系数和基础消耗。

```
ADD CPUABILITYCFG: CPUTYPE=X86_64, CPUGENERATE="1", CPUFREQUENCY=2, COEFFICIENT=500, BASECOST=0, CONFIRM=Y;
```
