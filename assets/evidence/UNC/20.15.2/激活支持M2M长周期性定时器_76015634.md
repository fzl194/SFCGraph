## [操作场景](#ZH-CN_OPI_0276015634)

本操作指导介绍在运行网络中激活支持M2M长周期性定时器的操作过程。

本特性针对长期处于静止或不经常移动状态的M2M类终端分配一个长周期定时器（extended periodic timer），用于延长终端位置更新（TAU/RAU）的周期，降低TAU/RAU频度，从而降低此类终端的功耗以及网络的信令负荷。

> **说明**
> 适用于MME。

## [操作步骤](#ZH-CN_OPI_0276015634)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开M2M长周期性定时器功能开关。
  [**SET SOFTPARAOFBIT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/软件参数管理/软件参数比特位/设置软件参数表比特位(SET SOFTPARAOFBIT)_72345783.md)
  > **说明**
  > 该软参控制是否下发M2M长周期定时器。
3. 配置长周期定时器控制参数。
  > **说明**
  > 若系统同时配置了基于所有用户、本网用户或外网用户以及指定IMSI前缀的长周期定时器下发策略，当系统识别到终端支持长周期定时器时，将首先匹配基于“指定IMSI前缀”的记录，其次匹配基于“外网用户”或“本网用户”的记录，最后匹配基于“所有用户”的记录。
  [**ADD M2MPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/M2M策略参数配置/增加M2M策略参数(ADD M2MPLCY)_72225443.md)
4. **可选：**若现网存在2G M2M终端，则需配置用户安全参数，避免发生串号。
    - 查询用户安全参数配置，若参数 “SECPLC” 取值不为 “AUTHANDPROTECTED” 且 “CIPHALG” 取值不为 “GEA_3-1&GEA_2-1&GEA_1-1” ，可执行 [**MOD GBAUTHCIPH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/Gb模式用户安全参数/修改Gb模式用户安全参数（MOD GBAUTHCIPH）_26305454.md) 修改配置。
      [**LST GBAUTHCIPH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/Gb模式用户安全参数/查询Gb模式用户安全参数(LST GBAUTHCIPH)_72345241.md)
    - 修改用户安全参数记录。
      [**MOD GBAUTHCIPH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/Gb模式用户安全参数/修改Gb模式用户安全参数（MOD GBAUTHCIPH）_26305454.md)

## [任务示例](#ZH-CN_OPI_0276015634)

任务描述

现网M2M终端集中在IMSI前缀为“12303”的用户范围中，M2M业务对应的APN为“M2M”，希望终端发起TAU的周期为12小时。

脚本

//打开M2M长周期性定时器功能开关。

```
SET SOFTPARAOFBIT:DT=DWORD_EX, PARANUM=31, VALUE=VALUE_1, POSITION=26;
```

//配置针对指定IMSI前缀用户的长周期定时器。

```
ADD M2MPLCY: SUBRANGE=IMSI_PREFIX, IMSIPRE="12303", APNNI="M2M",RAUTAUTMRSRC=USE_CONFIG, LONGRAUTAUTMR=12;
```

//配置用户安全参数，避免串号问题发生。

```
MOD GBAUTHCIPH: SUBRANGE=ALL_USER, SECPLC=AUTHANDPROTECTED, CIPHALG=GEA_3-1&GEA_2-1&GEA_1-1;
```
