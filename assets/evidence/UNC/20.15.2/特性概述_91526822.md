#### [适用NF](#ZH-CN_CONCEPT_0191526822)

SGSN、 GGSN、 MME、SGW-C、PGW-C

#### [定义](#ZH-CN_CONCEPT_0191526822)

- LTE到UMTS的网络重选
  UE移动出LTE和UMTS的重叠覆盖区进入单UMTS的覆盖区后，执行小区重选，发起RAU（路由区更新）过程，驻留到UMTS小区。
- UMTS到LTE的网络重选
  UE移动到LTE和UMTS的重叠覆盖区后，LTE具有高优先级，执行小区重选，发起TAU（跟踪区更新）过程，驻留到LTE小区。

#### [客户价值](#ZH-CN_CONCEPT_0191526822)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 对于运营商来说，LTE和UMTS网络之间的重选特性主要受益如下：UE驻留在当前系统，但将要选择驻留另外一个系统时，可以通过路由更新的方式接入新的系统，而不需要进行完整的附着过程，减少了信令交互，减少在空口传递IMSI的几率。<br>- 保护现有UMTS网络投资，便于网络间的平滑演进。 |
| 用户 | - 保证用户在UMTS和LTE网络间漫游。<br>- 良好的用户体验。 |

#### [应用场景](#ZH-CN_CONCEPT_0191526822)

支持LTE和UMTS网络之间的重选，MME和SGSN之间通过Gn接口互通。

#### [应用限制](#ZH-CN_CONCEPT_0191526822)

应用本特性有以下限制：

- 为保证网络重选后业务不中断，需要GGSN和PGW-C共建（同一个网元内）。
- 为保证鉴权集的一致性，推荐HLR和HSS共建，否则可能会在网络重选过程中多次鉴权，影响切换时延。
- 该特性中GTP消息中不支持GTPv0版本。

#### [原理概述](#ZH-CN_CONCEPT_0191526822)

- [**基本概念**](#ZH-CN_CONCEPT_0191526822__zh-cn_concept_0130428769_sub3_1)
- [**系统实现**](#ZH-CN_CONCEPT_0191526822__zh-cn_concept_0130428769_sub3_2)

[**基本概念**](#ZH-CN_CONCEPT_0191526822__zh-cn_concept_0130428769_sub3_1)

- UNC支持LTE到UMTS的网络重选（Inter和Intra）。MME和目标SGSN之间通过Gn接口互通。
- UNC支持UMTS到LTE的网络重选（Inter和Intra）。源SGSN和MME之间通过Gn接口互通。

[**系统实现**](#ZH-CN_CONCEPT_0191526822__zh-cn_concept_0130428769_sub3_2)

- LTE到UMTS的网络重选
  UE移动出LTE覆盖区同时该地区存在UMTS覆盖时，执行小区重选，发起RAU（路由区更新）过程，驻留到UMTS小区。在此过程中，目标SGSN通过Gn接口向源侧MME发起请求，如果MME能匹配到对应的用户信息，MME将系统保存的移动性管理上下文和承载上下文发送给目的SGSN，其中包括安全参数、QoS参数等信息，配合完成跨接入网类型路由区更新。
  **图1** LTE到UMTS的小区重选基本原理图

  <br>

  ![](特性概述_91526822.assets/zh-cn_image_0192721379_2.png)

  | 1、UE向SGSN发起RAU请求。 |
  | --- |
  | 2、SGSN从MME中获取UE的上下文。 |
  | 3、如果上下文中没有安全参数，执行安全模式操作。 |
  | 4、SGSN通知PGW-C/GGSN更新上下文信息。 |
  | 5、HLR取消原MME中的位置信息并更新SGSN上的位置信息。 |
  | 6、SGSN接受UE的RAU请求，UE完成LTE到UMTS的RAU。 |
  | 7、MME通知SGW-C删除原有承载，如果在RAU前存在S1连接，释放该连接。 |

  > **说明**
  > Intra重选场景下SGSN和MME融合，所以 [图1 LTE到UMTS的小区重选基本原理图](#ZH-CN_CONCEPT_0191526822__zh-cn_concept_0130428769_fig_05) 中不存在SGSN和MME之间交互的消息，其他消息流程不变。
- UMTS到LTE的网络重选
  UE移动出UMTS覆盖区同时该地区存在LTE覆盖时，执行小区重选，发起TAU（跟踪区更新）过程，驻留到LTE小区。在此过程中，目的MME通过Gn接口向源侧SGSN发起请求，如果SGSN能匹配到对应的用户信息，SGSN将系统保存的移动性管理上下文和承载上下文发送给目的MME，其中包括安全参数、QoS参数等信息，配合完成跨接入网类型跟踪区更新。
  **图2** UMTS到LTE的小区重选基本原理图

  <br>

  ![](特性概述_91526822.assets/zh-cn_image_0192721384_2.png)

  | 1、UE通过eNodeB向MME发送TAU请求。 |
  | --- |
  | 2、MME从SGSN获取UE的上下文。 |
  | 3、如果上下文中没有安全参数，执行安全模式操作。 |
  | 4、MME向SGW-C发送建立承载请求消息，SGW-C创建上下文，并向PGW-C发送更新承载请求消息。 |
  | 5、HLR/HSS取消原SGSN中的位置信息并更新MME上的位置信息。 |
  | 6、如果在TAU前存在Iu连接，释放该连接。 |
  | 7、MME接受UE的TAU请求，UE完成UMTS到LTE的TAU。 |

  > **说明**
  > Intra重选场景下SGSN和MME融合，所以 [图2 UMTS到LTE的小区重选基本原理图](#ZH-CN_CONCEPT_0191526822__zh-cn_concept_0130428769_fig_06) 中不存在SGSN和MME之间交互的消息，其他消息流程不变。
