#### [事件计费关键信元](#ZH-CN_TOPIC_0274112318)

融合计费中，GGSN-C/PGW-C/SMF根据CHF下发的计费事件（Triggers）与CHF进行交互，并通过下列信元完成基本的配额授权与用量上报。

- CHF下发的事件计费配额和阈值：
  Charging Data Response>MultipleUnitInformation>grantedUnit>serviceSpecificUnits
  Charging Data Response>MultipleUnitInformation>unitQuotaThreshold
- GGSN-C/PGW-C/SMF上报事件计费用量：Charging Data Request>MultipleUnitUsage>UsedUnitContainer>serviceSpecificUnits
- GGSN-C/PGW-C/SMF申请的事件计费配额：Charging Data Request>MultipleUnitUsage>RequestedUnit>serviceSpecificUnits

#### [事件计费支持的业务类型](#ZH-CN_TOPIC_0274112318)

UNC 支持SCUR（Session Charging with Unit Reservation）配额预留方式的会话计费。该方式是在传统的会话计费中上报事件计费，无论是流量、时长、还是事件计费，统一使用同一个在线计费会话来承载。 UNC 支持以下协议类型的事件计费：

- HTTP业务：基于用户对HTTP业务的访问次数进行计费。这里的访问次数，是触发HTTP Get/Post请求的次数，不是用户点击链接的次数。实际上，用户在打开一个主页的过程中，可能会触发多个HTTP Get/Post请求。 UNC 会当做多个事件进行计费。
  > **说明**
  > HTTP协议中connect报文是用于在MS/UE->proxy->server之间建立安全隧道，并不是真正的业务访问。因此HTTP协议中的connect报文不会触发事件计费。

- WAP业务：基于用户对某种WAP（Wireless Application Protocol）业务的访问次数进行计费。
- MMS业务：基于用户对MMS（Multimedia Messaging Service）服务的使用次数进行计费。

#### [建立计费会话](#ZH-CN_TOPIC_0274112318)

1：用户发起PDU会话建立请求。

2：GGSN-C/PGW-C/SMF向CHF发送Charging Data Request[Initial]消息，请求建立计费会话。该消息可携带预申请的事件计费配额（通过 [**SET CTXSTARTRATING**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务模板/用户模板/设置给OCS_CHF发送的消息初始携带的计费属性（SET CTXSTARTRATING）_09897210.md) 命令配置），预申请的事件计费配额数 **由 [ADD URR](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务策略/内容计费标识/增加URR（ADD URR）_09897158.md)** 命令中的 “EVENTRSUVALUE” 参数决定，并通过serviceSpecificUnits字段携带给CHF。

3：CHF向GGSN-C/PGW-C/SMF返回Charging Data Response[Initial]消息。如果GGSN-C/PGW-C/SMF预申请了事件计费配额，事件计费配额和阈值会通过serviceSpecificUnits字段和unitQuotaThreshold字段携带给GGSN-C/PGW-C/SMF。

- 若GGSN-C/PGW-C/SMF使能事件计费功能，CHF未下发对应的事件计费配额给GGSN-C/PGW-C/SMF，则GGSN-C/PGW-C/SMF异常去激活用户。
- 若GGSN-C/PGW-C/SMF未使能事件计费功能，CHF下发对应的事件计费配额给GGSN-C/PGW-C/SMF，则GGSN-C/PGW-C/SMF忽略CHF下发的事件计费配额。

> **说明**
> 当事件计费的申请配额数为0时，GGSN-C/PGW-C/SMF也会将serviceSpecificUnits带给CHF，以向CHF表明该RG需要进行事件计费。

4：GGSN-C/PGW-C/SMF发送PFCP Session Establishment Request消息，请求建立GGSN-C/PGW-C/SMF会话。消息中携带Create URR（Usage Reporting Rule）信元组，指示计费相关信息。Create URR包含的主要子信元如 [表1](#ZH-CN_TOPIC_0274112318__table5217133513420) ：

*表1 Create URR包含的主要子信元*

| 子信元 | 信元解释 |
| --- | --- |
| URR ID | 唯一标识一个Create URR。 |
| Measurement Method | 标识计费方式，包括流量计费、时长计费或事件计费。GGSN-C/PGW-C/SMF通过EVENT字段指示GGSN-U/PGW-U/UPF该RG是否支持事件计费，设置为1标识支持事件计费。<br>说明：整个PDU会话过程中某RG是否使能事件计费不会发生变化。 |
| Reporting Triggers | 标识计费上报的触发条件，包含Periodic Reporting（定期上报）、Volume Quota（流量配额到上报）、Event Quota（事件配额到上报）、Volume Threshold（流量阈值到上报）、Event Threshold（事件阈值到上报）、Linked Usage Reporting（关联URR）等条件。 |
| Event Quota | 标识事件配额。 |
| Event Threshold | 标识事件阈值。 |
| Linked URR ID | 上报某个URR时，也会上报其关联的URR，例如A link B，当B上报时，也会上报A。 |

5：GGSN-U/PGW-U/UPF返回PFCP Session Establishment Response消息，GGSN-C/PGW-C/SMF会话建立成功。

6：PDU会话建立成功。

#### [业务触发申请事件计费配额](#ZH-CN_TOPIC_0274112318)

7.a和7.b：用户访问数据业务，GGSN-U/PGW-U/UPF识别该数据业务需要进行事件计费。当判断该RG无可用事件计费配额，GGSN-U/PGW-U/UPF向GGSN-C/PGW-C/SMF进行新业务配额申请（Start Trigger）。GGSN-C/PGW-C/SMF与GGSN-U/PGW-U/UPF的事件计费功能配置不一致时处理如下：

- GGSN-C/PGW-C/SMF本地配置支持事件计费功能，GGSN-U/PGW-U/UPF本地配置不支持事件计费功能时。GGSN-U/PGW-U/UPF上报告警ALM-81054 CP和UP关键配置不一致，GGSN-U/PGW-U/UPF按照未开启事件计费功能处理业务。
- GGSN-C/PGW-C/SMF本地配置不支持事件计费功能时，GGSN-U/PGW-U/UPF本地配置支持事件计费功能。GGSN-C/PGW-C/SMF上报[ALM-81026 接口信元不一致](../../../../../网络运维/故障处理/UNC告警处理/业务告警/SMF&GW-C&GGSN/ALM-81026 接口信元不一致_34629541.md)告警。

8：GGSN-C/PGW-C/SMF收到GGSN-U/PGW-U/UPF发送的事件计费配额申请，根据本地配置的事件计费配额（通过 **[ADD URR](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务策略/内容计费标识/增加URR（ADD URR）_09897158.md)** 中的 “EVENTRSUVALUE” 参数值决定）和GGSN-U/PGW-U/UPF申请的事件计费配额之和作为事件计费配额，通过serviceSpecificUnits字段发给CHF。

9：GGSN-C/PGW-C/SMF发送PFCP Session Report Response给GGSN-U/PGW-U/UPF。

10：GGSN-C/PGW-C/SMF收到CHF下发的事件计费配额，配额通过serviceSpecificUnits字段携带，阈值通过unitQuotaThreshold字段携带。

11：GGSN-C/PGW-C/SMF通过GGSN-C/PGW-C/SMF Session Modification Request/Response消息将事件计费配额和阈值发送给GGSN-U/PGW-U/UPF。

#### [事件计费配额或阈值耗尽，用量上报和配额申请](#ZH-CN_TOPIC_0274112318)

12.a和12.b：GGSN-U/PGW-U/UPF对事件计费进行配额用量监控，在用量达到阈值，或配额耗尽时向GGSN-C/PGW-C/SMF进行用量上报和申请新的配额。用量上报中包括对应的计费事件Trigger和对应成功事件次数的时间戳。具体信元见 [表2](#ZH-CN_TOPIC_0274112318__table16660175152713) 。如果GGSN-C/PGW-C/SMF未返回足够配额则阻塞或重定向报文。

*表2 Usage Report包含的主要子信元*

| 子信元 | 信元解释 |
| --- | --- |
| URR ID | 标识上报的URR。 |
| UR-SEQN | 唯一标识URR的Usage Report。 |
| EVETH | 事件计费阈值到触发上报。 |
| EVEQU | 事件计费配额耗尽触发上报。 |
| Event Time Stamp | 事件计费的时间戳。 |

#### [计费会话去激活，事件计费用量上报](#ZH-CN_TOPIC_0274112318)

17：GGSN-C/PGW-C/SMF向GGSN-U/PGW-U/UPF发起用户去激活，获取最后的事件计费用量。

18：GGSN-U/PGW-U/UPF向GGSN-C/PGW-C/SMF上报去活的事件计费用量。

19：GGSN-C/PGW-C/SMF向CHF发送Charging Data Request Release消息上报去活的事件计费用量。

20：CHF向GGSN-C/PGW-C/SMF发送Charging Data Response Release消息。

> **说明**
> - 事件计费支持和时长、流量计费同时进行，先满足的计费Trigger会上报所有维度配额的用量。
> - GGSN-U/PGW-U/UPF支持区分收彩信和发彩信两种应用，也可以区分彩信单发和群发，并根据群发对象个数统计事件次数而进行计费处理。比如一次彩信收发，与彩信内容大小无关。对于彩信群发，可以设置是按单次事件收费还是根据接收者的数量按多次事件收费。具体请参考《UDG 产品文档》。
