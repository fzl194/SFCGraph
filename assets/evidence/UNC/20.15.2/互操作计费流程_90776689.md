# 互操作计费流程

基于N26接口的互操作场景组网图如 [图1](#ZH-CN_TOPIC_0190776689__fig597585817319) 所示，当5GS与EPS互操作时，PGW-C与SMF必须共部署。此时SMF与PGW-C采用一致的融合计费机制和融合计费接口，即共部署时的PGW-C支持融合计费。

![](互操作计费流程_90776689.assets/zh-cn_image_0192556615_2.png)

以切换流程为例，当发生5GS与EPS间的切换时，PGW-C/SMF与CHF间已经存在计费会话，则在切换时，PGW-C/SMF收到AMF发送的 [Nsmf_PDUSession_UpdateSmContext Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md) 消息，消息中携带的 **ratType** 信元，通过RAT更新的计费Trigger感知到切换事件，PGW-C/SMF关闭所有统计，并发送Charging Data Request [Update]消息到CHF，CHF收到计费消息就会生成CHF-CDR，更新计费会话，同时PGW-C/SMF启动新计数器和时间戳。其它切换流程步骤同 [基于N26接口的EPS Fallback切换](../../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/5GS切换到EPS/基于N26接口的EPS Fallback切换_97193684.md) 和 [基于N26接口EPS到5GS切换](../../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/EPS切换到5GS/基于N26接口EPS到5GS切换_98278350.md) 。
