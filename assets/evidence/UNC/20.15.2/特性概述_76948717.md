#### [适用NF](#ZH-CN_CONCEPT_0276948717)

MME

#### [定义](#ZH-CN_CONCEPT_0276948717)

本地VLR是指MME备份用户的签约数据，用户恢复业务过程中取用本地信息，减少MME和HSS之间的流量，降低业务恢复对HSS造成的信令风暴冲击。

#### [客户价值](#ZH-CN_CONCEPT_0276948717)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 在MME故障场景下，降低业务恢复对HSS造成的信令风暴冲击，确保网络的安全运行。 |
| 用户 | 无。 |

#### [应用场景](#ZH-CN_CONCEPT_0276948717)

当运营商部署MME链式备份特性的场景下，建议启用本特性，降低业务恢复对HSS造成的信令风暴冲击，确保网络的安全运行。

#### [对系统的影响](#ZH-CN_CONCEPT_0276948717)

- 如果开启延迟发起Update Location流程功能，则[如果N-MME和S-MME不是同一个MME的场景下](#ZH-CN_CONCEPT_0276948717__impact_ref)，业务恢复过程中，Update Location完成之前，HSS发起的签约数据变更流程不会被处理，需等待Update Location流程之后获取新的签约数据。
- 现网部署了[WSFD-102003 SRVCC](../WSFD-102003 SRVCC_66442619.md)特性，并且IMS支持eSRVCC功能的场景下，STN-SR（Session Transfer Number for SRVCC）由IMS域的网元动态分配。如果配置为延迟发起Update Location流程，此场景下，在Update Location完成之前，如果IMS域的网元分配了新的STN-SR，新的STN-SR无法立即生效，需等待MME发起的Update Location流程获取新的签约数据后生效。

#### [应用限制](#ZH-CN_CONCEPT_0276948717)

如果用户签约的4G APN个数大于11时，此用户的签约数据不会被备份，业务恢复流程中，N-MME向HSS获取签约数据。

#### [原理概述](#ZH-CN_CONCEPT_0276948717)

启用 [WSFD-201201 MME链式备份](../WSFD-201201 MME链式备份_75538862.md) 特性后，R-MME（Redundant MME）不备份用户的签约数据信息。用户恢复业务过程中需要获取签约数据，此时N-MME（New Serving MME）会向HSS获取签约数据。S-MME（Serving MME）故障影响的用户范围较大，大量用户短时间内获取签约数据，MME和HSS之间信令激增，造成HSS出现拥塞或过载。为解决此问题，引入本特性，在用户首次接入S-MME后，将用户签约数据备份到R-MME。业务恢复流程中，N-MME直接从R-MME获取签约数据。

**签约数据管理**

[WSFD-201201 MME链式备份](../WSFD-201201 MME链式备份_75538862.md) 特性中，采用 “一次呼叫恢复” 模式时，签约数据备份过程如 [图1](#ZH-CN_CONCEPT_0276948717__fig03) 所示。

**图1** 签约数据备份过程

<br>

![](特性概述_76948717.assets/zh-cn_image_0276971071_2.png)

- 用户首次接入S-MME时，S-MME将签约数据备份到R-MME：
    1. S-MME向R-MME发送Create Subscription Information Request消息，携带IMSI和Subscription Data信元。
    2. R-MME向S-MME回复Create Subscription Information Response消息。
- S-MME收到HSS修改签约数据通知后，S-MME将签约数据更新备份到R-MME：
    1. S-MME向R-MME发送Update Subscription Information Request消息，携带Temporary Sdup User Identifier、Base Subscription Data、APN Configuration Profile、和Alive APN Configuration ID List信元。
    2. R-MME向S-MME回复Update Subscription Information Response消息。
- S-MME删除用户签约数据的时候，需要通知R-MME删除用户的签约数据。例如HSS发起Cancel Location流程，且Cancellation-Type为“SUBSCRIPTION_WITHDRAWAL”，S-MME删除用户签约数据。
    1. S-MME向R-MME发送Delete Subscription Information Request消息。
    2. R-MME向S-MME回复Delete Subscription Information Response消息。

**签约数据获取**

[WSFD-201201 MME链式备份](../WSFD-201201 MME链式备份_75538862.md) 特性中，采用 “一次呼叫恢复” 模式时，主叫业务恢复过程中，签约数据获取方式如下：

- 如果N-MME和S-MME是同一个MME，N-MME向R-MME发送Subscription Data Inquiry Request消息获取签约数据，R-MME返回Subscription Data Inquiry Response消息。
- 如果N-MME和S-MME不是同一个MME，N-MME发起Update Location流程，向HSS获取签约数据。在业务忙时，业务恢复会触发大量的Update Location Request流程，可能造成HSS拥塞。 UNC 提供参数 “HSS注册策略（DULR）” （ [**SET RESTOFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/MME容灾管理/容灾功能管理/设置容灾功能(SET RESTOFUNC)_72345713.md) ）控制是否延迟发起此流程，以减缓对HSS造成的冲击。
  如果设置成延迟发起Update Location流程，则主叫业务的用户承载信息恢复过程如 [图2](#ZH-CN_CONCEPT_0276948717__fig04) 所示。和未启用本特性时过程差异是步骤3e，其他步骤的描述参见 [WSFD-201201 MME链式备份](../WSFD-201201 MME链式备份_75538862.md) 。主叫业务结束之后， UNC 会在30分钟之内，发起Update Location流程，向HSS获取签约数据。
  3e. N-MME与S-MME之间的Sdup接口状态正常时，N-MME向S-MME发送Cancel Location Notification。以解决S1–MME接口故障，但是MME状态正常时，快速删除S-MME上的用户上下文。
