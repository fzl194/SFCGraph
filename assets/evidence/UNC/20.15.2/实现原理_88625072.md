# 实现原理

APN拥塞控制功能开启后， UNC 统计单位时间内特定APN下每个用户附着次数，当附着次数大于等于配置的阈值时，判断该终端为异常终端。若MM backoff timer分配开关打开， UNC 将MM backoff timer在附着拒绝消息中携带给终端，在该定时器时间内，UE不允许发起附着流程。如果M2M终端在定时器时间内仍然发起了附着流程， UNC 会拒绝该流程。

> **说明**
> APN拥塞控制功能不对VoLTE优先语音永久在线的用户和紧急呼叫用户进行拥塞控制。

拥塞控制功能：

1. M2M终端进行附着流程， UNC 获取HLR/HSS上固定终端的签约数据。
2. UNC 判断M2M终端的签约APN和命令 [**ADD APNCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/基于APN的MM拥塞控制_流控_寻呼优化配置/增加APN控制参数配置(ADD APNCTRL)_26145470.md) 中参数 “SUBSCRIBEDAPN” 配置的APN是否相同。
    - 是，则执行[3](#ZH-CN_CONCEPT_0188625072__step3)。
    - 否，则执行标准附着流程。
3. UNC 通过命令 [**ADD APNCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/基于APN的MM拥塞控制_流控_寻呼优化配置/增加APN控制参数配置(ADD APNCTRL)_26145470.md) 中的参数 “APNTYPE” 判断此APN是否为APN拥塞控制的APN。
    - 是，则执行[4](#ZH-CN_CONCEPT_0188625072__step4)。
    - 否，则执行标准附着流程。
4. UNC 通过命令 [**ADD APNCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/基于APN的MM拥塞控制_流控_寻呼优化配置/增加APN控制参数配置(ADD APNCTRL)_26145470.md) 中的参数 “ATTACHTHRESH” 判断M2M终端附着行为是否异常。
    - 是，则执行[5](#ZH-CN_CONCEPT_0188625072__step5)。
    - 否，则执行标准附着流程。
5. UNC 通过命令 [**ADD APNCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/基于APN的MM拥塞控制_流控_寻呼优化配置/增加APN控制参数配置(ADD APNCTRL)_26145470.md) 中的参数 “BACKOFFSW” 判断MM backoff timer开关是否打开。
    - 是，UNC将MM backoff timer在附着拒绝消息中携带给终端，在该定时器时间内，M2M终端不允许发起附着流程。如果M2M终端在定时器时间内仍然发起了附着流程，UNC会拒绝该流程。
    - 否，UNC直接拒绝M2M终端的附着请求。
