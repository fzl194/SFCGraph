#### [Active UE从LTE网络到WLAN网络的切换](#ZH-CN_CONCEPT_0276971683)

Active UE从LTE网络到WLAN网络的切换流程如 [图1](#ZH-CN_CONCEPT_0276971683__fig4) 所示。

**图1** Active UE从LTE网络到WLAN网络的切换

<br>

![](实现原理_76971683.assets/zh-cn_image_0000001182063295_2.png "点击放大")

1. UE通过E-UTRAN接入EPC网络，PGW-C与SGW-C在S5接口建有GTP隧道。
2. UE发现非可信非3GPP接入网络WLAN，决定切换到该WLAN网络。
3. UE执行非3GPP接入系统的鉴权和授权流程，3GPP AAA Server查询HSS，获取IMSI、PGW-C标识并在授权过程中返回给非3GPP接入系统。
4. 鉴权和授权成功后，触发附着流程。UE向ePDG发起认证请求，ePDG向3GPP AAA Server请求UE切换前连接的PDN相关信息及用户签约信息。
5. ePDG向PGW-C发送Create Session Request消息，建立PDN连接。消息中携带IMSI、APN、RAT type、ePDG TEID for control plane、PDN Address Allocation、EPS Bearer Identity、Default EPS Bearer QoS、ePDG Address for the user plane、ePDG TEID of the user plane、APN-AMBR、Selection Mode、Dual Address Bearer Flag、Trace Information、Charging Characteristics、UE Local IP Address IE、UE UDP Port和Indication Flags等参数。PGW-C使用IMSI和APN找到PDN连接，如果同一APN下建立了多个PDN连接，PGW-C通过比较用户的IP地址确定是哪个PDN连接发生切换。
  > **说明**
  > Indication Flags的Handover Indication（HI）标记位，标识E-UTRAN和WLAN互操作。
  >
  > - 携带HI标记位，表示E-UTRAN和WLAN互操作按照切换处理
  > - 未携带或者忽略HI标记位时，表示E-UTRAN和WLAN互操作按照激活处理。
6. **可选：**E-UTRAN到ePDG切换会导致RAT，IP-CAN Type，AN-GW address等事件更新，如果PCRF签约了相关的Event-trigger，则PGW-C发起IP-CAN会话更新流程。
7. PGW-C向3GPP AAA Server更新PGW-C标识和APN信息，消息中包含PGW-C所在的PLMN信息。
8. PGW-C向ePDG返回Create Session Response消息，消息中携带PDN GW Address for the user plane、PDN GW TEID of the user plane、PDN GW TEID of the control plane、PDN Address Allocation、EPS Bearer Identity、EPS Bearer QoS、APN-AMBR、Charging ID、Cause等参数。
9. ePDG将PGW-C分配给UE的IP地址传给UE。附着流程完成。
10. UE和ePDG之间的隧道建立成功，ePDG和PGW-C间的GTP隧道建立成功，UE可以发送和接收IP数据报文。
11. **可选：**如果UE需要连接多个PDN，重复步骤3～9，完成其他PDN连接的建立。
12. PGW-C向SGW-C、MME发起3GPP网络中的承载删除流程，PGW-C、SGW-C、MME释放切换前的EPS承载信息。

#### [Active UE从WLAN网络到LTE网络的切换](#ZH-CN_CONCEPT_0276971683)

Active UE从WLAN网络到LTE网络的切换流程：

**图2** Active UE从WLAN网络到LTE网络的切换

<br>

![](实现原理_76971683.assets/zh-cn_image_0000001182183213_2.png "点击放大")

1. UE通过ePDG接入EPC网络，PGW-C与ePDG在S2b接口建有GTP隧道。
2. UE发现E-UTRAN系统，决定切换到该LTE网络。
3. UE向MME发起切换附着请求，Attach Request消息中携带在非3GPP网络中接入的APN。E-UTRAN将该附着请求路由至MME。
4. MME与HSS交互，对UE进行鉴权。
5. 鉴权成功后，MME执行位置更新流程和签约数据获取流程。MME根据签约信息获取切换前UE接入的PGW-C信息。
6. MME向SGW-C发送Create Session Request消息，建立PDN连接，消息中携带IMSI、MME Context ID、PDN-GW address、Handover Indication（HI标记位）、APN等信息。
7. SGW-C向PGW-C发送Create Session Request消息。PGW-C使用IMSI和APN找到PDN连接，如果同一APN下建立了多个PDN连接，PGW-C通过比较用户的IP地址确定是哪个PDN连接发生切换。PGW-C不会在此处立即切换非3GPP隧道到3GPP隧道。
8. **可选：**ePDG到E-UTRAN切换会导致RAT，IP-CAN Type，AN-GW address等事件更新，如果PCRF签约了相关的Event-trigger，则PGW-C发起IP-CAN会话更新流程。
  如果新的PCC规则要求为UE创建专有承载，则PGW-C先发送Create Session Response，再发送专有承载创建请求。专有承载的创建可能晚于non-3GPP隧道切换到3GPP隧道。
9. PGW-C回复Create Session Response消息给SGW-C。该消息携带UE在连接非3GPP接入系统时获取的IP地址。
10. SGW-C回复Create Session Response消息给MME。该消息携带UE的IP地址。
11. 无线承载和接入承载也在3GPP接入系统创建。
12. MME向SGW-C发送Modify Bearer Request消息，携带eNoeB地址、eNodeB TEID和Handover Indication。
13. SGW-C向PGW-C发送Modify Bearer Request消息，立即触发非3GPP接入隧道向3GPP接入隧道的切换。本步骤中，PGW-C删除非可信非3GPP接入的旧PCC规则，使用E-UTRAN接入的新PCC规则进行计费。
14. PGW-C向SGW-C发送Modify Bearer Response消息，表示收到承载更新请求消息。
15. SGW-C向MME发送Modify Bearer Response消息，携带EPS Bearer Identity，表示收到承载更新请求消息。
16. UE和SGW-C之间的无线承载和接入承载建立成功，SGW-C和PGW-C间的GTP隧道建立成功，UE通过E-UTRAN系统发送和接收数据报文。PGW-C通知3GPP AAA更新PGW-C地址，即删除用户与该PGW-C地址的绑定。
17. **可选：**如果UE需要连接多个PDN，重复步骤3～15，完成其他PDN连接的建立。
18. PGW-C发起非可信非3GPP接入系统中的PDN连接删除流程。
