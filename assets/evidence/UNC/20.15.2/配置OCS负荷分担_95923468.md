## [操作场景](#ZH-CN_OPI_0295923468)

UNC支持配置OCS group，group间是主备关系，group内绑定的OCS间是负荷分担关系。其中负荷分担又分为以下几种情况。

- OCS平均负荷分担
  同一group内的所有OCS平均负荷分担用户的在线计费消息处理。默认情况下采用此种方式。
- 基于用户号段进行OCS负荷分担
  可以通过以下方式实现基于用户号段选择OCS：
    - 根据用户的MSISDN/IMSI号段选择UserProfile，再根据UserProfile下绑定的在线计费模板来选择不同的OCS，实现将用户分布在不同的OCS上进行在线计费处理，降低对单个OCS性能的要求。
    - 根据用户的MSISDN/IMSI号段直接选择OCS，实现将用户分布在不同的OCS上进行在线计费处理。
  一般情况下，尽量避免同时配置上述两种方式选择OCS。当两种条件都满足的情况下，优先根据UserProfile下关联的OCS进行在线计费处理。
- 基于百分比例进行OCS负荷分担
  根据预先设置的百分比例选择OCS，百分比例的基数是累计选择此OCS的次数，实现将用户按比例分布在不同的OCS上进行在线计费，这是实现根据单个OCS性能选择OCS的一种方式。

> **说明**
> 适用于 GGSN、 PGW-C。

## [操作步骤](#ZH-CN_OPI_0295923468)

1. 配置在线计费中GGSN/PGW-C的设备标识。
  配置在线计费中通信的GGSN/PGW-C的设备标识，重复执行本步骤可添加多个Gy应用的本地信息。
  > **说明**
  > UNC 的Gx应用和Gy应用的本端主机名可以相同，但是同时要求 “REALMNAME” 参数和 “PRODUCTNAME” 参数也必须相同。
  **ADD DIAMLOCINFO**
2. 配置平均进行OCS负荷分担。
    a. 配置OCS信息。重复执行本步骤可增加多个OCS信息。
      **ADD OCS**
    b. 配置OCS组。重复执行本步骤可增加多个OCS组信息。
      **ADD OCSGROUP**
    c. 配置平均选择在线计费系统的OCS，不配置可选参数即可实现。
      **ADD OCSBINDING**
3. 配置基于用户号段进行OCS负荷分担。
    a. 配置OCS信息。重复执行本步骤可增加多个OCS信息。
      **ADD OCS**
    b. 配置IMSI及MSISDN号码段。
      **ADD IMSIMSISDNSEG**
    c. 配置IMSI及MSISDN号码段组。
      **ADD SUBSCRIBERIDSEGGRP**
    d. 配置OCS组。重复执行本步骤可增加多个OCS组信息。
      **ADD OCSGROUP**
    e. 配置基于用户号段IMSI/MSISDN选择在线计费系统的OCS。
      **ADD OCSBINDING**
4. 配置基于百分比例进行OCS负荷分担。
    a. 配置OCS信息。重复执行本步骤可增加多个OCS信息。
      **ADD OCS**
    b. 配置OCS组。重复执行本步骤可增加多个OCS组信息。
      **ADD OCSGROUP**
    c. 配置基于百分比例选择在线计费系统的OCS。
      **ADD OCSBINDING**
5. 配置主从OCS组。
  **ADD DCCTEMPLATE**

## [任务示例1](#ZH-CN_OPI_0295923468)

任务描述

如 [图1](#ZH-CN_OPI_0295923468__fig192110497307) 所示，GGSN/PGW-C与4台OCS连接。考虑到用户负荷，规划两个OCS组，OCS间属于主备关系，OCS组内的两台OCS平均负荷分担所有的在线计费用户。

**图1** 多个OCS平均负荷分担示例组网图

<br>

![](配置OCS负荷分担_95923468.assets/zh-cn_image_0295923395_2.png)

脚本

1. 配置GGSN/PGW-C设备标识。
  ```
  ADD DIAMLOCINFO: HOSTNAME="pgw1",REALMNAME="huawei.com",PRODUCTNAME="unc@huawei.com";
  ```
2. 配置OCS平均负荷分担。
  //配置OCS设备标识。
  ```
  ADD OCS:OCSHOSTNAME="ocs_host1",REALMNAME="ocs_realm";
  ```
  ```
  ADD OCS:OCSHOSTNAME="ocs_host2",REALMNAME="ocs_realm";
  ```
  ```
  ADD OCS:OCSHOSTNAME="ocs_host3",REALMNAME="ocs_realm";
  ```
  ```
  ADD OCS:OCSHOSTNAME="ocs_host4",REALMNAME="ocs_realm";
  ```
  ```
  ADD DIAMPEERADDR:HOSTNAME="ocs_host1",ADDRTYPE=IPv4,IPV4ADDRESS="192.168.40.10";
  ```
  ```
  ADD DIAMPEERADDR:HOSTNAME="ocs_host2",ADDRTYPE=IPv4,IPV4ADDRESS="192.168.40.11";
  ```
  ```
  ADD DIAMPEERADDR:HOSTNAME="ocs_host3",ADDRTYPE=IPv4,IPV4ADDRESS="192.168.40.12";
  ```
  ```
  ADD DIAMPEERADDR:HOSTNAME="ocs_host4",ADDRTYPE=IPv4,IPV4ADDRESS="192.168.40.13";
  ```
  ```
  ADD OCSGROUP:OCSGRPNAME="ocs_gra";
  ```
  ```
  ADD OCSGROUP:OCSGRPNAME="ocs_grb";
  ```
  ```
  ADD OCSBINDING:OCSGRPNAME="ocs_gra",OCSHOSTNAME="ocs_host1";
  ```
  ```
  ADD OCSBINDING:OCSGRPNAME="ocs_gra",OCSHOSTNAME="ocs_host2";
  ```
  ```
  ADD OCSBINDING:OCSGRPNAME="ocs_grb",OCSHOSTNAME="ocs_host3";
  ```
  ```
  ADD OCSBINDING:OCSGRPNAME="ocs_grb",OCSHOSTNAME="ocs_host4";
  ```
  //除此之外，如果同一个OCS有多个IP也可以配置OCS内部负荷分担，不同的用户会话将会负荷分担到不同链路上。如下是为OCS服务器配置IP地址，其IP地址为192.168.40.14、192.168.40.15、192.168.40.16、192.168.40.17。这样，一个OCS就有4个IP，所有使用这个OCS的用户会在这四条IP对应的链路上负荷分担。
  ```
  ADD OCS:OCSHOSTNAME="ocs_host5",REALMNAME="ocs_realm";
  ```
  ```
  ADD DIAMPEERADDR:HOSTNAME="ocs_host5",ADDRTYPE=IPv4,IPV4ADDRESS="192.168.40.14";
  ```
  ```
  ADD DIAMPEERADDR:HOSTNAME="ocs_host5",ADDRTYPE=IPv4,IPV4ADDRESS="192.168.40.15";
  ```
  ```
  ADD DIAMPEERADDR:HOSTNAME="ocs_host5",ADDRTYPE=IPv4,IPV4ADDRESS="192.168.40.16";
  ```
  ```
  ADD DIAMPEERADDR:HOSTNAME="ocs_host5",ADDRTYPE=IPv4,IPV4ADDRESS="192.168.40.17";
  ```
  ```
  ADD OCSGROUP:OCSGRPNAME="ocs-gr";
  ```
  ```
  ADD OCSBINDING:OCSGRPNAME="ocs-gr",OCSHOSTNAME="ocs_host5";
  ```
3. 配置主从OCS组。
  ```
  ADD DCCTEMPLATE:DCCTMPLTNAME="dcc-test",PRIOCSGRPNAME="ocs_gra",SECOCSGRPNAME="ocs_grb";
  ```
