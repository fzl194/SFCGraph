## [任务示例](#ZH-CN_OPI_0000001553482538)

任务描述

任务1：以AMF到UDM的局向为例，配置基于IP组生效的HTR流控。

任务2：以AMF到UDM的局向为例，配置基于网元类型生效的HTR流控。

任务3：以SMF到UDM的局向为例，配置基于IP组生效的HTR流控。

任务4：以SMF到UDM的局向为例，配置基于网元类型生效的HTR流控。

任务5：以SMF到PCF的局向为例，配置基于网元类型生效的HTR流控。

脚本

**任务1脚本**

//开启HTR流控License项的配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2SHFCUNC01", SWITCH=ENABLE;
```

//（可选）设置HTR流控全局配置。

```
SET HTTPHTRCFG: SWITCH=ON, STARTTHD=12, STOPTHD=1, TARGETTHD=6, PERIOD=5, STARTTIMES=4, STOPTIMES=8, TIMEOUT=2000, DECRATIO=10, INCRATIO1=5, INCRATIO2=18;
```

//（可选）配置HTR流控安全边界。

```
ADD HTTPHTRMS: LOCALNFTYPE=NFTypeAMF, PEERNFTYPE=NFTypeUDM, MINTHD=210, MAXTHD=1500;
```

//增加HTTP流控的IP地址组信息。

```
ADD HTTPFCIPGRP: INDEX=0, GROUP=0, IPTYPE=IPv4, IPV4ADDR="192.168.1.1", IPV4MASK="255.255.255.0";
ADD HTTPFCIPGRP: INDEX=1, GROUP=0, IPTYPE=IPv6, IPV6ADDR="2001:0db8::1", IPV6MASK=128;
```

//增加HTR流控生效的HTTP局向（基于IP组生效）。

```
ADD HTTPOFC: OFCIDX=1, OFCTYPE=IPGROUP, IPGROUP=0, IPGRPITEM=HTRIPGRP-1, LOCALNFTYPE=NFTypeAMF, OFCNAME="AMF2UDMIPHTR";
```

//（可选）配置由于HTR流控导致获取签约数据流程失败时，AMF发送给UE的拒绝原因值。

```
SET NGMMPROCTRL: PROT=UDM_GET_SUBSCRIBER_DATA, UDMSUBHTTPHTRFC=0;
```
