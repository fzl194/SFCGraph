## [操作场景](#ZH-CN_OPI_0000001327118049)

本操作指导介绍在运行网络中激活基于AMF的寻呼流控。

基于AMF的寻呼流控是指AMF支持区分业务优先级，保障高优先业务优先寻呼，并设置N2接口指定消息的接收/发送流控速率上限或者对于特定的寻呼消息不进行重发，从而控制N2接口上发送消息的速率，减少AMF对gNodeB的冲击。

## [任务示例](#ZH-CN_OPI_0000001327118049)

任务描述

AMF控制N2接口上发送PAGING消息速率及消息放通优先级。

脚本

//设置PAGING消息类型的固定速率门限。

```
SET N2FIXEDFCBYMSG:MSGTYPE=PAGING,FCSWITCH=On,THRESHOLD=2000;
```

//配置基于ARP识别Paging消息在N2口固定速率流控期间放通的优先级。

```
ADD NGPAGINGARPPRI: ARP=1, PRIORITY=PAGING_HIGH;
```

//配置基于DNN识别Paging消息在N2口固定速率流控期间放通的优先级。

```
ADD NGPAGINGDNNPRI: DNN="IMS", PRIORITY=PAGING_HIGH;
```

//配置N2模式下基于TALIST的寻呼消息不进行重发的TAC区间。

```
ADD NGTALISTPAGINGCFG: MCC="123", MNC="45",TACSTART="11BF85",TACEND="11BFFF";
```
