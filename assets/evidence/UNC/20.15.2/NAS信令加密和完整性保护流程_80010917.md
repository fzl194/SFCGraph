#### [业务场景](#ZH-CN_TOPIC_0000001480010917)

UE和网络完成鉴权流程后，通过NAS信令加密和完整性保护流程，协商后续通信过程中信令加密和完整性保护所使用的安全算法和密钥。NAS安全算法协商完成后，AMF与UE之间的NAS消息都会进行加密和完整性保护，提高网络的安全性。AMF对所有发送的NAS消息进行加密和完整性保护，对所有接收到的NAS消息进行完整性校验和解密。

#### [信令流程](#ZH-CN_TOPIC_0000001480010917)

![](NAS信令加密和完整性保护流程_80010917.assets/zh-cn_image_0000001447634097_2.png)

1. 内部流程：AMF启动完整性保护。
  相关机制：AMF结合配置的算法优先级和UE在初始NAS消息中上报的安全能力，选择保护算法、计算密钥，并启动完整性保护。
2. 消息交互：AMF发送[Security mode command](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Security mode command_76784934.md)给UE。
  触发条件：UE和网络完成鉴权流程后，AMF启动NAS信令加密和完整性保护流程。
  关键信元：
  | Selected NAS security algorithms | 加密和完整性保护的5G算法。 |
  | --- | --- |
  | nAS key set identifier | 网络侧分配的NAS密钥集标识，用于识别K<br>AMF<br>和鉴权成功后产生的安全上下文。 |
  | Replayed UE security capabilities | AMF返回的UE安全能力，UE用以判断是否被攻击者篡改过。 |
  相关机制：AMF发送的Security mode command已经通过选择的完整性保护算法进行了完整性保护。如果K <sub>AMF</sub> 有更新，则AMF会在该消息中携带Additional 5G security information信元请求UE重新推衍K <sub>AMF</sub> 。如果初始NAS消息经过保护但未通过完整性检查、或者AMF无法解密完整的初始NAS消息，则AMF会在该消息中携带Additional 5G security information信元，请求UE重新发送完整初始NAS消息。
3. 内部流程：UE进行完整性校验。
  相关机制：UE收到Security mode command消息后，对该消息进行完整性校验。校验通过后，比较AMF返回的UE安全能力是否与自身保存的安全能力匹配。如果匹配，则UE启动加密和完整性保护。如果AMF发送的Security mode command消息中要求UE重新推衍K <sub>AMF</sub> ，则UE将重新推衍K <sub>AMF</sub> 。
4. 消息交互：UE发送[Security mode complete](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Security mode complete_76784268.md)给AMF。
  触发条件：UE收到Security mode command后进行响应。
  关键信元：
  | NAS message container | 包含完整初始NAS消息。 |
  | --- | --- |
  相关机制：UE发送的Security mode complete已经经过了加密和完整性保护。如果AMF在Security mode command消息中携带了请求完整初始NAS消息的标志，或者UE发送的初始NAS消息未经过保护，那么Security mode complete消息将携带包含完整初始NAS消息的NAS message container。
5. 内部流程：AMF启动下行信令加密。
  相关机制：收到Security mode complete消息后对其进行完整性校验和解密，并开始对下行信令进行加密。
