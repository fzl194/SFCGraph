# Diff-Serv模型

Differentiated Service模型即区分服务模型，简称Diff-Serv，是一种适用于骨干网络，可满足多种服务需求的IP QoS模型。在Diff-Serv体系中，用户通过标记分组的DS（Differentiated Service）字段来申请不同层次的服务。DS字段的前6位称为DSCP，具有相同DSCP值的分组集合称为BA（Behaviour Aggregate）。路由器保留有DSCP到每跳转发行为PHB（Per-Hop Behaviours）的映射，PHB是指满足某类转发需求的行为，如流量监管、流量整形、队列管理等。分组进入路由器时，根据其DSCP标记划归一个BA并被选取特定的PHB进行转发。

- DSCP
  DSCP用来携带IP报文的分类信息，由RFC 2474定义，如下图所示：
  **图1** DSCP示意图
  ![](Diff-Serv模型_66204610.assets/zh-cn_image_0168257530_2.png)
  即DSCP是IPv4的数据报文头部里的TOS（Type of Service，服务类型）字节中的低6位，或IPv6的数据报文头部里的TC（Traffic Class，流量等级）字节中的低6位。网络中支持DiffServ技术的各设备根据DSCP值选择相应的转发行为。
- 每跳转发行为PHB
  PHB，是指满足业务转发需求的行为，如流量监管、流量整形、队列管理等。 UNC 支持DSCP到PHB的映射。
  根据用户报文的DSCP归划对应的具有相同DSCP值的分组集合BA并被选取特定的PHB进行转发。如果用户报文的DSCP未定义到PHB的映射， UNC 将选择采用缺省PHB进行转发处理。目前，IETF定义了四种标准的PHB：
  *表1 标准PHB*

  | PHB | 含义 | 适用场景 |
  | --- | --- | --- |
  | CS（Class Selector，类选择码） | CS表示类选择码，代表的服务等级与在现有网络中使用的IP Precedence相同。 | 主要用于协议报文，因为如果这些报文无法接收的话会引起协议中断，所以应该优先保障。 |
  | EF（Expedited Forwarding，加速转发） | 发出的信息流速率在任何情况下必须获得等于或大于设定的速率。 | 主要用于低丢包率、低延迟，确定带宽的转发服务，适用于信令的转发。 |
  | AF（Assured Forwarding，确保转发） | 在没有超过最大允许带宽时能够确保转发，一旦超出最大允许带宽，则将转发行为分为4 个服务等级，其中每个服务等级都被分配了不同的带宽资源；每个服务等级又可划分为高、中、低3 个不同的丢弃优先级，根据被丢弃的可能性划分出3档。 | 主要用于要求较低的延迟、低丢包率、高可靠性，适用于数据可靠性要求高的业务，如多媒体业务的转发。 |
  | BE（Best-Effort，尽力而为） | 对应于传统的IP分组投递服务，只关注可达性，其他方面不做任何要求。 | 适用于对带宽、延迟性能不敏感的WWW文件传输、E-Mail等业务。 |

- RFC推荐的DSCP
  DSCP共有64个取值（0-63），其中下述16个取值为RFC推荐的DSCP。
  *表2 常用DSCP值及其用途*

  | PHB | DSCP取值（二进制） | 常见用途 |
  | --- | --- | --- |
  | CS6、CS7 | 110 000、111 000 | CS6和CS7默认用于协议报文，比如说OSPF报文，BGP报文等应该优先保障，因为如果这些报文无法接收会引起协议中断。 |
  | EF | 101 110 | EF用于承载语音的流量，因为语音要求低延迟，低抖动，低丢包率，是仅次于协议报文的最重要的报文。<br>说明：EF提供的是低时延服务，应该具有最低的抖动和丢包率，因而必须限制EF的专用带宽，以免其他服务得不到可用带宽。 |
  | AF4 | - 100 010<br>- 100 100<br>- 100 110 | AF4用来承载语音的信令流量。<br>说明：语音流量必须优先于语音的信令，因为语音信令是语音的呼叫控制，在接通的时候等待几秒钟是可以忍受的，但是在通话过程的中断是绝对不能允许的。 |
  | AF3 | - 011 010<br>- 011 100<br>- 011 110 | AF3可以用来承载IPTV的直播流量。直播的实时性强，需要有连续性和大吞吐量的保证。 |
  | AF2 | - 010 010<br>- 010 100<br>- 010 110 | AF2可以用来承载VoD（Video on Demand）的流量。VoD要求的实时性不如IPTV直播强，允许有延迟或者缓冲。 |
  | AF1 | - 001 010<br>- 001 100<br>- 001 110 | AF1可以承载专线业务，因为IPTV和语音是运营商最关键的业务，需要最优先保证，其次才是专线业务。面向银行之类需要钻石级保证的业务，可以安排为AF4甚至为EF。 |
  | BE | 000 000 | 当前Internet尽力而为的服务。 |
  > **说明**
  > - AF类的DSCP二进制取值含义：
  >     - 前三位表示类别，即AF1~AF4。
  >     - 第4、5位用来表示丢弃优先级，数值越大，则丢弃优先级越高，即01（1）、10（2）、11（3）分别对应丢弃优先级低（L）、中（M）、高（H）。
  >     - 最后一位取值固定为0。
  > - 通常使用AF服务类等级加丢弃优先级表示AF类的DSCP值。例如，AF43表示服务类等级为AF4，丢包率为H；AF42表示服务类等级为AF4，丢包率为M；AF41表示服务类等级为AF4，丢包率为L。
