#### [适用NF](#ZH-CN_TOPIC_0215591144)

AMF、SMF 、NRF

#### [定义](#ZH-CN_TOPIC_0215591144)

UNC 与外部通信或 UNC 内部服务间通信时，支持安全传输协议TLS、IPsec、SSH、SSL等等，通过证书进行身份认证和数据加密。 UNC 提供证书加载、查询、删除、有效性检查和初始证书等管理功能，提升系统安全性。

#### [客户价值](#ZH-CN_TOPIC_0215591144)

| **受益方** | **受益描述** |
| --- | --- |
| 运营商 | 保护系统通信安全性，提升证书更新效率。 |
| 用户 | 用户不感知该特性。 |

#### [应用场景](#ZH-CN_TOPIC_0215591144)

- UNC对外业务接口：SBI接口支持TLS协议，通过证书进行认证和加密，涉及证书管理。
- UNC对外维护接口：与MAE、VNFM等对接时通过证书进行认证和加密，涉及证书管理。
- UNC内部服务间通信：内部服务间通信时通过证书进行认证和加密，涉及证书管理。

#### [对系统的影响](#ZH-CN_TOPIC_0215591144)

更新 UNC 内服务间通信证书需要重启网元所有POD。

#### [应用限制](#ZH-CN_TOPIC_0215591144)

仅支持PEM格式的证书，非PEM格式的证书需要转换为PEM格式 。

#### [原理概述](#ZH-CN_TOPIC_0215591144)

**证书类**

UNC 支持管理的证书类型如 [表1](#ZH-CN_TOPIC_0215591144__table18459104917428) 所示。

*表1 证书类型*

| 类型 | 类型 | 描述 |
| --- | --- | --- |
| CA证书 | CA证书 | 又称为信任证书、根证书，由CA（Certification Authority，证书颁发中心）颁发，在PKI（Public Key Infrastructure，公共密钥基础设施）管理范围内全局统一。 |
| 设备证书 | 设备证书 | 又称为身份证书、公钥证书、Certificate，由CA证书签发获得，通常用于加密会话或数据。 |
| 设备证书 | 私钥 | private key，与设备证书成对出现，用于解密设备证书加密的数据。 |
| 设备证书 | 私钥密码 | 用于对私钥文件进行加密保护。 |
| 证书吊销列表 | 证书吊销列表 | Certificate Revocation List（以下简称CRL），用于公布不再有效的数字证书。CRL是一种离线的证书状态信息，基本信息包括被吊销证书的序列号、吊销日期、吊销原因、颁发者等等。CRL一般情况下由PKI定期发布更新，PKI管理范围内全局统一。 |

**证书管理**

UNC 提供证书加载、查询、删除、有效性检查和初始证书等管理功能，如 [表2](#ZH-CN_TOPIC_0215591144__table18710517192716) 所示。

*表2 证书管理功能*

| 功能 | 描述 |
| --- | --- |
| 初始证书 | 为确保系统安全，<br>UNC<br>在系统内含有访问OM Portal和FusionStage使用的华为初始证书，并随软件包发布用于SBI接口加密和<br>UNC<br>内服务间通信使用的华为初始证书。 |
| 加载 | 运维人员可以通过OM Portal的证书管理功能、MML命令上传和加载CA证书、设备证书和CRL。 |
| 查询 | 运维人员可以通过OM Portal的证书管理功能、MML命令查看<br>UNC<br>已有的CA证书、设备证书和CRL。 |
| 删除 | 运维人员可以通过OM Portal的证书管理功能、MML命令删除<br>UNC<br>已有的CA证书、设备证书和CRL。 |
| 有效性检查 | UNC<br>支持定期检查证书有效期，上报证书即将到期告警、证书过期和证书无效告警。 |

> **注意**
> - 软件中含有仅用于部署阶段入网与初始化的华为初始证书，系统部署并初始化完成后客户应对初始证书进行更换。
> - 对于将华为初始证书作为业务证书使用而导致的安全风险和安全事件，由客户自行处置并承担后果。
> - 华为初始证书有效期参见各证书列表说明。
> - 初始证书过期后，使用初始证书的业务会中断。
> - 华为建议客户通过部署PKI系统对现网设备和软件进行证书的签发，并做好证书的生命周期管理（为保证安全性推荐使用最短有效期的证书）。
