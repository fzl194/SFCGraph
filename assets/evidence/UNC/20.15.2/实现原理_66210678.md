#### [报文优先级](#ZH-CN_TOPIC_0166210678)

UNC 将报文分为四个优先级，从高到低依次为优先级1~4，作为报文进入接收队列或发送队列的依据，找不到报文对应的优先级时，将报文加入“优先级4”的队列。建议为高优先级的业务配置入高优先级的队列，以便系统优先处理。

UNC 有两种方式确定报文的优先级：按照报文的DSCP进行映射，或者按照报文的业务类别进行映射。DSCP原理详见

- **DSCP优先级映射表**
  按照报文的DSCP进行优先级映射时，不同的DSCP取值对应不同的优先级，保存在DSCP优先级映射表中。缺省情况下的DSCP优先级映射表如下表所示。
  *表1 DSCP优先级缺省映射表*

  | 分类 | DSCP取值（二进制） | 映射优先级 |
  | --- | --- | --- |
  | CS6、CS7 | 110 000、111 000 | 优先级1 |
  | EF | 101 110 | 优先级2 |
  | AF4 | - 100 010<br>- 100 100<br>- 100 110 | 优先级2 |
  | AF3 | - 011 010<br>- 011 100<br>- 011 110 | 优先级2 |
  | AF2 | - 010 010<br>- 010 100<br>- 010 110 | 优先级3 |
  | AF1 | - 001 010<br>- 001 100<br>- 001 110 | 优先级3 |
  | BE | 000 000 | 优先级4 |
  > **说明**
  > - [表1](#ZH-CN_TOPIC_0166210678__table120761313615)中的映射关系为UNC默认配置，不能删除，可以通过**[MOD DSCPPRI](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/DSCP优先级映射管理/修改DSCP映射优先级配置表(MOD DSCPPRI)_26146208.md)**命令修改。
  > - 非系统默认的DSCP值（即不在[表1](#ZH-CN_TOPIC_0166210678__table120761313615)中包含的DSCP值）的映射优先级可以分别通过**[ADD DSCPPRI](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/DSCP优先级映射管理/增加DSCP映射优先级配置表(ADD DSCPPRI)_26306018.md)**命令、**[MOD DSCPPRI](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/DSCP优先级映射管理/修改DSCP映射优先级配置表(MOD DSCPPRI)_26146208.md)**命令、**[RMV DSCPPRI](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/DSCP优先级映射管理/删除DSCP映射优先级配置表(RMV DSCPPRI)_72345807.md)**命令新增、修改和删除。
  > - 删除非系统默认的DSCP值的映射优先级后，对应DSCP值的报文将入“优先级4”的队列。
- **业务类别优先级映射表**
  当发送数据报文按业务类别入发送队列时，采用下表的映射规则：
  *表2 业务类别优先级映射表*

  | 业务类别 | 优先级 |
  | --- | --- |
  | 会话类、流类 | 优先级2 |
  | 交互类 | 优先级3 |
  | 背景类 | 优先级4 |

#### [DSCP在UNC上的应用过程](#ZH-CN_TOPIC_0166210678)

UNC 需要处理接收和发送两个方向的报文，每个方向上的报文又分为信令报文和数据报文两种，共四种组合。

- 对接收的报文，UNC处理信令报文和数据报文的方式相同，即根据接收报文携带的DSCP值及UNC上的DSCP优先级映射表将报文放入不同优先级的调度队列。
- 对发送的报文，UNC对信令报文和数据报文的DSCP处理不同：
    - 对于信令报文，其DSCP由其通过哪个接口传输决定，按照DSCP优先级映射表进入发送队列。
    - 对于数据报文，其DSCP的填写方式由**[SET IFDSCP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/接口DSCP管理/设置接口DSCP配置(SET IFDSCP)_26306022.md)**命令中的“数据DSCP使用策略”参数决定，进入发送队列的优先级映射方式由**[MOD DSCPPRI](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/DSCP优先级映射管理/修改DSCP映射优先级配置表(MOD DSCPPRI)_26146208.md)**命令中的“数据报文的映射队列策略”参数决定，两个参数的取值如下表所示：
      *表3 数据报文的DSCP使用策略及映射队列策略*

      | 参数 | 取值 | 所属命令 | 说明 |
      | --- | --- | --- | --- |
      | 数据DSCP使用策略 | - “透传DSCP值”：根据**[SET IFDSCP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/接口DSCP管理/设置接口DSCP配置(SET IFDSCP)_26306022.md)**命令中的“SNDCP转GTP数据的DSCP值”参数填写DSCP值。<br>- “修改DSCP值”：根据数据报文的业务类型填写DSCP值。 | **[SET IFDSCP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/接口DSCP管理/设置接口DSCP配置(SET IFDSCP)_26306022.md)** | “透传DSCP值”适用于2G网络。 |
      | 数据报文的映射队列策略 | - “基于业务类型”：数据报文按**业务类别优先级映射表**进入发送队列。<br>- “基于DSCP”：数据报文按**DSCP优先级映射表**进入发送队列。 | **[MOD DSCPPRI](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/DSCP优先级映射管理/修改DSCP映射优先级配置表(MOD DSCPPRI)_26146208.md)** | - |

UNC 对不同报文的DSCP具体处理过程如下：

- **对接收的信令报文、数据报文的处理**
  **图1** 信令报文、数据报文接收处理示意图

  ![](实现原理_66210678.assets/zh-cn_image_0251970968_2.png)
  外部信令或数据报文进入 UNC 后， UNC 对报文依次进行如下DSCP处理：
    1. 根据报文携带的DSCP值及UNC上的DSCP优先级映射表，将报文放入相应的优先级队列进行排队。
    2. 根据高优先级的队列优先调度的原则分别取出队列内的IP报文送给后端处理。

- **对发送的信令报文的处理**
  **图2** 信令报文发送处理示意图

  ![](实现原理_66210678.assets/zh-cn_image_0251970965_2.png "点击放大")
  发送信令报文时， UNC 对报文依次进行如下DSCP处理：
    1. 根据报文所属的接口类型填写报文的DSCP值。
    2. 根据UNC上的DSCP优先级映射表，将报文放入4个优先级队列进行排队。
    3. 根据高优先级的队列优先调度原则分别取出队列内的IP报文进行发送处理。
  > **说明**
  > **[SET IFDSCP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/接口DSCP管理/设置接口DSCP配置(SET IFDSCP)_26306022.md)** 命令中介绍了支持单独配置DSCP的信令面接口及对应的DSCP系统初始设置值。
- **对发送的数据报文的处理**
  **图3** 数据报文发送处理示意图

  ![](实现原理_66210678.assets/zh-cn_image_0251970967_2.png "点击放大")
  发送数据报文时， UNC 对报文依次进行如下DSCP处理：
    1. 根据**[SET IFDSCP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/接口DSCP管理/设置接口DSCP配置(SET IFDSCP)_26306022.md)**命令中的“数据DSCP使用策略”填写DSCP值。
    2. 根据**[MOD DSCPPRI](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/QoS管理/DSCP/DSCP优先级映射管理/修改DSCP映射优先级配置表(MOD DSCPPRI)_26146208.md)**命令中的“数据报文的映射队列策略”将IP报文放入4个优先级队列进行排队。
    3. 根据高优先级的队列优先调度原则分别取出队列内的IP报文进行发送处理。
