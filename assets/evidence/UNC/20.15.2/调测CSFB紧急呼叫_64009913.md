# 调测CSFB紧急呼叫

## [操作场景](#ZH-CN_OPI_0164009913)

本操作介绍了通过调测手段检查 UNC 是否正确配置该功能的过程。

> **说明**
> 适用于MME

## [必备事项](#ZH-CN_OPI_0164009913)

前提条件

CSFB紧急呼叫功能已经激活，具体配置请参考 [激活CSFB紧急呼叫特性](激活CSFB紧急呼叫_64009912.md) 。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0164009913)

- 调测CSFB紧急呼叫业务。
    1. 创建用户跟踪。
      参数选择如下：
          - IMSI:被跟踪用户的IMSI。
          - 其它参数：选择默认值。
    2. 用户附着成功，拨打紧急语音号码，发起紧急呼叫业务。
    3. 查看跟踪结果。
          - 预期结果：UE发送给MME的Extended Service Request消息中的Service type信元指示为紧急呼叫（mobile originating CS fallback emergency call）。MME收到eNodeB透传的Extended Service Request消息后，向eNodeB发送UE Context Modification Request消息，携带CS Fallback Indicator信元并取值为“CS Fallback High Priority”，指示高优先级回落此CS业务。如 [图1](#ZH-CN_OPI_0164009913__fig1) ， [图2](#ZH-CN_OPI_0164009913__fig2) 所示。
            **图1** Extended Service Request消息中的Service type信元指示为紧急呼叫（样例）

            <br>
            **图2** UE Context Modification Request消息，携带CS Fallback Indicator信元并取值为“CS Fallback High Priority”（样例）

          - 异常结果：Extended Service Request消息及UE Context Modification Request消息中未正确携带上述信元。
            参考 [激活CSFB紧急呼叫特性](激活CSFB紧急呼叫_64009912.md) ，查看配置是否正确。
