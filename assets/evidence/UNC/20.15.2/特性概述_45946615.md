#### [适用NF](#ZH-CN_TOPIC_0245946615)

SGSN

#### [定义](#ZH-CN_TOPIC_0245946615)

SGSN Pool用户迁移特性是在SGSN Pool组网中，当用户在Pool内的SGSN上分布不均衡时，通过手工方式将部分用户从一个SGSN迁移到另一个SGSN上，从而实现对SGSN Pool内这些SGSN的维护而不影响用户业务。

#### [客户价值](#ZH-CN_TOPIC_0245946615)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | Pool区内用户灵活调度，方便网络维护及网络扩容。 |
| 用户 | 在进行SGSN维护时，用户业务不受影响。 |

#### [应用场景](#ZH-CN_TOPIC_0245946615)

SGSN Pool内SGSN之间用户的迁移可以应用于如下的场景。

- 迁移全部用户
  升级SGSN Pool内的一个或多个SGSN时，需要先将这些SGSN内的全部用户迁移到SGSN Pool内其他SGSN中，然后再对这些SGSN进行升级，避免升级影响用户的业务。
- 迁移部分用户
  如上例，当这些SGSN升级完毕后，需要将其他SGSN中的部分用户迁移回升级完毕的SGSN。
  通过EMS，指定SGSN中迁出的用户数或用户比例（当前SGSN静态用户数的比例），当迁出用户数达到指定的用户数或比例时，迁移操作自动停止。
- 迁移指定RNC/BSC下的用户
  将SGSN Pool1中的RNC/BSC由SGSN Pool2接管时，为了避免接管过程中对该RNC/BSC下的用户业务造成影响，先将需要接管的RNC/BSC也连入SGSN Pool2中（对这些RNC/BSC，SGSN Pool1和SGSN Pool2中所有的SGSN实际上构成了一个新的SGSN Pool），然后将SGSN Pool1各SGSN中位于RNC/BSC下的用户迁移到SGSN Pool2中的各SGSN中，迁移完毕后，可以断开RNC/BSC与SGSN Pool1的连接。
  通过EMS，SGSN能够只迁移指定RNC/BSC业务区的用户。同时可以支持迁移10个RNC/BSC业务区中的用户。
- 迁移指定的用户
  为了拨测或测试目的，需要将SGSN Pool内在SGSN1中注册的用户迁移到SGSN2。
  通过EMS，SGSN能够根据IMSI方式迁移指定的用户。
- 基于MSISDN号段迁移用户
  不推荐。

#### [可获得性](#ZH-CN_TOPIC_0245946615)

**涉及NF**

| 涉及NF | 支持版本 | 功能 |
| --- | --- | --- |
| MS | 无特殊要求 | - 根据网络侧的指示时间发起周期RAU。<br>- 根据网络侧指示重新发起Attach流程。 |
| BSC | 无特殊要求 | 不将新接入的用户路由到已经处于迁移状态的SGSN。 |
| RNC | 无特殊要求 | 不将新接入的用户路由到已经处于迁移状态的SGSN。 |
| SGSN | UNC 20.3.0及后续版本 | - 根据U2020的指示发起用户迁移任务。<br>- 定期向U2020上报迁移进度。 |
| U2020 | 无特殊要求 | - 根据POOL内SGSN的负荷情况，决定启动和停止用户迁移任务。<br>- 监控用户迁移进度。 |

本特性只有获得了License许可后才能获得该特性的服务，对应的License控制项为 “82206553 LKV2SGSNMG02 SGSN Pool用户迁移”。

本特性加载License后即可使用。

#### [对系统的影响](#ZH-CN_TOPIC_0245946615)

本特性对系统无影响。

#### [应用限制](#ZH-CN_TOPIC_0245946615)

在 UNC 存在进程复位的场景下，用户迁移功能不可用。请在进程复位完成后，再启动用户迁移。

#### [原理概述](#ZH-CN_TOPIC_0245946615)

用户迁移类型主要包括以下几种。

- 迁移全部用户
- 迁移部分用户
- 迁移指定RNC/BSC下的用户
- 迁移指定的用户
- 基于MSISDN号段迁移用户
