# 过程1：请求信息纠正前的预处理

请求信息纠正前的预处理是指AMF支持通过 **[ADD PDUACTCTRL](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NF发现和选择管理/PDU会话激活控制参数/增加PDU会话激活控制参数（ADD PDUACTCTRL）_19641130.md)** 配置对于指定用户的PDU会话建立处理策略。当UE发起PDU会话建立请求时，AMF会先判断是否存在该用户的PDU会话建立控制策略配置，根据 [图1](#ZH-CN_TOPIC_0000002116280013__fig1363414895319) 中的具体过程进行相应流程处理。

- AMF本地如果不存在该用户的PDU会话建立控制策略（即存在**[ADD PDUACTCTRL](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NF发现和选择管理/PDU会话激活控制参数/增加PDU会话激活控制参数（ADD PDUACTCTRL）_19641130.md)**配置），直接执行[过程2：请求信息纠正](过程2：请求信息纠正_73363451.md)。
- 此PDU会话建立控制策略配置对建立紧急PDU会话不生效。

**图1** 请求信息纠正前的预处理

<br>

![](过程1：请求信息纠正前的预处理_16280013.assets/zh-cn_image_0000002116374321_2.png)

1. 检查UE发起的PDU会话建立请求消息中是否携带请求DNN。
    - 是 => 执行[步骤2](#ZH-CN_TOPIC_0000002116280013__step2)。
    - 否 => 执行[步骤3](#ZH-CN_TOPIC_0000002116280013__step3)。
2. 检查是否存在该UE的ALL_DNN或SPECIAL_DNN或NOT_NULL_DNN的PDU会话建立控制配置。通过**[LST PDUACTCTRL](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NF发现和选择管理/PDU会话激活控制参数/查询PDU会话激活控制参数（LST PDUACTCTRL）_19483038.md)**令查询PDU会话建立控制配置记录。
    - 是 => 执行[步骤4](#ZH-CN_TOPIC_0000002116280013__li7290165415517)。
    - 否 => 执行[过程2：请求信息纠正](过程2：请求信息纠正_73363451.md)。
3. 检查是否存在该UE的ALL_DNN或NULL_DNN的PDU会话建立控制配置。通过**[LST PDUACTCTRL](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NF发现和选择管理/PDU会话激活控制参数/查询PDU会话激活控制参数（LST PDUACTCTRL）_19483038.md)**命令查询PDU会话建立控制配置记录。
    - 是 => 执行[步骤4](#ZH-CN_TOPIC_0000002116280013__li7290165415517)。
    - 否 => 执行[过程2：请求信息纠正](过程2：请求信息纠正_73363451.md)。
4. 检查存在的PDU会话建立控制配置记录中，“处理策略”是否为“直接拒绝”。
    - 是 => 拒绝PDU会话建立请求。
    - 否 => 执行[步骤5](#ZH-CN_TOPIC_0000002116280013__li4143161922918)。
5. 检查签约数据是否匹配。
    - 是 => 使用请求DNN或者Default DNN建立PDU会话。
    - 否 => 执行[步骤6](#ZH-CN_TOPIC_0000002116280013__li14569153017610)。
6. 检查是否执行纠正策略。
    - 是 => 执行纠正策略，参考[过程2：请求信息纠正](过程2：请求信息纠正_73363451.md)中配置了DNN纠正策略的场景。
    - 否 =>拒绝PDU会话建立请求。
