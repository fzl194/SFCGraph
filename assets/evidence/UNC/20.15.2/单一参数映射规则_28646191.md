# 单一参数映射规则

虚拟APN支持使用单一参数规则映射真实APN，如下：

- 网络切片（NS）
  网络切片只适用于PGW-C、SMF，当5G用户接入或者支持4/5互操作的用户在4G接入时，基于切片映射到真实APN的处理，通过映射后的APN区分不同切片的业务。
  映射方式为NS时，优先级最高，在所有虚拟APN映射规则之前应用。
- 缺省APN
  当前其他所有条件都匹配失败时，使用Default APN。如果缺省APN没有配置，则用户激活失败。
- 用户接入属性
  虚拟APN支持根据用户接入属性，包括IMSI、MSISDN、PLMN、IMEI、RAT、PCO、CC(Charging Characteristics)和Selection Mode参数，以及用户漫游类型映射出真实APN。当不同的用户携带相同的虚拟APN接入时，PGW-C根据用户所携带的属性映射到不同的真实APN进行不同的业务。各个参数的映射方式和应用场景如 [表1](#ZH-CN_CONCEPT_0228646191__tab_shuxing) 所示。

  *表1 根据用户接入属性匹配真实APN*

  | 用户接入属性 | 映射含义 | 应用场景 |
  | --- | --- | --- |
  | 根据IMSI匹配真实APN | 根据用户所在的IMSI号段匹配出真实APN，从而接入不同的目的网络。 | - 运营商通过IMSI号段进行业务规划。<br>- UNC通过IMSI号段区分移动虚拟网络运营商（Mobile Virtual Network Operator）。 |
  | 根据MSISDN匹配真实APN | 根据用户所在的MSISDN号段匹配出真实APN，从而接入不同的目的网络。 | 运营商通过MSISDN号段进行业务规划。 |
  | 根据PLMN匹配真实APN | 根据用户当前接入的网络的PLMN（SGSN/SGW-C）匹配出真实的APN，从而接入不同的目的网络。 | 运营商通过漫游用户的服务网络的PLMN号段进行业务规划。 |
  | 根据IMEI匹配真实APN | 根据用户所在的IMEI号段匹配出真实APN，从而接入不同的目的网络。 | 运营商通过IMEI号段分配不同的IP地址以满足特定的组网需求。 |
  | 根据RAT匹配真实APN | - 根据用户接入时携带上来的RAT（无线接入类型）属性和系统配置的RAT映射关系、RAT优先级进行判断，分析出用户是UTRAN、GERAN或WLAN等方式接入，解析出用户要接入的真实APN，从而接入不同的目的网络。<br>- 如果用户没有携带RAT，则完全根据UNC系统配置的RAT映射关系、RAT优先级进行判断。 | 运营商通过RAT属性进行业务规划。 |
  | 根据CC匹配真实APN | 根据用户接入时携带的CC(Charging Charactersitics)，匹配出用户要接入的真实APN，从而接入不同的目的网络。 | 运营商通过用户的CC号段进行业务规划。 |
  | 根据PCO匹配真实APN | 通过解析用户登录时携带的PCO中的用户名中域名部分，匹配出真实APN，从而接入用户希望访问的目的网络。<br>例如，用户使用虚拟APN“company”和一个域名分隔符形式的用户名“login@company1”连接网络，此用户名包含在PCO信元中。<br>- 如果未使能增强的PCO方式的虚拟APN功能（即配置Domain和APN参数），则从PCO信元中解析出域名“company1”作为用户的真实APN，UNC根据用户的真实APN处理其激活请求。<br>- 如果使能了增强的PCO方式的虚拟APN功能，仅配置有APN参数时，PCO信元中解析出域名“company1”与APN参数匹配上时，使用APN参数作为用户的真实APN。如果同时配置有Domain和APN参数时，PCO信元中解析出域名“company1”与Domain参数匹配上时，使用APN参数作为用户的真实APN。如果都没有匹配成功，则使用虚拟APN“company”下配置的default APN激活。 | 此方式适用于企业私有网络的用户接入的应用场景。运营商在<br>UNC<br>上为不同的企业设置不同的真实APN，不同的企业用户使用相同的接入APN，但提供不同的用户名，则<br>UNC<br>就可以根据用户名将用户接入相应的企业，应用相应企业的服务策略。 |
  | 根据用户的本地、漫游、拜访属性映射真实APN | 用户激活时，首先根据用户请求消息判断用户的本地、漫游、拜访属性信息，然后匹配映射规则进行虚拟APN映射，获得用户要接入的真实APN，从而接入不同的目的网络。<br>说明：如果映射得到的真实APN下又配置了基于SGSN/MME/SGW-C IP映射PLMN，可能导致上下文保存的用户漫游属性信息和根据请求消息获取的用户漫游属性信息存在差异。 | 运营商通过用户的本地、漫游、拜访属性进行业务规划，实施不同的计费、业务策略。 |
  | 根据Selection Mode匹配真实APN | 根据用户接入时携带上来的Selection Mode(选择模式)进行判断，解析出用户要接入的真实APN，从而接入不同的目的网络。 | 运营商通过Selection Mode进行业务规划。 |
  以根据IMSI匹配真实APN方式为例，介绍其业务流程，如 [图1](#ZH-CN_CONCEPT_0228646191__fig_imei) 所示，IMSI号码4600010123456和4600021123456的分别对应的UE1和UE2携带虚拟APN“example.com”向 UNC 发起激活请求， UNC 根据本机配置分别解析出UE1对应的真实APN是City A，UE2对应的真实APN是City B，然后 UNC 根据用户的真实APN处理其激活请求。

- 用户位置（LAC/TAC/RAC/SGSN IP/SGW IP）
  为了满足运营商根据用户所在位置区域进行差异化管理的需求，华为提出了支持基于用户位置信息的业务管理方案。 UNC 基于用户位置进行虚拟APN的映射，将不同位置区域映射到不同的真实APN，真实APN下配置特定的IP地址池，以此建立用户IP地址与用户位置区域的对应关系，以便其它设备根据用户的IP地址做相应的策略控制。
  UNC 将激活请求消息中的请求APN作为虚拟APN，基于LAC/TAC/RAC或基于SGSN IP/SGW-C IP映射真实APN。通过配置，将规划的LAC/RAC/TAC绑定在LAC group/RAC group/TAC group下，将规划的SGSN IP/SGW-C IP绑定在Serving Node Group下。再根据LAC group、RAC group、TAC group和Serving Node Group匹配虚拟APN，映射真实APN。
  如 [图2](#ZH-CN_CONCEPT_0228646191__fig_weizhi) 所示，以基于LAC映射真实APN为例进行介绍。UE1的激活请求消息中携带的LAC为0x1111， UNC 查找本地LAC group的配置，获得0x1111所属的LAC group为A。进而查找本地的虚拟APN配置，获得LAC group A对应的真实APN是City A。

- 备份方式（LOAD_BALANCE/REDUNDANCY）
  为了满足使用一个虚拟APN的大量用户接入， UNC 对所有用户采用备份方式，设置不同的真实APN来承担使用一个虚拟APN的大量用户接入PDN。 UNC 按照真实APN进行鉴权，生成话单。 UNC 采用的备份方式主要包括主备方式和负荷分担方式。
    - 基于主备方式（REDUNDANCY）的虚拟APN映射：如[图3](#ZH-CN_CONCEPT_0228646191__fig1490543345118)所示，所有用户使用虚拟APN“Net”接入网络，UNC上配置对应此虚拟APN的真实APN包括“Net 1”（主），“Net 2”（备）。则UNC首先将用户都映射到APN“Net1”，当检测到Net 1下的AAA服务器down之后，则后续新上线用户就直接映射到APN“Net 2”，然后UNC根据用户所属的真实APN处理其激活请求。
      **图3** 基于主备方式的虚拟APN映射

      <br>

      ![](单一参数映射规则_28646191.assets/zh-cn_image_0234838489_2.png "点击放大")
    - 基于负荷分担方式（LOAD_BALANCE）的虚拟APN映射：如[图4](#ZH-CN_CONCEPT_0228646191__fig13905183320517)所示，所有用户使用虚拟APN“Net”接入网络，UNC上配置对应此虚拟APN的真实APN包括“Net 1”，“Net 2”。则UNC把第一个用户映射到APN“Net 1”，把第二个用户映射到APN“Net 2”，把第三个用户映射到APN“Net 1”，把第四个用户映射到APN“Net 2”，依次类推，然后UNC根据用户所属的真实APN处理其激活请求。
