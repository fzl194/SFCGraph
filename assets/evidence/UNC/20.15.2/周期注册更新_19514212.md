1. 消息交互：UE发送[Registration request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Registration request_76783894.md)给(R)AN。
  触发条件：UE由于周期性注册更新定时器超时而发起注册流程。
  关键信元：
  | 5GS registration type | 指示5GS注册类型，周期注册时5GS registration type为periodic registration updating。 |
  | --- | --- |
  | 5GS mobile identity | 携带用户标识5G-GUTI或SUCI。优先选择携带5G-GUTI作为UE身份标识，若无法获取5G-GUTI，UE会携带自身的SUCI作为UE身份标识。<br><5G-GUTI> := <GUAMI（Globally Unique AMF Identifier）> <5G-TMSI><br><GUAMI> := <MCC> <MNC> <AMF Region ID> <AMF Set ID> <AMF Pointer><br>5G-TMSI：（Temporary Mobile Subscriber Identity）临时用户标识，网络侧AMF分配的临时用户标识。<br>SUPI：（Subscription Permanent Identifier）用户永久标识，5G网络中标识一个用户。<br>SUCI：（Subscription Concealed Identifier）用户隐藏标识，UE通过归属网络的公共密钥加密SUPI生成SUCI。 |
  | Requested NSSAI | 标识请求的网络切片。 |
  | 5G-S-TMSI | 5G-S-TMSI是5G-GUTI的部分字段，使用5G-S-TMSI可以缩短空口消息长度，提升空口效率。<br><5G-S-TMSI> := <AMF Set ID> <AMF Pointer><5G-TMSI> |
  | Uplink data status | 指示有上行数据待处理的保留的PDU会话。 |
  | PDU session status | 指示每个PDU会话的状态。 |
  相关机制：UE在注册请求消息中携带的用户标识，按照以下优先级递减的顺序排列：

  - 由之前成功注册的EPS网络分配的EPS (GUTI)转换的5G-GUTI。
    - UE正在尝试注册PLMN分配的可用的本地5G-GUTI。
    - 由UE正在尝试注册的PLMN的对等PLMN分配给UE的可用的本地5G-GUTI。
    - 任何其他PLMN分配的可用的本地5G-GUTI。
    - 如果无法获取5G-GUTI，则UE在注册请求中携带SUCI，比如首次开机注册。
  相关特性：注册是否携带网络切片以及Requested NSSAI的选择机制参见UNC特征：WSFD-110001 支持基于NSSAI选择切片。
  相关配置：周期性注册定时器T3512，通过UNC命令：SET NGMMPARA配置。
2. 内部流程：AMF选择流程。
  触发条件：(R)AN收到注册请求后进行AMF选择流程。
  相关机制：

  - 如果AN消息中未携带5G-S-TMSI，或者5G-S-TMSI不能指示一个合法的AMF时，(R)AN根据RAT（radio access technology）和请求的网络切片标识（Requested NSSAI）选择AMF。
    - 如果(R)AN不能选择合适的AMF，则将注册请求转发给(R)AN中已配置的default AMF进行AMF选择。
3. 消息交互：(R)AN发送[Registration request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Registration request_76783894.md)给新侧AMF。
  关键信元：
  | PLMN Identity | PLMN标识。<PLMN Identity>=<MCC> <MNC><br>MCC：（Mobile Country Code），移动国家码，唯一标识移动用户所属国家的三位数字代码。<br>MNC：（Mobile Network Code），移动网络码，用来在一个国家内唯一标识一个移动网络。 |
  | --- | --- |
  | TAC | Tracking Area Code，标识一个跟踪区域码，由运营商自行分配，用于网络侧跟踪UE的位置信息。 |
  相关机制：如果(R)AN是NG-RAN，N2参数包括PLMN标识、位置信息和与UE所在小区相关的小区标识、UE上下文等。
  相关特性：

  - AMF收到UE信令后可能会触发异常信令管控，参见UNC特性：WSFD-206008 5G UE信令控制。
    - AMF收到注册消息可能会触发系统过载控制，参见UNC特性：WSFD-010801 系统过载控制功能。
4. 内部流程：AUSF选择流程。
  触发条件：周期性注册中选择AUSF进行鉴权，紧急注册中AMF可以跳过AUSF选择流程以及鉴权流程进行注册。
  相关机制：AMF选择AUSF的机制参见UNC章节：注册管理流程的网元选择。
  相关配置：AMF通过配置决定注册流程是否开启鉴权功能，相关配置参见UNC命令：ADD NGUSRSECPARA **。**
5. 子流程：[5G AKA鉴权](../../../../../安全管理功能/WSFD-010301 鉴权功能/鉴权功能（5G）/鉴权流程（适用于5G）/5G AKA鉴权_80005829.md#ZH-CN_TOPIC_0000001480005829)和[NAS信令加密和完整性保护流程](../../../../../安全管理功能/WSFD-010303 NAS信令加密与完整性保护/NAS信令加密与完整性保护（5G）/NAS信令加密和完整性保护流程_80010917.md#ZH-CN_TOPIC_0000001480010917)。
  触发条件：周期性注册中选择AUSF进行鉴权，紧急注册中AMF可以跳过AUSF选择流程以及鉴权流程进行注册。
  相关特性：

  - 鉴权功能参见UNC特性：WSFD-010301 鉴权功能**。**
    - NAS加密参见UNC特性：WSFD-010303 NAS信令加密与完整性保护（5G）。
6. 消息交互：新侧AMF发送[Registration accept](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Registration accept_76785445.md)给UE。
  关键信元：
  | 5GS registration result | 用于指示注册结果。 |
  | --- | --- |
  | 5G-GUTI | AMF新分配的5G-GUTI。 |
  | Service area list | Service area list信元用于表示将允许跟踪区域列表或禁止跟踪区域列表从网络传输给UE。 |
  | TAI list | TAI list信元用于将网络中的跟踪区域列表传递给UE。 |
  | T3512-value | 周期性注册定时器T3512，通过UNC命令：SET NGMMPARA配置。 |
  相关机制： AMF根据AM策略关联更新本地的接入移动策略控制信息，并通知UE注册请求被接受。
7. 消息交互：UE发送[Registration complete](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Registration complete_76784509.md)给新侧AMF。
  触发条件：注册完成UE发送指示消息。
