1. 进入 “MML命令行-UNC” 窗口。
2. 配置支持基于HSS的P-CSCF故障恢复特性。

  ![](激活基于HSS的P-CSCF故障恢复（适用于承载更新）_21873364.assets/notice_3.0-zh-cn_2.png)

      [**ADD PGWCHARACT**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/P-GW属性/增加P-GW特性对接配置（ADD PGWCHARACT）_26305748.md)

## [任务示例](#ZH-CN_OPI_0000002521873364)

任务描述

激活基于HSS的P-CSCF故障恢复特性，假设仅本网HSS支持本特性，其他用户的归属HSS都不支持本特性。

脚本

// 进入 “MML命令行-UNC” 窗口。

//配置支持基于HSS的P-CSCF故障恢复特性。（实际部署时，需要根据HSS是否支持本特性，组合如下命令进行配置）

- 所有HSS都支持此特性：
  ```
  SET DMCMPT: SF2=FLSTID2_BIT16-1;
  ```
- 本网所有的HSS支持此特性，外网所有HSS不支持此特性：
  ```
  ADD DMCMPTBYIMSI: SUBRANGE=HOME_USER, NOID=0, SF2=FLSTID2_BIT16-1;
  ```
- 仅46123和46321为IMSI前缀的用户注册的HSS支持此特性：
  ```
  ADD DMCMPTBYIMSI: SUBRANGE=IMSI_PREFIX, IMSIPRE="46123", SF2=FLSTID2_BIT16-1;
  ADD DMCMPTBYIMSI: SUBRANGE=IMSI_PREFIX, IMSIPRE="46321", SF2=FLSTID2_BIT16-1;
  ```

//配置支持基于HSS的P-CSCF故障恢复机制的可选扩展PCO-based optional extension功能。

- 在S-GW上配置S-GW支持基于HSS的P-CSCF故障恢复功能。
  ```
  SET GLOBALIMS: HSSPCSCF=ENABLE;
  ```
- 在P-GW上配置P-GW支持基于HSS的P-CSCF故障恢复功能。
  ```
  SET GLOBALIMS: HSSPCSCF=ENABLE, HSSPCSCFMODE=PCSCFADDRUPDATE;
  ```
- 在MME上配置MME支持PCO-based optional extension功能。
  ```
  ADD PCSCFRESTOPLCY: SUBRANGE=ALL_USER, PCOOPEX=ON;
  ```
- 在MME上配置S-GW支持P-CSCF Restoration Indication的策略。
  ```
  ADD SGWCHARACT: RANGE=SPECIFIED, IPT=IPV4, IPV4="192.168.6.7", IPV4MSK="255.255.255.0", PCSCFRESTOIND=YES;
  ```
- 在MME上配置P-GW支持P-CSCF Restoration Indication的策略。
  ```
  ADD PGWCHARACT: RANGE=SPECIFIC, IPT=IPV4, IPV4="192.168.6.6", IPV4MASK="255.255.255.0", PCSCFRESTOIND=YES;
  ```

//打开特性License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2FRPH02", SWITCH=ENABLE;
```

//清除HSS信息。

```
CLR HSSINFO: HSSHTNAME="hss.epc.mnc456.mcc123.3gppnetwork.org";
```
