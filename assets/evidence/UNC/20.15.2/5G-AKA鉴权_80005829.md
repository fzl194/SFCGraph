#### [业务场景](#ZH-CN_TOPIC_0000001480005829)

UE首次接入网络、数据加密和完整性检查失败等场景，AMF调用AUSF的鉴权服务进行初始化鉴权。

1. 内部流程：AUSF选择流程。
  触发条件：初始注册中选择AUSF进行鉴权。
  相关机制：AMF选择老侧AMF的机制参见UNC章节注册管理流程的网元选择。
2. 消息交互：AMF发送[Nausf_UEAuthentication_CreateUeAuthContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nausf/N12/Nausf_UEAuthentication/Nausf_UEAuthentication_CreateUeAuthContext Request_56087661.md)给AUSF。
  触发条件：AMF接收到的Registration request消息发现用户为初始注册，则AMF调用AUSF的鉴权服务进行初始化鉴权。
  关键信元：
  | supiOrSuci | 表示UE的SUPI或SUCI。 |
  | --- | --- |
  | servingNetworkName | 表示服务网络名称。 |
  相关机制：当AMF判断UE已有合法5G-GUTI时，消息中包含SUPI，否则消息中包含SUCI。
3. 消息交互：AUSF发送Nudm_UEAuthentication_Get Request给UDM。
  相关机制：AUSF接收到Nausf_UEAuthentication_CreateUeAuthContext Request消息后，通过servingNetworkName检查AMF是否有资格使用该服务网络。如果授权使用，则AUSF向UDM发送Nudm_UEAuthentication_Get Request消息，消息中包含SUCI或SUPI、服务网络名称。
4. 消息交互：UDM发送Nudm_UEAuthentication_Get Response给AUSF。
  相关机制：如果消息中包含的是SUCI，那么UDM在处理请求前先将SUCI解密为SUPI。UDM根据SUPI获取用户真身份，根据用户签约信息（ **MOD AMDATA** ）选择鉴权方式。之后UDM生成鉴权向量5G HE AV（5G Home Environment Authentication Vector），包含RAND、AUTN、XRES*和K <sub>AUSF</sub> 并发送给AUSF。如果Nudm_UEAuthentication_Get Request包含SUCI时，则Nudm_UEAuthentication_Get Response消息中包含SUPI。
5. 消息交互：AUSF发送[Nausf_UEAuthentication_CreateUeAuthContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nausf/N12/Nausf_UEAuthentication/Nausf_UEAuthentication_CreateUeAuthContext Response_56287373.md)给AMF。
  触发条件：AUSF获取鉴权方式和鉴权向量后发送Nausf_UEAuthentication_CreateUeAuthContext Response给AMF。
  关键信元：
  | authType | 表示该用户所使用的鉴权方式。比如："5G_AKA"、"EAP_AKA_PRIME"、"EAP_TLS" |
  | --- | --- |
  | _links | - 如果选择5G-AKA鉴权方式，该信元中携带名称为“5g-aka”的成员和用于执行确认的URI。<br>- 如果选择EAP鉴权方式，该信元携带名称为“eap-session”的成员和用于执行EAP会话的URI。 |
  | 5gAuthData | 表示5G-AKA鉴权或EAP鉴权相关信息。 |
  相关机制：AUSF接收到Nausf_UEAuthentication_Authenticate Request消息后，通过servingNetworkName检查AMF是否有资格使用该服务网络。如果未授权使用，则AUSF使用原因值SERVING_NETWORK_NOT_AUTHORIZED返回失败。AUSF从UDM收到5G HE AV后，存储XRES*、SUPI或SUCI、K <sub>AUSF</sub> ，并通过XRES*、K <sub>AUSF</sub> 计算出HXRES*、K <sub>SEAF</sub> ，生成鉴权向量5G AV，包含RAND、AUTN、HXRES*和K <sub>SEAF</sub> 。AUSF移除5G AV中的K <sub>SEAF</sub> 生成5G SE AV（RAND、AUTN、HXRES*），并通过Nausf_UEAuthentication_CreateUeAuthContext Response消息将5G SE AV发送给AMF。
6. 消息交互：AMF发送[Authentication request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Authentication request_76784595.md)给UE。
  触发条件：AMF获取鉴权向量后向UE发起Authentication request消息，发起鉴权。
  关键信元：
  | ngKSI | ngKSI为网络侧分配的NAS密钥集标识，用于识别K<br>AMF<br>和鉴权成功后产生的安全上下文。 |
  | --- | --- |
  | ABBA | ABBA用于实现安全特性的降级保护。 |
  | Authentication parameter RAND | RAND为网络提供给UE的不可预知的随机数，UE使用RAND计算鉴权响应参数及安全保密参数。 |
  | Authentication parameter AUTN | AUTN为鉴权令牌，是网络提供给UE的参数，用于UE对网络进行鉴权。 |
  相关机制：如果UE鉴权失败，则UE向AMF返回Authentication failure并明确原因。
7. 消息交互：UE发送[Authentication response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Authentication response_76783899.md)给AMF。
  触发条件：UE收到鉴权请求后根据收到的AUTN鉴权网络，并根据鉴权结果向AMF发送响应消息。
  关键信元：
  | Authentication response parameter | 鉴权响应参数，UE将鉴权响应参数RES*发送给AMF。 |
  | --- | --- |
  相关机制：如果UE鉴权成功则UE计算鉴权响应RES*、KAUSF和KSEAF并发送Authentication response给AMF。AMF通过RES*计算出HRES*，比较HRES*和HXRES*以判断服务网络鉴权是否成功。如果HRES*和HXRES*不一致，则AMF认为服务网络鉴权失败。当UE在初始NAS消息中使用了SUCI，则AMF向UE发送Authentication reject消息拒绝鉴权。当UE在初始NAS消息中使用了5G GUTI，则AMF向UE重新获取SUCI并再次尝试鉴权。
8. 消息交互：AMF发送[Nausf_UEAuthentication_UeAuthConfirm Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nausf/N12/Nausf_UEAuthentication/Nausf_UEAuthentication_UeAuthConfirm Request_06927746.md)给AUSF。
  触发条件：AMF判断服务网络鉴权成功后将从UE收到的鉴权响应参数RES*发送给AUSF。
  关键信元：
  | resStar | 表示UE向AMF提供的RES*。 |
  | --- | --- |
  相关机制：AMF通过RES*计算出HRES*，比较HRES*和HXRES*以判断服务网络鉴权是否成功。如果HRES*和HXRES*一致，则AMF认为服务网络鉴权成功，AMF将从UE收到的鉴权响应参数RES*发送给AUSF。
9. 消息交互：AUSF发送[Nausf_UEAuthentication_UeAuthConfirm Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nausf/N12/Nausf_UEAuthentication/Nausf_UEAuthentication_UeAuthConfirm Response_56207425.md)给AMF。
  触发条件：AUSF判断归属网络鉴权成功后向AMF发送归属网络鉴权结果。
  关键信元：
  | authResult | 表示鉴权结果，比如AUTHENTICATION_SUCCESS。 |
  | --- | --- |
  | supi | 如果鉴权成功且AMF提供了SUCI，则该信元携带UE的SUPI。 |
  | kseaf | 如果鉴权成功，则携带Kseaf。 |
  相关机制：AUSF收到Nausf_UEAuthentication_UeAuthConfirm Request消息后，比较收到的RES*和存储的XRES*是否一致。
    - 如果一致，则认为归属网络鉴权成功，则AUSF发送消息中会同时包含K<sub>SEAF</sub>，后续AMF推衍ABBA和SUPI，并根据K<sub>SEAF</sub>推衍K<sub>AMF</sub>。
    - 如果不一致，则认为归属网络鉴权失败，则后续AMF向UE发送Authentication reject消息拒绝鉴权，或者重新发起身份识别流程后尝试再次鉴权。
10. 消息交互：AUSF发送Nudm_UEAuthentication_ResultConfirmation Request给UDM。
  相关机制：AUSF通过Nudm_UEAuthentication_ResultConfirmation Request消息通知UDM鉴权结果，包含SUPI、鉴权时间、认证方法和服务网络名称。
11. 消息交互：UDM发送Nudm_UEAuthentication_ResultConfirmation Response给AUSF。
  相关机制：UDM存储鉴权结果，并通过Nudm_UEAuthentication_ResultConfirmation Response响应AUSF。
