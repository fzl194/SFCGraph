## [操作场景](#ZH-CN_OPI_0000001304043206)

本操作指导介绍在运行网络中调测基于IMEI的NR差异化服务的操作过程。

## [必备事项](#ZH-CN_OPI_0000001304043206)

前提条件

- 请仔细阅读[WSFD-011311 基于IMEI的NR差异化服务特性概述](WSFD-011311 基于IMEI的NR差异化服务特性概述_04363010.md)。
- 完成[激活基于IMEI的NR差异化服务](激活基于IMEI的NR差异化服务_04203170.md)。

数据

该操作无需准备数据。

工具

- 测试终端。
- OM Portal跟踪。

## [操作步骤](#ZH-CN_OPI_0000001304043206)

1. 进入 “MML命令行-UNC” 窗口。
2. 检查AMF获取终端IMEISV的配置信息。
  **[LST NGPEIPLCY](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/业务安全管理/PEI策略管理/查询5G PEI策略（LST NGPEIPLCY）_09653007.md)** : SUBRANGE=ALL_USER, GETPEIPLCY=GETIMEISV;
  预期结果： “用户范围” 结果为 “所有用户” ， “PEI获取策略” 为 “获取IMEISV” ，与已规划配置一致，执行 [步骤 3](#ZH-CN_OPI_0000001304043206__step1641861191713) 。
  异常结果：查询结果与已规划配置不一致，参考 [激活基于IMEI的NR差异化服务](激活基于IMEI的NR差异化服务_04203170.md) 重新配置。
3. 检查AMF给gNodeB下发的Initial Context Setup Request和Handover Request消息中携带Masked IMEISV信元的配置信息。
  **[LST NGAPCMPT](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N2接口管理/NGAP兼容性参数管理/查询NGAP兼容性参数（LST NGAPCMPT）_09653275.md)** : MSKIMEISV=YES;
  预期结果： “是否携带Masked IMEISV信元” 结果为 “是” ，与已规划配置一致，执行 [步骤 4](#ZH-CN_OPI_0000001304043206__step20255174220910) 。
  异常结果：查询结果与已规划配置不一致，参考 [激活基于IMEI的NR差异化服务](激活基于IMEI的NR差异化服务_04203170.md) 重新配置。
4. 进行业务测试。
    a. 在UNC上创建用户跟踪任务和N2接口跟踪任务。
    b. 用户开机注册到5G网络。
      预期结果：用户注册成功。
    c. 观察用户消息跟踪和N2接口消息跟踪。
      预期结果：用户消息跟踪和N2接口跟踪中都可以观察到AMF向gNodeB发送的Initial Context Setup Request和Handover Request消息中携带“Masked IMEISV”信元，调测结束。
      异常结果：跟踪到的Initial Context Setup Request和Handover Request消息中未携带“Masked IMEISV”信元，执行 [步骤 5](#ZH-CN_OPI_0000001304043206__step16856162643819) 。
5. 收集信息并寻求技术支持。
    a. 在 “MML命令行-UNC” 窗口上执行命令 **[EXP MML](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md)** ，将当前配置数据导出为MML脚本文件并保存。
    b. 收集并保存上述所有查询信息。
    c. 收集归纳所有信息并联系华为技术支持解决。
