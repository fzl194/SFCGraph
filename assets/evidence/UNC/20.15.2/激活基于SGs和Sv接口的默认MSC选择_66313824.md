## [操作场景](#ZH-CN_OPI_0166313824)

本操作指导介绍在运行网络中激活基于SGs和Sv接口的默认MSC选择特性的操作过程。

在基于Sv接口部署SRVCC的场景下，当某些LAC没有在 UNC 或DNS Server配置对应的FQDN时，会导致MME寻址目标MSC失败，最终导致SRVCC流程失败。此时， UNC 将使用LAC=0xFFFF组装FQDN再次尝试DNS域名解析，寻址默认的MSC，提高SRVCC业务的成功率。

在基于SGs接口部署CSFB的场景下，当某些TAI没有配置对应的LAI时，会导致CSFB流程失败。此时， UNC 将使用TAI=FFFFFFFFFF映射至特定的LAI，并将该LAI指向默认的MSC，提高CSFB业务成功率。

> **说明**
> 适用于MME。

## [操作步骤](#ZH-CN_OPI_0166313824)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. 配置基于Sv接口选择默认MSC。
    a. 配置默认MSC地址。
      [**ADD IPV4DNSH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS Hostfile管理/增加IPV4 DNS Hostfile记录(ADD IPV4DNSH)_26145884.md)
    b. 配置LAC=0xFFFF对应的MSC。
      [**ADD DNSN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS NAPTR管理/增加DNS NAPTR记录(ADD DNSN)_72225569.md)
      > **说明**
      > FQDN中需要配置LAC=0xFFFF。
4. 配置基于SGs接口选择默认MSC。
    a. 配置缺省LAI记录。
      [**ADD TAILAI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/电路域联合业务/TAI与LAI对应关系/增加TAI与LAI对应关系(ADD TAILAI)_72345017.md)
      > **说明**
      > BGNLAI需要配置为FFFFFFFFFF。
    b. 配置默认LAI对应的MSC。
      > **说明**
      > 如果LAI已经配置对应MSC，跳过本步骤。
      [**ADD LAIVLR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/电路域联合业务/LAI与VLR号对应关系/增加LAI与VLR号对应关系(ADD LAIVLR)_72345015.md)

## [任务示例](#ZH-CN_OPI_0166313824)

任务描述

分别激活基于SGs和Sv接口的默认MSC选择。

脚本

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2DMSC01", SWITCH=ENABLE;
```

//配置基于Sv接口选择默认MSC。

//配置默认MSC地址。

```
ADD IPV4DNSH: HSINDEX=120, HOSTNAME="MSC1", ADDRSECTION=SECTION1, IPV4ADDR1="10.1.1.1";
```

//配置LAC=0xFFFF对应的MSC。

```
ADD DNSN: FQDN="LACFFFF.LAC.EPC.MNC001.MCC308.3GPPNETWORK.ORG", HSINDEX=120, ENTITY=MSC, INTYPE=Sv;
```

//配置基于SGs接口选择默认MSC。

//配置缺省LAI记录。

```
ADD TAILAI: BGNTAI="FFFFFFFFFF", LAI="308015101";
```

//配置默认LAI对应的MSC。

```
ADD LAIVLR: BGNLAI="308015101", ENDLAI="308015101", VLRNO="86139013201", DFLVLR=YES;
```
