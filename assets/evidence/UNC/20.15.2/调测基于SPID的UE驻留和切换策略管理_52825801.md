## [操作场景](#ZH-CN_OPI_0000001152825801)

调测基于SPID的UE驻留和切换策略管理。

## [必备事项](#ZH-CN_OPI_0000001152825801)

前提条件

- 请仔细阅读[WSFD-106207 基于SPID的UE驻留和切换策略管理（适用于AMF）](../WSFD-106207 基于SPID的UE驻留和切换策略管理（适用于AMF）_52057139.md)。
- 已完成[激活基于SPID的UE驻留和切换策略管理](激活基于SPID的UE驻留和切换策略管理_52313549.md)。

数据

无。

工具

OM Portal跟踪。

## [操作步骤](#ZH-CN_OPI_0000001152825801)

- 在相关网元均完成本特性的配置后，可以采用以下步骤检查特性工作是否正常：
    1. 进入 “MML命令行-UNC” 窗口。
    2. UDM有签约数据，或者本地配置。以本地配置为例：如配置了RFSP ID为1，策略为通过4G进行附着，不通过5G附着。NG RAN侧按照RFSP为1对应的频点优先级进行驻留控制。
      [**LST NGMMSUBDATA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/签约数据管理/查询用户移动性管理参数（LST NGMMSUBDATA）_09651400.md) :;
      ```
      %%LST NGMMSUBDATA:;%%
      RETCODE = 0  执行成功

      结果如下
      --------
                  用户范围  =  指定IMSI前缀
                  IMSI前缀  =  123031234567890
                   RAT限制  =  演进型通用陆地无线接入
            核心网类型限制  =  NULL
                  RFSP索引  =  1
      网络切片包含模式策略  =  网络切片包含模式C
                  寻呼DRX   =  0
      (结果个数 = 1)

      --- END
      ```
    3. 无线配置了SPID和驻留优先级的列表关系为3G优先。
    4. 用户在MME做了一次附着(SPID传递给RAN侧，RAN给了UE)。
      在 UNC 操作维护台上创建用户跟踪。用户成功附着后，可以在Initial Context Setup Request消息中携带 “SubscriberProfileIDforRFP” 信元值为1。

      **图1** Initial Context Setup Request(Attach Accept)用户跟踪

      <br>

      ![](调测基于SPID的UE驻留和切换策略管理_52825801.assets/zh-cn_image_0000001106385876_2.png)
    5. 具有45G接入能力的UE再次附着，直接通过4G进行附着，不会尝试附着到5G。
