#### [适用NF](#ZH-CN_TOPIC_0310243674)

GGSN、 PGW-C、SMF

#### [定义](#ZH-CN_TOPIC_0310243674)

授权修改CoA（Change-of-Authorization）是由AAA Server主动下发修改授权参数的消息，用于修改用户带宽使用策略等。

GGSN/PGW-C/SMF支持通过AAA Server下发的修改授权参数的消息，实现对用户业务流的限制，从而保证在业务高峰期网络负载均衡。

#### [客户价值](#ZH-CN_TOPIC_0310243674)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 实时变更业务控制策略和带宽，保证在业务高峰期网络负载均衡。 |
| 用户 | 用户不感知该特性。 |

#### [应用场景](#ZH-CN_TOPIC_0310243674)

CoA特性适用以下场景：

网络中同时部署PCRF/PCF和AAA Server，通过AAA Server下发策略规则后GGSN/PGW-C/SMF转发给PCRF/PCF，然后由PCRF/PCF决定最终策略规则下给用户。

网络中同时部署PCRF/PCF和AAA Server，但是用户上下文的PCC功能未能通过AAA鉴权响应消息指示功能关闭，此时GGSN/PGW-C/SMF将AAA下发的策略作为缺省策略信息携带给PCRF/PCF，作为PCRF/PCF决策最终策略的判断依据，GGSN/PGW-C/SMF最终执行PCRF/PCF的响应消息中携带的策略进行计费策略控制。此种方式为 **非RADIUS PCC** ，组网方式如 [图1](#ZH-CN_TOPIC_0310243674__fig72871120348) 所示。

#### [可获得性](#ZH-CN_TOPIC_0310243674)

**涉及网元**

| 涉及网元 | 支持版本 | 功能说明 |
| --- | --- | --- |
| GGSN/PGW-C/SMF | UNC 21.2.0及后续版本 | 接收AAA Server发送的CoA消息，对用户的授权参数进行控制，必要时通知其他网元用户授权参数的变更。 |
| AAA Server | 无特殊要求 | 用户的授权参数变更的发起方，负责保存用户的授权参数以及实时变更用户的授权参数。 |

**License支持**

本特性只有获得了License许可后才能获得该特性的服务，对应的License控制项为 “82200DAY LKV2UNCCOA1 支持CoA功能” 。

#### [对系统的影响](#ZH-CN_TOPIC_0310243674)

该特性会对每个使用AAA Server功能的用户进行处理，使用AAA Server功能的用户越多，该特性对系统的影响越大。

#### [应用限制](#ZH-CN_TOPIC_0310243674)

支持CoA消息基于承载上下文的业务修改，不支持基于会话级别的变更。

#### [原理概述](#ZH-CN_TOPIC_0310243674)

AAA server通过COA Request消息主动下发授权参数，变更用户的QoS信息和计费属性信息。

- 在2/3G网络中，AAA Server向GGSN发送CoA Request消息，消息中携带QoS-ID和计费属性信息。GGSN通过Update PDP Context Request消息通知左侧网元SGSN更新用户信息。
- 在4G网络中，AAA Server向PGW-C发送CoA Request消息，消息中携带QoS-ID和计费属性信息。PGW-C通过SGW-C向左侧网元MME发送Update Bearer Request消息更新用户信息。
- 在5G网络中，AAA Server向SMF发送CoA Request消息，消息中携带QoS-ID和计费属性信息。SMF通过Namf_Communication_N1N2MessageTransfer消息通知左侧网元AMF更新用户信息。

以2/3G网络为例，消息流程图如 [图2](#ZH-CN_TOPIC_0310243674__fig14657117806) 所示。

**图2** 消息流程图

<br>

![](特性概述_10243674.assets/zh-cn_image_0311137870_2.png)

1. AAA server向GGSN发送CoA Request消息，消息中携带QoS-ID和计费属性信息。
2. GGSN收到CoA Request消息后，处理CoA消息变更的属性，包含QoS-ID和计费属性信息。
3. GGSN发送Update PDP Context Request消息通知SGSN更新用户的QoS。
4. SGSN在QoS更新完成后，发送Update PDP Context Response消息给GGSN。更新成功后，用户数据流使用更新后的QoS和计费属性进行带宽控制和业务控制。
