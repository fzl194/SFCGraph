## [命令功能](#ZH-CN_MMLREF_0209653268)

**适用NF：NRF**

该命令用于查询NRF各类时长信息。

## [注意事项](#ZH-CN_MMLREF_0209653268)

无

#### [操作用户权限](#ZH-CN_MMLREF_0209653268)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0209653268)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| NFTYPE | NF类型 | 可选必选说明：可选参数<br>参数含义：该参数用于表示各定时器时长针对的NF类型。其中，DEFALUT范围为下面列举的具体NF外的其他NF类型。<br>数据来源：全网规划<br>取值范围：<br>- DEFAULT（DEFAULT）<br>- NRF（NRF）<br>- UDM（UDM）<br>- AMF（AMF）<br>- SMF（SMF）<br>- AUSF（AUSF）<br>- NEF（NEF）<br>- PCF（PCF）<br>- SMSF（SMSF）<br>- NSSF（NSSF）<br>- UDR（UDR）<br>- LMF（LMF）<br>- GMLC（GMLC）<br>- EIR_5G（EIR5G）<br>- SEPP（SEPP）<br>- UPF（UPF）<br>- N3IWF（N3IWF）<br>- AF（AF）<br>- UDSF（UDSF）<br>- BSF（BSF）<br>- CHF（CHF）<br>- NWDAF（NWDAF）<br>- CUSTOM_OCS（CUSTOM_OCS）<br>- SCP（SCP）<br>默认值：无<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0209653268)

查询NF类型为NRF的各类时长信息：

```
LST NRFTIMER: NFTYPE=NRF;
%%LST NRFTIMER:NFTYPE=NRF;%%
RETCODE = 0  执行成功

操作结果如下:
-------------------------
                                       NF类型  =  NRF
                              心跳超时次数  =  3
                        心跳周期时长(秒)  =  5
          服务发现缓存有效时长(秒)  =  86400
   服务发现缓存有效宽限时长(秒)  =  0
                        订阅有效时长(秒)  =  86400
订阅更新有效期误差宽限时长(秒)  =  300 
(结果个数 = 1)
```

## [输出结果说明](#ZH-CN_MMLREF_0209653268)

| 输出项名称 | 输出项解释 |
| --- | --- |
| NF类型 | 该参数用于表示各定时器时长针对的NF类型。其中，DEFALUT范围为下面列举的具体NF外的其他NF类型。 |
| 心跳超时次数 | 该参数用于表示NF与NRF之间的心跳超时次数（假设为N），当NRF连续N次未收到NF的心跳，则认为此NF心跳异常。 |
| 心跳周期时长(秒) | 该参数用于表示NF心跳周期时长。对于设置后新注册的NF立即生效，已注册的NF则在下一次心跳时生效。 |
| 服务发现缓存有效时长(秒) | 该参数用于表示系统对服务发现结果的缓存有效时长。在缓存有效时长内，该系统中的NF实例再次服务发现相同NF实例时，可以直接使用缓存中的结果，不需要再去NRF上处理。超过缓存有效时间， 服务发现结果缓存数据将被清除。系统中的NF实例再次服务发现相同NF实例时需要重新向NRF发起服务发现请求。 |
| 服务发现缓存有效宽限时长(秒) | 该参数用于表示NRF对服务发现结果的缓存有效宽限时长。NRF返回给NF的缓存有效时间会在“DISCVALIDTIMER”基础上再额外增加随机的宽限时长，随机宽限时长不超过本参数配置的值。目的是对NF服务发现的缓存时长进行离散化处理，防止NF侧的缓存结果中多个NF缓存时长同时失效，短时间触发大量重新服务发现而对NRF产生流量冲击。 |
| 订阅有效时长(秒) | 该参数用于表示NF服务订阅有效时长。在订阅有效时长内，订阅方NF订阅的内容生效且能正常收到被订阅NF的变化通知。超过此订阅有效时长，NF订阅的信息不再生效，被订阅方NF的变化将不再通知给订阅方NF。 |
| 订阅更新有效期误差宽限时长(秒) | 该参数用于表示NF订阅更新请求中有效期误差宽限时长。当NRF的订阅有效时长大于等于3600秒，订阅更新请求消息中携带的有效期和NRF生成的订阅有效期误差小于等于该宽限时长，NRF将向NF返回204状态码；误差大于该宽限时长或者订阅有效时长小于3600秒，NRF将向NF返回200状态码。 |
