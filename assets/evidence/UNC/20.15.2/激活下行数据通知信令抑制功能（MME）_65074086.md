## [操作场景](#ZH-CN_OPI_0165074086)

CPU过载或者Paging寻呼消息过多的场景下，当Downlink Data Notification消息的丢弃率达到设定的阈值时， UNC 通过在Downlink Data Notification Acknowledge消息中携带DL low priority traffic Throttling信元通知SGW-C抑制低优先级的Downlink Data Notification消息，以达到保护和提高高优先级业务被叫成功率的目的。

> **说明**
> 适用于MME。

## [操作步骤](#ZH-CN_OPI_0165074086)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开DDN信令抑制功能开关。
  **[**SET DDNTHROTPARA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/DDN信令抑制参数管理/设置DDN信令抑制参数(SET DDNTHROTPARA)_26305980.md)**

## [任务示例](#ZH-CN_OPI_0165074086)

任务描述

开启下行数据通知信令抑制特性，当Downlink Data Notification消息丢弃率达到80%时，抑制SGW-C发给UNC的Downlink Data Notification消息。DDN信令抑制时长为60秒，策略发送时长为2秒。

脚本

//打开DDN信令抑制功能开关。

```
SET DDNTHROTPARA: THROTSW=ON, THRESHOLD=80, DELAYVALUE=60, SNDDURATION =2;
```
