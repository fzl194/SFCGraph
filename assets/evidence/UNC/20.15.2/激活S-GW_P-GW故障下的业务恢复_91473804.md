## [操作场景](#ZH-CN_OPI_0191473804)

本操作指导介绍在运行网络中激活S-GW/P-GW故障下的业务恢复的操作过程。

在S-GW/P-GW故障或重启场景下，MME能感知网关故障，协助用户及时恢复VoLTE语音业务，实现网关设备的容灾功能。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0191473804)

前提条件

- 请仔细阅读[WSFD-201203 S-GW/P-GW故障下的业务恢复特性概述](特性概述_92137765.md)。
- 完成加载License。

- 建议同时开启[WSFD-102005 基于VoLTE的信令最短路径优选](../WSFD-102005 基于VoLTE的信令最短路径优选_70014684.md)特性，当MME检测到UE接入的P-GW或融合S/P-GW故障，促使用户回落到CS网络进行正常的语音业务，保证用户被叫业务正常进行。

## [操作步骤](#ZH-CN_OPI_0191473804)

1. 进入 “MML命令行-UNC” 窗口。
2. 配置本特性批量业务恢复功能。
  ![](激活S-GW_P-GW故障下的业务恢复_91473804.assets/notice_3.0-zh-cn_2.png)

  批量业务恢复会造成网元间的信令增加，可能导致网络拥塞。运营商需根据网络中VoLTE用户数量以及周边网元能力等因素，通过配置以下3个参数来避免在批量业务恢复过程中导致网络拥塞。
  [**SET GWOFFLOADPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/网关故障管理/设置网关迁移参数（SET GWOFFLOADPARA）_72345681.md)
3. **可选：**配置容灾APN特征参数。
  [**ADD GWRESTORAPN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/网关故障管理/增加网关容灾APN(ADD GWRESTORAPN)_72225761.md)

  > **说明**
  > - 当需要为IMS和紧急呼叫之外的其他APN提供容灾备份功能时，需要执行本命令。
  > - 此APN为支持容灾备份的APN，必须和 UDG 系统中配置的一致。否则可能导致主被叫业务无法一次恢复。
  > - UNC 默认支持IMS和紧急呼叫业务的恢复，无需对这两种业务额外配置APN。
4. 配置S-GW/P-GW故障场景下的业务恢复功能，运营商可根据实际需求进行如下配置。
  ![](激活S-GW_P-GW故障下的业务恢复_91473804.assets/notice_3.0-zh-cn_2.png)

  - MME向S-GW发送Echo消息的间隔周期可通过配置参数 “V2EI” ( [**SET GTPPUB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C协议参数配置/设置GTP-C协议参数(SET GTPPUB)_26145920.md) )来调整。为缩短VoLTE业务恢复时长，建议将 UNC 以及对端S-GW上的该参数值同时配置为 “60s” 。
    - UNC 上T3N3定时器的定时时长(T3RES*N3REQ)(通过命令 [**SET T3N3**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C协议参数配置/设置GTP-C T3_N3参数配置(SET T3N3)_26305730.md) 配置)需大于对端S-GW上该定时器的定时时长，否则当P-GW故障时，可能导致重选流程失败。
    -
      配置仅针对S-GW故障场景下的业务恢复功能。

      a. 配置Inter TAU过程中旧侧MME携带SGW Restoration Needed Indication至新侧MME功能。
            [**ADD GTPCV2CMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C V2协议兼容性/增加GTP-C V2协议兼容性(ADD GTPCV2CMPT)_26305734.md)
          b. 查询过滤故障状态的GTP-C路径开关是否已经打开。
            [**LST GTPPUB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C协议参数配置/查询GTP-C协议参数(LST GTPPUB)_72225599.md)
                  - 是，执行[4.d](#ZH-CN_OPI_0191473804__zh-cn_opi_0130428497_ADD_SGWCHARACT1)。
                  - 否，执行[4.c](#ZH-CN_OPI_0191473804__zh-cn_opi_0130428497_SET_GTPPUB1)。
          c. 打开过滤故障状态的GTP-C路径开关。
            [**SET GTPPUB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C协议参数配置/设置GTP-C协议参数(SET GTPPUB)_26145920.md)

            [**ADD PGWCHARACT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/P-GW属性/增加P-GW特性对接配置（ADD PGWCHARACT）_26305748.md)

## [任务示例](#ZH-CN_OPI_0191473804)

任务描述

启用S-GW/P-GW故障下的业务恢复特性，配置同时针对S-GW、P-GW故障场景下的业务恢复功能。

假设某运营商有24万个VoLTE语音用户，S-GW和P-GW合一部署，当S-GW\P-GW故障，希望迁移延迟时长为3分钟，并在延迟结束后的20分钟内完成对受影响用户的业务迁移，则迁移速率为：24*10000/20*60=200个，即每秒需要完成200个用户的迁移。

脚本

// UNC 上配置批量业务恢复功能参数。

```
SET GWOFFLOADPARA: DELAYTIMER=3, RATE=200, ITFOFFLOADSW=ON;
```

//配置S-GW网元故障重选特性相关信元。

```
ADD GTPCV2CMPT: IDX=0, MSGCLS=MM, MMMSGTYPE=CTX_RSP, CTXRSP=INDICATION, CTXRSPIND=SRNI-1;
```

//查询过滤故障状态的GTP-C路径开关是否已经打开。

```
LST GTPPUB:;
```

//(可选)打开过滤故障状态的GTP-C路径开关，如果此开关已打开，则不执行此步。

```
SET GTPPUB: DNSGTPCPATHFILTERSW=ON, V2EI=60;
```

//查询GTP原因值REJCAUSE是否为100。

```
LST GTPCAUSE:;
```

//(可选)配置特定原因失败重选P-GW功能。如果GTP原因值REJCAUSE为100，则不执行此步。

```
ADD GTPCAUSE: GTPVERSION=GTPV2, REJCAUSE=100;
```

//配置MME与S-GW间PRN功能以及MME ID传递功能。

```
ADD SGWCHARACT: RANGE=SPECIFIED, IPT=IPV4, IPV4="IPV4地址", IPV4MSK="255.255.255.255", PRN=YES, MMEID=YES;
```

//打开本特性的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2SRGF01", SWITCH=ENABLE;
```
