#### [适用NF](#ZH-CN_CONCEPT_0245188372)

AMF

#### [定义](#ZH-CN_CONCEPT_0245188372)

AMF支持基于PLMN（Public Land Mobile Network）配置NSSAI Inclusion Mode信元，来决定AMF控制UE在创建RRC（Radio Resource Control）连接时是否携带网络切片信息及携带此信息的方式。

#### [客户价值](#ZH-CN_CONCEPT_0245188372)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 可以为用户灵活选择网络切片，提升运营商的服务质量及运营商品牌。<br>- 可以规避漫游场景下冗余的信令流程，减少业务中断时间，减少客户投诉。 |
| 用户 | 用户不感知该特性。 |

#### [应用场景](#ZH-CN_CONCEPT_0245188372)

当运营商希望减少不同切片能力的网络之间的冗余信令流程，减少业务中断时间时，需要开启本特性，以指定3GPP接入方式下的网络切片包含模式及下发给UE的判断策略。

#### [可获得性](#ZH-CN_CONCEPT_0245188372)

**涉及NF**

| 涉及NF | 支持版本 | 功能说明 |
| --- | --- | --- |

#### [原理概述](#ZH-CN_CONCEPT_0245188372)

AMF应支持基于PLMN配置NSSAI Inclusion Mode，并在Registration Accept消息中将NSSAI inclusion mode发送给UE。

- AMF可以通过**[SET NGMMFUNC](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动性管理/设置5G移动性管理功能（SET NGMMFUNC）_09653748.md)**或**[ADD NGMMSUBDATA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/签约数据管理/增加用户移动性管理参数（ADD NGMMSUBDATA）_09651362.md)**命令配置NSSAI Inclusion mode参数，指定3GPP接入方式下的网络切片包含模式，可以根据运营商的要求控制UE在创建RRC连接时是否携带网络切片信息。有以下5种模式：
    - 不携带切片包含模式给UE（只有使用**[SET NGMMFUNC](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动性管理/设置5G移动性管理功能（SET NGMMFUNC）_09653748.md)**配置NSSAI Inclusion mode参数时才可配置此种模式）。
    - 模式A：初始注册、非UE能力变更触发的移动性注册更新场景下，UE在RRC连接建立时将携带Requested NSSAI给RAN；周期性注册更新、由于UE能力变更引起的移动性注册更新或者服务请求场景下，UE在RRC连接建立时将携带Allowed NSSAI给RAN。
    - 模式B：初始注册、非UE能力变更触发的移动性注册更新场景下，UE在RRC连接建立时将携带Requested NSSAI给RAN；周期性注册更新或由于UE能力变更引起的移动性注册更新场景下，UE在RRC连接建立时将携带Allowed NSSAI给RAN；用户面资源恢复触发服务请求场景下，UE携带涉及数传的PDU会话的切片集合；信令触发的服务请求场景下，UE携带信令涉及的网络切片信息。
    - 模式C：初始注册、非UE能力变更触发的移动性注册更新场景下，UE在RRC连接建立时将携带Requested NSSAI给基站；周期性注册更新、由于UE能力变更引起的移动性注册更新或者服务请求场景下，UE在RRC连接建立时不携带网络切片给基站。
    - 模式D：在任何场景下，UE在RRC连接建立时都不携带网络切片给基站。
  各种模式携带切片信息见 [表1](#ZH-CN_CONCEPT_0245188372__table15987632498) ，可根据实际需要配置对应的模式：
  *表1 切片包含模式在各个场景下携带的切片信息*

  | 场景 | 模式A | 模式B | 模式C | 模式D |
  | --- | --- | --- | --- | --- |
  | 初始注册 | Requested NSSAI | Requested NSSAI | Requested NSSAI | - |
  | 非UE能力变更触发的移动性注册更新 | Requested NSSAI | Requested NSSAI | Requested NSSAI | - |
  | UE能力变更引起的移动性注册更新 | Allowed NSSAI | Allowed NSSAI | - | - |
  | 周期性注册更新 | Allowed NSSAI | Allowed NSSAI | - | - |
  | 服务请求 | Allowed NSSAI | - 用户面资源恢复触发：涉及数传的PDU会话的网络切片信息<br>- 信令触发：信令涉及的网络切片信息 | - | - |
- 除AMF本地配置的NSSAI Inclusion mode参数外，还存在用户签约的NSSAI Inclusion mode，AMF可以通过**[SET NGMMFUNC](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动性管理/设置5G移动性管理功能（SET NGMMFUNC）_09653748.md)**命令配置决策使用哪个NSSAI Inclusion mode给UE下发切片包含模式。
