#### [业务场景](#ZH-CN_TOPIC_0000001479926121)

UE首次接入网络、数据加密和完整性检查失败等场景，AMF调用AUSF的鉴权服务进行初始化鉴权。

1. 内部流程：AUSF选择流程。
  触发条件：初始注册中选择AUSF进行鉴权。
  相关机制：AMF选择老侧AMF的机制参见UNC章节：注册管理流程的网元选择。
2. 消息交互：AMF发送[Nausf_UEAuthentication_CreateUeAuthContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nausf/N12/Nausf_UEAuthentication/Nausf_UEAuthentication_CreateUeAuthContext Request_56087661.md)给AUSF。
  触发条件：AMF接收到的Registration request消息发现用户为初始注册，则AMF调用AUSF的鉴权服务进行初始化鉴权。
  关键信元：
  | supiOrSuci | 表示UE的SUPI或SUCI。 |
  | --- | --- |
  | servingNetworkName | 表示服务网络名称。 |
  相关机制：当AMF判断UE已有合法5G-GUTI时，消息中包含SUPI，否则消息中包含SUCI。
3. 消息交互：AUSF发送Nudm_UEAuthentication_Get Request给UDM。
  相关机制：AUSF接收到Nausf_UEAuthentication_Authenticate Request消息后，通过servingNetworkName检查AMF是否有资格使用该服务网络。如果授权使用，则AUSF向UDM发送Nudm_UEAuthentication_Get Request消息，消息中包含SUCI或SUPI、服务网络名称。
4. 消息交互：UDM发送Nudm_UEAuthentication_Get Response给AUSF。
  相关机制：如果消息中包含的是SUCI，那么UDM在处理请求前先将SUCI解密为SUPI。UDM根据SUPI获取用户真身份，根据用户签约信息（ **MOD AMDATA** ）选择鉴权方式。之后UDM生成鉴权向量EAP-AKA' AV，包含RAND、AUTN、XRES、CK'、IK'并发送给AUSF。如果Nudm_UEAuthentication_Get Request包含SUCI时，则Nudm_UEAuthentication_Get Response消息中包含SUPI。
5. 消息交互：AUSF发送[Nausf_UEAuthentication_CreateUeAuthContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nausf/N12/Nausf_UEAuthentication/Nausf_UEAuthentication_CreateUeAuthContext Response_56287373.md)给AMF。
  触发条件：AUSF获取鉴权方式和鉴权向量后发送Nausf_UEAuthentication_CreateUeAuthContext Response给AMF。
  关键信元：
  | authType | 表示该用户所使用的鉴权方式。比如："5G_AKA"、"EAP_AKA_PRIME"、"EAP_TLS" |
  | --- | --- |
  | _links | - 如果选择5G-AKA鉴权方式，该信元中携带名称为“5g-aka”的成员和用于执行确认的URI。<br>- 如果选择EAP鉴权方式，该信元携带名称为“eap-session”的成员和用于执行EAP会话的URI。 |
  | 5gAuthData | 表示5G-AKA鉴权或EAP鉴权相关信息。 |
  相关机制：AUSF将EAP-Request/AKA'-Challenge发送给AMF，携带RAND、AUTN。
6. 消息交互：AMF发送[Authentication request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Authentication request_76784595.md)给UE。
  触发条件：AMF获取鉴权向量后向UE发起Authentication request消息，发起鉴权。
  关键信元：
  | ngKSI | ngKSI为网络侧分配的NAS密钥集标识，用于识别K<br>AMF<br>和鉴权成功后产生的安全上下文。 |
  | --- | --- |
  | ABBA | ABBA用于实现安全特性的降级保护。 |
  | Authentication parameter RAND | RAND为网络提供给UE的不可预知的随机数，UE使用RAND计算鉴权响应参数及安全保密参数。 |
  | Authentication parameter AUTN | AUTN为鉴权令牌，是网络提供给UE的参数，用于UE对网络进行鉴权。 |
  | EAP message | EAP message信元用于传送EAP消息。 |
  相关机制：如果UE鉴权失败，则UE向AMF返回Authentication failure并明确原因。失败原因为ngKSI already in use时，核心网可以重新选择ngKSI并向UE发起一个新的鉴权流程。
7. 消息交互：UE发送[Authentication response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Authentication response_76783899.md)给AMF。
  触发条件：UE收到鉴权请求后根据收到的AUTN鉴权网络，并根据鉴权结果向AMF发送响应消息。
  关键信元：
  | EAP message | EAP message信元用于传送EAP消息。 |
  | --- | --- |
8. 消息交互：AMF发送[Nausf_UEAuthentication_EapAuthMethod Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nausf/N12/Nausf_UEAuthentication/Nausf_UEAuthentication_EapAuthMethod Request_56287381.md)给AUSF。
  触发条件：AMF将从UE收到的鉴权响应发送给AUSF。
  关键信元：
  | eapPayload | EAP消息透传给AUSF。 |
  | --- | --- |
9. 消息交互：AUSF发送[Nausf_UEAuthentication_EapAuthMethod Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nausf/N12/Nausf_UEAuthentication/Nausf_UEAuthentication_EapAuthMethod Response_56207421.md)给AMF。
  触发条件：AUSF向AMF发送网络鉴权结果。
  关键信元：
  | eapPayload | EAP消息。 |
  | --- | --- |
  | kseaf | 如果鉴权成功，则携带Kseaf。 |
  | authResult | 表示鉴权结果，比AUTHENTICATION_SUCCESS。 |
  相关机制：消息中包含EAP Success和K <sub>SEAF</sub> 。如果AUSF在鉴权初始阶段从AMF收到SUCI，那么在Nausf_UEAuthentication_Authenticate Response消息中包含SUPI。
10. 消息交互：AMF发送[Authentication result](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/Authentication result_76784548.md)给UE。
  关键信元：
  | ngKSI | ngKSI信元表示由网络分配的NAS密钥集标识。 |
  | --- | --- |
  | ABBA | ABBA信元用于实现安全特性的降级保护。 |
11. 消息交互：AUSF发送Nudm_UEAuthentication_ResultConfirmation Request给UDM。
  相关机制：AUSF通过Nudm_UEAuthentication_ResultConfirmation Request消息通知UDM鉴权结果，包含SUPI、鉴权时间、认证方法和服务网络名称。
12. 消息交互：UDM发送Nudm_UEAuthentication_ResultConfirmation Response给AUSF。
  相关机制：UDM存储鉴权结果，并通过Nudm_UEAuthentication_ResultConfirmation Response响应AUSF。
