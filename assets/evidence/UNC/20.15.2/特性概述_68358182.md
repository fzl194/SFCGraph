#### [适用NF](#ZH-CN_CONCEPT_0168358182)

SGSN、 MME

#### [定义](#ZH-CN_CONCEPT_0168358182)

支持 ODB （Operator Determined Barring）特性，即运营商的限制，运营商能够通过设置ODB参数限制一些用户的业务和服务。

#### [客户价值](#ZH-CN_CONCEPT_0168358182)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | ODB业务有助于<br>运营商<br>根据需要来灵活控制用户可以使用的各种业务。 |
| 用户 | 无。 |

#### [应用场景](#ZH-CN_CONCEPT_0168358182)

运营商能够通过各种ODB业务来限制一些用户的业务和服务。如禁止呼出业务，当用户满足一定条件时（如欠费等），运营商能够禁止该用户的所有呼出。

#### [可获得性](#ZH-CN_CONCEPT_0168358182)

**涉及NF**

UNC 作为SGSN/MME启动本特性，涉及的网元如下表所示。

| 涉及NF | 支持版本 | 功能说明 |
| --- | --- | --- |
| SGSN/MME | UNC 20.0.0及后续版本 | 根据签约数据中的ODB参数，限制用户相应的业务。 |
| HLR/HSS | 无特殊要求 | 签约用户的ODB参数。 |

**License支持**

本特性只有获得了License许可后才能获得该特性的服务，对应的License控制项为 “82206154 LKV2ODB02 支持ODB” 。

#### [与其他特性的交互](#ZH-CN_CONCEPT_0168358182)

本特性不涉及与其他特性的交互。

#### [对系统的影响](#ZH-CN_CONCEPT_0168358182)

本特性对系统无影响。

#### [应用限制](#ZH-CN_CONCEPT_0168358182)

本特性需要和 HLR/ HSS配合完成，要求 HLR/ HSS支持ODB签约信息，能够根据签约数据决定是否触发ODB业务，以及向 VLR 发送用户的签约数据。

UNC 暂不支持以下ODB类型：

- 禁止所有区域间呼出
- 禁止非归属国家区间呼出
  即，用户禁止除到归属PLMN国家外的区域间呼出。
- 禁止非归属国区间国际呼出
  即，用户禁止除到归属PLMN国家外的区域间呼出以及国际呼出。
- 运营商自定义ODB类型1
- 运营商自定义ODB类型2
- 运营商自定义ODB类型3
- 运营商自定义ODB类型4

#### [原理概述](#ZH-CN_CONCEPT_0168358182)

*表1 UNC 支持的ODB类型*

| 支持的ODB类型 | 支持的网元 |
| --- | --- |
| 禁止所有呼出 | SGSN |
| 禁止所有国际呼出 | SGSN |
| 禁止除归属国外所有国际呼出 | SGSN |
| 禁止用户所有分组域业务 | SGSN、MME |
| 禁止漫游用户通过HPLMN的接入点接入 | SGSN、MME |
| 禁止漫游用户通过VPLMN的接入点接入 | SGSN、MME |
