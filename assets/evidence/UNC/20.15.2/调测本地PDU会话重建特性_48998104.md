# 调测本地PDU会话重建特性

- [操作场景](#ZH-CN_OPI_0248998104__1.3.1)
- [必备事项](#ZH-CN_OPI_0248998104__1.3.2)
- [操作步骤](#ZH-CN_OPI_0248998104__1.3.3)

## [操作场景](#ZH-CN_OPI_0248998104)

当运营商部署本地PDU会话重建时，需对 UNC 的本地PDU会话重建功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于AMF、SMF。

## [必备事项](#ZH-CN_OPI_0248998104)

前提条件

- 请仔细阅读[WSFD-205011 本地PDU会话重建特性概述](特性概述_48998101.md)。
- 完成[激活本地PDU会话重建特性](激活本地PDU会话重建特性_48998103.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 123450123456789 | 测试终端自带 | - |

## [操作步骤](#ZH-CN_OPI_0248998104)

AMF侧实现本地PDU会话重建功能的调测：

1. 进入 “MML命令行-UNC” 窗口。
2. 执行[**LST LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md)命令，查询License配置开关是否已打开。
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 3](#ZH-CN_OPI_0248998104__li72721737103914)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)命令打开本特性对应的License配置开关。
3. 在 UNC 上 [创建用户跟踪](../../../../../网络运维/日常维护/UNC基础运维操作/创建消息跟踪/创建用户跟踪/用户跟踪_72467812.md) 。
4. 用户开机附着到网络，在同一区域的SMF上激活数据业务的PDU会话连接，执行**[DSP USRSESSIONCTX](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/用户数据库管理/显示指定用户的会话信息（DSP USRSESSIONCTX）_09653734.md)**命令查询SMF信息。
  预期结果：参数 **“** 归属SMF的Instance ID **”** 的中标识区域字段的值与当前AMF的Instance ID标识区域字段的值相同。
5. 用户移动到另一个AMF的覆盖范围下，该AMF与原来的SMF属于不同的区域。
6. 在用户状态变为CM-IDLE态之后，查看用户跟踪。可以看到在用户进入CM-IDLE状态后，AMF向该UE发送Paging消息要求UE重新建立PDU会话连接的时间在00:02:00~00:04:00时间段内（此时用户进入CM-IDLE状态已超过60秒）。执行**[DSP USRSESSIONCTX](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/用户数据库管理/显示指定用户的会话信息（DSP USRSESSIONCTX）_09653734.md)**命令查询SMF信息。
  预期结果：参数 **“** 归属SMF的Instance ID **”** 的中标识区域字段的值与 [4](#ZH-CN_OPI_0248998104__li627212370398) 不同，但与当前新AMF的Instance ID标识区域字段的值相同。

SMF侧实现本地PDU会话重建功能的调测：

1. 进入 “MML命令行-UNC” 窗口。
2. 执行[**LST LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md)命令，查询License配置开关是否已打开。
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 3](#ZH-CN_OPI_0248998104__li149951237122918)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)命令打开本特性对应的License配置开关。
3. 在 UNC 上 [创建用户跟踪](../../../../../网络运维/日常维护/UNC基础运维操作/创建消息跟踪/创建用户跟踪/用户跟踪_72467812.md) 。
4. 用户开机附着到网络，在同一区域的SMF上激活数据业务的PDU会话连接（只存在缺省QoSFlow）。
5. 用户移动到另一个SMF的覆盖范围下，该SMF与原来的SMF属于不同的区域。
6. 在新的SMF上查看用户跟踪。
  预期结果：可以看到SMF发起PDU Session Release流程，AMF向UE发送的PDU Session Release Command消息中携带原因值为“reactivation requested”，要求UE重新建立PDU会话连接。
