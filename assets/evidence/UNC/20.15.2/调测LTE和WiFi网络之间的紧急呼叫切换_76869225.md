# 调测LTE和WiFi网络之间的紧急呼叫切换

- [操作场景](#ZH-CN_OPI_0276869225__1.3.1)
- [必备事项](#ZH-CN_OPI_0276869225__1.3.2)
- [操作步骤](#ZH-CN_OPI_0276869225__1.3.3)

## [操作场景](#ZH-CN_OPI_0276869225)

本操作指导用于验证LTE和WiFi网络之间的紧急呼叫切换业务功能。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0276869225)

前提条件

- 请仔细阅读[WSFD-104505 LTE和WiFi 网络之间的切换](../WSFD-104505 LTE和WiFi 网络之间的切换_76144612.md)。
- 已完成[激活LTE和WiFi网络之间的紧急呼叫切换](激活LTE和WiFi网络之间的紧急呼叫切换_76869184.md)。

数据

该操作无需准备数据。

工具

OM Portal跟踪。

## [操作步骤](#ZH-CN_OPI_0276869225)

- 在相关网元均完成本特性的配置后，可以采用以下步骤检查特性工作是否正常：
  如下以用户能够从WiFi网络成功切换到LTE网络，并根据签约数据中的PGW-C Identity选择PGW-C为例。
    1. 进入 “MML命令行-UNC” 窗口。
    2. [**DSP COMMMCTX**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/融合接入业务管理/融合用户数据库管理/显示移动性管理上下文的相关信息（DSP COMMMCTX）_58365337.md): QUERYOPT=BYIMSI, IMSI="IMSI";
      预期结果： UNC 上面没有用户的信息。
    3. 用户在HSS上签约了1个PDN上下文。该PDN上下文的APN名为APN1，PDN上下文的“PDN GW allocation type”是动态分配。
    4. 用户已经注册到WiFi网络，并在HSS中注册了PGW-C的IP。
    5. 创建用户跟踪、S1–MME和GTP-C接口跟踪任务。
    6. 增加PGW-C FQDN1对应的IP地址：
      [**ADD IPV4DNSH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS Hostfile管理/增加IPV4 DNS Hostfile记录(ADD IPV4DNSH)_26145884.md) : HSINDEX=1, HOSTNAME="PGW-C HOSTNAME", ADDRSECTION=SECTION1, IPV4ADDR1="10.10.10.10";
      预期结果： [**LST IPV4DNSH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS Hostfile管理/查询IPV4 DNS Hostfile记录(LST IPV4DNSH)_72345485.md) 能够显示 “IPV4DNSH” 配置信息。
    7. 增加PGW-C FQDN1对应的PGW-C配置，类型为GTP：
      [**ADD DNSN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS NAPTR管理/增加DNS NAPTR记录(ADD DNSN)_72225569.md) : FQDN="PDN GW FQDN1", HSINDEX=1, ENTITY=PGW, INTYPE=S5;
      预期结果： [**LST DNSN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS NAPTR管理/查询DNS NAPTR记录(LST DNSN)_26145892.md) 能够显示 “DNSN” 配置信息。
    8. 用户从WiFi网络切换到LTE网络。
      预期结果：
          - 从用户跟踪和S1–MME接口跟踪结果上看，用户发起了“Request type”为“handover”的附着。如[图1](#ZH-CN_OPI_0276869225__fig1)所示。
            **图1** Attach请求消息指示PDN创建类型是Handover（样例）

            ![](调测LTE和WiFi网络之间的紧急呼叫切换_76869225.assets/zh-cn_image_0000001165936321_2.png "点击放大")
          - 从用户跟踪和GTP-C接口跟踪结果上看，MME选择了[步骤6](#ZH-CN_OPI_0276869225__step6)中配置的“IPV4ADDR1”作为PGW-C IP，并包含在Create Session Request消息中。如[图2](#ZH-CN_OPI_0276869225__fig2)所示。
            **图2** MME从签约数据中提取到PGW-C Identity，作为Create Session Request消息中的PGW-C的IP地址来创建承载（样例）

            ![](调测LTE和WiFi网络之间的紧急呼叫切换_76869225.assets/zh-cn_image_0000001119056638_2.png "点击放大")
    9. 查询用户从WiFi网络切换到LTE网络后的状态：
      [**DSP COMMMCTX**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/融合接入业务管理/融合用户数据库管理/显示移动性管理上下文的相关信息（DSP COMMMCTX）_58365337.md) : QUERYOPT=BYIMSI, IMSI="IMSI";
      预期结果：用户状态为 “ECM-CONNECTED” 。
    10. 查询在[步骤8](#ZH-CN_OPI_0276869225__step7)中激活的用户的承载上下文信息：
      [**DSP SMCTX**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/系统管理/用户数据库管理/显示承载上下文(DSP SMCTX)_72226033.md) : QUERYOPT=BYIMSI, IMSI="IMSI", IDTYPE=BYBEARID/NSAPI;
      预期结果：上下文中的“PGW-C address of control plane(S5/S8)”与 [步骤6](#ZH-CN_OPI_0276869225__step6) 中配置的 “IPV4ADDR1” 相同。上下文中的 “Protocol type (S5/S8)” 是GTP。
