#### [命令功能](#ZH-CN_CONCEPT_0000001600600589)

该命令用于显示OSPFv3邻居信息。

#### [注意事项](#ZH-CN_CONCEPT_0000001600600589)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001600600589)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001600600589)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| NBRROUTERID | 邻居路由器标识 | 可选必选说明：可选参数<br>参数含义：邻居路由器标识。<br>数据来源：对端协商<br>取值范围：IPv4地址类型。点分十进制格式。<br>默认值：无 |
| PROCID | OSPFv3进程号 | 可选必选说明：可选参数<br>参数含义：OSPFv3进程号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～4294967295。<br>默认值：无 |
| IFNAME | 接口名称 | 可选必选说明：可选参数<br>参数含义：接口名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。<br>默认值：无<br>配置原则：请使用LST INTERFACE命令查看可用接口。 |

#### [使用实例](#ZH-CN_CONCEPT_0000001600600589)

显示设备OSPFv3进程号为1的所有节点邻居信息：

```
DSP OSPFV3PEER: PROCID=1;
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000001600600589)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 邻居路由器标识 | 该参数指定邻居路由器标识。 |
| OSPFv3进程号 | 该参数指定用户创建的OSPFv3进程号。 |
| OSPFv3区域标识 | 该参数指定用户创建的OSPFv3区域号。 |
| 接口名称 | 该参数指定本端建立邻居的接口名称。 |
| 邻居状态 | 该参数指定邻居状态。Down表示邻居的初始状态，表示没有邻居收到任何信息。在NBMA网络上，Hello报文在该状态时仍然可以收发，周期为Poll Interval（Hello Interval<Poll Interval）；Attempt只存在于NBMA网络上，表明正在尝试建立邻居关系。此时，Hello报文的收发周期为Hello Interval（Hello Interval<Poll Interval）；Init表明已经接收到了从邻居发送来的Hello报文；2Way表明已经接收到了从邻居发送过来的Hello报文，并且该Hello报文的Neighbor List中包含本地Router ID，即双方可以互通；ExStart为建立Adjacency的第一步，进行主从关系、DD Sequence Number的协商；Exchange表示开始进行LSDB同步操作，交互的报文有DD报文、LSR报文、LSU报文；Loading表示LSDB正在进行同步操作，交互的报文有LSR报文和LSU报文；Full说明邻居的LSDB已经同步完成，双方建立了Full邻接关系。 |
| 邻居路由器优先级 | 该参数指定邻居的指定路由器优先级。 |
| 邻居接口状态 | 该参数指定邻居接口的协议状态。 |
| 邻居Dead间隔（s） | 该参数指定邻居距离失效的时间。 |
| 实例号 | 该参数指定邻居的实例号。 |
| 主机名 | 该参数指定邻居的主机名。 |
| 邻居地址 | 该参数指定邻居IPv6地址。 |
