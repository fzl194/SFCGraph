# 调测到PCRF的数据

- [操作场景](#ZH-CN_OPI_0231422954__1.3.1)
- [对系统的影响](#ZH-CN_OPI_0231422954__1.3.2)
- [必备事项](#ZH-CN_OPI_0231422954__1.3.3)
- [操作步骤](#ZH-CN_OPI_0231422954__1.3.4)

## [操作场景](#ZH-CN_OPI_0231422954)

当运营商在部署分组交换网，新增UNC或PCRF时，UNC和PCRF完成互通数据配置后，需要通过调试手段检查GGSN/PGW-C与PCRF之间的链路是否连通，以及调测基本的 PCC 功能。

> **说明**
> 适用于 GGSN、 PGW-C。

## [对系统的影响](#ZH-CN_OPI_0231422954)

如果GGSN/PGW-C与PCRF之间的链路不通，则运营商无法应用PCC功能，相关的PCC用户无法激活，或者只能采用非PCC的方式激活。

## [必备事项](#ZH-CN_OPI_0231422954)

前提条件

- 请仔细阅读 [PCC基本功能（2G/3G/4G，基于Gx接口）](../../PCC基本功能（2G_3G_4G，基于Gx接口）_29056156.md) 。
- 完成[配置与PCRF对接数据](../激活PCC基本功能/配置与PCRF对接数据_30805096.md)。
- PCRF上已完成相应配置，并确保已配置到UNC的回程路由。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| [**ADD PCRF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF/增加PCRF（ADD PCRF）_09897101.md) | PCRF主机名（HOSTNAME） | pcrf_1<br>pcrf_2 | 已配置数据中获取 | 取自<br>[配置与PCRF对接数据](../激活PCC基本功能/配置与PCRF对接数据_30805096.md)<br>中配置的数据。 |

工具

该操作无需准备工具。

## [操作步骤](#ZH-CN_OPI_0231422954)

1. 进入 “MML命令行-UNC” 窗口。 执行 [**LST LICENSESWITCH**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) 命令查询 PCC基本功能 对应的License配置开关是否打开。
  ```
  LST LICENSESWITCH:LICITEM="LKV3W9SPCC11";
  ```
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0231422954__stp1)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)命令打开本特性对应的License配置开关。
2. 执行 [**DSP PCRFSTATUS**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF状态/显示PCRF状态（DSP PCRFSTATUS）_09897111.md) 命令，查看PCRF的状态是否正常。
  ```
  DSP PCRFSTATUS:PCRFNAME="pcrf_1";
  ```
  ```
  ......

  -------------------------
    PCRF主机名  =  pcrf_1
       POD名称  =  uncpod-0
      本端地址  =  10.8.10.1:16400
    本端子地址  =  -
      对端地址  =  10.11.21.59:3868
       Gx 状态  =  Normal
    本端主机名  =  pgw_1 
  (结果个数 = 1)
  ---    END
  ```
  ```
  DSP PCRFSTATUS:PCRFNAME="pcrf_2";
  ```
  ```
  ......

  -------------------------
  PCRF主机名    POD名称     本端地址            本端子地址    对端地址            Gx 状态  本端主机名
  pcrf_2        uncpod-0    10.8.10.2:16400      -            10.11.21.60:3868    Normal   pgw_1
  pcrf_2        uncpod-0    10.8.10.3:16400      -            10.11.21.61:3868    Normal   pgw_1
  (结果个数 = 2)
  ---    END
  ```
    - 如果检测PCRF状态正常，则GGSN/PGW-C和PCRF之间的链路互通调测结束。
    - 如果检测PCRF状态异常，请执行[步骤 3](#ZH-CN_OPI_0231422954__stp2)。
3. 执行 [**LST PCRF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF/查询PCRF（LST PCRF）_09897104.md) 命令，查看GGSN/PGW-C上的PCRF设备标识是否和规划值及PCRF上的配置一致。
  ```
  LST PCRF:HOSTNAME="pcrf_1";
  ```
  ```
  ......

  -------------------------
                         PCRF主机名  =  pcrf_1
                           PCRF域名  =  example.com
                            VPN实例  =  vpn_gxif
     Supported-Features动态协商开关  =  使能
                        Feature列表  =  RELEASE8-1&RELEASE9-1&RELEASE10-1
                             DSCP值  =  255
  (结果个数 = 1)
  ---    END
  ```
  ```
  LST PCRF:HOSTNAME="pcrf_2";
  ```
  ```
  ......

  -------------------------
                         PCRF主机名  =  pcrf_2
                           PCRF域名  =  example.com
                            VPN实例  =  vpn_gxif
     Supported-Features动态协商开关  =  使能
                        Feature列表  =  RELEASE8-1&RELEASE9-1&RELEASE10-1
                             DSCP值  =  255
  (结果个数 = 1)
  ---    END
  ```
    - 如果配置一致，请执行[步骤 5](#ZH-CN_OPI_0231422954__stp4)。
    - 如果配置不一致，请执行[步骤 4](#ZH-CN_OPI_0231422954__stp3)。
4. 修改GGSN/PGW-C上PCRF设备标识。
    a. 执行[**RMV PCRF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF/删除PCRF（RMV PCRF）_09897103.md)命令，删除原有配置。
    b. 执行[**ADD PCRF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF/增加PCRF（ADD PCRF）_09897101.md)命令，按照规划数据和PCRF上的数据重新配置GGSN/PGW-C的PCRF信息。
    c. 再次执行[步骤 2](#ZH-CN_OPI_0231422954__stp1)，查看PCRF状态。
    - 如果检测PCRF状态正常，则GGSN/PGW-C和PCRF之间的链路互通调测结束。
    - 如果检测PCRF状态异常，请执行[步骤 5](#ZH-CN_OPI_0231422954__stp4)。
5. 执行 [**LST LOGICINF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/计费和策略接口管理/逻辑接口/查询逻辑接口（LST LOGICINF）_09896726.md) 和 [**LST DIAMLOCINFO**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/计费和策略接口管理/Diameter管理/Diameter连接/Diameter本端信息/查询Diameter本端信息（LST DIAMLOCINFO）_09897274.md) 命令，查看UNC的Gx口设备标识与IP等是否和规划值及PCRF上的配置一致。
  ```
  LST LOGICINF:;
  ```
  ```
  ......

  -------------------------
  逻辑接口名称    逻辑接口的IPv4地址1     逻辑接口的IPv4掩码1  逻辑接口的IPv4地址2  逻辑接口的IPv4掩码2     VPN实例名称  逻辑接口的IP版本   ......
  gxif1/0/0         10.8.10.1              255.255.255.255          NULL                    NULL              vpn_gxif      IPv4              ......
  gxif1/0/1         10.8.10.2              255.255.255.255          NULL                    NULL              vpn_gxif      IPv4              ......
  gxif1/0/2         10.8.10.3              255.255.255.255          NULL                    NULL              vpn_gxif      IPv4              ......
  (结果个数 = 3)
  ---    END
  ```
  ```
  LST DIAMLOCINFO:;
  ```
  ```
  ......

  -------------------------
  本端主机名    本端域名       产品名称    Vendor-Id AVP值    Firmware-RevisionAVP值    Origin-State-Id AVP    Supported-Vendor-Id AVP

  pgw_2        huawei.com       unc           10415              0                          不使能                不使能

  ---    END
  ```
    - 如果配置一致，请执行[步骤 7](#ZH-CN_OPI_0231422954__stp6)。
    - 如果配置不一致，请执行[步骤 6](#ZH-CN_OPI_0231422954__stp5)。
6. 修改GGSN/PGW-C的Gx口设备标识。
    a. 执行[**RMV DIAMCONNECTION**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/计费和策略接口管理/Diameter管理/Diameter连接/Diameter链路/删除Diameter链路（RMV DIAMCONNECTION）_09897268.md)和[**RMV DIAMLOCINFO**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/计费和策略接口管理/Diameter管理/Diameter连接/Diameter本端信息/删除Diameter本端信息（RMV DIAMLOCINFO）_09897273.md)命令，删除原有配置。
    b. 执行[**ADD DIAMLOCINFO**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/计费和策略接口管理/Diameter管理/Diameter连接/Diameter本端信息/增加Diameter本端信息（ADD DIAMLOCINFO）_09897271.md)和[**ADD DIAMCONNECTION**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/计费和策略接口管理/Diameter管理/Diameter连接/Diameter链路/增加Diameter链路（ADD DIAMCONNECTION）_09897266.md)命令，按照规划数据重新配置GGSN/PGW-C的Gx口设备标识。
    c. **可选：**如果Gx接口IP与规划值不一致，请执行[**RMV LOGICINF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/计费和策略接口管理/逻辑接口/删除逻辑接口（RMV LOGICINF）_09896725.md)命令，删除原有配置。
    d. **可选：**执行[**ADD LOGICINF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/计费和策略接口管理/逻辑接口/增加逻辑接口（ADD LOGICINF）_09896723.md)命令，按照规划数据重新配置GGSN/PGW-C的Gx接口IP数据。
    e. 再次执行[步骤 2](#ZH-CN_OPI_0231422954__stp1)，查看PCRF状态。
    - 如果检测PCRF状态正常，则GGSN/PGW-C和PCRF之间的链路互通调测结束。
    - 如果检测PCRF状态异常，请执行[步骤 7](#ZH-CN_OPI_0231422954__stp6)。
7. 检查GGSN/PGW-C到PCRF的互通是否正常。
    a. 根据使用的路由方式，调测GGSN/PGW-C到PCRF的路由。
    b. 根据使用的可靠性组网方式，调测GGSN/PGW-C到PCRF的组网。
    c. 再次执行[步骤 2](#ZH-CN_OPI_0231422954__stp1)，查看PCRF状态。
    - 如果检测PCRF状态正常，则GGSN/PGW-C和PCRF之间的链路互通调测结束。
    - 如果检测PCRF状态异常，请执行[步骤 8](#ZH-CN_OPI_0231422954__stp7)。
8. 查看是否存在ID为 “81024” （PCRF无响应告警）的告警。
    - 如果产生告警，请参考[ALM-81024 PCRF无响应](../../../../../../../网络运维/故障处理/UNC告警处理/业务告警/SMF&GW-C&GGSN/ALM-81024 PCRF无响应_18580452.md)处理此问题。
    - 如果没有产生告警，请执行[步骤 9](#ZH-CN_OPI_0231422954__step437413411319)。
9. 收集信息并寻求技术支持。
    a. 在相应VNFC窗口下执行 [**EXP MML**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    b. 收集并保存调测过程中所有查询信息。
    c. 查看并收集PCRF设备配置及接口状态信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
