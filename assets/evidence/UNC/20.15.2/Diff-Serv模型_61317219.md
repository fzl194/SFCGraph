# Diff-Serv模型

Diff-Serv（Differentiated Services）是一种适用于骨干网络，可满足多种服务需求的IP QoS模型。该模型定义了加速转发EF（Expedited Forwarding）、确保转发AF（Assured Forwarding）等几类标准的转发服务。

在Diff-Serv体系中，将IP报文中ToS（Type of Service）字段重新定义，并命名为DS（Differentiated Services）字段，用户通过标记分组的DS字段来申请不同层次的服务。DS字段的前6位称为DSCP（Differentiated Services Code Point），具有相同DSCP值的分组的集合称为BA（Behavior Aggregate）。设备保留有DSCP到每跳转发行为PHB（Per-Hop Behaviors）的映射，PHB是指满足某类转发需求的行为，如队列管理等QoS行为。分组进入设备时，根据其DSCP标记划归到一个BA并选取特定的PHB进行转发。

如 [图1](#ZH-CN_CONCEPT_0161317219__fig_dc_fenix_nps_feature_qos_000201) 所示，不同Diff-Serv域之间用SLA（Service Level Agreement）和TCA（Traffic Conditioning Agreement）来联系，用以定义网络中的数据包优先级和流分类规则。不同的Diff-Serv域内有不同的PHB，Diff-Serv域中的内部设备根据PHB决定的策略来转发数据包，并根据分组的DSCP值，进行队列调度和分组丢弃。网络入口的边缘设备对数据业务使用流量策略，即流分类、流量控制，设置DSCP等。

目前，IETF定义了三种标准的PHB：加速转发EF（Expedited Forwarding）、确保转发AF（Assured Forwarding）和尽力而为BE（Best-Effort），BE是缺省的PHB。

- EF PHB
  加速转发被定义为这样的一种转发处理：从任何Diff-Serv节点发出的信息流速率在任何情况下必须获得等于或大于设定值的速率。EF PHB在DS域内不能被重新标记。重新标记仅允许在边界节点，并且要求新的DSCP（不同DS域可以选用不同的DSCP到PHB的映射）满足EF PHB的特性。当采用“隧道”技术时，外部分组也需标记为EF。
  制定EF PHB的目标是在DS域内模拟一种虚拟租用线（Virtual Leased Line）的转发效果，提供一种低丢包率、低延迟、高带宽的转发服务。
- AF PHB
  确保转发的推出是为了满足如下需求：用户在与ISP订购带宽服务时，允许业务量超出所订购的规格。对不超出所订购规格的流量要求确保转发的质量，对超出规格的流量希望是降低服务待遇继续转发，而不只是简单地被丢弃。
  当前定义了四类AF，即AF1、AF2、AF3、AF4。每一类AF业务的分组又可以细分为三种不同的丢弃优先级。AF编码点AFij表示AF类为i（1<=i<=4），丢弃优先级为j（1<=j<=3）。运营商在提供AF服务时，为每类AF分配不同的带宽资源。
  对AF PHB的一个特别要求是：流量控制不能改变同一信息流中分组的顺序。比如，某一业务流中的不同分组归属同一AF类，但在流量监管时被标记了不同的丢弃优先级，此时，虽然不同分组的丢包概率不同，但是不同组之间的相互顺序不能改变。这种机制特别适合于多媒体业务的传输。
- BE PHB
  即传统的IP分组投递服务，只关注可达性，其他方面不做任何要求。任何设备必须支持BE PHB。
