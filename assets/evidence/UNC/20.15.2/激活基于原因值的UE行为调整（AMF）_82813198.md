## [操作场景](#ZH-CN_OPI_0282813198)

当用户接入5G网络，当系统内部以及周边网元异常导致流程失败时，AMF可以控制给终端下发的原因值以调整终端的行为，辅助运营商实现对现网设备兼容性控制的需求。

> **说明**
> 给UE下发的原因值变化可能会导致相应拒绝类话统指标的变化，并对终端行为产生影响，在部署前需要评估影响。关于不同原因值的含义及对终端行为的影响请参见协议3GPP TS 24.501。

## [操作步骤](#ZH-CN_OPI_0282813198)

1. 进入 “MML命令行-UNC” 窗口。
2. **设置5G移动性管理流程对UE的调整参数** 。
  **[SET NGMMPROCTRL](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM流程管理/5G移动性管理流程控制参数/设置5G移动性管理流程控制参数（SET NGMMPROCTRL）_09652386.md)**

## [任务示例](#ZH-CN_OPI_0282813198)

任务描述

在AUSF发现流程、鉴权流程中当流程失败时希望UE能够尝试接入4G网络，24小时后再重新尝试接入5G，AMF给终端下发#27原因值（N1 mode not allowed）。

脚本

// **设置5G移动性管理流程对UE的调整参数** 。

```
SET NGMMPROCTRL:PROT=AUSF_DISCOVERY, AUSFDISCNOFOUND=27, AUSFDISCOTHER=27;
SET NGMMPROCTRL:PROT=AUTHENTICATION, AUTHRESCHKFAIL=27, AUTHMACFAIL=27,AUTHSYNCHFAIL=27;
```
