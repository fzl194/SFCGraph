## [操作场景](#ZH-CN_OPI_0301125477)

部署对漫游用户进行区分计费，可以当在线漫游用户业务低峰期时启用该功能，如果在业务高峰期才启用该功能，GGSN/PGW-C不能立刻得到所有BGP路由表中的ASN值，影响对漫游用户的计费。

> **说明**
> 适用于 GGSN、 PGW-C。

## [对系统的影响](#ZH-CN_OPI_0301125477)

配置该功能后，会导致业务刚开始时各个漫游区用户激活速率较慢，因为对于业务开始时激活的各个漫游区用户，需要到BGP路由表查询对应的ASN值。对后续激活的同区漫游用户不会有影响，因为之前的用户激活的时候如果查询到对应的ASN值，GGSN/PGW-C会将其保存到GTP路径上，后续漫游用户激活的时候，GGSN/PGW-C直接到GTP路径上取用之前保存的ASN值。

另外，不要频繁修改SGSN/SGW-C与GGSN/PGW-C间信令接口的IP地址，否则会导致重新发布路由，影响用户激活速率。

## [必备事项](#ZH-CN_OPI_0301125477)

前提条件

- 请仔细阅读[WSFD-109006 ASN漫游计费](../WSFD-109006 ASN漫游计费_95924631.md)。

- 不同PLMN间采用eBGP路由组网。
- 完成加载license（如果有需求，请联系华为技术支持工程师处理）。

## [操作步骤](#ZH-CN_OPI_0301125477)

1. 打开本特性的License开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
2. 配置ASN功能开关属性。
  **[SET ASNFUNC](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/GTP-C接口配置管理/ASN功能管理/设置ASN功能（SET ASNFUNC）_35519283.md)**
  > **说明**
  > 设置同步ASN的周期，同步刷新是指从刷新时间开始，到周期结束，整个路径和隧道上的ASN被刷新一次，一般情况下，ASN不会发生变化，所以在配置时，这个时间不要设置太小，太小会影响性能。
3. 配置是否指定SGSN/SGW-C对应的缺省ASN值。
  **[SET DEFAULTASN](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/GTP-C接口配置管理/缺省ASN管理/设置缺省ASN值（SET DEFAULTASN）_37559375.md)**
4. 设置SGSN/SGW-C的信令地址和ASN映射的本地表项。
  **[ADD STATICASN](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/GTP-C接口配置管理/ASN静态配置/增加静态ASN（ADD STATICASN）_39850023.md)**

## [任务示例](#ZH-CN_OPI_0301125477)

任务描述

使能UNC的ASN漫游计费功能。

脚本

//打开本特性的License开关。

```
SET LICENSESWITCH: LICITEM="LKV3W9ASNR11", SWITCH=ENABLE;
```

//配置ASN功能开关属性。

```
SET ASNFUNC:ASNSWITCH=ENABLE,SYNCINTERVAL=5;
```

//配置删除指定SGSN/SGW-C对应的缺省ASN值。

```
SET DEFAULTASN:ASNVALUE=UNSPECIFIEDASN;
```

//设置SGSN/SGW-C的信令地址和ASN映射的本地表项。

```
ADD STATICASN: SRVNODEIP=10.13.13.0, MASK=255.255.255.0, ASN=1;
```
