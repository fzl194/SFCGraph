## [操作场景](#ZH-CN_OPI_0000001558578942)

本操作指导用于介绍AMF支持通过配置LMFId、用户切片信息选择LMF的激活过程。

> **说明**
> 适用于AMF。

## [必备事项](#ZH-CN_OPI_0000001558578942)

前提条件

- 请仔细阅读 [WSFD-106407 LMF选择功能特性概述](WSFD-106407 LMF选择功能特性概述_58419026.md)。
- 基于网络侧发起的5G位置定位服务业务才会触发AMF选择LMF，所以，如果激活AMF选择LMF，还需要先完成[激活网络侧发起的5G位置定位服务](../WSFD-106701 网络侧发起的5G位置定位服务/激活网络侧发起的5G位置定位服务_46574621.md)。

## [操作步骤](#ZH-CN_OPI_0000001558578942)

1. AMF上配置对接的LMF基础信息。
  **[ADD PNFPROFILE](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/服务化接口管理/注册与服务发现/本地NRF功能管理/对端NF实例概述信息管理/增加对端NF实例概述信息（ADD PNFPROFILE）_09653772.md)**
2. **可选：**AMF上配置LMF支持的LMFID信息。
  **[ADD PNFLMFINFO](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/服务化接口管理/注册与服务发现/本地NRF功能管理/对端LMF信息管理/增加对端LMF的信息（ADD PNFLMFINFO）_02870338.md)**
3. **可选：**AMF上配置LMF支持的用户切片信息。
  **[ADD PNFNS](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/服务化接口管理/注册与服务发现/本地NRF功能管理/对端NF实例切片信息管理/增加对端NF的切片信息（ADD PNFNS）_09652622.md)**
4. AMF上配置LMF选择功能开关。
  **[SET NGLCSPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/5G定位服务管理/定位服务管理/设置5G定位服务参数（SET NGLCSPARA）_44007975.md)**
5. **可选：**位置定位安全加固配置。
    a. 建议增加AMF的位置定位服务白名单配置，确保仅限于合法的GMLC访问。
      **[ADD ALLOWSERNAME](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/白名单管理/基于地址的白名单管理/增加基于地址的白名单（ADD ALLOWSERNAME）_83813626.md)**
    b. 建议AMF和GMLC侧开启SBI接口加密，降低网络安全风险。
          - AMF侧开启SBI接口加密请参考[激活SBI接口加密特性(CERT)](../../安全管理功能/WSFD-010308 SBI接口加密/激活SBI接口加密特性(CERT)_70185215.md)。
          - GMLC侧开启SBI接口加密请参考对应的产品描述。
    c. 建议增加定位服务权限，用于AMF收到GMLC的定位请求时检查GMLC的定位请求是否越权，对越权的GMLC，AMF拒绝其定位请求。（用于B2B场景）
      **[ADD LCSPERMITCFG](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/5G定位服务管理/定位服务权限配置管理/增加定位服务权限配置（ADD LCSPERMITCFG）_62595587.md)**

## [任务示例](#ZH-CN_OPI_0000001558578942)

任务描述

- 任务1：网络中配置AMF基于LMFID选择LMF，当AMF接收GMLC的位置服务定位请求中LMF标识为0001时，AMF选择LMF1进行业务；当AMF接收GMLC的位置服务定位请求中LMF标识为0000时，AMF选择LMF0进行业务。
- 任务2：在AMF上配置LMF支持的用户切片信息，并开启切片选择开关。

脚本

**任务1脚本**

//AMF上配置LMF1的基本信息。

```
ADD PNFPROFILE: NFINSTANCEID="LMF_Instance_1", NFTYPE=NfLMF, NFSTATUS=Registered, IPADDRESSTYPE=IPTypeV4, IPV4ADDRESS1="10.18.0.2", PRIORITY=100;
```

//AMF上配置LMF0的基本信息。

```
ADD PNFPROFILE: NFINSTANCEID="LMF_Instance_0", NFTYPE=NfLMF, NFSTATUS=Registered, IPADDRESSTYPE=IPTypeV4, IPV4ADDRESS1="10.18.0.5", PRIORITY=1;
```

//AMF上配置LMF1、LMF0支持的LMFID信息。

```
ADD PNFLMFINFO: NFINSTANCEID="LMF_Instance_1", LMFID="0001";
ADD PNFLMFINFO: NFINSTANCEID="LMF_Instance_0", LMFID="0000";
```

//AMF上开启园区定位增强开关。

```
SET NGLCSPARA: CAMPLCSEN=ON;
```

**//设置位置定位安全加固配置。**

//增加AMF的位置定位服务白名单配置，确保仅限于合法的GMLC访问。

```
ADD ALLOWSERNAME: INDEX=0, NFINSTANCEID="AMF_Instance_0", SERVICENAME="namf-loc", LOCALADDR="10.16.1.5", PEERADDR="10.112.1.5";
```

//开启SBI接口加密，降低网络安全风险。

```
AMF侧请参考
激活SBI接口加密特性(CERT)
AMF侧配置脚本。
GMLC侧请参考对应产品描述。
```

//增加定位服务权限，用于AMF收到GMLC的定位请求时检查GMLC的定位请求是否越权，对越权的GMLC，AMF拒绝其定位请求。（用于B2B场景）

```
ADD LCSPERMITCFG: OPTYPE=Namf_Location_ProvidePositioningInfo, IPTYPE=IPTypeV4, IPV4ADDRESS="10.112.1.5", USRGRPID=20;
```

**任务2脚本**

//AMF上配置LMF的基本信息。

```
ADD PNFPROFILE: NFINSTANCEID="LMF_Instance_1", NFTYPE=NfLMF, NFSTATUS=Registered, IPADDRESSTYPE=IPTypeV4, IPV4ADDRESS1="10.18.0.2", PRIORITY=100;
```

//AMF上配置LMF支持的用户切片信息。

```
ADD PNFNS: INDEX=1,NFINSTANCEID="LMF_Instance_1", SST=1, SD="010101";
```

//AMF上开启切片选择开关。

```
SET NGLCSPARA: NSSW=YES;
```

**//设置位置定位安全加固配置。**

//增加AMF的位置定位服务白名单配置，确保仅限于合法的GMLC访问。

```
ADD ALLOWSERNAME: INDEX=0, NFINSTANCEID="AMF_Instance_0", SERVICENAME="namf-loc", LOCALADDR="10.16.1.5", PEERADDR="10.112.1.5";
```

//开启SBI接口加密，降低网络安全风险。

```
AMF侧请参考
激活SBI接口加密特性(CERT)
AMF侧配置脚本。
GMLC侧请参考对应产品描述。
```

//增加定位服务权限，用于AMF收到GMLC的定位请求时检查GMLC的定位请求是否越权，对越权的GMLC，AMF拒绝其定位请求。（用于B2B场景）

```
ADD LCSPERMITCFG: OPTYPE=Namf_Location_ProvidePositioningInfo, IPTYPE=IPTypeV4, IPV4ADDRESS="10.112.1.5", USRGRPID=20;
```
