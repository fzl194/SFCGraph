#### [适用NF](#ZH-CN_CONCEPT_0188625066)

MME

#### [定义](#ZH-CN_CONCEPT_0188625066)

LTE UE信令控制是指在终端反复发送Attach Request、PDN Connectivity Request、Service Request、Control Plane Service Request消息的场景下，当某一消息达到设定的阈值时， UNC 可以根据已配置的基于终端类型、信令流程的控制策略对UE的异常行为进行抑制，有效地减少网络中的异常信令，减轻由于UE的异常行为对网络侧的影响，提升网络可用性和可靠性。

#### [客户价值](#ZH-CN_CONCEPT_0188625066)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 减少网络中的异常信令。<br>- 提高网络业务流程成功率。<br>- 避免受控信令导致网络拥塞和设备过载。<br>- 提升网络可用性和可靠性。 |
| 用户 | 无。 |

#### [应用场景](#ZH-CN_CONCEPT_0188625066)

LTE网络中，在终端反复发送Attach Request、PDN Connectivity Request、Service Request、Control Plane Service Request，产生大量异常信令的情况下，利用异常信令控制功能可以减少网络的信令负荷。

UE反复发送Attach Request消息原因有很多，主要可以分两类：用户原因和网络原因。

- 用户原因有用户未签约、用户非法接入、用户欠费停机、用户漫游受限等。
- 网络原因有HSS/DNS/S-GW/P-GW/PCRF/AAA/OCS等设备短暂故障或拥塞，由于网元故障导致的频繁接入。

UE反复发送PDN Connectivity Request消息原因主要有：

- 用户原因（如用户欠费）导致承载建立失败。
- 周边网元故障（S-GW/P-GW/PCRF等）导致承载建立失败。
- 上层应用业务故障，导致的终端自动反复承载建立成功/去活。

UE Service Request、Control Plane Service Request消息激增的原因是智能终端的快速休眠和永久在线。

#### [对系统的影响](#ZH-CN_CONCEPT_0188625066)

- 当抑制措施设置为特定原因值拒绝时，如果原因值设置不当，可能会导致终端后续不再接入。
- Parking APN唤醒措施设置为网络侧分离方式时，唤醒会影响用户其他正常APN的业务，导致正在使用的业务中断。

#### [应用限制](#ZH-CN_CONCEPT_0188625066)

- 只有对端S-GW/P-GW支持时，才能使用Parking APN激活功能。
- 对于附着流程，与承载无关的失败场景（如鉴权失败，用户未开通4G业务导致失败），UNC不会对4G用户启用“Parking APN激活”信令抑制方式。
- 4G用户附着和服务请求异常抑制流程不支持Backoff Timer（Backoff Timer为UE和网络启动的定时器，在该定时器时间内，UE不允许发起PDN连接建立流程）。
- 系统唤醒附着流程使用Parking APN建立的缺省承载时，将同一UE所有使用Parking APN建立的承载同时唤醒；系统唤醒PDN连接流程使用Parking APN建立的承载时，将同一APN粒度并且使用Parking APN建立的承载全部唤醒。
- 对高优先级业务引发的Attach Request、Service Request、Control Plane Service Request或寻呼响应引发的Service Request、Control Plane Service Request消息不进行信令控制。
- 对紧急用户不进行信令抑制。
- 当存在GUL互操作时，当前信令抑制状态会失效，系统会重新对异常信令进行抑制判断。

#### [原理概述](#ZH-CN_CONCEPT_0188625066)

UNC 记录一个周期内终端请求的信令次数，当测量周期内信令数目达到阈值时（异常识别）， UNC 可以根据已配置的基于终端类型、信令流程的控制策略对UE的异常信令行为进行控制，实现对以下四种信令的控制（包括抑制措施和唤醒流程）：

- Attach Request
- PDN Connectivity Request
- Service Request
- Control Plane Service Request

> **说明**
> 本特性仅对以上四种信令异常的终端进行抑制，不影响其他正常终端的业务。
