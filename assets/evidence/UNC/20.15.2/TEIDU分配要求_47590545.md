# TEIDU分配要求

TEIDU（Tunnel Endpoint ID for user plane），全称隧道端点标识符，是UPF数据传输隧道的唯一标识，RAN侧或其他UPF需要根据目的UPF提供的数据面IP和TEIDU一起确认用户数据发送的目的地址。TEIDU由TEID和用户面逻辑接口（Sa/Sc/Pa）IP地址组成，以此唯一标识用户会话的核心网隧道信息CN Tunnel，如 [图1](#ZH-CN_TOPIC_0000001147590545__fig737372392913) 所示。

TEIDU属于用户面资源，要求同一UPF上TEIDU不可重叠，SMF上无此限制。非Full Mesh组网下，TEIDU和UE IP都由SMF网元分配。Full Mesh组网下，一个UPF会同时对接多个SMF网元：

TEIDU如果仍由SMF分配，则不同SMF分配给同一UPF的TEIDU可能重复，导致用户业务异常。因此Full Mesh组网场景下，推荐使用UPF分配TEIDU，保证用户面隧道信息CN Tunnel标识唯一，即使同一个SMF上出现TEIDU重叠，也不会影响业务。

- UDG上默认配置为由UPF分配TEIDU，只要UPF上配置为用户面分配TEIDU，则不允许SMF分配TEIDU。
- 系统开工时，UPF将分配TEIDU能力上报给SMF。如果系统开工后，UPF修改了分配TEIDU能力，则UPF根据记录的SMF列表，向每个SMF发起Association Update消息，SMF收到更新消息后记录UPF新修改的能力。
- 修改UPF的能力必须在UPF无用户时才可进行。如果UPF上存在用户，必须先锁定UPF，然后去活UPF上所有用户，当UPF无用户时才可进行，不允许有用户时修改UPF能力。
