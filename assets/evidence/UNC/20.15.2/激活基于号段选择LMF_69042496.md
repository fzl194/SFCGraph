## [任务示例](#ZH-CN_OPI_0000001269042496)

任务描述

网络中规划支持所有SUPI号段的LMF0为缺省LMF；规划LMF1支持SUPI前缀为12345~12346的号段。

配置AMF基于SUPI号段选择LMF，当SUPI前缀为12345~12346号段用户进行位置定位业务时，AMF选择LMF1进行业务；当SUPI前缀为12345~12346号段范围外的用户进行位置定位业务时，AMF选择LMF0进行业务。

脚本

//开启对端LMF的SUPI配置通配功能，如果AMF没有配置对端LMF支持的SUPI号段，则代表此LMF支持所有SUPI号段。

```
ADD PNFWILDCARD: NFTYPE=NfLMF, SUPICFGSWITCH=ON;
```

//AMF上配置缺省LMF0的基本信息。

```
ADD PNFPROFILE: NFINSTANCEID="LMF_Instance_0", NFTYPE=NfLMF, NFSTATUS=Registered, IPADDRESSTYPE=IPTypeV4, IPV4ADDRESS1="10.18.0.2", PRIORITY=100;
```

//AMF上配置LMF1的基本信息。

```
ADD PNFPROFILE: NFINSTANCEID="LMF_Instance_1", NFTYPE=NfLMF, NFSTATUS=Registered, IPADDRESSTYPE=IPTypeV4, IPV4ADDRESS1="10.18.0.5", PRIORITY=1;
```

//AMF上配置LMF1支持的SUPI号段信息。

```
ADD PNFSUPI: INDEX=1,NFINSTANCEID="LMF_Instance_1", QUERYTYPE=START_END, RANGESTART="12345",RANGEEND="12346";
```

//AMF上开启基于SUPI号段选择LMF。

```
SET NGLCSPARA: SUPISW=YES, GPSISW=NO;
```
