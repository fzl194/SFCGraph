# 调测SGSN的黑白名单

- [操作场景](#ZH-CN_OPI_0276495252__1.3.1)
- [必备事项](#ZH-CN_OPI_0276495252__1.3.2)
- [操作步骤](#ZH-CN_OPI_0276495252__1.3.3)

## [操作场景](#ZH-CN_OPI_0276495252)

当运营商需要部署SGSN的黑白名单时，需要对本功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于SGW-C、PGW-C、SMF。

## [必备事项](#ZH-CN_OPI_0276495252)

前提条件

- 请仔细阅读[WSFD-104509 SGSN的黑白名单特性概述](特性概述_76495250.md)。
- 完成[激活SGSN的黑白名单](激活SGSN的黑白名单_76495251.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息 | 国际移动用户识别码（IMSI） | 123000123456789 | 测试终端自带 | - |
| MME信息 | MME IP地址 | 10.20.0.3<br>10.200.0.2 | - | - |

工具

该操作无需准备其他工具。

## [操作步骤](#ZH-CN_OPI_0276495252)

1. 终端携带MME地址10.20.0.3接入网络。10.20.0.3在SGSN/MME/SGW-C的黑名单地址段内。
  预期结果：终端激活不成功。
2. 终端携带MME地址10.200.0.2接入网络。10.200.0.2不在SGSN/MME/SGW-C的黑名单地址段内。
  预期结果：终端激活成功。
  **[DSP PDUSESSION](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/查询PDP上下文信息/显示PDU会话（DSP PDUSESSION）_09653215.md)**
  预期结果：PDU会话信息中的CP的PFCP IPv4地址是不在SGSN/MME/SGW-C黑名单中的10.200.0.2。

  ```
  DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=DETAILED, WLNETWKTYPE=NW2G3G4G, PDUSESSIONID=5, QFI=1;
  RETCODE = 0  操作成功  

  结果如下 
  --------------  
              国际移动用户标识 = 123000123456789
                              ...
             CP的PFCP IPv4地址 = 10.200.0.2
                              ...
 
   (结果个数 = 1) 
  ---    END
  ```
