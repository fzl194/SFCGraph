# 实现原理

MME根据eNodeB上报的推荐寻呼的eNodeB进行寻呼优化，并在寻呼消息中将eNodeB上报的推荐寻呼的小区和小区覆盖等级通过寻呼辅助信息传给eNodeB，用于eNodeB进行寻呼。

实现覆盖等级寻呼包含如下步骤：

- eNodeB在S1连接释放时在UE Context Release Complete消息中携带覆盖等级寻呼辅助信息，具体包括Information on Recommended Cells and eNBs for Paging和Cell Identifier and Coverage Enhancement Level这两个可选信元。
  其中Information on Recommended Cells and eNBs for Paging又包含了Recommended Cells for Paging和Recommended eNBs for Paging这两个必选信元，即推荐小区列表和推荐eNodeB列表。
- MME收到eNodeB带上来的寻呼辅助信息有覆盖等级（即消息中携带Cell Identifier and Coverage Enhancement Level信元）后，将寻呼辅助信息保存在用户上下文中，将这些eNodeB标记为具有覆盖增强支持能力的eNodeB。
- MME在寻呼时，根据MME的配置“eNodeB覆盖增强检查开关”决定MME是否根据eNodeB的覆盖增强支持能力来决定在寻呼消息中携带呼辅助信息（AssistanceData for Paging）信元。
    - 当[**SET M2MCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/M2M控制参数/设置M2M控制参数(SET M2MCTRL)_72345369.md)命令的“eNodeB覆盖增强检查开关”参数为“ON”， MME在寻呼请求消息中将推荐寻呼的小区和小区覆盖等级通过寻呼辅助信息（Assistance Data for Paging）信元携带给有覆盖增强支持能力的待寻呼的eNodeB。
    - 当[**SET M2MCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/M2M控制参数/设置M2M控制参数(SET M2MCTRL)_72345369.md)命令的“eNodeB覆盖增强检查开关”参数为“OFF”，MME寻呼请求消息中将推荐寻呼的小区和小区覆盖等级通过寻呼辅助信息（Assistance Data for Paging）信元带给所有待寻呼的eNodeB，不考虑eNodeB的覆盖增强支持能力。
  > **说明**
  > - 通过命令[**DSP ENBATTR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/eNodeB属性信息管理/显示eNodeB属性(DSP ENBATTR)_72345861.md)可以查看eNodeB的覆盖增强寻呼能力指示，通过命令[**CLR ENBATTR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/eNodeB属性信息管理/清除eNodeB属性(CLR ENBATTR)_26146262.md)可以清除MME上eNodeB的覆盖增强寻呼能力指示。
  > - 寻呼辅助信息（Assistance Data for Paging）是否携带子信元Next Paging Area Scope受[**SET S1CMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1接口兼容性/设置S1接口兼容性(SET S1CMPT)_72345837.md)命令中的参数“Next Paging Area Scope开关”控制。
  > - 当用户上下文中存在推荐寻呼的eNodeB，且系统同时部署了**[WSFD-206001 精准寻呼](../../流控功能/WSFD-206001 精准寻呼_64557971.md)**，[**ADD S1PAGINGRULE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼规则管理/增加S1寻呼规则(ADD S1PAGINGRULE)_26306058.md)的参数“寻呼动作组合”配置包含“NEIGH_ENODEB（邻接eNodeB）”时，MME在寻呼邻接eNodeB时按照eNodeB上报的推荐寻呼的eNodeB列表进行寻呼，不使用学习到的eNodeB邻接关系和通过[**ADD ENBNSINDB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/eNodeB邻接关系测试/增加eNodeB邻接关系(ADD ENBNSINDB)_72345859.md)配置的eNodeB邻接关系。

**覆盖等级寻呼处理流程**

以CP-CIoT数传方式下举例介绍覆盖等级寻呼的处理流程。用户首次接入到eNodeB1下，随后移动到eNodeB2下，如 [图1](#ZH-CN_TOPIC_0276928325__fig547917541410) 所示。假设 MME 配置最多寻呼3次。

> **说明**
> - 最大寻呼次数为最大寻呼重发次数加1，最大寻呼重发次数为[**SET NBEMM**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/NB-MM协议参数管理/设置NB-S1模式MM协议参数（SET NBEMM）_26305584.md)的参数“N3413”的取值，配置最大寻呼重发次数为2，即可得到最大寻呼次数为3。
> - MME在Paging消息中Assistance Data for Paging的子信元Intended Number of Paging Attempts即MME下发寻呼的最大次数，MME下发寻呼达到最大次数而仍然没有得到UE响应时，认为寻呼失败。

![](实现原理_76928325.assets/zh-cn_image_0276928583_2.png)

<br>

1. UE先通过eNodeB1接入系统，覆盖等级为2。
2. 释放连接时eNodeB1通过UE Context Release Complete消息，将推荐寻呼的小区（TA1）和推荐的eNodeB1以及小区覆盖等级（级别为2）带给MME。
  > **说明**
  > UP-CIoT方式下，eNodeB1通过UE Context Suspend Request消息将推荐寻呼的小区和eNodeB以及小区覆盖等级带给MME。
3. UE移动到eNodeB2覆盖的区域范围内，该区域的覆盖等级为3。
4. 下行数据触发寻呼，首次寻呼时，MME下发的Paging Attempt Count为1，寻呼总次数为3，eNodeB1和eNodeB2使用上一次接入的覆盖等级2进行寻呼，寻呼失败。
5. MME寻呼失败后，会调整下发的Paging Attempt Count为2，继续给eNodeB发送寻呼请求，eNodeB1和eNodeB2根据eNodeB上的策略，调整寻呼的覆盖等级继续寻呼UE，当刚好调整寻呼等级为3时，寻呼成功。
6. 寻呼成功，UE发起Control Plane Service Request流程。
  > **说明**
  > UP-CIoT方式下，寻呼成功，UE发起的是Connection Resume流程。
