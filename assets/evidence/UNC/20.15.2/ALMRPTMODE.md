#### [命令功能](#ZH-CN_MMLREF_0000001126146204)

**适用网元：SGSN、MME**

该命令用于设置告警的上报模式。如果系统内某一个告警大量产生时，会对系统的性能和告警台产生冲击，此时可以根据实际的需求，配置告警的上报模式，可以有效降低大量告警对系统的影响。

#### [注意事项](#ZH-CN_MMLREF_0000001126146204)

- 系统初次上电运行时，会执行系统初始设置值。
- “批量统计周期(s)”的设置会引起告警的延迟上报，推荐值为60秒，在实际配置时不建议配置过长。

#### [本地用户权限](#ZH-CN_MMLREF_0000001126146204)

manage-ug；system-ug

#### [网管用户权限](#ZH-CN_MMLREF_0000001126146204)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_MMLREF_0000001126146204)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| ALMTYPE | 告警类型 | 可选必选说明：必选参数<br>参数含义：该参数用于指定告警类型。<br>数据来源：本端规划<br>取值范围：<br>- “S1apLinkDown(S1ap链路故障)”：**ALM-80589 S1ap链路故障**，对应的批量告警为**ALM-14005 批量S1ap链路故障**。<br>系统初始设置值：无<br>配置原则：根据实际需要配置。 |
| BATCHEDALMSWITCH | 批量告警上报开关 | 可选必选说明：分支可选参数<br>参数含义：该参数用于控制在大量告警产生时是否上报批量告警。<br>前提条件：该参数在<br>“告警类型”<br>参数配置后生效。<br>数据来源：本端规划<br>取值范围：<br>- “OFF(关)”<br>- “ON(开)”<br>系统初始设置值：<br>“ON(开)”<br>说明：- 当该参数设置为“OFF(关)”时，需要等到本次“批量统计周期(s)”结束，才会恢复批量告警。<br>- 当该参数设置为“ON(开)”时，批量告警功能立即生效。 |
| PERIOD | 批量统计周期（s） | 可选必选说明：分支可选参数<br>参数含义：该参数用于指定批量告警的统计周期。<br>前提条件：该参数在<br>“批量告警上报开关”<br>参数配置为<br>“ON(开)”<br>后生效。<br>数据来源：本端规划<br>取值范围：60s～300s<br>系统初始设置值：60s<br>配置原则：一般不建议修改。<br>说明：该参数和<br>“批量统计门限”<br>配合使用，组合后的含义是在某个时间周期内产生N个故障告警，如果N大于等于门限，则产生批量告警，否则按照正常告警上报。 |
| THRESHOLD | 批量统计门限 | 可选必选说明：分支可选参数<br>参数含义：该参数用于指定批量告警的统计门限。<br>前提条件：该参数在<br>“批量告警上报开关”<br>参数配置为<br>“ON(开)”<br>后生效。<br>数据来源：本端规划<br>取值范围：10～1000<br>系统初始设置值：100<br>配置原则：一般不建议修改。<br>说明：该参数和<br>“批量统计周期(s)”<br>配合使用，组合后的含义是在某个时间周期内产生N个故障告警，如果N大于等于门限则产生批量告警，否则按照正常告警上报。 |
| ORIENTALMSWITCH | 原始告警上报开关 | 可选必选说明：分支可选参数<br>参数含义：该参数用于控制批量告警上报时单条详细告警上报的开关。当该参数为<br>“ON(开)”<br>时，批量告警包含的单条告警也会上报，反之当该参数为<br>“OFF(关)”<br>时，批量告警包含的单条告警不上报。<br>前提条件：该参数在<br>“批量告警上报开关”<br>参数配置为<br>“ON(开)”<br>后生效。<br>数据来源：本端规划<br>取值范围：<br>- “OFF(关)”<br>- “ON(开)”<br>系统初始设置值：<br>“OFF(关)”<br>说明：该参数设置后，需要等下次<br>“批量统计周期(s)”<br>才能生效。 |
| CLEARTHRESHOLD | 批量恢复门限 | 可选必选说明：分支可选参数<br>参数含义：该参数用于设置系统批量告警“<br>**ALM-14005 批量S1ap链路故障**<br>”的恢复门限。<br>前提条件：该参数在<br>“批量告警上报开关”<br>参数配置为<br>“ON(开)”<br>后生效。<br>数据来源：本端规划<br>取值范围：0～1000<br>系统初始设置值：0<br>说明：该参数和<br>“批量统计门限”<br>配合使用，组合后的含义是在某个时间周期内当批量告警“<br>**ALM-14005 批量S1ap链路故障**<br>”中未恢复的原始告警“<br>**ALM-80589 S1ap链路故障**<br>”个数小于或等于恢复门限时，将批量告警“<br>**ALM-14005 批量S1ap链路故障**<br>”转为多条详细告警“<br>**ALM-80589 S1ap链路故障**<br>”进行上报，其中实际恢复门限为该参数和<br>“批量统计门限”<br>乘以“批量恢复百分比(%)”取值的最小值。转换后的告警上报时间为告警转换的时间，而不是原始告警上报时间。 |
| CLEARPERCENT | 批量恢复百分比(%) | 可选必选说明：分支可选参数<br>参数含义：该参数用于设置系统批量告警“ALM-14005 批量S1ap链路故障”的恢复门限。<br>前提条件：该参数在“批量告警上报开关”参数配置为“ON(开)”后生效。<br>数据来源：本端规划<br>取值范围：10~90，单位是百分比<br>系统初始设置值：90<br>说明：该参数和<br>“批量统计门限”<br>配合使用，组合后的含义是在某个时间周期内当批量告警“<br>**ALM-14005 批量S1ap链路故障**<br>”中未恢复的原始告警“<br>**ALM-80589 S1ap链路故障**<br>”个数小于或等于恢复门限时，将批量告警“<br>**ALM-14005 批量S1ap链路故障**<br>”转为多条详细告警“<br>**ALM-80589 S1ap链路故障**<br>”进行上报，其中实际恢复门限为“批量恢复门限”参数和<br>“批量统计门限”<br>乘以“批量恢复百分比(%)”取值的最小值。转换后的告警上报时间为告警转换的时间，而不是原始告警上报时间。 |

#### [使用实例](#ZH-CN_MMLREF_0000001126146204)

设置系统中，S1ap链路故障的批量告警上报开关为开，批量统计周期(s)为60，批量统计门限为100，原始告警上报开关为关：

SET ALMRPTMODE: ALMTYPE=S1apLinkDown, BATCHEDALMSWITCH=ON, PERIOD=60, THRESHOLD=100, ORIENTALMSWITCH=OFF;
