# 调测下行数据通知信令抑制功能（MME）

- [操作场景](#ZH-CN_OPI_0168057586__1.3.1)
- [必备事项](#ZH-CN_OPI_0168057586__1.3.2)
- [操作步骤](#ZH-CN_OPI_0168057586__1.3.3)

## [操作场景](#ZH-CN_OPI_0168057586)

CPU过载或者Paging寻呼消息过多的场景下，当Downlink Data Notification消息的丢弃率达到设定的阈值时， UNC 通过在Downlink Data Notification Acknowledge消息中携带DL low priority traffic Throttling信元通知SGW-C抑制低优先级的Downlink Data Notification消息，以达到保护和提高高优先级业务被叫成功率的目的。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0168057586)

前提条件

- 请仔细阅读[WSFD-010803 下行数据通知信令抑制特性概述](特性概述_62796711.md)。
- 完成[激活下行数据通知信令抑制功能](激活下行数据通知信令抑制功能（MME）_65074086.md)。

数据

该操作无需准备数据。

工具

该操作无需准备其他工具。

## [操作步骤](#ZH-CN_OPI_0168057586)

1. UNC 上CPU过载或者Paging寻呼消息过多的场景下，当Downlink Data Notification消息的丢弃率达到设定的阈值时，检查下行数据通知信令抑制是否已经进入抑制状态：
  进入 “MML命令行-UNC” 窗口。 执行命令 **[**DSP DDNTHROTPARA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/DDN信令抑制参数管理/显示DDN信令抑制状态(DSP DDNTHROTPARA)_26146172.md)** :
  ```
  %%DSP DDNTHROTPARA:;%%
  RETCODE = 0 操作成功

  输出结果如下：
  ---------
  抑制状态 = 已启动
  抑制比例（%） = 80
  抑制时长 = 30
  抑制时长单位 = 2秒
  (结果个数 = 1)

  ---    END
  ```
    - 预期结果： **抑制状态** 为 “已启动” 。
      > **说明**
      > 此时，会上报“ALM-80678 通知S-GW启动DDN信令抑制”告警。
    - 异常结果： **抑制状态** 为 “未启动” 。
