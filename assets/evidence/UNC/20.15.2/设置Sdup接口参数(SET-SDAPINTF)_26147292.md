#### [命令功能](#ZH-CN_MMLREF_0000001126147292)

**适用网元：MME**

本命令用于控制Sdup接口的管理参数。

#### [注意事项](#ZH-CN_MMLREF_0000001126147292)

- 该命令执行后立即生效。
- “Sdup接口探测间隔时长”不应小于链路管理类消息的T-Response和N-Request数值配置，参数见[**SET SDAPRETRSMT**](../Sdup消息重传参数管理/设置Sdup消息重传参数(SET SDAPRETRSMT)_26307104.md)（T-Response*N-Request）。
- 该命令执行后，“Sdup链路告警阈值”对新产生的告警生效。

#### [本地用户权限](#ZH-CN_MMLREF_0000001126147292)

manage-ug；system-ug

#### [网管用户权限](#ZH-CN_MMLREF_0000001126147292)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_MMLREF_0000001126147292)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| LPORT | Sdup接口侦听端口号 | 可选必选说明：可选参数<br>参数含义：本参数用于指定Sdup接口的侦听端口号。<br>接收端只处理侦听端口一致时的Sdup接口消息，端口不一致的消息将被接收端直接丢弃。<br>数据来源：全网规划<br>取值范围：20001~65534<br>系统初始设置值：29274<br>配置原则：Pool内所有MCR_VNFC配置的Sdup接口侦听端口号必须相同。 |
| SUDPCHK | Sdup接口UDP校验功能 | 可选必选说明：可选参数<br>参数含义：本参数用于控制Sdup接口的UDP层启用或关闭校验功能。<br>数据来源：本端规划<br>取值范围：<br>- “OFF(关闭)”<br>- “ON(开启)”<br>配置原则：当需要提升Sdup接口消息的传输可靠性，建议将参数设为ON。同一Pool内所有MCR_VNFC配置的Sdup接口UDP校验功能开关必须相同。<br>系统初始设置值：ON<br>说明：- 开关打开时，支持UDP校验功能，此时发送Sdup报文需要填写UDP校验和，接收Sdup报文需要检查UDP校验和，并且校验失败的报文将被丢弃。<br>- 开关关闭时，不支持UDP校验功能，此时发送Sdup报文不需要填写UDP校验和，接收Sdup报文需要检查UDP校验和，并且校验失败的报文将会继续接收，不会被丢弃。 |
| DETFUNC | Sdup接口探测功能 | 可选必选说明：可选参数<br>参数含义：本参数用于控制Sdup接口探测功能的启用或关闭。<br>数据来源：本端规划<br>取值范围：<br>- “OFF(关闭)”<br>- “ON(开启)”<br>系统初始设置值：ON<br>说明：- Sdup接口探测功能关闭后，系统无法检测链路故障，可能会造成部分容灾流程失败。<br>- Sdup接口探测功能启用后，系统将以“Sdup接口探测间隔时长”为间隔进行Sdup链路的状态检测。 |
| DETITVL | Sdup接口探测间隔时长 | 可选必选说明：可选参数<br>参数含义：本参数用于指定Sdup接口探测的时间间隔<br>数据来源：本端规划<br>取值范围：30s~300s<br>系统初始设置值：60 |
| ALMTHRSHD | Sdup链路告警阈值 | 可选必选说明：可选参数<br>参数含义：本参数用于指定Sdup链路探测失败后上报告警的阈值。当Sdup链路检测连续X个（X=告警阈值）周期探测无响应（或者收到失败响应），上报Sdup链路故障告警。<br>数据来源：本端规划<br>取值范围：1~20<br>系统初始设置值：5 |
| SRECOVERY | SDAP重启计数功能 | 可选必选说明：可选参数<br>参数含义：每个SDAP实体维护一个自身的重启计数器，当SDAP实体重启后，本参数控制SDAP实体的重启计数是否更新。<br>数据来源：本端规划<br>取值范围：<br>- “OFF(关闭)”:本端SDAP实体重启后，本端重启计数不会更新。<br>- “ON(开启)”：本端SDAP实体重启后，本端重启计数基于历史值+1。<br>系统初始设置值：ON<br>说明：- 每个SDAP实体的重启计数通过信令消息传递给对端SDAP实体，同时每个SDAP实体也维护每个对端SDAP实体的重启计数值。当发现对端SDAP实体的重启计数值发生变化时，会判断为对端SDAP实体重启，本端SDAP实体将通知业务层重新备份用户信息。<br>- 若本参数设置为“关闭”，SDAP实体重启后，自身的重启计数器不变化。对端SDAP实体也无法感知到某个SDAP实体的重启，用户信息的重新备份不及时，可能造成容灾流程的部分失败。 |
| IPPLCY | IP地址策略 | 可选必选说明：可选参数<br>参数含义：该参数指定Sdup接口IP地址选择策略。<br>数据来源：本端规划<br>取值范围：<br>- “IPV4”（仅使用IPv4地址）<br>- “IPV6”（仅使用IPv6地址）<br>- “IPV6PRE”（优先使用IPv6地址）<br>系统初始设置值：<br>“IPv4”<br>配置原则：<br>- 在IPv4网络中，配置成“IPV4(仅使用IPv4地址)”；在IPv4向IPv6改造过程中，如果Sdup接口本端和对端之间IPv6改造没有完成，配置成“IPV4(仅使用IPv4地址)”。<br>- 在IPv4向IPv6改造过程中，如果Sdup接口本端和对端之间IPv6改造完成，配置成“IPV6PRE(优先使用IPv6地址)”。<br>- IPv6网络改造完成后，配置成“IPV6(仅使用IPv6地址)”。 |

#### [使用实例](#ZH-CN_MMLREF_0000001126147292)

当现网需要启用“WSFD-201201 MME链式备份”特性时，本命令用于设置Sdup接口的管理参数，可执行此命令：

SET SDAPINTF: LPORT=20036, SUDPCHK=ON, DETFUNC=OFF, DETITVL=30, ALMTHRSHD=1, SRECOVERY=ON, IPPLCY=IPV4;
