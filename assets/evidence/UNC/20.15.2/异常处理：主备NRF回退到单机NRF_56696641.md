## [操作场景](#ZH-CN_OPI_0000001756696641)

单机改造主备后，主备NRF不能正常处理业务，也无法规避故障问题，希望通过回退到原单机的场景来保证NRF可以继续提供正常业务。

本操作指导介绍在运行网络中将现网原有单机改造后的主备NRF，回退为单机NRF的操作过程。

> **说明**
> 适用于NRF。

## [必备事项](#ZH-CN_OPI_0000001756696641)

前提条件

- 已完成[激活NRF主备容灾特性](../激活NRF主备容灾特性_56656841.md)。
- 已按照自身组网完成两个NRF之间的路由配置，对于IPv4组网，各配置场景请参考“初始配置与调测”中自动部署 [配置动态路由OSPF+BFD组网（IPv4）](../../../../../初始配置/UNC初始配置与调测/组网路由配置/配置VNF侧IP路由数据（非SDN）/自动部署（推荐）/配置动态路由OSPF+BFD组网（IPv4）_75096807.md) 、 [配置静态路由+BFD组网（IPv4）](../../../../../初始配置/UNC初始配置与调测/组网路由配置/配置VNF侧IP路由数据（非SDN）/自动部署（推荐）/配置静态路由+BFD组网（IPv4）_75096861.md) 、 [配置BGP over OSPF/静态路由+BFD（IPv4）](../../../../../初始配置/UNC初始配置与调测/组网路由配置/配置VNF侧IP路由数据（非SDN）/自动部署（推荐）/配置BGP over OSPF_静态路由+BFD（IPv4）_75096835.md) 或手动部署 [配置动态路由OSPF+BFD组网（IPv4）](../../../../../初始配置/UNC初始配置与调测/组网路由配置/配置VNF侧IP路由数据（非SDN）/手动部署/配置动态路由OSPF+BFD组网（IPv4）_75096780.md) 、 [配置静态路由+BFD组网（IPv4）](../../../../../初始配置/UNC初始配置与调测/组网路由配置/配置VNF侧IP路由数据（非SDN）/手动部署/配置静态路由+BFD组网（IPv4）_75096769.md) 、 [配置BGP over OSPF/静态路由+BFD（IPv4）](../../../../../初始配置/UNC初始配置与调测/组网路由配置/配置VNF侧IP路由数据（非SDN）/手动部署/配置BGP over OSPF_静态路由+BFD（IPv4）_75096771.md) 。

## [操作步骤](#ZH-CN_OPI_0000001756696641)

1. NRF1和NRF2上分别执行解除NRF的主备容灾配置。
    a. 进入 “MML命令行-UNC” 窗口。
    b. 关闭主备NRF容灾特性的License配置开关。
      **[SET LICENSESWITCH](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)**
    c. 删除主备NRF容灾实例通信地址信息。
      **[RMV DRCOMM](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/容灾管理/删除容灾实例地址(RMV DRCOMM)_51012926.md)**
    d. 删除主备容灾实例。
      **[RMV DRINST](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/容灾管理/删除容灾实例(RMV DRINST)_51012923.md)**
    e. 查询容灾配置信息，确保容灾配置信息已删除。
      **[DSP DRINFO](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/容灾管理/查询容灾信息(DSP DRINFO)_51012929.md)**
      > **说明**
      > 查询结果中，如果 “数据备份状态” 为 “容灾配置表已删除” ，表示容灾配置信息已删除；如果 “数据备份状态” 为 “ 容灾配置表仍然存在” ，表示容灾配置信息正在删除，请1分钟后再次执行此命令，如果连续5次查询都是配置信息正在删除，说明系统异常，请联系华为技术支持。

## [任务示例](#ZH-CN_OPI_0000001756696641)

任务描述

- 现网2个单机运行的NRF组成主备容灾组，NRF信息如下：
  NRF1：容灾实例标识为0，用于数据同步的通信IP地址为10.5.5.1。
  NRF2：容灾实例标识为1，用于数据同步的通信IP地址为10.5.5.2。
- **现在需要将改造后的主备NRF回退到2个单机NRF时，需要执行如下脚本。**

脚本

**//配置NRF1。**

//关闭主备NRF容灾特性的License配置开关。

```
SET LICENSESWITCH: LICITEM=" LKV2ASNRR01", SWITCH=DISABLE;
```

//删除主备NRF容灾实例通信地址信息。

```
RMV DRCOMM: DRINSTID=
1
, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.2";
RMV DRCOMM: DRINSTID=
0
, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.1";
```

//删除主备容灾实例。

```
RMV DRINST: DRINSTID=
0
; 
```

//查询容灾配置信息，确保容灾配置信息已删除。如果 “数据备份状态” 为 “容灾配置表已删除” ，表示容灾配置信息已删除；如果 “数据备份状态” 为 “ 容灾配置表仍然存在” ，表示容灾配置信息正在删除，请1分钟后再次执行此命令，如果连续5次查询都是配置信息正在删除，说明系统异常，请联系华为技术支持。

```
DSP DRINFO: PEERDRINSTID=
1
; 
```

**//配置NRF2。**

//关闭主备NRF容灾特性的License配置开关。

```
SET LICENSESWITCH: LICITEM=" LKV2ASNRR01", SWITCH=DISABLE;
```

//删除主备NRF容灾实例通信地址信息。

```
RMV DRCOMM: DRINSTID=
0
, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.1";
RMV DRCOMM: DRINSTID=
1
, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.2";
```

//删除主备容灾实例。

```
RMV DRINST: DRINSTID=
1
; 
```

//查询容灾配置信息，确保容灾配置信息已删除。如果 “数据备份状态” 为 “容灾配置表已删除” ，表示容灾配置信息已删除；如果 “数据备份状态” 为 “ 容灾配置表仍然存在” ，表示容灾配置信息正在删除，请1分钟后再次执行此命令，如果连续5次查询都是配置信息正在删除，说明系统异常，请联系华为技术支持。

```
DSP DRINFO: PEERDRINSTID=
0
; 
```
