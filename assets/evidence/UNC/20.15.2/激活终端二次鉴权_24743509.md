## [操作场景](#ZH-CN_OPI_0000001424743509)

需要使用终端二次鉴权时，进行终端二次鉴权激活。

> **说明**
> 适用于SMF。

## [必备事项](#ZH-CN_OPI_0000001424743509)

前提条件

- 请仔细阅读[二次鉴权特性概述](WSFD-108007 终端二次鉴权特性概述_73984008.md)。
- 完成[Radius鉴权接入配置](../../组网功能/WSFD-011305 Radius鉴权接入/激活Radius鉴权接入_50176279.md)。
- 完成[Radius功能配置](../../组网功能/WSFD-011306 Radius功能/激活RADIUS功能_15542173.md)。
  ![](激活终端二次鉴权_24743509.assets/notice_3.0-zh-cn_2.png)

  **UPLIST4RDS** 、 **UPFRDSCLIENTIP** 、 **UPFRDSSVR** 命令配置中的 **UPLISTNAME** 参数建议统一用小写。

数据

| 类别 | 参数 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| **ADD CPGTPUADDR** | IPV4ADDR | 192.168.0.20 | 全网规划 | 用于指定与UPF对接的本端GTP-U IPv4地址。(该参数在"IPVERSION"配置为"IPV4"时为条件必选参数。)<br>本特性中该参数取值是与UPF共同规划，是由SMF下发的分装GTPU报文的IP地址。 |
| **ADD UPLIST4RDS** | UPLISTNAME | uplist1 | 本端规划 | UP列表名称，用于指定UP列表名称。<br>取值范围：字符串类型，输入长度范围为1～63。长度1到63的非空格字符串。 |
| **ADD UPLIST4RDS** | UPINSTANCEID | up1 | 全网规划 | UP实例标识，用于指定UP实例标识。 |
| **ADD UPFRDSSVR** | SERVERTYPE | AUTHENTICATION | 本端规划 | 指定RADIUS服务器类型。<br>取值范围：<br>- “AUTHENTICATION（鉴权服务器）”：表示鉴权服务器。<br>- “ACCOUNTING（计费服务器）”：表示计费服务器。 |
| **ADD UPFRDSSVR** | SERVERIPV4 | 192.168.10.2 | 全网规划 | RADIUS服务器IPv4地址。DN-AAA的IP地址。 |
| **ADD UPFRDSCLIENTIP** | CLIENTYPE | AUTHENTICATION | 本端规划 | 指定RADIUS客户端类型。<br>取值范围：<br>- “AUTHENTICATION（鉴权客户端）”：表示鉴权客户端。<br>- “ACCOUNTING（计费客户端）”：表示计费客户端。<br>- “ACCT_AND_AUTH（计费和鉴权客户端）”：表示计费和鉴权客户端。 |
| **ADD UPFRDSCLIENTIP** | VPNINSTANCE | vpnInstance1 | 本端规划 | 指定VPN实例。本参数由UPF和DN-AAA共同规划，是UPF与DN-AAA对接的VPN。<br>取值范围：字符串类型，输入长度范围是1~31。 |
| **ADD UPFRDSCLIENTIP** | CLIENTIPV4 | 192.168.10.1 | 全网规划 | 用于指定RADIUS客户端IPv4地址。SMF下发至UPF，经UPF转发的ClientIP。 |

## [操作步骤](#ZH-CN_OPI_0000001424743509)

1. 进入 “MML命令行-UNC” 窗口。
2. 增加与UPF对接的本端GTP-U地址。
  **[ADD CPGTPUADDR](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/N4 GTP-U管理/N4 GTP-U地址管理/增加CP GTP-U地址（ADD CPGTPUADDR）_96805376.md)**
3. 增加中转使用的UP到RADIUS服务器使用的UPF列表。
  **[ADD UPLIST4RDS](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/UP列表/向RADIUS服务器使用的UP列表中增加UP（ADD UPLIST4RDS）_52749060.md)**
4. 可选配置：增加RADIUS服务器使用的UPF列表中指定UPF的功能属性。
  **[ADD RDSUPFCTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/RADIUS维护/RADIUS UPF控制/增加RADIUS服务器的UP列表中指定UP的功能属性（ADD RDSUPFCTRL）_35803148.md)**
5. 新增中转UPF与Radius服务器的绑定关系。
  **[ADD UPFRDSSVR](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/UPF中转Radius服务器/增加中转UPF与Radius服务器的绑定关系（ADD UPFRDSSVR）_35636449.md)**
6. 增加中转UPF与Radius客户端的绑定关系。
  **[ADD UPFRDSCLIENTIP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/UPF中转Radius客户端IP/增加中转UPF与Radius客户端的绑定关系（ADD UPFRDSCLIENTIP）_90026962.md)**
  > **说明**
  > - 配置“中转UPF与Radius服务器、客户端的绑定关系”之前需要先在UPF上执行**ADD NETWORKINSTVPNMAP**命令。否则SMF尝试建立N4-U会话失败，需延迟一段时间（最大15分钟）后才会再自动重试。
  > - 需先配置**[ADD UPFRDSSVR](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/UPF中转Radius服务器/增加中转UPF与Radius服务器的绑定关系（ADD UPFRDSSVR）_35636449.md)**命令，再配置**[ADD UPFRDSCLIENTIP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/UPF中转Radius客户端IP/增加中转UPF与Radius客户端的绑定关系（ADD UPFRDSCLIENTIP）_90026962.md)**命令，否则AAA计费业务无法正常进行。
  > - **[ADD UPFRDSCLIENTIP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/UPF中转Radius客户端IP/增加中转UPF与Radius客户端的绑定关系（ADD UPFRDSCLIENTIP）_90026962.md)**执行后SMF会立即触发建链，故需要其他配置都配置完成后最后执行，如果中间有其它命令修改，需要先执行**[RMV UPFRDSCLIENTIP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/UPF中转Radius客户端IP/删除中转UPF与Radius客户端的绑定关系（RMV UPFRDSCLIENTIP）_90026964.md)**再重新执行**[ADD UPFRDSCLIENTIP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/UPF中转Radius客户端IP/增加中转UPF与Radius客户端的绑定关系（ADD UPFRDSCLIENTIP）_90026962.md)**命令，否则SMF尝试建立N4-U会话失败，需延迟一段时间（最大15分钟）后才会再自动重试。

## [任务示例](#ZH-CN_OPI_0000001424743509)

任务描述

配置UPF中转Radius鉴权消息。

脚本

//增加与UPF对接的本端GTP-U地址。

```
ADD CPGTPUADDR:IPVERSION=IPV4,IPV4ADDR="192.168.0.20";
```

//向RADIUS服务器使用的UPF列表中增加UPF。

```
ADD UPLIST4RDS: UPLISTNAME="uplist1", UPINSTANCEID="up1";
```

//可选：增加RADIUS服务器的UPF列表中指定UPF的功能属性

```
ADD RDSUPFCTRL: UPLISTNAME="uplist1", UPINSTANCEID="up1",PREFERENCE=1,LOCKED=ENABLE;
```

//增加中转UPF与Radius客户端的绑定关系。

```
ADD UPFRDSSVR: SERVERTYPE=AUTHENTICATION, IPVERSION=IPV4, SERVERIPV4="192.168.10.2", UPLISTNAME="uplist1";
```

```
ADD UPFRDSCLIENTIP: CLIENTYPE=AUTHENTICATION, IPVERSION=IPV4, UPLISTNAME="uplist1", VPNINSTANCE="vpnInstance1", CLIENTIPV4="192.168.10.1";
```
