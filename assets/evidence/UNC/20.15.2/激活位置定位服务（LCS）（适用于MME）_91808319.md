## [操作场景](#ZH-CN_OPI_0191808319)

本操作指导介绍在运行网络中激活LCS特性的操作过程。

LCS（Location Services）是移动通信网络通过无线信号测量来确定UE地理位置信息及其移动速度的技术。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0191808319)

前提条件

- 请仔细阅读[WSFD-106401 位置定位服务（LCS）（适用于MME）特性概述](特性概述_91808305.md)。
- 3GPP协议定义的标准LCS组网（部署E-SMLC）场景或精简的LCS定位流程场景下，已经完成SLg和SLs接口数据配置，详细请参见[配置到E-SMLC的数据](../../../../../初始配置/UNC初始配置与调测/组网对接配置/配置AMF&MME&SGSN/配置MME/配置到E-SMLC的数据_29346481.md)和[配置到GMLC的数据](../../../../../初始配置/UNC初始配置与调测/组网对接配置/配置AMF&MME&SGSN/配置MME/配置到GMLC的数据_29306509.md)。
- 未部署E-SMLC网元的LCS组网场景下，已经完成SLg接口数据配置，详细请参见[配置到GMLC的数据](../../../../../初始配置/UNC初始配置与调测/组网对接配置/配置AMF&MME&SGSN/配置MME/配置到GMLC的数据_29306509.md)。
- 采用精简的LCS定位流程的场景下，要求E-SMLC能够根据MME上报的ECGI计算UE的位置信息并返回给MME。
- 完成加载License。

## [操作步骤](#ZH-CN_OPI_0191808319)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. **可选：**在未部署E-SMLC网元的LCS组网场景下，打开依赖特性“WSFD-110520 小区位置信息上报功能（S11接口）”的License配置开关。
  [**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
4. 增加发起定位请求的GMLC的权限。
  [**ADD GMLCAU**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/LCS/GMLC权限配置/增加GMLC权限配置(ADD GMLCAU)_26145794.md)
5. **可选：**设置LCS业务的运行参数。
  [**SET LCSPARA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/LCS/LCS软件参数表/设置LCS参数表记录(SET LCSPARA)_72225477.md)
  > **说明**
  > - 如果采用3GPP协议定义的标准LCS组网（部署E-SMLC），进行高精度的LCS定位，设置“4G LCS定位策略”为“PROTOCOL_POSITION(协议模式定位)”、“启用连接态LRC流程”为“NO(否)”。
  > - 如果采用未部署E-SMLC网元的LCS组网，进行较低精度的LCS定位，设置“4G LCS定位策略”为“NO_ESMLC_POSITION(无ESMLC定位)”。
  > - 如果采用精简的LCS定位，进行较低精度的LCS定位，设置“4G LCS定位策略”为“PROTOCOL_POSITION(协议模式定位)”、“启用连接态LRC流程”为“YES(是)”。

## [任务示例](#ZH-CN_OPI_0191808319)

任务描述

场景1：3GPP协议定义的标准LCS组网场景下，部署协议标准的LCS定位功能。

场景2：未部署E-SMLC网元的LCS组网场景下，部署LCS定位功能。

场景3：3GPP协议定义的标准LCS组网场景下，部署精简的LCS定位。

脚本

- 场景1
  //打开位置定位服务（LCS）的License配置开关。
  ```
  SET LICENSESWITCH: LICITEM="LKV2LCS02", SWITCH=ENABLE;
  ```
  //增加一条 “GMLC 标识” 为 “1” ， “支持的客户端类型” 为 “VALUE_ADDED_SERVICES（增值业务）” ， “支持的LCS业务类型” 为 “3” 的GMLC配置记录。
  ```
  ADD GMLCAU: GMLCID=1, CLTTYPE=VALUE_ADDED_SERVICES, SERTYPE=3;
  ```
  //设置LCS业务的运行参数，选择4G LCS定位策略为协议模式定位。

  ```
  SET LCSPARA: TR1=15, PAGTIMER=15, POSTIMER=15, MULIMSIMODE=REJECT, POSIDLE=60, GMLCAU=AUALL, LCS4GPOLICY=PROTOCOL_POSITION, LRC=NO;
  ```

- 场景2
  //打开位置定位服务（LCS）的License配置开关。

  ```
  SET LICENSESWITCH: LICITEM="LKV2LCS02", SWITCH=ENABLE;
  ```
  //增加一条 “GMLC 标识” 为 “1” ， “支持的客户端类型” 为 “VALUE_ADDED_SERVICES（增值业务）” ， “支持的LCS业务类型” 为 “3” 的GMLC配置记录。
  ```
  ADD GMLCAU: GMLCID=1, CLTTYPE=VALUE_ADDED_SERVICES, SERTYPE=3;
  ```
  //打开小区位置信息上报功能（S11接口）的License配置开关。
  ```
  SET LICENSESWITCH: LICITEM="LKV2LCR01", SWITCH=ENABLE;
  ```
  //设置LCS业务的运行参数，选择4G LCS定位策略为无ESMLC定位。
  ```
  SET LCSPARA: TR1=15, PAGTIMER=15, POSTIMER=15, MULIMSIMODE=REJECT, POSIDLE=60, GMLCAU=AUALL, LCS4GPOLICY=NO_ESMLC_POSITION;
  ```

- 场景3
  //打开位置定位服务（LCS）的License配置开关。
  ```
  SET LICENSESWITCH: LICITEM="LKV2LCS02", SWITCH=ENABLE;
  ```
  //增加一条 “GMLC 标识” 为 “1” ， “支持的客户端类型” 为 “VALUE_ADDED_SERVICES（增值业务）” ， “支持的LCS业务类型” 为 “3” 的GMLC配置记录。
  ```
  ADD GMLCAU: GMLCID=1, CLTTYPE=VALUE_ADDED_SERVICES, SERTYPE=3;
  ```
  //设置LCS业务的运行参数，选择4G LCS定位策略为协议模式定位，设置启用连接态LRC流程为YES。
  ```
  SET LCSPARA: TR1=15, PAGTIMER=15, POSTIMER=15, MULIMSIMODE=REJECT, POSIDLE=60, GMLCAU=AUALL, LCS4GPOLICY=PROTOCOL_POSITION, LRC=YES;
  ```
