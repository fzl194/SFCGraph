#### [适用NF](#ZH-CN_CONCEPT_0160375102)

GGSN、SGW-C、PGW-C、SMF

#### [定义](#ZH-CN_CONCEPT_0160375102)

N4接口是5G系统中SMF与UPF之间的接口，用于传输SMF/SGW-C/PGW-C/GGSN-C和UPF/SGW-U/PGW-U/GGSN-U间的控制面及用户面信息，通过该接口使控制面和用户面配合完成用户面规则下发和隧道建立功能。

#### [客户价值](#ZH-CN_CONCEPT_0160375102)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | UNC<br>的N4接口基于3GPP标准实现，可以实现SMF与不同厂家的UPF设备对接。 |
| 用户 | 用户不感知该特性。 |

#### [应用场景](#ZH-CN_CONCEPT_0160375102)

[图1](#ZH-CN_CONCEPT_0160375102__fig1) 是5G系统服务化架构接口图。N4接口是SMF与UPF之间的接口，用于传输SMF和UPF间的控制面信息及用户面信息。

**图1** 5G系统服务化架构接口图

<br>

![](WSFD-011136 N4接口_60375102.assets/zh-cn_image_0160374752_2.png)

> **说明**
> N4接口支持IPv4/IPv6双栈。
>
> UNC 中，GGSN-C/SGW-C/PGW-C与SMF合一部署，通过N4接口与UPF（GGSN-U/SGW-U/PGW-U）对接。即在 UNC 中，用于SGW-C和SGW-U之间传输数据的 Sxa接口 、用于PGW-C和PGW-U之间的传输数据的 Sxb接口 以及N4接口是合一的。
>
> UNC 中，SMF与UPF的配置分别支持与VPN绑定。SMF会选择与本端具有相同VPN的UPF进行建链，如果不存在与本端具有相同VPN的UPF，则与对端UPF建链失败。

#### [原理概述](#ZH-CN_CONCEPT_0160375102)

**N4接口协议栈**

N4接口协议栈如 [图2](#ZH-CN_CONCEPT_0160375102__fig25254624118) 所示。

**图2** N4接口协议栈

![](WSFD-011136 N4接口_60375102.assets/zh-cn_image_0160375049_2.png)

> **说明**
> - PFCP（Packet Forwarding Control Protocol）：SMF与UPF间控制面的数据转发协议。
> - GTP-U（GPRS Tunnelling Protocol for the user plane）：利用隧道协议在骨干网的GSN间传输用户数据，使用隧道协议传输数据，屏蔽上层协议影响，同时提供会话级的双向快速传输通道。GTP协议用户面除数据包外，还包括数据传输出错指示、支持的扩展头列表等用户面传输辅助信令。

**N4接口交互消息**

N4接口交互消息如 [表1](#ZH-CN_CONCEPT_0160375102__table1896510554181) 所示。

*表1 N4接口交互消息*

| 消息类型 | 消息名称 | 方向 | 作用 |
| --- | --- | --- | --- |
| 心跳检测消息 | PFCP Heartbeat Request | SMF->UPF<br>UPF->SMF | 用于探测通信路径和对端设备是否工作正常。 |
| 心跳检测消息 | PFCP Heartbeat Response | SMF->UPF<br>UPF->SMF | 用于探测通信路径和对端设备是否工作正常。 |
| 偶联管理消息 | PFCP Association Setup Request | SMF->UPF<br>UPF->SMF | 偶联建立，协商SMF和UPF之间的能力，如两端各自支持的功能。 |
| 偶联管理消息 | PFCP Association Setup Response | SMF->UPF<br>UPF->SMF | 偶联建立，协商SMF和UPF之间的能力，如两端各自支持的功能。 |
| 偶联管理消息 | PFCP Association Update Request | SMF->UPF<br>UPF->SMF | 偶联更新，如SMF支持的功能发生变化时，SMF发起偶联更新流程；UPF支持的功能发生变化、用户面可获得资源发生变化、请求释放偶联时，UPF发起偶联更新流程。 |
| 偶联管理消息 | PFCP Association Update Response | SMF->UPF<br>UPF->SMF | 偶联更新，如SMF支持的功能发生变化时，SMF发起偶联更新流程；UPF支持的功能发生变化、用户面可获得资源发生变化、请求释放偶联时，UPF发起偶联更新流程。 |
| 偶联管理消息 | PFCP Association Release Request | SMF->UPF | 偶联释放，如因为操作运维管理原因需要终结偶联。 |
| 偶联管理消息 | PFCP Association Release Response | UPF->SMF | 偶联释放，如因为操作运维管理原因需要终结偶联。 |
| 节点上报消息 | PFCP Node Report Request | UPF->SMF | 节点级信息上报，如UPF的N3/N9接口链路故障信息上报给SMF。 |
| 节点上报消息 | PFCP Node Report Response | SMF->UPF | 节点级信息上报，如UPF的N3/N9接口链路故障信息上报给SMF。 |
| 会话管理消息 | PFCP Session Establishment Request | SMF->UPF | 为用户建立N4接口会话。 |
| 会话管理消息 | PFCP Session Establishment Response | UPF->SMF | 为用户建立N4接口会话。 |
| 会话管理消息 | PFCP Session Modification Request | SMF->UPF | 更新PFCP会话，如新增一个PDR、更新一个PDR、删除一个PDR，触发PFCP会话更新。 |
| 会话管理消息 | PFCP Session Modification Response | UPF->SMF | 更新PFCP会话，如新增一个PDR、更新一个PDR、删除一个PDR，触发PFCP会话更新。 |
| 会话管理消息 | PFCP Session Deletion Request | SMF->UPF | 释放PFCP会话，如PDN删除，触发PFCP会话删除。 |
| 会话管理消息 | PFCP Session Deletion Response | UPF->SMF | 释放PFCP会话，如PDN删除，触发PFCP会话删除。 |
| 会话管理消息 | PFCP Session Report Request | UPF->SMF | 会话信息上报，如用户流量配额用完，触发UPF通过该消息上报用户流量。 |
| 会话管理消息 | PFCP Session Report Response | SMF->UPF | 会话信息上报，如用户流量配额用完，触发UPF通过该消息上报用户流量。 |

详细的消息及流程介绍请参见3GPP TS 29.244 V15.3.0。
