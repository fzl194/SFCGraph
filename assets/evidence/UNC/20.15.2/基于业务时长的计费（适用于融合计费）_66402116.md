# 基于业务时长的计费（适用于融合计费）

融合计费中，SMF根据CHF下发的计费事件（Triggers）与CHF进行交互，并通过下列两个信元完成基本的配额授权与用量上报，是否根据时长向终端用户收费由计费中心决定。有关融合计费系统的详细描述请参考 [WSFD-011206 支持融合计费](../../WSFD-011206 支持融合计费_62745898.md) 。

- CHF下发业务的时长配额：Charging Data Request>MultipleUnitInformation>grantedUnit>time
- SMF上报业务的时长用量：Charging Data Request>MultipleUnitUsage>UsedUnitContainer>time

融合计费系统使用连续时长方式统计时长，带配额管理的融合计费与不带配额管理的融合计费的业务时长开始、结束时长统计的时间点有所区别。

- 带配额管理
  业务时长统计从UPF首次收到授权的配额成功时开始，直到用户去激活时或者该配额管理实例终止结束。无论用户是否进行该配额的业务，都将全部时长作为有效计费时间。
  ![](基于业务时长的计费（适用于融合计费）_66402116.assets/zh-cn_image_0229961148_2.png)
- 不带配额管理
  业务时长统计从UPF首次收到业务数据包时开始，直到业务终止时结束，如图中RG1和RG2的业务统计时长。
  ![](基于业务时长的计费（适用于融合计费）_66402116.assets/zh-cn_image_0229961149_2.png)
  > **说明**
  > - 对于基于用户的计费，使用时长从用户激活上线时开始统计，直到用户下线去激活时结束，即无论用户是否进行业务，全部时长均作为有效计费时间，如图中User的业务统计时长。
  > - 离线RG的业务时长统计方式也可以修改为与带配额时长统计方式相同，通过[**SET CNVRGDCHGPARA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/全局配置/设置融合计费全局参数（SET CNVRGDCHGPARA）_09653056.md)命令的“OFLRGTIMECALC”参数控制，统计方式可以配置为从UPF首次收到授权的配额成功时开始，直到用户去激活时或者该配额管理实例终止结束。
