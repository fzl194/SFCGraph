## [操作场景](#ZH-CN_OPI_0166289935)

- 新增PGW-C/GGSN时，需配置与其对接的SGSN/SGW-C PLMN标识的获取方式，以判断接入用户归属的属性，为用户的接入策略和计费策略实施提供依据。
- 新增SGSN/SGW-C网元，且新增网元地址与原有网元不在同一网段时，配置新增网段到PLMN的映射关系。
- PGW-C/GGSN上新增APN时，配置该APN是否支持通过IP映射的方式获取用户接入的SGSN/SGW-C的PLMN标识。

> **说明**
> 适用于 GGSN、 SGW-C、PGW-C。

本节以PGW-C获取SGW-C为例进行介绍如何激活PLMN标识获取。

## [操作步骤](#ZH-CN_OPI_0166289935)

1. 进入 “MML命令行-UNC” 窗口。
2. 本地配置PGW-C的HPLMN。
  [**ADD NGHPLMN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/运营商管理/Home PLMN信息管理/增加5G Home PLMN（ADD NGHPLMN）_09651693.md)
3. 配置SGW-C缺省的PLMN信息。
  [**ADD DFTSRVNODEPLMN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入控制/缺省服务节点PLMN/增加默认PLMN（ADD DFTSRVNODEPLMN）_09653247.md)
4. 配置PGW-C获取SGW-C PLMN方式的优先级。
  [**SET PLMNPRIORITY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入控制/获取PLMN管理/获取PLMN方式的优先级/设置获取PLMN优先级（SET PLMNPRIORITY）_09653827.md)
5. 配置允许指定APN下的用户通过SGW-C IP映射获取SGW-C的PLMN标识的方式来获取PLMN标识。
    a. 指定APN实例名称时，设置允许此APN下的用户通过SGW-C IP映射方式来获取PLMN标识的开关。
      [**ADD APN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/APN管理/APN/增加APN配置（ADD APN）_09653747.md)
    b. 配置SGW-C IP与PLMN标识间的映射关系表。
      [**ADD SRVNODEPLMN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入控制/获取PLMN管理/服务节点PLMN/增加SGSN_SGW_PGW地址段和PLMN标识之间的映射关系（ADD SRVNODEPLMN）_09652097.md)

## [任务示例](#ZH-CN_OPI_0166289935)

任务描述

本地配置PGW-C的HPLMN，并配置PGW-C获取SGW-C PLMN方式的优先级，最终判断用户类型。

脚本

1、本地配置PGW-C的HPLMN。

```
ADD NGHPLMN: NOID=0, MCC="460", MNC="02";
```

2、配置SGW-C缺省的PLMN信息。

```
ADD DFTSRVNODEPLMN:NODETYPE=SGW,PLMNCODE="46002";
```

3、配置PGW-C按照优先级获取用户接入网络的SGW-C的PLMN标识。

//配置PGW-C获取SGW-C的PLMN标识的优先级。

```
SET PLMNPRIORITY:NODETYPE=PGWC,PRIORITYMODE=LOCAL,PGW1STPRIORITY=SERVING_NETWORK,PGW2NDPRIORITY=SGW_IP,PGW3RDPRIORITY=ULI;
```

4、配置允许指定APN下的用户通过SGW-C IP映射获取SGW-C的PLMN标识的方式来获取PLMN标识。

//指定APN实例名称时，设置允许此APN下的用户通过SGW-C IP映射方式来获取PLMN标识的开关。

```
ADD APN:APN="apn-test",SERVINGNODEPLMN=ENABLE;
```

//配置SGW-C IP与PLMN标识间的映射关系表。

```
ADD SRVNODEPLMN:SRVNODESTARTV4="192.168.1.1",SRVNODEENDV4="192.168.2.2",SRVNODEPLMN="46002",IPVERSION=IPV4;
```
