#### [适用NF](#ZH-CN_CONCEPT_0248669741)

AMF

#### [定义](#ZH-CN_CONCEPT_0248669741)

区域漫游限制是指 UNC 支持根据用户签约或者本地配置的移动性接入限制信息判断是否允许用户在当前位置注册到5G核心网的功能。 UNC 按以下两种策略实施区域漫游限制：

- 根据用户接入和移动签约数据中的“禁止区域（Forbidden Area）”信息控制用户的接入。
- AMF针对指定的区域在本地配置用户接入的策略，然后基于策略控制用户的接入。

#### [客户价值](#ZH-CN_CONCEPT_0248669741)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 满足运营商基于区域的差异化运营。 |
| 用户 | 无。 |

#### [应用场景](#ZH-CN_CONCEPT_0248669741)

运营商希望针对指定的区域，限制指定的用户接入或者禁止接入。

#### [应用限制](#ZH-CN_CONCEPT_0248669741)

- AMF本地配置的禁止区域发生变化时，仅针对新注册用户生效，对于已注册到本AMF的存量用户不生效。
- 区域漫游限制功能对于紧急注册不生效。
- 优先检查签约数据中的“禁止区域（Forbidden Area）”信息，再匹配AMF本地配置的策略。
- 签约数据中的“禁止区域（Forbidden Area）”信息对于漫游用户不生效。

#### [原理概述](#ZH-CN_CONCEPT_0248669741)

**5G移动性限制背景介绍**

网络的移动性管理既要保证UE可达和数据传输连续，又要防止UE接入某些受限区域或者请求不该使用的服务。移动性限制就是指3GPP内UE的接入访问受限、移动受限等。

5G协议中则明确定义了移动性接入限制信息有RAT限制、核心网限制、禁止区域、服务区限制四种类型：

- **RAT限制（RAT restriction ）**
- **核心网限制（Core Network type restriction）**
- **禁止区域****（****Forbidden Area****）**
- **服务区限制（Service Area Restriction）**

5G系统中的移动性限制是UE、基站和核心网相互协同完成的，作用如下：

1. UE在某区域尝试接入如果由于限制被核心网拒绝，UE支持根据注册被拒的原因值在本地生成和维护禁止区域列表，并且在一定时长内禁止尝试在该区域内注册到核心网。
2. 基站从核心网拿到用户移动性接入限制信息后，在切换流程选择目标网络时会过滤受限的RAT、跟踪区等。
3. 核心网作为移动性限制的决策中心，综合运营商的各种策略（如签约数据或AMF本地配置移动性接入限制信息）执行接入限制功能。

UNC 作为移动性限制的决策中心，支持实现移动性限制，并且根据不同类型的 **移动性接入限制信息** 实现不同的限制功能。包括：

- 根据**RAT限制（RAT restriction ）**和**核心网限制（Core Network type restriction）**实现用户接入控制功能，如[用户接入控制功能（适用于AMF）](../../../基本接入功能/WSFD-106003 用户接入控制功能/用户接入控制功能（适用于AMF）_48552634.md)。
- 根据**禁止区域****（****Forbidden Area****）**实现区域漫游限制功能，如本特性。
- 根据**服务区限制（Service Area Restriction）**实现服务区域限制功能。

本特性主要介绍区域漫游限制功能。

****区域漫游限制 **涉及的移动性接入限制信息介绍******

- **禁止（注册）区域****（****Forbidden Area****）****含义**
  禁止（注册）区域（Forbidden Area）表示禁止用户注册（除紧急注册）到网络的区域。该信息中指示了禁止区域列表，用于表明是否限制用户接入指定区域。
- **禁止区域获取方式**
    - AMF：AMF可以从用户签约数据获取禁止区域信息，也可以在本地针对5G网络中指定区域配置用户的接入策略。
    - UE：AMF不下发禁止区域列表给UE，UE支持根据注册被拒的原因值在本地生成和维护禁止区域列表，并且在一定时长内禁止区域尝试在该区域内注册到AMF。UE本地管理的禁止区域列表占满时，新增的禁止区域会替代原禁止区域，当UE开关机、插拔USIM卡或UE侧相关定时器超时，Forbidden Area会被清空。清空后，UE会再次发起小区选择，进而再发起到核心网的注册流程，生成本地禁止区域列表。
    - 基站：初始注册基站不会做区域漫游限制检查，核心网在注册流程结束后会将禁止区域列表下发给基站，该列表最多携带4096个TAC。基站用于切换场景下的目标小区筛选。

****区域漫游限制控制模式****

在注册、移动性注册和切换等业务流程中，当用户接入，AMF首先根据UE签约或AMF本地配置的 **禁止（注册）区域** **（** **Forbidden Area** **）** 信息来决策是否允许用户在当前位置注册到5G核心网， UNC 支持以下区域漫游限制控制模式，可以基于场景选择合适的模式，如 [表1](#ZH-CN_CONCEPT_0248669741__table165688181119) 。

- 模式一：以用户签约的“禁止区域（ForbiddenArea）”控制用户的接入。
  支持在UDM上定义 “禁止” 用户接入的区域，即签约的禁止区域（ForbiddenArea），然后基于签约的禁止区域（ForbiddenArea）对于用户进行接入限制。
- 模式二：以位置区域为对象，控制该区域用户的接入策略。
    - 支持在AMF上通过白名单的模式配置“允许”某类用户接入的策略，然后基于配置的接入策略对于用户进行接入限制。
    - 支持在AMF上通过黑名单的模式配置“禁止”某类用户接入的策略，然后基于配置的接入策略对于用户进行接入限制。
  *表1 基于场景选择合适的模式*

  | **限制模式** | **适用场景** |
  | --- | --- |
  | 模式一：针对指定的用户，在UDM签约时，禁止在指定的区域内接入。 | **覆盖范围较小**<br>（如涉及TAC数量少于10个）、<br>**组网稳定**<br>的区域内，且<br>**禁止少数用户**<br>接入的场景。 |
  | 模式二：针对指定的区域，限制指定的用户接入或者禁止接入。 | - 针对规划区域，允许少量用户接入的场景。<br>- 针对广大区域，限制少量用户接入的场景（不适合通过签约中的禁止区域指定，因为涉及的TAC数量过多）。 |

**区域漫游限制判断流程**

当上述的限制模式同时存在时，优先检查签约数据中的“禁止区域（Forbidden Area）”信息，再匹配AMF本地配置的策略。

> **说明**
> - 给UE分配的注册区不能包含禁止区域中的TA。
> - 本特性主要介绍下区域漫游限制流程。
> - 区域漫游限制流程是移动性限制的一部分，在进行区域漫游限制流程之前，必须先进行接入控制检查，如[用户接入控制功能（适用于AMF）](../../../基本接入功能/WSFD-106003 用户接入控制功能/用户接入控制功能（适用于AMF）_48552634.md)。检查通过了再执行区域漫游限制流程，否则拒绝UE接入。
