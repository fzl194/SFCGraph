# 调测流量阈值功能

## [操作场景](#ZH-CN_OPI_0295923507)

当运营商部署了离线计费业务，需要调测GGSN/SGW-C/PGW-C的流量阈值功能。

> **说明**
> 适用于SGW-C、PGW-C 、GGSN 。

## [必备事项](#ZH-CN_OPI_0295923507)

前提条件

- 请仔细阅读[Ga/Gy接口离线/在线计费](../../../../../业务专题/5G Core 计费解决方案/计费解决方案概述/计费原理/Ga_Gy接口离线_在线计费_87165686.md)。
- 完成[配置到CG的数据（GGSN/SGW-C/PGW-C）](../激活离线计费/配置到CG的数据（GGSN_SGW-C_PGW-C）_95923479.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 流量阈值 | 流量阈值（千字节）（VOLUMETHRESHOLD） | 20 | 已配置数据中获取 | **SET OFCTHRESHOLD** |
| APN实例名 | APN名称（APN） | apn-test | 已配置数据中获取 | 已通过命令<br>**ADD APN**<br>进行配置，可以使用命令<br>**LST APN**<br>查询。 |

工具

- 测试终端
- OM Portal
- CG客户端工具

## [操作流程](#ZH-CN_OPI_0295923507)

1. 激活用户。
2. 用户开始访问业务。
3. 访问业务产生的流量超过设置的流量阈值后，到CG上检查话单**causeForRecClosing**字段是否为**Volume Limit**。

本调测任务以APN实例下绑定流量阈值为20KByte为例说明调测过程。

## [操作步骤](#ZH-CN_OPI_0295923507)

1. 进入 “MML命令行-UNC” 窗口。 设置流量阈值是20KByte，时间阈值不生效。
  **SET OFCTHRESHOLD**
  > **说明**
  > 离线计费默认使能时长计费和流量计费，同时设置了时长阈值和流量阈值，当其中一个阈值先达到，则按此阈值生成话单，此后新产生的话单两种阈值重置，不进行累计。
2. 激活用户，测试终端使用“apn-test”APN接入网络，访问业务。
    - 如果测试终端成功接入网络，请执行[步骤 3](#ZH-CN_OPI_0295923507__stp2)。
    - 如果测试终端无法接入网络，请调测UNC的接入功能。
3. 用户访问业务，当使用的流量超过设置的流量阈值20KByte后，产生话单，去激活用户。
4. 登录CG客户端，查询此用户刚生成的话单（本实例是01:30左右）。如 [图1](#ZH-CN_OPI_0295923507__fig91541721122913) 所示，查看所生成话单中话单关闭原因字段是否是流量阈值到，且话单各List of Traffic Volumes流量总和加起来应该刚好超过流量阈值20Kbyte，如 [图2](#ZH-CN_OPI_0295923507__fig2) 所示。
  > **说明**
  > 当用户流量达到流量阈值的时候，会触发产生话单。但是数据包大小不一，所以话单中记录的用户总流量不可能完全等于流量阈值，一般都会大于流量阈值。
  **图1** CG客户端查询1

  ![](调测流量阈值功能_95923507.assets/zh-cn_image_0295923478_2.png)
  **图2** CG客户端查询2

  ![](调测流量阈值功能_95923507.assets/zh-cn_image_0295923559_2.png)

  <br>
    - 如果是，表明流量阈值功能正常，本调测任务结束。
    - 如果否，则继续执行[步骤 5](#ZH-CN_OPI_0295923507__stp4)。
5. 执行 **LST APNCHARGECTRL** apn-test命令，查看此APN下是否绑定了计费模板OFCTemplate。
  ```
  LST APNCHARGECTRL:APN="apn-test";
  ```
  ```
  RETCODE = 0  操作成功
  APN计费配置
  ------------
                                     APN名称  =  apn-test
                                            ......
                           PGW离线计费模板名  =  offlinecharge-test
                           SGW离线计费模板名  =  offlinecharge-test
                          GGSN离线计费模板名  =  offlinecharge-test
                                            ......
  (结果个数 = 1)
  ---    END
  ```
    - 如果是，请执行[步骤 6](#ZH-CN_OPI_0295923507__stp5)。
    - 如果否，请执行[配置离线计费参数（GGSN/SGW-C/PGW-C）](../激活离线计费/配置离线计费参数（GGSN_SGW-C_PGW-C）_95923590.md)重新配置，然后按步重新调测。
6. 执行 **LST OFCTEMPLATE** 命令，查看该APN实例绑定的OFCTemplate模板中的流量阈值VolumeThreshold配置是否正确。
  ```
  LST OFCTEMPLATE:OFCTEMPLATENAME="offlinecharge-test";
  ```
  ```
  RETCODE = 0  操作成功。

  离线计费模板配置
  -------------------------
                    离线计费模板名  =  offlinecharge-test
                                  ......                            
                流量阈值（千字节）  =  20
                                  ......                 
  (结果个数 = 1)
  ---    END
  ```
    - 如果是，请执行[步骤 7](#ZH-CN_OPI_0295923507__step1829245425110)。
    - 如果否，则返回[步骤 1](#ZH-CN_OPI_0295923507__step1)重新配置数据，然后按步重新调测。
7. 收集信息并寻求技术支持。
    a. 执行 **EXP MML** 命令将当前配置数据导出为MML脚本文件并保存。
    b. 收集并保存上述所有查询信息。
    c. 收集归纳所有信息并联系华为技术支持解决。
