# 实现原理（适用于4G）

本节从以下方面描述鉴权的基本原理：

- **[EPS鉴权原理](#ZH-CN_TOPIC_0170014630__p13514131316107)**
- **[EPS鉴权流程](#ZH-CN_TOPIC_0170014630__p1951516136104)**

除了基本的鉴权功能外， UNC 还提供了以下特性：

- **[预取鉴权集](#ZH-CN_TOPIC_0170014630__p14515813151019)**

[**EPS鉴权原理**](#ZH-CN_TOPIC_0170014630__sub5_2)

用户接入网络的时候，如果需要鉴权，则MME选取一组未使用过的鉴权四元组向量并向UE发起鉴权请求。该请求消息中携带所选取的鉴权向量中的RAND、AUTN和KSI <sub>ASME</sub> 。USIM根据AUTN对网络进行鉴权，如果鉴权成功，则UE会继续计算出RES并返回给MME。MME会使用鉴权向量组中的XRES和UE返回的RES比较，相同则鉴权成功，否则鉴权失败。鉴权成功后，UE计算并存储K <sub>ASME</sub> 值用于后续的加密及完整性保护流程。

[**EPS鉴权流程**](#ZH-CN_TOPIC_0170014630__sub5_3)

MME发起鉴权时，首先检查自身是否有鉴权集，如果没有鉴权集，则会向HSS索取鉴权集。获取鉴权向量的数量可以通过配置进行选择，获得鉴权向量后即可进行下面的EPS AKA流程。

[图1](#ZH-CN_TOPIC_0170014630__fig_1) 描述了MME到HSS取鉴权数据的流程，该过程的目的是MME从HSS取一组或多组EPS鉴权向量(RAND，AUTN，XRES，K <sub>ASME</sub> )。

**图1** MME到HSS取鉴权数据

<br>

![](实现原理（适用于4G）_70014630.assets/zh-cn_image_0170017201_2.png)

过程描述如下：

1. MME会向HSS发送Authentication Information Request消息，携带IMSI、Serving Network's Identity (SN id)、NT(Network Type)。
2. HSS接收到消息后，返回鉴权向量，HSS向MME返回Authentication Information Answer消息，如果请求了多EPS鉴权向量，则这些向量将被编序后传送给MME。

[图2](#ZH-CN_TOPIC_0170014630__fig_2) 描述了EPS AKA的流程，AKA（Authentication and Key Agreement）的作用包含两个，即安全认证和密钥同步协商。

**图2** EPS用户鉴权流程

<br>

![](实现原理（适用于4G）_70014630.assets/zh-cn_image_0170017202_2.png)

过程描述如下：

1. MME向ME/USIM发送“User Authentication Request”鉴权请求消息来触发鉴权过程，该消息中包含携带RAND、AUTN，KSI<sub>ASME</sub>(Key Set Identifier)。MME会从HSS获取多组鉴权向量，取一组发送给ME/USIM。
2. ME/USIM发送“User Authentication Response”消息响应MME。
    - USIM根据AUTN对网络进行鉴权，如果鉴权不成功，UE会向MME返回User Authentication Failure消息，并指明原因。
    - 如果UE鉴权成功，则UE会根据RAND继续计算出RES并返回给MME。MME会使用鉴权向量组中的XRES和UE返回的RES比较，相同则鉴权成功，否则鉴权失败并向UE发User Authentication Reject消息。
    - 如果MME鉴权成功，UE计算并存储K<sub>ASME</sub>值以便后续的加密及完整性保护。
    - 如果MME鉴权失败并且UE发送流程的消息中携带的是GUTI，则网络可以发起身份识别流程以请求IMSI决定是否二次鉴权；若UE发送流程的消息中携带的是IMSI，则给UE发送User Authentication Reject消息。

[**预取鉴权集**](#ZH-CN_TOPIC_0170014630__sub5_4)

出于安全考虑，MME中保存的每组鉴权集只能使用一次，MME中最多可以保存5组鉴权集，当保存的鉴权集用完后，MME需要向HSS请求新的鉴权集。

在用户附着、TAU等过程中才发起鉴权请求的话，该鉴权流程会导致接入或激活流程时间的延长，影响到用户的体验。

UNC 提供了预取鉴权集功能，在用户附着、TAU等过程中使用最后一组MME保存的鉴权集时，MME主动向HSS请求新的鉴权集。因为新的鉴权集流程与正在进行的用户附着、TAU等过程携带的信息无关，所以该用户附着、TAU等过程无需等待新的鉴权集流程的结束，从而提高了流程处理速度，有效的改善了用户体验。
