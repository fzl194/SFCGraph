## [操作场景](#ZH-CN_OPI_0188625065)

本操作指导介绍在运行网络中激活LTE精准寻呼的操作过程。

UNC 基于用户类型、业务类型选择相应的寻呼策略，综合用户的移动性进行寻呼规则过滤，选择合适的范围对用户进行寻呼，从而有效地减少整个网络中eNodeB的寻呼负荷，节省网络资源。

> **说明**
> - 适用于MME。
> - 本特性与**WSFD-102004 基于VoLTE的优先语音服务**特性中的**MPS优先级**业务存在冲突，启用前请先检查是否已配置MPS业务（[**SET MPSARP**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/MPS配置/MPS ARP信息配置/设置MPS ARP配置(SET MPSARP)_26305312.md)），如果已配置，建议先关闭MPS业务，再开启本特性，否则精准寻呼业务会失效。

## [必备事项](#ZH-CN_OPI_0188625065)

前提条件

- 请仔细阅读[WSFD-206001 精准寻呼（4G）](../WSFD-206001 精准寻呼（4G）_88625078.md)。
- 部署基于APN，QCI或ARP的精准寻呼策略时，S-GW需要支持在发送给MME的Downlink Data Notification消息中携带EPS Bearer ID信元。
- 建议由华为专业服务对本网进行评估后选定最优的精准寻呼参数。
- 已完成加载License。

## [操作步骤](#ZH-CN_OPI_0188625065)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. 打开eNodeB学习邻接关系和寻呼范围开关。
  [**SET S1PAGINGCTRL**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼策略管理/设置S1寻呼策略控制表(SET S1PAGINGCTRL)_72345845.md)
4. 配置寻呼策略及寻呼规则。
  > **说明**
  > - 部署了[WSFD-102301 基于CSFB的语音业务](../../../语音功能/WSFD-102301 基于CSFB的语音业务_70014756.md)、[WSFD-102001 VoLTE基础语音业务](../../../语音功能/WSFD-102001 VoLTE基础语音业务_67023607.md)或[WSFD-102601 LTE一键通基础功能特性概述（适用于MME）](../../../语音功能/WSFD-102600 Push to Talk（适用于MME）/WSFD-102601 LTE一键通基础功能（适用于MME）/特性概述_71266392.md)的情况下，启用LTE精准寻呼特性会增加被叫寻呼时延，在部署本特性时，建议针对CSFB、VoLTE语音或PTT业务使用TA List的寻呼范围，从而在不影响语音被叫时延的情况下节省网络资源。
  > - 针对数据业务和其他非语音类业务，“寻呼动作组合”推荐采用“最近访问eNodeB -> TA List”组合，即“ACTGRP（寻呼动作组合）”取值设置为“LAST_ENODEB-1”。这样能够有效减少寻呼范围的同时，明显减少对eNodeB侧的CPU消耗。
  [**ADD S1PAGINGRULE**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼规则管理/增加S1寻呼规则(ADD S1PAGINGRULE)_26306058.md)
5. 检查用户移动性，进行寻呼规则过滤。
  [**SET S1PAGINGCTRL**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼策略管理/设置S1寻呼策略控制表(SET S1PAGINGCTRL)_72345845.md)
  > **说明**
  > “ENB_STICK_TIME（eNodeB粘性时长）” 推荐设置为2min。
6. 配置指定TA List寻呼规则。
  [**ADD TALSTPAGINGPLCY**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/基于TA List寻呼策略管理/增加TA List寻呼策略配置(ADD TALSTPAGINGPLCY)_26306074.md)
  > **说明**
  > - 若针对指定的TA List关闭精准寻呼功能，设置 [**ADD TALSTPAGINGPLCY**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/基于TA List寻呼策略管理/增加TA List寻呼策略配置(ADD TALSTPAGINGPLCY)_26306074.md) 命令中 “CLSPRECISEPAG” 参数为 “YES” ， [步骤 4](#ZH-CN_OPI_0188625065__step3) 设置的精准寻呼规则不生效。
  > - 若需要关闭特定TA List的精准寻呼功能，且针对TA List范围的寻呼配置差异化的寻呼定时器时长，配置 [**ADD TALSTPAGINGPLCY**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/基于TA List寻呼策略管理/增加TA List寻呼策略配置(ADD TALSTPAGINGPLCY)_26306074.md) 命令中 “CLSPRECISEPAG” 参数为 “YES” ， “T3413” 、 “N3413” 和 “PAGINGDELTA” 为需要的值。
  > - “TALISTID” 需要在 [**ADD TALST**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/TA List管理/增加跟踪区列表(ADD TALST)_26305388.md) 中已经配置。

## [任务示例](#ZH-CN_OPI_0188625065)

任务描述

任务1：启用LTE精准寻呼。针对CSFB和VoLTE语音业务，采用TA List寻呼范围组合，针对数据业务和其他非语音类业务，使用“最近eNodeB -> TA List”寻呼范围组合。

任务2：在任务1的基础上，对指定的TA list关闭精准寻呼，直接对指定TA List下的所有eNodeB发起寻呼。

脚本

任务1：

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2PRPG02", SWITCH=ENABLE;
```

//打开学习最近eNodeB开关。

```
SET S1PAGINGCTRL: LASTENBSW=ON, NEIB_ENBSW=OFF, LASTTA=OFF, SUBT3413=3;
```

//配置寻呼策略及寻呼规则。

//配置CSFB语音业务不启用LTE精准寻呼。

```
ADD S1PAGINGRULE: GRPTYPE=ALL_USER, SVRTYPE=SGS_PAGING, SGSSI=CS_CALL, PRIORITY=100, ACTGRP=LAST_ENODEB-0&NEIGH_ENODEB-0&LAST_TAI-0, DESC="Voice of CSFB";
```

//配置VoLTE语音业务不启用LTE精准寻呼。

```
ADD S1PAGINGRULE: GRPTYPE=ALL_USER, SVRTYPE=S11_DOWNLINK, APNIND=ALL_APN, QCI=1, PRIORITY=100, ACTGRP=LAST_ENODEB-0&NEIGH_ENODEB-0&LAST_TAI-0;
ADD S1PAGINGRULE: GRPTYPE=ALL_USER, SVRTYPE=S11_DOWNLINK, APNIND=ALL_APN, QCI=5, PRIORITY=100, ACTGRP=LAST_ENODEB-0&NEIGH_ENODEB-0&LAST_TAI-0;
```

//配置数据业务和其他非语音业务启用LTE精准寻呼。

```
ADD S1PAGINGRULE: GRPTYPE=ALL_USER, SVRTYPE=S11_DOWNLINK, APNIND=ALL_APN, ACTGRP=LAST_ENODEB-1, DESC="data";
ADD S1PAGINGRULE: GRPTYPE=ALL_USER, SVRTYPE=OTHER, ACTGRP=LAST_ENODEB-1;
```

//检查用户移动性，进行寻呼规则过滤。

```
SET S1PAGINGCTRL: LOWMOBICHKSW=ON, ENB_STICK_TIME=2;
```
