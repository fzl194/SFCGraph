#### [适用NF](#ZH-CN_CONCEPT_0248043389)

SGSN、 GGSN、 MME、SGW-C、PGW-C

#### [定义](#ZH-CN_CONCEPT_0248043389)

本特性支持在PDP上下文激活流程/PDN连接建立流程中为UE同时分配IPv4和IPv6两种类型的地址，以便UE后续可以使用IPv4v6地址进行数据传输，为用户提供IPv4v6双栈接入服务。

#### [客户价值](#ZH-CN_CONCEPT_0248043389)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 相对于IPv4和IPv6单栈PDP，启用IPv4v6双栈可以节省SGSN/MME上的PDP/PDN资源。<br>- 通过部署IPv4v6双栈接入同时支持IPv4地址和IPv6地址，避免进行IPv4地址和IPv6地址之间的NAT转换。<br>- 以IPv4v6双栈接入为基础，进行网络部署，实现了现网向IPv6的平滑演进，充分保护运营商现网的投资。<br>- 基本不用对网络结构进行调整，用户可以同时使用IPv4地址和IPv6地址，可以解决以上两大问题。使网络中具备更多的IP资源，从而解决网络中IP资源不足的问题。 |
| 用户 | 同时享受IPv4和IPv6业务。 |

#### [应用场景](#ZH-CN_CONCEPT_0248043389)

本特性应用于网络中存在双栈类型的手机接入的情况。

**PDP上下文激活涉及的IPv4v6流程（适用于2/3G）**

PDP上下文激活涉及的IPv4v6流程如 [图3](#ZH-CN_CONCEPT_0248043389__fig1) 所示。

**图3** PDP上下文激活涉及的IPv4v6流程

<br>

![](特性概述_48043389.assets/zh-cn_image_0248043363_2.png)

此处只针对IPv4v6双栈控制流程作如下描述：

1. 通过[签约数据修改控制策略](#ZH-CN_CONCEPT_0248043389__p6)，使得该用户可以支持IPv4v6业务。
2. UE向GnGp SGSN发送激活PDP Context请求后，如果携带的PDP address为空，表示使用动态地址，则要求GGSN分配一个动态地址。GnGp SGSN根据一定的算法选择一个APN，并推导出GGSN。GGSN选择完成后，如果UE请求的是IPv4v6双栈，则启动[IPv4v6双栈控制策略](#ZH-CN_CONCEPT_0248043389__p7)。
3. PDP Type确定为IPv4v6双栈后，GnGp SGSN发送Create PDP Context Request消息给GGSN，消息中携带双栈标志(Dual Address Bearer Flag=1)和PDP Type（IPv4v6）。GGSN回复Create PDP Context Response消息给GnGp SGSN，且GnGp SGSN保存消息中的End User Address字段，并将原因值返回给UE。GGSN可以接受双栈请求，也可以拒绝双栈请求。当拒绝时，双栈会被纠正为单栈，GGSN直接下发单栈地址给UE，并携带原因值如下：
    - “New PDP type due to network preference”，表示GGSN选择了一个与UE请求不一致的PDP Type。
    - “New PDP type due to single address bearer only”，表示UE请求双栈，GGSN支持IPv4和IPv6单栈，但不支持双栈。
4. GnGp SGSN发送RAB Assignment Request消息给RNC，请求建立RAB，如果是IPv4v6双栈，消息中携带PDP Type Information Extension字段。
5. IPV6场景下，GGSN生成IPv6路由广播（前缀）并且经SGSN将其发送到MS。

> **说明**
> 若激活的类型不是双栈，SGSN给UE返回的Activate PDP Context Accept消息里会带上合适的原因值。

**其它业务涉及的IPv4v6流程（适用于2/3G）**

- RAU（Inter/Intra SGSN RAU）
    - Inter SGSN RAU
          - 旧侧：如果UE在旧侧的PDP上下文建立的是IPv4v6双栈，在消息SGSN Context Response中的PDP上下文中包含IPv4地址和IPv6地址。
          - 若新侧是UTRAN： GnGp SGSN保存IPv4v6双栈地址，在后续的Service Request中，启动[IPv4v6双栈控制策略](#ZH-CN_CONCEPT_0248043389__p7)。
          - 若新侧是GERAN：GnGp SGSN在重建承载时，启动[IPv4v6双栈控制策略](#ZH-CN_CONCEPT_0248043389__p7)。
    - Intra SGSN RAU
      新侧在进行PDP上下文更新时就启动了 [IPv4v6双栈控制策略](#ZH-CN_CONCEPT_0248043389__p7) 。
- TAU
  如果UE在旧侧的PDP上下文建立的是IPv4v6双栈，则在消息SGSN Context Response中的PDP上下文中包含IPv4地址和IPv6地址。
- 修改PDP上下文
  GGSN触发的修改PDP上下文流程中可能修改PDP地址，GnGp SGSN要保存GGSN下发的消息Update PDP Context Request中End user address里新增的PDP Type及其对应的PDP Address。
- Service Request
  UE和网络侧发起的Service Request流程，重建RAB，如果PDP上下文建立IPv4v6双栈，RAB Assignment Request消息中携带PDP Type Information Extension信元。
- Serving RNS Relocation流程
    - 旧侧：如果UE在旧侧的PDP上下文建立的是IPv4v6双栈，在消息Forward Relocation Request中的PDP上下文中包含IPv4地址和IPv6地址。
    - 新侧：新侧在重建承载时启动[IPv4v6双栈控制策略](#ZH-CN_CONCEPT_0248043389__p7)。如果重建保留IPv4v6，GnGp SGSN发送Relocation Request消息给RNC，消息中携带PDP Type Information extension信元。
- Handover
  如果UE在旧侧的PDP上下文建立的是IPv4v6双栈，在消息Forward Relocation Request中的PDP上下文中包含IPv4地址和IPv6地址。

**PDN连接涉及的IPv4v6流程（适用于4G）**

PDN连接涉及的IPv4v6流程如 [图4](#ZH-CN_CONCEPT_0248043389__fig2) 所示。

**图4** PDN连接涉及的IPv4v6流程

<br>

![](特性概述_48043389.assets/zh-cn_image_0248043364_2.png)

此处只针对IPv4v6双栈控制流程作如下描述：

1. 通过[签约数据修改控制策略](#ZH-CN_CONCEPT_0248043389__p6)，使得该用户可以支持IPv4v6业务。
2. UE向MME发送PDN连接请求后，MME根据APN选择一个PGW-C，建立默认承载，并且为默认承载分配承载ID。PGW-C选择完后，如果UE请求的是IPv4v6双栈，则启动[IPv4v6双栈控制策略](#ZH-CN_CONCEPT_0248043389__p7)。
3. PDN Type确定为IPv4v6双栈后，MME发送Create Session Request消息给SGW-C，消息中携带双栈标识（Dual Address Bearer Flag=1）和PDN Type。SGW-C将此消息发送给PGW-C。PGW-C回复Create Session Response给SGW-C，并透传给MME，携带双栈信息，包括PDN Type和PDN Address。PGW-C可以接受双栈请求，也可以拒绝双栈请求，当拒绝时，双栈会被纠正为单栈，PGW-C直接下发单栈地址给UE，并携带原因值如下：
    - “New PDP type due to network preference”，表示PGW-C选择了一个与UE请求不一致的PDN Type。
    - “New PDP type due to single address bearer only”，表示UE请求双栈，PGW-C支持IPv4和IPv6，但不支持双栈。
4. MME发送Setup Bearer Request消息给eNodeB，该消息中还包含一条Nas消息即Activate Default EPS Bearer Request。如果是IPv4v6双栈，Nas消息中携带双栈信息。
5. eNodeB向MME转发Activate default EPS bearer context accept消息。
6. SGW-C收到MME的Modify Bearer Request （EPS Bearer Identity）消息，建立S1-U缺省承载，并建立S1-U和S5/S8-U缺省承载的映射关系。
7. SGW-C向MME发送Modify Bearer Response （EPS Bearer Identity）。
8. IPV6场景下，PGW-C生成IPv6路由广播（前缀）并且经MME将其发送到UE。

**双栈的地址分配方式**

UNC 为用户创建一个上下文，此上下文中会同时分配两个IP地址，一个IPv4地址和一个IPv6地址。

UNC 支持静态地址分配方式IPv4v6双栈地址，指移动用户所分配的IP地址是UE在HSS注册分组业务时确定的IPv4v6地址（静态地址），在上下文激活过程中直接在信元中携带IPv4v6地址，通过激活请求消息发送给PGW-C。

#### [计费与话单](#ZH-CN_CONCEPT_0248043389)

针对双栈用户，在离线计费中GCDR和EGCDR话单只支持携带一个IPv4或者IPv6字段。
