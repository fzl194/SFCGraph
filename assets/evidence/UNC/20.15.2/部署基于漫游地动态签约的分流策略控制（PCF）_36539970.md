## [操作场景](#ZH-CN_OPI_0000001136539970)

当5G用户在漫游地签约本地专网业务，当用户进入专网区域后能自动激活专网业务提供的网络服务，以对接华为UNC为例，专网业务内容定义如下：

- 用户在漫游地进入特定专网区域（如NR Tai Tac=123）后可享受专网业务。
- 当用户进入专网区域后，PCF会动态下发专网DNN=dnn-ar给SMF。
- SMF基于新的DNN激活dnn-ar绑定的ULCL分流规则，用户使用园区专网提供的网络服务。

## [对系统的影响](#ZH-CN_OPI_0000001136539970)

本特性对系统无影响。

## [操作步骤](#ZH-CN_OPI_0000001136539970)

1. 登录UPCF Web UI界面。
2. 配置条件组。
  导航入口为 “ 策略管理 > 策略 > 条件组 ” 。
  > **说明**
  > - 当业务的使用需要设置控制条件时，需要配置条件组。若某规则不配置条件组，表示在同一业务里其他条件都匹配不上时，才触发此规则。
  > - 条件组是一个或多个条件的组合。条件组中的各条件之间可以是 “与” 的关系，也可以是 “或” 的关系。条件之间的与/或关系由参数 “关系” 的取值决定。
  需要配置1个条件组，即用户上线所在位置区域为专网区域场景，当用户位于非专网区域时，当前策略无需定义相关处理内容，用户可直接使用普通业务。条件组的数据规划如 [表2](#ZH-CN_OPI_0000001136539970__fup_cg) 所示。
  *表2 条件组*

  | 类别 | 参数 | 参数 | 取值样例 |
  | --- | --- | --- | --- |
  | 条件组1 | 基本信息 | 名称 | CG_Special |
  | 条件组1 | 基本信息 | 关系 | AND |
  | 条件组1 | 条件 | 名称 | CG_Special |
  | 条件组1 | 条件 | 对象属性 | - 对象：AmPolicySession<br>- 属性：NrTaiTac |
  | 条件组1 | 条件 | 操作符 | Equal |
  | 条件组1 | 条件 | 右值类型 | Value |
  | 条件组1 | 条件 | 右值 | 123<br>说明：“123”<br>：表示用户在漫游地进入专网区域后接入5G网络对应的小区TAC标识 |
3. 配置5G动作组。
  导航入口为 “ 策略管理 > 策略 > 5G动作组 ” 。
  5G动作组的数据规划如 [表3](#ZH-CN_OPI_0000001136539970__fup_ag) 所示。
  *表3 5G动作组*

  | 类别 | 参数 | 参数 | 取值样例 |
  | --- | --- | --- | --- |
  | 5G动作组1 | 基本信息 | 名称 | AG_SpecialDnn |
  | 5G动作组1 | 基本信息 | 类型 | AmPolicyAction |
  | 5G动作组1 | 属性 | LBOService | - 类型：Value<br>- 值：dnn-ar说明：- 该参数取值须与对端网元协商。<br>- 该参数配置当前仅支持由“-”、“.”、“0～9”、“a～f”以及“A～F”组成的长度为1～63字节的字符串，且不能以“.”开头、不能出现连续两个“.”。 |
4. 配置规则
  导航入口为 “ 策略管理 > 策略 > 规则 ” 。
  > **说明**
  > 此场景，规则的 “类型” 需要配置为 “5G Amf Pcc Rule” 。
  规则的数据规划如 [表4](#ZH-CN_OPI_0000001136539970__fup_rule) 所示。
  *表4 规则*

  | 类别 | 参数 | 参数 | 取值样例 |
  | --- | --- | --- | --- |
  | 规则1 | 基本信息 | 名称 | Rule_Dnn |
  | 规则1 | 基本信息 | 操作类型 | Change Rule |
  | 规则1 | 基本信息 | 类型 | 5G Amf Pcc Rule |
  | 规则1 | 条件组 | 关系 | AND<br>说明：当规则中只有一个条件组时，<br>“关系”<br>可设置为任意值。 |
  | 规则1 | 条件组 | 条件组 | CG_Special<br>说明：已在<br>[表2](#ZH-CN_OPI_0000001136539970__fup_cg)<br>中定义。 |
  | 规则1 | 5G动作组 | 5G动作组 | AG_SpecialDnn<br>说明：已在<br>[表3](#ZH-CN_OPI_0000001136539970__fup_ag)<br>中定义。 |
5. 配置策略。
  导航入口为 “ 策略管理 > 策略 > 策略 ” 。
  策略的数据规划如 [表5](#ZH-CN_OPI_0000001136539970__fup_policy) 所示。
  *表5 策略*

  | 类别 | 参数 | 参数 | 取值样例 |
  | --- | --- | --- | --- |
  | 策略1 | 基本信息 | 名称 | Policy_Dnn |
  | 策略1 | 基本信息 | 策略类型 | N15 Policy |
  | 策略1 | 触发器 | 名称 | - N15UserLocationChange<br>- N15PolicyCreateRequest |
  | 策略1 | 规则 | 名称 | Rule_Dnn<br>说明：已在<br>[表4](#ZH-CN_OPI_0000001136539970__fup_rule)<br>中定义。 |
6. 配置业务。
  导航入口为 “ 业务管理 > 业务 > 业务 ” 。
  业务的数据规划如 [表6](#ZH-CN_OPI_0000001136539970__fup_service) 所示。
  *表6 业务*

  | 类别 | 参数 | 参数 | 取值样例 |
  | --- | --- | --- | --- |
  | 业务1 | 基本信息 | 名称 | S_SelectedDnn |
  | 业务1 | 基本信息 | 类型 | VRM_BD_SERVICE_LOCAL |
  | 业务1 | 基本信息 | 是否记入基本配额 | 是 |
  | 业务1 | 基本信息 | 激活方式 | PCEF |
  | 业务1 | 基本信息 | 优先级 | 0<br>说明：运营商统一规划，用于设置该业务在互斥组中的优先级。当此业务不需配置相关互斥组时，此值可任意取值。 |
  | 业务1 | 基本信息 | QoS模式 | 替换 |
  | 业务1 | 基本信息 | 签约后才能使用 | 是 |
  | 业务1 | 策略 | 名称 | Policy_Dnn<br>说明：已在<br>[表5](#ZH-CN_OPI_0000001136539970__fup_policy)<br>中定义。 |
7. 登录“UPCF 业务管理”窗口。
8. 发放业务。
    a. 执行 **ADD PSUB** 增加一个用户。
      > **说明**
      > “USRIDENTIFIER” 的参数类型受软参VRM_SUBSRCIPTIONIDTYPE（系统支持的用户类型）控制，默认为 “IMSI” 。
      ```
      ADD PSUB: USRIDENTIFIER="46010381012****", USRMSISDN="1234567****", USRSTATE=Normal, USRPAIDTYPE=NULL, USRSTATION=Master, USRCONTACTMETHOD=SMS, USRBILLCYCLEDATE=NULL, USRSUBNETTYPE=
      LocalService
      ;
      ```
    b. 执行 **ADD PSRV** 给用户签约业务。
      ```
      ADD PSRV: USRIDENTIFIER="46010381012****", SRVNAME="
      S_SelectedDnn
      ", SRVUSAGESTATE=Normal, SRVROAMINGTYPE=NULL;
      ```

## [任务示例](#ZH-CN_OPI_0000001136539970)

任务描述

配置业务。

脚本

//添加条件组（CG_Special），即用户上线所在位置区域为专网区域。

```
ADD CONDITIONGROUP:CONDITIONGROUPNAME="CG_Special",RELATION=AND;
ADD CONDITION:CONDITIONGROUPNAME="CG_Special",CONDITIONNAME="90895_0101_C01",OBJECTATTRIBUTE="AmPolicySession.NrTaiTac",OPERATOR=Equal,RIGHTVALUETYPE=Value,RIGHTVALUE="123";
```

//添加动作组（AG_SpecialDnn）及动作组子参数。

```
ADD NGACTION:ACTIONNAME="AG_SpecialDnn",TYPE="AmPolicyAction";
ADD NGACTIONATTR:ACTIONNAME="AG_SpecialDnn",ATTRNAME="LBOService",TYPE=Value,VALUE="dnn-ar";
```

//添加规则（Rule_Dnn），用于关联已配置的条件组和动作组。

```
ADD RULE:RULENAME="Rule_Dnn",TYPE=Amf Pcc Rule 5G,RELATION=AND;
ADD RULECONDITIONGROUP:RULENAME="Rule_Dnn",CONDITIONGROUPNAME="CG_Special";
ADD RULENGACTION:RULENAME="Rule_Dnn",ACTIONNAME="AG_SpecialDnn";
```

//将规则关联到已配置的策略（Policy_Dnn）中。

```
ADD POLICY:POLICYNAME="Policy_Dnn",TYPE=N15 Policy;
ADD POLICYRULE:POLICYNAME="Policy_Dnn",RULENAME="Rule_Dnn";
ADD POLICYTRIGGER:POLICYNAME="Policy_Dnn",TRIGGERNAME="N15UserLocationChange";
ADD POLICYTRIGGER:POLICYNAME="Policy_Dnn",TRIGGERNAME="N15PolicyCreateRequest";
```

//将策略添加到业务（S_SelectedDnn）中。

```
ADD SERVICE:SERVICENAME="S_SelectedDnn",SERVICETYPE=VRM_BD_SERVICE_LOCAL,ACTIVATEDBY=PCEF,PRECEDENCE=0,QOSMODE=Replace,SUBSCRIPTIONFORCED=Yes,PROBILLDATE=No,QUOTAALLOCATIONMODE=None,ISSETSHAREDMODE=No,RESUBSCRIPTIONOPERATION=NULL,ISGROUPSERVICE=No,ISTEMPLATE=No,ISRECUSRLOC=No,HASBASESERVICE=No,ADDONLEVEL=Exhaust,CALCVALIDFORDYNSUBGROUP=No;
ADD SERVICEPOLICY:SERVICENAME="S_SelectedDnn",POLICYNAME="Policy_Dnn";
```
