# 调测公私网协同访问（含共享UPF选择）

- [操作场景](#ZH-CN_OPI_0315100010__1.3.1)
- [必备事项](#ZH-CN_OPI_0315100010__1.3.2)
- [操作步骤](#ZH-CN_OPI_0315100010__1.3.3)

## [操作场景](#ZH-CN_OPI_0315100010)

当已经部署公私网协同访问特性时，需要对本功能进行调测，确认本功能可以正常使用。

## [必备事项](#ZH-CN_OPI_0315100010)

前提条件

- 请仔细阅读[WSFD-223002 公私网协同访问（含共享UPF选择）特性概述](WSFD-223002 公私网协同访问（含共享UPF选择）特性概述_15100007.md)。
- 已经完成[激活公私网协同访问（含共享UPF选择）](激活公私网协同访问（含共享UPF选择）_15100009.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息 | 国际移动用户识别码（IMSI） | 123000123456789 | 测试终端自带 | - |
| 用户DNN信息 | DNN名称（DNN） | dnn1 | 测试终端自带 | - |
| 用户切片信息 | 切片/服务类型（SST）<br>切片分类器（SD） | SST=1<br>SD=010101 | 测试终端自带 | - |
| 用户位置信息 | 用户位置信息TAI（用户位置信息TAI） | 1230320101 | 测试终端自带 | - |

## [操作步骤](#ZH-CN_OPI_0315100010)

1. 终端使用“必备事项”中的“数据”接入公网网络，建立PDU会话。
2. 进入 “MML命令行-UNC” 窗口。
3. 查询用户PDU会话的会话ID。
  **[DSP PDUSESSION](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/查询PDP上下文信息/显示PDU会话（DSP PDUSESSION）_09653215.md)**
  预期结果：查询到PDU会话ID作为后续详细查询的PDU会话输入参数。
  ```
  DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=SIMPLE;
  RETCODE = 0  操作成功  

  结果如下 
  --------------  
  IMSI             PDU会话ID  APN或者DNN  请求的APN名称/DNN名称  无线接入类型  QoS流ID  QCI或者5QI  QoS流ARP优先级  QoS流ARP抢占能力  QoS流ARP被抢占能力  SNssaiSst/SNssaiSd  QosFlow 5QI 优先级  

  123000123456789  5          Huawei.com  Huawei.com             NR            1        5           1               MAY_PREEMPT       PREEMPTABLE         1/010101            NULL  

  (结果个数 = 1) 
  ---    END
  ```
4. 查询用户PDU会话的用户面信息。
  **[DSP PDUSESSION](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/查询PDP上下文信息/显示PDU会话（DSP PDUSESSION）_09653215.md)**
  预期结果：PDU会话信息中的用户面地址是预期为用户选择的共享UPF地址。
  ```
  DSP PDUSESSION:QUERYTYPE=IMSI, IMSI="123000123456789", DSPINFOTYPE=DETAILED, WLNETWKTYPE=NW5G, PDUSESSIONID=5, QFI=1;
  RETCODE = 0  操作成功  

  结果如下 
  --------------  
              国际移动用户标识 = 123000123456789
                              ...
               用户位置信息TAI = Huawei.com
                              ...
                        UPF ID = UPF_Instance_0
                              ...
             UP的PFCP IPv4地址 = 192.168.126.11
                              ...
  (结果个数 = 1) 
  ---    END
  ```
5. 终端使用“必备事项”中的“数据”接入私网网络，建立PDU会话。
6. 进入 “MML命令行-UNC” 窗口。
7. 参考 [步骤3](#ZH-CN_OPI_0315100010__cmd43086423411) 和 [步骤4](#ZH-CN_OPI_0315100010__cmd15716531164615) 查询用户PDU会话的信息。
  预期结果：查询到PDU会话的DNN信息、用户面信息与接入公网时一致，用户可以使用同一个DNN访问公网以及私网业务。
