**Attach/TAU**

本特性Attach/TAU流程如 [图1](#ZH-CN_TOPIC_0277260992__fig02) 和 [图2](#ZH-CN_TOPIC_0277260992__fig03) 所示，相较于传统4G流程MME侧做了如下特殊处理：

- 终端和MME间通过Attach/TAU流程协商数据传输方式。终端在Attach Request/TAU Request消息中携带UE network capability信元，指示终端是否支持CP-CIoT数传方式，MME本地可基于TAI配置eNodeB的CP-CIoT的能力（通过命令 [**ADD TAICIOT**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/CIoT能力配置/增加CIoT能力配置(ADD TAICIOT)_26305590.md) 配置实现）。
-
  终端接入时MME通过当前接入的TAI查询其对应的RAT是否为NB-IoT RAT，以确认终端是否通过NB-IoT RAN接入，当对端HSS支持NB-IoT RAT时，还需结合来自HSS的签约数据对终端进行接入控制。

  > **说明**
  > 当对端HSS支持NB-IoT RAT时需将命令 [**SET DMCMPT**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 中的参数 “NBIOTRAT（是否支持NBIoT RAT）” 设为 “SUPPORT” 。此时MME将结合来自HSS的Update Location Answer消息中的信元ARD（Access Restriction Data）的比特位“NB-IoT Not Allowed”对终端进行接入控制。
- 在Create Session Request/Modify Bearer Request消息中，MME会把S11-U的TEID以及NB-IoT RAT Type通知SGW-U，用于为终端创建基于S11-U接口的用户面承载。
- 在Attach/TAU流程中MME是否给eNodeB发送Connection Establishment Indication消息受控于软参 [**DWORD_EX28 BIT27**](../../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（SGSN_MME）/参数说明/移动性管理/DWORD_EX28 BIT27 控制对于CP CIOT模式下的NB-IoT用户，MME是否在Atta_eea6a0b6_05210953.md) 。
- 不发送Initial Context Setup Request消息通知eNodeB建立E-RAB承载，而是通过Downlink NAS Transport消息将Attach Accept/TAU Accept消息直接传递给终端。

**基于CP-CIoT的数据传输流程**

基于NAS信令的小数据传输流程包括终端发送数据流程 **MO（Mobile Originated）** 和接收数据流程 **MT（Mobile Terminated）** ，以下简称MO流程和MT流程。

NAS为终端和MME间的信令控制消息，为配合IoT终端对低功耗的诉求，在NAS层面增加了信元处理。这种机制下终端会通过Release assistance indication信元向MME指示释放S1连接的时机，以控制S1连接的占用时间，减少无效等待时间。

MO流程如 [图3](#ZH-CN_TOPIC_0277260992__fig04) 所示。

- Idle态终端需要发送上行数据时，在RRC连接建立完成后通过Control Plane Service Request消息将上行数据携带至MME。当终端进入连接态后，则通过ESM data transport消息传递上、下行数据。
- MME可根据终端在ESM data transport消息中携带的Release assistance indication信元指示eNodeB何时释放S1连接。
    - 若ESM data transport消息中未携带Release assistance indication信元，则同传统4G处理，由eNodeB触发释放S1连接。
    - 若ESM data transport消息中携带了Release assistance indication信元，该信元指示“Downlink data transmission subsequent to the uplink data transmission is not expected”时，则表示上行数据没有对应的下行应答，MME在将数据传给SGW-U后立即通知eNodeB释放S1连接。否则该信元指示“Downlink data transmission subsequent to the uplink data transmission is expected”时，则表示需要等待下行应答，MME在收到下行数据后再释放S1连接。
      > **说明**
      > 当ESM data transport消息中携带的Release assistance indication信元指示 “Downlink data transmission subsequent to the uplink data transmission is expected” 时，MME最长会等待15秒。若15秒内收不到下行数据，且此时eNodeB未触发释放S1连接，MME将主动触发释放S1连接。
