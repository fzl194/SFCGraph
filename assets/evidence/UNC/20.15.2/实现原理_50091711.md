#### [I-SMF选择策略](#ZH-CN_CONCEPT_0250091711)

- 移动性注册更新、Service Request、Xn Handover、N2 Handover流程，基于当前I-SMF（Intra-AMF的切换）或者老侧I-SMF（Inter-AMF的切换）是否存在，根据UE的位置信息存在不同的I-SMF选择策略，详细见[表1](#ZH-CN_CONCEPT_0250091711__table133838334185)：

*表1 I-SMF选择策略表1*

| 场景 | I-SMF是否存在 | UE是否在SMF服务区域 | UE是否在当前I-SMF服务区域 | I-SMF选择策略 |
| --- | --- | --- | --- | --- |
| 1 | 是 | 否 | 是 | 不变 |
| 2 | 是 | 否 | 否 | 变更 |
| 3 | 是 | 是 | 是 | 删除 |
| 4 | 是 | 是 | 否 | 删除 |
| 5 | 否 | 是 | 不涉及 | 不涉及 |
| 6 | 否 | 否 | 不涉及 | 新增 |

> **说明**
> 在触发I-SMF插入流程后，终端继续移动触发Xn或N2切换时还可能伴随I-SMF的改变或者I-SMF不变仅I-UPF改变。
>
> - I-SMF改变，新侧I-SMF收到请求后会向老侧I-SMF获取并更新用户上下文，建立新的用户面通路。
> - I-SMF不变I-UPF改变，I-SMF选择新的I-UPF，并将新I-UPF的地址信息更新给锚点SMF，锚点SMF辅助建立至PSA UPF的新的用户面通路。

- 会话建立、4G到5G Handover流程、4G到5G移动性注册更新流程，均只涉及到I-SMF的新增，如[表2](#ZH-CN_CONCEPT_0250091711__table0467134161920)所示。

*表2 I-SMF选择策略表2*

| 场景 | I-SMF是否存在 | UE是否在SMF服务区域 | UE是否在当前I-SMF服务区域 | I-SMF选择策略 |
| --- | --- | --- | --- | --- |
| 1 | 否 | 是 | 不涉及 | 不涉及 |
| 2 | 否 | 否 | 不涉及 | 新增 |

- 5G到4G Handover流程、5G到4G的TAU流程，只涉及到I-SMF的删除。
- 初始注册流程、周期注册更新、紧急注册都不涉及I-SMF的选择。
- 去注册流程、AN release、PDU会话修改、PDU会话释放、I-SMF都不变化。

步骤15： I-SMF向锚点SMF发送Nsmf_PDUSession_Update Response，该响应消息中携带UE向I-SMF发送的PDU会话修改命令Ack消息等。

- AMF发起PDU会话释放请求
