#### [适用NF](#ZH-CN_TOPIC_0160374912)

GGSN、SGW-C、PGW-C、SMF

#### [定义](#ZH-CN_TOPIC_0160374912)

随着智能终端快速普及，OTT（Over The Top）业务急剧增长，服务本地化、多样化的趋势愈来愈明显，传输资源消耗快速增长，对端到端的业务访问时延体验要求越来越高。传统的网关集中部署模式，已无法满足容量性能持续增长以及多样业务切片的需求。

CUPS（Control and User Plane Separation）架构即控制面与用户面分离架构，如 [图1](#ZH-CN_TOPIC_0160374912__fig1449571213390) 所示，通过提供GGSN-C/SGW-C/PGW-C/SMF和GGSN-U/SGW-U/PGW-U/UPF两种独立的NF形态实现控制面和用户面功能的分离，满足云化场景下控制面和用户面资源的不同要求。

#### [客户价值](#ZH-CN_TOPIC_0160374912)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 用户面贴近用户地理位置，业务数据可通过本地的CDN（Content Delivery Network）传输，不经过骨干网，缩短传输路径，降低传输成本。<br>- 支持用户面单独部署某种业务，满足不同业务对网络带宽、时延和可靠性的不同要求，提升业务质量。<br>- 平滑向5G演进，保护投资，节省成本。<br>- 一个控制面可以管理多个用户面，简化运维和组网。 |
| 用户 | 享受低时延业务体验。 |

#### [应用限制](#ZH-CN_TOPIC_0160374912)

- 不支持Non-3GPP接入。
- 不支持S4-SGSN接入。
- 仅支持与遵从R15及以上协议版本的Sx/N4接口交互。
- GGSN-C/SGW-C/PGW-C/SMF和GGSN-U/SGW-U/PGW-U/UPF设备必须同时为HUAWEI设备。

#### [原理概述](#ZH-CN_TOPIC_0160374912)

按照控制面与用户面分离理念，2/3G核心网的GGSN逻辑网元功能分裂为控制面（GGSN-C）和用户面（GGSN-U），4G核心网的SGW和PGW逻辑网元功能分裂为控制面（SGW-C、PGW-C）和用户面（SGW-U、PGW-U），5G核心网相应NF分为控制面（SMF）和用户面（UPF）。

**图2** 2/3/4G核心网CUPS逻辑架构

![](WSFD-107001 支持CUPS架构_60374912.assets/zh-cn_image_0169048437_2.png)

如 [图2](#ZH-CN_TOPIC_0160374912__fig189103136349) 所示，GGSN-C和GGSN-U分别与PGW-C、PGW-U合一部署，通过Gn/Gp-C、Gn/Gp-U接口与SGSN对接。SGW-C/PGW-C和SGW-U/PGW-U之间通过Sxa和Sxb接口交互。SGW-C和PGW-C位于核心层或汇聚层，SGW-U和PGW-U位于汇聚层或边缘。SGW-C和PGW-C可以独立部署或合一部署，SGW-U和PGW-U合一部署。

**图3** 5G核心网CUPS逻辑架构

![](WSFD-107001 支持CUPS架构_60374912.assets/zh-cn_image_0169047082_2.png)

如 [图3](#ZH-CN_TOPIC_0160374912__fig43014412447) 所示，SMF和UPF间通过N4接口交互。SMF可灵活部署在核心层或汇聚层，UPF部署在汇聚层。

CUPS各NF功能如 [表1](#ZH-CN_TOPIC_0160374912__table13959105111147) 所示。

*表1 CUPS逻辑NF功能*

| 逻辑NF | 主要功能 |
| --- | --- |
| SGW-C | 接入控制：<br>- 集中出信令接口到周边网元（MME、PGW-C、CG）。<br>- S1-U和S5/S8-U GTPU隧道的IP地址和TEID分配。<br>- 选择SGW-U。<br>- GTP接入信令处理。<br>- 用户会话承载（缺省和专有承载）管理。<br>- eNodeB间切换移动信令面锚点。<br>- ECM-IDLE用户寻呼消息转发。<br>- 基于GTP/PFCP协议的路径管理。<br>计费与策略控制：<br>- 跨运营商漫游计费（SGW-CDR话单）。 |
| SGW-U | 数据处理：<br>- S1-U和S5/S8-U GTP-U隧道封装解封装处理，S1-U到S5/S8-U的用户数据转发。<br>- 传输级报文标记。<br>- eNodeB间切换移动数据锚点（转发"end marker"、切到目标节点后发送"end marker"）。<br>- ECM-IDLE状态下触发DDN，并缓存下行用户。<br>数据路径管理：<br>- 基于GTP/PFCP协议的路径管理。<br>计费与策略控制：<br>- 用户QoS控制。<br>- 用户/PDP粒度的计费信息收集，包括Home Routed漫游场景。 |
| GGSN-C/PGW-C | 接入控制：<br>- 集中出信令接口到周边网元（SGW-C、CG、OCS Server）。<br>- PGW-U选择。<br>- PGW接入信令处理。<br>- 用户会话承载（缺省和专有承载）管理。<br>- 用户地址池分配。<br>- 用户接入鉴权认证控制。<br>- 基于GTP/PFCP协议的路径管理。<br>计费与策略控制：<br>- 离线计费话单。<br>- 在线计费配额管理。<br>- PCC相关功能：承载绑定（承载QoS & TFT）、SDF等控制策略、控制面相关事件报告（如RAT变更）。 |
| GGSN-U/PGW-U | 数据处理：<br>- S5/S8-U GTP-U隧道封装解封装处理，接入侧到Gi/SGi侧的用户数据转发。<br>- 传输级报文标记。<br>- 跨SGW切换切到目标节点后，发送"end marker"。<br>- 在上行报文中核查UE仅使用分配的IP地址。<br>路径管理：<br>- 基于GTP/PFCP协议的路径管理。<br>计费与策略控制：<br>- 上下行业务级计费数据收集（在线/离线等）。<br>- PCC相关功能：业务探测（SA，五元组）、上下行业务级门控、上下行业务级MBR执行、用户面相关事件报告（如应用探测报告）、重定向。 |
| SMF | 主要进行会话管理，支持如下全部或者部分功能：<br>- PDU会话建立、修改和释放，包括UPF和AN节点之间的隧道维护。<br>- UE IP地址分配和管理（包括可选授权）。<br>- 在UPF配置流量定向，将流量引到正确目的地。<br>- 终止对策略控制功能的接口。<br>- 计费数据收集和计费接口支持。<br>- 控制和协调UPF计费数据收集。<br>- 终止NAS消息的SM部分。<br>- 下行数据通知。<br>- AN特定SM信息的发起者，通过AMF通过N2发送给AN。<br>- 确定会话的SSC模式。 |
| UPF | 主要是用户面功能，支持以下全部或者部分功能：<br>- Intra / Inter-RAT移动性的锚点。<br>- 将PDU会话与DN(数据网络)互连。<br>- 分组路由和转发。<br>- 数据包检测。<br>- 策略规则执行的用户平面部分。<br>- 用量上报。<br>- 用户面的QoS处理。<br>- 上行链路流量验证（SDF到QoS流量映射）。<br>- 上行链路和下行链路中的传输级别数据包标记。<br>- 下行链路数据缓存和下行链路数据通知触发。<br>- 发送和转发一个或多个"end marker"到源NG-RAN节点。 |

#### [计费与话单](#ZH-CN_TOPIC_0160374912)

- GGSN-U/PGW-U收集在线计费的统计信息，上报给GGSN-C/PGW-C，GGSN-C/PGW-C汇总信息后，向OCS上报。
- GGSN-U/SGW-U/PGW-U收集离线计费的统计信息，上报给GGSN-C/SGW-C/PGW-C，GGSN-C/SGW-C/PGW-C完成离线计费功能，并生成话单发送至CG。
- UPF收集带或不带配额管理的融合计费统计信息，上报给SMF，SMF汇总信息后，向CHF上报。
