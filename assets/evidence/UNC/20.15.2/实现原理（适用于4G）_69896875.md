#### [设备标识检查流程](#ZH-CN_TOPIC_0169896875)

在Attach流程中，设备标识检查流程如 [图1](#ZH-CN_TOPIC_0169896875__fig15860192218239) 所示：

**图1** Attach流程中，设备标识检查流程

<br>

![](实现原理（适用于4G）_69896875.assets/zh-cn_image_0000001238746899_2.png)

流程描述：

1. UE发起附着请求流程。
2. MME向UE发送Identity Request或Authentication and Ciphering Request消息，请求UE的IMEI信息。
3. MME收到UE发送的Identity Response或Authentication and Ciphering Response消息（携带UE的IMEI信息）。
4. MME向EIR发送ME Identity Check Request消息（携带UE的IMEI信息），请求检查IMEI的合法性。
5. EIR检查IMEI的合法性后，向MME回复ME Identity Check Answer消息，MME根据该消息中携带的Result和Equipment Status信元进行相应的处理。
    - 如果Result信元值为Success，MME进一步根据Equipment Status信元值进行处理：
          - 如果Equipment Status信元值为WHITELISTED（0），表示该用户为白名单用户，MME允许用户接入网络。
          - 如果Equipment Status信元值为BLACKLISTED（1），表示该用户为黑名单用户，MME根据[**MOD S1IMEICFG**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/设备检查管理/S1模式IMEI配置/修改S1模式IMEI配置(MOD S1IMEICFG)_26145638.md)命令的参数“BLALLOW”决定是否允许用户接入网络，一般配置为拒绝用户，不再进行Attach流程。（向UE发送Attach Reject消息，原因值为“Illegal ME”。）
          - 如果Equipment Status信元值为GREYLISTED（2），表示该用户为灰名单用户，MME根据[**MOD S1IMEICFG**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/设备检查管理/S1模式IMEI配置/修改S1模式IMEI配置(MOD S1IMEICFG)_26145638.md)命令的参数“GRALLOW”决定是否允许用户接入网络，一般配置为允许用户接入网络，但是出于评估或其它目的，用户将被网络跟踪。
    - 如果Result信元值为User error，表示因用户原因（如用户在EIR中未注册）IMEI检查失败，MME根据[**MOD S1IMEICFG**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/设备检查管理/S1模式IMEI配置/修改S1模式IMEI配置(MOD S1IMEICFG)_26145638.md)命令的参数“USERERR”决定是否允许用户接入网络，一般配置为允许用户接入网络。
    - 如果Result信元值为Provider error，表示因设备原因（如资源受限）IMEI检查失败，MME根据[**MOD S1IMEICFG**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/设备检查管理/S1模式IMEI配置/修改S1模式IMEI配置(MOD S1IMEICFG)_26145638.md)命令的参数“PROVIDERR”决定是否允许用户接入网络，一般配置为允许用户接入网络。
6. MME按3GPP TS 23.401协议描述完成后续Attach流程的处理。

> **说明**
> - 如果MME中未保存UE的IMEI信息，MME需要首先通过Identity流程或SMC流程获取到UE IMEI信息后，再完成IMEI检查流程。
> - 如果用户未在MME中配置相关用户的IMEI检查策略，MME不进行IMEI检查流程，直接完成Attach流程的相关处理。
> - IMEI检查流程对应到Diameter应用中的ME-Identity-Check-Request/Answer（ECR/ECA）。

#### [设备标识检查功能查表流程](#ZH-CN_TOPIC_0169896875)

设备标识检查功能查表流程如 [图2](#ZH-CN_TOPIC_0169896875__fig2) 所示

**图2** 设备标识检查功能查表流程

<br>

![](实现原理（适用于4G）_69896875.assets/zh-cn_image_0000001194066928_2.png)

1. Attach流程中，在S1IMEICFG表中查找是否存在与用户IMSI前缀匹配的记录？
    - 是 => 步骤[2](#ZH-CN_TOPIC_0169896875__li2)
    - 否 => 步骤[3](#ZH-CN_TOPIC_0169896875__li3)
2. 记录中“IMEI检查策略”是否为“IMEISV”或“IMEI”？
    - 是 => 步骤[4](#ZH-CN_TOPIC_0169896875__li4)
    - 否 => Attach中不进行检查IMEI
3. 检查系统缺省记录中，“IMEI检查策略”是否为“IMEISV”或“IMEI”？
    - 是 => 步骤[4](#ZH-CN_TOPIC_0169896875__li4)
    - 否 => Attach中不进行检查IMEI
4. MME是否保存有用户的IMEI信息？
    - 是 => Attach中进行检查IMEI
    - 否 => 步骤[5](#ZH-CN_TOPIC_0169896875__li5)
5. MME向UE发起获取IMEI流程。 => Attach中进行检查IMEI
