# 过程1：请求信息纠正前的预处理

请求信息纠正前的预处理包括特定APN拒绝和APN NI转换。

- 特定APN拒绝
  如果运营商需要限制某一特定APN或某一类APN不允许接入网络。例如，指定请求APN为空的用户不允许接入网络，此时，可以通过特定APN拒绝功能配置APN为空的用户拒绝接入。
- APN NI转换
  如果运营商需要对指定用户进行APN转换。例如，某运营商对用户的签约APN进行了归一化改造，但终端用户请求的APN无法进行统一，此时，为避免PDP/PDN激活请求被拒，可以通过APN NI转换将终端用户请求的APN转换为归一化改造后的APN。

数据匹配前请求信息纠正流程说明如下：

1. 系统检查UE请求的APN是否为需要直接拒绝的特定APN。特定APN通过[**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md)命令指定。
    - 是 => 拒绝PDP/PDN激活请求。
    - 否 => 执行[步骤2](#ZH-CN_TOPIC_0186228993__step2)。
2. 系统检查UE请求的APN是否需要进行APN NI转换。需要进行转换的请求APN和转换后的APN均通过为[**ADD IMSIAPNCONVERT**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/APNNI转换管理/增加APNNI转换配置(ADD IMSIAPNCONVERT)_72225339.md)命令指定。
    - 是 =>UNC进行APN NI转换。然后，执行[步骤3](#ZH-CN_TOPIC_0186228993__step3)。
    - 否 => 执行[步骤3](#ZH-CN_TOPIC_0186228993__step3)。
3. UNC进行正常的签约数据匹配。
4. 系统检查签约数据匹配是否成功。
    - 是 => 接受PDP/PDN激活请求。
    - 否 => 进入“[过程2：请求信息纠正](过程2：请求信息纠正_86228994.md)”。
