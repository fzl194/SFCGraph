## [操作场景](#ZH-CN_OPI_0193914969)

本操作指导介绍在运行网络中激活Gr接口HTR流控功能的操作过程。

Gr接口HTR（Hard to Reach）流控功能是指当 UNC 的Gr接口业务流量过载而致使其链路拥塞和周边网元过载时，系统能够周期性的对比局向的请求和应答消息并进行统计，来即时自动调节Attach、Inter RAU（包括LTE->GERAN/UTRAN）、Intra RAU（包括LTE->GERAN/UTRAN）流程首次到HLR/STP的取鉴权集或位置更新请求流程的处理速率，从而控制通过Gr接口的业务量，来缓解接口拥塞或周边网元过载，以保证接口最大的业务成功率。

> **说明**
> 适用于 SGSN 。

## [任务示例](#ZH-CN_OPI_0193914969)

任务描述

升级过程中系统重启，预期会有大量的用户注册请求，因此系统重启前开启Gr接口HTR流控功能，打开局向流控开关，设置启动阈值和恢复阈值，当MS接入的请求速率首次超过启控阈值，系统自动调节Attach、Inter RAU（包括LTE->GERAN/UTRAN）、Intra RAU（包括LTE->GERAN/UTRAN）流程的首条消息的接入速率。对于被流控的请求，系统将给MS返回Attach Reject或RAU Reject消息，并携带 “#17 Network failure” 原因值。

脚本

//打开HTR流控开关，设置流控启动阈值为65，流控恢复阈值为85，打开系统自动添加局向功能。

```
SET HTR: HTRSWITCH=YES, HTRMANUALSWITCH=NO, HTRCONFIGOFCSWITCH=NO, HTRFAILPERCENT=65, HTRSUCCPERCENT=85;
```

//设置HTR局向索引和HTR局向名称，打开HTR局向流控开关。

```
ADD HTROFC: HTROFCINDEX=0, HTROFCNAME="HLR001", HTROFCSWITCH=YES;
```

//增加HTR号段信息：设置IMSI前缀和HTR局向索引。

```
ADD HTRIMSICFG: IMSIPRE="123030001", HTROFCINDEX=0;
```
