## [命令功能](#ZH-CN_MMLREF_0000001401605352)

该命令用于查询HTTP对于URL转义的参数。

## [注意事项](#ZH-CN_MMLREF_0000001401605352)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001401605352)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001401605352)

无

## [使用实例](#ZH-CN_MMLREF_0000001401605352)

如果想查询HTTP转义参数，可以用如下命令：

```
%%LST HTTPESCAPE:;%%
RETCODE = 0  操作成功

结果如下
--------
      URL中加号解码成空格开关  =  关闭
丢弃URL中存在特殊字符消息开关  =  开启
     URL中Json类型参数编码开关 =  关闭
     服务端对URL进行解码的时机 =  解析组件前
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001401605352)

| 输出项名称 | 输出项解释 |
| --- | --- |
| URL中加号解码成空格开关 | 该参数用于控制服务端对URL转义解码时，将加号解码成空格的开关状态。 |
| 丢弃URL中存在特殊字符消息开关 | 该参数用于控制服务端对URL的Path参数转义解码后，丢弃存在除大小写字母、数字、“-”、“_”、“.”、“;”、“:”、“/”、“?”以外字符的消息的开关状态。 |
| URL中Json类型参数编码开关 | 该参数用于控制客户端对URL中Json类型的Path参数及Query参数值进行转义编码的开关状态。开关开启后，如果Path参数或Query参数值是Json类型，会将参数值中除数字、大小写字母、“-”、“_”、“.”、“~”之外的其他字符，转换成百分号加上该字符的十六进制ASCII数字，比如“[”、“]”、“{”、“}”、“:”、“\”、“,”分别转换成“%5B”、“%5D”、“%7B”、“%7D”、“%3A”、“%22”、“%2C”。 |
| 服务端对URL进行解码的时机 | 该参数用于控制服务端对URL进行解码的时机。 |
