## [操作场景](#ZH-CN_OPI_0246574621)

本操作指导介绍在运行网络中激活网络侧发起的5G位置定位服务特性的操作过程。

网络侧发起的5G位置定位服务是移动通信网络通过无线信号测量来确定UE地理位置信息及其移动速度的技术，本特性中也称为LCS（Location Services）。

> **说明**
> 适用于AMF。

## [必备事项](#ZH-CN_OPI_0246574621)

- 完成包含此特性的License文件加载。

## [操作步骤](#ZH-CN_OPI_0246574621)

1. 进入 “MML命令行-UNC” 窗口。
2. **可选：** 当现网在UEG-S上部署LCS功能，且AMF网元版本为20.13.2及以上，UEG-S网元版本低于20.13.2时，需关闭对周期定位的ReportingAmount和ReportingInterval信元的校验。
  **[SET AMFSOFTPARAOFBIT](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/软件参数管理/设置AMF软件参数比特位（SET AMFSOFTPARAOFBIT）_09651487.md)**
  > **说明**
  > 在周期性定位或单次定位过程中，若期望AMF向LMF发送Nlmf_Location_DetermineLocation Request消息信元填充前不检查信元的有效性，需要将 “DWORD68 BIT32” 软参设置为 “1” 。
3. 设置LCS定位（MT-LR）的参数。
  [**SET NGLCSPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/5G定位服务管理/定位服务管理/设置5G定位服务参数（SET NGLCSPARA）_44007975.md)
  > **说明**
  > - 如果采用3GPP协议定义的标准LCS组网（部署LMF），进行精准LCS定位，“LCSSW(定位服务功能开关)”为“ON（开启）”，“LCSPLCY(5G定位服务策略)”为“PROTOCOL_POSITION（协议模式定位）”。
  > - 如果采用未部署LMF网元的LCS组网，进行较低精度的LCS定位，“LCSSW(定位服务功能开关)”为“ON（开启）”，“LCSPLCY(5G定位服务策略)”为“NO_LMF_POSITION（无LMF定位）”。
4. **可选：** 若LMF不支持显示订阅LPP、NRPPA等类型时，需开启 [DWORD63 BIT1](../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（AMF）/移动性管理/DWORD63 BIT1 控制5GC定位流程中LMF未向AMF发起位置定位订阅时，AMF是否通过查询N_1f6a1225_11038965.md) 软参以保证定位流程正常运行。
5. **可选：**位置定位安全加固配置。
    a. 建议增加AMF的位置定位服务白名单配置，确保仅限于合法的GMLC访问。
      **[ADD ALLOWSERNAME](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/白名单管理/基于地址的白名单管理/增加基于地址的白名单（ADD ALLOWSERNAME）_83813626.md)**
    b. 建议AMF和GMLC侧开启SBI接口加密，降低网络安全风险。
          - AMF侧开启SBI接口加密请参考[激活SBI接口加密特性(CERT)](../../安全管理功能/WSFD-010308 SBI接口加密/激活SBI接口加密特性(CERT)_70185215.md)。
          - GMLC侧开启SBI接口加密请参考对应的产品描述。
    c. 建议增加定位服务权限，用于AMF收到GMLC的定位请求时检查GMLC的定位请求是否越权，对越权的GMLC，AMF拒绝其定位请求。（用于B2B场景）
      **[ADD LCSPERMITCFG](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/5G定位服务管理/定位服务权限配置管理/增加定位服务权限配置（ADD LCSPERMITCFG）_62595587.md)**
6. **可选：**针对低功耗终端场景，配置周期性注册更新流程的超长时间间隔。
  **[ADD NGPRDREGTIMEDNN](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动管理定时器/增加基于DNN的周期性注册时长配置（ADD NGPRDREGTIMEDNN）_21861953.md)**
  > **说明**
  > 对于低功耗终端，AMF不支持InterAMF切换操作中给基站下发基于DNN配置的T3512。

## [任务示例](#ZH-CN_OPI_0246574621)

任务描述

设置网络侧发起的5G位置定位服务定位参数。

- 场景1：3GPP协议定义的标准LCS组网场景下，部署协议标准的LCS定位（MT-LR）功能。
- 场景2：未部署LMF网元的LCS组网场景下，部署无LMF的LCS定位（MT-LR）功能。

脚本

当AMF网元版本为20.13.2及以上，UEG-S网元版本低于20.13.2时。需关闭对周期定位的ReportingAmount和ReportingInterval信元的校验。

//关闭对周期定位的ReportingAmount和ReportingInterval信元的校验。

```
SET AMFSOFTPARAOFBIT: DT=Dw, DWORDNUM=68, VALUE=VALUE_1, POSITION=32;
```

- 场景1：标准的LCS定位（MT-LR）功能。
  //设置LCS定位的参数，选择5G定位服务策略为协议模式定位。

  ```
  SET NGLCSPARA: LCSSW=ON, POSEXPR=30, LCSPLCY=PROTOCOL_POSITION, LRC=YES, GUAMINOTIFY=NO, LCSIDCHK=NO, LMFRETRY=NO;
  ```
  //可选：若LMF不支持显示订阅LPP、NRPPA等类型时，需开启 “DWORD63 BIT1” 软参以保证定位流程正常。
  执行SET AMFSOFTPARAOFBIT命令设置软参DWORD63中BIT1的取值为“1”。
