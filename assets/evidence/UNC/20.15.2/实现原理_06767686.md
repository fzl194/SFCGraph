# 实现原理

以PDU会话建立流程为例，本章节重点描述基站与UPF之间的接口故障时，SMF选择UPF时如何识别并弃选故障的UPF，其它PDU会话建立详细流程请参考 [PDU会话建立](../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/会话流程/PDU会话建立_97353800.md) 。

**图1** UPF故障隔离流程示例（PDU会话建立）

<br>

![](实现原理_06767686.assets/zh-cn_image_0000001215336908_2.png)

1. UE向AMF发送PDU Session Establishment Request消息，请求建立会话。
2. AMF选择SMF。
3. AMF向选择的SMF发送Nsmf_PDUSession_CreateSMContext Request消息，请求创建SM上下文。Nsmf_PDUSession_CreateSMContext Request消息中包含UeLocation信元，SMF保存其中的gNodeB ID。
4. SMF向AMF返回Nsmf_PDUSession_CreateSMContext Response消息，确认接受创建PDU会话。
5. SMF选择UPF，详细原理可参考[UPF选择原理](../../../../业务专题/UNC UPF选择专题/5G用户访问5G网络时的用户面选择/UPF选择原理_06156304.md)。在选择UPF过程中，SMF根据[3](#ZH-CN_TOPIC_0000001206767686__li754044010516)中保存的gNodeB ID判断是否存在故障的UPF，如存在则弃选故障的UPF，如[图2](#ZH-CN_TOPIC_0000001206767686__fig766183135316)所示。
  **图2** 故障UPF判断机制

  <br>

  ![](实现原理_06767686.assets/zh-cn_image_0000001260177203_2.png)
6. SMF与选择的正常UPF建立连接，提供用于该PDU会话的信息。
