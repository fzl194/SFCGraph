#### [命令功能](#ZH-CN_MMLREF_0000001172225847)

**适用网元：MME**

该命令用于查询S11接口信令风暴抑制功能的控制与统计信息，控制信息包括：抑制用户数控制值、抑制分离速率（个/秒）、抑制功能启控状态，统计信息包括：流程未抑制的次数、流程被抑制的次数、抑制用户数、抑制用户被分离的次数、抑制用户被恢复的次数。

#### [注意事项](#ZH-CN_MMLREF_0000001172225847)

当 [**SET SIGSUPPRESS**](设置S11接口信令风暴抑制功能参数(SET SIGSUPPRESS)_72345767.md) 命令中的 “S11接口信令抑制开关” 从 “OFF(关闭)” 修改为 “ON(开启)” 时，统计信息中的 “流程未抑制的次数” 、 “流程被抑制的次数” 、 “抑制用户被分离的次数” 和 “抑制用户被恢复的次数” 会清零。

#### [本地用户权限](#ZH-CN_MMLREF_0000001172225847)

manage-ug；system-ug；monitor-ug

#### [网管用户权限](#ZH-CN_MMLREF_0000001172225847)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [输出结果说明](#ZH-CN_MMLREF_0000001172225847)

| 输出项名称 | 输出项解释 |
| --- | --- |
| RU名称 | 该参数用于显示SPP进程所属的资源单元名称。<br>取值范围：1～63位字符串 |
| 进程号 | 该参数用于显示业务进程所属的SPP进程序号。<br>取值范围：0～20 |
| 抑制用户数控制值 | 该参数用于显示系统每SPP进程计算抑制用户分离速率所使用的抑制用户数。如果<br>[**SET SIGSUPPRESS**](设置S11接口信令风暴抑制功能参数(SET SIGSUPPRESS)_72345767.md)<br>中配置的“信令抑制配置策略”为“SPECIFIC(自定义值)”，配置的“抑制用户数控制参考值”小于“系统实时的4G用户数”，那么该参数的取值为“系统实时的4G用户数”；否则，该参数的取值为配置的“抑制用户数控制参考值”。当S11接口信令风暴抑制功能处于“启控状态”下时，只有重新设置<br>[**SET SIGSUPPRESS**](设置S11接口信令风暴抑制功能参数(SET SIGSUPPRESS)_72345767.md)<br>命令时，该参数的取值才会刷新，当S11接口信令风暴抑制功能处于“未启控状态”下时，每五分钟刷新一次该参数的取值。<br>取值范围：0～20000000 |
| 抑制分离速率（个/秒） | 该参数用于显示系统每SPP进程对处于抑制状态的用户进行分离的速率。如果<br>[**SET SIGSUPPRESS**](设置S11接口信令风暴抑制功能参数(SET SIGSUPPRESS)_72345767.md)<br>中配置的“信令抑制配置策略”为“DEFAULT(系统缺省值)”，系统将以每SPP进程45次/秒的速率对抑制用户进行分离。否则，该参数的取值通过<br>“抑制用户数控制值”<br>/<br>“抑制时长（分）”<br>获得。<br>取值范围：10～200 |
| 抑制功能启控状态 | 该参数用于显示S11接口信令风暴抑制功能的控制状态。<br>取值范围：<br>- 启控状态：该状态下，S11接口闪断或中断导致的异常流程（例如Intra Handover/X2 Path Switch without S-GW Change、Service Request、Intra TAU/Combined TAU without S-GW Change流程的Modify Bearer Request处理超时，或者Intra Handover/X2 Path Switch、Intra TAU/Combined TAU流程进行上游网元选择失败）都将被抑制，UNC不直接分离用户。<br>- 未启控状态：该状态下，S11接口闪断或中断导致的异常流程速率超过“抑制分离速率（个/秒）”的80%，那么超过80%的异常流程将被抑制处理，其他情况下，不会被抑制，维持流程的常规失败处理。 |
| 流程未抑制的次数 | 该参数用于显示S11接口闪断或中断导致的异常流程（例如Intra Handover/X2 Path Switch without S-GW Change、Service Request、Intra TAU/Combined TAU without S-GW Change流程的Modify Bearer Request处理超时，或者Intra Handover/X2 Path Switch、Intra TAU/Combined TAU流程进行上游网元选择失败）因维持常规流程处理而未被抑制的次数。<br>取值范围：0～4294967295 |
| 流程被抑制的次数 | 该参数用于显示S11接口闪断或中断导致的异常流程（例如Intra Handover/X2 Path Switch without S-GW Change、Service Request、Intra TAU/Combined TAU without S-GW Change流程的Modify Bearer Request处理超时，或者Intra Handover/X2 Path Switch、Intra TAU/Combined TAU流程进行上游网元选择失败）中因进行延迟分离处理而被抑制的次数。<br>说明：延迟分离处理是指MME并不立即分离受影响的用户，而是在延迟一段时间后，再以较低的速率分离这些用户，是S11接口信令风暴抑制功能为避免因批量用户被分离后再次接入引发信令风暴的一种处理方法。<br>取值范围：0～4294967295 |
| 抑制用户数 | 该参数用于显示系统中存在的因S11接口闪断或中断导致的异常流程（例如Intra Handover/X2 Path Switch without S-GW Change、Service Request、Intra TAU/Combined TAU without S-GW Change流程的Modify Bearer Request处理超时，或者Intra Handover/X2 Path Switch、Intra TAU/Combined TAU流程进行上游网元选择失败）中因进行延迟分离处理而被抑制的用户数。<br>取值范围：0～4294967295 |
| 抑制用户被分离的次数 | 该参数用于显示抑制用户被系统主动平滑分离的次数。<br>取值范围：0～4294967295 |
| 抑制用户被恢复的次数 | 该参数用于显示当用户处于抑制状态时，因PDN承载到S-GW重建成功、4G到2/3G的RAU和Relocation、ESR或者SRVCC而被恢复为正常用户的次数。<br>取值范围：0～4294967295 |
