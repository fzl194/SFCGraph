## [命令功能](#ZH-CN_MMLREF_0209652363)

**适用NF：AMF**

该命令用于增加5G寻呼优先级策略。寻呼优先级用以指示NG-RAN应用不同的等级寻呼用户。

## [注意事项](#ZH-CN_MMLREF_0209652363)

- 该命令执行后立即生效。

- 不能为相同的DNN和ARP指定多个寻呼优先级。
- 5G寻呼优先级策略的匹配顺序按照如下顺序递减：1. 同时匹配DNN和ARP；2. 仅匹配DNN；3. 仅匹配ARP。
- 该命令配置的寻呼优先级策略只对SMF向AMF发送的Namf_Communication_N1N2MessageTransfer消息有效。

- 最多可输入128条记录。

#### [操作用户权限](#ZH-CN_MMLREF_0209652363)

G_1，管理员级别命令组；G_2，操作员级别命令组

## [参数说明](#ZH-CN_MMLREF_0209652363)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PLCYIDX | 策略索引 | 可选必选说明：必选参数<br>参数含义：该参数用于在AMF内唯一标识一条寻呼优先级策略。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~127。<br>默认值：无<br>配置原则：无 |
| DNN | 数据网络名称 | 可选必选说明：可选参数<br>参数含义：该参数用于指定应用寻呼优先级的数据网络名称（DNN）。<br>数据来源：全网规划<br>取值范围：字符串类型，输入长度范围是0~63。可输入的字符有字母、十进制数字、“-”和“.”，并且开头和结尾只能是数字或者字母。不能出现连续两个“.”。字母大小写不敏感。<br>默认值：无<br>配置原则：无 |
| ARPPL | ARP优先级 | 可选必选说明：可选参数<br>参数含义：该参数用于指定应用寻呼优先级策略的ARP优先级等级。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是1~15。<br>默认值：无<br>配置原则：无 |
| PAGINGPRIORITY | 寻呼优先级等级 | 可选必选说明：可选参数<br>参数含义：该参数用于指定寻呼优先级等级。数值越小，表示NG-RAN寻呼用户的优先等级越高。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是1~8。<br>默认值：8<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0209652363)

配置业务的DNN为“huawei.com”的用户寻呼优先级为3，策略索引为1，使业务在系统拥塞时，可以被优先处理，执行如下命令：

```
ADD NGPAGINGPRIPLCY: PLCYIDX=1,DNN="huawei.com", PAGINGPRIORITY=3;
```
