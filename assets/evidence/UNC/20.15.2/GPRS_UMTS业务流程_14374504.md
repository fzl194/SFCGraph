**MS先启动获取IPv6前缀流程。**

1. 无线终端（MS）发起激活请求。

2. SGSN向PGW-C发送Create PDP Context Request消息。

3. PGW-C获取到MS的激活请求后，向AAA Server发送鉴权请求。

P1：AAA Server在发送鉴权响应时，AAA Server支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

4. AAA Server将鉴权响应消息发送给PGW-C。

5. PGW-C发送PFCP Session Establishment Request消息给PGW-U。

P2：PGW-C支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

6. PGW-U返回PFCP Session Establishment Response消息给PGW-C。

P3：PGW-U支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

7. PGW-C返回Create PDP Context Response消息给SGSN。

8. SGSN返回Activate PDP Context Response消息给MS。

9. MS发送ICMPv6 Router Solicitation消息给PGW-U，发起IPv6地址自动配置协商。

10. PGW-U通过PFCP Session Report Request消息将IPv6地址自动协商请求发给PGW-C。

11. PGW-C返回PFCP Session Report Response消息给PGW-U。

12. PGW-U返回ICMPv6 Router Advertisement消息给MS，携带分配给MS的IPv6地址前缀。同时将Router Advertisement消息中的O-flag置位为1，表示后续可以通过DHCPv6方式获取其他的相关参数。

**MS获取IPv6地址前缀后，发起DHCPv6地址分配流程。**

13. MS发送DHCPv6 Solicit请求消息给PGW-U，消息中携带IA_PD option、Rapid Commit option和Prefix Exclude Option。

14~15. PGW-U通过PFCP Session Report Request消息将MS进行DHCPv6地址分配信息发给PGW-C，PGW-C返回响应。

16. （快速流程不包含此消息）PGW-U返回DHCPv6 Advertise消息给MS。

17. （快速流程不包含此消息）MS发送DHCPv6 Request消息给PGW-U。

18~19. （快速流程不包含此消息）PGW-U通过PFCP Session Report Request消息将MS协商信息发给PGW-C，PGW-C返回响应。

20. PGW-U返回DHCPv6 Reply消息给MS，携带为其分配的IPv6 delegated-prefix和Prefix Exclude Option。

21. MS下的终端设备申请接入网络时，MS为其从IPv6 delegated-prefix段中分配64位前缀。
