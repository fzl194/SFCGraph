## [操作步骤](#ZH-CN_OPI_0000001889769316)

1. 进入 “MML命令行-UNC” 窗口。
2. 基于运营商打开紧急呼叫触发的NI-LR流程开关，并配置流程策略。
  [**ADD NGNILRPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/5G定位服务管理/紧急定位服务管理/增加NI-LR功能参数（ADD NGNILRPARA）_44006465.md)
  > **说明**
  > 协议规定只能由源侧或目标侧的一侧进行上报，在 [**ADD NGNILRPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/5G定位服务管理/紧急定位服务管理/增加NI-LR功能参数（ADD NGNILRPARA）_44006465.md) 和 [**ADD LCSPARAEX**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/LCS/LCS扩展参数/增加LCS扩展参数(ADD LCSPARAEX)_26305624.md) 命令的 “HORPTTYPE” 参数中配置需保持一致。
3. **可选：** 若LMF不支持显示订阅LPP、NRPPA等类型时，需开启 [DWORD63 BIT1](../../../../../OM参考/软件参数/UNC软件参数/业务软件参数/软件参数（AMF）/移动性管理/DWORD63 BIT1 控制5GC定位流程中LMF未向AMF发起位置定位订阅时，AMF是否通过查询N_1f6a1225_11038965.md) 软参以保证定位流程正常运行。

## [任务示例](#ZH-CN_OPI_0000001889769316)

任务描述

标准的LCS组网进行紧急呼叫通话场景下，部署紧急呼叫NI-LR定位功能。

脚本

// 进入 “MML命令行-UNC” 窗口。

//基于运营商打开紧急呼叫触发的NI-LR流程开关，并配置流程策略。

```
ADD NGNILRPARA: NOID=0, NILR=YES, NILRPLCY=PROTOCOL_POSITION, RLSRPT=YES, HORPT=YES, HORPTTYPE=SOURCE;
```

//可选：若LMF不支持显示订阅LPP、NRPPA等类型时，需开启 “DWORD63 BIT1” 软参以保证定位流程正常。

执行SET AMFSOFTPARAOFBIT命令设置软参DWORD63中BIT1的取值为“1”。

```
SET AMFSOFTPARAOFBIT: DT=Dw, DWORDNUM=63, VALUE=VALUE_1, POSITION=1;
```
