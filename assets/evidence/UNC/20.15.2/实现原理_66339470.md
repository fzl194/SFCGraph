# 实现原理

分区域网络运维主要实现的功能如下：

**TAI组对象统计**

本特性之前， UNC 支持基于TAI对象的性能统计，此对象为配置对象，需要逐个配置才能实现性能统计。当TAI数量过多时，无法对所有的TAI对象进行性能统计。因此， UNC 提供了分区域网络运维功能来进行基于TAI组的性能统计。

**TAI对象合并**

UNC 缺省为当前已存在的每一个TAI创建一个TAI组进行性能统计测量，但最多只能提供1500个TAI组。为了实现更多TAI对象的性能统计，可以通过配置实现将多个TAI划分到一个TAI组中，以TAI组的更大区域范围粒度进行性能指标统计。 以上的TAI对象合并通过配置命令 [**ADD PERFOBJ**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/性能管理/性能对象管理/增加性能对象信息(ADD PERFOBJ)_26305998.md) 和 [**ADD PERFOBJRULE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/性能管理/性能对象规则管理/增加性能对象规则(ADD PERFOBJRULE)_72225873.md) 实现，以减少TAI组对象数过多带来的操作维护不便。

**满规格告警**

UNC 实现了满规格告警提示，当TAI组对象性能统计达到或超过规格1500个后， UNC 会提示告警 [ALM-12001 性能统计对象数达到或超过可测量最大规格](../../../../../网络运维/故障处理/UNC告警处理/业务告警/SGSN&MME/ALM-12001 性能统计对象数达到或超过可测量最大规格_89656068.md) ，告知用户统计对象超出规格，建议将TA划分成更大的TA组进行统计或建议替换统计对象。
