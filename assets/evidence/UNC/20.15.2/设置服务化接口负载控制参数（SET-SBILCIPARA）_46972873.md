## [命令功能](#ZH-CN_MMLREF_0000001146972873)

**适用NF：AMF、SMF**

该命令用于设置服务化接口负载控制参数。

## [注意事项](#ZH-CN_MMLREF_0000001146972873)

- 该命令执行后立即生效。

- 系统部署完成后，已经存在初始记录，参数的初始记录值如下表：

| SWITCH | THRESHOLD | STEPS | INTERVAL |
| --- | --- | --- | --- |
| OFF | 80 | 5 | 5 |

#### [操作用户权限](#ZH-CN_MMLREF_0000001146972873)

G_1，管理员级别命令组；G_2，操作员级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001146972873)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| SWITCH | 服务化接口负载控制开关 | 可选必选说明：可选参数<br>参数含义：该参数表示服务化接口负载控制的开关。<br>数据来源：本端规划<br>取值范围：<br>- “OFF（关）”：关闭服务化接口负载控制开关<br>- “ON（开）”：开启服务化接口负载控制开关<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST SBILCIPARA查询当前参数配置值。<br>配置原则：无 |
| THRESHOLD | 负载通知阈值(%) | 可选必选说明：可选参数<br>参数含义：该参数表示负载通知的阈值，当服务端的服务化接口负载超过阈值，会通知客户端。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~100。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST SBILCIPARA查询当前参数配置值。<br>配置原则：无 |
| STEPS | 负载通知步长(%) | 可选必选说明：可选参数<br>参数含义：该参数表示当服务端的服务化接口负载超过阈值通知客户端后，再次通知的步长。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~100。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST SBILCIPARA查询当前参数配置值。<br>配置原则：无 |
| INTERVAL | 负载获取间隔时长(分钟) | 可选必选说明：可选参数<br>参数含义：该参数表示系统获取实时负载间隔，当网元注册成功后，会首次获取本网元实时负载，后续按照该参数的时间间隔周期性获取。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是1~60，单位是分钟。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST SBILCIPARA查询当前参数配置值。<br>配置原则：<br>当期望负载的调整得到更快的应用，则设置较小值。但是会增加系统负荷。建议保持系统初始值。 |

## [使用实例](#ZH-CN_MMLREF_0000001146972873)

设置负载控制开关为开启，负载通知阈值为90，负载通知步长为6：

```
SET SBILCIPARA: SWITCH=ON, THRESHOLD=90, STEPS=6;
```
