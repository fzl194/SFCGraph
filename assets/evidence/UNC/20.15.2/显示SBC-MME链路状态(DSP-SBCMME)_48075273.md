#### [命令功能](#ZH-CN_CONCEPT_0000002148075273)

**适用网元：CBCF**

此命令用于查询SBC MME下SBc链路的连接状态。

#### [注意事项](#ZH-CN_CONCEPT_0000002148075273)

- 该命令执行后立即生效。
- 如果不输入MME索引号，则显示所有SBC MME下SBc链路的连接状态。

#### [本地用户权限](#ZH-CN_CONCEPT_0000002148075273)

manage-ug；system-ug；monitor-ug

#### [网管用户权限](#ZH-CN_CONCEPT_0000002148075273)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000002148075273)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| MMEIDX | MME索引 | 可选必选说明：可选参数<br>参数含义：该参数用于指定待查询的MME索引。<br>数据来源：本端规划<br>取值范围：0~127<br>默认值：无 |

#### [使用实例](#ZH-CN_CONCEPT_0000002148075273)

输入MME索引，查询指定的SBC MME数据：

```
%%DSP SBCMME: MMEIDX=0;%%
RETCODE = 0  操作成功。

输出结果如下：
-------------------------
     MME 索引  =  0
       进程号  =  6
       RU名称  =  USN_SP_RU_0064
   IP地址类型  =  IPv4
    本地地址1  =  10.2.62.20
    本地地址2  =  255.255.255.255
   本地端口号  =  29168
    对端地址1  =  172.27.168.1
    对端地址2  =  255.255.255.255
   对端端口号  =  29168
      VPN名称  =  _public_
     链路状态  =  正常
(结果个数 = 1)
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000002148075273)

| 输出项名称 | 输出项解释 |
| --- | --- |
| MME索引 | SBC MME的索引。 |
| 进程号 | SBc链路所在SGP进程的进程号。 |
| RU名称 | 查询所在的资源单元名称。 |
| IP地址类型 | SBc链路IP地址类型。 |
| 本地地址1 | SBc链路第一个本地IP地址。 |
| 本地地址2 | SBc链路第二个本地IP地址。 |
| 本地端口号 | SBc链路本地端口号。 |
| 对端地址1 | SBc链路第一个对端IP地址。 |
| 对端地址2 | SBc链路第二个对端IP地址。 |
| 对端端口号 | SBc链路对端端口号。 |
| VPN名称 | SBc链路的VPN名称。 |
| 链路状态 | 查询SBc链路状态的查询结果。<br>取值说明：<br>- “正常”：表示UNC网元与MME网元间之间的通信在该链路上正常。<br>- “故障”：表示UNC网元与MME网元间之间的通信在该链路上中断，需要检查网络状况，分析故障原因。详细操作参见[ALM-80718 SBc-MME链路故障](../../../../../../../../网络运维/故障处理/UNC告警处理/业务告警/SGSN&MME/ALM-80718 SBc-MME链路故障_48031301.md)告警处理。 |
