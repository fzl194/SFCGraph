# 方案背景

非Full Mesh组网如 [图1](#ZH-CN_TOPIC_0000001101470588__fig13752142314388) 左侧所示，一个SMF可以对接多个UPF，一个UPF仅归属于一个SMF，当SMF故障时，所有该SMF下的UPF均无法进行业务，造成资源的浪费。

因此引入Full Mesh组网，如 [图1](#ZH-CN_TOPIC_0000001101470588__fig13752142314388) 右侧所示，SMF和UPF全互联，任意SMF可管理所有UPF，任意UPF同时归属所有SMF。

3GPP协议23501中也明确表示，任意UPF可以被多个SMF管理。

非Full Mesh组网与Full Mesh组网在不同纬度的对比如 [表1](#ZH-CN_TOPIC_0000001101470588__table226691284020) 所示。

*表1 非Full Mesh组网与Full Mesh组网对比*

| **对比** | **非Full Mesh组网** | **Full Mesh**<br>**组网** |
| --- | --- | --- |
| **连接关系** | SMF与UPF一对多连接。 | SMF与UPF全互联。 |
| **SMF**<br>**故障** | 故障SMF不可用，同时对接的UPF也不可用，故障SMF及其对接UPF上的已在线用户业务受损。 | 故障SMF不可用，仅通过故障SMF接入的已在线用户业务受损。 |
| **UPF**<br>**故障** | 故障UPF不可用，同时对接的SMF可接入的用户数规格降低。 | 故障UPF不可用，仅故障UPF上的用户业务受损。 |
| **SMF**<br>**资源规划** | 1/N冗余 | 1/N冗余 |
| **UPF**<br>**资源规划** | X/N冗余（X为每个SMF对接的UPF数） | 1/N冗余 |
