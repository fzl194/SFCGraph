## [操作场景](#ZH-CN_OPI_0000001094028348)

本操作指导介绍在运行网络中激活支持 ODB 特性的操作过程。

支持ODB（Operator Determined Barring）特性，即运营商决定的闭锁，运营商能够通过设置ODB参数限制一些用户的业务和服务。

> **说明**
> 适用于AMF

## [必备事项](#ZH-CN_OPI_0000001094028348)

前提条件

- 请仔细阅读[WSFD-106005 支持ODB（适用于AMF）特性概述](特性概述_94308338.md)。
- 需要对端网元UDM支持ODB特性。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0000001094028348)

1. 进入 “MML命令行-UNC” 窗口。
2. 开启ODB注册和PDU会话建立流程参数检查开关。
  [**SET NGMMFUNC**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动性管理/设置5G移动性管理功能（SET NGMMFUNC）_09653748.md)
3. 配置注册流程ODB拒绝原因值。
  [**SET NGMMPROCTRL**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM流程管理/5G移动性管理流程控制参数/设置5G移动性管理流程控制参数（SET NGMMPROCTRL）_09652386.md)
4. 配置PDU会话建立流程ODB拒绝原因值。
  [**SET SMPDUCTRL**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NAS传输管理/PDU会话控制管理/设置PDU会话控制参数（SET SMPDUCTRL）_44008047.md)

## [任务示例](#ZH-CN_OPI_0000001094028348)

任务描述

配置AMF支持禁止用户所有分组域业务。

脚本

//开启ODB参数检查开关。

```
SET NGMMFUNC: REGODBPLCY=YES,PDUESTODBPLCY="YES"；
```

//配置注册流程ODB拒绝原因值。

```
SET NGMMPROCTRL: PROT=UDM_REGISTRATION, ODBREJCAUSE=0;
```

//配置PDU会话创建流程ODB拒绝原因值。

```
SET SMPDUCTRL: ODBREJCAUSE=0;
```
