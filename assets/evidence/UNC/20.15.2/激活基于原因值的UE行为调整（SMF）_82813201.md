## [操作场景](#ZH-CN_OPI_0282813201)

当用户接入5G网络，当系统内部以及周边网元异常导致流程失败时，SMF可以控制给终端下发的NAS原因值以调整终端的行为，辅助运营商实现对现网设备兼容性控制的需求。

> **说明**
> 给UE下发的原因值变化可能会导致相应拒绝类话统指标的变化，并对终端行为产生影响，在部署前需要评估影响。关于不同原因值的含义及对终端行为的影响请参见协议3GPP TS 24.501。

## [任务示例](#ZH-CN_OPI_0282813201)

任务描述

配置SMF会话创建流程异常情况下给UE下发的NAS原因值，其中异常来源为UPF，希望UPF响应失败后UE收到原因值后能尝试再次接入，且如果当某APN不允许该UE创建会话，希望UE收到原因值后不再尝试使用该APN继续创建会话。

脚本

//配置SMF原因值映射组。

```
ADD SMCAUSEGRP: CAUSEGRPID=17, CAUSEGRPNM="Default1";
```

//配置SMF流程中NAS原因值控制参数。

```
ADD SMFCAUSECTRL: PROCEDURETYPE=PDU_SESSION_EST_PROC,  NFTYPE=UPF, CAUSEGROUPID=17, CAUSEEXPIRATION=0;
```

//配置SMF原因值映射关系。

```
ADD SMCAUSEMAP: CAUSEGRPID=17, CAUSERANGE=DEFAULT, TCAUSE=27;
```
