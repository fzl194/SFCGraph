**UE获取IPv6前缀流程。**

1. UE向MME发送PDN Connectivity Request消息。

2. MME向SGW发送Create Session Request消息。

3. SGW向PGW-C发送Create Session Request消息。

4. PGW-C向AAA Server发送鉴权请求。

P1：AAA Server在发送鉴权响应时，AAA Server支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

5. AAA Server将鉴权响应消息发送给PGW-C。

6. PGW-C发送PFCP Session Establishment Request消息给PGW-U。

P2：PGW-C支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

7. PGW-U返回PFCP Session Establishment Response消息给PGW-C。

P3：PGW-U支持为Prefix Delegation终端分配IPv6地址，携带IPv6 Address，置位IPv6D，并通过IPv6 Prefix Delegation Bits field指示IPv6地址前缀长度。

8. PGW-C返回Create Session Response消息给SGW。

9. SGW返回Create Session Response消息给PGW-C。

10. UE发送ICMPv6 Router Solicitation消息给PGW-U，发起IPv6地址自动配置协商。

11. PGW-U通过PFCP Session Report Request消息将IPv6地址自动协商请求发给PGW-C。

12. PGW-C返回PFCP Session Report Response消息给PGW-U。

13. PGW-U返回ICMPv6 Router Advertisement消息给UE，携带分配给UE的IPv6地址前缀。同时将Router Advertisement消息中的O-flag置位为1，表示后续可以通过DHCPv6方式获取其他的相关参数。

**UE获取IPv6地址前缀后，发起DHCPv6地址分配流程。**

14. UE发送DHCPv6 Solicit请求消息给PGW-U，消息中携带IA_PD option、Rapid Commit option和Prefix Exclude Option。

15~16. PGW-U通过PFCP Session Report Request消息将UE进行DHCPv6地址分配信息发给PGW-C，PGW-C返回响应。

17. （快速流程不包含此消息）PGW-U返回DHCPv6 Advertise消息给UE。

18. （快速流程不包含此消息）UE发送DHCPv6 Request消息给PGW-U。

19~20. （快速流程不包含此消息）PGW-U通过PFCP Session Report Request消息将UE协商信息发给PGW-C，PGW-C返回响应。

21. PGW-U返回DHCPv6 Reply消息给UE，携带为其分配的IPv6 delegated-prefix和Prefix Exclude Option。

22. UE下的终端设备申请接入网络时，UE为其从IPv6 delegated-prefix段中分配64位前缀。
