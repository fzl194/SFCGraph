#### [适用NF](#ZH-CN_CONCEPT_0191532261)

SGSN、 GGSN、 MME、SGW-C、PGW-C

#### [定义](#ZH-CN_CONCEPT_0191532261)

- LTE到UMTS PS网络的切换
  UMTS与LTE共存的情况下，UE在LTE网络进行业务，当UMTS网络的覆盖（信号）优于LTE网络时，或者在eNodeB负荷较重等情况下，eNodeB会触发UE切换到UMTS网络。
- UMTS到LTE网络的切换
  UMTS与LTE共存的情况下，UE在UMTS网络进行业务，当UE移动到LTE覆盖范围等情况下，RNC可能会触发UE切换到LTE网络，以便为UE提供更好的服务。

#### [客户价值](#ZH-CN_CONCEPT_0191532261)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 运营商在现有UMTS等网络的基础上，将逐步引入LTE网络满足人们快速增长的无线数据业务需求，形成多系统长期共存的情况。因此给运营商带来如下受益：<br>- 保护UMTS运营商的已有投资，充分发挥两个网络的性能，同时提供更丰富的业务体验。<br>- 实现UMTS网络到LTE网络的平滑演进。 |
| 用户 | - 保证用户业务连续性：当UE移动出LTE网络覆盖范围时，如果目标区域存在UMTS网络，则可以切换到目标UMTS网络，保证业务连续性。或者当UE移动出UMTS网络覆盖范围时，如果目标区域存在LTE网络，则可以切换到目标LTE网络，保证业务连续性。<br>- 良好的用户体验：LTE可以提供更高的QoS，可以支撑更丰富的业务；UE在UMTS网络覆盖区域内移动，当移动到有LTE覆盖的区域时，可以切换到LTE，从而改善用户的业务体验。 |

#### [应用场景](#ZH-CN_CONCEPT_0191532261)

- 支持UE从LTE到UMTS PS网络的切换。
- 支持UE从UMTS到LTE网络的切换。

#### [对系统的影响](#ZH-CN_CONCEPT_0191532261)

- 在LTE发展初期，引入UMTS和LTE互操作缓解UMTS网络热点区域的网络阻塞。
- 引入UMTS和LTE互操作，在LTE的覆盖边缘会发生异系统间的互操作，增加系统负荷。

#### [应用限制](#ZH-CN_CONCEPT_0191532261)

应用本特性有以下限制：

- 为保证网络切换后业务不中断，需要GGSN和PGW-C共建（同一个网元内）。
- 为保证鉴权集的一致性，推荐HLR和HSS共建，否则可能会在网络重选过程中多次鉴权，影响切换时延。
- 该特性中GTP消息中不支持GTPv0版本。

#### [原理概述](#ZH-CN_CONCEPT_0191532261)

- [**基本概念**](#ZH-CN_CONCEPT_0191532261__p9636737102712)
- [**系统实现**](#ZH-CN_CONCEPT_0191532261__p116361837192711)

[**基本概念**](#ZH-CN_CONCEPT_0191532261__zh-cn_concept_0130428802_sub3_1)

UNC 支持UE从LTE到UMTS网络的切换（Inter和Intra），切换由源侧eNodeB发起，MME通知目标侧SGSN启动切换。

UNC 支持UE从UMTS到LTE网络的切换（Inter和Intra），切换由RNC发起，源侧SGSN通知目标侧MME启动切换。

[**系统实现**](#ZH-CN_CONCEPT_0191532261__zh-cn_concept_0130428802_sub3_2)

- LTE到UMTS PS网络的切换
  切换准备阶段，MME将当前创建的承载上下文和当前使用的安全上下文传递给目标侧SGSN。需要将安全上下文转成UMTS网络所需的安全上下文。如果对端SGSN是GnGp SGSN，还需要将承载上下文转换成PDP上下文。
  > **说明**
  > - LTE网络中的默认承载和专用承载切换到UMTS PS网络后，UMTS网络将其作为激活相同APN的多个PDP进行处理。对于专用承载，转换后的PDP承载仅提供GBR速率。
  > - LTE到UMTS PS网络的切换，要求终端在LTE侧和UMTS侧均支持TFT（Traffic Flow Template）功能。
  如果某些承载需要数据完整性，即：将源侧eNodeB没有成功发送的下行数据传递给目标侧RNC，MME依赖组网情况，当源侧eNodeB和目标侧RNC之间不能路由时可能需要在切换准备阶段建立间接转发隧道（Indirect Data Forwarding Tunnel），通过SGW进行数据转发。
  切换完成后，MME释放用户的承载。UE在UMTS网络发起切换过程简化的RAU过程。
  **图1** LTE到UMTS PS网络的切换基本原理图

  <br>

  ![](特性概述_91532261.assets/zh-cn_image_0192721226_2.png)

  | 1、eNodeB向MME发起切换请求，MME通知SGSN进行前向重定位。 |
  | --- |
  | 2、SGSN通知目标RNC建立无线资源，并向MME发送重定位响应。 |
  | 3、如果eNodeB和目标RNC之间没有直传链路，MME需要通知SGW-C建立非直传链路。 |
  | 4、MME通过eNodeB向UE发送切换命令。 |
  | 5、eNodeB进行数据转发：如果有直传链路，eNodeB直接将数据转发到目标RNC；如果没有直传链路，eNodeB先将数据转发到SGW，再由SGW将数据转发到目标RNC。 |
  | 6、UE完成切换并向目标RNC发送RRC重配置完成消息。 |
  | 7、SGSN通知MME目标侧已完成重定位，MME响应重定位完成。 |
  | 8、SGSN通知GGSN/PGW-C更新上下文信息。 |
  | 9、MME通知SGW-C删除会话，并通知eNodeB释放资源。 |

  > **说明**
  > Intra场景下SGSN和MME融合，所以 [图1 LTE到UMTS PS网络的切换基本原理图](#ZH-CN_CONCEPT_0191532261__zh-cn_concept_0130428802_fig_05) 中不存在SGSN和MME之间交互的消息，其他消息流程不变。
- UMTS到LTE网络的切换
  源侧SGSN会通知目标侧MME启动切换，并携带承载上下文和安全上下文，MME转换安全上下文，如果源侧SGSN为GnGp SGSN，则携带的是PDP上下文，MME需要将PDP上下文转换成EPS承载上下文。
  MME通知目标侧eNodeB，如果某些承载需要数据完整性保护，则MME基于组网情况决定是否建立间接转发隧道。
  切换完成后，UE会发起简化的TAU过程，MME向HSS发起位置更新。 UNC 可以在TAU过程中重新对UE进行鉴权。
  **图2** UMTS到LTE网络的切换基本原理图

  <br>

  ![](特性概述_91532261.assets/zh-cn_image_0192721284_2.png)

  | 1、UE激活态从UMTS覆盖区移动到LTE和UMTS重叠覆盖区，RNC向SGSN发起切换请求，SGSN通知MME进行重定位。 |
  | --- |
  | 2、MME通知SGW-C建立会话，通知eNodeB准备无线资源，eNodeB和SGW-C准备好资源后回送响应消息。 |
  | 3、如果RNC和eNodeB之间没有直传链路，MME需要通知SGW-C建立非直传链路。 |
  | 4、MME向SGSN确认目标侧已准备好资源，SGSN向RNC发送重定位命令。 |
  | 5、RNC进行数据转发：如果有直传链路，RNC直接将数据转发到eNodeB；如果没有直传链路，RNC先将数据转发到SGW-C，再由SGW-C将数据转发到eNodeB。 |
  | 6、RNC命令UE切换到eNodeB，UE完成切换，eNodeB向MME发送确认消息，MME紧接着向SGSN发送重定位完成消息。 |
  | 7、MME通知SGW-C/PGW-C切换已经完成，需要更新承载，如果已部署PCRF，PGW-C需要向PCRF上报RAT改变事件，以供决策。 |
  | 8、SGSN通知RNC释放原有的无线资源，RNC释放后向SGSN回送响应消息。 |

  > **说明**
  > Intra场景下SGSN和MME融合，所以 [图2 UMTS到LTE网络的切换基本原理图](#ZH-CN_CONCEPT_0191532261__zh-cn_concept_0130428802_fig_03) 中不存在SGSN和MME之间交互的消息，其他消息流程不变。
