# 实现原理

**网元间支持的能力的协商**

- HSS与MME之间
  SCEF通过HSS向MME订阅请求，HSS需要提前知道MME是否支持UE状态上报功能以及支持的UE状态上报事件类型。
  根据29272协议，本特性开启后，MME在Attach/TAU流程中，通过ULR消息中的“Supported Features：Feature-List-ID 2”信元（bit19=1）通知HSS其支持UE状态上报功能。MME需要通过命令 [**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md) 或 [**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 设置是否携带该标志位。
  HSS从SCEF收到订阅请求后，将订阅请求转发给MME。
  > **说明**
  > 如果MME收到订阅请求但MME不支持该功能时，MME在IDA响应中将Monitoring-Event标志位置为0。
- MME与MME之间
  当UE触发Attach或者Inter MME的TAU/Handover时，老侧MME可在S10接口通过Identification Response、Context Response或Forward Relocation Request消息将该用户的所有订阅任务相关的SCEF Reference ID和SCEF ID，以及当前剩余的上报次数传递给新侧。该功能可通过 [**ADD MMECHARACT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-SGSN_S10_S16_S3接口管理/MME属性/增加MME属性配置信息（ADD MMECHARACT）_26305766.md) 的参数 “NB-IoT状态上报订阅信息” 设置。

**UE状态上报流程**

SCEF通过HSS向MME做事件订阅流程如 [图1](#ZH-CN_TOPIC_0000001446602534__fig02) 所示，详细请参见3GPP 23682 5.6.1章节。

**图1** 通过HSS向MME做事件订阅流程

<br>

![](实现原理_46602534.assets/zh-cn_image_0000001498268193_2.png "点击放大")

<br>

1. SCS/AS发送Monitoring Request消息给SCEF。消息中携带包含在NB-IoT平台上的标识终端的External Identifier信元以及订阅时间相关参数，包括需要订阅事件类型的Monitoring Type、Maximum Number of Reports、Monitoring Duration、SCS/AS Reference ID for Deletion等信元（详细可参考3GPP TS 23.682中的Monitoring Event configuration and deletion via HSS）。
2. SCEF收到Monitoring Request消息后，为订阅请求任务分配一个SCEF Reference ID。根据SCEF ID和SCEF Reference ID可以在全网范围内唯一标识一个订阅任务。
3. SCEF发送Monitoring Request消息给HSS。
4. HSS完成External Identifier到IMSI的相互转换，并完成用户到MME的路由。
5. 当MME支持订阅事件的能力开放时，HSS将订阅信息通过Insert Subscriber Data Request（Monitoring Type，SCEF ID，SCEF Reference ID，Maximum Number of Reports，Monitoring Duration，SCEF Reference ID for Deletion，Chargeable Party Identifier）消息转发给MME。订阅请求的主要通用参数如[表1](#ZH-CN_TOPIC_0000001446602534__tab5)所示。
6. 如果订阅成功且在MME上可以立即获取到订阅请求的报告，MME在Insert Subscriber Data Answer消息中将满足订阅条件的事件报告返回给HSS；否则MME仅在Insert Subscriber Data Answer消息回复订阅成功。待订阅能力报告获取到后，MME再通过Reporting Information Request消息上报给SCEF。
7. HSS发送Monitoring Response将MME上报的报告转发给SCEF。SCEF发送给SCS/AS平台。

*表1 HSS下发订阅请求的主要参数*

| 订阅请求主要参数 | 参数说明 |
| --- | --- |
| Monitoring Type | 订阅事件类型，取值如下：<br>- LOSS_OF_CONNECTIVITY (0)<br>- UE_REACHABILITY (1)<br>- LOCATION_REPORTING (2)<br>- CHANGE_OF_IMSI_IMEI(SV)_ASSOCIATION (3)<br>- ROAMING_STATUS (4)<br>- COMMUNICATION_FAILURE (5)<br>- AVAILABILITY_AFTER_DDN_FAILURE (6)<br>- NUMBER_OF_UES_PRESENT_IN_A_GEOGRAPHICAL_AREA (7)<br>在同一个订阅任务中，可同时订阅多种事件类型。 |
| SCEF ID | SCEF 标识，该信元与SCEF Reference ID可以在全网范围内唯一标识一个订阅任务。 |
| SCEF Reference ID | 该信元与SCEF ID可以在全网范围内唯一标识一个订阅任务。<br>当SCEF为某个用户或者某批用户创建对某种能力的订阅任务时，SCEF就会为该订阅任务分配一个SCEF Reference ID。因此一个SCEF Reference ID可以对应多个用户，一个用户也可以有多个SCEF Reference ID。 |
| Maximum Number of Reports | 可选参数，表示某次订阅任务下发后，需要上报的报告数。MME上报的次数超过该数目后不再上报报告。该信元不携带时表示只上报一次订阅报告。<br>该参数与Monitoring Duration同时携带时，只要满足其中一个条件就停止上报。 |
| Monitoring Duration | 可选参数，标识某次订阅请求的生效时长。该时长超时后，MME就停止上报报告。<br>该参数与Maximum Number of Reports同时携带时，只要满足其中一个条件就停止上报。 |
| SCEF Reference ID for Deletion | 表示删除指定的订阅任务。<br>说明：当IDR消息中携带了此信元时，仅删除该用户关联SCEF Reference ID的任务。而不删除整系统范围内SCEF Reference ID对应的所有用户的订阅任务。 |

**订阅事件的处理**

如 [表2](#ZH-CN_TOPIC_0000001446602534__tab6) 所示为MME收到订阅请求后针对各类事件的处理过程。

*表2 NB-IoT UE状态订阅事件的处理过程*

| 订阅事件 | 订阅目的 | 订阅处理 |
| --- | --- | --- |
| Loss of connectivity（UE连接丢失） | 应用服务器通过SCEF向MME订阅此事件，可以向MME指定终端的TAU周期时长以及判断UE是否连接丢失的最大监测时长（即移动可达定时器）。 | 当移动可达定时器超时、用户分离时，MME向SCEF上报Loss of connectivity（UE连接丢失）事件。 |
| UE reachability（UE可达） | NB-IoT终端支持PSM或者eDRX模式时，大部分时间处于不可达状态。为了能感知NB-IoT终端的可达状态，应用服务器可通过SCEF向MME订阅UE可达事件，在UE可达时启动下行报文的传输。 | 订阅此事件中携带的参数除了如<br>[表2](#ZH-CN_TOPIC_0000001446602534__tab6)<br>通用参数外，还包括如下四个专用的参数：<br>- Reachability Type：可达类型，目前支持数据可达（Reachability for Data）。<br>- Maximum Latency：HSS会用此参数作为周期TAU/RAU定时器（T3412）时长发送给MME。<br>- Maximum Response Time：- PSM模式下，MME将此参数映射为Active Timer（T3342）发送给NB-IoT终端。<br>- eDRX模式下，MME按照该信元时长将订阅报告上报给SCEF，提前准备下发数据包。避免下行数据包发下来时，终端又进入了不可达状态。<br>- Suggested number of downlink packets：MME在DDN Ack消息中将此参数作为 Buffering Suggested Packet Count信元值发送给S-GW。通过[**SET M2MCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/M2M控制参数/设置M2M控制参数(SET M2MCTRL)_72345369.md)的参数“缓存报数取值来源”决定。 |
| Location Reporting（位置信息） | 位置信息，可获取UE的当前实际位置或者已知位置。 | MME向SCEF上报UE的位置信息。 |
| Communication failure（UE通信失败） | 检测UE的通信状态。 | MME从eNodeB收到S1 UE Context Release Request，并且携带异常原因值（O&M Intervention、 Unspecified Failure、User Inactivity等非正常释放的原因值）时，MME认为用户通信失败，向SCEF上报UE通信失败事件。<br>说明：该事件仅通过T6a口直接向SCEF上报。 |
| Availability after DDN Failure（DDN失败后可用） | 与UE reachability（UE可达）事件订阅目的一致，应用于应用服务器向UE发送下行数据的场景。MME寻呼终端失败后，等UE可用时才发送下行数据。 | MME收到订阅后仅上报一次。 |
