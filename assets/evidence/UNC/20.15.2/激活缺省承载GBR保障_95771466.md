## [操作场景](#ZH-CN_OPI_0000001195771466)

需要为WBB用户创建一个可以提供带宽保障的any to any的GBR专有承载时，需要激活缺省承载GBR保障。

> **说明**
> 适用于PGW-C、SMF。

## [必备事项](#ZH-CN_OPI_0000001195771466)

前提条件

- 请仔细阅读[WSFD-109304 缺省承载GBR保障](../WSFD-109304 缺省承载GBR保障_40691401.md)。
- 完成[激活PCC基本功能](../WSFD-109101 PCC基本功能/PCC基本功能（5G）/激活PCC基本功能_72467890.md)/[激活PCC基本功能](../WSFD-109101 PCC基本功能/PCC基本功能（2G_3G_4G，基于Gx接口）/激活PCC基本功能_29056159.md)。

数据

GBR带宽需合理规划，无线侧资源不足时，any to any GBR专有承载会创建失败。用户将不能使用该any to any GBR专有承载。

## [操作步骤](#ZH-CN_OPI_0000001195771466)

1. 进入 “MML命令行-UNC” 窗口。
2. 配置缺省GBR承载保障的相关参数。
  [**SET DFTGBRBEARER**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/基本功能/缺省GBR承载/设置缺省GBR承载参数（SET DFTGBRBEARER）_09897061.md)

## [任务示例](#ZH-CN_OPI_0000001195771466)

任务描述

开启指定APN的缺省承载GBR保障开关，缺省承载相关参数值详见 **必备事项>数据** 。

脚本

```
SET DFTGBRBEARER: APN="apn-test", DFTGBRBEARERSW=ENABLE, LOCQOSQCI=4, LOCQOSARP=15, LOCQOSPREEMPCAP=DISABLE, LOCQOSPREEMPVUL=ENABLE, LOCQOSMBRUL=64, LOCQOSMBRDL=64, LOCQOSGBRUL=64, LOCQOSGBRDL=64, SESSIONTERMINSW=DISABLE, ACCTYPE=RAT5G;
```
