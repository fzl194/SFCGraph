## [命令功能](#ZH-CN_MMLREF_0000001178258202)

**适用NF：AMF**

此命令用于增加等价PLMN的跟踪区域组成员，同一个跟踪区域组标识具有相同的等价PLMN策略。当基于用户位置采用不同等价PLMN策略时，需要执行此命令。

## [注意事项](#ZH-CN_MMLREF_0000001178258202)

- 该命令执行后立即生效。

- 相同跟踪区域组标识下的跟踪区域范围不能有重叠。
- 在ADD EPLMNGRP命令中同一个PLMN及同一个用户组下引用的不同跟踪区域组标识下对应的TAC范围也不能有重叠。

- 最多可输入4096条记录。

#### [操作用户权限](#ZH-CN_MMLREF_0000001178258202)

G_1，管理员级别命令组；G_2，操作员级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001178258202)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| TACGRPID | 跟踪区域组标识 | 可选必选说明：必选参数<br>参数含义：该参数用于标识使用相同等价PLMN策略的跟踪区域范围。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~4294967294。<br>默认值：无<br>配置原则：无 |
| BGNTAC | 跟踪区域码起始值 | 可选必选说明：必选参数<br>参数含义：该参数用于表示跟踪区编码的起始值。<br>数据来源：全网规划<br>取值范围：字符串类型，输入长度是6。TAC编码为16进制数，按照字符串格式输入，字符串长度为6，只能由数字（0-9），字母（A-F、a-f）组成。字母大小写不敏感。<br>默认值：无<br>配置原则：无 |
| ENDTAC | 跟踪区域码结束值 | 可选必选说明：必选参数<br>参数含义：该参数用于表示跟踪区编码的结束值。<br>数据来源：全网规划<br>取值范围：字符串类型，输入长度是6。TAC编码为16进制数，按照字符串格式输入，字符串长度为6，只能由数字（0-9），字母（A-F、a-f）组成。字母大小写不敏感。本参数表示的跟踪区编码不能小于“跟踪区编码起始值”。<br>默认值：无<br>配置原则：无 |
| DESC | 描述信息 | 可选必选说明：可选参数<br>参数含义：该参数是对区域成员的描述信息，在运维中起到助记的作用。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是0~32。<br>默认值：无<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0000001178258202)

增加一条5G用户跟踪区域范围记录，跟踪区域组标识为20，跟踪区编码起始值为120101，跟踪区编码终止值为120102，执行如下命令：

```
ADD EPLMNAREAMEM: TACGRPID=20, BGNTAC="120101", ENDTAC="120102";
```
