![](OCS组网和链路可靠性_95110432.assets/zh-cn_image_0301704146_2.png)

UNC 允许为用户配备主备OCS Group。每个OCS Group内包含一个或多个OCS服务器。

- 主备OCS Group之间支持热备份。当用户正在使用的主用OCS发生故障时，DCC会话可以动态迁移到备用OCS Group中的某一台状态正常的OCS上。由备用OCS继续执行信用控制，不中断用户业务。
- 同一个OCS Group内的OCS间采用负荷分担方式为配备此OCS Group的用户群进行服务，负荷分担方式如[表1](#ZH-CN_TOPIC_0295110432__table986491739175815)所示，详细配置请参考[配置OCS负荷分担](../激活Gy_Diameter在线计费/配置OCS组网可靠性配置实例/配置OCS负荷分担_95923468.md)。
  *表1 OCS负荷分担方式*

  | 负荷分担方式 | 说明 |
  | --- | --- |
  | 平均负荷分担 | 用户激活时，同一Group内的所有OCS平均负荷分担用户的在线计费消息处理。 |
  | 基于用户号段组（MSISDN/IMSI）的负荷分担 | 同一Group内的所有OCS均绑定了号段组，用户激活时，根据用户号段选择OCS，实现OCS的负荷分担。<br>同一号段组可以被Group内多个OCS绑定，绑定的优先级必须不同。优先级相同时，匹配到该号段组的用户，在这些OCS中平均承担负荷。 |
  | 基于百分比例的负荷分担 | 同一Group内的所有OCS按照设置的百分比例实现OCS的负荷分担。百分比例的基数是累计选择OCS的次数。 |
  > **说明**
  > 基于百分比例的负荷分担和基于用户号段组的负荷分担不能同时使用。

#### [OCS选择](#ZH-CN_TOPIC_0295110432)

PGW-C支持多种方式选择OCS：PCRF下发、AAA server下发和本地配置，本地配置有三种粒度：UserProfile、APN和整机，这些方式的优先级从高到低依次为：PCRF下发的OCS > AAA server下发OCS > UserProfile（Common Policy使用的UserProfile）配置的OCS > 本地APN配置OCS > 本地整机配置OCS，运营商可以根据网络部署情况灵活选择。

> **说明**
> 本地配置的三种粒度是基于OCS Group的，用户选中OCS Group后，再根据 [OCS负荷分担](#ZH-CN_TOPIC_0295110432__section15571144111474) 方式选中OCS。

| OCS选择方式 | 说明 |
| --- | --- |
| 通过PCRF指定OCS | 如果网络中部署PCRF管理用户在线计费信息时，PCRF在CCA-I消息中通过Primary-Event-Charging-Function-Name AVP和Secondary-Event-Charging-Function-Name AVP下发已在PGW-C上定义的OCS-ID，为用户指定OCS服务器。 |
| 通过AAA server指定OCS | 如果网络中部署AAA server管理用户在线计费信息时，AAA server在Access-Accept消息中下发已在PGW-C上定义的OCS-ID，为用户指定OCS服务器。 |
| 基于UserProfile选择OCS Group | 如果网络中部署基于UserProfile配置OCS Group，则用户激活时通过号段(IMSI/MSISDN/IMEISV)、接入类型（RAT）、Charging Characteristics属性、漫游状态或所处位置区匹配命令<br>**ADD UPBINDUPG**<br>选择UserProfile，进而选择UserProfile下绑定的DCC模板，从而可以确定OCS Group。 |
| 基于APN选择OCS Group | 如果网络中部署同一个APN下的用户使用相同OCS Group，则可以基于APN配置OCS Group。用户激活时，根据APN确定绑定的DCC模板，从而确定OCS Group。 |
| 基于计费属性CC选择OCS Group | 如果网络中部署相同计费属性的用户使用相同OCS Group，则可以基于CC配置OCS Group。用户激活时，根据CC确定绑定的DCC模板，从而确定OCS Group。 |

#### [链路可靠性-多Diameter链路](#ZH-CN_TOPIC_0295110432)

PGW-C支持与一个OCS的多个IP或多个SCTP端点建立多条直连Diameter链路，构成一个链路组，链路组内的多条链路间支持主备和负荷分担模式 （链路选择模式通过 **ADD DIAMCONNGRP** 命令中的“SELECTMODE”参数来设置） 。当链路组内的一条链路故障时，PGW-C选择链路组内的其他可用链路进行消息交互，增强信令组网的可靠性。

PGW-C支持通过TCP或SCTP协议来承载Diameter协议，可以分别构建多条Diameter链路：

- 采用TCP传输时，PGW-C与一个OCS的多个IP建立多条TCP链路，各链路间为主备或负荷分担模式。
- 采用SCTP传输时，PGW-C与一个OCS的多个SCTP端点间建立多条SCTP偶联，各偶联间为主备或负荷分担模式。SCTP功能的详细介绍请参见《UNC产品手册》中： WSFD-104508 SCTP。
