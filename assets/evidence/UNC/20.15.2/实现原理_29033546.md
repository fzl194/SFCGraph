#### [基于DNN的UL CL分流](#ZH-CN_TOPIC_0229033546)

该场景下在指定区域内，UE会话激活、位置移动都可能会触发SMF插入或删除UL CL UPF+PSA1的行为，详细如 [表1](#ZH-CN_TOPIC_0229033546__table91651929113013) 所示。

*表1 UE行为触发的UL CL流程*

| UE行为 | 流程概述 |
| --- | --- |
| UE会话激活 | SMF根据UE携带的DNN、TAI等为其选择UL CL UPF+PSA1并插入到会话中，详细流程如<br>[图1](#ZH-CN_TOPIC_0229033546__fig168603579263)<br>所示。 |
| UE移动导致位置区变化，触发的UL CL流程 | 当感知到UE位置区变化会触发SMF向用户会话插入或删除UL CL UPF+PSA1。如果UE从移出覆盖园区的TAI后会触发删除UL CL UPF+PSA1的流程，如<br>[图2](#ZH-CN_TOPIC_0229033546__fig247315910123)<br>所示。 |

**图1** UE会话的UL CL流程

<br>

![](实现原理_29033546.assets/zh-cn_image_0284386905_2.png)

以下步骤解释与图中1~6逐一对应：

1. UE发起会话激活流程，选择主锚点PSA0 UPF。
2. SMF根据UE激活流程中的DNN、TAI、切片以及本地DNAI的配置判定是否需要为UE插入UL CL UPF和辅锚点UPF，如果需要则为UE选择UL CL UPF+PSA1，并发送PFCP Session Establishment Request消息，携带对应的分流规则App ID。UL CL UPF+PSA1收到请求后回应Response消息，携带分配的N3上行隧道端点、UL CL UPF的N9下行隧道端点以及PSA1 UPF的N9隧道上行端点。
3. SMF向PSA0发送PFCP Session Modification Request消息，将其下行隧道端更新至N9接口。PSA0 UPF返回N9接口上行端隧道信息。
4. SMF向“UL CL UPF+PSA1”发送PFCP Session Modification Request消息，携带PSA0 UPF返回N9接口上行端隧道信息，将其上行隧道端更新至N9接口。
5. SMF发送Namf_Communication_N1N2MessageTransfer Request消息，携带N2 SM Information通知(R)AN侧，将N3隧道上行对端隧道信息更新为UL CL UPF。
  (R)AN向AMF发送PDU Session Resource Setup Response，建立AN隧道信息，响应消息中包含AN Tunnel Info、 List of Accepted/Rejected QFI等。之后SMF和AMF通过Nsmf_PDUSession_UpdateSMContext消息交互获取AN Tunnel Info等信息。
6. SMF向UL CL UPF+PSA1发送PFCP Session Modification Request，将(R)AN侧N3隧道下行端地址信息以及相应的PDR发送给UL CL UPF+PSA1，UL CL UPF+PSA1回应PFCP Session Modification Response。

**图2** UE位置变更触发的UL CL流程

<br>

![](实现原理_29033546.assets/zh-cn_image_0229646375_2.png)

1. SMF根据UE更新后的位置信息判定需要删除UL CL UPF，则首先要和主锚点PSA0 UPF交互，SMF生成新的N9/N3接口的PDR通过PFCP Session Modification Request消息带给PSA0 UPF，PSA0 UPF返回Response消息。
2. SMF发送PFCP Session Deletion Request消息给UL CL UPF+PSA1，UL CL UPF+PSA1通过PFCP会话ID识别需要删除的PFCP会话上下文，删除对应的会话。
3. SMF通知主锚点PSA UPF更新N9隧道下行对端端点信息，主锚点PSA0 UPF返回响应。

#### [基于PCF签约User Profile的UL CL分流](#ZH-CN_TOPIC_0229033546)

该场景下在指定区域内，UE会话激活、位置移动以及PCF侧签约信息的变化都可能会触发SMF插入或删除UL CL UPF+PSA1的行为，详细如 [表2](#ZH-CN_TOPIC_0229033546__table815864316529) 所示。

*表2 UE或网络侧行为触发的UL CL流程*

| UE或网络侧行为 | 流程概述 |
| --- | --- |
| UE激活会话 | SMF根据从PCF侧获取的User Profile、本地DNAI配置、DNN以及UE所处的位置区为其选择UL CL UPF+PSA1并插入到会话中，详细流程如<br>[图3](#ZH-CN_TOPIC_0229033546__fig1123514171632)<br>所示。 |
| UE移动导致位置区变化，移入UL CL覆盖区域 | PCF通过向AMF和SMF订阅UE位置信息，当感知到UE位置区变化会触发SMF向用户会话插入UL CL UPF+PSA1。如果UE从一个园区进入到另外一个园区，SMF在删除UL CL UPF+PSA1后插入新的UL CL+辅锚点UPF。<br>[图4](#ZH-CN_TOPIC_0229033546__fig20575154221218)<br>为用户位置区变化触发插入UL CL UPF+PSA1的流程示例。 |
| UE在PCF侧签约的User Profile信息变化 | 对于已经建立辅锚点会话的UE，当其PCF侧对应签约的User Profile发生变化时，PCF会将修改后的User Profile下发到SMF，SMF则根据新收到的User Profile决定是否删除UL CL UPF+PSA1。<br>SMF并不直接感知是UE位置区变化触发还是PCF侧签约发生变化触发的UL CL流程，该场景下触发的UL CL插入流程同<br>[图4](#ZH-CN_TOPIC_0229033546__fig20575154221218)<br>。 |

**图3** PCF签约UL CL User Profile场景下UE激活的流程

<br>

![](实现原理_29033546.assets/zh-cn_image_0278568566_2.png)

1. 1~3步，UE发起会话激活流程。
2. 4a~4b步，SMF与PCF建立SM策略关联。
    a. SMF向PCF发送Npcf_SMPolicyControl_Create Request消息，请求创建SM策略关联，消息中携带pduSessionId、pduSessionType、dnn等信息。
    b. PCF根据用户标识查找出用户签约的业务数据，根据用户的会话信息进行策略判断，生成规则，通过Npcf_SMPolicyControl_Create Response消息发送给SMF，消息中会包含**User Profile**以及PCF要求订阅的一些事件，当这些事件发生时，比如时区变更，位置变更时，需要SMF发送Npcf_SMPolicyControl_Update Request消息给PCF。
3. 5a~5b步，SMF根据DNN、S-NSSAI等信息选择主锚点PSA0 UPF。
4. 6a~6b步，SMF将PCF下发的User Profile以及本地配置的User Profile进行匹配，如果匹配成功且User Profile的属性为UL CL则结合DNN、DNAI、用户位置信息等为UE选择UL CL UPF+PSA1，并发送PFCP Session Establishment Request消息，携带对应的分流规则App ID。UL CL UPF+PSA1收到请求后回应Response消息，携带分配的N3上行隧道端点、UL CL UPF的N9下行隧道端点以及PSA1 UPF的N9隧道上行端点。
5. 7a~7b步，SMF向PSA0发送PFCP Session Modification Request消息，将其下行隧道端更新至N9接口。PSA0 UPF返回N9接口上行端隧道信息。
6. 8a~8b步，SMF向“UL CL UPF+PSA1”发送PFCP Session Modification Request消息，携带PSA0 UPF返回N9接口上行端隧道信息，将其上行隧道端更新至N9接口。
7. 第9步，SMF发送Namf_Communication_N1N2MessageTransfer Request消息，携带N2 SM Information通知(R)AN侧，将N3隧道上行对端隧道信息更新为UL CL UPF。
8. 10~12步，AMF通过(R)AN和UE完成信令交互发送PDU Session Resource Setup Request，请求中包含N2 SM Information，包含QFI、QoS Profile、CN Tunnel Info，发送给UE的N1 SM Container，包含PDU Session Establishment Accept、Allocated IPv4 Address。
  (R)AN向AMF发送PDU Session Resource Setup Response，建立AN隧道信息，响应消息中包含AN Tunnel Info、 List of Accepted/Rejected QFI等。
9. 13a~13b步，SMF和AMF交互获取AN Tunnel Info等信息。
10. 14a~14b步，SMF向UL CL UPF+PSA1发送PFCP Session Modification Request，将(R)AN侧N3隧道下行端地址信息以及相应的PDR发送给UL CL UPF+PSA1，UL CL UPF+PSA1回应PFCP Session Modification Response。

**图4** UE位置区变化或在PCF侧签约发生变化下的UL CL流程

![](实现原理_29033546.assets/zh-cn_image_0278598521_2.png)

1. UE激活成功，创建了到主锚点PSA0 UPF的会话。
2. PCF通过向AMF或SMF订阅UE位置信息感知到UE位置发生变化或其签约信息发生变化。
3. 1a~1b步，PCF决策出需要更新的策略信息，通过Npcf_SMPolicyControl_UpdateNotify Request消息，发送新的策略信息给SMF，携带新的User Profile信息。SMF收到请求后回应Npcf_SMPolicyControl_UpdateNotify Response消息。
4. 2a~2b步，SMF根据新的User Profile信息判定需要插入UL CL UPF，则首先要和主锚点PSA0 UPF交互，SMF生成新的N9/N3接口的PDR通过PFCP Session Modification Request消息带给PSA0 UPF，PSA0 UPF返回Response消息携带新的N9/N3接口隧道地址。
5. 3a~3b步，SMF选择UL CL UPF+PSA1，并发送PFCP Session Establishment Request消息，携带对应的分流规则App ID。UL CL UPF+PSA1收到请求后回应Response消息，携带分配的N3上行/N9下行隧道端点（UL CL UPF角色）和N9隧道上行端点（辅锚点UPF角色）。
6. 4a~4b步，SMF通知主锚点PSA UPF更新N9隧道下行对端端点信息，主锚点PSA UPF返回响应。
7. 5a~5b步，SMF给新UL CL UPF+PSA1发送N4会话更新请求，为UL CL UPF角色下发N9隧道上行对端端点，为辅锚点UPF角色下发N9隧道下行对端端点。UL CL UPF+PSA1回复N4会话更新响应。
8. 第6步，SMF发送Namf_Communication_N1N2MessageTransfer Request消息，携带N2 SM Information通知(R)AN侧，将N3隧道上行对端隧道信息更新为UL CL UPF。
9. 7a~7b步，SMF和AMF交互获取AN Tunnel Info等信息。
10. 8a~8b步，SMF向UL CL UPF+PSA1发送PFCP Session Modification Request，将(R)AN侧N3隧道下行端地址信息以及相应的PDR发送给UL CL UPF+PSA1，UL CL UPF+PSA1回应PFCP Session Modification Response。
