#### [相关概念](#ZH-CN_TOPIC_0000001603922473)

- **正常状态**
  当前没有进行HTR流控，则为“正常状态”。
- **流控状态**
  当前正在进行HTR流控，则为“流控状态”。
- **“正常状态” -> “****流控状态”**
  如果网元当前处于“正常状态”且满足“HTR流控起控条件”则进行HTR流控，同时状态变更为“流控状态”。
- **“流控状态”->****“正常状态”**
  如果网元当前处于“流控状态”且满足“HTR流控解控条件”则解除HTR流控，同时状态变更为“正常状态”。
- **启控失败率阈值**
  当网元处于“正常状态”，HTTP局向由于对端网元过载导致消息失败的失败率连续N个周期（N默认为4，由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “启控周期数（STARTTIMES）” 参数配置）超过该阈值时，网元会触发HTR流控。该阈值默认为12%，可以由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “启控失败率阈值(%)（STARTTHD）” 参数配置。
- **解控失败率阈值**
  当网元处于“流控状态”，经过流控调整，HTTP局向由于对端网元过载导致消息失败的失败率连续M个周期（M默认为8，由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “解控周期数（STOPTIMES）” 参数配置）不高于该阈值时，网元会停止HTR流控。该阈值默认为1%，可以由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “解控失败率阈值(%)（STOPTHD）” 参数配置。
- **目标失败率阈值**
  位于启控失败率阈值和解控失败率阈值之间，失败率位于该阈值与启控失败率阈值之间时，不调整放通消息量。该阈值默认为6%，可以由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “目标失败率阈值(%)（TARGETTHD）” 参数配置。
- **统计周期**
  HTR流控进行消息量统计、失败率计算的周期。默认5秒，可通过 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “统计周期(s)（PERIOD）” 参数进行配置。
- **HTR超时时长**
  当周边网元严重过载时，可能无法回复HTTP响应。当网元超过该时长还没收到周边网元回复的HTTP响应，则认为周边网元严重过载了。此场景会统计为由于周边网元过载导致消息失败。超时时长默认为2秒，可通过 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “HTR超时时长(ms)（TIMEOUT）” 参数配置。
- **初始放通量**
  初始放通量是指网元由正常状态进入HTR流控状态后，第一个HTR流控周期内，HTTP进程每秒放通到HTTP局向的首消息的消息数量。
- **放通阈值减少比例**
  当网元处于“流控状态”，消息失败率超过启动失败率阈值时，需要减少放通的消息量。放通阈值减少比例即在此场景下，每个统计周期相较于上一周期要减少的比例。默认值为10%，可以由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “放通阈值减少比例(%)（DECRATIO）” 参数配置。
- **放通阈值增加比例1**
  当网元处于“流控状态”，消息失败率大于解控失败率阈值但不大于目标失败率阈值时，需要增加放通的消息量。放通阈值增加比例1即在此场景下，每个统计周期相较于上一周期要增加的比例。默认值为5%，可以由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “放通阈值增加比例1(%)（INCRATIO1）” 参数配置。
- **放通阈值增加比例2**
  当网元处于“流控状态”，消息失败率不大于解控失败率阈值时，需要增加放通的消息量。放通阈值增加比例2即在此场景下，每个统计周期相较于上一周期要增加的比例。默认值为18%，可以由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “放通阈值增加比例2(%)（INCRATIO2）” 参数配置。
- **安全边界**
  HTR流控调整的消息放通量不会超出配置的安全边界。可通过 **[ADD HTTPHTRMS](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR流控安全边界管理/增加HTR流控安全边界配置（ADD HTTPHTRMS）_85910089.md)** 基于HTTP POD粒度配置各接口的流控安全边界。

#### [过载失败消息识别](#ZH-CN_TOPIC_0000001603922473)

HTR流控用于后端网元出现过载的场景，AMF/SMF/NCG通过如下方法识别后端网元出现过载：

- AUSF/UDM/PCF/OCS回复响应消息超时。
- 直连组网场景：AUSF/UDM/PCF/OCS回复的响应消息中HTTP状态码为503或者429。
  ![](实现原理_03922473.assets/zh-cn_image_0000001553123274_2.png)
- 间接路由组网场景：AUSF/UDM/PCF/OCS回复HTTP状态码为503或者429的响应消息给SCP，SCP透传给AMF/SMF/NCG，并携带Server域字段“<NFType>-<NF Instance ID>”。
  ![](实现原理_03922473.assets/zh-cn_image_0000001604402073_2.png)

#### [失败率统计](#ZH-CN_TOPIC_0000001603922473)

HTR流控基于HTTP局向统计消息量和失败率等，HTTP局向通过命令 **[ADD HTTPOFC](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP管理/HTTP局向管理/增加HTTP局向（ADD HTTPOFC）_35230482.md)** 进行配置，可以基于NF类型也可以基于IP组配置，推荐使用基于NF类型的配置方式。

按照配置的HTR流控周期，网元内的HTTP进程单独统计各HTTP局向对应的HTTP消息计算失败率，不同的消息类型单独统计，包括各请求消息数、成功消息数和过载失败消息数。各种 消息的失败率分开统计，取最高的失败率作为局向的失败率 。

- **HTR局向基于IP组：**一个IP组下面可以是单个网元的IP也可以是多个网元的IP，需要汇总统计到这个IP组下面所有IP的请求消息数、成功消息数、过载失败消息数。
- **HTR局向基于NF类型**：通过本端NF类型和对端NF类型的组合对应到一种接口，统计这种接口的请求消息数、成功消息数和过载失败消息数。

#### [流控算法](#ZH-CN_TOPIC_0000001603922473)

**启控条件**

当基于HTTP局向统计同时满足以下条件时，网元触发HTR流控，并上报 **[ALM-100576 HTTP HTR流控](../../../../../网络运维/故障处理/UNC告警处理/平台告警/ALM-100576 HTTP HTR流控_01513825.md)** 告警：

- 网元到该局向有正常的HTTP链路
- 首消息过载失败率连续N个周期大于起控失败率阈值
- N个连续周期中至少一个周期的过载失败的消息不全部是HTR超时导致的失败

其中，N默认为4，由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “启控周期数（STARTTIMES）” 参数配置。

满足以上条件但首消息的速率不大于安全边界中配置的最小流控速率时，仅上报 **[ALM-100576 HTTP HTR流控](../../../../../网络运维/故障处理/UNC告警处理/平台告警/ALM-100576 HTTP HTR流控_01513825.md)** 告警，不进行消息放通量调整。

**动态调整消息放通量**

网元进入流控状态后，从第二个周期开始，每个流控周期HTTP进程都需要根据前一个流控周期基于HTTP局向统计的过载失败率以及消息放通量计算本周期的HTR消息放通量。

*表1 消息放通量调整*

| 过载失败率 | 消息放通量 |
| --- | --- |
| 大于启控失败率阈值 | 上一周期消息放通量 * （1- 放通阈值减少比例）<br>说明：上一周期消息放通量小于安全边界的最小流控速率时，不调整消息放通量。 |
| 大于目标失败率阈值但不大于启控失败率阈值 | 保持不变 |
| 大于解控失败率阈值但不大于目标失败率阈值 | 上一周期消息放通量 * （1+ 放通阈值增加比例1） |
| 不大于解控失败率阈值 | 上一周期消息放通量 * （1+ 放通阈值增加比例2） |

> **说明**
> 当连续两个周期没有由于HTTP状态码为503或者429而失败的首消息时，系统认为过载失败率为0，消息放通量按照（上一周期消息放通量 * （1+ 放通阈值增加比例2））进行调整。

AMF网元由于HTR流控导致鉴权流程失败而回复给UE的拒绝原因值可以通过 **[SET NGMMPROCTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM流程管理/5G移动性管理流程控制参数/设置5G移动性管理流程控制参数（SET NGMMPROCTRL）_09652386.md)** 命令的 “AUTH HTR流控拒绝原因值（AUTHHTTPHTRFC）” 参数进行配置；由于HTR流控导致获取签约数据失败而回复给UE的拒绝原因值可以通过 **[SET NGMMPROCTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM流程管理/5G移动性管理流程控制参数/设置5G移动性管理流程控制参数（SET NGMMPROCTRL）_09652386.md)** 命令的 “获取UDM签约数据HTR流控拒绝原因值（UDMSUBHTTPHTRFC）” 参数进行配置。

SMF网元发送给UDM/PCF的请求消息由于HTR流控导致会话创建流程失败而回复给AMF的NAS原因值可以根据需要进行配置。配置方法是通过命令 **[ADD SMCAUSEGRP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/SM原因值管理/原因值映射组管理/增加SM原因值映射组配置（ADD SMCAUSEGRP）_09651511.md)** 和 **[ADD SMCAUSEMAP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/SM原因值管理/原因值映射管理/增加SM原因值映射配置（ADD SMCAUSEMAP）_09652445.md)** 配置原因值映射组，并通过 **[ADD SMFCAUSECTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/SM原因值管理/5GC会话流程原因值控制/增加SMF流程控制参数（ADD SMFCAUSECTRL）_09652124.md) 、 [ADD SPGWCAUSECTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/SM原因值管理/GTPv2会话流程原因值控制/增加SGW-C_PGW-C原因值控制参数（ADD SPGWCAUSECTRL）_09653621.md)** 和 **[ADD GGSNCAUSECTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/SM原因值管理/GTPv1 PDP流程原因值控制/增加GGSN-C原因值控制参数（ADD GGSNCAUSECTRL）_25120871.md)** 将原因值映射组与UDM/PCF网元类型和会话创建流程绑定。

**解控条件**

当HTTP局向同时满足以下条件时，停止HTR流控，并恢复 **[ALM-100576 HTTP HTR流控](../../../../../网络运维/故障处理/UNC告警处理/平台告警/ALM-100576 HTTP HTR流控_01513825.md)** 告警：

- 连续M个周期，基于HTTP局向统计的首消息速率小于等于放通量。
- 连续M个周期，基于HTTP局向统计的首消息失败率小于等于解控失败率阈值。

其中，M默认为8，由 **[SET HTTPHTRCFG](../../../../../OM参考/命令/UNC MML命令/平台服务管理/HTTP功能管理/HTTP流控管理/HTTP HTR流控管理/HTTP HTR全局配置管理/设置HTR流控全局配置（SET HTTPHTRCFG）_35071002.md)** 命令的 “解控周期数（STOPTIMES）” 参数配置。
