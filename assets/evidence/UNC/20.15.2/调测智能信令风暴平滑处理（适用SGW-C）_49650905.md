# 调测智能信令风暴平滑处理（适用SGW-C）

- [操作场景](#ZH-CN_OPI_0249650905__1.3.1)
- [必备事项](#ZH-CN_OPI_0249650905__1.3.2)
- [操作步骤](#ZH-CN_OPI_0249650905__1.3.3)

## [操作场景](#ZH-CN_OPI_0249650905)

当运营商需要部署SGW-C上智能信令风暴平滑处理时，需要调测该功能。

> **说明**
> 适用于SGW-C。

## [必备事项](#ZH-CN_OPI_0249650905)

前提条件

- 请仔细阅读[WSFD-010804 智能信令风暴平滑处理特性概述](特性概述_62734662.md)。
- 完成[激活智能信令风暴平滑处理（适用SGW-C）](激活智能信令风暴平滑处理（适用SGW-C）_49650904.md)。

数据

该操作无需准备数据。

工具

OM Portal跟踪

## [操作步骤](#ZH-CN_OPI_0249650905)

1. 在OM Portal上建立测试终端的用户跟踪。
2. 测试终端接入网络，等待UE进行空闲态。
3. 构造MME和SGW-C之间的S11接口故障，并触发用户的下行寻呼，导致触发延时删除功能。
4. 使用命令 **[DSP DDNTIMEOUTCNT](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/流控管理/DDN消息无响应处理策略/显示DDN超时延时删除功能运行信息记录（DSP DDNTIMEOUTCNT）_46314491.md)** 查询用户是否在当前处于DDN无响应延时删除状态，即 **current delay delete users number** 值不为0。
5. 恢复S11接口业务状态，验证DDN无响应业务恢复。
6. 再次执行 **[DSP DDNTIMEOUTCNT](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/流控管理/DDN消息无响应处理策略/显示DDN超时延时删除功能运行信息记录（DSP DDNTIMEOUTCNT）_46314491.md)** 命令，查看用户是否在当前处于DDN无响应延时删除状态，即 **restore from delay delete users number** 值不为0。
    - 如果是在从DDN无响应延时删除状态恢复，则DDN无响应延时删除状态恢复调测结束。
    - 如果不是在从DDN无响应延时删除状态恢复，请执行 [7](#ZH-CN_OPI_0249650905__step138845122215) 。
7. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务，执行[2](#ZH-CN_OPI_0249650905__step3341421184016)~[6](#ZH-CN_OPI_0249650905__cmd1808113204317)并保存报文。
    b. 执行**EXP MML**命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
