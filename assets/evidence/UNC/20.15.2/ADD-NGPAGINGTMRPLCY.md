## [命令功能](#ZH-CN_MMLREF_0209652229)

**适用NF：AMF**

该命令用于增加NG寻呼定时器策略。AMF可基于PDU会话的DNN、ARP、5QI以及PPI属性配置寻呼的定时器时长以及重寻呼的次数。

## [注意事项](#ZH-CN_MMLREF_0209652229)

- 该命令执行后立即生效。

- 不能为相同的DNN，FQI，ARP和PPI指定多个寻呼定时器策略。对于指定的TAList，该命令配置之后，使用SET NGMMPARA命令配置的T3513(秒)，N3513(次数)，重寻呼间隔递增值(s)将失效。
- 当用户所在TALIST的TAC在ADD NGTALISTPAGINGCFG配置的指定TAC区间范围内，则AMF不重发用户基于TALIST的寻呼，ADD NGPAGINGTMRPLCY命令配置的T3513(秒)，N3513(次数)将失效。

- 最多可输入2000条记录。

#### [操作用户权限](#ZH-CN_MMLREF_0209652229)

G_1，管理员级别命令组；G_2，操作员级别命令组

## [参数说明](#ZH-CN_MMLREF_0209652229)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PLCYIDX | 策略索引 | 可选必选说明：必选参数<br>参数含义：该参数用于在AMF中唯一标识一条寻呼定时器策略。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~1999。<br>默认值：无<br>配置原则：无 |
| DNN | 数据网络名称 | 可选必选说明：可选参数<br>参数含义：该参数用于指定应用5G寻呼定时器策略的数据网络名称。<br>数据来源：全网规划<br>取值范围：字符串类型，输入长度范围是0~63。可输入的字符有字母、十进制数字、“-”和“.”，并且开头和结尾只能是数字或者字母。不能出现连续两个“.”。字母大小写不敏感。<br>默认值：无<br>配置原则：<br>输入单空格将删除该参数已有配置项。 |
| FQI | 5G QoS标识 | 可选必选说明：可选参数<br>参数含义：该参数用于指定应用5G寻呼定时器策略的5G QoS标识。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是1~255。<br>默认值：无<br>配置原则：无 |
| ARPPL | ARP优先级别 | 可选必选说明：可选参数<br>参数含义：该参数用于指定应用5G寻呼定时器策略的ARP优先级别。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是1~15。<br>默认值：无<br>配置原则：无 |
| PPI | 寻呼策略指示 | 可选必选说明：可选参数<br>参数含义：该参数用于指定应用5G寻呼定时器策略的寻呼策略指示。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是0~7。<br>默认值：无<br>配置原则：<br>系统初始值为“255”，有效的PPI取值范围是0-7。 |
| T3513 | T3513(s) | 可选必选说明：可选参数<br>参数含义：此定时器用于控制AMF发起寻呼与UE响应的时间间隔。此定时器在AMF发送Paging Request消息后启动，在收到Service Request消息后停止；超时后，AMF重发Paging Request消息。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是3~12，单位是秒。<br>默认值：无<br>配置原则：无 |
| N3513 | N3513(times) | 可选必选说明：可选参数<br>参数含义：该参数用于指定在寻呼流程中，没有收到UE的响应消息，AMF重复发送Paging Request消息的最大次数。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~5，单位是次。<br>默认值：无<br>配置原则：无 |
| PAGINGDELTA | 重寻呼间隔递增值(s) | 可选必选说明：可选参数<br>参数含义：该参数用于指定在寻呼流程中，没有收到Service Request消息，AMF重复发送Paging Request消息的间隔递增时间值。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~20，单位是秒。<br>默认值：0<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0209652229)

增加一条NG定时器寻呼策略配置：策略索引为1，数据网络名称为"huawei.com"，T3513时间间隔为5，N3513重发次数为1，执行如下命令：

```
ADD NGPAGINGTMRPLCY: PLCYIDX=1,DNN="huawei.com",T3513=5,N3513=1;
```
