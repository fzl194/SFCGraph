#### [81203640 LKV2ABTVPB01 灰度升级解决方案增值包基本功能-UAM](#ZH-CN_TOPIC_0000001615088190)

| 适用NF | AMF/MME/SGSN |
| --- | --- |
| **功能描述** | 在使用普通升级方式对网元升级时，升级过程中会面临整系统复位，业务中断，用户被迫下线，冗余资源过大，升级过程复杂等问题。针对上述问题，UNC基于<br>Kubernetes<br>原生能力实现<br>Pod<br>滚动升级<br>，将业务网元的实例按规划分成若干批次，分批次逐步升级到高版本。当系统进行无损升级时，业务中断时间短，用户不用被迫下线，仅需预留少量冗余资源，在满足业务无损的情况下即可实现网元的升级。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 |
| **默认值** | 0 |
| **对应特性** | WSFD-227101 在线升级 |
| **相关控制项** | 无 |
| **应用场景** | 如果需要对网元进行升级操作，同时需要满足中断时间短和少量预留冗余资源时，可以使用无损升级的方式进行升级操作。 |

| **取值范围** | 0～1 UNC |
| **默认值** | 1 |
| **对应特性** | WSFD-209002 磁盘故障隔离 |
| **相关控制项** | 无 |
| **应用场景** | 存储设备通过网络和计算节点连接，其中导致存储设备整体故障的因素很多，包括供电中断、网络中断、网络拥塞丢包等，都会导致存储异常。对于5G Core网元，大部分业务都对存储依赖很低，存储设备故障如果保证业务不受损，可以大大提升产品竞争力。 |

#### [81203853 LKV2SHFCUNC01 服务化接口HTR流控 - UNC](#ZH-CN_TOPIC_0000001615088190)

| 适用NF | SMF/AMF/NCG |
| --- | --- |
| **功能描述** | 本License控制UNC的周边网元存在过载风险时，UNC能否基于周边网元对于用户各业务流程首消息返回的失败应答数的周期变化，自动调整向后端网元发送的请求消息数，避免后端网元出现严重过载，达到保护周边网元的目的。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 UNC |
| **默认值** | 1 |
| **对应特性** | WSFD-206009 网元间HTR流控 |
| **相关控制项** | 无 |
| **应用场景** | 由于5G网络部署越来越广泛，5G用户量剧增，UNC各周边接口和网元拥塞的可能性也越来越大，为了保证用户接通率、改善用户体验，引入网元间HTR流控功能。应用场景包括：<br>- 升级过程中系统重启该场景可以预期有大量用户重新注册，用户到周边网元的请求同时发起。系统重启前开启网元间HTR流控，预防周边链路拥塞或周边网元业务过载。<br>- 故障倒换该场景会突发大量到周边网元的接入请求消息，致使接口链路拥塞、相应网元过载，业务成功率明显下降。在倒换前提前开启本功能可以降低信令拥塞对周边网元的影响。 |
