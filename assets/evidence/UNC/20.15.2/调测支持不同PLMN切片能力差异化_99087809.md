# 调测支持不同PLMN切片能力差异化

- [操作场景](#ZH-CN_OPI_0000001299087809__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001299087809__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001299087809__1.3.3)

## [操作场景](#ZH-CN_OPI_0000001299087809)

本操作指导介绍在运行网络中调测支持不同PLMN下网络切片能力差异化功能的操作过程。

AMF、SMF支持多个PLMN，不同PLMN下的网络切片支持的能力可能不同，在业务识别时需要对不同PLMN下的网络切片进行区分，以满足运营商支持不同PLMN下网络切片能力差异化的需求。

> **说明**
> 适用于
>
> AMF、SMF、NRF。

## [必备事项](#ZH-CN_OPI_0000001299087809)

前提条件

- 请仔细阅读[WSFD-110009 支持不同PLMN切片能力差异化特性概述](WSFD-110009 支持不同PLMN切片能力差异化特性概述_91130449.md)。

- 完成[激活支持不同PLMN切片能力差异化](激活支持不同PLMN切片能力差异化_52527766.md)。

数据

该操作无需准备数据。

工具

该操作无需准备其他工具。

## [操作步骤](#ZH-CN_OPI_0000001299087809)

1. 进入 “MML命令行-UNC” 窗口。
2. 执行 [**LST NFNRFMGMTPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/服务化接口管理/注册与服务发现/NRF管理/NRF参数管理/NF与NRF间全局管理参数管理/查询NF与NRF间的全局管理参数（LST NFNRFMGMTPARA）_32041711.md) 命令，查询是否已配置NF向NRF注册及更新时支持携带perPlmnSnssaiList信元。
  预期结果： “注册携带PerPlmnSnssaiList开关” 的值为 “ON” 支持携带perPlmnSnssaiList信元，与已规划配置一致，执行 [步骤 3](#ZH-CN_OPI_0000001299087809__step17523144942811) 。

  ```
  %%LST NFNRFMGMTPARA:;%%
  RETCODE = 0 操作成功

  结果如下
  ------------------------
  注册携带PerPlmnSnssaiList开关 = ON
  （结果个数=1）

  ---- END
  ```
  异常结果：查询结果与已规划配置不一致，请参考 [激活支持不同PLMN切片能力差异化](激活支持不同PLMN切片能力差异化_52527766.md) 重新配置。
3. 执行 **[LST NFDISCPLCY](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/服务化接口管理/注册与服务发现/本地NRF功能管理/NF发现策略管理/查询NF的服务发现策略（LST NFDISCPLCY）_09651379.md)** 命令，查询是否已配置NF缓存服务发现时支持根据perPlmnSnssaiList匹配发现参数。
  预期结果： “NF缓存服务发现是否支持perPlmnSnssaiList开关” 的值为 “ON” ，与已规划配置一致，执行 [步骤 4](#ZH-CN_OPI_0000001299087809__step1595917105445) 。

  ```
  %%LST NFDISCPLCY:;%%
  RETCODE = 0 操作成功

  结果如下
  ------------------------
  NF缓存服务发现是否支持perPlmnSnssaiList开关 = ON
  （结果个数=1）

  ---- END
  ```
  异常结果：查询结果与已规划配置不一致，请参考 [激活支持不同PLMN切片能力差异化](激活支持不同PLMN切片能力差异化_52527766.md) 重新配置。
4. 执行 [**LST NGHPLMN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/运营商管理/Home PLMN信息管理/查询5G Home PLMN（LST NGHPLMN）_09653743.md) 命令，查询运营商的Home PLMN。
  预期结果：运营商Home PLMN信息与已规划配置一致，执行 [步骤 5](#ZH-CN_OPI_0000001299087809__step1297054132111) 。

5. 执行 [**LST NGSRVPLMN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/运营商管理/Serving PLMN信息管理/查询5G Serving PLMN（LST NGSRVPLMN）_09652615.md) 命令，查询运营商的Serving PLMN。
  预期结果：运营商Serving PLMN信息与已规划配置一致，执行 [步骤 6](#ZH-CN_OPI_0000001299087809__step14122181311225) 。

6. 执行 [**LST H2SRVPLMN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/运营商管理/Home PLMN到Serving PLMN关联信息管理/查询Home PLMN到Serving PLMN的对应关系（LST H2SRVPLMN）_48763590.md) 命令，查询运营商的Home PLMN到Serving PLMN的对应关系。
  预期结果：Home PLMN到Serving PLMN的对应关系与已规划配置一致，执行 [步骤 7](#ZH-CN_OPI_0000001299087809__step181871143112212) 。

7. 执行 [**LST PLMNNS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/运营商管理/PLMN内网络切片管理/查询网络切片（LST PLMNNS）_09652528.md) 命令，查询指定的PLMN新增支持的网络切片。
  预期结果： “PLMN 索引” 为 “1” 、 “2” 的PLMN已分别新增了 “010101” 、 “010111” 、 “FFFFFF” 的网络切片，与已规划配置一致，执行 [步骤 8](#ZH-CN_OPI_0000001299087809__step12389613102315) 。

  预期结果：SMF已配置了 “网络切片索引” 为 “1” 、 “2” 、 “3” 的切片，与已规划配置一致，执行 [步骤 9](#ZH-CN_OPI_0000001299087809__step20255174220910) 。

9. 进行业务测试。
    a. 在UNC上创建用户跟踪任务和HTTP接口跟踪任务。
    b. SMF向NRF发起注册流程。
    c. HTTP接口跟踪中观察，注册请求消息中，SMF的NFProfile信息是否携带PerPLmnSnssailist信元。
          - 预期结果：携带perPlmnSnssaiList信元，执行 [9.d](#ZH-CN_OPI_0000001299087809__li157611443115219) 。
            ![](调测支持不同PLMN切片能力差异化_99087809.assets/zh-cn_image_0000001327031480_2.png)
          - 异常结果，未携带perPlmnSnssaiList信元，执行[步骤 10](#ZH-CN_OPI_0000001299087809__step16856162643819)。
    d. 测试终端接入网络，访问**www.huawei.com**网页。
    e. HTTP接口跟踪中观察，AMF向NRF发起SMF服务发现流程NFDiscover request，NRF向AMF返回的响应中携带NFProfile，NFProfile中是否包含SMF实例的perPlmnSnssaiList。
          - 预期结果，NFProfile中包含SMF实例的perPlmnSnssaiList，“nfInstanceId”为“00000004-0000-0000-000000000001”，执行[9.f](#ZH-CN_OPI_0000001299087809__li1859815691020)。
            ![](调测支持不同PLMN切片能力差异化_99087809.assets/zh-cn_image_0000001379239129_2.png)
            ![](调测支持不同PLMN切片能力差异化_99087809.assets/zh-cn_image_0000001382244233_2.png)
          - 异常结果，NFProfile中未包含SMF实例的perPlmnSnssaiList，执行[步骤 10](#ZH-CN_OPI_0000001299087809__step16856162643819)。
    f. 重启测试终端。
    g. 测试终端再次访问**www.huawei.com**网页。
    h. 用户跟踪中观察，AMF使用缓存服务发现SMF。
          - 预期结果，匹配到目标SMF且SMF的InstanceId与[9.e](#ZH-CN_OPI_0000001299087809__li981885104717)中相同，调测结束。

            ![](调测支持不同PLMN切片能力差异化_99087809.assets/zh-cn_image_0000001330898754_2.png)
          - 异常结果，未匹配到目标SMF，执行[步骤 10](#ZH-CN_OPI_0000001299087809__step16856162643819)。
10. 收集信息并寻求技术支持。
    a. 在 “MML命令行-UNC” 窗口上执行命令 **[EXP MML](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md)** ，将当前配置数据导出为MML脚本文件并保存。
    b. 收集并保存上述所有查询信息。
    c. 收集归纳所有信息并联系华为技术支持解决。
