#### [IMS对 UNC 的要求](#ZH-CN_CONCEPT_0191686180)

在IMS网络中，主、被叫MS之间使用信令 PDP上下文 来传递 SIP 消息，使用媒体PDP上下文来传输媒体流。媒体PDP上下文与普通的交互类PDP上下文没有区别，SGSN无须特殊处理，信令PDP上下文需要SGSN进行特殊处理：

R5协议中，SGSN通过QoS参数中的IMS信令指示（Signalling Indication）和PDP上下文相关的L3和Gn消息中的PCO（Protocol Configuration Options）信元来支持IMS承载信令上下文的建立及信令的传递。

- IMS信令指示
  SGSN通过MS发送的PDP上下文消息中QoS IE携带的IMS信令指示来区分Interactive级别业务和IMS信令。若MS携带了IMS信令指示，可以使得IMS信令传输相对于其他交互式传输，在 RAN 侧具有更高的优先级、更低的延迟、峰值增强等特征。
  IMS信令指示只由RAN侧使用，SGSN将此参数传递给RAN侧和GGSN。
- PCO信元
  R5协议在PDP上下文相关的L3和Gn消息中增加了PCO（Protocol Configuration Options）信元，用于在MS与IMS之间传送协议参数、配置参数、错误码或者其他消息。PCO信元对SGSN透明。
  受到影响的用于携带可选信元PCO的L3消息如下：
    - Activate PDP context request
    - Activate PDP context accept
    - Activate PDP context reject
    - Activate Secondary PDP Context Request
    - Activate Secondary PDP Context Accept
    - Activate Secondary PDP Context Reject
    - Request PDP context activation
    - Request PDP context activation reject
    - Modify PDP context request (Network->MS)
    - Modify PDP context accept (MS->network)
    - Modify PDP context request (MS->network)
    - Modify PDP context accept (Network->MS)
    - Modify PDP Context Reject
    - Deactivate PDP context request
    - Deactivate PDP context accept
  受到影响的用于携带可选信元的PCO的Gn消息（GTP V1消息）如下：
    - Create PDP Context Request
    - Create PDP Context Response
    - Update PDP Context Request (by SGSN)
    - Update PDP Context Response(by GGSN)
    - Update PDP Context Request (by GGSN)
    - Update PDP Context Response(by SGSN)
    - Delete PDP Context Request
    - Delete PDP Context Response
    - PDU Notification Request
    - PDU Notification Reject Request

#### [IMS信令专用PDP上下文激活流程](#ZH-CN_CONCEPT_0191686180)

当MS由 GPRS 接入IMS时，MS和IMS CN 子系统之间的信令既可以承载在普通的PDP上下文之上，也可以承载在某个专用的PDP上下文之上，具体选择依赖于运营商的策略。

- 当承载于普通的PDP上下文之上时，对SGSN来说和普通的透明数据业务是一样的。
- 当承载于信令专用的PDP上下文时，SGSN将消息中携带的PCO信元以及QoS参数中的信令指示透传给RAN侧和GGSN。

1. 当MS准备接入IMS，并且期望给予IMS信令更高的处理优先级，MS可以激活一个专用于IMS信令的PDP上下文，即向SGSN发送“Activate PDP Context Request”消息。在该消息中，MS携带PCO IE，并且在其中携带0002H(IM CN Subsystem signalling Flag)参数。另外，如果想在空口方面也给予IMS信令更高的处理优先级，MS还会将UMTSQoS Profile IE的signalling Indication标志置位。
2. SGSN向GGSN发送“Create PDP Context Request”消息，消息中携带PCO信元和QoS信元。
3. GGSN根据收到的消息中信元所携带的参数进行相应处理，并反馈给SGSN“Create PDP Context Response”消息，消息中携带PCO信元和QoS信元。
4. SGSN根据从GGSN处接收到的消息进行相应处理。如果QoS信元中携带了Signalling Indication，SGSN通过Radio Access Bearer Setup过程，在RAN侧分配给MS更高的优先级。
5. SGSN向MS发送“Activate PDP Context Accept”消息，消息中携带PCO信元和QoS信元。
6. MS在PS域与IMS实体间建立信令交互，可以进行IMS信令传输。
