# 过程2：请求信息纠正

正常的签约数据进行匹配后，如果签约数据匹配失败，则需要进行请求信息纠正，以提升PDP/PDN请求签约匹配的成功率。 UNC 根据匹配失败情况，依次纠正PDP/PDN Type错误、PDP/PDN Address错误、APN错误。 UNC 每纠正一项，都会和本地存储的用户签约数据进行再匹配，再匹配成功后，再进行下一项的错误纠正。如果再匹配失败，则拒绝UE的PDP/PDN激活请求，激活流程失败。

请求信息纠正流程说明如下：

1. 查看PDP/PDN Type、PDP/PDN Adress和APN是否均为空。
    - 是 => 执行[签约数据匹配不唯一纠正](#ZH-CN_TOPIC_0186228994__sub2_4)。执行[步骤3](#ZH-CN_TOPIC_0186228994__li4298141541113)。
    - 否 => 执行[步骤2](#ZH-CN_TOPIC_0186228994__li151785619486)。
2. 查看匹配结果是否是匹配到多组的情况。
    - 是 => 执行[签约数据匹配不唯一纠正](#ZH-CN_TOPIC_0186228994__sub2_4)。执行[步骤3](#ZH-CN_TOPIC_0186228994__li4298141541113)。
    - 否 => 执行[步骤4](#ZH-CN_TOPIC_0186228994__li1835220218562)。
3. 查看是否仍存在多组签约数据。
    - 是 => 执行[多签约APN纠正](#ZH-CN_TOPIC_0186228994__sub2_3_2)。进入“[过程3：签约数据匹配/再匹配成功后再处理](过程3：签约数据匹配_再匹配成功后再处理_86228995.md)”。
    - 否 => 进入“[过程3：签约数据匹配/再匹配成功后再处理](过程3：签约数据匹配_再匹配成功后再处理_86228995.md)”。
4. 执行[PDP/PDN Type错误纠正](#ZH-CN_TOPIC_0186228994__p957045915318)。
5. 执行[PDP/PDN Address错误纠正](#ZH-CN_TOPIC_0186228994__p823619910588)。
6. 查看APN是否为“-”等非法字符。
    - 是 => 执行[步骤7](#ZH-CN_TOPIC_0186228994__li6528195922117)。
    - 否 => 执行[步骤8](#ZH-CN_TOPIC_0186228994__li947154522618)。
7. 查看是否是网络侧发起的激活请求。
    - 是 => 拒绝PDP/PDN激活请求。
    - 否 => 进入“[过程3：签约数据匹配/再匹配成功后再处理](过程3：签约数据匹配_再匹配成功后再处理_86228995.md)”。
8. 查看APN是否为空。
    - 是 => 进入“[过程3：签约数据匹配/再匹配成功后再处理](过程3：签约数据匹配_再匹配成功后再处理_86228995.md)”。
    - 否 => 执行[步骤9](#ZH-CN_TOPIC_0186228994__li2071911210284)。
9. 查看APN是否签约多组。
    - 是 => 执行[多签约APN纠正](#ZH-CN_TOPIC_0186228994__sub2_3_2)。进入“[过程3：签约数据匹配/再匹配成功后再处理](过程3：签约数据匹配_再匹配成功后再处理_86228995.md)”。
    - 否 => 执行[单签约APN纠正](#ZH-CN_TOPIC_0186228994__sub2_3_1)。进入“[过程3：签约数据匹配/再匹配成功后再处理](过程3：签约数据匹配_再匹配成功后再处理_86228995.md)”。

> **说明**
> 在多PDN的场景下， UNC 按照PDN连接建立请求的先后顺序逐个进行请求信息纠正。

**签约数据匹配不唯一纠正**

如果在进行签约数据匹配时， UNC 匹配到多组签约数据，即匹配签约数据不唯一，则需要 UNC 从这多组匹配到的签约数据中选择一组。 UNC 按照以下步骤进行依次筛选，选取合适的一组签约数据。 UNC 根据MML命令 [**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md) 下的参数“多组签约数据”的设置进一步进行筛选。

1. 对于匹配到的多组签约数据中相同APN的签约数据，优选PDP/PDN Type为双栈的，删除PDP/PDN Type为单栈的。
2. 如果存在和UE的请求APN相同的签约数据，则选择此组签约数据。如果存在多组和UE的请求APN相同的签约数据，则选择其中Context ID最小的一组。
3. 如果不存在和UE的请求APN相同的签约数据，则选择野卡签约数据。如果存在多组野卡签约数据，则选择其中Context ID最小的一组。

**PDP/PDN Type错误纠正**

签约数据匹配失败后是否进行PDP/PDN Type错误纠正，受控于MML命令 [**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md) 下的参数“PDP Type纠正”。

- 如果用户请求的PDP/PDN Type为IPv4v6类型，不会导致PDP/PDN Type签约数据不匹配，无需纠正。
- 如果用户请求的PDP/PDN Type为IPv4，但是签约的PDP/PDN Type为IPv6，则纠正为IPv6。
- 如果请求的PDP/PDN Type为IPv6，但是签约的PDP/PDN Type为IPv4，则纠正为IPv4。

**PDP/PDN Address错误纠正**

如果用户请求的PDP/PDN Address为空，代表用户允许使用任意的PDP/PDN Address。这种情况不会导致PDP/PDN Address签约数据不匹配，无需纠正。如果用户请求的PDP/PDN Address为静态（Address为某一指定IP），但是用户签约的PDP/PDN Address为动态，则纠正为动态。签约数据匹配失败后是否进行PDP/PDN Address错误纠正，受控于MML命令 [**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md) 下的参数“ PDP Address纠正”。

**单签约APN纠正**

单签约数据纠正方式通过MML命令 [**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md) 下的参数“单组签约数据”设置，纠正方式有如下三种：

- 不纠正（NO）：不进行APN纠正，直接拒绝PDP激活请求。
- 只选Local APNNI（SELECT_LOC_APN_ONLY）：
    - 对于接入网络的4G用户，查看HSS上的签约数据的APN是否为Local APN。如果是，选择该组签约数据。如果不是，则拒绝PDN激活请求。Local APN通过MML命令[**ADD PDPAPN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/本地APNNI管理/增加本地APN NI配置(ADD PDPAPN)_72345275.md)配置。若本地没有配置[**ADD PDPAPN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/本地APNNI管理/增加本地APN NI配置(ADD PDPAPN)_72345275.md)，则直接失败，拒绝PDN激活请求。
    - 对于接入网络的2/3G用户：如[图2](#ZH-CN_TOPIC_0186228994__fig2196177103316)所示。看本地是否通过[**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md)配置“要求签约APNNI”。
          - 如果未配置，查看Local APN是否已通过[**ADD PDPAPN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/本地APNNI管理/增加本地APN NI配置(ADD PDPAPN)_72345275.md)配置。若本地没有配置[**ADD PDPAPN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/本地APNNI管理/增加本地APN NI配置(ADD PDPAPN)_72345275.md)，则直接失败，拒绝PDP激活请求。如果已配置，再查看HLR上的签约数据的APN是否为Local APN。如果是，选择该组签约数据。如果不是，查看是否通过[**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md)配置“回落APNNI”。如果已配置，查看HLR上的签约数据的APN是否为回落APN。如果是，选择该组签约数据。如果不是，查看是否通过**[ADD APNPROFILE](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/APN功能配置/增加APN配置（ADD APNPROFILE）_67235537.md)**配置该APN的概要信息。如果配置了，选择该回落APN。如果没有配置，则拒绝PDP激活请求。如果没有配置“回落APNNI”，则拒绝PDP激活请求。
          - 如果已配置，查看HLR上的签约数据的APN是否为“要求签约APNNI”。如果不是，则拒绝PDP激活请求。如果是，与未配置“要求签约APNNI”处理流程相同。

**多签约APN纠正**

- 不纠正（NO）：不进行APN纠正，拒绝PDP/PDN激活请求。

          - 如果未配置，查看Local APN是否已通过[**ADD PDPAPN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/本地APNNI管理/增加本地APN NI配置(ADD PDPAPN)_72345275.md)配置。若本地没有配置[**ADD PDPAPN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/本地APNNI管理/增加本地APN NI配置(ADD PDPAPN)_72345275.md)，则直接失败，拒绝PDP激活请求。如果已配置，再查看HLR上的Sublist中是否存在Local APN的签约数据。如果存在，选择该组签约数据。如果不存在，查看是否通过[**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md)配置“回落APNNI”。如果已配置，查看HLR上Sublist中是否存在回落APN的签约数据。如果存在，选择该组签约数据。如果不存在，查看是否通过**[ADD APNPROFILE](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/APN功能配置/增加APN配置（ADD APNPROFILE）_67235537.md)**配置该APN的概要信息。如果配置了，选择该回落APN。如果没有配置，则拒绝PDP激活请求。如果没有配置“回落APNNI”，则拒绝PDP激活请求。

- 优选Local APNNI（SELECT_LOC_APN_FIRST）：
