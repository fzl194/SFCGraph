# 调测基于IMEI的语音策略控制

- [操作场景](#ZH-CN_OPI_0172659573__1.3.1)
- [必备事项](#ZH-CN_OPI_0172659573__1.3.2)

## [操作场景](#ZH-CN_OPI_0172659573)

调测基于IMEI的语音策略控制特性。

## [必备事项](#ZH-CN_OPI_0172659573)

前提条件

完成 [激活基于IMEI的语音策略控制](激活基于IMEI的语音策略控制_70014723.md) 。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0172659573)

- 营商A只允许特定型号终端的本网用户使用IMS VoPS业务，且优先使用VoLTE语音业务，不允许外网用户使用VoLTE语音业务。用户X和用户Y为运营商A本网用户，签约了IMS APN，都支持SRVCC，用户X的终端型号为运营商指定允许使用VoLTE语音业务的型号。用户Z为运营商A的外网用户，且签约了IMS APN。
  在相关网元均完成本特性的配置后，可以采用以下步骤检查特性工作是否正常：
    1. 分别创建用户X、Y、Z的用户跟踪。
    2. 用户X、Y、Z开机在LTE网络附着。
    3. 察MME向用户X、Y、Z发送的Attach Accept消息中EPS Network Feature Support信元的bit1的取值。验证结果如 [表1](#ZH-CN_OPI_0172659573__table567711115819) 所示。
      *表1 验证结果*

      | 用户 | EPS Network Feature Support信元的bit1 |
      | --- | --- |
      | X | **预期结果**<br>：bit1取值为1。 |
      | X | **异常结果**<br>：bit1取值为0。 |
      | Y | **预期结果**<br>：bit1取值为0。 |
      | Y | **异常结果**<br>：bit1取值为1。 |
      | Z | **预期结果**<br>：bit1取值为0。 |
      | Z | **异常结果**<br>：bit1取值为1。 |
      > **说明**
      > 当EPS Network Feature Support信元的所有bit值均为0时，Attach Accept消息中不携带此信元。
