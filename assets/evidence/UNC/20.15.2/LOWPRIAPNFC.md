#### [命令功能](#ZH-CN_MMLREF_0000001172345757)

![](设置低优先级APN流控参数(SET LOWPRIAPNFC)_72345757.assets/notice_3.0-zh-cn_2.png)

如果低优先级APN流控功能被关闭或者相关流控参数设置不正确，可能影响低优先级APN流控的效果。

**适用网元：MME**

该命令用于设置低优先级APN流控功能的相关参数。本功能在SPP上的MM模块部署。

低优先级用户（比如M2M）一般是和普通用户共用核心网设备，M2M业务很多都是行业应用，可能发生一个行业应用下的所有M2M终端在很短的时间内同时发起业务，导致移动网络拥塞（如一个电力公司下的所有电表在很短的时间内同时上报数据到电力公司），影响普通用户业务。为了防止这类低优先级业务影响其他用户的正常业务， UNC 系统能够基于当前CPU负荷，自动调节低优先级业务的接入消息允许处理速率。这些消息包括：Attach Request、Service Request消息。

#### [注意事项](#ZH-CN_MMLREF_0000001172345757)

- 该命令主要用于低优先级APN的接入控制，配置不当可能导致业务受损，建议保持系统初始设置值。如需修改，请联系华为技术支持。
- 该命令执行后立即生效。
- 系统初次上电运行时，会执行系统初始设置值。
- 此配置涉及基于延迟定时器的信令拥塞控制特性（特性编号：WSFD-215201，license部件编码：LKV2LTCC01），请在设置参数前使用[**DSP LICENSE**](../../../../../../../../平台服务管理/操作维护/License管理/显示License(DSP LICENSE)_00360098.md)命令确认对应特性license是否得到授权，执行[**LST LICENSESWITCH**](../../../../../../../../平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md)命令确认特性开关状态为“ENABLE(打开)”。

#### [参数说明](#ZH-CN_MMLREF_0000001172345757)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| FCSWITCH | 低优先级APN流控功能开关 | 可选必选说明：可选参数<br>参数含义：该参数用于设置是否开启低优先级APN流控功能。<br>数据来源：整网规划<br>取值范围：<br>- “ON(开启)”<br>- “OFF(关闭)”<br>系统初始设置值：<br>“OFF(关闭)”<br>配置原则：当低优先级APN用户（如M2M）业务可能影响普通用户正常业务时，可以通过设置该参数为<br>“ON(开启)”<br>控制低优先级APN业务接入。<br>说明：该参数与基于延迟定时器的信令拥塞控制特性license共同完成低优先级APN流控功能的开启。 |
| GRPID | APNNI组号 | 可选必选说明：条件必选参数<br>参数含义：本参数用于设置支持低优先级APN流控的APNNI组号。<br>前提条件：该参数在<br>“低优先级APN流控功能开关”<br>参数配置为<br>“ON(开启)”<br>时生效。<br>数据来源：本端规划<br>取值范围：0~15<br>系统初始设置值：无<br>配置原则：该参数的取值需要引用<br>[**ADD APNNIGROUP**](../../../../../业务安全管理/会话管理/APNNI信息管理/APNNI组管理/增加APNNI组(ADD APNNIGROUP)_26305508.md)<br>命令中的组号，因此在配置该参数前需要首先通过命令<br>[**ADD APNNIGROUP**](../../../../../业务安全管理/会话管理/APNNI信息管理/APNNI组管理/增加APNNI组(ADD APNNIGROUP)_26305508.md)<br>配置组号。 |
| MINTHD | 流控速率最小值（个/秒） | 可选必选说明：条件可选参数<br>参数含义：本参数用于控制每个SPP上每秒允许通过的低优先级APN接入消息包数的最小值，旨在将用户接入速率控制在一定范围内。<br>前提条件：该参数在<br>“低优先级APN流控功能开关”<br>参数配置为<br>“ON(开启)”<br>时生效。<br>数据来源：整网规划<br>取值范围：0～5000<br>系统初始设置值：10<br>说明：- 本参数设置越大，系统过载时允许处理的低优先级APN业务越多，可能对普通用户业务的影响越大。<br>- 本参数设置越小，系统过载时允许处理的低优先级APN业务越少，对普通用户业务的影响越小。<br>- 流控速率是基于Attach流程评估的能力，并非真正的流程消息通过量。 |
| MAXTHD | 流控速率最大值（个/秒） | 可选必选说明：条件可选参数<br>参数含义：本参数用于控制每个SPP上每秒允许通过的低优先级APN接入消息包数的最大值，旨在将用户接入速率控制在一定范围内。<br>前提条件：该参数在<br>“低优先级APN流控功能开关”<br>参数配置为<br>“ON(开启)”<br>时生效。<br>数据来源：整网规划<br>取值范围：0～5000<br>系统初始设置值：1000<br>配置原则：该参数的取值必须大于等于<br>“流控速率最小值”<br>的取值。<br>说明：- 本参数设置的越大，则在低优先级APN业务突发时对普通用户业务的影响越大。<br>- 本参数设置的越小，则在低优先级APN业务突发时对普通用户业务的影响越小，但是如果小于实际能力过多，可能在系统能容忍的有限的突发下发生过多的流控丢包。<br>- 流控速率是基于Attach流程评估的能力，并非真正的流程消息通过量。 |
| MINBOT | Back off timer最小值（秒） | 可选必选说明：条件可选参数<br>参数含义：本参数用于设置Back off timer的最小值，用于计算发给终端的附着拒绝消息中的Back off timer时长。<br>前提条件：该参数在<br>“低优先级APN流控功能开关”<br>参数配置为<br>“ON(开)”<br>后生效。<br>数据来源：本端规划<br>取值范围：1～11160<br>系统初始值：900 |
| MAXBOT | Back off timer最大值（秒） | 可选必选说明：条件可选参数<br>参数含义：本参数用于设置Back off timer的最大值，用于计算发给终端的附着拒绝消息中的Back off timer时长。<br>前提条件：该参数在<br>“低优先级APN流控功能开关”<br>参数配置为<br>“ON(开)”<br>后生效。<br>数据来源：本端规划<br>取值范围：1～11160<br>系统初始值：1800<br>配置原则：该参数的取值必须大于等于<br>“Back off timer最小值”<br>的取值。 |
| IMSI | 测试IMSI | 可选必选说明：条件可选参数<br>参数含义：本参数用于设置待调测用户的IMSI。<br>前提条件：该参数在<br>“低优先级APN流控功能开关”<br>参数配置为<br>“ON(开启)”<br>时生效。<br>数据来源：本端规划<br>取值范围：1～15位十进制数字<br>系统初始设置值：NULL<br>配置原则：本参数只用于调测场景。<br>说明：- 设置该参数后MME会强制拒绝本参数所指定IMSI对应用户的Attach流程和Service Request流程，并且携带T3346信元（Back off timer）。目的是为了测试终端在收到Back off timer后的行为。 |

#### [使用实例](#ZH-CN_MMLREF_0000001172345757)

当有大量的低优先级APN业务使系统过载，影响在网用户的普通业务，使用该命令对于低优先级APN进行接入限制，自动调节低优先级APN业务的接入消息允许处理速率。设置 “低优先级APN流控功能开关(FCSWITCH)” 参数为 “ON(开启)” 、 “APNNI组号(GRPID)” 参数为10、 “流控速率最小值(MINTHD)” 参数为10、 “流控速率最大值(MAXTHD)” 参数为300，配置格式如下：

SET LOWPRIAPNFC: FCSWITCH=ON, GRPID=10, MINTHD=10, MAXTHD=300;
