## [操作场景](#ZH-CN_OPI_0231422947)

GGSN/PGW-C与PCRF交互时，常见的异常场景如下：

- 基于号段选择PCRF分组失败
  特定号段的用户激活时选择不到号段绑定的PCRF分组。该种情况通常是由于GGSN/PGW-C上未基于号段绑定PCRF分组。
- 等待PCRF响应超时
  GGSN/PGW-C向PCRF发送CCR-I/U/T消息后，在Tx timer超时前，没有收到CCA响应消息。该种情况下可能是PCRF由于拥塞导致CCA回复失败，或者PCRF应用层故障导致无法正常回复CCA。DRA场景下还有可能是DRA的原因导致CCA消息丢失。
- Gx接口链路故障
  GGSN/PGW-C和PCRF/DRA之间无法建立连接，包括传输层TCP链路或者Diameter链路，导致网元间无法正常交互。该种情况下可能是物理链路故障，也可能是对端PCRF/DRA的链路层故障导致Diameter心跳消息DWR/DWA无法正常维护。
- CCA消息中携带异常返回码
  PCRF给GGSN/PGW-C回复的CCA-I/U/T响应消息中，如果Result-Code AVP或Experimental-Result-Code AVP携带了DIAMETER_SUCCESS(2001)和DIAMETER_LIMITED_SUCCESS(2002)以外的异常返回码，比如3XXX，4XXX，5XXX，表明业务层存在问题，可根据返回码的标准定义排查异常。
- CCA-I消息中下发的PCC规则全部安装失败
  按照协议要求，PCC用户成功激活的必要条件是必须安装有效的PCC规则。该场景中，PCRF成功响应CCA-I消息，也下发了PCC规则，但是PCC规则因未通过有效性检查而全部安装失败，例如PCRF下发的预定义规则未在网关上配置等。该种情况下通常是由于PCRF侧签约错误导致，此时GGSN/PGW-C会将该用户去活。
- CCA-I消息中未下发PCC规则
  该场景中，PCRF成功响应CCA-I消息，但未下发PCC规则。该种情况下通常是由于用户开户但未签约业务导致。
- Gx failover
  详细内容请参见 [Gx Failover功能](Gx Failover功能_31422950.md) 。

> **说明**
> 适用于 GGSN、 PGW-C。

## [任务示例](#ZH-CN_OPI_0231422947)

任务描述

规划GGSN/PGW-C在如下异常场景下的处理方式：

- 基于号段选择不到PCRF时，使用全局缺省的PCRF分组中PCRF进行交互，如果没有配置全局缺省的PCRF分组，且能匹配上本地策略，则激活为本地PCC用户，否则激活失败。
- Gx接口链路故障或CCA消息响应超时的情况下，用户激活时回滚为本地PCC用户；用户更新时回滚为本地PCC用户。
- GGSN/PGW-C收到3xxx异常返回码时，执行failover动作，使用容灾PCRF进行交互。

脚本

1. 配置Gx接口链路故障或CCA消息响应超时系统的处理动作。
  ```
  SET PCCFAILACTION:SELPEERFAILACT=DEFAULT,INITIALFAILACT=ROLLBACK,INITFAILLOCPCC=LOCAL_PCC,INITFAILRDSPCC=LOCAL_PCC,UPDATEFAILACT=ROLLBACK,UPDFAILLOCPCC=LOCAL_PCC;
  ```
  ```
  ADD PCCTEMPLATE:PCCTEMPNAME="templatetest",SELPEERFAILACT=DEFAULT,INITIALFAILACT=ROLLBACK,INITFAILRDSPCC=LOCAL_PCC,UPDATEFAILACT=ROLLBACK,UPDFAILLOCPCC=LOCAL_PCC;
  ```
  ```
  SET APNPCCFUNC:APN="apn-test",HOMEPCCSWITCH=ENABLE,ROAMPCCSWITCH=ENABLE,VISITPCCSWITCH=ENABLE,PCCTEMPLATE="templatetest";
  ```
2. 配置GGSN/PGW-C收到PCRF返回的异常码时的处理动作。
  ```
  ADD RESULTCODECTRL: PCCTEMPLATE="templatetest", INTFTYPE=INTFTYPE_GX, VENDORID=0, GXRESULTCODEVAL="3xxx", INITACTION=FAILOVER, UPDATEACTION=FAILOVER, TERMINACTION=FAILOVER;
  ```
3. **可选：**
  配置GGSN/PGW-C收到直连PCRF返回的异常码时的处理动作。
  ```
  ADD RESULTCODECTRL: PCCTEMPLATE="templatetest", INTFTYPE=INTFTYPE_GX, VENDORID=0, GXRESULTCODEVAL="3xxx", DIRECTINITACT=FAILOVER, DIRECTUPDACT=FAILOVER, DIRECTTERMINACT=FAILOVER;
  ```
