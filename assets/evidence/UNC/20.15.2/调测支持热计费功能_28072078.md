## [操作场景](#ZH-CN_OPI_0228072078)

本操作指导介绍在运行网络中调测支持热计费特性的操作过程。

> **说明**
> 适用于 GGSN、 SGW-C、PGW-C。

## [必备事项](#ZH-CN_OPI_0228072078)

前提条件

- 请仔细阅读[WSFD-011202 支持热计费功能（适用于GGSN、SGW-C、PGW-C）特性概述](特性概述_28072075.md)。
- 完成[激活支持热计费功能](激活支持热计费功能_28072077.md)。

## [操作步骤](#ZH-CN_OPI_0228072078)

1. 激活用户，测试终端使用“apn-test”APN接入网络，访问业务。
    - 如果测试终端成功接入网络，请执行[步骤 2](#ZH-CN_OPI_0228072078__stp2)。
    - 如果测试终端无法接入网络，调测UNC的接入功能，成功接入网络后请执行[步骤 2](#ZH-CN_OPI_0228072078__stp2)。
2. 用户访问业务，产生话单，一段时间超过设置的时间阈值或流量阈值后，去激活用户。
3. 登录CG客户端，查询此用户刚生成的话单。如 [图1](#ZH-CN_OPI_0228072078__fig1) 所示，查看所生成话单中chargingCharacteristics字段是否是热计费。
  > **说明**
  > 各计费在最终生成的话单中的chargingCharacteristics字段会有所区分。
  >
  > - 热计费，该字段显示是Hot billing。
  > - 普通计费，该字段显示是Normal billing。
  > - 预付费，该字段显示是Prepaid billing。
  > - 统一费率，该字段显示Flat billing。
  **图1** 用户计费属性查询

  <br>

  ![](调测支持热计费功能_28072078.assets/zh-cn_image_0228356598_2.png)

  <br>
    - 如果是Hot billing，表明热计费功能正常，本调测任务结束。
    - 如果不是Hot billing，请执行[步骤 4](#ZH-CN_OPI_0228072078__stp5)。
4. 执行 **[LST APNCHARGECTRL](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/APN计费配置/查询APN的计费配置（LST APNCHARGECTRL）_09896818.md)** 命令，查看此APN下是否绑定了正确的计费属性名 “Charging Characteristic名称” 。
  ```
  LST APNCHARGECTRL:APN="apn-test";
  ```
  ```
                 APN名称  =  apn-test
  ```
  ```
                        ......
  ```
  ```
            计费属性名称  =  testchgcha
                        ......
  ```
    - 如果是，请执行[步骤 5](#ZH-CN_OPI_0228072078__stp6)。
    - 如果否，请参考[配置计费属性CC（GGSN/SGW-C/PGW-C）](../../WSFD-011201 支持离线计费/激活离线计费/配置计费属性CC（GGSN_SGW-C_PGW-C）_95923501.md)重新配置该APN的计费属性实例，然后返回[步骤 1](#ZH-CN_OPI_0228072078__stp1)重新调测。
  > **说明**
  > - 如果配置AAA鉴权下发Charge-characteristic，则需要检查AAA鉴权响应消息是否下发Charge-characteristic:Hotbilling字段。
  > - 如果存在其他网元可以下发Charge-characteristic，需要检查其他网元是否下发Charge-characteristic:Hotbilling字段。
5. 执行 **[LST CHARGECHAR](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/基本计费属性/查询对本地用户、漫游用户、拜访用户所采用的计费属性（LST CHARGECHAR）_09896812.md)** 命令，查看对应用户类型下的计费属性是否为hotbilling(0x0100)。
  ```
  LST CHARGECHAR:CCNAME="testchgcha";             
  ```
  ```
                             计费属性名称 = testchgcha                                  
                         本地用户计费属性 = 0x0100                                  
                         漫游用户计费属性 = 0x0100                                  
                         拜访用户计费属性 = 0x0100                                  
         本地用户使用Serving Node计费属性 = 禁止                                     
         漫游用户使用Serving Node计费属性 = 禁止                                        
         拜访用户使用Serving Node计费属性 = 禁止
  ```
    - 如果是，请执行[步骤 6](#ZH-CN_OPI_0228072078__stp7)。
    - 如果否，请参考[配置计费属性CC（GGSN/SGW-C/PGW-C）](../../WSFD-011201 支持离线计费/激活离线计费/配置计费属性CC（GGSN_SGW-C_PGW-C）_95923501.md)重新配置计费属性实例，然后返回[步骤 2](#ZH-CN_OPI_0228072078__stp2)重新调测。
6. 查看后付费用户的计费选择配置情况，执行 [**LST CHARGEMETHOD**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/计费属性计费控制/查询计费方式（LST CHARGEMETHOD）_09896798.md) 命令。
    - 如果用户为后付费用户，而“Offline”配置为“DISABLE”，请将“Offline”配置为“ENABLE”并返回[步骤 2](#ZH-CN_OPI_0228072078__stp2)重新调测。
    - 如果用户为后付费用户，而“Offline”配置为“ENABLE”，请执行[步骤 7](#ZH-CN_OPI_0228072078__stp10)。
7. 查看计费开关配置情况，执行 **[LST CHARGECTRL](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/用户属性计费控制/查询计费控制配置（LST CHARGECTRL）_09896793.md)** 。
    - 如果用户所属类型被配置为“NONE”或者“ONLINE”，请重新配置为“OFFLINE”并返回[步骤 2](#ZH-CN_OPI_0228072078__stp2)重新调测。
    - 如果用户所属类型配置为“OFLINE”或者“BOTH”，请执行[步骤 8](#ZH-CN_OPI_0228072078__stp11)。
8. 收集信息并寻求技术支持。
    a. 执行 [**EXP MML**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    b. 收集并保存上述所有查询信息。
    c. 收集归纳所有信息并联系华为技术支持解决。
