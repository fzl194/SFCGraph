- [操作场景](#ZH-CN_OPI_0000001251740325__1.3.1)

## [操作场景](#ZH-CN_OPI_0000001251740325)

当网络中存在基站与UPF链路故障或者N4接口故障时，可以参考本章节激活UPF故障隔离特性，使SMF在选择UPF时屏蔽故障的用户面网元。

> **说明**
> 适用于SMF。

## [必备事项](#ZH-CN_OPI_0000001251740325)

前提条件

- 请仔细阅读[WSFD-213005 UPF故障隔离特性概述](特性概述_06900390.md)。
- 已在SMF侧通过**ADD UPNODE**增加故障的UPF。
- 已激活WSFD-107010 UPF选择特性，详细操作请参考[WSFD-107010 UPF选择](../../分布式网络功能/WSFD-107010 UPF选择_10789013.md)。

## [任务示例](#ZH-CN_OPI_0000001251740325)

任务描述

任务1：在SMF侧增加故障的UPF和基站，使SMF在基站与UPF之间的接口故障场景选择UPF时能够过滤故障的网元，实现UPF故障隔离功能。

任务2：在SMF上配置会话消息检测N4接口故障相关配置。

脚本

**任务1脚本：**

//增加故障RAN组和故障基站。

```
ADD FAULTRANGRP: RANGRPNAME="RANGRP1";
ADD FAULTRANBINDGRP: PLMN="12399", RANNODETYPE=GNB, RANNODEID=327697, NODEIDLEN=24, RANGRPNAME="RANGRP1";
```

//增加故障UPF组和故障UPF。

```
ADD FAULTUPGRP: UPFGRPNAME="UPFGRP1";
ADD FAULTUPBINDGRP: UPFID="UPF1", UPFGRPNAME="UPFGRP1";
```

//绑定故障的RAN组和UPF组。

```
ADD FAULTUPBINDRAN: RANGRPNAME="RANGRP1", UPFGRPNAME="UPFGRP1";
```

**任务2的脚本：**

//配置不同类型消息的超时间隔和发送次数阈值。此处理以PFCP Modification Request消息为例。

```
SET T1N1PARA: MSGTYPE=SessionModify, T1=2, N1=2;
```

//使能PFCP修改请求消息重发优化开关。

```
SET PFCPPARA: HBINTERVAL=30, HBT1=3, HBN1=5, DEACTIVEFLAG=ENABLE, DACTINTERVAL=900, MODREQRSOPTSW=ENABLE;
```
