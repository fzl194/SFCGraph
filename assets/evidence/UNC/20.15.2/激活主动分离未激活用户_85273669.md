## [操作场景](#ZH-CN_OPI_0185273669)

本操作指导介绍在运行网络中激活主动分离未激活用户特性的操作过程。

如果用户的PDP上下文在一定时期内没有被激活， UNC 就判定该用户为未激活用户，从而主动地将该用户分离。

> **说明**
> 适用于 SGSN 。

## [操作步骤](#ZH-CN_OPI_0185273669)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. 设置主动分离未激活用户特性参数。
    - 设置GSM模式下相关参数。
      [**SET GBDETACH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/分离非活动用户/Gb模式分离非活动用户参数/设置Gb分离非活动用户参数(SET GBDETACH)_72345095.md)
    - 设置UMTS模式下相关参数。
      [**SET IUDETACH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/分离非活动用户/Iu模式分离非活动用户参数/设置Iu分离非活动用户参数(SET IUDETACH)_26305310.md)
4. **可选：**通过特定APN配置分离未激活用户的规则。
  [**ADD APNNICHARACT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/APNNI属性/增加APNNI属性配置信息(ADD APNNICHARACT)_72345265.md)
  > **说明**
  > - 如果 “是否永久保留非活动用户” 参数设置为默认值 “NO” ，则在分离激活用户过程中根据APN设置分离非活动用户定时器时长。对于非永久在线用户，分离时间仅由 “分离非活动用户定时器(min)” 控制。
  >   如果分离非活动用户定时器已根据APN设置，SGSN会发起分离流程，在定时器超时后将用户分离，而不需要考虑 [步骤 5](#ZH-CN_OPI_0185273669__zh-cn_opi_0130429366_cmd884572513180642) 的设置。
  > - 如果“Reserve idle user forever”参数设置为“YES”，SGSN将根据APN在分离过程中永久保留未激活用户。
5. **可选：**通过特定 IMEI 配置分离未激活用户的规则。
  [**ADD IMEIFORFD**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/强制分离业务IMEI配置/增加强制分离策略(ADD IMEIFORFD)_26305298.md)
  > **说明**
  > 当未激活用户被分离时，如果用户携带的IMEI符合配置的IMEI匹配规则，SGSN会保留用户，而不是分离用户。如果不符合，SGSN必须分离未激活用户。

## [任务示例](#ZH-CN_OPI_0185273669)

任务描述

激活主动分离未激活用户特性，将路由区更新过程中的非激活用户分离定时器设置为360分钟。

脚本

开启License文件中需要的功能项。

```
SET LICENSESWITCH: LICITEM="LKV2DTUSR02", SWITCH=ENABLE;
```

在不同制式下，激活主动分离未激活用户特性。

//在GSM制式下，设置分离定时器超过360分时，主动分离未激活用户。

```
SET GBDETACH: DETACHSW=YES, NACTTMR=360, IMPDTCHTMOUT=YES;
```

//在UMTS制式下，设置分离定时器超过360分时，主动分离未激活用户。

```
SET IUDETACH: DETACHSW=YES, NACTTMR=360, IMPDTCH=EXPLICITLY;
```
