# 调测 VoLTE紧急呼叫 （适用于MME）

- [操作场景](#ZH-CN_OPI_0170014694__1.3.1)
- [必备事项](#ZH-CN_OPI_0170014694__1.3.2)

## [操作场景](#ZH-CN_OPI_0170014694)

VoLTE紧急呼叫调测用于验证终端发起的紧急呼叫功能。

## [必备事项](#ZH-CN_OPI_0170014694)

前提条件

数据配置完成，参见 [激活VoLTE紧急呼叫（适用于MME）](激活VoLTE紧急呼叫（适用于MME）_70014693.md) 。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0170014694)

1. 建立用户跟踪。
2. UE使用紧急附着接入调测MME。
3. 查看跟踪信息。
    -
      **预期结果：** E-URTAN向MME发送Attach Request消息(ePS-attach-type-value=ePS-emergency-attach)。MME根据配置参数取值判断，允许无卡终端接入网络，并且根据配置获取紧急APN名称，向SGW-C发起创建缺省承载请求。如 [图1](#ZH-CN_OPI_0170014694__fig_02) 所示。

      **图1** 用户跟踪（样例）

      <br>

      ![](调测VoLTE紧急呼叫（适用于MME）_70014694.assets/zh-cn_image_0170284328_2.png "点击放大")
    - **异常情况：** 跟踪不到eNodeB发起的附着消息，联系eNodeB的技术支持人员。
4. 查询用户承载信息。
  进入 “MML命令行-UNC” 窗口。
  执行 [**DSP SMCTX**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/系统管理/用户数据库管理/显示承载上下文(DSP SMCTX)_72226033.md) : QUERYOPT=BYIMEI, IMEI="333902920958476", IDTYPE=BYBEARID/NSAPI, BEARIDORNSAPI=5;
    - **预期结果：** 可观察到用户紧急呼叫的缺省承载上下文动态信息，指示被测UE发起紧急呼叫业务。
    - **异常情况：** 不存在用户的缺省承载上下文动态信息。根据 [激活VoLTE紧急呼叫（适用于MME）](激活VoLTE紧急呼叫（适用于MME）_70014693.md) 检查数据配置是否正确。
5. UE成功进行VoLTE紧急呼叫业务。
