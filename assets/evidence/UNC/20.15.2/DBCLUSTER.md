#### [命令功能](#ZH-CN_CONCEPT_0129626985)

该命令用于查询CSDB的子集群信息。

#### [注意事项](#ZH-CN_CONCEPT_0129626985)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0129626985)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0129626985)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| CLUSTERID | 子集群标识 | 可选必选说明：可选参数。<br>参数含义：该参数用于指定唯一一个子集群。<br>数据来源：本端规划<br>取值范围：1～10。<br>默认值：无。<br>配置原则：如果不指定子集群标识，则显示所有子集群的信息。 |

#### [使用实例](#ZH-CN_CONCEPT_0129626985)

查询 “子集群标识” 为 “2” 的信息：

DSP DBCLUSTER: CLUSTERID=2;

```
%%DSP DBCLUSTER: CLUSTERID=2;%%
RETCODE = 0  操作成功。

操作结果如下：

```

#### [输出结果说明](#ZH-CN_CONCEPT_0129626985)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 子集群标识 | 唯一标识一个子集群。<br>取值范围：1～10。 |
| VNFC名称 | 子集群使用者的VNFC名称。<br>取值范围：字符串类型，长度为1～31。 |
| DC名称 | 描述子集群所在的数据中心名称，当前暂不支持此功能，此字段显示值为NULL。<br>取值范围：字符串类型，长度为1～127。 |
| 资源单元个数 | 子集群所有的资源单元个数。<br>取值范围：1～1000。 |
| 控制节点状态 | 子集群的控制节点（PRP进程）状态。<br>取值范围：<br>- “正常”：正常运行的PRP进程个数大于1，CSDB可以正常工作，且具备可靠性功能。<br>- “不可靠”：正常运行的PRP进程个数为1，CSDB可以工作，但不具备可靠性功能。<br>- “不可用”：正常运行的PRP进程个数为0 ，CSDB无法正常工作。 |
| 存储节点状态 | 子集群的存储节点（DNP进程）状态。<br>取值范围：<br>- “正常”：本子集群内，一半以上的DNP进程是正常的，CSDB服务可用，且具备可靠性功能。<br>- “不可靠”：本子集群内，1个到DNP进程总数的一半的DNP进程是正常的，CSDB服务可用，但不具备可靠性功能。<br>- “不可用”：本子集群内，所有DNP进程都是异常的，CSDB服务不可用。 |
| 内存使用率(%) | 子集群存储节点平均内存占用率，单位百分比。<br>取值范围：0～100。 |
| CPU占用率(%) | 子集群存储节点平均CPU占用率，单位百分比。<br>取值范围：0～100。 |
| 资源单元亲和性 | 资源单元亲和性。<br>取值范围：<br>- “强反亲和性”：本子集群内，资源单元分布在不同的物理机上。<br>- “弱反亲和性”：本子集群内，最多两个资源单元可以分布在相同的物理机上。<br>- “弱三反亲和性”：本子集群内，最多三个资源单元可以分布在相同的物理机上。 |
| 副本模式 | 子集群的副本模式。<br>取值范围：<br>- “数据库级单副本”：本子集群内，CSDB服务端的所有记录只有主本，没有备本。<br>- “数据库级双副本”：本子集群内，CSDB服务端的所有记录都有主本和备本，并且主本、备本在不同的物理机上。<br>- “单双副本自适应”：本子集群内，CSDB服务端的业务表记录都有主本，根据记录在插件服务端的位置关系确定是否生成备本，主备本在不同的物理机上。 |
| 持久化标识 | 子集群是否持久化。<br>取值范围：<br>- “开启持久化”。<br>- “关闭持久化”。 |
| 持久化状态 | 子集群的持久化状态。<br>取值范围：<br>- “已经开启持久化”：子集群的进程持久化状态都是“已经开启持久化”。<br>- “已经关闭持久化”：子集群的进程持久化状态都是“已经关闭持久化”。<br>- “正在开启持久化”：子集群的部分进程的持久化状态是“正在开启持久化”。<br>- “正在关闭持久化”：子集群的部分进程的持久化状态是“正在关闭持久化”。<br>- “其他”：其他情况。 |
| 容灾子集群标识 | 容灾场景下唯一标识一个子集群。<br>取值范围：1~127 |
