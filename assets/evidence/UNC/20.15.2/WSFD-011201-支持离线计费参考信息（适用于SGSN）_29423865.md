#### [软参](#ZH-CN_TOPIC_0229423865)

本特性相关的软参如下：

- BYTE129 BIT1 控制是否在话单中MSISDN前带“91”字节
- BYTE129 BIT4 控制生成话单时如果MM信息中的NetworkCap为空时是否填充为E5C0
- BYTE129 BIT5 控制生成话单时如果MM信息中的NetworkCap为空时是否填充为0000
- BYTE129 BIT7 控制2G用户R4话单是否携带System Type字段
- BYTE130 控制非AIS SCDR中APN的编码为DOT形式还是LV形式
- BYTE131 控制AIS SCDR中APN的编码为DOT形式还是LV形式
- BYTE132 控制发生SGSN内系统间切换时是否生成M-CDR和S-CDR
- BYTE133 BIT2 控制是否进行MCDR duration的四舍五入处理
- BYTE133 BIT3 控制是否进行LCS-MT-CDR duration的四舍五入处理
- BYTE133 BIT4 控制在没有数据流量产生情况下是否生成S-CDR话单
- BYTE134 BIT1 控制UNC是否填写第一个流量节点的qosRequested字段
- BYTE134 BIT2 控制UNC是否填写第一个流量节点的qosNegotiated字段
- BYTE134 BIT3 控制用户QoS变更时UNC是否填写流量节点的QoSRequested字段
- BYTE134 BIT4 控制用户QoS变更UNC是否填写流量节点的qosNegotiated字段
- BYTE134 BIT5 控制非MS发起QoS变更时，是否填写流量节点的qosNegotiated字段
- BYTE134 BIT6 控制UNC清除SDB时，是否要清除计费控制块
- BYTE135 控制R4和R99话单消息中Data Record Format Version的取值
- BYTE136 控制UNC是否每间隔1小时打印创建SCDR失败的计数器
- BYTE138 BIT1 控制当系统忙时UNC生成话单的定时器时长是取固定值还是取随机值
- BYTE138 BIT2 控制当系统忙时UNC生成话单的定时器时长的取值区间
- BYTE138 BIT3 控制话单时长是否按周期进行拆分
- BYTE138 BIT4 控制费率变更点后第一张话单的周期是否取随机值且不按照配置周期时长拆分话单
- BYTE138 BIT5 控制话单生成周期定时器时长的取值区间
- BYTE138 BIT6 控制UNC是否启用SCDR夏令时标志校验功能
- BYTE138 BIT7 控制CG重选优化功能是否开启
- BYTE139 控制是否将S-CDR话单中APN字符串进行归一化处理
- BYTE141 BIT1-BIT2 控制SGSN每次发送话单数量
- BYTE141 BIT3 控制2G话单中是否仅填写GSM QoS（R98QoS）
- BYTE141 BIT4 控制2G/3G话单中是否仅填写GSM QoS（R98QoS）
- BYTE141 BIT5 控制是否产生R98格式的3G S-CDR话单
- BYTE141 BIT6 控制话单Duration字段填充
- BYTE141 BIT8 控制E_CDRF_TRANS_CMD_SEND_DATA消息携带Packet Transfer Command信元
- BYTE142 BIT3 控制R4和R6协议版本的S-CDR话单中IMSI和MSISDN的BCD码字节序
- BYTE142 BIT4 控制R4和R6协议版本的S-CDR话单中IMEI的BCD码字节序
- BYTE142 BIT5 控制S-CDR话单的RecordSequenseNumber字段填写方式
- BYTE142 BIT7 控制SCDR话单生成异常时，是否将计费CB表相关信息写入系统日志
- BYTE142 BIT8 控制CDP进程接收GTP发送的SCDR话单时，是否核查接收的话单重复性
- BYTE143 BIT1 控制R4话单中的ChargingID的ASN.1编码格式
- BYTE143 BIT2 控制Echo Request的编号方案
- BYTE143 BIT3 控制是否在话单中只使用IMEI
- BYTE143 BIT4 控制是否生成精确流量话单
- BYTE143 BIT5 控制第二张话单及之后的话单中填写的时长是否为0
- BYTE143 BIT6 设置Ga接口发送话单时，是否判断CG话单发送循环定时器消息的时间间隔
- BYTE144 BIT1 CDP进程控制话单压缩/解压缩功能
- BYTE144 BIT2 半成品话单发送前，控制话单字段核查并打印错误字段
- BYTE144 BIT3 CDP进程控制话单写硬盘速率
- BYTE144 BIT4 控制SCDR中SGSN地址填用户面地址还是信令面地址
- BYTE144 BIT5 在SDB表中因异常导致保存的APN长度为0时，控制SCDR中是否填写一个默认的APN
- BYTE144 BIT6 控制R6/R7话单中的APNNI及APNOI的格式及话单是否携带相关信元
- BYTE144 BIT7 控制在话单时间超长时，是否在生成Normal Release话单时拆分话单
- BYTE144 BIT8 控制是否调整费率变更时间戳的秒数
- DWORD33 BIT3 控制读取硬盘话单的策略
- DWORD33 BIT4 控制是否自动纠正错误话单的字段内容
- DWORD33 BIT13 用于控制S-CDR和M-CDR中话单中字段携带方式
- DWORD33 BIT14 控制当字段LAC为0值或FFFE值创建话单时是否允许激活PDP
- DWORD33 BIT15 控制当字段RAC/LAC/CELLID任意一个为全F创建话单时是否允许激活PDP
- DWORD33 BIT16 控制当字段RAC/LAC/CELLID同时为全F值创建话单时是否允许激活PDP
- DWORD33 BIT17 控制当字段networkCapability为全F值创建话单时是否允许激活PDP
- DWORD33 BIT18 控制当字段PDPAddress为非法值创建SCDR话单时是否允许激活PDP
- DWORD33 BIT19 控制是否对Ga接口话单消息进行编码后校验
- DWORD33 BIT20 控制是否在MCDR中记录的NodeID字段中添加CDP进程的模块号
- DWORD33 BIT21 控制是否在SMSCDR中记录的NodeID字段中添加CDP进程的模块号
- DWORD33 BIT22 控制是否在LCSCDR中记录的NodeID字段中添加CDP进程的模块号
- DWORD33 BIT23 控制当字段MSISDN为全F值创建SCDR话单时是否允许激活PDP
- DWORD33 BIT26 控制未配置CDPIP的CDP进程是否向UPP进程发起限呼
- DWORD33 BIT27 控制在费率变更时刻，是否将第二张话单的流量填写到第一张S-CDR话单中
- DWORD33 BIT28 控制是否启动增强型的错误话单纠正功能
- DWORD33 BIT29 控制是否启用话单池资源定时核查功能
- DWORD33 BIT31 控制SGSN在创建话单判断出PLMN字段值无效时如何填写PLMN
- DWORD33 BIT32 控制S-CDR话单中RNC Unsent Downlink Volume字段流量值超过1.9G时上报策略
- DWORD34 BIT1 本软参与BIT2联合控制费率变更处理中添加流量节点的定时器的长度
- DWORD34 BIT2 本软参与BIT1联合控制费率变更处理中添加流量节点的定时器的长度
- DWORD34 BIT3 本软参用于控制话单写硬盘时的话单池占有率的判断阈值
- DWORD34 BIT4 控制是否在SCDR中记录的NodeID字段中添加CDP进程的模块号
- DWORD34 BIT9 控制是否启用1分钟的SCDR失败等相关信息日志打印
- DWORD34 BIT10 控制是否直接将HLR签约数据中CC的值写入SCDR中的CC字段
- DWORD34 BIT11 控制新侧话单中CellID、PLMN ID、LAC、RA的填写方式

- DWORD34 BIT18 控制CauseForRecClosing是intraSGSNIntersystemChange的S-CDR话单的发送顺序
