- [业务流程](#ZH-CN_TOPIC_0192480008__1.3.1.1)

#### [业务流程](#ZH-CN_TOPIC_0192480008)

下面以3G到2G Inter-SGSN系统间切换为例，简要介绍切换的过程，如 [图1](#ZH-CN_TOPIC_0192480008__zh-cn_concept_0130427921_fig3) 所示。

> **说明**
> 详细的切换流程参见3GPP TS 23060 6.13.2。

**图1** 3G到2G系统间切换（Inter-SGSN）流程图

<br>

![](实现原理_92480008.assets/zh-cn_image_0192519413_2.png "点击放大")

1. 无线侧根据一定的算法判断并发起切换。
2. MS检测到新的无线区域的路由区发生改变，首先向New 2G-SGSN发起Routing Area Update Request。
3. New 2G-SGSN根据RAU Request中携带的old RAI到DNS上进行解析，获得old RAI所在的SGSN地址。New 2G-SGSN向该地址（Old 3G-SGSN）发送SGSN Context Request。
4. Old 3G-SGSN向RNC发送SRNS Context Request消息；RNC收到此消息后，开始缓存并且停止向MS发送数据PDU，并向Old 3G-SGSN返回SRNS Context Response。
5. Old 3G-SGSN将RNC传送过来的Uplink/Downlink PDU SN连同用户的MM Context和PDP Context一起通过SGSN Context Response发送给New 2G-SGSN；
6. 此时用户开始接入New 2G-SGSN，首先执行鉴权加密等安全流程，此安全流程与用户附着到2G网络的安全流程相同，在3GPP TS 23060 6.8章节中有详细描述。
7. New 2G-SGSN向Old 3G-SGSN发送SGSN Context Acknowledge消息，通知Old 3G-SGSN已经准备好接收属于该PDP上下文的数据包。
8. Old 3G-SGSN向RNC请求未发送给MS的数据包，RNC会将未发送给MS的数据包送给Old 3G-SGSN。
9. Old 3G-SGSN向New 2G-SGSN发送该用户的PDU。
10. 由于此时MS已经在New 2G-SGSN上完成注册，上下文已经迁移到New 2G-SGSN上，因此New 2G-SGSN要通过Update PDP Context流程将SGSN已经发生改变通知到GGSN。

与用户直接在New 2G-SGSN注册一样，SGSN向HLR发送Update GPRS Location请求，触发HLR向Old 3G-SGSN发送Cancel Location，Old 3G-SGSN收到Cancel Location后，向RNC发送Iu Release Command，释放RNC侧的资源。

最后New 2G-SGSN发送Routing Area Update Accept接受手机的路由区更新，手机响应Routing Area Update Complete后，切换完成。

> **说明**
> 在用户发生3G/2G切换时，仅仅是SGSN发生了变化，GGSN并没有发生变化。例如，当用户从3G区域切换到2G区域，2G的SGSN从3G-SGSN取到用户的上下文后，将向3G的GGSN发起Update PDP Context Request。

切换过程中，手机的数传过程有可能中断数秒，切换完成后，数传恢复。

对于2G到3G切换，流程也大致相同，在此不再赘述，请参考TS 23060 6.13.2.2。
