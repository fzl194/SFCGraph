## [操作场景](#ZH-CN_OPI_0230805098)

当网络中有PCRF，用户激活为PCC用户，需要为用户或用户业务提供灵活策略控制时，需要开启动态PCC功能，并根据业务情况来部署预定义规则/预定义规则组或动态规则。

- 预定义规则/预定义规则组
  当需要对报文进行七层解析或业务规划时针对用户的规则内容在用户激活前可以预先确定，建议采用预定义规则方式，以减少GGSN/PGW-C与PCRF之间的交互和对缓存资源的占用。即相应的QoS管理、计费控制策略等在GGSN/PGW-C上通过rule或者UserProfile配置，PCRF上只配置预定义规则名称/预定义规则组名称。
- 动态规则
  当用户激活前用户的规则内容无法确定，需要根据用户访问业务时的协商结果由AF（应用服务器）或其他网元发送请求给PCRF，请求建立动态规则时，建议采用动态规则方式。即GGSN/PGW-C上只需开启PCC功能开关，相应的QoS管理、计费控制策略等均在PCRF上配置。

GGSN/PGW-C可以基于APN、特定号段、全局选择PCRF组，并触发PCC功能，GGSN/PGW-C上基于不同粒度开启PCC功能的规划如 [表1](#ZH-CN_OPI_0230805098__tab1) 所示。

| --- | --- | --- | --- |
| 选择APN下绑定的PCRF组 | ENABLE | ENABLE | 指定APN下绑定PCRF组。 |
| 选择指定号段绑定的PCRF组 | ENABLE | INHERIT | 指定号段绑定PCRF组。 |
| 选择全局缺省的PCRF组 | ENABLE | INHERIT | 指定全局缺省的PCRF组，且指定号段未绑定PCRF组。 |

> **说明**
> 适用于 GGSN、 PGW-C。

## [操作流程](#ZH-CN_OPI_0230805098)

采用不同规则方式时需要执行的操作步骤如下：

- PCRF下动态规则时，需要执行[步骤 2](#ZH-CN_OPI_0230805098__step1)～[步骤 8](#ZH-CN_OPI_0230805098__step7)。
- PCRF下预定义规则时，需要执行[步骤 2](#ZH-CN_OPI_0230805098__step1)～[步骤 10](#ZH-CN_OPI_0230805098__step92671412440)。
- PCRF下预定义规则组时，需要执行[步骤 2](#ZH-CN_OPI_0230805098__step1)～[步骤 11](#ZH-CN_OPI_0230805098__step9)。

配置动态PCC功能的流程如下：

1. （可选）配置IMSI或MSISDN号码段。
2. 配置PCRF分组信息。
3. 配置全局缺省的PCC开关。
4. 配置APN的PCC开关，并绑定PCRF group到APN下。
5. （可选）配置本地规则。
6. （可选）配置业务策略组合。

## [操作步骤](#ZH-CN_OPI_0230805098)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. **可选：**配置IMSI或MSISDN号码段。
  [**ADD IMSIMSISDNSEG**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务公共/IMSI MSISDN号段/增加IMSI和MSISDN号段（ADD IMSIMSISDNSEG）_09897128.md)
  > **说明**
  > 规划特定号段的用户触发PCC功能时需要配置。
4. 配置PCRF分组信息。
    a. 配置PCRF。
      [**ADD PCRF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF/增加PCRF（ADD PCRF）_09897101.md)
    b. 配置PCRF分组。
      [**ADD PCRFGROUP**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF组/增加PCRF组（ADD PCRFGROUP）_09897090.md)
    c. 添加指定的PCRF到指定的PCRF分组中。
      [**ADD PCRFBINDGRP**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF绑定/增加PCRF与PCRF Group的关联关系（ADD PCRFBINDGRP）_09897096.md)
    d. **可选：**如果PCRF组的工作模式为主备模式，可使用如下命令修改PCRF分组内的缺省主用PCRF。
      [**SET MASTERPCRF**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF组/设置主用PCRF（SET MASTERPCRF）_09897094.md)
    e. **可选：**基于IMSI号段选择PCRF分组时，需要将指定的PCRF分组绑定到指定的号码段。

## [任务示例](#ZH-CN_OPI_0230805098)

任务描述

GGSN/PGW-C上规划全局使能PCC功能，APN“apn-test1”下使能PCC功能，APN“apn-test2”下继承全局使能开关。相应号段的用户通过指定APN接入时是否激活为PCC用户及GGSN/PGW-C选择哪个PCRF分组进行交互，如 [表2](#ZH-CN_OPI_0230805098__table_1) 所示。

*表2 用户选择的PCRF分组与配置的关系*

| 用户号段 | 接入的APN名称 | APN下的PCC开关 | 选择的PCRF分组 | 是否激活为PCC用户 |
| --- | --- | --- | --- | --- |
| 不区分号段 | apn-test1 | ENABLE | pcrf_group_2 | 是 |
| 18900000000～18949999999 | apn-test2 | INHERIT | pcrf_group_1（MSISDN号段绑定的PCRF分组） | 是 |
| 18950000000～18999999999 | apn-test2 | INHERIT | pcrf_group_2（MSISDN号段绑定的PCRF分组） | 是 |
| 其他号段 | apn-test2 | INHERIT | pcrf_group_1（全局缺省PCRF分组） | 是 |

脚本

1. 打开本特性的License配置开关。
  ```
  SET LICENSESWITCH:LICITEM="LKV3W9SPCC11",SWITCH=ENABLE;
  ```
2. 配置MSISDN号段。
  ```
  ADD IMSIMSISDNSEG:SEGMENTNAME="msisdnseg1",SEGMENTTYPE=MSISDN,SEGSTART="18900000000",SEGEND="18949999999";
  ```
  ```
  ADD IMSIMSISDNSEG:SEGMENTNAME="msisdnseg2",SEGMENTTYPE=MSISDN,SEGSTART="18950000000",SEGEND="18999999999";
  ```
3. 配置PCRF分组信息，设置pcrf_group_1组内的PCRF按配置比例进行负荷分担，pcrf_group_2组内的PCRF之间为主备备份。
  ```
  ADD PCRF:HOSTNAME="pcrf_1",REALMNAME="pcrf.huawei.com";
  ```
  ```
  ADD PCRF:HOSTNAME="pcrf_2",REALMNAME="pcrf.huawei.com";
  ```
  ```
  ADD PCRF:HOSTNAME="pcrf_3",REALMNAME="pcrf.huawei.com";
  ```
  ```
  ADD PCRF:HOSTNAME="pcrf_4",REALMNAME="pcrf.huawei.com";
  ```
  ```
  ADD PCRFGROUP:PCRFGRPNAME="pcrf_group_1",LOADBALANCEMODE=LOAD_BALANCE;
  ```
  ```
  ADD PCRFBINDGRP:PCRFGRPNAME="pcrf_group_1",PCRFHOSTNAME="pcrf_1",WEIGHT=40;
  ```
  ```
  ADD PCRFBINDGRP:PCRFGRPNAME="pcrf_group_1",PCRFHOSTNAME="pcrf_2",WEIGHT=60;
  ```
  ```
  ADD PCRFGROUP:PCRFGRPNAME="pcrf_group_2",LOADBALANCEMODE=MASTER_SLAVE;
  ```
  ```
  ADD PCRFBINDGRP:PCRFGRPNAME="pcrf_group_2",PCRFHOSTNAME="pcrf_3";
  ```
  ```
  ADD PCRFBINDGRP:PCRFGRPNAME="pcrf_group_2",PCRFHOSTNAME="pcrf_4";
  ```
4. 将指定的PCRF组和号码段绑定，并设置PCRF组“pcrf_group_1”为全局缺省的PCRF组。
  ```
  ADD GLBPCRFGROUP:IMSIMSISDNSEG="msisdnseg1",PCRFGRPNAME="pcrf_group_1",PRIORITY=1;
  ```
  ```
  ADD GLBPCRFGROUP:IMSIMSISDNSEG="msisdnseg2",PCRFGRPNAME="pcrf_group_2",PRIORITY=2;
  ```
  ```
  SET DFTGLBPCRFGRP:PCRFGRPNAME="pcrf_group_1";
  ```
5. 配置全局缺省的PCC开关。
  ```
  SET PCCFUNC:HOMEPCCSWITCH=ENABLE,ROAMPCCSWITCH=ENABLE,VISITPCCSWITCH=ENABLE,COMPOLICY=LOCALAAA;
  ```
6. 配置2个APN下的PCC开关，其中APN apn-test1的PCC开关为 “ENABLE” ，并绑定PCRF组 “pcrf_group_2” ，APN apn-test2的PCC开关为 “INHERIT” ，当通过该APN接入的用户的特定号段未与PCRF组绑定时，使用全局缺省的PCRF组“pcrf_group_1”。
  ```
  ADD PCRFGRPBNDAPN:APN="apn-test1",DEFAULTFLAG=DEFAULT,PCRFGRPNAME="pcrf_group_2";
  ```
  ```
  SET APNPCCFUNC:APN="apn-test1",HOMEPCCSWITCH=ENABLE,ROAMPCCSWITCH=ENABLE,VISITPCCSWITCH=ENABLE;
  ```
  ```
  SET APNPCCFUNC:APN="apn-test2",HOMEPCCSWITCH=INHERIT,ROAMPCCSWITCH=INHERIT,VISITPCCSWITCH=INHERIT;
  ```
