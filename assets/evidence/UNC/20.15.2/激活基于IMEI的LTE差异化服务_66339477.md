## [操作场景](#ZH-CN_OPI_0166339477)

本操作指导介绍在运行网络中激活基于IMEI的LTE差异化服务特性的操作过程。

基于IMEI的LTE差异化服务功能是指MME在获取终端IMEISV后通过S1接口将IMEISV以“Masked IMEISV”信元形式发送给eNodeB，便于eNodeB制定基于终端的类型、软件版本的控制策略，从而实现对不同终端的差异化控制。

> **说明**
> 适用于MME。

## [操作步骤](#ZH-CN_OPI_0166339477)

1. 进入 “MML命令行-UNC” 窗口。
2. 开启MME获取终端IMEISV功能。
    a. 查询用户获取IMEISV策略是否配置。
      [**LST S1IMEICFG**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/设备检查管理/S1模式IMEI配置/查询S1模式IMEI配置(LST S1IMEICFG)_72225319.md)
    b. **可选：**配置所需用户的获取IMEISV策略。
      [**ADD S1IMEICFG**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/设备检查管理/S1模式IMEI配置/增加S1模式IMEI配置(ADD S1IMEICFG)_26305448.md)
      > **说明**
      > - 如果[步骤 2.a](#ZH-CN_OPI_0166339477__a)查询结果中，所需号段对应的查询结果为空，则需要使用[**ADD S1IMEICFG**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/设备检查管理/S1模式IMEI配置/增加S1模式IMEI配置(ADD S1IMEICFG)_26305448.md)命令配置所需用户的获取IMEISV策略；如果[步骤 2.a](#ZH-CN_OPI_0166339477__a)查询结果中，所需号段对应的查询结果有内容，但不是获取IMEISV，则使用[**MOD S1IMEICFG**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/设备检查管理/S1模式IMEI配置/修改S1模式IMEI配置(MOD S1IMEICFG)_26145638.md)命令配置所需用户的获取IMEISV策略；如果[步骤 2.a](#ZH-CN_OPI_0166339477__a)查询结果中，所需号段对应的查询结果为获取IMEISV，则此步骤不需要执行。
      > - 用户可以根据参数选择获取用户的IMEISV的范围，推荐选用获取所有用户的IMEISV。
3. 设置IMEISV的eNodeB发送策略。
  [**SET S1CMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1接口兼容性/设置S1接口兼容性(SET S1CMPT)_72345837.md)

## [任务示例](#ZH-CN_OPI_0166339477)

任务描述

现网部署了支持IMEISV处理能力的eNodeB设备。无线LTE网络希望MME配合，通过这些eNodeB实现在调度，状态控制，功率分配等方面对所有的不同UE做差异化控制。

脚本

//开启MME获取终端IMEISV功能。

```
MOD S1IMEICFG: SUBRANGE=ALL_USER, GETIMEIPLC=IMEISV;
```

//MME将IMEISV发送给eNodeB。

```
SET S1CMPT: MSKIMEISVSCOPE=ALLENODEB;
```
