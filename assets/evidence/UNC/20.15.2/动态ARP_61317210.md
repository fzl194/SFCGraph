# 动态ARP

设备通过ARP报文动态学习、动态刷新IP地址和MAC地址之间的映射关系，整个过程包括ARP表项的创建、更新和老化，不需要网络管理员手工维护，大大减少了网络管理员的维护量。

动态ARP适用于拓扑结构复杂、带宽资源紧张、通信实时性要求高的网络。

- 动态ARP表项的创建和更新
  一般实现中，如果设备收到的ARP报文满足以下条件，系统将自动创建或更新ARP表项：
    - ARP报文的源IP地址与入接口IP地址在同一网段，目的IP地址是本接口IP地址。
- 动态ARP表项的老化
  在某条动态ARP表项老化时间到达之后，设备会向相应的设备发送ARP探测报文，如果在指定的探测次数内收不到对端对该探测报文的应答，就将该ARP表项老化掉。

在一段时间内，如果ARP表中的某一映射关系始终没有得到使用，这条映射关系就会被删除，动态ARP老化机制就是实现系统自动删除ARP表项操作的一套机制。通过动态ARP的老化机制，可以及时删除使用频率不高的ARP表项，节省ARP表的存储空间，进而加快设备发送数据报文之前查询ARP表的速度。

动态ARP老化机制相关概念如 [表1](#ZH-CN_CONCEPT_0161317210__ta2c0e6b33d01439d8da1f13641a61cc1) 所示。

*表1 动态ARP老化机制相关概念*

| 概念 | 说明 | 场景 |
| --- | --- | --- |
| 老化探测模式 | 在动态ARP表项老化之前，设备会对外发送ARP老化探测报文。老化探测报文可以是单播报文，也可以是广播报文。缺省情况下，设备以广播模式发送ARP老化探测报文。 | - 当对端设备的IP地址不变化而MAC地址频繁更新时，建议使用广播模式发送ARP老化探测报文。<br>- 当对端设备MAC地址不变，但是当前网络带宽资源特别紧缺，且ARP表项的老化超时时间设置较小时，建议使用单播模式发送ARP老化探测报文。 |
| 老化时间 | 动态ARP表项并非永远有效，每一条记录都有一个生存周期，到达生存周期仍得不到刷新的记录将被从ARP表中删除，这个生存周期被称作老化时间。 | 两台相互通信的设备，通过ARP协议可以学习到对方的IP地址和MAC地址之间的映射关系，并且保存在自己的ARP表中，之后两者就直接利用ARP表中的ARP表项进行通信。但是，当对端设备发生故障或者更换了网卡，本端设备没有得到任何关于对端状态变化的通告，于是本端设备仍会继续将数据报文发送给对端设备。这样就会造成网络流量的中断，导致中断的原因是本端设备的ARP表的信息没有得到及时更新。为了减少流量中断现象的发生，ARP表中的表项一般都会设定一个定时器，当到达定时器的某个动态ARP表项的老化时间后，设备会自动删除掉这个表项。 |
| 老化探测次数 | 在一个动态ARP表项老化之前，本端设备会先向对端设备发送ARP探测报文进行探测，如果超过设置的探测次数后仍没有应答来应答此ARP探测报文，该动态ARP表项将被删除。 | 设备通过设置ARP表项老化定时器，可以有效改善由于ARP表项的信息没有得到及时更新而造成流量中断的现象，但是这种现象并没有彻底消除，其原因在于时延。如果定时器的动态ARP表项的老化超时时间是N秒，本端设备只有等到N秒后才能检测到对端设备的状态发生了变化，而在此期间本端设备的ARP表的信息还是没有得到及时的更新。通过设置ARP表项的老化探测次数，本端设备可以更加及时感知对端设备状态的变化，维护自己的ARP表项信息。 |
