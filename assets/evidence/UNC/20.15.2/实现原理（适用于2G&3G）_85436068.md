# 实现原理（适用于2G&3G）

**用户数据管理流程**

SGSN与HSS间的用户数据管理流程请参考 [WSFD-010400 用户数据管理特性概述（适用于4G）](../WSFD-010400 用户数据管理（4G）/特性概述_70018151.md) 特性中的 [用户数据管理流程](../WSFD-010400 用户数据管理（4G）/实现原理（适用于4G）_70018152.md#ZH-CN_TOPIC_0170018152__p10632311459) ，SGSN与HLR间的用户数据管理功能相关的流程如下：

- Insert Subscriber Data流程
  **图1** Insert Subscriber Data流程

  <br>

  ![](实现原理（适用于2G&3G）_85436068.assets/zh-cn_image_0185454518_2.png)

  <br>
  如 [图1](#ZH-CN_CONCEPT_0185436068__zh-cn_concept_0130427302_fig4) 所示，SGSN和HLR之间的Insert Subscriber Data流程主要用于插入新增的用户签约数据，该流程由HLR发起。
- Purge MS流程
  **图2** Purge MS流程

  <br>

  ![](实现原理（适用于2G&3G）_85436068.assets/zh-cn_image_0185454519_2.png)

  <br>
  如 [图2](#ZH-CN_CONCEPT_0185436068__zh-cn_concept_0130427302_fig1) 所示，SGSN和HLR之间的Purge MS流程主要用于SGSN删除了用户的信息后通知HLR，该流程由SGSN发起，删除用户和信息的原因可能为UE长时间非活跃态导致的自动删除等。
- Cancel Location流程
  **图3** Cancel Location流程

  <br>

  ![](实现原理（适用于2G&3G）_85436068.assets/zh-cn_image_0185454520_2.png)

  <br>
  如 [图3](#ZH-CN_CONCEPT_0185436068__zh-cn_concept_0130427302_fig2) 所示，HLR和SGSN之间Gr接口上的Cancel Location流程主要用于删除SGSN上的用户记录，该流程由HLR触发，通过该流程通知SGSN取消用户的签约信息，或者通知SGSN进行某些流程，如SGSN改变等。
- Delete Subscriber Data流程
  **图4** Delete Subscriber Data流程

  <br>

  ![](实现原理（适用于2G&3G）_85436068.assets/zh-cn_image_0185454561_2.png)
  如 [图4](#ZH-CN_CONCEPT_0185436068__zh-cn_concept_0130427302_fig6) 所示，SGSN和HLR之间的Delete Subscriber Data流程主要用于删除部分或全部HLR存储在SGSN的数据，该流程由HLR触发，通过该流程HLR通知SGSN删除PDP上下文。
- Authentication流程，请参考 [WSFD-010301 鉴权功能（适用于SGSN）](../../WSFD-010301 鉴权功能/鉴权功能（2G&3G）_82934115.md) 。
