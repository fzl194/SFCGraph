## [操作场景](#ZH-CN_OPI_0000001180495435)

某些小区出现拥塞且不扩容，运营商规划基于小区负荷状态对某些用户或某些业务进行策略控制，从而解除/减缓小区拥塞状态时，需要激活基于小区负荷上报的无线资源优化功能。

## [对系统的影响](#ZH-CN_OPI_0000001180495435)

RAN侧和核心网之间传输携带GTP-U扩展头的IP报文，可能会影响网络中MTU值。

## [任务示例](#ZH-CN_OPI_0000001180495435)

任务描述

通过APN“apn1”接入的小区，存在小区拥塞。需要部署基于小区负荷上报的无线资源优化功能，通过限制低优先级用户/特定业务，来解除/减缓小区拥塞。

脚本

//打开本特性的License配置开关。

```
SET LICENSESWITCH:LICITEM="LKV3W9WOCR11",SWITCH=ENABLE;
```

//使能APN“apn1”支持基于小区负荷上报的无线资源优化功能。

```
SET APNREPORTATTR: APN="apn1", CONGESTIONRPT=ENABLE;
```

## [附加信息](#ZH-CN_OPI_0000001180495435)

相关任务

该特性为解决方案特性，还需要在RAN侧/PGW-U/PCRF上进行相应配置：

- PGW-U上需要使能基于小区负荷上报的无线资源优化功能。
- PCRF上规划策略时需要增加下发小区负荷状态变化上报trigger事件的配置，并规划基于小区负荷状态的策略。
- RAN侧需要打开小区拥塞状态上报开关等。

相关概念

无。
