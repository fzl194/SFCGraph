## [操作场景](#ZH-CN_OPI_0275415670)

随着物联网的广泛应用，NB-IoT终端被广泛应用在多种行业中，例如水、电表的上报、物流、智能停车等。目前运营商对于NB-IoT终端业务的接入控制较为薄弱，当大量的NB-IoT终端在很短的时间内同时发起业务（例如指定日期的水表上报），可能会造成系统过载并影响普通终端用户的业务接入。MME通过APN识别NB-IoT终端，并发送Back-off time信元来减缓NB-IoT终端业务的接入速度，确保普通终端业务的正常进行，预防系统进入拥塞状态。

> **说明**
> 适用于MME。

## [操作步骤](#ZH-CN_OPI_0275415670)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本功能的License配置开关。

    a. 增加APNNI组。

    c. 设置低优先级APN流控参数。
      [**SET LOWPRIAPNFC**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/低优先级APN流控管理/设置低优先级APN流控参数(SET LOWPRIAPNFC)_72345757.md)
4. 设置MME支持基于P-GW Back-off timer的APN级流控。
    a. 增加P-GW组。

## [任务示例](#ZH-CN_OPI_0275415670)

任务描述

网络存在NBIOT终端业务与普通终端业务共用一套核心网设备，最当出现大量的NBIOT业务情况时，可能会影响在网普通终端用户的业务，此时开启基于延迟定时器的信令拥塞控制，减缓NBIOT终端业务的接入速率，预防系统进入拥塞状态。

脚本

// 打开本特性的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2LTCC01", SWITCH=ENABLE;
```

//设置低优先级APN流控。

```
ADD APNNIGROUP: GRPID=2, DESC="M2M APNNI Group";
ADD APNNI: GRPID=2, APNNI="HUAWEI.COM"; 
SET LOWPRIAPNFC: FCSWITCH=ON, GRPID=2, MINTHD=10, MAXTHD=300;
```

//配置基于PGW-C的APN流控功能。

PGW-C脚本执行：

```
SET BACKOFFTIME:APNNAME="apn-op",OVERLOADSW=ENABLE,BACKOFFTIMER=600;
```

MME脚本执行：

```
ADD PGWGROUP: GRPID=2, DESC="M2M P-GW Group";
ADD PGWNODE: GRPID=2, SUFFIX="MNC000.MCC123.3GPPNETWORK.ORG"; 
SET PGWBACKOFF: FCSWITCH=ON, APNNIGRPID=2, PGWGRPID=2; 
```
