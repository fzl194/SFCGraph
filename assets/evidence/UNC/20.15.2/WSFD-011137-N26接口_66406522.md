#### [适用NF](#ZH-CN_CONCEPT_0166406522)

MME、AMF

#### [定义](#ZH-CN_CONCEPT_0166406522)

N26接口是AMF与MME之间的接口，提供AMF与MME之间的信令消息隧道，用于协助完成5G网络和LTE网络之间的互操作。

#### [客户价值](#ZH-CN_CONCEPT_0166406522)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 在现有LTE网络的基础上，可以逐步引入5G网络满足人们快速增长的无线数据业务需求，实现从LTE网络到5G网络的平滑演进。<br>- 保护LTE网络的已有投资，充分发挥4、5G网络的性能，提供更丰富的业务体验。 |
| 用户 | 保障在LTE和5G网络之间业务切换的连续性，提升用户的网络体验。 |

#### [应用限制](#ZH-CN_CONCEPT_0166406522)

- 该特性不支持无PDN的EPS与5GS互操作。
- 该特性不支持NB-IoT用户的EPS与5GS互操作。
- 该特性暂不支持Non-IP PDN， Ethernet /Unstructured PDU Session Type的EPS与5GS互操作。
- 该特性不支持通过Non-3gpp接入5G网络的5G用户向LTE网络进行重选或切换。

#### [原理概述](#ZH-CN_CONCEPT_0166406522)

**N26接口协议栈**

N26接口协议栈如 [图2](#ZH-CN_CONCEPT_0166406522__fig25254624118) 所示。

**图2** N26接口协议栈

![](WSFD-011137 N26接口_66406522.assets/zh-cn_image_0166444813_2.png)

- 系统支持AMF的N26接口和MME的S10接口IP地址合一，该配置场景下，如果已经完成MME的S10接口配置，则无需重复配置AMF的N26接口数据。
- AMF的N26接口和MME的S10接口地址独立场景下，需要分别完成AMF的N26接口和MME的S10接口的数据配置。

> **说明**
> GTP-C版本为GTPv2。
>
> AMF的N26接口部署方式：
>
> - N26接口独立部署：AMF的N26接口和MME的S10接口使用不同的IP地址。N26接口的IP地址使用ADD GTPCLEGRPMEM配置，S10接口的IP地址使用ADD GTPCLE配置。
> - N26接口融合部署：AMF的N26接口和MME的S10接口支持使用同一IP地址。AMF的N26接口和MME的S10接口的IP地址都使用ADD GTPCLE配置。

**N26接口交互消息**

N26接口交互消息如 [表1](#ZH-CN_CONCEPT_0166406522__table102313416814) 所示。

*表1 N26接口交互消息*

| 消息类型 | 消息 | 方向 | 作用 |
| --- | --- | --- | --- |
| 附着消息 | Identification Response | AMF->MME | 附着流程中，AMF将通过N26接口发送此消息给目标MME以回复之前发送过来的Identification Request消息。 |
| 重选消息 | Context Request | AMF<->MME | 目标侧的MME在EPS和5GS重选流程中通过N26接口向源侧AMF发送该消息，用于获取UE的MM和EPS承载上下文。<br>注：上下文请求消息也可由目标侧AMF在EPS和5GS重选流程中通过N26接口发送给源侧MME。 |
| 重选消息 | Context Response | AMF<->MME | 在EPS和5GS重选流程中，源侧的MME或AMF发送该消息用于响应之前的Context Request消息。 |
| 重选消息 | Context Acknowledge | AMF<->MME | 当之前的Context Response消息被接收且原因值为acceptance时，目标侧MME或AMF发送Context Acknowledge消息，用以响应之前的Context Response消息。 |
| 重定位消息 | Forward Relocation Request | AMF<->MME | 在EPS和5GS切换过程中，源侧MME通过N26接口发送该消息给目标侧AMF，作为切换过程中重定位流程的一部分。<br>注：重定位请求消息也可由源侧AMF通过N26接口发送给目标侧MME。 |
| 重定位消息 | Forward Relocation Response | AMF<->MME | 在EPS和5GS切换过程中，目标侧MME或AMF应当发送该消息用以响应之前的<br>Forward Relocation Request消息。 |
| 切换消息 | Forward Relocation Complete Notification | AMF<->MME | 在EPS和5GS切换过程中，目标侧MME通过N26接口发送该消息给源侧AMF用于指示切换已完成。<br>注：重定位完成通知消息也可由目标侧AMF通过N26接口发送给源侧MME。 |
| 切换消息 | Forward Relocation Complete Acknowledge | AMF<->MME | 在EPS和5GS切换过程中，源侧MME或AMF应当发送该消息用以响应之前的Forward Relocation Complete Acknowledge消息。 |
| 切换消息 | Relocation Cancel Request | AMF<->MME | 在EPS和5GS切换取消过程中，源侧MME通过N26接口发送该消息给目标侧AMF用于取消切换。<br>注：重定位取消请求消息也可由源侧AMF通过N26接口发送给目标侧MME。 |
| 切换消息 | Relocation Cancel Response | AMF<->MME | 在EPS和5GS切换取消过程中，目标侧MME或AMF应当发送该消息用以响应之前的Relocation Cancel Response消息。 |
