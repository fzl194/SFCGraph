被叫场景下，被叫用户处于CONNECTED状态时，查询实时位置信息具体流程描述如下：

1. IMS系统向HSS发送User Data Request请求获取用户在EPS的位置信息。
2. HSS在收到IMS触发的位置查询请求时，向MME发起Insert Subscriber Data Request消息，携带值为1的EPS Location Information Request和Current Location Request，标识HSS查询请求指示的是获取UE实时位置信息。
3. 被叫用户处于CONNECTED状态，由于在MME上的UE位置信息可能不是实时的，MME通过位置查询请求Location Reporting Control从eNodeB上获取UE实时的位置信息。
4. eNodeB向MME发起Location Report消息，消息中携带UE实时位置信息ECGI和TAI。
5. MME给HSS响应Insert Subscriber Data Answer消息，上报UE实时的位置信息为ECGI和TAI。该消息同时携带信元“Age Of Location Information”，用于指示返回的位置信息在本地已经缓存的时长，如果该信元值为0，表示查询返回的是用户当前最新的位置信息。
  > **说明**
  > 当软参DWORD_EX39 BIT29配置为1时，MME发给HSS的Insert Subscriber Data Answer消息中还会携带“Current-Location-Retrieved”信元，如果该信元值为0，则表示系统上报了用户当前最新的位置信息。
6. HSS将实时的UE位置信息返回给IMS。

被叫场景下，被叫用户处于IDLE状态时，查询实时位置信息具体流程描述如下：

1. IMS系统向HSS发送User Data Request请求获取用户在EPS的位置信息。
2. HSS在收到IMS触发的位置查询请求时，向MME发起Insert Subscriber Data Request消息，携带值为1的EPS Location Information Request和Current Location Request，标识HSS查询请求指示的是获取UE实时位置信息。
3. 被叫用户处于IDLE状态，Paging后，通过Service Request再次与系统建立连接；eNodeB会将UE的实时位置信息通过Initial UE请求消息将ECGI，TAI带给MME。
4. MME给HSS响应Insert Subscriber Data Answer消息，上报UE实时的位置信息为ECGI和TAI。该消息同时携带信元“Age Of Location Information”，用于指示返回的位置信息在本地已经缓存的时长，如果该信元值为0，表示查询返回的是用户当前最新的位置信息。
  > **说明**
  > 当软参DWORD_EX39 BIT29配置为1时，MME发给HSS的Insert Subscriber Data Answer消息中还会携带“Current-Location-Retrieved”信元，如果该信元值为0，则表示系统上报了用户当前最新的位置信息。
5. HSS将实时的UE位置信息返回给IMS。

> **说明**
> 被叫场景下，查询实时位置信息时，被叫用户处于其他状态的应答情况如下：
>
> - 如果被叫用户不存在，且IDR消息携带的EPS User State Request、EPS Location Information Request、Current Location Request信元值为1、0、1或0、0、1，则返回给HSS携带“DIAMETER_UNABLE_TO_COMPLY”原因值的IDA应答消息；其他用户不存在的位置信息查询，返回给HSS携带“DIAMETER_ERROR_USER_UNKNOWN”原因值的IDA应答消息。
> - 如果被叫用户处于Deregistered状态，且IDR消息携带的EPS User State Request信元值为1，返回给HSS携带用户状态为“Detached”的IDA应答消息，此时应答消息不携带位置信息；如果用户处于Deregistered状态，且IDR消息携带的EPS User State Request信元值为0，返回给HSS携带“DIAMETER_UNABLE_TO_COMPLY”原因值的IDA应答消息。

#### [被叫场景下，查询本地位置信息流程](#ZH-CN_TOPIC_0170014752)

如果HSS下发的请求携带值为1的EPS Location Information Request，Current Location Request值为0，则MME取本地缓存的位置信息。

![](实现原理_70014752.assets/zh-cn_image_0172473812_2.png "点击放大")

<br>

被叫场景下，查询本地位置信息具体流程描述如下：

1. IMS系统向HSS发送User Data Request请求获取用户在EPS的位置信息。
2. HSS在收到IMS触发的位置查询请求时，会给MME发起Insert Subscriber Data Request消息。
3. HSS查询请求指示为值为1的EPS Location Information Request，按本地位置信息查询处理，直接获取MME本地缓存的位置信息。MME给HSS响应Insert Subscriber Data Answer消息中，携带MME本地缓存的UE位置信息ECGI和TAI，并返回Age Of Location Information信元，表明返回的位置信息在本地已经缓存的时长。
4. HSS将本地的UE位置信息返回给IMS。

> **说明**
> HSS向MME发送的Insert Subscriber Data Request消息中，同时携带值为1的EPS User State Request时：
>
> - 如果用于处于IDLE状态，MME寻呼用户。
>     - 寻呼成功时，MME发送的Insert Subscriber Data Answer消息中，携带的用户状态为“CONNECTED_REACHABLE_FOR_PAGING”，并携带实时位置信息。
>     - 寻呼失败时，MME发送的Insert Subscriber Data Answer消息中，携带的用户状态为“CONNECTED_NOT_REACHABLE_FOR_PAGING”，并携带MME本地缓存的位置信息。
> - 如果用户处于CONNECTED状态时，MME发送的Insert Subscriber Data Answer消息中，携带的用户状态为“CONNECTED_REACHABLE_FOR_PAGING”，并携带实时位置信息。
