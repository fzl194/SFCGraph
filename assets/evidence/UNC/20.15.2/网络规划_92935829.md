# 网络规划

AMF Pool内，每个AMF对应的PLMN、AMF Region ID和AMF Set ID是相同的。所以在AMF新建部署的时候，需要为组AMF Pool的AMF规划相同的AMF对应的PLMN、AMF Region ID和AMF Set ID。由于5G部署初期，新建AMF与现网MME分别为4、5G用户服务，相互独立组Pool，建议AMF Pool ID独立规划，不与现网MME Pool ID耦合 。即AMF GUAMI和MME的MMEID规划遵循以下几个原则：

- 融合网元中AMF的 <AMF Region ID> <AMF Set ID><AMF Pointer> ≠ 现网存量的<AMF Region ID> <AMF Set ID><AMF Pointer>
- 融合网元中AMF的 <AMF Region ID> <AMF Set ID(2~9位)> ≠ 现网存量的<MMEGI>
- 融合网元中AMF的 <AMF Set ID(0~1位)><AMF Pointer> ≠ 同POOL内其它MME的<MMEC>
- 融合网元中MME的 <MMEGI> ≠ 现网存量的<MMEGI>
- 融合网元中MME的<MMEGI><MMEC> ≠ 现网存量<AMF Region ID> <AMF Set ID> <AMF Pointer>（即<AMF Region ID> <AMF Set ID> <AMF Pointer>映射后的mapped GUMMEI）

详细网设部署要求请参见对应局点网设文档。

> **说明**
> 一个TA覆盖的所有RAN节点所连接的AMF Pool/AMF（如果未规划进任何AMF Pool）必须相同。因此，一个AMF Pool内各AMF的TA List必然相同，不同AMF Pool的TA List必然不同，未规划进任何AMF Pool的AMF之间TA List也必然不同。
