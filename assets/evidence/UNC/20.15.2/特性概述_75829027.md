#### [适用NF](#ZH-CN_TOPIC_0275829027)

MME

#### [定义](#ZH-CN_TOPIC_0275829027)

本特性通过在网元间传递eDRX（Extended Discontinuous Reception）信元并计算出合理的寻呼时机来缩短对移动性、实时性较低的M2M终端的监控寻呼信道时长，从而减少网络信令负荷，节省终端耗电量，延长电池使用寿命。

#### [客户价值](#ZH-CN_TOPIC_0275829027)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 缩短M2M终端的监控寻呼信道时长，减少网络信令负荷，为行业用户延长电池寿命，提升运营竞争力。 |
| 用户 | 节省M2M终端耗电量，延长电池使用寿命，提升用户体验。 |

#### [应用场景](#ZH-CN_TOPIC_0275829027)

对使用电池供电、实时性要求较低的M2M终端可开启eDRX模式，如：智能抄表业务中的电表、智能工业中的传感器、智能宠物健康中的颈环等。如下图所示：

#### [可获得性](#ZH-CN_TOPIC_0275829027)

**涉及NF**

| 涉及NF | 支持版本 | 功能说明 |
| --- | --- | --- |
| MME | UNC 20.5.0及后续版本 | - 支持eDRX信元的传递。<br>- 根据eDRX信元计算出合理的寻呼时机。 |
| eNodeB | 3GPP R13及后续版本 | 在寻呼流程中，eNodeB需要根据MME携带的eDRX信元对UE进行寻呼。 |
| S/PGW-C | UNC 20.5.0及后续版本 | - 当开启eDRX模式时，根据MME在Downlink Data Notification（DDN） Acknowledge消息中携带的DL Buffering Duration信元来缓存数据。<br>- 要求S-GW在Downlink Data Notification消息里面携带EBI信元，否则MME在Data Notification Acknowledge消息中不会携带DL Buffering Duration信元给S-GW。 |
| UE | 3GPP R13及后续版本 | 需要支持eDRX模式。 |
| HSS | 无特殊要求 | - 允许MME以获取用户签约数据的方式从HSS获得eDRX周期。<br>- 支持HSS修改和删除eDRX周期。 |

#### [对系统的影响](#ZH-CN_TOPIC_0275829027)

本特性对系统无影响。

#### [应用限制](#ZH-CN_TOPIC_0275829027)

- 本特性不适用于对实时性要求较高的物联网行业应用中。例如，在智能宠物跟踪应用本特性可能会导致无法实时定位宠物位置。
- 本特性在紧急呼叫场景下不生效。
- 本特性在启用后，若由于当前用户不在寻呼时间窗口内，无法立即寻呼到用户，可能会导致Idle态用户MME Pool迁移失败。

#### [原理概述](#ZH-CN_TOPIC_0275829027)

与普通终端相比较，在部分物联网行业应用中，移动性较低并且实时性要求较低的终端，频繁地接收网络侧的寻呼会缩短其电池寿命、且增加网络的信令负荷。基于上述原因，本特性通过网元间eDRX信元的传递并计算出合理的寻呼时机来缩短M2M终端的监控寻呼信道时长，达到减少网络信令负荷，节省终端耗电量。

如左上图所示，传统网络中使用的DRX技术节省功耗，其寻呼周期最大为秒级（最大寻呼周期2.56s）。如右上图所示，为了适应物联网更低功耗、时延不敏感的业务，采用eDRX模式，寻呼周期可以达到分钟、小时级别，达到终端即省电又满足时延的应用要求。

eDRX周期获取方式有如下三种：

- MME支持使用UE请求值。
- MME支持使用MME的本地配置值。
- MME支持以获取用户签约数据的方式从HSS获得eDRX周期，并支持HSS修改和删除eDRX周期。
