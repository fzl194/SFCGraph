## [操作步骤](#ZH-CN_OPI_0000001833450249)

1. 进入 “MML命令行-UNC” 窗口。
2. 增加APN1的I-UPF部署模式配置为NOIUPF。
  **[ADD IUPFAPNDPMODE](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UPF选择管理/I-UPF部署策略/APN粒度的I-UPF部署模式/增加I-UPF在特定园区专用APN下的部署模式配置（ADD IUPFAPNDPMODE）_76930386.md)**
3. 增加APN2的I-UPF部署模式配置为ALLOWIUPF。
  **[ADD IUPFAPNDPMODE](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UPF选择管理/I-UPF部署策略/APN粒度的I-UPF部署模式/增加I-UPF在特定园区专用APN下的部署模式配置（ADD IUPFAPNDPMODE）_76930386.md)**

## [任务示例](#ZH-CN_OPI_0000001833450249)

任务描述

某运营商针对专用DNN“special1.com”和“special2.com”，各自绑定园区所在地UPF。希望专网DNN“special1.com”只允许用户在园区内接入，专网DNN“special2.com”支持用户在园区内外都接入时，开启本特性。

脚本

//APN special1.com下，禁止插入I-UPF，并且在会话创建失败或修改失败的响应消息中，携带的NAS拒绝原因值为“特定切片和DNN的资源不足”。

```
ADD IUPFAPNDPMODE: APN="special1.com", NGDEPLOYMODE=NOIUPF, NGFAILNASCAUSE=INSUFFICIENT_RESOURCES_FOR_SPECIFIC_SLICE_AND_DNN;
```

//APN special1.com下，允许插入I-UPF。

```
ADD IUPFAPNDPMODE: APN="special2.com", NGDEPLOYMODE=ALLOWIUPF;
```
