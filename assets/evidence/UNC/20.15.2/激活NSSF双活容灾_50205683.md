## [操作场景](#ZH-CN_OPI_0250205683)

本操作指导介绍在运行网络中激活NSSF双活容灾的操作过程。

NSSF通过跨DC（Data Center）建立双活容灾组，两个NSSF互相备份且都处于运行状态，AMF与两个NSSF均互联且有优先级区分，实现容灾管理。当高优先级NSSF网元故障，AMF检测到高优先级NSSF故障则将业务选择到低优先级的NSSF，低优先级的NSSF网元能够接管故障高优先级NSSF的业务，保证业务连续性。NSSF故障恢复后，AMF会自动回切高优先级NSSF进行业务。

> **说明**
> 适用于AMF 、NSSF 。

## [操作步骤](#ZH-CN_OPI_0250205683)

配置NSSF：每个NSSF都按如下操作配置

1. 进入 “MML命令行-UNC” 窗口。
2. 配置双活NSSF的容灾实例信息。
  **[ADD DRINST](../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/容灾管理/配置容灾实例(ADD DRINST)_51011025.md)**
3. 配置容灾实例通信地址信息。
  **[ADD DRCOMM](../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/容灾管理/增加容灾实例地址(ADD DRCOMM)_51012925.md)**
4. 等待30秒后查询容灾信息，若数据备份状态为容灾准备状态，则执行下一步，否则 [请收集信息并寻求技术支持。](调测NSSF双活容灾_50205684.md#ZH-CN_OPI_0250205684__cmd128569268384)
  **[DSP DRINFO](../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/容灾管理/查询容灾信息(DSP DRINFO)_51012929.md)**
5. 在双活NSSF上分别进行“关闭”和“打开”本特性的License配置开关，确保容灾功能成功激活。
  **[SET LICENSESWITCH](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)**

配置AMF

请参考“初始配置与调测”的 [配置AMF](../../../../初始配置/UNC初始配置与调测/组网对接配置/配置AMF&MME&SGSN/配置AMF_29301201.md) 配置AMF上的静态NSSF IP地址，并设置双活NSSF的不同优先级（本大区部署的NSSF为高优先级，对端大区部署的NSSF为低优先级）。

## [任务示例](#ZH-CN_OPI_0250205683)

任务描述

启用NSSF双活容灾功能，配置双活NSSF的容灾实例信息和容灾实例通信地址信息。

脚本

**//配置NSSF** **1**

//配置容灾实例信息。

```
ADD DRINST: DRINSTID=0, DRINSTNAME="DR1";
```

//配置容灾实例通信地址信息，包括本端和对端的数据。

```
ADD DRCOMM: DRINSTID=0, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.1", VPNNAME="DR_VPN";
```

```
ADD DRCOMM: DRINSTID=1, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.2", VPNNAME="DR_VPN";
```

**//配置NSSF2**

//配置容灾实例信息。

```
ADD DRINST: DRINSTID=1, DRINSTNAME="DR2";
```

//配置容灾实例通信地址信息，包括本端和对端的数据。

```
ADD DRCOMM: DRINSTID=1, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.2", VPNNAME="DR_VPN";
```

```
ADD DRCOMM: DRINSTID=0, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.1", VPNNAME="DR_VPN";
```

**//等待30秒后查询容灾信息，数据备份状态为容灾准备状态后，再执行下一步。**

```
DSP DRINFO: PEERDRINSTID=0;
```

**//配置NSSF** **1**

//关闭NSSF双活容灾功能的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2DLNSSR01", SWITCH=DISABLE;
```

//打开NSSF双活容灾功能的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2DLNSSR01", SWITCH=ENABLE;
```

**//配置NSSF** **2**

//关闭NSSF双活容灾功能的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2DLNSSR01", SWITCH=DISABLE;
```

//打开NSSF双活容灾功能的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2DLNSSR01", SWITCH=ENABLE;
```

> **说明**
> - 此处APPID可使用**[LST ME](../../../../../OM参考/命令/UNC MML命令/平台服务管理/系统管理/版本信息/查询网元配置信息（LST ME）_47084797.md)**命令查询结果中的ME ID。
> - 若两套NSSF的容灾实例标识都不为“0”，则两套NSSF都要进行复位。
