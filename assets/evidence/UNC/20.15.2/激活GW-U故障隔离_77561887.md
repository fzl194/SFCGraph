## [操作场景](#ZH-CN_OPI_0277561887)

当运营商需要部署GW-U故障隔离功能时，需要对本功能进行配置，确保本功能可以正常使用。

> **说明**
> 适用于 GGSN、 SGW-C、PGW-C。

## [必备事项](#ZH-CN_OPI_0277561887)

前提条件

- 请仔细阅读[WSFD-213004 GW-U故障隔离特性概述](特性概述_74590842.md)。
- 完成加载License。

## [操作步骤](#ZH-CN_OPI_0277561887)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  **[SET LICENSESWITCH](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)**
3. 配置SGW-U和S11的绑定关系。
  **[ADD UPBINDS11](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UPF选择管理/UPF绑定S11接口/增加SGW-U与SGW-C侧S11接口的绑定关系（ADD UPBINDS11）_09653045.md)**
4. 配置GGSN-U/PGW-U和Gn/Gp/S5/S8接口的绑定关系。
  **[ADD UPBINDGNGP](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UPF选择管理/UPF绑定GnGp或S5S8接口/增加GGSN与Gn_Gp接口或PGW-U与S5_S8接口的绑定关系（ADD UPBINDGNGP）_78029307.md)**
5. 设置PGW-C发送给SGW-C的失败原因值。
  **[ADD SPGWCAUSECTRL](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/SM原因值管理/GTPv2会话流程原因值控制/增加SGW-C_PGW-C原因值控制参数（ADD SPGWCAUSECTRL）_09653621.md)**

## [任务示例](#ZH-CN_OPI_0277561887)

任务描述

任务1：配置SGW-U故障隔离功能。

任务2：配置GGSN-U/PGW-U故障隔离功能。

脚本

**任务1：配置SGW-U故障隔离功能。**

//打开本特性license。

```
SET LICENSESWITCH: LICITEM="LKV3GWUISL01", SWITCH=ENABLE;
```

//设置SGW-U和S11接口绑定信息。

```
ADD UPBINDS11: NFINSTANCENAME="UPF1",S11IPTYPE=Ipv4,S11IPV4 ="192.168.1.2",PRIORITY=3;
ADD UPBINDS11: NFINSTANCENAME="UPF2",S11IPTYPE=Ipv4,S11IPV4 ="192.168.1.2",PRIORITY=3;
```

**任务2：配置GGSN-U/PGW-U故障隔离功能。**

//打开本特性license。

```
SET LICENSESWITCH: LICITEM="LKV3GWUISL01", SWITCH=ENABLE;
```

//配置GGSN-U/PGW-U和Gn/Gp/S5/S8接口绑定信息。

```
ADD UPBINDGNGP: NFINSTANCENAME="UPF1",GNGPIPTYPE=Ipv4,GNGPIPV4="192.168.1.12";
ADD UPBINDGNGP: NFINSTANCENAME="UPF2",GNGPIPTYPE=Ipv4,GNGPIPV4="192.168.1.12";
```

//配置PGW-C发送给SGW-C的失败原因值。

```
ADD SPGWCAUSECTRL: PROCEDURETYPE=CRT_SES_PROC, CAUSESOURCE=UPF, CAUSEEXPIRATION=73;
```
