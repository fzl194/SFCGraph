## [命令功能](#ZH-CN_MMLREF_0225120889)

**适用NF：AMF**

该命令用于查询AMF针对5G用户异常信令抑制的相关参数。

## [注意事项](#ZH-CN_MMLREF_0225120889)

无

#### [操作用户权限](#ZH-CN_MMLREF_0225120889)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0225120889)

无

## [输出结果说明](#ZH-CN_MMLREF_0225120889)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 信令抑制开关 | 该参数用于设置系统是否开启用户异常信令抑制功能。 |
| 初始注册异常识别门限(次) | 该参数用于设置识别异常初始注册行为的门限，即在指定的检测周期内如果用户的初始注册次数达到了此参数设置的值，则认为此用户的初始注册行为异常。 |
| 初始注册异常测量周期(秒) | 该参数用于设置异常初始注册行为的检测周期。 |
| 初始注册异常抑制时长(分钟) | 该参数用于设置异常初始注册的抑制时长。满足初始注册抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。 |
| 注册抑制措施 | 该参数用于设置当系统检测到终端发起的注册行为异常后采取的抑制措施。 |
| 注册拒绝原因值 | 该参数用于设置当采取特定原因拒绝的抑制措施时，系统给终端发送的注册拒绝消息中携带的协议原因值。 |
| 4G到5G初始注册信令抑制开关 | 该参数用于设置系统是否开启4G到5G初始注册以及无N26接口场景的4G到5G的重选和初始注册信令抑制功能。当本参数设置为“OFF（关闭）”，上述流程不进行信令抑制。当本参数设置为“ON（开启）”，上述流程进行信令抑制，涉及的异常识别门限、异常测量周期、抑制时长以及抑制措施使用初始注册流程配置。 |
| 服务请求异常识别门限(次) | 该参数用于设置识别异常服务请求行为的门限，即在一个测量周期内如果UE的服务请求次数达到了此参数设置的值，则认为此用户的服务请求行为异常。 |
| 服务请求异常测量周期(秒) | 该参数用于设置异常服务请求行为的测量周期。 |
| 服务请求异常抑制时长(分钟) | 该参数用于设置异常服务请求的抑制时长。执行服务请求抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。 |
| 服务请求抑制措施 | 该参数用于设置当系统检测到终端发起的服务请求行为异常后采取的抑制措施。 |
| 服务请求拒绝原因值 | 该参数用于设置当采取特定原因拒绝的抑制措施时，系统给终端发送的服务拒绝消息中携带的协议原因值。 |
| PDU会话建立异常识别门限(次) | 该参数用于设置识别异常PDU会话建立行为的门限，即在一个测量周期内如果终端PDU会话建立次数达到了此参数设置的值，则认为此终端的PDU会话建立行为异常。 |
| PDU会话建立异常测量周期(秒) | 该参数用于设置异常PDU会话建立行为的测量周期。 |
| PDU会话建立异常抑制时长(分钟) | 该参数用于设置异常PDU会话建立的抑制时长。执行PDU会话建立抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。 |
| PDU会话建立抑制措施 | 该参数用于设置当系统检测到终端发起的PDU会话创建行为异常后采取的抑制措施。 |
| PDU会话建立原因值 | 该参数用于设置当采取特定原因拒绝的抑制措施时，系统给终端发送的下行NAS传输消息中携带的协议原因值。 |
| 去注册原因值 | 该参数用于设置网络侧发起的去注册请求消息中携带的协议原因值。当系统检测到终端的注册（注册请求或注册失败）、服务请求或PDU会话建立行为异常， 并且采取网络侧去注册的抑制措施时，给终端发送的去注册请求消息（去注册类型为“re-registration not required”）中携带此原因值。 |
| Parking DNN | 该参数用于设置执行Parking DNN激活抑制策略时使用的DNN。异常PDU会话建立流程中执行Parking DNN抑制策略时使用Parking DNN建立一个假PDU会话。 |
| Parking DNN唤醒措施 | 该参数用于设置当对用户进行Parking DNN激活抑制的时间超时后，系统通知用户重新接入业务的方式。 |
| Parking切片业务类型 | 该参数用于设置执行Parking DNN激活抑制策略时使用的切片业务类型，如eMBB（1）、URLLC（2）、MIoT（3）等协议定义的标准SST，或者运营商自定义的SST。<br>异常PDU会话建立流程中执行Parking DNN抑制策略时，使用Parking切片和Parking DNN发现SMF。 |
| Parking切片细分标识 | 该参数用于设置执行Parking DNN激活抑制策略时使用的切片细分标识。切片细分标识是根据网络切片所提供的服务特点、所服务的对象差异，对某种网络切片的进一步细分。 |
| 4G到5G重选信令抑制开关 | 该参数用于设置系统是否开启用户4G到5G重选流程的异常信令抑制功能。 |
| 4G到5G重选异常识别门限(次) | 该参数用于设置识别异常4G到5G重选行为的门限，即在指定的检测周期内如果用户的4G到5G重选次数达到了此参数设置的值，则认为此用户的4G到5G重选行为异常。 |
| 4G到5G重选异常检测周期(秒) | 该参数用于设置异常4G到5G重选行为的检测周期。 |
| 4G到5G重选异常抑制时长(分钟) | 该参数用于设置异常4G到5G重选的抑制时长。执行4G到5G重选抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。 |
| 4G到5G重选抑制措施 | 该参数用于设置当系统检测到终端发起的4G到5G重选异常后采取的抑制措施。 |
| 4G到5G重选拒绝原因值 | 该参数用于设置当4G到5G重选采取特定原因拒绝的抑制措施时，4G到5G重选流程系统给终端发送的注册拒绝消息中携带的协议原因值。 |
| 4G到5G切换信令抑制开关 | 该参数用于设置系统是否开启用户4G到5G切换流程的异常信令抑制功能。 |
| 4G到5G切换异常识别门限(次) | 该参数用于设置识别异常4G到5G切换行为的门限，即在指定的检测周期内如果用户的4G到5G切换次数达到了此参数设置的值，则认为此用户的4G到5G切换行为异常。 |
| 4G到5G切换异常检测周期(秒) | 该参数用于设置异常4G到5G切换异常行为的检测周期。 |
| 4G到5G切换异常抑制时长(分钟) | 该参数用于设置异常4G到5G切换的抑制时长。执行4G到5G切换抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。 |
| 4G到5G切换抑制措施 | 该参数用于设置当系统检测到终端发起的4G到5G切换异常后采取的抑制措施。 |
| 4G到5G切换拒绝原因值 | 该参数用于设置当4G到5G切换采取特定原因拒绝的抑制措施时，4G到5G切换流程系统给MME发送的Forward Relocation Response消息中携带的拒绝原因值。 |
| 注册失败信令抑制开关 | 该参数用于设置系统是否开启用户注册失败流程的异常信令抑制功能。 |
| 注册失败异常识别门限(次) | 该参数用于设置识别异常注册失败行为的门限，即在指定的检测周期内如果用户的注册失败次数达到了此参数设置的值，则认为此用户的注册失败行为异常。 |
| 注册失败异常测量周期(秒) | 该参数用于设置异常注册失败行为的检测周期。 |
| 注册失败异常抑制时长(分钟) | 该参数用于设置异常注册失败的抑制时长。满足注册失败抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。 |
| 注册失败抑制措施 | 该参数用于设置当系统检测到终端发起的注册失败流程异常后采取的抑制措施。 |
| 注册失败拒绝原因值 | 该参数用于设置当采取特定原因拒绝的抑制措施时，系统给终端发送的注册拒绝消息中携带的协议原因值。 |
