#### [相关概念](#ZH-CN_TOPIC_0279067180)

*表1 相关概念*

| 位置信息 | 说明 |
| --- | --- |
| LAC | 位置区码。在一个PLMN（Public Land Mobile Network）内部唯一标识一个位置区，它的范围是0～65535。 |
| TAC | 跟踪区域码。TA是LTE/SAE系统为UE的位置管理新设立的概念，TAC是每个TA的编号。 |
| RAC | 路由区编码。在一个本地区域中唯一标识一个路由区域。一个Routing Area是SGSN寻呼处于Idle状态的终端的区域，是PS业务的寻呼区域。 |

#### [位置区域划分原则](#ZH-CN_TOPIC_0279067180)

*表2 位置区域划分原则*

| 类型 | 说明 |
| --- | --- |
| 3GPP接入 | - 一个区域可以是一个或多个LAC。<br>- 一个区域可以是一个或多个RAC。<br>- 一个区域可以是一个或多个TAC。<br>- 一个区域可以是LAC、RAC、TAC的组合。<br>- 一个LAC、RAC、TAC只能属于一个区域。<br>- 一个区域可以是一个或多个SGSN IP/SGW IP。<br>- 一个SGSN IP/SGW IP只能属于一个区域。 |
| WLAN接入 | - 一个区域可以是一个或多个ePDG IP。<br>- 一个ePDG IP只能属于一个区域。 |

#### [业务流程](#ZH-CN_TOPIC_0279067180)

1. 用户位置信息获取。
  UNC 从激活请求消息中获取用户位置信息（3GPP接入：LAC/TAC/RAC/SGSN IP/SGW IP；WLAN接入：ePDG IP）。
    - 对于3GPP接入，UNC从SGSN/MME发送的Create PDP Context Request/Create Session Request消息中获取用户位置信息，LAC/TAC/RAC的获取如[表3](#ZH-CN_TOPIC_0279067180__table974622901419)所示。
      *表3 LAC/TAC/RAC的获取*

      | 位置信息 | 获取途径（GTPv1/GTPv2消息） |
      | --- | --- |
      | LAC | 从ULI的以下子信元中获取，优先级从高到低依次为：CGI、SAI、RAI。 |
      | TAC | 从ULI的TAI子信元中获取。 |
      | RAC | 从ULI的RAI子信元中获取。 |
    - 对于WLAN接入，UNC从TGW发送的Create PDP Context Request消息中获取用户位置信息。
2. 虚拟APN映射。
  UNC 将激活请求消息中的请求APN作为虚拟APN，对于3GPP接入基于LAC/TAC/RAC或基于SGSN IP/SGW IP、对于WLAN接入基于ePDG IP映射真实APN。如 [图1](#ZH-CN_TOPIC_0279067180__fig223364812541) 所示，以基于LAC映射真实APN为例进行介绍。UE1的激活请求消息中携带的LAC为0x1111， UNC 查找本地LAC group的配置，获得0x1111所属的LAC group为A。进而查找本地的虚拟APN配置，获得LAC group A对应的真实APN是City A。

3. 基于位置的业务管理。
    - UPF根据位置信息，给UE分配特定IP段的IP。
    - 基于位置的性能统计上报。
      位置区A以真实APN“City A”上报性能统计数据，位置区B以真实APN“City B”上报性能统计数据。如果SGSN/MME支持当用户位置区域发生变化时去激活用户，则基于位置区域的性能统计更为准确，否则 UNC 上报的是基于用户初始位置区域的性能统计数据。
    - PCF根据位置信息规划计费和带宽控制等策略。
