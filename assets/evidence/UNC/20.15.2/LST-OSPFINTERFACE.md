#### [命令功能](#ZH-CN_CONCEPT_0000001600866061)

该命令用于查询OSPF接口配置。

#### [注意事项](#ZH-CN_CONCEPT_0000001600866061)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001600866061)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001600866061)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROCID | 进程号 | 可选必选说明：可选参数<br>参数含义：OSPF进程号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～4294967295。<br>默认值：无 |
| AREAID | 区域标识 | 可选必选说明：可选参数<br>参数含义：OSPF区域号。<br>数据来源：全网规划<br>取值范围：IPv4地址类型。点分十进制格式。<br>默认值：无 |
| IFNAME | 接口名 | 可选必选说明：可选参数<br>参数含义：指定特定的接口名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。<br>默认值：无<br>配置原则：请使用LST INTERFACE命令查看可用接口。 |

#### [输出结果说明](#ZH-CN_CONCEPT_0000001600866061)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 进程号 | 该参数指定用户创建的OSPF进程号。 |
| 区域标识 | 该参数指定用户创建的OSPF区域号。 |
| 接口名 | 该参数指定用户创建的特定的接口名称。 |
| 指定路由器优先级 | 该参数指定路由器优先级，优先级高的接口在DR选举时被首先考虑。如果一台设备的接口优先级为0，则它不会被选举为DR或BDR。 |
| 接口网络类型 | 该参数指定接口网络类型，包含P2P、P2MP、广播、NBMA。 |
| 在P2MP网络上忽略网络掩码 | 该参数指定在P2MP网络上忽略对网络掩码的检查。在P2MP网络上，当设备的掩码长度不一致时，使用此命令忽略对Hello报文中网络掩码的检查，从而可以正常建立OSPF邻居关系。 |
| 使能Smart-Discover | 该参数指定使能Smart-discover特性，当该多址网络上的DR、BDR的接口状态发生变化时，使能了Smart-discover功能的接口会主动在该网段发送Hello报文，参与到DR或者BDR的选举中。 |
| 邻居失效的时间（s） | 该参数指定OSPF的邻居失效的时间，邻居的失效时间是指：在该时间间隔内，若未收到邻居的Hello报文，就认为该邻居已失效。 |
| 邻居发送Hello包时间间隔（s） | 该参数指定接口发送Hello包时间间隔，hello interval的值写入Hello报文中后随之发送。hello interval的值越小，发现网络拓扑改变的速度越快，路由开销也就越大。确定接口和邻接路由器的参数要保持一致。 |
| 是否使能邻居失效定时器保守模式 | 该参数指定使能邻居失效定时器保守模式。使能后，即使邻居失效时间小于10s，仍按照实际配置值来判断邻居是否失效。 |
| 重发LSA的时间间隔（s） | 该参数指定接口重传LSA的时间间隔。当一台路由器向它的邻居发送一条＂链路状态广播＂（LSA）后，需要等到对方的确认报文。若在该重传LSA的时间间隔内未收到对方的确认报文，就会重传这条LSA。相邻路由器重传LSA时间间隔的值不要配置得太小，否则将会引起不必要的重传。 |
| 发送Hello报文的Poll时间间隔（s） | 该参数指定NBMA网络上发送轮询Hello报文的时间间隔。在NBMA网络上，当邻居失效后，路由器将按配置的轮询时间间隔定期地发送Hello报文。轮询时间间隔值至少应为Hello报文时间间隔的4倍，OSPF不支持NULL接口配置，单位是秒。 |
| 等待超时时间（s） | 该参数指定等待超时时间，在该时间间隔内，若未收到Backup Seen事件，则可以开始进行DR选举。通过合理配置Wait定时器的时间间隔，可以减少网络中DR、BDR的变化频率，减少网络振荡。 |
| 接口开销值 | 该参数指定接口开销值。 |
| 使能BFD | 该参数指定使能BFD功能。 |
| 接收BFD包的最小时间间隔（ms） | 该参数指定期望从对端接收BFD报文的最小接收间隔，单位是毫秒。 |
| 发送BFD包的最小时间间隔（ms） | 该参数指定向对端发送BFD报文的最小发送间隔，单位是毫秒。 |
| 本地检测倍数 | 该参数指定会话检测倍数，如果BFD会话在配置的检测周期内没有收到对端发来的BFD报文，则认为链路发生了故障，BFD会话的状态将会置为Down。 |
| 阻塞接口动态创建BFD特性 | 该参数指定阻塞接口动态创建BFD特性。 |
| 接口对LSA的传输延迟时间（s） | 该参数指定OSPF接口对LSA的传输延迟时间。 |
| 低链路质量Cost值 | 该参数指定低链路质量Cost值。 |
| 可达性抑制 | 该参数指定使能接口的可达性抑制。 |
| 使能MTU | 该参数指定使能接口MTU报文校验。 |
| 禁止自动使能MPLS LDP功能标志位 | 该参数指定个禁止OSPF接口自动使能MPLS LDP功能标志位。 |
| 使能LDP和OSPF同步功能 | 该参数指定使能LDP和OSPF同步功能。 |
| 阻止接口上运行LDP和OSPF同步功能 | 该参数指定是否阻止接口上运行LDP和OSPF同步功能。 |
| 禁止该接口接收和发送OSPF报文 | 该参数指定抑制接口收发OSPF报文，如果要使OSPF路由信息不被其他网络中的设备获得，并且使本地设备不接收网络中其他设备发布的路由更新信息，可使用此功能禁止此接口接收和发送OSPF报文。 |
| OSPF共网段虚拟系统使能标志 | 该参数指定使能了OSPF共网段虚拟系统标志。 |
| OSPF虚Router-id配置标志 | 该参数指定OSPF虚Router-id配置是否生效标志。 |
| OSPF配置的虚Router-id | 该参数指定OSPF的虚Router-id。 |
| 保持最大开销值标志 | 该参数指定配置本地设备通告链路最大开销值的时间。 |
| 保持最大开销值时间间隔（s） | 该参数指定OSPF在本地设备的中保持通告最大开销值的时间。 |
| 永久保持最大开销值标志 | 该参数指定在LDP会话重新建立之前，OSPF在本地设备中永久通告最大开销值。 |
| 抑制邻居时间间隔（s） | 该参数指定配置接口不建立OSPF邻居而等待LDP会话建立的时间间隔。 |
| BFD会话与接口链路状态绑定标志 | 该参数指定将BFD会话状态与OSPF Auto FRR进行绑定。BFD检测到接口链路故障后，BFD会话状态会变为Down并触发系统进行快速重路由，将流量从故障链路切换到备份链路上，从而达到流量保护的目的。 |
| 阻止指定OSPF接口的FRR能力 | 该参数指定阻止指定OSPF接口的FRR能力。 |
