# 业务流程

MSC POOL场景下的CSFB语音回落过程跟非MSC POOL场景下的CSFB语音回落过程完全一致，具体请参阅 [WSFD-102301 基于CSFB的语音业务](../../WSFD-102301 基于CSFB的语音业务_70014756.md) 。两种场景的差别仅在用户联合附着或联合位置更新时对MSC的选择机制不同，具体的选择机制如 [表1](#ZH-CN_TOPIC_0166375512__table_04) 所示。

*表1 Location Update流程MSC获取规则*

| 组网场景 | 生成规则 |
| --- | --- |
| 非MSC POOL | 根据<br>**[ADD TAILAI](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/电路域联合业务/TAI与LAI对应关系/增加TAI与LAI对应关系(ADD TAILAI)_72345017.md)**<br>和<br>[**ADD LAIVLR**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/电路域联合业务/LAI与VLR号对应关系/增加LAI与VLR号对应关系(ADD LAIVLR)_72345015.md)<br>命令，配置得出TA和VLR的对应关系，当用户接入时，通过用户所在的TA信息就确定了该用户归属的VLR。 |
| MSC POOL | MME根据用户当前所在的LAI以及IMSI的V值从POOL中选择对应的MSC。<br>说明：V值由MME根据IMSI号码通过一定的算法获得。V值和MSC之间的关系是多对一的关系，即每个V值只能对应一个MSC，一个MSC可以对应多个V值。 |
