#### [触发和上报位置事件](#ZH-CN_TOPIC_0000001766101465)

连续运动定位场景，比如物流车跟踪，相比于采用定时发起单次定位产生更多的信令交互，针对一个终端发起周期性定位，按周期上报定位结果时，可以有效提升定位效率。 [图1](#ZH-CN_TOPIC_0000001766101465__fig43544485372) 描述周期性位置订阅的信令流程。

**图1** 周期性位置定位流程

<br>

![](周期性5GC-MT-LR延迟流程_66101465.assets/zh-cn_image_0000001793001257_2.png)

**1：** LCS Client向GMLC发起位置服务请求，LCS Service Request提供了所请求位置报告的类型和相关参数。对于周期性定位，LCS Service Request包括连续位置报告之间的时间间隔、报告总数，并且可以包括位置QoS。

**2a-2b：** GMLC收到位置服务请求后，根据UE标识向UDM查询UE所在的AMF。UDM向GMLC返回的响应消息中携带查询到的AMF信息。

**3a-3b：** GMLC向UDM提供的AMF发送Namf_Location_Provide_PositioningInfo Request消息，请求AMF进行定位业务，消息中携带周期性定位相关信元。AMF收到请求后向GMLC返回响应消息。

*表1 Namf_Location_Provide_PositioningInfo Request消息中周期性定位相关参数*

| 信元 | 信元解释 |
| --- | --- |
| lcsLocation | 所请求的位置测量类型，如当前位置、初始位置、最后已知位置、延迟位置等。<br>周期性定位测量场景值为“DEFERRED_LOCATION”。 |
| ldrType | 延迟定位请求的类型。当lcsLocation设置为“DEFERRED_LOCATION”时该信元出现。<br>周期性定位测量场景值为“PERIODIC”。 |
| hgmlcCallBackURI | H-GMLC对延迟定位请求的回调URI。当lcsLocation设置为“DEFERRED_LOCATION”时该信元出现。<br>周期性定位测量场景，GMLC生成CallBackURI并发给AMF，AMF提供给LMF，用于LMF周期性上报位置信息。 |
| ldrReference | 延迟定位请求的LDR Reference Number。当lcsLocation设置为“DEFERRED_LOCATION”时该信元出现。<br>周期性定位场景GMLC支持生成ldrReference，用于LMF关联LMF和GMLC会话使用。 |
| periodicEventInfo | 用于延迟定位请求的周期性事件报告的信息。当ldrType设置为“PERIODIC”时该信元出现。 |
| locationNotificationUri | 上报位置变化的回调URI。 |
| lpHapType | 用于网络侧发起的低功耗和/高精度定位。 |

**4：** GMLC向LCS Client返回响应消息。

**5：** （可选）如果UE处于CM-IDLE状态：AMF首先对目标UE发起Paging流程，UE通过响应Paging切换为CONNECTED状态，并通过Service Request流程上报当前的NCGI信息。

**6** **：** AMF选择LMF，向LMF发送Nlmf_Location_DetermineLocation_Request消息，消息中携带LCS Correlation ID、NCGI、SUPI、AMF ID及周期性定位相关信元。

*表2 Nlmf_Location_DetermineLocation_Request消息周期性定位相关信元*

| 信元 | 信元解释 |
| --- | --- |
| ldrType | 延迟定位请求的类型，周期性定位测量场景值为“PERIODIC”。 |
| hgmlcCallBackURI | H-GMLC对延迟定位请求的回调URI。<br>周期性定位测量场景，GMLC生成CallBackURI并发给AMF，AMF提供给LMF，用于LMF周期性上报位置信息。 |
| ldrReference | 延迟定位请求的LDR Reference Number。 |
| periodicEventInfo | 用于延迟定位请求的周期性事件报告的信息。当ldrType设置为“PERIODIC”时该信元出现。 |

**7a-7b：** LMF发送Namf_Communication_N1N2MessageSubscribe Request消息给AMF，订阅N1 LCS消息。

**8a-8b** ：（可选）LMF发送Namf_Communication_N1N2MessageSubscribe Request消息给AMF，订阅N2 NRPPa消息和N1 LPP消息。

**9-13b：** LMF通过AMF向UE发送LCS Periodic-Triggered Invoke消息通知终端周期定位。LCS Periodic-Triggered Invoke消息封装在Namf_Communication_N1N2MessageTransfer Request消息的N1N2MessageTransferReqData信元中。如果UE支持周期性定位，则UE通过AMF向LMF返回消息，消息中携带LCS Periodic-Triggered返回结果。

**14-16：** LMF通过AMF向UE发送Release Complete消息。

**17a-17b：** LMF向AMF订阅小区位置更新事件。

**18：** LMF向AMF返回Nlmf_Location_DetermineLocation Response消息，返回UE当前的位置信息。

**19：** AMF发送Namf_Location_EventNotify消息给GMLC，上报UE当前位置信息。AMF将选择的LMF通知给GMLC，GMLC才能在取消低功耗/周期定位流程中携带正确的LMF ID。

**20：** GMLC发送Notification消息上报位置信息给LCS Client。

**21：** UE进行周期性测量，当有周期性位置事件上报时，UE发送UPLINK NAS TRANSPORT消息携带周期性位置事件给AMF，Additional information信元中携带deferred routing identifier，通过AMF向LMF透传位置事件。

**22-28b：** AMF通过deferred routing identifier找到LMF，将UE的周期事件封装在Namf_Communication_N1N2MessageTransfer Request消息中透传给LMF。LMF通过AMF向UE发送确认消息，终端收到后向LMF返回Release Complete消息。

**29：** （可选）如果需要进行位置估算，LMF执行 [公共子流程](公共子流程_46574620.md) 中的一个或者多个定位过程，根据获得的位置测量或者位置估算结果确定UE的位置及位置估算的时间戳。

**30：** LMF发送EventNotify消息将位置信息上报给GMLC，reportedEventType信元置为"PERIODIC_EVENT"表示周期性上报。同时携带callbackURI和ldrReference，用于定位GMLC实例和上下文。

**31：** GMLC发送EventNotify消息上报位置信息给LCS Client，消息中notifyType为“PERIODIC_EVENT”。

> **说明**
> 上述步骤 [21](#ZH-CN_TOPIC_0000001766101465__p03912521031) - [31](#ZH-CN_TOPIC_0000001766101465__p1573371441417) 可能会根据周期性定位参数重复执行。

#### [取消周期性定位流程](#ZH-CN_TOPIC_0000001766101465)

取消周期性定位流程分两种：UE触发的取消定位流程和网络侧触发的取消定位流程。

**UE触发的取消定位流程**

**图2** UE触发的取消周期定位流程

<br>

![](周期性5GC-MT-LR延迟流程_66101465.assets/zh-cn_image_0000001738799944_2.png)

**1:** （可选），如果UE处于CM-IDLE状态，则UE向AMF发起业务请求，建立与AMF之间的信令连接。

**2** **：** UE向AMF发送UPLINK NAS TRANSPORT消息，消息中封装LCS MSCancelDeferredLocation Invoke消息，Additional information信元中携带deferred routing identifier，在取消周期定位流程中携带LMF ID，通过AMF向LMF发起取消定位请求消息。

**3a-3b：** AMF根据deferred routing identifier，将LCS MSCancelDeferredLocation Invoke消息封装在Namf_Communication_N1MessageNotify Request消息中带给LMF。LMF返回Namf_Communication_N1MessageNotify Response消息。

**4-6：** LMF处理取消定位流程，通过Namf_Communication_N1N2MessageTransfer Request发送取消定位结果消息给AMF；AMF通过DOWNLINK NAS TRANSPORT消息转发给UE，向LMF发送Namf_Communication_N1N2MessageTransfer Response消息。

**7-8b：** UE返回Release Complete消息，此消息只是一个确认消息，AMF将消息透传给LMF。

**9-11：** （可选）LMF通过Namf_Communication_N1N2MessageTransfer Request发送LPP Abort消息给AMF。AMF通过DOWNLINK NAS TRANSPORT消息转发给UE，向LMF发送Namf_Communication_N1N2MessageTransfer Response消息。

**12：** LMF通过Namf_Communication_N1N2MessageTransfer Request发送Positioning Deactivation消息给AMF。

**13：** AMF将POSITIONING DEACTIVATION消息转发给(R)AN，通知(R)AN释放资源。

**14：** AMF返回Namf_Communication_N1N2MessageTransfer Response消息给LMF。

**15：** LMF通过Namf_Communication_N1N2MessageTransfer Request发送Measurement Abort消息给AMF。

**16：** AMF将MEASUREMENT ABORT消息转发给(R)AN，通知(R)AN停止测量。

**17：** AMF返回Namf_Communication_N1N2MessageTransfer Response消息给LMF。

**18a-18b：** LMF发送Namf_Communication_N1N2MessageUnSubscribe Request消息通知AMF取消订阅UE相关的订阅。

**19a-19b：** LMF发送Namf_EventExposure_Unsubscribe Request消息通知AMF取消位置订阅，AMF发起用户去注册流程。

**20-21：** LMF通知GMLC取消位置定位服务，GMLC通知LCS Client取消位置定位服务。

**网络侧触发的取消定位流程**

**图3** 网络侧触发的取消定位流程

<br>

![](周期性5GC-MT-LR延迟流程_66101465.assets/zh-cn_image_0000001785680101_2.png)

**1：** LCS Client向GMLC发送Ngmlc_Location_CancelLocation Request消息，请求取消周期性定位流程。

**2：** GMLC收到消息后，向AMF发送Namf_Location_CancelLocation Request消息，请求取消周期性定位流程，消息中携带hgmlcCallBackURI、ldrReference和servingLmfIdentification。

**3：** AMF向LMF发送Nlmf_Location_CancelLocation Request消息，请求取消周期性定位流程，消息中携带hgmlcCallBackURI和ldrReference。

**4：** （可选）如果UE处于CM-IDLE状态：AMF首先对目标UE发起Paging流程，UE通过响应Paging切换为CM-CONNECTED状态，并通过Service Request流程上报当前的NCGI信息。

**5-7：** AMF通过DOWNLINK NAS TRANSPORT消息封装LCS CancelDeferredLocation Invoke消息发送给UE，UE返回LCS CancelDeferredLocation的响应消息给AMF。AMF收到UE的响应消息后，给UE返回确认报文Release Complete。

**8-10：** （可选）LMF通过Namf_Communication_N1N2MessageTransfer Request发送LPP Abort消息给AMF。AMF通过DOWNLINK NAS TRANSPORT消息转发给UE，向LMF发送Namf_Communication_N1N2MessageTransfer Response消息。

**11：** LMF通过Namf_Communication_N1N2MessageTransfer Request发送Positioning Deactivation消息给AMF。

**12：** AMF将POSITIONING DEACTIVATION消息转发给(R)AN，通知(R)AN释放资源。

**13：** AMF返回Namf_Communication_N1N2MessageTransfer Response消息给LMF。

**14：** LMF通过Namf_Communication_N1N2MessageTransfer Request发送Measurement Abort消息给AMF。

**15：** AMF将MEASUREMENT ABORT消息转发给(R)AN，通知(R)AN停止测量。

**16：** AMF返回Namf_Communication_N1N2MessageTransfer Response消息给LMF。

**17a-17b：** LMF发送Namf_Communication_N1N2MessageUnSubscribe Request消息通知AMF取消订阅UE相关的订阅。

**18a-18b：** LMF发送Namf_EventExposure_Unsubscribe Request消息通知AMF取消位置订阅。

**19：** LMF向AMF返回Nlmf_Location_CancelLocation Response消息。

**20-21：** AMF向GMLC返回Namf_Location_CancelLocation Response消息，GMLC向LCS Client返回Ngmlc_Location_CancelLocation Response消息。

#### [UE进出MICO（Mobile Initiated Connection Only）模式定位流程](#ZH-CN_TOPIC_0000001766101465)

MICO模式的终端在CM-IDLE态时不侦听寻呼信道，不能接受寻呼，只能UE主动发起通讯。低功耗终端使用内置电池，为了能够保持更长时间工作，会进入MICO模式，通过不侦听寻呼信道帮助UE省电。

- 对于已经触发周期性定位的MICO模式的终端，在进入CM-IDLE态前终端会先取消周期性定位流程（参考[取消周期性定位流程](#ZH-CN_TOPIC_0000001766101465__section1598911414160)中的UE触发的取消周期定位流程）。LCS Client收到取消定位通知后会再次发起周期性定位，AMF发现UE处于MICO模式的CM-IDLE状态，会向GMLC返回响应消息，保存周期性定位相关参数，待UE进入CM-CONNECTED态后继续进行周期性定位流程（参考[触发和上报位置事件](#ZH-CN_TOPIC_0000001766101465__section18451155321317)）。相关流程如[图4](#ZH-CN_TOPIC_0000001766101465__fig1520232018619)所示。
- 如果初始触发周期性定位时，终端处于MICO模式的CM-IDLE态，AMF收到GMLC发起的周期性定位请求，会向GMLC返回响应消息，保存周期性定位相关参数，待UE进入CM-CONNECTED态后继续进行周期性定位流程（参考[触发和上报位置事件](#ZH-CN_TOPIC_0000001766101465__section18451155321317)）。相关流程如[图4](#ZH-CN_TOPIC_0000001766101465__fig1520232018619)的步骤4~步骤9所示。
