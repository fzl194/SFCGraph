## [命令功能](#ZH-CN_MMLREF_0209653820)

**适用NF：PGW-C、SMF**

该命令用于查询融合计费模板（Converged Charging Template），用于配置融合计费相关参数。

## [注意事项](#ZH-CN_MMLREF_0209653820)

无

#### [操作用户权限](#ZH-CN_MMLREF_0209653820)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0209653820)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| CCTMPLTNAME | 融合计费模板名称 | 可选必选说明：可选参数<br>参数含义：该参数用于设置融合计费模板名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是1~63。不区分大小写。<br>默认值：无<br>配置原则：无 |

## [输出结果说明](#ZH-CN_MMLREF_0209653820)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 融合计费模板名称 | 该参数用于设置融合计费模板名称。 |
| CHF交互使能开关 | 该参数用于配置融合计费模板中用户激活时是否需要到CHF交互。 |
| 初始RG个数 | 该参数用于配置融合计费模板中用户激活时发送的初始请求消息中允许携带RG的最大个数。 |
| CHF交互等待CHF响应开关 | 该参数用于配置融合计费模板中用户激活需要到CHF交互的情况下是否需要等待CHF响应。 |
| RG来源 | 该参数用于配置融合计费模板中用户激活需要到CHF交互的情况下所携带的RG来源。 |
| 配额空闲时间门限值(秒) | 该参数用于设置配额空闲时间门限值，数据包停止传送后，QHT立即开始计时，当数据包停止传送时间达到此门限值，向CHF上报使用的配额。 |
| 流量阈值触发百分比(%) | 该参数用于设置一个流量阈值，当有数据包到达时，SMF/PGW-C必须计算剩余的流量配额，如果剩余的配额小于等于流量阈值便触发一个Charging Data Request消息，消息中要包含已使用的数据流量。 |
| 时间阈值触发百分比(%) | 该参数用于设置一个时间阈值，当有数据包到达时，SMF/PGW-C必须计算剩余的时间配额，如果剩余的配额小于等于时间阈值便触发一个Charging Data Request消息，消息中要包含已使用的数据流量。 |
| 事件阈值触发百分比(%) | 该参数用于设置一个事件阈值，当有事件成功时，SMF/PGW-C必须计算剩余的事件配额，如果剩余的配额小于等于事件阈值便触发一个Charging Data Request消息，消息中要包含已使用的数据流量。 |
| 在线配额有效时长(分) | 该参数用于设置RG级配额的有效时长；SMF/PGW-C收到CHF下发的配额后按有效时长启定时器，超时后SMF/PGW-C立即向CHF上报配额用量并申请新配额。 |
| 计费条件改变阈值 | 该参数用于指定计费条件改变几次后会触发Charging Data Request Update消息。 |
| PDU流量阈值(MB) | 该参数用于指定PDU级触发Charging Data Request Update消息的流量阈值。 |
| PDU时长阈值(分钟) | 该参数用于指定PDU级别触发Charging Data Request Update消息的时长阈值。 |
| 业务级流量阈值(MB) | 该参数用于指定RG级别触发Charging Data Request Update消息的流量阈值。 |
| 业务级时长阈值(分钟) | 该参数用于指定RG级别触发Charging Data Request Update消息的时长阈值。 |
| QF流量阈值(MB) | 该参数用于指定QF级别触发Charging Data Request Update消息的流量阈值。 |
| QF时长阈值(分钟) | 该参数用于指定QF级别触发Charging Data Request Update消息的时长阈值。 |
| 最终配额动作指示终结方式 | 该参数用于配置融合计费模板中最终配额动作指示终结方式。 |
| FH处理动作 | 该参数用于设置Tx定时器（TxTimer）超时以及CHF返回某些错误码时的处理动作。 |
| 支持failover使能开关 | 该参数用于配置融合计费模板中是否支持failover功能使能。 |
| failback使能开关 | 该参数用于配置融合计费模板中是否支持自动failback。 |
| Tx定时器时长(秒) | 该参数用于设置向TOPO查询CHF和等待CHF响应消息的时长，当超过该时长，则认为查询CHF或CHF消息响应失败。 |
| 用户保持时长(分钟) | 该参数用于设置CHF故障后用户保持时长，当用户激活无可用CHF，或CHF响应消息超时，或CHF返回PDU级异常结果码(处理动作为FAILOVER)，且FHACTION的处理方式为CONTINUE时，该参数指定了允许用户保持业务的时长。超出该时长则去活用户。当取值为0时，不主动去活用户。 |
| 用户保持调整时长(分钟) | 该参数表示配置的HoldingTime超时后增加一个随机调整范围，在配置的范围内随机选取一个值作为HoldingTime的补充时长。 |
| 融合计费自动恢复功能开关 | 该参数用于设置融合计费用户因CHF故障保持用户在线时，是否支持通过与CHF重建会话恢复配额管理和配额上报。 |
| 融合计费自动恢复间隔 (分钟) | 该参数用于配置融合计费用户尝试与CHF重建会话的时间间隔。 |
| 业务停止时长(分钟) | 该参数用于配置PDU会话业务停止时长，超过该时长后SMF/PGW-C会向CHF发送Charging Data Request Release消息释放会话。 |
| 时间格式 | 该参数用于指定融合计费消息的时间格式。 |
| Update消息触发业务放通结束计费会话开关 | 用于指定融合计费Update消息触发业务放通时，SMF向CHF发送Terminate消息结束计费会话的开关是否使能。 |
| 最大携带的业务容器数量 | 用于指定N40接口消息中最大携带的业务容器数量。当消息中业务容器数量大于等于该值，会强制发送一个N40接口更新消息给CHF，并携带TriggerType为“MANAGEMENT_INTERVENTION”。取值0表示对消息中的业务容器数量不做限制。 |
| RAN-SecondaryRAT-Usage-Report上报CHF的阈值 | 用于指定RAN-SecondaryRAT-Usage-Report上报CHF更新消息的阈值。取值0表示对消息中的RAN-SecondaryRAT-Usage-Report个数不做限制。 |
| 最大可使用的业务个数 | 用于配置用户最多可以同时访问的业务（RG/RG+SID）个数，当用户访问的业务超过该配置时，用户会被去活。取值0表示用户同时访问的业务个数不受本命令限制。 |
| QHT超时触发的容器中Trigger的值 | 该参数用于控制业务QHT超时触发的NCHF消息容器中Trigger的值。 |
| 漫游用户最终配额动作指示终结方式 | 该字段用于配置漫游用户最终配额动作指示终结方式。 |
| 漫游用户的PDU流量阈值(MB) | 该参数用于指定漫游用户的PDU级触发Charging Data Request Update消息的流量阈值。 |
| 会话层异常返回码动作为Block时阻塞免费业务 | 该参数用于配置会话层异常返回码动作为Block时是否阻塞免费业务。 |
| 无配额更新开关 | 该参数用于控制业务无配额（阻塞或重定向）场景，是否允许在接入侧更新或PDU会话级阈值到触发上报时发起配额更新请求。 |
| SCP故障重选开关 | SMF和CHF采用Model C/D通信模式时，SCP故障是否进行SCP重选。 |
| QHT超时触发的N40消息的MUU中是否携带RSU | 该参数用于配置QHT超时触发的N40 ChargingDataUpdateRequest消息中的MultipleUnitUsage是否携带RequestedUnit。 |
