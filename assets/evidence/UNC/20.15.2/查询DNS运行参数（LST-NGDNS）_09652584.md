## [命令功能](#ZH-CN_MMLREF_0209652584)

**适用NF：AMF、SMF**

该命令用于查询DNS域名解析流程相关参数。

## [注意事项](#ZH-CN_MMLREF_0209652584)

无

#### [操作用户权限](#ZH-CN_MMLREF_0209652584)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0209652584)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| MMESWITCH | 优选融合MME开关 | 可选必选说明：可选参数<br>参数含义：该参数用于设置是否打开优选融合MME开关。当打开优选融合MME开关后，如果解析到的IP地址为融合MME，该IP地址会被优先选择。<br>数据来源：全网规划<br>取值范围：<br>- MME_ON（打开）<br>- MME_OFF（关闭）<br>默认值：无<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0209652584)

查询DNS运行参数：

```
LST NGDNS:;
%%LST NGDNS:;%%
RETCODE = 0  操作成功

结果如下
--------------
                   优选融合MME开关  =  打开
         N26接口本网IP地址选择策略  =  仅使用IPv6地址
                      总共发送次数  =  3
                响应超时时长(毫秒)  =  1000
一级Cache等待二级Cache响应时长(秒)  =  3
                        服务器组ID  =  11
                       DNS解析方式  =  本地数据优先
(结果个数 = 1)
```

## [输出结果说明](#ZH-CN_MMLREF_0209652584)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 优选融合MME开关 | 该参数用于设置是否打开优选融合MME开关。当打开优选融合MME开关后，如果解析到的IP地址为融合MME，该IP地址会被优先选择。 |
| N26接口本网IP地址选择策略 | 该参数表示IP地址策略。 |
| 总共发送次数 | 该参数用于设置如果DNS服务器无响应，DNS解析器发送查询请求的最大次数，发送查询次数超过设定后，则本次交互流程失败。 |
| 响应超时时长(毫秒) | 该参数用于设置DNS Client同DNS服务器间一次消息交互链路层等待的时长，DNS Client超过该时间没有收到响应，则本次消息交互失败，随后进行消息重发，重发最大次数参考“总共发送次数”。 |
| 一级Cache等待二级Cache响应时长(秒) | 一级Cache域名解析失败时，会请求二级Cache进行解析。该参数用于设置一级Cache等待二级Cache解析的最大时长，如果在该时间内没有收到二级Cache的响应，则一级Cache认为本次解析失败。 |
| 服务器组ID | 该参数表示二级Cache的DNS服务器组的ID，可通过LST DNSS命令查询获取。 |
| DNS解析方式 | 该参数用于表示DNS解析方式，即控制DNS查询时优先使用服务器数据或者优先使用本地配置数据。 |
