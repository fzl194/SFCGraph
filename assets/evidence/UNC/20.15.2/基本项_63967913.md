#### [81202446 LKV2MSCMG02 MSC Pool内的用户迁移(Gs接口)](#ZH-CN_TOPIC_0000001663967913)

| **适用NF** | SGSN |
| --- | --- |
| **功能描述** | MSC Pool<br>内的用户迁移（Gs接口）特性是Gs接口的一个增强型功能，当MSC Pool中某个MSC不可用时（如发生故障、升级等），<br>SGSN<br>可以下发启动用户迁移命令，将该MSC的状态信息标识为不可用，然后将该MSC上的用户迁移到MSC Pool中的其他MSC上。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 |
| **默认值** | 1 |
| **对应特性** | WSFD-104202 MSC Pool内的用户迁移（Gs接口） |
| **相关控制项** | 依赖WSFD-104407 支持Gs接口 |
| **应用场景** | 在MSC Pool组网场景下，如果某个MSC故障或者升级，会导致在该MSC下的用户的附着和位置更新失败或业务中断。为提升用户的联合附着和联合RA/LA更新成功率并减少业务中断时间，可以启用MSC Pool内的用户迁移（基于Gs口）特性。 |

#### [81203324 LKV2UEMA01 事件统一管理架构](#ZH-CN_TOPIC_0000001663967913)

| 适用NF | ALL |
| --- | --- |
| **功能描述** | 网络日常运行过程中各微服务经常会发生一些状态变化，这些状态的变化会触发一些事件，如N4接口故障等等都可以称为一个事件。<br>在上述事件触发后，微服务通过彼此间的互联能够相互感知对方的事件并作出适当响应来确保系统的稳定，避免有损用户业务的事情发生，即对事件进行管理。<br>UNC支持对上述事件的统一管理，通过对事件的统一管理来确保业务在保微服务系统框架下的平稳运行，从而保证用户业务的可靠性。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 |
| **默认值** | 1 |
| **对应特性** | WSFD-010004 事件统一管理架构 |
| **相关控制项** | 无 |
| **应用场景** | UNC产品采用微服务架构，服务间保持各自独立运行的同时还需要感知彼此的状态来使服务间通信可靠运行。<br>部署本特性后，UNC可以做到对上述事件的主动统一管理，当异常事件触发时，通过订阅事件主动感知并作出相应的措施，把对用户业务的影响降到最小。 |

#### [81203321 LKV2SSBA01 支持SBA架构](#ZH-CN_TOPIC_0000001663967913)

| 适用NF | SGSN、GGSN-C、MME、SGW-C/PGW-C、AMF、SMF、NRF、NSSF、SMSF、NCG |
| --- | --- |
| **功能描述** | 为了适配未来垂直行业多样化以及不确定的要求，5G网络架构进行了变革，将控制面功能抽象成为多个独立的网络服务，以软件化、模块化、服务化的方式来构建网络。每个网络服务和其他服务在业务功能上解耦，并且对外提供服务化接口，可以通过相同的接口向其它调用者提供服务，将多个耦合接口转变为单一服务接口，从而减少了接口数量。这种架构即是SBA（Service Based Architecture），基于服务的架构。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 |
| **默认值** | 1 |
| **对应特性** | WSFD-010000 支持SBA架构 |
| **相关控制项** | 无 |
| **应用场景** | SBA网络架构是5GC网络的必选架构。 |
