# MEC计费流程

- [网络架构](#ZH-CN_TOPIC_0252284255__1.3.3.1)
- [计费流程](#ZH-CN_TOPIC_0252284255__1.3.4.1)

随着物联网、人工智能、云计算、移动互联网、大数据和大视频等产业技术的蓬勃发展，以及围绕ICT开放生态的成熟，网络资源和计算能力逐步朝资源集中化和边缘化演进。

多址边缘计算MEC（Multi-access Edge Computing）为典型的资源边缘化模式，在移动网络边缘提供IT服务环境和云计算能力，实时完成移动网络边缘的业务处理。MEC将随着CT和IT深度融合趋势，物联网的兴起、人工智能技术的发展，以及企业对生产数据的安全性、实时性的诉求，持续快速的发展。

#### [网络架构](#ZH-CN_TOPIC_0252284255)

分流是MEC解决方案的基础，支撑着整个解决方案。3GPP针对于5G用户面的数据分流定义了UL CL（Uplink Classifier）功能，即上行分流器。UL CL是对上行业务数据分流，并对分流后的下行数据聚合的一个处理节点，如 [图1](#ZH-CN_TOPIC_0252284255__fig12510210191419) 所示。

**图1** UL CL架构图

<br>

![](MEC计费流程_52284255.assets/zh-cn_image_0252313116_2.png)

- 用户激活阶段，SMF将计费规则通过N4接口分别下发到主锚点（PSA1）和辅锚点（PSA2）。
- 用户访问业务阶段，主/辅锚点均可以对经过PSA的数据申请配额、收集业务使用量，并上报到SMF，SMF按UPF向CHF申请配额。即CHF对每个锚点UPF进行配额管理，每个UPF独享配额。
- UL CL UPF没有计费规则，不计费。通常情况下UL CL UPF与辅锚点合设。

#### [计费流程](#ZH-CN_TOPIC_0252284255)

**图2** MEC场景计费流程

<br>

![](MEC计费流程_52284255.assets/zh-cn_image_0252340528_2.png)

1~7：UE完成PDU会话建立流程，并获取计费规则，将计费规则下发到PSA1 UPF。若会话建立阶段，SMF已经申请了配额，则SMF会将计费规则和配额一同下发至PSA1 UPF。

P1：SMF根据UE激活的DNN和UE位置，选择UL CL+PSA2 UPF插入用户会话。

8~9：SMF将计费规则下发到UL CL+PSA2 UPF。

P2：当上行业务报文经过UL CL+PSA2 UPF时，报文按分流规则匹配，匹配上的报文访问本地DN。

10~11：UL CL+PSA2 UPF向SMF发送PFCP Session Report Request消息，申请配额。

12~13：SMF按UPF向CHF申请配额（消息中Multiple Unit Usage字段携带UPF ID），哪个UPF申请就向哪个UPF下发配额。后续计费流程与标准计费流程相同，可参见 [标准计费流程（未部署NCG场景）](../../../../../业务专题/5G Core 计费解决方案/计费解决方案概述/计费原理/N40接口融合计费/融合计费原理/标准计费流程（未部署NCG场景）_56728529.md) 。

P3：当上行业务报文经过UL CL+PSA2 UPF时，报文按分流规则匹配，未匹配上的报文分流至PSA1 UPF，访问Internet。

14~17：流程与10~13相同，PSA1 UPF申请配额，后续计费流程可参见 [标准计费流程（未部署NCG场景）](../../../../../业务专题/5G Core 计费解决方案/计费解决方案概述/计费原理/N40接口融合计费/融合计费原理/标准计费流程（未部署NCG场景）_56728529.md) 。
