#### [业务场景](#ZH-CN_CONCEPT_0281112294)

当空闲态UE从5GS移动到EPS覆盖区域时，UE执行TAU或初始注册，TAU过程就是将从5G的传输通道（红线）切换到EPS传输通道（蓝线）的过程，如 [图1](#ZH-CN_CONCEPT_0281112294__zh-cn_concept_0276784341_fig0724122217166) 所示。

**图1** N26接口5G到4G的空闲态移动示意图

<br>

![](基于N26接口的5GS到EPS的重选_81112294.assets/zh-cn_image_0276784625_2.png)

**当空闲态UE从5GC移动到EPC时，UE执行跟踪区更新或初始注册，判断条件如下（3GPP 23.502协议中描述）：**

- 在如下情况下，UE都是执行跟踪区更新流程：
  UE至少有过一个PDU会话，在互通过程中支持会话连续性，即UE有EPS Bearer ID和映射的EPS QoS参数。

- 在满足如下条件的情况下，UE发起初始注册流程：
  UE在5GC中只注册没有PDU会话，或者UE在5GC注册后只有在EPC互通时不支持会话连续性的PDU会话。

**为支持5GS和EPS间基于N26的互操作，在PDU会话建立/GBR QoS Flow建立时，5G核心网做如下相关操作：**

- QoS映射及TFT分配：如果部署了PCC，SMF+PGW-C将从PCF获取的5G QoS参数映射成EPS QoS映射，并根据获取的PCC规则分配TFT；否则，SMF+PGW-C在本地映射EPS QoS和分配TFT。SMF+PGW-C忽略不适合EPC的5G QoS参数（例如，QoS Notification control）。
- 分配EPS Bearer ID： SMF需要根据QoS Profile和运营商策略确定哪些QoS Flow需要分配EPS Bearer ID。当SMF判断需要为QoS流分配EPS Bearer ID（基于Dnninfo中的iwkEpsInd标志位，即UE在此DNN中是否签约了Interworking）时，由SMF请求服务AMF分配EPS Bearer ID。针对每个PDU会话，EPS承载ID分配给缺省EPS承载和专用EPS承载。
- UE、RAN、SMF+PGW-C都存储QoS flow和EBI、EPS QoS参数的关联关系。如果QoS flow删除了，UE、RAN、SMF+PGW-C也会删除与这个QoSflow关联的EPS QoS参数。

#### [信令流程](#ZH-CN_CONCEPT_0281112294)

UE注册到5G网络后，空闲态下移动到EPS的覆盖区进行的TAU流程如 [图2](#ZH-CN_CONCEPT_0281112294__zh-cn_concept_0276784341_fig1498644216491) 所示。

- [图2](#ZH-CN_CONCEPT_0281112294__zh-cn_concept_0276784341_fig1498644216491)以AMF与MME之间的N26接口独立部署为例，5G到4G的重选流程，MME使用4G-GUTI构造域名执行DNS查询获取AMF的IP地址（步骤3）。
- 如果是在MME的S10接口和AMF的N26接口IP地址合一的场景下，5G到4G的重选流程，MME可以直接使用4G-GUTI识别是否本设备AMF分配的GUTI，如果是，则识别为UAM内部的4/5G互操作；如果不是，则需通过DNS查找对端IP地址（步骤3）。

    10. SGW-C向PGW-C发送Modify Bearer Request消息，通知SGW-C的S5/S8-C接口F-TEID，以及SGW-U S5/S8-U接口的F-TEID。SMF+PGW-C根据PCF+PCRF之前下发的Policy Control Request Trigger，向PCF+PCRF上报RAT变化、UE位置变化等事件。PCF+PCRF下发更新后的策略。更新后的策略如果需要触发EPS Bearer操作，如承载激活、修改、删除等，在[19](#ZH-CN_CONCEPT_0281112294__zh-cn_concept_0276784341_li4797528165017)中执行。

    12. PGW-C向SGW-C返回Modify Bearer Response消息，该消息中主要包含各EPS Bearer的Charging ID信息。
    13. SGW-C向MME返回Create Session Response消息，其中包含了每个EPS Bearer的SGW-U S1-U接口F-TEID信息。
    14. MME向UDM+HSS发送Update Location Request消息，该消息中双注册标识被置0或不携带。
    15. UDM+HSS调用Nudm_UECM_DeregistrationNotification service operation，其中的DeregistrationData的DeregistrationReason为“5GS_TO_EPS_MOBILITY”。

    16. UDM+HSS向MME返回Update Location Ack消息。
    17. MME向UE发送TAU Accept消息，如果TAU Request消息中的Active Flag置位，则TAU Accept消息被携带在S1AP-MME接口的Initial Context Setup Request消息中发送给eNodeB。Initial Context Setup Request消息中包含了待建立的E-RAB信息。
    18. 如果新分配了4G-GUTI，UE向MME返回TAU Complete消息。
    19. **可选：**PCF+PCRF可能因为RAT变化发起专有承载建立/修改/删除流程，通过发起这些流程将相关修改同步给UE。
