#### [命令功能](#ZH-CN_MMLREF_0000001172345497)

**适用网元：SGSN、MME** **、AMF**

该命令用于增加一个DNS域名解析服务器。DNS服务器是网络中专门提供域名解析服务的服务器。

域名解析在4G系统中通常用于附着时根据APN解析P-GW地址和S-GW地址，Inter TAU时根据GUTI解析对端MME IP地址，Handover时根据TAI解析对端MME IP地址。

域名解析顺序：首先查找主机上HOSTFILE中的记录（通过 [**ADD IPV4DNSH**](../DNS Hostfile管理/增加IPV4 DNS Hostfile记录(ADD IPV4DNSH)_26145884.md) 添加）。如果没有相应的记录，再查找DNS Cache中的记录。如果没有相应的记录，再向DNS服务器发送解析请求进行解析。

#### [注意事项](#ZH-CN_MMLREF_0000001172345497)

- 本表最大记录数为38条。
- 该命令执行后立即生效。
- 该表最多可以配置38个服务器组，每个服务器组中最多可以配置6个不同的DNS服务器。不同的服务器组中可以配置相同的DNS服务器，整系统最多可以配置16个IP地址不同的DNS服务器。
- 业务进行DNS查询时，如果根据[**ADD DNSQ**](../DNS查询管理/增加DNS查询控制参数(ADD DNSQ)_26305702.md)配置的匹配原则找不到对应的服务器组，则会默认使用服务器组0进行查询，如果服务器组0未配置服务器，则会导致查询失败。
- 当不同服务器组中配置相同的DNS服务器时，服务器承载协议必须一致。但不建议在不同的服务器组下配置相同的DNS服务器，会对性能产生影响。
- 只有配置了DNS本端实体（通过[**ADD DNSLE**](../DNS本端实体管理/增加DNS实体绑定DNS Client IP(ADD DNSLE)_72225567.md)添加）和DNS服务器地址（通过[**ADD DNSS**](增加DNS服务器(ADD DNSS)_72345497.md)添加）之后，DNS服务器才能起作用。
- 对于一个组内优先级相同的DNS服务器，UNC按照轮选的方式进行选择。如果要使用DNS服务器的Round Robin选择功能，那么不允许将同一个DNS服务器配置到多个组内。

#### [本地用户权限](#ZH-CN_MMLREF_0000001172345497)

manage-ug；system-ug

#### [网管用户权限](#ZH-CN_MMLREF_0000001172345497)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_MMLREF_0000001172345497)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| GRPID | 服务器组ID | 可选必选说明：可选参数<br>参数含义：该参数用来指定DNS服务器所属的服务器组。进行DNS解析时，使用指定组内的DNS服务器。<br>数据来源：整网规划<br>取值范围：0~37<br>默认值：0 |
| IPT | IP地址类型 | 可选必选说明：可选参数<br>参数含义：IP地址类型。<br>数据来源：整网规划<br>取值范围：<br>- IPV4(IPV4)<br>- IPV6(IPV6)<br>默认值：IPV4(IPV4) |
| IP | IP地址 | 可选必选说明：条件必选参数<br>参数含义：该参数用来指定DNS服务器的IP地址。<br>该参数在"IP地址类型"参数配置为"IPV4(IPV4)"后生效。<br>数据来源：整网规划<br>取值范围：0.0.0.0~255.255.255.255<br>默认值：无<br>配置原则：<br>- IPv4地址不能为0.0.0.0。 |
| IPV6 | IPv6地址 | 可选必选说明：条件必选参数<br>参数含义：DNS服务器IPv6地址<br>该参数在"IP地址类型"参数配置为"IPV6(IPV6)"后生效。<br>数据来源：整网规划<br>取值范围：::～FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF<br>默认值：无<br>配置原则：IPv6地址必须是全球单播地址，不能为::、FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF、环回地址(::1)、链路本地地址(FE80::/10)和组播地址(FF00::/8)。 |
| PRI | 域名服务器优先级 | 可选必选说明：必选参数<br>参数含义：该参数用来指定DNS服务器的优先级。<br>数据来源：整网规划<br>取值范围：<br>- “PRI1(优先级1)”<br>- “PRI2(优先级2)”<br>- “PRI3(优先级3)”<br>- “PRI4(优先级4)”<br>- “PRI5(优先级5)”<br>- “PRI6(优先级6)”<br>默认值：无<br>配置原则：优先级从<br>“PRI1(优先级1)”<br>到<br>“PRI6(优先级6)”<br>依次递减。一个DNS服务器组内，可配的DNS服务器数量最多6台。同一个组内，不同的DNS服务器可以配置为相同优先级，也可以配置为不同的优先级。相同的DNS服务器采用轮选的方式负荷分担。<br>说明：同一服务器组内配置多个DNS服务器时，优先级高的优先使用。当高优先级的服务器都故障时，使用次优的服务器。 |
| DNSSNAME | DNS服务器名称 | 可选必选说明：可选参数<br>参数含义：该参数用来指定DNS服务器名称。<br>数据来源：整网规划<br>取值范围：0~32个字符<br>默认值：无 |
| PRO | DNS服务器承载协议 | 可选必选说明：可选参数<br>参数含义：该参数用来指定DNS服务器的承载协议，协议可以是TCP或者UDP。<br>数据来源：整网规划<br>- “TCP(TCP传输模式)”：表示仅支持TCP协议。<br>- “UDP_TCP(UDP_TCP传输模式)”：表示既支持UDP又支持TCP协议。<br>- “UDP(UDP传输模式)”：表示仅支持UDP协议。<br>默认值：<br>“UDP_TCP(UDP_TCP传输模式)”<br>配置原则：<br>- “DNS服务器承载协议”建议值为“UDP_TCP(UDP_TCP传输模式)”，即同时创建UDP、TCP两种类型的链路，优先选择UDP链路，当报文较长时（如响应中的IP个数超过28个时），切换为TCP链路。<br>- 系统如果配置了NAPTR类型的查询，该类型查询的响应消息一般比较长。此时建议配置为“TCP(TCP传输模式)”或者“UDP_TCP(UDP_TCP传输模式)”。 |

#### [使用实例](#ZH-CN_MMLREF_0000001172345497)

增加一个DNS服务器，属于ID为1的服务器组，IP地址类型为IPV4，IP地址为192.168.100.101，优先级为1，使用TCP承载协议：

ADD DNSS: GRPID=1, IPT=IPV4, IP="192.168.100.101", PRI=PRI1, PRO=TCP;
