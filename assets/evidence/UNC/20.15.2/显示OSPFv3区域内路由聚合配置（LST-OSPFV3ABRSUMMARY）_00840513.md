#### [命令功能](#ZH-CN_CONCEPT_0000001600840513)

该命令用于查询OSPFv3区域内路由聚合。

#### [注意事项](#ZH-CN_CONCEPT_0000001600840513)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001600840513)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001600840513)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROCID | OSPFv3进程号 | 可选必选说明：可选参数<br>参数含义：OSPFv3进程号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～4294967295。<br>默认值：无<br>配置原则：OSPFv3进程必须已经存在。 |
| AREAID | OSPFv3区域ID | 可选必选说明：可选参数<br>参数含义：OSPFv3区域ID。<br>数据来源：全网规划<br>取值范围：IPv4地址类型。点分十进制格式。<br>默认值：无 |
| IPADDRESS | IPv6地址 | 可选必选说明：可选参数<br>参数含义：IPv6地址。<br>数据来源：本端规划<br>取值范围：IPv6地址类型。<br>默认值：无 |
| IPV6MASKLEN | IPv6前缀长度 | 可选必选说明：可选参数<br>参数含义：IPv6前缀长度。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～128。<br>默认值：无 |

#### [使用实例](#ZH-CN_CONCEPT_0000001600840513)

显示OSPFv3区域内路由聚合配置：

```
LST OSPFV3ABRSUMMARY:;
```

```

RETCODE = 0  操作成功。

结果如下
-------------------
      OSPFv3进程号  =  1
      OSPFv3区域ID  =  0.0.0.1
          拓扑标识  =  0

      IPv6前缀长度  =  64
      发布路由标识  =  TRUE
      使能开销配置  =  TRUE
    聚合路由的开销  =  100
(结果个数 = 1)
---    END
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000001600840513)

| 输出项名称 | 输出项解释 |
| --- | --- |
| OSPFv3进程号 | 该参数指定用户创建的OSPFOSPFv3进程号。 |
| OSPFv3区域ID | 该参数指定用户创建的OSPFOSPFv3区域号。 |
| 拓扑标识 | 该参数指定用户创建的OSPFv3拓扑标识。 |
| IPv6地址 | 该参数指定聚合路由的IPv6地址。 |
| IPv6前缀长度 | 该参数指定聚合路由的IPv6前缀长度。 |
| 发布路由标识 | 该参数指定是否发布这条聚合路由。缺省时发布聚合路由。 |
| 使能开销配置 | 该参数指定是否配置聚合路由开销值。 |
| 聚合路由的开销 | 该参数用于配置聚合路由的开销，当此参数缺省时，则取所有被聚合的路由中最大的那个开销值作为聚合路由的开销，仅当使能开销配置被配置为true时生效。 |
