# 过程3：签约数据匹配/再匹配成功后再处理

在请求信息纠正完成且再匹配成功后， UNC 会对匹配成功的签约数据进行进一步的处理，包括野卡策略选择、别名APN、以及DNS解析失败后纠错。

签约数据匹配/再匹配成功后再处理流程说明如下：

1. 查看用户匹配的PDN Type是否是IPv4v6双栈。
    - 是 => 执行[步骤2](#ZH-CN_TOPIC_0186228995__li14857185194017)。
    - 否 => 执行[步骤3](#ZH-CN_TOPIC_0186228995__li14857185194017)。
2. 查看UNC双栈功能是否打开。
    - 是 => 执行[步骤3](#ZH-CN_TOPIC_0186228995__li14857185194017)。
    - 否 => 按照MML命令**[**SET SMFUNC**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/SM扩展功能管理/设置会话管理扩展功能(SET SMFUNC)_26145684.md)**中参数“PDP/PDN Type策略”的配置策略修改PDN/PDP Type，执行[步骤3](#ZH-CN_TOPIC_0186228995__li14857185194017)。
3. 查看匹配的签约数据是否为野卡签约数据。
    - 是 => 执行[步骤4](#ZH-CN_TOPIC_0186228995__li158573504019)。
    - 否 => 执行[步骤5](#ZH-CN_TOPIC_0186228995__step4)。
4. 根据[野卡签约数据策略](#ZH-CN_TOPIC_0186228995__xing)进行处理。
    - 拒绝=> 拒绝PDP/PDN激活请求。
    - 标准=> 执行标准的野卡签约数据处理，然后执行[步骤5](#ZH-CN_TOPIC_0186228995__step4)。
    - 定制=> 执行定制的野卡签约数据处理，然后执行[步骤5](#ZH-CN_TOPIC_0186228995__step4)。
5. 执行别名APN转换[别名APN转换](#ZH-CN_TOPIC_0186228995__alian)。
6. 执行[DNS解析失败后APN纠错](#ZH-CN_TOPIC_0186228995__dns)。

**野卡签约数据策略**

野卡签约数据策略分为拒绝、标准、定制3种，可以通过MML命令 [**ADD SMACTCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/激活过程管理/增加激活过程控制参数（ADD SMACTCTRL）_26305472.md) 根据需要进行选择。下面将分别介绍三种策略。

- 拒绝
  如果野卡签约数据策略为拒绝，则拒绝UE的PDP/PDN激活请求。
- 标准
  查看UE的请求APN是否为空，如果不为空，则使用UE的请求APN。如果为空，则使用 UNC 配置的Local APN。如果 UNC 未配置Local APN，则拒绝UE的PDP/PDN激活请求。
- 定制
  如果野卡签约数据为定制， UNC 根据定制策略对野卡签约数据进行处理。运营商可以从如下几个方面定制野卡签约数据策略：

  - “请求APNNI的处理策略”：
          - “STANDARD_MODEL(标准方式)”：直接使用用户请求的APN。
          - “IGNORE(直接忽略)”：忽略用户请求的APN。即，如果用户请求消息中携带了APN，将APN设置为空。
          - “IGNORE_DNS_FAIL(DNS解析失败后忽略)”：在解析失败后忽略请求APN，采用Local APN重新进行解析。
      > **说明**
      > 如果某些类型终端可以在签约数据匹配失败或DNS解析失败后，再次请求接入时自动纠正并携带正确的APN，建议此处选择“标准方式”的处理策略。
    - “优选非野卡签约数据”：在存在多组签约数据且存在非野卡签约数据的场景下，是否优选非野卡签约数据。如果优选非野卡，则在多组签约数据中选择Default APN签约数据、Local APN对应的签约数据或Context ID最小的签约数据
    - “只允许使用公共APNNI”：是否野卡只允许使用公共APNNI。如果只允许使用公共APNNI，UNC查看纠正后的APN是否是命令[**ADD PUBLICAPNNI**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/公共APNNI管理/增加公共APN NI配置(ADD PUBLICAPNNI)_26145682.md)配置的公共APN。如果是，UNC允许该野卡接入网络。如果不是，则拒绝UE的PDP/PDN激活请求。
    - “限制野卡激活数目”：考虑是否限制用户的使用野卡的PDP/PDN激活数目，以保证非野卡用户的成功接入。如果限制野卡激活数目，则在某用户使用野卡激活PDP/PDN的数目超过一定门限时，拒绝该用户后续PDP/PDN使用野卡签约数据。

别名APN纠错的具体流程说明如下：

1. 查看匹配的APN是否需要进行别名APN转换。
  > **说明**
  > 哪些APN需要进行别名APN转换是通过命令 [**ADD ALIASAPN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/别名APN管理/增加别名APN配置(ADD ALIASAPN)_72345281.md) 配置的。关于别名APN的详细信息，请参见 [WSFD-106203 别名APN](../../../WSFD-106203 别名APN_64082528.md) 。
    - 是 => 替换匹配的APN NI为[**ADD ALIASAPN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/别名APN管理/增加别名APN配置(ADD ALIASAPN)_72345281.md)配置的转换APNNI，执行[步骤2](#ZH-CN_TOPIC_0186228995__step2_2)。
    - 否 => 执行[步骤2](#ZH-CN_TOPIC_0186228995__step2_2)。
2. 是否根据IMSI和APN NI选择GGSN/P-GW路由。
    - 是 => 执行根据IMSI和APN NI选择GGSN/P-GW路由流程。
    - 否 => 执行[步骤3](#ZH-CN_TOPIC_0186228995__li9239420142710)。
  > **说明**
  > 有关根据IMSI和APN NI选择GGSN/P-GW路由的详细信息，请参见 “ [WSFD-205001 网关路由选择功能](../../../../网关选择功能/WSFD-205001 网关路由选择功能_63535123.md) ”。
3. 选择相对应的APN OI，然后将完整的APN（APN NI+APN OI）发往DNS服务器，进行DNS解析。
4. 收到DNS响应消息。
5. 查看DNS解析是否成功？
    - 是 => 请求信息纠正整体流程结束
    - 否 => 进行[DNS解析失败后APN纠错](#ZH-CN_TOPIC_0186228995__dns)。

DNS解析失败后APN纠错的具体流程说明如下：

1. DNS解释失败后，UNC查看APN是不是UE激活请求中携带的APN并且允许DNS解析失败后进行纠正。
    - 是 => 执行[步骤2](#ZH-CN_TOPIC_0186228995__step3_2)。
    - 否 => 拒绝PDP/PDN激活请求。
2. 查看UNC是否配置了Local APN。
    - 是 => 执行[步骤3](#ZH-CN_TOPIC_0186228995__step3_3)。
    - 否 => 拒绝PDP/PDN激活请求。
3. 查看Local APN是否和上次使用的APN相同。
    - 是 => 拒绝PDP/PDN激活请求。
    - 否 => 使用Local APN，重新选择APN OI，执行[步骤4](#ZH-CN_TOPIC_0186228995__li7970349114814)。
4. 使用Local APN重新选择APN OI。
5. 查看选择APN OI是否成功。
    - 是 => 进行DNS解析，请求信息纠正整体流程结束。
    - 否 => 拒绝PDP/PDN激活请求。
