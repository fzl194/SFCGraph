# 调测分区域网络运维

- [操作场景](#ZH-CN_OPI_0166339472__1.3.1)
- [必备事项](#ZH-CN_OPI_0166339472__1.3.2)

## [操作场景](#ZH-CN_OPI_0166339472)

验证 UNC 对TAI组性能对象进行统计情况，确保该功能生效。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0166339472)

前提条件

- 请仔细阅读[WSFD-011302 分区域网络运维特性概述](特性概述_66339468.md)。
- 完成[激活分区域网络运维](激活分区域网络运维_66339471.md)。

数据

该操作无需准备数据。

工具

OM Portal跟踪

## [操作步骤](#ZH-CN_OPI_0166339472)

1. UNC 与eNodeB对接完成，eNodeB上报
2. 查询系统中的TAI信息。
  进入 “MML命令行-UNC” 窗口。
  [**LST S1PAGING**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1接口寻呼数据/查询S1接口寻呼数据(LST S1PAGING)_72345841.md) : SERVICETYPE="服务类型";
  预期结果：输出报文中可以得到TAI信息（Tracking area ID），如下报文所示。
  ```
  %%LST S1PAGING: SERVICETYPE="USN";%%
  RETCODE = 0  操作成功

  操作结果如下：
  --------------------------------------------------------------------------------
  RU名称            进程号  移动国家码_移动网号  eNodeB类型    eNodeB标识   Tracking Area ID 

  USN_SP_RU_0065     2       12303               MACRO_ENB       270593     123034210       
  USN_SP_RU_0066     1       12303               MACRO_ENB       270353     123034201       
  (结果个数 = 2) 
  ```
3. 如果此功能生效，系统会将TAI作为TAI组（以此TAI值为名称）进行性能统计，所以根据上一步的输出结果，查询“TAI组名”为“TAIGP1”的TAI组信息。
  以查询“TAIGP1”为例， 进入 “MML命令行-UNC” 窗口。
  [**LST PERFOBJ**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/性能管理/性能对象管理/查询性能对象信息(LST PERFOBJ)_26306000.md) : MOC=TAIGP;
  预期结果：输出报文中存在以“TAIGP1”为“TAI组名”的TAI组信息，并且“TAI组类型”为“手动配置TAI组”。如下报文所示。
  ```
  %%LST PERFOBJ: MOC=TAIGP;%%
  RETCODE = 0  操作成功。

  操作结果如下
  -------------
  测量对象类型    TAI组名      TAI组类型       TAI组索引

  TAI组           TAIGP1   手动配置TAI组      1
  (结果个数 = 1)
  ---    END
  ```
