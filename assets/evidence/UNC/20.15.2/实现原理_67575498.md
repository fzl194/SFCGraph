S6a/S6d接口固定速率流控功能整体流程描述如下：

1. 系统默认开启固定速率流控开关为开，初始流控速率为50个/秒。系统平稳运行场景下，一般不会造成S6a/S6d接口拥塞或HSS/DRA过载。
2. 判断是否需要进行预先流控。
  在系统重启、重大节日或大型活动时，大量用户进行重新附着或系统间切换，需要进行预先流控。
    - 是 =>[3](#ZH-CN_CONCEPT_0167575498__step3)
    - 否 =>[4](#ZH-CN_CONCEPT_0167575498__step4)
3. 计算并设置固定流控速率。
  计算方法参考 [固定速率计算](#ZH-CN_CONCEPT_0167575498__section652454418119) 。 **[**SET FIXEDFC**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/固定速率流控管理/设置固定速率流控信息(SET FIXEDFC)_26146142.md)** 执行后立即生效。
4. 观察性能指标判断是否有突发需流控的情况。
  观察性能指标15分钟后，如果S6a/S6d接口负荷过高，HSS/DRA长时间拥塞，接入成功率极低为5%以下，可以适当调低接入速率。
    - 是 =>[3](#ZH-CN_CONCEPT_0167575498__step3)
    - 否 =>[5](#ZH-CN_CONCEPT_0167575498__step5)
5. 按照重新设置的流控速率控制Attach、Inter TAU/RAU、Intra TAU(GERAN/UTRAN->LTE)、Intra RAU(LTE->GERAN/UTRAN)流程首次到HSS/DRA的取鉴权集请求或位置更新请求的速率。

#### [固定速率计算](#ZH-CN_CONCEPT_0167575498)

S6a/S6d接口固定速率流控功能开启前要计算固定流控速率值。通过 **[**SET FIXEDFC**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/操作维护/设备管理/流控管理/业务流控管理/固定速率流控管理/设置固定速率流控信息(SET FIXEDFC)_26146142.md)** 设置固定流控速率，固定流控速率如下两种计算方法：

- **按实际用户数计算固定流控速率**
  S6a/S6d接口固定速率流控时，流控速率一般是基于如下目标来设置的：能够使系统的所有用户在期望的接入时长内全部接入，因此推荐流控速率按照“固定流控速率（个/秒）= **实际用户数/接入时长** ”值进行设置。
  一般可使用1800秒作为接入 **时长** 。
  **实际用户数** 取值如下：
  在 “MML命令行-UNC” 窗口上执行： **[**LST IMSIHLRHSS**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/签约数据管理/接口选择/查询IMSI对应的HLR_HSS接口（LST IMSIHLRHSS）_72345353.md)** ;查看GERAN/UTRAN参数是否配置了S6d。

  - 如果GERAN/UTRAN参数配置了S6d，那么 **实际用户数** =2G在线用户数+3G在线用户数+4G在线用户数。
    - 如果GERAN/UTRAN参数没有配置S6d，那么 **实际用户数** =4G在线用户数。
  在 “MML命令行-UNC” 窗口上执行： **[DSP COMUSRPDPNUM](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/融合接入业务管理/融合用户数据库管理/显示移动性管理用户数（DSP COMUSRPDPNUM）_55639265.md)** ;从返回的MML报文中获取“2G在线用户数”、“3G在线用户数”和“4G在线用户数”的值。
  > **说明**
  > “2G在线用户数”、“3G在线用户数”和“4G在线用户数”的值也可以通过性能指标获取。
  例如系统当前实际用户数为80万，需要在1800秒内接入，故“固定流控速率（个/秒）”为：800,000个 / 1800秒 = 445(个/秒)。

- **按HSS处理能力计算固定流控速率**
  已知HSS/DRA的处理能力时，可以通过进一步估算设置更合理的流控速率。

#### [业务流程](#ZH-CN_CONCEPT_0167575498)

本功能通过手动配置S6a/S6d接口的发送请求阈值，控制Attach、Inter TAU/RAU、Intra TAU(GERAN/UTRAN->LTE)、Intra RAU(LTE->GERAN/UTRAN)流程首次到HSS/DRA的取鉴权集请求或位置更新请求的速率，防止S6a/S6d接口拥塞或HSS/DRA过载，以确保整网接入成功率。

如下详细对比S6a/S6d接口固定速率流控功能关闭和开启两种情况下消息交互流程，从中可以看到S6a/S6d接口固定速率流控功能关闭情况下，只有少部分可以接入成功，而S6a/S6d接口固定速率流控功能开启情况下，大部分用户接入成功，明显提高用户接入网络成功率。

- **未进行S6a/S6d接口固定速率流控消息交互流程**

不开启S6a/S6d接口固定速率流控功能时，大量用户重新附着或系统间切换时，会造成S6a/S6d接口拥塞或HSS/DRA过载，仅少量用户能够成功接入。

未进行S6a/S6d接口固定速率流控消息交互流程如 [图2 未进行S6a/S6d接口固定速率流控消息交互流程图](#ZH-CN_CONCEPT_0167575498__fig5) 所示。

**图2** 未进行S6a/S6d接口固定速率流控消息交互流程图

<br>

![](实现原理_67575498.assets/zh-cn_image_0170994456_2.png)

未进行S6a/S6d接口固定速率流控消息交互流程描述如下：

1. 系统重启、重大节日、大型活动等场景下，大量用户重新附着时，发送Attach Request或TAU/RAU Request消息。
2. UNC向HSS/DRA发送取鉴权集请求（Authentication Information Request）。
3. **场景一：突发大量到HSS的取数据操作，致使S6a/S6d接口链路拥塞、HSS/DRA过载，无AIA应答。**
    a. 由于S6a/S6d接口链路拥塞、HSS/DRA过载，HSS/DRA未对取鉴权集请求做出AIA应答。
    b. UNC 向UE发送Attach/TAU/RAU Reject携带“#111（Protocol error unspecified）”原因值。
4. **场景二：突发大量到HSS的取数据操作，致使S6a/S6d接口链路拥塞、HSS/DRA过载，无ULA应答。**
    a. HSS接收到消息后，HSS向 UNC 返回Authentication Information Answer消息。
    b. UNC 向HSS/DRA发送位置更新请求（Update Location Request）。
    c. 由于S6a/S6d接口链路拥塞、HSS/DRA过载，HSS/DRA未对位置更新请求做出ULA应答。
    d. UNC 向UE发送Attach/TAU/RAU Reject携带“#111（Protocol error unspecified）”原因值。
5. **场景三：少量用户成功接入。**
    a. HSS接收到消息后，HSS向 UNC 返回Authentication Information Answer消息。
    b. UNC 向HSS/DRA发送位置更新请求（Update Location Request）。
    c. HSS/DRA响应Update Location Answer消息。
    d. Attach、TAU/RAU接受，用户接入成功。

- **进行S6a/S6d接口固定速率流控后消息流程**

在系统重启、重大节日、大型活动等特殊时间段有大量到HSS/DRA的AIR、ULR消息，需要进行S6a/S6d接口固定速率流控，使得AIR、ULR消息按照设定的固定流控速率接入，防止S6a/S6d接口拥塞或HSS/DRA过载，大部分用户能够成功接入。

进行S6a/S6d接口固定速率流控后消息流程如 [图3 进行S6a/S6d接口固定速率流控后消息流程图](#ZH-CN_CONCEPT_0167575498__fig4) 所示。

**图3** 进行S6a/S6d接口固定速率流控后消息流程图

<br>

![](实现原理_67575498.assets/zh-cn_image_0170994455_2.png)

进行S6a/S6d接口固定速率流控后消息流程描述如下：

1. 大量用户重新附着或系统间切换时，发送Attach Request或TAU/RAU Request消息。
2. **场景一：被流控限制接入的用户。**
  由于已经设置固定流控速率，当用户附着或系统间切换时，微服务USN控制Attach、Inter TAU/RAU、Intra TAU(GERAN/UTRAN->LTE)、Intra RAU(LTE->GERAN/UTRAN)流程请求消息首次到HSS/DRA的取鉴权集请求或位置更新请求的速率。
  超过新的流控速率的请求直接被微服务USN拒绝，UE接收到微服务USN返回的带有“#22 Congestion”原因值的消息，该原因值可通过 [**SET EMMPROCTRL**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/MM流程管理/S1模式MM流程控制参数/设置S1模式移动性管理流程控制参数(SET EMMPROCTRL)_72225199.md) 配置。
  UE接收到带有“#22 Congestion”原因值的消息，会继续在网络多次尝试，待重新接入。
3. **场景二：S6a/S6d接口固定速率流控，大部分用户成功接入。**
    a. UNC 向HSS/DRA发送取鉴权集请求（Authentication Information Request）。
    b. HSS接收到消息后，HSS向 UNC 返回Authentication Information Answer消息。
    c. UNC 向HSS/DRA发送位置更新请求（Update Location Request）。
    d. HSS/DRA响应Update Location Answer消息。
    e. Attach、TAU/RAU接受，用户接入成功。

> **说明**
> - S6a/S6d接口固定速率流控功能开启时，如果部分用户连续附着或系统间切换失败，且软参Byte_Ex21 Bit7取缺省值0， 会出现过多purge消息影响流控功能，那么需要打开此软参关闭Purge消息下发，待用户都接入后再关闭软参。
