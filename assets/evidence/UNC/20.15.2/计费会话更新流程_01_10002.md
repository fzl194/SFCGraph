# 计费会话更新流程

- [业务场景](#ZH-CN_TOPIC_0278888684__1.3.1.1)
- [信令流程](#ZH-CN_TOPIC_0278888684__1.3.2.1)

#### [业务场景](#ZH-CN_TOPIC_0278888684)

- 内部事件UPF触发：配额耗尽或剩余配额量达到阈值、配额空闲时间门限值达到、访问无配额的新业务等。
- 内部事件SMF触发：在线配额有效时长超期。
- 外部事件：接入条件更新。
- CHF发起重授权：比如用户重新充值成功，CHF通知SMF用户重新申请配额。

#### [信令流程](#ZH-CN_TOPIC_0278888684)

融合计费会话更新触发场景有如下几种：

- [内部事件UPF触发融合计费会话更新](#ZH-CN_TOPIC_0278888684__p7644163952011)
- [内部事件SMF触发融合计费会话更新](#ZH-CN_TOPIC_0278888684__p12413522193418)
- [外部事件触发融合计费会话更新](#ZH-CN_TOPIC_0278888684__p18565184733911)
- [CHF发起重授权触发融合计费会话更新](#ZH-CN_TOPIC_0278888684__p79601046125617)

1. 用户访问数据业务，在配额耗尽或剩余配额量达到阈值、配额空闲时间门限值达到、访问无配额的新业务等场景会触发计费会话更新流程。
  当到达下发的阈值或配额，满足计费事件上报时，UPF发送 [PFCP Session Report Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Report Request_33031211.md) 消息，消息中携带Usage Report信元，记录用户使用的流量、时间等信息。Usage Report包含的主要信元如 [表1](#ZH-CN_TOPIC_0278888684__table18994141118495) 所示。

  > **说明**
  > 当达到阈值或配额耗尽时，触发UPF发送PFCP Session Report Request消息申请新配额，并启动定时器，定时器时长= **T3RESPONSE** * **N3REQUEST** +4秒（ **T3RESPONSE** 及 **N3REQUEST** 参数由UPF上的 **SET UPPFCPPATH** 命令配置），当定时器超时，未收到配额更新消息时，UPF阻塞业务。

  *表1 Usage Report包含的主要信元*

  | 信元 | 信元解释 |
  | --- | --- |
  | URR ID | 标识上报的URR。 |
  | UR-SEQN | 唯一标识URR的Usage Report。 |
  | Start Time | 起始时间。 |
  | End Time | 结束时间。 |
  | Volume Measurement | 标识流量使用情况，包含总流量、上行流量和下行流量。 |
  | Duration Measurement | 标识时长使用情况：<br>- 当下发Create URR信元的Measurement Information的ISTM携带为0或不携带，时长统计从UPF首次收到业务数据包时开始，直到业务终止时结束。<br>- 当下发Create URR信元的Measurement Information的ISTM携带为1，时长统计从UPF首次收到授权的配额成功时开始，直到用户去激活时或者该配额管理实例终止结束。 |
  | Time of First Packet | 首包时间。 |
  | Time of Last Packet | 尾包时间。 |

4. CHF对SMF上报的用量进行扣费处理，并更新CHF-CDR。CHF会生成一张CDR原始话单，同时转换生成CDR最终话单，存储到最终话单临时文件中。当最终话单临时文件达到转正阈值（默认1M大小或半小时），则转正为最终话单文件。

1. SMF通过**ADD CCT**命令本地配置RG级配额的有效时长，SMF收到CHF下发的配额后会按照有效时长启动定时器，超时后SMF将向UPF查询配额用量并上报CHF。即SMF首先向UPF发送[PFCP Session Modification Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)消息，发起计费即时查询，查询配额用量。
2. UPF向SMF返回[PFCP Session Modification Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)消息，根据指示上报计费信息。
3. SMF向CHF发送[Nchf_ConvergedCharging_ChargingDataUpdate Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_f4a8bbee/Nchf_ConvergedCharging_ChargingDataUpdate Request_23690004.md)消息，更新计费会话，申请新的配额。当**SET CNVRGDCHGPARA**命令“MERGERGVTSW”参数配置为“ENABLE”时，支持对同时发生的RG级VT事件进行合并，一起上报给CHF。当“MERGERGVTSW”参数配置为“DISABLE”时，每个RG级VT事件单独上报CHF，不做消息合并。
4. CHF对SMF上报的用量进行扣费处理，并更新CHF-CDR。CHF会生成一张CDR原始话单，同时转换生成CDR最终话单，存储到最终话单临时文件中。当最终话单临时文件达到转正阈值（默认1M大小或半小时），则转正为最终话单文件。
5. CHF返回[Nchf_ConvergedCharging_ChargingDataUpdate Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_f4a8bbee/Nchf_ConvergedCharging_ChargingDataUpdate Response_23370256.md)消息，携带授权的配额。
6. SMF通过[PFCP Session Modification Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)消息下发配额到UPF。
7. UPF返回[PFCP Session Modification Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)消息。

  | "timeQuotaThreshold":100 | 代表时间配额门限为100秒。 |
  | "volumeQuotaThreshold":2000 | 代表流量配额门限为2000字节。 |
7. SMF本地保存配额信息，向UPF发送[PFCP Session Modification Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)消息，将申请的配额信息下发给UPF。
8. UPF本地保存配额，向SMF返回[PFCP Session Modification Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)消息。
