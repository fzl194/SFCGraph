## [操作场景](#ZH-CN_OPI_0170933935)

本操作指导介绍在运行网络中激活基于用户等级的QCI扩展特性的操作过程。

UNC 支持通过QCI（QoS Class Identifier）扩展功能可以将多个QCI设定为一个业务属性，每个QCI代表该业务的一个等级。S-GW/P-GW/eNodeB等网元获得扩展QCI信息后，在QoS控制时可为同类业务中高优先级用户提供高优先级保障。通过QCI来体现用户等级，对于每个Non-GBR(Guaranteed Bit Rate)/GBR业务，运营商可以区分多个等级的用户，对高优先级用户提供高优先级调度。

> **说明**
> 适用于MME

## [任务示例](#ZH-CN_OPI_0170933935)

任务描述

当网络侧下发扩展QCI时，根据运营商的配置，在MME上配置扩展QCI值为64向标准QCI值为8的转换关系。

脚本

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2QCIE02", SWITCH=ENABLE;
```

//（可选）设置扩展QCI值64转换为标准QCI值8。

```
ADD QCICONV: QCIEXT=64, QCISTEP=0, QCISTD=8;
```

//配置MME间移动时用户的QCI传递策略。

```
SET COMPATIBILITY: GTPV2EXTQCI=SUPPORT;
```
