## [操作场景](#ZH-CN_OPI_0295923545)

GGSN/PGW-C和OCS都可以指定CCR触发场景，OCS可以通过CCA消息携带的信元Trigger AVP指定触发CCR上报的场景，GGSN/PGW-C本地也可以配置CCR触发场景，OCS下发优先级更高。

GGSN/PGW-C在多种场景下支持向OCS发起CCR更新流程，配置的时候可以根据需要选择配置。在线计费过程中，当其中一个条件满足时，便会触发CCR上报。建议配置的配额门限的大小适度，要满足CCR与CCA之间下次新配额申请时使用。

> **说明**
> 适用于 GGSN、 PGW-C。

## [操作步骤](#ZH-CN_OPI_0295923545)

- 配置 UNC Gy接口与OCS之间的DCC会话建立方式。
  配置全局通用参数DCC模板，DCC模板名称。 “DCCTMPLTNAME” = “global”
  **MOD DCCTEMPLATE**
- 配置DCC模板。
  **ADD DCCTEMPLATE**
    1. 配置CCR-I消息中允许携带的RG的最大个数。配置参数 “CCRINITRGNUM”
    2. 配置剩余流量配额小于VQT后向OCS发起CCR。配置参数 “VQT”
    3. 配置剩余时长配额小于TQT后向OCS发起CCR。配置参数 “TQT”
      > **说明**
      > “VQT” 和 “TQT” 配置值不建议小于20，建议保持默认配置。
    4. 配置剩余事件配额小于UQT后向OCS发起CCR。配置参数 “UQT”
    5. 配置配额能使用的有效时长超时后向OCS发起CCR。配置参数 “VALIDTIME”
    6. 配置当用户数据包停止传送时长超过QHT后向OCS发起CCR。配置参数 “QHT”
      > **说明**
      > - 如果不想使用该功能，即无论用户停止数据传送多长时间也不触发CCR，则可以在OCS上指定CCA中携带的Quota-Holding-Time AVP值为0，或CCA中不携带Quota-Holding-Time AVP，且GGSN/PGW-C本地配置QHT为0。
      > - 在没有明确需求的情况下建议不使用QHT，配置为0。
      > - 当QHT不为0时QHT必须大于QCT（Quota Consumption Time）。
    7. 尚未申请过配额的新业务触发向OCS发起CCR。配置参数 “SERVICETRIGGER”
      > **说明**
      > 以下为配置 PDP上下文/EPS承载 更新场景（RAT改变、 SGSN/S-GW 地址改变、QoS改变、用户位置信息ULI改变 （包括用户MCC、MNC改变）、终端时区改变 触发的 PDP上下文/EPS承载 更新）触发CCR。
    8. 配置RAT改变发送CCR。配置参数 “RATTRIGGER”
    9. 配置 SGSN/S-GW 地址改变发送CCR。配置参数 “SGSNTRIGGER”
    10. 配置QoS改变发送CCR。配置参数 “QOSTRIGGER”
    11. 配置ULI改变发送CCR。配置参数 “ULITRIGGER”
    12. 配置终端时区改变发送CCR。配置参数 “UETZTRIGGER”
    13. 配置终端的MCC改变发送CCR。配置参数 “MCCTRIGGER”
    14. 配置终端的MNC改变发送CCR。配置参数 “MNCTRIGGER”
    15. 配置终端的LAC改变发送CCR。配置参数 “LACTRIGGER”
    16. 配置终端的eNodeB改变发送CCR。配置参数 “ENBTRIGGER”
    17. 配置终端的RAC改变发送CCR。配置参数 “RACTRIGGER”
    18. 配置终端的CellID改变发送CCR。配置参数 “CELLIDTRIGGER”
- 配置PDP上下文/EPS承载激活时发送CCR-I与OCS交互。
    1. 配置在线计费用户激活时候需要与OCS交互。
      **SET OCSINIT**
    2. 配置用户激活时CCR-I消息要携带的费率标识（RG），具体步骤请参见 [配置在线计费的费率标识](../配置在线计费的费率标识_95923388.md) 。
          a. 配置URR。
            **ADD URR**
          b. 配置上行和下行发起的业务对应的URR。
            **ADD URRGROUP**
    3. 配置在线计费用户激活时CCR-I消息中携带指定的费率标识。
      **SET CTXSTARTRATING**
      > **说明**
      > 此命令只对内容计费用户生效，PDP级别计费用户不适用该命令。PDP级别计费用户CCR-Initial中携带UserProfile下或者全局下配置的默认计费属性，其中UserProfile优先。如果UserProfile下没有配置CCR-Initial的计费属性，则使用配置在全局下的CCR-Initial的计费属性。如果全局下没有配置CCR-Initial的计费属性，则CCR-Initial不携带计费属性。
      **ADD DCCTEMPLATE** 命令配置CCR-I消息中允许携带的RG的最大个数时要与 **SET CTXSTARTRATING** 命令配合使用。如果在 **SET CTXSTARTRATING** 命令中指定了N个RG，则 **ADD DCCTEMPLATE** 命令配置CCR-I消息中允许携带的RG的最大个数至少要配置为N，才能使得上述N个RG都能被CCR-I消息带上申请配额。

## [任务示例](#ZH-CN_OPI_0295923545)

任务描述

以时长计费为例，配置单用户单DCC会话在线计费的各种CCR触发场景。

脚本

//配置DCC模板。

```
ADD DCCTEMPLATE:DCCTMPLTNAME="dcc-test", QHT=0,  VQT=0, TQT=20, VALIDTIME=30, RATTRIGGER=ENABLE, SGSNTRIGGER=ENABLE, QOSTRIGGER=ENABLE, ULITRIGGER=ENABLE, SERVICETRIGGER=ENABLE;
```

//配置 PDP上下文/EPS承载 激活时发送CCR-I与OCS交互。

```
SET OCSINIT:DCCTMPLTNAME="dcc-test",OCSINIT=ENABLE;
```

```
ADD URR: URRNAME="onlineURR", URRID=1100, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=100, ONLMETERINGTYPE=TIME;
```

```
ADD URRGROUP: URRGROUPNAME="urrgroup1", UPURRNAME1="onlineURR", DOWNURRNAME1="onlineURR";
```

```
ADD USERPROFILE:USERPROFILENAME="up-test", UPTYPE=PCC;
```

```
SET URRGRPBINDING: USERPROFILENAME="up-test", DFTURRGRPNAME="urrgroup1";
```

```
SET CTXSTARTRATING:USERPROFILENAME="up-test",CTXRURRGRPNAME1="urrgroup1";
```

//为APN实例"apn-test"绑定DCC template模板。

```
SET APNCHARGECTRL:APN="apn-test",DCCTEMPLATE="dcc-test";
```

//将UserProfile绑定到APN下APN实例名"apn-test"。

```
ADD USRPROFGROUP:USERPROFGNAME="upg-test";
```

```
ADD UPBINDUPG:USERPROFGNAME="upg_test",UPBINDTYPE=DEFAULT,USERPROFILENAME="up_test";
```

```
ADD APNUSRPROFG:APN="apn-test",USERPROFGNAME="upg-test";
```
