#### [命令功能](#ZH-CN_MMLREF_0000001172225199)

![](设置S1模式移动性管理流程控制参数(SET EMMPROCTRL)_72225199.assets/notice_3.0-zh-cn_2.png)

通过该命令配置不合适的原因值下发给UE后，可能导致UE无法进行业务。

**适用网元：MME**

在用户接入S1模式时， UNC 可通过此命令控制MM流程进行特殊处理以及特殊场景下指定的原因值下发，以满足运营商实现对现网设备兼容性特殊控制的需求。

#### [注意事项](#ZH-CN_MMLREF_0000001172225199)

- 系统初次上电运行时，会执行系统初始设置值。
- 此命令执行后立即生效。
- 该命令与**[ADD EMMPROCTRLIMSI](增加指定用户S1模式移动性管理流程控制参数(ADD EMMPROCTRLIMSI)_16407797.md)**命令配合使用，系统会首先匹配**[ADD EMMPROCTRLIMSI](增加指定用户S1模式移动性管理流程控制参数(ADD EMMPROCTRLIMSI)_16407797.md)**命令的配置记录，若发现系统中不存在或不匹配（用户范围不匹配）**[ADD EMMPROCTRLIMSI](增加指定用户S1模式移动性管理流程控制参数(ADD EMMPROCTRLIMSI)_16407797.md)**命令的配置记录才会匹配本命令的配置记录。
- 配置下发的原因值可能会对终端行为产生影响，在配置前评估影响。关于不同原因值的含义及对终端行为的影响请参见协议3GPP TS 24.301。
- 当Attach、RAU/TAU流程中HLR/HSS或MSC发送消息携带的原因值有多种情况时，将采用 [**ADD CAUSEGRP**](../../原因值管理/原因值映射组配置/增加原因值映射组配置(ADD CAUSEGRP)_72225173.md) 命令配置对应的原因值映射规则。原因值映射规则请参见命令 [**ADD CAUSEMAP**](../../原因值管理/原因值映射配置/增加原因值映射配置(ADD CAUSEMAP)_26145490.md) 。
- 链式备份恢复时，SR流程后ULR或者AIR流程失败时下发固定原因值#10（Implicitly detached），TAU流程后ULR或者AIR流程失败时下发固定原因值#9（UE identity cannot be derived by the network），不受此命令控制。

#### [本地用户权限](#ZH-CN_MMLREF_0000001172225199)

manage-ug；system-ug

#### [网管用户权限](#ZH-CN_MMLREF_0000001172225199)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [使用实例](#ZH-CN_MMLREF_0000001172225199)

当下发的缺省原因值导致位置更新流程异常时，设置 “PROT（流程类型）” 为 “UPDATE_LOCATION（更新位置流程）” ， “ULR拒绝原因值（HSS超时）” 为 “111” ， “ULR拒绝原因值（HSS拒绝）” 为 “2” ， “ULR拒绝原因值（Diameter链路异常）” 为 “15” 的S1模式移动性管理流程控制参数：

SET EMMPROCTRL: PROT=UPDATE_LOCATION, ULRHSSTMOUT=111, ULRHSSREJ=2, ULRDMERR=15;

当需要修改参数 “ULR拒绝原因值（HSS拒绝）” 、 “AIR拒绝原因值（HSS拒绝）” 或 “LUR拒绝原因值映射（MSC拒绝）” 为非0值时，则需要配置对应的原因值映射规则，以 “ULR拒绝原因值（HSS拒绝）” 为例，下面配置 “原因值组标识” 为 “1” 的映射规则，将原始原因值 “5420” 转换成 “目标原因值” 为 “15” ，配置顺序如下：

ADD CAUSEGRP: CAUSEGRPID=1, CAUSEGRPNM="S1-ULR-HSS-REJ";

ADD CAUSEMAP: CAUSEGRPID=1, CAUSERANGE=SPECIAL, BGCAUSE=5420, EDCAUSE=5420, TCAUSE=15;

SET EMMPROCTRL: PROT=UPDATE_LOCATION, ULRHSSREJ=1;

> **说明**
> 用户输入的ULRHSSREJ、AIRHSSREJ或SGSREJ必须在CAUSEGRP（原因值映射组）中存在，否则以上命令执行失败。
