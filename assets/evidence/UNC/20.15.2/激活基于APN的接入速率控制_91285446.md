## [操作场景](#ZH-CN_OPI_0191285446)

本操作指导介绍在运行网络中激活基于APN的接入速率控制特性的操作过程。

随着行业用户的服务器升级、故障排除的操作频率日益增加，并且有可能在网络业务高峰期进行相关操作，会对GGSN网元造成冲击。本特性主要对该类用户进行识别并对信令进行流控，以达到保护GGSN网元不过载。

> **说明**
> 适用于 SGSN 。

## [操作步骤](#ZH-CN_OPI_0191285446)

1. 进入 “MML命令行-UNC” 窗口。
2. 设置基于APN的接入速率控制。
    a. 增加APNNI组。
      [**ADD APNNIGROUP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/APNNI信息管理/APNNI组管理/增加APNNI组(ADD APNNIGROUP)_26305508.md)
    b. 增加APNNI。
      [**ADD APNNI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/APNNI信息管理/APNNI管理/增加APNNI(ADD APNNI)_26305506.md)
    c. 增加基于APN的接入速率控制参数。
      [**ADD APNCTRLPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/基于APN的信令控制参数/增加基于APN的信令控制参数(ADD APNCTRLPARA)_72345383.md)
3. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
  > **说明**
  > 如果同一个APN在多个APN组中分别设置了信令控制门限，则只要一个APN组中的相关信令速率超过门限值便开始该APN下的用户的信令控制。

## [任务示例](#ZH-CN_OPI_0191285446)

任务描述

某运营商的组网情况为GGSN集中部署在某区域，5个SGSN分布在各个分区域，各分区域的行业用户终端全部接入2个GGSN，这样可能导致在SGSN未过载的情况下2个GGSN已经过载。该行业终端的APNNI为 “HUAWEI.COM” 。

行业网关GGSN1的PDP激活处理能力为100，DNS配置的权重为7；GGSN2的PDP激活处理能力为200，DNS配置的权重为8，通过计算得PDP请求上下文控制出门限值为（100*7+200*8）*1.5/5=690。

脚本

//设置基于APN的信令控制。

//增加APNNI组。

```
ADD APNNIGROUP: GRPID=2, DESC="M2M APNNI Group";
```

//增加APNNI。

```
ADD APNNI: GRPID=2, APNNI="HUAWEI.COM";
```

//增加基于APN的接入速率控制参数。

```
ADD APNCTRLPARA: APNGRPID=2, ATTACHTHRESH=690, PDPCONTTHRESH=690;
```

//打开本功能的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2ACIU01", SWITCH=ENABLE;
```
