# 5G LAN组会话&UE会话创建流程

运营商为某企业开通5G LAN业务后，当签约同一个5G LAN会话组内有5G LAN终端首次发起PDU会话创建时，网络侧在为该终端创建UE会话的同时，还会创建5G LAN组会话。后续该5G LAN组会话组内再有其他5G LAN终端的PDU会话创建时，网络侧只创建UE会话，并将该5G LAN终端的UE会话关联到已有的5G LAN组内。详细流程如 [图1](#ZH-CN_TOPIC_0000001199814888__fig1123514171632) 所示。

**图1** 5G LAN组会话&UE会话创建流程

<br>

![](5G LAN组会话&UE会话创建流程_99814888.assets/zh-cn_image_0000001204096758_2.png "点击放大")

详细流程如下：

1. 5G LAN UE发起PDU会话创建请求给AMF，携带TAI、SUPI、DNN、Slice、PDU会话类型（Ethernet/IP）给AMF。
2. AMF根据DNN和位置选择SMF。
3. AMF向选择的SMF发起PDU会话创建请求，发送Nsmf_PDUSession_CreateSMContext Request消息给SMF，携带DNN、Slice等信息。
4. SMF与UDM交互获取用户基于DNN/切片/PDU会话类型的5G LAN组会话的会话标识。
    a. SMF向UDM获取用户签约数据，发送Nudm_SDM_GetUESessionManagementSubscriptionData Request消息给UDM，携带TAI、SUPI、DNN、Slice、PDU会话类型（Ethernet/IP）等信息。
    b. UDM在返回的Nudm_SDM_GetUESessionManagementSubscriptionData Response消息中携带SUPI、DNN、切片、PDU会话类型（Ethernet/IP）、QoS参数、sharedVnGroupDataIds，其中sharedVnGroupDataIds为当前用户归属的5G LAN组会话的会话标识。
    c. SMF根据获得的sharedVnGroupDataIds信息发送Nudm_SDM_GetSharedData Request消息给UDM。
    d. UDM通过Nudm_SDM_GetSharedData Response消息给SMF响应sharedVnGroupDataIds对应的sharedVnGroupDatas，包含所属的DNN、切片、PDU会话类型等信息。
      *表1 关键信元解释*

      | 相关消息 | 关键信元 | 信元解释 |
      | --- | --- | --- |
      | Nudm_SDM_GetUESessionManagementSubscriptionData Request | ![](5G LAN组会话&UE会话创建流程_99814888.assets/zh-cn_image_0000001226347926_2.png) | supported-features：“=1”表示请求5G LAN会话签约数据。 |
      | Nudm_SDM_GetUESessionManagementSubscriptionData Response | ![](5G LAN组会话&UE会话创建流程_99814888.assets/zh-cn_image_0000001226024878_2.png) | sharedVnGroupDataIds：指示当前用户归属的5G LAN组会话标识。 |
      | Nudm_SDM_GetSharedData Response | ![](5G LAN组会话&UE会话创建流程_99814888.assets/zh-cn_image_0000001270345429_2.png) | sharedVnGroupDatas：指示5G LAN组会话标识对应的DNN、PDU会话类型等信息。 |
5. SMF给AMF回复Nsmf_PDUSession_CreateSMContext Response消息后根据DNN、Slice以及PDU会话类型（Ethernet/IP） 选择支持5G LAN的UPF。
  > **说明**
  > SMF会基于命令 [**SET UPSELECTFLAG**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/UPF选择管理/UPF选择条件开关/设置UPF选择条件开关（SET UPSELECTFLAG）_09652250.md) 的参数 “5G LAN UPF选择开关” 选择支持5G LAN能力的UPF。
6. SMF向UPF发起第一次N4会话创建请求，创建5G LAN组会话。SMF向UPF发送PFCP Session Establishment Request消息，携带会话类型（组会话）、5G LAN组会话标识、PDR、FAR等。
  UPF收到SMF的N4会话创建请求后根据5G LAN组会话标识和UE会话标识确定两者之间的关联关系，便于在后续的数据报文传输流程中UPF可以快速定位到UE会话。
  UPF响应PFCP Session Establishment Response消息给SMF。
  *表2 关键信元解释*

  | 相关消息 | 关键信元 | 信元解释 |
  | --- | --- | --- |
  | PFCP Session Establishment Request | ![](5G LAN组会话&UE会话创建流程_99814888.assets/zh-cn_image_0000001229836144_2.png) | nglan-session-type-value：“group (1)”表示该N4会话为组会话。 |
7. SMF向UPF发起第二次N4会话创建请求，创建UE会话。SMF向UPF发送PFCP Session Establishment Request消息，携带会话类型（UE会话）、PDR、FAR、URR等，URR中指示上报MAC地址事件，后续UPF学习到新的MAC地址后基于URR的指示上报给SMF。
  UPF收到SMF的N4会话创建请求后分配F-TEID，并根据5G LAN组会话标识和UE会话标识确定两者之间的关联关系，便于在后续的数据报文传输流程中UPF可以快速定位到UE会话。
  UPF响应PFCP Session Establishment Response消息给SMF，携带分配的F-TEID。
  *表3 关键信元解释*

  | 相关消息 | 关键信元 | 信元解释 |
  | --- | --- | --- |
  | PFCP Session Establishment Request | ![](5G LAN组会话&UE会话创建流程_99814888.assets/zh-cn_image_0000001229996420_2.png) | nglan-session-type-value：“ue (0)”表示该N4会话为UE会话。 |
8. SMF向AMF发起Namf_Communication_N1N2MessageTransfer流程，携带N2 SM Information给AMF，里面包含UPF分配的N3上行TEID，用于通知（R）AN侧将N3隧道上行对端隧道信息更新至UPF。AMF向SMF回复响应消息。
9. AMF向（R）AN发送PDU Session Resource Setup Request，请求中包含N3上行隧道信息。
10. （R）AN与UE发起信令交互，将PDU Session ID等转发至UE，请求UE建立PDU会话。
11. （R）AN向AMF发送PDU Session Resource Setup Response消息，携带给UPF分配的N3下行隧道信息AN Tunnel Info等。
12. SMF和AMF通过Nsmf_PDUSession_UpdateSMContext消息交互获取AN Tunnel Info等信息。
13. SMF向UPF发送PFCP Session Modification Request，将（R）AN侧N3隧道下行隧道信息发送给UPF，UPF回复PFCP Session Modification Response消息给SMF。
