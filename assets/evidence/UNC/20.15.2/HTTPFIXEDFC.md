## [命令功能](#ZH-CN_MMLREF_0000001229053329)

该命令用于查询HTTP固定速率流控门限值信息。

## [注意事项](#ZH-CN_MMLREF_0000001229053329)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001229053329)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001229053329)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| GROUP | 流控组 | 可选必选说明：可选参数<br>参数含义：该参数用于指定HTTP固定速率流控组的唯一标识。该参数来源于<br>[**ADD HTTPFCIPGRP**](../HTTP流控组管理/增加HTTP流控组（ADD HTTPFCIPGRP）_29053323.md)<br>命令的“流控组”参数，可通过<br>[**LST HTTPFCIPGRP**](../HTTP流控组管理/查询HTTP流控组（LST HTTPFCIPGRP）_83813632.md)<br>命令查询获取。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是0~4095。<br>默认值：无<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0000001229053329)

查询HTTP固定速率流控门限值信息，可以用以下命令：

```
%%LST HTTPFIXEDFC:;%%
RETCODE = 0  操作成功

结果如下
------------------------
                       流控组 = 1
                   流控组类型 = 服务端组
单链路发送消息流控门限(条/秒) = 500
单链路接收消息流控门限(条/秒) = 600
整系统发送消息流控门限(条/秒) = 1500
整系统接收消息流控门限(条/秒) = 2500
                       状态码 = Too Many Requests
    是否携带Retry-After消息头 = YES
          Retry-After时长(秒) = 180
                         描述 = NULL
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001229053329)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 流控组 | 该参数用于指定HTTP固定速率流控组的唯一标识。该参数来源于<br>[**ADD HTTPFCIPGRP**](../HTTP流控组管理/增加HTTP流控组（ADD HTTPFCIPGRP）_29053323.md)<br>命令的“流控组”参数，可通过<br>[**LST HTTPFCIPGRP**](../HTTP流控组管理/查询HTTP流控组（LST HTTPFCIPGRP）_83813632.md)<br>命令查询获取。 |
| 流控组类型 | 该参数用于指定配置的流控组的类别。此参数无需配置，配置后也无实际意义。 |
| 单链路发送消息流控门限(条/秒) | 该参数用于指定发送消息流控门限。该门限按HTTP链路粒度设置。 |
| 单链路接收消息流控门限(条/秒) | 该参数用于指定接收消息流控门限。该门限按HTTP链路粒度设置。 |
| 整系统发送消息流控门限(条/秒) | 该参数用于指定发送消息流控门限。该门限按网元整系统粒度设置。 |
| 整系统接收消息流控门限(条/秒) | 该参数用于指定接收消息流控门限。该门限按网元整系统粒度设置。 |
| 状态码 | 该参数用于指定接收消息速率超过流控门限时，给对端网元响应的HTTP消息中的Status Code。该参数可以和“是否携带Retry-After消息头”参数配合使用，告知对端本端的过载情况的同时携带Retry-After来指示对端多久后再发送请求。 |
| 是否携带Retry-After消息头 | 该参数用于指定接收消息速率超过流控门限时，给对端网元响应的HTTP消息中是否携带Retry-After消息头。该参数可以和“状态码”参数配合使用，告知对端本端的过载情况的同时携带Retry-After来指示对端多久后再发送请求。 |
| Retry-After时长(秒) | 该参数用于指定Retry-After的时长。 |
| 描述 | 该参数由于填写流控的描述信息，建议包含NF类型，实例名等，例如UDM_UDM001。 |
