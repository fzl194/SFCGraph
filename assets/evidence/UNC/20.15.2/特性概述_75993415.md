#### [适用NF](#ZH-CN_TOPIC_0275993415)

MME

#### [定义](#ZH-CN_TOPIC_0275993415)

基于eNodeB覆盖等级的寻呼，指MME根据eNodeB上报的推荐寻呼的eNodeB进行寻呼优化，并在寻呼消息中将eNodeB上报的推荐寻呼的小区和小区覆盖等级通过寻呼辅助信息传给eNodeB，用以支持eNodeB覆盖等级的寻呼。

推荐寻呼的小区、eNodeB和小区覆盖等级由eNodeB通过UE Context Release Complete消息上报的Recommended Cells and eNBs for Paging和Cell Identifier and Coverage Enhancement Level信元带给MME。

#### [客户价值](#ZH-CN_TOPIC_0275993415)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 避免所有小区都以高覆盖等级接入系统造成的空口资源浪费，保障覆盖同时提高系统的容量，降低业务时延。 |
| 用户 | 以合适的覆盖等级接入系统，保证业务质量同时节省终端电池耗电量。 |

#### [应用场景](#ZH-CN_TOPIC_0275993415)

当有不同覆盖等级的小区接入系统时，开启本特性的MME与eNodeB配合可以有效寻呼不同覆盖等级小区下的用户，提高系统容量和寻呼效率。

#### [可获得性](#ZH-CN_TOPIC_0275993415)

**涉及NF**

| 涉及NF | 支持版本 | 功能说明 |
| --- | --- | --- |
| MME | UNC 20.5.0及后续版本 | MME接收并保存eNodeB上报的寻呼辅助信息，并根据寻呼辅助信息进行寻呼增强。 |
| eNodeB | 无特殊要求 | 根据终端当前覆盖等级、寻呼次数和RAN负荷自动智能调整寻呼终端时所用的覆盖等级。 |

**License支持**

本特性只有获得了License许可后才能获得该特性的服务，对应的License控制项为 “82207721 LKV2CELP02 eMTC基于eNodeB覆盖等级的寻呼 ”。

#### [与其他特性的交互](#ZH-CN_TOPIC_0275993415)

本特性与 [WSFD-206001 精准寻呼](../../流控功能/WSFD-206001 精准寻呼_64557971.md) 一起部署，且 [**ADD S1PAGINGRULE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼规则管理/增加S1寻呼规则(ADD S1PAGINGRULE)_26306058.md) 的参数 “ACTGRP” 配置包含 “NEIGH_ENODEB（邻接eNodeB）” 时，如果eNodeB通过UE Context Release Complete消息将推荐寻呼的eNodeB带给MME，MME在寻呼邻接eNodeB时按照eNodeB上报的推荐寻呼的eNodeB列表进行寻呼，不使用学习到的eNodeB邻接关系和通过 [**ADD ENBNSINDB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/eNodeB邻接关系测试/增加eNodeB邻接关系(ADD ENBNSINDB)_72345859.md) 配置的eNodeB邻接关系。

#### [应用限制](#ZH-CN_TOPIC_0275993415)

[**相关概念**](#ZH-CN_TOPIC_0275993415__sub3)

覆盖等级：覆盖等级与小区的信号质量有关，信号质量好的小区覆盖等级低，信号质量差的小区覆盖等级高。eNodeB对不同的覆盖等级下的UE采用不同的调制编码方案和重传技术，相对于接入覆盖等级高的UE，eNodeB能同时接入更多覆盖等级低的UE。

[**系统实现**](#ZH-CN_TOPIC_0275993415__sub3)

本特性License开启的情况下，eNodeB通过UE Context Release Complete消息将推荐寻呼的小区，eNodeB和小区覆盖等级（Recommended Cells and eNBs for Paging/ Cell Identifier and Coverage Enhancement Level）带给MME后，MME将这些信息保存在用户上下文中，并标记发送过小区覆盖等级（Cell Identifier and Coverage Enhancement Level）的eNodeB有覆盖增强寻呼能力。

MME在寻呼时利用这些信息做如下处理：

- 当用户上下文中存在推荐寻呼的小区或小区覆盖等级时:
    - 如果[**SET M2MCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/M2M控制参数/设置M2M控制参数(SET M2MCTRL)_72345369.md)命令的“CHK_ENB_CE_SW”参数为“ON”， MME将推荐寻呼的小区和小区覆盖等级通过寻呼辅助信息（Assistance Data for Paging）带给有覆盖增强支持能力的待寻呼eNodeB。
    - 如果[**SET M2MCTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/M2M管理/M2M控制参数/设置M2M控制参数(SET M2MCTRL)_72345369.md)命令的“CHK_ENB_CE_SW”参数为“OFF”，MME将推荐寻呼的小区和小区覆盖等级通过寻呼辅助信息（Assistance Data for Paging）带给所有待寻呼的eNodeB。
  > **说明**
  > 通过命令 [**DSP ENBATTR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/eNodeB属性信息管理/显示eNodeB属性(DSP ENBATTR)_72345861.md) 可以查看eNodeB的覆盖增强寻呼能力指示，通过命令 [**CLR ENBATTR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/eNodeB属性信息管理/清除eNodeB属性(CLR ENBATTR)_26146262.md) 可以清除MME上eNodeB的覆盖增强寻呼能力指示。
  >
  > 寻呼辅助信息（Assistance Data for Paging）是否携带子信元Next Paging Area Scope受 [**SET S1CMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1接口兼容性/设置S1接口兼容性(SET S1CMPT)_72345837.md) 命令中的参数“Next Paging Area Scope开关”控制。
- 当用户上下文中存在推荐寻呼的eNodeB，且系统同时部署了[WSFD-206001 精准寻呼](../../流控功能/WSFD-206001 精准寻呼_64557971.md)，[**ADD S1PAGINGRULE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼规则管理/增加S1寻呼规则(ADD S1PAGINGRULE)_26306058.md)的参数“ACTGRP”配置包含“NEIGH_ENODEB（邻接eNodeB）”时，MME在寻呼邻接eNodeB时按照eNodeB上报的推荐寻呼的eNodeB列表进行寻呼，不使用学习到的eNodeB邻接关系和通过[**ADD ENBNSINDB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/eNodeB邻接关系测试/增加eNodeB邻接关系(ADD ENBNSINDB)_72345859.md)配置的eNodeB邻接关系。

假设MME配置最大寻呼次数为3次，覆盖等级寻呼的处理流程如 [图1](#ZH-CN_TOPIC_0275993415__fig3) 所示：

**图1** 覆盖等级寻呼处理流程

<br>![](特性概述_75993415.assets/zh-cn_image_0000001372626333_2.png)<br>

1. UE先通过eNodeB1接入系统，覆盖等级为2。
2. 释放连接时eNodeB1通过UE Context Release Complete消息将推荐寻呼的小区和eNodeB以及小区覆盖等级带给MME。
3. UE通过eNodeB2下覆盖等级为3的小区接入系统。
4. 下行数据触发寻呼，首次寻呼时，MME下发的Paging Attempt Count为1，eNodeB1和eNodeB2使用上一次接入的覆盖等级2进行寻呼，寻呼失败。
5. MME调整下发的Paging Attempt Count为2，eNodeB1和eNodeB2调整寻呼的覆盖等级为3。
6. 寻呼成功。

> **说明**
> MME下发的Paging消息中Assistance Data for Paging的子信元Intended Number of Paging Attempts即MME下发寻呼的最大次数，MME下发寻呼达到最大次数而仍然没有得到UE响应时，认为寻呼失败。
>
> MME下发寻呼的最大次数为1+最大寻呼重发次数：
>
> - 对于特定QCI业务的寻呼，如果有匹配的QCI寻呼策略参数，最大寻呼重发次数为[**ADD QCIPAGINGINFO**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/QCI寻呼策略参数配置/增加QCI寻呼策略参数配置(ADD QCIPAGINGINFO)_72225211.md)的参数“N3413”的取值；否则，最大寻呼重发次数由[**SET EMM**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/MM协议参数管理/S1模式MM协议参数/设置S1模式MM协议参数(SET EMM)_72225207.md)配置，对于VoLTE紧急呼叫和VoLTE普通语音业务，最大寻呼重发次数为参数“HPN3413”的取值，对于其他低优先级业务，最大寻呼重发次数为“N3413”的取值。
> - 如果用户有可匹配的[**ADD S1PAGINGRULE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1寻呼规则管理/增加S1寻呼规则(ADD S1PAGINGRULE)_26306058.md)配置，参数“ACTGRP”的取值每包含一种寻呼动作(LAST_ENODEB(最近访问eNodeB)，NEIGH_ENODEB(邻接eNodeB)，LAST_TAI(最近访问TA))，最大寻呼重发次数累加1。
