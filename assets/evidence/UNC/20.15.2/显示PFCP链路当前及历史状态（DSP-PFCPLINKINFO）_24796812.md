## [命令功能](#ZH-CN_MMLREF_0000001124796812)

**适用NF：SMF、SGW-C、PGW-C、GGSN**

该命令用于查询PFCP链路当前及历史状态记录，包括接口类型、链路状态变更、变更时间戳、以及本端对端地址等信息。当前支持每个进程最多显示100条历史链路。

## [注意事项](#ZH-CN_MMLREF_0000001124796812)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001124796812)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001124796812)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| UPINSTANCEID | UP实例标识 | 可选必选说明：可选参数<br>参数含义：该参数用于指定UPF的实例标识。当需要查询指定UPF的路径信息时，可以指定该参数。<br>数据来源：全网规划<br>取值范围：字符串类型，输入长度范围是1~50。不区分大小写。<br>默认值：无<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0000001124796812)

- 查询所有链路状态。 DSP PFCPLINKINFO:;
  ```
  %%DSP PFCPLINKINFO: ;%%
  RETCODE = 0  操作成功

  结果如下
  ------------------------
              UP实例标识  =  upf_instance_1
                接口类型  =  N4
            本端IPv4地址  =  10.2.102.17
            本端IPv6地址  =  ::
            对端IPv4地址  =  192.168.208.11
            对端IPv6地址  =  ::
            链路状态变更  =  UP->DOWN
      链路状态变更时间戳  =  09:30:05 06/04/2021
                 POD名称  =  uncpod-0
  (结果个数 = 1)
  ```
- 查询UPF为“upf_instance_1”的UP链路状态。 DSP PFCPLINKINFO: UPINSTANCEID="upf_instance_1".
  ```
  %%DSP PFCPLINKINFO: UPINSTANCEID="upf_instance_1";%%
  RETCODE = 0  操作成功

  结果如下
  ------------------------
              UP实例标识  =  upf_instance_1
                接口类型  =  N4
            本端IPv4地址  =  10.2.102.17
            本端IPv6地址  =  ::
            对端IPv4地址  =  192.168.208.11
            对端IPv6地址  =  ::
            链路状态变更  =  UP->DOWN
      链路状态变更时间戳  =  09:30:05 06/04/2021
  (结果个数 = 1)
  ```

| 本端IPv4地址 | 该参数用于指示本端IPv4地址。 |
| 本端IPv6地址 | 该参数用于指示本端IPv6地址。 |
| 对端IPv4地址 | 该参数用于指示对端IPv4地址。 |

| 链路状态变更 | 该参数用于指示链路状态变更。 |

| POD名称 | 该参数用于显示POD名称。 |
