# 本地配置与签约数据的优先级

当用户接入，AMF首先根据UE签约数据或AMF本地配置的 **RAT限制信息（RAT restriction ）** 和 **核心网限制信息（Core Network type restriction）** 信息，以及UE当前使用的无线接入类型来决策是否允许用户接入。移动性接入限制信息获取方式以及优先级如下：

- **签约数据**
  AMF从签约数据获取移动性接入限制信息（AccessAndMobilitySubscriptionData）中获取 **RAT限制（RAT restriction ）** 信息和 **核心网限制（Core Network type restriction）** 信息。
- **AMF本地配置**
  为了在AMF上支持针对不同用户接入的灵活控制，AMF提供用户接入控制的本地策略配置，可配置本地策略适用的 **用户范围** 、指定限制的 **RAT类型** 和指定限制的 **核心网类型** 等。
  对于指定的用户（群），根据 [**ADD NGMMSUBDATA**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/签约数据管理/增加用户移动性管理参数（ADD NGMMSUBDATA）_09651362.md) 命令中“SUBRANGE”参数来匹配 **用户范围** ，优先级从高到低依次为：IMSI前缀（包括用户群组）>外网用户>本网用户>所有用户。如果其匹配成功，则按照本地配置的策略执行接入控制。
- **AMF本地配置****与****签约数据****的优先级**
  对于待接入的用户，同时存在签约信息和AMF本地配置的移动性接入限制信息，如果用户匹配了AMF本地配置的接入限制记录，则按照本地配置执行接入控制。只有当AMF本地未配置或者用户无任何匹配的接入限制记录时，AMF才根据签约数据检查接入控制。具体决策场景如 [表1](#ZH-CN_TOPIC_0248748809__table958362413522) 。

  *表1 移动性接入限制信息具体决策场景*

  | AMF本地配置（RAT限制） | AMF本地配置（<br>**核心网限制**<br>） | 签约数据（RAT限制） | 签约数据（<br>**核心网限制**<br>） | 决策结果 | 说明 |
  | --- | --- | --- | --- | --- | --- |
  | - | - | - | - | 允许接入。 | 如果AMF本地未配置或者用户无匹配的接入限制配置记录，且签约数据中没有RAT限制或者核心网限制，则表示不限制该用户使用NR接入，即该用户在5GC的接入控制检查通过。 |
  | - | - | NR | Any | 限制用户NR接入到5GC。 | 当该用户在AMF本地未配置或者用户无任何匹配的接入限制记录时，则AMF才根据签约数据检查接入控制。 |
  | - | - | NR_REDCAP | Any | 限制用户NR_REDCAP接入到5GC。 | 当该用户在AMF本地未配置或者用户无任何匹配的接入限制记录时，则AMF才根据签约数据检查接入控制。 |
  | - | - | EUTRA | Any | 限制用户EUTRA接入到5GC。 | 当该用户在AMF本地未配置或者用户无任何匹配的接入限制记录时，则AMF才根据签约数据检查接入控制。 |
  | - | - | Any | 5GC | 限制用户接入到5GC。 | 当该用户在AMF本地未配置或者用户无任何匹配的接入限制记录时，则AMF才根据签约数据检查接入控制。 |
  | - | - | Any | EPC | 限制用户从5G到4G的互操作流程。 | 当该用户在AMF本地未配置或者用户无任何匹配的接入限制记录时，则AMF才根据签约数据检查接入控制。 |
  | Null | Null | Any | Any | 允许接入。 | 如果用户匹配的AMF上的配置记录给出的RAT限制和核心网限制都是空值（Null），那么不论该用户签约的移动性接入限制信息配置为何值，AMF都对该用户做允许接入。 |
  | NR | Any | Any | Any | 限制用户NR接入到5GC。 | 如果其匹配了AMF本地配置的接入限制记录，则按照本地配置匹配记录给出的接入限制执行，不考虑签约数据。 |
  | NR_REDCAP | Any | Any | Any | 限制用户NR_REDCAP接入到5GC | 如果其匹配了AMF本地配置的接入限制记录，则按照本地配置匹配记录给出的接入限制执行，不考虑签约数据。 |
  | EUTRA | Any | Any | Any | 限制用户EUTRA接入到5GC（NAS组网场景）。 | 如果其匹配了AMF本地配置的接入限制记录，则按照本地配置匹配记录给出的接入限制执行，不考虑签约数据。 |
  | Any | 5GC | Any | Any | 限制用户接入到5GC。 | 如果其匹配了AMF本地配置的接入限制记录，则按照本地配置匹配记录给出的接入限制执行，不考虑签约数据。 |
  | Any | EPC | Any | Any | 限制用户从5G到4G的互操作流程。 | 如果其匹配了AMF本地配置的接入限制记录，则按照本地配置匹配记录给出的接入限制执行，不考虑签约数据。 |
  > **说明**
  > - “-”表示不存在配置或者签约。
  > - “Any”表示任意值或者任意值组合。
  > - “Null”表示记录给出的限制方式为空（参数全部清空），即不限制。
  > - NR_REDCAP作为NR的一种子类型，当AMF本地配置RAT限制为NR时，同时对NR_REDCAP用户也会产生限制；当AMF本地配置RAT限制为NR_REDCAP时，对普通NR用户无限制。
  > - 当UE在5GC接入控制检查失败时，AMF向UE返回注册拒绝，并通过[**SET NGMMPROCTRL**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM流程管理/5G移动性管理流程控制参数/设置5G移动性管理流程控制参数（SET NGMMPROCTRL）_09652386.md)配置携带相应原因值。UE的5G接入被拒绝后会尝试在4G接入，相关流程在EPC处理。
  > - 如果是**紧急注册**（5GS registration type=emergency registration），则跳过用户接入控制检查。
