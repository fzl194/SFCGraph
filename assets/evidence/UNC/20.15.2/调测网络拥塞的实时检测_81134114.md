# 调测网络拥塞的实时检测

## [操作场景](#ZH-CN_OPI_0000001381134114)

本操作指导介绍在 UNC 调测网络拥塞实时检测特性的操作过程。

> **说明**
> 适用于MME、SGW-C、PGW-C、AMF、SMF。

## [必备事项](#ZH-CN_OPI_0000001381134114)

前提条件

- 请仔细阅读[WSFD-205013 网络拥塞的实时检测](../WSFD-205013 网络拥塞的实时检测_80602062.md)。
- 完成[激活网络拥塞的实时检测](激活网络拥塞的实时检测_31333865.md)。

数据

该操作无需准备数据。

工具

OM Portal

## [操作步骤](#ZH-CN_OPI_0000001381134114)

1. 检查移动性管理流程ULI上报开关是否开启。
  执行 **[LST SMFUNC](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/SM扩展功能管理/查询会话管理扩展功能(LST SMFUNC)_72225363.md)** 命令，查看 “移动性管理流程ULI上报开关（MMULI）” 参数配置是否为“ON”。
    - 是 -> 执行[2](#ZH-CN_OPI_0000001381134114__step16880124184816)。
    - 否 -> 执行**[SET SMFUNC](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/SM扩展功能管理/设置会话管理扩展功能(SET SMFUNC)_26145684.md)**命令开启ULI上报开关。
2. 调测4G用户的网络拥塞实时检测功能。
    a. 创建4G用户跟踪。
    b. 用户开机上网，进行网络业务，发起会话创建流程。
    c. 查看4G用户跟踪中Create Session Request消息及PFCP Session Establishment Request消息中是否携带ULI信元。
          - 预期结果：Create Session Request消息及PFCP Session Establishment Request消息中均携带ULI信元，用户所处基站的位置信息会通过“eci”信元呈现。
          - 异常结果：Create Session Request消息或PFCP Session Establishment Request消息中未携带ULI信元，请联系华为技术支持解决。
      **图1** 4G用户跟踪消息（样例）

      <br>

      ![](调测网络拥塞的实时检测_81134114.assets/zh-cn_image_0000001428065240_2.png "点击放大")
    d. 4G用户进行跨小区移动，切换无线侧基站，发起会话修改流程。
    e. 查看4G用户跟踪中Modify Bearer Request消息及PFCP Session Modification Request消息中是否携带ULI信元。
          - 预期结果：Modify Bearer Request消息及PFCP Session Modification Request消息中均携带ULI信元，用户所处基站的位置信息会通过“eci”信元呈现。
          - 异常结果：Modify Bearer Request消息或PFCP Session Modification Request消息中未携带ULI信元，请联系华为技术支持解决。
3. 调测5G用户的网络拥塞实时检测功能。
    a. 创建5G用户跟踪。
    b. 用户开机上网，进行网络业务，发起会话创建流程。
    c. 查看5G用户跟踪中Nsmf_PDUSession_CreateSMContext Request消息及PFCP Session Establishment Request消息中是否携带ULI信元。
          - 预期结果：Nsmf_PDUSession_CreateSMContext Request消息及PFCP Session Establishment Request消息中均携带ULI信元，用户所处基站的位置信息会通过“gnb-id”信元呈现。
          - 异常结果：Nsmf_PDUSession_CreateSMContext Request消息或PFCP Session Establishment Request消息中未携带ULI信元，请联系华为技术支持解决。

      <br>

      ![](调测网络拥塞的实时检测_81134114.assets/zh-cn_image_0000001477984789_2.png "点击放大")
    d. 5G用户进行跨小区移动，切换无线侧基站，发起会话修改流程。
    e. 查看5G用户跟踪中Nsmf_PDUSession_UpdateSMContext Request消息及PFCP Session Modification Request消息中是否携带ULI信元。
          - 预期结果：Nsmf_PDUSession_UpdateSMContext Request消息及PFCP Session Modification Request消息中均携带ULI信元，用户所处基站的位置信息会通过“gnb-id”信元呈现。
          - 异常结果：Nsmf_PDUSession_UpdateSMContext Request消息或PFCP Session Modification Request消息中未携带ULI信元，请联系华为技术支持解决。
