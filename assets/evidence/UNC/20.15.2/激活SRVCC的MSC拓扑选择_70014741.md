> **说明**
> MSC的主机名格式为：<"TOPON"|"TOPOFF">.<SINGLE-LABEL-INTERFACE-NAME>.<CANONICAL-NODE-NAME>。
>
> - TOPON：代表优选拓扑最近的节点。TOPOFF：代表不优选拓扑最近的节点。
> - SINGLE-LABEL-INTERFACE-NAME：接口名称，仅作为一个标识，并非实际的物理接口。
> - CANONICAL-NODE-NAME：规范的节点名称，可以体现不同主机名间的拓扑关系。

## [操作步骤](#ZH-CN_OPI_0170014741)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. 配置基于Sv接口选择SGs/Sv合一的MSC。
  > **说明**
  > 组网上要求同时支持SGs和Sv接口的MSC需要具备IWF（Interworking Function）。
    a. 查询指定的VLR是否配置。
      [**LST VLR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/电路域联合业务/VLR管理/查询VLR配置信息(LST VLR)_72225125.md)
          - 如果未查询到记录，则执行[步骤 3.b](#ZH-CN_OPI_0170014741__substep_b)添加VLR配置信息。
          - 如果查询到记录，且“Sv/SGs合一”等于“不合一”，执行[步骤 3.c](#ZH-CN_OPI_0170014741__substep_c)
    b. 添加支持SGs/Sv合一类型的MSC/VLR。
      [**ADD VLR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/电路域联合业务/VLR管理/增加VLR配置信息(ADD VLR)_26305254.md)
    c. 将MSC/VLR修改为支持SGs/Sv合一类型的MSC/VLR。
      [**MOD VLR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/电路域联合业务/VLR管理/修改VLR配置信息(MOD VLR)_26145444.md)
4. 配置MSC主机名对应的Sv口MSC的IP地址。
  [**ADD IPV4DNSH**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/DNS/DNS Hostfile管理/增加IPV4 DNS Hostfile记录(ADD IPV4DNSH)_26145884.md)

## [任务示例](#ZH-CN_OPI_0170014741)

任务描述

运营商同时部署了VoLTE和CSFB业务，需要保证在进行VoLTE语音过程中进行数据业务，语音和数据业务正常。

脚本

// 进入 “MML命令行-UNC” 窗口。

//打开License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2MTSS01", SWITCH=ENABLE;
```

//修改VLR配置。

```
MOD VLR: VN="86139013201", SVSGS=CONSISTENT_SELECTION, MSCHOSTNAME="TOPON.SV.MALC1M.MSC.EPC.MNC001.MCC308.3GPPNETWORK.ORG";
```

//增加Sv接口MSC的A解析记录。

```
ADD IPV4DNSH: HSINDEX=1, HOSTNAME="TOPON.SV.MALC1M.MSC.EPC.MNC001.MCC308.3GPPNETWORK.ORG", ADDRSECTION=SECTION1, IPV4ADDR1="10.1.49.205";
```
