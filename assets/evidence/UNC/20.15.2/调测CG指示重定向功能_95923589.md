## [操作场景](#ZH-CN_OPI_0295923589)

当运营商部署了离线计费业务，在进行离线计费业务过程中，当CG故障或者CG负荷过重时，CG向GGSN/SGW-C/PGW-C发送重定向请求消息，需调测CG指示重定向功能。

> **说明**
> 适用于SGW-C、PGW-C 、GGSN 。

## [必备事项](#ZH-CN_OPI_0295923589)

前提条件

- 请仔细阅读[Ga/Gy接口离线/在线计费](../../../../../业务专题/5G Core 计费解决方案/计费解决方案概述/计费原理/Ga_Gy接口离线_在线计费_87165686.md)。
- GGSN/SGW-C/PGW-C与CG1，CG2的互通配置已经完成，请参见[配置到CG的数据（GGSN/SGW-C/PGW-C）](../激活离线计费/配置到CG的数据（GGSN_SGW-C_PGW-C）_95923479.md)。
  > **说明**
  > 当CG2被指定为CG1的重定向CG时，需要将CG1，CG2的话单类型配置一致，等级不一样。其中，CG1的等级高于CG2的等级。
- 需要在CG服务器上配置CG1的重定向CG是CG2。具体配置过程请参见CG的手册。
- 调测与CG1、CG2的互通数据已经完成。

## [操作流程](#ZH-CN_OPI_0295923589)

1. 激活用户，开始访问业务。
2. 停止CG1服务器的服务。
3. 通过用户跟踪任务检查CG1是否发送重定向消息，CG2是否能正确响应GGSN/SGW-C/PGW-C的计费消息。

当CG故障或者CG负荷过重的情况下，CG向GGSN/SGW-C/PGW-C发送重定向请求消息。本调测任务以CG故障为例说明GGSN/SGW-C/PGW-C执行CG指示重定向功能。

## [操作步骤](#ZH-CN_OPI_0295923589)

1. 在OM Portal上创建用户跟踪任务，查看跟踪消息。
2. 激活用户，进行业务。
3. 停止CG1服务器的服务（具体操作请参考CG的手册），以此模拟CG1服务器故障，1分钟后查看CG1的状态是否为abnormal。
  ```
  DSP CGSTATUS:;
  ```
  ```
  CG Status
  ------------------------
  POD名称           Ga IP地址        CG IP地址       CG端口号        CG类型        CG状态      Ga端口号
  uncpod-0      192.168.10.63      192.168.0.3         3386            R7          abnormal     10601
  uncpod-0      192.168.10.63      192.168.1.3         3386            R7          normal       10601

  (结果个数 = 2)
  ---    END
  ```
    - 如果CG状态为abnormal，执行[步骤 4](#ZH-CN_OPI_0295923589__stp3)。
    - 如果CG状态为normal，请继续等待，然后再重复执行[步骤 3](#ZH-CN_OPI_0295923589__stp2)查看，如果状态一直是normal，请联系CG技术支持处理。
4. 在跟踪台上观察消息跟踪，查看CG1是否向GGSN/SGW-C/PGW-C发送重定向请求消息，以及GGSN/SGW-C/PGW-C回应正确的响应消息。如 [图1](#ZH-CN_OPI_0295923589__fig1) 所示。
  **图1** Ga接口消息

  <br>

  ![](调测CG指示重定向功能_95923589.assets/zh-cn_image_0000001641706541_2.png "点击放大")
    - 如果是，执行[步骤 5](#ZH-CN_OPI_0295923589__stp4)。
    - 如果否，执行[步骤 6](#ZH-CN_OPI_0295923589__stp5)。
5. 在跟踪台上继续跟踪，查看GGSN/SGW-C/PGW-C是否将Data record transfer话单发送消息发到了CG1指定的重定向IP（CG2：192.168.1.3）上，并且收到了CG2成功的响应消息。
    - 如果是，表明CG重定向功能正常，本调测任务结束。
    - 如果否，执行[步骤 6](#ZH-CN_OPI_0295923589__stp5)。
6. 执行 **LST CG** 命令，查看CG的配置是否与对端的CG配置一致。
  ```
  LST CG:;
  ```
  ```
  RETCODE = 0 操作成功。  

  CG参数配置 
  ----------
  CG IPv4地址        CG端口号          等级       话单类型      wal值 
  192.168.0.3         3386              0           R7           0         
  192.168.1.3         3386              1           R7           0         
  (结果个数 = 2)
  ---    END
  ```
    - 如果是，执行[步骤 7](#ZH-CN_OPI_0295923589__stp6)。
    - 如果否，执行**RMV CG**命令删除不正确的CG配置，然后执行**ADD CG**重新配置，再执行[步骤 2](#ZH-CN_OPI_0295923589__stp1)重新调测。
7. 在跟踪台上查看Redirection request消息，检查 “Node address IPv4” 信元配置与CG上的配置是否一致。
    - 如果一致，请执行[步骤 8](#ZH-CN_OPI_0295923589__step1829245425110)。
    - 如果不一致，请先在CG服务器侧配置正确的重定向IP（具体配置方法请参考CG的随机手册），保证GGSN/SGW-C/PGW-C与CG两侧的配置一致，然后重新启动CG1使配置生效，启动后继续执行[步骤 2](#ZH-CN_OPI_0295923589__stp1)重新调测。
8. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务，执行 [步骤 2](#ZH-CN_OPI_0295923589__stp1) 并保存报文。
    b. 执行 **EXP MML** 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
