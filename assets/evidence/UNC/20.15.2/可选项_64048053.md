#### [81203640 LKV2ABTVPB01 灰度升级解决方案增值包基本功能-UAM](#ZH-CN_TOPIC_0000001664048053)

| 适用NF | AMF/MME/SGSN |
| --- | --- |
| **功能描述** | 在使用普通升级方式对网元升级时，升级过程中会面临整系统复位，业务中断，用户被迫下线，冗余资源过大，升级过程复杂等问题。针对上述问题，UNC基于<br>Kubernetes<br>原生能力实现<br>Pod<br>滚动升级<br>，将业务网元的实例按规划分成若干批次，分批次逐步升级到高版本。当系统进行无损升级时，业务中断时间短，用户不用被迫下线，仅需预留少量冗余资源，在满足业务无损的情况下即可实现网元的升级。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 |
| **默认值** | 0 |
| **对应特性** | WSFD-227101 在线升级 |
| **相关控制项** | 无 |
| **应用场景** | 如果需要对网元进行升级操作，同时需要满足中断时间短和少量预留冗余资源时，可以使用无损升级的方式进行升级操作。 |

#### [81202579 LKV6DIFAIS01 磁盘故障隔离](#ZH-CN_TOPIC_0000001664048053)

| 适用NF | SGSN/MME/GGSN-C/SGW-C/PGW-C/AMF/SMF/NRF/NSSF/SMSF |
| --- | --- |
| **功能描述** | 磁盘故障隔离（存储Bypass）是指将业务节点和存储设备进行解耦，当存储设备故障不可用后，业务节点运行不受影响，业务不受影响，提供受限运维功能，提升NF可靠性。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 UNC |
| **默认值** | 1 |
| **对应特性** | WSFD-209002 磁盘故障隔离 |
| **相关控制项** | 无 |
| **应用场景** | 存储设备通过网络和计算节点连接，其中导致存储设备整体故障的因素很多，包括供电中断、网络中断、网络拥塞丢包等，都会导致存储异常。对于5G Core网元，大部分业务都对存储依赖很低，存储设备故障如果保证业务不受损，可以大大提升产品竞争力。 |
