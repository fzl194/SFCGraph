# 调测基于SPID的UE驻留和切换策略管理

- [必备事项](#ZH-CN_OPI_0168358240__1.3.2)

## [操作场景](#ZH-CN_OPI_0168358240)

调测基于SPID的UE驻留和切换策略管理。

## [必备事项](#ZH-CN_OPI_0168358240)

前提条件

- 请仔细阅读[WSFD-106207 基于SPID的UE驻留和切换策略管理特性概述（适用于SGSN、MME）](特性概述_68358236.md)。
- 已完成[激活基于SPID的UE驻留和切换策略管理](激活基于SPID的UE驻留和切换策略管理_68358239.md)。

数据

无。

工具

OM Portal跟踪。

## [操作步骤](#ZH-CN_OPI_0168358240)

- 在相关网元均完成本特性的配置后，可以采用以下步骤检查特性工作是否正常：
    1. 进入 “MML命令行-UNC” 窗口。
    2. 执行 [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) 命令查询License配置开关是否已打开。
          - 如果“SWITCH”为“ENABLE”，请执行[步骤 3](#ZH-CN_OPI_0168358240__cmd51101620141416)。
          - 如果“SWITCH”为“DISABLE”，请执行[**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)命令打开本特性对应的License配置开关。
    3. 检查是否激活基于SPID的UE驻留和切换策略管理。
      [**DSP LICENSE**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/显示License(DSP LICENSE)_00360098.md) :;

      ```
      DSP LICENSE:;
      RETCODE = 0  操作成功。

      结果如下
      -------------------------
      License销售项名称    当前销售项的授权值    当前销售项的过期时间    当前销售项的销售量纲     当前销售项的描述信息                                                          

      LKV2SPID01           1000000               2017-07-28              Default                 Camp & Handover Based on SPID                                                      
      (结果个数 = 1)
      ---    END
      ```

      - 预期结果：License资源项 “基于SPID的UE驻留和切换策略管理” 为实际购买的license数量。
          - 异常结果：观察不到该License项。
            购买“基于SPID的UE驻留和切换策略管理”的License并激活该License。
    4. HSS有签约数据，或者本地配置。以本地配置为例：如配置了RFSP ID为1，策略为通过3G进行附着，不通过4G附着。RAN侧按照RFSP为1对应的频点优先级进行驻留控制。
      [**LST RFSP**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/RFSP管理/RFSP策略管理/RFSP参数配置/查询RFSP配置(LST RFSP)_72225221.md) :;
      ```
      LST RFSP:;
      %%LST RFSP:;%%
      RETCODE = 0  操作成功。

      输出结果如下
      -------------------------
       用户范围  IMSI前缀  运营商标识  RFSP来源  目标RFSP  使用扩展RFSP策略

       本网用户  NULL      NULL        配置优先  3         否              
       指定用户  30808     NULL        配置优先  1         否              
       外网用户  NULL      0           配置优先  2         否 
      (结果个数 = 3)
      ---    END
      ```
    5. 无线配置了SPID和驻留优先级的列表关系为3G优先。
    6. 用户在SGSN/MME做了一次附着(SPID传递给RAN侧，RAN给了UE)。
      在 UNC 操作维护台上创建用户跟踪。用户成功附着后，可以在Initial Context Setup Request(Attach Accept)消息中携带 “SubscriberProfileIDforRFP” 信元值为1。

      **图1** Initial Context Setup Request(Attach Accept)用户跟踪

      <br>

      ![](调测基于SPID的UE驻留和切换策略管理_68358240.assets/zh-cn_image_0178250200_2.png)
    7. 具有GUL接入能力的UE再次附着，直接通过3G进行附着，不会尝试附着到4G。
