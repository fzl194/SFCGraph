## [任务示例](#ZH-CN_OPI_0000001305594153)

任务描述

任务1：域路由组网方式下，开启MME的HSS全故障业务保活特性，且希望MME探测到HSS全故障后自动标识用户为HSS Bypass状态。

任务2：开启MME的HSS全故障业务保活特性，手动配置故障HSS，使用户进入HSS Bypass状态。

任务3：手动触发用户退出HSS Bypass状态。

任务4：域路由组网方式下，开启MME的HSS全故障业务保活特性，且希望DRA回复5012故障码时使用户进入HSS Bypass状态。

脚本

**任务1的脚本：**

//开启HSS全故障业务保活特性license。

```
SET LICENSESWITCH: LICITEM="LKV2HSSBP01",SWITCH=ENABLE;
```

//开启HSS Bypass功能开关并配置相关策略。

```
SET HSSBYPASS: BPSW=ON, RECOVERYACT=GRACEFUL_DEREG, SCANRATE=0, SCANINTERVAL=600, AUTOEXITSW=ON;
```

//配置HSS Bypass最小签约数据配置。

```
ADD HSSBPAPNSUB: APNSUBIDX=1, APNNI="cmnet", PDNTYPE=IPV4, APNAMBRULK=1000, APNAMBRDLK=1000, QCI=9, PRIORITY=1;
ADD HSSBPAPNGRP: APNGRPID=1, APNSUBIDX=1, DEFAULTAPN=YES;
ADD HSSBPUSRSUB: SUBRANGE=ALL_USER, APNGRPID=1;
```

//配置HSS故障返回码。

```
ADD HSSBPFAULTCODE: RANGE=ALL, FAULTCODE=3002;
ADD HSSBPFAULTCODE: RANGE=ALL, FAULTCODE=5012;
```

//配置触发重选路由的错误码。

```
ADD DMFOVERCODE:ERRCODE=3002;
ADD DMFOVERCODE:ERRCODE=5012;
```

//由于链路故障等原因发送Diameter消息失败时，开启软参BYTE_EX_B324 bit1使MME能够接收失败响应并进行响应处理，避免直连DRA故障场景无法触发用户进入HSS Bypass。

```
SET SOFTPARAOFBIT: DT=BYTE_EX_B, PARANUM=324, VALUE=VALUE_1, POSITION=1;
```

//（可选）开启软参BYTE_EX_B47 bit3，配置MME在匹配用户当前安全上下文时，忽略TSC标识。解决由于终端携带TSC错误导致用户安全检查失败而无法接入的问题。

```
SET SOFTPARAOFBIT: DT=BYTE_EX_B, PARANUM=47, VALUE=VALUE_1, POSITION=3;
```
