#### [适用NF](#ZH-CN_TOPIC_0000001361327113)

MME

#### [定义](#ZH-CN_TOPIC_0000001361327113)

在5G NSA组网下，用户首次更换了5G终端，可以通过NSA自动开户实现在不换卡、不换号、无需人工操作的场景下开通5G NSA业务功能，更方便快捷地体验5G网络服务，加速5G的快速发展。

#### [客户价值](#ZH-CN_TOPIC_0000001361327113)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 可以加速5G业务的快速发展。<br>- 在5G NSA组网下，运营商可以实现针对使用5G终端的用户进行NSA自动开户，节省资金投入，同时避免了人工配置5G NSA开户的运营成本。 |
| 用户 | 用户无需换卡换号和人工签约，使用5G终端之后就能快速便捷的体验5G网络服务。 |

#### [应用场景](#ZH-CN_TOPIC_0000001361327113)

本特性适用于5G NSA网络。运营商为快速发展5G用户，可以在NSA组网下部署该特性，即可高效快速的发展5G用户，满足5G终端用户享受更优质的网络服务的需求。

**涉及网元**

| 涉及网元 | 支持版本 | 功能说明 |
| --- | --- | --- |

| MME | UNC 20.9.2及后续版本 | - 支持NSA业务。<br>- 支持携带ue-dcnr-capability信元。 |
| UDM | UDM 20.9.0及后续版本 | - 存储NSA 组网下使用5G终端用户的签约信息，包括用户号码、业务签约数据等。<br>- 判断决策是否进行NSA自动开户。 |

**License支持**

本特性无需获得License许可即可获得该特性的服务。

本特性对系统无影响。

#### [应用限制](#ZH-CN_TOPIC_0000001361327113)

- 本特性建议在华为MME和华为UDM的场景下使用。若对接异厂商UDM，需要确认异厂商UDM对可选私有信元的兼容性，当无法兼容时则需要UDM解决兼容性问题后再开启本特性。
- 本特性只建议对本网用户开启，漫游用户归属地UDM厂商不确定，不建议对漫游用户开通NSA自动开户功能。
- 只有已经是LTE的用户才支持NSA自动开户。
- 2G/3G用户进行LTE/VoLTE自动开户时开通NSA场景下，不支持UDM与HLR/SAE-HSS网元分离部署的情况。

#### [原理概述](#ZH-CN_TOPIC_0000001361327113)

4G用户使用5G终端首次接入5G NSA网络时，通过MME接入UDM，MME在Update Location Request请求消息中会携带私有信元ue-dcnr-capability用来标识终端是否支持NR接入，UDM根据MME携带的ue-dcnr-capability信元，确定是否触发NSA自动开户。

用户进行NSA自动开户流程如 [图1](#ZH-CN_TOPIC_0000001361327113__fig1937711116810) 所示。

**图1** NSA自动开户流程

<br>

![](WSFD-011522 NSA自动开户特性概述_61327113.assets/zh-cn_image_0000001360967005_2.png)

1. 4G用户首次使用5G终端，终端能力和网络匹配，UE向MME发起5G NSA网络附着请求Attach Request消息。
2. UDM检查本地已有用户的EPS签约数据，完成鉴权流程。
3. 鉴权成功后，MME向UDM发送Update Location Request消息，消息中携带私有信元ue-dcnr-capability。
  *表1 Update Location Request消息中的关键信元*

  | 关键信元 | 信元解释 |
  | --- | --- |
  | ![](WSFD-011522 NSA自动开户特性概述_61327113.assets/zh-cn_image_0000001308047618_2.png) | ue-dcnr-capability：标识终端是否支持NR接入。取值如下：<br>ue-dcnr-supported (1) |
4. UDM接收到Update Location Request消息后，判断Update Location Request消息AVP中是否携带ue-dcnr-capability信元且取值为ue-dcnr-supported (1)。
    - 携带信元且取值为“ue-dcnr-supported (1)”：继续判断是否满足其他NSA自动开户的条件。若其他所有条件都满足，则进行NSA自动开户，且UDM向业务发放系统发起自动开户通知消息，同时将用户状态修改为已发送通知消息的NSA自动开户用户；否则NSA自动开户流程结束，附着流程会继续附着完成。
    - 未携带ue-dcnr-capability信元：不进行NSA自动开户，NSA自动开户流程结束，附着流程会继续附着完成。
5. UDM向MME返回Update Location Answer消息。
6. MME向UE返回附着成功响应消息Attach Accept。
