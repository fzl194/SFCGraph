#### [适用NF](#ZH-CN_TOPIC_0192480013)

SGSN

#### [定义](#ZH-CN_TOPIC_0192480013)

R98系统Gn接口采用GTPv0（GPRS Tunneling Protocol version 0）规范而R99系统Gn接口采用GTPv1（GPRS Tunneling Protocol version 1）规范。 UNC 同时支持GTPv0和GTPv1两种规范及两者之间的转换和映射。

#### [客户价值](#ZH-CN_TOPIC_0192480013)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 兼容GTPv0和GTPv1，增强了接口互通能力，降低对对端设备的要求，增加了组网灵活性。 |
| 用户 | 用户不感知该特性。 |

#### [应用场景](#ZH-CN_TOPIC_0192480013)

Gn接口同时支持GTPv0和GTPv1是SGSN默认支持的特性，适用于GPRS、UMTS R99/R4所有市场。

#### [原理概述](#ZH-CN_TOPIC_0192480013)

**相关概念**

- Gn/Gp接口：Gn和Gp接口是指UMTS/GPRS骨干网中GSN（GPRS Support Node）节点之间的接口。Gn接口是同一个PLMN内GSN节点间的接口，Gp接口是不同的PLMN内GSN间的接口。
- GTP：GTP是GPRS遂道协议，GTP协议包含控制面协议GTP-C（GTP Control Plane）和用户面协议GTP-U（GTP User Plane）两部分。GTP-C只在Gn/Gp接口使用；GTP-U在Gn/Gp接口和Iu接口使用。
  GTP-C的作用是利用隧道协议在骨干网的GSN间传输信令消息，主要信令包括路径管理、隧道管理、移动管理、位置管理，实现骨干网通道维护、会话建立/修改/删除、用户信息交互。信令要求可靠性传输。
  GTP-U的作用是利用隧道协议在骨干网的GSN间传输用户数据，使用隧道协议传输数据，屏蔽上层协议影响，同时提供会话级的双向快速传输通道。GTP协议用户面除数据包外，还包括数据传输出错指示、握手消息、支持的扩展头列表等用户面传输辅助信令。

**特性原理**

GTP协议有两个版本：v0和v1。GTP v1版本顺从协议3GPP TS 29.060；GTP v0版本顺从协议3GPP TS 09.60。

v0和v1的主要区别在于GTP消息头格式上的不同。

-
  在v0版本时遂道标识为数据面流标识（Flow Label Data I）和信令流标识（Flow Label Signalling）。v0版本的GTP消息头基本格式如 [图1](#ZH-CN_TOPIC_0192480013__zh-cn_concept_0130427936_fig1) 所示。

  **图1** GTPv0消息头

  <br>

  ![](WSFD-011004 Gn接口同时支持GTPv0和GTPv1特性概述_92480013.assets/zh-cn_image_0192533979_2.png)
  GPTv0的消息头长度固定为20个字节，各字节含义如下：

  - Version：GTP版本。
    - PT：Protocol Type，若为1则指明是GTP消息，为0则指明是GTP’消息。
    - SNN：标示是否包含SNDCP N-PDU Number。
    - Message Type：指示GTP消息的类型。
    - Length：GTP消息的长度，不含GTP头的长度。
    - Sequence Number：消息序列号。
    - SNDCP N-PDU Number：用于SGSN间路由区更新过程。
    - TID：隧道标识。
    - Flow Label：定义GTP流，暂时没有实际用处。

  > **说明**
  > GTP'协议是GSN与CGF（Charging Gateway Functionality）间以及不同CGF间的通讯协议。该协议属于应用层协议，底层的协议栈采用UDP/TCP以及IP。
-
  在v1版本时隧道标识为控制面TEID（Tunnel Endpoint IDentifier）和用户面TEID。TEID即隧道端点标识，用于GSN间、RAN和SGSN间的GPRS隧道协议。TEID可以标识接收GTP-C或GTP-U协议实体的隧道终点、一个PDP上下文或在Iu接口下的一个无线接入承载。v1版本的GTP消息头基本格式如 [图2](#ZH-CN_TOPIC_0192480013__zh-cn_concept_0130427936_fig2) 所示。

  **图2** GTPv1消息头

  <br>

  ![](WSFD-011004 Gn接口同时支持GTPv0和GTPv1特性概述_92480013.assets/zh-cn_image_0192533980_2.png)
  > **说明**
  > GTPv1消息头中(*)、(1)、(2)、(3)、(4)分别表示如下含义：
  >
  > - (*)该位保留，置为0。接收消息时不考虑该位。
  > - (1) 表示只有S标识符为1时，该字段有效。
  > - (2) 表示只有PN标识符为1时，该字段有效。
  > - (3) 表示只有E标识符为1时，该字段有效。
  > - (4) 表示只有当S，PN，E标识符中的一位或多位设置为1时，该字段有效。

由于对于只支持GTPv0的GSN来说，不能侦听GTPv1消息的端口，故对于对端发来的GTPv1消息无法作出回应；对于只支持GTPv1的GSN同理。

SGSN同时支持GTPv0和GTPv1：

- 当它收到对端发送的GTP消息时，能够根据该消息是GTPv0或GTPv1自动回复对应类型的消息。
- 当它向对端发送GTP消息时，它首先发送出GTPv1消息，如果没有收到成功响应，则默认为对端不支持v1版本，再以GTPv0的方式再发送一次消息。
