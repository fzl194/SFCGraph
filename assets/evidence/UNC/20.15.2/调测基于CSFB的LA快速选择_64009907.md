# 调测基于CSFB的LA快速选择

- [操作场景](#ZH-CN_OPI_0164009907__1.3.1)

## [操作场景](#ZH-CN_OPI_0164009907)

本操作介绍了通过调测手段检查 UNC 是否正确配置该功能的过程。

> **说明**
> 适用于MME

## [必备事项](#ZH-CN_OPI_0164009907)

前提条件

基于CSFB的LA快速选择功能已经激活，具体配置请参考 [激活基于CSFB的LA快速选择](激活基于CSFB的LA快速选择_64009906.md) 。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0164009907)

- 调测激活基于CSFB的LA快速选择业务。
    1. 进入 “MML命令行-UNC” 窗口。
    2. 创建用户跟踪。
      参数选择如下：
          - IMSI:被跟踪用户的IMSI。
          - 其它参数：选择默认值。
    3. 用户附着成功，发起CSFB业务流程。
    4. 查看跟踪结果。
          - 预期结果：CSFB回落时，MME将LAI封装在Initial Context Setup Request/UE Context Modification Request消息的Registered LAI信元中，发给eNodeB，且此LAI与LTE小区的LAI值一致，如 [图1](#ZH-CN_OPI_0164009907__fig1) 所示。
            **图1** Initial Context Setup Request消息中携带Registered LAI信元（样例）

            <br>

            ![](调测基于CSFB的LA快速选择_64009907.assets/cn_10_18_1451103_fig01_2.png "点击放大")

            <br>
          - 异常结果：Registered LAI信元不存在于Initial Context Setup Request/UE Context Modification Request消息内或者此LAI与LTE小区的LAI值不一致。
            参考 [激活特性/配置功能](激活基于CSFB的LA快速选择_64009906.md) ，查看配置是否正确。
