#### [适用NF](#ZH-CN_CONCEPT_0187065739)

SGSN

#### [定义](#ZH-CN_CONCEPT_0187065739)

Smartphone话务模型统计是指SGSN和网络优化工具协同工作，分析网络中Smartphone话务模型，从而进行准确的网络评估和规划的功能。

> **说明**
> 网络优化工具非 UNC 的标准配置软件，需要独立购买。

#### [客户价值](#ZH-CN_CONCEPT_0187065739)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 进行准确的网络评估和规划。<br>- 方便发现行为异常、对网络影响大的用户或终端类型，继而研究并制定相应的控制策略，降低对网络的影响。<br>- Smartphone可视化：- 分析现网的Smartphone终端的分布以及给出走势图，可预测客户所关注的Smartphone的趋势。<br>- 可按照客户的需求，给出客户关注的Smartphone的网络行为，分析Smartphone的话务模型。 |
| 用户 | 用户不感知该特性。 |

#### [应用场景](#ZH-CN_CONCEPT_0187065739)

GERAN和UTRAN网络中，在Smartphone终端被广泛使用的情况下，利用Smartphone话务模型统计功能可以优化网络部署，分析终端行为，从而制定相应的控制策略。

[**话务模型统计指标**](#ZH-CN_CONCEPT_0187065739__zh-cn_concept_0130429047_sub1)

Smartphone具有永远在线、快速休眠、发送心跳消息和推送机制的基本特点，请参考 [WSFD-206005 Smartphone控制基础功能](../WSFD-206005 Smartphone控制基础功能_85152745.md) 。

Smartphone的基本特点使其拥有不同于普通手机的话务模型，其具体特点根据现网情况的不同而不同，话务模型统计指标参见“相关主题-性能指标”。

[**话务模型统计流程**](#ZH-CN_CONCEPT_0187065739__zh-cn_concept_0130429047_sub1)

1. Smartphone话务模型统计功能的实现以系统已经配置了IMEI库或APN NI库，可识别出不同的终端类型。
2. 识别Smartphone终端类型
  系统通过 [**ADD IMEILIB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/Smartphone管理/终端类型识别/IMEI库管理/增加IMEI库记录（ADD IMEILIB）_26145734.md) 命令建立IMEI（International Mobile Equipment Identity）库或通过 [**ADD APNNILIB**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/Smartphone管理/终端类型识别/APNNI库管理/增加APNNI库记录（ADD APNNILIB）_26145736.md) 命令建立APN NI（Access Point Name Network Identifier）库来识别终端类型：
    - IMEI库
      系统内部建立各Smartphone终端类型与用户IMEI的对应关系数据库。移动性管理中，系统在用户首次发起的附着、RAU、Inter/Intra SGSN Relocation流程中获得用户IMEI信息，通过查询配置的IMEI库识别终端类型。如果系统在用户后续发起的附着、RAU、Inter/Intra SGSN Relocation流程中，获取到的IMEI和之前流程中获取的不同，则系统将重新查询配置的IMEI库识别终端类型。另外，系统也会在用户多次完成激活流程后，自动重新查询配置的IMEI库识别终端类型。
      > **说明**
      > IMEI的组成结构为TAC（Type Allocation Code）+SNR（Serial Number）+Spare digit，用于唯一标识一个用户终端。
    - APN NI库
      系统内部建立各Smartphone终端类型与用户APN对应关系的数据库。移动性管理中，系统通过附着、RAU、Handover等流程获得用户签约的APN；会话管理中，系统通过PDP上下文激活流程获得用户请求的APN，并且查询配置的APN NI库识别终端类型。
  > **说明**
  > IMEI库和APN NI库两者可以同时配置，优先级从高到低依次为：签约APN，IMEI，请求APN。
  系统可识别出的终端类型如下：
    - 4种已知类型：Android、Blackberry、iOS、Windows
    - 12种扩展类型：Custom Type 1～Custom Type 12
3. Smartphone话务模型统计流程如下：
    - 原始信息统计
      SGSN统计终端用户的原始行为数据，并将行为数据输出给网络优化工具，以进行数据分析。原始行为数据的具体内容请参考“相关主题-性能指标”。
    - 信息分析
      信息分析
      网络优化工具接收到SGSN发送的原始行为数据，分析后可获得不同Smartphone的话务模型、信令最多的用户、终端类型等结果，并通过CHR日志输出分析结果。
