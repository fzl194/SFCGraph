# RedCap用户4/5G互操作流程

- [信令流程](#ZH-CN_TOPIC_0000001538400088__1.3.6.1)
- [性能指标说明](#ZH-CN_TOPIC_0000001538400088__1.3.7.1)

由于5G NR目前尚未达成全地区覆盖，因此5G RedCap设备为双模终端，同时支持LTE和5G RedCap技术。即使在没有5G NR覆盖的地区，也可以使用现有的LTE网络连接。

全网络是否支持5G SA决定着5G RedCap单模的演进。长期来看，5G RedCap单模终端是发展趋势。

RedCap用户4/5G互操作场景如下：

- 5GS切换到EPS：RedCap用户基于N26口5GS到EPS重选和切换，与NR用户一致，均为标准4G接入，无特殊适配。
- EPS切换5GS:
    - [基于N26口EPS到5GS重选](#ZH-CN_TOPIC_0000001538400088__section1896023103313)：当空闲态UE从EPS覆盖区移动到5GS覆盖区，gNodeB不变，触发移动注册更新流程。AMF从gNodeB的INITIAL UE MESSAGE消息中获得RedCap指示，后续注册和移动性流程给周边网元携带的RatType取值为NR_REDCAP，详细请见[移动性注册更新流程-Inter AMF](RedCap用户接入5G核心网/注册流程/移动性注册更新流程_93049793.md#ZH-CN_TOPIC_0000001593049793__section06531330405)。
    - [基于N26口EPS到5GS切换](#ZH-CN_TOPIC_0000001538400088__section1896023103313)：AMF从gNodeB的**HANDOVER REQUEST ACKNOWLEDGE**请求中获得RedCap能力指示，后续注册和移动性流程给周边网元携带的RatType取值为NR_REDCAP。对切换中会话关联的SMF要单独发送一个**Nsmf_PDUSession_UpdateSMContext Request**消息通知RatType变更。

RedCap终端在4/5G互操作流程中，若gNodeB不支持RedCap，则以普通NR接入，其流程就同常规 [EPS切换到5GS](../../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/EPS切换到5GS_98118454.md) 流程，无特殊适配。

该流程相较于基于N26口EPS到5GS切换流程，主要差异点集中在如下步骤，其余步骤请参见 **[基于N26接口EPS到5GS切换](../../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/EPS切换到5GS/基于N26接口EPS到5GS切换_98278350.md)** 。

1. **Step 11**：AMF从HANDOVER REQUEST ACKNOWLEDGE消息中获取到RedCap Indication信元指示UE RedCap能力，并将其保存在本地用户上下文中。
  为实现此流程，AMF上的RedCap功能命令 [**SET AMFREDCAPFUNC**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G M2M管理/RedCap功能管理/设置AMF RedCap功能（SET AMFREDCAPFUNC）_17306402.md) 的参数 **REDCAPSW** 需要打开。
2. **Step 12**：AMF会通过Nsmf_PDUSession_UpdateSMContext Request消息通知SMF RatType的变更。
  为实现此流程，AMF上的RedCap功能命令 [**SET AMFREDCAPFUNC**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G M2M管理/RedCap功能管理/设置AMF RedCap功能（SET AMFREDCAPFUNC）_17306402.md) 的参数 **NTYSMFPLCY** 需配置为"IMMEDIATELY_NOTIFY_ON_CHANGE"。
3. **Step 28**：SMF通过消息Npcf_SMPolicyControlAPI_SmPolicyControlCreate Request携带RatType类型为NR_REDCAP通知PCF用户接入类型。
  为实现此流程，SMF上的RedCap开关 [**SET SMFFUNC**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/会话协议参数管理/会话管理拓展功能/5GC会话管理拓展功能/设置SMF扩展功能（SET SMFFUNC）_09653731.md) 的REDCAPSW参数需打开， [**SET REDCAPRATVALUE**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/RedCap用户RAT值/设置RedCap终端接入RAT填写值（SET REDCAPRATVALUE）_68425885.md) 的 **PCF** 参数需配置为“NR_REDCAP”。
4. **Step 34**：SMF通过消息PFCP Session Modification Request携带RatType类型为NR_REDCAP通知UPF用户接入类型。
  为实现此流程，SMF上的RedCap开关 [**SET SMFFUNC**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/会话协议参数管理/会话管理拓展功能/5GC会话管理拓展功能/设置SMF扩展功能（SET SMFFUNC）_09653731.md) 的REDCAPSW参数需打开， [**SET REDCAPRATVALUE**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/计费控制/RedCap用户RAT值/设置RedCap终端接入RAT填写值（SET REDCAPRATVALUE）_68425885.md) 的 **UPF** 参数需配置为“NR_REDCAP”。
