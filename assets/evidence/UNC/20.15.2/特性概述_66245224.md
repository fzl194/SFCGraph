#### [定义](#ZH-CN_TOPIC_0166245224)

QCI（QoS Class Identifier，QoS分类识别码）用于表示资源分配、分组调度的优先级，不同取值可以体现不同的业务QoS要求。3GPP定义了QCI取值为1~9时的标准QCI，但是标准QCI仅用于区分业务类型，并不能区分金、银、铜等用户级别。

基于用户等级的QCI扩展通过将一个标准QCI映射为多个扩展QCI，每个扩展QCI代表该业务的一个用户等级，使其不仅可以体现业务差异化，还可以体现同一业务不同用户使用时的业务体验差异。在QoS控制时，运营商可以通过扩展QCI区分多个等级的用户，为同类业务中高优先级用户提供高优先级保障。

#### [客户价值](#ZH-CN_TOPIC_0166245224)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 通过对用户群进行细分，为不同需求的用户群提供不同等级的服务，进行灵活业务控制，精细化运营。<br>- 在资源紧缺时，为高优先级用户优先分配资源；在资源相对富余时，更充分地发挥网络资源的效益，提高客户满意度。 |
| 用户 | 享受差异化的业务体验。 |

#### [应用场景](#ZH-CN_TOPIC_0166245224)

- 基于用户的差异化控制
  不同级别的用户体验相同业务时，运营商可以根据用户等级提供不同的QoS，使高级别的用户享有高优先级的服务，如准入、资源分配、调度等。基于用户等级的QoS控制，可以优先保障高价值用户的业务体验。
  运营商可以区分本地用户与漫游用户，运营商可对本地用户定义金、银、铜级别用户及免费用户通过对3GPP定义的标准QCI（1～9）进行扩展，将表示同一业务类型的标准QCI扩展为具有用户等级的多个扩展QCI，而每个扩展QCI分别属于不同的用户级别。
- 基于业务的差异化控制
  业务差分是指不同业务类型之间的QoS差别，比如流媒体业务和普通浏览业务的差异，主要体现在时延、丢包率、带宽及资源分配性质（GBR、non-GBR）等。不同类型的业务对时延、丢包、带宽及资源分配等有不同的需求，通过对不同的业务类别提供不同的QCI设置，可以达到提供不同业务类别体验质量的目的。

#### [应用限制](#ZH-CN_TOPIC_0166245224)

由于QCI扩展属于私有扩展，所以启用该特性的时候，需要全网部署功能网元eNodeB、MME、SGSN、PGW-C、SGW、PCRF和HSS支持QCI扩展。若一侧网元已开启扩展QCI License，而另一侧网元未开启扩展QCI License，可能会导致QCI处理冲突。

#### [原理概述](#ZH-CN_TOPIC_0166245224)

QCI是EPS QoS的一个重要参数，3GPP标准定义了QCI取值1~9时对应的QCI特征值，作为标准QCI。取值大于9并小于255的QCI称为扩展QCI，可以在 UNC 上配置标准QCI与扩展QCI间的映射关系。

标准QCI只和业务类型相关，和用户等级无关。有些运营商对用户进行了细分，希望不同级别的用户体验相同业务时可以根据用户等级提供不同的QoS，使高级别的用户享有高优先级的服务。由于标准QCI只和业务类型相关，无法直接区分用户等级，因此引入扩展QCI，即将一个标准QCI映射为多个扩展QCI，每个扩展QCI代表该业务的一个用户等级，从而根据扩展QCI为用户提供基于用户等级的细分服务。

UNC 支持以下QCI扩展功能：

- 支持EPS QoS QCI扩展
  支持HSS下发的用户签约信息扩展QCI，支持S-GW传递扩展QCI到MME，支持从源侧MME的承载上下文中接收扩展QCI。
- 支持差异化控制和异系统转换
  支持专有承载建立、修改，缺省承载修改流程中对eNodeB和UE的差异化控制。S1–MME接口传递网络侧下发的扩展的QCI，NAS消息中将扩展的QCI按照运营商配置转化为标准的QCI传递给UE。
- 扩展QCI功能由扩展QCI license控制
  QCI扩展功能由扩展QCI license控制。在license支持扩展QCI时实现上述功能；在license不支持扩展QCI时，MME对接收的扩展QCI值按协议处理（扩展值转换为标准值9）。
