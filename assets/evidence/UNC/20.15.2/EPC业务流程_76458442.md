# EPC业务流程

EPC网络中，Routing Behind MS的业务流程如 [图1](#ZH-CN_TOPIC_0276458442__fig243115811463) 所示。

**图1** Routing Behind MS业务流程图

<br>

![](EPC业务流程_76458442.assets/zh-cn_image_0276458820_2.png)

1. 无线设备（MS/UE）发起缺省承载建立请求。
2. MME向SGW-C发送Create Session Request消息。
3. SGW-C向SGW-U发送PFCP Session Establishment Request消息。
4. SGW-U向SGW-C回复PFCP Session Establishment Response消息。
5. SGW-C向PGW-C发送Create Session Request消息。
6. PGW-C获取到UE的缺省承载建立请求后，向AAA Server发送鉴权请求。
7. AAA Server检测UE是Routing Behind MS用户，将UE对应的PC下行路由封装在鉴权响应消息中发送给PGW-C。
8. PGW-C获取PC的下行路由，将该路由插入到PGW-C路由表中，并通过PFCP Session Establishment Request消息的Framed-Route信元携带手机后网段路由发送给PGW-U。后续，PGW-U会根据该手机后网段路由更新本地路由表，以便能找到对应的终端设备，进行下行转发，如步骤[16](#ZH-CN_TOPIC_0276458442__li1542596368175820)所示。
9. PGW-U给PGW-C响应PFCP Session Establishment Response消息。
10. PGW-C向SGW-C响应Create Session Response消息。
11. SGW-C向MME发送Create Session Response消息。
12. UE完成缺省承载建立。
13. PC通过UE发起上行业务。
14. PGW-U发现该报文源IP地址与UE的主机地址不一致，根据用户属性自动关闭Anti-spoofing功能。终端设备上行数据正常转发。
15. 网络侧发送到UE下某PC的下行数据。
16. PGW-U发现该报文目的IP地址与UE的主机地址不一致，根据网段路径将该报文关联到UE的承载上下文，再通过该承载发送到UE。UE将下行数据报文转发到目的PC。

UE发起去激活请求时，SGW-C/PGW-C可以感知UE下线，删除其对应路由信息并通知SGW-U/PGW-U。
