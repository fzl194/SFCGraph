# 调测EN-DC SON信息传送功能

- [操作场景](#ZH-CN_OPI_0000001514694133__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001514694133__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001514694133__1.3.3)

## [操作场景](#ZH-CN_OPI_0000001514694133)

当运营商部署EN-DC SON信息传送功能时，需对 UNC 的EN-DC SON信息传送功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0000001514694133)

前提条件

- 请仔细阅读[WSFD-104402 eNodeB SON信息传送功能特性概述](../特性概述_66292211.md)。
- 完成[激活 EN-DC SON信息传送功能](../激活信息传送功能/激活 EN-DC SON信息传送功能_64014514.md)。
- 完成eNodeB2与en-gNodeB之间的X2链路配置。

数据

该操作无需准备数据。

工具

测试终端

## [操作步骤](#ZH-CN_OPI_0000001514694133)

1. 进入 “MML命令行-UNC” 窗口。
2. 执行 [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) 命令查询License配置开关是否已打开。
    - 如果“SWITCH”为“ENABLE”，请执行[3](#ZH-CN_OPI_0000001514694133__cmd12357205912417)。
    - 如果“SWITCH”为“DISABLE”，请执行[**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)命令打开本特性对应的License配置开关。
3. 执行 **[LST S1CMPT](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1接口兼容性/查询S1接口兼容性(LST S1CMPT)_26146238.md)** 命令查询EN-DC SON开关是否已经打开。
    - 如果“ENDCSON”为“SUPPORT”，请执行[4](#ZH-CN_OPI_0000001514694133__cmd18271414345)。
    - 如果“ENDCSON”为“NOT_SUPPORT”，请执行**[SET S1CMPT](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1接口兼容性/设置S1接口兼容性(SET S1CMPT)_72345837.md)**命令打开本特性对应的EN-DC SON开关。
4. 执行 **DSP ENBGNBNEIBS** 命令查询eNodeB2邻接的en-gNodeB是否为当前组网中的en-gNodeB。
    - 如果“en-gNB ID”为目标en-gNodeB ID，请执行[5](#ZH-CN_OPI_0000001514694133__cmd19480755172748)。
    - 如果“en-gNB ID”不为目标en-gNodeB ID，请重新检查eNodeB2与en-gNodeB之间X2链路配置。
5. **Intra MME场景** ：
    a. 确认en-gNodeB、eNodeB1、eNodeB2配置在MME2下（使用[**DSP S1APLNK**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1AP链路/显示S1AP连接状态(DSP S1APLNK)_26146252.md)命令确认），并且en-gNodeB、eNodeB2之间已配置X2接口，en-gNodeB、eNodeB1未配置X2接口。
    b. MME建立分别与eNodeB1、eNodeB的S1接口跟踪。
    c. 无线侧触发eNodeB1进行X2地址学习流程。
    d. 观察MME上eNodeB1的接口跟踪出现eNB Configuration Transfer消息，以及消息中的EN-DC SON Configuration Transfer信元，检查该信元中包含Target en-gNodeB ID是否标示为en-gNodeB；并进一步检查EN-DC SON Configuration Transfer信元是否携带了Target eNodeB ID：
          - 若携带了Target eNodeB ID：检查Target eNodeB ID是否标示为eNodeB2，并进一步观察eNodeB2接口跟踪是否向en-gNodeB发起X2AP EN-DC Configuration Transfer消息。
          - 若未携带Target eNodeB ID：直接观察eNodeB2接口跟踪是否向en-gNodeB发起X2AP EN-DC Configuration Transfer消息。
    e. 观察MME上eNodeB1的接口跟踪出现MME Configuration Transfer消息，消息中携带X2 TNL Configuration Info信元，其中包含eNodeB1与en-gNodeB建链所需的IP、端口号等信息。
6. **Inter MME场景** ：
    a. 确认eNodeB1配置在MME1下，eNodeB2配置在MME2下（使用[**DSP S1APLNK**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/S1AP链路/显示S1AP连接状态(DSP S1APLNK)_26146252.md)命令确认），并且en-gNodeB、eNodeB2之间已配置X2接口，en-gNodeB、eNodeB1未配置X2接口；在MME1上打开eNodeB1，在MME2上打开eNodeB2的接口跟踪。
    b. 打开该S10接口的GTPC跟踪（在MME1或MME2上打开都可）。
    c. 无线侧触发eNodeB1进行X2地址学习流程。
    d. 观察MME1上eNodeB1的接口跟踪出现eNB Configuration Transfer消息，以及消息中的Target en-gNodeB ID是否标示为en-gNodeB；并进一步检查EN-DC SON Configuration Transfer信元是否携带了Target eNodeB ID：
          - 若携带了Target eNodeB ID：检查Target eNodeB ID是否标示为eNodeB2，并进一步观察eNodeB2接口跟踪是否向en-gNodeB发起X2AP EN-DC Configuration Transfer消息。
          - 若未携带Target eNodeB ID：直接观察eNodeB2接口跟踪是否向en-gNodeB发起X2AP EN-DC Configuration Transfer消息。
    e. 观察MME1与MME2的S10接口跟踪出现Configuration Transfer Tunnel消息，其中的Target en-gNodeB ID信元是否与eNB Configuration Transfer消息中信元一致。
    f. 观察MME2上eNodeB2的接口跟踪上出现MME Configuration Transfer消息，其中的EN-DC SON Configuration Transfer信元与之前eNB Configuration Transfer消息中的信元一致。
