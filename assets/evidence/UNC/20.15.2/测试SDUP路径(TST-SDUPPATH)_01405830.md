#### [命令功能](#ZH-CN_CONCEPT_0000002101405830)

**适用网元：MME**

该命令用于通过发送Echo消息的方法测试本端与对端MME之间的SDUP路径是否正常。无论路径正常与否，都返回报文显示路径地址信息及路径状态；

#### [注意事项](#ZH-CN_CONCEPT_0000002101405830)

- 该命令执行后立即生效。
- 该命令执行过程中会产生一条临时的维护路径，执行结束后删除该维护路径。该命令只探测路径是否正常，与该路径上的信令消息无关。
- 执行此命令时需要确保本端与对端均通过[**ADD SDAPLE**](../Sdup接口业务地址管理/增加SDAP本地实体(ADD SDAPLE)_26147290.md)命令配置SDUP链路的逻辑地址。
- 命令执行过程中如果发生进程复位，探测结果无法保证正确，请在进程正常后再次进行探测。
- 使用此命令需要开启License“82207526 LKV2MCR01 MME链式备份”。

#### [使用实例](#ZH-CN_CONCEPT_0000002101405830)

测试SDUP路径：

TST SDUPPATH: IPTYPE=IPV4, LOCIPV4ADDR="172.16.26.99", PEERIPV4ADDR="172.16.28.99";

```
%%TST SDUPPATH: IPTYPE=IPV4, LOCIPV4ADDR="172.16.26.99", PEERIPV4ADDR="172.16.28.99";%%
RETCODE = 0  操作成功

操作结果如下：
-------------------------
本端地址  =  172.16.26.99
对端地址  =  172.16.28.99
路径状态  =  路径正常
(结果个数 = 1)

---    END
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000002101405830)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 本端地址 | 参数含义：该参数指示测试的Sdup链路本端IP。<br>取值范围：<br>- 当“IP地址类型”设置为“IPV4（IPV4）”时为0.0.0.0～255.255.255.255。<br>- 当“IP地址类型”设置为“IPV6（IPV6）”时为::～FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF。 |
| 对端地址 | 参数含义：该参数指示测试的Sdup链路对端IP。<br>取值范围：<br>- 当“IP地址类型”设置为“IPV4（IPV4）”时为0.0.0.0～255.255.255.255。<br>- 当“IP地址类型”设置为“IPV6（IPV6）”时为::～FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF:FFFF。 |
| 路径状态 | 参数含义：该参数指示测试的Sdup链路路径状态。<br>取值范围：<br>- “路径正常”：表示本端收到对端响应，本端到对端的SDUP路径正常。<br>- “路径故障”：表示本端未收到对端响应，本端到对端的SDUP路径故障。<br>- “本端Sdaple不存在”：表示所输入本端地址未通过[**ADD SDAPLE**](../Sdup接口业务地址管理/增加SDAP本地实体(ADD SDAPLE)_26147290.md)命令配置。 |
