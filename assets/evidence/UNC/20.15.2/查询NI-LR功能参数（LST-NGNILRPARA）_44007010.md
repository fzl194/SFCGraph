## [命令功能](#ZH-CN_MMLREF_0244007010)

**适用NF：AMF**

该命令用于查询指定运营商的NI-LR功能的参数。

## [注意事项](#ZH-CN_MMLREF_0244007010)

无

#### [操作用户权限](#ZH-CN_MMLREF_0244007010)

G_1，管理员级别命令组；G_2，操作员级别命令组

## [参数说明](#ZH-CN_MMLREF_0244007010)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| NOID | 运营商标识 | 可选必选说明：可选参数<br>参数含义：该参数用于在UNC系统内唯一标识移动网络运营商。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围为0。<br>默认值：无<br>配置原则：<br>该参数取值必须和ADD NGMNO中配置的“NOID”参数取值相同。 |

## [使用实例](#ZH-CN_MMLREF_0244007010)

查询NOID为0的NI-LR功能参数，执行命令如下：

```
%%LST NGNILRPARA: NOID=0;%%
RETCODE = 0  操作成功

结果如下
--------
                  运营商标识  =  0
    是否允许网络触发定位功能  =  否
          网络触发的定位策略  =  协议模式定位
    紧急呼叫释放是否上报位置  =  否
                    水平精度  =  19
                    响应时间  =  低时延
          是否对GMLC进行重发  =  否
                是否重选GMLC  =  否
紧急会话切换的位置连续性开关  =  否
                切换上报类型  =  源侧上报
               LCS QoS Class  =  无效值
                    描述信息  =  NULL
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0244007010)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 运营商标识 | 该参数用于在UNC系统内唯一标识移动网络运营商。 |
| 是否允许网络触发定位功能 | 该参数用于AMF控制紧急呼叫场景是否发起定位流程。 |
| 网络触发的定位策略 | 该参数用于AMF配置紧急呼叫场景的定位策略。 |
| 紧急呼叫释放是否上报位置 | 该参数用于AMF控制紧急呼叫释放场景是否上报位置。 |
| 水平精度 | 该参数用于AMF配置定位请求的水平精度。 |
| 响应时间 | 该参数用于AMF配置定位请求消息的Response Time。 |
| 是否对GMLC进行重发 | 该参数用于指定AMF在NI-LR定位释放流程中，当AMF向GMLC发送Namf_Location_EventNotify Request消息，且GMLC返回5xx原因值时，AMF是否向该GMLC重发1次业务请求。 |
| 是否重选GMLC | 该参数用于指定当AMF在NI-LR定位触发位置上报流程中，当首次交互的GMLC返回5xx时，AMF是否重新选择新的GMLC进行业务请求。 |
| 紧急会话切换的位置连续性开关 | 该参数用于控制AMF是否支持紧急会话切换的位置连续性。 |
| 切换上报类型 | 该参数用于控制切换流程中由源侧AMF/MME还是目标侧AMF/MME上报切换通知信息。<br>对于Intra AMF N2切换流程，不受本参数控制。 |
| LCS QoS Class | 该参数用于配置AMF定位请求是否携带lcsQosClass。 |
| 描述信息 | 该参数用于设置NI-LR参数的描述信息。 |
