# 调测5G SA到LTE网络间重选

- [操作场景](#ZH-CN_OPI_0190799284__1.3.1)
- [必备事项](#ZH-CN_OPI_0190799284__1.3.2)
- [操作步骤](#ZH-CN_OPI_0190799284__1.3.3)

## [操作场景](#ZH-CN_OPI_0190799284)

当系统支持基于N26接口的5GS到EPS的重选，保持业务连续性，需要对本功能进行调测，确保功能可以正常使用。

> **说明**
> 适用于MME、SGW-C、PGW-C、AMF、SMF

## [必备事项](#ZH-CN_OPI_0190799284)

前提条件

- 请仔细阅读**LTE和5G SA网络间重选**实现原理。
- 用户在HSS+UDM中已签约EPC和5GC业务。
- MME和AMF配置完成N26接口，完成[基于N26接口5GC互操作的新建部署](../../../../业务专题/5G Core 4_5G互操作解决方案/新建部署指导_01291730.md)。

数据

该操作无需准备数据。

工具

- 测试终端
- OM Portal跟踪

## [操作步骤](#ZH-CN_OPI_0190799284)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) 查询LTE和5G SA网络间重选License配置开关是否打开。
  [**LST LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) :LICITEM="LKV2NRLSAM01";
  [**LST LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) :LICITEM="LKV2NRLSSM01";
  [**LST LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) :LICITEM="LKV2NRBL5";
    - 如果“SWITCH”为“ENABLE”，请执行[2](#ZH-CN_OPI_0190799284__cmd105004011261)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)命令打开本特性对应的License配置开关。
2. 验证UE从5G网络向LTE网络重选可实现。
    a. 在MME的OM Portal上创建用户跟踪任务。
    b. UE在5G网络注册，并且建立了PDU会话，后释放AN，处于空闲态。UE移动过程中从5G覆盖区域移动到EPS覆盖区域时，UE执行TAU流程。查看MME上用户消息跟踪。
      预期结果：
          - TAU Request消息中的UE Network Capability IE携带N1 MODE标识，指示UE支持N1 mode，如[图1](#ZH-CN_OPI_0190799284__fig151571026142912)所示。TAU Request消息会携带UE STATUS信元，表示UE曾经在5G网络注册过。另外，TAU Request中携带的old Native GUTI由5G-GUTI映射而来，如[图2](#ZH-CN_OPI_0190799284__fig12601493135)和[图3](#ZH-CN_OPI_0190799284__fig82051546191313)所示。
          - MME向AMF发送Context Request消息，获取上下文成功。Context Response消息包含MM，SM上下文，消息中RAT TYPE为NR，如[图4](#ZH-CN_OPI_0190799284__fig73751922151518)所示。
          - MME向SGW发送Create Session Reuqest请求建立承载，且SGW和PGW-C交互后，返回Create Session Response，完成承载创建相应。如[图5](#ZH-CN_OPI_0190799284__fig525253216429)所示。
          - MME向HSS/UDM发送Update Location请求，HSS/UDM向AMF发送Nudm_UEContextManagement_DeregistrationNotification消息。如[图6](#ZH-CN_OPI_0190799284__fig172761228114312)所示。
