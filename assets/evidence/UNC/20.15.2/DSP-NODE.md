#### [命令功能](#ZH-CN_TOPIC_0171678755)

本命令用于查询部署的节点配置信息。

若需要查询指定节点的配置信息，需要输入“网元ID”和“节点名称”参数。

> **说明**
> 该命令仅在Full-stack虚机场景下支持。

#### [注意事项](#ZH-CN_TOPIC_0171678755)

- 扩容完成后，执行此命令查询到的主机名称可能展示的是主机ID，下一个0点或12点系统会更新为正确的主机名称。
- 在强制重建虚拟机过程中，执行此命令查询到的节点信息有可能是重建前的节点信息。

#### [参数说明](#ZH-CN_TOPIC_0171678755)

| **参数标识** | **参数名称** | **参数说明** |
| --- | --- | --- |
| APPID | 网元ID | 可选必选说明：可选参数<br>参数含义：用于指示系统需要查询指定网元ID下的节点数据。当该参数与<br>“节点名称”<br>均为空时查询所有网元下的所有节点数据。<br>“网元ID”<br>可以通过点击<br>“首页”<br>查看<br>“应用ID”<br>获取。<br>取值范围：0～65535。<br>默认值：无。<br>配置原则：无。 |
| NODENAME | 节点名称 | 可选必选说明：可选参数<br>参数含义：用于指示系统需要查询哪个节点的数据。当该参数与<br>“网元ID”<br>均为空时查询所有网元下的所有节点数据；仅该参数为空时查询指定网元下的所有节点数据。操作员在对应FusionStage页面<br>“资源管理>节点管理>节点”<br>查看相应的节点获得。<br>取值范围：不超过128位字符串。<br>默认值：无。<br>配置原则：无。 |

#### [使用实例](#ZH-CN_TOPIC_0171678755)

1. “网元ID”和“节点名称”均为空时，查询所有网元下所有节点的配置信息，结果以进度上报形式展示，此处只展示最终结果数据：
  ```
  %%DSP NODE:;%%
  RETCODE = 0  操作成功  

  操作结果如下 
  ------------ 
  网元ID  虚拟机类型        节点名称       节点IP         虚拟机名称                主机名称      运行状态  管理状态    
  0       CSP_B             10.0.0.3       10.0.0.3       os-5g-3_A_0               M1-08-Slot15  Normal    Active     
  0       CSP_C             10.0.0.2       10.0.0.2       os-5g-3_B_0               M1-08-Slot8   Normal    Active     
  0       CSP_A             10.0.0.1       10.0.0.1       os-5g-3_A_1               M1-07-Slot7   Normal    Active
  (结果个数 = 5)  

  共有5个报告
  ---    END
  ```
2. 查询“网元ID”为“0”的所有节点配置信息，结果以进度上报形式展示，此处只展示最终结果数据：
  ```
  %%DSP NODE: APPID=0;%% 
  RETCODE = 0  操作成功  

  操作结果如下 
  ------------ 
  网元ID  虚拟机类型  节点名称       节点IP         虚拟机名称         主机名称     运行状态  管理状态    
  0       CSP_B       10.0.0.3       10.0.0.3       os-5g-3_A_0        M1-08-Slot15 Normal    Active     
  0       CSP_C       10.0.0.2       10.0.0.2       os-5g-3_B_0        M1-08-Slot8  Normal    Active     
  0       CSP_A       10.0.0.1       10.0.0.1       os-5g-3_A_1        M1-07-Slot7  Normal    Active 
  (结果个数 = 3)  

  共有5个报告
  ---    END
  ```
3. 查询“节点名称”为“10.0.0.1”的节点配置信息。指定“节点名称”时，可以选择指定“网元ID”或不指定“网元ID”。两种情况的操作结果相同，此处展示了指定“网元ID”为“0”的情况。结果以进度上报形式展示，此处只展示最终结果数据：
  ```
  %%DSP NODE: APPID=0, NODENAME="10.0.0.1";%% 
  RETCODE = 0  操作成功  

  操作结果如下 
  ------------       
      网元ID  =  0
  虚拟机类型  =  CSP_A
    节点名称  =  10.0.0.1      
      节点IP  =  10.0.0.1   
  虚拟机名称  =  os-5g-3_A_0     
    主机名称  =  M1-08-Slot15     
    运行状态  =  Normal     
    管理状态  =  Active 
  (结果个数 = 1)  

  共有5个报告
  ---    END
  ```

#### [输出结果说明](#ZH-CN_TOPIC_0171678755)

命令执行正常，会返回命令执行成功的提示信息。输出结果说明如 [表1](#ZH-CN_TOPIC_0171678755__table819976173217) 所示。

命令执行异常，会返回提示对应的失败信息。错误码列表如 [表2](#ZH-CN_TOPIC_0171678755__table698716490471) 所示。

*表1 输出项说明*

| 输出项名称 | 输出项解释 |
| --- | --- |
| 网元ID | 用于返回查询网元的ID。 |
| 虚拟机类型 | 用于返回虚拟机的类型 |
| 节点名称 | 用于返回查询的节点名称。 |
| 节点IP | 用于返回查询节点的IP地址。 |
| 虚拟机名称 | 用于返回查询虚拟机的名称。 |
| 主机名称 | 用于返回查询虚拟机所在主机名称（对于第三方虚拟化层该字段为空）。 |
| 运行状态 | 用于返回查询的节点运行状态。<br>取值范围：<br>- “Normal”: 虚拟机处于正常启动运行状态。<br>- “Unavailable”：虚拟机处于异常状态。<br>- “Stopped”：虚拟机处于关机状态。<br>- “Rebuild”：虚拟机处于重建状态。<br>- “Error”：虚拟机处于因存储故障导致的异常状态。<br>- “Zombie”：虚拟机处于自愈导致的异常状态。<br>- “Unknown”：虚拟机处于未知状态。<br>- “NotInstall”：虚拟机处于未纳管状态。说明：“NotInstall”<br>取的是系统缓存的节点状态而不是节点实时的状态。若系统中纳管失败的节点超过10个，仅10个纳管失败的节点显示<br>“NotInstall”<br>，其余的节点不刷新纳管失败的状态。 |
| 管理状态 | 用于返回查询的节点的I层状态。<br>取值范围：<br>- “Active”：虚拟机处于正常启动运行状态。<br>- “Stopped”：虚拟机处于关机状态。<br>- “Error”：虚拟机处于因存储故障导致的异常状态。<br>- “zombie”：虚拟机处于自愈导致的异常状态。<br>- “Rebuilding”：虚拟机处于重建状态。<br>- “Unknown”：虚拟机处于未知状态。 |
