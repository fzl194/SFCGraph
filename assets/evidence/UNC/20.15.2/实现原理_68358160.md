# 实现原理

基于LTE的GWCN网络共享模式下的主要业务流程包括：附着流程、Handover流程。

**附着流程**

附着流程如 [图1](#ZH-CN_CONCEPT_0168358160__fig2) 所示。

**图1** 附着流程

<br>

![](实现原理_68358160.assets/zh-cn_image_0192166564_2.png)

1. UE获取接入网侧的系统广播消息，并解析系统消息中的共享网络信息，获取候选PLMN ID列表。
2. UE在候选PLMN ID列表选择服务网络的PLMN ID。UE本身会维护“Forbidden PLMN List”，首先排除“Forbidden PLMN List”中的PLMN；然后会根据用户配置、运营商配置、支持的接入技术等因素对PLMN进行优先级排序，UE按照优先级选择PLMN。
3. 选择PLMN ID后，UE发起Attach Request请求，携带选择的PLMN。
4. eNodeB收到Attach Request消息后，根据携带的PLMN构造TAI信息，然后封装“Initial UE Message”消息发送给MME。MME收到“Initial UE Message”消息后，判断消息中携带的Selected PLMN否为允许接入的PLMN，如果允许则使用该PLMN作为UE的Serving PLMN；如果不允许，则拒绝接入。
5. MME、eNodeB、HSS、UE间进行其他流程，如：鉴权流程、位置更新流程、获取签约数据等，使用的PLMN都是MME选择的Serving PLMN。
6. 经过鉴权、接入控制等处理后，MME如果允许UE接入，则MME发送通过“Initial Context Setup Request”消息封装Attach Accept消息给eNodeB；如果不允许，则发Attach Reject消息给eNodeB。发给eNodeB的消息中的TAI、GUTI等信息中的PLMN是Serving PLMN。
7. eNodeB将Attach Accept/Reject消息发送给UE。

**Handover流程**

Handover流程如 [图2](#ZH-CN_CONCEPT_0168358160__fig3) 所示。

**图2** Handover流程

<br>

![](实现原理_68358160.assets/zh-cn_image_0192166563_2.png)

1. Source eNodeB向Target eNodeB发起S1-based handover，原因可能如下：
  到Target eNodeB没有X2连接。
  目标侧eNodeB告知Source eNodeB之前的X2-based handover失败。
  Source eNodeB收到动态信息。
2. Source eNodeB向Source MME发送Handover Required消息请求切换。
3. Source MME选择Target MME，并向Target MME发送Forward Relocation Request消息。**Source MME会在S10接口发往Target MME的Forward Relocation Request消息中携带Selected PLMN信元，该信元的信息来自于Source eNodeB在Handover Required消息中为UE选择的Target PLMN。**
4. **Target MME收到Forward Relocation Request消息后，首先判断消息中是否包含Selected PLMN信元。如果Forward Relocation Request消息中包含Selected PLMN信元，Target MME判断该PLMN是否合法。**
    - 如果合法，且MME根据漫游策略、接入策略控制判断允许用户接入，则正常执行Handover流程操作，使用该Selected PLMN作为Serving PLMN，并把该Serving PLMN下发给Target eNodeB。否则拒绝Handover流程。
    - 如果不合法，则判断Target ID信元中eNodeB ID的PLMN是否合法：如果合法，且根据漫游策略、接口控制策略判断允许用户接入，则使用该PLMN作为Serving PLMN，并把该Serving PLMN下发给Target eNodeB；否则拒绝Handover流程。
5. Target eNodeB向Target MME发送Handover Request Acknowledge消息。
6. 后续流程和Handover公共流程相同。
