# APN选择

MME根据UE请求参数和HSS签约参数进行APN选择，用于激活PDN连接。APN由两部分组成（3GPP TS 23.003）：

- APN NI（Network Identifier）
  APN NI是APN中的必选部分，用于标识需要接入的外部数据网络的类型。APN NI需要在HSS中进行了签约，激活时才允许使用该APNNI。
  APN NI是由一个标记或用点分隔的多个标记组成，例如“huawei.com”或 “service.huawei.com”。
- APN OI（Operator Identifier）
  APN OI是APN的可选部分，用于标识运营商类型。APN OI的格式完全符合DNS命令规范，由三个标记组成。APN OI使用".gprs"结尾，对于每个运营商，存在一个缺省值的APN OI，此APN OI由IMSI导出，形式为“MNCxxx.MCCyyy.gprs”或“apn.epc.MNCxxx.MCCyyy.3gppnetwork.org”。

MME收到UE的PDN连接建立请求消息后，解析出激活请求消息中的PDN类型、PDN地址和APN信息三元组，与签约三元组进行匹配。根据用户请求消息中三种参数的不同组合，MME进行APN选择分以下几种场景：

- 用户请求消息中包含PDN type， PDN address， APN
  如果签约数据中包含请求的PDN type、PDN address和APN，MME选择请求的APN。
- 用户请求消息中包含PDN type， PDN address
  如果存在一组签约数据中包含请求的PDN type对应的PDN address，MME选择签约数据中的APN。
  如果存在两组及以上签约数据包含请求的PDN type对应的PDN address，MME选择缺省APN。
- 用户请求消息中包含PDN type， APN
  签约数据中包含请求的PDN type对应的请求的APN，MME选择请求的APN。如果没有签约数据记录，只有野卡APN记录，MME选择UE请求的APN。
  > **说明**
  > 野卡：也叫通配符，指签约时不限制业务类型，签约APN为*，可以访问任何PDP/PDN Address、使用任何业务的卡。
  >
  > 野卡APN：如果HPLMN运营商允许用户接入给定PDN类型的任何网络，则可以给该用户增加签约野卡APN。
- 用户请求消息中包含PDN type
  如果签约数据中只包含一条带有野卡APN记录的PDN上下文，只需要MME上已经配置了缺省APN，MME会选择使用这个缺省APN。
- 用户请求消息中三个参数都不包含
  如果签约数据中只包含一条PDN上下文记录，且签约的PDN上下文记录中包含PDN address（或还有APN），MME会选择签约的APN。
  如果签约数据中只包含一条PDN上下文记录，且签约的PDN上下文记录中没有PDN address只有野卡APN，只需要MME上已经配置了缺省APN，MME会选择使用这个缺省APN。
  如果签约数据中只包含一条PDN上下文记录，且签约的PDN上下文记录中没有PDN address但有APN，MME会选择签约的APN。

**APN NI的选择**

APN NI部分的选择获取方式如下：

如果按照3GPP 23.060 附录A，APN选择模式为“ChosenBySGSN”时，需要使用MME配置的APN NI，则按照以下步骤进行：

1. 查询IMSI对应的缺省APN NI，如果存在，使用该APN NI。
2. 查询PDN TYPE对应的缺省APN NI，如果存在，使用该APN NI。
  > **说明**
  > “ChosenBySGSN”模式是指 UNC 提供APN，并且此APN未被签约。

**APN OI的选择**

APN OI按照特定的规则进行选择。

运营商往往希望控制用户使用拜访地或归属地的P-GW。 UNC 支持使用本地策略决定是否允许外网用户使用VPLMN分配的地址，从而确定APN OI的选择模式并判断用户的漫游属性，形成APN域名列表，用户可查询此列表获得相应的APN到DNS解析。APN OI选择模式的流程如 [图1](#ZH-CN_TOPIC_0175597698__fig040) 所示。

> **说明**
> 在软参DWORD_EX5 BIT17设置为1的情况下，APN OI列表成员的优先级顺序和使用方式说明如下：
>
> - H-HPLMN only：只允许本网用户接入HPLMN网络选择网关。形成的APN OI列表依次包含本地配置的APN OI（[**ADD GWSELPLCY**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md)）、签约APN级的APN OI Replacement、签约用户级的APN OI Replacement及IMSI提取的MCC/MNC组成的APN OI。在UE携带APN OI的场景下，如果签约APN级的APN OI Replacement、签约用户级的APN OI Replacement及IMSI提取的MCC/MNC组成的APN OI与UE携带的APN OI相同，则保留在列表中，否则，不进入APN OI列表。
> - C-HPLMN only：只允许外网用户接入HPLMN网络选择网关。形成的APN OI列表依次包含签约APN级的APN OI Replacement、签约用户级的APN OI Replacement及IMSI提取的MCC/MNC组成的APN OI。在UE携带APN OI的场景下，如果签约APN级的APN OI Replacement、签约用户级的APN OI Replacement及IMSI提取的MCC/MNC组成的APN OI与UE携带的APN OI相同，则保留在列表中，否则，不进入APN OI列表。
> - C-VPLMN only：只允许外网用户接入VPLMN网络选择网关。形成的APN OI列表依次包含本地配置的APN OI（[**ADD GWSELPLCY**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md)）及[**ADD HPLMN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/归属网络运营商管理/MNO管理/MNO网络配置表/增加本地PLMN(ADD HPLMN)_26146074.md)配置的APN OI。在UE携带APN OI的场景下，如果[**ADD HPLMN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/归属网络运营商管理/MNO管理/MNO网络配置表/增加本地PLMN(ADD HPLMN)_26146074.md)配置的APN OI与UE携带的APN OI相同，则保留在列表中，否则，不进入APN OI列表。
> - C-VPLMN first：允许外网用户首先接入VPLMN网络选择网关，仅当用户的APN到DNS解析失败时，可以允许外网用户再接入HPLMN网络选择网关。形成的APN OI列表依次包含C-VPLMN only模式+C-HPLMN only的APN OI列表。
>
> 根据以上四种情况得到APN域名列表后，按列表顺序依次将APN域名到DNS解析，直至解析成功。
>
> 签约APN级的APN OI Replacement在 UNC 上是否生效受命令 [**SET SMFUNC**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/SM扩展功能管理/设置会话管理扩展功能(SET SMFUNC)_26145684.md) 中的参数 “APNOIPLY” 的控制。如果该参数设置为NO，签约APN级的APN OI Replacement不会出现在APN OI列表中。
>
> 在软参DWORD_EX5 BIT17设置为0的情况下， UNC 仅根据本地配置的APN OI（ [**ADD GWSELPLCY**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md) ）进行DNS解析，不再根据上述列表进一步进行DNS解析。

流程的具体说明如下：

1. 判断签约数据匹配得到的APN NI是否由 UNC 提供。
  目的：当签约数据匹配得到的APN NI由 UNC 提供时，限制外网用户只能使用拜访地网络。
  出现签约数据匹配得到的APN NI由 UNC 提供的场景：匹配到的签约APN NI是野卡且UE没有携带请求的APN NI。
    - 是 => [判断用户是否为本网用户。](#ZH-CN_TOPIC_0175597698__p1691190141180638)
    - 否 => [根据用户IMSI的MCC/MNC是否在ADD HPLMN配置列表中，判断用户是否为本网用户。](#ZH-CN_TOPIC_0175597698__p912677673180638)
2. 根据用户IMSI的MCC/MNC是否在 [**ADD HPLMN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/归属网络运营商管理/MNO管理/MNO网络配置表/增加本地PLMN(ADD HPLMN)_26146074.md) 配置列表或者 [**ADD MVNONET**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/归属网络运营商管理/MVNO管理/MVNO网络标识配置表/增加MVNO网络配置信息(ADD MVNONET)_72345663.md) 配置列表中，判断用户是否为本网用户。
    - 是 => APN OI模式为H-HPLMN only，漫游属性为Home。
    - 否 => [3](#ZH-CN_TOPIC_0175597698__step3_3)
  > **说明**
  > 由IMSI提取MCC/MNC时，参考 [**ADD HPLMN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/归属网络运营商管理/MNO管理/MNO网络配置表/增加本地PLMN(ADD HPLMN)_26146074.md) 、 [**ADD CONNECTPLMN**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/互联PLMN管理/增加互联PLMN(ADD CONNECTPLMN)_26305852.md) 、 [**ADD MVNONET**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/归属网络运营商管理/MVNO管理/MVNO网络标识配置表/增加MVNO网络配置信息(ADD MVNONET)_72345663.md) 命令配置的MCC/MNC，按照最大匹配原则进行提取。
3. 根据 [**ADD GWSELPLCY**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md) 命令配置结果判断是否允许外网用户使用VPLMN分配的地址。
    - 是 => [4](#ZH-CN_TOPIC_0175597698__step3_4)
    - 否 => [5](#ZH-CN_TOPIC_0175597698__step3_5)
    - 标准方式 => [3.a](#ZH-CN_TOPIC_0175597698__step3_3_a)
  由于运营商无法控制外网用户的签约数据，在标准方式中需要进行以下判断来确认用户的APN OI模式。
    a. 查看用户是否签约允许使用VPLMN地址。
          - 是 => [3.b](#ZH-CN_TOPIC_0175597698__step3_3_b)
          - 否 => [5](#ZH-CN_TOPIC_0175597698__step3_5)
    b. 查看用户是否签约ODB（Operator-determined Barring），限制用户从VPLMN接入点接入网络。
          - 是 => [5](#ZH-CN_TOPIC_0175597698__step3_5)
          - 否 => [4](#ZH-CN_TOPIC_0175597698__step3_4)
4. 当允许外网用户使用VPLMN分配的地址时，系统是否禁止其从HPLMN接入点接入网络（通过 [**SET SDCFG**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/签约数据管理/签约数据信息/设置签约数据配置(SET SDCFG)_72225433.md) 命令中 **RAH** 参数值控制）。
    - 是 => APN OI模式为C-VPLMN only，漫游属性为Visiting
    - 否 => APN OI模式为C-VPLMN first，接入VPLMN网络时的漫游属性为Visiting，如果到用户的APN到DNS解析失败，则接入HLPMN网络，漫游属性为Roaming
5. 当禁止外网用户使用VPLMN分配的地址时，系统是否禁止其从HPLMN接入点接入网络（通过 [**SET SDCFG**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/签约数据管理/签约数据信息/设置签约数据配置(SET SDCFG)_72225433.md) 命令中 **RAH** 参数值控制）。
    - 是 => APN OI模式为C-HPLMN only，漫游属性为Roaming
    - 否 => 不允许外网用户接入网络
6. 判断用户是否为本网用户。
    - 是 => APN OI模式为H-HPLMN only，漫游属性为Home
    - 否 => 判断流程和 [3](#ZH-CN_TOPIC_0175597698__step3_3) 类似，具体流程如下：
      根据 [**ADD GWSELPLCY**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md) 命令配置结果判断是否允许外网用户使用VPLMN分配的地址。
          - 是 => APN OI模式为C-VPLMN only，漫游属性为Visiting
          - 否 => 不允许外网用户接入网络
          - 标准方式 => [6.a](#ZH-CN_TOPIC_0175597698__step3_6_a)
          a. 查看用户是否签约允许使用VPLMN地址。
                  - 是 => [6.b](#ZH-CN_TOPIC_0175597698__step3_6_b)
                  - 否 => 不允许外网用户接入网络
          b. 查看用户是否签约ODB，限制用户从VPLMN接入点接入网络。
                  - 是 => 不允许外网用户接入网络
                  - 否 => APN OI模式为C-VPLMN only，漫游属性为Visiting
