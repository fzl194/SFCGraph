[**GGSN/P-GW路由选择**](#ZH-CN_TOPIC_0170040601__sub1)

UNC 进行GGSN/P-GW路由选择流程如下：

1. **UNC 首先根据指定的IP地址或Hostname选择GGSN/P-GW路由。**
  UNC 可以为用户指定静态的GGSN/P-GW IP地址或Hostname

  - 如果UNC为用户指定了IP地址，则UNC将此IP地址看作是GGSN/P-GW的控制面IP地址，直接向该IP地址发送消息，无需进行DNS解析。
    - 如果UNC为用户指定了Hostname，则UNC将Hostname发送给DNS，解析出GGSN/P-GW IP地址。
  如果路由选择失败，或者系统未配置指定IP地址或Hostname， UNC 执行如下操作：
    - 如果选择的目标网元是P-GW，UNC执行[2](#ZH-CN_TOPIC_0170040601__procedure-2)。
    - 如果选择的目标网元是GGSN，UNC执行[3](#ZH-CN_TOPIC_0170040601__procedure-3)。
2. **UNC 根据HSS签约数据选择P-GW路由。**
  当 UNC 没有为用户指定P-GW IP地址或Hostname时， UNC 使用HSS中签约的P-GW IP地址，或根据HSS中签约的Hostname进行DNS解析得到P-GW IP地址。
  HSS携带的信息只限于P-GW网元，使用GGSN网元时不支持该功能。因为只有MME支持在3GPP网络与Non-3GPP网络之间的切换，从而动态上报P-GW ID给HSS，而且在传统的GPRS网络中，HLR中也不会签约静态的GGSN IP地址。
  如果HSS没有提供P-GW IP地址或Hostname， UNC 执行 [3](#ZH-CN_TOPIC_0170040601__procedure-3) 。
3. **UNC 根据APN域名选择GGSN/P-GW路由。**
  当 UNC 没有为用户指定GGSN/P-GW IP地址或Hostname，且HSS没有提供P-GW IP地址或Hostname时， UNC 通过DNS解析APN域名得到GGSN/P-GW IP地址。
  APN NI和APN OI按照一定的方式组合成DNS域名，然后到DNS服务器上进行解析，进而得到GGSN/P-GW的IP地址。有关APN NI和APN OI的选择，请参见“WSFD-010501 会话管理”。
  如果路由选择失败，则 UNC 选择GGSN/P-GW路由失败。

UNC 可以通过命令 [**ADD GWSELPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md) 为不同范围的用户配置不同的GGSN/P-GW路由选择方法和机制，以满足运营商对不同用户选择不同GGSN/P-GW，灵活部署网络的需求。在EPC网络中，用户范围的定义包括指定IMSI前缀用户、指定IMSI范围用户、外网用户、本网用户。在GPRS/UMTS网络中，用户范围的定义包括指定IMSI前缀用户、指定IMSI范围用户、外网用户、本网用户、本网本地用户、本网异地用户。其中本网用户和外网用户通过IMSI进行判断，本网本地用户和本网异地用户通过本地HLR进行判断。具体判断方法请参见 [WSFD-105002 漫游用户QoS限制](../../QoS功能/WSFD-105002 漫游用户QoS限制_93181076.md) 的“用户属地识别”部分描述。

> **说明**
> 在EPC网络中，由于无法获取HLR号码，不能对本网用户进一步细分本网本地或本网异地。

当一个UE使用相同的APN建立多个PDN/PDP连接时，如果配置了 [**ADD SCNDGWSELPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加相同APN建立多个PDP_PDN连接的网关选择策略(ADD SCNDGWSELPLCY)_26145946.md) ，则首先按照该命令配置的策略进行网关选择。如果没有匹配到，则按照 [**SET SMFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/SM扩展功能管理/设置会话管理扩展功能(SET SMFUNC)_26145684.md) 中参数 “LTE模式相同APN网关选择策略（LTEGWPREFER）” 和 “GU模式相同APN网关选择策略（GUGWPREFER）” 的整系统配置进行网关选择。通过 [**ADD SCNDGWSELPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加相同APN建立多个PDP_PDN连接的网关选择策略(ADD SCNDGWSELPLCY)_26145946.md) 命令对相同APN建立多PDN连接场景单独设置网关选择策略，避免了在该场景下，UE直接使用整系统配置策略而绕开DNS查询，选择到不合适的P-GW，导致承载创建失败。

[**根据DNS域名后缀选择DNS服务器**](#ZH-CN_TOPIC_0170040601__sub1)

DNS解析涉及到DNS服务器选择。

单击了解DNS解析详情。

- UNC 通过DNS解析获取网关IP地址的步骤如下：
  DNS提供域名和Hostname到主机IP地址的映射记录。 UNC 可以根据以下方法获取网关IP地址：

  - 使用本地主机信息表（Hostfile）：保存在本地UNC中，包含的记录比较少。
    - 使用DNS服务器：DNS服务器是网络中专门提供域名解析服务的服务器
    - 使用DNS Cache：是DNS服务器解析到的域名和IP地址在UNC上的缓存，用于快速解析域名，以减少访问DNS服务器的次数，其记录具有一定的生命周期，生命周期结束后，该记录失效。
    1. 首先查找UNC上Hostfile中的主机信息记录。
    2. 如果没有相应的记录，查找DNS Cache中的记录。
    3. 如果没有相应的记录，向DNS服务器发送解析请求进行解析。
  UNC 主要使用DNS服务器进行域名解析，Hostfile作为辅助手段。
  UNC 使用DNS服务器进行域名解析时，如果配置了多个IP地址，域名解析结果中将会根据优先级别和权重进行先后次序排列：
    - 优先级别高的排前面。
    - 同一级别的再按照选择权重来选择，权重越大，则排在前面的概率越高。

UNC 支持根据DNS域名后缀选择DNS服务器，优点如下：

- GPRS网络和EPS网络共存时，运营商可以为GPRS网元和EPS网元选择不同的DNS服务器。
- 多个运营商共享同一个UNC时，这些运营商可以指定各自的DNS服务器和DNS解析策略，以解析到一个特定的GGSN/P-GW。

DNS域名后缀的推荐原则如 [表1 域名后缀推荐原则](#ZH-CN_TOPIC_0170040601__tab1_1) 所示。

| 用户接入核心网接口 | 域名后缀 | 使用域名的逻辑实体 |
| --- | --- | --- |
| Gb/Iu | .gprs | SGSN |
| S1 | .org | MME |

[**多GGSN/P-GW路由功能**](#ZH-CN_TOPIC_0170040601__sub1)

UNC 通过DNS解析APN得到GGSN/P-GW IP地址列表，从中选取一个GGSN/P-GW IP地址可以有两种方式：

- 主备方式
  在解析到的GGSN/P-GW IP列表里， UNC 通过配置IP地址为不同的优先级来实现主备方式。 UNC 选择优先级最高的记录，作为主用GGSN/P-GW。如果主用GGSN/P-GW故障， UNC 选择列表中另一优先级最高的IP地址对应的GGSN/P-GW，避免业务损失。
  例如：从DNS服务器返回的APN1的解析结果为四个GGSN/P-GW IP地址，按优先级的高低排序分别为IP1、IP2、IP3、IP4，在所有GGSN/P-GW都正常时， UNC 一直选择IP1。在IP1发生故障后，对于后续的PDP上下文激活请求，DNS服务器返回的APN1解析结果仍然为按优先级的高低排序的IP1、IP2、IP3、IP4，此时， UNC 会选择IP2替代故障的IP1。
- 负荷分担方式
  在解析到的GGSN/P-GW IP列表里， UNC 通过配置IP地址为相同的优先级来实现负荷分担方式。 UNC 轮选优先级相同的IP地址，在这些GGSN/P-GW之间实现负荷分担。
  例如：在收到PDP上下文激活请求时，从DNS服务器返回的APN1解析结果为四个优先级相同的GGSN/P-GW IP，分别为IP1、IP2、IP3、IP4，在所有GGSN/P-GW都正常时， UNC 选择IP1；在再次收到PDP上下文激活请求时，从DNS服务器返回的APN1解析结果仍为优先级相同的IP1、IP2、IP3、IP4， UNC 将选择除IP1的另外一个IP地址，例如IP3，依次类推。

在DNS服务器解析的过程中，如果 UNC 要求DNS服务器查询NAPTR记录，则会首先解析出Hostname集合，每个Hostname再解析出对应的IP地址列表。 UNC 可以通过配置Hostname的优先级来选择首先选择优先级较高的Hostname进行解析，如果优先级相同， UNC 将轮选Hostname到DNS进行解析。
