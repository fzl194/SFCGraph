## [操作场景](#ZH-CN_OPI_0248552637)

本操作指导介绍在运行网络中激活用户接入控制功能的操作过程。

> **说明**
> - 对于待接入的用户，同时存在签约信息和AMF本地配置的移动性接入限制信息，本地配置的优先级高于签约数据。
> - 若签约数据或AMF本地配置给出的移动性接入限制信息为空，则表示不限制。

## [操作步骤](#ZH-CN_OPI_0248552637)

1. 进入 “MML命令行-UNC” 窗口。
2. AMF本地配置用户移动性接入限制信息。
  [**ADD NGMMSUBDATA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/签约数据管理/增加用户移动性管理参数（ADD NGMMSUBDATA）_09651362.md)
3. 配置拒绝用户接入的原因值。
  [**SET NGMMPROCTRL**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM流程管理/5G移动性管理流程控制参数/设置5G移动性管理流程控制参数（SET NGMMPROCTRL）_09652386.md)

## [任务示例](#ZH-CN_OPI_0248552637)

任务描述

启用用户接入控制功能，限制条件如下：

- 场景一：所有用户拒绝接入5GC网络，拒绝原因值不使用特殊指定原因值，发生该异常时系统将下发#27（N1 mode not allowed）原因值。
- 场景二：IMSI前缀为123431234567890，此号段内限制用户NR接入到5GC网络，拒绝原因值不使用特殊指定原因值。
- 场景三：IMSI前缀为123431234567891，此号段内限制用户EUTRA接入到5GC网络，拒绝原因值不使用特殊指定原因值。
- 场景四：IMSI前缀为123431234567892，此号段内限制用户从5G到4G的互操作流程，拒绝原因值不使用特殊指定原因值。

脚本

- 场景一：所有用户拒绝接入5GC网络。
  //AMF本地配置用户移动性接入限制信息。
  ```
  ADD NGMMSUBDATA: SUBRANGE=ALL_USER, CORERESTRICT=FIFTHGC-1;
  ```
  //拒绝用户接入的原因值。
  ```
  SET NGMMPROCTRL: PROT=OTHER_PROC, ACCRST=0;
  ```
- 场景二：IMSI前缀为123431234567890，此号段内限制用户NR接入到5GC。
  //AMF本地配置用户移动性接入限制信息。
  ```
  ADD NGMMSUBDATA: SUBRANGE=IMSI_PREFIX, IMSIPRE="123431234567890", RATRESTRICT=NR-1;
  ```
  //拒绝用户接入的原因值。
  ```
  SET NGMMPROCTRL: PROT=OTHER_PROC, ACCRST=0;
  ```
- 场景三：IMSI前缀为123431234567891，此号段内限制用户EUTRA接入到5GC网络。
  //AMF本地配置用户移动性接入限制信息。
  ```
  ADD NGMMSUBDATA: SUBRANGE=IMSI_PREFIX, IMSIPRE="123431234567891", RATRESTRICT=EUTRA-1;
  ```
  //拒绝用户接入的原因值。
  ```
  SET NGMMPROCTRL: PROT=OTHER_PROC, ACCRST=0;
  ```
- 场景四：IMSI前缀为123431234567892，此号段内限制用户从5G到4G的互操作流程。
  //AMF本地配置用户移动性接入限制信息。
  ```
  ADD NGMMSUBDATA: SUBRANGE=IMSI_PREFIX, IMSIPRE="123431234567892", CORERESTRICT=EPC-1;
  ```
  //拒绝用户接入的原因值。
  ```
  SET NGMMPROCTRL: PROT=OTHER_PROC, ACCRST=0;
  ```
