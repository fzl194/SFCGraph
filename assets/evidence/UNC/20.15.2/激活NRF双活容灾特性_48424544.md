## [操作场景](#ZH-CN_OPI_0248424544)

当运营商希望通过地理容灾确保业务所在地发生重大灾难NRF依然能够提供正常的数据业务，保证5GC中各NF业务流程的可靠性时，部署此特性。

本操作指导介绍在运行网络中分别在两个DC新增2台NRF用于设备间地理容灾或者将2个单机运行的NRF改造组成双活容灾的操作过程。

> **说明**
> 适用于NRF。

## [操作流程](#ZH-CN_OPI_0248424544)

请严格按照 [操作步骤](#ZH-CN_OPI_0248424544__OPIEworkSteps11508030172748) 的顺序进行操作。

## [操作步骤](#ZH-CN_OPI_0248424544)

1. 在NRF1和NRF2上分别配置双活容灾组。
    a. 配置双活容灾实例。
      **[ADD DRINST](../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/容灾管理/配置容灾实例(ADD DRINST)_51011025.md)**
    b. 配置双活NRF容灾实例通信地址信息，用于数据同步通道建立。
      **[ADD DRCOMM](../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/容灾管理/增加容灾实例地址(ADD DRCOMM)_51012925.md)**
      > **说明**
      > 此命令针对每个NRF需要配置两次，分别为本端NRF实例通信地址信息和对端NRF实例通信地址信息，并且需要先配置本端NRF实例通信地址信息。
2. NRF1和NRF2上的 [1](#ZH-CN_OPI_0248424544__cmd17947852202716) 都完成后，在NRF1和NRF2上分别打开双活容灾的License开关。
  **[SET LICENSESWITCH](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)**
3. **可选：**NRF1和NRF2上的 [2](#ZH-CN_OPI_0248424544__cmd2716193316285) 都完成后，在NRF1和NRF2上分别先关闭容灾通道开关，打开双活备份状态忽略开关。
  > **说明**
  > 如果是新建的2个NRF组成双活NRF，则不需要执行此步骤。
    a. 初始先关闭容灾通道开关。
      **[SET DBSOFTPARA](../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/软件参数管理/设置CSDB软件调试参数表(SET DBSOFTPARA)_80429704.md)**
    b. 打开双活备份状态忽略开关，忽略数据备份状态，NRF允许直接处理业务。
      **[SET NRFDRRTSTASW](../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NRF双活容灾参数/设置NRF双活备份状态忽略开关（SET NRFDRRTSTASW）_52071375.md)**
4. **可选：**在NRF1和NRF2上分别再打开容灾通道开关，关闭双活备份状态忽略开关，触发NRF间数据同步。
  > **说明**
  > 如果是新建的2个NRF组成双活NRF，则不需要执行此步骤。
    a. 对同一个NRF，请在 [3.a](#ZH-CN_OPI_0248424544__cmd64680333442) 执行关闭容灾通道开关1分钟后，打开容灾通道开关，数据可以实时同步。
      **[SET DBSOFTPARA](../../../../../OM参考/命令/UNC MML命令/平台服务管理/CSDB功能管理/CSDB管理/软件参数管理/设置CSDB软件调试参数表(SET DBSOFTPARA)_80429704.md)**
    b. 关闭双活备份状态忽略开关，NRF进入实时备份状态时，才允许处理业务。
      **[SET NRFDRRTSTASW](../../../../../OM参考/命令/UNC MML命令/业务服务管理/NRF业务及策略管理/NRF业务参数/NRF双活容灾参数/设置NRF双活备份状态忽略开关（SET NRFDRRTSTASW）_52071375.md)**

## [任务示例](#ZH-CN_OPI_0248424544)

任务描述

现网2个单机运行的NRF组成双活容灾组，NRF信息如下：

NRF1：容灾实例标识为0，用于数据同步的通信IP地址为10.5.5.1。

NRF2：容灾实例标识为1，用于数据同步的通信IP地址为10.5.5.2。

脚本

**//配置NRF1。**

//配置双活容灾实例。

```
ADD DRINST:DRINSTID=
0
, DRINSTNAME="nrf1";
```

//配置双活NRF容灾实例通信地址信息，用于数据同步通道建立。

```
ADD DRCOMM: DRINSTID=
0
, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.1", VPNNAME="_public_";
ADD DRCOMM: DRINSTID=
1
, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.2", VPNNAME="_public_";
```

**//配置NRF2。**

//配置双活容灾实例。

```
ADD DRINST:DRINSTID=
1
, DRINSTNAME="nrf2";
```

//配置双活NRF容灾实例通信地址信息，用于数据同步通道建立。

```
ADD DRCOMM: DRINSTID=
1
, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.2", VPNNAME="_public_";
ADD DRCOMM: DRINSTID=
0
, IPVERSION=COMM_IPV4, COMMIPV4="10.5.5.1", VPNNAME="_public_";
```

**//配置NRF1。**

//打开NRF双活容灾特性开关。

```
SET LICENSESWITCH: LICITEM="LKV2DLNRR01", SWITCH=ENABLE;
```

**//配置NRF2。**

//打开NRF双活容灾特性开关。

```
SET LICENSESWITCH: LICITEM="LKV2DLNRR01", SWITCH=ENABLE;
```

**//配置NRF1。**

//初始先关闭容灾通道开关。

```
SET DBSOFTPARA: PARAID=7, PARAVALUE=1;
```

//打开双活备份状态忽略开关，忽略数据备份状态，NRF允许直接处理业务。

```
SET NRFDRRTSTASW: FCSWITCH=FUNC_ON;
```

**//配置NRF2。**

//初始先关闭容灾通道开关。

```
SET DBSOFTPARA: PARAID=7, PARAVALUE=1;
```

//打开双活备份状态忽略开关，忽略数据备份状态，NRF允许直接处理业务。

```
SET NRFDRRTSTASW: FCSWITCH=FUNC_ON;
```

**//配置NRF1。**

//NRF1正常运行且与NRF2通信正常，在NRF1上关闭容灾通道开关1分钟后，再打开容灾通道开关，数据可以实时同步。

```
SET DBSOFTPARA: PARAID=7, PARAVALUE=0;
```

//关闭双活备份状态忽略开关，NRF进入实时备份状态时，才允许处理业务。

```
SET NRFDRRTSTASW: FCSWITCH=FUNC_OFF;
```

**//配置NRF2。**

//NRF2正常运行且与NRF1通信正常，在NRF2上关闭容灾通道开关1分钟后，再打开容灾通道开关，数据可以实时同步。

```
SET DBSOFTPARA: PARAID=7, PARAVALUE=0;
```

//关闭双活备份状态忽略开关，NRF进入实时备份状态时，才允许处理业务。

```
SET NRFDRRTSTASW: FCSWITCH=FUNC_OFF;
```
