## [命令功能](#ZH-CN_MMLREF_0000001192437462)

**适用NF：SGW-C、PGW-C、GGSN、SMF**

该命令用于查询指定实例名称的UPF粒度PFCP参数。

## [注意事项](#ZH-CN_MMLREF_0000001192437462)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001192437462)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001192437462)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| UPFINSTANCEID | UPF实例标识 | 可选必选说明：可选参数<br>参数含义：该参数用于指定UPF实例名称。<br>数据来源：全网规划<br>取值范围：字符串类型，输入长度范围是0~36。<br>默认值：无<br>配置原则：<br>本参数取值与ADD PNFPROFILE命令中的“NF实例标识”参数取值保持一致时，关联关系生效。 |

## [输出结果说明](#ZH-CN_MMLREF_0000001192437462)

| 输出项名称 | 输出项解释 |
| --- | --- |
| UPF实例标识 | 该参数用于指定UPF实例名称。 |
| 心跳间隔(秒) | 该参数用于配置控制面检查N4 PFCP心跳的时间间隔。 |
| 心跳消息发送次数阈值 | 该参数用于标识心跳消息发送次数阈值。当心跳消息发送次数到达阈值后SMF还未收到UPF的心跳响应，那么SMF会判定N4链路断开。 |
| 去活用户开关 | 该参数用于标识当UPF心跳故障时间超过允许的阈值或者UPF重启时，是否去激活该up上的用户。 |
| 去活间隔(秒) | 该参数用于标识UPF心跳故障允许的最大阈值，如果超过该阈值后，将会根据DeactiveFlag决定是否去激活用户。 |
| 心跳消息超时间隔(秒) | 该参数标识心跳消息超时间隔。如果在该时间内SMF没有收到UPF的心跳响应，那么SMF根据当前超时的次数决定处理方式，如果超时次数小于参数HBN1，那么会重发心跳，否则会判定与UPF之间的N4链路断开。 |
| 迁移间隔(秒) | 该参数用于设置辅锚点UPF迁移定时器，辅锚点UPF心跳异常后开启该定时器，定时器超时后心跳仍未恢复，则迁移该辅锚点UPF上的用户。 |
| 核查用户开关 | 该参数用于标识是否打开SMF与UPF用户核查功能。 |
| 惯性运行定时器(秒) | 该参数用于设置链路惯性运行定时器，UPF心跳异常后开启该定时器，定时器超时后心跳仍未恢复，该链路惯性运行定时器超时。 |
| 静默路径业务迁移开关 | 该参数用于指定SMF通过安全信令网关和UPF对接的场景下，当代理路径全部故障后，是否支持PFCP会话路径迁移到静默路径上。 |
| 静默路径业务迁移间隔(秒) | 该参数用于指定当代理路径全部故障后，业务迁移到静默路径上的时间间隔。 |
| 静默路径业务回迁间隔(秒) | 该参数用于指定当代理路径恢复后，业务启动迁回代理路径的间隔。 |
| 静默路径状态恢复间隔(秒) | 该参数用于指定当代理路径恢复业务启动回迁后，静默路径恢复静默状态的间隔。 |
| 静默路径发送偶联建立请求消息 | 该参数用于指定当代理路径恢复业务启动回迁后，静默路径恢复静默状态时是否通过代理路径发送偶联建立请求消息。 |
| 流量上报消息保序优化开关 | 该参数用于配置SMF对指定UPF是否使能流量上报消息保序优化开关。 |
| 发送偶联更新消息开关 | 该参数用于指定当对接的UPF存在多条路径时，是否支持在偶联建立成功之后发送偶联更新消息通知UPF路径信息。 |
