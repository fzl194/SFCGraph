#### [适用NF](#ZH-CN_CONCEPT_0193967875)

SGSN

#### [定义](#ZH-CN_CONCEPT_0193967875)

PFC（Packet Flow Context）是指通过为同一用户的多个 QoS 相近的 PDP 分配相同的PFI（Packet Flow Identifier），实现数据流的捆绑传送。

#### [客户价值](#ZH-CN_CONCEPT_0193967875)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | PFC级别上实现流控等QoS机制，方便运营商对同一<br>MS<br>的不同QoS类别的<br>PDP上下文<br>区分处理。 |
| 用户 | 保证良好的业务质量，提升用户业务体验。 |

#### [应用场景](#ZH-CN_CONCEPT_0193967875)

当 无线资源 不足，不能保证高优先级用户的无线传输带宽时，建议开启PFC功能，实现数据流的捆绑传送，来提高用户业务体验。

#### [原理概述](#ZH-CN_CONCEPT_0193967875)

**相关概念**

SGSN为 BSS 提供Gb模式下正在进行数据传输的用户相关信息，这些信息存储在BSS上下文（BSS context）中。

在BSS上可以包含多个MS的BSS上下文。每个MS都有一个对应的BSS上下文，包含该MS相关的所有BSS PFC。

BSS PFC由SGSN分配的PFI唯一标识。 PFC的ABQP（Aggregate BSS QoS Profile）包含了该PFC中所有PDP上下文的QoS属性信息。

PFC形式的数据传输举例如 [图1 MS的分组数据流传输举例](#ZH-CN_CONCEPT_0193967875__fig05) 所示，QoS相近的PDP Context 1和PDP Context 2组成一个PFC 1，有相同的PFI，ABQP包含了PFC 1中所有PDP上下文的QoS信息。PDP Context 3的QoS与PDP Context 1、PDP Context 2的QoS不同，单独组成PFC 2。

**图1** MS的分组数据流传输举例

<br>

![](特性概述_93967875.assets/zh-cn_image_0194107697_2.png)

由此可见，SGSN以BSS PFC的形式为BSS提供用户数据传输的QoS属性信息。BSS能基于PFC层级实施不同的QoS控制策略（如流量监管和资源分配等），使运营商对同一MS的不同QoS类别的PDP上下文区分处理。

**PFC的流量控制**

BSS可以进行流量控制，并且向SGSN发送每个PFC的流量控制信息。

PFC的流量控制示意图如 [图2 PFC流量控制](#ZH-CN_CONCEPT_0193967875__fig01) 所示。

每个PFC的缓存状态可以通过流量控制消息中携带的PFC Bucket Size、Leak Rate、Bucket Full Level（%）信元发送给SGSN。
