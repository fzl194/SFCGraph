本章节以SMF为例描述N6接口故障隔离特性。

#### [N6接口故障上报](#ZH-CN_TOPIC_0000001378700720)

当UPF开启路径探测功能后会向路径对端发送PING报文探测路径状态是否正常。当UPF检测到N6接口某条链路数据转发异常，并且该链路关联的DNN/APN开启了状态上报开关，UPF会向SMF上报该故障链路关联的DNN/APN信息。SMF收到UPF上报的故障信息后会进行相应的故障联动处理，处理流程如 [图1](#ZH-CN_TOPIC_0000001378700720__fig181425531) 所示，UPF实现请参考UEG产品的GWFD-021407 快速业务恢复特性。

**图1** N6接口故障上报

<br>

![](实现原理_78700720.assets/zh-cn_image_0000001380277150_2.png "点击放大")

当检测到N6接口故障后，UPF向SMF发送PFCP Node Report Request消息，消息中携带User Plane N6 Tunnel Failure Report信元，该信元中包含了故障链路关联的DNN/APN列表。

SMF收到携带User Plane N6 Tunnel Failure Report信元的PFCP Node Report Request消息后，解析User Plane N6 Tunnel Failure Report信元中的DNN/APN信息。如果故障DNN/APN配置为检测到故障后隔离，则SMF会保存故障的UPF和DNN/APN信息，上报 **[ALM-100536 N6接口故障的UPF黑名单](../../../../../网络运维/故障处理/UNC告警处理/业务告警/SMF&GW-C&GGSN/ALM-100536 N6接口故障的UPF黑名单_76661414.md)** 告警。对于使用故障DNN/APN的用户，SMF进行如下处理：

- 如果故障DNN/APN配置为SMF在用户面链路故障时进行恢复处理，则去激活当前已经激活在故障的UPF上的用户，待会话重新激活时选择其它可用的UPF。
- 为新激活用户选择UPF时，过滤掉上报故障的UPF，选择其它可用的UPF。

#### [N6接口故障恢复](#ZH-CN_TOPIC_0000001378700720)

当UPF检测到N6接口某条链路状态恢复正常，并且该链路关联的DNN/APN开启了状态上报开关，UPF会向SMF上报该链路关联的DNN/APN信息。SMF收到UPF上报的恢复信息后处理流程如 [图2](#ZH-CN_TOPIC_0000001378700720__fig050774918316) 所示。

**图2** N6接口故障恢复

<br>

![](实现原理_78700720.assets/zh-cn_image_0000001430157065_2.png "点击放大")

当检测到N6接口故障的链路恢复后，UPF向SMF发送PFCP Node Report Request消息，消息中携带User Plane N6 Tunnel Recovery Report信元，该信元中包含了故障恢复的链路关联的DNN/APN列表。

SMF收到携带User Plane N6 Tunnel Recovery Report信元的PFCP Node Report Request消息后，解析User Plane N6 Tunnel Recovery Report信元中的DNN/APN信息。SMF会清除上报故障恢复的UPF和DNN/APN信息，同一个UPF的所有故障DNN/APN信息都清除后，恢复 **[ALM-100536 N6接口故障的UPF黑名单](../../../../../网络运维/故障处理/UNC告警处理/业务告警/SMF&GW-C&GGSN/ALM-100536 N6接口故障的UPF黑名单_76661414.md)** 告警。对于使用故障恢复的DNN/APN新激活的用户恢复选择该UPF。

#### [故障信息老化](#ZH-CN_TOPIC_0000001378700720)

为保证N6接口故障上报和恢复消息功能的可靠性，UPF支持配置定时向SMF上报全量故障的DNN/APN信息。SMF支持配置故障DNN/APN信息的老化时间（该时间需要大于UPF配置的定时上报周期），如果当前时间距离最后一次收到某UPF报告故障信息的时长超过了配置的老化时间，则该UPF上报的DNN/APN会被清除。同时，SMF提供了手动清除故障UPF和DNN/APN信息的命令 **[CLR UPFAULTCACHE](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/可靠性管理/用户面故障管理/清除用户面故障缓存（CLR UPFAULTCACHE）_56347538.md)** ，执行该命令会触发 **[ALM-100536 N6接口故障的UPF黑名单](../../../../../网络运维/故障处理/UNC告警处理/业务告警/SMF&GW-C&GGSN/ALM-100536 N6接口故障的UPF黑名单_76661414.md)** 告警恢复。
