# 实现原理

- [VoNR典型业务场景](#ZH-CN_TOPIC_0000001158245283__1.3.6.1)
- [VoNR呼叫流程](#ZH-CN_TOPIC_0000001158245283__1.3.7.1)

在EPS Fallback的语音方案基础之上，无线侧eNodeB升级或支持了NR覆盖，使得用户得以通过VoNR进行语音业务。但用户在NR网络下发起呼叫，无论通话中终端是否移动到LTE网络，均称为VoNR呼叫。

- 切换到LTE往后实质上属于一种**4G语音**的解决方案：VoLTE。
- EPS Fallback、VoNR都属于**5G语音**的解决方案，5G语音的目标语音方案是VoNR。在建网初期采用EPS FB（5G回落4G）的过渡方案，当NR覆盖较广且网络调优后语音改造为VoNR。

因此当前部署VoNR有以下三种演进路径：

- VoLTE → EPS FB/VoNR：参考[WSFD-102702 EPS Fallback](../WSFD-102702 EPS Fallback_29036513.md)，需要4/5G互操作组网。
- EPS FB → VoNR：对于5G Core网络无需新增配置，由NGRAN决定语音业务为EPS FB或是VoNR方案。
- VoNR全新建：本特性描述场景。

有一种主流的语音演进方案，VoLTE → EPS FB → VoNR：基于VoLTE组网叠加部署NR和5GC，经EPS Fallback的过渡再演进至VoNR，可基于Option2和Option3/3a/3X组网实现演进。

#### [VoNR典型业务场景](#ZH-CN_TOPIC_0000001158245283)

VoNR基本呼叫的典型业务场景如 [表1](#ZH-CN_TOPIC_0000001158245283__table1692519271471) 所示。

*表1 典型业务场景*

| 业务场景 | 业务场景 | 说明 |
| --- | --- | --- |
| 注册流程 | 注册流程 | VoNR和EPS FB的注册流程相同，统称为Vo5G注册。Vo5G的注册过程跟VoLTE基本一样，主要区别在于5G将初始注册的Registration过程与会话建立的PDU Session过程拆分为两个独立的过程。4G在EPC网络附着时必须先建立数据APN，再建立IMS APN，才可进行后续会话的建立。而5G无需先建立数据DNN，即可直接建立IMS DNN，从而建立会话。 |
| VoNR呼叫流程 | MO侧呼叫流程 | Vo5G用户在IMS网络注册后，在NR发起呼叫。<br>[VoNR呼叫流程](#ZH-CN_TOPIC_0000001158245283__section834517102535)<br>。 |
| VoNR呼叫流程 | MT侧呼叫流程 | Vo5G用户在IMS网络注册后，在NR接收呼叫。相对于MO侧呼叫流程，MT侧呼叫流程主要是多了对UE的Paging过程及IMS域选过程。 |
| 5G到4G的互操作流程 | 基于PSHO的切换流程 | VoNR基于PSHO切换流程和基于切换的EPS FB流程基本相同，主要区别在于VoNR是在通话中由5G到4G进行切换，而EPS FB是在呼叫前便触发回落。 |
| 5G到4G的互操作流程 | 基于重定向的切换流程 | VoNR基于重定向的切换流程和基于重定向的EPS FB流程基本相同，主要区别在于VoNR是在通话中由5G到4G进行切换，而EPS FB是在呼叫前便触发回落。 |

#### [VoNR呼叫流程](#ZH-CN_TOPIC_0000001158245283)

**图2** VoNR呼叫流程

<br>

![](实现原理_58245283.assets/zh-cn_image_0000001214673025_2.png "点击放大")

1. P-CSCF向PCF发送AAR消息，请求获取UE的接入网类型。
2. PCF向SMF发送Npcf_SMPolicyControl_UpdateNotify Request消息通知SMF创建语音专有QoS Flow。
3. SMF响应Npcf_SMPolicyControl_UpdateNotify Response消息。
4. PCF向P-CSCF返回AAA消息。
5. SMF向AMF发送Namf_Communication_N1N2MessageTransfer，消息中携带SM相关信息。
6. AMF响应Namf_Communication_N1N2MessageTransfer Response消息。
7. AMF向NG RAN发送PDU Session Resource Modify Request消息。
8. AMF向UE发送PDU Session Resource Modification Command消息。
9. NG-RAN响应PDU session resource modify response消息，成功建立语音QOS flow。
10. UE响应PDU Session Resource Modification Accept消息。
11. AMF向SMF发送Nsmf_PDUSession_UpdateSMContext Request携带N2消息 PDU session resource modify response。
12. SMF响应Nsmf_PDUSession_UpdateSMContext Response消息。
13. SMF向UPF发送PFCP Modification Request。
14. UPF响应PFCP Modification Response。
15. SMF向PCF发送Npcf_SMPolicyControl_Update request消息，消息中携带用户当前的位置信息。
16. PCF向SMF返回Npcf_SMPolicyControl_Update response消息。
17. P-CSCF收到RAR消息。
18. PCSCF向PCF返回RAA响应。
