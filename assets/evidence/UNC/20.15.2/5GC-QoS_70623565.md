#### [5G QoS模型](#ZH-CN_TOPIC_0170623565)

如 [图1](#ZH-CN_TOPIC_0170623565__fig4161817175512) 所示，端到端EPS QoS模型是以承载粒度进行控制，EPS承载由无线承载、S1承载、S5/S8承载三部分组成，并且三种底层承载间一一对应。

不同于EPS QoS模型，5G网络中是以QoS flow粒度执行QoS控制，一个PDU会话可以由多个QoS flow进行控制。 在无线接入侧，一个无线承载可以对应多个QoS flow。N3接口上，同一个PDU会话采用同一个隧道流。

![](5GC QoS_70623565.assets/zh-cn_image_0170632689_2.png "点击放大")

[图2](#ZH-CN_TOPIC_0170623565__fig9312935102218) 为5GC QoS对上下行的用户面数据包进行分类、标记以及映射到QoS flows的过程，详细过程如下：

- 下行数据
  UPF根据下行PDRs的Packet Filter Sets对下行数据进行分类，并将数据包映射到QoS flow，然后执行该QoS flow对应的QoS控制。在N3接口用QFI标记该数据分组，将该数据发送给(R)AN。(R)AN收到分组数据后，根据QFI找到对应的DRB（data radio bearer）和QoS参数，执行QoS控制，将数据发送给UE。
- 上行数据
  UE通过QoS rules中的UL filters将上行数据包映射到QoS flow，并且对QoS flow进行标记。UE将QoS flow匹配到DRB，将数据发送到(R)AN。(R)AN收到后根据QFI执行QoS控制，并将分组数据发送到UPF。

每个PDU会话都有一个无QoS flow template的默认QoS rule，用该QoS rule建立的QoS flow类似于EPS网络中的默认承载，只要无法匹配到其它QoS flow的数据都映射到该QoS flow进行传输。

#### [5GC QoS参数](#ZH-CN_TOPIC_0170623565)

5G网络使用的QoS参数与4G网络不同，如 [表3](#ZH-CN_TOPIC_0170623565__table13834161210228) 所示。

*表1 5G网络与4G网络QoS参数对比*

| QoS类别 | 4G网络QoS参数 | 5G网络QoS参数 |
| --- | --- | --- |
| 关键QoS参数 | QCI （QoS Class Identifier ） | 5QI （5G QoS Identifier） |
| 关键QoS参数 | ARP（Allocation and Retention Priority ） | ARP |
| 关键QoS参数 | MBR（Maximum Bit Rate ）：最大带宽<br>GBR（Guaranteed Bit Rate）：保证带宽 | MFBR（Maximum Flow Bit Rate）：最大流比特率<br>GFBR（Guaranteed Flow Bit Rate）：保证流比特率 |
| 其他QoS参数 | APN-AMBR | Session-AMBR |
| 其他QoS参数 | UE-AMBR | UE-AMBR |
| 其他QoS参数 | - | RQA(Reflective QoS Attribute) |
| 其他QoS参数 | - | Notification control |

> **说明**
> 表中“-”代表不存在。

- 5QI
  5G QoS索引，作为特定分组转发行为(如丢包率和时延)的参考等级提供给业务数据流。5QI可以在(R)AN上实现配置，也可以动态分配，是QoS profile的一部分。

- ARP
  分配保留优先级。ARP信元包含priority level、pre-emption capability和pre-emption vulnerability。在资源受限情况下，决定是否接受新的QoS flow，以及决定QoS flow是否优先占用资源。priority level分为1到15个等级，1为最高优先级。
  pre-emption capability决定一个数据流是否可以抢占低优先级的资源。
  pre-emption vulnerability决定一个数据流是否可以被高优先级的数据流抢占资源。
- GFBR
  保证比特速率，系统通过资源预留的方式，保证数据流的比特速率在不超过GFBR时能够全部通过。
  超过GFBR的流量可以按照如下方式处理：

  - 拥塞时超过GFBR的流量会被丢弃。
    - 不拥塞时超过GFBR但小于MBR的流量可以通过。
  享有GFBR资源的承载被称为GBR承载，其它的承载被称为Non-GBR承载。GFBR分为上行GFBR和下行GFBR。
- MFBR
  最大比特速率，系统通过限制流量的方式禁止数据流的比特速率超过MFBR。MFBR应大于等于GFBR。目前受限于终端的能力，MFBR都等于GFBR。MFBR分为上行MFBR和下行MFBR。
- Session-AMBR（Session Aggregate Maximum Bit Rate）
  一个会话的所有Non-GBR QoS flows共享的最大比特速率。GBR QoS flows的流量不在Session-AMBR范围内。
  SMF从UDM获取签约的Session-AMBR发送给UPF和UE，由UPF和UE执行QoS控制。
- UE-AMBR（UE Aggregate Maximum Bit Rate）
  一个UE的所有Non-GBR QoS flows共享的最大比特速率。GBR QoS flows的流量不在UE-AMBR范围内。
  AMF从UDM获取签约的UE-AMBR，发送给(R)AN。由(R)AN进行执行QoS控制。
- RQA
  Reflective QoS属性，是一个可选参数。当QoS flow中指示了Reflective QoS属性，则(R)AN将RQI与QoS flow进行关联，用于指明该QoS flow是受制于Reflective QoS。 RQA在(R)AN建立UE上下文或在QoS flow增加和删除的时候，SMF通过N2节点通知给(R)AN。
- Notification control
  通知控制，是一个可选参数。当GFBR不再满足QoS flow时，(R)AN会通知SMF，SMF转发给PCF。用于5G核心网对该QoS flow进行对应的处理。

- QoS Flow
  QoS flow是5G QoS控制的最小粒度，一个PDU会话可能包含有多个QoS flow。每一个QoS Flow由QFI（QoS Flow ID）唯一识别。
  QoS flow有如下三个特征，如 [表2](#ZH-CN_TOPIC_0170623565__table1536116535) 所示。

  *表2 QoS flow的特征*

  | 特征名称 | 详细解释 |
  | --- | --- |
  | QoS profile | QoS flow的QoS parameters承载在QoS profile中，这些参数决定QoS flow的资源类型为GBR QoS flow还是Non-GBR QoS flow。<br>Non-GBR QoS flow包含参数：ARP、5QI、RQA（可选）。<br>GBR QoS flow包含参数：ARP、5QI、GFBR、MFBR、Notification control（可选），上下行最大包丢失率（可选）。<br>QoS profile由SMF通过AMF发送给(R)AN或者是在(R)AN上配置。 |
  | QoS rule | UE基于QoS规则执行上行用户面流量的分类和标记。包含QFI、 Packet Filter Set、 Precedence Value、QoS rule ID等内容。<br>QoS规则来源：<br>- PDU会话建立/修改流程中发送给UE。<br>- UE上配置。<br>- UE通过Reflective QoS机制推演获得。 |
  | UL and DL PDR(s) | 上下行的PDR(Packet Detection Rule)s，用户面数据报文检测规则，在UPF上用于上下行的用户面数据流的处理规则。<br>SMF通过N4接口发送到UPF。 |
