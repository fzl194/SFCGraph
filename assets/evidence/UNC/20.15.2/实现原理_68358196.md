#### [实时时区获取机制（适用于 SGSN、 MME）](#ZH-CN_CONCEPT_0168358196)

不同位置所对应的时区和夏令时是 UNC 通过如下MML命令配置实现。

- [**ADD AREATZ**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/多时区管理/区域时区参数配置/增加区域时区参数(ADD AREATZ)_72225267.md)：该命令用于给不同的位置区/路由区/跟踪区指定对应的时区标识。
- [**ADD TZLST**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/多时区管理/多时区参数/增加多时区参数(ADD TZLST)_26305400.md)：该命令用于配置时区标识所对应的时区和夏令时信息。

通过上述MML命令配置，可以指定某一特定区域（LAC/RAC/TAC）或区段所映射的时区和夏令时信息。在UE发生区域变更或进行PDP/EPS承载激活等流程时， UNC 根据UE携带的区域信息以及该区域与时区夏令时的映射关系，获取UE当前位置的实时时区信息。

> **说明**
> - 同一区域（LAC/RAC/TAC）不能部署在多个时区。否则，UNC时区上报可能会产生误差。
> - 在开通多时区特性的场景下，如果未配置区域与时区夏令时的映射信息，则 UNC 采用系统时区作为UE的实时时区。系统时区是通过MML命令 [**SET TZ**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/设备管理/时区夏令时管理/设置时区和夏令时信息（SET TZ）_33844133.md) 配置的。

#### [GPRS/UMTS网络中的实时时区上报机制](#ZH-CN_CONCEPT_0168358196)

在GPRS/UMTS网络场景下，不同业务流程中， UNC 向UE和GGSN上报实时时区和夏令时的机制是不同的，如 [表1](#ZH-CN_CONCEPT_0168358196__table_reportstrategy) 所示。

*表1 UNC 向UE和GGSN上报实时时区和夏令时的机制*

| 流程名称 | UNC<br>到GGSN | UNC<br>到GGSN | UNC<br>到UE | UNC<br>到UE | UNC<br>到UE |
| --- | --- | --- | --- | --- | --- |
| 流程名称 | 实时时区和夏令时上报策略 | 上报消息名称 | 实时时区和夏令时上报策略 | 信息上报位置 | 上报消息名称 |
| Attach | 不上报 | - | 始终上报 | Attach Accept消息之后 | GMM Information（注4） |
| Detach | 不上报 (注1) | - | 时区或夏令时发生变化才上报（注2） | Detach Request消息之后 | GMM Information |
| Intra SGSN RAU | 时区或夏令时发生变化才上报 | Update PDP Context Request | 始终上报 | Routing Area Update Accept消息之后 | GMM Information |
| Inter SGSN RAU | 始终上报 | Update PDP Context Request | 始终上报 | Routing Area Update Accept消息之后 | GMM Information |
| 2G和3G间的异系统切换 | 时区或夏令时发生变化才上报 | Update PDP Context Request | 时区或夏令时发生变化才上报 | Routing Area Update Accept消息之后 | GMM Information |
| Intra SGSN Relocation | 在伴随RAU流程场景下，时区或夏令时发生变化才上报 | Update PDP Context Request | 时区或夏令时发生变化才上报 | Intra SGSN Relocation之后的Intra SGSN RAU流程中的Routing Area Update Accept消息之后 | GMM Information |
| Inter SGSN Relocation | 不上报 | - | 时区或夏令时发生变化才上报 | Inter SGSN Relocation之后的Inter SGSN RAU流程中的Routing Area Update Accept消息之后 | GMM Information |
| Service Request | 始终上报 | Update PDP Context Request | 仅夏令时跳变上报 | Service Request安全流程完成后或Service Accept消息之后 | GMM Information |
| PDP激活 | 始终上报(注3) | Create PDP Context Request/Update PDP Context Request | 不上报 | - | - |
| PDP修改 | 始终上报 | Update PDP Context Request/Update PDP Context Response | 不上报 | - | - |
| PDP去激活（UE或SGSN发起） | 不上报 | - | 不上报 | - | - |
| PDP去激活（GGSN发起） | 始终上报 | Delete PDP Context Response | 不上报 | - | - |

> **说明**
> 注1：如果Detach流程伴随了PDP去激活的流程， UNC 会通过Delete PDP Context Request消息中携带时区和夏令时信息给GGSN。
>
> 注2：如果是用户关机触发了Detach流程， UNC 不会上报时区和夏令时信息给UE。
>
> 注3：在PDP激活流程中， UNC 向GGSN上报的是用户附着时或者RAU时的时区信息和UE当前位置区的夏令时。
>
> 注4： UNC 是否可以向UE下发GMM Information消息受 [**SET MMFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/MM扩展功能管理/设置移动性管理扩展功能(SET MMFUNC)_26145512.md) 命令下参数 “发送网络信息” 的控制。

在Gs接口场景下， UNC 对来自MSC的MM Information进行核查后转发。即， UNC 根据当前用户的缓存信息对来自MSC的MM Information中的时区和夏令时信息进行纠正，然后再转发给UE。

> **说明**
> 如果未开通多时区特性， UNC 不会进行核查，直接将来自MSC Server的MM Information消息转发给UE。

#### [EPC网络中的实时时区上报机制](#ZH-CN_CONCEPT_0168358196)

在EPC网络场景中，时区和夏令时的刷新是PDN级别的。如 [图1](#ZH-CN_CONCEPT_0168358196__fig3) 所示，当MME通过用户某个PDN下的EPS承载激活、修改、去激活流程通知S-GW时区或夏令时变化时，认为是通知S-GW该PDN下所有EPS承载的时区或夏令时变化。在时区和夏令时未再次发生变化的情况下，后续该PDN下的其他承载变化流程中不会再通知S-GW时区或夏令时变化。多PDN的场景下，各个PDN的时区或夏令时变化通知单独执行。

*表2 UNC 向UE和S-GW上报实时时区和夏令时的机制*

>
> 注2：在S1-Based inter-MME Handover流程中，旧测MME会通过Forward Relocation Request消息携带最新的TZ/DST给新侧MME，新侧MME将自身TZ/DST和旧测MME携带的TZ/DST进行比较，如果不相同，则上报旧测携带的TZ/DST。
>
> 注3：在专用EPS承载激活流程中， UNC 向S-GW上报的是用户附着时或者TAU时的时区和UE当前位置区的夏令时。
>
> 注4： UNC 是否可以向UE下发EMM Information消息受 [**SET MMFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/MM扩展功能管理/设置移动性管理扩展功能(SET MMFUNC)_26145512.md) 命令下参数 “发送网络信息” 的控制。在用户联合接入情况下，若希望优先对用户下发EMM Information消息，还需配置该命令中的参数 “PS网络信息优先” 。
>
> 注5：伴随S-GW变化的TAU或切换场景下， UNC 是否始终向S-GW上报实时时区和夏令时受 [**SET SMFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/会话管理/SM扩展功能管理/设置会话管理扩展功能(SET SMFUNC)_26145684.md) 命令下参数“切换后时区携带策略”的控制。建议该参数设置为YES，即始终上报。如果S-GW支持Create Session Request不携带时区信息，可以设置为NO， UNC 仅在时区或夏令时发生变化时才上报。
>
> 注6： [表2](#ZH-CN_CONCEPT_0168358196__table_reportstrategy_mme) 中，针对TAU流程和切换流程， UNC 到UE的实时时区和夏令时上报策略描述的是系统初始设置的上报策略。如果需要对这些系统初始设置的上报策略进行调整，则可以通过 [**SET MMFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/移动性管理/MM扩展功能管理/设置移动性管理扩展功能(SET MMFUNC)_26145512.md) 命令下参数 “EMM Information发送策略” 进行。

在SGs接口场景下， UNC 对来自MSC的MM Information进行核查后转发。即， UNC 根据当前用户的缓存信息对来自MSC的MM Information中的时区和夏令时信息进行纠正，然后再转发给UE。

> **说明**
> 如果未开通多时区特性， UNC 不会进行核查，直接将来自MSC Server的MM Information消息转发给UE。

另外， UNC 在如下两种场景下会通过SGs接口向MSC Server发送UE当前的时区和夏令时信息。

- 在CSFB场景下的被叫流程中，通过SGsAP Service Request发送。
- 在基于SGs接口实现短消息场景下，通过Uplink Unitdata发送。

#### [5G网络中的时区上报机制](#ZH-CN_CONCEPT_0168358196)

UE时区发生变更时，AMF在注册、切换、服务请求结束后，通过Configuration Update Command消息（3GPP 24.501）向UE下发网络标识、时区和夏令时等信息。包含以下信元（3GPP 24.008）：

*表3 信元*

| 信元 | 说明 |
| --- | --- |
| Full name for network | 网络名称全称，即运营商名称全称。 |
| Short name for network | 网络名称简称，即运营商名称简称。 |
| Local time zone | 时区，表示当前区域与通用协调时间（Coordinated Universal Time，UTC）的间隔（偏移量）。 |
| Universal time and local time zone | 通用协调时间和时区。 |
| Network daylight saving time | 夏令时DST，就是利用夏季天亮的早这一自然现象，人为地时间提前一个小时，充分利用光照资源，减少照明时间，从而节约能源。 |

在UE的位置变化时，对UE上的网络名称和时区信息进行刷新，提高用户体验。包含的信息以及配置原则如下：

- 网络名称，即运营商名称，包括运营商的全称（Full name for network）和简称（Short name for network）。配置原则为：
    - AMF可以配置是否下发网络名称，通过[**SET NGMMFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/MM协议参数管理/5G移动性管理/设置5G移动性管理功能（SET NGMMFUNC）_09653748.md)命令的“SNDNETINFO”参数控制。
    - AMF也可以基于区域范围和用户群组组合的粒度来灵活设置不同的网络名称，通过[**ADD AREACODE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/通用配置管理/通用区域编码管理/增加区域编码（ADD AREACODE）_44006351.md)、[**ADD AREAMEM**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/通用配置管理/通用区域成员管理/增加跟踪区成员（ADD AREAMEM）_44006352.md)、[**ADD NGUSRGRP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/通用配置管理/用户群组标识管理/增加5G用户群（ADD NGUSRGRP）_44006475.md)、[**ADD NGUSRGRPMEM**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/通用配置管理/用户群组成员管理/增加5G用户群成员（ADD NGUSRGRPMEM）_44006476.md)、[**ADD NITZPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NITZ管理/NITZ策略管理/增加NITZ策略（ADD NITZPLCY）_09652255.md)命令配置。
    - 当区域范围为所有区域，用户群组为所有用户时，则优先使用[**ADD NGMNO**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/运营商管理/5G 移动网络运营商管理/增加5G模式移动网络运营商信息（ADD NGMNO）_09652141.md)配置的整系统移动网络运营商名称，其次使用[**ADD NITZPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NITZ管理/NITZ策略管理/增加NITZ策略（ADD NITZPLCY）_09652255.md)配置的网络名称。
    - 当网络名称变化后，需要在注册/去注册/切换/SR等流程触发后，才会将变化后的网络名称下发给UE。
- 中文网络名称，即中文运营商名称。配置原则为：
    - AMF在Configuration Update Command消息中填入网络名称的中文编码下发给UE即可，中文网络名称可通过[**ADD NITZPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NITZ管理/NITZ策略管理/增加NITZ策略（ADD NITZPLCY）_09652255.md)或[**ADD NGMNO**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/运营商管理/5G 移动网络运营商管理/增加5G模式移动网络运营商信息（ADD NGMNO）_09652141.md)配置。
      > **说明**
      > 中文网络名称仅支持在中文环境下配置和下发。
- AMF上UE当前的时区和夏令时信息。配置原则为：
    - UE当前位置对应的时区和夏令时信息，通过[**ADD NGTZLST**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/多时区管理/5G多时区参数/增加5G多时区参数（ADD NGTZLST）_71994441.md)和**[ADD NGAREATZ](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/多时区管理/5G区域时区参数/增加5G区域时区参数（ADD NGAREATZ）_20674118.md)**命令配置。
    - 当时区或夏令时信息变化时，需要在注册/去注册/切换/服务请求等流程触发后，才会将变化后的时区信息下发给UE。
