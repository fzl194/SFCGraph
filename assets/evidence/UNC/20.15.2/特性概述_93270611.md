#### [适用NF](#ZH-CN_CONCEPT_0193270611)

MME

#### [定义](#ZH-CN_CONCEPT_0193270611)

本特性包含针对NSA用户的三个功能：安全参数传递功能，流量上报功能，5G用户信息对等网元间传递功能。

- **安全参数传递功能**：NSA Option3/3a/3x组网下，支持双连接能力的终端用户，在Attach Request或者TAU Request消息（周期性TAU请求除外）中，携带UE additional security capability信元，对于此类终端用户，系统支持在发送给eNodeB的消息中携带NR UE Security Capabilities信元，用于eNodeB、gNodeB和UE之间的安全协商。
- **流量上报功能**：支持将用户的5G流量统计报告上报给网关，供网关侧生成话单参考。
- **5G用户信息对等网元间传递功能**：在NSA Option3/3a/3x组网与2/3/4G组网共存场景下，当进行Inter切换时，将DCNR用户信息携带给对等网元，从而确保新侧MME/SGSN能够继续为DCNR用户提供连续服务。

#### [客户价值](#ZH-CN_CONCEPT_0193270611)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 为用户数据传输安全以及用户流量上报提供保障。<br>在NSA Option3/3a/3x组网与<br>2/3/<br>4G组网共存场景下，保证NSA用户的接入和服务质量，增加运营网络的商业价值。 |
| 用户 | 享受安全可靠的业务。 |

#### [应用场景](#ZH-CN_CONCEPT_0193270611)

该特性适用于NSA Option3/3a/3x组网以及NSA Option3/3a/3x组网与 2/3/ 4G组网共存场景。

#### [应用限制](#ZH-CN_CONCEPT_0193270611)

- 本特性不支持GTPv0版本中的GTP消息。
- 根据协议，5G流量上报的解决方案不支持端到端的计费流量准确性保证，MME作为5G流量统计报告的一个中间转发节点，无法识别到流量统计报告是否存在重复或者缺失。
- 在系统过载触发流控的场景下，S1接口中，携带流量统计报告的信令也可能被丢弃。
- 根据协议，在4G到2/3G RAU场景下，MME将从RAN侧获取的流量统计报告发送给S-GW，但P-GW在这种场景下收不到流量统计信息。
- MME会忽略处理eNodeB在E-RAB Modify Response消息中携带的流量统计报告。

[安全参数传递功能](#ZH-CN_CONCEPT_0193270611__li_01)

NSA双连接流程中的安全参数传递如 [图1](#ZH-CN_CONCEPT_0193270611__fig7405411183015) 所示。NSA的5G信息完整性及加密算法主要用于eNodeB/gNodeB和UE之间的用户面传输安全。核心网侧完成用户信息存储及转发。

**图1** NSA双连接安全参数传递流程

<br>

![](特性概述_93270611.assets/zh-cn_image_0230001177_2.png)

1. Attach Request和TAU Request（周期性TAU请求例外）中携带DCNR标志位和UE additional security capability信元。
2. eNodeB向MME发送Initial UE Message（携带UE additional security capability信元）。
3. MME需要在Security Mode Command消息中携带Replayed UE additional security capability信元给UE。该信元指示MME能够识别处理UE带上来的UE additional security capability信元。
4.
  MME在以下消息中携带NR UE Security Capabilities信元给eNodeB/gNodeB：Initial Context Setup Request/UE Context Modification Request/Path Switch Request Acknowledge/Handover Request。

  > **说明**
  > MME将安全上下文中的UE additional security capability信元映射到NR UE Security Capabilities信元，即最终MME通过NR UE Security Capabilities信元，把UE的通过UE additional security capability信元带上来的安全能力传递给eNodeB/gNodeB，用于eNodeB、gNodeB和UE之间的安全协商。

[**流量上报功能**](#ZH-CN_CONCEPT_0193270611__li_02)

eNodeB在消息中携带5G流量统计报告（Secondary RAT Usage Report List，参见3GPP TS 36413），MME从eNodeB发送来的S1接口消息中解析到Secondary RAT Usage Report List信元后，根据当前的流程状态进行上报计费流量给S-GW/P-GW。如果当前的流程状态是暂态，则直接通过已有消息上报到S-GW。如果是稳态，则由MME单独发起Change Notification流程进行流量上报。 在分离、PDN去连接以及4G到2/3G RAU流程中，通过调整消息处理时序，使得获取5G流量报告早于向S-GW释放承载相关资源。保证这些流程中计费流量的正确上报。

eNodeB在如下消息中携带5G流量统计报告（Secondary RAT Usage Report List）：

- E-RAB Modify Response
- E-RAB Release Response
- E-RAB Release Indication
- E-RAB Modification Indication
- UE Context Release Request
- UE Context Release Complete
- UE Context Suspend Request
- Secondary RAT Data Usage Report

MME通过如下消息中携带Secondary RAT Usage Data Report信元，向S-GW或P-GW上报5G用户流量，详细请参见3GPP TS 29274:

- Create Session Request
- Modify Bearer Request
- Delete Session Request
- Delete Bearer Response
- Delete Bearer Command
- Release Access Bearers Request
- Modify Access Bearers Request

5G流量统计报告转发流程如 [图2](#ZH-CN_CONCEPT_0193270611__fig01) 所示。

**图2** 5G流量统计报告转发流程

<br>

![](特性概述_93270611.assets/zh-cn_image_0193270763_2.png)

1. 从eNodeB中发来的消息中进行解码，判断消息中是否有计费流量报告信元Secondary RAT Usage Report List。
    - 是，执行[2](#ZH-CN_CONCEPT_0193270611__step2)。
    - 否，继续既有处理流程。
2. 根据UE network capability信元指示的DCNR能力，判断是否是DCNR用户。
    - 是，执行[3](#ZH-CN_CONCEPT_0193270611__step3)。
    - 否，继续既有处理流程。
3. 根据[**SET NSACTRL**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/NSA组网管理/NSA控制参数/设置NSA控制参数(SET NSACTRL)_26305942.md)的参数“NR流量上报”，判断MME是否支持上报NR流量统计报告功能。
    - 是，执行[4](#ZH-CN_CONCEPT_0193270611__step4)。
    - 否，继续既有处理流程。
4. MME判断所有流量统计报告是否已通过既有消息完成上报。
  > **说明**
  > MME会缓存5G流量统计报告，等流程中有消息发送给S-GW时，MME在消息中携带缓存的流量统计报告。等流程结束，MME会检查缓存的流量统计报告。
    - 是，流程结束。
    - 否，执行[5](#ZH-CN_CONCEPT_0193270611__step5)。
5. MME发起Change Notification消息给S-GW上报用户计费流量报告。流程结束。

[**5G用户信息对等网元间传递功能**](#ZH-CN_CONCEPT_0193270611__li_03)

NSA用户进行Inter流程时， SGSN/ MME通过Gn/S10接口传递5G用户信息至对等网元确保新侧MME /SGSN 能够继续为NSA用户提供连续服务，如 [表1](#ZH-CN_CONCEPT_0193270611__tab1_1) 所示。

*表1 功能概述*

| 接口 | 消息 | 信元 | 比特位 | 含义 |
| --- | --- | --- | --- | --- |
| Gn | SGSN Context Response，<br>Forward Relocation Request | MS Network Capability | Dual connectivity of E-UTRA with NR capability（octet 4, bit 1） | 指示终端的DCNR能力。 |
| Gn | SGSN Context Response，<br>Forward Relocation Request | Access Restriction Data | NR as Secondary RAT Not Allowed（octet 1 bit1） | 指示签约数据中是否限制终端的DCNR能力。 |
| S10 | Context Response，Forward Relocation Request，Identification Response | UE Network Capability | Dual connectivity with NR (DCNR) (octet 7, bit 5) | 指示终端的DCNR能力。 |
| S10 | Context Response，Forward Relocation Request，Identification Response | MS Network Capability | Dual connectivity of E-UTRA with NR capability（octet 4, bit 1） | 指示终端的DCNR能力。 |
| S10 | Context Response，Forward Relocation Request，Identification Response | UE additional security capability | - | 指示终端的附加安全能力。 |
| S10 | Context Response，Forward Relocation Request，Identification Response | Extended Access Restriction Data | NR as Secondary RAT Not Allowed（octet 1 bit1） | 指示签约数据中是否限制终端的DCNR能力。 |
| S10 | Context Response，Forward Relocation Request，Identification Response | Aggregate Maximum Bit Rate (AMBR)/MBR/GBR信元 | - | 指示QoS能力。 |
