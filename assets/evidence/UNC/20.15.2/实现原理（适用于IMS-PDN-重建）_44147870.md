#### [P-CSCF Restoration能力通知](#ZH-CN_TOPIC_0000001144147870)

根据3GPP协议，MME在Attach/TAU流程中，通过Update Location Request消息中的“Supported Features：Feature-List-ID 2”信元（bit16=1）通知HSS其支持P-CSCF Restoration特性。

UNC 上：通过命令 [**ADD DMCMPTBYIMSI**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md) 或 [**SET DMCMPT**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md) 设置MME支持P-CSCF Restoration特性。

> **说明**
> - 若现网所有用户对应的HSS均支持本特性时，通过命令[**SET DMCMPT**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md)配置MME支持P-CSCF Restoration特性。若部分用户对应的HSS不支持本特性时（例如外网用户归属的HSS都不支持本特性），则可通过命令[**ADD DMCMPTBYIMSI**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md)来配置。
> - 根据3GPP 23.380定义，只要有一个用户通过Update Location Request消息通知HSS MME支持P-CSCF Restoration特性，HSS即认为整个MME支持此特性。

#### [VoLTE被叫恢复](#ZH-CN_TOPIC_0000001144147870)

根据协议3GPP 23.380定义，在VoLTE被叫场景下，S-CSCF收到INVITE消息时，如果检测到被叫用户所注册的P-CSCF故障，则通知HSS。HSS通知该用户注册的MME触发P-CSCF Restoration流程使该用户进行IMS PDN连接重建。

-
  仅P-CSCF故障，且PCO-based optional extension功能关闭时，VoLTE被叫恢复流程如 [图1](#ZH-CN_TOPIC_0000001144147870__fig02) 所示。

  ![](实现原理（适用于IMS PDN 重建）_44147870.assets/zh-cn_image_0000002525820908_2.png "点击放大")

  > **说明**
  > [图1](#ZH-CN_TOPIC_0000001144147870__fig02) 中，当用户有IMS PDN和其他PDN时，MME下发Deactivate EPS Bearer Context Request消息，携带的原因值可通过DWORD_EX12 BIT30进行设置：
  >
  > - BIT30=0，携带原因值#36 Regular deactivation。
  > - BIT30=1，携带原因值#39 Reactivation requested。
-
  同时启用了 **[WSFD-201203 S-GW/P-GW故障下的业务恢复](../../WSFD-201203 S-GW_P-GW故障下的业务恢复_91473803.md)** 特性时，S-GW/P-GW、P-CSCF同时故障情况下，MME收到Insert Subscription Data Request (IDR-Flags bit8 = 1)消息时，

  - S-GW故障：
      发现S-GW故障，MME向用户下发Detach Request消息，分离类型为“reattach required”。
    - P-GW故障：
      发现P-GW故障，MME会批量通知用户重建语音PDN连接。如果PDN连接已经重建，则不做处理，否则MME向UE下发Deactivate EPS Bearer Context Request，携带原因值，要求用户重建PDN连接。

      > **说明**
      > Deactivate EPS Bearer Context Request消息携带的原因值可通过DWORD_EX12 BIT30进行设置。
    - S-GW和P-GW都故障：
      同 **[WSFD-201203 S-GW/P-GW故障下的业务恢复](../../WSFD-201203 S-GW_P-GW故障下的业务恢复_91473803.md)** 特性的处理，MME向UE下发Detach Request消息，分离类型为“reattach required”。
