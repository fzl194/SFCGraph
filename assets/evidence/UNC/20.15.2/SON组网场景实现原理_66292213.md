# SON组网场景实现原理

**业务流程**

- Intra MME配置传输流程
  只通过一个MME的转接实现eNodeB配置信息的传输，发生在两个eNodeB都注册到同一个MME的场景下。MME通过eNodeB注册流程记录eNodeB标识和S1连接的关系，所以可以根据eNodeB配置传输消息中的目标eNodeB标识选择S1连接进行转发。

  **图1** Intra MME配置传输流程示意图

  <br>

  ![](SON组网场景实现原理_66292213.assets/zh-cn_image_0166439071_2.png)

  1. eNodeB和MME设备SCTP偶联建立起来后，两设备间第一条S1AP消息，用于交换设备信息。
    2. MME记录eNodeB标识和S1链路的关系和eNodeB所在的TAC（Tracking Area Code）。
    3. eNodeB1发送eNB Configuration Transfer消息给MME，该消息包括了SON Configuration Transfer IE信元，该信元中包括了目标eNodeB的标识信息。
    4. MME在记录中若未找到目的eNodeB标识则会继续查找目的TAC。若存在，则判断是intra MME，MME根据步骤2记录的eNodeB和S1链路的关系，找到消息中的目标eNodeB即eNodeB2的S1链路，转发eNodeB1的配置消息。
- Inter MME配置传输流程
  通过S10接口间Configuration Transfer Tunnel消息传递eNodeB配置信息。该场景下两个eNodeB分别注册到不同MME上。MME通过eNodeB的注册流程记录eNodeB标识和S1连接的关系，后续可以根据eNodeB配置传输消息中的目的eNodeB标识选择S1连接进行转发。

  **图2** Inter MME配置传输流程示意图

  <br>

  ![](SON组网场景实现原理_66292213.assets/zh-cn_image_0166439069_2.png "点击放大")

  1. eNodeB1和eNodeB2分别向MME1和MME2发送S1 Setup Request消息，用于交换设备信息。
    2. MME1和MME2分别记录eNodeB标识和S1链路的关系以及eNodeB所在的TAC（Tracking Area Code）。
    3. eNodeB1发送eNB Configuration Transfer消息给MME，该消息包括了SON Configuration Transfer IE信元，该信元中包括了目标eNodeB的标识信息。
    4. MME在记录中若未找到目的eNodeB标识则会继续查找目的TAC。若不存在，则判断是inter MME，根据目的TAC构建消息发送至DNS服务器进行查询，得到目的MME设备地址。
    5. MME1构造Configuration Transfer Tunnel消息封装eNodeB配置信息转发给目的MME即MME2。
    6. 目的MME即MME2找到消息携带的目的eNodeB标识，根据查到的S1AP连接转发配置信息给目的eNodeB即eNodeB2。
