## [命令功能](#ZH-CN_MMLREF_0000001194753519)

该命令用于在事件诊断成功情况下，查询事件治理的自愈结果。

## [注意事项](#ZH-CN_MMLREF_0000001194753519)

本命令只能查询事件诊断和自愈成功的事件结果，事件诊断和自愈失败的结果请使用 [**DSP EVTGOVSTAT**](显示事件诊断失败状态（DSP EVTGOVSTAT）_94913407.md) 查询。

#### [操作用户权限](#ZH-CN_MMLREF_0000001194753519)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001194753519)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| SERVICE | 服务名称 | 可选必选说明：可选参数<br>参数含义：该参数表示策略模型绑定的服务名称，若不输入此参数，则查询整系统所有服务名称的事件诊断和自愈成功的结果。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是0~255。<br>默认值：无<br>配置原则：无 |
| EVENTID | 事件ID | 可选必选说明：可选参数<br>参数含义：该参数显示事件的ID，若不输入此参数，则查询整系统某服务的所有事件ID的事件诊断和自愈成功的结果。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~4294967295。<br>默认值：无<br>配置原则：无 |

## [输出结果说明](#ZH-CN_MMLREF_0000001194753519)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 服务名称 | 该参数表示策略模型绑定的服务名称，若不输入此参数，则查询整系统所有服务名称的事件诊断和自愈成功的结果。 |
| 事件ID | 该参数显示事件的ID，若不输入此参数，则查询整系统某服务的所有事件ID的事件诊断和自愈成功的结果。 |
| 事件策略 | 该参数表示事件治理的策略类型。<br>取值说明：<br>- “ASSOCIATIONPOLICY（关联诊断）”：关联诊断策略<br>- “POLLINGPOLICY（轮询诊断）”：轮询诊断策略 |
| 自愈时间 | 该参数用于显示事件自愈发生的时间。 |
| 对象类型 | 该参数用于显示事件自愈对象的类型。<br>取值说明：<br>- “PROCESS（进程）”：进程类型<br>- “POD（Pod）”：Pod类型 |
| 对象ID | 该参数表示被自愈对象的标识符。 |
| 上报次数 | 该参数用于显示单个上报事件中出现次数最多的嫌疑实例的上报次数。例如：在某个事件诊断的某一时刻，共有3个上报事件：事件一（嫌疑实例为A、B、C），事件二（嫌疑实例为A、B），事件三（嫌疑实例为A，C），则出现次数最多的嫌疑实例为A，出现次数为3。 |
| 事件数量 | 该参数用于显示单个事件所有上报嫌疑实例的事件总数。 |
