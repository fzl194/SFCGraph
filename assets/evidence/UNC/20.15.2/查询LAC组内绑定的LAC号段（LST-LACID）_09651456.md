# 查询LAC组内绑定的LAC号段（LST LACID）

## [命令功能](#ZH-CN_MMLREF_0209651456)

**适用NF：GGSN**

该命令用来查询指定LAC号段与LAC组的绑定关系或者配置的所有LAC号段和LAC组的绑定关系。

## [注意事项](#ZH-CN_MMLREF_0209651456)

无

#### [操作用户权限](#ZH-CN_MMLREF_0209651456)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0209651456)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| LACSECNUM | LAC段编号 | 可选必选说明：可选参数<br>参数含义：该参数用于指定LAC段编号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~23999。<br>默认值：无<br>配置原则：无 |
| LACGROUPNAME | LAC组名 | 可选必选说明：可选参数<br>参数含义：该参数用于指定LAC组名。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是1~32。不支持空格，不区分大小写。<br>默认值：无<br>配置原则：<br>该LACGROUPNAME必须已经由ADD LACGROUP配置过。 |

## [使用实例](#ZH-CN_MMLREF_0209651456)

假设运营商需要查看指定LAC号段与LAC组的绑定关系：“LAC组名”为“beijing”，“LAC号段”为2：

```
LST LACID: LACGROUPNAME="beijing", LACSECNUM=2;
RETCODE = 0 操作成功。

结果如下
--------
LAC段编号 = 2
LAC起始ID = 0x0001
LAC截止ID = 0x0010
LAC组名 = beijing
(结果个数 = 1)
--- END
```

| LAC组名 | 该参数用于指定LAC组名。 |
| LAC起始ID | 该参数用于指定LAC起始ID。 |
| LAC截止ID | 该参数用于指定LAC截止ID。 |
