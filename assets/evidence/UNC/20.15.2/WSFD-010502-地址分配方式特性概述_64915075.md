#### [适用NF](#ZH-CN_CONCEPT_0164915075)

GGSN、 PGW-C、SMF

#### [定义](#ZH-CN_CONCEPT_0164915075)

UNC 支持通过UDM分配、Radius分配、SMF分配、DHCP分配方式为用户分配IP地址，可以为用户分配IPv4地址、IPv6地址或双栈地址（IPv4和IPv6地址）。

#### [客户价值](#ZH-CN_CONCEPT_0164915075)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 可以根据不同的组网情况，灵活的选择地址分配方式。 |
| 用户 | 用户不感知。 |

#### [应用场景](#ZH-CN_CONCEPT_0164915075)

应用于UE使用IP地址接入网络接受服务的场景。

#### [原理概述](#ZH-CN_CONCEPT_0164915075)

**UDM分配方式**

UNC 支持UDM分配方式为用户分配IP地址，指用户所分配的IP地址是UE在HLR/HSS/UDM签约时确定的IP地址。为了让指定用户选择到特定的UPF，享受指定的服务，比如企业用户定制服务，可以在 UNC 上配置静态IP地址网段与UPF的绑定，为指定用户选择到特定的UPF。

- UE接入2/3G核心网场景下，承载上下文激活过程中，UE发送Activate PDP Context Request消息给SGSN，消息的PDP Address信元中携带UE的IPv4地址/IPv6前缀。SGSN验证UE提供的PDP地址和PDP上下文签约记录，然后通过Create PDP Context Request消息的End User Address信元将UE的IP地址信息发送给GGSN。GGSN将收到的静态IP地址/IPv6前缀发送给UE使用。
- UE接入4G核心网场景下，在默认承载上下文激活过程中，MME根据UE的请求和从HSS处获取的签约数据，在Create Session Request消息的PDN Address Allocation（PAA）信元中携带已获取的IPv4地址或IPv6前缀给SGW-C。SGW-C再通过Create Session Request消息将PAA信元发送给PGW-C，PGW-C将收到的静态IP地址发送给UE使用。
- UE接入5G核心网场景下，在PDU会话建立时，SMF通过调用UDM的服务操作Nudm_SDM_Get获取UE在UDM签约时确定的静态IP地址/前缀，之后通过PDU会话建立响应消息发送给UE。

**Radius地址分配方式**

UNC 支持Radius分配方式为用户分配IP地址，指为用户分配的IP地址是在PDP/承载/会话上下文激活阶段在用户鉴权过程中从Radius服务器获取的IP地址，通过Radius认证过程从Radius服务器获取用户IP地址，地址直接携带在服务器返回的接入响应消息中。

**SMF分配方式**

UNC 支持SMF为用户分配IP地址，指为用户分配的IP地址是在PDU会话建立阶段直接从SMF的本地地址池中获取的IP地址。

UNC 收到用户的激活请求，如果用户所属APN配置的是本地地址分配方式，则 UNC 从用户所属APN绑定的本地地址池或Radius地址池中为用户分配一个IP地址，这个地址可以是IPv4地址，也可以是IPv6地址前缀。

当用户去激活后， UNC 可设置为该用户分配的IP地址延迟释放，待本地地址池中IP地址的等待释放时间到达后， UNC 可以将该地址重新分配给其他用户。如果在租约期内该用户再次创建PDU会话，可继续使用原IP地址 。

**DHCP地址分配方式**

DHCP分配地址的方式是指移动用户所分配的IP地址是在PDU会话建立阶段从DHCP服务器上配置的地址池中获取的动态IP地址。 UNC 作为DHCP Client，代理用户向DHCP Server申请IP地址。

UNC 收到用户的激活请求，携带代理IP向DHCP Server/DHCPv6 Server申请IP地址，DHCP Server/DHCPv6 Server从与该代理IP相同网段的地址池中为用户分配一个IPv4/IPv6地址。
