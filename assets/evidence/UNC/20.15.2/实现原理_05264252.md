#### [Active UE从WLAN网络到5G网络的切换](#ZH-CN_CONCEPT_0000001505264252)

Active UE从WLAN网络到5G网络的切换流程如 [图1](#ZH-CN_CONCEPT_0000001505264252__fig1878713308528) 所示。

**图1** Active UE从WLAN网络到5G网络的切换

<br>

![](实现原理_05264252.assets/zh-cn_image_0000001562714481_2.png "点击放大")

1. UE向AMF发送NAS消息，该消息中包括S-NSSAI、DNN、PDU session ID、Requested PDU Session Type（Request Type）、[PDU Session Establishment Request](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/PDU session establishment request_76783928.md)等信息。消息中Request Type为“Existing PDU Session”，指示将当前会话从Non-3GPP网络切换到3GPP网络。
2. Active UE从WLAN网络到5G网络的切换流程中，AMF当前仅根据FQDN信息为PDU会话建立选择SMF。AMF在UE注册流程中会收到UDM发送的Nudm_SDM_GetMultipleDataSets Response消息携带PGW ID的FQDN；当PGW ID的FQDN发生变化，UDM会通过Nudm_SDM_SubscribeNotify Request消息通知AMF。
3. AMF向SMF发送[Nsmf_PDUSession_CreateSMContext](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_CreateSMContext_83734925.md)Request请求创建SM上下文。
4. **可选：**如果SMF没有为该PDU会话进行注册，SMF将通过Nudm_UECM_Registration （Nudm_UECM_RegistrationSMF Request）向UDM注册PDU会话。当SUPI、DNN、S-NSSAI相应的会话管理订阅数据不可用时，SMF通过Nudm_SDM_Get（Nudm_Get UE Session Management Subscription Data Request）消息获取会话管理订阅数据，通过Nudm_SDM_Subscribe（ Nudm_Subscribe Create Request）消息获取订阅数据变更的通知。
5. SMF向AMF返回[Nsmf_PDUSession_CreateSMContext](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_CreateSMContext_83734925.md)Response确认接受创建PDU会话。
6. SMF为PDU会话选择PCF。如果没有部署动态PCC，SMF可能采用本地策略获取PCC规则。
7. **可选：**SMF选择PCF后，SMF与PCF建立SM策略关联。
    a. SMF向PCF发送[Npcf_SMPolicyControl_Create](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Create_67572324.md)Request消息，请求创建SM策略关联，消息中携带pduSessionId、pduSessionType、dnn、notificationUri和Snssai等信息。
    b. PCF根据用户标识查找出用户签约的业务数据，根据用户的会话信息进行策略判断，生成规则，通过[Npcf_SMPolicyControl_Create](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Create_67572324.md)Response消息发送给SMF，消息中会包含PCF要求订阅的事件，当这些事件发生时，比如时区变更，位置变更，需要SMF发送[Npcf_SMPolicyControl_Update](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Update_67572331.md)Request消息给PCF。
8. SMF根据UE位置、DNN、S-NSSAI等信息选择UPF，并根据UE在UDM中签约时确定的IP地址为UE分配IP地址 ，也可以由UPF根据本地地址池为UE分配IP地址。
9. **可选：**如果之前PCF订阅的事件发生时，SMF会向PCF上报相应的信息，比如时区变更，位置变更。SMF也可以通过该流程向PCF上报分配的UE IP地址/前缀。
    a. SMF发送[Npcf_SMPolicyControl_Update](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Update_67572331.md)Request消息给PCF，其中携带事件及相关信息。
    b. PCF根据SMF上报的会话数据和本地的用户签约数据重新进行策略判断，生成新的规则，通过[Npcf_SMPolicyControl_Update](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Update_67572331.md)Response消息将新的规则下发给SMF。
10. SMF建立与UPF的连接，提供用于该PDU会话的数据监测，报告规则，CN隧道信息等。
11. SMF向AMF发送[Namf_Communication_N1N2MessageTransfer](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Namf/Namf简介/Namf接口服务操作介绍_84640741.md)消息，携带的信息包括发送给(R)AN的N2 SM Information，其中包含QFI、 QoS Profile、CN Tunnel Info等信息，发送给UE的N1 SM Container，其中包含[PDU Session Establishment Accept](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/PDU session establishment accept_76783876.md)、Allocated IPv4 Address等信息，通知(R)AN和UE需要建立PDU会话。
12. AMF向(R)AN发送[PDU Session Resource Setup Request](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PDU SESSION RESOURCE SETUP REQUEST_76784445.md)，请求中包含N2 SM Information，包含QFI、QoS Profile、CN Tunnel Info，发送给UE的N1 SM Container，包含[PDU Session Establishment Accept](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/PDU session establishment accept_76783876.md)、Allocated IPv4 Address。
13. (R)AN与UE发起信令交互，将SMF需要发送给UE的PDU Session ID、N1 SM Container消息转发至UE，请求UE建立PDU会话，消息中包含[PDU Session Establishment Accept](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/PDU session establishment accept_76783876.md)、Allocated IPv4 Address。
14. (R)AN向AMF发送[PDU Session Resource Setup Response](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PDU SESSION RESOURCE SETUP RESPONSE_76785226.md)，建立AN隧道信息，响应消息中包含AN Tunnel Info、 List of Accepted/Rejected QFI等。
15. AMF向SMF发送[Nsmf_PDUSession_UpdateSmContext](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext_83854801.md)Request（Nsmf_Update_SMContext Request），将从(R)AN接收到的N2 SM Information转发给SMF。
16. SMF向UPF发送[PFCP Session Modification Request](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)（N4 Session Modification Request），将AN隧道信息和相应的转发规则发送给UPF，关键信元如[表1](../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/PFCP交互流程/PFCP会话管理流程/PFCP会话修改流程_76785234.md)所示。UPF发送[PFCP Session Modification Response](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)(N4 Session Modification Response)。
17. SMF向AMF发送[Nsmf_PDUSession_UpdateSmContext](../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext_83854801.md)Response（Nsmf_Update_SMContext Response）。
18. SMF向ePDG发送Delete Bearer Request，消息中包含EPS Bearer Identity或 Linked EPS Bearer Identity。
19. ePDG发起资源释放流程。
20. ePDG向SMF回复Delete Bearer Response。
21. **可选：**当PDN连接对应的资源在PGW-C中释放时，PGW-C中通知3GPP AAA Server PDN断开。如果UE在3GPP AAA Server中不再有任何上下文，3GPP AAA Server将通知HSS。

#### [Active UE从5G网络到WLAN网络的切换](#ZH-CN_CONCEPT_0000001505264252)

Active UE从5G网络到WLAN网络的切换流程如 [图2](#ZH-CN_CONCEPT_0000001505264252__fig5) 所示。

**图2** Active UE从5G网络到WLAN网络的切换

<br>

![](实现原理_05264252.assets/zh-cn_image_0000001511914400_2.png "点击放大")

1. UE在5GS注册并建立PDU会话。注册流程详细请参考[初始注册](../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/注册流程/初始注册_47473401.md)，PDU会话建立流程详细请参考[PDU会话建立](../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/会话流程/PDU会话建立_97353800.md)。
2. UE发现非可信非3GPP接入网络WLAN，决定切换到该WLAN网络。
3. UE执行非3GPP接入系统的鉴权和授权流程，3GPP AAA Server查询HSS，获取IMSI、PGW-C标识并在授权过程中返回给非3GPP接入系统。
4. 鉴权和授权成功后，触发附着流程。UE向ePDG发起认证请求，ePDG向3GPP AAA Server请求UE切换前连接的PDN相关信息及用户签约信息。
5. ePDG向PGW-C发送Create Session Request消息，建立PDN连接。消息中携带IMSI、APN、RATtype、ePDG TEID for control plane、PDN Address Allocation、EPS Bearer Identity、Default EPS Bearer QoS、ePDG Address for the user plane、ePDG TEID of the user plane、APNAMBR、Selection Mode、Dual Address Bearer Flag、Trace Information、Charging Characteristics、UE Local IP Address IE、UE UDP Port和Indication Flags等参数。PGW-C使用IMSI和APN找到PDN连接，如果同一APN下建立了多个PDN连接，PGW-C通过比较用户的IP地址确定是哪个PDN连接发生切换。
6. **可选：**NG-RAN到ePDG切换会导致RAT，IP-CAN Type，AN-GW address等事件更新，如果PCF签约了相关的Event-trigger，则PGW-C发起IP-CAN会话更新流程。
7. PGW-C向3GPP AAA Server更新PGW-C标识和APN信息，消息中包含PGW-C所在的PLMN信息。
8. PGW-C向ePDG返回Create Session Response消息，消息中携带PDN GW Address for the user plane、PDN GW TEID of the user plane、PDN GW TEID of the control plane、PDN Address Allocation、EPS Bearer Identity、EPS Bearer QoS、APN-AMBR、Charging ID、Cause等参数。
9. ePDG将PGW-C分配给UE的IP地址传给UE，附着流程完成。附着流程详细请参考[WSFD-010102 Untrusted Non-3GPP网络用户接入](../../基本接入功能/WSFD-010102 Untrusted Non-3GPP网络用户接入/实现原理_80563510.md)。
10. **可选：**如果UE需要连接多个PDN，重复步骤3～9，完成其他PDN连接的建立。
11. PGW-C/SMF通过5G网络接入发起网络请求的PDU Session Resource Release Command，释放5GC和NG-RAN资源。
