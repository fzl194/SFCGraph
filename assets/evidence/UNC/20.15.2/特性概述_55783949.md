#### [适用NF](#ZH-CN_CONCEPT_0000001555783949)

PGW-C、SMF

#### [定义](#ZH-CN_CONCEPT_0000001555783949)

NG-RAN（NG Radio Access Network）和WLAN（Wireless Local Area Network）互操作指用户可以在3GPP 5G网络和非3GPP WLAN间进行切换，而无需中断业务。

#### [客户价值](#ZH-CN_CONCEPT_0000001555783949)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 支持5G网络和WLAN网络用户同时接入，为用户切换提供平滑的服务。<br>- 在5G网络上进行WiFi热点覆盖，减轻5G网络的带宽压力，利用WiFi资源为用户提供更稳定的服务。 |
| 用户 | 当用户在5G网络与WLAN之间切换时，保证业务连续性。 |

#### [应用场景](#ZH-CN_CONCEPT_0000001555783949)

5G网络和WLAN的切换可支持语音业务和数据业务的连续性。如语音通话，消息业务，浏览，文件下载，流媒体等场所下的连续性保障。应用场景包括：

- 用户从5G网络初始接入，切换到WLAN网络场景。
- 用户从WLAN网络初始接入，切换到5G网络场景。

#### [可获得性](#ZH-CN_CONCEPT_0000001555783949)

**涉及NF**

| 涉及NF | 版本支持 | 功能说明 |
| --- | --- | --- |
| UE | 无特殊要求 | 支持在WLAN接入和5G网络接入之间的切换。 |
| ePDG | 无特殊要求 | UE到PGW-C、SMF之间的上下行数据转发。 |
| AMF | UNC 20.10.2及后续版本 | 根据S-NSSAI、DNN等信息为PDU会话建立选择SMF。基于HSS提供的PGW-C Identity来获取PGW-C地址。 |
| PGW-C/SMF | UNC 20.10.2及后续版本 | UE IP地址分配和管理，控制和选择UPF。 |
| PGW-U/UPF | 无特殊要求 | - 根据本地地址池为UE分配IP地址。<br>- 建立PDU会话的CN隧道端口。<br>- 报文解析和策略执行、流量使用量上报。 |
| PCF | 无特殊要求 | 根据用户标识查找出用户签约的业务数据，根据用户的会话信息进行策略判断，生成规则。 |
| HSS/UDM | 无特殊要求 | - 通过N8接口与AMF交互PGW-C Identity。<br>- 通过SWx接口与3GPP AAA Server交互PGW-C Identity。<br>- 注册PDU会话，管理会话管理订阅数据。 |
| 3GPP AAA Server | 无特殊要求 | 存储PGW-C Identity。 |
| NG-RAN | 无特殊要求 | - 5G网络用户接入。<br>- 建立AN隧道端口。 |

本特性对系统无影响。

#### [应用限制](#ZH-CN_CONCEPT_0000001555783949)

- 本特性仅在N11SMF场景下支持（无I-SMF/V-SMF），不支持国际漫游场景、不支持UE进行跨区域移动到超出了原来SMF服务范围的新区域场景。
- AMF不支持通过IP方式发现SMF。
- 当SMF上同时存在WLAN会话和NG-RAN会话时，不支持WLAN会话先切到LTE网络，NG-RAN会话再切换到LTE网络。

#### [原理概述](#ZH-CN_CONCEPT_0000001555783949)

如 [图1 5G网络和WLAN网络之间切换示意图](#ZH-CN_CONCEPT_0000001555783949__fig1) 所示，UE从5G网络初始接入后切换到WLAN网络，或UE从WLAN网络初始接入后切换到5G网络，为了保证业务不中断，启用该特性提高用户体验。

**图1** 5G网络和WLAN网络之间切换示意图

<br>

![](特性概述_55783949.assets/zh-cn_image_0000001511744544_2.png "点击放大")

- UE从5G网络初始接入，切换到WLAN网络场景
    1. UE首先在5G网络完成PDU会话的建立和激活：AMF通过N11接口与SMF+PGW-C进行交互开始业务流程。随后通过NG-RAN及N3接口与UPF+PGW-U进行数据包传输。
    2. UE发现非可信，非3GPP协议的WLAN接入网络并与基站传递信息后，基站比较信号强弱，决定切换到该WLAN网络。
    3. UE发起非3GPP接入系统的鉴权和授权流程，3GPP AAA Server查询HSS+UDM，获取IMSI、SMF+PGW-C标识并在授权过程中返回给ePDG。
    4. 鉴权和授权成功后，触发切换流程。UE向ePDG发起认证请求，ePDG向3GPP AAA Server请求UE切换前连接的PDN相关信息及用户签约信息（包括切片，QoS等业务信息）。
    5. ePDG获得SMF+PGW-C标识后，通过S2b-C接口与SMF+PGW-C进行信令交互，完成会话建立后，向UE通知鉴权完成，此时切换完成，UE通过WLAN网络与核心网进行数据交换。
  > **说明**
  > 在UE从5G网络切换到WLAN网络的过程中，需要通过ePDG和3GPP AAA Server两个网元建立与核心网间的联系。其中：
  >
  > - 3GPP AAA Server起到获取，保存用户签约信息的作用，WLAN网络下PGW-C，ePDG等网元对用户签约信息的获取都需要通过拜访3GPP AAA Server达成。
  > - ePDG起到对用户进行鉴权的作用。同时，ePDG也会向3GPP AAA Server获取用户会话相关签约信息，发起用户的会话创建，建立用户的会话用户面通道，帮助用户完成切换流程。

- UE从WLAN网络初始接入，切换到5G网络场景
    1. UE通过WLAN网络与核心网进行数据交换：ePDG通过S2b-C接口与SMF+PGW-C进行交互开始业务流程，并通过S2b-U接口与UPF+PGW-U进行数据包传输。
    2. SMF+PGW-C通过S6b接口向3GPP AAA Server提供PGW-C Identity（3GPP AAA Server通过SWx接口把PGW-C Identity注册到HSS+UDM上）。
    3. UE发现5G网络，并与基站传递信息，基站比较信号强弱后决定切换到该5G网络。
    4. UE切换到5G网络后，发起PDU会话切换流程。AMF获取UDM提供的PGW-C Identity，继续会话建立流程。
