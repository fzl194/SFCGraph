## [操作场景](#ZH-CN_OPI_0275538863)

本操作指导介绍在运行网络中激活MME链式备份的操作过程。

MME Pool组网可以通过Pool内负荷分担实现Pool内MME间的数据业务的容灾，但在MME故障后的第一个TAU周期内，其之前服务的用户如果进行VoLTE业务，则仍可能因新MME上没有用户的上下文信息而经历主叫电话失败、接不到被叫电话等情况，影响用户体验。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0275538863)

前提条件

- 请仔细阅读[WSFD-201201 MME链式备份特性概述](特性概述_75543777.md)。
- 完成加载License。
- 完成激活[WSFD-104101 MME Pool](../../容灾功能/MME容灾方案/容灾原理/WSFD-104101 MME Pool_45935187.md)。
- MME Pool内，用户的安全配置必须都开启鉴权加密保护，即将[**MOD S1USRSECPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/S1模式用户安全参数/修改S1模式用户安全配置(MOD S1USRSECPARA)_26145650.md)命令设置“安全策略(SECPLC)”参数为“AUTHANDPROTECTED(鉴权并保护)”。
- IMS系统与HSS侧设定的T-ADS域选流程超时时长应大于6秒。如果IMS系统与HSS侧设定的T-ADS域选流程超时时长小于MME上业务恢复所需时长，IMS或HSS误判流程超时，最终导致T-ADS域选流程失败。
- IMS系统与HSS侧设定的位置/状态业务查询流程超时时长应大于6秒。如果IMS系统与HSS侧设定的位置/状态业务查询超时时长小于MME上业务恢复所需时长， IMS或HSS误判流程超时，最终导致位置/状态业务查询流程失败。
- 已经完成Sdup接口的配置，参见[配置到MME的数据（Sdup接口）](../../../../初始配置/UNC初始配置与调测/组网对接配置/配置AMF&MME&SGSN/配置MME/配置到MME的数据（Sdup接口）_82467052.md)。
- DRA已经完成S6a接口容灾配置。
- SGW-C已经完成容灾配置。

## [操作步骤](#ZH-CN_OPI_0275538863)

1. 进入 “MML命令行-UNC” 窗口。
2. 配置S6a接口容灾信息。
  [**ADD DMVLE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter虚拟本地实体/增加Diameter虚拟本地实体(ADD DMVLE)_26146296.md)
  > **说明**
  > HSS-DRA-MME组网下，将Pool内的一个MME的主机名配置为虚拟本端实体。此MME的选取原则如下：
  >
  > 根据MMEC，将Pool内所有MME从大到小进行排序。在MMEC较大的MME上，将MMEC较小的MME配置为虚拟本端实体，使MMEC较大的MME作为MMEC较小MME容灾信令实体。在MMEC最小的MME上，将MMEC最大的MME配置为虚拟本端实体，使MMEC最小的MME作为MMEC最大的MME的容灾信令实体。
3. 配置业务恢复模式。
  [**SET RESTOFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/MME容灾管理/容灾功能管理/设置容灾功能(SET RESTOFUNC)_72345713.md)
4. （可选）如希望仅备份指定号段内的用户信息时，配置用户号段。
  [**ADD RESTOUSRSEG**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/MME容灾管理/容灾用户号段管理/增加容灾用户号段参数 (ADD RESTOUSRSEG)_10674069.md)
5. 配置SGW-C对接的属性。
    - 所有的SGW-C支持NTSR功能。
      [**MOD SGWCHARACT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/S11接口管理/S-GW属性/修改S-GW特性对接配置(MOD SGWCHARACT)_26145970.md)
    - 部分SGW-C支持NTSR功能。
      [**ADD SGWCHARACT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/S11接口管理/S-GW属性/增加S-GW特性对接配置(ADD SGWCHARACT)_26305778.md) :
6. **可选：**配置容灾APN特征参数。
  [**ADD RESTOAPN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/MME容灾管理/容灾APN特征管理/增加容灾APN特征参数(ADD RESTOAPN)_26305924.md)
  > **说明**
  > - 当需要为“IMS”之外的其他IMS APN（例如“IMS-X”）提供容灾备份功能时，需要执行本命令。
  > - 此APN为支持容灾备份的APN，必须和UDG系统中配置的一致。否则可能导致主被叫业务无法一次恢复。
  > - 此步骤中，无需配置紧急呼叫APN，UNC可以获取已经配置的紧急呼叫APN。
7. 查询Context Request消息的GTP-C T3/N3参数。
  [**LST T3N3**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C协议参数配置/查询GTP-C T3_N3参数配置(LST T3N3)_72345521.md)
8. 设置Context Request Extended消息的GTP-C T3/N3参数，使其小于 [6](#ZH-CN_OPI_0275538863__cmd1721339228180640) 中的GTP-C T3/N3参数。
  [**SET T3N3**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/GTP-C接口管理/GTP-C协议管理/GTP-C协议参数配置/设置GTP-C T3_N3参数配置(SET T3N3)_26305730.md)
9. **可选：**调整4G鉴权集获取策略。
  > **说明**
  > 为减少业务恢复对HSS造成的信令冲击，建议执行本步骤，调整4G鉴权集获取策略。
  >
  > 执行 [**MOD S1USRSECPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/S1模式用户安全参数/修改S1模式用户安全配置(MOD S1USRSECPARA)_26145650.md) 命令后，S-MME会将未使用的鉴权集备份到R-MME。当S-MME故障时，业务恢复过程中，N-MME直接使用备份的鉴权集进行鉴权流程，如果鉴权成功，不会立即向HSS获取鉴权集。
  >
  > 执行 [**MOD S1USRSECPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/S1模式用户安全参数/修改S1模式用户安全配置(MOD S1USRSECPARA)_26145650.md) 命令后，4G用户从GERAN/UTRAN网络移动到E-UTRAN网络时，用户可能鉴权会失败，原因值为“#21 Synch failure”，MME会重新从HSS获取鉴权集并进行鉴权，Inter RAT流程成功。此过程会导致 [117491712 S1模式鉴权请求次数](../../../../../OM参考/性能指标/UNC业务性能指标/业务性能指标/SGSN_MME业务性能指标/USN性能指标/S1模式移动管理/S1模式安全性/117491712 S1模式鉴权请求次数_06831697.md) 和 [117491723 S1模式鉴权失败次数(#21 Synch failure)](../../../../../OM参考/性能指标/UNC业务性能指标/业务性能指标/SGSN_MME业务性能指标/USN性能指标/S1模式移动管理/S1模式安全性/117491723 S1模式鉴权失败次数(#21 Synch failure)_06831728.md) 指标值升高。
    a. 确认4G鉴权集获取策略是否存在如下情况之一。
          - “鉴权集数量”取值小于2。
          - “是否预取鉴权集”取值为NO。
      [**LST S1USRSECPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/S1模式用户安全参数/查询S1模式用户安全配置(LST S1USRSECPARA)_72225331.md) :;
          - 是，[9.b](#ZH-CN_OPI_0275538863__cmd1703984922180640)。
          - 否，[10](#ZH-CN_OPI_0275538863__set_licctrl)。
    b. 修改4G鉴权集获取策略。
      [**MOD S1USRSECPARA**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/业务安全管理/用户安全管理/S1模式用户安全参数/修改S1模式用户安全配置(MOD S1USRSECPARA)_26145650.md)
10. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)

## [任务示例](#ZH-CN_OPI_0275538863)

任务描述

针对所有用户启用MME链式备份特性。

脚本

//配置S6a接口容灾信息。

```
ADD DMVLE: LOINDEX=32, LOHSTNAME="mmec09.mmegi8001.mme.epc.mnc123.mcc456.3gppnetwork.org", LORLMNAME="epc.mnc123.mcc456.3gppnetwork.org";
```

//配置业务恢复模式。

```
SET RESTOFUNC: SRESTOM=FULL_RESTO, RESTOWORK=NORMAL, CHECKSCOPE=0;
```

//配置SGW-C对接的属性

```
ADD SGWCHARACT: RANGE=SPECIFIED, IPT=IPV4, IPV4="10.0.0.1", IPV4MSK="255.255.255.255", NTSR=YES;
```

//设置Context Request Extended消息的GTP-C T3/N3参数。

```
LST T3N3: GTPVER=GTPv2, V2MSGCLS=MM, V2SMMMSGTYPE=CONTEXT_REQ;
SET T3N3: GTPVER=GTPv2, V2MSGCLS=MM, V2SMMMSGTYPE=CONTEXT_REQ_EX, T3RES=1, N3REQ=2;
```

//打开本特性License开关。

```
SET LICENSESWITCH: LICITEM="LKV2MCR01", SWITCH=ENABLE;
```
