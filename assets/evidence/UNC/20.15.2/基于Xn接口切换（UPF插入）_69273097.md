# 基于Xn接口切换（UPF插入）

- [业务场景](#ZH-CN_TOPIC_0000001469273097__1.3.1.1)
- [信令流程](#ZH-CN_TOPIC_0000001469273097__1.3.2.1)

#### [业务场景](#ZH-CN_TOPIC_0000001469273097)

Handover流程用于将UE从一个源NG-RAN节点切换到一个目标NG-RAN节点，切换过程中使用Xn接口，基于Xn接口的NG-RAN间的Handover流程只发生在当前业务所在AMF的管理范围内。新的无线条件、负载均衡或特定的服务都可能触发切换流程，例如UE在AMF的管理范围内并处于连接态时，移动出了原有NG-RAN的服务范围，导致UE需要新的NG-RAN提供服务，此时会触发Handover流程。

该流程用于当用户移动出原UPF的用户服务范围，AMF未改变，SMF决定插入一个新的中间UPF（ Intermediate UPF，I-UPF）时发起的切换场景。

1. 交互流程：切换准备和切换开始。
  相关机制：源NG-RAN会根据目标NG-RAN所需要的消息格式提供必要的信息，并传输给目标NG-RAN。目标NG-RAN准备无线侧资源。
2. 消息交互：Target NG-RAN发送[PATH SWITCH REQUEST](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PATH SWITCH REQUEST_76784993.md)给AMF。
  关键信元：
  | User Location Information | 该IE用于提供UE的位置信息。 |
  | --- | --- |
  | UE Security Capabilities | 该信元用于携带UE的5G安全能力。 |
  | **PDU Session Resource**<br>**to be Switched in Downlink List** | 该信元用于携带需要切换的PDU会话列表，每个PDU会话包含会话ID和pathSwitchRequestTransfer。<br>pathSwitchRequestTransfer中包括目标gNB分配的N3接口AN隧道信息。 |
  | **PDU Session Resource Failed to Setup List** | 该信元用于携带需要拒绝的PDU会话列表 |
  相关机制：目标NG-RAN通知AMF用户已经移动到新的区域，并且提供需要切换的PDU会话列表。如果PDU会话的QoS流不被目标NG-RAN所接受或者目标NG-RAN不支持相应网络切片情况下，消息中会携带需要拒绝的PDU会话列表。
3. 消息交互：AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  关键信元：
  | ueLocation | 如果有UE的位置信息，该IE应包含UE的位置信息。 |
  | --- | --- |
  | n2SmInfoType | N2 SM消息类型，取值为PATH_SWITCH_REQ表示路径切换请求传输。 |
  相关机制：AMF携带NG-RAN侧的N2消息，并传递AN隧道信息。
4. 内部流程：UPF选择流程。
  相关机制：如果SMF选择一个新的UPF作为PDU会话的中间UPF，且需要使用不同的CN隧道信息，且CN隧道信息由UPF分配，则SMF向UPF (PSA) 发送消息请求分配CN隧道信息。
5. 消息交互：SMF发送[PFCP Session Modification Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)给UPF(PSA)。
  关键信元：
  | CP F-SEID | 指示控制面即SMF侧SEID，是控制面分配的标识会话的唯一标识，并用于后续信令交互。 |
  | --- | --- |
  相关机制：SMF向UPF (PSA) 发送消息请求分配N9接口的CN隧道信息。
6. 消息交互：UPF(PSA)发送[PFCP Session Modification Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)给SMF。
  关键信元：
  | Cause | 该信元应指示接受或拒绝相应的请求消息。 |
  | --- | --- |
  | UP F-SEID | 指示用户面即UPF侧SEID，是用户面分配的标识会话的唯一标识，并用于后续信令交互。 |
  | Local F-TEID | 指示UPF分配的用户面隧道端点标识。 |
  相关机制：如果UPF (PSA)分配UPF (PSA)的CN隧道信息(N9)，则向SMF提供CN隧道信息(N9)。
7. 消息交互：SMF发送[PFCP Session Establishment Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Establishment Request_32945585.md)给I-UPF。
  关键信元：
  | CP F-SEID | 指示控制面即SMF侧SEID，是控制面分配的标识会话的唯一标识，并用于后续信令交互。 |
  | --- | --- |
  相关机制：对于需要更新的PDU会话，如果UE已经移出了与服务NG-RAN节点连接的UPF服务区域，SMF选择一个I-UPF，向I-UPF发送消息。其中包含用于建立N9隧道的UPF(PSA)侧CN隧道信息，以及NG-RAN侧的AN隧道信息，如果该I-UPF的CN隧道信息由SMF分配，则SMF还向I-UPF提供I-UPF的上下行CN隧道信息。
8. 消息交互：I-UPF发送[PFCP Session Establishment Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Establishment Response_85720872.md)给SMF。
  关键信元：
  | Cause | 该信元应指示接受或拒绝相应的请求消息。 |
  | --- | --- |
  | UP F-SEID | 指示用户面即UPF侧SEID，是用户面分配的标识会话的唯一标识，并用于后续信令交互。 |
  | Local F-TEID | 指示UPF分配的用户面隧道端点标识。 |
  相关机制：如果I-UPF的CN隧道信息由UPF分配，则UPF向SMF发送I-UPF的上下行CN隧道信息。
9. 消息交互：SMF发送[PFCP Session Modification Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)给UPF(PSA)。
  关键信元：
  | CP F-SEID | 指示控制面即SMF侧SEID，是控制面分配的标识会话的唯一标识，并用于后续信令交互。 |
  | --- | --- |
  相关机制：SMF将I-UPF侧的N9接口的CN隧道信息传递给UPF(PSA)。
10. 消息交互：UPF(PSA)发送[PFCP Session Modification Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)给SMF。
  关键信元：
  | Cause | 该信元应指示接受或拒绝相应的请求消息。 |
  | --- | --- |
  相关机制：此时PDU会话UPF(PSA)开始通过I-UPF发送下行数据包给目标NG-RAN。为了保证目标NG-RAN数据包的顺序不混乱，在路径切换后，UPF(PSA)立即通过老的路径发送一个或多个“End Marker ”数据包给源侧NG-RAN，源侧NG-RAN再转发给目标NG-RAN。
11. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给AMF。
  关键信元：
  | N2SmInfoType | N2 SM消息类型，取值为PATH_SWITCH_REQ_ACK表示路径切换请求确认传输。 |
  | --- | --- |
  相关机制：SMF将I-UPF侧CN隧道信息传递给AMF。
12. 消息交互：AMF发送[PATH SWITCH REQUEST ACKNOWLEDGE](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PATH SWITCH REQUEST ACKNOWLEDGE_76785522.md)给Target NG-RAN。
  关键信元：
  | **PDU Session Resource**<br>**Switched List** | PDU会话资源切换列表。<br>包括PDU会话ID以及SMF通过AMF透传给NG-RAN的Path Switch Request Acknowledge Transfer IE。 |
  | --- | --- |
  | **PDU Session Resource**<br>**Released List** | PDU会话资源释放列表。<br>包括PDU会话ID以及SMF通过AMF透传给NG-RAN的Path Switch Request Unsuccessful Transfer IE。 |
  相关机制：携带N2 SM Information以及Failed PDU Sessions。NG-RAN侧获取到CN隧道信息，打通上行数据传输。
13. 交互流程：切换确认及资源释放。
  相关机制：目标NG-RAN通过发送Release Resources message给源NG-RAN，确认切换流程成功，触发源侧释放资源。
14. 子流程：[移动注册更新](../注册流程/移动注册更新_69433041.md#ZH-CN_TOPIC_0000001469433041)。
  相关机制：如果触发移动注册更新条件，则UE会发起移动性注册更新流程。
