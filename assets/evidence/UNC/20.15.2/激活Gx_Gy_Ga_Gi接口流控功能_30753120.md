## [操作场景](#ZH-CN_OPI_0230753120)

UNC 的周边网元存在过载风险时， UNC 能够基根据本地配置控制向各网元发送消息的速率，从而保护周边网元。

> **说明**
> 适用于 GGSN、 PGW-C。

## [操作步骤](#ZH-CN_OPI_0230753120)

1. 进入 “MML命令行-UNC” 窗口。
2. 设置 UNC 每秒发送给某PCRF的最大CCR消息数。
  **[ADD PCRF](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF/增加PCRF（ADD PCRF）_09897101.md)**
3. 设置 UNC 每秒发送给某OCS的最大消息数。
  **[ADD OCS](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/在线计费/OCS Diameter连接/OCS Server/增加OCS（ADD OCS）_09896954.md)**
4. 设置 UNC 每秒发送给CG的最大话单数
  **[ADD CG](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/离线计费/GTPP信令/CG管理/配置CG（ADD CG）_09896845.md)**
5. 设置 UNC 每秒发送某Radius计费服务器的最大消息数。
  **[ADD RDSSVR](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/RADIUS管理/连接管理/Radius服务器/增加RADIUS服务器（ADD RDSSVR）_09896756.md)**
6. **可选：**通过软参BYTE863配置Gx接口是否支持基于Gx集中点的WAL流控功能，以及流控阈值。
  **[SET SMFSOFTPARA](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/软件参数管理/设置SMF软参（SET SMFSOFTPARA）_09653002.md)**

## [任务示例](#ZH-CN_OPI_0230753120)

任务描述

设置 UNC 到周边网元PCRF/OCS/CG/AAA计费服务器的wal值，以保护周边网元，避免其系统过载。

脚本

//设置 UNC 每秒发送给某PCRF的最大CCR消息数。

```
ADD PCRF:HOSTNAME="pcrf1",REALMNAME="www.huawei.com", WALVALUE=1000;
```

//设置 UNC 每秒发送给某OCS的最大消息数。

```
ADD OCS:OCSHOSTNAME="ocs1",REALMNAME="www.huawei.com", WALVALUE=1000;
```

//设置 UNC 每秒发送给CG的最大话单数

```
ADD CG:IPVERSION=IPV4, IPV4ADDR="192.168.0.2",PORT=25009,PRIORITY=1,CDRTYPE=R9_PGW, WALVALUE=1000;
```

//设置 UNC 每秒发送某Radius计费服务器的最大消息数。

```
ADD RDSSVR: RDSSVRGRPNAME="rdsvr", SERVERTYPE=ACCOUNTING, IPVERSION=IPV4, SERVERIPV4="192.168.10.3", PORT=1813,  CIPHERKEY="123", WALVALUE=1000, CIPHERKEYCNFM="123";
```

//可选：通过软参BYTE863配置Gx接口是否支持基于Gx集中点的WAL流控功能，以及流控阈值。

```
SET SMFSOFTPARA:DT=BYTE,BYTENUM=863,BYTEVALUE=40;
```
