[**功能概述**](#ZH-CN_TOPIC_0277449219__sub1)

在大多数物联网行业应用中，终端发送数据报文频率较低且字节较小，但仅数据报文头所占用字节数有时就会超过数据，导致数据报文在传输过程中的有效字节数较低。因此本特性通过为UE在MME和S-GW/P-GW上创建Non-IP数据传输上下文来传输Non-IP数据（只有应用层数据，无UDP/IP传输层数据）来大幅度提升无线网络数据传输效率。支持Non-IP数据传输功能的流程为：

1. UE在附着请求建立Non-IP的PDN。
2. HSS下发Non-IP的签约数据给MME。
3. MME为该用户建立Non-IP的PDN。
4. UE使用该PDN，通过NAS消息携带不包含IP报头的数据进行数据传输。

> **说明**
> - Attach流程中由PDN Tpye=Non-IP信元和Non IP PDN Type Indicator=1来确定PDN类型为Non-IP。
> - 通过Non-IP接入方式进入网络时，终端在Attach Request消息中不携带IP地址。
> - 当现网中同一行业用户同时可以通过IP和Non-IP接入方式进入网络时，第5步中ULA消息中需要携带Additional Context Identifier信元来指向IP签约数据或Non-IP签约数据。

**[Non-IP数据传输Inter TAU流程](#ZH-CN_TOPIC_0277449219__sub1)**

Non-IP数据传输Inter TAU流程中目标MME向源MME发送Context Request消息时需要携带CIoT Optimization Support Indication信元，用来告知源MME，自己具有CIoT Optimization能力。其余流程与普通TAU流程一致。

> **说明**
> - Non-IP数据传输根据使用的数传方式，流程参考[WSFD-215101 基于信令面的数据传输](../../WSFD-215101 基于信令面的数据传输_75821879.md)或[WSFD-215102 NB-IoT用户面传输通道保活](../../WSFD-215102 NB-IoT用户面传输通道保活_75821880.md)。
> - Non-IP数据传输的MO与MT流程与[WSFD-215101 基于信令面的数据传输](../../WSFD-215101 基于信令面的数据传输_75821879.md)一致。

**[HSS发起IDR（Insert Subscriber Data Request）修改签约数据流程](#ZH-CN_TOPIC_0277449219__sub1)**

在修改签约数据时，对端HSS需要支持Non-IP签约数据的下发，流程触发场景为：

- 当有物联网行业用户同运营商签约了以Non-IP数据传输方式接入网络时，HSS发起IDR来增加签约数据。
- 当物联网行业用户同运营商修改接入方式（由IP变为Non-IP接入方式）时，HSS发起IDR来修改签约数据。
