## [命令功能](#ZH-CN_MMLREF_0000001428868621)

**适用NF：AMF、SMF**

该命令用于查询关键流程CHR上报管理参数。

## [注意事项](#ZH-CN_MMLREF_0000001428868621)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001428868621)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001428868621)

无

## [使用实例](#ZH-CN_MMLREF_0000001428868621)

查询关键流程CHR上报管理参数。

```
%%LST CHRMGMTPARA:;%%
RETCODE = 0  操作成功

结果如下
------------------------
  失败时允许上报CHR的流程  =  NF Discovery
        允许上报CHR的机制  =  NF Cache Update&Link Change
流程触发的CHR上报速率阈值  =  10
机制触发的CHR上报速率阈值  =  10
服务发现上报CHR的抑制时间  =  20
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001428868621)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 失败时允许上报CHR的流程 | 该参数用于指定失败时允许上报CHR的流程。 |
| 允许上报CHR的机制 | 该参数用于指定允许上报CHR的机制。 |
| 流程触发的CHR上报速率阈值 | 该参数用于指定流程触发的CHR上报速率阈值，用于控制整系统每秒允许上报的，由流程触发的CHR单据个数。 |
| 机制触发的CHR上报速率阈值 | 该参数用于指定机制触发的CHR上报速率阈值，用于控制整系统每秒允许上报的，由机制触发的CHR单据个数。 |
| 服务发现上报CHR的抑制时间 | 该参数用于指定每个Pod内同种服务发现条件组合的CHR上报抑制时间。 |
