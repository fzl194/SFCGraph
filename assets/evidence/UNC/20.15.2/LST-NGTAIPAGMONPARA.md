## [命令功能](#ZH-CN_MMLREF_0000001894289558)

**适用NF：AMF**

该命令用于查询NG TAI寻呼异常监控检测参数信息。

## [注意事项](#ZH-CN_MMLREF_0000001894289558)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001894289558)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001894289558)

无

## [使用实例](#ZH-CN_MMLREF_0000001894289558)

查询NG TAI组寻呼异常监控检测参数信息。

```
%%LST NGTAIPAGMONPARA:;%%
RETCODE = 0  操作成功

结果如下
------------------------
                     TAI寻呼异常监控开关  =  关闭
        N2模式TAI组寻呼成功率初始值（%）  =  90
     N2模式TAI组寻呼请求次数起始监控阈值  =  1000
      N2模式TAI组寻呼成功率降幅阈值（%）  =  5
N2模式TAI组二次寻呼请求次数增幅阈值（%）  =  20
         TAI寻呼消息启控连续异常周期个数  =  1
                 语音业务TAI寻呼流控开关  =  关闭
                        寻呼最大重试次数  =  0
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001894289558)

| 输出项名称 | 输出项解释 |
| --- | --- |
| TAI寻呼异常监控开关 | 该参数用来设置是否开启NG TAI寻呼异常监控功能。 |
| N2模式TAI组寻呼成功率初始值（%） | 该参数用于设置系统初始状态下N2模式TAI组寻呼成功率基准值。后续根据本初始值自学习N2模式TAI组寻呼成功率基准值。 |
| N2模式TAI组寻呼请求次数起始监控阈值 | 该参数用来设置最小业务量。在统计周期内，当实际业务量小于最小业务量时，TAI寻呼流控检测功能失效。业务量是指“指定TAI组的N2模式寻呼请求次数”的性能指标值。 |
| N2模式TAI组寻呼成功率降幅阈值（%） | 该参数用于设置N2模式TAI组寻呼成功率降幅阈值。该参数用于与“N2模式TAI组二次寻呼请求次数增幅阈值”联合判断是否发生寻呼风暴。TAI组寻呼成功率降幅和TAI组二次寻呼请求次数增幅均超过阈值，则AMF判定为发生寻呼风暴。 |
| N2模式TAI组二次寻呼请求次数增幅阈值（%） | 该参数用于设置N2模式TAI组二次寻呼请求次数增幅阈值。该参数用于与“N2模式TAI组寻呼成功率降幅阈值”联合判断是否发生寻呼风暴。TAI组寻呼成功率降幅和TAI组二次寻呼请求次数增幅均超过阈值，则AMF判定为发生寻呼风暴。 |
| TAI寻呼消息启控连续异常周期个数 | 该参数用于指定TAI对象实例连续寻呼异常周期个数。当异常周期数大于等于本参数值时，AMF启动寻呼流控。 |
| 语音业务TAI寻呼流控开关 | 该参数用于设置在TAI组对象处于寻呼风暴情况下，是否对语音寻呼进行流控。 |
| 寻呼最大重试次数 | 该参数用于指定在TAI对象处于寻呼风暴情况下，寻呼消息最大重试次数。 |
