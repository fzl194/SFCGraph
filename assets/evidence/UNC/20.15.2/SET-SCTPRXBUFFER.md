#### [命令功能](#ZH-CN_CONCEPT_0000001150932653)

![](设置SCTP接收缓冲区参数(SET SCTPRXBUFFER)_50932653.assets/notice_3.0-zh-cn_2.png)

该命令会改变SCTP通信缓存的使用方式，触发SGP进程内存占用率变化，影响S1接口和N2接口链路的接入。系统稳定运行场景下请慎用本功能。

**适用NF：SGSN、MME、AMF**

该命令用于设置系统中SCTP接收端缓冲区参数。设置参数时区分大缓冲区和小缓冲区，缓冲区分块规格包含max块、med块和min块。使用大缓冲区的接口有M3UA类、Diameter类、SBc、SGs和SLs；使用小缓冲区的接口有S1-MME和N2。SCTP会将接收到的报文根据大小存放到合适的块内。

#### [注意事项](#ZH-CN_CONCEPT_0000001150932653)

- 该命令执行后需要重启SGP进程才能生效。
- 该命令会影响SCTP内存分布。
- 该命令可以分别设置共享模式、私有模式下的接收端缓冲区参数。
- 该命令配置的参数仅在[SET SCTPBUFFCFG](设置SCTP缓冲区参数源(SET SCTPBUFFCFG)_03932862.md)的参数源类型设置为自定义时才会生效。
- 大缓冲区总量不能超过0XFFFFFFFF，大缓冲区总量 = BIGMINBFNUM * BIGMINBFSIZE + BIGMEDBFNUM * BIGMEDBFSIZE + BIGMAXBFNUM * BIGMAXBFSIZE。
- 小缓冲区总量不能超过0XFFFFFFFF，小缓冲区总量 = MINIMINBFNUM * MINIMINBFSIZE + MINIMEDBFNUM * MINIMEDBFSIZE + MINIMAXBFNUM * MINIMAXBFSIZE。

#### [本地用户权限](#ZH-CN_CONCEPT_0000001150932653)

manage-ug;system-ug

#### [网管用户权限](#ZH-CN_CONCEPT_0000001150932653)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [使用实例](#ZH-CN_CONCEPT_0000001150932653)

设置共享模式下的接收端缓冲区参数，指定大缓冲区min块的个数为100、大缓冲区min块的大小为2048字节、单偶联中大缓冲区min块最大数量为30：

```
SET SCTPRXBUFFER: MODE=SHARE, BIGMINBFNUM=100, BIGMINBFSIZE=2048, SHBMINLIM=30;
```
