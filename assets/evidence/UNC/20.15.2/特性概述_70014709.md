#### [适用NF](#ZH-CN_TOPIC_0170014709)

MME

#### [定义](#ZH-CN_TOPIC_0170014709)

UNC 根据终端的漫游属性、是否签约了IMS APN、UE的SRVCC能力、是否签约RFSP等因素精细化控制是否向终端提供VoLTE基础语音业务，以及在CSFB语音方案和VoLTE语音方案同时可用时的语音业务优选策略。

#### [客户价值](#ZH-CN_TOPIC_0170014709)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 丰富运营策略，可以为不同用户差异化提供VoLTE基础语音业务服务，增加收益。<br>- IMS网络部署早期难于实现国际漫游，网络容量也需要时间逐步扩展。针对不同用户提供精细化语音业务策略，可以使运营商从容部署IMS网络，语音业务稳步平滑过渡到IMS网络。<br>- 避免终端用户反复尝试不可用的语音业务承载方式，造成网络负荷的增加。 |
| 用户 | 根据网络的指示选择合适的语音业务承载方式，避免LTE接入时语音业务不可用导致的用户体验恶化。 |

#### [应用场景](#ZH-CN_TOPIC_0170014709)

以下场景下，请启用本特性：

- 运营商需要精细化控制终端用户LTE接入时的语音业务策略，如：
    - 因IMS网络的漫游还未开通，不允许外网用户使用VoLTE业务。
    - 同时部署CSFB和VoLTE语音业务时，如果终端也同时支持这两种语音业务，推荐本网用户优先使用VoLTE，VoLTE不可用时才使用CSFB。
    - 当MME被多个运营商共享的情况下，可以为不同的运营商提供差异化的语音服务。
- 运营商部署非GSMA标准要求的IMS域APN，例如“IMS-X”。

#### [原理概述](#ZH-CN_TOPIC_0170014709)

**相关概念**

终端最终选择何种语音业务是由终端和网络的语音能力决定的。网络语音能力由MME在Attach Accept/TAU Accept消息下发给终端。终端语音能力在Attach Request或TAU Request的 “Voice domain preference and UE's usage setting” 信元中携带。 “Voice domain preference and UE's usage setting” 信元包含UE's usage setting和Voice domain preference。

UE's usage setting有两种取值：

- Voice centric：此类终端获得网络对语音业务支持能力指示后，选择合适的网络和业务方式，确保语音业务可用。如终端无法在LTE网络获得语音业务，则终端关闭LTE能力，选择GSM/UMTS网络接入，确保语音业务可用。手机终端一般属于此类型。
- Data centric：此类终端获得网络对语音业务支持能力指示后，选择合适的网络和业务方式，确保数据业务可用，即使语音业务不可用的情况下，终端仍然选择驻留在LTE网络。数据卡终端一般属于此类型。

Voice domain preference不同取值下，终端的行为如 [表1](#ZH-CN_TOPIC_0170014709__table14225125252517) 所示。

*表1 Voice domain preference不同取值下终端的行为*

| Voice domain preference取值 | 终端支持的语音方式 | 网络注册流程 | 终端行为说明 |
| --- | --- | --- | --- |
| CS Voice only | 只支持CSFB语音业务。 | Combined Attach/TAU | 若在CS域注册失败，或网络返回SMS only或CSFB not preferred，则终端根据UE's usage setting采取相应的动作。 |
| CS voice preferred， IMS PS Voice as secondary | 同时支持CSFB和VoLTE语音业务，优先通过CSFB进行语音业务。 | Combined Attach/TAU | 若在CS域注册失败，或网络返回SMS only或CSFB not preferred，终端会尝试进行VoLTE语音业务。如果终端无法进行VoLTE语音业务，UE根据UE's usage setting采取相应的动作。 |
| IMS PS voice preferred， CS Voice as secondary | 同时支持CSFB和VoLTE语音业务，优先通过VoLTE进行语音业务。 | Combined Attach/TAU或EPS Attach/TAU | 如果终端无法进行VoLTE语音业务（如因网络下发不支持IMS voice），终端可以使用CSFB进行语音呼叫。此时，终端会进行Combined Attach/TAU。若Combined Attach/TAU时在CS域注册失败，或网络返回SMS only或CSFB not preferred，则终端根据UE's usage setting采取相应的动作。 |
| IMS PS Voice only | 仅支持VoLTE语音业务。 | Combined Attach with SMS only、Combined TAU with SMS only或EPS Attach/TAU | 如果UE无法进行VoLTE语音业务，终端根据UE's usage setting采取相应的动作。 |

> **说明**
> 在实际应用中，终端最终行为和各厂商实现有关。

**差异化的VoLTE语音服务**

[表2](#ZH-CN_TOPIC_0170014709__plc) 列举了在网络中同时部署VoLTE语音和CSFB语音业务的情况下，用户各种可能的语音策略，运营商可以根据具体情况进行组合。

*表2 用户语音策略*

| 语音策略序号 | 签约APN | VoLTE语音策略 |
| --- | --- | --- |
| 1 | IMS | 不允许使用VoLTE。 |
| 2 | IMS | 允许使用VoLTE，且指示用户优先使用VoLTE语音业务。 |
| 3 | IMS | 允许使用VoLTE，且不指示用户优先使用VoLTE语音业务。 |
| 4 | IMS-X | 允许使用VoLTE，且指示用户优先使用VoLTE语音业务。 |
| 5 | IMS-X | 允许使用VoLTE，且不指示用户优先使用VoLTE语音业务。 |

> **说明**
> 针对外网用户， [表2](#ZH-CN_TOPIC_0170014709__plc) 中“签约APN”不是“IMS”的情况下，例如“IMS-X”，建议部署 [WSFD-106203 别名APN](../../会话管理功能/WSFD-106203 别名APN_64082528.md) 特性，将漫游运营商的IMS APN转换成 UNC 设备归属运营商的IMS APN，简化网络部署。

例如运营商A可以：

- 允许本网用户使用VoLTE语音业务，且优先使用VoLTE语音业务，不允许外网用户使用VoLTE语音业务。
  采用 [表2](#ZH-CN_TOPIC_0170014709__plc) 中的2号策略（本网用户使用）和1号策略（外网用户使用）的组合。
- 允许本网用户使用VoLTE语音业务，仅允许来自运营商B和运营商C的外网用户使用VoLTE语音业务，其他外网用户不允许使用VoLTE语音业务。
  采用 [表2](#ZH-CN_TOPIC_0170014709__plc) 中的2号策略（本网用户使用）、3号策略（运营商B的外网用户使用）、3号策略（运营商C的外网用户使用）和1号策略（其他外网用户使用）的组合。

UE在进行VoLTE语音业务过程中移动到无E-UTRAN网络覆盖的区域时，会因为UE不支持SRVCC功能而导致通话中断，无法保证语音业务的连续性。为此， UNC 同时提供根据UE的SRVCC能力决定是否允许其使用VoLTE语音业务的功能。针对UE's usage setting为Voice Centric的终端，建议启用该功能。

当UE不支持SRVCC能力时，可以提供如下选择方式：

1. 对于语音连续性要求比较高的终端，建议检查UE的SRVCC能力，不允许用户使用VoLTE语音业务，直接使用CSFB语音解决方案。
2. 对于不关注语音连续性或者处于固定位置的终端，不需要检查UE的SRVCC能力，可直接使用VoLTE语音解决方案。

> **说明**
> 同时满足以下条件时，表示UE支持SRVCC功能：
>
> - 终端支持SRVCC。
> - 在HSS上为UE签约了SRVCC。

在用户ATTACH/TAU流程中，如果用户没有签约SRVCC， UNC 检查用户的SRVCC能力会因为UE未签约SRVCC而使这类用户无法使用VoLTE语音业务。为此，针对不同语音域优选参数（Voice domain preference）的Voice centric终端， UNC 可以通过命令 [**SET IMSVOPS**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/VoPS管理/设置VoPS配置(SET IMSVOPS)_72345711.md) 中 “基于语音域优选参数的SRVCC能力判断策略” 参数，配置所需的检查项。对于同时符合以下场景的用户，建议配置忽略检查其签约的SRVCC能力和终端的SRVCC能力。

- VoLTE终端开户或附着时未签约SRVCC。
- 终端附着成功后发起IMS域PDN连接建立请求，由IMS触发HSS为用户自动添加签约的SRVCC能力。

**流程实现**

附着或TAU的流程中，MME根据用户的签约数据、漫游属性等信息，决定是否向用户下发IMS VoPS指示。

- 在Attach/TAU流程中，MME通过Attach Accept/TAU Accept消息，在EPS network feature support信元中向UE下发网络侧VoLTE能力(IMS VoPS)，UE根据网络侧下发的IMS VoPS能力决定后续的处理。EPS Network Feature Support信元的bit 1为IMS VoPS指示位：
    - 0表示网络侧不允许UE使用VoLTE语音业务。
    - 1表示网络侧允许UE使用VoLTE语音业务。
- 在Attach/TAU流程中，MME向HSS发送Update Location Request消息或Notify Request消息，消息携带Homogeneous Support of IMS Voice Over PS Sessions信元，用于通知HSS MME是否支持VoLTE语音业务。可通过配置参数“NORPARA”([**SET DMCMPT**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性配置/设置Diameter兼容性(SET DMCMPT)_26306080.md)或[**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md))来决定向HSS发送哪种类型的消息。若仅针对部分用户，系统支持通过Update Location Request消息或Notify Request消息向HSS携带相关信元，则需要配置命令[**ADD DMCMPTBYIMSI**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/信令传输管理/Diameter管理/Diameter协议接口兼容性IMSI号段配置/增加IMSI对应的Diameter兼容性(ADD DMCMPTBYIMSI)_72225977.md)中的参数“用户范围”。
    - 0表示MME不支持VoLTE语音业务。
    - 1表示MME支持VoLTE语音业务。
- 在Combined Attach/TAU流程中，MME通过Attach/TAU Accept消息，在Additional update result信元中向UE下发CS fallback not preferred指示，UE根据网络侧下发的CS fallback not preferred指示决定后续处理。Additional Update Result信元的取值为1时表示CS Fallback Not Preferred，即优先使用VoLTE语音业务。
