#### [适用NF](#ZH-CN_CONCEPT_0272052698)

AMF

#### [定义](#ZH-CN_CONCEPT_0272052698)

5G UE信令控制是指在终端反复发送某些请求消息（例如：初始注册请求、PDU会话建立请求、服务请求消息、4G到5G的移动注册更新请求或者4G到5G的切换请求等）产生大量异常信令的场景下，当某一类消息达到设定的阈值时，AMF可以对UE进行异常信令管控，减轻UE的异常行为对网络侧的影响，提升网络可用性和可靠性。

#### [客户价值](#ZH-CN_CONCEPT_0272052698)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 减少网络中的异常信令。<br>- 提高网络业务流程成功率。<br>- 避免受控信令导致网络拥塞和设备过载。<br>- 提升网络可用性和可靠性。 |
| 用户 | 无。 |

#### [应用场景](#ZH-CN_CONCEPT_0272052698)

5G网络中，在终端反复发送某些请求消息（例如：注册请求、PDU会话建立请求、服务请求、4G到5G的移动注册更新请求或者4G到5G的切换请求等），产生大量异常信令的情况下，利用异常信令控制功能可以减少网络的信令负荷。

#### [对系统的影响](#ZH-CN_CONCEPT_0272052698)

- 当抑制措施设置为特定原因值拒绝时，如果原因值设置不当，可能会导致终端后续不再接入。
- Parking DNN唤醒措施设置为去注册方式时，唤醒会影响用户其他正常DNN的业务，导致正在使用的业务中断。

#### [应用限制](#ZH-CN_CONCEPT_0272052698)

- 只有对端SMF支持时，才能使用Parking DNN激活功能，且Parking DNN上不能开启ULCL功能，Parking DNN也不能为虚拟DNN。
- 系统唤醒PDU会话流程中使用Parking DNN建立的QoS Flow时，将同一DNN粒度并且使用Parking DNN建立的QoS Flow全部唤醒。
- 对高优先级业务引发的初始注册请求、服务请求、寻呼响应引发的服务请求或PDU会话建立请求消息不进行信令控制。
- 对于4/5G互操作场景，当前仅对4G到5G的初始注册请求消息、无N26接口场景4G到5G的重选和初始注册消息、有N26接口场景4G到5G的移动注册更新消息和4G到5G的切换消息进行信令控制。

#### [原理概述](#ZH-CN_CONCEPT_0272052698)

5G网络中，在终端反复发送某些请求消息（例如：注册请求、PDU会话建立请求、服务请求、4G到5G的移动注册更新请求或者4G到5G的切换请求等），产生大量异常信令的情况下，利用异常信令控制功能可以减少网络的信令负荷。

**异常信令出现场景**

- UE反复发送初始注册请求消息原因有很多，主要可以分两类：用户原因和网络原因。
    - 用户原因有：用户未签约、用户非法接入、用户漫游受限等。
    - 网络原因有：UDM/DNS/SMF/PCF等短暂故障或拥塞，由于网元故障导致的频繁接入。
- UE反复发送PDU会话建立请求消息原因主要有：
    - 用户原因导致会话建立失败。
    - 周边网元故障（SMF/PCF等）导致会话建立失败。
    - 上层应用业务故障，导致的终端自动反复会话建立成功/去活。
- 服务请求消息激增的原因是智能终端的快速休眠和永久在线。
- 4G到5G的移动注册更新消息和4G到5G的切换消息激增的原因可能是5G覆盖不连续、终端省电等原因。

**UE信令控制机制简介**

UNC 记录一个周期内UE如下五种请求消息的次数，当测量周期内某类信令数目达到阈值时，为了保护网络资源，AMF支持根据配置的抑制措施对用户的请求信令进行抑制。当抑制的时间达到一定时长后，系统会解除抑制，即允许用户重新接入。

- Registration Request（Initial）
- PDU Session Establishment Request
- Service Request
- Forward Relocation Request
- Registration Request（mobility-registration-updating）

> **说明**
> 本特性仅对以上信令异常的终端进行抑制，不影响其他正常终端的业务。

5G UE信令控制功能的具体过程如下：

1. UNC 判定5G用户Registration Request（Initial）、PDU Session Establishment Request、Service Request、Forward Relocation Request、Registration Request（mobility-registration-updating）消息数是否达到设定的阈值。详细请参见 [异常识别](#ZH-CN_CONCEPT_0272052698__p569923813289) 。
    - 如果消息数达到相应阈值，则系统判断用户信令异常，执行[2](#ZH-CN_CONCEPT_0272052698__step3)。
    - 如果消息数没有达到相应阈值，则系统判断用户行为正常，用户进行正常的业务流程，执行[4](#ZH-CN_CONCEPT_0272052698__step5)。
2. 系统针对不同的信令流程进行抑制。详细请参见[抑制措施](#ZH-CN_CONCEPT_0272052698__p19551118132913)。
3. 在抑制唤醒定时器超时后，针对不同的抑制规则，采取不同的抑制唤醒流程。详细请参见[抑制唤醒](#ZH-CN_CONCEPT_0272052698__p22911842182915)。
4. 进行正常的业务流程。

**异常识别**

UNC 判定测量周期内5G用户Registration Request（Initial）、PDU Session Establishment Request、Service Request、Forward Relocation Request、Registration Request（mobility-registration-updating）消息数是否达到设定的阈值。如果消息数达到相应阈值，则系统判断用户信令异常。如果消息数没有达到相应阈值，则系统判断用户行为正常。门限阈值和测量周期由 **[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)** 命令设置，各消息对应的配置参数如下表所示：

*表1 抑制消息的门限阈值和测量周期参数*

| 消息 | 阈值参数 | 测量周期参数 |
| --- | --- | --- |
| Registration Request（Initial） | 初始注册异常识别门限(次) | 初始注册异常测量周期(秒) |
| PDU Session Establishment Request | PDU会话建立异常识别门限(次) | PDU会话建立异常测量周期(秒) |
| Service Request | 服务请求异常识别门限(次) | 服务请求异常测量周期(秒) |
| Forward Relocation Request | 4G到5G切换异常识别门限(次) | 4G到5G切换异常测量周期(秒) |
| Registration Request（mobility-registration-updating） | 4G到5G重选异常识别门限(次) | 4G到5G重选异常测量周期(秒) |

**抑制措施**

5G UE信令控制的主要目的是对无效重复的Registration Request（Initial）、PDU Session Establishment Request、Service Request、Forward Relocation Request、Registration Request（mobility-registration-updating）异常信令进行控制和消除。

通过 **[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)** 命令针对不同的消息配置不同的抑制措施和抑制时长。5G UE信令控制流程抑制措施匹配如下：

*表2 抑制措施对应表*

| 消息 | 抑制措施1(高优先级) | 抑制措施2（中优先级） | 抑制措施3（低优先级） | 抑制措施4（更低优先级） |
| --- | --- | --- | --- | --- |
| Registration Request（Initial） | 特定原因值拒绝 | N/A | 网络侧去注册 | 丢弃 |
| PDU Session Establishment Request | 特定原因值拒绝 | Parking DNN激活 | 网络侧去注册 | 丢弃 |
| Service Request | 特定原因值拒绝 | N/A | 网络侧去注册 | 丢弃 |
| Forward Relocation Request | 特定原因值拒绝 | N/A | N/A | 丢弃 |
| Registration Request（mobility-registration-updating） | 特定原因值拒绝 | N/A | N/A | 丢弃 |

- 当达到设置的异常识别门限时，优先级高的抑制措施首先生效，当优先级高的抑制措施未能有效抑制，自动按照抑制措施的优先级顺序依次尝试每一项措施。
- 当前抑制策略生效后，如果抑制时间内终端再次发起请求，系统会采取下一优先级的抑制措施，无下一种抑制措施时，继续使用当前抑制措施进行抑制，直到抑制时间超时。
  > **说明**
  > 当选择“Parking DNN激活”作为抑制措施时，建议再选择“网络侧去注册”或者“丢弃”作为下一级抑制措施，否则当Parking DNN作为最后一种抑制措施，可能会导致建立多个Parking DNN会话，造成系统资源浪费。当从一个抑制策略切换到下一个抑制策略时，抑制时间重新计算。

例如：异常阈值设置为30次/小时。如果用户配置三种注册抑制措施，初始注册异常抑制时长为1小时。在1小时内的初始注册次数达到30次，系统会对终端的第30次进行特殊原因值拒绝，并记录时间T1；终端在时间T发起第31次初始注册，如果T-T1<1小时，系统进行去注册，并记录时间T2；终端时间在T发起第32次注册，如果T-T2<1小时，系统进行丢弃，并记录时间T3，后续一小时内的请求都按照丢弃处理，一小时后解除抑制。抑制解除后，重新开始异常检测。

**三种信令抑制措施具体说明如下：**

- **特定原因值拒绝**
    - 当系统检测到终端的初始注册行为异常后，给终端发送初始注册拒绝消息，携带配置的协议原因值。协议原因值由命令**[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)**中参数“注册拒绝原因值”设置。
    - 当系统检测到终端的PDU会话建立行为异常后，给终端发送PDU会话建立拒绝消息，携带配置的协议原因值。协议原因值由命令**[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)**中参数“PDU会话建立原因值”设置。
    - 当系统检测到终端的服务请求行为异常后，给终端发送服务请求拒绝消息，携带配置的协议原因值。协议原因值由命令**[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)**中参数“服务请求拒绝原因值”设置。
    - 当系统检测到终端的4G到5G重选请求行为异常后，给终端发送注册请求拒绝消息，携带配置的协议原因值。协议原因值由命令**[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)**中参数“4G到5G重选拒绝原因值”设置。
    - 当系统检测到终端的4G到5G切换请求行为异常后，给终端发送切换请求拒绝消息，携带配置的协议原因值。协议原因值由命令**[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)**中参数“4G到5G切换拒绝原因值”设置。
- **Parking DNN激活**
  当系统检测到终端的PDU会话建立行为异常后，使用配置的Parking DNN建立一个假会话。
    1. UNC收到5G用户发起的消息后，用特定的DNN（该DNN称为Parking DNN，通过**[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)**命令中的“PARKING DNN”参数设置）替换UE请求的DNN，用特定的切片（该切片称为Parking 切片，通过**[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)**命令中的“PARKING SST”和“PARKING SD”参数设置）替换UE请求的切片，为用户进行QoS Flow激活。
    2. AMF发给SMF的Nsmf_PDUSession_CreateSMContext Request请求消息使用Parking DNN替代用户原有请求的DNN，QoS Flow建立成功后，分配相关的IP地址、DNS地址等，终端停止反复PDU会话建立请求。
    3. 在切换和重选流程中，对于使用Parking DNN激活的PDU会话，旧侧UNC不将其转发到新侧，同时旧侧UNC通知SMF去激活此PDU会话。
    4. 对于使用Parking DNN激活的PDU会话，UDG不允许UE使用该PDU会话进行数据传输。
- **网络层去注册**
    - 注册抑制措施中如果配置了“网络侧去注册”，当系统检测到终端的注册行为异常后，给终端发送去注册请求消息，其中去注册类型为“re-registration not required”，并携带配置的协议原因值。
    - PDU会话建立抑制措施中如果配置了“网络侧去注册”，当系统检测到终端的PDU会话建立行为异常后，给终端发送去注册请求消息，其中去注册类型为“re-registration not required”，并携带配置的协议原因值。
    - 服务请求抑制措施中如果配置了“网络侧去注册”，当系统检测到终端的服务请求行为异常后，给终端发送去注册请求消息，其中去注册类型为“re-registration not required”，并携带配置的协议原因值。
  协议原因值由命令 **[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)** 中参数“去注册原因值”设置，系统初始注册拒绝原因值为#65 – maximum number of PDU sessions reached。
- **丢弃**
  系统直接丢弃终端后续的请求消息。

**抑制唤醒**

终端初始注册、PDU会话建立、服务请求、4G到5G的移动注册更新请求或者4G到5G的切换请求等被抑制后，为防止业务无法恢复，需要对该用户的抑制进行唤醒，采用唤醒方法取消对该用户的限制，帮助该用户恢复业务。

针对不同抑制规则可采用以下唤醒方法：

- 针对抑制规则“特定原因值拒绝”：在相应异常信令的抑制定时器超时后，系统取消对用户的抑制，给用户再次接入系统的机会。
- 针对抑制规则“Parking DNN激活”：Parking DNN可以通过配置的唤醒措施进行唤醒。
  唤醒措施通过 **[SET NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/设置5G信令抑制参数（SET NGSMARTPARA）_25121211.md)** 命令的 “Parking DNN唤醒措施” 参数配置，可配置为 “网络侧去激活” 或 “网络侧去注册” 。
    - 唤醒措施是“网络侧去激活”时，AMF通知SMF删除PDU会话，原因值为“REL_DUE_TO_REACTIVATION”，触发UE重建会话。
    - 唤醒措施是“网络侧去注册”时，AMF发起去注册流程，去注册类型为“re-registration required”，使用户重新注册网络。
- 针对抑制规则“网络侧去注册”：用户需要重新开机或在抑制定时器超时后，系统取消对用户的异常信令抑制。
  > **说明**
  > 如果针对异常初始注册请求消息设置了“丢弃”抑制规则，则用户重新开机的方式不能取消对用户异常信令的抑制。
- 针对抑制规则“丢弃”：在抑制定时器超时后，系统取消对用户的抑制，给用户再次接入系统的机会。

#### [计费与话单](#ZH-CN_CONCEPT_0272052698)

如果系统配置了Parking DNN，则Parking DNN激活的PDU会话不产生计费话单。
