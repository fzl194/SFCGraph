#### [适用NF](#ZH-CN_CONCEPT_0166346073)

MME

#### [定义](#ZH-CN_CONCEPT_0166346073)

LTE用户以Combined Attach或者Combined TAU方式接入到MME系统后，当UE以紧急呼叫方式发起CSFB主叫业务时，MME给此类业务以高优先级方式进行差异化处理，并指示eNodeB优先处理此CS回落业务，使CS域在系统拥塞的场景下优先处理CSFB紧急呼叫业务。

#### [客户价值](#ZH-CN_CONCEPT_0166346073)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 紧急呼叫业务是某些国家与地区的法律法规规定运营商必须满足的需求。 |
| 用户 | 在紧急情况下，用户可以得到及时、有效的求助。 |

#### [应用场景](#ZH-CN_CONCEPT_0166346073)

紧急呼叫业务为用户的人身和财产安全提供保障，网络部署中都需要激活该特性。

#### [原理概述](#ZH-CN_CONCEPT_0166346073)

CSFB紧急呼叫特性与 [WSFD-102301 基于CSFB的语音业务](../WSFD-102301 基于CSFB的语音业务_70014756.md) 特性的实现原理相似，业务流程基本相同，这里主要从流程角度介绍两个特性的差异。

- 连接态主叫-支持PS切换
    1. UE发送Extended Service Request消息给MME，请求进行CSFB语音业务。其中Extended Service Request消息中的Service type信元指示为紧急呼叫（mobile originating CS fallback emergency call）。
    2. MME发送UE Context Modification Request消息给eNodeB。该消息指示eNodeB，UE因CSFB业务需要回落到UTRAN/GERAN。由于是紧急呼叫触发的CS回落，因此其中回落消息中CS Fallback indicator信元取值为CS Fallback High Priority。
- 连接态主叫-不支持PS切换
  对于紧急呼叫业务，在支持PS切换和不支持PS切换场景下，信令处理流程相同。
- 空闲态主叫
  UE处于空闲态发起CSFB主叫语音业务时，建立空口连接，在RRC Connection Setup Complete消息里带上NAS扩展业务请求，进入RRC连接态，后续流程与连接态的CSFB主叫流程基本相同，主要差别是S1-AP UE Context Modification Request/Response消息变为S1-AP Initial UE Context Request/Response消息。
