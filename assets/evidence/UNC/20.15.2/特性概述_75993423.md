#### [适用NF](#ZH-CN_TOPIC_0275993423)

PGW-C

#### [定义](#ZH-CN_TOPIC_0275993423)

eMTC （enhanced Machine Type Communication）业务在正常场景下，终端发送的都是低频率数据包，但在终端异常场景（如超时重发等）下，可能在短时间大频率发送数据包，此时可能会导致网络拥塞并影响业务运行，所以3GPP协议引入终端发送数据的速率控制机制。基于APN的eMTC终端接入速率控制在PGW-C上配置速率门限，用于控制该UE的某PDN连接单位时间内上行和下行数据包的数目。

#### [客户价值](#ZH-CN_TOPIC_0275993423)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 避免由于终端异常而导致网络拥塞，有效提升整网的可靠性。 |
| 用户 | 提高普通终端用户业务的成功率和业务体验。 |

#### [应用场景](#ZH-CN_TOPIC_0275993423)

本特性应用于eMTC控制面优化（Control Plane Optimization）和用户面优化（User Plane Optimization）数据传输中，用于限制终端的数据传输速率，在终端异常情况下可以防止网络拥塞而影响业务，提升整网的可靠性。

#### [应用限制](#ZH-CN_TOPIC_0275993423)

用户在附着后短时间内发送一定的数据包后退网，当该用户再次入网时之前发送的数据包不会被 UNC 统计到单位时间内发送的数据包内， UNC 重新开始进行该用户的速率控制计时和计数。

#### [原理概述](#ZH-CN_TOPIC_0275993423)

**相关概念**

- 控制面优化
  控制面优化即CP优化（Control Plane Optimization），在终端和MME之间通过NAS（Non-access Stratum）消息传输业务层数据，在MME和SGW-C之间通过S11-U接口传递业务层数据，无需反复建立和删除空口无线数据承载DRB（Data Radio Bearer）和S1-U连接，有效提升了数据传输效率，同时也可以降低对网络的信令负荷。具体请参考 [WSFD-215101 基于信令面的数据传输（SGW-C）](../../NB-IoT业务功能/WSFD-215101 基于信令面的数据传输/WSFD-215101 基于信令面的数据传输（SGW-C）_77260994.md) 。
- 用户面优化
  用户面优化即UP优化（User Plane Optimization） ，在UE、eNodeB和MME上保存恢复DRB和S1-U连接的S1AP（S1 Application Protocol）关联、UE上下文和承载上下文等数据，后续能快速恢复承载，降低对网络的信令负荷。

基于APN的eMTC终端接入速率控制是在PGW-C上配置速率门限，对超过速率门限的数据包进行丢弃处理。PGW-C使用PCO /ePCO 信元将基于APN的eMTC终端接入速率控制参数发送给UE，用于控制该UE的某PDN连接单位时间内上行和下行数据包的数目。MME/SGW-C支持转发UE和PGW-C之间速率控制参数的消息。
