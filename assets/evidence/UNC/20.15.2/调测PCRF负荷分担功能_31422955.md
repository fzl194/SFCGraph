- [必备事项](#ZH-CN_OPI_0231422955__1.3.2)

## [操作场景](#ZH-CN_OPI_0231422955)

当运营商规划多个PCRF来负荷分担承载PCC业务时，需要通过调试手段检查PCRF组内的PCRF之间是否按规划比例进行负荷分担。

> **说明**
> 适用于 GGSN、 PGW-C。

## [必备事项](#ZH-CN_OPI_0231422955)

前提条件

- 请仔细阅读 [PCC基本功能（2G/3G/4G，基于Gx接口）](../../PCC基本功能（2G_3G_4G，基于Gx接口）_29056156.md) 。
- 完成[激活PCC基本功能](../激活PCC基本功能_29056159.md)。
- 完成[调测PCC业务](调测PCC业务_31422956.md)。

工具

## [操作流程](#ZH-CN_OPI_0231422955)

1. 激活大量用户。
2. 检查“PCC发送CCR-I消息数(PCRF)”指标统计值。
3. 检查GGSN/PGW-C与PCRF的链路状态。
4. 检查PCRF上的业务承载能力。

## [操作步骤](#ZH-CN_OPI_0231422955)

1. 激活大量PCC用户。
2. 所有PCC用户激活成功后，使用 U2020 /MAE客户端查看UNC上统计的“PCC发送CCR-I消息数(PCRF)”指标值。
    - 如果GGSN/PGW-C与各PCRF交互的CCR-I消息数的比例和GGSN/PGW-C上配置的PCRF组内各PCRF的负荷分担比例一致，则调测结束。
    - 如果不一致，请执行[步骤 3](#ZH-CN_OPI_0231422955__stp5)。
3. 进入 “MML命令行-UNC” 窗口。 执行 [**DSP PCRFSTATUS**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF状态/显示PCRF状态（DSP PCRFSTATUS）_09897111.md) 命令，查看各PCRF的状态是否正常。
  ```
  DSP PCRFSTATUS:PCRFNAME="pcrf_1";
  ```
  ```
  ......

  -------------------------
   PCRF主机名  =  pcrf_1
      POD名称  =  uncpod-0
     本端地址  =  10.8.20.1:13400
   本端子地址  =  -
     对端地址  =  10.11.21.59:3868
      Gx 状态  =  Normal
   本端主机名  =  pgw_1 
  (结果个数 = 1)
  ---    END
  ```
  ```
  DSP PCRFSTATUS:PCRFNAME="pcrf_2";
  ```
  ```
  ......

  -------------------------
  PCRF主机名     POD名称       本端地址              本端子地址     对端地址              Gx 状态   本端主机名
  pcrf_2         uncpod-0      10.8.20.1:13450        -             10.11.21.60:3868       Normal    pgw_1
  pcrf_2         uncpod-0      10.8.20.1:13450        -             10.11.21.61:3868       Normal    pgw_1
  (结果个数 = 2)
  ---    END
  ```
    - 如果检测PCRF状态正常，则执行[步骤 4](#ZH-CN_OPI_0231422955__stp6)。
    - 如果检测PCRF状态异常，请参考[配置与PCRF对接数据](../激活PCC基本功能/配置与PCRF对接数据_30805096.md)进行调测。
4. 进入 “MML命令行-UNC” 窗口。 执行 [**LST PCRFGROUP**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF组/查询PCRF组（LST PCRFGROUP）_09897093.md) 和 [**LST PCRFBINDGRP**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF绑定/查询PCRF与PCRF Group的关联关系（LST PCRFBINDGRP）_09897099.md) 命令检查指定PCRF组内各PCRF的负荷分担配置比例是否正确。
  ```
  LST PCRFGROUP:PCRFGRPNAME="pcrf_group_1";
  ```
  ```
  ......

  -------------------------
         PCRF组名称  =  pcrf_group_1
       负荷分担模式  =  主备
       宕机备份开关  =  使能
     主用PCRF主机名  =  NULL
  (结果个数 = 1)
  ---    END
  ```
  ```
  LST PCRFBINDGRP:PCRFGRPNAME="pcrf_group_1";
  ```
  ```
  ......

  -------------------------
  PCRF组名称       PCRF主机名称       PCRF百分比 
  pcrf_group_1       pcrf_1            40          
  pcrf_group_1       pcrf_2            60          
  (结果个数 = 2 )
  ---    END
  ```
    - 如果各PCRF上的配置的“权重”值与规划值一致，请执行[步骤 5](#ZH-CN_OPI_0231422955__stp7)。
    - 如果不一致，请执行[**MOD PCRFBINDGRP**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/PCRF Diameter连接/PCRF绑定/修改PCRF与PCRF Group的关联关系（MOD PCRFBINDGRP）_09897097.md)命令重新配置。
5. 请联系PCRF设备工程师，检查PCRF设备的业务承载性能。
    - 如果PCRF设备性能不足以承载当前业务量，请联系运营商重新规划PCRF。
    - 如果PCRF性能足以承载当前业务量，请联系华为技术支持工程师。
