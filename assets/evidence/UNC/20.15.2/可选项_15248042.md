#### [82200BNQ LKV2PCIAL01 基于初始接入位置的策略控制](#ZH-CN_TOPIC_0000001615248042)

| **适用NF** | GGSN-C、SGW-C/PGW-C、SMF |
| --- | --- |
| **功能描述** | 基于初始接入位置的策略控制是在UNC支持基于用户组的带宽管理基础上，增加基于初始接入位置维度的用户组，解决小区拥塞问题。例如对拥塞小区进行流控和业务限制，限制整个小区的P2P总流量等。主要用于解决日趋拥塞的网络带宽，针对拥塞小区的带宽控制。 |
| **实现描述** | 该控制项开启时，允许激活PDP上下文后使用基于初始接入位置的策略控制的功能。该控制项关闭时，不允许激活PDP上下文后使用基于初始接入位置的策略控制的功能。 |
| **取值范围** | 0～1 Bearer |
| **默认值** | 1 |
| **对应特性** | WSFD-211001 基于初始接入位置的策略控制 |
| **相关控制项** | 无 |
| **应用场景** | 网络中无线带宽资源比较紧张，在热点小区经常发生拥塞。该特性主要是在忙时针对拥塞的小区进行带宽控制，降低P2P/VoIP等业务对带宽资源的过度占用，提升大部分用户的使用体验。例如在9:00-11:00忙时对CBD区域基于小区控制P2P的带宽。 |

#### [81202579 LKV6DIFAIS01 磁盘故障隔离](#ZH-CN_TOPIC_0000001615248042)

| 适用NF | SGSN/MME/GGSN-C/SGW-C/PGW-C/AMF/SMF/NRF/NSSF/SMSF |
| --- | --- |
| **功能描述** | 磁盘故障隔离（存储Bypass）是指将业务节点和存储设备进行解耦，当存储设备故障不可用后，业务节点运行不受影响，业务不受影响，提供受限运维功能，提升NF可靠性<br>。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 UNC |
| **默认值** | 1 |
| **对应特性** | WSFD-209002 磁盘故障隔离 |
| **相关控制项** | 无 |
| **应用场景** | 存储设备通过网络和计算节点连接，其中导致存储设备整体故障的因素很多，包括供电中断、网络中断、网络拥塞丢包等，都会导致存储异常。对于5G Core网元，大部分业务都对存储依赖很低，存储设备故障如果保证业务不受损，可以大大提升产品竞争力。 |

#### [81203853 LKV2SHFCUNC01 服务化接口HTR流控 - UNC](#ZH-CN_TOPIC_0000001615248042)

| 适用NF | SMF/AMF/NCG |
| --- | --- |
| **功能描述** | 本License控制UNC的周边网元存在过载风险时，UNC能否基于周边网元对于用户各业务流程首消息返回的失败应答数的周期变化，自动调整向后端网元发送的请求消息数，避免后端网元出现严重过载，达到保护周边网元的目的。 |
| **实现描述** | 只有获得了License许可后才能获得该特性的服务。 |
| **取值范围** | 0～1 UNC |
| **默认值** | 1 |
| **对应特性** | WSFD-206009 网元间HTR流控 |
| **相关控制项** | 无 |
| **应用场景** | 由于5G网络部署越来越广泛，5G用户量剧增，UNC各周边接口和网元拥塞的可能性也越来越大，为了保证用户接通率、改善用户体验，引入网元间HTR流控功能。应用场景包括：<br>- 升级过程中系统重启该场景可以预期有大量用户重新注册，用户到周边网元的请求同时发起。系统重启前开启网元间HTR流控，预防周边链路拥塞或周边网元业务过载。<br>- 故障倒换该场景会突发大量到周边网元的接入请求消息，致使接口链路拥塞、相应网元过载，业务成功率明显下降。在倒换前提前开启本功能可以降低信令拥塞对周边网元的影响。 |
