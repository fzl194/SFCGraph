## [命令功能](#ZH-CN_MMLREF_0209653141)

**适用NF：AMF**

此命令用于查询当前配置的S1寻呼策略。

## [注意事项](#ZH-CN_MMLREF_0209653141)

无

#### [操作用户权限](#ZH-CN_MMLREF_0209653141)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0209653141)

无

## [使用实例](#ZH-CN_MMLREF_0209653141)

## [输出结果说明](#ZH-CN_MMLREF_0209653141)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 最近GNB开关 | 该参数用于指定是否启用最近gNodeB范围的寻呼。 |
| 最近GNB寻呼超时时长(s) | 该参数用于指定AMF等待最近gNodeB范围的寻呼响应的超时时长。 |
| 最近GNB寻呼超时重发次数(times) | 该参数用于指定AMF发起最近gNodeB范围的寻呼流程后没有收到UE的响应消息，重发Paging Request消息的次数。 |
| 邻接GNB开关 | 该参数用于指定是否启用邻接gNodeB范围的寻呼。 |
| 邻接GNB寻呼超时时长(s) | 该参数用于指定AMF等待邻接gNodeB范围的寻呼响应的超时时长。 |
| 邻接GNB寻呼超时重发次数(times) | 该参数用于指定AMF发起邻接gNodeB范围的寻呼流程后没有收到UE的响应消息，重发Paging Request消息的次数。 |
| 最近访问TA开关 | 该参数用于指定是否启用最近驻留TAI范围的寻呼。 |
| 最近驻留TAI寻呼超时时长(s) | 该参数用于指定AMF等待最近驻留TAI范围的寻呼响应的超时时长，仅在参数“最近访问TA开关”配置为“ON（ON）”时生效。 |
| 最近驻留TAI寻呼超时重发次数(times) | 该参数用于指定AMF发起最近驻留TAI范围的寻呼流程后没有收到UE的响应消息，重发NG Paging Request消息的次数，仅在参数“最近访问TA开关”配置为“ON（ON）”时生效。 |
| 邻接TA开关 | 该参数用于指定是否启用邻接TA范围的寻呼。 |
| 邻接TA寻呼超时时长(s) | 该参数用于指定AMF等待邻接TA范围的寻呼响应的超时时长。 |
| 邻接TA寻呼超时重发次数(times) | 该参数用于指定AMF发起邻接TA范围的寻呼流程后没有收到UE的响应消息，重发Paging Request消息的次数。 |
| 低移动性检查开关 | 该参数用于指定是否打开“低移动性检查”功能。 |
| GNB粘性时长(min) | 该参数用于指定低移动性检查时UE在一个gNodeB粘连的时间阈值，即UE在一个gNodeB下驻留的时间超过该参数才允许AMF对其使用精准寻呼。 |
| TA粘性时长(min) | 该参数用于指定低移动性检查时UE在一个TA粘连的时间阈值，即UE在一个TA下的驻留时间超过该参数才允许AMF对其使用精准寻呼。 |
| HO学习开关 | 该参数用于指定是否打开N2 Handover流程中学习邻接gNodeB关系的能力。 |
| SON学习开关 | 该参数用于指定是否打开SON流程中学习邻接gNodeB关系的能力。 |
| Path Switch学习开关 | 该参数用于指定是否打开Xn-based Handover流程中学习邻接gNodeB关系的能力。 |
| 邻接TA学习开关 | 该参数用于指定是否打开AMF学习邻接TA关系的能力。 |
| 邻接GNB老化时间(h) | 该参数用于指定邻接gNodeB的老化时间，当超过老化时间，相邻gNodeB之间的关系记录将被删除。 |
| 邻接TA老化时间(h) | 该参数用于指定邻接TA的老化时间，当超过老化时间，邻接TA之间的关系记录将被删除。邻接TA老化时间为0表示不启用老化。 |
| 全网寻呼开关 | 该参数用于指定打开或者关闭“全网寻呼”功能。<br>当前该功能尚未实现。 |
| 高优先级寻呼消息处理策略 | 该参数用于控制冲突场景下是否终止当前低优先级的寻呼消息，优先处理高优先级的寻呼消息。<br>该参数为"YES"时表示终止当前低优先级的寻呼消息，优先处理高优先级的寻呼消息。<br>该参数为"NO"表示继续处理当前正在处理的寻呼消息。 |
