# 设备标识检查机制

AMF通过MML命令 [**ADD NGPEIPLCY**](../../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/业务安全管理/PEI策略管理/增加5G PEI策略（ADD NGPEIPLCY）_09651346.md) 配置PEI的检查策略，PEI检查机制如 [图1](#ZH-CN_TOPIC_0000001177252939__fig11977115310414) 所示。

**图1** PEI检查机制

<br>

![](设备标识检查机制_77252939.assets/zh-cn_image_0000001141650594_2.png)

1. AMF根据“流程类型（PROTTYPE）”决定在哪些注册流程中进行PEI检查，建议在初始注册、移动性Inter注册、Inter切换后的注册、空闲态EPS到5GS注册、连接态EPS到5GS切换后的注册中开启PEI检查功能。
2. AMF根据“PEI获取策略（GETPEIPLCY）”决定向UE获取何种格式的PEI。建议获取IMEISV，此场景下当存在Security Mode Command流程时，AMF可以直接通过该流程向UE获取IMEISV，不需要再向UE发送Identity Request消息，从而减少信令消耗。此外，当PEI获取失败时，AMF根据“PEI获取失败是否允许接入（PEIACCESS）”决定是否允许用户接入。
3. 5G-EIR返回的检查结果如[表1 N5geir_EquipmentIdentityCheck_GetEquipmentStatus Response消息中的关键信元解释](#ZH-CN_TOPIC_0000001177252939__table16387127125013)所示。
  *表1 N5geir_EquipmentIdentityCheck_GetEquipmentStatus Response消息中的关键信元解释*

  | 状态码 | 关键信元 | 信元解释 |
  | --- | --- | --- |
  | 200 OK | EirResponseData | 设备标识检查成功，响应消息中包含设备状态：白名单（WHITELISTED）、灰名单（GREYLISTED）和黑名单（BLACKLISTED）。 |
  | 404 Not Found | ProblemDetails | 设备标识检查失败，响应消息中包含错误原因。 |
  | 307 Temporary Redirect | Location | 临时重定向，Location头字段中包含一个不同的URL，该URL指向同一个5G-EIR中可替代的服务实例或5G-EIR服务器集中可替代的服务实例。 |
  | 307 Temporary Redirect | 3gpp-Sbi-Target-Nf-Id | 临时重定向请求的目标NF实例ID。 |
  | 308 Permanent Redirect | Location | 永久重定向，Location头字段中包含一个不同的URL，该URL指向同一个5G-EIR中可替代的服务实例或5G-EIR服务器集中可替代的服务实例。 |
  | 308 Permanent Redirect | 3gpp-Sbi-Target-Nf-Id | 永久重定向请求的目标NF实例ID。 |
    - 当5G-EIR返回检查成功，且PEI处于白名单时，AMF允许用户接入。
    - 其它场景，AMF根据“PEI检查超时是否允许接入（CTFLAG）”、“是否允许黑名单用户接入（BALLOW）”、“是否允许灰名单用户接入（GRALLOW）”、“是否允许紧急注册用户接入（EMERGENCYSW）”、“是否允许EIR发现失败时接入（DISCFAIL）”、“是否允许EIR返回失败时接入（CHKFAIL）”等等参数配置决定后续动作。
