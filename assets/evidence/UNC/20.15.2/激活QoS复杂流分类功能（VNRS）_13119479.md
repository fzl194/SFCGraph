## [操作场景](#ZH-CN_OPI_0213119479)

本操作指导介绍在运行网络中激活QoS复杂流分类功能特性的操作过程。

QoS复杂流分类涉及到流策略、流分类、流行为三个概念。一个流行为和流分类可以组成一个CB对，在具体的流策略中绑定。流策略可以绑定到指定的以太主接口或者子接口的出入方向，经过的报文会匹配流分类中的规则，匹配上后会按照流动作中动作进行处理。

流分类中指定了具体的报文匹配规则。ACL规则的配置请见ACL配置手册。

流动作中指定了报文匹配成功后对应的处理动作。目前动作类型有：允许、拒绝、指定下一跳、级联流策略。

## [对系统的影响](#ZH-CN_OPI_0213119479)

该操作对系统正常运行无影响。

## [操作步骤](#ZH-CN_OPI_0213119479)

1. 配置ACL规则组。
  ACL4规则组参见 [激活支持ACL](../../../IPFD-012002 ACL/特性配置/激活支持ACL_06381760.md) 的配置，ACL6规则组参见 [激活支持ACL6](../../../IPFD-012002 ACL/特性配置/激活支持ACL6_06381763.md) 的配置。
2. 增加流行为。
  在 “MML命令行-UNC” 窗口上执行：
  [**ADD MQCBEHAVIOR**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/VNRS功能管理/IP服务/安全管理/MQC/分类行为/增加流行为（ADD MQCBEHAVIOR）_50281310.md) :BEHAVIORNAME="行为名称";
3. 增加对应的流动作。
  在 “MML命令行-UNC” 窗口上执行：
    - （可选）配置重定向下一跳动作：
      [**ADD QOSACTRDRNHP**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/VNRS功能管理/IP服务/安全管理/QoS管理/重定向下一跳信息/增加QoS重定向下一跳信息（ADD QOSACTRDRNHP）_00865777.md) :BEHAVIORNAME=" 流 行为名称", IPVERSION=IPv4, NEXTHOP="下一跳 地址 ";
    - （可选）配置重标记：
      [**ADD SQOSREMARK**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/VNRS功能管理/IP服务/安全管理/QoS管理/重标记/增加重标记配置（ADD SQOSREMARK）_00441017.md) :BEHAVIORNAME="流行为名称", V4DSCPVALUE="IPv4的DSCP值", V6DSCPVALUE="IPv6的DSCP值", DFVALUE="DF值";
    - （可选）配置deny动作：
      [**SET QOSACTFILTER**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/VNRS功能管理/IP服务/安全管理/QoS管理/流行为规则/设置流行为规则（SET QOSACTFILTER）_49802526.md) :BEHAVIORNAME="行为名称", FILTER=deny;
    - （可选）配置级联流策略动作：
      [**ADD QOSACTRDRPOLICY**](../../../../../../../OM参考/命令/UNC MML命令/平台服务管理/VNRS功能管理/IP服务/安全管理/QoS管理/级联流策略动作/增加流行为下级联流策略（ADD QOSACTRDRPOLICY）_00600429.md) :BEHAVIORNAME="行为名称", POLICYNAME="策略名称";

## [任务示例](#ZH-CN_OPI_0213119479)

任务描述

配置在接口中使用QoS复杂流分类功能。组网情况如 [图1](#ZH-CN_OPI_0213119479__zh-cn_opi_0134584227_fig_dc_fenix_nps_mml_cfg_qos_000401) 所示，VNF1的接口Ethernet 64/0/4 的IP地址为192.168.2.1， VNF2的接口Ethernet 64/0/5 的IP地址为192.168.2.3。添加acl1规则组，并添加“adv_3”，以VNF3的IP地址192.168.1.3为源IP地址，192.168.2.1为目的IP地址，动作为deny，还有“adv_4”号规则，以VNF4的IP地址192.168.1.4为源IP地址，192.168.2.1为目的IP地址，动作为permit。

在DeviceA上创建流分类class1，流分类引用acl1规则组，创建流行为behavior1，流动作为重定向下一跳到192.168.2.3。创建流策略policy1，绑定到接口Ethernet 64/0/3 出方向。

**预期结果** ：分别从VNF3 和VNF4上向VNF1和VNF2发送报文，结果应该是VNF1没收到VNF3和VNF4的报文，而VNF2能收到来自VNF4的报文。

脚本

//添加ACL规则组。添加组标识为acl1的规则组。

```
ADD ACLGROUP: ACLNAME="acl1";
```

//添加ACL规则。添加规则名称为"adv_3"和"adv_4"的规则。

```
ADD ACLRULEADV4: ACLNAME="acl1", ACLRULENAME="adv_3", ACLACTION=Deny, ACLPROTOCOL=0, ACLSOURCEIP="192.168.1.3", ACLSRCWILD="0.0.0.0", ACLDESTIP="192.168.2.1", ACLDESTWILD="0.0.0.0";
ADD ACLRULEADV4: ACLNAME="acl1", ACLRULENAME="adv_4", ACLACTION=Permit, ACLPROTOCOL=0, ACLSOURCEIP="192.168.1.4", ACLSRCWILD="0.0.0.0", ACLDESTIP="192.168.2.1", ACLDESTWILD="0.0.0.0";
```

//增加流分类。

```
ADD MQCCLASSIFIER: CLASSIFIERNAME="class1";
```

//配置流分类引用ACL。

```
ADD QOSRULEACL: CLASSIFIERNAME="class1", IPVERSION=IPv4, ACLTYPE=acl-name, ACLNAME="acl1";
```

//增加流行为。

```
ADD MQCBEHAVIOR: BEHAVIORNAME="behavior1";
```

//配置对应的动作。

```
ADD QOSACTRDRNHP: BEHAVIORNAME="behavior1", IPVERSION=IPv4, NEXTHOP="192.168.2.3";
```

//增加流策略。

```
ADD MQCPOLICY: POLICYNAME="policy1";
```

//流策略绑定流分类和流行为。

```
ADD MQCPOLICYNODE: POLICYNAME="policy1", BEHAVIORNAME="behavior1", CLASSIFIERNAME="class1";
```

//将流策略绑定到指定的接口。

```
ADD QOSAPPLICATION: POLICYNAME="policy1", IFNAME="Ethernet
64/0/3
", DIRECTION=outbound;
```

//使能流策略统计。

```
MOD MQCPOLICY: POLICYNAME="policy1", STCENABLE = enable;
```
