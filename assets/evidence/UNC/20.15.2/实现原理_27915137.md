# 实现原理

- [动态PCC](#ZH-CN_TOPIC_0227915137__1.3.2.1)
- [本地PCC](#ZH-CN_TOPIC_0227915137__1.3.3.1)

系统支持动态PCC和本地PCC管控热点小区用户带宽。

#### [动态PCC](#ZH-CN_TOPIC_0227915137)

GGSN-C/PGW-C/SMF通过Gx接口与PCRF交互时，业务流程如 [图1](#ZH-CN_TOPIC_0227915137__fig984713915716) 所示。

**图1** 基于初始接入位置的策略控制（动态PCC）

<br>

![](实现原理_27915137.assets/zh-cn_image_0231418419_2.png)

1. 获取并安装基于初始接入位置的带宽控制策略。
    a. 用户接入网络，MME/SGSN向PGW-C/GGSN-C发送Create PDP Context Request/Create Session Request。
    b. PGW-C/GGSN-C向PCRF发送CCR-Initial请求消息，消息中携带3GPP-User-Location-Info AVP。
    c. PCRF根据报文中携带的位置信息匹配用户所在服务区，决策使用的带宽控制策略。并通过CCA-Initial消息下发给PGW-C/GGSN-C。
    d. PGW-C/GGSN-C向PGW-U/GGSN-U发送PFCP Session Establishment Request消息，携带带宽控制策略。
      > **说明**
      > 规则中携带时间范围时，PGW-C/GGSN-C根据灵活时间的机制决定何时向PGW-U/GGSN-U下发带宽控制规则。
    e. PGW-U/GGSN-U安装规则并向PGW-C/GGSN-C返回PFCP Session Establishment Response消息。
    f. PGW-C/GGSN-C向MME/SGSN返回Create PDP Context Response/Create Session Response。
2. 对上行报文进行带宽控制。
    a. 用户访问业务，向PGW-U/GGSN-U发送Traffic Flow。
    b. PGW-U/GGSN-U收到用户的上行报文，根据安装的带宽规则进行带宽控制。用户的上行业务带宽超出允许值时，缓存/丢弃超出的报文。未超出允许值时，正常处理后转发报文。
3. 对下行报文进行带宽控制。
    a. 服务器向PGW-U/GGSN-U发送用户的业务报文。
    b. PGW-U/GGSN-U收到用户的下行报文，根据安装的带宽规则进行带宽控制。用户的下行业务带宽超出允许值时，缓存/丢弃超出的报文。未超出允许值时，正常处理后向UE转发报文。

GGSN-C/PGW-C/SMF通过N7接口与PCF交互时，与Gx接口类似，此处不再详细描述。

#### [本地PCC](#ZH-CN_TOPIC_0227915137)

**图2** 基于初始接入位置的策略控制（本地PCC）

<br>

![](实现原理_27915137.assets/zh-cn_image_0228104181_2.png)

1. 获取并安装基于初始接入位置的带宽控制策略。
    a. 用户接入网络，MME/SGSN向PGW-C/GGSN-C发送Create PDP Context Request/Create Session Request。
    b. PGW-C/GGSN-C根据3GPP-User-Location-Info AVP，决策使用的带宽策略。然后向PGW-U/GGSN-U发送PFCP Session Establishment Request消息，携带带宽控制策略。
    c. PGW-U/GGSN-U安装规则并向PGW-C/GGSN-C返回PFCP Session Establishment Response消息。
    d. PGW-C/GGSN-C向MME/SGSN返回Create PDP Context Response/Create Session Response。
2. 对上行报文进行带宽控制。
    a. 用户访问业务，向PGW-U/GGSN-U发送Traffic Flow。
    b. PGW-U/GGSN-U收到用户的上行报文，根据安装的带宽规则进行带宽控制。用户的上行业务带宽超出允许值时，缓存/丢弃超出的报文。未超出允许值时，正常处理后转发报文。
3. 对下行报文进行带宽控制。
    a. 服务器向PGW-U/GGSN-U发送用户的业务报文。
    b. PGW-U/GGSN-U收到用户的下行报文，根据安装的带宽规则进行带宽控制。用户的下行业务带宽超出允许值时，缓存/丢弃超出的报文。未超出允许值时，正常处理后向UE转发报文。
