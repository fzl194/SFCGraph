## [操作场景](#ZH-CN_OPI_0000001152313549)

本操作指导介绍在运行网络中激活基于SPID的UE驻留和切换策略管理特性的操作过程。

3GPP协议中定义了SPID（Subscriber Profile ID for RAT/Frequency Selection Priority），它定义了一组频段及其优先级。这个信元可以由用户签约信息指定，也可以由 UNC 指定，传递到无线侧之后，可以用于灵活控制终端的行为策略，例如指定终端驻留和切换频段的优先级等。

无线侧的SPID和核心网侧的RFSP ID为同一概念。

> **说明**
> 适用于AMF。

## [操作步骤](#ZH-CN_OPI_0000001152313549)

1. 进入 “MML命令行-UNC” 窗口。
2. 根据运营商规划，选择RFSP ID部署策略。
    - **场景一：运营商规划完全信赖用户在UDM的签约信息，在UDM直接规划好RFSP ID， UNC 本地不做任何RFSP** **控制** **。**
    - **场景二：假设不信赖UDM中的用户签约信息（例如对于漫游用户），或在UDM签约未规划或UDM不支持该功能。为了实现自定义的驻留优先级控制， UNC 可以在本地针对用户进行RFSP策略的配置：**
      [**ADD NGMMSUBDATA**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/签约数据管理/增加用户移动性管理参数（ADD NGMMSUBDATA）_09651362.md)

## [任务示例](#ZH-CN_OPI_0000001152313549)

任务描述

假设运营商需要某号段的用户优先使用签约的RFSP ID对应的频点优先级进行驻留控制（例如：5G用户回落4G后，优先使用签约RFSP ID对应的频点优先级进行驻留控制），如果没有签约RFSP ID，则按照配置的RFSP ID对应的频点优先级进行驻留控制。

脚本

//需要对30809号段的用户使用本地配置优先，即如果用户没有签约RFSP ID，按照RFSP ID为1对应的频点优先级进行驻留控制。

```
ADD NGMMSUBDATA: SUBRANGE=IMSI_PREFIX, IMSIPRE="30809", RFSPINDEX=1, RFSPPRI=CFG_FIRST;
```
