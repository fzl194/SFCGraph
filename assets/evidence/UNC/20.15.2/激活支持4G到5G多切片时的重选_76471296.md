## [操作场景](#ZH-CN_OPI_0276471296)

本操作指导介绍在运行网络中激活支持4G到5G多切片时的重选的操作过程。

空闲态UE移动出4G基站覆盖区域进入5G基站的覆盖区域，执行小区重选，发起移动注册更新流程，进行4G到5G的重选。

UE在4G网络时，不需要切片信息，4G网络一张网络满足所有应用；UE从4G移动到5G之后，在5G组网场景下，则需要通过对切片信息的判断，将UE接入不同的切片网络，分别为不同业务场景提供服务。比如当存在2B、2C两个切片网络，需要区分处理，让2B的用户接入到2B的切片网络，2C的用户接入到2C的切片网络。

本特性通过互操作过程中的切片信息，判断用户从4G网络重选到5G网络后，AMF是否需要重分配 、I-SMF是否需要插入、初始插入的I-SMF是否需要重选、I-UPF是否需要重选 ，让支持对应切片的网元为UE提供服务，最大化的保证用户在4G到5G多切片重选时的业务连续性。

> **说明**
> 适用于AMF、SMF 、NSSF 。

## [操作步骤](#ZH-CN_OPI_0276471296)

1. 进入 “MML命令行-UNC” 窗口。 -SMF
2. 5G用户在EPS网络建立PDN连接时，控制SMF是否向UDM获取签约的切片和DNN。
  [**SET SMFFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/会话协议参数管理/会话管理拓展功能/5GC会话管理拓展功能/设置SMF扩展功能（SET SMFFUNC）_09653731.md)
3. 指定PDN连接建立流程中根据DNN为用户分配的互操作默认切片。
  **[**ADD NSDNN**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/接口管理/服务化接口管理/注册与服务发现/网络切片内DNN管理/增加网络切片支持的DNN（ADD NSDNN）_09652268.md)**
4. 进入 “MML命令行-UNC” 窗口。 -AMF
5. AMF支持配置互操作专用切片发现I-SMF。
  [**ADD SMFSELPLCY**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/NF发现和选择管理/SMF选择策略管理/增加SMF选择策略（ADD SMFSELPLCY）_09653765.md)

## [任务示例](#ZH-CN_OPI_0276471296)

任务描述

用户附着在EPS并建立PDN连接。长时间无业务后进入空闲态。位置移动出原E-UTRAN覆盖区域到达了一个仅有NG-RAN基站的覆盖区域。注册流程被触发。在5G网络多切片情况下，保证在多切片场景下，4/5G互操作的业务连续性。

脚本

//控制SMF通过本地配置分配切片。为了避免发生SMF通过UDM获取不到签约的切片的异常场景，设置PDN连接建立流程中，SMF通过本地配置分配切片，需要设置REPSNSSAI参数为"NO"。

```
SET SMFFUNC: REPSNSSAI=NO;
```

//指定PDN连接建立流程中根据DNN为用户分配的互操作默认切片。

```
ADD NSDNN: NSIDX=0, DNN="123.com", DEFIWKIND=YES;
ADD NSDNN: NSIDX=1, DNN="abc.com", DEFIWKIND=YES;
```

//AMF支持配置互操作专用切片发现I-SMF。

```
ADD SMFSELPLCY:SUBRANGE=ALL_USER,ISMFSW=NO,TAISW=NO,IMSISW=NO,MSISDNSW=NO,RETRYSW=NO,PREFERSMFSW=NO,DNNFMT=DEFAULT,SMFLIST=DEFAULT,IWKNSSW=CFG,IWKSST=1,IWKSD="FFFFFF",ISMFRESEL=PROTOCOL;
```
