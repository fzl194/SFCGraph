#### [命令功能](#ZH-CN_MMLREF_0000001126146172)

**适用网元：MME**

该命令用于显示DDN（Downlink Data Notification）信令抑制状态。

#### [注意事项](#ZH-CN_MMLREF_0000001126146172)

无。

#### [本地用户权限](#ZH-CN_MMLREF_0000001126146172)

manage-ug；system-ug；monitor-ug

#### [网管用户权限](#ZH-CN_MMLREF_0000001126146172)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_MMLREF_0000001126146172)

无。

#### [使用实例](#ZH-CN_MMLREF_0000001126146172)

1. 显示DDN信令抑制状态。DSP DDNTHROTPARA:;
  ```
  %%DSP DDNTHROTPARA:;%%
  RETCODE = 0  操作成功。

  查询结果如下
  --------------
  抑制状态  =  未启动
  (结果个数 = 1)

  ---    END
  ```
2. 显示DDN信令抑制状态。DSP DDNTHROTPARA:;
  ```
  %%DSP DDNTHROTPARA:;%%
  RETCODE = 0  操作成功。

  查询结果如下
  --------------
       抑制状态  =  已启动
  抑制比例（%）  =  98
       抑制时长  =  3
   抑制时长单位  =  10分钟
  (结果个数 = 1)

  ---    END
  ```
3. 显示DDN信令抑制状态。DSP DDNTHROTPARA:;
  ```
  %%DSP DDNTHROTPARA:;%%
  RETCODE = 0  操作成功。

  查询结果如下
  --------------
       抑制状态  =  已启动
  抑制比例（%）  =  0
       抑制时长  =  3
   抑制时长单位  =  1小时
  (结果个数 = 1)

  ---    END
  ```

#### [输出结果说明](#ZH-CN_MMLREF_0000001126146172)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 抑制状态 | 该参数表示MME的DDN信令抑制的启动状态。<br>取值范围：<br>- 未启动：该状态下，MME未启动DDN信令抑制。<br>- 已启动：该状态下，MME已启动DDN信令抑制。 |
| 抑制比例（%） | 该参数表示MME根据当前状态制定的低优先级DDN信令的丢弃比例。即MME期望S-GW丢弃的DDN信令数占总DDN信令数的比例。 |
| 抑制时长 | 该参数表示DDN信令抑制策略的生效时长。该参数值是根据<br>[**SET DDNTHROTPARA**](设置DDN信令抑制参数(SET DDNTHROTPARA)_26305980.md)<br>中的“抑制时长”参数（DELAYVALUE）转换而来的。具体转换的原则如下：<br>- 15<=DELAYVALUE<=60，抑制策略中“抑制时长”取值（DELAYVALUE/2），“抑制时长单位”为“2秒”。<br>- 60<DELAYVALUE<=1800，抑制策略中“抑制时长”取值（DELAYVALUE/60），“抑制时长单位”为“1分钟”。<br>- 1800<DELAYVALUE<=18000，抑制策略中“抑制时长”取值（DELAYVALUE/600），“抑制时长单位”为“10分钟”。<br>- 18000<DELAYVALUE<=108000，抑制策略中“抑制时长”取值（DELAYVALUE/3600），“抑制时长单位”为“1小时”。<br>- 108000<DELAYVALUE<=1116000，抑制策略中“抑制时长”取值（DELAYVALUE/36000），“抑制时长单位”为“10小时”。 |
| 抑制时长单位 | 该参数表示DDN信令抑制策略生效时长的时间单位。该参数值是根据<br>[**SET DDNTHROTPARA**](设置DDN信令抑制参数(SET DDNTHROTPARA)_26305980.md)<br>中的“抑制时长”参数（DELAYVALUE）转换而来的。具体转换的原则参考本命令中输出项“抑制时长”的解释。 |
