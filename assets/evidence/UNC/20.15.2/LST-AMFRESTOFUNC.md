## [命令功能](#ZH-CN_MMLREF_0000001135374737)

**适用NF：AMF**

该命令用于查询AMF热备容灾功能的运行模式及相关参数。

## [注意事项](#ZH-CN_MMLREF_0000001135374737)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001135374737)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001135374737)

无

## [使用实例](#ZH-CN_MMLREF_0000001135374737)

查询AMF热备容灾，执行如下命令：

```
%%LST AMFRESTOFUNC:;%%
RETCODE = 0  操作成功

结果如下
-------
                    运行模式  =  运行
容灾用户上下文合法性校验范围  =  5
          延迟删除UE Context  =  是
                    备份类型  =  全量备份
         Binding头域携带策略  =  NULL
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001135374737)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 运行模式 | 该参数用于指定是否启用AMF热备容灾。 |
| 容灾用户上下文合法性校验范围 | 该参数用于指定备份用户上下文的NAS COUNT消息校验阈值。 |
| 延迟删除UE Context | 该参数用于指定是否延迟删除UE Context。<br>取值为YES时，原AMF故障后，容灾AMF接替原AMF为UE提供服务。如果这时UE因为4G/5G互操作流程离开5G网络到4G网络，容灾AMF作为Old AMF完成相关业务流程后，保留UE Context。<br>取值为NO时，上述场景下业务流程结束后，相关的UE Context会被系统删除。 |
| 备份类型 | 该参数用于设置AMF热备容灾的备份类型。 |
| Binding头域携带策略 | 该参数用于指定AMF和对端NF（SMF/UDM/PCF/SMSF）交互时，是否在3gpp-Sbi-Binding头域中携带备用AMF信息。 |
