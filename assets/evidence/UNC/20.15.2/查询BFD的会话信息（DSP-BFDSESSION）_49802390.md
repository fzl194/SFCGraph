#### [命令功能](#ZH-CN_CONCEPT_0000001549802390)

该命令用于显示BFD会话的状态信息，可以按照本地描述符和目的地址过滤。

#### [注意事项](#ZH-CN_CONCEPT_0000001549802390)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001549802390)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001549802390)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| LOCALDISCR | 本地标识符 | 可选必选说明：可选参数<br>参数含义：会话本地描述符。用于唯一标识本端会话。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～49152。<br>默认值：无 |
| DESTADDR | 目的IP地址 | 可选必选说明：可选参数<br>参数含义：会话检测链路的目的地址。<br>数据来源：对端协商<br>取值范围：字符串类型，输入长度范围为1～128。IPv4或者IPv6地址。<br>默认值：无 |

#### [使用实例](#ZH-CN_CONCEPT_0000001549802390)

显示目的地址为10.1.1.1的所有BFD会话的详细信息：

```
DSP BFDSESSION:DESTADDR="10.1.1.1";
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000001549802390)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 本地标识符 | 会话本地描述符。用于唯一标识本端会话。 |
| 目的IP地址 | 会话检测链路的目的地址。 |
| 远端标识符 | 会话远端描述符。用于唯一标识远端会话。 |
| 源IP地址 | 会话检测链路的源地址。 |
| 检测倍数 | 会话检测倍数，如果BFD会话在设置的检测周期内没有收到BFD报文，则认为链路发生了故障，BFD会话的状态将会置为Down。 |
| 最小发送间隔（ms） | 会话最小发送报文间隔。 |
| 最小接收间隔（ms） | 会话最小接收报文间隔。 |
| 单臂Echo会话的收包间隔（ms） | 单臂Echo最小接收报文间隔。 |
| 会话名称 | 会话名称。 |
| 会话状态 | 会话的当前状态。包括： UP：BFD会话处于Up状态； DOWN：BFD会话处于Down状态； 管理DOWN：如果对BFD会话执行了管理Down操作，将显示此种状态； INIT：BFD会话处于Init状态。 |
| 会话类型 | 会话类型，单跳IP会话、多跳IP会话或者LDP隧道会话。 |
| 出接口名称 | 出接口名称。 |
| 报文优先级 | 报文优先级，根据BFD绑定的业务的重要性，来设置BFD会话的优先级。在系统堵塞的情况下优先发送优先级高的BFD报文。 |
| 实际发送间隔（ms） | 实际发送报文的间隔。本地BFD报文实际发送时间间隔＝MAX { 本地配置的发送时间间隔，对端配置的接收时间间隔 }。 |
| 实际接收间隔（ms） | 实际接收报文的间隔。本地实际接收时间间隔＝MAX { 对端配置的发送时间间隔，本地配置的接收时间间隔 }。 |
| 检测时间（ms） | 检测时间间隔。本地实际检测时间＝本地实际接收时间间隔×对端配置的BFD检测倍数。 |
| 实际检测倍数 | 会话检测倍数，如果BFD会话在设置的检测周期内没有收到对端发来的BFD报文，则认为链路发生了故障，BFD会话的状态将会置为Down。 |
| 本地诊断字 | 最近一次会话Down的本地诊断原因。 |
| VPN实例名称 | VPN名称。 |
| UDP目的端口号 | 会话报文的目的端口号，单跳为3784，多跳会话是4784。 |
| 会话绑定应用 | 会话绑定应用。 |
| 描述信息 | 会话描述信息，一般用于描述该静态会话检测的链路等。 |
| RU名称 | 表示资源单元名称。 |
| 是否配置PST | 命令行或APP应用配置使能PST信息。 |
| 是否操作PST | 会话状态变化是否操作PST的标志。如果允许操作PST，当BFD检测到接口状态变为Down时，将更改PST，底层转发流量就能够通过PST了解接口是否发生故障。每个接口只允许1个BFD会话配置该参数。 |
