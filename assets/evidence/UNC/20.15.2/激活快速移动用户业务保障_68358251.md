## [操作场景](#ZH-CN_OPI_0168358251)

本操作指导介绍在运行网络中激活快速移动用户业务保障特性的操作过程。

在LTE网络中，部分运营商在高速铁路沿线提供了专用无线网络（简称专用网络），以保证高速铁路用户（简称高铁用户）的业务体验。在这种场景下，为了确保专用网络仅被高铁用户所使用，提高用户业务体验， UNC 提供了快速移动用户业务保障特性。

- 在整个高铁专用网络覆盖区域，在用户TAU过程中，UNC基于用户的上一次访问的TAI识别用户是否来自专用网络。如果用户由专用网络飘入公共网络，则指示用户重新接入到专用网络，确保用户的业务体验。
- 在用户密集的高速铁路低速区域，UNC基于设定的专用网络eNodeB群组中的用户停留时长，识别公共网络用户（简称公网用户）是否占用专用网络。如果公网用户占用专用网络资源，则指示用户迁出专用网络。

> **说明**
> 适用于MME。

## [任务示例](#ZH-CN_OPI_0168358251)

任务描述

前提要求：

- 运营商针对高速铁路用户仅希望使用扩展RFSP策略，不使用其他基于RFSP的驻留和切换策略。

任务一：专网专用

- 运营商规划TAC112为高速铁路专用网络，希望来自该专用网络的用户飘入公共网络时，指示其可以重新接入到专用网络。在运营商的规划中，RFSP ID 2代表了公共网络eNodeB上的专用网络频点具有较高优先级。

任务二：公网用户不占用专用网络

- 高速铁路专用网络eNodeB55～65为低速区域eNodeB群组，eNodeB5～10、80～85为入口eNodeB，且在低速区域eNodeB群组中的驻留时长阈值设置为30分钟，接入次数阈值设置为5次，并对RFSP进行下发抑制控制，抑制周期为10分钟，抑制周期内目标RFSP重发次数阈值为3。RFSP ID 3代表专用网络上的公共网络频点具有较高优先级。

脚本

//前提要求

//查询是否已使用扩展RFSP策略。

```
LST RFSP:;
```

//查询结果没有包含ALL_IMSI。

```
ADD RFSP: SUBRANGE=ALL_IMSI, TRFSP=65535, EXTPLCY=YES;
```

//配置允许MME通过Downlink NAS Transport消息中将RFSP ID信元发给eNodeB。

```
SET S1CMPT: SPID=YES;
```

//专网专用

//配置跟踪区群组信息。

```
ADD TAGP: TAGPID=1, TANAME="highspeed_railway";
ADD TAGPMEM: TAGPID=1, MCC="460", MNC="03", TAC="112";
```

//配置扩展RFSP策略组。

```
ADD EXTRFSP: TYPE=SPEC_MOVE_SUB, ARETYPE=TAGP, TAGPID=1, TRFSP=2;
```

//公网用户不占用专用网络

//配置低速区域eNodeB群组信息。

```
ADD ENBGP: ENBGPID=1, TYPE=INSIDE, STICK_TIME=30, ACC_TIMES=5, ENBGPNAME="lowspeed_inside";
ADD ENBGPMEM: ENBGPID=1, ENBTYPE=MACRO_ENODEB, MCC="460", MNC="03", BGNENBID=55, ENDENBID=65;
```

//配置入口eNodeB群组信息

```
ADD ENBGP: ENBGPID=2, TYPE=ENTRY, ENBGPNAME="highspeed_usercheck_entry";
ADD ENBGPMEM: ENBGPID=2, ENBTYPE=MACRO_ENODEB, MCC="460", MNC="03", BGNENBID=5, ENDENBID=10;
ADD ENBGPMEM: ENBGPID=2, ENBTYPE=MACRO_ENODEB, MCC="460", MNC="03", BGNENBID=80, ENDENBID=85;
```

//配置扩展RFSP策略组。

```
ADD EXTRFSP: TYPE=SPEC_MOVE_SUB, ARETYPE=ENBGP, ENBGPID=1, TRFSP=3, SEND_CTRL=YES, CTRL_PERIOD=10, RFSPSNDCUNT=3;
```

//打开本特性的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV2SGHM01", SWITCH=ENABLE;
```
