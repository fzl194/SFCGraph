## [操作场景](#ZH-CN_OPI_0276948720)

本操作指导介绍在运行网络中激活本地VLR的操作过程。

本地VLR是指MME备份用户的签约数据，用户恢复业务过程中取用本地信息，减少MME和HSS之间的流量，降低业务恢复对HSS造成的信令风暴冲击。

> **说明**
> 适用于MME。

## [必备事项](#ZH-CN_OPI_0276948720)

前提条件

- 请仔细阅读[WSFD-201202 本地VLR特性概述](特性概述_76948717.md)。
- 完成加载License。
- 完成激活[WSFD-201201 MME链式备份](../WSFD-201201 MME链式备份_75538862.md)，且恢复模式为一次呼叫恢复。
- 操作人员已经登录U2020/MAE。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0276948720)

1. 进入 “MML命令行-UNC” 窗口。
2. **可选：**配置延迟发起Update Location流程。
  [**SET RESTOFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/网络管理/MME容灾管理/容灾功能管理/设置容灾功能(SET RESTOFUNC)_72345713.md)
  > **说明**
  > 为避免对HSS造成信令冲击，建议执行本步骤，将 “HSS注册策略（DULR）” 配置为 “DELAY” 。
3. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)

## [任务示例](#ZH-CN_OPI_0276948720)

任务描述

启用本地VLR特性，配置延迟发起Update Location流程。

脚本

//配置延迟发起Update Location流程。

```
SET RESTOFUNC: DULR=DELAY;
```

//打开本特性License开关。

```
SET LICENSESWITCH: LICITEM="LKV2LVLR01", SWITCH=ENABLE;
```
