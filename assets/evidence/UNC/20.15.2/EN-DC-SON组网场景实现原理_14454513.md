# EN-DC SON组网场景实现原理

**业务流程**

- Intra MME配置传输流程
  只通过一个MME的转接实现eNodeB与en-gNodeB之间配置信息的传输，发生在eNodeB2已与en-gNodeB建立X2链路，且eNodeB1、eNodeB2、en-gNodeB都注册到同一个MME的场景下。MME通过eNodeB1注册流程记录eNodeB1标识和S1连接的关系及eNodeB2与en-gNodeB的X2链路关系，所以可以实现eNodeB1与en-gNodeB之间的配置信息转发并建立两者间的X2链路。

  **图1** Intra MME配置传输流程示意图

  <br>

  ![](EN-DC SON组网场景实现原理_14454513.assets/zh-cn_image_0000001514574241_2.png)

  1. eNodeB1向MME发起S1建链请求，用于交换设备信息。
    2. eNodeB2向MME发起S1建链请求，并携带Connected en-gNB List信元，该信元中包含邻接en-gNodeB标识和当前基站所支持的TAI即Supported TAs信元，用于交换设备信息。MME将eNodeB2添加到MME维护的S1链路表中，同时会将eNodeB2添加到en-gNodeB的邻接关系表中。
    3. eNodeB1向MME发送eNB Configuration Transfer消息请求转发邻接en-gNodeB配置信息，该消息携带了EN-DC SON Configuration Transfer信元，根据该信元，MME可判断当前eNodeB1所请求流程为EN-DC SON配置传输流程。EN-DC SON Configuration Transfer信元中包括了Target en-gNB ID子信元，且子信元中携带目标en-gNodeB的标识、Selected TAI等信息。
    4. 因为en-gNodeB不与MME直接互连，是通过与之提前建立X2链路的eNodeB2为锚点实现与MME之间配置信息传输，因此MME通过步骤[3](#ZH-CN_CONCEPT_0000001514454513__li2127295902180640)感知到当前流程为EN-DC SON配置传输后，会根据eNodeB1向MME发送的eNB Configuration Transfer消息寻找与当前en-gNodeB已完成X2建联的eNodeB。当前实现方式有如下两种：
          - **EN-DC SON Configuration Transfer信元中携带了Target eNB ID场景**：MME本地在步骤2记录的S1链路表中查找EN-DC SON Configuration Transfer信元中携带的Target eNB ID，假设查找到的结果是eNodeB2，则将eNodeB1的配置信息转发至eNodeB2；若未查找到任何结果，则丢弃eNB Configuration Transfer消息。
          - **EN-DC SON Configuration Transfer信元中未携带Target eNB ID场景**：MME检查EN-DC SON Configuration Transfer信元中若未携带的Target eNB ID，会进一步在邻接关系表中查询与此en-gNodeB邻接的eNodeB，并在多个eNodeB中选取一个作为Target eNB ID，假设被选取到的eNodeB是eNodeB2，则将eNodeB1的配置信息转发至eNodeB2；若在邻接关系表中未查询到与此en-gNodeB邻接的eNodeB，则丢弃eNB Configuration Transfer消息。
    5. eNodeB2接收到MME的消息后，通过与en-gNodeB已建立好的X2链路向en-gNodeB请求IP、端口号等信息。
    6. en-gNodeB将自身的IP、端口号等信息通过X2AP EN-DC Configuration Transfer消息返回给eNodeB2。
    7. eNodeB2将获取到的MME所需的邻接en-gNodeB信息通过eNB Configuration Transfer消息发送给MME。
    8. MME收到eNodeB2的消息后，将eNodeB1建链所需的邻接en-gNodeB信息通过EN-DC Configuration Transfer信元发送给eNodeB1。
    9. eNodeB1收到MME发送的消息中携带的en-gNodeB请求IP、端口号等信息后，与en-gNodeB之间成功建立X2链路。eNodeB1向MME发起eNB Configuration Update消息，消息中携带与eNodeB1新建立X2连接的en-gNodeB邻接关系信息，通知MME维护该eNodeB1最新的邻接关系表。

- Inter MME配置传输流程
  通过S10接口间Configuration Transfer Tunnel消息传递eNodeB与en-gNodeB之间的配置信息。如 [图2 Inter MME配置传输流程示意图](#ZH-CN_CONCEPT_0000001514454513__fig03) 所示，该场景下eNodeB1注册到MME1，eNodeB2与en-gNodeB注册到MME2上。MME通过eNodeB1/eNodeB2的注册流程分别记录eNodeB标识和S1连接的关系，后续可以根据eNodeB配置传输消息中的目的eNodeB标识选择S1连接进行转发。

  **图2** Inter MME配置传输流程示意图

  <br>

  ![](EN-DC SON组网场景实现原理_14454513.assets/zh-cn_image_0000001582086557_2.png)

  1. eNodeB1和eNodeB2分别向MME1和MME2发送S1 Setup Request消息，用于交换设备信息。
    2. MME1和MME2分别记录eNodeB标识和S1链路的关系以及eNodeB所在的TAC（Tracking Area Code）。
    3. eNodeB1向MME1发送eNB Configuration Transfer消息请求转发邻接en-gNodeB配置信息，该消息携带了EN-DC SON Configuration Transfer信元，根据该信元，MME1可判断当前eNodeB1所请求流程为EN-DC SON配置传输流程。EN-DC SON Configuration Transfer信元中包括了Target en-gNB ID子信元，且子信元中携带目标en-gNodeB的标识、Selected TAI等信息。
    4. MME1根据步骤[3](#ZH-CN_CONCEPT_0000001514454513__li452373015523)携带的目标en-gNodeB标识信息，进一步查询en-gNodeB的邻接关系表，若未在记录中查找到合适的eNodeB，则系统会判断当前流程为Inter流程，MME1会进一步根据Selected TAI信元携带的值查询DNS。
    5. DNS查询到除MME1自身以外的其他有效MME2，则会通过Configuration Transfer Tunnel消息将eNodeB1向MME1发送的配置转发请求封装后转发至目的MME即MME2。
    6. MME2根据MME1转发过来的配置转发请求消息，通过消息中携带的EN-DC SON Configuration Transfer信元判断当前流程为EN-DC SON配置传输流程。MME2进一步会根据eNodeB1向MME1发送的消息中携带的en-gNodeB标识，寻找与当前en-gNodeB已完成X2建联的eNodeB。当前实现方式有如下两种：
          - **EN-DC SON Configuration Transfer信元中携带了Target eNB ID场景**：MME2本地在记录中查找EN-DC SON Configuration Transfer信元中携带的Target eNB ID，MME2根据步骤2记录的eNodeB标识与S1链路的关系，找到消息中的目标eNodeB2的S1链路，转发eNodeB1的配置信息获取请求消息。
          - **EN-DC SON Configuration Transfer信元中未携带Target eNB ID场景**：MME2本地在记录中查找EN-DC SON Configuration Transfer信元中若未携带的Target eNB ID，会进一步在表中查询与此en-gNodeB邻接的eNodeB，并在多个eNodeB中选取一个作为Target eNB ID。MME根据步骤2记录的eNodeB标识与S1链路的关系，找到消息中的目标eNodeB2的S1链路，转发eNodeB1的配置信息获取请求消息。
    7. eNodeB2接收到MME2的消息后，通过与en-gNodeB已建立好的X2链路向en-gNodeB请求IP、端口号等信息。
    8. en-gNodeB将自身的IP、端口号等信息通过X2AP EN-DC Configuration Transfer消息返回给eNodeB2。
    9. eNodeB2将获取到的MME2所需的邻接en-gNodeB信息通过eNB Configuration Transfer消息发送给MME2。
    10. MME2将获取到的en-gNodeB IP、端口号等信息经过封装后，通过Configuration Transfer Tunnel消息发送至MME1。
    11. MME1收到MME2发送的MME Configuration Transfer消息后，将eNodeB1建链所需的邻接en-gNodeB信息通过EN-DC Configuration Transfer信元发送给eNodeB1。
    12. eNodeB1收到MME1发送的消息中携带的en-gNodeB请求IP、端口号等信息后，与en-gNodeB之间成功建立X2链路。eNodeB1向MME1发起eNB Configuration Update消息，消息中携带与eNodeB1新建立X2连接的en-gNodeB邻接关系信息，通知MME1维护该eNodeB1最新的邻接关系表。
