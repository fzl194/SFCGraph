# 调测跨域业务漫游访问（DNN纠错）（AMF）

## [操作场景](#ZH-CN_OPI_0000001211474976)

本操作指导介绍在调测跨域业务漫游场景下AMF上的DNN纠错功能。

## [必备事项](#ZH-CN_OPI_0000001211474976)

前提条件

- 请仔细阅读[WSFD-228001 跨域业务访问特性概述](特性概述_55914877.md)特性原理概述。
- 完成[部署跨域业务漫游访问（DNN纠错）（AMF）](部署跨域业务漫游访问（DNN纠错）（AMF）_11155002.md)。

数据

| 类别 | 参数名称 | 取值样例 | 说明 |
| --- | --- | --- | --- |
| 用户信息 | 国际移动用户识别码（IMSI） | 123000123456789 | - |
| 行业签约的专用DNN信息 | DNN名称（DNN） | Huawei.com | - |
| 运营商通用DNN信息 | DNN名称（DNN） | Huawei1.com | - |

## [操作步骤](#ZH-CN_OPI_0000001211474976)

1. 创建用户跟踪。
2. 用户在业务归属地激活PDU会话访问行业服务。
3. 查看用户跟踪结果。
    a. UE向AMF发送的PDU Session Establishment Request消息中携带了UE请求的DNN为 “Huawei1.com” 。
    b. AMF向UDM发送的Nudm_SubscriberDataManagement_Get Request消息请求UE的签约信息，UDM在返回的Nudm_SubscriberDataManagement_Get Response消息中返回用户签约的DNN信息“Huawei.com”
    c. AMF向SMF发送的PDU Session Establishment Request消息中携带DNN纠正后的专用DNN“Huawei.com”。
