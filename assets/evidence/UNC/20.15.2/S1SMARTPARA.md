#### [命令功能](#ZH-CN_MMLREF_0000001172225427)

**适用网元：MME**

该命令用于设置S1模式附着、服务请求、PDN连接建立流程异常抑制参数，包括异常识别门限、异常抑制的时长、Parking APN。当用户的附着请求，服务请求或PDN连接请求达到设定的阈值后，系统根据 [**ADD S1SMARTRULE**](../S1模式信令抑制规则管理/增加S1模式信令抑制规则(ADD S1SMARTRULE)_26145746.md) 配置的抑制措施进行抑制。当抑制的时间达到抑制时长后，系统会解除抑制，允许用户重新接入业务。

#### [注意事项](#ZH-CN_MMLREF_0000001172225427)

- 此命令执行后立即生效。
- 系统初次运行时，会执行系统初始设置值。
- 此命令涉及LTE UE信令控制特性（特性编号：WSFD- 206002，license部件编码：LKV2LUSC01），执行命令前请使用[**DSP LICENSE**](../../../../../../../平台服务管理/操作维护/License管理/显示License(DSP LICENSE)_00360098.md)命令确认对应特性license是否得到授权，执行[**LST LICENSESWITCH**](../../../../../../../平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md)命令确认特性开关状态为“打开”。

#### [参数说明](#ZH-CN_MMLREF_0000001172225427)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| ATTACHTHRESH | 附着异常识别门限（次） | 可选必选说明：可选参数<br>参数含义：该参数用于设置识别异常附着行为的门限，即在一个测量周期内如果用户的附着次数达到了此参数设置的值，则认为此用户的附着行为异常。<br>数据来源：整网规划<br>取值范围：2～1000<br>系统初始设置值：30<br>说明：- LTE UE信令控制（WSFD-206002）部署时此参数设置须大于“10”次，为了测试方便，当此参数设置小于或等于“10”时，需要将软参ByteEx21Bit7设置为1。 |
| ATTACHMEPERIOD | 附着异常测量周期（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于设置异常附着行为的测量周期。<br>数据来源：本端规划<br>取值范围：1～3600<br>系统初始设置值：3600 |
| ATTACHDURATION | 附着异常抑制时长（分钟） | 可选必选说明：可选参数<br>参数含义：该参数用于设置异常附着的抑制时长。执行附着抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。<br>数据来源：整网规划<br>取值范围：10~600<br>系统初始设置值：60<br>配置原则：<br>- 该参数的值和实际抑制时长有最多7分钟的误差。 |
| SRVREQTHRESH | 服务请求异常识别门限（次） | 可选必选说明：可选参数<br>参数含义：该参数用于设置识别异常服务请求行为的门限，即在一个测量周期内如果UE的服务请求次数达到了此参数设置的值，则认为此用户的服务请求行为异常。<br>数据来源：整网规划<br>取值范围：2~1000<br>系统初始设置值：100 |
| SRVREQMEPERIOD | 服务请求异常测量周期（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于设置异常服务请求行为的测量周期。<br>数据来源：本端规划<br>取值范围：1～3600<br>系统初始设置值：3600 |
| SRVREQDURATION | 服务请求异常抑制时长（分钟） | 可选必选说明：可选参数<br>参数含义：该参数用于设置异常服务请求的抑制时长。执行服务请求抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。<br>数据来源：整网规划<br>取值范围：10~600<br>系统初始设置值：60<br>配置原则：<br>- 该参数的值和实际抑制时长有最多7分钟的误差。 |
| CPSRVREQTHRESH | 控制面服务请求异常识别门限(次) | 可选必选说明：可选参数<br>参数含义：该参数用于设置识别异常控制面服务请求(CPSR)行为的门限，即在一个测量周期内如果UE的控制面服务请求次数达到了此参数设置的值，则认为此用户的控制面服务请求行为异常。<br>数据来源：全网规划<br>取值范围：2~1000<br>系统初始设置值：100 |
| CPSRVREQMEPERIOD | 控制面服务请求异常测量周期(秒) | 可选必选说明：可选参数<br>参数含义：该参数用于设置异常控制面服务请求行为的测量周期。<br>数据来源：本端规划<br>取值范围：1~3600<br>系统初始设置值：3600 |
| CPSRVREQDURATION | 控制面服务请求异常抑制时长(分钟) | 可选必选说明：可选参数<br>参数含义：该参数用于设置异常控制面服务请求的抑制时长。执行控制面服务请求抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。<br>数据来源：整网规划<br>取值范围：10~600<br>系统初始设置值：60<br>配置原则：<br>- 该参数的值和实际抑制时长有最多7分钟的误差。 |
| PDNCONNTHRESH | PDN连接异常识别门限（次） | 可选必选说明：可选参数<br>参数含义：该参数用于设置识别异常PDN连接建立行为的门限，即在一个测量周期内如果终端使用同一APN的PDN连接建立次数达到了此参数设置的值，则认为此终端的PDN连接建立行为异常。<br>数据来源：整网规划<br>取值范围：2~1000<br>系统初始设置值：30 |
| PDNCONNMEPERIOD | PDN连接异常测量周期（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于设置异常PDN连接行为的测量周期。<br>数据来源：本端规划<br>取值范围：1～3600<br>系统初始设置值：3600 |
| PDNCONNDURATION | PDN连接异常抑制时长（分钟） | 可选必选说明：可选参数<br>参数含义：该参数用于设置异常PDN连接的抑制时长。执行PDN连接抑制措施后，立即启动时长为抑制时长的定时器，在定时器超时后解除抑制，允许用户重新接入业务。<br>数据来源：整网规划<br>取值范围：10~600<br>系统初始设置值：60 |
| PARKINGAPN | Parking APN | 可选必选说明：可选参数<br>参数含义：该参数用于设置执行Parking APN激活抑制策略时使用的APN Network Identifier。异常附着或PDN连接建立流程中执行Parking APN抑制策略时使用Parking APN建立一个假承载。<br>数据来源：整网规划<br>取值范围：1~62位字符串<br>说明：“Parking APN”不配置或配置为“NULL”时，表示不允许使用Parking APN激活的抑制措施，即不进行抑制；当配置为其他，包括小写的“null”时，按照抑制策略进行抑制。 |

#### [使用实例](#ZH-CN_MMLREF_0000001172225427)

运营商A通过话务模型评估出4G终端正常的平均附着请求次数，服务请求次数和PDN连接次数后，设置附着异常识别门限为30次，附着异常测量周期为3600秒，服务请求异常识别门限为100次，服务请求异常测量周期为3600秒，PDN连接异常识别门限为30次， PDN连接异常测量周期为3600秒，抑制时长均为60分钟，由于服务请求流程和PDN连接流程配置了“Parking APN”的抑制措施，配置“Parking APN”为ParkingAPN；

SET S1SMARTPARA: ATTACHTHRESH=30, ATTACHMEPERIOD=3600, ATTACHDURATION=60, SRVREQTHRESH=100, SRVREQMEPERIOD=3600, SRVREQDURATION=60, PDNCONNTHRESH=30, PDNCONNMEPERIOD=3600, PDNCONNDURATION=60, PARKINGAPN="ParkingAPN";
