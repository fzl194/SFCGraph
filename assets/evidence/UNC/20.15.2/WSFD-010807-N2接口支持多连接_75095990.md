#### [适用 N F](#ZH-CN_TOPIC_0175095990)

AMF

#### [定义](#ZH-CN_TOPIC_0175095990)

本特性是指 UNC 支持在一个gNodeB和一个AMF之间的N2接口存在多条TNL连接（Transport Network Layer Association），减少单条TNL连接的负荷，增加链路可靠性。

#### [客户价值](#ZH-CN_TOPIC_0175095990)

| **受益方** | **受益描述** |
| --- | --- |
| 运营商 | - 减少单条TNL连接的负荷。<br>- 增加系统可靠性。<br>- 网络调整时，实现平滑用户迁移。 |
| 用户 | 减少了因N2接口负荷过大或故障不能使用网络的情况，享受更稳定的网络服务，提升用户体验。 |

#### [应用场景](#ZH-CN_TOPIC_0175095990)

- 需要减少单条TNL连接负荷时，可以使用该特性，建立多条TNL连接负荷分担。
- 需要提升系统可靠性时，可以使用该特性，建立多条TNL连接，当单条连接故障时，用户消息可以从其他连接转发。
- 在扩容或者组网调整时，可以使用该特性，通知(R)AN切换用户绑定的TNL连接，从而实现用户平滑迁移。

#### [原理概述](#ZH-CN_TOPIC_0175095990)

**基本概念**

**图1** 多TNL连接示意图

<br>

![](WSFD-010807 N2接口支持多连接_75095990.assets/zh-cn_image_0175095991_2.png)

**系统实现**

gNodeB和AMF建立连接时，互相通知对方用于建立TNL连接的地址信息。

先由AMF侧发起AMF Configuration Update流程，告知gNodeB侧AMF当前可以用于建立TNL连接的地址信息。gNodeB发起TNL连接建立流程，并在每条TNL连接建立成功之后首先发起RAN Configuration Update流程，告知AMF当前建立成功的TNL连接信息。

当用户侧或网络侧发起用户连接建立时，AMF根据负载均衡策略为用户选择一条TNL连接，绑定在该连接上，之后该用户选择此TNL连接进行业务和信令交互。

**涉及流程**

N2接口多连接涉及以下流程：

**TNL连接的建立**

gNodeB接入后，当AMF支持多偶联功能时，并且AMF上已经通过命令 **[**ADD SCTPLE**](../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/SCTP本地实体/增加SCTP本地实体(ADD SCTPLE)_11295747.md)** 配置了多个端点时，会触发TNL连接的建立流程。TNL连接建立流程如 [图2](#ZH-CN_TOPIC_0175095990__fig3633185092219) 所示。

**图2** TNL连接建立流程图

<br>

![](WSFD-010807 N2接口支持多连接_75095990.assets/zh-cn_image_0175095993_2.png)

1. AMF发送AMF Configuration Update消息，下发需要建立的TNL连接的信息。
2. gNodeB收到消息之后建立TNL连接，然后发送AMF Configuration Update Ack消息通知AMF创建的TNL连接的信息。
3. gNodeB在TNL连接上向AMF发送RAN Configuration Update消息，AMF收到消息之后确认TNL连接的建立情况。

**TNL连接的更新**

当通过命令 **[**MOD SCTPLE**](../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/SCTP本地实体/修改SCTP本地实体(MOD SCTPLE)_11295829.md)** 修改了端点信息时，AMF会发起TNL连接更新流程。如 [图3](#ZH-CN_TOPIC_0175095990__fig14417173018446) 所示。

**图3** TNL连接更新流程图

<br>

![](WSFD-010807 N2接口支持多连接_75095990.assets/zh-cn_image_0175095995_2.png)

1. AMF发送AMF Configuration Update消息，下发需要更新的TNL连接的信息。
2. gNodeB收到消息之后更新本地保存的TNL连接的相关参数信息并给AMF发送响应。

**TNL连接的释放**

当通过命令 **[**RMV SCTPLE**](../../../../OM参考/命令/UNC MML命令/业务服务管理/Pre 5G接入业务管理/控制面管理/S1接口管理/SCTP本地实体/删除SCTP本地实体(RMV SCTPLE)_11295837.md)** 删除了端点信息时，AMF会发起TNL连接释放流程。TNL连接释放流程如 [图4](#ZH-CN_TOPIC_0175095990__fig2078481463213) 所示。

**图4** TNL连接释放流程图

<br>

![](WSFD-010807 N2接口支持多连接_75095990.assets/zh-cn_image_0175095997_2.png)

1. AMF发送AMF Configuration Update消息，下发需要释放的TNL连接的信息。
2. gNodeB收到消息之后删除对应的TNL连接并向AMF发送响应。

**用户TNL连接绑定**

- 在注册流程、Service Request流程、切换流程中，在这些流程中AMF会为用户绑定一条TNL连接，后续消息都从该TNL连接收发。
- AMF可以随时触发更新用户和TNL连接的绑定关系，当AMF重新选择一条TNL连接发送用户消息，NG-RAN收到AMF从新的TNL连接发送的用户消息后，即认为该绑定关系更新。
- 当组网变更时，AMF可以发起释放流程，触发NG-RAN侧为用户选择一个新的AMF。
