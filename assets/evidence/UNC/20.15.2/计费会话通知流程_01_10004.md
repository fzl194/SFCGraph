# 计费会话通知流程

- [业务场景](#ZH-CN_TOPIC_0000001484997604__1.3.1.1)
- [信令流程](#ZH-CN_TOPIC_0000001484997604__1.3.2.1)

#### [业务场景](#ZH-CN_TOPIC_0000001484997604)

- CHF发起的用户重授权，比如用户重新充值成功，CHF通知SMF用户重新申请配额。
- CHF发起的用户去活。

#### [信令流程](#ZH-CN_TOPIC_0000001484997604)

融合计费会话通知触发场景有如下几种：

- [CHF发起的用户重授权](#ZH-CN_TOPIC_0000001484997604__p7644163952011)
- [CHF发起用户去活](#ZH-CN_TOPIC_0000001484997604__p73874195321)

**CHF发起的用户重授权** 如 [图1](#ZH-CN_TOPIC_0000001484997604__fig1010920514215) 所示，该流程仅适用于融合计费中的在线计费。

**图1** CHF发起的用户重授权流程图

<br>

![](计费会话通知流程_01_10004.assets/zh-cn_image_0000001484677628_2.png)

1. CHF向SMF发送[Nchf_ConvergedCharging_ChargingDataNotify Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_90e96f8e/Nchf_ConvergedCharging_ChargingDataNotify Request_72169905.md)消息，通知SMF重授权触发条件，消息中携带的信元举例如下所示：
  ```
  ......
  POST /nchf-convergedcharging/notify/v2/chargingdata/0890b418120f089410120a001a00b00006ffffffff HTTP/2.0
  host: 192.168.16.10:31000
  content-type: application/json;
  date: Fri, 10 Jan 2020 03:15:04 GMT
  content-length: 131
  user-agent: Go-http-client/2.0
  {
	
  "reauthorizationDetails":[

  		{

  			"quotaManagementIndicator":"ONLINE_CHARGING",

  			"ratingGroup":11

  		}

  	]
  ,
	
  "notificationType":"REAUTHORIZATION"

  }
  ```
  *表1 Nchf_ConvergedCharging_Notify Request 消息中关键信元解释*

  | 关键信元 | 信元解释 |
  | --- | --- |
  | "reauthorizationDetails":[ { "quotaManagementIndicator":"ONLINE_CHARGING", "ratingGroup":11} ] | 代表RG为11的业务需要重授权。 |
  | **"notificationType":"REAUTHORIZATION"** | 代表该Notify消息的类型为重授权。 |
2. SMF接收CHF的Notify消息，向CHF返回[Nchf_ConvergedCharging_ChargingDataNotify Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_90e96f8e/Nchf_ConvergedCharging_ChargingDataNotify Response_23849992.md)消息，如下所示：
  ```
  ......
  HTTP/2.0 
  204 No Content
  ```
3. SMF向UPF发送[PFCP Session Modification Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)消息，查询当前计费信息，通过携带Query URR向UPF请求指定URR的用量报告，或通过携带PFCPSMReq-Flags字段的QAURR信元指定查询所有URR的用量报告。
4. UPF通过PFCP会话标识识别需要修改的PFCP会话上下文，向SMF返回[PFCP Session Modification Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)消息，该消息包含UPF根据指示向SMF上报的计费信息。
5. SMF根据UPF上报的计费信息，向CHF发送[Nchf_ConvergedCharging_ChargingDataUpdate Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_f4a8bbee/Nchf_ConvergedCharging_ChargingDataUpdate Request_23690004.md)消息，请求重授权。
6. CHF向SMF响应[Nchf_ConvergedCharging_ChargingDataUpdate Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_f4a8bbee/Nchf_ConvergedCharging_ChargingDataUpdate Response_23370256.md)消息，携带新的计费事件。

**CHF发起用户去活** 如 [图2](#ZH-CN_TOPIC_0000001484997604__fig1336199112917) 所示。

**图2** CHF发起用户去活流程图

<br>

![](计费会话通知流程_01_10004.assets/zh-cn_image_0000001484237800_2.png)

1. CHF向SMF发送[Nchf_ConvergedCharging_ChargingDataNotify Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_90e96f8e/Nchf_ConvergedCharging_ChargingDataNotify Request_72169905.md)消息，该消息通过携带notificationType字段的ABORT_CHARGING信元，通知SMF去激活用户。
2. SMF向CHF返回[Nchf_ConvergedCharging_ChargingDataNotify Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_90e96f8e/Nchf_ConvergedCharging_ChargingDataNotify Response_23849992.md)消息。
3. SMF向该PDU会话对应的UPF发送[PFCP Session Deletion Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Deletion Request_86168956.md)消息。
4. UPF收到请求消息后将丢弃与该PDU会话相关的数据包，并释放与N4会话相关的所有隧道资源和上下文，并在向SMF返回的 [PFCP Session Deletion Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Deletion Response_85849356.md) 消息中携带配额的用量。
5. SMF向CHF发送[Nchf_ConvergedCharging_ChargingDataRelease Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_86c6dbcf/Nchf_ConvergedCharging_ChargingDataRelease Request_71969829.md) 消息，消息中携带UPF上报的配额用量。
6. CHF对SMF上报的用量进行扣费处理，并关闭CHF-CDR。
7. CHF向SMF返回[Nchf_ConvergedCharging_ChargingDataRelease Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_86c6dbcf/Nchf_ConvergedCharging_ChargingDataRelease Response_71849853.md)消息。
8. SMF发起PDU会话释放流程，后续流程详见[PDU会话释放（SMF发起）](../../../../../../快速入门/协议与流程/5G Core信令分析/业务流程/会话流程/PDU会话释放（SMF发起）_97353780.md)。
