## [命令功能](#ZH-CN_MMLREF_0209652276)

**适用NF：NRF**

该命令用于按照NF类型设置NRF的各类时长信息。运营商可以根据需要针对不同NF设置各类定时器时长信息。

## [注意事项](#ZH-CN_MMLREF_0209652276)

- 该命令执行后立即生效。

- 主备或双活组网的场景下，如果需要配置此命令，则两个NRF上均需执行此命令，且配置参数一致。
- 系统支持对特定NF实例的部分时长信息进行配置，通过ADD NRFNFTIMER命令实现。特定NF实例的时长信息最终以本命令和ADD NRFNFTIMER中配置的最小值为准。

## [参数说明](#ZH-CN_MMLREF_0209652276)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| NFTYPE | NF类型 | 可选必选说明：必选参数<br>参数含义：该参数用于表示各定时器时长针对的NF类型。其中，DEFALUT范围为下面列举的具体NF外的其他NF类型。<br>数据来源：全网规划<br>取值范围：<br>- DEFAULT（DEFAULT）<br>- NRF（NRF）<br>- UDM（UDM）<br>- AMF（AMF）<br>- SMF（SMF）<br>- AUSF（AUSF）<br>- NEF（NEF）<br>- PCF（PCF）<br>- SMSF（SMSF）<br>- NSSF（NSSF）<br>- UDR（UDR）<br>- LMF（LMF）<br>- GMLC（GMLC）<br>- EIR_5G（EIR5G）<br>- SEPP（SEPP）<br>- UPF（UPF）<br>- N3IWF（N3IWF）<br>- AF（AF）<br>- UDSF（UDSF）<br>- BSF（BSF）<br>- CHF（CHF）<br>- NWDAF（NWDAF）<br>- CUSTOM_OCS（CUSTOM_OCS）<br>- SCP（SCP）<br>默认值：无。<br>配置原则：无 |
| HBTIMES | 心跳超时次数 | 可选必选说明：可选参数<br>参数含义：该参数用于表示NF与NRF之间的心跳超时次数（假设为N），当NRF连续N次未收到NF的心跳，则认为此NF心跳异常。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是2~10，单位是次。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST NRFTIMER查询当前参数配置值。<br>配置原则：<br>- 心跳超时次数配置过小时，可能导致心跳正常的NF被误判为心跳异常，影响可靠性。<br>- 心跳超时次数配置过大时，可能导致NRF不能及时感知NF的心跳异常情况，同时造成一部分信令浪费。<br>- 实际配置由运营商统一规划。 |
| HEARTBEATTIMER | 心跳周期时长(秒) | 可选必选说明：可选参数<br>参数含义：该参数用于表示NF心跳周期时长。对于设置后新注册的NF立即生效，已注册的NF则在下一次心跳时生效。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是1~86400，单位是秒。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST NRFTIMER查询当前参数配置值。<br>配置原则：<br>- 心跳周期时长配置过小时，每个周期NF发送心跳请求，心跳相关的信令消息会很多。<br>- 心跳周期时长配置过大时，可能导致NRF不能及时感知NF的心跳异常情况。<br>- 实际配置由运营商统一规划。 |
| DISCVALIDTIMER | 服务发现缓存有效时长(秒) | 可选必选说明：可选参数<br>参数含义：该参数用于表示系统对服务发现结果的缓存有效时长。在缓存有效时长内，该系统中的NF实例再次服务发现相同NF实例时，可以直接使用缓存中的结果，不需要再去NRF上处理。超过缓存有效时间， 服务发现结果缓存数据将被清除。系统中的NF实例再次服务发现相同NF实例时需要重新向NRF发起服务发现请求。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~172800，单位是秒。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST NRFTIMER查询当前参数配置值。<br>配置原则：<br>- 服务发现缓存有效时长配置过小，系统中的NF实例再次服务发现相同NF实例时，可能每次都向NRF发送服务发现请求，导致系统内信令消息较多。<br>- 服务发现缓存有效时长配置过大，请求方NF不能及时感知服务提供方的NF变更。在缓存有效时长内，如果服务提供方的NF变更，请求方NF直接使用的缓存数据可能导致业务流程失败，造成信令浪费。<br>- 实际配置由运营商统一规划。 |
| DISCGRACETIMER | 服务发现缓存有效宽限时长(秒) | 可选必选说明：可选参数<br>参数含义：该参数用于表示NRF对服务发现结果的缓存有效宽限时长。NRF返回给NF的缓存有效时间会在“DISCVALIDTIMER”基础上再额外增加随机的宽限时长，随机宽限时长不超过本参数配置的值。目的是对NF服务发现的缓存时长进行离散化处理，防止NF侧的缓存结果中多个NF缓存时长同时失效，短时间触发大量重新服务发现而对NRF产生流量冲击。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~86400。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST NRFTIMER查询当前参数配置值。<br>配置原则：无 |
| SUBVALIDTIMER | 订阅有效时长(秒) | 可选必选说明：可选参数<br>参数含义：该参数用于表示NF服务订阅有效时长。在订阅有效时长内，订阅方NF订阅的内容生效且能正常收到被订阅NF的变化通知。超过此订阅有效时长，NF订阅的信息不再生效，被订阅方NF的变化将不再通知给订阅方NF。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是1~604800，单位是秒。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST NRFTIMER查询当前参数配置值。<br>配置原则：<br>- 订阅有效时长配置过小，如果订阅方NF希望实时感知被订阅方NF，就需要频繁发起订阅请求，增加系统内订阅相关信令。<br>- 订阅有效时长配置过大，订阅信息将会较长时间存储在NRF上，对NRF存储订阅信息的内存要求较高。<br>- 实际配置由运营商统一规划。 |
| SUBUPTGRTIMER | 订阅更新有效期误差宽限时长(秒) | 可选必选说明：可选参数<br>参数含义：该参数用于表示NF订阅更新请求中有效期误差宽限时长。当NRF的订阅有效时长大于等于3600秒，订阅更新请求消息中携带的有效期和NRF生成的订阅有效期误差小于等于该宽限时长，NRF将向NF返回204状态码；误差大于该宽限时长或者订阅有效时长小于3600秒，NRF将向NF返回200状态码。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~604800，单位是秒。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST NRFTIMER查询当前参数配置值。<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0209652276)

运营商希望减少系统中心跳相关的信令，增强心跳异常判断可靠性，调大心跳超时次数和心跳周期时长；同时，希望请求方NF能及时感知服务提供方的NF变更，减少NF订阅信息对NRF内存的冲击，调小服务发现有效时长和订阅有效时长信息： 配置NRF类型NF的心跳超时次数为5，心跳周期时长为70秒，服务发现结果缓存有效时长为36秒，服务发现缓存有效宽限时长为0，服务订阅有效时长为110秒。

```
SET NRFTIMER: NFTYPE=NRF, HBTIMES=5, HEARTBEATTIMER=70, DISCVALIDTIMER=36, DISCGRACETIMER=0, SUBVALIDTIMER=110;
```
