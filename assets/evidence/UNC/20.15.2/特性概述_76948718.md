#### [适用NF](#ZH-CN_CONCEPT_0276948718)

SGW-C、PGW-C

#### [定义](#ZH-CN_CONCEPT_0276948718)

E-UTRAN和WLAN（Wireless Local Area Network）互操作指用户可以在3GPP LTE网络和非3GPP WLAN间进行切换，而无需中断业务。

华为 UNC 支持通过基于GTPv2的S2b接口实现WLAN接入，并与基于GTPv2的S5/S8接口的E-UTRAN接入实现互操作。

#### [客户价值](#ZH-CN_CONCEPT_0276948718)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 支持LTE和WLAN网络用户同时接入，为用户切换提供平滑的服务。<br>- 核心网采用EPC架构，有利于WLAN网络向LTE-EPC网络平滑演进。 |
| 用户 | 当用户在LTE、WLAN网络覆盖范围内移动时，保证双向实时业务的连续性，提供良好的用户体验。 |

#### [应用场景](#ZH-CN_CONCEPT_0276948718)

对于实时性要求不高的数据业务，如浏览、消息业务、文件下载、流媒体等，LTE和WLAN的非优化切换即可支持该类业务的连续性。

#### [可获得性](#ZH-CN_CONCEPT_0276948718)

**涉及NF**

| 涉及NF | 版本支持 | 功能说明 |
| --- | --- | --- |
| UE | 无特殊要求 | 支持WLAN和LTE的双模终端。 |
| eNodeB | 无特殊要求 | LTE用户接入时，为UE提供无线资源及承载。若UE未携带MME信息，为UE进行MME选择。 |
| MME | UNC 20.5.0及后续版本 | 为LTE接入用户提供附着、去附着等移动性管理功能。<br>为UE选择接入的SGW-C和PGW-C。和SGW-C配合，为UE提供缺省承载建立、删除、更新等会话管理功能。 |
| ePDG | 无特殊要求 | 能够连接到3GPP AAA Server和PGW-C，为UE提供信令和数据业务通道。 |
| SGW-C | UNC 20.5.0及后续版本 | 为LTE接入用户提供数据路由和转发功能。 |
| PGW-C | UNC 20.5.0及后续版本 | 通过S5/S8接口和SGW-C互通，通过S2b接口和ePDG互通，通过S6b接口和3GPP AAA Server互通，将LTE/WLAN接入用户接入到EPC网络，为UE接入外部数据网提供基本承载功能。 |
| 3GPP AAA Server | 无特殊要求 | UE从Non-3GPP网络接入时，为其提供静态QoS信息。 |

#### [原理概述](#ZH-CN_CONCEPT_0276948718)

E-UTRAN和WLAN互操作的非漫游网络结构如 [图1](#ZH-CN_CONCEPT_0276948718__fig1) 所示，Home Routed漫游网络结构如 [图2](#ZH-CN_CONCEPT_0276948718__fig2) 所示，Local Breakout漫游网络结构如 [图3](#ZH-CN_CONCEPT_0276948718__fig3) 所示。

当UE从LTE网络切换到WLAN网络，PGW-C将与目标ePDG之间建立新的GTP隧道，源SGW-C和PGW-C之间的GTP隧道承载将被释放。

当UE从WLAN网络切换到LTE网络，PGW-C将与目标SGW-C之间建立新的GTP隧道承载，源ePDG和PGW-C之间的GTP隧道将被删除。
