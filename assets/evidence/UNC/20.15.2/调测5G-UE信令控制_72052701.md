# 调测5G UE信令控制

## [操作场景](#ZH-CN_OPI_0272052701)

本操作介绍了通过调测手段检查 UNC 是否正确配置5G UE信令控制功能的过程。

> **说明**
> 适用于AMF。

## [必备事项](#ZH-CN_OPI_0272052701)

前提条件

- 请仔细阅读[WSFD-206008 5G UE信令控制特性概述](特性概述_72052698.md)
- 完成[激活5G UE信令控制](激活5G UE信令控制_72052700.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户跟踪任务 | IMSI | 460123456789 | 测试终端自带 | - |
| 测试终端使用的DNN | apn-instance | huawei1.com | 从已配置数据中获取 | - |
| 地址池视图名称 | pool | testpool0 | 从已完成的配置数据中获取 | - |
| Internet公共知名网站 | - | www.example.com | 固定取值 | 用于判断测试终端能否正常使用网络基本业务。 |

## [操作步骤](#ZH-CN_OPI_0272052701)

1. 进入 “MML命令行-UNC” 窗口。
2. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) 命令查询License配置开关是否已打开。
    - 如果“SWITCH”为“ENABLE”，请执行[3](#ZH-CN_OPI_0272052701__step7311192619543)。
    - 如果“SWITCH”为“DISABLE”，请执行**[**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)**命令打开本特性对应的License配置开关。
3. 检查用户异常信令抑制功能是否开启。
  **[LST NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/查询5G信令抑制参数（LST NGSMARTPARA）_25120889.md)**
4. 检查异常行为识别门限、测量周期、抑制策略、抑制时长信息。
  **[LST NGSMARTPARA](../../../../../OM参考/命令/UNC MML命令/业务服务管理/5G接入业务管理/移动性管理/拥塞控制/异常信令抑制参数/查询5G信令抑制参数（LST NGSMARTPARA）_25120889.md)**
    - 预期结果：各项参数显示结果与配置的结果一致。
    - 异常结果：各项参数显示结果与配置的结果不一致。
      参考 [激活5G UE信令控制](激活5G UE信令控制_72052700.md) ，查看配置是否正确。
5. 查询相关软参配置。
  **[LST AMFSOFTPARAOFBIT](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/软件参数管理/查询AMF软件参数比特位（LST AMFSOFTPARAOFBIT）_09652441.md)**
    - 预期结果：显示结果与配置结果相一致。
    - 异常结果：显示结果与配置结果不一致。
      参考 [激活5G UE信令控制](激活5G UE信令控制_72052700.md) 进行配置。
6. 进行业务测试。
  **场景：对PDU会话建立异常信令流程，抑制措施全配。**
    a. 创建用户跟踪。
      参数选择如下：
          - IMSI：被跟踪用户的IMSI。
          - 其它参数：选择默认值。（PDU会话建立异常识别门限=N，PDU会话建立异常测量周期=M）。
    b. 用户注册成功，在设置的周期内重复发起N-1次PDU会话建立流程，都失败。
    c. 在测量周期M内，用户发起第N次PDU会话建立流程，查看跟踪结果。
      “特定原因值拒绝” 抑制措施生效。
          - 预期结果：达到异常识别门限，配置的第一种抑制措施“特定原因值拒绝”生效，AMF携带原因值为 “#65 – maximum number of PDU sessions reached。” 所示。
          - 异常结果： UNC 拒绝用户PDU会话建立请求的响应消息中原因值与查询到的参数值不一致。
            参考 [激活5G UE信令控制](激活5G UE信令控制_72052700.md) ，查看配置是否正确。
    d. 在抑制周期内，用户发起第N+1次PDU会话建立请求流程，查看跟踪结果。
      “Parking DNN激活” 抑制措施生效。
          - 预期结果： “Parking DNN激活” 抑制措施生效，AMF发送给用户的PDU会话建立响应消息使用Parking DNN替代用户原有请求的DNN。
          - 异常结果：AMF发送给用户的PDU会话建立响应消息使用Parking DNN与查询到的配置参数值不一致。
            参考 [激活5G UE信令控制](激活5G UE信令控制_72052700.md) ，查看配置是否正确。
    e. 在抑制周期内，用户发起第N+1次PDU会话建立流程，查看跟踪结果。
      “网络侧去注册” 抑制措施生效。
          - 预期结果： “网络侧去注册” 抑制措施生效，Deregistration Request消息使用分离原因值与配置的参数值一致。
          - 异常结果：Deregistration Request消息使用分离原因值与配置的参数值不一致。
            参考 [激活5G UE信令控制](激活5G UE信令控制_72052700.md) ，查看配置是否正确。
    f. 在新抑制周期内，用户发起第N+2次PDU会话建立流程，查看跟踪结果。
      “丢弃” 抑制措施生效。
          - 预期结果：“丢弃”抑制措施生效，导致PDU会话建立请求消息被丢弃。
          - 异常结果：PDU会话建立请求消息没有被丢弃。
      参考 [激活5G UE信令控制](激活5G UE信令控制_72052700.md) ，查看配置是否正确。
