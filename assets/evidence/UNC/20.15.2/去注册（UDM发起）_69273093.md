#### [业务场景](#ZH-CN_TOPIC_0000001469273093)

当UE无权限继续访问网络时，或者因为操作维护原因网络侧需要UE去注册，会发生网络侧发起的去注册流程。

1. 消息交互：UDM发送[Nudm_UECM_DeregistrationAMF3GPPAccessNotify Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nudm/N8/Nudm_UECM_DeregistrationNotify/Nudm_UECM_DeregistrationAMF3GPPAccessNotify Request_07167086.md)给AMF。
  触发条件：如果UDM想立即删除用户注册上下文和PDU会话则发起去注册。
  关键信元：
  | accessType | 表示用户去注册的接入类型，比如3GPP_ACCESS。 |
  | --- | --- |
  | deregReason | 表示用户去注册的原因。 |
2. 消息交互：AMF发送[De-registration request (UE terminated de-registration)](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/De-registration request (UE terminated de-registration)_76784523.md)给UE。
  触发条件：AMF收到消息后可发起隐式去注册和显示去注册两种。
    - 隐式去注册是指AMF不发送去注册消息给UE。
    - 显式去注册是指AMF发送去注册消息给UE。如果UE处于CM-IDLE态，AMF先寻呼UE，再发送去注册请求消息。
  关键信元：
  | De-registration type | 表示用户去注册的类型。 |
  | --- | --- |
3. 消息交互：AMF发送[Nudm_UECM_DeregistrationAMF3GPPAccessNotify Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nudm/N8/Nudm_UECM_DeregistrationNotify/Nudm_UECM_DeregistrationAMF3GPPAccessNotify Response_07206990.md)给UDM。
  相关机制：如果请求成功，AMF发送“204 No Content”响应。如果请求失败，则返回错误对应的HTTP状态码。
4. 消息交互：AMF发送[Nudm_SDM_Unsubscribe Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nudm/N8/Nudm_SDM_Unsubscribe/Nudm_SDM_Unsubscribe Request_07207074.md)给UDM。
  触发条件：AMF收到去注册消息后取消UDM签约数据变更通知的订阅。
  关键信元：
  | SUPI | 用户标识。 |
  | --- | --- |
  | URI | AMF订阅成功后UDM响应消息中返回的URI。用于标识订阅数据。 |
5. 消息交互：UDM发送[Nudm_SDM_Unsubscribe Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nudm/N8/Nudm_SDM_Unsubscribe/Nudm_SDM_Unsubscribe Response_56087233.md)给AMF。
  相关机制：如果请求成功，UDM发送“204 No Content”响应。如果请求失败，无有效订阅（例如，由于未知的subscriptionId值），则返回HTTP状态码“404 Not Found”。
6. 消息交互：AMF发送[Nsmf_PDUSession_ReleaseSmContext Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_ReleaseSmContext/Nsmf_PDUSession_ReleaseSmContext Request_83854765.md)给SMF。
  触发条件：当前UE存在PDU会话时，AMF发送消息给SMF释放PDU会话和相应的用户面资源。
  关键信元：
  | Cause | 释放的原因。比如REL_DUE_TO_REACTIVATION。 |
  | --- | --- |
  | ueLocation | UE的位置信息。 |
7. 消息交互：SMF发送[PFCP Session Deletion Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Deletion Request_86168956.md)给UPF。
  触发条件：SMF收到释放会话消息后发送消息给UPF释放会话相关的隧道资源。
  关键信元：
  | message type | 指示消息类型，比如pfcp session deletion request。 |
  | --- | --- |
8. 消息交互：UPF发送[PFCP Session Deletion Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Deletion Response_85849356.md)给SMF。
  关键信元：
  | Cause | 指示接受或拒绝相应的请求消息，比如request accepted。 |
  | --- | --- |
9. 消息交互：SMF发送[Nsmf_PDUSession_ReleaseSmContext Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_ReleaseSmContext/Nsmf_PDUSession_ReleaseSmContext Response_83854789.md)给AMF。
  相关机制：如果成功，SMF响应状态码“204 No Content”，或者“200 OK”，如果失败或重定向，SMF返回“4xx/5xx”的HTTP状态码。
10. 消息交互：SMF发送[Npcf_SMPolicyControl_Delete](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Delete_67572326.md)给PCF。
  触发条件：SMF收到AMF发送的删除PDU会话相关的SM上下文请求时，如果SMF判断需要向PCF请求删除PDU会话，则删除SM策略关联。
  关键信元：
  | userLocationInfo | 表示用户当前的位置信息。 |
  | --- | --- |
11. 消息交互：PCF发送[Npcf_SMPolicyControl_Delete](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Delete_67572326.md)给SMF。
  相关机制：如果请求被接受，PCF应响应“204 No Content”HTTP状态码，响应的消息体为空。如果失败或重定向时，PCF应返回HTTP状态码。
12. 消息交互：SMF发送[Nudm_SDM_Unsubscribe Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nudm/N10/Nudm_SDM_Unsubscribe/Nudm_SDM_Unsubscribe Request_06927390.md)给UDM。
  触发条件：如果SMF处理的是UE最后一个PDU会话，则SMF会发送消息取消订阅签约数据变更通知。
  关键信元：
  | SUPI | 用户标识。 |
  | --- | --- |
  | URI | SMF订阅成功后UDM响应消息中返回的URI。用于标识订阅数据。 |
13. 消息交互：UDM发送[Nudm_SDM_Unsubscribe Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nudm/N10/Nudm_SDM_Unsubscribe/Nudm_SDM_Unsubscribe Response_55967277.md)给SMF。
  相关机制：如果请求成功，UDM发送“204 No Content”响应。如果请求失败，无有效订阅（例如，由于未知的subscriptionId值），则返回HTTP状态码“404 Not Found”。
14. 消息交互：SMF发送[Nudm_UECM_DeregistrationSMF Request](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nudm/N10/Nudm_UECM_Deregistration/Nudm_UECM_DeregistrationSMF Request_07207050.md)给UDM。
  触发条件：SMF去注册，移除在UDM中存储的PDU会话与SMF ID、SMF地址以及DNN之间的联系。
  关键信元：
  | SUPI | 用户标识。 |
  | --- | --- |
15. 消息交互：UDM发送[Nudm_UECM_DeregistrationSMF Response](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nudm/N10/Nudm_UECM_Deregistration/Nudm_UECM_DeregistrationSMF Response_07207042.md)给SMF。
  相关机制：如果请求成功，UDM发送“204 No Content”响应。如果请求失败，则返回HTTP状态码。
16. 消息交互：AMF发送[Npcf_AMPolicyControl_Delete](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N15/Npcf_AMPolicyControl_Delete_17785209.md)给PCF。
  相关机制：如果AMF与PCF存在联系并且UE不再注册到网络，则删除AMF与PCF的AM策略关联关系。
  关键信元：
  | URI | 标识AM策略数据。 |
  | --- | --- |
17. 消息交互：PCF发送[Npcf_AMPolicyControl_Delete](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N15/Npcf_AMPolicyControl_Delete_17785209.md)给AMF。
  相关机制：如果请求被接受，PCF应响应“204 No Content”HTTP状态码，响应的消息体为空。如果请求失败，则返回HTTP状态码。
18. 消息交互：AMF发送[Npcf_UEPolicyControl_Delete](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N15/Npcf_UEPolicyControl_Delete_17785214.md)给PCF。
  相关机制：如果AMF与PCF之间存在与该UE相关的关联关系，且该UE在任何接入方式下都不再注册，则删除AMF与PCF的UE策略关联关系。
  关键信元：
  | URI | 标识AM策略数据。 |
  | --- | --- |
19. 消息交互：PCF发送[Npcf_UEPolicyControl_Delete](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N15/Npcf_UEPolicyControl_Delete_17785214.md)给AMF。
  相关机制：如果请求被接受，PCF应响应“204 No Content”HTTP状态码，响应的消息体为空。如果请求失败，则返回HTTP状态码。
20. 消息交互：UE发送[De-registration accept (UE terminated de-registration)](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/De-registration accept (UE terminated de-registration)_76785088.md)给AMF。
21. 消息交互：AMF发送[UE CONTEXT RELEASE COMMAND](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/UE CONTEXT RELEASE COMMAND_76785385.md)给(R)AN。
  触发条件：AMF释放N2信令连接。
  关键信元：
  | UE NGAP ID pair | UE NGAP ID对。 |
  | --- | --- |
  | Cause | 指示NGAP协议特定事件的原因。比如：deregister表示去注册。 |
22. 消息交互：(R)AN发送[UE CONTEXT RELEASE COMPLETE](../../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/UE CONTEXT RELEASE COMPLETE_76784409.md)给AMF。
  关键信元：
  | AMF UE NGAP ID | 标识UE在NG接口上的关联。 |
  | --- | --- |
  | RAN UE NGAP ID | 标识NG-RAN节点内NG接口的UE关联。 |
  | User Location Information | UE的位置信息。 |
