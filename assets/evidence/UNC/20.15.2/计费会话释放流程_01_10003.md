#### [信令流程](#ZH-CN_TOPIC_0278888685)

计费会话释放流程有如下几种场景：

- [UE发起PDU会话释放的计费会话释放](#ZH-CN_TOPIC_0278888685__p1350723722117)
- [CHF发起用户去活的计费会话释放](#ZH-CN_TOPIC_0278888685__p7832454112512)

1. 当UE不再需要相关业务时，将发起PDU会话释放请求流程，携带PDU Session ID等。
2. SMF释放该PDU会话的IP地址/前缀与相应的用户面资源，向该PDU会话对应的UPF发送[PFCP Session Deletion Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Deletion Request_86168956.md)，请求UPF上报对应的计费信息。
3. UPF收到请求消息后将丢弃与该PDU会话相关的数据包，并释放与N4会话相关的所有隧道资源和上下文，并将计费统计信息通过[PFCP Session Deletion Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Deletion Response_85849356.md)消息返回给SMF。
4. SMF向CHF发送[Nchf_ConvergedCharging_ChargingDataRelease Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_86c6dbcf/Nchf_ConvergedCharging_ChargingDataRelease Request_71969829.md)消息，消息中携带的信元举例如下所示：

  *表1 Nchf_ConvergedCharging_Release Request消息中关键信元解释*

  | 关键信元 | 信元解释 |
  | --- | --- |
  | "totalVolume":2059 | 代表上下行流量总和为2059字节。 |
  | "downlinkVolume":1488 | 代表下行流量为1488字节。 |
  | "quotaManagementIndicator":"ONLINE_CHARGING" | 代表配额管理标识，提示此业务需要配额管理。 |
  | "uplinkVolume":571 | 代表上行流量为571字节。 |
  | "triggerType":"FINAL" | 代表trigger的类型为业务终止。 |
  | "timeofLastUsage":"2019-11-06T19:49:12+08:00" | 代表最后一个业务包的时间戳。 |
  | "timeofFirstUsage":"2019-11-06T19:49:10+08:00" | 代表第一个业务包的时间戳。 |
  | "stopTime":"2020-01-10T11:15:12+08:00" | 代表停止PDU会话的时间。 |
  | "sessionStopIndicator":true | 代表SMF向CHF指示PDU会话已经终止。 |
  | "chargingId":11534353 | 代表PDU会话的计费标识为11534353。 |
  > **说明**
  > 融合计费场景中，当用户采用离线计费，即无配额管理时， [Nchf_ConvergedCharging_ChargingDataRelease Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_86c6dbcf/Nchf_ConvergedCharging_ChargingDataRelease Request_71969829.md) 消息中的quotaManagementIndicator信元标识为 **OFFLINE_CHARGING** 。
  >
  > 在 [Nchf_ConvergedCharging_ChargingDataRelease Request](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_86c6dbcf/Nchf_ConvergedCharging_ChargingDataRelease Request_71969829.md) 消息中，当用户正常去活时，triggerType信元填写FINAL；当用户异常去活时，由 **ADD N40DIAGTRIGGER** 命令的 “RELEASETRIGGER” 参数和 **SET CNVRGDCHGPARA** 命令的 “SPTABNTRIGGER” 参数控制triggerType信元填写FINAL或AbnormalRelease， **ADD N40DIAGTRIGGER** 命令优先级更高，具体控制原则如下：
  >
  > - 当**ADD N40DIAGTRIGGER**配置用户去活原因时，按照“RELEASETRIGGER”参数配置生效。
  > - 当**ADD N40DIAGTRIGGER**未配置用户去活原因时，如果**SET CNVRGDCHGPARA**命令的“SPTABNTRIGGER”参数配置为ENABLE，则支持triggerType填写AbnormalRelease；如果配置为DISABLE，则不支持triggerType填写AbnormalRelease，将triggerType改为FINAL。
5. CHF对SMF上报的用量进行扣费处理，并关闭CHF-CDR。
6. CHF向SMF返回[Nchf_ConvergedCharging_ChargingDataRelease Response](../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nchf/N40/Nchf_ConvergedCharging_Charging_86c6dbcf/Nchf_ConvergedCharging_ChargingDataRelease Response_71849853.md)消息，消息携带的信元举例如下所示：
  ```
  ......
  HTTP/2.0 
  204 No Content

  content-length: 0
  ```
7. PDU会话释放成功。
