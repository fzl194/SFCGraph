# 激活基于业务累计流量的策略控制

> ⚠ 源文档勘误（3 处）：
> ① 脚本 `ADD PCCPOLICYGRP:URRGROUPNAME="urr_01"` 但数据规划表 `URRGROUPNAME=urrgrp_01`（"urr_01" 是 URRNAME，不是 URRGROUPNAME）——脚本漏 `ADD URRGROUP` 步骤。配置生成须按对象链补 `ADD URRGROUP:URRGROUPNAME="urrgrp_01",UPURRNAME1="urr_01",DOWNURRNAME1="urr_01";` 再 `ADD PCCPOLICYGRP:URRGROUPNAME="urrgrp_01"`。依据费率标识链 [1-00009](task/UNC/20.15.2/1-00009.md) 标准结构 + [ADD PCCPOLICYGRP 命令 wiki](command/UNC/20.15.2/ADD-PCCPOLICYGRP.md)（URRGROUPNAME 引用 URRGROUP 的 URRGROUPNAME）。
> ② `SET PCCFUNC:MKPARSEFORMAT` 数据规划表有参数行（取值 UNSIGNED32），但任务示例脚本未演示该命令。
> ③ `MOD PCRF:UMCH`（使能流量监控拥塞处理）实现原理（Gx）有，但脚本完全无此命令——activation 未演示 UMCH 配置。
> feature wiki [2-00011](task/UNC/20.15.2/2-00011.md) 已脚注处置此 3 处差异。

## [操作场景](#ZH-CN_OPI_0227915156)

通过部署基于业务累计流量的策略控制功能，实现对特定业务的控制，如用户业务访问量超出限额时降低其访问速率或提高费率，达到网络公平使用的原则，使访问相同业务的用户得到同样的网络体验。

## [操作步骤](#ZH-CN_OPI_0227915156)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)
3. 设置Monitoring-Key的解析方式。
  [**SET PCCFUNC**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/PCC管理/基本功能/PCC公共参数/设置PCC功能（SET PCCFUNC）_09897057.md)
4. 配置URR标识与监控属性值的绑定关系。
  [**ADD URR**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务策略/内容计费标识/增加URR（ADD URR）_09897158.md)
5. 在PCC策略组中配置用于累计流量的URR标识。
  [**ADD PCCPOLICYGRP**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务策略/PCC策略组/增加PCC策略组（ADD PCCPOLICYGRP）_09897173.md)
6. 配置预定义规则。
  [**ADD RULE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务模板/规则/增加规则（ADD RULE）_09897200.md)
7. 配置预定义规则组。
    a. 创建用户模板。
      [**ADD USERPROFILE**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务模板/用户模板/增加用户模板（ADD USERPROFILE）_09897206.md)
    b. 将已配置的规则绑定到用户模板。
      [**ADD RULEBINDING**](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务模板/规则绑定/增加用户模板和规则的绑定关系（ADD RULEBINDING）_09897216.md)

## [任务示例](#ZH-CN_OPI_0227915156)

任务描述

配置基于业务的累计流量的策略控制。

脚本

//打开本特性的License配置开关。

```
SET LICENSESWITCH:LICITEM="LKV2FUPSAT01",SWITCH=ENABLE;
```

//配置URR标识与监控属性值的绑定关系。

```
ADD URR:URRNAME="urr_01", URRID=2000, USAGERPTMODE=MONITORINGKEY, MONITORINGKEY="2001";
```

//在PCC策略组中配置用于累计流量的URR标识。

```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pccpolicygrp_01",URRGROUPNAME="urr_01",FUPSESSIONEXC=ENABLE;
```

//配置业务使用的规则。

```
ADD RULE:RULENAME="rule_test2",POLICYTYPE=PCC,PRIORITY=20,POLICYNAME="pccpolicygrp_01";
```

//配置规则与用户模板的绑定关系。

```
ADD USERPROFILE:USERPROFILENAME="up_test",UPTYPE=PCC;
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="rule_test2";
```
