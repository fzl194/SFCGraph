## [操作场景](#ZH-CN_OPI_0301420555)

短时间内SGW-C向MME发送大量DDN消息会触发MME交互大量的寻呼信令，在SGW-C上激活下行数据通知信令抑制功能，从而达到保护MME的目的。

> **说明**
> 适用于SGW-C。

## [操作步骤](#ZH-CN_OPI_0301420555)

1. 进入 “MML命令行-UNC” 窗口。
2. 打开DDN Throttling开关。
  [**SET DDNTHROTTLING**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/流控管理/DDN Throttling功能开关/设置DDN Throttling功能（SET DDNTHROTTLING）_04284723.md)
3. 配置高优先级DDN消息的ARP。
  [**ADD DDNPRIORITYARP**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/流控管理/基于ARP的DDN消息优先级管理/增加基于ARP的DDN消息优先级配置（ADD DDNPRIORITYARP）_04284707.md)
4. 配置高优先级DDN消息的APN。
  [**ADD DDNPRIORITYAPN**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/接入管理/接入管理运维/流控管理/基于APN的DDN消息优先级管理/增加基于APN的DDN消息优先级配置（ADD DDNPRIORITYAPN）_04284706.md)

## [任务示例](#ZH-CN_OPI_0301420555)

任务描述

使能DDN Throttling功能并配置高优先级DDN消息的ARP和APN。

脚本

//使能DDN Throttling功能。

```
SET DDNTHROTTLING: DDNTHROTSWITCH=ENABLE;
```

//配置高优先级DDN消息的ARP为 **1** 、 **2** 、 **3** 。

```
ADD DDNPRIORITYARP: ARP=1,PRIORITY=DDN_HIGH;
```

```
ADD DDNPRIORITYARP: ARP=2,PRIORITY=DDN_HIGH;
```

```
ADD DDNPRIORITYARP: ARP=3,PRIORITY=DDN_HIGH;
```

//配置高优先级DDN消息的APN为 **ims** 。

```
ADD DDNPRIORITYAPN: APN="ims",PRIORITY=DDN_HIGH;
```
