#### [适用NF](#ZH-CN_TOPIC_0000001256413325)

AMF、SMF、NRF

#### [定义](#ZH-CN_TOPIC_0000001256413325)

SEPP（Security Edge Protection Proxy）安全边缘保护代理，作为5G信令网边界网关提供拓扑隐藏、安全防护等功能。5GC国际漫游场景下需要部署SEPP，AMF、SMF、NRF通过与SEPP对接，实现跨PLMN间NF注册、发现等NF交互流程的信令汇聚、拓扑隐藏和安全防护。

#### [客户价值](#ZH-CN_TOPIC_0000001256413325)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | - 简化5GC国际漫游组网的对接，快速实现信令适配，提升5GC国际漫游场景下的运维效率。<br>- 信令加密，拓扑隐藏，增强5GC国际漫游场景下的安全性。 |
| 用户 | 用户不感知该特性。 |

#### [应用场景](#ZH-CN_TOPIC_0000001256413325)

#### [原理概述](#ZH-CN_TOPIC_0000001256413325)

5GC国际漫游引入SEPP，实现跨PLMN间的NF交互的拓扑隐藏和信令集中转发功能，从而简化了漫游的组网对接，参考4G EPC组网场景下的DRA/DEA，但DRA/DEA只提供Diamter信令的Proxy转发，且依赖号段配置管理。 如 [图1](#ZH-CN_TOPIC_0000001256413325__fig497645611818) 所示。

SEPP同时对跨PLMN的信令安全传输进行了增强。

**拓扑隐藏**

在转发本PLMN的信令消息时，SEPP将本PLMN内NF的真实FQDN（NF的InterPlmnFQDN）修改为隐藏后的FQDN，避免对端PLMN的NF基于FQDN信息，获取本PLMN的拓扑结构。对于其他PLMN的NF发送至本PLMN内的消息，SEPP将本PLMN内NF隐藏后的FQDN进行拓扑还原，即本PLMN内的NF和对端PLMN的NF均不感知拓扑隐藏功能，如 [图2](#ZH-CN_TOPIC_0000001256413325__fig12629171618412) 所示。

**图2** 拓扑隐藏

<br>

![](WSFD-011144 支持与SEPP对接_56413325.assets/zh-cn_image_0000001211373452_2.png)

拓扑隐藏开启后，由于非Callback类消息的消息头中包含消息类型信息，SEPP可以根据协议定义的HTTP编码识别到包含FQDN信息的字段，对FQDN进行替换。由于Callback类消息（例如Namf_Communication_N1MessageNotify消息）的消息头中没有消息类型信息，因此各NF发送跨PLMN的Callback类消息时需要支持携带3gpp-sbi-callback扩展头，通过扩展头识别消息类型之后，SEPP可以根据协议定义的HTTP编码识别到包含FQDN信息的字段，对FQDN进行替换。3gpp-sbi-callback格式为"N<NF>_<服务名称>_<对应OpenAPI规范文件中回调服务操作的名称>"，例如“Nsmf_PDUSession_smContextStatusNotification”。

**信令汇聚**

当前产品支持如下两种信令汇聚方式，对于某个NF来说，在本PLMN内只会选择一种方式，其他PLMN不感知。

| 信令汇聚方式名称 | 方式介绍 | 说明 |
| --- | --- | --- |
| 3gpp-Sbi-Target-apiroot | 当判断是漫游用户消息需要发送至其他PLMN时，在服务请求消息的“3gpp-Sbi-Target-apiroot”中携带目标NF的地址信息，将消息发送至SEPP。SEPP收到转发消息后，基于“3gpp-Sbi-Target-apiroot”中目标NF的地址信息进行路由转发。 | 两者的区别在于3gpp-Sbi-Target-apiroot支持NF和SEPP之间配置基于TLS（Transport Layer Security）传输加密，进行传输数据保护，http Proxy不支持。<br>- 在不启用TLS场景下，两者无差异。<br>- 启用TLS场景下只支持“3gpp-Sbi-Target-apiroot”方式。 |
| http Proxy | 当判断是漫游用户消息需要发送至其他PLMN时，在服务请求消息的"authority"中携带目标NF的地址信息，将消息发送至SEPP。SEPP收到转发消息后，基于"authority"中目标NF的地址信息进行路由转发。 | 两者的区别在于3gpp-Sbi-Target-apiroot支持NF和SEPP之间配置基于TLS（Transport Layer Security）传输加密，进行传输数据保护，http Proxy不支持。<br>- 在不启用TLS场景下，两者无差异。<br>- 启用TLS场景下只支持“3gpp-Sbi-Target-apiroot”方式。 |

更多关于5GC国际漫游的介绍，请参考 [5G Core 国际漫游解决方案](../../../业务专题/5GC国际漫游解决方案/5GC国际漫游解决方案概述_11244470.md) 。
