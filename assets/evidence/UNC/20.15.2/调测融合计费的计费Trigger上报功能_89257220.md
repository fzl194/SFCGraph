# 调测融合计费的计费Trigger上报功能

- [操作场景](#ZH-CN_OPI_0289257220__1.3.1)
- [必备事项](#ZH-CN_OPI_0289257220__1.3.2)
- [操作步骤](#ZH-CN_OPI_0289257220__1.3.3)

## [操作场景](#ZH-CN_OPI_0289257220)

当运营商部署了融合计费功能，为了保证用户访问业务正常进行，需要调测计费Trigger上报功能。

## [必备事项](#ZH-CN_OPI_0289257220)

前提条件

- 请仔细阅读[WSFD-011206 支持融合计费](../../WSFD-011206 支持融合计费_62745898.md)。
- 完成加载License。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 测试终端使用的DNN | APN名称（APN） | apn-test | 已配置数据中获取 | 用户激活使用的DNN，取自接入配置中的DNN实例名。 |

工具

- 测试终端
- OM Portal

## [操作步骤](#ZH-CN_OPI_0289257220)

1. 进入 “MML命令行-UNC” 窗口。 执行 [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09651570.md) 命令查询UNC上内容计费基本功能对应的License配置开关是否打开。
  ```
  LST LICENSESWITCH:LICITEM="LKV3W9BCC12";
  ```
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0289257220__step7889143419175)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09654190.md)命令打开内容计费基本功能对应的License配置开关。
2. 在OM Portal上创建用户跟踪任务。
3. 测试终端使用 “apn-test” DNN接入网络。
4. 测试终端进行HTTP业务访问一段时间，当满足上报Trigger，如时间阈值到，查看SMF向CHF发送Charging Data Request [Update]消息中是否携带对应的计费Trigger。
    - 如果携带，调测完成。
    - 如果未携带，请执行[步骤 5](#ZH-CN_OPI_0289257220__step26275602172748)。
5. 检查CHF是否下发对应的Trigger类型。
    a. 查看用户跟踪消息，打开CHF给SMF返回的Charging Data Response [initial]/Charging Data Response [Update]消息，查看消息的triggers信元中是否包含了Session级或RG级对应的Trigger类型。
          - 如果CHF下发了对应的Trigger类型，请执行[步骤 5.b](#ZH-CN_OPI_0289257220__substep1664035711408)。
          - 如果CHF没有下发对应的Trigger类型，请执行[步骤 6](#ZH-CN_OPI_0289257220__step26116182112)。
    b. 查看Charging Data Response [initial]/Charging Data Response [Update]消息的triggers信元中Session级或RG级对应的trigger类型的生效原则是否错误。
      > **说明**
      > Trigger生效原则如下：
      >
      > - 如果Session级Trigger和RG级Trigger是完全不同的计费Trigger，按照各自的生效原则生效。
      > - 如果是Session级和RG级同时生效的计费Trigger，订阅存在冲突时，以Session级Trigger的规则优先，即如果Session级Trigger为立即上报，RG级Trigger是延迟上报，按照Session级立即上报处理。如果Session级Trigger不上报，只有RG级Trigger上报时，那么按照RG级生效原则上报且只上报该RG的所有数据。
      *表1 Trigger的上报方式*

      | 上报方式 | 详细说明 |
      | --- | --- |
      | 不上报 | 当计费Trigger发生时，不生成容器，不上报CHF。 |
      | 立即上报 | 当计费Trigger发生时，SMF立即将计费Trigger和当前的计费数据上报给CHF，之前延迟上报的也会同时上报上去。 |
      | 延迟上报 | 当计费Trigger发生时，SMF记录下当前的计费数据，但不立即触发Charging Data Request消息，而是在下一条Charging Data Request消息中将当前保存的计费数据上报给CHF。 |
          - 如果CHF下发的生效原则错误，请联系CHF端技术支持进行处理。
          - 如果CHF下发的生效原则正确，请执行[步骤 8](#ZH-CN_OPI_0289257220__step1829245425110)。
6. 执行 [**LST PDUTRIGGER**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/PDU级Trigger/查询PDU会话级的trigger参数（LST PDUTRIGGER）_09653638.md) / [**LST RGTRIGGER**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/RG级Trigger/查询RG级的trigger参数（LST RGTRIGGER）_09653261.md) 命令，查询SMF本地是否配置Session级或RG级的对应Trigger类型。
    - 如果本地已经配置，请执行[步骤 7](#ZH-CN_OPI_0289257220__step107941927143319)。
    - 如果本地未配置，请执行[**MOD PDUTRIGGER**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/PDU级Trigger/修改PDU会话级的trigger参数（MOD PDUTRIGGER）_09653719.md)/[**MOD RGTRIGGER**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/RG级Trigger/修改RG级的trigger参数（MOD RGTRIGGER）_09653135.md)命令，本地按照规划配置Session级或RG级的对应Trigger类型，并再次执行[步骤 3](#ZH-CN_OPI_0289257220__step2585105420194)。
7. 执行 [**LST PDUTRIGGER**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/PDU级Trigger/查询PDU会话级的trigger参数（LST PDUTRIGGER）_09653638.md) / [**LST RGTRIGGER**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/RG级Trigger/查询RG级的trigger参数（LST RGTRIGGER）_09653261.md) 命令，查询SMF本地配置的Session级或RG级对应Trigger类型的生效原则是否正确。
    - 如果本地配置的生效原则错误，请执行[**MOD PDUTRIGGER**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/PDU级Trigger/修改PDU会话级的trigger参数（MOD PDUTRIGGER）_09653719.md)/[**MOD RGTRIGGER**](../../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费管理/融合计费/RG级Trigger/修改RG级的trigger参数（MOD RGTRIGGER）_09653135.md)命令，本地按照规划配置Session级或RG级对应Trigger类型的生效原则，并再次执行[步骤 3](#ZH-CN_OPI_0289257220__step2585105420194)。
    - 如果本地配置的生效原则正确，请执行[步骤 8](#ZH-CN_OPI_0289257220__step1829245425110)。
8. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务，执行 [步骤 3](#ZH-CN_OPI_0289257220__step2585105420194) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
