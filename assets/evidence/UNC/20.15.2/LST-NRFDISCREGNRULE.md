## [命令功能](#ZH-CN_MMLREF_0000001171436539)

**适用NF：NRF**

该命令用于查询NRF服务发现区域识别规则。

## [注意事项](#ZH-CN_MMLREF_0000001171436539)

无

#### [操作用户权限](#ZH-CN_MMLREF_0000001171436539)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001171436539)

无

## [使用实例](#ZH-CN_MMLREF_0000001171436539)

查询NRF服务发现区域识别规则。

```
LST NRFDISCREGNRULE:;
%%LST NRFDISCREGNRULE:;%%
RETCODE = 0  操作成功

结果如下
--------
            网元类型  =  CUSTOM_OCS
区域关系精确匹配开关  =  打开
    区域信息起始位置  =  3
        区域信息长度  =  8
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001171436539)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 网元类型 | 该参数用于表示目标NF类型。 |
| 区域关系精确匹配开关 | 该参数用于表示在CHF发现OCS流程中是否根据CHF与OCS的区域关系来控制发现结果的精确匹配度。开关打开时，当CHF与OCS处于同区域，则NRF服务发现结果携带OCS的全量信元信息；当CHF与OCS处于不同区域，则NRF服务发现结果针对SUPI号段、GPSI号段、DNN列表、TAI列表等数据量较大的信元，只携带匹配上的信元信息。开关关闭时，发现过程中不进行区域关系的判断。 |
| 区域信息起始位置 | 该参数表示NF实例标识中大区或省份信息的起始位置。假设其取值为N，N表示NF Instance ID中的第N+1个字符。 |
| 区域信息长度 | 该参数表示NF实例标识中大区或省份信息的长度。 |
