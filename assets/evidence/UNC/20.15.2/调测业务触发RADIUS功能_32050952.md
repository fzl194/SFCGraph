# 调测业务触发RADIUS功能

- [操作场景](#ZH-CN_OPI_0232050952__1.3.1)
- [必备事项](#ZH-CN_OPI_0232050952__1.3.2)
- [操作步骤](#ZH-CN_OPI_0232050952__1.3.3)

## [操作场景](#ZH-CN_OPI_0232050952)

当运营商需要部署业务触发RADIUS功能时，需对 UNC 的业务触发RADIUS功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于 GGSN、 PGW-C、SMF。

## [必备事项](#ZH-CN_OPI_0232050952)

前提条件

- 请仔细阅读[WSFD-011306 Radius功能特性概述](特性概述_31467848.md)。

- 完成[配置业务触发RADIUS功能](激活RADIUS功能/配置业务触发RADIUS功能_33000859.md)。
- 设备状态正常，与周边网元的互通配置已经完成。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 测试终端使用的APN | APN名称（APN） | apn-test | 已配置数据中获取 | 取自<br>[配置业务触发RADIUS功能](激活RADIUS功能/配置业务触发RADIUS功能_33000859.md)<br>中配置的APN实例名。 |

工具

- 测试终端
- OM Portal跟踪工具

## [操作步骤](#ZH-CN_OPI_0232050952)

1. 在OM Portal上，选择Gi/SGi接口跟踪，在“Gi/SGi接口跟踪”参数配置对话框中选择“ACCT”，跟踪RADIUS计费消息。
2. 测试终端使用“apn-test”APN接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 3](#ZH-CN_OPI_0232050952__step4)。
    - 如果测试终端无法接入网络，请调测UNC的接入功能。
3. 使用测试终端，访问 **www.huawei.com** （该网页是 [配置业务触发RADIUS功能](激活RADIUS功能/配置业务触发RADIUS功能_33000859.md) 中配置的触发业务的网页），查看 UNC 是否给业务网关发送了RADIUS计费消息。
  **图1** 计费消息

  <br>

  ![](调测业务触发RADIUS功能_32050952.assets/zh-cn_image_0233078399_2.png "点击放大")

  <br>
    - 如果UNC给业务网关发送了RADIUS计费消息，即源地址、目的地址、消息ID正确，如[图1](#ZH-CN_OPI_0232050952__fig1)所示，则调测成功。
    - 否则，请执行[步骤 4](#ZH-CN_OPI_0232050952__step7)。
4. 检查APN下业务触发RADIUS相关配置是否正确。
    a. 执行 **[LST APNUSRPROFG](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务模板/APN用户模板组绑定/查询APN用户模板组绑定关系（LST APNUSRPROFG）_09897227.md)** 命令，查询APN绑定的User-profile是否正确。
      ```
      LST APNUSRPROFG:APN="apn-test";
      RETCODE = 0  操作成功。

      APN用户模板组绑定关系信息
      -------------------------
                          APN  =  apn-test
               用户模板组名称  =  upg_test
      (结果个数 = 1)
      ---    END
      ```
          - 如果APN绑定的“用户模板组名称”与规划值一致，请执行[4.b](#ZH-CN_OPI_0232050952__substepb)。
          - 如果APN绑定的“用户模板组名称”与规划值不一致，请参考[配置业务触发RADIUS功能](激活RADIUS功能/配置业务触发RADIUS功能_33000859.md)重新配置。
    b. 执行 **[LST RULEBINDING](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务模板/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_09897218.md)** 命令，查询User-profile绑定的Rule是否正确。
      ```
      LST RULEBINDING: USERPROFILENAME="up_test";
      RETCODE = 0  操作成功。

      用户模板与规则绑定信息
      -------------------------
            用户模板名称  =  up_test
                规则名称  =  rule_test1
                策略类型  =  SRV_TRIGGER
      (结果个数 = 1)
      ---    END
      ```
          - 如果User-profile绑定的“规则名称”和“优先级”与规划值一致，请执行[4.d](#ZH-CN_OPI_0232050952__substepc)。
          - 如果User-profile绑定的“规则名称”和“优先级”与规划值不一致，请参考[配置业务触发RADIUS功能](激活RADIUS功能/配置业务触发RADIUS功能_33000859.md)重新配置。
    c. 执行 **[LST RULE](../../../../../OM参考/命令/UNC MML命令/业务服务管理/会话管理/计费和策略的业务管理/业务模板/规则/查询规则（LST RULE）_09897204.md)** 命令，查询Rule的配置信息是否正确。
      ```
      LST RULE:RULENAME="rule_test1"; 
      RETCODE = 0  操作成功。

      规则信息
      -------------------------
            规则名称  =  rule_test1
            策略类型  =  SRV_TRIGGER
        流过滤器名称  =  flow-filter1
          全局优先级  =  10
            策略名称  =  NULL
                    ......
      (结果个数 = 1)
      ---    END
      ```
          - 如果rule绑定的“流过滤器名称”和“全局优先级”与规划值一致，请执行[步骤 5](#ZH-CN_OPI_0232050952__step1829245425110)。
          - 如果rule绑定的“流过滤器名称”和“全局优先级”与规划值不一致，请参考[配置业务触发RADIUS功能](激活RADIUS功能/配置业务触发RADIUS功能_33000859.md)重新配置。
5. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务并执行 [步骤 3](#ZH-CN_OPI_0232050952__step4) 并保存报文。
    b. 执行 **[EXP MML](../../../../../OM参考/命令/UNC MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md)** 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
