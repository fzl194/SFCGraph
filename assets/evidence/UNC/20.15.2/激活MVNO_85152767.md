## [操作场景](#ZH-CN_OPI_0185152767)

本操作指导介绍在运行网络中激活MVNO特性的操作过程。

MVNO（Mobile Virtual Network Operator）是指利用 MNO （Mobile Network Operator）授权的网络资源，提供业务（包括MNO提供的业务以及MVNO自己定制的业务），并对授权的资源进行维护的虚拟移动运营商。本特性是指MNO授权GSM/UMTS网络资源。

> **说明**
> 适用于 SGSN 。

[WSFD-207005 MVNO](../WSFD-207005 MVNO_85152766.md) 和 [WSFD-207006 基于LTE的MVNO](../WSFD-207006 基于LTE的MVNO_68260820.md) 可以共同部署，也可以独立部署。

- UNC作为融合网元时，应[共同部署](#ZH-CN_OPI_0185152767__zh-cn_opi_0130429108_con)[WSFD-207005 MVNO](../WSFD-207005 MVNO_85152766.md)和[WSFD-207006 基于LTE的MVNO](../WSFD-207006 基于LTE的MVNO_68260820.md)特性。
- 当UNC仅作为SGSN时，独立部署[WSFD-207005 MVNO](../WSFD-207005 MVNO_85152766.md)即可，操作步骤参见[未部署 **基于LTE的MVNO** 的情况下，部署 **MVNO**](#ZH-CN_OPI_0185152767__zh-cn_opi_0130429108_sep_one)。
- 当UNC已经作为MME部署了[WSFD-207006 基于LTE的MVNO](../WSFD-207006 基于LTE的MVNO_68260820.md)的情况下，UNC新增SGSN的功能，升级为融合网元，需要部署[WSFD-210004 MVNO](../WSFD-207005 MVNO_85152766.md)，操作步骤参见[部署 **基于LTE的MVNO** 的情况下，部署 **MVNO**](#ZH-CN_OPI_0185152767__zh-cn_opi_0130429108_sep_two)。

## [必备事项](#ZH-CN_OPI_0185152767)

前提条件

- 请仔细阅读[WSFD-207005 MVNO特性概述](特性概述_85662231.md)。
- 已完成加载License。
- 当UNC被共享时，UNC需要和其他核心网设备（例如GGSN、MSC、HLR等）连接，故在部署本特性之前，需要完成和其他核心网网元的对接配置。

脚本

- **示例1** ：MVNO通过租用IMSI号段方式向用户提供业务，MVNO和MNO使用相同的本端标识。
  //打开License配置开关。
  ```
  SET LICENSESWITCH: LICITEM="LKV2MVNO02", SWITCH=ENABLE;
  ```
  //增加MVNO标识。
  ```
  ADD MVNO: MVNOID=1, MVNON="MVNO1";
  ```
  //增加MVNO网络配置信息。
  ```
  ADD MVNONET: MVNOMCC="123", MVNOMNC="02", MATCHIMSI="111", MVNOCC="86", MVNOID=1;
  ```
  //增加MVNO资源配置信息
  ```
  ADD MVNORES: MVNOID=1, GMSNUM=10000, UMSNUM=10000, GPDPNUM=10000, UPDPNUM=10000;
  ```
  //增加MVNO功能配置信息
  ```
  ADD MVNOFUN: MVNOID=1, SMS=YES;
  ```
