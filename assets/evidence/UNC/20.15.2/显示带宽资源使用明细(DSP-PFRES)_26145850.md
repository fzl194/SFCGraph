#### [命令功能](#ZH-CN_MMLREF_0000001126145850)

**适用网元：SGSN**

该命令用于查看各个节点上的资源使用情况。

#### [注意事项](#ZH-CN_MMLREF_0000001126145850)

无

#### [本地用户权限](#ZH-CN_MMLREF_0000001126145850)

manage-ug；system-ug；monitor-ug

#### [网管用户权限](#ZH-CN_MMLREF_0000001126145850)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_MMLREF_0000001126145850)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| RUNAME | RU名称 | 可选必选说明：可选参数<br>参数含义：该参数用于指定资源单元名称。该参数可以通过<br>**DSP RU**<br>命令查询。<br>取值范围：1~63位字符串<br>默认值：无 |
| PN | 进程号 | 可选必选说明：可选参数<br>参数含义：该参数用于指定UPP进程的序号。<br>取值范围：0~20<br>默认值：无 |

#### [使用实例](#ZH-CN_MMLREF_0000001126145850)

查询资源单元为USN_SP_RU_0066上0号UPP进程的资源使用情况：

DSP PFRES: RUNAME="USN_SP_RU_0066", PN=0;

```

RETCODE = 0  操作成功。
```

#### [输出结果说明](#ZH-CN_MMLREF_0000001126145850)

| 输出项名称 | 输出项解释 |
| --- | --- |
| RU名称 | 资源单元名称。 |
| 进程号 | 待查询的UPP进程的序号。 |
| 用户面总带宽(kbits/s) | GTPU数据报文可使用的最大带宽(kbits/s)。<br>说明：此总带宽数值包含突发流量预留带宽。此值在LICENSE生效时(<br>**DSP LICENSE**<br>命令可以查看是否生效)，受LICENSE控制。在LICENSE无效时等于4Gbps平分给该RU内所有的Normal状态的UPP。 |
| 预防突发流量的预留带宽(kbits/s) | 预防突发流量的预留带宽(kbits/s)。<br>说明：此比例为系统默认值1%，不可设置。 |
| 总静态使用带宽(kbits/s) | 会话类和流类静态带宽的总和(kbits/s)。 |
| 总动态使用带宽(kbits/s) | 总动态使用带宽(kbits/s)。<br>说明：动态使用带宽是通过转发统计得到的实时流量。 |
| QoS1类业务静态带宽(kbits/s) | 预先为会话类业务分配的带宽(kbits/s)。<br>说明：其值通过签约的保证比特率计算得来。保证比特率可以通过命令<br>[**DSP SMCTX**](../../../../系统管理/用户数据库管理/显示承载上下文(DSP SMCTX)_72226033.md)<br>获得。 |
| QoS2类业务静态带宽(kbits/s) | 预先为流类业务分配的带宽(kbits/s)。<br>说明：其值通过签约的保证比特率计算得来。保证比特率可以通过命令<br>[**DSP SMCTX**](../../../../系统管理/用户数据库管理/显示承载上下文(DSP SMCTX)_72226033.md)<br>获得。 |
| QoS12类业务动态带宽(kbits/s) | 会话类和流类业务正在使用带宽的总和(kbits/s)。<br>说明：动态使用带宽是通过转发统计得到的实时流量。 |
| QoS3类业务动态带宽(kbits/s) | 交互类业务正在使用的带宽(kbits/s)。<br>说明：动态使用带宽是通过转发统计得到的实时流量。 |
| QoS4类业务动态带宽(kbits/s) | 背景类业务正在使用的带宽(kbits/s)。<br>说明：动态使用带宽是通过转发统计得到的实时流量。 |
| 目的地址非本板的转发带宽(kbits/s) | 目的地址非本板的转发带宽(kbits/s)。<br>说明：用于观察转板流量大小，验证网络规划合理性。 |
| QoS1类业务用户数 | 会话类业务用户数。 |
| QoS2类业务用户数 | 流类业务用户数。 |
| QoS3类业务用户数 | 交互类业务用户数。 |
| QoS4类业务用户数 | 背景类业务用户数。 |
| PDP数状态 | PDP上下文表的使用状态。<br>取值说明：<br>- 正常<br>- 忙<br>- 过载<br>- 拥塞<br>说明：- 正常：PDP数小于忙门限时的状态。<br>- 忙：PDP数大于等于忙门限，小于过载门限时的状态。<br>- 过载：PDP数大于等于过载门限，小于拥塞门限时的状态。<br>- 拥塞：PDP数大于等于拥塞门限时的状态。<br>- 过载和拥塞门限可以通过命令[**LST USRPDPCAP**](../../PDP资源管理/用户面PDP规格管理/查询用户面PDP规格表(LST USRPDPCAP)_72345453.md)获得。<br>忙门限计算方法：<br>- 如果过载恢复门限>20，忙恢复门限=过载恢复门限-10。否则，忙恢复门限=过载恢复门限/2。（如果过载恢复门限设置为1，则忙恢复门限为0，此时将无法恢复成正常状态。）<br>- 如果过载门限>20，忙门限=过载门限-10。否则，忙门限=过载门限/2。 （忙恢复门限大于0的情况下，如果忙门限等于忙恢复门限，则把忙门限加1。） |
| Path数状态 | GTPU路径表的使用状态。<br>取值说明：<br>- 正常<br>- 忙<br>- 过载<br>- 拥塞<br>说明：- 正常：路径数小于忙门限时的状态。<br>- 忙：路径数大于等于忙门限，小于过载门限时的状态。<br>- 过载：路径数大于等于过载门限，小于拥塞门限时的状态。<br>- 拥塞：路径数大于等于拥塞门限时的状态。<br>- 过载和拥塞门限可以通过命令[**LST GTPUPARA**](../../../GTP-U/GTP-U协议参数管理/查询GTP-U协议参数(LST GTPUPARA)_72345445.md)获得。<br>忙门限计算方法：参考<br>“PDP数状态”<br>的忙门限计算方法。 |
| 包转发速率(Packets/s) | SPU当前的包转发速率。 |
