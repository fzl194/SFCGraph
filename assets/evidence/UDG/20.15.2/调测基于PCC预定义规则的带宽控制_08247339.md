# 调测基于PCC预定义规则的带宽控制

- [操作场景](#ZH-CN_OPI_0208247339__1.3.1)
- [必备事项](#ZH-CN_OPI_0208247339__1.3.2)
- [操作步骤](#ZH-CN_OPI_0208247339__1.3.3)

## [操作场景](#ZH-CN_OPI_0208247339)

当采用基于PCC预定义规则的方式对用户的上行、下行用户面数据流量进行带宽控制时，运营商需对该功能进行调测，确保该功能可以正常使用。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0208247339)

前提条件

- 完成[基于PCC预定义规则的带宽控制](../激活基于业务感知的带宽控制/基于PCC预定义规则的带宽控制_06381676.md)。
- PCF/PCRF能够感知用户套餐的使用量动态下发预定义规则，并且已完成相关配置。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | 用户IMSI号（IMSI） | 633451234612111<br>633451234612112 | 测试终端自带 | - |
| 用户组名称 | 用户组名称（USERGROUPNAME） | ug-bwm | 已配置数据中获取 | 取自<br>[基于PCC预定义规则的带宽控制](../激活基于业务感知的带宽控制/基于PCC预定义规则的带宽控制_06381676.md)<br>中配置的<br>“USERGROUPNAME”<br>。 |

工具

- 测试终端
- 第三方抓包工具
- 可以控制发包速率的发包工具

## [操作步骤](#ZH-CN_OPI_0208247339)

1. 执行 [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令，查询License中是否允许使用基于业务感知带宽控制功能。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5TCSA01";
   ......
   -------------------------
          License项  =  LKV3G5TCSA01
        License名称  =  基于业务感知的带宽控制
               开关  =  打开
   (结果个数 = 1)
   ---    END
  ```
    - 如果“开关”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0208247339__step01)。
    - 如果“开关”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)命令打开本特性对应的License配置开关。
2. 打开镜像接口侧的抓包工具，准备抓取测试终端的出入报文。同时在OM Portal上创建 UDG N4接口跟踪任务。
3. 激活 “testapn” APN下的一个用户，该用户访问P2P业务。
  > **说明**
  > 测试前提条件：测试用户的套餐用量耗尽，业务访问量超出套餐限额；访问业务的带宽大于所配置PIR。
4. 查看镜像接口的抓包信息，利用Wireshark工具的统计功能计算S1-U/N3/SGi/N6/S5/S8接口侧用户发出接口报文速率，具体计算方法可参考 [调测针对HTTP应用基于用户的流量的带宽控制](调测针对HTTP应用基于用户的流量的带宽控制_84034126.md) 中的 [步骤 4](调测针对HTTP应用基于用户的流量的带宽控制_84034126.md#ZH-CN_OPI_0184034126__step100002399183112) 。
  > **说明**
  > 打开镜像接口侧的抓包工具，准备抓取测试终端的出入报文。通过设置不同的过滤条件（上行报文的目的IP为ServerIP，下行报文的目的IP为UE终端IP）可以同时观察上下行流量的带宽。在Server上抓包可以观察上行流量，在UE终端抓包可以观察下行流量。测试带宽管理功能需要访问业务的带宽大于所配置PIR。
  发包速率及误差范围在不同的场景下，应与不同的标准比较（CIR或者是PIR）。被控制流量的速率的大小，需要结合CAR的三种颜色对应的动作来判断。

  - 调测时配置：绿色 “PASS” ；黄色 “PASS” ；红色 “DISCARD” 。
      带宽控制的结果应为：流量的最大速率不超过峰值速率（PIR）。
      测试时需要使发包速率大于PIR，同样误差范围与PIR比较。
    - 调测时配置：绿色 “PASS” ；黄色 “DISCARD” ；红色 “DISCARD” 。
      带宽控制的结果应为：流量的最大速率不超过保证速率（CIR）。
      测试时需要使发包速率大于CIR，同样误差范围与CIR比较。
    - 调测时配置：绿色 “DISCARD” ；黄色 “DISCARD” ；红色 “DISCARD” 。
      带宽控制的结果应为：流量全部被丢弃。
      测试时发包速率没有限制，流量全部丢弃。
  以上行流量为例，通过工具观察由 UDG 出来的流量速率是否符合配置的带宽控制的速率。
    - 如果SGi/N6/S5/S8口上行报文流量带宽受控，则带宽管理功能正常，调测结束。
    - 如果SGi/N6/S5/S8口上行报文流量带宽不受控，则带宽管理功能不正常，请执行 [步骤 5](#ZH-CN_OPI_0208247339__step179601110531) 。
5. 查看N4接口跟踪消息，检查 UDG 收到的PFCP Session Establishment Request/PFCP Session Modification Request消息中携带的Activate Predefined Rules信元是否与规划值 “rule_p2pbwm” 一致。
    - 如果跟踪消息中的预定义规则名字与规划值一致 ，请执行 [步骤 6](#ZH-CN_OPI_0208247339__step9726152153613) 。
    - 如果跟踪消息中的预定义规则名字与规划值不一致 ，请检查SMF上的配置，确保SMF下发正确的预定义规则名称。
6. 执行 [**LST FLOWFILTER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/查询流过滤器（LST FLOWFILTER）_82837364.md) 、 [**LST PROTBINDFLOWF**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的协议绑定/查询流过滤器协议绑定关系（LST PROTBINDFLOWF）_82837373.md) 、 [**LST L7FILTER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/七层规则管理/七层过滤器/查询七层过滤器（LST L7FILTER）_86526660.md) 和 [**LST RULE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，检查七层过滤规则、流过滤器及rule规则是否与规划值一致。确保存在一条rule规则， “RULENAME” 的值为 “rule_p2pbwm” ， “POLICYTYPE” 的值为 “BWM” ， “POLICYNAME” 的值为 “cp_700_600” 。
    - 如果七层过滤规则、流过滤器及rule规则与规划值一致 ，请执行 [步骤 7](#ZH-CN_OPI_0208247339__step4) 。
    - 如果七层过滤规则、流过滤器及rule规则与规划值不一致 ，请参考 [基于PCC预定义规则的带宽控制](../激活基于业务感知的带宽控制/基于PCC预定义规则的带宽控制_06381676.md) 重新配置，重新执行 [步骤 3](#ZH-CN_OPI_0208247339__step2) ，注意要重新激活用户。
7. 查询用户对应的apn是否为用户组所绑定，用户组下是否配置了有效的带宽控制规则，确认用户报文是否进行了带宽控制。
    a. 执行 [**DSP SESSIONINFO**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，通过用户的IMSI查看用户激活所使用的APN（本例中用户IMSI为633451234612111）。
      ```
      DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="633451234612111";
      ```
      ```
      ......

      ```
      -----------------------------
      ```
      ```
                                       IMSI  =  633451234612111
      ```
      ```
                                          ......
      ```
      ```
                                   APN name  =  testapn
      ```
      ```
                                          ......
      ```
          - 如果IMSI绑定的“APN name”与规划值一致，请执行[步骤 7.b](#ZH-CN_OPI_0208247339__substepb)。
          - 如果IMSI绑定的“APN name”与规划值不一致，请参考[基于PCC预定义规则的带宽控制](../激活基于业务感知的带宽控制/基于PCC预定义规则的带宽控制_06381676.md)重新配置，重新执行[步骤 3](#ZH-CN_OPI_0208247339__step2)，注意要重新激活用户。
    b. 执行 [**LST APNBINDBWMUSRG**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理用户组APN绑定/查询带宽管理用户组APN绑定（LST APNBINDBWMUSRG）_82837489.md) 命令，查询带宽管理用户组下绑定的APN。
      ```
      LST APNBINDBWMUSRG:USERGROUPNAME="ug-bwm";
      ```
      ```
      ......
      -------------------------
           用户组名称  =  ug-bwm
                  APN  =  testapn
      (结果个数 = 1)
      ---    END
      ```
          - 如果带宽管理用户组绑定的“APN”与规划值一致，请执行[步骤 7.c](#ZH-CN_OPI_0208247339__substepc)。
          - 如果带宽管理用户组绑定的“APN”与规划值不一致，请参考[基于PCC预定义规则的带宽控制](../激活基于业务感知的带宽控制/基于PCC预定义规则的带宽控制_06381676.md)重新配置，重新执行[步骤 3](#ZH-CN_OPI_0208247339__step2)，注意要重新激活用户。
    c. 执行 [**LST BWMUSERGROUP**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理用户组/查询带宽管理用户组（LST BWMUSERGROUP）_82837471.md) 命令，查询用户组的相关信息。
      ```
      LST BWMUSERGROUP:QRYUSRGRPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug-bwm";
      ```
      ```
      ......
      -------------------------
                     用户组类型  =  特定用户组
                     用户组名称  =  ug-bwm
                   用户组优先级  =  1
                 用户级业务类型  =  Non TOS
               用户组级业务类型  =  Non TOS
      (结果个数 = 1)
      ---    END
      ```
          - 如果“用户级业务类型”与规划值一致，请执行[步骤 7.d](#ZH-CN_OPI_0208247339__substepd)。
          - 如果“用户级业务类型”与规划值不一致，请参考[基于PCC预定义规则的带宽控制](../激活基于业务感知的带宽控制/基于PCC预定义规则的带宽控制_06381676.md)重新配置，重新执行[步骤 3](#ZH-CN_OPI_0208247339__step2)，注意要重新激活用户。
    d. 执行 [**LST BWMRULE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理规则/查询带宽管理规则（LST BWMRULE）_82837481.md) 命令，查询用户组绑定的BWM Rule的相关信息。
      ```
      LST BWMRULE:USERGROUPTYPE=SPECIFIC_GROUP,USERGROUPNAME="ug-bwm";
      ```
      ```
       	                        ......

      ```
      ......
      -------------------------
                                  用户组名称  =  ug-bwm
                            带宽管理规则类型  =  用户级别特定
                            带宽管理规则名称  =  br-p2p
                          带宽管理规则优先级  =  1
                            带宽控制业务名称  =  my-bwm-service1

                                        漫游  =  未配置
                    上行带宽管理控制器名称一  =  bc-up-p2p
                    下行带宽管理控制器名称一  =  bc-dn-p2p
      ```
          - 如果Bwm Rule下的“带宽管理规则类型”、“带宽控制业务名称”、“上行带宽管理控制器名称一”和“下行带宽管理控制器名称一”与规划值一致，请执行[步骤 7.e](#ZH-CN_OPI_0208247339__substepe)。
          - 如果Bwm Rule下的“带宽管理规则类型”、“带宽控制业务名称”、“上行带宽管理控制器名称一”和“下行带宽管理控制器名称一”与规划值不一致，请参考[基于PCC预定义规则的带宽控制](../激活基于业务感知的带宽控制/基于PCC预定义规则的带宽控制_06381676.md)重新配置，重新执行[步骤 3](#ZH-CN_OPI_0208247339__step2)，注意要重新激活用户。
    e. 执行 [**LST BWMSERVICE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理业务/查询带宽管理业务（LST BWMSERVICE）_82837476.md) 、 [**LST BWMCONTROLLER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理控制器/查询带宽管理控制器（LST BWMCONTROLLER）_82837463.md) 命令，查询BWM Rule所对应BWM Service和BWM Controller的信息。
          - 如果BWM Service的“带宽控制业务名称”、“分类属性名称”、上行和下行的BWM Controller参数值与规划值一致，请执行[步骤 8](#ZH-CN_OPI_0208247339__step49451120657)。
          - 如果BWM Service的“带宽控制业务名称”、“分类属性名称”、上行和下行的BWM Controller参数值与规划值不一致，请参考[基于PCC预定义规则的带宽控制](../激活基于业务感知的带宽控制/基于PCC预定义规则的带宽控制_06381676.md)重新配置，重新执行[步骤 3](#ZH-CN_OPI_0208247339__step2)，注意要重新激活用户。
8. 收集信息并寻求技术支持。
    a. 在镜像接口或服务器上开启抓包工具，执行 [4](调测针对HTTP应用基于用户的流量的带宽控制_84034126.md#ZH-CN_OPI_0184034126__step2) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 联系华为技术支持解决。
