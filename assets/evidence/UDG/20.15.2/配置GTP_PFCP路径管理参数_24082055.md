## [操作场景](#ZH-CN_OPI_0124082055)

当运营商在部署分组交换网，新增 UDG 时，需配置 GTP/PFCP 路径管理参数。通过 GTP/PFCP 路径管理， UDG 可以进行到对端网元的路径管理，并控制路径中断后对已激活上下文的处理方式。

> **说明**
> -
>   非N4/Sxb接口的GTP路径管理应用以下对接网元之间。
>
>   - eNodeB与SGW-U之间。
>     - SGW-U与PGW-U之间。
>     - eNodeB/GnGp-SGSN和PGW-U之间。
>     - UPF和UPF之间。
>     - R(AN)和UPF之间。
> - N4/Sxb接口的GTP路径管理应用以下对接网元之间。
>     - PGW-C和PGW-U之间。
>     - SMF和UPF之间。
> - PFCP路径管理应用以下对接网元之间。
>     - SGW-C和SGW-U之间。
>     - PGW-C和PGW-U之间。
>     - SMF和UPF之间。
> - 路径管理在 UDG 和对端网元间是双向的，如果一方开启了本功能，为节省网络带宽资源，可以关闭另外一端的Echo消息开关。

> **说明**
> 适用于SGW-U、PGW-U、UPF。

## [操作步骤](#ZH-CN_OPI_0124082055)

1. 配置非N4接口的GTP路径管理参数。
  设置 UDG 上的Echo开关属性及请求间隔，设置Echo消息的重发时间间隔和最大尝试发送次数，配置GTP路径断开后 UDG 对该路径上已激活 PDP上下文/EPS承载 /PDU Session的处理。
  [**SET UPGTPPATH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/路径管理/GTP路径管理/GTP协议参数管理/GTP路径全局参数/设置GTP路径相关属性（SET UPGTPPATH）_82837227.md)

  > **说明**
  > - 只有在UDG启动路径管理功能时，才需执行本配置。
  > - 通过配置[**SET UPGTPPATH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/路径管理/GTP路径管理/GTP协议参数管理/GTP路径全局参数/设置GTP路径相关属性（SET UPGTPPATH）_82837227.md)的“LOGICINFTYPE”参数，可以针对指定类型接口间的路径启动路径探测功能，包括N3、S1-U、S11-U、N9c、S5-S\S8-S、Pa(N9a/S5-P/S8-P/S2b/Gn-U/Gp-U)接口。如果N3和S1-U接口合一为Saif抽象接口时，则仅能针对Saif抽象接口启动路径探测功能；如果N9c和S5-S/S8-S接口合一为Scif抽象接口时，则仅能针对Scif接口启动路径探测功能。如果N3、S1-U、S11-U、N9c、S5-S\S8-S需要启动独立接口间的路径探测功能，则必须按照独立接口配置。
  > - 不配置[**SET UPGTPPATH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/路径管理/GTP路径管理/GTP协议参数管理/GTP路径全局参数/设置GTP路径相关属性（SET UPGTPPATH）_82837227.md)的“LOGICINFTYPE”参数时，则针对所有非N4/Sxb接口间的路径启动路径探测功能。
2. 配置N4接口的GTP路径管理参数。
  设置 UDG 上的Echo开关属性及请求间隔，设置Echo消息的重发时间间隔和最大尝试发送次数，配置GTP路径断开后 UDG 对该路径上已激活PDU Session的处理。
  [**SET UPN4UPATH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/路径管理/GTP路径管理/GTP协议参数管理/N4U路径全局参数/设置N4U路径相关属性（SET UPN4UPATH）_18761567.md)

  > **说明**
  > 只有在 UDG 启动 路径管理功能 时，才需执行本配置。
3. 配置PFCP路径管理参数。
  设置 UDG 上的PFCP开关属性及请求间隔，设置PFCP Heartbeat消息的重发时间间隔和最大尝试发送次数，配置PFCP路径断开后 UDG 对该路径上已激活 PDP上下文/EPS承载 /PDU Session的处理。
  [**SET UPPFCPPATH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/路径管理/PFCP路径管理/PFCP参数管理/PFCP路径参数/设置PFCP路径相关属性（SET UPPFCPPATH）_82837240.md)

## [任务示例](#ZH-CN_OPI_0124082055)

任务描述

- 配置 UDG N9c接口的GTP路径探测参数。Echo请求间隔时间为60秒，Echo请求消息的重发时间间隔为10秒，最大尝试发送次数为5次。配置GTP数据路径断告警产生后发送20个Echo请求消息，之后去激活此路径上用户面的所有上下文。
- 配置N4接口的GTP路径探测参数。Echo请求间隔时间为60秒，Echo请求消息的重发时间间隔为3秒，最大尝试发送次数为5次。配置GTP数据路径断告警产生后不发送Echo请求消息，且去激活此路径上用户面的所有上下文。
- 配置 UDG 主动发送PFCP心跳请求的间隔时间为60秒。配置PFCP请求消息的重发时间间隔为3秒，最大尝试发送次数为5次。配置PFCP信令路径断告警产生后发送30个PFCP心跳请求消息，之后去激活此路径上用户面的所有上下文。

脚本

//设置N9c接口的GTP路径探测参数。

```
SET UPGTPPATH:V1DATAECHOSW=ENABLE,ECHOINTERVAL=60,T3RESPONSE=10,N3REQUEST=5,DEACTIVEFLAG=ENABLE,ECHOTIME=20,LOGICINFTYPE=N9c;
```

//设置N4/Sxb接口的GTP路径探测参数。

```
SET UPN4UPATH:N4ECHOSW=ENABLE,N4ECHOINTERVAL=60,N4T3RESPONSE=3,N4N3REQUEST=5,DEACTIVEFLAG=ENABLE,N4ECHOTIME=0;
```

//设置PFCP路径管理参数。

```
SET UPPFCPPATH:HEARTBEATINTERVAL=60,T3RESPONSE=3,N3REQUEST=5,DEACTIVEFLAG=ENABLE,HEARTBEATTIME=30;
```
