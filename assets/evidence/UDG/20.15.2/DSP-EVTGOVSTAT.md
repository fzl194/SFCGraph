## [命令功能](#ZH-CN_MMLREF_0000001194913407)

该命令用于在事件诊断失败情况下，查询事件诊断失败结果。

> **说明**
> 本命令只能查询事件诊断和自愈失败的事件结果，事件诊断和自愈成功的结果请使用 [**DSP EVTHEALREC**](显示事件治理自愈结果（DSP EVTHEALREC）_94753519.md) 查询。

#### [操作用户权限](#ZH-CN_MMLREF_0000001194913407)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001194913407)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| SERVICE | 服务名称 | 可选必选说明：可选参数<br>参数含义：该参数表示策略模型绑定的服务名称，若不输入此参数，则查询整系统所有服务名称的事件诊断和自愈成功的结果。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是0~255。<br>默认值：无<br>配置原则：无 |
| EVENTID | 事件ID | 可选必选说明：可选参数<br>参数含义：该参数显示事件的ID，若不输入此参数，则查询整系统某服务的所有事件ID的事件诊断和自愈成功的结果。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~4294967295。<br>默认值：无<br>配置原则：无 |

## [使用实例](#ZH-CN_MMLREF_0000001194913407)

该命令用于在事件诊断失败情况下，查询事件诊断失败结果。

```
%%DSP EVTGOVSTAT:;%%
RETCODE = 0  操作成功

结果如下
------------------------
服务名称  =  SDRS
事件ID   =  1
事件策略  =  关联诊断
进程ID  =  gtp-pod-6d746cb46-2jc8f192-168-1-236__103__0
失败阶段  =  自愈阶段
上报次数  =  5
事件数量  =  5
(结果个数 = 1)
```

## [输出结果说明](#ZH-CN_MMLREF_0000001194913407)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 服务名称 | 该参数表示策略模型绑定的服务名称，若不输入此参数，则查询整系统所有服务名称的事件诊断和自愈成功的结果。 |
| 事件ID | 该参数显示事件的ID，若不输入此参数，则查询整系统某服务的所有事件ID的事件诊断和自愈成功的结果。 |
| 事件策略 | 该参数表示事件治理的策略类型。<br>取值说明：<br>- “ASSOCIATIONPOLICY（关联诊断）”：关联诊断策略<br>- “POLLINGPOLICY（轮询诊断）”：轮询诊断策略 |
| 进程ID | 该参数表示诊断失败时嫌疑对象的进程ID。 |
| 失败阶段 | 该参数表示事件诊断失败所处的阶段。<br>取值说明：<br>- STAGEISOLATION（隔离阶段）<br>- STAGEDIAGNOSE（诊断阶段）<br>- STAGEHEALING（自愈阶段）<br>- STAGEIGENTITY（识别阶段）<br>- STAGEALARM（告警阶段） |
| 上报次数 | 该参数用于显示单个上报事件中出现次数最多的嫌疑实例的上报次数。例如：在某个事件诊断的某一时刻，共有3个上报事件：事件一（嫌疑实例为A、B、C），事件二（嫌疑实例为A、B），事件三（嫌疑实例为A，C），则出现次数最多的嫌疑实例为A，出现次数为3。 |
| 事件数量 | 该参数用于显示单个事件所有上报嫌疑实例的事件总数。 |
