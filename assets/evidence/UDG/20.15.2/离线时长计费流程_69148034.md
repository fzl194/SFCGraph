# 离线时长计费流程

离线计费中基于业务时长的计费业务流程如 [图1](#ZH-CN_CONCEPT_0169148034__fig2298129131910) 所示。

**图1** 离线计费中基于业务时长的计费特性业务流程图

<br>

![](离线时长计费流程_69148034.assets/zh-cn_image_0204998210.png)

1.
  用户激活

  - 用户发送会话建立请求。
    - SGW-C/PGW-C/SMF向PGW-U/UPF发送PFCP Session Establishment Request消息，消息中携带时间配额和计费方式。消息中的关键信元如[表1](#ZH-CN_CONCEPT_0169148034__table152619197258)所示。
      *表1 关键信元说明*

      | 关键信元 | 说明 |
      | --- | --- |
      | CC-Time | 该AVP携带用户业务对应的时间配额。 |
      | Quota-Consumption-Time | 该AVP携带值为0或不为0，表示连续时长计费方式或QCT计费方式。 |
      | Time-Quota-Mechanism<br>---Time-Quota-Type | 该AVP携带CTP或DTP计费方式。 |
      | Time-Quota-Mechanism<br>---Base-Time-Interval | 该AVP携带控制时长配额的基本时间间隔。 |
    - PGW-U/UPF向SGW-C/PGW-C/SMF返回PFCP Session Establishment Response消息。
    - SGW-C/PGW-C/SMF给用户返回会话建立响应。
2.
  用户访问业务

  - 用户开始访问业务。
    - PGW-U/UPF统计业务访问时长。当时间配额用完时，PGW-U/UPF向SGW-C/PGW-C/SMF发送PFCP Session Report Response消息上报业务统计时长。
    - SGW-C/PGW-C/SMF发送PFCP Session Report Response消息响应给PGW-U/UPF。
    - 当满足话单产生条件时，SGW-C/PGW-C/SMF向CG发送DataRecord Transfer Request。
    - CG返回DataRecord Transfer Response消息给SGW-C/PGW-C/SMF。
3.
  用户去活

  - 用户终止业务，UE发起终止会话请求。
    - SGW-C/PGW-C/SMF向用户回终止会话响应。
    - SGW-C/PGW-C/SMF通知PGW-U/UPF用户去活。
    - PGW-U/UPF向SGW-C/PGW-C/SMF上报业务统计结果。
    - SGW-C/PGW-C/SMF向CG发送话单。
    - CG向SGW-C/PGW-C/SMF返回成功响应。
