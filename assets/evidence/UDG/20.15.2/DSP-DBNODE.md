#### [命令功能](#ZH-CN_CONCEPT_0129626990)

该命令用于查询CSDB的节点信息，包括节点位置信息、状态信息、负载信息等。

#### [注意事项](#ZH-CN_CONCEPT_0129626990)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0129626990)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0129626990)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| CLUSTERID | 子集群标识 | 可选必选说明：必选参数。<br>参数含义：该参数用于指定特定子集群，可以通过<br>**[DSP DBCLUSTER](../集群管理/查询CSDB子集群信息（DSP DBCLUSTER）_29626985.md)**<br>命令查询获取。<br>数据来源：本端规划<br>取值范围：1～10。<br>默认值：无。 |
| RUNAME | 资源单元名称 | 可选必选说明：可选参数。<br>参数含义：该参数用于指定服务端进程所在的资源单元名称，可以通过<br>**[LST SERVICERUSTATE](../../../单体服务编排功能管理/系统管理/资源管理/RU信息/查询RU的信息(LST SERVICERUSTATE)_29626965.md)**<br>命令查询获取。<br>数据来源：本端规划<br>取值范围：字符串类型，长度为1～63。<br>默认值：无。 |
| NODETYPE | 节点类型 | 可选必选说明：必选参数。<br>参数含义：该参数用于指定服务端进程的类型。<br>数据来源：本端规划<br>取值范围：<br>- “CONTROL”：查询控制节点（PRP进程）的信息。<br>- “STORAGE”：查询存储节点（DNP进程）的信息。<br>默认值：无。 |
| NODENO | 同类进程实例号 | 可选必选说明：可选参数。<br>参数含义：该参数用于指定同类型进程在资源单元内的排序序号，从0开始排序。<br>数据来源：本端规划<br>取值范围：0～63。<br>默认值：无。<br>配置原则：<br>- 如果节点类型是“CONTROL”，本参数的值应为“0”。<br>- 如果节点类型是“STORAGE”，本参数的值必须小于STORAGE节点数的最大值。<br>- 不输入该参数可对指定类型的所有节点生效。 |
| NODESTATE | 节点状态 | 可选必选说明：可选参数。<br>参数含义：该参数用于指定服务端进程的状态。<br>数据来源：本端规划<br>取值范围：<br>- “NORMAL”<br>- “FAULT”<br>默认值：无。<br>配置原则：如果不指定节点状态，显示所有节点状态的节点信息。 |

#### [使用实例](#ZH-CN_CONCEPT_0129626990)

查询 “子集群标识” 为 “1” 、 “节点类型” 为 “STORAGE” 的节点信息：

DSP DBNODE: CLUSTERID=1, NODETYPE=STORAGE;

#### [输出结果说明](#ZH-CN_CONCEPT_0129626990)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 子集群标识 | 唯一标识一个子集群。<br>取值范围：1～10。 |
| 资源单元名称 | 描述节点所在的资源单元名称。<br>取值范围：字符串类型，长度为1～63。 |
| 节点类型 | 节点类型。<br>取值范围：<br>- “控制节点”。<br>- “存储节点”。 |
| 同类进程实例号 | 同类型节点在资源单元内的序号，从0开始排序。<br>取值范围：0～63。 |
| 节点通信信息 | 由虚拟化网络功能组件标识和节点在子集群内的序号组成。<br>取值范围：字符串类型，长度为1～63。 |
| 节点状态 | 节点状态。<br>取值范围：<br>- “正常”。<br>- “故障”。 |
| 节点主备状态 | 节点的主备状态。<br>取值范围：<br>- “主”。<br>- “备”。 |
| 内存大小(M) | 内存容量，单位MB。<br>取值范围：0～4294967295。 |
| 内存使用率(%) | 节点内存使用率，单位百分比。<br>取值范围：0～100。<br>说明：CSDB使用的内存分区是自动扩展和回收的，因此实际内存分区大小与总内存分区大小接近，内存使用率比较高。 |
| CPU占用率(%) | 节点CPU占用率，单位百分比。<br>取值范围：0～100。 |
| 队列使用率(%) | 节点队列使用率，单位百分比。<br>取值范围：0～100。 |
| 持久化状态 | 节点的持久化状态。<br>PRP节点的取值范围：<br>- “已经开启持久化”。<br>- “已经关闭持久化”。<br>DNP节点的取值范围：<br>- “已经开启持久化”。<br>- “已经关闭持久化”。<br>- “正在开启持久化”。<br>- “正在关闭持久化”。<br>- “其他”。 |
| Worker个数 | DNP进程上工作线程的个数。<br>取值范围：0～4294967295。 |
| 表空间计算方法 | DNP进程表空间的计算方法。<br>取值范围：<br>- “原模型法”：通过原模型文件设置DNP进程数据存储内存大小。<br>- “软参法”：通过软参设置DNP进程数据存储内存大小。<br>- “环境变量法”：通过环境变量设置DNP进程数据存储内存大小。<br>- “其他”：其他场景。 |
| 性能权重 | 节点的负载所占的权重值。<br>取值范围：整型，0～4294967295。 |
