# 调测基于用户/用户组/整机三级控制中的整机的带宽控制

- [操作场景](#ZH-CN_OPI_0184034130__1.4.1)
- [必备事项](#ZH-CN_OPI_0184034130__1.4.2)
- [操作步骤](#ZH-CN_OPI_0184034130__1.4.3)

## [操作场景](#ZH-CN_OPI_0184034130)

当运营商部署基于用户/用户组/整机三级控制中的整机的带宽管理功能，来对整机指定IP TOS等级的上行、下行用户面数据流量进行带宽控制时，需对 UDG 上的基于整机区分IP TOS等级的带宽管理功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0184034130)

前提条件

完成 [基于用户/用户组/整机三级的带宽控制](../激活基于业务感知的带宽控制/基于用户_用户组_整机三级的带宽控制_84034124.md) 。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 带宽管理规则名称 | 带宽管理规则名称（BWMRULENAME） | br_global_test | 已配置数据中获取 | 取自<br>[基于用户/用户组/整机三级的带宽控制](../激活基于业务感知的带宽控制/基于用户_用户组_整机三级的带宽控制_84034124.md)<br>中配置的<br>“BWMRULENAME”<br>。 |

工具

- 测试终端
- 第三方抓包工具
- 可以控制发包速率的发包工具

## [操作步骤](#ZH-CN_OPI_0184034130)

1. 执行 [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令，查询License中是否允许使用基于业务感知带宽控制功能。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5TCSA01";
   ......
   -------------------------
          License项  =  LKV3G5TCSA01
        License名称  =  基于业务感知的带宽控制
               开关  =  打开
   (结果个数 = 1)
   ---    END
  ```
    - 如果“开关”为“ENABLE”，请执行[3](#ZH-CN_OPI_0184034130__step2)。
    - 如果“开关”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)命令打开本特性对应的License配置开关。
2. 打开镜像接口侧的抓包工具，准备抓取测试终端的出入报文。
3. 在同一个 Instance 上激活，测试整机的带宽管理，需要同时发起多个用户的流量。
  激活同一个 Instance 上的2个用户，使用发包工具发送TOS域的值为BE（0）的用户报文，如 [图1](#ZH-CN_OPI_0184034130__fig4) 所示红色圈内标识DSCP域的值为0，表示TOS域的值为BE（0）。
  **图1** DSCP域值

  <br>

  ![](调测基于用户_用户组_整机三级控制中的整机的带宽控制_84034130.assets/fig_1_4d204.png "点击放大")

  <br>
  > **说明**
  > 激活同一个 Instance 上的2个用户，即在同一个S1-U/N3口地址上激活2个用户。查看这2个用户信息，显示Local SigAddr和Local DataAddr都相同，即可确定2个用户在一个 Instance 上。
  >
  > 测试限制条件：只有当前2个测试用户发出流量；访问业务的带宽大于所配置PIR。
  >
  > 用户流量可以匹配到相同的带宽控制规则（本例中2个用户报文的TOS值都等于0）。
4. 查看镜像接口的抓包信息，利用Ethereal工具的统计功能计算S1-U/N3/SGi/N6/S5/S8接口侧用户出接口报文速率。
  > **说明**
  > 打开镜像接口侧的抓包工具，准备抓取测试终端的出入报文。通过设置不同的过滤条件（上行报文的目的IP为ServerIP，下行报文的目的IP为UE终端IP）可以同时观察上下行流量的带宽。在Server上抓包可以观察上行流量，在UE终端抓包可以观察下行流量。测试带宽管理功能需要访问业务的带宽大于所配置PIR。
  >
  > 发包速率及误差范围在不同的场景下，应与不同的标准比较（CIR或者是PIR）。如果bwm控制器类型为CAR，则被控制流量的速率的大小，需要结合CAR的三种颜色对应的动作来判断。
    - 调测时配置：绿色 “PASS” ；黄色 “PASS” ；红色 “DISCARD” 。
      带宽控制的结果应为：流量的最大速率不超过峰值速率（PIR）。
      测试时需要使发包速率大于PIR，同样误差范围与PIR比较。
    - 调测时配置：绿色 “PASS” ；黄色 “DISCARD” ；红色 “DISCARD” 。
      带宽控制的结果应为：流量的最大速率不超过保证速率（CIR）。
      测试时需要使发包速率大于CIR，同样误差范围与CIR比较。
    - 调测时配置：绿色 “DISCARD” ；黄色 “DISCARD” ；红色 “DISCARD” 。
      带宽控制的结果应为：流量全部被丢弃。
      测试时发包速率没有限制，流量全部丢弃。
  以下行流量为例，通过工具观察由 UDG 出来的流量速率是否符合配置的带宽控制的速率，如 [图2](#ZH-CN_OPI_0184034130__fig1) 所示。
  **图2** 下行报文S1-U/N3/SGi/N6/S5/S8接口侧抓包信息

  <br>

  ![](调测基于用户_用户组_整机三级控制中的整机的带宽控制_84034130.assets/fig_1_4d501.png "点击放大")

  <br>
    a. 在抓包工具上选择 “ Statistics > IO Graphs ” ，查看S1-U/N3/SGi/N6/S5/S8接口侧包速率曲线图。
    b. 在“IO Graphs”窗口中设置曲线参数，如 [图3](#ZH-CN_OPI_0184034130__fig3) 所示。
      在“X Axis”栏选择时间单位为1秒，“Y Axis”栏选择单位为Bytes/Tick。
      在“Graphs”栏设置过滤条件：

      - “Graph 1”用于过滤两个用户（属于一个用户组）S1-U/N3口下行报文总和的流量，过滤条件设为：gtp，包速率曲线显示为黑色；
          - “Graph 2”用于过滤两个用户（属于一个用户组）SGi/N6/S5/S8口下行报文总流量，过滤条件设为：!gtp，包速率曲线显示为红色；
          - “Graph 3”用于过滤一个用户S1-U/N3接口侧下行报文流量，过滤条件设为：gtp && ip.addr == 10.3.5.5，包速率曲线显示为绿色；
          - “Graph 4”用于过滤另一个用户S1-U/N3接口侧下行报文流量，过滤条件设为：gtp && ip.addr == 10.3.5.6，包速率曲线显示为蓝色。
      **图3** 下行报文S1-U/N3/SGi/N6/S5/S8接口侧包速率曲线图

      <br>

      ![](调测基于用户_用户组_整机三级控制中的整机的带宽控制_84034130.assets/fig_1_4d302.png "点击放大")

      <br>
          - 如果整机（ Instance ）下的用户的总流量受控，则带宽管理功能正常，调测结束。
          - 如果整机（ Instance ）下的用户的总流量不受控，则带宽管理功能不正常，请执行 [6](#ZH-CN_OPI_0184034130__step4) 。
5. 查询全局带宽控制规则配置。
    a. 执行 [**LST BWMRULEGLOBAL**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理规则/查询全局带宽管理规则（LST BWMRULEGLOBAL）_82837485.md) 命令，查询全局用户组的相关信息。
      ```
      LST BWMRULEGLOBAL:BWMRULENAME="br_global_test";
      ```
      ```
      ......
      -------------------------
                       带宽管理规则名称  =  br_global_test
                             用户组名称  =  ug_test
                       带宽管理规则类型  =  全局级别特定
                     带宽管理规则优先级  =  1
                       带宽控制业务名称  =  bs_group_test
               上行带宽管理控制器名称一  =  bc-global-up
               下行带宽管理控制器名称一  =  bc-global-down
      ```
      ```
                                       ......
      ```
          - 如果全局带宽规则下的“带宽控制业务名称”、“上行带宽管理控制器名称一”和“下行带宽管理控制器名称一”与规划值一致，请执行[6.b](#ZH-CN_OPI_0184034130__substepe)。
          - 如果全局带宽规则下的“带宽控制业务名称”、“上行带宽管理控制器名称一”和“下行带宽管理控制器名称一”与规划值不一致，重新执行[4](#ZH-CN_OPI_0184034130__step3)，注意要重新激活用户。
    b. 执 [**LST BWMSERVICE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理业务/查询带宽管理业务（LST BWMSERVICE）_82837476.md) 、 [**LST BWMCONTROLLER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理控制器/查询带宽管理控制器（LST BWMCONTROLLER）_82837463.md) 命令，查询全局带宽规则所对应BWM Service和BWM Controller的信息。
          - 如果BWM Service的“TOS”、下行和上行的BWM Controller参数值与规划值一致，请执行[6](#ZH-CN_OPI_0184034130__cmd159451420459)。
          - 如果BWM Service的“TOS”、下行和上行的BWM Controller参数值与规划值不一致，请参考[基于用户/用户组/整机三级的带宽控制](../激活基于业务感知的带宽控制/基于用户_用户组_整机三级的带宽控制_84034124.md)重新配置，重新执行[4](#ZH-CN_OPI_0184034130__step3)，注意要重新激活用户。
6. 收集信息并寻求技术支持。
    a. 在镜像接口或服务器上开启抓包工具，执行 [5](#ZH-CN_OPI_0184034130__step3) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 联系华为技术支持解决。
