# 修改重定向Server的负荷分担方式

- [必备事项](#ZH-CN_OPI_0166620117__1.3.2)

## [操作场景](#ZH-CN_OPI_0166620117)

重定向Server的 负荷分担 方式需要修改。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0166620117)

前提条件

- 请仔细阅读[GWFD-110281 用户Portal](../../GWFD-110281 用户Portal_66620113.md)。
- 完成[配置用户Portal](../配置用户Portal_66620114.md)。

数据

该操作无需准备数据。

## [操作步骤](#ZH-CN_OPI_0166620117)

1. 配置整机的针对IPFARM配置的重定向Server的负荷分担方式。
  [**SET IPFARMGLOBAL**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/IP Farm 管理/IP Farm全局参数/设置IPFarm全局参数（SET IPFARMGLOBAL）_82837062.md)
  > **说明**
  > 负荷分担方式有下列三种：
  >
  > - ROUND_ROBIN：轮询方式。
  > - LEAST_RECENTLY_USED：选择最久未被使用的Server。
  > - LEAST_LOAD：选择最小负荷Server方式。

## [任务示例](#ZH-CN_OPI_0166620117)

任务描述

负荷分担方式由LEAST_LOAD改为LEAST_RECENTLY_USED。

脚本

//配置整机的针对IPFARM配置的重定向Server的负荷分担方式。

```
SET IPFARMGLOBAL:SERVERTYPE=REDIRECT,LBMETHOD=LEAST_RECENTLY_USED;
```
