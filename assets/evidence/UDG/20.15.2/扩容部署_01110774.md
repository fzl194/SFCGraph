    - 配置举例，每个已在Full Mesh组网里的所有SMF（SMF1和SMF2）都要做如下配置。
      ```
      //配置用户面节点实例。

      //配置Profile实例。
      ADD PNFPROFILE:NFINSTANCEID="UPF_Instance_4", NFTYPE=NfUPF, NFSTATUS=Registered, IPADDRESSTYPE=IPTypeV4, IPV4ADDRESS1="192.168.126.11", PORT=0;
      //配置用户面节点支持的DNN信息。
      ADD PNFNS: INDEX=3, NFINSTANCEID="UPF_Instance_4", SST=1, SD="000001";
      ADD PNFDNN:INDEX=3, NFINSTANCEID="UPF_Instance_4", DNN="huawei.com", PNFNSINDEX=3;
      //配置用户面节点支持的为SMF提供服务区域的信息。
      ADD PNFSMFSERAREA:NFINSTANCEID="UPF_Instance_4", SMFSERVINGAREA="UPAREA1";

      //配置用户面节点的位置特征和分流能力。

      ADD UPNODE:NFINSTANCENAME="UPF_Instance_4", LOCATION=Central, UPFUNCTION=None, LOCK=FALSE;

      //配置用户面节点的位置区信息，将位置区与TAI绑定。

      ADD UPAREA:AREANAME="UPAREA1", AREATYPE=N2TAI;
      ADD UPAREABINDN2TAI:AREANAME="UPAREA1", N2BEGINTAI="12303260101", N2ENDTAI="12303260102";

      //（可选）如果新建的UPF与已建立的UPF规划了相同的地址段，则SMF需要为新建的UPF绑定不同的IP Domain。

      ADD UPIPDOMAIN: UPNODE="IPv4+192.168.0.20", IPDOMAIN="Ipdomain-a", DNNSWITCH=DISABLE;
      ```
3. 已在Full Mesh组网里的所有UPF（UPF1和UPF2）的N4接口不需要再做数据配置。如果开启“基于SMF分配地址”和“基于SMF+APN/DNN分配地址”功能，需要已在Full Mesh组网里的所有UPF（UPF1和UPF2）对扩容的SMF（SMF3）分配地址，参见UDG产品手册的“基于SMF分配地址”和“基于SMF+APN/DNN分配地址”章节。
