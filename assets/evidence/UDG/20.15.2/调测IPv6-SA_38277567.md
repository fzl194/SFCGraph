# 调测IPv6 SA

- [操作场景](#ZH-CN_OPI_0138277567__1.3.1)
- [必备事项](#ZH-CN_OPI_0138277567__1.3.2)
- [操作步骤](#ZH-CN_OPI_0138277567__1.3.3)

## [操作场景](#ZH-CN_OPI_0138277567)

当运营商部署IPv6 SA时，需对 UDG 的IPv6 SA功能进行调测，确保本功能可以正常使用。该调测以七层解析业务阻塞功能应用场景为例进行调测。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0138277567)

前提条件

- 请仔细阅读 [GWFD-020352 IPv6 SA](../GWFD-020352 IPv6 SA_80136510.md) 。
- 完成[激活IPv6 SA](激活IPv6 SA_80136513.md)。
- 打开接入侧/DN侧镜像接口上的第三方抓包工具，准备抓取测试终端的出入报文。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN名称（APN） | apn-test | 已配置数据中获取 | 取自PCF下发的APN实例名。 |
| Internet公共知名网站 | - | www.huawei.com | 固定取值 | 用于判断<br>测试终端<br>能否正常使用网络基本业务。 |
| 阻塞触发地址 | URL | www.example.com*/* | 已配置数据中获取 | 取自<br>[激活IPv6 SA](激活IPv6 SA_80136513.md)<br>中配置的七层解析数据中的<br>“URL”<br>。 |

工具

- 测试终端
- 第三方抓包工具

## [操作步骤](#ZH-CN_OPI_0138277567)

1. 打开接入侧/DN侧镜像接口上的第三方抓包工具，准备抓取测试终端的出入报文。
2. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令，查询IPv6 SA的特性开关是否打开。
  [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) : LICITEM="LKV3G5SAV601";
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 3](#ZH-CN_OPI_0138277567__step2)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)命令打开本特性对应的License配置开关。
3. 测试终端 使用“apn-test”APN接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 4](#ZH-CN_OPI_0138277567__step3)。
    - 如果测试终端无法接入网络，请调测UDG的接入功能。
4. 测试终端 访问 **www.huawei.com** 网页。
    - 如果测试终端可以正常访问网页，请执行[步骤 5](#ZH-CN_OPI_0138277567__step4)。
    - 如果测试终端无法正常访问网页，请调测UDG的Web浏览功能。
5. 测试终端 访问 **www.example.com** 网页，查看镜像接口的抓包信息。
  > **说明**
  > 对于使用HTTP协议访问URL以 **www.example.com** 开头的任何网页，都应该触发七层解析业务阻塞业务。即在 UDG 的上行SGi/N6侧出接口抓取不到URL为 **www.example.com** 的报文。
  >
  > 在TCP的三次握手报文之后，可以在S1-U/N3接口侧观察到HTTP GET报文请求访问 **www.example.com/xxxx** 网址，但是在SGi/N6接口侧并没有任何由 UDG 转发的该HTTP GET报文。
    - 如果测试终端无法访问以**www.example.com**开头的任何网页，七层解析业务阻塞功能正常，调测结束。
    - 如果测试终端可以访问该网页，请执行[6](#ZH-CN_OPI_0138277567__cmd804232969163836)。
6. 检查七层解析业务阻塞相关配置是否正确。
    a. 执行 [**LST RULEBINDING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_86526204.md) 命令，查询UserProfile绑定的Rule是否正确。
      ```
      LST RULEBINDING:USERPROFILENAME="up_test";
      ```
      ```
      ......
      -------------------------
      用户模板名称  =  up_test
          规则名称  =  rule_test
            优先级  =  1
          策略类型  =  PCC
      (结果个数 = 1)
      ---    END
      ```
          - 如果UserProfile绑定的“规则名称”和“优先级”与规划值一致，请执行[步骤 6.b](#ZH-CN_OPI_0138277567__substepc)。
          - 如果UserProfile绑定的“规则名称”和“优先级”与规划值不一致，请参考[激活IPv6 SA](激活IPv6 SA_80136513.md)重新配置。
    b. 执行 [**LST RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，查询Rule的配置信息是否正确。
      ```
      LST RULE:RULENAME="rule_test";
      ```
      ```
      ......
      -------------------------
                                   规则名称  =  rule_test
                                   策略类型  =  PCC
                             黑名单规则标识  =  普通规则
                               流过滤器名称  =  flow-filter1
                                 时间段名称  =  NULL
                                 全局优先级  =  1
                              PCC策略组名称  =  pg_test
                                           ......
      (结果个数 = 1)
      ---    END
      ```
          - 如果Rule绑定的“流过滤器名称”、“PCC策略组名称”和“全局优先级”与规划值一致，请执行[步骤 6.c](#ZH-CN_OPI_0138277567__substepe)。
          - 如果Rule绑定的“流过滤器名称”、“PCC策略组名称”和“全局优先级”与规划值不一致，请参考[激活IPv6 SA](激活IPv6 SA_80136513.md)重新配置。
    c. 执行 [**LST FLTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的过滤器绑定/查询流过滤器的过滤器绑定关系（LST FLTBINDFLOWF）_82837368.md) 、 [**LST PROTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的协议绑定/查询流过滤器协议绑定关系（LST PROTBINDFLOWF）_82837373.md) 、 [**LST FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/查询过滤器（LST FILTER）_82837353.md) 和 [**LST L7FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/七层规则管理/七层过滤器/查询七层过滤器（LST L7FILTER）_86526660.md) 命令，查询Rule绑定的三四层和七层过滤信息是否正确。
      ```
      LST FLTBINDFLOWF:FLOWFILTERNAME="flow-filter1";
      ```
      ```
      ......
      -------------------------

        过滤器名称  =  filter_test
      (结果个数 = 1)
      ---    END
      ```
      ```
      LST PROTBINDFLOWF:FLOWFILTERNAME="flow-filter1";
      ```
      ```
      ......
      -------------------------
         流过滤器名称  =  flow-filter1
             协议名称  =  http
       七层过滤器名字  =  l7_test
      (结果个数 = 1)
      ---    END
      ```
      ```
      LST FILTER:FILTERNAME="filter_test";
      ```
      ```
      ......
      --------------------------------------------------------
                  过滤器名字  =  filter_test
      三四层IPv4协议输入类型  =  NULL
      三四层IPv6协议输入类型  =  字符串类型

      ```
      ```
      LST L7FILTER:L7FILTERNAME="l7_test";
      ```
      ```
      ......
      -------------------------
           七层过滤器名称  =  l7_test
         子七层过滤器名称  =  sl7-test
                      URL  =  www.example.com*/8
               客户端类型  =  NULL

      Referer关联计费开关  =  不使能

      (结果个数 = 1)
      ---    END
      ```
          - 如果Rule绑定的三四层和七层过滤信息与规划值一致，请执行[步骤 6.d](#ZH-CN_OPI_0138277567__substepf)。
          - 如果Rule绑定的三四层和七层过滤信息与规划值不一致，请参考[激活IPv6 SA](激活IPv6 SA_80136513.md)重新配置。
    d. 执行 [**LST SRVPBINDPCCPG**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组业务属性绑定/查询PCC组业务属性绑定（LST SRVPBINDPCCPG）_82837614.md) 、 [**LST PCCACTIONPROP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC动作属性/查询PCC动作属性（LST PCCACTIONPROP）_82837604.md) 命令，查询Rule的PCC动作策略组信息是否正确。
      ```
      LST SRVPBINDPCCPG:PCCPOLICYGRPNM="pg_test";
      ```
      ```
      ......
      -------------------------
        PCC策略组名称  =  pg_test
                      ......
      PCC动作属性名称  =  ap_test
                      ......
      (结果个数 = 1)
      ---    END
      ```
      ```
      LST PCCACTIONPROP:PCCACTPROPNAME="ap_test";
      ```
      ```
      ......
      -------------------------
                PCC动作属性名称  =  ap_test
      上行发起内容感知QoS的名称  =  NULL
             上行发起重定向名称  =  NULL
               上行发起上行门控  =  Discard
               上行发起下行门控  =  Discard
      下行发起内容感知QoS的名称  =  NULL
             下行发起重定向名称  =  NULL
               下行发起上行门控  =  Pass
               下行发起下行门控  =  Pass
      (结果个数 = 1)
      ---    END
      ```
          - 如果rule的PCC动作策略组信息与规划值一致，请执行[步骤 7](#ZH-CN_OPI_0138277567__step7)。
          - 如果rule的PCC动作策略组信息与规划值不一致，请参考[激活IPv6 SA](激活IPv6 SA_80136513.md)重新配置。
7. 执行 [**DSP SIGNATUREDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/识别特征库/查询协议特征库文件加载状态（DSP SIGNATUREDB）_82837717.md) 命令，查看 “Signature Database Load Status” 是否为 “Load Finish” 以及查看 “Signature Database Version” 是否正确。
    - 如果加载的版本正确，请执行[步骤 8](#ZH-CN_OPI_0138277567__step8)。
    - 如果尚未加载特征库版本或加载的版本不正确，请执行[**LOD SIGNATUREDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/识别特征库/加载协议特征库文件（LOD SIGNATUREDB）_82837716.md)加载或升级特征库版本。
8. 收集信息并寻求技术支持。
    a. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    b. 收集并保存上述所有查询信息。
    c. 收集归纳所有信息并联系华为技术支持解决。
