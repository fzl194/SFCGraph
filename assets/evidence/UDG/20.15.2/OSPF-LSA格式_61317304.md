# OSPF LSA格式

- [LSA头部信息](#ZH-CN_CONCEPT_0161317304__1.4.3.1)
- [Router-LSA](#ZH-CN_CONCEPT_0161317304__1.4.4.1)
- [Network-LSA](#ZH-CN_CONCEPT_0161317304__1.4.5.1)
- [Summary-LSA](#ZH-CN_CONCEPT_0161317304__1.4.6.1)
- [AS-External-LSA](#ZH-CN_CONCEPT_0161317304__1.4.7.1)

在AS内的每台 UDG ，根据设备的分类产生一种或多种LSA。LSA的集合形成了LSDB（Link-state Database）。OSPF中对路由信息的描述都是封装在LSA中发布出去的。常用的LSA包括：

- [Router-LSA](#ZH-CN_CONCEPT_0161317304__section_02) （Type1）
- [Network-LSA](#ZH-CN_CONCEPT_0161317304__section_03) （Type2）
- [Summary-LSA](#ZH-CN_CONCEPT_0161317304__section_04) （包括Network-summary-LSA（Type3）和ASBR-summary-LSA（Type4））
- [AS-External-LSA](#ZH-CN_CONCEPT_0161317304__section_05) （Type5）

#### [LSA头部信息](#ZH-CN_CONCEPT_0161317304)

所有的LSA都有相同的报文头，其格式如 [图1](#ZH-CN_CONCEPT_0161317304__f2a23ec7f5f43472f9906b1de93e485c5) 所示。

**图1** LSA的头格式

<br>

![](OSPF LSA格式_61317304.assets/zh-cn_image_0161317244.png)

*表1 LSA的头格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| LS age | 16比特 | LSA产生后所经过的时间，单位是秒。无论LSA是在链路上传输，还是保存在LSDB中，其值都会在不停的增长。 |
| Options | 8比特 | 可选项：<br>- E：允许泛洪AS-External-LSA。<br>- MC：转发IP组播报文。<br>- N/P：处理Type7 LSA。<br>- DC：处理按需链路。 |
| LS type | 8比特 | LSA的类型：<br>- Type1：Router-LSA。<br>- Type2：Network-LSA。<br>- Type3：Network-summary-LSA。<br>- Type4：ASBR-summary-LSA。<br>- Type5：AS-External-LSA。<br>- Type7：NSSA-LSA。 |
| Link State ID | 32比特 | 与LS Type一起描述路由域中唯一一个LSA。 |
| Advertising Router | 32比特 | 产生此LSA的<br>UDG<br>的Router ID。 |
| LS sequence number | 32比特 | LSA的序列号。其他设备根据这个值可以判断哪个LSA是最新的。 |
| LS checksum | 16比特 | 除了LS age外其它各域的校验和。 |
| length | 16比特 | LSA的总长度，包括LSA Header，以字节为单位。 |

#### [Router-LSA](#ZH-CN_CONCEPT_0161317304)

Router-LSA（Type1）：每个 UDG 都会产生，描述了 UDG 的链路状态和花费，在所属的区域内传播。Router-LSA格式如 [图2](#ZH-CN_CONCEPT_0161317304__faf71a006513c409f889d21b9d9b45d96) 所示。

**图2** Router-LSA格式

<br>

![](OSPF LSA格式_61317304.assets/zh-cn_image_0161317307.png)

*表2 Router-LSA格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Link State ID | 32比特 | 生成LSA的Router ID。 |
| V（Virtual Link） | 1比特 | 如果产生此LSA的<br>UDG<br>是虚连接的端点，则置为1，否则置为0。 |
| E（External） | 1比特 | 如果产生此LSA的<br>UDG<br>是ASBR，则置为1，否则置为0。 |
| B（Border） | 1比特 | 如果产生此LSA的<br>UDG<br>是ABR，则置为1，否则置为0。 |
| # links | 16比特 | LSA中所描述的链路信息的数量，包括<br>UDG<br>上处于某区域中的所有链路和接口。 |
| Link ID | 32比特 | UDG<br>所接入的目标，其值取决于连接的类型：<br>- 1：Router ID。<br>- 2：DR的接口IP地址。<br>- 3：网段/子网号。<br>- 4：虚连接中对端的Router ID。 |
| Link Data | 32比特 | 连接数据，其值取决于连接的类型：<br>- unnumbered P2P：接口的索引值。<br>- stub网络：子网掩码。<br>- 其它连接：<br>UDG<br>接口的IP地址。 |
| Type | 8比特 | UDG<br>连接的基本描述：<br>- 1：点到点连接到另一台<br>UDG<br>。<br>- 2：连接到传输网络。<br>- 3：连接到stub网络。<br>- 4：虚拟链路。 |
| # TOS | 8比特 | TOS数量。 |
| metric | 16比特 | 链路的开销值。 |
| TOS | 8比特 | 服务类型。 |
| TOS metric | 16比特 | 和指定TOS值相关联的度量。 |

#### [Network-LSA](#ZH-CN_CONCEPT_0161317304)

Network-LSA（Type2）：由DR（Designated Router）产生，Network-LSA中记录了这一网络上所有 UDG 的Router ID，描述本网段的链路状态，在DR所属的区域内传播。Network-LSA格式如 [图3](#ZH-CN_CONCEPT_0161317304__fc5b26448d0e94f1795ed9de0e5f3293f) 所示。

**图3** Network-LSA格式

<br>

![](OSPF LSA格式_61317304.assets/zh-cn_image_0161317376.png)

*表3 Network-LSA格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Link State ID | 32比特 | DR的接口IP地址。 |
| Network Mask | 32比特 | 该广播网或NBMA网络地址的掩码。 |
| Attached Router | 32比特 | 连接在同一个网络上的所有<br>UDG<br>的Router ID，也包括DR的Router ID。 |

#### [Summary-LSA](#ZH-CN_CONCEPT_0161317304)

Network-summary-LSA（Type3）：描述区域内某个网段的路由，并通告给其他区域（除了Totally Stub和Totally NSSA区域）。

ASBR-summary-LSA（Type4）：描述本区域到其他区域中的ASBR的路由，通告给除ASBR所在区域的其他区域（除了Stub区域、Totally Stub、NSSA区域和Totally NSSA区域）。

Type3和Type4的LSA有相同的格式，它们都是由ABR产生。Summary-LSA格式如 [图4](#ZH-CN_CONCEPT_0161317304__f6f3cf4acd9f64eb8b1998406bd38a5c5) 所示。

**图4** Summary-LSA格式

<br>

![](OSPF LSA格式_61317304.assets/zh-cn_image_0161317374.png)

*表4 Network-summary-LSA格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Link State ID | 32比特 | 通告的网络地址。 |
| Network Mask | 32比特 | 该广播网或NBMA网络地址的掩码。 |
| metric | 24比特 | 到目的地址的路由开销。 |
| TOS | 8比特 | 服务类型。 |
| TOS metric | 24比特 | 和指定TOS值相关联的度量。 |

> **说明**
> 通告缺省路由时，Link State ID和Network Mask都设置为0.0.0.0。

*表5 ASBR-summary-LSA格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Link State ID | 32比特 | ASBR的Router ID。 |
| Network Mask | 32比特 | 无意义，设置为0.0.0.0。 |
| metric | 24比特 | 到目的地址的路由开销。 |
| TOS | 8比特 | 服务类型。 |
| TOS metric | 24比特 | 和指定TOS值相关联的度量。 |

#### [AS-External-LSA](#ZH-CN_CONCEPT_0161317304)

AS-External-LSA（Type5）：由ASBR产生，描述到AS外部的路由，通告到所有的区域（除了Stub区域、Totally Stub、NSSA区域和Totally NSSA区域）。AS-External-LSA格式如 [图5](#ZH-CN_CONCEPT_0161317304__f33bb83bd299743a293eb9e0e6be77f0e) 所示。

**图5** AS-External-LSA格式

<br>

![](OSPF LSA格式_61317304.assets/zh-cn_image_0161317306.png)

*表6 AS-External-LSA字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Link State ID | 32比特 | 通告的网络地址。 |
| Network Mask | 32比特 | 通告的目的地址的掩码。 |
| E | 1比特 | 外部度量值类型：<br>- 0：第一类外部路由。<br>- 1：第二类外部路由。 |
| metric | 24比特 | 到目的地址的路由开销。 |
| Forwarding Address | 32比特 | 到所通告的目的地址的报文将被转发到这个地址。 |
| External Route Tag | 32比特 | 添加到外部路由上的标记。OSPF本身并不使用这个字段，它可以用来对外部路由进行管理。 |
| TOS | 8比特 | 服务类型。 |
| TOS metric | 24比特 | TOS附加距离信息。 |

> **说明**
> Type5的LSA可以用来通告缺省路由，此时Link State ID和Network Mask都设置为0.0.0.0。
