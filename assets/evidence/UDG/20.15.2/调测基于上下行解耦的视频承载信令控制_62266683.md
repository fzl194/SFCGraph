# 调测基于上下行解耦的视频承载信令控制

- [操作场景](#ZH-CN_OPI_0000001262266683__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001262266683__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001262266683__1.3.3)

## [操作场景](#ZH-CN_OPI_0000001262266683)

当运营商需要部署基于上下行解耦的视频承载信令控制功能时，需对本功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0000001262266683)

前提条件

- 请仔细阅读[GWFD-110302 基于上下行解耦的视频承载信令控制](../GWFD-110302 基于上下行解耦的视频承载信令控制_14445050.md)。
- 完成[激活基于上下行解耦的视频承载信令控制](激活基于上下行解耦的视频承载信令控制_62146691.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户跟踪任务 | 用户IMSI号（IMSI） | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN名称（APN） | apn1 | 已配置数据中获取 | 已通过<br>[**ADD APN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/添加APN配置（ADD APN）_82837014.md)<br>命令配置，可以使用<br>[**LST APN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/查询APN配置（LST APN）_82837017.md)<br>命令进行查询 |
| 业务触发协议 | 协议名称（PROTOCOLNAME） | example_video | 已配置数据中获取 | 取自<br>[激活基于上下行解耦的视频承载信令控制](激活基于上下行解耦的视频承载信令控制_62146691.md)<br>中配置的七层过滤规则。 |

工具

- 测试终端
- OM Portal

## [操作步骤](#ZH-CN_OPI_0000001262266683)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令查询UDG上本功能对应的License配置开关是否打开。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5SCUD01";
  ```
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0000001262266683__step19171111585013)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)命令打开本特性对应的License配置开关。
2. 在OM Portal上 **[创建UDG用户跟踪任务](../../../../../网络运维/日常维护/UDG基础运维操作/创建消息跟踪/创建用户跟踪/创建UDG用户跟踪任务_88792206.md)** 。
3. 测试终端使用 “apn1” APN接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 4](#ZH-CN_OPI_0000001262266683__step1712315319104)。
    - 如果测试终端无法接入网络，调测UDG的接入功能。
4. 测试终端浏览Web业务，访问 **www.huawei.com** 网页。
    - 如果测试终端可以正常访问网页，请执行 [步骤 5](#ZH-CN_OPI_0000001262266683__step135281929220) 。
    - 如果测试终端无法正常访问网页，请调测UDG的Web浏览功能。
5. 查看数据规划中相关规则的数据配置，根据相关规则，使用测试终端访问能触发基于下行链路建立专有承载/专有QoS Flow的业务。
6. 查看PGW-U/UPF是否立即向PGW-C/SMF发送PFCP Session Report Request消息，携带的Usage Report信元组中包含QoS类型的URR ID、Usage Report Trigger为start。
    - 如果PGW-U/UPF上报信元正确，则基于上下行解耦的视频承载信令控制功能生效，继续执行[步骤 7](#ZH-CN_OPI_0000001262266683__step2054514555016)。
    - 如果PGW-U/UPF没有上报或上报信元不全，则基于上下行解耦的视频承载信令控制功能没有正常生效，请执行[步骤 9](#ZH-CN_OPI_0000001262266683__step13676132835614)。
7. 查看PGW-C/SMF向PGW-U/UPF发送的PFCP Session Report Response消息，查询Create PDR信元组中携带的URR ID是否与PFCP Session Report Request消息携带的URR ID一致。
    - 如果一致，请执行 [10](#ZH-CN_OPI_0000001262266683__step123994101910) 。
    - 如果不一致，请执行 [步骤 8](#ZH-CN_OPI_0000001262266683__step6) 。
8. 执行 [**DSP SESSIONINFO**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，通过 测试终端 的IMSI查看 测试终端 建立专有承载/专有QoS Flow和使用的APN。
  ```
  DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
  ```
    - 如果测试终端使用的“APN name”与规划值一致，请执行[步骤 9](#ZH-CN_OPI_0000001262266683__step13676132835614)。
    - 如果测试终端使用的“APN name”与规划值不一致，请返回[步骤 3](#ZH-CN_OPI_0000001262266683__step3)使用规划的APN重新接入网络。
9. 检查APN下内容感知相关配置是否正确：
    a. 执行 [**LST RULEBINDING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_86526204.md) 命令，查询UserProfile绑定的Rule是否正确。
      ```
      %%LST RULEBINDING: USERPROFILENAME="up_test";%% 
      RETCODE = 0  操作成功  
      用户模板与规则绑定信息 
      ---------------------- 
      用户模板名称  =  up_test     
          规则名称  =  rule_test     
          策略类型  =  PCC 
      (结果个数 = 1)  
      ---    END 
      ```
          - 如果UserProfile绑定的“规则名称”与规划值一致，请执行 [步骤 9.b](#ZH-CN_OPI_0000001262266683__substep96381533142110) 。
          - 如果UserProfile绑定的“规则名称”与规划值不一致，请参考 [激活基于上下行解耦的视频承载信令控制](激活基于上下行解耦的视频承载信令控制_62146691.md) 重新配置。
    b. 执行 [**LST RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，查询Rule的配置信息是否正确。
      ```
      %%LST RULE: RULENAME="rule_test";%%
      RETCODE = 0 操作成功。  
      规则信息 
      --------
                             规则名称 = rule_test
                             策略类型 = PCC
                                    ......
                        流过滤器名称  =  flowfilter_test1
                                    ......
                       PCC策略组名称  =  ppg1 
                                    ......
      (结果个数 = 1) 
      ---    END 
      ```
          - 如果Rule绑定的“流过滤器名称”和“ PCC策略组名称” 与规划值一致，请执行 [步骤 9.c](#ZH-CN_OPI_0000001262266683__substep0785115011277) 。
          - 如果Rule绑定的“流过滤器名称”和“ PCC策略组名称” 与规划值不一致，请参考 [激活基于上下行解耦的视频承载信令控制](激活基于上下行解耦的视频承载信令控制_62146691.md) 重新配置。
    c. 执行 [**LST PCCPOLICYGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/查询PCC策略组（LST PCCPOLICYGRP）_82837609.md) 、 [**LST URR**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则/查询URR（LST URR）_82837632.md) 和 [**LST QOSPROP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/用户QOS控制/业务QOS控制/业务质量属性/查询QoS属性（LST QOSPROP）_82837652.md) 命令，查询PCC策略组信息是否正确。
      ```
      %%LST PCCPOLICYGRP: PCCPOLICYGRPNM="ppg1";%%
      RETCODE = 0  操作成功。  
      PCC策略组信息 
      -------------
               PCC策略组名称  =  ppg1
                            ......    
                 Qos属性名称  =  qos-property1
                            ......   
      (结果个数 = 1) 
      ---    END
      ```
      ```
      %%LST URR: URRNAME="URR_1";%%
      RETCODE = 0  操作成功。  
      使用量上报规则信息： 
      --------------------
         使用量上报规则名称  =  urr_1
                    URR标识  =  1
             使用量上报模式  =  QOS 
                           ......
      (结果个数 = 1) 
      ---    END 
      ```
      ```
      %%LST QOSPROP: QOSPROPNAME="qos-property1";%% 
      RETCODE = 0  操作成功  
      QoS属性信息 
      -----------              
                 Qos属性名称  =  qos-property1 
                            ......        
       QoS使用量上报规则名称  =  urr_1           
              上下行解耦开关  =  使能（开启） 
                            ......          
      (结果个数 = 1)  
      ---    END 
      ```
          - 如果Rule的PCC策略组信息与规划值一致，请执行 [步骤 9.d](#ZH-CN_OPI_0000001262266683__substep64174675011) 。
          - 如果Rule的PCC策略组信息与规划值不一致，请参考 [激活基于上下行解耦的视频承载信令控制](激活基于上下行解耦的视频承载信令控制_62146691.md) 重新配置。
    d. 执行 [**LST FLTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的过滤器绑定/查询流过滤器的过滤器绑定关系（LST FLTBINDFLOWF）_82837368.md) 、 [**LST FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/查询过滤器（LST FILTER）_82837353.md) 、 [**LST PROTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的协议绑定/查询流过滤器协议绑定关系（LST PROTBINDFLOWF）_82837373.md) 命令，查询rule绑定的三四层和七层过滤信息是否正确。
      ```
      %%LST FLOWFILTER:FLOWFILTERNAME="flowfilter_test1";%% 
      RETCODE = 0  操作成功  
      流过滤器信息
      -------------------------
                流过滤器名称  =  flowfilter_test1
                            ...... 
      (结果个数 = 1) 
      ---    END 
      ```
      ```
      %%LST FILTER:FILTERNAME="filter_test";%%
      RETCODE = 0  操作成功。  
      过滤器信息 
      ----------
                   过滤器名字  =  filter_test
       三四层IPv4协议输入类型  =  字符串类型
       三四层IPv6协议输入类型  =  NULL
           三四层协议数字类型  =  0
               三四层协议类型  =  ANY
                         ......
           手机IP地址配置模式  =  NULL 
                         ......
      (结果个数 = 1) 
      ---    END 
      ```
      ```
      %%LST PROTBINDFLOWF:FLOWFILTERNAME="flowfilter_test1";%%
      RETCODE = 0  操作成功。  
      流过滤器协议绑定信息 
      --------------------
         流过滤器名称  =  flowfilter_test1
             协议名称  =  example_video
       (结果个数 = 1) 
      ---    END
      ```
          - 如果Rule绑定的三四层过滤信息、七层过滤信息与规划值一致，请执行 [10](#ZH-CN_OPI_0000001262266683__step123994101910) 。
          - 如果Rule绑定的三四层过滤信息、七层过滤信息与规划值不一致，请参考 [激活基于上下行解耦的视频承载信令控制](激活基于上下行解耦的视频承载信令控制_62146691.md) 重新配置。然后重新执行 [步骤 5](#ZH-CN_OPI_0000001262266683__step135281929220) 。
10. 检查用户跟踪的上下行数据报文。查看基于IMSI的用户跟踪报文，查询用户的上行报文和下行报文中的GTP头中的TEIDU是否一致。
    - 如果不一致，基于上下行解耦的视频承载信令控制功能正常，调测结束。
    - 如果一致，请执行 [11](#ZH-CN_OPI_0000001262266683__step8) 。
11. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务，执行 [3](#ZH-CN_OPI_0000001262266683__step3) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
