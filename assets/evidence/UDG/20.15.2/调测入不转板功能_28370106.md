# 调测入不转板功能

- [操作场景](#ZH-CN_OPI_0000001428370106__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001428370106__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001428370106__1.3.3)

## [操作场景](#ZH-CN_OPI_0000001428370106)

激活入不转板功能后，请参考本页面进行调测，确保入不转板功能正常。

> **说明**
> ISU POD部署模式和APU POD部署模式都支持入不转板功能。为简化描述，如下以ISU POD描述为例。

## [必备事项](#ZH-CN_OPI_0000001428370106)

前提条件

已完成 [激活入不转板功能](激活入不转板功能_28210142.md) 。

数据

无。

## [操作步骤](#ZH-CN_OPI_0000001428370106)

1. 执行 [**LST LOGICINF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/逻辑接口管理/逻辑接口配置/接口/查询逻辑接口（LST LOGICINF）_86526672.md) ，检查逻辑接口数据是否与规划一致。
    - 一致，请执行[步骤 2](#ZH-CN_OPI_0000001428370106__step56681452374)。
    - 不一致，请删除错误的逻辑接口后重新配置，然后跳转到[步骤 2](#ZH-CN_OPI_0000001428370106__step56681452374)。
2. 执行 [**DSP IPSUIT**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/逻辑接口管理/IPSUIT 信息/显示IPsuit信息（DSP IPSUIT）_09988681.md) ，检查IPSUIT绑定的ISU POD及逻辑接口是否正确。
  IPSuit Name取值需与xxif1/Y/0中Y取值一致，xxif1/Y/0接口IP需与规划一致。记录Binding PodName取值，该参数说明该IPsuit和哪个ISU POD进行绑定。
  ```
  %%DSP IPSUIT:;%%
  RETCODE = 0  Operation succeeded

  IPSuit Info
  -----------
  Result  =  
                                IPSuit Name  =  
  1

                              Owner PodName  =  isu-pod-0
                            Binding PodName  =  
  isu-pod-0

               n3if1/
  1
  /0 Interface Info : 
                               IPV4 Address  =  10.182.21.122
                               IPV6 Address  =  fc00:21:183:3:0:0:0:112 
                               VPN Instance  =  VPN_Access
  ...
  ```
    - 正确，请执行[步骤 3](#ZH-CN_OPI_0000001428370106__step1111814311612)。
    - 不正确，请执行[步骤 6](#ZH-CN_OPI_0000001428370106__step134791061100)。
3. 获取ISU POD和RU ID的映射关系。
    a. 执行 [**LST VNFC**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/单体服务平台功能管理/操作维护/配置管理/VNFC信息/查询VNFC（LST VNFC）_59036046.md) ，获取含VNRS的管理代理标识（Management Agent ID）。
      ```
      LST VNFC:;
      ```
    b. 执行 [**DSP RU**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/单体服务公共功能管理/系统管理/资源管理/RU管理/显示资源单元信息（DSP RU）_59103857.md) ，获取ISU POD和RU ID的映射关系，然后执行 [步骤 4](#ZH-CN_OPI_0000001428370106__step15281353671) 。
      //此处以含VNRS的管理代理标识是 “VNRS_VNFC_999” 为例，实际请以 [步骤 3.a](#ZH-CN_OPI_0000001428370106__substep14411198241) 查询的为准。 “父资源名称（Parent Resource Name）” 即ISU POD的名称。
      ```
      DSP RU: SERVICEINSTANCE="VNRS_VNFC_999";
      ```
      ```
      %%DSP RU: SERVICEINSTANCE="VNRS_VNFC_999";%%
      RETCODE = 0  Operation succeeded

      The result is as follows:
      -------------------------
      RU ID  RU Name              RU Type         ...  Running State  Registration State  ...  Parent Resource Name ... 

      ...

      64
           VNRS_IP_RU_0064      VNRS_IP_RU      ...  normal         registered          ...  
      isu-pod-0
                   ...                    
      65     VNRS_IP_RU_0065      VNRS_IP_RU      ...  normal         registered          ...  isu-pod-1             ...                    
      66     VNRS_IP_RU_0066      VNRS_IP_RU      ...  normal         registered          ...  isu-pod-2             ...                    
      ...
      (Number of results = 7)

      ---    END
      ```
      获取ISU POD和RU ID的映射关系后，请执行 [步骤 4](#ZH-CN_OPI_0000001428370106__step15281353671) 。
4. 执行 [**DSP ROUTE**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/IP服务/路由管理/路由基础/IPv4路由表/显示IPv4路由表（DSP ROUTE）_00441129.md) ，检查生成的IPv4路由信息是否正确。
  对于N3/S1-U/N9c/判断 “路由标签（Route Tag）” 取值与RU ID相同的路由，目的IP是否包含IPSuit（ISU POD和RU ID有映射关系的IPSuit）中的所有逻辑接口IP，且 “路由接口名字（Route Interface Name）” 取值为 “PAE Group” 。若满足上述条件，说明路由信息正确。
  IPv6路由信息的查询使用 [**DSP ROUTE6**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/IP服务/路由管理/路由基础/IPv6路由表/查询IPv6路由表（DSP ROUTE6）_49961994.md) 命令，判断方式同IPv4。
  ```
  %%DSP ROUTE: ADDRESSFAMILY=ipv4unicast;%%
  RETCODE = 0  Operation succeeded

  The result is as follows:
  -------------------------
  Route VPN Name            Route Prefix     Route Mask Length  Route Protocol Name  ...  Route Tag  Is Route Active  Route Interface Name          ...  

  ...
  VPN_Access                10.182.21.122    32                 wlr_sp               ...  
  64
           TRUE             PAE Group                     ...       
  ...
  (Number of results = 30)

  ---    END
  ```
    - 正确，请执行[步骤 5](#ZH-CN_OPI_0000001428370106__step133461021053)。
    - 不正确，请执行[步骤 6](#ZH-CN_OPI_0000001428370106__step134791061100)。
5. 联系对接交换机/路由设备工程师，检查其上的路由信息是否正确。
    - 正确，调测结束。
    - 不正确，请执行[步骤 6](#ZH-CN_OPI_0000001428370106__step134791061100)。
6. 收集信息并寻求技术支持。
    a. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    b. 收集并保存上述所有查询信息。
    c. 收集归纳所有信息并联系华为技术支持解决。
