# 实现原理

DSCP值：IPv4报文头中的ToS（Type of Service）域可以用来对用户报文进行简单分类，DSCP（Differentiated Services Code Point）是ToS域的低6位（0～5bit）。本功能中 UDG 将IM业务类型映射为DSCP值带至BSC/RNC侧，由无线侧做相关资源优化处理。

DSCP值定义如 [表1](#ZH-CN_TOPIC_0239436128__table23113814115) 所示。

*表1 DSCP与业务类型映射表*

| DSCP | 业务类型 |
| --- | --- |
| 12 | QQ |
| 14 | MSN |
| 18 | Fetion |
| 20 | WAP2.0 |
| 22 | Default |
| 26 | HTTP |
| 28 | WAP1.x |
| 30 | FTP |
| 34 | MMS |
| 36 | RTSP |
| 38 | PING |
| 46 | TCP建链报文 |

**图1** DSCP方式的IM类业务无线资源管控

![](实现原理_39436128.assets/zh-cn_image_0239436331.png)

1. PCRF根据用户信息下发IM类业务优化预定义规则。
2. PGW-C向UDG传递IM类业务优化预定义规则。
3. PGW-U（ UDG ）通过SA识别出IM业务类型，基于识别结果匹配PCRF下发的预定义规则，并基于预定义规则定义的Remark动作，修改IM业务IP报文的DSCP值。这个值标识此用户报文是IM类业务报文。
4. PGW-U（ UDG ）将修改了DSCP的数据报文传递给SGSN，SGSN将此报文透传给BSC/RNC。
5. BSC/RNC收到用户报文后，基于报文的DSCP值识别出IM类业务报文，并按照对应的调度处理方式调整无线资源分配。
