- [Intra-Area-Prefix-LSA](#ZH-CN_CONCEPT_0161317235__1.4.10.1)

在AS内的每台 UDG ，根据 UDG 的分类产生一种或多种LSA。LSA的集合形成了LSDB（Link-state Database）。OSPFv3中对路由信息的描述都是封装在LSA中发布出去的。常用的LSA包括：

- [Router-LSA（Type1）](#ZH-CN_CONCEPT_0161317235__section_02)
- [Network-LSA（Type2）](#ZH-CN_CONCEPT_0161317235__section_03)
- [Inter-Area-Prefix-LSA（Type3）](#ZH-CN_CONCEPT_0161317235__section_04)
- [Inter-Area-Router-LSA（Type4）](#ZH-CN_CONCEPT_0161317235__section_05)
- [AS-external-LSA（Type5）](#ZH-CN_CONCEPT_0161317235__section_06)
- [Link-LSA（Type8）](#ZH-CN_CONCEPT_0161317235__section_08)
- [Intra-Area-Prefix-LSA（Type9）](#ZH-CN_CONCEPT_0161317235__section_09)

#### [LSA头部信息](#ZH-CN_CONCEPT_0161317235)

所有的LSA都有相同的报文头，其格式如 [图1](#ZH-CN_CONCEPT_0161317235__f41a34c02fa2042b1a328ef48805e5377) 所示。

**图1** LSA的报文头格式

<br>

![](OSPFv3 LSA格式_61317235.assets/zh-cn_image_0161317226.png)

*表1 LSA的报文头格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| LS age | 16比特 | LSA产生后所经过的时间，单位是秒。无论LSA是在链路上传输，还是保存在LSDB中，其值都会在不停的增长。 |
| LS type | 16比特 | LSA的类型：<br>- Type1：Router-LSA。<br>- Type2：Network-LSA。<br>- Type3：Inter-Area-Prefix-LSA。<br>- Type4：Inter-Area-Router-LSA。<br>- Type5：AS-external-LSA。<br>- Type7：NSSA-LSA。<br>- Type8：Link-LSA。<br>- Type9：Intra-Area-Prefix-LSA。 |
| Link State ID | 32比特 | 与LS Type一起描述路由域中唯一一个LSA。 |
| Advertising Router | 32比特 | 产生此LSA的<br>UDG<br>的Router ID。 |
| LS sequence number | 32比特 | LSA的序列号。其他<br>UDG<br>根据这个值可以判断哪个LSA是最新的。 |
| LS checksum | 16比特 | 除了LS age外其它各域的校验和。 |
| Length | 16比特 | LSA的总长度，包括LSA Header，以字节为单位。 |

#### [Router-LSA](#ZH-CN_CONCEPT_0161317235)

Router-LSA（Type1）：每个 UDG 都会产生，描述了 UDG 的链路状态和花费，在所属的区域内传播。Router-LSA格式如 [图2](#ZH-CN_CONCEPT_0161317235__f4c6dd80835a647ec9a2897e5e032659b) 所示。

**图2** Router-LSA格式

<br>

![](OSPFv3 LSA格式_61317235.assets/zh-cn_image_0161317292.png)

*表2 Router-LSA格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Nt (NSSA translation) | 1比特 | 如果产生此LSA的<br>UDG<br>是NSSA边界路由器，则置为1，否则置为0。当该位设置为1时，<br>UDG<br>无条件将NSSA-LSA转换为AS-external-LSA。 |
| x | 1比特 | 废弃位。 |
| V（Virtual Link） | 1比特 | 如果产生此LSA的<br>UDG<br>是虚连接的端点，则置为1，否则置为0。 |
| E（External） | 1比特 | 如果产生此LSA的<br>UDG<br>是ASBR，则置为1，否则置为0。 |
| B（Border） | 1比特 | 如果产生此LSA的<br>UDG<br>是ABR，则置为1，否则置为0。 |
| Options | 24比特 | UDG<br>支持的可选能力。 |
| Type | 8比特 | UDG<br>连接的基本描述：<br>- 1：点到点连接到另一台<br>UDG<br>。<br>- 2：连接到传输网络。<br>- 3：预留位。<br>- 4：虚拟链路。 |
| metric | 16比特 | 链路的开销值。 |
| Interface ID | 32比特 | 为接口分配的ID |
| Neighbor Interface ID | 32比特 | 邻居设备的接口ID。 |
| Neighbor Router ID | 32比特 | 邻居设备的Router ID。 |

#### [Network-LSA](#ZH-CN_CONCEPT_0161317235)

Network-LSA（Type2）：由广播网或NBMA网络中的DR产生，Network-LSA中记录了这一网络上所有 UDG 的Router ID，描述本网段的链路状态，在所属的区域内传播。Network-LSA格式如 [图3](#ZH-CN_CONCEPT_0161317235__f91a6c33cc52a4c909d3f230e467bd62a) 所示。

**图3** Network-LSA格式

<br>

![](OSPFv3 LSA格式_61317235.assets/zh-cn_image_0161317355.png)

*表3 Network-LSA格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Options | 24比特 | UDG<br>支持的可选能力。 |
| Attached Router | 32比特 | 连接在同一个网络上的所有<br>UDG<br>的Router ID，也包括DR的Router ID。 |

#### [Inter-Area-Prefix-LSA](#ZH-CN_CONCEPT_0161317235)

Inter-Area-Prefix-LSA（Type3）：描述区域内某个网段的路由，由ABR产生，并通告给其他相关区域。

Inter-Area-Prefix-LSA的格式如 [图4](#ZH-CN_CONCEPT_0161317235__fd17fc66e2df34132bc22a977cdfa2c2c) 所示。

**图4** Inter-Area-Prefix-LSA格式

<br>

![](OSPFv3 LSA格式_61317235.assets/zh-cn_image_0161317257.png)

*表4 Inter-Area-Prefix-LSA格式字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| PrefixLength | 8比特 | 前缀长度。 |
| PrefixOption | 8比特 | 前缀相关能力选项。 |
| Address Prefix | 32比特 | 前缀本身的编码。 |

#### [Inter-Area-Router-LSA](#ZH-CN_CONCEPT_0161317235)

Inter-Area-Router-LSA（Type4）：由ABR产生，描述到ASBR的路由，通告给除ASBR所在区域的其他相关区域。

Inter-Area-Router-LSA的格式如 [图5](#ZH-CN_CONCEPT_0161317235__fe9572134e5034967a6e8b071ace65c4b) 所示。

*表5 Inter-Area-Router-LSA字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Destination Router ID | 32比特 | LSA描述的<br>UDG<br>的Router ID。 |

#### [AS-External-LSA](#ZH-CN_CONCEPT_0161317235)

AS-External-LSA描述到AS外部的路由，由ASBR产生。

AS-External-LSA的格式如 [图6](#ZH-CN_CONCEPT_0161317235__f3074bf3db6304331830d1f112b6e3c8c) 所示。

**图6** AS-External-LSA格式

<br>

![](OSPFv3 LSA格式_61317235.assets/zh-cn_image_0161317345.png)

*表6 AS-External-LSA字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| E | 1比特 | 表示外部开销类型：<br>- 如果值为1，则表示Type 2外部开销类型。<br>- 如果值为0，则表示Type 1外部开销类型。 |
| F | 1比特 | 表示LSA中是否包括Forwarding Address：<br>- 如果值为1，则表示包含。<br>- 如果值为0，则表示不包含。 |
| T | 1比特 | 表示LSA中是否包括External Route Tag：<br>- 如果值为1，则表示包含。<br>- 如果值为0，则表示不包含。 |
| Referenced LS Type | 16比特 | 关联的链路状态类型。如果不为0，则表示具有这种类型的LSA和本LSA相关（参考下面的Referenced Link State ID）。 |
| Forwarding Address | 128比特 | 一个完全合格的全球IPv6地址。 |
| External Route Tag | 32比特 | 外部路由标识，可以用来在ASBR之间传递额外信息。 |
| Referenced Link State ID | 32比特 | 关联的链路状态ID。 |

#### [Link-LSA](#ZH-CN_CONCEPT_0161317235)

每个 UDG 都会为每个链路产生一个Link-LSA，描述到此Link上的link-local地址、IPv6前缀地址，并提供将会在Network-LSA中设置的链路选项，它仅在此链路内传播。

Link-LSA的格式如 [图7](#ZH-CN_CONCEPT_0161317235__fca73784010ed400789a1ac7ac51f5904) 所示。

**图7** Link-LSA格式

<br>

![](OSPFv3 LSA格式_61317235.assets/zh-cn_image_0161317342.png)

*表7 Link-LSA字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Rtr Priority | 8比特 | 接口的路由器优先级。 |
| Options | 24比特 | 广播或NBMA链路上DR生成的Network-LSA中可能设置的选项。 |
| Link-local Interface Address | 128比特 | 链路上始发<br>UDG<br>的链路本地地址。 |
| Number of prefixes | 32比特 | LSA中IPv6地址前缀数量。 |

#### [Intra-Area-Prefix-LSA](#ZH-CN_CONCEPT_0161317235)

每个 UDG 及DR都会产生一个或多个此类LSA，在所属的区域内传播。

- UDG 产生的此类LSA，描述与Route-LSA相关联的IPv6前缀地址。
- DR产生的此类LSA，描述与Network-LSA相关联的IPv6前缀地址。

Intra-Area-Prefix-LSA的格式如 [图8](#ZH-CN_CONCEPT_0161317235__f62dea819957741589a3f01f4e2b22fd5) 所示。

**图8** Intra-Area-Prefix-LSA格式

<br>

![](OSPFv3 LSA格式_61317235.assets/zh-cn_image_0161317222.png)

*表8 Intra-Area-Prefix-LSA字段解释*

| 字段名 | 长度 | 含义 |
| --- | --- | --- |
| Referenced LS Type | 16比特 | 描述IPv6前缀地址与Router-LSA或Network-LSA相关联。<br>- 如果Referenced LS Type字段是0x2001，则表示IPv6前缀地址与Router-LSA相关联。<br>- 如果Referenced LS Type字段是0x2002，则表示IPv6前缀地址与Network-LSA相关联。 |
| Referenced Link State ID | 32比特 | 相关联的链路状态ID。<br>- 如果Referenced LS Type字段是0x2001，则Referenced Link State ID应该是0。<br>- 如果Referenced LS Type字段是0x2002，则Referenced Link State ID应该是DR的接口ID。 |
| Referenced Advertising Router | 32比特 | 相关联的发布路由器的ID。<br>- 如果Referenced LS Type字段是0x2001，则Referenced Advertising Router字段应该是始发UDG的Router ID。<br>- 如果Referenced LS Type字段是0x2002，则Referenced Advertising Router字段应该是DR的Router ID。 |
