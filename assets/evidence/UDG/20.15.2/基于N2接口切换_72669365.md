# 基于N2接口切换

- [业务场景](#ZH-CN_TOPIC_0000001472669365__1.3.1.1)
- [信令流程](#ZH-CN_TOPIC_0000001472669365__1.3.2.1)

#### [业务场景](#ZH-CN_TOPIC_0000001472669365)

当NG-RAN之间不存在Xn接口时或者基于Xn接口的切换流程失败（比如，目标NG-RAN与源UPF之间无IP连接）时，需要通过N2接口进行切换。

基于N2接口的切换流程中，如果源NG-RAN向SMF指示源NG-RAN和目标NG-RAN之间存在直接转发路径，则SMF可以决定在切换流程中使用直接转发路径，即源NG-RAN将缓存的下行数据直接转发给目标NG-RAN，否则切换流程中要使用间接转发路径，源NG-RAN通过源UPF和目标UPF将缓存的下行数据转发给目标NG-RAN。

1. 内部流程：切换准备和切换开始。
  相关机制：Source NG-RAN决定启动基于N2接口的切换。
2. 消息交互：Source NG-RAN发送[HANDOVER REQUIRED](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/HANDOVER REQUIRED_76785269.md)给Source AMF。
  关键信元：
  | Handover Type | 表示源侧触发了哪种切换。<br>- Intra5GS: NG-RAN node to NG-RAN node<br>- 5GStoEPS: NG-RAN node to eNB<br>- EPSto5GS: eNB to NG-RAN node |
  | --- | --- |
  | Cause | 该信元指示NGAP协议特定事件的原因。 |
  | Target ID | 该IE标识切换的目标，Target ID可以是NG-RAN，也可以是E-UTRAN。 |
  | **PDU Session Resource List** | PDU Session Resource List携带的必选信元为PDU Session ID、HANDOVER REQUIRED Transfer。其中handover required transfer可以携带Direct Forwarding Path Availability信元（如果Direct Forwarding Path Availability信元取值为"direct path available"，则表示可以使用直接转发路径）。 |
  | Source to Target Transparent Container | 该IE用于通过核心网将无线相关信息从切换源侧透传给切换目标侧；由源侧NG-RAN节点产生，并发送到目标侧NG-RAN节点。 |
  相关机制：源NG-RAN通知有用户要进行切换。消息中指示是否支持直接转发路径。如果支持直接转发，则NG-RAN直接将下行数据转发给目标NG-RAN。如果不支持，则需要建立间接转发隧道，间接转发隧道是指源NG-RAN通过5G核心网将下行数据转发给目标NG-RAN。
3. 内部流程：AMF选择。
  相关机制：如果源AMF不能再服务UE，则源AMF根据HANDOVER REQUIRED消息中的targetID选择一个可提供服务的目标AMF。
4. 消息交互：Source AMF发送[Namf_Communication_CreateUEContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Namf/N14/Namf_Communication_CreateUEContext/Namf_Communication_CreateUEContext Request_84640897.md)给Target AMF。
  关键信元：
  | ueContext | 表示要创建的单个ueContext资源。 |
  | --- | --- |
  | targetId | 表示目标NG-RAN的标识。 |
  | sourceToTargetData | 该信元包含“源到目标透明容器”，携带N2信息。 |
  | pduSessionList | 该信元应包含N2SmInformation的列表，其中每个N2SmInformation包括从源NG-RAN接收的基于PDU会话ID的“HANDOVER REQUIRED Transfer” |
  相关机制：源AMF给目标AMF发送消息创建UE上下文。消息中包含用户的上下文信息、N2信息。如果源AMF和目标AMF是同一个AMF，则此步骤不需要。
5. 消息交互：Target AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  关键信元：
  | ueLocation | 包含UE的位置信息。 |
  | --- | --- |
  | n2SmInfo | 包含传递给SMF的N2信息。 |
  | n2SmInfoType | "HANDOVER_REQUIRED"表示N2消息为切换请求。 |
  相关机制：对于源NG-RAN指示的每个PDU Session，目标AMF发送Nsmf_PDUSession_UpdateSmContext请求给关联的SMF，建立目标AMF与SMF的关联，告知SMF需要切换的PDU会话ID、Target ID、T-AMF ID以及N2信息。PDU会话ID，表示N2切换的候选PDU会话。Target ID表示UE的位置信息。SM N2 Info包含Direct Forwarding Path Availability指示转发路径。
6. 内部流程：UPF选择。
  相关机制：SMF根据Target ID判断UE是否可以接受切换，以及UE是否移出了UPF的服务范围从而决定是否选择新的UPF。
7. 消息交互：SMF发送[PFCP Session Modification Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)给UPF(PSA)。
  关键信元：
  | Update FAR | 修改先前为PFCP会话创建的FAR。 |
  | --- | --- |
  相关机制：如果SMF选择一个新的UPF作为PDU Session的中间UPF，并且需要使用不同的CN Tunnel Info，则SMF向UPF（PSA）发送消息创建隧道信息。当核心网侧隧道信息由SMF分配时，SMF提供核心网UPF(PSA)侧隧道信息（N9）以及报文检测规则。
8. 消息交互：UPF(PSA)发送[PFCP Session Modification Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)给SMF。
  关键信元：
  | Cause | 该信元应指示接受或拒绝相应的请求消息。 |
  | --- | --- |
  | Create PDR | 该信元应包含与PFCP会话相关的PDR信息，可能存在多个该信元的实例。 |
  相关机制：如果UPF（PSA）分配了CN隧道信息（N9），则向SMF提供，并将核心网侧隧道信息（N9）与SMF提供的上行报文检测规则进行关联。
9. 消息交互：SMF发送[PFCP Session Establishment Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Establishment Request_32945585.md)给Target I-UPF。
  关键信元：
  | Create FAR | 创建FAR。 |
  | --- | --- |
  相关机制：如果SMF为PDU会话选择新的中间UPF，即Target I-UPF，如果CN隧道信息由Target I-UPF分配，则向Target I-UPF发送消息，提供在T-UPF上安装报文检测、执行和上报规则。该PDU会话的UPF（PSA）的CN隧道信息（N9）也提供给Target I-UPF，用于建立N9隧道。
10. 消息交互：Target I-UPF发送[PFCP Session Establishment Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Establishment Response_85720872.md)给SMF。
  关键信元：
  | Cause | 该信元应指示接受或拒绝相应的请求消息。 |
  | --- | --- |
  | Create PDR | 该信元应包含与PFCP会话相关的PDR信息，可能存在多个该信元的实例。 |
  相关机制：Target I-UPF向SMF发送N4会话建立响应消息，携带CN隧道信息(N9)和CN隧道信息（N3）。SMF启动定时器，释放Source I-UPF的资源。
11. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给Target AMF。
  关键信元：
  | ueLocation | 包含UE的位置信息。 |
  | --- | --- |
  | n2SmInfo | 包含N2信息。 |
  | n2SmInfoType | 表示N2消息类型。 |
  相关机制：SMF给Target AMF回复消息，消息中携带PDU会话ID，N2 SM信息。其中N2 SM信息包含了CN隧道信息（N3）和QoS参数。如果直接转发数据通道不可用，N2 SM信息中还包括指示转发数据不可用的标识。如果SMF没有接受PDU Session的N2切换，则SMF不会在响应消息中携带PDU会话的N2 SM信息，以避免在目标NG-RAN上建立无线资源。
12. 内部流程：监控响应消息。
  相关机制：AMF监控来自相关SMF的响应消息。等待切换候选PDU的最大延迟指示的最低值为AMF等待Nsmf_PDUSession_UpdateSmContext响应消息的最大时间。在最大等待时间超时或收到所有Nsmf_PDUSession_UpdateSmContext响应消息后，AMF继续执行N2切换流程。
13. 消息交互：Target AMF发送[HANDOVER REQUEST](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/HANDOVER REQUEST_76783992.md)给Target NG-RAN。
  关键信元：
  | Handover Type | 该信元指示切换类型。不同场景下，切换类型信元为取值如下：<br>Intra5GS: NG-RAN node to NG-RAN node<br>5GStoEPS: NG-RAN node to eNB<br>EPSto5GS: eNB to NG-RAN node |
  | --- | --- |
  | **PDU Session**<br>**Resource Setup List** | PDU会话资源建立列表。 |
  | HANDOVER REQUEST Transfer | SMF通过AMF透传给NG-RAN的HANDOVER REQUEST Transfer IE。 |
  | Cause | 该信元指示NGAP协议特定事件的原因。 |
  相关机制：Target AMF根据Target ID中的Global RAN Node ID信元确定Target NG-RAN。Target AMF给Target NG-RAN发送消息，请求建立无线侧网络资源，并传递Target I-UPF侧N3隧道信息。
14. 消息交互：Target NG-RAN发送[HANDOVER REQUEST ACKNOWLEDGE](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/HANDOVER REQUEST ACKNOWLEDGE_76785121.md)给Target AMF。
  关键信元：
  | **PDU Session Resource**<br>**Admitted List** | PDU会话资源准入列表。 |
  | --- | --- |
  | HANDOVER REQUEST ACKNOWLEDGE Transfer | NG-RAN通过AMF透传给SMF的HANDOVER REQUEST ACKNOWLEDGE Transfer IE。 |
  | **PDU Session Resource Failed to Setup List** | PDU会话资源建立失败列表。 |
  | Target to Source Transparent Container | 该IE用于通过核心网将无线相关信息从切换目标侧透传给切换源侧；由目标侧NG-RAN节点产生，并发送到源侧NG-RAN节点。 |
  相关机制：Target NG-RAN回复消息给Target AMF，此时Target NG-RAN做好接收分组数据单元的准备。消息中包括可切换的PDU会话列表和无法切换的PDU会话列表，可切换的PDU会话列表中每个PDU会话的N2 SM信息包含Target NG-RAN的N3隧道信息和N3转发隧道信息。
15. 消息交互：Target AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  关键信元：
  | ueLocation | 包含UE的位置信息。 |
  | --- | --- |
  | n2SmInfo | 包含N2信息。 |
  | n2SmInfoType | 表示N2消息类型。"HANDOVER_REQ_ACK"指示切换请求确认。 |
  相关机制：AMF给SMF发送消息，告知SMF Target NG-RAN的N3隧道信息和N3转发隧道信息。
16. 消息交互：SMF发送[PFCP Session Modification Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)给Target I-UPF。
  关键信元：
  | Update FAR | 修改先前为PFCP会话创建的FAR。 |
  | --- | --- |
  相关机制：SMF发送消息将Target NG-RAN的N3转发隧道信息更新到Target I-UPF。如果源NG-RAN指示建立转发隧道，则SMF向目标Target I-UPF发送分配N9转发通道标识。
17. 消息交互：Target I-UPF发送[PFCP Session Modification Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)给SMF。
  关键信元：
  | Cause | 该信元应指示接受或拒绝相应的请求消息。 |
  | --- | --- |
  | Create PDR | 该信元应包含与PFCP会话相关的PDR信息，可能存在多个该信元的实例。 |
  相关机制：Target I-UPF分配N9转发隧道信息，给SMF回复消息携带Target I-UPF的SM N9转发隧道信息。
18. 消息交互：SMF发送[PFCP Session Modification Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)给Source I-UPF。
  关键信元：
  | Update FAR | 修改先前为PFCP会话创建的FAR。 |
  | --- | --- |
  相关机制：SMF给Source I-UPF发送消息，告知Target I-UPF N9转发隧道信息，以及建立数据间接转发通道信息的指示。
19. 消息交互：Source I-UPF发送[PFCP Session Modification Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)给SMF。
  关键信元：
  | Cause | 该信元应指示接受或拒绝相应的请求消息。 |
  | --- | --- |
  | Create PDR | 该信元应包含与PFCP会话相关的PDR信息，可能存在多个该信元的实例。 |
  相关机制：Source I-UPF给SMF发送消息携带Source I-UPF侧N3转发隧道信息。
20. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给Target AMF。
  关键信元：
  | n2SmInfo | 包含N2信息。 |
  | --- | --- |
  | n2SmInfoType | 表示N2消息类型。"HANDOVER_CMD"指示切换命令传输。 |
  相关机制：SMF在消息中包含N2 SM information，包含发送给Source NG-RAN的N3转发隧道信息。如果是间接转发，SMF在消息中包含Source I-UPF的转发隧道信息，如果是直接转发，SMF在消息中包含Target NG-RAN的N3转发隧道信息。
21. 消息交互：Target AMF发送[Namf_Communication_CreateUEContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Namf/N14/Namf_Communication_CreateUEContext/Namf_Communication_CreateUEContext Response_84800841.md)给Source AMF。
  关键信元：
  | sourceToTargetData | 该信元包含“源到目标透明容器”，携带N2信息。 |
  | --- | --- |
  | pduSessionList | 该信元应包含N2SmInformation的列表，其中每个N2SmInformation包括从源NG-RAN接收的基于PDU会话ID的“HANDOVER REQUIRED Transfer”。 |
  相关机制：Target AMF将包含转发隧道信息的N2 SM信息携带给Source AMF。如果Source AMF和Target AMF是同一个AMF，则此步骤不需要。
22. 消息交互：Source AMF发送[HANDOVER COMMAND](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/HANDOVER COMMAND_76783896.md)给Source NG-RAN。
  关键信元：
  | Handover Type | 该信元指示切换类型。不同场景下，切换类型信元为取值如下：<br>Intra5GS: NG-RAN node to NG-RAN node<br>5GStoEPS: NG-RAN node to eNB<br>EPSto5GS: eNB to NG-RAN node |
  | --- | --- |
  | **PDU Session Resource**<br>**Handover**<br>**List** | PDU会话资源切换列表。 |
  | HANDOVER COMMAND Transfer | SMF通过AMF透传给NG-RAN的HANDOVER COMMAND Transfer IE。 |
  | **PDU Session Resource**<br>**to Release List** | PDU会话资源释放列表。 |
  | Target to Source Transparent Container | 该IE用于通过核心网将无线相关信息从切换目标侧透传给切换源侧，由Traget NG-RAN节点产生，并发送到Source NG-RAN节点。 |
  相关机制：Source AMF给Source NG-RAN发送消息，通知切换准备完成。消息中包括从Target NG-RAN处获取的接受切换的PDU会话列表和拒绝切换的PDU会话列表，以及每个会话的N2 SM信息。如果是直接转发，SM转发信息是Target NG-RAN的N3转发隧道信息，如果是间接转发，SM转发信息是Source I-UPF的N3转发隧道信息。
23. 消息交互：Source NG-RAN发送[HANDOVER COMMAND](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/HANDOVER COMMAND_76783896.md)给UE。
  关键信元：
  | Handover Type | 该信元指示切换类型。不同场景下，切换类型信元为取值如下：<br>Intra5GS: NG-RAN node to NG-RAN node<br>5GStoEPS: NG-RAN node to eNB<br>EPSto5GS: eNB to NG-RAN node |
  | --- | --- |
  相关机制：收到这条消息后，UE将释放被目标NG-RAN所拒绝的PDU会话资源。
24. 消息交互：Source NG-RAN发送[UPLINK RAN STATUS TRANSFER](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/UPLINK RAN STATUS TRANSFER_76785560.md)给Source AMF。
  关键信元：
  | RAN Status Transfer Transparent Container | 该IE由Source NG-RAN节点产生，传递到Target NG-RAN节点，用于5GC内部NG切换。 |
  | --- | --- |
  相关机制：Source NG-RAN向Source AMF发送上行运行状态迁移消息。如果UE的所有无线承载都不应被以PDCP状态保存处理，则源NG-RAN可以省略发送该消息。
25. 消息交互：Source AMF发送[Namf_Communication_N1N2MessageTransfer Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Namf/N14/Namf_Communication_N1N2MessageTransfer/Namf_Communication_N1N2MessageTransfer Request_36361062.md)给Target AMF。
  关键信元：
  | n2InfoContainer | 传递N2 information。 |
  | --- | --- |
  相关机制：如果AMF迁移，Source AMF发送消息给Target AMF。
26. 消息交互：Target AMF发送[Namf_Communication_N1N2MessageTransfer Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Namf/N14/Namf_Communication_N1N2MessageTransfer/Namf_Communication_N1N2MessageTransfer Response_36361182.md)给Source AMF。
  关键信元：
  | cause | 提供AMF上N1/N2消息传输处理的结果。 |
  | --- | --- |
27. 消息交互：Target AMF发送[DOWNLINK RAN STATUS TRANSFER](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/DOWNLINK RAN STATUS TRANSFER_76785430.md)给Target NG-RAN。
  关键信元：
