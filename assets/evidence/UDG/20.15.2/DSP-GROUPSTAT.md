## [命令功能](#ZH-CN_MMLREF_0000001221110027)

该命令用于查询消费组的状态，通过识别当前消费组的状态信息，判断是否存在消息积压。

> **说明**
> - 支持输入参数为空时的全量查询，支持按照group名称查询时进行模糊匹配。
> - 对部分不需要对外展示的group信息（如拨测）会进行过滤。

## [参数说明](#ZH-CN_MMLREF_0000001221110027)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| GROUP_NAME | group名称 | 可选必选说明：可选参数<br>参数含义：用于指示系统需要查询指定group的状态信息。<br>取值范围：字符串类型，取值范围2~256。<br>默认值：无。<br>配置原则：不输入参数，执行当前命令即可查询全量group名称。 |

## [输出结果说明](#ZH-CN_MMLREF_0000001221110027)

命令执行正常，会返回命令执行成功的提示信息。输出结果说明如 [表1](#ZH-CN_MMLREF_0000001221110027__table2340152719181) 所示。

命令执行失败则返回相应的错误，常见错误码如 [表2](#ZH-CN_MMLREF_0000001221110027__table11591014182115) 所示。

*表1 输出项解释*

| 输出项名称 | 输出项解释 |
| --- | --- |
| group名称 | 表示group的名称。 |
| topic名称 | 一个group下可以有多个topic。 |
| topic分区 | 一个topic可以有多个分区，topic分区表示查询到的分区ID。 |
| 消费位移 | 表示消费者已消费消息的偏移量。 |
| 消息位移 | 表示生产者已生产消息的偏移量。 |
| 待消费消息 | 消息位移和消费位移的差值，表示未被消费（积压）的消息数。 |
| 消费者ID | 消费者的ID。 |
| 消费者IP | 消费者的IP地址。 |
| 管理者 | 表示kafka Broker端完成整个消费组分区重新分配的管理者。 |
| 状态 | 表示消费者组当前状态，状态包括：<br>- Empty：组内没有任何成员，但消费组可能存在已提交且未过期的位移数据。<br>- Dead：组内没有任何成员，消费者组的信息已经被管理者删除。<br>- PreparingRebalance：消费组准备开启重平衡。<br>- CompletingRebalance ：消费组下所有成员已经加入，各个成员正在等待分配方案。<br>- Stable：稳定状态。<br>- Unknown：未知状态，表示状态未初始化或没有状态。 |
| 分配方式 | 表示消费者分区分配策略。 |
| 成员数 | 表示当前消费组成员数量。 |
