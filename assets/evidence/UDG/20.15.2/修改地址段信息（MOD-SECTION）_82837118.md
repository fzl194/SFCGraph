#### [命令功能](#ZH-CN_CONCEPT_0182837118)

**适用NF：PGW-U、UPF**

![](修改地址段信息（MOD SECTION）_82837118.assets/notice_3.0-zh-cn.png)

本命令属于高危命令，当修改后的地址段范围比原来缩小时，如果用户的地址属于删除的部分，用户会被去活。

该命令可以用于动态修改地址段，当系统需要修改地址段的起始地址、终止地址时，可以通过该命令来修改。

#### [注意事项](#ZH-CN_CONCEPT_0182837118)

- 该命令执行后立即生效。
- 该命令不支持对外部地址池的修改。
- 当地址池为本地地址池，可以不用锁定地址段，直接使用该命令修改地址段的范围。
- 当输入的地址段号存在时，就可以重新配置地址段的起始地址和终止地址。
- 地址段的起始地址和地址段的终止地址有可以修改的范围，修改后的地址如果超出这个范围，命令则会执行失败并显示可修改的范围。
- 起始地址和终止地址修改趋势不能相同（起始地址和终止地址的数值不能同时增大或缩小）。例如：地址段配置为10.0.100.5~10.0.100.250时，不能在一条命令中将地址段修改为10.0.100.0~10.0.100.255（同时缩小），也不能修改为10.0.100.10~10.0.100.255（同时增大）
- 地址段的起始地址和终止地址的变化范围必须在起始和终止子段的ip范围内。例如：地址段为10.0.100.5~10.0.100.250，mask设置为5，此时一个子段为2^5=32个地址，此时，起始ip范围为10.0.100.0~10.0.100.31，终止ip范围为10.0.100.224~10.0.100.255，此时不能将地址段改为10.0.100.32~10.0.100.223（超过了起始子段的ip范围），也不能将地址段改为10.0.99.255~10.0.101.0（超过了终止子段的ip范围）
- 当在一次命令执行中配置起始地址和终止地址时，不允许一个新增地址，另一个删除地址。如果配置起始地址和终止地址一个为删除地址一个为新增地址的话，必须分两次进行配置，第一次只配置起始地址或者终止地址，第二次配置另外一个。
- 当修改后的地址段范围比原来缩小时，如果用户的地址属于删除的部分，用户会被去活。

#### [操作用户权限](#ZH-CN_CONCEPT_0182837118)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [使用实例](#ZH-CN_CONCEPT_0182837118)

- 假设运营商已配置了一个IPv4的本地地址池pool1时，需要为其配置一个IPv4地址段：
  ```
  ADD SECTION:POOLNAME="pool1",SECTIONNUM=0,IPVERSION=IPV4,V4STARTIP="10.1.1.15",V4ENDIP="10.1.1.200";
  ```
- 假设运营商需要修改本地地址池pool1下的一个IPv4地址段的起始地址：
  ```
  MOD SECTION: POOLNAME="pool1", SECTIONNUM=0, IPVERSION=IPV4, V4STARTIP="10.1.1.0", V4ENDIP="10.1.1.200";
  ```
- 假设运营商需要修改本地地址池pool1下的一个IPv4地址段的终止地址：
  ```
  MOD SECTION: POOLNAME="pool1", SECTIONNUM=0, IPVERSION=IPV4, V4STARTIP="10.1.1.0", V4ENDIP="10.1.1.220";
  ```
