#### [命令功能](#ZH-CN_CONCEPT_0000001174954301)

该命令用于设置CSDB容灾通道参数，包括心跳间隔和消息重试次数。

#### [注意事项](#ZH-CN_CONCEPT_0000001174954301)

- 两个DC间的容灾通道故障后，控制消息为一秒重试一次，故容灾探测最长时间为心跳间隔加上消息重试次数。例如：
    - 心跳间隔设置为30秒，消息重试次数设置为30次，消息重试一次一秒，则容灾探测最长时间为60秒。
      ```
      心跳间隔30秒 + (消息重试次数30次 * 消息发送频率1次/秒) = 60秒
      ```
- 该命令存在系统初始记录，参数的初始设置值如下表：
  | 参数标识 | HBINTERNAL | MSGRETRYNUM |
  | --- | --- | --- |
  | 初始值 | 30 | 30 |
- 通常情况下，不建议修改心跳间隔和消息重试次数。心跳间隔设置过低会导致心跳太频繁，消息重试次数过低容易导致误判。心跳间隔和消息重试次数设置过高会导致容灾探测的时间变长。若需要修改，建议操作员按所需的最长探测时间分别配置心跳间隔和消息重试次数。
- 业务感知容灾数据备份通道故障时间，需在容灾探测时间上再加上一轮检查定时器周期（15秒）。例如：
    - 心跳间隔设置为30秒，消息重试次数设置为30次，则业务感知容灾通道故障的总探测时间区间为[30-75]秒，容灾数据备份通道故障告警上报时间区间同样为[30-75]秒。
      ```
      最长业务感知故障时间：心跳间隔30秒 + (消息重试次数30次 * 消息发送频率1次/秒) + 检查定时器周期15秒 = 75秒
      最短业务感知故障时间：心跳间隔0秒 + (消息重试次数30次 * 消息发送频率1次/秒) + 检查定时器周期0秒 = 30秒
      ```
    - 心跳间隔设置为10秒，消息重试次数设置为1次，则业务感知容灾通道故障的总探测时间区间为[1-26]秒。
      ```
      最长业务感知故障时间：心跳间隔10秒 + (消息重试次数1次 * 消息发送频率1次/秒) + 检查定时器周期15秒 = 26秒
      最短业务感知故障时间：心跳间隔0秒 + (消息重试次数1次 * 消息发送频率1次/秒) + 检查定时器周期0秒 = 1秒
      ```

#### [操作用户权限](#ZH-CN_CONCEPT_0000001174954301)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001174954301)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| CLUSTERID | 子集群标识 | 可选必选说明：必选参数。<br>参数含义：该参数用于指定唯一一个子集群，可以通过<br>**[DSP DBCLUSTER](../集群管理/查询CSDB子集群信息（DSP DBCLUSTER）_29626985.md)**<br>命令查询获取。<br>数据来源：本端规划<br>取值范围：1～10。<br>默认值：无。 |
| HBINTERNAL | 容灾通道心跳间隔 | 可选必选说明：可选参数。<br>参数含义：该参数用于指定容灾通道心跳间隔。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为10～300，单位为秒。<br>默认值：30。 |
| MSGRETRYNUM | 容灾通道消息重试次数 | 可选必选说明：可选参数。<br>参数含义：该参数用于指定容灾通道消息重试次数。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～30，单位为次。<br>默认值：30。 |

#### [使用实例](#ZH-CN_CONCEPT_0000001174954301)

设置 **“子集群标识”** 为 “1” 的容灾通道参数：

```
SET DRCHANNELPARA: CLUSTERID=1, HBINTERNAL=20, MSGRETRYNUM=20;
```
