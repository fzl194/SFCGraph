## [命令功能](#ZH-CN_MMLREF_0000001535550174)

该命令用于查询HTR流控全局配置信息。

> **说明**
> 无

#### [操作用户权限](#ZH-CN_MMLREF_0000001535550174)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001535550174)

无

## [使用实例](#ZH-CN_MMLREF_0000001535550174)

查询HTR流控全局配置信息，可以用如下命令：

```
%%LST HTTPHTRCFG:;%%
RETCODE = 0  操作成功

结果如下
--------
         HTR流控开关  =  打开
   启控失败率阈值(%)  =  12
   解控失败率阈值(%)  =  1
   目标失败率阈值(%)  =  6
         统计周期(s)  =  5
          启控周期数  =  4
          解控周期数  =  8
     HTR超时时长(ms)  =  2000
 放通阈值减少比例(%)  =  10
放通阈值增加比例1(%)  =  5
放通阈值增加比例2(%)  =  18
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001535550174)

| 输出项名称 | 输出项解释 |
| --- | --- |
| HTR流控开关 | 该参数用于设置是否开启HTR流控功能。 |
| 启控失败率阈值(%) | 该参数用于设置对端Server由于过载回复的失败率达到多少百分比时触发HTR流控。 |
| 解控失败率阈值(%) | 该参数用于设置对端Server由于过载回复的失败率降到多少百分比时解除HTR流控。 |
| 目标失败率阈值(%) | 该参数用于设置不调整放通消息量的失败率阈值。 |
| 统计周期(s) | 该参数用于设置HTR统计周期。 |
| 启控周期数 | 该参数用于设置连续几个周期满足启控条件开启HTR流控。 |
| 解控周期数 | 该参数用于设置连续几个周期满足解控条件解除HTR流控。 |
| HTR超时时长(ms) | 该参数用于设置等待后端Server回复响应时长，超过该时长可能后端Server严重过载，来不及给Client回复响应，Client统计超时次数，作为是否启动HTR的分析数据。 |
| 放通阈值减少比例(%) | 该参数用于设置进入HTR流控后失败率大于启控失败率阈值时放通消息量减少比例。 |
| 放通阈值增加比例1(%) | 该参数用于设置当HTR流控起控后，过载失败率在解控失败率阈值和目标失败率之间时放通消息量增加的比例。 |
| 放通阈值增加比例2(%) | 该参数用于设置当HTR流控起控后，过载失败率低于解控失败率阈值时放通消息量增加比例。 |
