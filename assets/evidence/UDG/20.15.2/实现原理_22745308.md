**说到EPS Fallback语音方案的实现，大家关注的主要有这几点：注册、呼叫、回落、两种回落流程的差异。接下来我们就从这几个方面解读下EPS Fallback语音。**

**本节从5G核心网角度描述，端到端的解决方案参见support网站： *技术支持>产品支持>云核心网>CS&IMS>Single Voice Core>CloudIMS>Single Voice Core* 产品文档。**

#### [EPS Fallback注册](#ZH-CN_TOPIC_0000001122745308)

EPS Fallback回落流程的前提是完成核心网侧、IMS侧的IMS DNN的默认QoS Flow建立及注册。

1. UE已经在5G网络上完成注册。
  EPS Fallback的注册过程跟VoLTE基本相同，主要区别在于5G将初始注册的Registration过程与会话建立的PDU Session过程拆分为两个独立的过程。用户在EPC网络附着时，同时伴随着到默认APN（通常签约为数据APN）的PDN连接建立，然后再建立IMS APN，才可进行后续音视频会话的建立。用户在5GC网络注册时，不伴随PDU会话的建立，当需要发起音视频会话时，建立IMS PDU会话。
2. 附着到5G网络后已建立IMS PDU会话，并且已在IMS网络上注册成功。
    - 建立IMS PDU会话流程和IMS网络注册流程参见support网站：*技术支持>产品支持>云核心网>CS&IMS>Single Voice Core>CloudIMS>Single Voice Core*产品文档。
    - IMS 语音PDU会话流程与普通数据PDU建立的区别包括以下三点，详细参见*Single Voice Core*产品文档。
          a. UE向AMF发送PDU_Session_Establishment Request消息，额外携带Extended protocol configuration options信元指示请求P-CSCF地址。
          b. 为了主被叫BSF查询时候能获取到对应的PCF地址，需要在会话建立的过程中完成N7和Rx接口会话绑定，实现BSF和PCF的会话绑定。
          c. NG RAN发起将AMF发送的PDU Session Establishment Accept消息转发给UE，将P-CSCF地址信息发送给UE。

#### [基于切换的EPS Fallback流程](#ZH-CN_TOPIC_0000001122745308)

从5GS到EPS的回落过程有两种方式，一种是基于切换的EPS Fallback，另一种是重定向的EPS Fallback。两种方式的选择主要取决于无线侧资源，如果无线侧支持切换，则从业务连续性的角度来看，更倾向于通过切换的方式进行回落，可避免重定向带来的时延大缺陷。重定向主要用于当前服务小区过载或者UE已经移动到小区边缘需要触发切换但是此时UE不支持切换。

EPS Fallback之前，由NG RAN确认发起切换还是重定向。

基于切换的EPS Fallback流程图如 [图3](#ZH-CN_TOPIC_0000001122745308__fig0663194219359) 所示。

**图3** 基于切换的EPS Fallback流程图

<br>

![](实现原理_22745308.assets/zh-cn_image_0000001122745382.png "点击放大")

UE已经在5G网络上完成注册，建立IMS PDU会话，并且已在IMS网络上注册成功。流程参见 [EPS Fallback注册](#ZH-CN_TOPIC_0000001122745308__section928763617554) 。

1. UE发起IMS语音业务。UE发送SIP invite消息给IMS。IMS发送AAR消息给PCF，PCF响应AAA消息触发创建专用QoS Flow的流程。
2. PCF向SMF发送Npcf_SMPolicyControl_UpdateNotify Request消息通知SMF创建语音专有QoS Flow。SMF响应Npcf_SMPolicyControl_UpdateNotify Response消息。
3. SMF向AMF发送Namf_Communication_N1N2MessageTransfer，消息中携带SM相关信息。AMF响应Namf_Communication_N1N2MessageTransfer Response消息。
4. AMF向NG-RAN发送N2 Session Request消息，通知NG-RAN建立语音QoS Flow资源，NG-RAN拒绝语音QoS Flow，响应N2 Session Response消息，并携带IMS Voice EPS Fallback or RAT Fallback Triggered原因值。
  > **说明**
  > 5QI 对应 4G 的 QCI ，是业务质量的索引，代表了资源类型、优先级、可靠性、丢包率等一组参数的取值集合， 语音专载对应标准化5QI和QCI如下表，因此后续语音专有承载创建流程中，5G语音QoS Flow的 5QI =1，4G语音EPS Bear的QCI=1。标准5QI参数特征参见 [相关概念](#ZH-CN_TOPIC_0000001122745308__section57436448284) 。
5. AMF向SMF发送Nsmf_PDUSession_UpdateSMContext Request携带IMS Voice EPS Fallback or RAT Fallback Triggered原因值。SMF响应Nsmf_PDUSession_UpdateSMContext Response消息。
6. NG RAN发起5GS to EPS Handover流程。
7. Handover之后的TAU流程。
8. PGW根据缓存的QCI=1的QoS Flow创建请求，在用户回落EPS网络后再发起IMS语音专有承载创建流程。
9. PGW向PCRF/PCF发送Npcf_SMPolicyControl_update request消息，消息中携带IP-CAN-Type、RAT-Type等信息，这些信息都是4G网络中的信元。
10. PCF之后向IMS发送RAR消息，IMS响应RAA消息。IMS网络向UE发送SIP消息，语音通话正常。

1. 创建语音专用QoS Flow的流程，同基于切换的EPS Fallback流程[1](#ZH-CN_TOPIC_0000001122745308__li68861345163418)-[5](#ZH-CN_TOPIC_0000001122745308__li15886445183414)。

    d. NG RAN向AMF发送UE Context Release Complete消息。
    e. AMF向SMF发送Nsmf_PDUSession_UpdateSMContext消息。

6. PCF之后向IMS发送RAR消息，IMS响应RAA消息。IMS网络向UE发送SIP消息，语音通话正常。

#### [基于切换和重定向的EPS Fallback差异](#ZH-CN_TOPIC_0000001122745308)

从上面的流程介绍，我们看到5GS到EPS的回落过程有两种方式，一种是基于切换的EPS Fallback，另一种是重定向的EPS Fallback。 两种流程的主要差异如下：

*表1 基于切换和重定向的EPS Fallback流程差异*

| 流程 | 基于切换的EPS Fallback | 基于重定向的EPS Fallback |
| --- | --- | --- |
| Qos Flow的建立 | - PCF通知SMF创建语音专有QoS Flow，当流程到达NG RAN时，NG RAN拒绝语音QoS Flow，响应N2 Session Response消息，并携带IMS Voice EPS Fallback Triggered原因值。<br>- 两种方式的选择主要取决于无线侧资源，如果无线侧支持HO，则从业务连续性的角度来看，更倾向于通过HO的方式进行回落，可避免重定向带来的时延大缺陷。EPS Fallback之前，NG RAN会通知UE测量LTE的信号，确认可回落的条件下，才会进行后续的步骤。 | - PCF通知SMF创建语音专有QoS Flow，当流程到达NG RAN时，NG RAN拒绝语音QoS Flow，响应N2 Session Response消息，并携带IMS Voice EPS Fallback Triggered原因值。<br>- 两种方式的选择主要取决于无线侧资源，如果无线侧支持HO，则从业务连续性的角度来看，更倾向于通过HO的方式进行回落，可避免重定向带来的时延大缺陷。EPS Fallback之前，NG RAN会通知UE测量LTE的信号，确认可回落的条件下，才会进行后续的步骤。 |
| 回落过程 | 5GS到EPS的HO流程：<br>- NG RAN发起5GS to EPS HO流程，发送Handover Required消息到AMF。AMF向SMF获取SM上下文，给MME发送承载信息并通知SGW建立承载。<br>- MME向E-UTRAN请求建立无线网络资源，包含承载信息，安全上下文，在E-UTRAN做好切换准备时，NG RAN通知UE切换到目的接入网络，下行数据通过UPF和SGW，从NG-RAN发送到目的E-UTRAN。 | AN Release流程：<br>- NG RAN发起重定向流程，发送UE Context Release Request消息到AMF，AMF向NG RAN回复UE Context Release Command消息，指示NG RAN释放和UE的无线侧连接。<br>- NG RAN发起和UE的NG RAN Connection Release流程并向AMF发送UE Context Release Complete消息，无论UE是否返回无线侧释放流程的Ack。 |
| TAU过程 | 当EPS Fallback回落到4G网络后，终端会发起一次TAU过程，将MME的地址记录到UDM中。此处的TAU为部分TAU，只需完成位置更新。 | 当EPS Fallback回落到4G网络后，需终端重新发起一次TAU过程，将MME的地址记录到UDM中。 |
| IMS语音专载的建立 | PGW发起的IMS语音专载建立流程，流程与VoLTE的IMS语音专载建立流程相同，此时携带4G的用户位置信息。 | PGW发起的IMS语音专载建立流程，流程与VoLTE的IMS语音专载建立流程相同，此时携带4G的用户位置信息。 |

EPS Fallback两种回落方式优缺点对比：

*表2 EPS Fallback两种回落方式优缺点*

| 维度 | 基于切换的EPS Fallback | 基于重定向的EPS Fallback |
| --- | --- | --- |
| 成功率 | 基于切换方式的回落，UE上报测量报告存在延迟，在移动性场景可能会影响切换成功率。<br>基于切换方式的<br>回落成功率略低于基于重定向方式的回落成功率<br>。 | 基于重定向方式的回落，UE选择质量较好的LTE小区接入，成功率与LTE VOLTE建立成功率基本相当。 |
| 时延 | 相对于基于重定向方式的<br>回落时长大约短<br>220ms<br>。 | 基于重定向方式的回落会先释放业务，然后重新建立，信令流程较多，相对于基于切换方式的回落时长大约长220ms。 |
| 数据业务影响 | 数据业务通过CN转到LTE，<br>中断时长较短<br>。 | 数据业务会中断，回落到LTE之后重新建立业务恢复，中断时长较长。 |
| 网规要求 | 需要配置LTE邻频点与准确的LTE邻区，<br>网规难度较大<br>。 | 需要配置LTE邻频点与邻区，但是对邻区准确性要求较低，网规难度较低。 |
| 接口要求 | 需要配置<br>N26<br>接口。 | 可不需要N26接口，推荐配置N26接口。 |
