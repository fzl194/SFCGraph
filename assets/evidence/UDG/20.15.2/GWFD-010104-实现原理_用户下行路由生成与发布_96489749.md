#### [用户下行路由定义及作用](#ZH-CN_TOPIC_0000001496489749)

用户下行路由指外部PDN/DN到UE的下行路由，用于下行报文（PDN/DN服务器发送给UE的报文）的回程寻址。

用户下行路由如 [图1](#ZH-CN_TOPIC_0000001496489749__fig11967123818486) 所示，其作用是PDN/DN Server和路径中的路由设备能将UE下行报文发送给正确的UPF，UPF接收并处理用户下行报文，然后转发给对应的基站，由基站发送给UE。

UE发起IP类型的PDU会话建立过程中，已建立UPF和基站之间的下行隧道，UPF本地已记录基站/UE的IP地址。用户下行路由的核心是UPF如何将其上的UE IP通知到PDN/DN Server及路径中的路由设备。

#### [用户地址来源](#ZH-CN_TOPIC_0000001496489749)

用户地址从哪里来？

IP类型的PDU会话建立流程中，SMF不仅选择唯一的锚点UPF为该UE提供服务，还会指示UPF或其他NF给UE分配IP地址，或者采用UE静态签约的IP地址。对于UPF来说，UE地址来源分为：

- 本地分配地址：UPF给UE分配IP地址。
  [**ADD POOL**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/地址池配置/添加地址池（ADD POOL）_82837132.md) 命令中 “地址池类型（POOLTYPE）” 设置为 “LOCAL” 时，表示由UPF给UE分配IP地址，且分配的是动态IP（同一用户不同时刻开机，分配到的IP地址可能不同）。
- 外部分配地址：其他NF给UE分配IP地址，或者采用UDM上的UE静态签约IP地址。
  [**ADD POOL**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/地址池配置/添加地址池（ADD POOL）_82837132.md) 命令中 “地址池类型（POOLTYPE）” 设置为 “EXTERNAL” 时，表示由其他NF给UE分配IP地址，不是本UPF给用户分配IP地址。当采用UE静态签约IP地址时，同一用户不同时刻开机，如签约IP地址未发生变更，分配到的IP地址不变。

不管是采用本地分配地址，还是采用外部分配地址，UE IP地址均由UPF对外发布。

#### [UPF对外发布方式](#ZH-CN_TOPIC_0000001496489749)

UPF通过N6外联口，实现与DC-GW的互通。UPF对外发布用户下行路由的方式，包括：

- 静态路由：PDN/DN Server、UPF与PDN/DN之间的路由设备上配置到UE IP地址段的静态路由或默认路由，根据静态路由/默认路由的配置逐跳发送到UPF。
  UPF需提供UE IP地址段、isu-pod上的子接口IP地址等。UPF上需配置到PDN/DN Server的静态路由。
- OSPF动态路由：UPF和DC-GW之间建立OSPF邻居，将wlr类型的路由发布出去。DC-GW获取到UPF上的UE IP地址段的路由，并发布给其他路由设备及PDN/DN服务器等。
  UPF与DC-GW之间建立OSPF邻居，不需要提供UE IP地址段，不需要提供isu-pod上的子接口IP地址等。
- BGP动态路由：UPF和DC-GW之间建立BGP邻居，将wlr类型的路由发布出去。DC-GW获取到UPF上的UE IP地址段的路由，并发布给其他路由设备及PDN/DN服务器等。
  UPF与DC-GW之间建立BGP邻居，不需要提供UE IP地址段。当采用静态路由实现BGP本端IP与BGP PEER IP之间的互通时，需要提供isu-pod上的子接口IP地址等。

#### [用户地址池配置](#ZH-CN_TOPIC_0000001496489749)

UPF上，用户IP地址段和APN的配置关系如 [图2](#ZH-CN_TOPIC_0000001496489749__fig1735482215274) 所示。

- 本地分配地址时，1和2都必须配置，且[**ADD POOL**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/地址池配置/添加地址池（ADD POOL）_82837132.md)命令中“地址池类型（POOLTYPE）”设置为“LOCAL”。
- 外部分配地址时，1必须配置，且[**ADD POOL**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/地址池配置/添加地址池（ADD POOL）_82837132.md)命令中“地址池类型（POOLTYPE）”设置为“EXTERNAL”；UPF校验用户IP地址是否合法时需要配置2，否则不需要。

#### [用户下行路由生成与发布（本地分配地址）](#ZH-CN_TOPIC_0000001496489749)

采用本地分配地址， “Pool Type” 为 “POOL_LOCAL” 。

```
%%LST POOL:;%%
RETCODE = 0  Operation succeeded

Pool Information
----------------
Pool Name   IP Address Type  ...  Has VPN  VPN Instance Name  ...  Pool Type   ...  

pool01      IPV4             ...  ENABLE   VRF_N6             ...  POOL_LOCAL  ... 
...
```

UE IP地址段以如下显示信息为例，即规划的UE IP地址段为10.1.1.0~10.1.1.255。

```
%%LST SECTION: POOLNAME="pool01";%%
RETCODE = 0  Operation succeeded

IP Section Information
----------------------
Segment Number  Pool Name  Start IPv4 Address  End IPv4 Address  Lock Section  IPv6 Prefix Start Address  IP Address Type  IPv6 Prefix End Address  IPv6 Prefix Length  Mask  
1               pool01     10.1.1.0            10.1.1.255        UNLOCK        ::                         IPV4             ::                       0                   0     
...
```

- 用户发起PDU会话建立流程前，在UPF上执行**DSP ROUTE**命令，查询到UPF上生成的黑洞路由（wlr_bh）如下：
  ```
  Route VPN Name  Route Prefix     Route Mask Length  Route Protocol Name  ...  Route Priority  ...  Is Route Active  ...  Route Flag  Route State    ...  

  VRF_N6          10.1.1.0         24                 wlr_bh               ...  50              ...  TRUE             ...       DB      Active Adv    ...
  ...
  ```
- 用户发起PDU会话建立流程中，UPF根据APN查找待给UE分配的IP地址池，分配1个空闲IP给UE。用户PDU会话建立后，在UPF上执行**DSP ROUTE**命令，查询到UPF上生成的黑洞路由（wlr_bh）和用户路由（wlr_ud）如下：
  ```
  Route VPN Name  Route Prefix     Route Mask Length  Route Protocol Name  ...  Route Priority  ...  Is Route Active  ...  Route Flag  Route State    ...  

  VRF_N6          10.1.1.0         24                 wlr_bh               ...  50              ...  TRUE             ...       DB      Active Adv    ...
  VRF_N6          10.1.1.32        27                 wlr_ud               ...  1               ...  TRUE             ...       D       Active NoAdv  ...
  ...
  ```

UPF上配置动态路由（OSPF/BGP）发布用户下行路由时，OSPF/BGP引入wlr类型的路由，即将wlr_bh路由的 “Route Prefix” 和 “Route Mask Length” 参数值发布给DC-GW。
