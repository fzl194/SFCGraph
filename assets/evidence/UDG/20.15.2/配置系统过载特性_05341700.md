## [操作场景](#ZH-CN_OPI_0205341700)

在系统负荷较重的情况下，建议开启流控策略。通过控制系统接入业务量来保证系统过载情况下业务的正常进行，避免造成整系统瘫痪。

## [对系统的影响](#ZH-CN_OPI_0205341700)

UDG 启动流控保护，首要目的是确保 UDG 和周边网元的安全运行，其次才是业务的尽量接入。因此超出系统处理能力时，流控机制启动，会丢弃部分消息，导致部分用户激活失败。

## [任务示例](#ZH-CN_OPI_0205341700)

任务描述

配置系统过载控制功能。

脚本

//配置APN实例。

```
ADD APN:APN="apn1";
```

//配置APN下用户的接入速率。

```
SET APNACCESSWAL:APN="apn1",WALNUMBER=30;
```

//配置用户去激活流控速率。

```
SET DEACTIVERATE:RATE=25;
```

//配置每个VMPod每秒发送的最大Error Indication消息数。

```
SET SOFTPARA: DT=DWORD, DWORDNUM=47, DWORDVALUE=35;
```
