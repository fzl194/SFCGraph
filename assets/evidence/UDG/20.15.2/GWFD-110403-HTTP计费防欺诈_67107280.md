#### [适用NF](#ZH-CN_TOPIC_0167107280)

PGW-U、UPF

#### [定义](#ZH-CN_TOPIC_0167107280)

针对非法工具通过修改HTTP报文的Host、URL字段将正常计费的报文伪装成免费的报文实现计费欺诈的行为， UDG 提供HTTP计费防欺诈特性阻止该类行为的发生。 UDG 支持通过定义欺诈行为特征库的方式来应对欺诈类型快速更迭的问题。

#### [客户价值](#ZH-CN_TOPIC_0167107280)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 防止计费欺诈给运营商带来的经济损失。 |
| 用户 | - |

#### [应用场景](#ZH-CN_TOPIC_0167107280)

以下场景中，可能存在伪造免费HTTP数据报文的计费欺诈行为，建议部署HTTP计费防欺诈特性加以阻止。

- 免费业务套餐包
  运营商为了吸引用户，推出了各种优惠费率或免费的业务套餐包，这类套餐的用户报文中会包含相同的关键字。这些特征被非法工具利用后，将用户正常计费业务报文构造成伪套餐包特征的报文。在HTTP Host字段中携带套餐包Host信息，真实的URL填写在HTTP Path字段、Referer字段或者其他扩展字段中，通过代理服务器来访问真实的网页地址，达到骗取优惠费率或者免费上网的目的。
- 在线计费FUI重定向
  在线计费用户欠费的情况下，通常会部署FUI（Final Unit Indication）重定向到充值页面，以方便用户进行充值操作。一般情况会将充值页面URL设置为免费。利用此特征，非法工具将用户业务报文伪造成充值页面URL，例如在HTTP Path字段中携带充值页面URL和用户业务真实URL，通过代理服务器访问真实业务，从而诈取计费免费。

防欺诈业务规则数据库：网络中存在疑似的欺诈场景时，建议部署防欺诈业务规则数据库对欺诈类型进行识别，进而对存在欺诈的用户进行策略控制以避免出现损失。每个防欺诈业务数据库支持的欺诈类型有所差异，具体信息请联系华为技术支持获取服务。

#### [对系统的影响](#ZH-CN_TOPIC_0167107280)

开启HTTP计费防欺诈特性后，由于匹配到规则的报文需要进行DNS解析或者HTTP解析，因此系统处理负荷将增加，报文转发性能和吞吐量将下降，具体下降程度需要根据话务模型和实施的业务特性来评估。

华为公司提供容量评估专业服务，请联系华为公司当地机构获取专业服务支持。

#### [应用限制](#ZH-CN_TOPIC_0167107280)

- 开启HTTP计费防欺诈功能后，需要关闭WAP网关、浏览器的全局代理设置。如果用户使用浏览器设置全局代理服务器上网，由于用户访问的所有报文都会发往设置的代理服务器上，UDG检测到的报文目的地址是代理服务器地址而不是业务服务器的地址。这样会对免费报文进行正常计费。
  > **说明**
  > 计费防欺诈业务规则数据库功能不受此限制。
- 本特性需要使用业务服务器的域名，而有的业务域名在各个国家和区域有所不同且会有经常变动，需要提前获取域名、定期维护并及时刷新。
- 防欺诈业务规则数据库中欺诈类型的更新是逐步完善的。每一次发布防欺诈业务规则数据库，其中仅包含已经被识别出的欺诈类型。
- 防欺诈业务规则数据库仅针对特殊费率进行防欺诈判断。
- 防欺诈业务规则数据库仅支持针对HTTP协议进行防欺诈判断。
- 防欺诈业务控制策略仅支持在UDG本地配置。
- 由于欺诈手段的不断变化更新，UDG无法确保覆盖现网所有的HTTP计费欺诈场景。

#### [原理概述](#ZH-CN_TOPIC_0167107280)

**相关概念**

- URL与Host：WWW中每个网站存储的每个网页，网站所提供的每种服务都有一个与之对应的统一资源定位器（Uniform Resource Locator，URL），用以表示该网站、网页或服务位于Internet的哪个子网的哪台服务器上，以及在这台服务器的哪个目录、哪个文件夹中，用哪种协议与其通信。URL组成格式为：scheme://machine.domain/full-path-of-file，其中scheme表示访问对象时采用的协议，如HTTP、FTP、Telnet、Gopher，后面部分表示主机名（machine）、域名（.domain）以及资源的路径（full-path-of-file），具体格式应当根据scheme来决定。HTTP URL格式为http://<host>:<port>/<path>?<searchpart>，以http://www.huawei.com:80/html为例，www.huawei.com为host，80为端口号，html为相对路径。可见，URL = Host + Path。
- 防欺诈业务规则数据库：UDG将已知的欺诈场景的特征固化为内部表达式以便于本身识别这些欺诈场景，该表达式的集合称为防欺诈业务规则数据库。

**系统实现**

**伪造免费HTTP** **数据报文**

对于免费业务套餐包，由于业务服务器的地址有多个，且可能动态变化，无法通过设置指定Server IP实现业务报文过滤，一般会在 UDG 上设置含有免费Host/URL的过滤条件实现，如L3/4：dst_port=80，L7：url = “0.huawei.com/*”，对应的UNC上的费率为优惠或者免费的RG/SID。终端非法工具利用该漏洞，将真实访问业务报文伪装成免费业务报文。

例如 [图1](#ZH-CN_TOPIC_0167107280__fig290343234412) 所示， UDG 上设置了免费套餐业务的过滤条件（0.huawei.com），用户访问业务（www.example.com）时，通过非法软件构造成免费报文（0.huawei.com/www.example.com），将报文送到指定代理服务器上。代理服务器忽略欺诈Host/URL，获取用户真实业务URL（www.example.com），转发到业务服务器。同样，用户下行报文也会经代理服务器处理后转发到 UDG 上，继而转发到用户终端上，实现诈取优惠费率访问真实业务。

对于网络中存在的上述HTTP计费欺诈场景， UDG 提供了以下解决方案：

- 可预先获取服务器IP
  对于能预先获取服务器IP的情况，通过修改 UDG 上的业务过滤条件，增加三四层IP过滤，如L3/4：dst_port=80，dst_ip=10.12.1.23，即可实现计费防欺诈。
- 不能预先获取服务器IP
  对于不能预先获取服务器IP或服务器IP动态变化的情况， UDG 采用免费网址域名Host与Server IP映射表实现。
  用户访问HTTP业务时，客户端在建立HTTP连接前会发起DNS域名（Host）解析流程，以获取服务器的IP，然后向该IP发起建链请求。 UDG 通过解析DNS响应报文，可以得到Host～Server IP的映射关系。依据此原理，在 UDG 的三四层规则中配置优惠费率Host名 （ [**ADD FILTER**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/增加过滤器（ADD FILTER）_82837348.md) ） ，解析Host地址的DNS数据包，可得到Host～Server IP的自学习映射表。
  当 UDG 收到用户的HTTP业务访问请求时，如果检测到报文的Server IP不在自学习映射表中，则判断该数据包为伪造报文，按照正常费率计费。如 [图2](#ZH-CN_TOPIC_0167107280__fig152319532579) 所示。

- **部署防欺诈业务规则数据库**
    - 实现原理：解析识别报文的七层信息后，将该信息与防欺诈业务规则数据库中的表达式进行比较。一旦匹配成功，则视作该用户利用欺诈方式访问业务。UDG支持对该场景中的用户流量进行动作控制、矫正费率或是阻断流量。
    - 方案对比：
          - 非防欺诈特征库功能：欺诈类型单一、欺诈行为的诊断周期长、无法主动识别欺诈行为。
          - 防欺诈特征库功能：主动识别欺诈行为、特征库定期更新保障识别率（更新需单独订制）、与CloudUDN交互实现欺诈行为的可视化。
    - 功能配置：
      实际策略选用的架构类型由匹配成功的URR所使用的配置架构类型决定。从 [**ADD AFPOLICY**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务公共参数管理/业务公共参数/设置业务公共参数（SET SRVCOMMONPARA）_82837309.md) 命令中获取欺诈处理策略，已经通过 [**ADD PCCPOLICYGRP**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/增加PCC策略组（ADD PCCPOLICYGRP）_82837606.md) 或 [**ADD CATEGORYPROP**](../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/分类属性/增加分类属性（ADD CATEGORYPROP）_82837504.md) 命令配置。
    - 效果呈现：
          - UDG上提供性能指标查看命中防欺诈业务规则特征库的上下行包数、上下行千字节数及在线用户数。
          - UDG上报信息给CloudUDN，由CloudUDN呈现具体的欺诈行为分析结果。支持的防欺诈主题包括免费流量趋势分析、免费业务分析、免费网站分析、欺诈用户分析和欺诈Proxy分析。
