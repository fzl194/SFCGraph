# 调测用户Portal

- [操作场景](#ZH-CN_OPI_0166620115__1.3.1)
- [必备事项](#ZH-CN_OPI_0166620115__1.3.2)
- [操作步骤](#ZH-CN_OPI_0166620115__1.3.3)

## [操作场景](#ZH-CN_OPI_0166620115)

当运营商需要部署用户Portal功能以实现在用户激活后首次使用浏览业务时向用户推送欢迎页面，或在用户浏览网页时定时向用户推送广告或提醒页面时，需对 UDG 的用户Portal功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0166620115)

前提条件

- 请仔细阅读[GWFD-110281 用户Portal](../GWFD-110281 用户Portal_66620113.md)。
- 完成[配置用户Portal](配置用户Portal_66620114.md)。
- 重定向设备状态正常，与UDG的互通配置已经完成。
- 镜像接口已经安装了第三方抓包工具。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 测试终端使用的APN | APN名称（APN） | apn-test | 已配置数据中获取 | 取自<br>[配置用户Portal](配置用户Portal_66620114.md)<br>中配置的APN实例名。 |
| Captive Portal触发地址 | URL | www.huawei.com/* | 已配置数据中获取 | 取自<br>[配置用户Portal](配置用户Portal_66620114.md)<br>中配置的七层过滤条件中的URL。 |
| Captive Portal目的地址 | 服务器IPv4地址（SERVERIPV4） | 192.168.10.123<br>192.168.10.124 | 已配置数据中获取 | 取自<br>[配置用户Portal](配置用户Portal_66620114.md)<br>中配置的IP Farm重定向服务器的目的地址。 |
| 心跳检测接口IP地址 | 逻辑接口的IPv4地址1（IPV4ADDRESS1） | 10.10.10.10 | 已配置数据中获取 | 取自<br>[配置用户Portal](配置用户Portal_66620114.md)<br>中配置的心跳检测接口的IP地址。 |

工具

- 测试终端
- OM Portal
- 第三方抓包工具

## [操作步骤](#ZH-CN_OPI_0166620115)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令查询 UDG 上 用户Portal功能 对应的License配置开关是否打开。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5CPPT01";
  ```
    - 如果License支持Portal功能为“DISABLE”，则设备不支持Captive Portal功能，请开启License后继续调测本功能。
    - 如果License支持Portal功能的为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0166620115__step2)。
2. 打开接入侧/DN侧镜像接口上的抓包工具，准备抓取测试终端的出入报文。
3. 测试终端 使用“apn-test”APN接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 4](#ZH-CN_OPI_0166620115__step4)。
    - 如果测试终端无法接入网络，请调测UDG的接入功能。
4. 使用 测试终端 浏览Web业务，访问 **www.huawei.com** 网页，查看镜像接口的抓包信息。
  **图1** UDG 接口抓包信息

  <br>

  ![](调测用户Portal_66620115.assets/zh-cn_image_0174413721.png "点击放大")
  > **说明**
  > 对于使用HTTP协议访问url以www.huawei.com开头的任何网页，都应该触发Captive Portal。即用户激活后的首次业务都被重定向到Captive Portal Server上。
  >
  > 如 [图1](#ZH-CN_OPI_0166620115__fig1) 中红框所示， 测试终端 首次浏览业务访问 **www.huawei.com** 网页，发送HTTP GET请求（消息9）， UDG 将丢弃此请求并模拟服务器向 测试终端 发送重定向报文（消息10）。同时 UDG 将模拟 测试终端 向用户请求的服务器发送拆链报文（消息11），并模拟服务器向 测试终端 发送拆链报文（消息12）。此时， 测试终端 上显示 www.example.com 网页。
  >
  > 重定向报文（消息10）如 [图2](#ZH-CN_OPI_0166620115__fig2) 所示，报文中携带重定向URL（www.example.com）。
  **图2** 重定向报文

  <br>

  ![](调测用户Portal_66620115.assets/zh-cn_image_0216613349.png)
    - 如果测试终端上看到跳转到“www.example.com”或“www.vmall.com”页面，UDG接口抓包结果与[图1](#ZH-CN_OPI_0166620115__fig1)、[图2](#ZH-CN_OPI_0166620115__fig2)一致，请执行[步骤 5](#ZH-CN_OPI_0166620115__step5)。
    - 如果测试终端未看到任何页面，或者测试终端上仍显示**www.huawei.com**网页，UDG接口抓包的目的IP与源IP一样，请执行[步骤 6](#ZH-CN_OPI_0166620115__step7)。
5. 测试终端 在 “www.example.com” 或 “www.vmall.com” 页面上重新访问 **www.huawei.com** 网页。
    - 如果测试终端上看到**www.huawei.com**网页，Captive Portal业务正常，调测结束。
    - 如果测试终端上仍看到“www.example.com”或“www.vmall.com”页面，或未看到任何页面，请执行[步骤 9](#ZH-CN_OPI_0166620115__step10)。
6. 检查APN下Captive Portal业务相关配置是否正确。
    a. 执行 [**LST RULEBINDING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_86526204.md) 命令，查询User Profile绑定的Rule是否正确。
      ```
      LST RULEBINDING:USERPROFILENAME="up_test";
      ```
      ```
      ......
      -------------------------
      用户模板名称    规则名称       优先级                策略类型                   
       up_test       rule_test1       10          Captive Portal Smart Redirect
       up_test       rule_test2       20                      PCC                           
      (结果个数 = 2)
      ---    END
      ```
          - 如果User Profile绑定的“规则名称”和“优先级”与规划值一致，请执行[步骤 6.b](#ZH-CN_OPI_0166620115__substep114481119175419)。
          - 如果User Profile绑定的“规则名称”和“优先级”与规划值不一致，请参考[配置用户Portal](配置用户Portal_66620114.md)重新配置。
    b. 执行 [**LST RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，查询Rule的配置信息是否正确。
      ```
      LST RULE:RULENAME="rule_test1",POLICYTYPE=SMARTREDIRECT;
      ```
      ```
      ......
      -------------------------
                                   规则名称  =  rule_test1
                                   策略类型  =  Captive Portal Smart Redirect
       	                            ......
                               流过滤器名称  =  ff_test1
       	                            ......
                                 全局优先级  =  10
       	                            ......
                 Captive Portal IP-Farm名称  =  farm_test
       	                            ......
      ---    END
      ```
      ```
      LST RULE:RULENAME="rule_test2",POLICYTYPE=PCC;
      ```
      ```
      ......
      -------------------------
                                   规则名称  =  rule_test2
                                   策略类型  =  PCC
       	                            ......
                               流过滤器名称  =  ff_test1
       	                            ......
                                 全局优先级  =  20
                              PCC策略组名称  =  pcc-pg_test
                                          ......
      ```
          - 如果Rule绑定的“流过滤器名称”、“Captive Portal IP-Farm名称”和“PCC策略组名称”与规划值一致，请执行[步骤 6.c](#ZH-CN_OPI_0166620115__substep104521819155412)。
          - 如果Rule绑定的“流过滤器名称”、“Captive Portal IP-Farm名称”和“PCC策略组名称”与规划值不一致，请参考[配置用户Portal](配置用户Portal_66620114.md)重新配置。
    c. 执行 [**LST PCCPOLICYGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/查询PCC策略组（LST PCCPOLICYGRP）_82837609.md) 、 [**LST URRGROUP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则组/查询URR组（LST URRGROUP）_82837637.md) 命令，查询Rule的PCC策略组信息是否正确。
      ```
      LST PCCPOLICYGRP:PCCPOLICYGRPNM="pcc-pg_test";
      ```
      ```
      ......
      -------------------------
             PCC策略组名称  =  pcc-pg_test
              计费属性名称  =  cp_test
                          ......
      ---    END
      ```
      ```
      LST URRGROUP:URRGROUPNAME="cp_test";
      ```
      ```
      ......
      -------------------------
                使用量上报规则组名称   =  cp_test
                                    ......
                    上行发起URR名称1   =  cbb_test
                    下行发起URR名称1   =  cbb_test
      (结果个数 = 1)
      ---    END
      ```
          - 如果Rule的PCC策略组信息与规划值一致，请执行[步骤 6.d](#ZH-CN_OPI_0166620115__substep1246051915547)。
          - 如果Rule的PCC策略组信息与规划值不一致，请参考[配置用户Portal](配置用户Portal_66620114.md)重新配置。
    d. 执行 [**LST FLOWFILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/查询流过滤器（LST FLOWFILTER）_82837364.md) 、 [**LST PROTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的协议绑定/查询流过滤器协议绑定关系（LST PROTBINDFLOWF）_82837373.md) 、 [**LST L7FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/七层规则管理/七层过滤器/查询七层过滤器（LST L7FILTER）_86526660.md) 、 [**LST FLTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的过滤器绑定/查询流过滤器的过滤器绑定关系（LST FLTBINDFLOWF）_82837368.md) 和 [**LST FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/查询过滤器（LST FILTER）_82837353.md) 命令，查询Rule绑定的流过滤信息是否正确。
      ```
      LST FLOWFILTER:FLOWFILTERNAME="ff_test1";
      ```
      ```
      ......
      -------------------------
              流过滤器名称  =  ff_test1
                         ......
      ---    END
      ```
      ```
      LST PROTBINDFLOWF:FLOWFILTERNAME="ff_test1";
      ```
      ```
      ......
      -------------------------
         流过滤器名称  =  ff_test1
             协议名称  =  http
       七层过滤器名字  =  l7-test
      (结果个数 = 1)
      ---    END
      ```
      ```

                                   ......

      ```

      ```

                                              ......
      ```

          - 如果Rule绑定的流过滤信息与规划值一致，请执行[步骤 7](#ZH-CN_OPI_0166620115__step8)。
          - 如果Rule绑定的流过滤信息与规划值不一致，请参考[配置用户Portal](配置用户Portal_66620114.md)重新配置。
7. 执行 [**LST IPFARM**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/IP Farm 管理/IP Farm参数/查询IPFarm（LST IPFARM）_82837053.md) 和 [**LST IPFARMSERVER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/IP Farm 管理/IP Farm服务器/查询IPFarmServer（LST IPFARMSERVER）_82837059.md) 命令，检查IP Farm中server的状态。
  ```
  LST IPFARM:IPFARMNAME="farm_test";
  ```
  ```
  ......
  -------------------------
             IP-Farm名称  =  farm_test
              服务器类型  =  重定向
                       ......
            健康检查标记  =  使能
              IP协议版本  =  IPV4
                       ......
        心跳检测接口名称  =  phif1/0/0
              接口IP地址  =  10.10.10.10
                       ......
  ---    END
  ```
  ```
  LST IPFARMSERVER:IPFARMNAME="farm_test";
  ```
  ```
  ......
  ------------------------
       IP-Farm名称  =  farm_test
          地址信息  =  192.168.10.123
               URL  =  www.example.com
   请求URL携带标识  =  不使能
       请求URL名称  =  dest_url
                ......
        服务器状态  =  up
    重定向缺省动作  =  block
  (结果个数 = 1)
  ---    END
  ```
8. 执行 [**LST LOGICINF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/逻辑接口管理/逻辑接口配置/接口/查询逻辑接口（LST LOGICINF）_86526672.md) 命令，检查IP Fam中server的心跳检测接口的相关配置是否正确。
  ```
  LST LOGICINF:NAME="phif1/0/0";
  ```
  ```
  ......
  -------------------------
            逻辑接口名称  =  phif1/0/0
     逻辑接口的IPv4地址1  =  10.10.10.10
     逻辑接口的IPv4掩码1  =  255.255.255.255
                       ......
  ---    END
  ```
    - 如果配置与规划值一致，请执行[步骤 9](#ZH-CN_OPI_0166620115__step10)。
    - 如果配置与规划值不一致，请参考[配置用户Portal](配置用户Portal_66620114.md)重新配置。然后重新执行[步骤 4](#ZH-CN_OPI_0166620115__step4)。
9. 收集信息并寻求技术支持。
    a. 在镜像接口或服务器上开启抓包工具，执行 [步骤 4](#ZH-CN_OPI_0166620115__step4) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 查看并收集对端设备配置及接口状态信息。
    e. 收集归纳所有信息并联系华为技术支持解决。
