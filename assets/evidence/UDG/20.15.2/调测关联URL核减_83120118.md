# 调测关联URL核减

- [操作场景](#ZH-CN_OPI_0183120118__1.3.1)
- [必备事项](#ZH-CN_OPI_0183120118__1.3.2)
- [操作步骤](#ZH-CN_OPI_0183120118__1.3.3)

## [操作场景](#ZH-CN_OPI_0183120118)

当运营商部署了关联URL核减，实现链接页面与主页面的关联计费、链接源访问控制时，需要调测该功能，确保该功能可正常使用。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0183120118)

前提条件

- 请仔细阅读[GWFD-020304 关联URL核减](../GWFD-020304 关联URL核减_83120116.md)。

- 完成[激活关联URL核减](激活关联URL核减_83120117.md)。
- 接入侧/PDN侧接口的镜像接口已经安装了第三方抓包工具。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | apn | apn-test | 已配置数据中获取 | 用户激活使用的APN，取自接入配置中的APN实例名。可以使用<br>[**LST APN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/查询APN配置（LST APN）_82837017.md)<br>命令进行查询。 |
| 测试终端浏览的主网站 | url | www.example.com | 已配置数据中获取 | 取自<br>[激活关联URL核减](激活关联URL核减_83120117.md)<br>中配置的URL。 |
| 测试终端浏览的链接 | url | www.huawei.com | 已配置数据中获取 | 取自<br>[激活关联URL核减](激活关联URL核减_83120117.md)<br>中配置的URL。 |

工具

- 测试终端
- OM portal跟踪工具

## [操作步骤](#ZH-CN_OPI_0183120118)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md) 命令，查询License中是否允许使用关联URL核减功能。
  ```
  LST LICENSESWITCH: LICITEM="LKV3G5RUCF01";
  ```
    - 若回显中“Switch”为“DISABLE”，请开启License后继续调测本功能。
    - 若回显中“Switch”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0183120118__step2)。
2. 打开镜像接口侧的抓包工具，准备抓取测试终端的出入报文。
3. 测试终端使用APN “apn-test” 接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 4](#ZH-CN_OPI_0183120118__step4)。
    - 如果测试终端无法接入网络，请调测UDG的接入功能。
4. 在测试终端上打开web浏览器，通过 “www.example.com” 页面上的链接访问 “www.huawei.com” 。检查镜像接口的抓包信息，查看HTTP报文头中是否携带Referer字段。
    - 如果是，请继续执行[步骤 5](#ZH-CN_OPI_0183120118__step5)。
    - 如果不是，说明HTTP报文头携带有问题，请检查浏览器设置或终端型号，确保HTTP报文头可以正常携带Referer字段。
5. 登录CG客户端，查看测试终端用户的话单，确认用户访问web业务的流量是否计入七层费率RG 30000中。
    - 如果是，说明关联URL计费功能正常，本调测任务结束。
    - 如果不是，说明关联URL计费异常，请执行[6](#ZH-CN_OPI_0183120118__cmd420455711183110)排查。
6. 执行命令 [**LST L7FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/七层规则管理/七层过滤器/查询七层过滤器（LST L7FILTER）_86526660.md) ，检查关联URL核减参数 “ISREFEREREN” 是否设为 “ENABLE” 。
  ```
  LST L7FILTER:L7FILTERNAME="l7_test";
  ```
  ```
  ......
  -------------------------
       
       七层过滤器名称  =  l7_test
     子七层过滤器名称  =  s17_test
                  URL  =  www.example.com/*
                     ......
  Referer关联计费开关  =  ENABLE
                     ......
  ---    END
  ```
    - 如果是，请继续执行[7](#ZH-CN_OPI_0183120118__cmd1207527453183110)。
    - 如果不是，请参考[激活关联URL核减](激活关联URL核减_83120117.md)正确配置，并重新调测。
7. 检查配置信息与规划值是否一致。
    a. 执行 [**LST RULEBINDING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_86526204.md) 命令，查询User Profile绑定的rule是否正确。
      ```
      LST RULEBINDING:USERPROFILENAME="up_test";
      ```
      ```
      ......
      -------------------------
      用户模板名称    规则名称     优先级    策略类型 
      up_test         rule_test1    1        PCC         
      up_test         rule_test2    2        PCC         
      (Number of results = 2)
      ---    END
      ```
          - 如果User Profile绑定的“Rule Name”和“Priority”与规划值一致，请执行[7.b](#ZH-CN_OPI_0183120118__cmd1340286638183110)。
          - 如果User Profile绑定的“Rule Name”和“Priority”与规划值不一致，请参考[激活关联URL核减](激活关联URL核减_83120117.md)重新配置。
    b. 执行 [**LST RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，查询 **RULE** 的配置信息是否正确。
      ```
      LST RULE:RULENAME="rule_test1";
      ```
      ```
      ......
      -------------------------
           规则名称  =  rule_test1
                   ......
       流过滤器名称  =  ff_test1
                   ......
      PCC策略组名称  =  ppg_test1
                   ......
      ---    END
      ```
          - 如果Rule绑定的“FLOWFILTER”和“PCC Policy Group”与规划值一致，请执行[7.c](#ZH-CN_OPI_0183120118__cmd1476006011183110)。
          - 如果Rule绑定的“FLOWFILTER”和“PCC Policy Group”与规划值不一致，请参考[激活关联URL核减](激活关联URL核减_83120117.md)重新配置。
    c. 执行 [**LST PCCPOLICYGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/查询PCC策略组（LST PCCPOLICYGRP）_82837609.md) 、 [**LST URRGROUP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则组/查询URR组（LST URRGROUP）_82837637.md) 和 [**LST URR**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则/查询URR（LST URR）_82837632.md) 命令，查询策略信息是否正确。
      ```
      LST PCCPOLICYGRP:PCCPOLICYGRPNM="ppg_test1";
      ```
      ```
      ......
      -------------------------
      PCC策略组名称  =  ppg_test1
                   ......
          URR组名称  =  cp_red
                   ......
      ---    END
      ```
      ```
      LST URRGROUP:URRGROUPNAME="cp_red";
      ```
      ```
      ......
      -------------------------
      使用量上报规则组名称  =  cp_red
         上行发起URR名称1   =  urr_red
         下行发起URR名称1   =  urr_red
                          ......
      ---    END
      ```
      ```
      LST URR:URRNAME="urr_red";
      ```
      ```
      ......
      -------------------------
      使用量上报规则名称  =  urr_red
          使用量上报模式  =  离线计费
                        ......
      ---    END
      ```
          - 如果L7Filter的“使用量上报规则组名称”、“上行发起URR名称1”与规划值一致，请执行[步骤 7.d](#ZH-CN_OPI_0183120118__substepg)。
          - 如果L7Filter的“使用量上报规则组名称”、“上行发起URR名称1”与规划值不一致，请参考[激活关联URL核减](激活关联URL核减_83120117.md)重新配置。
    d. 执行 [**LST FLOWFILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/查询流过滤器（LST FLOWFILTER）_82837364.md) 、 [**LST FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/查询过滤器（LST FILTER）_82837353.md) 命令，查询Rule绑定的Filter信息是否正确。
      ```
      LST FLOWFILTER:FLOWFILTERNAME="ff_test1";
      ```
      ```
      ......
      -------------------------
              流过滤器名称  =  ff_test1
                          ......
      ---    END
      ```
      ```
      LST FILTER:FILTERNAME="filter_test";
      ```
      ```
      ......
      -------------------------
                   过滤器名字  =  filter_test
       三四层IPv4协议输入类型  =  String Type
                             ......
               三四层协议类型  =  TCP
                             ......
             服务器起始端口号  =  80
             服务器结束端口号  =  80
                             ......
               Effective Flag  =  Yes
                             ......
      ---    END
      ```
          - 如果Rule绑定的“Filter Name”、Filter的“ Layer 3/Layer 4 Protocol Type”、“Server Start Port Number”和“Server End Port Number”与规划值一致，请执行[8](#ZH-CN_OPI_0183120118__cmd1952667090183110)。
          - 如果Rule绑定的“Filter name”、Filter的“Layer 3/Layer 4 Protocol Type”、“Server Start Port Number”和“Server End Port Number”与规划值不一致，请参考[激活关联URL核减](激活关联URL核减_83120117.md)重新配置。
8. 收集信息并寻求技术支持。
    a. 在镜像接口或服务器上开启抓包工具，执行 [步骤 4](#ZH-CN_OPI_0183120118__step4) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
