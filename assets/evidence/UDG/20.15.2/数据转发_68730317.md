#### [数据转发流程](#ZH-CN_CONCEPT_0268730317)

在PDU Session建立时，SMF会向UPF发起会话建立，通过PFCP Session Estabishment Request和PFCP Session Estabishment Response消息完成会话建立。在SMF发送的PFCP Session Estabishment Request消息中会携带一组PDR，后续数据流的转发路径会根据PDR的匹配情况进行转发，如 [图1](#ZH-CN_CONCEPT_0268730317__fig1526125725211) 所示。

**图1** 数据转发示意图

<br>

![](数据转发_68730317.assets/zh-cn_image_0268730383.png)

1. SMF向UPF下发一组PDR，UPF收到的这一组PDR均用于后续数据报文转发的匹配条件。
  这组PDR中，至少有一个用于从access到core方向的上行数据流的规则匹配。也至少有一个用于从core到access方向的下行数据流的规则匹配。
2. 用户数据报文发送至UPF。
3. UPF对用户报文进行PDR（Packet Detection Rule ）匹配，查找用户所属PDU（Protocol Data Unit） Session。
4. UPF根据PDR优先级、报文中的关键信息（如域名、IP等）进行PDR匹配，命中PDR。
5. UPF根据PDR中的相关信息或者本地配置的相关规则信息等，执行数据流的相关动作并转发。

#### [PDR概述](#ZH-CN_CONCEPT_0268730317)

**信息构成**

PDR的信息构成如 [图2](#ZH-CN_CONCEPT_0268730317__fig17832541161813) 所示。

**图2** PDR的信息构成图

<br>

![](数据转发_68730317.assets/zh-cn_image_0268730400.png)

流处理策略PDR主要包括流规则PDI（Packet Detection Information）和流动作信息。PDR主要参数信息如下。

- PDI：流规则，包含报文方向（source interface：0上行，1下行） 和 filter（SDF Filter/Application ID）。当入口数据流量的对应字段与PDI中的所有匹配条件都匹配成功则命中。
- 流动作：指明需要对数据流量执行的动作，包含以下动作。
    - FAR ID：指示转发动作FAR ID（转发动作包含转发、缓存、重传、丢弃等）。
    - QER ID：指示流控类动作的QER ID（流控类动作如CAR）。
    - URR ID：指示统计上报类动作的URR ID（统计上报类动作包含配额、阈值、时长、流量、上报原因等）。
- Precedence：PDR优先级。
- Activate Predefined Rules：预定义规则名称。

**PDR类型**

PDR分为动态PDR和预定义PDR。

- 动态PDR由PDI、FAR、QER、URR、Precedence组成。
- 预定义PDR由PDI、FAR、QER、URR、Precedence和Activate Predefined Rules组成，一条预定义PDR中可以携带多个Activate Predefined Rule。
    - 一条预定义PDR可以携带多个Activate Predefined Rules。
    - Activate Predefined Rules可以是本地配置的userprofile名称或者是本地配置的rule名称。

**PDR匹配**

PDR匹配包含动态PDR和预定义PDR匹配，PDR按照优先级匹配。取优先级最高的PDR，当动态PDR与预定义PDR优先级相同时，动态PDR优先级高于预定义PDR。

动态PDR匹配方式如下。

- 动态rule：在SDF filter中携带转发匹配条件，不携带Application ID参数。命中该rule的PDR，UPF会按照PDR中对应动作执行流转发。
- ADC rule：携带Application ID，且SDF filter为“Permit out ip from any to assigned”。UPF会根据Application ID匹配本地配置的Flowfilter/Flowfilter Group。命中该rule的PDR，UPF会按照本地的Flowfilter/Flowfilter Group配置执行流动作。

预定义PDR匹配：UPF会根据Activate Predefined Rules值匹配本地配置的userprofile或本地配置的rule，命中该PDR，UPF会按照userprofile或rule进行带宽、计费等策略控制。
