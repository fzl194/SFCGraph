#### [3GPP QoS机制](#ZH-CN_TOPIC_0000001242967127)

在3GPP移动网络演进过程中，为了根据不同业务特点为用户提供满意的服务，移动网络的QoS机制也在不断发展，经历了R97/98 QoS、R99 QoS、EPS QoS、5GC QoS等几个阶段，分别应用于GPRS、UMTS、EPS、5GC等网络制式。不同QoS机制采用了不同的QoS模型，其QoS控制机制、控制粒度以及控制参数等都有所区别。

- R97/98 QoS应用于GPRS网络，定义的QoS参数有时延级别、可靠性、最大数据流量、优先级、平均数据流量、重发需求等，这些QoS参数能够在UE与SGSN、GGSN之间传递。
- R99 QoS应用于UMTS网络，引入了端到端的QoS分层体系结构，涉及UE、接入网实体、核心网实体，不同接口的QoS参数处理必须保持一致。同时，R99版本的核心网IP承载还采用了IETF定义的Diff-Serv（差分服务）等QoS技术，并根据业务对时延、丢包率的要求，定义了会话类、流类、交互类以及背景类4种业务类型。
- EPS QoS应用于EPS网络，QoS控制的基本粒度是承载（Bearer），即相同承载上的所有数据流量将获得相同的QoS保障，不同类型的承载提供不同的QoS保障。同时，针对LTE网络无线接入网共享信道机制的特征，在EPS网络的承载处理（承载创建/修改/删除）过程中，取消了网络实体间复杂的QoS协商机制。EPS的QoS基于QoS等级标识（QCI）参数，其中QCI可用来代替UMTS系统中多个参数，即eNodeB可以从QCI推导出一组参数特征，从而减少了接口间传递的信令规模。EPS QoS详见[EPS QoS](EPS QoS_72271795.md)。
- 5GC QoS应用于5GS网络，QoS控制的基本粒度是QoS流（QoS Flow），不再基于端到端承载。与EPS QoS相比，在信令控制QoS的基础上，5GC QoS增加了反射QoS控制，即UE通过下行数据包携带的信息自行推演上行QoS信息。5GC QoS详见[5GC QoS](5GC QoS_72271794.md)。

#### [各3GPP QoS使用的主要QoS参数](#ZH-CN_TOPIC_0000001242967127)

- R97/98 QoS
  *表1 R97/98 QoS参数*

  | 参数名称（中文） | 参数名称（英文） | 说明 |
  | --- | --- | --- |
  | 优先级分类 | Precedence Class | 用来指示各种业务的重要性，如在资源有限或拥塞时保证哪些数据包先传送，丢弃哪些数据包。此参数指明继续GPRS业务时相关的优先级。例如：当发生异常（如网络拥塞）时，优先级低的包将首先被丢弃。 |
  | 时延分类 | Delay Class | 数据包在网络传送过程中的端到端时延。表明在GPRS网络中传输SDUs时的端到端的时延的级别。延迟级别分为四级，级别越高，延迟时间越长。 |
  | 可靠性分类 | Reliability Class | 定义了SDUs丢失、重复、错序或发生不可检测的错误的可能性。 |
  | 峰值吞吐量 | Peak Throughput Class | 在Gi接口和R接口测量到的吞吐率（字节/秒）。定义了单个PDP报文在网络中的最大速率期望值。描述单位时间内传递的数据位的最大值。 |
  | 平均吞吐量 | Mean Throughput Class | 在Gi接口和R接口测量到的吞吐率（字节/小时）。定义了单个PDP报文（在其生命周期内）在网络中的平均速率期望值。描述单位时间内传递的数据位的平均值。 |

- R99 QoS
  *表2 R99 QoS参数*

  | 参数名称（中文） | 参数名称（英文） | 说明 |
  | --- | --- | --- |
  | 业务分类 | Traffic class | 用来区分事务流对时延的敏感程度，将业务分为会话类、流类、交互类、背景类业务四种。 |
  | 最大上行速率 | Maximum bit rate for uplink | 单位时间内在UMTS的信令接入点传递的最大比特位数。单位：kbps。 |
  | 最大下行速率 | Maximum bit rate for downlink | 最大下行比特率。单位：kbps。 |
  | 上行保证速率 | Guaranteed bit rate for uplink | 可以保证的最大上行比特率。单位：kbps。 |
  | 下行保证速率 | Guaranteed bit rate for downlink | 可以保证的最大下行比特率。单位：kbps。 |
  | 传输顺序 | Delivery order | 是否要求顺序发送SDU。 |
  | 最大SDU长度 | Maximum SDU size | SDU的最大长度，单位为字节。 |
  | SDU误码率 | SDU bit error ratio | SDU在传递过程中，丢失或错误的比率。主要用来在UTRAN内配置协议、算法和错误检测方案。 |
  | 残留的误码率 | Residual BER | 表征在SDUs传递过程中，未被发现的误码率。 |
  | 是否传输错误的SDU | Delivery of erroneous SDUs | 表征当SDUs被检测到出错时，SDUs将继续传递，还是被丢弃。 |
  | 传输时延 | Transfer delay | 传输延迟。 |
  | 业务处理优先级 | Traffic Handling priority | 事务处理的优先级，应用于交互类业务。 |
- EPS QoS
  *表3 EPS QoS参数*

  | 参数名称（中文） | 参数名称（英文） | 说明 |
  | --- | --- | --- |
  | QoS分类识别码 | QCI（QoS Class Identifier） | 作为特定分组转发行为（如丢包率和时延）的参考等级提供给业务数据流（SDF）。该分组转发行为通过预先在接入网节点（如eNodeB）配置QCI相关参数来实现。<br>支持9类QCI业务类型。每个QCI值都对应一组参数，QCI与参数的对应关系可以由运营商定义。 |
  | 保证比特速率 | GBR（Guaranteed Bit Rate） | 系统通过预留资源等方式为保证数据流的比特速率在不超过GBR时能够全部通过，超过GBR的流量可以按照如下方式处理：拥塞时超过GBR的流量会被丢弃，不拥塞时超过GBR但小于MBR的流量可以通过。<br>享有GBR资源的承载被称为GBR承载，其它的承载被称为Non-GBR承载。 |
  | 最大比特速率 | MBR（Maximum Bit Rate） | 系统通过限制流量的方式禁止数据流的比特速率超过MBR。MBR应大于等于GBR。目前受限于终端的能力，MBR都等于GBR。 |
  | 每APN聚合最大比特速率 | APN-AMBR（per APN Aggregate Maximum Bit Rate） | APN-AMBR限制了一个用户在一个APN下的所有non-GBR承载共享的最大带宽。 |
  | 每UE聚合最大比特速率 | UE-AMBR（per Session Aggregate Maximum Bit Rate） | UE-AMBR限制了一个UE的所有non-GBR承载共享的最大带宽。 |

- 5GC QoS
  *表4 5GC QoS参数*

  | 参数名称（中文） | 参数名称（英文） | 说明 |
  | --- | --- | --- |
  | 5G QoS标识 | 5QI（5G QoS Identifer） | 作为特定分组转发行为(如丢包率和包时延)的参考等级提供给业务数据流（SDF）。5QI值可以是标准化的、预配置的或动态分配的。<br>每个5QI值都对应一组参数，对应关系可以由运营商定义。 |
  | 保证流比特速率 | GFBR（Guaranteed Flow Bit Rate） | 系统通过预留资源等方式为保证数据流的比特速率在不超过GFBR时能够全部通过，超过GFBR的流量可以按照如下方式处理：拥塞时超过GFBR的流量会被丢弃，不拥塞时超过GFBR但小于MFBR的流量可以通过。 |
  | 最大流比特速率 | MFBR（Maximum Flow Bit Rate） | 系统通过限制流量的方式禁止数据流的比特速率超过MFBR。MFBR应大于等于GFBR。 |
  | 每会话聚合最大比特速率 | Session-AMBR（per Session Aggregate Maximum Bit Rate） | Session-AMBR限制了一个PDU会话的所有non-GBR QoS flow共享的最大带宽。 |
  | 每UE聚合最大比特速率 | UE-AMBR（per Session Aggregate Maximum Bit Rate） | UE-AMBR限制了一个用户的所有non-GBR QoS flow共享的最大带宽。 |
  | 反射QoS属性 | RQA（Reflective QoS Attribute） | 反射QoS属性，是QoS流的可选参数，用于指示该QoS流承载的部分流量（不一定是全部流量）受反射QoS控制。 |
  | 通知控制 | Notification control | 当(R)AN侧不能满足GFBR时通知SMF，5GC发起N2信令流程来修改或移除QoS flow；当条件改善能够再次执行GFBR。 |
