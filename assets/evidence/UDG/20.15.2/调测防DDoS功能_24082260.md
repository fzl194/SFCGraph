# 调测防DDoS功能

- [操作场景](#ZH-CN_OPI_0124082260__1.4.1)
- [必备事项](#ZH-CN_OPI_0124082260__1.4.2)
- [操作步骤](#ZH-CN_OPI_0124082260__1.4.3)

## [操作场景](#ZH-CN_OPI_0124082260)

当运营商需要部署DDoS功能防止移动用户对PDN侧网络设备进行DDoS（Distributed Denial of Service）攻击时，需对 UDG 的DDoS功能进行调测，确保功能可以正常使用。

> **说明**
> 适用于SGW-U、PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0124082260)

前提条件

- 请仔细阅读 [GWFD-010253 防DDoS功能](../GWFD-010253 防DDoS功能_67574063.md) 。
- 完成[激活防DDoS功能](激活防DDoS功能_24082259.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN名称（APN） | apn-test | 已配置数据中获取 | 可以使用<br>[**LST APN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/查询APN配置（LST APN）_82837017.md)<br>命令进行查询。 |
| 使能防范DDoS功能的UserProfile | 用户模板名称（USERPROFILENAME） | up_test | 已配置数据中获取 | 取自<br>[激活防DDoS功能](激活防DDoS功能_24082259.md)<br>中配置的<br>“使能防范DDoS功能的UserProfile”<br>。此用户模板名称是SGW-C/PGW-C/SMF下发给<br>UDG<br>的，并且需要跟<br>UDG<br>上配置的名称一致。 |
| DDoS阈值参数 | DDoS防护阈值（THRESHOLD） | 10 | 已配置数据中获取 | 取自<br>[激活防DDoS功能](激活防DDoS功能_24082259.md)<br>中配置的<br>“防范DDoS攻击使用的流控阈值”<br>。 |

工具

- 测试终端
- 第三方抓包工具

## [操作步骤](#ZH-CN_OPI_0124082260)

1. 测试终端 使用"apn-test"APN接入网络。
2. 执行 [**DSP SESSIONINFO**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，通过 测试终端 的IMSI查看 测试终端 是否激活成功。
  ```
  DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
  ```
  ```
  ......
                                                           
  ```
  ```
  -------------------------------                                                     
  ```
  ```
                           IMSI  =  460000123456789                                    
  ```
  ```
               IPv4 PDP address  =  10.0.0.1                                           
  ```
  ```
                              ......                                                    
  ```
  ```
                       APN name  =  apn-test                                           
  ```
  ```
                              ......                                                    
  ```
  ```
  (Number of Results  =  1)                                                            
  ```
    - 如果测试终端激活成功，且使用的“APN name”与规划值一致，请记录测试终端激活使用的“IPv4 PDP address”，再执行[步骤 3](#ZH-CN_OPI_0124082260__step3)。
    - 如果测试终端激活成功，但使用的“APN name”与规划值不一致，请返回[步骤 1](#ZH-CN_OPI_0124082260__step1)使用规划的APN重新接入网络。
3. 打开接入侧/DN侧镜像接口上的抓包工具，准备抓取测试终端的出入报文。 将过滤条件设置为 测试终端 激活使用的 “PDP address” ，如 [图1](#ZH-CN_OPI_0124082260__fig1) 所示。
  **图1** 用户抓包信息

  <br>

  ![](调测防DDoS功能_24082260.assets/fig_1_4c201.png "点击放大")
4. 测试终端 发送大量TCP链接报文（SYN报文），报文速率超过配置阈值，并且持续3秒钟以上。查看 UDG 是否会将用户的TCP链接报文（SYN报文）的转发速率限制在配置的阈值以内。
  **图2** 工具计算出的包速率

  <br>

  ![](调测防DDoS功能_24082260.assets/fig_1_4c202.png "点击放大")
  > **说明**
  > DDoS功能允许3秒钟突发，所以前3秒内包速率较大。建议在发送报文3秒后查看抓包信息，以保证结果更精确。
    - 如果转发的用户SYN报文的速率小于配置的DDoS阈值，如[图2](#ZH-CN_OPI_0124082260__fig2)所示，实际包速率为7.088 packets/sec，小于DDoS阈值10，DDoS功能正常，调测结束。
    - 如果转发的用户SYN报文的速率大于配置的阈值，请执行[步骤 5](#ZH-CN_OPI_0124082260__step6)。
5. 执行 [**LST USERPROFILE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/用户模板/查询用户模板（LST USERPROFILE）_82837286.md) 命令，检查UserProfile的DDoS功能是否使能。
  ```
  LST USERPROFILE:USERPROFILENAME="up_test";

  ......

  -------------------------                                                                                                           
           用户模板名称  =  up_test                                      
  Alias Marking使能标记  =  不使能                                              
         防DDoS攻击标记  =  使能                                                       
                       ......                       
  (结果个数 = 1)                                         
  ```
    - 如果“防DDoS攻击标记”为“ENABLE”，请执行[步骤 6](#ZH-CN_OPI_0124082260__step7)。
    - 如果“防DDoS攻击标记”为“DISABLE”，请参考[激活防DDoS功能](激活防DDoS功能_24082259.md)重新配置。然后重新执行[步骤 4](#ZH-CN_OPI_0124082260__step4)。
6. 执行 [**LST DDOS**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务安全防护/用户攻击防护/DDoS防护/DDoS防护阈值/查询DDoS防攻击流控阈值（LST DDOS）_82837755.md) 命令，检查DDoS阈值当前配置与规划值是否一致。
  ```
  LST DDOS:;

  ......
                                                
  -------------------------                                                 
  DDoS防护阈值  =  10                                                
  (结果个数 = 1)                                       
  ```
    - 如果“DDoS防护阈值”与规划值一致，请执行[步骤 7](#ZH-CN_OPI_0124082260__step8)。
    - 如果“DDoS防护阈值”与规划值不一致，请参考[激活防DDoS功能](激活防DDoS功能_24082259.md)重新配置。然后重新执行[步骤 4](#ZH-CN_OPI_0124082260__step4)。
7. 收集信息并寻求技术支持。
    a. 在镜像接口或服务器上开启抓包工具，执行 [步骤 4](#ZH-CN_OPI_0124082260__step4) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
