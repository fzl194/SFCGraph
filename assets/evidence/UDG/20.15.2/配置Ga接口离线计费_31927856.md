## [操作场景](#ZH-CN_OPI_0000001331927856)

离线计费中， UDG 支持按用户使用的业务进行离线计费， UDG 与UNC消息交互时需要用到URR ID，上报计费信息。

> **说明**
> 适用于SGW-U、PGW-U。

## [操作步骤](#ZH-CN_OPI_0000001331927856)

配置业务级离线计费的费率标识。

1. 配置内容计费使用的URR Group。
    a. 配置URR。
      **ADD URR**
    b. 配置上行和下行发起的业务对应的URR。
      **ADD URRGROUP**
2. 配置内容计费使用的PCC策略组。
  **ADD PCCPOLICYGRP**
3. 配置过滤条件。
    a. 配置三四层过滤条件。
      **ADD FILTER**
    b. 配置流过滤器。
      **ADD FLOWFILTER**
    c. 绑定三四层过滤条件。
      **ADD FLTBINDFLOWF**
4. 配置内容计费使用的规则。
  **ADD RULE**
5. 配置内容计费规则绑定用户模板。
    a. 创建UserProfile实例。
      **ADD USERPROFILE**
    b. 将已配置的内容计费规则绑定到UserProfile。
      **ADD RULEBINDING**
6. **可选** ：设置URR上报失败后的动作处理参数。
  ****SET URRFAILACTION****
  > **说明**
  > - 该命令在离线计费新业务场景下生效。
  > - 当业务部署调整导致SGW-U/PGW-U的URR ID在SGW-C/PGW-C上不存在时(比如，先添加SGW-U/PGW-U上**ADD URR**，后添加SGW-C/PGW-C上**ADD URR**)，会导致SGW-C/PGW-C无法处理SGW-U/PGW-U的计费用量上报。为减小该场景下对用户业务的影响，可通过该命令设置SGW-U/PGW-U向SGW-C/PGW-C进行计费用量上报失败时的处理动作。
  > - 当计费URR上报失败，业务被阻塞时，如果想要放通业务，可以执行**SET URRFAILACTION**命令，配置“RETRYFAILACT”参数为“CONTINUE”，则URR对应的业务将不计费。在SGW-U/PGW-U向SGW-C/PGW-C进行用量上报恢复后，将恢复对该URR的计费。

## [任务示例](#ZH-CN_OPI_0000001331927856)

任务描述

用户进行离线计费，统计到URR ID为1000的URR中。

脚本

//配置离线计费的费率标识

```
ADD URR:URRNAME="offlineURR",URRID=1000,USAGERPTMODE=OFFLINE,OFFMETERINGTYPE=EVENT_VOLUME_TIME;
ADD URRGROUP:URRGROUPNAME="urrgroup1",UPURRNAME1="offlineURR",DOWNURRNAME1="offlineURR";
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="cg-test",URRGROUPNAME="urrgroup1";
```

//配置三四层信息

```
ADD FILTER:FILTERNAME="filter_red",L34PROTTYPE=STRING,L34PROTOCOL=TCP,SVRSTARTPORT=80,SVRENDPORT=80;
ADD FLOWFILTER:FLOWFILTERNAME="fg_red";
ADD FLTBINDFLOWF:FLOWFILTERNAME="fg_red",FILTERNAME="filter_red";
```

//配置内容计费规则

```
ADD RULE:RULENAME="Rule001",POLICYTYPE=PCC,FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="fg_red",POLICYNAME="cg-test",PRIORITY=110;
SET REFRESHSRV:REFRESHTYPE=USERPROFILE;
```

//配置内容计费规则绑定用户模板

```
ADD USERPROFILE:USERPROFILENAME="up_test";
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="Rule001";
```
