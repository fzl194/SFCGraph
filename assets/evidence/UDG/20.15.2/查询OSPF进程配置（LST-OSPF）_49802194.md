#### [命令功能](#ZH-CN_CONCEPT_0000001549802194)

该命令用于查询OSPF特性的进程的相关配置。

#### [注意事项](#ZH-CN_CONCEPT_0000001549802194)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001549802194)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001549802194)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROCID | 进程号 | 可选必选说明：可选参数<br>参数含义：OSPF进程号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～4294967295。<br>默认值：无 |

#### [使用实例](#ZH-CN_CONCEPT_0000001549802194)

查询OSPF特性的进程的相关配置：

```
LST OSPF:PROCID=1;
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000001549802194)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 进程号 | 该参数指定用户创建的OSPF进程号。 |
| VPN名称 | 该参数指定虚拟专用网(VPN)实例名称。 |
| 抑制接口收发OSPF报文 | 该参数指定是否抑制OSPF进程下所有接口收发OSPF报文标志。 |
| 使能BFD | 该参数指定是否为OSPF进程使能动态创建BFD会话特性。 |
| 是否配置BFD最小接收报文间隔 | 该参数指定是否配置了BFD最小接收报文间隔。 |
| 接收BFD包的最小时间间隔（ms） | 该参数指定期望从对端接收BFD报文的最小接收间隔，单位是毫秒，当是否配置BFD最小接收报文间隔为true时生效。 |
| 是否配置BFD最小发送报文间隔 | 该参数指定是否配置了BFD最小发送报文间隔。 |
| 发送BFD包的最小时间间隔（ms） | 该参数指定向对端发送BFD报文的最小发送间隔，单位是毫秒，当是否配置BFD最小发送报文间隔为true时生效。 |
| BFD检测乘数 | 该参数指定会话检测倍数，如果BFD会话在配置的检测周期内没有收到对端发来的BFD报文，则认为链路发生了故障，BFD会话的状态将会置为Down。 |
| 兼容RFC1583 | 该参数指定是否使能了兼容RFC1583的路由选择优先规则。 |
| VPN引入路由的tag值 | 该参数指定绑定VPN的OSPF进程引入外部路由的tag值，当配置VPN引入路由的tag值为true时生效。 |
| 计算链路开销参考值（Mbps） | 该参数指定计算链路开销时所依据的参考值， 单位是兆比特每秒。OSPF接口默认开销值的计算公式为：接口开销=带宽参考值/接口带宽，取计算结果的整数部分作为接口开销值（当结果小于1时取1）。 |
| 使能最大重传数 | 该参数指定是否使能了最大重传上限数目。 |
| 最大重传限制 | 该参数指定限制OSPF报文重传次数，并在达到指定的次数后断开邻居，避免在邻居收不到报文的情况下，一直重传造成死循环。当使能最大重传数为true时生效。 |
| Opaque LSA能力 | 该参数指定是否使能了Opaque LSA能力，Opaque LSA提供用于OSPF扩展的通用机制，包括OSPF GR功能。 |
| 使能可达性抑制 | 该参数指定是否使能了可达性抑制功能，抑制接口地址发布，从而可以重用接口地址。 |
| 接收时间间隔（ms） | 该参数指定LSA被接收的时间间隔，当取消使用智能定时器配置LSA接收的间隔时间为true时生效，单位是毫秒。 |
| 接收最长间隔时间（ms） | 该参数指定接收OSPF LSA的最长间隔时间，当取消使用智能定时器配置LSA接收的间隔时间为false时生效，单位是毫秒。 |
| 接收初始间隔时间（ms） | 该参数指定接收OSPF LSA的初始间隔时间，当取消使用智能定时器配置LSA接收的间隔时间为false时生效，单位是毫秒。 |
| 接收基数间隔时间（ms） | 该参数指定接收OSPF LSA的基数间隔时间，当取消使用智能定时器配置LSA接收的间隔时间为false时生效，单位是毫秒。 |
| 取消使用智能定时器配置LSA接收的间隔时间 | 该参数指定是否取消使用智能定时器配置LSA接收的间隔时间。 |
| 更新时间间隔（s） | 该参数指定配置除OSPF Router LSA和Network LSA外LSA的更新间隔时间，单位是秒。 |
| 更新最长间隔时间（ms） | 该参数指定更新Router LSA和Network LSA的最长间隔时间，单位是毫秒。 |
| 更新初始间隔时间（ms） | 该参数指定更新Router LSA和Network LSA的初始间隔时间，单位是毫秒。 |
| 更新基数间隔时间（ms） | 该参数指定更新Router LSA和Network LSA的基数间隔时间，单位是毫秒。 |
| 是否指定LSA更新的时间间隔为0 | 该参数指定是否取消默认的LSA的5秒的更新时间间隔。 |
| 去使能VPN路由环路检测 | 该参数指定禁止路由环路检测，直接进行路由计算。 |
| Stub路由器 | 该参数指定配置本进程为Stub路由器，No Config - 不配置；Stub On Hand - 配置Stub路由器； Stub On Boot - 主备倒换中为Stub路由器。 |
| On-Startup时间间隔（s） | 该参数指定路由器在发生主备倒换过程中保持为Stub路由器的时间间隔，当stub路由器配置为StubOnBoot时生效，单位是秒。 |
| 安全同步 | 该参数指定在OSPF与BGP联动场景中是否配置安全同步功能。 |
| 配置VPN引入路由的tag值 | 该参数指定VPN场景下OSPF进程引入的外部路由的tag值。 |
| SPF计算间隔时间（s） | 该参数指定OSPF的SPF计算间隔时间，单位是秒。 |
| SPF计算间隔时间（ms） | 该参数指定OSPF的SPF计算间隔时间，单位是毫秒。 |
| SPF计算最长间隔时间（ms） | 该参数指定通过智能定时器配置OSPF SPF计算的最长间隔时间，单位是毫秒。 |
| SPF计算初始间隔时间（ms） | 该参数指定通过智能定时器配置OSPF SPF计算的初始间隔时间，单位是毫秒。 |
| SPF计算基数间隔时间（ms） | 该参数指定通过智能定时器配置OSPF SPF计算的基数间隔时间，单位是毫秒。 |
| 禁止使用tag检测环路 | 该参数指定禁止使用VPN的路由标记检测环路。 |
| 使能ECA路由类型 | 该参数指定为VPN场景使能的ECA路由类型，以避免因不能正确识别OSPF VPN扩展团体属性的路由类型而导致不能正确解析的问题。 |
| 配置Domain ID为空 | 该参数指定删除OSPF进程下所有Domain ID配置。 |
| 路由器标识 | 该参数指定OSPF进程的路由器标识。 |
| 描述信息 | 该参数指定OSPF进程的描述信息。 |
| SPF正常模式标志 | 该参数指定使能SPF计算的保守模式，切换智能定时器SPF的时间间隔。 |
| 使能Sham-hello | 该参数指定使能Sham-hello功能，设备不仅通过Hello报文维持邻居关系，还可以通过LSU报文和LSAck报文维持邻居关系，从而更灵敏地感知OSPF邻居的存在，使邻居关系更加稳定。 |
| 是否自动使能MPLS LDP功能 | 该参数指定OSPF进程下所有可建立OSPF邻居的接口自动使能MPLS LDP的功能。 |
| OSPF共网段虚拟系统使能标志 | 该参数指定使能OSPF共网段虚拟系统功能。 |
| BFD会话与接口链路状态绑定标志 | 该参数指定将BFD会话状态与OSPF Auto FRR进行绑定。BFD检测到接口链路故障后，BFD会话状态会变为Down并触发系统进行快速重路由，将流量从故障链路切换到备份链路上，从而达到流量保护的目的。 |
| 是否为Stub链路发布最大开销值 | 该参数指定设备为Router LSA中的stub链路配置最大开销65535。 |
