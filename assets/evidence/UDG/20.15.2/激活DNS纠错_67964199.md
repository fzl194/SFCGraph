## [操作场景](#ZH-CN_OPI_0167964199)

运营商规划要求用户在访问internet时出现页面不存在的情况下，能引导用户访问第三方Platform，则需要在 UDG 上部署 DNS纠错 功能。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0167964199)

前提条件

- 请仔细阅读 [GWFD-110283 DNS纠错](../GWFD-110283 DNS纠错_67327493.md) 。
- 完成[激活PCC基本功能](../GWFD-020351 PCC基本功能/激活PCC基本功能_74096528.md)。

## [操作步骤](#ZH-CN_OPI_0167964199)

1. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)
2. **可选：**解析七层协议时，需要加载协议特征库文件。
  [**LOD SIGNATUREDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/识别特征库/加载协议特征库文件（LOD SIGNATUREDB）_82837716.md)
  > **说明**
  > 如果已经加载协议特征库文件，则使用本命令升级到最新版本的协议特征库文件。
3. 配置 DNS纠错 业务使用的三四层过滤条件。
    a. 配置 DNS纠错 业务使用的三四层过滤规则。
      [**ADD FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/增加过滤器（ADD FILTER）_82837348.md)
    b. 配置流过滤器。
      [**ADD FLOWFILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/增加流过滤器（ADD FLOWFILTER）_82837361.md)
    c. 绑定三四层过滤信息。
      [**ADD FLTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的过滤器绑定/增加流过滤器的过滤器绑定关系（ADD FLTBINDFLOWF）_82837366.md)
    d. 将新配置的Filter置为生效。
      [**SET REFRESHSRV**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/业务刷新/业务刷新（SET REFRESHSRV）_82837355.md)
4. 配置 DNS纠错 使用的扩展过滤器。
  [**ADD EXTENDEDFILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/扩展过滤器/增加扩展过滤器（ADD EXTENDEDFILTER）_82837379.md)
  > **说明**
  > 重复执行本步骤可以增加多个不同过滤条件的扩展过滤器。
5. 配置DNS错误码。
  [**ADD ERRORCODE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/重定向控制/重定向公共参数管理/ErrorCode/增加错误码（ADD ERRORCODE）_09678504.md)
6. 配置 DNS纠错 动作策略。
  [**ADD DNSOVERWRITING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/重定向控制/DNS重写控制/DNS重写/新增DNS重写动作（ADD DNSOVERWRITING）_82837549.md)
7. 配置 DNS纠错 使用的业务规则。
  [**ADD RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/增加规则（ADD RULE）_82837266.md)
  > **说明**
  > 如果运营商希望基于流过滤器组设置过滤条件时，配置 “流过滤器或流过滤器组选择(FILTERINGMODE)” 为 “FLOWFILTERGRP” ，并且使用 [**ADD FLOWFILTERGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器组/增加流过滤器组（ADD FLOWFILTERGRP）_82837384.md) 命令配置流过滤器组。

## [任务示例](#ZH-CN_OPI_0167964199)

任务描述

本例采用PCRF下发预定义规则的方式进行说明。

当用户访问internet时输入错误的域名，如www.example.com， UDG 执行DNS纠错动作，将DNS响应报文中的IP地址设置为第三方Platform的IP地址192.168.1.1，用户可以访问第三方Platform。

脚本

// 打开本特性的License配置开关。

```
SET LICENSESWITCH:LICITEM="LKV3G5DNSO01",SWITCH=ENABLE;
```

//加载协议特征库文件。

```
LOD SIGNATUREDB:LOADMODE=SPECIFIC,VERSION="01.0002.0358.01";
```

//配置 DNS纠错 使用的三四层过滤条件。

```
ADD FILTER:FILTERNAME="filter1",L34PROTTYPE=STRING,L34PROTOCOL=UDP,SVRSTARTPORT=53,SVRENDPORT=53;
```

```
ADD FLOWFILTER:FLOWFILTERNAME="flow-filter1";
```

```
ADD FLTBINDFLOWF:FLOWFILTERNAME="flow-filter1",FILTERNAME="filter1";
```

```
SET REFRESHSRV:REFRESHTYPE=USERPROFILE;
```

//配置 DNS纠错 使用的扩展过滤器。

```
ADD EXTENDEDFILTER:EXTFLTNAME="l7-filter1",URL="www.example.com/*";
```

//配置 DNS错误码 。

```
ADD ERRORCODE: ERRORCODENAME="dnserrorcode", ERRORCODEOP=EQUAL, ERRORCODESTART=3;
```

//配置 DNS纠错 动作。

```
ADD DNSOVERWRITING:DNSOVERWRTNAME="dec_1",EXTFLTTYPE1=AND,EXTFLTNAME1="l7-filter1",SERVERIP1="192.168.1.1",BINDERRCODENAME="dnserrorcode";
```

//配置 DNS纠错 规则。

```
ADD RULE:RULENAME="testrule",POLICYTYPE=SMARTREDIRECT,FILTERINGMODE=FLOWFILTER,FLOWFILTERNAME="flow-filter1",PRIORITY=20,POLICYNAME="dec_1";
```
