# 调测IM类业务无线资源管控

- [操作场景](#ZH-CN_OPI_0239436123__1.3.1)
- [必备事项](#ZH-CN_OPI_0239436123__1.3.2)
- [操作步骤](#ZH-CN_OPI_0239436123__1.3.3)

## [操作场景](#ZH-CN_OPI_0239436123)

当运营商部署了DSCP方式的IM类业务无线资源管控功能时，需要对该功能进行调测，以确保本功能可以正常使用。

> **说明**
> 适用于PGW-U。

## [必备事项](#ZH-CN_OPI_0239436123)

前提条件

- 请仔细阅读[GWFD-020359 IM类业务无线资源管控](../GWFD-020359 IM类业务无线资源管控_39436120.md)。
- 完成[激活IM类业务无线资源管控](激活IM类业务无线资源管控_39436118.md)。
- Gn/Gp镜像接口上已经安装了第三方抓包工具。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息 | 用户IMSI号（IMSI） | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN名称（APN） | apn-test | 与对端协商 | 取自<br>[激活IM类业务无线资源管控](激活IM类业务无线资源管控_39436118.md)<br>中配置的APN。 |

工具

- 测试终端
- 第三方抓包工具

## [操作步骤](#ZH-CN_OPI_0239436123)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令，查询License中是否允许使用IM类业务无线资源管控功能。
  > **说明**
  > 如下以检查IM类业务无线资源管控特性的license为例，依赖特性也需要检查。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5ITSR01";

  ......

  -------------------------
         License项  =  LKV3G5ITSR01
       License名称  =  IM类业务无线资源管控
              开关  =  打开
  (结果个数 = 1)
  ---    END
  ```
    - 如果License的“开关”为“打开”，请执行[步骤 2](#ZH-CN_OPI_0239436123__step2)。
    - 如果License的“开关”为“关闭”，则请打开本特性对应的License配置开关。
2. 打开 UDG PDN侧/接入侧镜像接口上的抓包工具，准备抓取测试终端的出入报文。
3. 测试终端使用 “apn-test” APN接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 4](#ZH-CN_OPI_0239436123__step13471154725015)。
    - 如果测试终端无法接入网络，调测UDG的接入功能。
4. 测试终端访问QQ业务，检查跟踪报文中的ToS值是否为AF12。
    - 如果是，说明UDG的SA识别及remark生效，本功能正常，调测完成。
    - 如果不是，请执行[步骤 5](#ZH-CN_OPI_0239436123__step1462111517301)。
  > **说明**
  > 如果存在多个IM类业务，请逐个业务进行调测。此处以调测QQ业务为例进行介绍。
5. 执行 [**LST USERPROFILE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/用户模板/查询用户模板（LST USERPROFILE）_82837286.md) 命令，查看Alias Marking功能是否关闭。
  ```
  LST USERPROFILE:USERPROFILENAME="up_test";
  ```
  ```
  ...... 
  ------------------------- 
                         用户模板名称  =  up_test
                Alias Marking使能标记  =  不使能
                                     ......                  
  (结果个数 = 1) 
  ---    END
  ```
    - 如果“Alias Marking使能标记”为“不使能”，请执行[步骤 6](#ZH-CN_OPI_0239436123__step7)。
    - 如果“Alias Marking使能标记”为“使能”，请参考[激活IM类业务无线资源管控](激活IM类业务无线资源管控_39436118.md)禁用Alias marking功能，然后返回[步骤 3](#ZH-CN_OPI_0239436123__step3)重新调测。
6. 执行 [**DSP SIGNATUREDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/识别特征库/查询协议特征库文件加载状态（DSP SIGNATUREDB）_82837717.md) 命令，查询知识库配置，确认 UDG 是否可以识别报文的七层协议类型。
  ```
  DSP SIGNATUREDB:;
  ```
  ```
  RETCODE = 0  操作成功。
  协议识别特征库信息
  ------------------
   特征库加载模式  =  更新为最新的特征库版本
     特征库版本号  =  01.0002.1063.01
   特征库加载状态  =  加载完成
  (结果个数 = 1) 
  ---    END
  ```
    - 如果协议解析的相关组件运行正常，可以识别报文的七层协议类型，请执行[步骤 7](#ZH-CN_OPI_0239436123__step5)。
    - 如果SIG知识库未正常加载，请参考[激活IM类业务无线资源管控](激活IM类业务无线资源管控_39436118.md)加载知识库，然后返回[步骤 3](#ZH-CN_OPI_0239436123__step3)重新调测。
7. 执行 [**DSP SESSIONINFO**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，通过测试终端的IMSI查看测试终端激活时使用的APN和用户模板。
  ```
  DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
  ```
    - 如果测试终端使用的“IPv4 PDP address”/“IPv6 PDP address”、“MSISDN”、“APN name”与规划值一致，请执行[步骤 8](#ZH-CN_OPI_0239436123__step14669123412113)。
    - 如果测试终端使用的“IPv4 PDP address”/“IPv6 PDP address”、“MSISDN”、“APN name”与规划值不一致，请参考[激活IM类业务无线资源管控](激活IM类业务无线资源管控_39436118.md)重新配置，然后返回[步骤 3](#ZH-CN_OPI_0239436123__step3)重新调测。
8. 检查IM类业务无线资源管控的相关配置是否正确。
    a. 执行 [**LST RULEBINDING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_86526204.md) 命令，查询用户模板和规则的绑定关系是否正确。
      ```
      LST RULEBINDING:USERPROFILENAME="up_test";

      ......

      -------------------------
      用户模板名称       规则名称      优先级         策略类型       
      up_test            rule_IM_01    20             REMARK_FPI
      (结果个数 = 2)
      ---    END
      ```
          - 如果用户模板名称绑定的“规则名称”与规划值一致，请执行[步骤 8.b](#ZH-CN_OPI_0239436123__substepd)。
          - 如果用户模板名称绑定的“规则名称”与规划值不一致，请参考[激活IM类业务无线资源管控](激活IM类业务无线资源管控_39436118.md)重新配置。
    b. 执行 [**LST RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，查询Rule的配置信息是否正确。
      ```
      LST RULE:RULENAME="rule_IM_01";

      ......

      -------------------------
                                   规则名称  =  rule_IM_01
                                   策略类型  =  REMARK_FPI
                                          ......
                               流过滤器名称  =  flowfilter_IM_01
                                          ......
                                 全局优先级  =  20
                              PCC策略组名称  =  NULL
                               分类属性名称  =  NULL
                                          ......
                            重标记或FPI选择  =  重标记                                
                             Remark配置类型  =  CLASS                                 
                             Remark分类类型  =  AF12 
                                          ......
      (结果个数 = 1)
      ---    END
      ```
          - 如果Rule的配置信息与规划值一致，请执行[步骤 8.c](#ZH-CN_OPI_0239436123__substepe)。
          - 如果Rule绑的配置信息与规划值不一致，请参考[激活IM类业务无线资源管控](激活IM类业务无线资源管控_39436118.md)重新配置。
    c. 执行 [**LST PROTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的协议绑定/查询流过滤器协议绑定关系（LST PROTBINDFLOWF）_82837373.md) 和 [**LST FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/查询过滤器（LST FILTER）_82837353.md) 命令，查询Rule绑定的三四层和七层过滤信息是否正确。
      ```
      LST PROTBINDFLOWF:FLOWFILTERNAME="flowfilter_IM_01";

      ......

      -------------------------
         流过滤器名称  =  flowfilter_IM_01
             协议名称  =  qq_im
      (结果个数 = 1)
      ---    END
      ```
      ```
      LST FILTER:FILTERNAME="filter_IM_any";

      ......

      -----------------------------------------------------------
                                     过滤器名字  =  filter_IM_any
                         三四层IPv4协议输入类型  =  字符串类型
                         三四层IPv6协议输入类型  =  NULL
                             三四层协议数字类型  =  0
                                 三四层协议类型  =  ANY
                                              ......
                                       生效标记  =  是
      (结果个数 = 1)
      ---    END
      ```
          - 如果Rule绑定的三四层和七层过滤信息与规划值一致，请执行[步骤 9](#ZH-CN_OPI_0239436123__step8)。
          - 如果Rule绑定的三四层和七层过滤信息与规划值不一致，请参考[激活IM类业务无线资源管控](激活IM类业务无线资源管控_39436118.md)重新配置，然后返回[步骤 3](#ZH-CN_OPI_0239436123__step3)重新调测。
9. 收集信息并寻求技术支持。
    a. 在镜像接口或服务器上开启抓包工具，执行 [步骤 3](#ZH-CN_OPI_0239436123__step3) 和 [步骤 4](#ZH-CN_OPI_0239436123__step13471154725015) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
