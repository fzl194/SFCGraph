# 调测HTTPS业务行为识别特性

- [操作场景](#ZH-CN_OPI_0000001256158489__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001256158489__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001256158489__1.3.3)

## [操作场景](#ZH-CN_OPI_0000001256158489)

当运营商完成部署HTTPS业务行为识别功能时，需要调测此功能，确保此功能可正常使用。

## [必备事项](#ZH-CN_OPI_0000001256158489)

前提条件

- 请仔细阅读[GWFD-110404 HTTPS业务行为识别](../GWFD-110404 HTTPS业务行为识别_56271761.md)。
- 完成[激活HTTPS业务行为识别特性](激活HTTPS业务行为识别特性_56038531.md)。
- 接入侧/PDN侧接口的镜像接口已经安装了第三方抓包工具。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN名称（APN） | apn-test | 已配置数据中获取 | 已通过<br>[**ADD APN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/添加APN配置（ADD APN）_82837014.md)<br>命令配置，可通过<br>[**LST APN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/查询APN配置（LST APN）_82837017.md)<br>命令查询。 |

工具

- 测试终端
- 第三方抓包工具

## [操作步骤](#ZH-CN_OPI_0000001256158489)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令，查询license中是否允许使用 HTTPS业务行为识别 功能。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5HSBI01";  
  RETCODE = 0  操作成功

  操作结果如下
  ------------
           License项  =  LKV3G5HSBI01
         License名称  =  HTTPS业务行为识别
                开关  =  打开
   (结果个数 = 1)  
   ---    END
  ```
    - 如果License的“开关”为“打开”，请执行[2](#ZH-CN_OPI_0000001256158489__step2)。
    - 如果License的“开关”为“关闭”，则执行[**SET LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)命令打开本特性对应的License配置开关。
2. 在OM Portal上 **[创建UDG用户跟踪任务](../../../../../网络运维/日常维护/UDG基础运维操作/创建消息跟踪/创建用户跟踪/创建UDG用户跟踪任务_88792206.md)** ，在定位条件栏输入用户IMSI ，在 “消息类型” 栏选择UP DATA、DOWN DATA、EMS SA消息类型，准备跟踪 测试终端 的出入报文。或者打开镜像接口上的抓包工具，准备抓取 测试终端 的出入报文。
3. 测试终端 使用APN “apn-test” 接入网络。
    - 如果测试终端成功接入网络，请执行[4](#ZH-CN_OPI_0000001256158489__step4)。
    - 如果测试终端无法接入网络，调测UDG的接入功能。
4. 执行 [**LST SUPPFINGERIDEN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/SA指纹识别/支持SA指纹识别的协议_64084148.md) 命令，查询当前 UDG 支持进行 HTTPS业务行为识别 功能的协议。
  ```
  LST SUPPFINGERIDEN:;
  RETCODE = 0  操作成功。

  支持SA指纹识别协议信息
  ----------------------
  应用子协议名称            应用协议名称

  a000dn_data               a000dn          
  (结果个数 = 1)
  ---    END
  ```
    - 如果当前UDG支持进行HTTPS业务行为识别功能的协议与规划值一致，请执行[5](#ZH-CN_OPI_0000001256158489__step5)。
    - 如果当前UDG支持进行HTTPS业务行为识别功能的协议与规划值不一致，请参考[激活HTTPS业务行为识别特性](激活HTTPS业务行为识别特性_56038531.md)重新配置后再执行本操作。
5. 观察EMS SA中 **fpsubprotocol、fpprotocol、isHttpsFraudFlag** 字段。
    - 如果上述字段有值，HTTPS业务行为识别功能正常，请[调测业务感知](../../../../业务专题/UDG业务感知专题/业务感知配置/调测业务感知_87804007.md)。
    - 如果上述字段无值，请执行[6](#ZH-CN_OPI_0000001256158489__step6)。
6. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务，执行 [2](#ZH-CN_OPI_0000001256158489__step2) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
