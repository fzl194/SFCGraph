# 调测基于业务的Shaping

- [操作场景](#ZH-CN_OPI_0183195648__1.3.1)
- [必备事项](#ZH-CN_OPI_0183195648__1.3.2)
- [操作步骤](#ZH-CN_OPI_0183195648__1.3.3)

## [操作场景](#ZH-CN_OPI_0183195648)

当用户进行P2P业务时，用户的访问流量需要控制在512kbps内。

## [必备事项](#ZH-CN_OPI_0183195648)

前提条件

- 已完成[激活基于业务的Shaping](激活基于业务的Shaping_83195645.md)。
- UDG接入侧/PDN侧的镜像接口已经安装了第三方抓包工具。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN | huawei.com | 已配置数据中获取 | 取自<br>[激活基于业务的Shaping](激活基于业务的Shaping_83195645.md)<br>中配置的APN实例名。 |
| 测试终端使用的BWMUSERGROUP | USERGROUPNAME | usergroup_test | 已配置数据中获取 | 取自<br>[激活基于业务的Shaping](激活基于业务的Shaping_83195645.md)<br>中配置的BwmUserGroup实例名。 |

## [操作步骤](#ZH-CN_OPI_0183195648)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令查询 UDG 上本特性对应的License配置开关是否打开。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5SBTS01";
  ```
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0183195648__step1)。
    - 如果“SWITCH”为“DISABLE”，则执行[**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md)命令打开本特性对应的License配置开关。
2. 执行 [**DSP SIGNATUREDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/识别特征库/查询协议特征库文件加载状态（DSP SIGNATUREDB）_82837717.md) 查询协议特征库文件加载状态。若加载成功则继续 [步骤 3](#ZH-CN_OPI_0183195648__step713692133183111) ，加载失败则检查特征库文件无误后重新加载。
  若加载成功，显示如下：
  ```
  协议识别特征库信息
  ------------------
     特征库加载模式  =  更新为最新的特征库版本
       特征库版本号  =  01.0002.1071.01
     特征库加载状态  =  加载完成
  (结果个数 = 1)
  ---   END
  ```
3. 执行 [**LST BWMUSERGROUP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理用户组/查询带宽管理用户组（LST BWMUSERGROUP）_82837471.md) 查看BwmUserGroup的配置是否正确。
  ```
  LST BWMUSERGROUP: QRYUSRGRPTYPE=SPECIFIC_GROUP, USERGROUPNAME="usergroup_test";
  ```
  ```
  用户组信息
  ----------
          用户组名称  =  usergroup_test              
          用户组类型  =  Specific User Group         
        用户组优先级  =  2      
      用户级业务类型  =  TOS
    用户组级业务类型  =  TOS
  (结果个数 = 1)
  ---    END
  ```
4. 执行 [**LST APNBINDBWMUSRG**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理用户组APN绑定/查询带宽管理用户组APN绑定（LST APNBINDBWMUSRG）_82837489.md) 检查APN是否绑定到BwmUserGroup下。
  ```
  用户组APN绑定信息
  -----------------
   用户组名称  =  usergroup_test
          APN  =  huawei.com
  (结果个数 = 1)
  ---    END
  ```
5. 执行 [**LST BWMRULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理规则/查询带宽管理规则（LST BWMRULE）_82837481.md) 获取以下信息。
    a. 获取BwmRule下配置的BwmService。
    b. 检查BwmService的配置是否是p2p。若是p2p则配置正确；否则需要重新配置BwmService，并将其绑定到BwmRule下。
    c. 检查BwmRule是否是用户级的规则，若是用户级的规则那么配置正确；否则需要在BwmUserGroup下增加用户级的带宽控制规则。
    d. 获取BwmRule下配置的BwmController。
6. 执行 [**LST BWMCONTROLLER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/带宽管理控制器/查询带宽管理控制器（LST BWMCONTROLLER）_82837463.md) 查看控制器下配置的各项参数。
  ```
  带宽管理控制器信息
  ------------------
                     带宽管理控制器名称  =  bwmc_test
                       带宽管理控制类型  =  Shaping
              承诺信息速率（千比特/秒）  =  0
       承诺突发能缓存信息的数量（字节）  =  0
              峰值信息速率（千比特/秒）  =  0
       峰值突发能缓存信息的数量（字节）  =  0
                                   绿色  =  NULL
                         绿色重标记类型  =  NULL
                           绿色重标记值  =  0
                           绿色重标记类  =  NULL
                                   黄色  =  NULL
                         黄色重标记类型  =  NULL
                           黄色重标记值  =  0
                           黄色重标记类  =  NULL
                                   红色  =  NULL
                         红色重标记类型  =  NULL
                           红色重标记值  =  0
                           红色重标记类  =  NULL
                    BWM CAR颜色是否敏感  =  不使能
                  BWM CAR优先级是否敏感  =  不使能
                      速率（千比特/秒）  =  50
                           队列深度（包）=  256
                           最大五元组数  =  2000
  (结果个数 = 1)
   ---    END
  ```
  用户进行TOS值为BE的业务，使用流量检测工具察看带宽控制结果，显示用户的访问流量被控制在300kbps以内。
7. 执行 [**DSP SHAPINGBUFUSED**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/用户QOS控制/流量管理/整形缓存使用率/查询整形使用的缓存（DSP SHAPINGBUFUSED）_86528803.md) 查看系统当前shaping buffer数信息。
  ```
  RETCODE = 0 操作成功。
 
  整形缓存使用信息
  --------
  POD名称                使用的缓存数  总缓存数  IPSQM使用的缓存数  IPSQM总缓存数
 
  isu-pod-0178-32-2-101  0             200032    0                  85128
  isu-pod-2178-32-2-121  0             200032    0                  85128
  (结果个数 = 2)
  --- END
  ```
  “使用的缓存数” 表示已经使用的Shaping Buffer的数目， “总缓存数” 表示总数，如果已经使用数目超过总数的80%，需要调整队列深度。
