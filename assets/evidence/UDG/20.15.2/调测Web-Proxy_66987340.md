# 调测Web Proxy

- [操作场景](#ZH-CN_OPI_0166987340__1.3.1)
- [必备事项](#ZH-CN_OPI_0166987340__1.3.2)
- [操作步骤](#ZH-CN_OPI_0166987340__1.3.3)

## [操作场景](#ZH-CN_OPI_0166987340)

当运营商需要部署Web Proxy功能以达到网络加速效果时，需对 UDG 的Web Proxy功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0166987340)

前提条件

- 请仔细阅读[GWFD-110282 Web Proxy](../GWFD-110282 Web Proxy_66666839.md)。
- 完成[配置Web Proxy](配置Web Proxy_66987339.md)。
- UDG 接入侧/PDN侧的镜像接口已经安装了第三方抓包工具。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | apn | apn-test | 已配置数据中获取 | 取自<br>[配置Web Proxy](配置Web Proxy_66987339.md)<br>中配置的APN实例名。 |
| Web Proxy触发地址 | 服务器IP地址（SVRIP） | 192.168.10.123 | 已配置数据中获取 | 取自<br>[配置Web Proxy](配置Web Proxy_66987339.md)<br>中配置的三四层过滤条件。 |
| Web Proxy目的地址 | 服务器IPv4地址（SERVERIPV4） | 192.168.253.251<br>192.168.253.253 | 已配置数据中获取 | 取自<br>[配置Web Proxy](配置Web Proxy_66987339.md)<br>中配置的IP Farm下重定向服务器的目的地址。 |

工具

- 测试终端
- OM Portal
- 第三方抓包工具

## [操作步骤](#ZH-CN_OPI_0166987340)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令，查询License中是否允许使用Web Proxy业务。
    - 如果License的“SWITCH”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0166987340__step2)。
    - 如果License的“SWITCH”不为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)命令打开本特性对应的License配置开关。
2. 打开接入侧/PDN侧镜像接口上的抓包工具，准备抓取 测试终端 的出入报文。
3. 测试终端 使用“apn-test”APN接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 4](#ZH-CN_OPI_0166987340__step4)。
    - 如果测试终端无法接入网络，调测UDG的接入功能。
4. 测试终端 进行Web浏览业务，访问 **192.168.10.123** 网页，查看镜像接口的抓包信息。
  **图1** 接入侧/PDN侧抓包信息

  <br>

  ![](调测Web Proxy_66987340.assets/zh-cn_image_0207948181.png)
  > **说明**
  > 如 [图1](#ZH-CN_OPI_0166987340__fig1) 所示，针对同一个上行报文，接入侧抓包目的IP为 “192.168.10.123” （ 测试终端 访问Web Proxy业务的触发地址），PDN侧抓包目的IP为 “192.168.253.253” （重定向server的地址）；针对同一个下行报文，PDN侧抓包源IP为 “192.168.253.253” ，接入侧抓包源地址为 “192.168.10.123” 。
    - 如果接入侧/PDN侧抓包结果与[图1](#ZH-CN_OPI_0166987340__fig1)一致，Web Proxy业务正常，调测结束。
    - 如果接入侧/PDN侧抓包的目的IP与源IP一样，请执行[步骤 5](#ZH-CN_OPI_0166987340__step5)。
    - 如果测试终端未看到任何页面，请执行[步骤 7](#ZH-CN_OPI_0166987340__step7)。
5. 执行 [**DSP SESSIONINFO**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，通过 测试终端 的IMSI查看 测试终端 激活时使用的APN。
  ```
  DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
  ```
  ```
  ......
  -------------------------
                           IMSI  =  460000123456789
   	                          ......
                            APN name =  apn-test
   	                          ......
  ---    END
  ```
    - 如果测试终端使用的“APN”与规划值一致，请执行[步骤 6](#ZH-CN_OPI_0166987340__step6)。
    - 如果测试终端使用的“APN”与规划值不一致，请返回[步骤 3](#ZH-CN_OPI_0166987340__step3)使用规划的APN重新接入网络。
6. 检查APN下Web Proxy业务相关配置是否正确。
    a. 执行 [**LST RULEBINDING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_86526204.md) 命令，查询User Profile绑定的rule是否正确。
      ```
      LST RULEBINDING:USERPROFILENAME="up_test";
      ```
      ```
      Rule Binding with User Profile Information
      -------------------------
      用户模板名称    规则名称       优先级       策略类型                   
      up_test       rule_test1    10          Web Proxy
      up_test       rule_test2    20          PCC                           
      (结果个数 = 2)
      ---    END
      ```
          - 如果User Profile绑定的“规则名称”和“优先级”与规划值一致，请执行[步骤 6.b](#ZH-CN_OPI_0166987340__substepc)。
          - 如果User Profile绑定的“规则名称”和“优先级”与规划值不一致，请参考[配置Web Proxy](配置Web Proxy_66987339.md)重新配置。
    b. 执行 [**LST RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，查询rule的配置信息是否正确。
      ```
      LST RULE:RULENAME="rule_test1",POLICYTYPE=WEBPROXY;
      ```
      ```
      规则信息
      -------------------------
                                  规则名称  =  rule_test1
                                  策略类型  =  Web Proxy
       	                                ......
                               流过滤器名称  =  flow-l34
       	                                ......
                                 全局优先级  =  10
       	                                ......
                     Web Proxy IP-Farm名称  =  farm_test
       	                                ......
      ---    END
      ```
      ```
      LST RULE:RULENAME="rule_test2",POLICYTYPE=PCC;
      ```
      ```
      规则信息
      -------------------------
                                  规则名称  =  rule_test2
                                  策略类型  =  PCC
       	                                ......
                               流过滤器名称  =  flow-filter1
       	                                ......
                                 全局优先级  =  20
                              PCC策略组名称  =  pcc-pg_test
                                        ......
      ```
          - 如果rule绑定的“流过滤器名称”、“Web Proxy IP-Farm名称”和“PCC策略组名称”与规划值一致，请执行[步骤 6.c](#ZH-CN_OPI_0166987340__substepd)。
          - 如果rule绑定的“流过滤器名称”、“Web Proxy IP-Farm名称”和“PCC策略组名称”与规划值不一致，请参考[配置Web Proxy](配置Web Proxy_66987339.md)重新配置。
    c. 执行 [**LST PCCPOLICYGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/查询PCC策略组（LST PCCPOLICYGRP）_82837609.md) 、 [**LST URRGROUP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则组/查询URR组（LST URRGROUP）_82837637.md) 命令，查询rule的PCC策略组信息是否正确。
      ```
      LST PCCPOLICYGRP:PCCPOLICYGRPNM="pcc-pg_test";
      ```
      ```
      PCC策略组信息
      -------------------------
            PCC策略组名称  =  pcc-pg_test
              计费属性名称  =  cp_test
       	                       ......
      ---    END
      ```
      ```
      LST URRGROUP:URRGROUPNAME="cp_test";
      ```
      ```
      ......
      -------------------------
           使用量上报规则组名称  =  cp_test
       	                         ......
               上行发起URR名称1  =  cbb_test
               下行发起URR名称1  =  cbb_test
      (结果个数 = 1)
      ---    END
      ```
          - 如果rule的PCC策略组信息与规划值一致，请执行[步骤 6.d](#ZH-CN_OPI_0166987340__substepe)。
          - 如果rule的PCC策略组信息与规划值不一致，请参考[配置Web Proxy](配置Web Proxy_66987339.md)重新配置。
    d. 执行 [**LST FLOWFILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/查询流过滤器（LST FLOWFILTER）_82837364.md) 、 [**LST PROTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的协议绑定/查询流过滤器协议绑定关系（LST PROTBINDFLOWF）_82837373.md) 和 [**LST FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/查询过滤器（LST FILTER）_82837353.md) 命令，查询rule绑定的filter信息是否正确。
      ```
      LST FLOWFILTER:FLOWFILTERNAME="flow-l34";
      ```
      ```
      流过滤器信息
      -------------------------
              流过滤器名称  =  flow-l34
       	               ......
      ---    END
      ```
      ```
      LST FLTBINDFLOWF:FLOWFILTERNAME="flow-l34";
      ```
      ```
      流过滤器的过滤器绑定信息
      -------------------------
      流过滤器名称  =  flow-l34
        过滤器名称  =  filter_test
      (结果个数 = 1)
      ---    END
      ```
      ```
      LST FILTER:FILTERNAME="filter_test";
      ```
      ```
      过滤器信息
      ------------------
                                   过滤器名字  =  filter_test
                         三四层IPv4协议输入类型  =  字符串类型
       	                                     ......
                                三四层协议类型  =  TCP
       	                                     ......
                                     生效标记  =  是	
      (结果个数 = 1)
      ---    END
      ```
          - 如果rule绑定的filter信息与规划值一致，说明Web Proxy业务相关配置正确，请执行[步骤 7](#ZH-CN_OPI_0166987340__step7)。
          - 如果rule绑定的filter信息与规划值不一致，请参考[配置Web Proxy](配置Web Proxy_66987339.md)重新配置。
7. 执行 [**LST IPFARM**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/IP Farm 管理/IP Farm参数/查询IPFarm（LST IPFARM）_82837053.md) 和 [**LST IPFARMSERVER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/IP Farm 管理/IP Farm服务器/查询IPFarmServer（LST IPFARMSERVER）_82837059.md) 命令，检查IP Farm中server的状态。
  ```
  LST IPFARM:IPFARMNAME="farm_test";
  ```
  ```
  IPFarm信息
  -------------------------
           IP-Farm名称  =  farm_test
             服务器类型  =  重定向
   	                 ......
            健康检查标记  =  使能
             IP协议版本  =  IPV4
   	                 ......
         心跳检测接口名称  =  phif1/0/0
              接口IP地址  =  10.10.10.10
   	                  ......
  ---    END
  ```
  ```
  LST IPFARMSERVER:IPFARMNAME="farm_test";
  ```
  ```
  ......
  ------------------------
      IP-Farm名称  =  farm_test
          地址信息  =  192.168.253.251
    请求URL携带标识  =  不使能
       请求URL名称  =  dest_url
   	             ......
         服务器状态  =  up
     重定向缺省动作  =  block
  (结果个数 = 1)
  ---    END
  ```
8. 执行 [**LST LOGICINF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/逻辑接口管理/逻辑接口配置/接口/查询逻辑接口（LST LOGICINF）_86526672.md) 命令，检查IP Farm中server的心跳检测接口的相关配置是否正确。
  ```
  LST LOGICINF:NAME="phif1/0/0";
  ```
  ```
  逻辑接口信息
  -------------------------
           逻辑接口名称  =  phif1/0/0
     逻辑接口的IPv4地址1  =  10.10.10.10
     逻辑接口的IPv4掩码1  =  255.255.255.255
   	                    ......
  ---    END
  ```
    - 如果配置与规划值一致，请执行[步骤 9](#ZH-CN_OPI_0166987340__step9)。
    - 如果配置与规划值不一致，请参考[配置Web Proxy](配置Web Proxy_66987339.md)重新配置。然后重新执行[步骤 4](#ZH-CN_OPI_0166987340__step4)。
9. 收集信息并寻求技术支持。
    a. 在镜像接口或服务器上开启抓包工具，执行 [步骤 4](#ZH-CN_OPI_0166987340__step4) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 查看并收集对端设备配置及接口状态信息。
    e. 收集归纳所有信息并联系华为技术支持解决。
