#### [命令功能](#ZH-CN_TOPIC_0171765322)

![](节点复位（RST NODE）_71765322.assets/notice_3.0-zh-cn.png)

- 执行该命令会影响所有运行在该节点上的业务，请慎重使用该命令。
- 在存储故障期间，执行该命令复位节点后，该节点以及节点中的容器和进程，在存储恢复前都无法启动。
- 硬复位可能导致文件系统损坏，强烈建议优先使用软复位。
- 本地盘形态下在主机异常时（如故障或下电）不建议下发此操作。
- 如果需要复位OMU多节点，在复位一个节点后需要等待节点重启成功且服务正常后再执行重启下一个节点的命令。

本命令用于复位指定网元ID下指定名称的节点。复位方式分为软复位和硬复位，软复位操作方式是由操作系统执行的安全复位，硬复位操作方式为强制下电复位。

> **说明**
> 该命令仅在Full-stack虚机场景下支持。

#### [注意事项](#ZH-CN_TOPIC_0171765322)

硬复位可能会导致文件系统损坏，可以通过执行 [**REB NODE**](节点重建（REB NODE）_71765323.md) 命令，选择重装OS的方式修复。

#### [参数说明](#ZH-CN_TOPIC_0171765322)

| **参数标识** | **参数名称** | **参数说明** |
| --- | --- | --- |
| APPID | 网元ID | 可选必选说明：必选参数<br>参数含义：用于指示系统需要复位指定网元ID下的节点数据。<br>取值范围：0～65535。<br>默认值：无。<br>配置原则：<br>“网元ID”<br>获取方式为：点击<br>“首页”<br>，查看<br>“应用ID”<br>即可。 |
| NODENAME | 节点名称 | 可选必选说明：必选参数<br>参数含义：用于指示系统需要复位哪个节点的数据。<br>取值范围：不超过128位字符串。<br>默认值：无。<br>配置原则：<br>操作员可以使用<br>[**DSP NODE**](节点查询（DSP NODE）_71678755.md)<br>命令查询获得。 |
| RESETTYPE | 复位类型 | 可选必选说明：必选参数<br>参数含义：标识指定网元ID和指定节点复位的方式。<br>取值范围：<br>- “soft（软复位）”<br>：通过给节点中的OS发送重启信号的方式重启节点。软复位是正常重启节点的流程，节点软复位过程中，运行在该虚拟机上的业务会受影响。<br>- “hard（硬复位）”<br>：复位时不知会节点OS，而是强制重启节点。使用硬复位可能会导致节点中的文件系统被破坏。节点复位过程中，运行在该节点上的业务会受影响。<br>默认值：无。<br>配置原则：优先使用<br>“soft（软复位）”<br>，不建议使用<br>“hard（硬复位）”<br>。 |

#### [使用实例](#ZH-CN_TOPIC_0171765322)

复位节点 “网元ID” 为“0”， “节点名称” 为“10.0.0.1”， “复位类型” 为“软复位”。

```
%%RST NODE: APPID=0, NODENAME="10.0.0.1", RESETTYPE=soft;%% 
RETCODE = 0  操作成功   
操作结果如下 
------------
    状态  =  Success 
详细信息  =  操作成功 
(结果个数 = 1)  
---    END
```
