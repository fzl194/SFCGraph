#### [UE或者PGW-C/SMF发起的L2TP用户去活](#ZH-CN_TOPIC_0319990151)

**图1** UE或者PGW-C/SMF发起的L2TP用户去活

<br>

![](L2TP去激活流程_19990151.assets/zh-cn_image_0000001085770838.png)

1. UE或者PGW-C/SMF主动去活L2TP用户，PGW-C/SMF向PGW-U/UPF发送PFCP Session Delete Request消息。
2. PGW-U/UPF收到PFCP Session Delete Request消息后，给LNS发送CDN（Call-Disconnect-Notify）消息，请求释放L2TP会话。
3. LNS给PGW-U/UPF发送ZLB（Zero-Length Body）消息，确认收到释放消息；
4. 如果L2TP会话释放后，对应的L2TP隧道没有其他的L2TP会话使用，则PGW-U/UPF发送StopCCN（Stop-Control-Connection-Notification）消息删除L2TP隧道；如果L2TP隧道还有其他的L2TP会话在使用，则不需要删除L2TP隧道。
5. PGW-U/UPF向PGW-C/SMF回复PFCP Session Delete Response消息。

#### [PGW-U/UPF发起的L2TP用户去活](#ZH-CN_TOPIC_0319990151)

**图2** PGW-U/UPF发起的L2TP用户去活

<br>

![](L2TP去激活流程_19990151.assets/zh-cn_image_0000001132552701.png)

1. PGW-U/UPF主动去活L2TP用户，比如通过[**DEA SESSION**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话去活/去活用户会话（DEA SESSION）_82837084.md)命令去活L2TP用户。PGW-U/UPF给LNS发送CDN消息，请求释放L2TP会话。
2. LNS给PGW-U/UPF发送ZLB消息，确认收到释放消息。
3. 如果L2TP会话释放后，对应的L2TP隧道没有其他的L2TP会话使用，则PGW-U/UPF发送StopCCN消息删除L2TP隧道；如果L2TP隧道还有其他的L2TP会话在使用，则不需要删除L2TP隧道。
4. PGW-U/UPF向PGW-C/SMF发送PFCP Session Report Request消息，上报L2TP会话已释放；PGW-C/SMF回复PFCP Session Report Response消息。
5. PGW-C/SMF向PGW-U/UPF发送PFCP Session Delete Request消息，删除L2TP的信息；PGW-U/UPF向PGW-C/SMF回复PFCP Session Delete Response消息。

#### [LNS发起的L2TP用户去活](#ZH-CN_TOPIC_0319990151)

**图3** LNS发起的L2TP用户去活

<br>

![](L2TP去激活流程_19990151.assets/zh-cn_image_0000001085642352.png)

1. LNS主动发起L2TP会话释放流程，给PGW-U/UPF发送CDN消息，请求释放L2TP会话。
2. PGW-U/UPF回复ZLB消息确认会话释放消息已收到。
3. 如果L2TP会话释放后，对应的L2TP隧道没有其他的L2TP会话使用，则PGW-U/UPF发送StopCCN消息删除L2TP隧道；如果L2TP隧道还有其他的L2TP会话在使用，则不需要删除L2TP隧道。
4. PGW-U/UPF向PGW-C/SMF发送PFCP Session Report Request消息，上报L2TP会话已释放；PGW-C/SMF回复PFCP Session Report Response消息。
5. PGW-C/SMF向PGW-U/UPF发送PFCP Session Delete Request消息，删除L2TP的信息；PGW-U/UPF向PGW-C/SMF回复PFCP Session Delete Response消息。

#### [LNS发起单栈释放流程](#ZH-CN_TOPIC_0319990151)

**图4** LNS发起单栈释放流程

<br>

![](L2TP去激活流程_19990151.assets/zh-cn_image_0000001132709899.png)

1. L2TP用户激活后，LNS给PGW-U/UPF发送IPCP Terminate-Request或者IPv6CP Terminate-Request消息，请求释放IPv4或者IPv6地址。
2. PGW-U/UPF收到IPCP或者IPv6CP终止请求消息后，根据用户激活时协商的IPCP或者IPv6CP判断消息合法时，则给LNS回复IPCP Terminate-Ack或者IPv6CP Terminate-Ack消息。
3. PGW-U/UPF收到LNS的地址释放请求后，发起L2TP用户的去活流程，PGW-U/UPF给LNS发送CDN消息，请求释放L2TP会话。
4. LNS回复ZLB消息确认会话释放消息已收到。
5. 如果L2TP会话删除后，对应的L2TP隧道没有其他的L2TP会话使用，则PGW-U/UPF发送StopCCN消息删除L2TP隧道；如果L2TP隧道还有其他的L2TP会话在使用，则不需要删除L2TP隧道。
6. PGW-U/UPF向PGW-C/SMF发送PFCP Session Report Request消息，上报L2TP会话已释放；PGW-C/SMF回复PFCP Session Report Response消息。
7. PGW-C/SMF向PGW-U/UPF发送PFCP Session Delete Request消息，删除L2TP的信息；PGW-U/UPF向PGW-C/SMF回复PFCP Session Delete Response消息。
