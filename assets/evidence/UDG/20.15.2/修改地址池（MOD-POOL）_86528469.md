#### [命令功能](#ZH-CN_CONCEPT_0186528469)

**适用NF：PGW-U、UPF**

![](修改地址池（MOD POOL）_86528469.assets/notice_3.0-zh-cn.png)

本命令属于高危命令，该操作会改变地址池的属性，可能会导致用户激活失败。

该命令是修改地址池，当系统需要修改地址池的地址租约使能、地址租期、小地址池标记、绑定VPN使能、VPN实例、外部地址检查、主用标识或告警使能时，可以通过该命令来修改。

#### [注意事项](#ZH-CN_CONCEPT_0186528469)

- 该命令执行后立即生效。
- 如果APN已经绑定到特定的VPN实例下，APN下要绑定的地址池必需和APN一样，绑定到同一个VPN实例下。
- 一个地址池只能绑定到一个VPN实例下。地址池下若不执行该命令，则默认配置为公网下的地址池，查询命令显示的地址池VPN实例名称为NULL。
- 如果一个地址池下配置有地址段或该地址池已经绑定地址池组，则禁止修改该地址池的VPN实例。
- 配置RELAESETIME为0且配置IPLEASE为ENABLE，则本次配置将执行失败。
- 不允许修改地址池类型。
- 修改后的CHECKIPVALID参数只对新激活的用户生效。
- 本地地址池下需要配置地址段，一个本地地址池最多可以配置64个地址段。
- 当需要修改的pool的IPVERSION为IPv6时，SINGLEIPFLAG字段修改为ENABLE，则本次配置失败。仅IPv4支持小地址池。

#### [操作用户权限](#ZH-CN_CONCEPT_0186528469)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0186528469)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| POOLNAME | 地址池名称 | 可选必选说明：必选参数<br>参数含义：地址池的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～79，单位是字节。由“_”、“-”、数字、大小写字母和“.”组成，不能以“.”开头且不能出现连续两个“.”，不区分大小写。<br>默认值：无<br>配置原则：无 |
| POOLTYPE | 地址池类型 | 可选必选说明：可选参数<br>参数含义：用于指定地址池类型。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- LOCAL：系统负责分配地址的地址池。<br>- EXTERNAL：外部地址的地址池。<br>- MULTICAST：组播地址的地址池。<br>默认值：无<br>配置原则：无 |
| IPLEASE | 地址租约使能标志 | 可选必选说明：条件可选参数<br>前提条件：该参数在“POOLTYPE”配置为“LOCAL”时为可选参数。<br>参数含义：指定是否使能IP地址租约功能。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：<br>- 对于外部地址池，IPLEASE参数不生效。<br>- 在配置IPLEASE参数值为ENABLE后上线的在线用户，配置IPLEASE参数值为DISABLE后，去活用户，该用户再次上线时，将不再具有地址租约功能。 |
| RELEASETIME | 地址租期（秒） | 可选必选说明：条件可选参数<br>前提条件：该参数在“POOLTYPE”配置为“LOCAL”时为可选参数。<br>参数含义：用户释放之后，对应用户IP地址的等待释放时间。取值为0时，表示用户去激活后，其使用的本地地址将被立即释放，且可重新分配给其他用户使用。取值非0时，表示用户去激活后，其使用的地址不会立即释放，而是等待配置的时间之后才能被使用。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～86400，单位是秒。<br>默认值：无<br>配置原则：<br>- 建议配置RELAESETIME的值大于0，并根据实际需要进行调整。<br>- 配置RELAESETIME为0且配置IPLEASE为ENABLE，则本次配置将执行失败。<br>- 已经配置IPLEASE为ENABLE，本次配置时只配置RELAESETIME为0而未配置IPLEASE，则本次配置将执行失败。<br>- 在远端地址池下配置RELAESETIME命令，配置将执行失败。<br>- 在配置RELAESETIME参数大于0后上线的在线用户，配置RELAESETIME参数为0后，去活用户，该用户使用的地址将被立即释放。<br>- 配置RELAESETIME不为0，如果集中去活（例如锁apn去活）大量用户后，马上激活，大量新用户接入，这时大量地址处于等待释放状态，可能会出现因无空闲地址而导致的用户入网失败的情况。 |
| SINGLEIPFLAG | 小地址池标志 | 可选必选说明：条件可选参数<br>前提条件：该参数在“POOLTYPE”配置为“LOCAL”时为可选参数。<br>参数含义：配置指定本地地址池是否为小地址池（每个地址子段只有一个地址的地址池）。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：<br>- 配置小地址池对性能影响较大，正常情况下不建议配置。<br>- 小容量地址池主要解决现网中经常出现的多块ISU同用一个地址池时，由于地址分配不均导致在用户激活过程中无可用的地址而激活失败的问题。<br>- 由于原有的地址池分配处理中每个地址段最多可以划分为1024个子段，对于小地址池来说，每个子段只有一个地址，则小地址池中每个地址段的地址个数不能超过1024个。对于小地址池中总的地址个数不做限制，理论上每个小地址池最多可以配置1024*16=65536个地址。<br>- 如果地址池先配置了地址段，再配置Single IP Flag属性，命令将执行失败。 |
| CHECKIPVALID | 检查地址合法性 | 可选必选说明：条件可选参数<br>前提条件：该参数在“POOLTYPE”配置为“EXTERNAL”时为可选参数。<br>参数含义：检查外部地址的合法性。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：<br>- IPv6地址池不支持白名单检查功能。<br>- ENABLE：检查外部地址的合法性。<br>- DISABLE：不检查外部地址的合法性。 |
| HASVPN | 绑定VPN | 可选必选说明：可选参数<br>参数含义：指定是否绑定VPN。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：<br>- 如果APN已经绑定到特定的VPN实例下，APN下要绑定的地址池必需和APN一样，绑定到同一个VPN实例下。<br>- 一个地址池只能绑定到一个VPN实例下。地址池下若不执行该命令，则默认配置为公网下的地址池，查询命令显示的地址池VPN实例名称为NULL。<br>- 如果是IPv4的本地地址池，配置VPN实例时必须在配置地址段section之前。<br>- 如果一个地址池下配置有地址段或该地址池已经绑定地址池组，则禁止修改该地址池的VPN实例。 |
| VPNINSTANCE | VPN实例名 | 可选必选说明：条件必选参数<br>前提条件：该参数在“HASVPN”配置为“ENABLE”时为必选参数。<br>参数含义：指定VPN实例名。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～31。区分大小写，不支持空格。<br>默认值：无<br>配置原则：该参数使用ADD VPNINST命令配置生成。 |
| ALARMSWITCH | 告警特性是否使能 | 可选必选说明：条件可选参数<br>前提条件：该参数在“POOLTYPE”配置为“LOCAL”时为可选参数。<br>参数含义：指定是否开启本地址池内ALM-81216 地址池占用率超门限告警功能。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：<br>- ENABLE：开启本地地址池内ALM-81216 地址池占用率超门限告警功能。<br>- DISABLE：关闭本地地址池内ALM-81216 地址池占用率超门限告警功能。 |
| IMSSW | IMS开关 | 可选必选说明：可选参数<br>参数含义：指定使能IP地址租约功能是否对IMS用户有效。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：<br>- ENABLE：IP地址租约功能对IMS用户有效。<br>- DISABLE：IP地址租约功能对IMS用户无效。 |
| REDUNDFUNC | 冗余备份功能开关 | 可选必选说明：条件可选参数<br>前提条件：该参数在“POOLTYPE”配置为“EXTERNAL”时为可选参数。<br>参数含义：指定是否开启外部地址池内静态地址用户路由冗余功能。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：<br>- ENABLE：开启外部地址池内静态地址用户路由冗余功能。<br>- DISABLE：关闭外部地址池内静态地址用户路由冗余功能。<br>- 设置ISU Group后静态地址用户路由冗余功能才会生效。 |
| MASTERFLAG | 主用标记 | 可选必选说明：条件可选参数<br>前提条件：该参数在“REDUNDFUNC”配置为“ENABLE”时为可选参数。<br>参数含义：标识是否是主用UPF。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：<br>- ENABLE：标识为主用UPF。<br>- DISABLE：标识为非主用UPF。<br>- 在终端静态签约地址且GRE冗余隧道进行容灾场景下，为了具备UE互访的能力，需要在路由优先级高的地址段归属的地址池上使能MasterFlag，同时对端UPF对应的地址池不能使能MasterFlag。 |

#### [使用实例](#ZH-CN_CONCEPT_0186528469)

假设运营商需要修改一个本地IPv4地址池pool1的地址租约使能：

```
MOD POOL:POOLNAME="pool1",POOLTYPE=LOCAL,IPLEASE=ENABLE,RELEASETIME=1;
```
