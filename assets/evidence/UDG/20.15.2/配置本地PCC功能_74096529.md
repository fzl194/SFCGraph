## [操作场景](#ZH-CN_OPI_0174096529)

当网络中无PCRF/PCF，若通过特定APN激活的用户需要激活为PCC用户，则需要配置本地PCC功能。

对于本地PCC功能（本地规则），UPF/SMF上需定义相同的规则组标识（即 [**ADD USERPROFILE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/用户模板/增加用户模板（ADD USERPROFILE）_82837279.md) 命令中的 “用户模板名称（USERPROFILENAME）” 参数值），UPF上定义该规则对应的业务流特征及业务的精细化管理动作。用户注册过程中，由SMF下发规则组标识，SMF通过规则组标识的方式激活UPF本地静态配置的规则来实现计费和策略的控制。SMF不支持实时更新规则。

> **说明**
> 适用于PGW-U、UPF。
>
> 配置本地PCC时， UDG 上需要配置rule和UserProfile，本页面不关注具体规则内容配置，具体规则内容需要根据具体业务策略确定，配置方法请参见 [业务感知配置](../../../../../业务专题/UDG业务感知专题/业务感知配置_90565483.md) 。

## [操作步骤](#ZH-CN_OPI_0174096529)

1. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)
2. 配置PCC策略组。
  [**ADD PCCPOLICYGRP**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/增加PCC策略组（ADD PCCPOLICYGRP）_82837606.md)
3. 配置流过滤器。
  [**ADD FLOWFILTER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/增加流过滤器（ADD FLOWFILTER）_82837361.md)
  流过滤器必须至少绑定一个filter或一个l7filter，否则所有报文都匹配不上该流过滤器。
4. 配置三四层过滤条件。
    a. 配置三四层过滤条件。
      [**ADD FILTER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/增加过滤器（ADD FILTER）_82837348.md)
    b. 配置三四层过滤条件与流过滤器的绑定关系。
      [**ADD FLTBINDFLOWF**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的过滤器绑定/增加流过滤器的过滤器绑定关系（ADD FLTBINDFLOWF）_82837366.md)
5. 配置规则。
  **[ADD RULE](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/增加规则（ADD RULE）_82837266.md)**
  若需配置多个规则，则需要多次执行 [步骤 5](#ZH-CN_OPI_0174096529__step3630133623814) 。
6. 配置规则与用户模板的绑定关系 。
    a. 创建UserProfile。
      [**ADD USERPROFILE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/用户模板/增加用户模板（ADD USERPROFILE）_82837279.md)
    b. 将已配置的rule绑定到UserProfile。
      [**ADD RULEBINDING**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/增加用户模板和规则的绑定关系（ADD RULEBINDING）_82837272.md)
7. 将新配置的Filter跟UserProfile置为生效。
  [**SET REFRESHSRV**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/业务刷新/业务刷新（SET REFRESHSRV）_82837355.md)

## [任务示例](#ZH-CN_OPI_0174096529)

任务描述

本举例中配置的是3/4层any to any的filter，未配置7层规则，表示所有报文都匹配。

脚本

//打开本特性的License配置开关。

```
SET LICENSESWITCH:LICITEM="LKV3G5PCCB01",SWITCH=ENABLE;
```

//配置流过滤器和三四层过滤条件。

```
ADD FILTER: FILTERNAME="filter_test", L34PROTTYPE=STRING, L34PROTOCOL=ANY;
ADD FLOWFILTER:FLOWFILTERNAME="flowfilter_test";
ADD FLTBINDFLOWF:FLOWFILTERNAME="flowfilter_test",FILTERNAME="filter_test";
```

//配置PCC策略组。 **当需要计费时，请配置参数URRGROUPNAME** 。

```
ADD PCCPOLICYGRP:PCCPOLICYGRPNM="pg_test",URRGROUPNAME="urrgp_test";
```

//配置规则。

```
ADD RULE:RULENAME="rule_test",POLICYTYPE=PCC,
FILTERINGMODE=FLOWFILTER,
FLOWFILTERNAME="flowfilter_test",PRIORITY=65535, POLICYNAME="pg_test";
```

//配置规则与用户模板的绑定关系。

```
ADD USERPROFILE:USERPROFILENAME="up_test";
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="rule_test";
```

//将新配置的Filter和UserProfile置为生效。

```
SET REFRESHSRV:REFRESHTYPE=ALL;
```
