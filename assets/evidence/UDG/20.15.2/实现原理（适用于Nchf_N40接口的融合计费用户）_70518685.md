#### [事件计费关键信元](#ZH-CN_TOPIC_0270518685)

融合计费中，PGW-C/SMF根据CHF下发的计费事件（Triggers）与CHF进行交互，并通过下列信元完成基本的配额授权与用量上报。

- CHF下发的事件计费配额和阈值：
  Charging Data Response>MultipleUnitInformation>grantedUnit>serviceSpecificUnits
  Charging Data Response>MultipleUnitInformation>unitQuotaThreshold
- PGW-C/SMF上报事件计费用量：Charging Data Request>MultipleUnitUsage>UsedUnitContainer>serviceSpecificUnits
- PGW-C/SMF申请的事件计费配额：Charging Data Request>MultipleUnitUsage>RequestedUnit>serviceSpecificUnits

#### [事件计费支持的业务类型](#ZH-CN_TOPIC_0270518685)

UDG 支持SCUR（Session Charging with Unit Reservation）配额预留方式的会话计费。该方式是在传统的会话计费中上报事件计费，无论是流量、时长、还是事件计费，统一使用同一个在线计费会话来承载。 UDG 支持以下协议类型的事件计费：

- HTTP业务：基于用户对HTTP业务的访问次数进行计费。这里的访问次数，是触发HTTP Get/Post请求的次数，不是用户点击链接的次数。实际上，用户在打开一个主页的过程中，可能会触发多个HTTP Get/Post请求。 UDG 会当做多个事件进行计费。
  > **说明**
  > HTTP协议中connect报文是用于在MS/UE->proxy->server之间建立安全隧道，并不是真正的业务访问。因此HTTP协议中的connect报文不会触发事件计费。

- WAP业务：基于用户对某种WAP（Wireless Application Protocol）业务的访问次数进行计费。
- MMS业务：基于用户对MMS（Multimedia Messaging Service）服务的使用次数进行计费。

#### [建立计费会话](#ZH-CN_TOPIC_0270518685)

1：用户发起PDU会话建立请求。

2：PGW-C/SMF向CHF发送Charging Data Request[Initial]消息，请求建立计费会话。该消息可携带预申请的事件计费配额，通过serviceSpecificUnits字段携带给CHF。具体请参考《UNC 产品文档》。

3：CHF向PGW-C/SMF返回Charging Data Response[Initial]消息。如果PGW-C/SMF预申请了事件计费配额，事件计费配额和阈值会通过serviceSpecificUnits字段和unitQuotaThreshold字段携带给PGW-C/SMF。

- 若PGW-C/SMF使能事件计费功能，CHF未下发对应的事件计费配额给PGW-C/SMF，则PGW-C/SMF异常去激活用户。
- 若PGW-C/SMF未使能事件计费功能，CHF下发对应的事件计费配额给PGW-C/SMF，则PGW-C/SMF忽略CHF下发的事件计费配额。

> **说明**
> 当事件计费的申请配额数为0时，PGW-C/SMF也会将serviceSpecificUnits带给CHF，以向CHF表明该RG需要进行事件计费。

4：PGW-C/SMF发送PFCP Session Establishment Request消息，请求建立PGW-C/SMF会话。消息中携带Create URR（Usage Reporting Rule）信元组，指示计费相关信息。Create URR包含的主要子信元如 [表1](#ZH-CN_TOPIC_0270518685__table5217133513420) ：

*表1 Create URR包含的主要子信元*

| 子信元 | 信元解释 |
| --- | --- |
| URR ID | 唯一标识一个Create URR。 |
| Measurement Method | 标识计费方式，包括流量计费、时长计费或事件计费。PGW-C/SMF通过EVENT字段指示PGW-U/UPF该RG是否支持事件计费，设置为1标识支持事件计费。<br>说明：整个PDU会话过程中某RG是否使能事件计费不会发生变化。 |
| Reporting Triggers | 标识计费上报的触发条件，包含Periodic Reporting（定期上报）、Volume Quota（流量配额到上报）、Event Quota（事件配额到上报）、Volume Threshold（流量阈值到上报）、Event Threshold（事件阈值到上报）、Linked Usage Reporting（关联URR）等条件。 |
| Event Quota | 标识事件配额。 |
| Event Threshold | 标识事件阈值。 |
| Linked URR ID | 上报某个URR时，也会上报其关联的URR，例如A link B，当B上报时，也会上报A。 |

5：PGW-U/UPF返回PFCP Session Establishment Response消息，PGW-C/SMF会话建立成功。

6：PDU会话建立成功。

#### [业务触发申请事件计费配额](#ZH-CN_TOPIC_0270518685)

7.a和7.b：用户访问数据业务，PGW-U/UPF识别该数据业务需要进行事件计费(通过 **[ADD URR](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则/增加URR（ADD URR）_82837629.md)** 命令的 “ONLMETERINGTYPE” 参数配置和PGW-C/SMF的Measurement Method信元指示决定)，如果判断该RG无可用事件计费配额，PGW-U/UPF向PGW-C/SMF进行新业务配额申请（Start Trigger）。PGW-C/SMF与PGW-U/UPF的事件计费功能配置不一致时处理如下：

- PGW-C/SMF本地使能事件计费功能，PGW-U/UPF本地不使能事件计费功能时。PGW-U/UPF上报[ALM-81054 CP和UP关键配置不一致](../../../../../网络运维/故障处理/用户面告警/ALM-81054 CP和UP关键配置不一致_93167109.md)告警，PGW-U/UPF按照未开启事件计费功能处理业务。
- PGW-C/SMF本地不使能事件计费功能时，PGW-U/UPF本地使能事件计费功能。PGW-C/SMF上报告警**ALM-81026 接口信元不一致**告警。

8：PGW-C/SMF收到PGW-U/UPF发送的事件计费配额申请，根据本地配置的事件计费配额和PGW-U/UPF申请的事件计费配额之和作为事件计费配额，通过serviceSpecificUnits字段发给CHF。

9：PGW-C/SMF发送PFCP Session Report Response给PGW-U/UPF。

10：PGW-C/SMF收到CHF下发的事件计费配额，配额通过serviceSpecificUnits字段携带，阈值通过unitQuotaThreshold字段携带。

11：PGW-C/SMF通过PGW-C/SMF Session Modification Request/Response消息将事件计费配额和阈值发送给PGW-U/UPF。

#### [事件计费配额或阈值耗尽，用量上报和配额申请](#ZH-CN_TOPIC_0270518685)

12.a和12.b：PGW-U/UPF对事件计费进行配额用量监控，在用量达到阈值，或配额耗尽时向PGW-C/SMF进行用量上报和申请新的配额。用量上报中包括对应的计费事件Trigger和对应成功事件次数的时间戳。具体信元见 [表2](#ZH-CN_TOPIC_0270518685__table16660175152713) 。如果PGW-C/SMF未返回足够配额则阻塞或重定向报文。

*表2 Usage Report包含的主要子信元*

| 子信元 | 信元解释 |
| --- | --- |
| URR ID | 标识上报的URR。 |
| UR-SEQN | 唯一标识URR的Usage Report。 |
| EVETH | 事件计费阈值到触发上报。 |
| EVEQU | 事件计费配额耗尽触发上报。 |
| Event Time Stamp | 事件计费的时间戳。 |

13：PGW-C/SMF向CHF发起Charging Data Request[Update]消息，请求更新计费会话。上报事件计费用量并申请新的事件计费配额。

#### [计费会话去激活，事件计费用量上报](#ZH-CN_TOPIC_0270518685)

17：PGW-C/SMF向PGW-U/UPF发起用户去激活，获取最后的事件计费用量。

18：PGW-U/UPF向PGW-C/SMF上报去活的事件计费用量。

19：PGW-C/SMF向CHF发送Charging Data Request Release消息上报去活的事件计费用量。

20：CHF向PGW-C/SMF发送Charging Data Response Release消息。

> **说明**
> - 当达到阈值或配额耗尽时，触发PGW-U/UPF发送PFCP Session Report Request消息申请新配额，并启动定时器，定时器时长=**T3RESPONSE*****N3REQUEST**+4秒（**T3RESPONSE**及**N3REQUEST**参数由[**SET UPPFCPPATH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/路径管理/PFCP路径管理/PFCP参数管理/PFCP路径参数/设置PFCP路径相关属性（SET UPPFCPPATH）_82837240.md)命令配置），当定时器超时，未收到配额更新消息时，PGW-U/UPF阻塞业务。
> - 事件计费支持和时长、流量计费同时进行，先满足的计费Trigger会上报所有维度配额的用量。
> - PGW-U/UPF支持区分收彩信和发彩信两种应用，也可以区分彩信单发和群发，并根据群发对象个数统计事件次数而进行计费处理。比如一次彩信收发，与彩信内容大小无关。对于彩信群发，可以设置是按单次事件收费还是根据接收者的数量按多次事件收费。具体请参考 [**ADD PCCPOLICYGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/增加PCC策略组（ADD PCCPOLICYGRP）_82837606.md) 配置。
