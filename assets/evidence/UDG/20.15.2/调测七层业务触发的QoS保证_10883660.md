# 调测七层业务触发的QoS保证

- [操作场景](#ZH-CN_OPI_0210883660__1.3.1)
- [必备事项](#ZH-CN_OPI_0210883660__1.3.2)
- [操作步骤](#ZH-CN_OPI_0210883660__1.3.3)

## [操作场景](#ZH-CN_OPI_0210883660)

当运营商需要部署七层业务触发的QoS保证功能时，需对 UDG 的七层业务触发的QoS保证功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于SGW-U、PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0210883660)

前提条件

- 请仔细阅读[GWFD-020358 业务触发的QoS保证](../../GWFD-020358 业务触发的QoS保证_67655191.md)。
- 完成[配置七层业务触发的QOS保证（PGW-U、UPF）](../激活业务触发的QoS保证/配置七层业务触发的QOS保证（PGW-U、UPF）_80361858.md)。

## [操作步骤](#ZH-CN_OPI_0210883660)

1. 执行 [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令查询 UDG 上本功能对应的License配置开关是否打开。
  ```
  LST LICENSESWITCH:LICITEM="
  LKV3G5STQE01
  ";
  ```
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0210883660__step19171111585013)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)命令打开本特性对应的License配置开关。
2. 在OM Portal上创建 [UDG N4接口跟踪](../../../../../../网络运维/日常维护/UDG基础运维操作/创建消息跟踪/创建接口跟踪/UDG N4接口跟踪_88792205.md) 任务。
3. 测试终端使用 “apn1” APN接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 4](#ZH-CN_OPI_0210883660__step1712315319104)。
    - 如果测试终端无法接入网络，调测UDG的接入功能。
4. 测试终端浏览Web业务，访问 **www.huawei.com** 网页。
    - 如果测试终端可以正常访问网页，请执行 [步骤 5](#ZH-CN_OPI_0210883660__step135281929220) 。
    - 如果测试终端无法正常访问网页，请调测 UDG 的Web浏览功能。
5. 查看数据规划中相关规则的数据配置，根据相关规则，使用测试终端访问能触发专有承载/专有QoS Flow创建或者更新的业务。
6. 查看PGW-U/UPF是否立即向PGW-C/SMF发送PFCP Session Report Request消息，携带的Usage Report信元组中包含QoS类型的URR ID、Usage Report Trigger为start。
    - 如果PGW-U/UPF上报信元正确，则业务触发QoS保证功能生效，继续执行[步骤 7](#ZH-CN_OPI_0210883660__step2054514555016)。
    - 如果PGW-U/UPF没有上报或上报信元不全，则业务触发QoS保证功能没有正常生效，请执行[步骤 9](#ZH-CN_OPI_0210883660__step13676132835614)。
7. 查看PGW-C/SMF向PGW-U/UPF发送的PFCP Session Report Response消息，查询Create PDR信元组中携带的URR ID是否与PFCP Session Report Request消息携带的URR ID一致。
    - 如果一致，业务触发QoS保证功能正常，调测结束。
    - 如果不一致，请执行 [步骤 8](#ZH-CN_OPI_0210883660__step6) 。
8. 执行 [**DSP SESSIONINFO**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，通过 测试终端 的IMSI查看 测试终端 建立专有承载/专有QoS Flow和使用的APN。
  ```
  DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
  ```
    - 如果测试终端使用的“APN name”与规划值一致，请执行[步骤 9](#ZH-CN_OPI_0210883660__step13676132835614)。
    - 如果测试终端使用的“APN name”与规划值不一致，请返回[步骤 3](#ZH-CN_OPI_0210883660__step3)使用规划的APN重新接入网络。
9. 检查APN下内容感知相关配置是否正确：
    a. 执行 [**LST RULEBINDING**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_86526204.md) 命令，查询UserProfile绑定的Rule是否正确。

          - 如果UserProfile绑定的“规则名称”和“优先级”与规划值不一致，请参考 [配置七层业务触发的QOS保证（PGW-U、UPF）](../激活业务触发的QoS保证/配置七层业务触发的QOS保证（PGW-U、UPF）_80361858.md) 重新配置。
    b. 执行 [**LST RULE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，查询Rule的配置信息是否正确。

    c. 执行 [**LST PCCPOLICYGRP**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/查询PCC策略组（LST PCCPOLICYGRP）_82837609.md) 、 [**LST URR**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则/查询URR（LST URR）_82837632.md) 和 [**LST QOSPROP**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/用户QOS控制/业务QOS控制/业务质量属性/查询QoS属性（LST QOSPROP）_82837652.md) 命令，查询PCC策略组信息是否正确。

    b. 执行 [**EXP MML**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
