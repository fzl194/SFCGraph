## [操作场景](#ZH-CN_OPI_0161317192)

IPsec功能调测用于检查采用IKE方式建立IPsec隧道的各项配置是否正确，以保证IKE自动协商可以正常创建和维护安全联盟，使IPsec功能正常运行。

本章节以普通IPv4 IPsec隧道场景为例进行说明。

**图1** IPv4 IPsec组网图

<br>

![](调测IPsec功能_61317192.assets/zh-cn_image_0279003331.png)

网络环境描述如下：

- 网络A属于10.1.1.0/24子网，通过接口interface2与 Device A 连接。
- 网络B属于10.1.2.0/24子网，通过接口interface2与 Device B 连接。
- Device A 和 Device B 路由可达，假设Device A往Device B方向下一跳地址为172.16.163.2/24，Device B往Device A方向下一跳地址为172.16.169.2/24。

*表1 设备接口IP地址（普通IPv4 IPsec隧道）*

| 设备名称 | 接口 | IP地址 |
| --- | --- | --- |
| Device A VNRS微服务 | Ethernet66/0/5 | 10.1.1.1/24 |
| Device A VNRS微服务 | Ethernet66/0/6 | 172.16.163.1/24（IPsec报文的本端物理出接口） |
| Device A VNRS微服务 | Tunnel10 | 192.168.1.1/32（需要双配） |
| Device B VNRS微服务 | Ethernet66/0/5 | 10.1.2.1/24 |
| Device B VNRS微服务 | Ethernet66/0/6 | 172.16.169.1/24（IPsec报文的本端物理出接口） |
| Device B VNRS微服务 | Tunnel10 | 192.168.1.2/32（需要双配） |
| Device A IPsec微服务 | Tunnel10 | 192.168.1.1/32（IKE协商IP地址） |
| Device B IPsec微服务 | Tunnel10 | 192.168.1.2/32（IKE协商IP地址） |
| PCA | - | 10.1.1.2/24 |
| PCB | - | 10.1.2.2/24 |
| - | Device A侧网关接口 | 172.16.163.2/24 |
| - | Device B侧网关接口 | 172.16.169.2/24 |

## [操作步骤](#ZH-CN_OPI_0161317192)

1. 检查本端设备和对端设备是否正常连通。
  在 “ MML命令行 ” 窗口 上执行：
  **[PING](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/系统管理/系统维护/Ping和Tracert/Ping/IP Ping（PING）_16499685.md)**
    - **预期结果**：收到对端网元的响应，请执行[步骤 2](#ZH-CN_OPI_0161317192__step19720142516325)。
      ```
      PING: IPVERSION=IPv4, DESTIPV4ADDRESS="172.16.169.1";

        PING 172.16.169.1: 56  字节，按CTRL_Q中断

          答复从 172.16.169.1: 字节数=56 顺序号=1 生存时间=255 往返时间=1 毫秒

          答复从 172.16.169.1: 字节数=56 顺序号=2 生存时间=255 往返时间=1 毫秒

          答复从 172.16.169.1: 字节数=56 顺序号=3 生存时间=255 往返时间=1 毫秒

          答复从 172.16.169.1: 字节数=56 顺序号=4 生存时间=255 往返时间=1 毫秒

          答复从 172.16.169.1: 字节数=56 顺序号=5 生存时间=255 往返时间=1 毫秒

        --- 172.16.169.1 ping 统计 ---
          发送包数: 5
          接收包数: 5
          0.00% 包丢失
          往返时间 最小/平均/最大 = 1/1/1 毫秒

      共有9个报告
      ---    END
      ```
    - **异常结果**：没有收到对端网元的响应，请参考[Ping不通的定位思路](../../../../../网络运维/故障处理/UDG故障案例/IP转发故障处理/Ping不通的定位思路_19947761.md)排查故障。
2. 检查本端IPsec隧道接口状态。
  在 “ MML命令行 ” 窗口 上执行：
  **[DSP IFSTATUS](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/IP服务/接口管理/接口状态信息/查询接口状态信息（DSP IFSTATUS）_49960906.md)**
    - **预期结果**：接口物理状态和协议状态都是Up，请执行[步骤 5](#ZH-CN_OPI_0161317192__step12764817135611)。“接口协议状态”显示接口的IPv4协议状态，“接口IPv6协议状态”显示接口的IPv6协议状态，请根据实际组网查看对应的接口协议状态。
      ```
      DSP IFSTATUS: IFNAME="Tunnel10";
      ```
      ```
      RETCODE = 0  操作成功  
      结果如下
      -------------------------
      接口名                  接口物理状态                接口协议状态                接口IPv6协议状态                接口带宽       

      Tunnel10             接口Up                      接口Up                      接口Down                        0Mbps 
      (结果个数 = 1) 
      ---   END
      ```
    - **异常结果**：接口物理状态和协议状态不是Up，请执行[步骤 3](#ZH-CN_OPI_0161317192__step67002321031)
3. 检查本端VNRS微服务和IPsec微服务的IPsec隧道接口IP地址是否配置一致。
  在 “ MML命令行 ” 窗口 上执行：
  **[LST IFIPV4ADDRESS](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/IP服务/接口管理/IPv4地址/查询接口IPv4地址（LST IFIPV4ADDRESS）_49960942.md)**
  在 “MML命令行” 窗口 上执行：
  **[LST IFIPV4ADDRESSIPSEC](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/接口管理/IPv4地址/查询接口IPv4地址（LST IFIPV4ADDRESSIPSEC）_80751064.md)**
    - **预期结果**：两个微服务的IPsec隧道接口的IP地址一致，请执行[步骤 4](#ZH-CN_OPI_0161317192__step961115441338)。
      ```
      LST IFIPV4ADDRESS: IFNAME="Tunnel10";
      ```
      ```
      RETCODE = 0  操作成功  

      结果如下
      -------------------------
                       接口名  =  Tunnel10             
                     IPv4地址  =  
      192.168.1.1 
        
                 IPv4地址掩码  =  255.255.255.0         
                 IPv4地址类型  =  主地址 
      (结果个数 = 1) 
      ---   END
      ```
      ```
      LST IFIPV4ADDRESSIPSEC: IFNAME="Tunnel10";
      ```
      ```
      RETCODE = 0  操作成功  

      结果如下
      -------------------------
                       接口名  =  Tunnel10             
                     IPv4地址  =  
      192.168.1.1 
        
                 IPv4地址掩码  =  255.255.255.0         
                 IPv4地址类型  =  主地址 
      (结果个数 = 1) 
      ---   END
      ```
    - **异常结果**：两个微服务的IPsec隧道接口的IP地址不一致，请按照网络规划修改IP地址。
4. 检查IPsec隧道接口是否绑定了IPsec策略。
  在 “MML命令行-UDG” 窗口上执行：
  [**LST IPSECINTFCFGIPSEC**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/IP安全管理/互联网密钥交换/IPsec接口配置/查询IPsec隧道接口（LST IPSECINTFCFGIPSEC）_26150755.md)
    - **预期结果：** 绑定的IPsec策略符合规划，请执行 [步骤 8](#ZH-CN_OPI_0161317192__step1037991120366) 。
      ```
      LST IPSECINTFCFGIPSEC:INTERFACENAME="Tunnel 0/0/1";
      ```
      ```
      RETCODE = 0  操作成功

      结果如下
      -------------------------
              接口名称  =  Tunnel10
      Tunnel口协议类型  =  IPv4IP安全
              策略名称  =  map1
            源接口名称  =  NULL
      (结果个数 = 1)
      ---    END
      ```
    - **异常结果：** 未绑定IPsec策略或绑定错误，请按照网络规划重新绑定IPsec策略。
5. 检查本端IPsec隧道接口和对端IPsec隧道接口是否正常连通。
  在 “ MML命令行 ” 窗口 上执行：
  **[PING](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/系统管理/系统维护/Ping和Tracert/Ping/IP Ping（PING）_16499685.md)**
    - **预期结果**：收到对端设备的响应，请执行[步骤 6](#ZH-CN_OPI_0161317192__step536859697211511)。
      ```
      PING: IPVERSION=IPv4, DESTIPV4ADDRESS="192.168.1.2";

        PING 192.168.1.2: 56  字节，按CTRL_Q中断

          答复从 192.168.1.2: 字节数=56 顺序号=1 生存时间=255 往返时间=1 毫秒

          答复从 192.168.1.2: 字节数=56 顺序号=2 生存时间=255 往返时间=1 毫秒

          答复从 192.168.1.2: 字节数=56 顺序号=3 生存时间=255 往返时间=1 毫秒

          答复从 192.168.1.2: 字节数=56 顺序号=4 生存时间=255 往返时间=1 毫秒

          答复从 192.168.1.2: 字节数=56 顺序号=5 生存时间=255 往返时间=1 毫秒

        --- 192.168.1.2 ping 统计 ---
          发送包数: 5
          接收包数: 5
          0.00% 包丢失
          往返时间 最小/平均/最大 = 1/1/1 毫秒

      共有9个报告
      ---    END
      ```
    - **异常结果**：没有收到对端设备的响应，请执行[步骤 8](#ZH-CN_OPI_0161317192__step1037991120366)。
6. 查询IKE安全联盟。
  在 “MML命令行-UDG” 窗口上执行：
  [**DSP IKESA**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/IP安全管理/互联网密钥交换/IKE安全联盟/显示IKE安全联盟（DSP IKESA）_25912245.md)
    - **预期结果：** 可以查询到IKE SA信息，且 “SA标识” 不等于NO STATE，请执行 [步骤 7](#ZH-CN_OPI_0161317192__step11142164211511) 。

    - **异常结果：** 未查询到IKE SA信息，或者 “SA标识” 等于NO STATE，请执行 [步骤 8](#ZH-CN_OPI_0161317192__step1037991120366) 。
7. 查询IKE IPsec安全联盟。
  在 “MML命令行-UDG” 窗口上执行：
  [**DSP IKEIPSECSA**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/IP安全管理/互联网密钥交换/IKE IPsec安全联盟/显示IKE IPsec安全联盟（DSP IKEIPSECSA）_80910988.md)
    - **预期结果：** 可以查询到IKE IPsec SA信息，调测结束。
    - **异常结果：** 未查询到IKE IPsec SA信息，可能是IPsec安全提议配置错误，或者其它配置问题，请执行 [步骤 8](#ZH-CN_OPI_0161317192__step1037991120366) 。
8. 检查IPSec相关配置。如果与网络规划不一致，请修改。
    - 查询IKE对等体。
      在 “MML命令行-UDG” 窗口上执行：
      [**LST IKEPEER**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/IP安全管理/互联网密钥交换/IKE对等体/查询IKE对等体（LST IKEPEER）_80432528.md)
    - 查询IKE安全提议。
      在 “MML命令行-UDG” 窗口上执行：
      [**LST IKEPROPOSAL**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/IP安全管理/互联网密钥交换/IKE安全提议/查询IKE提议（LST IKEPROPOSAL）_25830693.md)
    - 查询IPsec策略。
      在 “MML命令行-UDG” 窗口上执行：
      [**LST IPSECPOLICY**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/IP安全管理/互联网密钥交换/IPsec策略/查询IPsec策略（LST IPSECPOLICY）_80751066.md)
    - 查询IPsec策略的安全提议。
      在 “MML命令行-UDG” 窗口上执行：
      [**LST PROPATTACHIPSECPROPOSAL**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/IP安全管理/互联网密钥交换/绑定的IPsec安全提议/查询IPsec策略绑定提议（LST PROPATTACHIPSECPROPOSAL）_25830695.md)
    - 查询IPsec安全提议。
      在 “MML命令行-UDG” 窗口上执行：
      [**LST IPSECPROPOSALIPSEC**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/IPSEC功能管理/IP服务/IP安全管理/IPsec/IPsec提议/查询IPsec提议（LST IPSECPROPOSALIPSEC）_80910992.md)
