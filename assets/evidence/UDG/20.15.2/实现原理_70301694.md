# 实现原理

融合计费流程如 [图1](#ZH-CN_TOPIC_0170301694__fig1498485792219) 所示。

**图1** 融合计费流程

<br>

![](实现原理_70301694.assets/zh-cn_image_0170301707.png)

1：用户发起PDU会话建立请求。

2：SMF向CHF发起Charging Data Request[Initial]消息，消息中携带Charging ID、切片信息、CC、Multiple Unit Usage等信息，请求建立计费会话。

3：CHF向SMF返回Charging Data Response[Initial]消息，消息中携带计费Trigger，Multiple Unit Information等。

4：SMF发送PFCP Session Establishment Request消息，请求建立PFCP会话。消息中携带Create URR（Usage Reporting Rule）信元组，指示计费相关信息。Create URR包含的主要子信元如 [表1](#ZH-CN_TOPIC_0170301694__table5217133513420) ：

> **说明**
> - 对于一个业务，只能有一种计费类型，即离线计费或在线计费。
> - 计费类型全局规划，PGW-U/UPF可以通过[**ADD URR**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则/增加URR（ADD URR）_82837629.md)本地配置。

*表1 Create URR包含的主要子信元*

| 子信元 | 信元解释 |
| --- | --- |
| URR ID | 唯一标识一个Create URR。 |
| Measurement Method | 标识计费方式，包括流量计费、时长计费或事件计费。 |
| Reporting Triggers | 标识计费上报的触发条件，包含Periodic Reporting（定期上报）、Volume Quota（流量配额）、Time Quota（时间配额）、Volume Threshold（流量阈值）、Time Threshold（时间阈值）、Linked Usage Reporting（关联URR）等。 |
| Volume Quota | 流量配额。 |
| Time Quota | 时间配额。 |
| Volume Threshold | 标识流量阈值，包含总流量、上行流量和下行流量。 |
| Time Threshold | 标识时间阈值。 |
| Linked URR ID | 上报某个URR时，也会上报其关联的URR，例如A link B，当B上报时，也会上报A。 |

5：PGW-U/UPF返回PFCP Session Establishment Response消息，PFCP会话建立成功。

6：PDU会话建立成功。

7：用户访问数据业务，当到达下发的阈值或配额时，PGW-U/UPF发送PFCP Session Report Request消息，消息中携带Usage Report信元组，记录用户使用的流量、时间等信息。Usage Report包含的主要子信元如 [表2](#ZH-CN_TOPIC_0170301694__table18994141118495) ：

> **说明**
> 当达到阈值或配额耗尽时，触发PGW-U/UPF发送PFCP Session Report Request消息申请新配额，并启动定时器，定时器时长= **T3RESPONSE** * **N3REQUEST** +4秒（ **T3RESPONSE** 及 **N3REQUEST** 参数由 [**SET UPPFCPPATH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/路径管理/PFCP路径管理/PFCP参数管理/PFCP路径参数/设置PFCP路径相关属性（SET UPPFCPPATH）_82837240.md) 命令配置），当定时器超时，未收到配额更新消息时，PGW-U/UPF阻塞业务。

*表2 Usage Report包含的主要子信元*

| 子信元 | 信元解释 |
| --- | --- |
| URR ID | 标识上报的URR。 |
| UR-SEQN | 唯一标识URR的Usage Report。 |
| Start Time | 起始时间。 |
| End Time | 结束时间。 |
| Volume Measurement | 标识流量使用情况，包含总流量、上行流量和下行流量。 |
| Duration Measurement | 标识时长使用情况：<br>- 当Measurement Information的ISTM携带为0或不携带，时长统计从UPF首次收到业务数据包时开始，直到业务终止时结束。<br>- 当Measurement Information的ISTM携带为1，时长统计从UPF首次收到授权的配额成功时开始，直到用户去激活时或者该配额管理实例终止结束。 |
| Time of First Packet | 首包时间。 |
| Time of Last Packet | 尾包时间。 |

8：SMF返回PFCP Session Report Response响应消息。

9~10：SMF向CHF发起Charging Data Request[Update]消息，消息中携带Trigger、Multiple Unit Usage等信息，请求更新计费会话。CHF向SMF返回Charging Data Response[Update]消息。
