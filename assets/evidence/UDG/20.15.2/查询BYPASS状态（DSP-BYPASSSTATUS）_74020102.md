#### [命令功能](#ZH-CN_TOPIC_0274020102)

![](查询BYPASS状态（DSP BYPASSSTATUS）_74020102.assets/notice_3.0-zh-cn.png)

该功能仅支持华为虚拟化层软件FusionSphere，不支持第三方虚拟化层软件。

本命令用于查询节点当前的BYPASS状态。

若需要查询指定节点的BYPASS状态，需要输入“网元ID”和“节点名称”参数。

> **说明**
> 该命令仅在Full-stack虚机场景下支持。

#### [注意事项](#ZH-CN_TOPIC_0274020102)

无。

#### [参数说明](#ZH-CN_TOPIC_0274020102)

| **参数标识** | **参数名称** | **参数说明** |
| --- | --- | --- |
| APPID | 网元ID | 可选必选说明：必选参数。<br>参数含义：用于指示系统需要查询指定网元ID下的节点BYPASS状态。<br>取值范围：0～65535。<br>默认值：无。<br>配置原则：<br>“网元ID”<br>获取方式为：点击<br>“首页”<br>，查看<br>“应用ID”<br>即可。 |
| NODENAME | 节点名称 | 可选必选说明：可选参数<br>参数含义：用于指示系统需要查询哪个节点的BYPASS状态。<br>取值范围：不超过128位字符串。<br>默认值：无。<br>配置原则：<br>- 若不输入，则表示查询所有节点的信息。<br>- 用户可以通过[**DSP NODE**](../节点管理/节点查询（DSP NODE）_71678755.md)命令查看相应的节点名称。 |

#### [使用实例](#ZH-CN_TOPIC_0274020102)

1. 查询“网元ID”为“0”的所有节点BYPASS状态：
  ```
  %%DSP BYPASSSTATUS: APPID=0;%%
  RETCODE = 0  操作成功

  操作结果如下
  ------------
  网元ID  节点IP           节点名称         查询状态      手工进入BYPASS      节点BYPASS状态      详细信息
  0       10.0.0.1         10.0.0.1         Success       No                  Normal              Success             
  0       10.0.0.2         10.0.0.2         Success       No                  Normal              Success  
  0       10.0.0.3         10.0.0.3         Success       No                  Normal              Success             
  (结果个数 = 3)

  ---    END
  ```
2. 查询“网元ID”为“0”，“节点名称”为“10.0.0.1”的节点BYPASS状态：
  ```
  %%DSP BYPASSSTATUS: APPID=0, NODENAME="10.0.0.1";%%
  RETCODE = 0  操作成功

  操作结果如下
  ------------
              网元ID  =  0
              节点IP  =  10.0.0.1
            节点名称  =  10.0.0.1
            查询状态  =  Success
      手工进入BYPASS  =  Yes
      节点BYPASS状态  =  Bypass
            详细信息  =  Success
  (结果个数 = 1)

  ---    END
  ```

#### [输出结果说明](#ZH-CN_TOPIC_0274020102)

命令执行正常，会返回命令执行成功的提示信息。输出结果说明如 [表1](#ZH-CN_TOPIC_0274020102__table819976173217) 所示。

命令执行异常，会返回提示对应的失败信息。错误码列表如 [表2](#ZH-CN_TOPIC_0274020102__table698716490471) 所示。

*表1 输出项说明*

| 输出项名称 | 输出项解释 |
| --- | --- |
| 网元ID | 用于返回查询网元的ID。 |
| 节点IP | 用于返回查询节点的IP地址。 |
| 节点名称 | 用于返回查询的节点名称。 |
| 查询状态 | 用于返回本节点查询是否成功。 |
| 手工进入BYPASS | 用于返回查询的节点的手动设置BYPASS的操作状态。<br>取值范围：<br>- “No”：用于返回当前节点未设置手动进入Bypass。<br>- “Yes”：用于返回当前节点设置了手动进入Bypass。<br>- “Unknown”：用于返回当前节点的Bypass状态为Unknown状态。 |
| 节点BYPASS状态 | 用于返回查询的节点当前所处的BYPASS状态。<br>取值范围：<br>- “Normal”：用于返回当前节点的Bypass状态为非Bypass状态。<br>- “Bypass”：用于返回当前节点的Bypass状态为Bypass状态。<br>- “Unknown”：用于返回当前节点的Bypass状态为Unknown状态。 |
| 详细信息 | 用于返回成功和失败的详细信息。 |

*表2 错误码列表*

| 错误码 | 错误码解释 | 原因分析 | 处理建议 |
| --- | --- | --- | --- |
| 130001 | 没有符合条件的节点 | 所要查询的节点或者网元ID不存在，无相应BYPASS信息。 | 按照<br>[**DSP NODE**](../节点管理/节点查询（DSP NODE）_71678755.md)<br>命令查询的所有节点信息，选择确实存在的节点，查询其相应的BYPASS状态。 |
| 130002 | 操作失败 | bypass模式未激活。 | 请联系<br>华为<br>技术支持。 |
| 130004 | 部分成功 | 部分节点接口调用异常或者设置处理异常。 | 请联系<br>华为<br>技术支持。 |
| 130009 | 输入参数不合法 | 设置的参数值不在规定范围之内。 | 按照规定的范围进行参数值的设置。 |
