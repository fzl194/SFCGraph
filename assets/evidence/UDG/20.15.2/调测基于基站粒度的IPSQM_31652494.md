# 调测基于基站粒度的IPSQM

- [操作场景](#ZH-CN_OPI_0000001131652494__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001131652494__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001131652494__1.3.3)

## [操作场景](#ZH-CN_OPI_0000001131652494)

当运营商部署基于基站粒度的IPSQM，以实现对指定基站的下行用户面流量进行静态整形时，需对 UDG 的基于基站粒度的IPSQM功能进行调测，确保本功能可以正常使用。

本处以对4G用户进行IPSQM整形为例进行说明，5G用户类似。

> **说明**
> 适用于 SGW-U、UPF 。

## [必备事项](#ZH-CN_OPI_0000001131652494)

前提条件

- 请仔细阅读[GWFD-110941 基于基站粒度的IPSQM特性概述](GWFD-110941 基于基站粒度的IPSQM特性概述_77811903.md)。
- 完成[激活基于基站粒度的IPSQM](激活基于基站粒度的IPSQM_77772003.md)。

| 用户信息查询 | 用户IMSI号（IMSI） | 460000123456786 | 测试终端自带 | - |
| 用户信息查询 | 用户IMSI号（IMSI） | 460000123456789 | 测试终端自带 | - |

工具

- 测试终端
- 第三方抓包工具

## [操作步骤](#ZH-CN_OPI_0000001131652494)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令查询 UDG 上基于基站粒度的IPSQM功能对应的License配置开关是否打开。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5IPSM01";
  ```
    - 如果 “Switch” 为 “ENABLE” ，请执行[2](#ZH-CN_OPI_0000001131652494__cmd723752233175820)。
    - 如果 “Switch” 为 “DISABLE” ，则执行 [**SET LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md) 命令打开本特性对应的License配置开关。
2. 打开接入侧/PDN侧镜像接口上的抓包工具，准备抓取 测试终端 的出入报文。
3. 激活RAN下的多个用户，使用发包工具发送用户报文，访问业务的带宽大于所配置CIR。
4. 查看下行报文出流量是否符合配置的IPSQM静态整形的速率，如 [图1](#ZH-CN_OPI_0000001131652494__fig1) 所示。
  **图1** 下行报文SGi/S1-U接口抓包信息

  <br>

  ![](调测基于基站粒度的IPSQM_31652494.assets/zh-cn_image_0000001143224108.png "点击放大")
    - 如果符合，则带宽管理功能正常，调测结束。
    - 如果不符合，则带宽管理功能不正常，请执行 [步骤 5](#ZH-CN_OPI_0000001131652494__step4) 。
5. 执行 [**LST IPSQMSHAPING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/IPSQM控制/IPSQM Shaping配置/查询IPSQM整形配置（LST IPSQMSHAPING）_21153440.md) 命令，查看IPSQM静态整形带宽配置。
  ```
  LST IPSQMSHAPING:;
  RETCODE = 0  操作成功
  IPSQM配置信息
  ------------------------
                    
                           IP地址类型   =  IPv4
                             对端地址   =  192.168.1.235
                              VPN实例   =  _public_
             承诺信息速率（兆比特/秒）  =  100
                             描述信息   =  NULL
  (结果个数 = 1)
  ---    END
  ```
    - 如果配置与规划值一致，请执行[步骤 6](#ZH-CN_OPI_0000001131652494__step5)。
    - 如果配置与规划值不一致，请参考[激活基于基站粒度的IPSQM](激活基于基站粒度的IPSQM_77772003.md)重新配置，重新执行[步骤 3](#ZH-CN_OPI_0000001131652494__step2)，注意要重新激活用户。
6. 执行 [**LST IPSQMQDEPTH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/IPSQM控制/IPSQM整形队列深度/查询IPSQM缓存队列深度（LST IPSQMQDEPTH）_21153442.md) 命令，查询整形队列深度。
  ```
  LST IPSQMQDEPTH:;
  RETCODE = 0  操作成功
  IPSQM缓存队列深度
  -----------------
  队列深度（报文个数）  =  256
  (结果个数 = 1)
  ---    END
  ```
    - 如果配置与规划值一致，请执行[7](#ZH-CN_OPI_0000001131652494__cmd73876688183107)。
    - 如果配置与规划值不一致，请参考[激活基于基站粒度的IPSQM](激活基于基站粒度的IPSQM_77772003.md)重新配置，重新执行[步骤 3](#ZH-CN_OPI_0000001131652494__step2)，注意要重新激活用户。
7. 收集信息并寻求技术支持。
    a. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    b. 收集归纳所有信息并联系华为技术支持解决。
