# 地址分配方式

UDG 本地地址池分配地址有如下多种方式，也支持这些场景的任意叠加组合方式，例如基于位置+APN/DNN、SMF+APN/DNN的地址分配，多样的地址分配方式可以满足运营商多样化的地址管理和网络运维诉求。

- 基于APN/DNN分配地址
- 基于位置分配地址（详细请参考[GWFD-020421 基于位置的地址分配](../../GWFD-020421 基于位置的地址分配_02846765.md)）
- 基于SMF分配地址

*表1 UDG 地址分配方式及其使用场景*

| 地址分配方式 | 使用场景 |
| --- | --- |
| 基于APN/DNN分配地址 | 按照APN/DNN规划和管理用户地址池，适用于根据APN规划地址池的场景。 |
| 基于位置分配地址 | 按照位置规划和管理用户地址池，根据用户所处的位置分配特殊的IP地址，实现IP地址细分，便于运营商加强IP管理。 |
| 基于SMF分配地址 | 按照SMF Node ID规划和管理用户地址池，适用于运营商部署SMF内置BSF（Binding Support Function）的场景。 |
| 基于RADIUS下发地址池名称分配地址 | 按照运营商规划，适用于Radius管理端到端动态地址分配的场景。 |

上述方式可任意叠加组合，为不同的用户群选择不同的地址分配方式。 UDG 上地址分配可以基于全局（ [**SET IPALLOCRULE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/地址分配规则/设置地址分配规则（SET IPALLOCRULE）_82837152.md) ）、APN/DNN（ [**SET APNIPALLOCRULE**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/基于APN的地址分配规则/基于APN配置地址分配规则（SET APNIPALLOCRULE）_82837165.md) ）分别使能，如果设置了基于APN的地址分配规则，则使用APN下配置的地址分配规则；否则就使用基于全局的地址分配规则。

UDG 上地址分配规则支持三级优先级规则，三级优先级可分别选择上述APN/DNN、位置、SMF进行组合，如果第一优先级规则匹配成功，则使用第一优先级类型的地址池进行地址分配，否则依次是第二优先级和第三优先级类型的地址池进行地址分配。如果都匹配失败，则地址分配失败。
