# 其他增值功能

*表1 用户面地址分配的其他增值功能*

| 增值功能 | 详细解释 |
| --- | --- |
| 地址池优先级功能 | 同一粒度（APN/DNN、位置、SMF）下绑定多个地址池时，支持按地址池优先级分配地址。只有同类地址池之间才有优先级比较，不同类型的地址池间（IPv4和IPv6）没有优先级比较。<br>说明：优先级仅限于地址池，地址池下的地址段没有优先级顺序。 |
| 地址延迟释放功能 | 当新用户接入时，如果获取到的地址为之前用户刚释放的地址，则对地址敏感的周边网元其数据处理可能会存在异常（如可能存在将该用户与之前用户视为同一个用户）。为避免这类异常情况的发生，用户去激活后其使用的地址不能被立即释放，而适当等待一段时间后再释放。 |
| 地址租期续约功能 | 与地址延迟释放功能相反，当用户重新接入时，如果需要继续处理之前在网时可接续的业务，则<br>UDG<br>支持将该用户之前使用的IP地址再次分配给该用户，以接续中断的业务。 |
| 地址强制回收功能 | 如果要动态修改或删除某个地址池或地址段，但该地址池或地址段内已经有地址分配完成，这时可通过地址强制回收功能回收已分配的地址。<br>说明：执行地址强制回收之前，请首先锁定地址池或地址段。强制回收正在使用的地址，会使正常在线的用户将被强制下线，中断用户访问的业务。 |
| 防止地址冲突功能 | 当本地地址池中的某些IPv4/IPv6地址与系统或其他设备的地址冲突时，可以通过命令标识该IPv4/IPv6地址为冲突地址，这样可以限制这些IPv4/IPv6地址分配给用户，避免地址冲突。 |
