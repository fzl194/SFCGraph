# 调测N4/Sxb接口的GTP路径检测功能

- [操作场景](#ZH-CN_OPI_0000001144784945__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001144784945__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001144784945__1.3.3)

## [操作场景](#ZH-CN_OPI_0000001144784945)

N4/Sxb接口的GTP路径检测功能调测用于校验本局与对端SMF/PGW-C的GTP路径能否正常工作以及数据配置是否正确。

## [必备事项](#ZH-CN_OPI_0000001144784945)

前提条件

- 请仔细阅读[GWFD-010102 路径管理特性概述](GWFD-010102 路径管理特性概述_66891578.md)。
- 完成[配置GTP/PFCP路径管理参数](配置GTP_PFCP路径管理参数_24082055.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | 用户IMSI号（IMSI） | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN名称（APN） | apn-test | 已配置数据中获取 | 取自MS接入运营商自身IP网络中配置的APN实例名。 |

## [操作步骤](#ZH-CN_OPI_0000001144784945)

1. 测试终端使用“apn-test”APN发起接入网络请求。
2. 执行 [**LST UPN4UPATH**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/路径管理/GTP路径管理/GTP协议参数管理/GTP路径全局参数/查询路径相关属性（LST UPGTPPATH）_82837228.md) 命令，查看GTP路径配置信息是否与规划一致。
  ```
  LST UPN4UPATH:;
  ```
  ```
  RETCODE = 0  操作成功。

  GTP路径配置
  -----------
                 N4接口数据路径Echo开关  =  使能            
          N4接口发送GTP心跳请求的间隔时间  =  60
          N4接口的GTP请求消息的重发时间间隔  =  3
      N4接口的GTP请求消息的最大尝试发送次数  =  5
       是否去活N4 GTP-U路径上已激活的上下文  =  使能
     N4接口的GTP-U路径断告警后发送心跳消息的次数  =  0
  (结果个数 = 1)
  ---    END
  ```
    - 如果一致，请执行[步骤 3](#ZH-CN_OPI_0000001144784945__step1707882943201622)。
    - 如果不一致，请[配置GTP/PFCP路径管理参数](配置GTP_PFCP路径管理参数_24082055.md)参考重新配置。
3. 通过 [UDG N4接口跟踪](../../../../../网络运维/日常维护/UDG基础运维操作/创建消息跟踪/创建接口跟踪/UDG N4接口跟踪_88792205.md) 观察GTP路径Echo探测是否正常。包括Echo探测请求间隔是否和配置信息一致，且能正常收到Echo响应。
    - 如果一致，请执行 [步骤 4](#ZH-CN_OPI_0000001144784945__step5476525171313) 。
    - 如果不一致 ，请执行 [步骤 5](#ZH-CN_OPI_0000001144784945__step7) 。
4. 断开UPF与SMF之间的路径连接，通过 [UDG N4接口跟踪](../../../../../网络运维/日常维护/UDG基础运维操作/创建消息跟踪/创建接口跟踪/UDG N4接口跟踪_88792205.md) 观察GTP路径Echo探测失败后，是否去活用户。以 [步骤 2](#ZH-CN_OPI_0000001144784945__step38101840368) 为例，每隔60秒发送一次Echo Request给SMF，连续5次无响应，在第6次发送Echo Request消息后去活用户。
    - 如果一致，调测结束。
    - 如果不一致 ，请执行 [步骤 5](#ZH-CN_OPI_0000001144784945__step7) 。
5. 收集信息并寻求技术支持。
    a. 保存跟踪报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
