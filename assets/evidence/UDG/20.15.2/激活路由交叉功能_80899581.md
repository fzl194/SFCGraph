## [操作场景](#ZH-CN_OPI_0000002080899581)

UE激活后，能正常访问网络A中的服务器。另一网络B和网络A之间网络隔离，UE还需访问网络B中的指定服务器（服务器IP需固定，此处IP地址以10.8.8.8为例）。

该场景下，网络A和网络B是两个不同的VPN网络，需通过路由交叉功能，实现UE激活后，既能访问网络A中的服务器，又能访问网络B中的定制服务器。

  请参见 **操作步骤** 中的 [步骤 7](#ZH-CN_OPI_0000002080899581__step1533814599351) ~ [步骤 8](#ZH-CN_OPI_0000002080899581__step55314368506) 。

## [操作步骤](#ZH-CN_OPI_0000002080899581)

1. 配置外联口的VPN实例。
  ```
  ADD L3VPNINST:VRFNAME="
  VRF_networkB
  ";
  ADD VPNINSTAF:VRFNAME="
  VRF_networkB
  ",AFTYPE=ipv4uni;
  ```
2. 关闭自动配置开关。
    a. 查询自动配置开关。
      ```
      LST AUTOCONFIG:;
      ```
      如果自动配置开关值为 “TRUE” ，需要执行如下步骤关闭自动配置开关。
    b. （可选）查询自动部署维护助手“autoscaling_autoconfig.py”的状态，确保状态为“准备好状态（Ready）”或者查询无该脚本，以保证当前设备不在自动部署过程中。因为若系统当前有正在进行的自动部署，而自动部署生效有时延，此时若关闭自动配置开关，会影响系统配置效果。
      ```
      DSP OPSASSISTSTATE:;
      ```
    c. （可选）关闭自动配置开关。自动化配置命令只有在自动配置开关为关闭的状态下才可执行，因此需要提前关闭。
      ```
      SET AUTOCONFIG:SWITCHFLAG=FALSE;
      ```
3. 配置外联口自动部署模板。
  //如下脚本以SR-IOV bonding组网为例。SR-IOV bonding组网时， “AUTOCFGIFTYPE” 取值为 “ETHTRUNK” ；非SR-IOV bonding组网时， “AUTOCFGIFTYPE” 取值为 “VNIC” 。
  ```
  ADD AUTOSCALINGSERVICE:SERVICENAME="ServName_networkB",VPNNAME="
  VPN_networkB
  ",AFTYPE=IPv4,IPALLOCTYPE4=USER_CONFIG,AUTOCFGIFTYPE=ETHTRUNK,ETHTRUNKTMPID=
  6
  ,VLANID=
  20
  , BEGINADDR4="
  192.168.20.2
  ", ENDADDR4="
  192.168.20.126
  ", MASKLEN=
  25
  ; 
  ```
4. 配置BFD会话。
    - BFD双向检测时：
      ```
      ADD AUTOSCALINGSRBFD: SERVICENAME="ServName_networkB", IPVERSION=IPv4, MINRXINTERVAL=500, MINTXINTERVAL=500, MULTIPLIER=4;
      ```
    - BFD 单臂echo检测时：
      ```
      ADD AUTOSCALINGBFD: TEMPLATENAME="BFD_10", BFDTYPE=Static, SERVICENAME="ServName_networkB", IPVERSION=IPv4, DESTADDR4="
      192.168.20.1
      ", MINECHORXINT=500, DETECTMULTI=4, ONEARMECHO=TRUE;
      ```
5. 配置路由交叉相关数据（上行路由）。
  //此处以采用单向BFD为例。采用BFD双向检测时，不需要配置BFDTEMPLATENAME参数。
  ```
  ADD AUTOSCALINGSRROUTE: SERVICENAME="ServName_networkB", VRFNAME="
  VRF_networkA
  ", IPVERSION=IPv4, PREFIX4="
  10.8.8.8
  ", MASKLENGTH=
  32
  , NEXTHOPALLOCTYPE4=CONFIG, NEXTHOP4="
  192.168.20.1
  ", BFDENABLE=TRUE, BFDTEMPLATENAME="BFD_10";
  ```
6. 打开自动配置开关。
  ```
  SET AUTOCONFIG: SWITCHFLAG=TRUE;
  ```
  > **说明**
  > - 从自动部署开关打开到自动部署功能生效有时延，因此执行其他操作前（例如手动部署路由互通、关闭自动部署开关），务必先执行 **[**DSP OPSASSISTSTATE**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/操作维护/系统调测/开放可编程系统/显示系统助手的当前信息（DSP OPSASSISTSTATE）_00601449.md)** 命令查询自动部署维护助手“autoscaling_autoconfig.py”的状态为 “准备好状态（Ready）” ，否则结果不可预期。
  > - 在网关侧上配置到VNF的回程路由，目的地址为VNF上各逻辑接口的IP地址，下一跳为VNF上配置的各逻辑接口对应的外联口的IP地址。
  >   需要说明的是，配置VNF与DN/PDN通信的回程路由时，目的地址为UE的网段地址。 下一跳为VNF上外联口的IP地址，由于自动部署时外联口的IP地址是系统自动分配的，因此建议首先通过命令 [**LST IFIPV4ADDRESS**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/VNRS功能管理/IP服务/接口管理/IPv4地址/查询接口IPv4地址（LST IFIPV4ADDRESS）_49960942.md) 查看外联口的IP地址。
7. 配置VRF_networkA（源VRF）的相关数据。
    a. 修改L3VPN实例地址族。
      ```
      MOD VPNINSTAF: VRFNAME="
      VRF_networkA
      ", AFTYPE=ipv4uni, VRFRD="
      1:1
      ", VRFLABELMODE=perInstance;
      ```
    b. 增加VPN Target。 “VRFRTTYPE” 须设置为 “export_extcommunity” 。
      ```
      ADD VPNTARGET: VRFNAME="
      VRF_networkA
      ", AFTYPE=ipv4uni, VRFRTTYPE=export_extcommunity, VRFRTVALUE="
      1:1
      ";
      ```
    c. 使能全局BGP。如已使能BGP，则此处不需要重复配置。
      ```
      SET BGP: BGPENABLE=TRUE, ASNUM="
      60001
      ";
      ```
    d. 增加BGP VPN实例。如已配置，则此处不需要重复配置。
      ```
      ADD BGPVRF: VRFNAME="
      VRF_networkA"
      , DEFAULTAFTYPE=ipv4uni, VRFRIDAUTOSEL=FALSE;
      ```
    e. 增加BGP引入路由。如已配置，则此处不需要重复配置。
      ```
      ADD IMPORTROUTE: VRFNAME="
      VRF_networkA
      ", AFTYPE=ipv4uni, IMPORTPROTOCOL=wlr, MEDENABLE=FALSE;
      ```
8. 配置VRF_networkB（目的VRF）的相关数据。
    a. 修改L3VPN实例地址族。
      ```
      MOD VPNINSTAF: VRFNAME="
      VRF_networkB
      ", AFTYPE=ipv4uni, VRFRD="
      2:2
      ", VRFLABELMODE=perInstance;
      ```
    b. 增加VPN Target。 “VRFRTTYPE” 须设置为 “import_extcommunity” 。
      ```
      ADD VPNTARGET: VRFNAME="
      VRF_networkB
      ", AFTYPE=ipv4uni, VRFRTTYPE=import_extcommunity, VRFRTVALUE="
      1:1
      ";
      ```
