#### [命令功能](#ZH-CN_CONCEPT_0000001549801990)

该命令用于查询在OSPF进程下区域配置。

#### [注意事项](#ZH-CN_CONCEPT_0000001549801990)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001549801990)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001549801990)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROCID | 进程号 | 可选必选说明：可选参数<br>参数含义：OSPF进程号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～4294967295。<br>默认值：无 |
| AREAID | 区域号 | 可选必选说明：可选参数<br>参数含义：OSPF区域号。<br>数据来源：对端协商<br>取值范围：IPv4地址类型。点分十进制格式。<br>默认值：无<br>配置原则：规划的邻居所在的区域必须一致。 |

#### [使用实例](#ZH-CN_CONCEPT_0000001549801990)

查询在OSPF进程下区域配置：

```
LST OSPFAREA:;
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000001549801990)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 进程号 | 该参数指定用户创建的OSPF进程号。 |
| 区域号 | 该参数指定用户创建的OSPF区域号。 |
| 区域描述 | 该参数指定OSPF区域的描述信息。 |
| 区域类型 | 该参数指定OSPF区域类型。Normal-普通区域;Stub-Stub区域，不引入外部路由;Nssa-Nssa区域，可以引入外部路由并传播到整个NSSA区域，但不会学习外部路由。 |
| 产生缺省7类LSA到NSSA区域 | 该参数指定配置在ABR或者ASBR上产生缺省的Type 7 LSA到NSSA区域。 |
| 不引入外部路由 | 该参数指定禁止在NSSA区域引入外部路由。 |
| 禁止ABR向NSSA区域内发送Summary LSA | 该参数指定禁止ABR向NSSA区域内发送其他区域的Summary LSA。 |
| 配置N-bit位 | 该参数指定在DD报文中配置N-bit位的标志。 |
| 转换后生成的Type5 LSA的FA配置为0.0.0.0 | 该参数指定将Type 7 LSA通过该NSSA ABR转换后生成的Type 5 LSA的FA（Forwarding Address ）配置为0.0.0.0。 |
| 转换路由器 | 该参数指定允许将NSSA区域中的多个ABR配置成转换路由器。 |
| 失效时间（s） | 该参数指定转换路由器的失效时间，仅当区域类型配置为NSSA时生效。 |
| 将生成的Type7 LSA的FA置为0.0.0.0 | 该参数指定在NSSA区域的ABR上引入外部路由时，将生成的NSSA LSA的FA置为0.0.0.0。 |
| 禁止ABR向Stub区域内发送Summary LSA | 该参数指定禁止ABR向Stub区域内发送其他区域的Summary LSA。 |
| 发送到Stub或NSSA区域的Type3缺省路由的开销 | 该参数指定OSPF发送到Stub或NSSA区域的Type 3缺省路由的开销。 |
| 使能LDP和OSPF同步功能 | 该参数指定使能LDP和OSPF同步功能。 |
| 保持最大开销值标志 | 该参数指定配置本地设备通告链路最大开销值的时间。 |
| 保持最大开销值时间间隔（s） | 该参数指定OSPF在本地设备的中保持通告最大开销值的时间。 |
