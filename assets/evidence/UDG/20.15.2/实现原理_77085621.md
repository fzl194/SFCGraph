# 实现原理

L3/L4层业务级累计流量的策略控制流程如 [图1](#ZH-CN_CONCEPT_0177085621__fig19641708587) 所示。

**图1** L3/L4层业务累计流量的策略控制流程图

<br>

![](实现原理_77085621.assets/zh-cn_image_0184812613.png)

1. UE发起PDU Session Establishment流程。
2. AMF进行PGW-C/SMF选择，然后向PGW-C/SMF发送Nsmf_PDUSession_CreateSMContext消息。
3. PGW-C/SMF收到Nsmf_PDUSession_CreateSMContext消息后，再发送Npcf_SMPolicyControl_Create Request (SmPolicyContextData)请求会话策略，其中suppFeat字段指示是否支持Usage Monitoring Control功能。
4. PCRF/PCF从UDR获取策略签约数据。
5. PCRF/PCF进行特性协商，同时进行策略决策生成会话管理策略（包括FUP相关策略）。PCRF/PCF给PGW-C/SMF回复Npcf_SMPolicyControl_Create Response（SmPolicyDecision）消息，包括下发业务级流量阈值。业务级累计流量在PccRule信元组里的refUmData中定义。refUmData值是通过umDecs里的umID引用的。SmPolicyDecision包含的主要信元如[表1](#ZH-CN_CONCEPT_0177085621__table10486838958)所示：
  *表1 SmPolicyDecision包含的主要信元*

  | 主信元 | 子信元 | 信元解释 |
  | --- | --- | --- |
  | PccRule | - | PCCRule。 |
  | PccRule | pccRuleId | PCC规则标识。 |
  | PccRule | flowInfos | 业务流描述。 |
  | PccRule | refUmData | 对使用监控数据决策的引用。即业务级累计流量在PCCRule中定义，通过umID引用。 |
  | umDecs | - | DecisionData。 |
  | umDecs | umId | 标识使用监控策略数据在一个PDU会话。 |
  | umDecs | volumeThreshold | 总流量阈值。 |
  | policyCtrlReqTriggers | - | 标识达到流量阈值后上报的触发条件，如US_RE。 |
  | suppFeat | - | 标识特性协商结果，如UMC。 |
6. PGW-C/SMF根据Npcf_SMPolicyControl_Create Response消息中pccRule和umDecs等生成N4接口的URR规则，由于PccRule只对指定业务流生效，该URR只绑定在指定业务流对应的PDRs上。同时PGW-C/SMF建立URR ID和UMID之间的映射。
7. PGW-C/SMF向PGW-U/UPF发送PFCP Session Establishment Request（Create PDR）消息，SDF Filter中的Flow Description可以从PccRule中的flowInfos提取。
8. PGW-C/SMF收到PFCP Session Establishment Response消息后进行后续相关流程处理，PDU会话建立成功。
9. 用户开始访问指定数据业务，PGW-U/UPF检测到该业务的流量阈值耗尽，发送PFCP Session Report Request消息通知PGW-C/SMF，消息中携带Usage Report信元组，记录用户使用的流量等信息。
10. PGW-C/SMF根据URR ID和UMID之间映射关系，向PCRF/PCF通知流量阈值耗尽，业务累计流量上报信息从N4接口中Usage Report 中提取。
11. PGW-C/SMF发送Npcf_SMPolicyControl_Update Request (SmPolicyUpdateContextData)消息通知PCRF/PCF流量阈值耗尽。
12. PCRF/PCF进行策略决策，不再下发流量阈值，同时更新QoS规则，对用户进行带宽控制，上行带宽为1 Mbit/s，下行带宽为2 Mbit/s。Npcf_SMPolicyControl_Update Response消息中携带的authSessAmbr信元组负责更新QoS规则。
13. PGW-C/SMF收到Npcf_SMPolicyControl_Update Response消息后，检查是否下发新的流量阈值。如果没有下发新的流量阈值，则通知PGW-U/UPF不再上报流量。
14. PGW-C/SMF发送PFCP Session Modification Request消息，通知PGW-U/UPF更新业务规则。

L7层业务级累计流量的策略控制流程如 [图2](#ZH-CN_CONCEPT_0177085621__fig34434598516) 所示。

**图2** L7层业务级累计流量的策略控制流程图

<br>

![](实现原理_77085621.assets/zh-cn_image_0184815697.png)

1. PCRF/PCF/PGW-C/SMF/PGW-U/UPF支持预定义规则配置，其中
    - PGW-C/SMF配置的预定义PCC规则名和PCRF/PCF保持一致。
    - PGW-C/SMF配置预定义URR ID和umID对应关系，umID和PCRF/PCF保持一致。
    - PGW-C/SMF配置的预定义规则名和PGW-U/UPF保持一致。
    - PGW-C/SMF配置的预定义URR ID和PGW-U/UPF保持一致。
2. UE发起PDU Session Establishment流程。
3. AMF进行PGW-C/SMF选择，然后向PGW-C/SMF发送Nsmf_PDUSession_CreateSMContext消息。
4. PGW-C/SMF收到Nsmf_PDUSession_CreateSMContext消息后，再发送Npcf_SMPolicyControl_Create Request (SmPolicyContextData)请求会话策略，其中suppFeat字段指示是否支持Usage Monitoring Control功能。
5. PCRF/PCF从UDR获取策略签约数据。
6. PCRF/PCF进行特性协商，同时进行策略决策生成会话管理策略（包括预定义规则）。PCRF/PCF给PGW-C/SMF回复Npcf_SMPolicyControl_Create Response（SmPolicyDecision）消息，包括下发业务级流量阈值。
  > **说明**
  > - 预定义规则在pccRules中定义，只需定义pccRuleId和refCondData（可选）。
  > - umDecs中定义流量阈值，但是umId和pccRule无绑定关系。
7. PGW-C/SMF根据预定义规则名读取该预定义规则配置，获得预定义URR ID和UMID的映射关系，同时生成预定义规则PDR和预定义URR。
8. PGW-C/SMF向PGW-U/UPF发送PFCP Session Establishment Request（Create PDR）消息，消息中下发预定义规则。Create PDR包含的主要子信元如[表2](#ZH-CN_CONCEPT_0177085621__table3305054161418)所示：
  *表2 Create PDR包含的主要子信元*

  | 子信元 | 信元解释 |
  | --- | --- |
  | Activate Predefined Rule | 预定义规则名。 |
  | URR ID | 标识上报的URR。<br>说明：URR ID采用最高位bit来区分预定义URR和动态URR。高位bit为1，代表预定义URR ID。 |
9. PGW-C/SMF收到PFCP Session Establishment Response消息后进行后续相关流程处理，PDU会话建立成功。
10. 用户开始访问数据业务，PGW-U/UPF检测到指定的L7业务的流量阈值耗尽，发送PFCP Session Report Request消息通知PGW-C/SMF，消息中携带Usage Report信元组，记录用户使用的流量等信息。
11. PGW-C/SMF根据URR ID和UMID之间映射关系，向PCRF/PCF通知指定L7业务流量阈值耗尽，业务累计流量上报信息从N4接口中Usage Report 中提取。
12. PGW-C/SMF发送Npcf_SMPolicyControl_Update Request (SmPolicyUpdateContextData)消息通知PCRF/PCF流量阈值耗尽。
13. PCRF/PCF进行策略决策，不再下发流量阈值，同时更新QoS规则，对用户进行带宽控制，上行带宽为1 Mbit/s，下行带宽为2 Mbit/s。Npcf_SMPolicyControl_Update Response消息中携带的authSessAmbr信元组负责更新QoS规则。
14. PGW-C/SMF收到Npcf_SMPolicyControl_Update Response消息后，检查是否下发新的流量阈值。如果没有下发新的流量阈值，则通知PGW-U/UPF不再上报流量。
15. PGW-C/SMF发送PFCP Session Modification Request消息，通知PGW-U/UPF更新业务规则。
