#### [命令功能](#ZH-CN_CONCEPT_0000206312635690)

**适用NF：PGW-U、UPF**

该命令用来配置UPF向PDN服务器自动发送探测消息的路径，以检查UPF和PDN服务器之间的路由是否正常。

#### [注意事项](#ZH-CN_CONCEPT_0000206312635690)

- 该命令执行后立即生效。
- 该命令最大记录数为128。
- 配置探测路径后，若LST PDNAUTOTSTSW已使能，即进行一次本条探测路径的探测。
- 若执行探测时，确定在地址池下无可以探测的地址，会跳过该路径探测。
- 同时使能多条探测配置时，探测会持续一段时间，如使用DSP PDNAUTOTSTRESULT未查询到对应结果，请稍后查询。
- 当DOMAINVALUE不配置时，使用默认域名探测。
- 对于DOMAINVALUE，特殊字符的输入要求可以参考STR PDNROUTETST注意事项部分。

#### [操作用户权限](#ZH-CN_CONCEPT_0000206312635690)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [使用实例](#ZH-CN_CONCEPT_0000206312635690)

增加探测路径名称为test，用于检查地址池名称为pool-test中发布路由网段和PDN服务器10.1.1.2之间的路由是否正常。向PDN服务器发送PING探测消息：

```
ADD PDNROUTETSTPATH: PATHNAME="test", PATHTYPE=DNS, IPPOOL="pool-test", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.2", TESTMETHOD=PING;
RETCODE = 0  Operation succeeded
```
