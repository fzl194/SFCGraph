# 调测HTTPS业务地址识别

- [操作场景](#ZH-CN_OPI_0124082602__1.3.1)
- [必备事项](#ZH-CN_OPI_0124082602__1.3.2)
- [操作步骤](#ZH-CN_OPI_0124082602__1.3.3)

## [操作场景](#ZH-CN_OPI_0124082602)

> **说明**
> 适用于PGW-U、UPF。

当用户访问HTTPS协议承载的业务时，需要调测该功能，确保该功能可正常使用。

## [必备事项](#ZH-CN_OPI_0124082602)

前提条件

- 请仔细阅读[GWFD-110203 HTTPS业务地址识别](../GWFD-110203 HTTPS业务地址识别_71023371.md)。
- 完成[配置HTTPS业务地址识别](配置HTTPS业务地址识别_24082599.md)。
- 接入侧/PDN侧接口的镜像接口已经安装了第三方抓包工具。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | apn | apn-test | 已配置数据中获取 | 已通过<br>[**ADD APN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/添加APN配置（ADD APN）_82837014.md)<br>命令配置，可通过<br>[**LST APN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/查询APN配置（LST APN）_82837017.md)<br>命令查询。 |

工具

- 测试终端
- 第三方抓包工具

## [操作步骤](#ZH-CN_OPI_0124082602)

1. 打开镜像接口侧的抓包工具，准备抓取测试终端的出入报文。
2. 测试终端使用APN “apn-test” 接入网络。
    - 如果测试终端成功接入网络，请执行[步骤 3](#ZH-CN_OPI_0124082602__step4)。
    - 如果测试终端无法接入网络，请调测UDG的接入功能。
3. 在测试终端上访问https://www.example.com业务。检查用户是否可以正常访问HTTPS业务。
    - 如果测试终端可以正常访问网页，接入侧和PDN侧接口抓包显示测试终端的HTTPS报文可以通过UDG正常转发，请执行[步骤 4](#ZH-CN_OPI_0124082602__step5)。
    - 如果测试终端无法正常访问网页，请调测UDG的Web浏览功能。
4. 登录CG客户端，根据IMSI查看测试终端本次操作产生的话单，检查用户访问HTTPS业务的流量是否记入HTTPS RG 200中。
    - 如果是，说明HTTPS地址识别功能正常，本调测任务结束。
    - 如果不是，请执行[步骤 5](#ZH-CN_OPI_0124082602__step1560201915713)排查。
5. 执行 [**DSP SESSIONINFO**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，通过测试终端的IMSI查看测试终端激活时使用的APN。
  ```
  DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
  ```
  ```
  ......
  -------------------------
                                ...
                           IMSI  =  460000123456789
                                ...
                       APN Name  =  apn-test
                                ...
  ---    END
  ```
    - 如果测试终端使用的“APN Name”与规划值一致，请执行[步骤 6](#ZH-CN_OPI_0124082602__step7)。
    - 如果测试终端使用的“APN Name”与规划值不一致，请返回[步骤 2](#ZH-CN_OPI_0124082602__step3)使用规划的APN重新接入网络。
6. 检查HTTPS内容计费和DNS报文解析相关配置是否正确。
    a. 执行 [**LST RULEBINDING**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则绑定/查询用户模板和规则的绑定关系（LST RULEBINDING）_86526204.md) 命令，查询User Profile绑定的Rule是否正确。
      ```
      LST RULEBINDING:USERPROFILENAME="up_test";
      ```
      ```
      ......
      -------------------------
      用户模板名称           规则名称       优先级       策略类型 
      up_test                rule_dns         15          PCC         
      up_test                rule_https       35          PCC         
      (结果个数 = 2)
      ---    END
      ```
          - 如果User Profile绑定的“规则名称”和“优先级”与规划值一致，请执行[步骤 6.b](#ZH-CN_OPI_0124082602__substepc)。
          - 如果User Profile绑定的“规则名称”和“优先级”与规划值不一致，请重新配置。
    b. 执行 [**LST RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/查询规则（LST RULE）_82837270.md) 命令，查询Rule的配置信息是否正确。
      ```
      LST RULE:RULENAME="rule_https";
      ```
      ```
      ......
      -------------------------
                                     规则名称  =  rule_https
                                     策略类型  =  PCC
                                              ...
                                 流过滤器名称  =  ff_https
                                              ...
                                   全局优先级  =  120
                                PCC策略组名称  =  pcc_pg_example
                                              ...
      ---    END
      ```
      ```
      LST RULE:RULENAME="rule_dns";
      ```
      ```
      ......
      -------------------------
                                   规则名称  =  rule_dns
                                   策略类型  =  PCC
                                            ...
                               流过滤器名称  =  ff_dns
                                            ...
                                 全局优先级  =  100
                              PCC策略组名称  =  pcc_pg_example
                                            ...
      (结果个数 = 1)
      ---    END
      ```
          - 如果rule绑定的“流过滤器名称”、“PCC策略组名称”和“全局优先级”与规划值一致，请执行[步骤 6.c](#ZH-CN_OPI_0124082602__substepe)。
          - 如果rule绑定的“流过滤器名称”、“PCC策略组名称”和“全局优先级”与规划值不一致，请重新配置。
    c. 执行 [**LST FLOWFILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/查询流过滤器（LST FLOWFILTER）_82837364.md) 、 [**LST FLTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的过滤器绑定/查询流过滤器的过滤器绑定关系（LST FLTBINDFLOWF）_82837368.md) 、 [**LST PROTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的协议绑定/查询流过滤器协议绑定关系（LST PROTBINDFLOWF）_82837373.md) 、 [**LST FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/查询过滤器（LST FILTER）_82837353.md) 和 [**LST L7FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/七层规则管理/七层过滤器/查询七层过滤器（LST L7FILTER）_86526660.md) 命令，查询Rule绑定的三四层和七层过滤信息是否正确。
      ```
      LST FLOWFILTER:FLOWFILTERNAME="ff_https";
      ```
      ```
      ......
      -------------------------
              流过滤器名称  =  ff_https
                            ...
      ---    END
      ```
      ```
      LST FLOWFILTER:FLOWFILTERNAME="ff_dns";
      ```
      ```
      ......
      -------------------------
              流过滤器名称  =  ff_dns
                            ...
      ---    END
      ```
      ```

      ```
      ```

      ```

      ```

      七层过滤器名字   =  l7f_dns

      ```
      ```

      ```
      ```
      ......
      -------------------------
                  过滤器名字  =  filter_dns
      三四层IPv4协议输入类型  =  String Type
                             ...
          三四层协议数字类型  =  UDP
                             ...
            服务器起始端口号  =  53
            服务器结束端口号  =  53

      ```
      ```

      ```
      ```
      ......
      -------------------------
               七层过滤器名称  =  l7f_dns
             子七层过滤器名称  =  sl7_test

                              ...
      ---    END
      ```
          - 如果Rule绑定的三四层和七层过滤信息与规划值一致，请执行[步骤 6.d](#ZH-CN_OPI_0124082602__substepf)。
          - 如果Rule绑定的三四层和七层过滤信息与规划值不一致，请重新配置。
    d. 执行 [**LST PCCPOLICYGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/查询PCC策略组（LST PCCPOLICYGRP）_82837609.md) 、 [**LST URRGROUP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则组/查询URR组（LST URRGROUP）_82837637.md) 和 [**LST URR**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则/查询URR（LST URR）_82837632.md) 命令，查询HTTPS报文使用的分类组信息是否正确。
      ```
      LST PCCPOLICYGRP:PCCPOLICYGRPNM="pcc_pg_example";
      ```
      ```
      ......
      -------------------------
            PCC策略组名称  =  pcc_pg_example
          PCC动作属性名称  =  cp_https
                          ...
      ---    END
      ```
      ```
      LST URRGROUP: URRGROUPNAME="urrgp_https";
      ```
      ```
      ......
      -------------------------
      使用量上报规则组名称  =  urrgp_https
          上行发起URR名称1  =  urr_https   
                            ...
          下行发起URR名称1  =  urr_https
                            ...
      ---    END
      ```
      ```
      LST URR: URRNAME="urr_https";
      ```
      ```
      ......
      -------------------------

                                    URR标识  =  300 
                            使用量上报模式   =  离线计费
                                            ...
      ---    END
      ```
          - 如果PCC策略组、使用量上报规则组、使用量上报规则与规划值一致，请执行[步骤 6.e](#ZH-CN_OPI_0124082602__substeph)。
          - 如果PCC策略组、使用量上报规则组、使用量上报规则与规划值不一致，请重新配置。
    e. 执行 [**LST HOST**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/三四层规则管理/主机名/查询Host（LST HOST）_82837325.md) 命令，查询filter_https绑定的Host信息是否正确。
      ```
      LST HOST:HOSTNAME="ht_https";
      ```
      ```
      ......
      -------------------------
       Host配置名称  =  ht_https
               域名  =  www.example.com
             优先级  =  65535
      (结果个数 = 1)
      ---    END
      ```
          - 如果filter_https绑定的“Host配置名称”、“域名”与规划值一致，说明Host信息配置正确，请执行[步骤 7](#ZH-CN_OPI_0124082602__step8)。
          - 如果filter_https绑定的“Host配置名称”、“域名”与规划值不一致，请重新配置。
7. 检查 UDG 上的协议识别库和协议解析库是否正确。
    a. 执行 [**DSP SIGNATUREDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/识别特征库/查询协议特征库文件加载状态（DSP SIGNATUREDB）_82837717.md) 命令，查看协议识别特征库在 UDG 上是否已经成功加载。
          - 如果加载的版本正确，请执行[步骤 7.b](#ZH-CN_OPI_0124082602__step8-b)。
          - 如果尚未加载特征库版本或加载的版本不正确，请执行[**LOD SIGNATUREDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/识别特征库/加载协议特征库文件（LOD SIGNATUREDB）_82837716.md)加载或升级特征库版本。
    b. 执行 [**DSP PARSERDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/解析特征库/查询协议解析数据库加载状态（DSP PARSERDB）_82837720.md) 命令，查询 UDG 上使用的协议解析数据库的版本信息是否正确。
          - 如果协议解析库的版本与规划值一致，请执行[步骤 8](#ZH-CN_OPI_0124082602__step9)。
          - 如果协议解析库的版本与规划值不一致，请执行[**LOD PARSERDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/解析特征库/加载协议解析数据库（LOD PARSERDB）_82837719.md)加载或升级解析库版本。
    c. 执行 [**LST USRRELATEIDEN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/三四层规则管理/用户关联识别/查询用户关联识别（LST USRRELATEIDEN）_86526961.md) 命令查看指定协议的用户关联识别功能是否开启。
      ```
      LST USRRELATEIDEN:PROTOCOLNAME="facebook";
      ```
      ```
      ......
      -------------------------
                        协议名称  =  facebook
                用户关联识别开关  =  使能
      (结果个数 = 1)
      ---    END
      ```
8. 收集信息并寻求技术支持。
    a. 在镜像接口或服务器上开启抓包工具，执行 [步骤 3](#ZH-CN_OPI_0124082602__step4) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
