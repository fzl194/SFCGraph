## [命令功能](#ZH-CN_MMLREF_0000001497964757)

该命令用于查询CSLB负载不均衡检测功能的参数。

> **说明**
> 此功能仅在UPF中使用。

#### [操作用户权限](#ZH-CN_MMLREF_0000001497964757)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001497964757)

无

## [使用实例](#ZH-CN_MMLREF_0000001497964757)

查询当前CSLB负载不均衡检测功能的参数。

```
%%LST LOADCHECKINFO:;%%
RETCODE = 0  操作成功

结果如下
--------
CSLB负载不均衡检测开关  =  开启
       CPU基础阈值 (%)  =  50
       CPU阈值上限 (%)  =  50
       CPU阈值下限 (%)  =  30
    收包数阈值上限 (%)  =  80
    收包数阈值下限 (%)  =  40
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001497964757)

| 输出项名称 | 输出项解释 |
| --- | --- |
| CSLB负载不均衡检测开关 | 该参数用于控制CSLB负载不均衡检测功能开关。 |
| CPU基础阈值 (%) | 该参数用于指定CPU使用率的基础阈值。只有当某个RU的CPU使用率超过了此阈值，才会触发负载均衡检测。 |
| CPU阈值上限 (%) | 该参数用于指定CPU百分比阈值上限。若单个RU的CPU使用率比其他所有RU的CPU使用率的平均值高，且两者差大于（其他所有RU的CPU使用率的平均值*CPU阈值上限百分比），则进行下一步收包数检查。 |
| CPU阈值下限 (%) | 该参数用于指定CPU百分比阈值下限。若单个RU的CPU使用率比其他所有RU的CPU使用率的平均值低，且两者差大于（其他所有RU的CPU使用率的平均值*CPU阈值下限百分比），则进行下一步收包数检查。 |
| 收包数阈值上限 (%) | 该参数用于指定收包数百分比阈值上限。若单个RU的收包数比其他所有RU的收包数的平均值高，且两者差大于（其他所有RU的收包数的平均值*收包数阈值上限百分比），则上报告警。 |
| 收包数阈值下限 (%) | 该参数用于指定收包数百分比阈值下限。若单个RU的收包数比其他所有RU的收包数的平均值低，且两者差大于（其他所有RU的收包数的平均值*收包数阈值下限百分比），则上报告警。 |
