# 设置基于APN的下行数据DDN寻呼流控方式（SET APNDDNFLOWCTRL）

#### [命令功能](#ZH-CN_CONCEPT_0000203116858968)

**适用NF：UPF**

设置基于APN的下行数据DDN寻呼流控方式。

#### [注意事项](#ZH-CN_CONCEPT_0000203116858968)

- 该命令执行后立即生效。
- 系统最多支持配置10000条。
- 该命令需要与byte686和bit1230配合使用。
- 配置该命令时，对于语音类APN，建议配置时长模式，且时长检查时间不大于3s。
- 该命令存在系统初始记录，参数的初始设置值如下表：

| 参数标识 | FLOWCTRLMODE |
| --- | --- |
| 初始值 | DEFAULT |

#### [操作用户权限](#ZH-CN_CONCEPT_0000203116858968)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000203116858968)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| APN | APN | 可选必选说明：必选参数<br>参数含义：该参数用于指定APN实例名。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。只能由“-”、数字、大小写字母和“.”组成，不能以“.”开头且不能出现连续两个“.”。不支持空格及“_”、“#”、“$”、“&”、“%”、“^”、“（”、“）”、“，”、“/”、“;”、“:”、“””、“`”等特殊字符，不区分大小写。<br>默认值：无<br>配置原则：该参数必须已经通过ADD APN命令配置。 |
| FLOWCTRLMODE | 流控模式 | 可选必选说明：必选参数<br>参数含义：该参数用于控制下行数据报文触发DDN的流控模式。报文模式指按报文个数进行流控；时间模式指按时间进行流控。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DEFAULT：缺省流控模式。<br>- PKTNUM：报文流控模式，按报文个数进行流控。<br>- TIME：时间流控模式，按时间进行流控。<br>默认值：无<br>配置原则：无 |
| PACKETNUM | 报文个数 | 可选必选说明：条件必选参数<br>前提条件：该参数在“FLOWCTRLMODE”配置为“PKTNUM”时为必选参数。<br>参数含义：该参数表示收到多少下行报文后重复触发一次DDN寻呼。<br>数据来源：本端规划<br>取值范围：1~1000。<br>默认值：无<br>配置原则：无 |
| TIMEINTERVAL | 时间间隔 | 可选必选说明：条件必选参数<br>前提条件：该参数在“FLOWCTRLMODE”配置为“TIME”时为必选参数。<br>参数含义：该参数表示重复触发一次DDN寻呼间隔的时间，单位为秒。<br>数据来源：本端规划<br>取值范围：1~10。<br>默认值：无<br>配置原则：无 |

#### [使用实例](#ZH-CN_CONCEPT_0000203116858968)

设置基于APN的下行数据DDN寻呼流控方式：

```
SET APNDDNFLOWCTRL: APN="aaa", FLOWCTRLMODE=DEFAULT;
```
