## [命令功能](#ZH-CN_MMLREF_0000001228971839)

该命令用于查询HTTP协议层的属性配置信息。

> **说明**
> 无

#### [操作用户权限](#ZH-CN_MMLREF_0000001228971839)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001228971839)

无

## [输出结果说明](#ZH-CN_MMLREF_0000001228971839)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 信令路由开关 | 该参数用于配置信令路由开关。如果信令路由开关打开，HTTP链路管理按信令路由方式处理，如果信令路由开关关闭，HTTP链路管理按正常方式处理。 |
| http-server日志级别 | 该参数用于指定http-server日志级别。 |
| tcp-process日志级别 | 该参数用于指定tcp-process日志级别。 |
| 故障链路探测最长时间间隔(s) | 该参数用于设置故障链路探测的最长时间间隔。当链路状态变化为故障以后，该链路状态是否变化到正常需要通过重建探测，重建探测按时间递延方式触发，该参数即设置递延的最大时间。当递延到最大时间以后，在该链路对应的链路集老化前会一直以该间隔触发重建来探测该链路是否正常。时间间隔递延规则如下：<br>初始时间为1s，每次失败递延时间间隔增加1s，达到最大设置时间间隔之后，其值不再发生变化。参考递延时间间隔序列为（以最大值720s为例）：1s，2s，3s，... ，720s，720s，720s。 |
| 链路负载上报时间间隔(s) | 该参数用于配置链路的负载上报时间间隔。系统支持定时上报链路负载情况，该参数用于配置上报的时间间隔。 |
| 链路老化时间(min) | 该参数用于配置链路老化时间。当系统中的链路处于空闲状态下，为确保链路资源及时释放，该链路需要老化释放，通过本参数配置可修改链路空闲以后的老化时间。客户端老化时间与配置时间最多有5分钟误差。 |
| 客户端接收HTTP响应消息超时时间(s) | 该参数用于配置当系统作为客户端时，发送HTTP请求消息后，等待HTTP响应消息的超时时间。即超过该设置时间后，如果系统未收到对端设备的响应消息，则认为接收HTTP响应超时。 |
| 服务端接收HTTP请求消息体超时时间(s) | 该参数用于配置当系统作为服务端时，接收HTTP请求消息体的超时时间。系统作为服务端时，可能需要分多次接收对端同一个HTTP请求消息的消息体，如果在该设定超时时间内没有收齐，则认为接收该请求消息失败。 |
| 服务端回复HTTP响应消息超时时间(s) | 该参数用于配置当系统作为服务端时，接收到HTTP请求消息后，回复HTTP响应消息的超时时间。如果在该设置时间内未回复对应的响应消息，则生成状态码为504的HTTP响应消息，发送给对端设备，告知对端设备本端业务处理异常。 |
| 客户端HTTP PING探测开关 | 该参数用于配置本端为客户端时，HTTP PING探测功能是否打开。如果该功能打开，则HTTP会定时发送HTTP PING报文用于探测当前链路是否正常。如果该功能关闭，则HTTP不会定时发送HTTP PING探测报文，HTTP链路故障无法主动感知。 |
| 客户端HTTP PING探测时间间隔(s) | 该参数用于配置HTTP PING探测报文发送的时间间隔。HTTP PING报文按该设置值定时发送。 |
| 客户端HTTP PING探测次数 | 该参数用于设置HTTP PING报文的探测次数。当客户端按HTTP PING探测间隔持续发送HTTP PING报文，但是连续未收到本配置中设置次数的HTTP PING响应报文，则认为该链路故障。以探测时间间隔60秒、探测次数3次为例。故障发生后探测到故障的时间，最短为60×3=180秒，最长为60×(3+1)=240秒。 |
| 描述 | 该参数用于指定描述信息。 |
| Keepalive功能开关 | 该参数用于配置TCP的Keepalive机制开关。TCP协议中支持Keepalive功能，但该功能默认关闭，需要上层业务打开。 |
| Keepalive空闲时间(s) | 该参数用于配置TCP Keepalive的空闲时长，即在监控到TCP会话空闲以后，根据该配置值触发周期性的保活探测报文。TCP协议中默认该参数为7200秒，产品的默认值为1800秒。该参数配置过长，可能导致对端已经故障的链路长时间无法释放，该参数配置过短，可能会触发频繁保活探测。建议使用默认值配置。 |
| Keepalive探测间隔(s) | 该参数用于配置TCP的keepalive空闲时间以后，保活探测无响应，继续发送下一个保活探测报文的间隔时间。TCP协议中默认该参数为75秒，产品默认值为5秒。该参数配置过长，保活报文发送间隔变长，探测时间变长，该参数配置过短，保活报文发送间隔短，发送过于频繁。建议使用默认值。 |
| Keepalive探测次数 | 该参数用于配置TCP的Keepalive保活探测的次数。即保活报文无响应可继续发送保活报文的次数。TCP协议默认该探测次数为9次，产品默认设置为5次。该参数设置过大，探测时间变长，该参数设置过小，比如仅探测1次，可能出现误判断。建议使用默认值。 |
| 最大消息体大小(MBytes) | 该参数用于配置HTTP请求/响应中消息体的最大大小，单位是MBytes，超过该大小的消息将会被丢掉。 |
| JSON数据最多叶子节点个数 | 该参数用于配置HTTP请求消息中JSON数据的最大叶子节点个数，超过该大小的请求消息将会被丢掉。 |
| TCP会话删除延迟时间(s) | 该参数用于设置需要删除的TCP会话的延迟时间。 |
| 客户端最大并发流 | 该参数用于指定客户端最大并发流上限。 |
| 初始重建链间隔(ms) | 该参数用于限制初始建链时，HTTP重建的等待时长。 |
| IP可达自动探测的功能开关 | 该参数用于控制当检测到HTTP链路故障后，是否自动向对端IP发起Ping及Tracert，探测对端IP是否可达。 |
| 性能统计功能开关 | 该参数用于配置HTTP消息性能统计功能开关。 |
| 发送恢复链路集信息请求的时间间隔 | 该参数用于控制对于同一个链路集，SBILINK向TOPO发送链路恢复请求的时间间隔。 |
| 配置核查开关 | 该参数用于控制HTTP配置数据的核查功能开关。 |
| 链路核查开关 | 该参数用于控制HTTP链路的核查功能开关。 |
| 压缩报文阈值 (KBytes) | 该参数用于设置HTTP转发报文压缩阈值。 |
| Reqcb流控开关 | 该参数用于控制服务端和客户端的REQCB流控开关。 |
| TCP SYN包重传时间间隔(ms) | 该参数用于设置TCP SYN包重传间隔。 |
| TCP SYN包重传最大次数 | 该参数用于设置TCP SYN包重传最大次数。 |
| TCP建链成功后包重传最大次数 | 该参数用于配置TCP建链成功后，包重传最大次数。 |
| HTTP建链自动退避时间间隔(ms) | 该参数用于设置HTTP更换端口进行重试建链的退避时间间隔，0表示不退避。 |
| 使能HTTP1.1开关 | 该参数用于控制服务端是否支持HTTP1.1版本，服务端默认支持HTTP2.0版本，本参数开启后，服务端将同时支持HTTP2.0版本及HTTP1.1版本。 |
| HTTP Goaway处理开关 | 该参数用于控制客户端是否处理对端发送的Goaway帧，Goaway帧用于HTTP链路拆链。开关关闭时，收到对端Goaway帧后不立即断链，正常处理已发送的请求；开关打开后，收到对端Goaway帧后立即断链。 |
