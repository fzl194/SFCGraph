#### [命令功能](#ZH-CN_CONCEPT_0259104241)

该命令用于显示进程内的消息分区使用信息。

#### [注意事项](#ZH-CN_CONCEPT_0259104241)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0259104241)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0259104241)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| RUNAME | RU名称 | 可选必选说明：可选参数<br>参数含义：该参数用于指定RU名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。不支持空格，区分大小写。<br>默认值：无 |
| PROCID | 进程ID | 可选必选说明：可选参数<br>参数含义：网元内一个进程的进程ID。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～4294967295。<br>默认值：无 |
| SERVICEINSTANCE | 服务实例 | 可选必选说明：必选参数<br>参数含义：该参数表示大颗粒服务实例名称。<br>数据来源：本端规划<br>取值范围：字符串类型，通过LST VNFC命令获取。<br>默认值：无<br>配置原则：只能填写通过LST VNFC命令查询到的管理代理标识，但不能填写0，0表示VNFP。 |

#### [使用实例](#ZH-CN_CONCEPT_0259104241)

查询进程内消息分区的使用情况：

```
DSP PROCMSGAREA:RUNAME="VNODE_CSLB_VNFC_OMU_0001",PROCID=1001
,SERVICEINSTANCE="vnfc"
;
```

#### [输出结果说明](#ZH-CN_CONCEPT_0259104241)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 进程ID | 用于说明进程的编号。 |
| RU名称 | 表示资源单元的名称。 |
| 消息分区名 | 用于说明进程内消息分区的名字。 |
| 最大消息单元数目 | 用于说明进程内消息的最大消息单元数目。 |
| 已分配的消息单元数目 | 用于说明此进程内已分配的消息单元数目。 |
| 消息分配百分比（%） | 用于说明进程内消息分配的百分比（单位：%）。 |
| 已使用的消息分区数目 | 用于说明进程内的已经使用的消息分区数。 |
| 消息分区总数 | 用于说明进程内的消息分区的总数目。 |
