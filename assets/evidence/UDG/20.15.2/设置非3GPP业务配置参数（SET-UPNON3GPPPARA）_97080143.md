#### [命令功能](#ZH-CN_CONCEPT_0000206297080143)

**适用NF：UPF**

设置非3GPP业务配置参数。

#### [注意事项](#ZH-CN_CONCEPT_0000206297080143)

- 该命令执行后立即生效。
- 该命令最大记录数为1。
- 对于DETECTPERIOD参数： 1. 若用户数较多，周期配置较小，则可能存在一个周期内无法探测所有用户的情况。此时新一轮探测不会开始而是跳过，直到上一轮探测结束，下个周期才会再次启动探测。 2. 若周期先设为T1，经过t分钟后改为T2，则下轮探测时间为(T2 - (t % T2))分钟后，后续探测间隔才为T2。
- 该命令存在系统初始记录，参数的初始设置值如下表：

| 参数标识 | SWUWAL | AGINGTIME | DETECTSW | DETECTPERIOD | DATADETECTINTERVAL | DATAFAILTHRESH |
| --- | --- | --- | --- | --- | --- | --- |
| 初始值 | 1000 | 0 | DISABLE | 10 | 0 | 0 |

#### [操作用户权限](#ZH-CN_CONCEPT_0000206297080143)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000206297080143)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| SWUWAL | SWU接口UE接入速率 | 可选必选说明：可选参数<br>参数含义：该参数用于指定SWU接口UE接入速率。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～1200，单位是个/s。<br>默认值：无<br>配置原则：配置的接入速率值是整设备每秒支持的终端接入速率。 |
| AGINGTIME | ToH SESSION的老化时长 | 可选必选说明：可选参数<br>参数含义：该参数用于指定ToH SESSION的老化时长。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0，1440～10080，单位是分钟。取值为0表示ToH SESSION老化不开启。<br>默认值：无<br>配置原则：无 |
| DETECTSW | 会话内部探测开关 | 可选必选说明：可选参数<br>参数含义：设置非3GPP内部会话发送探测消息的功能开关。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：建议开启非3GPP内部会话发送探测消息的开关。 |
| DETECTPERIOD | 探测周期 | 可选必选说明：可选参数<br>参数含义：该参数用于指定非3GPP内部会话发送探测消息的周期。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为5～30，单位是分钟。<br>默认值：无<br>配置原则：无 |
| DATADETECTINTERVAL | 数据面报文检测时长 | 可选必选说明：可选参数<br>参数含义：该参数用于指定非3GPP会话检测IPsec数据面报文转发失败的时长。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0，5～30，单位是秒。<br>默认值：无<br>配置原则：此参数和数据报文转发失败阈值配合使用，当该参数与数据报文转发失败阈值都不为0时，配置生效。 |
| DATAFAILTHRESH | 数据报文转发失败阈值 | 可选必选说明：可选参数<br>参数含义：该参数用于指定非3GPP会话IPsec数据面转发失败的报文个数，此阈值用来判断是否发起会话去活。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0，1～10，单位是个。<br>默认值：无<br>配置原则：此参数和数据面报文检测时长配合，当会话在此时长内出现阈值个数的报文转发失败，则上报会话去活，防止会话静默故障。 |

#### [使用实例](#ZH-CN_CONCEPT_0000206297080143)

设置SWU接口UE接入速率为1000个/s，ToH SESSION的老化时长为1440分钟：

```
SET UPNON3GPPPARA: SWUWAL=1000, AGINGTIME=1440;
```
