## [命令功能](#ZH-CN_MMLREF_0000001983186620)

该命令用于查询消息跟踪模型。

> **说明**
> 无。

## [参数说明](#ZH-CN_MMLREF_0000001983186620)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| MEID | 网元ID | 可选必选说明：必选参数。<br>参数含义：标识网元ID，可以通过<br>[**LST ME**](../../系统管理/版本信息/查询网元配置信息（LST ME）_47084797.md)<br>命令查询获取。<br>取值范围：0~65535<br>默认值：无。<br>配置原则：无。 |
| TRACETYPE | 跟踪类型 | 可选必选说明：可选参数。<br>参数含义：跟踪类型，由网元在适配包中定义。<br>取值范围：0~2147483647<br>默认值：无。<br>配置原则：无。 |

## [使用实例](#ZH-CN_MMLREF_0000001983186620)

1. 查询“网元ID”为“12”，“跟踪类型”为“17”的消息跟踪模型：
  ```
  %%DSP MTSMODEL: MEID=12, TRACETYPE=17;%%
  RETCODE = 0  操作成功

  操作结果如下
  ------------
        网元ID  =  12
      跟踪类型  =  17
  拓展跟踪类型  =  17
  跟踪类型名称  =  E2E跟踪
  订阅服务名称  =  TestforCSPOMC|HttpDisCtrlService|UdmHttpService|UdmCcu
  (结果个数 = 1)

  ---    END
  ```
2. 查询“网元ID”为“12”，“跟踪类型”为“60102”的消息跟踪模型，跟踪类型“60102”配置了拓展跟踪类型：
  ```
  %%DSP MTSMODEL: MEID=12, TRACETYPE=60102;%%
  RETCODE = 0  操作成功

  操作结果如下
  ------------
  网元ID  跟踪类型  拓展跟踪类型  跟踪类型名称   订阅服务名称                                                   

  12      60102     10102         CSCF GTPC跟踪  cspjavacommonservicedemo                                       
  12      60102     60102         CSCF GTPC跟踪  TestforCSPOMC|CSPGoCommonServiceDemo
  (结果个数 = 2)

  ---    END
  ```

## [输出结果说明](#ZH-CN_MMLREF_0000001983186620)

如果同一个跟踪类型查出来多条记录，则代表此跟踪类型配置了拓展跟踪类型，此时跟踪类型以拓展跟踪类型为准。

命令执行正常，会返回命令执行成功的提示信息，输出结果说明 [表1](#ZH-CN_MMLREF_0000001983186620__table65351055114810) 如所示。

命令执行异常，会返回提示对应的失败信息。错误码列表如 [表2](#ZH-CN_MMLREF_0000001983186620__table202411337174912) 所示。

*表1 输出项解释*

| 输出项名称 | 输出项解释 |
| --- | --- |
| 网元ID | 标识网元ID。 |
| 跟踪类型 | 标识跟踪类型。 |
| 拓展跟踪类型 | 标识拓展跟踪类型，用于多网元融合时，区分同一类消息跟踪类型。 |
| 跟踪类型名称 | 标识跟踪类型名称。 |
| 订阅服务名称 | 标识订阅该跟踪类型的服务名称，多个服务使用“\|”分割。 |

*表2 错误码说明*

| 错误码 | 消息解释 | 原因分析 | 处理建议 |
| --- | --- | --- | --- |
| 101601 | 系统异常 | 服务离线、数据库异常等。 | 请等待几分钟后重试。 |
| 101602 | 网元不存在 | “网元ID”<br>输入非法，与实际环境不一致。 | 修改<br>“网元ID”<br>后重试。 |
| 101603 | 请求消息错误 | 请求消息错误。 | 请联系华为技术支持处理。 |
