# 调测基于位置的地址分配

- [操作场景](#ZH-CN_OPI_0202846777__1.3.1)
- [必备事项](#ZH-CN_OPI_0202846777__1.3.2)
- [操作步骤](#ZH-CN_OPI_0202846777__1.3.3)

## [操作场景](#ZH-CN_OPI_0202846777)

用户激活采用基于位置区域分配UE IP地址时，需要调测该功能，确保用户可以正常接入。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0202846777)

前提条件

- 请仔细阅读[GWFD-020421 基于位置的地址分配](../GWFD-020421 基于位置的地址分配_02846765.md)。
- 完成[激活基于位置的地址分配](激活基于位置的地址分配_02846771.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | 用户IMSI（IMSI） | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN/DNN实例名（APN name） | apn-test1 | 已配置数据中获取 | - |

工具

测试终端

## [操作步骤](#ZH-CN_OPI_0202846777)

1. 使用测试终端在规划的位置范围内通过“apn-test1”APN发起接入网络请求。
2. 执行 [**DSP SESSIONINFO**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，查看 测试终端的IP地址是否在 UDG 本地预先按位置规划的地址池内 。
  ```
  DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
  ```
  ```
  ......

  -------------------------------
                                   IMSI  =  460000123456789
                                       ......              
                               PDP Type  =  IPv4 
                                       ......    
                       IPv6 PDP address  =  10.10.1.15
                                       ......    
                               APN name  =  apn-test1
                                       ......       
  (Number of results = 1)
  ---    END
  ```
    - 如果测试终端使用的“IPv4 PDP address”在预先按位置规划的地址池内，则测试终端接入成功，调测结束。
    - 如果测试终端使用的“IPv4 PDP address”不在预先按位置规划的地址池内，执行[步骤 3](#ZH-CN_OPI_0202846777__step4)。
3. 继续查看 [**DSP SESSIONINFO**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话信息管理/会话信息查询/显示用户上下文（DSP SESSIONINFO）_86526407.md) 命令，确认测试终端IP地址类型。
  ```
  DSP SESSIONINFO:QUERYTYPE=IMSI,IMSI="460000123456789";
  ```
  ```
  ......

  -------------------------
           APN name  =  apn-test1
                   ......        
  IPv4 Address type  =  UPF ALLOC IP ADDRESS 
                   ......      
  (Number of results = 1)
  ---    END
  ```
    - 如果是本地地址池分配（UPF ALLOC IP ADDRESS），请执行[步骤 4](#ZH-CN_OPI_0202846777__step393310289141)。
    - 如果是外部地址池分配（EXTERNAL ALLOC IP ADDRESS），请返回UDG配置页面检查本地配置以及检查SMF上配置的地址分配能力，是否符合预先规划。如果不符合，请调整配置。如果符合，则执行[步骤 5](#ZH-CN_OPI_0202846777__step7)。
4. 查看当前使用的IP地址所归属的 UDG 上的地址池组，执行 [**LST POOLGRPMAP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/地址池组映射关系/显示地址池组映射关系（LST POOLGRPMAP）_82837150.md) 检查该地址池组绑定的位置信息是否与预期一致。
  ```
  LST POOLGRPMAP: MAPPINGNAME="mapping2",POOLGROUPNAME="poolgroup2";
  ```
  ```
  RETCODE = 0  Operation Success.  
  Mapping of Pool Group 
  --------------------- 
  Name of Pool Group Mapping    Type of Location Group    Name of Location Group    APN         SMF     Name of Pool Group  
  mapping1                      LAC                       lac1                      apn-test1   NULL    poolgroup1         (Number of results = 1) 
  ---    END
  ```
    - 如果一致，请执行[步骤 5](#ZH-CN_OPI_0202846777__step7)。
    - 如果不一致，请返回UDG配置页面重新配置。
5. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务 并保存报文。
    b. 执行 **[EXP MML](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md)** 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
