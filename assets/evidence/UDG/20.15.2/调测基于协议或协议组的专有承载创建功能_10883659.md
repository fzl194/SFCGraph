# 调测基于协议或协议组的专有承载创建功能

- [操作场景](#ZH-CN_OPI_0210883659__1.3.1)
- [必备事项](#ZH-CN_OPI_0210883659__1.3.2)
- [操作步骤](#ZH-CN_OPI_0210883659__1.3.3)

## [操作场景](#ZH-CN_OPI_0210883659)

当运营商开启了协议或协议组支持基于业务感知能力触发专有承载创建，并配置了触发专有承载创建的模式时，需对该功能进行调测，确保本功能可以正常使用。

> **说明**
> 适用于SGW-U、PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0210883659)

前提条件

- 请仔细阅读[GWFD-020358 业务触发的QoS保证](../../GWFD-020358 业务触发的QoS保证_67655191.md)。
- 完成[配置七层业务触发的QOS保证（PGW-U、UPF）](../激活业务触发的QoS保证/配置七层业务触发的QOS保证（PGW-U、UPF）_80361858.md)。

## [操作步骤](#ZH-CN_OPI_0210883659)

1. 执行 [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令查询 UDG 上本功能对应的License配置开关是否打开。
  ```
  LST LICENSESWITCH:LICITEM="
  LKV3G5STQE01
  ";
  ```
    - 如果“SWITCH”为“ENABLE”，请执行[步骤 2](#ZH-CN_OPI_0210883659__step19171111585013)。
    - 如果“SWITCH”为“DISABLE”，则执行[**SET LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)命令打开本特性对应的License配置开关。
2. 在OM Portal上创建 [UDG N4接口跟踪](../../../../../../网络运维/日常维护/UDG基础运维操作/创建消息跟踪/创建接口跟踪/UDG N4接口跟踪_88792205.md) 任务。
3. 测试终端使用 “apn1” APN接入网络，并进行HTTP业务。
4. 查看OM Portal跟踪结果，检查用户进行HTTP业务时，PGW-U/UPF是否立即向PGW-C/SMF发送PFCP Session Report Request消息，携带的Usage Report信元组中包含QoS类型的URR ID、Usage Report Trigger为start。
  > **说明**
  > 如果基于协议或协议组的专有承载创建功能不使能，用户在进行HTTP业务时，是不可能触发创建专有承载流程。
    - 如果PGW-U/UPF上报信元正确，基于协议或协议组的专有承载创建功能正常，继续执行[步骤 5](#ZH-CN_OPI_0210883659__step2054514555016)。
    - 如果PGW-U/UPF没有上报，请执行[步骤 6](#ZH-CN_OPI_0210883659__step6)。
5. 查看PGW-U/UPF向PGW-C/SMF发送的PFCP Session Report Request消息，查询rule-filter-type字段是否为l7-downlink，如 [图1](#ZH-CN_OPI_0210883659__fig486810451716) 所示。
  > **说明**
  > 执行 [**ADD SADEDICBEARER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/用户QOS控制/业务QOS控制/业务感知专有承载/增加业务感知专有承载配置（ADD SADEDICBEARER）_82837654.md) 命令将基于SA能力触发专有承载创建的模式配置为 **下行链路模式** ， UDG 将创建“rule-filter-type”字段为“l7-downlink”的模板，并关联一个特定的专有承载，基于该协议或协议组的所有用户下行报文汇聚到这个特定的专有承载，上行报文仍使用原专有承载。
  **图1** 用户跟踪任务结果（专有承载创建）

  <br>

  ![](调测基于协议或协议组的专有承载创建功能_10883659.assets/zh-cn_image_0210996586.png)
    - 如果rule-filter-type字段为 **l7-downlink** ，基于协议或协议组的专有承载创建的模式是下行链路模式，调测结束。
    - 如果rule-filter-type字段不为 **l7-downlink** ，请执行 [步骤 6](#ZH-CN_OPI_0210883659__step6) 。
6. 执行 [**LST SADEDICBEARER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/用户QOS控制/业务QOS控制/业务感知专有承载/查询业务感知专有承载配置（LST SADEDICBEARER）_86528486.md) 命令，检查“触发专有承载模式” 是否为“DOWNLINK_ONLY”。
  ```
  LST SADEDICBEARER:;
  ```
  ```
  RETCODE = 0  操作成功。  
  业务感知专有承载信息 
  -------------------- 
                   协议等级  =  Protocol
                   协议名称  =  http
                 协议组名称  =  NULL
            触发专有承载模式 =  DOWNLINK_ONLY 
  (结果个数 = 1) 
  ---    END 
  ```
    - 如果是，请执行[步骤 7](#ZH-CN_OPI_0210883659__step8)。
    - 否，请执行 [**ADD SADEDICBEARER**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/用户QOS控制/业务QOS控制/业务感知专有承载/增加业务感知专有承载配置（ADD SADEDICBEARER）_82837654.md) 命令，配置“TRIGGERMODE”为“DOWNLINK_ONLY”。重新执行 [步骤 3](#ZH-CN_OPI_0210883659__step3) 。
7. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务，执行 [步骤 3](#ZH-CN_OPI_0210883659__step3) 并保存报文。
    b. 执行 [**EXP MML**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
