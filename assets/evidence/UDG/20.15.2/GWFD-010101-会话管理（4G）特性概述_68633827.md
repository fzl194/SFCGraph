#### [适用NF](#ZH-CN_CONCEPT_0168633827)

SGW-U、PGW-U

#### [定义](#ZH-CN_CONCEPT_0168633827)

会话管理是 UDG 的基本功能， 是用户与外部PDN（ Packet Data Network）建立连接，进行数据业务的基础。华为 UDG 支持3GPP标准定义的E-UTRAN无线网络接入，实现LTE网络用户的接入。

#### [客户价值](#ZH-CN_CONCEPT_0168633827)

| 受益方 | 受益描述 |
| --- | --- |
| 运营商 | 会话管理功能使运营商得以向授权的用户提供从移动终端到数据网络间的数据传输服务，从而丰富其网络业务。 |
| 用户 | 会话管理使UE可以通过LTE网络接入外部PDN，享用Internet或企业网提供的数据业务。 |

#### [应用场景](#ZH-CN_CONCEPT_0168633827)

当用户需要通过LTE网络接入外部PDN时，需要支持本功能。

#### [原理概述](#ZH-CN_CONCEPT_0168633827)

**基本概念**

EPC会话管理的对象为EPS承载。每个PDN连接都由EPS承载（EPS Bearer）和IP连接组成。相关概念如 [表1](#ZH-CN_CONCEPT_0168633827__table117631943109) 所示，PDN连接、缺省承载、专有承载之间的关系请参照 [图1](#ZH-CN_CONCEPT_0168633827__fig1888153314814) 。

*表1 会话管理概念介绍*

| 概念 | 描述 |
| --- | --- |
| PDN连接 | EPS通过E-UTRAN网络和EPC网络，为UE提供了一条UE与PDN之间的IP连接，称为PDN连接。 |
| EPS承载 | 用于唯一标识基于GTP协议的UE和P-GW之间的共同QoS业务流。 |
| 缺省承载 | 当UE建立一个到某个PDN的PDN连接时，会建立一个EPS承载，这个承载在PDN连接的过程中不会被释放以提供到该PDN的永久性连接，这个承载被称为缺省承载。缺省承载被删除，PDN连接也将被释放。 |
| 专有承载 | 除缺省承载之外的其他连接到相同的PDN的承载称为专有承载。根据不同的QoS需求建立不同的专有承载，比如观看视频、浏览网页等。 |

- APN与PDN是一一对应。一个PDN是由一个APN进行代表的，不同的APN，可以连接到不同的PDN。一个终端可以通过多个APN接入到不同的PDN中，PDN中可以有多个服务器。
- 同一个APN下可建立多个PDN连接。同一个PDN连接必须建立一个缺省承载且只能建立一个缺省承载，但可以建立多个专有承载。由于无线侧的限制，一个终端最多只能建立11个承载。

**会话管理流程**

4G网络会话管理通过 [表2](#ZH-CN_CONCEPT_0168633827__table4913185915411) 中的流程管理用户的缺省和专有承载，以下介绍会话管理相关流程的主要应用场景，流程详情请查看5G Core解决方案 > 快速入门 > 协议与流程 > EPC信令分析。

*表2 会话管理流程介绍*

| 流程 | 描述 |
| --- | --- |
| 缺省承载激活 | 该流程由UE发起，可以是附着流程的一部分，也可以作为一个单独的流程，用于一个已附着UE向其他PDN申请建立连接。激活过程中，MME进行SGW-C/PGW-C选择，SGW-C/PGW-C完成IP地址分配、以及UE接入ISP（Internet service provider）或企业网的权限检查等。缺省承载激活成功后，用户即可通过EPC网络与PDN进行数据通信。 |
| 专有承载激活 | 该流程用于在UE和EPC之间建立一个使用特定QoS和TFT（Traffic Flow Template）的承载上下文，由网络侧发起。专有承载激活成功后，用户即可在特定的QoS保证下进行请求的数据业务。<br>说明：TFT：一组数据包过滤器的集合，UE使用TFT来决定业务数据流Service Data Flows和无线承载Radio Bearer之间的匹配关系。 |
| 承载去激活 | 该流程用于去激活一个或多个EPS承载上下文，可以由MME或SGW-C/PGW-C发起。如果UE的最后一个缺省承载被去激活，MME将发起去附着流程，向UE发送Detach Request消息。去激活过程中，UE的IP地址被释放，MME、SGW-C/PGW-C中的承载上下文被删除。承载去激活成功后，用户即不能使用该数据业务。<br>说明：UE的最后一个缺省承载只能在去附着流程中删除，该流程可以由UE、MME或HSS发起。 |
| 承载更新 | 该流程用于修改EPS承载上下文的参数，如QoS和TFT，可以由UE、HSS、PCRF或SGW-C/PGW-C发起。UE可以请求分配或修改一个特定的QoS，也可以请求新的GBR承载或修改已有的GBR承载。如果网络接受UE的请求，会发起EPS承载上下文激活、修改或释放流程。<br>说明：GBR主要体现为带宽管理，系统通过预留资源等方式为保证数据流的比特速率在不超过GBR时能够全部通过，超过GBR的流量可以按照如下方式处理：拥塞时超过GBR的流量会被丢弃，不拥塞时超过GBR但小于MBR的流量可以通过。 |
