#### [U面地址分配流程](#ZH-CN_TOPIC_0174936046)

UE IP地址是在PDU会话建立过程中被分配的，在整个PDU会话过程中固定唯一。 [图1](#ZH-CN_TOPIC_0174936046__fig1898820180423) 展示了PDU会话建立过程中用户面地址分配的过程。

**图1** PDU会话建立过程中用户面地址分配流程

![](地址分配流程_74936046.assets/zh-cn_image_0174936048.png)

> **说明**
> PDU会话建立流程详细过程可参见 [GWFD-010101 会话管理](../../GWFD-010101 会话管理_68632498.md) ，本特性中仅介绍和用户面地址分配相关的消息和信元。

1. 网元部署完成链路层互通后，在U面（UPF/PGW-U）配置本端信息（本端N4逻辑接口地址和数据面逻辑接口Sa/Sc/Pa地址），用于和C面（SMF/GW-C）建立连接。
2. U面（UPF/PGW-U）在偶联建立响应消息中上报本端地址分配能力。
    a. C面向U面主动发起偶联建立请求（PFCP Association Setup Request）。
    b. U面给C面返回偶联建立响应消息（PFCP Association Setup Response），消息中在私有扩展域携带UP Function Flag信元，其ADUP标识位置位表示U面具备地址分配能力。C面接收消息后在本地进行记录该信息。
3. 用户激活向C面发起会话建立请求，消息中携带Requested PDU Session Type信元指示UE请求的PDU会话类型（IPv4、IPv6、IPv4v6双栈）。
4. C面根据本地配置的地址分配能力决策由U面为UE分配其请求的指定PDU类型的IP地址（IPv4、IPv6或IPv4v6双栈）。
5. C面发起PFCP会话建立请求（PFCP Session Establishment Request），消息中携带Create PDR中的PDI信元下发的UE IP Address/UE IP address Pool Identity子信元或者Create Traffic Endpoint信元方式下发的UE IP Address子信元，通过其CHV4和CHV6 BIT位（如[表1](#ZH-CN_TOPIC_0174936046__table12248151812381)和[图2](#ZH-CN_TOPIC_0174936046__fig737372392913)和[图3](#ZH-CN_TOPIC_0174936046__fig0986018115414)所示），并结合V4、V6 BIT位指示UDG是否分配地址以及分配的地址类型。

7. C面获取用户IP地址后，给左侧网元返回用户激活响应消息，将分配的用户UE IP地址传递给UE。

#### [用户下行路由](#ZH-CN_TOPIC_0174936046)

UDG 从本地地址池为UE分配IP地址的同时，可自动为本地地址池生成用户下行子段路由，通过OSPF/BGP等协议发布到骨干网，用于下行报文的回程寻址。
