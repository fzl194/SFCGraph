#### [命令功能](#ZH-CN_CONCEPT_0182837117)

**适用NF：PGW-U、UPF**

![](锁定地址段（LCK SECTION）_82837117.assets/notice_3.0-zh-cn.png)

本命令属于高危命令，地址段被锁之后，被锁定地址段中的地址将不能分配给用户，地址池下的地址资源变少，可能导致用户激活失败。

该命令在系统需要回收或者重新使用地址段，可以用来锁定或解锁该地址段。

当有需要将已锁定的地址段进行解锁时使用本命令LCK SECTION，后将参数IsLocked设置为UNLOCK将地址段解锁。

#### [注意事项](#ZH-CN_CONCEPT_0182837117)

- 该命令执行后立即生效。
- 地址段锁定命令仅适用于本地地址池。
- 如果先执行LCK POOL锁定整个地址池，再执行LCK SECTION锁定某个地址段操作时，执行失败，因为地址池已被锁定。
- 如果先执行LCK SECTION，再执行锁定地址池命令LCK POOL，整个地址池均被锁定。
- 如果先执行LCK POOL，再执行LCK SECTION命令解锁某个地址段，执行失败。锁定地址池后不允许单独解锁地址池中的地址段。
- 如果某个地址段已经被锁定，执行LCK POOL再次锁定该地址段，执行成功。
- 如果某个地址段属于解锁状态，执行LCK POOL再次解锁该地址段，执行成功。
- 如果先执行LCK SECTION锁定某个地址段，再执行LCK POOL解锁整个地址池，命令执行后，地址池中的地址段全部解锁。
- 如果地址池中有地址处于分配状态，则该地址池不能被删除，所以当要删除地址池时，可以首先将该地址池使用LCK POOL锁定，则所有地址段均被锁定。等地址池中所有的地址都回收以后，就可以删除该地址池了。
- 如果地址正在通过ACT RECYCLE回收，其所在的地址池或地址段无法进行LCK SECTION，后加参数UNLOCK操作。地址回收完毕后，执行解锁命令LCK SECTION，后将参数IsLocked设置为UNLOCK才能成功。
- 地址池被锁之后，地址池中的地址将不会分配给用户，如果绑定锁定地址池的APN下地址不足，会导致用户因为无地址接入失败。地址段被锁之后，地址段中的地址将不分配给用户，但不影响地址池中的其他地址段。

#### [操作用户权限](#ZH-CN_CONCEPT_0182837117)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0182837117)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| POOLNAME | 地址池名称 | 可选必选说明：必选参数<br>参数含义：该参数用于指定已配置的地址池的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～79，单位是字节。由“_”、“-”、数字、大小写字母和“.”组成，不能以“.”开头且不能出现连续两个“.”，不区分大小写。<br>默认值：无<br>配置原则：无 |
| SECTIONNUM | 地址段号 | 可选必选说明：必选参数<br>参数含义：该参数用于指定配置Section的编号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～63。<br>默认值：无<br>配置原则：无 |
| ISLOCKED | 地址段锁定标志 | 可选必选说明：必选参数<br>参数含义：指定地址池锁定标志，本参数用于控制锁定或者解锁地址段。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- UNLOCK：解锁开关。<br>- LOCK：锁定开关。<br>默认值：无<br>配置原则：<br>- LOCK用于锁定地址池中的一个地址段，命令会立即起作用，即该地址段被禁用，不能用于分配IP地址。该地址段中已被分配的IP地址仍然照常使用，直到用户去活，或者系统强制回收。<br>- UNLOCK用于解除地址池中地址段的锁定状态。 |

#### [使用实例](#ZH-CN_CONCEPT_0182837117)

假设运营商需要锁定地址池pool1中地址段0：

```
LCK SECTION:POOLNAME="pool1",SECTIONNUM=0,ISLOCKED=LOCK;
```
