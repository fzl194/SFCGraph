#### [业务场景](#ZH-CN_TOPIC_0000001472735009)

当UE与AMF的会话状态不匹配，或者UE的网络切片不可用时，AMF发起会话释放。

1. 消息交互：AMF发送[Nsmf_PDUSession_ReleaseSmContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_ReleaseSmContext/Nsmf_PDUSession_ReleaseSmContext Request_83854765.md)给SMF。
  触发条件：当UE与AMF的会话状态不匹配，或者UE的网络切片不可用时。AMF发起会话释放。
  关键信元：
  | PDU Session ID | 标识一个PDU会话。 |
  | --- | --- |
  | Cause | 指示SM上下文释放的原因。 |
  | ueLocation | 包含UE的位置信息。 |
2. 消息交互：AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  触发条件：在释放SM上下文之前，如果需要N1、N2信令信息，AMF可以向SMF发送带有释放指示的消息。
  关键信元：
  | upCnxState | 用户面连接状态。 |
  | --- | --- |
  | ngApCause | 去激活PDU会话的用户面连接的NGAP原因。 |
3. 消息交互：SMF发送[PFCP Session Deletion Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Deletion Request_86168956.md)给UPF。
  触发条件：SMF给UPF发送此消息请求删除PFCP会话。
  关键信元：
  | message type | 指示PFCP消息类型，比如取值为 pfcp-session-deletion-request。 |
  | --- | --- |
4. 消息交互：UPF发送[PFCP Session Deletion Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Deletion Request_86168956.md)给SMF。
  关键信元：
  | Cause | 指示接受或拒绝相应的请求消息。比如取值为request-accepted。 |
  | --- | --- |
  | Usage Report | 如果在用户面功能中已经开通了一个URR，并且该PFCP会话被删除，并且在用户面功能中可以使用URR的流量使用测量，那么该信元将会出现。 |
  相关机制：UPF收到请求消息后将丢弃与该PDU会话相关的数据包，并释放与N4会话相关的所有隧道资源和上下文。
5. 消息交互：SMF发送[Nsmf_PDUSession_ReleaseSmContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_ReleaseSmContext/Nsmf_PDUSession_ReleaseSmContext Response_83854789.md)给AMF。
  相关机制：如果成功，SMF响应状态码“204 No Content”，或者“200 OK”，如果失败或重定向，SMF返回“4xx/5xx”的HTTP状态码。
6. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给AMF。
  关键信元：
  | n1SmMsg | 指示发送给UE的N1 SM Container。传递的消息是PDU session release command。 |
  | --- | --- |
  | n2SmInfo | 指示发送给(R)AN的N2 SM Information。传递的消息是PDU SESSION RESOURCE RELEASE COMMAND。 |
  相关机制：SMF收到请求消息后返回响应消息，包含发送给UE的N1 SM Container，通知UE释放PDU会话。当PDU会话的UP连接处在激活状态时，还包含发送给(R)AN的N2 SM Information，通知(R)AN释放PDU会话相关的资源。
7. 消息交互：AMF发送[PDU SESSION RESOURCE RELEASE COMMAND](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PDU SESSION RESOURCE RELEASE COMMAND_76785040.md)给(R)AN。
  触发条件：会话删除需要通知(R)AN和UE侧时，触发该消息。
  关键信元：
  | **PDU Session Resource to Release List** | PDU会话资源释放列表。 |
  | --- | --- |
  | **PDU Session Resource to Release Item** | PDU会话资源释放项。 |
  | PDU Session ID | 该IE用于标识一个UE的PDU会话。 |
  | PDU SESSION RESOURCE RELEASE COMMAND Transfer | SMF通过AMF透传给RAN的PDU SESSION RESOURCE RELEASE COMMAND Transfer IE。 |
  相关机制：当UE处于空闲态并且指示需要发送N1 SM消息时，AMF指示网络侧发起Service Request流程向(R)AN发送PDU Session ID、N1 SM container。当UE处于连接态时，AMF向(R)AN发送资源释放请求，请求中包含从SMF接收的N2 SM Information、N1 SM Container。
8. 交互流程：(R)AN请求UE释放会话。
  相关机制：(R)AN发送信令消息给UE，将SMF需要发送给UE的PDU Session ID、N1 SM Container（PDU session release command）消息转发至UE。
9. 消息交互：AMF发送[PDU session release command](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/PDU session release command_76785262.md)给UE。
  触发条件：会话释放需要通知UE侧时，触发该消息。
  关键信元：
  | PDU Session ID | 该IE用于标识一个UE的PDU会话。 |
  | --- | --- |
10. 消息交互：(R)AN发送[PDU SESSION RESOURCE RELEASE RESPONSE](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PDU SESSION RESOURCE RELEASE RESPONSE_76785168.md)给AMF。
  关键信元：
  | **PDU Session Resource Released List** | PDU会话资源释放列表。 |
  | --- | --- |
  | **PDU Session Resource Released Item** | PDU会话资源释放项。 |
  | PDU Session ID | 该IE用于标识一个UE的PDU会话。 |
  | PDU SESSION RESOURCE RELEASE RESPONSE Transfer | RAN通过AMF透传给SMF的PDU SESSION RESOURCE RELEASE RESPONSE Transfer IE。 |
  | User Location Information | 该IE用于提供UE的位置信息。 |
11. 消息交互：AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  触发条件：AMF向SMF发送消息将从(R)AN接收到的N2 SM Information转发给SMF。
  关键信元：
  | n2SmInfo | 从(R)AN接收到的N2 SM Information。 |
  | --- | --- |
  | n2SmInfoType | N2 SM消息类型，比如PDU_RES_REL_CMD 指示PDU会话资源释放命令传输。 |
  | ueLocation | UE的位置信息。 |
12. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给AMF。
  相关机制：如果请求成功，SMF响应状态码“204 No Content”，或者“200 OK”，如果失败或重定向，SMF返回“4xx/5xx”的HTTP状态码。
13. 消息交互：UE发送[PDU session release complete](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/PDU session release complete_76785085.md)给AMF。
  关键信元：
  | PDU Session ID | 标识一个PDU会话。 |
  | --- | --- |
  | PDU session authentication complete message identity | 标识message type类型，比如PDU session authentication complete。 |
14. 消息交互：AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  触发条件：AMF向SMF发送消息将从UE接收到的N1 SM Information转发给SMF。
  关键信元：
  | n1SmMsg | 从UE接收到的N1 SM Information。 |
  | --- | --- |
  | ueLocation | UE的位置信息。 |
15. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给AMF。
  相关机制：如果请求成功，SMF响应状态码“204 No Content”，或者“200 OK”，如果失败或重定向，SMF返回“4xx/5xx”的HTTP状态码。
16. 消息交互：SMF发送[Nsmf_PDUSession_NotifySMContextStatus Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_NotifySMContextStatus/Nsmf_PDUSession_NotifySMContextStatus Request_35255068.md)给AMF。
  触发条件：SMF释放会话后发送消息上报SM上下文的状态信息。
  关键信元：
  | statusInfo | 包含SM上下文的状态信息。 |
  | --- | --- |
  | resourceStatus | 指示SM上下文或PDU会话资源的状态。比如：RELEASED指示SM上下文或PDU会话资源释放。 |
17. 消息交互：AMF发送[Nsmf_PDUSession_NotifySMContextStatus Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_NotifySMContextStatus/Nsmf_PDUSession_NotifySMContextStatus Response_83734777.md)给SMF。
  相关机制：如果成功，目标NF响应状态码“204 No Content”并且响应的消息体为空。如果失败或重定向，目标NF返回“4xx/5xx”的HTTP状态码。
18. 消息交互：SMF发送[Npcf_SMPolicyControl_Delete](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Delete_67572326.md)给PCF。
  触发条件：当SMF收到AMF发送的删除PDU会话相关的SM上下文请求时，如果SMF判断需要向PCF请求删除PDU会话，则SMF发送消息请求删除SM策略关联。
  关键信元：
  | userLocationInfo | 表示用户当前的位置信息。 |
  | --- | --- |
  | nrLocation | 指示NR用户位置。 |
19. 消息交互：PCF发送[Npcf_SMPolicyControl_Delete](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Delete_67572326.md)给SMF。
  相关机制：如果请求被接受，PCF应响应“204 No Content”HTTP状态码，如果失败或重定向时，PCF应返回HTTP状态码。
