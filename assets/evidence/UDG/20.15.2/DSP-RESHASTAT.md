#### [命令功能](#ZH-CN_CONCEPT_0000001238001520)

该命令用于查询系统资源的HA状态，HA（High Availability）是为系统存储服务提供高可靠性的模块。当系统进行升级时，需要确保所有组件状态正常，可以使用该命令确定资源的进程、组件状态是否正常。

#### [注意事项](#ZH-CN_CONCEPT_0000001238001520)

该命令执行后立即生效。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001238001520)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001238001520)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| RESNAME | 资源名称 | 可选必选说明：可选参数<br>参数含义：该参数表示节点或容器资源的名称。<br>配置原则：当不输入时显示所有资源的HA状态。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。不支持空格，区分大小写。<br>默认值：无 |

#### [使用实例](#ZH-CN_CONCEPT_0000001238001520)

查询 “OMU1” 的HA状态：

```
DSP RESHASTAT:RESNAME="OMU1";
```

```
RETCODE = 0  操作成功
 
结果如下:
 -------------------------
 逻辑资源编号 = 1
     资源名称 = OMU1
       HA状态 = 正常    
 (结果个数 = 1)
  ---   END
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000001238001520)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 逻辑资源编号 | 表示逻辑资源编号，该编号在网元进行分配。 |
| 资源名称 | 表示节点或容器的资源名称。 |
| HA状态 | 表示HA状态。 包括<br>“正常”<br>、<br>“异常”<br>及<br>“暂停”<br>。 |
