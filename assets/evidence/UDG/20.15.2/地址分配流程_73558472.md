#### [地址分配总体流程](#ZH-CN_TOPIC_0173558472)

PGW-U/UPF地址分配原理如 [图1](#ZH-CN_TOPIC_0173558472__fig3131182010388) 所示。

**图1** PGW-U/UPF地址分配方式原理图

<br>

![](地址分配流程_73558472.assets/zh-cn_image_0233089636.png)

1. PGW-C/SMF主动向PGW-U/UPF发送偶联建立请求消息。PGW-U/UPF在偶联建立响应消息中会将用户面地址分配能力上报给PGW-C/SMF。
2. PDU会话建立时，PGW-C/SMF通过会话建立请求消息指示PGW-U/UPF具体的地址分配方式，消息中的UE IP Address信元包含用户接入到外部数据包网络时所需要使用的地址类型及IP地址。
3. PGW-U/UPF根据会话建立请求的UE IP Address信元参数决定地址分配方式：外部地址分配或用户面地址分配。
4. PGW-U/UPF根据具体的地址分配方式完成UE地址分配、路由发布、地址白名单或冲突检测功能。
5. 采用用户面地址分配时，PGW-U/UPF可为UE分配IP地址，并将分配好的IP地址在PDU会话建立响应中带给PGW-C/SMF。

#### [用户面地址分配能力上报](#ZH-CN_TOPIC_0173558472)

在偶联消息交互中，PGW-U/UPF会将用户面地址分配能力上报给PGW-C/SMF。PGW-U/UPF在偶联建立/响应消息（PFCP Association Setup Request/Response）或偶联更新请求/响应（PFCP Association Update Request/Response）消息中会携带UP Function Flag信元指示PGW-U/UPF是否具备用户面地址分配能力。UP Function Flag信元中的ADUP标志位标识PGW-U/UPF支持用户面地址分配方式。该标志位会恒定设为1，标识PGW-U/UPF支持用户面地址分配方式。

#### [地址分配方式选择](#ZH-CN_TOPIC_0173558472)

PGW-C/SMF通过会话建立请求消息（PFCP Session Establishment Request）中的UE IP Address信元指示PGW-U/UPF具体的地址分配方式。PGW-U/UPF根据UE IP Address信元中的CHV4、CHV6、V4、V6 标志位判断采用外部地址分配或用户面地址分配方式。其中CHV6和CHV4表示U面分配的地址，V4和V6表示C面分配的地址。UE IP Address信元如 [图2](#ZH-CN_TOPIC_0173558472__fig1267415492190) 所示，PGW-U/UPF地址分配方式选择逻辑如 [表1](#ZH-CN_TOPIC_0173558472__table1059943417228) 所示。

> **说明**
> 当设置 [BIT391](../../../../../../OM参考/软件参数/UDG软件参数/用户面软件参数/软件参数/地址管理/BIT391 控制是否使用PFCP消息中UE IP Address信元的Spare bit来决策UP分配用户地址_08683838.md) 为1时，PGW-U/UPF根据UE IP Address信元中的Spare作为扩展UPA，通过Spare、V4、V6 标志位判断采用外部地址分配或用户面地址分配方式。

#### [地址分配流程](#ZH-CN_TOPIC_0173558472)

**外部地址分配方式**

- **功能点1：路由发布**
  采用外部地址分配方式时，PGW-U/UPF的主要功能是路由发布。PGW-U/UPF将生成WLR（Wireless Route）路由，并通过动态路由协议（如OSPF）发布到骨干网上。路由发布的具体策略如 [图3](#ZH-CN_TOPIC_0173558472__fig64331419205) 所示。
  **图3** PGW-U/UPF路由发布策略

  <br>

  ![](地址分配流程_73558472.assets/zh-cn_image_0255922497.png)
  > **说明**
  > 当 [**ADD POOL**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/地址池配置/添加地址池（ADD POOL）_82837132.md) 中的 “POOLTYPE” 参数设置为 “EXTERNAL” ，该地址池即为EXTERNAL类型地址池。
- **功能点2：UE地址白名单检测**
  PGW-U/UPF支持对UE地址进行白名单检测。当UE地址的白名单检测功能开启时（ [**ADD POOL**](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话地址管理/地址池配置/添加地址池（ADD POOL）_82837132.md) 中的 “CHECKIPVALID” 参数设置为 “ENABLE” ），PGW-U/UPF将检查PGW-C/SMF下发的UE地址是否在配置的EXTERNAL类型的POOL内。如果不在，返回会话建立失败响应给PGW-C/SMF。
- **功能点3：UE地址冲突检测**
  PGW-U/UPF支持对UE地址进行冲突检测。当PGW-C/SMF下发UE地址但未下发掩码时，如果PGW-U/UPF检测到相同VPN属性下的UE地址冲突，则根据 [BYTE44 控制地址冲突检测机制](../../../../../../OM参考/软件参数/UDG软件参数/用户面软件参数/软件参数/地址管理/BYTE44 控制地址冲突检测机制_08922544.md) 配置决定去活原用户或者拒绝新用户接入。

**用户面地址分配方式**

- **功能点1：UE地址分配**
  采用用户面地址分配方式时，PGW-U/UPF在承载上下文激活/PDU会话建立阶段直接从PGW-U/UPF上配置的地址池中为UE分配IP地址，同时完成路由发布功能。PGW-U/UPF配置的地址池主要是由运营商规划的可用地址段。基本的UE地址分配过程如 [图4](#ZH-CN_TOPIC_0173558472__fig183671410125310) 所示。
  **图4** 基本UE地址分配过程

  <br>

  ![](地址分配流程_73558472.assets/zh-cn_image_0268044106.png)
  > **说明**
  > - 每个APN最多支持16个地址池。
  > - 只支持本地地址池优先级可配置。
  > - 只支持地址池级别的优先级配置，不支持地址池下的地址段的优先级配置。
  > - 只支持同类型地址池（IPv4地址池或者是IPv6地址池）间的优先级比较。

- **功能点2：UE地址****上报**
  PGW-U/UPF为UE分配IP地址后，将通过会话响应消息中的Created PDR或者Created Traffic Endpoint信元将UE的IP地址上报给PGW-C/SMF。
  > **说明**
  > 本地IPv4地址池不分配主机号为全0和全1的地址给UE。
- **功能点3：路由发布**
  PGW-U/UPF为UE分配IP地址后，将生成WLR路由，并通过动态路由协议（如OSPF）发布到骨干网上。用户面地址分配方式的详细细节请查看 [GWFD-010105 用户面地址分配](../../GWFD-010105 用户面地址分配_68664420.md) 。
- **功能点4：RADIUS下发地址池名称**
  SMF/PGW-C从RADIUS获取地址池名称，SMF/PGW-C将地址池名称下发到PGW-U/UPF，如果PGW-U/UPF命中该地址池名称，则为UE分配IP地址并生成WLR路由。如果PGW-U/UPF未命中该地址池名称，则UE分配地址失败，用户退网。
