#### [业务场景](#ZH-CN_TOPIC_0000001422895266)

PCF识别用户配置策略有修改时指示SMF更新会话策略，SMF更新数据后发起PDU会话修改。

1. 消息交互：PCF发送[Npcf_SMPolicyControl_UpdateNotify](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_UpdateNotify_67572328.md)给SMF。
  触发条件：PCF识别配置策略有修改，指示SMF更新会话策略，SMF更新数据后发起PDU会话修改。
  关键信元：
  | smPolicyDecision | SM策略信息。 |
  | --- | --- |
  | sessRules | 表示一组会话规则，是会话级QoS规则，用于进行PDU会话粒度的策略控制。 |
  | pduSessionId | 标识一个PDU会话。 |
  | supi | 标识用户SUPI。 |
2. 消息交互：SMF发送[Npcf_SMPolicyControl_UpdateNotify](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_UpdateNotify_67572328.md)给PCF。
  相关机制：SMF应响应“204 No Content”HTTP状态码，已成功更新策略，响应的消息体为空。SMF响应“200 OK”，表示PCF下发的PCC规则和/或会话规则部分未安装/激活成功。
3. 消息交互：SMF发送[Namf_Communication_N1N2MessageTransfer Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Namf/N11/Namf_Communication_N1N2MessageTransfer/Namf_Communication_N1N2MessageTransfer Request_36361278.md)给AMF。
  触发条件：QoS改变需要通知(R)AN和UE侧时，触发该消息。
  关键信元：
  | n1SmMsg | 指示发送给UE的N1 SM Container。传递的消息是PDU session modification command。 |
  | --- | --- |
  | n2SmInfo | 指示发送给(R)AN的N2 SM Information。传递的消息是PDU SESSION RESOURCE MODIFY REQUEST。 |
4. 消息交互：AMF发送[Namf_Communication_N1N2MessageTransfer Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Namf/N11/Namf_Communication_N1N2MessageTransfer/Namf_Communication_N1N2MessageTransfer Response_84800549.md)给SMF。
  关键信元：
  | Cause | 表示AMF上N1/N2消息传输处理的结果，比如N1_N2_TRANSFER_INITIATED表示AMF已向UE和/或AN发起N1/N2消息传输。 |
  | --- | --- |
5. 消息交互：AMF发送[PDU SESSION RESOURCE MODIFY REQUEST](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PDU SESSION RESOURCE MODIFY REQUEST_76784384.md)给(R)AN。
  触发条件：QoS改变需要通知(R)AN和UE侧时，触发该消息。
  关键信元：
  | **PDU SESSION RESOURCE MODIFY REQUEST List** | PDU会话资源修改请求列表。 |
  | --- | --- |
  | **PDU SESSION RESOURCE MODIFY REQUEST Item** | PDU会话资源修改请求项。 |
  | PDU SESSION RESOURCE MODIFY REQUEST Transfer | SMF通过AMF透传给(R)AN的PDU SESSION RESOURCE MODIFY REQUEST Transfer IE。 |
  | qosFlowAddOrModifyRequestList | 增加、删除、修改QoS flow场景携带该信元。 |
  | QoS Flow Identifier | 增加的Qos flow对应的QFI。 |
  | QOS flow to release list | 删除QoS flow携带该信元。 |
  | QoS Flow Level QoS Parameters | 增加或修改携带该信元，具体包括5QI、ARP等。 |
  | NAS-PDU | AMF发送给UE的消息，在RAN不解析。 |
6. 交互流程：(R)AN请求UE建立会话。
  触发条件：QoS改变需要通知(R)AN和UE侧时，触发该消息。
  相关机制：(R)AN发送信令消息给UE，将SMF需要发送给UE的PDU Session ID、N1 SM Container（PDU session modification command）消息转发至UE，请求UE更新QoS信息。
7. 消息交互：AMF发送[PDU session modification command](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/PDU session modification command_76783986.md)给UE。
  触发条件：QoS改变需要通知UE侧时，触发该消息。
  关键信元：
  | Authorized QoS rules | QoS规则信元用于指示UE使用的一组QoS规则。 |
  | --- | --- |
  | qos-rule-identifier | 标识QoS Rule。 |
  | rule operation code | 标识QoS Rule操作比如：<br>modify existing QoS rule and delete packet filters：修改已存在的QoS Rule以及包过滤器。<br>delete existing QoS rule：删除当前的QoS rule。<br>create new QoS rule：创建新的QoS rule。 |
  | Authorized Qos flow descriptions | QoS flow descriptions信元用于表示UE使用的QoS流描述的集合，每个QoS流描述包含一组参数集合。 |
  | QoS Flow Identifier | 标识qos flow。 |
  | qos flow operation code | 指示qos flow操作，比如：<br>create new qos flow description：创建新的QoS flow。<br>delete existing QoS flow description：删除当前已存在的QoS flow。 |
  | 5QI | 标识一个标准的5G QoS特征组合，5QI值和对应的这组QoS特征一一映射。 |
8. 消息交互：(R)AN发送[PDU SESSION RESOURCE MODIFY RESPONSE](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N2接口/相关消息解释/PDU SESSION RESOURCE MODIFY RESPONSE_76784667.md)给AMF。
  关键信元：
  | **PDU SESSION RESOURCE MODIFY RESPONSE Item** | PDU会话资源修改响应项。 |
  | --- | --- |
  | PDU Session ID | 该IE用于标识一个UE的PDU会话。 |
  | PDU SESSION RESOURCE MODIFY RESPONSE Transfer | (R)AN通过AMF透传给SMF的PDU SESSION RESOURCE MODIFY RESPONSE Transfer IE。 |
  | gtptunnel | AN侧隧道信息。 |
9. 消息交互：AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  触发条件：AMF向SMF发送消息将从(R)AN接收到的N2 SM Information转发给SMF。
  关键信元：
  | n2SmInfo | 从(R)AN接收到的N2 SM Information。 |
  | --- | --- |
  | n2SmInfoType | N2 SM消息类型，比如PDU_RES_MOD_RSP |
  | ueLocation | UE的位置信息。 |
10. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给AMF。
  相关机制：如果请求成功，SMF响应状态码“204 No Content”，或者“200 OK”，如果失败或重定向，SMF返回“4xx/5xx”的HTTP状态码。
11. 消息交互：SMF发送[PFCP Session Modification Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)给UPF。
  触发条件：QoS参数修改涉及UPF侧修改时触发该消息，将更新的QoS信息通知UPF。
12. 消息交互：UPF发送[PFCP Session Modification Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)给SMF。
  关键信元：
  | Cause | 指示接受或拒绝相应的请求消息。 |
  | --- | --- |
13. 消息交互：UE发送[PDU session modification complete](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N1接口/相关消息解释/PDU session modification complete_76785362.md)给AMF。
  关键信元：
  | PDU Session ID | 标识一个PDU会话。 |
  | --- | --- |
  | PDU session authentication complete message identity | 标识message type类型，比如PDU session authentication complete。 |
14. 消息交互：AMF发送[Nsmf_PDUSession_UpdateSmContext Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Request_35255324.md)给SMF。
  触发条件：AMF向SMF发送消息将从UE接收到的N1 SM Information转发给SMF。
  关键信元：
  | n1SmMsg | 从UE接收到的N1 SM Information。 |
  | --- | --- |
  | ueLocation | UE的位置信息。 |
15. 消息交互：SMF发送[Nsmf_PDUSession_UpdateSmContext Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Nsmf/N11/Nsmf_PDUSession_UpdateSmContext/Nsmf_PDUSession_UpdateSmContext Response_34935436.md)给AMF。
  相关机制：如果请求成功，SMF响应状态码“204 No Content”，或者“200 OK”，如果失败或重定向，SMF返回“4xx/5xx”的HTTP状态码。
16. 消息交互：SMF发送[PFCP Session Modification Request](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Request_32686661.md)给UPF。
  触发条件：QoS参数修改涉及UPF侧修改时触发该消息。将更新的QoS信息通知UPF。
  关键信元：
  | Remove PDR | SMF根据SM策略删除相应的PDR。 |
  | --- | --- |
  | Remove FAR | SMF根据SM策略删除相应的FAR。 |
  | Remove QER | SMF根据SM策略删除相应的QER。 |
  | Create PDR | SMF根据SM策略新增相应的PDR。 |
  | Create FAR | SMF根据SM策略新增相应的FAR。 |
  | Create QER | SMF根据SM策略新增相应的QER。 |
  | Update PDR | SMF根据SM策略更新相应的PDR。 |
  | QoS Flow Identifier | 指示QoS Flow标识。 |
17. 消息交互：UPF发送[PFCP Session Modification Response](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/非服务化接口协议/N4接口/相关消息解释/PFCP Session Modification Response_86008994.md)给SMF。
  关键信元：
  | Cause | 指示接受或拒绝相应的请求消息。 |
  | --- | --- |
18. 消息交互：SMF发送[Npcf_SMPolicyControl_Update](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Update_67572331.md)给PCF。
  触发条件：部署动态PCC策略，则SMF向PCF上报订阅事件。
  关键信元：
  | repPolicyCtrlReqTriggers | SMF向PCF上报发生的事件。比如：<br>SUCC_RES_ALLO表示资源分配成功。 |
  | --- | --- |
19. 消息交互：PCF发送[Npcf_SMPolicyControl_Update](../../../../../../../快速入门/协议与流程/5G Core信令分析/消息及信元/服务化接口协议/Npcf/N7/Npcf_SMPolicyControl_Update_67572331.md)给SMF。
  | Cause | 指示接受或拒绝相应的请求消息。 |
  | --- | --- |
