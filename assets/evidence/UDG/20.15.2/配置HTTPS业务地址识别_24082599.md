## [操作场景](#ZH-CN_OPI_0124082599)

> **说明**
> 适用于PGW-U、UPF。

当用户访问的业务采用HTTPS协议承载时，需要部署HTTPS业务地址识别特性，对业务实施计费与策略控制。

> **说明**
> UDG SSL/TLS证书识别HTTPS应用不需要配置，版本默认支持。

## [必备事项](#ZH-CN_OPI_0124082599)

前提条件

- 请仔细阅读[GWFD-110203 HTTPS业务地址识别](../GWFD-110203 HTTPS业务地址识别_71023371.md)。
- 完成加载license（如果有需求，请联系华为技术支持工程师处理）。

## [操作步骤](#ZH-CN_OPI_0124082599)

1. 打开本特性的License配置开关。
  [**SET LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)
2. 配置HTTPS报文的PCC策略组。
    a. 配置URR。
      [**ADD URR**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则/增加URR（ADD URR）_82837629.md)
    b. 配置使用量上报组属性URRGROUP，包括上下行发起的业务对应的URR。
      [**ADD URRGROUP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/计费控制/使用率上报规则组/增加URR组（ADD URRGROUP）_82837634.md)
    c. 配置PCC策略组，绑定 [步骤 2.b](#ZH-CN_OPI_0124082599__subd) 配置的使用量上报规则组。
      [**ADD PCCPOLICYGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/增加PCC策略组（ADD PCCPOLICYGRP）_82837606.md)
3. 配置HTTPS报文的解析规则。
    a. 配置HTTPS业务的域名信息。
      [**ADD HOST**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/三四层规则管理/主机名/增加Host（ADD HOST）_82837322.md)
    b. 配置三四层过滤条件。
      [**ADD FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/增加过滤器（ADD FILTER）_82837348.md)
      > **说明**
      > 如果能预先获取HTTPS Server IP列表，在三四层filter中配置端口和Server IP可实现HTTPS地址识别功能。这种情况下就无需配置 [步骤 4](#ZH-CN_OPI_0124082599__step33) 。
    c. 配置FlowFilter。
      [**ADD FLOWFILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/增加流过滤器（ADD FLOWFILTER）_82837361.md)
    d. 绑定三四层过滤条件。
      [**ADD FLTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的过滤器绑定/增加流过滤器的过滤器绑定关系（ADD FLTBINDFLOWF）_82837366.md)
    e. 配置HTTPS解析规则。
      [**ADD RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/增加规则（ADD RULE）_82837266.md)
      > **说明**
      > 如果运营商希望基于流过滤器组设置过滤条件时，配置 “流过滤器或流过滤器组选择(FILTERINGMODE)” 为 “FLOWFILTERGRP” ，并且使用 [**ADD FLOWFILTERGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器组/增加流过滤器组（ADD FLOWFILTERGRP）_82837384.md) 命令配置流过滤器组。
4. **可选：**配置DNS报文的解析规则。
  > **说明**
  > 若能预先获取HTTPS Server的IP列表，则无需配置DNS报文的解析规则；若不能预先获取HTTPS Server IP列表，只能获取域名信息，则需要配置DNS报文解析规则。
    a. 配置三四层过滤条件。
      [**ADD FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/过滤器/增加过滤器（ADD FILTER）_82837348.md)

      [**ADD L7FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/七层规则管理/七层过滤器/增加七层过滤器（ADD L7FILTER）_82837397.md)

      [**ADD FLOWFILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器/增加流过滤器（ADD FLOWFILTER）_82837361.md)
    d. 绑定三四层过滤条件。
      [**ADD FLTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的过滤器绑定/增加流过滤器的过滤器绑定关系（ADD FLTBINDFLOWF）_82837366.md)
    e. 绑定七层过滤条件。
      [**ADD PROTBINDFLOWF**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器的协议绑定/增加流过滤器协议绑定关系（ADD PROTBINDFLOWF）_82837370.md)
    f. 配置DNS解析规则。
      [**ADD RULE**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/增加规则（ADD RULE）_82837266.md)
      > **说明**
      > 如果运营商希望基于流过滤器组设置过滤条件时，配置 “流过滤器或流过滤器组选择(FILTERINGMODE)” 为 “FLOWFILTERGRP” ，并且使用 [**ADD FLOWFILTERGRP**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/流过滤器组/增加流过滤器组（ADD FLOWFILTERGRP）_82837384.md) 命令配置流过滤器组。
    g. 将新配置的Filter、L7Filter置为生效。
      [**SET REFRESHSRV**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/流过滤器管理/业务刷新/业务刷新（SET REFRESHSRV）_82837355.md)

    a. 配置知名端口。
      [**ADD WELLKNOWNPORT**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/三四层规则管理/知名端口/增加知名端口（ADD WELLKNOWNPORT）_82837332.md)
    b. 加载特定的知识库文件。如已加载，请跳过该步骤。
      [**LOD SIGNATUREDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/识别特征库/加载协议特征库文件（LOD SIGNATUREDB）_82837716.md)
    c. 加载协议解析库。如已加载，请跳过该步骤。
      [**LOD PARSERDB**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/协议识别/特征库管理/解析特征库/加载协议解析数据库（LOD PARSERDB）_82837719.md)

  [**ADD USRRELATEIDEN**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/三四层规则管理/用户关联识别/增加用户关联识别（ADD USRRELATEIDEN）_82837433.md)

  > **说明**
  > 配置User Profile分别绑定HTTPS业务规则和DNS报文解析规则。DNS报文解析规则优先级最高。
    a. 配置UserProfile。

## [任务示例](#ZH-CN_OPI_0124082599)

任务描述

在 UDG 上开启www.example.com HTTPS业务的内容计费功能。其中HTTPS报文的对端服务器端口号为443，DNS对端服务器端口号为53。

脚本

//打开本特性的License配置开关。

```
SET LICENSESWITCH: LICITEM="LKV3G5HSAP01",SWITCH=ENABLE;
```

//配置HTTPS报文的PCC策略组。

```
ADD URR:URRNAME="urr_https",URRID=300,USAGERPTMODE=OFFLINE;
```

```
ADD URRGROUP: URRGROUPNAME="urrgp_https", UPURRNAME1="urr_https", DOWNURRNAME1="urr_https";
```

```
ADD PCCPOLICYGRP: PCCPOLICYGRPNM="pccpg_https", URRGROUPNAME="urrgp_https";
```

//配置HTTPS报文的解析规则。

```
ADD HOST:HOSTNAME="ht_https",DOMAINVALUE="www.example.com";
```

```
ADD FILTER:FILTERNAME="filter_https",L34PROTTYPE=STRING,L34PROTOCOL=ANY,SVRIPMODE=HOSTNAME,HOSTNAME="ht_https",SVRSTARTPORT=443,SVRENDPORT=443;
```

```
ADD FLOWFILTER:FLOWFILTERNAME="ff_https";
```

```
ADD FLTBINDFLOWF:FLOWFILTERNAME="ff_https",FILTERNAME="filter_https";
```

```
ADD RULE:RULENAME="rule_https",POLICYTYPE=PCC,FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="ff_https",PRIORITY=120,POLICYNAME="pccpg_https";
```

//配置DNS报文的解析规则。

```
ADD FILTER:FILTERNAME="filter_dns",L34PROTTYPE=STRING,L34PROTOCOL=UDP,SVRSTARTPORT=53,SVRENDPORT=53;
```

```
ADD L7FILTER:L7FILTERNAME="l7f_dns",SUBL7FLTNAME="sl7_test",URL="*/*";
```

```
ADD FLOWFILTER:FLOWFILTERNAME="ff_dns";
```

```
ADD FLTBINDFLOWF:FLOWFILTERNAME="ff_dns",FILTERNAME="filter_dns";
```

```
ADD PROTBINDFLOWF:FLOWFILTERNAME="ff_dns",PROTOCOLNAME="dns",L7FILTERNAME="l7f_dns";
```

```
ADD RULE:RULENAME="rule_dns",POLICYTYPE=PCC,FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="ff_dns",PRIORITY=100,POLICYNAME="pccpg_https";
```

//将新配置的Filter置为生效。

```
SET REFRESHSRV:REFRESHTYPE=USERPROFILE;
```

//配置 UDG 的协议识别库和协议解析库。

```
LOD SIGNATUREDB:LOADMODE=LATEST;
```

```
LOD PARSERDB:LOADMODE=LATEST;
```

//配置开启facebook协议的用户关联识别功能。

```
ADD USRRELATEIDEN:PROTOCOLNAME="facebook",USRRLTIDENSW=ENABLE;
```

//配置HTTPS报文和DNS报文使用的策略组合。

```
ADD USERPROFILE:USERPROFILENAME="up_test";
```

```
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="rule_https";
```

```
ADD RULEBINDING:USERPROFILENAME="up_test",RULENAME="rule_dns";
```
