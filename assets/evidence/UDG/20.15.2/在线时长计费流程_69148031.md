# 在线时长计费流程

在线计费中基于业务时长的计费业务流程如 [图1](#ZH-CN_CONCEPT_0169148031__fig87541051175) 所示。

**图1** 在线计费中基于业务时长的计费特性业务流程图

<br>

![](在线时长计费流程_69148031.assets/zh-cn_image_0204997177.png)

1. 用户激活
    - 用户发送会话建立请求。
    - PGW-C/SMF在回激活应答之前发送CCR-I消息到OCS。
    - OCS允许用户激活，向PGW-C回CCA-I消息。
    - PGW-C/SMF向PGW-U/UPF发送PFCP Session Establishment Request消息。
    - PGW-U/UPF向PGW-C/SMF返回PFCP Session Establishment Response消息。
    - PGW-C/SMF给用户返回会话建立响应。
2. 用户访问业务
    - PGW-U/UPF收到用户访问业务请求。
    - PGW-U/UPF向PGW-C/SMF请求配额。
    - PGW-C/SMF发送PFCP Session Report Response消息响应给PGW-U/UPF。
    - PGW-C/SMF向OCS发送CCR-U进行配额申请。
    - OCS向PGW-C/SMF发送CCA-U返回RG1的配额，其中包含时间配额(CC-Time)和计费方式。消息中的关键信元如[表1](#ZH-CN_CONCEPT_0169148031__table681522169)所示。
    - PGW-C/SMF向PGW-U/UPF下发配额请求和计费方式。
    - PGW-U/UPF回应PFCP Session Modification Response消息给PGW-C/SMF。
    - 用户访问业务，PGW-U/UPF统计业务访问时长。
      *表1 关键信元说明*

      | 关键信元 | 说明 |
      | --- | --- |
      | CC-Time | 该AVP携带用户业务对应的时间配额。 |
      | Quota-Consumption-Time | 该AVP携带值为0或不为0，表示连续时长计费方式或QCT计费方式。 |
      | Time-Quota-Mechanism<br>---Time-Quota-Type | 该AVP携带CTP或DTP计费方式。 |
      | Time-Quota-Mechanism<br>---Base-Time-Interval | 该AVP携带控制时长配额的基本时间间隔。 |
      > **说明**
      > 如果同时携带QCT和TQM，则TQM优先；如果没有携带QCT和TQM，则根据PGW-C/SMF上配置的QCT值决定是采用QCT>0模式还是QCT＝0模式。
3. 用户去活
    - 用户终止业务，UE发起终止会话请求。
    - PGW-C/SMF向用户回终止会话响应。
    - PGW-C/SMF通知PGW-U/UPF用户去活。
    - PGW-U/UPF向PGW-C/SMF上报业务统计结果。
    - PGW-C/SMF向OCS发出信用控制结束请求CCR-T，汇报用户访问业务的配额使用情况。
    - OCS向PGW-C/SMF返回成功的信用控制应答CCA-T。OCS扣除相应的费用。
