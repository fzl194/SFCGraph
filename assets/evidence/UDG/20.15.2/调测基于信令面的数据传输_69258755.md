# 调测基于信令面的数据传输

- [操作场景](#ZH-CN_OPI_0269258755__1.3.1)
- [必备事项](#ZH-CN_OPI_0269258755__1.3.2)
- [操作步骤](#ZH-CN_OPI_0269258755__1.3.3)

## [操作场景](#ZH-CN_OPI_0269258755)

当运营商部署基于信令面的数据传输功能时，需对该功能进行调测，确保该特性正常生效。

> **说明**
> 适用于SGW-U。

## [必备事项](#ZH-CN_OPI_0269258755)

前提条件

- 请仔细阅读[GWFD-110606 基于信令面的数据传输](../GWFD-110606 基于信令面的数据传输_69258751.md)。
- 完成[激活基于信令面的数据传输](激活基于信令面的数据传输_69258754.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | APN | apncp | 已配置数据中获取 | 取自<br>[激活基于信令面的数据传输](激活基于信令面的数据传输_69258754.md)<br>中配置的APN实例名。 |

工具

- 测试终端
- OM Portal跟踪工具

## [操作步骤](#ZH-CN_OPI_0269258755)

1. 请确认NB-IoT基本功能已开启（详见 [激活NB-IoT终端标准接入](../GWFD-010296 NB-IoT终端标准接入/激活NB-IoT终端标准接入_74275194.md) ），并使用 [**LST LICENSESWITCH**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令查询是否开启了基于信令面的数据传输功能。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5DOVN01";
  ```
    - 如果 “Switch” 为 “ENABLE” ，请执行[步骤 3](#ZH-CN_OPI_0269258755__step2)。
    - 如果 “Switch” 为 “DISABLE” ，则执行 **[SET LICENSESWITCH](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)** 命令打开本特性对应的License配置开关。
2. 在OM Portal上 [创建UDG用户跟踪任务](../../../../../../网络运维/日常维护/UDG基础运维操作/创建消息跟踪/创建用户跟踪/创建UDG用户跟踪任务_88792206.md) ，在“参数配置”栏输入用户IMSI，在“消息类型”栏选择UP DATA和DOWN DATA消息类型。
3. 测试终端使用“apncp”APN接入网络，并进行下行数据业务。激活用户，并跟踪该用户消息。
4. 查看用户跟踪消息，打开SGW-C/PGW-C向SGW-U/PGW-U发送的PFCP Session Establishment Request消息，查看“source-interface-type”信元是否携带“s11-u”，如 [图1](#ZH-CN_OPI_0269258755__fig17978144611103) 所示。
  **图1** PFCP Session Establishment Request

  <br>

  ![](调测基于信令面的数据传输_69258755.assets/zh-cn_image_0276047155.png)
    - 如果“source-interface-type”信元携带“s11-u”，请执行 [步骤 5](#ZH-CN_OPI_0269258755__step41801848138) 。
    - 如果“source-interface-type”信元没有携带“s11-u”，请执行 [步骤 6](#ZH-CN_OPI_0269258755__step204091850132213) 。
5. 查看用户跟踪消息，打开SGW-U/PGW-U向SGW-C/PGW-C响应的PFCP Session Establishment Response消息，查看“local-fteid”信元中是否携带了SGW-U/PGW-U给S11-U接口分配的地址ipv4-address/ipv6-address，以及该地址与使用 **[LST LOGICINF](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/逻辑接口管理/逻辑接口配置/接口/查询逻辑接口（LST LOGICINF）_86526672.md)** 查询的结果是否一致。如图所示，使用 **[LST LOGICINF](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/逻辑接口管理/逻辑接口配置/接口/查询逻辑接口（LST LOGICINF）_86526672.md)** 查询得到的SGW-U/PGW-U上配置的S11-U接口地址10.2.151.15，PFCP Session Establishment Response消息中携带的ipv4-address也为10.2.151.15。
  ```
  LST LOGICINF:;
  ```
  ```
  RETCODE = 0  操作成功。
  逻辑接口信息
  ------------ 
  逻辑接口名称    逻辑接口的IPv4地址1    逻辑接口的IPv4掩码1    ......                          
  s11-uif1/1/0        10.2.151.15          255.255.255.255      ......
                             
  (结果个数 = 1)
  ---    END
  ```
  **图2** PFCP Session Establishment Response

  <br>

  ![](调测基于信令面的数据传输_69258755.assets/zh-cn_image_0276137072.png)
    - 如果“local-fteid”信元中携带了ipv4-address/ipv6-address，并与配置查询结果一致，则调测完成。
    - 如果“local-fteid”信元中没有携带ipv4-address/ipv6-address，或与配置查询结果不一致，请执行 [步骤 6](#ZH-CN_OPI_0269258755__step204091850132213) 。
6. 收集问题信息并寻求技术支持。
    a. 在镜像接口或服务器上开启第三方抓包工具，执行 [步骤 3](#ZH-CN_OPI_0269258755__step2) 并保存报文。
    b. 在相应窗口下执行 [**EXP MML**](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/配置管理/配置导出管理/导出MML文件（EXP MML）_47200033.md) 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 归纳所有信息并联系华为技术支持解决。
