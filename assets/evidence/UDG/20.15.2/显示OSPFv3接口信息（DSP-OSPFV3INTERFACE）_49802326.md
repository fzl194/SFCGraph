#### [命令功能](#ZH-CN_CONCEPT_0000001549802326)

该命令查询OSPFv3接口信息，用于查看接口下的OSPFv3相关配置信息包括接口的邻居状态、Hello包的发送配置、接口的OSPFv3网络类型等。

#### [注意事项](#ZH-CN_CONCEPT_0000001549802326)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001549802326)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001549802326)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROCID | OSPFv3进程号 | 可选必选说明：可选参数<br>参数含义：OSPFv3进程号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～4294967295。<br>默认值：无 |
| AREAID | OSPFv3区域号 | 可选必选说明：可选参数<br>参数含义：OSPFv3区域号。<br>数据来源：本端规划<br>取值范围：IPv4地址类型。点分十进制格式。<br>默认值：无 |
| IFNAME | 接口名称 | 可选必选说明：可选参数<br>参数含义：接口名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。<br>默认值：无<br>配置原则：请使用LST INTERFACE命令查看可用接口。 |

```
DSP OSPFV3INTERFACE: PROCID=1;
```

#### [输出结果说明](#ZH-CN_CONCEPT_0000001549802326)

| 输出项名称 | 输出项解释 |
| --- | --- |
| OSPFv3进程号 | 该参数指定用户创建的OSPFv3进程号。 |
| OSPFv3区域号 | 该参数指定用户创建的OSPFv3区域号。 |
| 接口名称 | 该参数指定用户创建的特定的接口名称。 |
| MTU值 | 该参数指定接口MTU值。 |
| GR状态 | 该参数指定接口GraceRestart状态。 |
| 接口开销值 | 该参数指定接口开销值。 |
| 发送Hello包的Poll时间间隔（s） | 该参数指定NBMA网络上发送轮询Hello报文的时间间隔。在NBMA网络上，当邻居失效后，路由器将按配置的轮询时间间隔定期地发送Hello报文。轮询时间间隔值至少应为Hello报文时间间隔的4倍，OSPF不支持NULL接口配置，单位是秒。 |
| 指定路由器的优先级 | 该参数指定路由器优先级，优先级高的接口在DR选举时被首先考虑。如果一台设备的接口优先级为0，则它不会被选举为DR或BDR。 |
| 接口对LSA的传输延迟时间（s） | 该参数指定接口对LSA的传输延迟时间，LSA在本设备的链路状态数据库（LSDB）中会随时间老化（每秒钟加1），但在网络的传输过程中却不会，所以有必要在发送之前在LSA的老化时间上增加本参数配置的一段时间。此配置对低速率的网络尤其重要。本参数不适用于null接口，单位是秒。 |
| 发送Hello包的时间间隔（s） | 该参数指定接口发送Hello包时间间隔，hello interval的值写入Hello报文中后随之发送。hello interval的值越小，发现网络拓扑改变的速度越快，路由开销也就越大。确定接口和邻接路由器的参数要保持一致，单位是秒。 |
| 抑制接口收发OSPFv3报文 | 该参数指定抑制接口收发OSPFv3报文，如果要使OSPFv3路由信息不被其他网络中的设备获得，并且使本地设备不接收网络中其他设备发布的路由更新信息，可以禁止此接口接收和发送OSPFv3报文。 |
| 邻居重传间隔（s） | 该参数指定邻居重传间隔，当一台设备向它的邻居发送一条LSA后，需要等到对方的确认报文。若在该重传LSA的时间间隔内未收到对方的确认报文，就会重传这条LSA。相邻设备重传LSA时间间隔的值不要配置得太小，否则将会引起不必要的重传。 |
| 使能MTU | 是否使能MTU，DD报文中的MTU值缺省为0，使用本命令可以手工设定指定接口在发送时填写DD报文中的MTU值域，即使用接口的实际MTU值填写。 |
| 超时时间（s） | 该参数指定等待超时时间，在该事件间隔内，若未收到Backup Seen事件，则可以开始进行DR选举。通过合理配置Wait定时器的时间间隔，可以减少网络中DR、BDR的变化频率，减少网络振荡。 |
| 接口网络类型 | 接口网络类型，包含P2P、P2MP、广播、NBMA、P2MP-NONBROADCAST、LOOPBACK，缺省情况下，接口的网络类型根据物理接口而定。以太网接口的网络类型为Broadcast，串口的网络类型为P2P，Frame-relay接口的网络类型为NBMA。 |
| 邻居失效的时间（s） | 邻居失效的时间，在该时间间隔内，若未收到邻居的Hello报文，就认为该邻居已失效。运行OSPF接口上的邻居失效时间dead interval必须大于发送Hello报文的时间间隔hello interval，且同一网段上的设备的dead interval值也必须相同。缺省情况下，邻居失效时间为发送Hello报文时间间隔的4倍。OSPF协议不支持NULL接口的配置。 |
| 备份指定路由器 | 该参数指定邻居关系中的备份指定路由器。 |
| 实例号 | 该参数指定OSPFv3接口所属的实例ID。 |
| 指定路由器 | 该参数指定邻居关系中的指定路由器。 |
| 高链路开销 | 该参数指定高质量链路开销。 |
| 低链路开销 | 该参数指定低质量链路开销，如果链路质量变差，需要增加链路开销值。 |
| 接口LSA数目 | 该参数指定接口LSA数目。 |
| 邻接路由器数目 | 该参数指定邻接路由器数目。 |
| 接口LSA校验和 | 该参数指定接口LSA校验和。 |
| 链路本地地址 | 该参数指定IPv6接口链路本地地址。 |
| 前缀长度 | 该参数指定接口IPv6地址前缀长度。 |
| 连接协议 | 该参数指定接口协议状态。 |
| 邻居数目 | 该参数指定接口上OSPFv3邻居数目。 |
| Hello包定时器 | 该参数指定接口发送Hello包的定时器时长。 |
| DR本地链路地址 | 该参数指定指定路由器的接口本地链路地址。 |
| BDR本地链路地址 | 该参数指定备份指定路由器的接口本地链路地址。 |
| 当前发送报文的高32比特序列号 | 该参数指定当前发送报文的高32比特序列号。 |
| 当前发送报文的低32比特序列号 | 该参数指定当前发送报文的低32比特序列号。 |
| 接口状态 | 该参数指定OSPFv3接口状态。 |
| IGP隧道类型 | 该参数指定IGP隧道类型。 |
| 使能BFD | 该参数指定使能BFD功能。 |
| 阻塞接口动态创建BFD特性 | 该参数指定阻塞接口动态创建BFD特性。 |
| BFD检测乘数间隔 | 该参数指定会话检测倍数，如果BFD会话在配置的检测周期内没有收到对端发来的BFD报文，则认为链路发生了故障，BFD会话的状态将会置为Down。 |
| 最小接收报文时间间隔（ms） | 该参数指定期望从对端接收BFD报文的最小接收间隔，单位是毫秒。 |
| 最小发送报文时间间隔（ms） | 该参数指定向对端发送BFD报文的最小发送间隔，单位是毫秒。 |
| 认证标志 | 该参数指定接口认证标志。 |
| OSPFv3自动扩容功能使能标志 | 该参数指定使能了OSPFv3共网段虚拟系统标志。 |
| OSPFv3虚Router-ID配置标志 | 该参数指定OSPFv3接口虚拟Router-id配置是否生效标志。 |
| OSPFv3配置的虚Router-ID | 该参数指定OSPFv3接口的虚拟Router-id。 |
| OSPFv3生效的虚Router-ID | 该参数指定OSPFv3接口生效的虚拟Router-id。 |
| IPsec SA名称 | 该参数指定接口绑定IPsec SA的名称。 |
| BFD会话与接口链路状态绑定标志 | 该参数指定将BFD会话状态与接口的链路状态进行绑定，配置此参数后，BFD会话状态与此接口链路状态绑定（当BFD会话状态变为Down时，接口的链路状态也会变为Down），从而达到快速感知故障的目的。 |
| 阻止指定OSPFv3接口的FRR能力 | 该参数指定阻止指定OSPFv3接口的FRR能力。 |
| 优雅迁移状态 | 该参数指定OSPFv3接口是否处于优雅迁移状态。 |
