# 调测支持OTT业务规则库功能

- [操作场景](#ZH-CN_OPI_0305695393__1.3.1)
- [必备事项](#ZH-CN_OPI_0305695393__1.3.2)
- [操作步骤](#ZH-CN_OPI_0305695393__1.3.3)

## [操作场景](#ZH-CN_OPI_0305695393)

当运营商部署了OTT业务规则库功能后，需要调测该功能，确保该功能可正常使用。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0305695393)

前提条件

- 请仔细阅读[GWFD-110321 支持内置百万级业务规则库特性概述](GWFD-110321 支持内置百万级业务规则库特性概述_05455408.md)。
- 完成[激活支持OTT业务规则库功能](激活支持OTT业务规则库功能_05455409.md)。

数据

| 类别 | 参数名称 | 取值样例 | 获取方法 | 说明 |
| --- | --- | --- | --- | --- |
| 用户信息查询 | IMSI | 460000123456789 | 测试终端自带 | - |
| 测试终端使用的APN | apn | apn-test | 已配置数据中获取 | 取自<br>[激活支持OTT业务规则库功能](激活支持OTT业务规则库功能_05455409.md)<br>中配置的APN实例名。 |

工具

测试终端

## [操作步骤](#ZH-CN_OPI_0305695393)

1. 执行 [**LST LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/查询License配置项开关（LST LICENSESWITCH）_09587900.md) 命令，查询license中是否允许支持内置百万级业务规则库功能。
  ```
  LST LICENSESWITCH:LICITEM="LKV3G5MSFM01";
  ```
  ```
  ......

  -------------------------
          License项  =  LKV3G5MSFM01
        License名称  =  Supporting Build-in Mega-class Service Flow Matching Database
               开关  =  使能
  (结果个数 = 1)
  ---    END
  ```
    - 如果回显中“开关”为“使能”，请执行[步骤 2](#ZH-CN_OPI_0305695393__step7889143419175)。
    - 如果回显中“开关”为“不使能”，请打开License开关。
2. 在OM Portal上创建N4接口跟踪任务。
3. 测试终端使用 “apn-test” DNN接入网络。
4. 查看N4接口跟踪任务，检查SMF给UPF发送的PFCP Session Modification Request消息中Create URR信元组Reporting Triggers、Volume Threshold、Time Threshold等信元，查询下发的触发条件和配额管理信息。
5. 测试终端访问IP地址为“10.13.4.3”的服务器，服务器信息已经在OTT业务规则库中配置，并且绑定在rule中。业务访问一段时间后，该业务费率的配额用尽，UPF向SMF发送PFCP Session Report Request消息，消息中携带URR ID，请求新的配额下发。查看SMF是否向UPF回复PFCP Session Report Response响应消息中是否携带URR ID对应的Create URR。
    - 如果下发，表示用户可以正常访问业务，调测完成。
    - 如果未下发，请执行[步骤 6](#ZH-CN_OPI_0305695393__step11)。
6. 收集信息并寻求技术支持。
    a. 在OM Portal上创建用户跟踪任务 并保存报文。
    b. 执行 **EXP MML** 命令将当前配置数据导出为MML脚本文件并保存。
    c. 收集并保存上述所有查询信息。
    d. 收集归纳所有信息并联系华为技术支持解决。
