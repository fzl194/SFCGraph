# 激活双故障Bypass

- [操作场景](#ZH-CN_OPI_0000001914147877__1.3.1)
- [必备事项](#ZH-CN_OPI_0000001914147877__1.3.2)
- [操作步骤](#ZH-CN_OPI_0000001914147877__1.3.3)

## [操作场景](#ZH-CN_OPI_0000001914147877)

在 UDG 上激活双故障Bypass功能，降低PCF双故障对用户业务的影响。

> **说明**
> 适用于PGW-U、UPF。

## [必备事项](#ZH-CN_OPI_0000001914147877)

前提条件

- 请仔细阅读[GWFD-112000 双故障Bypass](../GWFD-112000 双故障Bypass_68148336.md)。
- 依赖的特性已完成配置，相关特性参加[与其他特性的交互](GWFD-112000 双故障Bypass特性概述_14187557.md#ZH-CN_CONCEPT_0000001914187557__Dependency)。
- 已完成APN、User Profile、User Profile group创建，以及三者之间的关系绑定。
- 完成 **加载license** （如果有需求，请联系华为技术支持工程师处理），对应的License控制项为 82200HHX LKV3G5DFBP01 双故障Bypass 。

数据

> **说明**
> - 下面数据表格中的URRID、USAGERPTMODE、PCCPOLICYGRPNM、 RULENAME、POLICYTYPE、POLICYNAME、USERPROFILENAME要与SMF保持一致。
> - 语音承载的静态PCC规则中的RG与PCF下发的动态规则中的RG相同。
> - PCF下发的用于创建语音专有QoS Flow/专有承载的规则的优先级高于本地配置的用于创建语音专有QoS Flow/专有承载的规则；本地配置的用于创建语音专有QoS Flow/专有承载的规则优先级高于PCF下发的any to any的动态规则和本地配置的any to any的预定义规则。

## [任务示例](#ZH-CN_OPI_0000001914147877)

任务描述

激活双故障Bypass功能。

任务一：没有配置语音缺省QoS Flow/缺省承载静态PCC策略，在UPF上激活语音PCF故障bypass功能。

任务二：已配置语音缺省QoS Flow/缺省承载静态PCC策略，在当前配置基础上激活语音PCF故障bypass功能。

脚本

- **任务一：没有配置语音缺省QoS Flow/缺省承载静态PCC策略，在UPF上激活语音PCF故障bypass功能。**
    1. 配置语音缺省QoS Flow/缺省承载静态PCC策略。
      //配置流过滤器和三四层过滤条件。

      ```
      ADD FILTER: FILTERNAME="filter_test", L34PROTTYPE=STRING, L34PROTOCOL=ANY;
      ADD FILTERIPV6: FILTERNAME="filter_testV6", L34PROTTYPEV6=STRING, L34PROTOCOL=ANY;
      ADD FLOWFILTER:FLOWFILTERNAME="flowfilter_test";
      ADD FLTBINDFLOWF:FLOWFILTERNAME="flowfilter_test",FILTERNAME="filter_test";
      ADD FLTBINDFLOWF: FLOWFILTERNAME="flowfilter_test", FILTERNAME="filter_testV6";
      ```
      //增加使用量上报规则信息。

      ```
      ADD URR: URRNAME="urr_signaling_offline", URRID=2005, USAGERPTMODE=OFFLINE, OFFMETERINGTYPE=VOLUME, RG=20;
      ADD URR: URRNAME="urr_signaling_online", URRID=2015, USAGERPTMODE=ONLINE, ONLMETERINGTYPE=VOLUME, DFTQUOTAPARA=Byte, RG=25;
      ```
      //增加使用量上报规则组。

      ```
      ADD URRGROUP:URRGROUPNAME="urrg_signaling_01", UPURRNAME1="urr_signaling_offline", DOWNURRNAME1="urr_signaling_offline", UPURRNAME2="urr_signaling_online", DOWNURRNAME2="urr_signaling_online";
      ```
      //配置PCC策略组。
      ```
      ADD PCCPOLICYGRP: PCCPOLICYGRPNM="pccpolicygrp_signaling", FUPSESSIONEXC=DISABLE, TOKENFUNCFLAG=DISABLE, SIGNALASSOCIATE=ENABLE, ADCMUTEFLAG=ENABLE, HTTP2DEGRADESW=INHERIT, URRGROUPNAME="urrg_signaling_01", EVENTCHARGEFLAG=DISABLE, EVENTCHGPOINT=RESPONSE, MMSCHARGETYPE=SINGLE;
      ```
      //配置规则。
      ```
      ADD RULE: RULENAME="rule_ims_signaling", POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="flowfilter_test", PRIORITY=241, POLICYNAME="pccpolicygrp_signaling";
      ```
      //配置用户模板。
      ```
      ADD USERPROFILE: USERPROFILENAME="up_ims", ALIASMARKFLAG=DISABLE, DDOSCHECK=DISABLE, LOCRPTSWITCH=INHERIT;
      ```
      //配置规则与用户模板的绑定关系。
      ```
      ADD RULEBINDING:USERPROFILENAME="up_ims",RULENAME="rule_ims_signaling";
      ```
      //将新配置的Filter和UserProfile置为生效。
      ```
      SET REFRESHSRV:REFRESHTYPE=ALL;
      ```
    2. 配置语音专有QoS Flow/专有承载静态PCC策略。业务触发的专有QoS Flow/专有承载的规则需要与PCF/PCRF下发的动态规则对应的计费策略保持一致。
      //配置流过滤器和三四层过滤条件。SVRIP的取值需和现网SBC-Uip（媒体面IP）地址规划一致。SBC-Uip（媒体面IP）地址有多个时需要配置多条。

      ```
      ADD FILTER: FILTERNAME="filter_bypass", L34PROTTYPE=STRING, L34PROTOCOL=ANY, SVRIPMODE=IP, SVRIP="192.168.10.67", SVRIPMASKTYPE=LENGTHTYPE, SVRIPMASKLEN=32, MSSTARTPORT=0, MSENDPORT=65535, SVRSTARTPORT=0, SVRENDPORT=65535;
      ADD FILTERIPV6:FILTERNAME="filterV6_bypass", L34PROTTYPEV6=STRING, L34PROTOCOL=ANY, SVRIPMODE=IP, MSSTARTPORT=0, MSENDPORT=65535, SVRSTARTPORT=0, SVRENDPORT=65535, SVRIPV6="fc00:8028:5002:10D::181", SVRIPV6MASKTYPE=LENGTHTYPE, SVRIPV6MASKLEN=128; 
      ADD FLOWFILTER: FLOWFILTERNAME="flowfilter_bypass", TETHERDETFLAG=NONE; 
      ADD FLTBINDFLOWF: FLOWFILTERNAME="flowfilter_bypass", FILTERNAME="filter_bypass";
      ADD FLTBINDFLOWF: FLOWFILTERNAME="flowfilter_bypass", FILTERNAME="filterV6_bypass";
      SET REFRESHSRV: REFRESHTYPE=USERPROFILE;
      ```
      //增加使用量上报规则信息。
      ```
      ADD URR: URRNAME="urr_qos_01", URRID=1006, USAGERPTMODE=QOS;
      ADD URR: URRNAME="urr_voice_offline", URRID=1005, USAGERPTMODE=OFFLINE, OFFMETERINGTYPE=VOLUME, RG=10;
      ADD URR: URRNAME="urr_voice_online", URRID=1015, USAGERPTMODE=ONLINE, ONLMETERINGTYPE=VOLUME, DFTQUOTAPARA=Byte, RG=15;
      ```
      //增加使用量上报规则组。

      ```
      ADD URRGROUP:URRGROUPNAME="urrg_voice_01",UPURRNAME1="urr_voice_offline",DOWNURRNAME1="urr_voice_offline",UPURRNAME2="urr_voice_online",DOWNURRNAME2="urr_voice_online";
      ```
      //配置语音专有QoS Flow/专有承载的QoS参数。
      //UPF的URRID必须和SMF的URRID相同。

      ```
      ADD QOSPROP: QOSPROPNAME="qosprop_voice", GBRUPLKVALUE=64, GBRDNLKVALUE=64, MBRUPLKVALUE=64, MBRDNLKVALUE=64, QOSURRNAME="urr_qos_01", DECOUPLINGSW=DISABLE, KEYFLOWDETECTSW=DISABLE;
      ```
      //配置PCC策略组。

      ```
      ADD PCCPOLICYGRP: PCCPOLICYGRPNM="pccpolicygrp_voice", FUPSESSIONEXC=DISABLE, TOKENFUNCFLAG=DISABLE, SIGNALASSOCIATE=ENABLE, ADCMUTEFLAG=ENABLE, QOSPROPNAME="qosprop_voice", HTTP2DEGRADESW=INHERIT, URRGROUPNAME="urrg_voice_01", EVENTCHARGEFLAG=DISABLE, EVENTCHGPOINT=RESPONSE, MMSCHARGETYPE=SINGLE;
      ```
      //配置规则。本地配置静态规则优先级要低于PCF下发的动态规则的优先级。

      ```
      ADD RULE: RULENAME="rule_ims_voice", POLICYTYPE=PCC, FILTERINGMODE=FLOWFILTER, FLOWFILTERNAME="flowfilter_bypass", PRIORITY=242, POLICYNAME="pccpolicygrp_voice";
      ```
      //配置规则与用户模板的绑定关系。

      ```
      ADD RULEBINDING: USERPROFILENAME="up_ims", RULENAME="rule_ims_voice", POLICYTYPE=PCC;
      ```
      //将新配置的Filter和UserProfile置为生效。
      ```
      SET REFRESHSRV: REFRESHTYPE=ALL;
      ```
    3. 开启PCF Bypass功能。
      //开启PCF Bypass功能，并设置流上报模式和迟滞时间。

      ```
      SET IMSBYPASS: IMSBYPASSSW=ENABLE, QOSURRFLOWRPT=FLOW, QOSURRHYSTIMER=0;
      ```
