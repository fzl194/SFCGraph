#### [命令功能](#ZH-CN_CONCEPT_0182837327)

**适用NF：PGW-U、UPF**

该命令用来定义自定义协议类型，可以配置自定义协议所属协议大类，带服务器IP地址、端口范围、或HOST值的过滤器，以及协议识别匹配时的优先级，通过该命令的设置，系统可以根据设置的IP地址、端口或HOST值识别出7层协议类型。

#### [注意事项](#ZH-CN_CONCEPT_0182837327)

- 该命令执行后立即生效。
- 该命令最大记录数为100。
- PROTOCOLDEFINE中引用的FILTERNAME必须是FILTER配置过的过滤器名称。
- 执行本命令时，需要指定自定义协议所属的协议大类，必须为缺省协议列表中的协议大类分组名称，不能自定义。
- 被本命令引用的FILTERNAME不能配置MSIP，MSPORT和TOS，若配置系统会提示配置失败。
- 多个配置间的PRIORITY取值不能相同。
- 如果由于知识库升级或者回退导致自定义协议名称和系统默认的协议和子协议名称相同，系统将对自定义协议名称进行修改，在协议名称前增加“x-”前缀：
    - 如果自定义协议名称增加“x-”前缀后长度超过31，则不进行自动改名，同时上报“ALM-81032 业务过滤规则没有生效”告警，请按照告警指导处理此场景；。
    - 如果自定义协议名称增加“x-”前缀后和已经配置的自定义协议名称冲突，则不进行自动改名，同时上报“ALM-81032 业务过滤规则没有生效”告警，请按照告警指导处理此场景；。
    - 如果自定义协议名称增加“x-”前缀后和系统缺省协议列表中的协议大类/协议/协议子类名称冲突，则不进行自动改名，同时上报“ALM-81032 业务过滤规则没有生效”告警，请按照告警指导处理此场景；。
- 业务流使用ProtocolDefine配置数据得到的协议识别结果，使用方法类似于通过SA特征库识别出的应用层协议，可以用于计费、BWM、ADC(Application Detection Control)等特性。如果同时通过自定义协议和特征库得到两个协议识别结果，由软参BIT1303控制使用哪一个，缺省使用自定义协议的识别结果。
- PROTDEFINENAME不允许命名为"quic"、"Quic"或者"QUIC"。

#### [操作用户权限](#ZH-CN_CONCEPT_0182837327)

G_1，管理员级别命令组；G_2，操作员级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0182837327)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROTDEFINENAME | 自定义协议名称 | 可选必选说明：必选参数<br>参数含义：该参数用于设置自定义协议名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～31。不支持空格，不区分大小写。必须和默认的协议大类/协议/协议子类名称不同。<br>默认值：无<br>配置原则：自定义协议不能与默认协议重名。 |
| PROTGROUPNAME | 协议组名称 | 可选必选说明：必选参数<br>参数含义：该参数用于设置自定义协议分类所属的协议大类分组名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～31。不支持空格，不区分大小写。必须为系统缺省协议列表中的协议大类分组名称，不能自定义。<br>默认值：无<br>配置原则：该协议组只能为默认协议组，不支持自定义协议组。 |
| FILTERNAME | 过滤器名称 | 可选必选说明：必选参数<br>参数含义：该参数用于设置过滤器名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。不支持空格，不区分大小写。数据源为ADD FILTER配置。<br>默认值：无<br>配置原则：该参数使用ADD FILTER命令配置生成。 |
| PRIORITY | 优先级 | 可选必选说明：必选参数<br>参数含义：该参数用于设置协议识别匹配的优先级。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～65535。<br>默认值：无<br>配置原则：优先级数值越小，优先级越高。 |

#### [使用实例](#ZH-CN_CONCEPT_0182837327)

增加名称为protdefine、绑定的协议组名为p2p、绑定的过滤器名为filter的自定义协议：

```
ADD PROTOCOLDEFINE:PROTDEFINENAME="protdefine",PROTGROUPNAME="p2p",FILTERNAME="filter",PRIORITY=10;
```
