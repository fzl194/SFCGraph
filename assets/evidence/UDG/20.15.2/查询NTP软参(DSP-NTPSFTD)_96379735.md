#### [命令功能](#ZH-CN_TOPIC_0000001196379735)

该命令为专家维护命令，用于查询NTP软参值。

该命令仅适用于开局阶段调测或者维护的操作场景，当需要查询NTP软参配置信息时，操作员可以使用该命令实现这一目的。

#### [注意事项](#ZH-CN_TOPIC_0000001196379735)

无。

#### [参数说明](#ZH-CN_TOPIC_0000001196379735)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| NM | NTP功能模块 | 可选必选说明：必选参数。<br>参数含义：用于指定需要开启软参功能的模块号。<br>取值范围：0～65535<br>默认值：无。<br>配置原则：无。 |
| CTRL | 控制字 | 可选必选说明：必选参数。<br>参数含义：用于指定本次操作设置的软参控制字。<br>取值范围：0～65535<br>默认值：无。<br>配置原则：无。 |

#### [使用实例](#ZH-CN_TOPIC_0000001196379735)

1. 查询后台配置的时间同步策略。
  ```
  %%DSP NTPSFTD: NM=4, CTRL=1;%% 
  RETCODE = 0  操作成功
  操作结果如下
  ------------
  手工同步策略  =  priority_get_time_from_fst
  自动同步策略  =  auto_get_time_from_fst
  (结果个数 = 1)

  ---    END
  ```
2. 查询各节点生效的时间同步策略。
  ```
  %%DSP NTPSFTD: NM=4, CTRL=2;%%
  RETCODE = 0  操作成功
  操作结果如下
  ------------
  节点名称      手工同步策略                自动同步策略
  10.0.0.1      priority_get_time_from_fst  auto_get_time_from_fst
  10.0.0.2      priority_get_time_from_fst  auto_get_time_from_fst
  10.0.0.3      priority_get_time_from_fst  auto_get_time_from_fst
  10.0.0.4      priority_get_time_from_fst  auto_get_time_from_fst
  10.0.0.5      priority_get_time_from_fst  auto_get_time_from_fst
  (结果个数 = 5)

  ---    END
  ```

#### [输出结果说明](#ZH-CN_TOPIC_0000001196379735)

该命令执行正常，会返回命令执行成功的提示信息和相关输出项，各输出项含义请参见 [表1](#ZH-CN_TOPIC_0000001196379735__table1649912484420) 输出项解释。

该命令执行异常，会返回对应的错误码。错误码列表如 [表2](#ZH-CN_TOPIC_0000001196379735__table1091619591156) 所示。

*表1 输出项解释*

| **输出项名称** | **输出项解释** |
| --- | --- |
| 节点名称 | 用于返回查询的节点名称。 |
| 手工同步策略 | 用于返回节点当前生效的手动同步策略。<br>- manual_get_time_from_fst：强制从FST同步时间。<br>- priority_get_time_from_fst：优先从FST同步时间。<br>- priority_get_time_from_csp：优先从CSP同步时间。 |
| 自动同步策略 | 用于返回节点当前生效的自动同步策略。<br>- auto_get_time_from_fst：自动从FST同步时间。<br>- auto_get_time_from_csp：自动从CSP同步时间。 |
