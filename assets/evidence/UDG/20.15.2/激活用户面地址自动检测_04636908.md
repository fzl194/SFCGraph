## [操作场景](#ZH-CN_OPI_0204636908)

运营商规划使用用户面分配地址，在进行业务调测、版本升级、地址扩容或有用户业务访问异常投诉时，需要检查 UDG 到PDN/DN侧服务器之间的路由连通性，可激活本特性。

> **说明**
> 适用于PGW-U、UPF。

## [操作步骤](#ZH-CN_OPI_0204636908)

1. 启动PDN/DN侧路由探测功能。
  [**STR PDNROUTETST**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话连通性检测/网络侧连通性检测/PDN侧路由探测/启动PDN侧路由探测（STR PDNROUTETST）_70824402.md)
  > **说明**
  > - 在同一时间，只能执行一个探测命令，如果同时执行多个探测命令，后边执行的探测会失败。
  > - 同一时间只支持一个APN下的探测，未完成当前探测时，不允许再执行命令下发探测任务。
  > - 当出现误操作或者异常情况下路由探测时间过长，可执行[**STP PDNROUTETST**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话连通性检测/网络侧连通性检测/PDN侧路由探测/停止PDN侧路由探测（STP PDNROUTETST）_70824403.md)强制终止路由探测。
  > - 建议使用UDG地址池中的空闲IP地址作为探测源进行探测，如果被探测路由网段中的IP地址都被占用时，执行此命令可能会导致极少数（几个或者几十个）在线用户被去激活，请谨慎使用。
  > - TRACERT探测方式，在没有回全部探测响应的场景下，大概需要450秒左右才会返回探测的结果。
  > - 建议设备空载环境使用。
  > - 全量探测时，探测时长会成倍增加，此时探测任务最多探测5分钟；如果有未探测到的子段可以使用指定section的全量探测。
2. 查询PDN/DN侧路由探测结果。
  [**DSP PDNTSTRESULT**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/会话管理/会话连通性检测/网络侧连通性检测/PDN侧路由探测信息/查询PDN侧路由探测结果（DSP PDNTSTRESULT）_70824405.md)

## [任务示例](#ZH-CN_OPI_0204636908)

任务描述

调测 UDG 设备上用户业务访问是否正常，需要检查 UDG 到PDN/DN侧服务器之间的路由连通性。

脚本

1. 进入“MML命令行- UDG ”窗口。
2. 启动PDN/DN侧路由探测功能。
    - 任务一：以PING探测方式启动PDN/DN侧路由探测功能。
      //全量探测。
      ```
      STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=PING, WHOLEDETECT=ENABLE, SECTIONNUM=0;
      ```
      //非全量探测。

      ```
      STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, SRCIPV4ADDR="10.10.1.1", DSTIPV4ADDR="10.1.1.1", TESTMETHOD=PING;
      ```
    - 任务二：以DNS探测方式启动PDN/DN侧路由探测功能。
      //全量探测。
      ```
      STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=DNS, DOMAINVALUE="www.example.com", WHOLEDETECT=ENABLE, SECTIONNUM=0;
      ```
      //非全量探测。
      ```
      STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=DNS, DOMAINVALUE="www.example.com";
      ```
    - 任务三：以Tracert探测方式启动PDN/DN侧路由探测功能。Tracert探测方式不支持全量探测。
      ```
      STR PDNROUTETST: IPPOOL="testpool", IPVERSION=IPV4, DSTIPV4ADDR="10.1.1.1", TESTMETHOD=TRACERT;
      ```
3. 查询PDN/DN侧路由探测结果。
  //所有的探测结果，包括探测成功的结果和探测失败的结果。

  ```
  DSP PDNTSTRESULT:;
  ```
  //所有探测成功的结果。
  ```
  DSP PDNTSTRESULT:DISPLAYMODE=SUCCESS;
  ```
  //所有探测失败的结果。
  ```
  DSP PDNTSTRESULT: DISPLAYMODE=FAIL;
  ```
