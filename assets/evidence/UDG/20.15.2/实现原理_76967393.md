# 实现原理

**能力协商过程**

PCC用户激活后，PGW-U和RAN侧需要进行能力协商，通知彼此是否均为华为设备。PGW-U收到RAN侧发送的携带RAN侧能力的上行GTP-U报文后，通过如 [图1](#ZH-CN_TOPIC_0000001176967393__fig03) 所示的协商过程决定PGW-U是否将自身的能力通过GTP-U扩展头通知RAN侧。PGW-U同时支持在收到第一个下行数据报文时，主动向RAN侧发起能力协商过程，具体协商过程如 [图2](#ZH-CN_TOPIC_0000001176967393__fig04) 所示。

能力协商完成后， UDG 支持实时检测RAN侧能力的变化情况，便于实时调整 UDG 对数据报文的处理策略。

**GTP-U扩展头信息**

本功能中，RAN将RAN节点能力信息和小区负荷信息填充到上行GTP-U扩展头中，通知PGW-U RAN侧的能力和小区负荷情况，PGW-U将PGW-U能力信息填充到下行GTP-U扩展头中，通知RAN PGW-U的能力，详细的能力协商过程请参见 [图1](#ZH-CN_TOPIC_0000001176967393__fig03) 和 [图2](#ZH-CN_TOPIC_0000001176967393__fig04) 。

*表1 华为GTP-U头扩展包含的子信元类型定义*

| Sub Extension Header Type (Decimal) | Direction | Message Content |
| --- | --- | --- |
| 0 | 上行/下行 | 指示没有更多的子信元。 |
| 1 | 上行 | RAN节点能力信息，包括版本信息（Version）。 |
| 2 | 上行 | UMTS小区负荷信息，包括小区标识（CGI）、会话标识和小区负荷等级（0、1、2、3）。<br>- 0：无效值<br>- 1：正常（Normal）<br>- 2：拥塞（Congestion）<br>- 3：过载（Overload） |
| 3 | 上行 | LTE小区负荷信息，包括小区标识（ECGI）、会话标识和小区负荷等级（0、1、2、3）。<br>- 0：无效值<br>- 1：正常（Normal）<br>- 2：拥塞（Congestion）<br>- 3：过载（Overload） |
| 4 | 上行 | RAN与CN之间的心跳维护消息，只在上行报文中携带。 |
| 6 ～7 | - | Reserved |
| 8 | 下行 | GW能力信息，包括版本信息（Version）。 |
| 9 | 下行 | 业务优先级标识FPI。 |
| 10～15 | - | Reserved |
