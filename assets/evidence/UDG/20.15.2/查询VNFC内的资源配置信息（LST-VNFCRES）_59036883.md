#### [命令功能](#ZH-CN_CONCEPT_0259036883)

该命令用于查询VNFC内的资源配置信息。通过该命令可以查看各VNFC下部署的资源信息。

#### [注意事项](#ZH-CN_CONCEPT_0259036883)

无。

#### [操作用户权限](#ZH-CN_CONCEPT_0259036883)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0259036883)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| VNFCNAME | VNFC名称 | 可选必选说明：可选参数<br>参数含义：该参数用于表示资源所在的VNFC名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～31。不支持空格，区分大小写。<br>默认值：无 |

#### [输出结果说明](#ZH-CN_CONCEPT_0259036883)

| 输出项名称 | 输出项解释 |
| --- | --- |
| VNFC名称 | 用于表示资源所在的VNFC名称。 |
| 逻辑资源编号 | 用于表示逻辑资源编号，该编号由网元负责分配。 |
| 资源名称 | 用于表示资源名称，指节点或容器的名称。 |
| VNFM分配的资源编号 | 用于表示资源编号，该编号由VNFM层负责分配。 |
| VIM分配的资源编号 | 用于表示资源编号，该编号由VIM层负责分配。 |
| 资源实例类型 | 用于表示资源实例类型。 |
| HA组 | 用于表示资源的HA组。 在同一HA组的资源需要部署在不同的物理机上。显示为NULL时表示没有指定HA组，不受该限制。 OMU天然具有反亲和属性，需要部署在不同的物理机上。 |
| 亲和性组 | 用于表示资源的亲和性组。 如果两个资源在同一个亲和性组里，则这两个资源会尽量部署在同一个物理上。显示为NULL表示没有指定亲和性组，资源部署时不受该限制。 |
| 扩展组 | 用于表示资源的扩展组。 每个扩展组会独立判断Scale out/in（扩容/减容）的条件是否满足。显示为NULL时表示没有指定扩展组。 |
| 主机位置 | 用于表示资源的主机位置。为NULL时表示物理机未知。 |
| 主机名称 | 用于表示资源的主机名称。为NULL时表示物理机未知。 |
| 资源类型 | 用于表示资源类型。<br>- 虚拟机：资源类型为虚拟机。<br>- 容器：资源类型为容器。说明：Full-stack裸机场景下，资源类型为<br>**容器**<br>。 |
| 父资源名称 | 用于表示父资源名称，即资源所在Pod的名称。 |
| 虚拟节点模式 | 用于表示虚拟节点模式。<br>- online：在线模式，正常部署业务。<br>- offline：离线模式，不部署业务。VNFP和VNFC上都存在VNODE实例配置。<br>- offline-nps：离线模式，不部署业务。VNFP存在VNODE实例配置，VNFC上不存在VNODE实例配置。 |
