# 实现原理

关联URL核减查表流程如 [图1](#ZH-CN_TOPIC_0183120115__fig16634640476) 所示。简单概括该流程，即，当 UDG 使能了关联URL核减功能，在报文进行七层规则匹配过程中，会首先使用Referer URL进行七层规则匹配，如果匹配失败，才会使用Request URL再进行匹配。之后，使用命中的规则定义的策略进行计费和控制动作。

> **说明**
> Request URL是用户访问页面的请求URL。Referer URL是Referer字段携带的URL，即链接主页面的URL。

![](实现原理_83120115.assets/zh-cn_image_0204838854.png)

1. 用户报文进入UDG，开始进行解析，包括三四层和七层。
2. 根据协议识别结果，判断是否是HTTP/WAP2.0报文。
    - 如果是，则继续执行[3](#ZH-CN_TOPIC_0183120115__li3)。
    - 如果不是，则执行其他协议类型匹配过程，处理流程同[GWFD-110101 SA-Basic](../../业务感知功能/GWFD-110101 SA-Basic_24082506.md)一致。
3. 判断UDG上是否使能Referer关联开关，可通过命令[**SET LICENSESWITCH**](../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)设置**SWITCH**参数。
    - 如果是，则继续执行[4](#ZH-CN_TOPIC_0183120115__li4)。
    - 如果不是，则使用七层解析结果中的Request URL进行规则匹配。
4. 判断HTTP报文中是否携带Referer字段。
    - 如果是，则使用Referer URL进行七层规则匹配，然后执行[5](#ZH-CN_TOPIC_0183120115__li5)。
    - 如果不是，说明没有链接主页面，则使用Request URL进行规则匹配。
5. 判断Referer URL七层规则匹配是否成功。
  > **说明**
  > - Referer URL七层规则匹配的范围是使能了URL关联开关的[**ADD L7FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/七层规则管理/七层过滤器/增加七层过滤器（ADD L7FILTER）_82837397.md)组，可通过参数“ISREFEREREN”进行设置，详细描述可参见[**ADD L7FILTER**](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务过滤器管理/七层规则管理/七层过滤器/增加七层过滤器（ADD L7FILTER）_82837397.md)的命令帮助。
  > - Request URL七层规则匹配的范围是该用户所属UserProfile下的所有L7filter，无论是否使能URL关联开关。
    - 如果是，则使用匹配上的七层规则进行计费和策略控制。
    - 如果不是，则使用Request URL进行规则匹配。
