## [操作场景](#ZH-CN_OPI_0269972724)

本操作指导介绍在运行网络中激活Non-IP数据传输特性的操作过程。当Non-IP用户接入网络时，需要在 UDG 上开启Non-IP功能开关，并且为Non-IP用户配置目的服务器的地址。

> **说明**
> 适用于SGW-U、PGW-U。

## [必备事项](#ZH-CN_OPI_0269972724)

前提条件

- 请仔细阅读 [GWFD-110607 Non-IP数据传输](../GWFD-110607 Non-IP数据传输_69972721.md) 。
- 已开启NB-IoT基本接入功能（详见[激活NB-IoT终端标准接入](../GWFD-010296 NB-IoT终端标准接入/激活NB-IoT终端标准接入_74275194.md)）。
- 完成 [激活基于信令面的数据传输](../GWFD-110606 基于信令面的数据传输/激活基于信令面的数据传输_69258754.md) 。
- 完成加载License（本特性完整的功能需要同时在用户面和控制面开启License，如果有需求，请联系华为技术支持工程师处理）。

## [操作步骤](#ZH-CN_OPI_0269972724)

1. 打开本特性的License配置开关。
  **[SET LICENSESWITCH](../../../../../../OM参考/命令/UDG MML命令/平台服务管理/操作维护/License管理/设置License项的开关（SET LICENSESWITCH）_09587387.md)**
2. 配置Non-IP数据传输所使用的APN。
  **[ADD APN](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/APN管理/APN/添加APN配置（ADD APN）_82837014.md)**
3. 使能指定APN下的Non-IP功能开关。
  **[SET APNNONIPFUNC](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/M2M管理/APN的Non-IP配置/设置APN Non-IP配置（SET APNNONIPFUNC）_15595100.md)**
  > **说明**
  > - 本特性分成全局、非全局两种场景，当使用**[SET APNNONIPFUNC](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/M2M管理/APN的Non-IP配置/设置APN Non-IP配置（SET APNNONIPFUNC）_15595100.md)**配置之后，全局场景转变为非全局场景。
  >     - 全局场景下，不打开/打开License，全局默认关闭/开启该Non-IP功能。
  >     - 非全局场景下，打开License并使用**[SET APNNONIPFUNC](../../../../../../OM参考/命令/UDG MML命令/用户面服务管理/DN管理/M2M管理/APN的Non-IP配置/设置APN Non-IP配置（SET APNNONIPFUNC）_15595100.md)**命令配置指定APN的Non-IP功能之后，全局开启Non-IP功能转变为仅对特定的APN开启：NONIPSWITCH参数取值为**“ENABLE”**的APN才具有Non-IP功能。必须先打开License该命令配置才能生效。
  > - 需要在控制面上配置相同APN并使能Non-IP功能开关。
  > - 当UDG作为PGW-U时必须配置APN下的Non-IP功能开关。

## [任务示例](#ZH-CN_OPI_0269972724)

任务描述

一些IoT业务，发送的数据报文频率低且报文字节小，可以让这些业务使用Non-IP进行数据传输，因此需要对Non-IP数据传输功能进行配置。本实例中，仅名称为apn_nonip的APN具有Non-IP传输功能，其他APN则不具有Non-IP传输功能。

脚本

// 打开本特性的License配置开关。

```
SET LICENSESWITCH:LICITEM ="LKV3G5NOIP01", SWITCH=ENABLE;
```

//配置非全局场景下，Non-IP数据传输使用的APN。

```
ADD APN: APN="apnnonip";
```

//使能指定APN下的Non-IP功能开关。

```
SET APNNONIPFUNC:APN="apnnonip",NONIPSWITCH=ENABLE,USERPORT=1010;
```
