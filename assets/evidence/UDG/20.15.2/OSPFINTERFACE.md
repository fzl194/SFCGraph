#### [命令功能](#ZH-CN_CONCEPT_0000001600601065)

该命令显示OSPF接口信息，用于显示接口下的OSPF相关配置信息包括接口的邻居状态、Hello包的发送配置、接口的OSPF网络类型等。

#### [注意事项](#ZH-CN_CONCEPT_0000001600601065)

这条命令显示的是动态数据。

#### [操作用户权限](#ZH-CN_CONCEPT_0000001600601065)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

#### [参数说明](#ZH-CN_CONCEPT_0000001600601065)

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PROCID | OSPF进程号 | 可选必选说明：可选参数<br>参数含义：OSPF进程号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～4294967295。<br>默认值：无 |
| AREAID | 区域号 | 可选必选说明：可选参数<br>参数含义：OSPF区域号。<br>数据来源：对端协商<br>取值范围：IPv4地址类型。点分十进制格式。<br>默认值：无 |
| IFNAME | 接口名 | 可选必选说明：可选参数<br>参数含义：指定特定的接口名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。<br>默认值：无<br>配置原则：请使用LST INTERFACE命令查看可用接口。 |

#### [输出结果说明](#ZH-CN_CONCEPT_0000001600601065)

| 输出项名称 | 输出项解释 |
| --- | --- |
| OSPF进程号 | 该参数指定用户创建的OSPF进程号。 |
| 区域号 | 该参数指定用户创建的OSPF区域号。 |
| 接口名 | 该参数指定用户创建的特定的接口名称。 |
| 接口状态 | 该参数指定OSPF接口状态，根据OSPF接口状态机确定的当前接口状态，接口状态有Down、Waiting、P2P、DR、BDR、DROTHER、Loopback。 |
| 接口MTU值 | 该参数指定接口MTU值。 |
| 接口IP地址 | 该参数指定接口IP地址。 |
| GR状态 | 该参数指定接口GraceRestart状态。 |
| 接口开销值 | 该参数指定接口开销值。 |
| 认证类型 | 该参数指定接口的认证类型，分为None：None；Keychain：Keychain认证模式；HMAC-SHA256：HMAC-SHA256密文验证模式；Null：Null验证模式；Simple：简单验证模式；MD5：MD5密文验证模式；HMAC-MD5：HMAC-MD5密文验证模式。 |
| 使能Smart-discover | 该参数指定在接口上是否使能Smart-discover功能。若使能，设备的邻居状态或者多址网络（广播型或NBMA）上的DR、BDR发生变化时，不必等到Hello定时器到时，就立刻主动的向邻居发送Hello报文。 |
| 发送BFD包的最小时间间隔（ms） | 该参数指定向对端发送BFD报文的最小发送间隔，单位是毫秒。 |
| 发送Hello包的Poll时间间隔（s） | 该参数指定NBMA网络上发送轮询Hello报文的时间间隔。在NBMA网络上，当邻居失效后，路由器将按配置的轮询时间间隔定期地发送Hello报文。轮询时间间隔值至少应为Hello报文时间间隔的4倍，OSPF不支持NULL接口配置，单位是秒。 |
| 指定路由器优先级 | 该参数指定路由器优先级，优先级高的接口在DR选举时被首先考虑。如果一台设备的接口优先级为0，则它不会被选举为DR或BDR。 |
| 本地检测倍数 | 该参数指定会话检测倍数，如果BFD会话在配置的检测周期内没有收到对端发来的BFD报文，则认为链路发生了故障，BFD会话的状态将会置为Down。 |
| 使能BFD | 该参数指定使能BFD功能。 |
| P2MP网络上忽略对网络掩码的检查 | 该参数指定在P2MP网络上忽略对网络掩码的检查。在P2MP网络上，当设备的掩码长度不一致时，使用此命令忽略对Hello报文中网络掩码的检查，从而可以正常建立OSPF邻居关系。 |
| 接口对LSA的传输延迟时间（s） | 该参数指定接口对LSA的传输延迟时间，LSA在本设备的链路状态数据库（LSDB）中会随时间老化（每秒钟加1），但在网络的传输过程中却不会，所以有必要在发送之前在LSA的老化时间上增加本参数配置的一段时间。此配置对低速率的网络尤其重要。本参数不适用于null接口，单位是秒。 |
| 接收BFD包的最小时间间隔（ms） | 该参数指定期望从对端接收BFD报文的最小接收间隔，单位是毫秒。 |
| Hello报文间隔（s） | 该参数指定接口发送Hello包时间间隔，hello interval的值写入Hello报文中后随之发送。hello interval的值越小，发现网络拓扑改变的速度越快，路由开销也就越大。确定接口和邻接路由器的参数要保持一致，单位是秒。 |
| 阻塞接口动态创建BFD特性 | 该参数指定阻塞接口动态创建BFD特性。 |
| 按链路创建BFD会话标志 | 该参数指定按链路创建BFD会话。 |
| 使能BFD单臂Echo标志 | 该参数指定使能BFD单臂Echo标志。 |
| 抑制接口收发OSPF报文 | 该参数指定抑制接口收发OSPF报文，如果要使OSPF路由信息不被其他网络中的设备获得，并且使本地设备不接收网络中其他设备发布的路由更新信息，可使用此功能禁止此接口接收和发送OSPF报文。 |
| 认证密码 | 该参数指定接口认证的密码。 |
| 接口重传LSA的时间间隔（s） | 该参数指定接口重传LSA的时间间隔。当一台路由器向它的邻居发送一条＂链路状态广播＂（LSA）后，需要等到对方的确认报文。若在该重传LSA的时间间隔内未收到对方的确认报文，就会重传这条LSA。相邻路由器重传LSA时间间隔的值不要配置得太小，否则将会引起不必要的重传，本参数不适用于null接口，单位是秒。 |
| 发送DD报文时填MTU值 | 该参数指定在发送DD报文时填MTU值，DD报文中的MTU值缺省为0，使用本参数可以手工设定指定接口在发送时填写DD报文中的MTU值域，即使用接口的实际MTU值填写。 |
| 等待时间（s） | 该参数指定等待超时时间，在该事件间隔内，若未收到Backup Seen事件，则可以开始进行DR选举。通过合理配置Wait定时器的时间间隔，可以减少网络中DR、BDR的变化频率，减少网络振荡。 |
| 接口网络类型 | 网络类型，包含P2P、P2MP、广播、NBMA，缺省情况下，接口的网络类型根据物理接口而定。以太网接口的网络类型为Broadcast，Frame-relay接口的网络类型为NBMA。 |
| 验证字标识符 | 该参数指定接口认证的验证字标识符。 |
| 邻居失效的时间（s） | 该参数指定邻居的失效时间，在该时间间隔内，若未收到邻居的Hello报文，就认为该邻居已失效。运行OSPF接口上的邻居失效时间dead interval必须大于发送Hello报文的时间间隔hello interval，且同一网段上的设备的dead interval值也必须相同。本参数不适用于null接口，单位是秒。 |
| MD5/HMAC-MD5/HMAC-SHA256 密码 | 该参数指定接口MD5或HMAC-MD5或HMAC-SHA256认证的密码。 |
| 接口不透明的ID | 该参数指定接口扩展能力的ID。 |
| 接口之前状态 | 该参数指定上一个接口状态。 |
| 可达性抑制 | 该参数指定使能接口的可达性抑制。 |
| 去使能可达性抑制 | 该参数指定去使能接口的可达性抑制。 |
| 禁止自动使能MPLS LDP功能标志位 | 该参数指定个禁止OSPF接口自动使能MPLS LDP功能标志位。 |
| 过滤所有类型LSA | 该参数指定为所有类型LSA进行过滤。 |
| 过滤7类LSA | 该参数指定过滤7类LSA。 |
| 7类LSA过滤规则 | 该参数指定7类LSA过滤规则类型。 |
| 7类LSA过滤名称 | 该参数指定7类LSA过滤规则名称。 |
| 过滤3类LSA | 该参数指定过滤3类LSA。 |
| 3类LSA过滤规则 | 该参数指定3类LSA过滤规则类型。 |
| 3类LSA过滤名称 | 该参数指定3类LSA过滤规则名称。 |
| 过滤5类LSA | 该参数指定过滤5类LSA。 |
| 5类LSA过滤规则 | 该参数指定5类LSA过滤规则类型。 |
| 5类LSA过滤名称 | 该参数指定5类LSA过滤规则名称。 |
| 低链路开销 | 该参数指定低质量链路开销，如果链路质量变差，需要增加链路开销值。 |
| 使能LDP和OSPF同步功能 | 该参数指定使能LDP和OSPF同步功能。 |
| 阻止接口上运行LDP和OSPF同步功能 | 该参数指定阻止接口上运行LDP和OSPF同步功能。 |
| 保持最大开销值标志 | 该参数指定本地设备通告本接口最大开销值的时间。 |
| 保持最大开销值时间间隔（s） | 该参数指定OSPF在本地设备的中保持通告最大开销值的时间，单位是秒。 |
| 永久保持最大开销值 | 该参数指定在LDP会话重新建立之前，OSPF在本地设备中永久通告最大开销值。 |
| 抑制邻居时间间隔（s） | 该参数指定接口不建立OSPF邻居而等待LDP会话建立的时间间隔，单位是秒。 |
| OSPF共网段虚拟系统使能标志 | 该参数指定使能了OSPF共网段虚拟系统标志。 |

| BFD会话与接口链路状态绑定标志 | 该参数指定将BFD会话状态与接口的链路状态进行绑定，BFD会话状态与此接口链路状态绑定（当BFD会话状态变为Down时，接口的链路状态也会变为Down），从而达到快速感知故障的目的。 |
| 阻止指定OSPF接口的FRR能力 | 该参数指定阻止指定OSPF接口的FRR能力。 |
| 开销值来源 | 该参数指定接口开销值类型。 |
| 优雅迁移状态 | 该参数指定OSPF接口是否处于优雅迁移状态。 |
