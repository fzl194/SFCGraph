## [命令功能](#ZH-CN_MMLREF_0000001728195461)

该命令用于查询在复杂故障场景下自愈功能控制参数。

> **说明**
> 无

#### [操作用户权限](#ZH-CN_MMLREF_0000001728195461)

G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## [参数说明](#ZH-CN_MMLREF_0000001728195461)

无

## [使用实例](#ZH-CN_MMLREF_0000001728195461)

显示在复杂故障场景下自愈功能控制参数。

```
%%LST CFSHEALCTRL:;%%
RETCODE = 0  操作成功

结果如下
--------
         通信亚健康隔离功能开关  =  关闭
       亚健康诊断等待时间(分钟)  =  10
         Q922连续诊断周期（次）  =  3
网元级KPI异常检测与容灾功能开关  =  关闭
         容灾决策等待时间（秒）  =  300
                   容灾执行策略  =  手动
         单主机异常自愈功能开关  =  开启
     诊断隔离处理等待时间（秒）  =  120
                  异常阈值（%）  =  90
                      差值（%）  =  20
(结果个数 = 1)

---    END
```

## [输出结果说明](#ZH-CN_MMLREF_0000001728195461)

| 输出项名称 | 输出项解释 |
| --- | --- |
| 通信亚健康隔离功能开关 | 该参数用于控制在出现通信亚健康故障后是否诊断隔离。 |
| 亚健康诊断等待时间(分钟) | 该参数用于配置通信亚健康延迟诊断时间。 |
| Q922连续诊断周期（次） | 该参数用于指定Q922诊断周期N，目的是防止网络丢包震荡。如果连续N次查询都有“ALM-1011 控制平面丢包错包”告警，每次查询告警的定位信息中有相同的目的Pod，则以第N次查询结果作为诊断输入。 |
| 网元级KPI异常检测与容灾功能开关 | 在网元出现亚健康（如多个主机出现fabric平面亚健康/cpu过载/pod持续故障等）故障而导致业务KPI下降达到阈值时，该参数用于控制网元级KPI异常检测与容灾功能开关。 |
| 容灾决策等待时间（秒） | 该参数用于配置容灾决策的等待时间。 |
| 容灾执行策略 | 该参数用于配置容灾执行处理的策略。 |
| 单主机异常自愈功能开关 | 用于控制由于网元内单主机出现Fabric平面丢包/CPU过载/流控故障而导致KPI下降满足阈值时做对应自愈隔离处理的功能开关。 |
| 诊断隔离处理等待时间（秒） | 用于配置延后诊断隔离处理的等待时间。 |
| 异常阈值（%） | 用于配置POD的CPU使用率异常阈值，大于等于此值初步判断为异常。 |
| 差值（%） | 用于配置与同类型POD实例的CPU使用率平均值的差值，若某个POD的CPU使用率大于等于同类型POD实例的CPU使用率平均值与差值之和，则初步判断为异常。 |
