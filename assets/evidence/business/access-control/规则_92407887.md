#### [定义](#ZH-CN_TOPIC_0292407887)

规则主要由条件和动作组成，用于决定满足哪些条件的报文，执行哪些动作。规则中包含了如下参数：

- 规则名称（Rule Name）
  决定了规则的名字，在策略绑定规则、规则绑定User Profile、查询/修改/删除规则时会使用到。
- 策略类型（Policytype）
  为了将业务感知应用不同的使用场景，将这些场景归纳为不同的策略类型。如 [表1](#ZH-CN_TOPIC_0292407887__table8615351134513) 所示，目前 UDG 已经支持的策略类型有：
  *表1 策略类型表*

  | 策略类型 | 含义 | 相关特性 |
  | --- | --- | --- |
  | **PCC** | 计费与策略控制，代表该规则可以配置PCC策略，用于实现计费和策略控制功能。 | [GWFD-020351 PCC基本功能](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-020351 PCC基本功能_42369272.md)<br>、<br>[GWFD-020357 增强的ADC基本功能](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-020357 增强的ADC基本功能_84866817.md) |
  | **BWM** | 带宽管理，代表该规则可以配置带宽管理分类策略，其分类结果可以用于带宽管理的策略匹配功能。 | [GWFD-110311 基于业务感知的带宽控制](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-110311 基于业务感知的带宽控制_77219468.md) |
  | **HEADEN** | 头增强，代表该规则可以配置头增强策略，可以基于配置的头增强策略向HTTP/RTSP请求报文头域中插入字段。 | [GWFD-110261 HTTP头增强](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-110261 HTTP头增强_62736942.md)<br>、<br>[GWFD-110262 RTSP头增强](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-110262 RTSP头增强_62388776.md)<br>、<br>[GWFD-110263 HTTPS头增强](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-110263 HTTPS头增强_67112727.md) |
  | **WEBPROXY** | Web Proxy，代表该规则可以配置WebProxy的IPFarm对象名称，用于设置WebProxy选择的服务器地址池。 | [GWFD-110282 Web Proxy](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-110282 Web Proxy_66666839.md) |
  | **SMARTREDIRECT** | 智能重定向，代表该规则可以配置CaptivePortal业务对应的IPFarm的名称，HTTP智能重定向的名称或者DNS重写动作的名称。 | [GWFD-110284 HTTP智能重定向](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-110284 HTTP智能重定向_67075034.md) |
  | **ADC** | 代表该规则用于应用检测与控制功能。 | [GWFD-020357 增强的ADC基本功能](../../../../特性指南/UDG特性指南/智能策略控制功能/GWFD-020357 增强的ADC基本功能_84866817.md) |
  | **WORKER** | 通用策略，通过配置参数，定义具体的策略功能。通用策略当前支持的功能包含：to、nat和tethering_block。 | [UDG vTCP_OPT功能专题](../../../UDG vTCP_OPT功能专题/相关特性_51051778.md)<br>、<br>[UDG NAT功能专题](../../../UDG NAT功能专题/相关特性_60796403.md) |
- 策略名称（Policy）
  通过策略名称，将已经完成配置的策略绑定到规则上。策略名称随策略类型的变化而变化，如对于PCC策略，策略名称是 **[ADD PCCPOLICYGRP](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/PCC控制策略/PCC策略组/增加PCC策略组（ADD PCCPOLICYGRP）_82837606.md)** 配置的PCC策略组名称；对于BWM策略，策略名称是 **[ADD CATEGORYPROP](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/带宽控制/分类属性/增加分类属性（ADD CATEGORYPROP）_82837504.md)** 配置的分类属性名称；对于HEADEN策略，策略名称是 **[ADD HEADEN](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务控制策略/头增强控制/头增强/增加头增强（ADD HEADEN）_82837516.md)** 配置的头增强名称。
- 优先级（Priority）
  当用户报文命中了多条规则时，相同策略类型的规则中，优先级最高的生效。详见 [规则匹配流程](../业务感知过程/规则匹配流程_92407894.md) 。
- 流过滤器（组）（Flow Filter/Flow Filter Group）
  流过滤器上配置了不同类型的过滤器组合、协议组合，决定了规则采用的具体过滤条件。流过滤器既可直接绑定在规则上，也可先绑定成流过滤器组，再绑定到规则上。详见 [过滤条件](过滤条件_92407888.md) 。
- 其他参数
  与业务感知相关性不强的参数。比如规则生效的时间段、RADIUS消息触发标识和默认动作、重定向地址、Remark配置类型等。

更多关于规则配置的内容，详见 **[ADD RULE](../../../../../OM参考/命令/UDG MML命令/用户面服务管理/业务匹配策略/业务匹配公共配置/业务规则管理/规则/增加规则（ADD RULE）_82837266.md)** 。
