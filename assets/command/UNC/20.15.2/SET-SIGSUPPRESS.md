# SET SIGSUPPRESS（设置S11接口信令风暴抑制功能参数）

## 功能

**适用网元：MME**

此命令用于配置S11接口故障场景下的信令风暴抑制功能。当存在因S11接口闪断或中断导致大量用户集中分离的风险时，可以开启该功能来平滑异常流程（例如Intra Handover/X2 Path Switch without S-GW Change、Service Request、Intra TAU/Combined TAU without S-GW Change流程的Modify Bearer Request处理超时，或者Intra Handover/X2 Path Switch、Intra TAU/Combined TAU流程进行上游网元选择失败）导致的用户集中分离过程，达到降低S11接口信令风暴风险目的。

## 注意事项

- 此命令执行后立即生效。
- 开启此功能后，当S11接口闪断或中断导致的异常流程中用户被抑制而没有被立即分离时，用户在“抑制时长（分）”内的主被叫业务可能会受到影响。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| SIGSUPPSWITCH | S11接口信令抑制开关 | 可选必选说明：可选参数<br>参数含义：该参数用于设置S11接口信令风暴抑制功能是否开启。<br>数据来源：整网规划<br>取值范围：<br>- “OFF(关闭)”<br>- “ON(开启)”<br>系统初始设置值：<br>“OFF(关闭)”<br>配置原则：当存在因S11接口闪断或中断导致大量用户集中分离的风险时，可以通过开启S11接口信令风暴抑制功能来平滑处理用户集中分离过程，降低信令风暴发生的概率。<br>说明：当该参数从<br>“OFF(关闭)”<br>修改为<br>“ON(开启)”<br>时，<br>[**DSP SIGSUPPRESS**](显示S11接口信令风暴抑制功能的控制与统计信息(DSP SIGSUPPRESS)_72225847.md)<br>命令统计信息中的<br>“流程未抑制的次数”<br>、<br>“流程被抑制的次数”<br>、<br>“抑制用户被分离的次数”<br>和<br>“抑制用户被恢复的次数”<br>会清零。 |
| SIGSUPPCFGPLCY | 信令抑制配置策略 | 可选必选说明：条件可选参数<br>参数含义：该参数用于设置S11接口信令风暴抑制功能的<br>“抑制用户数控制参考值”<br>和<br>“抑制时长（分）”<br>的获取策略。<br>前提条件：该参数在<br>“S11接口信令抑制开关”<br>参数配置为<br>“ON(开启)”<br>后生效。<br>数据来源：整网规划<br>取值范围：<br>- “DEFAULT(系统缺省值)”：表示在S11接口信令风暴抑制功能启控下，UNC以每SPP进程45次/秒的速率将对抑制用户进行分离。<br>- “SPECIFIC(自定义值)”：表示通过指定“抑制用户数控制参考值”和“抑制时长（分）”参数来设置抑制分离速率，设置方法请参考“抑制用户数控制参考值”和“抑制时长（分）”参数描述。<br>系统初始设置值：<br>“DEFAULT(系统缺省值)”<br>配置原则：<br>- “DEFAULT(系统缺省值)”：当不明确周边网元的处理能力时，建议选择此配置值。<br>- “SPECIFIC(自定义值)”：当周边网元的处理能力不足或者过高导致系统缺省值不合理时，通过选择此配置值设置“抑制用户数控制参考值”和“抑制时长（分）”值。 |
| SUPPUSRNUM | 抑制用户数控制参考值 | 可选必选说明：条件必选参数<br>参数含义：该参数用于设置S11接口信令风暴抑制功能全系统的<br>“抑制用户数控制参考值”<br>。此参数是指S11接口闪断或故障可能影响的运营商规划系统接入的4G用户数，系统将通过<br>“抑制用户数控制参考值”<br>/<br>“抑制时长（分）”<br>来计算抑制用户的分离速率。<br>前提条件：该参数在<br>“信令抑制配置策略”<br>参数配置为<br>“SPECIFIC(自定义值)”<br>后生效。<br>数据来源：整网规划<br>取值范围：200～20000000<br>系统初始设置值：500000<br>配置原则：<br>UNC<br>系统预计规划的4G用户数。<br>说明：- 当此阈值配置的值比系统实际的4G用户数小时，系统将自动按实际用户数和“抑制时长（分）”确定被抑制用户的分离速率。<br>- 当此阈值配置比系统实际的4G用户数大时，系统将按此阈值配置的值和“抑制时长（分）”确定被抑制用户的分离速率。 |
| SUPPTIME | 抑制时长（分） | 可选必选说明：条件必选参数<br>参数含义：该参数用于设置S11接口信令风暴抑制功能全系统的<br>“抑制时长（分）”<br>数值。此参数是指当S11接口闪断或故障后，系统将在<br>“抑制时长（分）”<br>内分离完被抑制的用户。同时系统将基于<br>“抑制用户数控制参考值”<br>/<br>“抑制时长（分）”<br>计算被抑制用户的分离速率。<br>前提条件：该参数在<br>“信令抑制配置策略”<br>参数配置为<br>“SPECIFIC(自定义值)”<br>后生效。<br>数据来源：整网规划<br>取值范围：5～1440<br>系统初始设置值：15<br>配置原则：如果<br>“抑制时长（分）”<br>配置过小，将导致S11接口故障时，<br>UNC<br>对抑制用户的分离速率过快而触发周边网元拥塞；如果<br>“抑制时长（分）”<br>配置过大，对被抑制用户的主被叫业务影响较大。<br>说明：- 当用户因S11接口故障而被抑制后，用户被抑制的最大时长为“抑制时长（分）”设置的值；在被抑制期间，用户的主被叫业务可能会受到影响。<br>- 当本参数重新设置后，系统分离用户的时长从设置时刻开始重新计算。 |

## 使用实例

运营商在系统中规划有50万4G用户数，希望在S11接口闪断或中断时，信令风暴得到平滑处理，并且希望这些用户的主被叫业务受影响时长不超过15分钟，则设置 “S11接口信令抑制开关” 参数为 “ON(开启)” 、 “信令抑制配置策略” 参数为 “SPECIFIC(自定义值)” ， “抑制用户数控制参考值” 参数为 “500000” ， “抑制时长（分）” 参数为 “15” 。配置格式如下：

SET SIGSUPPRESS: SIGSUPPSWITCH=ON, SIGSUPPCFGPLCY=SPECIFIC, SUPPUSRNUM=500000, SUPPTIME=15;
