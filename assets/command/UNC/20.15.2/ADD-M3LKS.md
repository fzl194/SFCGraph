# ADD M3LKS（增加M3UA信令链路集）

## 功能

**适用网元：SGSN、MME、SMSF**

该命令用于增加M3UA信令链路集。

## 注意事项

- 该命令执行后立即生效。
- 本表最大记录数为1280。
- 每个本地实体与相邻的目的实体之间只能建立一个链路集。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| LSX | 链路集索引 | 可选必选说明：必选参数<br>参数含义：该参数用于唯一标识一个链路集的索引。<br>数据来源：本端规划<br>取值范围：0~1279<br>默认值：无 |
| DEX | 目的实体索引 | 可选必选说明：必选参数<br>参数含义：该参数用于指定目的实体索引，标志链路集的相邻目的实体。<br>数据来源：本端规划<br>取值范围：0~1279<br>默认值：无 |
| SLSM | 链路选择掩码 | 可选必选说明：可选参数<br>参数含义：该参数用于指示在该链路集下的多条M3UA链路之间如何实现信令业务的负荷分担。<br>数据来源：整网规划<br>取值范围：<br>- “B0000(B0000)”<br>- “B0001(B0001)”<br>- “B0010(B0010)”<br>- “B0011(B0011)”<br>- “B0100(B0100)”<br>- “B0101(B0101)”<br>- “B0110(B0110)”<br>- “B0111(B0111)”<br>- “B1000(B1000)”<br>- “B1001(B1001)”<br>- “B1010(B1010)”<br>- “B1011(B1011)”<br>- “B1100(B1100)”<br>- “B1101(B1101)”<br>- “B1110(B1110)”<br>- “B1111(B1111)”<br>默认值：<br>“B0000(B0000)”<br>说明：一个链路集中的链路数目和链路选择掩码的关系：1条链路，设置为全0；2条链路，设置1个BIT为1；3-4条链路，设置2个BIT为1；5-8条链路，设置3个BIT为1；9-16条链路，设置4个BIT为1； 到同一个目的实体，链路选择掩码和链路集选择掩码（<br>[**ADD M3DE**](../M3UA目的实体/增加M3UA目的实体(ADD M3DE)_72345901.md)<br>）不能够有相同的BIT位设置成1； 如果该链路集的业务模式为“OVERRIDE”，该参数不起作用。 |
| WM | 工作模式 | 可选必选说明：必选参数<br>参数含义：该参数用于指定该M3UA链路集所处的工作方式。<br>数据来源：整网规划<br>取值范围：<br>- “ASP（应用服务器进程）”<br>- “SGP（信令网关进程）”<br>- “IPSP（IP服务进程）”<br>默认值：无<br>说明：当本地实体类型为SGP，目的实体类型是ASP时，链路集的工作模式必须为SGP； 当本地实体类型为ASP，目的实体类型是SGP时，链路集的工作模式必须为ASP； 当目的实体类型为IPSP或SP时，链路集的工作模式必须为IPSP。 |
| TM | 业务模式 | 可选必选说明：可选参数<br>参数含义：该参数用于指定该M3UA链路集所采用的传输模式。<br>数据来源：与对端设备协商一致。<br>取值范围：<br>- “OVERRIDE（主从模式）”：即1＋N备份模式。在这种模式中，链路集中仅有一条链路处于激活状态，其余链路则均处于备份状态；当激活的链路故障时，处于备份状态的某条链路将被立即激活并传输业务。<br>- “LOADSHARE（负荷分担模式）”：在这种模式中，链路集中所有的链路均处于激活状态，各链路以负荷分担的方式传输业务。<br>默认值：<br>“LOADSHARE（负荷分担模式）”<br>说明：在SGSN与其他设备对接时，对接的链路集其两端的传输模式必须保持一致，否则，M3UA链路将不能正常工作。 |
| LSN | 链路集名 | 可选必选说明：可选参数<br>参数含义：该参数用于链路集名称，标识链路集。<br>数据来源：本端规划<br>取值范围：1~32位字符串<br>默认值：noname |

## 操作的配置对象

## 使用实例

增加链路集索引为1，目的实体索引为1，工作模式为ASP，链路集名为“TEST”的M3UA链路集：

ADD M3LKS: LSX=1, DEX=1, WM=ASP, LSN="TEST";
