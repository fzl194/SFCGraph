# LST ACTRL（查询接入控制）

## 功能

**适用NF：NCG**

该命令用于查询接入控制模块采用的IP地址、端口号等信息。

## 注意事项

无。

## 权限

manage-ug；system-ug；monitor-ug
G_1，管理员级别命令组；G_2，操作员级别命令组；G_3，用户级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| ACID | 接入控制标识 | 可选必选说明：可选参数<br>参数含义：用于在NCG中定义一条接入控制的数据记录。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～32。<br>默认值：无<br>配置原则：无 |
| AGID | 接入网元分组标识 | 可选必选说明：可选参数<br>参数含义：用于区分不同域的接入网元。<br>数据来源：对端协商<br>取值范围：字符串类型，输入长度范围为1～32。<br>默认值：无<br>配置原则：无 |
| IPRID | IP资源标识 | 可选必选说明：可选参数<br>参数含义：用于表示IP资源。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～32。<br>默认值：无<br>配置原则：无 |
| PORT | 端口 | 可选必选说明：可选参数<br>参数含义：用于表示接入控制模块采用的端口。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～65535。<br>默认值：无<br>配置原则：无 |
| RIPTYPE | 重定向IP地址类型 | 可选必选说明：可选参数<br>参数含义：用于表示重定向IP地址属于IPv4网络还是IPv6网络，需根据当前局点规划选择其中一种。<br>数据来源：对端协商<br>取值范围：枚举类型。<br>- IPV4：IPv4。<br>- IPV6：IPv6。<br>默认值：无<br>配置原则：无 |
| RIPV4 | 重定向IPv4 | 可选必选说明：条件可选参数<br>前提条件：该参数在“RIPTYPE”配置为“IPV4”时为条件可选参数。<br>参数含义：用于表示接入控制模块采用的重定向IPv4地址。<br>数据来源：本端规划<br>取值范围：IPv4地址类型。<br>默认值：无<br>配置原则：无 |
| RIPV6 | 重定向IPv6 | 可选必选说明：条件可选参数<br>前提条件：该参数在“RIPTYPE”配置为“IPV6”时为条件可选参数。<br>参数含义：用于表示接入控制模块采用的重定向IPv6地址。<br>数据来源：本端规划<br>取值范围：IPv6地址类型。<br>默认值：无<br>配置原则：无 |
| RPORT | 重定向端口 | 可选必选说明：可选参数<br>参数含义：用于接入控制模块采用的重定向端口。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～65535。<br>默认值：无<br>配置原则：无 |

## 使用实例

查询接入控制：

```
LST ACTRL:;
```

```
RETCODE = 0  操作成功

结果如下:
---------
接入网元分组标识   =  PS_GROUP_1
     IP资源标识   =  IP_Ga_CDRReceive_1st
           端口   =  9990
    IP地址类型    =  IPv4
     重定向IPv4   =  192.168.3.5
     重定向IPv6   =  ::
     重定向端口   =  9999
     接入控制标识 =  ps_actrl
```
