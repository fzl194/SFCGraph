# ADD M2MSIGCTRL（增加M2M信令控制配置）

## 功能

**适用网元：MME**

该命令用于配置M2M用户的信令控制策略。从用户每次连接可以收发数据包数、保持连接态时间、通信周期、通信时段四个纬度对M2M用户业务行为进行限制，以保护核心网免受异常信令冲击。本命令用于规范M2M UE的业务行为，对异常用户行为进行约束。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为32。
- 该命令对业务进行限制需在“WSFD-215401NB-IoT终端异常信令与流量管理”特性开启时生效。
- 如果用户在接入时携带了“mt access”、“highPriorityAccess”、“Mo ExCeption Data”和“emergency”，则本次接入不受限制。
- 配置前需充分调研UE发起业务流程的规律，以免影响正常的业务。
- 该命令是以用户粒度进行控制的。当用户存在多个APN时，用户所有APN对应的配置参数必须保持一致。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 使用实例

1. 某行业用户，用户终端的IMSI前缀为12303000，行业APN为HANGYE1.COM，单次通信的数据包个数为3，通信持续时长约为100s，未知该行业用户终端异常行为情况，需要对其进行观察。
  ADD M2MSIGCTRL: CFGINDEX=1, CTRLSW=OFF, COMPKGSW=ON, COMPKG=4, COMTIMESW=ON, COMTIME=110;
  ADD M2MPLCY: SUBRANGE=IMSI_PREFIX, IMSIPRE="12303000", APNNI="HANGYE1.COM", SIGCTRLSW=ON, SIGCTRLINDEX=1;
2. 某行业用户，用户终端的IMSI前缀为12304000，行业APN为HANGYE2.COM，单次通信的数据包个数为3，通信持续时长约为100s，该终端正常情况下每小时需要上报2次数据。
  ADD M2MSIGCTRL: CFGINDEX=2, CTRLSW=ON, COMPKGSW=ON, COMPKG=4, COMTIMESW=ON, COMTIME=110, COMPERIODSW=ON, COMPERIODUNIT=HOUR, COMPERIOD=1, COMPERIODSERTIME=3;
  ADD M2MPLCY: SUBRANGE=IMSI_PREFIX, IMSIPRE="12304000", APNNI="HANGYE2.COM", SIGCTRLSW=ON, SIGCTRLINDEX=2;
  SET LICENSESWITCH: LICITEM="LKV2PTST01", SWITCH=ENABLE
3. 某行业用户，用户终端的IMSI前缀为12305000，行业APN为HANGYE3.COM，单次通信的数据包个数为3，通信持续时长约为100s，每天的22:00～23:00上报一次日报表。
  ADD M2MSIGCTRL: CFGINDEX=3, CTRLSW=ON, COMPKGSW=ON, COMPKG=4, COMTIMESW=ON, COMTIME=110, COMTIMEINTERVALSW=ON, COMFRE=DAY, BEGINTIME=22&00&00, ENDTIME=23&00&00, SERVICETIME=2;
  ADD M2MPLCY: SUBRANGE=IMSI_PREFIX, IMSIPRE="12305000", APNNI="HANGYE3.COM", SIGCTRLSW=ON, SIGCTRLINDEX=3;
  SET LICENSESWITCH: LICITEM="LKV2PTST01", SWITCH=ENABLE;
