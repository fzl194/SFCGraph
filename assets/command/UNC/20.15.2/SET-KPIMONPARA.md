# SET KPIMONPARA（设置KPI监控参数）

## 功能

**适用网元：SGSN、MME**

该命令用于设置KPI监控功能的参数，用来控制是否允许打开KPI监控功能。

## 注意事项

- 该命令设置成功后，最长不超过1分钟生效。
- 在设置开启KPI监控功能后，可以通过命令来设置KPI指标的门限值、最小业务量等信息。
- 如果监控开关设置为“限时关闭”，当时间超过“关闭时长”时会自动开启监控功能。因此，会出现通过命令查询出的实际值（打开）与配置导出值（限时关闭）不一致。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| MONGRANULARITY | 监控粒度 | 可选必选说明：必选参数<br>参数含义：该参数用来设置监控的粒度。<br>数据来源：本端规划。<br>取值范围：<br>- “SYSTEM(整系统)”：通过整系统粒度监控<br>- “PROCESS(单进程)”：通过单个进程粒度监控<br>系统初始设置值：请参考<br>[系统默认配置](#ZH-CN_MMLREF_0000001172345799__info)<br>。<br>配置原则：系统目前仅支持<br>“PROCESS(单进程)”<br>。 |
| MONSWITCH_SYS | 监控开关 | 可选必选说明：条件可选参数<br>参数含义：该参数用来设置整系统粒度的监控功能的开关状态。<br>数据来源：本端规划。<br>取值范围：<br>- “ON(打开)”：开启监控功能<br>- “OFF(关闭)”：关闭监控功能<br>系统初始设置值：<br>“ON(打开)”<br>配置原则：系统目前不支持。<br>说明：- 如果监控开关设置为“限时关闭”，当时间超过“关闭时长”时会自动开启监控功能。因此，会出现通过命令查询出的实际值（打开）与配置导出值（限时关闭）不一致。 |
| MONSWITCH_PRO | 监控开关 | 可选必选说明：条件可选参数<br>参数含义：该参数用来设置单进程粒度的监控功能的开关状态。<br>数据来源：本端规划。<br>取值范围：<br>- “ON(打开)”：开启监控功能<br>- “OFF(关闭)”：关闭监控功能<br>系统初始设置值：<br>“ON(打开)”<br>说明：- 如果监控开关设置为“限时关闭”，当时间超过“关闭时长”时会自动开启监控功能。因此，会出现通过命令查询出的实际值（打开）与配置导出值（限时关闭）不一致。 |
| OFFTY | 关闭类型 | 可选必选说明：条件必选参数<br>参数含义：该参数用来设置监控开关为关闭状态时，关闭的类型。<br>数据来源：本端规划。<br>取值范围：<br>- “LIMIT(限时关闭)”：限时关闭<br>- “UNLIMIT(长期关闭)”：长期关闭<br>系统初始设置值：无。<br>说明：- 当选择监控开关为“关闭”时，并且在特定场景下需要关闭监控功能一段时间时，该参数设置为“LIMIT(限时关闭)”。<br>- 当选择监控开关为“关闭”时，并且需要一直关闭监控功能时，该参数设置为“UNLIMIT(长期关闭)”。<br>- 如果监控开关设置为“限时关闭”，当时间超过“关闭时长”时会自动开启监控功能。因此，会出现通过命令查询出的实际值（打开）与配置导出值（限时关闭）不一致。 |
| STOPTIMER | 关闭时长(分) | 可选必选说明：条件必选参数<br>参数含义：该参数用来设置关闭监控功能多长时间。<br>数据来源：本端规划。<br>取值范围：30-1440<br>系统初始设置值：无<br>说明：当选择监控开关为“关闭”，选择关闭类型为“限时关闭”时，可依据具体情况来设置关闭监控功能多长时间。 |
| ALMSWITCH | 告警开关 | 可选必选说明：条件可选参数<br>参数含义：该参数用来设置当开启监控功能后，检测到整系统有故障时，是否允许上报告警。<br>数据来源：本端规划。<br>取值范围：<br>- “ON(打开)”：允许上报告警<br>- “OFF(关闭)”：不允许上报告警<br>系统初始设置值：<br>“ON(打开)”<br>配置原则：系统目前不支持。 |
| RECOVERSWITCH | 告警开关 | 可选必选说明：条件可选参数<br>参数含义：该参数用来设置当开启监控功能后，检测到某个进程故障时，是否允许上报告警。<br>数据来源：本端规划。<br>取值范围：<br>- “ON(打开)”：允许上报告警<br>- “OFF(关闭)”：不允许上报告警<br>系统初始设置值：<br>“ON(打开)” |
| ALARMPERIOD | 告警间隔周期（分） | 可选必选说明：条件可选参数<br>参数含义：该参数和<br>“告警次数”<br>一起控制指定时间范围内允许上报告警的个数，时间范围称为告警间隔周期。<br>数据来源：本端规划。<br>取值范围：30-1440<br>系统初始设置值：60。<br>说明：请谨慎使用该参数，以免系统上报过多告警。 |
| ALARMTIMES | 告警次数 | 可选必选说明：条件可选参数<br>参数含义：该参数和<br>“告警间隔周期（分）”<br>一起控制指定时间范围内允许上报告警的个数，上报告警的个数称为告警次数。<br>数据来源：本端规划。<br>取值范围：1-65535<br>系统初始设置值：3。<br>说明：请谨慎使用该参数，以免系统上报过多告警。 |
| SELFRECOVERSWITCH | 自愈开关 | 可选必选说明：条件可选参数<br>参数含义：该参数用来设置当开启监控功能和告警功能后，检测到某个进程故障时，是否允许自愈处理。<br>数据来源：本端规划。<br>取值范围：<br>- “ON(打开)”：允许自愈处理<br>- “OFF(关闭)”：不允许自愈处理<br>系统初始设置值：<br>“ON(打开)” |

## 使用实例

设置监控粒度为单进程粒度，监控开关为关闭状态，并且仅限关闭监控功能30分钟。

```
SET KPIMONPARA: MONGRANULARITY=PROCESS, MONSWITCH_PRO=OFF, OFFTY=LIMIT, STOPTIMER=30;
```
