# MOD OP（修改本地用户属性）

## 功能

该命令用于修改网元侧已配置的本地用户的属性信息，包括密码、用户组、接入方式、用户状态、接入限制数、老化周期、密码有效期和允许登录时间段。具体有哪些本地用户，可以通过 [**LST OP**](查询本地用户（LST OP）_59036769.md) 命令查出。

## 注意事项

- 该命令执行后立即生效。
- 本地用户的属性（密码、状态、接入类型）变更时，会强制下线在线用户。
- 本地用户的其他属性变更不会对当前在线用户产生影响，已在线用户重新登录之后此配置变更生效。
- 当本地用户属性变更时，为降低安全风险，建议该用户重新登录。
- 修改网元侧已配置的用户的属性信息后，MANO侧该用户的属性信息要修改保持一致。
- 使用该命令配置用户允许二次授权后，用户状态正常时才能进行二次授权。状态不正常包括：用户未执行过首次登录修改初始密码、用户密码过期、用户被锁定等。

## 权限

G_1，管理员级别命令组

| CFMPASSWORD | 确认密码 | 可选必选说明：可选参数<br>参数含义：本地用户的确认密码。<br>数据来源：本端规划<br>取值范围：密码类型，密码长度范围为8～64。<br>默认值：无<br>配置原则：必须与第一次输入的密码一致。 |

| USERSTATE | 用户状态 | 可选必选说明：可选参数<br>参数含义：用户的状态。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- block：用户状态为阻塞。<br>- active：用户状态为激活。<br>默认值：无 |
| MAXACCESSNUM | 接入限制数目 | 可选必选说明：可选参数<br>参数含义：接入限制数目。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～10，2147483647。<br>默认值：无<br>配置原则：当不配置的情况下，缺省值为2147483647，此时不检查用户接入限制个数。可以通过将参数设置为缺省值，取消该配置。使用当前账号登录系统时需确保该参数值不小于2。 |

## 使用实例

使用实例修改用户，用户名为“omuser”，密码为“*****”，确认密码为“*****”，SSH服务为使能，HTTP服务为去使能，用户组为“manage-ug”，用户状态为激活，最大连接数为10，用户老化天数为300天，密码过期天数为500天，用户过期时间为2015-10-10，是否限制登录时间为YES，允许登录起始星期为星期一，允许登录结束星期为星期五，允许登录开始时间为01:00:00，允许登录结束时间为23:59:00，不允许二次授权，重置后首次登录不强制修改密码：

```
MOD OP: USERNAME="omuser", PASSWORD="*****", CFMPASSWORD="*****", SERVICESSH=TRUE, SERVICEHTTP=FALSE,USERGROUPNAME="manage-ug", USERSTATE=active,MAXACCESSNUM=10,USERAGINGPERIOD=300,PASSWORDEXPIREINDAYS=500,ENDDATE=2015&10&10,LOGINLIMIT=YES,STARTWEEK=Mon,ENDWEEK=Fri,STARTTIME=01&00&00,ENDTIME=23&59&00,CONFIRMAUTH=FALSE,FIRSTLGISWITCH=FALSE;
```
