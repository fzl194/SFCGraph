# ADD ZC（增加区域码）

## 功能

**适用网元：SGSN、MME**

此命令用于增加区域码记录。区域码用于定义允许或者禁止用户漫游的区域，借助一个完全的区域码列表，运营商可以确定用户是否能在此区域漫游。用户不允许在此区域漫游的情况下， UNC 回复给用户的拒绝消息中携带的原因值可通过软参配置，2G/3G用户可使用软参BYTE_EX64配置，4G用户可使用软参BYTE_EX209配置。

## 注意事项

- 此命令最大记录数为20000。
- 此命令配置后，需要把移动性扩展功能([**SET MMFUNC**](../../MM扩展功能管理/设置移动性管理扩展功能(SET MMFUNC)_26145512.md))中的“区域码”设置为“YES”，配置才能生效。
- 可以为一个ZC配置多个LAI或RAI等位置区域，一个ZC代表一组位置区域。
- 此命令配置后，如果用户签约了ZC（区域码），则只能从签约ZC对应的本命令的位置接入系统，若签约ZC没有在本命令配置也不允许接入。
- 配置此命令时，需要同时考虑2G/3G/4G情况，对允许用户接入的TAI和LAI（或者TAI和RAI）需同时进行配置，避免用户在未配置的制式接入时受限。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 使用实例

1. 增加一条位置区域码记录，区域码为0x1234，标识类型为LA，移动国家代码为123，移动网号为03，位置区域码为1，区域名称为test1：
  ADD ZC: ZC="0x1234", IDTYPE=LA, MCC="123", MNC="03", LAC="1", ZCN="test1";
2. 增加一条路由区域码记录，区域码为12，标识类型为RA，移动国家代码为123，移动网号为03，位置区域码为10，路由区域码为1，路由区域码范围为5，区域名称为test2：
  ADD ZC: ZC="12", IDTYPE=RA, MCC="123", MNC="03", LAC="10", RAC="1", RACRANGE="5", ZCN="test2";
