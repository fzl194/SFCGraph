# MOD S1LOCRPT（修改S1模式实时小区位置订阅）

## 功能

**适用网元：MME**

此命令用于修改S1模式实时小区位置订阅，根据用户当前所在的位置，控制是否订阅该用户的实时小区位置。

针对用户在相应的位置区域中设置对应的订阅策略时，需要执行此命令。

## 注意事项

- 此命令执行后不是立即生效，而是等待用户下一次发起数据业务时生效。
- 此命令配置后会对于订阅流程产生影响。如果修改为取消订阅时，当用户完成当前数据业务，进入到空闲态时，自动取消订阅；如果修改为开启订阅，则在用户再次发起服务请求时，启动订阅。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| AREA | 区域范围 | 可选必选说明：必选参数<br>参数含义：该参数用于指定待修改的区域范围。<br>数据来源：整网规划<br>取值范围：<br>- “TA(指定跟踪区)”：表示该区域标识类型为跟踪区。<br>默认值：无<br>说明：- 区域范围表示进行实时位置订阅所适用的位置区域。<br>- 区域范围目前只支持跟踪区粒度的范围。<br>- 相同粒度区域范围的各记录的区域码区段不能交叠。 |
| MCC | 移动国家码 | 可选必选说明：条件必选参数<br>参数含义：该参数用于指定待修改的移动国家代码。<br>前提条件：该参数在“AREA(区域范围)”设置为“TA(指定跟踪区)”时，才需要配置。<br>数据来源：整网规划<br>取值范围：3位的十进制数字<br>默认值：无 |
| MNC | 移动网号 | 可选必选说明：条件必选参数<br>参数含义：该参数用于指定待修改的移动网号。<br>前提条件：该参数在“AREA(区域范围)”设置为“TA(指定跟踪区)”时，才需要配置。<br>数据来源：整网规划<br>取值范围：2～3位的十进制数字<br>默认值：无 |
| TAC | 跟踪区域码 | 可选必选说明：条件必选参数<br>参数含义：该参数用于指定待修改的跟踪区域码。<br>前提条件：该参数在“AREA(区域范围)”设置为“TA(指定跟踪区)”时，才需要配置。<br>数据来源：整网规划<br>取值范围：0x0000～0xFFFF<br>默认值：无<br>说明：- 该参数需要与已有记录中的“TAC(跟踪区域码)”保持一致，执行命令前先通过[**LST S1LOCRPT**](查询S1模式实时小区位置订阅(LST S1LOCRPT)_72345171.md)命令确认已有记录。 |
| TACRANGE | 跟踪区域码范围 | 可选必选说明：条件可选参数<br>参数含义：该参数用于指定待修改的跟踪区域码范围。<br>前提条件：该参数在“AREA(区域范围)”设置为“TA(指定跟踪区)”时，才需要配置。<br>数据来源：整网规划<br>取值范围：0x0000～0xFFFF<br>默认值：无<br>配置原则：该参数的取值需要大于或等于“TAC”。<br>说明：- 该参数与“TAC(跟踪区域码)”参数构成的TAC区段，方便客户修改由“TAC(跟踪区域码)”指定记录的TAC区段，如果构成TAC区段与指定记录外的TAC区段交叠，则命令执行失败。<br>- 如果不输入，表示对指定记录的“TACRANGE(跟踪区域码范围)”不做修改。 |
| SUBSWITCH | 订阅开关 | 可选必选说明：可选参数<br>参数含义：该参数用于指定是否对该范围内的用户发起实时位置订阅。<br>数据来源：整网规划<br>取值范围：<br>- “ON(打开)”<br>- “OFF(关闭)”<br>默认值：无 |

## 使用实例

修改一条S1模式实时小区位置订阅记录，区域范围为TA，移动国家码为111，移动网号为222，跟踪区域码为0x3333，修改跟踪区域码范围为0x4444，修改订阅开关为ON：

MOD S1LOCRPT: AREA=TA, MCC="111", MNC="222", TAC="0x3333", TACRANGE="0x4444", SUBSWITCH=ON;
