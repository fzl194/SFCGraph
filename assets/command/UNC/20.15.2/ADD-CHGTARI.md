# ADD CHGTARI（增加费率时段配置）

## 功能

**适用网元：SGSN**

该命令用于设置普通计费属性用户、预付费计费属性用户、包月制计费属性用户或实时计费属性用户的某天（含工作日、周休日和节假日）的费率时段。该命令与星期表配置（ [**SET CHGWKDY**](../计费星期配置/设置星期配置(SET CHGWKDY)_26305210.md) ）和节假日配置（ [**ADD CHGHOLI**](../计费节假日配置/增加节假日配置(ADD CHGHOLI)_26145382.md) ）相结合，进行灵活的费率时段配置。该命令的配置是为了实现对S-CDR话单进行不同费率时段的计费。

## 注意事项

- 该命令执行后立即生效。
- 在费率时段配置表中，“费率类型”、“起始时间”、“结束时间”和“计费属性”唯一确定一条记录。
- 新增费率时段与具有相同“费率类型”和“计费属性”的已有费率时段不得相互重叠。
- 具有相同“计费属性”和“费率类型”的记录总数不能超过24，本表记录总数不能超过288。
- 任一费率时段的长度不得小于10分钟。
- 任一费率时段的“起始时间”、“结束时间”和零点的间隔或等于0，或大于等于10分钟。
- 任两条费率时段的间隔或等于0，或大于等于10分钟。

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| TT | 费率类型 | 可选必选说明：必选参数<br>参数含义：该参数用于指定费率时段的费率类型。<br>数据来源：整网规划<br>取值范围：<br>- “WORK(工作日)”：表示该费率时段适用于工作日。<br>- “WEEKEND(周休日)”：表示该费率时段适用于周休日。<br>- “FESTIVAL(节假日)”：表示该费率时段适用于节假日。<br>默认值：无 |
| ST | 起始时间 | 可选必选说明：必选参数<br>参数含义：该参数用于指定费率时段的起始时间。<br>数据来源：整网规划<br>取值范围：00&00&00~23&59&59<br>默认值：无 |
| ET | 结束时间 | 可选必选说明：必选参数<br>参数含义：该参数用于指定费率时段的结束时间。<br>数据来源：整网规划<br>取值范围：00&00&00~23&59&59<br>默认值：无 |
| CC | 计费属性 | 可选必选说明：必选参数<br>参数含义：该参数用于指定该费率时段是对普通计费属性用户、预付费计费属性用户、包月制计费属性用户还是对实时计费属性用户有效。<br>数据来源：整网规划<br>取值范围：<br>- “NORMAL(普通计费)”：表示普通计费属性，按照此种方式计费的用户按照常规的方式支付费用。<br>- “PREPAID(预付费)”：表示预付费计费属性，按照此种方式计费的用户在获取某种服务之前需要预支付一定的费用。<br>- “FLATRATE(包月制)”：表示包月制计费属性，按照此种方式计费的用户在一个月内的收费是固定的。<br>- “HOTBILLING(实时计费)”：表示实时计费属性，按照此种方式计费的用户将在短时间或流量达到某个值时及时生成话单，保证运营商对此类用户及时收费。<br>默认值：无 |
| TARIACCFLG | 费率变更允许的用户类型 | 可选必选说明：可选参数<br>参数含义：该参数用于指定该费率时段是对访问类型为Home、Roaming、Visiting的用户有效。<br>数据来源：整网规划<br>取值范围：<br>- “HOME_USER（本地用户）”<br>- “ROAM_USER（漫游用户）”<br>- “VISIT_USER（拜访用户）”<br>默认值：无<br>说明：系统默认所有字段全选。 |

## 使用实例

1. 增加一条预付费计费属性的费率时段记录：费率类型为工作日，起始时间为0点10分，结束时间为10点30分，配置格式为：
  ADD CHGTARI: TT=WORK, ST=00&10&00, ET=10&30&00, CC=PREPAID;
2. 增加一条普通计费属性的费率时段记录：费率类型为周休日，起始时间为0点0分，结束时间为0点0分，配置格式为：
  ADD CHGTARI: TT=WEEKEND, ST=00&00&00, ET=00&00&00, CC=NORMAL;
  > **说明**
  > 起始时间和结束时间都为0点0分，表示一整天。
