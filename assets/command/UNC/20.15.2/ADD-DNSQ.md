# ADD DNSQ（增加DNS查询控制参数）

## 功能

**适用网元：SGSN、MME**

该命令用于增加DNS查询控制的相关参数，包括DNS的查询方式和DNS服务器组ID等。

## 注意事项

- 本表最大记录数为1024。
- 该命令执行后立即生效。
- 该命令可以控制使不同的DNS域名发向不同的服务器组查询。但是如果域名完全相同，由于UNC的DNS cache针对相同域名只能存储一份记录，UNC会使用cache中已经存在的来自于某一服务器组的记录，而不会再次向不同的服务器组发起查询。如果有这种应用场景，推荐您使用[**ADD GWSELPLCY**](../../GnGp-GGSN_S5_S8接口管理/GGSN_P-GW选择/增加GGSN_P-GW选择策略（ADD GWSELPLCY）_26145944.md)配置出不同的域名。
- 该命令执行后，需要执行[**CLR DNSC**](../../../系统管理/DNS维护管理/清除DNS Cache/清除DNS缓存(CLR DNSC)_72345945.md)来清除L1，L2 缓存，以保证使用最新的配置进行查询。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| SUBRANGE | 用户范围 | 可选必选说明：必选参数<br>参数含义：该参数用于指定命令生效的用户范围。<br>数据来源：整网规划<br>取值范围：<br>- “All（所有用户）”<br>- “SPECIFY（指定用户）”<br>默认值：无<br>配置原则：用户范围+IMSI前缀+域名后缀不能重复。<br>说明：- 当“用户范围”选择为“All（所有用户）”时，只要查询请求的域名与配置的“域名后缀”匹配，则此次查询与该配置匹配成功，UNC会按照该配置指定的方式进行查询。<br>- 当“用户范围”选择为“SPECIFY（指定用户）”时，只有当查询请求的用户IMSI与配置的“IMSI前缀”匹配、查询请求的域名与配置的“域名后缀”匹配，并且查询的网元类型是GGSN、S-GW、P-GW之一，则此次查询与该配置匹配成功，UNC会按照该配置指定的方式进行查询。 |
| IMSIPRE | IMSI前缀 | 可选必选说明：条件必选参数<br>参数含义：该参数用于表示IMSI前缀。<br>前提条件：当<br>“用户范围”<br>为<br>“SPECIFY（指定用户）”<br>时，此参数为必选参数。<br>数据来源：整网规划<br>取值范围：5~15位数字<br>默认值：无 |
| DNSUF | 域名后缀 | 可选必选说明：必选参数<br>参数含义：该参数用于表示域名后缀。<br>数据来源：整网规划<br>取值范围：1~255个字符<br>默认值：无<br>配置原则：域名后缀支持*（通配符）。域名后缀配置*（通配符）记录后，如果没有最长域名后缀匹配的记录，所有向DNS服务器发起的解析将都使用域名后缀配置*的记录；IMSI前缀+域名后缀配置*（通配符）记录后，IMSI前缀下如果没有最长域名后缀匹配的记录，该IMSI前缀的所有向DNS服务器发起的解析都将使用域名后缀配置*的记录。<br>说明：- 按照域名后缀选择合适的DNS服务器。域名后缀在比较时从后向前进行最大匹配。<br>- 域名后缀支持配置为*。任何域名和*比较，都认为匹配。<br>- 域名后缀不能以“.”开始，也不能以“.”结束。<br>- 相同域名后缀下的IMSI前缀不能相同，相同IMSI前缀下的域名后缀不能相同。<br>- 该参数只能由字母（A-Z或者a-z）、数字（0-9）、连字符（-）、通配符（*）和点（.）组成。<br>- 按照协议RFC1035规定，Hostname最大有效字符数为253，并且每个Label最大长度为63个字节。 |
| ANALYSETYPE | DNS解析方式 | 可选必选说明：可选参数<br>参数含义：该参数用于表示DNS解析方式，即控制DNS查询时优先使用server数据或者优先使用本地配置数据。<br>数据来源：整网规划<br>取值范围：<br>- “E_HOSTFILE_SERVER（本地数据优先）”：本地配置数据优先。<br>- “E_SERVER_HOSTFILE（服务器数据优先）”：服务器数据优先。<br>默认值：E_HOSTFILE_SERVER（本地数据优先）<br>说明：- 当此参数不配置时，解析方式会默认为本地数据优先，以保持继承性。 |
| DNSQUERYMODE | DNS查询方式 | 可选必选说明：可选参数<br>参数含义：该参数用于表示DNS查询方式，控制DNS查询时使用A记录还是NAPTR记录查询方法。<br>数据来源：整网规划<br>取值范围：<br>- “AAAA/A(AAAA/A)”：域名到IP的映射。<br>- “NAPTR(NAPTR)”：FQDN（标准规范域名）到域名的映射。<br>默认值：无<br>说明：- 当此参数不配置时，查询方式确定规则为：当查询对等网元时，根据后缀来区分，即“GPRS”后缀对应为A类型，“ORG”对应为NAPTR类型；当查询上游网元时，根据查询消息中携带的接口类型来区分，即“Gn/Gp”接口对应为A类型，“Sx”接口对应NAPTR类型，但"Gn/Gp"接口查询支持DCNR能力的GGSN时，选择NAPTR类型。 |
| GROUPID | DNS服务器组ID | 可选必选说明：可选参数<br>参数含义：该参数用于表示服务器组ID。该值对应于<br>[**LST DNSS**](../DNS服务器管理/查询DNS服务器(LST DNSS)_26305708.md)<br>的组ID。<br>前提条件：该参数必须已经通过<br>[**ADD DNSS**](../DNS服务器管理/增加DNS服务器(ADD DNSS)_72345497.md)<br>配置。如果未输入该参数，系统默认使用0作为该参数的输入值。即使ID为0的服务器组未通过<br>[**ADD DNSS**](../DNS服务器管理/增加DNS服务器(ADD DNSS)_72345497.md)<br>配置，此处也可以引用0作为输入参数。但是如果ID为0的服务器组未配置，会导致向该服务器组发起的DNS查询失败。<br>数据来源：整网规划<br>取值范围：0~37<br>默认值：无 |
| DESC | 描述 | 可选必选说明：可选参数<br>参数含义：该参数用于描述本条规则的文字说明，目的是在配置的时候可以将对象属性、配置原因、背景等进行描述，以便在查询时能够在大量配置数据中清晰的掌握配置的原因。<br>数据来源：整网规划<br>取值范围：0~32位字符串<br>默认值：无 |

## 操作的配置对象

- [[UNC@20.15.2@ConfigObject@DNSQ]] · DNS查询控制参数（DNSQ）

## 使用实例

添加DNS查询参数，包括DNS服务器组ID：

ADD DNSQ: SUBRANGE=All, DNSUF="gprs", DNSQUERYMODE=AAAA/A, GROUPID=1, DESC="For huawei rule01";
