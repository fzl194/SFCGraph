# ADD CDRSTOR（增加话单存储）

## 功能

**适用NF：NCG**

该命令用于配置原始话单、第一份最终话单以及第二份最终话单以及可能重复帧的存储规则。CG需要分别增加原始话单、第一份最终话单、第二份最终话单以及可能重复帧的存储规则。

CG支持定制最终话单文件名命名，通过配置可以在文件名中体现各类信息。例如：

- 增加序列号
  序列号能清晰的体现每个最终话单文件产生的先后顺序。
- 加入局点信息
  当采用大本地网配置时，通常要将各个归属地的话单派发到不同的计费中心（本文档中又称BS（Billing System））。这时可以在最终话单文件名中加入局点信息，方便管理。
- 加入时间信息
  当计费中心要求在最终话单文件名中体现最终话单的生成时间时，可以在最终话单文件名中加入时间信息。
- 加入话单张数信息
  当计费中心要求在最终话单文件名中体现最终话单文件中包含的最终话单数量时，可以在最终话单文件名中进行配置。
- 加入接入点信息
  当多对CG的RU的最终话单送往同一计费中心的同一文件夹下时，需要在话单名上加入接入点信息。用于区分不同对RU上的话单，避免话单被覆盖。

话单文件名定制配置项有：

- %0xL或%xL
- %Y%m%d%H%M%S或%y%m%d%H%M%S或%Y%m%d%I%M%S或%y%m%d%I%M%S
- %p
- %w
- %c或%C
- %a或%A
- %B
- %o
- %0xN

> **说明**
> - 最终话单文件名的默认配置为“b%08L.dat”。默认话单文件名的取值范围为“b00000001.dat”~“b99999999.dat”。
> - 用户自行配置文件名时，必须配置[表1](#ZH-CN_CONCEPT_0251174277__table_0365FEC2)中列出的“增加序列号信息”，且只能增加一次，用于标示第二份最终话单的生成顺序，12/24小时制的年月日时分秒配置项需要和序列号配置项一起使用。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为2048。
- 该命令除最终话单文件名的最大序列号外其他参数均可动态生效。
- 在话单存储中配置的话单相关路径为相对路径，在RU上实际保存话单文件的绝对路径为“/opt/CG_VNFC/run/vrpv8/product/”加上在话单存储中配置的相对路径。
- 当前版本不支持Bi口SingleIP功能，请勿开启。

## 使用实例

- 增加原始话单的存储规则：
  ```
  ADD CDRSTOR: CDRSTORID="OriginalCDR", ISSINGLEBI=No, FILETYPE=Original;
  ```
- 增加第一份最终话单的存储规则：
  ```
  ADD CDRSTOR: CDRSTORID="FinalCDR_1stCopy", ISSINGLEBI=No, FILETYPE=FirstFinal;
  ```
- 增加第二份最终话单的存储规则：
  ```
  ADD CDRSTOR: CDRSTORID="FinalCDR_2ndCopy", ISSINGLEBI=No, FILETYPE=SecondFinal, AGID="PS_GROUP_1";
  ```
- 增加可能重复帧的存储规则：
  ```
  ADD CDRSTOR: CDRSTORID="PotentialDuplicatesCDR", ISSINGLEBI=No, FILETYPE=PotentialDup, AGID="PS_GROUP_1", MNAME="AP66_1", PDDESTPATH="./frontsave/destpath/";
  ```
- 增加全部话单文件的存储规则：
  ```
  ADD CDRSTOR: CDRSTORID="AllCDRFile", ISSINGLEBI=No, FILETYPE=AllCDRFiles;
  ```
