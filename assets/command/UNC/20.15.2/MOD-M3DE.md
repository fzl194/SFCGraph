# MOD M3DE（修改M3UA目的实体）

## 功能

**适用网元：SGSN、MME、SMSF**

该命令用于修改M3UA目的实体配置数据。

## 注意事项

该命令执行后立即生效。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| DEX | 目的实体索引 | 可选必选说明：必选参数<br>参数含义：该参数用于指定唯一标识目的实体的索引。<br>数据来源：本端规划<br>取值范围：0~1279<br>默认值：无<br>配置原则：无 |
| SLSSM | 链路集选择掩码 | 可选必选说明：可选参数<br>参数含义：该参数用于指定链路集选择掩码。链路集选择掩码用于路由选择，实现到同一个目的实体的多条路由之间的负荷分担。<br>数据来源：整网规划<br>取值范围：<br>- “B0000（B0000）”<br>- “B0001（B0001）”<br>- “B0010（B0010）”<br>- “B0011（B0011）”<br>- “B0100（B0100）”<br>- “B0101（B0101）”<br>- “B0110（B0110）”<br>- “B0111（B0111）”<br>- “B1000（B1000）”<br>- “B1001（B1001）”<br>- “B1010（B1010）”<br>- “B1011（B1011）”<br>- “B1100（B1100）”<br>- “B1101（B1101）”<br>- “B1110（B1110）”<br>- “B1111（B1111）”<br>默认值：无<br>配置原则：无<br>说明：到一个目的实体的路由数目和链路集选择掩码的关系：1条路由，设置为全0；2条路由，设置1个BIT为1；3-4条路由，设置2个BIT为1；5-8条路由，设置3个BIT为1；9-16条路由，设置4个BIT为1； 到同一个目的实体，信令链路集选择掩码和信令链路选择掩码（<br>[**LST M3LKS**](../M3UA链路集/查询M3UA信令链路集(LST M3LKS)_26146312.md)<br>）不能够有相同的BIT位设置成1。 |
| DEN | 目的实体名 | 可选必选说明：可选参数<br>参数含义：该参数用于标识目的实体。<br>数据来源：与对端网元协商<br>取值范围：长度不超过32的字符串<br>默认值：无<br>配置原则：无 |

## 操作的配置对象

- [[configobject/UNC/20.15.2/M3DE]] · M3UA目的实体（M3DE）

## 使用实例

将索引为1的M3UA目的实体链路集选择掩码修改为B0010，目的实体名修改为M3DE1：

```
MOD M3DE: DEX=1, SLSSM=B0010, DEN="M3DE1";
```
