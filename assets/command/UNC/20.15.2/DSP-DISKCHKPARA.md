# DSP DISKCHKPARA（查询存储检测参数）

## 功能

本命令用于查询节点当前的磁盘检测参数信息。

- 若需要查询所有节点的磁盘检测参数，需要输入“网元ID”并选择“查询存储检测类型”。
- 若需要查询指定节点的磁盘检测参数，需要输入“网元ID”和“节点名称”，并选择“查询存储检测类型”。

> **说明**
> 该命令仅在Full-stack虚机场景下支持。

## 注意事项

无。

## 参数

| **参数标识** | **参数名称** | **参数说明** |
| --- | --- | --- |
| APPID | 网元ID | 可选必选说明：可选参数<br>参数含义：用于指示系统需要查询的网元ID。<br>取值范围：0～65535。<br>默认值：无。<br>配置原则：<br>“网元ID”<br>获取方式为：点击<br>“首页”<br>，查看<br>“应用ID”<br>即可。 |
| TYPE | 查询存储检测类型 | 可选必选说明：必选参数<br>参数含义：用于指示系统需要查询指定网元ID下的节点存储检测参数类型。<br>取值范围：<br>- “DiskFault(存储故障)”：节点存储故障的检测参数信息。<br>- “DiskSubHealth(存储亚健康)”：节点存储亚健康的检测参数信息。<br>默认值：无。<br>配置原则：无。 |
| NODENAME | 节点名称 | 可选必选说明：可选参数<br>参数含义：用于指示系统需要查询哪个节点的存储检测参数。<br>取值范围：不超过128位字符串。<br>默认值：无。<br>配置原则：<br>- 若不输入，则表示查询所有节点的信息。<br>- 用户可以通过[**DSP NODE**](节点查询（DSP NODE）_71678755.md)命令查看相应的节点名称。 |

## 使用实例

1. 查询“网元ID”为“0”的所有节点“存储故障检测”参数信息：
  ```
  %%DSP DISKCHKPARA: TYPE=DiskFault, APPID=0;%%
  RETCODE = 0  操作成功

  操作结果如下
  ------------
  网元ID  节点IP    节点名称   查询类型  检测开关 存储异常检测周期(个) 存储异常检测阈值（%） IO时延（ms）  dd读写速率（KB/s）  
  0       10.0.0.1  10.0.0.1   存储故障   开启      19                    20                   200            20              
  0       10.0.0.2  10.0.0.2   存储故障   开启      19                    20                   200            20
  0       10.0.0.3  10.0.0.3   存储故障   开启      19                    20                   200            20
                 
  (结果个数 = 3)

  ---    END
  ```
2. 查询“网元ID”为“0”，“节点名称”为“10.0.0.1”的“存储亚健康检测”参数信息：
  ```
  %%DSP DISKCHKPARA: TYPE=DiskSubHealth, APPID=0, NODENAME="10.0.0.1";%%
  RETCODE = 0  操作成功

  操作结果如下
  ------------
                   网元ID  =  0
                   节点IP  =  10.0.0.1
                 节点名称  =  10.0.0.1
                 查询类型  =  存储亚健康
                 检测开关  =  关闭
     存储异常检测周期(个)  =  17
    存储异常检测阈值（%）  =  50
             IO时延（ms）  =  200
       dd读写速率（KB/s）  =  20
  (结果个数 = 1)

  ---    END
  ```
