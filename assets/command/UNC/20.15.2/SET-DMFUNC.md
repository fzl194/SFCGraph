# SET DMFUNC（设置Diameter配置）

## 功能

**适用网元：SGSN、MME**

该命令用于设置Diameter全局参数。

## 注意事项

该命令执行后生效。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| RECONNMETHOD | 重连方式 | 可选必选说明：可选参数<br>参数含义：该参数表示<br>UNC<br>连续重连的方式。说明：取值为恒定时长（重连时间不变），可变时长（累加重连时间）。<br>数据来源：整网规划<br>取值范围：<br>- “Change (可变时长)”:表示累加重连时长。<br>- “Invariableness (恒定时长)”:表示重连时长不变。<br>系统初始设置值：Invariableness (恒定时长)<br>说明：- 可变时长：重连时长为n*“重连定时器(s)”+“等待回复定时器(s)”设置的值，其中n从1开始递增，最大值为20。<br>- 恒定时长：重连时长为“重连定时器(s)”+“等待回复定时器(s)”设置的值。 |
| RECONNTIMER | 重连定时器（s） | 可选必选说明：可选参数<br>参数含义：该参数表示当<br>UNC<br>建立连接失败时，需等待重新试连的时间。如果超时，<br>UNC<br>会显示失败信息，并尝试重连。<br>数据来源：整网规划<br>取值范围： 1～100<br>系统初始设置值：30 |
| WAITANSWERTIMER | 等待回复定时器（s） | 可选必选说明：可选参数<br>参数含义：该参数表示<br>UNC<br>发出一条请求信息，收到回复的时间。如果超时，<br>UNC<br>再次初始化一条新连接发往对端。<br>数据来源：整网规划<br>取值范围： 1～100<br>系统初始设置值：14 |
| WATCHDOGTIMER | 看门狗定时器（s） | 可选必选说明：可选参数<br>参数含义：该参数表示<br>UNC<br>发送DWR（看门狗）的间隔时间。 如果超时，<br>UNC<br>重新发送DWR消息。<br>数据来源：整网规划<br>取值范围： 1～100<br>系统初始设置值：15<br>说明：- Diameter层故障检测时间应大于SCTP层故障探测时间。即“看门狗定时器(s)”*(“连接侦测次数(times)”+1)大于SCTP层故障探测时间。<br>- “连接恢复定时器（ms）”时长需小于“看门狗定时器(s)”时长 。<br>- 此参数需要偶联重新建链时才能生效。 |
| CONNDETECTNUM | 连接侦测次数（times） | 可选必选说明：可选参数<br>参数含义：该参数表示连接侦测的次数，当尝试链接次数超过配置的参数（默认为3次），<br>UNC<br>发送相同数量的DWR信息至对端，并等待回复。<br>数据来源：整网规划<br>取值范围：1～255<br>系统初始设置值： 3 |
| MSGVERNUMBER | 消息版本号 | 可选必选说明：可选参数<br>参数含义：在消息头中将会出现的版本号，诸如CER，DWR，DPR消息。对于其他的协议版本， 该消息的版本值为1。<br>数据来源：整网规划<br>取值范围：1<br>系统初始设置值：1 |
| DIAMPEERRSPTIMER | 等待对端响应定时器（s） | 可选必选说明：可选参数<br>参数含义：该参数表示发送请求消息（不包括DWR和CER）后等待对端回复的时间。如果超时，则此次交互失败。<br>数据来源：全网规划<br>取值范围：1～14<br>系统初始设置值：13<br>说明：- 不建议修改此参数，修改此参数值可能导致上层业务流程成功率降低。<br>- 此参数配置只对S6a/S6d和SLg接口生效。S13接口为固定值60(s)，不可修改。 |
| FAILOVER | 重选路由功能开关 | 可选必选说明：可选参数<br>参数含义：该参数用于指定是否需要重选路由。<br>数据来源：全网规划<br>取值范围：<br>- “YES（是）”<br>- “NO（否）”<br>系统初始设置值：<br>“NO（否）”<br>说明：- 当该参数设置为“YES（是）”，且[**ADD_DMFOVERCODE**](../Diameter协议接口重选路配置/增加触发重选路由的错误码(ADD DMFOVERCODE)_26146294.md)命令中配置了“触发重选路由的错误码”，若系统收到对端返回的错误码为配置值时，则系统重选路由重发消息。当该参数设置为“NO（否）”，系统不会重选路由。<br>- 当该参数设置为“YES（是）”，当Diameter链路故障时，系统会将在“等待对端响应定时器”超时前未收到应答的请求消息重选路由发送。当该参数设置为“NO（否）”，系统不会将在“等待对端响应定时器”超时前未收到应答的请求消息重新选择其他可用链路发送。 |
| DWRSW | 心跳消息故障检测开关 | 可选必选说明：可选参数<br>参数含义：该参数用于指定是否通过心跳消息（DWR/DWA）进行增强的Diameter连接故障检测和恢复。<br>数据来源：全网规划<br>取值范围：<br>- “ON（开启）”<br>- “OFF（关闭）”<br>系统初始设置值：<br>“OFF（关闭）”<br>说明：- 如果开关开启，会增大连接恢复时长，如果现网网络不稳定，会出现diameter链路没有变为DOWN但是请求消息无法发送的情况。<br>- 如果出现DIAM层故障但是SCTP层链路正常，请打开此开关。<br>- 开关打开后，包括两种使用场景：- 故障检测：当此参数为“ON（开启）”时，如果检测到Diameter连接中断，将关闭SCTP连接。并且在第一次心跳探测超时后，即切换到其它连接。当此参数为“OFF（关闭）”时，如果检测到Diameter连接中断，仅将此连接设置为故障状态并切换到其它连接，不关闭SCTP连接。Diameter连接故障检测时长为：“看门狗定时器(s)”*(“连接侦测次数(times)”+1)。<br>- 连接恢复：当此参数为“ON（开启）”时，在CER/CEA交互完成后，需要再进行“连接恢复次数”次DWR/DWA交互后，才能从此连接发送消息。CER/CEA交互后，连接恢复时长为：“连接恢复次数”*“连接恢复定时器（ms）”。当此参数为“OFF（关闭）”时，在CER/CEA交互后，则认为此连接正常。 |
| DWRNUM | 连接恢复次数 | 可选必选说明：可选参数<br>参数含义：该参数用于指定在Diameter连接恢复过程中，DWR/DWA消息交互次数，连续指定次数交互成功后，才可用此连接发送消息。<br>数据来源：全网规划<br>取值范围：1～10<br>系统初始设置值：3<br>说明：当<br>“心跳消息故障检测开关”<br>为<br>“ON（开启）”<br>时，该字段有效。 |
| DWRCTIMER | 连接恢复定时器（ms） | 可选必选说明：可选参数<br>参数含义：该参数用于指定在Diameter连接恢复过程中，每次DWR/DWA消息交互超时时长。<br>数据来源：全网规划<br>取值范围：50~6000<br>系统初始设置值：3000<br>说明：- 当“心跳消息故障检测开关”为“ON（开启）”时，该字段有效。<br>- “连接恢复定时器（ms）”时长需小于“看门狗定时器（s）”时长 。 |
| CONGTIMER | 对端拥塞避免定时器（s） | 可选必选说明：可选参数<br>参数含义：该参数用于指定对端拥塞避免定时器时长。<br>数据来源：全网规划<br>取值范围：1~3600<br>系统初始设置值：30<br>说明：当通过<br>[**ADD DMPE**](../Diameter对端实体/增加Diameter对端实体(ADD DMPE)_72225963.md)<br>或者<br>[**MOD_DMPE**](../Diameter对端实体/修改Diameter对端实体(MOD DMPE)_72345885.md)<br>打开某对端的<br>“对端拥塞避免开关”<br>后，SGSN/MME收到该对端的响应消息且携带DIAMETER_TOO_BUSY错误码时，启动拥塞避免定时器，定时器超时前不向该对端发送请求消息。 |

## 操作的配置对象

- [[UNC@20.15.2@ConfigObject@DMFUNC]] · Diameter配置（DMFUNC）

## 使用实例

设置diameter全局信息，设置重连定时器为30秒，等待回复定时器为14秒，看门狗定时器为15秒，连接侦测次数为3次，消息版本号为1，等待对端响应定时器为12s，不支持重选路由功能，心跳消息故障检测开关为OFF，对端拥塞避免定时器为30秒，运行以下命令:

**SET DMFUNC: RECONNTIMER=30, WAITANSWERTIMER=14, WATCHDOGTIMER=15, CONNDETECTNUM=3, MSGVERNUMBER=1, DIAMPEERRSPTIMER=12, FAILOVER=NO, DWRSW=OFF, CONGTIMER=30;**
