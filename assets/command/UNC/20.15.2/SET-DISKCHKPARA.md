# SET DISKCHKPARA（设置存储检测参数）

## 功能

![](设置存储检测参数（SET DISKCHKPARA）_80543423.assets/notice_3.0-zh-cn_2.png)

该命令为高危命令，须由运维人员设置，另外，dd读写速率，IO时延需根据环境的实际情况进行填写。

本命令用于手动设置部署的节点当前磁盘检测的参数。

- 若需要设置所有节点的磁盘检测参数，需要输入“网元ID”并选择“设置存储检测类型”。
- 若需要设置指定节点的磁盘检测参数，需要输入“网元ID”和“节点名称”，并选择“设置存储检测类型”。

> **说明**
> 该命令仅在Full-stack虚机场景下支持。

## 注意事项

- 该命令仅限系统管理员可以执行。
- 该命令存在系统初始记录，参数的初始设置值如下表：
  | 设置存储检测类型 | 存储异常检测周期（个） | 存储异常检测阈值（%） | IO时延（ms） | dd读写速率（KB/s） |
  | --- | --- | --- | --- | --- |
  | 存储故障 | 3 | 100 | 1000 | 0 |
  | 存储亚健康 | 10 | 50 | 100 | 50 |

## 参数

| **参数标识** | **参数名称** | **参数说明** |
| --- | --- | --- |
| APPID | 网元ID | 可选必选说明：必选参数<br>参数含义：用于指示系统需要设置的网元ID。<br>取值范围：0～65535。<br>默认值：无。<br>配置原则：<br>“网元ID”<br>获取方式为：点击<br>“OM Portal首页”<br>，查看<br>“应用ID”<br>即可。 |
| TYPE | 设置存储检测类型 | 可选必选说明：必选参数<br>参数含义：用于指示系统需要设置指定网元ID下的节点存储检测参数类型。<br>取值范围：<br>- “DiskFault(存储故障)”：节点存储故障的检测参数信息。<br>- “DiskSubHealth(存储亚健康)”：节点存储亚健康的检测参数信息。<br>默认值：无。<br>配置原则：当选择该参数后，节点将列出该设置参数类型对应的可设置存储检测参数。 |
| NODENAME | 节点名称 | 可选必选说明：该参数在<br>“TYPE”<br>配置为<br>“DiskFault(存储故障)”<br>或<br>“DiskSubHealth(存储亚健康)”<br>其中之一时为条件可选参数。<br>参数含义：用于指示系统需要设置哪个节点的存储检测参数。<br>取值范围：不超过128位字符串。<br>默认值：无。<br>配置原则：<br>- 若不输入，则表示设置所有节点的存储检测参数。<br>- 用户可以通过[**DSP NODE**](节点查询（DSP NODE）_71678755.md)命令查看相应的节点名称。 |
| SUBSWICHER | 存储亚健康检测开关 | 可选必选说明：该参数在<br>“TYPE”<br>配置为<br>“DiskSubHealth(存储亚健康)”<br>时为条件可选参数。<br>参数含义：用于设置节点存储亚健康检测开关的状态。<br>取值范围：<br>- “ON(开启存储亚健康检测)”<br>：用于开启节点存储亚健康检测。<br>- “OFF(关闭存储亚健康检测)”：用于关闭节点存储亚健康检测。<br>默认值：无。<br>配置原则：根据业务需要选择开启或者关闭节点存储亚健康检测开关：打开存储亚健康检测开关，则当存储状态超过检测阈值后会检测出存储亚健康并上报<br>“ALM-135644 节点存储亚健康”<br>告警；关闭存储亚健康检测开关，则不会检测存储的亚健康状态。 |
| PERIOD | 存储异常检测周期（个） | 可选必选说明：该参数在<br>“TYPE”<br>配置为<br>“DiskFault(存储故障)”<br>或<br>“DiskSubHealth(存储亚健康)”<br>其中之一时为条件可选参数。<br>参数含义：用于设置节点存储故障或者存储亚健康检测周期，标识检测周期为几个默认检测时长。<br>取值范围：3~30。<br>默认值：无。<br>配置原则：根据业务需要设置存储故障或者存储亚健康检测周期。 |
| THREDHOLD | 存储异常检测阈值（%） | 可选必选说明：该参数在<br>“TYPE”<br>配置为<br>“DiskFault(存储故障)”<br>或<br>“DiskSubHealth(存储亚健康)”<br>其中之一时为条件可选参数。<br>参数含义：用于设置节点存储故障或者存储亚健康检测阈值。<br>取值范围：10~100。<br>默认值：无。<br>配置原则：根据业务需要设置存储故障或者存储亚健康检测阈值：当存储检测超过设置阈值，则会检测到存储故障或者存储亚健康。 |
| AWAITSVCTM | IO时延（ms） | 可选必选说明：该参数在<br>“TYPE”<br>配置为<br>“DiskFault(存储故障)”<br>或<br>“DiskSubHealth(存储亚健康)”<br>其中之一时为条件可选参数。<br>参数含义：用于设置节点存储故障或者存储亚健康检测的IO时延阈值。<br>取值范围：0~2000。<br>默认值：无。<br>配置原则：<br>- 根据业务需要设置存储故障或者存储亚健康检测的IO时延阈值：当存储检测超过设置阈值，则会检测到存储故障或者存储亚健康。<br>- 存储故障检测的“IO时延”参数值必须大于存储亚健康检测的“IO时延”。 |
| DDWRITEREADSPEED | dd读写速率（KB/s） | 可选必选说明：该参数在<br>“TYPE”<br>配置为<br>“DiskFault(存储故障)”<br>或<br>“DiskSubHealth(存储亚健康)”<br>其中之一时为条件可选参数。<br>参数含义：用于设置节点存储故障或者存储亚健康检测的dd读写速率阈值。<br>取值范围：0~1024。<br>默认值：无。<br>配置原则：<br>- 根据业务需要设置存储故障或者存储亚健康检测的dd读写速率阈值：当存储检测dd读写速率小于设置阈值，则会检测到存储故障或者存储亚健康。<br>- 存储故障检测的“dd读写速率”参数值必须小于存储亚健康检测的“dd读写速率”。 |

## 使用实例

1. 设置“网元ID”为“0”的所有节点“存储故障检测”参数信息：
  ```
  %%SET DISKCHKPARA: TYPE=DiskFault, APPID=0;%%
  RETCODE = 0  操作成功

  操作结果如下
  ------------
    设置类型  =  存储故障 
    详细信息  =  操作成功
  (结果个数 = 1)

  ---    END
  ```
2. 设置“网元ID”为“0”，“节点名称”为“10.0.0.1”的“存储亚健康检测”参数信息：
  ```
  %%SET DISKCHKPARA: TYPE=DiskSubHealth, APPID=0, NODENAME="10.0.0.1", DDWRITEREADSPEED=0;%%
  RETCODE = 0  操作成功

  操作结果如下
  ------------
    设置类型  =  存储亚健康 
    详细信息  =  操作成功
  (结果个数 = 1)

  ---    END
  ```
