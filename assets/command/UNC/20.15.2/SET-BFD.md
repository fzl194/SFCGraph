# SET BFD（设置BFD全局属性）

## 功能

![](设置BFD全局属性（SET BFD）_00840937.assets/notice_3.0-zh-cn_2.png)

本命令属于高危命令，操作不当会去使能BFD，导致BFD功能不可用，并且所有历史BFD配置全部删除，请谨慎使用并联系华为技术支持协助操作。

该命令用于设置BFD全局属性。为了减小设备故障对业务的影响，提高网络的可靠性，网络设备需要能够尽快检测到与相邻设备间的通信故障，以便及时采取措施，保证业务继续进行。

## 注意事项

- 该命令执行后立即生效。
- 全局去使能BFD后，BFD所有配置无法恢复。
- 如果有业务在使用BFD功能，有可能导致去使能BFD失败，具体原因请参考提示信息。
- 该命令存在系统初始记录，参数的初始设置值如下表：

| BFDENABLE | DELAYUPTIMER | TOSEXP | MPLSPASSIVE |
| --- | --- | --- | --- |
| FALSE | 0 | 7 | FALSE |

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| BFDENABLE | BFD使能标志 | 可选必选说明：必选参数<br>参数含义：BFD使能标志，控制BFD全局使能或是全局去使能。<br>数据来源：本端规划<br>取值范围：布尔类型，输入格式为“TRUE”或者“FALSE”。<br>默认值：无 |
| DELAYUPTIMER | 会话延迟Up时间（s） | 可选必选说明：可选参数<br>参数含义：在与其它厂商互通的场景中，如果设备整机重启或者是资源重启，则需要将链路切换到备份链路上。待设备整机重启成功或者是资源重启成功后，则需要将流量重新切换到主用链路上。此时主用链路上BFD会话的状态为Up。由于有的厂商处理机制是直接根据BFD会话的状态进行链路切换的，其它厂商设备指向设备的路由可能还没有生成，在这种情况下，可能会导致流量丢失。为避免这种情况的发生，需要BFD会话在建立并协商Up之前延迟一段时间。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～600，单位是秒。<br>默认值：无 |
| TOSEXP | 动态BFD会话报文优先级 | 可选必选说明：可选参数<br>参数含义：报文优先级。根据BFD绑定的业务的重要性，来设置BFD会话的优先级。在系统堵塞的情况下优先发送优先级高的BFD报文。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～7。<br>默认值：无<br>配置原则：取值越大，BFD报文优先级越高。 |
| MPLSPASSIVE | 使能MPLS被动会话 | 可选必选说明：可选参数<br>参数含义：是否使能BFD支持创建MPLS被动会话。如果使能被动动态创建BFD会话功能，在接收到源端发送的携带BFD TLV的LSP请求报文后建立反向BFD会话。<br>数据来源：本端规划<br>取值范围：布尔类型，输入格式为“TRUE”或者“FALSE”。<br>默认值：无 |

```
SET BFD:BFDENABLE=TRUE;
```
