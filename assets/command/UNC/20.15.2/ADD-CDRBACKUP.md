# ADD CDRBACKUP（增加话单备份）

## 功能

**适用NF：NCG**

该命令用于将CG上的原始话单、第一份最终话单或第二份最终话单备份到第三方服务器或本地磁盘上实现数据可靠性备份。从而提高数据的安全性。目前的备份方案有：

- 将话单实时备份到第三方服务器。
- 将历史话单一次性备份到第三方服务器（建议该任务的第三方服务器与已有的实时备份任务的第三方服务器保持一致）。
- 将话单实时备份到本地磁盘上。

仅某些局点有特殊要求，才配置话单备份任务。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为1280。
- 当配置话单备份到本地时，会占用CG上的硬盘空间。CG长时间运行后，可能引发**ALM-82010 磁盘空间不足**告警，若不及时处理，可能导致CG停止接收话单、软交换限呼、话单丢失等问题。因此，为避免话单堆积，请务必设置“目的侧话单文件的保存天数”参数为0～180天，以保证备份的过期话单能及时被删除。
- 建议配置的备份任务数不超过10个，否则会有内存不足的风险。
- CG在单个RU上最多支持创建1024个话单备份任务。
- 配置话单备份任务时，需要在每一个RUID上配置Omi或Bi&Omi的IP地址。“RUID”在客户端的“MML命令行 - UNC”窗口执行[**LST SERVICERUSTATE**](../../../../../平台服务管理/单体服务编排功能管理/系统管理/资源管理/RU信息/查询RU的信息(LST SERVICERUSTATE)_29626965.md)命令来查看。
- 请确保备份的第三方FTP/SFTP服务器支持LIST和DELE命令，否则可能导致CG备份话单异常或者话单文件在第三方服务器的过期删除功能失效。
- 配置话单备份任务后产生的话单文件才会被备份，配置话单备份任务前产生的话单文件不会被备份，但对于自定义源路径话单文件的备份任务不影响。
- 备份目的服务器为“REMOTE(第三方服务器)”时，支持配置“是否一次性备份任务”。当“是否一次性备份任务”配置为“是”时，“话单文件开始时间”和“话单文件结束时间”之间的历史话单文件才会被备份，且不支持自定义源路径话单文件和第二份最终话单文件，不支持上报[**ALM-82033 备份分发话单延时**](../../../../../../../../网络运维/故障处理/UNC告警处理/业务告警/NCG/ALM-82033 备份分发话单延时_02884208.md)告警。执行[**DSP CDRBACKUP**](显示备份任务状态（DSP CDRBACKUP）_51174248.md)命令查询到任务状态为“停止”时，表示历史话单文件备份完成，可执行[**RMV CDRBACKUP**](删除话单备份（RMV CDRBACKUP）_51174245.md)命令删除该任务。
- 在话单备份中配置的话单相关路径为相对路径，在RU上实际保存话单文件的绝对路径为“/opt/CG_VNFC/ *VNFCID* / *RUID* /vrpv8/product/”加上在话单备份中配置的相对路径。其中，“VNFCID”，在客户端的“MML命令行 - UNC”窗口执行[**DSP VNFC**](../../../../../平台服务管理/单体服务平台功能管理/操作维护/配置管理/VNFC信息/显示VNFC（DSP VNFC）_59036773.md)命令来查看。“RUID”，在客户端的“MML命令行 - UNC”窗口执行[**LST SERVICERUSTATE**](../../../../../平台服务管理/单体服务编排功能管理/系统管理/资源管理/RU信息/查询RU的信息(LST SERVICERUSTATE)_29626965.md)命令来查看。
