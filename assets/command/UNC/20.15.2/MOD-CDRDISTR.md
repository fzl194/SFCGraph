# MOD CDRDISTR（修改话单分发）

## 功能

![](修改话单分发（MOD CDRDISTR）_51174256.assets/notice_3.0-zh-cn_2.png)

如果修改了密钥相关参数，需要执行“UPL CDRDISTR”。否则，业务将会断连。

**适用NF：NCG**

该命令用于修改某通道下或所有通道下第二份最终话单的分发规则。

## 注意事项

- 当CG使用PULL方式分发话单时，计费中心取走话单后应立即删除取走的话单，防止保存的话单数量过多导致磁盘空间不足。

- 当前版本不支持Bi口SingleIP功能，请勿开启。
- 该命令执行后立即生效。
- 当配置分发话单到本地时，会占用CG RU上的硬盘空间。CG长时间运行后，可能引发**ALM-82010 磁盘空间不足**告警，若不及时处理，可能导致CG停止接收话单、软交换限呼、话单丢失等问题。为避免话单堆积，设置“目的侧话单文件的保存天数”参数为0～180天，以保证分发的过期话单能及时被删除。
- 当配置了具体通道下的话单分发规则后，先前配置为“All”的话单分发，就变为除具体通道外其他所有通道下的话单分发任务。因此修改先前配置为“All”的话单分发任务只会对除具体通道外其他所有通道生效。
- 当“是否将文件转移至本地路径”参数配置为“YES(是)”时，会将所有成功分发至计费中心（本文档中又称BS（Billing System））的原话单文件（包括压缩文件）压缩至指定路径，同时将原文件删除。“文件转移本地路径”不能设置为已有话单路径。
- CG最多支持创建120个PULL方式的话单分发任务。当修改话单分发任务的“分发方式”为“PULL(PULL方式)”时，需要确认PULL方式话单分发任务未超过允许的最大值。
- CG在单个RU上最多支持创建1024个除PULL方式以外的话单分发任务。
- 当CG使用PULL方式分发话单时，为确保对接安全，请定期修改CG开放给BS的密码，并确保BS同步修改。
- 如果修改了密钥相关参数，需要执行[**UPL CDRDISTR**](上传SFTP密钥文件到BS侧（UPL CDRDISTR）_51174252.md)。否则，业务将会断连。
- 不能将内部分发任务类型修改为非内部分发类型，也不能将非内部分发任务类型修改为内部分发类型。
- 当分发方式不为内部分发类型时，任务名不能配置为“InnerConfig”（不区分大小写）。
- 修改BS侧主用或者备用IP地址时，必须输入登录BS侧的用户名和密码。
- 如果修改ISOMIIP参数，若ISOMIIP参数配置为“YES(是)”时，提供给该用户取话单的IP为Omi的IP地址；若ISOMIIP参数配置为“NO(否)”时，提供给该用户取话单的IP为Bi的IP地址。修改ISOMIIP参数会自动重启FTP或SFTP服务，已经登录客户端的账号会被强制下线。
  > **注意**
  > 当增加PULL(PULL方式)话单分发任务执行报错，并且错误为“RETCODE = 27568 按照密码策略校验密码失败。”时，可执行 [**DSP SERVICE**](../../业务系统管理/业务管理/显示业务状态（DSP SERVICE）_51174328.md) 命令查看进程状态，确认CDM进程全部为激活状态。若已激活，参考“开放给BS侧的密码”参数说明排查输入的密码是否符合密码策略。

## 使用实例

> **说明**
> 使用FTP传输协议方式时，传输数据时不加密，安全性低。使用SFTP传输协议方式时，会对传输数据进行加密，安全性较高。建议和计费中心协商选择SFTP协议。

修改标识为“Distri_1st_scdr_BS1”话单分发任务中CG的用户名和密码及分发文件的定时扫描间隔。示例如下：

```
MOD CDRDISTR: CDRDISTRID="Distri_1st_scdr_BS1", ISSINGLEBI=NO, TRANSMODE=PUSH, AGID="PS_GROUP_2", TPTYPE=FTP, IPTYPE=IPV4, PDESTIPV4="10.11.12.13", USERNAME="scdr", PASSWORD="*****", SRCHFI=120;
```
