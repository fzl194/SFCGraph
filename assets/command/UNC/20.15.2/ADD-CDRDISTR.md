# ADD CDRDISTR（增加话单分发）

## 功能

**适用NF：NCG**

该命令用于配置第二份最终话单的分发任务，包括话单分发的方式、采用的协议、计费中心（本文档中又称BS（Billing System））IP地址等信息。

CG支持PULL和PUSH两种方式分发话单到计费中心（本文档中又称BS（Billing System））。

- PULL方式
  在CG的每对RU上开放第二份最终话单文件的存放目录给计费中心获取。大多数情况下，CG开放第二份最终话单的默认存储目录给计费中心获取。少数情况下，有局点要求在本地磁盘上新建一个目录，将默认存储路径下的第二份最终话单先备份到该目录下，然后开放此目录给计费中心获取。
- PUSH方式
  CG采用FTP或者SFTP协议主动将话单分发到计费中心指定目录下。

不同的分发方式的准备工作，请参见 [表1](#ZH-CN_CONCEPT_0251174254__table_038FF8DB) 。

## 注意事项

- 当CG使用PULL方式分发话单时，为确保正常执行，请在初始化部署完成后执行“[**RST VNFC**](../../../../../平台服务管理/单体服务公共功能管理/系统管理/复位系统/重启系统（RST VNFC）_59103634.md)”命令重新启动系统，使CG默认安全策略生效。
- 当CG使用PULL方式分发话单时，BS取走话单后应立即删除取走的话单，防止保存的话单数量过多导致CG磁盘空间不足。

- 该命令执行后立即生效。
- 该命令最大记录数为4095。
- 当配置分发话单到本地时，会占用CG上的硬盘空间。CG长时间运行后，可能引发“**ALM-82010 磁盘空间不足**”告警，若不及时处理，可能导致CG停止接收话单、话单丢失等问题。为避免话单堆积，设置“目的侧话单文件的保存天数”参数为0～180天，以保证分发的过期话单能及时被删除。
- 当“是否将文件转移至本地路径”参数配置为“YES(是)”时，会将所有成功分发至计费中心的原话单文件（包括压缩文件）压缩至指定路径，同时将原文件删除。“文件转移本地路径”不能设置为已有话单路径。
- CG最多支持创建120个PULL方式的话单分发任务。
- CG在单个RU上最多支持创建1024个除PULL方式以外的话单分发任务。
- 当CG使用PULL方式分发话单时，为确保对接安全，请定期修改CG开放给BS的密码，并确保BS同步修改。
- PUSH方式下，CG支持话单按通道分发话单到计费中心。每对CG至少需要增加一次话单分发任务。
- 配置话单分发任务前需先执行[**ADD IPRESOURCE**](../IP资源/增加IP资源（ADD IPRESOURCE）_51174282.md)增加话单分发采用的IP资源。
- 配置话单分发任务时，则需在每一个RUID上配置Bi或Bi&OMi的IP地址。“RUID”在客户端的“MML命令行 - UNC”窗口执行[**LST SERVICERUSTATE**](../../../../../平台服务管理/单体服务编排功能管理/系统管理/资源管理/RU信息/查询RU的信息(LST SERVICERUSTATE)_29626965.md)命令来查看。
- 增加PUSH或者LOCAL话单分发任务后产生的话单才会被分发，PULL分发不涉及。
- 内部分发任务为默认记录，任务名为“InnerConfig”（不区分大小写），不支持手动新增或删除。
- 在话单分发中配置的话单相关路径为相对路径，在RU上实际保存话单文件的绝对路径为“/opt/CG_VNFC/ *VNFCID* / *RUID* /vrpv8/product/”加上在话单分发中配置的相对路径。其中，“VNFCID”，在客户端的“MML命令行 - UNC”窗口执行[**DSP VNFC**](../../../../../平台服务管理/单体服务平台功能管理/操作维护/配置管理/VNFC信息/显示VNFC（DSP VNFC）_59036773.md)命令来查看。“RUID”，在客户端的“MML命令行 - UNC”窗口执行[**LST SERVICERUSTATE**](../../../../../平台服务管理/单体服务编排功能管理/系统管理/资源管理/RU信息/查询RU的信息(LST SERVICERUSTATE)_29626965.md)命令来查看。 举例说明：在话单分发中配置的“开放给BS的路径”为“./third”，在RU上实际话单分发中配置的“开放给BS的路径”为“/opt/CG_VNFC/2/66/vrpv8/product/third”。
- 当前版本不支持Bi口SingleIP功能，请勿开启。
  > **注意**
  > 当增加PULL(PULL方式)话单分发任务执行报错，并且错误为“RETCODE = 27568 按照密码策略校验密码失败。”时，可执行 [**DSP SERVICE**](../../业务系统管理/业务管理/显示业务状态（DSP SERVICE）_51174328.md) 命令查看进程状态，确认CDM进程全部为激活状态。若已激活，参考“开放给BS侧的密码”参数说明排查输入的密码是否符合密码策略。

## 使用实例

> **说明**
> 使用FTP传输协议方式时，传输数据时不加密，安全性低。使用SFTP传输协议方式时，会对传输数据进行加密，安全性较高。建议和计费中心协商选择SFTP协议。

- 配置计费中心到CG上的第二份最终话单的默认存储路径下取话单。示例如下：
  ```
  ADD CDRDISTR: CDRDISTRID="Distribution_1st", ISSINGLEBI=NO, TRANSMODE=PULL, OTPTYPE=FTP, OUSERNAME="BS", OPASSWORD="*****", CPASSWD="*****", ISOMIIP=NO;
  ```
- 因为“./third”不是第二份最终话单的默认存储路径，因此首先需要配置LOCAL方式下将第二份最终话单分发到“./third”目录下：
  ```
  ADD CDRDISTR: CDRDISTRID="Distribution_1st_local1", ISSINGLEBI=NO, TRANSMODE=LOCAL, AGID="PS_GROUP_2", MNAME="AP67_1", ISCUTOMDISTR=NO, SOURCECHL="All", DESTPATH="./third", TRANSPER=EVERYDAY, CMRRULE=UNCOMPRESS, SRCHFI=60;
  ```
- 再配置PULL方式下，计费中心到“./third”目录下取话单，其中1为相应的VNFC ID，需在SPU上查看：
  ```
  ADD CDRDISTR: CDRDISTRID="Distribution_1st_pull", ISSINGLEBI=NO, TRANSMODE=PULL, OTPTYPE=FTP, ODESTPATH="./third", OUSERNAME="BS1", OPASSWORD="*****", CPASSWD="*****", ISOMIIP=NO;
  ```

