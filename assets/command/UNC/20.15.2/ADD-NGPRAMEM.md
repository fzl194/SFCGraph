# ADD NGPRAMEM（增加PRA位置区成员）

## 功能

**适用NF：AMF**

该命令用于为指定的PRA添加位置区成员。

## 注意事项

- 该命令执行后立即生效。

- 最多可输入10000条记录。
- 在为PRA增加区域成员前首先需要通过ADD NGPRA配置PRA基本信息。
- 单个PRA区域内可配置的位置成员的数量只受AMF整系统PRA位置成员数量限制。
- 当“LOCTYPE”参数选择“TAI”时，产品最大可实现TAC个数为16，当超过限制，即使配置也不生效。
- 当“LOCTYPE”参数选择“GNBID”时，产品最大可实现最大基站个数为64，当超过限制，即使配置也不生效。
- 当“LOCTYPE”参数选择“NRCELLID”时，产品最大可实现最大小区个数为256，当超过限制，即使配置也不生效。
- 区域类型为“HIGH_RAIL_AREA”的PRA区域仅支持配置位置区类型为“TAI”的位置区成员。
- 当按照起始值和结束值的区间配置方式时，一次配置的区域成员的数量是（结束值-起始值+1）。
- GNBIDBEGIN和GNBIDEND需要转换为2进制比较，如果之中一个或者两个的位数在22到32范围内，判断GNBIDBEGIN和GNBIDEND的位数是否相同且等于GNBIDLEN，若不同则报错。
- 两条记录间的MCC相同时，2位MNC与3位MNC的前两个数字不能相同。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 使用实例

为ID为8388616的PRA添加跟踪区成员，TAC为0x900001、0x900002和0x900003，执行如下命令：

```
ADD NGPRAMEM: PRAID=8388616, LOCTYPE=TAI, MCC="123", MNC="45", TACBEGIN="900001", TACEND="900003", DESC="for East Lake Campus";
```
