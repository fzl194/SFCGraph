# ADD M3DE（增加M3UA目的实体）

## 功能

**适用网元：SGSN、MME、SMSF**

该命令用于增加M3UA目的实体，目的实体是从本地实体的角度出发，在本地实体所在的所有信令网络中可见的实体。

## 注意事项

- 该命令执行后立即生效。
- 相同网络指示语（NI）下，M3UA本地实体编码（OPC）与目的实体编码（DPC）相同时，该M3UA实体工作于代理模式。
- 代理模式和转接模式是相对于M3UA组网时SG（信令网关）侧而言的，SG和AS（应用服务器）之间是否是同一点码是用来定义代理和转接方式的一个特征，SG和AS属于一个点码称为代理模式，否则称为转接模式。
- M3UA目地实体个数之和不能超过1280。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| DEX | 目的实体索引 | 可选必选说明：必选参数<br>参数含义：该参数用于指定唯一标识目的实体的索引。<br>数据来源：本端规划<br>取值范围：0~1279<br>默认值：无<br>配置原则：无 |
| LEX | 本地实体索引 | 可选必选说明：必选参数<br>参数含义：该参数用于指定唯一标识本地实体的索引。<br>数据来源：本端规划<br>取值范围：0~63<br>默认值：无<br>配置原则：无<br>说明：该本地实体索引必须在M3UA本地实体表中已经配置，可通过<br>[**LST M3LE**](../M3UA本地实体/查询M3UA本地实体(LST M3LE)_72225993.md)<br>查看。 |
| DPC | 目的实体编码 | 可选必选说明：必选参数<br>参数含义：该参数用于指定该目的实体在No.7信令网中的信令点编码。<br>数据来源：与对端网元协商<br>取值范围：0x1~0x3FFF（对应14位编码）或0x1~0xFFFFFF（对应24位编码）<br>默认值：无<br>配置原则：无<br>说明：- 信令点编码结构与对应的本地实体的信令点编码结构一致； 目的实体编码为16进制数，可以输入0x为前缀。即输入“0xB23”和“B23”均表示实体编码为“0xB23”；<br>- 目的实体编码（DPC）不允许配置为0；<br>- 对于同一个本地实体（由LEX决定）而言，目的实体编码（DPC）唯一。 |
| SLSSM | 链路集选择掩码 | 可选必选说明：可选参数<br>参数含义：该参数用于指定链路集选择掩码。链路集选择掩码用于路由选择，实现到同一个目的实体的多条路由之间的负荷分担。<br>数据来源：整网规划<br>取值范围：<br>- “B0000（B0000）”<br>- “B0001（B0001）”<br>- “B0010（B0010）”<br>- “B0011（B0011）”<br>- “B0100（B0100）”<br>- “B0101（B0101）”<br>- “B0110（B0110）”<br>- “B0111（B0111）”<br>- “B1000（B1000）”<br>- “B1001（B1001）”<br>- “B1010（B1010）”<br>- “B1011（B1011）”<br>- “B1100（B1100）”<br>- “B1101（B1101）”<br>- “B1110（B1110）”<br>- “B1111（B1111）”<br>默认值：<br>“B0000（B0000）”<br>配置原则：无<br>说明：到一个目的实体的路由数目和链路集选择掩码的关系：1条路由，设置为全0；2条路由，设置1个BIT为1；3-4条路由，设置2个BIT为1；5-8条路由，设置3个BIT为1；9-16条路由，设置4个BIT为1； 到同一个目的实体，信令链路集选择掩码和信令链路选择掩码（<br>[**ADD M3LKS**](../M3UA链路集/增加M3UA信令链路集(ADD M3LKS)_72225989.md)<br>）不能够有相同的BIT位设置成1。 |
| DET | 目的实体类型 | 可选必选说明：必选参数<br>参数含义：该参数用于指定该目的实体在M3UA链路中的应用模式。应用模式确定了目的实体类型。<br>数据来源：整网规划<br>取值范围：<br>- “ASP（应用服务器进程）”<br>- “SGP（信令网关进程）”<br>- “IPSP（IP服务进程）”<br>- “SP（信令点）”<br>默认值：无<br>配置原则：<br>- 当本地实体为ASP时，目的实体类型可以为“SP（信令点）”，“SGP（信令网关进程）”。<br>- 当本地实体为SGP时，目的实体类型可以为“SP（信令点）”，“ASP（应用服务器进程）”。<br>- 当本地实体为IPSP时，目的实体类型必须是“IPSP（IP服务进程）”。 |
| ADJF | 相邻标记 | 可选必选说明：可选参数<br>参数含义：该参数用于指定SGSN与对端目的实体之间是否存在直达的M3UA信令路由。<br>数据来源：整网规划<br>取值范围：<br>- “FALSE（否）”<br>- “TRUE（是）”<br>默认值：<br>“TRUE（是）”<br>配置原则：无<br>说明：当对端信令设备与SGSN之间存在着直达的M3UA信令路由时，必须将该信令设备定义为相邻目的实体； 当对端信令设备与SGSN之间的No.7信令消息需要经过信令网关转发时，必须将该信令设备定义为非相邻目的实体。 |
| RC | 路由上下文 | 可选必选说明：可选参数<br>参数含义：该参数用于指定目的实体的路由上下文。<br>数据来源：整网规划<br>取值范围：0~4294967295<br>默认值：无<br>配置原则：无<br>说明：- 不输入时为无效值即0xFFFFFFFF，显示为NULL。<br>- M3DE路由上下文的配置配合远端相应M3LE的路由上下文，当远端上的本地实体的网络指示语和实体编码相同时，远端用不同的路由上下文来区分不同的本地实体。 |
| PVER | M3UA协议版本 | 可选必选说明：可选参数<br>参数含义：该参数用于指定M3UA目的实体使用的是哪个版本的协议实现的。<br>数据来源：整网规划<br>取值范围：<br>- “DRAFT10（支持DRAFT10）”<br>- “RFC3332（支持RFC3332）”<br>默认值：<br>“RFC3332（支持RFC3332）”<br>配置原则：无<br>说明：M3UA本地实体（SGSN）默认的协议版本为RFC3332，当对端为DRAFT10时，本端能自动适配。 |
| DEN | 目的实体名 | 可选必选说明：可选参数<br>参数含义：该参数用于标识目的实体。<br>数据来源：与对端网元协商<br>取值范围：长度不超过32的字符串<br>默认值：<br>“noname”<br>配置原则：无 |
| STPF | 信令转接点功能 | 可选必选说明：可选参数<br>参数含义：该参数用于指定该信令点是否支持信令转接功能。<br>数据来源：整网规划<br>取值范围：<br>- “DISABLE(否)”<br>- “ENABLE(是)”<br>默认值：<br>“DISABLE(否)”<br>配置原则：无 |
| NMD | 网络模式 | 可选必选说明：可选参数<br>参数含义：该参数用于指定该信令点的网络模式是否为独占信令。<br>数据来源：整网规划<br>取值范围：<br>- “INCLUSIVE(非独占信令点)”<br>- “EXCLUSIVE(独占信令点)”<br>默认值：<br>“EXCLUSIVE(独占信令点)”<br>配置原则：无<br>说明：独占模式中，一个NI、OPC、DPC组合对应于多个处理AS，根据业务来划分不同的处理AS。非独占模式中，一个NI、OPC、DPC组合只对应于一个处理AS。目前<br>UNC<br>仅支持独占模式。 |

## 操作的配置对象

- [[UNC@20.15.2@ConfigObject@M3DE]] · M3UA目的实体（M3DE）

## 使用实例

增加M3UA目的实体，DEX为1，LEX为1，DPC为“0x123”，DET为“SGP”，DEN为“TEST”,STPF为“DISABLE”,NMD为“EXCLUSIVE”：

```
ADD M3DE: DEX=1, LEX=1, DPC="123", DET=SGP, DEN="TEST", STPF=DISABLE, NMD=EXCLUSIVE;
```
