# SET L2TPKEY（设置L2TP加密参数）

## 功能

**适用NF：PGW-C、SMF、GGSN**

该命令用于配置对L2TP业务中的敏感信息在N4接口传输时进行加密所使用的参数。

## 注意事项

- 该命令执行后立即生效。

- 加密开关在默认情况下设置为DISABLE，即明文携带L2TP私有信元，存在一定安全风险，建议和对端UPF同时开启加密功能。

- 系统部署完成后，已经存在初始记录，参数的初始记录值如下表：

| ENSWITCH | KEYCHANGE | TIMELEN |
| --- | --- | --- |
| DISABLE | NEVER | 1 |

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| ENSWITCH | 加密开关 | 可选必选说明：可选参数<br>参数含义：该参数用于控制是否启用加密功能。<br>数据来源：全网规划<br>取值范围：<br>- DISABLE（不使能）<br>- ENABLE（使能）<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST L2TPKEY查询当前参数配置值。<br>配置原则：无 |
| PW | 密钥口令 | 可选必选说明：该参数在"ENSWITCH"配置为"ENABLE"时为条件必选参数。<br>参数含义：该参数用于指定生成加解密密钥所需的口令。<br>数据来源：全网规划<br>取值范围：Pwd，取值范围是1~432。密码类型。明文输入密钥口令时，输入长度范围为16~30；只能输入0~9，A~F，不区分大小写。加密输入时，输入长度范围为1~432。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST L2TPKEY查询当前参数配置值。<br>配置原则：<br>该配置值需与UPF配置保持一致，否则将导致L2TP业务失败。 |
| KEYCHANGE | 密钥变更 | 可选必选说明：该参数在"ENSWITCH"配置为"ENABLE"时为条件必选参数。<br>参数含义：该参数用于控制是否定期变更密钥。<br>数据来源：全网规划<br>取值范围：<br>- “NEVER（不变更）”：表示不变更密钥。<br>- “BYTIME（时长）”：表示按时长进行密钥变更，系统根据配置的时长间隔重新生成新的密钥用于加密。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST L2TPKEY查询当前参数配置值。<br>配置原则：无 |
| TIMELEN | 时长(分钟) | 可选必选说明：该参数在"KEYCHANGE"配置为"BYTIME"时为条件必选参数。<br>参数含义：该参数用于指定变更密钥的触发条件，当当前所使用的密钥满足配置的变更条件时，系统进行密钥变更。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是1~65535，单位是分钟。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST L2TPKEY查询当前参数配置值。<br>配置原则：<br>配置值过小时，可能导致系统频繁变更密钥，频繁变更密钥对系统性能消耗较大。建议配置时长大于60分钟。 |
| CFMPW | 确认密钥口令 | 可选必选说明：该参数在"ENSWITCH"配置为"ENABLE"时为条件必选参数。<br>参数含义：该参数用于确认密钥口令。<br>数据来源：全网规划<br>取值范围：Pwd，取值范围是1~432。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST L2TPKEY查询当前参数配置值。<br>配置原则：<br>配置需要与密钥口令完全一致，口令才会生效。 |

## 操作的配置对象

- [[UNC@20.15.2@ConfigObject@L2TPKEY]] · L2TP加密配置（L2TPKEY）

## 使用实例

设置加密开关为关闭：

```
SET L2TPKEY: ENSWITCH=DISABLE;
```
