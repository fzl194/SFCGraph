# SET QCI2ARP（设置标准QCI到ARP的映射规则）

## 功能

**适用NF：PGW-C、SMF**

该命令用于配置ARP和标准QCI的对应关系，从而通过标准QCI来映射出相应的承载ARP取值。

## 注意事项

- 该命令执行后只对新激活用户生效。

- 系统部署完成后，已经存在初始记录，参数的初始记录值如下表：

| QCI | ARPPCI | ARPPL | ARPPVI |
| --- | --- | --- | --- |
| 1 | 0 | 7 | 0 |
| 2 | 0 | 7 | 0 |
| 3 | 0 | 7 | 0 |
| 4 | 0 | 7 | 0 |
| 5 | 0 | 3 | 0 |
| 6 | 1 | 11 | 0 |
| 7 | 1 | 11 | 0 |
| 8 | 1 | 11 | 0 |
| 9 | 1 | 11 | 0 |
| 65 | 0 | 3 | 1 |
| 66 | 0 | 7 | 0 |
| 69 | 0 | 3 | 1 |
| 70 | 0 | 6 | 0 |

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| QCI | 标准QCI | 可选必选说明：必选参数<br>参数含义：该参数用于指定标准QCI。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是1~9，65~66，69~70。<br>默认值：无。<br>配置原则：无 |
| ARPPCI | ARP的可抢占能力 | 可选必选说明：可选参数<br>参数含义：该参数用于指示资源限制时，业务是否可以抢占更低优先级别的业务资源。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是0~1。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST QCI2ARP查询当前参数配置值。<br>配置原则：<br>参数值为0时代表可抢占其他低优先级业务，1为不可抢占。 |
| ARPPL | ARP的优先级别 | 可选必选说明：可选参数<br>参数含义：该参数表示ARP的优先级别。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是1~15。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST QCI2ARP查询当前参数配置值。<br>配置原则：无 |
| ARPPVI | ARP的被抢占能力 | 可选必选说明：可选参数<br>参数含义：该参数表示资源限制时，业务资源是否可以被更高优先级别的业务抢占。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是0~1。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST QCI2ARP查询当前参数配置值。<br>配置原则：<br>参数值为0时代表可被高优先级业务抢占，1为不可被抢占。 |

## 操作的配置对象

- [标准QCI到ARP的映射规则（QCI2ARP）](configobject/UNC/20.15.2/QCI2ARP.md)

## 使用实例

配置ARP和标准QCI的对应关系：

```
SET QCI2ARP:QCI=9,ARPPCI=0,ARPPL=1,ARPPVI=1;
```
