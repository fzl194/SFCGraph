# SET DNS（设置DNS运行参数）

## 功能

**适用网元：SGSN、MME**

该命令用于设置DNS域名解析流程相关参数，如发送查询报文后等待服务器应答的时长、超时重发次数等参数。DNS解析器（SPP进程上负责域名解析的实体）向DNS服务器发送域名解析请求，获取对应的IP地址信息的时候会依照这些参数进行处理。

同时，DNS探测开关、DNS探测间隔、缺省探测域名等参数也用于服务器状态探测。当DNS探测开关打开时，DNS解析器会定期向已配置的DNS服务器发送探测查询报文，并根据响应状况更新服务器状态，以避免解析域名时向一个已经故障的服务器发送查询请求报文。

系统每隔3s（缺省）发送一次探测报文以检测DNS服务器是否故障。不管DNS服务器是否故障，连续3次（缺省故障门限）DNS服务器没有响应，系统就会上报DNS服务器对应的DNS链路故障告警。

DNS服务器是否故障完全由探测报文确定，与DNS解析请求是否成功无关。发送间隔、故障门限等可通过此命令进行配置。

## 注意事项

- 该命令执行后立即生效。
- DNS探测开关DDS的默认值为打开。如果修改为关闭，系统将无法识别与DNS服务器之间的UDP链路状态，认为UDP链路状态都是正常的。
- DNS模块实现了两级cache，即一级cache，二级cache。一级cache缓存最近查询成功的域名信息，二级cache可以缓存最近查询成功和查询失败的域名信息。 只有在一级cache查询失败后，才会向二级cache发起查询。

## 使用实例

1.设置DNS运行参数： “DNS探测开关” 打开； “DNS探测间隔(s)” 为10秒； “故障判定阈值” 设置为5； “持续失败最大时长（s）” 为0，即域名持续解析失败时允许继续向DNS服务器查询该域名； “二级Cache提前刷新时间（s）” 为0，即二级Cache不提前刷新， “非5G用户是否选择到融合的PGW-C/SMF” 为YES：

SET DNS: DDS=ON, DDI=10, SFT=5, FAILTM=0, RCUPDTM=0, NON5GSEL=YES;

2.现网在割接、网络改造、地址重规划等场景下，会在DNS Server上修改TAC/RAC/APN域名的解析记录。若需要使用新的解析记录，则需要等待本地缓存记录老化或者手工清理缓存（ [**CLR DNSC**](../../../系统管理/DNS维护管理/清除DNS Cache/清除DNS缓存(CLR DNSC)_72345945.md) ）后才能生效。为了防止手动清理存在遗漏的情况，可以通过配置提前老化功能，加快本地缓存记录的老化。例如，希望本地新增缓存记录在3个小时内到服务器刷新一次，可以使用如下命令：
