# ADD NGIPV4DNSH（增加IPv4 DNS Hostfile记录）

## 功能

**适用NF：AMF**

该命令用于配置网元接口所对应的IPv4地址信息，该信息用于网元选择时DNS查询功能。

如果配置了多个IP地址，域名解析结果中将会根据“PRIORITY（优先级）”和“WEIGHT（权重）”进行先后次序排列：

- 优先级别高的排前面。
- 同一优先级别的再按照权重来选择，权重越大，则排在前面的概率越高。

## 注意事项

- 该命令执行后立即生效。

- 该表中最多能配置1024个不同主机名。每个主机名最多配置8个IP地址区间，每个区间对应一条记录。相同主机名不能配置重复的IP地址。
- 同一域名对应的IP地址信息，必须全部配置在本地或者全部配置在DNS服务器上。
- 一个主机名最多能对应64个IP地址，且64个地址中至少有一个必须配置为有效地址。
- 同一主机名只能有唯一主机名索引。
- 该命令执行后，可能导致HTTP的FQDN链路信息地址发生变更，建议通过执行CHK SBILINKFQDNIP，核查HTTP的链路信息。

- 最多可输入8192条记录。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 使用实例

网络升级过程中，网络新增一个MME网元，该网元对应的域名配置IPv4型的地址，具体操作为：增加一个主机名索引值为2，主机名为HUAWEI1.COM.GTP.APN.EPC.MNC123.MCC456.3GPPNETWORK.ORG，地址区间号为SECTION1，IPv4地址1为10.10.10.10，优先级1为10，权重1为100。

```
ADD NGIPV4DNSH: HSINDEX=2, HOSTNAME="HUAWEI1.COM.GTP.APN.EPC.MNC123.MCC456.3GPPNETWORK.ORG", ADDRSECTION=SECTION1, IPV4ADDR1="10.10.10.10", PRIORITY1=10, WEIGHT1=100;
```
