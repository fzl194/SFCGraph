# TST DNS（测试DNS通信）

## 功能

**适用网元：SGSN、MME**

该命令用于测试指定域名能否被解析。

## 注意事项

- 该命令执行后立即生效。
- 本命令进行的DNS查询动作，不受[**ADD DNSQ**](../DNS查询管理/增加DNS查询控制参数(ADD DNSQ)_26305702.md)配置的相关限制（如:查询类型，查询顺序），只受本命令自身的参数控制。
- 本命令只是测试用户输入的某个指定域名能否被正确的解析，但是解析的域名是否符合流程预期，通过本命令的输出结果无法判断。所以，如果发生由于DNS查询失败导致业务流程失败，而TST DNS能正常解析对应域名的情况，请参考下表，检查DNS NAPTR查询结果中“网元类型”、“接口类型”和“协议类型”是否与业务流程预期一致。AAAA/A查询不涉及如下检查。

>

>
> [3] 默认情况下，MME选择Gn接口类型；如果软参Byte1Bit1设置为1，MME则会优选S3接口类型，如果S3接口查询失败，再选择Gn接口类型。
