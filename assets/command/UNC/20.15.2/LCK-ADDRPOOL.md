# LCK ADDRPOOL（锁定/解锁地址池）

## 功能

**适用NF：PGW-C、SMF、GGSN**

该命令用来锁定或解锁指定地址池，当地址池计划不再使用时，可以通过本命令将地址池锁定，保证地址池中的地址不会再分配给终端用户。等到该地址池中的所有地址都释放后，即可删除或修改。

## 注意事项

- 该命令执行后立即生效。

- 该命令仅适用于本地或DHCP类型的地址池。
- 如果先锁定整个地址池，再执行锁定或者解锁某个地址段操作时，执行失败。
- 如果先执行锁定某个地址段命令，再执行锁定地址池命令，整个地址池均被锁定。
- 如果某个地址池已经被锁定，执行LCK ADDRPOOL再次锁定该地址池，执行成功。
- 如果某个地址池属于解锁状态，执行LCK ADDRPOOL再次解锁该地址池，执行成功。
- 如果先执行LCK SECTION锁定某个地址段，再执行LCK ADDRPOOL解锁整个地址池，命令执行后，地址池中的地址段全部解锁。
- 如果地址池中有地址处于使用状态（通过DSP POOLUSAGE查询），则该地址池不能被删除，所以当要删除地址池时，可以首先将该地址池锁定，等地址池中所有的地址都回收以后，就可以删除该地址池了。
- 如果地址正在通过ACT RECYCLE回收，其所在的地址池无法进行解锁操作。地址回收完毕后，才能执行解锁操作。
- 地址池被锁之后，地址池中的地址将不会分配给用户，如果绑定锁定地址池的APN下地址不足，会导致用户因为无地址接入失败。地址段被锁之后，地址段中的地址将不分配给用户，但不影响地址池中的其他地址段。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| POOLNAME | 地址池名称 | 可选必选说明：必选参数<br>参数含义：该参数用于指定地址池的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是1~79。不支持空格及特殊字符“#”、“$”和“&”等，由“-”、“_”、数字、字母和“.”组成，不能以“.”开头或结尾，不区分大小写。<br>默认值：无<br>配置原则：无 |
| ISLOCKED | 地址池锁定标志 | 可选必选说明：必选参数<br>参数含义：该参数用于指定地址池锁定标志。<br>数据来源：本端规划<br>取值范围：<br>- UNLOCK（解锁）<br>- LOCK（锁定）<br>默认值：无<br>配置原则：<br>- 使用LCK ADDRPOOL命令锁定地址池后，命令会立即起作用，该地址池将不会再有地址被分配出去，不影响之前分配的地址。<br>- 使用LCK ADDRPOOL命令解锁地址池之后，该地址池立即可以正常分配地址。 |

## 使用实例

锁定地址池“pool1”：

```
LCK ADDRPOOL:POOLNAME="pool1",ISLOCKED=LOCK;
```
