# STR OFFLOADAMF（启动AMF迁移任务）

## 功能

![](启动AMF迁移任务（STR OFFLOADAMF）_09653235.assets/notice_3.0-zh-cn_2.png)

启动AMF的用户迁移任务，如果迁移速率设置不合理，可能导致本系统过载及对周边网元造成冲击。

**适用NF：AMF**

此命令用于启动AMF的用户迁移任务。

当AMF Pool中的某个AMF出现过载、AMF需要进行迁移或升级、网络负载出现不均衡的情况下，可以启动用户迁移任务，能够及时调整网络运行状况，实现资源合理利用，并有效减少用户接入损耗和业务中断。

此命令执行后，任务状态变为“迁移中”或“已结束”，迁移类型为“迁移全部用户”、“迁移百分比用户”、“迁移部分用户”的迁移任务可执行“DSP OFFLOADAMF”命令查询AMF迁移任务状态及待迁移用户数和已迁移用户数等进度信息，执行“STP OFFLOADAMF”命令停止AMF迁移任务，迁移任务状态变为“已停止”。

## 注意事项

- 该命令执行后立即生效。

- AMF不支持多个迁移任务并行进行。
- 如果有迁移任务状态为“迁移中”，执行此命令不会发起新的任务。
- 迁移类型为“迁移全部用户”、“迁移百分比用户”、“迁移部分用户”：如果已迁移用户数已经达到待迁移用户数或AMF当前处于注册态的5G用户数已经为0，则停止迁移任务。
- 启动迁移任务时会根据待迁移用户数/速率计算预估迁移时长，预估迁移时长加预留的保护时间（30分钟）后，如果迁移任务还没有停止，就强制停止迁移任务。
- 迁移类型为“指定IMSI”：IMSI参数指定所要迁移单个用户的国际移动用户标识，该类型迁移任务不支持“DSP OFFLOADAMF”和“STP OFFLOADAMF”命令，可通过用户跟踪等手段查看迁移进度。
- 不支持容灾用户的迁移。如果迁移类型为指定IMSI，若命令执行结果返回“记录不存在”，表示该用户是容灾用户或者该用户在系统中不存在。
- 当启动AMF的用户迁移任务，CPU和内存使用率会升高，和周边网元交互消息量增大，待迁移完成后，系统会恢复正常。
- 如果待迁移的用户为eDRX用户且用户不在eDRX寻呼窗口内，则系统会启动延迟寻呼定时器延迟迁移，这会导致迁移时长延长，最大延迟时间为60分钟。
- eDRX用户迁移时，源侧AMF会在N14接口通过私有信元携带eDRX周期和寻呼时间窗口时长（是否携带私有信元受SET AMFSBICMPT命令的SNDPRVEDRXINFO参数控制），目标侧AMF会保存并使用eDRX周期和寻呼时间窗口。如果从支持eDRX功能的AMF向不支持eDRX功能的低版本AMF迁移，会导致eDRX用户迁移到目标AMF后，下行寻呼成功率降低，直到用户在目标AMF上重新进行一次注册流程并协商关闭eDRX功能后，寻呼成功率才能恢复正常。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| TYPE | 迁移类型 | 可选必选说明：可选参数<br>参数含义：该参数用来指定AMF的迁移类型。<br>数据来源：本端规划<br>取值范围：<br>- “ALL（全部用户）”：全部用户<br>- “PART（部分用户）”：部分用户<br>- “RATE（百分比用户）”：百分比用户<br>- “IMSI（指定IMSI）”：指定IMSI<br>默认值：ALL<br>配置原则：<br>具体选择哪种迁移类型由具体的运营场景决定，比如:<br>1）AMF Pool中的某个AMF需要执行升级、退网等操作，需要将该AMF上的所有用户迁移到Pool内其它AMF，则选择“ALL”；<br>2）AMF Pool中的某个AMF过载，需要将该AMF上的部分用户迁移到Pool内的其它AMF，或者AMF Pool中实例化了一个新的AMF，需要将其它AMF上的存量用户迁移一部分到该新AMF，则选择“PART”或者“RATE”；<br>3）测试场景下，运维人员需要将指定IMSI的拨测用户迁移到目标AMF以便进行业务观测，则选择“IMSI”。 |
| OFFLOADNUM | 迁移总数 | 可选必选说明：该参数在"TYPE"配置为"PART"时为条件必选参数。<br>参数含义：该参数用来指定AMF迁移用户总数。<br>如果输入的数值大于当前系统用户数，则为迁移当前系统用户数。系统用户数可通过DSP NGUSERNUM命令查询。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是0~17000000。<br>默认值：无<br>配置原则：无 |
| OFFLOADRATE | 迁移比率(百分比) | 可选必选说明：该参数在"TYPE"配置为"RATE"时为条件必选参数。<br>参数含义：该参数用来指定AMF待迁移用户数占整系统用户数的比率。<br>如果输入100，系统按照“全部用户”处理。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是1~100，单位是百分比。<br>默认值：无<br>配置原则：无 |
| IMSI | IMSI | 可选必选说明：该参数在"TYPE"配置为"IMSI"时为条件必选参数。<br>参数含义：该参数用于指定待迁移用户的IMSI。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是5~15。只允许输入十进制数字（0-9）。<br>默认值：无<br>配置原则：无 |
| RATE | 迁移速率(千/分钟) | 可选必选说明：该参数在"TYPE"配置为"ALL"、"PART"、"RATE"时为条件可选参数。<br>参数含义：该参数用于指定AMF用户迁移的速率。<br>迁移速率的设置需要考虑本AMF，目标AMF、NG-RAN、UDM、PCF等周边NF的负载情况，不宜设置过大，否则可能导致上述NF过载。<br>如果不输入，按照每分钟迁移当前系统用户数的1%的速度进行迁移。<br>注意，由于实现原因，AMF在系统内部分发迁移任务时迁移速率按照向下取整，故存在实际迁移速率略微低于本设置值的可能。<br>实际的最大迁移速度会产生100用户/分钟的误差，即可能会略低于该设置值。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围是1~500。<br>默认值：无<br>配置原则：无 |
| TAMF | 目标AMF名称 | 可选必选说明：可选参数<br>参数含义：该参数用于指定用户需要迁移到的目标AMF的名称。（和TAMFIPADDR共同用于配置目标AMF）。<br>- TAMF获取方式：如果目标AMF为UNC，则在目标AMF上执行LST AMFINFO查询AMF名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是0~150。可输入的字符有字母、十进制数字、“-”和“.”，并且开头和结尾只能是数字或者字母。字母大小写不敏感。<br>默认值：无<br>配置原则：<br>当同时配置“目标AMF名称”和“目标AMF IP信息”时，使用“目标AMF IP信息”作为迁移用户的手段。<br>如果只配置“目标AMF名称”和“目标AMF IP信息”其中一个参数时，以该配置参数选择目标AMF。<br>如果两个参数都不配置，表示不指定目标AMF，相关业务迁出时，系统按照AMF Set中各AMF在NRF中注册的优先级和能力进行目标AMF选择。 |
| TAMFIPADDR | 目标AMF IP信息 | 可选必选说明：可选参数<br>参数含义：该参数用于指定用户需要迁移到的目标AMF的ApiRoot的IP信息。（和TAMF共同用于配置目标AMF）。<br>输入项格式为scheme + IP:PORT格式，当“是否启用TLS”为“YES”时，scheme为“https://”，否则为“http://”。以IPv4举例：http://10.2.3.4:3060或者https://10.2.3.4:3060。<br>针对IPv6地址输入时需要添加“[ ]”，例如当“是否启用TLS”为“YES”，“端口号”为“80”，“IPv6地址”为“fc00:2000:3000:fc00:2000::15”时，“TAMFIPADDR”参数为“https://[fc00:2000:3000:fc00:2000::15]:80”。<br>TAMFIPADDR获取方式：如果目标AMF为UNC，则在目标AMF上执行LST HTTPLE查询Server端是否启用TLS、IP和端口信息。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围是0~150。可输入的字符有大小写字母、十进制数字、“:”、“.”、“[”、“]”和“/”。<br>默认值：无<br>配置原则：<br>当同时配置“目标AMF名称”和“目标AMF IP信息”时，使用“目标AMF IP信息”作为迁移用户的手段。<br>如果只配置“目标AMF名称”和“目标AMF IP信息”其中一个参数时，以该配置参数选择目标AMF。<br>如果两个参数都不配置，表示不指定目标AMF，相关业务迁出时，系统按照AMF Set中各AMF在NRF中注册的优先级和能力进行目标AMF选择。 |
| WAITHLCOMUE | 等待高延迟UE迁移完成 | 可选必选说明：可选参数<br>参数含义：该参数用于指定AMF是否等待处于空闲态使用eDRX的用户都被迁移完成之后才结束迁移任务。<br>如果本参数配置为“NO(否)”，迁移结束时可能会存在部分eDRX用户未迁移。<br>如果本参数配置为“YES(是)”则迁移的时长会延迟（eDRX用户最大延迟60分钟）。如需停止迁移，请执行STP OFFLOAD。<br>数据来源：全网规划<br>取值范围：<br>- “NO（否）”：否<br>- “YES（是）”：是<br>默认值：无<br>配置原则：<br>如果AMF已开启eDRX功能，建议迁移时将该参数设置为“YES(是)”。 |

## 使用实例

启动AMF迁移任务，迁移全部用户，但不指定目标AMF，不指定迁移速率，执行如下命令：

```
STR OFFLOADAMF: TYPE=ALL;
```
