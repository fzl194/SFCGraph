# MOD SCTPPARA（修改SCTP协议参数）

## 功能

**适用网元：SGSN、MME、AMF** 、 **SMSF**

该命令用于修改SCTP偶联配置的协议参数表中指定的记录。

## 注意事项

- 该命令不能修改SCTP协议参数索引。
- 该命令执行后，“RTO初始值”、“RTO ALPHA值”、“RTO BETA值”、“最大入流个数”和“最大出流个数”需要偶联重新建链才能生效，其他参数在与以上参数的旧值符合“配置原则”的前提下，才能动态生效。（例如，原来的“RTO最小值”和“RTO初始值”分别为5和8，修改后的值分别为9和12。因为“RTO初始值”的修改不能实时生效，链路的当前“RTO初始值”仍为8，而要修改的“RTO最小值”为9，大于当前生效的“RTO初始值”8，不符合“配置原则”，此次修改不能实时生效，需要重新建链才生效。）
- 需要根据现网网络状况、接口类型，与对端网元协商合适的SCTP协议参数。
- Diameter链路不支持ADLER32校验和算法。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

| SCTPPARAINDEX | SCTP协议参数索引 | 可选必选说明：必选参数<br>参数含义：该参数在系统范围内部唯一标识一条SCTP协议参数。<br>数据来源：本端规划<br>取值范围：0~65534<br>默认值：无<br>配置原则：该参数在系统范围内唯一。建议从0开始按顺序取值。 |

| RTOALPHA | RTO ALPHA值（%） | 可选必选说明：可选参数<br>参数含义：该参数用于计算与更新RTO定时器的时长，具体使用方法参考RFC2960协议的相关描述。<br>数据来源：本端规划<br>取值范围：1%~100%<br>默认值：无<br>配置原则：此参数值必须小于<br>“RTOBETA(RTO BETA值)”<br>。<br>说明：正常情况下，不需更改此参数。取值为<br>“12”<br>表示<br>“RTOALPHA(RTO ALPHA值)”<br>为0.12。 |
| RTOBETA | RTO BETA值（%） | 可选必选说明：可选参数<br>参数含义：该参数用于计算与更新RTO定时器的时长，具体使用方法参考RFC2960协议的相关描述。<br>数据来源：本端规划<br>取值范围：1%~100%<br>默认值：无<br>配置原则：此参数值必须大于<br>“RTOALPHA(RTO ALPHA值)”<br>。<br>说明：正常情况下，不需更改此参数。取值为<br>“25”<br>表示<br>“RTOALPHA(RTO ALPHA值)”<br>为0.25。 |
| HBINTERVAL | 心跳间隔（ms） | 可选必选说明：可选参数<br>参数含义：该参数用于定义本端向远端IP地址发送心跳消息的周期。当偶联SCTP在指定时间内没有将数据发送至远端IP地址，则偶联SCTP会向该IP地址发送心跳消息，检查远端IP地址是否激活。<br>数据来源：本端规划<br>取值范围：10ms~60000ms<br>默认值：无 |
| MAXASSOCRETR | 偶联最大重发次数 | 可选必选说明：可选参数<br>参数含义：该参数用于指定在一个SCTP偶联上允许消息连续重传的最大次数。如果SCTP偶联消息连续重发次数大于本参数，则系统认为对端SCTP端点不可达，于是会自动放弃SCTP偶联，并通知SCTP用户层该SCTP偶联不可用。<br>数据来源：本端规划<br>取值范围：1~254<br>默认值：无<br>配置原则：正常情况下，该参数取值不得大于所有偶联远端地址的最大路径重发次数总和，并且不得小于最大路径重发次数。 |
| MAXPATHRETR | 路径最大重发次数 | 可选必选说明：可选参数<br>参数含义：该参数用于指定在一个SCTP偶联上允许向某个特定的IP地址进行消息连续重传的最大次数。 如果某个SCTP偶联上向某个特定的IP地址进行消息连续重传的最大次数大于最大路径重发次数，则认为对端IP地址不可达，此时，如果数据配置了多个对端目的IP，本局将自动将当前使用的目的IP切换为下一个可用的IP地址，需要指出的是，如果本局优先使用主用IP，则当主用IP从不可用状态变为可用状态时，该SCTP偶联将自动切换到主用IP上。<br>数据来源：本端规划<br>取值范围：1~254<br>默认值：无 |
| MAXINSTREAMS | 最大入流个数 | 可选必选说明：可选参数<br>参数含义：该参数用于修改最大的入流个数。<br>数据来源：本端规划<br>取值范围：2~17<br>默认值：无 |
| MAXOUTSTREAMS | 最大出流个数 | 可选必选说明：可选参数<br>参数含义：该参数用于修改最大的出流个数。<br>数据来源：本端规划<br>取值范围：2~17<br>默认值：无 |
| CNGLEVELNO | 拥塞结束门限 | 可选必选说明：可选参数<br>参数含义：该参数用于指定当SCTP偶联进入拥塞状态以后，SCTP偶联的信令负荷解除拥塞状态的门限，为百分数。门限是指信令发送缓冲区被占用的百分比。当SCTP偶联进入拥塞状态后，如果后续已经使用的发送缓冲区与总发送缓冲区的比值小于该参数指定的门限时，则系统将解除SCTP偶联的拥塞状态。<br>数据来源：本端规划<br>取值范围：1~100<br>默认值：无<br>配置原则：此参数值必须小于<br>“CNGLEVELLOW(拥塞最低门限)”<br>。 |
| CNGLEVELLOW | 拥塞最低门限 | 可选必选说明：可选参数<br>参数含义：该参数用于指定SCTP偶联的信令负荷处于低拥塞状态时的门限，为百分数。当发送缓存与总发送缓存的比值大于该参数指定的门限并且小于<br>“CNGLEVELHIGH(拥塞最高门限)”<br>时，偶联处于低拥塞状态（从无拥塞状态算起）。<br>数据来源：本端规划<br>取值范围：1~100<br>默认值：无<br>配置原则：此参数值必须大于<br>“CNGLEVELNO(拥塞结束门限)”<br>并且小于<br>“CNGLEVELHIGH(拥塞最高门限)”<br>。 |
| CNGLEVELHIGH | 拥塞最高门限 | 可选必选说明：可选参数<br>参数含义：该参数用于指定SCTP偶联的信令负荷处于高拥塞状态时的门限，为百分数。当遇忙发送缓存与总发送缓存的比值大该参数设定门限时，偶联处于高拥塞状态。<br>数据来源：本端规划<br>取值范围：1~100<br>默认值：无<br>配置原则：此参数值必须大于<br>“CNGLEVELLOW(拥塞最低门限)”<br>。 |
| CHKSUMTX | 发送消息时是否计算校验和 | 可选必选说明：可选参数<br>参数含义：该参数用于指定本端SCTP端点发送SCTP消息时的计算校验和是否可用。<br>数据来源：本端规划<br>取值范围：<br>- “NO(否)”：表示本端SCTP端点发送SCTP消息时，不支持计算SCTP校验和。<br>- “YES(是)”：表示本端SCTP端点发送SCTP消息时，支持计算SCTP校验和。<br>默认值：无<br>配置原则：建议值为<br>“YES(是)”<br>。若取值为<br>“NO(否)”<br>，则对端SCTP端点必须支持不检查SCTP消息中的“校验和”字段，否则SCTP偶联将无法建立。 |

| CHKSUMTYPE | 校验和算法类型 | 可选必选说明：可选参数<br>参数含义：该参数用于指定本端的SCTP协议支持哪种校验和算法。<br>数据来源：本端规划<br>取值范围：<br>- “ADLER32(ADLER32)”：表示本端的SCTP协议支持的校验和算法为ADLER32。<br>- “CRC32(CRC32)”：表示本端的SCTP协议支持的校验和算法为CRC32。<br>默认值：无<br>配置原则：SCTP检验和算法需要在SCTP偶联的两端保持一致。否则，一端发送的SCTP消息将因校验和不正确而被接收的另一端丢弃。 |
| BINDTMR | 绑定定时器时长（ms） | 可选必选说明：可选参数<br>参数含义：该定时器用于控制SCTP消息绑定发送的时延，该定时器超时后，系统会把多个数据包绑定在一起发送。<br>数据来源：本端规划<br>取值范围：10ms~200ms<br>默认值：无<br>配置原则：该参数最小粒度为10ms。<br>说明：此参数只在M3UA链路上有效。S1链路的绑定定时器时长可用<br>[**SET S1APPARA**](../../S1接口管理/S1AP协议参数/设置S1AP协议参数(SET S1APPARA)_72225935.md)<br>的BUNDLETIME参数设置。 |
| DELAYTMRLEN | 延迟定时器时长（ms） | 可选必选说明：可选参数<br>参数含义：该定时器用于控制SCTP数据应答消息延迟发送的时延，即收到对端的数据后延时多长时间后回响应。<br>数据来源：本端规划<br>取值范围：10ms~500ms<br>默认值：无<br>配置原则：该参数最小粒度为10ms。 |
| IPV4MTU | IPv4 MTU大小 | 可选必选说明：可选参数<br>参数含义：该参数用于指定SCTP使用的IPv4 MTU大小。<br>数据来源：本端规划<br>取值范围：96~65534<br>默认值：无<br>说明：产品接收和发送基于IPv4的SCTP包的MTU规格是1600字节，系统会自动使用该规格与此参数值进行比较，若设定值大于1600则该MTU实际值保持不变，反之则取该参数设置值。 |
| IPV6MTU | IPv6 MTU大小 | 可选必选说明：可选参数<br>参数含义：该参数用于指定SCTP使用的IPv6 MTU大小。<br>数据来源：本端规划<br>取值范围：1280~65534<br>默认值：无<br>说明：产品接收和发送基于IPv6的SCTP包的MTU规格是1600字节，系统会自动使用该规格与此参数值进行比较，若设定值大于1600则该MTU实际值保持不变，反之则取该参数设置值。 |

## 使用实例

将SCTP协议参数索引号为3的参数表中记录的RTO最小值改为350，RTO最大值为900，RTO初始值为400，RTO ALPHA值为13，RTO BETA值为45，SCTP协议参数名为sctppara2：

MOD SCTPPARA: SCTPPARAINDEX=3, RTOMIN=350, RTOMAX=900, RTOINIT=400, RTOALPHA=13, RTOBETA=45, SCTPPARANAME="sctppara2";
