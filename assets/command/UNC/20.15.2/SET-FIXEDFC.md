# SET FIXEDFC（设置固定速率流控信息）

## 功能

**适用网元：SGSN、MME**

该命令用于设置S6a/S6d接口固定速率流控信息。

## 注意事项

- 系统初次运行时，会执行系统初始设置值。
- 此命令执行后立即生效。
- “统一HTR流控”功能优先级高于“固定速率流控”。启用“统一HTR流控”功能后，即“OFFICESWITCH（局向流控开关）”取值为“ON”时，“固定速率流控”功能失效。

## 权限

manage-ug；system-ug
G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| FCSWITCH | 固定速率流控开关 | 可选必选说明：可选参数<br>参数含义：该参数用于指定是否开启S6a/S6d接口的“固定速率流控”功能。<br>数据来源：整网规划<br>取值范围：<br>- “ON（开启）”<br>- “OFF（关闭）”<br>系统初始设置值：<br>“ON (开启)”<br>说明：- ON(开启)：当系统承载的用户量较大，或者HSS的处理能力较弱时，尤其是系统重启后大量用户并发接入时，需要开启“固定速率流控”功能。<br>- OFF(关闭)：系统用户量较小，或者HSS的处理能力非常强时，可以关闭“固定速率流控”功能。 |
| FCTHD | 固定流控速率（个/秒） | 可选必选说明：可选参数<br>参数含义：该参数用于指定在开启固定速率流控功能后，UE“注册请求”速率阈值。系统通过控制“注册请求”中向HSS发送的请求数，来实现对HSS的流控。该参数是基于整系统（SPP进程数*每SPP进程期望的流控值）的流控值；注册请求指UE发起的Attach请求、Inter TAU请求、Inter RAU（S6d接口）请求、Intra TAU(GERAN/UTRAN->LTE)请求和Intra RAU(LTE->GERAN/UTRAN S6d接口)请求。<br>数据来源：整网规划<br>取值范围：0~1000000<br>系统初始设置值：0<br>说明：- 当“固定速率流控开关”为“ON (开启)”时，该字段有效。<br>- 建议根据实际用户数和期望接入时长来配置此参数，即“固定流控速率（个/秒）”为：实际用户数/接入时长，例如系统当前实际注册用户数为80万，需要在1800秒内接入，故“固定流控速率（个/秒）”为：800,000个 / 1800秒 = 445(个/秒)。<br>- 如果通过上面公式计算的值与HSS的实际处理能力存在较大的差别，则以HSS的处理能力为准。一般一个“注册请求”会触发到HSS的一次取鉴权集请求和位置更新请求，故“固定流控速率（个/秒）”为：HSS处理能力 / 2。<br>- 当该参数取值为0时，每个SPP进程的流控速率为50个/秒。<br>- 总速率请使用[**DSP FIXEDFC**](显示固定速率流控信息(DSP FIXEDFC)_26305952.md)命令查询。 |

## 操作的配置对象

- [[configobject/UNC/20.15.2/FIXEDFC]] · 固定速率流控信息（FIXEDFC）

## 使用实例

当系统重启时，需要开启固定速率流控，设置固定流控速率为400个/秒，运行以下命令:

**SET FIXEDFC: FCSWITCH=ON, FCTHD=400;**
