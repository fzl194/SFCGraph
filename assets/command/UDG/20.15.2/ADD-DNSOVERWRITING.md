# ADD DNSOVERWRITING（新增DNS重写动作）

## 功能

**适用NF：PGW-U、UPF**

该命令用于配置一个DNS重写动作。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为2000。
- 一条DNS重写配置中最多可以配置5组扩展过滤器和5组错误码范围。
- 扩展过滤器分为AND组、OR组和NOT组，AND组的个数不得超过3个。如果系统需对业务流执行对应的DNS重写动作，需满足下述条件：匹配AND组中所有成功或者匹配OR组中任意一个ExtFltName成功，并且匹配NOT组所有ExtFltName失败。如果没有配置，默认为所有业务资源访问都可以执行DNS重写动作。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 使用实例

假设运营商需要配置一个名称为test的DNS重写策略，扩展过滤器1为AND类型，名称为testextflt，指定重写的IPv4和IPv6地址，绑定错误码名称为errcode，则配置如下：

```
ADD DNSOVERWRITING: DNSOVERWRTNAME="test", EXTFLTTYPE1=AND, EXTFLTNAME1="testextflt", SERVERIP1="10.0.0.1", SERVERIPV6_1="fc00:1234:5678:abcd:1:1", BINDERRCODENAME="errcode";
```
