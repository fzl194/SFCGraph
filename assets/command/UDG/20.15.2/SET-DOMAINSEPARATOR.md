# SET DOMAINSEPARATOR（配置前后缀分隔符）

## 功能

**适用NF：SGW-U、PGW-U、UPF**

该命令用于配置作为前缀和后缀的分隔符。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为1。
- 同一字符不能同时作为前缀和后缀分隔符。
- 系统会把“#”作为系统默认的前缀分隔符，把“＠”作为系统默认的后缀分隔符。当该命令被执行后该默认配置会失效。
- 该命令存在系统初始记录，参数的初始设置值如下表：

| 参数标识 | PREFIX | SUFFIX |
| --- | --- | --- |
| 初始值 | # | @ |

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PREFIX | 前缀域名分隔符 | 可选必选说明：可选参数<br>参数含义：该参数用于指定作为前缀的分隔符。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～4。“@”、“#”、“%”、“/”或四者的组合。<br>默认值：无<br>配置原则：<br>- 输入单空格将删除该参数已有配置项。<br>- 域名作为前缀，域名与用户名以前缀分隔符相隔。 如用户为“HUAWEI.com#user1”，域名是“HUAWEI.com”，作为前缀以分隔符“#”与用户名连接。可以指定一个或多个作为前缀的域名分隔符，在进行域名分割时，用配置的第一个域名分隔符来匹配整个用户名，如果匹配则以该域名分割符为准，否则以第二个分割符来匹配，以此类推。<br>- 通过域名分隔符，可以将域名从用户名中分离出来，在本地配置的L2TP组中进行查找，看是否有这样的域名存在。如果有，则表明用户是VPN用户，需要与用户所属的LNS（L2TP Network Server）建立L2TP隧道。 |
| SUFFIX | 后缀域名分隔符 | 可选必选说明：可选参数<br>参数含义：该参数用于指定作为后缀的分隔符。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～4。“@”、“#”、“%”、“/”或四者的组合。<br>默认值：无<br>配置原则：<br>- 输入单空格将删除该参数已有配置项。<br>- 域名作为后缀，域名与用户名以后缀分隔符相隔。 如用户为“user1#HUAWEI.com”，域名是“HUAWEI.com”，作为后缀以分隔符“#”与用户名连接。可以指定一个或多个作为后缀的域名分隔符，在进行域名分割时，用配置的第一个域名分隔符来匹配整个用户名，如果匹配则以该域名分割符为准，否则以第二个分割符来匹配，以此类推。<br>- 通过域名分隔符，可以将域名从用户名中分离出来，在本地配置的L2TP组中进行查找，看是否有这样的域名存在。如果有，则表明用户是VPN用户，需要与用户所属的LNS（L2TP Network Server）建立L2TP隧道。 |

## 操作的配置对象

- [[configobject/UDG/20.15.2/DOMAINSEPARATOR]] · 前后缀分隔符（DOMAINSEPARATOR）

## 使用实例

设置前缀分割符为“@”和“%”，后缀分隔符为“#”和“/”：

```
SET DOMAINSEPARATOR:PREFIX="@%",SUFFIX="#/";
```
