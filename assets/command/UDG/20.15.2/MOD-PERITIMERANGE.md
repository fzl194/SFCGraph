# MOD PERITIMERANGE（修改周期时间段）

## 功能

**适用NF：PGW-U、UPF**

该命令用于修改指定时间段下，特定序号的周期时间段信息，如果运营商需要修改系统的业务生效的周期时间段，则用此命令进行设置。

## 注意事项

- 该命令执行后立即生效。
- 周期时间段的开始时间不能大于等于结束时间。
- 如果时间段被BwmRule绑定，则修改后的周期时间段配置不能与BwmRule绑定的其它时间段下的周期时间段配置重叠。
- 如果时间段被BwmRule绑定，则修改后的周期时间段时间间隔不能小于30分钟。
- 对于PCC用户，如果在时间段内出现夏令时切换或系统时间修改，以TimeRange开始时间为准，按照结束时间与开始时间之差（不考虑夏令时）标识的时间长度计算结束时间。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 使用实例

假设运营商需要修改系统的业务在周五的15点到17点生效，则按如下命令进行配置：

```
MOD PERITIMERANGE:TIMERANGENAME="t1",PERITMRANGESEQ=1,PERITIMETYPE=DAYSOFWEEK,DAYOFWEEK=FRIDAY,STARTTIME=15&00,ENDTIME=17&00;
```
