# MOD BWMCONTROLLER（修改带宽管理控制器）

## 功能

**适用NF：PGW-U、UPF**

该命令可以修改已经创建的BwmController的控制类型、保证带宽、峰值速率、car算法、shaping速率、shaping缓存队列深度、五元组个数、业务等级规格、用户公平使能、工作模式、最大丢包率、丢包率差值、保障模式等相关参数。

## 注意事项

- 该命令执行后立即生效。
- BwmController命令定义一套业务控制动作模板，只有被绑定在BwmRule下才会产生实际的业务控制器实例。
- 在同一个对象应用范围内，相同的业务控制器参数模板名字，只会创建一个业务控制器实例，如果一个业务控制器参数模板应用到多个BwmRule中，表明这些BwmRule共享一个业务控制器实例。但在不同对象应用范围，虽然是相同的业务控制器参数模板，会创建不同的业务控制器实例。
    - 比如，BwmController A绑在BwmUserGroup 1的BwmRule a和BwmRule b下， 表示符合BwmRule a和BwmRule b的流量共同按照BwmController A的参数进行控制，并采用同一个业务控制器实例。但在BwmUserGroup 2的BwmRule c下绑定BwmController A，表示在BwmUserGroup 2下的符合BwmRule c的流量采用BwmController A的参数进行业务控制，但BwmUserGroup 2的业务控制器实例和BwmUserGroup 1的业务控制器实例是不同的。
- 一个业务控制器被BwmRule绑定时，不允许将该控制器从car的配置修改成shaping的配置，或从shaping的配置修改成car的配置。
- 如果配置了PIR参数并且PIR参数不为0，则PIR参数的值不能小于CIR参数的值。建议用户级控制时，不要配置过大，否则达不到带宽控制的效果；用户组级控制时，不要配置过小，否则会加剧系统性能消耗，开启前请联系华为技术支持。
- 如果需要取消五元组数量限制，可以把最大五元组数修改为0。
- 修改SrvLevelSpec参数时，该BwmController下的所有ServiceLevel参数大于SrvLevelSpec参数值的BCSrvLevelPly配置将会被删除。
- 如果BwmController已经被绑定到BwmRule，则不允许修改BwmController的业务等级规格，只有解除绑定后才可以修改。
- 该配置不合理可能导致丢包过多。修改配置前请确认该配置是用户级规则使用，还是组级规则使用，组级规则的CIR调整会影响较多的用户，不能随意改小。
- 如果带宽控制器绑定到组级BwmRule，则每个业务容器内队列深度最大允许缓存报文数为300000，如果该带宽控制器绑定到用户级BwmRule，其最大值保持2560不变。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

| MAXFDNUM | 最大五元组数 | 可选必选说明：可选参数<br>参数含义：该参数用于表示业务最大可以使用的五元组个数，该参数可用于控制业务流个数。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围为0～4294967295。<br>默认值：无<br>配置原则：<br>- 如果该参数不配或者配置为0，则表示不限制最大五元组数。<br>- 如果bwm控制器被绑定在用户组级BwmRule下，则控制的是用户组内业务流数不能超过参数的值。<br>- 如果bwm控制器被绑在用户级BwmRule下，则控制的是单用户的业务流数不能超过参数的值。<br>- 该参数只对非Tos类业务生效，即只有配置了该参数的bwm控制器，与非Tos类的BwmService绑定在同一个BwmRule下，才会控制五元组个数，否则不会根据该参数控制五元组个数。<br>- 该参数将影响使用此控制器的业务流总数。业务流数达到参数的值后，新业务流将被阻断和拆链。因此必须慎重，不合适的配置会影响业务正常开展。 |
| COLORISAWARE | BWM CAR颜色是否敏感 | 可选必选说明：条件可选参数<br>前提条件：该参数在“CTRLTYPE”配置为“CAR”时为可选参数。<br>参数含义：该参数用于表示上一级颜色感知开关。<br>数据来源：全网规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：系统目前BWM CAR有三级，先后做了用户级CAR->用户组级CAR->整机CAR，如果想在后一级CAR参考前一级CAR的结果，可以将该参数配置使能，否则配置不使能。 |
| PRIORITYISAWARE | BWM CAR优先级是否敏感 | 可选必选说明：条件可选参数<br>前提条件：该参数在“CTRLTYPE”配置为“CAR”时为可选参数。<br>参数含义：该参数用于表示优先级感知开关。<br>数据来源：全网规划<br>取值范围：枚举类型。<br>- DISABLE：不使能。<br>- ENABLE：使能。<br>默认值：无<br>配置原则：如果需要区分优先级，当拥塞时优先转发某些业务的报文，则可以将该参数配置为使能。 |
| CFGDOMAINNAME | 配置域名称 | 可选必选说明：可选参数<br>参数含义：该参数表示命令所属公共配置域的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～31。<br>默认值：无<br>配置原则：输入单空格将删除该参数已有配置项。 |
| SRVLEVELSPEC | 业务等级规格 | 可选必选说明：条件可选参数<br>前提条件：该参数在“CTRLTYPE”配置为“CAR” 或 “SHAPING”时为可选参数。<br>参数含义：该参数用于表示该带宽管理控制器最大可支持的业务等级规格。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围为1~30。<br>默认值：无<br>配置原则：<br>- 针对组级带宽管理控制器最多可以配置30个不同的业务等级，支持基于此规格执行基于业务等级的优先调度。<br>- 针对用户级带宽管理控制器最多可以配置5个不同的业务等级。<br>- 如果无需执行多个业务等级的带宽管理，该参数建议配置为1。 |
| USERFAIREN | 用户公平调度 | 可选必选说明：条件可选参数<br>前提条件：该参数在“CTRLTYPE”配置为“SHAPING”时为可选参数。<br>参数含义：该参数用于表示该带宽管理控制器在组级带宽管理时是否支持用户级公平调度功能使能。如果期望该带宽控制器在组级带宽控制时，支持限制重度用户超预期占用管道带宽或新用户在初始业务发起阶段优先占用管道带宽时，可以配置该参数为ENABLE。<br>数据来源：全网规划<br>取值范围：枚举类型。<br>- DISABLE：带宽管理控制器在组级带宽管理时用户级公平调度功能不使能。<br>- ENABLE：带宽管理控制器在组级带宽管理时用户级公平调度功能使能。<br>默认值：无<br>配置原则：如果期望该带宽控制器在组级带宽控制时，支持限制重度用户超预期占用管道带宽或新用户在初始业务发起阶段优先占用管道带宽时，可以配置该参数为ENABLE。 |
| WORKMODE | 控制模式 | 可选必选说明：条件可选参数<br>前提条件：该参数在“CTRLTYPE”配置为“SHAPING”时为可选参数。<br>参数含义：该参数用于表示该带宽管理控制器在执行组级带宽控制策略时的工作模式为自动模式还是手动模式。<br>数据来源：全网规划<br>取值范围：枚举类型。<br>- MANUAL：BWM控制器的类型为手动模式。<br>- AUTO：BWM控制器的类型为自动模式。<br>默认值：无<br>配置原则：带宽管理控制器在执行组级带宽控制策略时，当该参数不配或配置为手动模式时，表示BwmController为手动调整模式，基于配置的每个ServiceLevel的CIRRate、PIRRate、ShapRate进行管控。 带宽管理控制器在执行组级带宽控制策略时，当该参数配置为自动模式时，表示BwmController为自动调整模式，基于每个ServiceLevel的业务分布，在满足MaxPktLostRate的要求下自动调整每个ServiceLevel的CIRRate、PIRRate、ShapRate进行管控，确保基于优先级的公平调度。 带宽管理控制器在执行用户级带宽控制策略时，该参数不生效，仅支持手动模式工作。 |
| ASSUREMODE | 保障模式 | 可选必选说明：条件可选参数<br>前提条件：该参数在“WORKMODE”配置为“AUTO”时为可选参数。<br>参数含义：该参数用于表示该带宽管理控制器的在自动计算每个ServiceLevel的丢包率时，出现最大丢包率和带宽控制器出口速率仅能满足一个条件时的优先保障模式。<br>数据来源：全网规划<br>取值范围：枚举类型。<br>- EXPFIRST：BWM控制器的类型为体验优先模式，优先保障每个ServiceLevel的丢包率不高于配置的最大丢包率。<br>- RATEFIRST：BWM控制器的类型为转发速率优先模式，优先保障整个带宽控制器的出口速率不超过配置值。<br>默认值：无<br>配置原则：当该参数配置为EXPFIRST时，表示BwmController在自动模式工作时，如果出现最大丢包率和带宽控制器出口速率仅能满足一个条件时，优先保障每个ServiceLevel的丢包率不高于配置的最大丢包率。 当该参数配置为RATEFIRST时，表示BwmController在自动模式工作时，如果出现最大丢包率和带宽控制器出口速率仅能满足一个条件时，优先保障整个带宽控制器的出口速率不超过配置值。 |
| MAXPKTLOSTRATE | 最大丢包率 | 可选必选说明：条件可选参数<br>前提条件：该参数在“WORKMODE”配置为“AUTO”时为可选参数。<br>参数含义：该参数用于表示此带宽控制器允许的最大丢包比率。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围为0～10000。单位是万分比。<br>默认值：无<br>配置原则：对于某一类业务为了保证其体验可控，可配置该参数控制该ServiceLevel业务的最大丢包率。 |
| PKTLOSTRATEDTL | 丢包率差值 | 可选必选说明：条件可选参数<br>前提条件：该参数在“WORKMODE”配置为“AUTO”时为可选参数。<br>参数含义：该参数用于表示此带宽控制器两个有数据的ServiceLevel之间的丢包率差值。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围为0～10000。单位是万分比。<br>默认值：无<br>配置原则：为了保证不同ServiceLevel在自动模式下丢包率的公平调度，高优先级业务丢包率理论上应低于低优先级业务，可以通过该参数配置周期内基于业务分布，自动调整两个ServiceLevel之间的丢包率差值。 |

## 使用实例

- 如果已经创建了BWM控制器“bc1”，如果需要修改它的参数，例如期望速率平稳在200kbps，队列深度为2000，业务流数不能超过4000个，则使用如下的命令修改：
  ```
  MOD BWMCONTROLLER: BWMCNAME="bc1", CTRLTYPE=SHAPING, RATE=200, QUEDEPTH=2000, MAXFDNUM=4000, USERFAIREN=DISABLE, WORKMODE=MANUAL;
  ```
- 如果已经创建了BWM控制器“bc2”，并且它对五元组数量做了限制，如果需要取消五元组限制，则可以使用下面的命令修改：
  ```
  MOD BWMCONTROLLER:BWMCNAME="bc2",MAXFDNUM=0;
  ```
