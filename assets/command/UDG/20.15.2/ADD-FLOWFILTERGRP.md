# ADD FLOWFILTERGRP（增加流过滤器组）

## 功能

**适用NF：PGW-U、UPF**

该命令用于配置流过滤器组。当用户需要多个流过滤条件同时存在时，可以配置流过滤器组。

## 注意事项

- 该命令执行后只对新激活用户生效。
- 系统支持配置100000个流过滤器组。当配置记录数大于规格的90%时，会上报“ALM-135602215 配置数量超阈值”告警。当配置记录数小于等于配置规格90%时，恢复“ALM-135602215 配置数量超阈值”告警。阈值可以通过MOD CFGTHRESHOLD命令修改。
- 单个流过滤器组中可以配置128个流过滤器。系统最多可支持200000条流过滤器组和流过滤器的绑定关系。
- 流过滤器组没有配置任何逻辑分组的流过滤器，则没有流能命中该流过滤器组。
- 配置流过滤器组数量较多时，对性能会产生一定影响，且影响大小与配置数量有关，请确定是否必须配置，或共用已存在的配置。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| FLWFLTRGRPNAME | 流过滤器组名称 | 可选必选说明：必选参数<br>参数含义：该参数用于指定流过滤器组名字。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。不区分大小写。<br>默认值：无<br>配置原则：无 |
| FLOWFILTERNAME | 流过滤器名称 | 可选必选说明：可选参数<br>参数含义：该参数用于指定流过滤器名字。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。不支持空格，不区分大小写。<br>默认值：无<br>配置原则：该参数使用ADD FLOWFILTER命令配置生成。 |
| FLWFLTRLOGICGRP | 流过滤器逻辑分组 | 可选必选说明：可选参数<br>参数含义：该参数用于指定流过滤器的逻辑分组。当该参数未设置时，流过滤器的逻辑分组默认为或组。只有一条流同时命中流过滤器组内的与、或、非逻辑分组，该流才算命中该流过滤器组。<br>数据来源：本端规划<br>取值范围：枚举类型。外置规则库定义属性的流过滤器的逻辑分组只能设置为或。<br>- AND：指定流过滤器的逻辑分组为与组。如果一条流命中与组内所有流过滤器，则该流命中与逻辑分组；如果配置了或逻辑分组或者非逻辑分组，而与逻辑分组没有配置流过滤器，则所有流可以命中与逻辑分组。<br>- OR：指定流过滤器的逻辑分组为或组。如果一条流命中或组内任何一个流过滤器，则该流命中或逻辑分组；如果配置了与逻辑分组或者非逻辑分组，而或逻辑分组没有配置流过滤器，则所有流可以命中或逻辑分组。<br>- NOT：指定流过滤器的逻辑分组为非组。如果一条流未命中非逻辑分组内任何流过滤器，则流命中该逻辑分组；如果配置了与逻辑分组或者或逻辑分组，而非逻辑分组没有配置流过滤器，则所有流可以命中该逻辑分组。<br>默认值：OR<br>配置原则：无 |
| CFGDOMAINNAME | 配置域名称 | 可选必选说明：可选参数<br>参数含义：该参数表示命令所属公共配置域的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～31。<br>默认值：无<br>配置原则：无 |

## 使用实例

配置流过滤器组testflowfiltergrp，在流过滤器组中增加流过滤器testflowfilter：

```
ADD FLOWFILTERGRP: FLWFLTRGRPNAME="testflowfiltergrp",FLOWFILTERNAME="testflowfilter";
```
