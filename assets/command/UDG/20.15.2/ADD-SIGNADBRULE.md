# ADD SIGNADBRULE（增加特征库规则配置）

## 功能

**适用NF：PGW-U、UPF**

该命令用于配置特征库中应用协议、应用子协议的动态识别规则，可以配置的识别规则有HOST、服务器IP地址、端口以及优先级。

## 注意事项

- 为了防止频繁刷新规则对系统造成性能影响，识别规则添加、修改、删除后，不会立即生效，当最后一次执行本命令的90秒后所有规则才生效。
- 该命令最大记录数为1000。
- 该命令仅支持应用协议、应用子协议。
- 配置优先级时，建议配置时按照步长为5配置，例如5、10、15，便于后续插入新的优先级。
- 配置动态协议识别规则时指定优先级，则系统以动态协议识别规则识别出的协议为准。如果不指定优先级，则动态协议识别规则与协议特征知识库文件的识别规则的优先级相同。
- 该命令与LOD SIGNATUREDB命令建议间隔5分钟，否则可能会导致加载知识库失败。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

| PORTSTART | 服务器起始端口号 | 可选必选说明：条件必选参数<br>前提条件：该参数在“OPERATOR”配置为“EQUAL”、“GT” 或 “RANGE”时为必选参数。<br>参数含义：该参数用于设置服务器起始端口号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～65535。<br>默认值：无<br>配置原则：无 |

## 使用实例

假如运营商想要指定IP地址为10.0.0.1/16的业务是facebook_others协议，配置如下：

```
ADD SIGNADBRULE:SIGNADBRULENAME="test_signadbrule",PROTOCOLNAME="facebook_others",IPMODE=IP,IPVERSION=IPV4,IPV4ADDR="10.0.0.1",MASKVALUE=16;
```
