# SET BANDWIDTHMNG（设置带宽管理参数）

## 功能

**适用NF：PGW-U、UPF**

![](设置带宽管理参数（SET BANDWIDTHMNG）_82837465.assets/notice_3.0-zh-cn.png)

本命令属于高危命令，Range发生变化后，其他带宽管理命令（bwmcontroller，bwmservice等）也需要同步修改。

该命令用于设置带宽控制器中用户组级业务带宽和连接数配置的生效范围，以及带宽资源动态统计和调整周期。

系统由多个业务处理单元组成，每个处理单元都是独立运行，如果带宽控制器中用户组级业务带宽和连接数是整机配置，就需要周期性统计汇总各业务处理单元的业务使用状态，然后统计结果，动态调整每个单元可以使用的资源，如CAR桶参数、连接数阈值等。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为1。
- 使用本命令配置业务带宽和连接数生效范围将影响到带宽数据统计汇总范围，进而影响最终带宽控制效果（通过流量监测工具查看）。使用本命令配置的资源动态统计和调整周期，将影响带宽数据统计汇总周期。
- 该命令对系统性能有影响，SYSTEM需要整机级汇总信息，执行命令前请重新进行容量评估，如果无法评估请联系华为技术支持。
- Range发生变化后，其他带宽管理命令（bwmcontroller，bwmservice等）也需要同步修改。
- 该命令存在系统初始记录，参数的初始设置值如下表：

| 参数标识 | RANGE | PERIOD | CIRMINTHRES | PIRMINTHRES | BWMSUBSCRCTRL | ADJUSTVALUE | GRPBWMSHAPSW |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 初始值 | CPU | 5 | 0 | 0 | PCC_RULE_PRIOR | 10 | DISABLE |

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| RANGE | 带宽管理生效范围 | 可选必选说明：可选参数<br>参数含义：该参数用于设置带宽控制器中用户组级业务带宽和连接数的生效范围。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- CPU：设置带宽控制器中用户组级业务带宽和连接数是基于业务处理单元生效。<br>- SYSTEM：设置带宽控制器中用户组级业务带宽和连接数是整机生效。<br>默认值：无<br>配置原则：<br>- CPU：当前已经有用户组级带宽配置的前提下，建议配置该参数。<br>- SYSTEM：如果是新建局点，建议配置该参数。 |
| PERIOD | 统计周期（秒） | 可选必选说明：条件可选参数<br>前提条件：该参数在“RANGE”配置为“SYSTEM”时为可选参数。<br>参数含义：该参数用于设置带宽资源动态统计和调整周期。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为5～60，单位是秒。<br>默认值：无<br>配置原则：<br>- 如果配置周期值过大，会影响带宽资源动态调整的实时性，最终影响业务控制效果。一般情况下，建议保留默认配置。<br>- 如果现网业务流量比较平稳，没有频繁波动，可以考虑适当增加周期值。 |
| CIRMINTHRES | 承诺信息速率最低保护值（千比特每秒） | 可选必选说明：条件可选参数<br>前提条件：该参数在“RANGE”配置为“SYSTEM”时为可选参数。<br>参数含义：该参数用于系统根据整机带宽计算出的各业务处理单元承诺信息速率的最低保护值。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～629145600，单位是千比特每秒。<br>默认值：无<br>配置原则：RANGE设置为SYSTEM时，如果业务处理单元上流量占比不均，对于占比很小的处理单元，分到的带宽就会相对小，为确保分到小带宽的处理单元出现流量突增时能够平滑过渡，可通过此参数给这个业务处理单元预留一部分带宽。注意如果单个处理单元实际控制流量比此参数值小，则系统总带宽会比带宽控制器实际配置的带宽大一些。 参数值建议：如果多个处理单元实际流量均衡，建议保持初始配置；如果多个处理单元间实际流量差异大，需要对此参数进行调增，建议调整值最小值为（控制总带宽/处理单元数）* ADJUSTVALUE / 100，最大值为（控制总带宽/处理单元数），如有疑问请联系华为工程师。 |
| PIRMINTHRES | 峰值信息速率最低保护值（千比特每秒） | 可选必选说明：条件可选参数<br>前提条件：该参数在“RANGE”配置为“SYSTEM”时为可选参数。<br>参数含义：该参数用于系统根据整机带宽计算出的各业务处理单元峰值信息速率的最低保护值。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～629145600，单位是千比特每秒。<br>默认值：无<br>配置原则：RANGE设置为SYSTEM时，如果业务处理单元上流量占比不均，对于占比很小的处理单元，分到的带宽就会相对小，为确保分到小带宽的处理单元出现流量突增时能够平滑过渡，可通过此参数给这个业务处理单元预留一部分带宽。注意如果单个处理单元实际控制流量比此参数值小，则系统总带宽会比带宽控制器实际配置的带宽大一些。 参数值建议：如果多个处理单元实际流量均衡，建议保持初始配置；如果多个处理单元间实际流量差异大，需要对此参数进行调增，建议调整值最小值为（控制总带宽/处理单元数）* ADJUSTVALUE / 100，最大值为（控制总带宽/处理单元数），如有疑问请联系华为工程师。 |
| BWMSUBSCRCTRL | 用户级业务带宽控制选项 | 可选必选说明：可选参数<br>参数含义：该参数用于配置用户级业务带宽控制器是否使用PCC Rule下的QoS参数。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- PCC_RULE_IGNORE：表示不使用PCC Rule中的QoS参数做用户级业务带宽控制，使用BWM Rule的带宽参数做用户级业务带宽控制。<br>- PCC_RULE_PRIOR：表示优先使用PCC Rule下的QoS参数做用户级业务带宽控制。如未匹配上PCC Rule，则使用BWM Rule中的带宽参数做用户级业务带宽控制。<br>- BWM_RULE_IGNORE：表示在命中PCC动态规则场景，不使用BWM rule做用户级业务带宽控制，使用PCC动态规则下的QoS参数做用户级业务带宽控制。如果PCC动态规则未下发QoS参数， 则不做用户级业务带宽控制。<br>默认值：无<br>配置原则：如果对用户带宽控制的参数在PCC Rule中和BWM Rule中一样，则每种类型均可选择；如果不同，则以实际规划的业务带宽控制效果进行选择。 |
| ADJUSTVALUE | 调整系数 | 可选必选说明：条件可选参数<br>前提条件：该参数在“RANGE”配置为“SYSTEM”时为可选参数。<br>参数含义：该参数用于设置整机带宽控制功能对于流量占比低于平均比例的调整系数。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～100，单位是百分比。<br>默认值：无<br>配置原则：无 |
| GRPBWMSHAPSW | 基于Shaping的组级带宽控制开关 | 可选必选说明：可选参数<br>参数含义：该参数用于控制基于Shaping的组级带宽控制功能。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- DISABLE：不使能（关闭）。<br>- ENABLE：使能（开启）。<br>默认值：无<br>配置原则：无 |

## 使用实例

假如运营商希望设置带宽控制器中的用户组级业务带宽和连接数基于整机生效，并设置资源统计和调整周期：

```
SET BANDWIDTHMNG: RANGE=SYSTEM, PERIOD=6, CIRMINTHRES=25, PIRMINTHRES=26;
```
