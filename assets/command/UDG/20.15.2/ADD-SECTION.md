# ADD SECTION（添加地址池IP地址段）

## 功能

**适用NF：PGW-U、UPF**

该命令为地址池增加地址段。系统在激活用户时需要为对应的地址池分配地址段，可用该命令配置对应地址池的地址段。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为40000。当配置记录数大于规格的98%时，会上报“ALM-135602215 配置数量超阈值”告警。当配置记录数小于等于配置规格95%时，恢复“ALM-135602215 配置数量超阈值”告警。阈值可以通过MOD CFGTHRESHOLD命令修改。
- 一个地址池最多可以配置64个地址段。起始地址不能大于终止地址，本地地址池每个IPv4地址段最多可以配置32768个地址，外部地址池每个IPv4地址段最多配置1048576个地址，本地地址池每个IPv6地址段可以配置1048576个地址，外部地址池每个IPv6地址段可以配置1048576个地址。
- 同一个pool下各个地址段里的地址不能相互重叠，且同一个VPN下，各个地址段里的地址不能相互重叠。注意，当两个地址池均未配置VPN，代表其VPN相同。
- 地址池增加前需确认规划的地址与现网中的地址是否存在重叠，如重叠需确认是否合理。如果地址池规划不合理，可能导致业务中断、受损。
- 系统上需要激活由系统本地分配地址的用户时，必须先执行命令ADD POOL配置本地地址池，本地地址池中必须配置本地地址段。
- 本地地址池的IPv4地址段中A、B、C类地址中的广播地址和主机地址为不可用地址，不会分配给用户。
- 本地地址池的IPv6地址只支持冒分十六进制表示法，不支持0位压缩表示法和内嵌IPv4地址表示法。
- 配置IPv6类型地址池内地址时，接口ID部分可以随意配置，不参与地址段下地址个数统计。
- 组播地址池的IPv4地址段范围为232.0.1.0~232.255.255.255。
- UE地址段应规划使用规整地址段（必须是一个完整子网中的所有IP地址，详见ADD SECTION），避免UPF生成过多对外发布的路由对外部设备的冲击。
- 新建时根据目标网络规划好UE地址段，例如规划UE地址段为10.10.1.0~10.10.127.255/255.255.128.0，子段IP的个数与局点最大可能的ISU Pod个数一致。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| POOLNAME | 地址池名称 | 可选必选说明：必选参数<br>参数含义：该参数用于指定已配置的地址池的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～79，单位是字节。由“_”、“-”、数字、大小写字母和“.”组成，不能以“.”开头且不能出现连续两个“.”，不区分大小写。<br>默认值：无<br>配置原则：该参数通过ADD POOL命令配置生成。 |
| SECTIONNUM | 地址段号 | 可选必选说明：必选参数<br>参数含义：该参数用于指定配置Section的编号。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为0～63。<br>默认值：无<br>配置原则：无 |
| IPVERSION | IP地址类型 | 可选必选说明：必选参数<br>参数含义：该参数用于指定匹配地址池的地址类型。<br>数据来源：本端规划<br>取值范围：枚举类型。<br>- IPV4：IPv4地址类型。<br>- IPV6：IPv6地址类型。<br>默认值：IPV4<br>配置原则：无 |
| V4STARTIP | 地址段的起始IPv4地址 | 可选必选说明：条件必选参数<br>前提条件：该参数在“IPVERSION”配置为“IPV4”时为必选参数。<br>参数含义：该参数用于指定IPv4地址段的起始地址。<br>数据来源：全网规划<br>取值范围：IPv4地址类型。<br>默认值：无<br>配置原则：无 |
| V4ENDIP | 地址段的终止IPv4地址 | 可选必选说明：条件必选参数<br>前提条件：该参数在“IPVERSION”配置为“IPV4”时为必选参数。<br>参数含义：该参数用于指定IPv4地址段的终止地址。<br>数据来源：全网规划<br>取值范围：IPv4地址类型。<br>默认值：无<br>配置原则：结束IP地址必须大于或等于起始IP地址。 |
| V6PREFIXSTART | IPv6前缀起始地址 | 可选必选说明：条件必选参数<br>前提条件：该参数在“IPVERSION”配置为“IPV6”时为必选参数。<br>参数含义：该参数用于指定IPv6地址段的起始前缀。<br>数据来源：全网规划<br>取值范围：IPv6地址类型。冒号分十六进制，格式为X:X:X:X:X:X:X:X。除了组播地址、链路本地地址、环回地址或未指定的地址为非法地址外，其他都为合法地址。<br>默认值：无<br>配置原则：无 |
| V6PREFIXEND | IPv6前缀结束地址 | 可选必选说明：条件必选参数<br>前提条件：该参数在“IPVERSION”配置为“IPV6”时为必选参数。<br>参数含义：该参数用于指定IPv6地址段的终止前缀。<br>数据来源：全网规划<br>取值范围：IPv6地址类型。冒号分十六进制，格式为X:X:X:X:X:X:X:X。除了组播地址、链路本地地址、环回地址或未指定的地址为非法地址外，其他都为合法地址。<br>默认值：无<br>配置原则：一个IPv6地址前缀只能配置一个前缀地址段，前缀地址段的起始地址必须小于或等于终止地址，前缀地址段内的地址总数不能超过1048576，且同一VPN下，各个前缀地址段里的地址不能相互重叠。 |
| V6PREFIXLENGTH | IPv6前缀长度 | 可选必选说明：条件必选参数<br>前提条件：该参数在“IPVERSION”配置为“IPV6”时为必选参数。<br>参数含义：该参数用于指定IPv6地址段的前缀长度。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围为49～64。<br>默认值：无<br>配置原则：无 |
| MASK | 掩码 | 可选必选说明：可选参数<br>参数含义：该参数用于控制系统发布用户wlr_ud网段路由掩码长度。 该参数配置为N，代表发布用户wlr_ud路由覆盖的地址个数为2的N次方（2^N）。 如果地址段内的地址个数小于2的N次方（2^N），则按照原有规则生成路由。 改变系统发布用户wlr_ud网段路由掩码长度会对系统信令处理能力产生影响，对外发布的路由条数没有特别要求的情况下，不建议修改该参数。<br>数据来源：全网规划<br>取值范围：整数类型，取值范围为5～15。如果地址段内的地址个数小于2的N次方（2^N），则按照原有规则生成路由。<br>默认值：无<br>配置原则：<br>- 该参数仅用于本地地址池。 该参数用于控制系统发布用户wlr_ud网段路由掩码长度。 该参数配置为N，代表发布用户wlr_ud路由覆盖的地址个数为2的N次方（2^N）。 如果参数Mask所覆盖的地址个数大于地址段的地址个数，该参数值将无法生效，如果参数Mask所覆盖的地址个数和地址段内的地址个数不匹配，可能造成该地址段对应的用户路由条数过多，路由条数可能超过1025，该参数值将无法生效。 该参数适用于已配置了入不转板功能的系统。 在入不转板使能时，需要保证wlr-ud网段数量大于等于ISU个数，如果网段数量小于ISU个数，可能会导致部分POD分配不到地址子段，部分用户激活到未分配地址子段的POD上，可能会出现流量入转板。 改变系统发布用户wlr_ud网段路由掩码长度可能冲击交互机等设备的路由表，会对系统信令处理能力产生影响，对外发布的路由条数没有特别要求的情况下，不建议修改该参数。 如果该参数设置太大，当开启入不转板功能时，可能会导致部分POD分配不到地址子段，部分用户激活到未分配地址子段的POD上，可能会出现流量入转板。<br>- 业务内部会对配置的地址段，使用mask值进行额外的内部分段处理，mask的值是5到15。例如一个地址段配置了200个IP地址，mask的值如果设置成5，则每个内部地址子段的地址数就是2的5次方，即32个。子段数量就是section总的地址数200除以32，也就是业务内部会额外分成7段地址子段。在开启入不转板功能的场景下，每个section内部的地址子段数量建议不少于isu pod数量，请依据该描述合理设置mask的值。 |

## 使用实例

- 假设运营商已配置了一个IPv4的本地地址池pool1时，需要为其配置一个IPv4地址段：
  ```
  ADD SECTION:POOLNAME="pool1",SECTIONNUM=0,IPVERSION=IPV4,V4STARTIP="10.1.1.0",V4ENDIP="10.1.1.200";
  ```
- 假设运营商需为IPv6的本地地址池pool2配置一个IPv6地址段：
  ```
  ADD SECTION:POOLNAME="pool2",SECTIONNUM=0,IPVERSION=IPV6,V6PREFIXSTART="fc01:1111:2222:0000::",V6PREFIXEND="fc01:1111:2222:ffff::",V6PREFIXLENGTH=64;
  ```
