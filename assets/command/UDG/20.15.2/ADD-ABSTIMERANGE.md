# ADD ABSTIMERANGE（增加绝对时间段）

## 功能

**适用NF：SGW-U、PGW-U、UPF**

该命令用于配置绝对时间段信息，从设定的起始年月日时分到终止年月日时分之间的时间段，该时间段无法循环出现，也没有周期。设置绝对时间段的目的是保证系统的业务在该绝对时间段范围内生效。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为2400。
- 每个时间段下最多可以配置12个绝对时间段。
- 绝对时间段的开始时间不能大于等于结束时间。
- 绝对时间段的结束时间不能小于等于系统当前时间。
- 如果时间段被BwmRule绑定，则新增的绝对时间段配置不能与BwmRule绑定的其它时间段下的绝对时间段配置重叠。
- 如果时间段被BwmRule绑定，则新增的绝对时间段配置时间间隔不能小于30分钟。
- 开始时间和开始日期必须同时输入，结束时间和结束日期必须同时输入。
- 对于PCC用户，如果在时间段内出现夏令时切换或系统时间修改，以TimeRange开始时间为准，按照结束时间与开始时间之差（不考虑夏令时）标识的时间长度计算结束时间。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| TIMERANGENAME | 时间段名称 | 可选必选说明：必选参数<br>参数含义：该参数用于设置时间段的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～31。不支持空格，不区分大小写，以字母开头。<br>默认值：无<br>配置原则：该参数使用ADD TIMERANGE命令配置生成。 |

| ENDTIME | 结束时间，格式HH:MM | 可选必选说明：可选参数<br>参数含义：该参数用于设置绝对时间段的结束时间。只输入结束时间不输入开始时间时，表明系统的业务立即生效。如果不设置该值，表明系统的业务从开始时间起一直生效。<br>数据来源：本端规划<br>取值范围：时间类型，输入格式是HH:MM。<br>默认值：无<br>配置原则：无 |
| ENDDATE | 结束日期，格式MM/DD/YYYY | 可选必选说明：可选参数<br>参数含义：该参数用于设置绝对时间段的结束日期。<br>数据来源：本端规划<br>取值范围：日期类型，输入格式是MM/DD/YYYY。<br>默认值：无<br>配置原则：无 |
| CFGDOMAINNAME | 配置域名称 | 可选必选说明：可选参数<br>参数含义：该参数表示命令所属公共配置域的名称。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～31。<br>默认值：无<br>配置原则：无 |

## 使用实例

假设运营商需要配置系统的业务在2015年8月20日的10点到2015年8月20日的11点生效，则按如下命令进行配置：

```
ADD ABSTIMERANGE:TIMERANGENAME="t1",ABSTIMERANGESEQ=1,STARTTIME=10&00,STARTDATE=2015&08&20,ENDTIME=11&00,ENDDATE=2015&08&20;
```
