# SET FLOWAGETIME（设置五元组老化时间）

## 功能

**适用NF：PGW-U、UPF**

![](设置五元组老化时间（SET FLOWAGETIME）_82837291.assets/notice_3.0-zh-cn.png)

本命令属于高危命令，可能会影响业务。

该命令用于配置头增强五元组、三四层五元组、TCP信令五元组、任意协议五元组和本地分流五元组的老化时间。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为1。
- 设置五元组老化时间时，每种类型的老化时间只能配置一条。如果五元组老化时间配置过大，会造成单用户和整机五元组使用数量太多的情况。
- 如果配置了业务触发专载的协议五元组老化时间设置超过120s，且需要通过SET UPGLBPMPARA命令修改业务触发的专有承载老化时间大于等于五元组老化时间。
- 该命令存在系统初始记录，参数的初始设置值如下表：

| 参数标识 | HEADENAGETIME | L34AGETIME | TCPSIGAGETIME | ANYAGETIME | TRAFFCLSAGETIME | LBOAGETIME |
| --- | --- | --- | --- | --- | --- | --- |
| 初始值 | 300 | 0 | 0 | 30 | 300 | 300 |

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| HEADENAGETIME | 头增强五元组老化时间（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于设置HTTP/RTSP/HTTPS头增强五元组老化时间。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为20～15300，单位是秒。<br>默认值：无<br>配置原则：推荐使用初始设置值。 |
| L34AGETIME | 三四层五元组老化时间（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于指定三四层流（即不进行七层协议识别或解析的五元组）的老化时间。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为5～7200，单位是秒。<br>默认值：无<br>配置原则：<br>- 如果三四层五元组老化时间未配置，则采用任意协议五元组老化时间。<br>- 对于Web Proxy业务的三四层流，采用任意协议五元组老化时间。 |
| TCPSIGAGETIME | TCP信令五元组老化时间（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于指定TCP信令流（即只有建链报文，没有业务报文，且需要进行七层协议识别或解析的五元组）的老化时间。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为5～7200，单位是秒。<br>默认值：无<br>配置原则：推荐使用初始设置值。 |
| ANYAGETIME | 任意协议五元组老化时间（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于指定任意协议五元组节点的老化时间。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为20～7200，单位是秒。<br>默认值：无<br>配置原则：在获取p2p类协议的五元组老化时间时，如果五元组使用数量超过总数量的80%，采用软参byte107配置的老化时间，如果没有超过总数量的80%，采用任意协议五元组老化时间。 |
| TRAFFCLSAGETIME | 流量分类五元组老化时间（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于设置流量分类五元组老化时间，此参数当前不支持。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为20～15300，单位是秒。<br>默认值：无<br>配置原则：推荐使用初始设置值。 |
| LBOAGETIME | 本地分流五元组老化时间（秒） | 可选必选说明：可选参数<br>参数含义：该参数用于设置本地分流五元组老化时间，此参数当前不支持。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为20～15300，单位是秒。<br>默认值：无<br>配置原则：推荐使用初始设置值。 |

## 使用实例

将五元组老化时间的初始设置值修改为其他值：

```
SET FLOWAGETIME:HEADENAGETIME=63,L34AGETIME=8,TCPSIGAGETIME=9,ANYAGETIME=60,LBOAGETIME=65;
```
