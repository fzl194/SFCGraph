# STP NODE（停止节点）

## 功能

![](停止节点（STP NODE）_45545153.assets/notice_3.0-zh-cn.png)

停止节点，节点上数据可能会丢失，导致业务受到影响。停止节点后，节点在存储故障期间无法启动。

本命令用于关闭网元下指定名称的节点。

> **说明**
> 该命令仅在Full-stack虚机场景下支持。
>
> - 执行该命令会影响所有运行在该节点上的业务，同时会造成部分数据丢失，请慎重使用该命令。
> - 执行该命令可能会提示操作异常，需要通过执行[DSP NODE](节点查询（DSP NODE）_71678755.md)命令来查看节点是否成功关闭。
> - 在存储故障期间，执行该命令停止节点后，若再执行STR NODE命令启动节点，该节点以及节点中的容器和进程，在存储故障恢复前都无法启动。
> - 执行该命令返回成功，表明命令成功下发到MANO，节点下电是否成功需要稍后执行[DSP NODE](节点查询（DSP NODE）_71678755.md)命令来查看。

> **说明**
> - 避免对同一个节点反复执行下电操作。该行为可能会导致MANO执行多次下电任务，请稍后使用[**DSP NODE**](节点查询（DSP NODE）_71678755.md)命令查看具体执行结果。
> - 同时只允许下电一个OMU/ETCD节点。
> - 避免短时间内对多个节点执行下电操作。该行为可能会导致多个管理节点被同时下电，从而影响业务功能。
> - NRSMaster为保证可靠性会在下电操作中超时重试一次，导致MML执行一次下电操作而在MANO上执行了两次下电任务，请稍后使用[**DSP NODE**](节点查询（DSP NODE）_71678755.md)命令查看具体执行结果
> - 若下电节点恰好为MML服务所在节点，由于后台返回操作结果的时间会晚于MML服务下线时间，会导致MML服务收不到返回的操作结果，请稍后使用[**DSP NODE**](节点查询（DSP NODE）_71678755.md)命令查看具体执行结果。
> - 若下电节点恰好为MML向NRSMaster实例下发命令所在节点，会导致MML收到100031错误返回码，请稍后使用[**DSP NODE**](节点查询（DSP NODE）_71678755.md)命令查看具体执行结果。
> - 避免在三方系统和OM Portal上同时操作节点上下电。
> - 节点处于非Normal状态时，对其进行下电操作，MANO可能不会对该节点执行下电操作，请稍后使用[**DSP NODE**](节点查询（DSP NODE）_71678755.md)命令查看具体执行结果。
> - 由于节点规格与运行的服务不同，每次下电操作耗时也存在差异。同时，下电节点会导致节点上所有的进程与服务被下线，建议谨慎使用，避免频繁执行本命令。

## 参数

| **参数标识** | **参数名称** | **参数说明** |
| --- | --- | --- |
| NODENAME | 节点名称 | 可选必选说明：必选参数。<br>参数含义：用于指示系统需要关闭的节点名称。<br>取值范围：不超过128位的字符串。<br>默认值：无。<br>配置原则：操作员可以使用<br>[**DSP NODE**](节点查询（DSP NODE）_71678755.md)<br>命令查询获得。 |

## 操作的配置对象

## 使用实例

关闭 “节点名称” 为 “10.0.0.1” 。

```
%%STP NODE: NODENAME="10.0.0.1";%% 
RETCODE = 0  操作成功   
操作结果如下 
------------
    状态  =  Success 
详细信息  =  操作成功 
(结果个数 = 1)  
---    END
```
