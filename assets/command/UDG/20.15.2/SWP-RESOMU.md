# SWP RESOMU（倒换主备操作维护单元）

## 功能

![](倒换主备操作维护单元（SWP RESOMU）_04480462.assets/notice_3.0-zh-cn.png)

本命令为高危命令，操作不当会引起操作维护单元承载的业务受影响，如果在存储故障期间执行该命令后，原备OMU可以正常升主，原主OMU会复位，且在存储故障恢复前原主OMU无法启动，请谨慎使用并联系华为技术支持协助操作。

在硬件替换、升级或打补丁等场景中，需要通过主动倒换OMU进程来达到原主用OMU所在资源上只有备用进程的要求，从而避免在替换这个资源时所造成的业务损失，此时可使用本命令。

## 注意事项

- 该命令执行后立即生效。
- 该命令只能在HA模式下使用，非HA模式不支持倒换。可以通过[**DSP OMU**](../../../../单体服务公共功能管理/系统管理/系统维护/倒换主备资源单元/显示主备操作维护单元信息（DSP OMU）_59103381.md)命令查询“倒换状态”是否为“就绪”。
- 执行本命令后，可以使用[**DSP OMU**](../../../../单体服务公共功能管理/系统管理/系统维护/倒换主备资源单元/显示主备操作维护单元信息（DSP OMU）_59103381.md)命令查询OMU主备状态，对照倒换前的OMU主备状态，以确定倒换是否成功。
- 该命令操作不当，可能引起操作维护单元承载的业务受影响。
- 该命令仅支持VNFP下发，如果需要在VNFC进行主备倒换操作，需要使用[**SWP OMU**](../../../../单体服务公共功能管理/系统管理/系统维护/倒换主备资源单元/倒换主备操作维护单元（SWP OMU）_59104194.md)。

## 使用实例

OMU1为VNFP主OMU名称，OMU2为VNFP备OMU名称，现在要执行VNFP主备倒换：

```
SWP RESOMU: RESNAME="OMU2";
```
