# MOD POLICYCONDITION（修改业务质差检测策略）

## 功能

**适用NF：PGW-U、UPF**

该命令用于修改业务质差检测策略。

## 注意事项

该命令执行后立即生效。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| POLICYCNDNAME | 策略参数名 | 可选必选说明：必选参数<br>参数含义：该参数用于指定业务质量检测的策略名称。<br>数据来源：本端规划<br>取值范围：字符串类型，不区分大小写，长度1~63。<br>默认值：无<br>配置原则：无 |
| UPLINKAVGRATE | 上行平均速率基线（Kbps） | 可选必选说明：可选参数<br>参数含义：配置上行平均速率基线，UPF检测到业务流实际的上行平均速率小于该值时，认为该业务流质差。<br>数据来源：本端规划<br>取值范围：整数类型。4294967295为无效值，不进行质差检测。<br>默认值：无<br>配置原则：若已配置过该参数，当不使用该参数进行检测时，可配置为4294967295。 |
| DNLINKAVGRATE | 下行平均速率基线（Kbps） | 可选必选说明：可选参数<br>参数含义：配置下行平均速率基线，UPF检测到业务流实际的下行平均速率小于该值时，认为该业务流质差。<br>数据来源：本端规划<br>取值范围：整数类型。4294967295为无效值，不进行质差检测。<br>默认值：无<br>配置原则：若已配置过该参数，当不使用该参数进行检测时，可配置为4294967295。 |

## 操作的配置对象

- [[UDG@20.15.2@ConfigObject@POLICYCONDITION]] · 业务质差检测策略（POLICYCONDITION）

## 使用实例

将质差策略名为policyname的上行平均速率基线修改为50Kbps：

```
MOD POLICYCONDITION: POLICYCNDNAME="policyname", UPLINKAVGRATE=50;
```
