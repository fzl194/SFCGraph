# SET LICCTRLALM（设置License容量告警配置）

## 功能

该命令用于设置系统的License容量告警配置。

当资源项使用率超过配置的故障告警门限，且持续超过“持续时间”时，则上报故障告警ALM-100046 资源达到LICENSE扩容门限；当资源项使用率低于配置的恢复告警门限，且持续超过“持续时间”时，则恢复此告警。

当用户需要修改告警门限和告警采样时间时，需要执行此命令。

> **说明**
> - 该命令执行后立即生效。
>
> - 执行该命令须确认输入参数不全为空。
>
> - 系统部署完成后，已经存在初始记录，参数的初始记录值如下表：
>
> | PER | REPER | TM |
> | --- | --- | --- |
> | 80 | 70 | 3 |

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| PER | 容量告警百分比(%) | 可选必选说明：可选参数<br>参数含义：该参数用于指定容量告警百分比。当系统中已使用的资源控制项数目在“持续时间”内的平均值与License允许的数目的比值大于该参数设定的值时，系统将产生扩容告警。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是1~99，单位是百分比。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST LICCTRLALM查询当前参数配置值。<br>配置原则：无 |
| REPER | 容量告警恢复百分比(%) | 可选必选说明：可选参数<br>参数含义：该参数用于指定容量告警恢复百分比。系统已经产生该资源控制项的扩容告警后，当已使用的资源控制项数目在“持续时间”内的平均值与License允许的数目的比值小于该参数设定的值时，系统将恢复该扩容告警。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是1~99，单位是百分比。“容量告警恢复百分比(%)”取值不能超过“容量告警百分比(%)”。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST LICCTRLALM查询当前参数配置值。<br>配置原则：无 |
| TM | 持续时间(h) | 可选必选说明：可选参数<br>参数含义：该参数用于指定系统中计算已使用的资源控制项数目平均值的“持续时间”，当该参数设置的时间内的平均值与License允许的数目的比值大于系统设定的“容量告警百分比”时，系统将产生扩容告警。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围是1~24，单位是小时。<br>默认值：无。执行命令并不输入该参数时，该参数保持系统当前配置不变，可通过LST LICCTRLALM查询当前参数配置值。<br>配置原则：无 |

## 操作的配置对象

- [[configobject/UDG/20.15.2/LICCTRLALM]] · License容量告警配置（LICCTRLALM）

## 使用实例

设置系统中原来的容量告警配置的容量告警百分比为85％，容量告警恢复百分比为75％，持续时间为2小时：

```
%%SET LICCTRLALM: PER=85, REPER=75, TM=2;%%
RETCODE = 0  操作成功

---    END
```
