# ADD IFCONTROLFLAP（添加接口震荡抑制）

## 功能

该命令用于添加接口震荡抑制。

网络应用中，由于物理信号干扰、链路层配置错误等原因可能导致设备接口频繁地交替出现Up和Down状态，造成路由协议反复震荡，对设备和网络产生较严重影响，甚至可能造成部分设备瘫痪，网络不可使用。

控制接口震荡特性用于降低接口状态变化对网络稳定性的影响。

给每个接口设置一个值，称为惩罚值（penalty value）。惩罚值的初始值为0，值越大说明该接口越不稳定。当某个接口的状态发生Up/Down变化时，就给予其惩罚，即：接口Down信号每到达一次就将其惩罚值增加一个固定值400，作为惩罚，而接口Up信号每到达一次，惩罚值进行一次指数衰减计算。当惩罚值达到一个设定的抑制门限（suppress value，简称suppress）时，就将该接口状态抑制住，即不上报该接口状态。惩罚值可随着时间推移按指数（半衰期法则）递减，当惩罚值下降到一个设定的重用门限（reuse value，简称reuse）时，就解除对该接口的抑制，重新上报告该接口状态信息。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为65535。
- 该命令的必选参数IFNAME可通过LST INTERFACE命令查询到。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 参数

| 参数标识 | 参数名称 | 参数说明 |
| --- | --- | --- |
| IFNAME | 接口名 | 可选必选说明：必选参数<br>参数含义：该参数用于指定震荡抑制的接口名。<br>数据来源：本端规划<br>取值范围：字符串类型，输入长度范围为1～63。接口名称由接口类型+接口编号组成。<br>默认值：无<br>配置原则：请使用LST INTERFACE命令查看可用接口。 |
| SUPPRESS | 接口抑制门限 | 可选必选说明：可选参数<br>参数含义：该参数用于指定接口抑制门限值。当接口抑制惩罚值超过此值时，接口被抑制。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～20000。接口抑制门限取值必须大于接口重用门限，且小于接口抑制门限最大值。<br>默认值：2000 |
| REUSE | 接口重用门限 | 可选必选说明：可选参数<br>参数含义：该参数用于指定接口可重用门限值。当接口惩罚值小于此值时，接口抑制被解除。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～20000。接口重用门限取值必须小于接口抑制门限。<br>默认值：750 |
| CEILING | 接口抑制门限最大值 | 可选必选说明：可选参数<br>参数含义：该参数用于指定接口抑制门限最大值。当接口抑制惩罚值超过此值时，接口抑制惩罚值不再增加。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1001～20000。接口抑制门限最大值取值必须大于接口抑制门限。<br>默认值：6000 |
| DECAYOK | 接口Up半衰期（s） | 可选必选说明：可选参数<br>参数含义：该参数用于指定当接口Up时，惩罚值半衰期。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～900，单位是秒。<br>默认值：54 |
| DECAKNG | 接口Down半衰期（s） | 可选必选说明：可选参数<br>参数含义：该参数用于指定当接口Down时，惩罚值半衰期。<br>数据来源：本端规划<br>取值范围：整数类型，取值范围为1～900，单位是秒。<br>默认值：54 |

## 操作的配置对象

- [接口震荡抑制（IFCONTROLFLAP）](configobject/UDG/20.15.2/IFCONTROLFLAP.md)

## 使用实例

添加接口震荡抑制：

```
ADD IFCONTROLFLAP:IFNAME="ethernet64/0/3",SUPPRESS=1001,REUSE=1000,CEILING=1002,DECAYOK=50,DECAKNG=50;
```
