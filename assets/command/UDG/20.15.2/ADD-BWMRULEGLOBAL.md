# ADD BWMRULEGLOBAL（增加全局带宽管理规则）

## 功能

**适用NF：PGW-U、UPF**

该命令用于增加一条全局业务带宽管理规则。当运营商希望规划整机用户使用相同的带宽控制规则进行控制时，执行该命令。

## 注意事项

- 该命令执行后立即生效。
- 该命令最大记录数为8。
- ADD BWMRULE命令与ADD BWMRULEGLOBAL命令同属于BwmRule对象，共享配置记录空间，即两个命令的总记录数最大为5000。
- 如果用户激活或者更新时，所属的用户组配置下没有用户级或用户组级的带宽管理规则，则该用户后续业务不进行带宽管理规则的匹配，后续再增加带宽管理规则的配置，也不生效。
- 配置全局带宽管理规则时，必须指定上行、下行或双向的带宽控制器，必须指定一个带宽管理业务和优先级，可以选择业务规则的时段，无需指定用户属性（接入类型、漫游类型）。
- 需要为每一个业务控制规则指定优先级，且不能相同。进行业务规则匹配时，优先匹配优先级高的业务控制规则，优先级的值越小，优先级越高。
- 全局的业务带宽管理规则只支持配置TOS类业务，如果将非TOS业务绑定到全局规则上，会提示失败，并且所有的全局带宽管理规则下的带宽管理业务的TOS值不能相同。
- 该命令设定后的数据，需要通过LST BWMRULEGLOBAL命令进行查看。
- BwmRuleGlobal不支持配置Shaping控制类型的BwmController。

## 权限

G_1，管理员级别命令组；G_2，操作员级别命令组

## 使用实例

假如运营商需要增加全局的带宽管理规则，名称为“testbwmruleglobal”，并分别配置上下行的带宽控制器：

```
ADD BWMRULEGLOBAL:BWMRULENAME="testbwmruleglobal",BWMSERVICENAME="bs1",UPBWMCTRLNAME1="testupbwmctrl",DNBWMCTRLNAME1="testdnbwmctrl",BWMRULEPRI=5;
```
