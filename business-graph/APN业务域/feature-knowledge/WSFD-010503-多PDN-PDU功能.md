## 1. 概述

### 1.1 特性定义（是什么）

多 PDN/PDU（Packet Data Network）功能是指**一个 UE 同时创建多个 PDN 连接（EPC）/ PDU 会话（5GC）来访问外部多个 PDN 网络**，满足用户同时访问多个网络的要求。

本特性是**会话管理基础（WSFD-010501）的多并发实例扩展**：WSFD-010501 定义"一个 UE 如何建立一个 PDN 连接/PDU 会话"的端到端流程（附着、建立、修改、释放），而 WSFD-010503 解答的是"一个 UE 能否在同一时刻并存 N 个这样的会话实例、并存数量上限是多少、如何在 APN 粒度约束并发"。

> 来源：`output/UNC 20.15.2 产品文档(裸机容器) 05/网络部署/特性部署/UNC特性指南/会话管理功能/WSFD-010503 支持多PDN_PDU功能/WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"定义"章节）

## 2. 核心概念

### 2.1 关键术语

| 术语 | 全称 | 说明 |
|------|------|------|
| PDN | Packet Data Network | 分组数据网络，用户访问的外部数据网络（如 Internet、企业专网） |
| PDN 连接 | PDN Connection | EPC 场景下 UE 到一个 PDN 的 IP 连接，每个连接由 PGW-C 分配独立 IP |
| PDU 会话 | PDU Session | 5GC 场景下 UE 到一个 DN 的会话，是 PDN 连接在 5G 的演进形态 |
| 缺省承载 | Default Bearer | EPC 中每个 PDN 连接建立时创建的默认 Non-GBR 承载 |
| 专有承载 | Dedicated Bearer | 在 PDN 连接之上为特定 QoS 需求触发的额外承载，每个 PDN 连接可触发一个或多个 |
| APN | Access Point Name | 接入点名称，标识 UE 访问的 PDN；本特性的单 APN 并发限制以 APN 为粒度 |
| TEID | Tunnel Endpoint Identifier | 隧道端点标识，多 PDN 场景下存在 TEID=0 的 PDN 连接并存问题（见软参 DWORD1021 BIT1） |
| 多 PDN / 多 PDU | Multi-PDN / Multi-PDU | 一个 UE 同时维护多个 PDN 连接或 PDU 会话的能力 |

> 来源：`WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"定义/原理概述"章节）、`WSFD-010503 支持多PDN_PDU功能参考信息_70362993.md`（"软参"章节）

### 2.2 对象模型

WSFD-010503 的核心是"并发会话实例 + 并发限制策略"，对象关系如下：

```
一个 UE（用户）
  │
  ├──[EPC 多 PDN]──→ 并发 1..N 个 PDN 连接（全局上限 11）
  │     │
  │     ├── 每个 PDN 连接 → 独立 IP 地址（由 PGW-C 分配）
  │     ├── 每个 PDN 连接 → 可经同一或不同 PGW-C
  │     ├── 每个 PDN 连接 → 一个缺省承载 + 0..N 个专有承载
  │     └── 单 APN 粒度约束 → APNACTNUM（PDNNUM/IPV4ADDRNUM/IPV6ADDRNUM）
  │
  └──[5GC 多 PDU]──→ 并发 1..N 个 PDU 会话（全局上限 15）
        │
        ├── 每个 PDU 会话 → 经由一个或多个 UPF
        └── 每个 PDU 会话 → 独立 DN
```

核心对象说明：

| 对象 | 作用 | 本特性角色 |
|------|------|-----------|
| PDN 连接 | EPC 中 UE 到 PDN 的 IP 连接 | 多 PDN 功能的并发实例单元，全局上限 11 |
| PDU 会话 | 5GC 中 UE 到 DN 的会话 | 多 PDU 功能的并发实例单元，全局上限 15 |
| APN | 接入点名称 | 作为单 APN 并发限制配置的粒度键 |
| APNACTNUM | APN 激活数目限制配置对象 | 定义"单 APN 单用户"下的 PDN 连接数/IP 地址数上限及拒绝原因值 |
| 缺省承载 | PDN 连接的默认承载 | 每个 PDN 连接的承载基础；多 PDN 的每条连接独立建立 |

> 来源：`WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"原理概述"章节）、`激活支持多PDN功能（适用于4G）_66843676.md`（ADD APNACTNUM 数据规划表）

### 2.3 在 APN 业务域的角色

WSFD-010503 在 APN 业务域中扮演"并发承载使能"的角色，是 WSFD-010501（单会话建立）向"多业务并行访问"扩展的关键环节：

1. **多网络并行访问**：允许用户同时访问 Internet + 企业专网 + IMS 等多个 PDN，每个连接独立 IP、独立承载
2. **多业务不中断**：用户在不中断当前业务的情况下启用另一种业务（如通话时仍保有 Internet 连接）
3. **并发规格治理**：通过全局上限（EPC:11 / 5GC:15）和单 APN 粒度上限，防止单用户过度占用资源
4. **与 WSFD-010501 的承接关系**：单 PDN/PDU 的建立流程完全复用 WSFD-010501；本特性只负责"并发上限判定 + 多实例并存"

> 来源：`WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"定义/客户价值/原理概述"章节）

## 3. 原理与流程

### 3.1 实现原理

#### 3.1.1 EPC 支持多 PDN 功能

EPC 支持一个 UE 同时创建多个 PDN 连接，通过一个或多个 PGW-C 与多个 PDN 网络同时进行 IP 业务交换。

**建立机制**：
1. UE 发起 PDN 连接建立请求
2. 由 MME 决定每个 PDN 连接应该使用的 SGW-C 及 PGW-C
3. 在缺省承载建立过程中或之后，PGW-C 针对每个 PDN 连接为 UE 分配一个独立的 IP 地址
4. UE 使用这些独立的 IP 地址访问对应的 PDN
5. 每个 PDN 连接均可触发一个或多个专有承载的建立

**并发规格（文档依据）**：
- **每个用户最多同时存在 11 个 PDN 连接**；超过 11 时，MME 拒绝建立新的 PDN 连接
- **单 APN 粒度可配**：MME 可通过 `ADD APNACTNUM` 配置特定 APN 下、同一用户最多可以同时存在的 PDN 连接数目和可分配的 IPv4/IPv6 地址数目；达到配置阈值时拒绝新连接

> 来源：`WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"原理概述/EPC支持多PDN功能"章节）

#### 3.1.2 5GC 支持多 PDU 功能

5GC 支持一个 UE 同时创建多个 PDU 会话，通过一个或多个 UPF 同时与多个 DN 网络进行 IP 业务。

**并发规格（文档依据）**：
- **每个用户最多同时存在 15 个 PDU 会话**；超过 15 时，SMF 拒绝新的 PDU 会话建立
- 单个 PDU 会话的建立流程参见 WSFD-010501（会话管理）的会话建立流程

> 来源：`WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"原理概述/5GC支持多PDU功能"章节）

#### 3.1.3 与 WSFD-010501（会话管理）的关系

| 维度 | WSFD-010501（会话管理） | WSFD-010503（多 PDN/PDU 功能） |
|------|------------------------|------------------------------|
| 定位 | 单 PDN/PDU 的建立/修改/释放流程 | 单 UE 多 PDN/PDU 并发实例的并存与上限治理 |
| 流程复用 | 定义基础会话流程 | 每个并发实例复用 010501 的建立流程 |
| 关键判定 | 单会话的附着/激活是否成功 | 是否允许再创建一个新会话实例（并发数是否达上限） |
| IP 地址 | 分配机制 | EPC 下每个 PDN 连接独立 IP；多 PDN = 多 IP |
| 承载关系 | 缺省承载 + 专有承载 | 每个 PDN 连接独立的缺省承载 + 各自的专有承载 |
| 拒绝机制 | 单会话鉴权/资源不足拒绝 | 并发超限拒绝（55/MULTI_PDN_FOR_APN_NOT_ALW_55、SMF 拒绝 PDU 建立） |

**核心结论**：WSFD-010503 不是新的会话建立机制，而是 WSFD-010501 的多实例并发治理层。脱离 010501 无法独立完成会话建立；脱离 010503 则只能维持单会话。

> 来源：`WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"原理概述"全章节，含对 WSFD-010201/WSFD-010501 的引用）

#### 3.1.4 TEID=0 并存控制（软参）

多 PDN 建立场景下，若已存在 TEID 为 0 的 PDN 连接，控制 TEID 也为 0 的新 PDN 连接建立时，是否会删除已有 TEID 为 0 的 PDN 连接，由软参 `DWORD1021 BIT1` 控制。

> 来源：`WSFD-010503 支持多PDN_PDU功能参考信息_70362993.md`（"软参"章节）

### 3.2 业务流程（EPC 多 PDN 并发建立）

```
┌─────────────────────────────────────────────────────────────────┐
│ 1. 初始状态                                                       │
│    UE 已附着，已有 0..K 个 PDN 连接（K < 11）                     │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. UE 发起新 PDN 连接建立请求                                      │
│    （携带目标 APN，如 "Internet" 或 "WAP"）                        │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. MME 并发判定（本特性核心）                                      │
│                                                                   │
│  3a. 全局判定：当前 UE 的 PDN 连接总数是否已达 11？                 │
│      → 是：MME 拒绝新 PDN 连接建立                                 │
│                                                                   │
│  3b. 单 APN 判定（若配置了 APNACTNUM）：                           │
│      当前 UE 在该 APN 下的 PDN 连接数 / IPv4 地址数 / IPv6 地址数   │
│      是否已达 PDNNUM / IPV4ADDRNUM / IPV6ADDRNUM 阈值？            │
│      → 是：MME 拒绝，向 UE 回 PDN 连接拒绝消息                     │
│         携带原因值 PDNCONNREJCAUSE（如 MULTI_PDN_FOR_APN_NOT_ALW_55）│
└──────────────────────────────┬──────────────────────────────────┘
                               │ 判定通过
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. MME 选择 SGW-C 与 PGW-C                                        │
│    （可与已有 PDN 连接共用，或选择新的网元）                        │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 5. 缺省承载建立（复用 WSFD-010501 / WSFD-010201 流程）              │
│    PGW-C 为该 PDN 连接分配独立 IP 地址                             │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────────┐
│ 6. 新 PDN 连接建立完成                                             │
│    UE 并发 PDN 连接数 +1，可独立访问对应 PDN                       │
└─────────────────────────────────────────────────────────────────┘
```

5GC 多 PDU 流程结构与之对偶，区别在于：判定网元为 SMF（而非 MME）、全局上限为 15（而非 11）、当前文档未提供单 DNN 粒度配置（与 4G 单 APN 配置能力存在代际差异）。

> 来源：`WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"原理概述"章节）、`调测支持多PDN功能（适用于4G）_70780933.md`（调测步骤反映的流程）

## 4. 配置规则

### 4.1 激活步骤

多 PDN/PDU 功能无需独立的"激活"开关（本特性无 License、无应用限制），其能力随 UNC 基础会话管理天然可用。配置重点是**单 APN 粒度的并发数限制**（仅 EPC 场景）：

```
步骤1：会话管理基础就绪
  └── 已激活 WSFD-010501 会话管理（4G/5G 基础会话流程可用）

步骤2：（可选）配置单 APN 并发数限制
  └── ADD APNACTNUM → 为特定 APN 配置 PDNNUM/IPV4ADDRNUM/IPV6ADDRNUM/PDNCONNREJCAUSE

步骤3：（可选）修改已有 APN 并发数限制
  └── MOD APNACTNUM

步骤4：（可选）查询 APN 并发数限制
  └── LST APNACTNUM

步骤5：（可选）删除 APN 并发数限制
  └── RMV APNACTNUM

注意：未配置 APNACTNUM 的 APN，仅受单用户全局上限（EPC:11 / 5GC:15）约束。
```

> 来源：`激活支持多PDN功能（适用于4G）_66843676.md`（"操作场景/操作步骤"章节）、`WSFD-010503 支持多PDN_PDU功能参考信息_70362993.md`（"命令"章节）

### 4.2 MML命令清单

| 命令 | 用途 | 多 PDN/PDU 场景关联 |
|------|------|--------------------|
| ADD APNACTNUM | 增加 APN 激活数目限制 | 配置"单 APN 单用户"的 PDN 连接数 / IPv4 地址数 / IPv6 地址数上限及拒绝原因值 |
| MOD APNACTNUM | 修改 APN 激活数目限制 | 修改已配置的 APN 并发阈值 |
| RMV APNACTNUM | 删除 APN 激活数目限制 | 删除某 APN 的并发限制（恢复仅受全局上限约束） |
| LST APNACTNUM | 查询 APN 激活数目限制 | 查询当前 APN 并发限制配置 |

> 来源：`WSFD-010503 支持多PDN_PDU功能参考信息_70362993.md`（"命令"章节）

### 4.3 参数说明

#### 4.3.1 ADD APNACTNUM 关键参数

| 参数 | 取值样例 | 说明 |
|------|---------|------|
| APNNI（APN 网络标识） | Internet / WAP | APN 名称，作为单 APN 并发限制的粒度键 |
| PDNNUM（PDN 连接数目） | 2 | 该 APN 下同一用户最多可同时存在的 PDN 连接数 |
| IPV4ADDRNUM（IPv4 地址数目） | 2 | 该 APN 下同一用户最多可分配的 IPv4 地址数 |
| IPV6ADDRNUM（IPv6 地址数目） | 2 | 该 APN 下同一用户最多可分配的 IPv6 地址数 |
| PDNCONNREJCAUSE（PDN 连接拒绝原因） | MULTI_PDN_FOR_APN_NOT_ALW_55 | 达到阈值时系统向终端回复的拒绝原因值 |

> 来源：`激活支持多PDN功能（适用于4G）_66843676.md`（"必备事项/数据"章节）

#### 4.3.2 相关软参

| 软参 | 说明 |
|------|------|
| DWORD1021 BIT1 | 多 PDN 建立场景下，若已存在 TEID 为 0 的 PDN 连接，控制 TEID 也为 0 的新 PDN 连接建立时，是否会删除已有 TEID 为 0 的 PDN 连接 |

> 来源：`WSFD-010503 支持多PDN_PDU功能参考信息_70362993.md`（"软参"章节）

### 4.4 约束条件

| 约束类型 | 约束内容 | 文档依据 |
|---------|---------|---------|
| EPC 单用户全局 PDN 上限 | 每个用户最多同时存在 **11 个** PDN 连接；超过 11 时 MME 拒绝新 PDN 连接建立 | "原理概述/EPC支持多PDN功能" |
| 5GC 单用户全局 PDU 上限 | 每个用户最多同时存在 **15 个** PDU 会话；超过 15 时 SMF 拒绝新 PDU 会话建立 | "原理概述/5GC支持多PDU功能" |
| 单 APN 粒度上限（EPC） | 可通过 ADD APNACTNUM 配置 PDNNUM/IPV4ADDRNUM/IPV6ADDRNUM；达到阈值拒绝 | "原理概述"及激活文档 |
| 每个并发 PDN 连接独立 IP | PGW-C 针对每个 PDN 连接为 UE 分配一个独立 IP 地址 | "原理概述/EPC支持多PDN功能" |
| 单 APN 限制仅 4G 文档化 | 当前产品文档仅提供 EPC(4G) 场景的单 APN 配置流程；5GC 场景未见等价 DNN 粒度配置 | 文档覆盖范围观察 |

> 来源：`WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（"原理概述"全章节）、`激活支持多PDN功能（适用于4G）_66843676.md`

## 5. 配置案例

### 5.1 典型场景：单 APN 多 PDN 连接并发限制（4G）

**场景描述**：用户需同时访问 Internet 和 WAP 两个网络（多 PDN 并发）。为防止单用户在同一 APN 下过度占用 PDN 连接和 IP 地址资源，对 APN "Internet" 和 "WAP" 各配置 PDN 连接最大数目为 2、IPv4/IPv6 地址最大数目为 2，超限回复拒绝原因值 "MULTI_PDN_FOR_APN_NOT_ALW_55"。

**配置步骤和 MML 命令序列**：

```
// 配置 APN "Internet" 下的并发限制
ADD APNACTNUM: APNNI="Internet", PDNNUM=2, IPV4ADDRNUM=2, IPV6ADDRNUM=2, PDNCONNREJCAUSE=MULTI_PDN_FOR_APN_NOT_ALW_55;

// 配置 APN "WAP" 下的并发限制
ADD APNACTNUM: APNNI="WAP", PDNNUM=2, IPV4ADDRNUM=2, IPV6ADDRNUM=2, PDNCONNREJCAUSE=MULTI_PDN_FOR_APN_NOT_ALW_55;
```

**并发行为说明**（基于上述配置）：
- 该用户在 APN "Internet" 下发起第 1、2 个 PDN 连接 → 建立成功（K=1, K=2）
- 该用户在 APN "Internet" 下发起第 3 个 PDN 连接 → UNC 向 UE 发送 PDN 连接拒绝消息，携带原因值 "Multiple PDN for APN not allowed"（MULTI_PDN_FOR_APN_NOT_ALW_55）
- 同时，该用户仍可在 APN "WAP" 下发起最多 2 个独立的 PDN 连接（单 APN 粒度独立计数）
- 跨 APN 总数仍受全局上限 11 约束

> 来源：`激活支持多PDN功能（适用于4G）_66843676.md`（"任务示例/脚本"章节）、`调测支持多PDN功能（适用于4G）_70780933.md`（调测预期结果）

**场景变体**：

| 变体 | 场景说明 | 核心差异 |
|------|---------|---------|
| 仅全局上限、无单 APN 限制 | 不配置 APNACTNUM | 仅受单用户全局 11 PDN（EPC）/ 15 PDU（5GC）约束 |
| 单 APN 高并发（如专网聚合） | Internet 配置 PDNNUM 较大值 | 允许同一 APN 下多 PDN，用于业务流量分流 |
| 5GC 多 PDU 会话 | UE 同时建立多个 PDU 会话到不同 DN | 全局上限 15；当前文档未提供单 DNN 粒度配置 |

## 6. 验证与调测

### 6.1 验证方法

#### 6.1.1 查询命令

| 验证目的 | 命令 | 说明 |
|---------|------|------|
| 查询 APN 并发限制配置 | LST APNACTNUM | 查看已配置 APN 的 PDNNUM/IPV4ADDRNUM/IPV6ADDRNUM/PDNCONNREJCAUSE |

#### 6.1.2 信令跟踪与并发调测（4G）

调测步骤（以 APN "Internet" 下 PDNNUM=2 为例）：
1. 在 UNC 操作维护台上创建用户跟踪任务
2. 用户开机发起附着，并发起针对 APN "Internet" 的 PDN 连接建立请求 → 预期：附着成功，PDN 连接 1 建立成功
3. 该用户使用相同 APN 再次发起 PDN 连接建立请求 → 预期：PDN 连接 2 建立成功（达到 PDNNUM=2）
4. 该用户使用相同 APN 再次发起 PDN 连接建立请求 → 预期：UNC 向 UE 发送 PDN 连接拒绝消息，携带原因值 "Multiple PDN for APN not allowed"

> 来源：`调测支持多PDN功能（适用于4G）_70780933.md`（"操作步骤"章节）

### 6.2 常见问题与排查

| 问题现象 | 可能原因 | 排查方法 |
|---------|---------|---------|
| UE 第 N 个 PDN 连接被拒绝（原因值 55） | 该 APN 下 PDN 连接数已达 PDNNUM 阈值 | LST APNACTNUM 查询配置；确认 UE 已建立的 PDN 连接数 |
| UE 第 N 个 PDN 连接被拒绝（非 55 原因值） | 单用户全局 PDN 总数已达 11 | 查询 UE 当前并发 PDN 连接总数；检查是否触达全局上限 |
| IPv4/IPv6 地址分配失败 | 该 APN 下 IP 地址数已达 IPV4ADDRNUM/IPV6ADDRNUM 阈值 | LST APNACTNUM 查询；核对 UE 已分配的 IP 地址数 |
| 5G PDU 会话被拒绝 | 单用户 PDU 会话总数已达 15 | 查询 UE 当前并发 PDU 会话总数；确认是否触达 5GC 全局上限 |
| 多 PDN 场景下 TEID=0 连接异常删除 | 软参 DWORD1021 BIT1 配置 | 检查 DWORD1021 BIT1 的设置，确认 TEID=0 并存控制策略 |
| 配置 APNACTNUM 后未生效 | 配置未提交或 APNNI 拼写不一致 | LST APNACTNUM 核对 APNNI；确认与 UE 请求中的 APN 完全一致 |

> 来源：综合 `WSFD-010503 支持多PDN_PDU功能特性概述_63778196.md`（原理与规格）、`WSFD-010503 支持多PDN_PDU功能参考信息_70362993.md`（软参）、`调测支持多PDN功能（适用于4G）_70780933.md`（调测预期）整理
