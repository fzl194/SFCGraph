# WSFD-010400 用户数据管理 知识文档

> 聚焦 APN 业务域的 UNC（SGSN/MME/AMF/SMF）控制面用户签约数据与上下文数据管理底座特性
> 与 WSFD-010501（会话管理）、WSFD-010502（地址分配）、WSFD-010301（鉴权功能）构成"签约数据 → 鉴权 → 会话 → 地址分配"纵向链路

## 1. 概述

### 1.1 特性定义（是什么）

UNC 的用户数据管理功能是指 **SGSN / MME / AMF / SMF 中用户数据的插入、修改与删除过程**。按接入代际不同，定义略有差异：

| 接入代际 | 控制面 NF（UNC 侧） | 签约数据源（后端） | 定义原文 |
|---------|--------------------|------------------|---------|
| 2G&3G | SGSN | HLR / HSS | "用户数据管理是指 SGSN 中用户数据的插入、删除与修改过程" |
| 4G | MME | HSS | "用户数据管理是指 MME 中用户数据的插入、删除与修改过程" |
| 5G | AMF、SMF | UDM | "用户数据管理是指 AMF、SMF 中用户数据的插入、修改与删除过程" |

**移动用户数据由两部分组成**（三代理绩统一表述）：

1. **签约数据（静态）**：移动用户在 UDM / HSS / HLR 中签约的信息，包括用户标识（IMSI/SUPI）、计费属性、APN/DNN 签约、QoS、ODB（运营商闭锁）、接入限制、GMLC 号码等
2. **动态上下文数据**：用户接入网络过程及会话/承载/PDP上下文过程中动态生成的数据，包括用户当前位置信息（TAI/RAI/ECGI）、当前会话为 UE 分配的 IP 地址、用户实际使用的 QoS 资源、连接的 S-GW/P-GW/GGSN 的 IP 地址、使用的 APN 等

本特性是 **APN 业务域的签约数据基础**：APN/DNN 签约、每 APN 的默认 QoS、APN-AMBR、VPLMN Address Allowed（漫游网关允许）、PDN Address（静态IP签约）、APN-OI Replacement 等所有 APN 相关签约字段，都通过本特性从 UDM/HSS/HLR 下发到 UNC 控制面，供 WSFD-010501 会话管理、WSFD-010502 地址分配、WSFD-010301 鉴权等特性读取使用。

> 来源：`WSFD-010400 用户数据管理（5G）_60374923.md`（"定义"章节）、`WSFD-010400 用户数据管理特性概述（适用于4G）_70018151.md`（"定义"章节）、`WSFD-010400 用户数据管理（适用于SGSN）特性概述_85436067.md`（"定义"章节）

## 2. 核心概念

### 2.1 关键术语

| 术语 | 全称 | 说明 |
|------|------|------|
| 签约数据 | Subscription Data | 用户在 UDM/HSS/HLR 中静态开户的数据（IMSI/SUPI、APN/DNN、QoS、计费属性、ODB 等） |
| 动态上下文数据 | Dynamic Context Data | 用户接入过程动态生成的数据（MM context、PDP/EPS/PDU context、当前位置、分配的IP等） |
| MM Context | Mobility Management Context | 移动性管理上下文（MM State、GUTI/P-TMSI、TAI/RAI、鉴权向量等） |
| PDP Context | Packet Data Protocol Context | 2/3G 会话上下文（PDP State、PDP Type/Address、APN、QoS、TEID 等） |
| EPS Bearer Context | EPS 承载上下文 | 4G 会话上下文（EPS Bearer ID、TFT、QoS、S1-u/S5/S8 TEID 等） |
| PDU Session Context | PDU 会话上下文 | 5G 会话上下文（DNN、PDU Session ID、QoS Flow、IP Address/Prefix 等） |
| UDM | Unified Data Management | 5G 统一数据管理网元（基于 SBA，Nudm 服务化接口） |
| HSS | Home Subscriber Server | 归属用户服务器（4G/EPC，S6a/S6d 接口，Diameter 协议） |
| HLR | Home Location Register | 归属位置寄存器（2/3G，Gr 接口，MAP 协议） |
| ODB | Operator Determined Barring | 运营商决定的闭锁业务 |
| APN-OI Replacement | APN Operator Identifier Replacement | 构建 PDN 网关 FQDN 时代替 APN OI 的域名（用户级与 APN 级，APN 级优先） |
| SUPI | Subscription Permanent Identifier | 5G 用户永久标识（对应 4G IMSI） |
| NSSAI | Network Slice Selection Assistance Information | 5G 网络切片选择辅助信息 |
| Super-Charger | Super-Charger 功能 | HLR/HSS 与 SGSN 均支持时，用户漫游不通知原 SGSN 删除数据（2/3G 优化） |
| Insert Subscriber Data | 插入签约数据流程 | UDM/HSS/HLR 主动向 UNC 插入/更新签约数据 |
| Cancel Location | 取消位置流程 | UDM/HSS/HLR 触发，删除 UNC 上的用户记录 |
| Purge UE/MS | 清除 UE/MS 流程 | UNC 发起，通知后端签约数据库用户数据已被本地清除 |
| Nudm_SDM | Nudm Subscriber Data Management | 5G 服务化接口（Get/Subscribe/Unsubscribe/Notification） |
| Nudm_UECM | Nudm UE Context Management | 5G 服务化接口（Get/Registration/Deregistration/Update/Notification） |

> 来源：`WSFD-010400 用户数据管理特性概述（适用于4G）_70018151.md`（表1 HSS签约数据、表2 MM上下文和EPS承载上下文）、`WSFD-010400 用户数据管理（适用于SGSN）特性概述_85436067.md`（表1 HLR GPRS签约数据、表2 MM上下文和PDP上下文）、`WSFD-010400 用户数据管理（5G）_60374923.md`（用户数据定义、Nudm接口流程）

### 2.2 对象模型（签约数据 + 动态上下文）

用户数据管理特性的对象模型由"**两类数据 × 三代际**"构成，核心结构如下：

```
┌─────────────────────────────────────────────────────────────────────┐
│ 后端签约数据库（数据源）                                                │
│                                                                     │
│   5G: UDM  ←─ Nudm 服务化接口 (HTTP/2) ─→  AMF / SMF                  │
│   4G: HSS  ←─ S6a (Diameter) ─→  MME                                 │
│       HSS  ←─ S6d (Diameter) ─→  SGSN                                │
│   2/3G: HLR ←─ Gr (MAP) ─→  SGSN                                     │
│                                                                     │
│   存储内容：                                                          │
│     - 用户标识 (IMSI/SUPI, MSISDN, IMEI/IMEISV)                      │
│     - APN/DNN 签约（每 APN：Context ID, APN, PDN Type,                │
│       PDN Address(静态), QoS, APN-AMBR, Charging Characteristics,    │
│       VPLMN Address Allowed, PDN GW identity）                        │
│     - 接入限制 (Access-Restriction-Data, Subscriber-Status, ODB)      │
│     - 计费属性 (3GPP-Charging-Characteristics)                        │
│     - QoS 签约 (UE-AMBR, Default QCI/ARP)                             │
│     - 5G 独有：NSSAI、allowed/default SSC Mode                       │
│     - APN-OI Replacement（用户级 + APN 级）                            │
└──────────────────────────┬──────────────────────────────────────────┘
                           │
                           ▼ Insert/Update/Delete/Subscribe/Notify
┌─────────────────────────────────────────────────────────────────────┐
│ UNC 控制面本地缓存（SGSN/MME/AMF/SMF）                                │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────┐      │
│   │ 签约数据缓存（静态，从后端复制）                              │      │
│   │   - 减少 UNC ↔ UDM/HSS/HLR 的信令交互                       │      │
│   │   - 用户在服务区内活动时一直保留                              │      │
│   │   - 用户分离后一段时间（PURGTMR 可配）主动清除                │      │
│   └──────────────────────────────────────────────────────────┘      │
│                                                                     │
│   ┌──────────────────────────────────────────────────────────┐      │
│   │ 动态上下文数据（运行时生成）                                  │      │
│   │   - MM Context（移动性管理状态、位置、鉴权向量）              │      │
│   │   - 会话上下文：                                              │      │
│   │     2/3G: PDP Context（PDP State, APN in Use, QoS Negotiated）│      │
│   │     4G:   EPS Bearer Context（Bearer ID, TFT, QoS, TEID）    │      │
│   │     5G:   PDU Session Context（DNN, IP/Prefix, QoS Flow）    │      │
│   │   - 当前连接的用户面网元 IP/TEID（GGSN/SGW/PGW/UPF）         │      │
│   └──────────────────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────────────────┘
```

核心对象说明：

| 对象 | 作用 | APN 业务域角色 |
|------|------|---------------|
| 签约数据缓存 | 本地保存从 UDM/HSS/HLR 复制的签约数据，减少信令交互 | **APN/DNN 签约、APN-AMBR、PDN Address（静态IP）的核心载体** |
| MM Context | 移动性管理上下文（用户当前状态、位置、鉴权向量） | 承载 Access Restriction、ODB 等接入控制字段 |
| PDP Context（2/3G） | PDP 会话上下文 | 包含 APN Subscribed、APN in Use、QoS Negotiated |
| EPS Bearer Context（4G） | EPS 承载上下文 | 包含 APN in Use、APN Restriction、PDN Type、IP Address、默认承载 |
| PDU Session Context（5G） | PDU 会话上下文 | 包含 DNN、PDU Session ID、分配的 IP/Prefix |

## 3. 原理与流程

### 3.1 用户数据管理功能（三代理绩统一描述）

UNC 的用户数据管理功能包括以下几个方面的内容（三代理绩表述高度一致）：

| # | 功能 | 2G&3G (SGSN) | 4G (MME) | 5G (AMF/SMF) |
|---|------|--------------|----------|--------------|
| 1 | **初始签约数据拉取** | 用户首次附着或 RAU 到新 SGSN 时，SGSN 主动向 HLR/HSS 请求签约数据 | 用户首次附着或 TAU 到新 MME 时，MME 主动向 HSS 请求签约数据 | 用户首次注册或 TAU 到新 AMF/SMF 时，AMF/SMF 主动向 UDM 请求签约数据 |
| 2 | **签约数据主动推送** | HLR/HSS 签约数据改变（如 QoS）时，主动向 SGSN 插入更新后数据 | HSS 签约数据改变时，主动向 MME 插入更新后数据 | UDM 签约数据改变时，主动向已订阅的 AMF/SMF 推送变化通知 |
| 3 | **特性兼容性检查** | SGSN 检查签约特性（ODB、业务等），不支持则通知 HLR/HSS，由 HLR/HSS 决定是否允许接入 | MME 检查签约特性（ODB、业务等），不支持则通知 HSS | AMF/SMF 检查签约特性（业务功能项），不支持则通知 UDM |
| 4 | **本地缓存策略** | 用户在服务区内活动时一直保留数据；分离后一段时间（SET SDBTMR PURGTMR 可配）主动清除 | 用户在服务区内活动时一直保留数据；分离后一段时间（可由维护人员设置）主动清除 | 用户在服务区内活动时一直保留数据；分离后一段时间主动清除 |
| 5 | **漫游数据清理** | 用户漫游到其它 SGSN 时，HLR/HSS 通知原 SGSN 删除（Super-Charger 优化除外） | 用户漫游到其它 MME 时，HSS 通知原 MME 删除 | （5G 文档未展开，由 Deregistration 流程承载） |

> 来源：三代理绩特性概述的"原理概述/用户数据管理功能"章节

### 3.2 5G 用户数据管理流程（Nudm 服务化接口，★本特性演进重点）

#### 3.2.1 用户数据管理流程（Nudm_SDM 服务族）

```
┌─────────────────────────────────────────────────────────────────┐

│   用途：AMF/SMF 主动获取用户数据，如 UE 的 NSSAI、DNN 签约、       │
│         SSC Mode、静态IP/前缀等                                    │

└─────────────────────────────────────────────────────────────────┘

│   用途：AMF/SMF 订阅 UDM 中用户数据的变化通知                      │
└─────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────┐
│ Nudm_SDM_Unsubscribe（AMF/SMF → UDM）                            │
│   用途：AMF/SMF 取消订阅 UDM 中用户数据的变化通知                  │
└─────────────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────────────┐
│ Nudm_SDM_Notification（UDM → AMF/SMF）                           │
│   用途：UDM 通知订阅过数据变化通知的 AMF/SMF 相关用户数据变化       │
│   （对应 4G 的 Insert/Delete Subscriber Data 主动推送语义）        │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.2.2 UE 上下文管理流程（Nudm_UECM 服务族）

```
┌─────────────────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────────┘
```

### 3.3 4G 用户数据管理流程（MME ↔ HSS，Diameter/S6a）

4G 共 6 个用户数据管理相关流程（除 Notification 仅适用 MME 外，其他同时适用 MME 和 SGSN）：

| # | 流程 | 发起方 | 用途 |
|---|------|-------|------|
| 1 | **Insert Subscriber Data** | HSS | 插入新增的用户签约数据（含 APN、QoS 等改变后的主动推送） |
| 2 | **Delete Subscriber Data** | HSS | 删除部分或全部 HSS 存储在 MME 的数据（如 EPS 签约数据 APN 配置侧、区域签约等） |
| 3 | **Purge UE** | MME | MME 删除用户信息后通知 HSS（UE 长时间非活跃态自动删除等） |
| 4 | **Update Location** | MME | 更新 HSS 上的位置信息；通知 HSS 当前服务 MME 标识；更新 MME 签约数据；提供终端信息给 HSS |
| 5 | **Cancel Location** | HSS | 删除 MME 上的用户记录（取消签约信息，或通知 MME 进行 MME 改变等流程） |
| 6 | **Notification**（仅 MME） | MME | 通知 HSS 多类信息（见下表） |

**Notification 流程的具体触发条件**（4G 独有，MME→HSS）：

| 触发场景 |
|---------|
| Inter-MME Update Location 流程没有发生，通知 HSS 终端升级的信息 |
| Inter-MME Update Location 流程没有发生，通知 HSS 需要发送 Cancel Location 消息给当前的 SGSN |
| **通知 HSS 动态分配给 APN 的 PDN 网关的分配、更改和移除信息** ★APN 业务域相关 |
| 通知 HSS 用户变为可达 |
| 通知 HSS IMEI 或 SRVCC 发生改变 |
| 通知 HSS Paging Response 时的签约数据未被证实 |

> ★关键：Notification 流程中"通知 HSS 动态分配给 APN 的 PDN 网关的分配/更改/移除"是 APN 业务域的**反向数据流**（UNC→HSS），用于 HSS 后续 PDN GW 选择（静态分配保持不变，动态分配可更新）

**Authentication 流程**：4G 实现原理文档明确"Authentication 流程请参考 WSFD-010301 鉴权功能"，本特性不独立承载鉴权流程。

> 来源：`实现原理（适用于4G）_70018152.md`（"用户数据管理流程"章节，含图1-图6）

### 3.4 2G&3G 用户数据管理流程（SGSN ↔ HLR/Gr-MAP + SGSN ↔ HSS/S6d-Diameter）

**双接口架构**：SGSN 与 HLR 通过 **Gr 接口（MAP 协议）** 交互，与 HSS 通过 **S6d 接口（Diameter 协议）** 交互。

SGSN 与 HSS 间的流程**复用 4G MME↔HSS 流程**（Insert/Delete/Purge/Update/Cancel/Notification，参考 4G 实现原理）。

SGSN 与 HLR（Gr 接口）的用户数据管理流程：

| # | 流程 | 发起方 | 用途 |
|---|------|-------|------|
| 1 | **Insert Subscriber Data** | HLR | 插入新增的用户签约数据（HLR 发起） |
| 2 | **Purge MS**（注意是 MS，非 UE） | SGSN | SGSN 删除用户信息后通知 HLR（UE 长时间非活跃态自动删除等） |
| 3 | **Cancel Location** | HLR | HLR 和 SGSN 之间 Gr 接口上删除 SGSN 上的用户记录（取消签约信息，或通知 SGSN 进行 SGSN 改变等流程） |
| 4 | **Delete Subscriber Data** | HLR | 删除部分或全部 HLR 存储在 SGSN 的数据（HLR 通知 SGSN 删除 PDP 上下文） |

**Authentication 流程**：2/3G 实现原理文档明确"Authentication 流程请参考 WSFD-010301 鉴权功能（适用于SGSN）"。

> 来源：`实现原理（适用于2G&3G）_85436068.md`（"用户数据管理流程"章节，含图1-图4）

### 3.5 ★签约数据如何影响地址分配（与 WSFD-010502 的关系，文档依据）

这是本特性在 APN 业务域的**最关键纵向链路**。文档依据来自 WSFD-010502 地址分配方式特性概述"UDM 分配方式"章节：

```
┌─────────────────────────────────────────────────────────────────┐
│ 5G 场景：SMF 通过 Nudm_SDM_Get 获取静态 IP                        │
│                                                                 │
│   1. UE 发起 PDU 会话建立请求                                     │
│   2. SMF 调用 UDM 服务操作 Nudm_SDM_Get                          │
│   3. UDM 返回 UE 在 UDM 签约时确定的静态 IP 地址/前缀              │
│      （★本特性承载的签约数据）                                     │
│   4. SMF 通过 PDU 会话建立响应消息将静态 IP 发送给 UE              │
│                                                                 │
│   → 若本特性未下发静态IP签约，SMF 无法使用 UDM 分配方式            │
│   → 签约数据的存在性直接决定地址分配方式的可选性                    │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ 4G 场景：MME 从 HSS 获取静态 IP                                   │
│                                                                 │
│   1. 默认承载上下文激活过程                                        │
│   2. MME 根据 UE 的请求和从 HSS 处获取的签约数据                   │
│      （★本特性通过 Insert/Update Location 下发）                  │
│   3. MME 在 Create Session Request 消息的 PAA 信元中携带           │
│      已获取的 IPv4 地址或 IPv6 前缀给 SGW-C                       │
│   4. SGW-C 通过 Create Session Request 将 PAA 信元发送给 PGW-C    │
│   5. PGW-C 将收到的静态 IP 地址发送给 UE 使用                      │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│ 2/3G 场景：SGSN 验证 PDP 地址与签约记录                            │
│                                                                 │
│   1. UE 发送 Activate PDP Context Request 给 SGSN                │
│      消息的 PDP Address 信元携带 UE 的 IPv4 地址/IPv6 前缀         │
│   2. SGSN 验证 UE 提供的 PDP 地址和 PDP 上下文签约记录             │
│      （★本特性从 HLR 获取的签约数据）                              │
│   3. SGSN 通过 Create PDP Context Request 的 End User Address     │
│      信元将 UE 的 IP 地址信息发送给 GGSN                          │
│   4. GGSN 将收到的静态 IP 地址/IPv6 前缀发送给 UE 使用              │
└─────────────────────────────────────────────────────────────────┘
```

> 来源：`WSFD-010502 地址分配方式特性概述_64915075.md`（"UDM分配方式"章节，三代理绩完整流程描述）

### 3.6 ★签约数据如何影响鉴权（与 WSFD-010301 的关系）

签约数据中的鉴权相关字段通过本特性下发，影响鉴权决策：

- **2/3G**：Authentication Vectors（鉴权三元组/五元组）虽由 HLR/Gr-MAP 下发，但鉴权流程本身归属 WSFD-010301 鉴权功能（本特性 2/3G 实现原理文档明确指向）
- **4G**：Authentication Vector（RAND/XRES/KASME/AUTN）存储在 MM Context 中（表2），由 HSS 通过 Authentication 流程下发（归属 WSFD-010301）
- **接入控制类签约**：Access-Restriction-Data、Subscriber-Status、Operator-Determined-Barring、Network-Access-Mode、HPLMN-ODB、Regional-Subscription-Zone-Code、Roaming-Restricted-Due-To-Unsupported-Feature 等字段（表1加粗字段），由本特性下发，影响用户是否允许接入

> 来源：4G 特性概述表1（接入限制类加粗字段）、2/3G 特性概述表1（同字段）

### 3.7 签约数据存储分拆机制（2/3G 与 4G 共有，SET SYS 配置）

**GPRS 签约数据与 EPS 签约数据的分开存储**：

- 表1中加粗字段是 GPRS 签约数据和 EPS 签约数据公有字段，共 **10 个**
- 除 MSISDN、GMLC-Number 字段外，其他 **8 个字段实现了分开存储**
- 通过 `SET SYS` 命令的 **"SUBSTORAG"** 参数配置存储方式
- **修改签约数据存储方式后，在用户发生移动性管理流程时生效**（非立即生效）

> 来源：4G 特性概述表1备注、2/3G 特性概述表1备注（原文统一）

## 5. 配置案例

### 5.1 典型场景一：5G SMF 获取静态 IP 签约数据（Nudm_SDM_Get，★APN 业务域核心）

**场景描述**：企业用户定制服务场景，UE 在 UDM 签约时确定了静态 IP 地址/前缀，SMF 在 PDU 会话建立时通过 Nudm_SDM_Get 获取该静态 IP，并通过 PDU 会话建立响应发送给 UE。

**配置/触发流程（原始消息流）**：

```
1. UE → AMF：PDU Session Establishment Request（NAS 消息）
   携带：DNN、PDU Session ID、Request Type、SSC Mode 等

2. AMF → SMF：调用 SMF 的 Nsmf_PDUSession_CreateSMContext（透传 SM NAS）

3. SMF → UDM：Nudm_SDM_Get
   ★ 本特性核心服务操作
   请求获取：DNN 签约、allowed/default SSC Mode、UE 在 UDM 签约时确定的静态 IP/前缀

4. UDM → SMF：Nudm_SDM_Get Response
   返回：静态 IP 地址/前缀（来自本特性管理的签约数据）

5. SMF 决策地址分配方式：UDM静态分配（因签约数据中存在静态IP）

6. SMF → AMF → UE：PDU Session Establishment Accept
   携带：静态 IP 地址/前缀
```

> 来源：`WSFD-010502 地址分配方式特性概述_64915075.md`（"UDM分配方式"章节，5G场景原文："SMF 通过调用 UDM 的服务操作 Nudm_SDM_Get 获取 UE 在 UDM 签约时确定的静态 IP 地址/前缀，之后通过 PDU 会话建立响应消息发送给 UE"）

### 5.2 典型场景二：4G HSS 主动插入更新后的 APN 签约数据（Insert Subscriber Data）

**场景描述**：用户已在 MME 附着，运营商标通过 HSS 修改了用户的 APN 签约（如新增 APN、修改 QoS、修改 APN-AMBR）。HSS 主动向 MME 插入更新后的签约数据。

**配置/触发流程（基于 4G 实现原理图1）**：

```
1. 运营商在 HSS 修改用户签约数据（新增 APN / 修改 QoS / 修改 APN-AMBR 等）

2. HSS → MME：Insert Subscriber Data Request（Diameter，S6a 接口）
   携带：更新后的 APN 签约上下文（Context Identifier、APN、PDN Type、
        EPS subscribed QoS profile、Subscribed-APN-AMBR、
        VPLMN Address Allowed、PDN GW identity 等）

3. MME 检查签约特性兼容性
   ├── 支持：本地缓存更新，后续会话管理流程使用新签约
   └── 不支持：通知 HSS，由 HSS 决定是否允许用户接入本 MME

4. MME → HSS：Insert Subscriber Data Answer

5. 后续该用户发起 PDN 连接建立时，MME 使用更新后的 APN 签约数据
```

> 来源：`实现原理（适用于4G）_70018152.md`（图1 Insert Subscriber Data 流程 + 4G 特性概述"用户数据管理功能"章节第2条）

### 5.3 典型场景三：2/3G SGSN 清除非活跃用户数据（Purge MS）

**场景描述**：用户长时间处于非活跃状态，SGSN 自动删除用户信息后通知 HLR（SET SDBTMR PURGTMR 配置超时时间）。

**配置/触发流程（基于 2/3G 实现原理图2）**：

```
1. UE 长时间非活跃（超过 SET SDBTMR PURGTMR 配置的时间）

2. SGSN 自动删除用户信息（释放占用资源）

3. SGSN → HLR：Purge MS（Gr 接口，MAP 协议）
   ★ 本特性独有流程（2/3G 为 Purge MS，4G 为 Purge UE，5G 为 Deregistration）

4. HLR 记录该 UE 的 MM 和 PDP 上下文已从 SGSN 上删除
   （MS PS Purged for GPRS 字段置位）
```

> 来源：`实现原理（适用于2G&3G）_85436068.md`（图2 Purge MS 流程）+ 2/3G 特性概述"用户数据管理功能"章节第4条

### 5.4 典型场景四：4G MME 通知 HSS 动态 PDN GW 分配（Notification，★APN 业务域相关）

**场景描述**：MME 为用户的某 APN 动态分配了 PDN 网关，需通知 HSS 更新 APN-PDN 网关 ID 关系列表（含野卡 APN 的 PDN 签约上下文），供 HSS 后续 PDN GW 选择。

**配置/触发流程（基于 4G 实现原理图6）**：

```
1. MME 为用户 APN 动态分配/更改/移除 PDN 网关

2. MME → HSS：Notification（Diameter，S6a 接口）
   携带：APN - PGW relation #n（APN 与 PDN 连接的动态分配 PDN 网关标识）
   ★ 反向数据流：UNC → HSS

3. HSS 更新 APN-PDN网关ID关系列表（含野卡APN的PDN签约上下文）

4. 后续该用户 PDN GW 选择时，静态分配保持不变，动态分配可使用更新后的 PDN GW
```

> 来源：`实现原理（适用于4G）_70018152.md`（图6 Notification 流程，原文触发条件之一："通知 HSS 动态分配给 APN 的 PDN 网关的分配、更改和移除信息"） + 4G 特性概述表1（APN-PDN网关ID关系列表字段）

### 6.5 常见问题与排查

| 问题现象 | 可能原因 | 排查方法 |
|---------|---------|---------|
| 5G PDU 会话建立失败（UDM 静态地址分配方式不生效） | UDM 未签约静态 IP/前缀，Nudm_SDM_Get 返回空 | 核对 UDM 签约：DNN、Session Management Subscription 中是否包含静态 IP/前缀 |
| 5G SMF 签约数据未及时更新 | SMF 未订阅 UDM 数据变化（Nudm_SDM_Subscribe 未发起） | 跟踪 Nudm_SDM_Subscribe/Notification 消息；检查 SMF 订阅状态 |
| 5G AMF/SMF 获取签约数据失败 | Nudm 接口（HTTP/2）网络不通；UDM 服务不可用 | 检查 AMF/SMF 与 UDM 间 SBA 连接；检查 UDM 服务健康状态 |
| 4G APN 签约数据未更新 | HSS Insert Subscriber Data 未送达 MME；MME 特性兼容性检查拒绝 | 跟踪 S6a 接口 Insert Subscriber Data 消息；检查 MME 是否通知 HSS 不支持某特性 |
| 4G PDN 连接建立时静态 IP 不生效 | HSS 签约 PDN Address 为空（动态）；PDN GW Allocation Type 配置问题 | 核对 HSS 表1 PDN Address 字段；PDN GW Allocation Type（静态/动态） |
| 4G HSS 位置信息未更新 | MME Update Location 未发起；Notification 流程异常 | 跟踪 S6a Update Location / Notification 消息 |
| 2/3G PDP 激活失败（静态地址校验不通过） | HLR 签约 PDP Address 与 UE 提供不一致；PDP 上下文签约记录缺失 | 核对 HLR 表1 PDP Address 字段；SGSN 验证 PDP 地址与签约记录 |
| 2/3G SGSN 用户数据未及时清理 | SET SDBTMR PURGTMR 配置过长；Super-Charger 功能导致不清理 | 检查 PURGTMR 配置；评估 Super-Charger 启用状态 |
| 2/3G 漫游用户签约数据异常 | SGSN↔HLR（Gr-MAP）或 SGSN↔HSS（S6d-Diameter）双接口数据不一致 | 核对 HLR 与 HSS 签约数据；检查双接口信令 |
| GPRS/EPS 签约数据存储方式切换未生效 | SET SYS SUBSTORAG 修改后未触发移动性管理流程 | 触发用户附着/TAU/RAU 流程使配置生效 |
| APN 限制配置异常传播 | APN 限制不应在 SGSN 间转发（2/3G 约束） | 检查移动性管理过程中 APN 限制字段的传播路径 |
