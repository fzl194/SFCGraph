## 1. 概述

### 1.1 特性定义（是什么）

本特性是 **UNC 控制面（C 面）用户接入控制决策特性的总集**：UNC 在用户接入（注册/附着/路由区更新/切换）过程中，根据**用户签约数据**和（或）**UNC 本地配置**判断是否允许用户接入网络，并决策用户可使用的无线接入方式（RAT）和核心网类型。

**同一特性ID含两个实现独立的子特性**（★重点）：

| 子特性 | 适用NF | 接入域 | 定义原文 | 控制依据 | 数据来源 |
|--------|--------|--------|---------|---------|---------|
| **子特性A：AMF 侧** | AMF | 5GC | "UNC 支持根据用户签约或者本地配置的移动性接入限制信息判断是否允许用户注册到5G核心网的功能" | RAT限制（RAT restriction）+ 核心网类型限制（Core Network type restriction） | UDM 签约数据 + AMF 本地配置 |
| **子特性B：SGSN/MME 侧** | SGSN、MME | 2G&3G&4G（GERAN/UTRAN/E-UTRAN） | "UNC 支持先根据IMSI号段将用户进行分类，再对于每一类用户按照用户属性进行区分而控制用户接入 GERAN、UTRAN和 E-UTRAN网络" | 签约ARD（Access Restriction Data）+ 签约APN + 卡类型（SIM/USIM） | HLR/HSS 签约数据 + UNC 本地配置 |

> 来源：`用户接入控制功能（适用于AMF）/特性概述_48552635.md`（"定义"章节）、`用户接入控制功能（适用于SGSN、MME）/特性概述_68358176.md`（"定义"章节）

### 1.8 应用场景

#### 子特性A（AMF 侧）应用场景

- 运营商希望根据用户签约/AMF本地配置的"RAT限制（RAT restriction）"信息控制用户的无线接入方式
- 运营商希望根据用户签约/AMF本地配置的"核心网类型限制（Core Network type restriction）"信息控制用户的核心网接入方式

#### 子特性B（SGSN/MME 侧）应用场景

- 若运营商希望**拒绝某个号段接入 GERAN、UTRAN 或 E-UTRAN**，则可以启用**根据签约ARD控制用户接入**特性
- 若运营商希望**控制某类用户是否允许接入 GERAN、UTRAN 或 E-UTRAN，但 HLR/HSS 不支持 ARD** 时，则可以启用**根据签约APN控制用户接入**特性
- 若运营商已经强制要求对 GERAN 用户使用 SIM 卡、UTRAN 用户使用 USIM 卡，并希望对它们进行灵活的接入控制，就需要启用**按SIM卡/USIM卡控制用户接入**特性

> 来源：`用户接入控制功能（适用于AMF）/特性概述_48552635.md`（"应用场景"章节）、`用户接入控制功能（适用于SGSN、MME）/特性概述_68358176.md`（"应用场景"章节）

| 限制类型 | 说明 |

| **紧急注册跳过** | 用户接入控制功能对于**紧急注册不生效**，如果是紧急注册，则**跳过用户接入控制检查** |

## 2. 核心概念

### 2.1 关键术语

#### 通用术语

| 术语 | 全称 | 说明 |
|------|------|------|
| 用户接入控制 | User Access Control | UNC 在用户接入（注册/附着/路由区更新/切换）过程中判断是否允许用户接入网络的功能 |
| UNC | Universal Network Controller | 统一网络控制器（控制面网元集合：AMF/SGSN/MME/SMF/PGW-C/GGSN） |
| RAT | Radio Access Technology | 无线接入技术（如 NR/EUTRA/UTRAN/GERAN） |
| IMSI | International Mobile Subscriber Identity | 国际移动用户标识，用于用户号段识别 |
| IMSI前缀 | IMSI Prefix | IMSI 号段的匹配前缀，用于本地配置的用户范围划分 |

#### 子特性A（AMF 侧）专属术语

| 术语 | 全称 | 说明 |
|------|------|------|
| 移动性接入限制信息 | Mobility Access Restriction Data | 5G协议定义的移动性限制信息，包含RAT限制、核心网限制、禁止区域、服务区限制四种类型 |
| **RAT限制** | **RAT restriction** | 限制的3GPP内的无线接入方式，指示受限的RAT类型列表（如限制NR/EUTRA接入5GC） |
| **核心网限制** | **Core Network type restriction** | 限制的核心网接入方式，指示受限的核心网类型列表（如限制5GC/EPC接入） |
| 禁止区域 | Forbidden Area | 5G移动性限制的一种，由WSFD-105003区域漫游限制特性实现 |
| 服务区限制 | Service Area Restriction | 5G移动性限制的一种 |
| ARD（5G侧） | Access Restriction Data | 2/3/4G签约数据中的接入限制数据，在互操作流程中可部分转换为5G的RAT限制 |
| RedCap | Reduced Capability | 降低能力的NR终端子类型（v02+新增针对RedCap用户的RAT接入限制原则） |
| MRL | Mobility Restriction List | 移动性限制列表，核心网通过该信元将限制信息下发给基站 |

#### 子特性B（SGSN/MME 侧）专属术语

| 术语 | 全称 | 说明 |
|------|------|------|
| **ARD** | **Access Restriction Data** | 接入限制数据。3GPP R6协议在HLR/HSS签约数据中加入的ARD信息，指示是否拒绝用户接入GERAN、UTRAN或E-UTRAN |
| AccessRestrictionData | - | HLR/HSS与SGSN/MME间传递ARD的信元（bit0~bit7，bit0/bit1/bit4分别代表UTRAN/GERAN/E-UTRAN，bit6代表NB-IoT，取值1=拒绝，0=允许） |
| 特殊APNNI | - | HLR/HSS中为用户签约的特殊APN网络标识，用于控制用户接入GERAN/UTRAN/E-UTRAN |
| SIM卡 | Subscriber Identity Module | 用户识别模块（智能卡），GSM/UMTS网络可用，鉴权信息为三元组 |
| USIM卡 | UMTS Subscriber Identity Module | UMTS用户识别模块，GSM/UMTS/LTE网络可用，鉴权信息为五元组 |
| Gb模式 | - | GERAN接入模式（对应SGSN的Gb接口） |
| Iu模式 | - | UTRAN接入模式（对应SGSN的Iu接口） |
| S1模式 | - | E-UTRAN接入模式（对应MME的S1接口） |

### 2.2 对象模型

#### 子特性A（AMF 侧）对象模型

AMF 侧基于"移动性接入限制信息（RAT限制 + 核心网限制）"的二元决策模型：

```
┌─────────────────────────────────────────────────────────────────┐
│ 子特性A（AMF 侧）配置与决策体系                                   │
│                                                                 │
│   ┌────────────────────────────────────────────────────┐        │
│   │  数据来源1：UDM 签约数据                            │        │
│   │    AccessAndMobilitySubscriptionData               │        │
│   │      ├── RAT限制（RAT restriction）                │        │
│   │      └── 核心网限制（Core Network type restriction）│        │
│   └────────────────────────────────────────────────────┘        │
│                                                                 │
│   ┌────────────────────────────────────────────────────┐        │
│   │  数据来源2：AMF 本地配置（ADD NGMMSUBDATA）          │        │
│   │    按用户范围（SUBRANGE）匹配，优先级：              │        │
│   │    IMSI前缀 > 外网用户 > 本网用户 > 所有用户        │        │
│   │      ├── IMSIPRE（IMSI前缀）                       │        │
│   │      ├── RATRESTRICT（RAT限制）                    │        │
│   │      ├── CORERESTRICT（核心网类型限制）             │        │
│   │      └── NIM（网络切片包含模式策略）                │        │
│   └────────────────────────────────────────────────────┘        │
│                                                                 │
│   决策优先级：本地配置 > 签约数据                                │
│                                                                 │
│   ┌────────────────────────────────────────────────────┐        │
│   │  AMF 接入控制检查（注册/移动性注册更新/切换）       │        │
│   │    输入：当前UE使用的RAT类型 + 移动性接入限制信息   │        │
│   │    输出：允许接入 / 拒绝接入（携带原因值）          │        │
│   │    拒绝原因值配置：SET NGMMPROCTRL（ACCRST参数）    │        │
│   └────────────────────────────────────────────────────┘        │
└─────────────────────────────────────────────────────────────────┘
```

#### 子特性B（SGSN/MME 侧）对象模型

SGSN/MME 侧基于"IMSI号段分类 + 用户属性匹配（ARD/APN/卡类型）"的多维决策模型：


## 3. 原理与流程

### 3.1 子特性A（AMF 侧）实现原理

#### 3.1.1 5G移动性限制背景

5G协议明确定义了**移动性接入限制信息**有四种类型：

1. **RAT限制（RAT restriction）** ← 本特性实现
2. **核心网限制（Core Network type restriction）** ← 本特性实现
3. **禁止区域（Forbidden Area）** ← 由 WSFD-105003 区域漫游限制实现
4. **服务区限制（Service Area Restriction）** ← 由服务区域限制特性实现

5G系统中的移动性限制由 UE、基站、核心网协同完成：
- **UE**：在某区域尝试接入被拒后，根据原因值在本地维护禁止区域列表
- **基站（NG-RAN）**：从核心网拿到移动性接入限制信息后，在切换流程选择目标网络时过滤受限RAT/跟踪区
- **核心网（AMF）**：作为移动性限制的**决策中心**，综合运营商策略（签约数据或AMF本地配置）执行接入限制

UNC 作为移动性限制的决策中心，根据不同类型的移动性接入限制信息实现不同的限制功能：
- 根据 **RAT限制** 和 **核心网限制** 实现**用户接入控制功能**（本特性）
- 根据 **禁止区域** 实现**区域漫游限制功能**（WSFD-105003）
- 根据 **服务区限制** 实现**服务区域限制功能**

#### 3.1.2 RAT限制与核心网限制详解

**RAT限制（RAT restriction）**：
- 表示限制的3GPP内的**无线接入方式**
- 指示受限的RAT（Radio Access Technologies）类型列表，用于表明是否限制用户接入 **NG-RAN** 或 **E-UTRAN**
- 对应于2/3/4G签约数据中的 **ARD（Access Restriction Data）**，在互操作流程中可以部分转换
- 获取方式：
  - AMF 可从用户签约数据获取 RAT 限制信息
  - AMF 也可在本地配置 RAT 限制信息
  - 注册流程结束后，核心网把 RAT 限制信息下发给基站，基站在发起切换流程时可基于该信息判断是否允许用户切换到目标 RAT

**核心网限制（Core Network type restriction）**：
- 表示限制的**核心网接入方式**
- 指示受限的核心网类型列表，用于表明是否限制用户接入 **5GC** 或者 **EPC**
- 获取方式：
  - AMF 可从用户签约数据获取核心网限制信息
  - AMF 也可在本地配置核心网限制信息

#### 3.1.3 ★本地配置与签约数据的优先级（核心机制）

当用户接入时，AMF 首先根据 UE 签约数据或 AMF 本地配置的 RAT 限制信息和核心网限制信息，以及 UE 当前使用的无线接入类型来决策是否允许用户接入。

**决策优先级原则**：

1. **本地配置匹配优先**：对于待接入的用户，同时存在签约信息和 AMF 本地配置的移动性接入限制信息，**如果用户匹配了 AMF 本地配置的接入限制记录，则按照本地配置执行接入控制**
2. **签约数据兜底**：只有当 AMF 本地未配置或者用户无任何匹配的接入限制记录时，AMF 才根据签约数据检查接入控制
3. **本地配置 Null 表示允许**：如果用户匹配的 AMF 上的配置记录给出的 RAT 限制和核心网限制都是空值（Null），那么不论该用户签约的移动性接入限制信息配置为何值，AMF 都对该用户**做允许接入**

**本地配置用户范围匹配优先级**（SUBRANGE参数）：
- IMSI前缀（包括用户群组）> 外网用户 > 本网用户 > 所有用户

**移动性接入限制信息具体决策场景（12种组合，文档原文表1）**：

| AMF本地配置（RAT限制） | AMF本地配置（核心网限制） | 签约数据（RAT限制） | 签约数据（核心网限制） | 决策结果 | 说明 |
|---|---|---|---|---|---|
| - | - | - | - | 允许接入 | 本地未配置且签约无限制 => 不限制 |
| - | - | NR | Any | 限制用户NR接入到5GC | 本地未匹配 => 按签约 |
| - | - | NR_REDCAP | Any | 限制用户NR_REDCAP接入到5GC | 本地未匹配 => 按签约 |
| - | - | EUTRA | Any | 限制用户EUTRA接入到5GC | 本地未匹配 => 按签约 |
| - | - | Any | 5GC | 限制用户接入到5GC | 本地未匹配 => 按签约 |
| - | - | Any | EPC | 限制用户从5G到4G的互操作流程 | 本地未匹配 => 按签约 |
| Null | Null | Any | Any | 允许接入 | 本地配置Null => 不论签约如何均允许 |
| NR | Any | Any | Any | 限制用户NR接入到5GC | 本地匹配 => 按本地配置 |
| NR_REDCAP | Any | Any | Any | 限制用户NR_REDCAP接入到5GC | 本地匹配 => 按本地配置 |
| EUTRA | Any | Any | Any | 限制用户EUTRA接入到5GC（NAS组网场景） | 本地匹配 => 按本地配置 |
| Any | 5GC | Any | Any | 限制用户接入到5GC | 本地匹配 => 按本地配置 |
| Any | EPC | Any | Any | 限制用户从5G到4G的互操作流程 | 本地匹配 => 按本地配置 |

**RedCap 说明**：NR_REDCAP 作为 NR 的一种子类型：
- 当 AMF 本地配置 RAT 限制为 **NR** 时，同时对 **NR_REDCAP** 用户也会产生限制
- 当 AMF 本地配置 RAT 限制为 **NR_REDCAP** 时，对**普通 NR 用户无限制**

> 来源：`用户接入控制功能（适用于AMF）/用户接入控制功能（适用于AMF）实现原理/本地配置与签约数据的优先级_48748809.md`

#### 3.1.4 用户接入控制判断流程

在**注册、移动性注册更新和切换**等业务流程中，当用户接入，AMF 根据 UE 签约或 AMF 本地配置的 **RAT限制信息** 和 **核心网限制信息**，以及 **UE 当前使用的无线接入类型** 来决策是否允许用户接入。

判断流程：
1. 获取移动性接入限制信息（本地配置优先 > 签约数据兜底）
2. 对比 UE 当前 RAT 类型与 RAT 限制列表
3. 对比 UE 当前核心网类型与核心网限制列表
4. 若命中任一限制 => 拒绝接入，向 UE 返回注册拒绝原因值（通过 SET NGMMPROCTRL 配置原因值）
5. 若均未命中限制 => 通过接入控制检查，AMF 进一步检查区域漫游限制（WSFD-105003）

> 来源：`用户接入控制功能（适用于AMF）/特性概述_48552635.md`（"原理概述"章节，图1 用户接入控制判断流程）

### 3.2 子特性A（AMF 侧）典型场景业务流程

UNC 支持不同注册和移动性场景的接入控制：

| 场景 | 子场景 | 用户状态 | 实现策略（接入限制信息未更新场景） |
|------|--------|---------|----------------------------------|
| 初始注册 | 用户初始注册 | 用户"首次"接入目标AMF | 在UDM获取签约数据之后进行接入控制检查；可能涉及AMF重定向（AMF re-allocation） |
| 移动性注册更新 | Intra AMF | 已接入用户，位置移动 | AMF无需再针对用户的接入控制检查，仅涉及区域漫游限制检查 |
| 移动性注册更新 | Inter AMF | 用户"首次"接入目标AMF | 由**目标AMF**执行用户接入控制检查，老侧AMF将签约数据等UE上下文发送给目标AMF |
| 切换 | Intra AMF切换 | 已接入用户，位置移动 | AMF无需再针对用户进行接入控制检查；源侧基站已基于AMF下发的禁止区域过滤 |
| 切换 | Inter AMF切换 | 用户"首次"接入目标AMF | 目标AMF在切换准备阶段进行接入控制检查，决策是否允许用户在目标AMF上接入 |
| 5GS与EPS互操作 | 基于N26接口的EPS到5GS注册更新 | 用户"首次"接入目标AMF | 目标AMF从MME收到ARD信息后转换为5G上下文的RAT限制，再执行检查 |
| 5GS与EPS互操作 | 基于N26接口的EPS到5GS切换 | 用户"首次"接入目标AMF | 目标AMF执行检查，与Inter AMF切换类似；AMF给NG-RAN下发的MRL只能携带RAT限制 |

**互操作ARD转换关键点**（★重点）：
- 基于N26接口的EPS到5GS流程中，目标AMF将4G上下文中 ARD 的 **ENA（WB-E-UTRAN Not Allowed）** 和 **NR in 5GS Not Allowed** 分别转换为5G上下文中的 **RAT Restriction** 的 **EUTRA** 和 **NR** 标志
- 基于N26接口的EPS到5GS切换场景，AMF给NG-RAN下发的 Mobility Restriction List 信元中**只能携带 RAT Restriction 信息**

### 3.4 子特性B（SGSN/MME 侧）实现原理

#### 3.4.1 用户接入控制判断流程（详细步骤）

1. 当用户发起**附着或路由区更新**时，UNC 首先比较用户的 IMSI 与配置的 **IMSI 前缀**是否匹配
   - 是 => 步骤2
   - 否 => 允许接入
2. UNC 判断是否配置了**接入控制属性列表**
   - 是 => 步骤3
   - 否 => 步骤4
3. UNC 根据用户属性（**签约ARD/APN/卡类型**）判断是否至少有一个用户属性与配置的属性相匹配
   - 是 => 步骤4（采取与配置相同的策略）
   - 否 => 步骤5（采取与配置相反的策略）
4. UNC 采取和配置策略相同的接入控制策略
5. UNC 采取和配置策略相反的接入控制策略
6. UNC 判断配置的控制策略
   - 拒绝（REJECT）=> 拒绝接入（携带原因值CAUSE）
   - 允许（ALLOW）=> 允许接入

#### 3.4.2 三种控制方式详解

**方式1：根据签约的ARD控制用户接入**

- **ARD 定义**：3GPP R6协议中在 HLR/HSS 签约数据中加入了 ARD 信息，指示是否拒绝用户接入 GERAN、UTRAN 或 E-UTRAN
- **信元传递**：ARD 信息在 HLR/HSS 和 SGSN/MME 之间通过 **Insert Subscriber Data Request** 消息或 **Update Location Answer** 消息中的 **AccessRestrictionData 信元**传送
- **bit 位定义**：AccessRestrictionData 信元包括 bit0~bit7 八个bit位：
  - bit0：是否允许用户接入 **UTRAN**
  - bit1：是否允许用户接入 **GERAN**
  - bit4：是否允许用户接入 **E-UTRAN**
  - bit6：是否允许 **NB-IoT** 用户接入网络
  - 取值：**1 = 拒绝接入，0 = 允许接入**
- **默认假设**：如果 UNC 支持 ARD 特性但未从 HLR/HSS 收到 RAT 消息，UNC 将假定用户签约时不需要限制 RAT
- **ARD 生效流程**：
  1. 当用户发起附着或路由区更新时，若 IMSI 号段匹配且 UE 通过 UTRAN 接入，UNC 根据 ARD 信息"禁止接入UMTS"拒绝用户接入
  2. 当 HLR 发起用户签约数据修改时，若 IMSI 号段匹配且 UE 通过 UTRAN 接入，若 ARD 信息变更为"禁止接入UMTS"，UNC 会直接将该用户分离
- **拒绝原因值配置**：通过 ADD GBARD / ADD IUARD / ADD S1ARD 命令的"原因值"参数设置

**方式2：根据签约的APN控制用户接入**

- **机制**：在 HLR/HSS 中为用户签约一个**特殊的 APNNI**，此 APNNI 用来控制用户是否允许接入 GERAN、UTRAN 或 E-UTRAN
- **示例签约**：
  - 正常签约用户：Internet, WAP
  - GERAN接入用户：Internet, WAP, **GERAN_USER**
  - UTRAN接入用户：Internet, WAP, **UTRAN_USER**
  - E-UTRAN接入用户：Internet, WAP, **E_UTRAN_USER**
  - （GERAN_USER/UTRAN_USER/E_UTRAN_USER 是标识接入特定无线网的用户的特殊 APNNI）
- **生效流程**：用户请求接入时，UNC 首先根据用户 IMSI 号段找到对应的 APNNI 列表，然后比对该列表中的 APNNI 和用户在 HLR 中签约的特殊 APNNI，比对一致则允许接入，否则拒绝
- **应用限制**：这种接入控制方式可能导致**电路域和分组域区分不同接入网用户的标准不同**

**方式3：根据卡类型控制用户接入（只适用于GERAN/UTRAN接入用户）**

- **机制**：UNC 通过分析用户签约数据中的鉴权信息是**三元组还是五元组**来确定是 SIM 卡还是 USIM 卡，从而区分 GERAN 用户和 UTRAN 用户，实现接入控制
- **两种控制方式**：
  - **拒绝SIM卡用户接入UTRAN网络**：UNC 向 HLR 请求鉴权参数，若含**三元组**（SIM卡），则拒绝接入 UTRAN
  - **拒绝USIM卡用户接入GERAN网络**：UNC 向 HLR 请求鉴权参数，若含**五元组**（USIM卡），则拒绝接入 GERAN
- **依赖鉴权功能**：本方式要求先启用附着和 Inter RAU 流程的鉴权功能（WSFD-010301），因为 UNC 需要向 HLR 请求鉴权参数以判断卡类型

> 来源：`用户接入控制功能（适用于SGSN、MME）/特性概述_68358176.md`（"原理概述"章节）、`用户接入控制功能（适用于SGSN、MME）/用户接入控制功能（适用于SGSN、MME）实现原理_68358178.md`

### 3.5 ★APN 接入权限机制（重点）

**重要结论（★）**：APN 级接入权限控制**仅存在于子特性B（SGSN/MME 侧）**，AMF 侧无 APN 级控制。

**APN 接入权限机制（SGSN/MME 侧的"根据签约的APN控制用户接入"）**：

1. **不是基于 APN 黑白名单的常规理解**：本机制**不是**直接对某个 APN 配置黑/白名单，而是通过在 HLR/HSS 中为用户签约一个**特殊的 APNNI**（如 GERAN_USER/UTRAN_USER/E_UTRAN_USER），UNC 在用户附着时比对签约 APNNI 列表，实现基于 APNNI 的接入控制。

2. **控制逻辑**：
   - 运营商在 HLR/HSS 中为不同接入类型的用户签约不同的特殊 APNNI
   - UNC 本地配置（ADD GBARD/IUARD/S1ARD）的 APNNI 参数指定允许/拒绝接入的特殊 APNNI
   - 用户附着时，UNC 比对用户的签约 APNNI 列表与本地配置的 APNNI，按 CTRLTYPE 执行允许/拒绝

3. **典型用例**：
   - 只允许 APNNI=wcdma 的用户接入 E-UTRAN：`ADD S1ARD: SUBRANGE=SPECIAL_IMSIPRE, IMSIPRE="12303001", APNNI="wcdma", ARD=NO, CTRLTYPE=ALLOW;`
   - 拒绝 APNNI=wcdma 的用户接入 E-UTRAN：`ADD S1ARD: SUBRANGE=SPECIAL_IMSIPRE, IMSIPRE="12303001", APNNI="wcdma", ARD=NO, CTRLTYPE=REJECT, CAUSE=EPS_SERVICE_NOT_ALLOWED;`

> 来源：`用户接入控制功能（适用于SGSN、MME）/特性概述_68358176.md`（"根据签约的APN控制用户接入功能原理"章节）、`激活用户接入控制功能特性（适用于SGSN、MME）_64082494.md`（任务示例脚本四、脚本五）

### 3.6 ★与 GWFD-010151（UDG 接入控制）的 C-U 关系（重点）

| 维度 | WSFD-106003（本特性，UNC 控制面） | GWFD-010151（UDG 接入控制，用户面） |
|------|----------------------------------|------------------------------------|
| **网络面** | 控制面（C 面） | 用户面（U 面） |
| **网元角色** | AMF / SGSN / MME | SGW-U / PGW-U / UPF（UDG） |
| **决策时机** | 用户接入（注册/附着/路由区更新/切换）阶段 | 报文转发/会话建立阶段 |
| **控制对象** | 用户级接入权限（是否允许接入网络 + 可用RAT/核心网类型） | 报文级/会话级控制（基于APN/过滤规则的报文处理） |
| **控制依据** | 签约数据 + 本地配置（移动性接入限制/ARD/APN/卡类型） | APN 配置 + 过滤规则 |
| **决策结果** | 允许接入 / 拒绝接入（拒绝原因值） | 报文允许通过 / 丢弃 / 重定向等 |
| **License** | AMF侧无 / SGSN&MME侧 LKV2ARD02 | （参见 GWFD-010151 文档） |
| **是否直接交互** | **否** | **否** |

**C-U 关系结论**：
- **本特性（WSFD-106003）与 GWFD-010151 是并列的两层接入控制**：本特性在 C 面决策"用户是否可以接入网络"，GWFD-010151 在 U 面执行"用户报文/会话是否允许通过"
- 两者**无直接接口交互**：C 面接入控制通过后，用户才进入 U 面；U 面接入控制是会话建立后的报文级控制
- **APN 维度的差异**：本特性（SGSN/MME侧）的 APN 控制是基于**特殊 APNNI 签约**的接入允许/拒绝（附着阶段）；GWFD-010151 的 APN 控制是基于 **APN 配置**的会话级报文处理（会话阶段）
- 本特性文档**未直接引用** GWFD-010151（两者无直接依赖关系），关系基于特性语义推断，Stage 3 横向验证

| **ARD仅适用GERAN/UTRAN/E-UTRAN** | 卡类型控制只适用于 GERAN/UTRAN 接入用户（不适用E-UTRAN） |
| **License强依赖** | 必须加载 License 控制项 82206571 LKV2ARD02 并打开 License 开关 |

### 6.6 常见问题与排查

#### 子特性A（AMF 侧）常见问题

| 问题现象 | 可能原因 | 排查方法 |
|---------|---------|---------|
| 用户接入被拒但本地配置未生效 | 存量用户，本地配置变更不立即生效 | 等待下一次移动性注册流程；或触发用户重新注册 |
