# 批次6：APN 业务域 Task 命令引用单条化修复（command_refs / task_summary 命令分隔符清理）

> **修复日期**：2026-06-23
> **修复目标**：清理 03-task-layer.md 中用 `/` 或 `+` 合并多命令的写法，使每条命令独立单条、可被前端索引
> **根因**：批次5 已将 `command_refs` 改回纯文本命令名，但仍残留 `/` 或 `+` 作为命令间分隔符（如 `SET LICENSESWITCH / SET LICENSESWITCH`、`ADD POOL/082(ADD POOL REDUNDFUNC)`、`SET IPALLOCRULE/APNIPALLOCRULE`）；前端 graph_parser 按 command_name 逐条匹配 04 的 MMLCommand，合并写法会导致部分命令无法索引
> **参照标杆**：`业务图谱体系/访问限制场景/three-layer-graph/03-task-layer.md`（command_refs 逗号分隔单条，task_summary 描述性嵌入命令亦单条）

## §3 同名命令去重规则

前端 graph_parser 的解析逻辑：`command_refs` 的纯文本 command_name → 匹配 04 的 MMLCommand.command_name → 建立 Task→Command 边。同一 command_name（如 `SET LICENSESWITCH`）在 04 中可能有 CMD-UDG-001 与 CMD-UNC-001 两条定义（U/C 侧），graph_parser 自动匹配两侧建两条边。

因此 `command_refs` 中同名命令重复书写（如 `SET LICENSESWITCH / SET LICENSESWITCH` 表达 U/C 侧各一条）是冗余的，应去重为单条 `SET LICENSESWITCH`，让前端自动建立 U/C 双边。

| 修复前 | 修复后 | 理由 |
|--------|--------|------|
| `SET LICENSESWITCH / SET LICENSESWITCH`（T-007 line 93） | `SET LICENSESWITCH` | 同名去重，前端自动匹配 CMD-UDG-001 + CMD-UNC-001 双边 |
| `SET REDUNDUSER / SET REDUNDUSER`（T-708 line 1149） | `SET REDUNDUSER` | 同名去重，前端自动建立双边 |
