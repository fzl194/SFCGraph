### 0.3 ★别名APN 双视角核对（核心差异点）

| 维度 | GGSN/PGW-C/SMF 套 | SGSN/MME 套 |
|------|------------------|-------------|
| 适用NF | GGSN、PGW-C、SMF | SGSN、MME |
| License控制项 | LKV2AAPN01（82200BNM 别名APN-USM） | LKV2ALIASAPN02（82207546 别名APN） |
| 核心命令 | ADD APNALIAS / RMV APNALIAS / LCK APNALIAS | ADD ALIASAPN / MOD ALIASAPN / RMV ALIASAPN / LST ALIASAPN |
| 参数主键 | SUBRANGE(ALL_USER/SPECIFIC_NS) + SST + SD + ALIASAPN + CONVERTAPN | SUBRANGE(ALL_USER/IMSI_PREFIX/IMSI_RANGE) + IMSIPRE/BEGIMSI/ENDIMSI + OLDAPN + NEWAPN |
| ★映射方向 | **别名APN（ALIASAPN）→ 转换APN（CONVERTAPN=真实APN）**：用户请求的别名APN 被转换为真实APN | **原始APN（OLDAPN）→ 转换APN（NEWAPN=别名APN）**：协商后的APN 映射成别名APN，用别名APN发起DNS解析 |
| 用户范围维度 | 切片维度（ALL_USER / SPECIFIC_NS + SST + SD） | IMSI维度（ALL_USER / IMSI_PREFIX / IMSI_RANGE） |
| 锁定/去活支持 | 有（LCK APNALIAS + DEA SMCTX + SET DEACTIVERATE） | 无（仅 ADD/MOD/RMV/LST，无锁定与批量去活） |
| 记录上限 | 1000（系统）/ 500（每转换APN） | 1024 |
| 触发时机 | 用户激活时由 GGSN/PGW-C/SMF 转换 | DNS解析前由 SGSN/MME 转换 |

> **★关键洞察**：两套网元的"别名"语义相反。GGSN/PGW-C/SMF 侧 ALIASAPN 是"用户请求的、需被转换掉的"APN；SGSN/MME 侧 NEWAPN 才是"转换后的别名APN"。这一差异决定了参数命名与映射方向的不对称，命令层必须双套并列建模，不能合并。

### 0.4 全局字段声明（status）

> **适用范围**：本文件 §1 所有 MMLCommand。
> **声明**：本文件 §1 所有 MMLCommand 的 `status` 字段值均为 `active`（Schema §11.3 MMLCommand.status 必备）。所有命令均在产品文档正式登记，无 `deprecated` 或 `planned` 状态。
> **依据**：SET APNREPORTATTR 中 INTELLIGENTSEL/LOCATIONREPORT/MAPTRANSDATA 参数手册标注「该参数已弃用」，但命令本身仍为 active；弃用仅限参数级，不影响命令状态。

## 4. ConfigObject 间关系边（§11.7）

### 4.1 refers_to 边（4条）

| 起点 | 关系 | 终点 | 说明 |
|------|------|------|------|
| APNALIAS (G套) | `refers_to` | APN | G套别名APN映射的 CONVERTAPN 必须是 ADD APN 已配置的真实APN（手册强约束） |
| APNREPORTATTR | `belongs_to` | APN | ADD APN 时自动为 APNREPORTATTR 生成一条记录（每APN一条） |
| APNACTNUM | `belongs_to` | APN | APN激活数目限制按 APNNI 关联（APNNI 可通配 `*`） |
| ALIASAPN (S套) | `refers_to` | APN（DNS解析目标） | S套 NEWAPN 为转换后别名APN，用于发起DNS解析（非 ADD APN 配置依赖） |

### 4.2 governs / activates 边（2条）

| 起点 | 关系 | 终点 | 说明 |
|------|------|------|------|
| APNALIASLOCK | `governs` | APNALIAS | 锁定状态（LOCKED=ENABLE）控制别名APN接入：新激活失败，配合 DEA SMCTX 去活已在线用户 |
| DEACTIVERATE | `governs` | DEA SMCTX | 去活速率控制批量去活的执行节奏（过高致 CPU 升高） |

### 4.3 depends_on 边（3条）

| 起点 | 关系 | 终点 | 说明 |

## 9. 使用实例脚本（保留手册原文）

### 9.1 WSFD-010503 多PDN/PDU（来源：激活支持多PDN功能_适用于4G）

```
//配置APN"Internet"下的数据。
ADD APNACTNUM: APNNI="Internet", PDNNUM=2, IPV4ADDRNUM=2, IPV6ADDRNUM=2, PDNCONNREJCAUSE=MULTI_PDN_FOR_APN_NOT_ALW_55;

//配置APN"WAP"下的数据。
ADD APNACTNUM: APNNI="WAP", PDNNUM=2, IPV4ADDRNUM=2, IPV6ADDRNUM=2, PDNCONNREJCAUSE=MULTI_PDN_FOR_APN_NOT_ALW_55;
```

### 9.2 WSFD-106203 G套 别名APN（来源：激活别名APN_适用于GGSN_PGW-C_SMF）

```
//1. 开启别名APN功能License。
SET LICENSESWITCH: LICITEM="LKV2AAPN01", SWITCH=ENABLE;

//2. 配置真实APN并配置APN的上报属性。
ADD APN: APN="apn5";
SET APNREPORTATTR: APN="apn5", PCF=SERVICE, CHF=SERVICE;

//3. 别名APN名称为"apn1"和"apn2"，配置真实APN为"apn5"。
ADD APNALIAS: SUBRANGE=ALL_USER, ALIASAPN="apn1", CONVERTAPN="apn5";
ADD APNALIAS: SUBRANGE=ALL_USER, ALIASAPN="apn2", CONVERTAPN="apn5";
```

### 9.3 WSFD-106203 S套 别名APN（来源：激活别名APN_适用于SGSN_MME，关键差异：方向相反 + IMSI维度）

```
//设置别名APN License开关为"ENABLE"。
SET LICENSESWITCH: LICITEM="LKV2ALIASAPN02", SWITCH=ENABLE;

//添加别名APN映射表的记录，将APN(example.com)映射为别名APN(huawei.com)。
ADD ALIASAPN: SUBRANGE=IMSI_PREFIX, IMSIPRE="12345", OLDAPN="example.com", NEWAPN="huawei.com";
```
