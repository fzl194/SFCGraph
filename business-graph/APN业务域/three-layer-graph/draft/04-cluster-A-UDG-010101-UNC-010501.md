## 5. MMLCommand 关键参数集（UNC 配置类命令全参数）

> **Schema 参考**：§11.4 CommandParameter。`required_mode` 取值 `required / optional / conditional_required`（Schema §11.4 必备字段）。本节 5 个配置类命令的参数表已从 UNC MML 手册原文抽取全量参数。

### 5.1 ADD PDPAPN（UNC 增加本地 APN NI 配置，6 参数）

| 参数 | 类型 | 取值范围 | `required_mode` | 默认值 | 说明 |
|------|------|---------|-----------------|--------|------|
| SUBRANGE | enum | ALL_USER / IMSI_PREFIX / IMSI_RANGE | `required` | 无 | 指定签约用户的范围（ALL_USER=所有用户；IMSI_PREFIX=指定IMSI前缀；IMSI_RANGE=指定IMSI范围） |
| IMSIPRE | 数字 | 1~15 位数字 | `conditional_required` | 无 | IMSI 前缀；SUBRANGE=IMSI_PREFIX 时必填 |
| BEGIMSI | 数字 | 1~15 位数字 | `conditional_required` | 无 | 起始 IMSI；SUBRANGE=IMSI_RANGE 时必填 |
| ENDIMSI | 数字 | 1~15 位数字 | `conditional_required` | 无 | 终止 IMSI；SUBRANGE=IMSI_RANGE 时必填 |
| PDPTYPE | enum | PT_IPV4 / PT_IPV6 / PT_PPP / PT_IPV4_IPV6 / PT_ALL | `required` | 无 | 指定 PDP 类型（PT_IPV4=IPV4协议；PT_IPV6=IPV6协议；PT_PPP=点对点通信协议；PT_IPV4_IPV6=IPV4和IPV6协议；PT_ALL=所有类型） |
| APNNI | string | 1~62 位字符串 | `required` | 无 | 指定 APN 网络标识地址（★必须可被 DNS 解析，否则激活失败 CR-010501-01） |

> **★手册关键约束**：本表最大记录数 1024（CR-010501-02）；当 SUBRANGE=IMSI_PREFIX/IMSI_RANGE 且 IMSI+PDP类型组合同时匹配多条记录时，使用 PDPTYPE 非 PT_ALL 的记录（CR-010501-03）。

### 5.2 MOD PDPAPN（UNC 修改本地 APN NI 配置，5 参数）

| 参数 | 类型 | 取值范围 | `required_mode` | 默认值 | 说明 |
|------|------|---------|-----------------|--------|------|
| SUBRANGE | enum | ALL_USER / IMSI_PREFIX / IMSI_RANGE | `required` | 无 | 指定签约用户的范围 |
| IMSIPRE | 数字 | 1~15 位数字 | `conditional_required` | 无 | IMSI 前缀；SUBRANGE=IMSI_PREFIX 时必填 |
| IMSI | 数字 | 1~15 位数字 | `conditional_required` | 无 | IMSI；SUBRANGE=IMSI_RANGE 时必填 |
| PDPTYPE | enum | PT_IPV4 / PT_IPV6 / PT_PPP / PT_IPV4_IPV6 / PT_ALL | `required` | 无 | 指定 PDP 类型 |
| APNNI | string | 1~62 位字符串 | `optional` | 无 | 指定 APN 网络标识地址（修改后的值；★配错导致 DNS 解析失败→激活失败 CR-010501-01） |

> **★与 ADD 的差异**：MOD 用 IMSI（单值）定位记录，ADD 用 BEGIMSI+ENDIMSI（范围）新建；APNNI 在 MOD 中可选（仅修改时填）。

### 5.3 SET GBSM（UNC 设置 Gb 模式 SM 协议参数，8 参数）

| 参数 | 类型 | 取值范围 | `required_mode` | 默认值 | 说明 |
|------|------|---------|-----------------|--------|------|
| T3385 | int | 1~86400 秒 | `optional` | 8 秒 | 网络侧激活定时器时长（ACTIVATE PDP CONTEXT REQUEST 等待响应） |
| N3385 | int | 0~5 次 | `optional` | 4 次 | 重发 ACTIVATE PDP CONTEXT REQUEST 消息的次数 |
| T3386 | int | 1~86400 秒 | `optional` | 8 秒 | 网络侧修改定时器时长（MODIFY PDP CONTEXT REQUEST 等待响应） |
| N3386 | int | 0~5 次 | `optional` | 4 次 | 重发 MODIFY PDP CONTEXT REQUEST 消息的次数 |
| T3395 | int | 1~86400 秒 | `optional` | 8 秒 | 网络侧去激活定时器时长（DEACTIVATE PDP CONTEXT REQUEST 等待响应） |
| N3395 | int | 0~5 次 | `optional` | 4 次 | 重发 DEACTIVATE PDP CONTEXT REQUEST 消息的次数 |
| PFT_USED | enum | NO / YES | `optional` | YES | 是否启用 PFT（Packet Flow Timer，数据包流定时器） |
| SM_BSS_PFT_T9 | int | 6~11160 秒 | `optional` | 40 秒 | BSS 保留 PFC 的时长定时器（T9 定时器） |

> **★手册关键约束**（CR-010501-04）：修改 GBSM 协议配置信息可能导致某些类型的会话无法激活成功；系统初次运行时执行系统初始设置值。
