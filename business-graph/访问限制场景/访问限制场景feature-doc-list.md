## 访问限制场景特性说明

### 核心机制：三种策略动作（均承载于PCC体系）

1. **PCC 阻塞**（DISCARD）→ PCCPOLICYGRP 包装 → 依赖 GWFD-020351/WSFD-109101 PCC基本功能
2. **头增强**（HEADEN 插入 MSISDN+防欺诈）→ 可直接被 RULE 引用 → 依赖 GWFD-110261/262/263
3. **URL 重定向**（REDIRECT 指定目标 URL）→ PCCACTIONPROP 引用 REDIRECT → PCCPOLICYGRP 包装 → 依赖 GWFD-110284 + GWFD-110283(DNS纠错变体)

### 核心特性分组

| 分组 | 特性 | 角色 |
|------|------|------|
| **A. PCC动作载体**（根能力，三场景共享） | GWFD-020351 / WSFD-109101 | DISCARD/REDIRECT/HEADEN 执行与下发框架 |
| **B. ADC应用检测**（触发动作） | GWFD-020357 / WSFD-109102 | L7 DPI检测，APP_STA/STO 事件触发访问限制 |
| **C. 头增强族** | GWFD-110261/262/263 | 插入 MSISDN/防欺诈头 |
| **D. 重定向族** | GWFD-110284 / GWFD-110283 | URL重定向 + DNS重写两种变体 |
| **E. URL过滤与Portal** | GWFD-110471 / 110281 / 110282 | URL过滤 + Portal/Web Proxy |
| **F. 防欺诈与接入** | GWFD-110401 / WSFD-211001 | 头防欺诈 + 初始接入位置策略 |
