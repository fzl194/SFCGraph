## 七、建议优先级

构建业务图谱时，建议按以下顺序提取知识：

1. **P0（立即）**：SA专题(12) + 计费专题(2) + PCC策略(50) = 64篇
   - 这些是SA核心原理，直接支撑业务图谱的对象和关系定义
   - 计费场景已有知识库和06文件，需验证其他场景的Schema适用性

2. **P1（次优先）**：本地分流(8) + ULCL分流专题(60) + CUPS&MEC(2) = 70篇
   - 本地分流场景的业务图谱实例化

3. **P2（按需）**：4G&5G互操作(34) + 接入控制(19) + FUP(15) = 68篇
   - 互操作场景含QoS映射，接入控制和FUP是独立子场景

4. **P3（补充）**：计费专题(200) + PCC之SM策略(79) + QoS(2) + 协议信令(168) = 449篇
   - 深度配置和信令知识，用于填充实例化细节和验证
   - 文件量大，建议按具体特性按需提取
   - Gx/Gy接口信令特别有价值：理解PCF-UNC和OCS-UNC的参数交互
