# GWFD-110313 基于智能Shaping的组级带宽控制 知识文档

## 概述

### 特性定义（组级Shaping，整流而非丢弃）

基于智能Shaping的组级带宽控制是指 UDG 通过基于 Shaping 的分级带宽管控，使整形后的用户报文出口速率稳定，自动模式下可以基于每个优先级业务的分布自动调整业务的出口带宽，保障高低优先级业务的调度公平。

核心设计理念：

- **整形（Shaping）而非直接丢弃**：报文入队后不丢包，丢包只发生在入队判断前（即缓存队列满时），避免直接丢包对用户体验造成影响。
- **削峰填谷**：通过缓存机制实现报文的缓存与延迟转发，使出口流量速率平滑稳定，整形令牌的缓存对周期统计的速率指标影响小。
- **智能调度**：支持基于用户的公平调度能力，自动模式下可基于业务分布动态调整各优先级保障带宽，保障高低优先级业务的调度公平。
- **性能优化**：当入流量速率小于整形速率时，能够直接做出队判断，该流量不需通过队列缓存，提升产品性能。

> 基于PCC标准架构的流量经营类业务功能，主要基于网络资源、客户的业务感知针对性展开网络运营，是运营商开展MBB商业运营的基础，实际部署时，需要在遵循当地法律允许的目的和范围内提供服务，以保证终端用户的通信自由和业务知情权。

## 核心概念

### 关键术语

| 术语 | 说明 |
|------|------|
| **Shaping（整形）** | 流量整形技术，通过缓存超额报文并按目标速率平稳转发，实现削峰填谷效果，使出口流量速率稳定。与直接丢弃超额报文的Policing（限速）不同，Shaping优先缓存而非丢弃 |
| **组级（Group Level）** | 带宽控制的粒度层级，针对用户组（而非单个用户或单个业务流）进行带宽控制。本特性仅在组级带宽控制规则中生效 |
| **智能调度** | 自动模式下基于每个优先级业务的分布自动调整业务的出口带宽，根据丢包率差值、最大丢包率等参数动态计算各ServiceLevel的保障带宽，确保高低优先级业务的丢包率符合预期 |
| **令牌桶（Token Bucket）** | 流量整形的底层机制。周期性触发添加令牌，根据保障带宽速率对所有优先级的令牌桶添加令牌；报文出队时从对应优先级的令牌桶中取令牌，取到令牌则出队转发，取不到则继续缓存 |
| **队列调度（Queue Scheduling）** | 按优先级从高到低依次对队列中的报文进行出队调度。对某个优先级队列中的报文依次处理，从对应优先级的令牌桶中取令牌，直到取不到令牌或队列为空后再处理下一个优先级队列 |
| **用户公平调度** | 同一优先级内的用户间公平地共享带宽；当带宽有剩余时可以被其他用户借用，同一优先级内用户间借用的优先级高于跨优先级借用 |
| **ServiceLevel（业务等级）** | 使用同一控制器的不同业务流间的优先级，值越小优先级越高。配置优先级后，优先保证高优先级的带宽 |
| **公平丢包率策略** | 自动模式下通过WorkMode、MaxPktLostRate、PktLostRateDtl、AssureMode等参数自动计算和调整每个周期的不同ServiceLevel的保障带宽，确保不同优先级的丢包率符合预期 |

### Shaping vs Policing vs 普通BWM差异

| 维度 | Shaping（整形） | Policing（限速） | 普通BWM（带宽管理） |
|------|----------------|------------------|---------------------|
| **超额报文处理** | 缓存到队列，延迟转发（削峰填谷） | 直接丢弃超额报文 | 通常基于CAR限速，超额丢弃 |
| **出口流量特征** | 速率平稳，出口速率稳定 | 突发后立即丢弃，速率波动大 | 取决于具体实现 |
| **丢包时机** | 仅在入队前（缓存队列满时）丢包，入队后不丢包 | 超过承诺速率即丢包 | 超过限速即丢包 |
| **时延影响** | 存在缓存时延（可配置MAXINCRDELAY） | 无额外时延 | 无额外时延 |
| **用户体验** | 更平滑，避免突发丢包导致速率波动 | 突发场景体验较差 | 常规体验 |
| **性能开销** | 存在一定性能消耗（队列管理、令牌调度） | 开销较小 | 开销较小 |
| **本特性定位** | **本特性采用此方式** | 不适用 | 本特性是BWM的增强形式 |

### 组级 vs 用户级 vs 业务级带宽控制

| 层级 | 控制粒度 | 说明 | 本特性关系 |
|------|----------|------|------------|
| **组级** | 用户组 | 针对一个用户组内所有用户的总带宽进行控制，组内用户共享带宽池 | **本特性作用层级** |
| **用户级** | 单个用户 | 针对单个用户的带宽进行控制，用户间独立 | 本特性的用户公平调度在组内对用户级进行调度 |
| **业务级** | 单个业务流 | 针对单个业务流（如某个APP的流量）进行控制 | 本特性通过ServiceLevel对组内不同业务流进行优先级区分 |

本特性是**组级带宽控制**，在组级总带宽约束下：
- 通过 ServiceLevel 区分组内不同优先级的业务流（业务级维度）。
- 通过 UserFairEn 实现组内用户间的公平调度（用户级维度）。
- 通过 Shaping 实现整流而非直接丢弃（控制方式维度）。

## 原理与流程

### 实现原理（组内多用户共享带宽、智能调度）

本特性包含三个层次的实现原理，逐步增强：

#### 原理一：基于Shaping的分级带宽控制

这是最基础的整形管控机制，核心流程如下：

1. **判断队列存在性**：首先判断对应优先级的队列是否存在。
2. **取令牌决策**：
   - 如果队列不存在，则直接从对应的令牌桶中取令牌。如果能够取到令牌则直接出队转发；如果令牌不足则创建对应优先级的队列，并缓存该报文。
   - 如果队列存在，则缓存至优先级对应的队列中。
3. **周期性添加令牌**：周期性触发添加令牌，根据保障带宽速率对所有优先级的令牌桶添加令牌。
4. **按优先级出队**：完成添加令牌后，对所有优先级队列从高到低依次出队。对某个优先级队列中的报文依次处理，从对应优先级的令牌桶中取令牌，直到取不到令牌或队列为空后再处理下一个优先级队列中的报文。

**关键设计**：
- 当入流量速率小于整形速率时，直接做出队判断，不需要队列缓存，提升性能。
- 当出口速率不足、缓存队列满时，优先丢弃低优先级报文。
- 智能调控算法保障高低优先级业务体验影响可控，低优先级业务影响高于高优先级业务。

#### 原理二：支持用户公平调度的基于Shaping的分级带宽管控

在基础分级带宽算法之上，增加用户间公平调度机制：

**解决的问题**：
- 新发起业务的用户与已经进入高速转发阶段的用户丢包率一致，导致新发起业务的用户长时间无法达到预期速率，影响用户体验。
- 大量低优先级用户占用带宽的不公平现象。

**实现机制**（通过 BwmController 命令的 UserFairEn 参数控制）：

| 公平调度规则 | 说明 |
|-------------|------|
| 同优先级用户间共享 | 同一优先级内的用户间公平地共享带宽 |
| 带宽借用 | 当带宽有剩余时可以被其他用户借用 |
| 借用优先级 | 同一优先级内用户间借用的优先级高于跨优先级借用 |
| 借用原则 | 同一优先级内用户间借用遵循"先到先得"原则 |
| 新用户保障 | 支持为新加入的用户预留资源，启动阶段快速获取带宽 |

#### 原理三：基于优先级业务分布和公平丢包率策略执行智能分级带宽管控

在用户公平调度之上，进一步增加自动调整能力：

**解决的问题**：
- 传统方式需要手动为每一个优先级的业务配置固定保障带宽。
- 不同优先级业务的占比每天存在较大波动，固定配置可能导致低优先级业务过高保障，高优先级业务整体丢包率和QoE影响大于低优先级业务的不公平问题。

**实现机制**（通过 BwmController 命令的以下参数自动计算和调整）：

| 参数 | 作用 |
|------|------|
| WorkMode（工作模式） | MANUAL（手动模式）或 AUTO（自动模式），自动模式下自动调整各ServiceLevel保障带宽 |
| MaxPktLostRate（最大丢包率） | 带宽控制器允许的最大丢包比率 |
| PktLostRateDtl（丢包率差值） | 两个有数据的ServiceLevel之间的丢包率差值 |
| AssureMode（保障模式） | 决策冲突时的优先保障模式：EXPFIRST（体验优先）或 RATEFIRST（转发速率优先） |

**自动调整逻辑**：
1. 自动模式下，系统周期性自动计算和调整每个周期的不同ServiceLevel的保障带宽。
2. 确保不同优先级的丢包率符合预期丢包率（由MaxPktLostRate和PktLostRateDtl参数控制）。
3. 在决策冲突时（最大丢包率和带宽控制器出口速率仅能满足一个条件时），根据AssureMode选择体验优先或转发速率优先。
4. 确保高优先级业务的QoE影响小于低优先级业务。

| 带宽管理控制器名称 | BWMCNAME | bc1 | 带宽管理控制器的名字 |
| 带宽管理控制类型 | CTRLTYPE | SHAPING | 固定为SHAPING，表示整形类型 |
| 速率（千比特/秒） | RATE | 400 | SHAPING速率，报文将被控制按该速率平稳转发出去 |
| 队列深度（包） | QUEDEPTH | 25 | SHAPING缓存队列深度，以包为单位 |
| 业务等级规格 | SRVLEVELSPEC | 10 | 该带宽管理控制器最大可支持的业务等级规格 |
| 用户公平使能 | USERFAIREN | ENABLE | 组级带宽管理时是否支持用户级公平调度功能使能 |
| 工作模式 | WORKMODE | AUTO / MANUAL | AUTO为自动模式，MANUAL为手动模式 |
| 最大丢包率 | MAXPKTLOSTRATE | 10000 | 此带宽控制器允许的最大丢包比率（自动模式参数） |
| 丢包率差值 | PKTLOSTRATEDTL | 50 | 两个有数据的ServiceLevel之间的丢包率差值（自动模式参数） |
| 保障模式 | ASSUREMODE | EXPFIRST | 自动计算丢包率出现冲突时的优先保障模式 |

**WORKMODE（工作模式）详细说明：**

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| MANUAL（手动模式） | 手动为每个优先级业务配置固定保障带宽 | 业务占比稳定，不需要动态调整 |
| AUTO（自动模式） | 基于每个优先级业务的分布自动调整保障带宽 | 业务占比波动大，需要动态公平调度 |

**ASSUREMODE（保障模式）详细说明：**

| 模式 | 说明 |
|------|------|
| EXPFIRST（体验优先） | 当最大丢包率和带宽控制器出口速率仅能满足一个条件时，优先保障用户体验（降低丢包率） |
| RATEFIRST（转发速率优先） | 当最大丢包率和带宽控制器出口速率仅能满足一个条件时，优先保障转发速率 |

#### ADD BCSRVLEVELPLY 核心参数

| 参数 | 英文名 | 取值样例 | 说明 |
|------|--------|----------|------|
| 带宽管理控制器名称 | BWMCNAME | bc1 | 关联的带宽管理控制器名称 |
| 业务等级 | SERVICELEVEL | 2 | 不同业务流间的优先级，值越小优先级越高 |
| 整形速率 | SHAPRATE | 100 | 此业务等级的最高峰值带宽比例（百分比），实际承诺信息速率 = SHAPRATE * RATE(BWMCONTROLLER) |
| 最大丢包率 | MAXPKTLOSTRATE | 10000 | 此带宽控制器允许的最大丢包比率 |
| 最大增加时延 | MAXINCRDELAY | 100 | 此业务等级因Shaping队列缓存的最大增加时延 |

**SHAPRATE计算规则**：
- SHAPRATE 是百分比，实际承诺信息速率（CIR） = SHAPRATE * RATE(BWMCONTROLLER的RATE参数)。
- 例如：BWMCONTROLLER RATE=400 kbit/s，SHAPRATE=100，则该ServiceLevel的实际CIR = 100% * 400 = 400 kbit/s。

### 约束条件

| 约束类别 | 说明 |
|----------|------|
| 适用层级 | 仅在组级带宽控制规则中生效 |
| 特性依赖 | 必须先开启 GWFD-110311（基于业务感知的带宽控制）和 GWFD-020354（基于业务的Shaping） |
| License依赖 | 必须先开启依赖特性的License，再开启本特性License |
| 特征库 | 必须已加载特征库和解析库 |
| 性能影响 | 开启后Shaping流控会存在一定的性能消耗，需提前进行性能影响评估 |
| PCRF/PCF | 只支持下发预定义规则的shaping |
| 计费与话单 | 本特性不涉及计费与话单 |
| 特性规格 | 本特性无特殊规格 |
| 缓存丢包 | 丢包只发生在入队判断前（队列满时），入队后不丢包 |
| 优先级丢弃 | 缓存队列规格满时，优先丢弃低优先级的报文 |

## 配置案例

### 典型场景：某用户组共享带宽池（含完整MML）

**场景描述**：
某运营商需要为特定用户组配置组级带宽控制，要求：
- 用户组总出口带宽限制为 400 kbit/s。
- 支持用户间公平调度，避免新用户被饿死。
- 采用自动模式，根据业务分布动态调整各优先级保障带宽。
- 最大允许丢包率为 1%（10000万分之一）。
- 优先级间丢包率差值为 0.5%（50万分之一）。
- 体验优先模式。
- 配置3个业务等级，ServiceLevel=2的整形速率比例为100%。

**配置脚本**：

```mml
// ==========================================
// Step 1: 打开本特性的License配置开关
// ==========================================
SET LICENSESWITCH:LICITEM="LKV3G5FSHP01",SWITCH=ENABLE;

// ==========================================
// Step 2: 配置带宽管理控制器（自动模式）
// ==========================================
// 参数说明：
//   BWMCNAME     = bc1         - 控制器名称
//   CTRLTYPE     = SHAPING     - 控制类型为整形
//   RATE         = 400         - 整形速率 400 kbit/s
//   QUEDEPTH     = 40          - 队列深度 40 包
//   SRVLEVELSPEC = 3           - 最大支持3个业务等级
//   USERFAIREN   = ENABLE      - 开启用户公平调度
//   WORKMODE     = AUTO        - 自动模式（智能调度）
//   MAXPKTLOSTRATE = 5000      - 最大丢包率 0.5%
//   PKTLOSTRATEDTL = 50        - 丢包率差值
//   ASSUREMODE   = EXPFIRST    - 体验优先
// ==========================================
ADD BWMCONTROLLER:BWMCNAME="bc1",CTRLTYPE=SHAPING,RATE=400,QUEDEPTH=40,SRVLEVELSPEC=3,USERFAIREN=ENABLE,WORKMODE=AUTO,MAXPKTLOSTRATE=5000,PKTLOSTRATEDTL=50,ASSUREMODE=EXPFIRST;

// ==========================================
// Step 3: 配置带宽管理控制器业务等级策略
// ==========================================
// ServiceLevel=2, 整形速率比例100%, 最大丢包率10000, 最大增加时延100ms
// 实际CIR = 100 * 400 = 400 kbit/s
// ==========================================
ADD BCSRVLEVELPLY:BWMCNAME="bc1",SERVICELEVEL=2,SHAPRATE=100,MAXPKTLOSTRATE=10000,MAXINCRDELAY=100;
```

**手动模式替代配置（二选一）**：

如果业务占比稳定，不需要自动调整，可使用手动模式：

```mml
// 手动模式：固定保障带宽配置
ADD BWMCONTROLLER:BWMCNAME="bc1",CTRLTYPE=SHAPING,RATE=400,QUEDEPTH=25,SRVLEVELSPEC=10,USERFAIREN=ENABLE,WORKMODE=MANUAL;

// 配置业务等级策略
ADD BCSRVLEVELPLY:BWMCNAME="bc1",SERVICELEVEL=2,SHAPRATE=100,MAXPKTLOSTRATE=10000,MAXINCRDELAY=100;
```

**两种模式对比**：

| 维度 | 手动模式 (MANUAL) | 自动模式 (AUTO) |
|------|-------------------|-----------------|
| 保障带宽 | 固定配置，不随业务分布变化 | 周期性自动调整 |
| 适用场景 | 业务占比稳定 | 业务占比波动大 |
| 配置复杂度 | 需手动为每个优先级配置 | 系统自动计算 |
| 丢包率控制 | 取决于手动配置是否合理 | 确保丢包率符合预期（MaxPktLostRate + PktLostRateDtl） |
| 冲突处理 | 无 | AssureMode决定（体验优先/速率优先） |
| QUEDEPTH建议 | 25（较小） | 40（较大，因自动调整需要更多缓冲） |
| SRVLEVELSPEC | 10（支持更多等级） | 3（精简等级） |

## 验证与调测

### 调测步骤

**前提条件**：已完成激活基于智能Shaping的组级带宽控制的全部配置。

#### 步骤1：验证License开关

执行 `LST LICENSESWITCH` 查询本特性对应的License配置开关是否打开：

```mml
LST LICENSESWITCH:LICITEM="LKV3G5FSHP01";
```

- 如果 SWITCH 为 ENABLE，继续下一步。
- 如果 SWITCH 为 DISABLE，执行 `SET LICENSESWITCH` 打开开关。

#### 步骤2：验证业务等级策略

执行 `LST BCSRVLEVELPLY` 查询带宽管理控制器业务等级策略是否生效：

```mml
LST BCSRVLEVELPLY:;

// 预期输出示例：

BWM Controller Service Level Policy Information

      Service Level  =  2
           CIR Rate  =  50
           ShapRate  =  0
           PIR Rate  =  1000
     MaxPktLostRate  =  10000
 Max Increase Delay  =  NULL
(Number of results = 1)

```

#### 步骤3：验证带宽管理控制器配置

| 工作模式 | WorkMode | AUTO / MANUAL | 确认工作模式正确 |
| 最大丢包率 | MaxPktLostRate | 配置值 | 确认丢包率参数生效 |
