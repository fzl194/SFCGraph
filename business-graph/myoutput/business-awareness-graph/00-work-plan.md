# 业务感知业务图谱工作计划

## 目标

本目录用于沉淀一版：

```text
面向业务图谱展示与讨论的业务图谱展示稿
```

强调：

- 业务图谱长什么样
- 业务图谱使用哪些对象 schema
- 业务感知这个领域如何作为例子把图谱讲清楚

不强调：

- 全量命令细节
- 参数级穷举
- 一次性读完全量原始语料

## 当前范围

第一批必读原始语料固定为 8 块：

1. `UDG业务感知专题`
2. `GWFD-110101 SA-Basic`
3. `GWFD-020351 PCC基本功能`
4. `GWFD-110311 基于业务感知的带宽控制`
5. `WSFD-109002 内容计费基本功能`
6. `WSFD-011206 支持融合计费`
7. `WSFD-109101 PCC基本功能`
8. `5G Core 计费解决方案`

## 迭代策略

采用：

```text
先规划
-> 分轮阅读
-> 边读边写
-> 每轮回看并调整骨架
```

每轮只处理一个有限目标：

### Round 1

目标：

- 从 UDG 业务感知专题中提炼：
  - 业务感知定义
  - 业务感知核心过程
  - 第一批一级场景候选
  - 业务图谱对象如何挂到业务感知域

输出：

- `01-source-scope.md`
- `02-schema-and-view-model.md`
- `notes/r1-udg-topic.md`
- `03-global-graph-skeleton.md` 第一版

### Round 2

目标：

- 从 UDG 特性侧补足：
  - SA-Basic 的识别与执行底座
  - PCC 基本功能的规则与动作骨架
  - 基于业务感知的带宽控制在用户面侧的落点

输出：

- `notes/r2-udg-features.md`
- `03-global-graph-skeleton.md` 第二版

### Round 3

目标：

- 从 UNC 特性与方案侧补足：
  - 控制面内容计费
  - 融合计费
  - PCC 基本功能
  - 计费解决方案中的控制面对象链和运行链

输出：

- `notes/r3-unc-features-and-solution.md`
- `03-global-graph-skeleton.md` 第三版
- `04-report-view.md` 第一版

## 工作纪律

1. 所有结论优先来自原始语料，不直接复用历史抽象作为事实。
2. 历史讨论只作为：
   - schema 参考
   - 命名对齐参考
   - 缺口检查参考
3. 不一次性通读全部文档。
4. 每轮都要显式记录：
   - 读了哪些文档
   - 学到了哪些稳定结论
   - 哪些还只是候选判断
5. 第一版图中：
   - `重定向`
   - `增强与安全`
   暂不单列一级主块，但必须在骨架中保留落点。
