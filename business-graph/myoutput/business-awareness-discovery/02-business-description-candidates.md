# 业务感知相关业务描述文件盘点

## 1. 说明

本文件只关注 `业务描述层` 的原始材料，不下钻命令和软件参数。

这里要找的不是：

- 如何配置
- 使用哪些命令
- 参数怎么取值

这里要找的是：

- 业务感知是什么
- 业务感知解决哪些业务问题
- 业务感知有哪些典型场景
- 业务感知和计费、PCC、带宽控制、重定向、本地分流之间是什么关系

## 2. 当前判断的主要材料簇

从整个产品文档往下看，业务感知相关的 `业务描述层` 材料主要分成 8 簇：

1. `业务感知专题`
2. `5G基础知识 / 一望5G / 业务解决方案解读：业务感知（SA）`
3. `5G基础知识 / 一望5G / 5G PCC策略系列`
4. `业务专题 / 计费解决方案`
5. `业务专题 / 本地分流 / MEC / ULCL相关解决方案`
6. `5G基础知识 / 一望5G / 计费解决方案`
7. `快速入门 / 网元介绍 / 产品描述`
8. `调测、运维、高危操作、License描述中对业务现象和能力边界的描述`

其中前 4 簇应优先纳入主清单，第 5 到 7 簇是重要补充或旁支，第 8 簇用于补充业务后果、验证、风险和能力边界描述。

## 5. 外围补充材料

### 5.1 运维 / 高危操作 / 调测

这类材料不负责定义业务感知，但能补：

1. 哪些业务动作有明显风险
2. 哪些现象说明业务可能失效
3. 哪些业务配置修改会影响内容计费、带宽控制、规则识别

当前可保留的典型主题：

- `添加、删除、修改内容计费数据`
- `SA特征库规则加载条件`
- `PCC策略组/用户模板/带宽管理用户组删除后的业务影响`

### 5.2 用户Portal / 重定向类业务说明

虽然用户说当前不进命令层，但以下材料属于业务描述层，值得保留：

- `GWFD-110281 用户Portal特性概述`
- `GWFD-110281 用户Portal实现原理`

纳入原因：

- 它们直接描述了“重定向业务到底长什么样、业务目标是什么、用户体验会如何变化”。
- 这对业务感知中的 `重定向引导` 支线很重要。

### 5.3 快速入门 / 产品描述 / 典型组网

这类材料不直接给出场景方案，但会补：

1. UDG/UPF 与 UNC/SMF 分别在业务感知链路上扮演什么角色
2. 智能PCC、计费管理、MEC 这些能力在产品能力地图中的位置
3. 全产品视角下，差异化服务、边缘分流、体验保障这些业务诉求如何被产品描述

当前应保留的典型主题：

- `UNC产品描述/业务功能/SMF+PGW-C+SGW-C+GGSN业务功能`
- `UDG产品描述/业务功能`
- `UNC/UDG产品描述/华为5G Core解决方案`
- `UDG产品描述/典型组网`

当前判断：

- 这类材料属于 `背景与角色说明材料`
- 它们不替代专题页，但对后续梳理“业务感知业务知识版图”很有帮助

### 5.4 License 描述中的能力摘要

这类材料容易被忽略，但它会以高度压缩的方式给出：

1. 某个能力的功能描述
2. 典型应用场景
3. 对应特性和相关控制项

当前已确认对业务感知有价值的内容：

- `SA-Basic`
- `用户Portal`
- 若干 `SA-*` 子类能力

当前判断：

- 这类材料不进入主链定义
- 但属于 `能力摘要型补充材料`，适合用来检查“某个能力是否真的属于业务感知族”
