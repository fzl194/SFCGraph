# WSFD-109002 内容计费基本功能 知识文档

## 概述

### 定义

内容计费（Content-based Charging），即基于业务粒度的计费，是一种通过包过滤和分析技术识别出用户上下行数据的业务类型，针对不同的业务实施不同费率的计费方法。内容计费可通过在线计费系统（OCS）、离线计费系统（CG）或融合计费系统（CHF）实现。

内容计费对应3GPP协议中定义的**基于流的计费（Flow based Charging）**。

### 适用NF

GGSN、SGW-C、PGW-C、SMF

> 与UDG侧区别：UDG侧适用NF为SGW-U、PGW-U、UPF。UNC侧重**控制面**（计费规则管理与上报），UDG侧重**用户面**（业务识别与流量统计）。

### 涉及网元与角色

|----------|----------|----------|

## 核心概念

### UNC侧内容计费功能定位

UNC（SMF/PGW-C）在内容计费中的角色是**控制面管理**：

1. **从PCRF/PCF或本地配置中获取计费规则**：如果计费规则中定义了内容计费，则SMF/PGW-C将计费规则的业务信息通过PDI（Packet Detection Information）下发给UPF/PGW-U。
2. **接收用户面的计费统计结果**：UPF/PGW-U根据SA功能对业务流进行解析和计费规则匹配，匹配成功的业务标记对应URR ID，触发上报后上报计费信息。
3. **上报计费系统**：SMF/PGW-C将计费信息上报到OCS/CG/CHF。

### 核心配置对象（计费三件套）

SMF/PGW-C侧的内容计费配置围绕三个核心对象展开：

| 配置对象 | MML命令 | 作用 |
|----------|---------|------|
| URR（使用量上报规则） | ADD URR | 定义一条费率规则，包含费率标识（RG/SID）、计费模式、统计类型 |
| URRGROUP（使用量上报规则组） | ADD URRGROUP | 将在线URR和离线URR组合在一起，支持上下行分别配置 |
| PCCPOLICYGRP（PCC策略组） | ADD PCCPOLICYGRP | 将URRGROUP绑定为PCC策略组，供RULE引用 |

三件套关系：`URR -> URRGROUP -> PCCPOLICYGRP`，每条业务独立配置一组三件套。

### 费率标识

费率标识用于UNC与CHF交互时标识计费粒度：

| 标识类型 | 说明 | 使用场景 |
|----------|------|----------|
| RG（Rating Group） | 费率组，必须配置 | 基本的费率标识 |
| (RG, SID) | 费率组 + 业务ID，可选配置 | 更细粒度的配额管理和更丰富的话单信息 |

费率标识组成类型参数：
- **ONLYRG**：仅使用RG作为费率标识
- **RGSID**：同时使用RG和SID作为费率标识

### 匹配规则配置对象

| 配置对象 | MML命令 | 作用 |
|----------|---------|------|
| RULE（规则） | ADD RULE | 将PCC策略组绑定，定义匹配优先级 |
| USERPROFILE（用户模板） | ADD USERPROFILE | 规则组容器，承载多条RULE |
| RULEBINDING（规则绑定） | ADD RULEBINDING | 将RULE绑定到USERPROFILE |
| SET URRGRPBINDING | SET URRGRPBINDING | 设置USERPROFILE的缺省URR组 |

> 与UDG侧区别：UNC侧**不配置**FILTER、L7FILTER、FLOWFILTER、FLTBINDFLOWF、PROTBINDFLOWF等过滤条件。这些过滤条件由PCF/PCF动态下发或在UPF侧本地配置。UNC侧只配置"费率规则"和"规则绑定"。

## 原理与流程

### 内容计费功能逻辑

```
PCRF/PCF --> SMF/PGW-C（获取计费规则）
                |
                | PDI下发业务信息
                v
           UPF/PGW-U（SA解析 + 计费规则匹配）
                |
                | 匹配成功标记URR ID，触发上报
                v
           SMF/PGW-C（接收计费信息）
                |
                | 上报计费信息
                v
           OCS / CG / CHF
```

详细步骤：
1. SMF/PGW-C从PCRF/PCF或本地配置中获取计费规则。
2. SMF/PGW-C将计费规则的业务信息通过PDI下发给UPF/PGW-U。
3. UPF/PGW-U根据SA功能对业务流进行解析，进行计费规则匹配。
4. 匹配成功的业务标记对应的URR ID。
5. 触发上报条件后，UPF/PGW-U上报计费信息给SMF/PGW-C。
6. SMF/PGW-C将计费信息上报到计费系统（OCS/CG/CHF）。

### 计费信息上报

SMF/PGW-C按照内容计费的RG向计费系统上报计费信息。

内容计费支持不同的资源统计方式：
- **流量计费**：参见WSFD-109004 基于业务流量的计费
- **时长计费**：参见WSFD-109003 基于业务时长的计费

### URR配置规则

#### 关键参数

| 参数 | 说明 | 约束 |
|------|------|------|
| URRNAME | 使用量上报规则名称 | 本端规划 |
| URRID | URR标识 | 全网规划，SMF和UPF上必须一致 |
| USAGERPTMODE | 使用量上报模式 | ONLINE（在线）/ OFFLINE（离线） |
| ONLCOMPOUNDTYPE | 在线计费标识组成类型 | ONLYRG / RGSID |
| ONLINERG | 在线计费组 | 本端规划 |
| ONLINESID | 在线业务标识（可选） | 本端规划 |
| ONLMETERINGTYPE | 在线计费统计类型 | VOLUME / DURATION / FREE / EVENT_VOLUME_TIME等 |
| OFFCOMPOUNDTYPE | 离线计费标识组成类型 | ONLYRG / RGSID |
| RG | 离线计费组 | 本端规划 |
| OFFSID | 离线业务标识（可选） | 本端规划 |
| OFFMETERINGTYPE | 离线计费统计类型 | VOLUME / DURATION / FREE等 |

#### 在线URR示例

```
ADD URR: URRNAME="urr_online_URL", URRID=2300, USAGERPTMODE=ONLINE,
  ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=100, ONLMETERINGTYPE=VOLUME;
```

#### 离线URR示例

```
ADD URR: URRNAME="urr_offline_URL", URRID=12500, USAGERPTMODE=OFFLINE,
  OFFCOMPOUNDTYPE=ONLYRG, RG=1100, OFFMETERINGTYPE=VOLUME;
```

#### 计费模式与统计类型

每个业务通常需要配置两条URR：
- 一条USAGERPTMODE=ONLINE，配ONLMETERINGTYPE
- 一条USAGERPTMODE=OFFLINE，配OFFMETERINGTYPE

> PGW-C/GGSN侧在线URR支持ONLMETERINGTYPE=EVENT_VOLUME_TIME（事件+流量+时长），SMF侧通常使用VOLUME（按流量计费）。

### URRGROUP配置规则

#### 关键参数

| 参数 | 说明 | 约束 |
|------|------|------|
| URRGROUPNAME | 使用量上报规则组名称 | 本端规划 |
| UPURRNAME1 | 上行发起URR名称1（通常是在线URR） | 已通过ADD URR配置 |
| DOWNURRNAME1 | 下行发起URR名称1（通常是在线URR） | 已通过ADD URR配置 |
| UPURRNAME2 | 上行发起URR名称2（通常是离线URR） | 已通过ADD URR配置 |
| DOWNURRNAME2 | 下行发起URR名称2（通常是离线URR） | 已通过ADD URR配置 |

#### 核心约束

1. **融合计费场景**：当SET USRPROFCHARGE、SET APNCHARGECTRL、ADD CHARGEMETHOD命令的"RGAPPLIED"配置为"DEFAULT"时，URRGROUP下**不能同时配置离线和在线URR**。

2. **纯在线/纯离线场景**：当"RGAPPLIED"配置为"ONLINERGONLY"或"OFFLINERGONLY"时，URRGROUP下**建议同时配置离线和在线URR**。

3. **UPURRNAME1/2和DOWNURRNAME1/2只是编号**，没有优先级语义。1和2用于区分不同计费模式的URR（如1=在线，2=离线）。

4. 上下行可以使用相同的URR，也可以使用不同的URR。

### PCCPOLICYGRP配置规则

| 参数 | 说明 | 约束 |
|------|------|------|
| PCCPOLICYGRPNM | PCC策略组名称 | 本端规划 |
| URRGROUPNAME | URR组名称 | 已通过ADD URRGROUP配置 |

作用：将URRGROUP绑定为PCC策略组，供RULE引用。

### RULE配置规则

| 参数 | 说明 | 约束 |
|------|------|------|
| RULENAME | 规则名称 | SMF和UPF上必须一致 |
| POLICYTYPE | 策略类型 | 固定取值PCC |
| PRIORITY | 全局优先级 | 数字越小优先级越高 |

| 参数 | 说明 | 约束 |
|------|------|------|

### SET URRGRPBINDING（缺省费率）

| 参数 | 说明 | 约束 |
|------|------|------|
| USERPROFILENAME | 用户模板名称 | 已通过ADD USERPROFILE配置 |
| DFTURRGRPNAME | 缺省URR组名称 | 已通过ADD URRGROUP配置 |

核心约束：
- **建议每个UserProfile都绑定缺省费率URR组**。
- 如果UserProfile下不绑定默认计费属性，用户的数据包按全局计费属性计费，否则不计费。

### RULEBINDING配置规则

| 参数 | 说明 | 约束 |
|------|------|------|
| USERPROFILENAME | 用户模板名称 | 已通过ADD USERPROFILE配置 |
| RULENAME | 规则名称 | 已通过ADD RULE配置 |

SMF、UPF上USERPROFILE下绑定的RULENAME需要保持一致，不一致策略无法按预期生效。

### USRPROFGROUP / UPBINDUPG（用户模板组绑定）

| 参数 | 说明 | 约束 |
|------|------|------|
| USERPROFGNAME | 用户模板组名称 | 本端规划 |
| UPBINDTYPE | 用户模板绑定类型 | SPECIFIC / DEFAULT |
| PRIORITY | 优先级 | 取值越小优先级越高 |
| USERPROFILENAME | 用户模板名称 | 已通过ADD USERPROFILE配置 |

### APN / APNUSRPROFG（DNN实例绑定）

| 参数 | 说明 | 约束 |
|------|------|------|
| APN | APN名称/DNN名称 | 本端规划 |
| USERPROFGNAME | 用户模板组名称 | 已通过ADD USRPROFGROUP配置 |

约束：每个DNN实例只能绑定一个UserProfile组。如UserProfile组和DNN实例已配置且已绑定，只需执行ADD UPBINDUPG命令将UserProfile绑定到UserProfile组即可。

### ADD QUOTAEXHAUSTACT（配额耗尽动作，可选）

当计费事件触发SMF发起Charging Data Request[Update]更新消息申请配额，Total Volume到达后仍未收到CHF响应消息，且SMF本地通过SET FAILHANDLING命令配置的TXTIMER超时时长未到达时，RG配额耗尽后的默认动作根据ACTION参数配置生效：

| ACTION取值 | 动作 | 说明 |
|------------|------|------|
| BLOCK | 阻塞业务 | SMF在N4接口下发该业务URR绑定的FAR动作为drop，在线RG配额耗尽后阻塞业务 |
| REDIRECT | 重定向 | SMF在N4接口下发该业务URR绑定的FAR动作为Redirect Information，将当前业务重定向到指定地址 |
| FORWARD | 继续转发 | SMF在N4接口下发该业务URR绑定的FAR动作为FORW，在线RG配额耗尽后允许继续转发业务 |

## 配置案例

### 典型场景：SMF侧内容计费（融合计费）

根据以下4个场景配置：

| 条件 | 计费类型 | RG | URR ID | Rule Name | User Profile |
|------|----------|-----|--------|-----------|-------------|
| 访问www.huawei.com | online | 100 | 2300 | Rule001_dns | userprofile001 |
| 访问www.huawei.com | offline | 1100 | 12500 | Rule001_dns | userprofile001 |
| 访问IMS业务 | online | 200 | 3300 | Rule002_IMS | userprofile002 |
| 访问IMS业务 | offline | 1200 | 13500 | Rule002_IMS | userprofile002 |
| any to any业务流 | online | 300 | 1300 | Rule003_any | userprofile001 |
| any to any业务流 | offline | 1300 | 11500 | Rule003_any_IPv6 | userprofile001 |
| 异常场景 | online | 400 | 4300 | - | - |
| 异常场景 | offline | 1400 | 14500 | - | - |

### 完整MML配置脚本

#### 步骤1：配置URR

```
// 访问www.huawei.com的在线/离线URR
ADD URR: URRNAME="urr_online_URL", URRID=2300, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=100, ONLMETERINGTYPE=VOLUME;
ADD URR: URRNAME="urr_offline_URL", URRID=12500, USAGERPTMODE=OFFLINE, OFFCOMPOUNDTYPE=ONLYRG, RG=1100, OFFMETERINGTYPE=VOLUME;

// 访问IMS业务的在线/离线URR
ADD URR: URRNAME="urr_online_IMS", URRID=3300, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=200, ONLMETERINGTYPE=VOLUME;
ADD URR: URRNAME="urr_offline_IMS", URRID=13500, USAGERPTMODE=OFFLINE, OFFCOMPOUNDTYPE=ONLYRG, RG=1200, OFFMETERINGTYPE=VOLUME;

// any to any业务流的在线/离线URR
ADD URR: URRNAME="urr_online_any", URRID=1300, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=300, ONLMETERINGTYPE=VOLUME;
ADD URR: URRNAME="urr_offline_any", URRID=11500, USAGERPTMODE=OFFLINE, OFFCOMPOUNDTYPE=ONLYRG, RG=1300, OFFMETERINGTYPE=VOLUME;

// 异常场景的在线/离线URR（免费）
ADD URR: URRNAME="URR_abnormal_online", URRID=4300, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=ONLYRG, ONLINERG=400, ONLMETERINGTYPE=FREE;
ADD URR: URRNAME="URR_abnormal_offline", URRID=14500, USAGERPTMODE=OFFLINE, OFFCOMPOUNDTYPE=ONLYRG, RG=1400, OFFMETERINGTYPE=FREE;
```

#### 步骤2：配置URRGROUP

```
ADD URRGROUP: URRGROUPNAME="UrrGp_URL", UPURRNAME1="urr_online_URL", DOWNURRNAME1="urr_online_URL", UPURRNAME2="urr_offline_URL", DOWNURRNAME2="urr_offline_URL";
```

### PGW-C/GGSN侧配置案例（在线计费）

PGW-C/GGSN侧使用在线计费时，配置步骤类似但有以下差异：

1. 使用ONLCOMPOUNDTYPE=RGSID（同时使用RG和SID）
2. URR参数中增加ONLINESID
3. ONLMETERINGTYPE支持EVENT_VOLUME_TIME

```
// 打开内容计费License开关
SET LICENSESWITCH: LICITEM="LKV3W9BCC12", SWITCH=ENABLE;

// 配置在线计费URR，费率标识(RG,SID)=(10,20)
ADD URR: URRNAME="onlineURR", URRID=1100, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=RGSID, ONLINERG=10, ONLINESID=20, ONLMETERINGTYPE=EVENT_VOLUME_TIME;

// 配置URRGROUP
ADD URRGROUP: URRGROUPNAME="urrgroup001", UPURRNAME1="onlineURR", DOWNURRNAME1="onlineURR";

// 配置PCC策略组和Rule
ADD PCCPOLICYGRP: PCCPOLICYGRPNM="cg-test", URRGROUPNAME="urrgroup001";
ADD RULE: RULENAME="Rule002", POLICYTYPE=PCC, POLICYNAME="cg-test", PRIORITY=120;

// 配置UserProfile和Rule绑定
ADD USERPROFILE:USERPROFILENAME="up-test", UPTYPE=PCC;
ADD RULEBINDING:USERPROFILENAME="up-test",RULENAME="Rule002";

// 配置UserProfile组和APN
ADD USRPROFGROUP:USERPROFGNAME="upg-test";
ADD UPBINDUPG:USERPROFGNAME="upg-test",UPBINDTYPE=DEFAULT,USERPROFILENAME="up-test";
ADD APN:APN="apn-test";
ADD APNUSRPROFG:APN="apn-test",USERPROFGNAME="upg-test";
```

### PGW-C/GGSN侧全局粒度Session级计费

PGW-C/GGSN还支持全局粒度配置，使用SET GLBURRGROUP命令：

```
ADD URR: URRNAME="onlineURR", URRID=1100, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=RGSID, ONLINERG=10, ONLINESID=20, ONLMETERINGTYPE=EVENT_VOLUME_TIME;
SET GLBURRGROUP:UPONLURRNAME="onlineURR", DNONLURRNAME="onlineURR";
```

> 全局粒度配置时，GGSN/PGW-C中的所有用户使用相同的费率标识进行用户级的在线计费。UserProfile粒度配置可以使不同APN的不同UserProfile下的用户使用不同的费率标识。

### PGW-C/GGSN侧UserProfile粒度Session级计费

不绑定Rule的UserProfile下的用户数据包不会创建五元组，不占用五元组资源，适合用户级计费（非内容计费）：

```
ADD URR: URRNAME="onlineURR", URRID=1100, USAGERPTMODE=ONLINE, ONLCOMPOUNDTYPE=RGSID, ONLINERG=10, ONLINESID=20, ONLMETERINGTYPE=EVENT_VOLUME_TIME;
ADD URRGROUP: URRGROUPNAME="urrgroup001", UPURRNAME1="onlineURR", DOWNURRNAME1="onlineURR";

ADD USERPROFILE:USERPROFILENAME="up-test", UPTYPE=PCC;
SET URRGRPBINDING:USERPROFILENAME="up-test",DFTURRGRPNAME="urrgroup001";

ADD USRPROFGROUP:USERPROFGNAME="upg-test";
ADD UPBINDUPG:USERPROFGNAME="upg-test",UPBINDTYPE=DEFAULT,USERPROFILENAME="up-test";
ADD APN:APN="apn-test";
ADD APNUSRPROFG:APN="apn-test",USERPROFGNAME="upg-test";
```

## 验证与调测

### 对系统的影响

本特性对系统无影响（UNC侧）。

### 应用限制

对加密协议报文，例如SSL（Security Socket Layer）、FTPS报文等，不能应用内容计费特性。

### License配置开关

内容计费费率标识的配置不依赖于内容计费License，但内容计费费率标识需在打开内容计费License后生效。仅在激活内容计费时，需要执行SET LICENSESWITCH命令。
