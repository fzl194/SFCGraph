# 计费场景新三层图谱实施计划

> 输出目录：`business-graph/计费场景/新三层图谱`
> 约束：只新增文件，不修改已有文件。
> 依据：`三层图谱本体标准定义.md`、`三层图谱Schema-最终版-v0.1.md`、`计费案例核心概念标准定义.md`，以及 `business-graph/计费场景` 下既有材料。

---

## 1. 工作目标

在计费场景下，按照根目录新 schema 产出三份独立文档：

1. 业务图谱
2. 特性图谱
3. 命令图谱

三份文档都必须满足：

- 对象类型严格服从根目录定义
- 关系类型严格服从根目录定义
- 属性命名尽量复用根目录 schema 里的定义
- 内容优先复用 `business-graph/计费场景` 现有沉淀，不重新发明概念

---

## 2. 串行拆分

### Phase 1：业务图谱

目标：先完成 `BusinessDomain / NetworkScenario / ConfigurationSolution / BusinessRule / DecisionPoint / SemanticObject` 的新 schema 版本。

子步骤：

1. 对齐旧业务图谱到新 schema 的映射
2. 抽取业务层对象和属性
3. 抽取业务层关系
4. 标记对 `Feature` 和 `ConfigTask` 的外部引用
5. 形成第一版业务图谱文档

### Phase 2：特性图谱

目标：在业务图谱稳定后，完成 `Feature / SubFeature / FeatureRule / License / ConfigTask` 的新 schema 版本。

子步骤：

1. 整理正式特性集合
2. 判断 `SubFeature`
3. 抽取 `ConfigTask`
4. 抽取特性依赖、License、规则
5. 形成第一版特性图谱文档

### Phase 3：命令图谱

目标：在特性图谱稳定后，完成 `MMLCommand / CommandParameter / ConfigObject / CommandRule` 的新 schema 版本。

子步骤：

1. 收敛命令集合
2. 收敛参数集合
3. 收敛配置对象集合
4. 抽取命令级规则与对象关系
5. 形成第一版命令图谱文档

## 4. 旧模型到新模型的映射原则

本轮业务图谱先执行以下映射：

| 旧对象 | 新对象 | 处理原则 |
|---|---|---|
| `DeliverySolution` | `ConfigurationSolution` | 直接改名，保留原实例含义 |
| `EngineeringTask` | `ConfigTask` | 不在业务图谱内部展开定义，只作为外部引用 |
| `DomainSemanticObject` | `SemanticObject` | 直接改名 |
| `ConfigObject` | 命令图谱对象 | 从业务图谱中剥离，仅在后续命令图谱展开 |
| `Scope` | 子对象 | 不再作为一级实体 |
| `Participant` | 子对象 | 不再作为一级实体 |
| `ValidationPlan` | 不入主本体 | 当前业务图谱不单独保留 |
