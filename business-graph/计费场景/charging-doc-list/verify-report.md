## 4. Gx接口 (待确认~25篇)

60个命中信元中，约25个是核心计费信元：
- Charging-Rule-Definition/Install/Name/Base-Name → 计费规则定义/安装/名称
- Charging-Characteristics/Address/Identifier → 计费特征/地址/标识
- Bearer-Control-Mode/Usage → 承载使用量
- Usage-Monitoring-Information/Report → 用量监控
- Quota-Management → 配额管理

其余35个命中的Rule信元(ADC-Rule等)和通用信元属于策略控制，非纯计费。

## 5. PFCP (N4+Sx) (待确认~30篇)

脚本解析batch-05格式有误导致显示已收录=0。实际PFCP有更多Usage Report/URR相关信元未收录。

## 6. 初始配置 (追加~3篇)

93个命中中约90个为误匹配：
- "CC"命中Country Code → MCC/MNC相关配置
- "策略"命中非计费策略(AMF发现策略等)
- "PCC"命中"智能PCC解决方案"选项
- "PCF"命中NF发现配置

真正计费相关的仅~3个SMF配置文件。
